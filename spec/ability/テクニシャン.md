# 仕様書: テクニシャン

参照URL: https://wiki.pokemonwiki.com/wiki/%E3%83%86%E3%82%AF%E3%83%8B%E3%82%B7%E3%83%A3%E3%83%B3
調査日: 2026-04-30

## 2. 基本効果（1段落）
元威力が60以下の攻撃技に対し、威力補正レイヤーで1.5倍を適用する。

## 3. 詳細仕様

### 3.1 適用条件
- イベント: `Event.ON_CALC_POWER_MODIFIER`
- 判定対象: `ctx.move.data.power`
- 発動条件:
	- `ctx.move.data.power is not None`
	- `ctx.move.data.power <= 60`
- 効果: 補正値 `value` に対して `value * 6144 // 4096` を返す（1.5倍）。

### 3.2 非発動条件
- `ctx.move.data.power is None`（元威力未定義の技）
- `ctx.move.data.power >= 61`

### 3.3 補正レイヤー上の位置づけ
- テクニシャンは「威力補正（Power Modifier）」として扱う。
- 攻撃補正（Attack Modifier）やダメージ補正（Damage Modifier）では扱わない。
- `spec/damage.md` の威力補正テーブルの通り、倍率 1.5（6144）を使用する。

### 3.4 他補正との関係
- 判定は「元威力」で行うため、他の補正で最終威力が60を超えても発動したまま。
- 逆に、他補正で最終威力が60以下になっても、元威力が61以上なら発動しない。

### 3.5 現実装での明確化
- 連続技は各ヒット共通の `data.power` で判定する。
- 可変威力技のうち `data.power=None` のものは非発動として扱う。
- 可変威力技の個別例外（世代差を含む厳密再現）は、必要時に技側ハンドラで追加対応する。

## 4. 処理順・優先度
- 発動タイミング: `DamageCalculator.calc_final_power()` 内の威力補正フェーズ。
- 処理順: `ON_CALC_POWER_MODIFIER` 参加要素（特性・持ち物・技・場）と同一レイヤーで合成。
- 係数表現: 固定小数点（基準 4096）で管理し、テクニシャンは 6144 を返す。

## 5. テスト項目（実装検証用）
- 発動ケース:
	- 威力60技（例: でんこうせっか）で `ON_CALC_POWER_MODIFIER == 6144`。
- 非発動ケース:
	- 威力61以上の技（例: １０まんボルト）で `ON_CALC_POWER_MODIFIER == 4096`。
- 境界ケース:
	- `data.power=None` の技で非発動。
	- 連続技でヒットごとに判定がぶれないこと。
	- 他の威力補正と同時に掛かったときに固定小数点合成が破綻しないこと。
