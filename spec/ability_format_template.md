# 特性仕様書の統一形式

## セクション構成

### 1. 基本情報
```
# 仕様書: [特性名]

参照URL: [Wiki URL]
```

### 2. 基本効果（1段落）
機能をシンプルに、実装に直結する表現で記述。

### 3. 詳細仕様
事象や条件ごとにサブセクションを分ける：
- **フォルムチェンジ関連** → フォルムチェンジの1つであり、タイプ・ステータスが...
- **無効化・上書き** → かがくへんかガスで無効化できる/できない、特性を変える技の効果...
- **交代・入場時** → 交代すると解除される、場に出たときの発動タイミング...
- **その他の特性との相互作用** → 具体的な特性名と相互作用内容

### 4. 処理順・優先度
- 発動タイミング明記（ターン開始時、終了時など）
- 同時発動時の優先度
- すばやさ補正の考慮有無

### 5. テスト項目（実装検証用）
実装を検証するための具体的なテストケース：
- 発動ケース
- 非発動ケース  
- 例：「フォルムチェンジ発動後、タイプが正しく変更される」

## 削除対象
- アニメでの使用例
- 見た目描写（演出など）
- 実装と無関係な世代別差異
- 余談や背景情報

## 保持対象
- Game仕様の詳細（ランク変化、優先度、条件）
- 他の特性との相互作用
- 交代・入場時の挙動
- フォルムチェンジの詳細（タイプ・ステータス変更）
- 無効化・上書きの判定
- コピー・交換可否
