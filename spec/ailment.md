# 状態異常（Ailment）仕様書（第9世代準拠）

更新日: 2026-04-30
対象: シングルバトル実装（`src/jpoke`）

## 1. 対象状態異常

本リポジトリで扱う状態異常は以下の6種。

- どく
- もうどく
- まひ
- やけど
- ねむり
- こおり

## 2. 共通ルール

### 2.1 排他

- 同時に保持できる状態異常は1つのみ。
- 既に状態異常を持つ場合、新規付与は失敗する（強制上書きオプションを除く）。

### 2.2 付与前の無効化判定

- `Event.ON_BEFORE_APPLY_AILMENT` を通して、特性・場状態・地形による無効化を判定する。
- このため、`しんぴのまもり`、`ミストフィールド`、`はれ`中のこおり無効などを統一的に扱う。

### 2.3 付与後処理

- 付与時に状態異常ハンドラを登録し、解除時に解除する。
- HP変更は `battle.modify_hp` 経由で行う。

## 3. 個別仕様

## 3.1 どく

- ターン終了時に最大HPの1/8ダメージ（小数点以下切り捨て、最小1）。
- どくタイプ・はがねタイプには通常無効。
- 例外として、付与元が特性`ふしょく`かつ有効時は、どく・はがねにも付与可能。

## 3.2 もうどく

- ターン終了時に `n/16` ダメージ（`n` は経過回数、1始まり）。
- `n` は毎ターン増加し、ダメージ上限は `15/16`。
- 交代時は経過をリセットし、再登場時は `1/16` から再開。
- どくタイプ・はがねタイプには通常無効（`ふしょく`例外あり）。

## 3.3 まひ

- 行動前に25%で行動不能。
- すばやさは1/2。
- でんきタイプには付与不可。

## 3.4 やけど

- ターン終了時に最大HPの1/16ダメージ（小数点以下切り捨て、最小1）。
- 物理技ダメージ補正は0.5倍。
- ほのおタイプには付与不可。

## 3.5 ねむり

- 行動前にカウントを消費し、カウントが0で解除。
- カウントが残る間は原則行動不能。
- 例外として `いびき` と `ねごと` はねむり中でも行動可能。
- 本実装での通常付与カウントは `2〜4` 行動（`ねむる`は個別仕様側で固定値を扱う）。

## 3.6 こおり

- 行動前に20%で自然回復（回復時はそのターン行動可）。
- 回復しなければ行動不能。
- こおりタイプには付与不可。
- ほのおタイプの攻撃技でダメージを受けた場合、こおりは解除される。
- `はれ` 中は新規こおり付与不可（既存のこおりは自動解除しない）。

### こおり中でも使用時に自身のこおりを解除する技

- かえんぐるま
- かえんボール
- クロスフレイム
- シャカシャカほう
- スチームバースト
- せいなるほのお
- ねっさのだいち
- ねっとう
- ハイドロスチーム
- フレアドライブ
- めらめらバーン
- もえつきる

実装メモ:
これらの「使用時に自身のこおりを解除する」挙動は、状態異常ハンドラではなく技個別効果として扱い、`handlers/move.py` 側の技ハンドラに実装する。
