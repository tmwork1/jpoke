## ハンドラ一覧

### 1. はれ_power_modifier（威力補正）— はれと共有

| 項目 | 値 |
|------|----|
| イベント | Event.ON_CALC_POWER_MODIFIER |
| subject_spec | `attacker:self`（AttackContext: 攻撃フェーズ） |
| priority | 100（デフォルト） |

**ロジック:**
1. `battle.weather_for(ctx.defender).name == ""` → 防御側がばんのうがさを持つ場合は無効 → 早期リターン
2. `ctx.move.type == "ほのお"` → `apply_fixed_modifier(value, 6144)`（×1.5）
3. `ctx.move.type == "みず"` → `apply_fixed_modifier(value, 2048)`（×0.5）

### 2. はれ_prevent_freeze（こおり防止）— はれと共有

| 項目 | 値 |
|------|----|
| イベント | Event.ON_BEFORE_APPLY_AILMENT |
| subject_spec | `target:self`（EventContext: 非攻撃フェーズ） |
| priority | 100（デフォルト） |

**ロジック:**
1. `value == "こおり"` → `value = ""` で無効化

### 3. おおひでり_block_move（みず技ブロック）— おおひでり固有

| 項目 | 値 |
|------|----|
| イベント | Event.ON_TRY_MOVE_1 |
| subject_spec | `attacker:self`（AttackContext: 攻撃フェーズ） |
| priority | 10 |
| 根拠 | .internal/spec/turn.md: 「おおあめ・おおひでりによる技失敗」Priority 10 |

**ロジック:**
1. `ctx.move.type == "みず"` → `battle.add_event_log(ctx.attacker, LogCode.MOVE_FAILED, payload={"reason": "おおひでり"})`
2. `HandlerReturn(value=False, stop_event=True)` を返す

**注意:**
- 攻撃技・変化技を問わずブロックする
- ON_TURN_END ハンドラは不要（カウントダウンなし）

## 天候優先度

`data/field.py` の `WEATHER_PRIORITY` に登録済み:

```python
WEATHER_PRIORITY = {
    "おおひでり": 1,
    "らんきりゅう": 2,
    ...
}
```

`WeatherManager.apply` は `new_priority >= current_priority` のときのみ上書きを許可する。
