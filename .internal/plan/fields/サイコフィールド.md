## ハンドラ一覧

### 1. サイコフィールド_power_modifier（エスパー技威力補正）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_CALC_POWER_MODIFIER |
| subject_spec | `attacker:self`（AttackContext: 攻撃フェーズ） |
| priority | 100（デフォルト） |
| 根拠 | 他のフィールドの威力補正ハンドラと同じデフォルト値 |

**ロジック:**
1. `ctx.move.type == "エスパー"` かつ `not battle.query.is_floating(ctx.attacker)` → `apply_fixed_modifier(value, 5325)`（×1.3）

### 2. サイコフィールド_block_priority_move（先制技無効化）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_TRY_MOVE_1 |
| subject_spec | `defender:self`（AttackContext: 攻撃フェーズ） |
| priority | 100 |
| 根拠 | .internal/spec/turn.md: 「サイコフィールドによる無効化」Priority 100 |

**ロジック:**
1. `ctx.move.priority > 0` かつ `not battle.query.is_floating(ctx.defender)` → `MOVE_FAILED` ログ + `HandlerReturn(value=False, stop_event=True)`

### 3. tick_terrain（カウントダウン）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_TURN_END |
| subject_spec | `source:self`（EventContext: 非攻撃フェーズ） |
| priority | 140 |
| 根拠 | .internal/spec/turn.md: 「サイコフィールドの継続・終了」Priority 140 |

**ロジック:** 1P側のみ `battle.terrain_manager.tick_down()` を呼ぶ（共通関数 `tick_terrain` を使用）。

## データ登録（data/field/terrain.py）

```python
"サイコフィールド": FieldData(
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.FieldHandler(
            h.サイコフィールド_power_modifier,
            subject_spec="attacker:self",
        ),
        Event.ON_TRY_MOVE_1: h.FieldHandler(
            h.サイコフィールド_block_priority_move,
            priority=100,
            subject_spec="defender:self",
        ),
        Event.ON_TURN_END: h.FieldHandler(
            h.tick_terrain,
            subject_spec="source:self",
            priority=140,
        ),
    },
),
```
