# まきびし 実装計画

## 概要

相手陣サイドフィールド。交代で場に出た地面にいるポケモンに層数依存のダメージを与える。
最大3層まで重ね掛け可能。

## 技データ（data/move.py）

```python
"まきびし": MoveData(
    type="じめん",
    category="status",
    pp=20,
    target="foe_side",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.まきびし_set_field,
        ),
    }
)
```

## フィールドデータ（data/field/side_field.py）

```python
"まきびし": FieldData(
    max_count=3,
    handlers={
        Event.ON_SWITCH_IN: h.FieldHandler(
            h.まきびし_damage,
            subject_spec="source:self",
        ),
    },
)
```

## ハンドラ実装

### まきびし_set_field（handlers/move_status.py）

- `battle.get_side(ctx.defender).activate("まきびし", 1)` で1層設置
- すでに3層（max_count=3）の場合は `activate` が `False` を返すため失敗

### まきびし_damage（handlers/field.py）

```python
def まきびし_damage(battle, ctx, value):
    # あつぞこブーツ・マジックガード等の設置技免疫チェック
    if battle.query.is_hazard_immune(ctx.source):
        return HandlerReturn(value=value)
    # ひこうタイプ・ふゆう等の浮遊チェック
    if battle.query.is_floating(ctx.source):
        return HandlerReturn(value=value)

    field = battle.get_side(ctx.source).get("まきびし")
    damage_ratio = {
        1: -1/8,
        2: -1/6,
    }.get(field.count, -1/4)  # 3層以上は1/4

    success = battle.modify_hp(ctx.source, r=damage_ratio)
    return HandlerReturn(value=success)
```

## 優先度

| イベント | priority | 根拠 |
|----------|----------|------|
| ON_SWITCH_IN（ダメージ） | 100（デフォルト） | 既存の ステルスロック・どくびし と同じ |

## 注意点

- `modify_hp(r=...)` は `min(-1, int(max_hp * r))` で最低1ダメージを保証する
- `is_floating` はひこうタイプを初期値として ON_CHECK_FLOATING イベントで判定
- `is_hazard_immune` は ON_CHECK_HAZARD_IMMUNE イベントで判定
