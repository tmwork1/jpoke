## ハンドラ一覧

### 1. はれ_power_modifier（威力補正）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_CALC_POWER_MODIFIER |
| subject_spec | `attacker:self` （AttackContext: 攻撃フェーズ） |
| priority | 100（デフォルト） |

**ロジック:**
1. `battle.weather_for(ctx.defender).name == ""` → 防御側がばんのうがさを持つ場合は天候が無効扱い → 補正なしで早期リターン
2. `ctx.move.type == "ほのお"` → `apply_fixed_modifier(value, 6144)`（×1.5）
3. `ctx.move.type == "みず"` → `apply_fixed_modifier(value, 2048)`（×0.5）

**注意:** 攻撃フェーズなので subject_spec は `attacker:self`。ばんのうがさチェックは防御側（`ctx.defender`）を参照する。

### 2. はれ_prevent_freeze（こおり防止）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_BEFORE_APPLY_AILMENT |
| subject_spec | `target:self` （EventContext: 非攻撃フェーズ） |
| priority | 100（デフォルト） |

**ロジック:**
1. `value == "こおり"` → `value = ""` で無効化

### 3. tick_weather（カウントダウン）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_TURN_END |
| subject_spec | `source:self` （EventContext: 非攻撃フェーズ） |
| priority | 10 |
| 根拠 | .internal/spec/turn.md: 「にほんばれ等の天気終了」Priority 10 |

**ロジック:** 1P側のみ `battle.weather_manager.tick_down()` を呼ぶ（共通関数 `tick_weather` を使用）。

## おおひでりとの共通化

`おおひでり` も `はれ_power_modifier` と `はれ_prevent_freeze` を共有して使用する（FieldData の handlers に同じ関数を登録）。
