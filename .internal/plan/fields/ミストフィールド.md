## ハンドラ一覧

### 1. ミストフィールド_power_modifier（ドラゴン技威力補正）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_CALC_POWER_MODIFIER |
| subject_spec | `defender:self`（AttackContext: 攻撃フェーズ。防御側ポケモンが接地しているかチェック） |
| priority | 100（デフォルト） |

**ロジック:**
1. `ctx.move.type == "ドラゴン"` かつ `not battle.query.is_floating(ctx.defender)` → `apply_fixed_modifier(value, 2048)`（×0.5）

### 2. ミストフィールド_prevent_ailment（状態異常防止）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_BEFORE_APPLY_AILMENT |
| subject_spec | `target:self`（EventContext: 非攻撃フェーズ） |
| priority | 100（デフォルト） |

**ロジック:**
1. `not battle.query.is_floating(ctx.target)` → `value = ""` で無効化、`stop_event=True`
   - （エレキフィールドと異なり、ねむり以外の全状態異常もブロックする）

### 3. ミストフィールド_prevent_confusion（こんらん防止）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_BEFORE_APPLY_VOLATILE |
| subject_spec | `target:self`（EventContext: 非攻撃フェーズ） |
| priority | 100（デフォルト） |

**ロジック:**
1. `value == "こんらん"` かつ `not battle.query.is_floating(ctx.target)` → `value = ""` で無効化、`stop_event=True`
   - （ねむけは防がない。ねむけ→ねむりの移行はねむりがブロックされることで防ぐ）

### 4. tick_terrain（カウントダウン）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_TURN_END |
| subject_spec | `source:self`（EventContext: 非攻撃フェーズ） |
| priority | 140 |
| 根拠 | .internal/spec/turn.md: 「ミストフィールドの継続・終了」Priority 140 |

**ロジック:** 1P側のみ `battle.terrain_manager.tick_down()` を呼ぶ（共通関数 `tick_terrain` を使用）。

## ねむる失敗チェック

`src/jpoke/handlers/move_status.py` の `ねむる_check` に以下のチェックを追加する:

1. `battle.terrain.name == "ミストフィールド"` かつ `not battle.query.is_floating(ctx.attacker)` → 失敗ログ + `HandlerReturn(value=False, stop_event=True)`

**根拠:** wiki「効果を受けているポケモンのねむるは失敗する。」  
HP 回復や状態異常解除をする前の ON_TRY_MOVE_1 段階で失敗させることで、ねむるの副作用を防ぐ。
