# しんぴのまもり 実装計画書

## 概要

サイドフィールド「しんぴのまもり」のハンドラ実装。
5ターンの間、他のポケモンからのわざによる状態異常・こんらん・ねむけを防ぐ。

## 参照仕様書

`.internal/spec/fields/しんぴのまもり.md`

## 実装するハンドラ

### 1. しんぴのまもり_prevent_ailment

| 項目 | 値 |
|---|---|
| イベント | `ON_BEFORE_APPLY_AILMENT` |
| subject_spec | `target:self` |
| priority | 100 (default) |

**ロジック**:
- `ctx.is_foe_target()` が True（自分自身への状態異常付与でない）
- かつ `ctx.can_bypass_status_guard(battle)` が False（すりぬけ等でない）
- の場合のみ `value = ""` にして無効化

**注意**: `is_foe_target()` チェックが必要。自分が使うねむる（source==target）は防がない。

### 2. しんぴのまもり_prevent_confusion

| 項目 | 値 |
|---|---|
| イベント | `ON_BEFORE_APPLY_VOLATILE` |
| subject_spec | `target:self` |
| priority | 100 (default) |

**ロジック**:
- `value in ["こんらん", "ねむけ"]`（こんらん・ねむけのみ対象）
- かつ `ctx.is_foe_target()` が True
- かつ `ctx.can_bypass_status_guard(battle)` が False
- の場合のみ `value = ""` にして無効化

### 3. しんぴのまもり_tick_side_field

| 項目 | 値 |
|---|---|
| イベント | `ON_TURN_END` |
| subject_spec | `source:self` |
| priority | **130**（spec/turn.md で確認）|

**ロジック**: `tick_side_field(battle, ctx, value, name="しんぴのまもり")` を呼ぶ。

## data/ への登録

`data/field/side_field.py` の `SIDE_FIELD["しんぴのまもり"]` に3つのハンドラを登録する。

## テスト計画

- 状態異常を防ぐこと
- こんらんを防ぐこと
- ねむけを防ぐこと
- ねむる（自分自身に使う）は防がないこと
- すりぬけ特性で貫通できること（abilities テストで確認）
- ターンカウントダウンが正しく動作すること（test_サイドフィールドカウント減少 で確認）
