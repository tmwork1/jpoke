# かえんのまもり 実装計画書

## 概要

かえんのまもり揮発性状態の実装。
使用したターンに受ける攻撃技を無効化し、直接攻撃をしてきた相手をやけど状態にする。
ターン終了時に解除される。

## 実装対象

### 1. かえんのまもり_protect（Event.ON_TRY_MOVE_1）

**ファイル**: `handlers/volatile.py`  
**登録**: `data/volatile.py`

- `_run_protect(battle, ctx, value, ailment_on_contact="やけど", protect_non_attack=False)` を呼び出す
  - `protect_non_attack=False` により変化技を防がない
  - `ailment_on_contact="やけど"` により直接攻撃者にやけどを付与する
  - `battle.query.is_contact(ctx)` が True の場合のみやけどを付与（ぼうごパット持ちは False になる）
  - `ON_CHECK_PROTECT` イベントで守りが無視される場合（ふかしのこぶし等）は無効化もやけども行わない
- subject_spec: `"defender:self"` （AttackContext）
- priority: 100
  - 根拠: `.internal/spec/turn.md` の `Event.ON_TRY_MOVE_1` より「まもる・キングシールド等による無効化」は Priority 100

### 2. かえんのまもり_remove_volatile（Event.ON_TURN_END）

**ファイル**: `handlers/volatile.py`  
**登録**: `data/volatile.py`

- ターン終了時にかえんのまもり状態を解除する
- subject_spec: `"source:self"` （EventContext）
- priority: 未指定（デフォルト）
  - 根拠: turn.md に ON_TURN_END でのまもる系解除の明示的な priority 記載なし。他のまもる系（まもる・キングシールド等）と同様に priority 指定なし

### 3. かえんのまもり_apply_volatile_to_attacker（Event.ON_STATUS_HIT）

**ファイル**: `handlers/move_status.py`  
**登録**: `data/move.py`

- 技使用時（自分への変化技として成功）、自分にかえんのまもり状態（count=1）を付与する
- subject_spec: 未指定（デフォルト `"attacker:self"`）
  - 根拠: MoveHandler のデフォルト subject_spec = `"attacker:self"`

## テスト方針

- 技を使うとかえんのまもり状態が付与されることを確認する
- 接触技を受けると攻撃者がやけどになることを確認する
- 非接触攻撃技を受けてもやけどにならないことを確認する
- 変化技は防がれないことを確認する（まもる系共通テストで検証済み）
- ターン終了でかえんのまもり状態が解除されることを確認する（まもる系共通テストで検証済み）

## 注意点

- `data/volatile.py` の登録順は五十音順に従う
- 交代による解除は `switch_manager._switch_out` の `remove_all_volatiles` で自動的に処理される
- まもる系共通の攻撃技無効化・変化技への反応・ターン終了解除はパラメトリックテストで一括検証
