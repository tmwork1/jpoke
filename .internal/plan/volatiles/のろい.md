# のろい (状態変化) 実装計画書

## 仕様サマリ

- ターン終了時（Priority 100）に最大HPの1/4ダメージを受ける。
- 交代で解除される。バトンタッチで引き継がれる。
- マジックガード特性があるとダメージを受けない（`ON_MODIFY_NON_MOVE_DAMAGE` 経由で自動機能）。

## 実装対象イベント

| イベント | subject_spec | priority | 実装関数 | 根拠 |
|---------|-------------|----------|--------|------|
| `ON_TURN_END` | `source:self` | 100 | `のろい_damage` | `.internal/spec/turn.md` Priority 100 のろいダメージ行 |

## ハンドラ実装 (`handlers/volatile.py`)

```python
def のろい_damage(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """のろい状態のターン終了時ダメージ"""
    battle.modify_hp(ctx.source, r=-1/4)
    return HandlerReturn(value=value)
```

- `r=-1/4` で最大HPの1/4ダメージ（切り捨て、最小1は `modify_hp` 内で保証）。
- `Pokemon.hp` への直接代入は禁止。必ず `battle.modify_hp(...)` を使う。

## データ登録 (`data/volatile.py`)

```python
"のろい": VolatileData(
    handlers={
        Event.ON_TURN_END: h.VolatileHandler(
            h.のろい_damage,
            subject_spec="source:self",
            priority=100,
        ),
    }
),
```

## 配置順（五十音順）

`handlers/volatile.py` および `data/volatile.py` において、のろいは「ねをはる」の後、「バインド」の前に配置する。
