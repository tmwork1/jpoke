# みがわり 実装計画書

## 仕様要約

みがわりは揮発性状態として実装される。攻撃を代わりに受けるみがわりHPを保持し、HPが0になると消滅する。

## イベントと priority

### ON_BEFORE_APPLY_MOVE（変化技の無効化）

| event | subject_spec | priority | 根拠 |
|-------|-------------|----------|------|
| ON_BEFORE_APPLY_MOVE | defender:self | 30 | turn.md: ON_APPLY_MOVE priority 30「みがわりによる無効化」 |

処理: 変化技かつ `check_hit_substitute` が True の場合、MOVE_IMMUNEDログを出してstop_event=True

### ON_MODIFY_MOVE_DAMAGE（ダメージの肩代わり）

| event | subject_spec | priority | 根拠 |
|-------|-------------|----------|------|
| ON_MODIFY_MOVE_DAMAGE | defender:self | 20 | turn.md: ON_MODIFY_DAMAGE priority 20「みがわりに被弾」 |

処理:
1. `check_hit_substitute` が False なら何もしない
2. SUBSTITUTE_HITログを出力
3. ダメージをみがわりHPから削る（min(volatile.hp, damage)）
4. `ctx.substitute_damage` に肩代わりダメージを保存（後続処理用）
5. みがわりHPが0になったら消滅
6. 本体へのダメージは0を返す

## check_hit_substitute の判定ロジック（move_executor.py）

以下の条件のいずれかに該当すればみがわりを貫通する（Falseを返す）：
1. `ctx.move.target != "foe"` → 自分対象・全体・フィールドなど
2. `ctx.move.has_label("bypass_substitute")` → スキルスワップ等
3. `ctx.move.has_label("sound")` → 音技（第六世代以降の仕様）
4. ON_CHECK_HIT_SUBSTITUTE イベントで False が返る → すりぬけ特性など

## VolatileData の handlers

```python
"みがわり": VolatileData(
    handlers={
        Event.ON_BEFORE_APPLY_MOVE: VolatileHandler(
            みがわり_immune,
            subject_spec="defender:self",
            priority=30,
        ),
        Event.ON_MODIFY_MOVE_DAMAGE: VolatileHandler(
            みがわり_block_damage,
            subject_spec="defender:self",
            priority=20,
        ),
    }
),
```

## handlers/volatile.py の関数

### みがわり_immune
- `check_hit_substitute` を呼び出す
- True かつ変化技なら MOVE_IMMUNED ログ + stop_event=True
- そうでなければ True を返す

### みがわり_block_damage
- `check_hit_substitute` が False なら元のダメージを返す
- SUBSTITUTE_HIT ログを出力
- damage = min(volatile.hp, damage) でみがわりHPを削る
- `ctx.substitute_damage` に保存
- volatile.hp == 0 なら volatile_manager.remove
- 本体ダメージとして 0 を返す
