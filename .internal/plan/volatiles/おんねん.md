# おんねん 実装計画書

## 概要

おんねん揮発性状態の実装。
攻撃技でひんしにされたとき、その技の PP を 0 にする。

## 実装対象

### 1. おんねん_deplete_attacking_move_pp（Event.ON_MOVE_KO）

**ファイル**: `handlers/volatile.py`  
**登録**: `data/volatile.py`

- おんねん状態のポケモンが攻撃技でひんしになったとき、その技の PP を 0 にする
- 発動しない条件:
  - `ctx.move.pp == 0`（すでに PP が 0）
  - `ctx.move.has_label("non_onnen")`（わるあがき等）
- subject_spec: `"defender:self"` （AttackContext — おんねん持ちが defender）
- priority: 10
  - 根拠: `.internal/spec/turn.md` の `Event.ON_DAMAGE`（ON_MOVE_KO）より「防御側のおんねんによるPP消失」は Priority 10

### 2. おんねん_remove_volatile（Event.ON_TRY_ACTION）

**ファイル**: `handlers/volatile.py`  
**登録**: `data/volatile.py`

- おんねん状態のポケモンが行動しようとするとき（次の自身の行動順）に状態を解除する
- subject_spec: `"attacker:self"` （AttackContext — おんねん持ちが attacker）
- priority: 10
  - 根拠: `.internal/spec/turn.md` の `Event.ON_TRY_ACTION` より「自身のおんねん状態の解除」は Priority 10

## わるあがき除外の実装方針

`utils/type_defs.py` に `"non_onnen"` ラベルを追加し、
`data/move.py` の わるあがき の labels に `"non_onnen"` を追加する。

## テスト方針

- 基本動作: 攻撃技でひんしになると PP が 0 になる
- 非発動: ひんしにならないと PP が 0 にならない
- 非発動: PP が既に 0 の技でひんしになっても再発動しない
- 非発動: わるあがきでひんしになっても PP を消費しない
- 解除: 次のターンに行動するとおんねん状態が解除される

## 注意点

- `ctx.move.pp = 0` は Move オブジェクトを直接変更する（技データの共有問題に注意）
  - 各 Pokemon が個別に Move インスタンスを持つ設計なら問題なし
- 交代による解除は `switch_manager._switch_out` の `remove_all_volatiles` で自動処理される
