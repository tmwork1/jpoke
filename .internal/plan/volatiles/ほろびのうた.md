# ほろびのうた (状態変化) 実装計画書

## 仕様サマリ

- ターン終了時（Priority 120）にカウントを1減らす。
- カウントが0になった時（ON_VOLATILE_END）にひんしにする。
- 交代で解除される。バトンタッチで引き継がれる。
- マジックガードでも防げない（`reason="perish"` を使用）。
- きあいのタスキ等で耐えることができない。

## 実装対象イベント

| イベント | subject_spec | priority | 実装関数 | 根拠 |
|---------|-------------|----------|--------|------|
| `ON_TURN_END` | `source:self` | 120 | `ほろびのうた_tick_volatile` | `.internal/spec/turn.md` Priority 120 ほろびのうた行 |
| `ON_VOLATILE_END` | `source:self` | 100 | `ほろびのうた_faint` | ON_VOLATILE_END のデフォルト priority（既存の他ハンドラを参照） |

## ハンドラ実装 (`handlers/volatile.py`)

```python
def ほろびのうた_tick_volatile(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    return tick_volatile(battle, ctx, value, volatile="ほろびのうた")


def ほろびのうた_faint(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """ほろびのうたでひんしになる処理。マジックガードでも防げない。"""
    mon = ctx.source
    battle.modify_hp(mon, v=-mon.hp, reason="perish")
    return HandlerReturn(value=value)
```

- `reason="perish"` を指定することでマジックガードの無効化を回避する。
- `HPChangeReason` の `"perish"` は `utils/type_defs.py` に定義済み。
- `マジックガード_ignore_damage` の除外リストに `"perish"` が含まれている。
- `Pokemon.hp` への直接代入は禁止。必ず `battle.modify_hp(...)` を使う。

## データ登録 (`data/volatile.py`)

```python
"ほろびのうた": VolatileData(
    handlers={
        Event.ON_TURN_END: h.VolatileHandler(
            h.ほろびのうた_tick_volatile,
            subject_spec="source:self",
            priority=120
        ),
        Event.ON_VOLATILE_END: h.VolatileHandler(
            h.ほろびのうた_faint,
            subject_spec="source:self",
        )
    }
),
```
