# 特性実装計画: じゅくせい

更新日: 2026-07-11

## スコープ

- 仕様書: `.internal/spec/abilities/じゅくせい.md`
- 実装状態: 未実装
- 実装方針: じゅくせい専用のイベントハンドラは追加せず、既存のきのみ発動処理（`handlers/item.py` の
  共通ヘルパー・個別ハンドラ、および `なげつける` の個別処理）内で
  `mon.ability.name == "じゅくせい"`（効果を受け取る本人の特性）を判定し、効果量を2倍にする
  （`_gluttony_denominator` のくいしんぼうと同様のインライン判定パターン）。
- `なげつける`/`おちゃかい`/`むしくい`/`ついばむ`はいずれも最終的に対象ポケモンが
  「きのみを消費して効果を受ける」処理（`_heal_berry` 等の共通ヘルパー、または
  `なげつける_apply_item_effect` の個別分岐）を通るため、その時点の
  「効果を受け取るポケモン」の特性を見れば、経由した技を区別せず正しく判定できる
  （むしくい・ついばむで奪って自分が食べた場合は「自分」が効果を受け取る側になるため
  自然に対応できる）。

## 対象外（変更しない）

- 状態異常回復系（カゴ・クラボ・チーゴ・ナナシ・モモン・ラム・キーの各のみ）:
  効果量ではなく発動可否のみのため2倍化の余地がなく、そのままにする。
- 急所ランク上昇（サンのみ）・命中率補正（ミクルのみ）・行動順（イバンのみ）:
  一次情報で明示的に2倍化されないと記載されているため変更しない。
- 型半減以外の被弾時ランク上昇アイテム（きゅうこん・じゅうでんち・ひかりごけ・ゆきだま等、
  `_boost_stat_on_type_hit` 経由）: きのみではないため対象外（一次情報に明記）。
- `_cure_ailment_berry` / `_cure_ailment_berry_on_apply` / `_apply_ailment_from_item`:
  上記の状態異常回復・付与系のため変更なし。

## 実装ゴール（完了定義）

- じゅくせい所持時、HP/PP回復系きのみの回復量が2倍になる（端数切り捨て後に2倍。切り捨て前に
  2倍にしてから丸めるわけではない）。
- じゅくせい所持時、能力ランク上昇系きのみの上昇量が2倍になる（サンのみ・ミクルのみは対象外）。
- じゅくせい所持時、17種の半減系きのみ（タイプ相性半減・ホズのみ含む）は「1/2の被ダメージ」では
  なく「1/4の被ダメージ」になる（1/2の補正を2回掛けるのではなく、1回の乗算で1/4を算出する）。
- じゅくせい所持時、ジャポのみ/レンブのみは「最大HPの1/4（切り捨て）」の反動ダメージになる
  （「最大HPの1/8（切り捨て）を2倍」ではない）。
- じゅくせい所持時、ナゾのみは最大HPの1/4（切り捨て）の2倍の回復になる。
- なげつける・おちゃかい・むしくい/ついばむ経由でも同様に2倍化される。ほおばるも同様に2倍化
  されるが専用の告知は行わない（既存実装がアイテム告知イベントを個別に持たないため、現状の
  実装のままで一次情報の「特性バーが現れない」という見た目の違いは区別しない＝そもそも
  じゅくせい自体の特性発動ログを追加しないため矛盾しない）。
- 既存テスト（各きのみのテスト）を壊さない（じゅくせいを持たない場合は従来通り）。

## 実装対象イベント

- 既存の各きのみが登録済みのイベント（`Event.ON_HP_CHANGED` / `Event.ON_CALC_DAMAGE_MODIFIER` /
  `Event.ON_DAMAGE_HIT` / `Event.ON_HIT` 等）は変更しない。判定条件・効果量の計算のみを変更する。

## 変更対象（ファイル/関数単位）

- `src/jpoke/handlers/item.py`
  - `_is_ripen(mon)` を新規追加（`mon.ability.name == "じゅくせい"` の判定ヘルパー）。
  - `_berry_heal_amount(mon, *, r=None, v=None)` を新規追加。
    `r` 指定時は `max(1, int(mon.max_hp * r))` を算出してから、`v` 指定時はそのまま、
    じゅくせい所持時に2倍にして返す（`modify_hp` の `r` 引数を素通しせず、内部で先に
    端数を切り捨ててから2倍することで一次情報の丸め仕様を再現する）。
  - `_heal_berry` — 回復量の算出を `_berry_heal_amount` 経由に変更し、
    `battle.modify_hp(mon, v=...)` を呼ぶ（イア・ウイ・バンジ・フィラ・マゴ・オボン・オレンの
    各のみが対象）。
  - `_boost_on_quarter_hp` — `battle.modify_stats(mon, {stat: amount})` の `amount` を
    じゅくせい所持時に2倍にする（チイラ・リュガ・カムラ・ヤタピ・ズアの各のみが対象）。
  - `_boost_on_attack_category` — 同様に `amount` を2倍にする（アッキ・タラプのみが対象）。
  - `_modify_super_effective_damage` — じゅくせい所持時、`modifier` を1回の乗算で
    2乗した値（0.5→0.25）に置き換えてから `value = int(value * modifier)` を計算する
    （17種の半減系きのみ + ホズのみが対象。`_boost_stat_on_type_hit` はきのみ以外の
    アイテム専用ヘルパーのため変更しない）。
  - `_retaliate_on_category` — じゅくせい所持時、反動ダメージの割合を `-1/8` ではなく
    `-1/4` に直接置き換える（ジャポのみ・レンブのみが対象。`-1/8` を2倍にするのではなく、
    1回の乗算で`-1/4` を算出する）。
  - `ナゾのみ_heal_on_super_effective` — `_berry_heal_amount(mon, r=1/4)` 経由に変更する。
  - `ヒメリのみ_restore_pp` / `ヒメリのみ_restore_pp_on_item_enabled` — PP回復上限を
    `_is_ripen` 判定で `10` → `20` に変更する。
  - `なげつける_apply_item_effect` — 対象ポケモン（`ctx.defender`）がじゅくせい所持のとき、
    HP回復・PP回復・能力ランク上昇の各分岐（ヒメリ・オレン・オボン・フィラ/ウイ/マゴ/バンジ/
    イア・チイラ/リュガ・アッキ/カムラ/ヤタピ/ズア・タラプ）の効果量を2倍にする
    （サンのみ・スターのみ・ミクルのみは仕様通り対象外/別ロジックのため要注意。
    スターのみは能力ランク上昇系きのみのため対象、`+2` を `+4` にする）。
- `src/jpoke/data/ability.py` — `じゅくせい` は専用ハンドラを持たないため変更なし
  （インライン判定の理由を`AbilityData()`のコメントで明記）。
- `tests/abilities/test_ability_sa.py`（五十音順で該当するファイル）— テストを追加する。

## テスト観点

1. じゅくせい所持時、HP回復系きのみ（オボンのみ等）の回復量が2倍になる（端数切り捨て後に2倍。
   例: 最大HP150でオボンのみ発動時、37ではなく74回復する）。
2. じゅくせい所持時、能力ランク上昇系きのみ（例: リュガのみ）の上昇量が2倍になる。
3. じゅくせい所持時、17種の半減系きのみ（例: オッカのみ）が被ダメージを1/4にする
   （1/2の2回掛けと異なる端数になり得るケースで検証する）。
4. じゅくせい所持時、ジャポのみ/レンブのみの反動ダメージが最大HPの1/4になる。
5. じゅくせい所持時、ナゾのみの回復量が2倍になる。
6. じゅくせい所持時、ヒメリのみのPP回復量が20になる。
7. じゅくせい所持時、サンのみ・ミクルのみの効果量は変化しない（対象外の確認）。
8. じゅくせい非所持時は従来通りの効果量になる（回帰なし）。
9. なげつける・おちゃかいで対象に強制消費させたきのみもじゅくせい所持側の効果として2倍になる。
10. むしくい/ついばむで相手のきのみを奪って食べた場合、奪った側（じゅくせい所持側）の効果として
    2倍になる。

## リスクと対策

- リスク: `_heal_berry` の `r` 引数を `battle.modify_hp` に素通ししていた既存実装を
  `_berry_heal_amount` 経由の `v` 指定に変更するため、じゅくせいを持たない場合の
  丸め結果が変わらないことを回帰確認する必要がある。
  - 対策: `_berry_heal_amount` の非じゅくせい時の計算式を `modify_hp` の `r` 分岐
    （`max(1, int(max_hp * r))`）と完全に一致させる。
- リスク: `_modify_super_effective_damage` はホズのみ（`super_effective_only=False`）を
  含め17種全て共有しているため、対象外にすべきアイテム（`_boost_stat_on_type_hit`側）と
  混同しないよう関数を分けたまま変更する。
- リスク: `なげつける_apply_item_effect` は共通ヘルパーを使わない個別実装のため、
  `_heal_berry` 側の変更が自動的に反映されない。両方を変更する。
- リスク: かがくへんかガス解除・コアパニッシャーによる特性消滅と同一攻撃内で
  半減系きのみ/回復系きのみが連続発動する場合の効果対象可否は、一次情報上は
  「発動した時点で有効なポケモン本人の特性」で決まる。既存のイベント発火順序・
  特性無効化タイミングに従うため、じゅくせい側で追加のガス/かたやぶり判定は書かない
  （`mon.ability.name` を参照するだけで十分）。
