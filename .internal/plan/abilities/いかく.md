# 特性実装計画: いかく

更新日: 2026-07-10

## 仕様参照

- `.internal/spec/abilities/いかく.md`

## 実装ゴール

- 場に出たとき（`ON_SWITCH_IN`）、または特性を変える技・かがくへんかガス解除などで特性が有効化したとき
  （`ON_ABILITY_ENABLED`）、相手のこうげきを1段階下げる。
- みがわり状態の相手には無効。
- 相手が既にこうげき最低ランクなら不発（`battle.modify_stats` が0扱いにするため個別ガード不要）。
- しろいきり・かいりきバサミ/クリアボディ/しろいけむり/メタルプロテクト/フラワーベール・
  クリアチャームによる無効化、ミラーアーマーによる反射、ばんけんによる上昇反転、
  きもったま/せいしんりょく/どんかん/マイペースによる無効化は各アイテム・特性側の
  `ON_BEFORE_MODIFY_STAT` ハンドラで処理する（いかく側は `battle.modify_stats` を呼ぶだけでよい）。
- 対象はシングルバトルのみのため、複数対象・素早さ順の記述（ダブル/トリプル/ローテーション）は対象外。

## 実装方針

- `Event.ON_SWITCH_IN` と `Event.ON_ABILITY_ENABLED` の2イベントに同一関数を登録する。
  `subject_spec="source:self"`。
- priority は `.internal/spec/turn.md` の `Event.ON_SWITCH_IN` 表 100行目「いかく（特性）」に従い
  デフォルト値（100）のまま指定しない。
- `Event.ON_ABILITY_ENABLED` は `turn.md` に掲載がないため、既存の他特性（ON_ABILITY_ENABLED を
  使う特性は全て priority 未指定＝デフォルト100）に合わせてデフォルト値のまま指定しない。
- `battle.foe(source)` で相手を取得し、`target.has_volatile("みがわり")` の場合は
  `battle.modify_stats` を呼ばずに終了する（みがわりは相手のランク低下技/効果を防ぐため）。
- ランク低下自体の無効化・反射・反転（しろいきり、かいりきバサミ、クリアボディ、しろいけむり、
  メタルプロテクト、フラワーベール、クリアチャーム、ミラーアーマー、ばんけん、きもったま、
  せいしんりょく、どんかん、マイペース）は `ON_BEFORE_MODIFY_STAT` を経由して各特性・アイテム側の
  ハンドラで処理済み（本計画の対象外）。本レビューで以下の欠落・不整合を修正した:
  - どんかん・マイペースに `いかく` を無効化する `ON_BEFORE_MODIFY_STAT` ハンドラが未登録だった
    → 追加。
  - ばんけんのハンドラが `battle.modify_stats` を別呼び出しで +1 するだけで、元の -1 の変化を
    打ち消していなかったため最終ランクが 0 になっていた（仕様は +1 になるべき）
    → `value` の `"atk"` を直接 `+1` に書き換える方式に修正。
  - ビビリだま（アイテム）が `ON_MODIFY_STAT`（実際に変化が確定した後）に登録されていたため、
    クリアボディ等でいかくが完全に無効化されたケースで `ON_MODIFY_STAT` 自体が発火せず、
    本来発動すべきビビリだまが発動していなかった
    → `ON_BEFORE_MODIFY_STAT`（priority=50、無効化系ハンドラより先）に変更し、
    無効化の可否に関わらず「いかくによるこうげき変化が試みられたか」を先行判定する方式に修正。
    ミラーアーマー所持者自身は跳ね返した時点では発動しない例外を追加。

## テスト観点

1. いかくを持つポケモンが場に出ると相手のこうげきが1段階下がる。
2. みがわり状態の相手には無効。
3. どんかん・マイペースの相手には無効（第八世代以降）。
4. ばんけんの相手はこうげきが1段階下がらず、逆に1段階上がる。
5. クリアボディ等でいかくが無効化されても、ビビリだま所持者のすばやさは上がる。
6. ミラーアーマー所持者自身は跳ね返した時点ではビビリだまが発動しない。
