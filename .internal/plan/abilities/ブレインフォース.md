# 特性実装計画: ブレインフォース

更新日: 2026-07-12
実装状況: **✅ 完了**

## 仕様参照

- `.internal/spec/abilities/ブレインフォース.md`

## 実装ゴール

✅ 効果バツグンの攻撃技を使用したとき、与えるダメージを1.25倍にする。
✅ 相性判定はタイプ追加・相性上書き系の効果（もりののろい、ハロウィン、フリーズドライ、
   フライングプレス、らんきりゅう等）を経由した「最終的な相性」で判定する。
✅ 抜群のダメージを半減するきのみ（オッカのみ等）が発動した場合でも、相性判定そのものは
   変化しないためブレインフォースの効果は発動したままになる（きのみ側は最終ダメージを
   別途補正するのみで、タイプ相性の計算結果には影響しないため実装上は自然に満たされる）。

## 実装方針

✅ **効果抜群判定とダメージ1.25倍化**
- `Event.ON_CALC_DAMAGE_MODIFIER` で `attacker:self` として登録する（攻撃側の特性のため
  `AttackContext` の `attacker` ロールに合わせる）
- `battle.query.is_super_effective(ctx)` を使って判定する
  - 内部で `damage_calculator.calc_def_type_modifier(ctx)` を呼び、`modifier/4096 > 1` を
    満たす場合のみ True を返す
  - `calc_def_type_modifier` は `Event.ON_CALC_DEF_TYPE_MODIFIER` の結果であり、タイプ追加・
    相性上書き系の効果を経由した「最終的な相性」を反映するため、これらの効果との相互作用は
    追加実装不要
- 判定が真のとき、`value` を `apply_fixed_modifier(value, 5120)`（1.25倍固定小数点補正）で
  更新する
- たつじんのおび（アイテム、1.2倍・modifier=4915）と同一イベントで独立して累積するため、
  両方が有効な場合は自動的に1.25×1.2=1.5倍相当になる。固定小数点演算を逐次適用（各ステップで
  端数を切り捨て）するため、厳密な計算結果は modifier=6143（1.5倍ちょうどの6144より1小さい）
  になる。追加実装は不要。

### priority について

- `ON_CALC_DAMAGE_MODIFIER` は `.internal/spec/turn.md` に priority の記載がない（ダメージ計算内部
  イベントのため対象外）。既存の同種ハンドラ（`いろめがね_boost_ineffective`、
  `たつじんのおび_boost_super_effective` など）と同様に `priority` を指定せず既定値（100）を
  使う。

```python
"ブレインフォース": AbilityData(
    handlers={
        Event.ON_CALC_DAMAGE_MODIFIER: h.AbilityHandler(

            subject_spec="attacker:self",

    }

```

- `ブレインフォース_boost_effective()`: `is_super_effective` が真のとき最終ダメージ補正を
  1.25倍にする。

## テスト観点 ✅ 完了

✅ 1. 効果が「抜群」の技を使用した際、ダメージ補正が1.25倍（modifier=5120）になる。
