# トレース 実装計画

## スコープ
- 優先度: H
- 仕様書: `.internal/spec/abilities/トレース.md`
- 実装方針: 入場時に相手アクティブのコピー可能な特性へ差し替える。シングルバトル限定のため
  相手候補は常に1体（ランダム選択は不要）。相手が不在（瀕死で未交代）/ 相手がかがくへんか
  ガス発動中で自分のトレース自体が無効化されている場合は不発のまま保留し、後から相手側の
  変化（入場・特性有効化）を契機に再判定する。交代でベンチに戻る際は既存の
  `Pokemon.base_ability_name` の仕組みで自動的に元の `トレース` へ戻る（実装済み、対応不要）。

## 実装対象イベント
- `Event.ON_SWITCH_IN`（`source:self` と `source:foe` の双方に登録）
- `Event.ON_ABILITY_ENABLED`（`source:self` と `source:foe` の双方に登録）

## 仕様要点（実装に反映）
- 自分が入場したとき（`source:self`）、相手アクティブの特性をコピーする。
  - コピー不可（`uncopyable` フラグ）または相手が不在なら不発のまま保留する。
- 保留から再判定するトリガーを以下の2系統で拾う（`source:foe`）。
  - 相手が新たに入場したとき（`Event.ON_SWITCH_IN`）: 相手不在で保留していたケースに対応
    （例: 味方の交代技で相手を瀕死にし、自分がその交代先としてトレースで出た場合）。
  - 相手の特性が有効化されたとき（`Event.ON_ABILITY_ENABLED`）: 相手のかがくへんかガスが
    解除された、なりきり等で相手の特性が変わった等のケースに対応。
- 自分の特性（トレース自身）が有効化されたとき（`Event.ON_ABILITY_ENABLED`, `source:self`）にも
  再判定する。かがくへんかガス発動中は自分のトレースというハンドラ自体が
  `enabled_ignoring` の仕組みで自動的に呼ばれなくなるため、ガス側の無効化チェックを
  明示的に書く必要はない（`.claude/loop/_common.md` の既存メモリ方針どおり）。ガスが
  解除された瞬間にこのハンドラで再度コピー判定を行う。
- 相手が瀕死（`fainted`）の場合はコピーしない（保留のまま）。`battle.actives` は死に出し
  交代が完了するまで瀕死ポケモンを保持し続けるため、明示的なガードが必要。
- コピー成功後は `battle.change_ability` の内部処理でトレースのハンドラ自体が解除される
  ため、複数登録した各ハンドラが二重発動することはない（`change_ability` は
  `base_name` が既に一致する場合に早期リターンする点でも冪等）。
- とくせいガードによる不発は `battle.change_ability` 内の `is_change_blocked` /
  `ON_CHECK_ABILITY_DISABLE` の仕組みで自動的に処理される（アイテム側 `とくせいガード` の
  ハンドラがブロック＋メッセージを担当するため、トレース側での追加対応は不要）。
- コピー不可判定は特性フラグ `uncopyable` を基準に統一する。

## 変更対象
- `src/jpoke/data/ability.py`
  - `トレース` に `Event.ON_SWITCH_IN` / `Event.ON_ABILITY_ENABLED` それぞれについて
    `source:self` と `source:foe` の2ハンドラを登録（計4エントリ）。
- `src/jpoke/handlers/ability.py`
  - `_トレース_try_copy(battle, mon)`: コピー可否判定＋`change_ability` 呼び出しの共通処理。
  - `トレース_copy_ability`: `source:self` 用（自分の入場・自分の特性有効化）。
  - `トレース_copy_ability_on_foe_change`: `source:foe` 用（相手の入場・相手の特性有効化）。

## 対象外（今回は実装しない／既存のフレームワークに委ねる）
- ダブル/トリプルバトル固有の複数候補ランダム選択・対角除外（CLAUDE.md によりシングル
  バトル限定のため対象外）。
- とくせいガードで一度不発したメッセージが以後表示されなくなる仕様（表示制御のみの
  細部であり、`is_change_blocked` によるブロック自体は毎回正しく機能するため実害はない）。
- かがくのちから/レシーバー経由の即時トレース発動（レシーバーはダブル専用のため対象外）。

## テスト観点
- 発動: 相手のコピー可能特性を正しくコピーする（いかく等、即時発動系も含む）。
- 非発動: `uncopyable` 特性のみが相手にいる場合は不発。
- 非発動: 相手が瀕死で場を去っている場合はコピーしない。
- 保留からの再判定: 相手が瀕死のまま自分だけ交代 → 相手の死に出し交代で再判定して発動する。
- 保留からの再判定: 相手がかがくへんかガス発動中（自分のトレースも無効化）→ 相手が交代し
  ガスが解除されると新しい相手をコピーする。
- 境界: コピー後に交代すると `トレース` に戻る（既存実装で対応済み）。
