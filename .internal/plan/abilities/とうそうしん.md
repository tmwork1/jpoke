# 特性実装計画: とうそうしん

更新日: 2026-07-12

## 仕様参照

- `.internal/spec/abilities/とうそうしん.md`

## 実装ゴール

- 使用する攻撃技の威力が、相手と同性のとき 1.25 倍（5120/4096）になる。
- 相手と異性のとき 0.75 倍（3072/4096）になる。
- 性別不明（双方どちらか）のときは補正なし。
- こんらんの自傷ダメージ（内部技 `"_こんらん"`）には影響しない（第五世代以降の仕様）。

## 実装方針

- `Event.ON_CALC_POWER_MODIFIER`、`subject_spec="attacker:self"` で実装する。
  - 一次情報（ポケモンWiki）の「効果」節に「使用する攻撃技の**威力**が…」と明記されており、
    ATK/SATK 実数値ではなく技の威力を変動させる特性のため、`ON_CALC_ATK_MODIFIER` ではなく
    `ON_CALC_POWER_MODIFIER` が対象イベントになる。
  - `.internal/spec/turn.md` に本イベントの記載はないため、既存の同種ハンドラ（技の威力を変動させる
    特性）である `テクニシャン`（`ON_CALC_POWER_MODIFIER`, `subject_spec="attacker:self"`）を
    根拠に priority・登録形式を決定する。
- `mon.gender` の値（`"male"`, `"female"`, `""` 等）を確認して実装。性別不明は `""` を想定。
- 威力を変動させる他の特性（`テクニシャン`, `ちからもち` 等）と同様、こんらんの自傷ダメージ
  （内部技 `"_こんらん"`）は `ctx.move.name == "_こんらん"` で明示的に除外する。
  自傷ダメージも通常の `roll_damage` 経由でダメージ計算パイプライン（`ON_CALC_POWER_MODIFIER`
  含む）を通るため、除外しないと自傷ダメージにも威力補正がかかってしまう。

## 変更候補

- `src/jpoke/data/ability.py` — `とうそうしん` に handlers を登録する（`ON_CALC_POWER_MODIFIER`）
- `src/jpoke/handlers/ability.py` — `とうそうしん_modify_power` を実装する
- `tests/abilities/test_ability_ta.py` — テストを追加する

## テスト観点

1. 同性対決 → 威力が 1.25 倍（`power_modifier == 5120`）。
2. 異性対決 → 威力が 0.75 倍（`power_modifier == 3072`）。
3. 性別不明（片方または双方）→ 補正なし（`power_modifier == 4096`）。
4. こんらんの自傷ダメージには補正がかからない（`power_modifier == 4096`）。

    if ctx.move.name == "_こんらん":
        return HandlerReturn(value=value)
    a_gender = ctx.attacker.gender
    d_gender = ctx.defender.gender if ctx.defender else ""
