# 特性実装計画: スキルリンク

更新日: 2026-07-11

## 仕様参照

- `.internal/spec/abilities/スキルリンク.md`

## 実装ゴール

- 2～5回攻撃する連続攻撃技（`min_hits != max_hits` の通常連続技）が成功したとき、必ず最大回数（5回）攻撃になる。
- トリプルキック/トリプルアクセル（3回固定・毎ヒット命中判定）やネズミざん（10回固定・毎ヒット命中判定）は、
  初回ヒットが命中すれば残りのヒットは命中判定なしで必ず当たる。
- ダブルニードル・にどげり（2回固定）、すいりゅうれんだ（3回固定）など、元々ヒット数が固定
  （`min_hits == max_hits`）の技には影響しない。
- いかさまダイス（4回/5回抽選）を持っていても、スキルリンクの効果が優先されて必ず最大ヒット数になる。

## 実装方針

- ヒット数決定: `Event.ON_MODIFY_HIT_COUNT`、`subject_spec="attacker:self"` で実装。
  - `move_executor._resolve_hit_count` がベースのヒット数（乱数抽選結果）を算出したあとに発火する。
  - `ctx.move.max_hits > 1` の場合のみ `value = ctx.move.max_hits` に上書きする
    （`max_hits <= 1` の通常技・固定回数技は `min_hits == max_hits` であり、
    ここでの条件 `max_hits > 1` は 2～10 回の連続技のみを対象にする）。
  - `いかさまダイス`（`ON_MODIFY_HIT_COUNT`, priority=90）と同一イベントを取り合うが、
    特性側はデフォルト優先度（100）のまま登録し、いかさまダイスより後に実行させることで
    「後勝ちの上書き」によりスキルリンクの効果を優先させる（`data/item.py` のコメント参照）。
- 毎ヒット命中判定の無効化: `Event.ON_MODIFY_HIT_CHECK_EACH_TIME`、`subject_spec="attacker:self"` で実装。
  - `move_executor` は技の `check_hit_each_time` フラグ（トリプルキック等）を初期値として
    このイベントを発火し、結果が `True` なら毎ヒット命中判定、`False` なら初回ヒットのみ判定する。
  - スキルリンクは常に `value=False` を返し、毎ヒット判定を無効化する（初回ヒットのみ判定）。
  - いかさまダイスも同じ値（`False`）を返すため、優先度の取り合いは発生しない。
- 両イベントとも `.internal/spec/turn.md` に個別の記載はない（ダメージ計算内部の技実行フロー用イベント）。
  優先度は既存の `いかさまダイス`（同一2イベントを利用）の実装パターンに合わせる。

  トリプルキック/トリプルアクセル/ネズミざんが初回ヒットのみ判定されるテストを追加する（実装済み）
- `tests/items/test_item_a.py` — いかさまダイスと併せ持つ場合にスキルリンクが優先されるテストを追加する（実装済み）

## テスト観点

1. 2～5回攻撃する連続攻撃技（タネマシンガン等）使用時、必ず5回ヒットする。
2. トリプルキック/トリプルアクセル/ネズミざんは、初回ヒットが命中すれば残りは命中判定なしで全ヒットする。
3. ダブルニードル・にどげり・すいりゅうれんだなど固定回数技には影響しない（対象外のため個別テスト省略）。
4. いかさまダイスと併せ持つ場合、いかさまダイスの抽選（4回/5回）ではなくスキルリンクの最大ヒット数が優先される。
