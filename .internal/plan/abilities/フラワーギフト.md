# 特性実装計画: フラワーギフト

更新日: 2026-07-12

## 仕様参照

- `.internal/spec/abilities/フラワーギフト.md`

## 実装ゴール

- 天気が晴れ（にほんばれ/おおひでり）のとき、**物理技のこうげき補正**が 1.5 倍になる。
- 天気が晴れのとき、**特殊技のとくぼう補正**が 1.5 倍になる。
- 晴れ以外では発動しない。
- ノーてんき/エアロックで天候が無効な場合、および自身がばんのうがさを持つ場合は発動しない
  （`battle.weather_for(mon)` が両方をまとめて判定する）。

## 実装方針

- `Event.ON_CALC_ATK_MODIFIER`（attacker 側）と `Event.ON_CALC_DEF_MODIFIER`（defender 側）の
  2 ハンドラで実装する。
- `battle.weather_for(mon).sunny` で天候（ばんのうがさ・ノーてんき/エアロック考慮済み）を確認する。
- **こうげき側**: `ctx.move.category == "physical"` のときのみ `apply_fixed_modifier(value, 6144)`
  を適用する（こんじょう・サンパワー・ちからもち等と同じ、ON_CALC_ATK_MODIFIER は物理/特殊
  どちらの技でも発火するため、物理技のみに絞るのはハンドラ側の責務）。
- **とくぼう側**: `not battle.query.deals_physical_damage(ctx.attacker, ctx.move)` のときのみ
  適用する（サイコショック等、特殊技だが防御側実数値は「ぼうぎょ」参照になる技を除外するため。
  `ability_paradox.py` の `apply_def_modifier` と同じ判定パターン）。
- ワンダールーム中の実数値入れ替えとの相互作用は `.internal/spec/fields/ワンダールーム.md` の
  「範囲外（現行実装非対象）」に明記の通り対象外（本計画でも対応しない）。
- フォルムチェンジ（はなやかなすがた）は `turn.md` ON_SWITCH_IN priority 140 に記載あり。本計画では能力補正のみ対象とし、フォルムチェンジは別途対応。
- `flags=["uncopyable", "mold_breaker_ignorable"]` は data/ability.py に既定済み。

## テスト観点

1. 晴れ中・物理技 → こうげき補正が 1.5 倍になる。
2. 晴れ中・特殊技 → こうげき補正は変化しない（1.0 倍のまま）。
3. 雨等他天候では変化なし。
4. 晴れ中・特殊技を受けたとき → とくぼう補正が 1.5 倍になる。
5. 晴れ中・物理技を受けたとき → とくぼう補正は変化しない（1.0 倍のまま）。
6. ばんのうがさを持つ場合は発動しない。

```python
def フラワーギフト_modify_atk(battle, ctx, value):
    if battle.weather_for(ctx.attacker).sunny and ctx.move.category == "physical":
        value = apply_fixed_modifier(value, 6144)  # 1.5倍
    return HandlerReturn(value=value)


def フラワーギフト_modify_def(battle, ctx, value):
    if (
        battle.weather_for(ctx.defender).sunny
        and not battle.query.deals_physical_damage(ctx.attacker, ctx.move)
    ):
        value = apply_fixed_modifier(value, 6144)  # 1.5倍
    return HandlerReturn(value=value)
```
