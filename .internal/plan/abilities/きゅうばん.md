# 特性実装計画: きゅうばん

更新日: 2026-07-11
実装状況: **✅ 完了**

## 仕様参照

- `.internal/spec/abilities/きゅうばん.md`

## 実装ゴール

✅ 相手の交代させる技（ほえる・ふきとばし・ともえなげ・ドラゴンテール等）による強制交代を防ぐ。
✅ 相手のレッドカードによる強制交代を防ぐ（アイテム自体は消費される）。
✅ 自分からの交代（だっしゅつボタン・だっしゅつパック・使うと交代する技）は制限しない。
✅ かたやぶり等で特性が無視された場合は通常どおり交代させられる（`mold_breaker_ignorable`）。

## 実装方針

✅ **`Event.ON_TRY_BLOW` での無効化ハンドラとして実装**
- `ON_TRY_BLOW` は「制御系イベント」（`enums/event.py` 参照）であり、
  `core/move_executor.py`（ふきとばし・ほえる等の `_force_switch_random`）・
  `handlers/move_status.py`（`on_blow_apply`）から強制交代を試みる直前に発火される、
  ターン内の主要フェーズ（`.internal/spec/turn.md`）とは独立した内部サブイベントのため
  turn.md には掲載がない。同種の無効化ハンドラである `ねをはる`
  （`data/volatile.py` の `Event.ON_TRY_BLOW`）と同様に priority は指定しない
  （きゅうばん・ねをはるのいずれも「無効化するかどうか」のみを返すハンドラであり、
  複数登録されていても実行順に依存しないため）。
- `subject_spec="defender:self"`（`AttackContext` の防御側ロールと一致）。
- ハンドラは常に `value=False, stop_event=True` を返して技を無効化する
  （特性が無効化されている場合はそもそもハンドラが登録されないため、
  かたやぶり等によるチェックをハンドラ内で重複して行う必要はない）。
- レッドカードの強制交代は `handlers/item.py` の
  `_レッドカード_try_force_switch` 内で `foe.ability.name in ("きゅうばん", "ばんけん")`
  を直接判定して防ぐ（アイテムの消費は先に行われるため、きゅうばんが発動しても
  レッドカードは消費される）。`foe.ability.name` は特性が無効化されていれば
  空文字を返すため、かたやぶり等での無視も自動的に反映される。
- `ばんけん`（いかく反撃 + 同様の強制交代無効化）は、強制交代を防ぐ点は同じだが
  `.internal/spec/abilities/ばんけん.md`「特性バーは現れずに交代が無効化される」の通り
  発動演出が無い点が異なるため、`きゅうばん_block_blow` は共用せず専用の
  `ばんけん_block_blow`（`_announce_ability_triggered` を呼ばない）を実装する
  （`.internal/plan/abilities/ばんけん.md` 参照。2026-07-12 のばんけんレビューで分離）。

## 実装完了内容

### src/jpoke/handlers/ability.py
```python
def きゅうばん_block_blow(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """きゅうばん特性: 強制交代技の効果を防ぐ。"""
    _announce_ability_triggered(battle, ctx.defender)
    return HandlerReturn(value=False, stop_event=True)
```

### src/jpoke/data/ability.py
```python
"きゅうばん": AbilityData(
    flags={
        "mold_breaker_ignorable"
    },
    handlers={
        Event.ON_TRY_BLOW: h.AbilityHandler(
            h.きゅうばん_block_blow,
            subject_spec="defender:self",
        ),
    },
),
```

### src/jpoke/handlers/item.py（レッドカードとの連携）
`_レッドカード_try_force_switch` 内で、レッドカード発動時に攻撃側（強制交代させられる側）が
きゅうばん・ばんけんを持つ場合は交代をスキップする（アイテムは消費済み）。

## テスト観点

✅ 1. ふきとばし・ほえるを受けても交代しない（`tests/moves_status/`）。
✅ 2. ともえなげ・ドラゴンテールを受けてもダメージ後に交代しない（`tests/moves_attack/`）。
✅ 3. 自分からのだっしゅつボタン使用は制限されない（自動・専用テストなし）。
✅ 4. レッドカードの効果で交代させられないが、アイテムは消費される（`tests/items/test_item_yarawa.py`）。
✅ 5. かたやぶりの技には無視されて交代させられる（`tests/abilities/test_ability_ka.py`）。
