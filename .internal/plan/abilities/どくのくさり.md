# 特性実装計画: どくのくさり

更新日: 2026-07-12

## 仕様参照

- `.internal/spec/abilities/どくのくさり.md`

## 実装ゴール

- 攻撃技でダメージを与えたとき、30%の確率で相手をもうどく状態にする（直接攻撃でなくても発動）。
- こらえるでHP1のまま耐えたときやばけのかわ/アイスフェイスで攻撃を肩代わりされたときなど、
  技が成功して与えたダメージが0だったときでも発動することがある。
- みがわり状態に攻撃を防がれたときは発動しない。
- もうどく状態にできない相手（どく・はがねタイプ等）には無効
  （`AilmentManager._can_apply_by_type` が処理する）。
- りんぷん・おんみつマントを持つ相手には発動しない
  （`Event.ON_MODIFY_SECONDARY_CHANCE` 経由で確率0になる、既存の共通処理）。
- 連続攻撃技は1発ヒットするごとに発動判定がある。

## 実装方針

- `Event.ON_HIT`、`subject_spec="attacker:self"` で実装する。
  - `Event.ON_DAMAGE_HIT` は `actual_damage<=0` のとき発火しないため採用しない
    （`src/jpoke/core/move_executor.py` の `_execute_hit` 参照）。こらえるでHP1耐えした場合や
    ばけのかわ/アイスフェイスで身代わりされた場合など、技が成功してダメージが0のケースでも
    発動する仕様（どくげしょう・すなはき・じきゅうりょくと同じ既存パターン）を満たすため、
    常に発火する `Event.ON_HIT`（`.internal/spec/turn.md` 「ダメージ発生後の処理（みがわりに被弾しても
    発動）」）を使用する。
  - みがわりに阻まれた場合はハンドラ内で `ctx.substitute_damage` を見て除外する。
- `battle.resolve_secondary_chance(ctx, 0.3)` で確率を取得してランダム判定後、
  `battle.ailment_manager.apply(ctx.defender, "もうどく", source=ctx.attacker, ctx=ctx)` を呼ぶ。
- タイプ耐性（どく・はがねタイプ無効）は `ailment_manager.apply` 内部の `_can_apply_by_type` が
  処理するため、ハンドラ側で個別のタイプ判定は不要。
- りんぷん・おんみつマントは `resolve_secondary_chance` が内部で発火する
  `Event.ON_MODIFY_SECONDARY_CHANCE` により確率が0になるため、ハンドラ側で個別の判定は不要。
- **どくしゅ**（どく付与）と類似パターン。違いは直接攻撃限定でない点と、付与する状態異常が
  もうどくである点。

## 変更候補

- `src/jpoke/handlers/ability.py` — `どくのくさり_maybe_badly_poison` を実装する
- `tests/abilities/test_ability_ta.py` — テストを追加する

## テスト観点

1. 攻撃命中時に30%でもうどく付与される。
2. 確率外ではもうどくにならない。
3. こらえるでHP1のまま耐えたとき（実ダメージ0）でも発動する。
4. みがわりに攻撃を防がれたときは発動しない。
5. どくタイプへは `ailment_manager` の耐性チェックで弾かれる。
6. りんぷん所持相手には発動しない。
7. おんみつマント所持相手には発動しない。
8. 連続攻撃技はヒットごとに発動判定がある。

## 疑似コード

```python
def どくのくさり_maybe_badly_poison(battle, ctx, value):
    if ctx.substitute_damage:
        return HandlerReturn(value=value)
    if battle.random.random() < battle.resolve_secondary_chance(ctx, 0.3):
        battle.ailment_manager.apply(
            ctx.defender, "もうどく", source=ctx.attacker, ctx=ctx,
        )
    return HandlerReturn(value=value)
```

## 既知の制約（対応スコープ外）

- みらいよち/はめつのねがいのダメージ適用（`src/jpoke/handlers/field.py` の
  `みらいよち_damage`/`はめつのねがい_damage`）は `battle.modify_hp` を直接呼ぶのみで
  `Event.ON_HIT`/`AttackContext` を発火しない実装になっているため、現状はこの経路からの
  どくのくさり発動には対応していない。あくしゅう等、他の「攻撃技全般で発動する特性」にも
  共通する構造的な制約であり、フィールド技の実装フレームワーク側の課題のため本タスクでは
  対応しない。
