# 特性実装計画: ひひいろのこどう

更新日: 2026-07-12

## 仕様参照

- `.internal/spec/abilities/ひひいろのこどう.md`

## 実装ゴール

- 場に出たとき（`ON_SWITCH_IN`）天候を「はれ」にする。継続ターンは5（あついいわ所持で8）。
  発動優先度・不発条件・強天候による上書き不可・特性バー表示条件はひでりと同じ
  （`_activate_weather` 共通ヘルパーに委譲）。
- とくせいが再度有効化されたとき（`ON_ABILITY_ENABLED`）も同様に発動する。
- 場が「はれ」の間、物理技の攻撃を5461/4096倍（約1.33倍）にする。
  - 対象は物理技全般（ボディプレス・あなをほるも含む）。
  - こんらんの自傷ダメージ（内部技 `_こんらん`）には適用しない
    （ちからもち等の既存パターンに合わせる）。
  - イカサマを受ける側（自分の実数値が相手の攻撃力として使われる側）としては、
    `subject_spec="attacker:self"` によりそもそもハンドラが発火しないため、
    追加のガードは不要。
- 以下の場合は攻撃補正が得られない（`battle.weather_for(ctx.attacker)` が「なし」天候を返す
  ことで自動的に反映されるため、特性側での個別チェックは不要）:
  - 天気がにほんばれでなくなったとき。
  - 場にノーてんき/エアロックのポケモンがいるとき。
  - 自身がばんのうがさを持っているとき。
- 自身ではなく、ばんのうがさを持つ相手に攻撃するときは補正を受ける
  （`weather_for` は `ctx.attacker` 基準でチェックするため自動的に満たされる）。
- テラスタル中のテラバーストの分類決定（`テラバースト_modify_move_category`）は
  `mon.ranked_stats["atk"]` という生のランク補正値を使っており `ON_CALC_ATK_MODIFIER` を
  経由しないため、この特性の補正は分類決定に影響しない（既存実装のまま、変更不要）。

## 実装方針

- 天候発動は `ひでり` 等と同じ `_activate_weather` ヘルパー（`handlers/ability.py`）を使う。
- 攻撃補正は `ON_CALC_ATK_MODIFIER`（`subject_spec="attacker:self"`）で
  `battle.weather_for(ctx.attacker).sunny` と `ctx.move.category == "physical"` を判定して
  `apply_fixed_modifier(value, 5461)` を適用する。
  - `ctx.move.name != "_こんらん"` の除外条件を追加する（ちからもちの実装パターンに準拠）。

## イベント別実装計画（turn.md準拠）

1. `Event.ON_SWITCH_IN`（Priority 100, turn.md「ひひいろのこどう（特性）」行）
   - `ひひいろのこどう_activate_weather` → `_activate_weather(weather="はれ", count=5)`
2. `Event.ON_ABILITY_ENABLED`（turn.md未掲載の内部イベント。他の天候始動特性と同じくデフォルト
   優先度100のまま登録する）
   - `ひひいろのこどう_activate_weather` を同様に接続。
3. `Event.ON_CALC_ATK_MODIFIER`（turn.md未掲載。ダメージ計算内部のイベントのため、既存の同種
   攻撃補正特性＝ちからもち・こだいかっせい等の `ON_CALC_ATK_MODIFIER` ハンドラと同じ扱いとし、
   デフォルト優先度100のまま `subject_spec="attacker:self"` で登録する）
   - `ひひいろのこどう_modify_atk` → `battle.weather_for(ctx.attacker).sunny` かつ
     `ctx.move.category == "physical"` かつ `ctx.move.name != "_こんらん"` のとき
     `apply_fixed_modifier(value, 5461)`。

## テスト観点

1. 登場時に「はれ」が発動する（`ひでり` 系と共通の天候発動テストで間接確認済み）。
2. はれ中は物理技の攻撃が5461/4096倍になり、特殊技は補正なし
   （`test_ひひいろのこどう_はれ中に攻撃が1_33倍`）。
3. はれ以外の天候では補正なし（`test_ひひいろのこどう_はれ以外では補正なし`）。
4. こんらんの自傷ダメージには補正がかからない（要追加）。
5. ボディプレス使用時は防御実数値を代入した攻撃力に補正がかかる（要追加）。
6. イカサマを受ける側としては補正がかからない（要追加）。

## 実装ステップ

3. `tests/abilities/test_ability_ha.py` にこんらん・ボディプレス・イカサマ（受け側）のテストを
   追加する。
