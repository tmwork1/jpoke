# 特性実装計画: どくげしょう

更新日: 2026-07-12

## 仕様参照

- `.internal/spec/abilities/どくげしょう.md`

## 実装ゴール

- 物理技を受けると、攻撃者の場に「どくびし」を1層設置する（最大2層）。
- 特殊技では発動しない。
- すでに2層のどくびしがある場合は設置しない。
- HP1でこらえるを使用したときやみねうちを受けたときなど、実HPダメージが0のときも発動する。
- みがわりに攻撃を防がれたとき（実HPダメージ0）は発動しない。

## 実装方針

- `Event.ON_HIT`、`subject_spec="defender:self"`、priority省略（デフォルト100）で実装。
  - `Event.ON_DAMAGE_HIT` は `actual_damage<=0` のとき発火しない
    （`src/jpoke/core/move_executor.py` の実装を確認済み）ため、こらえる／みねうちで
    実HPダメージ0のときも発動する仕様を満たせない。既存の同種特性
    （こぼれダネ・じきゅうりょく）と同じく、常に発火する `Event.ON_HIT` を使用し、
    みがわりに阻まれた場合はハンドラ内で `ctx.substitute_damage` を見て除外する。
- 物理技判定後、攻撃者のサイドにどくびしを1層追加する。
- `battle.get_side(ctx.attacker)` で攻撃者側の SideFieldManager を取得し、
  `foe_side.activate("どくびし", 1)` を呼ぶ（技「どくびし」のハンドラ
  `どくびし_set_field` と同じ呼び方。最大層数チェック・カウント加算・
  イベントログ出力は `activate()` 内部で処理されるため、呼び出し側で
  独自に再実装しない）。

## 変更候補

- `src/jpoke/data/ability.py` — `どくげしょう` に handlers を登録する
- `src/jpoke/handlers/ability.py` — `どくげしょう_set_toxic_spikes` を実装する
- `tests/abilities/test_ability_ta.py` — テストを追加する

## リスクと対策

- リスク: どくびしの layer 追加に公開 API がない可能性がある。
  - 対策: 既存の「どくびし設置技（どくびし）」の move ハンドラ実装
    （`src/jpoke/handlers/move_status.py` の `どくびし_set_field`）を参照し、
    同じ `SideFieldManager.activate()` を使う。

## テスト観点

1. 物理技受けると攻撃者側にどくびしが1層設置される。
2. 特殊技では発動しない。
3. すでに2層の場合は追加されない。
4. こらえるでHP1のまま耐えた（実HPダメージ0）ときも発動する。
5. みがわりに攻撃を防がれたとき（実HPダメージ0）は発動しない。

def どくげしょう_set_toxic_spikes(battle, ctx, value):
    if ctx.move.category != "physical":
        return HandlerReturn(value=value)

        return HandlerReturn(value=value)
