# 特性実装計画: きんちょうかん

更新日: 2026-07-11

## スコープ

- 優先度: H
- 仕様書: `.internal/spec/abilities/きんちょうかん.md`
- 実装状態: 実装済み（本計画書はレビュー時に事後作成）
- 実装方針: `Event.ON_CHECK_NERVOUS`（`battle.query.is_nervous(pokemon)` から発火）に
  `subject_spec="source:foe"` でハンドラを登録し、常に `True` を返すことで
  「相手（source の foe）はきのみを使用できない」を表現する。実際のきのみ使用可否判定は
  各きのみハンドラ側（`handlers/item.py`）が `battle.query.is_nervous(mon)` を呼び出して
  個別にガードする方式とする（きんちょうかん側は判定結果を返すのみで、きのみ発動処理には
  関与しない）。
  自身がひんしになった瞬間に効果が失われる（`.internal/spec/abilities/きんちょうかん.md` の
  「特性の仕様」節）ため、`is_nervous` の判定対象（きんちょうかん所持ポケモン）が
  ひんしかどうかをハンドラ内で明示的にチェックする（ハンドラ登録上、対象が場を離れる
  タイミングとは独立に `is_nervous` は都度評価されるため、`fainted` を見ないと
  ひんし直後も効果が残ってしまう）。
- じんばいったい（`.internal/spec/abilities/じんばいったい.md`）はきんちょうかんと同一の
  きのみ禁止効果を持つため、同じハンドラ関数 `きんちょうかん_check_nervous` を共有する。

## 実装ゴール（完了定義）

- きんちょうかんを持つポケモンが場にいる間、相手の場のポケモンはきのみを使用できない
  （`battle.query.is_nervous(foe)` が `True` を返す）。
- きんちょうかん自身・味方には影響しない（`subject_spec="source:foe"` により保証）。
- きんちょうかんのポケモンがひんしになった瞬間、効果が失われる
  （交代して後続が出るのを待たない）。
- なげつける/むしくい/ついばむ/ほおばる/おちゃかい、特性はんすう、しぜんのめぐみ、
  トレーナーのバッグ使用によるきのみ消費はきんちょうかんの影響を受けない
  （`handlers/item.py` 側で `ON_FORCE_BERRY_TRIGGER` 等の強制発動経路に対しては
  `is_nervous` 判定を行わない設計とする）。
- 場に出たときのログ表示は `.internal/spec/turn.md` の `Event.ON_SWITCH_IN` priority 30 に従う。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_SWITCH_IN`（`.internal/spec/turn.md` L31, priority=30） —
  登場時に特性発動ログを記録する（`announce_ability_triggered` を流用）。
- `Event.ON_CHECK_NERVOUS`（`.internal/spec/turn.md` に掲載なし。`battle.query.is_nervous` は
  `is_trapped` 等の他の `ON_CHECK_*` 系クエリと同様、単一ハンドラが真偽値を返す
  質問イベントであり、優先度が意味を持たないため既定値のままとする） —
  相手のきのみ使用可否を判定する。

## 実装ステップ（レビュー時点で確認した内容）

1. 仕様書で「相手は全員きのみを使用できなくなる」「自身・味方には影響しない」
   「ひんしになった瞬間に効果が失われる」ことを確認する。
2. `is_nervous` クエリ（`core/query.py`）が `Event.ON_CHECK_NERVOUS` を
   `EventContext(source=pokemon)` で発火し、既定値 `False` を返す構成になっていることを確認する。
3. `きんちょうかん_check_nervous` を実装する。`subject_spec="source:foe"` により
   「判定対象 (`ctx.source`) の foe がきんちょうかん所持者と一致する場合のみ」発火する。
   ハンドラ内で `battle.foe(ctx.source)`（＝きんちょうかん所持者本体）の `fainted` を確認し、
   ひんし済みなら `False`（元の値）を返して効果を無効化する。
4. `src/jpoke/data/ability.py` の `きんちょうかん` / `じんばいったい` に
   `Event.ON_SWITCH_IN`（priority=30, `.internal/spec/turn.md` L31/L32）・
   `Event.ON_CHECK_NERVOUS` の handlers を登録する。
5. 各きのみの自動発動ハンドラに `is_nervous` チェックを追加する。ただし
   `Event.ON_FORCE_BERRY_TRIGGER`（なげつける/むしくい/ついばむ/ほおばる/おちゃかい経由の
   強制発動）にも登録されている状態異常・こんらん回復系きのみ
   （カゴ/クラボ/チーゴ/ナナシ/モモン/ラム/キーのみ）は、強制発動経路まで一律に
   ブロックすると仕様（強制発動はきんちょうかんで防げない）に反するため、
   本レビューでは対象外とし、別途個別の追加調査が必要な既知の未対応事項として残す。
6. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. きんちょうかんのポケモンが場にいる間、相手はきのみを使用できない
   （`battle.query.is_nervous` が `True`）。
2. きんちょうかん自身・味方には影響しない。
3. きんちょうかんのポケモンがひんしになった瞬間、効果が失われる。
4. じんばいったいもきんちょうかんと同じ効果を持つ。
5. イバンのみ・オボンのみなど、既存の相互作用テストが引き続き通る。
6. 弱点半減系きのみ（オッカのみ等）・HP1/4上昇系きのみ（カムラのみ等）・
   被弾時ランク上昇きのみ（アッキのみ）・被弾反撃きのみ（ジャポのみ）・
   ナゾのみが、相手のきんちょうかん下では発動しない。
