# 特性実装計画: ふしぎなうろこ

更新日: 2026-07-12

## 仕様参照

- `.internal/spec/abilities/ふしぎなうろこ.md`

## 実装ゴール

- 自分が状態異常（どく/もうどく/やけど/まひ/ねむり/こおり）のとき、防御（ダメージ計算で参照される実際のステータス）を1.5倍にする。
- こんらん・メロメロなどの状態変化（volatile）だけでは発動しない（`ailment.is_active` は状態異常のみを見る）。
- 通常特殊技には発動しないが、サイコショック/サイコブレイク/しんぴのつるぎのように防御を参照する特殊技には発動する。
- こんらんの自傷ダメージ（内部技 `_こんらん`）には発動しない（第五世代以降の仕様）。
- シェルアームズを受けるとき、分類（物理/特殊）の判定自体には影響しない
  （`シェルアームズ_modify_move_category` は `ranked_stats` のみを参照し `ON_CALC_DEF_MODIFIER` の
  結果を見ないため、実装上は自動的に無関係になる）。物理技に決まった場合はその後の
  `ON_CALC_DEF_MODIFIER` で通常どおり1.5倍が乗る。
- ワンダールーム中は防御と特防の参照が入れ替わるため、結果的に特防を1.5倍にする効果になる
  （追加実装は不要、既存のワンダールーム処理と組み合わせるだけで成立する）。
- かたやぶり・こだいのちからガス等で無効化される（`mold_breaker_ignorable` フラグ）。

## 実装対象イベント

- `Event.ON_CALC_DEF_MODIFIER`（`defender:self`）— ダメージ計算の防御補正を計算するタイミング。
  `.internal/spec/turn.md` に個別記載なし（ダメージ計算内部イベント）。既存の同種特性（`くさのけがわ` 等、
  `ON_CALC_DEF_MODIFIER` に登録する特性群）と同じくデフォルト priority=100 とする。

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `ふしぎなうろこ` に `Event.ON_CALC_DEF_MODIFIER` の handler を登録する
  （`flags={"mold_breaker_ignorable"}` 済み）
- `src/jpoke/handlers/ability.py` — `ふしぎなうろこ_boost_B` を実装する
- `tests/abilities/test_ability_ha.py` — テストを追加する

## 実装ステップ

1. `ctx.defender.ailment.is_active` で状態異常中か確認する。
2. `ctx.move.name != "_こんらん"` でこんらんの自傷ダメージを除外する。
3. `battle.query.deals_physical_damage(ctx.attacker, ctx.move)` で「防御実数値を参照するダメージ計算か」
   を判定する（`move.category == "physical"` の技に加え、`physical_damage` フラグを持つ特殊技
   〈サイコショック等〉も対象になる）。
4. 条件を満たす場合 `apply_fixed_modifier(value, 6144)` で1.5倍にする。

## テスト観点

1. 各状態異常（どく/もうどく/まひ/やけど/ねむり/こおり）で物理技を受けると防御補正が1.5倍になる
   （`def_modifier == 6144`）。
2. かたやぶり持ちの物理技には貫通され無効化される（`def_modifier == 4096`）。
3. 通常の特殊技（`physical_damage` フラグなし）には発動しない。
4. サイコショックのように防御を参照する特殊技には発動する。
5. こんらんの自傷ダメージには発動しない。
6. こんらん（volatile）のみで状態異常でない場合は発動しない。
7. ワンダールーム中は物理技でも実質的に特防が1.5倍になる（防御/特防の入れ替えとの組み合わせ）。

        and ctx.move.name != "_こんらん"

## リスクと対策

- リスク: こんらんの自傷ダメージ（`_こんらん`）も状態異常中であれば条件を満たしてしまい、
  除外条件を書き忘れると誤って1.5倍が乗ってしまう（第四世代までの仕様が誤って再現される）。
  - 対策: `ctx.move.name != "_こんらん"` を明示的に条件へ追加する（`くさのけがわ_boost_B` 等と
    同じパターン）。
