# 特性実装計画: やるき

更新日: 2026-07-13

## 仕様参照

- `.internal/spec/abilities/やるき.md`
- `.internal/spec/abilities/ふみん.md`（バトルでの効果はふみんと同一のため参照）

## 実装ゴール

- ねむり状態にならない（付与を防ぐ）。
- ねむけ状態にならない（付与を防ぐ）。
- すでにねむり状態のポケモンの特性がやるきに変わった場合（特性の書き換え・入れ替え・
  かがくへんかガス/かたやぶりの解除・メガシンカ等）、即座にねむりを回復する。
- すでにねむり状態のやるきのポケモンを場に出すと、即座にねむりを回復する
  （どくびし等サイドの状態異常付与判定より後に発動する）。
- かたやぶりの効果がある技でねむり状態にされた場合、技終了直後（かたやぶりの
  効果が切れた直後）に回復する。
- かたやぶりの効果でねむけ状態にされた場合、そのねむけからのねむり移行には
  かたやぶりの効果が及ばないため、やるきが有効なら通常通りねむりを防ぐ。
- なやみのタネはふみんに対しては失敗するが、やるきに対しては成功する
  （なやみのタネ側の実装で判定済み、やるき側の対応は不要）。

## 実装方針

ふみんと全く同じ効果のため、実装方針もふみんに準拠する
（`handlers/ability.py` の `やるき_prevent_volatile` / `やるき_cure_sleep_on_enable` は
ふみん用の共通ヘルパーを呼び出すだけの薄いラッパーとして既に実装済み）。

- `Event.ON_BEFORE_APPLY_AILMENT`（`target:self`）で `prevent_sleep_ailment`
  （`_prevent_ailment` 共通ヘルパー、blocked_ailments=["ねむり"]）を登録し、
  ねむり付与を防ぐ。`mold_breaker_ignorable` フラグにより、かたやぶりの技中は
  この判定自体が無効化される。
- `Event.ON_BEFORE_APPLY_VOLATILE`（`target:self`）で `やるき_prevent_volatile`
  （`_prevent_volatile` 共通ヘルパー、blocked_volatiles=["ねむけ"]）を登録し、
  ねむけ付与を防ぐ。
- `Event.ON_ABILITY_ENABLED`（`source:self`）で `やるき_cure_sleep_on_enable` を
  登録し、特性が有効化された時点ですでにねむり状態なら
  `battle.ailment_manager.remove(mon)` で即座に回復する。
  - `AbilityManager.change_ability` / `swap_ability` /
    `remove_disabled_reason`（かがくへんかガス・かたやぶりの解除）から
    `ON_ABILITY_ENABLED` が発火するため、これらのケースを一括でカバーする。
- `Event.ON_SWITCH_IN`（`source:self`）にも同じ `やるき_cure_sleep_on_enable` を
  登録する。理由: `AbilityManager` は交代による特性ハンドラの再登録時に
  `ON_ABILITY_ENABLED` を発火しない（`switch_manager` はハンドラ登録のみを行う）。
  そのため、「元の特性がやるきのポケモンが、特性を書き換えられた状態でねむり
  状態にされ、交代でベンチに戻る（このとき `Pokemon.reset_on_switch_out` が
  `base_ability` と異なる特性を元のやるきへ戻すが、ねむりは治さない）→ 再度場に
  出す」という経路では `ON_ABILITY_ENABLED` が発火せず、ねむりが治らないままに
  なってしまう。`.internal/spec/turn.md` の `ON_SWITCH_IN` に「Priority 100: やるき
  （特性）による状態異常回復」が明記されているため、`ON_SWITCH_IN` にも登録する
  ことでこの経路を正しくカバーする。
  - `ON_SWITCH_IN` の同一 Priority 100 には「どくびしによる毒付与」もあり、
    どくびしの毒付与判定（既にねむり状態のため重ね掛け不可で不発）が
    やるきの回復より先に実行される必要がある。どくびしはサイドの場の効果として
    交代の都度ではなく設置時に一度だけハンドラ登録されるのに対し、やるきは
    交代のたびに新規にハンドラ登録されるため、登録順（＝実行順）は自然に
    どくびしが先になる。

## 変更候補

- `src/jpoke/data/ability.py` — `やるき` に `ON_SWITCH_IN` handler
  （`やるき_cure_sleep_on_enable`）を追加登録する
- `src/jpoke/handlers/ability.py` — 既存の `やるき_cure_sleep_on_enable` を
  `ON_SWITCH_IN` からも呼び出せるよう docstring を更新する（関数自体の変更は不要）
- `tests/abilities/test_ability_ya.py` — テストを追加する

## テスト観点

1. ねむり状態にならない（付与失敗）。
2. ねむけ状態にならない（付与失敗）。
3. なやみのタネ等で特性がやるきに書き換わったとき、すでにねむり状態なら即座に治る。
4. かたやぶりの技でねむり状態にされても、技終了直後に回復する。
5. すでにねむり状態のやるきのポケモンを場に出すと、即座に回復する。
6. 5. と同時にどくびしが設置されている場合、どくびしの毒付与は不発し、
   その後やるきでねむりが治る。
