# 特性実装計画: フラワーベール

更新日: 2026-07-12

## 仕様参照

- `.internal/spec/abilities/フラワーベール.md`

## 実装ゴール

- 自分がくさタイプのとき、状態異常にならない。
- 自分がくさタイプのとき、ねむけ状態にならない。
- 自分がくさタイプのとき、相手由来の能力ランク低下を防ぐ（自分自身の技/特性による
  自発的なランク低下は防がない）。
- くさタイプでないポケモンには効果なし（シングルでは自分のみが対象。味方全体を守る
  効果はダブル以降の仕様であり本プロジェクトの対象外）。
- かたやぶり系には無効。

## 実装方針

- 3 ハンドラ構成:
  - `Event.ON_BEFORE_APPLY_AILMENT`（状態異常ブロック、`_prevent_ailment` 共通ヘルパーを使用）
  - `Event.ON_BEFORE_APPLY_VOLATILE`（ねむけブロック、`_prevent_volatile` 共通ヘルパーを
    `blocked_volatiles=["ねむけ"]` で使用。きよめのしお・ふみん等と同じパターン）
  - `Event.ON_BEFORE_MODIFY_STAT`（能力低下ブロック、`_block_stat_drop_by_foe` 共通ヘルパーを
    使用。クリアボディと同じパターンで、`ctx.source` が自分自身（None または target と同一）の
    場合は防がない）
- `"くさ" in mon.types` でくさタイプを確認（各ハンドラの冒頭でガード）。
- `flags={"mold_breaker_ignorable"}` は data/ability.py に既定済み。

## 変更候補

- `src/jpoke/data/ability.py` — `フラワーベール` に handlers を登録する
- `src/jpoke/handlers/ability.py` — `フラワーベール_prevent_ailment`、`フラワーベール_prevent_stat_drop`、
  `フラワーベール_prevent_volatile` を実装する
- `tests/abilities/test_ability_ha.py` — テストを追加する

## テスト観点

1. くさタイプへの状態異常 → 無効。
2. くさタイプへのねむけ状態 → 無効。
3. くさタイプへの相手由来の能力低下 → 無効。
4. くさタイプ自身の技/特性による自発的な能力低下（リーフストーム等） → 防がない。
5. くさタイプでないポケモンには保護なし。
6. かたやぶり系では無効。
