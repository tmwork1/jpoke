# 特性実装計画: おわりのだいち

更新日: 2026-07-11

## 仕様参照

- `.internal/spec/abilities/おわりのだいち.md`
- `.internal/spec/abilities/ひでり.md`（おおひでりの詳細はひでりを参照）

## 実装ゴール

- 場に出たとき（`ON_SWITCH_IN`）天候を「おおひでり」にする。
- とくせいが再度有効化されたとき（`ON_ABILITY_ENABLED`）も同様に発動する。
- おおひでり中は、みずタイプの技（攻撃技・変化技問わず）を失敗させる。
- おおひでり中は、はれ相当の効果（ほのお技威力1.5倍・みず技威力0.5倍・こおり付与無効）を適用する。
- 通常天候（はれ/あめ/すなあらし/ゆき）では上書きされず、他の強天候（らんきりゅう）は上書き可能。
- 場からこの特性のポケモンがいなくなったとき（`ON_SWITCH_OUT`）・とくせいが無効化されたとき
  （`ON_ABILITY_DISABLED`）おおひでりを解除する。ただし相手も同じ特性を持つ場合は解除しない。

## 実装方針

- 天候の発動・解除は、あめふらし/ひでり/はじまりのうみ/デルタストリームと共通の
  `_activate_weather` / `_deactivate_strong_weather` ヘルパー（`handlers/ability.py`）を使う。
  強天候は count=1 で登録する（`ON_TURN_END` でのカウントダウン対象外にするため）。
- みずタイプ技の失敗・はれ相当効果（威力補正・凍結無効）・天候の上書き優先度制御は、
  特性側ではなく `data/field/weather.py` の `"おおひでり"` エントリと
  `WeatherManager.apply`（`core/field_manager.py`）の優先度比較に委譲する。
  これにより、はれの `はれ_power_modifier` / `はれ_prevent_freeze` ハンドラをそのまま再利用できる。
- ノーてんき/エアロックによる効果抑制は `ON_CHECK_WEATHER_ENABLED` の共通導線（天候システム側）に従う。

## イベント別実装計画（turn.md準拠）

1. `Event.ON_SWITCH_IN`（Priority 100, turn.md「おわりのだいち（特性）」行）
   - `おわりのだいち_activate_weather` → `_activate_weather(weather="おおひでり", count=1)`
2. `Event.ON_ABILITY_ENABLED`（とくせいなし解除・スキルスワップ等での再有効化時、既存の他天候始動特性と同じ扱い）
   - `おわりのだいち_activate_weather` を同様に接続。
3. `Event.ON_SWITCH_OUT` / `Event.ON_ABILITY_DISABLED`
   - `おわりのだいち_deactivate_strong_weather` → `_deactivate_strong_weather(weather="おおひでり")`。
   - 現在の天候が「おおひでり」かつ、相手の特性名がこの特性と異なる場合のみ解除する
     （相手も同じ特性なら解除しない＝仕様書「相手の特性が同じ天候を発生させるものなら解除しない」）。
4. `Event.ON_TRY_MOVE_1`（Priority 10, turn.md「おおあめ・おおひでりによる技失敗」行）
   - `data/field/weather.py` の `"おおひでり"` に `おおひでり_block_water_move` を登録済み。
   - みずタイプの技を `stop_event=True` で失敗させる（新規ハンドラは不要、field側で実装済み）。
5. `Event.ON_CALC_POWER_MODIFIER` / `Event.ON_BEFORE_APPLY_AILMENT`
   - `"おおひでり"` に `はれ_power_modifier` / `はれ_prevent_freeze` を流用登録済み。
   - みず技は 4. で失敗するため、この補正が実際に効くのはほのお技威力補正とこおり無効のみ。
6. 天候の上書き制御
   - `WEATHER_PRIORITY`（`data/field/weather.py`）で `おおひでり` は優先度1、通常天候は0、
     らんきりゅうは2。`WeatherManager.apply` が優先度比較で上書き可否を判定するため、
     特性側に個別の上書き禁止ロジックは不要。
7. `Event.ON_TURN_END`
   - `"おおひでり"` には `tick_weather` を登録しない（強天候はターン経過で自然解除されない）。

## テスト観点

1. 登場時に「おおひでり」が発動し、count=1で登録される。
2. みずタイプの攻撃技・変化技が失敗する。
3. 通常天候（はれ/あめ/すなあらし/ゆき/おおあめ）では上書きされる。らんきりゅうでは上書きされない。
4. 相手も同じ特性を持つ場合、退場しても天候は解除されない。
5. 退場（交代・ひんし・とくせいなし化）でおおひでりが解除される。

## 実装ステップ（実装済み）

1. `data/field/weather.py` に `おおひでり` を追加し、はれ相当効果とみず技失敗を登録。
2. `data/ability.py` の `おわりのだいち` に `ON_SWITCH_IN` / `ON_ABILITY_ENABLED` /
   `ON_SWITCH_OUT` / `ON_ABILITY_DISABLED` ハンドラを接続。
3. `handlers/ability.py` に `おわりのだいち_activate_weather` /
   `おわりのだいち_deactivate_strong_weather` を実装（共通ヘルパー経由）。
4. `tests/abilities/test_ability__group.py` に登場時発動・上書き可否・退場時解除のテストを追加。
