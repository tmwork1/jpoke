# ふゆう の実装計画

## 概要
地面にいない状態として扱われ、じめんタイプ攻撃を無効化する特性。ON_CHECK_FLOATINGイベントで浮遊判定を返すことで実装。

## 実装対象のイベント
1. **`ON_CHECK_FLOATING`** - 浮遊判定イベント

## 実装内容

### イベント: ON_CHECK_FLOATING（浮遊判定）
- **担当**: `handlers/ability.py` の `ふゆう_float` ハンドラ（`うなぎのぼり` と共用。両特性とも「地面にいない」扱いになる効果が同一のため同じ関数を登録する）
- **処理**:
  - ふゆう持ちのポケモン → True を返す（無条件）
  - 優先度指定は不要：`じゅうりょく`（field）、`うちおとす`/`ねをはる`（volatile）、`くろいてっきゅう`（item）が value 入力に関わらず常に `False, stop_event=True` を返すため、これらの登録順がふゆうハンドラの前後どちらであっても最終結果は変わらない
  - `とくせいなし`/`かがくへんかガス`（特性無効化）、`かたやぶり`（`mold_breaker_ignorable` フラグ経由の一時無効化）が発動している間はふゆうの特性ハンドラ自体が `_check_handler_validity` によりスキップされるため、個別の分岐処理は不要

### 実装順序
1. `handlers/ability.py` に `ふゆう_float` ハンドラを追加
2. `AbilityData` に `flags={"mold_breaker_ignorable"}` と handlers を登録
3. テスト追加

## 実装の詳細ポイント
- `return HandlerReturn(value=True)` で浮遊扱いを返す（無条件・stop_event なし）
- priority は指定不要（デフォルト100）。じゅうりょく・うちおとす・ねをはる・くろいてっきゅうの各ハンドラは入力値を無視して無条件に上書き＋stop_eventするため、登録順序に依存せず正しく動作する
- subject_spec は `source:self`（自身の浮遊判定）
- `flags={"mold_breaker_ignorable"}` を付与し、かたやぶり・シャドーレイ等の技を受ける際に特性が一時無効化されるようにする（`handlers/ability.py` の `かたやぶり_disable_foe_ability` 等が参照）
- とくせいなし・かがくへんかガスによる無効化は `AbilityManager`／`_check_handler_validity` の共通機構でカバーされるため、専用の分岐は不要

## テスト項目
- ふゆう持ちが浮遊状態になる（`test_ふゆう_浮いている`）
- じめん技が通らないことを確認（`test_ふゆう_じめん技が通らない`）
- かたやぶり持ちの相手のじめん技は通ることを確認（`test_ふゆう_かたやぶりでじめん技が通る`）
- うちおとす状態になると浮遊が無効化される（`tests/volatiles/test_volatile_a.py::test_うちおとす_ふゆう特性より優先される`）
- ねらいのまとを持っていても浮遊による無効化（フリーフォール）には影響しない（`tests/items/test_item_na.py::test_ねらいのまと_浮遊による無効化には効果がない`）
- くろいてっきゅうを持つと浮遊が無効化される（`tests/items/test_item_ka.py` の該当テスト）
- だいちのはどう・ミストバースト・グラススライダー・ワイドフォース等「使用者の接地」を参照する技で浮遊が正しく反映される（`tests/moves_attack/` 各所）
- じゅうりょく状態・ねをはる状態は generic な `is_floating` 経路（`tests/test_field.py`, `tests/volatiles/test_volatile_satana.py`）で確認済み。特性固有の分岐がないためふゆう専用テストは追加不要
