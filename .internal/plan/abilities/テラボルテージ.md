# 特性実装計画: テラボルテージ

更新日: 2026-07-12

## 仕様参照

- `.internal/spec/abilities/テラボルテージ.md`
- `.internal/spec/abilities/かたやぶり.md`（無視できる特性の一覧・詳細仕様の参照先）

## 実装ゴール

- 場に出たとき・技を出すときに、相手の防御的な特性（`mold_breaker_ignorable` フラグを持つ特性）を
  無視して技を出せる。無視できる特性の範囲は `かたやぶり` と完全に同一（メタルプロテクト/
  ファントムガード/プリズムアーマーは無視できない）。
- かたやぶり・ターボブレイズと効果は完全に同一であり、場に出たときのメッセージのみ異なる
  （「<ポケモン名>は 弾ける オーラを 放っている!」）。
- かがくへんかガス等でこの特性自体が無効化されている間は効果を発揮しない。

## 実装方針

- かたやぶり・ターボブレイズと同じ挙動のため、既存の共有ハンドラ
  `かたやぶり_disable_foe_ability` / `かたやぶり_restore_foe_ability`
  （`src/jpoke/handlers/ability.py`）をそのまま再利用する。新規ハンドラは追加しない。
- `Event.ON_BEGIN_MOVE` で相手の無視できる特性を無効化し、`Event.ON_END_MOVE` で復元する
  （`subject_spec="attacker:self"`、`AttackContext` に合わせたロール指定）。
- 場に出たときのメッセージ表示は既存の汎用ハンドラ `announce_ability_triggered` を使う。
- **既存実装の欠落**: 現在の `テラボルテージ` エントリは `Event.ON_SWITCH_IN` にのみ
  `announce_ability_triggered` を登録しており、`Event.ON_ABILITY_ENABLED` への登録が
  抜けていた（ターボブレイズの過去レビューで発見・修正された欠落と同種）。`かたやぶり`
  （および `いかく`・`そうだいしょう` 等の「場に出たときに発動する特性」全般）は
  `ON_SWITCH_IN` と `ON_ABILITY_ENABLED` の両方に同一ハンドラを登録し、かがくへんかガスの
  効果が切れて特性が再有効化されたときにも発動メッセージを再度流している。`かたやぶり.md`
  備考に「ターボブレイズ/テラボルテージはかたやぶりと全く同一の効果を持つ」と明記されている
  以上、この非対称は実装漏れと判断し、`ON_ABILITY_ENABLED` の登録を追加する。
  （`ON_BEGIN_MOVE`/`ON_END_MOVE` 自体はガス解除時のハンドラ再登録で自動的に復元されるため
  機能面の欠落ではなく、発動ログが再表示されない見た目上の欠落）

## 変更候補

- `src/jpoke/data/ability.py` — `テラボルテージ` に `Event.ON_ABILITY_ENABLED` ハンドラ
  （`h.announce_ability_triggered`, `subject_spec="source:self"`）を追加する
- `src/jpoke/handlers/ability.py` — 変更なし（既存の `かたやぶり_disable_foe_ability` /
  `かたやぶり_restore_foe_ability` / `announce_ability_triggered` を再利用）
- `tests/abilities/test_ability_ta.py` — テストを追加する

## テスト観点

1. 場に出たときに特性が開示される（`ability.revealed`）。
2. 相手の無視できる特性（例: がんじょう）を無視して一撃で倒せる。
3. メタルプロテクト/ファントムガード/プリズムアーマーは無視できない。
4. かがくへんかガスの効果が切れて特性が再有効化されたとき、特性発動ログが再度記録される。

## 検証コマンド

```powershell
python -m pytest tests/abilities/test_ability_ta.py -k "テラボルテージ" -v
python -m pytest tests/ -v
```

## 疑似コード

```python
# data/ability.py
"テラボルテージ": AbilityData(
    handlers={
        Event.ON_SWITCH_IN: h.AbilityHandler(
            h.announce_ability_triggered,
            subject_spec="source:self",
        ),
        Event.ON_ABILITY_ENABLED: h.AbilityHandler(
            h.announce_ability_triggered,
            subject_spec="source:self",
        ),
        Event.ON_BEGIN_MOVE: h.AbilityHandler(
            h.かたやぶり_disable_foe_ability,
            subject_spec="attacker:self",
        ),
        Event.ON_END_MOVE: h.AbilityHandler(
            h.かたやぶり_restore_foe_ability,
            subject_spec="attacker:self",
        ),
    }
),
```
