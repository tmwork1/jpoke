# 特性実装計画: こおりのりんぷん

更新日: 2026-07-11

## 仕様参照

- `.internal/spec/abilities/こおりのりんぷん.md`

## 実装ゴール

- 特殊技を受けたときのダメージを半減する（`ctx.move.category == "special"` で判定）。
- サイコショック/サイコブレイク/しんぴのつるぎのように、ぼうぎょを参照しつつ分類は
  `special` のままの技には効果が乗る（`category` のみで判定するため自然に満たされる）。
- ダメージ固定技には影響しない（`power=0` の技は `calc_damages` の通常フローを通らず
  `ON_MODIFY_MOVE_DAMAGE` で最終ダメージが上書きされるため、`ON_CALC_DAMAGE_MODIFIER` 自体が
  発火しない。ハンドラ側で明示的なガードは不要）。
- 分類が変わる技（フォトンゲイザー・テラバースト等）は、`ON_CALC_DAMAGE_MODIFIER` 発火時点で
  `ctx.move.category` が既に変更後の分類になっているため、変更後の分類で自然に判定される。
- ワンダールーム状態でも特殊技のダメージ半減は適用される（`category` 判定のみのため、
  防御側の参照ステータス入れ替えとは独立して機能する）。
- かたやぶり等の効果がある技に対しては無効（`mold_breaker_ignorable` フラグで対応済み。
  ハンドラ側に無効化ロジックを重複実装しない）。

## 実装方針

- `Event.ON_CALC_DAMAGE_MODIFIER`、`subject_spec="defender:self"` で実装。
- priority は turn.md に記載のない内部（ダメージ計算内）イベントのため、既存の同一イベント・
  同一 `subject_spec="defender:self"` ハンドラ（ハードロック・マルチスケイル・もふもふ等）に
  合わせ、`AbilityHandler` のデフォルト値 `priority=100` を使用する。
- `ctx.move.category == "special"` のとき `apply_fixed_modifier(value, 2048)`
  （4096 基準で 0.5 倍）を適用する。
- `flags={"mold_breaker_ignorable"}` を付与し、かたやぶり等による特性無効化はフレームワーク
  側の共通処理に委譲する（ハンドラ内で個別に判定しない）。

## 変更候補

- `src/jpoke/data/ability.py` — `こおりのりんぷん` に `flags={"mold_breaker_ignorable"}` と
  `Event.ON_CALC_DAMAGE_MODIFIER` ハンドラを登録する（実装済み）。
- `src/jpoke/handlers/ability.py` — `こおりのりんぷん_reduce_special_damage` を実装する
  （実装済み）。
- `tests/abilities/test_ability_ka.py` — `test_こおりのりんぷん_かたやぶりで無効` /
  `test_こおりのりんぷん_特殊技のダメージ半減` にてカバー（実装済み）。

## テスト観点

1. 特殊技を受けたとき `damage_modifier` が 2048（0.5倍）になる。
2. 物理技を受けたときは `damage_modifier` が 4096（等倍）のまま変化しない。
3. かたやぶりを持つ相手の技に対しては無効（`damage_modifier` が 4096 のまま）。
4. サイコショック等、ぼうぎょを参照するが分類は特殊のままの技にも効果が乗る。
