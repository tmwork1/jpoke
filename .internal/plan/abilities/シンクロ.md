# 特性実装計画: シンクロ

更新日: 2026-06-03

## スコープ

- 優先度: H
- 仕様書: `.internal/spec/abilities/シンクロ.md`
- 実装状態: 未実装
- 実装方針: シンクロ所持者がどく・まひ・やけど状態にされたとき、その状態にした相手にも同じ状態異常を付与する。`Event.ON_APPLY_AILMENT` で発動させる。このイベントは `core/ailment_manager.py` ですでに発火済み。

## 実装ゴール（完了定義）

- 相手から受けた技でどく・まひ・やけど状態にされたとき、その相手にも同じ状態異常を付与する。
- かえんだま・どくどくだまによる自傷状態異常では発動しない。
- 相手がすでに状態異常の場合、タイプ耐性がある場合は発動しない。
- ねむり・こおり状態にされても発動しない。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_APPLY_AILMENT` — 状態異常付与後に発火。subject_spec="target:self"（シンクロ所持者が target）。`ctx.is_foe_target()` で自傷（かえんだまなど）を除外する。

## ctx の役割
ailment_manager.py が発火する apply_ctx は:
- `ctx.target` = 状態異常を受けたポケモン（シンクロ所持者）
- `ctx.source` = 状態異常を付与したポケモン（相手）
- `value` = 付与された状態異常名（"どく"/"まひ"/"やけど"）

`ctx.is_foe_target()` は `source != target` のとき True → 相手による付与

## 実装ステップ

1. 仕様書 `.internal/spec/abilities/シンクロ.md` で発動条件（どく/まひ/やけどのみ、自傷除外）と例外条件を確認する。
2. `src/jpoke/handlers/ability.py` に `シンクロ_return_ailment` を実装する。
   - `ctx.is_foe_target()` が False のとき（自傷）は return する。
   - value（付与された状態異常名）が "どく"/"まひ"/"やけど" でなければ return する。
   - `battle.ailment_manager.apply(ctx.source, value, source=ctx.target, ctx=ctx)` で相手に返す。
3. `src/jpoke/data/ability.py` の `シンクロ` に handler を登録する（subject_spec="target:self"）。
4. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. 相手の技でどく状態にされたとき、相手もどく状態になる。
2. 相手の技でまひ状態にされたとき、相手もまひ状態になる。
3. 相手の技でやけど状態にされたとき、相手もやけど状態になる。
4. ねむり・こおり状態にされても発動しない。
5. かえんだまによる自傷やけどでは発動しない。
6. 相手がすでに状態異常の場合は発動しない（付与できない）。
