# 特性実装計画: ダウンロード

更新日: 2026-07-12

## 仕様参照

- `.internal/spec/abilities/ダウンロード.md`

## 実装ゴール

- 入場時に相手の防御と特防を比較し、防御 < 特防なら攻撃+1、防御 ≧ 特防なら特攻+1を得る。
- 比較に使う防御・特防は、ランク補正（パワートリック・ガードシェアで書き換わった実数値を含む）を
  考慮した値を使用し、持ち物・特性・すなあらし・ゆきによる補正は除外する（`Pokemon.ranked_stats` は
  `Pokemon.stats`（実数値）× `rank_modifier` のみで構成されるため、この条件を自然に満たす）。
- ワンダールーム下では、実数値は据え置いたまま防御・特防のランク補正のみを入れ替えて比較する
  （本家の既知の仕様）。
- 上げる能力が既に最大ランクの場合は発動しない（`Battle.modify_stats` がランク上限で変化なしを
  返すため、追加の分岐は不要）。

## 実装方針

- `ON_SWITCH_IN` で `battle.foe(mon)` から相手を取得し、`Pokemon.ranked_stats["def"]` /
  `["spd"]` を比較する。
- `battle.get_global_field("ワンダールーム").is_active` が真の場合のみ、
  `foe.stats["def"] * foe.rank_modifier("spd")` と `foe.stats["spd"] * foe.rank_modifier("def")`
  を比較する（ランク補正のみ入れ替え）。
- ランク変化は `battle.modify_stats(mon, {boost_stat: +1}, source=mon)` を呼ぶ（CLAUDE.md の
  状態変更ルールに従い `Pokemon.rank` への直接代入はしない）。
- ランク変化が実際に適用された場合のみ `_announce_ability_triggered` でログを出す。
- かがくへんかガスによる無効化・発動順（素早さ順）・トレース等との同時発動順は
  イベント発火側／ターン制御側の責務のため、このハンドラでは扱わない。
- ダブルバトル・トリプルバトルでの複数体合算比較は対象外（本プロジェクトはシングルバトルのみ）。

## テスト観点

1. 防御 < 特防の相手に対して攻撃+1を得る。
2. 防御 ≧ 特防の相手に対して特攻+1を得る。
3. ランク補正（能力変化）を考慮した実数値で比較する。
4. ワンダールーム下では、ランク補正のみ入れ替えて比較する。
