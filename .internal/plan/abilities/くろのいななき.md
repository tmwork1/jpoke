# 特性実装計画: くろのいななき

更新日: 2026-07-11

## 仕様参照

- `.internal/spec/abilities/くろのいななき.md`

## 実装ゴール

- 攻撃技で他のポケモンをひんしにさせたとき、自分のとくこうが1段階上がる。
- 上がる能力（とくこう）を除けば `じしんかじょう`（こうげき版）・`しろのいななき`（こうげき版）と同様の効果。
- すでにとくこうが最大ランクまで上がっているときは発動しない（`battle.modify_stats` が
  変化なしを返すため追加のガードは不要）。

## 実装方針

- `Event.ON_MOVE_KO`、`subject_spec="attacker:self"` で実装する。
- `move_executor._execute_hit` が「攻撃技で相手をひんしにした」ときのみ `ON_MOVE_KO` を発火するため、
  ハンドラ側での追加のひんし判定・攻撃技判定は不要。
  ばけのかわ等のフォルムチェンジ消費ダメージで結果的にひんしになった場合も発火する。
- `じしんかじょう`・`しろのいななき` と共通の内部ヘルパー `_boost_on_move_ko(battle, ctx, value, stats)`
  を使い、上昇させる能力のみ `{"spa": +1}` に差し替える。
- priority: `turn.md` に `ON_MOVE_KO` の記載はないため、同一イベントを使う既存ハンドラ
  （`しろのいななき`, `じしんかじょう`, `ソウルハート`, `ビーストブースト`, `ゆうばく`, `とびだすなかみ`）
  にならい priority 指定なし（デフォルト）とする。これらは対象・効果が異なり相互干渉しないため
  順序依存がない。

## 変更候補

- `src/jpoke/data/ability.py` — `くろのいななき` に handlers を登録する（実装済み）
- `src/jpoke/handlers/ability.py` — `くろのいななき_boost`（実装済み、`_boost_on_move_ko` を利用）
- `tests/abilities/test_ability__group.py` — `じしんかじょう`/`しろのいななき`/`くろのいななき` の
  パラメタライズテストとしてまとめて検証する

## テスト観点

1. 攻撃技で相手を倒すと C が1段階上がる。
2. 相手を倒せなければ発動しない。
3. ばけのかわのフォルムチェンジ消費ダメージ（本来のダメージが0でも）で倒した場合も発動する。
4. C がすでに最大ランク（+6）のときは発動しない。

## 検証コマンド

```powershell
python -m pytest tests/abilities/test_ability__group.py -k "倒すと能力上昇系" -v
python -m pytest tests/ -v
```

## data/ability.py 登録（実装済み）

```python
"くろのいななき": AbilityData(
    handlers={
        Event.ON_MOVE_KO: h.AbilityHandler(
            h.くろのいななき_boost,
            subject_spec="attacker:self",
        )
    }
),
```
