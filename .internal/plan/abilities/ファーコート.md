# 特性実装計画: ファーコート

更新日: 2026-07-12

## 仕様参照

- `.internal/spec/abilities/ファーコート.md`

## 実装ゴール

- 攻撃技を受けるとき、自分の防御（ダメージ計算で参照される実際のステータス）を2倍にする。
- 通常特殊技には発動しないが、サイコショック/サイコブレイク/しんぴのつるぎのように防御を参照する特殊技には発動する。
- こんらんの自傷ダメージ（内部技 `_こんらん`）には発動しない。
- シェルアームズを受けるとき、分類の判定（物理/特殊のどちらになるか）にファーコートの効果は影響しない（分類判定は `defender.ranked_stats` を参照する `シェルアームズ_modify_move_category` が行い、ファーコートは `ON_CALC_DEF_MODIFIER` の別イベントで乗算されるだけなので、実装上は自動的に無関係になる）。技が物理技に決まった場合は防御を2倍にしてダメージ計算をする。
- ボディプレスで攻撃力として参照する自分の防御実数値には影響しない（`ON_CALC_ATK_MODIFIER` 側の処理であり、本特性は `ON_CALC_DEF_MODIFIER`・`defender:self` にのみ登録するため自動的に無関係になる）。
- ワンダールーム中は防御と特防の参照が入れ替わるため、結果的に特防を2倍にする効果になる（追加実装は不要、既存のワンダールーム処理〈`ワンダールーム_def_modifier`〉と組み合わせるだけで成立する）。
- かたやぶり・こだいのちからガス等で無効化される（`mold_breaker_ignorable` フラグ）。

## 実装対象イベント

- `Event.ON_CALC_DEF_MODIFIER`（`defender:self`）— ダメージ計算の防御補正を計算するタイミング。`.internal/spec/turn.md` に個別記載なし（ダメージ計算内部イベント）。既存の同種特性（`くさのけがわ` 等、`ON_CALC_DEF_MODIFIER` に登録する特性群）と同じくデフォルト priority=100 とする。

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `ファーコート` に `Event.ON_CALC_DEF_MODIFIER` の handler を登録済み（`flags={"mold_breaker_ignorable"}` 済み）
- `src/jpoke/handlers/ability.py` — `ファーコート_boost_B` を実装済み（要修正: こんらん除外漏れ）
- `tests/abilities/test_ability_ha.py` — テストを追加する

## 実装ステップ

1. `ctx.move.name != "_こんらん"` でこんらんの自傷ダメージを除外する。
2. `battle.query.deals_physical_damage(ctx.attacker, ctx.move)` で「防御実数値を参照するダメージ計算か」を判定する（`move.category == "physical"` の技に加え、`physical_damage` フラグを持つ特殊技〈サイコショック等〉も対象になる）。
3. 条件を満たす場合 `apply_fixed_modifier(value, 8192)` で2倍にする。

## テスト観点

1. 物理技を受けると防御補正が2倍になる（`def_modifier == 8192`）。
2. 特殊技（`physical_damage` フラグなし）には発動しない（`def_modifier == 4096`）。
3. サイコショックのように防御を参照する特殊技には発動する。
4. こんらんの自傷ダメージには発動しない。
5. かたやぶりで無効化される。

def ファーコート_boost_B(battle, ctx, value):
    if (
        ctx.move.name != "_こんらん"
        and battle.query.deals_physical_damage(ctx.attacker, ctx.move)

        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)

## リスクと対策

- リスク: こんらんの自傷ダメージ（`_こんらん`）も `category="physical"` のため、除外条件を書き忘れると誤って2倍が乗ってしまう。
  - 対策: `ctx.move.name != "_こんらん"` を明示的に条件へ追加する（`くさのけがわ_boost_B` 等と同じパターン）。実際、レビュー時点の実装にはこの条件が欠落しておりバグとして修正した。
