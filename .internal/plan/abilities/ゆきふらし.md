# 特性実装計画: ゆきふらし

更新日: 2026-07-14

## 仕様参照

- `.internal/spec/abilities/ゆきふらし.md`

## 実装ゴール

- 場に出たとき（`ON_SWITCH_IN`）天候を「ゆき」にする。継続ターンは5（つめたいいわ所持で8）。
- とくせいが再度有効化されたとき（`ON_ABILITY_ENABLED`。スキルスワップ・トレース・なりきり・
  かがくへんかガス解除など、ゆきふらしに書き換わった/再有効化された瞬間）も同様に発動する。
- 既に場がゆき状態のときは発動せず、継続ターンも書き換わらない（特性バーも表示されない）。
- 場がおおひでり・おおあめ・らんきりゅうのときは、天候をゆきに変えられないが、特性バー（発動ログ・
  `ability.revealed`）は表示される。
- 通常天候（はれ/あめ/すなあらし）は上書きする。
- 退場・ひんしになっても発動済みのゆき自体は消えない（ゆきふらし自身に解除ロジックは不要。ゆきは
  `ON_TURN_END` の周期処理で自然に切れる）。
- ゆきふらし自体にあられ状態のダメージを防ぐ効果は無い（このシミュレータはChampions＝第九世代準拠
  のため「あられ」自体が存在せず、この点はそもそも該当しない）。

## 実装方針

- 天候の発動は、あめふらし/ひでり/すなおこし/ひひいろのこどう/おわりのだいち/はじまりのうみ/
  デルタストリームと共通の `_activate_weather` ヘルパー（`handlers/ability.py`）を使う。
- 同一天候時の不発動・強天候（おおひでり・おおあめ・らんきりゅう）による上書き不可・天候上書きの
  優先度比較は、特性側ではなく `WeatherManager.apply`（`core/field_manager.py`, `WEATHER_PRIORITY`）に
  委譲する。
- 「強天候に阻まれて天候は変わらないが特性バーは表示される」動作は `_activate_weather` 内で、
  `WeatherManager.apply` が `False` を返した場合に `battle.weather_manager.current_name != weather`
  （＝阻まれたのが同一天候ではなく、より強い天候によるものだったか）を判定して実現する
  （同一天候時は `current_name == weather` になるため表示されない）。
- つめたいいわによる継続ターン延長（5→8）は `handlers/item.py` の `つめたいいわ_resolve_field_count`
  （`ON_MODIFY_DURATION`）に委譲されており、ゆきふらし側での重複実装は不要。
- 複数の天候始動特性が同時に場に出た場合の「すばやさが高い順に発動し、後発が上書きする」動作は
  `EventManager._sort_handlers`（`core/event_manager.py`）の Priority + 素早さソートと、通常天候同士は
  `WEATHER_PRIORITY` が全て同値（0）であることの組み合わせで実現されており、特性側の実装は不要。
- ゆき状態下のこおりタイプ防御補正（`ゆき_boost_def`, `handlers/field.py`）はゆき天候自体の効果であり、
  ゆきふらし特性固有のロジックではないため、この特性のハンドラでは扱わない。

## イベント別実装計画（turn.md準拠）

1. `Event.ON_SWITCH_IN`（Priority 100, turn.md「ゆきふらし（特性）」行）
   - `ゆきふらし_activate_weather` → `_activate_weather(weather="ゆき", count=5)`
2. `Event.ON_ABILITY_ENABLED`（turn.md未掲載の内部イベント。あめふらし/ひでり/すなおこし等
   既存の天候始動特性と同じくデフォルト優先度100のまま登録する。スキルスワップ等でゆきふらしに
   書き換わった瞬間に即時発動させるための接続で、他の天候始動特性と同一パターン）
   - `ゆきふらし_activate_weather` を同様に接続。

## テスト観点

1. 登場時に「ゆき」が発動し、count=5で登録される
   （`tests/abilities/test_ability__group.py::test_天候始動特性_登場時に発動`）。
2. 既に雪のときは再入場しても発動せず、継続ターンも据え置き・特性バーも表示されない
   （あめふらし等の共通テストで間接的に確認）。
3. おおひでり・おおあめ・らんきりゅうのときは天候を変えられないが、特性バー
   （`ability.revealed`）は表示される（あめふらし等の共通テストで間接的に確認）。
4. はれ/あめ/すなあらしの通常天候は上書きする（あめふらし等の共通テストで間接的に確認）。
5. つめたいいわ所持時は継続ターンが8になる（`tests/items/test_item_misc.py::test_天候延長アイテム`
   で天候システム側として確認済み）。
