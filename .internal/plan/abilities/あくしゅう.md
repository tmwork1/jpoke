# 特性実装計画: あくしゅう

更新日: 2026-07-10

## 仕様参照

- `.internal/spec/abilities/あくしゅう.md`

## 実装ゴール

- 攻撃技でダメージを与えたとき、10%の確率で相手をひるませる。
- みがわりに当たった場合は発動しない（ON_DAMAGE_HIT は damage > 0 のときのみ発火）。
- 元々ひるみの追加効果を持つ技（アイアンヘッド等）・一撃必殺技には重複して発動しない。
- 特性りんぷん・アイテムおんみつマント・にじ状態（確率2倍）等の補正は
  `battle.resolve_secondary_chance` 経由で共通処理される。

## 実装方針

- `Event.ON_DAMAGE_HIT`、`subject_spec="attacker:self"` で実装。
- `ctx.move.has_flag("ohko")` および元々ひるみ効果を持つ技名の一覧
  （`_INNATE_FLINCH_MOVES`）に該当する場合は判定自体を行わない。
  - `handlers/item.py` の `おうじゃのしるし`/`するどいキバ`
    （`flinch_on_hit_10pct`）と同一のリストをミラーする。
- 10%の乱数判定後、`battle.volatile_manager.apply(ctx.defender, "ひるみ", ...)` で付与。
- 既存の **ほのおのからだ・せいでんき** および `おうじゃのしるし`/`するどいキバ` と同一パターン。

## 変更候補

- `src/jpoke/data/ability.py` — `あくしゅう` に handlers を登録する
- `src/jpoke/handlers/ability.py` — `あくしゅう_maybe_flinch` を実装する
- `tests/abilities/test_ability_a.py` — テストを追加する

## テスト観点

1. 攻撃技が命中したとき 10% でひるみが付与される。
2. 確率外ではひるみを付与しない。
3. 元々ひるみの追加効果がある技（エアスラッシュ等）には重複して発動しない。
4. 一撃必殺技（じわれ等）には効果が無い。

## 既知の未対応ケース

- HPが1でこらえるを使用した相手への攻撃や、ばけのかわ/アイスフェイスで攻撃を肩代わりされた場合など
  「技は成功したが実ダメージが0」のケースでもあくしゅうは発動しうる仕様だが、現行実装は
  `Event.ON_DAMAGE_HIT` が `actual_damage > 0` のときのみ発火する
  （`src/jpoke/core/move_executor.py`）ため未対応。おうじゃのしるし/するどいキバ
  （`Event.ON_DAMAGE_HIT` を使用する道具全般）にも共通する既知の制約であり、
  本レビューでは対応範囲外とする。

## 疑似コード

```python
def あくしゅう_maybe_flinch(battle, ctx, value):
    if ctx.defender is None or ctx.move.has_flag("ohko") or ctx.move.name in _INNATE_FLINCH_MOVES:
        return HandlerReturn(value=value)
    chance = battle.resolve_secondary_chance(ctx, 0.1)
    if battle.random.random() < chance:
        battle.volatile_manager.apply(ctx.defender, "ひるみ", source=ctx.attacker)
    return HandlerReturn(value=value)
```
