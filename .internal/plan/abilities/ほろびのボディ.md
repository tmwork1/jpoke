# 特性実装計画: ほろびのボディ

更新日: 2026-06-05

## 仕様参照

- `.internal/spec/abilities/ほろびのボディ.md`

## 実装ゴール

- 直接攻撃を受けると、自分と攻撃者の両方がほろびのうた状態になる。
- 直接攻撃でない技では発動しない。
- すでにほろびのうた状態のポケモンには重複して付与しない。
- 攻撃者がひんし（じばく等）のときは発動しない。

## 実装方針

- `Event.ON_DAMAGE_HIT`、`subject_spec="defender:self"`、priority=20 で実装。
- priority は `turn.md` ON_DAMAGE priority 20「のろわれボディ等: 攻撃技を受けた」に準拠。
- `battle.query.is_contact(ctx)` 後、自分と攻撃者の両方に "ほろびのうた" volatile を付与する。

## 変更候補

- `src/jpoke/data/ability.py` — `ほろびのボディ` に handlers を登録する
- `src/jpoke/handlers/ability.py` — `ほろびのボディ_on_damage` を実装する

## テスト観点

1. 直接攻撃受けると自分と攻撃者の両方にほろびのうたが付与される。
2. 非接触技では発動しない。
3. すでにほろびのうた状態には二重付与しない。
4. 攻撃者がじばくでひんしなら発動しない。

```python
def ほろびのボディ_on_damage(battle, ctx, value):
    if not battle.query.is_contact(ctx):
        return HandlerReturn(value=value)
    attacker = ctx.attacker
    if attacker is None or attacker.fainted:
        return HandlerReturn(value=value)
    defender = ctx.defender
    triggered = False
    for mon in (defender, attacker):
        if not mon.check_volatile("ほろびのうた"):
            battle.volatile_manager.apply(mon, "ほろびのうた", count=3, source=defender)
            triggered = True
    if triggered:
        announce_ability_triggered(battle, ctx, value, mon=defender)
    return HandlerReturn(value=value)
```
