# 特性実装計画: かがくへんかガス

更新日: 2026-07-11

## 仕様参照

- `.internal/spec/abilities/かがくへんかガス.md`

## 実装ゴール

- この特性を持つポケモンが場にいる間、`gas_proof` フラグを持つ特性（ARシステム・
  アイスフェイス・うのミサイル・かがくへんかガス自身・ぎょぐん・じんばいったい・
  スワームチェンジ・ぜったいねむり・ダルマモード・テラスチェンジ・ばけのかわ・
  バトルスイッチ・マイティチェンジ・マルチタイプ・リミットシールド）を除き、
  相手（シングルバトルのため対象は1体のみ）の特性の効果が発動しなくなる。
- 発動順は `.internal/spec/turn.md` の `Event.ON_SWITCH_IN` 表に従い、テラスチェンジ
  （priority=10）の後、他の場に出たときに発動する特性（デフォルトpriority=100）
  より先（priority=20）に発動する。
- 自分が場に出たとき既に場にいる相手の特性を無効化し、逆に自分が場にいる間に
  新たに相手が交代してきた場合もその特性を即座に無効化する。
- 自分（かがくへんかガスのポケモン自身）が交代で退場する、またはいえき等で
  とくせいなし状態になり自身の特性が無効化されたときに、相手の特性無効化を解除する。
- この特性自身は `uncopyable` フラグを持ち、トレース・なかまづくり・うつしえで
  コピーできない。また `protected` フラグは持たない
  （いえき/コアパニッシャーを受けるととくせいなし状態になり、これにより
  かがくへんかガスの効果は切れる）。
- ただしスキルスワップ/さまようたましいによる特性の交換だけは対象外
  （`protected` フラグを持つ特性と同様に失敗する）。

## 実装方針

- `Event.ON_SWITCH_IN`
  - `subject_spec="source:self"`, priority=20:
    自分の登場時、`_announce_ability_triggered` でメッセージを出し、相手の特性が
    `gas_proof` でなければ `battle.add_ability_disabled_reason(foe, "かがくへんかガス")`
    を呼ぶ。
  - `subject_spec="source:foe"`, priority=20:
    既に自分がガスを発動している状態で相手が新たに交代してきたとき（トリガーは
    相手の `ON_SWITCH_IN`）、その相手の特性が `gas_proof` でなければ同様に無効化する。
    こちらはメッセージを出さない（既にガスは発動済みのため）。
- `Event.ON_ABILITY_ENABLED`, `subject_spec="source:self"`:
  自分の特性（かがくへんかガス）がとくせいなし状態などから復帰し再び有効化した
  ときに、`ON_SWITCH_IN` と同じ無効化処理を再実行する。
- `Event.ON_SWITCH_OUT`, `subject_spec="source:self"`:
  自分が退場する（ひんし含む）ときに、`battle.remove_ability_disabled_reason(相手, "かがくへんかガス")`
  で相手の特性無効化を解除する。
- `Event.ON_ABILITY_DISABLED`, `subject_spec="source:self"`:
  自分の特性が（いえき等で）とくせいなし状態になったときに同様に解除する。
- 上記4イベントはいずれも `turn.md` に個別記載がないため、`ON_SWITCH_IN` の
  発動イベント（priority=20）を除き、デフォルト優先度（100）のまま指定しない。
- `gas_proof` フラグは「無効化できない特性」のみに付与し、`data/ability.py` の
  該当エントリに追加する。

## 本レビューで修正した既存の不整合

- `じんばいったい` に `gas_proof` フラグが欠落していた
  （`.internal/spec/abilities/じんばいったい.md` は明記済みだったが実装漏れ）→ 追加。
- `スキルスワップ_can_apply` / `さまようたましい_swap_ability_on_contact` が
  `protected` フラグのみで判定しており、`かがくへんかガス`（`protected` を
  持たない）を対象にした交換を防げていなかった → `base_name == "かがくへんかガス"`
  の個別判定を追加。
- `うつしえ_change_ability` / `なりきり_change_ability` が対象の
  `ability.name`（無効化中は空文字列になる）をそのままコピー元にしていたため、
  かがくへんかガスで特性が抑制されている相手からコピーすると使用者の特性が
  空になってしまっていた → とくせいなし状態（`has_volatile("とくせいなし")`）
  の場合のみ空文字列を、それ以外（かがくへんかガス等の抑制のみ）の場合は
  `base_name` をコピーする `_ability_name_to_copy` ヘルパーを追加して修正。
- `AbilityManager.change_ability` が特性ハンドラを解除してから
  `ON_ABILITY_DISABLED` を発火していたため、なりきり等でかがくへんかガス
  保持者自身の特性が変わっても `かがくへんかガス_gas_deactivate` が発火せず、
  相手の特性が無効化されたまま戻らなくなっていた → `swap_ability` と同様に
  「発火してから解除する」順序に修正。

## テスト観点

1. かがくへんかガスのポケモンが場に出ると、相手の特性が無効化される。
2. かがくへんかガスのポケモンが場を離れる（交代/ひんし）と、相手の特性が再び有効化される。
3. `gas_proof` フラグを持つ特性（例: マルチタイプ、ばけのかわ、じんばいったい）は
   かがくへんかガスでも無効化されない。
4. かがくへんかガスの相手にスキルスワップ/さまようたましいを使っても特性は交換されない。
5. かがくへんかガス保持者自身が使うなりきりで、特性が抑制されている相手の特性を
   コピーすると、抑制前の本来の特性がコピーされ、かがくへんかガス保持者自身の特性が
   変わったことでガス効果が解除され、相手の特性が再有効化される。
6. とくせいなし状態の相手をなりきり/うつしえでコピーすると空の特性になる
   （既存の回帰確認）。
