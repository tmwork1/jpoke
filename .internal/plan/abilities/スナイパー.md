# 特性実装計画: スナイパー

更新日: 2026-07-11

## スコープ

- 仕様書: `.internal/spec/abilities/スナイパー.md`
- 実装状態: 実装済み
- 実装方針: `Event.ON_CALC_DAMAGE_MODIFIER` で急所時のみ追加で1.5倍を乗算する。急所自体の基礎倍率1.5倍（`core/damage.py` 内で先に適用済み）と掛け合わさることで、通常時と比べ2.25倍のダメージになる。

## 実装ゴール（完了定義）

- 攻撃技が相手の急所に当たったとき、最終ダメージが（急所基礎倍率1.5倍と合わせて）2.25倍になる。
- 急所に当たらなかったときは補正が発動しない。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_CALC_DAMAGE_MODIFIER` — ダメージ補正（やけど補正の後、まもる貫通系補正の前）。`.internal/spec/turn.md` に掲載のないダメージ計算内部イベントのため、同種のダメージ補正特性（`いろめがね` 等）の登録に合わせてデフォルト優先度で登録する。

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `スナイパー` に `Event.ON_CALC_DAMAGE_MODIFIER` の handler を登録する（実装済み）
- `src/jpoke/handlers/ability.py` — `スナイパー_boost_critical` を実装する（実装済み）
- `tests/abilities/test_ability_sa.py` — テストを追加する（実装済み）

## 実装ステップ

1. 仕様書 `.internal/spec/abilities/スナイパー.md` で発動条件（急所ヒット時のみ）を確認する。
2. `src/jpoke/handlers/ability.py` に `スナイパー_boost_critical` を実装する。`ctx.critical` が真のときのみ `apply_fixed_modifier(value, 6144)`（1.5倍）を適用する。
3. `src/jpoke/data/ability.py` の `スナイパー` に `Event.ON_CALC_DAMAGE_MODIFIER` の handler を `subject_spec="attacker:self"` で登録する。
4. `tests/abilities/test_ability_sa.py` にテストを追加し、全テストが通ることを確認する。

## テスト観点

1. 急所ヒット時（`critical_rank=3` の確定急所技を使用）に `damage_modifier` が 6144（1.5倍）になる。
2. 急所に当たらないときは補正が発動しない（`いろめがね` 等の既存ダメージ補正テストと同様の非発動確認は他特性テストで担保）。
3. リーサル計算は対象外（`.internal/progress/ability.md` の該当行が n/a）。急所自体が乱数依存のため、リーサル計算では確定急所以外を考慮しない方針。
