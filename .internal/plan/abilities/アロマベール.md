# 特性実装計画: アロマベール

更新日: 2026-07-10

## 仕様参照

- `.internal/spec/abilities/アロマベール.md`

## 実装ゴール

- 自分（シングルバトルでは味方は対象外）が以下の揮発性状態にならない。
  - アンコール / いちゃもん / かいふくふうじ / かなしばり / ちょうはつ / メロメロ
- 発動源（技・特性メロメロボディ/のろわれボディ・アイテムあかいいと等）を問わず一律で無効化する。
- かたやぶり等の特性無視効果を持つ技で狙われた場合は無効化できない
  （既存の `ON_BEFORE_MODIFY_STAT`/`ON_BEFORE_APPLY_VOLATILE` 系ハンドラと同様、
  `mold_breaker_ignorable` フラグ + かたやぶり側ハンドラによる一時的な特性無効化で対応済み）。
- 既に付与された状態変化を後から治すことはできない（付与前の無効化のみ）。

## 実装方針

- `ON_BEFORE_APPLY_VOLATILE` で揮発性状態の付与を無効化する。
  どんかん・マイペース・ふみん・スイートベール・せいしんりょく等の既存実装と同パターンで、
  共通ヘルパー `_prevent_volatile(battle, ctx, value, blocked_volatiles=[...])` を利用する。
- `subject_spec="target:self"`: `ON_BEFORE_APPLY_VOLATILE` は
  `EventContext(source=source, target=target)` で発火されるため、
  「揮発性状態を付与されそうな自分自身」を対象にする役割指定として正しい。
- `flags={"mold_breaker_ignorable"}` を付与し、かたやぶり等が発動中は
  `battle.add_ability_disabled_reason(defender, "かたやぶり")` により
  一時的に特性が無効化される既存の仕組みに乗せる
  （`_check_handler_validity` が `subject.ability.enabled` を見てハンドラをスキップする）。
- 現行世代（第八世代以降）ではかたやぶり使用者自身のアロマベールは無視されない。
  これは「かたやぶり側ハンドラが `ctx.defender`（＝攻撃対象）のみを無効化し、
  攻撃者自身のアビリティには触れない」という既存実装で自然に満たされる。

## テスト観点

1. アンコール/いちゃもん/かいふくふうじ/かなしばり/ちょうはつ/メロメロの直接付与を防ぐ。
2. あくび（ねむけ）・いばる・のろい等、メンタル系以外の状態変化は防がない。
3. アイテムあかいいとによるメロメロ付与を防ぐ（`tests/items/test_item_a.py` で確認済み）。
4. 特性メロメロボディ/のろわれボディの接触時付与を防ぐ。
5. かたやぶり等の特性無視効果を持つ技で対象になった場合は防げない。
