# 特性実装計画: かたいツメ

更新日: 2026-07-11

## スコープ

- 仕様書: `.internal/spec/abilities/かたいツメ.md`
- 実装状態: 実装済み
- 実装方針: `ON_CALC_POWER_MODIFIER` で攻撃側として `is_contact(ctx)` が True の技（接触技）に
  5325/4096倍の固定小数点補正を適用する。

## 実装ゴール（完了定義）

- 接触技を使用したときの威力が1.3倍（正確には5325/4096倍）になる。
- 接触技でない技（でんきショック等）には補正がかからない。
- パンチグローブを持っている場合、パンチ技には効果が発動しない
  （`is_contact()` が `Event.ON_CHECK_CONTACT` を経由するため、パンチグローブ側の
  接触判定無効化ハンドラが自動的に反映される）。
- ぼうごパットを持っている相手に対しても発動する（`is_contact()` は
  `is_contact_reaction()` と異なりぼうごパットの影響を受けない）。
- みがわりに防がれた場合も発動する（威力計算自体はダメージ判定前に行われるため）。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_CALC_POWER_MODIFIER` — ダメージ計算時の威力補正イベント（`AttackContext`,
  `attacker:self`）。ターン全体のイベント一覧（`.internal/spec/turn.md`）には掲載されていない
  ダメージ計算内部イベントのため、同種の威力補正特性（てつのこぶし・がんじょうあご・
  パンクロック・メガランチャー・きれあじ）と同じ priority（`AbilityHandler` 既定値 100）を
  踏襲する。

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `かたいツメ` に `Event.ON_CALC_POWER_MODIFIER` の handler を登録する
- `src/jpoke/handlers/ability.py` — `かたいツメ_modify_power` を実装する
- `tests/abilities/test_ability_ka.py` — テストを追加する

## 実装ステップ

1. 仕様書 `.internal/spec/abilities/かたいツメ.md` で発動条件・例外条件を確認する。
2. `src/jpoke/handlers/ability.py` に `かたいツメ_modify_power(battle, ctx, value)` を実装する。
   `battle.query.is_contact(ctx)` が True のとき `apply_fixed_modifier(value, 5325)` を返す。
3. `src/jpoke/data/ability.py` の `かたいツメ` に `Event.ON_CALC_POWER_MODIFIER` の
   `AbilityHandler(h.かたいツメ_modify_power, subject_spec="attacker:self")` を登録する。
4. `tests/abilities/test_ability_ka.py` にテストを追加し、全テストが通ることを確認する。

## テスト観点

1. 接触技（たいあたり等）を使用したとき威力補正倍率が5325/4096になる。
2. 接触技でない技（でんきショック等）には補正がかからない（4096のまま）。
3. パンチグローブ所持時はパンチ技に補正がかからない
   （`tests/items/test_item_ha.py` の `test_パンチグローブ_パンチ技の接触判定を無効化`
   により `is_contact()` がFalseになることは既に確認済み）。
