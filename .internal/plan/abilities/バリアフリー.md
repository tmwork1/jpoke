# 特性実装計画: バリアフリー

更新日: 2026-06-05

## 仕様参照

- `.internal/spec/abilities/バリアフリー.md`

## 実装ゴール

- 場に出たとき、敵味方全体のリフレクター・ひかりのかべ・オーロラベールを消す。
- すでに壁が一切ない場合はアナウンスなし。

## 実装方針

- `Event.ON_SWITCH_IN`、`subject_spec="source:self"` で実装。priority は指定しない
  （`.internal/spec/turn.md` ON_SWITCH_IN の「100 バリアフリー（特性）」= Handler のデフォルト値と一致）。
- `battle.side_managers` を両サイドループし、`SideFieldManager.deactivate(name)`（公開 API）で
  リフレクター・ひかりのかべ・オーロラベールを解除する。
- 1つでも解除された場合のみ `_announce_ability_triggered` でログを記録する。

## 変更候補

- `src/jpoke/data/ability.py` — `バリアフリー` に ON_SWITCH_IN handler を登録する
- `src/jpoke/handlers/ability.py` — `バリアフリー_remove_screens` を実装する
- `tests/abilities/test_ability_ha.py` — テストを追加する

## テスト観点

1. 相手側にリフレクターがあるとき → 入場で解除される。
2. 自分側にひかりのかべがあるとき → 入場で解除される。
3. 両サイドのオーロラベールが同時に解除される。
4. 壁がない場合 → アナウンスなし。
5. しんぴのまもり・しろいきり・設置技（まきびし等）は解除されない。

## 疑似コード

```python
_BARRIER_SCREENS: tuple[str, ...] = ("リフレクター", "ひかりのかべ", "オーロラベール")

def バリアフリー_remove_screens(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    mon = ctx.source
    triggered = False
    for side in battle.side_managers:
        for screen in _BARRIER_SCREENS:
            if side.deactivate(screen):
                triggered = True
    if triggered:
        _announce_ability_triggered(battle, mon)
    return HandlerReturn(value=value)
```
