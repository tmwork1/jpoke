# 特性実装計画: うのミサイル

更新日: 2026-07-10

## スコープ

- 優先度: H
- 仕様書: `.internal/spec/abilities/うのミサイル.md`
- 実装状態: 実装済み（レビューにより flags・ダイビング溜めターン・スキルスワップ/さまようたましいの
  例外を追補）
- 実装方針: なみのり/ダイビング成功時にフォルムチェンジ、攻撃を受けてダメージが入ったとき獲物を吐き出して反撃ダメージ＋副効果を与える。ウッウ専用のため所持者チェックが必要。

## 実装ゴール（完了定義）

- ウッウがなみのり/ダイビングを成功させると HP に応じてフォルムチェンジする（HP 1/2 超 → うのみのすがた、1/2 以下 → まるのみのすがた）。
  - なみのりは技が成功（`move_applied`）した時点で判定する。
  - ダイビングは1ターン目の溜め行動が成功した時点で判定が確定し、2ターン目の攻撃がまもる等で無効化されても維持する（パワフルハーブで1ターンで繰り出した場合も同様）。
- 咥えたすがたのとき物理・特殊問わずダメージを受けると獲物を吐き出し、相手最大 HP の 1/4 ダメージ + 副効果（うのみ: ぼうぎょ -1、まるのみ: まひ）を与える。
- みがわりで防いだとき・ダメージ 0 のとき・姿を隠しているとき（一部例外除く）は発動しない。
- ウッウ以外のポケモンがこの特性を得ても発動しない。
- かがくへんかガスで無効化されない（gas_proof）。
- 上書き（いえき・なかまづくり・シンプルビーム・なやみのタネ等）に対しては protected フラグで抵抗するが、
  SV Ver.3.0.0 以降はスキルスワップ/さまようたましいでの交換・トレース等によるコピーは可能（`protected`
  フラグは持つが `うのミサイル` の base_name を個別に除外し、`uncopyable` フラグは付与しない）。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_MOVE_CHARGE`（priority=90） — ダイビングの溜めターン（1ターン目）成功時にフォルムチェンジする。
  技自身の「ダイビング」揮発状態付与ハンドラ（priority=100）より先に実行し、揮発状態が未付与
  （＝1ターン目）であることを条件にする。根拠は `.internal/spec/turn.md` の `Event.ON_MOVE_CHARGE` 表に追記済み。
- `Event.ON_MOVE_END` — なみのり成功後にフォルムチェンジを行う
- `Event.ON_DAMAGE_HIT` — 攻撃を受けたとき獲物を吐き出し反撃ダメージと副効果を与える
- `Event.ON_SWITCH_OUT` — 交代時にフォルムを「通常のすがた」に戻す

## テスト観点

1. なみのりが命中するとHP半分超なら「うのみのすがた」になる。
2. なみのりが命中するとHP半分以下なら「まるのみのすがた」になる。
3. ダイビングは1ターン目の溜めターンで成功時点でフォルムチェンジし、2ターン目がまもるで無効化されても維持される。
4. 「うのみのすがた」のとき攻撃を受けると通常に戻り相手のぼうぎょが -1 になる。
5. 「まるのみのすがた」のとき攻撃を受けると通常に戻り相手がまひ状態になる。
6. みがわりで防いだ場合は吐き出しが発動しない。
7. 交代するとフォルムが「通常のすがた」に戻る。
8. ウッウ以外のポケモンがこの特性を持っても発動しない。
9. いえき・なかまづくり・なやみのタネ・シンプルビームは protected フラグにより失敗する。
10. スキルスワップ・さまようたましいでは例外的に交換できる。
11. なりきりでコピーできる（uncopyable フラグを持たないため）。

## 既知の制限（対応見送り）

以下は一次情報で言及されているが、エンジン全体の共通処理に起因する既知の制限であり、
`うのミサイル` 固有の対応は行わない（他の `Event.ON_DAMAGE_HIT` 系反応特性にも共通する制約のため）。

- **HP1でこらえるを使用した際の0ダメージ発動**: `move_executor._execute_hit` は
  `actual_damage <= 0` の場合に `Event.ON_DAMAGE_HIT` を発火しないため、実際のダメージが
  0でもこらえるで生存した場合に発動する仕様を再現できない。
- **みらいよち/はめつのねがいの反撃**: これらのダメージは `handlers/field.py` から
  `battle.modify_hp` を直接呼び出す実装になっており、`Event.ON_DAMAGE_HIT` を経由しないため
  反撃が発動しない。
- **ウッウ以外がウッウにへんしんした状態でスキルスワップ等により本特性を獲得した場合**の
  フォルムチェンジ抑制は、種族名ベースの判定（`mon.name == "ウッウ"`）のみで対応しており、
  これらが複合した極端なケースまでは検証していない。
