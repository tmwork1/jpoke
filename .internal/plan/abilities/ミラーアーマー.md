# ミラーアーマー 実装計画

## スコープ
- 優先度: H
- 仕様書: `spec/ability/ミラーアーマー.md`
- 実装方針: `ON_BEFORE_MODIFY_STAT` で相手由来の能力ランク低下を検知し、低下量を source（相手）側へ転送して target（自分）側の低下をゼロに置き換える。

## 実装対象イベント
- `Event.ON_BEFORE_MODIFY_STAT`

## 仕様要点（実装に反映）
- `source != target`（相手由来）かつ低下値を含む場合のみ反射。
- 低下値だけを source 側へ `battle.modify_stats(source, ...)` で適用し、自分側の戻り値からは低下分を除去（ゼロ化）する。
- 上昇値は通常どおり受ける（除去しない）。
- ランク低下以外の技本体の効果（ダメージ等）はこのハンドラに含まれないため素通り。
- まけんき / かちき は `ON_MODIFY_STAT` フェーズなので競合なし（本特性は `ON_BEFORE_MODIFY_STAT`）。
- `priority=140`: クリアチャーム（既定100）や しろいきり（130）の無効化ハンドラより後に判定し、
  それらで既にランク低下が無効化されている場合は反射しないようにする（`びびり` の
  `ON_BEFORE_MODIFY_STAT` ハンドラと同じパターン。優先度は `.internal/spec/turn.md` に未掲載のため、
  同種の既存ハンドラ（びびり・しろいきり・クリアチャーム）を参照して決定）。

## 変更対象（想定）

  ```python
      flags={"mold_breaker_ignorable"},
      handlers={
          Event.ON_BEFORE_MODIFY_STAT: h.AbilityHandler(
              h.ミラーアーマー_reflect_stat_drop,

          )
      }
  ),
  ```

### `src/jpoke/handlers/ability.py`
- `ミラーアーマー_reflect_stat_drop` を追加する。
  ```python
  def ミラーアーマー_reflect_stat_drop(battle, ctx, value):
      # 反射由来の低下・自己デメリット・相手が存在しない場合は素通り
      if not ctx.is_foe_target() or ctx.stat_change_reason == "ミラーアーマー":
          return HandlerReturn(value=value)
      # 低下分のみ抽出
      drops = {stat: v for stat, v in value.items() if v < 0}
      if drops:
          # 低下を source（相手）側へ反射。
          # modify_stats の target は ctx.source（元の攻撃者）、source は ctx.target
          # （ミラーアーマーのポケモン）にする。ctx.source を target・source 両方に
          # 使うと「自己デメリット」扱いになりまけんき/かちきが発動しなくなるため誤り。
          battle.modify_stats(ctx.source, drops, source=ctx.target, reason="ミラーアーマー")
          # 自分側から低下分を取り除く（上昇分は残す）
          value = {stat: v for stat, v in value.items() if v > 0}
          # だっしゅつパック: 自身のランクは実際には変化しないが、跳ね返した時点で発動する
      return HandlerReturn(value=value)
  ```

## テスト観点
| ケース | 内容 |
| --- | --- |
| 発動: いかく | いかくポケモンが入場 → ミラーアーマー保持側の攻撃は下がらず、いかく使用側の攻撃が下がる |
| 発動: 追加効果の能力低下 | 相手の攻撃技の追加効果（例: アシッドボム）で下がるはずの特防を反射する |
| 非発動: 自己デメリット | からをやぶる等、自分が下げる効果は反射しない |
| 非発動: かたやぶり | かたやぶり系使用者からの能力低下は反射しない |
| 非発動: かがくへんかガス | かがくへんかガス下では特性が無効化され、反射しない |
| 境界: 相手が最低ランク | 反射先（相手）が既に最低ランクの場合、ランクが変わらない旨が正しく処理される |
| 境界: 上昇と低下が混在 | 低下分のみ反射し、上昇分は自分が受ける |

## 実装前確認事項

### 無限反射の防止・まけんき/かちきの正しい発動
反射呼び出しは `battle.modify_stats(ctx.source, drops, source=ctx.target, reason="ミラーアーマー")` とする。
`modify_stats` の `target` 引数（実際にランクが変化するポケモン）は元の攻撃者 `ctx.source`、
`source` 引数（変化の原因）はミラーアーマーのポケモン `ctx.target` にする。
この結果生じる `ON_MODIFY_STAT` の新しいコンテキストは `target=元の攻撃者, source=ミラーアーマーのポケモン`
となり、両者は別ポケモンのため `まけんき`/`かちき` の `ctx.source != ctx.target` 判定が真になり正しく発動する
（`target` と `source` の両方に `ctx.source`（元の攻撃者自身）を使うと「自己デメリット」扱いになり
まけんき/かちきが発動しなくなるため誤り）。

#### `reason` を `EventContext` に追加して無限反射を防止
1. `src/jpoke/types/` に `StatChangeReason` を `HPChangeReason` と同形式で追加する。
   ```python
   StatChangeReason = Literal[
       "",                 # 通常（理由なし）
       "ミラーアーマー",   # ミラーアーマーによる反射
   ]
   ```
2. `EventContext` に `stat_change_reason: StatChangeReason = ""` フィールドを追加する。
3. `status_manager.py` の `modify_stats` 呼び出し側で `EventContext(target=target, source=source, stat_change_reason=reason)` として渡す。
4. `ミラーアーマー_reflect_stat_drop` の反射呼び出し時に `reason="ミラーアーマー"` を渡す。
5. ハンドラ冒頭で `if ctx.stat_change_reason == "ミラーアーマー": return HandlerReturn(value=value)` としてガードする
   （反射で生じた `ON_BEFORE_MODIFY_STAT` を再度ミラーアーマーが処理して無限反射するのを防ぐ）。
