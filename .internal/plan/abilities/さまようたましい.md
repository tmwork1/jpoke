# 実装計画: さまようたましい

作成日: 2026-06-04
更新日: 2026-07-11（実装済みコードに合わせてレビュー・更新）

## スコープ

- 仕様書: `.internal/spec/abilities/さまようたましい.md`
- 実装状態: 実装済み（`src/jpoke/handlers/ability.py` の `さまようたましい_swap_ability_on_contact`）

## 効果
直接攻撃でダメージを受けたとき、攻撃者と自分の特性を入れ替える。

入れ替え不可の特性（ARシステム・アイスフェイス・ふしぎなまもり等）は全て `protected` フラグを持つため、`has_flag("protected")` でチェックできる。

例外は以下の2つ:
- **うのミサイル**: `protected` フラグを持つが、SV Ver.3.0.0 以降（本プロジェクトの対象範囲）はさまようたましいでの交換が可能になったため、`base_name == "うのミサイル"` を個別に除外して交換を許可する
  （`.internal/spec/abilities/うのミサイル.md` 参照）。
- **かがくへんかガス**: `protected` フラグを持たない（いえき/コアパニッシャーでとくせいなし状態になれるため）が、
  さまようたましいでの交換だけは対象外のため `base_name == "かがくへんかガス"` を個別に判定して除外する
  （`.internal/spec/abilities/かがくへんかガス.md` 参照。2026-07-11 のかがくへんかガスレビューで追加）。

ぼうごパットによる発動阻止・みがわり貫通不可は、`battle.query.is_contact_reaction()` が
`ON_CHECK_CONTACT_REACTION` を発火して判定するため、ハンドラ側で個別に実装する必要はない。
特性の実際の入れ替えは `battle.ability_manager.swap_ability()`（`battle.swap_ability()` 経由）を使う。
このメソッドは両者の `ON_ABILITY_DISABLED` → 入れ替え → 両者の `ON_ABILITY_ENABLED` をまとめて発火するため、
片方だけ入れ替わることがなく、入れ替え後の特性（いかく等）も正しく即時発動する。

## イベント／ハンドラ

| イベント | priority | subject_spec | 関数名 |
|---------|---------|-------------|--------|
| `ON_DAMAGE_HIT` | 20 | `defender:self` | `さまようたましい_swap_ability_on_contact` |

turn.md に直接掲載なし。同じ「直接攻撃を受けたときに特性を書き換える」系統の `ミイラ`・`とれないにおい` が
`ON_DAMAGE_HIT` priority=20 で実装済みのため、それに合わせる。

## 実装（実装済みコード: `src/jpoke/handlers/ability.py`）

```python
def さまようたましい_swap_ability_on_contact(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """さまようたましい特性: 直接攻撃を受けたとき攻撃者と特性を入れ替える。

    例外: うのミサイルは protected フラグを持つが、SV Ver.3.0.0 以降は
    さまようたましいでの交換が可能になったため base_name で個別に除外する。

    かがくへんかガスは protected フラグを持たないが、とくせいなし状態に
    なっていてもさまようたましいでの交換は発動しないため base_name で
    個別に判定する。
    """
    attacker = ctx.attacker
    defender = ctx.defender
    if (
        not battle.query.is_contact_reaction(ctx)
        or attacker is None
        or attacker.fainted
        or (attacker.ability.has_flag("protected") and attacker.ability.base_name != "うのミサイル")
        or attacker.ability.base_name == "かがくへんかガス"
        or defender is None
    ):
        return HandlerReturn(value=value)
    battle.ability_manager.swap_ability(defender, attacker)
    return HandlerReturn(value=value)
```

**補足**:
- KO されたときも発動する仕様 → `ON_DAMAGE_HIT` は KO 後にも発火するため追加のガード不要
- `is_contact_reaction` はぼうごパットによる無効化と接触判定を両方内包する
- 攻撃者が fainted の場合（じばくなど）は発動しない
- みがわりが身代わりになった場合、defender の実HPが減らないため `ON_DAMAGE_HIT` 自体が発火せず自動的に不発動

## data/ability.py 登録（実装済み）

## 既知の制限（未対応・優先度低）

仕様書に記載の以下のケースは、反動技（アフロブレイク等）のハンドラが `Event.ON_HIT` で登録されており、
`さまようたましい`／`ミイラ`／`とれないにおい` の特性入れ替え（`Event.ON_DAMAGE_HIT`、`ON_HIT` より後に発火）
より前に反動判定が走ってしまうため、現状は仕様通りに再現できていない:

- 「いしあたまのポケモンから反動技を受けた場合、特性を交換した後に反動ダメージが発生する」
  （`.internal/spec/abilities/さまようたましい.md` および `.internal/spec/abilities/いしあたま.md` に記載）

この制限は `さまようたましい` 固有ではなく、`ミイラ`・`とれないにおい` にも同様に存在する既存の
アーキテクチャ上の制約（反動技を含む全ハンドラの発火イベントの見直しが必要）であり、本レビューでは
対応範囲外として保留する。

## テストケース（実装済み: `tests/abilities/test_ability_sa.py`, `tests/items/test_item_ha.py`）
- 直接攻撃を受ける → 攻撃者と特性が入れ替わる
- 直接攻撃でない技 → 発動しない
- 攻撃者の特性が `protected`（おもかげやどし等）→ 発動しない
- うのミサイルは例外的に入れ替わる
- かがくへんかガスは入れ替わらない（とくせいなし状態でも）
- ひんしになっても発動する（特性交換後にひんし処理）
- みがわり状態では発動しない
- 同士うちでさまようたましい同士 → 同特性を交換する
- ぼうごパットにより発動が防がれる
