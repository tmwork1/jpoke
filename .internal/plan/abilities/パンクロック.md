# 特性実装計画: パンクロック

更新日: 2026-07-12

## スコープ

- 仕様書: `.internal/spec/abilities/パンクロック.md`
- 実装状態: 実装済み
- 実装方針: `ON_CALC_POWER_MODIFIER` で自身が音の技を使用したときの威力を5325/4096倍にし、
  `ON_CALC_DAMAGE_MODIFIER` で自身が音の技を受けたときのダメージを2048/4096倍（半減）にする。
  どちらもダメージ計算内部（攻撃技のみ通過する）イベントのため、変化技には自然に発動しない。
  かたやぶり系との相互作用は `flags={"mold_breaker_ignorable"}` を AbilityData 全体に付与し、
  既存の `かたやぶり_disable_foe_ability` / `きんしのちから_disable_foe_ability` の
  「防御側の特性を丸ごと無効化する」仕組みに委ねる（こおりのりんぷん等の既存特性と同一パターン）。

## 実装ゴール（完了定義）

- 自身が音の攻撃技を使用したとき、威力が5325/4096倍になる。
- 自身が音の攻撃技を受けたとき、ダメージが2048/4096倍（半減）になる。
- 音の技であっても、変化技を使用・被弾したときは効果が発動しない。
- 音の攻撃技を受けても、攻撃側がかたやぶり系の効果を持つ場合は被ダメ軽減が発動しない
  （威力上昇側はかたやぶりと無関係のため対象外）。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_CALC_POWER_MODIFIER` — ダメージ計算内の威力補正イベント（攻撃技のみ通過）。
  `.internal/spec/turn.md` に未掲載のダメージ計算内部イベントのため、かたいツメ・がんじょうあご等
  既存の威力補正系特性ハンドラと同じ登録パターン（priority指定なし）に合わせる。
- `Event.ON_CALC_DAMAGE_MODIFIER` — ダメージ計算内のダメージ補正イベント（攻撃技のみ通過）。
  同様に既存のこおりのりんぷん等のダメージ軽減系特性ハンドラに合わせ、priority指定なし。

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `パンクロック` に `flags={"mold_breaker_ignorable"}` と
  上記2イベントの handlers を登録する（実装済み）。
- `src/jpoke/handlers/ability.py` — `パンクロック_modify_power`（attacker:self）、
  `パンクロック_reduce_damage`（defender:self）を実装する（実装済み）。
- `tests/abilities/test_ability_ha.py` — テストを追加・確認する。

## 実装ステップ

1. 仕様書 `.internal/spec/abilities/パンクロック.md` で発動条件・例外条件を確認する。
2. `src/jpoke/handlers/ability.py` に `パンクロック_modify_power` / `パンクロック_reduce_damage` を実装する。
   `ctx.move.has_flag("sound")` で音技判定し、`apply_fixed_modifier` で固定小数点補正を適用する。
3. `src/jpoke/data/ability.py` の `パンクロック` に `flags={"mold_breaker_ignorable"}` と
   `ON_CALC_POWER_MODIFIER` (attacker:self) / `ON_CALC_DAMAGE_MODIFIER` (defender:self) を登録する。
4. `tests/abilities/test_ability_ha.py` にテストを追加し、全テストが通ることを確認する。

## テスト観点

1. 音の攻撃技（例: バークアウト）を使用したとき、威力が5325/4096倍になる。
2. 音の攻撃技を受けたとき、ダメージが2048/4096倍（半減）になる。
3. 音の変化技（例: うたう）を使用・被弾したときは、威力補正・ダメージ補正のいずれも発動しない
   （`ON_CALC_POWER_MODIFIER` / `ON_CALC_DAMAGE_MODIFIER` 自体が発火しないことを確認する）。
4. かたやぶり持ちの音の攻撃技を受けたときは被ダメ軽減が無効化される。
