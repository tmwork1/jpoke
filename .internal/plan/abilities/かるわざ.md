# 特性実装計画: かるわざ

更新日: 2026-07-11
実装状況: **✅ 完了**

## 仕様参照

- `.internal/spec/abilities/かるわざ.md`

## 実装ゴール

✅ 場に出た後に自分のアイテムが消費・奪取・除去されたとき、すばやさが2倍になる。
✅ アイテムを再取得するとすばやさが元に戻る。
✅ 場に出た時点からアイテムがない場合は発動しない。
✅ 引っ込んで再び場に出た場合はリセットされ、未発動扱いになる。
✅ かがくへんかガス等で特性が無効化されている間は効果が止まり、無効化中にアイテムを失っても
  発動状態には移行しない。無効化解除後は、無効化前の発動状態（発動中／未発動）がそのまま
  引き継がれる。

## 実装方針

✅ `ability.state` で状態管理を実装
  - `""`: 未初期化（交代で戻ってきた直後など）
  - `"idle"`: 未発動状態（アイテムを持っている。消失で発動可能）
  - `"active"`: 発動状態（アイテムを消失し、すばやさ2倍が適用中）
  - `"inactive"`: 発動不可状態（在場中にアイテムを持ったことがない、またはかるわざを得た
    時点で既にアイテムを持っていない）

✅ `Event.ON_SWITCH_IN` / `Event.ON_ABILITY_ENABLED` で `state == ""` のときのみ
  アイテム所持状況から初期状態（`idle` / `inactive`）を設定する（`かるわざ_init_state`）。
  - `ON_SWITCH_IN` 時は交代処理で必ず `state` がリセット（`""`）されているため、常に初期化される。
  - `ON_ABILITY_ENABLED` は「新たにかるわざを得た場合」（スキルスワップ等で `Ability`
    インスタンスが差し替わり `state` が `""` になる場合）と、「かがくへんかガス解除等で
    同一インスタンスが無効化から復帰した場合」（`state` は既に確定済み）の両方で発火する。
    後者では `state` を保持し再判定しないことで、無効化前の発動状態を引き継ぐ
    （仕様「かがくへんかガスが解除されればかるわざは再度発動する」に対応）。
✅ `Event.ON_ITEM_LOST`（`item_manager._change_item` がアイテム喪失時に発火する既存イベント）
  で `state == "idle"` のときのみ `"active"` へ遷移させる（`かるわざ_activate_on_item_lost`）。
  - 特性が無効化されている間はこのハンドラ自体が発火しない（`event_manager` が
    `subject.ability.enabled_ignoring(...)` でスキップする）ため、無効化中にアイテムを
    失ってもこの遷移は起きない。これにより仕様「かるわざが発動していないときに
    かがくへんかガスが発動し、その間にアイテムを消費してからかがくへんかガスが解除された
    場合、かるわざは発動しない」に対応する。
  - `remove_item` / `consume_item` / `swap_items` / `take_item` 等、道具を失うすべての経路は
    最終的に `_change_item` を経由するため、なげつける・はたきおとす・どろぼう・トリック・
    ふしょくガス等どの手段で失った場合でも一律に検知できる。
✅ `DomainEvent.ON_CALC_SPEED` で `state == "active"` のときすばやさ実数値を2倍にする
  （ランク補正ではない）。アイテムを再取得していた場合はここで `"idle"` に戻す
  （`かるわざ_modify_speed`）。
✅ `Event.ON_ABILITY_DISABLED` のハンドラは登録しない。`state` は無効化中も
  同一インスタンス上に保持されるため、明示的なリセットは不要（リセットすると
  無効化解除後に発動状態を引き継げなくなり、上記の仕様に反する）。

### priority について

`.internal/spec/turn.md` にかるわざの記載はなく、`ON_SWITCH_IN` テーブルにも「かるわざ」の
行はない（アナウンス等の他効果と競合しないため）。`ON_ABILITY_ENABLED` / `ON_ITEM_LOST` /
`DomainEvent.ON_CALC_SPEED` も turn.md 未掲載のイベントであり、既存の同種ハンドラ
（`AbilityHandler` のデフォルト優先度）に倣いデフォルト優先度のまま登録する。

## 実装完了内容

### src/jpoke/data/ability.py
```python
"かるわざ": AbilityData(
    handlers={
        Event.ON_SWITCH_IN: h.AbilityHandler(
            h.かるわざ_init_state,
            subject_spec="source:self",
        ),
        Event.ON_ABILITY_ENABLED: h.AbilityHandler(
            h.かるわざ_init_state,
            subject_spec="source:self",
        ),
        Event.ON_ITEM_LOST: h.AbilityHandler(
            h.かるわざ_activate_on_item_lost,
            subject_spec="source:self",
        ),
        DomainEvent.ON_CALC_SPEED: h.AbilityHandler(
            h.かるわざ_modify_speed,
            subject_spec="source:self",
        ),
    }
),
```

### src/jpoke/handlers/ability.py
- `かるわざ_init_state()`: 入場時・特性有効化時（未初期化の場合のみ）にアイテム有無から
  状態を設定
- `かるわざ_activate_on_item_lost()`: アイテムを失った瞬間（特性が有効な場合のみ）に
  `idle` → `active` へ遷移
- `かるわざ_modify_speed()`: `active` 中のすばやさ2倍化と、アイテム再取得時の
  `active` → `idle` 復帰

## テスト観点 ✅ 完了

✅ 1. アイテム消費（きのみ使用）後にすばやさが2倍になる。
✅ 2. はたきおとす・トリック等でアイテムを失ってもすばやさが2倍になる。
✅ 3. リサイクルでアイテムを再取得するとすばやさが元に戻る。
✅ 4. 場に出た時点からアイテムがない場合は発動しない。
✅ 5. 引っ込んで再び出た場合、アイテムがなくても発動しない（リセット確認）。
✅ 6. 発動中にかがくへんかガスが発動しても、解除後は発動状態が継続する（すばやさ2倍のまま）。
✅ 7. 未発動時にかがくへんかガスが発動し、その間にアイテムを失っても、ガス解除後に発動しない。
