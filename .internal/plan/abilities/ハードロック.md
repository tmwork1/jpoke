# 特性実装計画: ハードロック

更新日: 2026-07-12

## 仕様参照

- `.internal/spec/abilities/ハードロック.md`

## 実装ゴール

- 相手から受ける効果バツグンの技のダメージを3/4倍にする。
- タイプ相性自体が変わる要因（もりののろい/ハロウィンで追加されたタイプ、
  タールショットで追加された弱点、フリーズドライ/フライングプレスによる
  相性変化、らんきりゅうによる弱点無効化）は正しく反映しつつ、相性自体は
  変わらない効果（弱点半減のきのみ等）には影響されないようにする。
- かたやぶり・かがくへんかガス等で無効化される（`mold_breaker_ignorable` フラグで
  一元的に処理されるため、ハンドラ側での個別判定は不要。
  `.claude/loop/_common.md` および memory の既存方針に従う）。

## 実装方針

- `Event.ON_CALC_DAMAGE_MODIFIER`（`AttackContext`, `subject_spec="defender:self"`）で
  `battle.query.is_super_effective(ctx)` を判定し、真なら `value` に固定小数点で
  3072/4096（0.75倍）を乗算する。
  - `is_super_effective` は `damage_calculator.calc_def_type_modifier(ctx)` を経由し、
    `Event.ON_CALC_DEF_TYPE_MODIFIER` の結果（タイプ相性の一元計算。もりののろい・
    タールショット・フリーズドライ・らんきりゅう等の相性変化要因を含む）を参照する。
    このためハードロック側で個別の相性変化要因を判定する必要はない。
  - 弱点半減のきのみ等、相性自体を変えずダメージだけを軽減する効果は
    別の `ON_CALC_DAMAGE_MODIFIER` ハンドラとして独立に乗算されるため、
    `is_super_effective` の判定には影響しない（乗算の可換性によりどちらが
    先に適用されても最終値は変わらない）。
  - `Event.ON_CALC_DAMAGE_MODIFIER` は `.internal/spec/turn.md` に未掲載
    （ダメージ計算内部イベント）のため、既存の同種ハンドラ
    （`フィルター`/`プリズムアーマー`/`ブレインフォース` 等）と同様に
    priority のデフォルト値をそのまま用いる。
- `AbilityData` に `flags={"mold_breaker_ignorable"}` を付与し、シャドーレイ・
  フォトンゲイザー・かたやぶり等の無視技で無効化される（プリズムアーマーとの違い＝
  プリズムアーマーは `mold_breaker_ignorable` を持たず、かたやぶりに対しても有効）。
- ハンドラ関数 `ハードロック_reduce_effective` は `フィルター` / `プリズムアーマー` と
  完全に同一効果（0.75倍軽減）のため共用する。

## テスト観点

1. 効果バツグンの技を受けたとき、`damage_modifier` が3072（0.75倍）になる。
2. かたやぶりを持つ相手の攻撃に対しては無効化され、`damage_modifier` が4096のままになる。
3. シャドーレイ・フォトンゲイザー等の特性無視技に対しても無効化される
   （`test_move_sa.py` / `test_move_fa.py` に既存カバレッジあり）。

## 保留・論点

- なし（実装済み。相性変化要因の判定は `ON_CALC_DEF_TYPE_MODIFIER` に一本化されており
  個別分岐の追加は不要）。
