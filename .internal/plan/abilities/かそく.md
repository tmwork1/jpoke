# 特性実装計画: かそく

更新日: 2026-07-11

## 仕様参照

- `.internal/spec/abilities/かそく.md`

## 実装ゴール

- 場に出てから最初の行動を選択した後、毎ターン終了時にすばやさを1段階上げる。
- 交代で出たターン・まだ行動を選択していないターンは発動しない。
- すばやさランクが最大（+6）の場合は発動しない（`battle.modify_stats` が変化なしなら False を返すため自然に不発）。

## 実装方針

- `Event.ON_TURN_END`（`.internal/spec/turn.md` の ON_TURN_END 表で priority=150、ムラっけ・スロースタート等と同格）で
  `Pokemon.acted_since_switch_in` を確認し、真の場合にすばやさ+1を適用する。
  - `Pokemon.executed_move is not None` は使わない。`executed_move` は技が
    `ON_TRY_ACTION` を通過し実際に実行された場合のみセットされるため、状態異常
    （まひ・ねむり・こおり・こんらん自傷等）でPPを消費せず行動が失敗したときに
    Falseのままとなり、「最初の行動が状態異常/状態変化によりPPを消費することなく
    失敗したときでも発動する」という仕様に反する。
  - `acted_since_switch_in` は `MoveExecutor.run_move` の `finally` 節で行動の成否に
    関わらず True になり、`Pokemon.reset_on_switch_in` で場に出るたびにリセットされる
    ため、仕様の「場に出てから一度でも行動を選択した」を正しく表現できる。
- 交代直後のターンは `acted_since_switch_in` が未セットのため自然に不発になる。

## 変更候補

- `src/jpoke/data/ability.py`
- `src/jpoke/handlers/ability.py`

## テスト観点

1. 初回行動後のターン終了時にすばやさ+1される。
2. 交代で出たターン（行動前）は発動しない。
3. すばやさ最大時は発動しない。
4. 交代で引っ込んで再度出直した場合、再び1回行動するまで発動しない。
5. 最初の行動がまひ（フルパラ）でPPを消費せず失敗した場合でも発動する。
