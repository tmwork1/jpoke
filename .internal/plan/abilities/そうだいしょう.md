# 特性実装計画: そうだいしょう

更新日: 2026-07-11

## 仕様参照

- `.internal/spec/abilities/そうだいしょう.md`

## 実装ゴール

- 特性発動時点（場に出たとき、または他の特性からそうだいしょうに書き変わったとき）で、
  その戦闘で自分側がひんしになった延べ回数×10%の威力補正率を確定させる（最大 +50%）。
- 威力補正率は発動時点で確定し、以後その戦闘中に味方がひんしになっても変化しない。
- 一度ひんしになった味方が復活しても延べ回数は減らない。復活後に再度ひんしになった場合は
  さらに加算される（＝現在ひんし状態の頭数ではなく、延べ発生回数をカウントする）。
- ひんし延べ回数が0のときはアナウンスなし・補正なし。

## 実装方針

- ひんし延べ回数は特性側では持たず、`PlayerState.total_fainted_count`（プレイヤー単位の
  累積カウンター）で管理する。`StatusManager.modify_hp` が `target.fainted` に遷移した
  瞬間に `+1` する（かたきうち用 `ally_fainted_turn` の更新と同じ箇所）。これにより、
  さいきのいのり等でHPが回復して `fainted` が解除されても、過去の発動回数はカウントに
  残り続ける。
- `Event.ON_SWITCH_IN` と `Event.ON_ABILITY_ENABLED`（他特性からの書き変わり）の両方に
  同一ハンドラ `そうだいしょう_announce_on_entry` を登録し、発動時点で
  `min(state.total_fainted_count, 5)` を `mon.ability.count`（int）に確定保存する
  （`いかく` と同じ「入場 + 特性書き変わり」の二重登録パターン）。
- `Event.ON_CALC_POWER_MODIFIER` の `そうだいしょう_modify_power` は `ctx.attacker.ability.count`
  を読み取るだけで、その場で再計算はしない。
  `apply_fixed_modifier(value, 4096 * (10 + count) // 10)` で適用する
  （`count` は発動時点で 0〜5 にクランプ済みのため、ここでの追加クランプは不要）。

## 変更候補

- `src/jpoke/handlers/ability.py` — `そうだいしょう_announce_on_entry`、`そうだいしょう_modify_power` を実装する
- `tests/abilities/test_ability_sa.py` — テストを追加する

## テスト観点

1. 味方1体ひんし状態で入場 → 攻撃技の威力が 1.1 倍。
2. 味方5体（以上）ひんし状態で入場 → 威力が 1.5 倍（上限 50%）。
3. 入場後に味方がひんしになっても、その戦闘中は補正が変わらない（発動時点で確定）。
4. ひんし味方なし → アナウンスなし・補正なし。
5. 一度ひんしになった味方が回復（復活）しても、延べ回数は減らない
   （再度そうだいしょうを場に出したときの補正率が下がらない）。

```python
# core/status_manager.py: modify_hp 内、target.fainted になった分岐
player_states[player].ally_fainted_turn = battle.turn
player_states[player].total_fainted_count += 1  # 復活しても減らない延べ回数

# handlers/ability.py
def そうだいしょう_announce_on_entry(battle, ctx, value):
    mon = ctx.source
    player = battle.get_player(mon)
    state = battle.player_states[player]
    fainted_count = min(state.total_fainted_count, 5)
    mon.ability.count = fainted_count
    if fainted_count:
        _announce_ability_triggered(battle, mon)
    return HandlerReturn(value=value)

def そうだいしょう_modify_power(battle, ctx, value):
    fainted_count = ctx.attacker.ability.count
    if not fainted_count:
        return HandlerReturn(value=value)
    modifier = 4096 * (10 + fainted_count) // 10
    return HandlerReturn(value=apply_fixed_modifier(value, modifier))
```
