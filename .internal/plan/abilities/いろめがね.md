## 実装ゴール

✅ 使用した技が「効果は今ひとつ」（タイプ相性 0 < 倍率 < 1）だったとき、与えるダメージを2倍にする。
✅ 相性判定そのものは変化させない（無効化・抜群判定には影響しない）。
✅ タイプ相性補正はもりののろい/ハロウィン/タールショット/フリーズドライ/フライングプレス/
   らんきりゅう/テラスシェルなど「最終的な相性」を変える効果をすべて反映した後の値で判定する。

## 実装方針

✅ **いまひとつ判定とダメージ2倍化**
- `Event.ON_CALC_DAMAGE_MODIFIER` で `attacker:self` として登録する（攻撃側の特性のため `AttackContext` の
  `attacker` ロールに合わせる）
- `battle.query.is_not_very_effective(ctx)` を使って判定する
  - 内部で `damage_calculator.calc_def_type_modifier(ctx)` を呼び、`0 < modifier/4096 < 1` を満たす場合のみ
    True を返す（無効化時は modifier=0 のため False になり発動しない）
  - `calc_def_type_modifier` は `Event.ON_CALC_DEF_TYPE_MODIFIER` の結果であり、タイプ追加・相性上書き系の
    効果（もりののろい、ハロウィン、タールショット、フリーズドライ、フライングプレス、らんきりゅう、
    テラスシェル等）を経由した「最終的な相性」を反映するため、これらの効果との相互作用は追加実装不要。
- 判定が真のとき、`value` を `apply_fixed_modifier(value, 8192)`（2倍固定小数点補正）で更新する。

### priority について

- `ON_CALC_DAMAGE_MODIFIER` は `.internal/spec/turn.md` に priority の記載がない（ダメージ計算内部イベントの
  ため対象外）。既存の同種ハンドラ（`かんそうはだ_modify_fire_damage` など）と同様に `priority` を
  指定せず既定値（100）を使う。
