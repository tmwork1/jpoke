# 特性実装計画: てんきや

更新日: 2026-07-12

## ユーザーコメント
ポワルンの姿違いをdata/ps-champ-ja/pokedex.jsonに追加した


## スコープ

- 優先度: H
- 仕様書: `.internal/spec/abilities/てんきや.md`
- 実装状態: 実装済み
- 実装方針: `ON_SWITCH_IN` と `ON_FIELD_CHANGE`（天気変化）で天気に対応したフォルムへ即時フォルムチェンジする。ノーてんき/エアロック発動中は通常フォルムに戻す。かがくへんかガス発動中は変化できない（`AbilityHandler` が特性無効化中は発火しない共通機構を利用）。かがくへんかガスの効果が切れた瞬間は `ON_ABILITY_ENABLED` で即座にフォルムチェンジする。

## 実装ゴール（完了定義）

- にほんばれ/おおひでり → ほのおタイプの「たいようのすがた」
- あめ/おおあめ → みずタイプの「あまみずのすがた」
- ゆき → こおりタイプの「ゆきぐものすがた」
- それ以外 → ノーマルタイプの「ポワルンのすがた」
- ノーてんき/エアロック発動中は通常フォルムに固定される。
- かがくへんかガス発動中はフォルムチェンジできず、効果が切れた瞬間に即座にフォルムチェンジできる。
- フォルムチェンジが発生したとき、みずびたし/もりののろい/ハロウィン/ミラータイプによるタイプ変化を解消する。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_SWITCH_IN`（priority=140: turn.md L111 参照、他のフォルムチェンジ特性と同じ値） — 登場時に現在の天気に合わせたフォルムに変化する
- `Event.ON_FIELD_CHANGE` — 天気が変わったとき即座にフォルムを切り替える（turn.md未掲載イベント。同種の`ぎたい`/`アイスフェイス`ハンドラに合わせ priority 省略）
- `Event.ON_ABILITY_ENABLED` — かがくへんかガス等の効果が切れて特性が再び有効になった瞬間に即座にフォルムを同期する（turn.md未掲載イベント。同種の`ぎたい`ハンドラに合わせ priority 省略）

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `てんきや` に `ON_SWITCH_IN` / `ON_FIELD_CHANGE` / `ON_ABILITY_ENABLED` の handlers を登録する
- `src/jpoke/handlers/ability.py` — `てんきや_sync_form` を実装する。フォルムが実際に変わったときは `みずびたし`/`もりののろい`/`ハロウィン` の volatile を解除し、`move_override_types`（ミラータイプ）をクリアする
- `tests/abilities/test_ability_ta.py` — テストを追加する

## 実装ステップ

1. 仕様書で天気 → フォルム/タイプのマッピングを確認する。
2. `てんきや_sync_form` を実装する（現在の天気に対応したフォルムへ `mon.set_form`）。ノーてんき/エアロック発動中は常に「ポワルンのすがた」に戻す。フォルムが実際に変わった場合はみずびたし/もりののろい/ハロウィンのvolatileを解除し、move_override_typesをクリアする。
3. `src/jpoke/data/ability.py` の `てんきや` に `ON_SWITCH_IN`（priority=140）、`ON_FIELD_CHANGE`、`ON_ABILITY_ENABLED` の handler を登録する。
4. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. にほんばれ状態で登場するとたいようのすがたになる。
2. バトル中に天気があめに変わると即座にあまみずのすがたになる。
3. ノーてんき/エアロックが発動中はフォルムが変化しない（通常フォルムに固定）。
4. かがくへんかガス発動中は天候が変わってもフォルムチェンジせず、かがくへんかガスの効果が切れた瞬間に即座に天候に対応したフォルムへ変化する。
5. みずびたしでみずタイプ化されたポワルンが天候変化でフォルムチェンジすると、みずびたしの効果が解消され天候対応タイプになる。
