# 特性実装計画: ほうし

更新日: 2026-07-12

## 仕様参照

- `.internal/spec/abilities/ほうし.md`

## 実装ゴール

- 直接攻撃を受けたとき、30%の確率でどく・まひ・ねむりのいずれかにする。
  確率内訳: どく 9%、まひ 10%、ねむり 11%（合計 30%）。
- 直接攻撃でない技（ぼうごパット所持者への攻撃・えんかく/パンチグローブで
  非接触化された技を含む）では発動しない。
- 攻撃側がくさタイプ・特性ぼうじん・アイテムぼうじんゴーグルのいずれかを
  持つ場合は無効（`ほうし` ハンドラ内で直接チェックする。粉・胞子技の
  `powder` フラグ判定とは別経路のため、ailment_manager 側の汎用ロジックでは
  カバーされない）。
- ちからずくでは追加効果とは扱われないため、ちからずくの無効化対象外。
- しんぴのまもり・さわぐ状態による無効化は `ailment_manager.apply` が発火する
  `ON_BEFORE_APPLY_AILMENT` を経由して汎用的に処理される（ほうし側での
  個別対応は不要）。
- 連続攻撃技のヒット中に新たにねむり状態を付与した場合、その連続攻撃技は
  直ちに中断される（`MoveExecutor` の連続技ループ側で汎用対応する）。

## 実装方針

- `Event.ON_DAMAGE_HIT`、`subject_spec="defender:self"` で実装。
- `battle.query.is_contact_reaction(ctx)` で接触判定（ぼうごパット等の反応系
  無効化も含む）を行った後、攻撃側のくさタイプ／ぼうじん／ぼうじんゴーグルを
  チェックして無効なら何もしない。
- 乱数で状態異常を決定して `battle.ailment_manager.apply` を呼ぶ。
- 状態異常の無効化（まひ耐性・しんぴのまもり等）は `ailment_manager.apply` 側で
  処理される。
- 連続攻撃技の中断は `src/jpoke/core/move_executor.py` の
  `_execute_move` 内、連続技ヒットループに汎用ロジックとして追加する
  （ループ開始前の睡眠状態を記録し、ヒット後に新規にねむり状態へ移行して
  いれば break する。ねごとで元々ねむり状態のまま行動しているケースは
  対象外とする）。

## テスト観点

1. 直接攻撃受けて 30% 以内で状態異常が発動する（乱数境界値で どく/まひ/ねむり）。
2. 非接触技では発動しない。
3. どく耐性・まひ耐性を持つ相手には該当異常が付与されない。
4. 攻撃側がくさタイプ・ぼうじん・ぼうじんゴーグル所持のいずれかの場合は発動しない。
5. 連続攻撃技のヒット中にねむり状態を付与した場合、それ以降のヒットが中断される。

```python
def ほうし_maybe_inflict_ailment_on_contact(battle, ctx, value):
    if (
        not battle.query.is_contact_reaction(ctx)
        or ctx.attacker.has_type("くさ")
        or ctx.attacker.ability.name == "ぼうじん"
        or ctx.attacker.has_item("ぼうじんゴーグル")
    ):
        return HandlerReturn(value=value)
    r = battle.random.random()
    if r < 0.09:
        ailment = "どく"
    elif r < 0.19:
        ailment = "まひ"
    elif r < 0.30:
        ailment = "ねむり"
    else:
        return HandlerReturn(value=value)
    battle.ailment_manager.apply(ctx.attacker, ailment, source=ctx.defender)
    return HandlerReturn(value=value)
```
