# ビビリだま 実装計画

## 効果概要
いかく でこうげきが変化したとき、すばやさ（S）を +1 する（1度発動すると消費）。
あまのじゃく所持者はいかくで逆にこうげきが上昇するが、その場合も発動する。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_MODIFY_STAT` |
| subject_spec | `"target:self"` |
| priority | 100（デフォルト） |
| once | False（`battle.item_manager.consume_item` で都度消費するため `once` は不要） |
| 消費条件 | すばやさが実際に+1されたときのみ `_announce_and_consume_item(battle, mon)` |

### ハンドラロジック（疑似コード）

```python
def ビビリだま_boost_speed_on_intimidate(battle, ctx, value):
    mon = ctx.target
    if (
        "atk" in value
        and ctx.stat_change_reason == "いかく"
        and battle.modify_stats(mon, {"spe": +1})
    ):
        _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)
```

`value` は `battle.modify_stats` が返す `actual_changes`（`{stat: 実際の変化量}`）であり、
`Event.ON_MODIFY_STAT` はランクが実際に変化した場合のみ発火する
（`core/status_manager.py` の `modify_stats` 参照）。したがって:

- `"atk" in value` は「こうげきランクが実際に変化した」ことを意味する。符号を条件にしないのは、
  あまのじゃく所持者がいかくを受けた場合にこうげきが実際には上昇する（符号が反転する）ため。
  この場合も本アイテムは発動する（`.internal/spec/items/ビビリだま.md` 参照）。
- すでにこうげきが最低ランク（あまのじゃくの場合は最高ランク）まで下がっていて実際には
  ランクが変化しなかった場合、`actual_changes` に `"atk"` キーが含まれないため自然に不発になる。
- `battle.modify_stats(mon, {"spe": +1})` の戻り値が空 dict（すばやさが既に最大、または
  あまのじゃく所持者ですばやさが既に最小のため+1が-1に反転してさらに不発）の場合は
  発動・消費しない。あまのじゃく所持者への符号反転は `あまのじゃく_reverse_stat`
  （`Event.ON_BEFORE_MODIFY_STAT`, `subject_spec="target:self"`）が自動的に適用するため、
  本ハンドラ側で特別な分岐は不要。

## 修正した既存実装の問題点
- 旧実装は `value.get("atk", 0) < 0` を条件にしていたため、あまのじゃく所持者がいかくを
  受けてこうげきが上昇したケースで発動しなかった（`"atk" in value` に修正）。
- 旧実装は `battle.modify_stats(mon, {"spe": +1})` の戻り値を確認せず無条件に
  `_announce_and_consume_item` を呼んでいたため、すばやさが既に最大（またはあまのじゃく
  所持者ですばやさが既に最小）で発動しないはずのケースでもアイテムが消費されていた。

## 既知の未実装範囲（本レビューではスコープ外）
一次情報には以下の詳細な相互作用が記載されているが、単体で大規模な事前イベント設計変更を
要するため本レビューでは対応しない（`.internal/spec/items/ビビリだま.md` に経緯を残す）。

- しろいきり状態、特性かいりきバサミ・クリアボディ・しろいけむり・メタルプロテクト・
  フラワーベール・きもったま・せいしんりょく（第八世代以降）でいかくの効果自体が
  無効化された場合も、本来はビビリだまが発動する。現在の実装は `Event.ON_MODIFY_STAT`
  （こうげきランクが実際に変化した後にのみ発火）を起点にしているため、これらの特性が
  `Event.ON_BEFORE_MODIFY_STAT` でこうげきの変化量を除去した場合（`_block_stat_drop_by_foe`
  等）は `"atk"` キー自体が `actual_changes` に現れず、本アイテムは発動しない。
  正しく実装するには、いかく側に「発動確定」を通知する専用イベント（こうげきランク低下の
  成否とは独立に、いかくを受けたこと自体を通知するイベント）を追加する必要があり、
  影響範囲が本アイテム単体を超えるため見送る。
  - なお、どんかん・マイペースは本コードベースでは現状いかくのランク低下自体を
    無効化する実装がないため、この問題は顕在化しない。
- 特性ミラーアーマーでいかくを跳ね返した場合、跳ね返した側が持つビビリだまは
  その場では発動せず、以後の能力変動時に発動する（保留機構）。現在は未実装。
- クリアチャームでいかくの効果自体を防いだ場合も、上記と同様の理由で本来は
  ビビリだまが発動するが未実装。
- きょうせい（Symbiosis）を介した複数ポケモン間でのビビリだまの受け渡しは、
  対象特性自体が本コードベースで未実装（`ABILITIES["きょうせい"] = AbilityData()`）
  かつダブルバトル以降の仕様であるため対象外。

## 注意
だっしゅつパック（`ON_MODIFY_STAT` でランク低下を検知）のパターンを参照する。

## テスト観点
- いかく によってこうげきが下がったとき S が +1 されること
- あまのじゃく所持者がいかくを受けてこうげきが上昇したときも S が +1 されること
- こうげきが既に最低ランクで実際に変化しなかった場合は発動しないこと
- すばやさが既に最大ランクの場合は発動・消費されないこと
- 発動後にアイテムが消費されること
