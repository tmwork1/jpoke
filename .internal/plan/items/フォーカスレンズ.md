# フォーカスレンズ 実装計画

## 効果概要
そのターンに、すでに行動している相手に対して使う技の命中率を1.2倍にする。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_MODIFY_ACCURACY` |
| subject_spec | `"attacker:self"` |
| priority | 100（デフォルト値。明示指定なし） |
| once | False |
| 消費条件 | なし（非消費アイテム） |

### ハンドラロジック（疑似コード）
```python
def フォーカスレンズ_boost_accuracy_second(battle, ctx, value):
    # 一撃必殺技には効果がない
    if ctx.move.has_flag("ohko"):
        return HandlerReturn(value=value)

    # 交代してきたばかりで技を未使用の相手には効果がない（第五世代以降の仕様）
    defender_player = battle.get_player(ctx.defender)
    if battle.player_states[defender_player].has_switched:
        return HandlerReturn(value=value)

    # アナライズ特性と同様の後攻判定ロジック（action_orderが未確定の場合のフォールバックあり）
    attacker_player = battle.get_player(ctx.attacker)
    is_second = battle.query.is_second_actor(attacker_player)
    if is_second is None:
        is_second = ctx.defender.executed_move is not None

    if is_second:
        value = apply_fixed_modifier(value, 4915)  # 1.2倍(4915/4096)
    return HandlerReturn(value=value)
```

### 実装上の注意
- `battle.query.is_second_actor` は `turn_controller.action_order`（このターンの行動順。交代・技いずれの行動も含む）を参照して先後を判定する。
- 相手が「このターンに交代してきた」場合、`action_order` 上は相手が先に行動した扱いになるが、
  フォーカスレンズの効果は発動しない（第五世代以降の仕様）。そのため `player_states[defender_player].has_switched`
  で明示的に除外する。
- `action_order` が未確定（`is_second_actor` が `None`）の場合は、アナライズ同様に
  `ctx.defender.executed_move is not None` をフォールバックとして用いる（`has_switched` は前段で除外済みのため
  フォールバック側では判定しない）。
- 一撃必殺技（`ohko` フラグ）は命中率が固定形式で扱われるため、他の命中率上昇効果同様に対象外とする
  （`しょうりのほし`・`はりきり`・`ふくがん`・`ひかりのこな` 等と同じパターン）。

## テスト観点
- 後攻時（自分が遅く、相手がすでに技を使用済み）に命中率が 1.2 倍になること
- 先攻時（自分が速い）は命中率が変化しないこと
- 一撃必殺技には効果がないこと
- 相手がそのターンに交代してきた場合は効果がないこと（技を使用していないため）
- 非所持では命中率が変化しないこと
