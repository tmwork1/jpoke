# ソクノのみ 実装計画

更新日: 2026-07-05

## スコープ

- 仕様書: `.internal/spec/items/_タイプ半減きのみ.md`（ソクノのみ節）
- 実装状態: 実装済み

## 効果概要

- 効果バツグンの`でんき`タイプの技を受けたとき、ダメージを1/2にして消費する（1度きり）。
- そうでん・プラズマシャワー・エレキスキンの効果ででんきタイプになった攻撃技は、元のタイプに
  関係なくソクノのみの効果対象になる。
  - でんきタイプになったどろぼう・ほしがる・ついばむ・むしくい・やきつくすの攻撃を抜群で
    受けた場合、ソクノのみの効果が優先して発動してダメージを半減し、技の効果は受けない。
  - でんきタイプになったはたきおとすを抜群で受けた場合、はたきおとすの威力は1.5倍になるが、
    ソクノのみの効果でダメージは半減できる。
  - でんきタイプになったポルターガイストの攻撃を受けた場合、ソクノのみでダメージを半減して
    攻撃を受ける。
- これらの相互作用は `ctx.move.type` を参照する共通処理のみで実現され、個別対応は不要。
  ソクノのみの `ON_CALC_DAMAGE_MODIFIER` ハンドラは技の後続効果（どろぼうの奪取・はたきおとす
  の除去等）より先に発火し、そこでアイテムを消費するため、技側の効果発動時には
  `has_item()` が False になっている。

## 実装方針

- 通常戦闘: `Event.ON_CALC_DAMAGE_MODIFIER`（`subject_spec="defender:self"`）で発火。
  タイプ半減きのみ共通処理 `handlers/item.py` の `_modify_super_effective_damage` を
  `type_="でんき"`, `modifier=2048/4096` で呼び出す。
  - `ctx.move.type == "でんき"` かつ `battle.damage_calculator.calc_def_type_modifier(ctx) > 4096`
    で効果バツグン判定する。
  - priority: 他のタイプ半減きのみ・ダメージ補正系アイテムと同じ既定値（`.internal/spec/turn.md` に
    `ON_CALC_DAMAGE_MODIFIER` 個別の優先度記載なし。ダメージ計算内部イベントのため、既存の
    同種きのみハンドラの priority に合わせてデフォルト値を使用）。
  - 条件成立時に `_announce_and_consume_item` でログ記録とアイテム消費を行う。
- リーサル計算（簡易KO判定用の `LethalEvent`）: `LethalEvent.ON_BEFORE_HIT`（`subject="defender"`）で
  `handlers/lethal.py` の `ソクノのみ_resist_electric` → 共通処理 `_type_resist_berry(..., "でんき")`
  を呼び出す。通常ハンドラ側で既にダメージ半減済みのため、リーサル側ではアイテム消費と
  `item_enabled` の更新のみ行う。

## 登録

  `lethal_handlers`（`LethalEvent.ON_BEFORE_HIT`）を登録済み。
- `fling_power=10`。

  - `damage_modifier == 2048`（1/2倍）であること。
  - 使用後にアイテムが消費されていること（`not has_item()`）。

  仕組み上自動的に成立するため追加実装は不要）。
