# ヤタピのみ 実装計画

## 効果概要
HP が最大値の 1/4 以下になった瞬間に とくこう（C）を +1 する（消費）。
なげつける・むしくい・ついばむ・ほおばる・おちゃかいで消費するときは、残りHPに関わらず発動する
（`ON_HP_CHANGED` に `value=mon.max_hp` を渡す強制発動経由）。
こんらんの自傷ダメージ（`reason="self_attack"`）でHPが発動圏内になっても発動しない
（第五世代以降の仕様）。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_HP_CHANGED` |
| subject_spec | `"target:self"` |
| priority | 100（`.internal/spec/turn.md` に本イベントの記載なし。同種のピンチきのみ実装（チイラのみ等）に合わせる） |
| once | False（Handler登録時の設定は他のピンチきのみと同様。実際の一度きりの消費は `consume_item` によるアイテム剥奪で保証する） |
| 消費条件 | 発動時に `battle.item_manager.consume_item(mon)`（`_announce_and_consume_item` 経由） |

### ハンドラロジック（共通処理 `_boost_on_quarter_hp` を使用）
```python
def ヤタピのみ_boost_spatk(battle, ctx, value):
    """ヤタピのみ: HP1/4以下でとくこう+1。"""
    return _boost_on_quarter_hp(battle, ctx, value, stat="spa", amount=+1)
```

`_boost_on_quarter_hp` の内部ロジック（チイラのみ・カムラのみ・リュガのみ・ズアのみと共通）:
```python
def _boost_on_quarter_hp(battle, ctx, value, stat, amount):
    mon = ctx.target
    forced = value >= mon.max_hp  # なげつける・ほおばる等による強制発動
    if not forced:
        if ctx.hp_change_reason == "self_attack":
            return HandlerReturn(value=value)  # こんらんの自傷ダメージでは発動しない
        if mon.hp * 4 > mon.max_hp:
            return HandlerReturn(value=value)
    if battle.modify_stats(mon, {stat: amount}):  # すでにランクが最大の場合は不発・消費しない
        _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)
```

たんじゅん・じゅくせい（2段階上昇）やあまのじゃく（1段階下降）の影響は、
`ON_BEFORE_MODIFY_STAT` を経由する既存の共通処理（`たんじゅん_double_stat` /
`あまのじゃく_reverse_stat`）が自動的に適用するため、本ハンドラ側での個別対応は不要。

## テスト観点
- HP が 1/4 超 → 1/4 以下になった瞬間にとくこうが +1 されること
- 発動後にアイテムが消費されること
- とくこうランクが +6 のとき発動してもランク上限で止まる（不発・不消費となる）こと
- こんらんの自傷ダメージでHPが1/4以下になっても発動しないこと
