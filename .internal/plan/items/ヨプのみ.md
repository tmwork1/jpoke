# ヨプのみ 実装計画

更新日: 2026-07-05

## スコープ

- 仕様書: `.internal/spec/items/_タイプ半減きのみ.md`（ヨプのみ節）
- 実装状態: 実装済み

## 効果概要

- 効果バツグンの`かくとう`タイプの技を受けたとき、ダメージを1/2にして消費する（1度きり）。
- フライングプレス特有の注意点: フライングプレスは技タイプ自体は`かくとう`のままで、
  `ひこう`タイプとの複合相性（ダメージ計算内部でひこうタイプの相性を追加で掛け合わせる）で
  効果抜群かどうかが決まる。ヨプのみは技タイプが`かくとう`と一致するため、複合込みの相性判定
  （`calc_def_type_modifier`）で抜群になったときのみ発動する。バコウのみ（`ひこう`）は
  フライングプレスの技タイプが`ひこう`ではないため発動しない。

## 実装方針

- 通常戦闘: `Event.ON_CALC_DAMAGE_MODIFIER`（`subject_spec="defender:self"`）で発火。
  タイプ半減きのみ共通処理 `handlers/item.py` の `_modify_super_effective_damage` を
  `type_="かくとう"`, `modifier=2048/4096` で呼び出す。
  - `ctx.move.type == "かくとう"` かつ `battle.damage_calculator.calc_def_type_modifier(ctx) > 4096`
    で効果バツグン判定する。フライングプレスは `フライングプレス_add_flying_type`
    （`Event.ON_CALC_DEF_TYPE_MODIFIER`）がヨプのみのハンドラより先に発火し、ひこう相性を
    掛け合わせた後の値が渡ってくるため、複合相性の判定は共通処理の変更なしに正しく動作する。
  - priority: 他のタイプ半減きのみ・ダメージ補正系アイテムと同じ既定値（`.internal/spec/turn.md` に
    `ON_CALC_DAMAGE_MODIFIER` 個別の優先度記載なし。ダメージ計算内部イベントのため、
    既存の同種きのみハンドラの priority に合わせてデフォルト値を使用）。
  - 条件成立時に `_announce_and_consume_item` でログ記録とアイテム消費を行う。
- リーサル計算（簡易KO判定用の `LethalEvent`）: `LethalEvent.ON_BEFORE_HIT`（`subject="defender"`）で
  `handlers/lethal.py` の `ヨプのみ_resist_fighting` → 共通処理 `_type_resist_berry(..., "かくとう")` を呼び出す。
  通常ハンドラ側で既にダメージ半減済みのため、リーサル側ではアイテム消費と `item_enabled` の更新のみ行う。

## テスト

- `tests/test_item.py` の `test_タイプ半減実`（パラメータ化テスト）で
  `("ヨプのみ", "かくとう", "カビゴン")` を検証。
  - `damage_modifier == 2048`（1/2倍）であること。
  - 使用後にアイテムが消費されていること（`not has_item()`）。
- `test_ヨプのみ_フライングプレスのひこう複合相性が抜群なら発動する`
  くさ単タイプ（キモリ）がフライングプレス（かくとう等倍×ひこう2倍=抜群）を受けたとき発動することを確認。
- `test_ヨプのみ_フライングプレスのひこう複合相性が等倍なら発動しない`
  はがね単タイプ（ギアル）がフライングプレス（かくとう2倍×ひこう0.5倍=等倍）を受けても
  発動しないことを確認。
