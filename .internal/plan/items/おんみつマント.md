# おんみつマント 実装計画

## 効果概要
技の追加効果（secondary effect）を受けない（非消費）。特性りんぷんの持ち物版。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_MODIFY_SECONDARY_CHANCE` |
| subject_spec | `"defender:self"` |
| priority | 100（デフォルト値をそのまま使用。明示指定はしない） |
| once | False |
| 消費条件 | なし（非消費アイテム） |

### ハンドラロジック（疑似コード）
```python
def おんみつマント_negate_secondary_effect(battle, ctx, value):
    # 追加効果の確率を 0 にする。以降のハンドラに上書きされないよう stop_event=True で確定させる。
    return HandlerReturn(value=0, stop_event=True)
```

## 確認事項（実装済み）
- `battle.resolve_secondary_chance(ctx, chance)` は `AttackContext`（`ctx.attacker` / `ctx.defender`）を受け取って
  `Event.ON_MODIFY_SECONDARY_CHANCE` を発火する（`core/battle.py` の型注釈は `EventContext` だが、実際に渡されるのは
  `AttackContext`。`EventContext` と `AttackContext` は共に `BaseContext` を継承する別クラスであり、継承関係にはない）。
  したがって `subject_spec` は `"defender:self"` が正しい（特性りんぷんの登録と同一）。
- `ctx.defender` が追加効果を受ける側（防御側 = アイテム所持者）を指すことを確認済み。
- りんぷん（`handlers/ability.py` の `りんぷん_block_secondary_chance`）も同じ event・同じ役割で
  `subject_spec="defender:self"`, `stop_event=True` を使っており、実装を揃えた。

## 仕様上防げる効果と実装状況
- 通常の追加効果（まひ・やけど等）: `move.py` の `apply_ailment_to_defender` 等が `resolve_secondary_chance` を経由するため対応済み。
- 特性どくしゅ/どくのくさりのどく/もうどく: 両特性のハンドラが `resolve_secondary_chance` を呼ぶため対応済み。
- 特性あくしゅう・道具おうじゃのしるし/するどいキバのひるみ: アイテム版 `flinch_on_hit_10pct` は対応済みだったが、
  特性版 `あくしゅう_maybe_flinch` が `resolve_secondary_chance` を呼んでいなかったため本レビューで修正した。
- なげつけるで投げられたアイテムの追加効果: `なげつける_apply_item_effect`（`handlers/move_attack.py`）が
  `resolve_secondary_chance` を経由せず状態異常/ひるみを直接付与しているため、現状はおんみつマントで防げない。
  `なげつける` 自体が `.internal/progress/move.md` 上「保留」のため、本タスク（アイテム側レビュー）ではスコープ外とし、
  なげつけるの実装完了時に合わせて対応する。
- 自分のランクを上げる追加効果（攻撃技に付随する自己バフ）: `modify_attacker_stats` も同じ
  `resolve_secondary_chance(ctx, chance)` を呼ぶため、防御側がおんみつマントを持つ場合に誤ってブロックされうる
  （`ON_MODIFY_SECONDARY_CHANCE` は効果が自分向けか相手向けかを区別しない）。りんぷん側にも同一の制約があり、
  区別するには `AttackContext` に効果対象を示すフィールドを追加するアーキテクチャ変更が必要になるため、
  本タスクでは既知の制限として記録するに留め、修正はしない。

## テスト観点
- おんみつマント持ちへの追加効果（まひ・やけど等）が発動しないこと
- 本体ダメージ自体は通常通り受けること
- 追加効果の確率 0% 化により完全無効になること
- 特性あくしゅうによるひるみを防ぐこと
- 特性どくしゅによるどく状態を防ぐこと
