# ミクルのみ 実装計画

参照仕様書: `.internal/spec/items/_ランク上昇ピンチきのみ.md`（ミクルのみ節）

## 効果概要
HP が最大値の 25% 以下になったとき消費され、次に使う技の命中率を 1.2 倍（4915/4096倍）にする。
イバンのみと同様、HP 変化直後にフラグを立て、実際のアイテム消費は次の行動時まで遅延する。

## 実装仕様

### イベント 1: HP 閾値トリガー（フラグ付与）

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_HP_CHANGED` |
| subject_spec | `"target:self"` |
| priority | 100（デフォルト） |
| 消費条件 | 発動時点ではアイテムは消費せず `mon.item.count = 1` のフラグのみ立てる |
| 除外条件 | こんらんの自傷ダメージ（`ctx.hp_change_reason == "self_attack"`）では発動しない（第五世代以降の仕様。イバンのみと同様） |

### イベント 2: 命中率 1.2 倍適用（フラグ消費）

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_MODIFY_ACCURACY` |
| subject_spec | `"attacker:self"` |
| priority | 100（デフォルト） |
| 除外条件 | 一撃必殺技（`ctx.move.has_flag("ohko")`）には倍率を適用しない。ただしアイテムは消費される |

### イベント 3: 命中判定のない技を使った場合のフラグ解除

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_END_MOVE` |
| subject_spec | `"attacker:self"` |
| priority | 100（デフォルト） |
| 目的 | 命中率が `None`（必中）の技を使うと `ON_MODIFY_ACCURACY` が発火しないため、行動完了時に未消費のフラグを解除する |
| 除外条件 | 行動不能（状態異常・ひるみ等で `action_success is False`）や、溜め技の溜めターンでは解除しない（効果は持続する） |

### ハンドラロジック（疑似コード）
```python
def ミクルのみ_set_accuracy_flag(battle, ctx, value):
    mon = ctx.target
    if ctx.hp_change_reason == "self_attack":
        return HandlerReturn(value=value)
    hp_after = mon.hp
    hp_before = hp_after + value
    if hp_before * 4 > mon.max_hp and hp_after * 4 <= mon.max_hp:
        mon.item.count = 1
        _announce_item_triggered(battle, mon)
    return HandlerReturn(value=value)


def ミクルのみ_boost_accuracy(battle, ctx, value):
    mon = ctx.attacker
    if mon.item.count == 1:
        battle.item_manager.consume_item(mon)
        if not ctx.move.has_flag("ohko"):
            value = apply_fixed_modifier(value, 4915)  # 1.2倍
    return HandlerReturn(value=value)


def ミクルのみ_clear_flag_after_move(battle, ctx, value):
    mon = ctx.attacker
    if mon.item.count != 1:
        return HandlerReturn(value=value)
    executor = battle.move_executor
    if executor.action_success is False:
        return HandlerReturn(value=value)
    if (
        executor.move_success is None
        and executor.move_applied is None
        and not executor.move_missed
    ):
        return HandlerReturn(value=value)
    battle.item_manager.consume_item(mon)
    return HandlerReturn(value=value)
```

## 注意
- フラグ管理は `mon.item.count`（イバンのみと同じ手法）を用いる。この手法では、フラグが立っている間
  アイテムスロットが実際には空にならないため、「効果発動中に別のミクルのみを再入手した場合はすぐに
  発動しない」という仕様（詳細な仕様参照）が構造上自然に満たされる（再入手経路自体が発生し得ないため）。
- `ON_MODIFY_ACCURACY` は `move.accuracy is None`（必中技）のときは発火しない
  （`core/move_executor.py` の `_check_hit` 参照）ため、イベント3で補完する。

## テスト観点
- HP が 25% 超 → 25% 以下になった瞬間にフラグが立ち、アイテムは消費されずに保持されること
- こんらんの自傷ダメージでは発動しないこと
- フラグが立った後、次の技の命中率が 1.2 倍になり、アイテムが消費されること
- 技が外れた場合でも消費されること
- 一撃必殺技では倍率が適用されないが、消費はされること
- 命中判定のない変化技を使った場合でも効果が消費されること
- 状態異常等で行動できなかった場合は効果が持続すること
