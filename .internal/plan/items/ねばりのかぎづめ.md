# ねばりのかぎづめ 実装計画

## 効果概要
バインド技（まきつく・ほのおのうず等）でバインド状態を発生させたとき、継続ターンを 7 ターンに固定する。

道具の効果は「相手をバインド状態にするとき」（＝バインド付与時点）にしか判定されない。
付与後にねばりのかぎづめを入手しても継続ターンは延びず、逆に付与後に手放しても継続ターンは減らない
（`.internal/spec/items/ねばりのかぎづめ.md` 詳細な仕様、`しめつけバンド`と同様の「付与時確定」挙動）。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_MODIFY_DURATION` |
| subject_spec | `"attacker:self"` |
| priority | 100 |
| once | False |
| 消費条件 | なし（非消費アイテム） |

`ON_MODIFY_DURATION` は `handlers/move_attack.py` の `apply_bind_to_defender`（`AttackContext`）内、
`battle.query.get_volatile_duration` を通じてバインド付与時に1回だけ発火し、
継続ターン数をその場で確定させて `Volatile.count` に保存する。
攻撃側（ねばりのかぎづめ持ち）は `ctx.attacker` に当たるため `subject_spec="attacker:self"`。
`バインド_damage`（ターン終了時処理）では以降イベントを再発火せず、`volatile_manager.tick` で
確定済みのカウントを単純に減算する。

## 発火側

`handlers/move_attack.py` の `apply_bind_to_defender` で `ON_MODIFY_DURATION` を
`battle.query.get_volatile_duration` 経由で発火し、結果を `volatile_manager.apply` の
`count` 引数として渡す（既存実装。`しめつけバンド`の `ON_MODIFY_BIND_DAMAGE` 発火と同じ関数内で行う）。

```python
def apply_bind_to_defender(battle, ctx, value):
    count = battle.query.get_volatile_duration(ctx, "バインド", battle.random.randint(4, 5))
    bind_damage_ratio = battle.events.emit(Event.ON_MODIFY_BIND_DAMAGE, ctx, 1/8)
    battle.volatile_manager.apply(
        ctx.defender, "バインド", count=count, source=ctx.attacker, bind_damage_ratio=bind_damage_ratio
    )
    return HandlerReturn(value=value)
```

`get_volatile_duration` は `[name, count]` の2要素リストで `ON_MODIFY_DURATION` を発火し、
更新後の `count` を返す（`core/query.py`）。

### ハンドラロジック（アイテム側）
```python
def ねばりのかぎづめ_fix_bind_duration(_battle, _ctx, value):
    if value[0] == "バインド":
        value[1] = 7
    return HandlerReturn(value=value)
```

## テスト観点
- ねばりのかぎづめ持ちがバインド技を使ったとき継続ターンが 7 ターンになること
- ねばりのかぎづめがない場合は 4〜5 ターン（乱数）になること
- バインド付与後にねばりのかぎづめを入手しても継続ターンが延びないこと（付与時確定）
- バインド付与後にねばりのかぎづめを失っても継続ターンが減らないこと（付与時確定）
