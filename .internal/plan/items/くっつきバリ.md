# くっつきバリ 実装計画

## 効果概要
仕様書: `.internal/spec/items/くっつきバリ.md`

2つの効果を持つ：
1. **ターン終了時ダメージ**: 所持者は毎ターン最大HPの1/8のダメージを受ける。
2. **直接攻撃者へのアイテム転送**: アイテムを持たない相手から直接攻撃を受けると、相手のアイテムになる。

## 実装仕様

### 効果1: ターン終了時ダメージ

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_TURN_END` |
| subject_spec | `"source:self"` |
| priority | 150（`.internal/spec/turn.md` ON_TURN_END: 「150 くっつきバリ」） |
| once | False |
| 消費条件 | なし（非消費） |

### 効果2: 直接攻撃者へのアイテム転送

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_DAMAGE_HIT` |
| subject_spec | `"defender:self"` |
| priority | 30（`.internal/spec/turn.md` ON_DAMAGE: 「30 くっつきバリが攻撃側に渡る」） |
| once | False |
| 消費条件 | 転送時に自分のアイテムを失う（`item_manager.remove_item(mon)` + `item_manager.gain_item(attacker, "くっつきバリ")`） |

priority=30 は、どろぼう・ほしがる・はたきおとす（ON_DAMAGE_HIT priority=100）・
特性マジシャン（ON_DAMAGE_HIT priority=100、既定値）・特性わるいてぐせ（priority=180）よりも
前に発動させるために必須。これにより「くっつきバリの転移判定が先に行われ、既に転移していた場合は
どろぼう等の効果自体が発動しない」という仕様（`.internal/spec/items/くっつきバリ.md` 34行目）を満たす。

**レビューで修正した前提**: 当初 `どろぼう_steal_item` / `はたきおとす_remove_item` は
`Event.ON_HIT`（`ON_DAMAGE_HIT` より前に発火するイベント）に登録されており、
くっつきバリの転移判定より先にアイテムが除去されて転移が一切発生しない不具合があった。
本レビューで両ハンドラの登録イベントを `Event.ON_DAMAGE_HIT` に修正済み
（`.internal/spec/turn.md` ON_DAMAGE 「100 はたきおとす等のアイテム効果」に整合）。

### ハンドラロジック（疑似コード）
```python
def くっつきバリ_damage_on_turn_end(battle, ctx, value):
    """くっつきバリ: ターン終了時に最大HPの1/8ダメージ。"""
    mon = ctx.source
    assert mon is not None
    if battle.modify_hp(mon, r=-1/8):
        _announce_item_triggered(battle, mon)
    return HandlerReturn(value=value)


def くっつきバリ_transfer_on_contact(battle, ctx, value):
    """くっつきバリ: 接触攻撃者がアイテムを持っていない場合、攻撃者に転送する。"""
    mon = ctx.defender
    assert mon is not None
    if battle.query.is_contact(ctx) and not ctx.attacker.has_item():
        _announce_item_triggered(battle, mon)
        battle.item_manager.remove_item(mon)
        battle.item_manager.gain_item(ctx.attacker, "くっつきバリ")
    return HandlerReturn(value=value)
```

## 実装ステップ

1. `src/jpoke/handlers/item.py` に `くっつきバリ_damage_on_turn_end` と `くっつきバリ_transfer_on_contact` を追加。
2. `src/jpoke/data/item.py` の `くっつきバリ` に両イベントのハンドラを登録（priority は上記の通り明示する）。
3. `src/jpoke/data/move.py` の `どろぼう` / `ほしがる` / `はたきおとす` のアイテム関連ハンドラを
   `Event.ON_HIT` から `Event.ON_DAMAGE_HIT` に変更する（くっつきバリとの発動順を正しくするため）。
4. `tests/test_item.py` にテストを追加し、全テストが通ることを確認。

## 既知の制限（未対応）

- ばけのかわ/アイスフェイスがダメージを0に変換して肩代わりした場合、現在の実装では
  `Event.ON_DAMAGE_HIT` 自体が発火しないため、くっつきバリの転移は発動しない
  （本来の仕様では発動するべき）。この修正には技実行フロー（`MoveExecutor._execute_hit`）
  および該当特性ハンドラの横断的な変更が必要なため、本アイテム単体のレビューでは未対応。
  `.internal/spec/items/ゴツゴツメット.md` の既知の制限と同種の課題。
- リーサル計算（`LethalEvent.ON_TURN_END`）は未実装（`.internal/progress/item.md` でも
  リーサル実装/リーサルテストは `-` のまま）。くろいヘドロ・たべのこし同様の
  ターン終了時ダメージのため、将来的に `handlers/lethal.py` へ追加する余地がある
  （とつげきチョッキ・ヒメリのみと同様、実装/テストは完了だがリーサルは別途対応する扱い）。

## テスト観点

1. 毎ターン終了時に最大HPの1/8ダメージを受けること
2. 接触技を受け、攻撃者がアイテムなしのとき、くっつきバリが攻撃者に転送されること
3. 攻撃者がすでにアイテムを持っている場合は転送されないこと
4. 非接触技（飛び道具等）では転送されないこと
5. 転送後、自分のアイテム欄が空になること
6. 転送された攻撃者が次のターン終了時にダメージを受けること
7. はたきおとす・どろぼうでアイテムなしの攻撃者から攻撃を受けた場合、技自体の効果（はたき落とす/奪う）
   ではなくくっつきバリの転移が優先され、攻撃者がくっつきバリを得ること
8. はたきおとすで攻撃者がすでにアイテムを持っている場合は転移せず、通常通りはたき落とされること
9. どろぼうで攻撃者がすでにアイテムを持っている場合は転移も奪取も発生しないこと
