# ゴツゴツメット 実装計画

## 効果概要
直接攻撃（接触技）を受けた際に、攻撃した相手に最大 HP の 1/6 のダメージを与える。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_CHECK_CONTACT` |
| subject_spec | `"defender:self"` |
| priority | 100 |
| once | False |
| 消費条件 | なし（非消費アイテム） |

### ハンドラロジック（実装）
```python
def ゴツゴツメット_chip_contact_attacker(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    mon = ctx.defender
    assert mon is not None
    if _apply_contact_item_chip(battle, ctx, ratio=1/6):
        _announce_item_triggered(battle, mon)
    return HandlerReturn(value=value)
```
- `_apply_contact_item_chip`（`handlers/item.py` 内の共通ヘルパー）が `battle.query.is_contact(ctx)` を判定し、
  接触技であれば `battle.modify_hp(ctx.attacker, r=-ratio)` で攻撃側に固定割合ダメージを与える。
- `battle.query.is_contact` は `Event.ON_CHECK_CONTACT` を発火するため、えんかく/パンチグローブ/ぼうごパットによる
  接触判定の無効化を自動的に反映する。
- `modify_hp` の理由は既定値（`""`）のため、マジックガードの間接ダメージ無効化判定（`ハンドラ側で reason not in {...}` の判定）に
  自動的に該当し、マジックガード所持者へは発動しない。
- `Event.ON_DAMAGE_HIT` はみがわりが実ダメージを完全に肩代わりした場合（`actual_damage<=0`）は発火しないため、
  自身のみがわりで防いだ場合は自動的に不発になる。一方、攻撃側のみがわりは `ctx.attacker` への直接の `modify_hp` であり
  みがわりのHPを経由しないため、自然に貫通する。
- 連続攻撃技は `MoveExecutor` がヒットごとに `Event.ON_DAMAGE_HIT` を発火するため、追加実装なしでヒットごとに発動する。

## 注意
- `handlers/ability.py` の `さめはだ`/`てつのトゲ`（`さめはだ_chip_contact_attacker`）と同じ `_apply_contact_item_chip` パターンの踏襲。
- さめはだ/てつのトゲ（特性、`ON_SWITCH_IN` で先に登録される）とゴツゴツメット（アイテム、後に登録される）は同一イベント・同一優先度・
  同一 subject（defender）のため、登録順に基づく安定ソートにより「特性 → アイテム」の順で発動する（合計 7/24 ダメージの仕様を満たす）。
- 既知の未対応事項: ばけのかわ/アイスフェイスがダメージを 0 に変換して肩代わりした場合、`actual_damage<=0` となり
  `Event.ON_DAMAGE_HIT` 自体が発火しないため、現状ゴツゴツメットは発動しない（仕様上は発動すべき）。この修正は
  `MoveExecutor._execute_hit` のヒット判定ロジック（および `ばけのかわ`/`アイスフェイス` ハンドラ）に及ぶ横断的な対応が
  必要なため、本アイテム単体の実装では対応していない。

## テスト観点
- 接触技を受けたとき相手に最大 HP の 1/6 ダメージが入ること
- 非接触技を受けたときはダメージが入らないこと
- マジックガード所持者には発動しないこと
- さめはだ/てつのトゲと併用した場合、特性ダメージの後にアイテムダメージが入り合計 7/24 ダメージになること
- 連続攻撃技はヒットごとに発動すること
- 攻撃側のみがわりを貫通して本体にダメージを与えること
- 自身のみがわりで防いだ場合は発動しないこと
- 相手が接触ダメージでひんしになるケース
