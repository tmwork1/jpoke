# ビアーのみ 実装計画

更新日: 2026-07-05

## スコープ

- 仕様書: `.internal/spec/items/_タイプ半減きのみ.md`（ビアーのみ節）
- 実装状態: 実装済み

## 効果概要

- 効果バツグンの`どく`タイプの技を受けたとき、ダメージを1/2にして消費する（1度きり）。
- 一次情報（ポケモンWiki `.internal/wiki/items/ビアーのみ.html`）を確認したが、他のタイプ半減きのみ
  （ソクノのみ・ヨプのみ等）にあるような特殊な相互作用（タイプ変化技・特性による判定変化など）の
  記載はなく、共有仕様書の「詳細な仕様」欄は空欄のままで問題ない。

## 実装方針

- 通常戦闘: `Event.ON_CALC_DAMAGE_MODIFIER`（`subject_spec="defender:self"`）で発火。
  タイプ半減きのみ共通処理 `handlers/item.py` の `_modify_super_effective_damage` を
  `type_="どく"`, `modifier=2048/4096` で呼び出す。
  - `ctx.move.type == "どく"` かつ `battle.damage_calculator.calc_def_type_modifier(ctx) > 4096`
    で効果バツグン判定する。
  - priority: 他のタイプ半減きのみ・ダメージ補正系アイテムと同じ既定値（`.internal/spec/turn.md` に
    `ON_CALC_DAMAGE_MODIFIER` 個別の優先度記載なし。ダメージ計算内部イベントのため、
    既存の同種きのみハンドラの priority に合わせてデフォルト値を使用）。
  - 条件成立時に `_announce_and_consume_item` でログ記録とアイテム消費を行う。
- リーサル計算（簡易KO判定用の `LethalEvent`）: `LethalEvent.ON_BEFORE_HIT`（`subject="defender"`）で
  `handlers/lethal.py` の `ビアーのみ_resist_poison` → 共通処理 `_type_resist_berry(..., "どく")`
  を呼び出す。通常ハンドラ側で既にダメージ半減済みのため、リーサル側ではアイテム消費と
  `item_enabled` の更新のみ行う。

## 登録

- `src/jpoke/data/item.py` の `"ビアーのみ"` に `handlers`（`ON_CALC_DAMAGE_MODIFIER`）と
  `lethal_handlers`（`LethalEvent.ON_BEFORE_HIT`）を登録済み。
- `fling_power=10`。

## テスト

- `tests/test_item.py` の `test_タイプ半減実`（パラメータ化テスト）で
  `("ビアーのみ", "どく", "プリン")` を検証（どく技はフェアリータイプのプリンに抜群）。
  - `damage_modifier == 2048`（1/2倍）であること。
  - 使用後にアイテムが消費されていること（`not has_item()`）。
- リーサル側のテスト `test_タイプ半減きのみ_代表種`（`tests/test_lethal.py`）は代表種3件
  （オッカのみ・ヤチェのみ・ロゼルのみ）のみを検証する設計であり、共有ロジックの動作確認が目的の
  ため、ビアーのみを個別に追加する必要はない。
