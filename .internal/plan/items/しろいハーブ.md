# しろいハーブ 実装計画

## 効果概要
ランク低下が発生したとき即座に低下分を打ち消す（消費・1回のみ）。上昇は打ち消さない。
また、以下のタイミングでも「既に能力がマイナスになっている」場合は即座に発動する。
- 場に出た瞬間（バトンタッチで下がった能力を引き継いだ場合など）
- アイテムを入手した瞬間（トリック等で既にランクが下がっている状態で入手した場合）
なげつけるで投げつけられた場合は、投げた側ではなく対象（相手）の下がったランクをリセットする。

## 実装仕様

### 1. 通常の能力低下キャンセル

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_MODIFY_STAT` |
| subject_spec | `"target:self"` |
| priority | 100（デフォルト） |
| once | True（消費アイテム） |
| 消費条件 | 低下を打ち消したとき `battle.item_manager.consume_item(mon)`（`_announce_and_consume_item` 経由） |

`value` は `actual_changes`（`{stat: 変化量}`）。既に `battle.modify_stats` 内で
`target.modify_stat` によりランクへ反映済みの値が渡されるため、以下の条件で発動判定する。

```python
def しろいハーブ_cancel_stat_drop(battle, ctx, value):
    mon = ctx.target
    # 今回の変化で「実際に下がった」かつ「結果のランクがマイナス」の場合のみ発動
    # （+2 を -2 して結果+0になるようなケースは発動しない）
    triggered = any(v < 0 and mon.rank[s] < 0 for s, v in value.items())
    if triggered and _reset_negative_ranks(battle, mon, reason="しろいハーブ"):
        _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)
```

発動時は、この変化以外で既にマイナスになっている能力も含めて **全てのマイナスランクを
まとめて0にリセットする**（くすぐるなどで複数能力が同時に下がった場合や、ねばねばネット＋
バトンタッチの引き継ぎが重なった場合に対応するため）。
リセット処理は `Event.ON_BEFORE_MODIFY_STAT`／`Event.ON_MODIFY_STAT` を経由せず
`mon.rank[s] = 0` を直接書き換える（`くろいきり_reset_all_ranks` と同様の方式）。
これにより、この復元自体がかちき・びんじょう等の「ランク変化」をトリガーとする効果を
再度誘発しないようにする。

### 2. 場に出た瞬間・アイテム入手時の即時発動

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_SWITCH_IN` / `Event.ON_ITEM_GAINED` |
| subject_spec | `"source:self"` |
| priority | `ON_SWITCH_IN`: 160（`.internal/spec/turn.md` ON_SWITCH_IN: 「160 しろいハーブの発動」）<br>`ON_ITEM_GAINED`: デフォルト（turn.md に掲載なし。`くちたけん_form_change` 等の
既存アイテムの ON_ITEM_GAINED ハンドラを参考にデフォルト優先度とした） |
| once | True |

バトンタッチによるランク引き継ぎは `switch_manager.run_switch` 内で `new.rank[stat] = v` を
直接代入する実装であり、`ON_MODIFY_STAT` を経由しないため、既存の
`しろいハーブ_cancel_stat_drop` だけでは検知できない。そのため `ON_SWITCH_IN` で
場に出た時点のランクを確認し、マイナスがあれば即座にリセットする専用ハンドラを追加する。

同様に、トリック等で戦闘中にアイテムを入手する場合も `battle.item_manager._change_item`
経由で `Event.ON_ITEM_GAINED` が発火する（対象が場に出ている場合のみ）ため、同じハンドラを
登録する。

priority=160 はねばねばネットによる素早さ低下（ON_SWITCH_IN priority=100）より後になるよう
設定している。これにより「ねばねばネットで素早さが下がった後、しろいハーブが発動して
（バトンタッチで引き継いだ分も含めて）全ての下がった能力をリセットする」という仕様を、
ねばねばネット自身の `ON_MODIFY_STAT` 経由の発動（既存の1.のハンドラ）と組み合わせて
実現する。

```python
def しろいハーブ_reset_if_already_lowered(battle, ctx, value):
    mon = ctx.source
    if _reset_negative_ranks(battle, mon, reason="しろいハーブ"):
        _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)
```

### 3. なげつけるで投げつけられた場合

`なげつける_apply_item_effect`（`handlers/move_attack.py`、`Event.ON_HIT` に登録済み）に
分岐を追加し、`item_name == "しろいハーブ"` のとき `ctx.defender` の下がったランクを
直接リセットする（`battle.modify_stats` を経由しない直接代入 + `LogCode.STAT_CHANGED` ログ）。
なげつけるは「投げた側」のアイテムなので、アイテム発動宣言（`ITEM_TRIGGERED`）や消費処理は
行わない（投げた側のアイテムは `なげつける_consume_item` で別途消費される）。

`ON_HIT` は `ON_DAMAGE_HIT`（くだけるよろい等の発動イベント）より前に発火するため、
「投げられたしろいハーブ→くだけるよろい」という仕様上の順序が自然に満たされる
（`.internal/spec/items/しろいハーブ.md` 詳細な仕様 参照）。

## 注意
- `ON_MODIFY_STAT` の `value` は `{stat: change}` 辞書形式（`battle.modify_stats` の
  `actual_changes`）。だっしゅつパックの `だっしゅつパック_reserve_switch` が同イベントを
  使っているため参照した。
- どろぼう・ほしがる・はたきおとす・マジシャン・わるいてぐせによるアイテム剥奪は
  既存のアイテム管理機構（`battle.item_manager`）側で「しろいハーブが発動する前に
  アイテムが失われる」処理順になっていることを前提とし、本アイテム側で追加の
  優先順位制御は行わない。

## テスト観点
- ランク低下が発生したとき、低下を打ち消して発動すること
- ランク上昇のみの場合は発動しないこと
- 上昇と低下が混在する場合、低下分のみ打ち消すこと
- 発動後にアイテムが消費されること
- 消費後はランク低下を打ち消せないこと
- なげつけるで相手に投げつけると、相手の下がったランクがリセットされること
- バトンタッチで下がった能力を引き継いだポケモンがしろいハーブを持っていた場合、
  場に出た瞬間にリセットされること
- 既にランクが下がっている状態でアイテムを入手した場合、即座にリセットされること
