# だいこんごうだま 実装計画

更新日: 2026-07-04

## スコープ

- 仕様書: `.internal/spec/items/だいこんごうだま.md`
- 実装状態: 実装済み
- 実装方針: `ON_SWITCH_IN` / `ON_ITEM_GAINED` でディアルガをオリジンフォルムにフォルムチェンジし、`ON_CALC_POWER_MODIFIER` でドラゴン・はがね技を 1.2 倍にする。
  加えて `ON_CHECK_ITEM_CHANGE` で（1）ディアルガが持っている間の奪取/交換/除去防止と、（2）通常の姿のディアルガへの譲渡防止を行う。

## 実装ゴール（完了定義）

- ディアルガがだいこんごうだまを持って登場するとディアルガ(オリジン)にフォルムチェンジする
- ディアルガ(オリジン)がドラゴンまたははがね技を使うと威力が 4915/4096 倍になる
- `removable=False` はすでに ItemData に設定済み（なげつける威力0で失敗）
- ディアルガ以外が持っても効果なし
- ディアルガが持っている間はトリック・すりかえ・ほしがる・どろぼう・特性マジシャン/わるいてぐせ・
  ふしょくガス・はたきおとすによる奪取/交換/除去を防ぐ（`ON_CHECK_ITEM_CHANGE` を `False` で止める）
  - はたきおとすの威力補正（1.5倍）も、除去できないため発生しない
- 通常の姿のディアルガへはトリック・すりかえ等で渡すことができない

## 実装対象イベント

- `Event.ON_SWITCH_IN` — 登場時にフォルムチェンジを発動する
- `Event.ON_ITEM_GAINED` — バトル中にアイテムを入手したとき
- `Event.ON_CALC_POWER_MODIFIER` — ドラゴン/はがね技威力 1.2 倍
- `Event.ON_CHECK_ITEM_CHANGE` — 奪取/交換/除去の可否判定（`ItemManager.can_change_item` から発火。ターン進行に関する `.internal/spec/turn.md` 掲載イベントではないため、既存の
  同種ハンドラ（`くちたけん_prevent_item_change` 等）に倣い priority は既定値(100)のまま登録する）

## 変更対象（ファイル/関数単位）

## 実装ステップ

1. `src/jpoke/handlers/item.py` に `だいこんごうだま_form_change` と `だいこんごうだま_modify_power` を実装する。
2. `src/jpoke/handlers/item.py` に共通処理 `_dedicated_item_prevent_item_change` /
   `_dedicated_item_prevent_transfer_to_base_form` と、それらを呼び出す
   `だいこんごうだま_prevent_item_change` / `だいこんごうだま_prevent_transfer_to_base_form` を実装する。
3. `src/jpoke/data/item.py` の `だいこんごうだま` に各 handler を登録する（`ON_CHECK_ITEM_CHANGE` は
   `target:self` と `target:foe` の2つを登録する）。
4. `src/jpoke/handlers/move_attack.py` の `はたきおとす_power` を修正し、`ItemManager.can_change_item`
   が `False` を返す場合は威力補正をかけないようにする。
5. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. ディアルガがだいこんごうだまを持って登場するとディアルガ(オリジン)になる。
2. ディアルガ(オリジン)がドラゴン技を使うと威力が約 1.2 倍になる。
3. ディアルガ(オリジン)がはがね技を使うと威力が約 1.2 倍になる。
4. ディアルガ以外が持っても効果がない。
5. ディアルガ(オリジン)がはたきおとす・どろぼうで奪われない。
6. ディアルガ(オリジン)がトリック・すりかえで交換されない。
7. 通常の姿のディアルガへトリック等で渡すことができない。
8. はたきおとすの威力補正がディアルガ(オリジン)に対してかからない。

## 疑似コード

```python
BOOSTED_TYPES = {"ドラゴン", "はがね"}

def だいこんごうだま_form_change(battle, ctx, value):
    mon = ctx.source
    if mon.name != "ディアルガ":
        return HandlerReturn(value=value)
    mon.set_form("ディアルガ(オリジン)")
    return HandlerReturn(value=value)

def だいこんごうだま_modify_power(battle, ctx, value):
    if ctx.attacker.name != "ディアルガ(オリジン)":
        return HandlerReturn(value=value)
    if ctx.move.type not in BOOSTED_TYPES:
        return HandlerReturn(value=value)
    return HandlerReturn(value=apply_fixed_modifier(value, 4915))

def だいこんごうだま_prevent_item_change(battle, ctx, value):
    # ディアルガ（オリジン/通常どちらでも）が持っている間は除去/交換を防ぐ
    mon = ctx.target
    if mon is not None and mon.name.startswith("ディアルガ"):
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)

def だいこんごうだま_prevent_transfer_to_base_form(battle, ctx, value):
    # 通常の姿のディアルガへは渡せない
    mon = ctx.target
    if mon is not None and mon.name == "ディアルガ":
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

## リスクと対策

- リスク: `はたきおとす_power` の修正は全アイテム共通のロジックに影響するため、既存の除去可能アイテム
  （たべのこし等）の威力補正が変わらないことをテストで確認する（`test_はたきおとす_相手のアイテムを失わせる`
  等の既存テストが通ることで担保）。
- リスク: `can_change_item` はイベント発火を伴うが、副作用（アイテム消費や状態変更）を持つハンドラは
  現状 `ON_CHECK_ITEM_CHANGE` に登録されていないため、威力計算時に事前判定として呼び出しても安全。
