# バンジのみ 実装計画

## 効果概要
HP が最大値の 25% 以下になった瞬間に最大 HP の 33%（端数切り捨て）を回復する（消費）。
回復と同時に、とくぼうが上がりにくい性格（にがい味が嫌い）のポケモンはこんらんする。
イアのみ・フィラのみ・ウイのみ・マゴのみと回復量・発動条件は同じで、嫌う味と対応する性格が異なるのみ。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_HP_CHANGED` |
| subject_spec | `"target:self"` |
| priority | 100（`.internal/spec/turn.md` に ON_HP_CHANGED の個別記載はなし。既存の同種ハンドラ `_heal_berry` 系が priority=100（`ItemHandler` のデフォルト）で登録されているため、それに合わせる） |
| once | False（`ItemHandler` は消費アイテムでも once を使わず、`battle.item_manager.consume_item` で明示的に消費する既存の共通パターンに合わせる） |
| 消費条件 | 発動時に `_announce_and_consume_item(battle, mon)`（内部で `battle.item_manager.consume_item` を呼ぶ既存の共通処理） |
| こんらん付与 | 発動時、`mon.nature` が `("やんちゃ", "のうてんき", "うっかりや", "むじゃき")` のいずれかなら `battle.volatile_manager.apply_confusion(mon, source=mon)` |

### ハンドラロジック

共通処理 `_heal_berry`（`handlers/item.py`）の `confuse_natures: tuple[str, ...] | None` パラメータに
にがい味が嫌いな性格を渡す、バンジのみ専用のラッパー関数 `バンジのみ_heal_on_quarter_hp` を追加する
（イアのみ・ウイのみと同じ実装パターン）。

```python
def バンジのみ_heal_on_quarter_hp(battle, ctx, value) -> HandlerReturn:
    return _heal_berry(
        battle, ctx, value, denominator=4, heal_r=1/3,
        confuse_natures=("やんちゃ", "のうてんき", "うっかりや", "むじゃき"),
    )
```

- こんらんの発動条件チェック（どんかん／ミストフィールド等の無効化）は `volatile_manager.apply`
  内部の `ON_BEFORE_APPLY_VOLATILE` フックが担うため、本ハンドラ側で個別に判定しない
  （CLAUDE.md の方針・既存の `どくくぐつ_confuse_on_poison` 等と同じ扱い）。
- `source=mon`（自傷）として付与することで、しんぴのまもり等の「相手由来のみ防ぐ」ガードに
  誤って阻止されないようにする（`ctx.is_foe_target()` が false になるため防がれない＝実機仕様と一致）。
- lethal 計算（`handlers/lethal.py` の `バンジのみ_heal`）はHP分布のみを扱うため、こんらん付与は
  対象外（HPを直接変えない効果のため、既存の他アイテムと同様に lethal 側では未実装のままでよい）。
- マゴのみは本レビューの対象外のため、既存の共通関数 `heal_on_quarter_hp`（こんらん未対応）のまま
  変更しない。将来的にレビュー時に同様の専用ラッパーへ切り替える想定。

## テスト観点
- HP が 25% 超 → 25% 以下になった瞬間に 33% 回復すること（回復量は既存テストで確認済み）
- 発動後にアイテムが消費されること（既存テストで確認済み）
- とくぼうが上がりにくい性格（やんちゃ・のうてんき・うっかりや・むじゃき）のとき、発動と同時にこんらんすること
- それ以外の性格では発動してもこんらんしないこと
