# スターのみ 実装計画

## 効果概要
HP が最大値の 1/4 以下になった瞬間に、こうげき・ぼうぎょ・とくこう・とくぼう・すばやさのうち
ランダムな1つ（ランクが最大でないもの）が +2 される（消費）。
なげつける・むしくい・ついばむ・ほおばる・おちゃかいで消費するときは、残りHPに関わらず発動する
（`ON_HP_CHANGED` に `value=mon.max_hp` を渡す `force_trigger_berry` 経由）。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_HP_CHANGED` |
| subject_spec | `"target:self"` |
| priority | 100（`.internal/spec/turn.md` に本イベントの記載なし。同種のピンチきのみ実装（チイラのみ等）に合わせる） |
| once | False（Handler登録時の設定はチイラのみ等と同様。実際の一度きりの消費は `consume_item` によるアイテム剥奪で保証する） |
| 消費条件 | 発動時に `battle.item_manager.consume_item(mon)`（`_announce_and_consume_item` 経由） |

### ハンドラロジック（疑似コード）
```python
def スターのみ_random_boost(battle, ctx, value):
    mon = ctx.target
    forced = value >= mon.max_hp  # なげつける・ほおばる等による強制発動
    if not forced:
        if ctx.hp_change_reason == "self_attack":
            return HandlerReturn(value=value)  # こんらんの自傷ダメージでは発動しない
        if mon.hp * 4 > mon.max_hp:
            return HandlerReturn(value=value)
    # すでにランクが最大の能力は選ばれない（5箇所全て最大なら発動しない）
    candidates = [s for s in ("atk", "def", "spa", "spd", "spe") if mon.rank[s] < 6]
    if candidates:
        stat = battle.random.choice(candidates)
        if battle.modify_stats(mon, {stat: +2}):
            _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)
```

たんじゅん・じゅくせい（4段階上昇）やあまのじゃく（2段階下降）の影響は、
`ON_BEFORE_MODIFY_STAT` を経由する既存の共通処理（`たんじゅん_double_stat` /
`あまのじゃく_reverse_stat`）が自動的に適用するため、本ハンドラ側での個別対応は不要。
ただし、あまのじゃく持ちのポケモンについて「ランクが最小の能力は選ばれない」という
候補除外の反転（本来上げ候補となるはずのランク+6の能力を候補に含め、ランク-6の能力を除外する）
は、既存の類似実装（`ムラっけ_boost_stats` 等）でも未対応であり、本実装でも対応しない
（既知の限界として明記する）。

## テスト観点
- HP が 1/4 超 → 1/4 以下になった瞬間にランダムな能力が +2 されること
- 5種（A/B/C/D/S）のいずれかが上昇すること
- 発動後にアイテムが消費されること
- 乱数固定テストで各能力の上昇を確認すること
- ランクがすでに最大（+6）の能力は選択候補から除外されること
- 5箇所全てのランクが最大のときは発動しない（消費されない）こと
- こんらんの自傷ダメージでHPが1/4以下になっても発動しないこと
