# レンブのみ 実装計画

## 効果概要
特殊技でダメージを受けたとき、攻撃者に最大 HP の 1/8 のダメージを与える（消費）。ジャポのみの特殊版。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_DAMAGE_HIT` |
| subject_spec | `"defender:self"` |
| priority | 90（`.internal/spec/abilities/マジシャン.md`: 「弱点半減のきのみ/ナゾのみ/ジャポのみ/レンブのみはマジシャンより先に発動する」。マジシャン特性ハンドラ（priority=100）より確実に先に発動させるため、素早さに依存しない priority=90 を指定する。ひかりごけ等と同様の対応） |
| once | 消費アイテム（発動時に `_announce_and_consume_item` で消費） |
| 消費条件 | `battle.modify_hp(ctx.attacker, r=-1/8)` が実際に変化した場合のみ消費（攻撃者がマジックガード持ちの場合や既にひんしの場合は不発・非消費） |

### ハンドラロジック（疑似コード、実装は `_retaliate_on_category` 共通関数としてジャポのみと共有）
```python
def _retaliate_on_category(battle, ctx, value, category):
    mon = ctx.defender
    if ctx.move.category == category:
        if battle.modify_hp(ctx.attacker, r=-1/8):
            _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)

def レンブのみ_retaliate_special(battle, ctx, value):
    return _retaliate_on_category(battle, ctx, value, "special")
```

## テスト観点
- 特殊技でダメージを受けたとき相手に 1/8 ダメージが入ること
- 物理技・変化技では発動しないこと
- 発動後にアイテムが消費されること
- 攻撃者がマジックガード持ちの場合は発動せず、アイテムも消費されないこと
- 攻撃者がマジシャン持ちの場合、レンブのみが先に発動して消費されるためアイテムを奪われないこと（素早さに依存しない）
- ジャポのみ（物理版）と対称的な実装であること
