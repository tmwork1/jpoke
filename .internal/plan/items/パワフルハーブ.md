# パワフルハーブ 実装計画

## 効果概要
溜め技（ソーラービーム等）の溜めターンをスキップして 1 ターンで発動する（消費）。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_MOVE_CHARGE` |
| subject_spec | `"attacker:self"` |
| priority | 100 |
| once | True（消費アイテム） |
| 消費条件 | 発動時に `battle.consume_item(mon)` |

### ハンドラロジック（疑似コード）
```python
def パワフルハーブ_skip_charge(battle, ctx, value):
    mon = ctx.attacker
    # おおひでり下のダイビングのように、溜めをスキップした結果 ON_TRY_MOVE_1 で
    # 天候により技自体が無効化される場合はアイテムを消費しない
    weather_blocks_move = (
        (battle.weather.name == "おおひでり" and ctx.move.type == "みず")
        or (battle.weather.name == "おおあめ" and ctx.move.type == "ほのお")
    )
    if not weather_blocks_move:
        _announce_and_consume_item(battle, mon)
    # 溜めをスキップして即発動（消費有無に関わらず ON_MOVE_CHARGE 自体は停止する）
    return HandlerReturn(value=True, stop_event=True)
```

### 実装済みハンドラとの連携
- パワフルハーブは priority=100 で登録され、アイテムはポケモンの登場時、技側ハンドラは
  技使用の直前に登録されるため、同じ priority を持つハンドラ同士では常にアイテムが
  技側の既定ハンドラ（例: `ソーラービーム系_charge` の溜め処理、priority=100）より
  先に実行される。
- ただし「にほんばれ/おおひでりでソーラービーム系が溜めをスキップする」
  「あめ/おおあめでエレクトロビームが溜めをスキップする」という天候由来の自動スキップは、
  パワフルハーブの消費より優先されなければならない（一次情報参照）。そのため、
  これらの天候チェックは `ソーラービーム系_weather_skip` / `エレクトロビーム_weather_skip`
  として priority=90（パワフルハーブより先）に分離して登録し、天候によるスキップが
  成立する場合はそこで `stop_event=True` を返して以降のパワフルハーブ判定に到達させない。
  天候によるスキップが成立しない場合のみ、通常通りパワフルハーブ（priority=100）→
  技側の既定の溜め処理（`ソーラービーム系_charge` 等、priority=100）の順で処理される。
- `メテオビーム`（天候による自動スキップ機構を持たない）はこの分離が不要であり、
  従来通り単一ハンドラ（priority=100）のままで良い。

## テスト観点
- ソーラービーム等の溜め技が 1 ターンで発動すること
- 発動後にアイテムが消費されること
- 消費後は溜め技が 2 ターンかかること
- はれ状態時の溜めスキップと競合しないこと（天候優先でアイテムは消費されない）
- おおひでり下でダイビングを使ったとき、技が失敗してもアイテムが消費されないこと
