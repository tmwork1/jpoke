# タラプのみ 実装計画

## 効果概要
特殊技でダメージを受けた直後、とくぼう（spd）を +1 する（消費）。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_DAMAGE_HIT` |
| subject_spec | `"defender:self"` |
| priority | 100（デフォルト値。`.internal/spec/turn.md` に ON_DAMAGE_HIT の記載はないため、同種の被弾トリガー系アイテム〔アッキのみ・ゴツゴツメット・きゅうこん等〕の既存実装に倣いデフォルト優先度を採用） |
| once | 未使用（`Handler.once` は指定しない＝False のまま。消費は下記の通り `modify_stats` が実際に変化を返した場合のみ `ItemManager.consume_item` を呼んで行う） |
| 消費条件 | ランクが実際に変化した場合のみ `battle.item_manager.consume_item(mon)` を呼ぶ（共通ヘルパー `_announce_and_consume_item` 経由） |

### ハンドラロジック（疑似コード）
共通ヘルパー `_boost_on_attack_category`（アッキのみと共用）を利用する。

```python
def タラプのみ_boost_spdef(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """タラプのみ: 特殊技でダメージを受けたときとくぼう+1。"""
    return _boost_on_attack_category(battle, ctx, value, "special", "spd", +1)


def _boost_on_attack_category(battle, ctx, value, category, stat, amount):
    mon = ctx.defender
    # 特性ちからずくの対象技（追加効果あり技）を受けたときは発動しない。
    if (
        ctx.attacker.ability.name == "ちからずく"
        and ctx.move.has_flag("secondary_effect")
    ):
        return HandlerReturn(value=value)
    if ctx.move.category == category:
        # source=mon とすることで、あまのじゃくにより反転した下降も
        # 自発的な変化として扱われ、しろいきり/フラワーベールで防がれない。
        # ランク上限/下限で実際に変化しなかった場合は消費しない。
        if battle.modify_stats(mon, {stat: amount}, source=mon):
            _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)
```

## 注意
- アッキのみの特殊版（アッキ=物理被弾でB+1、タラプ=特殊被弾でD+1）。ハンドラ実装は共通ヘルパー `_boost_on_attack_category` を共用する。
- じゅくせい持ちでも効果量は+1のまま（スター以外のピンチきのみと違い2倍にならない）
- HP1/4閾値とは無関係。特殊技でダメージを受けたら即発動
- ダメージが実際に発生しなかった場合（無効/まもる/みがわり等で`ON_DAMAGE_HIT`自体が発火しない場合）は当然発動しない
- すでにとくぼうランクが最大のときは`modify_stats`が変化なし（falsy）を返すため発動・消費ともにしない
- あまのじゃく/たんじゅんの影響は`ON_MODIFY_STAT_CHANGE`側の特性ハンドラが処理するため、本アイテムのハンドラ側で個別対応は不要
- かたやぶり等により対象の特性が無効化されている技で発動した場合、あまのじゃく/たんじゅんの効果は出ない。これはかたやぶり側が`ON_BEGIN_MOVE`〜`ON_END_MOVE`の間、対象の特性を無効化する既存の仕組みで自動的に処理されるため、本アイテムのハンドラ側で個別対応は不要
- マジシャン（技を受けた瞬間に持ち物を奪う特性）やきつくす（技のダメージ処理後に持ち物を燃やす効果）でアイテムを失った場合は、タラプのみ自体が発動せず奪われる/燃やされる。これらは既存のアイテム奪取/消失の仕組みで処理されるため、本アイテムのハンドラ側で個別対応は不要
- 特性ちからずくの対象技（追加効果あり技）を受けたときは発動しない（`ctx.attacker.ability.name == "ちからずく" and ctx.move.has_flag("secondary_effect")`で明示的に判定する必要がある。だっしゅつボタン・かいがらのすず・レッドカードと同様の既存パターン）。
  - **レビューで発見した不具合**: 従来の実装（共通ヘルパー`_boost_on_attack_category`）はこの判定が欠落しており、アッキのみ・タラプのみ共通でちからずく所持者の追加効果あり技を受けても発動していた。本レビューで判定を追加して修正した。

## リーサル実装（lethal.py）
- `LethalHandler` `タラプのみ_boost_spd`（`LethalEvent.ON_HIT`, subject="defender"）。
  - 特殊技を受けた直後にとくぼう+1して消費する（`item_enabled` state のみ）。
  - とくぼうランクがすでに+6のときは発動しない。
  - 通常ハンドラと同様、ちからずく所持者の追加効果あり技を受けたときは発動しない
    （`ctx.attacker.ability.name == "ちからずく" and ctx.move.has_flag("secondary_effect")`）。
    こちらもレビューで判定漏れを発見し、アッキのみのリーサルハンドラとあわせて修正した。
  - あまのじゃく/たんじゅんによるランク反転・倍化は、リーサル計算の簡略化方針上もともと
    モデル化されていない（既存の他のランク変化系リーサルハンドラも同様）ため、本アイテムの
    リーサルハンドラでも対象外とする。

## テスト観点
- 特殊技でダメージを受けたとき spd が +1 されること
- 物理技では発動しないこと
- 発動後にアイテムが消費されること
- すでにとくぼうランクが最大のときは発動・消費ともにしないこと
- あまのじゃくによる反転（下降）はしろいきりで防げないこと（自発的変化扱い）
- 特性ちからずく所持者の追加効果あり特殊技（例: でんきショック）を受けたときは発動しないこと
- （リーサル）特殊技を受けた直後にとくぼう+1し2発目のダメージが減少すること
- （リーサル）物理技では発動しないこと
- （リーサル）ちからずく所持者の追加効果あり特殊技では発動しないこと
