# おおきなねっこ 実装計画

## 効果概要
HPを吸収する効果（吸収技・ちからをすいとる・アクアリング・ねをはる・やどりぎのタネ）で
回復するHP量を1.3倍（正確には5324/4096倍、五捨五超入）にする（非消費）。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_CALC_DRAIN` |
| subject_spec | `"attacker:self"`（吸収技・ちからをすいとる／AttackContext）と `"source:self"`（アクアリング・ねをはる・やどりぎのタネ／EventContext）の2つを登録 |
| priority | 100（デフォルト） |
| once | False |
| 消費条件 | なし（非消費アイテム） |

`Event.ON_CALC_DRAIN` は AttackContext（吸収技・ちからをすいとる）と EventContext
（アクアリング・ねをはる・やどりぎのタネ）の両方から発火される。`Handler.subject_spec` は
コンテキスト型ごとに異なるロール名（`attacker`/`source`）でしか解決できないため、
`ItemData.handlers[Event.ON_CALC_DRAIN]` には `subject_spec`違いの `ItemHandler` を
2つ登録する（`list[Handler]` はフレームワークが対応済み）。

## 発火側の改修（4箇所）

### 1. `handlers/move_attack.py` の `_drain_hp()`
吸収技（ギガドレイン・きゅうけつ等）の回収量計算。AttackContextで発火。
```python
def _drain_hp(battle: Battle, ctx: AttackContext, damage: int, heal_ratio: float) -> None:
    damage = damage or ctx.substitute_damage
    heal_amount = int(damage * heal_ratio)
    heal_amount = battle.events.emit(Event.ON_CALC_DRAIN, ctx, heal_amount)
    battle.modify_hp(ctx.attacker, v=heal_amount, reason="drain")
```

### 2. `handlers/move_status.py` の `ちからをすいとる_apply()`
相手のこうげき実数値分回復する処理。AttackContextで発火。
```python
def ちからをすいとる_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    assert ctx.defender is not None
    recover_amount = ctx.defender.ranked_stats["atk"]
    recover_amount = battle.events.emit(Event.ON_CALC_DRAIN, ctx, recover_amount)
    battle.modify_hp(ctx.attacker, v=recover_amount)
    battle.modify_stats(ctx.defender, {"atk": -1}, source=ctx.attacker)
    return HandlerReturn(value=value)
```

### 3. `handlers/volatile.py` の `アクアリング_self_heal()` / `ねをはる_self_heal()`
ターン終了時回復。`ON_TURN_END` は `subject_spec="source:self"` で登録されているため、
ハンドラに渡される `ctx`（EventContext）は既に `ctx.source == mon` になっている。
新たに `EventContext` を組み立てる必要はなく、`ctx` をそのまま emit に使う。
```python
def アクアリング_self_heal(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    mon = ctx.source
    heal = max(1, mon.max_hp // 16)
    heal = battle.events.emit(Event.ON_CALC_DRAIN, ctx, heal)
    return HandlerReturn(value=battle.modify_hp(mon, v=heal, source=mon))


def ねをはる_self_heal(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    heal = max(1, ctx.source.max_hp // 16)
    heal = battle.events.emit(Event.ON_CALC_DRAIN, ctx, heal)
    return HandlerReturn(value=battle.modify_hp(ctx.source, v=heal, source=ctx.source))
```
（`r=1/16` で `modify_hp` に丸めを任せていた実装を、先に基準回復量を算出してから
`ON_CALC_DRAIN` を発火する形に変更。1/16は2のべき乗なので端数処理の結果は変わらない。）

### 4. `handlers/volatile.py` の `やどりぎのタネ_drain_hp()`
仕様上、やどりぎのタネによる回復量への補正は「回復するポケモン（to_mon）」の所持アイテムで
判定する（設置側=from_monが持っていても無効）。`ctx.source == from_mon` なので、
`ctx.derive(source=to_mon)` で to_mon を主体とする EventContext を作って emit する。
```python
def やどりぎのタネ_drain_hp(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    from_mon = ctx.source
    to_mon = battle.foe(from_mon)
    damage = battle.modify_hp(from_mon, r=-1/8, reason="drain")
    if damage:
        heal = battle.events.emit(Event.ON_CALC_DRAIN, ctx.derive(source=to_mon), -damage)
        battle.modify_hp(to_mon, v=heal, reason="drain")
    return HandlerReturn(value=damage)
```

## ハンドラロジック（アイテム側）

端数処理は「通常時の回復量を算出し、端数処理も行った後におおきなねっこの効果を計算する」
「五捨五超入」「正確な倍率は5324/4096倍」という仕様に厳密に従う。
`apply_fixed_modifier`（`// 4096` による切り捨て）ではなく、`round_half_down` を使う。

```python
def おおきなねっこ_boost_drain(_battle: Battle, _ctx: Any, value: int) -> HandlerReturn:
    """おおきなねっこ: HPを吸収する効果の回復量を1.3倍(5324/4096倍、五捨五超入)にする。"""
    return HandlerReturn(value=round_half_down(value * 5324 / 4096))
```


## ヘドロえきとの整合性
`_drain_hp()` は `battle.modify_hp(ctx.attacker, v=heal_amount, reason="drain")` を呼ぶため、
`modify_hp` 内部の `ON_MODIFY_HEAL` 発火時点では既に `おおきなねっこ` で1.3倍された
`heal_amount` が渡る。ヘドロえき側 (`ヘドロえき_reverse_drain`) はこの値の符号を反転させるだけ
なので、「ヘドロえきによるダメージも1.3倍になる」という仕様は発火順序（先に
`ON_CALC_DRAIN`、後から`modify_hp`内の`ON_MODIFY_HEAL`）だけで自動的に満たされる。
追加の実装は不要。

## テスト観点
- 吸収技（ギガドレイン等）でおおきなねっこ持ちの回収量が1.3倍になること
- 端数処理が五捨五超入・5324/4096倍であること（切り捨てや1.3倍の単純計算と異なる値になる
  ケースを用意して区別する）
- ちからをすいとるの回復量が1.3倍になること
- アクアリング・ねをはるのターン終了時回復が1.3倍になること
- やどりぎのタネの回復量は、回復側（吸い取る側）がおおきなねっこを持つ場合のみ1.3倍になり、
  設置された側（吸われる側）だけが持っていても影響しないこと
