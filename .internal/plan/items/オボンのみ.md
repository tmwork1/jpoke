# オボンのみ 実装計画

## 効果概要
HP が最大 HP の 50% 以下になった瞬間に最大 HP の 25%（端数切り捨て、最低1）を回復する（消費）。
オレンのみ・フィラのみ・ウイのみ・マゴのみ・バンジのみ・イアのみと同じ「HP 回復ピンチきのみ」
グループに属し、閾値・回復量以外の発動条件（自傷こんらん・きんちょうかん・かいふくふうじ等）は共通。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_HP_CHANGED` |
| subject_spec | `"target:self"` |
| priority | 100（`.internal/spec/turn.md` に `ON_HP_CHANGED` の個別記載はなし。既存の同種ハンドラ（`heal_on_quarter_hp` 等）が `ItemHandler` のデフォルト priority=100 で登録されているため、それに合わせる） |
| once | False（`ItemHandler` は消費アイテムでも once を使わず、`battle.item_manager.consume_item` で明示的に消費する既存の共通パターンに合わせる） |
| 消費条件 | 発動時に `_announce_and_consume_item(battle, mon)`（内部で `battle.item_manager.consume_item` を呼ぶ既存の共通処理）。ただし回復が実際に発生した場合のみ消費する |

### ハンドラロジック

共通処理 `_heal_berry`（`handlers/item.py`）をそのまま利用する。

```python
def _heal_berry(battle, ctx, value, *, denominator, heal_r=None, heal_v=None,
                 confuse_natures: tuple[str, ...] | None = None) -> HandlerReturn:
    mon = ctx.target
    assert mon is not None
    # value >= mon.max_hp はほおばる等による強制発動（HP閾値チェックを無視する）
    forced = value >= mon.max_hp
    if not forced:
        # こんらんの自傷ダメージでは発動しない（第五世代以降の仕様）
        if ctx.hp_change_reason == "self_attack":
            return HandlerReturn(value=value)
        # 相手のきんちょうかん・じんばいったいの影響下では発動しない
        if battle.query.is_nervous(mon):
            return HandlerReturn(value=value)
    if forced or mon.hp * denominator <= mon.max_hp:
        if heal_r is not None:
            healed = battle.modify_hp(mon, r=heal_r)
        else:
            healed = battle.modify_hp(mon, v=heal_v)
        # かいふくふうじ等で回復が完全に無効化された場合は消費しない
        if healed:
            _announce_and_consume_item(battle, mon)
            if confuse_natures is not None and mon.nature in confuse_natures:
                battle.volatile_manager.apply_confusion(mon, source=mon)
    return HandlerReturn(value=value)


def オボンのみ_heal_on_half_hp(battle, ctx, value) -> HandlerReturn:
    return _heal_berry(battle, ctx, value, denominator=2, heal_r=1/4)
```

- 「現在の HP が閾値以下」という状態ベースの判定にしているため、「閾値を跨いだ瞬間」だけでなく
  閾値以下の状態で `ON_HP_CHANGED` が発火するたび判定される。実際に一度しか発動しないのは、
  発動後にアイテムが消費されて `ItemHandler` 自体が解除されるため（`item.enabled` チェックとは別に、
  アイテム変更時に `unregister_handlers` されるため再登録された状態にならない）。
- かいふくふうじによる回復無効化は `Event.ON_MODIFY_HEAL`（`battle.modify_hp` 内部で発火）側の
  `かいふくふうじ_block_heal` が既に担っており、本ハンドラは戻り値（実際に回復した HP 量）を
  見て消費するかどうかだけ判定すればよい。かいふくふうじ側の実装を変更する必要はない。
- きんちょうかん・じんばいったいの判定は `battle.query.is_nervous(mon)` を利用する
  （`Event.ON_CHECK_NERVOUS` を発火し、相手の特性ハンドラが判定する既存の仕組み）。
  なげつける・むしくい・ついばむ・ほおばる・おちゃかい等の強制発動（`value >= mon.max_hp`）では
  現在の HP 量を無視して発動するため、この判定もスキップする（ウィキに強制発動時の
  きんちょうかん無効化についての記載がないため、現状の仕様書どおり通常発動時のみ判定する）。
- こんらんの自傷ダメージ（`reason="self_attack"`）による HP 減少は、既存の `ききかいひ` 特性の
  実装と同じパターンで `ctx.hp_change_reason == "self_attack"` を除外することで、
  第五世代以降の「こんらん自傷では発動しない」仕様を再現する。アイテムは消費されずに残るため、
  次に別要因でダメージを受けたときは通常どおり発動する（ウィキの仕様と一致）。
- ぶきよう・マジックルーム状態での無効化は `EventManager._check_handler_validity` の
  `subject.item.enabled` チェックで汎用的に処理されるため、本ハンドラ側で個別に判定しない。
- 特性じゅくせいによる回復量2倍化は本アイテム固有の仕様ではなく `じゅくせい` 特性側の実装課題
  （`.internal/progress/ability.md` で実装="-" として別管理されている）。本レビューでは対象外とし、
  `_heal_berry` 側での実装は行わない。
- さしおさえ・ダイマックスは本プロジェクトの対象（ポケモンチャンピオンズシングルバトル）に
  存在しない状態・システムのため実装しない（仕様書には参照用として記載を残す）。
- lethal 計算（`handlers/lethal.py` の `オボンのみ_heal` → `_heal_at_pinch`）は HP 分布のみを扱う
  簡略化されたモデルであり、自傷こんらん・きんちょうかん等の細かい発動条件は元々対象外
  （他の同種きのみの lethal 実装と同じ扱いのため変更しない）。

## テスト観点
- HP が 50% 超 → 50% 以下になった瞬間に 25% 回復すること（既存テストで確認済み）
- HP が 50% 超のときは発動しないこと（既存テストで確認済み）
- 発動後にアイテムが消費されること
- こんらんの自傷ダメージ（`reason="self_attack"`）で HP が 1/2 以下になっても発動しないこと。
  その後、自傷以外のダメージで再度 HP 変動が起きると発動すること
- 相手が きんちょうかん／じんばいったい を持つ場合は発動しないこと
- かいふくふうじ状態のときは発動せず、アイテムも消費されないこと
