# しんかのきせき 実装計画

## 効果概要
未進化のポケモンのぼうぎょ・とくぼうを 1.5 倍にする（非消費）。

## 実装仕様

### イベント 1: ぼうぎょ（B）1.5 倍

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_CALC_DEF_MODIFIER` |
| subject_spec | `"defender:self"` |
| priority | 100 |
| once | False |

### イベント 2: とくぼう（D）1.5 倍（物理/特殊で同一イベントを使う場合は条件分岐）

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_CALC_DEF_MODIFIER` |
| subject_spec | `"defender:self"` |
| priority | 100 |
| once | False |

### ハンドラロジック（疑似コード）
```python
# モジュール定数: POKEDEX の pre_evolution を逆引きして「進化先がある」名前集合を構築
_HAS_EVOLUTION = frozenset(d.pre_evolution for d in POKEDEX.values() if d.pre_evolution)

# メルタンは進化先（メルメタル）を持つが、進化手段が特殊（通常の進化データと異なる）ため
# 内部的には「進化の余地がある」と判定されず、しんかのきせきの効果を得られない例外。
_EVIOLITE_NO_EFFECT = frozenset({"メルタン"})

def しんかのきせき_boost_defenses(_battle, ctx, value):
    if ctx.defender.name in _HAS_EVOLUTION and ctx.defender.name not in _EVIOLITE_NO_EFFECT:
        value = apply_fixed_modifier(value, 6144)  # 1.5倍
    return HandlerReturn(value=value)
```

## 注意
`Pokemon.pre_evolution` は「この個体が進化してきた元の種族名」を返す。
各エントリの `pre_evolution` を集めた集合 `_HAS_EVOLUTION` に自分の名前が含まれる
= 何らかのポケモンが自分を進化前として持つ = 進化先が存在する（未進化）という判定になる。

ただしメルタンは例外（`_EVIOLITE_NO_EFFECT`）として明示的に除外する
（.internal/spec/items/しんかのきせき.md の「効果の発動判定」参照）。

なお `pokedex.json` の `0670-000`（フラエッテ）の `pre_evolution` が自己参照（`"フラエッテ"`）に
なっているデータ不備を発見し、`"フラベベ"` に修正した（フラベベがしんかのきせきの効果を得られない
不具合の原因だったため）。

## テスト観点
- 未進化のポケモンのぼうぎょ・とくぼうが 1.5 倍になること
- 最終進化のポケモンは倍率が変化しないこと
- 進化後ポケモンに持たせた場合も効果がないこと
- メルタンに持たせても効果が無いこと（進化先を持つが例外扱い）
