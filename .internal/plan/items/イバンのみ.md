# イバンのみ 実装計画

参照: `.internal/spec/items/_ランク上昇ピンチきのみ.md` の「イバンのみ」節（イバンのみ専用の
`.internal/spec/items/イバンのみ.md` は作成しない。チイラのみ等と同じ「ピンチきのみ」ファミリー
としてまとめて仕様化する既存方針に合わせる）。

## 効果概要
HP が最大値の 25% 以下になった瞬間にフラグを立て（消費はしない）、次に技を選択したターンの
行動順を先制させたうえでアイテムを消費する（1回のみ）。技自体の優先度は変化しない。

## 実装仕様

### イベント 1: HP 閾値トリガー（フラグ付与のみ、消費はしない）

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_HP_CHANGED` |
| subject_spec | `"target:self"` |
| priority | 100（デフォルト） |
| once | False |
| 状態管理 | `mon.item.count` を 0→1 にする（`Item` の汎用カウンタフィールドを流用。消費はまだしない） |
| 除外条件 | こんらんの自傷ダメージ（`ctx.hp_change_reason == "self_attack"`）では発動しない（ピンチきのみファミリー共通仕様） |

### イベント 2: 行動順の先制付与（ここで消費）

| 項目 | 内容 |
|------|------|
| イベント | `DomainEvent.ON_CALC_BACK_TIER` |
| subject_spec | `"attacker:self"` |
| priority | 100（デフォルト。せんせいのツメ・クイックドロウの登録と同様） |
| once | False |
| 消費条件 | `mon.item.count == 1` かつ相手のきんちょうかん・じんばいったいの影響下でないとき、`battle.item_manager.consume_item(mon)` を呼び、行動ティア +1 する |

### ハンドラロジック（実装済み・実際のコード）
```python
def イバンのみ_boost_priority(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    mon = ctx.attacker
    if mon.item.count == 1 and not battle.query.is_nervous(mon):
        battle.item_manager.consume_item(mon)
        return HandlerReturn(value=value + 1)
    return HandlerReturn(value=value)


def イバンのみ_set_priority_flag(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    mon = ctx.target
    assert mon is not None
    if ctx.hp_change_reason == "self_attack":
        return HandlerReturn(value=value)
    hp_after = mon.hp
    hp_before = hp_after + value
    if hp_before * 4 > mon.max_hp and hp_after * 4 <= mon.max_hp:
        mon.item.count = 1
        _announce_item_triggered(battle, mon)
    return HandlerReturn(value=value)
```

## 注意
- タラプのみ（特殊被弾でD+1）とは別アイテム。混同しないこと。
- `ON_CALC_BACK_TIER` はターン開始時の行動順計算（技選択直後・交代処理より前）で発火するため、
  「相手のきんちょうかん/じんばいったい所持ポケモンが交代で引っ込んだターンでは発動しない」という
  仕様を、消費時点（`イバンのみ_boost_priority`）での `battle.query.is_nervous` 判定として自然に満たせる。
- くいしんぼう特性所持時は閾値が HP1/2 以下になる仕様があるが、`くいしんぼう` 特性自体が
  本コードベースで未実装（`.internal/progress/ability.md` 参照）のため、この項目では対応しない。
  くいしんぼう実装時に本ハンドラ側の閾値判定を見直すこと。
- クイックドロウ特性が同一ポケモンで発動した場合、HPが1/4以下でもイバンのみは消費されない
  （クイックドロウが発動しなかったときに限りイバンのみが発動する）という仕様があるが、
  `ON_CALC_BACK_TIER` の同一 attacker に対するハンドラ実行順序に依存する実装が必要になり
  影響範囲が特性側（`クイックドロウ_maybe_fast_attack`）にも及ぶため、本レビューでは未対応の
  既知の制限とする。クイックドロウ側のレビュー時に合わせて対応を検討する。

## テスト観点
- HP が 25% 超 → 25% 以下になった瞬間にフラグが立ち、アイテムはまだ消費されないこと
- こんらんの自傷ダメージで HP が 25% 以下になってもフラグが立たないこと
- フラグが立った後、次のターンに先制（行動ティア +1）で行動し、そのタイミングでアイテムが消費されること
- 先制後はフラグが解除され、2ターン目は通常行動順になること
- 相手がきんちょうかん/じんばいったいを発動している間はフラグが立ってもアイテムが消費されず、
  行動順も変化しないこと
