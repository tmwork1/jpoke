# しめつけバンド 実装計画

## 効果概要
バインドダメージを最大 HP の 1/6 に増加する（通常は最大 HP の 1/8）。

道具の効果は「相手をバインド状態にするとき」（＝バインド付与時点）にしか判定されない。
付与後にしめつけバンドを入手してもダメージは増えず、逆に付与後に手放してもダメージは減らない
（`.internal/spec/items/しめつけバンド.md` 詳細な仕様、`ねばりのかぎづめ`と同様の「付与時確定」挙動）。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_MODIFY_BIND_DAMAGE` |
| subject_spec | `"attacker:self"` |
| priority | 100 |
| once | False |
| 消費条件 | なし（非消費アイテム） |

`ON_MODIFY_BIND_DAMAGE` は `handlers/move_attack.py` の `apply_bind_to_defender`（`AttackContext`）内で
バインド付与時に1回だけ発火し、ダメージ倍率をその場で確定させて `Volatile.bind_damage_ratio` に保存する。
攻撃側（しめつけバンド持ち）は `ctx.attacker` に当たるため `subject_spec="attacker:self"`。
`バインド_damage`（ターン終了時処理）では以降イベントを再発火せず、保存済みの `bind_damage_ratio` をそのまま使う。

## 発火側の改修

`handlers/move_attack.py` の `apply_bind_to_defender` で `ON_MODIFY_BIND_DAMAGE` を発火し、
結果を `volatile_manager.apply` の `bind_damage_ratio` kwarg として渡す。

### `handlers/move_attack.py` の `apply_bind_to_defender` を改修
```python
def apply_bind_to_defender(battle, ctx, value):
    count = battle.query.get_volatile_duration(ctx, "バインド", battle.random.randint(4, 5))
    bind_damage_ratio = battle.events.emit(Event.ON_MODIFY_BIND_DAMAGE, ctx, 1/8)
    battle.volatile_manager.apply(
        ctx.defender, "バインド", count=count, source=ctx.attacker, bind_damage_ratio=bind_damage_ratio
    )
    return HandlerReturn(value=value)
```

### `handlers/volatile.py` の `バインド_damage`（イベント再発火なし）
```python
def バインド_damage(battle, ctx, value):
    mon = ctx.source
    battle.volatile_manager.tick(mon, "バインド")
    if not mon.has_volatile("バインド"):
        return HandlerReturn(value=value)

    r = mon.volatiles["バインド"].bind_damage_ratio  # 付与時に確定済み
    battle.modify_hp(mon, r=-r)
    return HandlerReturn(value=value)
```

### ハンドラロジック（アイテム側）
```python
def しめつけバンド_boost_bind_damage(battle, ctx, value):
    return HandlerReturn(value=1/6)
```

## テスト観点
- しめつけバンド持ちがバインド技を使ったとき毎ターンのバインドダメージが最大 HP の 1/6 になること
- しめつけバンドがない場合は最大 HP の 1/8 のバインドダメージになること
- バインド付与後にしめつけバンドを入手してもダメージ倍率が増加しないこと（付与時確定）
- バインド付与後にしめつけバンドを失ってもダメージ倍率が減少しないこと（付与時確定）
