# ヒメリのみ 実装計画

## 効果概要
持たせたポケモンの技のPPが0になったとき、その技のPPを10回復して消費する
（じゅくせい所持時は20）。PPが0の技が複数ある場合の回復対象選定・
なげつける/むしくい・ついばむ/ほおばる・おちゃかいでの発動・マジックルーム/ぶきよう
との相互作用は .internal/spec/items/ヒメリのみ.md を参照。

## 実装仕様

### 1. 自分の技使用でPPが0になったとき（メインルート）

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_PP_CONSUMED`（move_executor._consume_pp が技使用のたびに発火） |
| コンテキスト型 | `AttackContext` |
| subject_spec | `"attacker:self"` |
| priority | 100（デフォルト） |
| ハンドラ | `h.ヒメリのみ_restore_pp` |

`ctx.move` が直前に使用した技そのものなので、複数技のPP0優先順位を考える必要はない
（`ctx.move.pp == 0` を見るだけでよい）。

### 2. 場に出た直後／アイテムが再有効化されたとき／強制発動されたとき

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_SWITCH_IN` / `Event.ON_ITEM_ENABLED` / `Event.ON_FORCE_BERRY_TRIGGER` |
| コンテキスト型 | `EventContext` |
| subject_spec | `"source:self"` |
| priority | `ON_SWITCH_IN` は 100（デフォルト。.internal/spec/turn.md の「100 きのみの発動」枠）、`ON_ITEM_ENABLED`/`ON_FORCE_BERRY_TRIGGER` は 50（turn.md 未掲載イベントのため、同じ「きのみ」種別である他のきのみ（クラボのみ・チーゴのみ等の状態異常回復きのみ）の既存登録に合わせる） |
| ハンドラ | `h.ヒメリのみ_restore_pp_if_any_move_empty`（3イベント共通の単一関数） |

- `ON_SWITCH_IN`: 場に出た時点でPPが0の技があれば即座に発動する。
- `ON_ITEM_ENABLED`: マジックルーム解除・ぶきよう解除等でアイテムが無効化状態から
  有効に戻ったとき、その時点でPPが0の技があれば発動する
  （item_manager.remove_disabled_reason が有効状態の変化時にのみ発火するため、
  無効化されていなかった場合は発火自体されず二重発動の心配はない）。
- `ON_FORCE_BERRY_TRIGGER`: むしくい・ついばむ・ほおばる・おちゃかい経由の強制発動
  （`battle.item_manager.force_trigger_berry`）で、PPが0の技があれば発動する。
  対象がいなければ何もせず戻り、force_trigger_berry 側の最終フォールバックで
  無条件に消費される（他の状態異常回復きのみと同じ挙動）。

回復対象の選定は「PPが0の技が複数ある場合、最後に使われた技（`mon.pp_consumed_move`）が
PP0であればそれを優先し、なければ技リストの先頭にあるPP0の技」というルール
（一次情報: 第五世代以降の仕様）を `himeri_select_move_to_restore()` に共通化する。

### 3. なげつけるで投げられたとき

なげつけるの効果は `handlers/move_attack.py` の `なげつける_apply_item_effect` に
アイテムごとの分岐として既に実装されている（`elif item_name == "ヒメリのみ":`）。
このハンドラは Event 経由ではなく `なげつける` 技自身の `ON_HIT` ハンドラ内で
直接処理されるため、`data/item.py` 側に別途登録は不要。回復対象の選定は
`himeri_select_move_to_restore()` を共通利用する。

### 4. うらみ・ぶきみなじゅもんでPPが直接減らされたとき

うらみ・ぶきみなじゅもんは `Move.modify_pp()` で対象のPPを直接減らすため、通常の
PP消費経路（`move_executor._consume_pp`）を経由せず `Event.ON_PP_CONSUMED` が
自動では発火しない。`handlers/move_status.py` の `うらみ_deplete_pp` と
`handlers/move_attack.py` の `ぶきみなじゅもん_reduce_defender_pp` の両方で、
PP減少後に改めて `Event.ON_PP_CONSUMED` を明示的に発火する
（`ctx.derive(attacker=defender, defender=battle.foe(defender), move=move)` で
「PPが減った側」を attacker ロールに読み替えた `AttackContext` を作り、
`ヒメリのみ_restore_pp` の `subject_spec="attacker:self"` にマッチさせる）。

## 注意
- `himeri_pp_restore_cap()` はじゅくせい判定込みの共通回復量（10 / 20）を返す既存関数。
- キョダイゲンスイ（ダイマックス関連効果）は対象範囲外（チャンピオンズ形式に
  ダイマックスは存在しない）のため実装しない。

## テスト観点
- 自分の技のPPが0になったとき、その技のPPが回復し（じゅくせい時は20）、アイテムが
  消費されること。PPが0でない場合は発動しないこと。
- 場に出た直後にPPが0の技を持っていた場合に即座に発動すること。
- うらみ・ぶきみなじゅもんでPPが0になったときに発動すること。
- むしくい・ついばむでヒメリのみを奪われて食べられたとき、奪った側の技のPPが
  回復すること（対象技がなければ発動せず、アイテムのみ消費されること）。
- なげつけるでヒメリのみを投げたとき、相手のPPが0の技が回復すること。
- マジックルーム／ぶきようでアイテムが無効化されている間はPPが0になっても発動せず、
  無効化解除時にPPが0のままなら即座に発動すること。
- PPが0の技が複数ある場合、最後に使われた技が優先されること。
