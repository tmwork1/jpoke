# じゃくてんほけん 実装計画

## 効果概要
効果抜群のダメージを受けたとき、こうげき（A）と とくこう（C）を +2 ずつ上昇させる（消費）。
ただしダメージ固定技（一撃必殺技を除く）はタイプ相性上抜群であっても発動しない。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_DAMAGE_HIT`（`AttackContext`、`.internal/spec/turn.md`未掲載イベント。実際にダメージを与えた場合のみ`MoveExecutor._execute_hit`から発火されるため、まもる・特性無効化・みがわり・ばけのかわ/アイスフェイスで防いだ場合は自動的に発火しない） |
| subject_spec | `"defender:self"` |
| priority | 100（他のON_DAMAGE_HITハンドラ同様の既定値。既存の同種アイテムハンドラ（じゅうでんち等）に合わせる） |
| once | 不要（発動可否は`battle.modify_stats`の戻り値で判定し、都度消費処理を行う） |
| 消費条件 | `battle.modify_stats`で実際にランクが変化した場合のみ`battle.item_manager.consume_item(mon)`を呼ぶ |

### 発動条件の詳細
- `ctx.move.has_flag("fixed_damage")`が真の場合は発動しない（いかりのまえば・カタストロフィ・
  ちきゅうなげ・ナイトヘッドなど、タイプ相性を無視した固定ダメージ技）。
  - 一撃必殺技（`ohko`フラグ）は`fixed_damage`フラグを持たせず、通常通りタイプ相性判定を行う
    （耐えた場合に弱点なら発動する仕様のため）。
- `battle.query.is_super_effective(ctx)`（内部で`calc_def_type_modifier`を使用）で判定する。
  - かんそうはだ/もふもふ/あついしぼう/たいねつ等のダメージ量を変える特性は
    `ON_CALC_DAMAGE_MODIFIER`側の補正であり、`calc_def_type_modifier`には影響しないため、
    自然に「元のタイプ相性」でのみ判定される。
  - らんきりゅう・タールショットはタイプ相性自体（`ON_CALC_DEF_TYPE_MODIFIER`）を変更するため、
    自然に反映される。
- たんじゅん/あまのじゃくによるランク変化量の増減・反転、ランク上限/下限によるクランプは
  `battle.modify_stats`が`ON_BEFORE_MODIFY_STAT`を経由して汎用的に処理するため、
  本ハンドラ側で個別対応する必要はない。
- まけんき/かちきは`Event.ON_MODIFY_STAT`（技のランクダウン効果発生時、`ON_DAMAGE_HIT`より前に
  発火）で処理されるため、同時発動時は自然に先着する。
- どろぼう/はたきおとす/マジシャンは`Event.ON_DAMAGE_HIT`に登録されるが、本ハンドラの方が
  priorityが高い（先に評価される）ため、アイテムを失う前にランクが上昇する。
  優先度の妥当性は各アイテム・特性の既存実装（`data/item.py`, `data/ability.py`）を参照。

### ハンドラロジック（疑似コード）
```python
def じゃくてんほけん_boost_on_super_effective(battle, ctx, value):
    mon = ctx.defender
    if ctx.move.has_flag("fixed_damage"):
        return HandlerReturn(value=value)
    if battle.query.is_super_effective(ctx):
        changes = battle.modify_stats(mon, {"atk": +2, "spa": +2})
        if changes:  # ランク上限などで不発の場合は消費しない
            _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)
```

## 既知の制限
- HPが1の状態でこらえるにより攻撃技のダメージが0に補正された場合、`ON_DAMAGE_HIT`自体が
  発火しないため発動しない（本来は発動するべき）。詳細は`.internal/spec/items/じゃくてんほけん.md`の
  備考を参照。技実行フローの横断的な修正が必要なため本アイテム単体では未対応。

## テスト観点
- 効果抜群のダメージを受けたとき A・C が +2 ずつ上昇すること
- 効果抜群でない（等倍・いまひとつ）の攻撃では発動しないこと
- 発動後にアイテムが消費されること
- ランク上限（+6）で片方のみ上限の場合はもう片方だけ上昇し、アイテムが消費されること
- 両方ランク上限（+6）の場合は発動しない（アイテムが消費されない）こと
- ダメージ固定技（例: ちきゅうなげ）で弱点タイプでも発動しないこと
- 一撃必殺技（例: じわれ）を弱点タイプで受けて倒れなかった場合は発動すること
- まもる・特性等でダメージを無効化された場合は発動しないこと
- みがわりで攻撃を防いだ場合は発動しないこと
