# もうどく 実装計画書

## 実装対象

もうどくによるターン終了時ダメージ処理。

## ハンドラ概要

| ハンドラ名 | イベント | プライオリティ | subject_spec | 処理内容 |
|-----------|---------|---------------|-------------|---------|
| `もうどく_damage` | `ON_TURN_END` | 90 | `source:self` | ターン経過でダメージ増加 |

プライオリティ根拠: `.internal/spec/turn.md` の `Event.ON_TURN_END` セクションに「90: どく・もうどくダメージ」と明記されている。

## ダメージ計算ロジック

```
1. ailment_manager.tick(mon) で経過ターン数をインクリメント
2. 有効ターン数 = min(15, elapsed_turns)  ← 上限15
3. damage = max(1, mon.max_hp * 有効ターン数 // 16)  ← 最小1
4. battle.modify_hp(mon, v=-damage, reason="poison")
```

## 実装場所

- ハンドラ実装: `src/jpoke/handlers/ailment.py` → `もうどく_damage` 関数
- ハンドラ登録: `src/jpoke/data/ailment.py` → `AILMENTS["もうどく"]`

## 注意点

- 経過ターン数の上限は 15（最大ダメージは最大HP × 15/16）
- ダメージの最小値は 1（最大HPが小さいポケモンへの対応）
- 交代時は `Ailment.reset_on_switch_out()` が `elapsed_turns = 0` にリセットする（自動）
- `battle.modify_hp()` を使うこと（`mon.hp` への直接代入禁止）
- `tick()` を先に呼んでから計算すること（`elapsed_turns=0` はダメージ0になるため）
