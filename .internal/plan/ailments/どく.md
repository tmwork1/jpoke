# どく 実装計画書

## 概要

どく状態異常の実装。ターン終了時に最大HPの1/8（最小1）のダメージ。

## 実装対象

### 1. どく_damage（Event.ON_TURN_END）

**ファイル**: `handlers/ailment.py`  
**登録**: `data/ailment.py`

- どく状態のポケモンのターン終了時に最大HPの1/8ダメージを与える
- ダメージ計算: `max(1, mon.max_hp // 8)`（切り捨て、最小1）
- `modify_hp(mon, v=-damage, reason="poison")` で呼び出す
  - reason="poison" により `ON_MODIFY_POISON_DAMAGE` が発火され、ポイズンヒール特性等が正しく動作する
- subject_spec: `"source:self"` （EventContext）
- priority: 90
  - 根拠: `.internal/spec/turn.md` の `Event.ON_TURN_END` より「どく・もうどくダメージ」は Priority 90

## テスト方針

- どく状態付与後に `end_turn` を呼び、HP が `max_hp - max_hp // 8` になることを確認する
- マジックガード・ポイズンヒールとのインタラクションは各特性のテストで確認する

## 注意点

- `modify_hp` に `v=0` を渡すと処理がスキップされるため、`max(1, ...)` で最小値1を保証する
- どくタイプ・はがねタイプへの付与不可は `AilmentManager._can_apply_by_type` で実装済み
- マジックガード（`ON_MODIFY_NON_MOVE_DAMAGE` で0に変換）とポイズンヒール（`ON_MODIFY_POISON_DAMAGE` で回復に変換）は特性側で実装済み
