# 計画書: コットンガード

## 仕様要約

- 自分の「ぼうぎょ」ランクが3段階上がる
- タイプ: くさ、変化、PP: 12（champions 基準）、対象: 自分
- 純粋な自己ランク上昇技（プラスのみ、混在なし）

出典: `.internal/spec/moves/コットンガード.md`

---

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100 (省略) | `attacker:self` (省略) | ぼうぎょ+3 を適用 |

---

## Priority 根拠

`.internal/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`コスモパワー_modify_attacker_stats` 等）がすべて priority=100 (デフォルト) で登録されているため、100 を使用する。

---

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: `コスモパワー_modify_attacker_stats` の後、`こらえる_apply` の前（五十音順、実装済み）

```python
def コットンガード_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"def": 3})
```

### `src/jpoke/data/moves/move_ka.py`

`"コットンガード"` エントリに `target="self"` を指定する（`target` を省略するとデフォルトの
`"foe"` になり、`is_blocked_by_protect` / `check_hit_substitute`（みがわり判定）が
誤って「相手を対象とする技」として扱われ、相手のまもる・みがわりで本来防がれない
はずの自己バフが防がれてしまうため必須）:

```python
    "コットンガード": MoveData(
        type="くさ",
        category="status",
        pp=12,
        target="self",
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.コットンガード_modify_attacker_stats,
            )
        }
    ),
```

---

## 注意点・エッジケース

- ランクがすでに+6の場合、`battle.modify_stats` が空の結果を返し技が失敗する（フレームワーク側で処理、ハンドラ側にガード不要）
- 追加効果・急所判定は対象外（変化技のため）
- `target="self"` を指定しないと、相手の「まもる」「みがわり」で誤って防がれてしまうため必須
  （`かたくなる`・`こうごうせい` 等と同様のパターン）
- リーサル計算: 対象外（ダメージを与えない変化技、n/a）
