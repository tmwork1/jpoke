# 計画書: がむしゃら

## 仕様要約

- ノーマルタイプ / 物理 / 威力-（固定ダメージ） / 命中100 / PP5 / 対象: 相手1体 / 優先度0 / 直接攻撃
- 命中すると、相手の残りHPから自分の残りHPを引いた数値分の固定ダメージを与える。
- 相手の残りHPが自分の残りHP以下の場合、無効化されたときと同様に技が失敗する。
- ダメージ計算式（攻撃・防御・威力・急所・タイプ相性倍率）を使用しない固定ダメージ技。
- タイプ相性による無効化（0倍）のみ適用される（ノーマル技のためゴーストタイプに無効）。
- 特性おやこあいでも連続攻撃にはならない。
- リーサル計算は対象外（この技だけでは相手を倒せないため）。

## ハンドラ構成

### `がむしゃら_can_use`（`handlers/move_attack.py`, `Event.ON_TRY_MOVE_1`, priority=30）

相手の残りHPが自分の残りHP以下の場合、`LogCode.MOVE_IMMUNED` を記録して
`HandlerReturn(value=False, stop_event=True)` を返し、技を失敗させる。

```python
def がむしゃら_can_use(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """がむしゃらの使用可否を判定する。

    相手の残りHPが自分の残りHP以下の場合、無効化されたときと同様に失敗する。
    """
    if ctx.defender.hp <= ctx.attacker.hp:
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_IMMUNED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="がむしゃら")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### `がむしゃら_modify_damage`（`handlers/move_attack.py`, `Event.ON_MODIFY_MOVE_DAMAGE`）

`がむしゃら_can_use` を通過した場合のみ到達するため、常に正の値になるが、
念のため `max(0, ...)` で下限0を保証する（既存実装を流用）。

```python
def がむしゃら_modify_damage(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """がむしゃらのダメージを計算する（相手の残りHP−自分の残りHP）。"""
    value = max(0, ctx.defender.hp - ctx.attacker.hp)
    return HandlerReturn(value=value)
```

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_TRY_MOVE_1` に「30 | カウンター・ミラーコート: 適格ダメージ無」
の行があり、同様に「使用条件を満たさない場合に技自体を失敗させる」固定ダメージ技のパターンである
がむしゃらも、同じ priority=30 に「がむしゃら: 相手の残りHPが自分以下」として追加する。

タイプ相性判定（ゴースト無効）は `ON_TRY_MOVE_1` より後（`_check_hit_by_type`）で行われるため、
HP条件チェックが先に評価される。相手がゴーストタイプかつHP条件も満たさない場合は
`がむしゃら_can_use` の時点で失敗が確定し、型判定には到達しない
（いずれの経路でもダメージなし・技失敗という結果は変わらない）。

## 実装箇所

### data/move.py

```python
    "がむしゃら": MoveData(
        type="ノーマル",
        category="physical",
        pp=5,
        power=0,
        accuracy=100,
        flags={"contact", "fixed_damage"},
        handlers={
            Event.ON_TRY_MOVE_1: h.MoveHandler(
                ha.がむしゃら_can_use,
                subject_spec="attacker:self",
                priority=30,
            ),
            Event.ON_MODIFY_MOVE_DAMAGE: h.MoveHandler(
                ha.がむしゃら_modify_damage,
                subject_spec="attacker:self",
            )
        }
    ),
```

- 修正前は `flags={"contact"}` のみで `fixed_damage` が欠落していたため追加する
  （じゃくてんほけん・ナゾのみの誤発動防止。ノーマル技は元々抜群を取れないため実害はほぼ無いが、
  他の固定ダメージ技との整合性のため設定する）。
- 修正前は HP条件を満たさない場合の失敗処理が無く、`ON_MODIFY_MOVE_DAMAGE` で
  `max(0, ...)` により黙って0ダメージになるだけだった（技としては「成功」扱いになってしまい、
  やけっぱち・じだんだ等の「前のターンの行動が失敗したか」の判定に反映されない誤りがあった）。
  `ON_TRY_MOVE_1` での明示的な失敗処理に修正する。

### handlers/ability.py（`おやこあい_modify_hit_count`）

```python
def おやこあい_modify_hit_count(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """おやこあい特性: 単発攻撃技を2ヒット化する。

    がむしゃらはHPの変動有無に関わらずいかなる状況でも連続攻撃にならないため対象外とする。
    """
    if ctx.move.name == "がむしゃら":
        return HandlerReturn(value=value)
    if ctx.move.is_attack and ctx.move.max_hits == 1:
        value = 2
    return HandlerReturn(value=value)
```

- 修正前は `ctx.move.is_attack and ctx.move.max_hits == 1` のみの判定だったため、
  がむしゃらが単発の物理技として2ヒット化されてしまう誤りがあった（`.internal/spec/abilities/おやこあい.md`
  に「がむしゃら - きのみの効果などでHPが変動した場合も連続攻撃にならない」と明記されている）。
  がむしゃらを名指しで除外する。
- いのちがけ・一撃必殺技等、他にもおやこあいの対象外規定がある技は本計画書の対象外
  （それぞれの技のレビュー時に個別対応する）。

## リーサル計算

- `.internal/progress/move.md` の「リーサル実装」列は n/a（対象外）。
  がむしゃらのダメージは常に「相手の残りHP−自分の残りHP」であり、
  結果として相手の残りHPは自分の残りHP以上（≧1）にしかならないため、この技単独では
  相手をひんしにできない。よってリーサル計算ハンドラは実装しない。

## 注意点・エッジケース

- 相手の残りHPが自分の残りHPと等しい場合も無効（「以下」であって「未満」ではない）。
- みがわり状態でも、みがわり人形の残りHPではなく本体の残りHPを使ってHP条件判定・ダメージ計算を行う
  （`ctx.attacker.hp` / `ctx.defender.hp` は元々ポケモン本体のHPを参照するため、追加対応は不要）。
- タイプ相性でゴーストタイプに無効化される場合は `_check_hit_by_type`（`ON_TRY_MOVE_1` より後）で
  処理されるため、既存の型無効化ロジックをそのまま利用できる（追加対応不要）。
