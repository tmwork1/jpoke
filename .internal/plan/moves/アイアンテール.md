# アイアンテール 実装計画書（再レビュー）

## 仕様要約

`.internal/spec/moves/アイアンテール.md` 参照。

* タイプ: はがね / 分類: 物理 / 威力: 100 / 命中率: 75 / PP: 16（Championsでは全技PPが
  8・12・16・20に統一。SVまでのPPは15） / 対象: 相手1体
* 直接攻撃。追加効果として、30%の確率で相手の『ぼうぎょ』ランクを1段階下げる。

## 既存実装の確認結果と問題点

`src/jpoke/handlers/move_attack.py` の `アイアンテール_lower_defender_def` が
`Event.ON_DAMAGE_HIT` で `modify_defender_stats(battle, ctx, value, stats={"def": -1}, chance=0.3)`
を実行しており、同種のぼうぎょダウン追加効果を持つ `Gのちから_lower_defender_def` /
`ほのおのムチ_lower_defender_def` と同じ実装パターンになっている。イベント・優先度
（デフォルト100・`.internal/spec/turn.md` の「100 スケイルショットによる能力変化」相当）も
問題ない。

一方、`src/jpoke/data/moves/move_a.py` の `MoveData` エントリに以下の誤りがあった。

* `pp=15`（SVまでの値）になっていたが、`src/jpoke/data/ps-champ-ja/moves.json` /
  `.internal/champions/changes_from_sv.md`（技のPPが8・12・16・20の4段階に統一）によれば
  Championsでは `16` が正しい値のため修正する。

`flags={"contact", "secondary_effect"}` は一次情報（直接攻撃○、ちからずく対象技）と
一致しており問題ない。
