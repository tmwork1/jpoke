# 計画書: 攻撃技_壁解除系（かわらわり・サイコファング）

## 仕様要約

### かわらわり

- かくとう物理技（威力75・命中100・PP15(16)・優先度0・直接攻撃）
- 命中時、相手側の「ひかりのかべ」「リフレクター」「オーロラベール」を解除する
- 壁解除は追加効果ではない（りんぷん・おんみつマント・ちからずくの対象外）
- タイプ相性/特性/まもる等で無効化された場合は壁を解除できない

### サイコファング

- エスパー物理技（威力85・命中100・PP10・優先度0・直接攻撃・かみつく技）
- 命中時、相手側の「ひかりのかべ」「リフレクター」「オーロラベール」を解除する
- 壁解除は追加効果ではない（りんぷん・おんみつマント・ちからずくの対象外）
- タイプ相性/特性/まもる等で無効化された場合は壁を解除できない

## Priority 根拠

### ON_HIT（壁解除）

`.internal/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに `priority=30: かわらわり等の効果発動` と記載がある。

ただし `ON_APPLY_MOVE` はコードの `enums/event.py` に存在しないイベントであり、実際の壁解除処理は「ダメージを与えた後」に行われる必要がある（無効化されたときは壁を解除しない仕様のため）。

実際の参考実装として以下が存在する：

- `レイジングブル_break_screens`（`handlers/move_attack.py` l.1968）: 同様の壁3種解除を `ON_HIT priority=100` で実装
- `こうそくスピン_clear_field`（`handlers/move_attack.py` l.595）: 場の設置物解除を `ON_HIT priority=100` で実装

これらを根拠として `ON_HIT priority=100`（デフォルト）とする。

## 実装箇所

### handlers/move_attack.py

`かわらわり` は「か」行、`サイコファング` は「さ」行の五十音順の位置に追加する。

```python
def かわらわり_break_screens(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """かわらわり: 相手側のリフレクター・ひかりのかべ・オーロラベールを解除する。

    壁解除は追加効果ではないため、りんぷん・おんみつマント・ちからずくの対象外。
    技が無効化されなかった場合（ON_HITに到達した場合）のみ発動する。
    """
    defender_side = battle.get_side(ctx.defender)
    for wall in ("リフレクター", "ひかりのかべ", "オーロラベール"):
        defender_side.deactivate(wall)
    return HandlerReturn(value=value)


def サイコファング_break_screens(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """サイコファング: 相手側のリフレクター・ひかりのかべ・オーロラベールを解除する。

    壁解除は追加効果ではないため、りんぷん・おんみつマント・ちからずくの対象外。
    技が無効化されなかった場合（ON_HITに到達した場合）のみ発動する。
    """
    defender_side = battle.get_side(ctx.defender)
    for wall in ("リフレクター", "ひかりのかべ", "オーロラベール"):
        defender_side.deactivate(wall)
    return HandlerReturn(value=value)
```

### data/move.py

`"かわらわり"` と `"サイコファング"` の各エントリに `handlers` を追加する（`ha` は `handlers.move_attack` のエイリアス）。

## 注意点・エッジケース（review-test 向け）

### 壁解除タイミング

- `ON_HIT` はダメージが実際に通った後に発火するため、タイプ相性無効・特性無効・まもる等で無効化された場合は自動的に壁解除されない（イベント自体が発火しない）
- `SideFieldManager.deactivate` は対象フィールドが存在しない場合は何もしないため、壁が張られていないケースのガード不要

### 壁解除は追加効果でない点

- `labels` に `"secondary_effect"` を付けない
- `ON_MODIFY_SECONDARY_CHANCE` を経由しないため `ちからずく` の威力上昇対象にならない
- `りんぷん`・`おんみつマント` で防がれない

### みがわり状態への挙動

- turn.md の `ON_HIT` 説明には「みがわりに被弾しても発動」とある
- みがわり状態の相手に命中した場合も `ON_HIT` は発火し、壁解除が実行される
- 仕様書に例外記述はないため、この挙動は正しい

### かわらわりとサイコファングの壁解除範囲

- 両技とも「ひかりのかべ」「リフレクター」「オーロラベール」の3種のみを解除する
- きりばらい（move_status.py l.458）は「しんぴのまもり」「しろいきり」も解除するが、かわらわり・サイコファングの仕様にはない

### テストケース例

1. かわらわりが命中後、相手側のリフレクターが解除される
2. かわらわりが命中後、相手側のひかりのかべが解除される
3. かわらわりが命中後、相手側のオーロラベールが解除される
4. かわらわりが命中後、壁3種がすべて同時に解除される
5. まもるで無効化されたときは壁が解除されない
6. 壁が張られていない場合でもエラーなく動作する
7. サイコファングが命中後、相手側のリフレクターが解除される
8. サイコファングが命中後、まもるで無効化されたときは壁が解除されない
