# 計画書: ひっくりかえす

## 仕様要約

- タイプ: あく / 分類: 変化 / PP 20（champions/move_list.txt には未掲載だが、
  PP20はchampions PP統一4段階の1つに一致するため変更不要） / 命中: 必中
  （`accuracy=None`） / 優先度: 0 / 対象: 相手1体選択
- 効果: 相手にかかっているすべての能力ランク変化のプラスマイナスを反転する
- 失敗条件: 相手のランク補正がすべて±0の場合は失敗する
- まもる状態で防がれる（wiki: まもる ○）
- マジックコート状態で反射される（wiki: マジックコート ○）
- よこどりでは奪われない（wiki: よこどり ×）
- みがわり状態の相手には防がれる（wiki「技の仕様」節に明記）
- **重要**: この技によるランク反転は、クリアボディ／ミラーアーマー等の特性・
  しろいきり状態・クリアチャームで防げない。また、たんじゅん／あまのじゃくの
  影響を受けない。さらに、まけんき／かちき／だっしゅつパック／びんじょう／
  ものまねハーブのいずれも発動しない。つまり通常のランク変化イベント
  （`ON_BEFORE_MODIFY_STAT` / `ON_MODIFY_STAT`）を一切経由してはならない。

## Priority 根拠

`.internal/spec/turn.md` に `ON_STATUS_HIT` 自体の記載はない（ダメージ計算内部と同様の
汎用変化技適用イベントの扱い）。同種の「失敗条件チェック＋効果適用を1つの
`ON_STATUS_HIT` ハンドラにまとめる」パターンは `フラワーヒール_heal_defender`
（HP満タンなら失敗、そうでなければ回復）が既に採用しており、`priority=100`
（`MoveHandler` のデフォルト）で登録されている。本技も同じパターンとして
`priority=100` を採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`（実装済み・変更なし）

```python
def ひっくりかえす_invert_ranks(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ひっくりかえすの効果: 相手の全能力ランク変化を反転させる。

    全ランクが0の場合は技が失敗する。
    """
    mon = ctx.defender
    if all(v == 0 for v in mon.rank.values()):
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="能力ランクに変化がない"),
        )
        return HandlerReturn(value=False, stop_event=True)
    for stat in mon.rank:
        mon.rank[stat] = -mon.rank[stat]
    return HandlerReturn(value=value)
```

- `mon.rank[stat]` を直接反転することで、`battle.modify_stats()` が内部で発火する
  `ON_BEFORE_MODIFY_STAT`（クリアボディ／ミラーアーマー／しろいきり／
  クリアチャームによるブロック）、`ON_MODIFY_STAT`（まけんき／かちき／
  だっしゅつパック／びんじょう／ものまねハーブの発動、`stat_lowered_this_turn`
  ／`stat_raised_this_turn` フラグの設定によるうっぷんばらし／しっとのほのお／
  みわくのボイスへの連動）を一切経由しない。これは wiki の「技の仕様」節に
  記載された一連の除外仕様と一致する、意図的な実装である。
- 既存の `パワースワップ_swap_ranks`（`atk_rank[stat], def_rank[stat] = ...`）も
  同様に `rank` 辞書を直接操作しており、本技と同じ設計方針。
- ランクは既に -6〜+6 の範囲に収まっているため、符号反転後も範囲外にならず、
  クランプ処理は不要。

## 注意点・エッジケース（review-test 向け）

- **全ランク±0で失敗**: 相手の6能力（こうげき・ぼうぎょ・とくこう・とくぼう・
  すばやさ、命中率・回避率を含む `rank` 辞書の全キー）がすべて0のとき、技は
  失敗しランクは変化しない。
- **複数ランクの同時反転**: 複数の能力にランク変化がある場合、すべて同時に
  符号反転する。
- **プラス→マイナス、マイナス→プラスの両方**: 上昇していたランクが下降し、
  下降していたランクが上昇することを確認する。
- **みがわり**: 相手がみがわり状態のとき、この技は防がれる（`bypass_substitute`
  フラグを付与していないため、変化技の汎用みがわり判定で対応済み。個別の
  ハンドラ実装は不要）。
- **まもる**: まもる状態の相手には防がれる（汎用処理で対応済み）。
- **マジックコート**: マジックコート状態の相手に使用すると技が反射される
  （汎用処理で対応済み）。反射後は使用者と対象が入れ替わった状態で
  `ひっくりかえす_invert_ranks` が再評価される。
- **必中判定と回避中の相手**: 必中技だが、そらをとぶ等で「姿を隠している」
  相手には当たらない（汎用の必中技×溜め技回避処理で対応済み。個別実装不要）。
- **subject_spec**: `MoveHandler` のデフォルト `attacker:self` を使用。
  `AttackContext` 内で `ctx.defender` を直接参照するため、対象選択自体には
  影響しない。

## 本実装での対象外（スコープ外）

- Zワザによる付加効果（自分のこうげきを1段階上げる）。本プロジェクトは
  Zワザ非対応のためスコープ外。
