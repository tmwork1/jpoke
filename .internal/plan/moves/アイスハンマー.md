# アイスハンマー 実装計画書（再レビュー）

## 仕様要約

`.internal/spec/moves/アイスハンマー.md` 参照。

* タイプ: こおり / 分類: 物理 / 威力: 100 / 命中率: 90 / PP: 12（Championsでは全技PPが
  8・12・16・20に統一。SVまでのPPは10） / 対象: 相手1体
* 直接攻撃（`contact`）、パンチ技（`punch`。てつのこぶしで威力1.2倍）。
* 技が成功すると、自分の『すばやさ』ランクが1段階下がる（確定効果）。

## 既存実装の確認結果と問題点

`src/jpoke/data/moves/move_a.py` の `アイスハンマー` エントリ、
`src/jpoke/handlers/move_attack.py` の `アイスハンマー_lower_attacker_spe` を確認した。

* `pp=12` は既に正しい（Championsのpp統一ルールに合致）。
* `flags={"contact", "punch", "secondary_effect"}` のうち `secondary_effect` が誤り。
  `.internal/wiki/abilities/ちからずく.html` の一次情報によれば、ちからずくの対象技は
  「相手のランク補正を下げる技」「自分のランク補正を上げる技」等に限られ、
  「自分のランク補正を下げる技」は対象外。同種の技である `アームハンマー`
  （`.internal/spec/moves/アームハンマー.md` 参照）でも同様の判断で `secondary_effect` を
  付与していない。アイスハンマーの速さダウンは自分自身への確定効果であるため、
  `secondary_effect` フラグを削除する。
* `アイスハンマー_lower_attacker_spe` ハンドラの実装自体（`modify_attacker_stats` で
  `spe: -1` を確定発動）は仕様と一致しており修正不要。

```python
"アイスハンマー": MoveData(
    type="こおり",
    category="physical",
    pp=12,
    power=100,
    accuracy=90,
    flags={"contact", "punch"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.アイスハンマー_lower_attacker_spe,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 命中後、自分の『すばやさ』ランクを1段階下げる（確定効果） |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def アイスハンマー_lower_attacker_spe(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"spe": -1})
```

`modify_attacker_stats`（`handlers/move.py`）の `chance` 既定値は1のため常に発動する。
自分自身への確定効果であり、`resolve_secondary_chance` 経由の「追加効果」判定
（ちからずく等）の対象にはならない（`flags` に `secondary_effect` を付与しない）。

## Priority 根拠

自分自身への確定ランク低下は既存の同種ハンドラ（アームハンマー・りゅうせいぐん・
リーフストーム・いじげんラッシュ等）と同じ `ON_HIT` / priority 100（デフォルト）の
パターンに従う。`ON_HIT` は `.internal/spec/turn.md` に「10 自分のランクが下がる反動」として
掲載されているが、本プロジェクトの既存の同種ハンドラはいずれも優先度指定なし
（デフォルト100）で実装されているため、既存実装との一貫性を優先してそのパターンに従う。

## 注意点・エッジケース（review-test 向け）

1. **ちからずく等「追加効果」系特性・アイテムとの非該当**
   - 自分自身への確定ランク低下は「追加効果」ではないため、ちからずく・てんのめぐみ
     （`secondary_effect` フラグを参照する特性）の対象にならない。`flags` から
     `secondary_effect` を削除するのが正しい。
2. **まもる貫通なし**
   - 通常の攻撃技であり、`まもる` 等の対象になる。個別のガードは不要
     （`unprotectable` フラグ等は付与しない）。
3. **てつのこぶし（Iron Fist）との相互作用**
   - `punch` フラグにより、てつのこぶし所持ポケモンが使用すると威力が1.2倍になる。
4. **テスト方針**
   - 100%固定効果のため `secondary_chance` の調整は不要。命中後に自分の
     『すばやさ』ランクが1段階下がることを確認する通常テストを追加する。
   - ちからずく所持時も威力上昇なし・すばやさ低下は通常通り発動することを確認する
     テストを追加する（`アームハンマー` の既存テストと同パターン）。
