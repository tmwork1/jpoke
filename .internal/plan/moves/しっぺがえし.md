# しっぺがえし 実装計画書

参照: .internal/spec/moves/しっぺがえし.md

## 現状

* `src/jpoke/data/move.py` の `"しっぺがえし"` エントリは以下の通り実装済み。
  - type="あく", category="physical", pp=10, power=50, accuracy=100
  - flags={"contact"}
  - `Event.ON_CALC_POWER_MODIFIER`: `しっぺがえし_double_power_when_second`（`MoveHandler` の
    デフォルト `subject_spec="attacker:self"` を使用）

## 発見した不具合と修正

* `しっぺがえし_double_power_when_second` は修正前、`battle.query.is_second_actor(attacker_player)`
  のみを条件にしていた。しかし `is_second_actor` はそのターンの行動順（優先度・素早さ）だけを見て
  判定しており、「相手がその場に交代してきたばかりで、まだ技を使用していない」ケースを
  区別できていなかった。
* `_run_switch_phase`（`src/jpoke/core/turn_controller.py`）では、交代したプレイヤーの
  インデックスも `action_order` に追加される。そのため、相手が交代しただけで攻撃していない
  ターンでも、しっぺがえし使用者は「相手より後に行動した」と誤判定され、威力2倍が誤って
  適用されてしまう。
* 一次情報（.internal/spec/moves/しっぺがえし.md 参照）では、第五世代以降は交代直後で技を
  未使用の相手に対しては威力2倍にならないと明記されている。
* 既存の類似実装 `フォーカスレンズ_boost_accuracy_second`（`src/jpoke/handlers/item.py`）が、
  同様の「相手がそのターンに行動済みか」を判定する際に
  `battle.player_states[defender_player].has_switched` を先にチェックして、交代直後の相手を
  除外している。この既存パターンに合わせて修正する。

```python
def しっぺがえし_double_power_when_second(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """しっぺがえし: 相手（対象）がそのターンにすでに行動していると威力が2倍になる。

    そのターンに交代してきたばかりで技を未使用の相手には威力補正が乗らない
    （第五世代以降の仕様）。
    """
    defender_player = battle.get_player(ctx.defender)
    if battle.player_states[defender_player].has_switched:
        return HandlerReturn(value=value)

    attacker_player = battle.get_player(ctx.attacker)
    if battle.query.is_second_actor(attacker_player):
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)
```

* なお、「相手が反動技（リチャージ）で行動できなかった場合は威力2倍にならない」というケースは、
  `action_order` が行動順決定時に一括で確定し、リチャージによる行動失敗はその後の実行時
  （`ON_TRY_ACTION`）に判明するため、本プロジェクトの `is_second_actor` ベースの判定では
  区別できない。この点は他の類似実装（フォーカスレンズ等）にも存在する既知の制限であり、
  現状は対応を見送る（`.internal/spec/moves/しっぺがえし.md` に既知の制限として明記）。
