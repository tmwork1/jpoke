# 計画書: こんげんのはどう

## 仕様要約

- みずタイプ / 特殊 / 威力110 / 命中85 / PP10 / 対象: 相手全員（シングルバトルでは相手1体） / 優先度0 / 非接触
- `pulse`（波動）フラグを持つ。特性メガランチャーで威力1.5倍（既存の汎用ハンドラで処理済み）。
- 追加効果なし。

## ハンドラ構成

追加効果を持たず、`pulse` フラグのみで威力補正（メガランチャー）を表現できるため、専用ハンドラは不要。
`handlers={}` と `flags={"pulse"}` のみで基本ダメージ計算に委ねる。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、priority の検討対象イベントは存在しない。

## 実装箇所

### data/move.py への更新

既存の `"こんげんのはどう"` エントリは以下の通りで、仕様と一致している（変更不要）。

```python
    "こんげんのはどう": MoveData(
        type="みず",
        category="special",
        pp=10,
        power=110,
        accuracy=85,
        flags={"pulse"},
        handlers={},  # 追加効果なし
    ),
```

- `flags` に `contact` を含まないため非接触 → 仕様と一致。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ シングルバトルでは仕様と一致。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `flags={"pulse"}` により、メガランチャーの汎用ハンドラ
  （`handlers/ability.py:メガランチャー_modify_power`、`_modify_by_move_condition` で
  `move_flag="pulse"` を判定）が自動的に威力を1.5倍にする。

## リーサル計算

追加効果・確率分岐を持たない通常攻撃であり、リーサルハンドラは不要（`.internal/progress/move.md` で n/a）。

## 注意点・エッジケース

- 追加効果がないため、まもる・おうじゃのしるし・オウムがえし等の一般的な技判定は
  フレームワーク側の共通処理（`handlers/move_attack.py` の基本ダメージ計算経路）で
  自動的に処理され、本技固有のハンドラ実装は不要。
- 専用技（グラードン専用）としての使用可否制限は、本プロジェクトが技の学習可否を
  検証しない方針のため対象外。
