# 計画書: ずつき

## 仕様要約（.internal/spec/moves/ずつき.md 参照）

- ノーマルタイプ / 物理 / 威力70 / 命中100 / PP15 /
  対象: 1体選択 / 優先度0 / 接触あり
- 追加効果として、命中時に30%の確率で相手をひるませる。
- 追加効果に分類されるため、ちからずく（威力1.3倍・効果無効化）の対象技になる。
- `contact`（接触技）フラグを持つ。

## 既存実装との差分

既存実装（`data/move.py` の `"ずつき"` エントリ）は仕様と一致しており、修正は不要。

1. **基本情報は仕様通り**
   `type="ノーマル", category="physical", pp=15, power=70, accuracy=100` で一致。

2. **追加効果ハンドラ本体（`ずつき_apply_flinch`）は仕様通り実装済み**
   `apply_volatile_to_defender(battle, ctx, value, volatile="ひるみ", chance=0.3)`
   で、30%発動の仕様と一致している。変更不要。

3. **`flags` は既に `{"contact", "secondary_effect"}` が設定済み**
   接触技かつ追加効果を持つ技として正しく分類されている。変更不要。

## ハンドラ構成

### handlers/move_attack.py: `ずつき_apply_flinch`（既存・変更なし）

```python
def ずつき_apply_flinch(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_defender(battle, ctx, value, volatile="ひるみ", chance=0.3)
```

- `apply_volatile_to_defender`（`handlers/move.py`）は内部で `battle.resolve_secondary_chance` を
  呼び出すため、`Event.ON_MODIFY_SECONDARY_CHANCE` 経由でちからずく（確率0化・威力1.3倍）の
  ハンドラが自動的に適用される（`secondary_effect` フラグが前提）。
- イベント: `Event.ON_DAMAGE_HIT`（ダメージが実際に適用された後に発動。
  .internal/spec/turn.md の `## Event.ON_DAMAGE` 参照）。
  ダメージが発生しなかった場合（無効化・まもる等）はイベント自体が発火しないため、
  追加効果も発生しない仕様を自然に満たす。
- 優先度: デフォルト100（`MoveHandler` のデフォルト値）。
- subject_spec: `MoveHandler` デフォルト（`attacker:self`）のまま。

### data/move.py（既存・変更なし）

```python
"ずつき": MoveData(
    type="ノーマル",
    category="physical",
    pp=15,
    power=70,
    accuracy=100,
    flags={"contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ずつき_apply_flinch,
        )
    }
),
```

## リーサル計算

対象外（`.internal/progress/move.md` の「リーサル実装」列が `n/a`。ひるみ付与のみでHP分布に影響しない
追加効果のためリーサル計算ハンドラは不要）。

## 注意点・エッジケース

- ノーマルタイプの相手など、技自体が無効化された場合、`Event.ON_DAMAGE_HIT` 自体が発火しないため
  追加効果も発生しない。個別のガードは不要。
- まもるで防がれた場合も同様にダメージが発生せず `Event.ON_DAMAGE_HIT` が発火しないため、
  追加効果も発生しない。
- ひるみは `Event.ON_CHECK_ACTION` で消費される（`.internal/spec/turn.md` 参照）。
  ON_DAMAGE_HIT での付与後、相手が次のターンに行動しようとする際に判定される。
  すでに行動済みのポケモンにひるみが付与されてもそのターンは効果がない。
