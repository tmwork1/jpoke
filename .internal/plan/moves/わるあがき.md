# わるあがき 実装計画書

## 仕様要約（.internal/spec/moves/わるあがき.md 参照）

- 選択できる技が1つもないときに使用される特殊な物理技。威力50、必中、PP実質無制限。
- タイプなし扱い（タイプ相性・タイプ一致補正の影響を受けない）。
- 命中後、自分の最大HPの1/4（五捨五超入で丸め、最低1）を反動として受ける。
  この反動は与えたダメージ量に依存せず、特性いしあたま・マジックガードでも防げない
  （通常の反動技〔ワイルドボルト等〕とは異なる特殊な性質）。

## 実装方針

わるあがきは実在のポケモンが習得する技ではなく、`MOVES_SYMBOL`
（`data/moves/move_symbol.py`）に定義される記号技。
`CommandManager.resolve_move_from_command` が選択可能技なしの場合に
`Move("わるあがき")` を直接生成して使用するため、通常の技のような
習得可否・PP管理は考慮不要。実装対象はダメージ計算そのものと反動ダメージのみ。

### ハンドラ構成

| イベント | Priority | 役割 |
|----------|----------|------|
| `Event.ON_HIT` | 100（デフォルト、`MoveHandler` の既定値） | 自分の最大HPの1/4を反動ダメージとして与える |

### Priority 根拠

- `.internal/spec/turn.md` の `Event.ON_HIT` 節には反動ダメージ専用の記載はないが、
  同種の反動技（ワイルドボルト等）が `ON_HIT` をデフォルト優先度（100）で
  登録しているため、それに合わせる。

### subject_spec

- `ON_HIT`: `attacker:self`（`MoveHandler` のデフォルト）。
  攻撃フェーズ（`AttackContext`）のイベントであり、反動を受けるのは使用者（attacker）自身のため。

## 実装箇所

### handlers/move_attack.py（五十音順: わ行）

```python
def わるあがき_self_damage(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """わるあがきの反動ダメージ: 自分の最大HPの1/4（五捨五超入で丸め、最低1）を受ける。

    与えたダメージ量に依存しない固定割合の反動であり、特性いしあたま・マジックガード
    のどちらでも無効化されない特殊な性質を持つため、reason="self_cost" を用いて
    一般的な反動ダメージ（reason="recoil"）向けの無効化ハンドラの対象から外す。
    """
    recoil = max(1, round_half_down(ctx.attacker.max_hp / 4))
    return HandlerReturn(
        value=battle.modify_hp(ctx.attacker, v=-recoil, reason="self_cost", source=ctx.attacker)
    )
```

### data/moves/move_symbol.py（既存の「わるあがき」MoveData エントリーを修正）

## 修正が必要な既存実装上の誤り（レビュー時に発見）

1. **威力が40になっていた（正しくは50）。** wikiの技データ表・Pokémon LEGENDS アルセウス表の
   両方で威力50と明記されている。
2. **反動の丸め処理が単純切り捨てになっていた。** `battle.modify_hp(ctx.attacker, r=-1/4, ...)`
   は内部で `int(max_hp * r)` により0方向への切り捨てを行うため、
   例えば最大HP=119（119/4=29.75）の場合、正しくは30ダメージになるべきところ29ダメージに
   なってしまう。`round_half_down(max_hp / 4)` を用いて明示的に丸める必要がある。
3. **反動の `reason` が `"recoil"` になっていた。** `いしあたま_ignore_recoil`
   （`ON_MODIFY_NON_MOVE_DAMAGE` で `hp_change_reason == "recoil"` を無効化）や
   `マジックガード_ignore_damage`（`"recoil"` を含む多くの reason を無効化）に
   一致してしまい、いしあたま・マジックガード所持時に反動ダメージが誤って消えてしまう。
   `reason="self_cost"` に変更することでどちらの特性でも防げない仕様を再現する。

## 注意点・エッジケース（review-test 向け）

1. **タイプなし判定**: `move.type == ""` は `TYPE_MODIFIER.get(move_type, {})` が空辞書になり
   タイプ相性の影響を受けず、`move_type in attacker.data.types` も常に `False` になるため
   タイプ一致補正も乗らない。既存の `_calc_atk_type_modifier` / `calc_def_type_modifier`
   の実装のみで対応済みであり、わるあがき側での追加対応は不要
   （`_こんらん` の自傷ダメージと同様のパターン）。
2. **命中率**: `accuracy` を指定しない（`None`）ことで必中になる
   （`move_executor.py` の `_check_hit` が `accuracy is None` の場合 `True` を返す）。
3. **反動ダメージの発生条件**: `Event.ON_HIT` は技が実際に命中した場合のみ発火するため、
   まもるで防がれた場合・命中しなかった場合は自然と反動が発生しない。追加のガードは不要。
4. **ダメージが0でも反動は発生する**: こらえる・ばけのかわ・アイスフェイス等で最終ダメージが
   0になった場合でも、反動計算は `value`（与ダメ）を参照せず `ctx.attacker.max_hp` のみから
   算出するため、常に規定量の反動を受ける。実装上は元々この仕様を満たしていた。
5. **かなしばり・アンコール**: 相手の直前使用技がわるあがきの場合に失敗させる処理は
   `handlers/move_status.py` の `かなしばり_can_apply`（技名判定）・`non_encore` フラグ判定
   （アンコール側）に既に実装済みのため、わるあがき側の追加対応は不要。
6. **おんねん**: `non_onnen` フラグにより、わるあがきで相手をひんしにしてもPP消失効果が
   発動しないよう既に対応済み。
