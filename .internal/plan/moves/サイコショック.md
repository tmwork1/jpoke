# サイコショック 実装計画書

## 仕様要約（.internal/spec/moves/サイコショック.md 参照）

* タイプ: エスパー / 分類: 特殊 / 威力: 80 / 命中: 100 / PP: 12（Champions基準）
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果: 相手の『とくぼう』ではなく『ぼうぎょ』でダメージ計算をする（分類は特殊のまま）。追加効果なし。

## 現状の実装状況（バグ2件）

`src/jpoke/data/moves/move_sa.py` の `"サイコショック"` エントリには以下2件の不備があった。

1. **`pp=10` になっている（バグ）**
   `src/jpoke/data/ps-champ-ja/moves.json` では `サイコショック	エスパー	特殊	80	100	12	接×	守○` と
   PP=12。Gen9本家基準の10のままになっており、Champions移行漏れ。
2. **`flags` に `"physical_damage"` が欠落している（バグ）**
   ダメージ計算で防御側の『ぼうぎょ』を参照させるためのフラグが無く、`handlers={}` のみで
   「追加効果なし」とコメントされていた。この欠落により、実際には分類どおり『とくぼう』で
   ダメージ計算されてしまっていた（技の説明文と矛盾）。

`"physical_damage"` フラグは `src/jpoke/types/literals.py` に定義済みで、
`src/jpoke/core/query.py` の `Query.deals_physical_damage()` が

```python
move_category = self.resolve_move_category(attacker, move)
return (
    move_category == "physical"
    or move.has_flag("physical_damage")
)
```

の形で判定し、`src/jpoke/core/damage.py` の `_calc_final_defense()` が
`deals_physical_damage()` の結果で `"def"` / `"spd"` のどちらの防御ステータスを
参照するか切り替える。既存の仕組みをそのまま利用でき、新規ハンドラは不要。
同種の技である `しんぴのつるぎ`（`.internal/spec/moves/しんぴのつるぎ.md` /
`.internal/plan/moves/しんぴのつるぎ.md`）で先行して同じ修正が行われている。

### 修正後

```python
"サイコショック": MoveData(
    type="エスパー",
    category="special",
    pp=12,
    power=80,
    accuracy=100,
    flags={"physical_damage"},
    handlers={},  # 追加効果なし
),
```

## ハンドラ構成

新規ハンドラは不要。`flags` に `"physical_damage"` を追加するだけで、
`Query.deals_physical_damage()` 経由でダメージ計算・関連特性
（例: ステータス依存の特性やアイテム判定で `deals_physical_damage()` を参照するもの、
`handlers/ability.py` や `handlers/ability_paradox.py`、`handlers/field.py` の
既存参照箇所）にも自動的に反映される。

## Priority 根拠

新規イベント登録なし（既存の `ON_CALC_DEF_MODIFIER` 等のイベントフローに変更を加えない。
`deals_physical_damage()` の判定分岐を利用するのみ）。

## 注意点・エッジケース（review-test 向け）

1. **ダメージ計算での防御ステータス切り替え（修正の主眼）**
   - 相手の『とくぼう』が高く『ぼうぎょ』が低い場合、通常の特殊技より大ダメージになることを
     確認する。
   - 相手の『ぼうぎょ』ランクが上がっているとダメージが減ること、逆に『とくぼう』ランクが
     変化してもダメージが変わらないことを確認する。
2. **分類は特殊のまま**
   - 攻撃側の実攻撃力は『とくこう』（特殊）を参照する（`_calc_final_attack` は
     `move.category == "special"` のときに `"spa"` を参照するため、`physical_damage` フラグの
     追加で変化しない）。物理技として扱われる特性・アイテム（てつのこぶし等）の対象にはならない。
   - `resolve_move_category()` が返すカテゴリも `"special"` のまま変化しない
     （ちからもち等、物理カテゴリのみに反応する特性の対象外であることを確認する）。
3. **急所時のランク補正**
   - 急所時は防御側のランク下降を無視する処理（`_calc_final_defense` の
     `min(r_rank, 1)`）がそのまま適用される（既存ロジックを流用するため個別対応不要）。
4. **PP修正**
   - `pp=10`（Gen9本家基準）→ `pp=12`（Champions基準）に修正する。
5. **テスト方針**
   - とくぼうランクが変化してもダメージが変わらないこと、ぼうぎょランクが変化するとダメージが
     変わることを確認するテストを追加する（`しんぴのつるぎ` の既存テストと同型）。
   - 通常のダメージが発生することを確認するテストを追加する。
