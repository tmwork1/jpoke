# 実装計画書: リーフストーム（再レビュー）

## 仕様要約

`.internal/spec/moves/リーフストーム.md` 参照。

* タイプ: くさ / 分類: 特殊 / 威力: 130（Champions・第六世代以降の値） / 命中率: 90 /
  PP: 8（Champions。SVまでの本家基準は5） / 対象: 相手1体
* 直接攻撃ではない。命中して技が成功した後、確定で自分の『とくこう』ランクを2段階下げる
  （反動によるランク低下であり、追加効果ではない）。

## 既存実装の確認結果と問題点

`src/jpoke/handlers/move_attack.py` の `リーフストーム_sharply_lower_spa_C` が
`modify_attacker_stats(battle, ctx, value, stats={"spa": -2})`（`chance` 省略でデフォルト100%）
を実行しており、同系統の反動技である `オーバーヒート_lower_attacker_spa` /
`りゅうせいぐん_sharply_lower_spa_C` と全く同じ実装パターンになっている。
`Event.ON_HIT` での登録・`h.MoveHandler` のデフォルト priority（100）も同系統技と同一で
問題ない。

一方、`src/jpoke/data/moves/move_ra.py` の `MoveData` エントリに以下の誤りがあった。

* `pp=5`（Gen9本家基準の値）になっていたが、`src/jpoke/data/ps-champ-ja/moves.json:934` /
  `.internal/progress/move.md` によればChampionsでは`8`が正しい値のため修正する
  （`オーバーヒート` も同じ本家PP5→Champions PP8のパターンで既に修正済み）。

`power=130`・`accuracy=90` はすでにChampions値（第六世代以降の値）と一致しており問題ない。
`flags` を持たない点も一次情報（直接攻撃×、追加効果なし＝ちからずく非対象）と一致しており
問題ない。

  `MoveHandler` のデフォルト priority（100）で登録する。
