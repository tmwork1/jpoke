## Priority 根拠

`.internal/spec/turn.md` には `ON_STATUS_HIT` の内部 priority 表は掲載されていない。
既存の実数値操作ハンドラ（`パワートリック_swap_stats`・`スピードスワップ_swap_speed`・
`ガードシェア_equalize_stats`等）はすべて priority 100（`MoveHandler` デフォルト）を使用しており、
パワーシフトも同一パターンで priority 100 を採用する（パワートリックの計画書と同じ根拠）。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

新規関数は追加しない。既存の `パワートリック_swap_stats` をそのまま `パワーシフト` の
`ON_STATUS_HIT` ハンドラとして登録する。

```python
def パワートリック_swap_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """パワートリック: 使用者のこうげきとぼうぎょの実数値を入れ替える。

    ランク変化は行わず、実数値のみを入れ替える。
    インデックス 1 = こうげき、インデックス 2 = ぼうぎょ
    （[HP, 攻撃, 防御, 特攻, 特防, 素早さ]）。
    """
    atk = ctx.attacker
    atk_stat, def_stat = atk.get_raw_stat(1), atk.get_raw_stat(2)
    atk.set_raw_stat(1, def_stat)
    atk.set_raw_stat(2, atk_stat)
    return HandlerReturn(value=value)
```

パワーシフトとパワートリックは効果が完全に同一（ウィキ本文「パワートリックと同じ効果を持つ」）
であるため、専用の `パワーシフト_swap_stats` 関数は作らず既存関数を共有する。
これは `あさのひざし_heal_self` を複数の日照回復技（`あさのひざし`・`こうごうせい`・
`モーンフォレスト`等）で共有する既存パターンと同一の設計である。

### `src/jpoke/data/moves/move_ha.py`

`"パワーシフト"` エントリを新規追加する。ps-champ-jaでカバーされる技のため
`type`/`category`/`pp`/`accuracy`/`priority`/`crit_ratio`/`target` は明示指定不要
（`data/move.py` の `common_setup()` が `data/ps-champ-ja/moves.json` から自動的に埋める。
`target="self"` もps-champ-ja側の `"target": "self"` から自動導出される）。

```python
"パワーシフト": MoveData(
    # パワートリックと完全に同一効果（自分のこうげき・ぼうぎょの実数値を入れ替える）のため
    # ハンドラ関数を共有する。
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.パワートリック_swap_stats,
        ),
    }
),
```

挿入位置: 五十音順「は」行 — `"パワーシェア"` の直後、`"パワージェム"` の直前
（シェア→シフト→ジェムの順。シ→ジの清濁順、シェ→シフの母音行順に基づく）。

## 注意点・エッジケース

- **対象は自分のみ**: `ctx.defender` は参照しない
- **`target="self"` は自動導出**: ps-champ-ja側の `"target": "self"` から
  `_TARGET_MAP["self"] = "self"` として自動的に設定されるため、move_ha.py側での明示指定は不要
  （パワートリックと同一パターン）
- **実数値直接操作の根拠**: `パワートリック_swap_stats` と同じ実装のため、
  `battle.modify_stat()`（ランク変化量加算API）ではなく `get_raw_stat`/`set_raw_stat`
  （実数値そのものの入れ替え）を使用する
- **ランク変化は変わらない**: `mon.rank["atk"]`・`mon.rank["def"]` は変化しない
- **ON_MODIFY_STAT 非発火**: 直接代入のため、ランク変化に反応する特性（かちき・まけんき等）は発動しない
- **再度使用で元に戻る**: パワーシフトを2回使用すると実数値が元の状態に戻る
- **パワートリックとの相互作用**: パワーシフト使用後にパワートリックを使用（またはその逆）しても、
  実数値の入れ替え自体は同一ロジックのため、通算で偶数回になれば元に戻り、奇数回になれば
  入れ替わった状態になる（ウィキ「パワーシフト/パワートリックを使用したポケモンがもう一度
  パワーシフト/パワートリックを使用するとステータスは元に戻る」の通り、技名を問わず
  「入れ替え操作の回数」で決まる。ハンドラ関数を共有しているため実装上も自動的にこの挙動になる）
- **交代による解除**: 場を離れると入れ替えた実数値は元に戻る（`Pokemon.reset_on_switch_out()` が
  `update_stats()` を呼ぶため、実装済み・確認済み）
- **PP について**: `src/jpoke/data/ps-champ-ja/moves.json` 準拠で 12（ウィキ記載の本家値10ではない）
- **習得可能ポケモンが存在しない点**: SVでは本技を習得できるポケモンが存在しないが、
  `Pokemon(move_names=[...])` により任意の技を割り当てられる本シミュレータの設計上、
  実装・テストの妨げにはならない（既存の習得可否チェックを行わない設計と整合）
