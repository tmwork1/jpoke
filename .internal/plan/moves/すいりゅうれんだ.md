# すいりゅうれんだ 実装計画書

## 仕様要約（.internal/spec/moves/すいりゅうれんだ.md 参照）

* タイプ: みず / 分類: 物理 / 威力: 25（1ヒットあたり） / 命中: 100 / PP: 5
* 対象: 相手1体 / 優先度: 0 / 接触あり / パンチ技
* 効果:
  * 3回連続で攻撃する固定回数の連続攻撃技（命中判定は1発目のみ、各ヒット威力は固定25）。
  * 必ず急所に当たる（急所ランク+3 = 確率100%）。急所判定は各ヒットごとに独立して行われる。
  * 追加効果なし。

## 現状の実装状況

`src/jpoke/data/move.py` に既に登録済み。

```python
"すいりゅうれんだ": MoveData(
    type="みず",
    category="physical",
    pp=5,
    power=25,
    accuracy=100,
    critical_rank=3,
    multi_hit={
        "min": 3,
        "max": 3,
        "check_hit_each_time": False,
        "power_sequence": (),
    },
    flags={"contact", "punch"},
    handlers={},  # 追加効果なし
),
```

内容は仕様書と一致しており、修正不要。

* `critical_rank=3` → `CRIT_RATES[3] == 1`（`core/move_executor.py`）により確定急所となる。
  この機構は `あんこくきょうだ` 等の既存の確定急所技と共通。
* `multi_hit.min == multi_hit.max == 3` かつ `flags` に `check_hit_each_time` を含まないため、
  `_resolve_hit_count` は常に3を返し、命中判定は1発目のみ行われる（`トリプルダイブ` と同一パターン）。
* `power_sequence` が空タプルのため、`_resolve_hit_power` は各ヒットとも `move.power`（25）を返す。

## 急所無効化との関係

確定急所ではあるが、防御側の特性カブトアーマー・シェルアーマーを持つポケモンに対しては
急所に当たらない（急所無効化特性が確定急所効果より優先される）。この無効化は
カブトアーマー・シェルアーマー側のハンドラで処理されるため、本技側の実装への影響はない。

## 注意点・エッジケース（review-test 向け）

1. **確定急所は各ヒットごとに独立判定**
   - `.internal/spec/moves/_multi_hit.md` の「7.1 急所判定」の通り、各ヒットごとに急所判定が行われる。
     `critical_rank=3` により全ヒットが確定急所となる。
2. **命中判定は1発目のみ**
   - `check_hit_each_time` フラグを持たないため、1発目が外れた場合は残りのヒットも発生しない。
3. **パンチ技フラグ**
   - `flags={"punch"}` により、てつのこぶし・パンチグローブの対象となる（フレームワーク側で汎用処理）。
4. **テスト方針**
   - `_resolve_hit_count` が常に3を返すことを確認するテスト。
   - `run_move` 実行後、`battle.move_executor.critical` が `True` であることを確認するテスト
     （`あんこくきょうだ` の確定急所テストと同一パターン）。
   - ダメージが発生することを確認する基本テスト。
