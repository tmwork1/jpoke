# 実装計画書: ボルテッカー

参照: .internal/spec/moves/ボルテッカー.md

## 概要

物理技。相手に与えたダメージの1/3を自分も受ける反動技。追加効果として10%の確率で相手をまひ状態にする。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---------|----------|--------------|------|
| ON_HIT | 100（`MoveHandler` デフォルト） | attacker:self | 与ダメの1/3を反動として攻撃者に与える |
| ON_DAMAGE_HIT | 100（`MoveHandler` デフォルト） | attacker:self | 10%の確率で相手をまひ状態にする |

Priority 根拠: `.internal/spec/turn.md` の `Event.ON_HIT` に反動ダメージ専用の記載行はないが、
同系統の反動技（とっしん・ワイルドボルト・じごくぐるま・アフロブレイク）と同じく
`MoveHandler` のデフォルト priority=100 を使用する（既存実装との整合性を優先）。
`ON_DAMAGE_HIT` の追加効果も、他の追加効果技（かみなりのキバ等）と同様デフォルト priority=100 を使用する。

## handlers/move_attack.py

共通ヘルパー `_recoil(battle, ctx, value, ratio)` と `apply_ailment_to_defender(...)` を利用する（既存実装）。

```python
def ボルテッカー_apply_paralysis_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="まひ", chance=0.1)


def ボルテッカー_recoil(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return _recoil(battle, ctx, value, 1/3)
```

## data/move.py

```python
    "ボルテッカー": MoveData(
        type="でんき",
        category="physical",
        pp=16,
        power=120,
        accuracy=100,
        flags={"contact", "recoil", "secondary_effect"},
        handlers={
            Event.ON_HIT: h.MoveHandler(
                ha.ボルテッカー_recoil,
            ),
            Event.ON_DAMAGE_HIT: h.MoveHandler(
                ha.ボルテッカー_apply_paralysis_to_defender,
            )
        }
    ),
```

## リーサル計算

対象外（`.internal/progress/move.md` の「リーサル実装」列は n/a）。
反動技は攻撃者自身のHPを減らすのみで、防御側の HP 分布 (`StateDist`) には影響しないため
リーサル計算ハンドラの実装対象にはならない。まひの追加効果も確率依存かつ攻撃側の
HP分布に影響しないため対象外。

## 注意点・エッジケース

* `reason="recoil"` を指定することで、いしあたま特性・マジックガード特性により反動ダメージが無効化される
  （特性側の実装に依存する汎用処理のため、本技側での分岐は不要）。
* みがわりに命中した場合も `ON_HIT` は発動するため反動ダメージは発生する。
  `_recoil` は `value`（実際の被ダメージ、みがわり有効時は0）が0の場合
  `ctx.substitute_damage`（みがわりへの与ダメージ）にフォールバックして反動量を算出する
  （共通ヘルパー `_drain_hp` と同じパターン）。
* みがわりに完全に阻まれた場合（`actual_damage <= 0`）、`core/move_executor.py` の実装上
  `Event.ON_DAMAGE_HIT` は発火しないため、まひの追加効果は発動しない（`Event.ON_HIT` は発火するため
  反動は発生する点に注意）。
* `secondary_effect` フラグにより、ちからずく特性使用時はまひの追加効果が発動しなくなる
  （`resolve_secondary_chance` 経由で処理される既存の汎用処理）。
* PP はチャンピオンズのPP統一仕様（8/12/16/20の4段階）により16。Gen9本家のPP15とは異なる
  （`src/jpoke/data/ps-champ-ja/moves.json` を正とする）。
