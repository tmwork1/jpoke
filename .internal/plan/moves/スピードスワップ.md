# 計画書: スピードスワップ

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 12（`src/jpoke/data/ps-champ-ja/moves.json` 準拠） / 命中: - （必中） / 対象: 相手
- 使用者と相手の「すばやさ（S）」の**実数値**を入れ替える
- ランク変化（`rank["spe"]`）は変化しない。実数値のみを入れ替える
- まもる: 防がれる / マジックコート: 跳ね返されない（`flags={"unreflectable"}` が必要） /
  みがわり: 貫通する（`flags={"bypass_substitute"}` が必要） / 命中判定: なし（`accuracy=None`）

## 現状の実装状況（レビュー後）

| 項目 | 状態 |
|------|------|
| `data/move.py` エントリ | 実装済み（type・category・pp=12・accuracy=None・flags・handlers 登録済み） |
| `handlers/move_status.py` 関数 | 実装済み（`スピードスワップ_swap_speed`） |

### レビューで発見・修正したバグ

1. **PP誤り**: `pp=10`（Gen9本家基準）になっていたが、`src/jpoke/data/ps-champ-ja/moves.json` では
   `12` が正。ガードシェア・ガードスワップ等と同じ移行漏れバグだったため `pp=12` に修正した。
2. **命中判定誤り**: `accuracy=100` になっていたが、これだと `ON_MODIFY_ACCURACY` を経由して
   回避率ランク補正の影響を受けてしまい、必中（命中率「-」）にならない。`accuracy=None` に修正した。
3. **`flags` 未設定**: `unreflectable`・`bypass_substitute` が未設定だったため、デフォルトの
   まま（マジックコートで跳ね返される・みがわりで防がれる）になっていた。一次情報の
   「マジックコート: ×」「相手のみがわり状態を無視する」に反するため、
   `flags={"unreflectable", "bypass_substitute"}` を追加した。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 | 実装場所 |
|----------|----------|--------------|------|----------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | すばやさ実数値を両者間で入れ替える | `handlers/move_status.py` |

`ON_STATUS_HIT` は攻撃フェーズ（`AttackContext`）で発火するイベントであり、コンテキストは
`attacker` / `defender` ロールを持つ。ハンドラのシグネチャは `ctx: AttackContext` を用いる
（`ガードシェア_equalize_stats` 等、他の swap/share 系ハンドラと同一）。

## Priority 根拠

`.internal/spec/turn.md` には `ON_STATUS_HIT` の内部 priority 表は掲載されていない。既存の変化技ハンドラ（`ガードシェア_equalize_stats`・`ガードスワップ_swap_ranks`・`すりかえ_swap_items` 等）はすべて priority 100（`MoveHandler` デフォルト）を使用している。スピードスワップも同一パターンで priority 100 を採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: 五十音順「す」行 — `すりかえ_swap_items` の直前に挿入する（す→すり の順）。

実数値の入れ替えは `_stats_manager.stats[5]`（インデックス5 = すばやさ）を直接操作する。ガードシェアが `stats[2]`, `stats[4]`（ぼうぎょ・とくぼう）を操作する実績パターンを踏襲する。`battle.modify_stat()` は実数値の書き換えAPIではなくランク変化量の加算APIであるため使用しない。

```python
def スピードスワップ_swap_speed(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """スピードスワップ: 使用者と相手のすばやさの実数値を入れ替える。

    ランク変化は行わず、実数値のみを入れ替える。
    インデックス 5 = すばやさ（[HP, 攻撃, 防御, 特攻, 特防, 素早さ]）。
    """
    atk_stats = ctx.attacker._stats_manager.stats
    def_stats = ctx.defender._stats_manager.stats
    atk_stats[5], def_stats[5] = def_stats[5], atk_stats[5]
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

既存の `"スピードスワップ"` エントリに `handlers` を追加する。PP は `src/jpoke/data/ps-champ-ja/moves.json` 準拠で 12。
必中技のため `accuracy=None`。マジックコートで跳ね返されず、みがわりを貫通するため
`flags={"unreflectable", "bypass_substitute"}` を設定する。

```python
"スピードスワップ": MoveData(
    type="エスパー",
    category="status",
    pp=12,
    accuracy=None,  # 必中
    # マジックコートで跳ね返されず、みがわりを貫通する
    flags={"unreflectable", "bypass_substitute"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.スピードスワップ_swap_speed,
        ),
    }
),
```

## 注意点・エッジケース

- **実数値直接操作の根拠**: ガードシェア（`_stats_manager.stats[2]`, `[4]`）と同じパターンで実数値を直接書き換える。`battle.modify_stat()` はランク変化量加算APIのため「入れ替え」に使用不可
- **ランク変化は変わらない**: `mon.rank["spe"]` は変化しない。よっておいかぜやすりかえ等によるランク補正は入れ替え後も各ポケモンの rank["spe"] に従う
- **ON_MODIFY_STAT 非発火**: 直接代入のため、ランク変化に反応する特性（かちき・まけんき等）は発動しない。これはゲーム仕様上も正しい（スピードスワップは「ランク変化」ではなく「実数値交換」の技）
- **トリックルーム中の行動順への影響**: 入れ替え後のすばやさ実数値はその後のターンから行動順計算に反映される。本プロジェクトはターン開始時に行動順を確定させる実装のため、使用したターン自体の行動順には影響しない（第七世代相当。第八世代以降・Championsの「使用ターンから影響する」仕様との差異は行動順決定タイミングの実装アーキテクチャ上の制約であり、既存のガードスワップ等と同様に許容する）
- **おいかぜ・ルームサービス等との相互作用**: これらはすばやさのランク変化（rank["spe"]）やフィールド状態で速度を修正するため、実数値スワップ後もそれぞれの補正が独立して適用される
- **ワンダールーム中**: ワンダールームはぼうぎょとくぼうの参照を入れ替えるが、すばやさには影響しない。スピードスワップと干渉なし
- **0値・極端な実数値**: 双方のすばやさが同値の場合は入れ替えても変化なし。`ON_TRY_MOVE_2` の失敗条件（ランク上限・下限）はランク変化技に対する判定であり、実数値スワップ技のスピードスワップには適用されない
- **ガードシェア・パワーシェアとの類似**: シェア系（平均化）ではなく「入れ替え」であることに注意。コードパターンはガードシェア（stats[2], stats[4]）に近いが、平均化ではなく swap である
- **交代時リセット**: `Pokemon.reset_on_switch_out()` が `update_stats()` を呼んで種族値ベースの実数値に再計算するため、スピードスワップによる `stats[5]` の書き換えは交代で自動的にリセットされる。専用の実装は不要（ガードシェア・パワートリック等と共通の既存修正でカバーされる）
- **みがわり貫通・マジックコート非対応**: 一次情報の判定節に「みがわり: ○（無視する）」「マジックコート: ×」と明記されている。`MoveData.flags` に `"bypass_substitute"`・`"unreflectable"` を設定することで実現する
- **PP**: `src/jpoke/data/ps-champ-ja/moves.json` の値（12）を正とする。Gen9本家基準の10は移行漏れのバグだったため修正する
