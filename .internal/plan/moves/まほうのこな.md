# 実装計画書: まほうのこな

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 20 / 命中: 100 / 対象: 1 体選択（ブリムオン専用技）
- 相手のタイプを「エスパー」単タイプに変える（タイプを追加するのではなく、完全に置き換える）
- 以下の場合は失敗する
  - 相手がすでにエスパー単タイプである
  - 相手がテラスタルしている
  - 相手がアルセウス・シルヴァディである（持ち物の有無・特性の状態に関わらず無効。種族名で判定する）
- 粉技（`flags={"powder"}`）。くさタイプ・特性ぼうじん・道具ぼうじんゴーグルにはコア共通ルールで無効化される

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_BEFORE_APPLY_MOVE` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手がすでにエスパー単タイプ／テラスタル中／アルセウス・シルヴァディなら失敗 |
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手に「まほうのこな」volatile を付与 |
| `ON_VOLATILE_START` | 100（デフォルト） | `source:self` | `volatile_override_type` を `"エスパー"` に設定し `added_types` をクリア |
| `ON_VOLATILE_END` | 100（デフォルト） | `source:self` | `volatile_override_type` を `None` に戻す |

## Priority 根拠

- `ON_BEFORE_APPLY_MOVE` priority 100: `.internal/spec/turn.md` の `Event.ON_APPLY_MOVE`（実装上は `ON_BEFORE_APPLY_MOVE`）テーブルに
  「みずびたし等: すでに該当タイプ」が priority 100 として明記されている。まほうのこなも同じ「タイプ変更技の失敗判定」カテゴリに属するため同じ priority を採用する
  （テラスタル・アルセウス／シルヴァディの追加失敗条件も同一関数内で判定するため同じ priority でよい）
- `ON_STATUS_HIT` priority 100（デフォルト）: みずびたし・もりののろい・ハロウィンと同じパターン
- `ON_VOLATILE_START` / `ON_VOLATILE_END` priority 100（デフォルト）: みずびたし volatile と同じパターン

## タイプ上書きの実装方針

みずびたしと全く同じ設計を採用する（`.internal/plan/moves/みずびたし.md` 末尾に「まほうのこなも同じパターンで実装可能」と明記済み）。

1. 揮発性状態「まほうのこな」を新設し、`ON_VOLATILE_START` で `volatile_override_type = "エスパー"` に設定、`added_types` をクリアする
2. `ON_VOLATILE_END` で `volatile_override_type = None` に戻す
3. `Pokemon.types` は既に `volatile_override_type` を参照する実装になっているため、`model/pokemon.py` の変更は不要

## 実装箇所

### 1. `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「ま」行、`みかづきのまい` 等より前、`まもる` 系の前後の適切な位置
（既存の `まもる_apply` 等の直前、`ホバー`／`ほろびのうた` の後）。

```python
def まほうのこな_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """まほうのこなの効果: 相手にまほうのこな状態を付与してエスパー単タイプに変える。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="まほうのこな")


def まほうのこな_can_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """まほうのこなの使用条件チェック: 相手がすでにエスパー単タイプ、テラスタル中、
    またはアルセウス・シルヴァディの場合は失敗する。"""
    defender = ctx.defender
    if (
        defender.types == ["エスパー"]
        or defender.terastallized
        or defender.name in ("アルセウス", "シルヴァディ")
    ):
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="まほうのこな")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### 3. `src/jpoke/handlers/volatile.py`

五十音順の挿入位置: `ハロウィン_*` の後、`みがわり_*` の前の「ま」行。

```python
def まほうのこな_clear_type(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """まほうのこな解除時: volatile_override_type を None に戻す。"""
    ctx.source.volatile_override_type = None
    return HandlerReturn(value=value)


def まほうのこな_set_type(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """まほうのこな付与時: volatile_override_type をエスパーに設定し added_types をクリアする。"""
    ctx.source.volatile_override_type = "エスパー"
    ctx.source.added_types.clear()
    return HandlerReturn(value=value)
```

## 粉技無効化

`flags={"powder"}` を付与するだけで、`core/move_executor.py` の
`_check_grass_type_powder_immunity`（ON_TRY_MOVE_2 相当・コア共通ルール）、
特性ハンドラ `ぼうじん_block_powder`、道具ハンドラ `ぼうじんゴーグル_block_powder_move` が
自動的に適用される。まほうのこな側で個別実装する必要はない。

もりののろいでくさタイプが追加された相手にまほうのこなを使うと失敗するという wiki の記述は、
`Pokemon.types`（`added_types` を考慮する）を用いた `has_type("くさ")` 判定で自然に成立するため、
まほうのこな固有の特別対応は不要。

## 注意点・エッジケース

- アルセウス・シルヴァディの無効化は特性（マルチタイプ／ARシステム）ではなく **種族名** で判定する。
  実際のゲーム仕様では持ち物なしでも無効になるため、特性の `ability_override_type`（持ち物依存）に頼ると
  無道具時に判定が漏れる。`mon.name` は道具の有無に関係なくフォーム名を返さないため
  （アルセウス・シルヴァディはプレート／メモリで別 pokedex エントリに分かれておらず `"アルセウス"` `"シルヴァディ"` のまま）、
  種族名判定で正しく扱える
- テラスタル中の相手は `ON_BEFORE_APPLY_MOVE` の時点で明示的に失敗させる（みずびたしのように volatile だけ付与してテラスタルタイプに隠れさせる方式ではなく、wiki に「テラスタルしている相手に対しては失敗する」と明記されているため）
- `ON_VOLATILE_START` で `added_types.clear()` を行うのは、ハロウィン・もりののろい状態が付与されている相手にまほうのこなを使った場合、追加タイプが残らないようにするため
- 交代すると `reset_on_switch_out` で `volatile_override_type = None` にリセットされるため、エスパータイプの効果は消える（状態異常ではなく揮発性状態のため）
- みがわり状態・マジックコート・まもるは既存の共通ルール（`みがわり_immune` 等）で処理されるため個別対応不要
