## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_HIT` セクションより：

```
| 100 | ダイマックスわざの効果 |
| 100 | ニードルガード等の効果 |
```

追加効果（揮発性状態付与）は同表に「Priority 100: ダイマックスわざの効果」と同列。
既実装の `はやてがえし_apply`（ひるみ100%）および `かみなりのキバ_apply_volatile_to_defender`（ひるみ10%）が
いずれも `MoveHandler` デフォルト（priority=100）で登録されているため、同値を採用する。

## 注意点・エッジケース（review-test 向け）

### ひるみの発動タイミング

ひるみは `Event.ON_CHECK_ACTION` で消費される（`.internal/spec/turn.md` Priority 40「ひるみで動けない」）。
ON_HIT での付与後、相手が次のターンに行動しようとする際に判定される。
すでに行動済みのポケモンにひるみが付与されてもそのターンは効果がない。

### こおりのキバの排他制御

仕様: 「10%でこおり状態にするか、10%でひるみ状態にする（どちらか一方）」

こおり系計画書で登録した `こおりのキバ_apply_ailment_to_defender`（こおりのみ）は、
本計画書の `こおりのキバ_apply_flinch_or_freeze` 複合ハンドラに**置き換える**。
複合ハンドラは 1 回の確率判定（10%）を行い、発動した場合はこおりかひるみをランダム選択する。
したがって、こおり付与とひるみ付与の両方が同時に発動することはない。

### ほのおのキバの排他制御

仕様: 「10%でやけど状態にするか、10%でひるみ状態にする（どちらか一方）」

やけど系計画書で `ほのおのキバ_apply_ailment_to_defender`（やけど10%）が実装されている場合、
本計画書の `ほのおのキバ_apply_flinch_or_burn` 複合ハンドラに置き換える。
実装順次第では、やけど系計画書が先行実装されている可能性がある。

### ねこだましの特殊性

ねこだましは priority=3 の先制技で、かつ「すでに行動をしたポケモンにしか使えない」制約がある
（`.internal/spec/turn.md` ON_TRY_MOVE_1 priority=30「ねこだまし等: すでに行動をした」）。
ひるみ付与確率は 100%（chance 省略）。
ひるみ付与自体のハンドラは単純な `apply_volatile_to_defender` で問題ない。

### いびきのねむり条件

いびきは「ねむり状態でないと使えない」制約がある（ON_TRY_MOVE_1 priority=30「いびき・ねごと: ねむり状態でない」）。
ひるみ付与ハンドラ側でこの条件を重複チェックする必要はない（発動前に manager で処理済み）。
