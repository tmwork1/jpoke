# こうそくスピン 実装計画書

## 仕様要約

- `.internal/spec/moves/こうそくスピン.md` 参照
- ノーマル物理・威力50・命中100・PP40・対象1体（相手）
- 命中後、以下を発動する
  1. 使用者自身の **バインド** 状態を解除する
  2. 使用者自身の **やどりぎのタネ** 状態を解除する
  3. 使用者のサイドの **まきびし・どくびし・ステルスロック・ねばねばネット** を解除する
  4. 100%の確率で使用者の **すばやさ** ランクを1段階上げる
- 上記1〜4は技が命中しなかった場合・まもる等に防がれた場合・タイプ相性や特性で無効化された
  場合は発動しない。一方、みがわりに防がれた場合やダメージが0だった場合（ばけのかわ等）でも発動する。
- 1〜4は「追加効果」ではあるが、りんぷん・おんみつマントの影響は受けない。
  使用者の特性がちからずくの場合のみ、威力1.3倍と引き換えに1〜4のいずれも発動しない。
- 同じ実装（`_clear_spin_effects`）をキラースピンと共有する。

## 実装方針

### ハンドラ構成（`handlers/move_attack.py`）

| イベント | Priority | 役割 |
|----------|----------|------|
| `Event.ON_HIT` | 100 | バインド・やどりぎのタネ解除、設置物解除（`こうそくスピン_clear_field` → `_clear_spin_effects`） |
| `Event.ON_HIT` | 100（デフォルト） | すばやさ+1（`こうそくスピン_boost_attacker_spe`） |

### Priority 根拠

`.internal/spec/turn.md` の Event.ON_HIT セクションには直接掲載がないが、
同ドキュメントの「Event.ON_DAMAGE」セクション（実装上は `Event.ON_DAMAGE_HIT` に相当）に
「100 | こうそくスピン等で場の状態解除」と記載がある。
ただし `Event.ON_DAMAGE_HIT` は実ダメージが0（みがわりに阻まれた場合等）のとき発火しない仕様であり、
本技は「みがわりに防がれても解除できる」「ダメージが0でも解除できる」という要件を持つため、
`かいがらのすず`（`handlers/item.py` 内のコメント参照）と同じ設計判断で、
常に発火する `Event.ON_HIT` を採用し、priority のみ turn.md の値（100）を踏襲する。

### subject_spec

`MoveHandler` のデフォルト `attacker:self`（`AttackContext` の攻撃フロー、`skip_subject_check=True`）。

## 実装箇所

### handlers/move_attack.py（五十音順: こうそくスピンは「こ」行）

```python
def _is_sheer_force_blocked(ctx: AttackContext) -> bool:
    """ちからずく特性による追加効果無効化を判定する。

    りんぷん・おんみつマントの影響を受けない効果（resolve_secondary_chance を
    経由しない効果）でも、ちからずくの追加効果無効化だけは受けるケースで使う。
    """
    return ctx.attacker.ability.name == "ちからずく" and ctx.move.has_flag("secondary_effect")


def _clear_spin_effects(battle: Battle, ctx: AttackContext) -> None:
    """こうそくスピン・キラースピン共通: バインド・やどりぎのタネを解除し、設置物を除去する。

    りんぷん・おんみつマントを持つ側がいても解除は発動するため
    resolve_secondary_chance は経由しないが、使用者がちからずくの場合は
    （威力1.3倍化と引き換えに）解除自体が発動しない。
    """
    if _is_sheer_force_blocked(ctx):
        return
    battle.volatile_manager.remove(ctx.attacker, "バインド")
    battle.volatile_manager.remove(ctx.attacker, "やどりぎのタネ")
    side = battle.get_side(ctx.attacker)
    for hazard in ("まきびし", "どくびし", "ステルスロック", "ねばねばネット"):
        side.deactivate(hazard)


def こうそくスピン_boost_attacker_spe(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """こうそくスピン: 使用者のすばやさを1段階上げる。

    りんぷん・おんみつマントを持つ側がいても発動するため resolve_secondary_chance
    は経由せず battle.modify_stats を直接呼ぶ。使用者がちからずくの場合のみ発動しない。
    """
    if _is_sheer_force_blocked(ctx):
        return HandlerReturn(value=value)
    battle.modify_stats(ctx.attacker, {"spe": 1}, source=ctx.attacker)
    return HandlerReturn(value=value)


def こうそくスピン_clear_field(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """こうそくスピン: バインド・やどりぎのタネを解除し、自分のサイドの設置物を除去する。"""
    _clear_spin_effects(battle, ctx)
    return HandlerReturn(value=value)
```

### data/move.py（既存の「こうそくスピン」MoveData エントリ、変更なし）

```python
"こうそくスピン": MoveData(

    category="physical",

```

## 注意点・エッジケース（review-test 向け）

1. **やどりぎのタネ解除**: 旧実装では未対応だった。ウィキの Champions 説明文
   「自分にかかっているバインド やどりぎのタネ状態と…解除する」を根拠に追加する。
2. **りんぷん・おんみつマント無効**: `battle.volatile_manager.remove` / `side.deactivate` /
   `battle.modify_stats` を直接呼び、`resolve_secondary_chance` を経由しないことで実現する
   （`うちおとす_apply_grounded` 等と同じ設計パターン）。
3. **ちからずくブロック**: 第9世代では、ちからずく使用時は威力1.3倍になる代わりに
   解除・すばやさ上昇のいずれも発動しない。`ctx.attacker.ability.name == "ちからずく"` を
   直接判定する（`だっしゅつボタン_reserve_switch` 等と同じパターンで、
   `resolve_secondary_chance` を経由しないため通常の `ちからずく_disable_secondary_effect`
   ハンドラは通らない）。
4. **みがわり・ばけのかわ**: `Event.ON_HIT` は実ダメージが0でも必ず発火するため、
   追加のガードは不要。
5. **命中しなかった場合**: `_execute_hit` に到達しないため `Event.ON_HIT` 自体が
   発火せず、自然に解除されない。
6. **キラースピンとの共有**: `_clear_spin_effects` を両技で共有するため、
   本計画の変更はキラースピンの「バインド・やどりぎのタネ解除」「設置物解除」にも
   同様に適用される（キラースピンの毒付与は対象への効果のため対象外、従来通り
   `resolve_secondary_chance` を経由する）。
