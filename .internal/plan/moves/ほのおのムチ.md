# 実装計画書: ほのおのムチ（再レビュー）

## 仕様要約

`.internal/spec/moves/ほのおのムチ.md` 参照。

* タイプ: ほのお / 分類: 物理 / 威力: 90（Champions。SVまでは80） / 命中率: 100 / PP: 16（Champions。SVまでは15） / 対象: 相手1体
* 直接攻撃。追加効果として、100%の確率で相手の『ぼうぎょ』ランクを1段階下げる。

## 既存実装の確認結果と問題点

`src/jpoke/handlers/move_attack.py` の `ほのおのムチ_lower_defender_def` が
`Event.ON_DAMAGE_HIT` で `modify_defender_stats(battle, ctx, value, stats={"def": -1})`
（`chance` 省略でデフォルト100%）を実行しており、同種のぼうぎょダウン追加効果を持つ
`Gのちから_lower_defender_def` と同じ実装パターンになっている。イベント・優先度
（デフォルト100・`.internal/spec/turn.md` の「100 スケイルショットによる能力変化」相当）も
問題ない。

一方、`src/jpoke/data/moves/move_ha.py` の `MoveData` エントリに以下の誤りがあった。

* `power=80`（SVまでの値）になっていたが、`src/jpoke/data/ps-champ-ja/moves.json` /
  `.internal/progress/move.md` によれば Champions では `90` が正しい値のため修正する。
* `pp=15`（SVまでの値）になっていたが、`src/jpoke/data/ps-champ-ja/moves.json` /
  `.internal/progress/move.md` によれば Champions では `16` が正しい値のため修正する。

`flags={"contact", "secondary_effect"}` は一次情報（直接攻撃○、ちからずく対象技）と
一致しており問題ない。`lethal_handlers={LethalEvent.ON_HIT: LethalHandler(l.ほのおのムチ_lower_def)}`
も `Gのちから_lower_def` と同一パターンで問題ない。

## ハンドラ構成（変更なし）

| ほのおのムチ | `Event.ON_DAMAGE_HIT` | デフォルト（100） | `attacker:self`（デフォルト、`modify_defender_stats` 内部で `ctx.defender` を操作） | 命中後、100%の確率で相手のぼうぎょランクを1段階下げる |

* `.internal/spec/turn.md` の `## Event.ON_DAMAGE`（実装上の `Event.ON_DAMAGE_HIT` に相当）の
  「100 スケイルショットによる能力変化」に準じ、ダメージ適用後にランク変化を発動させる。

## リーサル計算

`.internal/progress/move.md` の該当行は「リーサル実装: x」「リーサルテスト: x」。
`handlers/lethal.py` の `ほのおのムチ_lower_def` は `Gのちから_lower_def` と同一パターンで
`ctx.move_secondary` が真のときのみ `ctx.defender.rank["def"]` を1段階下げており、
仕様・実装と整合している。修正不要。

## テスト

`tests/moves_attack/test_move_ha.py` の `test_ほのおのムチ_ぼうぎょ1段階低下が発動する` で
追加効果の発動を確認済み。`tests/test_lethal.py` の
`test_ほのおのムチ_ぼうぎょダウン_secondary有り` / `_secondary無し` で
リーサル計算のランク反映を確認済み。今回の修正（威力・PP）はこれらのテストの
アサーションに直接影響しないため、既存テストのままで問題ない。
