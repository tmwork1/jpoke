# きゅうけつ 実装計画書

参照: .internal/spec/moves/きゅうけつ.md

## 概要

物理・むしタイプのドレイン技。相手に与えたダメージの50%だけ自分のHPを回復する。
既存のドレイン技（ギガドレイン・すいとる・ドレインパンチ等）と同一の共通処理
`_drain_hp`（`handlers/move_attack.py`）を再利用する。

## MoveData（data/move.py）

```python
"きゅうけつ": MoveData(
    type="むし",
    category="physical",
    pp=10,
    power=80,
    accuracy=100,
    flags={"contact", "heal"},
    handlers={
        Event.ON_HIT: h.MoveHandler(ha.きゅうけつ_drain, priority=20)
    }
),
```

* `flags` に `"heal"` を含める（HPを吸収する技であることを示す）。
* `subject_spec` は `MoveHandler` のデフォルト `"attacker:self"` を使用（明示指定不要）。

## ハンドラ（handlers/move_attack.py）

```python
def きゅうけつ_drain(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """きゅうけつの回復量を計算する。"""
    _drain_hp(battle, ctx, value, heal_ratio=0.5)
    return HandlerReturn(value=value)
```

* 共通関数 `_drain_hp(battle, ctx, damage, heal_ratio)` を呼び出す。
  * `damage`（与えたダメージ）が0の場合（みがわりに命中した場合）は `ctx.substitute_damage` を使う。
  * `heal_amount = int(damage * heal_ratio)` で算出し、`Event.ON_CALC_DRAIN` を発火して
    `おおきなねっこ`（1.3倍）・`ヘドロえき`（回復→反動ダメージへの変換）を適用する。
  * `battle.modify_hp(ctx.attacker, v=heal_amount, reason="drain")` で攻撃者のHPを回復する。

## イベント・priority

* `Event.ON_HIT`（ダメージ発生後の処理）: priority=20
  * `.internal/spec/turn.md` の `Event.ON_HIT` 表「20 | HP吸収技による回復」に対応。
  * 既存の全ドレイン技ハンドラ（ウッドホーン・ギガドレイン・すいとる・ドレインパンチ等）と同じ
    priority=20 で登録し、処理順の一貫性を保つ。
