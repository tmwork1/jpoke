# ウッドハンマー 実装計画書

参照: .internal/spec/moves/ウッドハンマー.md

## 概要

物理・くさタイプの反動技。相手に与えたダメージの1/3を自分も受ける。
共通ヘルパー `_recoil`（handlers/move_attack.py）を再利用して実装する。

既存のグループ計画書 `.internal/plan/moves/反動技_033.md`（反動量1/3の技をまとめた計画書。
ウェーブタックル・すてみタックル・フレアドライブ・ブレイブバードと共通）で既に
ウッドハンマーの登録内容が計画・実装済みとなっている。
本ファイルは一次情報（Wiki）・仕様書との照合結果として、既存実装に誤り・欠落が
ないことを個別に確認した記録を兼ねる。

## MoveData（data/moves/move_a.py）

既存定義（PPを15→16に修正。詳細は「実装状況の確認結果」参照）:

```python
"ウッドハンマー": MoveData(
    type="くさ",
    category="physical",
    pp=16,
    power=120,
    accuracy=100,
    flags={"contact", "recoil"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.ウッドハンマー_recoil,
        )
    }
),
```

* `flags` に `"contact"`（直接攻撃）・`"recoil"`（反動技。すてみ特性の1.2倍威力補正、
  いしあたま特性の反動無効化判定に使用）を含める。
* `subject_spec` は `MoveHandler` のデフォルト `"attacker:self"` を使用。
* `priority` は `MoveHandler` のデフォルト値100を使用する。

## ハンドラ（handlers/move_attack.py）

既存実装（変更なし、確認済み）:

```python
def ウッドハンマー_recoil(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return _recoil(battle, ctx, value, 1/3)
```

* 共通ヘルパー `_recoil(battle, ctx, value, ratio)` を再利用する。
  `recoil = max(1, round_half_down(damage * ratio))` で反動量を計算し、
  `battle.modify_hp(ctx.attacker, v=-recoil, reason="recoil", source=ctx.attacker)`
  で自分のHPを減らす。
* `damage = value or ctx.substitute_damage` により、みがわりに命中した場合は
  みがわりへの与ダメージを基準に反動量を算出する。
* `reason="recoil"` を指定することで、いしあたま特性
  （`ability.py` の `いしあたま_ignore_recoil`、`ctx.hp_change_reason == "recoil"`
  を検査して無効化）・マジックガード特性
  （`マジックガード_ignore_damage`、`hp_change_reason` が除外リストに含まれないため無効化）
  が自動的に反動を無効化する。
* すてみ特性（`すてみ_boost_power`、`ctx.move.has_flag("recoil")` を検査）により、
  `flags={"recoil"}` を持つ本技は自動的に威力1.2倍の恩恵を受ける。

## イベント・priority

* `Event.ON_HIT`（ダメージ発生後の処理）: priority=100（`MoveHandler` のデフォルト）
  * `.internal/spec/turn.md` の `Event.ON_HIT` 表「100 | 反動技による反動ダメージ」に対応。
  * 既存の同系統反動技（ウェーブタックル・すてみタックル・フレアドライブ・
    ブレイブバード・アフロブレイク・じごくぐるま・とっしん・はめつのひかり・
    もろはのずつき・ボルテッカー・ワイルドボルト）と同じ priority=100 で
    登録されており、処理順の一貫性が保たれている。

## 実装状況の確認結果

一次情報（`.internal/wiki/moves/ウッドハンマー.html`）・仕様書
（`.internal/spec/moves/ウッドハンマー.md`）と照合した結果、以下の点を確認した。

* タイプ（くさ）・分類（物理）・威力（120）・命中率（100）が一致。
* 反動比率1/3、共通ヘルパー `_recoil` を使用した実装が仕様と一致。
* `flags={"contact", "recoil"}` が接触技・反動技の両方の性質を正しく表現している。
* がんじょう特性は `ON_BEFORE_DAMAGE_APPLY`（メイン攻撃ダメージの適用時）のみに
  作用し、反動ダメージの `modify_hp` 呼び出しには影響しないため、本技の反動による
  瀕死をがんじょうが防がない仕様と実装が一致している。
* **PPの誤り**: `data/moves/move_a.py` の既存定義は `pp=15`（本家Gen9基準）だったが、
  `src/jpoke/data/ps-champ-ja/moves.json` では `16` と定義されている
  （`.internal/champions/changes_from_sv.md` の「全技PPを8/12/16/20に統一」対象技のため）。
  championsへの移行漏れのバグと判断し、`pp=16` に修正した。
