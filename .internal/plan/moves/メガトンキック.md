# 計画書: メガトンキック

## 仕様要約

- ノーマルタイプ / 物理 / 威力120 / 命中75 / PP8（champions基準） / 対象: 1体選択 / 優先度0 / 接触あり
- 追加効果なし（通常攻撃のみ）

## ハンドラ構成

追加効果を持たないため、専用ハンドラは不要。`handlers={}` のみで基本ダメージ計算に委ねる。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、priority の検討対象イベントは存在しない。

## 実装箇所

### data/moves/move_ma.py への更新

```python
    "メガトンキック": MoveData(
        type="ノーマル",
        category="physical",
        pp=8,
        power=120,
        accuracy=75,
        flags={"contact"},
        handlers={},  # 追加効果なし
    ),
```

- `pp` は Gen9本家では5だが、`src/jpoke/data/ps-champ-ja/moves.json` により本プロジェクトでは8（PPは8/12/16/20の4段階に統一されているため）。
- `flags={"contact"}` により直接攻撃（接触）を表現 → 仕様と一致。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `critical_rank` 省略時のデフォルトは `0`（通常の急所判定）→ 仕様と一致。

## 注意点・エッジケース

- 追加効果がないため、まもる・おうじゃのしるし・オウムがえし等の一般的な技判定は
  フレームワーク側の共通処理（`handlers/move_attack.py` の基本ダメージ計算経路）で
  自動的に処理され、本技固有のハンドラ実装は不要。
