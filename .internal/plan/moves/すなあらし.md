# 実装計画書: すなあらし

## 仕様

- タイプ: いわ / カテゴリ: 変化 / PP: 8（champions/move_list.txt準拠）
- 命中率: なし（必中） / 対象: field
- 効果: 5 ターン間、天候を「すなあらし」にする

強天候（おおひでり・おおあめ・らんきりゅう）発動中は `weather_manager.apply` 内で上書き拒否され失敗。
すでに「すなあらし」の場合も `apply` が `False` を返し失敗。

## イベント・Priority

| イベント | Priority | subject_spec | 役割 |
|----------|----------|-------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self` | すなあらし天候を5ターン設定 |

**根拠**: `ON_STATUS_HIT` は `turn.md` に独立エントリなし。`にほんばれ_activate_weather`・`あまごい_activate_weather` が共に priority=100 デフォルトのため同パターンを採用。

## 実装

### handlers/move_status.py

```python
def すなあらし_activate_weather(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return HandlerReturn(value=battle.weather_manager.apply("すなあらし", 5, source=ctx.attacker))
```

### data/moves/move_sa.py

```python
"すなあらし": MoveData(
    type="いわ",
    category="status",
    pp=8,
    target="field",
    flags={"wind"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.すなあらし_activate_weather,
        ),
    }
),
```

## 注意事項

- `weather_manager.apply` の戻り値 bool をそのまま `HandlerReturn(value=...)` に渡す
- さらさらいわによる 8 ターン延長は `ON_MODIFY_DURATION` ハンドラ（item 側）が担当
- すなあらし状態のターン終了ダメージ・いわタイプとくぼう補正は `data/field/weather.py` の
  「すなあらし」エントリ（`すなあらし_turn_end` / `すなあらし_boost_spd`）が担当。本技固有の実装は不要
- かがくへんかガス/かたやぶり無効化チェックは書かない
