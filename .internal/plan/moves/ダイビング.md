# 計画書: ダイビング

作成日: 2026-06-26

---

## 仕様要約

- 2ターン技（物理・みず・威力80・命中100・PP12・直接攻撃）
  - PPはチャンピオンズ仕様（`src/jpoke/data/ps-champ-ja/moves.json` 基準）。Gen9本家のPP10ではない点に注意
- 1ターン目: 水中に潜り「ダイビング」揮発状態になる（チャージターン）
  - チャージ中は行動が強制（FORCED）固定、PPは消費しない
  - 交代できない（ゴーストタイプを除く）
- 2ターン目: 揮発状態を解除して攻撃する
- 潜伏中は「なみのり」「うずしお」以外の技を受けない
- 「なみのり」「うずしお」はダイビング状態の相手に命中し、威力が2倍になる

## Priority 根拠

### ON_MOVE_CHARGE (priority=100)

`.internal/spec/turn.md` の `Event.ON_MOVE_CHARGE` テーブルに「隠れる技: 状態変化」が priority=100 として記載されている。

### ON_TRY_MOVE_1 (priority=50)

`.internal/spec/turn.md` の `Event.ON_TRY_MOVE_1` テーブルに「姿を隠していることによる無効化」が priority=100 として記載されている。ただし現実装では priority=50 を使用している。これは「姿を隠す無効化」がパワフルハーブによる溜め中断（priority=70）より前に発生させる必要があるため、`あなをほる` / `そらをとぶ` / `シャドーダイブ` と同一の priority=50 を採用している（同種ハンドラ参照）。

### handlers/volatile.py の関数（実装済み）

```python
def ダイビング_boost_power(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ダイビング状態の相手に対して なみのり・うずしお の威力を2倍にする。"""
    if ctx.move.name in ("なみのり", "うずしお"):
        value *= 2
    return HandlerReturn(value=value)


def ダイビング_can_hit_hidden_target(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """ダイビング状態の回避判定"""
    return can_hit_hidden_target(battle, ctx, value, "ダイビング")


def ダイビング_remove_volatile(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """ダイビング状態の解除"""
    return remove_volatile(battle, ctx, value, volatile="ダイビング")
```

## 注意点・エッジケース（review-test 向け）

1. **チャージ中断後の揮発状態解除**
   - ねむり・こおりなどで行動不能になったターン、揮発状態は解除される（turn_controller または move_executor 側で処理される）
   - ハンドラ側での追加ガードは不要

2. **威力2倍の対象技**
   - 「なみのり」「うずしお」のみ。他のみずタイプ技（バブルこうせんなど）は2倍にならない
   - `HIDDEN_MOVE_ALLOWED_MOVES["ダイビング"] = ["なみのり", "うずしお"]` の定数定義が根拠

3. **命中可能技の回避判定**
   - 一次情報ではロックオン状態・ノーガード特性・どくどく（毒タイプ、第六世代以降）でも
     ダイビング中の相手に技が命中するとされるが、現状の `can_hit_hidden_target`
     （`handlers/volatile.py`）は `HIDDEN_MOVE_ALLOWED_MOVES` の技名一致のみで判定しており、
     これらの救済は未実装（`あなをほる` / `そらをとぶ` と共通の既知の未実装事項。
     詳細は `.internal/spec/volatiles/ダイビング.md` を参照）
   - 本レビューでは共有ハンドラへの影響が大きいため対応を見送り、現状維持とした

4. **ON_TRY_MOVE_1 の priority=50 について**
   - turn.md では priority=100 と記載があるが、パワフルハーブ（priority=70）が溜めをスキップするより前に回避判定を行う必要があるため priority=50 を使用している
   - `あなをほる` / `そらをとぶ` / `シャドーダイブ` と同一設計

5. **パワフルハーブとの相互作用**
   - パワフルハーブ所持時は1ターン目にチャージをスキップして即攻撃する
   - `data/item.py` のパワフルハーブハンドラ（`ON_MOVE_CHARGE` priority=70）が溜め状態を解除し技を続行させる
   - ダイビング側での追加処理は不要

6. **ゴーストタイプの交代禁止除外**
   - `check_trapped_not_ghost` ヘルパーが共通で処理する
   - ゴーストタイプ以外は交代コマンドが ON_CHECK_TRAPPED で禁止される

7. **forced=True の意味**
   - VolatileData の `forced=True` フラグにより、チャージ中のポケモンはそのターン技を強制的に使用する扱いになる
   - PP は ON_MODIFY_COMMAND_OPTIONS で FORCED を返し stop_event=True とすることで消費されない
