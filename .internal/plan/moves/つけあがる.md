# つけあがる 実装計画書

## 仕様要約（.internal/spec/moves/つけあがる.md 参照）

* タイプ: あく / 分類: 物理 / 威力: 20（変動） / 命中: 100 / PP: 10
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 使用者の正のランク変化合計1段階ごとに威力+20（命中率・回避率も対象、HPは対象外）。
  負のランク変化は合計に含めない（下限0）。追加効果なし。

## 現状の実装状況

`src/jpoke/data/move.py` に登録済み。

```python
"つけあがる": MoveData(
    type="あく",
    category="physical",
    pp=10,
    power=20,
    accuracy=100,
    flags={"contact"},
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
            ha.つけあがる_calc_power,
        ),
    }
),
```

`MoveHandler` の `subject_spec` はデフォルト値 `"attacker:self"` を使用（`ON_CALC_POWER_MODIFIER` は
`AttackContext` を渡すイベントであり、使用者自身のランクを参照するため一致している）。

## ハンドラ構成

| event | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_CALC_POWER_MODIFIER` | 100（デフォルト） | `attacker:self` | 使用者の正のランク変化合計から威力倍率を計算する |

### handlers/move_attack.py

```python
def つけあがる_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """つけあがる: 使用者の能力ランク上昇段階の合計1段階ごとに威力が20増加する（基本威力20）。

    基本威力 20 に対して (1 + rank_sum) 倍の modifier を掛ける。
    A/B/C/D/S/命中率/回避率の正ランク合計を使う（HPは対象外）。
    アシストパワーと同様の計算。
    """
    attacker = ctx.attacker
    rank_sum = sum(max(0, v) for k, v in attacker.rank.items() if k != "hp")
    return HandlerReturn(value=apply_fixed_modifier(value, 4096 * (1 + rank_sum)))
```

`アシストパワー_boost_power_by_rank`（`handlers/move_attack.py`）と全く同じ計算式であり、
両者とも使用者の `rank` 辞書（`hp` を除く全キー: `atk`/`def`/`spa`/`spd`/`spe`/`accuracy`/`evasion`）の
正の値のみを合計する。

## Priority 根拠

`ON_CALC_POWER_MODIFIER` は `.internal/spec/turn.md` に独立した行が存在しないイベント（ダメージ計算内部の
威力補正チェーン）。既存の `アシストパワー_boost_power_by_rank`（同一イベント・デフォルト priority 100）
と同じパターンを採用し、priority のデフォルト値（100）をそのまま使用する。

## 注意点・エッジケース（review-test 向け）

1. **命中率・回避率ランクも合計に含める**
   - アシストパワーの備考「全能力6段階上がった状態で使うと威力が860となり」から逆算すると、
     5能力（A/B/C/D/S）のみの合計では最大30、威力620にしかならない。
     命中率・回避率を含めた7能力×6段階=42で初めて 20+20×42=860 と一致するため、
     命中率・回避率のランクも合計に含める必要がある。
   - 修正前の実装は `("atk", "def", "spa", "spd", "spe")` のみを対象としており、
     命中率・回避率を除外していた誤りがあったため本レビューで修正した。

2. **負のランク変化は無視する**
   - `max(0, v)` により、下がったステータスは0扱いとし、他の上昇分を相殺しない。

3. **てんねん特性との関係**
   - 相手のてんねんは威力計算そのものには影響しない（使用者のダメージ計算時のこうげきランク補正が
     無視されるのみで、つけあがるの威力自体はてんねんの影響を受けない）。本技側の実装に対応は不要。

4. **テスト方針**
   - ランク上昇なし（威力20）、A+1（威力40）、A+2/D+1合計+3（威力80）、A-1（威力20のまま）、
     ACC+1（威力40）の各ケースを確認する。
