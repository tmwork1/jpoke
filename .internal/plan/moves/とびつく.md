# とびつく 実装計画書

## 仕様要約（.internal/spec/moves/とびつく.md 参照）

* タイプ: むし / 分類: 物理 / 威力: 50 / 命中: 100 / PP: 20
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 追加効果として100%の確率で相手の『すばやさ』ランクを1段階下げる。

## 現状の実装状況（修正前）

`src/jpoke/data/move.py` のタイプ・分類・威力・命中・PPは仕様書と一致していたが、以下2点の誤り・欠落があった。

```python
"とびつく": MoveData(
    type="むし",
    category="physical",
    pp=20,
    power=50,
    accuracy=100,
    flags={"contact"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.とびつく_lower_defender_atk,
        )
    }
),
```

```python
def とびつく_lower_defender_atk(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"atk": -1})
```

1. **下げる能力値の誤り**: `stats={"atk": -1}` となっており、相手の『こうげき』を下げる実装になっていた。
   一次情報（`.internal/wiki/moves/とびつく.html`）・`.internal/spec/moves/_move_secondary_1.md`
   （とびつく: defender 確率1 S-1）とも、下がるのは『すばやさ』であり、実装が誤っていた。
2. **`flags` に `"secondary_effect"` が欠落**: `.internal/spec/moves/_move_secondary_1.md` に確率1の
   追加効果として記載されているため、ちからずくとの相互作用対象技であるべきだが、フラグが
   付与されておらず `ctx.move.has_flag("secondary_effect")` が常に False になっていた。

### 修正後

```python
"とびつく": MoveData(
    type="むし",
    category="physical",
    pp=20,
    power=50,
    accuracy=100,
    flags={"contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.とびつく_lower_defender_spd,
        )
    }
),
```

```python
def とびつく_lower_defender_spd(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spe": -1})
```

関数名は同種の速度低下技（`がんせきふうじ_lower_defender_spd`・`こごえるかぜ_lower_defender_spd` 等）の
命名パターンに合わせ `_lower_defender_spd`（この文脈の `spd` は "speed" の略であり、`Stat` 型の
`"spd"`（とくぼう）とは無関係）とした。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 100%の確率で相手の『すばやさ』ランクを1段階下げる |

`modify_defender_stats`（`handlers/move.py`）は `chance` 省略時デフォルト1（100%）で
`battle.resolve_secondary_chance` を経由するため、ちからずくによる追加効果無効化は汎用的に処理される。
本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（能力ランク低下）は既存の同種ハンドラ（がんせきふうじ・こごえるかぜ 等）と
同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。`.internal/spec/turn.md` にも
`ON_DAMAGE_HIT` は他の追加効果技と共通のタイミングとして扱われており、個別の priority 検討は不要。

### handlers/move_attack.py

`とびつく_lower_defender_atk` を `とびつく_lower_defender_spd` にリネームし、`stats={"atk": -1}` を
`stats={"spe": -1}` に修正する。

### data/move.py

`"とびつく"` エントリの `flags` に `"secondary_effect"` を追加し、ハンドラ参照を
`ha.とびつく_lower_defender_spd` に更新する。

## リーサル計算について

`.internal/progress/move.md` の該当行「リーサル実装」「リーサルテスト」列は `n/a`。本技の追加効果は相手の
『すばやさ』ランクを下げるのみでHPに影響しないため、`handlers/lethal.py` に専用ハンドラは不要であり、
対象外（n/a）のままで妥当。

## 注意点・エッジケース（review-test 向け）

1. **下げる能力値の誤り**
   - 修正前は『こうげき』を下げる実装になっていた。テストで `rank["spe"] == -1` を確認し、
     `rank["atk"] == 0` のまま変化しないことも合わせて検証できると再発防止になる。
2. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` を追加したことで、ちからずく所持時は威力が1.3倍になり、
     すばやさ低下効果が発動しなくなる（`がんせきふうじ` 等と同じ挙動）。
3. **テスト方針**
   - `secondary_chance` を発動させる設定で、すばやさランク変化が発動することを確認するテストを
     `tests/moves_attack/test_move_ta.py` に追加する（`とびかかる` の既存テストと同じパターン）。
