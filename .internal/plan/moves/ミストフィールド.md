# 実装計画書: ミストフィールド

## 仕様要約

- タイプ: フェアリー / カテゴリ: status / PP: 12（`src/jpoke/data/ps-champ-ja/moves.json` 準拠） / 対象: 全体の場（target="field"）
- 5 ターンの間、地形を「ミストフィールド」にする
- すでにミストフィールドが発動している場合は失敗する
- 詳細は `.internal/spec/moves/ミストフィールド.md` / `.internal/spec/fields/ミストフィールド.md` を参照。
  本計画書は技本体（ON_STATUS_HIT で地形を発動する処理）のみを扱う。地形自体の効果（ドラゴン技威力補正・
  状態異常／こんらん防止・カウントダウン等）は `.internal/plan/fields/ミストフィールド.md` を参照

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト、AttackContext） | `battle.terrain_manager.apply("ミストフィールド", 5)` を呼び地形を発動する |

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「天気・フィールド: すでに同じ」が priority 100 として記載されている。同種ハンドラ `エレキフィールド_activate_terrain`・`グラスフィールド_activate_terrain`・`サイコフィールド_activate_terrain` と同じパターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

```python
def ミストフィールド_activate_terrain(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ミストフィールド: 地形をミストフィールドにする。"""
    return HandlerReturn(value=battle.terrain_manager.apply("ミストフィールド", 5))
```

### `src/jpoke/data/moves/move_ma.py`

```python
"ミストフィールド": MoveData(
    type="フェアリー",
    category="status",
    pp=12,
    target="field",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ミストフィールド_activate_terrain,
        ),
    }
),
```

## 注意点・エッジケース

- `terrain_manager.apply` はすでに同じ地形が発動している場合に `False` を返す（技は失敗扱いになる）
- `エレキフィールド_activate_terrain` / `グラスフィールド_activate_terrain` / `サイコフィールド_activate_terrain` と同一パターン
- エレキフィールド・グラスフィールド・サイコフィールドと同時に発動することはできない（`terrain_manager` が排他管理する）
- グランドコートを持つポケモンが発動させた場合は継続ターンが8ターンに延長される（`ON_MODIFY_DURATION` 経由、`.internal/spec/items/グランドコート.md` を参照）
