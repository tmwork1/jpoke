# 計画書: サイコファング

## 仕様要約

- エスパータイプ / 物理 / 威力85 / 命中100 / PP12（champions基準。move_list.txtに準拠） /
  対象: 相手1体 / 優先度0 / 直接攻撃 / かみつき技（がんじょうあご対象）
- 相手の場のリフレクター・ひかりのかべ・オーロラベールを解除してから攻撃する。
- 壁の解除はダメージ計算前に行われるため、サイコファングのダメージ自体は壁の軽減効果を受けない。
- 壁解除は追加効果ではない（りんぷん・おんみつマント・ちからずくの対象外）。
- かわらわり（.internal/plan/moves/かわらわり.md）と全く同一の壁解除ロジックを持つ。

## ハンドラ構成

`サイコファング_break_screens`（`handlers/move_attack.py`）で
`defender_side.deactivate()` を呼び出し、リフレクター・ひかりのかべ・オーロラベールを解除する。
実装内容はかわらわりの `かわらわり_break_screens` と同一。

```python
def サイコファング_break_screens(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """サイコファング: 相手側のリフレクター・ひかりのかべ・オーロラベールを解除する。

    壁解除は追加効果ではないため、りんぷん・おんみつマント・ちからずくの対象外。
    ダメージ計算より前に解除する必要があるため、Event.ON_BEFORE_APPLY_MOVE で発動する
    （まもる・タイプ相性無効で技自体がここまで到達しない場合は壁を解除しない）。
    """
    defender_side = battle.get_side(ctx.defender)
    for wall in ("リフレクター", "ひかりのかべ", "オーロラベール"):
        defender_side.deactivate(wall)
    return HandlerReturn(value=value)
```

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_APPLY_MOVE`（コード上の定数は `Event.ON_BEFORE_APPLY_MOVE`。
コメントで「技の無効化チェック」とあり、まもる・タイプ相性無効の判定はこれより前段
（`ON_TRY_MOVE_1`/型判定/`ON_TRY_MOVE_2`）で完了しているため、このイベントに到達する時点で
まもる・タイプ相性による無効化は既に排除されている）に、
「30 | かわらわり等の効果発動」という行が明記されている。この priority=30 を
かわらわりと同じくそのまま使用する。

このイベントはダメージ計算（`roll_damage`）より前に発火するため、壁解除の効果が
今回のダメージ計算に反映される（＝壁の軽減を受けない）。

同じ priority=30 で登録されているみがわりの技無効化ハンドラ（`みがわり_immune`）は、
対象技が変化技の場合のみ `stop_event=True` で無効化する。物理技であるサイコファングに対しては
`stop_event` を発生させないため、みがわりに防がれた場合でも壁解除ハンドラは実行される
（一次情報の「みがわり状態で技を防がれたときも壁を解除できる」と一致）。

## 実装箇所

### data/moves/move_sa.py

```python
    "サイコファング": MoveData(
        type="エスパー",
        category="physical",
        pp=12,
        power=85,
        accuracy=100,
        flags={"bite", "contact"},
        handlers={
            Event.ON_BEFORE_APPLY_MOVE: h.MoveHandler(
                ha.サイコファング_break_screens,
                priority=30,
            ),
        },
    ),
```

- 修正前は `pp=10`（Gen9本家基準のPP）になっていたが、champions基準では
  `src/jpoke/data/ps-champ-ja/moves.json` の値（PP12）が正のため `pp=12` に修正する。
- 修正前は `Event.ON_HIT`（ダメージ計算後）に登録されており、サイコファング自身の
  ダメージがリフレクター/ひかりのかべ/オーロラベールの軽減を受けてしまう誤りがあったため、
  `Event.ON_BEFORE_APPLY_MOVE`（priority=30）に修正する（かわらわりと同じ修正）。
- `flags={"bite", "contact"}` は変更不要（がんじょうあご判定・直接攻撃判定に必要）。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `critical_rank` 省略時のデフォルトは `0`（通常の急所判定）→ 仕様と一致。

### handlers/move_attack.py

`サイコファング_break_screens` の実装内容自体（壁を解除する処理）は変更不要。
docstring を実際の発動イベント（`Event.ON_BEFORE_APPLY_MOVE`）に合わせて更新する。

## リーサル計算

対象外（n/a）。壁解除効果はHPやダメージ量の分布に影響を与えないため、
同種の壁解除技であるかわらわり（n/a）・レイジングブル（n/a）と同様に

## 注意点・エッジケース

- 技が外れた場合、命中判定チェック（`_check_hit`）で `_execute_move` が中断され
  `Event.ON_BEFORE_APPLY_MOVE` に到達しないため、壁は解除されない（仕様通り）。
- まもるに防がれた場合、`まもる_protect` は `Event.ON_TRY_MOVE_1`（本ハンドラより前段）で
  `stop_event=True` を返して技全体を中断するため、壁は解除されない（仕様通り）。
- タイプ相性で無効化された場合、`_check_hit_by_type`（`ON_TRY_MOVE_2` より前）で
  `_execute_move` が中断されるため、壁は解除されない（第五世代以降の仕様と一致）。
- みがわり状態への命中時は、`みがわり_immune`（同じ `Event.ON_BEFORE_APPLY_MOVE`, priority=30）が
  物理技に対しては `stop_event` を発生させないため、壁解除ハンドラは実行される（仕様通り）。
- がんじょうあご特性は `handlers/ability.py` の `がんじょうあご_modify_power` が
  `flags` の `"bite"` を汎用的に判定して1.5倍にするため、本技側で個別対応は不要。
