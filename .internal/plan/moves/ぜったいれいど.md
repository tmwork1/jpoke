# 計画書: ぜったいれいど

## 仕様要約

（`.internal/spec/moves/ぜったいれいど.md`、`.internal/spec/moves/_ohko.md` 参照）

* こおりタイプ・特殊、威力なし（一撃必殺技）、命中30固定（ただし使用者がこおりタイプでない場合は
  基礎命中率20）、PP5、非接触技。
* 命中した場合、対象のHPを0にする。反動・追加効果・急所判定なし。
* 対象が「こおり」タイプの場合、本技は無効となる（本技専用の仕様。通常のタイプ相性表では
  こおり×こおりは0.5倍でしかないため、専用のブロック処理が必要）。
* 一撃必殺技共通の無効化条件（がんじょう・きあいのタスキ等での生存、まもる等でのブロック、
  はりきりの命中率ペナルティ対象外、おうじゃのしるし等の追加効果ハンドラ対象外）は
  既存の `flags={"ohko"}` を参照する各特性・アイテムハンドラで共通対応済み。

## 既存実装の状況

`data/move.py` の `ぜったいれいど` エントリは実装済みで、`ha.ohko_damage`
（`handlers/move_attack.py`）を `Event.ON_MODIFY_MOVE_DAMAGE` に `priority=90` で登録する、
`じわれ`・`つのドリル`・`ハサミギロチン` と同一パターンを踏襲していた。

レビューの結果、以下の2点が未実装だった。

1. **こおりタイプへの無効化**（本技固有の仕様）が未実装だった。
2. **使用者がこおりタイプでない場合の命中率20への低下**が未実装だった
   （`accuracy=30` の固定値のみで、使用者タイプによる分岐がなかった）。

## イベント選定の根拠

* **こおりタイプへの無効化**: `Event.ON_TRY_MOVE_2`、`priority=120`。
  `.internal/spec/turn.md` の `Event.ON_TRY_MOVE_2` セクションに
  「120 | こおりタイプ: ぜったいれいど無効」と明記されている
  （相性判定 `ON_BEFORE_APPLY_MOVE` より後、`メロメロ` の性別判定などと同じ優先度帯）。
  `メロメロ_check_gender`（`handlers/move_status.py`）と同様、対象タイプを判定して
  `MOVE_IMMUNED` ログを出し `stop_event=True` で技を失敗させるパターンを踏襲する
  （タイプに基づく無効化のため `MOVE_FAILED` ではなく `MOVE_IMMUNED` を使う。
  `core/move_executor.py` の `_check_grass_type_powder_immunity` も同様に `MOVE_IMMUNED` を使用）。

* **命中率の使用者タイプ依存低下**: `Event.ON_MODIFY_ACCURACY`。
  `.internal/spec/turn.md` に個別記載のない計算内部イベントだが、既存の `かみなり_accuracy`・
  `ふぶき_accuracy`（`handlers/move_attack.py`）が同一イベントで天候条件に応じて命中率の値を
  直接上書きする実装パターンを採用しており、priorityは指定せずデフォルト値（100）を用いる。
  使用者がこおりタイプでない場合のみ value を20に上書きし、こおりタイプの場合は
  base値（30）のまま変更しない。

## ハンドラ構成

| 対象 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| ぜったいれいど（技） | `Event.ON_MODIFY_MOVE_DAMAGE` | 90 | `attacker:self`（`ha.ohko_damage` は既存共通ハンドラ） | 対象のHPと同値を確定ダメージとして設定する |
| ぜったいれいど（技） | `Event.ON_TRY_MOVE_2` | 120 | `attacker:self` | 対象がこおりタイプの場合、技を無効化する |
| ぜったいれいど（技） | `Event.ON_MODIFY_ACCURACY` | 100（デフォルト） | `attacker:self` | 使用者がこおりタイプでない場合、命中率を20に上書きする |

```python
"ぜったいれいど": MoveData(
    type="こおり",
    category="special",
    pp=5,
    power=0,
    accuracy=30,
    flags={"ohko"},
    handlers={
        Event.ON_TRY_MOVE_2: h.MoveHandler(
            ha.ぜったいれいど_check_ice_immunity,
            priority=120,
        ),
        Event.ON_MODIFY_ACCURACY: h.MoveHandler(
            ha.ぜったいれいど_modify_accuracy,
        ),
        Event.ON_MODIFY_MOVE_DAMAGE: h.MoveHandler(
            ha.ohko_damage,
            priority=90,
        ),
    }
),
```

## テスト

* 対象がこおりタイプの場合、技が無効化されダメージを与えないこと
  （`test_一撃必殺技_タイプ相性で無効化される` に `("ぜったいれいど", <こおりタイプのポケモン>)`
  を追加する）。
* 命中したとき、対象のHPが0になること（既存の `test_一撃必殺技_命中時は相手を一撃で倒す` は
  `つのドリル` を代表技として使うが、`ぜったいれいど` 固有の命中テストも
  `tests/moves_attack/` に追加する）。
* 使用者がこおりタイプでない場合、命中率が20として扱われること（乱数を固定して、
  20%ちょうどの境界で命中/失敗が切り替わることを確認する）。
* 使用者がこおりタイプの場合、命中率が30のまま（20に低下しない）であること。

## 注意点・エッジケース

* こおりタイプへの無効化は、じわれのひこうタイプ無効化（既存のタイプ相性・浮遊判定に委譲）とは
  異なり、通常のタイプ相性表では倍率0にならないため、本技専用のハンドラが必要になる。
* 命中率の低下は「使用者のタイプ」に依存し、「対象のタイプ」には依存しない
  （使用者がこおりタイプなら対象のタイプによらず基礎命中率30、使用者がこおりタイプでなければ
  対象のタイプによらず基礎命中率20。ただし対象がこおりタイプの場合はいずれにせよ無効化される）。
