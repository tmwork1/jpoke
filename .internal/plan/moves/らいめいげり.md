# らいめいげり 実装計画書

## 仕様要約（.internal/spec/moves/らいめいげり.md 参照）

* タイプ: かくとう / 分類: 物理 / 威力: 90 / 命中: 100 / PP: 10
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 追加効果として、100%の確率で相手の『ぼうぎょ』ランクを1段階下げる。

## 現状の実装状況（修正前）

`src/jpoke/data/moves/move_ra.py` に登録済みだが、`flags` に `"secondary_effect"` が
欠落しているため、ちからずく特性による威力1.3倍化・追加効果無効化が適用されない誤りがあった。
同じ「100%相手ランク低下」パターンの `うらみつらみ` `ほのおのムチ` は `secondary_effect` を
保持しており、`らいめいげり` も .internal/wiki/abilities/ちからずく.html に記載の
「相手のランク補正を下げる技」に該当するため、本来は対象技として扱うべきである。

## 修正後

```python
"らいめいげり": MoveData(
    type="かくとう",
    category="physical",
    pp=10,
    power=90,
    accuracy=100,
    flags={"contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.らいめいげり_lower_defender_def,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 100%の確率で相手の『ぼうぎょ』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def らいめいげり_lower_defender_def(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"def": -1})
```

`modify_defender_stats`（`handlers/move.py`）の `chance` 既定値は1のため、
`battle.resolve_secondary_chance(ctx, 1)` を経由してちからずくのポケモンが使用した場合は
確率が0化され効果が発動しなくなる。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（能力ランク低下）は既存の同種ハンドラ（うらみつらみ・
ほのおのムチ・りんごさん 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）の
パターンに従う。`ON_DAMAGE_HIT` は `.internal/spec/turn.md` に未掲載のイベント
（ダメージ計算内部）だが、既存の同種ハンドラの実装パターンを踏襲するため
個別の priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` を追加したことで、ちからずく所持時は威力1.3倍（90→117）になり、
     ぼうぎょ低下の追加効果は発動しなくなる。
2. **接触フラグ**
   - `flags={"contact"}` により、接触時発動効果・特性（『さめはだ』『てつのとげ』等）の対象となる。
     個別のガードは不要（各特性・アイテム側で判定される）。
3. **テスト方針**
   - 100%固定効果のため `secondary_chance` の調整は不要。ぼうぎょランクの変化を確認する
     テスト（`test_らいめいげり_ぼうぎょ1段階低下が発動する`）を追加する。
