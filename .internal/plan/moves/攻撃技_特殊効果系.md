## 対象技一覧

## 仕様要約

### トライアタック
- ダメージ後、20%の確率で相手を状態異常にする
- まひ・やけど・こおりのいずれかをランダムで付与（各1/3）
- 単一乱数判定：r < base/3 → まひ、r < base*2/3 → やけど、r < base → こおり

### いにしえのうた
- ダメージ後、10%の確率で相手をねむり状態にする
- 音系の技（sound ラベル）
- 相手のみがわり状態でも効果が届く（音技の通常挙動）

### しおづけ
- ダメージ後、100%の確率で相手をしおづけ揮発状態にする
- しおづけ状態ハンドラは `handlers/volatile.py` に実装済み（毎ターン最大HPの1/16ダメージ、みず・はがねタイプは1/8）
- `apply_volatile_to_defender` ヘルパーで付与可能

### とどめばり
- この技で相手を倒すとこうげきランクが3段階上がる
- ON_DAMAGE_HIT ではなく「相手がひんしになった場合」のタイミングが必要
- turn.md の ON_DAMAGE の priority 100 に「とどめばりによるこうげき上昇」が明記されている
- 実装上は ON_DAMAGE_HIT ハンドラで `ctx.defender.hp == 0`（または alive==False）を確認する

### みずあめボム
- ダメージ後、100%の確率で相手をあめまみれ揮発状態にする
- あめまみれ状態ハンドラは `handlers/volatile.py` に実装済み（3ターン間毎ターンS-1段階）
- `apply_volatile_to_defender` ヘルパーで付与可能
- 命中率85%（accuracy=85 は既に data に登録済み）

### みわくのボイス
- ダメージ後、100%の確率で「そのターンに能力ランクが1つでも上がった相手」をこんらん状態にする
- 相手が後攻の場合、通常はこの技を先に使う必要がある
- 場に出た時にランクが上昇した場合（いかく等）も対象
- 音系の技（sound ラベル）
- Pokemon モデルに `stat_raised_this_turn: bool` フィールドの追加が必要
  - `battle.modify_stats` でランクが実際に上昇したとき当フラグを True にする
  - `reset_turn_state()` でリセットする
- ON_SWITCH_IN 時のいかく等によるランク低下ではなく**上昇**した場合もトラッキングする

### しっとのほのお
- ダメージ後、100%の確率で「そのターンに能力ランクが1つでも上がった相手」をやけど状態にする
- 仕様・条件は `みわくのボイス` と同じ（状態異常付与が異なる）
- 音系ではない通常の特殊技
- `stat_raised_this_turn` フラグに依存（みわくのボイスと同一の新フィールド）

## Priority 根拠

- `.internal/spec/turn.md` の `Event.ON_DAMAGE` テーブルに「priority 10: 追加効果（特殊なもの除く）」と記載
- `.internal/spec/turn.md` の `Event.ON_DAMAGE` テーブルに「priority 100: とどめばりによるこうげき上昇」と記載
- **ON_DAMAGE_HIT は turn.md に未掲載のイベント**。既存の同種ハンドラ（かえんほうしゃ、かみなり等）が `priority=100`（MoveHandler デフォルト）を使用しているため、これに倣い priority=100 とする

> turn.md 記述:
> ```
> | 10 | 追加効果（特殊なもの除く）  ← ON_DAMAGE
> | 100 | とどめばりによるこうげき上昇 ← ON_DAMAGE
> ```
>
> ON_DAMAGE_HIT は内部処理順位であり、既存ハンドラ実装上は全て priority=100（MoveHandler デフォルト）を使用。

### 2. battle.modify_stats でフラグ更新

`src/jpoke/core` の modify_stats（または StatusManager）の処理で、ランクが実際に上昇した場合に `target.stat_raised_this_turn = True` をセットする。

ただし、ランク上昇がクリアボディ等で無効化された場合はフラグを立てない。

### 3. handlers/move_attack.py に追加する関数

```python
def トライアタック_apply_ailment_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """トライアタックの追加効果: 20%の確率でまひ/やけど/こおりのいずれかを付与する。

    単一乱数で判定: r < base/3 → まひ、r < base*2/3 → やけど、r < base → こおり。
    """
    base = battle.resolve_secondary_chance(ctx, 0.2)
    r = battle.random.random()
    if r < base / 3:
        ailment = "まひ"
    elif r < base * 2 / 3:
        ailment = "やけど"
    elif r < base:
        ailment = "こおり"
    else:
        return HandlerReturn(value=value)
    return HandlerReturn(value=battle.ailment_manager.apply(
        ctx.defender, ailment, source=ctx.attacker, ctx=ctx
    ))


def いにしえのうた_apply_ailment_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="ねむり", chance=0.1)


def しおづけ_apply_volatile_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_defender(battle, ctx, value, volatile="しおづけ")


def とどめばり_boost_attacker_A(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """とどめばりの追加効果: この技で相手を倒すとこうげきランクが3段階上がる。"""
    if not ctx.defender.alive:
        battle.modify_stats(ctx.attacker, {"A": 3}, source=ctx.attacker)
    return HandlerReturn(value=value)


def みずあめボム_apply_volatile_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_defender(battle, ctx, value, volatile="あめまみれ", count=3)


def みわくのボイス_apply_confusion_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """みわくのボイスの追加効果: そのターンにランクが上がった相手をこんらん状態にする。"""
    if not ctx.defender.stat_raised_this_turn:
        return HandlerReturn(value=value)
    return apply_confusion_to_defender(battle, ctx, value)


def しっとのほのお_apply_ailment_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """しっとのほのおの追加効果: そのターンにランクが上がった相手をやけど状態にする。"""
    if not ctx.defender.stat_raised_this_turn:
        return HandlerReturn(value=value)
    return apply_ailment_to_defender(battle, ctx, value, ailment="やけど")
```

## 注意点・エッジケース

### トライアタック
- まひ/やけど/こおりの各1/3は「単一乱数で切り分け」であり、3回独立した判定ではない
- 各状態異常の付与に `ailment_manager.apply` を使うため、すでに状態異常の場合・タイプ免疫（でんき→まひ免疫等）・みずのベール等の特性チェックは ailment_manager 内部で自動処理される
- `resolve_secondary_chance` による確率調整（てんのめぐみ2倍等）が適用される

### いにしえのうた
- 音技（sound ラベル）であるため、みがわり状態でも効果が届く
- ねむり付与は ailment_manager の通常チェックが適用（ふみん等でブロック）
- さわぐ状態中はねむり付与に失敗する（ailment_manager 内部で処理済み）

### しおづけ
- 「しおづけ」揮発状態はすでに `handlers/volatile.py` に実装済み
- すでにしおづけ状態の場合は `volatile_manager.apply` 内部で重複チェックされる（または上書きされる）ため、ハンドラ側でのガード不要
- チャンピオンズ仕様: 通常1/16、みず・はがねタイプは1/8（SV とは異なる）
  → volatile ハンドラで既に実装済み

### とどめばり
- `ctx.defender.alive` は ON_DAMAGE ハンドラ時点（ダメージ付与後の最初の処理）で既に False になっている
- ランク上昇は `battle.modify_stats` で実装（攻撃側に対して、クリアボディ等は関係なし）
- turn.md に「priority 100: とどめばりによるこうげき上昇」と明記されているため priority=100

### みずあめボム
- 「あめまみれ」状態は `handlers/volatile.py` に実装済み（3ターン間毎ターンS-1段階）
- `volatile_manager.apply(..., count=3, ...)` で3ターン分を指定する
- 命中率が85%であることに注意（ON_CHECK_ACCURACY で適用）

### みわくのボイス / しっとのほのお
- **Pokemon モデルへの `stat_raised_this_turn: bool` フィールドの追加が必須**
- StatusManager（または battle.modify_stats）の処理で実際にランクが上昇した場合に True を設定する必要がある
- `reset_turn_state()` でターン開始時にリセットする
- 「場に出た時にランクが上がった場合」（ON_SWITCH_IN での特性いかく等）も True になる
  → ただし、いかくは A を**下げる**特性なので上昇フラグには影響しない
  → ダウンロード、グラスシード等のランク上昇がトリガーとなる
- みわくのボイスは音技のため「みがわり状態でも通過する」が、こんらん付与はみがわりで防がれる可能性がある（音技の「みがわり貫通」は攻撃ダメージには適用されるが、追加効果の扱いは実装依存）
  → `apply_confusion_to_defender` のラップ内で volatile_manager.apply_confusion が呼ばれ、そこでみがわりチェックが行われる
- しっとのほのお・みわくのボイスは「後攻で使う」設計だが、先攻でも発動自体は可能（その場合は「そのターン既にランクが上がっていた場合のみ有効」）
- ランク上昇の記録は **対象側（defender）の `stat_raised_this_turn`** を参照する
