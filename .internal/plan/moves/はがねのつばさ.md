# はがねのつばさ 実装計画書

## 仕様要約

`.internal/spec/moves/はがねのつばさ.md` を参照。
はがねタイプ・物理・威力70・命中90・PP25の直接攻撃技。
追加効果として10%の確率で自分の「ぼうぎょ」ランクを1段階上げる。

本技は `.internal/plan/moves/攻撃技_攻撃側ランク変化系.md`（攻撃側ランク変化技の一括実装計画）に含まれている。
以下はその内容のうち本技に関する部分の抜粋。

## ハンドラ構成

`Event.ON_HIT` に `MoveHandler` デフォルト設定（`subject_spec="attacker:self"`, `priority=100`）で登録する。
`modify_attacker_stats` は `handlers/move.py` の既存共通関数を使用する。

| 技 | event | priority | subject_spec | 役割 |
|---|---|---|---|---|
| はがねのつばさ | ON_HIT | 100 | attacker:self | B+1（10%） |

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_HIT` セクションに「自分のランクを上げる追加効果」は priority=10 と記載があるが、
この値は ON_HIT イベント内の処理グループの優先度を示すものであり、`MoveHandler` の `priority` 引数（イベント内での
個別ハンドラ順序）とは別軸である。既実装の自分ランク上昇技（`メタルクロー_boost_attacker_A` 等）と同様に
`MoveHandler` デフォルト（priority=100）で登録する。

## 実装箇所

### `src/jpoke/handlers/move_attack.py`

```python
def はがねのつばさ_boost_attacker_B(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"def": 1}, chance=0.1)
```

既存の `メタルクロー_boost_attacker_A`（はがねタイプ・物理・自分Aランク上昇10%）と同型のパターン。
関数名は「自分自身のランクを上げる」ことを表すため `_boost_attacker_B` とする
（旧実装では `_boost_defender_B` という誤った名称になっていたため、レビューで修正）。

### `src/jpoke/data/move.py`

```python
"はがねのつばさ": MoveData(
    type="はがね",
    category="physical",
    pp=25,
    power=70,
    accuracy=90,
    flags={"contact", "secondary_effect"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.はがねのつばさ_boost_attacker_B,
        )
    }
),
```

## リーサル計算

- 対象外（`n/a`）。自分自身の防御ランク上昇はダメージ量計算そのものには直接影響しないため、
  専用のリーサルハンドラは不要（`.internal/progress/move.md` の当該行も `n/a` 指定）。

## エッジケース

- 発動確率は10%。`chance=0.1` は `modify_attacker_stats` 内の `battle.resolve_secondary_chance` /
  `battle.random.random()` 判定で制御される。
- ランク上限（+6）は `battle.modify_stats` 側で処理されるため、ハンドラ側に境界チェックは不要。
- ON_HIT はダメージ発生後に発火するため、みがわりに命中した場合でも自分自身への効果は発動する。
