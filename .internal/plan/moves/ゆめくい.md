# 計画書: ゆめくい

## 仕様要約（.internal/spec/moves/ゆめくい.md 参照）

- エスパータイプ / 特殊 / 威力100 / 命中100 / PP15 / 対象: 1体選択 / 優先度0 / 接触なし
- ねむり状態の相手にのみダメージを与えられる（ねむり状態でない相手に使うと失敗するが、
  技を選択すること自体はできる。選択してから使用するまでに相手がねむり状態になった場合は成功し、
  逆に使用時までにねむり状態が回復していた場合は失敗する＝判定は使用直前に行う）。
- 特性 ぜったいねむり（ゆめうつつ状態）のポケモンに対しても効果がある。
- 相手に与えたダメージの半分だけ自分のHPを回復する（回復量は第五世代以降四捨五入）。
- おおきなねっこ所持時は回復量が1.3倍。ヘドロえき所持の相手を攻撃した場合、回復効果がダメージに変わる。
- かいふくふうじ状態のポケモンは回復できない。
- まもるで防がれる。みがわりに阻まれた場合はみがわりへのダメージを基準に回復量を算出する
  （現行世代では第四世代までの「みがわり無効」仕様は撤廃されている）。

## 既存実装との差分・修正内容

既存実装は `data/moves/move_ya.py` の `"ゆめくい"` エントリ、
`handlers/move_status.py` の `ゆめくい_check_sleep`、`handlers/move_attack.py` の `ゆめくい_drain`。

### バグ修正: `ゆめくい_check_sleep` の判定条件

修正前:
```python
def ゆめくい_check_sleep(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ゆめくいの使用条件チェック: 相手がねむり状態でない場合に失敗させる。"""
    if not ctx.defender.has_ailment("ねむり"):
        ...
```

`has_ailment("ねむり")` は状態異常名が厳密に `"ねむり"` の場合のみ True を返す。
特性ぜったいねむりは `ゆめうつつ`（`types/ailment.py` の別の `AilmentName`）を付与するため、
この判定では ぜったいねむり 特性のポケモンに対して常に失敗してしまい、
仕様「特性 ぜったいねむり のポケモンに対しても効果がある」に反していた。

同種の判定（自分の眠り状態を確認する `いびき_check_sleep`）では
`ctx.attacker.ailment.is_sleep`（`ねむり` と `ゆめうつつ` の両方で True になるプロパティ、
`model/ailment.py` 参照）を使っており、この慣例に合わせて修正した。

修正後:
```python
def ゆめくい_check_sleep(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ゆめくいの使用条件チェック: 相手がねむり状態（ゆめうつつ含む）でない場合に失敗させる。

    特性 ぜったいねむり のポケモンは「ゆめうつつ」状態を持つため、
    is_sleep で判定して対象に含める。
    """
    if not ctx.defender.ailment.is_sleep:
        ...
```

## ハンドラ構成

### handlers/move_status.py: `ゆめくい_check_sleep`（修正）

- イベント: `Event.ON_BEFORE_APPLY_MOVE`
  （`.internal/spec/turn.md` に未掲載のイベントのため、同種の使用条件チェック
  `ちからをすいとる_can_apply` / `みずびたし_can_apply` 等の既存実装を参考に、
  `MoveHandler` のデフォルト優先度100・デフォルト `subject_spec="attacker:self"` のまま採用。
  これらは選択時ではなく技の実行直前に判定されるため、
  「選択後に相手が寝た場合は成功／使用時に相手が起きていた場合は失敗」という
  タイミング仕様を自然に満たす）。
- 判定対象は `ctx.defender.ailment.is_sleep`（`ねむり` および `ゆめうつつ` を含む）。

### handlers/move_attack.py: `ゆめくい_drain`（既存・変更なし）

```python
def ゆめくい_drain(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """ゆめくいの回復量を計算する。"""
    _drain_hp(battle, ctx, value, heal_ratio=0.5)
    return HandlerReturn(value=value)
```

- イベント: `Event.ON_HIT`、優先度20（`.internal/spec/turn.md` の `## Event.ON_HIT` に
  「20 | HP吸収技による回復」と明記されており、`ギガドレイン_drain` 等の既存ドレイン技と同じ優先度）。
- `_drain_hp`（`handlers/move.py` 系ヘルパー、`handlers/move_attack.py` 内定義）は
  `int(damage * heal_ratio)` で回復量を算出する。`heal_ratio=0.5` の場合、端数は必ず `.0` か `.5` に
  なるため、Python の `int()`（0方向への切り捨て）と `round_half_down`（五捨五超入）は常に同じ結果になり、
  第五世代以降の「四捨五入」＝実際のゲーム内丸め仕様（0.5は切り捨て）と整合している。追加修正不要。
- `_drain_hp` は内部で `Event.ON_CALC_DRAIN` を発火し、`data/item.py` に登録された
  おおきなねっこ（1.3倍）のハンドラが自動適用される。
- 回復は `battle.modify_hp(ctx.attacker, v=heal_amount, reason="drain")` 経由で行われ、
  `modify_hp` が `v > 0` のとき `Event.ON_MODIFY_HEAL` を発火するため、
  ヘドロえき（回復をダメージに変換、`ability.py` の `ヘドロえき_reverse_drain`）と
  かいふくふうじ（回復無効化、`volatile.py` の `かいふくふうじ_block_heal`）が
  個別実装なしに自動適用される。
- `damage = damage or ctx.substitute_damage` により、みがわりに阻まれた場合はみがわりへの
  ダメージを基準に回復量を算出する。現行世代の「みがわりに阻まれても本来の技と同様に扱う」
  仕様と一致しており、ゆめくい固有の実装は不要。

### data/moves/move_ya.py（既存・変更なし）

```python
"ゆめくい": MoveData(

    category="special",
    pp=15,
    power=100,
    accuracy=100,
    flags={"heal"},
    handlers={

```

## 注意点・エッジケース

- ねむり状態でない相手に対する使用時判定は `ON_BEFORE_APPLY_MOVE`（技の実行直前）で行われるため、
  技選択後・実行前に相手が寝た場合は成功、逆に実行前に相手が起きた場合は失敗という
  タイミング仕様を追加実装なしに満たす。
- 特性ぜったいねむり（ゆめうつつ）の相手にも成功するかどうかは `is_sleep` プロパティで判定する
  （本レビューで修正済み）。
- 「第六世代以降、かいふくふうじ状態では選択自体ができない／攻撃前にかいふくふうじにされた場合は
  技自体が失敗する」という仕様は、`heal` フラグを持つ全ての回復技に共通する一般的な仕様であり、
  本コードベースには現時点でこの選択制限・実行時失敗メカニズムを実装した箇所が見当たらない
  （`ON_MODIFY_HEAL` による回復量ゼロ化のみ実装済み）。これはゆめくい固有の欠落ではなく
  `heal` フラグを持つ全技に共通する仕様ギャップのため、本レビューのスコープ外として着手しない。
