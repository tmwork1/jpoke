# 実装計画書: アイススピナー

参照仕様書: .internal/spec/moves/アイススピナー.md

## 実装方針

アイススピナーは通常の物理攻撃技（こおりタイプ・威力80・命中100）に、命中後のダメージ処理の
一環として「場のフィールドを解除する」効果を追加した技。この効果は「追加効果」ではないため
（一次情報: .internal/wiki/moves/アイススピナー.html 技の仕様節）、`flags` に `"secondary_effect"` を
付与しない（付与するとちからずくで誤って威力が上がったり、りんぷん／おんみつマントで誤って
フィールド解除が防がれてしまう）。

同系統の「アイアンローラー」との違いは、フィールドが存在しない状態でも技自体は失敗しない点。
そのため `ON_TRY_MOVE_1` での存在チェック（アイアンローラーの `_check_terrain` 相当）は不要で、
`ON_DAMAGE_HIT` でのフィールド解除ハンドラのみを実装する。

### データ定義（`src/jpoke/data/moves/move_a.py`）

* `type="こおり"`
* `category="physical"`
* `power=80`
* `accuracy=100`
* `pp=16`（`src/jpoke/data/ps-champ-ja/moves.json` に基づく。Gen9本家は15だがポケモンチャンピオンズでは
  16に統一されている）
* `flags={"contact"}`（`"secondary_effect"` は付与しない。上記の通りフィールド解除は追加効果
  ではないため）
* `handlers`: `Event.ON_HIT`（priority=180）と `Event.ON_DAMAGE_HIT`（priority=180）の2本
  （後述の「みがわり／ばけのかわ／アイスフェイス対応」参照）

### ハンドラ実装（`src/jpoke/handlers/move_attack.py`）

```python
def アイススピナー_clear_terrain(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """アイススピナー: ダメージ後にフィールドを解除する。

    使用者がいのちのたまの反動・ゴツゴツメット・さめはだ等、自身へのダメージ処理中に
    ひんしになった場合はフィールドを解除しない。
    """
    if ctx.attacker.fainted:
        return HandlerReturn(value=value)
    battle.terrain_manager.remove()
    return HandlerReturn(value=value)


def アイススピナー_clear_terrain_on_zero_damage_hit(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """アイススピナー: みがわり/ばけのかわ/アイスフェイスにダメージを肩代わりされた場合、
    フィールドを解除する（value=実際のダメージ量が0のときのみ）。
    """
    if value == 0 and not ctx.attacker.fainted:
        battle.terrain_manager.remove()
    return HandlerReturn(value=value)
```

`battle.terrain_manager.remove()` はフィールドが存在しない場合 `False` を返すのみで副作用が
ないため、フィールド未使用時に安全に呼び出せる（アイアンローラーと異なり事前チェック不要）。

#### みがわり／ばけのかわ／アイスフェイス対応（`Event.ON_DAMAGE_HIT` だけでは不足）

`.internal/wiki/moves/アイススピナー.html` 技の仕様節には「対象のみがわり状態や、特性ばけのかわ／
アイスフェイスにダメージを肩代わりされたときはフィールドを解除できる」と明記されている。
しかし `src/jpoke/core/move_executor.py` の `_execute_hit` は、`battle.modify_hp` で計算される
`actual_damage`（対象の実HP減少量）が `0` 以下の場合、`Event.ON_DAMAGE_HIT` を発火せずに
処理を打ち切る実装になっている。みがわり・ばけのかわ・アイスフェイスはいずれも
`Event.ON_MODIFY_MOVE_DAMAGE` でダメージ量を `0` に上書きしてから独自に肩代わり処理を行うため
（`みがわり_block_damage` / `ばけのかわ_block_damage` / `アイスフェイス_block_physical`）、
これらが発動したヒットでは `actual_damage == 0` となり `ON_DAMAGE_HIT` が発火しない。
そのため `Event.ON_DAMAGE_HIT` のみをフックする実装では一次情報の記述を満たせない
（レビューで実機テストにより確認: `test_アイススピナー_みがわりに肩代わりされてもフィールドを
解除する` が実装前は失敗していた）。

この問題は既存の `いのちのたま`（`src/jpoke/data/item.py`）でも同様に起きており、
「`ON_DAMAGE_HIT` は `actual_damage<=0` のとき発火しないため採用せず、常に発火する
`Event.ON_HIT`（`value`=実際のダメージ量、みがわり被弾時も含め必ず発火）を使う」という
既存の解決パターンがある。アイススピナーもこれに倣い、`value == 0` のときのみフィールドを
解除する専用ハンドラを `Event.ON_HIT` に追加する。

#### いのちのたま等の反動でひんしになった場合の除外

一次情報には「自分のいのちのたまの反動で使用者がひんしになったときはフィールドを解除できない」
「対象の持ち物ゴツゴツメット、特性さめはだ...などの効果...で使用者がひんしになったときは
フィールドを解除できない」とも明記されている。`いのちのたま`の反動は `Event.ON_HIT`
priority=160 で実装されており、`ゴツゴツメット`・`さめはだ` は `Event.ON_DAMAGE_HIT`
priority=90/20 で実装されている（いずれも本ハンドラの priority=180 より早い）。
`_check_handler_validity`（`src/jpoke/core/event_manager.py`）には fainted 時にハンドラを
自動スキップする仕組みが無いため、両ハンドラとも明示的に `ctx.attacker.fainted` を
チェックしてから解除処理を行う。

* `Event.ON_HIT` 側（`アイススピナー_clear_terrain_on_zero_damage_hit`）の priority は
  いのちのたまの反動（priority=160）より確実に後で実行されるよう `180` とする
  （turn.md に `ON_HIT` 側でのこの効果の掲載はないが、同一効果の `ON_DAMAGE_HIT` 側の
  値=180 に揃え、既存の「反動より後」という順序要件を満たすことを優先した）。
* `Event.ON_DAMAGE_HIT` 側（`アイススピナー_clear_terrain`）の priority=180 は
  `.internal/spec/turn.md` の「| 180 | アイアンローラー等でフィールド消失 |」に直接記載がある値。
  ゴツゴツメット（90）・さめはだ（20）より後に実行されるため、`ctx.attacker.fainted` の
  チェックのみで両方の反動ケースを正しく除外できる。

## 相互作用の確認

* **まもる**: `flags` に `"unprotectable"` を付与しないため通常通り防がれ、`_execute_hit` 自体が
  呼ばれず `ON_HIT`／`ON_DAMAGE_HIT` のどちらも発火しないためフィールドは解除されない
  （一次情報通り）。
* **マジックコート**: 攻撃技のため対象外（一次情報通り反射されない）。
* **ちからずく**: `"secondary_effect"` フラグを付与しないため、`ちからずく_boost_power` 等の
  既存ハンドラは反応せず威力は上がらない。
* **りんぷん／おんみつマント**: 同様に `"secondary_effect"` フラグを持たないため、
  `りんぷん_block_secondary_chance` 等は反応せずフィールド解除は防がれない。
* **くさのけがわ／クォークチャージ**: ダメージ計算過程（`ON_HIT`／`ON_DAMAGE_HIT` より前）で
  通常通り特性の影響を受け、その後にフィールドが解除される（既存のダメージ計算ロジックに委ねる）。
* **みがわり／ばけのかわ／アイスフェイス**: 上記の通り `Event.ON_HIT` 側ハンドラが
  `value == 0` を検出してフィールドを解除する。
* **いのちのたま／ゴツゴツメット／さめはだ等の反動で使用者がひんしになった場合**:
  両ハンドラの `ctx.attacker.fainted` チェックにより解除しない。

## テスト方針

`tests/moves_attack/test_move_a.py` に以下を確認するテストを追加・拡張する。

* 命中時、場のフィールド（エレキフィールド・グラスフィールド・ミストフィールド・
  サイコフィールド）が解除されることを確認するテスト
* フィールドが存在しない状態でも通常通りダメージを与えて技が成功することを確認するテスト
* まもるで防がれた場合、フィールドが解除されないことを確認するテスト
* 対象がみがわり状態でダメージを肩代わりしても、フィールドは解除されることを確認するテスト
* 対象の特性ばけのかわでダメージを肩代わりされても、フィールドは解除されることを確認するテスト
* 使用者がいのちのたまの反動でひんしになった場合、フィールドを解除しないことを確認するテスト
* 対象がみがわりで肩代わりしても、使用者がいのちのたまの反動でひんしになった場合は
  フィールドを解除しないことを確認するテスト（`ON_HIT` 側ハンドラの除外判定の確認）
* 対象の持ち物ゴツゴツメットの反動で使用者がひんしになった場合、フィールドを解除しない
  ことを確認するテスト
