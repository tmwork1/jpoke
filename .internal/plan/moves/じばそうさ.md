# 計画書: じばそうさ

## 仕様要約

- タイプ: でんき、変化、PP: 20、対象: 味方全員（シングルバトルでは自分のみ）、必中
- 特性が「プラス」または「マイナス」のポケモンの「ぼうぎょ」「とくぼう」ランクをそれぞれ1段階ずつ上げる
- 使用者自身の特性が「プラス」/「マイナス」の場合、自分にも効果がある（シングルバトルでは使用者＝対象）
- 特性が「プラス」/「マイナス」でないポケモンには効果がない（失敗する）
- いえき・かがくへんかガスによりプラス/マイナスが消えている場合も効果なし
- みがわり状態・まもる状態でも効果がある
- よこどり可（本プロジェクト未実装のため対象外）（Zワザ時は「とくぼう1段階上昇」の付加効果、Zワザ非対応のため未実装）

出典: `.internal/wiki/moves/じばそうさ.html`、`.internal/spec/moves/じばそうさ.md`

## Priority 根拠

- `ON_BEFORE_APPLY_MOVE` priority 100: `.internal/spec/turn.md` の「Event.ON_APPLY_MOVE」セクション
  （line 407、コード上の enum 名は `Event.ON_BEFORE_APPLY_MOVE`）に「じばそうさ等: 特性不一致」が
  priority 100 として明記されている。既存実装（`なかまづくり_can_apply`、`なりきり_can_apply` 等）も
  同一パターンで priority 100（デフォルト）を使用している
- `ON_STATUS_HIT` priority 100: `.internal/spec/turn.md` に個別記載なし。既存の同種ハンドラ
  （`コスモパワー_modify_attacker_stats` 等）がすべて priority=100（デフォルト）で登録されているため、
  100 を使用する

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: `じこさいせい_heal_self` の後、`じゅうでん_apply` の前（五十音順）

```python
def じばそうさ_can_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """じばそうさの失敗条件: 使用者の特性がプラス/マイナスでない場合は失敗させる。"""
    if ctx.attacker.ability.name not in ("プラス", "マイナス"):
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="じばそうさ")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def じばそうさ_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """じばそうさの効果: ぼうぎょ・とくぼうをそれぞれ1段階上げる。"""
    return modify_attacker_stats(battle, ctx, value, stats={"def": 1, "spd": 1})
```

`Stat` の値は `"def"` / `"spd"` を使用する（`"B"` / `"D"` のような表記は
`src/jpoke/types/literals.py` の `Stat` リテラルに存在しないため誤り）。

### `src/jpoke/data/moves/move_sa.py`

`"じばそうさ"` エントリの `handlers` に加えて `target="self"` を指定する:

```python
    "じばそうさ": MoveData(
        type="でんき",
        category="status",
        pp=20,
        target="self",
        handlers={
            Event.ON_BEFORE_APPLY_MOVE: h.MoveHandler(
                hs.じばそうさ_can_apply,
            ),
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.じばそうさ_modify_attacker_stats,
            ),
        }
    ),
```

`target="self"` が必須である理由は「注意点・エッジケース」を参照。

## 注意点・エッジケース

- シングルバトルでは対象は常に使用者自身のため、「味方全員」効果は使用者に対してのみ発動する
- 失敗条件チェックは `ON_BEFORE_APPLY_MOVE` で使用者の特性を確認する。いえき状態・かがくへんかガスにより
  特性が消えている場合は `ability.name` が変わるため、別途ガードは不要
- ぼうぎょ・とくぼうの両方を1つの `stats` dict にまとめて `modify_attacker_stats` に渡せる
  （キーは `Stat` リテラルの `"def"` / `"spd"`）
- どちらか一方のランクが最大（+6）の場合は、そのランクの上昇のみ失敗し、もう一方は成功する
  （`modify_attacker_stats` / ランク変化マネージャー内部処理）
- **`MoveData.target` は `"self"` を明示的に指定する必要がある。** 本フレームワークでは
  `MoveExecutor.run_move` が `defender = battle.foe(attacker)`（常に相手）としてコンテキストを
  構築するため、`target` を既定値の `"foe"` のままにすると、実際には自分にしか効果がない本技が、
  以下の理由で相手の状態によって誤って無効化される:
  - `check_hit_substitute`（`ON_BEFORE_APPLY_MOVE` priority 30 の `みがわり_immune`）が
    `ctx.move.target == "foe"` を条件に相手のみがわり状態を参照し、変化技を無効化してしまう
  - ちくでん・ひらいしん・でんきエンジン等の「でんきタイプ吸収」特性ハンドラ
    （`handlers/ability.py` の `_apply_type_absorb` 系）も `ctx.move.target == "foe"` を条件に
    発動判定するため、相手がこれらの特性を持つ場合に技が誤って無効化されてしまう
  - `target="self"` を指定すると `check_hit_substitute` が早期に `False` を返し、上記の
    誤発動を防げる（実装時に手動テストで確認済み）
- 対象が「ちくでん」「ひらいしん」「でんきエンジン」の場合の無効化は、上記の通り `target="self"`
  指定によって発動条件自体が満たされなくなるため、ハンドラ側に追加ガードは不要
