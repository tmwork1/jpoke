# 計画書: エレクトロビーム

## 仕様要約

出典: `.internal/spec/moves/エレクトロビーム.md`, `src/jpoke/data/ps-champ-ja/moves.json`

- タイプ: でんき / 分類: 特殊 / 威力: 130 / 命中: 100 / PP: 12
- 接触: × / まもる: ○
- **1ターン目**: 100% の確率で自分の「とくこう」を1段階上げ、溜め状態に入る（追加効果でない）
- **2ターン目**: 攻撃する
- **天気があめ/おおあめ**: 溜めをスキップして即攻撃（とくこう上昇は発動する）
- **パワフルハーブ**: 溜めをスキップして即攻撃（とくこう上昇は発動する）。ただし天気が
  あめ/おおあめでもある場合は天気が優先され、パワフルハーブは消費されない
- **ばんのうがさ**: あめの恩恵を受けず、通常通り溜める

### メテオビームとの比較

| 技 | SpAtk 変化 | 天気スキップ条件 | 天気による威力変化 |
|---|---|---|---|
| エレクトロビーム | +1（必ず） | あめ / おおあめ | なし |
| メテオビーム | +1（必ず） | なし | なし |
| ソーラービーム | なし | にほんばれ / おおひでり | あめ/すなあらし/ゆきで半減 |

## ハンドラ構成（実装済み）

### move_attack.py ハンドラ（`ON_MOVE_CHARGE` に登録）

| 関数名 | Priority | subject_spec | 役割 |
|---|---|---|---|
| `エレクトロビーム_boost_spa` | 50 | `attacker:self` | とくこうを1段階上げる（追加効果でない、必ず発動） |
| `エレクトロビーム_weather_skip` | 90 | `attacker:self` | あめ/おおあめ時に溜めをスキップして攻撃に進む（`stop_event=True`） |
| `エレクトロビーム_charge` | 100（デフォルト） | `attacker:self` | 揮発状態なし: 揮発付与して停止 / 揮発あり(2ターン目): 通過 |

3ハンドラ構成にした理由（旧計画からの変更点）:
- 旧計画では「天気による溜めスキップ」を `エレクトロビーム_charge` 内に埋め込む2ハンドラ構成
  だったが、これだとパワフルハーブ（`ItemHandler` デフォルト priority=100）より後（同一
  priority 100 かつ登録順で item が先）に判定されるため、あめ下でパワフルハーブと併用した際に
  天気の判定より先にパワフルハーブが消費されてしまい、`.internal/spec/moves/エレクトロビーム.md`
  の「天気があめの場合はパワフルハーブが優先されず消費されない」という仕様に反する。
- そのため天気判定を独立ハンドラ `エレクトロビーム_weather_skip`（priority=90）として切り出し、
  パワフルハーブより先に判定・`stop_event=True` で以降のハンドラ（パワフルハーブ含む）を
  停止させることで、天気優先の仕様を正しく満たす（ソーラービーム/ソーラーブレードと同一パターン）。

### volatile.py ハンドラ（揮発状態 "エレクトロビーム" に登録）

| 関数名 | Event | subject_spec | 役割 |
|---|---|---|---|
| `force_command`（共用） | `ON_MODIFY_COMMAND_OPTIONS` | `source:self` | 同技を強制選択させる |
| `エレクトロビーム_remove_volatile` | `ON_HIT` | `attacker:self` | 攻撃命中後に揮発状態を解除する |

### lethal.py ハンドラ（`LethalEvent.ON_BEFORE_MOVE` に登録）

| 関数名 | subject_spec | 役割 |
|---|---|---|
| `エレクトロビーム_boost_spa` | なし（`ctx.attacker` を直接参照） | `ctx.move_secondary` が True のときのみ攻撃側のとくこうを+1する（メテオビームと同一パターン） |

## Priority 根拠

- **`ON_MOVE_CHARGE` priority 50 (boost_spa)**: `.internal/spec/turn.md` の `ON_MOVE_CHARGE` テーブルにはエレクトロビームの記載がないが、同じく「溜め中に自分のとくこう+1」を持つメテオビームの実装が priority=50 (`data/moves/move_ma.py`) を採用している。パワフルハーブが `ON_MOVE_CHARGE` 内でデフォルト priority=100 で処理されるより前に boost を確実に実行するため、同じ priority=50 を採用する。
- **`ON_MOVE_CHARGE` priority 90 (weather_skip)**: `.internal/spec/turn.md` に記載がないため、同じ天気スキップ機構を持つソーラービーム/ソーラーブレードの実装（`ソーラービーム系_weather_skip` priority=90）を踏襲する。パワフルハーブ（`ItemHandler` デフォルト priority=100）より先に判定することで、天気による自動スキップがパワフルハーブの消費に優先されるようにする。
- **`ON_MOVE_CHARGE` priority 100 (charge, デフォルト)**: `.internal/spec/turn.md` の `ON_MOVE_CHARGE` テーブルで `| 100 | メテオビーム: とくこう上昇 |`（ソーラービームや隠れる技も同 100）と記載。溜め系技の状態変化は priority 100 が標準。
- **`ON_HIT` volatile 解除**: ソーラービーム/メテオビームと同じ priority デフォルト（100）を踏襲。みがわりに被弾しても `ON_HIT` は発動するため、揮発解除は確実に行われる。
- **`LethalEvent.ON_BEFORE_MOVE` (lethal boost_spa)**: メテオビームの lethal 実装と同一イベントを踏襲。

## 実装済みコード概要（参照用）

### `src/jpoke/handlers/move_attack.py`

```python
def エレクトロビーム_boost_spa(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """エレクトロビーム: とくこうを1段階上げる（追加効果ではないため必ず発動）。"""
    battle.modify_stats(ctx.attacker, {"spa": 1}, source=ctx.attacker)
    return HandlerReturn(value=value)


def エレクトロビーム_charge(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """エレクトロビーム: 1ターン目に溜め状態へ移行、2ターン目は通過する。"""
    attacker = ctx.attacker
    if not attacker.has_volatile("エレクトロビーム"):
        battle.volatile_manager.apply(attacker, "エレクトロビーム", count=1, source=attacker)
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def エレクトロビーム_weather_skip(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """エレクトロビーム: あめ/おおあめ下では溜めずにそのターンに攻撃する。"""
    attacker = ctx.attacker
    if not attacker.has_volatile("エレクトロビーム") and battle.weather_for(attacker).rainy:
        return HandlerReturn(value=value, stop_event=True)
    return HandlerReturn(value=value)
```

def エレクトロビーム_remove_volatile(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:

## 注意点・エッジケース（review-test 向け・確認済み）

1. **とくこう上昇は追加効果でない**: `battle.resolve_secondary_chance` を呼ばず、`battle.modify_stats` を直接呼ぶ（メテオビームと同じパターン）。実装済み・テスト済み。

2. **ばんのうがさ持ちはあめの恩恵を受けない**: `battle.weather_for(attacker).rainy` を使うことで自動的にばんのうがさ判定が入る（`battle.weather.rainy` は使わない）。実装済み・テスト済み（`test_エレクトロビーム_ばんのうがさ持ちはあめでも2ターンかかる`）。

3. **パワフルハーブとの相互作用**: `エレクトロビーム_weather_skip`（priority=90）がパワフルハーブ（デフォルト priority=100）より先に判定され、あめ下では `stop_event=True` でパワフルハーブの消費を防ぐ。実装済み・テスト済み（`test_エレクトロビーム_あめとパワフルハーブ両方所持時は天候優先で消費されない` および `tests/items/test_item_ha.py` 側の同等テスト）。

4. **あめ + 2ターン目の挙動**: 1ターン目にあめでスキップした場合、揮発状態は付与されない。そのため「揮発状態あり」が2ターン目の判定に使われるのは通常（あめなし）の2ターン目のみ。テスト済み。

5. **PP数**: `src/jpoke/data/ps-champ-ja/moves.json` 準拠で pp=12（実装済み・解決済み。旧計画に記載されていた pp=10 との差異は修正済み）。

6. **ON_HIT での揮発解除**: ソーラービーム/メテオビームと同様に `ON_HIT` で揮発状態を解除する。これにより攻撃がみがわりに当たった場合でも解除される（`ON_HIT` は「みがわりに被弾しても発動」と `.internal/spec/turn.md` に記載）。実装済み・テスト済み。

7. **VolatileData の `forced=True`**: 溜め中は次ターンの行動が同技に強制される。`ON_MODIFY_COMMAND_OPTIONS` に `force_command` を登録することで実現（ソーラービーム/メテオビームと同一パターン）。実装済み。

8. **ちからずくとの関係**: Champions ではこの技に関してちからずくの特別な相互作用は発生しない（本家 SV との差異。`.internal/spec/moves/エレクトロビーム.md` 参照）。実装上もちからずく判定は不要であり、現状の実装は正しい。

9. **ノーてんき/エアロック/メガソーラー特性との相互作用**: ウィキには「使用者の特性がメガソーラーであるときは、場があめ状態であれば溜めずに使用できる」との記載があるが、これは天気に依存する溜め技（ソーラービーム系）向けの共通テンプレート記述であり、実際の `weather_for`/`ON_CHECK_WEATHER_ENABLED`/`ON_CHECK_WEATHER_IMMUNE` の汎用機構（`ノーてんき`/`エアロック`/`メガソーラー`/`ばんのうがさ` の各特性・アイテム側で実装済み）を通じて自動的に反映される。エレクトロビーム固有のハンドラを追加する必要はない。
