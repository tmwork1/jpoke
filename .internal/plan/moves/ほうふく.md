# 計画書: ほうふく

参照: .internal/spec/moves/ほうふく.md
参照: .internal/plan/moves/攻撃技_反撃系.md（カウンター・ミラーコート・ほうふく・メタルバーストの共通実装計画）

## 概要

* あく / 物理 / 威力 - / 命中 100 / PP 10 / 優先度 0 / 接触あり
* そのターンに最後に相手から受けた攻撃技のダメージの1.5倍（切り捨て）を固定ダメージとして相手に与える。
* ダメージを受けていない場合は失敗する。
* メタルバーストと発動条件・ダメージ計算式は同一（タイプ・接触判定のみ異なる）。

## 前提: Pokemon モデルのフィールド

`.internal/plan/moves/攻撃技_反撃系.md` で追加済みの以下のフィールドを利用する（`src/jpoke/model/pokemon.py`）。

```python
self.last_physical_damage_received: int = 0  # 今ターン受けた物理ダメージ合計
self.last_special_damage_received: int = 0   # 今ターン受けた特殊ダメージ合計
self.last_damage_received: int = 0           # 今ターン最後に受けたダメージ量
```

`reset_turn_state` でターンごとに0にリセットされ、`src/jpoke/core/move_executor.py` の
`_execute_hit` 内で `actual_damage` 確定後に defender 側へ記録される（全攻撃技共通の処理）。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_TRY_MOVE_1` | 30 | `attacker:self` | `last_damage_received <= 0` なら失敗（stop_event=True） |
| `ON_MODIFY_MOVE_DAMAGE` | 100（デフォルト） | `attacker:self` | `int(last_damage_received * 1.5)` を返す |

### Priority 根拠

* `ON_TRY_MOVE_1` priority=30: `.internal/spec/turn.md` の
  「30 | カウンター・ミラーコート: 適格ダメージ無」と同一カテゴリ
  （技固有の「受けたダメージの有無」チェック）のため priority=30 を採用する。
* `ON_MODIFY_MOVE_DAMAGE` はターン内イベント一覧（`.internal/spec/turn.md`）に記載がない
  ダメージ計算内部イベントのため、既存の同種ハンドラ（`がむしゃら`, `ちきゅうなげ`, `いのちがけ`,
  `メタルバースト`）に倣いデフォルト priority を採用する。

## 実装箇所（実装済み）

### `src/jpoke/handlers/move_attack.py`

```python
def ほうふく_check_can_use(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ほうふくの使用可否を判定する。

    そのターンダメージを受けていない場合は失敗する。
    """
    if ctx.attacker.last_damage_received <= 0:
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="ほうふく")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def ほうふく_modify_damage(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ほうふくの固定ダメージを計算する（最後に受けたダメージ × 1.5、切り捨て）。"""
    return HandlerReturn(value=int(ctx.attacker.last_damage_received * 1.5))
```

### `src/jpoke/data/moves/move_ha.py`

## タイプ無効化の扱い

* `src/jpoke/core/move_executor.py` の `_execute_move` は `ON_TRY_MOVE_1` 通過後、
  `_check_hit_by_type` でタイプ相性 0倍（無効）を判定し、無効な場合はダメージ計算前に
  技自体を失敗させる。そのため万一 0倍（無効）となる相手が存在する場合でも、
  `ほうふく_modify_damage` に到達する前に技が失敗し、固定ダメージが上書きされることはない
  （現行のタイプ相性表では、あくタイプの攻撃技が0倍になる防御タイプは存在しないため、
  実際にこの分岐が発生することはない）。
* 半減・等倍・弱点等の通常の相性倍率は `ON_MODIFY_MOVE_DAMAGE` で確定させる固定値には影響しない
  （`.internal/spec/moves/固定ダメージ.md` 7.2 と整合。フェアリータイプ相手でも半減されない）。

## リーサル計算

* 対象外（`n/a`）。攻撃側が受けたダメージに依存するため、防御側HP分布の事前計算要素がない。
