# 計画書: ドラゴンエナジー

参照仕様書: .internal/spec/moves/ドラゴンエナジー.md

## 仕様要約

* 使用者の残りHPが多いほど威力が大きくなる（威力 = floor(150 × 残りHP ÷ 最大HP)、最小値1）。
* しおふき・ふんかと全く同じ威力計算式（`.internal/spec/moves/_variable_power.md` 第3節）。
* PP は champions 換算で 8（base PP5 → 8。しおふき・ふんかと同一の変換規則）。

## 既存実装の確認結果

`src/jpoke/data/moves/move_ta.py` にはすでに `MOVES["ドラゴンエナジー"]` が定義されているが、
以下の2点が仕様と不一致だったため修正する。

1. `pp=5` になっていた。champions では base PP5 の技は PP8 に統一されるため、
   `pp=8` に修正する（`しおふき`・`ふんか`と同じ base PP5 技であり、`src/jpoke/data/ps-champ-ja/moves.json`
   でも両者とも PP8 として記載されている）。
2. `ON_CALC_POWER_MODIFIER` ハンドラとして `ha.しおふき_calc_power`（別の技の名前付き関数）を
   直接流用していた。効果自体は同一で計算結果に誤りはないが、CLAUDE.md の
   「固有効果のロジックは handlers/* に名前付き関数で実装し、data/*.py からその関数を登録する」
   という約束と、`ふんか`が`しおふき`と同じ計算式でも独自の名前付き関数
   （`ふんか_calc_power`）を持っている既存の慣例に合わせ、`ドラゴンエナジー_calc_power` を
   新規に定義して登録する（ロジックは`しおふき_calc_power`と完全に同一）。

## ハンドラ構成

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| ドラゴンエナジー | `Event.ON_CALC_POWER_MODIFIER` | デフォルト（100） | `attacker:self`（`MoveHandler` デフォルト） | 使用者の残りHP割合から技の実威力の modifier を算出して返す |

* `Event.ON_CALC_POWER_MODIFIER` は `.internal/spec/turn.md` に priority の記載がないため、
  既存の同種ハンドラ（`しおふき_calc_power`・`ふんか_calc_power`ほか、`ON_CALC_POWER_MODIFIER`
  を使う全ての `MoveHandler`）が明示的な priority 指定なし＝デフォルト値（100）で登録されている
  慣例に合わせ、今回もデフォルトのままとする。

```python
def ドラゴンエナジー_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """ドラゴンエナジー: 自分のHP比率に比例して威力が決まる（最大150）。しおふき・ふんかと同計算。

    ON_CALC_POWER_MODIFIER では modifier = floor(HP比率 × 4096) を返す。
    final_power = round_half_down(150 × modifier / 4096) = round_half_down(150 × HP比率)
    """
    modifier = int(ctx.attacker.hp * 4096 / ctx.attacker.max_hp)
    modifier = max(1, modifier)  # 威力が最低1になるよう保証
    return HandlerReturn(value=modifier)
```

```python
"ドラゴンエナジー": MoveData(
    type="ドラゴン",
    category="special",
    pp=8,
    power=150,
    accuracy=100,
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
            ha.ドラゴンエナジー_calc_power,
        ),
    }
),
```

## リーサル計算

`.internal/progress/move.md` の該当行は「リーサル実装: n/a」「リーサルテスト: n/a」のため対象外。
理由: 威力は攻撃側自身の現在HPにのみ依存し、相手（`defender`）のHP分布 `StateDist` を
直接左右する要素ではないため、`lethal.py` 側の専用ハンドラは不要。

## テスト

`tests/moves_attack/test_move_ta.py` に、`しおふき`・`ふんか`の既存テストに倣い以下を追加する。

* HP満タン時の威力が150になること
* HP半分のとき威力が150未満・1以上になること

## 注意点・エッジケース

* `src/jpoke/data/ps-champ-ja/moves.json` にドラゴンエナジー自体の記載はない
  （専用技を覚えるレジドラゴが現行の選出対象に含まれていないためと考えられる）。
  ただし README の対象範囲は「第9世代で実装されていない技は実装しない」であり、本技は
  第8世代・第9世代ともに実装済みの技のため実装対象内。PP は威力計算式が同一のしおふき・
  ふんか（ともに `move_list.txt` にPP8として記載）に合わせて8とする。
* しおふき・ふんかの実装（`pp=5`）およびそのspec/plan文書の記載はこのレビューの対象外のため
  変更しない。両技のPP不一致に気づいた場合は、それぞれのレビューエントリで別途修正する。
