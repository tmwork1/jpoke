# ほろびのうた 実装計画書

## 仕様要約

- ノーマルタイプ・変化技（PP=8。`src/jpoke/data/ps-champ-ja/moves.json` の値。第九世代本家のPP=5は
  チャンピオンズのPP4段階統一（8/12/16/20）によりPP=8に変更されている）
- **場に出ているすべてのポケモン**（使用者自身を含む）を「ほろびのうた」揮発性状態にする
- ほろびのうた状態は count=3 で付与され、ターン終了時にカウントを1ずつ減らす
- カウントが 0 になったターン終了時に対象ポケモンは強制ひんし（HP=0）になる
- **交代すると効果は消える**（揮発性状態なので switch_manager が `remove_all_volatiles` で自動解除）
- 失敗条件：**場の全員（自分+相手）がすでにほろびのうた状態**の場合のみ失敗する
  （turn.md に priority=130 で明記。「誰にも効果がない場合は失敗する」という一般的な
  ゲーム仕様に基づく。片方だけがすでに状態でも、もう片方に効果があれば技は成功する）
- 音技（`sound` ラベル）のため、みがわり状態の相手にも効果が発生する
- 命中判定なし（必中）
- まもる系（まもる・キングシールド等）には防がれない（仕様書より）。
  `MoveData.flags` に `"unprotectable"` を設定して `is_blocked_by_protect` を False にする
  （既存の `ないしょばなし`（音技かつ変化技）と同じパターン）
- すでにほろびのうた状態の対象にはカウントがリセットされない（別途キャンセル）

## 現状の実装状況

| コンポーネント | 状態 | 備考 |
|---|---|---|
| `data/volatile.py` の `"ほろびのうた"` VolatileData | 実装済み | `ON_TURN_END`（tick）・`ON_VOLATILE_END`（faint）登録済み |
| `handlers/volatile.py` の `ほろびのうた_tick_volatile` | 実装済み | `tick_volatile` ヘルパー呼び出し |
| `handlers/volatile.py` の `ほろびのうた_faint` | 実装済み | `battle.modify_hp(mon, v=-mon.hp)` でHP=0 |
| `data/moves/move_ha.py` の `"ほろびのうた"` MoveData | 実装済み | `ON_BEFORE_APPLY_MOVE` / `ON_STATUS_HIT` ハンドラ登録済み。`flags={"sound", "unprotectable"}` |
| `handlers/move_status.py` の `ほろびのうた_apply` / `ほろびのうた_can_apply` | 実装済み | 場の全ポケモンへの付与・失敗判定 |

## ハンドラ構成

| イベント | Priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_BEFORE_APPLY_MOVE` | 130 | `attacker:self`（MoveHandlerデフォルト） | 失敗チェック：場の全員（`battle.actives`）がすでにほろびのうた状態なら失敗 |
| `ON_STATUS_HIT` | 100（MoveHandlerデフォルト） | `attacker:self` | 場の全ポケモン（自分+相手）に揮発性状態を付与 |

## Priority 根拠

- `ON_BEFORE_APPLY_MOVE priority=130`：`.internal/spec/turn.md` の ON_TRY_MOVE_2 セクションに
  「ほろびのうた: すでにほろびのうた状態」が priority=130 として明記されている

  > | 130 | ほろびのうた: すでにほろびのうた状態 |

  ただし、turn.md の記載イベント名は `ON_TRY_MOVE_2` だが、既存の `あくび` 等の実装では
  「すでに揮発性状態なら失敗」チェックを `ON_BEFORE_APPLY_MOVE priority=130` として登録している
  パターンに倣う（参考: `あくび_can_apply` の `ON_BEFORE_APPLY_MOVE priority=130`）。

- `ON_STATUS_HIT priority=100`：MoveHandlerデフォルト値。既存の全変化技効果ハンドラが
  priority=100 を使用しており、ほろびのうたも同様（turn.md に記載なし）。

## 実装箇所

### handlers/move_status.py（五十音順: ほ行、ほたるびの直後）

```python
def ほろびのうた_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ほろびのうたの効果: 場の全ポケモンにほろびのうた状態を付与する（count=3）。

    使用者自身も対象になる。音技のためみがわりを貫通する。
    すでにほろびのうた状態のポケモンには付与されない（volatile_manager.apply が False を返す）。
    """
    triggered = False
    for mon in battle.actives:
        if battle.volatile_manager.apply(
            mon,
            "ほろびのうた",
            count=3,
            source=ctx.attacker,
        ):
            triggered = True
    return HandlerReturn(value=triggered)


def ほろびのうた_can_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ほろびのうたの失敗チェック: 全員がすでにほろびのうた状態なら失敗する。"""
    if all(mon.has_volatile("ほろびのうた") for mon in battle.actives):
        battle.add_event_log(
            ctx.attacker,
            LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="ほろびのうた_すでに状態"),
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### data/moves/move_ha.py（既存の `"ほろびのうた"` MoveData にハンドラを追加）

```python
"ほろびのうた": MoveData(
```

## 注意点・エッジケース（review-test 向け）

1. **使用者自身も対象になる**
   仕様書の「場に出ている全てのポケモンを対象」に従い、`battle.actives` でループして
   自分を含む全員に付与する。`ctx.defender` のみへの付与では使用者に効果が出ない。

2. **すでにほろびのうた状態のポケモンはスキップされる**
   `volatile_manager.apply` は既に同じ揮発性状態があれば `False` を返す。
   ループ内での重複チェックは不要。

3. **ON_BEFORE_APPLY_MOVE の失敗チェックは場の全員（`battle.actives`）を確認する**
   「誰にも効果がない場合は失敗する」という一般的な仕様のため、`ctx.defender` のみの
   判定では不十分（使用者だけがすでに状態でも相手には付与できるため失敗にしてはならない）。
   `all(mon.has_volatile("ほろびのうた") for mon in battle.actives)` で
   場の全員がすでに状態である場合のみ失敗させる。片方だけがすでに状態の場合は技が成功し、
   まだ状態でない側にのみ付与される（`ほろびのうた_apply` 内の `volatile_manager.apply` が
   すでに付与済みの対象をスキップする）。

4. **音技（`sound` フラグ）によるみがわり貫通**
   `MoveData.flags={"sound", ...}` はすでに設定済み。`move_executor.check_hit_substitute`
   が `target == "foe"` かつ `sound` フラグを持つ技をみがわり判定なしとして扱うため
   （コア側の汎用ルール）、個別技のハンドラ側での追加実装は不要。

5. **まもる系を貫通する**
   仕様書より「まもる/ワイドガード等あらゆるガード状態を無視して有効」とある。
   `Move.is_blocked_by_protect` は `target == "foe" and not has_flag("unprotectable")` で
   判定されるため、`MoveData.flags` に `"unprotectable"` を追加することで
   `まもる_protect`（`_check_protect_success` 経由）が常に失敗しブロックされなくなる。
   既存の `ないしょばなし`（`flags={"sound", "unprotectable"}`）と同じパターン。

6. **交代による解除はハンドラ不要**
   `switch_manager._switch_out` が `remove_all_volatiles` を呼ぶため、
   ON_SWITCH_OUT ハンドラの追加は不要。

7. **のどスプレーとの連携**
   仕様書に「技が成功するとのどスプレーが発動するが、場にいる全てのポケモンが既に
   ほろびのうた状態の場合は発動しない」とある。
   `triggered = False` のまま `HandlerReturn(value=False)` を返せば
   技が成功したとみなされないため、のどスプレーが不当に発動しない。
   （triggered が False の場合は value=False を返す実装になっている）

8. **ひんし後のカウント消費タイミング**
   ターン終了時の `ON_TURN_END priority=120` で tick が行われ、count=0 になると
   `ON_VOLATILE_END` が発火し `ほろびのうた_faint` が `battle.modify_hp(mon, v=-mon.hp)` を実行する。
   この挙動は既実装の `data/volatile.py` / `handlers/volatile.py` で完結しており、
   技のハンドラ側での追加実装は不要。
