# 計画書: フェイント

## 仕様要約

- ノーマルタイプ / 物理 / 威力30 / 命中100 / PP12（Champions基準） / 対象: 1体選択 / 優先度+2 / 非接触
- まもる貫通（`unprotectable` フラグ）。命中時、対象の以下の揮発性状態を解除してから攻撃する:
  まもる・みきり・ニードルガード・トーチカ・キングシールド・スレッドトラップ・かえんのまもり・ファストガード
- まねっこ（Copycat）でコピー不可（`non_copycat` フラグ）
- 詳細は `.internal/spec/moves/フェイント.md` を参照

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| Event.ON_HIT | 100（`MoveHandler` デフォルト） | attacker:self | 命中時にdefenderのまもる系揮発性状態を全て解除する |

`Event.ON_HIT` は「ダメージ発生後の処理（みがわりに被弾しても発動）」であり、対象がみがわり状態でも
発動する仕様（`まもる・みがわり両方の状態でも解除される`）と一致する。
また、技が外れた場合・タイプ相性で無効化された場合・優先度技無効特性
（じょおうのいげん等）で無効化された場合は `ON_BEFORE_APPLY_MOVE` や `ON_TRY_MOVE_1` の時点で
処理が打ち切られ `ON_HIT` に到達しないため、専用の除外チェックは不要。

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_HIT` セクションに「ニードルガード等の効果」が priority=100 で
明記されている。フェイントの状態解除も同種の「命中後の固有効果」であるため、
`MoveHandler` のデフォルト優先度100（明示的指定なし）を採用する。

## 実装箇所

### handlers/move_attack.py への実装

`_FAINT_REMOVE_VOLATILES` タプルで解除対象の揮発性状態を列挙し、`いじげんホール_remove_protect` /
`いじげんラッシュ_remove_protect` と共通のタプルを使い回す（3技とも「まもる系統の状態を
解除して攻撃する」という同一効果を持つため）。

```python
_FAINT_REMOVE_VOLATILES: tuple[str, ...] = (
    "まもる", "みきり", "トーチカ", "キングシールド",
    "ニードルガード", "スレッドトラップ", "かえんのまもり", "ファストガード",
)


def フェイント_remove_protect(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """フェイントの追加効果: 相手のまもる系揮発性状態を解除する。"""
    defender = ctx.defender
    for volatile in _FAINT_REMOVE_VOLATILES:
        if defender.has_volatile(volatile):
            battle.volatile_manager.remove(defender, volatile)
            battle.add_event_log(
                defender,
                LogCode.VOLATILE_REMOVED,
                payload=VolatilePayload(volatile=volatile, display_reason="フェイント")
            )
    return HandlerReturn(value=value)
```

- ワイドガード・トリックガード・たたみがえし・ブロッキングは、シングルバトル専用の本プロジェクトでは
  無効果（ワイドガード）または未実装（トリックガード・たたみがえし・ブロッキング）のためタプルに含めない。

### data/moves/move_ha.py への更新

```python
"フェイント": MoveData(
    type="ノーマル",
    category="physical",
    pp=12,
    power=30,
    accuracy=100,
    priority=2,
    flags={"unprotectable", "non_copycat"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.フェイント_remove_protect,
        )
    }
),
```

- `pp=12` は `src/jpoke/data/ps-champ-ja/moves.json` 準拠（本家Gen5以降のPP10とは異なる）。
- `flags={"unprotectable"}` により `Move.is_blocked_by_protect` が False を返し、
  まもる状態のポケモンにも命中する。
- `flags` に `"non_copycat"` を追加し、まねっこ（Copycat）でのコピーを禁止する
  （`まねっこ_can_use` ハンドラが `non_copycat` フラグを持つ技のコピーを失敗させる）。

## リーサル計算

対象外（`.internal/progress/move.md` の該当行は「リーサル実装」「リーサルテスト」ともに `n/a`）。
フェイントは確率要素のない固定効果（まもる系解除＋通常ダメージ）のため、リーサル計算上は
通常の物理攻撃技と同様に扱われる。

## 注意点・エッジケース

### まもる状態がない場合

- 対象がまもる系統の状態でない場合は、通常のダメージ計算のみが行われる
  （`_FAINT_REMOVE_VOLATILES` のどの揮発性状態も持たないため、ループ内で何も解除されない）。

### タイプ相性・技外れ時は解除されない

- ゴーストタイプへのノーマル技は無効化されるため、`ON_BEFORE_APPLY_MOVE` で処理が打ち切られ
  `ON_HIT` に到達しない。よってまもる状態は解除されない（仕様通り）。
- 命中判定に失敗した場合も同様に `ON_HIT` に到達しない。

### みがわり状態への被弾でも解除される

- `Event.ON_HIT` はみがわりへの被弾でも発火するイベントであるため、対象がみがわり状態でも
  まもる系統の状態解除は正常に行われる（実装済み・追加対応不要）。

### まねっこ（Copycat）との相互作用

- フェイントは `non_copycat` フラグを持つため、まねっこで直前にフェイントが使われていても
  コピーできず失敗する。
