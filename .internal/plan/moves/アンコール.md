# 計画書: アンコール

出典: `.internal/spec/moves/アンコール.md`

## 仕様要約

- ノーマルタイプ / 変化 / 命中100 / PP8（champions 基準） / 対象: 相手1体
- 相手にアンコール揮発性状態を付与する（3ターン固定。Pokémon Champions 仕様）
- アンコール状態の相手は最後にPPを消費した技しか使えなくなる（詳細は `.internal/spec/volatiles/アンコール.md`）
- **相手を対象とする変化技のため `MoveData(target=...)` は省略時デフォルトの `"foe"` のままでよい**
  （まもる・マジックコートに正しく反応するために必要な設定であり、既定値のままで問題ない）
- **みがわりを貫通しない**ため `flags={"bypass_substitute"}` は付与しない
  （原作Wikiにみがわり貫通の記載がなく、`ちょうはつ`等とは異なり通常の対象foe変化技として扱う）
- 技「アンコール」自体は `non_encore` ラベルを持つ（`flags={"non_encore"}`）ため、
  アンコールをアンコールで固定することはできない

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_BEFORE_APPLY_MOVE` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手が未行動、または `non_encore` ラベルの技を最後に使っていれば失敗させる（`アンコール_can_apply`） |
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手にアンコール揮発性状態を付与する（count=3, move_name=相手の最後の技名）（`アンコール_apply`） |

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_APPLY_MOVE`（コード上は `Event.ON_BEFORE_APPLY_MOVE` に対応）に
「100 | アンコール: 失敗条件」と記載されている。
付与側の `ON_STATUS_HIT` は turn.md に個別記載がないため、既存の `apply_volatile_to_defender` を使う
他のハンドラ（`ちょうはつ_apply`、`いちゃもん_apply_volatile_to_defender`、`ロックオン_apply_volatile_to_defender`）
と同様に priority=100 デフォルトを採用する。

## 実装箇所

### handlers/move_status.py（実装済み）

挿入位置（五十音順）: `アロマセラピー_apply` の後、`いえき_apply` の前

```python
def アンコール_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """アンコールの効果を発動する。"""
    move = ctx.defender.executed_move
    return apply_volatile_to_defender(
        battle, ctx, value, volatile="アンコール", count=3, move_name=move.name
    )


def アンコール_can_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """アンコールの失敗条件を判定する。

    - 相手がまだ技を使っていない（executed_move が None）場合は失敗する
    - 相手の最後に使った技が non_encore ラベルを持つ場合は失敗する
    """
    move = ctx.defender.executed_move
    if not move or move.has_flag("non_encore"):
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="アンコール失敗")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース

- アンコール揮発性状態の持続ターン数（3 ターン）は `アンコール_apply` が `count=3` を明示的に渡している
- すでにアンコール状態の相手には失敗する（`volatile_manager.apply` が既存 volatile への重複付与を失敗として扱う）
- みがわり状態の相手には無効（`bypass_substitute` フラグを設定していないため `みがわり_immune` が処理）
- 揮発性状態「アンコール」自体のコマンド制限・技固定・解除条件・かなしばり等との重複挙動は
  `.internal/spec/volatiles/アンコール.md` / `.internal/plan/volatiles/アンコール.md` を参照
- **リーサル計算**: 対象外（ダメージを与えない変化技、n/a）
