# 計画書: おはかまいり（再レビュー）

## 仕様要約

* 基本威力50。その戦闘で味方がひんしになった数に応じて威力が50ずつ上がる
  （`.internal/spec/moves/おはかまいり.md` 参照）。
* 本プロジェクトはシングルバトル専用かつバトル中の復活手段が存在しないため、
  「控えポケモン（bench）のうち現在ひんしの数」＝「累計ひんし回数」として扱ってよい。
* 直接攻撃（接触技）ではない。

## 既存実装の状況

`おはかまいり` はすでに実装済み（`data/move.py`・`handlers/move_attack.py`・テスト3件）。
今回のレビューで以下の誤りを検出した。

### 1. `flags={"contact"}` の誤設定（バグ）

`.internal/wiki/moves/おはかまいり.html` の「直接攻撃」欄は `×`（接触技ではない）だが、
既存の `data/move.py` のエントリには `flags={"contact"}` が設定されていた。

このフラグは `core/query.py` の `is_contact()` / `is_contact_reaction()` が参照しており、
接触ありと誤判定されると、さめはだ・ゴツゴツメット・ほのおのからだ等の「相手の直接攻撃を
受けたときに反応する」効果が誤発動してしまう（実機ではおはかまいりを受けてもこれらは発動しない）。

同じくゴーストタイプの非接触物理技である `ポルターガイスト`（`data/move.py`）は `flags` を
指定していない（`MoveData.flags` のデフォルトは空集合）ため、それに倣い `flags` 自体を削除する。

### 2. `.internal/progress/move.md` のPP列が誤り

該当行のPP列が `12` になっていたが、一次情報・実装（`pp=10`）ともに `10` が正しいため修正する。

## 修正方針

* `data/move.py` の `おはかまいり` エントリから `flags={"contact"}` を削除する（`flags` 自体を省略）。
* `handlers/move_attack.py` の `おはかまいり_calc_power` はロジック自体に誤りがないため変更しない
  （下記「ハンドラ構成」に既存実装を再掲）。
* `.internal/progress/move.md` の `おはかまいり` 行のPP列を `12` → `10` に修正し、
  仕様書・計画書列を `x`、レビュー列を `✓` に更新する。

## ハンドラ構成（既存・変更なし）

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| おはかまいり | `Event.ON_CALC_POWER_MODIFIER` | デフォルト（100） | `attacker:self`（デフォルト） | 控えのひんし数に応じて威力modifierを掛ける |

`Event.ON_CALC_POWER_MODIFIER` は `.internal/spec/turn.md` に priority の掲載がないため、
同種の可変威力技（`けたぐり_calc_power`・`くさむすび_calc_power`・`じたばた_calc_power` 等）が
いずれも `MoveHandler` のデフォルト値（priority=100, subject_spec="attacker:self"）のまま
登録されていることを根拠に、既存実装のデフォルト設定を踏襲する。

```python
def おはかまいり_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """おはかまいり: ひんしになった味方の数に応じて威力が増加する。

    基本威力 50 に対して (1 + ひんし味方数) 倍の modifier を掛ける。
    ひんし味方数は控えポケモン（bench）のうちひんしのもの。
    場に出ている自分自身はひんしでないため bench に含まれない。
    """
    player = battle.get_player(ctx.attacker)
    state = battle.player_states[player]
    fainted_count = sum(1 for p in state.bench if p.fainted)
    return HandlerReturn(value=apply_fixed_modifier(value, 4096 * (1 + fainted_count)))
```

```python
"おはかまいり": MoveData(
    type="ゴースト",
    category="physical",
    pp=10,
    power=50,
    accuracy=100,
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
            ha.おはかまいり_calc_power,
        ),
    }
),
```

## 注意点・エッジケース

* 本プロジェクトにバトル中の復活手段が存在しない前提のため、
  「復活してもカウントは減らない」「累計100回以上で最大威力5050」という一次情報の記述は
  実質的に到達不可能な仕様であり、追加実装は不要（`.internal/spec/moves/おはかまいり.md` に明記済み）。
* マルチバトル・テラレイドバトルの除外規定は、本プロジェクトがシングルバトル専用のため対象外。
