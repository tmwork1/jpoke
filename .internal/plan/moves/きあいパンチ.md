# きあいパンチ 実装計画書

## 仕様要約（.internal/spec/moves/きあいパンチ.md 参照）

* タイプ: かくとう / 分類: 物理 / 威力: 150 / 命中: 100 / PP: 20
* 対象: 相手1体 / 優先度: -3 / 接触あり（直接攻撃）
* 効果: 行動前（技を出すまでの間）に攻撃技によるダメージを受けていた場合、技が失敗する。
  第5世代以降の仕様では、不発時にPPを消費しない。
* パンチ技（`flags` に `"punch"` を含む）。特性『てつのこぶし』・アイテム『パンチグローブ』の対象。
* `ねごと`（`non_negoto`）で選ばれない。

## 現状の実装状況

`src/jpoke/data/move.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。

```python
"きあいパンチ": MoveData(
    type="かくとう",
    category="physical",
    pp=20,
    power=150,
    accuracy=100,
    priority=-3,
    flags={"contact", "non_negoto", "punch"},
    handlers={
        Event.ON_MODIFY_PP_CONSUMED: h.MoveHandler(
            ha.きあいパンチ_suppress_pp_on_fail,
            subject_spec="attacker:self",
        ),
        Event.ON_TRY_MOVE_1: h.MoveHandler(
            ha.きあいパンチ_check_move,
            subject_spec="attacker:self",
            priority=30,
        ),
    },
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_MODIFY_PP_CONSUMED` | 100（デフォルト） | attacker:self | 不発する場合はPP消費量を0にする |
| `ON_TRY_MOVE_1` | 30 | attacker:self | 使用者がそのターン中に既に攻撃技のダメージを受けていれば失敗させる |

```python
def きあいパンチ_check_move(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """きあいパンチの発動可否を判定する。

    行動前に実際の攻撃ダメージを受けていた場合は不発になる。
    """
    if ctx.attacker.hits_taken > 0:
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="きあいパンチ")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def きあいパンチ_suppress_pp_on_fail(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """きあいパンチ: 不発する場合はPPを消費しない（第五世代以降の仕様）。"""
    if ctx.attacker.hits_taken > 0:
        return HandlerReturn(value=0)
    return HandlerReturn(value=value)
```

`Pokemon.hits_taken` は `core/move_executor.py` の `_execute_hit` 内で、攻撃技のヒットにより実際に
HPが減少した（`actual_damage > 0`）場合にのみインクリメントされ、`reset_turn_state()` によって
ターン開始時に0へリセットされる。これにより「そのターン中に既に攻撃技のダメージを受けたか」を
判定できる。

`_consume_pp`（`core/move_executor.py`）は `ON_TRY_MOVE_1` より前（`ON_TRY_ACTION` 成功直後）に
呼ばれるため、`きあいパンチ_check_move` の失敗判定だけではPP消費を止められない。
`ON_MODIFY_PP_CONSUMED` で同じ条件（`hits_taken > 0`）を先読みして消費量を0にすることで、
「不発時にPPを消費しない」という第5世代以降の仕様を実現する。

## Priority 根拠

* `ON_TRY_MOVE_1`: `.internal/spec/turn.md` には本技固有の行は掲載されていないが、
  「そのターンに既に生じた別の行動の結果（受けたダメージの有無）に基づいて技の成否を判定する」
  という点で、同種の判定を行う既存ハンドラ（カウンター・ミラーコートの `_can_use` 系、
  priority=30）と同じ性質を持つ。よって priority=30 に統一する
  （`ON_TRY_MOVE_1` 表内の「30 | カウンター・ミラーコート: 適格ダメージ無」と同じ区分）。
* `ON_MODIFY_PP_CONSUMED`: `.internal/spec/turn.md` に専用の記載がないイベントだが、本技の
  PP消費ハンドラは他に競合するハンドラが存在しないため、既存の同種ハンドラ
  （`ねごと_suppress_pp`）に合わせてデフォルト値（100）のままとする。

## 実装箇所

### handlers/move_attack.py（修正）

1. `きあいパンチ_suppress_pp_on_fail` を新規追加。`ON_MODIFY_PP_CONSUMED` に登録し、
   不発条件（`hits_taken > 0`）を満たす場合はPP消費量を0にする。
2. `みねうち_modify_damage`: 対象のHPがすでに1でダメージが0になるケースで
   `ctx.defender.hits_taken` を増加させていなかったため、`ctx.defender.hits_taken += 1` を追加した。
   実ダメージが0でも「攻撃を無効化した」わけではなく、きあいパンチの不発条件を満たすため。

`きあいパンチ_check_move` 自体はロジック修正不要（priority のみ `data/move.py` 側で明示化）。

### handlers/ability.py（修正）

`ばけのかわ_block_damage`（Mimikyu の特性）は、攻撃ダメージを0にして肩代わりする際に
`Pokemon.hits_taken` を増加させていなかったため、きあいパンチの使用者がばけのかわで
攻撃を肩代わりしても不発しないという仕様との不整合があった。
一次情報（きあいパンチ Wiki「集中状態について」節）に「HP1のときにみねうちを受けたときや、
ばけのかわでダメージを肩代わりしたときはダメージを無効化したことにはならず、きあいパンチは
不発する」と明記されているため、`ばけのかわ_block_damage` 内で `ctx.defender.hits_taken += 1`
するよう修正した。

### data/move.py（修正）

* `ON_TRY_MOVE_1` ハンドラに `priority=30` を明示的に指定した（デフォルト値100からの変更）。
* `ON_MODIFY_PP_CONSUMED` ハンドラを新規追加し、`きあいパンチ_suppress_pp_on_fail` を登録した。

## 注意点・エッジケース（review-test 向け）

1. **みがわりへの被弾**
   - みがわりが身代わりでダメージを受けた場合、使用者自身はダメージを受けていない扱いとなり
     不発しない（`みがわり_block_damage` は `hits_taken` を増加させない。既存仕様どおり）。
2. **HP1でのみねうち／ばけのかわでの肩代わり**
   - 実ダメージが0でも「無効化」には該当しないため不発する（上記の handlers 修正で対応）。
3. **一撃必殺技をきあいのタスキ等で耐えた場合（第5世代以降）**
   - 実機では一撃必殺技をきあいのタスキ／きあいのハチマキ／ばけのかわで耐えた場合に限り、
     きあいパンチは不発しないという例外がある（Wiki「集中状態について＞第五世代以降」節）。
   - この例外は一撃必殺技＋耐え系アイテム／特性＋同ターンのきあいパンチという極めて稀な組み合わせ
     でのみ関係し、一般的なばけのかわ／みねうちの挙動（上記2.）と矛盾する特殊な例外仕様のため、
     実装の複雑化・副作用のリスクに対して効果が小さいと判断し、本実装では対応しない
     （既知の未実装エッジケースとして本ファイルに明記する）。
4. **不発時のPP消費**
   - 第5世代以降の仕様に合わせ、不発時はPPを消費しないよう `ON_MODIFY_PP_CONSUMED` で対応した
     （上記実装箇所を参照）。
5. **テスト方針**
   - `test_きあいパンチ_攻撃ダメージを受けると失敗` / `test_きあいパンチ_行動前にダメージを受けず成功` /
     `test_きあいパンチ_みがわりへの被弾では中断しない` に加え、
     `test_きあいパンチ_HP1でみねうちを受けると失敗` / `test_きあいパンチ_ばけのかわで肩代わりすると失敗` /
     `test_きあいパンチ_不発時はPPを消費しない` を追加する。
