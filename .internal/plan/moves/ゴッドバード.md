## 仕様要約

- ひこうタイプ・物理・威力 140・命中率 90・PP 8（`src/jpoke/data/ps-champ-ja/moves.json` 準拠。base game の PP 5 はチャンピオンズ移行時の統一ルール（8/12/16/20）により 8 に変更されている）
- 急所ランク+1（`critical_rank=1`）
- 直接攻撃 ×（非接触技）
- 2ターン技
  - 1ターン目: 溜め（`ゴッドバード` 揮発状態が付与される）。攻撃・追加効果・急所判定は行わない。天候等によるスキップ条件は無い（ソーラービームと異なり、必ず1ターン溜める）。
  - 2ターン目: 相手を攻撃する。命中判定・急所判定・追加効果はすべてこのターンにのみ発生する。攻撃終了後に揮発状態が解除される。
- 追加効果として、30%の確率で相手をひるませる（`flags={"secondary_effect"}`）
- あなをほる・そらをとぶ・ダイビングと異なり、姿を隠す（半透明化）効果は無い。1ターン目も相手からの通常攻撃を回避しない。したがって:
  - 交代禁止（トラップ）は無い
  - 相手の特定技のみ命中するような回避判定は無い
  - 威力を2倍にするような特殊技の存在も無い
- まもる: ○（2ターン目のみ判定）
- おうじゃのしるし・するどいキバの追加ひるみ効果は重複しない（第五世代以降の仕様。本プロジェクトは既に `_INNATE_FLINCH_MOVES` に登録済み）
- 使用者の特性ちからずくの影響を受けると威力が上がるがひるみ効果は無効化される。1ターン溜める効果・急所に当たりやすい効果は変わらない（`secondary_effect` フラグ経由で既存の ちからずく 実装が汎用対応済み）

## ハンドラ構成

### `data/move.py`（技データ）

| イベント | priority | subject_spec | ハンドラ関数 | 役割 |
|---------|----------|-------------|------------|------|
| `ON_MOVE_CHARGE` | 100（デフォルト） | — | `lambda b,c,v: h.charge_into_volatile(b,c,v,"ゴッドバード")` | 1ターン目に揮発状態を付与して技を停止する |
| `ON_DAMAGE_HIT` | 100（デフォルト） | — | `ゴッドバード_apply_flinch` | 2ターン目のダメージ命中時に30%でひるみを付与する |

### `data/volatile.py`（揮発状態データ）

| イベント | priority | subject_spec | ハンドラ関数 | 役割 |
|---------|----------|-------------|------------|------|
| `ON_MODIFY_COMMAND_OPTIONS` | 100（デフォルト） | `source:self` | `force_command` | 2ターン目のコマンドを FORCED に強制 |
| `ON_HIT` | 100（デフォルト） | `attacker:self` | `ゴッドバード_remove_volatile` | 攻撃ターン終了後に揮発状態を解除 |

あなをほる・そらをとぶ・ダイビングと異なり、`ON_CHECK_TRAPPED`（交代禁止）・`ON_CALC_POWER_MODIFIER`（特定技の威力2倍）・`ON_TRY_MOVE_1`（隠れ技の回避判定）は登録しない。ゴッドバードは姿を隠さないため。この点は `ソーラービーム`/`ソーラーブレード` の揮発状態（`ON_MODIFY_COMMAND_OPTIONS` と `ON_HIT` のみ）と同型。

## Priority 根拠

| イベント | Priority | 根拠 |
|---------|----------|------|
| `ON_MOVE_CHARGE` | 100（デフォルト） | `.internal/spec/turn.md` に「隠れる技: 状態変化」priority=100 の記載はあるが、ゴッドバードは隠れる技ではない。既存の非隠れ系2ターン技（ソーラービーム・ソーラーブレード）も明示priorityを指定せずデフォルト100で実装されており、それに合わせる |
| `ON_DAMAGE_HIT`（ひるみ） | 100（デフォルト） | `.internal/spec/turn.md` に `ON_DAMAGE_HIT` の記載は無いが、既存の同種ひるみハンドラ（かみつく・いわなだれ・つららおとし等、すべて `_INNATE_FLINCH_MOVES` 内）が全てデフォルト priority=100 で実装されているため、それに合わせる |
| `ON_MODIFY_COMMAND_OPTIONS` | 100（デフォルト） | 既存の `force_command` 利用箇所（あなをほる・そらをとぶ・ダイビング・ソーラービーム・ソーラーブレード）と同じデフォルト priority=100 を採用 |
| `ON_HIT`（揮発状態解除） | 100（デフォルト） | `.internal/spec/turn.md` Event.ON_HIT に掲載なし。既存の同種ハンドラ（ソーラービーム_remove_volatile 等）の priority=100（デフォルト）を根拠として採用 |

## 実装箇所

### `data/moves/move_ka.py`

```python
"ゴッドバード": MoveData(
    type="ひこう",
    category="physical",
    pp=8,
    power=140,
    accuracy=90,
    critical_rank=1,
    flags={"secondary_effect"},
    handlers={
        Event.ON_MOVE_CHARGE: h.MoveHandler(
            lambda b, c, v: h.charge_into_volatile(b, c, v, "ゴッドバード"),
        ),
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ゴッドバード_apply_flinch,
        )
    }
),
```

### `data/volatile.py`

```python
"ゴッドバード": VolatileData(
    forced=True,
    handlers={
        Event.ON_MODIFY_COMMAND_OPTIONS: h.VolatileHandler(
            h.force_command,
            subject_spec="source:self",
        ),
        Event.ON_HIT: h.VolatileHandler(
            h.ゴッドバード_remove_volatile,
            subject_spec="attacker:self",
        ),
    }
),
```

### `handlers/volatile.py`

```python
def ゴッドバード_remove_volatile(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """ゴッドバード溜め状態の解除"""
    return remove_volatile(battle, ctx, value, volatile="ゴッドバード")
```

### `types/volatile.py`

`python scripts/generate_literals/generate_volatile_literal.py` を実行して `VolatileName` に `"ゴッドバード"` を自動反映する（`data/volatile.py` の `VOLATILES` 辞書定義順から生成されるため、手動編集は不要）。

## 注意点・エッジケース（review-test 向け）

### あなをほる系との違い

- ゴッドバードは「姿を隠す」効果を持たないため、1ターン目でも通常の攻撃を受ける（回避しない）。したがって `ON_TRY_MOVE_1` による命中判定上書きは実装しない。
- 交代禁止（トラップ）も発生しない。`ON_CHECK_TRAPPED` は登録しない。

### ソーラービーム系との違い

- ソーラービームは「にほんばれ/おおひでり」下では1ターン目を省略できる（`ソーラービーム_weather_skip`）。ゴッドバードには天候によるスキップ条件が存在しないため、天候チェックハンドラは実装しない。
- ソーラービームは天候（あめ・おおあめ・すなあらし・ゆき）で威力半減するが、ゴッドバードにはそのような威力変化条件は無い。

### 急所ランク・ちからずくとの関係

- `critical_rank=1` は `MoveData` の標準フィールドであり、ちからずく等の特性による威力・追加効果の変化とは独立している。ちからずくが `secondary_effect` フラグを見て追加効果（ひるみ）のみを無効化する一方、`critical_rank` は変化しない（既存実装のまま、変更不要）。

### PP について

- `src/jpoke/data/ps-champ-ja/moves.json` の値（PP 8）を正とする。base game の PP 5 は本プロジェクトの対象外（ポケモンチャンピオンズ シングルバトルのみ対象）。
