# バブルこうせん 実装計画書

## 仕様要約（.internal/spec/moves/バブルこうせん.md 参照）

* タイプ: みず / 分類: 特殊 / 威力: 65 / 命中: 100 / PP: 20
* 対象: 相手1体 / 優先度: 0 / 非接触
* 効果: 追加効果として、10%の確率で相手の『すばやさ』ランクを1段階下げる。

## 現状の実装状況

`src/jpoke/data/moves/move_ha.py` に既に登録済み。基本データ（type/category/pp/power/accuracy）・
`flags={"secondary_effect"}`・ハンドラ登録のいずれも仕様と一致しており、修正は不要。

```python
"バブルこうせん": MoveData(
    type="みず",
    category="special",
    pp=20,
    power=65,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.バブルこうせん_lower_defender_spd,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手の『すばやさ』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def バブルこうせん_lower_defender_spd(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spe": -1}, chance=0.1)
```

`modify_defender_stats`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は `flags={"secondary_effect"}` さえ
付与されていれば汎用的に処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（能力ランク低下）は既存の同種ハンドラ（オーロラビーム・じゃれつく・
うらみつらみ・はるのあらし・ひやみず・とびかかる 等）と同じ `ON_DAMAGE_HIT` / priority 100
（デフォルト）のパターンに従う。`ON_DAMAGE_HIT` は `.internal/spec/turn.md` に未掲載のイベント
（ダメージ計算内部）だが、既存の同種ハンドラの実装パターンを踏襲するため個別の priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` が付与されているため、ちからずく所持時は威力1.3倍（65→84）になり、
     すばやさ低下の追加効果は発動しなくなる。
2. **テスト方針**
   - `secondary_chance` を0/1に固定し、すばやさランクの変化有無を確認するテストを追加する
     （`test_バブルこうせん_相手にダメージを与える` / `test_バブルこうせん_すばやさ1段階低下が発動する` /
     `test_バブルこうせん_すばやさ1段階低下が発動しない`）。
