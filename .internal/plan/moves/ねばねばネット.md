# 実装計画書: ねばねばネット

## 仕様要約

- 仕様書: `.internal/spec/moves/ねばねばネット.md`
- タイプ: むし / カテゴリ: 変化 / PP: 20 / 対象: 相手の場（target="foe_side"）
- 使用後、相手はポケモンを交代する度に、出てきたポケモンの「すばやさ」ランクが 1 段階下がる
- ひこうタイプや特性「ふゆう」のポケモンは影響を受けない（詳細は `.internal/spec/fields/ねばねばネット.md`）
- 永続効果（ターン制限なし、count=1 で登録）
- サイドフィールド側の `ON_SWITCH_IN` priority は turn.md に「ねばねばネットによる素早さ低下」priority 100 として記載されている（フィールド側の実装計画は `.internal/plan/fields/ねばねばネット.md` を参照）

## Priority 根拠

`Event.ON_STATUS_HIT` は `.internal/spec/turn.md` に掲載されていない技内部イベントのため、同種の設置技ハンドラ（`ステルスロック_set_field`・`どくびし_set_field`）が `MoveHandler` のデフォルト priority（100）を使用しているのと同一パターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: `ねごと_suppress_pp` の後、`ねむりごな_apply_ailment_to_defender` の前（実装済み）。

```python
def ねばねばネット_set_side_field(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ねばねばネット: 相手陣営に「ねばねばネット」を設定する（永続）。"""
    side = battle.get_side(ctx.defender)
    if not side.activate("ねばねばネット", 1):
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/moves/move_na.py`

既存の `"ねばねばネット"` エントリ（実装済み）。

```python
"ねばねばネット": MoveData(
    type="むし",
    category="status",
    pp=20,
    target="foe_side",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ねばねばネット_set_side_field,
        ),
    }
),
```

## 注意点・エッジケース

- `ステルスロック_set_field` と同じく `ctx.defender` の陣営を使う（相手側へ設置）
- `side.activate("ねばねばネット", 1)` の count=1 は「永続」を意味する（ステルスロックと同様）
- ひこうタイプ・ふゆう特性の判定はサイドフィールドの ON_SWITCH_IN ハンドラ側で処理されるため、技ハンドラ側でのガードは不要
- おいかぜ・リフレクター・ひかりのかべ・しんぴのまもりは自陣営（ctx.attacker）を使うが、ねばねばネットは相手陣営（ctx.defender）を使う点に注意
- 技ハンドラ内の `ctx` の型は `AttackContext`（`EventContext` ではない）。技ハンドラは常に `AttackContext` を受け取る
