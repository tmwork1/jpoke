# ゴールドラッシュ 実装計画書

## 仕様要約（.internal/spec/moves/ゴールドラッシュ.md 参照）

* タイプ: はがね / 分類: 特殊 / 威力: 120 / 命中: 95（Champions基準） / PP: 8（チャンピオンズ基準）
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果: 命中して技が成功した後、自分の『とくこう』ランクが2段階下がる（Champions基準、確定効果）。

## 現状の実装状況（レビュー時点）

`src/jpoke/data/moves/move_ka.py` に登録済みだが、以下の誤りがあった。

1. `pp=5` — Gen9本家基準（SV）のPPのまま。Championsの8/12/16/20統一ルールに基づき
   `pp=8` に修正する（同系統のオーバーヒート・りゅうせいぐんと同じ変換）。
2. `accuracy=100` — SV基準のまま。Championsでの下降修正（100→95）が未反映。`accuracy=95` に修正する。
3. `flags={"secondary_effect"}` — 自分自身への確定ランク低下は「追加効果」ではないため、
   ちからずく等の対象にしてはならない。オーバーヒート・りゅうせいぐんに倣い `flags` を外す
   （`secondary_effect` を持たせない）。
4. `handlers/move_attack.py` の `ゴールドラッシュ_lower_spa_C` が `stats={"spa": -1}` になっている。
   ChampionsではSVの1段階から2段階に変更されているため `stats={"spa": -2}` に修正する。
   同系統の命名規則（`りゅうせいぐん_sharply_lower_spa_C`）に合わせ、関数名も
   `ゴールドラッシュ_sharply_lower_spa_C` に変更する。
5. `lethal_handlers` が未登録。リーサル実装列は `-`（未実装）のため新規実装する。

修正後の登録（`src/jpoke/data/moves/move_ka.py`）:

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 命中後、自分の『とくこう』ランクを2段階下げる（確定効果） |

`src/jpoke/handlers/move_attack.py` の実装（修正後）:

```python
def ゴールドラッシュ_sharply_lower_spa_C(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"spa": -2})
```

`modify_attacker_stats`（`handlers/move.py`）の `chance` 既定値は1のため常に発動する。
自分自身への確定効果であり、`resolve_secondary_chance` 経由の「追加効果」判定
（ちからずく等）の対象にはならない（`flags` に `secondary_effect` を付与しない）。

## Priority 根拠

自分自身への確定ランク低下は既存の同種ハンドラ（オーバーヒート・りゅうせいぐん等）と同じ
`ON_HIT` / priority 100（デフォルト）のパターンに従う。`ON_HIT` は
`.internal/spec/turn.md` に個別掲載されていないが、既存の同種ハンドラ（`オーバーヒート_lower_attacker_spa`
等、命中直後に自分のランクを下げる実装）を参照して決定した。

## リーサル計算

`.internal/progress/move.md` のリーサル実装列は `-`（未実装）のため新規実装する。

`src/jpoke/handlers/lethal.py` に以下を追加する（`オーバーヒート_lower_attacker_spa` /
`りゅうせいぐん_lower_spa` と同じパターン）。

```python
def ゴールドラッシュ_lower_spa(battle: Battle, ctx: LethalContext, hp_dist: StateDist) -> StateDist:
    """ゴールドラッシュ: 命中後、攻撃側のとくこうを2段階下げる。"""
    ctx.attacker.rank["spa"] = clamp_stats(ctx.attacker.rank["spa"] - 2)
    return hp_dist
```

`data/moves/move_ka.py` の `ゴールドラッシュ` エントリに
`lethal_handlers={LethalEvent.ON_HIT: LethalHandler(l.ゴールドラッシュ_lower_spa)}` を追加する。

## 注意点・エッジケース（review-test 向け）

1. **ちからずく等「追加効果」系特性・アイテムとの非該当**
   - 自分自身への確定ランク低下は「追加効果」ではないため、ちからずく・てんのめぐみ
     （`secondary_effect` フラグを参照する特性）の対象にならない。`flags` から
     `secondary_effect` を除去する。
2. **まもる貫通なし**
   - 通常の攻撃技であり、`まもる` 等の対象になる。個別のガードは不要
     （`unprotectable` フラグ等は付与しない）。
3. **PPソースの欠落**
   - `src/jpoke/data/ps-champ-ja/moves.json` に `ゴールドラッシュ` 自体の行が存在しない
     （同様にサーフゴー・デオキシス・マギアナ専用技のサイコブースト・フルールカノンも
     未掲載であり、対応するポケモンがChampions未実装であることが原因と推測される）。
     このため直接引用はできず、同系統の変換規則（SV基準PP5→Champions PP8）を根拠として
     PP8を採用した。将来 `move_list.txt` に本行が追加された場合は値を再確認すること。
4. **テスト方針**
   - 100%固定効果のため `secondary_chance` の調整は不要。命中後に自分の『とくこう』
     ランクが2段階下がることを確認する既存テスト（`test_ゴールドラッシュ_特攻1段階低下が発動する`）を
     -1→-2 の期待値に修正する（テスト名も実態に合わせて更新する）。
   - `t.calc_lethal` を用いたリーサルテストを新規追加する
     （`test_オーバーヒート_とくこうダウン` / `test_りゅうせいぐん_とくこうダウン` と同じパターン）。
