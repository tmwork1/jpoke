# どくどくのキバ 実装計画書

## 仕様要約（.internal/spec/moves/どくどくのキバ.md 参照）

* タイプ: どく / 分類: 物理 / 威力: 50 / 命中: 100 / PP: 15
* 対象: 相手1体 / 優先度: 0 / 接触あり（直接攻撃）
* 効果: 追加効果として、50%の確率で相手を『もうどく』状態にする。
* かみつき技（`flags` に `"bite"` を含む）。特性『がんじょうあご』の対象。

## 現状の実装状況

`src/jpoke/data/move.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。
以下のとおり仕様と一致していることを確認した（修正不要）。

```python
"どくどくのキバ": MoveData(
    type="どく",
    category="physical",
    pp=15,
    power=50,
    accuracy=100,
    flags={"bite", "contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.どくどくのキバ_apply_toxic_to_defender,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 50%の確率で相手を『もうどく』状態にする |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def どくどくのキバ_apply_toxic_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="もうどく", chance=0.5)
```

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を
経由するため、ちからずくによる追加効果無効化（確率0化）・威力1.3倍化、`flags={"bite"}` による
がんじょうあごの威力1.5倍化は、いずれも汎用ハンドラ側で処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（状態異常付与）は既存の同種ハンドラ（どくづき・どくばり・
どくばりセンボン 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。
個別の priority検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **がんじょうあごとの相互作用**
   - `flags={"bite"}` により、がんじょうあご所持ポケモンが使用すると威力が1.5倍（50→75）になる。
     既存の がんじょうあご テストファイル側で他のかみつき技と同様の網羅がされていれば個別追加は不要。
2. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・もうどく付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
3. **もうどく状態異常の重複判定**
   - 対象が既に状態異常（`ailment`）を持つ場合、`ailment_manager.apply` 側で重複付与されない
     ことを既存の共通実装が保証している（本技固有のガードは不要）。
4. **世代差**
   - 第五世代までは追加効果発動率30%だったが、第六世代以降50%に変更されている。
     本プロジェクトは現行仕様（50%）のみ対象。
5. **テスト方針**
   - `secondary_chance` を1に固定し、もうどく付与を確認するテストが既存
     （`test_どくどくのキバ_もうどくが発動する`）。
