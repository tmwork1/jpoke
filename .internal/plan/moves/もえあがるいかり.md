# もえあがるいかり 実装計画書

## 仕様要約（.internal/spec/moves/もえあがるいかり.md 参照）

* タイプ: あく / 分類: 特殊 / 威力: 90 / 命中: 100 / PP: 10
* 対象: 自分以外の全員（本プロジェクトはシングルバトル専用のため、実質「相手1体」として扱う）
* 優先度: 0 / 接触なし（直接攻撃ではない）
* 効果: 追加効果として、20%の確率で相手をひるませる。
* ファイヤー（ガラルのすがた）専用技。

## 現状の実装状況

`src/jpoke/data/moves/move_ma.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。
以下のとおり仕様と一致していることを確認した（修正不要）。

```python
"もえあがるいかり": MoveData(
    type="あく",
    category="special",
    pp=10,
    power=90,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.もえあがるいかり_apply_flinch,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 20%の確率で相手をひるませる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def もえあがるいかり_apply_flinch(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_defender(battle, ctx, value, volatile="ひるみ", chance=0.2)
```

`apply_volatile_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を
経由するため、ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は汎用ハンドラ側で処理される。
本技側に個別のガードは不要。あくのはどう（同一効果・威力80）と全く同じ実装パターンである。

## Priority 根拠

ダメージを与えた技の追加効果（ひるみ付与）は既存の同種ハンドラ（あくのはどう・かみなりのキバ 等）
と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。個別の priority検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・ひるみ付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
2. **おうじゃのしるし・するどいキバとの相互作用**
   - `handlers/item.py` の `_INNATE_FLINCH_MOVES` に「もえあがるいかり」が既に含まれており、
     元々ひるみ追加効果を持つ技として、おうじゃのしるし等による追加のひるみ判定が重複しないよう
     処理済み（修正不要）。
3. **テスト方針**
   - `secondary_chance` を1に固定し、ひるみの付与有無を確認するテストが既存
     （`test_もえあがるいかり_ひるみが発動する`、`tests/moves_attack/test_move_ma.py`）。
   - テストは五十音順のファイル配置ルールに従い、`もえあがるいかり`（ま行）は
     `tests/moves_attack/test_move_ma.py` に置く（`test_move_yarawa.py` への配置は誤りだったため
     本レビューで移動した）。
