# 実装計画: ぼうぎょしれい

## 仕様サマリ

- タイプ: むし / 分類: 変化 / PP: 10（Gen9本家準拠。champions/move_list.txt に本技自体が未収録のため） / 命中率: なし（自分対象）
- 効果: 自分の防御と特防をそれぞれ1段階ずつ上げる
- まもる: × / マジックコート: × / 直接攻撃: ×
- 対象: 自分

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自分の防御と特防を1段階ずつ上げる |

## Priority 根拠

`.internal/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の自分のランク上昇ハンドラ（`コスモパワー_modify_attacker_stats`（B+1,D+1）、
`こうそくいどう_modify_attacker_stats`（S+2）など）がすべて priority=100 デフォルトで
実装されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py への追加（実装済み）

挿入位置: 五十音順で `ほろびのうた_*` の後、`まきびし_*` の前（ほ → ぼ → ま の順）

```python
def ぼうぎょしれい_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ぼうぎょしれいの効果: 自分のぼうぎょ・とくぼうを1段階ずつ上げる。"""
    return modify_attacker_stats(battle, ctx, value, stats={"def": 1, "spd": 1})
```

### data/moves/move_ha.py の更新

`ぼうぎょしれい` の MoveData に `target="self"` を追加する（修正済み）。

```python
"ぼうぎょしれい": MoveData(
    type="むし",
    category="status",
    pp=10,
    target="self",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ぼうぎょしれい_modify_attacker_stats,
        ),
    }
),
```

## 注意点・エッジケース

- 対象は自分であるため `target="self"` を必ず設定する。デフォルトの `target="foe"` のままだと
  `Move.is_blocked_by_protect` / `Move.is_reflectable` が誤って True になり、相手の「まもる」で
  阻止されたり「マジックコート」で跳ね返されたりする不具合になる（レビュー時に発見・修正。
  `かたくなる` で過去に同種の不具合が確認された箇所と同じパターン）。
- `modify_attacker_stats` を使うことで、`subject_spec="attacker:self"`（`MoveHandler` のデフォルト）
  により対象が正しく自分になる
- 防御・特防の両方を1つの `stats` dict にまとめて `modify_attacker_stats` に渡せる。片方だけ
  ランク上限（+6）に達している場合はそのランクの上昇のみ失敗し、もう一方は成功する
  （`battle.modify_stats()` 内部処理）
- ランク上限チェックはフレームワーク側で処理されるため、ハンドラ側のガードは不要
- 命中率なし（自分対象）のため accuracy 設定不要。テストは命中率固定不要
- PPは champions/move_list.txt に本技自体の収録がないため、Gen9本家の値（10）をそのまま採用する
  （詳細は `.internal/spec/moves/ぼうぎょしれい.md` の「PPについて」を参照）
- リーサル計算は対象外（ダメージを与えない変化技のため、`.internal/progress/move.md` の
  リーサル実装列は n/a）
