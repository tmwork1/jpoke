## 仕様要約

- 命中し技が成功した場合のみ、攻撃後に自分の最大HPの1/2（**切り上げ**、最低1）を反動ダメージとして
  受ける（まもる・命中失敗・タイプ無効化などで不成立の場合は反動を受けない）。
- 与えたダメージが0でも、技自体が成立していれば反動を受ける（こらえる・ばけのかわ等）。
- いしあたま・マジックガードの双方で反動を防げる（既存の `HPChangeReason = "recoil"` に該当）。
- すてみでは威力が上がらない（`flags={"recoil"}` を付与しない）。

### 発見された誤り

1. **イベント選択が誤り**: `Event.ON_PAY_HP`（技使用前の無条件HP前払い）を使っており、
   まもる・命中失敗・タイプ無効化時にも反動ダメージを受けてしまう。一次情報は
   「技が成功した場合のみ反動を受ける」ため、`Event.ON_HIT`（命中時発火、実ダメージ0でも発火）
   に変更する必要がある。
2. **`HPChangeReason` が `"self_cost"` になっていた**: `"self_cost"` はいしあたま・マジックガード
   どちらでも防げない理由のため、「両方で防げる」という本技の一次情報と矛盾する。
   既存の `"recoil"`（いしあたま・マジックガード双方で防げる）に変更する。
   - `.internal/plan/moves/失敗反動技.md` の「reason の選択」節に「`self_cost` はみがわり・いのちがけ・
     クロロブラスト等の意図的な自己HP消費」という記載があるが、これは本技の一次情報
     （.internal/wiki/moves/クロロブラスト.html）で明示されている特性相互作用と矛盾するため、
     本技に関する記載は誤りとして扱う（他技の記載自体は本レビューでは変更しない）。
3. **端数処理が切り捨てになっていた**: `cost = max(1, ctx.attacker.max_hp // 2)` は切り捨て。
   一次情報は「小数点以下切り上げ」のため `(ctx.attacker.max_hp + 1) // 2` に修正する。
4. **PPがGen9本家の値のまま**: `pp=5` はSV本家の値。チャンピオンズはPP8/12/16/20の4段階に
   統一されており、`move_list.txt` に該当行はないが、Gen9本家PP5の技は例外なくPP8になっている
   規則（オーバーヒート・ハイドロカノン・はかいこうせん・りゅうせいぐん・てっていこうせん等）
   から類推し、`pp=8` に修正する。

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_DAMAGE`（実装上の `Event.ON_HIT` に相当。`いのちのたま`・
`かいがらのすず`・`だっしゅつボタン`・`レッドカード` が同じ理由でこのイベントを使い、
turn.md記載のpriority値をそのまま流用している既存パターンを踏襲する）に
「100 | 反動技による反動ダメージ」とあり、これは `MoveHandler` の既定 priority（100）と一致する。
既存の `_recoil`（アフロブレイク・ウェーブタックル・ウッドハンマー等）もこの既定値のまま
`Event.ON_HIT` に登録しているため、本技も priority を明示せずデフォルトのまま実装する。

## 実装箇所

### `src/jpoke/handlers/move_attack.py`

既存の `クロロブラスト_pay_hp` を以下に置き換える（関数名も実態に合わせて変更）。

```python
def クロロブラスト_recoil(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """クロロブラスト: 命中時に自分の最大HPの1/2（切り上げ）を反動ダメージとして受ける。

    ダメージ量に関わらず固定量。いしあたま・マジックガードの双方で防げる（reason="recoil"）。
    Event.ON_HIT は実ダメージ0（こらえる等）でも発火するため0ダメージ成功時も反動を受ける仕様を
    満たし、まもる・命中失敗・タイプ無効化時はそもそも発火しないため反動を受けない仕様も満たす。
    """
    cost = max(1, (ctx.attacker.max_hp + 1) // 2)
    battle.modify_hp(ctx.attacker, v=-cost, reason="recoil", source=ctx.attacker)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース（review-test 向け）

- **まもるで防がれると反動なし**: 対象がまもるを使ったとき、使用者のHPが減らないこと。
- **命中失敗で反動なし**: 命中率0（命中判定失敗）のとき、使用者のHPが減らないこと。
- **命中して0ダメージでも反動あり**: 対象のHPを1にして`こらえる`を付与し、大ダメージを与えても
  対象のHPが1のまま（実ダメージ0）である状況で、使用者は通常通り反動ダメージを受けること。
- **いしあたまで反動無効**: いしあたま特性の使用者は反動ダメージを受けないこと
  （てっていこうせん・ビックリヘッドとの違いを明確にするため、必ずテストで区別する）。
- **マジックガードで反動無効**: マジックガード特性の使用者は反動ダメージを受けないこと。
- **切り上げの確認**: 最大HPが奇数のポケモンで、消費HPが `(max_hp + 1) // 2` になることを
  テストで確認する（偶数の場合は切り捨てと同じ値になり区別できないため、奇数個体が必要）。
- **最低コスト1**: `max(1, (max_hp + 1) // 2)` により、最大HPが1でもコストは1になる。
- **反動の発生順序**: 相手へのダメージ適用より後に自分のHPが減ること
  （＝両者のHPが0になる場面で、相手が先に倒れることの間接的な確認になる）。
- **すてみで威力上昇しない**: `flags={"recoil"}` を付与していないため、すてみ特性を持っていても
  威力は上がらないことを確認する（フラグを追加しないことが正しい実装）。
- **てっていこうせん・ビックリヘッドとの共通テストからの分離**: `tests/moves_attack/test_move__group.py`
  の `test_HPコスト技グループ_最大HPの半分を消費する` は「使用前に無条件でHP消費し、切り上げをしない」
  という誤った前提でクロロブラストをパラメトライズしていたため、クロロブラストを外し
  `tests/moves_attack/test_move_ka.py` に専用テストを追加する（てっていこうせんが既に同様の理由で
  分離されている前例に倣う）。
- **うのミサイル・レンブのみとの順序は対象外**: .internal/spec/moves/クロロブラスト.md の該当節を参照。
  既存アーキテクチャ全体に関わる制約のため本レビューでは対応しない。
