# 計画書: トリックフラワー

## 仕様要約

- くさタイプ / 物理 / 威力70 / 命中-（必中） / PP12 / 対象: 1体選択 / 優先度0 / 非接触
- 必ず急所に当たる（急所ランク3）。追加効果なし。

## ハンドラ構成

追加効果を持たず、必中（`accuracy=None`）と急所ランク3の指定のみで確定急所・必中を
表現できるため、専用ハンドラは不要。`handlers={}` と `accuracy=None`, `critical_rank=3`
のみで基本ダメージ計算に委ねる。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、priority の検討対象イベントは存在しない。

## 実装箇所

### data/moves/move_ta.py への更新

既存の `"トリックフラワー"` エントリは以下の通りで、仕様と一致している（変更不要）。

```python
    "トリックフラワー": MoveData(
        type="くさ",
        category="physical",
        pp=12,
        power=70,
        critical_rank=3,
        handlers={},  # 追加効果なし
    ),
```

- `accuracy` を指定していないため `None`（必中）になる → 仕様と一致。
- `flags` を指定していないため `contact` フラグは付与されない → 非接触（仕様と一致）。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `critical_rank=3` により、既存の急所判定共通ロジック（シェルアーマー/カブトアーマー・
  おまじない・かたやぶりとの相互作用を含む）で確定急所として処理される。
- `accuracy=None` により、既存の命中判定共通ロジック（`_check_accuracy`）で必中として
  処理される（相手の回避率ランク・自分の命中率ランクを無視）。

## リーサル計算

追加効果・確率分岐を持たない通常攻撃であり、急所ランク3は `calc_lethal` 呼び出し側が
`critical` 引数で急所有無を指定する既存の枠組みで表現されるため、本技固有の
リーサルハンドラは不要（`.internal/progress/move.md` で n/a）。

## 注意点・エッジケース

- 追加効果がないため、まもる・おうじゃのしるし・オウムがえし等の一般的な技判定は
  フレームワーク側の共通処理（`handlers/move_attack.py` の基本ダメージ計算経路）で
  自動的に処理され、本技固有のハンドラ実装は不要。
- 確定急所（`critical_rank=3`）とシェルアーマー/カブトアーマー・おまじない・かたやぶりの
  相互作用は既存の急所判定共通ロジックで処理されるため、本計画では扱わない
  （`.internal/plan/moves/こおりのいぶき.md`・`やまあらし.md` 相当のロジックと同一）。
- 必中（`accuracy=None`）は既存の命中判定共通ロジックで処理されるため、本計画では扱わない
  （`.internal/plan/moves/あてみなげ.md` 相当のロジックと同一）。
