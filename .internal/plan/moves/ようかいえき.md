# ようかいえき 実装計画書

## 仕様要約（.internal/spec/moves/ようかいえき.md 参照）

* タイプ: どく / 分類: 特殊 / 威力: 40 / 命中: 100 / PP: 30
* 対象: 相手1体 / 優先度: 0 / 非接触
* 効果: 追加効果として、10%の確率で相手の『とくぼう』ランクを1段階下げる。

## 現状の実装状況

`src/jpoke/data/moves/move_ya.py` に既に登録済みで、仕様と一致している（変更不要）。

```python
"ようかいえき": MoveData(
    type="どく",
    category="special",
    pp=30,
    power=40,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ようかいえき_lower_defender_spd,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手の『とくぼう』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def ようかいえき_lower_defender_spd(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spd": -1}, chance=0.1)
```

`modify_defender_stats`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は `flags={"secondary_effect"}` さえ
付与されていれば汎用的に処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（能力ランク低下）は既存の同種ハンドラ（きあいだま・だいちのちから・
あわ 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。
`ON_DAMAGE_HIT` は `.internal/spec/turn.md` に未掲載のイベント（ダメージ計算内部）だが、
既存の同種ハンドラの実装パターンを踏襲するため個別の priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` が付与されているため、ちからずく所持時は威力1.3倍（40→52）になり、
     とくぼう低下の追加効果は発動しなくなる。
2. **テスト方針**
   - `secondary_chance` を0/1に固定し、とくぼうランクの変化有無を確認するテストを追加する
     （ようかいえき_相手にダメージを与える／ようかいえき_とくぼう1段階低下が発動する／
     ようかいえき_とくぼう1段階低下が発動しない）。
