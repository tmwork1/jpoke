# 実装計画書: ジャングルヒール

## 仕様要約

- タイプ: くさ / カテゴリ: 変化 / PP: 10（champions/move_list.txt に本技の行が存在しないため
  Gen9本家値を採用） / 命中: なし（必中） / 対象: 味方全体
- 自分と味方全体の HP をそれぞれ最大 HP の 1/4 ずつ回復し、状態異常を治す（**端数は切り捨て**）
- シングルバトルでは「味方全体」＝使用者自身のみが対象
- `MoveData.target="self"` を明示する必要がある（デフォルト値 `"foe"` のままだと、
  `まもる` / `おうごんのからだ` 等の `move.target == "foe"` 判定に誤って合致してしまう）
- 使用者の HP が満タンで、かつ状態異常もない場合は技が失敗する（一方が満たされなければ成功する）
- `heal` フラグあり（`かいふくふうじ` の対象・`ヒーリングシフト` の優先度+3対象になる）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | HP満タン・状態異常無時の失敗チェック → 自分のHPを最大HPの1/4（切り捨て）回復 → 状態異常を治す |

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_APPLY_MOVE` セクションに以下の記載がある：

> | 100 | ジャングルヒール: HP満タン・状態異常無 |

このイベントはコード上の `ON_STATUS_HIT`（`いのちのしずく`・`みかづきのいのり` 等の
自己回復技と同様のパターン）に対応する。`みかづきのいのり` は「タイプとPPを除けば基本的に
ジャングルヒールと同様の効果」であり、既にこの priority（100=デフォルト）・判定パターンを
本技から踏襲して実装済みのため、本技側も同一の priority・判定パターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

`みかづきのいのり_apply` と同型の関数を新規実装する（じ→じゃ→じゅ の五十音順で
`じばそうさ_modify_attacker_stats` の後、`じゅうでん_apply` の前に配置）。

```python
def ジャングルヒール_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ジャングルヒール: 自分のHPを最大HPの1/4回復し、状態異常を治す。

    HPが満タンかつ状態異常もない場合は失敗する。端数は切り捨て。
    """
    mon = ctx.attacker
    if mon.hp == mon.max_hp and not mon.ailment.is_active:
        return HandlerReturn(value=False, stop_event=True)
    battle.modify_hp(mon, r=1 / 4)
    battle.ailment_manager.remove(mon)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/moves/move_sa.py`

既存の `"ジャングルヒール"` エントリを修正する（`target="self"` の明示・`handlers` の実装追加）:

```python
"ジャングルヒール": MoveData(
    type="くさ",
    category="status",
    pp=10,
    target="self",
    flags={"heal"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(hs.ジャングルヒール_apply),
    },
),
```

`target="self"` を明示することで、以下の判定が正しく機能する:

- `Move.is_blocked_by_protect`（`target == "foe"` が前提）→ `まもる` に防がれない
- `おうごんのからだ_block_status_move` など（`target == "foe"` が前提）→ 誤ってブロックされない

## 注意点・エッジケース

- シングルバトルでは「味方全体」の回復対象は使用者自身のみ。ダブルバトルは対象外のため、複数対象
  への回復処理・みがわり判定は実装しない
- 失敗チェックは `mon.hp == mon.max_hp and not mon.ailment.is_active` の両方が真の場合のみ。
  片方だけ満たす場合（例: HP満タンだが状態異常あり）は成功し、状態異常のみ治す
- `HandlerReturn(value=False, stop_event=True)` を返すと `move_executor.py` 内で `MOVE_FAILED`
  ログが記録される
- 端数処理（切り捨て）は `battle.modify_hp(mon, r=1/4)` で行う（`modify_hp` の `r=` 引数は内部で
  `int()` により切り捨てを行うため、追加の丸め処理は不要）
- `かいふくふうじ` 状態のポケモンがこの技を使用した場合の無効化は、`かいふくふうじ` ハンドラ側で
  `heal` フラグを参照して処理されるため、本ハンドラ側での追加チェックは不要
