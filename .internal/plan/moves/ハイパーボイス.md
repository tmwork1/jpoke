# 実装計画書: ハイパーボイス

参照仕様書: .internal/spec/moves/ハイパーボイス.md

## 仕様要約

- ノーマルタイプ / 特殊 / 威力90 / 命中100 / PP12（Champions基準） / 対象: 相手全体
  （シングルバトルでは相手1体） / 優先度0 / 非接触
- `sound`（音の技）フラグを持つ。みがわり貫通は `move_executor.py` の
  `check_hit_substitute` が `sound` フラグを見て汎用処理するため、本技固有のハンドラは不要。
- 追加効果なし。
- ダブルバトルで複数対象にヒットする場合はダメージ0.75倍（既存の汎用ハンドラ
  `handlers/move_attack.py:reduce_damage_in_double_battle` で処理）。

## ハンドラ構成

追加効果を持たず、`sound` フラグと既存の範囲攻撃技共通ハンドラのみで挙動が完結するため、
新規の専用ハンドラは不要。

* `ON_CALC_DAMAGE_MODIFIER`: `move_attack.reduce_damage_in_double_battle`
  （`.internal/spec/moves/ばくおんぱ.md` / `.internal/spec/moves/こんげんのはどう.md` と同一の
  既存汎用ハンドラを流用する）

## Priority 根拠

`ON_CALC_DAMAGE_MODIFIER` は `.internal/spec/turn.md` に未掲載のイベント（ダメージ計算内部処理）のため、
既存の同種ハンドラ（`ばくおんぱ`・`こんげんのはどう`・`じしん` 等の範囲攻撃技に登録済みの
`reduce_damage_in_double_battle`）に倣い、`MoveHandler` のデフォルト優先度（`priority=100`）を
そのまま使用する。本技専用の新規ハンドラは実装しないため、priority の新規検討は不要。

## 実装箇所

### data/move.py への更新

既存の `src/jpoke/data/moves/move_ha.py` の `"ハイパーボイス"` エントリは以下の通りで、
仕様と一致している（変更不要）。

```python
    "ハイパーボイス": MoveData(
        type="ノーマル",
        category="special",
        pp=12,
        power=90,
        accuracy=100,
        flags={"sound"},
        handlers={
            Event.ON_CALC_DAMAGE_MODIFIER: h.MoveHandler(
                ha.reduce_damage_in_double_battle,
            )
        },
    ),
```

* `flags` に `contact` を含まないため非接触 → 仕様と一致。
* `target` 省略時のデフォルトは `"foe"`（相手1体）→ シングルバトルでは仕様と一致。
* `priority` 省略時のデフォルトは `0` → 仕様と一致。
* `flags={"sound"}` により、以下の既存汎用機構が自動的に働く。
  - `move_executor.py: check_hit_substitute` によるみがわり貫通
  - `handlers/ability.py` のぼうおん・パンクロック等、音技を参照する特性の判定
  - `handlers/item.py` ののどスプレー等、音技を参照するアイテムの判定
  - `handlers/volatile.py` のじごくづき（音技禁止）判定
* `ON_CALC_DAMAGE_MODIFIER` の `reduce_damage_in_double_battle` により、
  `battle.option.double_battle` が真かつ実際に複数対象にヒットする場合のみダメージ0.75倍。
  シングルバトル専用の本プロジェクトでは通常は影響しないが、`double_battle` オプションを
  有効にしたテスト構成（`tests/test_battle_option.py`）との整合のため既存実装のまま維持する。

## テスト方針

`tests/moves_attack/test_move_ha.py` に以下を追加・確認する。

* 通常のダメージ攻撃技として相手にダメージを与えることを確認するテスト
* 音の技であるため、みがわり状態の相手に対して本体にダメージを与える（みがわりを貫通する）ことを
  確認するテスト（実装済み: `test_ハイパーボイス_みがわりを貫通して本体にダメージを与える`）

以下は既存の共通テストで既にカバーされているため、本技専用の追加テストは不要。

* `tests/test_battle_option.py` のダブルバトル0.75倍補正テスト
  （`test_ダブルバトル_複数対象になり得る技はダメージ0_75倍` 等）
* `tests/moves_attack/test_move_sa.py` のじごくづきによる音技禁止テスト
  （`test_じごくづき_じごくづき状態の相手は音技を使えない`）
* `tests/items/test_item_na.py` ののどスプレー関連テスト
* `tests/abilities/test_ability_a.py` のうるおいボイス関連テスト
* `tests/abilities/test_ability_ha.py` のフェアリースキン関連テスト

## リーサル計算

  （`.internal/progress/move.md` のリーサル実装/リーサルテスト列は n/a）。

## 注意点・エッジケース

* ダブルバトル自体は本プロジェクトの対象範囲外（シングルバトルのみ）だが、既存の
  `reduce_damage_in_double_battle` ハンドラは他の範囲攻撃技と共通のテスト（`test_battle_option.py`）で
  カバーされているため、本技のエントリからも参照したままとする。
* `sound` フラグに紐づく各種特性・アイテム側の挙動（ぼうおん・パンクロック・のどスプレー等）は
  それぞれの特性・アイテム側のハンドラで汎用的に処理されるため、本技固有の実装は不要。
