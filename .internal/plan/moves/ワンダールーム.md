# 実装計画書: ワンダールーム

## 仕様

### 技データ

- タイプ: エスパー
- カテゴリ: 変化（`category="status"`）
- 威力/命中率: なし（`power`/`accuracy` とも未設定 = 常に命中）
- PP: 12（championsルールにより8/12/16/20の4段階へ統一。`src/jpoke/data/ps-champ-ja/moves.json` 準拠。
  本家 Gen5〜9 では PP10）
- 優先度: 0（`MoveData.priority` のデフォルト値をそのまま使用。第五世代では -7 だったが、
  第六世代以降は 0 に変更されており、対象世代（チャンピオンズ＝Gen9ベース）では 0 が正しい）
- 対象: 全体の場（`target="field"`。デフォルト値 `"foe"` のままだと `まもる`/`マジックコート` の
  判定が `move.target == "foe"` を前提にしているため誤って防がれる/跳ね返されてしまう）
- 直接攻撃: しない（`flags` に `"contact"` を含めない。デフォルトの空集合のままでよく、
  `flags` キー自体を明示する必要はない）

### 効果

場を『ワンダールーム』状態（5ターン、発動ターンを含む）にする。すでに発動中の場合は
再使用によって即座に解除される（トグル動作）。フィールド自体の詳細効果（防御・特防の実数値
参照入れ替え・カウントダウン）は `.internal/plan/fields/ワンダールーム.md` / `.internal/spec/fields/ワンダールーム.md`
を参照。技自体はフィールドの発動・解除のみを行う。

### 参照仕様書

- `.internal/spec/moves/ワンダールーム.md`（一次情報の生スクレイプ）
- `.internal/spec/fields/ワンダールーム.md`（キュレーション済みのフィールド仕様）

## subject_spec と priority

### イベント: `ON_STATUS_HIT`

- **ctx型**: `AttackContext`（変化技の `ON_STATUS_HIT` は攻撃フローとして発火するため
  `ctx.attacker` / `ctx.defender` を参照する）
- **subject_spec**: `"attacker:self"`（`MoveHandler` のデフォルト値を使用）
- **priority**: 100

**根拠**: `ON_STATUS_HIT` は `.internal/spec/turn.md` に独立エントリが存在しない。同種の
全体の場を発動する変化技 `トリックルーム_activate_global_field` / `マジックルーム_activate_global_field` /
`じゅうりょく_activate_global_field` も priority=100 のデフォルトを使用しているため、
これと同一パターンとして priority=100 を採用する。

## ハンドラ構成

### 1. `src/jpoke/handlers/move_status.py`

既存実装（五十音順の挿入位置: `わるだくみ_modify_attacker_stats` の後）:

```python
def ワンダールーム_activate_global_field(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    manager = battle.global_manager
    if manager.fields["ワンダールーム"].is_active:
        return HandlerReturn(value=manager.deactivate("ワンダールーム"))
    return HandlerReturn(value=manager.activate("ワンダールーム", 5))
```

- すでに『ワンダールーム』状態が有効な場合、`deactivate` でフィールドを即座に解除する
  （トリックルーム・マジックルームと同様、再使用による解除効果がある）。
- 未発動の場合は `activate` で5ターン（発動ターンを含む）発動する。

### 2. `src/jpoke/data/moves/move_wa.py`

`"ワンダールーム"` エントリ:

```python
"ワンダールーム": MoveData(
    type="エスパー",
    category="status",
    pp=12,
    target="field",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ワンダールーム_activate_global_field,
        ),
    }
),
```

### 3. `src/jpoke/data/field/global_field.py`

フィールド本体の効果（防御・特防の実数値参照入れ替え・カウントダウン）は既存の
`GLOBAL_FIELD["ワンダールーム"]` に実装済み。詳細は `.internal/plan/fields/ワンダールーム.md` を参照。

## 注意事項

- `target="field"` を明示しないと `MoveData.target` のデフォルト値 `"foe"` になり、
  `is_blocked_by_protect`（`まもる`）・`マジックコート_reflect` の判定が `move.target == "foe"` を
  条件にしているため、誤って「まもるで防がれる」「マジックコートで跳ね返される」動作になる
  （wiki の判定表でも まもる: ×、マジックコート: × と明記されている一方、オウムがえし: ○ で
  返すことは可能）
- 技の優先度（0）は `MoveData.priority` を明示せずデフォルトのままでよい
- ワンダールームによる防御・特防の実数値参照入れ替え自体は技ハンドラの責務ではなく、
  フィールドハンドラ（`ON_CALC_DEF_MODIFIER`）の責務。技ハンドラはフィールドの発動・解除のみを行う
- PPは `src/jpoke/data/ps-champ-ja/moves.json` 準拠で12（本家Gen5〜9のPP10からの変更）
