# 計画書: とける

## 仕様要約

- 自分の「ぼうぎょ」ランクを2段階上げる
- タイプ: どく、変化、PP: 20、対象: 自分

出典: `.internal/spec/moves/とける.md`

---

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100 (省略) | `attacker:self` (省略) | ぼうぎょを2段階上げる |

---

## Priority 根拠

`.internal/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`てっぺき_modify_attacker_stats` が B+2 で同パターン）が priority=100 (デフォルト) で登録されているため、100 を使用する。

---

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: `とぐろをまく_modify_attacker_stats` の後（「とけ」は「とぐろ」の後、「トリックルーム」の前）

```python
def とける_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"def": 2})
```

### `src/jpoke/data/moves/move_ta.py`

`"とける"` エントリに `handlers` を追加する:

```python
    "とける": MoveData(
        type="どく",
        category="status",
        pp=20,
        target="self",
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.とける_modify_attacker_stats,
            ),
        }
    ),
```

---

## 注意点・エッジケース

- `てっぺき` と同じ 防御+2 効果。実装パターンが完全に同一
- ランクがすでに最大（+6）の場合、`battle.modify_stats` が空の変化辞書を返し技は失敗する。ハンドラ側にガード不要
- 自分を対象とする変化技のため、相手の `まもる` では防がれない（`MoveHandler` の `subject_spec` はデフォルトの `attacker:self` のままでよい）
