# 実装計画書: ミルクのみ

## 仕様

### 技データ

- タイプ: ノーマル
- カテゴリ: 変化（`category="status"`）
- PP: 8（championsルールにより8/12/16/20の4段階へ統一。`src/jpoke/data/ps-champ-ja/moves.json` に単独行は
  存在しないが、同一効果の `じこさいせい`・`タマゴうみ`・`なまける`・`はねやすめ`・`ねむる`・
  `こうごうせい`・`つきのひかり` がいずれもchampionsPP8であることから類推した。一次情報の
  「第八世代までPP10・第九世代PP5」はGen9本家基準の値であり、championsの4段階統一とは無関係の
  ため採用しない）
- 対象: 自分（`target="self"`。デフォルト値 `"foe"` のままだと `まもる`/`マジックコート` の
  判定が `move.target == "foe"` を前提にしているため誤って防がれる/跳ね返されてしまう）
- label: `heal`

### 効果

使用者の HP を最大HPの 1/2 分回復する。HP が満タンの場合は失敗する（`じこさいせい`・`タマゴうみ`
と同一実装）。

### 参照仕様書

- `.internal/spec/moves/ミルクのみ.md`
- `.internal/spec/moves/じこさいせい.md`（同一効果の実装パターン）
- `.internal/spec/moves/タマゴうみ.md`（PPの類推根拠が同一のため参照）

## subject_spec と priority

### イベント: `ON_STATUS_HIT`

- **ctx型**: `AttackContext`（変化技の `ON_STATUS_HIT` は攻撃フローとして発火するため
  `ctx.attacker` / `ctx.defender` を参照する。`EventContext` の `source`/`target` ではない）
- **subject_spec**: `"attacker:self"`（`MoveHandler` のデフォルト値を使用）
- **priority**: 100

**根拠**: `.internal/spec/turn.md` の `ON_APPLY_MOVE` テーブルに「自己回復技: HP満タン」が priority 100
として記載されている。`ON_STATUS_HIT` は `turn.md` に独立エントリが存在しないが、同種の変化技
自己回復ハンドラ `じこさいせい_heal_self`・`タマゴうみ_heal_self` も
`h.MoveHandler(hs.<関数名>)` で priority=100 のデフォルトを使用しているため、これと同一パターン
として priority=100 を採用する。

## ハンドラ構成

### 1. `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: `ミラータイプ_can_apply` の後、`みをけずる_apply` の前
（`data/moves/move_ma.py` の定義順: ミラータイプ → ミルクのみ → みわくのボイス に合わせる）。

```python
def ミルクのみ_self_heal(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ミルクのみ: 自分のHPを最大HPの1/2回復する（じこさいせい・タマゴうみと同一効果）。"""
    mon = ctx.attacker
    if mon.hp == mon.max_hp:
        return HandlerReturn(value=False, stop_event=True)
    battle.modify_hp(mon, r=1/2)
    return HandlerReturn(value=value)
```

### 2. `src/jpoke/data/moves/move_ma.py`

`"ミルクのみ"` エントリ:

```python
"ミルクのみ": MoveData(
    type="ノーマル",
    category="status",
    pp=8,
    target="self",
    flags={"heal"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ミルクのみ_self_heal,
        ),
    },
),
```

## リーサル計算

対象外（n/a）。`じこさいせい`等の同種自己回復技と同様、攻撃者自身の回復は防御側のHP分布に
影響しないため、リーサル計算（防御側が生存するか否かの判定）には影響しない。

## 注意事項

- `battle.modify_hp(mon, r=r)` の `r` は小数で指定。端数処理は `battle.modify_hp` 側に委ねる
  （`int(max_hp * r)` による切り捨て）
- HP満タン判定は `mon.hp == mon.max_hp` で行い、失敗時は `HandlerReturn(value=False, stop_event=True)`
  を返す（既存実装は `mon.hp >= mon.max_hp` になっていたが、HPが最大値を超えることはないため
  意味は同じ。他の同種ハンドラとの表記統一のため `==` に修正する）
- `target="self"` を明示しないと `MoveData.target` のデフォルト値 `"foe"` になり、
  `is_blocked_by_protect`（`まもる`）・`マジックコート_reflect` の判定が `move.target == "foe"` を
  条件にしているため、誤って「まもるで防がれる」「マジックコートで跳ね返される」動作になる
  （既存実装は `target` 未指定でこのバグを含んでいたため修正が必要）
- `flags={"heal"}` は既存実装に設定済み。かいふくふうじ・ヒーリングシフトの汎用処理が
  この flag を参照するため、本技専用の実装は不要
