# 計画書: じこあんじ

## 仕様要約（.internal/spec/moves/じこあんじ.md）

- タイプ: ノーマル / カテゴリ: 変化 / PP: 12 / 命中: 必中（`accuracy=None`） / 対象: 相手1体
- 自分の能力ランク（A/B/C/D/S/命中/回避）を相手の現在のランクと完全に一致させる（上書き）
- 第六世代以降は急所ランクに関する効果（本プロジェクトでは `きゅうしょアップ`
  揮発性状態＝`critical_rank`）もコピー対象
- ランクコピーは通常のランク変化イベントを経由しない
  （`たんじゅん`・`あまのじゃく`・`クリアボディ`・`だっしゅつパック`・`かちき`・
  `まけんき`・`びんじょう`・`ものまねハーブ`は反応しない）
- 例外: `しろいハーブ`のみ、コピー直後に負のランクが残っていれば個別に発動する
- `まもる`を無視するが `トリックガード`／`ダイウォール`には防がれる（`unprotectable`）
- `マジックコート`／`マジックミラー`で跳ね返されない（`unreflectable`）
- `みがわり`状態の相手にも成功する（`bypass_substitute`）
- `おうごんのからだ`の相手には失敗する（既存の`おうごんのからだ`実装のみで対応済み、
  追加実装不要）
- `よこどり`／`オウムがえし`はいずれも本プロジェクトで未実装のため対応不要

|---|---|---|---|
| `Event.ON_STATUS_HIT` | 100（デフォルト） | `"attacker:self"` | 使用者のランク・急所ランクを相手のランク・急所ランクへ直接代入し、しろいハーブの特例処理を行う |

## Priority 根拠

`.internal/spec/turn.md` に `ON_STATUS_HIT` の記載はない（ダメージ計算内部イベントの扱い）。
類似の直接ランク代入系ハンドラ（`ガードスワップ_swap_ranks`・`パワースワップ_swap_ranks`）が
いずれも `priority=100`（`MoveHandler` のデフォルト）で登録されているため、同じ 100 を採用する。

### `src/jpoke/handlers/move_status.py`

`じこあんじ_copy_ranks` を以下のとおり修正する（急所ランクのコピー・しろいハーブ特例を追加）。
ランクのコピーは `mon.rank` dict を直接代入する。`battle.modify_stats()` はデルタ加算
＋通常のランク変化イベント（`ON_BEFORE_MODIFY_STAT`/`ON_MODIFY_STAT`）を経由するAPIで
あり、「相手の現在値で上書き」かつ「たんじゅん・あまのじゃく・クリアボディ・
だっしゅつパック等を発動させない」という仕様には不向きなため、直接代入を採用する。

```python
def じこあんじ_copy_ranks(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """じこあんじ: 相手の能力ランク変化・急所ランクをすべて自分にコピーする。

    相手のランクは変化しない。direct代入により、たんじゅん・あまのじゃく・
    クリアボディ・だっしゅつパック等のランク変化に反応する特性・アイテムを
    経由しない。ただし、しろいハーブのみコピー直後にマイナスのランクを
    打ち消す特例がある（一次情報: .internal/wiki/moves/じこあんじ.html 技の仕様節）。
    """
    attacker = ctx.attacker
    defender = ctx.defender
    rank_stats: list[Stat] = ["atk", "def", "spa", "spd", "spe", "accuracy", "evasion"]
    for stat in rank_stats:
        attacker.rank[stat] = defender.rank[stat]

    # 急所ランクに関する効果（きゅうしょアップ状態）も第六世代以降コピー対象。
    attacker.critical_rank = defender.critical_rank

    # しろいハーブ: マイナスのランクをコピーした直後に発動する（じこあんじ固有の特例）。
    if attacker.item.name == "しろいハーブ":
        changed = {s: -v for s, v in attacker.rank.items() if v < 0}
        if changed:
            for s in changed:
                attacker.rank[s] = 0
            battle.add_event_log(
                attacker, LogCode.STAT_CHANGED,
                payload=StatChangePayload(stats=changed, display_reason="しろいハーブ"),
            )
            battle.add_event_log(
                attacker, LogCode.ITEM_TRIGGERED,
                payload=ItemPayload(item=attacker.item.name),
            )
            attacker.item.revealed = True
            battle.item_manager.consume_item(attacker)

    return HandlerReturn(value=value)
```

`ItemPayload` を `jpoke.core.event_logger` からのimportに追加する。

## 注意点・エッジケース（review-test 向け）

- **相手のランクが全て0**: 自分のランクも全て0になる（失敗しない）。
- **相手のランクは変化しない**: コピーは片方向。
- **急所ランクのコピー**: `きあいだめ`で相手に`きゅうしょアップ`（`critical_rank=2`）を
  付与した状態でじこあんじを使うと、使用者にも`critical_rank=2`が付与される。
  相手が`critical_rank=0`なら使用者の`きゅうしょアップ`は消える（クリアされる）。
- **しろいハーブ**: 使用者が`しろいハーブ`を持ち、コピー結果に負のランクが含まれる場合、
  当該ランクが全て0に戻り、アイテムが消費される。`だっしゅつパック`は同条件でも
  発動しない点と対比してテストする。
- **たんじゅん・あまのじゃく**: 使用者がこれらの特性を持っていても、コピーされる値は
  そのまま（変化量が変わらない）。
- **みがわり**: 相手が`みがわり`状態でも成功する（`bypass_substitute`）。
- **まもる**: 相手が`まもる`状態でも成功する（`unprotectable`）。
- **マジックコート**: 跳ね返されない（`unreflectable`）。
- **PP**: `src/jpoke/data/ps-champ-ja/moves.json`の値（12）を採用。Gen9本家値（10）ではない。
- **subject_spec**: `MoveHandler`のデフォルト`"attacker:self"`を使用。`AttackContext`内で
  `ctx.attacker`/`ctx.defender`を直接参照するため、対象選択には影響しない。
