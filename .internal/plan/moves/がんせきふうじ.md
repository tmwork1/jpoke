# がんせきふうじ 実装計画書

## 仕様要約（.internal/spec/moves/がんせきふうじ.md 参照）

* タイプ: いわ / 分類: 物理 / 威力: 60 / 命中: 95 / PP: 15
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果: 追加効果として100%の確率で相手の『すばやさ』ランクを1段階下げる。

## 現状の実装状況（修正前）

`src/jpoke/data/move.py` にはタイプ・分類・威力・命中・PPとも仕様書と一致する値で登録済みだったが、
`flags` に `"secondary_effect"` が設定されておらず、`.internal/spec/moves/_move_secondary_1.md`
（がんせきふうじ: defender 確率1 S-1）と食い違っていた。

```python
"がんせきふうじ": MoveData(
    type="いわ",
    category="physical",
    pp=15,
    power=60,
    accuracy=95,
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.がんせきふうじ_lower_defender_spd,
        )
    }
),
```

### 修正後

```python
"がんせきふうじ": MoveData(
    type="いわ",
    category="physical",
    pp=15,
    power=60,
    accuracy=95,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.がんせきふうじ_lower_defender_spd,
        )
    }
),
```

`flags={"secondary_effect"}` がないと、`ちからずく_boost`（`handlers/ability.py`）による威力1.3倍化と
`ちからずく_disable_secondary_effect` によるすばやさ低下無効化のいずれも発動しなくなってしまう
（`ctx.move.has_flag("secondary_effect")` が常に False になるため）。同種の実装として
`あわ_lower_defender_spd`・`からみつく_lower_defender_spd` が全く同じパターンで
`flags={"secondary_effect"}` を持っている。

追加効果ハンドラ（`handlers/move_attack.py` の `がんせきふうじ_lower_defender_spd`）は仕様と一致しており、
修正不要。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 100%の確率で相手の『すばやさ』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def がんせきふうじ_lower_defender_spd(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spe": -1})
```

`modify_defender_stats`（`handlers/move.py`）は `chance` 省略時デフォルト1（100%）で
`battle.resolve_secondary_chance` を経由するため、ちからずくによる追加効果無効化は
汎用的に処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（能力ランク低下）は既存の同種ハンドラ（あわ・からみつく 等）と同じ
`ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。`.internal/spec/turn.md` にも
`ON_DAMAGE_HIT` は他の追加効果技と共通のタイミングとして扱われており、個別の priority 検討は不要。
