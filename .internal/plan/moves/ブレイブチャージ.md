# 計画書: ブレイブチャージ

## 仕様要約（.internal/spec/moves/ブレイブチャージ.md 参照）

* エスパータイプ / 変化 / 命中なし / PP15（champions未収録のため第九世代基準を採用） / 対象: 自分
* 自分の『とくこう』『とくぼう』ランクをそれぞれ1段階ずつ上げる
* 自分の状態異常を回復する
* 2つの効果は互いに独立しており、どちらか片方（あるいは両方）が不発でも技自体は失敗しない

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略可） | とくこう・とくぼうランクを1段階ずつ上げ、状態異常を回復する |

失敗条件チェック用ハンドラは不要（本技はランク上限・状態異常なしのいずれの場合でも失敗しない）。

## Priority 根拠

`.internal/spec/turn.md` に ブレイブチャージ（および `ON_STATUS_HIT` イベント自体）の記載なし。
類似の自己対象ランク変化技（`めいそう`, `コスモパワー`, `ビルドアップ`）が `ON_STATUS_HIT` priority
100（`MoveHandler` のデフォルト）を使用しているため、同パターンを採用する。

## 実装（実装済み）

### handlers/move_status.py

```python
def ブレイブチャージ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ブレイブチャージ: 自分のとくこう・とくぼうを1段階ずつ上げ、状態異常を回復する。

    ランク上限・状態異常なしのいずれの場合でも技自体は失敗しない。
    """
    modify_attacker_stats(battle, ctx, value, stats={"spa": 1, "spd": 1})
    battle.ailment_manager.remove(ctx.attacker)
    return HandlerReturn(value=value)
```

`modify_attacker_stats` の戻り値（ランク変化が完全に阻まれた場合は失敗扱いになりうる）を使わず、
常に呼び出し時の `value`（技の成否フラグ）をそのまま返すことで、ランク上限・状態異常なしのいずれの
場合でも技自体を失敗させない仕様を満たしている。
`battle.ailment_manager.remove()` は対象に状態異常がない場合は何もせず `False` を返すだけなので、
安全に無条件で呼び出せる。

挿入位置: 五十音順で「ブ」の位置（`ふるいたてる_modify_attacker_stats` の後、`へびにらみ_apply_ailment_to_defender` の前）。

## リーサル計算

対象外（`n/a`）。自分の能力ランク変化・状態異常回復のみで、相手のHPには影響しないため。

## テスト（実装済み: tests/moves_status/test_move_fa.py）

* `test_ブレイブチャージ_とくこうとくぼうが上がり状態異常が治る`: 通常時、とくこう・とくぼうが
  +1ずつ上がり、まひが治る
* `test_ブレイブチャージ_ランク上限でも技は失敗しない`: とくこう・とくぼうが+6のときでも技は
  成功し、状態異常（まひ）は治る
* `test_ブレイブチャージ_状態異常がなくても技は失敗しない`: 状態異常がなくても技は成功し、
  とくこう・とくぼうは+1ずつ上がる

## 注意点・エッジケース

* `Pokemon.hp` への直接代入禁止 → 本技はHPを扱わないため該当なし
* `subject_spec` は `AttackContext` のロールと一致させる（自己対象のためデフォルト
  `"attacker:self"` を省略）
* champions の `move_list.txt` に本技のエントリが存在しない点はレビュー時に確認済み。
  フィオネ・マナフィ（ミュウ的ポジションの専用技持ち）はチャンピオンズのシングル対戦では
  実質使用不可のため、PP再編対象から漏れていると判断し、第九世代基準のPP15を維持する。
