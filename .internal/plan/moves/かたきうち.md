# 計画書: かたきうち

## 仕様要約

* 基本威力70。味方がひんしになった次のターンに使用すると威力が2倍（140）になる
  （`.internal/spec/moves/かたきうち.md` 参照）。
* 「次のターン」判定はシビアで、ひんしになった**そのターン中**の使用や、
  2ターン以上経過してからの使用では威力は上がらない。
* 直接攻撃（接触技）。

## 既存実装の状況

`data/move.py` にエントリはあるが `handlers={}`（追加効果なし扱い）となっており、
本技の核心である威力2倍効果が全く実装されていない（バグ）。テストも存在しない。

## 必要な状態管理

「味方がひんしになったターン」はポケモン単位ではなくプレイヤー（チーム）単位の情報であり、
かつ「ひんしになった味方」と「かたきうちを使うポケモン」は別個体（交代後の後続）である。
そのため既存の `Pokemon` の `_this_turn` 系フラグ（例: `stat_lowered_this_turn`）や
`failed_or_immobile_last_turn`（ポケモン自身の前ターン行動を見るだけ）では表現できない。

`PlayerState` に新規フィールドを追加し、プレイヤー単位で「直近に味方がひんしになったターン」を
記録する。

### `core/player_state.py`

```python
self.ally_fainted_turn: int | None = None  # 味方が直近にひんしになったターン（かたきうち用）
```

* ターンをまたいでリセットする必要はない（絶対ターン番号を保持し、使用時に
  `現在のターン - 1` と比較するだけで判定できるため）。
* `__deepcopy__` は既存の `fast_copy` に委譲されており、`int | None` はプリミティブなので
  追加のディープコピー対象指定は不要。

### `core/status_manager.py` の `modify_hp`

既に `stat_lowered_this_turn` 等、特定技専用の状態をこの階層で更新している前例があるため、
同様の書き方で「対象がひんしになった瞬間、そのポケモンが所属するプレイヤーの
`ally_fainted_turn` に現在のターンを記録する」処理を追加する。

```python
if v < 0:
    if target.fainted:
        self.battle.judge_winner()
        # かたきうち用: 味方がひんしになったターンを記録する
        player = self.battle.get_player(target)
        self.battle.player_states[player].ally_fainted_turn = self.battle.turn

    self._events.emit(Event.ON_HP_CHANGED, ctx, -v)
```

これにより、味方・相手を問わず「ひんしになったポケモンが所属するプレイヤー」の状態が
更新される（かたきうちの判定時は自分側の記録のみを参照するため、相手のひんしで
誤って威力が上がることはない）。

## ハンドラ構成

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| かたきうち | `Event.ON_CALC_POWER_MODIFIER` | デフォルト（100） | `attacker:self`（デフォルト） | 味方が前ターンにひんしになっていれば威力を2倍にする |

`Event.ON_CALC_POWER_MODIFIER` は `.internal/spec/turn.md` に priority の掲載がないため、
同種の可変威力技（`おはかまいり_calc_power`・`からげんき_double_power_when_ailment` 等）が
いずれも `MoveHandler` のデフォルト値（priority=100, subject_spec="attacker:self"）のまま
登録されていることを根拠に、デフォルト設定のまま登録する。

```python
def かたきうち_calc_power(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """かたきうち: 味方が前のターンにひんしになっていた場合、威力が2倍になる。"""
    player = battle.get_player(ctx.attacker)
    state = battle.player_states[player]
    if state.ally_fainted_turn == battle.turn - 1:
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)
```

```python
"かたきうち": MoveData(
    type="ノーマル",
    category="physical",
    pp=5,
    power=70,
    accuracy=100,
    flags={"contact"},
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
            ha.かたきうち_calc_power,
        ),
    }
),
```

## テスト

`tests/moves_attack/test_move_ka.py` に以下を追加する。

* 前のターンに味方（控えのポケモン）がひんしになった直後のターンで使用 → 威力2倍（140）で
  ダメージ計算されることを確認。
* 味方がひんしになったそのターン中（強制交代後、同一ターン扱いになるケースは本プロジェクトの
  ターン進行上発生しないため、代わりに）ひんしから2ターン経過後に使用 → 威力が上がらない
  （70のまま）ことを確認。
* 味方が一度もひんしになっていない状態で使用 → 威力が上がらない（70のまま）ことを確認。
* 相手側のポケモンがひんしになった次のターンに使用しても、自分側の記録には影響しないため
  威力は上がらないことを確認（相手をひんしにしてから自分が行動するケース）。

## 注意点・エッジケース

* シングルバトル専用のため「ダブルバトルで味方がひんしになったそのターン」のケースは
  発生しない。ただし本プロジェクトでも「ひんしになったターンそのもの」で本技を使うこと自体は
  ない（ひんしになったポケモンは交代するまで行動できないため）。テストでは
  「2ターン以上経過後は威力が上がらない」ことで実質的に同じ境界条件を確認する。
