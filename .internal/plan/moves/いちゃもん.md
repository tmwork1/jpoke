# 計画書: いちゃもん

参照仕様書: .internal/spec/moves/いちゃもん.md

## 仕様要約

- あくタイプ / 変化 / 命中100 / PP16（`src/jpoke/data/ps-champ-ja/moves.json` 準拠） / 対象: 1体選択
- 相手を「いちゃもん」揮発性状態にする。いちゃもん状態のポケモンは、いちゃもん付与時点で
  相手が直前に選択・実行していた技（`ctx.defender.executed_move`）をコマンド選択時に
  選べなくなる。
- 対象が場に出てから技を未使用（`executed_move is None`）でも付与自体は成功する
  （かなしばりと異なり、未行動を理由に失敗しない）。
- 揮発性状態「いちゃもん」自体の詳細（コマンド制限・解除条件）は既存の
  `.internal/spec/volatiles/いちゃもん.md` / `handlers/volatile.py` の
  `いちゃもん_modify_command_options` に実装済み。本計画は技側（`いちゃもん_apply`）のみを対象とする。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略時デフォルト） | 相手にいちゃもん付与（`move_name` に相手の直前実行技を記録） |

## Priority 根拠

`.internal/spec/turn.md` には `ON_STATUS_HIT` の記載がなく、既存の同種ハンドラ
（`かなしばり_apply`、`メロメロ_apply` 等）がすべてデフォルト priority=100 の
`MoveHandler` として登録されているため、これに合わせる。

## 実装箇所

### handlers/move_status.py

`いちゃもん_apply`（既存関数を修正、五十音順で「いたみわけ」の後・「いとをはく」の前）:

```python
def いちゃもん_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """いちゃもんの効果: 相手がいちゃもん状態になり、直前に選択した技を出せなくなる。

    相手が場に出てから技を使用していない場合でも付与自体は成功する
    （その場合は move_name="" となり、実質的に禁止技は発生しない）。
    """
    move = ctx.defender.executed_move
    return apply_volatile_to_defender(
        battle, ctx, value, volatile="いちゃもん", move_name=move.name if move else ""
    )
```

`かなしばり_apply` と同じパターンで `ctx.defender.executed_move` から相手の直前実行技を
取得し、`apply_volatile_to_defender` の `move_name` キーワード引数に渡す。
これにより `いちゃもん_modify_command_options`（`handlers/volatile.py`）が参照する
`mon.volatiles["いちゃもん"].move_name` に正しい技名が設定される。

### data/moves/move_a.py

`"いちゃもん"` エントリの PP を修正する（Gen9基準の15のまま移行漏れになっていたため、
Champions基準の16に修正）。ハンドラ登録自体は変更なし。

```python
    "いちゃもん": MoveData(
        type="あく",
        category="status",
        pp=16,
        accuracy=100,
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.いちゃもん_apply,
            ),
        }
    ),
```

## 注意点・エッジケース

- `move_name` を渡し忘れると `いちゃもん_modify_command_options` が空文字列と比較することになり、
  技側からの付与が実質的に何も制限しない状態になる（修正前の実装で発生していたバグ）。
- かなしばりと異なり「相手が未行動なら失敗」という `can_apply` 系ハンドラは不要
  （Wiki: 「場に出てから技を使用していないポケモンもいちゃもん状態になる」）。
- まもる: `target == "foe"` の変化技として自動的に防がれる（専用フラグ不要）。
- マジックコート: `マジックコート_reflect` が「対象が相手・変化技」であるすべての技を
  汎用的に跳ね返す実装のため、専用対応は不要。
- 「すでにいちゃもん状態」の場合は `volatile_manager.apply` が内部で
  `target.has_volatile(name)` を見て失敗を返す既存の共通処理に委ねる（`メロメロ` 等と同じパターン）。
