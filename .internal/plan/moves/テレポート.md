## 仕様要約

- タイプ: エスパー / 分類: 変化 / PP: 20 / 命中: なし（必中） / 優先度: -6
- 対象: 自分
- 効果: 控えのポケモンと交代する。とらわれ状態（にげられない・バインド・フェアリーロック・
  かげふみ・ありじごく・じりょく）を無視して確実に交代できる（第八世代以降のトレーナー戦仕様）。
- 失敗条件: 交代できる控えのポケモン（生存個体）がいない場合。
- バトンタッチと異なり、能力ランク・揮発性状態の引き継ぎは行わない。

## Priority 根拠

- `ON_TRY_MOVE_1` priority=30:
  `.internal/spec/turn.md` の `Event.ON_TRY_MOVE_1` 節に「30 | テレポート: トレーナー戦・交代不可」と
  明記されている。同 priority 帯には「たくわえる: カウントが3」「みちづれ: 最後の成功行動が
  みちづれ」等、技固有の失敗条件チェックが並んでおり、同じ位置づけで実装する。
- `ON_STATUS_HIT` priority=100:
  `.internal/spec/turn.md` には ON_STATUS_HIT の個別 priority は掲載されていない。
  同じ交代技である `バトンタッチ_apply`（`.internal/plan/moves/バトンタッチ.md` 参照）が
  デフォルト priority（100）を使用しているため、これに倣う。

## 実装箇所

### 1. `src/jpoke/handlers/move_status.py`

五十音順で「テ」は「て」行に属するため、既存の「て」始まりの関数群
（`てんしのキッス_apply` 等）の近くに追加する（最終配置は `sort_handlers.py` に委ねる）。

```python
def テレポート_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """テレポートの効果: とらわれ状態を無視して控えのポケモンと交代する。

    バトンタッチと異なり、能力ランクや揮発性状態の引き継ぎは行わない。
    """
    player = battle.get_player(ctx.attacker)
    battle.player_states[player].interrupt = Interrupt.PIVOT
    return HandlerReturn(value=value)


def テレポート_check(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """テレポートの失敗条件チェック: 控えに生きているポケモンがいない場合は失敗する。

    とらわれ状態（にげられない・バインド・フェアリーロック等）でも交代可能なため、
    トラップチェックを経由せず控えポケモンの生存のみを確認する（バトンタッチ_check と同様）。
    """
    mon = ctx.attacker
    player = battle.get_player(mon)
    state = battle.player_states[player]
    if not any(m.alive for m in state.bench):
        battle.add_event_log(
            mon, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="テレポート_交代不可"),
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース（review-test 向け）

1. **`target="self"` の明示**
   - 未設定のまま（デフォルト `"foe"`）だと `Move.is_blocked_by_protect` /
     `Move.is_reflectable` が誤判定し、まもるで防がれる・マジックコートで反射される
     という誤動作になる。`アクアリング` と同様に必ず明示する。

2. **とらわれ状態を無視する交代**
   - `バトンタッチ_check` / `バトンタッチ_apply` と同じロジックを流用する。
     `battle.command_manager.get_available_switch_commands(player)`（トラップチェックあり）
     ではなく、`state.bench` の生存確認のみで判定し、`interrupt = Interrupt.PIVOT` を
     直接設定する。

3. **控えがいない場合**
   - `テレポート_check`（`ON_TRY_MOVE_1`）で失敗させる。`ON_STATUS_HIT` 側では
     控え不在チェックを重複させない（`ON_TRY_MOVE_1` で弾かれるため到達しない）。

4. **優先度 -6**
   - `.internal/wiki/moves/テレポート.html` に明記された値。バトンタッチ（優先度0）とは異なる点に注意。

5. **ランク・揮発性状態の引き継ぎなし**
   - バトンタッチと違い `baton_pass_data` 相当の処理は不要。`Interrupt.PIVOT` を設定するだけで
     交代先の選択・繰り出しは既存の共通インフラ（`SwitchManager.run_interrupt_switch`）に委ねる。
