# 計画書: ばくれつパンチ

## 仕様要約（.internal/spec/moves/ばくれつパンチ.md 参照）

- かくとうタイプ / 物理 / 威力100 / 命中50 / PP5 /
  対象: 1体選択 / 優先度0 / 接触あり（直接攻撃）/ パンチ技
- 追加効果として、命中時に100%の確率で相手をこんらん状態にする。
- 追加効果に分類されるため、ちからずく（威力1.3倍・効果無効化）の対象技になる。
- パンチ技のため、特性「てつのこぶし」・アイテム「パンチグローブ」所持時は威力が1.2倍になる
  （共通処理のため本技固有の対応は不要）。

## 既存実装との差分

既存実装（`data/moves/move_ha.py` の `"ばくれつパンチ"` エントリ、`handlers/move_attack.py` の
`ばくれつパンチ_apply_confusion_to_defender`）は仕様と一致しており、差分なし。

## ハンドラ構成

### handlers/move_attack.py: `ばくれつパンチ_apply_confusion_to_defender`（既存・変更なし）

```python
def ばくれつパンチ_apply_confusion_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ばくれつパンチの追加効果: 相手をこんらん状態にする（確率100%）。"""
    return apply_confusion_to_defender(battle, ctx, value)
```

- `apply_confusion_to_defender`（`handlers/move.py`）は内部で `battle.resolve_secondary_chance` を
  呼び出すため、`Event.ON_MODIFY_SECONDARY_CHANCE` 経由でちからずく（確率0化・威力1.3倍）の
  ハンドラが自動的に適用される（`secondary_effect` フラグが前提）。
- 内部で `battle.volatile_manager.apply_confusion` を呼び、こんらんの付与ターン数（2〜5）は
  共通処理に委譲される。
- イベント: `Event.ON_DAMAGE_HIT`（ダメージが実際に適用された後に発動。
  .internal/spec/turn.md の `## Event.ON_DAMAGE` 参照）。
  ダメージが発生しなかった場合（無効化・まもる等）はイベント自体が発火しないため、
  追加効果も発生しない仕様を自然に満たす。
- 優先度: デフォルト100（`MoveHandler` のデフォルト値）。
- subject_spec: `MoveHandler` デフォルト（`attacker:self`）のまま。

### data/moves/move_ha.py（既存・変更なし）

```python
"ばくれつパンチ": MoveData(
```

## 注意点・エッジケース

- 命中率50%のため外れることが多いが、命中判定自体は共通のダメージフローで処理されるため
  本技固有の対応は不要。
- まもるで防がれた場合、`Event.ON_DAMAGE_HIT` 自体が発火しないため追加効果も発生しない。
  個別のガードは不要。
- 既にこんらん状態の相手に対しては `volatile_manager.apply_confusion` 側で重複付与しない
  （`VolatileManager` の既存ロジックに準拠、本技側での重複ガードは不要）。
- パンチ技の威力補正（てつのこぶし・パンチグローブ）は `flags={"punch"}` を通じて
  特性・アイテム側のハンドラが処理するため、本技側の実装は不要。
