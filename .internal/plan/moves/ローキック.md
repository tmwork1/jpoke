# ローキック 実装計画書

## 仕様要約（.internal/spec/moves/ローキック.md 参照）

* タイプ: かくとう / 分類: 物理 / 威力: 65 / 命中: 100 / PP: 20
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 追加効果として100%の確率で相手の『すばやさ』ランクを1段階下げる。

## 現状の実装状況（修正前）

`src/jpoke/data/moves/move_ra.py` にはタイプ・分類・威力・命中・PPとも仕様書と一致する値で
登録済みだったが、`flags` に `"secondary_effect"` が設定されておらず、
`.internal/spec/moves/_move_secondary_1.md`（ローキック: defender 確率1 S-1）と食い違っていた。

```python
"ローキック": MoveData(
    type="かくとう",
    category="physical",
    pp=20,
    power=65,
    accuracy=100,
    flags={"contact"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ローキック_lower_defender_spd,
        )
    }
),
```

### 修正後

```python
"ローキック": MoveData(
    type="かくとう",
    category="physical",
    pp=20,
    power=65,
    accuracy=100,
    flags={"contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ローキック_lower_defender_spd,
        )
    }
),
```

`flags={"secondary_effect"}` がないと、`ちからずく_boost`（`handlers/ability.py`）による威力1.3倍化と
`ちからずく_disable_secondary_effect` によるすばやさ低下無効化のいずれも発動しなくなってしまう
（`ctx.move.has_flag("secondary_effect")` が常に False になるため）。同種の実装として
`がんせきふうじ_lower_defender_spd`・`こごえるかぜ_lower_defender_spd` が全く同じパターンで
`flags={"secondary_effect"}` を持っている。

追加効果ハンドラ（`handlers/move_attack.py` の `ローキック_lower_defender_spd`）は仕様と一致しており、
修正不要。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 100%の確率で相手の『すばやさ』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def ローキック_lower_defender_spd(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spe": -1})
```

`modify_defender_stats`（`handlers/move.py`）は `chance` 省略時デフォルト1（100%）で
`battle.resolve_secondary_chance` を経由するため、ちからずくによる追加効果無効化は
汎用的に処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（能力ランク低下）は既存の同種ハンドラ（がんせきふうじ・こごえるかぜ 等）と
同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。`.internal/spec/turn.md` には
`ON_DAMAGE_HIT` の個別記載はなく、他の追加効果技と共通のダメージ計算内部タイミングとして扱われるため、
個別の priority 検討は不要。

### data/moves/move_ra.py

## リーサル計算について

`.internal/progress/move.md` の該当行「リーサル実装」列は `-` だが、本技の追加効果は相手の『すばやさ』ランクを
下げるのみでHPに影響しない（攻撃・防御・特攻・特防のようにダメージ計算に直接介入するランクではない）。
同種の `がんせきふうじ`・`こごえるかぜ`（いずれもすばやさ-1の100%追加効果）にも
`handlers/lethal.py` の専用ハンドラは存在せず、対象外（n/a）として扱われている。ローキックも同様に
`handlers/lethal.py` に専用ハンドラは不要であり、n/a が妥当。`.internal/progress/move.md` の表記を
n/a に修正する。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` を追加したことで、ちからずく所持時は威力が1.3倍になり、
     すばやさ低下効果が発動しなくなる（`がんせきふうじ` 等と同じ挙動）。
2. **テスト方針**
   - `secondary_chance` を1に固定し、すばやさランク変化が発動することを確認するテストを
     `tests/moves_attack/test_move_yarawa.py` に追加する（`がんせきふうじ` の既存テストと同じパターン）。
   - ちからずく所持時に威力1.3倍・すばやさ低下無効化になることを確認するテストを追加する
     （`がんせきふうじ` の既存テストと同じパターン）。
