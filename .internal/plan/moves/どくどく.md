# どくどく 実装計画書

## 仕様要約（.internal/spec/moves/どくどく.md 参照）

* タイプ: どく / 分類: 変化 / 威力: - / 命中: 90 / PP: 12（champions基準）
* 対象: 相手1体 / 優先度: 0
* 効果: 相手を『もうどく』状態にする。使用者がどくタイプの場合は必中になる。

```python
"どくどく": MoveData(
    type="どく",
    category="status",

    accuracy=90,
    handlers={

            hs.どくどく_accuracy,

        ),
    }

```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_MODIFY_ACCURACY` | 100（デフォルト） | attacker:self | 使用者がどくタイプなら必中（`None`）にする |
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 相手を『もうどく』状態にする |

`src/jpoke/handlers/move_status.py` の実装:

```python
def どくどく_accuracy(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """どくどくの命中率補正。使用者がどくタイプの場合は必中になる。"""
    if "どく" in ctx.attacker.types:
        return HandlerReturn(value=None)  # 必中
    return HandlerReturn(value=value)


def どくどく_apply_ailment_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="もうどく")
```

`どくどく_apply_ailment_to_defender` は既存実装のまま変更なし。`apply_ailment_to_defender`
（`handlers/move.py`）が `battle.ailment_manager.apply` を経由するため、どくタイプ・
めんえき等による無効化は汎用実装側で処理される。本技側に個別のガードは不要。

`ctx.attacker.types` は現在の実際のタイプ（テラスタル・へんげんじざい・リベロ等の変化を
反映済み）を返すため、命中判定前にどくタイプへ変化した場合も自動的に必中となる
（`めざめるダンス_modify_type` 等、既存実装と同じ参照方法）。

マジックコートで跳ね返した場合、`ctx.attacker` は跳ね返した側ではなく元の使用者を指す
（既存の `マジックコート_reflect` の仕組みに従うため、本技側での追加対応は不要）。

## Priority 根拠

`.internal/spec/turn.md` に `ON_MODIFY_ACCURACY` / `ON_STATUS_HIT` の記載はないため、
既存の同種ハンドラを参照した。
`ON_MODIFY_ACCURACY` は `かみなり_accuracy`（`handlers/move_attack.py`）と同じ
デフォルトpriority・`subject_spec="attacker:self"` パターンに従う。
`ON_STATUS_HIT` は既存の状態異常付与技（`どくガス` 等）と同じ `MoveHandler` 既定値
（priority100・`attacker:self`）に従う。

`.internal/progress/move.md` の該当行「リーサル実装」列は `n/a`（対象外）。

## 注意点・エッジケース（review-test 向け）

1. **PP修正**: `src/jpoke/data/ps-champ-ja/moves.json` 基準で12（Gen9本家の10ではない）。
2. **どくタイプ使用時の必中**: `ctx.attacker.types` に `"どく"` が含まれる場合、
   命中率90を無視して必ず命中する。
3. **どくタイプでない使用者の場合**: 通常どおり命中率90で判定する。
4. **まもる・みがわり**: 通常の変化技と同様に防がれる（既存の共通実装が対応済み、
   本技固有のガードは不要）。
5. **マジックコート・オウムがえし**: 既存の共通実装（`ctx.move.is_reflectable`）が
   対応済みのため、本技固有の対応は不要。
6. **テスト方針**: `moves_status` にどくタイプ使用時の必中テスト、どくタイプでない
   使用者が外れるテスト、命中時にもうどく状態になるテストを追加する。
