# 実装計画: クロスサンダー

## 仕様サマリ

- タイプ: でんき / 分類: 物理 / 威力: 100 / 命中率: 100 / PP: 8（本家PP5→チャンピオンズPP8変換）
- 効果: 同じターン中に直前でクロスフレイムが命中していた場合、威力が2倍になる（通称デュアルクロス）
- 発動条件: 先発技（クロスフレイム）が命中しなかった／タイプ相性・まもる・特性吸収
  （ちくでん・もらいび等）で無効化された場合は発動しない
- まもる: ○ / マジックコート: ×（変化技ではない） / 直接攻撃: ×

詳細は `.internal/spec/moves/クロスサンダー.md` を参照。

## 対象イベント

2つのイベントを使う。

1. `Event.ON_CALC_POWER_MODIFIER` — 威力倍率を計算するタイミング。ここでクロスフレイムが
   直前に命中していたかどうかを判定し、威力を2倍にする。
2. `Event.ON_HIT` — 技が命中した直後に発火するイベント。まもる・タイプ相性・特性吸収・
   命中失敗のいずれの場合も発火しないため、「命中して成功した」ことの記録に使える。
   ここでクロスサンダー自身が命中したターン番号を記録する。

### なぜ `battle.last_used_move_name` を使わないか

`battle.last_used_move_name` は「まねっこ」用に技実行の発動確定時点
（`core/move_executor.py` の `_execute_move` 冒頭、命中判定より前）で更新される
汎用フィールドである。クロスサンダー自身の発動時点で既に自分の技名で上書きされて
しまうため、`ON_CALC_POWER_MODIFIER`（ダメージ計算内で発火し、上書きより後）で
参照すると必ず「クロスサンダー自身」と比較することになり、常に不成立になってしまう。
また、このフィールドは「命中したか」を問わず技が失敗しなければ更新されるため、
クロスサンダー側が要求する「命中して初めて成立」という条件とも一致しない。

このため、クロスサンダー・クロスフレイム専用の新規フィールドを `Battle` に追加し、
それぞれの `ON_HIT` ハンドラで自分が命中したターン番号を記録する方式を採る
（`りんしょう` が `battle.round_used_turn` で同ターン内の連鎖使用を判定する
パターンと同様の設計だが、りんしょうは「使用（発動）したか」で足りるため
`ON_BEGIN_MOVE` で判定・記録しているのに対し、クロスサンダーは「命中したか」まで
要求するため `ON_HIT` を使う点が異なる）。

## Priority

`.internal/spec/turn.md` に `ON_HIT` の記載はあるが（priority 10〜100 の一覧）、本ハンドラは
既存ハンドラと競合する処理ではなく、単に自分の命中ターンを記録するだけの副作用のため、
デフォルト優先度（`MoveHandler` のデフォルト `priority=100`）で問題ない。

`ON_CALC_POWER_MODIFIER` は `.internal/spec/turn.md` に記載がない（ダメージ計算内部のイベント）。
既存の同種ハンドラ（`かたきうち_calc_power`・`からげんき_double_power_when_ailment`・
`ライジングボルト_calc_power`）がいずれも `MoveHandler(func, subject_spec=...)` のみで
priority を明示指定していない（デフォルト値を使用）ため、本技も同様にデフォルトを使う。

## 実装手順

### core/battle.py

`Battle.__init__` に、クロスサンダー・クロスフレイムがそれぞれ直近に命中した
ターン番号を保持するフィールドを追加する（`round_used_turn` と同じ配置場所）。

```python
self.fusion_bolt_used_turn: int | None = None  # クロスサンダー: 直近に命中したターン番号（クロスフレイムとの威力2倍判定用）
self.fusion_flare_used_turn: int | None = None  # クロスフレイム: 直近に命中したターン番号（クロスサンダーとの威力2倍判定用）
```

### handlers/move_attack.py

`クロスポイズン_apply_poison_to_defender` の直前（五十音順: くらいつく → クロスサンダー →
クロスフレイム → クロスポイズン）に4つの関数を追加する。

```python
def クロスサンダー_calc_power(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """クロスサンダー: 同じターン中に直前でクロスフレイムが命中していた場合、威力が2倍になる。

    「命中していた」の判定はクロスフレイム側の ON_HIT ハンドラ（クロスフレイム_record_hit）が
    記録する battle.fusion_flare_used_turn を参照する。まもる・タイプ相性・ちくでん等の特性で
    無効化された場合や命中しなかった場合は ON_HIT 自体が発火しないため、記録されず倍化しない。
    """
    if battle.fusion_flare_used_turn == battle.turn:
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)


def クロスサンダー_record_hit(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """クロスサンダー: 命中したターンを記録する（クロスフレイムとの威力2倍判定用）。"""
    battle.fusion_bolt_used_turn = battle.turn
    return HandlerReturn(value=value)


def クロスフレイム_calc_power(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """クロスフレイム: 同じターン中に直前でクロスサンダーが命中していた場合、威力が2倍になる。

    判定方法はクロスサンダー_calc_power と対になる（battle.fusion_bolt_used_turn を参照）。
    """
    if battle.fusion_bolt_used_turn == battle.turn:
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)


def クロスフレイム_record_hit(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """クロスフレイム: 命中したターンを記録する（クロスサンダーとの威力2倍判定用）。"""
    battle.fusion_flare_used_turn = battle.turn
    return HandlerReturn(value=value)
```

`apply_fixed_modifier` は本ファイル冒頭で既に import 済み
（`from jpoke.utils.math import apply_fixed_modifier, ...`）。

### data/moves/move_ka.py

クロスサンダー・クロスフレイムの両方の `MoveData` に `ON_CALC_POWER_MODIFIER` と
`ON_HIT` ハンドラを追加する。PP はクロスサンダーのみ 5→8 に修正する
（クロスフレイムの PP 修正は本レビューの対象外。担当スコープはクロスサンダーのみのため、
クロスフレイム側は威力2倍判定を機能させるために必要な最小限の handlers 追加に留める）。

```python
"クロスサンダー": MoveData(
    type="でんき",
    category="physical",
    pp=8,
    power=100,
    accuracy=100,
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
            ha.クロスサンダー_calc_power,
            subject_spec="attacker:self",
        ),
        Event.ON_HIT: h.MoveHandler(
            ha.クロスサンダー_record_hit,
            subject_spec="attacker:self",
        ),
    },
),
```

```python
"クロスフレイム": MoveData(
    type="ほのお",
    category="special",
    pp=5,
    power=100,
    accuracy=100,
    flags={"thaw"},
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
            ha.クロスフレイム_calc_power,
            subject_spec="attacker:self",
        ),
        Event.ON_HIT: h.MoveHandler(
            ha.クロスフレイム_record_hit,
            subject_spec="attacker:self",
        ),
    },
),
```

## テスト方針

- クロスフレイムが直前に命中 → クロスサンダーの威力が200になることを確認
  （ダメージ計算経由、または `battle.damage_calculator` 相当の検証手段で確認）
- クロスフレイムが不在のターン（何も直前に成功していない）→ 威力は100のまま
- クロスフレイムが直前に命中したが、そのフレイム自身がまもるで防がれていた場合 →
  威力は100のまま（デュアルクロス不成立）
- クロスフレイムが直前に外れた場合 → 威力は100のまま
- ターンをまたぐと効果が持続しない（前のターンにクロスフレイムが命中していても
  次のターンのクロスサンダーは威力2倍にならない）ことを確認
