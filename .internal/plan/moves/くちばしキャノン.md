# くちばしキャノン 実装計画書

## 仕様要約（.internal/spec/moves/くちばしキャノン.md 参照）

* タイプ: ひこう / 分類: 物理 / 威力: 120（Champions） / 命中: 100 / PP: 15
* 対象: 相手1体 / 優先度: -3 / 接触なし（弾のわざ）
* 効果: 加熱してから攻撃を繰り出すまでの間に直接攻撃を受けると、攻撃者をやけど状態にする。
  やけど付与は追加効果扱いではなく、技はそのまま発動する。使用者が加熱後にひんしになった
  場合でもやけど効果は発動する（一次情報）。
* `non_negoto`（ねごとで選ばれない）・`bullet`（ぼうだんの対象）ラベルを持つ。

## 実装方針（改訂）

当初は「使用者自身の行動開始時点（`ON_TRY_ACTION`）で `Pokemon.contact_hitter`（接触技を
受けた際に core 側が記録する他ポケモン参照）を確認する」設計だったが、以下2点の問題があった。

1. **ひんし時にやけどが発動しない**: `turn_controller.py` の `_run_move_phase` は
   `attacker.fainted` の場合そのポケモンの `run_move` 自体を呼び出さないため、使用者が
   加熱後にひんしになると `ON_TRY_ACTION` ハンドラが実行されず、一次情報と矛盾していた。
2. **`Pokemon.contact_hitter` が他ポケモンへの参照を保持する専用フィールド**になっており、
   `Battle.__deepcopy__` の汎用参照付け替え処理でしか救済できない特殊な状態だった
   （`memory` スコープでは Pokemon/Move オブジェクト参照を保持できない制約があるため）。

これを解消するため、**「くちばしキャノン」揮発性状態＋新設 `Event.ON_BEFORE_MOVE`** による
設計に変更した。加熱を「使用者自身に一時的な揮発性状態を付与する」ことで表現し、やけど付与は
ダメージ適用直後の `Event.ON_DAMAGE_HIT` で即座に判定する。これにより `contact_hitter` の
ような他ポケモン参照フィールドが不要になり、ひんし時のやけど付与も自然に成立する。

## Event.ON_BEFORE_MOVE（新設）

`.internal/spec/turn.md` の `Event.ON_BEFORE_MOVE`（技使用前の処理、priority=100で
「くちばしキャノンの準備行動」が記載）に対応する、通常戦闘用の `Event` を新設した
（従来は `LethalEvent.ON_BEFORE_MOVE` という致死率計算専用の同名イベントのみ存在していた）。

* emit: `core/turn_controller.py` の `_run_before_move_phase`（新設フェーズ）。
  行動順解決（`_resolve_action_order`）の後・実際の技実行（`_run_move_phase`）の前に、
  `self.action_order` の各プレイヤーについて予約コマンドを `command_to_move` で技に解決し、
  その技のハンドラを一時的に登録した上で `EventContext(source=mon)` として発火、直後に解除する
  （`run_move` 内の register/unregister とは独立した、この1イベントのみを対象とする短命な
  登録）。交代・瀕死・技コマンド以外（`command.is_type("move")` が False）は対象外。
* handle: `handlers/move_attack.py` の `くちばしキャノン_start_heating`
  （`subject_spec="source:self"`）。

## ハンドラ構成

### data/moves/move_ka.py（くちばしキャノン MoveData）

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_BEFORE_MOVE` | 100 | source:self | 「くちばしキャノン」揮発性状態を自身に付与する（加熱開始） |

### data/volatile.py（"くちばしキャノン" VolatileData）

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100 | defender:self | 接触技のダメージ直後に攻撃者をやけど状態にする |
| `ON_TRY_ACTION` | 5 | attacker:self | 使用者自身の行動開始時点で加熱を終了する（揮発性状態を解除） |
| `ON_TURN_END` | 100 | source:self | 行動が発生しなかった場合（瀕死・強制交代等）のフォールバック解除 |

```python
def くちばしキャノン_start_heating(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """くちばしキャノンの予備動作: この技を選んだ時点で体を加熱させる。"""
    battle.volatile_manager.apply(ctx.source, "くちばしキャノン")
    return HandlerReturn(value=value)


def くちばしキャノン_burn_on_contact(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """くちばしキャノンの加熱中に接触技を受けた場合、その攻撃者を即座にやけど状態にする。"""
    if battle.query.is_contact_reaction(ctx):
        battle.ailment_manager.apply(ctx.attacker, "やけど", source=ctx.defender, ctx=ctx)
    return HandlerReturn(value=value)


def くちばしキャノン_end_heating(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """くちばしキャノンの加熱を終了する。"""
    return remove_volatile(battle, ctx, value, volatile="くちばしキャノン")
```

## Priority 根拠

### `ON_BEFORE_MOVE` priority=100（MoveData）

`.internal/spec/turn.md` の記載どおり priority=100（他に同イベントを使うハンドラが無いため
競合なし）。

### `ON_TRY_ACTION` priority=5（VolatileData、加熱終了）

まひ（`ON_TRY_ACTION` priority=120 相当）・こおり（priority=10）・こんらん（priority=110）等の
行動可否判定が `stop_event=True` を返しても、それより前の priority=5 で加熱終了が必ず実行される
よう、既存の「行動可否判定より前に必ず処理を行う」パターン（`かえんぐるま_thaw_attacker` 等と
同区分）を踏襲した。これにより「相手の優先度がより低い場合」（カウンター・ドラゴンテール等）に、
使用者自身の行動開始と同時に加熱が終了し、以降の（自分より遅い）相手の接触技には反応しなく
なる（一次情報と一致）。

### `ON_DAMAGE_HIT` priority=100（VolatileData、やけど付与）

同イベントに他の競合ハンドラがないため既定値のまま。

## 注意点・エッジケース

1. **ぼうごパット**: 接触技の使用者がぼうごパットを持つ場合、やけど効果は発動しない
   （`is_contact_reaction` が `ON_CHECK_CONTACT_REACTION` を経由して判定するため、既存の
   ぼうごパット実装により自動的に反映される）。
2. **やけど耐性（ほのおタイプ等）**: `ailment_manager.apply` 内の汎用免疫判定により、
   ほのおタイプの攻撃者はやけどにならない。
3. **みがわり**: くちばしキャノンの使用者自身がみがわり状態の場合、接触技をみがわりが
   肩代わりすると `_execute_hit` 内の `actual_damage <= 0` 早期returnにより
   `Event.ON_DAMAGE_HIT` 自体が発火しないため、やけど効果は発動しない（一次情報と一致）。
4. **相手の優先度がより低い場合**: カウンター（-5）・ドラゴンテール（-6）等、くちばしキャノン
   （-3）より優先度の低い技を相手が使う場合、使用者自身の `ON_TRY_ACTION`（priority=5）で
   加熱が既に終了しているため、やけど効果は発動しない（一次情報と一致）。
5. **まひ・ねむり・こおり・こんらん等で行動できない場合**: 加熱終了は行動可否判定より前の
   priority=5 で行われるが、加熱自体は `ON_BEFORE_MOVE`（行動順解決直後）で既に完了して
   いるため、使用者がこれらの状態で行動できない場合でもやけど効果は正しく発動する。
6. **ひんし時のやけど付与（解消済み）**: 使用者が接触技でひんしになった場合でも、
   `Event.ON_DAMAGE_HIT` はダメージ適用直後・ひんし判定より前に発火するため、やけど効果は
   正しく発動する（`test_くちばしキャノン_使用者がひんしになってもやけど効果は発動する`）。
7. **さいはい／まねっこ／オウムがえし**: 一次情報では「加熱動作は技の使用とみなされない」
   ため、これらの技で技Aの代わりにくちばしキャノンが選ばれることはない、という仕様がある。
   ただし `さいはい`／`まねっこ`／`オウムがえし` 自体が本実装では未実装のため、対応不要
   （それぞれの技を実装する際に、直前に使用した技を参照する既存ロジックで自然に満たされる
   想定）。
8. **テスト**: `tests/moves_attack/test_move_ka.py` の
   `test_くちばしキャノン_ほのおタイプの相手はやけどにならない` /
   `test_くちばしキャノン_ぼうごパットを持つ相手はやけどにならない` /
   `test_くちばしキャノン_まひで行動不能でもやけどは発動する` /
   `test_くちばしキャノン_みがわりで防いだ場合はやけどにならない` /
   `test_くちばしキャノン_接触技を受けると攻撃者がやけどになる` /
   `test_くちばしキャノン_使用者がひんしになってもやけど効果は発動する` /
   `test_くちばしキャノン_相手の優先度がより低い場合はやけどにならない` /
   `test_くちばしキャノン_非接触技を受けてもやけどにならない`
