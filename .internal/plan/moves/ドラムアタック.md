# ドラムアタック 実装計画書

## 仕様要約（.internal/spec/moves/ドラムアタック.md 参照）

* タイプ: くさ / 分類: 物理 / 威力: 80 / 命中: 100 / PP: 10（Gen9基準。Champions側PPは後述）
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果: 100%の確率で相手の『すばやさ』ランクを1段階下げる（追加効果）。

## PP（Champions基準）

`src/jpoke/data/ps-champ-ja/moves.json` にはドラムアタックの項目が存在しない（専用技のため一覧から漏れている）。
ただし `.internal/champions/changes_from_sv.md` の「すべての技のPPが8・12・16・20の4段階に統一」という規則と、
同一バッチの他技（とびかかる 15→16、ひやみず 20→20、げきりん 10→12、ドラゴンダイブ 10→12、
ドラゴンテール 10→12、りゅうのはどう 10→12 等）から読み取れる変換規則
（Gen9基準PPを4の倍数へ切り上げ: 5→8, 10→12, 15→16, 20→20）に従うと、
Gen9基準PP=10のドラムアタックはChampions基準でPP=12となる。
本レビューで `data/moves/move_ta.py` のPPを10→12に修正する。

## 現状の実装状況

`src/jpoke/data/moves/move_ta.py` に登録済み（レビューにより `secondary_effect` フラグの欠落とPP値を修正）。

```python
"ドラムアタック": MoveData(
    type="くさ",
    category="physical",
    pp=12,
    power=80,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ドラムアタック_lower_defender_spd,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 100%の確率で相手の『すばやさ』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def ドラムアタック_lower_defender_spd(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spe": -1})
```

`modify_defender_stats`（`handlers/move.py`）は `chance` 省略時に100%として扱い、
`battle.resolve_secondary_chance` を経由するため、ちからずくによる追加効果無効化（確率0化）は
汎用的に処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（ランク変化）は既存の同種ハンドラ（とびかかる・ひやみず・うらみつらみ 等）と
同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。`.internal/spec/turn.md` に
`ON_DAMAGE_HIT` は掲載されていないため、既存の同種ハンドラを参照して決定した。

## 注意点・エッジケース（review-test 向け）

1. **効果の対象は相手（防御側）のすばやさ**であり、自分（攻撃側）のランクではない。
   一次情報（ポケモンWiki）の説明文「こうげき することで あいての すばやさを さげる」の通り。
2. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` が必要。ちからずく所持時は `resolve_secondary_chance` が
     確率を0にし、すばやさランク低下が発動しなくなる（威力1.3倍化は `ちからずく_boost` 側で処理）。
3. **直接攻撃ではない**ため `contact` フラグは付与しない（ぼうおん等の接触依存特性・アイテムの対象外）。
4. **テスト方針**
   - `battle.actives[1].rank["spe"] == -1` を確認する（100%発動のため乱数固定は不要）。
