# 計画書: はいすいのじん

## 仕様要約

- タイプ: かくとう / カテゴリ: 変化 / PP: 5 / 対象: 自分 / まもる: 無視 / マジックコート: 無効
- 自分の こうげき・ぼうぎょ・とくこう・とくぼう・すばやさ ランクを1段階ずつ上げる
- 自分が『にげられない』状態になる（交代・逃げることができなくなる）
- はいすいのじん起因のにげられない状態をすでに持っている場合は失敗する（2回目以降失敗）
- すべての能力ランクがすでに+6の場合も失敗する（一般的な複合ランク変化技のルール）
- `MoveData.target` を明示的に `"self"` にする必要がある
  （デフォルト値 `"foe"` のままだと `is_blocked_by_protect` / `is_reflectable` に加え、
  `いたずらごころ`（あくタイプ無効化）・`おうごんのからだ`（変化技無効化）等の
  `ctx.move.target == "foe"` 判定にも誤って合致してしまう）
- 自分自身が発生源のにげられない状態のため、相手が場を離れても解除されない
  （`にげられない`共通ハンドラの修正が必要。詳細は「既存バグの修正」参照）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 | 登録場所 |
|---|---|---|---|---|
| `ON_TRY_MOVE_1` | 30 | `attacker:self` | はいすいのじん起因のにげられない状態ならば失敗（`.internal/spec/turn.md` 明記） | `data/moves/move_ha.py` |
| `ON_STATUS_HIT` | 100（標準） | `attacker:self` | 全5能力+1段階 + にげられない付与（すでに他起因でにげられない状態なら付与スキップ） | `data/moves/move_ha.py` |

全能力ランクが+6で1つも上げられない場合の失敗は、`battle.modify_stats()` の戻り値が
空辞書になることを利用し、`ON_STATUS_HIT` ハンドラが `HandlerReturn(value=False)` を返すことで
`move_executor._execute_status_hit` が自動的に `MOVE_FAILED` ログを出す仕組みに委譲する
（`ちょうのまい_modify_attacker_stats` 等、既存の複合ランク変化技と同じパターン）。
そのため `ON_TRY_MOVE_1` の失敗チェックには「全能力ランクが+6」の条件を含めない
（含めると `modify_attacker_stats` 経由の自動失敗と二重判定になり、`ON_TRY_MOVE_1` と
`ON_STATUS_HIT` のどちらで失敗判定すべきかという役割分担が不明瞭になるため）。

## Priority 根拠

- `ON_TRY_MOVE_1` priority 30: `.internal/spec/turn.md` に「30 | はいすいのじん: すでに状態変化」と
  明記されている。
- `ON_STATUS_HIT` は `.internal/spec/turn.md` に掲載がないイベント（技実行フロー内部の状態変化技専用
  イベント）。既存の同種ハンドラ（`ちょうのまい_modify_attacker_stats`、`とぐろをまく_modify_attacker_stats`、
  `ソウルビート_pay_hp_and_modify_attacker_stats`）が標準 priority（指定なし）で登録されているため、
  それに合わせる。

## 実装箇所

### `handlers/move_status.py`

「は」行の適切な位置（`はいすいのじん_apply` → `はいすいのじん_can_apply` の順、五十音順を維持）。

```python
def はいすいのじん_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """はいすいのじんの効果: すべての能力を1段階ずつ上げる。

    すべての能力ランクがすでに+6で1つも上げられない場合は失敗する（他の複合ランク
    変化技と同様、battle.modify_stats() の戻り値が空辞書になることで自動的に失敗
    扱いになる）。
    すでに他の要因でにげられない状態の場合は、にげられない付与をスキップする。
    """
    mon = ctx.attacker
    result = battle.modify_stats(mon, {"atk": 1, "def": 1, "spa": 1, "spd": 1, "spe": 1}, source=mon)
    if not result:
        return HandlerReturn(value=False)
    if not mon.has_volatile("にげられない"):
        battle.volatile_manager.apply(mon, "にげられない", source=mon, move_name="はいすいのじん")
    return HandlerReturn(value=value)


def はいすいのじん_can_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """はいすいのじんの失敗条件チェック（ON_TRY_MOVE_1 priority=30）。

    すでにはいすいのじんによってにげられない状態になっている場合は失敗する。
    """
    mon = ctx.attacker
    if (mon.has_volatile("にげられない")
            and mon.volatiles["にげられない"].move_name == "はいすいのじん"):
        battle.add_event_log(
            mon, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="はいすいのじん_すでに状態変化"),
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

## 既存バグの修正: `にげられない_remove_on_foe_switch`

`handlers/volatile.py` の `にげられない_remove_on_foe_switch`（`にげられない` volatile の
`ON_SWITCH_OUT` ハンドラ、`subject_spec="source:foe"`）は、元々「にげられない状態を与えた相手
（発生源）が場を離れたら解除する」という、とおせんぼう・くらいつく等の **相手を対象とする**
にげられない付与技のための処理だった。

はいすいのじんは自分自身が発生源かつ対象（`source=mon` で自分に付与）のため、この共通ハンドラを
そのまま適用すると「対戦相手が交代しただけ」で誤ってはいすいのじん起因のにげられない状態が
解除されてしまうバグがあった（一次情報「使用時に相手の場にいたポケモンが場から去っても、
この技によるにげられない状態は解除されない」に反する）。

`Volatile.move_name` が `"はいすいのじん"` の場合はスキップするガードを追加して修正した。

```python
def にげられない_remove_on_foe_switch(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    foe = battle.foe(ctx.source)
    if not foe.has_volatile("にげられない"):
        return HandlerReturn(value=value)
    if foe.volatiles["にげられない"].move_name == "はいすいのじん":
        return HandlerReturn(value=value)
    battle.volatile_manager.remove(foe, "にげられない")
    return HandlerReturn(value=value)
```

くらいつく等の相手を対象とする付与では `move_name` が設定されない（`apply()` 時に
`move_name` を渡していない）ため、この修正による影響はない。

## 注意点・エッジケース（review-test 向け）

### 2回目使用時の失敗

- `ON_TRY_MOVE_1` priority=30 でチェックし、`stop_event=True` で中断する。
  `move.md` の仕様通り、はいすいのじん起因のにげられない状態のときのみ失敗し、他の要因（例:
  くろいまなざし等）によるにげられない状態では失敗しない（ランクは通常通り上昇する）。

### 全能力ランク+6での失敗

- `ON_STATUS_HIT` の `はいすいのじん_apply` 内で `battle.modify_stats()` の戻り値が空辞書になる
  ことで自動的に失敗する。にげられない状態も付与されない（ランクも状態変化もどちらも発生しない、
  一般的な複合ランク変化技の全失敗パターン）。

### target="self" の重要性

- `is_blocked_by_protect` / `is_reflectable` が `False` になることに加え、`いたずらごころ`
  （あくタイプの相手に優先度技を無効化する特性）・`おうごんのからだ`（他のポケモンからの
  変化技を無効化する特性）等、`ctx.move.target == "foe"` を前提とする特性ハンドラの誤爆も防ぐ。

### にげられない状態の交代解除に関する既存バグ修正

- 上記「既存バグの修正」を参照。`tests/volatiles/test_volatile_satana.py` に
  回帰テストを追加。
