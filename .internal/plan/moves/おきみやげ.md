# 計画書: おきみやげ

## 仕様要約

- タイプ: あく / カテゴリ: 変化 / PP: 12（チャンピオンズ仕様。本家 SV 基準は 10） / 命中: 100 / 対象: 1体選択（相手）
- 使用者が自分のHPを0にしてひんしになり、相手のこうげき・とくこうをそれぞれ2段階下げる
- 第五世代以降: 命中しなかった場合・まもる/みがわりで防がれた場合は使用者はひんしにならない
- 第五世代以降: こうげき/とくこうランクが共に最低でも、ランク低下は失敗するが使用者はひんしになる
- クリアボディ等でランク低下が防がれた場合でも使用者はひんしになる
- まもる: ○（防がれる）/ マジックコート: ×（反射されない）/ よこどり: ×
- Zワザ版: 控えから出るポケモンのHPを全回復（対象外。本プロジェクトはZワザ非対応）

|----------|----------|--------------|------|----------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（`MoveHandler` デフォルト） | 使用者をひんし化・相手のこうげき/とくこうを2段階下げる | `handlers/move_status.py` |

## Priority 根拠

`.internal/spec/turn.md` には `ON_STATUS_HIT` 内部の priority 表がない。既存の変化技ハンドラ（`おいかぜ_set_side_field`、`おたけび_modify_defender_stats`、`いやしのねがい_apply` など）がすべて `MoveHandler` のデフォルト priority 100 を使用している。おきみやげも同一パターンとして priority 100 を採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: 五十音順「お」行 — `おかたづけ_cleanup` の後、`おたけび_modify_defender_stats` の前に挿入する（実装済み）。

```python
def おきみやげ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """おきみやげ: 使用者をひんしにし、相手のこうげき・とくこうを2段階ずつ下げる。"""
    battle.faint(ctx.attacker)
    modify_defender_stats(battle, ctx, value, stats={"atk": -2, "spa": -2})
    return HandlerReturn(value=value)
```

技ハンドラは `handlers/move.py` の `MoveHandler` 基盤を経由するため、`ON_STATUS_HIT` であっても
コンテキストは常に `AttackContext`（`attacker` / `defender` を直接参照できる）である点に注意する
（`EventContext` ではない）。

### `src/jpoke/data/moves/move_a.py`

```python
"おきみやげ": MoveData(
    type="あく",
    category="status",
    pp=12,
    accuracy=100,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.おきみやげ_apply,
        ),
    }
),
```

## 注意点・エッジケース

- **ひんし化のタイミング**: `ON_STATUS_HIT` は命中判定・まもる/みがわり判定を通過した後に発火する。第五世代以降の「命中しなかった場合/まもる・みがわりで防がれた場合はひんしにならない」という仕様は、ON_STATUS_HIT が呼ばれない（それ以前のフェーズで技が失敗する）ことで自然に実現される。ハンドラ側での追加ガードは不要
- **ランク最低でもひんし化する**: 第五世代以降「こうげき/とくこうランクが共に -6 でも使用者はひんしになる」。`battle.faint(ctx.attacker)` を `modify_defender_stats` より先に呼ぶことで、ランク低下の成否に関わらずひんし化が保証される
- **クリアボディ等でランク低下が防がれた場合も使用者はひんし**: wiki仕様「クリアボディ等により能力低下が無かった場合でも使用者はひんしになる」。`battle.faint` を `modify_defender_stats` より先に呼ぶことで保証される
- **`battle.faint` を使う**: `battle.faint(target)` は内部で `battle.modify_hp(target, v=-target.max_hp)` を呼ぶユーティリティであり、いやしのねがい（`battle.faint(mon)`）と同一パターン
- **戻り値は `value` を維持する**: `modify_defender_stats` の戻り値（ランク低下が特性等で完全に阻まれた場合に `False` 相当になり得る）を採用せず、常に受け取った `value`（技自体の成否フラグ）をそのまま返す。これにより、ランク低下が失敗しても「おきみやげ」自体は成功として扱われる（使用者はひんしになっているため）
- **`target` フィールドのデフォルト**: MoveData の `target` デフォルトは `"foe"` であり、おきみやげは相手1体が対象のためデフォルトのまま変更不要
- **すてゼリフとの違い**: すてゼリフは使用者が「交代」するのに対し、おきみやげは「ひんし」になる点が異なる
- **Zおきみやげ**: 控えポケモンのHPを全回復する効果があるが、Zワザは現状対象外のためハンドラには含めない
- **PP は 12**: `src/jpoke/data/ps-champ-ja/moves.json` に記載の値（チャンピオンズ仕様で全技PPが8/12/16/20に統一されているため、本家 SV の10から変更されている）
