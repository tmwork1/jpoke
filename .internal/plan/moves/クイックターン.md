# クイックターン 実装計画書

## 仕様要約（.internal/spec/moves/クイックターン.md 参照）

* タイプ: みず / 分類: 物理 / 威力: 60 / 命中: 100 / PP: 20
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 攻撃後、使用者が手持ちのポケモンと交代する（追加効果とはみなされない）。
* とんぼがえり・ボルトチェンジと全く同じ効果のわざ。

## 現状の実装状況

`src/jpoke/data/moves/move_ka.py` に既に登録済み。

```python
"クイックターン": MoveData(
    type="みず",
    category="physical",
    pp=20,
    power=60,
    accuracy=100,
    flags={"contact"},
    handlers={
        Event.ON_HIT: h.MoveHandler(ha.pivot)
    }
),
```

内容は仕様書（src/jpoke/data/ps-champ-ja/moves.json: `クイックターン みず 物理 60 100 20 接○ 守○`）と一致しており、修正不要。

## ハンドラ構成

| Event | 関数 | subject_spec | priority | 役割 |
|---|---|---|---|---|
| ON_HIT | `pivot` | attacker:self | 100（デフォルト） | 交代可能な控えがいれば `player_states[player].interrupt = Interrupt.PIVOT` を設定 |

`pivot`（`handlers/move_attack.py`）は `とんぼがえり`・`ボルトチェンジ` と共通の実装であり、専用の新規関数は不要。
実際の交代処理（プレイヤーが控えを選択して入れ替える）は `SwitchManager.run_interrupt_switch(Interrupt.PIVOT)`
（`turn_controller.py` の `_run_move_phase` 内、だっしゅつボタン・ききかいひの後、だっしゅつパックの前に実行）
が担う共通インフラであり、本技側の実装には含まれない。

みずタイプ技であるため、特性かんそうはだ・ちょすい・よびみずによる無効化は
`_apply_type_absorb`（`handlers/ability.py` の `かんそうはだ_absorb_water` / `ちょすい_absorb_water` /
`よびみず_absorb_water`）が既存の共通処理として担っており、無効化時は `Event.ON_HIT` 自体が発火しないため
交代も発動しない。本技固有の対応は不要。

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_HIT` 節（ダメージ発生後の処理、みがわりに被弾しても発動）を確認した。
一覧に交代技専用の行は明記されていないが、既存の `とんぼがえり`・`ボルトチェンジ` も同じ `pivot` 関数を
デフォルト priority（100）で `Event.ON_HIT` に登録しており、他の priority=100 の処理（コアパニッシャー・
なげつける等）とは独立した状態変更（`interrupt` フラグ設定のみ）であるため相互作用がなく、priority の
新規検討は不要。既存パターンを踏襲しデフォルト値のままとする。

## 注意点・エッジケース（review-test 向け）

1. **控えがいない場合**
   - `pivot` は `battle.command_manager.get_available_switch_commands(player)` が空の場合、
     interrupt を設定しない。ダメージは通常通り与えられるが交代は発動しない。実装済み・動作確認済み。

2. **まもる・命中しなかった場合**
   - `Event.ON_BEFORE_APPLY_MOVE` で無効化される、または命中判定に失敗した場合は
     `Event.ON_HIT` 自体が発火しないため、交代も発動しない。既存の move_executor.py の
     フロー（本技固有の実装ではない）に依存しており、対応済み。

3. **かんそうはだ・ちょすい・よびみずで無効化された場合**
   - `_apply_type_absorb` がみずタイプ技を吸収する際に `Event.ON_HIT` 自体を止める
     （無効化として扱う）ため、`pivot` は発火せず交代も発動しない。とんぼがえり・ボルトチェンジには
     ないクイックターン固有の判定ポイントだが、既存の共通吸収特性実装に依存しており対応済み。

4. **対象を倒した場合の交代順序**
   - `Event.ON_HIT` は攻撃側の interrupt を設定し、`_run_move_phase` 内で
     攻撃側の PIVOT 交代→（ターン終了時の）防御側の瀕死交代、の順に処理されるため、
     「クイックターンの交代先を見せた後に相手が次のポケモンを選択する」という仕様と一致する。
     とんぼがえりで動作確認済みのフローと共通のため、本技固有の再検証は不要。

5. **レッドカード・だっしゅつボタン・にげごし/ききかいひとの相互作用**
   - これらの防御側アイテム・特性は `Event.ON_DAMAGE_HIT`（レッドカードは実ダメージ0の場合のみ
     `Event.ON_HIT`）で `battle.run_switch(...)` を直接呼び出し、防御側プレイヤーの交代を
     即座に実行する。攻撃側の `interrupt = Interrupt.PIVOT` フラグとは別プレイヤーの状態を
     操作するため、通常のケース（実ダメージ>0）では相互に干渉しない。とんぼがえりと共通の
     既存実装のまま対応不要。
