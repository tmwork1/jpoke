# 計画書: ニードルガード

## 仕様要約

- タイプ: くさ / カテゴリ: 変化 / PP: 8（champions基準。src/jpoke/data/ps-champ-ja/moves.json 参照） / 命中: なし（必中） / 対象: 自分 / 優先度: +4
- 使用ターン中、相手の攻撃技・変化技を防ぐ
- 接触技（直接攻撃）で攻撃してきた相手に、相手の最大HPの1/8ダメージを与える
- まもる系共通の連続使用失敗ルールが適用される（`まもる系_連続使用失敗チェック`、`Event.ON_TRY_MOVE_2` デフォルト priority）
- `"ニードルガード"` volatile を自分に付与し、`ON_TRY_MOVE_1` で攻撃を無効化しつつ接触カウンターダメージを与える
- ターン終了時に volatile を解除する

詳細は .internal/spec/moves/ニードルガード.md・.internal/spec/volatiles/ニードルガード.md を参照。

## Priority 根拠

### `ON_TRY_MOVE_2`: priority 100（デフォルト）
`.internal/spec/turn.md` の `Event.ON_TRY_MOVE_1` 30番に「まもる系: 連続使用により失敗」と記載があるが、
既存の守る系技（まもる・みきり・こらえる・キングシールド・トーチカ等）がすべて `Event.ON_TRY_MOVE_2`
のデフォルト priority で実装済みのため、同一パターンに合わせる。理由は .internal/spec/moves/まもる.md 参照。

### `ON_STATUS_HIT`: priority 100（デフォルト）
`.internal/spec/turn.md` に `ON_STATUS_HIT` の内部 priority 表がない。既存の変化技ハンドラ（`まもる_apply`・
`トーチカ_protect` を付与する各技のハンドラ・`キングシールド_apply` 等）がすべて priority 100（デフォルト）
を使用しているため、同一パターンで 100 を採用する。

### `ON_TRY_MOVE_1`: priority 100
`.internal/spec/turn.md` の `Event.ON_TRY_MOVE_1` セクションに明記：

```
| 100 | まもる・キングシールド等による無効化 |
```

既存の `まもる` / `キングシールド` / `トーチカ` volatile ハンドラがすべて priority 100 を明示しているため、同一の 100 を採用する。

### `ON_TURN_END`: priority 100（デフォルト）
`.internal/spec/turn.md` に `ON_TURN_END` の「まもる系 volatile 解除」の記載はない。既存の `まもる_remove_volatile`・
`かえんのまもり_remove_volatile`・`キングシールド_remove_volatile`・`トーチカ_remove_volatile` 等が
すべてデフォルト priority を使用しているため、同一パターンで 100 を採用する。

### 2. `src/jpoke/handlers/volatile.py` — `_run_protect` の拡張

`_run_protect` は `chip_on_contact: float | None = None` パラメータを持ち、接触時に攻撃者へ
固定割合ダメージを与える処理を行う。接触判定は `battle.query.is_contact_reaction(ctx)` を使用する
（`ぼうごパット` 等の反応阻害要因を考慮した判定。かたいツメ等「自分の技が接触技であること」に
由来する効果の判定に使う `is_contact` とは区別する。.internal/spec/volatiles/トーチカ.md も同じ関数を使用）。

```python
def _run_protect(battle: Battle,
                 ctx: EventContext,
                 value: Any,
                 stats_change_on_contact: dict[Stat, int] | None = None,
                 ailment_on_contact: AilmentName | None = None,
                 chip_on_contact: float | None = None,
                 protect_non_attack: bool = True) -> HandlerReturn:
    if not _check_protect_success(battle, ctx, protect_non_attack):
        battle.add_event_log(ctx.defender, LogCode.PROTECT_FAILED, ...)
        return HandlerReturn(value=value)

    battle.add_event_log(ctx.defender, LogCode.PROTECT_SUCCEEDED, ...)

    if battle.query.is_contact_reaction(ctx):
        if stats_change_on_contact:
            battle.modify_stats(ctx.attacker, stats_change_on_contact, source=ctx.defender)
        if ailment_on_contact:
            battle.ailment_manager.apply(ctx.attacker, ailment_on_contact, source=ctx.defender)
        if chip_on_contact is not None:
            battle.modify_hp(ctx.attacker, r=-chip_on_contact, reason="")

    return HandlerReturn(value=False, stop_event=True)
```

`chip_on_contact` の `modify_hp` は `reason=""`（その他のダメージ）で呼び出す。これは
`ON_MODIFY_NON_MOVE_DAMAGE` を経由するため、マジックガード特性で無効化される
（一次情報の「相手の特性がマジックガードの場合はニードルガードによるダメージを受けない」に合致）。

## 注意点・エッジケース

### まもる系の連続使用失敗

`_run_protect` 自体は連続使用判定を行わない。連続使用失敗は技本体の `Event.ON_TRY_MOVE_2` に登録した
`まもる系_連続使用失敗チェック`（`executed_move` が `protect` フラグを持つ技かどうかで判定する簡略化
モデル）が担う。ニードルガードもこのパターンに従う。

### `chip_on_contact` 追加による既存 `_run_protect` への影響

`_run_protect` は複数の守る系技（まもる・かえんのまもり・キングシールド・スレッドトラップ・トーチカ・
ニードルガード）で共有される。`chip_on_contact=None`（デフォルト）では既存動作と同一になるため、
ニードルガード追加による他技への影響はない。

### 接触判定は `battle.query.is_contact_reaction(ctx)` で行う

さめはだ・てつのトゲ・トーチカのどく付与等の接触カウンター系と同じ関数を使用する。ぼうごパットを
持つ攻撃者はこの判定が False になるため、ニードルガードのダメージも発生しない
（一次情報の「相手がぼうごパットを持つ場合はニードルガードによるダメージを受けない」に合致）。

### ダイマックス技・Zワザは対象外

一次情報の「ダイマックス技や第7世代のZワザの攻撃技は貫通し、1/4のダメージを受ける」は、
本プロジェクトの対象範囲（第9世代・champions基準）にダイマックス・Zワザが存在しないため対応不要。

### 変化技への保護

ニードルガードはデフォルト（`protect_non_attack=True`）で変化技も防ぐ。「なみだめ」・「ふきとばし」・
「ほえる」がニードルガードを貫通する仕様は、それらの技側で `is_blocked_by_protect=False` ラベル等に
より管理されるため、ニードルガードハンドラ側では対応不要。

### フェイント等による解除

フェイント・ゴーストダイブ・シャドーダイブ・いじげんホール・いじげんラッシュは `_FAINT_REMOVE_VOLATILES`
（`handlers/move_attack.py`）に `"ニードルガード"` を含めて実装済みで、ニードルガード状態を取り除いて
ダメージを与える（接触カウンターダメージは発生しない）。
