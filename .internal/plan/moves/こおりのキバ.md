# こおりのキバ 実装計画書

## 仕様要約（.internal/spec/moves/こおりのキバ.md 参照）

* タイプ: こおり / 分類: 物理 / 威力: 65 / 命中: 95 / PP: 15
* 対象: 相手1体 / 優先度: 0 / 接触あり（直接攻撃）
* 効果: 追加効果として、10%の確率で相手を『こおり』状態にし、10%の確率で相手をひるませる。
  2つの効果はそれぞれ独立に判定される（同時発生し得る）。
* かみつき技（`flags` に `"bite"` を含む）。特性『がんじょうあご』の対象。

## 現状の実装状況（レビューにより修正）

修正前は単一乱数（`r < base*0.5` → こおり、`r < base` → ひるみ）で排他的に判定しており、
「2つの追加効果はそれぞれ個別に判定される」という仕様（同時発生し得る）に反していた。
`かみなりのキバ` と同じパターン（こおり付与・ひるみ付与をそれぞれ独立した `MoveHandler` で登録）に修正した。

```python
"こおりのキバ": MoveData(
    type="こおり",
    category="physical",
    pp=15,
    power=65,
    accuracy=95,
    flags={"bite", "contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: [
            h.MoveHandler(ha.こおりのキバ_apply_freeze),
            h.MoveHandler(ha.こおりのキバ_apply_flinch),
        ]
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手を『こおり』状態にする |
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手をひるませる |

`src/jpoke/handlers/move_attack.py` の実装:

```python
def こおりのキバ_apply_flinch(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_defender(battle, ctx, value, volatile="ひるみ", chance=0.1)


def こおりのキバ_apply_freeze(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="こおり", chance=0.1)
```

`apply_ailment_to_defender` / `apply_volatile_to_defender`（いずれも `handlers/move.py`）が
`battle.resolve_secondary_chance` を経由するため、ちからずくによる追加効果無効化（確率0化）・
威力1.3倍化、`flags={"bite"}` によるがんじょうあごの威力1.5倍化は、いずれも汎用ハンドラ側で
処理される。本技側に個別のガードは不要。こおりとひるみは別々の `MoveHandler` として登録されており、
それぞれ独立に `battle.random.random()` で抽選されるため、仕様どおり同時発生し得る。

## Priority 根拠

ダメージを与えた技の追加効果（状態異常・ひるみ付与）は既存の同種ハンドラ（かみなり・かみなりパンチ・
かみなりのキバ 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。個別の priority
検討は不要。

## 実装箇所

### handlers/move_attack.py

`こおりのキバ_apply_flinch_or_freeze`（単一乱数で排他判定していた誤実装）を削除し、
`こおりのキバ_apply_freeze` / `こおりのキバ_apply_flinch` の2関数に分割した。

### data/move.py

`こおりのキバ` の `handlers[Event.ON_DAMAGE_HIT]` を単一の `MoveHandler` からリスト
（こおり付与・ひるみ付与の2つ）に変更した。

## 注意点・エッジケース（review-test 向け）

1. **がんじょうあごとの相互作用**
   - `flags={"bite"}` により、がんじょうあご所持ポケモンが使用すると威力が1.5倍（65→97）になる。
     既存の がんじょうあご テストファイル側で他のかみつき技と同様の網羅がされていれば個別追加は不要。
2. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・こおり/ひるみ付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
3. **おうじゃのしるし・するどいキバとの相互作用**
   - `handlers/item.py` の `_INNATE_FLINCH_MOVES` に「こおりのキバ」が既に含まれており、
     元々ひるみ追加効果を持つ技として、おうじゃのしるし等による追加のひるみ判定が重複しないよう
     処理済み（修正不要）。
4. **2つの追加効果の独立性**
   - こおりとひるみはそれぞれ個別に `battle.random.random()` で抽選されるため、両方同時に発動する
     可能性がある。修正前のテストは排他制御を前提としていたため誤りであり、`かみなりのキバ` と
     同様に「こおり付与」「ひるみ付与」をそれぞれ個別に確認するテストへ修正した。
5. **テスト方針**
   - `secondary_chance` を1に固定し、こおり・ひるみそれぞれの付与有無を確認するテストへ修正
     （`test_こおりのキバ_こおりが発動する` / `test_こおりのキバ_ひるみが発動する`）。
     排他制御を前提とした `test_こおりのキバ_同時発動しない` は仕様に反するため削除する。
