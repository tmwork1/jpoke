# 計画書: フラフラダンス

## 仕様要約

- ノーマルタイプ / 変化 / 命中100 / PP20 / 対象: 自分以外全体（シングルでは相手1体）
- 相手を2〜5ターンの間『こんらん』状態にする（`apply_confusion` の仕様に準拠）
- ダンス技（flags: dance）
- マジックコート・マジックミラーの影響を受けない（flags: unreflectable）
- マイペース特性には無効（フレームワーク側が処理）
- こんらん済みの場合は失敗（フレームワーク側が処理）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | 相手にこんらん付与 |

## Priority 根拠

てんしのキッス・あやしいひかりと同種の `ON_STATUS_HIT` priority 100 を採用する。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `フェザーダンス` の後、`ふしょくガス` の前（五十音順）。

```python
def フラフラダンス_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """フラフラダンスの効果: 相手をこんらん状態にする。"""
    return HandlerReturn(value=battle.volatile_manager.apply_confusion(
        ctx.defender, source=ctx.attacker
    ))
```

`battle.volatile_manager.apply_confusion()` は内部で2〜5ターンのランダム値を生成するため、
ハンドラ側で自前に `randint` を呼ぶ必要はない（`あやしいひかり`・`てんしのキッス` と同じ実装）。

### data/moves/move_ha.py への更新

`"フラフラダンス"` エントリのハンドラを追加する。

```python
    "フラフラダンス": MoveData(
        type="ノーマル",
        category="status",
        pp=20,
        accuracy=100,
        flags={"dance", "unreflectable"},
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.フラフラダンス_apply,
            ),
        }
    ),
```

## 注意点・エッジケース

- てんしのキッス・あやしいひかりと全く同じ実装パターン（`apply_confusion` ヘルパーを使用）
- ダンス技であるため、おどりこ特性のポケモンにコピーされる（フレームワーク側が処理。
  おどりこ自体は本プロジェクトで未実装 = 保留のため実装上の対応は不要）
- シングルバトルでは事実上1体対象として動作する
- 他のこんらん付与変化技と異なり、マジックコート・マジックミラーで跳ね返されないため
  `flags` に `unreflectable` を含める（`model/move.py` の `is_reflectable` 判定を参照）
- こんらん済み判定・マイペース無効はいずれもフレームワーク側が処理するためハンドラ内ガード不要
