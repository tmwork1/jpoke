# 実装計画: いたみわけ

## 仕様サマリ

- タイプ: ノーマル / 分類: 変化 / PP: 20 / 命中率: 必中（`accuracy=None`）
- 効果: 自分と相手の現在HPを合計し、2で割った値（端数切り捨て）に均等化する
- 最大HPを超えて回復することはできない（`battle.modify_hp` が自動的にクランプする）
- まもる: ○ / マジックコート: × / おうじゃのしるし: × / 直接攻撃: ×
- HP回復技として分類されない（データ上 `heal` フラグを持たない扱い）ため、以下が成立する
  - かいふくふうじ状態でも使用・回復ができる
  - マジックガードで無効化されない（HP減少側も無効化対象外）
  - みがわり状態の相手には防がれる（第六世代以降、すりぬけなら貫通）
  - おうごんのからだ特性を持つ相手には防がれる（他の変化技と共通の汎用処理）
- 対象: 相手1体

## 現状の実装状況

`data/moves/move_a.py` にエントリが存在し、`handlers/move_status.py` に
`いたみわけ_equalize_hp` が実装済み。

```python
"いたみわけ": MoveData(
    type="ノーマル",
    category="status",
    pp=20,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.いたみわけ_equalize_hp,
        )
    }
),
```

```python
def いたみわけ_equalize_hp(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """両者の現在HPを平均化する。"""
    shared_hp = (ctx.attacker.hp + ctx.defender.hp) // 2
    for mon in (ctx.attacker, ctx.defender):
        battle.modify_hp(mon, v=shared_hp - mon.hp, reason="pain_split")
    return HandlerReturn(value=value)
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自分と相手のHPを均等化する |

## Priority 根拠

`.internal/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の同種変化技ハンドラ（`いとをはく_modify_defender_stats` など）が priority=100
デフォルトで実装されているため、同様に採用する。

## 注意点・エッジケース

- `shared_hp` は攻守どちらの `modify_hp` 呼び出しよりも前に一度だけ計算するため、
  攻撃側→防御側の順で `modify_hp` を適用しても結果は変わらない
- `shared_hp` が対象の最大HPを超える場合でも、`battle.modify_hp` 内部で
  `Pokemon.modify_hp` がHPを `0〜max_hp` にクランプするため、追加のガードは不要
- まもる／みがわり／おうごんのからだ等の無効化は変化技共通のフレームワーク処理
  （`move_executor.check_hit_substitute` など）に委譲されるため、本ハンドラ内に
  重複実装は不要
- ダイマックス相手のHP参照補正（ダイマックスレベルによる除算）は本プロジェクトの
  対象範囲（チャンピオンズシングル）に存在しない仕様のため実装不要
