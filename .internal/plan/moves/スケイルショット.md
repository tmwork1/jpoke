# スケイルショット 実装計画書

## 仕様要約（.internal/spec/moves/スケイルショット.md 参照）

* タイプ: ドラゴン / 分類: 物理 / 威力: 25（1ヒットあたり） / 命中: 90 / PP: 20
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果:
  * 2～5回連続で攻撃する可変回数の連続攻撃技（命中判定は1発目のみ、各ヒット威力は固定25）。
  * 急所判定は各ヒットごとに独立して行われる。
  * **全ての攻撃が終了した後**、当たった回数に関係なく使用者自身の `ぼうぎょ` -1・`すばやさ` +1。
    技が外れた／無効化された場合は発動しない。あまのじゃく・たんじゅんは
    `battle.modify_stats` 経由の汎用ハンドラで自動的に反映される。
    追加効果扱いではないため、ちからずく/りんぷん/おんみつマントの影響を受けない
    （`flags` に `secondary_effect` を含めないことで表現する）。

## 現状の実装状況

`src/jpoke/data/moves/move_sa.py` に既に登録済み。

```python
"スケイルショット": MoveData(
    type="ドラゴン",
    category="physical",
    pp=20,
    power=25,
    accuracy=90,
    multi_hit={
        "min": 2,
        "max": 5,
        "check_hit_each_time": False,
        "power_sequence": (),
    },
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.スケイルショット_apply_stat_change,
        )
    }
),
```

* `flags` を持たない（＝ `contact` を含まない）ため直接攻撃技ではない。仕様書の「接触なし」と一致。
* `flags` に `secondary_effect` を含まないため、ちからずく/りんぷん/おんみつマントの対象にならない。
  仕様書の「追加効果として扱われない」という要件を満たす。
* `multi_hit.min=2, max=5` かつ `check_hit_each_time` を指定しないため、`_resolve_hit_count` が
  `MULTI_HIT_DISTRIBUTION_2_TO_5` に従ってヒット回数を決定し、命中判定は1発目のみ行われる
  （`スイープビンタ` と同一パターン）。
* `power_sequence` が空タプルのため、`_resolve_hit_power` は各ヒットとも `move.power`（25）を返す。

### レビューで発見したバグと修正

`handlers/move_attack.py` の `スケイルショット_apply_stat_change` は修正前、

```python
if ctx.hit_index != ctx.hit_count:
    return HandlerReturn(value=value)
```

という条件で「最後のヒットかどうか」を判定していた。これは `ctx.hit_count`
（乱数で事前に決定された予定ヒット数、例: 4）と `ctx.hit_index`（現在のヒット番号）を
比較するものだが、**連続ヒットの途中で相手がひんしになった場合**
（例: 4ヒット予定のうち2ヒット目で相手が倒れて残り2ヒットが行われない）、
`ctx.hit_index`（2）が `ctx.hit_count`（4）に到達しないままループが終了するため、
この条件では自己ランク変化が一切発動しない不具合があった。

仕様書に明記した通り「全ての攻撃が終了した後、当たった回数に関係なく」発動するのが正しい
挙動であり（Pokemon Showdown の `selfBoost` 実装でも、多段技の実際のヒット数に関わらず
技全体の成否（`moveResult`）のみを条件に1回だけ適用される）、相手がひんしになって
残りのヒットが行われなかった場合でも発動しなければならない。

修正後の条件:

```python
if ctx.hit_index != ctx.hit_count and not ctx.defender.fainted:
    return HandlerReturn(value=value)
```

`ctx.defender.fainted` を追加のOR条件にすることで、
「最後の予定ヒット」または「相手がひんしになり連続ヒットが打ち切られたヒット」の
いずれかで発動するようになる。`ON_DAMAGE_HIT` は `actual_damage > 0` の場合のみ発火し
（技が外れた・無効化された場合は発火しない）、`core/move_executor.py` の
`_execute_hit` は当該ヒット処理の最後に `ON_DAMAGE_HIT` を発火してから
ひんし判定でループを打ち切るため、この修正で意図した挙動になる。

## ハンドラ構成

| イベント | 対象 | 関数 | 概要 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | attacker:self | `スケイルショット_apply_stat_change` | 最終ヒット（または相手ひんしによる打ち切り時点）で自分の `ぼうぎょ` -1・`すばやさ` +1 |

## Priority 根拠

`ON_DAMAGE_HIT` は `.internal/spec/turn.md` に未掲載のイベント（ダメージ計算内部の処理）。
同種の既存ハンドラ（`スケイルノイズ_lower_attacker_def` の `ON_HIT` 登録や、
`へんしょく_copy_move_type` の `ON_DAMAGE_HIT` 登録など）がいずれも priority 未指定
（デフォルト値 100）であるため、本ハンドラも priority を指定せずデフォルト値を用いる。
他の同一ヒット内ハンドラとの実行順序に依存しない自己完結的な処理のため問題ない。

## 注意点・エッジケース（review-test 向け）

1. **ヒット回数は2～5回の乱数分布**
   - `_resolve_hit_count` の戻り値が2〜5の範囲に収まることを確認するテスト
     （既存の `test_スケイルショット_複数ヒットする`）。
2. **命中判定は1発目のみ**
   - 1発目が外れた場合は残りのヒットも発生せず、自己ランク変化も発動しない。
3. **通常ケース: 全ヒット命中後のランク変化**
   - 既存テスト `test_スケイルショット_最終ヒット時にぼうぎょマイナス1_すばやさプラス1` で確認済み。
4. **エッジケース: 相手がひんしになり連続ヒットが打ち切られた場合**
   - `defender.hp = 1` を設定して1ヒット目で相手を確実にひんしにさせ、それでも
     `attacker.rank["def"] == -1` かつ `attacker.rank["spe"] == 1` になることを確認するテストを追加する。
   - 修正前の実装ではこのケースでランク変化が発動しないバグがあったため、
     テストで確実に検出できるようにする。
5. **あまのじゃく・たんじゅん**
   - `battle.modify_stats` → `ON_BEFORE_MODIFY_STAT` の汎用ハンドラで自動的に反映されるため、
     本技専用の実装は不要（既存の特性テストで担保される範囲）。
