# マジカルアクセル 実装計画書

## 仕様要約（.internal/spec/moves/マジカルアクセル.md 参照）

* タイプ: フェアリー / 分類: 物理 / 威力: 100 / 命中: 100 / PP: 12（Champions基準、本家PP10からの変換）
* 対象: 相手1体 / 優先度: 0 / **接触なし**（物理技だが直接攻撃ではない）
* 効果: 追加効果として30%の確率で相手を「こんらん」状態にする。
* この技に対する アンコール・まねっこ は失敗する（`non_encore` / `non_copycat` フラグで表現）。
  ものまね・さいはいは本プロジェクト未実装のため対応フラグなし。

## 現状の実装状況（レビュー前）

`src/jpoke/data/moves/move_ma.py` に既に登録済みだったが、以下の誤りがあった。

```python
"マジカルアクセル": MoveData(
    type="エスパー",                                       # 誤り: 正しくはフェアリータイプ
    category="special",                                    # 誤り: 正しくは物理技
    pp=10,                                                  # 誤り: Championsの4段階(8/12/16/20)に含まれない
    power=100,
    accuracy=100,
    flags={"non_copycat", "secondary_effect"},              # 誤り: "non_encore" が不足
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.マジカルアクセル_apply_confusion_to_defender,
        )
    }
),
```

こんらん付与ハンドラ自体（`ha.マジカルアクセル_apply_confusion_to_defender` =
`apply_confusion_to_defender(battle, ctx, value, chance=0.3)`）は確率30%で仕様と一致しており、修正不要。

## 修正後の実装

```python
"マジカルアクセル": MoveData(
    type="フェアリー",
    category="physical",
    pp=12,
    power=100,
    accuracy=100,
    flags={"non_copycat", "non_encore", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.マジカルアクセル_apply_confusion_to_defender,
        )
    }
),
```

## ハンドラ構成

| Event | 関数 | subject_spec | priority | 役割 |
|---|---|---|---|---|
| `Event.ON_DAMAGE_HIT` | `マジカルアクセル_apply_confusion_to_defender` | `attacker:self`（デフォルト） | 100（デフォルト） | 30%の確率で相手をこんらん状態にする |

`ON_DAMAGE_HIT` は既存の同種こんらん付与技（ばくれつパンチ・ねんりき・サイケこうせん・
ぼうふう 等）と同一イベントであり、`.internal/spec/turn.md` に個別の priority 記載がない
ダメージ命中後の追加効果ハンドラ群と同様、`MoveHandler` のデフォルト priority（100）を
そのまま使用する。これは既存のこんらん付与ハンドラすべてが priority 未指定（デフォルト値）
である前例を踏襲したもの。

## Priority 根拠

* `Event.ON_DAMAGE_HIT` は `.internal/spec/turn.md` に個別の priority 記載がないイベントであり、
  CLAUDE.md の指針に従い既存の同種ハンドラ（`ばくれつパンチ_apply_confusion_to_defender` 等、
  すべて priority 未指定＝デフォルト100）を参照して同じ扱いとする。

## 実装箇所

### handlers/move_attack.py

修正不要（`マジカルアクセル_apply_confusion_to_defender` は既存実装のまま、
`apply_confusion_to_defender` 汎用ヘルパー（`handlers/move.py`）を呼び出すのみ）。

### data/moves/move_ma.py

`"マジカルアクセル"` エントリの `type` を `"エスパー"` → `"フェアリー"` に、
`category` を `"special"` → `"physical"` に、`pp` を `10` → `12` に、
`flags` に `"non_encore"` を追加する。

## 注意点・エッジケース（review-test 向け）

1. `type` を `"フェアリー"` に修正 → タイプ相性判定（フェアリー技として、はがね・どく・
   ドラゴンタイプへの相性が変わる）に影響する。既存テストは type 相性を直接検証していない
   ため、既存テストは通る想定だが、新規にタイプ判定のテストは追加しない
   （本レビューの主眼はフラグ・PP・分類の修正であり、タイプ相性は共通ロジックでカバーされる）。
2. `category` を `"physical"` に修正 → ダメージ計算で攻撃側の「こうげき」実数値を参照する
   ようになる（従来は誤って「とくこう」を参照していた）。既存テストはダメージ量を検証して
   いないため影響なし。
3. `flags` に `"contact"` を含めない → 一次情報の「直接攻撃：×」の通り、接触判定系の
   特性・アイテム（さめはだ等）のトリガー対象外であることを確認する（既存実装のまま、
   `"contact"` フラグは付与されていないため変更不要）。
4. `flags` に `"non_encore"` を追加 → アンコールで固定できないことを確認する
   （`アンコール_can_apply` は `move.has_flag("non_encore")` を見て失敗させる）。
5. `flags` の `"non_copycat"` は既存実装済みで、まねっこ失敗テストも既に存在する
   （`tests/moves_status/test_move_ma.py` の
   `test_まねっこ_スターモービル専用技はコピーできない`）→ 変更不要。
6. PP を `12` に修正 → PP消費・わるあがき遷移等の一般テストは既存の共通ロジックで
   カバーされるため専用テスト不要（他のPP修正済み技と同様の扱い）。
7. `secondary_effect` フラグにより、ちからずく・てんのめぐみ等の特性判定対象になる
   （既存の共通ロジックで処理されるため専用ハンドラ不要）。
8. リーサル計算: `.internal/progress/move.md` の「リーサル実装」列が `n/a` のため対象外。
