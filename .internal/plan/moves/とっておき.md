# とっておき 実装計画書

## 仕様要約（.internal/spec/moves/とっておき.md 参照）

* タイプ: ノーマル / 分類: 物理 / 威力: 140 / 命中: 100 / PP: 5
* 対象: 相手1体 / 優先度: 0 / 直接攻撃
* 効果: 自分が場に出た後で、とっておき以外に覚えている技全てを1回以上使ったことがなければ失敗する。
  * 「使用した」とみなす条件は PP が実際に消費されたことであり、ねむり等で PP を消費できなかった
    技は使用したことにならない。逆に PP が消費されていれば、技自体が失敗していても条件を満たせる。
  * とっておき以外に覚えている技が無い場合は失敗する。
  * 一度控えに戻ると、次に場に出たとき再びとっておき以外の技を1回以上使わなくてはいけなくなる
    （場に出てからの使用履歴としてリセットされる）。
  * とっておきを覚えていないポケモンが、他の技が出る技経由でとっておきを使った場合は失敗する。
  * **Champions では条件を満たしていない場合、とっておき自体をコマンドとして選択できなくなる**
    （スカーレット・バイオレット以前は選択自体は可能で、行動時に失敗するのみ）。
* 本プロジェクトでは、ものまね/スケッチ/レベルアップによる技構成変更・へんしん・Zワザ・
  ダイマックスわざは未実装のため、これらとの相互作用は対象外（該当する技・特性が実装されていない）。

## 現状の実装状況（レビュー前）

`src/jpoke/data/move.py` にタイプ・威力・命中率・PP・分類は登録済みだったが、
`handlers={}` となっており、とっておき最大の特徴である「他の技を全て使い切っていないと
失敗する」条件が一切実装されていなかった。

`src/jpoke/model/pokemon.py` の `__deepcopy__` には `pp_consumed_moves` という
未定義属性への参照だけが残っており（`fast_copy` は存在しない属性を無視するため実害は無いが）、
本来この属性を使って実装される予定だったことが伺える。今回この属性を実際に追加する。

## 実装方針

### 1. 技の使用履歴を追跡する仕組み（Pokemonモデル・MoveExecutor）

- `Pokemon.pp_consumed_moves: set[MoveName]` を新設する。
  「場に出てから実際に PP を消費して使用した技名」の集合。
- `Pokemon.reset_on_switch_in()` でリセットする（交代で控えに戻り、再度場に出た際に
  未使用状態に戻る仕様と一致する）。
- `MoveExecutor._consume_pp()` で PP 消費量 `v > 0` のとき、`ctx.attacker.pp_consumed_moves`
  に技名を追加する。
  - PP 消費は `ON_TRY_ACTION`（まひ・ねむり等の行動可否判定）を通過した後にのみ行われるため、
    「PPを消費できなかった技は使用したことにならない」という仕様と自然に一致する。
  - ねごとのサブ技実行時は `ねごと_suppress_pp` により `v=0` となるため、サブ技側は
    使用済みに計上されない（「他の技が出る技経由で使用した技は、自分がPPを消費した技しか
    使ったことにならない」という仕様と一致する）。
  - これは技固有の効果ではなく PP 消費という汎用処理の一部であるため、`handlers/` ではなく
    `core/move_executor.py` / `core/query.py` に直接実装する
    （`acted_since_switch_in` 等の既存の汎用フラグと同じ扱い）。

### 2. 発動条件判定（PokemonQuery）

- `PokemonQuery.can_use_last_resort(pokemon) -> bool` を新設する。
  - `pokemon.has_move("とっておき")` が False なら False
    （他の技が出る技経由でとっておきを覚えていないポケモンが使った場合に対応）。
  - とっておき以外の技名一覧が空なら False。
  - とっておき以外の技名が全て `pp_consumed_moves` に含まれていれば True。
- ハンドラ（`handlers/move_attack.py`）とコマンド生成（`core/command_manager.py`）の
  両方から同じ判定ロジックを再利用するため、`core/query.py` に集約する。

### 3. 発動失敗ハンドラ（handlers/move_attack.py・data/move.py）

- `Event.ON_TRY_MOVE_1`（priority=30, `.internal/spec/turn.md` に
  「30 | とっておき: 未使用技がある」と明記）で `can_use_last_resort` が False の場合、
  `LogCode.MOVE_FAILED` を出して技を失敗させる。

### 4. コマンド選択の制限（core/command_manager.py）

- Champions 仕様に合わせ、`get_available_action_commands()` の技コマンド生成時に、
  `can_use_last_resort` を満たさないとっておきは選択肢から除外する
  （PP0 の技を除外する既存処理と同じ場所に条件を追加する）。
- 除外の結果、選択可能な技が0件になった場合は既存処理により自動的にわるあがきが選択肢に
  追加される。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_TRY_MOVE_1` | 30（`.internal/spec/turn.md`「とっておき: 未使用技がある」に対応） | attacker:self（`MoveHandler` 既定値） | 他の技を全て使用済みでなければ失敗させる |

```python
def とっておき_check_used_all_moves(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """とっておきの発動条件チェック。

    自分がとっておきを覚えていない場合(他の技が出る技経由での使用)は失敗する。
    場に出てから、とっておき以外に覚えている技をすべて1回以上PP消費して使用して
    いなければ失敗する。とっておき以外に覚えている技が無い場合も失敗する。
    """
    mon = ctx.attacker
    if not battle.query.can_use_last_resort(mon):
        battle.add_event_log(
            mon, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="とっておき"),
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

## 実装箇所

### src/jpoke/model/pokemon.py

- `__init__` に `self.pp_consumed_moves: set[MoveName] = set()` を追加。
- `reset_on_switch_in()` で `self.pp_consumed_moves = set()` にリセット。
- `__deepcopy__` は既に `pp_consumed_moves` を `keys_to_deepcopy` に含んでいたため変更不要。

### src/jpoke/core/move_executor.py

- `_consume_pp()` で `v > 0` の場合に `ctx.attacker.pp_consumed_moves.add(move.name)` を追加。

### src/jpoke/core/query.py

- `PokemonQuery.can_use_last_resort(pokemon)` を新設。

### src/jpoke/core/command_manager.py

- `get_available_action_commands()` の `move_indexes` 算出条件に、とっておき用の
  `can_use_last_resort` チェックを追加。

### src/jpoke/handlers/move_attack.py

- `とっておき_check_used_all_moves` を新設。

### src/jpoke/data/move.py

- `"とっておき"` の `handlers` に `Event.ON_TRY_MOVE_1` ハンドラを登録。
