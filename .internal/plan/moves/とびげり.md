# 実装計画書: とびげり

参照: `.internal/spec/moves/とびげり.md`, `.internal/plan/moves/失敗反動技.md`

## 現状の実装状況（レビュー時点）

* `data/move.py` に登録済み。ただし以下の誤りがあった。
  * `power=85, accuracy=90` → 第四世代の値のまま。第五世代以降の値である `power=100, accuracy=95` に修正する。
  * `flags={"contact", "gravity_restricted"}` → `recoil` フラグが欠落している。
    `.internal/plan/moves/失敗反動技.md` の注記の通り、失敗反動技グループは実際のゲームで
    すてみ（Reckless）の威力上昇対象であり、本プロジェクトでは `ctx.move.has_flag("recoil")` で
    すてみを判定しているため、`recoil` フラグが無いとすてみが発動しない。
* `handlers/move_attack.py` の `とびげり_crash` は `reason="self_cost"` で減算していたが、これは誤り。
  `.internal/plan/moves/失敗反動技.md` 注記2（とびひざげりレビュー時の知見）の通り、
  実際のゲームではマジックガード所持時にとびげりが外れても反動ダメージを受けない。
  `マジックガード_ignore_damage`（`handlers/ability.py`）は `self_cost` を除外リストに含めているため、
  `self_cost` のままではマジックガードで防げないバグになる。`reason="fixed_recoil"` に修正した
  （`fixed_recoil` は いしあたま では防げず、マジックガードでは防げる区分）。
* `Event.ON_TRY_MOVE_1`（priority=30）で `gravity_restricted_fail` を発火し、
  じゅうりょく中は技を失敗させる。`tests/test_field.py::test_じゅうりょく_使用禁止技ブロック` で
  パラメタライズ済み（修正不要）。

## ハンドラ構成

| event | priority | subject_spec | 役割 |
|-------|----------|-------------|------|
| `ON_TRY_MOVE_1` | 30 | `attacker:self` | じゅうりょく中は失敗（`gravity_restricted_fail`、既存の共通ハンドラを流用） |
| `ON_MISS` | 100（デフォルト） | `attacker:self` | 外れたとき自分の最大HPの1/2ダメージ（`reason="fixed_recoil"`） |

Priority根拠: `.internal/plan/moves/失敗反動技.md` を参照（`ON_MISS` は `.internal/spec/turn.md` 未掲載のため、
既存の同種ハンドラ（からぶりほけん・かかとおとし等）に倣いデフォルト値100を採用）。
`ON_TRY_MOVE_1` の priority=30 は既存の `gravity_restricted_fail` 登録済みの技（とびはねる等）と同じ値を踏襲。

## 修正内容

### `src/jpoke/data/move.py`

```python
"とびげり": MoveData(
    type="かくとう",
    category="physical",
    pp=10,
    power=100,
    accuracy=95,
    flags={"contact", "gravity_restricted", "recoil"},
    handlers={
        Event.ON_TRY_MOVE_1: h.MoveHandler(
            h.gravity_restricted_fail,
            subject_spec="attacker:self",
            priority=30,
        ),
        Event.ON_MISS: h.MoveHandler(
            ha.とびげり_crash,
        ),
    }
),
```

変更点: `power` 85→100、`accuracy` 90→95、`flags` に `"recoil"` を追加。

さらに `handlers/move_attack.py` の `とびげり_crash` の `reason` を `"self_cost"` → `"fixed_recoil"` に修正する。

```python
def とびげり_crash(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """とびげりが外れた場合の失敗反動ダメージ。自分の最大HPの1/2を受ける。

    マジックガードで防げるが、いしあたまでは防げない確定反動として扱う
    （reason="fixed_recoil"。"self_cost" だとマジックガードで防げなくなってしまうため区別する）。
    """
    battle.modify_hp(ctx.attacker, v=-max(1, ctx.attacker.max_hp // 2), reason="fixed_recoil", source=ctx.attacker)
    return HandlerReturn(value=value)
```

修正後は `python scripts/sort_data/sort_moves.py` と `python scripts/sort_handlers.py src/jpoke/handlers/move_attack.py`
を実行し順序を維持する。

## テスト観点

1. 命中時: ON_MISSが発火せず反動なし（既存テストで対応済み）。
2. 外れたとき: 使用者のHPが `max(1, max_hp // 2)` 減る（既存テストで対応済み）。
3. 残HPが反動ダメージ未満でもひんしになる（既存テストで対応済み）。
4. じゅうりょく中は技が失敗する（`tests/test_field.py` の共通パラメタライズテストで対応済み）。
5. `flags` に `"recoil"` が含まれる（すてみの対象）ことを確認する新規テストを追加する。
6. すてみ特性で威力が1.2倍になることを確認する新規テストを追加する
   （`tests/abilities/test_ability_sa.py::test_すてみ_反動技の威力が1_2倍になる` にパラメータ追加）。
7. `power`/`accuracy`/`pp` が仕様通り（100/95/10）であることを確認する回帰テストを追加する。
8. いしあたま所持時も失敗反動ダメージを受けることを確認する新規テストを追加する。
9. マジックガード所持時は失敗反動ダメージを受けないことを確認する新規テストを追加する。
