## 注意点・エッジケース（review-test 向け）

1. **やまあらし の SVでの使用可否**
   - `.internal/wiki/moves/やまあらし.html` に「SV で使用不可」と記載がある。
   - ただし `src/jpoke/data/ps-champ-ja/moves.json` には登録されており、プロジェクト対象（ポケモンチャンピオンズシングルバトル）では使用可能として扱う。

2. **必中（トリックフラワー）と命中率なし**
   - `トリックフラワー` の `accuracy` フィールドが未設定（`None`）のままで、既存実装では必中扱いとなっている。
   - 計画書対象の変更は不要（既登録・既実装）。

3. **カブトアーマー / シェルアーマー との相互作用**
   - 確定急所（`critical_rank=3`）であっても、相手の `カブトアーマー` / `シェルアーマー` 特性が発動すれば急所無効になる。
   - `ON_MODIFY_CRITICAL_RATE` イベントハンドラ側で `return HandlerReturn(value=0.0)` とすることで対応済み（カブトアーマー実装側の責務）。ハンドラ側の対応は不要。

4. **きょううん / ピントレンズ との重複**
   - `critical_rank=1` 技に対してきょううん（+1）が加算されると rank=2 となり確率 1/2 になる。
   - 上限 `clamp_critic` で rank=3 超えは 3 に丸められ、rank=3 以上は必ず急所。

5. **`critical_rank=3` は「確定急所」として十分か**
   - `.internal/spec/critical.md` によると Stage 3以上で確率 1（100%）。
   - `clamp_critic` が rank を 3 上限にクランプし、`CRIT_RATES[3] = 1.0` を返すため確定急所として正しく機能する。

6. **contact ラベルの有無**
   - `やまあらし`：直接攻撃（接触）のため `contact` が必要。
   - `エアカッター`：非接触（特殊技かつ直接攻撃なし）のため `contact` は不要。現行登録で正しい。
   - `アクアカッター`：非接触のため `contact` は不要。現行登録で正しい（`slash` のみ）。

7. **テスト方針**
   - `critical_rank=1` の確率テストは確率制御（accuracy=100 固定のように random を固定）が必要。
   - `critical_rank=3`（やまあらし・トリックフラワー・こおりのいぶき）は乱数に関係なく急所が確定するため、シンプルなアサートでテスト可能。
