# 計画書: ないしょばなし

## 仕様要約

- ノーマルタイプ / 変化 / 必中（`accuracy=None`） / PP20 / 対象: 1体選択 / 音技（`sound`）
- 相手の『とくこう』ランクを1段階下げる
- まもるで防がれない（`まもる状態やダイウォールを無視する。`と明記）ため `unprotectable` フラグを付与する
- みがわりは貫通する（`sound` フラグによるフレームワーク側の一般ルール。個別対応不要）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | 相手のとくこうランクを1段階下げる |

## Priority 根拠

`Event.ON_STATUS_HIT` は `.internal/spec/turn.md` に未掲載のイベントのため、既存の同種ハンドラ
（ドわすれ・とおせんぼう等の能力ランク変化・状態付与ハンドラ）と同じデフォルト優先度100を使用する。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `ドわすれ` の後、`なかまづくり` の前（五十音順）。

```python
def ないしょばなし_modify_defender_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spa": -1})
```

### data/moves/move_na.py への更新

`"ないしょばなし"` エントリを以下のとおりとする。

```python
    "ないしょばなし": MoveData(
        type="ノーマル",
        category="status",
        pp=20,
        accuracy=None,  # 必中
        flags={"sound", "unprotectable"},
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.ないしょばなし_modify_defender_stats,
            )
        }
    ),
```

## 注意点・エッジケース

- 命中判定は「100%」ではなく「必中」（`accuracy=None`）。ランク補正・回避率に関係なく必ず命中する
- まもるで防がれない（`まもる状態やダイウォールを無視する。`と明記）ため `unprotectable` フラグが必要
  （デフォルトでは target=foe の変化技はまもるで防がれる仕様のため、明示的なフラグ付与が必須）
- みがわりには `sound` フラグにより自動的に貫通する（`move_executor.check_hit_substitute` が
  sound フラグから汎用的に判定するため、個別のフラグ・ハンドラは不要）
- マジックコートには反射される（`unreflectable` フラグを付けない。category=status/target=foe の
  一般ルールに従う）
- とくこうランク変化は既存の `modify_defender_stats` ヘルパーをそのまま利用する（`ドわすれ` の
  `modify_attacker_stats` 呼び出しと対称的なパターン）
- 特性『かちき』の相手への差し引き反転は `かちき` 特性側の一般仕様（`battle.modify_stats` 経由）で
  自動的に処理されるため、本技のハンドラでの個別対応は不要
