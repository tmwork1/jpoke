# 攻撃技_こんらん系 実装計画書

## 仕様要約

- ダメージを与えた後、確率でこんらん状態を付与する
- こんらんは揮発性状態（volatile）であり、`volatile_manager.apply_confusion` で付与する
- こんらんのターン数は 2〜5 ターンのランダム（`apply_confusion` 内部で `randint(2, 5)` が設定される）
- 確率は技ごとに異なる（0.1〜1.0）
- マイペース持ちは `ON_TRY_MOVE_2` (priority=140) でこんらん無効になる（ハンドラ側でのガード不要）
- ミストフィールド展開中はこんらん無効（volatile_manager.apply 内部で処理）

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_HIT` セクション：

> | 100 | ダイマックスわざの効果 |

追加効果（状態異常付与）の既存ハンドラ（`ほうでん_apply_ailment_to_defender` 等）がすべて priority=100 で登録されているため、こんらん付与も同様に priority=100 とする。

`.internal/spec/turn.md` の `Event.ON_HIT` の記載（priority=100）:
- 技の追加効果全般が priority=100 で処理される

（ワンダースチームは実際には `ON_DAMAGE_HIT` で登録されるが、`MoveHandler` のデフォルト値
priority=100・subject_spec=attacker:self は変わらないため、上記根拠がそのまま適用される。）

## 実装コード

### ハンドラ関数の実装パターンについての注意

`apply_volatile_to_defender(battle, ctx, value, volatile="こんらん", chance=...)` は使用**不可**。  
理由: `volatile_manager.apply` に `count=None` が渡され、こんらんのランダムターン数（2〜5）が設定されない。  
正しいパターン: `volatile_manager.apply_confusion` + 手動確率チェック を使う。

## 注意点・エッジケース

### ハンドラの実装パターン

- **`apply_volatile_to_defender(volatile="こんらん")` は使用禁止**: `volatile_manager.apply` に `count=None` が渡るため、こんらんのランダムターン数（2〜5）が設定されない。必ず `volatile_manager.apply_confusion` を呼ぶ。
- 確率が 1.0 の技（おしゃべり・ばくれつパンチ）は確率チェック不要。`apply_confusion` を直接呼ぶ。
- 確率が 1.0 未満の技では `battle.resolve_secondary_chance(ctx, chance)` で効果確率補正（ちからずく等）を適用してから判定する。

### こんらん無効化はハンドラ側でのガード不要

- **マイペース**: `ON_TRY_MOVE_2` priority=140 で manager 内部が処理する
- **ミストフィールド**: `volatile_manager.apply` 内部で無効化される
- **みがわり**: こんらんは みがわりを貫通しない（`volatile_manager.apply` がブロック）

### すでにこんらん状態の場合

- `volatile_manager.apply_confusion` が内部で `apply` を呼ぶ。`apply` はすでに同じ揮発性状態がある場合は `False` を返してスキップする。

### ぼうふうの既存ハンドラとの共存

- `ON_MODIFY_ACCURACY` ハンドラ（天候による命中率補正）と `ON_HIT` ハンドラ（こんらん付与）は別イベントなので競合しない。

### 未登録 MoveData の実データ確認

おしゃべり / シグナルビーム / マジカルアクセル / ロッククライム の type / power / accuracy / labels は `.internal/spec/` 内の詳細仕様書で確認してから実装すること。本計画書の仮定義は概算値。

### ばくれつパンチの命中率

仕様書では accuracy=50 で登録されているが、実際の技の命中率は確認が必要（技によっては命中補正系の特性と相互作用する可能性あり）。
