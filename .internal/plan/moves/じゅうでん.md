# 計画書: じゅうでん

## 仕様要約

- タイプ: でんき、変化、PP: 20、対象: 自分、命中判定なし、優先度: 0
- 使用者自身に **じゅうでん** 揮発性状態を付与する
  - じゅうでん状態中に次に使うでんきタイプの攻撃技の威力が2倍になる
  - でんき技を選択すると消滅する（第九世代仕様）。でんき技を選択しない限り何ターンでも継続
- 使用者自身の **とくぼうを1段階上げる**（第四世代以降）
- とくぼうのランク上昇はじゅうでん状態が解除されても消えない

出典: `.internal/wiki/moves/じゅうでん.html`、`.internal/wiki/volatiles/じゅうでん_(状態変化).html`

## ハンドラ構成

| イベント | Priority | subject_spec | 役割 |
|----------|----------|-------------|------|
| `ON_STATUS_HIT` | 100 (デフォルト) | なし (MoveHandler は skip_subject_check=True) | 使用者にじゅうでん状態付与 + とくぼう+1 |

### Priority 根拠

`.internal/spec/turn.md` に `ON_STATUS_HIT` 内の個別 priority 記載なし。
既存の自己状態付与技（アクアリング、きあいだめ等）および自己ランク変化技（めいそう、ちょうのまい等）はすべてデフォルト priority=100 で登録されている。
じゅうでんはその両方を同時に行う複合技であるため、同様に priority=100 を使用する。

## 実装箇所

### 1. `src/jpoke/handlers/move_status.py`

挿入位置: `じばそうさ_modify_attacker_stats` の後、`しんぴのまもり_set_side_field` の前（五十音順「じゅ」は「じ」グループ末尾）

```python
def じゅうでん_apply(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """じゅうでんの効果: 自分にじゅうでん状態を付与し、とくぼうを1段階上げる。"""
    mon = ctx.attacker
    battle.volatile_manager.apply(mon, "じゅうでん", source=mon)
    battle.modify_stats(mon, {"D": 1}, source=mon)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース

1. **とくぼう上昇はじゅうでん状態消滅後も持続**: ランク変化は揮発性状態とは独立して管理されるため、じゅうでん状態が消えてもとくぼうランクはそのまま残る。volatile_manager と modify_stats は互いに独立しているので実装上も問題ない

2. **すでにじゅうでん状態の場合**: `volatile_manager.apply` が内部で重複チェックを行うため、ハンドラ側にガード不要。ただし状態が重なってもでんき技の威力は2倍のまま増えず、1回使用で消滅する

3. **とくぼうランク+6 で技を使った場合**: ランク最大時の失敗判定は `turn.md` Priority 130 でマネージャー内部処理されるため、ハンドラ側にガード不要。volatile 付与は成功する

4. **じゅうでん状態の消滅タイミング（第九世代）**: でんき技を選択した行動完了時に消滅。消滅処理は `handlers/volatile.py` の `じゅうでん_boost_electric` 内の `remove_volatile` で実装済み

5. **バトンタッチ**: じゅうでん状態は引き継がれない（wiki 仕様）。バトンタッチのハンドラ側で除外する必要があるが、それはバトンタッチ実装の責務

6. **交代による消滅**: じゅうでん状態は交代で消滅する。これは volatile_manager の通常フローで処理される

7. **`target="self"` の明示**: MoveData に `target="self"` を設定することで、自己対象の変化技として正しく動作する
