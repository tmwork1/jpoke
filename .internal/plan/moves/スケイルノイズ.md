# 実装計画書: スケイルノイズ

参照仕様書: .internal/spec/moves/スケイルノイズ.md

## 仕様要約

* タイプ: ドラゴン / 分類: 特殊 / 威力: 110 / 命中率: 100 / PP: 8（champions版） / 対象: 1体選択
  / 優先度: 0 / 接触なし / 音技（sound）
* 技が成功しダメージを与えた後、自分の『ぼうぎょ』ランクが1段階下がる
  （確率100%・確定効果、追加効果には分類されない）。
* 音技のため `みがわり` 状態の相手にも本体へダメージが通る。

## 既存実装の確認結果

`src/jpoke/data/moves/move_sa.py` に既に以下の通り実装済みであることを確認した。

```python
"スケイルノイズ": MoveData(
    type="ドラゴン",
    category="special",
    pp=5,
    power=110,
    accuracy=100,
    flags={"sound"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.スケイルノイズ_lower_attacker_def,
        )
    }
),
```

* `handlers/move_attack.py` の `スケイルノイズ_lower_attacker_def` は
  `modify_attacker_stats(battle, ctx, value, stats={"def": -1})` を呼び出しており、`chance` を
  指定していないためデフォルトの100%（確定効果）で発動する。`battle.modify_stats` 経由のため、
  あまのじゃく等の特性補正も自動的に適用される。
* `Event.ON_HIT` の `MoveHandler` は `subject_spec` を省略した場合デフォルトで `"attacker:self"`
  になる（`src/jpoke/handlers/move.py` の `MoveHandler.__init__`）。本ハンドラは攻撃者自身の
  ランクを変化させるため、この既定値と一致しており問題ない。
* `flags` は `{"sound"}` のみで `secondary_effect` を含まない。一次情報の判定表に
  「追加効果: ×」と明記されており、自己ランクダウンは「追加効果」に分類されないため、
  ちからずくとの相互作用を避けるためにも `secondary_effect` を持たないこの実装は正しい
  （`アーマーキャノン`・`ぶちかまし` 等の同系統技と同じ方針）。接触もパンチ技でもないため他の
  flag も不要。`sound` フラグにより `みがわり` 貫通・`ぼうおん`・`パンクロック` は
  `ctx.move.has_flag("sound")` を参照する汎用ロジック（`handlers/volatile.py`・
  `handlers/ability.py`）で自動的に処理されるため、個別の対応は不要。
* `Event.ON_HIT` は `.internal/spec/turn.md` で「自分のランクが下がる反動」が priority=10 と
  記載されているが、既存の同種実装（`アーマーキャノン_lower_attacker_stats`、
  `インファイト_lower_defender_stats`、`ばかぢから_lower_attacker_stats` 等）はいずれも
  `priority` を指定せずデフォルト（100）のまま登録されている。本技も同じ実装パターンの
  ため、既存実装のデフォルト優先度を踏襲し変更しない。
* 一次情報（ポケモンWiki技の仕様欄）にまひ等の追加状態異常を示す記載は存在しない
  （「効果: 成功すると、自分のぼうぎょが1段階下がる。」「追加効果: ×」のみ）。自分の防御
  ランクダウン以外の効果は無いと判断し、まひ付与の実装は行わない。

### 不整合の修正: PP

`src/jpoke/data/ps-champ-ja/moves.json`（正）では `スケイルノイズ` の PP は 8 と記載されている
（champions では全技PPが 8/12/16/20 の4段階に統一されているため）。既存実装は
`pp=5`（Gen9本家Wikiの値のまま移行漏れ）になっており、`pp=8` に修正する。
`.internal/progress/move.md` は既に PP 列が `8` になっており、修正不要（実装側を進捗表に
合わせる）。

### リーサル計算の確認結果

`.internal/progress/move.md` の「リーサル実装」列は `-`（未判定）。スケイルノイズが下げるのは
攻撃者自身の『ぼうぎょ』であり、以降の与ダメージには影響しない（`ばかぢから`が自分の
『こうげき』を下げる＝以降の与ダメージに影響するのとは異なり、`アーマーキャノン`・
`ぶちかまし`・`インファイト`と同じ「防御側の能力」低下パターンに該当する）。よって
`lethal_handlers` は登録不要であり、`n/a` が妥当と判断する。`.internal/progress/move.md` の
リーサル実装・リーサルテスト列を `-` から `n/a` に修正する。
