# かみなり 実装計画書

## 仕様要約（.internal/spec/moves/かみなり.md 参照）

* タイプ: でんき / 分類: 特殊 / 威力: 110 / 命中: 70 / PP: 10
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果: 追加効果として、30%の確率で相手を『まひ』状態にする。
* 天気が『あめ』のときは必ず命中する。天気が『にほんばれ』のときは命中率が50になる。
* `そらをとぶ` の溜め状態（回避状態）の相手にも命中する。

## 現状の実装状況

`src/jpoke/data/move.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。
レビュー時点で以下の誤りを検出し修正した：

* `かみなり_apply_paralysis_to_defender` のまひ付与確率が `chance=0.1`（10%）になっていたが、
  正しくは `chance=0.3`（30%）である。第一世代のみ10%であり、本プロジェクトの対象世代（第九世代）では
  30%が正しい。`.internal/spec/moves/_move_secondary_1.md` 側の一覧データも同様に誤っていたため修正した。

```python
"かみなり": MoveData(
    type="でんき",
    category="special",
    pp=10,
    power=110,
    accuracy=70,
    flags={"secondary_effect"},
    handlers={
        Event.ON_MODIFY_ACCURACY: h.MoveHandler(
            ha.かみなり_accuracy,
            subject_spec="attacker:self"
        ),
        Event.ON_DAMAGE_HIT: h.MoveHandler(

        ),
    }

```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_MODIFY_ACCURACY` | 100（デフォルト） | attacker:self | 天候による命中率補正（あめ: 必中、にほんばれ: 50） |
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 30%の確率で相手を『まひ』状態にする |

`src/jpoke/handlers/move_attack.py` の実装（修正済み）:

```python
def かみなり_accuracy(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """かみなりの天候による命中率補正。雨: 必中、晴れ: 50%
    攻撃側がばんのうがさを持つ場合、晴れでも命中率低下なし。
    防御側がばんのうがさを持つ場合、雨でも必中にならない。
    """
    if battle.weather_for(ctx.defender).rainy:
        return HandlerReturn(value=None)  # 必中
    elif battle.weather_for(ctx.attacker).sunny:
        return HandlerReturn(value=50)
    return HandlerReturn(value=value)


def かみなり_apply_paralysis_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="まひ", chance=0.3)
```

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は汎用ハンドラ側で処理される。本技側に個別のガードは不要。

`battle.weather_for(mon)` はばんのうがさを持つポケモンに対して晴れ/雨系天候を「なし」として返すため、
攻撃側・防御側どちらがばんのうがさを持っていても正しく無効化される（`かみなり_accuracy` 側の実装で対応済み）。

`そらをとぶ` 状態の相手に命中する仕様は `handlers/volatile.py` の
`HIDDEN_MOVE_ALLOWED_MOVES["そらをとぶ"]` に `かみなり` が登録済みであることで実現しており、
本技側の追加実装は不要。

## Priority 根拠

* `ON_MODIFY_ACCURACY` はダメージ計算内部のイベントであり `.internal/spec/turn.md` に priority 記載がないため、
  既存の同種ハンドラ（`ふぶき_accuracy`）と同じデフォルト priority（100）のパターンに従う。
* `ON_DAMAGE_HIT` によるまひ付与も同様に、既存の同種ハンドラ（`10まんボルト`・`かみなりのキバ`・
  `かみなりパンチ` 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。

## 注意点・エッジケース（review-test 向け）

1. **天候による命中率補正**
   - あめ（防御側から見た天候）: 必中。ランク補正（回避・命中ランク）も無視される
     （`ON_MODIFY_ACCURACY` が `None` を返すと `_check_hit` がランク補正前にTrueを返すため）。
   - にほんばれ（攻撃側から見た天候）: 命中率50。ランク補正は適用される。
   - `battle.weather_for` によりばんのうがさ所持ポケモンには晴れ/雨の効果が及ばない
     （`tests/test_item.py` の `test_ばんのうがさ_雨でかみなりが必中にならない` で検証済み）。
2. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・まひ付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
3. **そらをとぶとの相互作用**
   - `tests/moves_attack/test_move_sa.py` の `test_そらをとぶ_かみなりは命中する` で検証済み。
4. **テスト方針**
   - `secondary_chance` を1.0に固定し、まひ付与を確認するテスト
     （`test_かみなり_まひが発動する`）は既存。
   - 天候による命中率補正（あめ: 必中、にほんばれ: 命中率50）を確認するテストを追加する
     （`test_かみなり_あめ中は必中になる` / `test_かみなり_にほんばれ中は命中率が50になる`）。
