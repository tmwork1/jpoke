# シェルアームズ 実装計画書

## 仕様要約（.internal/spec/moves/シェルアームズ.md 参照）

* タイプ: どく / 分類: 特殊（判定次第で物理技に変化） / 威力: 90 / 命中: 100 / PP: 12
  （champions/move_list.txt 準拠。Gen9本家の10ではない）
* 対象: 相手1体 / 優先度: 0
* 接触: 物理技として計算されたときのみ直接攻撃になる（特殊技のときは直接攻撃にならない）
* 効果: 追加効果として、20%の確率で相手を『どく』状態にする。
* 分類判定: `攻撃/防御` の比が `特攻/特防` の比より高ければ物理技になる
  （ランク補正・パワートリック・パワーシェア・ガードシェア込みの実数値で比較）。
  等しい場合は実機ではランダム50%だが、本実装では特殊技のまま据え置く
  （「Priority根拠・実装方針」参照）。

## 現状の実装状況

`src/jpoke/data/moves/move_sa.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。
レビュー時点で以下の不備を発見し修正した。

1. **PP誤り**: `pp=10`（Gen9本家値）になっていた。champions/move_list.txt の値は12。
   → `pp=12` に修正。
2. **分類判定の誤り**: 使用者の `攻撃実数値 > 特攻実数値` のみで判定していたが、
   仕様上は対象の防御・特防実数値も比較に含める必要がある（テラバースト等の
   「攻撃 vs 特攻のみ」比較とは異なる、シェルアームズ固有の仕様）。
   → `攻撃 × 対象の特防 > 特攻 × 対象の防御` の比較に修正。
3. **接触判定の欠落**: `flags={"contact"}` を常時付与しており、特殊技として計算された
   場合でも直接攻撃扱いになっていた。
   → `Event.ON_CHECK_CONTACT` ハンドラを追加し、物理技のときのみ直接攻撃になるよう修正。

修正後の登録内容:

```python
"シェルアームズ": MoveData(
    type="どく",
    category="special",
    pp=12,
    power=90,
    accuracy=100,
    flags={"contact", "secondary_effect"},
    handlers={
        Event.ON_MODIFY_MOVE_CATEGORY: h.MoveHandler(
            ha.シェルアームズ_modify_move_category,
        ),
        Event.ON_CHECK_CONTACT: h.MoveHandler(
            ha.シェルアームズ_check_contact,
        ),
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.シェルアームズ_apply_poison_to_defender,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_MODIFY_MOVE_CATEGORY` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 攻撃/防御比と特攻/特防比を比較し分類を決定する |
| `ON_CHECK_CONTACT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 物理技のときのみ直接攻撃と判定する |
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 20%の確率で相手を『どく』状態にする |

`src/jpoke/handlers/move_attack.py` の実装:

```python
def シェルアームズ_apply_poison_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="どく", chance=0.2)


def シェルアームズ_check_contact(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """シェルアームズ: 物理技のときのみ直接攻撃になる（特殊技のときは直接攻撃にならない）。"""
    return HandlerReturn(value=ctx.move.category == "physical")


def シェルアームズ_modify_move_category(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """シェルアームズ: 攻撃/防御の比を特攻/特防の比と比較し、高い方の分類になる。"""
    attacker = ctx.attacker
    defender = ctx.defender
    atk = attacker.ranked_stats["atk"]
    spa = attacker.ranked_stats["spa"]
    de = defender.ranked_stats["def"]
    spd = defender.ranked_stats["spd"]
    if atk * spd > spa * de:
        return HandlerReturn(value="physical")
    return HandlerReturn(value=value)
```

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を
経由するため、ちからずくによる追加効果無効化・威力1.3倍化は汎用ハンドラ側で処理される。
本技側に個別のガードは不要。

`くさのけがわ`・`ふしぎなうろこ`・`ファーコート`・`てんねん`・`スロースタート` は、
いずれも `ctx.move.category`（`resolve_move_category` で確定済みの値）や
`battle.query.deals_physical_damage()` を参照する汎用実装になっており、
分類が確定した後のダメージ計算フェーズで動作するため、本技固有の追加対応は不要。

## Priority根拠・実装方針

* `ON_MODIFY_MOVE_CATEGORY` / `ON_CHECK_CONTACT` / `ON_DAMAGE_HIT` は `.internal/spec/turn.md`
  に priority 記載のない「ダメージ計算内部」のイベントである。CLAUDE.md の指示に従い、
  同種の既存ハンドラ（`テラバースト_modify_move_category`、`えんかく_nullify_contact`、
  `どくばり_apply_poison_to_defender` 等）を参照し、いずれもデフォルト priority=100・
  `attacker:self` で競合なく動作していることを確認したうえで同じ構成を採用した。
* **分類が同点の場合に乱数を使わない理由**: `resolve_move_category()`
  （`core/move_executor.py`）は `Event.ON_MODIFY_MOVE_CATEGORY` を再emitする実装であり、
  ダメージ計算中に `battle.query.deals_physical_damage()` 経由（`くさのけがわ` 等の
  特性ハンドラから）で複数回呼び出される。ここで乱数を使うと、同一ターン内の呼び出し
  ごとに分類の判定結果がぶれてしまい、ダメージ計算と直接攻撃判定で矛盾が生じる致命的な
  バグになる。そのため、既存の `テラバースト_modify_move_category`（同点なら特殊技に
  固定）と同じ方針を踏襲し、同点時は乱数を使わず特殊技のまま据え置く。実機の50%抽選は
  未実装だが、同点になるケース自体が稀（実数値が完全一致する必要がある）であり、
  既存の類似技と同じ簡略化として許容する。

## 注意点・エッジケース（review-test 向け）

1. **分類判定は攻撃側の実数値だけで決まらない**
   - 対象（防御側）の防御・特防実数値も比較に含まれる。使用者の特攻が攻撃より高くても、
     対象の防御が特防より大幅に低ければ物理技になり得る。
2. **接触判定は分類確定後に評価する**
   - `core/move_executor.py` の `execute()` は `ctx.move.category` を早い段階
     （`ON_TRY_ACTION` より前）で確定させるため、`is_contact()` が呼ばれる時点では
     常に確定済みの分類を参照できる。
3. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・どく付与無効化になる。
     分類が変わる効果はちからずくでも通常どおり発動する（ダメージ計算とは独立した処理）。
4. **テスト方針**
   - `test_シェルアームズ_AがCより高いとき物理技になる`: 使用者の攻撃・特攻のみで
     分類が決まる典型ケース（既存）。
   - `test_シェルアームズ_相手の防御実数値も分類判定に影響する`: 対象の防御・特防の
     ランク変化により分類判定が逆転することを確認する（新規）。
   - `test_シェルアームズ_物理技のときのみ直接攻撃になる`: `さめはだ` を用いて、
     物理技のときのみ直接攻撃として反動を受けることを確認する（新規）。
   - `test_シェルアームズ_どくが発動する`: 20%でどくを付与することを確認する（既存）。
