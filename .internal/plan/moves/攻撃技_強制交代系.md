# 計画書: 攻撃技（追加効果）強制交代系

## 仕様要約

ダメージを与えた後、相手の控えポケモンを強制的に場に出す技。

| 技 | タイプ | 分類 | 威力 | 命中 | 優先度 | 交代先選択 |
|---|---|---|---|---|---|---|
| ともえなげ | かくとう | 物理 | 60 | 90 | -6 | 控えポケモンからランダムに選択 |
| ドラゴンテール | ドラゴン | 物理 | 60 | 90 | -6 | 控えポケモンからランダムに選択 |

- ダメージ処理後に強制交代が発生する（`ON_DAMAGE_HIT` を利用）
- ダイマックスしている相手には追加効果（強制交代）が無効
- まもる等の防御技で技自体を防がれる（攻撃技扱いのため通常の防御が有効）
- きゅうばん・ねをはる状態の相手には `ON_TRY_BLOW` 経由で無効化される
- 控えポケモンが存在しない場合は交代処理を行わない
- 交代先はほえる・ふきとばしと同様、控えポケモン（瀕死・タマゴを除く）からランダムに選ばれる
  （`.internal/wiki/moves/ともえなげ.html` の「技の仕様」節参照。ドラゴンテールも同節を参照する旨が記載されている）

## 現状の実装状況

| 技 | MoveData 登録 | ON_DAMAGE_HIT ハンドラ | 状態 |
|---|---|---|---|
| ともえなげ | あり | `ともえなげ_force_switch` | 実装済み |
| ドラゴンテール | あり | `ドラゴンテール_force_switch` | 実装済み |

既存の強制交代技（ふきとばし・ほえる）は変化技であり、`ON_BEFORE_APPLY_MOVE` (priority=30) で `on_blow_apply` を呼んで `ON_TRY_BLOW` を先にエミットする構造になっている。
攻撃技型は `ON_DAMAGE_HIT` でダメージ後に交代処理を行い、その中で `ON_TRY_BLOW` をエミットしてきゅうばん等の無効化を処理する。

## ハンドラ構成

| 技 | イベント | priority | subject_spec | 役割 |
|---|---|---|---|---|
| ともえなげ | `ON_DAMAGE_HIT` | 100 | `attacker:self`（MoveHandler デフォルト） | きゅうばん等チェック後、控えポケモンをランダムに強制交代 |
| ドラゴンテール | `ON_DAMAGE_HIT` | 100 | `attacker:self`（MoveHandler デフォルト） | きゅうばん等チェック後、控えポケモンをランダムに強制交代 |

MoveHandler のデフォルト（`subject_spec="attacker:self"`, `priority=100`）を使用する。

## 実装箇所

### `src/jpoke/handlers/move_attack.py`

`blow` 関数（`move_status.py` 内）と同様、ともえなげ・ドラゴンテールも控えポケモンから `battle.random.choice(commands)` でランダムに交代先を選ぶ。
共通ヘルパー `_force_switch_random` を切り出し、両方の技から呼び出す。攻撃技型は `ON_DAMAGE_HIT` から直接 `ON_TRY_BLOW` をエミットして無効化チェックを行う。

交代先候補の取得には `handlers/move.py` の共有ヘルパー `get_forced_switch_commands(battle, player)` を使う。
このヘルパーは `CommandManager.get_available_switch_commands` を経由せず控えの生存ポケモンから直接コマンドを
構築するため、にげられない・バインド・フェアリーロックや特性かげふみ・ありじごく・じりょくによる交代制限を
無視して発動できる（強制交代技の仕様どおり）。

```python
def _force_switch_random(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """攻撃技型の強制交代共通ロジック（控えポケモンからランダムに選択）。"""
    result = battle.events.emit(Event.ON_TRY_BLOW, ctx, True)
    if not result:
        battle.add_event_log(ctx.attacker, LogCode.MOVE_IMMUNED)
        return HandlerReturn(value=value)
    player = battle.get_player(ctx.defender)
    state = battle.player_states[player]
    commands = get_forced_switch_commands(battle, player)
    if commands:
        command = battle.random.choice(commands)
        battle.run_switch(player, state.team[command.index])
    return HandlerReturn(value=value)


def ともえなげ_force_switch(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ともえなげ: ダメージ後に相手を強制交代させる（控えポケモンからランダムに選択）。"""
    return _force_switch_random(battle, ctx, value)


def ドラゴンテール_force_switch(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ドラゴンテール: ダメージ後に相手を強制交代させる（控えポケモンからランダムに選択）。"""
    return _force_switch_random(battle, ctx, value)
```

## 注意点・エッジケース（review-test 向け）

### きゅうばん・ねをはる無効化

- きゅうばん（特性）と ばんけん（特性）は `Event.ON_TRY_BLOW` に `defender:self` でハンドラを登録しており、`ON_TRY_BLOW` をエミットすると自動的に無効化される（実装済み: `handlers/ability.py` の `きゅうばん_block_blow`）
- ねをはる状態も `Event.ON_TRY_BLOW` に `defender:self` でハンドラを登録済み（`handlers/volatile.py` の `ねをはる_block_blow`、`data/volatile.py` の `"ねをはる"` エントリ）

### とらわれ状態・交代制限特性の無視

- 相手がにげられない/バインド状態であるときや、場がフェアリーロック状態であるとき、
  特性かげふみ/ありじごく/じりょくの効果を受けているときでも交代させることができる
  （強制交代技は交代制限を無視する）。
- `CommandManager.get_available_switch_commands` は `Query.can_switch`（`ON_CHECK_TRAPPED` 経由）を
  通すためこれらの状態を無視できない。そのため `handlers/move.py` の専用ヘルパー
  `get_forced_switch_commands(battle, player)` を使い、控えの生存ポケモンから直接コマンドを構築する
  （とらわれ状態の判定を一切経由しない）。

### ダイマックス無効化

- 仕様上「ダイマックスしている相手には追加効果が無効」だが、現状コードベースにはダイマックス状態を参照するフラグ（`is_dynamaxed` 等）が存在しない
- ダイマックス状態管理が実装された際には `_force_switch_random` 内に `if ctx.defender.is_dynamaxed: return HandlerReturn(value=value)` のガードを追加する
- 現時点では `ON_TRY_BLOW` 未登録のため、ダイマックスによる無効化は無視する（実装スコープ外）

### ほえる・ふきとばしとの違い

| 項目 | ほえる・ふきとばし | ともえなげ・ドラゴンテール |
|---|---|---|
| 種別 | 変化技 | 攻撃技 |
| 無効化チェックタイミング | `ON_BEFORE_APPLY_MOVE` (priority=30) | `ON_DAMAGE_HIT` 内で `ON_TRY_BLOW` エミット |
| まもる貫通 | 貫通 | 貫通しない |
| 交代先選択 | ランダム | ランダム（`blow` と同じロジック） |

### 控えが存在しない場合

- `get_forced_switch_commands` が空リストを返した場合は交代処理をスキップする（失敗扱い）
- ハンドラ側でガードを省略せず明示的に確認する（`commands` が空なら何もしない）

### みがわりと追加効果

- みがわり状態の相手にダメージを与えた場合、`ON_DAMAGE_HIT` は発火されない（みがわりに当たる場合は `ON_MODIFY_DAMAGE` で処理される）
- したがって みがわりを破壊した場合のみ強制交代が発動する（manager 側で保証済みのためハンドラ内のガード不要）

### secondary_effect ラベルと ちからずく

- 強制交代は「追加効果」扱いだが、ともえなげ・ドラゴンテールに `secondary_effect` ラベルは付けない
- 仕様上、ちからずくの影響を受けるかは不明確なため、現時点では `secondary_effect` ラベルなしで実装する
