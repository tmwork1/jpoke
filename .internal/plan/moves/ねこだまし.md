# ねこだまし 実装計画書

## 仕様要約（.internal/spec/moves/ねこだまし.md 参照）

* タイプ: ノーマル / 分類: 物理 / 威力: 40 / 命中: 100 / PP: 10
* 対象: 相手1体 / 優先度: +3 / 接触あり（直接攻撃）
* 効果: 100%の確率で相手をひるませる。場に出てから最初の行動として使用した場合のみ成功する。

## 現状の実装状況（修正前）

`src/jpoke/data/move.py` にはひるみ付与の `ON_DAMAGE_HIT` ハンドラのみが登録されており、
本技の根幹となる「場に出てから最初の行動でなければ失敗する」という成功条件が
一切実装されていなかった。

```python
"ねこだまし": MoveData(
    type="ノーマル",
    category="physical",
    pp=10,
    power=40,
    accuracy=100,
    priority=3,
    flags={"contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ねこだまし_apply_flinch,
        )

```

## 「場に出てから最初の行動か」の判定方法

`であいがしら`（`.internal/plan/moves/であいがしら.md` 参照）と全く同じ成功条件を持つため、
既存の `Pokemon.acted_since_switch_in` フラグと `であいがしら_check_first_turn` と
同型のハンドラをそのまま流用する。

* `model/pokemon.py` の `acted_since_switch_in` は既存実装済みのため変更不要。
* `core/move_executor.py` の `run_move()` の `finally` 節での更新も既存実装済みのため変更不要。

## ハンドラ構成（修正後）

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_TRY_MOVE_1` | 30 | attacker:self | 場に出てから最初の行動でなければ失敗させる |
| `ON_DAMAGE_HIT` | （既定値） | defender:self | 100%の確率でひるみを付与する（既存実装） |

```python
def ねこだまし_check_first_turn(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ねこだましの発動可否を判定する。

    場に出てから最初の行動でなければ不発になる。
    """
    if ctx.attacker.acted_since_switch_in:
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="ねこだまし")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

## Priority 根拠

* `.internal/spec/turn.md` の `Event.ON_TRY_MOVE_1` 節に「30 | ねこだまし等: すでに行動をした」
  という専用の行が存在する。よって `priority=30` を明示的に指定する
  （`であいがしら` と同じ priority）。

## 注意点・エッジケース（review-test 向け）

1. **基本ケース**
   - 場に出た直後の最初の行動として使用した場合は成功する（ひるみ付与）。
   - 何らかの技を1回使用した（成功・失敗を問わない）後に使用した場合は失敗する。
2. **状態異常等で最初の行動が潰れた場合**
   - まひ等で最初の行動ができなかった場合も「行動済み」扱いとなり、次のターンの
     ねこだましは失敗する（`であいがしら` と同様の挙動）。
3. **交代後の成功**
   - 交代で場に出た直後の最初の行動としてねこだましを使用した場合は成功する。
4. **未対応の既知エッジケース**
   - トリプルバトル・ローテーションバトル固有の例外、バッグ／ボールの使用、
     にげる失敗後の成功条件緩和は、本プロジェクトの対象（シングルバトル、
     アイテム使用・にげるコマンド無し）の範囲外のため未対応。
   - Champions のコマンド選択自体がキャンセルされる挙動は、`であいがしら` と同様に
     本プロジェクトでは対象外（選択自体は可能なまま不発とする）。
5. **テスト方針**
   - 既存の `test_ねこだまし_ひるみが発動する` に加え、
     `test_ねこだまし_場に出てから最初の行動でなければ失敗する` /
     `test_ねこだまし_交代後の最初の行動では成功する` を追加する。
   - `tests/moves_status/test_move_na.py` に混入していた重複テストは
     `tests/moves_attack/` 側に一本化する（本技は攻撃技であるため）。
