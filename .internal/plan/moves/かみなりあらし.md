# かみなりあらし 実装計画書

## 仕様要約（.internal/spec/moves/かみなりあらし.md 参照）

* タイプ: でんき / 分類: 特殊 / 威力: 100 / 命中: 80 / PP: 10
* 対象: 相手全員（本プロジェクトはシングルバトルのみ対象のため、実質的に相手1体） / 優先度: 0 / 接触なし
* 風の技（`flags={"wind"}`）
* 効果: 追加効果として、20%の確率で相手を『まひ』状態にする。
* 天気が『あめ』のときは必ず命中する（`かみなり`と異なり、にほんばれ時の命中率低下はない）。

## 現状の実装状況

`src/jpoke/data/move.py` に登録済み、`src/jpoke/handlers/move_attack.py` にまひ付与ハンドラ
（`かみなりあらし_apply_paralysis_to_defender`, chance=0.2）は実装済みだったが、
**あめ時に必ず命中する仕様が未実装**だった（`ON_MODIFY_ACCURACY` ハンドラが登録されていなかった）ため追加した。

```python
"かみなりあらし": MoveData(
    type="でんき",
    category="special",
    pp=10,
    power=100,
    accuracy=80,
    flags={"wind", "secondary_effect"},
    handlers={
        Event.ON_MODIFY_ACCURACY: h.MoveHandler(
            ha.かみなりあらし_accuracy,
            subject_spec="attacker:self"
        ),
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.かみなりあらし_apply_paralysis_to_defender,
        ),
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_MODIFY_ACCURACY` | 100（デフォルト） | attacker:self | 天候による命中率補正（あめ: 必中。それ以外は補正なし） |
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 20%の確率で相手を『まひ』状態にする |

`src/jpoke/handlers/move_attack.py` の実装:

```python
def かみなりあらし_accuracy(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """かみなりあらしの天候による命中率補正。あめのときのみ必中。
    かみなりと異なり、にほんばれ時の命中率低下はない。
    防御側がばんのうがさを持つ場合、あめでも必中にならない。
    """
    if battle.weather_for(ctx.defender).rainy:
        return HandlerReturn(value=None)  # 必中
    return HandlerReturn(value=value)


def かみなりあらし_apply_paralysis_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="まひ", chance=0.2)
```

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は汎用ハンドラ側で処理される。本技側に個別のガードは不要。

`battle.weather_for(mon)` はばんのうがさを持つポケモンに対してあめ系天候を「なし」として返すため、
防御側がばんのうがさを持っていても正しく無効化される（`かみなりあらし_accuracy` 側の実装で対応済み）。

## Priority 根拠

* `ON_MODIFY_ACCURACY` はダメージ計算内部のイベントであり `.internal/spec/turn.md` に priority 記載がないため、
  既存の同種ハンドラ（`かみなり_accuracy`）と同じデフォルト priority（100）のパターンに従う。
* `ON_DAMAGE_HIT` によるまひ付与も同様に、既存の同種ハンドラ（`かみなり`・`10まんボルト`・
  `かみなりのキバ` 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。

## 注意点・エッジケース（review-test 向け）

1. **天候による命中率補正**
   - あめ（防御側から見た天候）: 必中。ランク補正（回避・命中ランク）も無視される
     （`ON_MODIFY_ACCURACY` が `None` を返すと `_check_hit` がランク補正前にTrueを返すため）。
   - にほんばれ等その他の天候: 補正なし（`かみなり`と異なり命中率低下しない）。
2. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・まひ付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
3. **テスト方針**
   - `secondary_chance` を1.0に固定し、まひ付与を確認するテスト（`test_かみなりあらし_まひが発動する`）は既存。
   - 天候による命中率補正（あめ: 必中）を確認するテストを追加する
     （`test_かみなりあらし_あめ中は必中になる`）。
