# 計画書: やどりぎのタネ

## 仕様要約

- くさタイプ / 変化 / 命中90 / PP12 / 対象: 1体選択
- 相手を『やどりぎのタネ』状態にする。毎ターン終了時、相手の最大HPの1/8を減らし
  その分自分のHPを回復する（詳細は `.internal/spec/volatiles/やどりぎのタネ.md`）
- 自分が交代しても効果は引き継がれる（`やどりぎのタネ` 揮発性状態自体の仕様）
- くさタイプのポケモンには技が失敗する（技固有の失敗条件。専用ハンドラで判定）
- すでにやどりぎのタネ状態の場合は失敗する
  （`VolatileManager.apply()` の「既に同じ揮発性状態があれば失敗」という
  汎用ルールで自動的に処理されるため、専用の重複チェックハンドラは不要）
- そうしょく特性による無効化は特性側（`ability.py` の `そうしょく_absorb_grass`、
  `Event.ON_BEFORE_APPLY_MOVE`, subject_spec="defender:self"）が汎用的に処理するため
  技側の実装は不要

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_BEFORE_APPLY_MOVE` | 130 | attacker:self（省略時デフォルト） | くさタイプの対象には失敗させる |
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | 相手にやどりぎのタネ状態を付与 |

## Priority 根拠

`.internal/spec/turn.md` では「くさタイプ: やどりぎ無効」「やどりぎのタネ: すでにやどりぎ状態」は
いずれも `Event.ON_TRY_MOVE_2` の priority130 として記載されている。ただし本プロジェクトの
実装では、型・状態変化系の無効化判定は既存の `もりののろい` 等と同様に
`Event.ON_BEFORE_APPLY_MOVE`（コード内コメントでは「技の無効化チェック」を担当するイベント）に
統一して実装する慣例があり、priority値のみ turn.md の値（130）を踏襲する。
「すでにやどりぎ状態」の失敗は前述のとおり `VolatileManager.apply()` が汎用的に処理するため、
ハンドラ側では「くさタイプの対象には失敗」のみを実装する。
`ON_STATUS_HIT` priority は既存の状態変化付与技のパターン（デフォルト100）に従う。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `もりののろい` の後、`ゆきげしき` の前（五十音順）。

```python
def やどりぎのタネ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """やどりぎのタネの効果: 相手をやどりぎのタネ状態にする。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="やどりぎのタネ")


def やどりぎのタネ_can_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """やどりぎのタネの使用条件チェック: くさタイプのポケモンには失敗する。"""
    if ctx.defender.has_type("くさ"):
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="やどりぎのタネ")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### data/moves/move_ya.py への登録

## 揮発性状態側の実装（参考）

`やどりぎのタネ` 揮発性状態そのもの（ターン終了時のダメージ・回復処理、マジックガード、
ヘドロえき、おおきなねっこ、バトンタッチでの引き継ぎ、こうそくスピン・キラースピンによる
解除等）は `data/volatile.py` の `"やどりぎのタネ": VolatileData(...)` と
`handlers/volatile.py` の `やどりぎのタネ_drain_hp` で実装済み。本計画書の対象は
「技を当てて状態を付与する」部分のみであり、揮発性状態自体の実装は対象外
（別途 `.internal/progress/volatile.md` で管理）。

## 注意点・エッジケース

- くさタイプ無効化は技固有のハンドラ（`やどりぎのタネ_can_apply`）で実装する
  （どく/はがねタイプのように `ailment_manager` 側で一律判定される汎用ルールは存在しないため）
- そうしょく無効化・かたやぶり等での無視は特性側の実装に委ねる（技側の実装は不要）
- すでにやどりぎのタネ状態の失敗チェックは `VolatileManager.apply()` が汎用的に処理する
- 毎ターンのHP吸収・回復処理は `ON_TURN_END` priority80 で揮発性状態の効果として
  別途動作する（本ハンドラとは分離）
- 自分が交代しても効果が持続する点は揮発性状態側の実装（バトンタッチの引き継ぎ対象
  volatile リストへの登録、`from_mon`/`battle.foe(from_mon)` による位置ベースの
  ダメージ・回復対象決定）に委ねる
- `apply_volatile_to_defender` の標準ラッパーパターンを使用
