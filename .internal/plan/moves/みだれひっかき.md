# みだれひっかき 実装計画書

## 仕様要約（.internal/spec/moves/みだれひっかき.md 参照）

* タイプ: ノーマル / 分類: 物理 / 威力: 18（1ヒットあたり） / 命中: 80 / PP: 15
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果:
  * 2～5回連続で攻撃する可変回数の連続攻撃技（命中判定は1発目のみ、各ヒット威力は固定18）。
  * 急所判定は各ヒットごとに独立して行われる。
  * 追加効果なし。

## 現状の実装状況

`src/jpoke/data/moves/move_ma.py` に既に登録済み。

```python
"みだれひっかき": MoveData(
    type="ノーマル",
    category="physical",
    pp=15,
    power=18,
    accuracy=80,
    flags={"contact"},
    multi_hit={
        "min": 2,
        "max": 5,
        "check_hit_each_time": False,
        "power_sequence": (),
    },
    handlers={},  # 追加効果なし
),
```

内容は仕様書と一致しており、修正不要。

* `multi_hit.min=2, max=5` かつ `flags` に `check_hit_each_time` を含まないため、
  `core/move_executor.py` の `_resolve_hit_count` が `MULTI_HIT_DISTRIBUTION_2_TO_5`
  （2回37.5%、3回37.5%、4回12.5%、5回12.5%）に従ってヒット回数を決定し、
  命中判定は1発目のみ行われる（`スイープビンタ` と同一パターン）。
* `power_sequence` が空タプルのため、`_resolve_hit_power` は各ヒットとも `move.power`（18）を返す。
* 急所判定（`_check_critical`）はヒットごとに独立して呼び出されるため、既存の汎用機構のみで
  「各ヒットごとに独立して急所判定」という仕様を満たす。

## 注意点・エッジケース（review-test 向け）

1. **ヒット回数は2～5回の乱数分布**
   - `.internal/spec/moves/_multi_hit.md` の「2.2 可変回数技（2～5回）」の通り、
     `_resolve_hit_count` が乱数でヒット回数を決定する。テストでは戻り値の範囲（2〜5）を確認する。
2. **命中判定は1発目のみ**
   - `check_hit_each_time` フラグを持たないため、1発目が外れた場合は残りのヒットも発生しない。
3. **急所判定は各ヒットごと**
   - 急所ランク補正なし（`critical_rank=0`、デフォルト値）。既存の急所機構をそのまま利用する。
4. **テスト方針**
   - `_resolve_hit_count` の戻り値が2〜5の範囲に収まることを確認するテスト
     （`スイープビンタ` の複数ヒットテストと同一パターン）。
   - ダメージが発生することを確認する基本テスト。
