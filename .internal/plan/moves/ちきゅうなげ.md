# ちきゅうなげ 実装計画書（再レビュー）

## 仕様要約（.internal/spec/moves/ちきゅうなげ.md 参照）

* タイプ: かくとう / 分類: 物理 / 威力: -（固定ダメージ） / 命中: 100 / PP: 20
* 対象: 相手1体 / 優先度: 0 / 直接攻撃技
* 効果: 使用者のレベルと同じ固定ダメージを与える。本プロジェクトはレベル50固定のため実質常に50ダメージ。
* タイプ相性は無効化（ゴースト）のみ適用され、それ以外の相性補正は受けない（固定ダメージ技共通仕様）。

## 現状の実装状況

`src/jpoke/data/move.py` に既に登録済み。

```python
"ちきゅうなげ": MoveData(
    type="かくとう",
    category="physical",
    pp=20,
    power=0,
    accuracy=100,
    flags={"contact", "fixed_damage"},
    handlers={
        Event.ON_MODIFY_MOVE_DAMAGE: h.MoveHandler(
            ha.level_fixed_damage,
            subject_spec="attacker:self",
        )
    }
),
```

`handlers/move_attack.py` の `level_fixed_damage`（既存の共通ハンドラ。ナイトヘッドと共用）:

```python
def level_fixed_damage(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """使用者レベルと同値の固定ダメージを計算する。"""
    return HandlerReturn(value=ctx.attacker.level)
```

### 今回のレビューで検出した問題点（`いかりのまえば` と同種の priority バグ）

`.internal/plan/moves/いかりのまえば.md` で修正された問題と同じ原因が `ちきゅうなげ` にも
残っていた。`level_fixed_damage` が `priority` 未指定（デフォルト100）のまま
`Event.ON_MODIFY_MOVE_DAMAGE` に登録されているため、以下のハンドラより **後** に実行される。

- `みがわり_block_damage`（`data/volatile.py`, priority=20）
- `こらえる_endure`（`data/volatile.py`, priority=95）

固定ダメージ技は「通常のダメージ計算式を経由せず、`ON_MODIFY_MOVE_DAMAGE` ハンドラ内で
初めて基礎ダメージ値（レベル値）が確定する」という実装方式のため、`level_fixed_damage` は
「基礎ダメージの確定」の役割を持ち、みがわり・こらえる等の「確定済みダメージへの修正」
ハンドラより **前** に実行されなければならない。

#### 実機確認（修正前）

みがわり状態の相手にちきゅうなげを使用（本体HP110、みがわりHP27）:

```python
battle.volatile_manager.apply(defender, "みがわり", hp=27)
battle.step()
# 結果: 本体HPが110→60（50ダメージがそのまま本体に適用される）
#       みがわりHPは27のまま変化なし
# 期待値: みがわりが27ダメージ分肩代わりして消滅し、本体HPは110のまま変化しない
```

こらえる状態（残りHP1）でちきゅうなげを受ける:

```python
battle.modify_hp(defender, -(defender.hp - 1))  # 残りHP1
battle.volatile_manager.apply(defender, "こらえる")
battle.step()
# 結果: HPが1→0（ひんし）
# 期待値: こらえるが発動しHP1で耐える
```

いずれも `.internal/spec/turn.md` に掲載のない未掲載イベント（固定ダメージ技の基礎ダメージ確定）
における priority 未設定に起因する。

なお、`がんじょう`・`きあいのタスキ`（いずれも `ON_MODIFY_MOVE_DAMAGE` priority=100）との
競合は理論上あり得るが、レベル50における実際のHP実数値は
`floor((2*base+iv+ev/4)*level/100)+level+10` であり、`base=1`（最低値）でも
最小 HP は 76 程度になる（`level+10=60` が下限に加算されるため）。
ちきゅうなげの最大ダメージは使用者レベル（本プロジェクトでは50固定）であり、
どのポケモンも満タンHPが50を上回るため、満タンHPからの一撃ひんしは実質発生しない。
したがって `がんじょう`・`きあいのタスキ` との実害のある競合は生じない
（`いかりのまえば` の計画書における同種の記述を参照）。

## 修正方針

`level_fixed_damage` の登録に `priority=15` を追加する
（`みがわり`=20 より前、`ばけのかわ`=10 より後）。

```python
"ちきゅうなげ": MoveData(
    type="かくとう",
    category="physical",
    pp=20,
    power=0,
    accuracy=100,
    flags={"contact", "fixed_damage"},
    handlers={
        Event.ON_MODIFY_MOVE_DAMAGE: h.MoveHandler(
            ha.level_fixed_damage,
            subject_spec="attacker:self",
            priority=15,
        )
    }
),
```

`level_fixed_damage` はナイトヘッドとも共用のハンドラだが、`priority` は
`data/move.py` の登録側（`MoveHandler` 呼び出し）で個別に指定するため、
ナイトヘッド側の登録には影響しない（今回のタスク範囲外のため変更しない）。

## ハンドラ構成

| Event | 対象 | subject_spec | priority | 関数 |
|---|---|---|---|---|
| ON_MODIFY_MOVE_DAMAGE | ちきゅうなげ | attacker:self | 15 | `ha.level_fixed_damage` |

## Priority 根拠

`.internal/spec/turn.md` に `ON_MODIFY_MOVE_DAMAGE`（固定ダメージ技のダメージ確定）は
個別掲載されていない未掲載イベントのため、既存の同種ハンドラ
（`みがわり`=20, `こらえる`=95, `ばけのかわ`=10, `いかりのまえば`の`half_damage`=15）を
参照し、`みがわり`より前に基礎ダメージを確定させる必要から `priority=15` とした
（`いかりのまえば.md` と同じ値・同じ根拠）。

## 注意点・エッジケース（review-test 向け）

1. **ゴーストタイプへの無効化**
   - `_check_hit_by_type`（`move_executor.py`）が `calc_def_type_modifier` を
     参照して別途無効化判定を行うため、`fixed_damage` フラグや `power=0` とは
     独立して正しく無効化される。実装済み・修正不要（既存テストで確認済み）。

2. **みがわり・こらえるとの相互作用**
   - 上記の priority 修正により正しく動作することをテストで確認する。

3. **がんじょう・きあいのタスキ**
   - レベル50におけるHP実数値の下限（60程度）がちきゅうなげの最大ダメージ（50）を
     常に上回るため、満タンHPからの一撃ひんしは発生せず、実害のある競合は生じない。

4. **使用者レベル依存の固定ダメージ**
   - 本プロジェクトは全ポケモンレベル50固定（Pokemon Champions ルール）のため、
     実質常に50固定ダメージとなる。`level=50` を明示したテストで確認する。
