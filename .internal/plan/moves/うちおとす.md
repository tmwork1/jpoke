# 計画書: うちおとす

## 仕様要約（.internal/spec/moves/うちおとす.md 参照）

- いわタイプ / 物理 / 威力50 / 命中100 / PP15 / 対象: 1体選択 / 優先度0 / 非接触
- 命中時、地面にいない（浮いている）相手を「うちおとす」揮発性状態にして地面に落とす。
  元々地面にいる相手には何も付与しない。
- この状態付与は「追加効果」ではないため、ちからずく/りんぷん/おんみつマントの影響を受けない。
- そらをとぶ/とびはねる中の相手にも命中し、命中すれば中断させる。

## 既存実装との差分

既存実装（`handlers/move_attack.py` の `うちおとす_apply_grounded`）は
`apply_volatile_to_defender` ヘルパー経由で無条件にうちおとす状態を付与しており、以下2点が仕様と乖離している。

1. **地面にいる相手にも付与してしまう**
   `apply_volatile_to_defender` は対象の浮遊状態を確認せずに揮発性状態を付与するため、
   すでに地面にいる相手（例: ノーマルタイプでふゆうを持たない相手）にもうちおとす状態が付与される。
   → 命中前に `battle.query.is_floating(ctx.defender)` を確認し、
   浮いていない場合は何も付与せずに終了する必要がある。

2. **「追加効果」の確率解決経路を通ってしまう**
   `apply_volatile_to_defender` は内部で `battle.resolve_secondary_chance` を呼び出す。
   これは `Event.ON_MODIFY_SECONDARY_CHANCE` を発火し、りんぷん・おんみつマントの
   ハンドラ（`りんぷん_block_secondary_chance` / `おんみつマント_negate_secondary`）が
   無条件に確率を0にしてしまう。
   うちおとすの状態付与は「追加効果」ではないため、これらに阻害されてはならない。
   → `apply_volatile_to_defender` を使わず、`battle.volatile_manager.apply` を直接呼び出す
   （`battle.volatile_manager.apply` 自体は `Event.ON_BEFORE_APPLY_VOLATILE` のみを経由し、
   `ON_MODIFY_SECONDARY_CHANCE` は経由しないため、この問題を回避できる）。

3. **そらをとぶ中の相手に命中しない**
   `handlers/volatile.py` の `HIDDEN_MOVE_ALLOWED_MOVES["そらをとぶ"]` に
   `"うちおとす"` が含まれていないため、そらをとぶ滞空中の相手にうちおとすを撃っても
   `そらをとぶ_can_hit_hidden_target` によって必ず外れてしまう。
   → リストに `"うちおとす"` を追加する。

## ハンドラ構成（修正後）

### handlers/move_attack.py: `うちおとす_apply_grounded` の修正

```python
def うちおとす_apply_grounded(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """うちおとす: 命中時に相手にうちおとす揮発性状態を付与する。

    ひこうタイプ・ふゆう特性による浮遊状態を無効化する。
    元々地面にいる相手には付与しない。
    「追加効果」には該当しないため、ちからずく/りんぷん/おんみつマントの影響を受けない
    （resolve_secondary_chance を経由しないよう volatile_manager.apply を直接呼ぶ）。
    """
    if not battle.query.is_floating(ctx.defender):
        return HandlerReturn(value=value)
    battle.volatile_manager.apply(ctx.defender, "うちおとす", source=ctx.attacker)
    return HandlerReturn(value=value)
```

- イベント: `Event.ON_HIT`（既存のまま。.internal/spec/turn.md の ON_HIT priority=100
  「バインド状態付与」「ダイマックスわざの効果」と同種の“追加効果ではない技固有の効果”に該当し、
  priority省略時のデフォルト100と整合する）。
- subject_spec: 既存のまま（`MoveHandler` デフォルト、attacker側）。

## 注意点・エッジケース

- くろいてっきゅう/じゅうりょく状態などで「地面にいる」扱いになっている場合は、
  `battle.query.is_floating` が既にFalseを返すため、本修正の `is_floating` チェックのみで
  「地面にいる要因が優先される」仕様も自然に満たされる（個別のハンドラ追加は不要）。
- でんじふゆう/テレキネシス状態の相手に命中した場合、うちおとす状態を付与すれば
  `ON_CHECK_FLOATING` は "うちおとす" ハンドラが stop_event=True で確定させるため、
  でんじふゆう自体を明示的に解除しなくても浮遊判定は正しくFalseになる
  （テレキネシスは本プロジェクト未実装のため対応不要）。
- ふうせんを割ったことによる「地面に落ちる」効果とうちおとす状態付与の先後関係
  （どちらもON_HIT priority=100）は、本フレームワークの既存仕様（同priorityは素早さ順で不定）に委ね、
  本技専用の追加対応は行わない。
- 仕様上、そらをとぶ滞空中にうちおとす（またはかぜおこし等の既存の命中可能技）が命中すると
  そらをとぶは中断され2ターン目の攻撃を行わずその場でダメージを受けるはずだが、
  現状の実装（`そらをとぶ_remove_volatile` は `subject_spec="attacker:self"` で登録されており、
  そらをとぶ状態のポケモン自身が攻撃した後に解除する用途にのみ使われている）を確認した結果、
  かぜおこし等の既存の命中可能技を含め、この中断処理自体が未実装であることが判明した。
  これはそらをとぶ全体に関わる既存の仕様ギャップであり、うちおとす固有の問題ではないため、
  本計画のスコープ外として対応しない（別途そらをとぶのレビュー時に扱う）。
