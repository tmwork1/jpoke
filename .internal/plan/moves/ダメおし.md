# ダメおし 実装計画書

参照: .internal/spec/moves/ダメおし.md

## 現状（レビュー前）

* `src/jpoke/data/moves/move_ta.py` の `"ダメおし"` エントリは実装済みだったが、以下の不備があった。
  - `pp=10`（本家Gen9基準の移行漏れ）。`src/jpoke/data/ps-champ-ja/moves.json` では `12` であり、
    `.internal/champions/changes_from_sv.md` の「全技PPは8/12/16/20の4段階に統一」方針に反していた。
  - `Event.ON_CALC_POWER_MODIFIER` に `ダメおし_double_power_when_hit` を登録し、
    `ctx.defender.hits_taken > 0` を条件に威力を2倍にする実装自体は仕様通りだったが、
    `hits_taken` の加算漏れ（後述）により一部のケースで威力2倍が発動しなかった。

## 修正内容

### data/moves/move_ta.py

* `pp=10` → `pp=12` に修正（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）。

### handlers/move_attack.py（変更なし）

```python
def ダメおし_double_power_when_hit(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ダメおし: 同ターン中に対象が既にダメージを受けていたら威力が2倍になる。"""
    if ctx.defender.hits_taken > 0:
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)
```

* `subject_spec` はデフォルト（`MoveHandler` の `attacker:self`）のままでよい。攻撃側の技効果として
  発動し、`ctx.defender`（対象）の状態を参照するため、`AttackContext` の役割上問題ない。
* `Event.ON_CALC_POWER_MODIFIER` に priority 指定なし（デフォルト）。`.internal/spec/turn.md` に
  本イベント専用の記載はないが、既存の同種ハンドラ（`テクニシャン_boost_power`、
  `からげんき_calc_power` 等）もすべてデフォルト priority のため、これに合わせる。

### hits_taken の加算漏れ修正（`ダメおし` の判定条件の前提となる共有ロジック）

`.internal/spec/moves/ダメおし.md`「実装上の制約」節に記載の通り、一次情報の「技の仕様」節に
挙げられている以下のダメージ源で `Pokemon.hits_taken` が加算されていなかったため、
既存の類似実装（`みねうち_modify_damage`、`ばけのかわ_block_damage`）と同じパターンで
手動インクリメントを追加した。

1. **`こらえる_endure`**（`src/jpoke/handlers/volatile.py`）: HP1で実ダメージが0になる場合、
   `mon.hits_taken += 1` を追加。`みねうち_modify_damage` と同じパターン。
2. **`アイスフェイス_block_physical`**（`src/jpoke/handlers/ability.py`）: 物理技のダメージを
   肩代わりする際、`ctx.defender.hits_taken += 1` を追加。`ばけのかわ_block_damage` と同じパターン。
3. **反動ダメージ全般**（`core/status_manager.py` の `modify_hp`）: `reason in ("recoil",
   "fixed_recoil")` のとき `target.hits_taken += 1` する処理を、`v < 0` 判定ブロック内に追加。
   これにより `_recoil()`（すてみタックル系共通ヘルパー、`handlers/move_attack.py`）、
   クロロブラスト、とびげり/とびひざげり/サンダーダイブの失敗時反動（`reason="fixed_recoil"`）、
   てっていこうせんの反動を一括でカバーできる（ビックリヘッドは第9世代に存在しないため
   本プロジェクトでは実装対象外）。
   `move_damage`（通常の攻撃技ヒット）はこの処理に含めない
   （`core/move_executor.py` 側で既存の専用インクリメント処理があるため、二重加算を避ける）。
4. **いのちのたまの反動ダメージ**（`いのちのたま_recoil`、`src/jpoke/handlers/item.py`）:
   `ctx.attacker.hits_taken += 1` を追加。`reason` は付与しない（`reason="recoil"` にすると
   いしあたまで無効化されてしまい、いのちのたまの反動が本来ブロックされない一次情報の仕様に
   反するため、`status_manager.modify_hp` 側の一括処理ではなく個別インクリメントで対応する）。
5. **`_apply_contact_item_chip`**（`src/jpoke/handlers/item.py`、ゴツゴツメット用ヘルパー）:
   ダメージが実際に発生した場合 `ctx.attacker.hits_taken += 1` を追加。
6. **`_retaliate_on_category`**（`src/jpoke/handlers/item.py`、ジャポのみ/レンブのみ用ヘルパー）:
   同様に `ctx.attacker.hits_taken += 1` を追加。
7. **`_apply_contact_counter_chip`**（`src/jpoke/handlers/ability.py`、さめはだ/てつのトゲ用
   ヘルパー）: 同様に `ctx.attacker.hits_taken += 1` を追加。

いずれも「実際にダメージ（0ダメージを含む攻撃の肩代わり）が発生した場合のみ加算し、
無効化された場合は加算しない」という既存の `hits_taken` の意味論を維持している。
これらの修正は `きあいパンチ`（`hits_taken` を不発判定に使用）の挙動にも影響するが、
「そのターンに攻撃ダメージ（肩代わり・反動含む）を受けていれば不発する」という
`きあいパンチ` の一次情報上の仕様とも整合するため、副作用ではなく straight-up な不具合修正である。

### 対応を見送った既知の限界

`.internal/spec/moves/ダメおし.md`「実装上の制約」節を参照。わるあがきの反動、ゆうばく/とびだすなかみ/
うのミサイル、ターン中強制交代からの設置技ダメージは、実装コストに対して発生頻度が極めて低い
エッジケースと判断し、対応を見送った。
