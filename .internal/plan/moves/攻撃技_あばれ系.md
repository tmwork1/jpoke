# 計画書: 攻撃技_あばれ系（あばれる・げきりん・だいふんげき・はなびらのまい）

## 仕様要約

- 4技とも「2〜3ターン連続で同じ技を強制使用し、終了後に使用者が混乱する」という共通仕様
- 初回使用時に `あばれる` 揮発性状態を付与し、ランダムに 2〜3 ターン分のカウントを設定する
- `あばれる` 状態中は `Command.FORCED` が強制返却され、`volatile.move_name` に記録された技名を使い続ける
- カウントが 0 になった（揮発性状態が解除された）直後に混乱を付与する

| 技名 | タイプ | 区分 | 威力 | 命中 |
|------|--------|------|------|------|
| あばれる | ノーマル | 物理 | 120 | 100 |
| げきりん | ドラゴン | 物理 | 120 | 100 |
| だいふんげき | ほのお | 特殊 | 90 | 100 |
| はなびらのまい | フェアリー | 特殊 | 120 | 100 |

### `あばれる_apply` の動作詳細

```python
def あばれる_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """あばれる系技の初回命中時に揮発性状態を付与する。

    あばれる・げきりん・はなびらのまい・だいふんげきで共用。
    すでにあばれる状態の場合（強制行動の2ターン目以降）は何もしない。

    ターン数は 2〜3 ターンのランダム（最初の使用時に決定）。
    """
    attacker = ctx.attacker
    if attacker.has_volatile("あばれる"):
        return HandlerReturn(value=value)
    count = battle.random.randint(2, 3)
    battle.volatile_manager.apply(
        attacker, "あばれる", count=count, source=attacker, move_name=ctx.move.name
    )
    return HandlerReturn(value=value)
```

`move_name=ctx.move.name` で技名を揮発性状態に記録することで、`get_forced_move_name` → `command_manager` により強制行動時に同じ技が選択される。

## Priority 根拠

- `ON_HIT` のデフォルト priority: `.internal/spec/turn.md` の `Event.ON_DAMAGE` に「追加効果（特殊なもの除く） priority=10」と記載。ただし `ON_HIT` と `ON_DAMAGE` は別イベント。MoveHandler のデフォルト priority は既存の類似ハンドラ（例: `かみつく_apply_volatile_to_defender`、`フレアドライブ_recoil` 等）を参照し、priority 指定なし（内部的に 100 として扱われる）。
- `ON_DAMAGE_HIT` priority 180: `.internal/spec/turn.md` の `Event.ON_DAMAGE` section に「あばれる状態終了でこんらん: priority 180」と明記されている。

## 注意点・エッジケース（review-test 向け）

### 1. 強制行動の仕組み
`force_command` が `ON_MODIFY_COMMAND_OPTIONS` で `Command.FORCED` を返す。`command_manager` は `Command.FORCED` を受け取ると `query.get_forced_move_name(attacker)` を呼び、`volatile.move_name` に記録された技名（`"げきりん"` 等）を取得して `Move(move_name)` を生成する。PP は消費しない。

### 2. すでにあばれる状態中の初回命中
`あばれる_apply` 内の `if attacker.has_volatile("あばれる")` ガードにより 2ターン目以降の命中時は二重付与されない。

### 3. カウント管理
`volatile_manager.tick` を呼ぶと count が 1 減少し、count が 0 になると揮発性状態が自動除去される。`あばれる_tick` では `volatile_manager.tick(mon, "あばれる")` の後に `not mon.has_volatile("あばれる")` を確認し、状態が除去されていたら `apply_confusion` を呼ぶ。

### 4. ねむりによるあばれる中断
`.internal/spec/turn.md` の `Event.ON_TURN_END` priority 150 に「ねむりによるあばれる中断」が記載されている。この処理は現在の `あばれる` volatile 実装に含まれていない可能性がある（要確認）。ねむり状態になった場合に `あばれる` 状態を解除するハンドラが必要かもしれない。

### 5. `だいふんげき` のカテゴリ誤記
`data/move.py` 内で `"だいふんげき"` の `category` が「物理」になっている（仕様は「特殊」）。ハンドラ追加と同時に修正が必要。

### 6. `だいふんげき` の威力
`.internal/progress/move.md` では威力 120 と記載されているが、仕様（ポケモン公式）では威力 90。`data/move.py` の現在値 120 は誤記の可能性があるため確認が必要。

### 7. 混乱の継続ターン数
`apply_confusion` の実装で混乱ターン数がどのように設定されるか（1〜4ターン）を確認しておくこと。仕様では「1〜4ターン」の混乱。

### 8. みがわり状態と `ON_HIT`
命中時の `ON_HIT` は `みがわり` に当たった場合でも発動する（event コメントより: 「ダメージ発生後の処理（みがわりに被弾しても発動）」）。`あばれる_apply` はみがわりを貫通して揮発性状態を付与する設計となる。実際の仕様を確認すること。

### 9. `さわぐ` との類似性の参照
`さわぐ` は同じく `forced=True` でカウント管理・強制行動を行っており、`さわぐ_tick_volatile` と類似した処理。ただし `さわぐ` はターン終了時カウント、`あばれる` はダメージ後カウントという違いがある。
