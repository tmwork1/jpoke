## 仕様要約（.internal/spec/moves/コールドフレア.md より）

- こおりタイプ・特殊・威力140・命中90・PP5・対象1体・優先度0・非接触
- 2ターン技。1ターン目は攻撃せず「コールドフレア」揮発状態になる（溜めターン）。
  - `あなをほる` / `そらをとぶ` / `ダイビング` / `シャドーダイブ`（隠れる技）とは異なり、
    姿を隠さない。回避判定（`can_hit_hidden_target` 相当）は不要。
  - `ソーラービーム` / `ソーラーブレード` / `メテオビーム` / `エレクトロビーム` と同じ
    「その場で溜める」技グループ。
- 2ターン目に攻撃する。命中判定・急所判定は攻撃ターンのみ。
- 追加効果として、命中時に30%の確率で相手をやけど状態にする（`ちからずく` の対象になる通常の追加効果）。
- まもる: ○、マジックコート: ×（変化技でないため対象外）、よこどり: ×（変化技でないため対象外）、
  オウムがえし: ○
- PPは `src/jpoke/data/ps-champ-ja/moves.json` に記載がないため、本家（Wiki）の値である5を採用する。

## 採用した実装方針

コールドフレアは「姿を隠さない・天候スキップがない・チャージ中に追加効果（能力上昇等）がない」
最もシンプルな溜め技であるため、`あなをほる` の charge 登録スタイルと
`ソーラービーム`/`メテオビーム`/`エレクトロビーム` の VolatileData 構成（隠れる技グループの
`ON_CHECK_TRAPPED` ・ 回避判定は持たない）を組み合わせて実装した。

### data/moves/move_ka.py: MoveData（技ターン制御）

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_MOVE_CHARGE` | 100（デフォルト） | `attacker:self`（デフォルト） | 1ターン目: 「コールドフレア」揮発状態を付与して技を止める。2ターン目: 素通り |
| `ON_DAMAGE_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 命中時30%でやけど付与（既存実装、変更なし） |

```python
"コールドフレア": MoveData(
    type="こおり",
    category="special",
    pp=5,
    power=140,
    accuracy=90,
    flags={"secondary_effect"},
    handlers={

            lambda b, c, v: h.charge_into_volatile(b, c, v, "コールドフレア"),
        ),
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.コールドフレア_apply_burn_to_defender,
        )
    }
),
```

### data/volatile.py: VolatileData「コールドフレア」（揮発状態ハンドラ）

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_MODIFY_COMMAND_OPTIONS` | 100（デフォルト） | `source:self` | 行動を FORCED 固定・stop_event=True |
| `ON_HIT` | 100（デフォルト） | `attacker:self` | 攻撃完了後に揮発状態を解除する |

```python
        ),

    }

```

## Priority 根拠

### ON_MOVE_CHARGE (priority=100・デフォルト)

`.internal/spec/turn.md` の `Event.ON_MOVE_CHARGE` テーブルに「隠れる技: 状態変化」が priority=100 として
記載されている。コールドフレアは姿を隠さないが、揮発状態を付与してターンを止めるという処理内容は
同一であり、`ソーラービーム` / `メテオビーム` / `エレクトロビーム` の charge ハンドラも同じ
priority=100（デフォルト）を採用しているため、これに合わせた。

パワフルハーブ（`data/item.py` の `ON_MOVE_CHARGE`、`ItemHandler` デフォルト priority=100）との
順序は、item ハンドラが move ハンドラより先に登録される実装上の慣例（`エレクトロビーム` 系のコメント
参照）により、同一 priority でもパワフルハーブが先に処理され、溜めをスキップして即攻撃に切り替わる。
コールドフレア固有の追加処理は不要。

### ON_MODIFY_COMMAND_OPTIONS / ON_HIT（揮発状態側）

`.internal/spec/turn.md` に記載がないイベントのため、CLAUDE.md の方針に従い既存の同種ハンドラ
（`ソーラービーム` / `メテオビーム` / `エレクトロビーム` の VolatileData）を参照し、同一の
priority=100（デフォルト）・subject_spec を採用した。

### ON_CHECK_TRAPPED / 回避判定を持たない根拠

一次情報（Wiki）の判定表に「回避」に関する記載がなく、姿を隠す効果は無いため、
`あなをほる` / `そらをとぶ` / `ダイビング` / `シャドーダイブ` が持つ `ON_CHECK_TRAPPED`
（`check_trapped_not_ghost`）や `ON_TRY_MOVE_1` の `can_hit_hidden_target` は登録しない。
これは `ソーラービーム` / `メテオビーム` / `エレクトロビーム` と同じ扱いである。
