# 計画書: せいちょう

## 仕様要約

- 自分の「こうげき」「とくこう」ランクがそれぞれ1段階ずつ上がる
- 天候が「にほんばれ」（はれ / おおひでり）の時は2段階ずつ上がる
- タイプ: くさ（LEGENDS Z-A・Championsでノーマルから変更）、分類: 変化、PP: 20、対象: 自分

出典: `.internal/spec/moves/せいちょう.md`、`src/jpoke/data/ps-champ-ja/moves.json`

---

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100 (省略) | `attacker:self` (省略) | 天候に応じてこうげき・とくこうを1段階または2段階上げる |

## Priority 根拠

`.internal/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`つるぎのまい_modify_attacker_stats` 等）がすべて priority=100 (デフォルト) で登録されているため、100 を使用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: `スレッドトラップ_apply` の後（「せ」行は「す」の後、「そうでん」の前）

天候チェックは `battle.weather_for(ctx.attacker)`（使用者視点の有効天候）を使う。これにより
以下が自動的に考慮される:

- 使用者が **ばんのうがさ** を持つ場合、にほんばれ/おおひでりの補正を無効化（1段階扱い）
- 場に **ノーてんき/エアロック** 持ちがいる場合、天候効果自体を無効化（1段階扱い）

`weather.sunny` は「にほんばれ」と「おおひでり」の両方をカバーしている。

```python
def せいちょう_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """せいちょう: 天候がにほんばれ・おおひでりの時はこうげき・とくこうを2段階、通常時は1段階上げる。
    攻撃側がばんのうがさを持つ場合、晴れでも1段階のみ。
    """
    n = 2 if battle.weather_for(ctx.attacker).sunny else 1
    return modify_attacker_stats(battle, ctx, value, stats={"atk": n, "spa": n})
```

### `src/jpoke/data/moves/move_sa.py`

`"せいちょう"` エントリ:

```python
"せいちょう": MoveData(
    type="くさ",
    category="status",
    pp=20,
    target="self",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.せいちょう_modify_attacker_stats,
        ),
    }
),
```

## 注意点・エッジケース

- `target="self"` を明示しないと `MoveData.target` のデフォルト値 `"foe"` になり、
  `is_blocked_by_protect`（まもる）・マジックコートの判定が `move.target == "foe"` を
  条件にしているため、誤って「まもるで防がれる」「マジックコートで跳ね返される」動作になる
- `type` は第一世代〜スカーレット・バイオレットまでノーマルタイプだったが、LEGENDS Z-A・
  Championsではくさタイプに変更されているため `"くさ"` を指定する
- `battle.weather_for(mon).sunny` プロパティが「にほんばれ」と「おおひでり」の両方を `True` として返すことを確認済み（`あさのひざし_heal_self` で同じプロパティを使用）
- 天候チェックは `ON_STATUS_HIT` ハンドラ内で行う（技発動時点の天候を参照）
- ランクがすでに最大（+6）の場合の失敗は `battle.modify_stats` 内部処理でハンドリングされるため、ハンドラ側にガード不要
- 晴れ時にこうげきまたはとくこうのどちらかがすでに+6の場合、その上昇のみ失敗し、もう一方は成功する
- 「エアロック」「ノーてんき」特性・「ばんのうがさ」による天候無効化は `battle.weather_for(mon)` が
  汎用的に処理するため、ハンドラ内追加対応不要
- 使用者の特性が「たんじゅん」の場合、`battle.modify_stats` が倍化を汎用的に処理するため、
  にほんばれ中は4段階上昇する（ハンドラ側追加対応不要）
