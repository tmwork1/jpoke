# らいげき 実装計画書

* タイプ: でんき / 分類: 物理 / 威力: 130 / 命中: 85 / PP: 5

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 20%の確率で相手を『まひ』状態にする |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def らいげき_apply_paralysis_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="まひ", chance=0.2)
```

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を
経由するため、ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は汎用ハンドラ側で
処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（状態異常付与）は既存の同種ハンドラ（かみなり・かみなりパンチ・
かみなりのキバ 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。個別の
priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **おうじゃのしるし・するどいキバとの相互作用**
   - らいげきは元々ひるみの追加効果を持たないため、`handlers/item.py` の
     `_INNATE_FLINCH_MOVES` には含まれない（含めるべきでもない）。おうじゃのしるし・
     するどいキバ所持ポケモンが使用すると、20%のまひ判定とは独立に10%の追加ひるみ判定が
     行われる（汎用処理のため個別実装不要）。
2. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・まひの追加効果は
     発動しなくなる。既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
3. **テスト方針**
   - `secondary_chance` を1に固定し、まひの付与を確認するテストが既存
     （`test_らいげき_まひが発動する`）。
