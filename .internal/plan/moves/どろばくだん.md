# 計画書: どろばくだん

## 仕様要約（.internal/spec/moves/どろばくだん.md 参照）

- じめんタイプ / 特殊 / 威力65 / 命中85 / PP10 /
  対象: 相手1体 / 優先度0 / 接触なし
- `bullet`（弾のわざ）フラグを持つ。ぼうだん特性で無効化される。
- 追加効果として、命中時に30%の確率で相手の命中率を1段階下げる。
- 追加効果に分類されるため、ちからずく（威力1.3倍・効果無効化）の対象技になる。

## 既存実装との差分

既存実装（`data/move.py` の `"どろばくだん"` エントリ）は仕様と一致しており、修正は不要。

1. **基本情報は仕様通り**
   `type="じめん", category="special", pp=10, power=65, accuracy=85` で一致。

2. **追加効果ハンドラ本体（`どろばくだん_lower_acc`）は仕様通り実装済み**
   `modify_defender_stats(battle, ctx, value, stats={"accuracy": -1}, chance=0.3)`
   で、30%発動・命中率-1段階の仕様と一致している。変更不要。

3. **`flags` は既に `{"bullet", "secondary_effect"}` が設定済み**
   弾のわざ（ぼうだん対象）かつ追加効果を持つ技として正しく分類されている
   （`contact` フラグは付与されておらず、直接攻撃ではない仕様と一致）。変更不要。

## ハンドラ構成

### handlers/move_attack.py: `どろばくだん_lower_acc`（既存・変更なし）

```python
def どろばくだん_lower_acc(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"accuracy": -1}, chance=0.3)
```

- `modify_defender_stats`（`handlers/move.py`）は内部で `battle.resolve_secondary_chance` を
  呼び出すため、`Event.ON_MODIFY_SECONDARY_CHANCE` 経由でちからずく（確率0化・威力1.3倍）の
  ハンドラが自動的に適用される（`secondary_effect` フラグが前提）。
- イベント: `Event.ON_DAMAGE_HIT`（ダメージが実際に適用された後に発動。
  .internal/spec/turn.md の `## Event.ON_DAMAGE` 参照）。
  ダメージが発生しなかった場合（無効化・まもる・ぼうだん等）はイベント自体が発火しないため、
  追加効果も発生しない仕様を自然に満たす。
- 優先度: デフォルト100（`MoveHandler` のデフォルト値）。
- subject_spec: `MoveHandler` デフォルト（`attacker:self`）のまま。

### data/move.py（既存・変更なし）

```python
"どろばくだん": MoveData(
    type="じめん",
    category="special",
    pp=10,
    power=65,
    accuracy=85,
    flags={"bullet", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.どろばくだん_lower_acc,
        )
    }
),
```

影響しない追加効果のためリーサル計算ハンドラは不要）。

## 注意点・エッジケース

- じめんタイプの技が無効化されるタイプ（ひこうタイプ・ふゆう特性等）の相手など、技自体が
  無効化された場合、`Event.ON_DAMAGE_HIT` 自体が発火しないため追加効果も発生しない。
  個別のガードは不要。
- まもるで防がれた場合、およびぼうだん特性で無効化された場合も同様にダメージが発生せず
  `Event.ON_DAMAGE_HIT` が発火しないため、追加効果も発生しない。
- おうじゃのしるし・するどいキバの10%ひるみ追加抽選（`handlers/item.py` の
  `flinch_on_hit_10pct`）は `_INNATE_FLINCH_MOVES` に「どろばくだん」が含まれないため、
  本技の命中率低下効果とは独立して常に判定される。どろばくだん側の実装への影響はない。
