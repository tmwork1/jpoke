# 計画書: てっぺき

## 仕様要約

- 自分の「ぼうぎょ」ランクを2段階上げる
- タイプ: はがね、変化、PP: 16（champions 基準）、対象: 自分
- すでに+6の場合はランクが変化せず技は失敗する

出典: `.internal/spec/moves/てっぺき.md`

---

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100 (省略) | `attacker:self` (省略) | ぼうぎょ+2 を適用 |

---

## Priority 根拠

`.internal/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`つるぎのまい_modify_attacker_stats` 等）がすべて priority=100
（デフォルト）で登録されているため、100 を使用する。

---

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: `つるぎのまい_modify_attacker_stats` の後（「つるぎのまい」の次が「てっぺき」、五十音順）

```python
def てっぺき_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"def": 2})
```

### `src/jpoke/data/moves/move_ta.py`

`"てっぺき"` エントリに `target="self"` を追加する（`まもる`/`マジックコート` 誤反応防止）。

```python
    "てっぺき": MoveData(
        type="はがね",
        category="status",
        pp=16,
        target="self",
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.てっぺき_modify_attacker_stats,
            )
        }
    ),
```

---

## 注意点・エッジケース

- `MoveData.target` のデフォルトは `"foe"` のため、自分対象の変化技には必ず `target="self"` を
  明記する（省略すると `is_blocked_by_protect` / `is_reflectable` が誤って `True` になり、
  まもる・マジックコートに誤反応する）
- `turn.md` Priority 130「ランク補正上げ: すでに最大」はマネージャー内部で処理されるため、
  ハンドラ側にガード不要
