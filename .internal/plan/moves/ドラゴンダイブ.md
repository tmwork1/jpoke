# ドラゴンダイブ 実装計画書

## 仕様要約（.internal/spec/moves/ドラゴンダイブ.md 参照）

* タイプ: ドラゴン / 分類: 物理 / 威力: 100 / 命中: 75 / PP: 12（Champions）
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 追加効果として20%の確率で相手をひるませる。ちいさくなる状態の相手には必ず命中し、
  ダメージ（威力）が2倍になる（`minimize` フラグにより汎用処理）。

## 現状の実装状況

`src/jpoke/data/moves/move_ta.py` に登録済み（レビューにより `pp` の値誤り（Gen9本家基準の10のまま）を
Champions基準の12に修正）。

```python
"ドラゴンダイブ": MoveData(
    type="ドラゴン",
    category="physical",
    pp=12,
    power=100,
    accuracy=75,
    flags={"minimize", "contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ドラゴンダイブ_apply_flinch,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 20%の確率で相手をひるませる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def ドラゴンダイブ_apply_flinch(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_defender(battle, ctx, value, volatile="ひるみ", chance=0.2)
```

`apply_volatile_to_defender`（`handlers/move.py`）は `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）は汎用的に処理される。本技側に個別のガードは不要。

ちいさくなる状態の相手に対する必中・威力2倍は `flags={"minimize"}` により
`handlers/volatile.py` の `ちいさくなる_boost_power` / `ちいさくなる_guaranteed_hit` が
汎用的に処理するため、本技固有のハンドラは不要（`ふみつけ` 等の他 `minimize` フラグ技と共通）。

## Priority 根拠

ダメージを与えた技の追加効果（ひるみ付与）は既存の同種ハンドラ（あくのはどう・ずつき 等）と
同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。`.internal/spec/turn.md` にも
`ON_DAMAGE_HIT` は他の追加効果技と共通のタイミングとして扱われており、個別の priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"contact", "secondary_effect"}` が必要。ちからずく所持時は `resolve_secondary_chance` が
     確率を0にし、ひるみが発動しなくなる（威力1.3倍化は `ちからずく_boost` 側で処理）。
2. **威力・命中・PPはChampions基準**
   - `src/jpoke/data/ps-champ-ja/moves.json` により PP12 に統一されている（Gen9本家のPP10ではない）。
3. **ちいさくなるとの相互作用**
   - `minimize` フラグにより必中・威力2倍。専用テストは `tests/volatiles/test_volatile_satana.py` の
     `minimize_enhance_moves` パラメータ化テストでカバーする。
4. **テスト方針**
   - `secondary_chance=1.0` でひるみ付与を確認する（`tests/moves_attack/test_move_ta.py`）。
