# 実装計画: ギアチェンジ

## 仕様サマリ

- タイプ: はがね / 分類: 変化 / 威力: - / 命中率: - / PP: 10 / 対象: 自分
- 効果: 自分のこうげきランクを1段階、すばやさランクを2段階上げる
- まもる: 防がれない（自分対象） / マジックコート: 反射不可（自分対象） / 直接攻撃: いいえ
- よこどり: 対象になる（○）

出典: .internal/spec/moves/ギアチェンジ.md

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | こうげき+1、すばやさ+2 |

## Priority 根拠

`.internal/spec/turn.md` には `ON_STATUS_HIT` の詳細表はないが、既存の `りゅうのまい_modify_attacker_stats` /
`ちょうのまい_modify_attacker_stats` / `つるぎのまい_modify_attacker_stats`（いずれも `ON_STATUS_HIT`
priority 100 = デフォルト）が同じ自己強化ランクアップパターンであるため、同じ priority を採用する。
ランク上限チェック（すでに最大のとき失敗）は `battle.modify_stats()` 側でマネージャー内部処理される
ため、ハンドラ側にガード不要。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `きんぞくおん_modify_defender_stats` の後、`くすぐる_modify_defender_stats` の前（五十音順。
最終的な並びは `scripts/sort_handlers.py` で整形する）。

```python
def ギアチェンジ_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ギアチェンジの効果: 自分のこうげきを1段階、すばやさを2段階上げる。"""
    return modify_attacker_stats(battle, ctx, value, stats={"atk": 1, "spe": 2})
```

### data/moves/move_ka.py の更新

`target="self"` を指定する（`target` を指定しないとデフォルトの `"foe"` になり、
`is_blocked_by_protect` / `is_reflectable` がまもる・マジックコートに誤反応してしまうため必須）。
既存実装は `handlers={}` の空スタブ（コメント「追加効果なし」）になっており、効果が
一切実装されていなかったため修正する。

```python
"ギアチェンジ": MoveData(
    type="はがね",
    category="status",
    pp=10,
    target="self",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ギアチェンジ_modify_attacker_stats,
        ),
    }
),
```

## 注意点・エッジケース

- `target="self"` を指定し忘れると、まもる・マジックコートの判定にデフォルト値 `"foe"` の挙動が
  誤って適用されるため、必ず指定する
- こうげき・すばやさともに+6の場合の失敗は `battle.modify_stats()` 側でマネージャー内部処理されるため、
  ハンドラ内ガード不要
- 上昇幅がこうげき+1・すばやさ+2と非対称な点が `りゅうのまい`（各+1）や `つるぎのまい`（こうげき+2のみ）
  と異なるため、`stats` 引数の値取り違えに注意する
- `modify_attacker_stats` ヘルパーを利用することで、ランク変化ログ出力まで一括処理される
