# 計画書: こごえるかぜ

## 仕様要約（.internal/spec/moves/こごえるかぜ.md 参照）

- こおりタイプ / 特殊 / 威力55 / 命中95 / PP16（Champions基準。本家は15） /
  対象: 1体選択（本家は相手全員だが本プロジェクトはシングル専用） / 優先度0 / 非接触
- 追加効果として、命中時に相手の『すばやさ』ランクを1段階下げる（100%）。
- 追加効果に分類されるため、ちからずく（威力1.3倍・効果無効化）の対象技になる。
- `wind`（風技）フラグを持つ。

## 既存実装との差分

既存実装（`data/move.py` の `"こごえるかぜ"` エントリ）は以下の点で仕様と乖離していた。

1. **PPがChampions基準になっていない**
   既存: `pp=15`（本家の値）。
   → `src/jpoke/data/ps-champ-ja/moves.json` に準拠し `pp=16` に修正する。

2. **`secondary_effect` フラグが付与されていない**
   既存: `flags={"wind"}` のみ。
   → 100%の追加効果（すばやさダウン）を持つため、ちからずくの対象技として扱う
     `flags={"wind", "secondary_effect"}` に修正する
     （`あわ` / `あやしいかぜ` / `かみなりあらし` 等、同様に追加効果を持つ風技の既存実装に倣う）。

3. **追加効果ハンドラ本体（`こごえるかぜ_lower_defender_spd`）は仕様通り実装済み**
   `modify_defender_stats(battle, ctx, value, stats={"spe": -1})`（chance引数省略＝デフォルト1.0=100%）
   で、100%発動の仕様と一致している。変更不要。

## ハンドラ構成

### handlers/move_attack.py: `こごえるかぜ_lower_defender_spd`（既存・変更なし）

```python
def こごえるかぜ_lower_defender_spd(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spe": -1})
```

- `modify_defender_stats`（`handlers/move.py`）は内部で `battle.resolve_secondary_chance` を
  呼び出すため、`Event.ON_MODIFY_SECONDARY_CHANCE` 経由でちからずく（確率0化・威力1.3倍）の
  ハンドラが自動的に適用される（`secondary_effect` フラグが前提）。
- イベント: `Event.ON_DAMAGE_HIT`（ダメージが実際に適用された後に発動。
  .internal/spec/turn.md の `## Event.ON_DAMAGE` 参照）。
  ダメージが発生しなかった場合（無効化・まもる等）はイベント自体が発火しないため、
  追加効果も発生しない仕様を自然に満たす。
- 優先度: デフォルト100（`MoveHandler` のデフォルト値）。
- subject_spec: `MoveHandler` デフォルト（`attacker:self`）のまま。

### data/move.py（修正）

```python
"こごえるかぜ": MoveData(
    type="こおり",
    category="special",
    pp=16,
    power=55,
    accuracy=95,
    flags={"wind", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.こごえるかぜ_lower_defender_spd,
        )
    }
),
```

## リーサル計算

対象外（`.internal/progress/move.md` の「リーサル実装」列が `-`。すばやさダウンはHP分布に影響しないため
リーサル計算ハンドラは不要。今後の運用ルールでは能力ランク変化のみの追加効果は `n/a` 相当だが、
既存表記の `-` はそのまま維持し、対象外として扱う）。

## 注意点・エッジケース

- ノーマルタイプなどに技自体が無効化された場合、`Event.ON_DAMAGE_HIT` 自体が発火しないため
  追加効果も発生しない。個別のガードは不要。
- まもるで防がれた場合も同様にダメージが発生せず `Event.ON_DAMAGE_HIT` が発火しないため、
  追加効果も発生しない。
- すでに『すばやさ』ランクが-6の場合、`battle.modify_stats` 側でランク変化が発生しない
  （既存の共通処理に委ねるため、本技のハンドラ内で個別対応は不要）。
