# 実装計画書: みずびたし

## 仕様要約

- タイプ: みず / カテゴリ: 変化 / PP: 20 / 命中: 100 / 対象: 1 体選択
- 相手の「タイプ」を「みず」タイプに変える（タイプを追加するのではなく、完全に置き換える）
- 以下の場合は失敗する
  - 相手がすでにみず単タイプである
  - 相手がテラスタルしている
  - 相手がアルセウス・シルヴァディである（持ち物の有無・特性の状態に関わらず無効。種族名で判定する）
  - （`.internal/spec/turn.md` の `ON_APPLY_MOVE` priority 100 に「みずびたし等: すでに該当タイプ」として明記）

## 現状の実装状況

| 項目 | 状態 |
|------|------|
| `data/moves/move_ma.py` エントリ | 登録済み |
| `handlers/move_status.py` 関数 | 実装済み（テラスタル・アルセウス／シルヴァディ判定を追加修正） |
| `data/volatile.py` エントリ | 登録済み |
| `handlers/volatile.py` 関数 | 実装済み |
| `types/volatile.py` VolatileName | 追加済み |
| `model/pokemon.py` `volatile_override_type` | 実装済み（まほうのこなでも再利用されている） |

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_BEFORE_APPLY_MOVE` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手がすでにみず単タイプ／テラスタル中／アルセウス・シルヴァディなら失敗 |
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手に「みずびたし」volatile を付与 |
| `ON_VOLATILE_START` | 100（デフォルト） | `source:self` | `volatile_override_type` を `"みず"` に設定し `added_types` をクリア |
| `ON_VOLATILE_END` | 100（デフォルト） | `source:self` | `volatile_override_type` を `None` に戻す |

## Priority 根拠

- `ON_BEFORE_APPLY_MOVE` priority 100: `.internal/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「みずびたし等: すでに該当タイプ」が priority 100 として明記されている
  （テラスタル・アルセウス／シルヴァディの追加失敗条件も同一関数内で判定するため同じ priority でよい）
- `ON_STATUS_HIT` priority 100（デフォルト）: ハロウィン・もりののろいと同じパターン
- `ON_VOLATILE_START` / `ON_VOLATILE_END` priority 100（デフォルト）: ハロウィン volatile と同じパターン

## タイプ上書きの実装方針

みずびたしは「タイプを追加」ではなく「タイプをみずに変える」技であり、`added_types` の append だけでは実現できない。

**採用方針: `volatile_override_type` フィールドを使用する（実装済み）**

1. `src/jpoke/model/pokemon.py` に `self.volatile_override_type: Type | None = None` フィールドを持つ
2. `types` プロパティが `volatile_override_type` を参照する（`ability_override_type` より低い優先度）
3. `reset_on_switch_out()` で `volatile_override_type = None` をリセットする
4. volatile の `ON_VOLATILE_START` で `volatile_override_type = "みず"` に設定し、`added_types` をクリアする
5. volatile の `ON_VOLATILE_END` で `volatile_override_type = None` に戻す

`types` プロパティの優先順位:
```
テラスタル > ability_override_type > volatile_override_type > data.types
```

失敗判定は「すでに `types == ["みず"]`」に加えて「テラスタル中」「アルセウス・シルヴァディ」を
`ON_BEFORE_APPLY_MOVE` で明示的にチェックする。

## 実装箇所

### 1. `src/jpoke/handlers/move_status.py`

```python
def みずびたし_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """みずびたしの効果: 相手に「みずびたし」状態を付与してみずタイプに変える。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="みずびたし")


def みずびたし_can_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """みずびたしの使用条件チェック: 相手がすでにみずタイプのみ、テラスタル中、
    またはアルセウス・シルヴァディの場合は失敗する。"""
    defender = ctx.defender
    if (
        defender.types == ["みず"]
        or defender.terastallized
        or defender.name in ("アルセウス", "シルヴァディ")
    ):
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="みずびたし")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### 3. `src/jpoke/handlers/volatile.py`

```python
def みずびたし_set_type(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """みずびたし付与時: volatile_override_type をみずに設定し added_types をクリアする。"""
    ctx.source.volatile_override_type = "みず"
    ctx.source.added_types.clear()
    return HandlerReturn(value=value)


def みずびたし_clear_type(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """みずびたし解除時: volatile_override_type を None に戻す。"""
    ctx.source.volatile_override_type = None
    return HandlerReturn(value=value)
```

### 4. `src/jpoke/data/volatile.py`

"みずびたし": VolatileData(

        ),
        Event.ON_VOLATILE_END: h.VolatileHandler(
            h.みずびたし_clear_type,

## 注意点・エッジケース

- **`volatile_override_type` は `pokemon.py` の `__init__`・`reset_on_switch_out`・`types` プロパティの 3 箇所すべてで扱われている（まほうのこなと共用）**
- 失敗判定は「すでにみずタイプのみである」ことを確認するが、みずタイプに加えて他タイプ（ハロウィンや もりののろい などで追加されたタイプを含む）を持っている場合は成功させる必要がある。判定式 `ctx.defender.types == ["みず"]` はリストの完全一致であるため、複合タイプのケースで正しく成功する
- `ON_VOLATILE_START` で `added_types.clear()` を行うのは、ハロウィン・もりののろい状態が付与されている相手にみずびたしを使った場合、追加タイプが残らないようにするため
- **テラスタル中の相手は `ON_BEFORE_APPLY_MOVE` の時点で明示的に失敗させる**。wiki に「テラスタルしている相手に対しては失敗する」と明記されているため、volatile 自体を付与しない（まほうのこなと同じ扱い）
- アルセウス・シルヴァディの無効化は特性（マルチタイプ／ARシステム）ではなく **種族名** で判定する。
  実際のゲーム仕様では持ち物なしでも無効になるため、特性の `ability_override_type`（持ち物依存）に頼ると
  無道具時に判定が漏れる。`mon.name` は道具の有無に関係なくフォーム名を返さないため
  （アルセウス・シルヴァディはプレート／メモリで別 pokedex エントリに分かれておらず `"アルセウス"` `"シルヴァディ"` のまま）、
  種族名判定で正しく扱える。ただしへんしん・かわりものでこれらに変身したポケモンには有効（`mon.name` は
  変身後の姿を返さないため、変身によるすり替わりは自然に区別できる）
- 交代すると `volatile_override_type = None` にリセットされるため、みずタイプの効果は消える（`reset_on_switch_out` でリセット）
- みずびたし状態の相手にハロウィン・もりののろいを使うと、`added_types` に追加されるが `volatile_override_type = "みず"` が優先されるため「みず＋ゴースト」タイプになる。これは `types` プロパティが `[volatile_override_type] + extra` を返すことで自然に処理される
- `まほうのこな`（相手をエスパータイプにする）は同じ設計で既に実装済み
