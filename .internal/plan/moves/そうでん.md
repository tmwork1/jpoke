# 計画書: そうでん

## 仕様要約

- でんきタイプ / 変化 / 命中なし（必中） / PP20 / 対象: 相手1体
- 命中した相手を「そうでん状態」にし、そのターン中に相手が使う技のタイプをでんきタイプに変換する
- ターン終了時にそうでん状態は解除される（1ターン限定）
- 相手がすでに行動していた場合は失敗する（`ON_TRY_MOVE_1` priority 30 で明示的にチェックする。
  `ふいうち_try_move` と同様、フレームワーク側が自動判定するわけではないため専用ハンドラが必要）
- みがわり状態の相手には防がれる
- この技自体がでんきタイプのため、ひらいしん・ちくでん・でんきエンジンを持つポケモンには無効
- `わるあがき`はそうでん状態でもでんきタイプにならない
- 交代で出てきたポケモンに対しては成功する
- マジックコート・マジックミラーで跳ね返されない（変化技で対象が相手であるにもかかわらず例外的に
  跳ね返されない。`flags={"unreflectable"}` が必要）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_TRY_MOVE_1` | 30 | `attacker:self`（MoveHandler デフォルト、`skip_subject_check=True`） | 相手がすでに行動済みなら失敗させる（技ハンドラ） |
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（MoveHandler デフォルト） | 相手にそうでん状態を付与する（技ハンドラ） |
| `ON_MODIFY_MOVE_TYPE` | 110 | `attacker:self` | そうでん状態の攻撃側が出す技のタイプをでんきに変換する（volatile ハンドラ） |
| `ON_TURN_END` | 110 | `source:self` | ターン終了時にそうでん状態を解除する（volatile ハンドラ） |

## Priority 根拠

- **`ON_TRY_MOVE_1` priority 30**: `.internal/spec/turn.md` の `ON_TRY_MOVE_1` 節に「ふいうち: 対象行動済み・変化技選択」が priority 30 に記載されている。そうでんの「相手がすでに行動済みなら失敗」も同種の対象行動状態判定のため、同じ priority 30 に合わせる。実装は `ふいうち_try_move` / `_ふいうち系_can_apply` の `def_state.command_reserved()` チェックと同じパターンを用いる。
- **`ON_STATUS_HIT` priority 100**: `MoveHandler` のデフォルト値。既存の `タールショット_apply_volatile_to_defender` や `ちょうはつ_apply_volatile_to_defender` と同パターン。
- **`ON_MODIFY_MOVE_TYPE` priority 110**: `.internal/spec/turn.md` には未掲載のイベント。`めざめるダンス_modify_type`（`data/moves/move_ma.py`, `MoveHandler` デフォルト priority 100）など使用者自身のタイプを参照する技タイプ変換より後に評価し、そうでんによる「でんき」への変換を確実に優先させるため、それらより大きい 110 とする。
- **`ON_TURN_END` priority 110**: `.internal/spec/turn.md` の `ON_TURN_END` 節に「ちょうはつの終了」「いちゃもんの終了」「アンコールの終了」等が priority 110 に記載されており、1ターン限りの揮発状態の解除はすべて priority 110 に統一されている。

## 実装箇所（レビュー時点の実装内容）

### 1. `types/volatile.py` / `types/move.py`

`VolatileName` / `MoveName` の Literal に `"そうでん"` を追加済み。

### 2. `handlers/move_status.py`

```python
def そうでん_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """そうでんの効果: 相手をそうでん状態にする（そのターン中のみ）。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="そうでん")


def そうでん_try_move(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """そうでんの発動条件を判定する。

    相手（対象）がそのターンすでに行動済み（コマンド消費済み）の場合は失敗する。
    """
    def_player = battle.get_player(ctx.defender)
    def_state = battle.player_states[def_player]
    if not def_state.command_reserved():
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="そうでん")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### 3. `data/moves/move_sa.py`

```python
    "そうでん": MoveData(
        type="でんき",
        category="status",
        pp=20,
        flags={"unreflectable"},
        handlers={
            Event.ON_TRY_MOVE_1: h.MoveHandler(
                hs.そうでん_try_move,
                priority=30,
            ),
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.そうでん_apply,
            ),
        }
    ),
```

`category="status"` は `MoveCategory` Literal（`"status"` / `"physical"` / `"special"`）を用いる実装上の表記であり、
`.internal/progress/move.md` 上の日本語表記「変化」と対応する。`target` は明示していないが `MoveData` のデフォルト値
`"foe"` が仕様の「相手1体選択」と一致するため追加設定不要。

### 4. `handlers/volatile.py`

```python
def そうでん_move_type(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """そうでん状態による技タイプ変換: 攻撃側の使う技をでんきタイプに変換する。

    わるあがきはでんきタイプに変換されない。
    """
    if ctx.move and ctx.move.name != "わるあがき":
        return HandlerReturn(value="でんき")
    return HandlerReturn(value=value)


def そうでん_turn_end(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """そうでん状態のターン終了処理: ターン終了時に無条件で解除する。"""
    battle.volatile_manager.remove(ctx.source, "そうでん")
    return HandlerReturn(value=value)
```

## 注意点・エッジケース

- **「相手がすでに行動済みの場合は失敗」**: `.internal/spec/turn.md` の `ON_TRY_MOVE_1` priority 30 は「ふいうち: 対象行動済み・変化技選択」を指し、これは対象（相手）の行動状態を確認する共通パターンの一例である。フレームワークが自動判定するわけではないため、`そうでん_try_move` で明示的に `battle.player_states[def_player].command_reserved()` を確認し、`False`（＝すでにコマンド消費済み＝行動済み）なら失敗させる。
- **マジックコートで跳ね返されない**: `MoveData.is_reflectable` は `category == "status" and target in ("foe", "foe_side") and not has_flag("unreflectable")` で判定される。そうでんはデフォルトのままだと `is_reflectable=True` になってしまうため、`flags={"unreflectable"}` を明示的に設定する必要がある。
- **みがわりによる防御**: `bypass_substitute` フラグを付けていないため、既存のみがわり判定処理により自動的に防がれる。ハンドラ内にガード不要。
- **ひらいしん等による無効**: 技タイプが `"でんき"` であるため、ひらいしん・ちくでん・でんきエンジン等の「でんきタイプ技を無効化する」特性判定は変化技・攻撃技を問わず技タイプに基づいて汎用的に処理される。ハンドラ内にガード不要。
- **`わるあがき` 除外**: Wikiの仕様に明記されており、`そうでん_move_type` 内で `ctx.move.name != "わるあがき"` の条件でスキップする。
- **ターン終了解除タイミング**: `ON_TURN_END` priority 110 は `ちょうはつの終了` 等と同順。同一ターンに付与・解除が行われることを想定した1ターン効果。
- **`count` 指定なし**: `apply_volatile_to_defender` に `count` を渡さない（`None` のまま）。そうでん状態はカウントダウン式ではなく `そうでん_turn_end` によるターン終了時の無条件解除のみで管理する。
- **`ON_MODIFY_MOVE_TYPE` のコンテキスト**: `resolve_move_type` が発火する `AttackContext` の `attacker` がそうでん状態を持つポケモンなので `subject_spec="attacker:self"` が正しい。
