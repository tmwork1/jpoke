# 実装計画書: タールショット

## 仕様要約

- タイプ: いわ / カテゴリ: 変化 / PP: 20 / 命中: 100 / 対象: 1 体選択
- 命中した相手をタールショット状態にし、相手の「すばやさ」ランクを1段階下げる
- タールショット状態自体の効果（ほのおタイプへの弱点追加）は揮発性状態側で実装する

詳細は `.internal/spec/moves/タールショット.md`・`.internal/spec/volatiles/タールショット.md` を参照。

## 現状の実装状況

| 項目 | 状態 |
|------|------|
| `data/moves/move_ta.py` エントリ | 登録済み（type=いわ, category=status, pp=20, accuracy=100） |
| `handlers/move_status.py` 関数 | 実装済み（`タールショット_apply`） |
| `data/volatile.py` タールショット状態エントリ | 登録済み（`ON_CALC_DEF_TYPE_MODIFIER` ハンドラ） |
| `handlers/volatile.py` 関数 | 実装済み（`タールショット_boost_fire`） |

## ハンドラ構成（技本体）

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（`MoveHandler` デフォルト） | `attacker:self`（`MoveHandler` デフォルト） | タールショット状態を付与し、すばやさを1段階下げる単一の複合ハンドラ |

`ON_STATUS_HIT` は `move_executor.py` の `_execute_status_hit` から `AttackContext` を伴って発火されるため、
ハンドラは `ctx: AttackContext` を受け取り `ctx.defender` を直接参照する（`EventContext` ではない）。
同種のパターンは `どくのいと_apply`（`handlers/move_status.py`）を参照。

揮発性状態側のハンドラ構成（`ON_CALC_DEF_TYPE_MODIFIER` / priority=100 / `subject_spec="defender:self"`）は
`.internal/plan/volatiles/タールショット.md` を参照。

## Priority 根拠

`.internal/spec/turn.md` には `ON_STATUS_HIT` の記載なし。既存の状態変化技ハンドラはすべて
`MoveHandler` のデフォルト priority=100 を使用しており、タールショットも同じくデフォルトを踏襲する。

## 実装箇所（技本体）

### `src/jpoke/handlers/move_status.py`

```python
def タールショット_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """タールショットの効果: 相手にタールショット状態を付与し、すばやさを1段階下げる。"""
    assert ctx.defender is not None
    battle.volatile_manager.apply(ctx.defender, "タールショット", source=ctx.attacker)
    return HandlerReturn(value=battle.modify_stats(ctx.defender, {"spe": -1}, source=ctx.attacker))
```

### `src/jpoke/data/moves/move_ta.py`

```python
"タールショット": MoveData(
    type="いわ",
    category="status",
    pp=20,
    accuracy=100,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.タールショット_apply,
        ),
    }
),
```

## 注意点・エッジケース

- すばやさ低下は状態付与の成否に関わらず毎回発動する（どくのいとと異なり、状態付与とすばやさ低下は
  同時に処理される単一ハンドラであり、どちらか一方の成否のみで技全体の成否を判定しない）
- タールショット状態はターン経過では解除されず、交代でのみ解除される（`switch_manager` の
  `remove_all_volatiles` で自動処理）
- みがわり状態の相手には技全体が無効（通常の状態変化技と同様の挙動。特性『すりぬけ』所持者は無視できる）
- PP は `src/jpoke/data/ps-champ-ja/moves.json` に本技のエントリが存在しないため確認できず、一般に知られる
  本技のPP値（20）を採用した（詳細は `.internal/spec/moves/タールショット.md` 参照）
