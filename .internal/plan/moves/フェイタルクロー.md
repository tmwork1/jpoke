# フェイタルクロー 実装計画書

## 仕様要約

`.internal/spec/moves/フェイタルクロー.md` 参照。

* タイプ: どく、分類: 物理、威力: 80、命中: 100、PP: 16、接触あり
* 追加効果: 確率30%（Champions）で、どく/まひ/ねむりのいずれか（各1/3）を付与する
* Champions で「切る技」に追加（きれあじの対象。`flags` に `"slash"` が必要）
* 急所ランク補正なし（LEGENDS アルセウス限定の仕様であり、Champions には存在しない）

## ハンドラ構成

| イベント | Priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `Event.ON_DAMAGE_HIT` | 100（`MoveHandler` デフォルト） | `attacker:self`（`MoveHandler` デフォルト） | 単一乱数でどく/まひ/ねむりのいずれかを判定・付与 |

### Priority 根拠

`.internal/spec/turn.md` に `Event.ON_DAMAGE_HIT` 専用のエントリはないが、同種の確率付与追加効果
（どくづき・バーンアクセル等）がすべて `Event.ON_DAMAGE_HIT`・`MoveHandler` デフォルト
（priority=100、subject_spec="attacker:self"）を使用しているため、同じパターンを踏襲する。

## 実装コード

```python
def フェイタルクロー_apply_ailment_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """フェイタルクローの追加効果: 単一乱数で どく/まひ/ねむり のいずれかを付与する。

    r < base/3 → どく、r < base*2/3 → まひ、r < base → ねむり（チャンピオンズ: 確率0.3）。
    """
    base = battle.resolve_secondary_chance(ctx, 0.3)
    r = battle.random.random()
    if r < base / 3:
        ailment = "どく"
    elif r < base * 2 / 3:
        ailment = "まひ"
    elif r < base:
        ailment = "ねむり"
    else:
        return HandlerReturn(value=value)
    return HandlerReturn(value=battle.ailment_manager.apply(
        ctx.defender, ailment, source=ctx.attacker, ctx=ctx
    ))
```

`どく/まひ/こおり` の3択・共通ヘルパー `apply_ailment_to_defender` は使わず専用関数とする理由:
単一の乱数で「発動有無」と「どの状態か」を同時に判定する必要があり、
`apply_ailment_to_defender`（発動判定と付与状態異常が1:1の共通ヘルパー）では表現できないため。

## 注意点・エッジケース

1. **どく・はがねタイプへの無効化／まひ・ねむり無効特性**
   選ばれた状態異常が対象の耐性に引っかかる場合は `ailment_manager.apply` 内部で無効化される。
   発動判定自体は単一乱数のためやり直しはしない（無効化されても「不発扱い」にはならず、
   単に状態異常が付与されないだけ）。ハンドラ側に重複ガードは不要。

2. **りんぷん・おんみつマント**
   `battle.resolve_secondary_chance` 内の `ON_MODIFY_SECONDARY_CHANCE` ハンドラ
   （`りんぷん_block_secondary_chance` 等）で確率0扱いにされるため、ハンドラ側の対応は不要。

3. **てんのめぐみ・にじ状態**
   `てんのめぐみ_boost_secondary_chance` により発動確率が2倍になる。
   `resolve_secondary_chance` を通す時点で自動的に反映される。

4. **急所ランク補正を実装しないこと**
   一見「フェイタルクローは急所に当たりやすい」という通説があるが、これは
   Pokémon LEGENDS アルセウス限定の仕様であり、SV/Champions には存在しない
   （`.internal/spec/moves/フェイタルクロー.md` の「急所ランク補正について」参照）。
   `critical_rank` を設定しないこと。

5. **きれあじとの相互作用**
   `flags` に `"slash"` を含めることで `きれあじ_modify_power`（`ON_CALC_POWER_MODIFIER`、
   `subject_spec="attacker:self"`）が自動的に威力1.5倍を適用する。専用の相互作用ハンドラは不要。

## リーサル計算

対象外（`.internal/progress/move.md` の「リーサル実装」列が `n/a`）。追加効果が状態異常付与のみで
即死判定に影響する固定ダメージ等を持たないため、リーサル計算の対象としない。
