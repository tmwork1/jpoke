# 計画書: くらいつく

## 仕様要約（.internal/spec/moves/くらいつく.md 参照）

- あく / 物理 / 威力80 / 命中100 / PP10 / 対象: 相手1体 / 優先度0 / 接触あり / `bite`
- 命中時、相手と自分の両方を『にげられない』状態にする。
- この効果は**追加効果とはみなされない**（`secondary_effect` フラグを付与しない）。
  - ちからずくで威力が上がらず、りんぷん・おんみつマントでも無効化されない。
- 本技固有の判定として、以下の場合は**双方とも**『にげられない』状態にならない。
  - 片方でもゴーストタイプである場合
  - 片方でもすでに『にげられない』状態である場合
- この技で相手をひんしにした場合、使用者（攻撃側）は『にげられない』状態にならない。

## 既存実装との差分

既存実装（`data/move.py` の `"くらいつく"` エントリ）は `handlers={}`（コメント「追加効果なし」）
となっており、『にげられない』状態の付与が未実装だった。

1. **にげられない状態の付与（相手・自分の双方）が未実装**
   → 追加効果ハンドラを新規実装し、`Event.ON_DAMAGE_HIT` に登録する。
2. **『にげられない』揮発性状態に、付与元が場を離れた際の解除ハンドラが未実装**
   → `.internal/spec/volatiles/にげられない.md` の消滅条件（付与した側が場からいなくなると解除）を
     満たすため、`data/volatile.py` の `"にげられない"` エントリに `Event.ON_SWITCH_OUT` ハンドラを
     追加する。これは `"バインド"` の `バインド_remove`（`subject_spec="source:foe"`）と同一パターンで、
     くらいつく以外の『にげられない』付与技（くろいまなざし・とおせんぼう・かげぬい等）にも
     共通して必要な一般的な修正である。

## ハンドラ構成

### handlers/move_attack.py: `くらいつく_apply_no_escape`（新規）

```python
def くらいつく_apply_no_escape(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """くらいつくの効果: 命中時、相手と自分の両方をにげられない状態にする。

    追加効果とはみなされないため、apply_volatile_to_defender/attacker は使わず
    battle.volatile_manager.apply を直接呼び出す（ちからずく・りんぷんの影響を受けない）。
    片方でもゴーストタイプ、あるいは片方でもすでににげられない状態の場合は
    双方とも付与しない。相手をひんしにした場合、使用者はにげられない状態にならない。
    """
    attacker = ctx.attacker
    defender = ctx.defender
    if attacker.has_type("ゴースト") or defender.has_type("ゴースト"):
        return HandlerReturn(value=value)
    if attacker.has_volatile("にげられない") or defender.has_volatile("にげられない"):
        return HandlerReturn(value=value)
    battle.volatile_manager.apply(defender, "にげられない", source=attacker)
    if not defender.fainted:
        battle.volatile_manager.apply(attacker, "にげられない", source=defender)
    return HandlerReturn(value=value)
```

- イベント: `Event.ON_DAMAGE_HIT`（turn.md未掲載。ダメージが実際に適用された後に発動させる
  既存の同種ハンドラ「かげぬい」に倣う）。ダメージが発生しなかった場合（無効化・まもる・
  みがわりに防がれた場合等）はイベント自体が発火しないため、追加の個別ガードは不要。
- 優先度: デフォルト100（`MoveHandler` のデフォルト値。turn.md未掲載の技固有効果のため
  既存の同種ハンドラ（かげぬい等）に倣う）。
- subject_spec: `MoveHandler` デフォルト（`attacker:self`）のまま。
- ゴーストタイプ判定・すでに『にげられない』状態かどうかの判定は、`にげられない` 揮発性状態
  自体の `ON_CHECK_TRAPPED`（動的なゴースト無視判定）とは別に、本ハンドラ内で技の効果発動
  時点の判定として直接行う（本技固有の「双方とも不発」仕様のため）。

### handlers/volatile.py: `にげられない_remove_on_foe_switch`（新規・一般的な修正）

```python
def にげられない_remove_on_foe_switch(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """にげられない状態の解除処理（にげられない状態を与えた相手が場を離れたときに解除する）"""
    foe = battle.foe(ctx.source)
    battle.volatile_manager.remove(foe, "にげられない")
    return HandlerReturn(value=value)
```

### data/volatile.py

```python
"にげられない": VolatileData(
    handlers={
        Event.ON_CHECK_TRAPPED: h.VolatileHandler(
            h.check_trapped_not_ghost,
            subject_spec="source:self",
        ),
        Event.ON_SWITCH_OUT: h.VolatileHandler(
            h.にげられない_remove_on_foe_switch,
            subject_spec="source:foe",
            priority=100,
        ),
    }
),
```

- 優先度: 100（既存の同種ハンドラ「バインド」の `ON_SWITCH_OUT`（解除）に合わせる。
  `.internal/spec/volatiles/バインド.md` の Priority 表参照）。
- この修正により、シングルバトルの構造上「foe」は常に対戦相手の場のポケモン1体のみを
  指すため、くらいつくの相互付与（自分・相手の双方が『にげられない』状態を持つ）でも、
  片方が場を離れれば`もう片方の『にげられない』状態も自動的に解除される（`_switch_out` は
  ひんし後の交代でも呼ばれるため、ひんしによる消滅も自然に満たされる）。

## リーサル計算

対象外（`.internal/progress/move.md` の「リーサル実装」列が `n/a`）。
にげられない状態の付与はHP分布に影響しないため、リーサル計算ハンドラは不要。

## 注意点・エッジケース

- ノーマルタイプなどに技自体が無効化された場合、`Event.ON_DAMAGE_HIT` 自体が発火しないため
  追加効果も発生しない。個別のガードは不要。
- まもるで防がれた場合、相手のみがわりに防がれた場合も同様にダメージが発生せず
  `Event.ON_DAMAGE_HIT` が発火しないため、効果は発生しない。
- 自分（使用者）がみがわり状態であっても、本効果には影響しない（ダメージは実HPに適用され、
  `Event.ON_DAMAGE_HIT` は通常通り発火する）。
- `にげられない_remove_on_foe_switch` の追加は、くらいつく以外の『にげられない』付与技
  （くろいまなざし・とおせんぼう・かげぬい・クモのす・アンカーショット・サウザンウェーブ・
  たこがため）にも同様に適用される一般的な修正。既存テストへの影響がないことを
  `python -m pytest tests/ -v` で確認する。
