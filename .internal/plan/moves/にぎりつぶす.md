# 計画書: にぎりつぶす

## 仕様要約

- ノーマルタイプ / 物理 / 威力可変（1〜120） / 命中100 / PP5 / 対象: 相手1体選択 / 優先度0 / 接触
- 対象（相手）の残りHPが多いほど威力が高くなる。
- 威力 = max(1, round_half_down(120 × 対象の残りHP ÷ 対象の最大HP))
  - 端数は五捨五超入（`round_half_down`）で丸める（第五世代以降の仕様、
    `.internal/spec/moves/にぎりつぶす.md` 参照）。
- 追加効果なし。まもるで防がれる。おうじゃのしるし発動、オウムがえし反射可能。

## ハンドラ構成

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| にぎりつぶす | `ON_CALC_POWER_MODIFIER` | 100（デフォルト） | `attacker:self` | 対象の残りHP比率から威力を決定 |

## Priority 根拠

`.internal/spec/turn.md` に `ON_CALC_POWER_MODIFIER` イベントの記載はない（ダメージ計算内部イベントのため）。
既存の同種ハンドラ（`ハードプレス_calc_power`、`エレキボール_calc_power`、
`ジャイロボール_calc_power` 等）が全て priority=100（デフォルト）を使用しているため、
同様に **priority=100** を採用する。`MoveHandler` のデフォルト priority は 100、
subject_spec は `"attacker:self"` であるため、引数省略で問題ない。

## 実装箇所

### handlers/move_attack.py

```python
def にぎりつぶす_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """にぎりつぶす: 対象の残りHP / 最大HP の比率で威力を決定する（最大120）。

    威力 = max(1, round_half_down(120 × 現在HP / 最大HP))
    端数は五捨五超入で処理する（第五世代以降の仕様）。
    """
    power = max(1, round_half_down(120 * ctx.defender.hp / ctx.defender.max_hp))
    return HandlerReturn(value=power * 4096)
```

`power=1` をプレースホルダとして `data/move.py` に設定し、`ON_CALC_POWER_MODIFIER` で
`value` を `target_power * 4096` に差し替えることで、`damage_calculator._calc_final_power()`
の `final_power = round_half_down(1 * power_modifier / 4096) = target_power` として
最終威力を確定させる（`.internal/plan/moves/攻撃技_威力変動系.md` の実装方式と同一パターン）。

### data/move.py

既存の登録内容は仕様と一致しているため変更不要（`power` プレースホルダ・ハンドラ登録済み）。

```python
"にぎりつぶす": MoveData(
```

## 注意点・エッジケース

- 相手HPが1のときも `max(1, ...)` により威力は最低1になる（round_half_down のみでは
  端数次第で0になり得るため必須）。
- 相手HPが満タンのとき威力は必ず120（端数なし）。
- リーサル計算（`core/lethal.py`）は代表HP値（分布中の最小値）を使って通常の
  `battle.calc_damages` を呼ぶ設計のため、この `ON_CALC_POWER_MODIFIER` ハンドラは
  リーサル計算時にもそのまま発火する。ただし複数のHP分布枝が存在する場合、各枝ごとの
  厳密な威力差は反映されない（`.internal/progress/move.md` のにぎりつぶす行は
  `リーサル実装 n/a` / `リーサルテスト n/a` のまま、本タスクでは対象外）。

## テスト方針

`tests/moves_attack/test_move_na.py` に以下を配置（五十音順）:

1. `test_にぎりつぶす_相手HP1のとき威力1`: 相手HP=1で威力が最低1になることを確認。
2. `test_にぎりつぶす_相手HP半分のとき威力60`: 端数丸め（五捨五超入）で
   `round_half_down(120 × 117 / 235) = 60` になることを確認（floor では59になり誤り）。
3. `test_にぎりつぶす_相手HP満タンのとき威力120`: 満タン時に威力が120になることを確認。
