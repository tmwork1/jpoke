# 実装計画書: ひけん・ちえなみ

## 仕様要約（.internal/spec/moves/ひけん・ちえなみ.md 参照）

- タイプ: あく / カテゴリ: 物理 / 威力: 65 / 命中: 90 / PP: 15 / 対象: 相手1体
- flags: `contact`（直接攻撃）、`slash`（きれあじで威力1.5倍）、`secondary_effect`（ちからずく等の対象）
- 命中後、相手陣営に「まきびし」を1層設置する（最大3層、超過時は追加効果のみ不発）
- 使用者が特性「ちからずく」を持つ場合、威力1.3倍と引き換えに追加効果（まきびし設置）が発動しない
- りんぷん・おんみつマントは対象個体ではなく場を変化させる効果のため影響しない

## 修正内容

### `src/jpoke/data/moves/move_ha.py`

```python
"ひけん・ちえなみ": MoveData(
    type="あく",
    category="physical",
    pp=15,
    power=65,
    accuracy=90,
    flags={"contact", "slash", "secondary_effect"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.ひけん・ちえなみ_set_spikes,
        ),
    },

```

### `src/jpoke/handlers/move_attack.py`

`ひけん・ちえなみ_set_spikes` にちからずくガードを追加する。
`_is_sheer_force_blocked` は同ファイル内の `こうそくスピン_boost_attacker_spe` 等で使われている
既存の共通ヘルパー（`ctx.attacker.ability.name == "ちからずく" and ctx.move.has_flag("secondary_effect")`）を再利用する。

```python
def ひけん・ちえなみ_set_spikes(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ひけん・ちえなみ: 命中後、相手陣営に「まきびし」を1層設置する（最大3層）。

    追加効果のため、使用者が特性「ちからずく」の場合は発動しない
    （威力1.3倍化と引き換え）。りんぷん・おんみつマントは対象個体ではなく
    場を変化させる効果のため影響を受けない（resolve_secondary_chance は経由しない）。
    """
    if _is_sheer_force_blocked(ctx):
        return HandlerReturn(value=value)
    side = battle.get_side(ctx.defender)
    side.activate("まきびし", 1)
    return HandlerReturn(value=value)
```

きれあじによる威力1.5倍、ちからずくによる威力1.3倍は、それぞれ既存の
`きれあじ_modify_power`（`ctx.move.has_flag("slash")`）、
`ちからずく_boost`（`ctx.move.has_flag("secondary_effect")`）が
`flags` の追加のみで自動的に適用されるため、個別の実装は不要。

## 注意点・エッジケース

- `side.activate("まきびし", 1)` はすでに3層設置済みの場合 False を返すが、戻り値は無視してよい
  （攻撃自体は成功しているため `stop_event` は不要。がんせきアックスと同じパターン）
- まもる状態で無効化された場合・命中しなかった場合は `Event.ON_HIT` 自体が発火しないため、
  ハンドラ側でのガードは不要
- りんぷん・おんみつマントは `resolve_secondary_chance` を経由する効果にのみ影響するため、
  本ハンドラでは考慮不要（意図的に経由しない）
