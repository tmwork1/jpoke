# 計画書: ソウルビート

## 仕様要約

- ドラゴンタイプ / 変化 / 命中なし / PP8（`src/jpoke/data/ps-champ-ja/moves.json` 準拠） / 対象: 自分
- 自分のHPを最大HPの1/3（切り捨て）だけ減らす
- 自分のすべての能力ランク（こうげき・ぼうぎょ・とくこう・とくぼう・すばやさ）を1段階ずつ上げる
- ダンス技かつ音系の技（flags: `dance`, `sound`）
- 失敗条件（HPが最大HPの1/3以下、または対象5ランクすべてが+6）はHP消費前に判定し、
  失敗時はHPを消費しない（`.internal/spec/moves/はらだいこ.md` と同一パターン）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_BEFORE_APPLY_MOVE` | 100（デフォルト） | attacker:self（省略） | 失敗条件チェック（HP不足／全能力+6） |
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | HP1/3消費＋全能力+1段階 |

## Priority 根拠

`.internal/spec/turn.md` の `ON_APPLY_MOVE`（コード上のイベント名は `ON_BEFORE_APPLY_MOVE`。
「技が無効化されないかの最終判定」）に `100 | みがわり: すでに状態変化` `100 | みがわり: HP不足`
`100 | ソウルビート: HP不足・全能力+6` と明記されている。

HPコスト＋失敗条件を持つ変化技（みがわり・はらだいこ）はいずれも `ON_TRY_MOVE_1` ではなく
`ON_BEFORE_APPLY_MOVE`（デフォルトpriority=100）で自己完結の失敗チェックを行うのが確立された
パターンであり、本技もこれに従う。

（旧版の本計画書では `ON_TRY_MOVE_1` priority=30 としていたが、はらだいこ・みがわりの実装と
`.internal/spec/turn.md` の実際のドキュメント構成に合わせて `ON_BEFORE_APPLY_MOVE` に修正した。
`.internal/spec/turn.md` 側の該当行も修正済み）

## 実装箇所

### handlers/move_status.py への実装

`ソウルビート_check`（失敗条件チェック）と `ソウルビート_pay_hp_and_modify_attacker_stats`
（効果本体）の2関数で構成する。`はらだいこ_can_apply` / `はらだいこ_apply` と同様、
**失敗条件チェックはHP消費より前に行い、全ランクがすでに+6の場合はHPを消費させない**。

```python
def ソウルビート_check(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ソウルビートの使用条件チェック: HPが最大HPの1/3以下、またはこうげき・ぼうぎょ・とくこう・とくぼう・すばやさが
    すべて+6ならば失敗する。"""
    mon = ctx.attacker
    if (
        mon.hp <= mon.max_hp // 3
        or all(mon.rank[stat] >= 6 for stat in ("atk", "def", "spa", "spd", "spe"))
    ):
        battle.add_event_log(
            mon, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="ソウルビート")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def ソウルビート_pay_hp_and_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ソウルビートの効果: 最大HPの1/3を消費し、すべての能力を1段階ずつ上げる。"""
    mon = ctx.attacker
    battle.modify_hp(mon, r=-1/3)
    return modify_attacker_stats(battle, ctx, value, stats={"atk": 1, "def": 1, "spa": 1, "spd": 1, "spe": 1})
```

挿入位置: `そうでん_check` の後、`たくわえる_apply` の前（五十音順）。

### data/moves/move_sa.py への登録

```python
    "ソウルビート": MoveData(
        type="ドラゴン",
        category="status",
        pp=8,
        target="self",
        flags={"dance", "sound"},
        handlers={
            Event.ON_BEFORE_APPLY_MOVE: h.MoveHandler(
                hs.ソウルビート_check,
                priority=100,
            ),
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.ソウルビート_pay_hp_and_modify_attacker_stats,
            ),
        }
    ),
```

`target="self"` を明示しないとデフォルト値 `"foe"` になり、`is_blocked_by_protect` /
`is_reflectable` の判定でまもる・マジックコートの対象になってしまう（はらだいこ計画書と同じ注意点）。

## リーサル計算

自分にのみ作用し相手にダメージを与えないため、リーサル計算（`lethal_handlers`）は対象外（n/a）。

## 注意点・エッジケース

- **HPが足りない失敗判定**（現在HP <= 最大HP // 3）は `ソウルビート_check` で判定し、HP消費前に失敗させる
- **全能力+6での失敗判定**も同じ `ソウルビート_check` で判定し、HP消費前に失敗させる
  （一部ランクのみ+6の場合は失敗せず、上げられる分だけ上げてHPを消費する）
- HP減算は `battle.modify_hp(mon, r=-1/3)` を使用する。`Pokemon.hp` への直接代入は禁止
- 1/3 切り捨ての丸め規則は `battle.modify_hp` の `int(max_hp * r)` 実装により
  `max_hp // 3` と一致する（浮動小数点誤差はmax_hp=1000以下の範囲で発生しないことを確認済み）
- ダンス技（labels: dance）かつ音系の技（labels: sound）のため、おどりこ特性のポケモンへのコピーと
  じごくづきによる音技封じはフラグベースでフレームワーク側が処理する
- data/moves/move_sa.py の既存エントリで pp=5・target未設定（デフォルト"foe"）と誤っていたため修正した
