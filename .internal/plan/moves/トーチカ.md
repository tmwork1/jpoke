# 計画書: トーチカ

## 仕様要約

- タイプ: どく / カテゴリ: 変化 / PP: 8（champions基準） / 命中: なし（必中） / 対象: 自分 / 優先度: +4
- 使用ターン中、相手の攻撃技・変化技を防ぐ（まもると同様に変化技も防ぐ）
- 接触技（直接攻撃）で攻撃してきた相手をどく状態にする
- まもる系共通の連続使用失敗ルールが適用される（`ON_TRY_MOVE_2` デフォルト priority「まもる系: 連続使用により失敗」）
- `"トーチカ"` volatile を自分に付与し、`ON_TRY_MOVE_1` で攻撃を無効化しつつ接触した相手をどく状態にする
- ターン終了時に volatile を解除する

## ハンドラ構成

| イベント | priority | subject_spec | 役割 | 実装場所 |
|----------|----------|--------------|------|----------|
| `ON_TRY_MOVE_2` | 100（デフォルト） | `attacker:self` | まもる系連続使用失敗チェック（`まもる系_連続使用失敗チェック`） | `handlers/move_status.py` |
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self` | 自分に `"トーチカ"` volatile を付与する（`トーチカ_apply`） | `handlers/move_status.py` |
| `ON_TRY_MOVE_1`（volatile側） | 100 | `defender:self` | 攻撃を無効化し、接触技で攻撃した相手をどく状態にする（`トーチカ_protect`） | `handlers/volatile.py` |
| `ON_TURN_END`（volatile側） | 100（デフォルト） | `source:self` | ターン終了時に `"トーチカ"` volatile を解除する（`トーチカ_remove_volatile`） | `handlers/volatile.py` |

## Priority 根拠

### `ON_TRY_MOVE_2`: priority 100（デフォルト）

`.internal/spec/turn.md` の `Event.ON_TRY_MOVE_1` に「30: まもる系: 連続使用により失敗」の記載があるが、
既存実装（まもる・みきり・こらえる・キングシールド・ニードルガード等）はすべて `Event.ON_TRY_MOVE_2` の
デフォルト priority（100）で連続使用失敗チェックを実装しているため、同一パターンに合わせる
（詳細は `.internal/spec/moves/まもる.md` 参照）。

### `ON_STATUS_HIT`: priority 100（デフォルト）

`.internal/spec/turn.md` に `ON_STATUS_HIT` の内部 priority 表がない。既存の変化技ハンドラ
（`まもる_apply`・`キングシールド_apply`・`ニードルガード_apply` 等）がすべて priority 100（デフォルト）を
使用しているため、同一パターンで 100 を採用する。

### `ON_TRY_MOVE_1`（volatile側）: priority 100

`.internal/spec/turn.md` の `Event.ON_TRY_MOVE_1` に「100: まもる・キングシールド等による無効化」と記載されている。
既存の `まもる` / `キングシールド` / `ニードルガード` volatile ハンドラがすべて priority 100 を明示しているため、
同一の 100 を採用する。

### `ON_TURN_END`（volatile側）: priority 100（デフォルト）

`.internal/spec/turn.md` に `ON_TURN_END` の「まもる系 volatile 解除」の記載はない。既存の
`まもる_remove_volatile`・`キングシールド_remove_volatile`・`ニードルガード_remove_volatile` 等がすべて
デフォルト priority を使用しているため、同一パターンで 100 を採用する。

## 実装箇所（実装済み）

### `src/jpoke/handlers/volatile.py`

`_run_protect` 共通関数（`stats_change_on_contact` / `ailment_on_contact` / `chip_on_contact` /
`protect_non_attack` を受け取る）を再利用し、`ailment_on_contact="どく"` を渡すことで実装する。

```python
def トーチカ_protect(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """トーチカの保護判定。接触した相手をどく状態にする"""
    return _run_protect(battle, ctx, value, ailment_on_contact="どく")


def トーチカ_remove_volatile(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    return remove_volatile(battle, ctx, value, volatile="トーチカ")
```

`_run_protect` 内で `ailment_on_contact` が指定されている場合、接触時に
`battle.ailment_manager.apply(ctx.attacker, ailment_on_contact, source=ctx.defender)` を呼ぶ。
どくタイプ・はがねタイプ免疫、状態異常の重複防止、`ぼうごパット` による接触判定無効化などは
`ailment_manager.apply` および `is_contact` 判定側の共通処理に委譲されるため、トーチカ固有の実装は不要。

### `src/jpoke/handlers/move_status.py`

def トーチカ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:

## 注意点・エッジケース

### まもる系の連続使用失敗

`_run_protect` 自体は連続使用失敗を判定しない。連続使用失敗は技側の `ON_TRY_MOVE_2`
（`まもる系_連続使用失敗チェック`）で処理され、`protect` フラグを持つ技同士で共有される
（詳細は `.internal/spec/volatiles/まもる.md` 参照）。

### 変化技への保護

`protect_non_attack` のデフォルトは `True` のため、トーチカは変化技も防ぐ
（まもる・みきり・ニードルガードと同様。キングシールド・かえんのまもり・スレッドトラップ・
ファストガードは変化技を防げない点で異なる）。

### まねっこでコピーされない

`まねっこ_can_use` は `copied_move.has_flag("protect")` を持つ技を一律コピー不可とするため、
トーチカに `non_copycat` フラグを個別付与する必要はない（`flags={"protect"}` のみで十分）。

### ふかしのこぶし・フェイント等による貫通

`ON_CHECK_PROTECT`（`ふかしのこぶし_bypass_protect`）と `_FAINT_REMOVE_VOLATILES`
（フェイント・ゴーストダイブ・シャドーダイブ・いじげんホール・いじげんラッシュ）は
まもる系共通のリストで処理されており、トーチカもすでに含まれている。トーチカ固有の実装は不要。

### 特性「ふしょく」の例外

どくタイプ・はがねタイプの使用者が「ふしょく」持ちの場合、防いだ攻撃者をどく状態にできる例外は
未実装（スコープ外）。`ailment_manager.apply` 側の一般ロジックに委ねる。
