# こなゆき 実装計画書

## 仕様要約（.internal/spec/moves/こなゆき.md 参照）

* タイプ: こおり / 分類: 特殊 / 威力: 40 / 命中: 100 / PP: 25
* 対象: 相手全員（シングルバトルでは相手1体と同義） / 優先度: 0 / 接触なし
* 効果: 追加効果として、10%の確率で相手を『こおり』状態にする。

## 現状の実装

既に実装済み。以下の内容でレビューし、仕様との齟齬がないことを確認した。

```python
"こなゆき": MoveData(
    type="こおり",
    category="special",
    pp=25,
    power=40,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.こなゆき_apply_freeze_to_defender,
        )
    }
),
```

```python
def こなゆき_apply_freeze_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="こおり", chance=0.1)
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手を『こおり』状態にする |

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を
経由するため、ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は汎用ハンドラ側で処理される。
本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（状態異常付与）は既存の同種ハンドラ（こおりのキバ・コールドフレア 等）
と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。個別の priority検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・こおり付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
2. **テスト方針**
