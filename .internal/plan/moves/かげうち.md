# 計画書: かげうち

## 仕様要約

- ゴーストタイプ / 物理 / 威力40 / 命中100 / PP30 / 対象: 1体選択 / 優先度+1 / 接触
- 追加効果なし（通常攻撃のみ、優先度+1のみが特徴）

## ハンドラ構成

追加効果を持たないため、専用ハンドラは不要。`handlers={}` のみで基本ダメージ計算に委ねる。
優先度は `priority=1` を `MoveData` に指定するだけでよく、`turn_controller.py` の行動順決定処理が
既存の共通ロジックとして `priority` を参照するため、専用実装は不要。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、priority の検討対象イベントは存在しない。
技自体の行動順における優先度（`MoveData.priority=1`）は `.internal/spec/turn.md` の「技の優先度」に
関する記述通り、通常の優先度制御の枠組みで処理される。

## 実装箇所

### data/move.py への更新

既存の `"かげうち"` エントリは以下の通りで、仕様と一致している（変更不要）。

```python
    "かげうち": MoveData(
        type="ゴースト",
        category="physical",
        pp=30,
        power=40,
        accuracy=100,
        priority=1,
        flags={"contact"},
        handlers={},  # 追加効果なし
    ),
```

- `flags={"contact"}` → 接触判定あり（仕様と一致）。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致。
- `priority=1` → 優先度+1（仕様と一致）。
- `critical_rank` 省略時のデフォルトは `0`（通常の急所判定）→ 仕様と一致。

## 注意点・エッジケース

- 追加効果がないため、まもる・おうじゃのしるし・オウムがえし等の一般的な技判定は
  フレームワーク側の共通処理（`handlers/move_attack.py` の基本ダメージ計算経路）で
  自動的に処理され、本技固有のハンドラ実装は不要。
- 優先度+1のため、同一ターン内で通常優先度の技より先に発動する。すばやさ比較は
  同優先度同士の場合のみ行われる（`.internal/spec/turn.md` 参照）。
