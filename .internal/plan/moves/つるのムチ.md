# 計画書: つるのムチ

## 仕様要約

- くさタイプ / 物理 / 威力45 / 命中100 / PP25 / 対象: 1体選択 / 優先度0 / 接触
- 追加効果なし（通常攻撃のみ）

## ハンドラ構成

追加効果を持たないため、専用ハンドラは不要。`handlers={}` のみで基本ダメージ計算に委ねる。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、priority の検討対象イベントは存在しない。

## 実装箇所

### data/move.py（data/moves/move_ta.py）

既存の `"つるのムチ"` エントリは仕様と一致している（変更不要）。

```python
    "つるのムチ": MoveData(
        type="くさ",
        category="physical",
        pp=25,
        power=45,
        accuracy=100,
        flags={"contact"},
        handlers={},  # 追加効果なし
    ),
```

- `flags={"contact"}` を指定 → 直接攻撃（仕様と一致）。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `critical_rank` 省略時のデフォルトは `0`（通常の急所判定）→ 仕様と一致。

## リーサル計算

追加効果がなく通常ダメージのみのため、`handlers/lethal.py` に専用ハンドラは不要（n/a）。

## 注意点・エッジケース

- 追加効果がないため、まもる・おうじゃのしるし・オウムがえし等の一般的な技判定は
  フレームワーク側の共通処理（`handlers/move_attack.py` の基本ダメージ計算経路）で
  自動的に処理され、本技固有のハンドラ実装は不要。
