# 実装計画書: ワイルドボルト

参照: .internal/spec/moves/ワイルドボルト.md

## 概要

物理技。相手に与えたダメージの1/4を自分も受ける反動技。追加効果なし。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---------|----------|--------------|------|
| ON_HIT | 100（`MoveHandler` デフォルト） | attacker:self | 与ダメの1/4を反動として攻撃者に与える |

Priority 根拠: `.internal/spec/turn.md` の `Event.ON_HIT` に反動ダメージ専用の記載行はないが、
同系統の反動技（とっしん・じごくぐるま・アフロブレイク・ボルテッカー）と同じく
`MoveHandler` のデフォルト priority=100 を使用する（既存実装との整合性を優先）。

## handlers/move_attack.py

共通ヘルパー `_recoil(battle, ctx, value, ratio)` を利用する（既存実装）。

```python
def ワイルドボルト_recoil(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return _recoil(battle, ctx, value, 1/4)
```

## リーサル計算

対象外（`.internal/progress/move.md` の「リーサル実装」列は n/a）。
反動技は攻撃者自身のHPを減らすのみで、防御側の HP 分布 (`StateDist`) には影響しないため
リーサル計算ハンドラの実装対象にはならない。

## 注意点・エッジケース

* `reason="recoil"` を指定することで、いしあたま特性・マジックガード特性により反動ダメージが無効化される
  （特性側の実装に依存する汎用処理のため、本技側での分岐は不要）。
* みがわりに命中した場合も `ON_HIT` は発動するため反動ダメージは発生する。
  `_recoil` は `value`（実際の被ダメージ、みがわり有効時は0）が0の場合
  `ctx.substitute_damage`（みがわりへの与ダメージ）にフォールバックして反動量を算出する
  （共通ヘルパー `_drain_hp` と同じパターン。既に `_recoil` に実装済み）。
* PP はチャンピオンズのPP統一仕様（8/12/16/20の4段階）により16。Gen9本家のPP15とは異なる
  （`src/jpoke/data/ps-champ-ja/moves.json` を正とする）。
