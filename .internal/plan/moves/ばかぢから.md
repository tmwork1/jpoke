# 実装計画書: ばかぢから

参照仕様書: .internal/spec/moves/ばかぢから.md

## 仕様要約

* タイプ: かくとう / 分類: 物理 / 威力: 120 / 命中率: 100 / PP: 5 / 対象: 1体選択 / 優先度: 0 / 直接攻撃
* 技が成功しダメージを与えた後、自分の『こうげき』『ぼうぎょ』ランクが1段階ずつ下がる
  （確率100%・確定効果、追加効果には分類されない）。

## 既存実装の確認結果

`src/jpoke/data/moves/move_ha.py` に既に以下の通り実装済みであることを確認した。

```python
"ばかぢから": MoveData(
    type="かくとう",
    category="physical",
    pp=5,
    power=120,
    accuracy=100,
    flags={"contact"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.ばかぢから_lower_attacker_def,
        )
    },
    lethal_handlers={
        LethalEvent.ON_HIT: LethalHandler(l.ばかぢから_lower_atk)
    }
),
```

* `flags` に `secondary_effect` を含めていない点は、仕様書の「追加効果には分類されない」と
  整合している（シールドダスト等の追加効果無効化と無関係に発動するため）。
* `handlers/move_attack.py` の `ばかぢから_lower_attacker_def` は
  `modify_attacker_stats(battle, ctx, value, stats={"atk": -1, "def": -1})` を呼び出しており、
  `chance` を指定していないためデフォルトの100%（確定効果）で発動する。`battle.modify_stats`
  経由のため、あまのじゃく等の特性補正も自動的に適用される。
* `Event.ON_HIT` は `.internal/spec/turn.md` で「自分のランクが下がる反動」が priority=10 と
  記載されているが、既存の同種実装（`インファイト_lower_defender_stats`、
  `アームハンマー_lower_attacker_spe`、`オーバーヒート_lower_attacker_spa` 等）は
  いずれも `priority` を指定せずデフォルト（100）のまま登録されている。本技も同じ関数群と
  同一パターンのため、既存実装のデフォルト優先度を踏襲し変更しない。

### リーサル計算の確認結果

`handlers/lethal.py` の `ばかぢから_lower_atk` は `ctx.attacker.rank["atk"]` のみを
1段階下げ、`ぼうぎょ` ランクの低下は反映していない。これは以下の既存の同種実装と整合した
意図的な仕様である。

* 自分の「攻撃側の能力（こうげき・とくこう）」を下げる技（`オーバーヒート_lower_attacker_spa`
  など）は、以降のダメージ計算に影響するためリーサル計算でも `rank` を更新する。
* 自分の「防御側の能力（ぼうぎょ・とくぼう・すばやさ）」のみを下げる技
  （`インファイト_lower_defender_stats`、`アームハンマー_lower_attacker_spe`、
  `アーマーキャノン_lower_attacker_stats` 等）は、攻撃側自身の以降の与ダメージに影響しない
  ため、対応する `lethal_handlers` 自体が登録されていない。

ばかぢからは「こうげき」（以降の与ダメージに影響）と「ぼうぎょ」（影響しない）の両方を
下げる技であり、`ばかぢから_lower_atk` が「こうげき」のみを反映する現状の実装は上記の
方針と一致しているため、修正不要と判断した。
