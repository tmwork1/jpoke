# 計画書: じごくづき

## 仕様要約

- あくタイプ / 物理 / 威力80 / 命中100 / PP15 / 対象: 相手1体 / 直接攻撃あり
- 追加効果（100%）として、命中した相手を『じごくづき』揮発性状態にする
- じごくづき状態の詳細（2ターンの間、音の技を使えなくする効果・解除条件など）は揮発性状態側の実装
  （`.internal/spec/volatiles/じごくづき.md`、`data/volatile.py`、`handlers/volatile.py`）に委ねる
- 詳細仕様は `.internal/spec/moves/じごくづき.md` を参照

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | デフォルト | attacker:self（省略時デフォルト） | 相手にじごくづき状態を付与 |

## Priority 根拠

`ON_DAMAGE_HIT` は `.internal/spec/turn.md` に未掲載のダメージ計算内部イベント（技命中・ダメージ適用後に発火し、
みがわりで防がれた場合は発火しない）。同種の追加効果ハンドラである `しおづけ_apply_volatile_to_defender`・
`みずあめボム_apply_volatile_to_defender`（いずれも `ON_DAMAGE_HIT`、priority省略）に倣い、デフォルト priority を使用する。

## 実装箇所（実装済み）

### handlers/move_attack.py

```python
def じごくづき_apply_volatile_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """じごくづきの追加効果: 2ターンの間、相手が音技を使えなくなる。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="じごくづき", count=2)
```

### data/move.py

```python
    "じごくづき": MoveData(
        type="あく",
        category="physical",
        pp=15,
        power=80,
        accuracy=100,
        flags={"contact", "secondary_effect"},
        handlers={
            Event.ON_DAMAGE_HIT: h.MoveHandler(
                ha.じごくづき_apply_volatile_to_defender,
            )
        }
    ),
```

## 注意点・エッジケース

- `secondary_effect` フラグにより、ちからずく特性（威力1.3倍・追加効果無効化）、りんぷん特性・おんみつマント
  （追加効果無効化）は `apply_volatile_to_defender` 内の `resolve_secondary_chance` を通じて汎用的に処理される。
  ハンドラ側に個別のチェックは不要。
- みがわり状態の相手に技が防がれた場合、`ON_MODIFY_MOVE_DAMAGE` でダメージが0になり `actual_damage <= 0` となるため
  `ON_DAMAGE_HIT` 自体が発火しない（`move_executor._execute_hit` の共通処理）。ハンドラ側にみがわりチェックは不要。
- じごくづき状態そのもの（コマンド制限・行動ブロック・ターンカウント）は `data/volatile.py` の "じごくづき" エントリに
  実装済み（`ON_MODIFY_COMMAND_OPTIONS` / `ON_TRY_ACTION`(priority=100) / `ON_TURN_END`(priority=110)）。
  技側のハンドラは volatile 付与のみを担当する。
- リーサル計算は対象外（`.internal/progress/move.md` でリーサル実装=n/a）。じごくづき状態はダメージを発生させず、
  今後の行動制限のみに影響するためリーサル計算（HP分布）には影響しない。

## 修正履歴（レビュー時に発見した誤り）

- **`flags` に `secondary_effect` が欠落していた**: ちからずく特性の威力1.3倍・追加効果無効化が機能していなかった。
  `flags={"contact"}` → `flags={"contact", "secondary_effect"}` に修正。
- **登録イベントが `ON_HIT` になっていた**: `ON_HIT` はみがわりに防がれた場合（`actual_damage <= 0`）でも発火するため、
  みがわり状態の相手にもじごくづき状態が付与されてしまう不具合があった。他の同種ハンドラ
  （`しおづけ`・`みずあめボム` 等）に合わせて `ON_DAMAGE_HIT` に修正。
