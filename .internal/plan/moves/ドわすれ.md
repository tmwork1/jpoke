# 実装計画: ドわすれ

## 仕様サマリ

- タイプ: エスパー / 分類: 変化 / 威力: - / 命中率: - / PP: 20 / 対象: 自分
- 効果: 自分のとくぼうランクを 2 段階上げる
- まもる: 防がれない（自分対象） / マジックコート: 反射不可（自分対象） / 直接攻撃: いいえ
- よこどり: 対象になる（○）

出典: .internal/spec/moves/ドわすれ.md

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自分のとくぼうを 2 段階上げる |

## Priority 根拠

`.internal/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`つるぎのまい_modify_attacker_stats`, `わるだくみ_modify_attacker_stats` 等）が
すべて priority=100（デフォルト）で登録されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py

既存実装済み（`ドわすれ_modify_attacker_stats`）。

```python
def ドわすれ_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"spd": 2})
```

### data/moves/move_ta.py の修正

`target="self"` を指定する（`target` を指定しないとデフォルトの `"foe"` になり、
`is_blocked_by_protect` / `is_reflectable` がまもる・マジックコートに誤反応してしまうため必須）。
既存実装ではこの指定が漏れていたため追加する。

```python
"ドわすれ": MoveData(
    type="エスパー",
    category="status",
    pp=20,
    target="self",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ドわすれ_modify_attacker_stats,
        )
    }
),
```

## 注意点・エッジケース

- `target="self"` を指定し忘れると、まもる・マジックコートの判定にデフォルト値 `"foe"` の挙動が
  誤って適用されるため、必ず指定する
- とくぼうがすでに最大（+6）の場合の失敗は `battle.modify_stats()` 側でマネージャー内部処理されるため、
  ハンドラ側にガード不要
- `つるぎのまい_modify_attacker_stats` / `りゅうのまい_modify_attacker_stats` と全く同じ実装パターン
  （stats の内容のみ異なる）
