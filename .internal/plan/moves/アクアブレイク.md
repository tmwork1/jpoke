# 実装計画書: アクアブレイク

参照仕様書: .internal/spec/moves/アクアブレイク.md

## 実装方針

アクアブレイクは「20%の確率で相手のぼうぎょランクを1段階下げる」という追加効果を持つ
攻撃技であり、既存の共通ハンドラ `modify_defender_stats`（handlers/move.py）を利用して実装する。
専用の効果処理は不要。

* `type="みず"`
* `category="physical"`
* `pp=12`（champions基準。src/jpoke/data/ps-champ-ja/moves.json 準拠）
* `power=85`
* `accuracy=100`
* `flags={"contact", "secondary_effect"}`
  * `"contact"`: 直接攻撃
  * `"secondary_effect"`: 追加効果を持つ技（ちからずく等の対象技であることを示す）
  * `"slash"` は付与しない（切る技に該当しないため。`.internal/spec/abilities/きれあじ.md` 参照）

## ハンドラ

`Event.ON_DAMAGE_HIT` で `handlers/move_attack.py` の `アクアブレイク_lower_defender_def` を呼び出す。

```python
def アクアブレイク_lower_defender_def(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"def": -1}, chance=0.2)
```

`modify_defender_stats` は内部で `battle.resolve_secondary_chance(ctx, chance)` を呼び出すため、
ちからずくによる追加効果無効化は自動的に処理される。

### priority 根拠

`.internal/spec/turn.md` の `Event.ON_DAMAGE`（実装上は `Event.ON_DAMAGE_HIT`）セクションでは
「追加効果（特殊なもの除く）」が priority=10 に分類される。
一方、みずがための発動（`ability.py` の `みずがため` エントリ）は
`Event.ON_DAMAGE_HIT priority=20` で登録済み。
`.internal/spec/abilities/みずがため.md` の仕様「アクアブレイク/シェルブレードを受けた場合、
追加効果の後にみずがためが発動する」を満たすため、アクアブレイクの追加効果は
みずがため（priority=20）より先に発動する必要があり、`priority=10` を明示する。

```python
Event.ON_DAMAGE_HIT: h.MoveHandler(
    ha.アクアブレイク_lower_defender_def,
    priority=10,
)
```

## テスト方針

`tests/moves_attack/test_move_a.py` に以下を追加する（シェルブレード・かみくだく等と同様のパターン）。

* `secondary_chance=0.0` で追加効果不発時にぼうぎょランクが変化しないことを確認するテスト
* `secondary_chance=1.0` で追加効果発動時にぼうぎょランクが-1になることを確認するテスト

`tests/abilities/test_ability_ma.py` には、みずがためとの発動順序を検証するテスト
（`test_みずがため_アクアブレイクとシェルブレードでは追加効果の後に発動する`）が
シェルブレードと共通のパラメータ化テストとして既に存在する。
