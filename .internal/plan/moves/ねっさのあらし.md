# ねっさのあらし 実装計画書

## 仕様要約（.internal/spec/moves/ねっさのあらし.md 参照）

* タイプ: じめん / 分類: 特殊 / 威力: 100 / 命中: 80 / PP: 10
* 対象: 相手全員（本プロジェクトはシングルバトルのみ対象のため、実質的に相手1体） / 優先度: 0 / 接触なし
* 風の技（`flags={"wind"}`）
* 効果: 追加効果として、20%の確率で相手を『やけど』状態にする。
* 天気が『あめ』のときは必ず命中する（`かみなりあらし`・`こがらしあらし`と同様、にほんばれ時の命中率低下はない）。
* そらをとぶ状態の相手には、あめによる必中補正であっても命中しない（`ぼうふう`・`かみなり`と異なる点）。

## 現状の実装状況

`src/jpoke/data/move.py` に登録済み、`src/jpoke/handlers/move_attack.py` にやけど付与ハンドラ
（`ねっさのあらし_apply_burn_to_defender`, chance=0.2）は実装済みだったが、
**あめ時に必ず命中する仕様が未実装**だった（`ON_MODIFY_ACCURACY` ハンドラが登録されていなかった）ため追加する。

そらをとぶ状態への回避判定は `handlers/volatile.py` の `HIDDEN_MOVE_ALLOWED_MOVES["そらをとぶ"]` に
`ねっさのあらし` が含まれていないため、既存実装のままで正しく「命中しない」挙動になる（変更不要）。

```python
"ねっさのあらし": MoveData(
    type="じめん",
    category="special",
    pp=10,
    power=100,
    accuracy=80,
    flags={"wind", "secondary_effect"},
    handlers={
        Event.ON_MODIFY_ACCURACY: h.MoveHandler(
            ha.ねっさのあらし_accuracy,
            subject_spec="attacker:self"
        ),
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ねっさのあらし_apply_burn_to_defender,
        ),
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_MODIFY_ACCURACY` | 100（デフォルト） | attacker:self | 天候による命中率補正（あめ: 必中。それ以外は補正なし） |
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 20%の確率で相手を『やけど』状態にする |

`src/jpoke/handlers/move_attack.py` に追加する実装:

```python
def ねっさのあらし_accuracy(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ねっさのあらしの天候による命中率補正。あめのときのみ必中。
    かみなりあらし・こがらしあらしと同様、にほんばれ時の命中率低下はない。
    防御側がばんのうがさを持つ場合、あめでも必中にならない。
    """
    if battle.weather_for(ctx.defender).rainy:
        return HandlerReturn(value=None)  # 必中
    return HandlerReturn(value=value)
```

`ねっさのあらし_apply_burn_to_defender`（既存）は変更しない。

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は汎用ハンドラ側で処理される。本技側に個別のガードは不要。

`battle.weather_for(mon)` はばんのうがさを持つポケモンに対してあめ系天候を「なし」として返すため、
防御側がばんのうがさを持っていても正しく無効化される（`ねっさのあらし_accuracy` 側の実装で対応済み）。

## Priority 根拠

* `ON_MODIFY_ACCURACY` はダメージ計算内部のイベントであり `.internal/spec/turn.md` に priority 記載がないため、
  既存の同種ハンドラ（`かみなりあらし_accuracy`・`こがらしあらし_accuracy`・`ぼうふう_accuracy`）と
  同じデフォルト priority（100）のパターンに従う。
* `ON_DAMAGE_HIT` によるやけど付与も同様に、既存の同種ハンドラ（`ねっとう`・`ねっぷう`等）と
  同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う（既存実装のまま変更なし）。

## 注意点・エッジケース（review-test 向け）

1. **天候による命中率補正**
   - あめ（防御側から見た天候）: 必中。ランク補正（回避・命中ランク）も無視される
     （`ON_MODIFY_ACCURACY` が `None` を返すと `_check_hit` がランク補正前にTrueを返すため）。
   - にほんばれ等その他の天候: 補正なし（`かみなり`・`ぼうふう`と異なり命中率低下しない）。
2. **そらをとぶ状態の相手への非命中**
   - あめ中でも `そらをとぶ` 状態の相手には命中しない（`HIDDEN_MOVE_ALLOWED_MOVES` に未登録のため）。
     専用テストは不要（既存の `そらをとぶ` 揮発状態テストで汎用的にカバーされる想定）。
3. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・やけど付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
4. **テスト方針**
   - `secondary_chance` を1.0に固定し、やけど付与を確認するテスト
     （`test_ねっさのあらし_やけどが発動する`）は既存のため変更なし。
   - 天候による命中率補正（あめ: 必中）を確認するテストを追加する
     （`test_ねっさのあらし_あめ中は必中になる`）。
