# じならし 実装計画書

## 仕様要約（.internal/spec/moves/じならし.md 参照）

* タイプ: じめん / 分類: 物理 / 威力: 60 / 命中: 100 / PP: 20
* 対象: 自分以外の全員（シングルバトルでは相手1体） / 優先度: 0 / 接触なし
* 効果: 追加効果として100%の確率で相手の『すばやさ』ランクを1段階下げる。
* グラスフィールド状態の相手には威力半減（`handlers/field.py` の `グラスフィールド_power_modifier` で
  `ctx.move.name in ["じしん", "じならし", "マグニチュード"]` として汎用的に処理済み・対応済み）。
* あなをほる状態には当たらない（`handlers/volatile.py` の `あなをほる` ヒット対象リストは
  `["じしん", "マグニチュード"]` のみで `じならし` を含まない・対応済み）。

## 現状の実装状況（修正前）

`src/jpoke/data/move.py` にはタイプ・分類・威力・命中・PPとも仕様書と一致する値で登録済みだったが、
`flags` に `"secondary_effect"` が設定されておらず、`.internal/spec/moves/_move_secondary_1.md`
（じならし: defender 確率1 S-1）と食い違っていた。

```python
"じならし": MoveData(
    type="じめん",
    category="physical",
    pp=20,
    power=60,
    accuracy=100,
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.じならし_lower_defender_spd,
        )
    }
),
```

### 修正後

```python
"じならし": MoveData(
    type="じめん",
    category="physical",
    pp=20,
    power=60,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.じならし_lower_defender_spd,
        )
    }
),
```

`flags={"secondary_effect"}` がないと、`ちからずく_boost`（`handlers/ability.py`）による威力1.3倍化と
`ちからずく_disable_secondary_effect` によるすばやさ低下無効化のいずれも発動しなくなってしまう
（`ctx.move.has_flag("secondary_effect")` が常に False になるため）。同種の実装として
`あわ_lower_defender_spd`・`からみつく_lower_defender_spd`・`がんせきふうじ_lower_defender_spd` が
全く同じパターンで `flags={"secondary_effect"}` を持っている。

追加効果ハンドラ（`handlers/move_attack.py` の `じならし_lower_defender_spd`）は仕様と一致しており、
修正不要。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 100%の確率で相手の『すばやさ』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def じならし_lower_defender_spd(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spe": -1})
```

`modify_defender_stats`（`handlers/move.py`）は `chance` 省略時デフォルト1（100%）で
`battle.resolve_secondary_chance` を経由するため、ちからずくによる追加効果無効化は
汎用的に処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（能力ランク低下）は既存の同種ハンドラ（あわ・からみつく・がんせきふうじ 等）と
同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。`.internal/spec/turn.md` にも
`ON_DAMAGE_HIT` は他の追加効果技と共通のタイミングとして扱われており、個別の priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` を追加したことで、ちからずく所持時は威力が1.3倍になり、
     すばやさ低下効果が発動しなくなる（`がんせきふうじ` 等と同じ挙動）。
2. **グラスフィールドとの相互作用**
   - グラスフィールド状態の相手には威力が半減する（既存の `グラスフィールド_power_modifier` で対応済み）。
     テストは `tests/test_field.py::test_グラスフィールド_じならし弱化` に既存。
3. **あなをほるとの相互作用**
   - あなをほる状態の相手には命中せず、追加効果も発動しない（既存の `あなをほる` 実装で対応済み）。
4. **テスト方針**
   - `secondary_chance` を1に固定し、すばやさランク変化が発動することを確認するテストを
     `tests/moves_attack/test_move_sa.py` に追加する（`あわ` の既存テストと同じパターン）。
   - ちからずく所持時に威力1.3倍・すばやさ低下無効化になることを確認するテストを追加する
     （`がんせきふうじ` の既存テストと同じパターン）。
