# ふんえん 実装計画書

## 仕様要約（.internal/spec/moves/ふんえん.md 参照）

* タイプ: ほのお / 分類: 特殊 / 威力: 80 / 命中: 100 / PP: 15
* 対象: 相手1体（シングルバトル） / 優先度: 0 / 接触なし
* 効果: 追加効果として、30%の確率で相手を『やけど』状態にする。
* 『ほのお』タイプの相手にはやけど耐性により付与できない
  （`AilmentManager._can_apply_by_type` の汎用処理で保証済み）。

## 現状の実装状況

`src/jpoke/data/moves/move_ha.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。
以下のとおり仕様と一致していることを確認した（修正不要）。

```python
"ふんえん": MoveData(
    type="ほのお",
    category="special",
    pp=15,
    power=80,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ふんえん_apply_burn_to_defender,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 30%の確率で相手を『やけど』状態にする |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def ふんえん_apply_burn_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="やけど", chance=0.3)
```

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は汎用ハンドラ側で処理される。本技側に
個別のガードは不要。また `AilmentManager._can_apply_by_type` により、『ほのお』タイプの相手には
やけどが付与されない（同ロジックは他の全やけど付与技と共通）。

## Priority 根拠

ダメージを与えた技の追加効果（状態異常付与）は既存の同種ハンドラ（ほうでん・かえんほうしゃ 等）と
同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。個別の priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・やけど付与効果が無効化される。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
2. **ほのおタイプへの耐性**
   - `AilmentManager._can_apply_by_type` が汎用的にやけど付与を無効化するため、本技側の実装・
     テストで個別に再実装する必要はない。
3. **テスト方針**
   - `secondary_chance` を1に固定し、やけど付与を確認するテストが既存（
     `test_ふんえん_やけどが発動する`）。
