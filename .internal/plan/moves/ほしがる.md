# ほしがる 実装計画書

## 仕様要約（.internal/spec/moves/ほしがる.md 参照）

* タイプ: ノーマル / 分類: 物理 / 威力: 60 / 命中: 100 / PP: 25
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 自分が道具を持っていない場合、命中時に相手の持ち物を奪って自分の持ち物にする。
* どろぼう（あくタイプ）と全く同じ追加効果を持つ（威力・命中・PP・タイプ以外は共通）。

## 現状の実装状況

`src/jpoke/data/moves/move_ha.py` に既に登録済み。どろぼう（`src/jpoke/data/moves/move_ta.py`）と
完全に同一のハンドラ（`ha.どろぼう_steal_item`）を共有している。

```python
"ほしがる": MoveData(
    type="ノーマル",
    category="physical",
    pp=25,
    power=60,
    accuracy=100,
    flags={"contact"},
    handlers={
        # .internal\spec\turn.md ON_DAMAGE: 「100 はたきおとす等のアイテム効果」
        # くっつきバリの転移判定（priority=30）より後に発動する必要があるため ON_DAMAGE_HIT を使用する。
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.どろぼう_steal_item,
        )
    }
),
```

## ハンドラ構成

| Event | subject_spec | 関数 | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | `attacker:self` （デフォルト） | `どろぼう_steal_item` | ダメージ命中後、`battle.item_manager.take_item(ctx.defender)` を呼び出し、条件を満たせば道具を奪う |

`take_item` 内部の判定ロジック・`ON_CHECK_ITEM_CHANGE` 経由の各種ガード（ねんちゃく・マルチタイプ・
こだいかっせい/クォークチャージ・専用道具等）は、どろぼう実装計画書（.internal/plan/moves/どろぼう.md）の
記載と完全に共通であり、本技固有の追加実装は不要。

## Priority 根拠

`.internal/spec/turn.md` の `ON_DAMAGE` セクション（コード上は `Event.ON_DAMAGE_HIT` に対応）に
「100 はたきおとす等のアイテム効果」と明記されている。くっつきバリの転移判定
（`ON_DAMAGE_HIT` priority=30）や、さめはだ・てつのトゲ等の接触ダメージ特性
（`ON_DAMAGE_HIT` デフォルト優先度=100、同時刻ハンドラ）より後に道具を奪う必要があるため、
`MoveHandler` のデフォルト優先度（100）をそのまま使用する（`どろぼう`・`はたきおとす` と同一パターン）。

第五世代以降の仕様では、ほしがるの効果は相手のダメージ被弾特性・ひんし判定より後に発動し、
相手をひんしにさせた場合でも道具を奪える。`ON_DAMAGE_HIT` はダメージ適用後
（`core/move_executor.py` の `_execute_hit` 内、`modify_hp` 実行後）に発火するため、
この仕様を自然に満たす。またダメージが0（例: みがわりに完全に防がれた場合）は
`actual_damage <= 0` の分岐で `ON_DAMAGE_HIT` 自体が発火しないため、
「みがわりに防がれた場合は奪えない」という仕様も自動的に満たされる。

## 実装箇所

### handlers/move_attack.py

既存の `どろぼう_steal_item`（どろぼうと共有）で完結しており、追加実装は不要。

```python
def どろぼう_steal_item(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """どろぼう・ほしがるのアイテム奪取効果。"""
    battle.item_manager.take_item(ctx.defender)
    return HandlerReturn(value=value)
```

### data/moves/move_ha.py

登録済み・修正不要（`どろぼう`・`ほしがる` ともに同一ハンドラを参照）。

## 注意点・エッジケース（review-test 向け）

どろぼう実装計画書（.internal/plan/moves/どろぼう.md）と共通。特に以下を確認する:

1. **自分がすでに道具を持っている場合** → 追加効果不発（ダメージのみ）。
2. **相手が道具を持っていない場合** → 追加効果不発。
3. **相手の特性ねんちゃく** → `ON_CHECK_ITEM_CHANGE` 経由でブロックされ、既存実装で対応済み。
4. **相手の特性ホズのみ等の道具消費が先に発動するケース** → タイプ半減実がノーマルタイプ用の
   ホズのみのように、道具消費が先に発動して奪取が失敗する場合がある
   （既存テスト `tests/items/test_item_ha.py::test_ホズのみ_ほしがるでは奪われない` で確認済み）。
