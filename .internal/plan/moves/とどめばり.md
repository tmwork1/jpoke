# 計画書: とどめばり（再レビュー）

## 仕様要約

`.internal/spec/moves/とどめばり.md` 参照。

* この技で相手を倒すと、自分の『こうげき』ランクが3段階上がる。
* 相手を倒すと自身が発動するランク上昇特性（じしんかじょう等）より先に、
  この技自身の効果が発動する。
* 相手のミイラ等で自分の特性が変化した場合は、特性変化後にこの技の効果が発動する。
* 第八世代以降、ばけのかわのフォルムチェンジ消費ダメージで相手を倒した場合も
  この技の効果は発動する（技自体のダメージは0でも発動する）。

## 既存実装の問題点（今回のレビューで検出）

### 1. `Event.ON_DAMAGE_HIT` に登録されていたため、ばけのかわ由来のKOで発動しない

`core/move_executor.py` の `_execute_hit` は、技のダメージ適用 (`modify_hp`) の結果
`actual_damage <= 0` の場合、`Event.ON_DAMAGE_HIT` を発火せずに早期リターンしていた。

ばけのかわの `ON_MODIFY_MOVE_DAMAGE` ハンドラ（`ばけのかわ_block_damage`）は、
技自体のダメージを0にしつつ、フォルムチェンジ消費ダメージとして
`battle.modify_hp(ctx.defender, r=-1/8)` を直接呼んでいる。このダメージで
相手がひんしになった場合でも、後続の `modify_hp` による技ダメージは0
（`actual_damage == 0`）となるため、`ON_DAMAGE_HIT` に登録されていた
`とどめばり_boost_attacker_atk` は発火しなかった。

実機確認（修正前）:

```
ピカチュウ「とどめばり」 → コイキング（ばけのかわ、HP1）
結果: コイキングはフォルムチェンジ消費ダメージでひんし化するが、
      ピカチュウのこうげきランクは0のまま（上がらない）
```

この問題は `Event.ON_MOVE_KO` を使うじしんかじょう等の特性でも同様に発生していた
（`.internal/spec/abilities/ばけのかわ.md` に「攻撃側の特性じしんかじょう/ビーストブースト/
しろのいななき/くろのいななき/きずなへんげは発動する」「とどめばりで倒されたときは、
使用者の攻撃が3段階上がる」と明記されている）。

### 2. ミイラ/さまようたましい/とれないにおいが`priority`未指定で速度依存の順序になっていた

`data/ability.py` の `ミイラ`・`さまようたましい`・`とれないにおい` は、いずれも
`Event.ON_DAMAGE_HIT` に登録されていたが `priority` を指定しておらず、デフォルト値
`100` が使われていた。とどめばりの効果ハンドラも同じ `Event.ON_DAMAGE_HIT`・
`priority=100` で登録されていたため、両者が同時にトリガーされる場面（相手を倒すのと
同時に自分の特性がミイラ等に変化する場合）で、`EventManager._sort_handlers` が
同優先度のハンドラを素早さで順序決定してしまい、攻撃側の方が素早い場合に
「特性変化前」の特性（たんじゅん/あまのじゃく）でとどめばりの効果が適用される
誤りが発生していた。

実機確認（修正前）:

```
ピカチュウ（あまのじゃく、素早さ90）「とどめばり」 → カビゴン（ミイラ、素早さ低）
結果: 攻撃側の特性がミイラに変化する前に atk-3 が適用され、
      その後ミイラに変化した（本来は atk+3 になるべき）
```

同様の反応特性（てつのトゲ・のろわれボディ・くだけるよろい・みずがため・
ふうりょくでんき・いかりのつぼ等）はいずれも `Event.ON_DAMAGE_HIT` に
`priority=20` を明示しており、この慣例に合わせる。

## 修正方針

### `core/move_executor.py`

`_execute_hit` の `actual_damage <= 0` の早期リターン内で、`ctx.defender.fainted`
（このヒットで新たにひんしになった場合のみ真になる。ループ先頭の
`if ctx.defender.fainted or ctx.attacker.fainted: break` により、既にひんしの
状態でこのヒット処理に入ることはない）を判定し、真であれば `Event.ON_MOVE_KO`
のみを追加で発火してから return する。`Event.ON_DAMAGE_HIT`・`hits_taken` 等
既存のダメージ量に依存する処理は変更しない（実ダメージ0の場合に不要な副作用が
発生するリスクを避けるため）。

### `data/move.py`

とどめばりの効果ハンドラの発火イベントを `Event.ON_DAMAGE_HIT` から
`Event.ON_MOVE_KO` に変更する。`Event.ON_MOVE_KO` は `move_executor.py` で
相手がひんしになったときのみ発火するため（ばけのかわ経由のひんしも含む）、
ハンドラ内の生存判定が不要になる。

じしんかじょう等のKOトリガー特性（`Event.ON_MOVE_KO`、`priority` 未指定＝
デフォルト100）より必ず先に発動するよう、`priority=50` を明示する
（`.internal/spec/turn.md` に `Event.ON_MOVE_KO` 自体の掲載はないため、既存の
同種ハンドラ間の相対順序から根拠づける）。

### `data/ability.py`

`ミイラ`・`さまようたましい`・`とれないにおい` の `Event.ON_DAMAGE_HIT` ハンドラに
`priority=20` を明示し、同種の「受けた時の特性反応」（てつのトゲ等）と同じ優先度に
揃える。とどめばりが `Event.ON_MOVE_KO`（`ON_DAMAGE_HIT` より後に発火する別イベント）
に移動したことで直接の競合は解消されるが、素早さ依存で結果が変わる状態は
それ自体が仕様と異なるバグのため、根本原因として合わせて修正する。

## ハンドラ構成（修正後）

| 技名/特性名 | イベント | priority | subject_spec | 役割 |
|---|---|---|---|---|
| とどめばり | `Event.ON_MOVE_KO` | 50 | attacker:self | 相手を倒すとこうげきランク+3 |
| ミイラ | `Event.ON_DAMAGE_HIT` | 20 | defender:self | 直接攻撃で攻撃者の特性をミイラにする |
| さまようたましい | `Event.ON_DAMAGE_HIT` | 20 | defender:self | 直接攻撃で攻撃者の特性をさまようたましいにする |
| とれないにおい | `Event.ON_DAMAGE_HIT` | 20 | defender:self | 直接攻撃で攻撃者の特性をとれないにおいにする |

## リーサル計算

対象外（`.internal/progress/move.md` リーサル実装列: n/a）。とどめばりの効果は
相手を倒した後にのみ発動する自己バフであり、当該ターンの被弾側の生死判定
（`LethalContext`）には影響しない。
