# 計画書: きょけんとつげき

## 仕様要約

（`.internal/spec/moves/きょけんとつげき.md` 参照）

* 物理・ドラゴンタイプ、威力120・命中100・PP5、直接攻撃。
* 技が成功した場合（タイプ無効化・まもる等でのブロック・命中失敗を除く）、使用者に専用の
  揮発性状態「きょけんとつげき」を付与する。
* この状態の間、相手から使用者への技は必ず命中し（一撃必殺技含む）、ダメージが2倍になる。
* 状態は「使用者の次の行動が始まるタイミング」で、行動できるか否かに関わらず解除される。
* ダメージ固定技は威力計算を経由しないため、2倍の対象外。

## 既存実装の問題点

`docs/data/move.py` の `きょけんとつげき` は `handlers={}` で「追加効果なし」というコメントが
付いていたが、これは誤り。本技は上記の重大なデメリット効果（次の相手の攻撃を必中・2倍にする）を
持つため、未実装だった。

## イベント選定の根拠

* **状態の付与**: `Event.ON_HIT`（`.internal/spec/turn.md` 掲載）。
  「ダメージ発生後の処理（みがわりに被弾しても発動）」であり、まもる・キングシールド等による
  無効化（`ON_TRY_MOVE_1`、命中判定より前）やタイプ相性による無効化（`ON_TRY_MOVE_2`、
  命中判定より前）で技が不発になった場合はそもそも発火しないため、「まもるやフェアリータイプ
  無効化時はデメリットが発動しない」という一次情報の仕様を自然に満たす。
  命中に失敗した場合も同様に `ON_HIT` 自体が発火しないため対応不要。
* **相手の技を必中にする**: `Event.ON_MODIFY_ACCURACY`（`.internal/spec/turn.md` 掲載外だが、
  既存の `ちいさくなる_guaranteed_hit`（`data/volatile.py`「ちいさくなる」エントリ）と同じ
  パターンを踏襲。`value=None` を返すことで `MoveExecutor._check_hit` が必中として扱う
  （`core/move_executor.py` 158〜168行目）。一撃必殺技も同じ `ON_MODIFY_ACCURACY` を経由して
  命中判定を行うため、追加の分岐なしで「一撃必殺技を受けたときも必中になる」仕様を満たす。
* **相手の技のダメージを2倍にする**: `Event.ON_CALC_DAMAGE_MODIFIER`（`core/damage.py`
  266行目付近の `_calc_final_power` とは別に、`calc_damages` 内でタイプ相性・やけど補正の後、
  まもる貫通補正の前に適用される最終ダメージ倍率。既存の `リフレクター_reduce_damage`
  （`data/field/side_field.py`「リフレクター」エントリ）と同じ「defender:self」パターンを
  踏襲し、`apply_fixed_modifier(value, 8192)` で2倍にする）。
  `calc_damages` は技の威力が `None`/0 の場合（固定ダメージ技）は最初に `return [0]` で
  打ち切られ、`ON_CALC_DAMAGE_MODIFIER` 自体が発火しないため、「ダメージ固定技によるダメージは
  増えない」という一次情報の仕様を自然に満たす。
* **状態の解除**: `Event.ON_TRY_ACTION`（`core/move_executor.py` 274行目 `self.action_success
  = self._events.emit(Event.ON_TRY_ACTION, ctx, True)`）。
  この技と全く同じ「次の自分の行動が始まるまで持続し、行動可否に関わらず解除される」性質を持つ
  `みちづれ`（`みちづれ_remove`、`data/volatile.py`「みちづれ」エントリ）が
  `subject_spec="attacker:self", priority=10` を使用しているため、これを踏襲する。
  `priority=10` は、こんらんの自傷判定（`こんらん_try_action` priority=110）・まひ判定
  （`まひ_action` priority=120）・ひるみ判定（`ひるみ_block_action` priority=40）よりも
  早く実行されるため、「こんらんで自分を攻撃するより前に解除される」「状態異常や状態変化で
  動けなかったときでも解除される」という一次情報の仕様を満たす。

## ハンドラ構成

| 対象 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| きょけんとつげき（技） | `Event.ON_HIT` | デフォルト（100） | `attacker:self` | 使用者に「きょけんとつげき」状態を付与する |
| きょけんとつげき（状態） | `Event.ON_MODIFY_ACCURACY` | デフォルト（100） | `defender:self` | 相手からの技を必中にする |
| きょけんとつげき（状態） | `Event.ON_CALC_DAMAGE_MODIFIER` | デフォルト（100） | `defender:self` | 相手からの技のダメージを2倍にする |
| きょけんとつげき（状態） | `Event.ON_TRY_ACTION` | 10 | `attacker:self` | 自身の次の行動開始時に状態を解除する |

```python
def きょけんとつげき_apply_self_volatile(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """きょけんとつげき成功時、自身に『きょけんとつげき』状態を付与する。

    この状態は自身の次の行動が始まるタイミングで解除される。
    """
    return apply_volatile_to_attacker(battle, ctx, value, volatile="きょけんとつげき")
```

```python
def きょけんとつげき_guaranteed_hit(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """きょけんとつげき状態: 相手から受ける技が必ず命中する（一撃必殺技も含む）。"""
    return HandlerReturn(value=None, stop_event=True)


def きょけんとつげき_double_damage(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """きょけんとつげき状態: 相手から受ける技のダメージを2倍にする。

    ダメージ固定技は威力計算を経由しないため対象外となる。
    """
    return HandlerReturn(value=apply_fixed_modifier(value, 8192))


def きょけんとつげき_remove(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """きょけんとつげき状態: 自身の行動が始まるタイミングで解除する。"""
    return remove_volatile(battle, ctx, value, volatile="きょけんとつげき")
```

## テスト

* 命中したとき、使用者に「きょけんとつげき」状態が付与されること。
* 「きょけんとつげき」状態の間、相手の技が命中率0%指定でも必中すること。
* 「きょけんとつげき」状態の間、相手の技のダメージが2倍になること。
* まもるで防がれたとき、状態が付与されないこと。
* フェアリータイプに無効化されたとき、状態が付与されないこと。
* 技が外れたとき、状態が付与されないこと。
* 自身の次の行動が始まると状態が解除されること（まひ等で行動不能になった場合も解除されること）。

## 注意点・エッジケース

* 「バッグから道具を使った場合、その間だけダメージ2倍が継続する」という一次情報の記述は、
  本プロジェクトの対象（ポケモンチャンピオンズシングルバトル）ではバトル中のどうぐ使用を
  想定していないため対応しない。
* 「さいはいで強制的にきょけんとつげきを出した場合」「特性おどりこで行動した場合」に関する
  一次情報の記述は、いずれも `Event.ON_TRY_ACTION` を経由する行動開始時に解除される
  という基本挙動でカバーされる想定だが、組み合わせの個別テストは行わない
  （`さいはい`・`おどりこ` 自体は実装済み）。
