# 実装計画書: いやしのはどう

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 12（champions基準。src/jpoke/data/ps-champ-ja/moves.json準拠） /
  命中: なし（必中） / 対象: 1体選択（相手）
- 相手（defender）の HP を最大 HP の **1/2** 回復する（端数**切り上げ**。一次情報の「技の仕様」節に
  明記されている、この技固有の丸め方向）
- 使用者が特性 **メガランチャー** を持つ場合、回復量が最大 HP の **3/4** になる
  （端数は五捨五超入 = `round_half_down`）
- `heal` フラグあり（`かいふくふうじ` の対象になる）。`pulse` フラグあり（はどう技、メガランチャー
  判定用。ただし本技は威力を持たないため `ON_CALC_POWER_MODIFIER` 経由のメガランチャー効果は発動せず、
  回復量側の補正は本技専用のハンドラ内で直接判定する）
- 相手の HP が満タンの場合は技が失敗する

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_APPLY_MOVE` セクションに以下の記載がある：

> | 100 | いやしのはどう等: HP満タン |

このイベントはコード上の `ON_BEFORE_APPLY_MOVE` に対応する。ただし、同種の回復技（`じこさいせい`・`なまける`・`いのちのしずく`・`こうごうせい`・`つきのひかり`・`あさのひざし`）はすべて `ON_STATUS_HIT`（priority=100）内で HP 満タンチェックと回復処理を一括して行うパターンを採用している。`いやしのはどう` も同パターンで実装し、priority はデフォルト（100）とする。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

```python
def いやしのはどう_heal_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """いやしのはどう: 相手の最大HPの1/2（メガランチャー所持時は3/4）を回復する。

    通常の端数は切り上げ、メガランチャー所持時は五捨五超入（round_half_down）で丸める。
    HPが満タンの場合は失敗する。
    """
    mon = ctx.defender
    if mon.hp == mon.max_hp:
        return HandlerReturn(value=False, stop_event=True)
    if ctx.attacker.ability.name == "メガランチャー":
        heal = round_half_down(mon.max_hp * 3 / 4)
    else:
        heal = (mon.max_hp + 1) // 2
    battle.modify_hp(mon, v=heal)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース

- **HP 満タン時の失敗**: `mon.hp == mon.max_hp` で判定し、`HandlerReturn(value=False, stop_event=True)` を返す。`move_executor.py` 側で `MOVE_FAILED` ログが記録されるため、ハンドラ内でのログ記録は不要
- **回復対象は defender（相手）**: `じこさいせい` などの自己回復技と異なり、`ctx.defender` を参照する点に注意。`heal` フラグは attacker 側の `かいふくふうじ` ではなく defender 側の回復無効化にも適用される（`ON_MODIFY_HEAL` は target=defender で発火するため、defender が かいふくふうじ 状態なら汎用処理で回復量が 0 になる）
- **かいふくふうじとの相互作用**:
  - 使用者（attacker）が かいふくふうじ 状態の場合は `heal` フラグを持つ技として汎用処理
    （`かいふくふうじ_try_action`）で選択できない
  - 対象（defender）が かいふくふうじ 状態の場合は `ON_MODIFY_HEAL` の汎用処理
    （`かいふくふうじ_block_heal`）で回復量が 0 になる
  - いずれも本ハンドラ側での追加チェックは不要
- **回復量の端数処理**: 一次情報（技の仕様節）により、通常時は**切り上げ**が明記されている
  （他の多くの自己回復技のような切り捨てとは異なる）。`(mon.max_hp + 1) // 2` で切り上げ除算を行う
  （`クロロブラスト_recoil` 等の既存パターンに合わせる）。`battle.modify_hp` の `r=` 引数は内部で
  `int()`（切り捨て）を使うため、切り上げが必要な本技では `v=` 引数に計算済みの値を渡す
- **HP が 1 でも減っていれば成功**: `mon.hp == mon.max_hp` で判定するため、回復量が 0 になるケース（最大 HP が 2 未満など）でも満タンでない限り成功扱いとなる
- **メガランチャーとの相互作用**: メガランチャー特性を持つポケモンが使うと、回復量が defender の
  最大 HP の 3/4 になる。本技は威力を持たない変化技のため `ON_CALC_POWER_MODIFIER` 経由の汎用
  メガランチャー処理（`メガランチャー_modify_power`）は発動せず、本ハンドラ内で
  `ctx.attacker.ability.name == "メガランチャー"` を直接判定する（かたやぶり等の特性無効化は
  攻撃側自身の特性発動可否には影響しないため追加考慮不要）。端数は五捨五超入
  （`round_half_down`）で丸める（一次情報の記載どおり）
