## 仕様要約

- ひこうタイプ・物理・威力 90・命中率 95・PP 15・優先度 0・直接攻撃
- 2ターン技
  - 1ターン目: 空中へ飛び上がる（`そらをとぶ` 揮発状態が付与される）。攻撃は行わない。
  - 2ターン目: 相手を攻撃する。攻撃終了後に揮発状態が解除される。
- 空中にいる間は以下の技以外は命中しない:
  - かぜおこし・かみなり・たつまき・ぼうふう・うちおとす（第五世代以降）
  - スカイアッパー（第三〜七世代 / 本実装スコープ外）
  - サウザンアロー（本実装では別技側で対応）
- かぜおこし・たつまき が命中した場合、威力が 2 倍になる
- 1ターン目はPPを消費しない（ためターン）
- そらをとぶ状態中は交代できない（ゴーストタイプを除く）
- まもる: ○（2ターン目のみ判定）
- じゅうりょく状態では技が失敗する（`ON_TRY_MOVE_1` 失敗条件 priority=30、別実装）

```python
def そらをとぶ_boost_power(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """そらをとぶ状態の相手に対して かぜおこし・たつまき の威力を2倍にする。"""

    """そらをとぶ状態の回避判定"""
    return can_hit_hidden_target(battle, ctx, value, "そらをとぶ")

def そらをとぶ_remove_volatile(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """そらをとぶ状態の解除"""
```

### 追加すべきテスト（`tests/moves_attack/test_move_sa.py` への追加）

```python
def test_そらをとぶ_2ターンで攻撃する():
    """1ターン目はダメージを与えず、2ターン目にダメージを与える"""
    battle = t.start_battle(
        team0=[Pokemon("ピカチュウ", move_names=["そらをとぶ"])],
        team1=[Pokemon("カビゴン")],
        accuracy=100,
    )
    defender = battle.actives[1]
    hp_before = defender.hp

    # 1ターン目: 揮発状態付与のみ、ダメージなし
    t.run_move(battle, 0)
    assert defender.hp == hp_before
    assert "そらをとぶ" in battle.actives[0].volatiles

    # 2ターン目: ダメージあり、揮発状態解除
    t.run_move(battle, 0)
    assert defender.hp < hp_before
    assert "そらをとぶ" not in battle.actives[0].volatiles


def test_そらをとぶ_1ターン目はPPを消費しない():
    """1ターン目（溜めターン）はPPを消費しない"""
    battle = t.start_battle(
        team0=[Pokemon("ピカチュウ", move_names=["そらをとぶ"])],
        team1=[Pokemon("カビゴン")],
        accuracy=100,
    )
    attacker = battle.actives[0]
    initial_pp = attacker.moves[0].pp

    # 1ターン目: PP消費なし
    t.run_move(battle, 0)
    assert attacker.moves[0].pp == initial_pp

    # 2ターン目: PP消費あり
    t.run_move(battle, 0)
    assert attacker.moves[0].pp == initial_pp - 1


def test_そらをとぶ_空中は通常技を回避する():
    """1ターン目のそらをとぶ中は通常技を受けない"""
    battle = t.start_battle(
        team0=[Pokemon("ピカチュウ", move_names=["そらをとぶ"])],
        team1=[Pokemon("カビゴン", move_names=["たいあたり"])],
        accuracy=100,
    )
    attacker = battle.actives[0]
    hp_before = attacker.hp

    # 1ターン目: そらをとぶを使用（揮発状態付与）
    t.run_move(battle, 0)
    # 相手のたいあたりはミスする
    t.run_move(battle, 1)
    assert attacker.hp == hp_before


def test_そらをとぶ_かみなりは命中する():
    """そらをとぶ状態でもかみなりは命中する"""
    battle = t.start_battle(
        team0=[Pokemon("カビゴン", move_names=["かみなり"])],
        team1=[Pokemon("カビゴン", move_names=["そらをとぶ"])],
        accuracy=100,
    )
    attacker, defender = battle.actives
    # 1ターン目: defenderがそらをとぶ（揮発状態付与）
    t.run_move(battle, 1)
    # かみなりが命中する
    t.run_move(battle, 0)
    assert battle.move_executor.move_success
    assert defender.hp < defender.max_hp
```

## 注意点・エッジケース（review-test 向け）

### 揮発状態解除タイミング

- `ON_HIT` ハンドラ（`そらをとぶ_remove_volatile`）は `attacker:self` で登録されている。
  - `attacker` は「そらをとぶを使った側（飛んでいる側）」。
  - ダメージヒット後に自身の揮発状態を解除する仕組み。
- 攻撃がミスした場合（命中判定失敗）は `ON_HIT` が発火しないため、揮発状態が次ターンも残る。ただし `forced=True` の揮発状態は次ターンのコマンドが FORCED に強制されるため、実質的に攻撃失敗 → 次ターンも同じ攻撃が実行される（攻撃失敗ターンが挿入される形になる）。

### まひ・こんらんで攻撃できなかった場合

- `.internal/spec/volatiles/そらをとぶ.md` に「まひ・こんらん・なまけなどで2ターン目に攻撃できなかったときも解除される（状態変化が解除され、攻撃もできない）」とある。
- `ON_HIT` は命中 → ダメージ後に発火するが、まひ等で攻撃そのものが不発になった場合は `ON_HIT` が発火しない。
- このケースの揮発状態解除は現在の実装では未対応の可能性がある。レビューで確認が必要。

### HIDDEN_MOVE_ALLOWED_MOVES リストの確認

現在のリスト: `["かぜおこし", "たつまき", "かみなり", "ぼうふう"]`
- スカイアッパー: 技が未実装のため対象外
- サウザンアロー: うちおとす揮発状態側で浮遊無効化を実装済み（スコープ外と明記）
- ロックオン状態による命中: ロックオン揮発状態側で実装済み
- ノーガード特性による命中: 特性側で実装済み

### かぜおこし・たつまき の威力 2 倍と回避判定の関係

- `そらをとぶ_can_hit_hidden_target` で「かぜおこし」「たつまき」は回避されずに命中する。
- その後 `そらをとぶ_boost_power` で威力が 2 倍になる。
- これら 2 つのハンドラが独立して動作するため、回避されないことと威力 2 倍は別々に保証されている。

### `ON_CALC_POWER_MODIFIER` のイベントタイプ

- `AttackContext` を受け取るハンドラ（`そらをとぶ_boost_power`）は `defender:self` で登録。
- `subject_spec="defender:self"` は「防御側（飛んでいる側）の揮発状態が、攻撃側の威力計算に影響を与える」パターン。

### 交代禁止の実装

- `ON_CHECK_TRAPPED` の `check_trapped_not_ghost` は汎用ヘルパー関数であり、そらをとぶ専用ではない。
- ゴーストタイプは交代禁止が無効になる（既実装済み）。
