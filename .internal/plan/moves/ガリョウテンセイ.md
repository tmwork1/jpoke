# ガリョウテンセイ 実装計画書

## 仕様要約（.internal/spec/moves/ガリョウテンセイ.md 参照）

* タイプ: ひこう / 分類: 物理 / 威力: 120 / 命中: 100 / PP: 8
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 命中して技が成功した後、確定で自分の『ぼうぎょ』『とくぼう』ランクが各1段階下がる
  （インファイトのひこうタイプ版）。
* レックウザの技依存メガシンカ（メガストーン不要）は本プロジェクトのメガシンカ実装
  （持ち物トリガーのみ）の対象外のため実装しない（詳細は仕様書参照）。

## 現状の実装状況・修正内容

`src/jpoke/data/moves/move_ka.py` に既に登録されていたが、以下の誤りがあったため修正する。

### 修正前

```python
"ガリョウテンセイ": MoveData(
    type="ひこう",
    category="physical",
    pp=5,
    power=120,
    accuracy=100,
    flags={"contact", "secondary_effect"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.ガリョウテンセイ_lower_defender_spd,
        )
    }
),
```

### 修正後

```python
"ガリョウテンセイ": MoveData(
    type="ひこう",
    category="physical",
    pp=8,
    power=120,
    accuracy=100,
    flags={"contact"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.ガリョウテンセイ_lower_attacker_stats,
        )
    }
),
```

### 修正点

1. **PP**: 5 → 8。ポケモンチャンピオンズでは全技PPが8/12/16/20の4段階に統一されており、
   本家基準PP5の技（インファイト・だいもんじ等）はチャンピオンズPP8に対応する
   （`src/jpoke/data/ps-champ-ja/moves.json` に本技自体は未収録のため類推適用。仕様書参照）。
2. **flags**: `secondary_effect` を削除。自分自身への確定ランク低下は「追加効果」に
   該当せず、ちからずく・てんのめぐみの対象にもならない（インファイトと同じ扱い）。
   このフラグが残っていると `resolve_secondary_chance` 経由でちからずく所持時に
   ランク低下が誤って無効化されてしまう。
3. **ハンドラ関数名**: `ガリョウテンセイ_lower_defender_spd` →
   `ガリョウテンセイ_lower_attacker_stats`。実装ロジック自体は `modify_attacker_stats`
   で正しく自分（ctx.attacker）のランクを下げていたが、関数名が「相手のすばやさを
   下げる」という誤った内容を示しており、実態（自分の防御・特防を下げる）と
   一致していなかったため、インファイトの命名規則
   （`インファイト_lower_attacker_stats`）に合わせて修正する。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 確定で自分の『ぼうぎょ』『とくぼう』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（修正後）:

```python
def ガリョウテンセイ_lower_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ガリョウテンセイ: 命中時に確定で自分の『ぼうぎょ』『とくぼう』ランクが1段階ずつ下がる。

    インファイトのひこうタイプ版。自分自身のランクを下げる確定効果のため、
    ちからずくの対象外（secondary_effect フラグ非付与）。
    """
    return modify_attacker_stats(battle, ctx, value, stats={"def": -1, "spd": -1})
```

`modify_attacker_stats`（`handlers/move.py`）は ON_HIT では value（実際のダメージ量）を
そのまま次のハンドラへ引き継ぐため、本技側で追加のダメージ処理は不要。

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_HIT` には「自分のランクが下がる反動」が priority 10 と
記載されているが、同種の既存実装（インファイト・オーバーヒート）はいずれも明示的な
priority 指定なし（`MoveHandler` デフォルトの100）で登録されており、本技も既存実装との
一貫性を優先してデフォルト値のまま登録する。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags` に `secondary_effect` を付与しないため、ちからずく所持時も
     `resolve_secondary_chance` を経由せず、常に自分のランク低下が発動する
     （威力上昇は別途 `ちからずく_boost` 側で処理されるため、ここでの実装には影響しない）。
2. **命名不整合の修正**
   - 既存の関数名 `ガリョウテンセイ_lower_defender_spd` は実装内容（自分の防御・特防を
     下げる）と一致しない誤った命名だったため修正。ロジック自体（`ctx.attacker` を
     対象にする）は元から正しかった。
3. **テスト方針**
   - インファイトの既存テスト（`test_インファイト_ぼうぎょととくぼう1段階低下が発動する`
     `test_インファイト_ちからずくは無関係で発動する`）と同じパターンで、
     `test_ガリョウテンセイ_ぼうぎょととくぼう1段階低下が発動する`
     `test_ガリョウテンセイ_ちからずくは無関係で発動する` を追加する。
