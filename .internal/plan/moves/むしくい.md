# 計画書: むしくい

## 仕様要約（.internal/spec/moves/むしくい.md 参照）

- むしタイプ / 物理 / 威力60 / 命中100 / PP20 / 対象: 相手1体 / 優先度0 / 接触あり
- 相手が「バトルに効果のある」きのみを持っていれば、命中後にそのきのみを奪い、
  自分（攻撃者）が即座に消費してその効果を受ける。
- ついばむ（ひこうタイプ）と全く同じ追加効果を持つ姉妹技で、ハンドラを共有する。
- 追加効果ではないため、りんぷん・おんみつマント・ちからずくの影響を受けない。
- `src/jpoke/data/ps-champ-ja/moves.json:894` に記載あり（むし / 物理 / 60 / 100 / 20 / 接触あり / まもる有効）。

## 既存実装との差分

既存実装（`data/moves/move_ma.py` の `"むしくい"` エントリ、
`handlers/move_attack.py` の `むしくい_steal_and_use_berry`）は、以下2点で仕様と食い違っていたため修正した。

### 修正1: みがわりに防がれた場合もきのみを奪ってしまう不具合

仕様書（一次情報）:「対象のみがわり状態に防がれた場合、きのみを食べることはできない。」

修正前の実装は `defender.has_item()` のみを見ており、みがわりに阻まれたかどうかを
判定していなかった。このため、みがわり越しに命中した場合でも実体のきのみを奪ってしまっていた
（実機で再現・確認済み）。

`Event.ON_MODIFY_MOVE_DAMAGE` でみがわりがダメージを肩代わりすると `ctx.substitute_damage`
（`AttackContext` のフィールド）に肩代わり量が記録される
（`handlers/volatile.py` の `みがわり_block_damage`）。これを利用し、
`むしくい_steal_and_use_berry` の先頭で `ctx.substitute_damage > 0` の場合は
何もせず処理を打ち切るようにした。

### 修正2: ねんちゃく持ちをこの技でひんしにさせた場合に奪えない不具合

仕様書（一次情報、第五世代以降）:「対象が特性ねんちゃくである場合、きのみを食べることは
できない。ただし、ねんちゃくのポケモンをこの技でひんしにさせた場合はきのみを食べることができる。」

修正前の実装は `take_item` → `can_change_item` → `Event.ON_CHECK_ITEM_CHANGE` を経由し、
`ねんちゃく_prevent_item_change`（`handlers/ability.py`）が対象の生死に関係なく常に
奪取を阻止していた。

この仕様はむしくい・ついばむに固有の挙動であり、他の道具奪取技（ぬすむ・すりかえ等）には
適用されない（ねんちゃくは通常、生死に関係なく道具の奪取・交換・除去を防ぐ）。
そのため、`ねんちゃく_prevent_item_change` を無条件に変更するのではなく、
呼び出し元が明示的に指定した場合のみねんちゃくを無視できる仕組みを追加した。

* `core/context.py` の `EventContext` に `ignore_sticky_hold: bool = False` を追加。
* `core/item_manager.py` の `can_change_item` / `swap_items` / `take_item` に
  `ignore_sticky_hold` キーワード引数を追加し、`EventContext` に伝播させる
  （デフォルト `False` のため既存の全呼び出し元は影響を受けない）。
* `handlers/ability.py` の `ねんちゃく_prevent_item_change` で
  `ctx.dry_run or ctx.ignore_sticky_hold` の場合は阻止しないよう変更。
* `むしくい_steal_and_use_berry` から
  `battle.item_manager.take_item(defender, ignore_sticky_hold=will_faint)` を呼び出す。
  当初は `Event.ON_HIT` の時点で確定済みの `defender.fainted` を利用していたが、
  後述の「既知の制限（解消済み）」の修正で発動タイミングを `Event.ON_MODIFY_MOVE_DAMAGE`
  （HP反映前）へ移したため、`defender.fainted` はまだ確定していない。代わりに
  `value >= defender.hp`（がんじょう等の生存補正を経た確定ダメージが現在HP以上か）で
  「この技で対象をひんしにさせるか」を予測し、同じ役割を果たす。

なお、ねんちゃく以外にアイテム奪取を阻止する特性（マルチタイプ・ARシステム）は
いずれも非きのみ専用アイテム（メガランチャー等の対象ではないプレート・ドライブ類）を
対象としており、`defender.item.is_berry()` が真の時点でこれらが競合することはない。

## ハンドラ構成

| Event | subject_spec | 関数 | 役割 |
|---|---|---|---|
| ON_MODIFY_MOVE_DAMAGE (priority=110) | attacker:self / defender:self | `むしくい_steal_and_use_berry` | defender がきのみを持っていれば奪取し、attacker が即座に消費する |

### handlers/move_attack.py: `むしくい_steal_and_use_berry`（再修正: ON_HIT → ON_MODIFY_MOVE_DAMAGE）

```python
def むしくい_steal_and_use_berry(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """むしくい・ついばむ: 相手のバトルに効果のあるきのみを奪って自分が消費する。

    Event.ON_MODIFY_MOVE_DAMAGE の終盤（がんじょう・きあいのタスキ等HP1残し補正より後、
    priority=110）で発動する。HP反映（battle.modify_hp）より前のこのタイミングで奪う
    ことで、被弾側自身のHP閾値で発動するきのみ（オボンのみ等、Event.ON_HP_CHANGED で
    反応する）より確実に先に奪取を成立させる。本家の「ダメージを与えると同時に相手の
    きのみを奪う」仕様に対応する。

    みがわりに防がれた場合はきのみを食べられない（実体への攻撃ではないため）。
    take_item はねんちゃくチェックを内包し、attacker がアイテムを
    持っている場合は失敗して何もしない。ただし、この技で対象をひんしにさせる場合は
    ねんちゃくによる阻止を無視して奪取できる（第五世代以降の仕様）。
    HP反映前のため、value（がんじょう等の生存補正を経た確定ダメージ）が
    defender の現在HP以上かどうかで「このヒットでひんしになるか」を判定する。
    """
    if ctx.substitute_damage > 0:
        return HandlerReturn(value=value)
    defender = ctx.defender
    attacker = ctx.attacker
    if not defender.has_item() or not defender.item.is_berry():
        return HandlerReturn(value=value)
    will_faint = value >= defender.hp
    # take_item でdefenderのきのみをattackerに移す（ねんちゃく等で失敗する場合あり）
    if not battle.item_manager.take_item(defender, ignore_sticky_hold=will_faint):
        return HandlerReturn(value=value)
    # attackerがきのみを得たので効果を発動して消費する
    battle.item_manager.force_trigger_berry(attacker, track_loss=False)
    return HandlerReturn(value=value)
```

- イベント: `Event.ON_MODIFY_MOVE_DAMAGE`（ダメージ量確定・HP反映前の処理。
  .internal/spec/turn.md の `## Event.ON_MODIFY_DAMAGE` 参照）。当初 `Event.ON_HIT`
  で実装していたが、`Event.ON_HIT` は `battle.modify_hp` によるダメージ反映
  （`Event.ON_HP_CHANGED` の同期発火を含む）より後に発火するため、被弾側自身の
  HP閾値きのみ（オボンのみ等）が先に自動発動してしまう不具合があった
  （下記「既知の制限（解消済み）」参照）。
- 優先度: 110。.internal/spec/turn.md の `Event.ON_MODIFY_DAMAGE` セクションに記載の
  「60 | みねうち・てかげん」「100 | がんじょう」「100 | きあいのタスキ」
  「100 | きあいのハチマキ」（いずれもHP1残し等の生存補正）より確実に後で実行する
  ことで、`value` にこれらの補正を反映した最終的な確定ダメージを受け取り、
  「このヒットでひんしになるか」を正しく判定できるようにする。
  「20 | みがわりに被弾」より後であることも必須（`ctx.substitute_damage` の
  設定を待つ必要があるため）。
- subject_spec: `MoveHandler` デフォルト（`attacker:self`）のまま。

### handlers/ability.py（修正）

```python
def ねんちゃく_prevent_item_change(battle: Battle, ctx: EventContext, value: bool) -> HandlerReturn:
    """ねんちゃく特性: 相手から受けるアイテム交換・奪取・除去を防ぐ。

    はたきおとすの威力補正判定（dry_run=True）では、除去自体は行われないため
    特性を発動・表示させず、威力補正の対象として扱う（除去は別途阻止される）。
    ignore_sticky_hold（むしくい・ついばむが対象をひんしにさせた場合等、
    第五世代以降の仕様）が指定されている場合も阻止しない。
    """
    if ctx.dry_run or ctx.ignore_sticky_hold:
        return HandlerReturn(value=value)
    if ctx.source != ctx.target:
        _announce_ability_triggered(battle, ctx.target)
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

## 既知の制限（解消済み）

一次情報には「チイラのみなどのピンチきのみ、オボンのみなどの回復きのみ、アッキのみは、
消費される前に食べることができる」とある。これは、相手自身のHP閾値ベースのきのみ
（例: オボンのみ）が自動発動するより先に、むしくいの奪取が優先されるべきという仕様である。

当初の実装（`Event.ON_HIT` への登録）では、HP閾値ベースのきのみの発動チェック
（`Event.ON_HP_CHANGED`）が `core/status_manager.py` の `modify_hp` 内でダメージ適用と
同時（同期的）に行われ、これが `Event.ON_HIT` より前に発生するため、むしくいの命中で
相手のHPがちょうど閾値を下回った場合に相手自身のオボンのみ等が自動的に先に発動・消費
されてしまい、むしくいは奪えないという不具合があった（`tests/abilities/test_ability_ha.py::test_はんすう_むしくいで奪ったきのみも対象になる`
で顕在化・実機でも再現確認済み）。

この不具合を、むしくいの奪取ハンドラ（`むしくい_steal_and_use_berry`）の登録先を
`Event.ON_HIT` から `Event.ON_MODIFY_MOVE_DAMAGE`（ダメージ量確定後・HP反映前、
がんじょう等の生存補正より後のpriority=110）へ移す形で解消した。この位置は
`battle.modify_hp` の呼び出し（＝`Event.ON_HP_CHANGED` の同期発火）より確実に前になる
ため、被弾側自身のHP閾値きのみより先にむしくいの奪取が成立する。

これに伴い、「この技で対象をひんしにさせた場合」の判定（ねんちゃく無視の条件）も
`defender.fainted`（HP反映後に確定）から `value >= defender.hp`（HP反映前に、
がんじょう等の生存補正を経た確定ダメージが現在HP以上かどうかで予測）に変更した
（上記「修正2」参照）。

## 注意点・エッジケース（review-test 向け）

1. **きのみを持たない場合**: 追加効果は発生しない（既存テストで確認済み）。
2. **きのみ以外のアイテム**: 奪取は発生しない（既存テストで確認済み）。
3. **タンガのみ等ダメージ計算前消費型のきのみ**: ダメージ計算時に先に消費されるため、
   命中後の奪取処理では既にきのみを失っており不発になる（既存テストで確認済み）。
4. **ねんちゃく（対象が生存する場合）**: 奪取自体が失敗し attacker は効果を受けない
   （既存テストで確認済み、修正後も変わらず維持）。
5. **ねんちゃく（対象をこの技でひんしにさせた場合）**: 奪取が成功し attacker が
   効果を受ける（今回修正・新規テスト追加）。
6. **みがわり**: 命中してもみがわりに防がれた場合はきのみを食べられない
   （今回修正・新規テスト追加）。
7. **被弾側自身のHP閾値きのみ（オボンのみ等）との競合**: むしくいの奪取が
   被弾側自身の`Event.ON_HP_CHANGED`ベースの自動発動より先に成立する
   （`Event.ON_MODIFY_MOVE_DAMAGE`への移行で解消。
   `tests/abilities/test_ability_ha.py::test_はんすう_むしくいで奪ったきのみも対象になる`
   で確認済み）。
8. **がんじょう・きあいのタスキ等との併用**: これらのHP1残し補正
   （`Event.ON_MODIFY_MOVE_DAMAGE` priority=100）はむしくいの奪取処理（priority=110）
   より先に確定するため、生存した場合に誤って「ひんしにさせた」と判定してねんちゃくを
   無視することはない。
