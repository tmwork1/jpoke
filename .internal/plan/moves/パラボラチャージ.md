# 計画書: パラボラチャージ

参照: .internal/spec/moves/パラボラチャージ.md
参照: .internal/plan/moves/HP吸収技.md（HP吸収技グループの共通計画書。本技もこのグループに属する）

## 仕様要約

* タイプ: でんき / 分類: 特殊 / 威力: 65 / 命中: 100 / PP: 20 / 対象: defender / 優先度: 0 / 接触: なし
* 相手に与えたダメージの半分だけ、自分のHPを回復する（追加効果なし）。

## 実装方針

既存の `_drain_hp`（`src/jpoke/handlers/move_attack.py`）ヘルパーをそのまま再利用する。
`heal_ratio=0.5` で、端数は `round_half_up`・最低1回復という共通仕様（ギガドレイン等の他の
HP吸収技と同一）に従うため、専用の丸め処理は不要。

## ハンドラ構成

| イベント | Priority | subject_spec | 役割 |
|---------|---------|--------------|------|
| ON_HIT | 20 | attacker:self | 与ダメ×0.5 を四捨五入（最低1）で回復 |

Priority 根拠: `.internal/spec/turn.md` の `Event.ON_HIT` テーブル「priority=20: HP吸収技による回復」
（ギガドレイン・すいとる等、同種のHP吸収技ハンドラと同じ priority を用いる）。

## handlers/move_attack.py

```python
def パラボラチャージ_drain(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """パラボラチャージの回復量を計算する。"""
    _drain_hp(battle, ctx, value, heal_ratio=0.5)
    return HandlerReturn(value=value)
```

## data/moves/move_ha.py

```python
    "パラボラチャージ": MoveData(
        type="でんき",
        category="special",
        pp=20,
        power=65,
        accuracy=100,
        flags={"heal"},
        handlers={
            Event.ON_HIT: h.MoveHandler(ha.パラボラチャージ_drain, priority=20)
        }
    ),
```

## 注意点・エッジケース

* `_drain_hp` 内で `damage = damage or ctx.substitute_damage` を処理するため、みがわり破壊時も
  正しく計算される。
* `battle.modify_hp` 内部でHP最大値超えを防ぐ処理が保証されている。
* 相手の特性『ヘドロえき』は `hp_change_reason == "drain"` を条件とする汎用実装
  （`ヘドロえき_reverse_drain`）で自動的に対応済みのため、本技専用の実装は不要。
* アイテム『おおきなねっこ』も `Event.ON_CALC_DRAIN` の汎用実装で対応済みのため、本技専用の
  実装は不要。
* 『かいふくふうじ』状態では `flags={"heal"}` により選択不可となる（`かいふくふうじ_try_action`）。

## リーサル計算

* `.internal/progress/move.md` のリーサル実装・リーサルテスト列は n/a（対象外）。
  理由: 本技は攻撃側の回復技であり、`handlers/lethal.py` の対象（相手の被弾期待値分布）に
  影響する特殊な追加効果を持たないため、通常のダメージ計算のみでリーサル判定が成立する。
