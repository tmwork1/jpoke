# ボディプレス 実装計画書

参照: .internal/spec/moves/ボディプレス.md

## 現状

* `src/jpoke/data/moves/move_ha.py` の `"ボディプレス"` エントリは以下の通り実装済み。
  - type="かくとう", category="physical", power=80, accuracy=100
  - flags={"contact"}
  - `handlers={}`（追加効果なし。ダメージ計算式自体が特殊なため専用ハンドラは不要）
* 攻撃力の入れ替え（自分の『こうげき』ではなく自分自身の『ぼうぎょ』実数値・ランク補正を使う）は
  `src/jpoke/core/damage.py` の `_calc_final_attack` 内で `move.name == 'ボディプレス'` により
  ハードコードされている。この分岐の後は通常の物理技と同じ `ON_CALC_ATK_RANK_MODIFIER` /
  `ON_CALC_ATK_MODIFIER` フローに乗るため、アイテム・特性・状態異常の補正（はりきり・
  こだわりハチマキ・スロースタート・わざわいのおふだ・やけど等）は使用者側の状態を正しく参照する。
  追加のハンドラ実装は不要。

## 発見した不具合と修正

### PP 値の不一致（実装バグ）

* `data/moves/move_ha.py` の PP が `10`（Gen9本家基準）になっていたが、
  `src/jpoke/data/ps-champ-ja/moves.json` では `12` と記載されている
  （チャンピオンズでは全技PPが8/12/16/20の4段階に統一されているため）。
  `.internal/progress/move.md` の PP 列はすでに `12` で記載されており、実装側が移行漏れだった。
  → `pp=12` に修正。

### パラドックス特性（クォークチャージ/こだいかっせい）との相互作用

* `src/jpoke/handlers/ability_paradox.py` の `apply_atk_modifier` は、イカサマとの相互作用
  レビュー（`.internal/plan/moves/イカサマ.md` 参照）で既に技名分岐を撤廃済みで、常に
  `ctx.attacker.paradox_boost_stat` を技分類（物理/特殊）で判定するのみになっている。
  これはボディプレスにも正しく当てはまる（使用者自身の『こうげき』スロットが強化対象のときのみ
  1.3倍補正が乗り、『ぼうぎょ』が強化対象でも乗らない）。修正不要、現状維持。

### ワンダールームとの相互作用（スコープ外）

* `.internal/spec/fields/ワンダールーム.md` の「範囲外（現行実装非対象）」に、実数値参照を入れ替える
  技・特性との複雑な相互作用は現行実装のスコープ外と明記されている。ボディプレス（自分の
  『ぼうぎょ』を攻撃として使う技）とワンダールーム（防御側の実数値参照を入れ替えるフィールド）の
  組み合わせも同種の複雑な相互作用のため、本タスクでは実装しない。
  仕様書にのみ挙動を記載し、実装は見送る。

## リーサル計算

* `.internal/progress/move.md` の「リーサル実装」「リーサルテスト」列は `n/a`。対象外
  （攻撃力の参照元入れ替えはダメージ計算内で完結し、専用のリーサルハンドラを要しない）。
