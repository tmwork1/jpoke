# こごえるせかい 実装計画書

## 仕様要約（.internal/spec/moves/こごえるせかい.md 参照）

* タイプ: こおり / 分類: 特殊 / 威力: 65 / 命中: 95 / PP: 10
* 対象: 相手全員（シングルバトルでは相手1体） / 優先度: 0 / 非接触
* 効果: 追加効果として100%の確率で相手の「すばやさ」を1段階下げる。

## 現状の実装状況

`src/jpoke/data/move.py` に登録済み。ただし `flags={"secondary_effect"}` が
欠落していたため、レビューで追加した（`ちからずく` によるすばやさ低下無効化・威力1.3倍が
発動しないバグがあった。同じ「100%確率ですばやさ1段階低下」の `がんせきふうじ` /
`こごえるかぜ` は既に `flags={"secondary_effect"}` を持っており、本技も揃える）。

```python
"こごえるせかい": MoveData(
    type="こおり",
    category="special",
    pp=10,
    power=65,
    accuracy=95,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.こごえるせかい_lower_defender_spd,
        )
    }
),
```

## ハンドラ構成

`handlers/move_attack.py` に既存実装済み。

```python
def こごえるせかい_lower_defender_spd(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spe": -1})
```

`modify_defender_stats` は `chance` 省略時デフォルト1（100%）で `ctx.defender` の
「すばやさ」を1段階下げる共通ヘルパー（`handlers/move.py`）。仕様の「100%の確率で
すばやさ1段階ダウン」と一致する。`こごえるかぜ_lower_defender_spd` と全く同じ実装パターン
（威力・命中率のみ異なる別技）であり、`subject_spec` も `attacker:self` / `defender:self`
のロールと一致した `AttackContext` を受け取る形になっている。

## Priority 根拠

`Event.ON_DAMAGE_HIT` は `.internal/spec/turn.md` に未掲載（ダメージ計算内部のイベント）。
同種の追加効果ハンドラ（`こごえるかぜ_lower_defender_spd` 等、能力ランク低下のみを行う
`ON_DAMAGE_HIT` ハンドラ群）はいずれも priority を指定していない（デフォルト値）ため、
本技も同様にデフォルトのままとする。

## 注意点・エッジケース（review-test 向け）

1. **ダメージ確認**
   - 通常のダメージ計算が行われることを確認する。
2. **追加効果の確認**
   - 100%の確率で相手の「すばやさ」ランクが1段階下がることを確認する
     （確率技ではないため乱数固定は不要）。
3. **まもるとの相互作用**
   - 「まもる」で防がれた場合、ダメージも追加効果も発生しないことを確認する。
