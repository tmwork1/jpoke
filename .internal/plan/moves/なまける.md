## Priority 根拠

`.internal/spec/turn.md` に `ON_STATUS_HIT` 内の自己回復技の記載なし。
既存の `じこさいせい_heal_self` が priority=100 (デフォルト) で登録されており、完全に同じ処理のため 100 を使用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: `なきごえ_modify_defender_stats` の後（「なまける」は「なきごえ」の後、「なみだめ」の前）

```python
def なまける_heal_self(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """なまける: 最大HPの1/2を回復する。"""
    mon = ctx.attacker
    if mon.hp == mon.max_hp:
        return HandlerReturn(value=False, stop_event=True)
    battle.modify_hp(mon, r=1/2)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/moves/move_na.py`

`"なまける"` エントリに `target="self"` を明示し、`pp` を championsルール準拠の 8 に修正する:

```python
    "なまける": MoveData(
        type="ノーマル",
        category="status",
        pp=8,
        target="self",
        flags={"heal"},
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.なまける_heal_self,
            ),
        }
    ),
```

## 注意点・エッジケース

- `じこさいせい` と全く同じ処理。`labels=["heal"]` はエントリに既記載のため変更不要
- HP満タン時の失敗チェック（`mon.hp == mon.max_hp` で早期リターン）は `turn.md` の `ON_APPLY_MOVE` Priority 100「自己回復技: HP満タン」で管理される場合もあるが、既存の `じこさいせい_heal_self` がハンドラ内でもチェックしているため同パターンに従う
- `かいふくふうじ` による使用不可チェックは `turn.md` Priority 100「かいふくふうじで技が出せない」でフレームワーク側が処理するため、ハンドラ内不要
- `target="self"` を明示しないと `MoveData.target` のデフォルト値 `"foe"` になり、`まもる`・`マジックコート` の判定が `move.target == "foe"` を前提とするため誤って防がれる/跳ね返される不具合になる（`じこさいせい` の過去レビューで判明した既知パターン）
- `pp` は Gen9本家基準（5）ではなく `src/jpoke/data/ps-champ-ja/moves.json` 準拠の 8 を使用する
- 回復量の端数処理（一次情報は第五世代以降切り上げ）について、本実装は `じこさいせい` 等と同様に切り捨て（`r=1/2`）のままとしている。詳細は `.internal/spec/moves/なまける.md` の備考を参照
