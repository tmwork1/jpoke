# ホイールスピン 実装計画書（再レビュー）

## 仕様要約

`.internal/spec/moves/ホイールスピン.md` 参照。

* タイプ: はがね / 分類: 物理 / 威力: 100 / 命中率: 100 / PP: 5（`src/jpoke/data/ps-champ-ja/moves.json`
  に記載なし＝チャンピオンズ固有の上書きなし。一次情報どおり5） / 対象: 相手1体
* 直接攻撃（`contact`）。
* 技が成功すると、自分の『すばやさ』ランクが2段階下がる（確定効果）。

## 既存実装の確認結果と問題点

`src/jpoke/data/moves/move_ha.py` の `ホイールスピン` エントリ、
`src/jpoke/handlers/move_attack.py` の `ホイールスピン_sharply_lower_attacker_spe` は
以下の点で仕様と一致していた。

* `type="はがね"`、`category="physical"`、`pp=5`、`power=100`、`accuracy=100` は
  一次情報と一致しており妥当。
* `flags={"contact", "secondary_effect"}` のうち `secondary_effect` が誤り。
  `.internal/wiki/abilities/ちからずく.html` の一次情報によれば、ちからずくの対象技は
  「相手のランク補正を下げる技」「自分のランク補正を上げる技」等に限られ、
  「自分のランク補正を下げる技」は対象外。既存実装の `アームハンマー`・
  `いじげんラッシュ`・`インファイト`（いずれも `.internal/spec/moves/` に個別仕様書あり）
  でも同様の判断で `secondary_effect` を付与していない。ホイールスピンのすばやさ
  ダウンも自分自身への確定効果であるため、`secondary_effect` フラグを削除する。

```python
"ホイールスピン": MoveData(
    type="はがね",
    category="physical",
    pp=5,
    power=100,
    accuracy=100,
    flags={"contact"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.ホイールスピン_sharply_lower_attacker_spe,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 命中後、自分の『すばやさ』ランクを2段階下げる（確定効果） |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def ホイールスピン_sharply_lower_attacker_spe(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"spe": -2})
```

`modify_attacker_stats`（`handlers/move.py`）の `chance` 既定値は1のため常に発動する。
自分自身への確定効果であり、`resolve_secondary_chance` 経由の「追加効果」判定
（ちからずく等）の対象にはならない（`flags` に `secondary_effect` を付与しない）。

## Priority 根拠

自分自身への確定ランク低下は既存の同種ハンドラ（アームハンマー・いじげんラッシュ・
インファイト等）と同じ `ON_HIT` / priority 100（デフォルト）のパターンに従う。
`.internal/spec/turn.md` の `Event.ON_HIT` セクションに記載の「自分のランクが下がる反動」
（概念上の優先度10）はイベント内の処理グループを示すものであり、`MoveHandler` の
`priority` 引数はイベント内の個別順序を指定するものであるため、既存パターン
（priority 引数省略＝100）に合わせて変更なし。

## 注意点・エッジケース（review-test 向け）

1. **ちからずく等「追加効果」系特性との非該当**
   - 自分自身への確定ランク低下は「追加効果」ではないため、ちからずくの対象にならない。
     `flags` から `secondary_effect` を削除するのが正しい。
2. **まもる貫通なし**
   - 通常の攻撃技であり、`まもる` 等の対象になる。個別のガードは不要
     （`unprotectable` フラグ等は付与しない）。
3. **おうじゃのしるし・するどいキバとの相互作用**
   - `handlers/item.py` の `flinch_on_hit_10pct` は `ctx.move.is_attack` かつ
     `_INNATE_FLINCH_MOVES` に含まれない技であれば汎用的にひるみ判定を行うため、
     `secondary_effect` フラグの有無に関係なく発動する。追加対応は不要。
4. **テスト方針**
   - 100%固定効果のため `secondary_chance` の調整は不要。命中後に自分の
     『すばやさ』ランクが2段階下がることを確認する通常テストは既存
     （`test_ホイールスピン_素早さ2段階低下が発動する`）で充足している。
   - `secondary_effect` フラグ削除の回帰確認として、ちからずく所持時も威力上昇が
     発生しないことを確認するテストを追加する（`アームハンマー`・`インファイト`の
     既存パターンに倣う）。
