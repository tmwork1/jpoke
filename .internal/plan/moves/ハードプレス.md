# 計画書: ハードプレス

## 仕様要約

- はがねタイプ / 物理 / 威力可変（1〜100） / 命中100 / PP12（`src/jpoke/data/ps-champ-ja/moves.json`
  基準。Gen9本家PP10 → Championsは8/12/16/20の4段階に統一されており、base PP10は12に変換される）
  / 対象: 相手1体選択 / 優先度0 / 接触あり
- 対象（相手）の残りHPが多いほど威力が高くなる。
- 威力 = max(1, floor(100 × 対象の残りHP ÷ 対象の最大HP))
  - 端数は切り捨て（`.internal/spec/moves/ハードプレス.md` 参照。にぎりつぶすと異なり
    五捨五超入ではなく単純floorで確定する）。
- 追加効果なし。まもるで防がれる。おうじゃのしるし発動（第五世代以降）。マジックコート対象外。

## ハンドラ構成

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| ハードプレス | `ON_CALC_POWER_MODIFIER` | 100（デフォルト） | `attacker:self` | 対象の残りHP比率から威力を決定 |

## Priority 根拠

`.internal/spec/turn.md` に `ON_CALC_POWER_MODIFIER` イベントの記載はない（ダメージ計算内部イベントのため）。
既存の同種ハンドラ（`にぎりつぶす_calc_power`、`しおふき_calc_power`、`ドラゴンエナジー_calc_power` 等）が
全て priority=100（デフォルト）を使用しているため、同様に **priority=100** を採用する。
`MoveHandler` のデフォルト priority は 100、subject_spec は `"attacker:self"` であるため、引数省略で問題ない。

## 実装箇所

### handlers/move_attack.py

```python
def ハードプレス_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """ハードプレス: 対象の残りHP / 最大HP の比率で威力を決定する（最大100）。

    威力 = max(1, floor(100 × 現在HP / 最大HP))
    """
    power = max(1, int(100 * ctx.defender.hp / ctx.defender.max_hp))
    return HandlerReturn(value=power * 4096)
```

`power=1` をプレースホルダとして `data/moves/move_ha.py` に設定し、`ON_CALC_POWER_MODIFIER` で
`value` を `target_power * 4096` に差し替えることで、`damage_calculator._calc_final_power()`
の `final_power = round_half_down(1 * power_modifier / 4096) = target_power` として
最終威力を確定させる（`.internal/plan/moves/攻撃技_威力変動系.md` の実装方式と同一パターン）。

実装は既存コードに存在し、仕様と一致しているため変更不要（レビュー確認済み）。

### data/moves/move_ha.py

```python
"ハードプレス": MoveData(
    type="はがね",
    category="physical",
    pp=12,
    power=1,
    accuracy=100,
    flags={"contact"},
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
            ha.ハードプレス_calc_power,
        ),
    }
),
```

`pp` は本レビューで `10`（Gen9本家値の移行漏れ）から `12`（Champions値）に修正した。

## 注意点・エッジケース

- 相手HPが1のときも `max(1, ...)` により威力は最低1になる。
- 相手HPが満タンのとき威力は必ず100（端数なし）。
- `int()` によるPython切り捨ては非負値に対して `floor` と等価であるため、`floor(100 × HP / max_HP)`
  の実装として問題ない。
- リーサル計算（`core/lethal.py`）は代表HP値（分布中の最小値）を使って通常の
  `battle.calc_damages` を呼ぶ設計のため、この `ON_CALC_POWER_MODIFIER` ハンドラは
  リーサル計算時にもそのまま発火する。ただし複数のHP分布枝が存在する場合、各枝ごとの
  厳密な威力差は反映されない（`.internal/progress/move.md` のハードプレス行は
  `リーサル実装 n/a` / `リーサルテスト n/a` のまま、本タスクでは対象外）。

## テスト方針

`tests/moves_attack/test_move_ha.py` に既存の以下のテストが配置済み（五十音順）:

3. `test_ハードプレス_相手HP満タンのとき威力100`: 満タン時に威力が100になることを確認。
