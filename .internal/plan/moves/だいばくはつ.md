## 仕様要約

* タイプ: ノーマル / 分類: 物理 / 威力: 250 / 命中: 100 / PP: 5 / 対象: defender
* 使用前に自分の現在HPを全て消費する（`Event.ON_PAY_HP`）
* HP消費後にHPが0になっても技は発動する（`ON_PAY_HP` はヒット処理より前に発火されるため）
* 命中判定・タイプ相性・まもるにより攻撃が不発になっても、使用者は必ずひんしになる
* `flags={"explosion"}` を保有する（しめりけによる失敗判定用）

## 現状の実装状況

`data/move.py` に登録済み・`flags={"explosion"}`・`handlers/move_attack.py` に `だいばくはつ_pay_hp` 実装済み。
実装内容は本計画書の内容と一致しているため、修正不要（レビューのみ）。

### Priority 根拠

`.internal/spec/turn.md` の `Event.ON_PAY_HP` テーブルには以下の記載がある。

```
| Priority | 処理 |
|----------|------|
| - | じばく等のダメージ・瀕死判定(外れても必ず自滅する技のみ。...) |
```

「じばく等」として明記されており、本実装が対象（`だいばくはつ` も同種の自滅技）。priority は
未指定（デフォルト値）で実装する。既存の `いのちがけ_pay_hp` / `じばく_pay_hp` も priority 未指定で
実装されており、それと同じパターンを踏襲する。

## 実装箇所（既存実装のまま・変更なし）

`src/jpoke/handlers/move_attack.py`:

```python
def だいばくはつ_pay_hp(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """だいばくはつ: 使用前に現在HPを全消費する。"""
    battle.modify_hp(ctx.attacker, v=-ctx.attacker.hp, reason="self_cost", source=ctx.attacker)
    return HandlerReturn(value=value)
```

## リーサル計算について

`.internal/progress/move.md` の だいばくはつ 行は「リーサル実装」「リーサルテスト」ともに `n/a`。
使用者自身がひんしになる技であり、相手へのダメージ算出は通常の物理技と同一のため、専用のリーサル
ハンドラを追加する必要はない（対象外のまま据え置く）。

## 注意点・エッジケース（テストで確認済み）

* **技後にひんしになるか**: `ON_PAY_HP` でHP全消費するため、技使用後、即座にひんしになる。
  → `test_だいばくはつ_使用後に攻撃者がひんしになる` で確認済み。
* **HP0でも攻撃が発動するか**: `ON_PAY_HP` はヒット処理より前に発火される（`いのちがけ` と同一パターン）。
  → `test_だいばくはつ_HP消費後も攻撃が相手に届く` で確認済み。
* **しめりけによる失敗**: `flags={"explosion"}` を保有しているため、相手が「しめりけ」特性を持つ場合は
  `ON_TRY_MOVE_1` で失敗し、HPも消費されない。
  → `test_だいばくはつ_しめりけ持ちには技が失敗する` で確認済み。
* **ハンドラ側ガード不要**: `ON_PAY_HP` 発火時に使用者が生存していることはバトル側で保証されている
  （行動可能チェック済み）ため、`if not attacker.alive` 等の重複ガードは不要。
* **レッドカード等との相互作用**: 攻撃者が自滅技でひんしになった場合、レッドカードなど「相手を
  強制的に交代させるアイテム」は発動しない（`tests/test_item.py` の
  `test_レッドカード_自滅技でひんしになった攻撃者には発動しない` で確認済み）。
