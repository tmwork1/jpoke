# 計画書: じしん

## 仕様要約

- じめんタイプ / 物理 / 威力100 / 命中100 / PP10 / 対象: 自分以外の全員（シングルでは相手1体） / 優先度0 / 非接触
- 追加効果なし（通常攻撃のみ）
- 『あなをほる』状態の相手には命中し、ダメージが2倍になる
- 『グラスフィールド』の影響を受けている（接地している）相手には威力が半減する

## ハンドラ構成

- じしん自体には追加効果がないため、`data/move.py` の `じしん` エントリは `handlers={}` のみで
  基本ダメージ計算に委ねる。
- 『あなをほる』状態の相手への威力2倍は、`handlers/volatile.py` の `あなをほる_boost_power`
  （`Event.ON_CALC_POWER_MODIFIER`、`subject_spec="defender:self"`、対象は `あなをほる` volatile
  自体に登録済み）で処理される。`HIDDEN_MOVE_ALLOWED_MOVES["あなをほる"]` に `じしん` が含まれることで
  命中判定も回避されずに命中する。
- 『グラスフィールド』での威力半減は、`handlers/field.py` の `グラスフィールド_power_modifier`
  （`Event.ON_CALC_POWER_MODIFIER`、フィールド側に登録済み）で処理される。
- 上記2点はいずれも既存実装済みのため、じしん側で新規に実装するハンドラはない。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、priority の検討対象イベントは存在しない。
（`あなをほる_boost_power` および `グラスフィールド_power_modifier` は既存実装であり、それぞれ
`あなをほる` 状態・フィールドの計画書で priority 根拠が示されている）。

## 実装箇所

### data/move.py への更新

既存の `"じしん"` エントリは以下の通りで、仕様と一致している（変更不要）。

```python
    "じしん": MoveData(
        type="じめん",
        category="physical",
        pp=10,
        power=100,
        accuracy=100,
        handlers={},  # 追加効果なし
    ),
```

- `flags` を指定していないため `contact` フラグは付与されない → 非接触（仕様と一致）。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `critical_rank` 省略時のデフォルトは `0`（通常の急所判定）→ 仕様と一致。

## 注意点・エッジケース

- 『あなをほる』状態の相手への威力2倍・命中確定は `あなをほる` 状態側のハンドラで処理されるため、
  じしん固有のハンドラを重複実装しない。
- 『グラスフィールド』での威力半減も同様にフィールド側のハンドラで処理されるため、
  じしん固有のハンドラを重複実装しない。
- 追加効果がないため、まもる・おうじゃのしるし・オウムがえし等の一般的な技判定は
  フレームワーク側の共通処理（`handlers/move_attack.py` の基本ダメージ計算経路）で
  自動的に処理され、本技固有のハンドラ実装は不要。
