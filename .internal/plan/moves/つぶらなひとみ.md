# 実装計画書: つぶらなひとみ

## 仕様要約

- タイプ: フェアリー / カテゴリ: 変化 / PP: 20（`src/jpoke/data/ps-champ-ja/moves.json` 準拠） / 命中: 100 / 対象: 1 体選択
- 優先度: +1（必ず先制できる）
- 相手の「こうげき」ランクを 1 段階下げる

## 現状の実装状況

| 項目 | 状態 |

| `data/moves/move_ta.py` エントリ | 登録済み（priority=1、handlers あり） |
| `handlers/move_status.py` 関数 | 実装済み |

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（`MoveHandler` デフォルト） | 相手の「こうげき」を 1 段階下げる |

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_STATUS_HIT` には明示記載なし。同種ハンドラ `なきごえ_modify_defender_stats` と同じパターンを採用し、priority はデフォルト（100）とする。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「つ」行 — `ちょうはつ_apply` の後、`つぼをつく_modify_attacker_stats` の前（「つぶ」は「つぼ」の前）に挿入する。

```python
def つぶらなひとみ_modify_defender_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """つぶらなひとみの効果: 相手のこうげきを1段階下げる。"""
    return modify_defender_stats(battle, ctx, value, stats={"atk": -1})
```

### `src/jpoke/data/moves/move_ta.py`

`"つぶらなひとみ"` エントリに handlers を登録する。priority=1 は先制効果を表すためすでに設定済み。
PP は `src/jpoke/data/ps-champ-ja/moves.json` の値（20）に従う（本家 Gen9 の 30 ではない）。

```python
"つぶらなひとみ": MoveData(
    type="フェアリー",
    category="status",
    pp=20,
    accuracy=100,
    priority=1,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.つぶらなひとみ_modify_defender_stats,
        ),
    }
),
```

## 注意点・エッジケース

- 優先度 +1 はすでに `data/moves/move_ta.py` に設定済みのため、ハンドラ側では不要
- ランク下限チェック（-6 で失敗）は `modify_stats` → `modify_stat` 内部の `clamp_stats` が行うため、ハンドラ側では追加チェック不要
- しろいきり・クリアボディ等によるランク変化無効はイベント `ON_TRY_MOVE_2`・`ON_BEFORE_MODIFY_STAT` 側で処理されるため、ハンドラ側では不要
- 対象は相手のため `target` はデフォルト値 `"foe"` のままでよい（`target="self"` を設定してはいけない）。
  `Move.is_blocked_by_protect` / `Move.is_reflectable` が `target in ("foe", "foe_side")` を前提に
  判定するため、まもる／マジックコートで正しく防がれる／跳ね返される
