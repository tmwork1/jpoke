# ひのこ 実装計画書

参照: .internal/spec/moves/ひのこ.md

## 現状

* `src/jpoke/data/moves/move_ha.py` の `"ひのこ"` エントリは以下の通り実装済み。
  - type="ほのお", category="special", pp=25, power=40, accuracy=100
  - `Event.ON_DAMAGE_HIT`: `ひのこ_apply_burn_to_defender`（10%でやけど付与、`apply_ailment_to_defender` 利用）
* 上記のやけど付与は仕様と一致しており修正不要。
* 一方、`flags` に `secondary_effect` のみが設定されており `"thaw"` が欠落していた。
  ひのこはほのおタイプの攻撃技であるため、被弾した相手がこおり状態の場合は解凍する
  効果を持つ（`.internal/spec/ailments/こおり.md` 31-32行目）。この解凍処理は
  `src/jpoke/data/ailment.py` の `"こおり"` エントリに登録された
  `こおり_cure_by_thaw_move`（`Event.ON_DAMAGE_HIT`, subject_spec="defender:self"）が
  汎用的に処理するが、`ctx.move.has_flag("thaw")` の判定に依存するため、
  `flags` に `"thaw"` を追加しないと発動しない。本レビューで追加する。

## 追加実装

### data/moves/move_ha.py の変更

```python
"ひのこ": MoveData(
    type="ほのお",
    category="special",
    pp=25,
    power=40,
    accuracy=100,
    flags={"secondary_effect", "thaw"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ひのこ_apply_burn_to_defender,
        )
    }
),
```

`だいもんじ`・`かえんほうしゃ` 等、既存の同種特殊ほのお技と同じ扱い。
ひのこ自体は「使用者がこおり状態のとき無条件で解凍してから攻撃する」対象技
（かえんぐるま・かえんボール・せいなるほのお・もえつきる等）には含まれないため、
使用者側の解凍処理（`Event.ON_TRY_ACTION` の thaw_attacker ハンドラ）は不要。

### .internal/spec/moves/_move_labels.md の変更

`thaw` セクションに `ひのこ` を五十音順で追加する（ハイドロスチームとフレアドライブの間）。

### handlers/move_attack.py

変更不要。既存の `ひのこ_apply_burn_to_defender` をそのまま使用する。

## Priority の根拠

* `ON_DAMAGE_HIT` は追加効果・こおり解凍付与の標準イベントであり、他のやけど付与技
  （かえんほうしゃ・だいもんじ等）と同じ priority（デフォルト）を使用する。
  本技独自の priority 検討は不要。
