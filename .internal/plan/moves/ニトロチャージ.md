# ニトロチャージ 実装計画

参照: .internal/spec/moves/ニトロチャージ.md

## 現状（レビュー時点）

`src/jpoke/data/moves/move_na.py` に `MoveData` は登録済みで、追加効果（自分のすばやさ+1、
100%）を処理する `Event.ON_HIT` ハンドラ `ニトロチャージ_boost_attacker_spe` も実装済みだが、
以下の問題がある。

* `flags` に `"secondary_effect"` が設定されておらず、ちからずく（`ちからずく_boost` /
  `ちからずく_disable_secondary_effect`）の対象として扱われない
  （一次情報「特性ちからずくのポケモンが使用すると威力が1.3倍になるが、すばやさは
  上がらなくなる。」を満たせていない）。

## 対応内容

### 1. `data/moves/move_na.py`

* `flags={"contact"}` → `flags={"contact", "secondary_effect"}` に修正する。

### 2. `handlers/move_attack.py`

変更なし。既存の `ニトロチャージ_boost_attacker_spe` はそのまま維持する
（`オーラウイング_boost_attacker_spe` / `くさわけ_boost_attacker_spe` と同一パターン）。

```python
def ニトロチャージ_boost_attacker_spe(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"spe": 1})
```

## ハンドラ構成

| 技名 | イベント | priority | subject_spec | 役割 |
|---|---|---|---|---|
| ニトロチャージ | ON_HIT | 100（デフォルト） | attacker:self | S+1（100%、ちからずく対象） |

## Priority 根拠

* `Event.ON_HIT` は `.internal/spec/turn.md` の該当セクションに「自分のランクを上げる追加効果」が
  priority=10 と記載されている。ただし既存の同種実装（`オーラウイング_boost_attacker_spe`,
  `オーラぐるま_boost_attacker_spe`, `くさわけ_boost_attacker_spe`,
  `こうそくスピン_boost_attacker_spe` 等）はいずれも `MoveHandler` のデフォルト priority
  （未指定＝100）で登録されており、turn.md の priority=10 は ON_HIT イベント内での処理グループの
  優先度を示すものであるため、既存パターンに合わせてデフォルト priority のまま登録する
  （変更不要）。

## ちからずくとの相互作用

* `flags={"secondary_effect"}` を追加することで、既存の汎用処理
  `ちからずく_boost`（ON_CALC_POWER_MODIFIER, 威力1.3倍）と
  `ちからずく_disable_secondary_effect`（追加効果確率を0にする）が自動的に適用される。

## リーサル計算

`.internal/progress/move.md` の該当行「リーサル実装」列は元は `-` だったが、`くさわけ` /
`オーラぐるま` など同種の「自分のすばやさランクを上げるだけの追加効果を持つ攻撃技」は
いずれも `n/a` になっている。本技の追加効果も相手へのダメージ確定計算（HP分布）には
一切影響しないため、同様に対象外（`n/a`）として扱う（進捗表の記載を `n/a` に修正する）。

## テスト

`tests/moves_attack/test_move_na.py` に以下を追加する。

* 命中時、自分の『すばやさ』ランクが1段階上がること（100%）。
* 特性『ちからずく』を持つ場合、威力が1.3倍になり、すばやさランクは上がらないこと。
