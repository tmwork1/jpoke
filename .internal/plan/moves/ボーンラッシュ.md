# ボーンラッシュ 実装計画書

## 仕様要約（.internal/spec/moves/ボーンラッシュ.md 参照）

* タイプ: じめん / 分類: 物理 / 威力: 30（Champions基準、1ヒットあたり） / 命中: 90 / PP: 12
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果:
  * 2～5回連続で攻撃する連続攻撃技（命中判定は1発目のみ、各ヒット威力は固定30）。
  * 追加効果なし。

## 現状の実装状況

`src/jpoke/data/moves/move_ha.py` に既に登録済み。

```python
"ボーンラッシュ": MoveData(
    type="じめん",
    category="physical",
    pp=12,
    power=30,
    accuracy=90,
    multi_hit={
        "min": 2,
        "max": 5,
        "check_hit_each_time": False,
        "power_sequence": (),
    },
    handlers={},  # 追加効果なし
),
```

* `multi_hit.min=2`, `multi_hit.max=5` のため、`_resolve_hit_count`（`core/move_executor.py`）は
  2～5の乱数（37.5% / 37.5% / 12.5% / 12.5%）で回数を決定する。
* `flags` に `check_hit_each_time` を含まないため、命中判定は1発目のみ行われる
  （`スケイルショット` / `ミサイルばり` 等と同一パターン）。
* `power_sequence` が空タプルのため、`_resolve_hit_power` は各ヒットとも `move.power`（30）を返す。
* `flags` は空集合。接触フラグ（`contact`）を含まないため非接触技として扱われる
  （一次情報の「直接攻撃: ×」と一致）。

## ハンドラ構成

固有ハンドラなし（`handlers={}`）。可変回数連続ヒットの挙動は `core/move_executor.py` 側の
汎用機構（`multi_hit`）で処理されるため、`handlers/move_attack.py` への追加実装は不要。

## 注意点・エッジケース（review-test 向け）

1. **命中判定は1発目のみ**
   - `check_hit_each_time` フラグを持たないため、1発目が外れた場合は残りのヒットも発生しない。
2. **各ヒット威力は固定30**
   - `power_sequence` を使わず、威力上昇パターン（トリプルアクセル等）とは異なる。
3. **ヒット回数は2～5回の可変**
   - `_resolve_hit_count` が返す値は2以上5以下であることを確認する。
4. **PP・威力はChampions基準**
   - `MOVES["ボーンラッシュ"].pp == 12`、`MOVES["ボーンラッシュ"].power == 30` であることを確認する
     （Gen9本家基準のPP=10・威力=25ではない）。
5. **リーサル計算は対象外**
   - 連続攻撃技全般はリーサル計算の対象外（`.internal/progress/move.md` のリーサル実装列がn/a）。
