# 実装計画書: フルールカノン（再レビュー）

## 仕様要約

`.internal/spec/moves/フルールカノン.md` 参照。

* タイプ: フェアリー / 分類: 特殊 / 威力: 130 / 命中率: 90 /
  PP: 8（Champions。SVまでの本家基準は5） / 対象: 相手1体
* 直接攻撃ではない。命中して技が成功した後、確定で自分の『とくこう』ランクを2段階下げる
  （反動によるランク低下であり、追加効果ではない）。

## 既存実装の確認結果と問題点

`src/jpoke/handlers/move_attack.py` の `フルールカノン_sharply_lower_spa_C` が
`modify_attacker_stats(battle, ctx, value, stats={"spa": -2})`（`chance` 省略でデフォルト100%）
を実行しており、同系統の反動技である `りゅうせいぐん_sharply_lower_spa_C` /
`オーバーヒート_lower_attacker_spa` / `サイコブースト_sharply_lower_spa_C` と全く同じ実装
パターンになっている。`Event.ON_HIT` での登録・`h.MoveHandler` のデフォルト priority（100）も
同系統技と同一で問題ない。

一方、`src/jpoke/data/moves/move_ha.py` の `MoveData` エントリに以下の誤りがあった。

* `pp=5`（Gen9本家基準の値）になっていたが、同系統の反動技（りゅうせいぐん・オーバーヒート・
  リーフストーム・サイコブースト）がいずれも本家PP5→ChampionsPP8に統一されていることから、
  フルールカノンも `8` に修正する（`.internal/spec/moves/フルールカノン.md` 「基本情報」節に
  根拠を記載）。
* `lethal_handlers` が未登録だった。同系統技（りゅうせいぐん・オーバーヒート・
  ゴールドラッシュ・サイコブースト）はいずれも `lethal_handlers` に登録済みであり、
  `.internal/progress/move.md` のフルールカノン行の「リーサル実装」列が `n/a` になっていたのは
  実態と食い違う誤り（他の同系統技はすべて `x`）。新規実装する。

`power=130`・`accuracy=90` はすでに一次情報（ポケモンWiki）・Champions値と一致しており
問題ない。`flags` を持たない点も一次情報（直接攻撃×、追加効果なし＝ちからずく非対象）と
一致しており問題ない。

## リーサル計算

`.internal/progress/move.md` の該当行は「リーサル実装: n/a」となっていたが、同系統技
（りゅうせいぐん・オーバーヒート・ゴールドラッシュ・サイコブースト）がいずれも
`x`（実装済み）であることと矛盾するため、新規実装対象として扱う。
`handlers/lethal.py` に `りゅうせいぐん_lower_spa` / `オーバーヒート_lower_attacker_spa` /
`サイコブースト_lower_spa` と同一パターンで `フルールカノン_lower_spa` を新規実装する。

```python
def フルールカノン_lower_spa(battle: Battle, ctx: LethalContext, hp_dist: StateDist) -> StateDist:
    """フルールカノン: 命中後、攻撃側のとくこうを2段階下げる。"""
    ctx.attacker.rank["spa"] = clamp_stats(ctx.attacker.rank["spa"] - 2)
    return hp_dist
```

* イベント: `LethalEvent.ON_HIT`（りゅうせいぐん・オーバーヒート・サイコブーストと同一）
* subject: `"attacker"`
* `src/jpoke/data/moves/move_ha.py` の `フルールカノン` エントリに
  `lethal_handlers={LethalEvent.ON_HIT: LethalHandler(l.フルールカノン_lower_spa)}` を追加する。
