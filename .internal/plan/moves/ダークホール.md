# ダークホール 実装計画書

## 仕様要約

- 相手を **ねむり** 状態にする変化技（あくタイプ、命中率50、PP10）
- 第七世代以降、**ダークライ以外が使用すると必ず失敗する**（種族制限）
- マジックコートで跳ね返された場合は、跳ね返す側の種族制限は適用されない

## 実装方針

### ハンドラ構成

| イベント | Priority | 役割 |
|----------|----------|------|
| `ON_TRY_MOVE_1` | 30 | 種族チェック：使用者がダークライでなければ失敗 |
| `ON_STATUS_HIT` | 100（デフォルト） | 相手をねむり状態にする |

### Priority 根拠

- `ON_TRY_MOVE_1 priority=30`：`.internal/spec/turn.md` の ON_TRY_MOVE_1 一覧で、使用者自身の前提条件チェック
  （例:「とっておき: 未使用技がある」）が priority=30 に分類されている。ダークホールの種族チェックも
  使用者自身の状態のみで判定する前提条件チェックであるため同じ priority=30 を採用する。

### マジックコート反射との整合性

`core/move_executor.py` の反射処理（`ON_CHECK_REFLECT`）は `ON_TRY_MOVE_1` より後段で発火し、
反射時は `ctx.attacker` が跳ね返した側に差し替わるが、`ON_TRY_MOVE_1` は既に通過済みのため
再評価されない。したがって、ダークライが使用した際に種族チェックを1回だけ通過させれば、
その後マジックコートで跳ね返されても種族チェックが再度走ることはなく、
「跳ね返す側は種族制限を受けない」という仕様を追加コードなしで満たす。

### へんしん・イリュージョンとの整合性

種族チェックは `ctx.attacker.name`（`Pokemon.name` プロパティ、`self.data.name`）で判定する。

- へんしんでダークライに変身した場合、`self.data` がダークライのデータに置き換わるため
  `ctx.attacker.name == "ダークライ"` となり成功する（仕様どおり）。
- イリュージョンは見た目のみの偽装であり `self.data.name` は変化しないため、
  イリュージョンでダークライに化けているだけでは種族チェックを通過できない（仕様どおり）。

いずれも既存の `Pokemon.name` の実装のみで対応でき、ダークホール側の追加実装は不要。

### subject_spec

- `ON_TRY_MOVE_1`: `attacker:self`（`MoveHandler` デフォルト）
- `ON_STATUS_HIT`: `attacker:self`（`MoveHandler` デフォルト）

## 実装箇所

### handlers/move_status.py（五十音順: ダークホールは「た」行、たくわえる〜ちいさくなるの間）

```python
def ダークホール_check_species(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ダークホールの発動条件チェック。

    第七世代以降、ダークライ以外が使用すると必ず失敗する（種族制限）。
    マジックコートで跳ね返された場合はこのチェックを再度通らないため、
    跳ね返す側は種族制限を受けずに成功する。
    """
    if ctx.attacker.name != "ダークライ":
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name),
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def ダークホール_apply_sleep(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ダークホールの効果: 相手をねむり状態にする。"""
    return apply_ailment_to_defender(battle, ctx, value, ailment="ねむり")
```

### data/moves/move_ta.py（既存の「ダークホール」MoveData エントリーを修正）

"ダークホール": MoveData(

## 注意点・エッジケース（review-test 向け）

1. **本プロジェクトの `POKEDEX` にダークライが未収録**：現状、ダークホールは常に失敗する
   （`ctx.attacker.name` が `"ダークライ"` になり得ないため）。テストは非ダークライ使用時の失敗
   ケースのみ実装可能。ダークライ実装後に成功ケースのテストを追加すること。
2. **ものまね/へんしん経由の失敗**：ものまねでダークホールを技として覚えても、使用者がダークライでな
   ければ `ON_TRY_MOVE_1` で失敗する。既存実装のみで対応可能なため追加コード不要。
3. **オウムがえし等の強制技経由**：オウムがえし・ねこのて・まねっこ・ゆびをふるでダークホールが選択され
   た場合も、実際の使用者（`ctx.attacker`）がダークライでなければ失敗する。これらの技は通常の
   `move_executor` パイプラインを経由して技を実行するため、`ON_TRY_MOVE_1` が正しく発火し追加実装は
   不要。
