# 計画書: ドゲザン

## 仕様要約（.internal/spec/moves/ドゲザン.md 参照）

- あくタイプ / 物理 / 威力85 / 命中- (必中) / PP12 / 対象: 相手1体 / 優先度0 / 接触あり
- 追加効果なし。急所ランク+1（急所に当たりやすい）。

## 既存実装との差分

既存実装（`data/move.py` の `"ドゲザン"` エントリ）には以下の欠落がある。

1. **PPが誤り（10 → 12）**
   `src/jpoke/data/ps-champ-ja/moves.json` では `ドゲザン あく 物理 85 - 12 接○ 守○` となっており、
   PPは12が正。本家Gen9基準の10が移行漏れとして残っていたバグのため、12に修正する。

2. **`critical_rank=1` が未設定**
   一次情報（`.internal/wiki/moves/ドゲザン.html`）の「判定」欄で「急所: ○」となっており、
   急所に当たりやすい技である。同じ「切る技」の `つじぎり`（`critical_rank=1`）と同様の
   扱いとし、`critical_rank=1` を設定する。

3. **`accuracy` は未設定のままでよい**
   `MoveData.accuracy` のデフォルトは `None` であり、`core/move_executor.py` は
   `accuracy is None` を必中として扱う。ドゲザンは省略時点で既にデフォルトの `None` が
   適用されるため、動作上のバグではない。ただし他の必中技（`あてみなげ` `でんげきは`
   `スピードスター` 等）は明示的に `accuracy=None` を書くスタイルで統一されているため、
   可読性のため明示する。

## ハンドラ構成

専用ハンドラなし（`handlers={}`）。追加効果を持たない技のため、`critical_rank` と
`accuracy=None` の設定のみで既存の共通ロジック（`core/move_executor.py` の命中判定・
急所判定）が自動的に適用される。

### data/move.py（修正後）

```python
"ドゲザン": MoveData(
    type="あく",
    category="physical",
    pp=12,
    power=85,
    accuracy=None,
    critical_rank=1,
    flags={"contact", "slash"},
    handlers={},  # 追加効果なし
),
```

## 注意点・エッジケース

- 急所ランクは通常のダメージ計算の急所判定に組み込まれるため、まもる等で防がれた場合は
  ダメージ自体が発生せず急所判定も行われない（既存の共通ロジックに準拠）。
- 必中技のため、命中率・回避率ランクや回避率無視特性（ノーガード等）に関係なく必ず命中する。
- 追加効果を持たないため、ちからずくの影響を受けない（威力・効果とも変化しない）。
- ドドゲザンの専用技。
