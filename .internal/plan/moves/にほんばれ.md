# 実装計画書: にほんばれ

## 仕様要約

- タイプ: ほのお / カテゴリ: 変化（内部 category="status"） / PP: 8（champions/move_list.txt準拠。
  一次情報のPP5は不採用） / 対象: 全体の場（target="field"）
- 5ターンの間、天候を「はれ」にする
- ほのおタイプ技のダメージが 1.5 倍、みずタイプ技が半減（天候「はれ」自体の一般仕様。本技固有の実装は不要）
- こおり状態にならなくなる（同上）
- おおひでり・おおあめ等の永続天候が存在する場合は失敗する（turn.md: 「天気変更技: おおひでり等で失敗」priority 100）
- 既に「はれ」状態の場合も再発動にはならず失敗する（`WeatherManager.apply` が同名天候への再適用を拒否する）

## 現状の実装状況

| `handlers/move_status.py` 関数 `にほんばれ_activate_weather` | 実装済み |

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト、`AttackContext`） | 天候を「はれ」に変更 |

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「天候・フィールド: すでに同じ」「天気変更技: おおひでり等で失敗」がいずれも priority 100 として記載されている。同種ハンドラ `あまごい_activate_weather`・`すなあらし_activate_weather` が priority デフォルト（100）を使用しているため、同一パターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

関数はすでに実装済み。

```python
def にほんばれ_activate_weather(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return HandlerReturn(value=battle.weather_manager.apply("はれ", 5, source=ctx.attacker))
```

### `src/jpoke/data/moves/move_na.py`

```python
"にほんばれ": MoveData(
    type="ほのお",
    category="status",
    pp=8,
    target="field",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.にほんばれ_activate_weather,
        ),
    }
),
```

## 注意点・エッジケース

- `weather_manager.apply` が既存天候と同じ場合・おおひでり等の永続天候が存在する場合に失敗を返す
  （`WeatherManager.apply` 内部の `WEATHER_PRIORITY` 比較と `ExclusiveFieldManager.apply` の
  `current_name == name` チェックによる）。ハンドラ側での重複ガードは不要
- `あまごい` と完全に同じ登録パターン（`target="field"`）
- あついいわ所持時の持続ターン延長は item 側 `ON_MODIFY_DURATION` ハンドラが処理するため、
  本技側での対応は不要
