# 計画書: ロッククライム

## 仕様要約（.internal/spec/moves/ロッククライム.md 参照）

- ノーマルタイプ / 物理 / 威力90 / 命中85 / PP20 /
  対象: 1体選択 / 優先度0 / 接触あり（直接攻撃）
- 追加効果として、命中時に20%の確率で相手をこんらん状態にする。
- 追加効果に分類されるため、ちからずく（威力1.3倍・効果無効化）の対象技になる。
- `src/jpoke/data/ps-champ-ja/moves.json` には掲載なし（SVで使用不可の技）。威力・命中・PPは
  ポケモンWiki記載の値（第四世代以降変更なし）を採用する。

## 既存実装との差分

既存実装（`data/moves/move_ra.py` の `"ロッククライム"` エントリ、`handlers/move_attack.py` の
`ロッククライム_apply_confusion_to_defender`）は仕様と一致しており、差分なし。

## ハンドラ構成

### handlers/move_attack.py: `ロッククライム_apply_confusion_to_defender`（既存・変更なし）

```python
def ロッククライム_apply_confusion_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ロッククライムの追加効果: 20%の確率で相手をこんらん状態にする。"""
    return apply_confusion_to_defender(battle, ctx, value, chance=0.2)
```

- `apply_confusion_to_defender`（`handlers/move.py`）は内部で `battle.resolve_secondary_chance` を
  呼び出すため、`Event.ON_MODIFY_SECONDARY_CHANCE` 経由でちからずく（確率0化・威力1.3倍）の
  ハンドラが自動的に適用される（`secondary_effect` フラグが前提）。
- イベント: `Event.ON_DAMAGE_HIT`（ダメージが実際に適用された後に発動。
  .internal/spec/turn.md の `## Event.ON_DAMAGE` 参照）。
  ダメージが発生しなかった場合（無効化・まもる等）はイベント自体が発火しないため、
  追加効果も発生しない仕様を自然に満たす。
- 優先度: デフォルト100（`MoveHandler` のデフォルト値）。
- subject_spec: `MoveHandler` デフォルト（`attacker:self`）のまま。

### data/moves/move_ra.py（既存・変更なし）

```python
"ロッククライム": MoveData(
    type="ノーマル",
    category="physical",
    pp=20,
    power=90,
    accuracy=85,
    flags={"contact", "secondary_effect"},

```

## リーサル計算

対象外（`.internal/progress/move.md` の「リーサル実装」列が `n/a`。こんらん付与のみでHP分布に影響しない
追加効果のためリーサル計算ハンドラは不要）。

## 注意点・エッジケース

- ノーマルタイプの相手など、技自体が無効化された場合、`Event.ON_DAMAGE_HIT` 自体が発火しないため
  追加効果も発生しない。個別のガードは不要。
- まもるで防がれた場合も同様にダメージが発生せず `Event.ON_DAMAGE_HIT` が発火しないため、
  追加効果も発生しない。
- 既にこんらん状態の相手に対しては `volatile_manager.apply_confusion` 側で重複付与しない
  （`VolatileManager` の既存ロジックに準拠、本技側での重複ガードは不要）。
- おうじゃのしるし・するどいキバによる追加ひるみ効果は `_INNATE_FLINCH_MOVES`（`handlers/item.py`）
  に含まれないため通常通り適用される（本技固有のハンドラ実装は不要）。
