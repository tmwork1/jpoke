# うたかたのアリア 実装計画書

## 仕様要約（.internal/spec/moves/うたかたのアリア.md 参照）

* タイプ: みず / 分類: 特殊 / 威力: 90 / 命中: 100 / PP: 12（Champions基準。本家は10）
* 対象: 相手1体 / 優先度: 0 / 非接触 / 音のわざ
* 効果: この技でダメージを受けたポケモンの『やけど』状態を治す（追加効果）。やけど以外は治さない。
* ちからずく対象技（追加効果があるため）。りんぷん・おんみつマント所持の相手には発動しない。

## 実装状況（レビュー前）

`src/jpoke/data/move.py` に既に登録されていたが、以下の誤りがあった。

1. PP が 10 になっていた（Champions基準では 12 が正しい。`.internal/progress/move.md` / `src/jpoke/data/ps-champ-ja/moves.json` 参照）。
2. `flags` に `"secondary_effect"` が付いておらず、ちからずくによる威力上昇・追加効果無効化が発動しない状態だった。
3. `Event.ON_HIT` に登録されていた。`.internal/spec/turn.md` の `## Event.ON_DAMAGE`
   （実装上は `Event.ON_DAMAGE_HIT` に相当。`はたきおとす` の実装コメント参照）に
   「100 | うたかたのアリアでやけど回復」と明記されており、`Event.ON_DAMAGE_HIT` を使うべきところが誤っていた。
4. ハンドラが `battle.resolve_secondary_chance` を経由しておらず、ちからずく・りんぷん・おんみつマントによる
   無効化が一切反映されていなかった。
5. リーサル計算ハンドラが未実装だった（`.internal/progress/move.md` のリーサル実装列が `-`）。

レビューにより上記を修正する。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 追加効果として防御側のやけどを治す |

### handlers/move_attack.py

```python
def うたかたのアリア_cure_defender_burn(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """うたかたのアリア: 追加効果として、命中時に防御側のやけどを治す。

    やけど以外の状態異常は治さない。ちからずくで無効化され威力が上がる代わりに発動しなくなる。
    りんぷん・おんみつマントを持つ相手には発動しない（`resolve_secondary_chance` 経由で判定）。
    """
    chance = battle.resolve_secondary_chance(ctx, 1)
    if chance < 1 and battle.random.random() >= chance:
        return HandlerReturn(value=value)
    mon = ctx.defender
    if mon.ailment.name == "やけど":
        battle.ailment_manager.remove(mon)
    return HandlerReturn(value=value)
```

`handlers/move.py` の `apply_ailment_to_defender` と同じ「`resolve_secondary_chance` → `chance < 1` なら
`battle.random.random() >= chance` で確定的にスキップ」というガードパターンを踏襲する（今回はやけどの
「付与」ではなく「解除」なので専用の除去処理を直接書く）。`chance` が 1 のまま（ちからずく・りんぷん・
おんみつマントのいずれも無い通常時）は乱数を消費せず必ず発動する。

## Priority 根拠

`.internal/spec/turn.md` の `## Event.ON_DAMAGE` セクションに「100 | うたかたのアリアでやけど回復」と明記されている。
このセクションは実装上の `Event.ON_DAMAGE_HIT` に対応する（`data/move.py` の `はたきおとす` エントリのコメント
「.internal\spec\turn.md ON_DAMAGE: 「100 はたきおとす等のアイテム効果」...ON_DAMAGE_HIT を使用する」で明示的に
対応関係が確認できる）。優先度は同セクション記載の 100（`MoveHandler` のデフォルト）でよい。

`ON_DAMAGE_HIT` はダメージが実際に適用された後に発火するため、まもる等で技が無効化された場合や
ダメージ計算自体が行われなかった場合は発動しない、という仕様（一次情報の「技を無効化した相手のやけどは
治せない」）を自然に満たす。

同セクションの「150 レッドカードの発動・交代」より優先度が小さいため、だっしゅつボタン/レッドカードの
効果より先にやけど回復が発動する（一次情報の記載通り）。

## リーサル計算

`.internal/progress/move.md` のリーサル実装列が `-`（未実装）だったため新規実装する。

### handlers/lethal.py

```python
def うたかたのアリア_cure_defender_burn(battle: Battle, ctx: LethalContext, hp_dist: StateDist) -> StateDist:
    """うたかたのアリア: 命中後、追加効果有効時に防御側のやけど状態を治す。"""
    if ctx.move_secondary and ctx.defender.ailment.name == "やけど":
        from jpoke.model.ailment import Ailment
        ctx.defender.ailment = Ailment()
    return hp_dist
```

`れんごく_apply_やけど`（やけど付与の逆パターン）に倣い、`ctx.move_secondary` で追加効果の有効・無効を
切り替える。やけど付与ではなく解除なので `ctx.defender.ailment = Ailment()`（空＝状態異常なし）を代入する。

登録は `LethalEvent.ON_HIT`、`subject` は指定しない（`れんごく` と同様。技のハンドラは
`_get_move_handlers` で無条件に呼ばれるため `subject` によるフィルタリングは行われない）。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力が1.3倍になり、やけど回復が発動しなくなる
     （`ちからずく_boost` / `ちからずく_disable_secondary_effect` で汎用的に処理される）。
2. **りんぷん・おんみつマントとの相互作用**
   - 対象がりんぷんまたはおんみつマントを持つ場合、`resolve_secondary_chance` が確率を0にし、やけど回復が
     発動しない。本プロジェクトはシングルバトルのみのため、一次情報にある「複数体にダメージを与えた場合は
     治る」という多体戦特有の分岐は考慮不要（常に発動しない扱いでよい）。
3. **みがわり貫通**
   - 音のわざのため、みがわり状態の相手にも本体のやけどを治す効果が届く（フレームワーク側の
     `ctx.move.has_flag("sound")` 判定で汎用的に処理される）。
4. **まひ等、やけど以外の状態異常には無効**
   - `mon.ailment.name == "やけど"` の判定のみ行い、他の状態異常は治さない。既存テストで確認済み。
5. **使用者がひんしになった場合でも発動する（Champions仕様）**
   - とびだすなかみ/うのミサイル・レンブのみ・みちづれ等で使用者が先にひんしになった場合、本家シリーズ
     （SVまで）ではやけど回復が発動しないが、Championsでは発動する。攻撃側の生死をチェックするガードは
     追加しない。
6. **テスト方針**
   - 既存テスト（`test_うたかたのアリア_まひ状態の相手には影響しない` /
     `test_うたかたのアリア_やけど状態の相手のやけどが治る`）に加え、ちからずく・りんぷん・おんみつマントで
     やけど回復が発動しないことを確認するテストを追加する。
   - リーサル計算については `secondary=True/False` で `やけど` によるターン終了時ダメージの有無が変わることを
     確認するテストを追加する（`test_れんごく_やけど付与_secondary有り` と対になる形）。
