# 実装計画書: ゆきげしき

## 仕様要約

- タイプ: こおり / カテゴリ: 変化 / PP: 8（champions/move_list.txt準拠） / 対象: 全体の場（target="field"）
- 5ターンの間、天候を「ゆき」にする
- こおりタイプのポケモンの「ぼうぎょ」が 1.5 倍になる（`.internal/spec/fields/ゆき.md` の一般仕様。固有実装は不要）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 天候を「ゆき」に変更 |

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「天候・フィールド: すでに同じ」が priority 100 として記載されている。同種ハンドラ `あまごい_activate_weather`・`にほんばれ_activate_weather`・`すなあらし_activate_weather` が priority デフォルト（100）を使用しているため、同一パターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

関数はすでに実装済みのため、追加不要。

```python
def ゆきげしき_activate_weather(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return HandlerReturn(value=battle.weather_manager.apply("ゆき", 5, source=ctx.attacker))
```

### `src/jpoke/data/moves/move_ya.py`

既存の `"ゆきげしき"` エントリ（PP は champions 準拠の 8 に修正）。

```python
"ゆきげしき": MoveData(
    type="こおり",
    category="status",
    pp=8,
    target="field",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ゆきげしき_activate_weather,
        ),
    }
),
```

## 注意点・エッジケース

- ハンドラ関数・登録はすでに存在するが、PP が Gen9 本家基準の 10 のままだったため
  champions 準拠の 8 に修正する
- `weather_manager.apply` が既存天候と同じ場合の失敗処理は manager 側に委ねられる。ハンドラ側でのガードは不要
- `あまごい`・`にほんばれ`・`すなあらし` と同じ登録パターン
- つめたいいわによる持続ターン延長（5→8）は item 側の `ON_MODIFY_DURATION` ハンドラが処理するため、
  ゆきげしき固有の実装は不要
