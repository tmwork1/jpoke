# 計画書: かいふくふうじ

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 15 / 命中: 100 / 対象: 相手
- 相手に「かいふくふうじ」揮発性状態を5ターン付与する
- 揮発状態側（HP回復無効化・ターンカウント・交代解除）は `.internal/spec/volatiles/かいふくふうじ.md`
  および `.internal/plan/volatiles/かいふくふうじ.md` にて実装済み
- 相手がすでにかいふくふうじ状態の場合は失敗する（`volatile_manager.apply` が False を返す）
- 特性アロマベールに守られている相手には効果がない（フレームワーク側で阻止、本技側の実装は不要）
- まもる: ○（防がれる） / みがわり: ○（防がれる、非接触・非音技のため） / マジックコート: ○（第五世代以降反射される）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 | 実装場所 |
|----------|----------|--------------|------|----------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self` | 相手へかいふくふうじ揮発性状態を付与する（count=5） | `handlers/move_status.py` の `かいふくふうじ_apply` |

揮発状態側（付与後のHP回復無効化・ターン経過・解除）は既に `data/volatile.py` +
`handlers/volatile.py` に実装済みのため追加不要。

## Priority 根拠

`.internal/spec/turn.md` の `ON_STATUS_HIT` には内部 priority 表が存在しない。
既存の変化技ハンドラ（`かいでんぱ_modify_defender_stats`、`かなしばり_apply` 等）が
すべて priority 100（`MoveHandler` デフォルト）を使用しており、かいふくふうじも
同一パターンとして priority 100 を採用する。

## 実装箇所（実装済み・確認結果）

### `src/jpoke/handlers/move_status.py`

```python
def かいふくふうじ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """かいふくふうじの効果: 相手に「かいふくふうじ」状態を付与する（5 ターン）。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="かいふくふうじ", count=5)
```

`apply_volatile_to_defender`（`handlers/move.py`）は `battle.volatile_manager.apply` を
呼び出す共通ヘルパーであり、対象がすでに同じ揮発性状態を持つ場合は自動的に失敗する
（重複チェックの追加実装は不要）。

### `src/jpoke/data/moves/move_ka.py`

```python
"かいふくふうじ": MoveData(
    type="エスパー",
    category="status",
    pp=15,
    accuracy=100,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.かいふくふうじ_apply,
        ),
    }
),
```

## 注意点・エッジケース

- **すでにかいふくふうじ状態の場合の失敗**: `volatile_manager.apply` が既に揮発性状態が
  ある場合 `False` を返すため、ハンドラ側での重複チェックは不要
- **アロマベール無効化**: `ON_BEFORE_APPLY_VOLATILE` でアロマベール側のハンドラが
  かいふくふうじの付与を阻止する（`tests/abilities/test_ability__group.py` の
  `test_揮発状態耐性` で汎用的にテスト済み）ため、技側の実装は不要
- **みがわり貫通なし**: サイコノイズ（音技）と異なり、かいふくふうじ自体は音技フラグを
  持たないためみがわりを貫通しない（フレームワークの共通処理）
- **対象範囲**: 本家は「相手全員」だが、シングルバトルのみの本プロジェクトでは
  `MoveData.target` のデフォルト `"foe"`（相手1体）のままで機能的に等価
