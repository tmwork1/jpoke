# ついばむ 実装計画書

## 仕様要約（.internal/spec/moves/ついばむ.md 参照）

* タイプ: ひこう / 分類: 物理 / 威力: 60 / 命中: 100 / PP: 20
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 相手がバトルに効果のあるきのみを持っていれば、命中後にそれを奪って自分が消費する
  （むしくいと全く同じ追加効果）。

`src/jpoke/data/moves/move_ta.py` に既に登録済み。むしくいと共通のハンドラ関数
`ha.むしくい_steal_and_use_berry` を `Event.ON_MODIFY_MOVE_DAMAGE`（priority=110）に
登録している（当初は `Event.ON_HIT` に登録していたが、むしくい側の不具合修正
[`.internal/plan/moves/むしくい.md`「既知の制限（解消済み）」参照] に伴い、共有ハンドラの
登録イベントを両技とも同時に変更した）。

```python
"ついばむ": MoveData(
    type="ひこう",
    category="physical",
    pp=20,
    power=60,
    accuracy=100,
    flags={"contact"},
    handlers={
```

内容は仕様書と一致しており、修正不要。

## ハンドラ構成

| Event | subject_spec | 関数 | 役割 |
|---|---|---|---|
| ON_MODIFY_MOVE_DAMAGE (priority=110) | attacker:self / defender:self | `むしくい_steal_and_use_berry` | defender がきのみを持っていれば奪取し、attacker が即座に消費する |

`むしくい_steal_and_use_berry`（`handlers/move_attack.py`）の内部処理:

1. `ctx.substitute_damage > 0` ならみがわりに防がれたとして何もしない
2. `defender.has_item()` かつ `defender.item.is_berry()` を確認（きのみでなければ何もしない）
3. `value >= defender.hp`（がんじょう等の生存補正を経た確定ダメージが現在HP以上か）で
   このヒットでひんしになるかを予測する
4. `battle.item_manager.take_item(defender, ignore_sticky_hold=will_faint)` で attacker に移す
   （`take_item` 内部で `can_change_item` を通してねんちゃく等のチェックを行う。
   奪取失敗時は何もしない）
5. `battle.item_manager.force_trigger_berry(attacker, track_loss=False)` で attacker に
   強制的にきのみ効果を発動・消費させる

**専用ハンドラの新規実装は不要**（むしくいの既存実装を流用）。

## 注意点・エッジケース（review-test 向け）

1. **きのみを持たない場合**: 追加効果は発生しない（むしくい側で既にテスト済みのパターン）。
2. **きのみ以外のアイテム**: 奪取は発生しない。
3. **タンガのみ等ダメージ計算前消費型のきのみ**: ダメージ計算時に先に消費されるため、
   奪取処理が発火する前に既にきのみを失っており不発になる。
4. **ねんちゃく（対象が生存する場合）**: defender がねんちゃくを持つ場合、奪取自体が
   失敗し attacker は効果を受けない。
5. **ねんちゃく（対象をこの技でひんしにさせた場合）**: 奪取が成功し attacker が効果を受ける。
6. **みがわり**: `ctx.substitute_damage > 0` の場合は何もしないため、みがわりに防がれた
   場合はきのみを食べられない。
7. **被弾側自身のHP閾値きのみ（オボンのみ等）との競合**: `Event.ON_MODIFY_MOVE_DAMAGE`
   （HP反映前）で奪取するため、被弾側自身の `Event.ON_HP_CHANGED` ベースの自動発動より
   先に奪取が成立する。

これらのエッジケースは `むしくい_steal_and_use_berry` を対象とした
`tests/moves_attack/test_move_ma.py` の むしくい 用テストで既に網羅されている
（同一関数を共有するため、ついばむ側は代表的な1ケースのみで十分）。
