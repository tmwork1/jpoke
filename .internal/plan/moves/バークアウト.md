# 実装計画: バークアウト

## 仕様サマリ

- タイプ: あく / 分類: 特殊 / 威力: 55 / 命中率: 95 / PP: 16（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）
- 対象: defender（原作は相手全員対象だが、本プロジェクトはシングルバトルのみのため単体扱い）
- 効果: 追加効果として、100%の確率で相手の『とくこう』ランクを1段階下げる
- flags: `sound`（音技・みがわり貫通）, `secondary_effect`（100%発動の追加効果を持つため、
  ちからずくの威力上昇・追加効果無効化の対象になる）
- まもる: ○ / マジックコート: ×（変化技ではないため対象外） / 直接攻撃: ×

参照: `.internal/spec/moves/バークアウト.md`、`.internal/spec/moves/_move_secondary_1.md`
（対象=defender、C-1=100%の技として掲載済み）。
同種の「攻撃技命中時に防御側ランクを100%下げる」技（Gのちから・ほのおのムチ・りんごさん等）は
`.internal/plan/moves/攻撃技_防御側ランク変化系.md` にまとめて計画済みで、バークアウトも同ファイルの
対象技一覧・実装状況表に含まれている（C-1・100%・`ON_HIT`系の登録済み技として記載）。
本ファイルはバークアウト単体のレビュー時に見つかった残課題（`secondary_effect` フラグ未付与・
PPの移行漏れ）の修正内容を個別に記録する。

## 対象イベント

`Event.ON_DAMAGE_HIT` — ダメージ適用後に発火する「追加効果発動」イベント。
`かみくだく`・`からみつく`・`がんせきふうじ` 等、同種の防御側ランク変化技がすべて
`Event.ON_DAMAGE_HIT` で実装済みであり、既存実装と統一する。

## Priority

`MoveHandler` のデフォルト priority=100 を使用（明示指定なし）。
既存の同種ハンドラ（`かみくだく_lower_defender_def` 等）もすべて priority 未指定で統一されている。

## 実装状況（レビュー時点）

### 既存実装（変更なし）

- `handlers/move_attack.py` の `バークアウト_lower_spa_C`（`modify_defender_stats` で
  `stats={"spa": -1}`, chance未指定＝デフォルト100%）
- `data/moves/move_ha.py` の `Event.ON_DAMAGE_HIT` 登録

### 今回のレビューで修正した点

1. **PP移行漏れ**: `pp=15`（Gen9本家値）になっていたが、`src/jpoke/data/ps-champ-ja/moves.json` では
   `16` であり、チャンピオンズは全技PPを8/12/16/20の4段階に統一しているため `pp=16` に修正。
2. **`secondary_effect` フラグ未付与**: `flags={"sound"}` のみで `secondary_effect` が
   付いていなかった。100%発動の追加効果（とくこう-1）を持つため、`ちからずく`（威力1.3倍・
   追加効果無効化）の対象になるべきだが、フラグ欠落によりこの相互作用が機能していなかった。
   `flags={"secondary_effect", "sound"}` に修正。
3. **リーサル計算は対象外（n/a）と判定**: 進捗表では `-`（未実装）表記だったため、当初は
   Gのちから・ほのおのムチ・りんごさん・ルミナコリジョン（いずれも防御側の100%ランク変化技）に
   倣って `handlers/lethal.py` にハンドラを追加しようとしたが、以下の理由により効果が
   まったく無いことが判明したため実装を見送り、`n/a` に修正した。
   - Gのちから／ほのおのムチは物理技で defender の『ぼうぎょ』を下げる、りんごさん／
     ルミナコリジョンは特殊技で defender の『とくぼう』を下げる、というように、
     いずれも自分の技のダメージ計算で参照される防御側ステータス（物理→ぼうぎょ、
     特殊→とくぼう）と一致しているため、後続ヒットのダメージが変化し `hp_dist` に
     影響する。
   - バークアウトは特殊技だが、下げるのは defender の『とくこう』（攻撃用ステータス）で
     あり、特殊技のダメージ計算が参照するのは defender の『とくぼう』であるため、
     `defender.rank["spa"]` を変化させても `battle.calc_damages` の結果は一切変わらない
     （`src/jpoke/core` 配下に defender の spa を参照するダメージ計算コードが存在しないことを
     確認済み）。
   - 同じ理由で `ひやみず`（特殊技で defender の『こうげき』を下げる）・`うらみつらみ`
     （特殊技で defender の『こうげき』を下げる）・`がんせきふうじ`（物理技で defender の
     『すばやさ』を下げる）も進捗表で `n/a` になっており、本ケースもこれらと同じ扱いが
     正しい。

## テスト方針

- 通常ダメージ確認（`accuracy=100` 固定）
- とくこう1段階低下（`secondary_chance=1.0`）
- `sound` フラグによるみがわり貫通確認（`ハイパーボイス` の既存テストと同パターン）
- `ちからずく` との相互作用確認（威力1.3倍・とくこう低下は発動しない。`がんせきふうじ` の
  既存テストと同パターン）
- リーサル計算は n/a のためテスト追加不要

## 注意事項

- かがくへんかガス/かたやぶり無効化チェックは書かない
- 原作は「相手全員」が対象の技だが、本プロジェクトの対象範囲（シングルバトルのみ）では
  単体（defender）として扱えば十分であり、複数対象向けの分岐は不要
