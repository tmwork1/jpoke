# いじげんホール 実装計画書

## 仕様要約（.internal/spec/moves/いじげんホール.md 参照）

* タイプ: エスパー / 分類: 特殊 / 威力: 80 / 命中: -（必中） / PP: 5
* 対象: 相手1体 / 優先度: 0 / 直接攻撃なし
* フラグ: 必中技、まもる貫通（unprotectable）、みがわり貫通（bypass_substitute）
* 効果: 対象がまもる系統の状態であれば、それを取り除いてから攻撃する。みがわり状態は無視して本体にダメージを与える（みがわり自体は解除しない）。追加効果なし。

## 現状の実装状況（レビュー時点）

`src/jpoke/data/move.py` に登録済みだったが、`flags={"unprotectable", "bypass_substitute"}` のみで
まもる系統の状態を解除する追加効果（`handlers`）が欠落していた。

```python
# 修正前
"いじげんホール": MoveData(
    type="エスパー",
    category="special",
    pp=5,
    power=80,
    accuracy=None,
    flags={"unprotectable", "bypass_substitute"},
),
```

`unprotectable` フラグにより「まもるで防がれない」動作自体は `model/move.py` の
`is_blocked_by_protect` で正しく処理され、`bypass_substitute` フラグにより
「みがわりを貫通して本体にダメージを与える」動作も `move_executor.py` の
`check_hit_substitute` で正しく処理されている。

しかし、`.internal/spec/volatiles/まもる.md` に明記されている
「フェイント / ゴーストダイブ / シャドーダイブ / いじげんホール / いじげんラッシュ の
ダメージを受けた場合もまもる状態が取り除かれる」という効果（＝まもる系揮発性状態そのものの解除）は
実装されていなかった。フェイント（`ha.フェイント_remove_protect`）には同等の実装が既にあるため、
同じパターンでいじげんホール専用のハンドラを追加する。

## ハンドラ構成

| Event | 関数 | subject_spec | 役割 |
|-------|------|--------------|------|
| `Event.ON_HIT` | `いじげんホール_remove_protect` | `attacker:self`（`MoveHandler`のためsubject_specは省略、`AttackContext`固定） | ダメージ発生後、相手のまもる系揮発性状態（まもる・みきり・トーチカ・キングシールド・ニードルガード・スレッドトラップ・かえんのまもり）を解除する |

既存の `フェイント_remove_protect`（`handlers/move_attack.py`）と全く同じロジック
（対象揮発性状態リスト `_FAINT_REMOVE_VOLATILES` を走査し、該当するものを
`battle.volatile_manager.remove()` で除去、`LogCode.VOLATILE_REMOVED` をログに残す）を用いる。
ログの `reason` のみ `"いじげんホール"` に差し替える。

## Priority 根拠

`.internal/spec/turn.md` の `Event.ON_HIT` 一覧には「まもる系統の状態解除」の行は明記されていない
（掲載されているのは自分ランク変化・HP吸収・ヘドロえき・コアパニッシャーのみ、いずれも priority 10〜100）。
既存の `フェイント_remove_protect` が `Handler` のデフォルト priority（100、`core/handler.py` 参照）で
登録されている前例に倣い、本技も同様にデフォルト priority（明示指定なし）で登録する。
ダメージ処理そのものは `ON_HIT` 発火前に完了しているため、他の `ON_HIT` ハンドラ
（HP吸収・ヘドロえき等）との priority 競合は発生しない。

## 実装箇所

### handlers/move_attack.py

```python
def いじげんホール_remove_protect(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """いじげんホールの追加効果: 相手のまもる系揮発性状態を解除する。"""
    defender = ctx.defender
    for volatile in _FAINT_REMOVE_VOLATILES:
        if defender.has_volatile(volatile):
            battle.volatile_manager.remove(defender, volatile)
            battle.add_event_log(
                defender,
                LogCode.VOLATILE_REMOVED,
                payload={"volatile": volatile, "reason": "いじげんホール"}
            )
    return HandlerReturn(value=value)
```

`_FAINT_REMOVE_VOLATILES` は既存のモジュールレベル定数（フェイント用に定義済み）をそのまま再利用する。

### data/move.py

```python
"いじげんホール": MoveData(
    type="エスパー",
    category="special",
    pp=5,
    power=80,
    accuracy=None,
    flags={"unprotectable", "bypass_substitute"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.いじげんホール_remove_protect,
        )
    }
),
```

## 注意点・エッジケース（review-test 向け）

1. **まもる状態を解除して攻撃する**
   - `volatile1={"まもる": 1}` を持つ相手に対して技を撃つと、ダメージを与えた上で
     `defender.has_volatile("まもる")` が `False` になることを確認する。
2. **まもる状態がない場合は通常攻撃する**
   - まもる系状態がない相手には通常通りダメージが入ることを確認する（既存テストで担保済み）。
3. **みがわり状態は解除しない**
   - `bypass_substitute` によりみがわりの背後の本体にダメージが入るが、みがわり自体の
     `hp` は変化しないことを確認する（既存テストで担保済み）。
4. **ニードルガード・トーチカ等の他のまもる系状態も解除できる**
   - フェイントの既存テストと同様のパターンで、他のまもる系揮発性状態も解除対象になることを
     必要に応じて確認する（本レビューでは「まもる」のみ追加テストとする。フェイント側で
     ニードルガード解除は既にテスト済みのため重複は避ける）。
