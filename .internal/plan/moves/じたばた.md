# 計画書: じたばた（再レビュー）

## 仕様要約

* 使用者の残りHPが少ないほど威力が大きくなる（`.internal/spec/moves/じたばた.md` 参照）。
* X = floor(残りHP × 48 / 最大HP) を算出し、X の範囲に応じて威力 20〜200 の6段階から決定する。
* きしかいせいと全く同じ威力計算式を使用する（タイプのみノーマルで異なる）。

## 既存実装の確認結果

`src/jpoke/handlers/move_attack.py` の `じたばた_calc_power` と共通関数 `_hp_low_to_power`
（`きしかいせい_calc_power` と共用）を確認した結果、仕様書の式・威力段階と完全に一致しており、
`src/jpoke/data/move.py` の `MoveData`（type/category/pp/power/accuracy/flags/handlers）も
一次情報と一致している。実装の修正は不要。

## ハンドラ構成（既存・変更なし）

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| じたばた | `Event.ON_CALC_POWER_MODIFIER` | デフォルト（100） | `attacker:self`（`MoveHandler` デフォルト） | 使用者の残りHP割合から技の実威力を算出して返す |

* `Event.ON_CALC_POWER_MODIFIER` は `.internal/spec/turn.md` に priority の記載がないため、
  既存の同種ハンドラ（`きしかいせい_calc_power` ほか、`ON_CALC_POWER_MODIFIER` を使う全ての
  `MoveHandler`）が明示的な priority 指定なし＝デフォルト値（100）で登録されている慣例に
  合わせ、今回もデフォルトのままとする。

```python
def じたばた_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """じたばた: 自分の残りHPが少ないほど威力が高くなる（きしかいせいと同計算）。"""
    return HandlerReturn(value=_hp_low_to_power(ctx.attacker.hp, ctx.attacker.max_hp) * 4096)
```

```python

    flags={"contact"},

        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(

        ),
    }
),
```

## リーサル計算

`.internal/progress/move.md` の該当行は「リーサル実装: n/a」「リーサルテスト: n/a」のため対象外。
理由: 威力は攻撃側自身の現在HPにのみ依存し、相手（`defender`）のHP分布 `StateDist` を
直接左右する要素ではないため、`lethal.py` 側の専用ハンドラは不要。

確認されていたが、`きしかいせい`（`test_move_ka.py`）と同水準の境界網羅（40/80/100/150）が
