# 計画書: だいちのはどう

## 仕様要約

- ノーマルタイプ・特殊・威力50・命中100・PP10
- `pulse`（波動）ラベル（メガランチャーで威力1.5倍）
- 使用者が**地面に接地している**とき（`not battle.query.is_floating(attacker)`）、
  フィールド状態に応じてタイプが変化し、威力が2倍になる

| フィールド | 変化後タイプ |
|------------|------------|
| エレキフィールド | でんき |
| グラスフィールド | くさ |
| ミストフィールド | フェアリー |
| サイコフィールド | エスパー |

- フィールドがないとき、または使用者が浮いているときはノーマルタイプのまま威力変化なし
- 対象が地面にいるかは関係なく、攻撃側の接地状態のみで判定する
- フェアリースキン・フリーズスキン等の「スキン系」特性によるタイプ変換は受けない
  （`ON_MODIFY_MOVE_TYPE` でだいちのはどう自身がタイプを上書きするため、スキン系より先に実行する必要はない。ただしスキン系はノーマルタイプにのみ作用するため、フィールドでタイプ変化した後のスキン系は適用されない）

## Priority 根拠

`ON_MODIFY_MOVE_TYPE` と `ON_CALC_POWER_MODIFIER` は `.internal/spec/turn.md` に個別行の記載がない（ダメージ計算内部のイベント）。

既存の類似ハンドラを根拠とする：

- `テラバースト_modify_move_type`（`data/move.py:4484`）: `MoveHandler(ha.テラバースト_modify_move_type)` → priority=100
- `オーラぐるま_check_move_type`（`data/move.py:441`）: `MoveHandler(ha.オーラぐるま_check_move_type)` → priority=100
- `テラバースト_stellar_power`（`data/move.py:4490`）: `MoveHandler(ha.テラバースト_stellar_power)` → priority=100

これらすべて MoveHandler のデフォルト priority=100 を使用している。だいちのはどうも同様に priority=100 とする。

## 実装箇所

### `src/jpoke/handlers/move_attack.py` に追加する関数

```python
def だいちのはどう_modify_move_type(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """だいちのはどう: 接地かつフィールドありのとき技タイプをフィールドに対応したタイプに変換する。"""
    if battle.query.is_floating(ctx.attacker):
        return HandlerReturn(value=value)
    terrain = battle.terrain.name
    type_map = {
        "エレキフィールド": "でんき",
        "グラスフィールド": "くさ",
        "ミストフィールド": "フェアリー",
        "サイコフィールド": "エスパー",
    }
    new_type = type_map.get(terrain)
    if new_type is not None:
        value = new_type
    return HandlerReturn(value=value)


def だいちのはどう_power_modifier(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """だいちのはどう: 接地かつフィールドありのとき威力を2倍にする。"""
    if battle.query.is_floating(ctx.attacker):
        return HandlerReturn(value=value)
    terrain = battle.terrain.name
    if terrain in {"エレキフィールド", "グラスフィールド", "ミストフィールド", "サイコフィールド"}:
        value = apply_fixed_modifier(value, 8192)  # ×2倍
    return HandlerReturn(value=value)
```

`apply_fixed_modifier` は `from jpoke.utils.math import apply_fixed_modifier` でインポート済みのものを使う（`move_attack.py` 冒頭の既存インポートを確認して追記すること）。

## 注意点・エッジケース（review-test 向け）

### 接地判定

- `battle.query.is_floating(ctx.attacker)` は `ON_CHECK_FLOATING` イベントを発火し、
  以下の全条件を考慮する：
  - ひこうタイプ（Pokemon.has_type で確認）
  - 特性ふゆう（`handlers/ability.py:ふゆう_float`）
  - でんじふゆう状態（`data/volatile.py:でんじふゆう_check_floating`）
  - じゅうりょく発動中は浮遊が打ち消される（`handlers/field.py:じゅうりょく_grounded`）
- ハンドラ内で直接タイプチェックは不要。`is_floating` が全て吸収している。

### フィールドとタイプブースト（二重効果）

- エレキフィールド時にタイプが「でんき」になると、同ターン中 `エレキフィールド_power_modifier`
  （`handlers/field.py:162`）が別途 `ON_CALC_POWER_MODIFIER` で発動し、さらに 1.3倍される。
  仕様書備考に記載あり：「フィールドの効果によりさらに威力が1.3倍される」。
  グラスフィールド（くさ技 1.3倍）、サイコフィールド（エスパー技 1.3倍）も同様。
  これはハンドラ間の連携によって自然に実現されるため、個別対応は不要。

### ミストフィールドのフェアリー技減衰との関係

- ミストフィールドは相手が接地している場合にドラゴン技を 0.5倍にするが、
  フェアリー技への弱体化効果はない。
  だいちのはどうがフェアリータイプになってもミストフィールドによる威力低下は発生しない。

### スキン系特性との排他

- フェアリースキン・フリーズスキン等はノーマルタイプの技にのみ作用する。
  フィールドありかつ接地時は `ON_MODIFY_MOVE_TYPE` でノーマル以外に書き換わるため、
  スキン系は実質的に無効になる（仕様通り）。
  ハンドラの実行順は `priority=100` で同一だが、MoveHandler が AbilityHandler より先に評価される場合のみ正常に動作する。
  実装環境でのハンドラ登録順（move → ability）を確認すること。

### そうでん状態との排他

- そうでん状態は `ON_MODIFY_MOVE_TYPE` で「でんき」に上書きする（`data/volatile.py:289`）。
  priority は `VolatileHandler` のデフォルト 100 であり、MoveHandler と同値。
  実行順が不定になる可能性がある。仕様書には「そうでんの効果を受けると
  いずれのフィールドでもでんきタイプになる」とあり、そうでん優先が正しい挙動。
  実装時に priority 調整または実行順の確認が必要。

### フィールドなしまたは浮遊時

- タイプ・威力ともにノーマル・50 のまま。ガード分岐はハンドラ内で完結する。

### ダイマックス技

- ダイアタックとして使う場合のタイプ変化はダイマックス未実装のため対象外。
