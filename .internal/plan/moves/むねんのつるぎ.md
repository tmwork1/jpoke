# むねんのつるぎ 実装計画書

参照: .internal/spec/moves/むねんのつるぎ.md

## 概要

物理・ほのおタイプのドレイン技。相手に与えたダメージの50%だけ自分のHPを回復する。
他の多くのドレイン技（ギガドレイン・すいとる・ドレインパンチ等）は共通処理 `_drain_hp`
（`handlers/move_attack.py`、`int(damage * heal_ratio)` で端数切り捨て）を再利用しているが、
むねんのつるぎは公式Wikiに明記された仕様上、**回復量の端数を切り上げる**点が異なるため、
`_drain_hp` を再利用せず専用ロジックで実装する。

`.internal/plan/moves/HP吸収技.md`（ドレイン技グループの計画書）では、むねんのつるぎも
他のドレイン技と同様に `_drain_hp(heal_ratio=0.5)` を使う想定になっていたが、
一次情報（Wiki）と照合した結果これは誤りであるため、本ファイルで個別に上書きする。

## MoveData（data/moves/move_ma.py）

既存定義（変更なし、確認済み）:

```python
"むねんのつるぎ": MoveData(
    type="ほのお",
    category="physical",
    pp=10,
    power=90,
    accuracy=100,
    flags={"contact", "slash", "heal"},
    handlers={
        Event.ON_HIT: h.MoveHandler(ha.むねんのつるぎ_drain, priority=20)
    }
),
```

* `flags` に `"contact"`（直接攻撃）・`"slash"`（きれあじで威力1.5倍）・`"heal"`
  （HPを吸収する技、かいふくふうじ／ヒーリングシフトの対象判定）を含める。
* `subject_spec` は `MoveHandler` のデフォルト `"attacker:self"` を使用。

## ハンドラ（handlers/move_attack.py）

```python
def むねんのつるぎ_drain(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """むねんのつるぎの回復量を計算する。

    他の多くのドレイン技（`_drain_hp`）は端数を切り捨てるが、むねんのつるぎは
    公式仕様上、回復量の端数を切り上げる点が異なるため専用に実装する。
    """
    damage = value or ctx.substitute_damage
    heal_amount = (damage + 1) // 2  # 端数切り上げ
    heal_amount = battle.events.emit(Event.ON_CALC_DRAIN, ctx, heal_amount)
    battle.modify_hp(ctx.attacker, v=heal_amount, reason="drain")
    return HandlerReturn(value=value)
```

* `damage`（与えたダメージ）が0の場合（みがわりに命中した場合）は `ctx.substitute_damage` を使う
  （`_drain_hp` と同じフォールバック）。
* `heal_amount = (damage + 1) // 2` で切り上げ計算する
  （`damage` は整数のため、端数は必ず0か0.5。`(damage + 1) // 2` は
  `ceil(damage / 2)` と等価。他のヘルパー `てっていこうせん_pay_hp` / `しっぽきり_apply`
  で使われている `(x + 1) // 2` の切り上げパターンと同じ書き方に揃える）。
* `Event.ON_CALC_DRAIN` を発火して `おおきなねっこ`（1.3倍）を適用する。
  発火順序は `_drain_hp` と同じく「切り上げ後の値」に対して倍率をかける。
* `ヘドロえき`（回復→反動ダメージへの変換）は `ON_MODIFY_HEAL` 経由で
  `battle.modify_hp(reason="drain")` を通す限り自動的に適用されるため、
  `_drain_hp` を使わなくても対応済み。

## イベント・priority

* `Event.ON_HIT`（ダメージ発生後の処理）: priority=20
  * `.internal/spec/turn.md` の `Event.ON_HIT` 表「20 | HP吸収技による回復」に対応。
  * 既存の全ドレイン技ハンドラ（ウッドホーン・ギガドレイン・すいとる・ドレインパンチ等）と同じ
    priority=20 で登録し、処理順の一貫性を保つ（既存実装のまま変更不要）。

## テスト方針

* `tests/moves_attack/test_move__group.py` の
  `test_drain技グループ_回復量が与ダメのheal_ratio倍になる` からは除外する
  （むねんのつるぎは切り上げのため、他の技と同じ `int(damage * heal_ratio)` の
  アサーションが常には成立しない）。
* `tests/moves_attack/test_move_ma.py` に以下を用意する:
  * `test_むねんのつるぎ_使用後に攻撃者のHPが回復する`（既存、基本動作）
  * `test_むねんのつるぎ_回復量の端数は切り上げになる`（新規、
    与ダメ101 → 回復51 になることを確認し、切り捨て(50)との違いを検証する）
