# であいがしら 実装計画書

## 仕様要約（.internal/spec/moves/であいがしら.md 参照）

* タイプ: むし / 分類: 物理 / 威力: 100（Champions。SVまでは90） / 命中: 100 / PP: 10
* 対象: 相手1体 / 優先度: +2 / 接触あり（直接攻撃）
* 効果: 場に出てから最初の行動として使用した場合のみ成功する。それ以外は失敗する。

## 現状の実装状況（修正後）

`src/jpoke/data/move.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。

```python
"であいがしら": MoveData(
```

### 修正前の問題点

* `power=90` のままだった（Champions での威力変更 90→100 が未反映）。
* `handlers={}`（追加効果なしのコメント）のままで、「場に出てから最初の行動でなければ
  失敗する」という本技の根幹となる成功条件が一切実装されていなかった。

## 「場に出てから最初の行動か」の判定方法

既存コードには「交代してから経過したターン数」や「最初の行動を済ませたか」を直接表す
状態がなかったため、`Pokemon` に新しいフラグ `acted_since_switch_in: bool` を追加した。

* `model/pokemon.py`
  * コンストラクタで `self.acted_since_switch_in: bool = False` を初期化。
  * `reset_on_switch_in()` で `False` にリセットする（場に出た直後は未行動扱い）。
* `core/move_executor.py` の `MoveExecutor.run_move()`
  * `try/finally` の `finally` 節（`attacker.failed_or_immobile_last_turn` を記録している
    箇所の直後）で `attacker.acted_since_switch_in = True` を無条件に設定する。
  * この位置は「今回の行動が呼び出し元の `run_move()` を通じて完全に処理し終えた後」で
    あるため、以下の性質を持つ:
    1. 通常の技使用（成功・失敗・タイプ相性による無効・命中判定でのミスを問わない）は
       すべて「行動済み」として記録される。
    2. まひ・ねむり・こおり・こんらん自傷・ひるみ・なまけ等で `ON_TRY_ACTION` が失敗し、
       `_execute_move` が呼ばれなかった場合も、`run_move()` 自体は最後まで実行されるため
       同様に「行動済み」として記録される（一次情報どおり、これらは最初の行動を
       消費した扱いになる）。
    3. であいがしら自身の成功判定（`ON_TRY_MOVE_1`）は `_execute_move` の内部、つまり
       `finally` 節よりも前に実行されるため、「今回の行動が正真正銘の最初の行動である
       場合」はこのフラグがまだ `False` のまま判定される → 正しく成功できる。
       フラグが `True` に更新されるのは行動が完全に終わった後なので、次回以降の
       行動でのみ「既に行動済み」と判定される。
    4. メトロノーム・ねごと等が `battle.run_move(attacker, chosen_move)` を再帰的に
       呼び出すケースでは、外側の `run_move()`（メトロノーム自体の実行）の `finally` 節は
       内側の再帰呼び出しがすべて完了した後にしか実行されない。そのため、外側の
       行動が「場に出てから最初の行動」であった場合、内側で選ばれた技が
       であいがしらであっても、その判定タイミングではフラグはまだ `False` のままであり、
       一次情報どおり「最初の行動を済ませた扱いにならず成功する」という挙動と一致する。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_TRY_MOVE_1` | 30 | attacker:self | 場に出てから最初の行動でなければ失敗させる |

```python
def であいがしら_check_first_turn(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """であいがしらの発動可否を判定する。

    場に出てから最初の行動でなければ不発になる。
    """
    if ctx.attacker.acted_since_switch_in:
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="であいがしら")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

## Priority 根拠

* `.internal/spec/turn.md` の `Event.ON_TRY_MOVE_1` 節に「30 | ねこだまし等: すでに行動をした」
  という専用の行が存在し、であいがしらはこの「ねこだまし等」に該当する。よって
  `priority=30` を明示的に指定する。

## 注意点・エッジケース（review-test 向け）

1. **基本ケース**
   - 場に出た直後の最初の行動として使用した場合は成功する。
   - 何らかの技を1回使用した（成功・失敗を問わない）後に使用した場合は失敗する。
2. **状態異常等で最初の行動が潰れた場合**
   - まひ等で最初の行動ができなかった場合も「行動済み」扱いとなり、次のターンの
     であいがしらは失敗する（一次情報どおり）。
3. **未対応の既知エッジケース**
   - メトロノーム等で偶然であいがしらが選ばれた場合に成功する挙動は上記の実装方針で
     自然にカバーされるが、これは意図した設計であり、特別なテストは必須としない
     （既存の `ねごと`/メトロノーム実装のテストで代替する）。
   - トリプルバトル・ローテーションバトル固有の例外、バッグ／ボールの使用、
     にげる失敗後の成功条件緩和は、本プロジェクトの対象（シングルバトル、
     アイテム使用・にげるコマンド無し）の範囲外のため未対応。
4. **テスト方針**
   - `test_であいがしら_相手にダメージを与える`（既存）に加え、
     `test_であいがしら_場に出てから最初の行動でなければ失敗する` /
     `test_であいがしら_場に出た直後は成功する` を追加する。
