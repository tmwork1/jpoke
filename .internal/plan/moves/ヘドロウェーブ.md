# ヘドロウェーブ 実装計画書

仕様書: `.internal/spec/moves/ヘドロウェーブ.md`

## 概要

`.internal/plan/moves/攻撃技_どく系.md`（どく状態異常を付与する攻撃技グループの計画書）の
対象技の一つ。本ファイルはヘドロウェーブ単体のレビュー用に切り出した個別計画書であり、
グループ計画書の内容と矛盾しない。

- タイプ: どく / 分類: 特殊 / 威力95 / 命中100 / PP12（Champions基準）
- 追加効果: 10%の確率で相手を『どく』状態にする

## ハンドラ構成

| イベント | Priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `Event.ON_DAMAGE_HIT` | 100（`MoveHandler` デフォルト） | `attacker:self`（`MoveHandler` デフォルト） | ダメージ命中後に10%の確率でどくを付与 |

### Priority 根拠

`.internal/spec/turn.md` の「Event.ON_DAMAGE」節（`emit: core/move_executor.py` の
`Event.ON_DAMAGE_HIT`。実装上のイベント名は `ON_DAMAGE_HIT` だが、`turn.md` の
概念名は「ON_DAMAGE」）priority=10「追加効果（特殊なもの除く）」に該当する処理。
`MoveHandler` のデフォルト priority=100 を用いる既存実装（`ヘドロこうげき`・
`ヘドロばくだん`・`どくばり` 等、同一グループの他技）と同じパターンを採用する。

なお `.internal/plan/moves/攻撃技_どく系.md` には `Event.ON_HIT` と記載されているが、
これはグループ計画書執筆当時の草案であり、実際の実装・テストはいずれも
`Event.ON_DAMAGE_HIT`（ダメージ適用後、追加効果発動用のイベント）を使用している。
本ファイルでは実装済みの `Event.ON_DAMAGE_HIT` を正とする。

## 実装コード

`src/jpoke/handlers/move_attack.py`（五十音順の適切な位置、既存関数を流用）:

```python
def ヘドロウェーブ_apply_poison_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="どく", chance=0.1)
```

`apply_ailment_to_defender`（`handlers/move.py`）は `resolve_secondary_chance` で
ちからずく・てんのめぐみ等の補正を適用した後、`ailment_manager.apply` でどく付与を
試みる。どく・はがねタイプへの無効化やめんえき等の免疫チェックは `ailment_manager.apply`
内部で処理されるため、本ハンドラ側に重複ガードは不要。

`src/jpoke/data/moves/move_ha.py`:

```python
"ヘドロウェーブ": MoveData(
    type="どく",
    category="special",
    pp=12,
    power=95,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ヘドロウェーブ_apply_poison_to_defender,
        )
    }
),
```
