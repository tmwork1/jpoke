# トリプルアクセル 実装計画書

## 仕様要約（.internal/spec/moves/トリプルアクセル.md 参照）

* タイプ: こおり / 分類: 物理 / 威力: 20（1発目）/40（2発目）/60（3発目） / 命中: 90 / PP: 12
  （チャンピオンズはPP8/12/16/20の4段階に統一。第9世代本家基準は10）
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果:
  * 最大3回連続で攻撃する連続攻撃技。攻撃が命中するたびに威力が上がる
    （1発目20→2発目40→3発目60）。
  * 命中判定は各ヒットごとに行う（`check_hit_each_time`）。外れた時点で終了。
  * 威力上昇は「試行したヒット回数」に連動するため、みがわりに防がれても次の威力は上がる。
  * スキルリンク／いかさまダイス所持時は初回ヒットのみ命中判定を行う。
  * 追加効果なし。

## 現状の実装状況

`src/jpoke/data/moves/move_ta.py` に既に登録済み。

```python
"トリプルアクセル": MoveData(
    type="こおり",
    category="physical",
    pp=12,
    power=20,
    accuracy=90,
    flags={"contact", "check_hit_each_time"},
    multi_hit={
        "min": 3,
        "max": 3,
        "check_hit_each_time": True,
        "power_sequence": (20, 40, 60),
    },
    handlers={},  # 追加効果なし
),
```

* `multi_hit.min == multi_hit.max == 3` のため、`_resolve_hit_count`
  （`core/move_executor.py`）は常に3を返す。
* `power_sequence == (20, 40, 60)` のため、`_resolve_hit_power` は
  `hit_index`（1始まり）に応じて20→40→60を返す。
* `flags` に `check_hit_each_time` を含み、`multi_hit.check_hit_each_time` も
  `True` のため、`ON_MODIFY_HIT_CHECK_EACH_TIME` が上書きされない限り
  ヒットごとに命中判定が行われる。
* スキルリンク（`data/ability.py`）・いかさまダイス（`data/item.py`）は
  `Event.ON_MODIFY_HIT_CHECK_EACH_TIME` に `False` を返すハンドラを登録しており、
  これらを持つ攻撃者は初回ヒットのみ命中判定される
  （`handlers/ability.py` の `スキルリンク_modify_hit_check_each_time`、
  `handlers/item.py` の `いかさまダイス_modify_hit_check_each_time`）。

## ハンドラ構成

固有ハンドラなし（`handlers={}`）。連続攻撃・威力上昇・毎ヒット命中判定の挙動は
`core/move_executor.py` 側の汎用機構（`multi_hit`）で処理されるため、
`handlers/move_attack.py` への追加実装は不要
（`トリプルキック` と同一パターン。ただしタイプ・威力テーブル・PPが異なる）。

## Priority 根拠

該当なし（固有ハンドラを持たないため priority 検討対象外）。
`ON_MODIFY_HIT_COUNT` / `ON_MODIFY_HIT_CHECK_EACH_TIME` は `.internal/spec/turn.md` 未掲載の
イベントだが、`いかさまダイス` の既存ハンドラ（同イベント、priority未指定=既定値100）に
合わせ、`スキルリンク` 側の `ON_MODIFY_HIT_CHECK_EACH_TIME` ハンドラも既定priorityとした
（両ハンドラとも常に `False` を返すため優先度による衝突は発生しない）。

## 注意点・エッジケース（review-test 向け）

1. **威力が回数ごとに上昇する**
   - `power_sequence == (20, 40, 60)` を確認する。
2. **各ヒットごとに命中判定される**
   - 2発目の命中判定に外れた場合、そこで打ち切られる
     （`トリプルキック` の同種テストパターン）。
3. **スキルリンク／いかさまダイス所持時は初回ヒットのみ判定**
   - 2発目以降で本来外れる状況を作っても、命中判定自体が行われず全3発命中する。
4. **PPはチャンピオンズ基準の12**
   - 第9世代本家基準の10ではない点に注意（`src/jpoke/data/ps-champ-ja/moves.json` 参照）。
