# 実装計画: つるぎのまい

## 仕様サマリ

- タイプ: ノーマル / 分類: 変化 / 威力: - / 命中率: - / PP: 20 / 対象: 自分
- 効果: 自分のこうげきランクを 2 段階上げる
- ダンス技（`dance` フラグ）
- まもる: 防がれない（自分対象） / マジックコート: 反射不可（自分対象） / 直接攻撃: いいえ
- よこどり: 対象になる（○）

出典: .internal/spec/moves/つるぎのまい.md

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自分のこうげきを 2 段階上げる |

## Priority 根拠

`.internal/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`わるだくみ_modify_attacker_stats`, `ドわすれ_modify_attacker_stats` 等）が
すべて priority=100（デフォルト）で登録されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: 五十音順で `つ〜と` の間（`つるぎのまい` は既に実装済み。`てっぺき_*` の直前）

```python
def つるぎのまい_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"atk": 2})
```

### data/moves/move_ta.py の更新

`target="self"` を指定する（`target` を指定しないとデフォルトの `"foe"` になり、
`is_blocked_by_protect` / `is_reflectable` がまもる・マジックコートに誤反応してしまうため必須）。

```python
"つるぎのまい": MoveData(
    type="ノーマル",
    category="status",
    pp=20,
    target="self",
    flags={"dance"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.つるぎのまい_modify_attacker_stats,
        ),
    }
),
```

## 注意点・エッジケース

- `target="self"` かつ `flags={"dance"}` はエントリに既記載のため変更不要
- ランクがすでに最大（+6）の場合の失敗は `battle.modify_stats()` 側でマネージャー内部処理されるため、
  ハンドラ側にガード不要
- 踊り技として「おどりこ」特性への連鎖は既存フレームワーク内で処理される
- 対象は自分のため命中率なし。テストは命中率固定不要

## リーサル計算

対象外（`.internal/progress/move.md` の該当行は `n/a`）。ダメージを与えない自己バフ技のため
リーサル計算ハンドラは不要。
