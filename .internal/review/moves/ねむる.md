## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_na.py` の `"ねむる"` エントリに **`target="self"` が欠落**していた
  バグを修正した。これにより相手の `まもる` で誤って防がれ、`マジックコート` で誤って
  跳ね返される不具合があった（実際に再現・修正を確認）
- `src/jpoke/data/moves/move_na.py` の `"ねむる"` の PP を **5 → 8** に修正した
  （`src/jpoke/data/ps-champ-ja/moves.json` 準拠。Gen9本家値からの移行漏れバグ）
- `src/jpoke/handlers/move_status.py` の `ねむる_apply` を修正: 従来は
  「HP全回復 → `ailment_manager.remove()` → `ailment_manager.apply()`」の順で実装されており、
  特性（やるき・ふみん・スイートベール・きよめのしお・リミットシールド）やゆめうつつ
  （ぜったいねむり特性、uncurable）によってねむり付与自体が失敗するケースでも、HP回復と
  状態異常解除だけが先に完了してしまい「ねむらずに全回復する」不具合があった。
  `ailment_manager.apply(mon, "ねむり", count=3, source=mon, overwrite=True)` を先に試行し、
  成功した場合のみHPを回復する順序に変更して修正した
- `ねむる_check`（`ON_TRY_MOVE_1`）に「場にさわぐ状態のポケモンがいる場合は失敗する」チェックを
  追加した（`mon.has_volatile("さわぐ") または mon.has_volatile("さわがしい")`）。従来は
  この条件が未実装で、相手がさわぐ状態でもねむるが成立してしまっていた
