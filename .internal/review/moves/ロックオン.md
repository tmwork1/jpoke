## 実装（handlers/ / data/）

**バグ発見・修正あり。**

### 問題1: 揮発状態の付与対象が誤っていた

`src/jpoke/handlers/move_status.py` の `ロックオン_apply` が
`apply_volatile_to_defender` を使っており、ロックオン状態が**相手**に付与されていた。
仕様（第五世代以降は自分にロックオン状態が付与される）と矛盾するバグ。

```python
# 修正前
def ロックオン_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_defender(battle, ctx, value, volatile="ロックオン", count=2)

# 修正後
def ロックオン_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ロックオンの効果: 自分をロックオン状態にし、次のターンの自分の技を相手に必中させる。"""
    return apply_volatile_to_attacker(battle, ctx, value, volatile="ロックオン", count=2)
```

### 問題2: PPが本家Gen9基準（5）のままだった

`src/jpoke/data/ps-champ-ja/moves.json` ではロックオンのPPは8（champions側でPP4段階統一により変更）。
`src/jpoke/data/moves/move_ra.py` を `pp=5` から `pp=8` に修正した。

### 問題3: `unreflectable` フラグが未設定だった

一次情報（判定セクション）でマジックコート「×」（反射不可）と明記されているが、
MoveData に `flags` が未設定だった。`flags={"unreflectable"}` を追加した。

## テスト

`tests/moves_status/test_move_ra.py` にロックオンの技本体（`ON_STATUS_HIT` ハンドラ）の
テストが存在しなかったため、以下4件を追加した。

- `test_ロックオン_自分にロックオン状態が付与される`: 使用者（相手ではない）にロックオン
  状態が付与されることを確認（問題1のバグの再発防止）
- `test_ロックオン_すでにロックオン状態なら失敗する`: 既にロックオン状態のときに再使用
  してもカウントが変化しない（重複付与されない）ことを確認
- `test_ロックオン_まもるで防がれる`: まもる状態の相手に使うと防がれ、ロックオン状態が
  付与されないことを確認
- `test_ロックオン_マジックコートで跳ね返されない`: `unreflectable` フラグにより
  マジックコート状態の相手に使っても跳ね返されず、自分にロックオン状態が付与されることを
  確認（問題3の修正の確認）

揮発性状態自体の必中効果・ターン経過解除・交代解除は
`tests/volatiles/test_volatile_hamayarawa.py` で既にテスト済みのため重複実装していない。

`python -m pytest tests/ -v` を実行し、4644 passed, 1 skipped（既存の無関係な skip）で
全件パスすることを確認した。ログは `.loop/test_logs/ロックオン.log` に保存。
