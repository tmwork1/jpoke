  - 一次情報（ポケモンWiki）をもとに、基本情報（タイプ・分類・威力・命中・PP）、
    最大3回連続攻撃・命中するたびに威力上昇（20→40→60）・毎ヒット命中判定・
    スキルリンク/いかさまダイス所持時の挙動・からぶりほけんの発動条件・各種判定フラグを記載。
  - PPは `src/jpoke/data/ps-champ-ja/moves.json` の値（12）を正として記載
    （第9世代本家基準は10だが、チャンピオンズはPP8/12/16/20の4段階に統一されているため）。

## 実装計画書
- 新規作成（`.internal/plan/moves/トリプルアクセル.md`）。
  - 既存実装（`data/moves/move_ta.py`）が仕様と一致するよう照合し、`multi_hit.power_sequence`

  - スキルリンク側に不足していた `ON_MODIFY_HIT_CHECK_EACH_TIME` ハンドラの追加を計画に明記。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ta.py` の `トリプルアクセル` エントリを修正。
  - `pp=10`（第9世代本家基準）になっていたが、`src/jpoke/data/ps-champ-ja/moves.json`・
    `.internal/progress/move.md` の値（12）と食い違っていたため `pp=12` に修正
    （PPのソースオブトゥルースは `move_list.txt`。詳細は CLAUDE.md 参照）。
  - `power_sequence == (20, 40, 60)` は一次情報（1発目20/2発目40/3発目60）と一致しており
    変更なし。
- `src/jpoke/handlers/ability.py` / `src/jpoke/data/ability.py`（スキルリンク）を修正。
  - 一次情報・`.internal/spec/abilities/スキルリンク.md`（「トリプルキック/トリプルアクセルの
    最初の1回が当たった場合、必ず攻撃は3回続く」）に対し、既存実装は
    `Event.ON_MODIFY_HIT_COUNT`（ヒット数を最大化）のみを登録しており、トリプルキック/
    トリプルアクセルは元々 `min == max == 3` のため実質効果がなく、
    毎ヒット命中判定（`check_hit_each_time`）を無効化する処理が欠落していた
    （いかさまダイス側には既に同等のハンドラが存在していた）。
  - `スキルリンク_modify_hit_check_each_time` を新規追加し、
    `Event.ON_MODIFY_HIT_CHECK_EACH_TIME` に登録（`subject_spec="attacker:self"`）。
    いかさまダイス（`いかさまダイス_modify_hit_check_each_time`）と同様に常に `False` を返す。
  - `src/jpoke/enums/event.py` の `ON_MODIFY_HIT_CHECK_EACH_TIME` のコメントを
    「handle: item.py（いかさまダイス）/ ability.py（スキルリンク）」に更新。
