## 実装（handlers/ / data/）
- **バグ修正**: `src/jpoke/data/moves/move_na.py` の `"ねをはる"` エントリに `target="self"`
  が指定されておらず、`MoveData.target` のデフォルト値 `"foe"` のままになっていた。
  `Move.is_blocked_by_protect` は `target == "foe"` を基準に判定するため、このままでは
  相手の『まもる』でねをはるが防がれてしまい、`src/jpoke/data/ps-champ-ja/moves.json` の「守×」
  （まもるで防がれない）に反する。同様に `Move.is_reflectable` も `target in ("foe",
  "foe_side")` を基準にするため、マジックコート等で誤って反射される状態になっていた。
  `target="self"` を追加して修正した。
- `handlers/move_status.py` の `ねをはる_apply`、`handlers/volatile.py` の
  `ねをはる_self_heal` / `ねをはる_block_blow` / `ねをはる_check_floating`、
  `data/volatile.py` の `"ねをはる"` エントリ（`ON_TURN_END` priority 70、
  `ON_CHECK_TRAPPED`、`ON_CHECK_FLOATING`、`ON_TRY_BLOW`）は仕様・`.internal/spec/turn.md` と
  整合しており変更なし。
- `heal` フラグは付与していない（仕様どおり。ねをはるはデータ上HP回復技に分類されず、
  かいふくふうじ状態でも使用自体は可能なため）。
