# ちょうのまい レビュー結果

## 仕様書
- 新規作成（`.internal/spec/moves/ちょうのまい.md`）。.internal/wiki/moves/ちょうのまい.html を参照し、
  タイプ・分類・PP・対象・効果・判定フラグ（急所/命中判定/追加効果/まもる/マジックコート/
  よこどり/オウムがえし）を記載した。

## 実装計画書
- 既存の `.internal/plan/moves/ちょうのまい.md` は `category="変化"` や `labels=["dance"]`、
  `stats={"C":1,"D":1,"S":1}` など現行の型・命名規則（`category="status"`, `flags={"dance"}`,
  `stats={"spa":1,"spd":1,"spe":1}`）と乖離した古い記述だったため、現行実装に合わせて全面的に
  書き直した。
- `target="self"` の必要性（設定しないとデフォルトの `"foe"` になり、まもる／マジックコートに
  誤反応する）を明記し、リーサル計算が対象外である旨のセクションを追加した。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ta.py` の `ちょうのまい` MoveData に `target="self"` が
  欠落していたバグを修正した。デフォルトの `target="foe"` のままだと
  `Move.is_blocked_by_protect` / `Move.is_reflectable` が真になり、相手の『まもる』で
  防がれたり『マジックコート』で跳ね返されたりする誤動作になっていた。
- `handlers/move_status.py` の `ちょうのまい_modify_attacker_stats`（とくこう・とくぼう・
  すばやさを各+1）自体は仕様通りで変更なし。

## リーサル計算
- 対象外（`.internal/progress/move.md` のリーサル実装列は `n/a`）。ダメージを与えない自己バフ技のため。

## テスト
