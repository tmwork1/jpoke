## 実装（handlers/ / data/）
- **バグ修正**: `src/jpoke/handlers/move_attack.py` の `こおりのキバ_apply_flinch_or_freeze` は
  単一の乱数値で `r < base*0.5` → こおり、`r < base` → ひるみ、という排他的判定になっており、
  一次情報の「2つある追加効果の発動判定はそれぞれ個別に行われる」（同時発生し得る）という仕様に
  反していた。実際には各効果10%ずつ独立判定すべきところ、実質的に「合計10%でどちらか一方のみ」
  という誤った挙動になっていた。
- `こおりのキバ_apply_freeze`（`apply_ailment_to_defender` で「こおり」を10%付与）と
  `こおりのキバ_apply_flinch`（`apply_volatile_to_defender` で「ひるみ」を10%付与）の
  2関数に分割し、`かみなりのキバ` と同じパターンに統一した。
- `src/jpoke/data/move.py` の `こおりのキバ` エントリの `handlers[Event.ON_DAMAGE_HIT]` を
  単一の `MoveHandler` からリスト（こおり付与→ひるみ付与の2つ）に変更した。
- `python scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` と
  `python scripts/sort_data/sort_moves.py` を実行して並び替えを反映した。
- `handlers/item.py` の `_INNATE_FLINCH_MOVES`（おうじゃのしるし等の重複防止リスト）には
  既に「こおりのキバ」が含まれており、修正不要だった。
