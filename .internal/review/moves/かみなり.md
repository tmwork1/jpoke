# かみなり レビュー結果

## 仕様書
- 新規作成（`.internal/spec/moves/かみなり.md`）。一次情報（`.internal/wiki/moves/かみなり.html`）を基に、
  タイプ・分類・威力（110、第五世代までは120）・命中率・PP・追加効果（30%まひ）・
  天候による命中率補正（あめ: 必中、にほんばれ: 命中率50）・そらをとぶ状態への命中仕様を記載した。
- 併せて `.internal/spec/moves/_move_secondary_1.md` の一覧データにあった誤り
  （かみなりのまひ付与確率が `0.1` と記載されていた）を `0.3` に修正した。

## 実装計画書
- 新規作成（`.internal/plan/moves/かみなり.md`）。既存の類似技（`ふぶき`・`かみくだく`）のパターンに
  準拠し、ハンドラ構成・priority根拠・エッジケース（天候補正、ちからずくとの相互作用、
  そらをとぶとの相互作用）を明記した。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/move_attack.py` の `かみなり_apply_paralysis_to_defender` の
  まひ付与確率が `chance=0.1`（10%）になっていたバグを発見し、`chance=0.3`（30%）に修正した
  （一次情報: 「追加効果として、30%(第一世代は10%)の確率で相手をまひ状態にする」）。
  本プロジェクトの対象世代（第九世代）では30%が正しい。
- `かみなり_accuracy`（天候による命中率補正: あめ必中・にほんばれ命中率50、ばんのうがさ考慮）は
  仕様と一致しており修正不要。
- `そらをとぶ` 状態への命中仕様は `handlers/volatile.py` の
  `HIDDEN_MOVE_ALLOWED_MOVES["そらをとぶ"]` に登録済みで修正不要。
- `data/move.py` のMoveData定義（type/category/pp/power/accuracy/flags）は仕様と一致しており修正不要。
- ついでに `.internal/progress/move.md` の `かみなり` 行のPP列が `12` と誤っていたため `10` に修正し、
  隣接する `かみなりのキバ`・`かみなりパンチ` 行のPP列も `16` から実際の `15` に修正した
  （一次情報で確認）。また `かみなり` 行の効果欄から実装されていない「フリーフォール」への言及を除去した。

## リーサル計算
- 対象外（`.internal/progress/move.md` のリーサル実装列が `n/a`。まひ付与などの確率依存効果は
  他の同種技と同様にリーサル計算の対象としていない）。

## テスト
- `tests/moves_attack/test_move_ka.py` の既存テスト `test_かみなり_まひが発動する` の
  docstringが「10%でまひを付与する」という誤った記述だったため「30%でまひを付与する」に修正した。
- 天候による命中率補正を検証するテストを追加した:
  - `test_かみなり_あめ中は必中になる`（`ふぶき_ゆき中は必中になる` と同じパターン）
  - `test_かみなり_にほんばれ中は命中率が50になる`
- 既存の関連テスト（`tests/moves_attack/test_move_sa.py::test_そらをとぶ_かみなりは命中する`、
  `tests/test_item.py::test_ばんのうがさ_雨でかみなりが必中にならない`）は仕様と一致しており修正不要。
- `python scripts/sort_tests.py tests/moves_attack/test_move_ka.py` → `python scripts/generate_test_list.py` を実行し、
  `python -m pytest tests/ -v` で全3280件（+スキップ1件）がパスすることを確認した。
