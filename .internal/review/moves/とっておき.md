# とっておき レビュー結果

## 仕様書
- `.internal/spec/moves/とっておき.md` は既存だったが、対象の Champions における重要な仕様変更点
  （条件を満たしていない場合、選択すること自体ができなくなる）が反映されていなかったため、
  「技の仕様」節に以下を追記した。
  - スカーレット・バイオレットまでは選択自体は可能で行動時に失敗するのみだが、
    **Champions では条件を満たしていない場合はとっておき自体を選択できない**旨を明記。

## 実装計画書
- `.internal/plan/moves/とっておき.md` が存在しなかったため新規作成した。
  - 技の使用履歴追跡（`Pokemon.pp_consumed_moves`）、判定ロジック（`PokemonQuery.can_use_last_resort`）、
    発動失敗ハンドラ（`ON_TRY_MOVE_1` priority=30）、コマンド選択制限（`CommandManager`）の
    4つの実装ポイントを整理して記載した。

## 実装（handlers/ / data/）
- `data/move.py` の `"とっておき"` は `power=140, accuracy=100, pp=5` は正しかったが、
  `handlers={}` となっており、技の核心的な発動条件（他の技を全て使い切っていないと失敗する）が
  一切実装されていなかった。以下を新規実装した。
  - `model/pokemon.py`: `pp_consumed_moves: set[MoveName]` 属性を追加（場に出てから PP を
    消費して使用した技名の集合）。`reset_on_switch_in()` でリセット。
    （`__deepcopy__` の `keys_to_deepcopy` には既にこの属性名への参照が残っており、
    未使用のまま放置されていたことが判明したため、これを実際に活用する形にした）
  - `core/move_executor.py`: `_consume_pp()` で実際に PP が消費された（`v > 0`）場合のみ
    `pp_consumed_moves` に技名を記録するようにした。これにより、まひ・ねむり等で行動できず
    PP を消費できなかった技は「使用した」扱いにならず、ねごとのサブ技（PP消費0）も
    使用済みに計上されない（仕様通り）。
  - `core/query.py`: `PokemonQuery.can_use_last_resort(pokemon)` を新設し、
    「とっておきを覚えているか」「他の技を覚えているか」「他の技を全て使用済みか」を判定する
    ロジックを一元化した。
  - `handlers/move_attack.py`: `とっておき_check_used_all_moves` を新設し、`can_use_last_resort`
    が False の場合に技を失敗させる。
  - `data/move.py`: `"とっておき"` に `Event.ON_TRY_MOVE_1`（priority=30、
    `.internal/spec/turn.md` の「とっておき: 未使用技がある」に対応）でこのハンドラを登録した。
  - `core/command_manager.py`: `get_available_action_commands()` の技コマンド生成条件に
    `can_use_last_resort` チェックを追加し、Champions 仕様通り条件未達成時はコマンド自体を
    選択肢から除外するようにした。
- `scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` と
  `scripts/sort_data/sort_moves.py` を実行済み。
