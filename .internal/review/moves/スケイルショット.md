## 実装（handlers/ / data/）
- `handlers/move_attack.py` の `スケイルショット_apply_stat_change` にバグを発見し修正した。
  - 修正前: `if ctx.hit_index != ctx.hit_count: return HandlerReturn(value=value)`
    という条件のみで「最終ヒットかどうか」を判定していたため、連続ヒットの途中で相手が
    ひんしになり残りのヒットが行われなかった場合（例: 4ヒット予定のうち2ヒット目で相手が倒れる）に、
    `hit_index` が `hit_count` に到達せず自己ランク変化が発動しないバグがあった。
  - 仕様書に明記した「全ての攻撃が終了した後、当たった回数に関係なく」発動するという挙動
    （Pokemon Showdown の `selfBoost` 実装でも技全体の成否のみを条件に1回発動する）に反するため、
    `ctx.defender.fainted` を追加のOR条件として修正:
    `if ctx.hit_index != ctx.hit_count and not ctx.defender.fainted: return ...`
- `data/moves/move_sa.py` のエントリ（型・威力・命中・PP・multi_hit設定・ハンドラ登録）は
  仕様書と一致しており修正不要。
