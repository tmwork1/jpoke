# とおせんぼう レビュー結果

## 仕様書
- 新規作成（`.internal/spec/moves/とおせんぼう.md`）。一次情報（ポケモンWiki）と `src/jpoke/data/ps-champ-ja/moves.json`、
  および同系統の `.internal/spec/moves/くろいまなざし.md` を参照して作成した。
  - PP8（必中）、まもるでは防がれない（`守×`）、みがわりでは防がれる、ゴーストタイプには
    『にげられない』状態が無効、等を明記
  - 一次情報にあるトリックガード・ダイウォールによる無効化、姿を隠した相手への非命中は
    本プロジェクト未実装のためスコープ外として記載

## 実装計画書
- 既存の `.internal/plan/moves/とおせんぼう.md` を修正した。
  - PP を `5` → `8`（Championsの値。`src/jpoke/data/ps-champ-ja/moves.json` 準拠）に修正
  - `accuracy=100` → `accuracy=None`（必中）に修正
  - まもるで防がれない（第六世代以降）ことに対応する `flags={"unprotectable"}` の付与を追記
  - ゴーストタイプ無効化の根拠を「`ON_TRY_MOVE_2` priority 120」という誤った記述から、実際の
    フレームワーク機構である `にげられない` 揮発性状態側 `Event.ON_CHECK_TRAPPED` の
    `check_trapped_not_ghost` に修正（`.internal/plan/moves/くろいまなざし.md` の記述と整合させた）
  - 同様に「すでに逃げられない状態なら失敗」の根拠を `VolatileManager.apply()` の重複付与
    チェックに修正
  - 挿入位置の記述を実装済みの実際の位置（`てんしのキッス` の後、`でんじは` の前）に修正

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ta.py` の `"とおせんぼう"` エントリを修正した。

  - `accuracy=100` → `accuracy=None`（必中技だったため100%命中ではなく命中判定自体を行わない

  揮発性状態を defender に付与）は正しく、修正不要だった
