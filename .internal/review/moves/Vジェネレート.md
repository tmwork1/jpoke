# Vジェネレート レビュー結果

## 仕様書

- `.internal/spec/moves/` に個別の仕様書は存在しない。100%発動の自分ランク低下
  効果（『ぼうぎょ』『とくぼう』『すばやさ』各-1）は、他の同種の自分ランク
  低下技（インファイト、ぶちかまし、オーバーヒート など）と同様に
  `.internal/spec/moves/_move_secondary_1.md` の一覧表（206行目、
  `Vジェネレート attacker 1 0 -1 0 -1 -1 ...`）で管理する方針であり、
  一次情報（Wiki）と照合した結果、効果自体は正しく記載されていた。
  - タイプ: ほのお、分類: 物理、威力: 180、命中率: 95、PP: 5、直接攻撃: ○
    （一次情報のインフォボックスと一致）。
- 個別ファイルは作成しないという既存の方針（Gのちから・3ぼんのや 等の
  レビュー実績と同様）を踏襲し、新規作成は行っていない。

## 実装計画書

- `.internal/plan/moves/Vジェネレート.md` は存在しない。同種の単純な自分ランク
  低下技は `.internal/plan/moves/攻撃技_攻撃側ランク変化系.md`（バッチ計画書）で
  まとめて管理する慣例のため、個別の新規作成は行わなかった。
- **不備を発見し修正した**: `攻撃技_攻撃側ランク変化系.md` は本バッチ計画時点
  で Vジェネレートを「MoveData 未登録（新規追加が必要）」の一覧に
  `Ｒジェネレート`（technicalな誤字、かつ全角Ｖ）として誤記載していたが、
  実際には計画作成前からすでに `data/move.py` に登録済みであった。
  該当行を削除し、実装済みである旨と本レビューでの修正内容（後述）を明記した。
- 同ファイル内の他の Vジェネレート 関連記述（一覧表・ハンドラ構成表・
  コード例・エッジケース節）もすべて全角`Ｖ`表記だったため、実装・Wiki上の
  正式表記である半角`V`に統一し、コード例内の `accuracy=80` を `95` に、
  ハンドラ名を実装と一致する `Vジェネレート_lower_attacker_def_spd_spe` に
  修正した。

## 実装（handlers/ / data/）

- **不備を発見し修正した**: `src/jpoke/data/move.py` の `"Vジェネレート"` の
  `accuracy` が `80` になっていたが、一次情報（Wiki）によると命中率は
  `95` である。`accuracy=95` に修正した。
- **不備を発見し修正した**: `src/jpoke/handlers/move_attack.py` の
  ハンドラ関数名が `Vジェネレート_lower_defender_spd_spe` となっていたが、
  実際には `modify_attacker_stats` を用いて攻撃者自身のランクを下げる処理
  であり、`defender` という命名は実態と矛盾していた（アーマーキャノン等の
  正しい命名パターンに倣い）`Vジェネレート_lower_attacker_def_spd_spe` に
  改名した。`data/move.py` 側の参照も合わせて更新し、
  `python scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` を
  実行したが並び順自体に変更はなかった。
- `Event.ON_HIT`（デフォルト priority=100、`subject_spec="attacker:self"`）
  での登録は、同種の自分ランク低下技（インファイト、りゅうせいぐん 等）と
  一貫しており妥当。`.internal/spec/turn.md` の `Event.ON_HIT` セクション
  （L474〜L489）に記載の「自分のランクが下がる反動」（概念上の優先度10）は
  イベント内の処理グループを示すものであり、`MoveHandler` の `priority`
  引数はイベント内の個別順序を指定するものであるため、既存パターン
  （priority 引数省略=100）に合わせて変更なし。
- `flags={"contact", "secondary_effect"}` は一次情報の「直接攻撃: ○」と
  一致しており妥当。変更なし。
- `stats={"def": -1, "spd": -1, "spe": -1}` によりぼうぎょ・とくぼう・
  すばやさが同時に1段階ずつ下がる実装は仕様通り。変更なし。
- `python scripts/sort_data/sort_moves.py` を実行したが、値のみの変更の
  ため並び順に変更はなかった。

## テスト

- `tests/moves_attack/test_move_yarawa.py` の
  `test_Vジェネレート_防御特防素早さが各1段階低下する` で、命中時に
  使用者の `def`/`spd`/`spe` ランクがそれぞれ-1になることを確認済み。
  `start_battle(accuracy=100)` で命中率を固定しているため、
  `accuracy` フィールドの修正（80→95）はこのテストの結果に影響しない。
  過不足なし。変更なし。
- `python scripts/sort_tests.py tests/moves_attack/test_move_yarawa.py`、
  `python scripts/generate_test_list.py` を実行したが差分なし。
- `python -m pytest tests/ -v` を実行し、Vジェネレート関連のテスト
  （`test_Vジェネレート_防御特防素早さが各1段階低下する`）は成功。
  結果は `.loop/test_logs/Vジェネレート.log` に保存した。
  なお同時実行中の並列レビュー作業（他技・特性のファイル編集）の影響で、
  実行のたびに無関係なテスト（メガシンカのABILITIES未登録KeyError、
  だっしゅつパック/ボタン、てんねん 等）が数件失敗する現象が見られたが、
  実行ごとに失敗するテストの組み合わせが変動しており、Vジェネレートの
  変更（`data/move.py`・`handlers/move_attack.py`）とは無関係であることを
  確認した。
