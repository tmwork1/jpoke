# わるあがき レビュー結果

## 仕様書
- 既存の `.internal/spec/moves/わるあがき.md` を一次情報（ポケモンWiki）と照合し、以下を修正:
  - 「タイプ: ノーマル」という誤った記述を、対象世代（第九世代・金銀/ポケモンスタジアム2以降）
    の実際の仕様である「タイプなし（タイプ相性・タイプ一致補正の影響を受けない）」に修正。
  - 反動ダメージの丸め処理（五捨五超入、端数0は最低1）を明記。
  - 特性いしあたま・マジックガードのどちらでも反動ダメージを防げないという、
    他の反動技と異なる特殊な性質を明記。
  - アンコール・かなしばりが直前技わるあがきに対して失敗すること、おんねん状態が
    わるあがきによるひんしに対して発動しないことを追記。
  - 威力補正特性についてテクニシャンだけでなく、かたいツメ・はりきり・こだわりハチマキ/
    メトロノームも対象になることを追記。

## 実装（handlers/ / data/）
- `data/moves/move_symbol.py`: `power=40` → `power=50` に修正
  （ポケモンWikiの技データ表・Pokémon LEGENDS アルセウス表の両方で威力50と明記されているため）。
- `handlers/move_attack.py` の `わるあがき_self_damage`:
  - 反動計算を `battle.modify_hp(ctx.attacker, r=-1/4, ...)`（内部で `int()` による
    0方向への切り捨てのみ）から `max(1, round_half_down(ctx.attacker.max_hp / 4))` を用いた
    明示的な五捨五超入の丸めに変更。
    例: 最大HP119（119/4=29.75）の場合、修正前は29ダメージ、修正後は正しく30ダメージになる。
  - `reason="recoil"` → `reason="self_cost"` に変更。
    `reason="recoil"` のままだと `いしあたま_ignore_recoil`（`ON_MODIFY_NON_MOVE_DAMAGE` で
    `hp_change_reason == "recoil"` を無効化）や `マジックガード_ignore_damage`
    （`"recoil"` を含む多くの reason を無効化）に一致してしまい、いしあたま・マジックガード
    所持時に反動ダメージが誤って消えてしまうバグがあった。
    `"self_cost"`（自己HP消費、マジックガード・いしあたまで防げない）に変更することで、
    ポケモンWikiに明記された「わるあがきの反動はいしあたま・マジックガードでも防げない」
    という仕様を正しく再現するよう修正。

## テスト
- `tests/moves_attack/test_move_yarawa.py` に以下を追加:
  - タイプ相性の影響を受けないこと（ゴースト対面でも通る）
  - タイプ一致補正を受けないこと（`atk_type_modifier == 4096`）
  - 威力が50固定であること（`final_power == 50`）
  - 使用後に反動ダメージを受けること
  - 反動ダメージが与えたダメージ量に依存しないこと（固定ダメージにしても
    `round_half_down(max_hp/4)` になること）
  - 反動ダメージが最大HPの1/4を五捨五超入で丸めた値になること
    （ゼニガメ: 最大HP119、期待値30。修正前の実装では29になっていた）
- `tests/abilities/test_ability_a.py` に `test_いしあたま_わるあがきの反動ダメージは防げない` を追加。
- `tests/abilities/test_ability_ma.py` に `test_マジックガード_わるあがきの反動ダメージは防げない` を追加。
- `python -m pytest tests/ -v` で3650件全件パスを確認
  （実行順によって無関係な既存テスト〔`test_くらいつく_*`等〕が乱数依存で稀に失敗する
  既存のflakinessを確認したが、わるあがき関連の変更には起因しないことをスタッシュ比較で確認済み）。
