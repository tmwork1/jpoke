# かたくなる レビュー結果

## 仕様書
- 新規作成（`.internal/spec/moves/かたくなる.md`）。一次情報（`.internal/wiki/moves/かたくなる.html`）に基づき、
  タイプ・分類・威力・命中率・PP・範囲・優先度・直接攻撃・効果・判定
  （急所/命中判定/追加効果/まもる/おうじゃのしるし/マジックコート/よこどり/オウムがえし）を記載した。
- PPについて: `src/jpoke/data/ps-champ-ja/moves.json` に「かたくなる」自体の項目が存在しないことを確認した。
  ただし一次情報に「ソード・シールドで使用不可」等の除外カテゴリの記載はなく、第9世代で通常に
  使用可能な技であるため、README.md の「対象範囲」定義（第9世代で実装されていない技は実装しない）
  には該当しない。よって champions未収録は「PP統一ルールの対象外（ロースターに習得者がいない等）」
  と判断し、PPはGen9本家の値（30）を採用する旨を明記した。

## 実装計画書
- 新規作成（`.internal/plan/moves/かたくなる.md`）。ハンドラ構成表・priority根拠
  （`ON_STATUS_HIT` は `.internal/spec/turn.md` に記載がないため、既存の自分対象ランク上昇ハンドラと
  同じ priority=100 デフォルトを採用）と、`target="self"` を設定する必要がある理由
  （デフォルトの `target="foe"` のままだと `Move.is_blocked_by_protect` /
  `Move.is_reflectable` が誤って True になる不具合）を明記した。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ka.py` の `かたくなる` エントリに `target="self"` を追加した。
  - 修正前は `target` 未指定（デフォルト `"foe"`）だったため、`Move.is_blocked_by_protect`
    （`target == "foe"` で判定）や `Move.is_reflectable` が誤って True になり、相手の「まもる」で
    阻止されたり「マジックコート」で跳ね返されたりする可能性があった不具合を修正した。
  - PPは既存の30のまま変更なし（Gen9本家準拠、championsに項目なし）。
- `src/jpoke/handlers/move_status.py` の `かたくなる_modify_attacker_stats` に、他のハンドラと
  同様の日本語docstring（「かたくなるの効果: 自分の防御を1段階上げる。」）を追加した。
  ハンドラ本体のロジック（`modify_attacker_stats(battle, ctx, value, stats={"def": 1})`）は
  仕様と一致しており変更なし。
