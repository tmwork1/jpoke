# こごえるせかい レビュー結果

## 仕様書
- 新規作成（`.internal/spec/moves/こごえるせかい.md`）。.internal/wiki/moves/こごえるせかい.html を参照し、
  タイプ・分類・威力・命中・PP・対象・優先度・追加効果（100%の確率で相手のすばやさを1段階下げる）・
  各種判定フラグ（まもる/おうじゃのしるし/マジックコート/よこどり/オウムがえし）を記載した。

## 実装計画書
- 新規作成（`.internal/plan/moves/こごえるせかい.md`）。既存の同型技（`がんせきふうじ`・`こごえるかぜ`）の
  計画書構成を踏襲し、ハンドラ構成・Priority根拠・リーサル計算対象外の理由を明記した。

## 実装（handlers/ / data/）
- `src/jpoke/data/move.py` の `こごえるせかい` エントリに `flags={"secondary_effect"}` が
  欠落していたバグを修正した。これにより `ちからずく` 特性使用時に威力1.3倍・すばやさ低下無効化が
  発動しない不整合があった（同一の「100%確率ですばやさ1段階低下」技である `がんせきふうじ` /
  `こごえるかぜ` は既に同フラグを保持していた）。
  `python scripts/sort_data/sort_moves.py` を実行済み。
- `handlers/move_attack.py` の `こごえるせかい_lower_defender_spd` 実装自体は
  `modify_defender_stats(battle, ctx, value, stats={"spe": -1})` で仕様と一致しており、変更なし。

## リーサル計算
- 対象外（`.internal/progress/move.md` のリーサル実装列は n/a）。すばやさランクを下げるのみで
  HP・ダメージ計算に影響しないため、専用のリーサル計算ハンドラは不要。
