# スキルスワップ レビュー結果

## 仕様書
- 新規作成（`.internal/spec/moves/スキルスワップ.md`）。
- `.internal/wiki/moves/スキルスワップ.html`（ポケモンWiki、本レビューで取得）を一次情報として、
  基本情報（エスパー・変化・威力なし・命中「-」＝必中・PP・相手1体選択・優先度0・非接触）、
  効果、失敗条件（`protected` フラグ・とくせいガード・おうごんのからだ）、
  判定フラグ（まもる○・みがわり×・マジックコート×＝反射されない・よこどり／トリックガード
  は未実装のため対象外）を記載した。

## 実装計画書
- 既存の `.internal/plan/moves/スキルスワップ.md` を実装・一次情報と照合し、以下の誤りを修正した。
  - PP: 旧計画の「10」から一次情報（`src/jpoke/data/ps-champ-ja/moves.json`）準拠の「12」に修正。
    ポケモンWikiの技データ表は第三世代時点のPP（10）を掲載しているが、現行版の値は
    move_list.txtを正とする。
  - 命中: 旧計画は「命中: 必中（命中判定なし）」と書きつつ実装は `accuracy=100` の
    ままだった不整合を、`accuracy=None` に統一。
  - マジックコートの反射可否について、旧計画は「反射されない」と正しく記述していたが、
    実装の `flags` に `unreflectable` が付与されておらず矛盾していたバグを発見・修正
    （後述）。
  - おうごんのからだの無効化処理のイベント名を、旧計画の誤記「`ON_TRY_MOVE_1` priority
    110」から実装と一致する「`ON_BEFORE_APPLY_MOVE`
    (`おうごんのからだ_block_status_move`, priority 100, `subject_spec="defender:self"`)」
    に修正。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_sa.py` の「スキルスワップ」エントリを修正:
  - `pp=10` → `pp=12`（`src/jpoke/data/ps-champ-ja/moves.json` の「スキルスワップ	エスパー	変化	-	-	12	接×	守○」に準拠。Gen3基準の値が移行漏れとして残っていた）
  - `accuracy=100` → `accuracy=None`（必中技。move_list.txt でも命中欄は「-」で必中を表す。
    命中判定自体が行われないため、回避ランクや命中ランクの影響を受けない）
  - `flags={"bypass_substitute"}` → `flags={"bypass_substitute", "unreflectable"}`
    （マジックコート・マジックミラーで跳ね返されない、という一次情報「マジックコート: ×」の
    記載を反映。`unreflectable` フラグが欠落しており、修正前はマジックコート状態の相手に
    使用すると誤って反射されてしまうバグがあった）
- `src/jpoke/handlers/move_status.py` の `スキルスワップ_can_apply` /
  `スキルスワップ_swap_ability`: 既存実装のまま問題なし。使用者・対象双方の `protected`
  フラグと `is_change_blocked`（とくせいガード等）を判定済みで、一次情報・仕様書の
  失敗条件と一致することを確認した。
- `src/jpoke/core/ability_manager.py` の `swap_ability`: 既存実装のまま問題なし。
  `is_change_blocked` の二重チェック、`ON_ABILITY_DISABLED` → ハンドラ解除 →
  特性入れ替え（使用者側が先に登録される順序）→ ハンドラ再登録 → `ON_ABILITY_ENABLED`
  という一連の処理により、入れ替え直後の特性発動（いかく等）と発動順
  （「第五世代以降ではすばやさに依らずスキルスワップ使用者側から先に発動する」）が
  自動的に再現されることを確認した。
