# バリアーラッシュ レビュー結果

## 仕様書
- 新規作成（`.internal/spec/moves/バリアーラッシュ.md`）。
- 一次情報（`.internal/wiki/moves/バリアーラッシュ.html`）と `src/jpoke/data/ps-champ-ja/moves.json`（714行目）を確認し、
  以下を明記した。
  - タイプ: エスパー / 分類: 物理 / 威力: 70（SV基準）→ 90（Champions基準） / 命中: 90 /
    PP: 10（SV基準）→ 12（Champions基準） / 対象: 相手1体 / 優先度: 0 / 接触あり
  - 追加効果: 100%の確率で**自分（攻撃側）**の『ぼうぎょ』ランクを1段階上げる
  - オドシシ・アヤシシの専用技
- 依頼文には「バリアーラッシュは自分が壁（リフレクター等）を張っている場合に威力が上昇する技」と
  記載されていたが、一次情報（ポケモンWiki本文・Champions技一覧の効果欄）を確認したところ誤りだった。
  実際の効果は「命中後100%の確率で自分のぼうぎょランクを1段階上げる」のみで、壁の有無による
  威力補正は存在しない。既存実装にも壁判定・威力補正のロジックは存在せず、そのような効果を
  追加していない（一次情報に基づき仕様書に明記した）。

## 実装計画書
- 新規作成（`.internal/plan/moves/バリアーラッシュ.md`）。
- Champions基準PP・威力の根拠（`src/jpoke/data/ps-champ-ja/moves.json` 714行目）を明記した。
- `.internal/spec/turn.md` の `Event.ON_HIT`「自分のランクを上げる追加効果 priority=10」との対応関係を
  確認したが、同種の既存ハンドラ（グロウパンチ・コメットパンチ・メタルクロー等）がいずれも
  `MoveHandler` デフォルト値（priority=100）のまま登録されており、明示的にpriority=10を
  指定している実装例が存在しないため、既存の同種ハンドラとの一貫性を優先しデフォルト値のまま
  据え置いた（本レビューでは変更しない旨を計画書に明記）。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ha.py` の `MoveData` を修正した。
  - `power` を 70 → 90 に修正（Champions基準への移行漏れ。`src/jpoke/data/ps-champ-ja/moves.json` 714行目参照）。
  - `pp` を 10 → 12 に修正（同上）。
- `src/jpoke/handlers/move_attack.py` の関数名を `バリアーラッシュ_boost_defender_B` から
  `バリアーラッシュ_boost_attacker_B` に修正した。実装内容（`modify_attacker_stats(..., stats={"def": 1})`）は
  攻撃側自身のランクを変更しているにもかかわらず、旧関数名は「defender」を含んでおり、
  `グロウパンチ_boost_attacker_A` 等の命名規則（`_boost_attacker_<変化するステータスの略号>`）と
  矛盾していたため（実際の挙動自体は元から正しかった＝命名のみのバグ）。
  併せてdocstringを追加した。
- `flags={"contact", "secondary_effect"}` および `Event.ON_HIT` への登録・`subject_spec`
  （`MoveHandler` 既定値 `attacker:self`）は仕様通りであり、変更なし。

## リーサル計算
- 対象外。自分のぼうぎょランク変化はその場のダメージ・HPに影響しないため、同一パターンの他技
  （グロウパンチ・コメットパンチ等）と同様にリーサル計算ハンドラの対象外とした。
  進捗表の「リーサル実装」「リーサルテスト」は既に n/a になっており、変更なし。

## テスト
- `tests/moves_attack/test_move_ha.py` に以下を追加した。
  - `test_バリアーラッシュ_secondary_effectフラグを持つ`: `flags` に `secondary_effect` が
    含まれることを確認。
  - `test_バリアーラッシュ_Champions基準の威力とPPを持つ`: `power == 90`, `pp == 12` を確認。
  - `test_バリアーラッシュ_相手にダメージを与える`: 直接攻撃の物理技として相手にダメージを与えることを確認。
- 既存の `test_バリアーラッシュ_防御1段階上昇が発動する` に、相手（防御側）のぼうぎょランクが
  変化しないことの確認（`defender.rank["def"] == 0`）を追加した。
- `python -m pytest tests/ -v` を複数回実行し、バリアーラッシュ関連の4テストは常に成功することを
  確認した。全体としては、テスト実行順序に依存する既存の（本レビュー対象外の）フレーキーな
  失敗が実行のたびに異なるテストで1〜3件発生する状態だったため、レビュー対象外の変更を加えていない
  ベースブランチ（`git stash` で本レビューの変更を除いた状態）でも同様の現象
  （`くらいつく`関連3件失敗等、実行のたびに異なる）が再現することを確認済み。バリアーラッシュの
  変更によって新たに発生した失敗ではないため、本レビューでは対応しない。
  最終ログは `.loop/test_logs/バリアーラッシュ.log` に保存した
  （3949 passed, 1 failed（フレーキーで無関係）, 1 skipped）。
