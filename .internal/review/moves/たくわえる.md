## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ta.py` の `"たくわえる"` エントリに `target="self"` が欠落していたバグを修正
  - `MoveData.target` のデフォルトは `"foe"` のため、この欠落により `Move.is_blocked_by_protect`（変化技かつ `target=="foe"` で True）と `Move.is_reflectable`（変化技かつ `target in ("foe","foe_side")` で True）が誤って成立し、相手の まもる・マジックコートに誤反応する状態だった
  - `handlers/move_status.py` の `たくわえる_check_can_use` / `たくわえる_apply` 自体のロジックは仕様通り（カウント3で失敗、ランク上昇＋カウントインクリメント）で変更なし

## 参考: 発見したが本タスクのスコープ外とした事項
- `はきだす`・`のみこむ` の PP が `src/jpoke/data/ps-champ-ja/moves.json` 基準では12だが、実装（`data/moves/move_ha.py` / `data/moves/move_na.py`）は10のまま（移行漏れ）。`.internal/progress/move.md` の `はきだす` 行のPP列も10のままで、champions側と食い違っている
- `のみこむ` の `MoveData` にも `target="self"` が未設定（自分を対象とするHP回復技だが、たくわえると同種のバグを抱えている可能性がある）
- これらは `はきだす`・`のみこむ` 自体のレビュータスクで対応すべき事項のため、本タスクでは実装ファイルを変更していない
