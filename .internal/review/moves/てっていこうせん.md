# てっていこうせん レビュー結果

## 仕様書

- 新規作成（`.internal/spec/moves/てっていこうせん.md`）
- 一次情報（`.internal/wiki/moves/てっていこうせん.html`）から以下を明記した:
  - 基本情報（はがね/特殊/威力140/命中95/PP5/相手1体/優先度0/直接攻撃なし）
  - HP消費は最大HPの1/2を**切り上げ**（既存実装は切り捨てだった）
  - `Event.ON_PAY_HP` がヒット処理より前に発火するため、まもる・タイプ無効化・命中失敗でも
    使用者のHPは減ること
  - 特性との相互作用: **マジックガードでは反動を防げるが、いしあたまでは防げない**
    （クロロブラストは逆に両方で防げるため仕様が異なる。混同しないよう明記）
  - すてみで威力が上がらないこと（`flags={"recoil"}` を付与しないことで対応）
  - HP0でも技が発動し、相討ちの場合は使用者側が負けになる裁定
  - `ダメおし` との相互作用（反動ダメージも「そのターンに受けたダメージ」として扱われる）は
    反動技全般に共通する `ダメおし` 側の課題であり、本技単独では対応しないことを明記（スコープ外）

## 実装（handlers/ / data/）

- `src/jpoke/handlers/move_attack.py` の `てっていこうせん_pay_hp` に2件のバグを発見・修正:
  1. **端数処理が切り捨てだった**: `max(1, ctx.attacker.max_hp // 2)` → 一次情報は「小数点以下
     切り上げ」のため `max(1, (ctx.attacker.max_hp + 1) // 2)` に修正。
  2. **`HPChangeReason` が `"self_cost"` になっていた**: `"self_cost"` はマジックガード・
     いしあたまどちらでも防げない理由のため、「マジックガードでは防げるがいしあたまでは防げない」
     という本技の仕様と矛盾していた。新設した `"fixed_recoil"` に変更した。
- `src/jpoke/types/literals.py` の `HPChangeReason` に `"fixed_recoil"` を追加
  （マジックガードで防げるがいしあたまでは防げない反動。てっていこうせん専用）。
  `マジックガード_ignore_damage` の除外リストには含めない（＝防がれる）ことで対応し、
  `いしあたま_ignore_recoil` は `reason == "recoil"` のみを見るため影響しない
  （いずれも既存コードの変更は不要、新しい reason 値の追加のみで両特性の相互作用を正しく表現できた）。
- `data/move.py` の登録内容（`Event.ON_PAY_HP` / `subject_spec="attacker:self"`）は変更なし。
- `python scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` を実行済み（並び順に変化なし）。
