## 実装（handlers/ / data/）
`src/jpoke/data/moves/move_sa.py` の `"ソウルビート"` エントリと `src/jpoke/handlers/move_status.py`
の `ソウルビート_check` / `ソウルビート_pay_hp_and_modify_attacker_stats` を確認し、以下4点の
バグを発見・修正した。

1. **`target` 未指定バグ（重大）**: `target` を指定しておらず、`MoveData` のデフォルト値
   `"foe"` が適用されていた。ソウルビートは自分が対象の技であり、本来 `target="self"` である
   べき。`target` 未指定のままだと `Move.is_blocked_by_protect` が `True` になり、相手がまもる
   状態のときに誤って失敗する、および `Move.is_reflectable` が `True` になり、マジックコート・
   マジックミラーで誤って跳ね返される、という観測可能なバグだった。`target="self"` を明示して
   修正した。
2. **PP誤り**: 旧実装は `pp=5`（本家Scarlet/Violetの基準値）だったが、
   `src/jpoke/data/ps-champ-ja/moves.json`（L434）は `8`。PPはchampionsのmove_list.txtを正とする
   プロジェクト方針に基づき `pp=8` に修正した（`.internal/progress/move.md` のPP列はすでに
   `8` と正しく記載されていたため、実装側のみが誤っていた）。
3. **HP消費前の失敗判定漏れ（重大）**: 旧実装は「HP不足」のみを事前チェックし
   （`ON_BEFORE_APPLY_MOVE`）、「対象5ランクすべてが+6」の場合は `ON_STATUS_HIT` 側の
   `modify_attacker_stats` が空の変化結果を返すことで失敗扱いになる設計だった。しかし
   `battle.modify_hp(mon, r=-1/3)` が `modify_attacker_stats` より**先**に呼ばれていたため、
   全ランクがすでに+6のときでもHPだけ消費されてしまうバグがあった
   （`はらだいこ_can_apply` / `はらだいこ_apply` の参照実装と異なり、失敗条件がHP消費より
   後に評価される順序になっていた）。`ソウルビート_check` に「対象5ランクすべてが+6」の判定を
   追加し、HP消費前に判定するよう修正した。
4. **`display_reason` の命名不統一**: 旧実装は `display_reason="ソウルビート_HP不足"` と
   していたが、他の同種チェック（`はらだいこ`・`みがわり`等）は技名のみ（例:
   `display_reason="はらだいこ"`）を渡す慣例のため、`display_reason="ソウルビート"` に統一した。

### `.internal/spec/turn.md` の修正
- L257「30 | ソウルビート: HPが足りない」（`ON_TRY_MOVE_1`）と L337「130 | ソウルビート等:
  全能力最大」（`ON_TRY_MOVE_2`）の2行は、実際の実装（および `はらだいこ`・`みがわり` の
  確立済みパターン）と矛盾していたため削除した。
- 代わりに `ON_APPLY_MOVE`（コード上のイベント名は `ON_BEFORE_APPLY_MOVE`）セクションの
  「100 | みがわり: HP不足」の直後に「100 | ソウルビート: HP不足・全能力+6」を追加し、
  実装・計画書・turn.mdの3者を一致させた。

### HP消費量の丸め誤差確認
- `battle.modify_hp(mon, r=-1/3)` は `int(max_hp * (-1/3))` で計算するが、Python の浮動小数点
  誤差により `max_hp // 3`（失敗判定式）と異なる結果になる可能性を懸念し、`max_hp` を
  1〜999の範囲で総当たり検証した。結果、すべての値で `-v == max_hp // 3` が成立し
  （`max_hp // 3 == 0` となる `max_hp <= 2` の極小ケースを除く）、丸め誤差による不整合は
  発生しないことを確認した。

修正後、`python scripts/sort_handlers.py src/jpoke/handlers/move_status.py` と
`python scripts/sort_data/sort_moves.py` を実行したが、並び替えは発生しなかった
（既存の並び順が五十音順を維持していたため）。

## テスト
既存テストは1件（HP消費・全能力+1のみを確認）で、修正した失敗条件・`target="self"`の
挙動をカバーしていなかったため、以下4件を追加した（`tests/moves_status/test_move_sa.py`）。

- `test_ソウルビート_HPが最大の3分の1以下なら失敗しHPも減らない`: HP不足時に失敗し、
  HP・ランクとも変化しないことを確認。
- `test_ソウルビート_すべてのランクが最大なら失敗しHPも減らない`: 全能力+6での失敗時に
  HPが消費されないこと（今回修正したバグの回帰テスト）を確認。
- `test_ソウルビート_一部のランクのみ最大でも成功しHPを消費する`: 一部ランクのみ+6でも
  失敗せず、上げられる分だけ上げてHPを消費することを確認。
- `test_ソウルビート_相手のまもるを無視して成功する`: `target="self"`修正の回帰テストとして、
  相手がまもる状態でも無視して成功することを確認。

`python scripts/sort_tests.py tests/moves_status/test_move_sa.py` と
`python scripts/generate_test_list.py` を実行し、`python -m pytest tests/ -v` で全件成功を
確認した（`test_むしくい_ねんちゃく持ちをひんしにさせた場合は奪える` はRNGシード依存の
既知flakyテストで、単体再実行では成功することを確認済み。本レビューの変更とは無関係）。
