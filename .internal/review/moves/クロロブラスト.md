# クロロブラスト レビュー結果

## 仕様書
- 新規作成（`.internal/spec/moves/クロロブラスト.md`）。
  一次情報（.internal/wiki/moves/クロロブラスト.html）に基づき、以下を明記した。
  - 基本情報（くさ/特殊/威力150/命中95/PP5→チャンピオンズ換算8/相手1体/優先度0/接触なし）。
  - 反動ダメージは技が成功した場合のみ発生し、まもる・命中失敗・タイプ無効化時は発生しない
    （てっていこうせん・ビックリヘッドは使用前に無条件で消費する点で異なる）。
  - 命中して実ダメージ0（こらえる・ばけのかわ等）でも反動は発生する。
  - いしあたま・マジックガードの双方で反動を防げる（てっていこうせん・ビックリヘッドはマジックガード
    のみで防げ、いしあたまでは防げない点で異なる）。
  - すてみで威力上昇しない、爆発技には含まれない、うのミサイル・レンブのみとの発動順序が既存
    アーキテクチャ上の既知の限界であること、などを記載。

## 実装計画書
- 新規作成（`.internal/plan/moves/クロロブラスト.md`）。
  仕様書をもとに、`Event.ON_HIT`（`いのちのたま`・`かいがらのすず`・`だっしゅつボタン`・`レッドカード`
  と同じ理由で、実ダメージ0でも発火するイベントを採用）で `reason="recoil"` の反動を実装する設計、
  および発見した実装上の誤りと修正内容を記載した。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/move_attack.py`: `クロロブラスト_pay_hp` を `クロロブラスト_recoil` に変更し、
  以下の誤りを修正した。
  1. **イベント選択の誤り**: `Event.ON_PAY_HP`（技使用前に無条件でHPを消費）を使っていたため、
     まもる・命中失敗・タイプ無効化時にも反動を受けてしまっていた。一次情報「技が成功した場合のみ
     反動を受ける」に合わせ `Event.ON_HIT`（命中時発火、実ダメージ0でも発火）に変更した。
  2. **`HPChangeReason` の誤り**: `"self_cost"`（いしあたま・マジックガードどちらでも防げない）に
     なっていたが、一次情報は「いしあたま・マジックガードの双方で防げる」ため、既存の `"recoil"`
     に変更した（新規の `HPChangeReason` 追加は不要）。
  3. **端数処理の誤り**: `ctx.attacker.max_hp // 2`（切り捨て）になっていたが、一次情報は
     「小数点以下切り上げ」のため `(ctx.attacker.max_hp + 1) // 2` に修正した。
- `src/jpoke/data/moves/move_ka.py`: `Event.ON_PAY_HP` 登録を `Event.ON_HIT` に変更し、`pp=5` を
  `pp=8` に修正した（PPの根拠は下記「進捗表」節を参照）。
