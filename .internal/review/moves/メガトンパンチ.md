# メガトンパンチ レビュー結果

## 仕様書
- 対象外（新規作成せず）。一次情報（.internal/wiki/moves/メガトンパンチ.html）を確認したところ、
  ノーマルタイプ・物理・威力80・命中率85・PP20・直接攻撃・パンチ技で、追加効果は無し。
  第一世代から第九世代まで数値・効果に変更はない。
  本コードベースでは「追加効果なし」の単純な攻撃技（例: DDラリアット、マッハパンチ、
  ドレインパンチ等）は仕様書・計画書を作成しない運用のため、`.internal/progress/move.md` の
  仕様書・計画書列が `-` のまま据え置きとした。

## 実装計画書
- 対象外（新規作成せず、理由は仕様書の項と同様）。

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/data/moves/move_ma.py` の定義
  （`type="ノーマル"`, `category="physical"`, `pp=20`, `power=80`, `accuracy=85`,
  `flags={"contact", "punch"}`, `handlers={}`）は一次情報・champions/move_list.txt
  （本技は掲載なしのためGen9本家PPをそのまま採用）と整合していることを確認した。
  パンチ技フラグにより特性『てつのこぶし』の威力1.2倍は
  `handlers/ability.py` の `てつのこぶし_modify_power` が汎用的に処理するため、
  個別実装は不要（既存の実装通り）。

## リーサル計算
- 対象外（`.internal/progress/move.md` のリーサル実装・リーサルテスト列は `n/a`）。
  追加効果・特殊なダメージ計算を持たない通常攻撃技のため、既存の汎用リーサル計算で
  問題ないことを確認した。

## テスト
- `tests/moves_attack/test_move_ma.py` にダメージ確認テストを新規追加した
  （`test_メガトンパンチ_相手にダメージを与える`）。てつのこぶしによる威力補正は
  `tests/abilities/test_ability_ta.py` 等で汎用的に検証済みのため、本技専用のテストは
  追加していない。
- `python -m pytest tests/ -v` で全4022件（1件skip）がパスすることを確認した
  （実行ログ: `.loop/test_logs/メガトンパンチ.log`）。
  なお実行中に `tests/moves_attack/test_move_ma.py::test_むしくい_ねんちゃく持ちをひんしにさせた場合は奪える`
  が1回だけ失敗する事象を観測したが、まひの完全麻痺（25%）に起因する既存テストの
  非決定性であり、メガトンパンチとは無関係のため本レビューでは修正していない
  （再実行では複数回連続でパスすることを確認済み）。
