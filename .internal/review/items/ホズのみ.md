# ホズのみ レビュー結果

## 仕様書
- 変更なし。`.internal/spec/items/_タイプ半減きのみ.md`（タイプ半減きのみ共通仕様書）内の「ホズのみ」節が
  一次情報（ポケモンWiki）と一致していることを確認した。
  - 効果:ノーマルタイプの攻撃技を受けたときに消費して、1度だけダメージを1/2にする
    （ノーマル技は通常抜群にならないため、他のタイプ半減きのみと違い相性に関係なく発動する）。
  - 詳細な仕様: ほしがるでは通常奪えない（先に消費されるため）。技のタイプが変わって
    ノーマルでなくなったときや、あくタイプのどろぼうを受けたときは奪われる。ノーマルスキンで
    ノーマル化したついばむ・むしくい・はたきおとすを受けた場合は技の効果よりホズのみが優先され、
    はたきおとすの威力1.5倍は維持される。わるあがきは第五世代以降発動しない。

## 実装（handlers/ / data/）
- 変更なし。既存実装を確認し、仕様との整合性に問題がないことを確認した。
  - `src/jpoke/data/item.py` の `ホズのみ` は `Event.ON_CALC_DAMAGE_MODIFIER` ハンドラ
    （`h.ホズのみ_modify_super_effective_damage`, `subject_spec="defender:self"`）と
    `LethalEvent.ON_BEFORE_HIT` の `lethal_handlers`（`l.ホズのみ_resist_normal`,
    `subject="defender"`）を登録済み。
  - `src/jpoke/handlers/item.py` の `ホズのみ_modify_super_effective_damage` は共通関数
    `_modify_super_effective_damage` を `type_="ノーマル", modifier=2048/4096,
    super_effective_only=False` で呼び出しており、効果バツグン判定を行わずタイプ一致のみで
    発動する（ホズのみ専用の重要パラメータが正しく分岐されていることを確認）。
  - `src/jpoke/handlers/lethal.py` の `ホズのみ_resist_normal` も同様に `_type_resist_berry` を
    `super_effective_only=False` で呼び出しており、瀕死判定でも整合している。
  - わるあがきは `type=""` として定義されており、`ctx.move.type != type_` の判定で
    自然に対象外となることを確認した（追加の特別処理は不要）。
  - どろぼう（あくタイプ）はタイプ不一致のため発動せず奪取が成立し、ほしがる（ノーマルタイプ）は
    ダメージ計算時点でホズのみが先に消費されるため後続の奪取が失敗することを、実際に
    バトルを実行して動作確認した。
