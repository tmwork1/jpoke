# サイコシード レビュー結果

## 仕様書
- `.internal/spec/items/_フィールドシード.md` の「サイコシード」節が「効果」「詳細な仕様」ともに空欄だったため、一次情報（`.internal/wiki/items/サイコシード.html`）をもとに以下を追記した。
  - 効果: サイコフィールド中に使用してとくぼう+1（1度発動すると消費）、なげつける威力10、ウッウロボ素材（エスパータイプ、素材ポイント4）
  - 詳細な仕様: 地面にいなくても発動すること、すでにとくぼうが最大（+6）のときは消費しないこと、発動タイミング（登場時／フィールド新規展開時）

## 実装計画書
- `.internal/plan/items/サイコシード.md` に以下の誤り・欠落があったため修正した（`.internal/plan/items/エレキシード.md` の最新版に準拠させた）。
  - `Event.ON_SWITCH_IN` の priority が未記載（デフォルト100扱い）だったが、`.internal/spec/turn.md` の ON_SWITCH_IN 表では「120 サイコシードの発動」と定義されているため 120 に修正し、根拠を明記した。
  - `Event.ON_ITEM_ENABLED`（かがくへんかガス解除等）のイベント記載が欠落していたため追記した。
  - `Event.ON_FIELD_CHANGE` / `Event.ON_ITEM_ENABLED` は turn.md に記載がないためデフォルト優先度（100）を採用する旨と根拠（既存の同種ハンドラ実装を参照）を明記した。
  - 「とくぼうがすでに最大のときは消費しない」という一次情報の仕様が計画書に欠落していたため追記した。

## 実装（handlers/ / data/）
- `src/jpoke/data/item.py`: 「サイコシード」の `Event.ON_SWITCH_IN` ハンドラに `priority=120` が付与されておらず、他のフィールドシード系（エレキシード・グラスシード）と異なり `.internal/spec/turn.md` の規定（「120 サイコシードの発動」）に反していたため `priority=120` を追加した。
- `src/jpoke/handlers/item.py`: 共通ヘルパー `_terrain_seed_boost` を使用しており、ランク上限時の不消費ロジックは既に正しく実装済みだったため変更なし。

## テスト
- `tests/test_item.py` に既存のパラメータ化テスト（`test_フィールドシード_フィールドなしでは発動しない` / `test_フィールドシード_ランク最大時は消費しない` / `test_フィールドシード_発動`）でサイコシードは既にカバーされていたが、これらは全て「登場前にフィールドを設定」または「登場後にフィールドを新規展開」のケースであり、「フィールド展開済みの場に交代で登場する」ケース（ON_SWITCH_IN の priority=120 が効くケース）を検証するテストが無かったため、`test_サイコシード_展開済みサイコフィールドに登場して発動` を新規追加した（`test_グラスシード_展開済みグラスフィールドに登場して発動` と同形式）。
- `python scripts/sort_tests.py tests/test_item.py` と `python scripts/generate_test_list.py` を実行して並び替え・一覧更新を行った。
- `python -m pytest tests/ -v` を実行し、2843 passed, 1 skipped（既存の無関係なskip）で全テスト成功を確認。ログは `.loop/test_logs/サイコシード.log` に保存。
