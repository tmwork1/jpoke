# リンドのみ レビュー結果

## 仕様書
- 変更なし。`.internal/spec/items/_タイプ半減きのみ.md`（タイプ半減きのみ共通仕様書）内の「リンドのみ」節が
  一次情報（ポケモンWiki `.internal/wiki/items/リンドのみ.html`）と一致していることを確認した。
  - 効果:効果バツグンのくさわざを受けたときに消費して、1度だけダメージを1/2にする。
  - 一次情報を確認したが、他のタイプ半減きのみ（ソクノのみ・ヨプのみ等）にあるような特殊な
    相互作用（タイプ変化技・特性による判定変化など）の記載はなく、「詳細な仕様」欄は空欄のままで
    問題ないことを確認した。

## 実装計画書
- 全面更新（`.internal/plan/items/リンドのみ.md`）。旧内容は「仕様書未整備・テスト未実装」という
  古い定型文のままで、共有仕様書・共有ヘルパーの実態と乖離していたため、他のタイプ半減きのみ
  （ヨロギのみ等）のレビュー後の計画書と同じ形式に揃え、共通仕様書の参照先・イベント
  （`ON_CALC_DAMAGE_MODIFIER` / `LethalEvent.ON_BEFORE_HIT`）・効果バツグン判定条件
  （`calc_def_type_modifier(ctx) > 4096`）・priority根拠（`.internal/spec/turn.md` に個別記載がないため
  既存の同種きのみハンドラに合わせた既定値を使う旨）を明記した。

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/data/item.py` の `リンドのみ` 登録内容（`ON_CALC_DAMAGE_MODIFIER` ハンドラと
  `lethal_handlers`、`fling_power=10`）に誤りはなく、五十音順の並びも正しい。
- `src/jpoke/handlers/item.py` の `リンドのみ_modify_super_effective_damage`（`type_="くさ"`,
  `modifier=2048/4096`）、`src/jpoke/handlers/lethal.py` の `リンドのみ_resist_grass`
  （`_type_resist_berry(..., "くさ")`）とも、共有ヘルパー `_modify_super_effective_damage` /
  `_type_resist_berry` を正しく利用している。
- 共有ヘルパー自体の重大バグ（アイテム消費漏れ、効果抜群判定の閾値誤り `> 1` → `> 4096`、
  ホズのみ用 `super_effective_only` パラメータ追加）は直前のレビュー（ナモのみ・ヨロギのみ）で
  既に修正済みであることを確認し、リンドのみ側で追加の修正は不要と判断した。

## テスト
- 変更なし。`tests/test_item.py` の既存パラメータ化テスト `test_タイプ半減実` に
  `("リンドのみ", "くさ", "ゼニガメ")` が含まれており、ゼニガメ（みずタイプ単）に対して
  くさ技が効果バツグンになる組み合わせとして正しいことを確認した。
  - `damage_modifier == 2048`（1/2倍）であることと、発動後にアイテムが消費されている
    （`not has_item()`）ことの両方を検証済み。
- リーサル側のテスト `test_タイプ半減きのみ_代表種`（`tests/test_lethal.py`）は代表種3件
  （オッカのみ・ヤチェのみ・ロゼルのみ）のみを検証する設計であり、リンドのみを個別に追加する
  必要はないことを確認した（共有ロジックの動作確認が目的のため）。
- `python -m pytest tests/ -v` を実行し、リンドのみ関連の全テスト（`test_タイプ半減実` を含む
  25件）がパスすることを確認した。結果は `.loop/test_logs/リンドのみ.log` に保存済み
  （3144 passed, 2 failed, 1 skipped）。
  - 失敗した2件（`tests/moves_attack/test_move_a.py::test_いかりのまえば_相手HP半分のダメージ`）は
    `accuracy=100` 未指定によるテスト順序・乱数状態依存の既存 flaky テストであり、単体実行では
    パスすることを確認済み。リンドのみ・アイテム実装とは無関係。
