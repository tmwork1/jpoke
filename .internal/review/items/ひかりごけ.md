  かたやぶりのアビリティ無効化（`ON_BEGIN_MOVE`〜`ON_END_MOVE` 区間）により自動的に仕様通りの挙動になることを明記した。

  「使い捨てアイテムの効果はマジシャンで奪う前に発動する」という順序を、素早さに関わらず保証する必要があることと、
  その実現方法（priority=90）を記述した。
- 同じ `_boost_stat_on_type_hit` ヘルパーを使う `きゅうこん` / `じゅうでんち` / `ゆきだま` にも同様の発動順の課題が

- テスト観点に、ランク上限時不発・たんじゅん・あまのじゃく・あまのじゃく最小時不発・かたやぶり時の挙動・
  マジシャンとの発動順を追加した。

## 実装（handlers/ / data/）
- `src/jpoke/data/item.py` の `ひかりごけ` の `ON_DAMAGE_HIT` ハンドラ登録に `priority=90` を追加した。
  - 不具合の内容: マジシャン特性（`ON_DAMAGE_HIT`, subject=attacker, priority=100（デフォルト））と
    ひかりごけ（`ON_DAMAGE_HIT`, subject=defender, 修正前はpriority=100（デフォルト）だった）は、
    フレームワークの `_sort_handlers` が同一priority時に `-素早さ` でタイブレークするため、
    攻撃側が防御側より素早い場合にマジシャンが先に発動し、ひかりごけが発動する前に道具を奪ってしまっていた。
  - `.internal/spec/abilities/マジシャン.md`・`.internal/spec/items/ひかりごけ.md` はいずれも「ひかりごけの効果が先に発動する
    ため奪われない」と明記しており、素早さに関わらず常にひかりごけが先でなければならない。
  - 修正: ひかりごけの `ON_DAMAGE_HIT` ハンドラに `priority=90` を指定し、マジシャン（priority=100）より
    確実に先に発動するようにした（`いかさまダイス` の `priority=90` と同様の先例パターン）。
  - 修正後、実際にピカチュウ（マジシャン・みずでっぽう、素早さが高い）対カビゴン（ひかりごけ、素早さが低い）の
    組み合わせで検証し、ひかりごけが先に発動・消費され、マジシャンは奪う対象がなく発動しないことを確認した。
- `_boost_stat_on_type_hit` 共通ヘルパー自体（ランク上限時の不発判定を含む、じゅうでんちレビューで修正済みの内容）は
  変更なし。ひかりごけはこの修正の恩恵をすでに受けている状態だったため、追加修正は不要だった。
- たんじゅん・あまのじゃく・かたやぶりに関する固有ロジックは追加していない。これらは既存のフレームワーク機構
  （`たんじゅん_double_stat` / `あまのじゃく_reverse_stat` を `ON_BEFORE_MODIFY_STAT` で発火、
  `かたやぶり` の `mold_breaker_ignorable` フラグによる防御側特性の技実行中無効化）によって自動的に
  仕様通りに動作することを確認した。
