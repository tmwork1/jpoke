## 実装（handlers/ / data/）
仕様書の「特性を変える技・特性の効果で特性が書き変わることや、無効化されることがない」のうち、
**無効化のブロックのみが実装されており、変更のブロックが一切実装されていない**という大きな欠落を発見し、修正した。

- `src/jpoke/core/ability_manager.py`
  - `AbilityManager.is_change_blocked(mon)` を新設し、`ON_CHECK_ABILITY_DISABLE` を発火して
    特性変更のブロック可否を判定できるようにした。
  - `change_ability()` に変更前のブロックチェックを追加。これによりなかまづくり・なやみのタネ・
    シンプルビーム・なりきり（使用者自身の変更）・トレースが一括してブロックされるようになった。
  - `swap_ability()` にも両者のブロックチェックを追加し、スキルスワップ・さまようたましいで
    片方だけ入れ替わることがないようにした。
- `src/jpoke/handlers/move_status.py`
  - `シンプルビーム_can_apply` / `なかまづくり_can_apply` / `なやみのタネ_can_apply` /
    `スキルスワップ_can_apply` / `なりきり_can_apply` に `is_change_blocked()` チェックを追加し、
    既存の `protected` フラグと同様に技自体を失敗させるようにした（PP消費のみで不発）。
- `src/jpoke/core/handler.py` / `src/jpoke/core/event_manager.py` / `src/jpoke/model/effect.py`
  - `Handler` に `ignored_disable_reasons` フィールドを追加し、`GameEffect.enabled_ignoring()` で
    指定した無効化理由を除いて有効判定できるようにした。
  - `EventManager._check_handler_validity()` の item 無効化判定をこの新メソッド経由に変更。
- `src/jpoke/handlers/item.py` / `src/jpoke/data/item.py`
  - `ItemHandler` に `ignored_disable_reasons` パラメータを追加。
  - `とくせいガード` のハンドラに `ignored_disable_reasons=frozenset({"ぶきよう"})` を指定し、
    所持者の特性がぶきようであっても効果が発動するようにした（仕様書に明記された挙動）。
  - `とくせいガード_check_ability_disable` にアイテム公開・ログ出力（`_announce_item_triggered`）を追加した
    （従来は引数がすべて `_` 始まりで未使用のまま、無条件でブロックするだけの実装だった）。

## 既知の簡略化点（未対応）
- かがくへんかガスとの相互作用で「メッセージが出ない場合がある」という細かい仕様（発動タイミング依存）は
  未対応。本実装ではブロック時は常にメッセージを出す。
- トレースの「保留」状態などトレース自身の詳細仕様は本レビューの対象外（トレース実装のレビューで対応）。
