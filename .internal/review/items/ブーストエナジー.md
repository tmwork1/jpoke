# ブーストエナジー レビュー結果

## 仕様書
- 一次情報（`.internal/wiki/items/ブーストエナジー.html`）と照合したところ、`## 効果` セクションが
  未記入（空欄）だったため、以下を追記した。
  - こだいかっせい/クォークチャージ持ちに持たせたとき、登場直後に消費してランク補正込みで
    最も高い能力を1.3倍（素早さのみ1.5倍）にする。同値時の優先順位（攻撃→防御→特攻→特防→
    素早さ）、HPを対象に含めないことを明記。
  - なげつけるはパラドックスポケモン（こだいかっせい/クォークチャージ持ち）が使用すると失敗し、
    それ以外は威力30であることを明記。
  - removable=False（専用道具相当の授受制限）であることを追記。
- `## 詳細な仕様` 以下は一次情報と一致しており、変更なし。

  リスクと対策を追記した。

## 実装（handlers/ / data/）
仕様書と照合した結果、以下の欠落・不具合を発見し修正した。

1. **`refresh_paradox_charge_state` の特性ガード漏れ（潜在バグ）**
   - `src/jpoke/handlers/ability_paradox.py`
   - この関数はこだいかっせい/クォークチャージ側の `ON_SWITCH_IN` 等に加え、
     アイテム側の `ON_ITEM_ENABLED`/`ON_ITEM_GAINED` からも呼ばれる共通処理だが、
     アイテム側の呼び出しは保持者の特性を問わず発火するため、パラドックス特性を
     持たないポケモンがブーストエナジーを取得しただけで誤ってブーストが発動してしまう
     不具合があった（仕様書「特性こだいかっせい/クォークチャージを持たないポケモンに
     持たせても特に効果は無い」に反する）。関数冒頭に特性チェックを追加して修正した。
2. **トリック・すりかえ・はたきおとす・どろぼう等による授受が防げていなかった**
   - `removable=False` は `ItemData` 上のフラグに過ぎず、実際に授受を防ぐには
     `ON_CHECK_ITEM_CHANGE` ハンドラの個別登録が必要（他の専用道具＝だいこんごうだま等と
     同様のパターン）だったが、ブーストエナジーには未登録だった。
   - `handlers/item.py` に `ブーストエナジー_prevent_item_change` を新規実装し、
     `data/item.py` の `ON_CHECK_ITEM_CHANGE` に登録した。
     - 保持者自身がパラドックス特性持ちの場合は外部からの授受（渡す/奪われる）を防ぐ。
       ただしブースト消費に伴う自己消費（`source is mon`）は防がない。
     - `swap_items` 経由（トリック・すりかえ・どろぼう等、`ctx.source is None`）の場合は
       交換相手（`battle.foe(mon)`）の特性も判定し、パラドックス特性持ちへ渡ることも防ぐ。
     - はたきおとす等の単純消失処理（`ctx.source` に実行者が入る）では相手側の判定は行わない
       （破壊のみで transfer ではないため）。
3. **なげつけるがパラドックス特性持ちでも成功してしまっていた**
   - `handlers/move_attack.py` の `なげつける_check_item` に、はっきんだま×ギラティナと
     同様のパターンで「攻撃者がこだいかっせい/クォークチャージ持ちかつブーストエナジー所持」の
     場合に失敗する条件を追加した。

修正後、`python scripts/sort_handlers.py src/jpoke/handlers/item.py` と
`python scripts/sort_data/sort_items.py` を実行済み。
