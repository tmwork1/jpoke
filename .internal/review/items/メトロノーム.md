## 実装（handlers/ / data/）
- **修正**: `メトロノーム_update_count` が `Event.ON_MOVE_END` に登録されていたため、以下のケースで
  カウントが正しく更新されない不具合があった。
  - `Event.ON_MOVE_END` は技がタイプ相性・特性・まもるなどで無効化された場合には発火しない
    （`move_executor._execute_move` が `ON_MOVE_END` 発火前に `return` するため）。そのため、
    仕様書にある「無効化されたときはカウントが0にリセットされる」という挙動が実装されていなかった。
  - ミス時は `ON_MOVE_END` 自体は発火するが、ハンドラ側で命中判定を確認していなかったため、
    ミスした技をヒットしたものとして扱い、誤ってカウントを増加させていた。
  - 修正として、常に発火する `Event.ON_END_MOVE` にハンドラを移し、`battle.move_executor` の
    `action_success` / `move_success` / `move_applied` / `move_missed` を参照して以下のように場合分けした。
    - 状態異常等で技を出せなかった場合（`action_success is False`）: カウントを維持
    - 溜め技の溜めターンなど、実際に攻撃を試みていない場合: カウントを維持（従来の挙動を保持）
    - 無効化・失敗・ミスの場合: カウントを0にリセット
    - 成功の場合: 従来通りカウントを更新
  - `src/jpoke/handlers/item.py` の `メトロノーム_update_count` を修正し、
    `src/jpoke/data/item.py` の登録イベントを `ON_MOVE_END` → `ON_END_MOVE` に変更した。
  - `python scripts/sort_handlers.py src/jpoke/handlers/item.py` と
    `python scripts/sort_data/sort_items.py` を実行済み。
