## 実装計画書
- `.internal/plan/abilities/トレース.md` を全面的に補充した。旧計画書は「候補が複数ならランダム
  選択」「ON_SWITCH_IN / ON_SWITCH_OUT のみ登録」という当初案のままで、仕様書にある保留・
  再判定系の仕様（相手不在時の保留、かがくへんかガス発動中の保留、それぞれの解除時の
  再判定）が反映されていなかったため、以下を追記・修正した。
  - シングルバトル限定のためランダム選択は不要である旨を明記。
  - 交代時の元特性への復帰は `Pokemon.base_ability_name` の既存の仕組みで対応済みであり、
    `Event.ON_SWITCH_OUT` への個別登録は不要である旨を明記（旧計画書の誤り）。
  - 保留・再判定を実現するための `Event.ON_SWITCH_IN` / `Event.ON_ABILITY_ENABLED` への
    `source:self` / `source:foe` 4エントリ登録方針を明記。
  - とくせいガードのブロックは `battle.change_ability` 内の `is_change_blocked` /
    `ON_CHECK_ABILITY_DISABLE` で自動処理されるため個別対応不要である旨を明記。
  - 対象外とする範囲（ダブル/トリプル固有仕様、とくせいガード不発メッセージの表示制御、
    かがくのちから/レシーバー経由の即時発動）を明記。

## 実装（handlers/ / data/）
- 旧実装は `Event.ON_SWITCH_IN`（`source:self`）のみで一度だけ判定する単純なものだった。
  以下のバグ・欠落を修正した。
  - **バグ修正**: 相手が瀕死のまま自分だけ交代した場合（例: 味方の交代技で相手を倒し、
    その交代先としてトレースを出した場合）、瀕死の相手のポケモンが `battle.actives` に
    残っているため、その特性を誤ってコピーしてしまう問題があった。`foe.fainted` を
    チェックして瀕死の相手からはコピーしないよう修正した。
  - **欠落追加**: 相手不在時の保留から、後で相手が入場したときに再判定してコピーする
    仕組み（`Event.ON_SWITCH_IN`, `subject_spec="source:foe"`）を追加した。
  - **欠落追加**: 相手のかがくへんかガスが発動している間は自分のトレース自体が無効化され
    保留になるが、ガスが解除された瞬間に再判定してコピーする仕組み
    （`Event.ON_ABILITY_ENABLED`, `subject_spec="source:self"` / `"source:foe"`）を追加した。
  - 上記に伴い `handlers/ability.py` の `トレース_copy_ability` を共通処理
    `_トレース_try_copy(battle, mon)` に切り出し、新規 `トレース_copy_ability_on_foe_change`
    （`source:foe` 用）を追加した。`data/ability.py` の `トレース` エントリを
    `Event.ON_SWITCH_IN` / `Event.ON_ABILITY_ENABLED` それぞれに `source:self` /
    `source:foe` の2ハンドラずつ（計4エントリ）登録するよう変更した。
  - とくせいガードによるブロックは既存の `battle.change_ability` の仕組みで自動的に
    処理されることを確認した（追加実装不要）。
