# すてみ レビュー結果

## 仕様書
- 変更なし。`.internal/spec/abilities/すてみ.md` は一次情報（`.internal/wiki/abilities/すてみ.html`）の対象技一覧（16技）、
  補正率（4915/4096 = 1.2倍）、対象外技（わるあがき、ビックリヘッド/てっていこうせん/クロロブラスト、
  じばく/だいばくはつ/ミストバースト、反動で動けなくなる技、反動で能力が下がる技）と一致していることを確認した。

## 実装計画書
- 変更なし。`.internal/plan/abilities/すてみ.md` の実装方針（`Event.ON_CALC_POWER_MODIFIER` /
  `subject_spec="attacker:self"` / `ctx.move.has_flag("recoil")` / `apply_fixed_modifier(value, 4915)`）は
  仕様書と一致しており、CLAUDE.md のハンドラ約束事にも沿っている。

## 実装（handlers/ / data/）
- 変更なし。`handlers/ability.py` の `すてみ_boost_power` は `ctx.move.has_flag("recoil")` で反動技を判定し
  `apply_fixed_modifier(value, 4915)` を適用する実装で、仕様と一致していることを確認した。
- `data/ability.py` の `すてみ` エントリは `Event.ON_CALC_POWER_MODIFIER` に `subject_spec="attacker:self"` で
  登録されており、他の威力補正特性（かたいツメ、テクニシャン等）と同じパターンになっていることを確認した。
- `"recoil"` フラグが付与されている技を全 `data/moves/*.py` から洗い出し、仕様書の対象技表16件
  （アフロブレイク、ウェーブタックル、ウッドハンマー、かかとおとし、サンダーダイブ、じごくぐるま、
  すてみタックル、とっしん、とびげり、とびひざげり、はめつのひかり、フレアドライブ、ブレイブバード、
  ボルテッカー、もろはのずつき、ワイルドボルト）と完全一致することを確認した。
- じばく/だいばくはつ/ミストバースト（`ON_PAY_HP` による自滅コスト）、わるあがき、ビックリヘッド系
  （てっていこうせん・クロロブラスト、`ON_HIT` の専用反動ハンドラ）には `"recoil"` フラグが付与されて
  おらず、正しく対象外になっていることを確認した（火傷等ダメージ由来の反動ではない技も同様に対象外）。

## リーサル計算
- 対象外（`.internal/progress/ability.md` の「リーサル実装」列が `n/a`）。すてみは通常のダメージ計算
  （`ON_CALC_POWER_MODIFIER`）経由で処理されるため `handlers/lethal.py` への個別実装は不要と判断した。

## テスト
- 変更なし。`tests/abilities/test_ability_sa.py` の `test_すてみ_反動技の威力が1_2倍になる` で
  反動技（すてみタックル・かかとおとし・とびげり・とびひざげり・サンダーダイブ）が 4915、
  反動なし技（たいあたり）が 4096（補正なし）になることを検証済み。
  `tests/moves_attack/test_move_sa.py` / `test_move_ta.py` 側でも各技の `"recoil"` フラグ付与を
  個別に確認しており、過不足なし。
- `python -m pytest tests/ -v` を実行し、すてみ関連テストはすべて成功した。
  無関係な既知flakyテスト（`test_ごりむちゅう_特殊技には効果がない` 等の乱数依存テスト）の単発失敗は
  個別再実行でパスすることを確認済み。
