## 実装（handlers/ / data/）
- **`Event.ON_SWITCH_IN` の priority を修正**: `src/jpoke/data/ability.py` のこだいかっせいが
  `priority=200` で登録されており、`.internal/spec/turn.md` が定める `140`（いかく等の入場効果より後、
  しろいハーブ等〈160〉より前）と食い違っていた。しろいハーブを装備した状態でバトンタッチにより
  下降ランクを引き継いで登場すると、本来はランクが戻る前に能力を選ぶべきところ、逆順で発動し
  ランク回復後の値で選んでしまう不具合を確認・修正した（`priority=140` に修正。
  クォークチャージは既に別レビューで同様の修正済みだったが、こだいかっせい側は未修正だった）。
- **天候の持続ターン切れ（自然消滅）で `ON_FIELD_CHANGE` が発火しない不具合を修正**:
  `src/jpoke/core/field_manager.py` の `ExclusiveFieldManager.tick_down_current`（天候・地形の
  ターン終了時カウントダウン）は、カウントが0になって効果が解除されても `current_name` の
  リセットや `ON_FIELD_CHANGE` の発火を行っていなかった（`remove()` による明示的な解除でのみ
  発火していた）。このため、にほんばれが持続ターン切れで自然に終了しても
  `refresh_paradox_charge_state` が呼ばれず、こだいかっせいの効果がいつまでも消えず、
  未消費のブーストエナジーへのフォールバックも起きない不具合があった
  （仕様書「にほんばれ状態が消滅すると一旦特性の効果が消えるが、即座にブーストエナジーを
  消費して特性を再発動させる」に違反）。`tick_down_current` がカウント0到達時に
  `remove()` と同様の後処理（`current_name` リセット・`ON_FIELD_CHANGE` 発火）を行うよう修正した。
  この修正は共通コード（天候・地形全般）に影響するため、クォークチャージ/こだいかっせいに加え、
  同様に `ON_FIELD_CHANGE` を参照するぎたい/てんきやの自然消滅時の再判定にも波及する
  （既存テストに回帰がないことを確認済み）。
- **ワンダールーム状態での実数値入れ替えが選択ロジックに未反映だった不具合を修正**:
  `ability_paradox.py` の `_select_paradox_boost_stat` は `mon.ranked_stats`（ランク補正込みの
  実数値）をそのまま比較しており、ワンダールーム状態でのぼうぎょ/とくぼうの実数値入れ替えを
  考慮していなかった（仕様書「パワーシェア/ガードシェア/パワートリック/スピードスワップ/
  ワンダールームによる実数値変動も考慮する」「実数値の変動はステータスの比較より先に考慮する」に
  違反）。`battle.get_global_field("ワンダールーム").is_active` を見て、def/spd の実数値
  （ランク補正を除く）を入れ替えて比較するよう修正した（`handlers/ability.py` の
  `ビーストブースト_boost_best_stat_on_ko` と同じ方式）。`battle` 引数を追加する必要があったため
  呼び出し元 `_activate_paradox_boost` も合わせて修正した。
- **ワンダールームの発生・解除で補正対象が入れ替わらない不具合を修正**:
  `apply_def_modifier` は技の分類（物理/特殊）だけから防御側の参照能力を決定しており、
  ワンダールーム状態で実際の防御実数値参照が入れ替わる（`handlers/field.py` の
  `ワンダールーム_def_modifier`）ことを考慮していなかった（仕様書「防御か特防が上昇している
  ときにワンダールーム状態が発生した場合や、解除された場合では、その度に補正が掛かっている
  能力も入れ替わる」に違反）。`battle.get_global_field("ワンダールーム").is_active` のときは
  判定対象の能力（def/spd）を入れ替えるよう修正した。
- 上記以外（`refresh_paradox_charge_state` の場条件・アイテム消費判定、`modify_speed`、
  `apply_atk_modifier` のこんらん自傷除外、`flags={"uncopyable"}`、スキルスワップ失敗条件、
  ブーストエナジーの授受防止）は既存のクォークチャージレビューで修正済みの共通コードであり、
  仕様と一致していたため変更なし。
