# どくのくさり レビュー結果

## 仕様書
- 変更なし。`.internal/wiki/abilities/どくのくさり.html`（一次情報）と照合し、誤り・欠落なし。

## 実装計画書
- 修正した。旧計画書は `Event.ON_DAMAGE_HIT` を使う方針だったが、実装の不備（後述）と
  同じ理由でこれは仕様を満たせないため `Event.ON_HIT` + `ctx.substitute_damage` 判定に更新した。
- 「ちからずく使用時は追加効果が発動しない」という実装ゴールの記載を削除した。ちからずくは
  特性であり、1体のポケモンが「どくのくさり」と同時に持つことはできないため、この項目は
  そもそも成立しないテスト不能な誤記だった。
- タイプ耐性・りんぷん・おんみつマントは既存の共通処理（`AilmentManager._can_apply_by_type` /
  `resolve_secondary_chance` が発火する `Event.ON_MODIFY_SECONDARY_CHANCE`）で自動的に処理される
  旨を明記した。
- みらいよち/はめつのねがい経由の発動について、現状の実装（`handlers/field.py` の
  `みらいよち_damage`/`はめつのねがい_damage`）は `battle.modify_hp` を直接呼ぶのみで
  `Event.ON_HIT`（`AttackContext`）を発火しないため対応できていない旨を「既知の制約」として
  記載した（フィールド技実装フレームワーク側の構造的な課題のため本タスクでは対応せず）。

## 実装（handlers/ / data/）
- 修正した。`src/jpoke/data/ability.py` の `どくのくさり` ハンドラ登録を `Event.ON_DAMAGE_HIT`
  から `Event.ON_HIT` に変更した。
  - `Event.ON_DAMAGE_HIT` は `actual_damage<=0` のとき発火しない
    （`src/jpoke/core/move_executor.py` の `_execute_hit`）。仕様書に明記されている
    「こらえるでHP1のまま耐えたときやばけのかわ/アイスフェイスで攻撃を肩代わりされたときなど、
    技が成功したなら与えたダメージが0だったときでもどくのくさりが発動することがある」という
    要件を満たせておらず、実際に検証したところ発動していなかった（バグ）。
    どくげしょう・すなはき・じきゅうりょくと同じ既存パターン（`Event.ON_HIT` を使用）に合わせて
    修正した。
- `src/jpoke/handlers/ability.py` の `どくのくさり_maybe_badly_poison` に
  `ctx.substitute_damage` のチェックを追加した（みがわりに阻まれたときに発動しないようにするため。
  `Event.ON_HIT` はみがわり被弾時にも発火するため、ハンドラ内で明示的に除外する必要がある）。
