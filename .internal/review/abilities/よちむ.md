## 実装（handlers/ / data/）
- **重大な不具合を修正**: `_move_power` が `move.data.power` をそのまま参照していたため、威力欄が「―」と表示される変動威力技（一撃必殺技、カウンター系、くさむすび・けたぐり・ジャイロボールなど）を実質的に威力0～1として誤判定していた。これらの技は内部データ上の `data.power` がダメージ計算用のプレースホルダ値であり、よちむの見なし威力とは異なるため、仕様書の対応表（150/120/80）を明示的なマッピングとして実装し、`_move_power` から参照するよう修正した。
  - 影響例: くさむすび（本来見なし威力80）を持つ相手に対し、修正前は威力40のたいあたり等の通常技が優先して読み取られてしまっていた。
  - じわれ・ぜったいれいど・つのドリル・ハサミギロチン（見なし威力150）、カウンター・ミラーコート・メタルバースト・ほうふく（見なし威力120）についても同様に修正。
  - しおふき・アシストパワー・テラバースト（ステラタイプ含む）など「技説明に数値が表記される技」は元々 `data.power` が表示威力と一致しており、修正の影響を受けない（既存動作を維持）。
- `よちむ_reveal_strongest_move` 内の `if mon is None: return ...` と `if not foe.moves: return ...` の重複ガードを削除した。`ON_SWITCH_IN` は必ず有効な `source` を伴って発火し、ポケモンは必ず1つ以上の技を持つため、イベント発火側で前提が保証されている（`きけんよち_warn_threat` など既存の同種ハンドラも同様に無条件で `battle.foe(source)` を使用しており、実装パターンを揃えた）。
- Priority・subject_spec・登録イベント（`ON_SWITCH_IN`）は仕様書・`.internal/spec/turn.md`（優先度100）と一致しており変更なし。
