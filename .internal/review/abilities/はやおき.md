## 実装（handlers/ / data/）
- 修正した。`handlers/ability.py` の `はやおき_extra_decrement`（`data/ability.py` の
  `ABILITIES["はやおき"]` から `Event.ON_TRY_ACTION` priority=9 で登録）は既に実装済みだったが、
  バグを発見し修正した。
  - **バグ内容**: ねごと使用時、ねごと自身の `ON_TRY_ACTION`（`sleep_talk_active=False`）で
    はやおきの追加tick + 通常tick の計2回消費した後、ねごとが選んだサブ技の実行でも
    `ON_TRY_ACTION` が再度発火する（`sleep_talk_active=True`）。既存の `ねむり_check_action`
    （`handlers/ailment.py`）はこの2回目の発火時に `sleep_talk_active` を見て通常tickをスキップ
    するガードを持っていたが、`はやおき_extra_decrement` にはこのガードが無く、サブ技実行時にも
    無条件で追加tickしてしまい、1ターンで合計3回消費される不具合があった
    （再現: `ailment0=("ねむり", 5)` ではやおき+ねごとを1回実行すると count が 5→2 になり、
    正しくは 5→3 であるべきだった）。
  - **修正**: `ねむり_check_action` と同じ `if mon.sleep_talk_active: return HandlerReturn(value=value)`
    ガードを `はやおき_extra_decrement` の先頭に追加し、ねごとのサブ技実行中は追加tickを行わない
    ようにした。修正後は再現手順で count が 5→3 となり、1ターンあたり2回消費が正しく維持される
    ことを確認した。
