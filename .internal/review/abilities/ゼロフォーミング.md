## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/handlers/ability.py` の `ゼロフォーミング_clear_field`、`src/jpoke/data/ability.py` の `Event.ON_TERASTALLIZE` への登録はすでに実装済みで、仕様と整合していることを確認した。
  - `flags={"uncopyable"}` のみで `protected` は付与されていない → 他特性による上書きは可能（仕様どおり）
  - `Event.ON_ABILITY_ENABLED` にはハンドラを登録していない → かがくへんかガス解除後に再発動しない仕様（おもかげやどしとの違い）を正しく実装している
  - `battle.weather_manager.remove()` / `battle.terrain_manager.remove()` は `ON_CHECK_WEATHER_ENABLED` や設置場所（浮遊等）を経由せず直接解除するため、ノーてんき/エアロック無効化中でも解除でき、ふうせん等で非接地でもフィールドを解除できる仕様を満たしている
  - `mon.terastallize()` で `terastallized` が恒久的に True になり `can_terastallize()` が False を返すため、テラスタルは1バトル中1回のみ・`ON_TERASTALLIZE` も1回のみ発火する（交代しても再発動しない）ことを確認した
