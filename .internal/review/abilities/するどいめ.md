# するどいめ レビュー結果

## 仕様書
- 変更なし（一次情報 `.internal/wiki/abilities/するどいめ.html` と照合し、既存の `.internal/spec/abilities/するどいめ.md` の記述内容（命中率ランク低下無効、第六世代以降の回避率ランク無視〔低下分も含む〕、ミラクルスキン等の非ランク変動には無効、フィールド効果、備考）はすべて一致していることを確認した）。

## 実装計画書
- `.internal/plan/abilities/するどいめ.md` を実際の実装内容に合わせて修正した。
  - 回避率無視の実装方針を「`ON_MODIFY_ACCURACY` で攻撃側として相手の回避率ランクを0扱い」という記述から、実際に使われている「`ON_GET_STAT_RANK`（subject_spec="attacker:self"）で命中判定直前の rank 辞書の `evasion` を常に0にする」という記述に修正した。
  - 命中率ランク低下防止の実装方針に `_block_stat_drop_by_foe` の再利用である旨を明記した。
  - テスト観点を実装内容（かたやぶりでは低下する／回避率の上昇・低下いずれも無視される）に合わせて更新した。

## 実装（handlers/ / data/）
- `src/jpoke/data/ability.py` の `するどいめ` 登録（`ON_BEFORE_MODIFY_STAT` → `するどいめ_block_ACC_drop` / `ON_GET_STAT_RANK` → `するどいめ_ignore_evasion`、`mold_breaker_ignorable` フラグ）はロジック・subject_spec とも仕様と一致しており問題なし。
- `しんがん` からも同じ2関数が再利用登録されており、既レビュー済みの `しんがん` との実装パターンの一貫性を確認した。
- `src/jpoke/handlers/ability.py` の `するどいめ_block_ACC_drop` の型ヒントが `ctx: AttackContext` になっていたが、`ON_BEFORE_MODIFY_STAT` は `EventContext`（`status_manager.modify_stats` が生成）を渡すため誤りだった。同系の `かいりきバサミ_block_A_drop` / `クリアボディ_block_stat_drop` / `はとむね_block_B_drop` に合わせて `EventContext` に修正した（実行時の挙動には影響しない型ヒントのみの修正）。
- `するどいめ_ignore_evasion` の `ctx: AttackContext` は `ON_GET_STAT_RANK`（`move_executor._check_hit` が attacker/defender ロールで発火）の実態と一致しており正しい。
- 一撃必殺技（つのドリル等）は `.internal/spec/moves/_ohko.md` の通り通常の命中ランク計算経路（`_check_hit`）を通るため、`するどいめ` の回避率無視は一撃必殺技にも正しく適用される。すながくれで見つかったような「一撃必殺技除外漏れ」バグの類似パターンは該当しない（すながくれは `ON_MODIFY_ACCURACY` の固定倍率補正であり必中技・一撃必殺技を明示的に除外する必要があるが、するどいめはランクベースの補正でそのような除外規定がない）。
