# ふくつのたて レビュー結果

## 仕様書
- 変更なし。一次情報（.internal/wiki/abilities/ふくつのたて.html）と照合し、効果（場に出たときぼうぎょ+1段階、
  第九世代では戦闘中1度のみ発動）・ふとうのけんとの対応関係・かがくへんかガスとの相互作用（じばく/
  だいばくはつでガス使用者がひんしになりガスが解除された際に発動する挙動）・備考の記述に誤り・欠落が
  ないことを確認した。

## 実装計画書
- 更新した。旧版は `mon.ability.state == "used"` による使用済み管理を疑似コードに記載していたが、
  実装の実物は `battle.add_ability_disabled_reason(mon, "consumed")` ＋ `per_battle_once` フラグ
  （`Ability.reset_on_switch_out` が交代後も "consumed" 理由を維持する仕組み）を使っており、
  スタットキーも `"B"` ではなく `"def"` だった。これらを実装に合わせて修正し、ふとうのけんの詳細仕様
  （既に最大まで上がっていて不発でも特性は消費される／かがくへんかガス解除時に再発動する）を
  参照すべき旨と、対応する `Event.ON_ABILITY_ENABLED` ハンドラ登録を明記した。
- テスト観点にかがくへんかガス関連（発動中は発動しない／解除後に発動する）と、既に最大値で不発でも
  消費される観点を追加した。

## 実装（handlers/ / data/）
- 変更なし。`data/ability.py` の `ふくつのたて` エントリ（`per_battle_once` フラグ、
  `Event.ON_SWITCH_IN` / `Event.ON_ABILITY_ENABLED` への `ふくつのたて_boost_B` 登録、
  `subject_spec="source:self"`）は `ふとうのけん` と同一パターンで仕様通りだった。
- `handlers/ability.py` の `ふくつのたて_boost_B` は `modify_stats` の成否に関わらず
  `add_ability_disabled_reason(mon, "consumed")` を先に呼んでおり、「既に最大でぼうぎょが
  上がらなかった場合でも特性は消費される」という仕様（ふとうのけんの詳細仕様を参照）を満たしている
  ことを実機的に確認した（`test_ふくつのたて_Bが既に最大でも特性は消費される` で検証）。
- `.internal/spec/turn.md` の `Event.ON_SWITCH_IN` priority表で `ふくつのたて（特性）` は Priority 100 と
  定義されており、`Handler` のデフォルト priority（100）と一致することを確認した（変更不要）。
