# ノーマルスキン レビュー結果

## 仕様書
- 変更なし。`.internal/wiki/abilities/ノーマルスキン.html`（一次情報）から再変換した内容と
  既存の `.internal/spec/abilities/ノーマルスキン.md` が完全に一致することを確認した。

## 実装計画書
- 修正した。
  - `.internal/spec/turn.md` に `ON_MODIFY_MOVE_TYPE` / `ON_CALC_POWER_MODIFIER` が未掲載であることを確認し、
    既存のスキン系特性（スカイスキン・ドラゴンスキン・フェアリースキン・フリーズスキン）と同じ既定
    priority（100）・`subject_spec: attacker:self` を採用する根拠を明記した。
  - ノーマルスキンが他のスキン系特性と変換の向きが逆（ノーマル以外→ノーマル、他は
    ノーマル→特定タイプ）であることを明記し、共通ヘルパー `_skin_modify_move_type` /
    `_skin_boost_power` を使わない理由を追記した。
  - 実装レビューで発見した「わるあがき（`type=""`）がノーマルタイプ・威力補正の対象に誤って
    含まれる」バグとその修正方針を明記した。

## 実装（handlers/ / data/）
- 修正した。`src/jpoke/handlers/ability.py` の `ノーマルスキン_modify_move_type` /
  `ノーマルスキン_boost_power` が `type=""`（わるあがき等のタイプなし技）を除外していなかった
  バグを修正した。
  - 修正前: `わるあがき` を使うと技タイプが `ノーマル` に変換され、威力も4915/4096倍に
    補正されてしまっていた（仕様書「わるあがきを使用したときにこの特性は発動しない」に違反）。
  - 修正後: `value` / `ctx.move.data.type` が `""` の場合も対象外とし、わるあがきの
    タイプ・威力は変化しないことを確認した。
  - `data/ability.py` の登録内容（イベント・priority・subject_spec）は仕様・計画と一致しており
    変更なし。
