# ぶきよう レビュー結果

## 仕様書
- 変更なし（`.internal/wiki/abilities/ぶきよう.html` の一次情報と `.internal/spec/abilities/ぶきよう.md` の記述を
  項目単位で照合し、「特性の仕様」「第五世代以降」の各箇条書きが一致していることを確認した。
  「第四世代のみ」節・「所有ポケモン」表・「こんなときに使おう」節は対象範囲（チャンピオンズ）に
  無関係な情報のため、仕様書へ転記しない方針を維持する）

## 実装計画書
- 修正した。旧計画書は `Event.ON_CHECK_ITEM_ENABLED` を新設する設計を記述していたが、実際の実装は
  `かがくへんかガス` と同じ `model/effect.py` の `_disabled_reasons` 汎用機構
  （`item_manager.add_disabled_reason` / `remove_disabled_reason` と
  `EventManager._check_handler_validity` による発火時判定）に収斂しており、計画書と実装が乖離していた。
  実装済みの設計に合わせて計画書を全面的に書き直した
  （`Event.ON_SWITCH_IN` / `ON_ABILITY_ENABLED` / `ON_ABILITY_DISABLED` への登録、
  道具ハンドラ側の `ignored_disable_reasons` による例外化など）

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/data/ability.py` の `ぶきよう` エントリ、`handlers/ability.py` の
  `ぶきよう_disable_item` / `ぶきよう_enable_item`、`item_manager.py` の
  `add_disabled_reason` / `remove_disabled_reason`、`event_manager.py` の
  `_check_handler_validity` による汎用ゲーティングを確認し、以下の仕様項目が正しく実現されていることを
  個別に検証した:
  - `くろいてっきゅう`（グラウンド化・すばやさ半減の両方を無視）
  - `だいこんごうだま` 等のフォルムチェンジ専用道具（フォルムチェンジ・威力補正が発動しない）
  - `ふうせん`（じめん技を防げない）
  - `なげつける`（使用側が失敗し道具を消費しない、現行世代仕様）
  - `とくせいガード`（`ignored_disable_reasons=frozenset({"ぶきよう"})` により例外的に発動し、
    ぶきよう自身の上書き・無効化を防ぐ）
  - `ポルターガイスト`（道具の所持自体は判定対象のため成功する）
  - `おみとおし`（無効化中の道具も公開できる）
  修正の必要な誤り・欠落は見つからなかった
- `.internal/spec/turn.md` の `Event.ON_SWITCH_IN` 優先度表に `ぶきよう（特性）` の行が漏れていたため追加した
  （実装は `AbilityHandler` のデフォルト優先度100で他の多数の特性と同格）

## リーサル計算
- 対象外（`.internal/progress/ability.md` のリーサル実装列が `n/a`。ぶきようはダメージ計算自体を変えず、
  対戦相手の道具依存効果を消す間接効果のため、現行のリーサル計算モデルの対象外という判断は妥当）

## テスト
- `tests/abilities/test_ability_ha.py` に以下を追加した:
  - `test_ぶきよう_くろいてっきゅうの効果を無視する`: グラウンド化・すばやさ半減の両方が無視されることを、
    ひこうタイプ（ピジョット）の免疫維持とすばやさ計算の両面で検証
  - `test_ぶきよう_フォルムチェンジアイテムの効果が発動しない`: `だいこんごうだま` を持つディアルガが
    オリジンフォルムにならないことを検証
- 既存テスト（`test_ぶきよう_アイテムが無効`、および `test_なげつける_ぶきようを持つ攻撃側は失敗する` /
  `test_ふうせん_ぶきようで無効化される` / `test_ポルターガイスト_ぶきようで無効化されていても成功する` /
  `test_おみとおし_ぶきようでアイテム無効化中でも公開される` / `test_とくせいガード_ぶきようでも特性無効化のブロックは発動する` /
  `test_ねらいのまと_ぶきようで効果が無効化される` 等、他ファイルに分散した既存テスト）は実装と一致しており
  修正不要と判断した
- `python scripts/sort_tests.py tests/abilities/test_ability_ha.py` と
  `python scripts/generate_test_list.py` を実行し、`.internal/tests/abilities.md` を更新した

## 既知の未対応事項
- 第七世代以降の仕様「こだわり系アイテムで技を固定した“後”に `ぶきよう` を得た場合、固定技以外を
  選ぼうとするとPP消費なしで技が失敗する」は未実装（現状は `こだわり` 揮発状態のコマンド制限が
  `ぶきよう` の影響を受けずそのまま固定され続ける）。スキルスワップ等で既にこだわり済みのポケモンへ
  `ぶきよう` を後から付与するという極めて限定的な状況でのみ顕在化する差異のため、
  `.internal/plan/abilities/ぶきよう.md` の「未対応・要確認」に明記したうえで今回は対応を見送った
