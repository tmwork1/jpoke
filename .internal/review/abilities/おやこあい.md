## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `おやこあい_reduce_second_damage` に最低1ダメージ保証が抜けていたバグを修正した
  - 修正前: `value //= 4`（1ヒット目のダメージが1〜3の場合、2ヒット目が0ダメージになってしまう）
  - 修正後: `value = max(1, value // 4)`
  - `.internal/spec/damage.md` の「最低保証: タイプ相性0以外 → 最低1ダメージ」、および同一特性のリーサル計算実装（`handlers/lethal.py` の `_add_second_hit` は既に `max(1, state.value // 4)` を使用）と整合させた
- その他のハンドラロジック（ヒット数2化の対象判定、命中判定・急所判定・乱数判定が2回とも独立して行われる既存アーキテクチャとの整合性、PPが1回分しか消費されない点など）は仕様書と一致していることを確認した
- `Event.ON_MODIFY_HIT_COUNT` に既に登録されているスキルリンク・いかさまダイスとの適用条件排他性を確認し、優先度競合がないことを確認した（`priority`は両ハンドラともデフォルト100のまま）

## 既知の未対応エッジケース（今回は対象外）
- 仕様書の「親の攻撃で相手の特性やアイテムが発動しておやこあい側がねむり状態になった場合、子の攻撃は発動しない」（例: ほうし特性による接触時のねむり付与）は、現在の共有ヒットループ（`MoveExecutor._execute_move`）が `fainted` のみをチェックしており、ターン中に新規付与された「ねむり」状態を検知していないため未対応。この対応には「今回の行動中に新規に状態異常が付与されたか」を判定する状態追跡の追加が必要であり、共有ループ（他の連続技にも影響する）への変更を伴うため、本レビューのスコープ外として見送った。将来的な対応が必要な場合は別タスクとして計画すること。
