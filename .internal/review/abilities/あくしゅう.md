# あくしゅう レビュー結果

## 仕様書
- 既存の `.internal/spec/abilities/あくしゅう.md` は一次情報（ポケモンWiki）と照合し概ね一致していたが、以下を補足した。
  - 「元々ひるみの追加効果がある技を使用してもあくしゅうの効果は発動しない」の項に、対象技は
    `.internal/spec/items/おうじゃのしるし.md`（おうじゃのしるし/するどいキバと同一仕様）の一覧を参照する旨を追記。
  - 一撃必殺技（じわれ等）にはあくしゅうの効果が無い旨を追記
    （道具のおうじゃのしるし/するどいキバと同一仕様。一次情報のあくしゅうページには明記が無いが、
    あくしゅうページ自身が「バトルでの効果は道具のおうじゃのしるし/するどいキバと同じ効果がある」と
    明記しているため、道具側の仕様書に基づき補足した）。

## 実装計画書
- `.internal/plan/abilities/あくしゅう.md` を更新した。
  - 実装ゴールに「元々ひるみの追加効果を持つ技・一撃必殺技には重複して発動しない」を追加。
  - 実装方針に `_INNATE_FLINCH_MOVES` リストと `ctx.move.has_flag("ohko")` による除外ロジックを明記
    （`handlers/item.py` の `flinch_on_hit_10pct` と同一パターン）。
  - テスト観点に「元々ひるみ効果がある技には重複しない」「一撃必殺技には効果が無い」を追加。
  - 「既知の未対応ケース」の節を新設し、HP1+こらえる/ばけのかわ/アイスフェイス等で実ダメージ0でも
    あくしゅうが発動しうる仕様が、現行の `Event.ON_DAMAGE_HIT`（`actual_damage > 0` のみ発火）の
    アーキテクチャ上未対応であることを明記した（`おうじゃのしるし`/`するどいキバ` 等
    `ON_DAMAGE_HIT` を使う道具全般に共通する既知の制約であり、本レビューの対応範囲外とした）。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `あくしゅう_maybe_flinch` にバグを発見し修正した。
  - **修正前**: 攻撃技でダメージを与えるたびに無条件で10%判定していたため、元々ひるみの追加効果を
    持つ技（アイアンヘッド・エアスラッシュ等）や一撃必殺技（じわれ等）でも重複してあくしゅうの
    ひるみ判定が行われてしまっていた（`handlers/item.py` の `おうじゃのしるし`/`するどいキバ`
    （`flinch_on_hit_10pct`）が既に持っている `_INNATE_FLINCH_MOVES` 判定・`ohko` フラグ判定が
    あくしゅう側には存在しなかった）。
  - **修正後**: `handlers/item.py` の `_INNATE_FLINCH_MOVES`（同一のフローズンセットをミラー）と
    `ctx.move.has_flag("ohko")` の除外判定を追加し、道具版と同じ挙動になるよう修正した。
- `src/jpoke/data/ability.py` の登録内容自体は仕様通りで変更なし。
