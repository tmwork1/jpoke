# みずがため レビュー結果

## 仕様書
- 変更なし。一次情報（https://wiki.pokemonwiki.com/wiki/みずがため）と照合したが、既存の
  `.internal/spec/abilities/みずがため.md` の記述は効果・特性の仕様（発動条件・タイプ変更技への
  対応・連続攻撃技・アクアブレイク/シェルブレードの発動順序など）を含めすべて一致しており、
  誤り・欠落はなかった。

## 実装計画書
- 更新日を更新し、以下を補充した。
  - こんがりボディとの比較を修正: 発動条件は近いがこんがりボディはダメージ自体を無効化する
    吸収技で `Event.ON_BEFORE_APPLY_MOVE` 実装のため、みずがための「ダメージを受けた後に
    発動する」`Event.ON_DAMAGE_HIT` 実装とは異なる旨を明記（実装パターンとしてはくだけるよろい・
    てつのトゲ等の ON_DAMAGE_HIT priority=20 グループが近い）。
  - アクアブレイク/シェルブレードの追加効果とみずがための発動順序（priority=10 が
    priority=20 より先に発動する必要がある）を実装方針・変更候補・テスト観点に追記。
  - 変更候補・検証コマンドのテストファイルパスを実体に合わせて `tests/test_ability.py` →
    `tests/abilities/test_ability_ma.py` に修正。
  - 疑似コードの関数名・stats キーを実装（`みずがため_boost_B_on_water`、`{"def": +2}`）に
    合わせて修正。

## 実装（handlers/ / data/）
- `みずがため_boost_B_on_water`（`handlers/ability.py`）・`data/ability.py` の登録内容自体は
  仕様どおりで変更なし。
- **不具合を発見し修正**: 仕様書に明記されている「アクアブレイク/シェルブレードを受けた場合、
  追加効果の後にみずがためが発動する」という順序が、実装では逆になっていた。
  - `アクアブレイク`（`data/moves/move_a.py`）・`シェルブレード`（`data/moves/move_sa.py`）の
    `ON_DAMAGE_HIT` ハンドラ（ぼうぎょ低下の追加効果）が `priority` 未指定のためデフォルトの
    `100` で登録されており、みずがための `priority=20` より後に実行されていた
    （`.internal/spec/turn.md` の ON_DAMAGE priority=10「追加効果（特殊なもの除く）」と矛盾）。
  - 実際に B=+5 の状態からアクアブレイクを受けさせて検証したところ、修正前は
    みずがための B+2 が先に適用され +6 にクランプされた後、アクアブレイクの B-1 が
    適用されて最終的に B=+5 となっていた（仕様上は追加効果 B-1 が先: 5→4、その後
    みずがための B+2 でクランプされず 4→6 になるべき）。
  - `アクアブレイク_lower_defender_def` / `シェルブレード_lower_defender_def` の
    `ON_DAMAGE_HIT` ハンドラに `priority=10` を明示し、仕様書・turn.md を根拠として
    コメントを追加して修正した。
