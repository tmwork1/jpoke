# ステルスロック レビュー結果

## 仕様書

- 新規作成（`.internal/spec/fields/ステルスロック.md`）
- wiki の内容をもとに以下を記載:
  - ダメージ計算テーブル（×1/4=1/32、×1/2=1/16、×1=1/8、×2=1/4、×4=1/2）
  - マジックガード・あつぞこブーツによる免疫
  - らんきりゅう下でひこうタイプのダメージが変化しない旨
  - 解除条件（こうそくスピン・きりばらい・おかたづけ・キョダイフウゲキ）

## 実装計画書

- 新規作成（`.internal/plan/fields/ステルスロック.md`）
- ハンドラの実装方針（TYPE_MODIFIER 直接参照、イベント経由回避）を記載
- `// 4096` 整数除算を避ける旨の注意点を明記

## 実装（handlers/ / data/）

- `src/jpoke/handlers/field.py` の `ステルスロック_damage` を修正
  - **バグ修正**: `calc_def_type_modifier(tmp_ctx) // 4096` の整数除算により、×1/4（例: ルカリオ）と ×1/2（例: カイリキー）の場合にダメージが 0 になっていた
  - 修正内容: `AttackContext` と `ON_CALC_DEF_TYPE_MODIFIER` イベントを経由するのをやめ、`TYPE_MODIFIER["いわ"]` から直接タイプ相性を計算する
  - 副次効果: らんきりゅう下でも正しいダメージが計算されるようになった

## テスト

- `test_ステルスロック_x1_2`: ×1/2耐性（カイリキー）→ 最大HPの1/16ダメージ を追加
- `test_ステルスロック_x1_4`: ×1/4耐性（ルカリオ）→ 最大HPの1/32ダメージ を追加
- `test_ステルスロック_マジックガード無効`: マジックガード持ちはダメージなし を追加
- `test_ステルスロック_x4` のdocstringを「×4弱点ポケモン（リザードン）」と明確化
