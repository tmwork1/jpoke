# トライアタック（Tri Attack）仕様書

参照: .internal/wiki/moves/トライアタック.html（ポケモンWiki）

## 基本情報

* タイプ: ノーマル
* 分類: 特殊
* 威力: 80
* 命中: 100
* PP: 12（Champions基準。src/jpoke/data/ps-champ-ja/moves.json 準拠。Gen9本家は10）
* 対象: 相手1体
* 優先度: 0
* 接触: なし

## 効果

* 追加効果として、20%の確率で相手を『まひ』『やけど』『こおり』のいずれかの状態にする
  （第一世代は追加効果なし。第二世代以降で追加効果が発生する。本プロジェクトの対象世代では20%）。
* まひ・やけど・こおりの3つの状態異常はそれぞれ等確率（各1/3）で選ばれる。
  単一の乱数で判定する（3回独立抽選ではない）: `r = random()` として
  `r < base/3` → まひ、`base/3 <= r < base*2/3` → やけど、`base*2/3 <= r < base` → こおり、
  `r >= base` → 追加効果なし（`base` は `resolve_secondary_chance` で補正後の発動確率）。
* 各状態異常は通常の状態異常付与ルールに従う（タイプ免疫・特性・既に状態異常の場合の無効化等は
  `ailment_manager.apply` 内部で処理される）。

## 判定フラグ

* 急所: 通常判定
* 命中判定: あり（命中率100）
* 追加効果: あり（20%・まひ/やけど/こおりのいずれか、各1/3）
  * 特性りんぷん・持ち物おんみつマントを持つ相手には発動しない（`resolve_secondary_chance` 経由）。
  * 特性ちからずくのポケモンが使用すると威力が1.3倍になるが追加効果は発動しなくなる
    （`secondary_effect` フラグによる汎用処理）。
  * 特性てんのめぐみで発動確率が2倍になる（`resolve_secondary_chance` 経由）。
* まもる: 防がれる
* おうじゃのしるし: 第五世代以降は対象（この技自体はひるみ効果を持たないため、
  おうじゃのしるしによる10%ひるみ追加効果は通常通り重複して発動しうる。アイテム側の
  `おうじゃのしるし` 実装で汎用処理される）
* マジックコート: 反射不可（変化技ではないため対象外）
* よこどり: 対象外（変化技ではないため）
* オウムがえし: 反射可能
