# ほおばる（Stuff Cheeks）仕様書

参照: .internal/wiki/moves/ほおばる.html（ポケモンWiki）

## 基本情報

* タイプ: ノーマル
* 分類: 変化
* 威力: なし
* 命中: なし（必中扱い）
* PP: 12（`src/jpoke/data/ps-champ-ja/moves.json` L812 準拠。本家シリーズ（第八世代〜SV）のPPは10だが、
  Championsでは12に統一されている）
* 対象: 自分
* 優先度: 0
* 直接攻撃: なし

## 効果

* 自分が「きのみ」を持っている場合、そのきのみを即座に強制消費してその効果を発動させ
  （HPしきい値等の通常の発動条件を無視して発動する）、さらに自分の防御ランクを2段階上げる。
* きのみを持っていない場合は技が失敗する。
* Champions説明文: 「持っているきのみを食べる｡ 自分の防御を2段階上げる｡ きのみを持っていないと
  この技は使えない｡」

## 技の仕様（wikiより）

* きのみを持っていないときにほおばるを選ぼうとした場合:
  * 本家シリーズ（SVまで）では「きのみを 持っていないので 技がだせない!」というメッセージが出て
    選択がキャンセルされる。他に選べる技が無い場合はわるあがきになる。
  * **Pokémon Champions**では技を選ぶことができる。ターン開始時点できのみを持っていなくても、
    自分の行動までにきのみを入手できれば技を成功させられる。
    * 本プロジェクトの対象はChampionsシングルバトルのみのため、「選択自体を禁止する」本家仕様は
      実装せず、実行時（`Event.ON_TRY_MOVE_1`）に条件判定を行うだけでよい（`ゲップ`と同じ方針。
      `.internal/spec/moves/ゲップ.md` 参照）。
* ほおばるを使用したときは、持ち物の発動を封じる効果を無視してきのみを食べることができる。
  * 使用者が相手のきんちょうかんの影響を受けているときでもきのみを食べることができる。
    → 本プロジェクトの実装では、強制発動時（`value >= mon.max_hp`）に `is_nervous` チェックを
    バイパスする既存の共通ロジック（`src/jpoke/handlers/item.py` の `_heal_berry` 等）により
    自然に満たされる。
  * 使用者の特性がぶきようであるときや、場がマジックルーム状態であるときでもきのみを食べることが
    できる。
    → **既知の実装上の制限**: 本プロジェクトの `Item.enabled` ゲート機構（`ぶきよう`/
    `マジックルーム` 下では `item.name` が空文字列扱いになり `is_berry()` が False を返す）
    により、このケースでは効果発動だけでなく消費自体もスキップされる。この制限は `おちゃかい`
    にも共通する既存の未対応エッジケースであり（`.internal/spec/moves/おちゃかい.md` 参照）、
    本技のレビューでは踏み込んだ修正を行わない。
* ほおばるできのみを食べて効果が発動することで特性ほおぶくろが発動する。
  * 半減のきのみ/イバンのみ/ジャポのみ/レンブのみを食べてもきのみの効果は発動しないが、
    ほおぶくろは発動する。
  * 状態異常を回復するきのみを状態異常でないときにほおばるで食べてもほおぶくろは発動しない。
  * 基礎ポイントを下げるきのみをほおばるで食べてもほおぶくろは発動しない。
  * → 特性ほおぶくろは本プロジェクトでは未実装（`AbilityData()` の空データ）のため、
    相互作用の実装は対象外。
* ほおばるできのみを食べることでゲップが使用可能になる。
  * 半減の実/イバンのみ/ジャポのみ/レンブのみなど、バトルにおける効果があるきのみを食べたなら、
    発動/非発動を問わずゲップを使用することができる。
  * 基礎ポイントを下げるきのみはバトルにおける効果が無いので、食べるアニメも発動せず、
    ゲップも使用できない。
  * → 本プロジェクトの `battle.item_manager.consume_item()` は消費時に無条件で
    `mon.ate_berry = True` を設定するため、既存の `ゲップ` 実装と自然に整合する
    （努力値操作系のきのみは本プロジェクト未対応のため対象外）。
* すでに自分の防御ランクが最大まで上がっているときに使うと失敗し、きのみも消費されない。
  * あまのじゃく持ちの場合は防御上昇効果が下降に反転するため、防御ランク-6のときに失敗し、
    +6のときは成功してきのみを消費し防御ランクが下がる（`.internal/spec/abilities/あまのじゃく.md`）。
  * 防御ランクが+4〜+5のときにリュガのみ/アッキのみを持ってほおばるを使うと、先にほおばるの
    効果で防御が+6段階になるので、きのみとほおぶくろの効果は発動しないが、きのみは消費される。
    このときもゲップは使用可能になる。
    → `battle.modify_stats` の内部でランク上限チェックが行われるため、ほおばる側での
    追加実装は不要（既存実装のまま自然に満たされる）。
* ほおばるで消費したきのみはものひろい/キョダイサイセイで復活する。使用者ならばしゅうかく/
  リサイクルでも復活させることや、はんすうで再使用することができる。
  → 各特性・技のハンドラが管理するため、ほおばる側での追加実装は不要。
* ゲップとは異なりまねっこ/ねごとで選ばれる。使用者がきのみを持っていれば成功するが、
  持っていなければ失敗する。
  → ゲップと異なり `non_negoto`/`non_copycat` フラグを付与しない。
* 持ち物に影響を与える技の1つだが、自身の持ち物にしか影響しないのでマックスレイドバトル/
  テラレイドバトルでも使用できる。→ レイドバトル自体が本プロジェクト対象外のため考慮しない。

## 判定

* 急所: なし
* 命中判定: なし（必中扱い）
* 追加効果: なし
* まもる: 効果なし（まもるで防がれない）
* おうじゃのしるし（第五世代以降）: 効果なし
* マジックコート: 反射されない

## イベント / 優先度

* `Event.ON_TRY_MOVE_1` priority=30: きのみを持っていない場合に失敗させる
  （`.internal/spec/turn.md` の `Event.ON_TRY_MOVE_1` テーブルに「ほおばる: きのみを持っていない」が
  priority 30 として明記されている）。
* `Event.ON_TRY_MOVE_2` priority=130: 防御ランクが既に上限（あまのじゃく考慮）の場合に失敗させる
  （`.internal/spec/turn.md` の `Event.ON_TRY_MOVE_2` テーブルに「ほおばる: ぼうぎょ最大」が
  priority 130 として明記されている）。
* `Event.ON_STATUS_HIT`（デフォルト優先度）: きのみを強制消費して効果を発動し、防御ランクを
  2段階上げる。

## リーサル計算

* 対象外（`.internal/progress/move.md` のリーサル実装列は n/a）。自己強化技であり、ダメージ計算・
  被ダメージ量には直接影響しないため。
