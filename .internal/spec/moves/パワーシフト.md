## 実在性・効果内容の確認

- ポケモンWiki（`.internal/wiki/moves/パワーシフト.html`、記事ID 55786）に掲載があり、
  `Pokémon LEGENDS アルセウス` および `スカーレット・バイオレット` の両方に登場する正規の技として
  記載されている
- `src/jpoke/data/ps-champ-ja/moves.json` （Pokemon Showdown champions mod 由来）にも
  `num: 829, type: "ノーマル", category: "Status", pp: 12, target: "self"` として掲載されており、
  技として実装されている（Showdown 側の技コードとして機能する）ことが確認できる
- ウィキ「備考」節: 「スカーレット・バイオレットでは習得できるポケモンがおらず、ゆびをふるでも
  出現しないため使用手段が無いが、カテゴリ:スカーレット・バイオレットで使用不可能な技として
  設定されているわけではない」。すなわち **SVにおいて現在この技を習得できるポケモンは存在しない**
  が、技自体はゲーム内部データ・エンジンとして正規に実装されている（グリッチ技や使用不可能技の
  扱いではない）。README.md の対象範囲「第9世代で実装されていない技は実装しない」の基準に照らすと、
  本技はSV内部で実装されている技に該当するため実装対象となる
- 効果はポケモンWiki「技の仕様」節に明記: 「パワートリックと同じ効果を持つ」

## 効果

使用者自身の **こうげき（A）・ぼうぎょ（B）の実数値** を入れ替える。パワートリックと完全に同一の効果。

- ランク補正（`rank["atk"]` / `rank["def"]`）は一切変化しない。実数値のみを交換する。
- 実数値は種族値・個体値・努力値・性格補正・ランク補正等をすべて含んだ最終的な計算値であり、
  この技はその「攻撃実数値」と「防御実数値」そのものを入れ替える。
- 相手のこうげき・ぼうぎょには一切影響しない（対象は自分のみ）。
- 同じポケモンがもう一度パワーシフト（またはパワートリック）を使用すると、入れ替えた実数値が
  元に戻る（2回使用で元の状態に戻る。ウィキ「パワーシフト/パワートリックを使用したポケモンが
  もう一度パワーシフト/パワートリックを使用するとステータスは元に戻る」）。
- 場を離れる（交代する）と、実数値は種族値ベースで再計算され元の状態に戻る
  （`Pokemon.reset_on_switch_out()` が `update_stats()` を呼ぶため）。

## 判定

- 急所: なし（変化技）
- 命中判定: なし（必中技）
- 追加効果: なし
- まもる: 防がれない。`target="self"` のため `Move.is_blocked_by_protect`
  （`target == "foe"` が条件）が自動的に False になる
- マジックコート・マジックミラー: 跳ね返されない。`target="self"` のため
  `Move.is_reflectable`（`target in ("foe", "foe_side")` が条件）が自動的に False になる
- みがわり: 対象が自分自身のため、みがわりの有無に関係なく発動する

## イベントフロー

```
ON_STATUS_HIT (priority=100, MoveHandler デフォルト)
  → パワートリック_swap_stats（パワートリックとハンドラ関数を共有）
    → attacker.get_raw_stat(1) ⇔ attacker.get_raw_stat(2)
      （インデックス1=こうげき、インデックス2=ぼうぎょ）
```

パワーシフトとパワートリックは効果が完全に同一（ウィキ本文に明記）であるため、
`src/jpoke/handlers/move_status.py` の `パワートリック_swap_stats` 関数を両技で共有する
（`あさのひざし_heal_self` を複数の技で共有する既存パターンと同様）。

直接代入のため `ON_MODIFY_STAT` イベントは発火しない。ランク変化トリガー系の特性
（かちき・まけんき等）は発動しない。これはゲーム仕様上も正しい挙動である
（パワーシフトは「ランク変化」ではなく「実数値交換」の技のため）。
