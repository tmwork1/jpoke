## 効果

自分が **ねむり** 状態であるときのみ成功する。自分が覚えている技の中から `non_negoto` フラグを
持たない技をランダムに1つ選び、その技をそのまま実行する。

- ねごと自体のPPのみ1消費する。選ばれた技のPPは消費しない。
- 急所・命中判定・追加効果・まもるなどの判定はすべて選ばれた技に依存する。

## 失敗条件

1. 使用者が **ねむり** 状態でない場合、失敗する。
   - ねむり状態回復の判定はねごとを出す前に行われるため、ねごとを選択したターンに起きた場合は失敗する。
   - 特性 `ぜったいねむり` の場合は常にねむり状態として扱われるため、ねごとは常に使用条件を満たす。
   - この失敗判定は **PP消費より後**（`ON_TRY_MOVE_1`）で行われるため、失敗してもねごと自体のPPは消費される。
2. `non_negoto` フラグを持たない技を1つも覚えていない場合、失敗する（この場合もPPは消費される）。

## 選ばれない技（`non_negoto` フラグ）

以下に該当する技はねごとで選ばれない：

- ねごと自身
- 他の技が出る技（オウムがえし・さきどり・しぜんのちから・ねこのて・まねっこ・ゆびをふる 等）
- スケッチ・ものまね
- 溜め技（そらをとぶ・あなをほる・ダイビング・ソーラービーム・ソーラーブレード・
  エレクトロビーム・メテオビーム・フリーズボルト・コールドフレア・ゴッドバード・
  ゴーストダイブ・シャドーダイブ・とびはねる 等）
  - はめつのねがい・みらいよちは溜め技ではなく「相手陣営に設置して即座に行動終了する」技のため、
    ねごとで選ばれる（対象外にしない）
- がまん・さわぐ
- きあいパンチ・くちばしキャノン・トラップシェル
- ゲップ
- ダイマックスほう（champions対象外のため実装不要）
- おいわい・てをつなぐ
- わるあがき

（オウムがえし・さきどり・しぜんのちから・ねこのて・がまん・トラップシェルは本プロジェクト未実装のため、
実装時に `non_negoto` フラグを付与すること）

## 選ばれる技（他の「他のわざが出るわざ」との違い）

まもる状態になる技、相手を交代させる技（ほえる等）、反射技（カウンター等）、ちゅうもくのまと状態と
なる技、みらいよち・はめつのねがい、さいはい、てだすけ、フェイント、きょじゅうざん・きょじゅうだん、
ハッピータイム・てかげん、かなしばり・いちゃもん・ふういん状態により出せない技など、多くの
「他のわざが出るわざ」と異なりねごとでは選択・成功しうる。優先度が異なる技でもねごとと同じ優先度
（0）で使用される。

## PPが0の技が選ばれた場合

選ばれた技の残りPPが0であっても成功する。ねごとで選ばれた技のPPは消費されないため、
0のまま実行される。通常の技実行では残りPP0の技はわるあがきに置き換わるが、ねごと経由の
サブ実行ではこの置き換えを行わない。

## こだわり系アイテムとの相互作用

こだわり系アイテム（こだわりスカーフ・こだわりハチマキ・こだわりメガネ）を持った状態でねごとを
使用した場合、ロックされる技は **ねごとで選ばれた技ではなく「ねごと」自体** である
（第五世代以降の仕様。本実装ターゲット）。
そのため、こだわり系アイテムを持っていても次のターン以降もねごとを選択・成功させることができる。

## イベントフロー

```
（ON_TRY_ACTION 通過直後に move_executor が ねごと自体のPPを1消費する）

ON_TRY_MOVE_1 (priority=30, attacker:self)
  → ねむり状態でなければ失敗 (stop_event=True)
    （この時点ではすでにPP消費済みのため、失敗してもねごとのPPは消費される）

ON_MODIFY_PP_CONSUMED (attacker:self)
  → ねごとのサブ実行中（sleep_talk_active）は選ばれた技のPP消費を0にする

ON_STATUS_HIT (attacker:self)
  → non_negotoフラグを持たない技からランダムに1つ選択
  → 候補技がない場合は失敗 (stop_event=True)
  → sleep_talk_active フラグを立てた上で battle.run_move(attacker, chosen) を実行
    - 実行中はねごと自身の ON_TRY_MOVE_1 / ON_STATUS_HIT ハンドラを一時的に解除する
      （再帰・自己チェックの誤発火を防ぐため）
  → 実行後 sleep_talk_active フラグを戻す
```

`Event.ON_TRY_MOVE_1` を採用する理由: `.internal/spec/turn.md` の `Event.ON_TRY_MOVE_1` テーブルに
「30 | いびき・ねごと: ねむり状態でない」と明記されており、姉妹技である `いびき` も同イベント・
同priorityでねむりチェックを実装している。`Event.ON_TRY_ACTION` はPP消費より前に発火するため、
そちらに置くと「ねむり状態でないために失敗した場合もPPは消費される」という wiki 仕様を
満たせない。

## 実装上の注意点

- `move_executor.run_move` の「PP0の技はわるあがきに置き換える」処理は
  `attacker.sleep_talk_active` が真の間はスキップする（コア側の分岐）。
- こだわり系アイテムの `こだわり_lock_move`（`ON_MOVE_END`）は `sleep_talk_active` が真の間は
  ロックしない。ねごと自身の `ON_MOVE_END`（サブ実行完了後、`sleep_talk_active=False` に戻ってから
  発火する）でロックが行われるため、ロック対象は自然に「ねごと」自体になる。
- `_execute_move` 内で `non_negoto` フラグを持つ技（ねごと自身）は `last_used_move_name` を
  更新しない。サブ実行された技（通常は non_negoto を持たない）が `last_used_move_name` を
  更新するため、オウムがえし等の「最後に使われた技」参照は選ばれた技を正しく指す
  （コア側で既に対応済み）。
- `handlers/ailment.py` の `ねむり_check_action`（`ON_TRY_ACTION` で発火し、ねむりのターン
  カウントを消費する）は `sleep_talk_active` が真の間は `tick` を呼ばない。
  `battle.run_move(attacker, chosen)` によるサブ実行でも `ON_TRY_ACTION` が再度発火するため、
  ガードなしだと1ターンでねむりカウントが2回消費されてしまう。

## ポケモンチャンピオンズ向けの簡略化

- 第三世代限定の仕様（残りPP0技の失敗・スケッチ/ものまね不可コピー等）は対象外
- Zワザ関連の付加効果は実装しない
- コンテスト効果は実装しない
- 「選ばれない技しか覚えていないためにねごとが失敗した場合はこだわり系アイテムでロックされない」という
  更に細かい仕様（Gen5以降）は、現状のコアアーキテクチャでは対応していない
  （`_execute_move` は `ON_STATUS_HIT` の成否に関わらず `ON_MOVE_END` を発火するため）。
  ねごとの候補が0件になるケース自体が非常に稀（覚えている技全てが `non_negoto` の場合のみ）
  であるため、初期実装では対象外とする。
