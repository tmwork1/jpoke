## 効果

相手を **アンコール** 状態（揮発性状態）にする。
アンコール状態のポケモンは、最後にPPを消費した技しか選択・使用できなくなる。
ねごとで実行されたサブ技はPPを消費しないため対象にならず、ねごと自身が判定対象になる。

- 相手がまだ一度もPPを消費する行動をしていない（`pp_consumed_move` が `None`）
  場合は失敗する。
- 相手が最後にPPを消費した技が `non_encore` ラベルを持つ場合は失敗する
  （わるあがき・アンコール・ねごと・へんしん・ものまね・スケッチ・オウムがえし・
  ダイマックスほう等。詳細は `.internal/spec/volatiles/アンコール.md` を参照）。
  PP消費後に失敗した技（ねむり状態でないのに使ったねごと等）も判定の対象になる。
- すでにアンコール状態の相手には失敗する（重複付与不可）。

揮発性状態「アンコール」自体の詳細仕様（コマンド制限・技固定・解除条件など）は
`.internal/spec/volatiles/アンコール.md` を参照。

## 判定

- 急所: なし（変化技）
- 命中判定: あり（100%、回避率等の補正を受ける）
- 追加効果: なし（本体効果が100%発動するため「追加効果」扱いではない）
- まもる: 防がれる（`Move.is_blocked_by_protect` は対象が `foe` の変化技を自動的に対象とするため、専用フラグは不要）
- マジックコート: 跳ね返される（`マジックコート_reflect` が「対象が相手・変化技」であるすべての技を汎用的に跳ね返すため、専用対応は不要）
- みがわり: 貫通しない（`flags` に `bypass_substitute` を設定していないため、`みがわり_immune` によりみがわり状態の相手には無効）

## イベントフロー

```
ON_BEFORE_APPLY_MOVE (priority=100, .internal/spec/turn.md の ON_STATUS_HIT「アンコール: 失敗条件」に対応)
  → アンコール_can_apply
    → move = ctx.defender.pp_consumed_move
    → move が None、または non_encore ラベルを持つ場合は失敗
ON_STATUS_HIT (priority=100, MoveHandler デフォルト)
  → アンコール_apply
    → move = ctx.defender.pp_consumed_move
    → volatile_manager.apply(defender, "アンコール", count=3, move_name=move.name)
```

持続ターン数（3ターン）は Pokémon Champions 仕様として固定。
原作の「行動後に受けた場合はターン数が変動する」という世代差分は実装を単純化して省略している。
