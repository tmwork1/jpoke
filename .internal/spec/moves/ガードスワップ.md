## 効果

使用者と相手の **ぼうぎょ（B）・とくぼう（D）のランク補正** を入れ替える。
実数値（種族値・努力値・個体値・性格補正等）は一切変化せず、ランク変化量のみが
両者間で交換される。

- こうげき（A）・とくこう（C）・すばやさ（S）・命中率・回避率のランクは対象外。
- 交換後もランクは常に ±6 の範囲に収まる（既存のランク値同士を入れ替えるだけのため、
  範囲外になることはない）。
- 双方のランクが同値（0 同士等）の場合は見かけ上変化しない。

## 判定

- 急所: なし（変化技）
- 命中判定: なし（必中技。ただし「姿を隠している」相手（そらをとぶ中の相手等）には当たらない）
- 追加効果: なし（本体効果がそのまま発動するため「追加効果」扱いではない）
- まもる: 防がれる（`Move.is_blocked_by_protect` は対象が `foe` の変化技を自動的に対象とするため、専用フラグは不要）
- マジックコート: 跳ね返されない。`MoveData` に `flags={"unreflectable"}` を設定する必要がある
  （デフォルトでは対象が `foe` の変化技はすべて跳ね返り対象になるため、明示的な除外が必要）
- みがわり: 貫通する。`MoveData` に `flags={"bypass_substitute"}` を設定する必要がある

## イベントフロー

```
ON_STATUS_HIT (priority=100, MoveHandler デフォルト)
  → ガードスワップ_swap_ranks
    → attacker.rank["def"] ⇔ defender.rank["def"]
    → attacker.rank["spd"] ⇔ defender.rank["spd"]
```

`battle.modify_stats()` はランク変化量（デルタ）を加算するAPIのため、
「現在値そのものを入れ替える」目的には使用できない。`Pokemon.rank` dict の
該当キーを直接入れ替える方式を採る（`パワースワップ`・`ハートスワップ` と同一パターン）。

直接代入のため `ON_MODIFY_STAT` イベントは発火しない。これにより、クリアボディ・
しろいけむり・フラワーベール・メタルプロテクト・ミラーアーマー等のランク低下無効特性や、
しろいきり・クリアチャームの影響を受けずにランクを入れ替える（ゲーム仕様上正しい挙動）。
同様にあまのじゃく・たんじゅんによる反転も発生しない。
まけんき・かちき・だっしゅつパック（ランク低下トリガー）や、びんじょう・ものまねハーブ
（ランク上昇トリガー）も発動しない。
