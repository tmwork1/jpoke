# ニードルガード (状態変化) 仕様書

対象: Pokémon Champions シングルバトル

## 概要

ニードルガードとは揮発性状態の一つ。
技「ニードルガード」を使用することで付与される（自分対象）。
そのターンに受ける攻撃技・変化技を無効化し、直接攻撃をしてきた相手に相手の最大HPの1/8のダメージを与える。
第六世代以降の技。

## 技の基本情報

| 項目 | 値 |
|------|-----|
| タイプ | くさ |
| 分類 | 変化 |
| PP | 8（champions基準。src/jpoke/data/ps-champ-ja/moves.json 628行目参照。Gen9本家は10） |
| 優先度 | +4 |
| 範囲 | 自分 |
| 直接攻撃 | なし |
| 急所 | なし |
| 命中判定 | なし |
| 追加効果 | なし |
| まもる貫通 | × |

## 効果

- 使用したターンに受ける**攻撃技・変化技**を無効化する（まもると同様に変化技も防ぐ）
- 直接攻撃技を防いだ場合、攻撃者に攻撃者自身の最大HPの1/8ダメージを与える
  （端数切り捨て、切り捨てた結果0になる場合は1ダメージ）

## 付与条件

- 技「ニードルガード」を使用したとき（自分に付与、count=1）

## 消滅条件

- ターン終了時に解除される（count=1 のため、使用したターンのみ有効）
- **交代**によって解除される

## 詳細仕様

### 攻撃技・変化技の無効化

- そのターンに受ける攻撃技（`is_attack=True`）および変化技（`is_attack=False`）を無効化する
- **変化技も防げる**（`protect_non_attack=True`）
  - まもる / みきり / ニードルガード / トーチカ が変化技を防ぐ
  - キングシールド / かえんのまもり / スレッドトラップ / ブロッキング とは異なる
- まもる系と同じ `ON_TRY_MOVE_1` イベントで処理される（priority=100）

### 接触カウンターダメージ

- 直接攻撃（`is_contact=True`、かつ `ぼうごパット` 等の反応阻害要因を考慮した
  `query.is_contact_reaction` で判定）を防いだ場合のみ、攻撃者に攻撃者自身の最大HPの1/8
  ダメージを与える
  - 非接触の攻撃技（れいとうビーム等）を防いでもダメージは発生しない
  - タイプ相性・特性でダメージが無効な直接攻撃技であっても、第七世代以降はダメージを与えられる
    （第六世代ではダメージを与えられなかったが、本プロジェクトは第九世代基準のため対象外）
  - 連続攻撃技かつ直接攻撃でも、ダメージは1回だけ発動する（`_run_protect` 内で1回だけ
    `modify_hp` を呼ぶため重複発生しない）
  - `ぼうごパット` を持つ攻撃者は反応判定が False になるためダメージを受けない
    （`query.is_contact_reaction` が `ON_CHECK_CONTACT_REACTION` を発火し、
    `ぼうごパット` ハンドラが False を返す）
  - 攻撃者の特性が `マジックガード` の場合はダメージを受けない
    （`modify_hp` の `reason=""` が `ON_MODIFY_NON_MOVE_DAMAGE` を経由し、
    `マジックガード` ハンドラが無効化する）
  - 使用者（ニードルガード側）がみがわり状態でもダメージを与える。相手がみがわり状態でも、
    本体にダメージを与える（`modify_hp` が対象ポケモン本体のHPを直接操作するため、
    みがわりの身代わり判定を経由しない）

### 防御が無視されるケース（今世代スコープ外の記述を除く）

- `ふかしのこぶし` 特性の直接攻撃では守りを貫通し、ダメージも発生しない
  - `ON_CHECK_PROTECT` ハンドラで `ふかしのこぶし` が False を返す
- `ハイパードリル` も守りを貫通（技ハンドラ側で処理）
- `フェイント` / `ゴーストダイブ` / `シャドーダイブ` / `いじげんホール` / `いじげんラッシュ` を受けた場合
  - ニードルガード状態が取り除かれてダメージを受け、接触カウンターダメージは発生しない
  - 攻撃がタイプ相性で無効な場合はニードルガード状態が取り除かれない
- ダイマックスわざ・Zワザによる貫通（1/4ダメージを受ける）は第9世代（champions基準）に
  存在しないため対象外

### 連続使用による失敗率上昇

- まもる/みきり/こらえる/ニードルガード/トーチカ/スレッドトラップ/かえんのまもり/キングシールド を連続使用すると成功率が低下する
  - 成功するごとに次ターンの成功率が 1/3 になる（下限 1/729）
  - 連続使用しなかった場合・失敗した場合はリセットされる
  - **注意**: 本実装では連続使用失敗チェックは `ON_TRY_MOVE_2` のデフォルト priority で行われる
    （turn.md 上の記載は `ON_TRY_MOVE_1` priority=30 だが、既存の守る系技すべてがこのパターンで
    実装済みのため踏襲する）

## ターン処理の優先度

`.internal/spec/turn.md` の `Event.ON_TRY_MOVE_1` より:

| Priority | 処理 |
|----------|------|
| 30 | まもる系: 連続使用により失敗（実装では `Event.ON_TRY_MOVE_2` デフォルト priority） |
| 100 | ニードルガード状態による攻撃技・変化技の無効化・接触カウンターダメージ |

`Event.ON_TURN_END`:
- priority 未指定（デフォルト）で source:self に対して解除する

## 関連インタラクション

- **ぼうごパット**: 攻撃者の `is_contact_reaction` を False にするため、ダメージを受けない
- **マジックガード**: `ON_MODIFY_NON_MOVE_DAMAGE` で無効化されるため、ダメージを受けない
- **ふかしのこぶし**: `ON_CHECK_PROTECT` でFalseを返し、守りを貫通する（ダメージも発生しない）
- **まもる / みきり / キングシールド / トーチカ / スレッドトラップ / かえんのまもり**: 連続使用成功率共有
- **フェイント / ゴーストダイブ / シャドーダイブ / いじげんホール / いじげんラッシュ**: ニードルガード状態を取り除く

## 専用ポケモン

- X・Yではブリガロンの専用技、BDSPではノクタスの専用技だったが、champions基準（第9世代）では
  一般習得技（オーガポン等が第9世代で新規習得）として扱う
