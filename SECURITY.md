# Security Policy

## このライブラリの性質について

jpoke はポケモンチャンピオンズのシングルバトルをローカルで再現する、
イベント駆動のシミュレーション・ダメージ計算ロジックライブラリです。

- ネットワーク通信を行いません（外部サーバーとの通信・API呼び出しは一切なし）
- ユーザー認証機能を持ちません
- ユーザーデータ（個人情報・アカウント情報等）を保存・送信しません
- 外部からの入力を受け付けて処理するネットワークサービスではなく、呼び出し側の
  Python プロセス内で完結するライブラリです

このため、想定される攻撃対象領域（アタックサーフェス）は一般的な Web サービスや
ネットワークサーバーと比べて限定的です。想定されるリスクは主に、ライブラリを
組み込んだアプリケーション経由での意図しない動作（クラッシュ・無限ループ・
不正な入力によるロジック破綻など）に留まります。

## サポート対象バージョン

jpoke は現在 0.x 系（プレリリース相当）です。**PyPI で公開されている最新版のみ
サポート対象** とし、過去バージョンへの遡及的なセキュリティ修正は行いません。

| バージョン | サポート状況 |
|---|---|
| 最新版（PyPI 公開の最新リリース） | ✅ サポート対象 |
| それ以外の過去バージョン | ❌ サポート対象外 |

## 脆弱性の報告方法

脆弱性（機微な内容を含む可能性がある不具合）を発見した場合は、公開の Issue では
なく [GitHub Security Advisories](https://github.com/tmwork1/jpoke/security/advisories/new)
経由での非公開報告をお願いします。修正内容や影響範囲を公開前に確認できます。

一方で、クラッシュ・計算結果の誤りなど機微でない一般的なバグ報告については、
通常どおり [GitHub Issues](https://github.com/tmwork1/jpoke/issues) で問題ありません。

報告いただく際は、以下の情報を含めていただけると調査がスムーズです。

- jpoke のバージョン（`jpoke.__version__`）
- Python のバージョン・OS
- 再現手順（可能であれば最小限のコード例）
- 想定される影響

本プロジェクトは個人が非公式（fan-made）で開発・保守しているため、対応にお時間を
いただく場合があります。あらかじめご了承ください。
