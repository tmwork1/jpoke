# 場に出たときに発動する特性の実装

## 概要
ON_SWITCH_IN イベントで発動する、または登場時に情報開示が必要な特性54個の実装計画。
- 基本要件: `LogCode.ABILITY_TRIGGERED` 記録時に `mon.ability.revealed = True` を設定
- ログ開示: `LogCode.ABILITY_TRIGGERED` で記録（実装パターンで統一）

---

## 実装パターン分類

### Pattern A: 天候・地形・場の状態を変更（11個）
**優先度**: 100（Priority）、実装: `common.activate_weather()` / `activate_terrain()` / `activate_field()`

- あめふらし → Event.ON_SWITCH_IN: `activate_weather("あめ")`
- ひでり → Event.ON_SWITCH_IN: `activate_weather("はれ")`
- すなおこし → Event.ON_SWITCH_IN: `activate_weather("すなあらし")`
- ゆきふらし → Event.ON_SWITCH_IN: `activate_weather("ゆき")`
- おわりのだいち → Event.ON_SWITCH_IN: `activate_weather("おおひでり")`
- はじまりのうみ → Event.ON_SWITCH_IN: `activate_weather("おおあめ")`
- デルタストリーム → Event.ON_SWITCH_IN: `activate_weather("らんきりゅう")`
- グラスメイカー → Event.ON_SWITCH_IN: `activate_terrain("グラスフィールド")`
- エレキメイカー → Event.ON_SWITCH_IN: `activate_terrain("エレキフィールド")`
- サイコメイカー → Event.ON_SWITCH_IN: `activate_terrain("サイコフィールド")`
- ミストメイカー → Event.ON_SWITCH_IN: `activate_terrain("ミストフィールド")`

**実装スニペット例** (あめふらし)：
```python
def あめふらし_on_switch_in(battle: Battle, ctx: BattleContext, value: Any) -> HandlerReturn:
    """あめふらし特性: 場に出たときに雨へ変更する。"""
    mon = ctx.source
    common.activate_weather(battle, "あめ")
    mon.ability.revealed = True  # ← ハンドラ実行時に特性開示
    idx = battle.get_player_index(mon)
    battle.event_logger.add(
        battle.turn, idx, LogCode.ABILITY_TRIGGERED,
        payload={"ability": "あめふらし", "success": True}
    )
    return HandlerReturn(value=value)
```

### Pattern D: 相手ポケモンの能力を変更（100 Priority）（2個）
- ダウンロード → Event.ON_SWITCH_IN: 相手の防御 vs 特防で補正能力を決定
- トレース → Event.ON_SWITCH_IN: 相手の特性をコピー + ON_SWITCH_IN 再emit

**実装スニペット例** (トレース)：
```python
def トレース_on_switch_in(battle: Battle, ctx: BattleContext, value: Any) -> HandlerReturn:
    """トレース特性: 相手の有効特性をコピーする。"""
    mon = ctx.source
    foe = battle.foe(mon)
    
    if not foe or "uncopyable" in foe.ability.data.flags:
        return HandlerReturn(value=value)
    
    battle.set_ability(mon, foe.ability.name)
    mon.ability.revealed = True  # ← ハンドラ実行時に特性開示
    
    idx = battle.get_player_index(mon)
    battle.event_logger.add(
        battle.turn, idx, LogCode.ABILITY_TRIGGERED,
        payload={"ability": "トレース", "success": True}
    )
    
    # コピー後にコピー先特性の登場処理も発動させる
    battle.events.emit(Event.ON_SWITCH_IN, BattleContext(source=mon))
    return HandlerReturn(value=value)
```

## テスト観点共通化

全特性共通:
1. ✓ `assert mon.ability.revealed is True` — ハンドラが実行されて `LogCode.ABILITY_TRIGGERED` を記録した後
2. ✓ `assert mon.ability.revealed is False` — ハンドラが条件を満たさずログを記録しない場合（非発動ケース）
3. ✓ 相手ポケモンがいない場合も動作（エラー起きない）

Pattern別:
- Pattern A: 対応天候/地形が正しく set される
- Pattern C: 相手ポケモン状態が正しく変更される
- Pattern D: 自分のステータス/特性が正しく更新される
- Pattern E: 相手に状態を付与できる
- Pattern H: ハンドラが実行され LogCode.ABILITY_TRIGGERED が記録される（実際の効果は別途イベントで実装）

## 実装リスク管理

- **ON_SWITCH_IN 再emit**: トレースのように再emit する特性は、無限ループ防止チェック必須
- **能力無効化との相互作用**: かがくへんかガス発動時に他の登場特性が無効化されるケース
- **優先度順序**: spec/turn.md に記載された Priority を厳守
- **相手ポケモン未出現**: 対戦初手やデータ不整合時のエラーハンドリング

## 重要な実装注記

### 1. ability.revealed の設定タイミング
```python
# src/jpoke/core/switch.py の register_switch_in_handlers() ではハンドラ登録のみ
def register_switch_in_handlers(self, mon: Pokemon):
    # mon.revealed は初期状態のまま（False）
    mon.ability.register_handlers(...)
```

**重要**: `revealed` フラグはハンドラの処理が実際に発動し、`LogCode.ABILITY_TRIGGERED` を記録するタイミングで設定される。
- ハンドラ内で LogCode を記録する際に、必ず `mon.ability.revealed = True` を設定する。
- ハンドラの処理結果に応じて LogCode を記録しない場合、`revealed` は False のままになる。
  - 例: かわりもの は相手の特性が「コピー不可」の場合、ハンドラを実行しても LogCode を記録せず、`revealed` は False

**2026年5月4日 修正**: 天候・地形・状態異常系の LogCode 記録用ヘルパー関数を追加
```python
# src/jpoke/handlers/ability.py に以下を追加:
- _activate_weather_with_log(): 天候変更 + LogCode 記録
- _activate_terrain_with_log(): 地形変更 + LogCode 記録
- _apply_ailment_with_log(): 状態異常付与 + LogCode 記録
```

- ぜったいねむり（状態異常）

### 2. LogCode.ABILITY_TRIGGERED と revealed の連動
```python
# Pattern H で announce_ability_on_switch_in() を共用する場合:
def announce_ability_on_switch_in(battle: Battle, ctx: BattleContext, value: Any) -> HandlerReturn:
    """ON_SWITCH_IN でハンドラが実行されたことをログ記録し、特性を開示する。"""
    mon = ctx.source
    mon.ability.revealed = True  # ← ログ記録と同時に特性開示
    idx = battle.get_player_index(mon)
    battle.event_logger.add(
        battle.turn, idx, LogCode.ABILITY_TRIGGERED,
        payload={"ability": mon.ability.name, "success": True}
    )
    return HandlerReturn(value=value)
```

**Pattern H の役割**:
- ON_SWITCH_IN 時に特性を「開示」（ログ記録 + revealed = True）する。
- 実際の効果（攻撃-1 など）は別途のイベント（ON_ABILITY_ENABLED など）で実装されている。
- ON_SWITCH_IN では効果を発動させず、情報開示だけを行う。

### 3. ON_SWITCH_IN 再emit（トレース）の無限ループ防止
```python
# トレース実装時は「同じ特性を再度トレースしない」チェックが必須
if mon.ability.name == copied_ability:
    return HandlerReturn(value=value)  # 再emit しない
battle.events.emit(Event.ON_SWITCH_IN, BattleContext(source=mon))
```

### 4. 相手ポケモン参照時の安全性
```python
foe = battle.foe(mon)
if foe is None:  # 初手・交代先不在の場合も安全に
    return HandlerReturn(value=value)
```
