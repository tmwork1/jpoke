# 強天候システム 実装計画書

更新日: 2026-05-01

## ユーザーコメント
- 通常天候と強天候の共通効果を実装するとき、毎回`weather in {Weather.SUN, Weather.EXTREME_SUN}`のような判定式を書くのではなく、天候オブジェクト側にプロパティを通すようにしたい

## 概要

強天候（おおひでり / おおあめ / らんきりゅう）は通常天候より優先度が高い特殊天候カテゴリである。  
3 種を共通の導線に乗せ、各特性（おわりのだいち / はじまりのうみ / デルタストリーム）から発動する。

## 実装ゴール

| 天候 | 発動特性 | 固有効果 |
|------|----------|---------|
| おおひでり | おわりのだいち | みずタイプ攻撃技を無効化、晴れ相当全効果 |
| おおあめ | はじまりのうみ | ほのおタイプ攻撃技を無効化、雨相当全効果 |
| らんきりゅう | デルタストリーム | ひこうタイプ弱点（でんき/いわ/こおり）を軽減 |

共通ルール:
- 強天候は通常天候で上書き不可（通常天候技失敗）。
- 強天候同士は後発が勝つ（上書き可）。
- カウントダウンなし（永続）。
- ノーてんき / エアロック下では天候状態を保持したまま効果のみ無効。

## 型・データ整備

### `src/jpoke/utils/type_defs.py`

`Weather` 列挙型に 3 値を追加する。

```python
EXTREME_SUN  = "おおひでり"
EXTREME_RAIN = "おおあめ"
STRONG_WINDS = "らんきりゅう"
```

強天候かどうかを判定するヘルパーセットを定義する。

```python
STRONG_WEATHERS: frozenset[Weather] = frozenset({
    Weather.EXTREME_SUN, Weather.EXTREME_RAIN, Weather.STRONG_WINDS
})
```

### `src/jpoke/data/field.py`

`WeatherData` に 3 天候分のエントリを追加する。  
`duration=None`（無限）、`strong=True` フラグを持たせて通常天候と区別する。

```python
WeatherData(name="おおひでり", strong=True, duration=None),
WeatherData(name="おおあめ",   strong=True, duration=None),
WeatherData(name="らんきりゅう", strong=True, duration=None),
```

### `src/jpoke/data/ability.py`

各特性エントリに `ON_SWITCH_IN` ハンドラを接続する（個別特性の実装ステップ参照）。

## イベント別実装計画（turn.md 準拠）

### Event.ON_SWITCH_IN（Priority 100）

**対象ハンドラ**: `ability.py` → 各強天候特性  
**処理**: `common.activate_weather(weather=<強天候>)` を呼ぶ。  
通常天候始動特性（ひでり・あめふらし等）と同じ Priority 100 に配置する。

```
おわりのだいち → activate_weather("おおひでり")
はじまりのうみ → activate_weather("おおあめ")
デルタストリーム → activate_weather("らんきりゅう")
```

**`activate_weather` 内の上書きルール判定**:

| 現在の天候 | 要求天候 | 結果 |
|-----------|---------|------|
| 通常天候 | 強天候 | 強天候で上書き（常に成功） |
| 強天候 A | 強天候 B（別種） | 後発で上書き（常に成功） |
| 強天候 X | 強天候 X（同種） | 変化なし（スキップ） |
| 強天候 | 通常天候 | 失敗（後述の ON_CHECK_IMMUNE で処理） |

---

### Event.ON_CHECK_MOVE（Priority 10）

**対象ハンドラ**: `field.py` または `handlers/weather.py`  
**行**: `おおあめ・おおひでりによる技失敗`（turn.md Priority 10）  
**処理**:

- 現在天候が `おおひでり` かつ 使用技がみずタイプの攻撃技 → `stop_event=True` で失敗扱い。
- 現在天候が `おおあめ` かつ 使用技がほのおタイプの攻撃技 → `stop_event=True` で失敗扱い。
- `らんきりゅう` はここで技を失敗させない（タイプ無効化ではなくダメージ軽減のみ）。

ノーてんき / エアロック有効時はガードし、この失敗判定をスキップする。

---

### Event.ON_CHECK_IMMUNE（Priority 100）

**対象ハンドラ**: `field.py` または `handlers/weather.py`  
**行**: `天気変更技: おおひでり等で失敗`（turn.md Priority 100）  
**処理**:

- 現在天候が強天候（`STRONG_WEATHERS`）のとき、通常天候変更技（にほんばれ / あまごい / すなあらし / ゆきげしき）を失敗させる。
- `stop_event=True` を返し、天候変更を阻止する。
- 強天候変更特性（別の強天候）による発動は対象外（`ON_SWITCH_IN` 経路は技と別の経路を通る）。

---

### Event.ON_CALC_POWER_MODIFIER

**対象ハンドラ**: 既存天候補正ハンドラ  
**処理**: 既存の晴れ / 雨補正ロジックで `weather in {Weather.SUN, Weather.EXTREME_SUN}` のように強天候を同一扱いにする。

| 天候 | タイプ | 倍率 |
|------|--------|------|
| おおひでり | ほのお | ×1.5 |
| おおひでり | みず | ×0.5（実際は ON_CHECK_MOVE で失敗するため到達しない） |
| おおあめ | みず | ×1.5 |
| おおあめ | ほのお | ×0.5（同上） |
| らんきりゅう | 全タイプ | 補正なし（ウェザーボールも変化なし） |

---

### Event.ON_CALC_DEF_TYPE_MODIFIER（らんきりゅう専用）

**対象ハンドラ**: `handlers/weather.py` または `handlers/field.py`  
**Priority**: タイプ相性計算の後段に配置（既存の相性補正に上乗せ）  
**処理**:

- 天候が `らんきりゅう` のとき:
  - 防御側がひこうタイプを含む。
  - 攻撃側の技タイプが `でんき / いわ / こおり` のいずれか。
  - 上記両条件を満たす場合、ダメージ補正に `×0.5` を適用する。
- これにより: 2倍弱点 → 等倍、4倍弱点 → 2倍。
- ひこうタイプでないポケモンには影響しない。
- ステルスロックなどの固定割合ダメージには適用しない。

**じゃくてんほけん連携**:  
弱点が等倍に下がった場合、じゃくてんほけんは発動しない。  
相性補正後の実効倍率で判定されるよう、`ON_DAMAGE` での発動判定を補正後値で評価する。

---

### Event.ON_MODIFY_ACCURACY

**対象ハンドラ**: 既存命中補正ハンドラ  
**処理**:

| 技 | おおひでり | おおあめ | らんきりゅう |
|----|-----------|---------|------------|
| かみなり | 命中50% | 必中 | 変化なし（通常70%） |
| ぼうふう | 命中50% | 必中 | 変化なし（通常70%） |

既存の `if weather == Weather.SUN` → `Weather.SUN` と `Weather.EXTREME_SUN` を条件に加える。  
既存の `if weather == Weather.RAIN` → `Weather.RAIN` と `Weather.EXTREME_RAIN` を条件に加える。

---

### Event.ON_BEFORE_APPLY_AILMENT

**対象ハンドラ**: 既存こおり付与判定ハンドラ  
**処理**: `おおひでり` を晴れ相当として扱い、こおり状態の付与を無効化する。

```python
if weather in {Weather.SUN, Weather.EXTREME_SUN}:
    return HandlerReturn(stop_event=True)
```

---

### Event.ON_MODIFY_MOVE_TYPE / ウェザーボール分岐

**対象ハンドラ**: ウェザーボール処理  
**処理**:

| 天候 | タイプ | 威力 |
|------|--------|------|
| おおひでり | ほのお | 2倍（50→100） |
| おおあめ | みず | 2倍（50→100） |
| らんきりゅう | ノーマル（変化なし） | 等倍（50） |

`らんきりゅう` では天候無し相当として扱う。  
`おおひでり / おおあめ` は通常の晴れ / 雨分岐に合流させる。

---

### Event.ON_STATUS_HIT（回復技補正）

**対象ハンドラ**: 回復技（こうごうせい / あさのひざし / つきのひかり）  
**処理**:

| 天候 | 回復量 |
|------|--------|
| おおひでり | 最大HP の 2/3 |
| おおあめ | 最大HP の 1/4 |
| らんきりゅう | 最大HP の 1/2（変化なし） |

既存の `Weather.SUN` 分岐に `Weather.EXTREME_SUN` を追加する形で拡張する。  
`Weather.RAIN` 分岐に `Weather.EXTREME_RAIN` を追加する形で拡張する。

---

### Event.ON_TURN_END_1（Priority 10 : 天気終了 / Priority 20 : 天気ダメージ）

**Priority 10 — 天気終了判定**:
- `おおひでり / おおあめ / らんきりゅう` は `tick_weather` に登録しない（カウントダウン非対象）。
- 終了条件は発動ポケモンの退場・特性書き換え時のみ（`ON_SWITCH_OUT` / `ON_ABILITY_CHANGE` 経由で解除する）。

**Priority 20 — 天気ダメージ**:
- 強天候ではすなあらし・あられのダメージは発生しない（通常天候と排他）。

---

### Event.ON_CHECK_ABILITY_ENABLED / Event.ON_FIELD_CHANGE

**対象ハンドラ**: ノーてんき / エアロック 共通判定  
**処理**:
- ノーてんき / エアロック 有効時は天候の効果（タイプ無効化・ダメージ補正・弱点軽減）を抑制する。
- ただし `WeatherManager` が保持する天候名は残す（特性解除後に効果が復帰できるよう）。
- 既存の `is_weather_suppressed()` 判定を強天候にも適用する。

## テスト観点

### 共通（3 天候共通）

1. 登場時に対応する強天候が発動する。
2. 通常天候始動特性（ひでり等）が強天候中に起動しても上書きされない。
3. 強天候 A 発動中に別の強天候 B 特性が登場したとき、後発が勝つ。
4. 強天候中に通常天候技（にほんばれ等）を使うと失敗する。
5. ノーてんき / エアロック 同居時: 天候状態は残り、全効果が抑制される。

### おおひでり 固有

6. みずタイプ攻撃技が失敗する。
7. ほのおタイプ攻撃技の威力が 1.5 倍になる。
8. かみなり / ぼうふう の命中が 50% になる。
9. こおり状態にならない。
10. ソーラービームが溜め不要で発動する。
11. ウェザーボールがほのおタイプ・威力 100 になる。
12. こうごうせい等の回復量が 2/3 になる。

### おおあめ 固有

13. ほのおタイプ攻撃技が失敗する。
14. みずタイプ攻撃技の威力が 1.5 倍になる。
15. かみなり / ぼうふう が必中になる。
16. ソーラービームの威力が半減する。
17. ウェザーボールがみずタイプ・威力 100 になる。
18. こうごうせい等の回復量が 1/4 になる。

### らんきりゅう 固有

19. ひこうタイプ（でんき/いわ/こおり 被弾）の弱点が半減される（2倍→等倍、4倍→2倍）。
20. ひこうタイプでないポケモンへのダメージが変化しない。
21. じゃくてんほけんが等倍で発動しない。
22. ウェザーボールがノーマルタイプ・威力 50 のまま変化しない。
23. 回復技（こうごうせい等）の回復量が 1/2 のまま変化しない。

## 実装ステップ

1. **型定義**: `Weather` 列挙型に強天候 3 値・`STRONG_WEATHERS` を追加する。
2. **データ登録**: `WeatherData` に強天候 3 エントリを追加する。
3. **`activate_weather` 上書きルール**: 強天候 ↔ 通常天候の上書き可否を実装する。
4. **ON_SWITCH_IN**: 各特性に `activate_weather(<強天候>)` を接続する。
5. **ON_CHECK_MOVE**: おおひでり/おおあめ の技失敗ハンドラを実装する。
6. **ON_CHECK_IMMUNE**: 通常天候変更技の失敗ハンドラを実装する。
7. **ON_CALC_POWER_MODIFIER**: 既存晴れ/雨補正に強天候を合流させる。
8. **ON_CALC_DEF_TYPE_MODIFIER**: らんきりゅう弱点軽減ハンドラを実装する。
9. **ON_MODIFY_ACCURACY**: かみなり/ぼうふうの強天候分岐を追加する。
10. **ON_BEFORE_APPLY_AILMENT**: おおひでりでこおり無効を追加する。
11. **ウェザーボール・回復技分岐**: 強天候対応を追加する。
12. **tick 除外**: `ON_TURN_END_1` の天気終了判定から強天候を除外する。
13. **テスト**: 上記テスト観点 1–23 をカバーするテストを追加する。
