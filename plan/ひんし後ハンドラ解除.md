# ひんし後ハンドラ解除 — 設計調査メモ

## 問題

ポケモンがひんしになった瞬間（`modify_hp` → `hp == 0`）にハンドラは解除されない。  
解除されるのは `run_faint_switch` → `run_switch` → `unregister_switch_out_handlers` であり、  
**ひんしから死に出し交代完了までの間、ハンドラは登録されたまま発火し続ける。**

## ターン内のタイムライン（典型例）

```
ON_TURN_END_3  ← 毒・砂・のろいダメージ → ここでひんし発生
ON_TURN_END_4
ON_TURN_END_5  ← だっぴ, あめうけざら等 → ひんし後も発火する ★
run_faint_switch → run_switch → unregister_switch_out_handlers ← ここで初めて解除
ON_TURN_END_6
```

## 現在の対処：`not mon.alive` ガード

ひんし後に発火してはいけないハンドラには手動ガードが入っている：

| ハンドラ | ガード |
|---------|--------|
| `だっぴ_on_turn_end` | `not mon.alive` |
| `ポイズンヒール` 系 | `not mon.alive` |
| `さいせいりょく_on_turn_end` | `not mon.alive` |
| `ももんのみ` 等のアイテム | `not mon.alive`（要確認） |

## ひんし後に発火すべきハンドラ（正常動作）

| ハンドラ | イベント | 理由 |
|---------|---------|------|
| `おんねん` | `ON_HP_CHANGED` (`ctx.fainted`) | 攻撃者への PP 削除はひんし時のみ |
| `みちづれ` | `ON_HP_CHANGED` (`ctx.fainted`) | 攻撃者も道連れにする |

これらは `ctx.fainted` フラグを使い、**ひんしになった瞬間** の `ON_HP_CHANGED` 内で処理している。  
ハンドラが解除されていないことを利用している設計ではなく、タイミングが `ON_HP_CHANGED` に依存している。

# `ON_FAINTED` イベント導入計画

## 背景・問題

`modify_hp` で `hp == 0` になった後、ハンドラは `run_faint_switch` 完了まで解除されない。  
そのため `ON_TURN_END_*` 等でひんしポケモンのハンドラが引き続き発火し、  
各ハンドラに `not mon.alive` ガードを手動で書き続ける必要がある。

```
【現状】
ON_TURN_END_3  ← 毒・砂ダメージ → ひんし発生
ON_TURN_END_4  ← ハンドラまだ生きている ★
ON_TURN_END_5  ← だっぴ, サンパワー等も発火してしまう ★
run_faint_switch → unregister_switch_out_handlers ← ここで初めて解除

【目標】
ON_TURN_END_3  ← ひんし発生 → emit ON_FAINTED → unregister_handlers
ON_TURN_END_4  ← ハンドラ解除済み（発火しない）
ON_TURN_END_5  ← 同上
run_faint_switch（交代処理のみ）
```

---

## 変更ステップ

### A. `Event.ON_FAINTED` を追加

**ファイル:** `src/jpoke/enums/event.py`

`ON_HP_CHANGED` の直後に追加する。

```python
# emit: core/pokemon_state.py（HP が 0 になった直後）
# handle: volatile.py（おんねん・みちづれ等のひんし時効果）
ON_FAINTED = auto()
```

---

### B. `pokemon_state.py` で `ON_FAINTED` を emit しハンドラを解除する

**ファイル:** `src/jpoke/core/pokemon_state.py` — `StatusManager.modify_hp`

```python
# 変更前
if target.fainted:
	self.battle.judge_winner()

# 変更後
if target.fainted:
	self.battle.events.emit(
		Event.ON_FAINTED,
		BattleContext(target=target, hp_change_reason=reason),
	)
	self.battle.switch_manager.unregister_switch_out_handlers(target)
	self.battle.judge_winner()
```

> **注意:** `unregister_switch_out_handlers` はすでに `run_switch` 内でも呼ばれる。  
> 2 回呼んでも `events.off` は冪等（未登録ハンドラの off は無視される）ため問題ない。  
> ただし `run_switch` 側での重複呼び出しを避けたい場合は `fainted` フラグで分岐してもよい。

---

### C. `おんねん`・`みちづれ` を `ON_HP_CHANGED` → `ON_FAINTED` に移す

**ファイル:** `src/jpoke/handlers/volatile.py`

現在は `ON_HP_CHANGED` 内で `ctx.fainted` を見て発火している。  
`ON_FAINTED` 導入後はその直前に `ON_FAINTED` が emit されるため、そちらへ移す。

```python
# 変更前（ON_HP_CHANGED ハンドラ内）
def おんねん(battle, ctx, value):
	if ctx.fainted:
		ctx.move.pp = 0
		...

# 変更後（ON_FAINTED ハンドラ）
def おんねん_on_fainted(battle, ctx, value):
	if ctx.move is None:
		return HandlerReturn()
	ctx.move.pp = 0
	...
```

`みちづれ` も同様に `ON_FAINTED` ハンドラへ移す。

> `ON_FAINTED` は `target` として ctx を作るため、`ctx.attacker` で攻撃者を参照できるかを確認すること。  
> 必要に応じて `BattleContext` に `source`（攻撃者）を追加して emit する。

---

### D. `not mon.alive` ガードを除去する

B で `ON_FAINTED` 発火直後にハンドラを解除するため、以下のガードは不要になる。

| ファイル | 行 | ハンドラ | 除去するガード |
|---------|---|---------|--------------|
| `handlers/ability.py` | L154 | `だっぴ_on_turn_end` | `not mon.alive` |
| `handlers/ability.py` | L466 | `さいせいりょく_on_switch_out` | `not mon.alive` |
| `handlers/ability.py` | L503 | `サンパワー_on_turn_end` | `not mon.alive` |
| `handlers/ability.py` | L518 | `しぜんかいふく_on_switch_out` | `not mon.alive` |
| `handlers/ability.py` | L1899 | `ムラっけ_on_turn_end` | `not mon.alive` |

> `on_switch_out` ハンドラ（さいせいりょく・しぜんかいふく）はひんし退場時も発火するが、  
> B の変更でハンドラが解除済みのため、ガードなしでも発火しない。ただし **テストで確認すること**。

---

## テスト方針

1. 毒ダメージで ON_TURN_END_3 にひんし → ON_TURN_END_5 の `だっぴ` 等が発火しないこと
2. `おんねん` のひんし時 PP 削減が引き続き動作すること
3. `みちづれ` のひんし時道連れが引き続き動作すること
4. `さいせいりょく` が生存交代時のみ発動し、ひんし退場時は発動しないこと
5. `ムラっけ` のターン終了処理がひんし後に発火しないこと
