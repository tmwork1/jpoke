# こだいかっせい・クォークチャージ 実装計画

## 目的
- こだいかっせい と クォークチャージ を、発動条件だけ差し替える共通基盤として実装する。
- 既存のイベント駆動設計を崩さず、交代・天候/地形変化・特性無効化・持ち物消費を一貫して扱えるようにする。

## 現状整理
- src/jpoke/data/ability.py では両特性が flags のみで、handler は未登録。
- src/jpoke/data/item.py の ブーストエナジー は consumable 定義のみで、自動発動処理が無い。
- src/jpoke/model/pokemon.py には、両特性の「どの能力が上がっているか」「何由来で発動中か」を保持する状態がまだ無い。
- かがくへんかガス による特性有効/無効の共通ゲートは既にあるため、発動済み補正の一時停止と復帰はこの基盤に乗せるのが自然。
- ひひいろのこどう、ハドロンエンジン、マジックルーム は未実装なので、今回の計画では接続しやすい形で基盤を作り、個別挙動は後続タスクに分ける。

## 実装方針

### 1. 共通状態の追加
- Pokemon に以下の場持続状態を追加する。
	- 特性ブーストが現在有効か
	- 上昇対象の能力値
	- 発動源 item / weather / terrain
	- 同一滞在中にブーストエナジーを起動済みか
- ベンチに戻ったときに確実にリセットされるよう、bench_reset() に寄せて管理する。

### 2. 共通 helper の実装
- src/jpoke/handlers/ability.py か src/jpoke/handlers/common.py に、両特性共通で使う helper を追加する。
	- 発動条件判定
	- 比較対象能力の決定
	- 発動処理
	- 解除処理
	- 再判定処理
- こだいかっせい と クォークチャージ の差分は、以下だけに閉じ込める。
	- 参照する場の条件: はれ / エレキフィールド
	- ログメッセージ
	- 発動源の表現

### 3. 上昇能力の決定ロジック
- 比較対象は HP を除く A/B/C/D/S とする。
- ランク補正込みで比較し、同値は A → B → C → D → S の順で決定する。
- 比較時には、こだわり系・おいかぜ など既存の最終補正は含めず、現在コードが直接持っている実数値とランク変化を基準にする。
- 発動後にランクが変わっても上昇箇所は固定し、再発動したときだけ再計算する。

### 4. 補正の差し込み位置
- 素早さ上昇は DomainEvent.ON_CALC_SPEED で処理する。
- 攻撃/特攻/防御/特防の上昇は Event.ON_CALC_ATK_MODIFIER と Event.ON_CALC_DEF_MODIFIER に差し込む。
- move context を見て、どの実数値に補正が掛かる場面かを判定する。
	- A 上昇は通常の物理技だけ
	- C 上昇は特殊技だけ
	- B 上昇は防御参照だけでなく ボディプレス の攻撃側計算にも反映
	- D 上昇は特殊受け側計算に反映
- これにより、ランク変化とは別枠の補正として扱い、てんねん や急所の既存ランク処理と混線させない。

### 5. 発動契機の接続
- ON_SWITCH_IN に handler を登録し、場に出た直後の後発タイミングで発動判定を行う。
- 設置技や いかく の後で判定できるよう、priority は既存の入場効果より遅くする。
- 天候/地形が変わった直後に再判定できるよう、WeatherManager / TerrainManager の activate/deactivate 直後に共通再評価フックを入れる。
- 場条件で発動中に条件が切れたら補正を解除し、ブーストエナジー未消費ならその場で再発動を試みる。

### 6. ブーストエナジーの扱い
- 場条件が無いときのみ消費して発動する。
- 場条件がある状態で場に出た場合は消費せず、場条件由来の発動を優先する。
- 場条件由来の効果が切れた瞬間に、未消費ならブーストエナジーへフォールバックする。
- 一度アイテム由来で起動した後は、その滞在中に再入手しても再起動しない設計にする。
- マジックルーム対応は move 実装が前提なので、helper 側で「item.enabled を見る」形にして後から接続しやすくする。

### 7. 特性無効化との整合
- かがくへんかガス と とくせいなし により未発動のまま抑止されるケースは、解除時即発動ではなく、次回の再判定契機まで保留する。
- 既に発動済みの補正は、特性が無効化された間だけ効果を止め、再有効化時に同じ能力へ復帰させる。
- 補正復帰時に「再計算」しないことで、仕様通りガス前に選ばれた能力を維持する。

## 実装ステップ
1. Pokemon に専用状態を追加し、bench_reset() で初期化する。
2. 共通 helper を実装し、こだいかっせい / クォークチャージ の AbilityData に handler を登録する。
3. ON_CALC_SPEED / ON_CALC_ATK_MODIFIER / ON_CALC_DEF_MODIFIER へ補正処理を接続する。
4. ON_SWITCH_IN からの発動と、天候/地形変化時の再評価フックを実装する。
5. ブーストエナジー消費とフォールバック再発動を実装する。
6. かがくへんかガス / とくせいなし / 交代リセットの回帰テストを追加する。
7. ひひいろのこどう / ハドロンエンジン / マジックルーム は別タスクで接続する。

## テスト観点
- はれ の場で こだいかっせい が発動し、最高能力だけが上がる。
- エレキフィールド の場で クォークチャージ が発動し、最高能力だけが上がる。
- 同値時に A → B → C → D → S の順で選ばれる。
- 場条件なしで場に出たとき、ブーストエナジーを消費して発動する。
- 場条件ありで場に出たとき、ブーストエナジーを消費しない。
- 場条件由来の効果が切れたとき、未消費のブーストエナジーで即再発動する。
- 素早さ上昇時は calc_effective_speed() に反映される。
- A/C/B/D 上昇時はダメージ計算にだけ反映され、ランク変化としては扱われない。
- かがくへんかガス中は補正が止まり、解除後は同じ能力に復帰する。
- 交代すると補正対象・発動源・アイテム起動済み状態がリセットされる。

## 後続タスク
- ひひいろのこどう / ハドロンエンジン 実装後、この共通 helper を使って即時発動まで接続する。
- マジックルーム実装後、ブーストエナジー起動可否を item.enabled に連動させる。
- ノーてんき / エアロック 実装後、こだいかっせい の場条件判定へ抑止を追加する。
