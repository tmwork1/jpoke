# いたずらごころ 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/いたずらごころ.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `DomainEvent.ON_MODIFY_MOVE_PRIORITY`
- `Event.ON_FIELD_CHANGE`
- `Event.ON_BEFORE_MODIFY_STAT`
- `Event.ON_MODIFY_STAT`

## 仕様要点（実装に反映）
- 全ての 変化技 の 優先度 を1加算して使用する。
- 第七世代 以降では、この特性の効果で優先度を上げた技は敵の あく タイプ に無効化される。
- 他の技が出る変化技 で 攻撃技 の 溜め技 ・ 数ターン行動する技 を出した場合、1ターン目はいたずらごころの効果で優先度が上がるが、2ターン目以降の行動にいたずらごころの効果は無い。
- 他の技が出る技で ジオコントロール を出した場合は、2ターン目にもいたずらごころの効果がある。
- メガシンカ により他の特性のポケモンがいたずらごころに変化したとき、 第六世代 ではそのターンの行動順に影響は無い。 第七世代 ではそのターンからいたずらごころは適用される。逆にメガシンカによりいたずらごころから別の特性に変化した...

## 実装フロー
1. `src/jpoke/data/ability.py` の `いたずらごころ` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `いたずらごころ` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
