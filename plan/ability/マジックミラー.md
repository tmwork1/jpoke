# マジックミラー 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/マジックミラー.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_BEFORE_MODIFY_STAT`
- `Event.ON_MODIFY_STAT`
- `Event.ON_CHECK_DEF_ABILITY`

## 仕様要点（実装に反映）
- 戦闘中、常に マジックコート状態 になる。
- 跳ね返せる変化技、跳ね返せない変化技については マジックコート (状態変化)#対象技 を参照とする。
- かたやぶり 系の効果がある 変化技 は跳ね返せない。
- 設置技 の変化技を受けたときは跳ね返せるが、すでに設置されているこれらの場の状態の効果を防いだり跳ね返したりすることはできない。
- がんせきアックス / ひけん・ちえなみ など 攻撃技 による 追加効果 や、 どくげしょう など特性による効果は跳ね返せない。

## 実装フロー
1. `src/jpoke/data/ability.py` の `マジックミラー` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `マジックミラー` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
