# ふしょく 実装計画

## スコープ
- 優先度: H
- 仕様書: `spec/ability/ふしょく.md`
- 実装方針: どく・もうどく付与の共通経路を単一の制御点に寄せ、`ふしょく` は「どく/はがねタイプへの通常無効だけを外す」例外として扱う。状態異常そのものを防ぐ他要因は既存の `Event.ON_BEFORE_APPLY_AILMENT` 判定をそのまま通す。

## 現状整理
- `src/jpoke/data/ability.py` の `ふしょく` は `AbilityData()` のみで、ハンドラ未登録。
- 状態異常付与は `src/jpoke/handlers/common.py` の `apply_ailment()` から `battle.ailment_manager.apply()` に集約されている。
- `src/jpoke/core/pokemon_state.py` の `AilmentManager.apply()` は `Event.ON_BEFORE_APPLY_AILMENT` を発火し、`めんえき`・`ふみん`・`やるき`・`ミストフィールド`・`しんぴのまもり` などの無効化要因を共通処理している。
- `どくどく` のような状態異常技は `common.apply_ailment()` を経由している一方、`どくびし` は直接 `ailment_manager.apply()` を呼んでおり、こちらも共通制御点側を直せば追従できる。
- `src/jpoke/core/move_executor.py` の変化技処理は `Event.ON_CHECK_IMMUNE` を発火するが、`src/jpoke/handlers/move.py` のタイプ無効判定 `_is_type_immune()` は一撃必殺技ハンドラからしか参照されていない。
- 実挙動確認では、`ふしょく` なしの `どくどく` でも `コイル`・`フシギダネ` に通常どおり `もうどく` が入る。つまり、毒タイプ/鋼タイプへの毒無効は現状未実装で、`ふしょく` 以前の共通仕様欠落になっている。

## 仕様要点（実装に反映）
- `ふしょく` 持ちが与える `どく` / `もうどく` は、通常は無効の どくタイプ・はがねタイプ にも通る。
- `めんえき`、`ミストフィールド`、`しんぴのまもり`、`みがわり`、既存の別状態異常など、「タイプ相性以外」の無効要因は貫通しない。
- 特性無効化中や特性変更後は通常どおりの毒無効判定に戻る。
- 対象外は「毒付与そのもの」であり、ダメージ・命中・追加効果抽選など他の処理順は変えない。

## 未対応・要確認
- 毒タイプ/鋼タイプへの毒無効は「分散している」のではなく、少なくとも通常の毒付与経路では未実装。`common.apply_ailment()` 経由と `AilmentManager.apply()` 直呼びの両方を一度に直せる位置が必要。
- `ON_BEFORE_APPLY_AILMENT` は現在「状態異常そのものを防ぐ」用途で使われているため、`ふしょく` をここへ直接載せるより、タイプ無効の判定ヘルパーを `AilmentManager.apply()` 側に寄せる方が副作用が小さい。
- `どくびし` や `トーチカ` など、技以外の毒付与経路にも `source` が渡る。`source` の特性参照条件を広げすぎると、本来 `ふしょく` の影響を受けない経路まで通してしまうおそれがある。

## 実装フロー
1. `AilmentManager.apply()` に、どく/もうどく付与時のタイプ無効判定ヘルパーを追加する。現状は通常の毒無効そのものが未実装なので、ここを一次制御点にする。
2. 共通ヘルパーでは「対象が どく/はがね か」「付与元 `source` が存在し、`ふしょく` が有効か」を判定し、`ふしょく` 有効時だけタイプ無効を解除する。
3. `めんえき` などの既存 `ON_BEFORE_APPLY_AILMENT` はそのまま残し、`ふしょく` はそれらを素通ししないことを確認する。
4. `src/jpoke/core/move_executor.py` / `src/jpoke/handlers/move.py` の `ON_CHECK_IMMUNE` 側には原則手を入れず、毒付与そのものの可否だけを状態異常レイヤで扱う。
5. `src/jpoke/data/ability.py` の `ふしょく` に必要最小限のフラグを追加する。専用ハンドラが不要なら `AbilityData()` のままでもよく、無理にイベント登録を増やさない。
6. 追加効果毒技、変化技、場由来の毒付与で挙動差が出ないことをテストで固定する。

## テスト観点
- 発動ケース: `ふしょく` 持ちの `どくどく` が はがねタイプ・どくタイプ の相手に成功する。
- 発動ケース: `ふしょく` 持ちの追加効果毒技で、ダメージ後の毒付与判定が成功する。
- 非発動ケース: `ふしょく` なしでは従来どおり、はがねタイプ・どくタイプ への毒付与は失敗する。
- 無効化ケース: 相手が `めんえき` のときは `ふしょく` 持ちでも失敗する。
- 無効化ケース: `ミストフィールド`・`しんぴのまもり`・`みがわり`・既存状態異常中には失敗する。
- 境界ケース: `かがくへんかガス` や `かたやぶり` 系の既存特性有効/無効判定と矛盾しない。
- 境界ケース: `どくびし` や `トーチカ` など、`ふしょく` を持たない付与元の経路は従来どおりである。
