# ほろびのボディ 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/ほろびのボディ.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`
- `Event.ON_CALC_POWER_MODIFIER`

## 仕様要点（実装に反映）
- 直接攻撃 で ダメージ を受けたとき、攻撃してきた相手と自分を ほろびのうた 状態にし、3ターン後に ひんし にする。
- 特性 ぼうおん のポケモンに対しても効果がある。
- 攻撃で自分が ひんし になったときでも発動する。
- 特性が発動するタイミングで、攻撃者がすでにひんしになっているときは発動しない。
- 相手が ぶんまわす / ワイドブレイカー など、直接攻撃の全体攻撃を使用し、攻撃者の味方か、ほろびのボディの左側にいるポケモンが持つ ゴツゴツメット / ジャポのみ / さめはだ などのダメージを受けてひんしになった場合が該当する...

## 実装フロー
1. `src/jpoke/data/ability.py` の `ほろびのボディ` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `ほろびのボディ` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
