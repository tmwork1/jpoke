# ミラクルスキン 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/ミラクルスキン.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_FIELD_CHANGE`
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_BEFORE_MODIFY_STAT`
- `Event.ON_MODIFY_STAT`

## 仕様要点（実装に反映）
- この特性のポケモンが受ける 変化技 の 命中率 は50になる。
- 必中技 など、 命中 判定が無い技を回避することはできない。 あくび / ほえる / どく タイプが使用した どくどく などや、 自分が対象の変化技 、 味方の場 / 相手の場 / 全体の場が対象の技 などが該当する。
- 状態/ もちもの / ランク補正 など、 命中 が変動する他の効果はミラクルスキンの後に計算する。
- 例えば 回避率 が6段階上がったミラクルスキンのポケモンが命中75の変化技を受ける場合、ミラクルスキンの効果で命中が50に変化した後にランク補正の効果で1/3倍 (小数点以下切り捨て) になり、結果として命中率は16%になる。
- 詳しい仕様は 命中#第五世代以降 を参照。

## 実装フロー
1. `src/jpoke/data/ability.py` の `ミラクルスキン` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `ミラクルスキン` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
