# リベロ 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/リベロ.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_CALC_ATK_TYPE_MODIFIER`

## 仕様要点（実装に反映）
- 技を出す直前に、使用者のタイプがその技と同じタイプに変化する。 交代 して手持ちに戻ると元のタイプに戻る。
- 第九世代 では、1回場に出るごとにつき1回しか発動しない。
- へんげんじざい#特性の仕様 を参照。

## 実装フロー
1. `src/jpoke/data/ability.py` の `リベロ` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `リベロ` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
