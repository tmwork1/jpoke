# きけんよち 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/きけんよち.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_FIELD_CHANGE`

## 仕様要点（実装に反映）
- 場に出たとき、相手が自分に対して 効果バツグン の 攻撃技 を覚えているときか、 一撃必殺技 を覚えているときに、「<特性所持者>は みぶるいした!」というメッセージが表示される。
- 一部対象わざは世代によって異なる。
- 共通: ウェザーボール / だいちのはどう は 場の状態 に関係なく ノーマル タイプとみなす。
- 第四世代: カウンター / ミラーコート / メタルバースト には反応しない。
- 第五世代 以降: カウンター/ミラーコート/メタルバーストのタイプが効果抜群になるときは反応する。

## 実装フロー
1. `src/jpoke/data/ability.py` の `きけんよち` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `きけんよち` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
