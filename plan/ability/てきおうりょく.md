# てきおうりょく 実装計画

## スコープ
- 優先度: H
- 仕様書: `spec/ability/てきおうりょく.md`
- 実装の核: STAB 計算のみを拡張し、他の倍率計算へ影響を広げない。

## 実装対象イベント
- `Event.ON_CALC_ATK_TYPE_MODIFIER`

## 実装方針（確定）
1. `src/jpoke/data/ability.py` の `てきおうりょく` に `Event.ON_CALC_ATK_TYPE_MODIFIER` を登録する。
2. `src/jpoke/handlers/ability.py` に `てきおうりょく_on_calc_atk_type_modifier` を追加する。
3. ハンドラでは「通常 STAB=1.5」を「てきおうりょく STAB=2.0」へ置換する。
4. テラスタル時は既存 STAB 判定の結果を入力として扱い、以下のみ分岐する。
- 元タイプ一致 + テラ一致: 2.25
- テラ一致のみ: 2.0
- テラで失った元タイプ一致: 1.5
- ステラ時: 既存実装に合わせ、てきおうりょく補正を適用しない

## 実装メモ
- 補正位置は `ON_CALC_POWER_MODIFIER` ではなく `ON_CALC_ATK_TYPE_MODIFIER` 固定。
- こうすることで `いのちのたま` など最終倍率と独立し、既存テストの差分を最小化できる。
- `かがくへんかガス` / `とくせいなし` で無効化中はイベントゲートにより自動で不発にする。

## テストケース
1. 非テラスタルでタイプ一致技: 2.0倍になる。
2. 非一致技: 補正が掛からない。
3. テラ一致かつ元タイプ一致: 2.25倍になる。
4. テラ一致のみ: 2.0倍になる。
5. テラで失った元タイプ技: 1.5倍に下がる。
6. ステラテラスタル時: てきおうりょく補正が乗らない。
7. かがくへんかガス中: 補正が消える。
