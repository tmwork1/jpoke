# しゅうかく 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/しゅうかく.md`
- 実装方針: バトル効果を先に実装し、例外は段階的に拡張する。
- 非スコープ: 第八世代のフィールド遭遇効果（本リポジトリはバトルロジック中心のため対象外）。
- 非スコープ: きのみ消費後に別アイテムへ差し替わる履歴追跡（実装簡略化のため当面対応しない）。

## 実装対象イベント（第一候補）
- `Event.ON_TURN_END_5`

## イベント選定理由
- しゅうかく判定は「ターン終了時に1回」のため、ターン終了イベントに集約する。
- 天候カウントは先行フェーズで減算されるため、判定時点の天候をそのまま参照できる。
- 既存実装でもターン終了時に発動する特性が `ON_TURN_END_5` を利用している。

## 仕様要点（実装に反映）
- ターン 終了時、自分が最後に使用した きのみ を50%の確率で復活させる。 天気 が にほんばれ ・ おおひでり 状態のときは必ず復活する。
- もちもの を持っているときは発動しない。
- 消費した きのみ が復活するかどうかの判定は、毎 ターン 終了時に1回だけ行われる( ターン#5.ターン終了時の処理 も参照)。
- ターン経過でにほんばれ状態が終了する場合、しゅうかくの発動判定より早く天気が戻るためそのターンのしゅうかく発動率は50%となる。
- ノーてんき / エアロック のポケモンが場にいる場合、にほんばれ / おおひでり 状態でも発動率は50%となる。
- 復活後、使用条件を満たすきのみは即座に使用される。

## 実装フロー
1. アイテム全般に `lost` と `lost_cause` を追加する。
	- `lost`: そのアイテムが持ち主の手元から失われたかどうか
	- `lost_cause`: 喪失理由（例: `consume`, `steal`, `remove`, `swap`, `transfer`）
	- しゅうかく判定は `lost=True` かつ `lost_cause=consume` を基本条件にする。
2. きのみ消費/喪失/交換の主要経路から上記プロパティを更新する。
	- 自発消費: `lost=True`, `lost_cause=consume`（しゅうかく対象）
	- 奪取/破壊/交換/譲渡: `lost=True`, `lost_cause!=consume`（しゅうかく対象外）
3. `src/jpoke/handlers/ability.py` に `しゅうかく_on_turn_end` を追加する。
	- 発動条件: しゅうかく有効、場にいる、持ち物なし、復活候補あり
	- 確率条件: 通常50%、有効な晴れ下は100%
	- 抑制条件: ノーてんき/エアロック下では晴れ扱い無効
4. `src/jpoke/data/ability.py` の `しゅうかく` に `Event.ON_TURN_END_5` ハンドラを登録する。
5. 復活後にきのみ即時使用判定を行い、必要ならその場で消費まで反映する。
6. 特性無効化（`かがくへんかガス` / `とくせいなし`）との相互作用を確認する。

## 段階実装
### Phase 1（先行実装）
- ON_TURN_END_5 での復活判定（50%/晴れ100%/持ち物チェック）
- 自発消費したきのみの復活
- かがくへんかガス中は不発、解除後は再び有効

### Phase 2（例外拡張）
- やきつくす/はたきおとす/むしくい/ついばむ/ふしょくガス起因の対象外処理
- トリック/すりかえ/どろぼう/ほしがる等の受け渡し起因の対象外処理

## テスト観点
- 50%判定の成功/失敗を乱数固定で再現できる。
- 晴れ継続ターンは100%、晴れ終了ターンは50%になる。
- ノーてんき/エアロック下では晴れ中でも50%になる。
- 持ち物あり、候補なし、控えにいる状態では発動しない。
- 復活後、使用条件を満たすきのみは即時使用される。
- かがくへんかガス中は発動せず、解除後に再発動する。
- Phase 2 対象の喪失経路で誤復活しない。
- 非スコープ（差し替え履歴）に対しては未対応であることを確認する。
