# しゅうかく 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/しゅうかく.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_FIELD_CHANGE`
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_CALC_POWER_MODIFIER`

## 仕様要点（実装に反映）
- ターン 終了時、自分が最後に使用した きのみ を50%の確率で復活させる。 天気 が にほんばれ ・ おおひでり 状態のときは必ず復活する。
- 第八世代 のみ、手持ちの先頭にこのとくせいを持つポケモンがいると、出現可能な くさ タイプの 野生ポケモン が50%の確率で出現する。 ひんし 状態のときも効果がある。
- もちもの を持っているときは発動しない。
- 消費した きのみ が復活するかどうかの判定は、毎 ターン 終了時に1回だけ行われる( ターン#5.ターン終了時の処理 も参照)。
- ターン経過でにほんばれ状態が終了する場合、しゅうかくの発動判定より早く天気が戻るためそのターンのしゅうかく発動率は50%となる。

## 実装フロー
1. `src/jpoke/data/ability.py` の `しゅうかく` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `しゅうかく` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
