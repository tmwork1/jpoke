# きよめのしお 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/きよめのしお.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_CHECK_DEF_ABILITY`
- `Event.ON_CALC_POWER_MODIFIER`
- `Event.ON_CALC_ATK_MODIFIER`

## 仕様要点（実装に反映）
- 状態異常 ・ ねむけ 状態にならない。
- ゴースト タイプの 攻撃技 を受けたとき、相手の こうげき ・ とくこう を半減して ダメージ を計算する。
- この特性のポケモンが使用する ねむる は失敗する。
- 持たせた かえんだま / どくどくだま の効果は発動しない。
- こんらん 状態などの 状態変化 は防げない。

## 実装フロー
1. `src/jpoke/data/ability.py` の `きよめのしお` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `きよめのしお` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
