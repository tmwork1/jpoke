# トレース 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/トレース.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`
- `Event.ON_BEFORE_APPLY_AILMENT`

## 仕様要点（実装に反映）
- 場に出たとき、自分の特性を相手の場にいるポケモンと同じ 特性 にする。対象となる相手が複数いる場合、ランダムで1体が選ばれる。
- 手持ちに戻るか、戦闘が終了すると特性はトレースに戻る。
- 発動すると、 第四世代 では「<特性所有者>は (対象が味方であるときは無し)/やせいの/相手の <対象の ニックネーム >の <コピーした特性名>を トレースした!」というメッセージが出る。 第五世代 以降では、「<特性所有者>...
- 相手の場に複数匹同じ名前のポケモンがいる場合、コピー元になった個体の判別はできない。ただし、 第九世代 の場合は特性バーと同時にポケモンのアイコンが表示されるので、外見が異なるポケモンであれば判別できる。
- 場に出たときに発動する特性 をコピーした場合、世代により仕様が異なる。

## 実装フロー
1. `src/jpoke/data/ability.py` の `トレース` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `トレース` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
