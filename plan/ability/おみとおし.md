# おみとおし 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/おみとおし.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_FIELD_CHANGE`
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`

## 仕様要点（実装に反映）
- 場に出たときに相手の場にいるポケモンが もちもの を持っている場合、何を持っているかが分かる。
- 相手の場にもちものを持つポケモンが複数いる場合、相手全員のもちものが分かる ( 第六世代 以降) 。
- 第五世代 以前は、ランダムで1体のポケモンのもちものが分かる。
- 相手の場に 持ち物 を持つポケモンがいないときは発動しない。
- 持ち物がない相手が交代して、持ち物があるポケモンが繰りだされたときなどは発動しない。

## 実装フロー
1. `src/jpoke/data/ability.py` の `おみとおし` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `おみとおし` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
