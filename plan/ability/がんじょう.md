# がんじょう 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/がんじょう.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_CHECK_DEF_ABILITY`

## 仕様要点（実装に反映）
- 一撃必殺技 を受けても無効化する。
- 第五世代 以降では、HPが満タンのとき一撃で ひんし になるダメージを受けても、HPを1残して持ちこたえる。
- 特性 かたやぶり の効果がある一撃必殺技は防げない。
- 一撃必殺技 ががんじょうで無効化されるとき、 第四世代 までは「<ポケモン>は がんじょうで きかなかった!」というメッセージが出る。 第五世代 以降はがんじょうの特性バーと「<ポケモン>には 効果が ないようだ…」というメッセー...
- 一撃必殺技を受けたときは、 命中 判定が行われる前にがんじょうで無効化される。

## 実装フロー
1. `src/jpoke/data/ability.py` の `がんじょう` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `がんじょう` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
