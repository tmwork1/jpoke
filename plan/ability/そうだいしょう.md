# そうだいしょう 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/そうだいしょう.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_CALC_POWER_MODIFIER`
- `Event.ON_CALC_ATK_MODIFIER`
- `Event.ON_CALC_DEF_MODIFIER`

## 仕様要点（実装に反映）
- 場に出たとき、その戦闘で ひんし になった味方ポケモンの数に応じて、使用する 攻撃技 の 威力 が上昇する効果が発動する。ひんし1匹につき10%ずつ補正率が上がる。最大補正率は+50%。
- その戦闘で味方が ひんし になってから場に出たときに発動し、特性バーと「<ポケモン>は 倒された 仲間から 力を もらった!」とメッセージが出る。
- いかく など、多くの 場に出たときに発動する特性 とは補正込みの すばやさ が高い順に発動する。
- その戦闘でひんしになった味方がいないときは特性バーとメッセージは現れない。
- 戦闘が開始する前からひんしになっていた味方はカウントしない。

## 実装フロー
1. `src/jpoke/data/ability.py` の `そうだいしょう` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `そうだいしょう` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
