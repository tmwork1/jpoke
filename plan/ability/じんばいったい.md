# じんばいったい 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/じんばいったい.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`

## 仕様要点（実装に反映）
- 戦闘中、相手は きのみ を使用できなくなる。
- 自分の 攻撃技 で他のポケモンを ひんし にさせたとき、はくばじょうのすがたの場合は こうげき が 1段階 上がる。こくばじょうのすがたの場合は とくこう が1段階上がる。
- きんちょうかん と しろのいななき 、またはきんちょうかんと くろのいななき の2つの特性の効果をあわせ持つ。場に出ると、じんばいったいの特性バーが現れ「バドレックスは ふたつの 特性を あわせ持つ!」というメッセージが出た後、き...
- 場に出た直後、 テラスチェンジ / かがくへんかガス の後、 設置技 の効果の前にじんばいったいのメッセージが現れる。他のじんばいったいやきんちょうかんのポケモンと同時に場に出た場合は すばやさ 順に発動する。それ以外の 場に出た...
- この特性は コピーできない 。 とくせいなし 状態にもならず、 かがくへんかガス によって無効化されることもない。

## 実装フロー
1. `src/jpoke/data/ability.py` の `じんばいったい` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `じんばいったい` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
