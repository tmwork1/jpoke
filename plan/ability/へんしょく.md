# へんしょく 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/へんしょく.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_CHECK_DEF_ABILITY`

## 仕様要点（実装に反映）
- 他のポケモンから 攻撃技 の ダメージ を受けた後、その技と同じ タイプ に自分のタイプが変化する。
- タイプが変化する技 ・ 技のタイプを変える技 ・ 技のタイプを変える特性 の効果でタイプが変わっている技を受けた場合、最終的な技のタイプに変化する。
- 他の技が出る技 で繰り出された攻撃技を受けたときは、その攻撃技のタイプに変化する。
- 受けた技のタイプをすでに自分が持っている場合、特性は発動しない。
- 自分が複合タイプである場合、そのいずれかのタイプの技に対しては発動せず、単タイプにはならない。いずれのタイプでもない技を受けると発動して単タイプになる。

## 実装フロー
1. `src/jpoke/data/ability.py` の `へんしょく` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `へんしょく` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
