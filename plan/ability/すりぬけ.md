# すりぬけ 実装計画

## スコープ
- 優先度: H
- 仕様書: `spec/ability/すりぬけ.md`
- 実装の核: 「敵側の壁/しんぴのまもり/しろいきり/みがわり」の参照判定だけを貫通させる。

## 実装対象イベント
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`
- 壁/しんぴのまもり/しろいきりを参照する既存計算イベント
- 通常は `Event.ON_CALC_DEF_MODIFIER` と状態付与前イベント（`Event.ON_BEFORE_APPLY_AILMENT`, `Event.ON_BEFORE_MODIFY_STAT`）

## 実装方針（確定）
1. `src/jpoke/data/ability.py` の `すりぬけ` に `ON_CHECK_HIT_SUBSTITUTE` ハンドラを登録する。
2. `src/jpoke/handlers/ability.py` に `すりぬけ_check_hit_substitute` を実装し、攻撃側が `すりぬけ` かつ対象が敵なら貫通を返す。
3. 壁/しんぴのまもり/しろいきりは、参照側のハンドラ（field/side_field）に「攻撃側がすりぬけなら敵側だけ無視」の条件を追加する。
4. 味方側の場状態は無視しないことを条件分岐で固定する。
5. `Event.ON_CHECK_SUBSTITUTE` と `Event.ON_CHECK_HIT_SUBSTITUTE` の責務を揃え、みがわり判定の二重適用を防ぐ。

## 実装メモ
- 「無視」は「状態を消す」ではない。場状態の残りターンはそのまま維持する。
- かわらわり系の壁破壊とは切り分ける（すりぬけ実装で壁削除をしない）。
- 特性無効化中は貫通しない。

## テストケース
1. 相手リフレクター中、物理技ダメージ軽減が無効化される。
2. 味方リフレクター中、自分側の軽減は無効化されない。
3. 相手みがわり中、対象技が本体へ通る。
4. 味方みがわりは無視しない。
5. しんぴのまもり/しろいきりを敵側だけ無視する。
6. かがくへんかガス中は貫通しない。
