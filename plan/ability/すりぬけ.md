## 効果まとめ（第9世代シングル）

| 効果                                   | 対象フィールド                               | 既実装 |
| -------------------------------------- | -------------------------------------------- | ------ |
| 壁のダメージ軽減を無視                 | リフレクター / ひかりのかべ / オーロラベール | ✓      |
| しんぴのまもりによる状態異常無効を無視 | しんぴのまもり                               | ✓      |
| しんぴのまもりによる揮発状態無効を無視 | しんぴのまもり                               | ✓      |
| しろいきりによる能力低下ガードを無視   | しろいきり                                   | ✓      |
| みがわりを無視して攻撃できる           | みがわり（volatile）                         | ✓      |

---

## 設計方針

`python.instructions.md` の責務設計原則：
> 相手の技が自分の特性で無効化される場合、**効果を所持している側** がハンドラを実装する。

`ON_CHECK_DEF_ABILITY_ENABLED` / `ctx.check_def_ability_enabled(battle)` という既存パターンと対称的に、`ON_CHECK_INFILTRATE` イベントと `ctx.check_infiltrate(battle)` メソッドを新設する。

- **すりぬけ側**が `ON_CHECK_INFILTRATE` ハンドラを登録し `True` を返す
- **フィールド・volatile ハンドラ側**は `ctx.check_infiltrate(battle)` で問い合わせるだけ
- `field.py` / `volatile.py` が特性名 `"すりぬけ"` をハードコードしない（疎結合）
- 将来同挙動の特性が追加されても `data/ability.py` に追加するだけで済む

```python
ON_CHECK_INFILTRATE = auto()
# 攻撃側がすりぬけ系特性を持つかを問い合わせる。base=False、True で貫通。
```

---

### ② `src/jpoke/core/context.py`

`check_def_ability_enabled` と対称的なメソッドを追加：

```python
def check_infiltrate(self, battle: Battle) -> bool:
    """攻撃側のすりぬけ特性が有効かどうかを返す。"""
    return battle.events.emit(
        Event.ON_CHECK_INFILTRATE,
        self,
        False,
    )
```

---

### ③ `src/jpoke/data/ability.py`

```python
"すりぬけ": AbilityData(
    handlers={
        Event.ON_CHECK_INFILTRATE: AbilityHandler(
            h.すりぬけ_check_infiltrate,
            subject_spec="attacker:self",
        )
```

---

### ④ `src/jpoke/handlers/ability.py`

```python
def すりぬけ_check_infiltrate(battle: Battle, ctx: BattleContext, value: bool) -> HandlerReturn:
    """すりぬけ特性: 攻撃側が壁・みがわり等を貫通する。"""
    if ctx.attacker.ability.enabled:
        return HandlerReturn(value=True, stop_event=True)
    return HandlerReturn(value=value)
```

---

### ⑤ `src/jpoke/handlers/field.py`

各ハンドラに `ctx.check_infiltrate(battle)` チェックを追加。`_attacker_has_infiltrator` ヘルパーは不要。

#### 変更するハンドラ一覧

| 関数名                            | 変更内容                                      |
| --------------------------------- | --------------------------------------------- |
| `リフレクター_reduce_damage`      | すりぬけ有効なら `value` をそのまま返す       |
| `ひかりのかべ_reduce_damage`      | 同上                                          |
| `オーロラベール_reduce_damage`    | 同上                                          |
| `しんぴのまもり_prevent_ailment`  | すりぬけ有効なら `value`（元の状態異常名）をそのまま返す |
| `しんぴのまもり_prevent_volatile` | すりぬけ有効なら `value`（揮発状態名）をそのまま返す |
| `しろいきり_prevent_stat_drop`    | すりぬけ有効なら `value` をそのまま返す       |

#### 変更イメージ（リフレクターの例）

```python
def リフレクター_reduce_damage(battle: Battle, ctx: BattleContext, value: Any) -> HandlerReturn:
    if getattr(ctx, "critical", False) or ctx.check_infiltrate(battle):
        return HandlerReturn(value=value)
    if ctx.move.category == "物理":
        value = value * 2048 // 4096
    return HandlerReturn(value=value)
```

---

### ⑥ `src/jpoke/handlers/volatile.py`

同様に `ctx.check_infiltrate(battle)` チェックを追加。

#### 変更イメージ（`みがわり_immune` の例）

```python
def みがわり_immune(battle: Battle, ctx: BattleContext, value: Any) -> HandlerReturn:
    hit_substitute = battle.move_executor.hit_substitute(ctx)
    immune = hit_substitute and ctx.move.category == "変化" and not ctx.check_infiltrate(battle)
    if immune:
        battle.add_event_log(ctx.defender, LogCode.MOVE_IMMUNE,
                             payload={"move": ctx.move.name, "reason": "みがわり"})
        return HandlerReturn(value=True, stop_event=True)
    return HandlerReturn(value=False)
```

`みがわり_modify_damage` も同様に `ctx.check_infiltrate(battle)` で貫通判定を追加。

## みがわり無視の実装方針

みがわりは実装済み（`data/volatile.py` + `handlers/volatile.py`）。

- `ON_CHECK_IMMUNE` → `みがわり_immune`: 変化技を無効化（`stop_event=True`）
- `ON_MODIFY_DAMAGE` → `みがわり_modify_damage`: 攻撃技のダメージを肩代わり

両ハンドラに `ctx.check_infiltrate(battle)` チェックを追加する。
