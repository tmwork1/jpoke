# ぎゃくじょう 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/ぎゃくじょう.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_BEFORE_MODIFY_STAT`

## 仕様要点（実装に反映）
- 相手の 攻撃技 を受けて、自分の HP が最大HPの1/2超から1/2以下に減らされたとき、 とくこう が 1段階 上がる。
- オボンのみ / ヤタピのみ などの きのみ を所持しており、ぎゃくじょうときのみの発動条件を同時に満たした場合、基本的にはぎゃくじょうが発動してからきのみが発動する。
- ナゾのみ はぎゃくじょうの前に発動判定がある。ナゾのみで回復してなおHPが半分以下でなければぎゃくじょうは発動しない。
- 連続攻撃技 を受けた場合、途中でHPが半分になっても全ての攻撃が終わるまでぎゃくじょうは発動しない。攻撃を受ける前にHPが1/2超あり、攻撃が終わった後のHPが1/2以下になったときにぎゃくじょうが発動する。
- 連続攻撃技を受けたときは各攻撃ごとにきのみの発動判定があるため、きのみが発動してからぎゃくぎょうが発動するようになる。

## 実装フロー
1. `src/jpoke/data/ability.py` の `ぎゃくじょう` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `ぎゃくじょう` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
