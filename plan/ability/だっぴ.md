# だっぴ 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/だっぴ.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_CALC_POWER_MODIFIER`
- `Event.ON_CALC_ATK_MODIFIER`
- `Event.ON_CALC_DEF_MODIFIER`

## 仕様要点（実装に反映）
- ターン 終了時に、自分の 状態異常 が30%の確率で回復する。
- 第三世代 のみ、発動率は1/3となる。
- こんらん などの 状態変化 は治せない。
- 発動のタイミングは ターン の終了時になる。詳細なタイミングは以下のようになる。
- どく / もうどく / やけど / あくむ 状態や、特性 ナイトメア のダメージ判定より前なので、だっぴが発動するターンはこれらのダメージを受けない。

## 実装フロー
1. `src/jpoke/data/ability.py` の `だっぴ` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `だっぴ` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
