# いろめがね 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/いろめがね.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_CALC_POWER_MODIFIER`
- `Event.ON_CALC_ATK_MODIFIER`

## 仕様要点（実装に反映）
- 使用した技が「 効果は今ひとつ 」だったとき、与える ダメージ が2倍になる。相性判定そのものは変化しない。
- 技が無効化されたときは特性の効果も発動しない。
- 相手が複合タイプを持つときは、最終的な相性が今ひとつのときに効果が発動する。片方のタイプに対して半減でも、もう片方に対して抜群のため最終的な相性が等倍になったときは特性は発動しない。
- 全体攻撃技を使用した場合、効果が今ひとつだった相手に対してのみ与えるダメージが増える。
- 第五世代 以降は みらいよち / はめつのねがい の ダメージ 発生時に使用者の特性がいろめがねで場に出ているならば、いろめがねの効果は発動する。

## 実装フロー
1. `src/jpoke/data/ability.py` の `いろめがね` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `いろめがね` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
