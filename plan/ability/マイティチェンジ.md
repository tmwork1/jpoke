# マイティチェンジ 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/マイティチェンジ.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_CALC_POWER_MODIFIER`
- `Event.ON_CALC_ATK_MODIFIER`
- `Event.ON_CALC_DEF_MODIFIER`

## 仕様要点（実装に反映）
- 「ナイーブ フォルム 」の イルカマン が他のポケモンに 交代 したとき、「マイティフォルム」に フォルムチェンジ する。
- 使うと交代する技 / 相手を交代させる技 や、 だっしゅつボタン / だっしゅつパック / レッドカード の効果で交代したときも発動する。
- 手持ちに戻った時点で フォルムチェンジ は行われる。
- ナイーブフォルムのイルカマンが ひんし になって場から去ったときは発動しない。
- マイティフォルムのイルカマンはひんしになってもナイーブフォルムに戻らない。戦闘中に復活した場合、次に場に出たときもマイティフォルムのままとなる。

## 実装フロー
1. `src/jpoke/data/ability.py` の `マイティチェンジ` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `マイティチェンジ` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
