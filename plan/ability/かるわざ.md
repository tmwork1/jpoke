# かるわざ 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/かるわざ.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `DomainEvent.ON_MODIFY_MOVE_PRIORITY`
- `Event.ON_SWITCH_IN`
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_BEFORE_MODIFY_STAT`

## 仕様要点（実装に反映）
- 場に出た後に自分の 持ち物 が無くなると すばやさ が2倍になる。再び持ち物を手にするとすばやさは元に戻る。場に出たときから持ち物がない場合は発動しない。
- 一度 持ち物 を消費して、かるわざを発動させてから 交代 で引っ込んだ場合、再び場に出てきても「初めから何も持っていなかった」扱いになり、かるわざは発動しない。
- 場に出たときと同時に持ち物を消費したときはかるわざは発動する。
- 道具を消費する手段は問わない。以下の状況でかるわざは発動する。
- 普通に道具を消費した場合 ( なげつける / しぜんのめぐみ / ほおばる を使用したときも有効)。

## 実装フロー
1. `src/jpoke/data/ability.py` の `かるわざ` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `かるわざ` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
