# マルチタイプ 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/マルチタイプ.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_CHECK_DEF_ABILITY`
- `Event.ON_CALC_POWER_MODIFIER`
- `Event.ON_CALC_ATK_MODIFIER`

## 仕様要点（実装に反映）
- 持って いる プレート に応じて自身の フォルム と タイプ が変わる（ プレート#一覧 を参照）。
- 第七世代 のみ、持っている Zクリスタル に応じて自身のフォルムとタイプが変わる。
- この特性により アルセウス のタイプが変わると、それに合わせて フォルム も変わる。
- 手持ち画面に映される小アイコンの姿に持たせた プレート が反映されるかは、世代によって異なる。
- 第八世代 以前では持たせたプレートに関係無く通常時のフォルムが表示される。 通信対戦 の対戦相手は、見せ合い時点ではプレートの有無と種類を判別できない。

## 実装フロー
1. `src/jpoke/data/ability.py` の `マルチタイプ` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `マルチタイプ` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
