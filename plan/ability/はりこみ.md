# はりこみ 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/はりこみ.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_CALC_POWER_MODIFIER`
- `Event.ON_CALC_ATK_MODIFIER`
- `Event.ON_CALC_DEF_MODIFIER`

## 仕様要点（実装に反映）
- ポケモンチェンジ で出てきたポケモンへ攻撃するときの こうげき ・ とくこう が2倍になる。
- 使うと交代するわざ で交代して出てきた相手に攻撃したときも発動する。
- 持ち物 レッドカード ・ だっしゅつボタン ・ だっしゅつパック 、特性 にげごし ・ ききかいひ の効果で交代して出てきた相手に攻撃したときも発動する。
- 初手で繰り出された相手や、死に出しで繰り出された相手に攻撃しても発動しない。
- かわりもの が発動したポケモンに対しては発動しない。

## 実装フロー
1. `src/jpoke/data/ability.py` の `はりこみ` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `はりこみ` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
