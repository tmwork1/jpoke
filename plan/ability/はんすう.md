# はんすう 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/はんすう.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_BEFORE_MODIFY_STAT`
- `Event.ON_MODIFY_STAT`

## 仕様要点（実装に反映）
- きのみ を消費したとき、次の ターン の終了時にまた同じきのみを食べる。途中で 交代 すると発動しなくなる。
- 発動するまでに複数のきのみを食べた場合、最後に消費した1つだけがはんすうの対象となり、最後にきのみを食べた次のターンの終了時に発動する。
- 特性で 持ち物 が復活するわけではない。 きのみ を食べたポケモンなら、持ち物があるときでもはんすうは発動する。
- 食べた きのみ にHP割合などの発動条件があったしても、はんすうで食べる際には影響が無い。
- オボンのみ / カムラのみ なども、はんすう発動時のHP割合に関係なく効果を得られる。

## 実装フロー
1. `src/jpoke/data/ability.py` の `はんすう` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `はんすう` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
