# さいせいりょく 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/さいせいりょく.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_CHECK_DEF_ABILITY`

## 仕様要点（実装に反映）
- 他のポケモンに 交代 すると、自身の最大 HP の1/3が回復する (小数点以下は切り捨て)。
- 使うと交代する技 / 相手を交代させる技 や、 だっしゅつボタン / だっしゅつパック / レッドカード の効果で 交代 したときも発動する。
- しっぽきり で交代する場合、技の効果でHPが減少した後にさいせいりょくの発動判定がある。途中で きのみ の発動圏内に入った場合、きのみの発動が挟まれる。
- さいせいりょくによる回復は、交代先のポケモンを選択するタイミングですでに完了している。
- 本来他の特性を持つポケモンが、 特性を変える技 / 特性 の効果で一時的にさいせいりょくを得ているときに交代した場合、さいせいりょくの効果は発動する。

## 実装フロー
1. `src/jpoke/data/ability.py` の `さいせいりょく` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `さいせいりょく` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
