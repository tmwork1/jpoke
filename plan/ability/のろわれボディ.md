# のろわれボディ 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/のろわれボディ.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`
- `Event.ON_CHECK_DEF_ABILITY`
- `Event.ON_CALC_POWER_MODIFIER`

## 仕様要点（実装に反映）
- 他のポケモンから 攻撃技 による ダメージ を受けたとき、30%の確率でその技を かなしばり 状態にする。非 直接攻撃 に対しても発動する。
- この特性で発動した かなしばり 状態は、発動したターンを含めて4ターンの間続く。
- ダメージ を受けなかったときは発動しない。
- 変化技 などダメージが無い技に対しては発動しない。 いたみわけ に対しても発動しない。
- 相性 / まもる 状態などで攻撃を防いだときは発動しない。

## 実装フロー
1. `src/jpoke/data/ability.py` の `のろわれボディ` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `のろわれボディ` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
