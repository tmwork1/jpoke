# にげごし 実装計画

## スコープ
- 優先度: H
- 仕様書: `spec/ability/にげごし.md`
- 実装の核: HPが 50% をまたいだ瞬間に、割り込み交代（野生戦は逃走）を一度だけ予約する。

## 実装対象イベント/処理点
- HP変化の集中点（`Battle.modify_hp` または同等処理）
- `Event.ON_DAMAGE`（被ダメ由来）
- `Event.ON_TURN_END_1..3`（スリップ由来）
- 既存の割り込み交代処理（だっしゅつパック系）

## 実装方針（確定）
1. `src/jpoke/data/ability.py` の `にげごし` に「発動条件判定ハンドラ」を登録する。
2. 条件は「変更前HP > 1/2 かつ 変更後HP <= 1/2」。
3. 発動時は即時交代ではなく「強制交代フラグ」を立て、既存割り込み解決フェーズに乗せる。
4. 交代不能状態（かげふみ/バインド等）はこの特性だけ例外的に無視する。
5. 同一滞在中の再発動を防ぐフラグを `Pokemon` に保持し、交代でリセットする。

## 非発動ルール
- こんらん自傷ダメージでは発動しない。
- いたみわけ由来のHP低下では発動しない。
- 特性無効化中（かがくへんかガス/とくせいなし）は発動しない。

## テストケース
1. 被ダメで 50% をまたぐと交代予約される。
2. 50%ちょうどからさらに減っても新規発動しない。
3. 毒/やけど等のスリップで 50% をまたいでも発動する。
4. こんらん自傷では発動しない。
5. いたみわけでは発動しない。
6. 交代不能状態でも発動して引っ込む。
