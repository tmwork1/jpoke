# にげごし 実装計画

## スコープ
- 優先度: H
- 仕様書: `spec/ability/にげごし.md`
- 実装の核: HP が「1/2超 -> 1/2以下」に落ちたとき、既存の割り込み交代基盤 `Interrupt.EMERGENCY` を立てて処理させる。

## 事前確認済みの実装面
- `src/jpoke/core/turn.py` には `Interrupt.EMERGENCY` の解決ポイントがすでにあり、技後とターン終了各段で割り込み交代を処理する。
- `src/jpoke/core/switch.py` の `run_interrupt_switch()` が、だっしゅつボタン・だっしゅつパックと同じ系統の割り込み交代を一括処理する。
- `src/jpoke/data/ability.py` の `にげごし` は未登録で、現状は `flags=["undeniable"]` のみ。
- HP 半分閾値の近縁実装として `ぎゃくじょう` が `Event.ON_DAMAGE` 上で「変更前HP / 変更後HP」を計算している。
- 一方で `CommandManager.get_available_switch_commands()` は通常交代の可否判定に `is_trapped()` を使うため、にげごし専用に交代禁止を無視するなら別導線が必要。

## 実装方針
1. `src/jpoke/handlers/ability.py` に、にげごし専用ハンドラを追加する。
2. 技ダメージ起点は `Event.ON_DAMAGE` で受け、`ぎゃくじょう` と同様に連続技を最終ヒット後にまとめて判定する。
3. スリップ起点は `Event.ON_TURN_END_1..4` のうち実際にHP減少が起きる段を対象にし、`Battle.modify_hp(..., reason=...)` の reason を手掛かりに対象要因だけを拾う。
4. 発動条件は共通化し、「変更前HP > 最大HP/2」「変更後HP <= 最大HP/2」「瀕死でない」「交代先が存在する」を同時に満たしたときだけ `player.interrupt = Interrupt.EMERGENCY` を立てる。
5. 割り込みの実行順は既存の `Turn.run()` に委ね、だっしゅつボタン優先・にげごし後続という現実装の順序を崩さない。

## 実装タスク
1. `src/jpoke/data/ability.py` の `にげごし` にハンドラ登録を追加する。
2. `src/jpoke/handlers/ability.py` に「閾値判定」と「割り込み予約」を分けた小関数を追加し、技ダメージ用とターン終了用から共用する。
3. スリップ起点で除外すべき理由を列挙する。
	- 少なくとも `こんらん自傷` と `いたみわけ` は非発動。
	- 実際に `Event.ON_TURN_END_*` を通らない要因は対象から外す。
4. 交代可否判定を整理する。
	- 交代先不在では不発。
	- 交代禁止系を無視する仕様を採るなら、通常の `get_available_switch_commands()` とは別に「強制交代用の候補取得」を追加する。
	- 交代禁止系を無視しない仕様を採るなら、仕様書側を現在コードと矛盾しない文面に寄せる。
5. 発動ログが必要なら `LogCode.ABILITY_TRIGGERED` を既存特性と同様に記録する。

## 判定ルール
### 発動
- 攻撃技でダメージを受け、最終的に HP が半分以下になった。
- どく・やけど・天候・しおづけ・バインド等の定期ダメージで HP が半分以下になった。

### 非発動
- ダメージ前からすでに HP が半分以下だった。
- 瀕死になった。
- 交代先がいない。
- `こんらん自傷`、`いたみわけ` など仕様書で除外済みの減少要因。
- 特性無効化中。

## テスト計画
1. 単発攻撃で `1/2超 -> 1/2以下` になった直後に `Interrupt.EMERGENCY` が立ち、技後の割り込み交代で引っ込む。
2. 連続技で途中ヒット時には発動せず、最終ヒット後にのみ発動する。
3. HP がちょうど半分の状態からさらに減っても発動しない。
4. どく・やけど・すなあらし・バインド等のターン終了ダメージで条件を満たした場合に発動する。
5. `こんらん自傷` と `いたみわけ` では発動しない。
6. だっしゅつボタン同時発動時に、だっしゅつボタンが先、にげごしが後になる。
7. 交代先不在では `Interrupt.EMERGENCY` が立たない。

## 未解決点
- 交代禁止系を無視するかどうかは、`spec/ability/にげごし.md` の簡略記述よりも各詳細仕様との整合確認が必要。
- シングル前提の現実装では「野生戦では逃走」はスコープ外。手持ち交代だけを対象にする。
