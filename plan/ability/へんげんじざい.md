# へんげんじざい 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/へんげんじざい.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `DomainEvent.ON_MODIFY_MOVE_PRIORITY`
- `Event.ON_SWITCH_IN`
- `Event.ON_FIELD_CHANGE`
- `Event.ON_CHECK_HIT_SUBSTITUTE`

## 仕様要点（実装に反映）
- 技 を出す直前に、自分の タイプ がその技のタイプに変化する。 交代 して手持ちに戻ると元のタイプに戻る。
- 第九世代 では、1回場に出るごとにつき1回しか発動しない。
- 攻撃技 を使用して発動するときは、特性が発動した後に ダメージ 計算が行われるため、 タイプ一致ボーナス を得られる。
- 変化技 を使用するときも発動する。
- どくどく を使用した場合、 どく タイプに変わった後に 命中 判定が行われ、技は必中になる。

## 実装フロー
1. `src/jpoke/data/ability.py` の `へんげんじざい` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `へんげんじざい` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
