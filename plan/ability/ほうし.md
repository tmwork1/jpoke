# ほうし 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/ほうし.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_CALC_POWER_MODIFIER`

## 仕様要点（実装に反映）
- 直接攻撃 で ダメージ を受けたとき、一定確率で相手を どく ・ まひ ・ ねむり 状態のいずれかにする 。
- 第三世代 では、10%の確率で特性が発動する。いずれかの 状態異常 が等確率 (約3.3%) で選ばれる。
- 第四世代 では、30%の確率で特性が発動する。いずれかの状態異常が等確率 (10%) で選ばれる。
- 第五世代 以降では、30%の確率で特性が発動する。9%の確率でどく状態、10%の確率でまひ状態、11％の確率でねむり状態にする 。
- 直接攻撃 の ダメージ を 相性 ・ まもる 状態で無効化したときは発動しない。

## 実装フロー
1. `src/jpoke/data/ability.py` の `ほうし` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `ほうし` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
