# ダウンロード 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/ダウンロード.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_FIELD_CHANGE`
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`

## 仕様要点（実装に反映）
- 場に出たときの相手の 防御 と 特防 の値を比較して、防御＜特防なら自分の 攻撃 が、防御≧特防なら 特攻 が 1段階 上がる。
- 値を比較する際、 防御 ・ 特防 補正の効果については、以下のように扱う。
- ランク補正 / パワートリック / ガードシェア の効果を考慮する
- 持ち物 / 特性 / すなあらし / ゆき の効果を除外する。
- ワンダールーム 状態のときは、相手の防御と特防のランク補正のみ入れ替えた能力を比較する。(防御×特防ランク補正率)＜(特防×防御ランク補正率)なら攻撃が、そうでなければ特攻が上がる。ワンダールーム状態では有効でない方の能力が上がる...

## 実装フロー
1. `src/jpoke/data/ability.py` の `ダウンロード` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `ダウンロード` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
