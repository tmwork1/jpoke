# くいしんぼう 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/くいしんぼう.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_CHECK_DEF_ABILITY`
- `Event.ON_CALC_POWER_MODIFIER`

## 仕様要点（実装に反映）
- 本来は残りHPが最大 HP の1/4以下になったときしか発動しない きのみ を、最大HPの1/2以下になったときに発動できる。
- もとから最大HPの1/2以下で発動するきのみ（ オボンのみ など）に対しては効果は無い。発動条件が残りHPでないきのみ（ ラムのみ など）も同様に効果は無い。
- 対象となるきのみは以下。
- フィラのみ ( 第七世代 以降)
- ウイのみ (第七世代以降)

## 実装フロー
1. `src/jpoke/data/ability.py` の `くいしんぼう` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `くいしんぼう` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
