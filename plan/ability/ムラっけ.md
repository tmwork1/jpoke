# ムラっけ 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/ムラっけ.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_BEFORE_MODIFY_STAT`
- `Event.ON_MODIFY_STAT`

## 仕様要点（実装に反映）
- ターン 終了時、自分の こうげき ・ ぼうぎょ ・ とくこう ・ とくぼう ・ すばやさ の内、いずれか1つの ステータス が 2段階 上がり、別のいずれか1つのステータスが1段階下がる。
- 第七世代 以前では、 命中率 ・ 回避率 も上下するステータスとして選ばれることがある。
- すでに+6の能力を上げたり、-6の能力を下げたりする変化は選ばれない。
- 上げる能力に+5の能力が選ばれ、結果1段階しか上がらないことは起こりうる。
- 下がる能力はそのターン上がらなかった能力から選ばれる。1ターンに同じ能力が上下することはない。

## 実装フロー
1. `src/jpoke/data/ability.py` の `ムラっけ` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `ムラっけ` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
