# ミイラ 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/ミイラ.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`
- `Event.ON_BEFORE_APPLY_AILMENT`

## 仕様要点（実装に反映）
- 直接攻撃 で ダメージ を受けたとき、相手の 特性 をミイラにする。
- 上書きできない特性 に対しては発動しない。
- 上書きされたミイラは 交代 することで元に戻る。戦闘終了によっても元に戻る。
- ミイラによってミイラになったポケモンが別のポケモンから 直接攻撃 を受けた場合でも、同様に相手をミイラにする。
- 発動すると攻撃者の特性バーが現れ「<攻撃者>は とくせいが ミイラになっちゃった!」とメッセージと共にミイラに書き換わる。

## 実装フロー
1. `src/jpoke/data/ability.py` の `ミイラ` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `ミイラ` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
