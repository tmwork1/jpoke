# ミラーアーマー 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/ミラーアーマー.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_FIELD_CHANGE`
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`

## 仕様要点（実装に反映）
- 自身に対する ランク補正 を下げる効果を相手に跳ね返す。
- 変化技 / 攻撃技 による 追加効果 、 ダイマックスわざ による ランク 低下、 いかく / わたげ などの とくせい などの効果で、他のポケモンによりミラーアーマーのポケモンの能力が下げられようとしたとき、効果を跳ね返して下げ...
- ちからをすいとる を受けた場合、技の使用者の 攻撃 ランクを下げるが、回復量はミラーアーマーのポケモンの攻撃実数値と攻撃ランクにより決まる。ちからをすいとるの使用者が みがわり 状態などで跳ね返した効果が無かったときでも回復は発動...
- すてゼリフ を受けた場合、技の使用者の 攻撃 / 特攻 ランクを下げた後に使用者は交代する。すてゼリフ使用者のみがわり状態などで跳ね返した効果が無かったときも相手は交代する。
- おきみやげ を受けた場合、技の使用者の攻撃/特攻ランクを下げた後に使用者は ひんし になる。

## 実装フロー
1. `src/jpoke/data/ability.py` の `ミラーアーマー` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `ミラーアーマー` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
