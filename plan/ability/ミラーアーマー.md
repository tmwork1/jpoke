# ミラーアーマー 実装計画

## スコープ
- 優先度: H
- 仕様書: `spec/ability/ミラーアーマー.md`
- 実装方針: `ON_BEFORE_MODIFY_STAT` で相手由来の能力ランク低下を検知し、低下量を source（相手）側へ転送して target（自分）側の低下をゼロに置き換える。

## 実装対象イベント
- `Event.ON_BEFORE_MODIFY_STAT`

## 仕様要点（実装に反映）
- `source != target`（相手由来）かつ低下値を含む場合のみ反射。
- 低下値だけを source 側へ `battle.modify_stats(source, ...)` で適用し、自分側の戻り値からは低下分を除去（ゼロ化）する。
- 上昇値は通常どおり受ける（除去しない）。
- ランク低下以外の技本体の効果（ダメージ等）はこのハンドラに含まれないため素通り。
- `かたやぶり` 系で防御側特性が無効化される場合は発動しない（`mold_breaker_ignorable` flag を付与）。
- `かがくへんかガス` 下では発動しない（AbilityHandler が自動的に無効化する）。
- まけんき / かちき は `ON_MODIFY_STAT` フェーズなので競合なし（本特性は `ON_BEFORE_MODIFY_STAT`）。

## 変更対象（想定）

### `src/jpoke/data/ability.py`
- `ミラーアーマー` エントリに `flags=["mold_breaker_ignorable"]` と `ON_BEFORE_MODIFY_STAT` ハンドラを追加する。
  ```python
  "ミラーアーマー": AbilityData(
      flags=["mold_breaker_ignorable"],
      handlers={
          Event.ON_BEFORE_MODIFY_STAT: h.AbilityHandler(
              h.ミラーアーマー_reflect_stat_drop,
              subject_spec="target:self",
          )
      }
  ),
  ```

### `src/jpoke/handlers/ability.py`
- `ミラーアーマー_reflect_stat_drop` を追加する。
  ```python
  def ミラーアーマー_reflect_stat_drop(battle, ctx, value):
      # 反射由来の低下・自己デメリット・相手が存在しない場合は素通り
      if ctx.stat_change_reason == "ミラーアーマー":
          return HandlerReturn(value=value)
      if ctx.source is None or ctx.source is ctx.target:
          return HandlerReturn(value=value)
      # 低下分のみ抽出
      drops = {stat: v for stat, v in value.items() if v < 0}
      if not drops:
          return HandlerReturn(value=value)
      # 低下を反射（source は元の攻撃者のまま保つことでまけんき等が正しく発動）
      battle.modify_stats(ctx.source, drops, source=ctx.source, reason="ミラーアーマー")
      # 自分側から低下分を取り除く（上昇分は残す）
      filtered = {stat: v for stat, v in value.items() if v > 0}
      return HandlerReturn(value=filtered)
  ```

## テスト観点
| ケース | 内容 |
| --- | --- |
| 発動: いかく | いかくポケモンが入場 → ミラーアーマー保持側の攻撃は下がらず、いかく使用側の攻撃が下がる |
| 発動: 追加効果の能力低下 | 相手の攻撃技の追加効果（例: アシッドボム）で下がるはずの特防を反射する |
| 非発動: 自己デメリット | からをやぶる等、自分が下げる効果は反射しない |
| 非発動: かたやぶり | かたやぶり系使用者からの能力低下は反射しない |
| 非発動: かがくへんかガス | かがくへんかガス下では特性が無効化され、反射しない |
| 境界: 相手が最低ランク | 反射先（相手）が既に最低ランクの場合、ランクが変わらない旨が正しく処理される |
| 境界: 上昇と低下が混在 | 低下分のみ反射し、上昇分は自分が受ける |

## 実装前確認事項

### 無限反射の防止
`source=ctx.target`（自分）で呼ぶことで「自己デメリット扱い」にする案は **不正**。  
反射先がまけんき / かちき を持つ場合、本来「相手に下げられた」と判定されるべきだが、
source を自分にするとまけんき / かちき が発動しなくなり仕様違反になる。

#### 正しい対策: `reason` を `BattleContext` に追加して反射フラグを伝播させる
1. `src/jpoke/utils/type_defs.py` に `StatChangeReason` を `HPChangeReason` と同形式で追加する。
   ```python
   StatChangeReason = Literal[
       "",                 # 通常（理由なし）
       "ミラーアーマー",   # ミラーアーマーによる反射
   ]
   ```
2. `BattleContext.__init__` に `stat_change_reason: StatChangeReason = ""` フィールドを追加する。
3. `pokemon_state.py` の `modify_stats` 呼び出し側で `BattleContext(target=target, source=source, stat_change_reason=reason)` として渡す（`reason` は文字列のまま渡し、型は `StatChangeReason` に絞る）。
4. `ミラーアーマー_reflect_stat_drop` の反射呼び出し時に `reason="ミラーアーマー"` を渡す。
5. ハンドラ冒頭で `if ctx.stat_change_reason == "ミラーアーマー": return HandlerReturn(value=value)` としてガードする。

この設計であれば、反射された低下の `source` は元の攻撃者のまま保たれるため、
まけんき / かちき が正しく発動する。
