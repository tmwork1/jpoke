# シンクロ 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/シンクロ.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_FIELD_CHANGE`
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`

## 仕様要点（実装に反映）
- 相手から受けた技で どく ・ まひ ・ やけど 状態にされたとき、その状態にしてきた相手も同じ 状態異常 にする。
- このとくせいを持つポケモンが手持ちの先頭にいる場合、同じ 性格 の 野生ポケモン が出現しやすくなる。シンクロを持つポケモンが ひんし 状態であっても効果がある ( エメラルド ～ 第八世代 まで)。
- 条件が満たされた場合、基本的に50%の確率で同じ性格になる。
- 第三世代 から 第五世代 までの 徘徊系ポケモン には効果がない( トルネロス 、 ボルトロス 、 ライコウ など)。
- エメラルド では、草むら等によるランダムエンカウントでのみ効果があり、 伝説のポケモン (シンボルエンカウント)には効果がない。

## 実装フロー
1. `src/jpoke/data/ability.py` の `シンクロ` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `シンクロ` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
