# なまけ 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/なまけ.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `DomainEvent.ON_MODIFY_MOVE_PRIORITY`
- `Event.ON_SWITCH_IN`
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_CALC_POWER_MODIFIER`

## 仕様要点（実装に反映）
- 2回に1回しか行動することができない。
- この特性に対して シンプルビーム / なかまづくり / なやみのタネ は失敗する。 特性#バトルにおける特性の変化 も参照。
- この特性に対して なりきり / うつしえ / トレース でコピーすることや、うつしえ/ ミイラ で上書きすること、 スキルスワップ / さまようたましい で交換することはできる。 いえき / コアパニッシャー で とくせいなし 状...
- この特性がなかまづくりを使用して相手に特性をコピーすることや、なりきりを使用して自身の特性を上書きすることもできる。
- 溜め技 を使った場合、次のターンになまけるので攻撃できない。 パワフルハーブ で溜めずに使用すれば攻撃できる。

## 実装フロー
1. `src/jpoke/data/ability.py` の `なまけ` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `なまけ` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
