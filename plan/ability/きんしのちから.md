# きんしのちから 実装計画

## スコープ
- 優先度: H
- 仕様書: `spec/ability/きんしのちから.md`
- 実装の核: 変化技ターン限定で「あとだし相当の行動順補正」と「かたやぶり相当の特性無視」を両立する。

## 実装対象イベント
- `DomainEvent.ON_MODIFY_MOVE_PRIORITY`（行動順側）
- `Event.ON_CHECK_DEF_ABILITY`（特性無視側）
- 必要に応じて `Event.ON_CHECK_MOVE`（こうこうのしっぽ/せんせいのツメとの競合調整）

## 実装方針（確定）
1. `src/jpoke/data/ability.py` の `きんしのちから` に、優先度補正用と特性無視用の2系統ハンドラを登録する。
2. 行動順補正は「変化技を選択したターンのみ」有効にする。
3. 特性無視は「実行直前の技分類が変化技なら有効」とし、`ねごと`/`さいはい` 由来の実行技で最終判定する。
4. `かたやぶり_check_def_ability` を再利用しつつ、条件を「きんしのちから有効 + 対象技が変化技」に限定する。
5. `とくせいガード` で無視されない仕様を維持する（既存 `item` ハンドラ優先度に合わせる）。

## 実装メモ
- 行動順は「優先度値を上げ下げする」のではなく、同優先度内の後順位化が必要。
- 既存 `DomainEvent` で表現不能なら、speed 計算に「後行フラグ」を足す拡張を別コミットで追加する。
- かがくへんかガスで無効化されると「後行のみ残る/特性無視だけ残る」を作らないよう、判定タイミングを仕様通りに分離する。

## テストケース
1. 変化技選択ターンで同優先度内の後行になる。
2. 攻撃技選択ターンでは後行にならない。
3. 変化技実行時のみ相手特性を無視する。
4. `ねごと`/`さいはい` 由来の実行技分類で無視判定が切り替わる。
5. `とくせいガード` 持ちには無視が通らない。
6. かがくへんかガス中は特性無視が不発になる。
