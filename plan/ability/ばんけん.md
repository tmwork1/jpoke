# ばんけん 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/ばんけん.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`
- `Event.ON_CHECK_DEF_ABILITY`

## 仕様要点（実装に反映）
- いかく を受けても こうげき が下がらず、逆に 1段階 上がる。
- 交代させる技 や レッドカード の効果による交代を防ぐ。
- いかく を受けたとき、特性バーが表れて攻撃が上がる。
- ビビリだま を持っているときは、攻撃が上がった後にビビリだまの効果が発動する。
- クリアチャーム を持っているときでも、ばんけんの発動が優先されて攻撃が上がる。

## 実装フロー
1. `src/jpoke/data/ability.py` の `ばんけん` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `ばんけん` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
