# ゆうばく 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/ゆうばく.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`
- `Event.ON_CHECK_DEF_ABILITY`

## 仕様要点（実装に反映）
- 直接攻撃 で ひんし になったとき、攻撃してきた相手に対して、相手の最大 HP の1/4のダメージを与える (小数点以下は切り捨て。ただし最低でも1以上)。
- マジックガード の特性を持つポケモンにはダメージを与えられない。 第五世代 以降での特性バーはゆうばくのみ現れる。
- ぼうごパット を持つポケモンはゆうばくのダメージを受けない。このとき、ゆうばくの特性バーが出た直後に「<所持者>は ぼうごパットで 防いだ！」とメッセージが流れる。
- しめりけ の特性を持つポケモンが場に出ているときは発動しない。特性バーも現れない。
- かたやぶり の効果がある直接攻撃技を受けてひんしになった場合も発動する。

## 実装フロー
1. `src/jpoke/data/ability.py` の `ゆうばく` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `ゆうばく` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
