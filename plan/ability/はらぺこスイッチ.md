# はらぺこスイッチ 実装計画

## スコープ
- 優先度: L
- 仕様書: `spec/ability/はらぺこスイッチ.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_CALC_POWER_MODIFIER`
- `Event.ON_CALC_ATK_MODIFIER`
- `Event.ON_CALC_DEF_MODIFIER`

## 仕様要点（実装に反映）
- ターン の終了時に、「まんぷくもよう」と「はらぺこもよう」の2つの フォルム が1ターンおきに入れ替わる。フォルムにより オーラぐるま のタイプが変わる。
- 発動するタイミングは ターン の終わり。あらゆるスリップダメージの発生や 状態変化 ・ 場の状態 の解除が行われた後になる ( ターン#5.ターン終了時の処理 も参照)。
- はらぺこスイッチのポケモン同士では、すばやさの高い方が先にフォルムチェンジする。 トリックルーム 状態が発動している間はその効果を考慮する。
- ダルマモード / リミットシールド / スワームチェンジ / ぎょぐん のポケモンとは、すばやさに関係なくはらぺこスイッチが後に発動する。
- 交代 によりモルペコを繰りだしたとき、そのターンから即座にはらぺこスイッチが発動してはらぺこもようになる。

## 実装フロー
1. `src/jpoke/data/ability.py` の `はらぺこスイッチ` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `はらぺこスイッチ` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
