# よちむ 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/よちむ.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_FIELD_CHANGE`
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`

## 仕様要点（実装に反映）
- 場に出たときに、相手が覚えている 技 を1つ読み取る。威力の高い技が優先される。
- 変化技 など、ダメージが無い技は威力0とみなされる。
- 威力が変動する技 の場合、以下のように扱う。
- 技の説明を開いたとき、威力の欄に数値が表記されている場合、表記通りの威力とみなされる。技の効果で威力が変動する場合でも変わらない (例: しおふき は150、 アシストパワー は20)。
- ステラ タイプの テラバースト も威力80とみなされる

## 実装フロー
1. `src/jpoke/data/ability.py` の `よちむ` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `よちむ` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
