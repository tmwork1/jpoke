# かぜのり 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/かぜのり.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_FIELD_CHANGE`
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`

## 仕様要点（実装に反映）
- 味方の場が おいかぜ 状態になったときや、おいかぜ状態の場に出たときに こうげき が 1段階 上がる。
- 以下の 風の技 を受けても無効化し、こうげきが1段階上がる。
- 相手の場が おいかぜ 状態になっても発動しない。
- コートチェンジ で味方の場においかぜ状態が移動してきても発動しない。
- おいかぜ状態の場に他のポケモンと同時に死に出しされた場合、 いかく など多くの 場に出たときに発動する特性 とは補正込みの素早さが高い順に発動する。

## 実装フロー
1. `src/jpoke/data/ability.py` の `かぜのり` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `かぜのり` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
