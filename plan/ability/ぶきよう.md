# ぶきよう 実装計画

## スコープ
- 優先度: H
- 仕様書: `spec/ability/ぶきよう.md`
- 実装方針: `かがくへんかガス` と同様にチェックイベントを制御点にする。`Event.ON_CHECK_ITEM_ENABLED` を導入し、`ぶきよう` はこのイベントで道具効果だけを無効化する。道具の所持・受け渡し API（`swap_items` / `take_item` / `remove_item`）は既存挙動を維持する。

## 現状整理
- `src/jpoke/data/ability.py` の `ぶきよう` は `AbilityData()` のみで、専用ハンドラ未登録。
- 道具変更可否は `src/jpoke/core/item_manager.py` の `can_change_item()`（`Event.ON_CHECK_ITEM_CHANGE`）に集約されている。
- 道具効果は `src/jpoke/data/item.py` の各イベントハンドラで発動し、入場時に `SwitchManager.register_switch_in_handlers()` で登録される。
- 現状の `item.enabled` 更新は主に `Item.lose()` / `Item.consume()` で行われ、特性状態による道具効果 ON/OFF の再計算はない。
- 特性有効状態は `AbilityManager.refresh_ability_enabled_states()` で再計算され、`かがくへんかガス` の入退場時にも更新される。

## 仕様要点（実装に反映）
- `ぶきよう` が有効な間、所持道具のバトル効果は不発。
- 道具の受け渡し・喪失そのもの（`トリック` / `すりかえ` / `どろぼう` など）は通常どおり可能。
- `かがくへんかガス` 等で `ぶきよう` が無効化されている間は、道具効果が通常どおり発動する。
- `ぶきよう` 復帰時は即座に道具効果が再び止まる（残存ハンドラの取りこぼしを許さない）。

## 未対応・要確認
- `ON_CHECK_ITEM_ENABLED` は新規追加イベントのため、イベント一覧コメントと呼び出し箇所の対応を維持する必要がある。
- `item.enabled` を後から切り替える場合、イベントハンドラの登録/解除を同期しないと効果漏れが起きる。
- `check_on_empty=True` の `remove_item()` は道具未所持でも判定イベントを通す。`ぶきよう` は「道具変更の可否」ではなく「道具効果の可否」なので干渉しない設計に固定する。

## 実装フロー
1. `Event` に `ON_CHECK_ITEM_ENABLED` を追加し、`ItemManager.refresh_item_enabled_states()` から一括判定を発火する。
2. `ItemManager` の基準値は `should_enable = mon.alive and mon.has_item()` とし、追加条件はイベントハンドラ側に寄せる。
3. `ぶきよう` に `Event.ON_CHECK_ITEM_ENABLED` ハンドラを登録し、特性有効時のみ `False` を返す。
4. `item.enabled` が変化した場合だけ、`item.unregister_handlers()` / `item.register_handlers()` を差分適用してイベント登録を同期する。
5. `Battle.refresh_ability_enabled_states()` の直後に `item` 側再計算を呼び、`かがくへんかガス` 入退場・交代時の追従を保証する。
6. `ItemManager.set_item()` でも再計算を呼び、`トリック/すりかえ` 直後の同ターン判定を安定化する。

## テスト観点
- 発動ケース: `ぶきよう` + `いのちのたま` で火力補正と反動がともに発生しない。
- 発動ケース: `ぶきよう` + `オボンのみ` で HP 半分以下でも消費回復しない。
- 非発動ケース: `かがくへんかガス` 登場中は `ぶきよう` 持ちの `オボンのみ` が通常どおり発動する。
- 復帰ケース: `かがくへんかガス` 退場後、同ポケモンの道具効果が再び不発になる。
- 境界ケース: `トリック` / `すりかえ` で道具を受け取った直後も、`ぶきよう` 有効中なら受け取った道具効果が不発。
- 回帰ケース: `ぶきよう` 非所持ポケモンの既存道具挙動（`いのちのたま`、回復きのみ）が変わらない。
