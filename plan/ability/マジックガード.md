# マジックガード 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/マジックガード.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `Event.ON_SWITCH_IN`
- `Event.ON_FIELD_CHANGE`
- `Event.ON_CHECK_HIT_SUBSTITUTE`
- `Event.ON_CHECK_SUBSTITUTE`

## 仕様要点（実装に反映）
- 相手の 攻撃技 以外による ダメージ を無効化する。具体的には以下のダメージを受けない。
- わざ の補助効果によるダメージ
- 反動技 による反動ダメージ : ビックリヘッド / てっていこうせん / クロロブラスト の反動、 とびげり / とびひざげり / かかとおとし / サンダーダイブ の失敗時の反動も受けない。
- 例外として、 わるあがき の反動ダメージは受ける。
- ニードルガード

## 実装フロー
1. `src/jpoke/data/ability.py` の `マジックガード` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `マジックガード` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
