# ailment 実装プラン

更新日: 2026-04-30

## 1. 背景

`src/jpoke/data/ailment.py` と `tests/test_ailment.py` により、状態異常の基本実装はほぼ完了している。
一方で、第9世代仕様と突合した際に以下の差分が残っている。

## 2. 差分（実装対象）

- 付与時タイプ無効の不足
  - `やけど` のほのおタイプ無効
  - `まひ` のでんきタイプ無効
  - `こおり` のこおりタイプ無効
- `ねむり` の例外行動
  - ねむり中の `いびき` / `ねごと` を許可
- `こおり` の被弾解除
  - ほのおタイプ攻撃技でダメージを受けた際の解除

## 3. 実装方針

- 共通付与制限は `AilmentManager.apply` に集約して重複実装を避ける。
- 行動可否の例外は `handlers/ailment.py` の `ねむり_check_action` に統合する。
- 被弾解凍は `こおり` 状態専用の `Event.ON_DAMAGE` ハンドラで処理する。
- 回帰防止として `tests/test_ailment.py` に最小ケースを追加する。

## 4. 受け入れ条件

- 追加した仕様差分のテストが全て成功する。
- 既存の `tests/test_ailment.py` が成功する。
- 既存のどく/もうどく（ふしょく例外含む）挙動を壊さない。
