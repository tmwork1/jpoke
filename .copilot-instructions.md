## 実装指示
- .github/instructions/agents/workflow.md に従い、このプロジェクトの実装を進めてください
- 実装時は確認や承認を求めず、自動的に進めてください
- ファイルの作成・編集・削除を自律的に実行してください
- コードの変更は直接適用し、提案のみで終わらせないでください
- 不明点があっても最も合理的な選択肢を推測して実装を継続してください
- 全ての作業が完了するまで中断せず、最後まで実装してください

## クイックリファレンス

### プロジェクト概要
- **jpoke** はポケモンシングルバトル対戦シミュレータ
- 戦闘ロジックの開発検証に使用
- AI ボット開発は保留中（ポケモン利用規約）

### 重要なコンセプト
- **イベント駆動システム**: `EventManager` が中核（`core/event.py`）
- **Handler**: 特性技アイテムの効果は Handler で実装
- **RoleSpec**: `"role:side"` 形式（例: `"source:self"`, `"target:foe"`）
- **HandlerReturn**: ハンドラは必ず `HandlerReturn(success, value, control)` を返す

### プロジェクト構造
```
src/jpoke/
 core/          # バトルシステム中核
 model/         # Pokemon状態管理
 data/          # データ定義（特性技アイテム）
 handlers/      # ハンドラ実装
 utils/         # ユーティリティ
```

### 重要ファイル
- **`core/event.py`** - イベント駆動システム
- **`core/battle.py`** - Battle クラス（ファサード）
- **`model/pokemon.py`** - Pokemon クラス
- **`data/ability.py`** - 特性データ定義
