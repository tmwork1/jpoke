# ねがいごと 仕様書

## 概要

- カテゴリ: 自陣サイドフィールド（場の状態として管理）
- 継続ターン: 使用した次のターン終了時に発動して解除（合計 2 ターン分カウント）
- 対象: 使用者の自陣にいるポケモン

## 技の基本情報

| 項目 | 値 |
|------|-----|
| タイプ | ノーマル |
| 分類 | 変化 |
| PP | 10 |
| 威力 | なし |
| 命中 | 必中 |
| 優先度 | 0 |

## 効果一覧

| 効果 | イベント |
|------|----------|
| すでに「ねがいごと」状態の場合は失敗する | ON_BEFORE_APPLY_MOVE |
| 自陣に「ねがいごと」フィールドを設置し、回復量を使用者の最大HPの1/2（小数切り捨て）に設定する | ON_STATUS_HIT |
| ターン終了時にカウントを1減らし、0になったらフィールドを解除してHP回復を発動 | ON_TURN_END（priority=50） |
| フィールド解除時、現在の場にいるポケモンのHPを回復する | ON_FIELD_DEACTIVATE |

## 詳細な仕様

### 技の使用時

- すでに自陣がねがいごと状態のときはねがいごとが失敗する
- 成功した場合、自陣に「ねがいごと」フィールドが設置される
- 回復量は **使用者（技を使ったポケモン）の最大HPの1/2**（第五世代以降、小数切り捨て）

### フィールドの発動タイミング

- 第五世代以降: 天候・みらいにこうげきの処理の後、たべのこし等の処理の前（`ON_TURN_END` priority=50）
- 使用したターンを含めて「次のターン終了時」に発動するため、フィールドの初期カウントは 2

### フィールドの発動（HP回復）

- 発動時に「ON_FIELD_DEACTIVATE」イベントを経由して回復処理が行われる
- 回復対象は「そのポケモンが使用した場所」の現在の場にいるポケモン（使用者ではない）
- 使用者が交代またはひんしになった場合でも、フィールドは消滅せず発動は継続する
- **かいふくふうじ**状態のポケモンは回復できない（`ON_MODIFY_HEAL` イベントでブロックされる）
- 第五世代以降: HP が満タンのポケモンは回復されない（`battle.modify_hp` で自動的に処理）

### その他の仕様

- 場の状態であるため、変化技とは見なされない。まもる・みがわり・おうごんのからだがある場合でも回復は有効
- ねがいごと状態の位置はサイドチェンジ・コートチェンジで入れ替わらない（本プロジェクトでは非対象）

## 発動優先度

| イベント | Priority | 根拠 |
|----------|----------|------|
| ON_BEFORE_APPLY_MOVE（失敗チェック） | 100（デフォルト） | ON_APPLY_MOVE「ねがいごと: 前ターン効果残り」Priority 100 相当 |
| ON_STATUS_HIT（フィールド設置） | 100（デフォルト） | 既存の変化技ハンドラに合わせたデフォルト |
| ON_TURN_END（カウント減少・発動） | 50 | docs/spec/turn.md: 「ねがいごと」Priority 50 |
