# アンコール (状態変化) 仕様書

対象: Pokémon Champions シングルバトル

## 概要

アンコールとは、ポケモンの揮発性状態の一つ。
技「アンコール」を使用することで相手に付与される。

## 効果

- アンコール状態のポケモンは、最後にPPを消費した技と同じ技しか使えない。
- コマンド選択画面でアンコールされた技のみ選択可能になる（固定技のPPが0の場合はわるあがき）。
- 行動実行時も固定技に書き換えられる（ON_MODIFY_MOVE により強制）。

## 付与条件（技アンコールの効果）

- 対象のポケモンが場に出てから一度でも技を使っていること（executed_move が存在すること）
- 対象の最後に使った技が non_encore ラベルを持たないこと
  - non_encore ラベルの技: わるあがき、アンコール、ねごと、へんしん、ものまね、スケッチ、オウムがえし、ダイマックスほう
- 対象がすでにアンコール状態でないこと

付与成功時のターン数: 3ターン（Champions仕様）

## 消滅条件

- 使用させている技のPPが無くなる（実質的にわるあがき強制 → コマンド選択で反映される）
- ターン数の経過（3ターン後 → ON_TURN_END priority=110 でカウント終了）
- 交代（揮発状態は交代時に自動解除される）
- メンタルハーブ（Champions 世代では使用済み）

## ターン処理の優先度

`docs/spec/turn.md` の `Event.ON_MODIFY_MOVE` より:

| Priority | 処理 |
|----------|------|
| 20 | アンコール（技の固定） |

`docs/spec/turn.md` の `Event.ON_TURN_END` より:

| Priority | 処理 |
|----------|------|
| 110 | アンコールの終了 |

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE`（技の無効化最終判定。コード上は `Event.ON_BEFORE_APPLY_MOVE`）
（技「アンコール」の失敗条件チェック）より:

| Priority | 処理 |
|----------|------|
| 100 | アンコール: 失敗条件 |

## コマンド制限

- ON_MODIFY_COMMAND_OPTIONS で固定技のコマンドのみを残す。
- 固定技の PP が 0 の場合、コマンドが空になり、上位ロジックでわるあがきになる。
- アンコール状態中でも交代コマンドは選択可能（かなしばり等がなければ）。

## 詳細な仕様（Champions版）

- バトンタッチで引き継がれない。
- かなしばり・かいふくふうじ等と重複した場合、どちらか一方が解除されるまでわるあがきを使用する。
  - コマンド選択レベルで：アンコール制限で固定技のみ残り、かなしばり制限で固定技が除外 → コマンドなし → わるあがき
- 技「アンコール」自体は non_encore ラベルを持つのでアンコールできない。
