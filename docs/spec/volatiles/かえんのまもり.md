# かえんのまもり (状態変化) 仕様書

対象: Pokémon Champions シングルバトル

## 概要

かえんのまもりとは揮発性状態の一つ。
技「かえんのまもり」を使用することで付与される（自分対象）。
そのターンに受ける攻撃技を無効化し、直接攻撃をしてきた相手をやけど状態にする。
第九世代（スカーレット・バイオレット）専用技。

## 技の基本情報

| 項目 | 値 |
|------|-----|
| タイプ | ほのお |
| 分類 | 変化 |
| PP | 10 |
| 優先度 | +4 |
| 範囲 | 自分 |
| 直接攻撃 | なし |
| 急所 | なし |
| 命中判定 | なし |
| 追加効果 | なし |
| まもる貫通 | × |

## 効果

- 使用したターンに受ける**攻撃技**を無効化する（まもると異なり**変化技は防げない**）
- 直接攻撃技を防いだ場合、攻撃者をやけど状態にする
- **そのターンの最後に行動する場合は失敗する**（シングルバトルでは自分が後攻の場合、
  守る対象がなく必ず失敗する）

## 付与条件

- 技「かえんのまもり」を使用したとき（自分に付与、count=1）

## 消滅条件

- ターン終了時に解除される（count=1 のため、使用したターンのみ有効）
- **交代**によって解除される

## 詳細仕様

### 攻撃技の無効化

- そのターンに受ける攻撃技（`is_attack=True` の技）を無効化する
- **変化技は防げない**（`protect_non_attack=False`）
- まもる系と同じ `ON_TRY_MOVE_1` イベントで処理される（priority=100）

### やけどの付与

- 直接攻撃（`is_contact=True`）を防いだ場合のみ攻撃者をやけど状態にする
- 非接触の攻撃技（れいとうビーム等）を防いでもやけどにならない
- タイプ相性・特性でダメージが無効でも直接攻撃を防げばやけど状態にする（ wiki 仕様）
  - **注意**: 本実装では `_run_protect` の `is_contact` チェックにより接触判定を行う
- 連続攻撃技かつ直接攻撃でも、やけど判定は1回だけ行われる（`ailment_manager.apply` が重複付与を防ぐ）
- `ぼうごパット` を持つ攻撃者は接触判定が False になるためやけどにならない
  - `query.is_contact` が `ON_CHECK_CONTACT` を発火し、`ぼうごパット` ハンドラが False を返す

### 防御が無視されるケース

- `ふかしのこぶし` 特性の直接攻撃では守りを貫通し、やけどにもならない
  - `ON_CHECK_PROTECT` ハンドラで `ふかしのこぶし` が False を返す
- `ハイパードリル` / `パワフルエッジ` も守りを貫通（`unprotectable` フラグにより
  `is_blocked_by_protect` が False になり `_check_protect_success` の時点で
  防御そのものが不成立になるため、やけども付与されない）

### 連続使用による失敗率上昇

- まもる/みきり/こらえる/ニードルガード/トーチカ/スレッドトラップ/かえんのまもり を連続使用すると成功率が低下する
  - 成功するごとに次ターンの成功率が 1/3 になる（下限 1/729）
  - 連続使用しなかった場合・失敗した場合はリセットされる
  - **注意**: 本実装は確率的に成功率が逓減する仕様ではなく、「直前に protect
    フラグを持つ技を成功させていた場合は必ず失敗する」という簡略化した実装
    （`まもる系_連続使用失敗チェック`）になっている。まもる等の他の protect
    系技と共通の仕様であり、本実装では `Event.ON_TRY_MOVE_2`（デフォルト
    priority=100）で行われる

### フェイント等による解除

- フェイント/ゴーストダイブ/シャドーダイブ/いじげんホール/いじげんラッシュ を受けた場合、
  かえんのまもり状態は取り除かれてダメージを受ける。これらの技に対しても攻撃者を
  やけど状態にすることはできない
  - タイプ相性により無効だった場合は、かえんのまもり状態は取り除かれない
  - これらによりかえんのまもり状態が取り除かれた場合、次ターンのかえんのまもりは
    連続使用と見なされず必ず成功する
  - **注意**: 本実装ではフェイント/いじげんホール/いじげんラッシュは `_FAINT_REMOVE_VOLATILES`
    経由でかえんのまもり等の volatile を明示的に解除する。一方ゴーストダイブ/
    シャドーダイブは `unprotectable` フラグのみで守りを無視しており、volatile
    自体は明示的に解除されない（シングルバトルでは同ターン中の他への影響はないが、
    「次ターンを連続使用と見なさない」という仕様までは再現できていない簡略実装）

### そのターンの最後に行動する場合の失敗

- かえんのまもりはそのターンに自分より後に行動する相手の技から身を守るための
  技であるため、自分がそのターンの最後に行動する場合は守る対象がなく失敗する
  （ポケモンWiki「技の仕様」節に記載されている、かえんのまもり固有の失敗条件）
- 本実装では `battle.query.is_second_actor()`（シングルバトル想定）を用いて
  `Event.ON_TRY_MOVE_2` で判定する（`かえんのまもり_check`）
- 失敗時は executed_move を None にリセットするため、次ターンの連続使用判定には
  影響しない

## ターン処理の優先度

`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` より:

| Priority | 処理 |
|----------|------|
| 30 | まもる系: 連続使用により失敗（本実装では `ON_TRY_MOVE_2` で行う） |
| 100 | かえんのまもり状態による攻撃技の無効化 |

`docs/spec/turn.md` に記載のない本実装独自のハンドラ:

| Event | Priority | 処理 |
|-------|----------|------|
| `ON_TRY_MOVE_2` | 100（デフォルト） | まもる系: 連続使用により失敗（`まもる系_連続使用失敗チェック`） |
| `ON_TRY_MOVE_2` | 100（デフォルト） | かえんのまもり: そのターン最後の行動による失敗（`かえんのまもり_check`） |

`Event.ON_TURN_END`:
- priority 未指定（デフォルト）で source:self に対して解除する

## 関連インタラクション

- **ぼうごパット**: 攻撃者のis_contactをFalseにするため、やけどにならない
- **ふかしのこぶし**: `ON_CHECK_PROTECT` でFalseを返し、守りを貫通する（やけども付与されない）
- **ハイパードリル / パワフルエッジ**: `unprotectable` フラグにより守りを無視する（やけども付与されない）
- **フェイント / いじげんホール / いじげんラッシュ**: かえんのまもり状態を解除してダメージを与える
- **ゴーストダイブ / シャドーダイブ**: `unprotectable` フラグにより守りを無視するが、volatile 自体は明示的には解除されない
- **まもる / ニードルガード / トーチカ / キングシールド / スレッドトラップ**: 連続使用成功率共有（かえんのまもりとの連続使用で失敗率上昇）
- **ワイドガード / ファストガード**: 連続使用成功率共有に含まれるが、かえんのまもりの失敗率に影響しない（ワイドガード・ファストガード側の連続使用カウントのみ変動）
