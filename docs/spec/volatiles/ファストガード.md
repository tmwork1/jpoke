# ファストガード (状態変化) 仕様書

対象: Pokémon Champions シングルバトル

## 概要

ファストガードとは揮発性状態の一つ。
技「ファストガード」を使用することで付与される（自分対象）。
そのターンに受ける、実効優先度+1以上の技（先制技）を無効化する。

## 技の基本情報

| 項目 | 値 |
|------|-----|
| タイプ | かくとう |
| 分類 | 変化 |
| PP | 16（champions基準。docs/champions/move_list.txt 744行目参照。Gen9本家は15） |
| 優先度 | +3 |
| 範囲 | 自分 |
| 直接攻撃 | なし |
| 急所 | なし |
| 命中判定 | なし |
| 追加効果 | なし |
| まもる貫通 | × |

技本体（PP・連続使用失敗等）の詳細は `docs/spec/moves/ファストガード.md` を参照。

## 効果

- 使用したターンに受ける、**実効優先度+1以上の技**を無効化する（攻撃技・変化技いずれも対象）
- 実効優先度0以下の技は防げない（まもるとの最大の違い）
- **接触時の追加効果はなし**（キングシールド・ニードルガード・トーチカ等と異なる）

## 付与条件

- 技「ファストガード」を使用したとき（自分に付与、count=1）

## 消滅条件

- ターン終了時に解除される（count=1 のため使用したターンのみ有効）
- **交代**によって解除される
- フェイント / ゴーストダイブ / シャドーダイブ / いじげんホール / いじげんラッシュ のダメージを
  受けた場合も取り除かれる（`_FAINT_REMOVE_VOLATILES` に `"ファストガード"` を含む）

## 詳細仕様

### 優先度判定

- `Event.ON_TRY_MOVE_1`（`defender:self`、priority=100）で、これから着弾する技の**実効優先度**を
  `battle.speed_calculator.calc_move_priority(ctx.attacker, ctx.move)` で算出し、1以上であれば
  `_run_protect` に処理を委譲して無効化する
- 技データの静的な `move.priority`（例: つぶらなひとみは+1で固定）だけでなく、`いたずらごころ`
  （変化技の優先度+1）・`はやてのつばさ`（ひこうタイプかつHP満タン時に攻撃技の優先度+1）・
  `ヒーリングシフト` 等の特性による**動的な優先度変化**も考慮する必要があるため、`ctx.move.priority`
  を直接参照せず `speed_calculator.calc_move_priority()` を経由する
  （`じょおうのいげん_block_priority`・`はやてがえし_can_apply` と同じパターン）
- ファストガード自身の優先度+3より早く出される技（優先度+4以上、または同優先度でより素早い場合）は
  ファストガードの volatile が付与される前に着弾しているため、このハンドラに到達せず防げない
  （ターン順序自体で制御されるため、ハンドラ側で個別対応する必要はない）

### 変化技の無効化

- `protect_non_attack=True`（デフォルト）のため、実効優先度+1以上であれば変化技も防ぐ
  （つぶらなひとみ、いたずらごころの効果による技など）

### 防御対象

- `_check_protect_success` により `ctx.move.target == "foe"` かつ `unprotectable` フラグを
  持たない技のみが対象になる（`is_blocked_by_protect`）
- `フェイント`は優先度+2の先制攻撃技だが `unprotectable` フラグを持つため防げない
- 特性`ふかしのこぶし`の直接攻撃技は `ON_CHECK_PROTECT` で防御判定自体が無効化されるため防げない
  （ファストガード状態自体は取り除かれない）

### フェイントによる解除

- フェイント・ゴーストダイブ・シャドーダイブ・いじげんホール・いじげんラッシュ を受けた場合、
  ファストガード状態が取り除かれてダメージを受ける
- 攻撃がタイプ相性で無効の場合は取り除かれない

### 連続使用による失敗率上昇

- まもる / みきり / こらえる / ニードルガード / トーチカ / スレッドトラップ / かえんのまもり /
  キングシールド / ファストガード を連続使用すると成功率が低下する
  - 本来の仕様: 成功するごとに次ターンの成功率が 1/3 になる（下限 1/729）
  - 連続使用しなかった場合・失敗した場合・交代を挟んだ場合はリセットされる
  - **実装済み**: `まもる系_連続使用失敗チェック`（`handlers/move_status.py`、
    `Event.ON_TRY_MOVE_2` デフォルト priority）として実装済み。ただし確率モデルではなく
    「直前ターンに `protect` フラグを持つ技を成功させていた場合、今ターンは必ず失敗する」
    という簡略化モデルを採用している（確率的な1/3ずつの低下は未実装・スコープ外。
    詳細は `docs/spec/moves/まもる.md` 参照）。

## ターン処理の優先度

`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` より:

| Priority | 処理 |
|----------|------|
| 30 | まもる系: 連続使用により失敗（実装では `Event.ON_TRY_MOVE_2` デフォルト priority。`docs/spec/moves/まもる.md` 参照） |
| 100 | ファストガード状態による、実効優先度+1以上の技の無効化 |

`Event.ON_TURN_END`:
- priority 未指定（デフォルト）で source:self に対して解除する

## 関連インタラクション

- **ふかしのこぶし**: `ON_CHECK_PROTECT` でFalseを返し、守りを貫通する（ファストガード状態は
  取り除かれない）
- **フェイント / ゴーストダイブ / シャドーダイブ / いじげんホール / いじげんラッシュ**: ファストガード
  状態を取り除く
- **いたずらごころ / はやてのつばさ / ヒーリングシフト**: 動的に優先度を上げる特性。
  `calc_move_priority` で算出した実効優先度がファストガードの判定対象になる
- **まもる / みきり / キングシールド / ニードルガード / トーチカ / スレッドトラップ /
  かえんのまもり**: 連続使用成功率共有
