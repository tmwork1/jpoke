# アクアリング (状態変化) 仕様書

対象: Pokémon Champions シングルバトル

## 概要

アクアリングとは揮発性状態の一つ。
アクアリング（技）を使用することで付与される。

## 効果

| 効果 | 値 |
|------|-----|
| ターン終了時回復 | 最大HPの **1/16**（小数点以下切り捨て、最小1） |

- ターン終了時（Priority 70）に最大HPの1/16を回復する。
- 小数点以下は切り捨て。ただし回復量の最小は1。

## 付与条件

- アクアリング（技）を使用したとき

## 消滅条件

- **交代**によって解除される（バトンタッチは除く）

## 詳細仕様

- バトンタッチで引き継がれる。
- かいふくふうじ状態では回復できない。かいふくふうじ状態が解除されるターンでもアクアリングの回復はできない。
- おおきなねっこを持っているポケモンは回復量が1.3倍になる。
  - 本来の回復量の端数処理を行った後におおきなねっこの効果を計算する。
  - バトンタッチで引き継がれたときは、技の使用者ではなくバトン先のポケモンが持っているおおきなねっこにより回復量が増える。
- ターン経過では解除されない。

## ターン処理の優先度

`docs/spec/turn.md` の `Event.ON_TURN_END` より:

| Priority | 処理 |
|----------|------|
| 70 | アクアリング |

## 関連インタラクション

- **かいふくふうじ**: アクアリングの回復を無効化する
- **おおきなねっこ**: 回復量を1.3倍にする
- **ヘドロえき**: 相手の回復を無効化する（`ON_MODIFY_HEAL` イベント経由）

## 実装上の注意

- 回復量計算: `max_hp // 16`（整数切り捨て）、結果が0の場合は1とする
- `modify_hp(source, r=1/16, source=source)` で呼び出すと `v = int(max_hp * 1/16)` となり、
  max_hp < 16 の場合に v=0 になるリスクがある。現実のポケモンではmax_hpが十分大きいため問題にならないが、
  厳密には `v = max(1, max_hp // 16)` を使うべき。
- かいふくふうじの回復ブロックは `ON_MODIFY_HEAL` イベント経由で自動的に機能する。
