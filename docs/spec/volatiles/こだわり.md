# 仕様書: こだわり（揮発状態）

調査日: 2026-06-20
参照URL:
- https://wiki.pokemonwiki.com/wiki/こだわりスカーフ
- https://wiki.pokemonwiki.com/wiki/こだわりハチマキ
- https://wiki.pokemonwiki.com/wiki/こだわりメガネ

## 概要

こだわり状態とは、こだわり系アイテム（こだわりスカーフ・こだわりハチマキ・こだわりメガネ）を持ったポケモンが最初に技を使用した後に付与される揮発状態。
最初に選択した技しか使用できなくなる。

## 付与条件

- こだわり系アイテムを持ったポケモンが技を使用したとき（ON_MOVE_END）に付与される
- 既にこだわり状態の場合は重複付与しない（idempotent）

付与時に `move_name` として使用した技名を記録する。

## 効果

- ON_MODIFY_COMMAND_OPTIONS において、固定技（move_name と一致する技）コマンドのみを選択可能にする
- 交代コマンドは引き続き選択可能
- 固定技の PP が 0 の場合、コマンドが空になり、上位ロジックでわるあがきになる
- かなしばり等と重複した場合、固定技が除外されコマンドなし → わるあがき

## 消滅条件

- 交代（揮発状態は交代時に自動解除される）
- アイテムが消滅・移動した場合でも、こだわり状態自体は継続する
  （アイテムを失った後も、こだわり状態が残っている間は技固定が維持される）

## こだわり系アイテムの固有効果（volatile.py の外）

| アイテム | イベント | 効果 |
|---------|---------|------|
| こだわりスカーフ | DomainEvent.ON_CALC_SPEED | 素早さを 1.5 倍（×6144/4096） |
| こだわりハチマキ | Event.ON_CALC_ATK_MODIFIER | 物理技の攻撃補正を 1.5 倍（×6144/4096） |
| こだわりメガネ | Event.ON_CALC_ATK_MODIFIER | 特殊技の攻撃補正を 1.5 倍（×6144/4096） |

これらは `handlers/item.py` に実装し、`data/item.py` で登録する。

## ターン処理の優先度

`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` より:

| Priority | 処理 |
|----------|------|
| 10 | こだわり系アイテム・ごりむちゅうで技固定 |

（これはコマンド制限ではなく、技使用時のこだわり固定チェックに関する記述。
 実装では ON_MODIFY_COMMAND_OPTIONS でコマンド制限、ON_MOVE_END でロック付与を行う。）

## コマンド制限

- ON_MODIFY_COMMAND_OPTIONS で固定技コマンドのみを残す
- 固定技の PP が 0 の場合、コマンドが空になり、上位ロジックでわるあがきになる
- こだわり状態中でも交代コマンドは選択可能
