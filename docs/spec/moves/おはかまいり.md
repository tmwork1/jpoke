# おはかまいり（Last Respects）仕様書

参照: docs/wiki/moves/おはかまいり.html（ポケモンWiki）

## 基本情報

* タイプ: ゴースト
* 分類: 物理
* 威力: 50（可変。効果参照）
* 命中: 100
* PP: 10
* 対象: 相手1体
* 優先度: 0
* 接触: なし（直接攻撃 ×）

## 効果

* その戦闘で味方がひんしになるたび、威力が50ずつ上がる（基本威力50 + 50 × ひんし味方数）。
  * 例: ひんし0体 → 威力50、ひんし1体 → 威力100、ひんし2体 → 威力150。
* 戦闘が開始する前からひんしになっていた味方はカウントしない。
* ひんしになるわざ（いのちがけ等）などを使い、自らひんしになった味方もカウントする。
* ひんしになった味方が戦闘中に復活しても威力は減らない。復活させたその味方が再びひんしになることでも威力が上がる（累計カウント）。
  * 1回の戦闘でひんしと復活を繰り返し、味方が累計100回以上ひんしになることで最大威力は5050になる（実機の仕様、要出典情報がWikiに記載）。
  * 使用者自身がひんしになった回数もカウントする。
* マルチバトル・テラレイドバトルの味方のひんしはカウントしない（本プロジェクトはシングルバトル専用のため対象外の注記）。

## 本プロジェクトでの実装方針

* 本プロジェクト（ポケモンチャンピオンズ シングルバトル）には、バトル中に手持ちポケモンを復活させる
  手段（きのみ・アイテム・技によるひんしからの復活）が存在しない。そのため「ひんしになった味方の
  累計カウント」と「現在ひんし状態の控えポケモン数」は常に一致する。
  → 控えポケモン（bench）のうち `fainted` なものの数をそのまま威力計算に用いてよい。
* 戦闘開始前からひんしのポケモンを手持ちに含める運用は本プロジェクトでは想定しないため、
  「開始前からひんしだった味方を除外する」ための特別な処理は不要。

## 判定フラグ

* 急所: あり（通常判定）
* 命中判定: あり（命中率100）
* 追加効果: なし
* まもる: 防がれる
* おうじゃのしるし: 発動する（第五世代以降）
* マジックコート: 反射不可（変化技ではないため対象外）
