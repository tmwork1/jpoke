# サイドチェンジ（Ally Switch）仕様書

参照: docs/wiki/moves/サイドチェンジ.html（ポケモンWiki）
参照: docs/champions/move_list.txt L306-307

## 基本情報

* タイプ: エスパー
* 分類: 変化
* 威力: -
* 命中: -（必中、命中判定なし。自分が対象の技のため命中判定自体が行われない）
* PP: 16（`docs/champions/move_list.txt` が正。ポケモンWiki記載の第五世代基準値は15だが、
  championsでは16に調整されている）
* 対象: 自分（自分と味方1体の位置を入れ替える。ダブル/トリプルバトル専有の効果）
* 優先度: +2（第七世代以降。第六世代以前は+1）
* 接触: なし（直接攻撃×）

## 効果（本来のダブル/トリプルバトルでの効果）

* 自分と味方1体の位置を入れ替える。ダブルバトル用。
* 「自分が対象の技」なので命中判定は行われずに成功する。
  * 味方がまもる/みがわり状態であるとき、トリックガードを使用したとき、姿を隠しているとき
    （溜め技中）でも成功する。
  * 味方がダイマックスしているとき、ダイウォールを使用したときでも成功する。
* 入れ替わる先の場所にポケモンがいない場合や、すでにひんしになっている場合は失敗する。
* シングルバトル/ローテーションバトル/マルチバトル/バトルロイヤル/マックスレイドバトル/
  テラレイドバトルで使用すると失敗する。
* 判定: 命中判定 × / 追加効果 × / まもる × / おうじゃのしるし（第五世代以降）× /
  マジックコート × / よこどり × / オウムがえし ×
* 第九世代仕様（連続使用の成功率低下）:
  * 技が成功した後、次に使うサイドチェンジの成功率は約1/3ずつ低下していく。
  * サイドチェンジが失敗した直後に、もう一度サイドチェンジを使った場合は必ず成功する。
  * 他の味方がサイドチェンジを使ったターンに使用したときは成功する。
  * サイドチェンジが成功してからPPを消費した行動を取らなかった場合も、その直後に使う
    サイドチェンジは失敗する可能性がある。
  * まもる系の技の成功率とは区別され、これらの技とサイドチェンジを交互に使用するときなら
    どちらも必ず成功する（＝まもる系専用の連続失敗カウンタとは別のカウンタで判定される）。
  * さいはいによるサイドチェンジは、直接サイドチェンジを使ったときと同一視される。
* トリプルバトルでは端にいるポケモンがもう片端のポケモンと入れ替わる（使用者が中央にいる場合は
  失敗する。中央にポケモンが残っていても中央と端で入れ替わることはできない）。
* サイドチェンジにより、自分以外の1体が対象の技/味方1体が対象の技の対象位置に技の使用者自身が
  移動していた場合、その技は失敗する（例: いやしのはどう）。
* ねらいうち、および特性スクリューおびれ/すじがねいりのポケモンが使用した技は、サイドチェンジの
  影響を無視して技の対象に選択したポケモンを狙って使用される。
* Zワザによる付加効果: 自分のすばやさが2段階上がる。

## シングルバトルでの扱い（本プロジェクトのスコープ）

* 本プロジェクトの対象はポケモンチャンピオンズのシングルバトルのみであり、`Battle` は常に
  各プレイヤー1体ずつの場（`battle.actives` は常に2体以下）しか扱わない。
* ポケモンWikiに明記の通り、シングルバトルで使用すると失敗する（入れ替え先の味方が存在しないため）。
* 自分と味方の位置交換、および連続使用による成功率低下はダブル/トリプルバトル専有の効果であり、
  本プロジェクトの対象範囲外。類似技「てだすけ」「コーチング」「いかりのこな」等、ダブル専有技と
  同じ扱いとし、`docs/progress/move.md` で本技を全列「ダブル専用」（対象外）として扱う。

## 実装方針

* 通常のダメージ計算を伴わない変化技として、追加効果なし（`handlers={}`）で実装する。
* 位置交換処理・連続使用成功率低下カウンタはダブル専有機能のため未実装のままとする。
* `target="self"` を明示する（後述の注意点参照）。
