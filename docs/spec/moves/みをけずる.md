## 効果

自分の最大HPの1/2（小数点以下切り捨て）を消費して、自分の『こうげき』『とくこう』『すばやさ』
ランクを2段階ずつ上げる。

## 失敗条件

- 現在のHPが最大HPの半分以下の場合は失敗し、HPは減らない
  （`現在HP <= 最大HP // 2` が失敗条件。境界値である「ちょうど半分」のときも失敗する）。
- 『こうげき』『とくこう』『すばやさ』のランクがすでにすべて最大（+6）まで上がっている場合、
  ランクは変化せず、HPも消費されない（技自体が失敗扱いになる）。
  一部のランクのみ最大（+6）で他が変化可能な場合は、変化可能なランクだけが上昇し、
  HPは通常通り消費される。

## あまのじゃく（Contrary）との相互作用

- 使用者の特性が**あまのじゃく**である場合、ランク変化の効果が反転し、
  『こうげき』『とくこう』『すばやさ』ランクが2段階ずつ下がる。
  このときも、いずれかのランクが実際に変化すればHPは通常通り消費される。
- あまのじゃくであっても、対象の3ランクがすべて最低（-6）まで下がっているときに使用すると、
  ランク変化が発生せず技は失敗し、HPは消費されない
  （`battle.modify_stats` の返り値が空になる場合を汎用的に判定するため、
  あまのじゃく特性の有無にかかわらず同一のロジックで扱える）。

## 技の仕様（一次情報より）

- HPが最大HPの1/2以下のときに使用すると失敗する。
- すでに『こうげき』『とくこう』『すばやさ』が最大まで上がっている場合、HPは減らない。
- この技でHPを消費した後に受けるダメおしの威力は2倍にならない
  （`battle.modify_hp` は反動・確定反動以外の理由では `hits_taken` を増加させないため、
  実装上は追加対応不要で自然に満たされる）。
- ゆびをふるで選ばれない（対象: ゆびをふる自体が未実装のため、現時点では対応不要。
  ゆびをふるを実装する際に除外リストへ追加すること）。

## 判定（一次情報より）

| 項目 | 値 |
|------|-----|
| 急所 | × |
| 命中判定 | × |
| 追加効果 | × |
| まもる | ×（自分を対象とする技のため防がれない） |
| おうじゃのしるし | × |
| マジックコート | ×（自分を対象とする技のため跳ね返されない） |

`まもる`・`マジックコート` はいずれも `move.target == "foe"` を前提に判定されるため、
実装上は `MoveData.target="self"` を明示する必要がある。
