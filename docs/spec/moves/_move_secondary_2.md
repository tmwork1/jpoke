## 端数処理

「反動」比率を damage に乗算した際の端数は、第五世代以降 `round_half_down`（五捨五超入。
きっちり0.5の場合のみ切り捨て、それ以外は四捨五入と同じ）で丸めるのが正式仕様
（参照: docs/wiki/moves/すてみタックル.html「技の仕様」節）。

一方「HP吸収」比率の端数処理は「反動」とは**別の丸め方**であることが判明している。
`docs/wiki/moves/ウッドホーン.html`（ギガドレイン・きゅうけつ等の同種ページも同文言）の
「技の仕様」節には「回復量を算出する際、小数点以下は第四世代までは切り捨て、
第五世代以降は四捨五入する」と明記されており、これは `round_half_down` ではなく
標準的な四捨五入（`round_half_up`、ちょうど0.5は切り上げ）である。
反動量・HP吸収量はいずれも最低1で下限が保証される（`max(1, ...)`）。

* 反動（`handlers/move_attack.py` の `_recoil`）: 2026-07 の すてみタックル レビューで
  `int()` 切り捨てから `round_half_down` に修正済み。反動比率が1/2の場合は端数が必ず0か0.5であり
  `int()` 切り捨てと `round_half_down` の結果は一致するため実質的な影響はないが、1/3・1/4比率では
  端数が0.5を超える場合に反動量が1不足するバグがあったため修正した。
* HP吸収（`_drain_hp`）: 2026-07 の ウッドホーン レビューで `int()` 切り捨てから
  `max(1, round_half_up(...))` に修正済み。吸収比率0.5の技（ウッドホーン・きゅうけつ等）は
  ダメージが奇数のとき端数が0.5となり、四捨五入では切り上げられるため、修正前の `int()` 切り捨て
  （常に切り捨て）とは回復量が1異なるケースがあった（例: 与ダメ101→切り捨て50、四捨五入51）。
  比率0.75の技（デスウイング・ドレインキッス）についても同様に `round_half_up` を適用するのが
  一次情報に沿った実装であるが、個別のWikiページでの文言確認は各技の個別レビューで再確認すること。
* ドレインキッスについては個別レビューで `docs/wiki/moves/ドレインキッス.html`「技の仕様」節を確認済み。
  「回復量を算出する際、小数点以下は四捨五入する。」と明記されており、`round_half_up` の実装は正しい
  （例: 与ダメ101→ round_half_up(75.75)=76、`int()` 切り捨てなら75になりバグとなる）。
