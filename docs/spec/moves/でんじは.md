## 効果

相手を **まひ** 状態にする。

## 失敗条件

でんじはは変化技だが、例外的にタイプ相性の影響を受ける（ポケモンWiki記載）。

1. `ailment_manager.apply(defender, "まひ")` が内部で行う汎用チェックにより、以下の場合は失敗する
   - 対象がすでに **状態異常**（どく・もうどく・まひ・やけど・ねむり・こおり）を持っている
   - 対象が **でんきタイプ** である（まひ無効。第六世代以降の仕様）
2. でんじは固有の失敗条件として、対象が **じめんタイプ** である場合は無効になる
   （でんき技はじめんタイプに対して効果がないため）。ただし対象が **ねらいのまと**
   を持っている場合はタイプ無効が解除され、有効になる。

## 防がれる条件

以下の場合、でんじはは防がれる（`ailment_manager.apply` 内、`ON_BEFORE_APPLY_AILMENT` 経由の
汎用判定、および `ON_BEFORE_APPLY_MOVE` 経由の特性判定）：

- じゅうなん・ぜったいねむり・きよめのしお特性（まひ無効）
- フラワーベール特性・しんぴのまもり状態・ミストフィールド状態
- みがわり状態（使用者にすりぬけがある場合は無視）
- ちくでん特性（第三世代のみ有効、第四世代以降は無効）
- でんきエンジン特性（でんき技を無効化し、すばやさが1段階上がる）
- ひらいしん特性（でんき技を自分に引き寄せ、とくこうが1段階上がる）

## イベントフロー

```
ON_BEFORE_APPLY_MOVE (priority=130, MoveHandler)
  → じめんタイプ無効チェック（ねらいのまと所持時は無効化解除）
     - calc_def_type_modifier(ctx) == 0 なら MOVE_IMMUNED を記録して失敗

ON_STATUS_HIT (priority=100, MoveHandler デフォルト)
  → ailment_manager.apply(defender, "まひ")
     - すでに状態異常 OR でんきタイプなら失敗（apply内部の汎用チェック）
     - ON_BEFORE_APPLY_AILMENT を発火し、じゅうなん等の特性・
       しんぴのまもり等の場の状態・みがわりによる無効化を判定
```

## ポケモンチャンピオンズ向けの簡略化

- Z変化技付加効果（とくぼう+1段階）は実装しない
- コンテスト効果は実装しない
- おうじゃのしるし・こんらんのくぎ等の追加効果判定は対象外
  （でんじはに追加効果〈secondary_effect〉は存在しない）
