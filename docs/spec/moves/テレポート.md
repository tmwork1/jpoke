# テレポート（Teleport）仕様書

参照: docs/wiki/moves/テレポート.html（ポケモンWiki）

## 基本情報

* タイプ: エスパー
* 分類: 変化
* 威力: -
* 命中: なし（必ず自分に成功する）
* PP: 20（`src/jpoke/data/ps-champ-ja/moves.json` にエントリなし。全世代共通で PP20 かつ既存の
  4段階PP制（8/12/16/20）にも適合するため、原作値を維持する）
* 対象: 自分
* 優先度: -6（ピカブイ・第八世代以降。本プロジェクトの対象範囲である第九世代準拠）
* 接触: なし

## 効果

第七世代（USUM）までは「野生ポケモン戦のみ使用可能で戦闘を終了させる技」であり、
トレーナー戦（＝本プロジェクトが対象とするシングルバトル）では効果が無かった。
しかし **ピカブイ・第八世代以降は仕様が変更**され、トレーナー戦でも以下のように機能する。

* トレーナーのポケモンが使用すると、**優先度-6** で控えのポケモンと交代する
  （とんぼがえり等とは異なり、攻撃は行わず技そのものが交代技として機能する）。
* 交代できる控えのポケモン（生存しているポケモン）がいない場合は技が失敗する。
* 使用者が以下の状態・特性の影響を受けていても、**これらをすべて無視して確実に交代できる**
  （トレーナー戦限定の仕様。野生ポケモン戦ではこれらの効果でテレポートが失敗しうるが、
  本プロジェクトの対象外）。
  * バインド状態
  * にげられない状態
  * フェアリーロック状態（場の状態）
  * 特性かげふみ・ありじごく・じりょくの効果を受けている場合
* 第八世代（BDSP除く）以降、野生ポケモンがテレポートを使用した場合は戦闘が終了するが、
  本プロジェクトはトレーナー戦（シングルバトル）のみが対象のため無関係。

## 判定フラグ

* 急所: なし（変化技）
* 命中判定: なし（自分対象のため必ず命中）
* 追加効果: なし
* まもる: 防がれない（対象が自分のため）
* マジックコート・マジックミラー: 反射されない（対象が自分のため）
* おうじゃのしるし・よこどり・オウムがえし: 対象外（変化技だが追加効果を持たないため影響なし）

## バトンタッチとの違い

* 能力ランク変化や揮発性状態を交代先のポケモンに引き継がせる効果はない
  （`docs/spec/moves/バトンタッチ.md` 参照）。
* 優先度が -6 である点がバトンタッチ（優先度 0）と異なる。

## 実装上の注意

* `MoveData` には `target="self"` を必ず指定する
  （欠落すると `Move.is_blocked_by_protect` / `Move.is_reflectable` が
  `target == "foe"` を基準に判定するため、まもる・マジックコートに誤反応する）。
* `priority=-6` を設定する。
* とらわれ状態（にげられない・バインド・フェアリーロック・かげふみ・ありじごく・じりょく）を
  無視して交代できる点は `バトンタッチ` と同様の仕様のため、`バトンタッチ_check` /
  `バトンタッチ_apply`（`handlers/move_status.py`）と同じ実装パターン
  （`get_available_switch_commands` のとらわれ状態チェックを経由せず、
  控えの生存ポケモンの有無のみで判定する）を踏襲する。
  ただしテレポートはランク・揮発性状態の引き継ぎを行わないため、その部分の実装は不要。
