## 効果

使用者自身の HP を、最大HPの 1/2 分回復する。

HP が満タンの場合は失敗する。

## かいふくふうじとの相互作用

- 使用者が **かいふくふうじ** 状態の場合、`heal` フラグを持つ技として選択できない
  （`かいふくふうじ_try_action` が汎用的に処理するため、本技専用の実装は不要）。

## 特性 ヒーリングシフト との相互作用

`heal` フラグを持つ回復技のため、使用者が `ヒーリングシフト` を持つ場合は優先度が+3される
（`docs/spec/abilities/ヒーリングシフト.md` 参照。特性側は `heal` フラグを汎用的に参照するため、
本技専用の実装は不要）。

## 判定（一次情報より）

| 項目 | 値 |
|------|-----|
| 急所 | × |
| 命中判定 | × |
| 追加効果 | × |
| まもる | ×（自分を対象とする技のため防がれない） |
| おうじゃのしるし | × |
| マジックコート | ×（自分を対象とする技のため跳ね返されない） |
| よこどり | ○ |
| オウムがえし | × |

`まもる`・`マジックコート` はいずれも `move.target == "foe"` を前提に判定されるため、
実装上は `MoveData.target="self"` を明示する必要がある（未設定だとデフォルト値 `"foe"` になり、
誤って防がれる/跳ね返される）。

## 備考

- 一次情報の「技の仕様」節には「回復量の計算において、第四世代まで小数点以下は切り捨てる。
  第五世代以降は切り上げる。」と明記されている。ただし本実装は同種の自己回復技
  （`じこさいせい`・`はねやすめ`・`ミルクのみ` 等）との実装一貫性を優先し、
  `battle.modify_hp(mon, r=1/2)`（端数切り捨て）を採用している。最大HPが奇数の場合、
  理論値と1HPの差が生じ得る既知の簡略化。この差異は自己回復技ファミリー全体で共通するため、
  将来的に一括で見直すことが望ましい。
