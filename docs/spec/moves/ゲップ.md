# ゲップ（Belch）仕様書

参照: docs/wiki/moves/ゲップ.html（ポケモンWiki）

## 基本情報

* タイプ: どく
* 分類: 特殊
* 威力: 120
* 命中: 90
* PP: 12（`docs/champions/move_list.txt` / `docs/progress/move.md` 準拠。本家シリーズのPPは10だが、Championsでは12）
* 対象: 相手1体
* 優先度: 0
* 直接攻撃: なし
* フラグ:
  * `non_negoto`（ねごとで選ばれない）
  * `non_copycat`（まねっこで選ばれない）

## 効果

* その戦闘で「きのみ」を食べたことがあるポケモンでなければ使用できない（追加効果なし。ダメージのみ）。
* 本家シリーズ（SVまで）: 使用条件を満たしていない状態でゲップを選ぼうとすると、選択そのものがキャンセルされる
  （「〈ポケモン名〉は きのみを 食べていないので 技が だせない!」）。ゲップ以外に選べる技が無い場合は
  わるあがきになる。
* **Pokémon Champions**: ゲップを選択すること自体は可能。ターン開始時点で使用条件（きのみを食べたことがある）を
  満たしていなくても、自分の行動が実行されるまでにきのみを食べることができれば成功する。
  逆に、行動時点で条件を満たしていなければ技は失敗する
  （「戦闘中にきのみを食べていない場合 この技は失敗する。」）。
  * 本プロジェクトの対象はChampionsシングルバトルのみのため、「選択自体を禁止する」本家仕様は実装せず、
    実行時（`Event.ON_TRY_MOVE_1`）に条件判定を行うだけでよい。これはプレイヤー・AIが行動選択時に
    ゲップを選べること、かつ選択後に条件を満たさなければ技が失敗する、というChampionsの仕様と一致する。

## 使用条件（「きのみを食べた」の判定）

* 条件が成立する（`ate_berry` フラグが立つ）ケース:
  * 持たせたきのみを通常通り消費して食べたとき。
  * なげつけるで相手から投げつけられたきのみを食べたとき（食べるアニメが発生する場合のみ）。
  * むしくい・ついばむで相手のきのみを奪って食べたとき。
  * ほおばる・おちゃかいの効果で自分のきのみを食べたとき。
  * なげつけるで消費したきのみを次ターンはんすうで食べたとき。
  * きのみを食べた後にリサイクル・しゅうかく・ものひろいなどできのみが復活しても、既に成立した条件は消えない。
  * きのみを食べた後にひんし・交代・げんきのかけら等での復活があっても、条件は戦闘中ずっと保持される。
* 条件が成立しないケース:
  * なげつける・ギフトパスで自分のきのみを手放したとき（食べていない）。
  * しぜんのめぐみできのみを消費したとき。
  * むしくい・ついばむを受けて自分のきのみを食べられたとき（食べたのは相手）。
  * どろぼう・ほしがる・トリック・すりかえ、特性マジシャン・わるいてぐせ・きょうせいなどできのみが
    他のポケモンに渡ったとき。
  * やきつくす・はたきおとす・ふしょくガスの効果できのみを失ったとき。
* 条件は戦闘（バトル）単位でのみ有効。バトンタッチで後続に引き継がれない。
  へんしんしても、自身が食べていない限り成立しない（コピー元が食べていても無関係）。

## 判定フラグ

* 急所: あり（通常判定）
* 命中判定: あり（命中率90）
* 追加効果: なし
* まもる: 防がれる
* おうじゃのしるし: 第五世代以降は対象になる
* マジックコート: 対象外（変化技ではないため無関係）
* よこどり: 対象外
* オウムがえし: 対象外（コピー不可）
* 音のわざ: 該当しない

## 選択・コピー不可の詳細

* ねごと・ねこのて・まねっこ・ゆびをふる・さいはいでは選ばれない。
* おうむがえし・さきどりでもコピーできない。
* 第八世代まではものまねでコピー可能だったが、第九世代以降はコピー不可。
* 本プロジェクトでは、まねっこ（Copycat）が「直前に使用された技が `non_copycat` フラグを持つ場合は失敗する」
  という実装（`まねっこ_can_use`、`src/jpoke/handlers/move_status.py`）になっているため、ゲップに
  `non_copycat` フラグを付与することでこの仕様を満たす。
  ねこのて・ゆびをふる・さいはい・おうむがえし・さきどり・ものまねは本プロジェクトでは未実装
  （`handlers={}` の実装保留、またはそもそも技データが存在しない）のため、これらとの相互作用は対応不要。

## イベント / 優先度

* `docs/spec/turn.md` の `## Event.ON_TRY_MOVE_1` に「30 | ほおばる: きのみを持っていない」と同種の
  失敗条件が並んでおり、ゲップの「きのみを食べていない場合に失敗」もこれに準ずるため priority=30、
  subject_spec="attacker:self" とする。

## リーサル計算

* 対象外（`docs/progress/move.md` のリーサル実装列は n/a）。追加効果のない通常ダメージ技であり、
  失敗条件は「きのみを食べたか」という状態依存でリーサル計算の対象にする性質のものではない。
