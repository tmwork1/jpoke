# かなしばり 仕様書

## 基本情報

| 項目 | 値 |
|------|-----|
| タイプ | ノーマル |
| 分類 | 変化 |
| PP | 20 |
| 命中率 | 100（第五世代以降。第三世代まで55、第四世代80） |
| 範囲 | 相手1体 |
| 優先度 | 0 |
| 直接攻撃 | × |
| まもる | 防がれる |
| マジックコート | 跳ね返される（第五世代以降。第四世代までは跳ね返されない） |

## 効果

相手を **かなしばり** 状態（揮発性状態）にする。
かなしばり状態のポケモンは、付与時点で相手が最後にPPを消費した技
（`ctx.defender.last_pp_consumed_move`）を、4ターンの間コマンド選択できなくなる
（第五世代以降。世代によりターン数は異なるが、本プロジェクトは第五世代以降の
仕様のみ扱う）。
ねごとで実行されたサブ技はPPを消費しないため対象にならず、ねごと自身が封じられる。

## 失敗条件

- 対象がそのバトルで一度もPPを消費する行動をしていない場合
  （`last_pp_consumed_move` が `None`）は失敗する。
- 対象が直前に使用した技が「わるあがき」の場合は失敗する。
- 対象がすでにかなしばり状態の場合は失敗する
  （`volatile_manager.apply` が既存の揮発性状態を検知して `False` を返す）。

揮発性状態「かなしばり」自体の詳細仕様（コマンド制限・解除条件など）は
`docs/spec/volatiles/かなしばり.md` を参照。

## 判定

- 急所: なし（変化技）
- 命中判定: あり（100%、回避率等の補正を受ける）
- 追加効果: なし（本体効果が100%発動するため「追加効果」扱いではない）
- まもる: 防がれる（`Move.is_blocked_by_protect` は対象が `foe` の変化技を自動的に対象とするため、専用フラグは不要）
- マジックコート: 跳ね返される（`マジックコート_reflect` が「対象が相手・変化技」であるすべての技を汎用的に跳ね返すため、専用対応は不要）
- よこどり: 対象外（よこどりは自分または場を対象とする変化技のみを対象とするため、相手を対象とする本技は元々対象外）

## イベントフロー

```
ON_BEFORE_APPLY_MOVE (priority=100, MoveHandler デフォルト)
  → かなしばり_can_apply
    → move = ctx.defender.last_pp_consumed_move
    → move が None または move.name == "わるあがき" の場合、失敗ログを出して stop_event=True

ON_STATUS_HIT (priority=100, MoveHandler デフォルト)
  → かなしばり_apply
    → move = ctx.defender.last_pp_consumed_move
    → volatile_manager.apply(defender, "かなしばり", count=4, move_name=move.name)
      → 既にかなしばり状態の場合は自動的に失敗する
```

## 本実装での対象外（スコープ外）

- のろわれボディ特性によるかなしばり付与: 特性実装時に対応済み（`tests/abilities/test_ability_na.py`）。
- メンタルハーブによる解除: アイテム実装時に対応済み（`tests/items/test_item_ma.py`）。
- アロマベール特性による予防: 特性実装時に対応済み（`tests/abilities/test_ability__group.py`）。
- ダイマックスわざへの変換による回避: Champions ではダイマックスは対象外。
