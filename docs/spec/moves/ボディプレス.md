# ボディプレス（Body Press）仕様書

参照: docs/wiki/moves/ボディプレス.html（ポケモンWiki）
参照: docs/spec/moves/イカサマ.md（実数値の参照元入れ替えという同種の技）
参照: docs/spec/fields/ワンダールーム.md（実数値参照入れ替えとの相互作用）

## 基本情報

* タイプ: かくとう
* 分類: 物理
* 威力: 80
* 命中: 100
* PP: 12（チャンピオンズ準拠。src/jpoke/data/ps-champ-ja/moves.json）
* 対象: 相手1体
* 優先度: 0
* 接触: あり（直接攻撃○）

## 効果

自分の『こうげき』とランク補正の代わりに、**自分自身の『ぼうぎょ』実数値とぼうぎょランク補正**を
用いてダメージ計算を行う（技の分類は物理のまま）。

## 技の仕様（重要な相互作用）

* 使用者の特性・持ち物などは、通常の物理技と同様に「攻撃に影響する効果」のみがダメージに影響し、
  「防御に影響する効果」は影響しない。
  * はりきり・こだわりハチマキなど、物理技の攻撃補正（`ON_CALC_ATK_MODIFIER`）を上げる効果は
    通常通り適用され、ダメージが増える。
  * スロースタートなど、物理技の攻撃補正を下げる効果も通常通り適用され、ダメージが減る。
  * ファーコート・しんかのきせきなど、**防御補正**（`ON_CALC_DEF_MODIFIER`、防御側として発動時）
    を上げる効果は、ボディプレスの使用者が自分のダメージを計算する際には参照されないため、
    ダメージは増えない。
  * こだいかっせい/クォークチャージで自分の「こうげき」が強化対象になっている場合、防御の値を
    1.3倍にしてダメージを計算する。「ぼうぎょ」が強化対象になっていてもダメージは増えない
    （パラドックス特性の補正は技が参照する実数値の種類に関わらず、常に技の分類（物理/特殊）に
    対応する攻撃側スロットでのみ判定されるため）。
  * わざわいのおふだ等、相手の特性により自分の「こうげき」補正が下がっている状態で使用すると、
    防御の値を3/4倍にしてダメージを計算する。わざわいのつるぎ等により自分の「ぼうぎょ」補正が
    下がっていても、ボディプレスのダメージは減らない
    （わざわいのつるぎは防御側の計算（`ON_CALC_DEF_MODIFIER`）にのみ作用し、ボディプレスが
    参照する攻撃側の計算には影響しないため）。
  * ゆき状態でこおりタイプが受ける防御アップ（防御側の天候ボーナス）は、ボディプレスの使用者
    自身の計算には影響しない。
* 物理技のため、やけど状態だとダメージが半減する（他の物理技と同様、通常の
  `ON_CALC_ATK_MODIFIER` フローで処理される）。
* パワートリック状態（自分の『こうげき』と『ぼうぎょ』実数値が入れ替わっている）で使用すると、
  入れ替え後の実数値（＝元の『こうげき』実数値）と『ぼうぎょ』ランク補正を参照する
  （実数値のみが入れ替わり、ランク補正は入れ替わらないため）。
* 特性てんねんを持つ相手に使用した場合、使用者自身の『ぼうぎょ』ランクは無視される
  （てんねんは`ON_CALC_ATK_RANK_MODIFIER`にフックしており、技が参照する実数値の種類に
  関わらず動作する）。
* ワンダールーム状態で使用すると、実数値は通常通り『ぼうぎょ』を参照するが、ランク補正は
  『とくぼう』のものを参照する（本家の既知の仕様）。
  * **現行実装では対象外**（`docs/spec/fields/ワンダールーム.md` の「範囲外」に記載の通り、
    ワンダールームと実数値参照を入れ替える技・特性との複雑な相互作用は現行実装のスコープ外）。
* ちいさくなる状態の相手に使用しても、威力2倍や必中の効果はない
  （ボディプレスの技フラグに `minimize` を含まないため）。
* テラレイドバトルにおけるがっちりぼうぎょ等の相互作用は、シングルバトル以外の要素のため
  本プロジェクトの対象外（README.md「対象範囲」参照）。

## 判定フラグ

* 急所: 通常判定（急所ランク0）
* 命中判定: あり（命中率100）
* 追加効果: なし
* まもる: 防がれる
* おうじゃのしるし: 発動する（第五世代以降、通常のダメージ技と同様の一般ロジックで処理）
* マジックコート: 反射不可（変化技ではないため対象外）

## 実装方針

* `src/jpoke/core/damage.py` の `_calc_final_attack` 内で `move.name == 'ボディプレス'` を判定し、
  `attacker.stats["def"]` と `attacker.rank_modifier("def")` を最終攻撃力計算に使う特別分岐が
  すでに存在する。参照元が『ぼうぎょ』に入れ替わるだけで、その後の `ON_CALC_ATK_RANK_MODIFIER` /
  `ON_CALC_ATK_MODIFIER`（アイテム・特性・状態異常の補正）は通常の物理技と同じフローで発火する
  ため、追加のハンドラは不要。
* パラドックス特性（`src/jpoke/handlers/ability_paradox.py`）は、参照する実数値の出どころに
  関わらず「使用者自身の攻撃側スロット」（物理技なら『こうげき』）のみを見て判定するため、
  技名による分岐は不要（イカサマとの相互作用レビューで既に技名分岐は撤廃済み）。

## リーサル計算

* `docs/progress/move.md` の「リーサル実装」「リーサルテスト」列は `n/a`。対象外
  （攻撃力の参照元入れ替えはダメージ計算内で完結し、専用のリーサルハンドラを要しない）。
