# みずびたし 仕様書

## 基本情報

| 項目 | 値 |
|------|-----|
| タイプ | みず |
| 分類 | 変化 |
| PP | 20 |
| 命中率 | 100 |
| 範囲 | 相手1体 |
| 優先度 | 0 |
| 直接攻撃 | × |

## 効果

相手のタイプを **みず単タイプ** に変える（複合タイプであっても単タイプに置き換える）。

## 失敗条件（ON_BEFORE_APPLY_MOVE）

以下の場合、みずびたしは失敗する：

1. 対象がすでにみず単タイプである
   - もりののろい・ハロウィンなどでタイプが追加されており見かけ上「みず」を含む複合タイプになっている場合は、
     対象は「みず単タイプ」ではないため成功し、みず単タイプに置き換えられる（追加タイプは消える）
2. 対象がテラスタルしている
3. 対象がアルセウスまたはシルヴァディである（種族固有の無効化。持ち物の有無・特性の状態に関わらず無効。
   へんしん・かわりものでこれらに変身したポケモンには有効）

## 技自体がみずタイプであることによる無効化（コア共通ルール）

このハンドラでは判定しない。技タイプが「みず」であることにより、`core/move_executor.py` の
タイプ相性・タイプ無効化の共通ルールにより以下の場合に自動的に無効化される：

- 対象が特性 **ちょすい** / **かんそうはだ** / **よびみず** を持つ
  （みず技自体を無効化する特性のため、相手をみず単タイプに変更できない）
- そうでん状態や特性ノーマルスキンなどにより技のタイプが変更された場合も、成功すれば相手をみず単タイプに
  変化させる。このときはみず技を無効化する特性に対しては有効になるが、変更後のタイプを無効化する特性に
  対しては失敗するようになる（技タイプ判定に連動するため、みずびたし側での個別実装は不要）

## 防がれる条件

- みがわり状態（状態変化技はみがわりに防がれる共通ルール `みがわり_immune` が適用される。
  第六世代以降は特性すりぬけで無視すれば有効）
- マジックコートで反射される（`unreflectable` フラグを付与しないため既定で反射可能）
- まもるで防がれる（`unprotectable` フラグを付与しないため既定で防御可能）
- オウムがえしでコピーされる

## タイプ変化の実装方式

専用の揮発性状態「みずびたし」を付与し、`Pokemon.volatile_override_type` を使って現在のタイプを
みず単タイプに上書きする。

- `ON_VOLATILE_START`: `volatile_override_type = "みず"` に設定し、`added_types` をクリアする
  （もりののろい・ハロウィンで追加されていたタイプを消す）
- `ON_VOLATILE_END`: `volatile_override_type = None` に戻す

状態異常ではなく揮発性状態のため、交代すると元のタイプに戻る（揮発性状態がリセットされるため。
バトンタッチでも引き継がれない）。

## イベントフロー

```
ON_BEFORE_APPLY_MOVE (priority=100, MoveHandler デフォルト)
  → みずびたし_can_apply: 既にみず単タイプ／テラスタル中／アルセウス・シルヴァディなら失敗 (stop_event=True)

ON_STATUS_HIT (priority=100, MoveHandler デフォルト)
  → みずびたし_apply: volatile_manager.apply(defender, "みずびたし")
```

揮発性状態「みずびたし」のタイプ変更処理は `data/volatile.py` に実装：

```
ON_VOLATILE_START → みずびたし_set_type
ON_VOLATILE_END   → みずびたし_clear_type
```

## ポケモンチャンピオンズ向けの簡略化

- フォルムチェンジ・メガシンカなどで姿が変わるとタイプが本来のものに戻るという wiki の記述は、
  フォルムチェンジ自体の実装（各フォルムチェンジ技・特性側）で `data.types` を参照し直すため、
  みずびたし側での特別対応は不要
- ダイマックス／キョダイマックスは対象範囲外のため考慮しない
