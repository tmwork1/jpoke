# もりののろい 仕様書

## 基本情報

| 項目 | 値 |
|------|-----|
| タイプ | くさ |
| 分類 | 変化 |
| PP | 20 |
| 命中率 | 100 |
| 範囲 | 相手1体 |
| 優先度 | 0 |
| 直接攻撃 | × |

ボクレー・オーロットの専用技。

## 効果

相手の「タイプ」に **くさタイプを追加** する（既存のタイプに上書きせず、複合タイプに追加する）。
すでにタイプを2つ持っているポケモンに使用した場合も上書きされず、一時的に3タイプになる。

## 失敗条件（ON_BEFORE_APPLY_MOVE）

以下の場合、もりののろいは失敗する：

1. 対象がすでにくさタイプを持っている（複合タイプの一部として持っている場合を含む）
2. 対象がテラスタルしている

## 技自体がくさタイプであることによる無効化（コア共通ルール）

このハンドラでは判定しない。技タイプが「くさ」であることにより、`core/move_executor.py` の
タイプ相性・タイプ無効化の共通ルールにより以下の場合に自動的に無効化される：

- 対象が特性 **そうしょく** を持つ（くさ技自体を無効化する特性のため、相手にくさタイプを追加できない）

## 防がれる条件

- みがわり状態（状態変化技はみがわりに防がれる共通ルール `みがわり_immune` が適用される。
  第六世代以降は特性すりぬけで無視すれば有効）
- マジックコートで反射される（`unreflectable` フラグを付与しないため既定で反射可能）
- まもるで防がれる（`unprotectable` フラグを付与しないため既定で防御可能）
- オウムがえしでコピーされる

## タイプ追加の実装方式

専用の揮発性状態「もりののろい」を付与し、`Pokemon.added_types` にくさタイプを追加する
（`Pokemon.types` プロパティが `added_types` を考慮した現在のタイプを返す）。

- `ON_VOLATILE_START`: `added_types` にくさタイプを追加する
- `ON_VOLATILE_END`: `added_types` からくさタイプを除去する

状態異常ではなく揮発性状態のため、交代すると元のタイプに戻る（揮発性状態がリセットされるため。
バトンタッチでも引き継がれない）。

## イベントフロー

```
ON_BEFORE_APPLY_MOVE (priority=100, MoveHandler デフォルト)
  → もりののろい_can_apply: 既にくさタイプ／テラスタル中なら失敗 (stop_event=True)

ON_STATUS_HIT (priority=100, MoveHandler デフォルト)
  → もりののろい_apply: volatile_manager.apply(defender, "もりののろい")
```

揮発性状態「もりののろい」のタイプ追加処理は `data/volatile.py` に実装：

```
ON_VOLATILE_START → もりののろい_add_type
ON_VOLATILE_END   → もりののろい_remove_type
```

## 技の仕様（一次情報より）

- もりののろい状態のポケモンが「ハロウィン」の効果を受けると、もりののろい状態は消える。
  ハロウィン状態のポケモンがもりののろいの効果を受けると、ハロウィン状態は消える
  （どちらも `added_types` を用いて実装されており、それぞれの volatile が独立して付与・解除されるため、
  実装上は特別な相互作用処理は不要。両方の volatile が同時に付与されていても `added_types` に
  「くさ」「ゴースト」が共存するだけであり、公式の「一方を受けるともう一方が消える」という仕様とは
  異なる可能性がある点に注意。ポケモンチャンピオンズでは実戦上の影響が乏しいため簡略化して問題ない）
- 交代するか、ひんしになるか、戦闘が終了するとタイプは元に戻る（揮発性状態のリセットで実現）
- フォルムチェンジ・メガシンカ・きずなへんげ・ウルトラバースト・テラスタルによりもりののろい状態は消える
  （テラスタルは失敗条件で対応。フォルムチェンジ等は各フォルムチェンジ実装側で `data.types` を
  参照し直すため、もりののろい側での特別対応は不要）
- もりののろい状態はバトンタッチで引き継がれない（揮発性状態のため自動的に対応済み）
- 特性へんげんじざい・リベロ・へんしょくが発動すると、もりののろい状態は消える
  （各特性実装側で対応する事項であり、もりののろい側での特別対応は不要）
- 特性ぎたいが発動しても、もりののろい状態は消えない（何もしないことで実現済み）

## ポケモンチャンピオンズ向けの簡略化

- ダイマックス／キョダイマックスは対象範囲外のため考慮しない
- BDSPでは使用不可能な技だが、対象範囲外（ポケモンチャンピオンズシングルバトル）のため考慮しない
