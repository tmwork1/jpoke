## 効果

使用者自身の **こうげき（A）・ぼうぎょ（B）の実数値** を入れ替える。

- ランク補正（`rank["atk"]` / `rank["def"]`）は一切変化しない。実数値のみを交換する。
- 実数値は種族値・個体値・努力値・性格補正・ランク補正等をすべて含んだ最終的な計算値であり、
  この技はその「攻撃実数値」と「防御実数値」そのものを入れ替える。
- 相手のこうげき・ぼうぎょには一切影響しない（対象は自分のみ）。
- 同じポケモンがもう一度パワートリックを使用すると、入れ替えた実数値が元に戻る
  （2回使用で元の状態に戻る）。
- 場を離れる（交代する）と、実数値は種族値ベースで再計算され元の状態に戻る
  （`Pokemon.reset_on_switch_out()` が `update_stats()` を呼ぶため）。

## 判定

- 急所: なし（変化技）
- 命中判定: なし（必中技）
- 追加効果: なし
- まもる: 防がれない。`target="self"` のため `Move.is_blocked_by_protect`
  （`target == "foe"` が条件）が自動的に False になる
- マジックコート・マジックミラー: 跳ね返されない。`target="self"` のため
  `Move.is_reflectable`（`target in ("foe", "foe_side")` が条件）が自動的に False になる
- みがわり: 対象が自分自身のため、みがわりの有無に関係なく発動する

## イベントフロー

```
ON_STATUS_HIT (priority=100, MoveHandler デフォルト)
  → パワートリック_swap_stats
    → attacker._stats_manager.stats[1] ⇔ attacker._stats_manager.stats[2]
      （インデックス1=こうげき、インデックス2=ぼうぎょ）
```

`battle.modify_stat()` はランク変化量（デルタ）を加算するAPIのため、
「実数値そのものを入れ替える」目的には使用できない。`Pokemon._stats_manager.stats`
（実数値配列）の該当インデックスを直接入れ替える方式を採る
（`スピードスワップ`・`ガードシェア` と同一パターン）。

直接代入のため `ON_MODIFY_STAT` イベントは発火しない。ランク変化トリガー系の特性
（かちき・まけんき等）は発動しない。これはゲーム仕様上も正しい挙動である
（パワートリックは「ランク変化」ではなく「実数値交換」の技のため）。
