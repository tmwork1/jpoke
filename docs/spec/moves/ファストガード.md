# ファストガード（Quick Guard）仕様書

参照: docs/wiki/moves/ファストガード.html（ポケモンWiki）、
docs/wiki/volatiles/まもる_(状態変化).html（ポケモンWiki）、
src/jpoke/data/ps-champ-ja/moves.json

## 基本情報

* タイプ: かくとう
* 分類: 変化
* 威力: なし
* 命中: なし（必中）
* PP: 16（champions基準。src/jpoke/data/ps-champ-ja/moves.json 744行目参照。Gen9本家は15）
* 対象: 自分（味方の場。ダブル・トリプルバトルでは味方全体が対象になるが、
  本プロジェクトはシングルバトル専用のため自分のみが対象になる）
* 優先度: +3
* 接触: なし

## 効果

* 使用したターンの間、自分に「ファストガード」揮発性状態を付与する。そのターンに受ける、
  **実効優先度が+1以上の技（先制技）を無効化する**。まもると異なり、優先度0以下の通常技は
  防げない。付与条件・防御対象・優先度判定の詳細は docs/spec/volatiles/ファストガード.md 参照。
* 連続で使う度に成功率が1/3になっていく（最初は必ず成功、以降は1/3, 1/9, ... と下がり、下限は1/3^6=1/729）。
  失敗するとカウントはリセットされる。
  * 本プロジェクトでは、まもる系技共通の連続使用失敗判定（`まもる系_連続使用失敗チェック`）を採用し、
    「直前ターンに守る系技（`protect` フラグを持つ技）を成功させていた場合、今ターンは必ず失敗する」
    という簡略化モデルで実装する（確率的に成功率が1/3ずつ低下していく詳細モデルは
    未実装・スコープ外とする。詳細は docs/spec/volatiles/まもる.md 参照）。
  * まもる・みきり・こらえる・キングシールド・ニードルガード・トーチカ・スレッドトラップ・
    かえんのまもり等、`protect` フラグを持つ技同士であれば異なる技を挟んでも連続使用扱いになる。

## 判定フラグ

* 急所: なし（攻撃技ではないため対象外）
* 命中判定: なし（必中）
* 追加効果: なし
* まもる: 対象外（自分を対象とする変化技のため通常は問題にならない）
* おうじゃのしるし等: 発動しない
* マジックコート・マジックミラー: 反射不可（自分対象技のため対象外）
* よこどり: 対象外（自分対象技のため通常はよこどりされない）
* オウムがえし: 対象外（コピー不可）
* まねっこ: コピー不可（`protect` フラグを持つ守る系技は一律コピー対象から除外される。
  `まねっこ_can_use` が `protect` フラグを判定するため、個別に `non_copycat` フラグを
  付与する必要はない）

## 優先度が高い技のブロック条件

一次情報「技の仕様」節より：

* 優先度は+3で、この技より早く出される技（優先度+4以上、または同優先度でより素早い場合）は
  防ぐことができない（ファストガード自身が実行される前に既に着弾しているため）。
* 実効優先度が+1以上の技であれば、変化技・攻撃技を問わず防ぐことができる
  （つぶらなひとみ、いたずらごころの効果による技など）。
  * `つぶらなひとみ`のように技データ自体の優先度が+1以上の技だけでなく、
    `いたずらごころ`・`はやてのつばさ`・`ヒーリングシフト` 等の特性で**動的に優先度が
    上がった**技も防ぐ対象になる。実装上は技データの静的な `priority` ではなく
    `battle.speed_calculator.calc_move_priority()` で実効優先度を算出して判定する
    （`じょおうのいげん`・`はやてがえし` と同じパターン。詳細は
    docs/spec/volatiles/ファストガード.md 参照）。
* `フェイント`は先制攻撃技（優先度+2）だが防ぐことができない
  （`unprotectable` フラグにより `is_blocked_by_protect` が False になる）。
* 特性`ふかしのこぶし`の直接攻撃技は優先度が高くても防ぐことができない。ただし
  ふかしのこぶしで無視されてもファストガード状態自体は取り除かれない。

## Zワザ・ダイマックスわざによる貫通（対象外）

一次情報によれば、優先度が高いZワザ/ダイマックスわざを受けた場合、防げる変化技なら防ぐが、
攻撃技ならダメージを1/4に抑えて受ける、という記述があるが、本プロジェクトの対象範囲
（README.md「対象範囲」、champions基準）ではZワザ・ダイマックスは存在しないため対象外。

## 連続使用による失敗

`まもる` 系統技（`protect` フラグを持つ技）と連続使用成功率を共有する。詳細な失敗条件・実装状況は
`docs/spec/volatiles/まもる.md`「連続使用による失敗率上昇」節を参照
（本実装では簡略化した「直前ターンに `protect` フラグ技を成功させていた場合は必ず失敗」という
判定になっている。確率的な 1/3 逓減は未実装・スコープ外として保留）。

## Zワザによる付加効果（対象外）

一次情報によれば、Zワザ使用時は自分のぼうぎょが1段階上がる付加効果があるが、本プロジェクトの
対象範囲ではZワザは対象外。

## ポケモンチャンピオンズ向けの簡略化

- ダブルバトル・トリプルバトル固有の挙動（味方全体への保護効果）は対象外
- Zワザ・ダイマックスわざによる付加効果・貫通は対象外
- 第五世代のみの仕様（変化技は優先度が高くても防げない、確率的な連続使用失敗判定）は対象外
  （championsは第九世代基準のため第六世代以降の仕様を採用する）
