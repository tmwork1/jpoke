# クロスサンダー（Fusion Bolt）仕様書

参照: docs/wiki/moves/クロスサンダー.html（ポケモンWiki）

## 基本情報

* タイプ: でんき
* 分類: 物理
* 威力: 100
* 命中: 100
* PP: 8（ポケモンチャンピオンズでは全技のPPが8/12/16/20の4段階に統一されている。
  本技は `src/jpoke/data/ps-champ-ja/moves.json` に未収録のため、本家基準PP（5）が同じ
  ガリョウテンセイ（`src/jpoke/data/ps-champ-ja/moves.json` 未収録・チャンピオンズPP=8として
  既に実装済み）・インファイト（同PP=8）・だいもんじ（同PP=8）と同様に
  「本家PP5→チャンピオンズPP8」の変換規則を適用した値を採用する）
* 対象: 相手1体
* 優先度: 0
* 接触: なし（直接攻撃ではない）

## 効果

* 同じターン中に、直前に成功（命中）した技がクロスフレイムだった場合、威力が2倍になる。
  この現象は俗に「デュアルクロス」と呼ばれる（コンビネーションわざの一種）。
* 本プロジェクトはシングルバトルのみが対象のため、「直前に成功した技」は事実上
  「同じターン中に相手（または味方が入れ替わる状況はないため常に相手）がこのターン
  クロスフレイムを命中させていたかどうか」と等価になる。

### デュアルクロスの発動条件（一次情報より）

* 先発技（クロスフレイム）が命中しなかった場合は発動しない。
* 先発技がタイプ相性で無効化された場合は発動しない。
* 先発技がまもる等でブロックされた場合は発動しない。
* 先発技がちくでん・もらいび等の特性で吸収・無効化された場合は発動しない。
* 先発技の戦闘アニメが再生されずに失敗した場合は発動しない。
* 2つの技の間に別の技が成功した場合は発動しない（本プロジェクトのシングル環境では
  1ターンにつき最大2アクションしかないため、この条件は実質的に常に満たされる）。

## 判定フラグ

* 急所: あり
* 命中判定: あり（命中率100）
* 追加効果: なし
* まもる: 防がれる
* おうじゃのしるし: 発動する（第五世代以降）
* マジックコート: 反射不可（変化技ではないため対象外）
* よこどり: 対象外（変化技ではないため）
* オウムがえし: 反射可能

## 実装メモ

* 「直前に成功した技」の判定は、技実行中に汎用的に更新される
  `battle.last_used_move_name`（技の成否のみを見て命中・被ブロックを考慮しない）を
  そのまま使うと、クロスサンダー自身の発動時点で既に自分自身の名前で上書きされて
  しまうため使用できない（発動条件が「命中」まで要求するため、より厳密な判定が必要）。
* そのため、クロスサンダー・クロスフレイムそれぞれの `Event.ON_HIT` ハンドラで
  専用のターン番号フィールド（`battle.fusion_bolt_used_turn` /
  `battle.fusion_flare_used_turn`）に命中したターンを記録し、`Event.ON_CALC_POWER_MODIFIER`
  側で「相手技のフィールドが現在のターン番号と一致するか」を見て倍化する。
  `ON_HIT` はまもる・タイプ相性・特性吸収・命中失敗のいずれでも発火しないため、
  一次情報の発動条件（命中して初めて成立）を自然に満たす。
* この実装はクロスサンダー・クロスフレイムの双方が対になって初めて機能するため、
  本レビュー（クロスサンダー担当）の範囲でクロスフレイム側にも対になる
  `Event.ON_HIT` ハンドラ（命中ターンの記録のみ）を追加している。
  クロスフレイム自体の仕様書・計画書・進捗行の作成は別レビューの範囲とする
  （`docs/progress/move.md` のクロスフレイム行は本レビューでは変更しない）。
