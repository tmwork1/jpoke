# むしくい（Bug Bite）仕様書

参照: docs/wiki/moves/むしくい.html（ポケモンWiki）
docs/champions/move_list.txt:894（むし / 物理 / 60 / 100 / 20 / 接触あり / まもる有効）

## 基本情報

* タイプ: むし
* 分類: 物理
* 威力: 60
* 命中: 100
* PP: 20
* 対象: 相手1体
* 優先度: 0
* 接触: あり（直接攻撃）
* フラグ: `contact`（追加効果に確率発動要素はない）

## 効果

* 相手が「バトルに効果のある」きのみを持っていれば、命中後にそのきのみを奪い、
  自分（攻撃者）がそのきのみの効果を受ける（消費する）。
* ついばむ（ひこうタイプ・同威力・同命中・同PP）と全く同じ追加効果を持つ、姉妹技。

## 判定フラグ

* 急所: あり（通常判定）
* 命中判定: あり（命中率100）
* 追加効果: なし（確率発動要素はない。きのみを持っていれば必ず発動し、追加効果扱いではないため
  りんぷん・おんみつマント・ちからずくの影響を受けない）
* まもる: 防がれる
* おうじゃのしるし: 対象外（追加効果がないため無関係）
* マジックコート: 反射不可（変化技ではないため対象外）
* オウムがえし: コピー可能

## きのみ奪取の詳細

* 発動タイミングはダメージ命中後（`Event.ON_HIT`）。
* 相手がきのみを持っていない場合、または「バトルに効果のある」きのみでない場合は不発。
* 相手がみがわりに防がれた場合はきのみを食べられない（実体への攻撃ではないため）。
* タンガのみ等、ダメージ計算前に自動発動して消費されるきのみは、命中後の奪取処理が発火する前に
  既に消費されているため、むしくいによる奪取は不発になる
  （例: むしくいはむしタイプなので、むしタイプが弱点のポケモンが持つタンガのみは
  むしくいより先に消費され、奪えない）。
* チイラのみ等のピンチきのみ、オボンのみ等の回復きのみ、アッキのみは、相手自身の消費条件
  （HP閾値等）が満たされる前にむしくいが奪って食べることができる
  （※本実装では、通常攻撃と同様にダメージ適用と同時にHP閾値ベースのきのみが処理されるため、
  この優先関係は未実装。詳細は計画書の「既知の制限」を参照）。
* 対象がひんしになったときでも効果は発動する。
* 対象の特性ぶきよう・さしおさえ（さしおさえは本プロジェクト未実装）状態であっても、
  奪取自体は発動する（対象側の道具無効化は「使う」ことを妨げるだけで「奪われる」ことは妨げない）。
* マジックルーム状態でも効果は発動する。
* 使用者や対象がきんちょうかんの効果を受けていても、きのみを食べてその効果を受けられる。
* 使用者が特性ぶきようであっても、奪ったきのみを食べて効果を得られる
  （新しく得たアイテムは別インスタンスとして扱われ、既存の無効化状態を引き継がないため、
  本実装は追加対応なしでこの仕様を満たす）。
* 使用者の持ち物いのちのたまによる反動ダメージや、かいがらのすずによる回復は
  むしくいの効果より後に発生する（`Event.ON_HIT` の優先度差で自然に満たされる）。

### ねんちゃく（Sticky Hold）との関係

* 相手が特性ねんちゃくを持つ場合、通常は奪取自体が失敗し追加効果は発生しない。
* ただし、この技で対象をひんしにさせた場合は、ねんちゃくによる阻止を無視してきのみを
  奪うことができる（第五世代以降の仕様）。ねんちゃく以外のアイテム変更禁止効果
  （マルチタイプ・ARシステム等）には影響しない。
