# みねうち（False Swipe）仕様書

参照: docs/wiki/moves/みねうち.html（ポケモンWiki）

## 基本情報

* タイプ: ノーマル
* 分類: 物理
* 威力: 40
* 命中: 100
* PP: 40
* 対象: 相手1体
* 優先度: 0
* 接触: あり

## 効果

* 相手の残りHPがなくなるようなダメージを与えるときも、必ずHPが1残るようになる。
  * 与えるダメージを「相手の現在HP − 1」に制限する。
  * 急所に当たった場合でも必ずHPは1残る。
* すでにHPが1の相手に使用したときもHPが1残る（＝実際に与えるダメージは0になる）。
  * ダメージが0になったとしても、攻撃を無効化したことにはならない。具体的には以下の効果が発生する。
    * ダメージを受けたときに発動する特性が発動する（例: せいでんき・どくのトゲ・てつのトゲ等）。
    * 持っていたふうせんが割れる。
    * さかさバトルで効果抜群ならじゃくてんほけんが発動する（本プロジェクトはさかさバトル非対応のため対象外）。
  * 技の効果が発動したことを明示するメッセージは流れない。
    * こらえるを使用している相手のHPを1にしたとき、現行世代（第五世代以降）ではこらえるの効果が優先されてメッセージが流れる（最終的なHPはどちらの効果でも1で変わらない）。
    * 持ち物きあいのタスキでHPが満タンの相手のHPを1にしたときも、みねうちの効果が優先されてタスキは消費されない。
    * 特性がんじょうでHPが満タンの相手のHPを1にしたときも、みねうちの効果が優先されて特性バーは現れない（発動しない）。
* 技の効果は追加効果ではないので、使用者の特性がちからずくであるときや、対象がりんぷん/おんみつマントを持つときでもHPを1残す効果は適用される。
* みがわり状態に防がれたときはみがわりのHPが残る効果はなく、通常の技と同様にみがわりのHPを削り、みがわり状態を解除できる（本体のHPを対象とした1残し効果はみがわりには適用されない）。
* Zワザ・ダイマックスわざにした場合は通常の攻撃技となり、相手のHPを1残す効果は失われる（本プロジェクトはZワザ・ダイマックスわざ非対応のため対象外）。
* このわざだけでは相手を倒すことができない。

## 判定フラグ

* 急所: あり
* 命中判定: あり
* 追加効果: なし（技の効果自体は追加効果ではない）
* まもる: 防がれる
* おうじゃのしるし: 発動する
* マジックコート: 反射不可（変化技ではないため対象外）
* よこどり: 対象外（変化技ではないため）
* オウムがえし: コピー可能

## 優先度（docs/spec/turn.md 参照）

* `Event.ON_MODIFY_MOVE_DAMAGE`（turn.md 表記: `Event.ON_MODIFY_DAMAGE`）: 60番「みねうち・てかげん」
  * がんじょう（100）・きあいのタスキ（100）・きあいのハチマキ（100）より必ず先に発動させる必要がある。
    先に「相手の現在HP − 1」へ補正しておくことで、これらの特性・持ち物が誤って発動・消費されるのを防ぐ。

## 既知の未実装範囲（備考）

* 「すでにHPが1の相手に使用したとき、ダメージを受けたときに発動する特性（せいでんき等）が発動する」効果は未実装。
  現在の実装は`Event.ON_MODIFY_MOVE_DAMAGE`でダメージ値を0に補正するのみで、`actual_damage <= 0`のため
  `Event.ON_DAMAGE_HIT`（せいでんき等が購読するイベント）が発火しない。
  実際のダメージが0であっても攻撃自体は命中している、という特殊な扱いのため、コアエンジン
  （`core/move_executor.py`）側の設計変更が必要になる可能性があり、本レビューではスコープ外として見送る。
* 「そうでん状態で使用したときやさかさバトルで効果抜群ならじゃくてんほけんが発動する」は、
  さかさバトル自体が本プロジェクトの対象範囲外（README.md参照）のため対象外。
