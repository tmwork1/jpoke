# ゆびをふる（Metronome）仕様書

参照:
* docs/wiki/moves/ゆびをふる.html（ポケモンWiki）
* docs/spec/moves/へんしん.md（同様に実装保留とした前例）

## 基本情報

* タイプ: ノーマル
* 分類: 変化
* 威力: -
* 命中: - （必中）
* PP: 10（`src/jpoke/data/ps-champ-ja/moves.json` に本技のエントリが存在しないため、README記載の
  フォールバック規則「チャンピオンズ側でルールが異なる場合はそちらを優先し、記載がなければ
  第9世代の値を基本とする」に従い、第9世代（スカーレット・バイオレット）基準の値10を採用する）
* 対象: 不定（選ばれた技に依存する。プレースホルダーとしてデフォルトの`foe`を採用）
* 優先度: 0
* 直接攻撃: なし

## 効果（第五世代以降 / 本プロジェクトが対象とするルール基準）

指をふり自分の脳を刺激して、一部を除いたほぼ全ての技の中からランダムに1つを選び、その技を
本技の代わりに繰り出す。

### 選ばれる技の範囲
* そのソフト（本プロジェクトの対象範囲であるスカーレット・バイオレット/チャンピオンズ）で
  使用できない技は選ばれない。
* 上記に加え、以下のような技は選ばれない（ポケモンWiki「ゆびをふるで出る技」表の第9世代列より。
  網羅的な一覧ではなく、代表的な分類のみ記載する）:
  * 他の技が出る技（ゆびをふる自身、オウムがえし、まねっこ、ねごと、さきどり、へんしん、
    ものまね、スケッチ等）
  * まもる系統の技（まもる、みきり、トーチカ、キングシールド、ワイドガード等）
  * 行動順・反撃に関する技（きあいパンチ、カウンター、ミラーコート、みちづれ、このゆびとまれ、
    でんこうそうげき、フェイント等）
  * だいばくはつ・こらえる等の一部の技、わるあがき
  * 第五世代以降の伝説・幻ポケモンの専用技の一部（全てではない）
  * 第八世代以降では、一般ポケモンの専用技の一部（全てではない）や、特に特徴のない技でも
    選ばれないことがある（Zワザ・ダイマックス技を含む）
* 選ばれた技の優先度に関係なく、ゆびをふる自体の優先度（0）で繰り出される。

## 技の仕様

* ゆびをふるで出た技は、アンコールの対象にできない。
* 第三世代以降、ねごとで選ばれない。
* ねこのてで選ばれない。
* まねっこでコピーできない（第二世代以降一貫して×）。
* この技に対するさいはいは失敗する。
* ゆびをふるに対するものまねは、第八世代まではゆびをふる自体をコピーする。第九世代では失敗する。
* 同じ技が連続して選ばれない限り、メトロノーム（アイテム）の効果は発生しない
  （複数ターン行動する技が選ばれた場合は効果を受ける）。
* じゅうりょく状態で使用できない技が選ばれた場合、じゅうりょくの影響で失敗する。
* かいふくふうじ状態のとき、回復技が選ばれても失敗する。
* こだわり系アイテムを持っている状態でゆびをふるを使い、数ターン続く技や溜め技が選ばれた場合、
  最初のターンは成功するが2ターン目の行動は必ず失敗する。

## 判定フラグ

* 急所: 選ばれた技に依存する
* 命中判定: 選ばれた技に依存する
* 追加効果: 選ばれた技に依存する
* まもる・みきり・トーチカ: 貫通する（防がれない）
* マジックコート: 反射されない
* よこどり: 対象外
* オウムがえし: コピー不可

## 実装状況

実装済み。既存のねごと（`ねごと_select_and_execute`）と同じ「`battle.run_move()` を直接呼ぶ」
パターンで実装する（`handlers/move_status.py` の `ゆびをふる_select_and_execute`）。
候補は `_METRONOME_EXCLUDED_MOVES`（代表的な分類のみの非網羅的な除外セット）と
`protect` フラグで除外する。選ばれた技のPPは消費しない（`ゆびをふる_suppress_pp`）。
伝説等の専用技の一部除外はスコープ外。詳細は `docs/plan/moves/ゆびをふる.md` を参照。
