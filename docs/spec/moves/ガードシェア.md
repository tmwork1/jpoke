# ガードシェア（Guard Split）仕様書

参照: docs/wiki/moves/ガードシェア.html（ポケモンWiki）

## 基本情報

* タイプ: エスパー
* 分類: 変化
* 威力: -
* 命中: - （必中）
* PP: 12（Champions基準。src/jpoke/data/ps-champ-ja/moves.json参照。Gen9本家では10）
* 対象: 相手1体
* 優先度: 0
* 接触: なし

## 効果

* 自分と相手の「ぼうぎょ」「とくぼう」の実数値を、両者の合計を1/2ずつに分けた値（平均値、小数点以下切り捨て）に書き換える。
* ランク変化は一切行わない。実数値そのものを書き換える。
* 参照する値は「つよさをみる」画面で確認できる実数値のみ。
* ランク補正・持ち物・特性による補正は、ガードシェアで書き換えた後の実数値に対してかかる。
* 実数値を変更されたポケモンが急所に攻撃を受けた場合も、変更後の実数値でダメージを計算する。
* ガードシェアで変更した実数値は交代でリセットされる（種族値ベースの値に戻る）。
* フォルムチェンジ（ウルトラバーストを含む）で効果は消えてリセットされる。
* メガシンカ: 第七世代では実数値がリセットされるが、Championsでは維持される
  （メガシンカにより防御・特防の種族値が変わるポケモンであっても実数値は変わらない）。
* へんしん状態になった場合、ガードシェアの効果は消える。ガードシェアで実数値を変更された
  ポケモンに対してへんしんした場合は、変更後の実数値をコピーする。
* ダイマックス/キョダイマックス/テラスタルでは基本的にリセットされない
  （ダイマックス/テラスタルと同時にフォルムチェンジするポケモンの場合はリセットされる）。
* テラレイドバトルで使用すると失敗する（本プロジェクトのシングルバトル対象外のため考慮不要）。

## 判定フラグ

* 急所: 対象外（変化技のため急所判定自体が発生しない）
* 命中判定: 必中（命中判定自体が行われない）
* 追加効果: なし
* まもる: 防がれる
* おうじゃのしるし: 対象外
* マジックコート: 反射不可
* よこどり: 対象外（コピー不可）
* オウムがえし: コピー不可（「対象の状態を参照する技」に分類され例外的にコピー対象から除外される）
* みがわり: 相手がみがわり状態の場合は防がれる

## 備考

* パワーシェア（こうげき・とくこうの実数値平均化）、スピードスワップ（すばやさの実数値入れ替え）、
  ガードスワップ（ぼうぎょ・とくぼうの**ランク**入れ替え）と類似・対比される技。
* 本プロジェクトの対象範囲はポケモンチャンピオンズシングルバトルのみのため、
  「自分以外の1体が対象のわざ」カテゴリの意味も含め、対象は常に相手1体で固定となる
  （ダブルバトルで味方を対象にする運用は対象外）。

## 本プロジェクトでの実装範囲

* 実数値の書き換え・交代時のリセットまでを実装する。
* フォルムチェンジ・メガシンカ・へんしん・ダイマックス/テラスタルとの相互作用は、
  それぞれの機能側の仕様書（該当する特性・状態変化の仕様書）で扱う極めて限定的なケースであり、
  現状 `docs/spec/abilities/` 側に「ガードシェアで変化した実数値を考慮する」旨の記述がある
  特性（てんきや・てんねん・ばけのかわ・アイスフェイス・はらぺこスイッチ 等）についてのみ
  個別に対応する。
