# カウンター（Counter）仕様書

参照: docs/wiki/moves/カウンター.html（ポケモンWiki）

## 基本情報

* タイプ: かくとう
* 分類: 物理
* 威力: -（固定ダメージ技）
* 命中: 100
* PP: 20
* 対象: 相手1体
* 優先度: -5
* 接触: あり（直接攻撃）

## 効果

* そのターンに相手から受けた物理技のダメージを2倍にして、同じ相手に返す（ダメージ固定技）。
* 使用したターンに相手から物理技によるダメージを受けていない場合は失敗する。
* 複数回物理ダメージを受けていた場合（連続技など）は、**最後の1回分のダメージのみ**を2倍にする
  （合計ダメージではない）。
* 相手から物理技を受けたが、無効化された場合や命中しなかった場合は跳ね返す対象にならない
  （＝そのターン他に物理ダメージを受けていなければ失敗する）。
* 味方から受けた物理技には反撃しない（本プロジェクトはシングルバトル限定のため、通常は考慮不要）。

## 判定フラグ

* 急所: 発生しない（ダメージ固定技のため）
* 命中判定: あり（命中率100）
* 追加効果: なし
* まもる: 防がれる（第五世代以降）
* おうじゃのしるし: 発動する（第五世代以降。ひるみ効果は追加効果扱いのため本来は対象外だが、
  本プロジェクトでは おうじゃのしるし 自体が「攻撃技でひるみ追加」の一般規則として実装されるべきもので、
  カウンター固有の実装は不要）
* マジックコート: 反射不可（変化技ではないため対象外）
* よこどり: 対象外（変化技ではないため）
* オウムがえし・さきどり・まねっこ: コピー不可（第四世代以降）。本プロジェクトでは まねっこ のみ実装済みのため、
  `non_copycat` フラグで対応する
* ゆびをふる・ねこのて: 選ばれない（両技とも未実装のため対応不要）
* ちょうはつ状態でも使用できる（変化技ではないため、ちょうはつの影響を受けない。特別な実装は不要）
* とつげきチョッキを持っていても使用できる（物理技のため、特別な実装は不要）
* はりきりの効果で命中率が下がる（物理技全般に適用される既存の一般規則に従うため、固有実装は不要）
* カウンター自身も物理技に属している（ふいうちの成立対象になる。かみつき等の判定フラグは持たない）
* 反撃時、相手の特性『さめはだ』『てつのトゲ』、持ち物『ゴツゴツメット』などの直接攻撃反応効果を受ける
  （直接攻撃の一般規則に従うため、固有実装は不要）

## 優先度根拠

`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` テーブルに以下の記述がある：

```
| 30 | カウンター・ミラーコート: 適格ダメージ無 |
```

したがって、使用可否判定ハンドラは `Event.ON_TRY_MOVE_1` の priority=30 で実装する。

ダメージ計算の固定化（`Event.ON_MODIFY_MOVE_DAMAGE`）は turn.md に個別記載が無い内部イベントのため、
既存の同種ハンドラ（ミラーコート等）に合わせてデフォルト priority=100 を採用する。

## 既知の未対応エッジケース（対象外）

以下は第五世代以降の公式仕様として存在するが、いずれも「反撃対象の元HPがすでに1で、
きあいのハチマキ／がんじょう等の効果でダメージが0に補正されたことにより、実際にはHPが
減少しなかった」という極めて限定的な状況でのみ発生する。この場合、本来は「無効化された
攻撃」とは異なりカウンターは失敗せず固定1ダメージを返す仕様だが、本実装ではダメージ量が
0になった時点でその物理攻撃自体を「受けていない」ものとして扱うため、この極小エッジケースは
未対応（対象外）とする。

* 相手からの物理攻撃でHPが0まで減る予定だったが、きあいのハチマキ／なかよし度の効果で
  HP1で耐えた場合（耐えた結果HPが変化しなかったとき）→ 本来は1ダメージを返す
* 相手からの物理攻撃を受けたが、みねうち・てかげんなどにより残りHP1のまま変化しなかった場合
  → 本来は1ダメージを返す
* 相手からの物理攻撃を『ばけのかわ』の発動により無効化した場合 → 本来は1ダメージを返す
