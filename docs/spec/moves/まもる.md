# まもる（Protect）仕様書

参照: docs/wiki/moves/まもる.html（ポケモンWiki）、docs/wiki/volatiles/まもる_(状態変化).html（ポケモンWiki）、
docs/champions/move_list.txt

## 基本情報

* タイプ: ノーマル
* 分類: 変化
* 威力: なし
* 命中: なし（必中）
* PP: 8（champions基準。docs/champions/move_list.txt 参照。Gen9本家は10）
* 対象: 自分
* 優先度: +4
* 接触: なし

## 効果

* 使用したターンの間、自分に「まもる」揮発性状態を付与する。相手から受ける攻撃技・変化技の大部分を
  無効化する（付与条件・防御対象・解除条件などの詳細は docs/spec/volatiles/まもる.md 参照）。
* 連続で使う度に成功率が1/3になっていく（最初は必ず成功、以降は1/3, 1/9, ... と下がり、下限は1/3^6=1/729）。
  失敗するとカウントはリセットされる。
  * 本プロジェクトでは、まもる系技共通の連続使用失敗判定（`まもる系_連続使用失敗チェック`）を採用し、
    「直前ターンに守る系技（`protect` フラグを持つ技）を成功させていた場合、今ターンは必ず失敗する」
    という簡略化モデルで実装する（確率的に成功率が1/3ずつ低下していく詳細モデルは
    未実装・スコープ外とする。詳細は docs/spec/volatiles/まもる.md 参照）。
  * 同じ守る系技の連続使用に限らず、まもる・みきり・こらえる・キングシールド・ニードルガード・トーチカ・
    スレッドトラップ・かえんのまもり等、`protect` フラグを持つ技同士であれば異なる技を挟んでも連続使用扱いになる。
  * 交代を挟んだ場合、または直前ターンに `protect` フラグを持たない技を使った場合はリセットされ、
    次のまもるは連続使用扱いにならず成功する。

## 判定フラグ

* 急所: なし（攻撃技ではないため対象外）
* 命中判定: なし（必中）
* 追加効果: なし
* まもる: 対象外（自分を対象とする変化技のため通常は問題にならない）
* おうじゃのしるし等: 発動しない
* マジックコート・マジックミラー: 反射不可（自分対象技のため対象外）
* よこどり: 対象外（自分対象技のため通常はよこどりされない）
* まねっこ: コピー不可（`non_copycat` フラグ）
* ゆびをふる・ねこのて: 選ばれない（本プロジェクトではゆびをふる・ねこのて自体が未実装のため対応不要）

## 優先度（docs/spec/turn.md 参照）

* `Event.ON_TRY_MOVE_2`（`attacker:self`）: まもる系連続使用失敗チェック。turn.md では
  `Event.ON_TRY_MOVE_1` の30番「まもる系: 連続使用により失敗」に該当する記載があるが、既存実装
  （みきり・キングシールド・こらえる等）はすべて `Event.ON_TRY_MOVE_2` のデフォルト priority（100）で
  実装されているため、同一パターンに合わせる（docs/spec/moves/こらえる.md も参照）。
* `Event.ON_STATUS_HIT`（`attacker:self`）: 自分に「まもる」揮発状態を付与する。turn.md に個別 priority
  行の記載がないため、既存の変化技ハンドラと同様にデフォルト（100）を採用する。
* まもる volatile 自体（`Event.ON_TRY_MOVE_1` priority=100 で技を無効化、`Event.ON_TURN_END` で解除）は
  docs/spec/volatiles/まもる.md を参照。

## 既知の未実装範囲（備考）

* 連続使用時の確率的な成功率低下（1/3ずつ低下、下限1/729）は未実装。本プロジェクトはまもる系共通の
  簡略化モデル（連続使用で必ず失敗）を採用する（docs/spec/volatiles/まもる.md 参照）。
* `executed_move` は「直前に実際に実行された技」を保持するフィールドだが、まひ・ねむり・ひるみ等で
  行動自体ができなかったターンを挟んでも値がリセットされない（`ON_TRY_ACTION` 失敗時に更新されないため）。
  そのため、本来は「動けなかったターンを挟むと連続使用とみなされず成功する」仕様のところ、まれに
  連続使用扱いのまま判定されるケースがありうる（`いちゃもん`・`アンコール` 等、他機能が同じ
  `executed_move` を「直前に実行した技（ターンをまたいでも可）」として利用しているため、
  この用途のために独立したフィールドを設けていない）。既知の限界としてスコープ外とする。
* さいはい・おどりこ等ダブルバトル専用技との相互作用は、本プロジェクトの対象範囲（シングルバトル）外のため未実装。
* ダイマックスわざ・Zワザは第9世代（champions基準）に存在しないため対象外。
