# 技仕様書: おちゃかい（Teatime）

**対象世代**: 第9世代（ポケモンチャンピオンズ）
**バトル形式**: シングルバトル

---

# 基本データ

- タイプ: ノーマル
- 分類: 変化
- 威力: なし
- 命中率: なし（必中扱い）
- PP: 12（`src/jpoke/data/ps-champ-ja/moves.json` L152 準拠。第八世代本家は10）
- 対象: 自分含む場の全ポケモン（実装上は `target="self"` を採用。理由は「技の仕様」節を参照）
- 優先度: 0
- 直接攻撃: いいえ
- 専用技（ポットデス）

---

# 基本効果

場にいるポケモン全員（自分・相手の両方）が持っている「きのみ」を、発動条件を無視して即座に
強制消費させ、その効果を受けさせる。

消費順序（本プロジェクトはシングルバトルのみのため、実質的には使用者→相手の順）:
1. 使用者
2. 使用者の味方（ダブル/トリプル用。シングルでは存在しない）
3. 相手から見て左側
4. 相手から見て右側（ダブル/トリプル用。シングルでは存在しない）

きのみを消費できるポケモンが1体もいない場合でも、技自体は成功したものとして扱う
（失敗判定は行わない）。

---

# 技の仕様（wikiより）

- 使用すると「おちゃかいをして みんなで きのみを 食べた!」というメッセージが出る。
  きのみを消費できるポケモンがいない場合でもこのメッセージは表示され、続けて
  「しかし 何も 起こらなかった!」と表示される。
  → 本プロジェクトは技専用の演出テキストを実装しない方針（`おかたづけ` 等と同様）のため、
  この文言自体は実装対象外。技の成否判定への影響のみを実装する。
- 発動条件があるきのみ（例: HPが1/2以下で発動するオボンのみ等）であっても、その条件を
  無視して消費させることができる。
- 半減のきのみ/ジャポのみ/レンブのみ/イバンのみや、状態異常でないときのラムのみなど、
  効果が発動しないきのみを消費した場合も、消費自体は行われる（見た目上の演出のみの違いで、
  本プロジェクトの戦闘結果には影響しない）。
- 基礎ポイントを下げるきのみ（努力値操作系）も消費されるが、本プロジェクトは
  努力値システムを扱わないため対象外。
- マックスレイドバトル/テラレイドバトルで使用したときは失敗する
  → レイドバトル自体が本プロジェクトの対象外のため実装しない。
- 姿を隠している（そらをとぶ・あなをほる・ダイビングで待機中の）ポケモンはきのみを
  消費しない。
- みがわり状態のポケモンもきのみを消費する（みがわりは技の直接的な対象選択を経由しないため、
  実装上は自然に満たされる）。
- まもる/トリックガード状態を無視して発動するが、ダイウォールには防がれる
  → ダイウォール（ダイマックス関連の技）は本プロジェクト対象外のため考慮しない。
- きのみを消費したポケモンが特性ほおぶくろ/はんすうを持つ場合、効果が発動した場合に限り
  特性の効果も発動する（効果が発動しなかった場合は発動しない）
  → 両特性とも本プロジェクトでは未実装（`AbilityData()` の空データのまま）のため、
  相互作用の実装は対象外。
- きのみを消費したポケモンは基本的に「ゲップ」を使えるようになる（効果が発動しなかった
  場合も含む。ただし努力値操作系のきのみを消費した場合は使えない）
  → 本プロジェクトの `battle.item_manager.consume_item()` は消費時に無条件で
  `mon.ate_berry = True` を設定するため、既存の `ゲップ` 実装と自然に整合する。
- 特性ねんちゃくのポケモンもきのみを消費する
  → `ねんちゃく_prevent_item_change` は `ctx.source != ctx.target` の場合のみ阻止するため、
  自分自身の消費（source=target）では阻止されず、実装上自然に満たされる。
- 特性きんちょうかんの影響を受けているポケモンもきのみを消費する
  → きんちょうかんはトレーナー変化技・特性発動の一部を封じる効果であり、
  アイテム消費処理自体には関与しないため、実装上自然に満たされる。
- 特性ぶきようやマジックルーム状態で持ち物が使えないポケモンもきのみを消費し、
  その効果を受ける
  → **既知の実装上の制限**: 本プロジェクトの `Item.enabled` ゲート機構（`ぶきよう`/
  `マジックルーム` 下では `item.name` が空文字列扱いになり `is_berry()` が False を返す）
  により、このケースでは効果発動だけでなく消費自体もスキップされる。この制限は
  `ほおばる`（Stuff Cheeks）にも共通する既存の未対応エッジケースであり、本技の
  レビューでは踏み込んだ修正を行わない。
- おちゃかいで消費したきのみはものひろい/しゅうかく/リサイクル/キョダイサイセイで復活する
  → `battle.item_manager.consume_item()` が `last_lost_item_name` を更新するため、
  ものひろい/しゅうかく/リサイクルの復元条件は既存実装のまま自然に満たされる。
  キョダイサイセイ（ダイマックス関連）は対象外。
- 特性いたずらごころの効果で優先度+1にして使用した場合の追加ルール（あくタイプ/
  サイコフィールドによる無効化、じょおうのいげん/ビビッドボディ/テイルアーマーによる
  技全体の失敗、ファストガードを無視して消費させる、でんきタイプ化時のちくでん等の対応）は、
  優先度が上がった状態異常技全般に共通する汎用ルールであり、おちゃかい固有の実装は不要
  （ポットデスはいたずらごころを持たないため実戦では起こらない組み合わせでもある）。

# 判定

- 急所: なし
- 命中判定: なし（必中扱い）
- 追加効果: なし
- まもる: 効果なし（まもるで防がれない）
- トリックガード: 効果なし（防がれない）
- おうじゃのしるし（第五世代以降）: 効果なし
- マジックコート: 反射されない
