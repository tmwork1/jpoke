## 効果の詳細

- 使用者と対象の特性を入れ替える。
- 入れ替わった特性は、片方が交代・ひんしになってももう片方はそのまま
  （`base_ability_name` に基づき、交代時に元の特性へ戻る）。
- 自分と対象の特性が同じ場合でも、第六世代以降（本プロジェクトが対象とする
  SVを含む）は成功する（第五世代のみ失敗する仕様だった）。
- 入れ替わった特性が「場に出たときに発動する特性」（いかく等）であった場合、
  第五世代以降は入れ替えた直後に即座に発動する。発動順は、すばやさに依らず
  使用者側の新しい特性が先に発動する。
- 対象がみがわり状態でも成功する。
- 対象がとくせいなし状態でも成功する（無効化された特性の名前（空文字列）を
  そのまま入れ替える）。

## 失敗条件

以下のいずれかに該当する場合、技は失敗する。

- 自分と対象のどちらかの特性が「上書き・変更できない特性」（`protected`
  フラグを持つ特性: ARシステム・アイスフェイス・イリュージョン・うのミサイル
  （第八世代までは対象、第九世代=本プロジェクト対象範囲では入れ替え可能）・
  おもかげやどし・ぎょぐん・クォークチャージ・こだいかっせい・
  しれいとう・じんばいったい・スワームチェンジ・ぜったいねむり・ダルマモード・
  テラスシェル・テラスチェンジ・ゼロフォーミング・ばけのかわ・バトルスイッチ・
  はらぺこスイッチ・ふしぎなまもり・マイティチェンジ・マルチタイプ・
  リミットシールド等）である場合。使用者の特性がかたやぶりであっても、
  この失敗条件は回避できない。
  - かがくへんかガスは `protected` フラグを持たない（いえき/コアパニッシャー
    を受けるととくせいなし状態になれるため）が、スキルスワップ/さまようたましい
    による交換だけは対象外のため、`base_name` による個別判定で失敗させる
    （`docs/spec/abilities/かがくへんかガス.md` 参照）。
- 対象の特性がおうごんのからだ（Good as Gold）である場合は、変化技全般の
  無効化処理により防がれる。ただし使用者の特性がかたやぶりであれば成功する
  （おうごんのからだ自体はスキルスワップで入れ替え可能）。
- 自分か対象のどちらかの持ち物がとくせいガードである場合。

## 判定フラグ

- 急所: 対象外（変化技）
- 命中判定: なし（必中技。`accuracy=None`）
- 追加効果: なし
- まもる: 防がれる（`unprotectable` フラグなし。デフォルトの変化技扱い）
- みがわり: 防がれない（`bypass_substitute`）
- おうじゃのしるし: 対象外（変化技）
- マジックコート・マジックミラー: 反射不可（`unreflectable`）
- よこどり: 対象外（Champions未採用の技のため未実装）
- オウムがえし: 対象外（変化技のため無関係。オウムがえし自体も未実装）
- トリックガード: 対象外（Champions未採用の技のため未実装）
- 特性おうごんのからだ: 防がれる（変化技共通の無効化処理。使用者の特性が
  かたやぶりの場合は防がれずに成功する）

## 実装方針

- イベント: `ON_BEFORE_APPLY_MOVE`（priority 100, `docs/spec/turn.md` の
  `Event.ON_APPLY_MOVE` セクションに「スキルスワップ: 失敗条件」として記載）
  - 使用者または対象の特性が `protected` フラグを持つ場合、使用者または対象の
    `base_name` が「かがくへんかガス」の場合、または
    `battle.ability_manager.is_change_blocked(ctx.attacker)` /
    `is_change_blocked(ctx.defender)` が真の場合（とくせいガード等）に失敗させる
    （シンプルビーム・いえき・なりきり等と同一パターン）。
    - なりきり・いえきは対象側のみをチェックするが、スキルスワップは使用者・
      対象の両方の特性を書き換えるため、双方のチェックが必要な点が異なる。
  - おうごんのからだによる無効化は `Event.ON_BEFORE_APPLY_MOVE`
    （`おうごんのからだ_block_status_move`, priority 100, `defender:self`）
    で汎用的に処理されるため、スキルスワップ固有のガードは不要。
- イベント: `ON_STATUS_HIT`（priority 100、変化技の既定値）
  - `battle.ability_manager.swap_ability(ctx.attacker, ctx.defender)` を呼び出し、
    使用者と対象の特性を入れ替える。
  - `swap_ability` は内部でも `is_change_blocked` を再チェックしており
    （とくせいガードを途中で失った等のケースを想定した二重防御）、
    `ON_ABILITY_DISABLED` → ハンドラ解除 → 特性入れ替え → ハンドラ再登録 →
    `ON_ABILITY_ENABLED` の順に発火するため、入れ替え直後の特性発動
    （いかく等）や発動順（使用者側の新特性が先）は自動的に再現される。

## 本実装での対象外（スコープ外）

- よこどり・オウムがえし・トリックガードによる相互作用（いずれも
  Championsのルール上使用不可のため未実装）。
- ダイマックス関連の失敗条件（本プロジェクトの対象範囲外）。
