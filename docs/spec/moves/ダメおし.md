# ダメおし（Assurance）仕様書

参照: docs/wiki/moves/ダメおし.html（ポケモンWiki）

## 基本情報

* タイプ: あく
* 分類: 物理
* 威力: 60（第六世代以降。第五世代までは50だが本プロジェクトは現行世代のみ対応のため考慮不要）
* 命中: 100
* PP: 12（`src/jpoke/data/ps-champ-ja/moves.json` 準拠。本家基準は10だが、
  `docs/champions/changes_from_sv.md` により全技PPは8/12/16/20の4段階に統一されている）
* 対象: 相手1体
* 優先度: 0
* 接触: あり

## 効果

* そのターン中に対象（相手）がすでにダメージを受けていた場合、威力が2倍（120）になる。

## 判定フラグ

* 急所: 通常通り発生する
* 命中判定: あり（命中率100）
* 追加効果: なし
* まもる: 防がれる
* おうじゃのしるし: 効果あり
* マジックコート: 反射不可（変化技ではないため対象外）
* よこどり: 対象外（変化技ではないため）
* オウムがえし: コピー可能（本プロジェクトはオウムがえし自体が未実装のため対応不要）

## 「そのターンにダメージを受けた」の判定詳細（一次情報: 技の仕様節）

以下のダメージ源を受けていた場合に威力が2倍になる（第八世代以降＝本プロジェクトの対象世代の挙動）。

### 2倍になる（○）

* 攻撃技を受けたときのダメージ
* 特性によるダメージ（さめはだ/てつのトゲ/ヘドロえき/ゆうばく/とびだすなかみ/うのミサイル等）
* ゴツゴツメット/ジャポのみ/レンブのみによるダメージ
* 反動技の反動ダメージ（すてみタックル系、わるあがき/クロロブラストの反動、
  とびひざげり系の技失敗時の反動、てっていこうせん使用時の反動を含む。ビックリヘッドは
  第9世代に存在しないため本プロジェクトでは実装対象外＝対応不要）
* いのちのたまの反動ダメージ
* 設置技のダメージ（交代時＝ターン中に強制交代等で入れ替わったときに受けた場合）
* HP1のときのみねうちやこらえるなどにより、実際のダメージが0になった場合
  （ダメおしを受ける前にそのポケモンのHPが回復していれば影響が表れる）
* ばけのかわ/アイスフェイスでダメージを肩代わりした相手（実際のHPは減っていなくても対象になる）

### 2倍にならない（×）

* みがわりでダメージを防いだ相手（みがわりの身代わりが受けただけで、本体はダメージを受けていない扱い）
* こんらんによる自傷ダメージ
* みがわり/のろい（呪い）/はらだいこ/ソウルビート/しっぽきり/みをけずる使用時の自己HP消費ダメージ
* いたみわけの使用、または受けたときによるダメージ
* 天候（すなあらし等）など、前のターン終了時の処理で受けただけのダメージ（どく/やけどの
  定期ダメージ等、残りHPを毎ターン終了時に減少させる処理全般を含む）
* 設置技のダメージ（死に出し時＝ひんしによる交代でそのターン最初に受けた場合。
  本プロジェクトは第八世代以降のみ対応のため、死に出し直後の設置技ダメージは対象外）

## 特性との相互作用

* テクニシャン: ダメおしの効果が発動しなかったとき（威力60のまま）はテクニシャンの補正で威力90になる。
  ダメおしの効果が発動し威力120になったときはテクニシャンの補正はかからない。
  **既知の制限**: 本プロジェクトの `テクニシャン_boost_power`
  （`src/jpoke/handlers/ability.py`）は技の**変動前**の基礎威力（`ctx.move.data.power`）のみを参照して
  判定しており、ダメおし自身の威力変動（`ON_CALC_POWER_MODIFIER` での2倍補正）を考慮していない。
  これは `アクロバット`/`ソーラービーム` 等、威力が変動する技全般に共通するテクニシャン側の実装課題
  （`docs/spec/abilities/テクニシャン.md` 参照）であり、本仕様書・本レビューの対象外として、
  テクニシャンの実装レビュー（`docs/progress/ability.md` のテクニシャン行）側で扱う。

## 実装上の制約（既知の限界）

* `hits_taken`（`Pokemon` クラス、「そのターンに攻撃を受けたか」を表すカウンタ）は、以下の経路で
  加算される：
  1. 通常の攻撃技ヒット（`core/move_executor.py`、`actual_damage > 0` のとき）
  2. HP1でのみねうち/こらえるによる0ダメージ（`みねうち_modify_damage` / `こらえる_endure` で
     手動インクリメント）
  3. ばけのかわ/アイスフェイスによる肩代わり（`ばけのかわ_block_damage` /
     `アイスフェイス_block_physical` で手動インクリメント）
  4. 反動ダメージ（`reason in ("recoil", "fixed_recoil")`。`core/status_manager.py` の
     `modify_hp` で一括インクリメント。すてみタックル系・とびひざげり系失敗反動・
     てっていこうせん・クロロブラスト等を含む）
  5. いのちのたまの反動ダメージ（`いのちのたま_recoil` で手動インクリメント）
  6. ゴツゴツメット（`_apply_contact_item_chip`）、ジャポのみ/レンブのみ
     （`_retaliate_on_category`）、さめはだ/てつのトゲ（`_apply_contact_counter_chip`）による
     ダメージ（各ヘルパー関数で手動インクリメント）
* 以下は一次情報上は2倍対象（○）だが、本プロジェクトでは未対応（既知の限界。発生頻度が
  極めて低いエッジケースのため対応を見送る）:
  * わるあがきの反動ダメージ: 実装上 `reason="self_cost"` を用いている（いしあたま非対象という
    別の目的のため）。`self_cost` は みがわり/のろい/はらだいこ 等の「2倍にならない」自己HP消費と
    共用のタグであり、これらと区別する追加のリーズンを新設しない限り、わるあがきの反動のみを
    対象に含めることができない。わるあがきはPP切れ時のみ使用される技であり、かつ同ターンに
    相手からダメおしを受ける状況が必要という極めて稀な組み合わせのため対応を見送る。
  * 特性ゆうばく（相手を倒したときに攻撃者がダメージを受ける）、とびだすなかみ/うのミサイル
    （ウッウ専用の特性ギミック）: シングルバトルでは、ゆうばく発動時点で対象が瀕死になっており、
    第八世代以降の仕様では死に出し後にターンが終了したものとみなされるため、そのポケモンが
    同ターン中に別の技（ダメおし含む）を受ける状況が発生しない。対応不要。
  * 設置技のダメージ（交代時＝ターン中の強制交代で受けた場合）: `どろぼう`/`ふきとばし`等による
    ターン中強制交代からの設置技ダメージと、その後の`ダメおし`との組み合わせは、本プロジェクトの
    `hits_taken` では追跡していない。発生頻度が極めて低いエッジケースのため対応を見送る。

## リーサル計算

* 威力が「相手がそのターンにダメージを受けたか」という一時的なターン内状態
  （`Pokemon.hits_taken`）に依存するため、通常のダメージ計算
  （`battle.calc_damages`）が `Event.ON_CALC_POWER_MODIFIER` を発火させる時点の
  `ctx.defender.hits_taken` を参照すれば足り、専用の `lethal_handlers` は不要。
* `docs/progress/move.md` 上も「リーサル実装: n/a」として扱う。
