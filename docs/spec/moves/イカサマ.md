# イカサマ（Foul Play）仕様書

参照: docs/wiki/moves/イカサマ.html（ポケモンWiki）
参照: docs/spec/abilities/クォークチャージ.md / docs/spec/abilities/こだいかっせい.md（パラドックス特性との相互作用）

## 基本情報

* タイプ: あく
* 分類: 物理
* 威力: 95
* 命中: 100
* PP: 16（チャンピオンズ準拠。src/jpoke/data/ps-champ-ja/moves.json）
* 対象: 相手1体
* 優先度: 0
* 接触: あり（直接攻撃○）

## 効果

自分の『こうげき』とランク補正の代わりに、相手（対象）の『こうげき』実数値とランク補正を用いて
ダメージ計算を行う。

## 技の仕様（重要な相互作用）

* 持ち物・特性・状態異常などの効果は、攻撃力の参照元（相手の実数値）に関わらず、他の物理技と
  同じように**イカサマの使用者側**の状態に伴ってダメージを計算する。
  * 使用者がやけど状態だとダメージが半減する。対象がやけど状態であっても影響を受けない。
  * 使用者の持ち物がこだわりハチマキの場合、対象の攻撃を1.5倍にしてダメージ計算を行う。
    対象の持ち物の影響は受けない。
  * 使用者の特性がちからもちの場合、対象の攻撃を2倍にしてダメージ計算を行う。対象の特性が
    ちからもちでも影響は受けない。
  * 使用者が他のポケモンの特性わざわいのおふだの効果を受けている場合、対象の攻撃を3/4倍に
    してダメージ計算を行う。使用者の特性がわざわいのおふだのときは影響がない。
* 対象の攻撃ランクが変化している場合はダメージもそれに伴う。使用者の攻撃ランクが変化していても
  影響はない。
  * 対象の特性がてんねんである場合、対象自身のこうげきランクはてんねんにより無視される。
  * 使用者の特性がてんねんである場合、対象の攻撃ランクは影響するが、ぼうぎょランクはてんねん
    により無視される（防御側の計算は通常通りのため、この技固有の対応は不要）。
* 対象の攻撃がパワーシェア/パワートリック/へんしん/フォルムチェンジなどで変動していた場合、
  変動後の攻撃を用いる。
* レベルは使用者の数値に依存する。
* 使用者があくタイプ（テラスタル込み）ならタイプ一致ボーナスが付く。対象があくタイプなだけでは
  ボーナスは付かない。
* パラドックス特性（クォークチャージ/こだいかっせい）は「使用者自身」の攻撃側スロット（この技は
  物理技のため『こうげき』）が強化対象のときのみ補正が乗る。対象（相手）の攻撃が特性で上昇して
  いても、イカサマを受けたときのダメージは変わらない
  （docs/spec/abilities/クォークチャージ.md 「特性の効果はランク補正上昇とは異なる」参照）。

## 判定フラグ

* 急所: 通常判定（急所ランク0）
* 命中判定: あり（命中率100）
* 追加効果: なし
* まもる: 防がれる
* おうじゃのしるし: 発動する（第五世代以降）
* マジックコート: 反射不可（変化技ではないため対象外）
* よこどり: 対象外
* オウムがえし: 反射可能

## 実装方針

* `src/jpoke/core/damage.py` の `_calc_final_attack` 内で `move.name == 'イカサマ'` を判定し、
  `defender.stats["atk"]` と `defender.rank_modifier("atk")` を最終攻撃力計算に使う特別分岐が
  すでに存在する。攻撃力の参照元のみが入れ替わり、その後の `ON_CALC_ATK_RANK_MODIFIER` /
  `ON_CALC_ATK_MODIFIER`（アイテム・特性・状態異常の補正）は通常の物理技と同じフローで発火する
  ため、追加のハンドラは不要。
* パラドックス特性（`src/jpoke/handlers/ability_paradox.py`）は、参照する実数値の出どころに
  関わらず「使用者自身の攻撃側スロット」のみを見て判定する（技名による分岐は不要）。
