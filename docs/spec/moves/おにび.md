## 効果

相手を **やけど** 状態にする。

## 失敗条件

おにびは技固有の失敗チェックを持たず、`ailment_manager.apply(defender, "やけど")` が
内部で行う汎用チェックにより以下の場合は失敗する：

1. 対象がすでに **状態異常**（どく・もうどく・まひ・やけど・ねむり・こおり）を持っている
2. 対象が **ほのおタイプ** である（やけど無効）

失敗時は `MoveExecutor._execute_status_hit` が `ON_STATUS_HIT` の戻り値が偽であることを
検知し、汎用的に `LogCode.MOVE_FAILED` を記録する（技固有の失敗ログ処理は不要）。

## 防がれる条件

以下の場合、おにびは防がれる（`ailment_manager.apply` 内、`ON_BEFORE_APPLY_AILMENT` 経由の
汎用判定）：

- しんぴのまもり状態（使用者にすりぬけがある場合は無視）
- みがわり状態（使用者にすりぬけがある場合は無視）
- みずのベール特性・もらいび特性（もらいび発動時は無効化されタイプ一致技として発動）

## イベントフロー

```
ON_STATUS_HIT (priority=100, MoveHandler デフォルト)
  → ailment_manager.apply(defender, "やけど")
     - すでに状態異常 OR ほのおタイプなら失敗（apply内部の汎用チェック）
     - ON_BEFORE_APPLY_AILMENT を発火し、みずのベール等の特性・
       しんぴのまもり等の場の状態・みがわりによる無効化を判定
```

## ポケモンチャンピオンズ向けの簡略化

- Z変化技付加効果は実装しない
- コンテスト効果は実装しない
