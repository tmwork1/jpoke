## 効果の詳細

- 使用者の特性を、対象（相手）の現在の特性と同じにする。
  - ダブルバトルでは自分と味方全員が対象になるが、本プロジェクトはシングルバトルのみを
    対象とするため、実質的に「使用者自身の特性を対象の特性にする」効果になる
    （`なりきり` と同一の効果）。
- 失敗条件:
  - 対象の特性が「コピーできない特性」（`uncopyable` フラグ）の場合、失敗する
    （例: かわりもの、ぜったいねむり、ばけのかわ、トレース）。
  - 使用者自身の特性が「上書きできない特性」（`protected` フラグ）の場合、失敗する
    （例: アイスフェイス、ぜったいねむり、ばけのかわ）。
    - シングルバトルでは「自分か味方のどちらかが上書きできない特性のときは、無視して
      もう片方だけ変更する」という分岐先（味方）が存在しないため、使用者自身が
      上書き不可なら技全体が失敗する。
  - 使用者と対象がすでに同じ特性の場合、失敗する
    （分岐先の「味方」が存在しないシングルバトルでは、これも技全体の失敗になる）。
  - 使用者の特性変更がとくせいガード等（`ON_CHECK_ABILITY_DISABLE`）で防がれる場合、
    失敗する。
    - 対象がとくせいガードを持っていても、それによって「対象の特性がコピーされること」
      自体は防がれない（とくせいガードは所持者自身の特性が変更されることのみを防ぐ）。
  - 対象の特性が「おうごんのからだ」の場合も失敗するが、これは `おうごんのからだ` 自身が
    持つ「自分を対象とする変化技を無効化する」効果（`ON_BEFORE_APPLY_MOVE`）による
    ものであり、うつしえ側に追加の実装は不要。
- 対象の特性がとくせいなし状態（いえき等）の場合は失敗せず、無効化された特性の名前
  （空文字列）をそのままコピーする（`なりきり` と同一の仕様。詳細は
  `docs/spec/moves/なりきり.md` の「特性コピーに関する仕様」を参照）。
  - 一方、対象の特性がかがくへんかガスにより抑制されている（とくせいなし状態ではない）
    場合は、抑制前の本来の特性（`base_name`）をコピーする。かがくへんかガスは対象の
    特性そのものを消去するのではなく効果の発動のみを抑制するため
    （`docs/spec/abilities/かがくへんかガス.md` 参照）。
- まもる状態の相手にも効果が発動する（`unprotectable` フラグ）。
- マジックコート・マジックミラーで跳ね返されない（`unreflectable` フラグ）。
  - ミラクルスキンの効果対象にはなる（命中率50%に固定され得る）。命中率が100と
    50を超えるため、実際にはミラクルスキンにより命中率が50に低下する。
- みがわり状態の相手には防がれる（`bypass_substitute` フラグを持たない変化技のため、
  既存の汎用みがわり判定でブロックされる）。
- ゆびをふるに選ばれない（`ゆびをふる` 自体が本プロジェクトで未実装のため、対応する
  除外フラグは存在しない。`ゆびをふる` 実装時に追って対応する）。
- ノーマルタイプの技だが、変化技のためタイプ相性による無効化はない。

## Pokémon Champions での補足

- コンテスト効果は実装しない。
- PP は `src/jpoke/data/ps-champ-ja/moves.json` に記載がないため、Gen9本家のPP（10）をそのまま
  採用する。タギングル（本技の専用習得者）は同ファイルの他の技と同様に個別の変更履歴が
  無く、専用技のため走査対象から漏れている可能性がある。今後 `move_list.txt` に追記が
  あれば値を更新すること。

## 実装方針

- イベント: `ON_BEFORE_APPLY_MOVE`（priority 100） — 失敗条件の判定
  （`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` に `なりきり: 失敗条件` などが
  priority 100 で記載されているのと同じ位置づけ）
- イベント: `ON_STATUS_HIT`（priority 100・デフォルト） — 使用者の特性を対象の特性に
  書き換える（`battle.change_ability` を使用）
- パターン: `なりきり`（`handlers/move_status.py`）とほぼ同一の失敗判定・効果構造を持つ。
  ただし、うつしえは「使用者自身の特性がprotected」「使用者と対象の特性が同じ」という
  追加の失敗条件を持つ点が `なりきり` と異なる（Wiki記載の仕様に基づく）。
- `まもる`／`マジックコート`／`マジックミラー` への耐性は、`unprotectable` /
  `unreflectable` の技フラグで表現する（`unreflectable` は本レビューで新設したフラグ。
  `Move.is_reflectable` プロパティ経由で `マジックコート_reflect` /
  `マジックミラー_reflect` 双方から参照する）。
