# まほうのこな 仕様書

## 基本情報

| 項目 | 値 |
|------|-----|
| タイプ | エスパー |
| 分類 | 変化 |
| PP | 20 |
| 命中率 | 100 |
| 範囲 | 相手1体 |
| 優先度 | 0 |
| 直接攻撃 | × |
| フラグ | 粉技（powder） |

ブリムオンの専用技。

## 効果

相手のタイプを **エスパー単タイプ** に変える（複合タイプであっても単タイプに置き換える）。

## 失敗条件（ON_BEFORE_APPLY_MOVE）

以下の場合、まほうのこなは失敗する：

1. 対象がすでにエスパー単タイプである
   - もりののろい・ハロウィンなどでタイプが追加されており見かけ上「エスパー」を含む複合タイプになっている場合は、
     対象は「エスパー単タイプ」ではないため成功し、エスパー単タイプに置き換えられる（追加タイプは消える）
2. 対象がテラスタルしている
3. 対象がアルセウスまたはシルヴァディである（種族固有の無効化。持ち物の有無・特性の状態に関わらず無効）

## 粉技としての無効化（コア共通ルール）

このハンドラでは判定しない。`flags={"powder"}` を設定することで、
`core/move_executor.py` の共通ルール（`_check_grass_type_powder_immunity`, ON_TRY_MOVE_2 相当）により
以下の場合に自動的に無効化される：

- 対象が **くさタイプ**（もりののろいでくさタイプが追加されている場合を含む。`Pokemon.types` が
  added_types を考慮するため追加で判定は不要）
- 対象が特性 **ぼうじん** を持つ（`ぼうじん_block_powder`）
- 対象が道具 **ぼうじんゴーグル** を持つ（`ぼうじんゴーグル_block_powder_move`）

## 防がれる条件

- みがわり状態（状態変化技はみがわりに防がれる共通ルール `みがわり_immune` が適用される）
- マジックコートで反射される（`unreflectable` フラグを付与しないため既定で反射可能）
- まもるで防がれる（`unprotectable` フラグを付与しないため既定で防御可能）

## タイプ変化の実装方式

みずびたし（みずタイプへの単タイプ化）と同様に、専用の揮発性状態「まほうのこな」を付与し、
`Pokemon.volatile_override_type` を使って現在のタイプをエスパー単タイプに上書きする。

- `ON_VOLATILE_START`: `volatile_override_type = "エスパー"` に設定し、`added_types` をクリアする
  （もりののろい・ハロウィンで追加されていたタイプを消す）
- `ON_VOLATILE_END`: `volatile_override_type = None` に戻す

状態異常ではなく揮発性状態のため、交代すると元のタイプに戻る（揮発性状態がリセットされるため）。

## イベントフロー

```
ON_BEFORE_APPLY_MOVE (priority=100, MoveHandler デフォルト)
  → まほうのこな_can_apply: 既にエスパー単タイプ／テラスタル中／アルセウス・シルヴァディなら失敗 (stop_event=True)

ON_TRY_MOVE_2 (コア共通ルール)
  → 粉技無効化判定（くさタイプ／ぼうじん／ぼうじんゴーグル）

ON_STATUS_HIT (priority=100, MoveHandler デフォルト)
  → まほうのこな_apply: volatile_manager.apply(defender, "まほうのこな")
```

揮発性状態「まほうのこな」のタイプ変更処理は `data/volatile.py` に実装：

```
ON_VOLATILE_START → まほうのこな_set_type
ON_VOLATILE_END   → まほうのこな_clear_type
```

## ポケモンチャンピオンズ向けの簡略化

- フォルムチェンジによるタイプ復帰は、フォルムチェンジ自体の実装（各フォルムチェンジ技・特性側）で
  `data.types` を参照し直すため、まほうのこな側での特別対応は不要
- ダイマックス／キョダイマックスは対象範囲外のため考慮しない
