# うらみ 仕様書

## 基本情報

| 項目 | 値 |
|------|-----|
| タイプ | ゴースト |
| 分類 | 変化 |
| PP | 12（`docs/champions/move_list.txt` 準拠。Gen9 本家は10） |
| 命中率 | 100 |
| 範囲 | 相手1体 |
| 優先度 | 0 |
| 直接攻撃 | × |
| まもる | 防がれる |

## 効果

相手が最後にPPを消費した技（`ctx.defender.last_pp_consumed_move`）の残りPPを4減らす。
ねごとで実行されたサブ技はPPを消費しないため対象にならず、ねごと自身のPPが減る。

## 失敗条件

以下のいずれかに該当する場合、うらみは失敗する。

- 相手が場に出てから一度もPPを消費する行動をしていない（`last_pp_consumed_move` が `None`）
- 相手が最後にPPを消費した技が **わるあがき**（わるあがきのPPは概念上無限であり、
  対象にならない）
- 相手が最後にPPを消費した技のPPがすでに0である（別の効果で0まで減らされていた場合など）

上記のいずれにも該当しない場合は、対象技のPPを4減らして成功する
（減算後にPPがマイナスになる場合は0で下限クリップされる。
`Move.modify_pp` が実装済みのクリップ処理をそのまま利用する）。

### 対象外として設計上考慮しないケース（Champions シングル対象外・本エンジン非対象）

- ものまね/スケッチ、レベルアップにより最後に使用した技を忘れているポケモン
  （本エンジンは戦闘中の「技を忘れる」処理を実装していないため対象外）
- マックスレイドバトル/テラレイドバトルのレイドボスポケモン、スターモービル
  （ダブルバトル用ギミックであり本プロジェクトの対象範囲外）

### `last_pp_consumed_move` の更新タイミングによる自然な整合

`last_pp_consumed_move` は `core/move_executor.py` の `_consume_pp` で実際の
PP消費量が1以上のときにのみ更新される。相手の直前の行動が状態異常（ねむり等）・
状態変化（ひるみ等）・特性なまけ等でPPを消費せずに失敗していた場合
（`ON_TRY_ACTION` が偽を返した場合）や、ねごとのサブ技のように消費量が0の場合は
更新されない。そのため「うらみは最後にPPを消費した技を対象にする」という仕様
（第二世代・第五世代以降の挙動）は、本エンジンでは `last_pp_consumed_move` の
更新条件自体によって自然に満たされており、うらみ側で追加のケア
（前回のPP消費技を遡って探す処理等）は不要。

## 判定

- 急所: なし（変化技）
- 命中判定: あり（100%、回避率等の補正を受ける）
- 追加効果: なし
- まもる: 防がれる（`Move.is_blocked_by_protect` が対象 `foe` の変化技を自動的に対象とするため専用フラグ不要）
- マジックコート: 第五世代以降は跳ね返される（`マジックコート_reflect` が
  「対象が相手・変化技であるすべての技」を汎用的に跳ね返すため専用対応は不要）
- みがわり: **貫通する**（みがわり状態の相手にも効果がある）。
  `Move.flags` に `bypass_substitute` を付与する必要がある
- ダイマックスわざに対して使用した場合はそのベースとなった技のPPを減らす
  （本エンジンはダイマックス機構自体が対象範囲外のため考慮不要）
