# スレッドトラップ（Thread Trap）仕様書

参照: docs/wiki/moves/スレッドトラップ.html（ポケモンWiki、本worktreeには未生成のため未参照）、
docs/spec/volatiles/スレッドトラップ.md（付与される揮発性状態）、
docs/champions/move_list.txt

## 基本情報

* タイプ: むし
* 分類: 変化
* 威力: なし
* 命中: なし（必中）
* PP: 10（`docs/champions/move_list.txt` にはスレッドトラップ自体の単独エントリが存在しないため
  championsのPP値を確認できない。第九世代本家の値である10をそのまま採用する。「なみだめ」等、
  まもる系技を無効化する旨の説明文中に「スレッドトラップ」という技名の言及はあるが、
  スレッドトラップ自身の独立した行としては掲載されていない。ワナイダー専用技であり、
  champions図鑑にワナイダーが含まれない可能性がある（`docs/spec/moves/はいすいのじん.md` と
  同様の扱い）。champions基準のPP統一ルール（`docs/champions/changes_from_sv.md`
  「すべての技のPPが8・12・16・20の4段階に統一」）は移行元データが存在する技にのみ適用可能で
  あるため、本技は対象外として本家PPを維持する）
* 対象: 自分
* 優先度: +4
* 接触: なし

## 効果

* 使用したターンの間、自分に「スレッドトラップ」揮発性状態を付与する。そのターンに受ける
  **攻撃技のみ**を無効化する（**変化技は防げない**点でまもると異なる）。防いだ攻撃が
  直接攻撃技だった場合、その技を使用した相手の素早さ（S）ランクを1段階下げる。
  付与条件・防御対象・ランクダウンの相互作用などの詳細は docs/spec/volatiles/スレッドトラップ.md 参照。
* 連続で使う度に成功率が1/3になっていく（最初は必ず成功、以降は1/3, 1/9, ... と下がり、下限は1/3^6=1/729）。
  失敗するとカウントはリセットされる。
  * 本プロジェクトでは、まもる系技共通の連続使用失敗判定（`まもる系_連続使用失敗チェック`）を採用し、
    「直前ターンに守る系技（`protect` フラグを持つ技）を成功させていた場合、今ターンは必ず失敗する」
    という簡略化モデルで実装する（確率的に成功率が1/3ずつ低下していく詳細モデルは
    未実装・スコープ外とする。詳細は docs/spec/volatiles/まもる.md 参照）。
  * まもる・みきり・こらえる・キングシールド・ニードルガード・トーチカ・スレッドトラップ・
    かえんのまもり等、`protect` フラグを持つ技同士であれば異なる技を挟んでも連続使用扱いになる。

## 判定フラグ

* 急所: なし（攻撃技ではないため対象外）
* 命中判定: なし（必中）
* 追加効果: なし
* まもる: 対象外（自分を対象とする変化技のため通常は問題にならない）
* おうじゃのしるし等: 発動しない
* マジックコート・マジックミラー: 反射不可（自分対象技のため対象外。`target="self"` により
  `Move.is_reflectable` が自動的に `False` になるため個別の `unreflectable` フラグは不要）
* よこどり: 対象外（自分対象技のため通常はよこどりされない）
* まねっこ: コピー不可（`protect` フラグを持つ守る系技は一律コピー対象から除外される。
  `まねっこ_can_use` が `protect` フラグを判定するため、個別に `non_copycat` フラグを
  付与する必要はない）
* ゆびをふる・ねこのて・さいはい: 選ばれない（本プロジェクトではゆびをふる・ねこのて・さいはい
  自体が未実装のため対応不要）

## 優先度（docs/spec/turn.md 参照）

* `Event.ON_TRY_MOVE_2`（`attacker:self`）: まもる系連続使用失敗チェック。turn.md では
  `Event.ON_TRY_MOVE_1` の30番「まもる系: 連続使用により失敗」に該当する記載があるが、既存実装
  （まもる・みきり・こらえる・キングシールド等）はすべて `Event.ON_TRY_MOVE_2` のデフォルト
  priority（100）で実装されているため、同一パターンに合わせる
  （docs/spec/moves/まもる.md・docs/spec/moves/キングシールド.md も参照）。
* `Event.ON_STATUS_HIT`（`attacker:self`）: 自分に「スレッドトラップ」揮発状態を付与する。turn.md に
  個別 priority 行の記載がないため、既存の変化技ハンドラと同様にデフォルト（100）を採用する。
* スレッドトラップ volatile 自体（`Event.ON_TRY_MOVE_1` priority=100 で攻撃技を無効化、
  `Event.ON_TURN_END` で解除）は docs/spec/volatiles/スレッドトラップ.md を参照。

## 既知の未実装範囲（備考）

* 連続使用時の確率的な成功率低下（1/3ずつ低下、下限1/729）は未実装。本プロジェクトはまもる系共通の
  簡略化モデル（連続使用で必ず失敗）を採用する（docs/spec/volatiles/まもる.md 参照）。
* `executed_move` は「直前に実際に実行された技」を保持するフィールドだが、まひ・ねむり・ひるみ等で
  行動自体ができなかったターンを挟んでも値がリセットされない（まもる系共通の既知の限界。
  詳細は docs/spec/moves/まもる.md 参照）。
* さいはい・おどりこ等ダブルバトル専用技との相互作用は、本プロジェクトの対象範囲（シングルバトル）外のため未実装。
* ダイマックスわざ・Zワザは第9世代（champions基準）に存在しないため対象外。
* ふかしのこぶし・かんつうドリルによる守り貫通（素早さランクも下がらない）は
  docs/spec/volatiles/スレッドトラップ.md を参照。
