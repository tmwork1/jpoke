## 効果

相手を **いちゃもん** 状態（揮発性状態）にする。
いちゃもん状態のポケモンは、直前に選択した技（いちゃもん付与時点の
`ctx.defender.selected_move`）をコマンド選択時に選択できなくなる。
ねごとでサブ技を実行した場合も、選択した技はねごと自身である。

- 対象が場に出てから技を選択していない場合（`selected_move` が `None`）でも
  いちゃもんの付与自体は成功する。この場合は禁止される技が存在しないため、
  実質的に行動制限は発生しない。
- ダイマックス・キョダイユウゲキ関連の相互作用は本プロジェクトの対象範囲外
  （ポケモンチャンピオンズは第9世代基準でダイマックス機構自体が存在しない）。

揮発性状態「いちゃもん」自体の詳細仕様（コマンド制限・解除条件など）は
`docs/spec/volatiles/いちゃもん.md` を参照。

## 判定

- 急所: なし（変化技）
- 命中判定: あり（100%、回避率等の補正を受ける）
- 追加効果: なし（本体効果が100%発動するため「追加効果」扱いではない）
- まもる: 防がれる（`Move.is_blocked_by_protect` は対象が `foe` の変化技を自動的に対象とするため、専用フラグは不要）
- マジックコート: 跳ね返される（`マジックコート_reflect` が「対象が相手・変化技」であるすべての技を汎用的に跳ね返すため、専用対応は不要）

## イベントフロー

```
ON_STATUS_HIT (priority=100, MoveHandler デフォルト)
  → いちゃもん_apply
    → move_name = ctx.defender.selected_move.name（未選択なら ""）
    → volatile_manager.apply(defender, "いちゃもん", move_name=move_name)
```

`いちゃもん_apply` で `move_name` を明示的に渡す点が実装上の要点。
`ctx.defender.selected_move` から直前に選択された技名を取得する
（「最後にPPを消費した技」を参照する `かなしばり` とは参照先が異なる）。
`かなしばり` と異なり、対象が未選択（`selected_move is None`）でも
失敗させない（`can_apply` のような失敗判定ハンドラは不要）。

`selected_move`（`Pokemon.selected_move`）はPP消費と同時点
（`MoveExecutor.run_move` 内、タイプ相性判定・`ON_TRY_MOVE_2`・くさタイプ粉技無効
判定より前）で確定する。これにより、タイプ相性等で技自体が不発になった場合でも
「その技を選択して使用した」こと自体は成立しているため、`selected_move` は
正しくその技を指す（PPは既に消費済みのため）。全体ターン不発（まひ・こおり・
ひるみ等で `ON_TRY_ACTION` が失敗した場合）はPPが消費されないため
`selected_move` も更新されない。
