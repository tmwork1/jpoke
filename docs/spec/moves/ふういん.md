## 効果

自分を **ふういん** 状態（揮発性状態）にする。
ふういん状態のポケモンが覚えている技と同じ技を、相手のポケモンは使用できなくなる。

- 使用者自身がふういんを覚えている場合、ふういん自体も封印対象になる
  （ただし封印は相手側にのみ効果があるため、使用者自身の行動には影響しない）。
- 第五世代以降は相手の技構成によらず必ず成功する
  （第四世代までは相手が使用者と共通の技を1つも覚えていない場合失敗するが、対象外の世代仕様）。

揮発性状態「ふういん」自体の詳細仕様（ブロック判定・解除条件など）は
`docs/spec/volatiles/ふういん.md` を参照。

## 失敗条件

- すでにふういん状態の場合は失敗する
  （`volatile_manager.apply` が既存の揮発性状態を検知して `False` を返す）。

## 判定

- 急所: なし（変化技）
- 命中判定: なし（必中。`accuracy` を設定しない）
- 追加効果: なし
- まもる: 防がれない（対象が `self` のため `is_blocked_by_protect` が自動的に False になる）
- マジックコート: 跳ね返されない（対象が `self` のため `is_reflectable` が自動的に False になる）
- よこどり: 未実装のため対象外
- オウムがえし: 未実装のため対象外

## イベントフロー

```
ON_STATUS_HIT (priority=100, MoveHandler デフォルト)
  → ふういん_apply
    → apply_volatile_to_attacker(battle, ctx, value, volatile="ふういん")
      → volatile_manager.apply(attacker, "ふういん", source=attacker)
      → 既にふういん状態の場合は自動的に失敗する
```

## 本実装での対象外（スコープ外）

- 第四世代までの「相手に共通技が無い場合失敗する」仕様: 対象外世代のため実装しない。
- テラレイドバトルでの失敗: Champions シングルバトルには存在しない概念のため対象外。
- プレッシャー特性によるPP消費増加: `プレッシャー_extra_pp`（`handlers/ability.py`）が
  `Event.ON_MODIFY_PP_CONSUMED` で `defender:self`（=攻撃者の相手が該当）に汎用対応済み。
  対象が `self` の技にも適用されるため、ふういん専用の対応は不要。
