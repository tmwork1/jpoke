## 効果の詳細

- 相手の『こうげき』ランクを2段階上げる。
- 相手の『ぼうぎょ』ランクを2段階下げる。
- 命中判定を行わない必中技（自分の命中率、相手の回避率に関係なく必ず命中する）。
- 失敗条件: 相手の『こうげき』ランクがすでに+6、かつ『ぼうぎょ』ランクがすでに-6の場合のみ、
  技全体が失敗する（どちらのランク変化も適用されない）。
  - こうげきが+6でもぼうぎょが-6未満であれば、ぼうぎょダウンのみ適用されて技は成功する。
  - ぼうぎょが-6でもこうげきが+6未満であれば、こうげきアップのみ適用されて技は成功する。
- みがわり状態の相手には無効。
- まもるで防がれる。マジックコートで跳ね返される。
- ゆびをふるでは選ばれない（変化技全般の共通仕様）。
- くさタイプの技だが、変化技のためタイプ相性による無効化はない。
- スコヴィランの専用技。

## Pokémon Champions での補足

- コンテスト効果は実装しない。
- PP は Pokémon Champions で 8/12/16/20 の4段階に統一されているため、Gen9本家の15ではなく
  16 を採用する（`docs/champions/changes_from_sv.md`）。

## 実装方針

- イベント: `ON_STATUS_HIT`（priority 100・デフォルト） — こうげき+2、ぼうぎょ-2 を
  1回の `modify_stats` 呼び出しにまとめて適用する。
- `docs/spec/turn.md` に `ON_STATUS_HIT` の明示的な記載はないが、同種の変化技はすべて
  デフォルト priority 100 を使用しており、本技も同パターンを採用する。
- 失敗判定は専用の `ON_BEFORE_APPLY_MOVE` ハンドラを設けず、`modify_stats` に
  `{"atk": 2, "def": -2}` をまとめて渡すことで実現する。両方とも境界（+6/-6）で
  変化不可の場合のみ戻り値が空dict（偽値）になり、フレームワーク側
  （`move_executor._execute_status_hit`）が `ON_STATUS_HIT` の最終結果が偽であることを
  検知して技を失敗とする。片方のみ境界に達している場合は、変化可能な方のみ適用され
  技は成功する。
- accuracy は必中のため `MoveData` に指定しない（省略時は `None` となり、命中判定を
  バイパスする。`あくび`等の必中変化技と同じ実装パターン）。
