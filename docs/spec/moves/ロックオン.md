# ロックオン（Lock-On）仕様書

参照: docs/wiki/moves/ロックオン.html（ポケモンWiki）

## 基本情報

* タイプ: ノーマル
* 分類: 変化
* 威力: -
* 命中: -（必中。第四世代以降。第三世代までは命中率100の判定ありだったが本プロジェクトは対象外）
* PP: 8（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）
* 対象: 相手1体選択
* 優先度: 0
* 接触: なし（直接攻撃ではない）

## 効果

* 使用すると自分が「ロックオン」状態になる（第五世代以降。詳細は
  `docs/spec/volatiles/ロックオン.md` を参照）。
* ロックオン状態の自分が次のターンに使う技は、命中率を無視して相手に必ず命中する。
* すでに自分がロックオン状態の場合、ロックオンの使用は失敗する
  （`VolatileManager.apply` が同名状態の重複付与を拒否するため自然に成立する。
  シングルバトルでは「別の相手を狙って使用した場合」の分岐は問題にならない）。
* 相手がひんし・交代すると、そのタイミングでロックオン状態は解除される。

## 判定フラグ

* 急所: 対象外（変化技のため）
* 命中判定: なし（必中）
* 追加効果: なし
* まもる: 防がれる（○）
* みがわり: 防がれる
* おうじゃのしるし: 対象外（変化技のため）
* マジックコート: 反射不可（×）
* よこどり: 対象外（未実装の技のため本プロジェクトでは判定不要）
* オウムがえし: コピー可能（○）

## 実装上の注意

* 効果の付与対象は**自分（attacker）**であり、技の選択対象（相手）ではない。
  `apply_volatile_to_attacker` を使う（`apply_volatile_to_defender` を使うと誤り）。
* 潜伏貫通（そらをとぶ・あなをほる等の相手にも必中させる効果）は本プロジェクトの
  実装スコープでは未対応（詳細は `docs/spec/volatiles/ロックオン.md` を参照）。

## 備考

* 第九世代ではこころのめ（同効果の技）が削除されたため、ロックオン状態を発生させる
  唯一の技となった。
* Zワザ化した場合、自分の素早さが1段階上がる（本プロジェクトでは対象外）。
