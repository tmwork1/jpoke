## 効果

**特性が「プラス」または「マイナス」の味方全員**の「ぼうぎょ」「とくぼう」ランクを
それぞれ1段階ずつ上げる。本プロジェクトはシングルバトルのみのため、実質的に
「使用者自身の特性がプラス/マイナスの場合、自身のぼうぎょ・とくぼうがそれぞれ
1段階ずつ上がる」技として機能する。

使用者自身の特性がプラス/マイナスでない場合は失敗する。

## 失敗条件

- 使用者の特性が「プラス」「マイナス」以外の場合は失敗する
  - いえき状態やかがくへんかガスにより特性が変化/無効化されている場合、
    `ability.name` の参照結果自体が変わるため、専用の追加ガードは不要
- ぼうぎょ・とくぼうが両方すでにランク+6の場合、実質的に失敗扱いとなる
  （ランク変化マネージャー側で個別に処理されるため、片方のみ+6の場合は
  もう片方のみ上昇して技自体は成功扱いとなる）

## 判定・技の仕様（出典: ポケモンWiki）

- 急所判定: 対象外（変化技）
- 命中判定: 対象外（必中技）
- 追加効果: なし
- **まもる**: 防がれない（味方対象の技のため）
- **マジックコート**: 跳ね返されない（味方対象の技のため）
- **よこどり**: 奪われる（本プロジェクト未実装のため対象外）
- **オウムがえし**: コピーされない（味方対象の技のため）
- 使用者自身が**みがわり状態**であっても効果がある
- 味方が**ちくでん/ひらいしん/でんきエンジン**を持っていても、でんきタイプ技として
  これらの特性に対して発動せず、特性自体も発動しない（味方対象の技のため）
- 必中技だが、姿を隠している（そらをとぶ等ため状態の）味方には当たらない
  （シングルバトルでは使用者自身が対象のため、使用者自身がため状態のまま
  この技を使用すること自体が発生しないので実質的に無関係）

## 対象範囲についての注記

本家では「味方全員」が対象の技だが、本プロジェクトの対象はポケモンチャンピオンズ
シングルバトルのみであり、味方は使用者自身のみのため、機能的には
「使用者自身が対象の自己強化技」として扱われる。

本技は防御側（相手）を一切参照しない自己完結型の効果のため、`MoveData.target` は
`"self"` を指定する。これにより、フレームワーク内部でも技実行時のみがわり判定
（`check_hit_substitute`）や、でんきタイプ吸収系特性（ちくでん・ひらいしん・
でんきエンジン等）の発動判定（`ctx.move.target == "foe"` を条件とする）が、
実際には無関係な**相手側**の状態を参照しないようにする。`target` を既定値の
`"foe"` のままにすると、相手がみがわり状態のときや、相手がちくでん等の特性を
持つときに本技が誤って無効化されるバグとなる。

## イベントフロー

```
ON_BEFORE_APPLY_MOVE (priority=100)
  → 使用者の特性がプラス/マイナスでない場合、失敗させる

ON_STATUS_HIT (priority=100, MoveHandler デフォルト)
  → 使用者のぼうぎょ・とくぼうをそれぞれ1段階上げる
```

## ポケモンチャンピオンズ向けの注記

- Zワザ追加効果（とくぼう1段階上昇）は実装しない（Zワザ非対応のため）
- コンテスト効果は実装しない
- ダブル/トリプルバトル固有の記述（隣接していない味方への効果等）は
  シングルバトル専用の本プロジェクトでは対象外
