# グラススライダー（Grassy Glide）仕様書

参照: docs/wiki/moves/グラススライダー.html（ポケモンWiki）

## 基本情報

* タイプ: くさ
* 分類: 物理
* 威力: 55（第九世代。第八世代では70）
* 命中: 100
* PP: 20
* 対象: 相手1体
* 優先度: +1（自分がグラスフィールド状態の効果を受けている場合） / 0（それ以外）
* 接触: あり（直接攻撃）

## 効果

* 使用者が「グラスフィールド」の効果を受けている（＝グラスフィールドが展開されており、
  かつ使用者が接地している）とき、優先度が+1される。
* 接地判定はひこうタイプ・特性「ふゆう」・ふうせん等、通常の「浮いている」判定
  （`battle.query.is_floating`）に従う。グラスフィールドが展開されていても使用者が
  浮いていれば優先度は上がらない。
* 追加効果なし。

### 優先度上昇の判定タイミングについて

* 本家ゲームでは、行動選択後・実行前にグラスフィールドの展開状況や使用者の接地状態が
  変化した場合、実行時点の状態で優先度が即座に再判定される（例: 行動選択後に
  グラスフィールドが張り替えられた／ふうせんが割られた等）。
* 本プロジェクトの行動順決定は `TurnController.resolve_action_order` でターン開始時に
  一括計算する簡略化されたモデルであり、上記のような実行直前の動的な再判定は行わない
  （既存のターン処理全体の仕様であり、本技固有の実装では対応しない）。

## 判定フラグ

* 急所: 通常判定（急所ランク0）
* 命中判定: あり（命中率100）
* 追加効果: なし
* まもる: 防がれる
* おうじゃのしるし: 発動する（第五世代以降）
* マジックコート: 反射不可（変化技ではないため対象外）
* よこどり: 対象外（変化技ではないため）

## テラスタルとの関係

* テラスタイプが「くさ」と一致する場合、最終威力が60未満なら60に底上げされる
  （`docs/spec/terastal.md` 「威力補正（60底上げ）」を参照）。
* この底上げ補正は本家仕様では「先制技（優先度+1以上）は対象外」だが、グラススライダーは
  グラスフィールドによって優先度が+1になっている場合でも例外的に底上げが適用される。
* 本プロジェクトの実装では、底上げ補正の除外判定 (`damage.py`
  `_can_apply_terastal_power_floor`) が技の**基本優先度**（`MoveData.priority`、
  本技は0）を参照しており、グラスフィールドによる動的な優先度上昇（イベントハンドラ経由）
  を見ないため、結果的に本家仕様の「例外」が自然に再現される。

## 優先度上昇の実装箇所

* 優先度上昇はグラススライダー自身のハンドラではなく、地形「グラスフィールド」側の
  `DomainEvent.ON_MODIFY_MOVE_PRIORITY` ハンドラ（`グラスフィールド_boost_move_priority`,
  `src/jpoke/data/field/terrain.py` / `src/jpoke/handlers/field.py`）として実装する。
* 理由: 優先度判定 (`SpeedCalculator.calc_move_priority`) はターン開始時、
  各ポケモンの技がまだ実行されていない段階で発火する。技（`Move`）自身のハンドラは
  `move_executor` がその技の実行を開始した時点で `register_handlers` される
  （`core/move_executor.py`）ため、ターン開始時点ではまだ登録されておらず、
  技側にハンドラを持たせても発火しない。
  一方、地形のハンドラは地形が展開されている間ずっと登録されているため
  （`FieldManager._activate_field` / `_deactivate_field`）、ターン開始時の
  優先度判定にも正しく反映される。トリックルームの素早さ反転
  （`DomainEvent.ON_CHECK_SPEED_REVERSE`）も同様に地形側のハンドラとして実装されている。

## 既知の制限

* 相手の「サイコフィールド」（先制技無効化）・「ファストガード」・「じょおうのいげん」等、
  技の優先度を参照して先制技を無効化・妨害する効果は、いずれも `ctx.move.priority`
  （技の基本優先度、本技は常に0）を参照して判定している
  （`src/jpoke/handlers/field.py` `サイコフィールド_block_priority_move`,
  `src/jpoke/handlers/volatile.py` `ファストガード_protect` 等）。
  そのため、グラスフィールドで実際に優先度+1になっているターンであっても、
  これらの効果によってグラススライダーが無効化・妨害されることはない
  （本家仕様と異なる）。
  これは「いたずらごころ」「はやてのつばさ」等、既存の動的優先度変化アビリティ全般に
  共通する既存のアーキテクチャ上の制限であり、本レビューでは対応しない。
