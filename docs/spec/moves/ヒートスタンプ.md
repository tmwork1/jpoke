# ヒートスタンプ（Heat Crash）仕様書

参照: docs/wiki/moves/ヒートスタンプ.html（ポケモンWiki）
参照: docs/spec/moves/_variable_power.md（変動威力技共通仕様、第8節「体重依存威力技」に体重"差"依存の欄がないため本ファイルで個別に定義）

## 基本情報

* タイプ: ほのお
* 分類: 物理
* 威力: 可変（40〜120。下記参照）
* 命中: 100
* PP: 12（`docs/champions/move_list.txt` 準拠。Gen9本家は10だが、チャンピオンズでは全技PP 8/12/16/20 の4段階に統一されているため12）
* 対象: 相手1体
* 優先度: 0
* 接触: あり（直接攻撃）

## 効果

* 自分（使用者）の体重と相手（対象）の体重の比率によって威力が変わる。
* 比率 = 自分の体重 ÷ 相手の体重 として、以下の表で威力を決定する。

  | 相手の体重 | 比率 | 威力 |
  |---|---|---|
  | 自分の1/5以下 | 5以上 | 120 |
  | 自分の1/4以下（1/5超） | 4以上5未満 | 100 |
  | 自分の1/3以下（1/4超） | 3以上4未満 | 80 |
  | 自分の1/2以下（1/3超） | 2以上3未満 | 60 |
  | それより重い（1/2超） | 2未満 | 40 |

* ヘビーボンバーと全く同じ威力計算式を使用する（自分と対象の体重比率で威力が変化する技はこの2種のみ）。
* 相手が「ちいさくなる」状態の場合、上記の体重比率計算を行わず、`minimize` ラベルの共通処理により
  必中・威力2倍（現在の基礎威力に対して2倍）が適用される（第6世代以降の仕様）。
  * この場合、体重比率で決定するはずだった威力ではなく、`MoveData.power`（プレースホルダー値=1）に
    対して2倍の補正がかかる実装になっている点に注意（`ちいさくなる_boost_power` ハンドラが担当）。

## 判定フラグ

* 急所: 通常通り発生する。
* 命中判定: あり（命中率100。ただし相手が「ちいさくなる」状態の場合は必中）。
* 追加効果: なし。
* まもる: 防がれる。
* おうじゃのしるし: ひるみ効果あり（通常の直接攻撃技として扱う）。
* マジックコート: 反射不可（変化技ではないため対象外）。
* よこどり: 対象外（変化技ではないため）。
* オウムがえし: コピー可能（本プロジェクトはオウムがえし自体が未実装のため対応不要）。
* ダイマックス相手への使用: 本プロジェクトはダイマックス非対応のため対象外。

## リーサル計算

* 威力は自分・相手双方の体重（静的な値。バトル中に変動しない）にのみ依存し、HPの分布（`StateDist`）には
  影響しないため、専用のリーサルハンドラは不要（`docs/progress/move.md` のリーサル実装/リーサルテスト列はn/a）。
