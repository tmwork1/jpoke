# クロロブラスト（Chloroblast）仕様書

参照: docs/wiki/moves/クロロブラスト.html（ポケモンWiki）
参照: docs/spec/moves/てっていこうせん.md, ビックリヘッド（未作成）（同系統の「自己HP半分消費」技だが、
本技のみ特性・タイミングの仕様が異なる点に注意）
参照: src/jpoke/data/ps-champ-ja/moves.json, docs/champions/changes_from_sv.md（PP統一ルール）

## 基本情報

* タイプ: くさ
* 分類: 特殊
* 威力: 150
* 命中: 95
* PP: 5（Gen9本家）→ **8**（チャンピオンズ換算。下記参照）
* 対象: 相手1体
* 優先度: 0
* 接触: なし

### PPについて

`src/jpoke/data/ps-champ-ja/moves.json` に `クロロブラスト` の行が存在しない（該当データなし）。
ただし `changes_from_sv.md` により全技PPは8/12/16/20の4段階に統一されており、
Gen9本家PP5の技は例外なくPP8になっている（`move_list.txt` 内で実例確認済み:
オーバーヒート 5→8、ハイドロカノン 5→8、はかいこうせん 5→8、りゅうせいぐん 5→8、
同系統の `てっていこうせん` 5→8）。この規則から類推し、本技もPP8として扱う。

## 効果

* 命中すると、通常のダメージ計算（攻撃・防御・威力・急所・タイプ相性など）で相手にダメージを与えた後、
  使用者は自分の最大HPの1/2（**小数点以下切り上げ**、最低1）を反動ダメージとして受ける。
* 追加効果なし。

## 反動ダメージの発生条件（重要・てっていこうせん/ビックリヘッドとの違い）

一次情報「技の仕様」節に明記されている通り、本技の反動は **技が成功した場合のみ** 発生する。
これは「使用前に無条件でHPを消費する」`てっていこうせん`・`ビックリヘッド` とは根本的に異なる。

* まもる状態や特性（そうしょく等）によって技が無効化された場合、反動ダメージを受けない。
* 命中しなかった場合、反動ダメージを受けない。
* 姿を隠している相手（あなをほる中など）に当たらなかった場合、反動ダメージを受けない。
* 上記いずれでもなく技が成功した場合は、たとえ与えたダメージが0であっても
  （例: 対象がHP1で`こらえる`を使用したとき、`ばけのかわ`が発動したとき）反動ダメージを受ける。

## 特性との相互作用（重要・てっていこうせん/ビックリヘッドとの違い）

* 使用者の特性が **いしあたま** のときも、**マジックガード** のときも、反動ダメージを受けない
  （一次情報に両方明記。てっていこうせん・ビックリヘッドはマジックガードのみで防げ、
  いしあたまでは防げない仕様であり、本技はその逆＝両方で防げる）。
* この「いしあたま・マジックガード双方で防げる」は既存の `HPChangeReason = "recoil"`
  （通常の反動技=フレアドライブ等が使う理由）にそのまま該当する。
  - `いしあたま_ignore_recoil` は `reason == "recoil"` を見て無効化する。
  - `マジックガード_ignore_damage` は `"recoil"` を除外リストに含まないため防がれる。
  - 専用の `HPChangeReason` を新設する必要はない。
* 使用者の特性が **すてみ** であっても、技の威力は上がらない
  （すてみは `move.has_flag("recoil")` を見て判定するため、本技にその flag を付与しないことで
  対応する。反動ダメージの理由=`HPChangeReason` と、すてみが参照する `flags={"recoil"}` は
  別概念であることに注意）。

## 反動ダメージのタイミング

* 相手へのダメージ計算・HP減少処理より **後** に、使用者自身のHPが減る
  （てっていこうせんとは逆の順序。てっていこうせんは攻撃前にHPを消費する）。
* `Event.ON_HIT` は「命中時（実ダメージが0でも発火する）」イベントであり、まもる・命中失敗・
  タイプ無効化などで技が不成立の場合はそもそも発火しない。これにより「反動は技成功時のみ・
  実ダメージ0でも発生する」という一次情報の仕様を両方満たせる
  （`いのちのたま`・`かいがらのすず`・`だっしゅつボタン`・`レッドカード` が同じ理由で
  `Event.ON_DAMAGE_HIT`（実ダメージ<=0で発火しない）ではなく `Event.ON_HIT` を採用している
  既存パターンと同一）。

## HP0でのひんし・全滅判定

* 反動によりHPが0になるときでも技は発動できる（技自体は反動より前に成立している）。
* この技で両者のHPが0になった場合、**対象（相手）が先に倒れてから使用者が倒れる**
  （てっていこうせんとは逆。てっていこうせんは反動を先に処理するため使用者が先に倒れる）。
  お互いのポケモンが全滅した場合、対人戦では技の使用者側の勝ちとなる
  （一次情報。相手へのダメージ適用 → 反動ダメージ適用、の順で処理することで再現できる）。
* きあいのタスキ／きあいのハチマキ、特性がんじょうを持っていても、それらは反動ダメージには
  介入しない（通常の被ダメージ処理＝相手からの攻撃時のみに介入するため、既存実装のままで対応済み）。

## 爆発技・しめりけとの関係

* 爆発する技には含まれない（`flags={"explosion"}` を付与しない）。特性しめりけのポケモンが
  場にいても成功する。

## うのミサイル・レンブのみとの順序（既知の限界・対応不要）

* 一次情報: 「相手のうのミサイルや、レンブのみが発動した後に反動ダメージを受ける。相手の
  レッドカード、自身のいのちのたまの反動、かいがらのすずの回復効果より前に反動ダメージを受ける」。
* 本プロジェクトの `うのミサイル`・`レンブのみ` は `Event.ON_DAMAGE_HIT`（実ダメージ>0でのみ発火）
  で実装されており、`Event.ON_HIT`（本技の反動が発火するイベント。ON_DAMAGE_HIT より必ず先に
  発火する）より後に発火する。つまり本技の反動は現状の実装では両アイテム/特性より **先に**
  発火してしまい、一次情報の順序と逆になる。
* これは `いのちのたま`・`かいがらのすず`・`だっしゅつボタン`・`レッドカード` が「実ダメージ0でも
  発火する」ことを優先して `Event.ON_HIT` を採用した際に共通して生じている既存の設計上の制約であり
  （`src/jpoke/data/item.py` の該当コメント参照）、本技固有の問題ではない。反動技全般に関わる
  アーキテクチャ変更が必要なため、本レビューでは対応しない（スコープ外。てっていこうせん.md の
  「ダメおしとの相互作用」と同種の既知の限界として扱う）。
* レッドカード・いのちのたま・かいがらのすずとの順序（本技の反動が先）は、`Event.ON_HIT` の
  priority を `docs/spec/turn.md` の「反動技による反動ダメージ = 100」に合わせることで満たされる
  （レッドカード150・いのちのたま160・かいがらのすず160はいずれもこれより後）。
