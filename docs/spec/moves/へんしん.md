# へんしん（Transform）仕様書

参照:
* docs/wiki/moves/へんしん.html（ポケモンWiki）
* docs/spec/abilities/かわりもの.md（同一のコピー処理を共有する特性）

## 基本情報

* タイプ: ノーマル
* 分類: 変化
* 威力: -
* 命中: - （必中。命中ランク補正・回避率補正の影響を受けない）
* PP: 12（Champions基準。`src/jpoke/data/ps-champ-ja/moves.json` 準拠）
* 対象: 相手1体
* 優先度: 0
* 直接攻撃: なし
* フラグ:
  * non_encore（アンコールで固定できない）
  * non_copycat（まねっこで選ばれない）
  * unprotectable（まもる・みきり・トーチカで防がれない）
  * unreflectable（マジックコート・マジックミラーで跳ね返されない）

## 効果（第五世代以降 / 本プロジェクトが対象とするルール基準）

相手のポケモンに変身し、以下をコピーする。

### コピーするもの
* 見た目・種族相当の姿（表示上のみ。専用アイテムの効果までは得られない）
* タイプ
* 特性
* 覚えている技（最大4つ、技の種類そのもの）
  * 変身後の技のPPは一律5になる。ただし元の技の最大PPが1の場合のみ1になる
    （Champions基準ではPP1の技は存在しないため、実質的には常に5になる）
* HP以外の実数値ステータス（こうげき・ぼうぎょ・とくこう・とくぼう・すばやさ）
* 対象（コピー元）の能力ランク補正
* 性別・たかさ・おもさ

### コピーしないもの
* 種族そのもの（専用アイテムの効果は得られない。固有ポケモン専用技は、変身先がそのポケモンで
  あれば使用可能）
* HP（現在値・最大値とも）
* レベル
* 性格・個体値
* なつき度・なかよし度
* アイテム
* 状態異常
* ほとんどの状態変化（へんしん使用者自身の能力ランク補正は変身時に消滅し、対象のランク補正で
  上書きされる）
* 技の残りPP/最大PP（上記の一律PP5化を除く）

変身中は「へんしん」状態（揮発状態）となり、この状態を持つポケモンが場を離れる
（交代・ひんし等）と全て元に戻る。

## 成功条件・失敗条件（第五世代以降 / 本プロジェクト基準）

* 自身が既にへんしん状態である場合は失敗する。
* 対象がみがわり状態である場合は失敗する（第六世代以降は、自身が特性すりぬけであっても失敗する）。
* 対象が特性イリュージョンの効果で他のポケモンに化けている場合は失敗する
  （化けていない本来の姿であれば成功する）。
* 対象がへんしん状態である場合は失敗する。
* 対象がスターモービルである場合は失敗する。
* 自身または対象がテラスタルしているとき、オーガポン/テラパゴスに対しては失敗する
  （お互いテラスタルしていなければオーガポンに対しても成功する）。
* 自身がステラタイプにテラスタルしているときは失敗する。
* 必中技だが、対象があなをほる/そらをとぶ/ダイビング/シャドーダイブ状態で潜伏している場合は
  当たらない（ロックオン状態や特性ノーガードの効果を受ければ成功する）。

## 技の仕様

* まもる状態やマジックコート状態の相手を対象にしても失敗しない（まもるは貫通する。マジックコートは
  対象が相手のため技自体が反射されうるが、変化技であってもへんしんは反射されない特殊な技として
  扱われる）。
* こだわり系アイテムを持った状態でこの技を使用したとき、変身先もへんしんを技として覚えていれば
  へんしんでこだわる（変身先がへんしんを覚えていなければ技をこだわったことにはならない）。
* 成功したときでも、かなしばり状態は解除されず継続する。
* この技はねこのて/ゆびをふるで選ばれない。
* この技はものまね/まねっこでコピーできない。
* この技はオウムがえしでコピーできない。
* この技はアンコールの対象にできない（non_encoreフラグ）。

## 判定フラグ

* 急所: 対象外（変化技のため）
* 命中判定: 必中（上記の潜伏中相手への失敗を除く）
* 追加効果: なし
* まもる・みきり・トーチカ: 防がれない
* マジックコート: 反射されない
* よこどり: 未実装の技のため対象外
* オウムがえし: コピー不可
* みがわり: 対象がみがわり状態の場合はそもそも失敗条件に該当し発動しない（貫通ではなく不発）

## 実装状況

本技は「相手の種族・実数値ステータス・特性・技・ランク補正を丸ごとコピーし、退場時に完全復元する」
という大規模な変身機構を要する。同機構は特性かわりもの（メタモン専用の隠れ特性）とも共有される
基盤であり、現在の `Pokemon` モデルには種族・実数値ステータス・技リスト・特性を一時的に上書きし、
交代/瀕死時に復元する仕組みが存在しない。

本プロジェクトでは複雑度に見合わないため実装を見送り、`handlers={}` のまま実装保留とする
（前例: スケッチ・ゆびをふる）。詳細は `docs/plan/moves/へんしん.md` を参照。
