# しっぽきり（Shed Tail）仕様書

参照: docs/wiki/moves/しっぽきり.html（ポケモンWiki）、docs/spec/moves/交代技.md、docs/spec/volatiles/みがわり.md

## 基本情報

* タイプ: ノーマル
* 分類: 変化
* 威力: -
* 命中: -（命中判定なし）
* PP: 12（`docs/champions/move_list.txt` 基準。本家SVでは10だが、チャンピオンズは全技PPを8/12/16/20に統一しているため12）
* 対象: 自分
* 優先度: 0
* 接触: なし（直接攻撃ではない）

## 効果

* 使用者の**最大HPの1/2（小数点以下切り上げ）** を消費して「みがわり」を生成する。
* みがわりのHP自体は通常の `みがわり` と同じく **使用者の最大HPの1/4（小数点以下切り捨て）**。
* みがわり状態を引き継いだまま、控えのポケモンに交代する（バトンタッチと異なりランク変化や
  みがわり以外の状態変化は引き継がない）。
* 成功すると「〈使用者〉は しっぽを 切って みがわりにした!」というメッセージが出る。

## 失敗条件

以下のいずれかに該当する場合は失敗し、HPは減少しない。

1. 使用者がすでにみがわり状態のとき
   （「しかし 〈使用者〉の 身代わりは すでに でていた」というメッセージが出て交代もしない）。
2. HPを消費した結果、使用者のHPが0以下になってしまうとき
   （= 現在HPが消費コスト `ceil(max_hp/2)` 以下のとき）。
   通常の `みがわり`（消費コスト `floor(max_hp/4)` 以下で失敗）と同様、
   「自分の残りHPを消費してHPが0になる場合は使用不可」という一般則に従う。
3. 交代できる控えのポケモンがいないとき。このとき使用者はみがわり状態になることもできない。

## 判定フラグ

* 急所: 対象外（ダメージを与えない）
* 命中判定: なし
* 追加効果: なし
* まもる: 無効化されない（自分対象の変化技のためまもる判定自体が発生しない）
* おうじゃのしるし: 対象外（第五世代以降、この技には影響しない）
* マジックコート: 反射不可

## 技の仕様（詳細）

* 成功した際、HP減少により残りHPの割合で発動するきのみ（オボンのみ等）の発動圏に入った場合、
  きのみを消費してから交代する。
* 使用者の特性が `さいせいりょく` である場合、成功して交代できた後に特性の効果でHPが回復する。
* ランク補正や、みがわり状態以外の状態変化（やどりぎのタネ等）は交代先に引き継がない。

## 交代処理との関係

* 交代処理の共通仕様は `docs/spec/moves/交代技.md` を参照。
  ありじごく・かげふみ・じりょく・ねをはる・くろいまなざし／クモのす等の交代制限効果を無視して交代できる。
* 交代先の選択は使用者側プレイヤーが行う。
