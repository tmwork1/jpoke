# さきおくり（Quash）仕様書

参照: docs/wiki/moves/さきおくり.html（ポケモンWiki）
参照: src/jpoke/data/ps-champ-ja/moves.json L312

## 基本情報

* タイプ: あく
* 分類: 変化
* 威力: -
* 命中: 100（必中ではない。命中判定・ランク補正の対象）
* PP: 16
* 対象: 相手1体選択
* 優先度: 0
* 接触: なし（直接攻撃×） / まもる: 効果あり（さきおくりが防がれる） / マジックコート: 反射不可 /
  よこどり: 不可 / オウムがえし: コピー可 / おうじゃのしるし: 第五世代以降は無効

## 効果（本来のダブル/トリプルバトルでの効果）

* 対象となったポケモンの行動順を、そのターンの最後にする。
* 第七世代以前は、ターン開始時に決定済みの行動順を並び替えて対象を最後尾にする方式だった
  （同一ターンに複数体がさきおくりを受けた場合、後からさきおくりを受けたポケモンほど
  さらに後ろに並ぶ）。
* 第八世代以降は、対象を「さきおくり状態」（volatile）にする方式に変更された。
  * さきおくり状態は優先度よりも優先して適用されるため、優先度で上回っている相手よりも
    行動順が遅くなる。
  * 複数体がさきおくり状態になっている場合、それらの中での行動順は通常どおり優先度・すばやさ等で
    決定される。
  * さきおくり状態はターン終了時に解除される（さいはいで先に行動しても解除されない）。
* 失敗条件:
  * 対象がそのターンにすでに行動済みの場合は失敗する。
* 無効化:
  * みがわり状態の相手には無効。
  * 特性「おうごんのからだ」を持つ相手には無効。
* 行動順の再修正:
  * 「おさきにどうぞ」「コンビネーション技」「トラップシェル」「りんしょう」等、さらに行動順を
    変動させる効果がある場合、さきおくりを受けたポケモンであってもそれらの効果が優先され、
    行動順が早くなることがある。
  * 行動順が変化しても、対象が選択した技の優先度の情報自体は失われない
    （例: さきおくりを受けても、サイコフィールド／ビビッドボディ／じょおうのいげんによる
    優先度無効化の判定や、あくタイプの相手にいたずらごころの効果を無効化する判定〈第七世代以降〉は
    通常どおり働く）。
* Zワザによる追加効果: 自分のすばやさランクが1段階上がる（本プロジェクトの対象外）。
* さきおくり自体の優先度は0のため、素早く行動できるポケモン（あるいは特性「いたずらごころ」等で
  優先度を上げられるポケモン）でなければ、対象が行動する前に使用できず使いにくい。
  この技が実際に意味を持つのは、対象を「使用者だけでなく味方も含めた全ての行動者より後」に
  押し下げられるダブル/トリプルバトルであり、公式解説でも「ダブルバトル以上で効果を発揮する技」と
  明記されている（1対1の対戦では対象は元々次に行動する唯一の相手であり、押し下げても
  行動順は変化しない）。

## シングルバトルでの扱い（本プロジェクトのスコープ）

* 本プロジェクトの対象はポケモンチャンピオンズのシングルバトルのみであり、`Battle` は常に
  各プレイヤー1体ずつの場（`battle.actives` は常に2体以下）しか扱わない。
* シングルバトルでは、行動側とその対象（相手1体）しか存在しないため、「対象の行動順を最後にする」
  という本技の核心効果は意味を持たない。対象がすでに行動済みなら本技は失敗するだけであり、
  対象が未行動ならそもそも次に行動するのは対象自身であり、さきおくりを使っても使わなくても
  行動順は変化しない。
* この「常に空振りの分岐にしかならない」という性質は、`docs/spec/turn.md`（`Event.ON_APPLY_MOVE`
  priority 100「さきおくり等: すでに行動済み」）に記載された失敗条件チェックを実装したとしても、
  シングルバトル上で観測可能な効果を生まない。逆効果の類似技「おさきにどうぞ」、および
  「このゆびとまれ」「いかりのこな」「てだすけ」等、同様に行動順操作・対象リダイレクトを
  主目的とするダブル/トリプル専有技と同じ扱いとし、`docs/progress/move.md` で本技を全列
  「ダブル専用」（対象外）として扱う。

## 実装方針

* 通常のダメージ計算を伴わない変化技として、追加効果なし（`handlers={}`）で実装する。
* 行動順操作（対象の行動を最後にする処理）はダブル/トリプル専有機能のため未実装のままとする。
