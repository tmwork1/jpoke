## 効果

相手を **ねむけ** 状態（揮発性状態）にする。
ねむけ状態のポケモンは **次のターン終了時** にねむり状態になる。

## 失敗条件（ON_BEFORE_APPLY_MOVE priority=130）

以下の場合、あくびは失敗する：

1. 対象がすでに **ねむけ** 状態である
2. 対象がすでに **状態異常**（どく・もうどく・まひ・やけど・ねむり・こおり）を持っている

## 防がれる条件

以下の場合、あくびは防がれる（ON_BEFORE_APPLY_AILMENT/ON_BEFORE_APPLY_VOLATILE での判定）：

- しんぴのまもり状態（使用者にすりぬけがある場合は無視）
- みがわり状態（第六世代以降では使用者にすりぬけがある場合は無視）
- フラワーベール特性の影響下
- おうごんのからだ特性

## 注意：無効化（ねむけ付与後もねむりを防ぐ）

ねむけ状態になった後でも、以下の要素がある場合はねむりにならない（`ねむけ_remove_and_apply_sleep` の中の `ailment_manager.apply` が失敗する）：

- すでに状態異常になっている
- 特性 やるき / ふみん / ぜったいねむり / きよめのしお
- 特性 リーフガード かつ 天気が晴れ
- 特性 スイートベール の影響下
- エレキフィールド状態で地面にいる
- さわぐ状態のポケモンが場にいる（第五世代以降）

## イベントフロー

```
ON_BEFORE_APPLY_MOVE (priority=130)
  → すでにねむけ・状態異常なら失敗 (stop_event=True)

ON_STATUS_HIT (priority=100, MoveHandler デフォルト)
  → volatile_manager.apply(defender, "ねむけ", count=1)
```

ねむけ揮発性状態のターン終了処理は `data/volatile.py` の「ねむけ」に既実装：

```
ON_TURN_END (priority=110)  ← ねむけ_tick_volatile でカウント減
ON_VOLATILE_END             ← ねむけ_remove_and_apply_sleep でねむり付与
```

## さわぐとの相互作用

- 第五世代以降（本実装ターゲット）：あくびは成功しねむけ付与できる。
  ねむけが解除される時点（次ターン終了時）にさわぐ状態が残っていれば、ねむりは防がれる。
  この処理は `さわぐ_prevent_sleep` / `さわがしい_prevent_sleep` が担当しており、
  あくびハンドラ側では何も対応不要。

## ポケモンチャンピオンズ向けの簡略化

- Z変化技付加効果（すばやさ+1）は実装しない
- コンテスト効果は実装しない
