# ハロウィン 仕様書

## 基本情報

| 項目 | 値 |
|------|-----|
| タイプ | ゴースト |
| 分類 | 変化 |
| PP | 20 |
| 命中率 | 100 |
| 範囲 | 相手1体 |
| 優先度 | 0 |
| 直接攻撃 | × |

バケッチャ・パンプジンの専用技。

## 効果

相手の「タイプ」に **ゴーストタイプを追加** する（既存のタイプに上書きせず、複合タイプに追加する）。
すでにタイプを2つ持っているポケモンに使用した場合も上書きされず、一時的に3タイプになる。

## 失敗条件（ON_BEFORE_APPLY_MOVE）

以下の場合、ハロウィンは失敗する：

1. 対象がすでにゴーストタイプを持っている（複合タイプの一部として持っている場合を含む）
2. 対象がテラスタルしている
   - 一次情報（ポケモンWiki）の該当ページにはこの技自体のテラスタル関連の記述がないが
     （SVで使用不可の技のためページが更新されていない）、同一構造の技である「もりののろい」の
     一次情報には「テラスタルした相手に使用したときは失敗する」と明記されている。テラスタル中は
     `Pokemon.types` の基準タイプがテラスタイプに完全に置き換わり、追加タイプという概念と相容れない
     という理屈は両技で共通のため、ポケモンチャンピオンズでは同様に失敗させる

## 技自体がゴーストタイプであることによる無効化（コア共通ルール）

もりののろい（くさ技）に対する **そうしょく** のような、ゴーストタイプの技全般を無効化する特性は
存在しない。このハンドラ側で判定すべき技タイプ由来の無効化条件は無い。

## 防がれる条件

- みがわり状態（状態変化技はみがわりに防がれる共通ルール `みがわり_immune` が適用される。
  第六世代以降は特性すりぬけで無視すれば有効）
- マジックコートで反射される（`unreflectable` フラグを付与しないため既定で反射可能）
- まもるで防がれる（`unprotectable` フラグを付与しないため既定で防御可能）
- オウムがえしでコピーされる

## タイプ追加の実装方式

専用の揮発性状態「ハロウィン」を付与し、`Pokemon.added_types` にゴーストタイプを追加する
（`Pokemon.types` プロパティが `added_types` を考慮した現在のタイプを返す）。

- `ON_VOLATILE_START`: `added_types` にゴーストタイプを追加する
- `ON_VOLATILE_END`: `added_types` からゴーストタイプを除去する

状態異常ではなく揮発性状態のため、交代すると元のタイプに戻る（揮発性状態がリセットされるため。
バトンタッチでも引き継がれない）。

## イベントフロー

```
ON_BEFORE_APPLY_MOVE (priority=100, MoveHandler デフォルト)
  → ハロウィン_can_apply: 既にゴーストタイプ／テラスタル中なら失敗 (stop_event=True)

ON_STATUS_HIT (priority=100, MoveHandler デフォルト)
  → ハロウィン_apply: volatile_manager.apply(defender, "ハロウィン")
```

揮発性状態「ハロウィン」のタイプ追加処理は `data/volatile.py` に実装：

```
ON_VOLATILE_START → ハロウィン_add_type
ON_VOLATILE_END   → ハロウィン_remove_type
```

## 技の仕様（一次情報より）

- ハロウィン状態のポケモンが「もりののろい」の効果を受けると、ハロウィン状態は消える。
  もりののろい状態のポケモンがハロウィンの効果を受けると、もりののろい状態は消える
  （どちらも `added_types` を用いて実装されており、それぞれの volatile が独立して付与・解除されるため、
  実装上は特別な相互作用処理は不要。両方の volatile が同時に付与されていても `added_types` に
  「くさ」「ゴースト」が共存するだけであり、公式の「一方を受けるともう一方が消える」という仕様とは
  異なる可能性がある点に注意。ポケモンチャンピオンズでは実戦上の影響が乏しいため簡略化して問題ない）
- 交代するか、ひんしになるか、戦闘が終了するとタイプは元に戻る（揮発性状態のリセットで実現）
- フォルムチェンジ・メガシンカ・きずなへんげ・ウルトラバースト・テラスタルによりハロウィン状態は消える
  （テラスタルは失敗条件で対応。フォルムチェンジ等は各フォルムチェンジ実装側で `data.types` を
  参照し直すため、ハロウィン側での特別対応は不要）
- ハロウィン状態はバトンタッチで引き継がれない（揮発性状態のため自動的に対応済み）
- 特性へんげんじざい・リベロ・へんしょくが発動すると、ハロウィン状態は消える
  （各特性実装側で対応する事項であり、ハロウィン側での特別対応は不要）
- 特性ぎたいが発動しても、ハロウィン状態は消えない（何もしないことで実現済み）
- へんしん/ミラータイプはハロウィンの効果を受けている状態・受けていない状態もコピーする
  （`ミラータイプ_apply` が `added_types` をリセットする既存実装で対応。へんしんは
  `data.types` ベースでコピーするため元々対象外）
- ハロウィン状態のポケモンがシンクロノイズを使用すると、ゴーストタイプのポケモンにもダメージを与えられる
  （シンクロノイズ側の実装で対応する事項）
- にげられない状態のポケモンがハロウィンによりゴーストタイプになると、にげられない状態は継続したままだが
  交代させることができるようになる（にげられない側の実装で対応する事項）
- たこがため状態のポケモンがハロウィンによりゴーストタイプになっても、たこがため状態は解除されない
- ゴーストタイプ以外でのろいを使おうとしているポケモンにゴーストタイプを追加すると、発動する効果は
  「鈍い」ではなく「呪い」になる（のろい側の実装で対応する事項）

## ポケモンチャンピオンズ向けの簡略化

- ダイマックス／キョダイマックスは対象範囲外のため考慮しない
- SVでは使用不可能な技だが、`docs/champions/move_list.txt` に記載されているためポケモン
  チャンピオンズでは使用可能（PP20・まもるで防御可能）
