## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `ききかいひ_switch_on_half_hp`（にげごしと共用）で、
  発動を除外する `hp_change_reason` が `{"self_attack", "pain_split"}` のみで
  `"self_cost"` が欠けていた。仕様書上「自身がみがわり/いたみわけ/はらだいこ/のろい
  (呪い)/ソウルビートを使用したときのダメージ」は発動しないとされているが、`のろい`
  はすでに `reason="self_cost"` を正しく付与していたにもかかわらず除外対象外になって
  おり、ききかいひ/にげごしが誤発動しうるバグだった。`"self_cost"` を除外セットに追加した。
- `src/jpoke/handlers/move_status.py` の `はらだいこ_apply` / `みがわり_apply` /
  `ソウルビート_pay_hp_and_modify_attacker_stats` が、自己HP消費の `battle.modify_hp()`
  呼び出しに `reason="self_cost"` を付与していなかった（デフォルトの `reason=""` の
  まま呼んでいた）。この結果、
  - 上記のききかいひ/にげごし除外条件を追加しても、はらだいこ・みがわり・ソウルビート
    自体では正しく除外されない状態だった。
  - さらに `マジックガード_ignore_damage` は `reason` が
    `{"move_damage", "pain_split", "self_attack", "self_cost", "perish"}` のいずれでも
    ない場合にダメージを無効化する実装のため、`reason=""` のままだとマジックガード
    持ちがはらだいこ/みがわり/ソウルビートを使ってもHP消費が無効化されてしまう
    （実際には消費されるべき）副次的なバグも存在した。
  - `battle.modify_hp(..., reason="self_cost", source=mon)` に修正し、両方のバグを解消した。
