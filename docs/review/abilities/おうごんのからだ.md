## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `おうごんのからだ_block_status_move` を修正した。
  - 従来は `ctx.move.category == "status" and ctx.move.target == "foe"` のみで判定しており、
    「自分含む全員が対象の技」であるほろびのうたが `target` 未指定（デフォルト `"foe"`）のため
    技全体（使用者自身への効果も含む）を無効化してしまうバグがあった
    （`docs/spec/abilities/おうごんのからだ.md`「自分含む全員が対象の技 (おちゃかい・ほろびのうた)」
    「ただし、いずれの場合もこの特性のポケモン自身が使用する技が自身の特性で防がれることはない」
    に反する）。
  - `_OUGON_NO_KARADA_EXCLUDED_MOVES = {"ほろびのうた"}` を追加し、ほろびのうたを技全体の
    無効化対象から除外した（おちゃかいは既存の `target="self"` によりもとから対象外）。
- `src/jpoke/handlers/move_status.py` を修正した。
  - 共通ヘルパー `_blocked_by_ougon_no_karada(battle, mon)` を追加。おうごんのからだが有効
    （`mon.ability.enabled`。かがくへんかガス・かたやぶり等による無効化を自動的に考慮）かつ
    ポケモン名が一致する場合に特性発動ログを記録して `True` を返す。
  - `おちゃかい_force_consume_berries`: 使用者以外できのみを強制消費する直前にこのヘルパーで
    免疫判定を行うよう修正。従来はおうごんのからだ所持者であってもきのみが強制消費されて
    いたバグを修正した。
  - `ほろびのうた_apply`: 使用者以外へほろびのうた状態を付与する直前にこのヘルパーで免疫判定を
    行うよう修正。上記の全体無効化バグ修正と対になる変更で、使用者自身には通常通り状態が
    付与されるようになった。
- 動作確認: ほろびのうた・おちゃかいのいずれも「おうごんのからだ所持者への効果のみ防ぎ、
  使用者自身の効果は防がない」こと、かたやぶり所持ポケモンが使用した場合は免疫が無効化される
  ことをスクリプトで確認した。また `ぼうおん`（ほろびのうたを技全体で無効化する既存実装、
  使用者自身への効果まで防いでしまう既知の制約が残る）には影響がないことを確認した
  （このぼうおん側の制約は本レビューのスコープ外として計画書に明記した）。
