## 実装計画書
- 更新日・実装方針を修正した。

  - テスト観点に「まひで最初の行動がPP消費なしで失敗しても発動する」を追加した。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `かそく_boost_speed`: 判定条件を `mon.executed_move is not None` から `mon.acted_since_switch_in` に修正した。
  - `executed_move` は技が `ON_TRY_ACTION` を通過し実際に実行された場合のみセットされるため、まひ（フルパラ）・ねむり・こおり・こんらん自傷等でPPを消費せず行動が失敗した最初の行動では発動しないバグがあった（仕様書「最初の行動が状態異常/状態変化によりPPを消費することなく失敗したときでも発動するようになる」に反していた）。
  - `acted_since_switch_in` は `MoveExecutor.run_move` の `finally` 節で行動の成否に関わらず True になり、`Pokemon.reset_on_switch_in` で場に出るたびにリセットされるため、仕様の意図を正しく表現できる。
- `src/jpoke/data/ability.py` の `かそく` ハンドラ登録: `priority` が未指定（デフォルト100）だったため、`docs/spec/turn.md` の ON_TURN_END 表（priority=150、ムラっけと同格）に合わせて `priority=150` を追加した。
