## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `ふとうのけん_boost_A` にバグを発見し修正した。
  - 修正前: `battle.add_ability_disabled_reason(mon, "consumed")` を
    `if battle.modify_stats(...)` の **成功時のみ** 呼んでいたため、こうげきが既に
    最大まで上がっていて特性が不発だった場合に「発動済み」フラグが立たず、次回入場時に
    再度発動できてしまう不具合があった。
  - 仕様書（第九世代以降の仕様）: 「すでに攻撃が最大まで上がっているため特性が不発した場合
    でも、その戦闘で特性を発動させることはできなくなる。」と明記されており、姉妹特性
    ふくつのたて（`ふくつのたて_boost_B`）・かんろなミツ（`かんろなミツ_lower_foe_evasion`）
    ではすでに `add_ability_disabled_reason(mon, "consumed")` を `modify_stats` の成否に
    関わらず無条件で呼ぶ実装になっており、ふとうのけんのみがこのパターンから逸脱していた。
  - 修正後: `add_ability_disabled_reason` を `modify_stats` の判定より前に無条件で呼ぶよう
    順序を入れ替え、ふくつのたてと同じパターンに揃えた。

- 全テスト実行結果は `C:\Users\tmtmh\Documents\pokemon\jpoke\.loop\test_logs\ふとうのけん.log`
  に保存した。`tests/test_examples_smoke.py` の13件は、ローカル環境に `jpoke` パッケージが
  pip インストールされていない（`pip show jpoke` が未検出）ため、`subprocess.run(sys.executable, ...)`
  経由で examples スクリプトを起動する際に `ModuleNotFoundError: No module named 'jpoke'` と
  なって失敗している。これは今回の変更（ふとうのけん）と無関係かつ、mainブランチ環境自体で
  同様に発生する既存の環境依存の問題（`pip install -e .` 未実施）であり、間欠的な flaky 失敗
  ではなく決定的に常時発生するため、このレビュー作業の対象外として扱った。
  `--deselect tests/test_examples_smoke.py` を指定した場合は 5389 passed, 1 skipped, 0 failed。
