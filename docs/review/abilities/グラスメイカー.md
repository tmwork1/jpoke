# グラスメイカー レビュー結果

## 仕様書
- 変更なし（`docs/spec/abilities/グラスメイカー.md` は一次情報（`docs/wiki/abilities/グラスメイカー.html`）と照合し、内容に誤り・欠落なし）

## 実装計画書
- 新規作成（`docs/plan/abilities/グラスメイカー.md` が存在しなかったため、`docs/plan/abilities/エレキメイカー.md` を参考に作成した）

## 実装（handlers/ / data/）
- `src/jpoke/data/ability.py` の `グラスメイカー` に `Event.ON_ABILITY_ENABLED` のハンドラ登録を追加した。
  - 仕様書に「ポケモンの特性がグラスメイカーに書き換わった場合、即座に特性が発動する」とあるが、
    `ON_SWITCH_IN` のみ登録されており、スキルスワップ・トレース・かがくへんかガス解除等での
    特性書き換え時に再発動しない欠落があった。
  - `エレキメイカー`/`ハドロンエンジン` と同様のパターン（`ON_SWITCH_IN` と `ON_ABILITY_ENABLED` の
    両方で `グラスメイカー_activate_terrain` を登録）に揃えた。
  - `handlers/ability.py` の `グラスメイカー_activate_terrain` は既存の共通ヘルパー
    `_activate_terrain` を利用しており変更不要。

## リーサル計算
- 対象外（`docs/progress/ability.md` の「リーサル実装」列が `n/a`。グラスメイカーは場の状態を
  発生させるのみでダメージ計算に直接関与しないため対象外）

## テスト
- `tests/abilities/test_ability_ka.py` に `test_グラスメイカー_特性再有効化時にも発動する` を追加した。
  かがくへんかガスで特性が無効化された状態から解除されたときに `ON_ABILITY_ENABLED` 経由で
  グラスフィールドが展開されることを確認する（`エレキメイカー` の同種テストを参考に作成）。
- `tests/abilities/test_ability__group.py` の `ability_terrain_pairs` に既に `グラスメイカー` が
  含まれており、登場時展開・既存フィールド上書き不可のテストはカバー済みのため変更なし。
