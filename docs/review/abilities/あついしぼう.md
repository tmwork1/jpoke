## 仕様書
- 変更なし。一次情報（`docs/wiki/abilities/あついしぼう.html`）と照合し、既存の
  `docs/spec/abilities/あついしぼう.md` の内容（第五世代以降の仕様: こうげき・とくこうを
  半減してダメージ計算、タイプ相性判定は変わらない、ダメージ固定技は対象外、
  かたやぶりで無効、ランク補正には影響しない）は一致していることを確認した。

## 実装計画書
- 新規作成。`docs/plan/abilities/あついしぼう.md` が存在しなかったため、既存の実装
  （`ON_CALC_ATK_MODIFIER` / `subject_spec="defender:self"`）をもとに新規作成した。
  priority は turn.md に記載のない内部イベントのため、同一パターンの既存ハンドラ
  （たいねつ・きよめのしお・わざわいのうつわ/おふだ）に合わせてデフォルト値
  （`priority=100`）を採用する旨を根拠として明記した。

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/data/ability.py` の `あついしぼう` エントリ
  （`flags={"mold_breaker_ignorable"}` + `ON_CALC_ATK_MODIFIER` ハンドラ登録）と
  `src/jpoke/handlers/ability.py` の `あついしぼう_reduce_fire_ice`
  （炎・氷タイプの技を受けたとき `apply_fixed_modifier(value, 2048)` で 0.5 倍）を
  仕様書・計画書と照合し、誤り・欠落がないことを確認した。
  - ダメージ固定技への非適用は `calc_damages` が `_calc_final_attack`（ON_CALC_ATK_MODIFIER
    発火元）を経由しない技（`fixed_damage` フラグ技）で自然に担保されており、ハンドラ側の
    追加ガードは不要と判断した。
  - かたやぶり無効化はフラグ `mold_breaker_ignorable` に委譲されており、ハンドラ内に
    重複ロジックは無い。
