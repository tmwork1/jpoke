# アナライズ レビュー結果

## 仕様書
- 変更なし。`docs/spec/abilities/アナライズ.md` を一次情報（`docs/wiki/abilities/アナライズ.html`）と
  照合し、効果（そのターンで最後に行動するなら威力1.3倍）、行動順の判定に関する仕様（ダブル/トリプル
  での3体以上判定、テラレイドバトルでの扱い、行動順に影響する各種効果、交代による発動、行動できな
  かった場合・ひんしになった場合の第八/第九世代差異、しれいとうとの相互作用）、威力補正の仕様（正確な
  補正値5325/4096、こんらん自傷は対象外、みらいよち/はめつのねがいは対象外、さいはい/おどりこ
  との相互作用）を確認したが、一次情報と齟齬はなく欠落もなかった。

## 実装計画書
- 新規作成。`docs/plan/abilities/アナライズ.md` が存在しなかった（`docs/progress/ability.md` の計画書列は
  `x` になっていたが実体がなく不整合だったため作成した）。
  仕様書をもとに、プロジェクトがシングルバトル専用であることを踏まえ、行動順判定を
  `battle.query.is_second_actor()`（2人対戦想定の「相手より先に行動記録に現れなければ後攻」判定）
  で行う方針、`ON_CALC_POWER_MODIFIER` の優先度は既存の同種威力補正特性に準拠する方針、
  こんらん自傷（内部技`_こんらん`）を除外する方針を明記した。

## 実装（handlers/ / data/）
- `handlers/ability.py` の `アナライズ_boost_power` に `ctx.move.name == "_こんらん"` の場合は
  威力補正を行わずに早期returnするガードを追加した。
  - 修正前は、こんらんの自傷ダメージ（内部技`_こんらん`、`ON_CALC_POWER_MODIFIER`を経由する）にも
    後攻判定次第で1.3倍補正がかかってしまうバグがあった（`handlers/field.py`のオーロラベールなど
    既存箇所で使われている `ctx.move.name != "_こんらん"` ガードと同じパターンで修正）。
    修正前後でテストを実行し、修正前は失敗（5325 != 4096）・修正後は成功することを確認した。
  - `battle.query.is_second_actor()` を用いた後攻判定自体（`turn_controller.action_order` の先頭要素が
    自分でなければ後攻とみなす）はシングルバトル1vs1の前提で妥当であることを確認した。相手が交代した
    場合、交代は必ず移動フェーズより前の交代フェーズで `action_order` に記録されるため、相手の交代後は
    自分が常に「後攻」と判定される（一次情報の「トレーナーが交代させた場合も発動する」と整合）。
  - `data/ability.py` の登録（`Event.ON_CALC_POWER_MODIFIER`, `subject_spec="attacker:self"`）は
    `AttackContext` のロールと一致しており変更なし。

    交代すると威力補正がかかることを確認（交代による発動の検証）。
  - `test_アナライズ_こんらんの自傷では威力上昇しない`: 後攻状態でこんらんの自傷ダメージが発生しても
    威力補正がかからないことを確認（上記バグ修正の検証）。
