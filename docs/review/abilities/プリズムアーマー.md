# プリズムアーマー レビュー結果

## 仕様書
- 変更なし。一次情報（wiki.pokemonwiki.com/wiki/プリズムアーマー）と全項目（効果・特性の仕様・備考・関連項目）を照合し、一致を確認した。

## 実装計画書
- 新規作成。`docs/plan/abilities/プリズムアーマー.md` が存在しなかったため、仕様書および
  既存のハードロック・フィルターの計画書を参考に新規作成した。実装方針として、
  ハードロックと同じハンドラ関数 `ハードロック_reduce_effective` を共用しつつ、
  `mold_breaker_ignorable` フラグを付与しないことでかたやぶり耐性を実現している旨を明記した。

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/data/ability.py` の `プリズムアーマー` エントリは
  `Event.ON_CALC_DAMAGE_MODIFIER`（`subject_spec="defender:self"`）で
  `h.ハードロック_reduce_effective` を使用しており、`flags` に
  `mold_breaker_ignorable` を含まない（ハードロック・フィルターとの差分どおり、
  かたやぶり・特性無視技に対しても効果が維持される）。仕様書・計画書と一致しており
  修正不要と判断した。

## リーサル計算
