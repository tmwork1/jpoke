# きんしのちから レビュー結果

## 仕様書
- 変更なし。`docs/spec/abilities/きんしのちから.md` の内容を一次情報
  （`docs/wiki/abilities/きんしのちから.html`）と突き合わせたところ、効果節・特性の仕様節とも
  一言一句相違なく完全に一致していることを確認した。

## 実装計画書
- `docs/plan/abilities/きんしのちから.md` は実装状態が「未実装」のまま放置されており、実装対象
  イベントも実際に登録されているイベント（`DomainEvent.ON_CALC_BACK_TIER` / `Event.ON_BEGIN_MOVE` /
  `Event.ON_END_MOVE`）と異なる架空のイベント名（`DomainEvent.ON_MODIFY_MOVE_PRIORITY` /
  `Event.ON_ACTIVATE_MOLD_BREAKER` / `Event.ON_DEACTIVATE_MOLD_BREAKER`）が書かれていたため、実装
  済みの内容に合わせて全面的に修正した。
  - 実装状態を「実装済み」に更新
  - 実装方針・実装対象イベントを実際のイベント（あとだし/かたやぶりと共有する
    `ON_CALC_BACK_TIER` / `ON_BEGIN_MOVE` / `ON_END_MOVE`）に修正し、優先度指定なしとする根拠
    （あとだし・かたやぶりの既存登録に合わせた）を明記
  - `ctx.move` が「選択した技」ではなく「実際に使用される技」を指すため、ねごと・さいはい等で
    技が入れ替わった場合の挙動も正しく再現されることを補足
  - 変更対象ファイル・関数名を実装済みの実名（`きんしのちから_delay_status_move` 等）に修正し、
    せんせいのツメ/イバンのみとの相互作用（実装漏れがあったため今回追加）を変更対象に追記
  - テスト観点にせんせいのツメ/イバンのみとの相互作用を追加

## 実装（handlers/ / data/）
- `handlers/ability.py` の `きんしのちから_delay_status_move` / `きんしのちから_disable_foe_ability` /
  `きんしのちから_restore_foe_ability` と `data/ability.py` の登録内容は仕様書・かたやぶり/あとだし
  の既存パターンと整合しており、修正なし。
- `handlers/item.py` の `せんせいのツメ_priority_boost` には「所有者の特性がきんしのちからで変化技
  選択時は発動しない」判定があったが、`イバンのみ_boost_priority` には同等の判定が欠落しており、
  仕様書「変化技を選択したターンでは、持たせたせんせいのツメ/イバンのみは発動することがない」に
  反していたため追加した（バグ修正）。
