# ヒーリングシフト レビュー結果

## 仕様書
- 変更なし。`docs/spec/abilities/ヒーリングシフト.md` を一次情報（`docs/wiki/abilities/ヒーリングシフト.html`）
  と照合した。優先度+3の対象技リスト（自分HP回復技・相手HP回復技・味方全体HP回復技・HP吸収攻撃技・
  その他の回復技、全33種）、除外条件（アクアリング・グラスフィールド・ねをはる・やどりぎのタネは
  状態変化技のため対象外、いたみわけ・プレゼント/かふんだんごの回復効果は回復技と見なされない、
  Z変化わざ・キョダイダンエンは対象外）、はやてがえしとの相互作用（吸収技の優先度が上がっても
  はやてがえしは成立し、変化技には成立しない）を正しく反映しており誤り・欠落はなかった。
  なお仕様書に記載の「かいふくしれい」「じょうか」は `src/jpoke/data/ps-champ-ja/moves.json` に
  技単体のエントリが存在せず（かいふくふうじの説明文中の言及のみ）、本プロジェクトの対象範囲
  （チャンピオンズ）では未実装の技のため、リストへの記載自体は一次情報通りで妥当。

## 実装計画書

  - `ctx.move.has_label("heal")` → `ctx.move.has_flag("heal")`（`has_label` というメソッドは

    検証コマンド欄も同様に修正。

## 実装（handlers/ / data/）
- 変更なし。`handlers/ability.py` の `ヒーリングシフト_modify_priority` は
  `AttackContext` を受け取り `ctx.move.has_flag("heal")` が True のとき優先度に+3する実装で、
  `data/ability.py` では `DomainEvent.ON_MODIFY_MOVE_PRIORITY` に `subject_spec="attacker:self"`
  で登録されており、コンテキストロールと一致していた。
  `data/moves/*.py` を全件確認し、仕様書記載の33技すべてに `"heal"` フラグが付与されており、
  除外対象（アクアリング・ねをはる・やどりぎのタネ・いたみわけ・プレゼント・かふんだんご等）には
  付与されていないことを確認した。過不足なし。
  `はやてがえし`（`handlers/move_attack.py` の `_はやてがえし_can_apply`）は優先度上昇の有無に
  関わらず「攻撃技か否か」で成否判定しており、本特性による優先度上昇時の挙動（吸収技には成功、
  変化技には失敗）を追加実装なしで正しく満たしていることを確認した。
