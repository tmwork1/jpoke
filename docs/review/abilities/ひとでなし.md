  同イベントは `docs/spec/turn.md` に記載がない（ダメージ計算内部のイベント）が、
  既存の同種ハンドラ（きょううん・するどいツメ・ピントレンズ・きゅうしょアップ〈volatile〉）が
  すべて priority 未指定（デフォルト100）の単純な加算/上書きであり、最終的に

  未指定でよいと判断し、その根拠を計画書に明記した。

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/handlers/ability.py` の `ひとでなし_modify_critical_rank`
  （`AttackContext` を受け取り、`ctx.defender.has_ailment("どく", "もうどく")` のとき
  `value = 10` として `clamp_critic` により急所ランク3＝確定急所にする）、
  `src/jpoke/data/ability.py` の `ON_CALC_CRITICAL_RANK` ハンドラ登録
  （`subject_spec="attacker:self"`）とも仕様・計画書と一致していた。
- カブトアーマー/シェルアーマーの急所無効化（`value=0, stop_event=True`）は
  ひとでなしとの実行順（どちらが先でも最終値は0になる）に依存せず正しく機能することを確認した。
- 急所判定は `core/move_executor.py` の `_execute_hit` でヒットごとに行われるため、
  どくの追加効果を持つ技（どくづき等）でその場で毒を付与した場合でも、急所判定が
  ダメージ計算（＝追加効果判定より前）に行われる実装になっており、仕様どおり
  「その攻撃自体には効果が無い」動作を確認した（追加でテストを追加、後述）。
- ミイラ/さまようたましい/とれないにおいの特性書き換えハンドラは接触ダメージ後
  （`ON_DAMAGE_HIT`）に登録されており、急所判定より後に発火するため、仕様どおり
  「その攻撃に限り効果がある」動作になることをコードレベルで確認した（変更なし）。
- かがくへんかガスによる無効化は特性システム側の共通処理で担っており、ひとでなし側に
  個別のガスチェックは不要（プロジェクト規約どおり）。実装に該当コードがないことを確認した。

### 対象外・既知の制限として記録した事項（本タスクのスコープ外）
- みらいよち/はめつのねがい (`handlers/move_attack.py` の `みらいよち_charge` /
  `はめつのねがい_charge`) は、溜め時点で `battle.roll_damage(ctx.attacker, ctx.defender,
  ctx.move)` を `critical` 引数を渡さず（デフォルト `False`）呼び出しており、
  攻撃技として急所判定イベント（`ON_CALC_CRITICAL_RANK`）自体を発火させていない。
  そのため、仕様書に記載した「みらいよち/はめつのねがいは、攻撃時にひとでなしの使用者が
  場に残っている場合のみ必ず急所に当たる」という一次情報の挙動は、現在の実装では
  再現できない（急所は常に発生しない）。
  この制限はひとでなし固有の問題ではなく、みらいよち/はめつのねがいの遅延攻撃全体に
  影響する（きょううん・するどいツメ・ピントレンズ・スナイパー等、急所関連の特性/道具
  すべてに同様に非対応）。修正には遅延攻撃の急所判定を着弾時点に遅延させる設計変更が
  必要で、単一特性のレビューの範囲を超えるため、今回は修正せず記録のみ行う。
  （ダブルニードルは本リポジトリに未実装のため、多段攻撃での確定急所についても現時点では
  実機的な検証はできない。既存の連続技ループ実装（ヒットごとに `_check_critical` を再実行）
  からは、実装済みになれば正しく動作すると判断できる。）
