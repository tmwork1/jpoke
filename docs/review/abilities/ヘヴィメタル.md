## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/data/ability.py` の `"ヘヴィメタル"` エントリは `flags={"mold_breaker_ignorable"}` のみを持ち、`src/jpoke/model/pokemon.py` の `Pokemon.weight` が `self.ability.name` の `match` で `w *= 2` している。
- `GameEffect.name` は無効化理由がある場合に空文字を返す仕様であり、`かたやぶり_disable_foe_ability` ハンドラが `mon.ability.has_flag("mold_breaker_ignorable")` を見て `battle.add_ability_disabled_reason(mon, "かたやぶり")` を設定するため、かたやぶり系の技を受けた場合は `self.ability.name` が空文字になり `match` 文がヒットせず正しく無視される。実装は仕様と整合しており、修正不要と判断した。
- かるいしとの重ね掛け順序（ヘヴィメタル→かるいしの順で0.5倍）も `weight` プロパティのコード順で仕様通りになっていることを確認した。

## テスト
- 既存の `tests/abilities/test_ability__group.py::test_体重操作系` で通常時のおもさ2倍化は確認済みだったが、「かたやぶり系の効果がある技を受けた場合、この特性は無視される」という仕様書記載のエッジケースを検証する実戦的なテストが存在しなかったため、`tests/abilities/test_ability_ha.py` に以下を追加した。
  - `test_ヘヴィメタル_かたやぶりで無効化されおもさが基本値になる`: かたやぶり持ちのくさむすびを受けたとき、ドータクン(187.0kg)のおもさがヘヴィメタルで2倍化されず基本値のまま扱われ、威力が100（374kg想定の120ではなく187kg相当の100）になることを確認。
  - `test_ヘヴィメタル_おもさが基本値の2倍になる`: 通常時（かたやぶりでない攻撃）はヘヴィメタルによりドータクンのおもさが374.0kgとして扱われ、くさむすびの威力が120になることを確認。
- `python -m pytest tests/ -v` を実行し、5438 passed, 1 skipped で全件成功を確認した（結果は `.loop/test_logs/ヘヴィメタル.log` に保存）。
