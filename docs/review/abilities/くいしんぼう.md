# くいしんぼう レビュー結果

## 仕様書
- `docs/spec/abilities/くいしんぼう.md` の「特性の仕様」節が空だったため、一次情報（ポケモンWiki
  「詳細な仕様」節）をもとに以下を補充した。
  - HP1/4超-1/2以下になった瞬間に対応するきのみを発動できること
  - 特性が後からくいしんぼうに書き換わった瞬間の即時発動
  - 場に出た時点で既にHP1/4超-1/2以下の場合の即時発動
  - かたやぶりの効果がある技を受けた場合も発動すること
  - きんちょうかん・かがくへんかガスとの相互作用（解除後の再発動条件、イバンのみの例外扱いなど）
  - ミイラ/さまようたましい/とれないにおいのポケモンを直接攻撃した際の反動ダメージでは
    特性変更が優先されて発動しないこと
- 「備考」に、現行実装が既存のHP閾値系きのみ発動ロジック（`Event.ON_HP_CHANGED` 依存）を
  踏襲しており、特性書き換え時の即時発動・場に出た瞬間の即時発動等は他のHP閾値系きのみ全般で
  未対応のため本特性でも対象外とする旨を明記した。
- 「関連項目」に対象14種のきのみ名を追記した。

## 実装（handlers/ / data/）
- `src/jpoke/data/ability.py` の `くいしんぼう` エントリが空の `AbilityData()` のまま、
  かつ `src/jpoke/handlers/item.py` 側にも対応する分岐が一切無く、効果が完全に未実装だった
  （進捗表では実装=-のとおり）ため、新規実装した。
- `src/jpoke/handlers/item.py` に `_gluttony_denominator(mon, denominator)` を追加。
  denominator=4（1/4以下発動）のときのみ、くいしんぼう所持で2（1/2以下発動）に緩和する
  （denominator=2のオボンのみ等には影響しない）。
- 以下の関数内のハードコードされたHP閾値判定を `_gluttony_denominator` 経由に変更した。
  - `_heal_berry`（イアのみ・ウイのみ・バンジのみ・フィラのみ・マゴのみが対象）
  - `_boost_on_quarter_hp`（カムラのみ・ズアのみ・チイラのみ・ヤタピのみ・リュガのみが対象）
  - `イバンのみ_set_priority_flag`
  - `サンのみ_apply_focus_energy`
  - `スターのみ_random_boost`
  - `ミクルのみ_set_accuracy_flag`
- `src/jpoke/data/ability.py` の `くいしんぼう` エントリはハンドラ登録不要（インライン判定の
  ため）だが、その理由をコメントで明記した。
