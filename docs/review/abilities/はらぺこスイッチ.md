## 実装計画書
- 全面的に更新した（実装済みの内容に合わせて記述を刷新し、以下の誤り・欠落を修正）。
  - `Event.ON_TURN_END_5` という存在しないイベント名を記載していた点を
    `Event.ON_TURN_END`（priority を分けて対応）に修正。
  - モルペコのフォルムを `zukan.json`（実在しないファイル名。実際は
    `pokedex.json`）の新規エントリで管理する方針だったが、実装はより単純な
    `Ability.is_hangry` フラグのみで完結しており、この点を実装済みの内容に
    合わせて修正。
  - オーラぐるまのタイプ変換を特性側 `ON_MODIFY_MOVE_TYPE` に登録する計画
    だったが、実際は技側（`オーラぐるま` の `ON_MODIFY_MOVE_TYPE`）に実装
    されており、特性が無効化されていてもフォルムに応じたタイプになる仕様と
    整合することを確認して記述を修正。
  - `ON_TURN_END` の priority 根拠、`ON_SWITCH_OUT` の優先度根拠
    （turn.md 未掲載イベントのため既存の同種ハンドラを参照した旨）を明記。
  - スキルスワップ/さまようたましいでの交換不可、テラスタル中ひんし時の
    まんぷく復帰など、今回のレビューで発見した実装ギャップの実装ステップを追記。
  - 「既知の制限」節を新設し、`なりきり` で自発的に特性を手放した場合に
    `is_hangry` がリセットされる未対応ケースを明記（一次情報が明言していない
    ため今回は実装見送り）。

## 実装（handlers/ / data/）
- `src/jpoke/data/ability.py`: `はらぺこスイッチ` の `Event.ON_TURN_END` に
  `priority=161` を追加。従来は `AbilityHandler` のデフォルト値
  （`priority=100`）のままで、`docs/spec/turn.md` の
  「160 ダルマモード/リミットシールド/スワームチェンジ/ぎょぐんによる
  フォルムチェンジ」より**先に**実行される状態だった。一次情報の
  「すばやさに関係なくはらぺこスイッチが後に発動する」という要件を満たして
  いなかったバグ。同じ priority=160 のままだと `_sort_handlers` の
  `(priority, -speed)` tie-break ですばやさ次第で前後してしまうため、161に
  設定して必ず後に回るようにした。
- `src/jpoke/handlers/ability.py`:
  - `はらぺこスイッチ_on_switch_out`: `mon.fainted` の判定を追加。従来は
    `not mon.terastallized` のみでフォルムリセットを判断していたため、
    テラスタル中にひんしになった場合でも `is_hangry` が維持されたままに
    なるバグがあった。一次情報「テラスタルしてひんしになった場合は、
    まんぷくもように戻る」に反していたため、`mon.fainted or not
    mon.terastallized` に修正。
  - `さまようたましい_swap_ability_on_contact`: 攻撃者の特性が
    `はらぺこスイッチ` の場合も交換を防ぐよう base_name 判定を追加
    （従来は `かがくへんかガス` のみ除外しており、モルペコがさまようたましい
    持ちに接触攻撃すると特性が入れ替わってしまうバグがあった）。
- `src/jpoke/handlers/move_status.py`: `スキルスワップ_can_apply` の
  `UNSWAPPABLE_ABILITIES` タプルに `はらぺこスイッチ` を追加（従来は
  `protected` フラグを持たないため通常のスキルスワップで交換できてしまう
  バグがあった）。
- `オーラぐるま`（`data/moves/move_a.py` / `handlers/move_attack.py`）は
  レビューの結果、既存実装が仕様と一致しており変更なし。
