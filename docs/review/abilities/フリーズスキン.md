## 実装計画書
- 実装方針・変更候補・テスト観点を修正した（既存記述はフェアリースキンの計画書がレビュー時に更新される前の
  古い内容のままで、以下の欠落・誤りがあった）:
  - 「スカイスキン・フェアリースキン・ノーマルスキンと同一パターン」という記述を修正。
    実装（共通ヘルパー `_skin_modify_move_type` / `_skin_boost_power`）を確認したところ、
    フリーズスキンが共用しているのはスカイスキン・フェアリースキン・ドラゴンスキンであり、
    ノーマルスキンは変換方向が逆（非ノーマル→ノーマル）のため別実装で共通ヘルパーを使わない。
  - ウェザーボール/さばきのつぶて/しぜんのめぐみ/だいちのはどう/マルチアタック/めざめるダンスが
    スキン系特性の対象外になる旨、およびテラバースト/テラクラスターのテラスタル判定条件が
    未記載だったため追記した。
  - 変更候補のテストファイルパスを `tests/test_ability.py`（存在しない）から
    実際に使われている `tests/abilities/test_ability__group.py` に修正した。
  - テスト観点にウェザーボール等の対象外判定・めざめるダンスの対象外判定・テラバーストの
    テラスタル有無による挙動を追加した。

## 実装（handlers/ / data/）
- 変更なし。`handlers/ability.py` の `フリーズスキン_modify_move_type` / `フリーズスキン_modify_power` は
  スカイスキン・ドラゴンスキン・フェアリースキンと共通の `_skin_modify_move_type` / `_skin_boost_power`
  ヘルパーを呼び出しており、対象外技（ウェザーボール等・テラバースト系）の判定を含めて仕様と一致している
  （この共通ヘルパーはフェアリースキンのレビュー時に既に修正済みで、フリーズスキンにも反映されている）。
  `data/ability.py` の登録内容（`ON_MODIFY_MOVE_TYPE` / `ON_CALC_POWER_MODIFIER`、
  `subject_spec="attacker:self"`）も他のスキン系特性と一致しており問題なし。

## テスト
- 変更なし。`tests/abilities/test_ability__group.py` の `SKIN_CASES`
  （スカイスキン/ドラゴンスキン/フェアリースキン/フリーズスキン共通）に対して以下が既にカバーされている:
  - `test_スキン系_ノーマル技を対応タイプに変換する`
  - `test_スキン系_変換した技の威力が4915倍になる`
  - `test_スキン系_もともと対応タイプの技は威力補正を受けない`
  - `test_スキン系_対象外タイプの技はタイプも威力も変化しない`
  - `test_スキン系_タイプが変わる技は結果がノーマルタイプでも対象外になる`
  - `test_スキン系_めざめるダンスはタイプが変わる技として対象外になる`
  - `test_スキン系_テラスタル後もノーマル技は対応タイプに変換される`
  - `test_スキン系_テラバーストはテラスタルしていなければ対象になる`
- `python -m pytest tests/ -v` で全5410件（+1 skipped、既存の無関係なskip）がパスすることを確認した。
