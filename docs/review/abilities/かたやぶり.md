## 実装（handlers/ / data/）
- `かたやぶり_disable_foe_ability` / `かたやぶり_restore_foe_ability`（`ターボブレイズ` / `テラボルテージ` も同一ハンドラを共有）自体に誤りはなかった。`mold_breaker_ignorable` フラグによるフィルタリングも仕様書の一覧と完全一致していた。
- レビュー中に、かたやぶりの仕様書に明記されている「やるき/ふみん/めんえき/じゅうなん/みずのベール/マグマのよろい/マイペース/どんかん/すいほう/ねつこうかんは、かたやぶりに無視されて状態異常/状態変化にされても技終了直後に特性の効果で回復する」という相互作用について、`ふみん`（`ふみん_cure_sleep_on_enable`）以外の9特性に `Event.ON_ABILITY_ENABLED` の回復ハンドラが実装されておらず、無視された状態異常/状態変化が回復せず残り続けるバグを発見したため修正した。
  - `src/jpoke/handlers/ability.py` に共通ヘルパー `_cure_ailment_on_enable` / `_cure_volatile_on_enable` を追加し、以下の特性ごとの回復ハンドラを実装した。
    - `やるき_cure_sleep_on_enable`（ねむり）
    - `めんえき_cure_poison_on_enable`（どく/もうどく）
    - `じゅうなん_cure_paralysis_on_enable`（まひ）
    - `みずのベール_cure_burn_on_enable`（やけど）
    - `マグマのよろい_cure_freeze_on_enable`（こおり）
    - `ねつこうかん_cure_burn_on_enable`（やけど）
    - `マイペース_cure_confusion_on_enable`（こんらん）
    - `どんかん_cure_volatile_on_enable`（ちょうはつ/メロメロ）
    - `すいほう_cure_burn_on_enable`（やけど）
  - さらに、`すいほう` 自体にやけど無効化ハンドラ（`prevent_burn_ailment`）が一切登録されておらず、`docs/spec/abilities/すいほう.md` に明記された「やけど状態にならない」という基本効果自体が未実装だったため、`Event.ON_BEFORE_APPLY_AILMENT` への登録を追加した。
  - `docs/spec/abilities/パステルベール.md` に「パステルベールのポケモン自身はどく/もうどく状態になっても直後に回復する」とあるが、こちらも回復ハンドラが未実装だったため `パステルベール_cure_poison_on_enable` を追加した（本プロジェクトはシングルバトル専用のため「味方は回復しない」側の仕様は対象外）。
  - 各特性の `data/ability.py` エントリに `Event.ON_ABILITY_ENABLED` ハンドラ（`subject_spec="source:self"`）を追加登録した。
