- 変更なし。`docs/spec/abilities/あとだし.md` は一次情報（docs/wiki/abilities/あとだし.html）と照合し、
  効果・特性の仕様（優先度は変わらない、せんせいのツメ/イバンのみ発動時の効果消失、トリックルーム下でも
  後攻、あとだし同士・こうこうのしっぽ/まんぷくおこう・きんしのちからとの速度比較）を正しく反映していた。
  第四世代限定の仕様（本プロジェクトの対象範囲外）は記載されておらず妥当。

## 実装計画書
- 変更なし。`docs/plan/abilities/あとだし.md` の実装方針（`DomainEvent.ON_CALC_BACK_TIER` で後攻ティア -1
  を付与し、優先度自体は変更しない）は仕様書と整合しており、CLAUDE.md のハンドラ約束事にも反する点はない。

## 実装（handlers/ / data/）
- 変更なし。`handlers/ability.py` の `あとだし_delay_move_order` は `AttackContext` を受け取り
  `subject_spec="attacker:self"` で登録されており、コンテキストロールと一致していた。
  `core/speed_calculator.py` の `resolve_action_order` で `(move_priority, back_tier, speed_key)` の
  タプルにより「優先度→後攻ティア→素早さ」の順で行動順を決定しており、あとだし・せんせいのツメ
  （+1）・イバンのみ（+1）が同一ポケモンに揃うとティアが相殺され「あとだしの効果が無くなり素早さ
  順に戻る」という一次情報の挙動を正しく再現していることを確認した。重複ガードや不要な分岐もなかった。

## リーサル計算
