## 実装計画書
- 「テスト観点」から「かがくへんかガス発動中は回復しない」を削除（かがくへんかガスによる無効化は

## 実装（handlers/ / data/）
- 変更なし（`handlers/ability.py` の `さいせいりょく_heal_on_withdraw` は `battle.modify_hp(mon, r=1/3,
  reason="bench_heal")` で端数切り捨ての1/3回復を行い、`reason="bench_heal"` により
  `かいふくふうじ_block_heal` の対象外とすることでかいふくふうじ無効の仕様を満たしている。
  `data/ability.py` の `ON_SWITCH_OUT` 登録も priority 未指定で `しぜんかいふく` 等と同パターン。
  かがくへんかガスによる無効化は `EventManager` の `ability.enabled_ignoring` チェックで汎用的に
  処理されており、ハンドラ側に重複ガードは不要で実際に存在しないことを確認した）
