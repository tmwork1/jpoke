## 実装（handlers/ / data/）
- `src/jpoke/handlers/field.py` の `らんきりゅう_type_modifier` にバグを発見し修正した。
  - 修正前: エアロック・ノーてんきが場にいてもひこうタイプの弱点軽減（0.5倍）が無条件で適用されていた。
  - 原因: `らんきりゅう`のフィールドハンドラは天候が「らんきりゅう」である間ずっと`EventManager`に登録され続けており、`ON_CALC_DEF_TYPE_MODIFIER`イベント自体はエアロック・ノーてんきによる無効化（`ON_CHECK_WEATHER_ENABLED`）を自動的には考慮しない。同じ天候依存の`はれ_power_modifier`/`あめ_power_modifier`は`battle.weather_for(ctx.defender).name == ""`で明示的にガードしているが、`らんきりゅう_type_modifier`にはこのガードが欠落していた。
  - 修正: 関数冒頭に `if battle.weather_for(ctx.defender).name == "": return HandlerReturn(value=value)` を追加し、はれ/あめと同じパターンでエアロック・ノーてんき下では効果を発動しないようにした。
  - `data/ability.py`（デルタストリームのハンドラ登録）・`data/field/weather.py`（らんきりゅうのエントリ登録）自体には誤りはなかった。
