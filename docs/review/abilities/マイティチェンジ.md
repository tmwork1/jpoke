# マイティチェンジ レビュー結果

## 仕様書
- 変更なし。一次情報（https://wiki.pokemonwiki.com/wiki/マイティチェンジ）と照合し、既存の
  `docs/spec/abilities/マイティチェンジ.md` の内容（効果・発動契機・非発動条件・上書き/コピー/
  スキルスワップ不可・専用特性である旨）はすべて一致していることを確認した。

## 実装計画書
- 変更なし。`docs/plan/abilities/マイティチェンジ.md` の設計方針（`ON_SWITCH_OUT` での
  フォルム更新、`alive` 判定必須、`set_form` の流用）は仕様書と整合しており、実装済みの内容とも
  一致していることを確認した。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `マイティチェンジ_change_form` に、フォルム変化が実際に
  発生したとき（`mon.set_form(...)` が `True` を返したとき）のみ `_announce_ability_triggered`
  を呼び出す修正を追加した。従来はフォルムが変化しても特性発動ログ（`LogCode.ABILITY_TRIGGERED`）
  が一切記録されておらず、計画書の「発動ログは `LogCode.ABILITY_TRIGGERED` を利用する」という
  記述と実装が乖離していた。既存の他特性（いかく等）と同じ `_announce_ability_triggered` ヘルパーを
  再利用し、`mon.ability.revealed` の更新とログ記録を統一した。
- `data/ability.py` 側のハンドラ登録（`Event.ON_SWITCH_OUT`, `subject_spec="source:self"`）は
  仕様・他の同系統特性（ダルマモード、バトルスイッチ、うのミサイル、リミットシールド等）の
  登録パターンと一致しており、変更なし。
- `flags={"uncopyable", "protected", "gas_proof"}` は仕様書の「上書きできない／かがくへんかガスで
  無効化されない／コピーできない」との対応を確認済み。`uncopyable`/`protected` は
  `test_move_a.py` の `うつしえ`・`test_move_sa.py` の `シンプルビーム`/`スキルスワップ` 等の
  汎用パラメタライズドテストで代表特性を用いて検証される設計のため、マイティチェンジ個別の
  追加テストは不要と判断した。`gas_proof` は `tests/abilities/test_gas.py` の一覧に既に含まれて
  いることを確認した。
