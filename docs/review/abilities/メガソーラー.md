## 実装（handlers/ / data/）
以下の2件のバグを修正した（いずれも既存実装のレビューで新規発見）。

1. **ねごと・まねっこ等でのネスト時に天候が永久に「はれ」のまま復元されないバグ**
   - 原因: ねごと/まねっこは選んだ技を `battle.run_move(attacker, ...)` で
     同じ攻撃者に対して再帰的に実行するため、外側の技実行中に内側の技の
     `ON_BEGIN_MOVE`/`ON_END_MOVE` が同じ攻撃者へもう一度発火する。旧実装は
     `ability.state` の `"active"`/`""` のみで管理していたため、内側の `ON_END_MOVE` が
     先に `state = ""` にリセットしてしまい、外側の `ON_END_MOVE` が
     「既に非アクティブだから何もしない」と誤判定し、天候が「はれ」のまま
     復元されなくなっていた。
   - 修正: `Ability.weather_override_depth` で発動階層を数え、深度が0に戻る
     最も外側の呼び出し時のみ本来の天候へ復元するよう変更した。
2. **メガソーラー所持者自身が天候変更技（にほんばれ等）を使うと常に失敗し、
   天候変更が反映されないバグ**
   - 原因: `weather_manager.apply()` は `current_name == name` の場合「既に同じ天候」として
     即座に失敗を返す。メガソーラーが天候を「はれ」に仮想上書きしたままにほんばれを使うと、
     常に「既にはれ」と誤判定されて技が失敗し、`ON_END_MOVE` で本来の天候（例: あめ）へ
     巻き戻されてしまっていた。一次情報「この特性であっても技のにほんばれは使用でき、
     場をにほんばれ状態に変えることができる」と矛盾する挙動だった。
   - 修正: `ctx.move.name` が天候変更技（にほんばれ・あまごい・すなあらし・ゆきげしき・
     さむいギャグ）のいずれかに一致する場合は、`メガソーラー_activate` /
     `メガソーラー_deactivate` の両方で天候の仮想上書き自体をスキップするようにした。

併せて、以下のコード品質改善を行った。
- `handlers/ability.py` のモジュールレベルグローバル辞書 `_メガソーラー_saved`
  （`id(mon)` キー、オブジェクト再利用時の衝突リスクあり）を廃止し、
  `model/ability.py` の `Ability` クラスに `saved_weather_name` / `saved_weather_count` /
  `weather_override_depth` の専用属性を追加した（`cud_chew_item`/`cud_chew_turns`
  （はんすう）と同じ、特性固有の一時状態を `Ability` に持たせる既存の慣習に合わせた）。
  `reset_on_switch_out()` でもこれらをリセットするようにした。
