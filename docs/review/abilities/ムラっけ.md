# ムラっけ レビュー結果

## 仕様書
- 既存の `docs/spec/abilities/ムラっけ.md` を一次情報（ポケモンWiki, `docs/wiki/abilities/ムラっけ.html`）と
  照合したところ、「効果」「特性の仕様」「備考」の内容・記載順とも一致していた。変更なし。

## 実装計画書
- `docs/plan/abilities/ムラっけ.md` を実装・仕様書に合わせて全面的に更新した。
  - 存在しないイベント名 `ON_TURN_END_3` を、実際に登録されている `Event.ON_TURN_END`
    （`docs/spec/turn.md` 掲載の priority=150、かそく/ナイトメア/スロースタート/はんすう等と同一
    priority）に修正した。
  - `subject_spec="source:self"`（`EventContext` を受け取る非攻撃フェーズのハンドラであること）を明記した。
  - 実装方針を実際のハンドラ（`ムラっけ_boost_stats`）の手順（up_candidates/down_candidates の
    選出ロジック、`battle.modify_stats(mon, ..., source=mon, reason="ムラっけ")`、境界チェックは
    `battle.modify_stats` 内部に委譲）に合わせて具体化した。
  - しろいきり/フラワーベール/クリアチャームで下降を防げない仕様について、`source=mon`（自分自身）を
    明示することでこれらのハンドラの「相手由来のみブロック」判定（`ctx.source != ctx.target`）に
    自然に該当しないため実装変更が不要であることを明記した。
  - テスト観点を実装済みテストの内容（+2/-1が別能力になる、全能力+6で下降のみ、全能力-6で上昇のみ）に
    合わせて修正した。
  - 「保留・論点」を解消済みとして更新した（乱数は `battle.random` の既存共通方式を使用）。

## 実装（handlers/ / data/）
- `src/jpoke/data/ability.py` の `ムラっけ` エントリ（`Event.ON_TURN_END`, `subject_spec="source:self"`,
  `priority=150`）と `src/jpoke/handlers/ability.py` の `ムラっけ_boost_stats` を仕様書・計画書と
  照合した結果、以下の点で仕様と一致していることを確認した。変更なし。
  - priority=150 は `docs/spec/turn.md` の記載と一致。
  - +6以上／-6以下の能力は候補から除外しており、「すでに+6の能力を上げたり、-6の能力を下げたりする
    変化は選ばれない」を満たす。
  - 下降候補は上昇で選ばれた能力を除外しており、「1ターンに同じ能力が上下することはない」を満たす。
  - 全能力+6のときは `up_candidates` が空になり上昇のみスキップ、全能力-6のときは `down_candidates` が
    空になり下降のみスキップされる。
  - `source=mon`（自分自身）を明示しているため、しろいきり/フラワーベール/クリアチャームの
    「相手由来のみブロック」判定に該当せず、能力低下を防げない仕様と一致する。
