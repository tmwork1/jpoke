# にげごし レビュー結果

## 仕様書
- `docs/wiki/abilities/にげごし.html`（一次情報）を確認した。にげごしのwiki記事自体は
  「詳細な仕様はききかいひ#特性の仕様を参照」とした上で捕獲時に影響が出やすい効果のみを
  抜粋する形式であるため、`docs/wiki/abilities/ききかいひ.html`（グソクムシャ専用特性
  「ききかいひ」は名前が異なるだけでにげごしと完全に同一効果）の「特性の仕様」節を一次情報
  として全面的に参照した。
- 既存の仕様書（`docs/spec/abilities/にげごし.md`、調査日2026-06-03）は「ききかいひを参照」
  とする簡略な要約のみで、以下が欠落していたため全面的に拡充した。
  - 自己HP消費で不発になる技の具体列挙（みがわり/はらだいこ/のろい(呪い)/ソウルビート）。
    旧仕様書は「こんらん」「いたみわけ」のみ記載していた。
  - 発動の処理順（きのみとの優先順位、連続攻撃技、だっしゅつボタン/だっしゅつパック/
    レッドカードとの同時発動時の順序、ターン終了時処理での交代先繰り出しタイミングなど）
    がまるごと欠落していた。
  - 特性ちからずくに関する記述。一次情報（ききかいひのwiki記事）には「特性ちからずくの
    効果が発動した技のダメージでは発動しない」と書かれているが、これは第七世代からSVまでの
    仕様であり、**Championsではこの制限が撤廃されている**（`docs/spec/abilities/ぎゃくじょう.md`
    ・`docs/spec/abilities/いかりのこうら.md` に記載済みの同種の修正と同じパターン）ことを
    確認し、その旨を明記した。

## 実装（handlers/ / data/）
- `src/jpoke/data/ability.py` の `にげごし` 登録（`Event.ON_HP_CHANGED` /
  `subject_spec="target:self"` / `ききかいひ_switch_on_half_hp`）はききかいひと一致しており
  変更なし。
- `src/jpoke/handlers/ability.py` の `ききかいひ_switch_on_half_hp`（にげごしと共用）に、
  第七世代からSVまでの仕様である「ちからずくの効果が発動した技のダメージでは発動しない」
  という判定ブロックが残っていた（`ctx.source.ability.name == "ちからずく"` かつ
  `secondary_effect` フラグを持つ技かどうかを判定して除外する処理）。
  - **バグ修正**: 上記の「仕様書」節で述べた通り、この制限はChampionsでは撤廃されている
    （ぎゃくじょう/いかりのこうらの各仕様書・実装で既に反映済みの修正と同じ内容）。
    にげごし専用のバグではなく、ききかいひ側の先行レビュー（2026-07-11実施）が
    いかりのこうら（2026-07-10）・ぎゃくじょう（2026-07-11）で確立された「Championsでは
    ちからずく除外が撤廃されている」という知見を見落としたまま実装に混入させたものと判断し、
    にげごしのレビューを機に該当ブロックを削除した（共用ハンドラのためききかいひにも
    同時に適用される）。
  - docstringを、実際の除外条件（`self_attack`/`pain_split`/`self_cost`のみ）と一致するよう
    修正した。
- `self_cost`（みがわり/はらだいこ/のろい(呪い)/ソウルビートの自己HP消費）を除外条件に
  含める修正、および `はらだいこ_apply`/`みがわり_apply`/
  `ソウルビート_pay_hp_and_modify_attacker_stats`（`src/jpoke/handlers/move_status.py`）への
  `reason="self_cost"` 付与は、先行するききかいひ側レビューで既に修正済みであり、にげごしにも
  共用ハンドラ経由でそのまま適用されていることを確認した（追加修正不要）。
