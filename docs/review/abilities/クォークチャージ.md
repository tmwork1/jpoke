## 実装（handlers/ / data/）
- **`Event.ON_SWITCH_IN` の priority を修正**: `src/jpoke/data/ability.py` のクォークチャージが
  `priority=200` で登録されており、`docs/spec/turn.md` が定める `140`（いかく等の入場効果より後、
  しろいハーブ等〈160〉より前）と食い違っていた。`しろいハーブ` を装備した状態で
  バトンタッチにより下降ランクを引き継いで登場すると、本来はランクが戻る前に能力を選ぶべき
  ところ、逆順で発動しランク回復後の値で選んでしまう不具合を確認・修正した
  （`priority=140` に修正）。
- **`Event.ON_ABILITY_ENABLED` の登録漏れを修正**: こだいかっせいには登録されているが
  クォークチャージには無く、かがくへんかガス解除時などに補正状態を再評価する経路が
  欠けていた。こだいかっせいと同じ `priority=200` で追加した。
- **スキルスワップの失敗条件の欠落を修正**: `スキルスワップ_can_apply`
  (`src/jpoke/handlers/move_status.py`) は `protected` フラグと `かがくへんかガス` の
  base_name のみを判定しており、クォークチャージ/こだいかっせいは `uncopyable` フラグしか
  持たないため、使用者・対象のどちらかがこの特性でもスキルスワップが成立してしまっていた
  （仕様書「使用者か対象のどちらかがこの特性であるとき、スキルスワップは失敗する」に違反）。
  `かがくへんかガス` と同様に base_name の個別除外リストにクォークチャージ/こだいかっせいを
  追加した。
- **こんらんの自傷ダメージへの誤補正を修正**: `ability_paradox.py` の
  `apply_atk_modifier` / `apply_def_modifier` が内部技 `"_こんらん"` を除外しておらず、
  攻撃/防御が強化対象のときにこんらんの自傷ダメージにも1.3倍補正がかかっていた
  （仕様書「特性で攻撃/防御が上がってもこんらんのダメージには影響しない」に違反）。
  他の同種修正（こだわりハチマキ・でんきだま等）と同じパターンで `ctx.move.name == "_こんらん"`
  時は補正をスキップするよう修正した。
- **ブーストエナジーの授受防止の判定条件を修正**: `src/jpoke/handlers/item.py` の
  `ブーストエナジー_prevent_item_change` が `mon.ability.name`（無効化中は空文字になる）で
  判定しており、かがくへんかガスで特性が無効化されている間はトリック等でブーストエナジーを
  奪える不具合があった（仕様書「特性が変更されたり、無効化されたりしているときでも…
  ブーストエナジーが渡る効果や、奪う効果は無効となる」に違反）。`mon.ability.base_name`
  （無効化状態に関わらない元の特性名）で判定するよう修正した。
  なお、テツノイワオ/テツノカシラの例外（ひひいろのこどう/ハドロンエンジン未実装のため
  現状のデータでは両者ともクォークチャージのみを持つ）は現時点で再現しようがないため対応しない。
- 上記以外（`refresh_paradox_charge_state` の場条件・アイテム消費判定、
  `modify_speed`、`flags={"uncopyable"}`）は仕様と一致しており変更なし。
