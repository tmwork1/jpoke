# どくげしょう レビュー結果

## 仕様書
- 変更なし。一次情報（`docs/wiki/abilities/どくげしょう.html`）と照合したが、誤り・欠落は見つからなかった。

## 実装計画書
- 修正した。
  - `Event.ON_DAMAGE_HIT`（priority=20）を使う方針だったが、`ON_DAMAGE_HIT` は
    `actual_damage<=0` のとき発火しない（`src/jpoke/core/move_executor.py` で確認）ため、
    仕様書の「HP1でこらえるを使用したときや、みねうちを受けたときなど、0のダメージを
    受けたときは発動する」を満たせない誤りだった。既存の同種特性（こぼれダネ・
    じきゅうりょく）と同じ `Event.ON_HIT` + `ctx.substitute_damage` 判定の方式に修正した。
  - 実装方針・リスクと対策・テスト観点・疑似コードを実装の修正内容に合わせて更新した。

## 実装（handlers/ / data/）
- 修正した。
  - `src/jpoke/handlers/ability.py` の `どくげしょう_set_toxic_spikes`:
    - `Event.ON_DAMAGE_HIT` 前提の実装だったため、HP1こらえる・みねうちなど実HPダメージ0の
      被弾で発動しないバグがあった。`ctx.substitute_damage` によるみがわり除外を追加した
      うえで `Event.ON_HIT` に載せ替えた。
    - どくびしのカウント加算・発動ログ出力を独自に再実装していた箇所
      （`field.count >= 2` チェック、`field.count += 1` + `add_event_log` の手動呼び出し）を、
      技「どくびし」のハンドラ（`どくびし_set_field`）と同じ
      `foe_side.activate("どくびし", 1)` の呼び出しに統一した。`activate()` が内部で
      最大層数チェック・カウント加算・ログ出力を行うため、重複実装を解消した。
  - `src/jpoke/data/ability.py` の `どくげしょう` 登録を `Event.ON_DAMAGE_HIT`
    （priority=20）から `Event.ON_HIT`（priority省略・デフォルト100）に変更し、
    こぼれダネ・じきゅうりょくと同じ理由コメントを追加した。
