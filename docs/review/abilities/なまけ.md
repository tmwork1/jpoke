## 実装（handlers/ / data/）
- `src/jpoke/data/ability.py` の `なまけ` の `Event.ON_TRY_ACTION` ハンドラの `priority` を `10` → `15` に修正した。
  - **発見したバグ**: `なまけ_try_action`（ability, priority=10）と `ねむり_check_action` / `こおり_action`（ailment, priority=10, `src/jpoke/data/ailment.py`）が同点の priority で登録されており、`EventManager._sort_handlers` の安定ソートにより実装上たまたま `なまけ` 側が先に実行される順序になっていた。
    このため、一次情報が明記する「ねむり/こおり状態で行動できない間、第五世代以降はXを消費しない」という仕様に反し、**眠り・氷結で行動できないターンでも `なまけ` の行動可否カウンタ（X）が誤って消費・反転**していた（実機動作を再現するスクリプトで実際に不具合を確認）。
  - 修正として `なまけ` の `ON_TRY_ACTION` priority を `15`（ねむり/こおり判定の `10` より後、PP切れ判定の `20` より前）に変更し、ねむり/こおりが行動不能で `stop_event` する場合に `なまけ_try_action` が一切実行されないようにした。
  - `handlers/ability.py` の `なまけ_init` / `なまけ_try_action` 自体のロジックは仕様通りで変更なし。
  - なお、リチャージ（`Event.ON_TRY_ACTION`, priority=10, `技の反動で動けない`）との組み合わせ（なまけの行動可能ターンにギガインパクト等を使い反動で動けなくなる場合、一次情報は「反動が優先されつつXは消費される」としている）は、`リチャージ_block_action` が `stop_event` する実装のため本修正だけでは正しく再現できない。これは `なまけ` 単体の不具合ではなく `リチャージ` / `きょけんとつげき` 側の設計に踏み込む必要があるため、本レビューでは対応せず計画書に申し送りとして記載した。
