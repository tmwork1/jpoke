## 実装計画書
- 実装方針を修正した。旧記述はふくつのたてパターン（`battle.foe(mon)` の EVA を単純に↓1）を
  参照していたが、実際に必要なのはいかくパターン（発動メッセージは必ず表示し、みがわり状態の
  相手には効果を及ぼさない）である旨を明記した。
- テスト観点に「みがわり状態の相手には効果がない」を追加した。
- 疑似コードを実装の実物（`かんろなミツ_lower_foe_evasion`）に合わせて修正した。

## 実装（handlers/ / data/）
- `handlers/ability.py` の `かんろなミツ_lower_foe_evasion` にみがわり状態チェックが欠落していた
  （仕様書「みがわり状態の相手には効果が無い」に反する）ため、いかくの実装パターンに合わせて
  `foe.has_volatile("みがわり")` のチェックを追加し、みがわり状態のときは `modify_stats` を呼ばずに
  終了するよう修正した。
- あわせて、発動メッセージ（`_announce_ability_triggered`）を `modify_stats` の成否に関わらず必ず
  表示するよう変更した（いかくと同一挙動。クリアボディ等で効果が無くても特性は発動したものとして
  扱われ、メッセージは出る）。
- `data/ability.py` の登録内容（`per_battle_once` フラグ、`Event.ON_SWITCH_IN`、`subject_spec="source:self"`）
  は仕様通りで変更なし。
