## 実装（handlers/ / data/）
- 修正: `src/jpoke/handlers/ability.py` の `ばんけん_boost_atk_on_intimidate`
  （`ON_BEFORE_MODIFY_STAT`）に、いかく発動時の特性バー表示（`_announce_ability_triggered`）が
  抜けていたため追加した。あまのじゃく（発動してもメッセージが流れない）とは異なり、
  ばんけんは「いかくを受けたとき、特性バーが表れて攻撃が上がる」仕様のため、実際に
  `mon.ability.revealed` が `False` のままになっており、観測（`observation_builder.py`）上も
  相手にばんけんの存在が伝わらないバグになっていた。あわせて、こうげきが既に最大ランク（+6）で
  いかくが不発だったときは発動しない仕様（あまのじゃくの同種仕様・ビビリだまの
  `at_limit` 判定と同じロジック）も未実装だったため、`ctx.target.rank["atk"] < 6` の
  ガードを追加した。
- 修正: `src/jpoke/data/ability.py` の `ばんけん` エントリで `Event.ON_TRY_BLOW` が
  きゅうばんと同一の `きゅうばん_block_blow`（`_announce_ability_triggered` を呼ぶ）を
  共用していたため、逆にほえる等を防いだ際に特性バーが表示されてしまうバグになっていた。
  発動演出を行わない専用の `ばんけん_block_blow`（`src/jpoke/handlers/ability.py` に新規追加）に
  差し替えた。レッドカード側（`handlers/item.py` の `_レッドカード_try_force_switch`）は
  もともと `foe.ability.name in ("きゅうばん", "ばんけん")` で発動演出なしに交代のみを
  防ぐ実装になっており、こちらは修正不要だった。
  `flags={"mold_breaker_ignorable"}` は既定済みで、かたやぶり時の無効化も正しく動作する
  ことを確認した。
