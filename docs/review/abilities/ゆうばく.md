# ゆうばく レビュー結果

## 仕様書
- 一次情報（ポケモンWiki `docs/wiki/abilities/ゆうばく.html`）と照合した結果、既存の記述は一致しており誤り・欠落はなかった。
- 備考節に、実装上の既知の制限として「ゴツゴツメット/ジャポのみ/レンブのみとの発動順序」を追記した。
  現在のエンジンではこれらの持ち物のダメージ（`Event.ON_DAMAGE_HIT`）がゆうばくのダメージ
  （`Event.ON_MOVE_KO`）より先に適用されるため、仕様通りの「ゆうばくが先」という順序と逆になる。
  `Event.ON_MOVE_KO` が `Event.ON_DAMAGE_HIT` より後に発火する設計はミイラ等の特性書き換え順序を
  守るために意図的なものであり（`ソウルハート_boost_spa_on_ko` のdocstring参照）、本特性単体の
  レビューでエンジン全体の発火順序を変更するのは影響範囲が広いため見送った。数値上の合計ダメージは
  攻撃者の残りHPで頭打ちになるため変わらないが、片方の効果だけで丁度ひんしになるHPのとき、
  表示される特性バー/メッセージが本来と異なる場合がある。
- 関連項目にゴツゴツメット/ジャポのみ/レンブのみ/しめりけを追記した。

## 実装計画書
- 実装ゴール・実装方針を実際の実装（`is_contact_reaction`によるぼうごパット対応、`reason="ability"`に
  よるマジックガード対応、しめりけ判定）に合わせて具体化した。
- 上記の既知の制限（ゴツゴツメット等との発動順序）と、かたやぶりによるしめりけ無視が
  シングル環境では到達不可能な分岐であること（かたやぶりを持つ攻撃者自身がしめりけを同時に
  持つことはできないため）を「実装上の既知の制限」節として追記した。
- テスト観点を実際に追加したテスト内容に合わせて更新した。
- 疑似コードを実際の実装（`ゆうばく_damage_attacker_on_ko`）に合わせて更新した。

## 実装（handlers/ / data/）
- 変更なし。`handlers/ability.py` の `ゆうばく_damage_attacker_on_ko` と `data/ability.py` の
  登録内容を精査したが、以下の点で仕様と一致しており誤りはなかった:
  - `battle.query.is_contact_reaction(ctx)` によりぼうごパットでの無効化を自動的に反映
  - `reason="ability"` により `マジックガード_ignore_damage` が自動的にダメージを無効化
  - `attacker.ability.enabled` チェックによりかがくへんかガス等でしめりけが無効化されている
    場合は正しくゆうばくが発動する
  - `attacker.fainted` ガードにより、反動等で攻撃者が既にひんしの場合は発動しない
  - `battle.modify_hp` は`みがわり`の身代わりHPを経由せず攻撃者本体のHPを直接変更するため、
    攻撃者がみがわり状態でも本体にダメージが入る仕様と自然に一致
- ゴツゴツメット/ジャポのみ/レンブのみとの発動順序の既知の制限については、上記の通りドキュメント化のみとし、
  コード変更は見送った（詳細は仕様書・計画書参照）。
