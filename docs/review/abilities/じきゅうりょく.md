## 仕様書
- 変更なし。`docs/spec/abilities/じきゅうりょく.md` は一次情報（ポケモンWiki）の内容と一致していることを確認した。

## 実装計画書
- `docs/plan/abilities/じきゅうりょく.md` を修正した。
  - 実装方針を、実際に採用すべき `Event.ON_HIT`（＋ `ctx.substitute_damage` によるみがわり除外）に
    書き換えた。旧記述は `Event.ON_DAMAGE_HIT` としていたが、これは実HPダメージが0以下のとき
    発火しないため、こらえるでHP1のまま耐えたときに発動しない仕様漏れがあった（後述）。
  - 実装ゴール・テスト観点に「こらえるでの0ダメージでも発動」「みがわりでは発動しない」を追加。
  - 変更候補のファイル名・関数名を実際のもの（`tests/abilities/test_ability_sa.py`,
    `じきゅうりょく_boost_B_on_hit`）に修正。
  - 疑似コードを実装と一致させ、リーサル計算（`LethalEvent.ON_HIT`, `じきゅうりょく_boost_def`）の
    節を追加。

## 実装（handlers/ / data/）
- バグを修正した。`src/jpoke/data/ability.py` の `じきゅうりょく` 登録が `Event.ON_DAMAGE_HIT` を
  使用していたが、このイベントは `actual_damage<=0` のとき発火しない
  （`src/jpoke/core/move_executor.py` 参照）。仕様書には「HP1のときにこらえるで攻撃を耐えたとき
  など、0のダメージを受けたときは攻撃を無効化したことにはならず、じきゅうりょくは発動する」と
  明記されており、こらえるで耐えた場合に発動しないバグとなっていた。
  - `こぼれダネ`（同じ「0ダメージでも発動、みがわりのみ除外」仕様を持つ既存特性）と同じパターンに
    合わせ、`Event.ON_HIT` に変更した。
  - `src/jpoke/handlers/ability.py` の `じきゅうりょく_boost_B_on_hit` に
    `if ctx.substitute_damage: return HandlerReturn(value=value)` を追加し、みがわりに阻まれた
    場合は発動しないようにした（タイプ相性0倍・まもる等は `ON_BEFORE_APPLY_MOVE` で `move_applied`
    が False になるため `_execute_hit` 自体に到達せず、`Event.ON_HIT` も発火しないので既存仕様通り
    発動しない）。
- `def` キーによる `battle.modify_stats` 呼び出し・ぼうぎょ最大ランク時の不発動（`modify_stats` が
  偽値を返すことで制御）は元から仕様通りだったため変更なし。
