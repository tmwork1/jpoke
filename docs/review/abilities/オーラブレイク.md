# オーラブレイク レビュー結果

## 仕様書
- 変更なし。一次情報（`docs/wiki/abilities/オーラブレイク.html`）と照合したが、効果（威力補正率
  1.33倍→0.75倍への反転）・特性の仕様（倍率3072/5448、累積しない、登場時アナウンス、かたやぶりで
  オーラブレイクの影響が無視される、ターボブレイズ/テラボルテージはオーラブレイクの影響を受けない）・
  備考（ジガルデ専用特性・とくせいカプセル/パッチで変更不可）まで正確に転記されており、誤り・欠落は
  なかった。

## 実装計画書
- 変更なし。実装方針（`ON_SWITCH_IN`/`ON_ABILITY_ENABLED` でアナウンス、フェアリーオーラ/ダークオーラの
  `ON_CALC_POWER_MODIFIER` 側で威力反転ロジックを持たせる設計）は現在の実装と一致しており、修正不要
  だった。

## 実装（handlers/ / data/）
- 変更なし。`data/ability.py` の `オーラブレイク` は `mold_breaker_ignorable` フラグ・`ON_SWITCH_IN`/
  `ON_ABILITY_ENABLED` のアナウンスハンドラのみを持ち、威力反転ロジックは `handlers/ability.py` の
  `ダークオーラ_boost_power` / `フェアリーオーラ_boost_power` 内で
  `ctx.attacker.ability.name == "オーラブレイク" or ctx.defender.ability.name == "オーラブレイク"` により
  判定している。`GameEffect.name` は無効化中（かがくへんかガス・かたやぶり等）に空文字を返す実装のため、
  オーラブレイクがかたやぶりで無効化された場合は自動的に判定から外れ、フェアリー/あく技の威力が1.33倍
  （通常のオーラ効果）に戻る。これは一次情報の「オーラブレイクが無視されるときは、フェアリー/あく技の
  威力が1.33倍になって使われることになる」と整合している。
- かたやぶり・ターボブレイズ・テラボルテージはいずれも `ON_BEGIN_MOVE`/`ON_END_MOVE` で
  `かたやぶり_disable_foe_ability`/`かたやぶり_restore_foe_ability`（`subject_spec="attacker:self"`）を
  使い、自分が攻撃している間だけ相手（`ctx.defender`）の `mold_breaker_ignorable` フラグ付きの特性を
  無効化する実装になっており、オーラブレイク側にも同フラグが付与済みで整合していた。
  逆にターボブレイズ/テラボルテージ自身には `mold_breaker_ignorable` フラグが付いていないため、
  オーラブレイクの影響（オーラブレイクは他特性を無効化する効果を持たない）を受けない点も一次情報と
  一致していた。
- 3072/5448 の固定小数点倍率は `apply_fixed_modifier`（4096基準）で正しく `apply_fixed_modifier(value, 3072)`
  = 0.75倍、`apply_fixed_modifier(value, 5448)` ≒ 1.330倍として実装されており誤りはなかった。
- なお、かたやぶり攻撃者＋オーラブレイク防御者＋別のオーラ提供特性という3特性が絡む組み合わせは
  単体2体制のシングルバトルでは（1体が同時に2つの特性を持つことはないため）原理的に成立せず、対象範囲
  （チャンピオンズシングルバトル）では検証不要と判断した。

## リーサル計算
- 対象外（`docs/progress/ability.md` の該当行は「リーサル実装」「リーサルテスト」ともに `n/a`）。

  同じダメージ計算ロジックを再利用するため、専用のリーサルハンドラは不要。
