# じしんかじょう レビュー結果

## 仕様書
- 変更なし。`docs/spec/abilities/じしんかじょう.md` の内容を一次情報（ポケモンWiki）と照合し、
  効果・特性の仕様（最大ランク時不発動、複数体撃破時の合算、状態異常/天気/反動/ゴツゴツメット等
  攻撃技以外のダメージでは不発動、ばけのかわのフォルムチェンジ消費ダメージでは発動、一撃必殺技・
  固定ダメージ技での発動可否、ミイラ/さまようたましい/とれないにおいとの優先関係、
  かがくのちから/レシーバー・かがくへんかガスとの相互作用）に誤り・欠落がないことを確認した。

## 実装計画書
- 新規作成（`docs/plan/abilities/じしんかじょう.md` が存在しなかった。`docs/progress/ability.md`
  では計画書列が `x` になっていたが実体が欠落していたため、`くろのいななき.md` の計画書を
  テンプレートとして新規作成した）。
- `Event.ON_MOVE_KO` / `subject_spec="attacker:self"` で実装し、`しろのいななき`（こうげき上昇で
  完全に同一効果）と共通ハンドラ `しろのいななき_boost`（内部で `_boost_on_move_ko` を利用）を
  流用する方針を明記した。priority は `turn.md` に `ON_MOVE_KO` の記載がないため、同一イベントを
  使う既存ハンドラ（しろのいななき・くろのいななき・ソウルハート・ビーストブースト・ゆうばく・
  とびだすなかみ）にならいデフォルト（指定なし）とする根拠を記載した。

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/data/ability.py` の `じしんかじょう` エントリは `Event.ON_MOVE_KO` /
  `subject_spec="attacker:self"` で `h.しろのいななき_boost` を登録しており、仕様・計画書と一致。
- `src/jpoke/handlers/ability.py` の `しろのいななき_boost`（`_boost_on_move_ko` で `{"atk": +1}` を
  適用）は `しろのいななき` と `じしんかじょう` の両方から参照されており、効果が完全一致するため
  共通化は妥当と判断（誤り・欠落なし）。
- `ON_MOVE_KO` は `move_executor` の攻撃ダメージ適用処理内でのみ発火し、相手がひんしになった
  ときだけ発火する実装になっており、「攻撃技によるダメージ以外では発動しない」仕様と整合している。
