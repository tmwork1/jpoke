## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/handlers/ability.py` の `アロマベール_prevent_volatile`
  （共通ヘルパー `_prevent_volatile` を利用）と `src/jpoke/data/ability.py` の
  `ON_BEFORE_APPLY_VOLATILE` 登録・`mold_breaker_ignorable` フラグは仕様書と一致しており、
  誤り・欠落は見つからなかった。
  - `subject_spec="target:self"` は `ON_BEFORE_APPLY_VOLATILE` が
    `EventContext(source=source, target=target)` で発火されることと整合している。
  - かたやぶり側は `ON_BEGIN_MOVE`/`ON_END_MOVE` で `ctx.defender`（攻撃対象）のみの
    アロマベールを一時的に無効化する実装になっており、
    「かたやぶり使用者自身のアロマベールは無視されない」という第八世代以降の仕様と自然に一致する。
