# こんじょう レビュー結果

## 仕様書
- 変更なし（`docs/spec/abilities/こんじょう.md` は一次情報（`docs/wiki/abilities/こんじょう.html`）と照合し、内容が一致していることを確認した）

## 実装計画書
- 新規作成（`docs/plan/abilities/こんじょう.md` が存在しなかったため、仕様書と実装内容をもとに新規作成した。
  `ON_CALC_ATK_MODIFIER` / `ON_CALC_BURN_MODIFIER` は `docs/spec/turn.md` に priority の記載が無いため、
  既存の同種ハンドラ（`はりきり_modify_atk`, `とうそうしん_modify_atk`, `からげんき_ignore_burn_modifier` 等）を
  根拠として明記した）

## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `こんじょう_modify_atk` を修正した。
  - 修正前は `ctx.attacker.ailment.is_active and ctx.move.category == "physical"` のみで判定しており、
    こんらんの自傷ダメージ（内部技 `"_こんらん"`、`category="physical"`）にも誤って1.5倍補正がかかっていた。
  - 仕様書に明記されている「こんらんで自傷ダメージを受けるときは、第五世代以降では特性の効果は無い
    （Champions仕様）」に合わせ、`ctx.move.name != "_こんらん"` の条件を追加した。
    同様のパターンは `こだわりハチマキ_boost_physical`（`item.py`）に既にあり、それを参考にした。
  - `こんじょう_ignore_burn_penalty` は既存実装のままで問題なし（`ON_CALC_BURN_MODIFIER` は
    やけど＋物理技のときのみ意味を持つ値であり、常に4096を返しても他ケースへの副作用が無いため）。
  - `data/ability.py` の登録内容（`ON_CALC_ATK_MODIFIER` priority=デフォルト100、
    `ON_CALC_BURN_MODIFIER` priority=200）は仕様・既存パターンと整合しており変更なし。

## リーサル計算
- 対象外（`docs/progress/ability.md` の「リーサル実装」列が `n/a`。
  `ON_CALC_ATK_MODIFIER` / `ON_CALC_BURN_MODIFIER` は通常のダメージ計算パイプライン内で処理されるため、
  リーサル計算専用のハンドラは不要と判断した。`src/jpoke/handlers/lethal.py` にも該当エントリは無い）

## テスト
- `tests/abilities/test_ability_ka.py` に `test_こんじょう_こんらん自傷ダメージには補正なし` を追加した。
  - どく状態＋こんらん状態のコラッタでこんらんの自傷を発生させ、`atk_modifier == 4096`
    （補正なし）になることを確認する。修正前のコードでは `6144` になり失敗することを確認済み。
  - 既存の `test_こんじょう_行動可能な状態異常で攻撃1_5倍`（どく・もうどく・やけど）は変更なし。
