# ほのおのたてがみ レビュー結果

## 仕様書
- 新規作成（`docs/spec/abilities/ほのおのたてがみ.md`）。一次情報
  （https://wiki.pokemonwiki.com/wiki/ほのおのたてがみ）に基づき、以下を記載した。
  - 効果: ほのおタイプの攻撃技を使うとき、自分のこうげき・とくこうを1.5倍にしてダメージを計算する。
  - 特性の仕様: タイプ一致ボーナスとは補正箇所が異なり両立する／技のタイプが変わった場合は変更後の
    タイプで判定する。
  - 備考: メガカエンジシの専用特性（ポケモンチャンピオンズで新規追加）。同型の特性
    （いわはこび・はがねつかい・りゅうのあぎと・トランジスタ・すいほう）との関係を明記。

## 実装計画書
- 新規作成（`docs/plan/abilities/ほのおのたてがみ.md`）。同型の特性（いわはこび・はがねつかい・
  りゅうのあぎと）の計画書を参考に、`Event.ON_CALC_ATK_MODIFIER`（`subject_spec="attacker:self"`、
  priority は同型ハンドラに合わせデフォルト値）で実装する方針を記載した。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` に `ほのおのたてがみ_modify_atk` を新規実装（既存の
  `_modify_by_move_condition(ctx.move, value, modifier=6144, move_type="ほのお")` を再利用）。
- `src/jpoke/data/ability.py` の `"ほのおのたてがみ": AbilityData()`（空スタブ）を、
  `Event.ON_CALC_ATK_MODIFIER` ハンドラ登録に修正した。
- いずれも五十音ソートスクリプトは実行していない（マージ時に一括整形予定）。

## リーサル計算
- 対象外（`docs/progress/ability.md` のリーサル実装・リーサルテスト列はいずれも `n/a`）。
  `ON_CALC_ATK_MODIFIER` は通常のダメージ計算パイプラインの一部でありリーサル計算でも
  同一ハンドラが使われるため、同型の いわはこび／はがねつかい／りゅうのあぎと と同様に
  専用のリーサルハンドラは不要。

## 付随修正（無関係な既存テストの flaky 修正）
本タスクとは無関係だが、全件テスト実行中に間欠的に失敗する既存テストを2件発見し、原因調査の上で
修正した（急所（クリティカル）判定により低耐久の防御側が一撃で瀕死になり、追加効果
〈ひるみ／やけど〕が発動しないまま `assert` が失敗するケース）。

- `tests/moves_attack/test_move_sa.py::test_ずつき_ひるみが発動する`
  - 防御側を `ピカチュウ`（低耐久）から `カビゴン`（高耐久）に変更し、急所判定でも
    瀕死にならないようにした。
- `tests/moves_attack/test_move_ha.py::test_ひゃっきやこう_やけどが発動する`
  - 防御側を `フーディン`（低耐久）から `バンギラス`（高耐久・ゴースト技を半減できるあく複合）
    に変更した（ノーマル複合ポケモンはゴースト技が無効になるため選択肢から除外した）。
- いずれも修正後に30回連続実行してすべて成功することを確認した。
