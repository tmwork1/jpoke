## 実装（handlers/ / data/）
- **道具交換ブロックの欠落を修正**: `マルチタイプ_block_item_change` は自分の持つプレートの
  奪取・交換のみをブロックしており、仕様書の「アルセウスに対するトリック/すりかえ/ギフトパス、
  またはアルセウスが使うトリックは自分か相手のどちらかがプレートを持っているときのみ失敗する」
  （`EventContext.is_exchange` で判定する設計は ARシステムの先行実装・`context.py` の
  docstring で既に想定されていた）が未実装だった。
  - `handlers/ability.py` の `マルチタイプ_block_item_change` を、`is_exchange=True` の場合に
    相手（`battle.foe(mon)`）の道具もプレートかどうか判定するよう修正（ARシステムの
    `ARシステム_prevent_item_change` と同一パターン）。
  - 修正前は `Pokemon('アルセウス', item_name='')` に対して `battle.item_manager.swap_items()`
    を呼ぶと、相手がプレートを持っていてもアルセウスにプレートが渡ってしまう不具合があった
    （再現・修正確認済み）。
- **プレートアイテムの登録漏れを修正**: `src/jpoke/data/item.py` の `ITEMS` 辞書に
  `せいれいプレート` しか登録されておらず、残り16種（ひのたまプレート・もりのプレート・
  しずくプレート・いかずちプレート・つららのプレート・こぶしのプレート・もうどくプレート・
  だいちのプレート・あおぞらプレート・ふしぎのプレート・たまむしのプレート・がんせきプレート・
  もののけプレート・りゅうのプレート・こわもてプレート・こうてつのプレート）が `ItemName` 型にも
  `ITEMS` 辞書にも存在しなかった。そのため `Pokemon(..., item_name="ひのたまプレート")` は
  `KeyError` になり、マルチタイプの効果を17種のうち1種でしか検証できていなかった
  （ARシステムのメモリ登録漏れと同種の問題。プレートは「対応タイプの技威力を1.2倍にする」という
  マルチタイプ特性とは独立した効果を持つため、`せいれいプレート` と同じ形式
  （`fling_power=90` + `ON_CALC_POWER_MODIFIER` ハンドラ）で16種を追加登録した）。
  - `handlers/item.py` に `<プレート名>_modify_power_by_type` を16件追加（既存の
    `_modify_power_by_type` 共通ヘルパー・`せいれいプレート_modify_power_by_type` と同一パターン）。
  - `data/item.py` の `ITEMS` 辞書に16件追加登録。
  - `python scripts/generate_literals/generate_item_literal.py` を実行して `ItemName` 型を再生成。
  - なげつける時のこわもてプレート特有の補正（あくタイプへの補正が乗り威力108になる仕様）は
    未実装のまま（`せいれいプレート` 含め既存プレートでも実装されていない挙動であり、
    なげつける自体の技タイプが道具のタイプへ変化する仕組みがないため対応していない。
    既知の限定的なギャップとして残す）。
  - 五十音順ソート（`sort_data/sort_items.py` / `sort_handlers.py`）はマージ後に一括整形するため
    今回は未実行。
- 上記以外の既存実装（`マルチタイプ_apply_type`、`_apply_multitype` 共通ロジック、
  `flags={"uncopyable", "protected", "gas_proof"}`、はたきおとす/どろぼう/ほしがる/マジシャン/
  わるいてぐせでの `can_change_item()` 経由の一方的除去防止、さばきのつぶてのタイプ決定、
  みずびたし/まほうのこなの `defender.name in ("アルセウス", "シルヴァディ")` 判定）は
  仕様と一致しており変更なし。
