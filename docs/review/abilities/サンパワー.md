## 実装（handlers/ / data/）
- 通常特性ハンドラ（`サンパワー_modify_atk` / `サンパワー_take_sun_damage`、`Event.ON_TURN_END`
  priority=30）は仕様・計画と一致しており変更なし。
- リーサル計算ハンドラを修正した。
  - `src/jpoke/handlers/lethal.py` の `サンパワー_take_sun_damage()`: 「攻撃者HP未追跡のためスタブ」
    として常に `hp_dist` をそのまま返す no-op になっていたが、`LethalContext.hp_dist` は
    **defender（計算対象ポケモン）のHP分布のみを追跡する**ため、attacker側のサンパワー自傷は
    そもそも `hp_dist` に反映できない（`いのちのたま` の反動が lethal_handlers に登録されていない
    のと同じ理由）。一方で **defender がサンパワーを持つ場合**の自傷は `hp_dist` に直接反映すべき
    効果であり、`かんそうはだ_weather_hp`（`subject="defender"`）と全く同じパターンで実装すべき
    ものだった。`battle.weather_for(ctx.defender).sunny` のとき
    `_damage(hp_dist, max(1, ctx.defender.max_hp // 8))` を適用するよう修正。
  - `src/jpoke/data/ability.py` の `LethalHandler(func=l.サンパワー_take_sun_damage, subject="attacker")`
    を `subject="defender"` に修正（`_get_pokemon_handlers` は `subject` と一致するロールのポケモンが
    そのハンドラを持つ場合のみ拾うため、`subject="attacker"` のままでは defender がサンパワーを
    持っていても永久に発動しない no-op だった）。

## リーサル計算
- 上記の通り新規実装（`-` → `x`）。defender がサンパワーを持ち晴れ状態のとき、ターン終了時の
