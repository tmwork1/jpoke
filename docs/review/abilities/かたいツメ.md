# かたいツメ レビュー結果

## 仕様書
- 変更なし（`docs/spec/abilities/かたいツメ.md` は一次情報（`docs/wiki/abilities/かたいツメ.html`）と
  内容が一致していることを確認した。効果・補正率（5325/4096）・ぼうごパット/パンチグローブ/みがわり/
  ミイラ系特性との相互作用の記述に誤り・欠落なし）

## 実装計画書
- 新規作成（`docs/plan/abilities/かたいツメ.md` が存在しなかったため、`docs/progress/ability.md` の
  計画書列が `x` になっていたのに実体がない状態だった。既存実装・仕様書をもとに新規作成した。
  実装方針は「てつのこぶし」「がんじょうあご」「パンクロック」「メガランチャー」「きれあじ」と
  同一パターン（`ON_CALC_POWER_MODIFIER` での固定小数点威力補正）であることを明記した）

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/handlers/ability.py` の `かたいツメ_modify_power` は
  `battle.query.is_contact(ctx)` で接触判定を行い、`apply_fixed_modifier(value, 5325)` を
  適用しており仕様書と一致する。`is_contact()` は `Event.ON_CHECK_CONTACT` を経由するため、
  パンチグローブ所持時のパンチ技非接触化やぼうごパットの影響を受けない仕様も自動的に満たされている。
  `data/ability.py` の登録（`subject_spec="attacker:self"`, 既定priority=100）も
  同種ハンドラと一致しており妥当。

## リーサル計算
- 対象外（`docs/progress/ability.md` の「リーサル実装」「リーサルテスト」列は `n/a`。
  威力補正はダメージ計算に直接組み込まれるため専用のリーサルハンドラは不要で、
  同種の威力上昇特性（がんじょうあご・きれあじ・てつのこぶし・パンクロック・メガランチャー）も
  同じく `n/a` になっており整合している）
