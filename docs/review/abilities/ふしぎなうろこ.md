## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `ふしぎなうろこ_boost_B` を修正。
  - 修正前は状態異常中かつ物理ダメージ判定 (`deals_physical_damage`) のみを条件にしており、
    こんらんの自傷ダメージ（内部技 `_こんらん`）中に自身が状態異常であれば誤って1.5倍の防御補正が
    乗ってしまうバグがあった（仕様書に明記された「こんらんの自傷ダメージには第五世代以降で効果が無い」
    という記述に反する）。同種の防御バフ特性 `くさのけがわ_boost_B` 等では
    `ctx.move.name != "_こんらん"` の除外条件が既に付いていたのに対し、`ふしぎなうろこ_boost_B` にだけ
    抜けていた。
  - `ctx.move.name != "_こんらん"` の条件を追加して修正した。
  - `data/ability.py` 側の登録（`Event.ON_CALC_DEF_MODIFIER` / `subject_spec="defender:self"` /
    `flags={"mold_breaker_ignorable"}`）は仕様・計画と一致しており変更なし。
  - シェルアームズの分類判定への非干渉、ワンダールームでの特防1.5倍化については、既存の
    `deals_physical_damage` / `ワンダールーム_def_modifier` との組み合わせで自動的に成立しており、
    追加実装は不要と確認した。
