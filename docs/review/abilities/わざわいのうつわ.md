## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `わざわいのうつわ_reduce_C` に技の分類判定
  （`ctx.move.category == "special"`）が欠落していたバグを修正した。
  - 修正前は分類を判定せず、防御側がこの特性を持つ場合、物理技であっても攻撃側の
    攻撃補正を0.75倍にしてしまっていた（本来は特攻のみが対象で、物理技には無関係）。
  - `わざわいのおふだ`（物理版・こうげき対象）等の兄弟特性で `ctx.move.category` に
    相当する判定が既に他の特性（サンパワー・ちからもち等）で標準的に行われているのに対し、
    わざわいのうつわ側だけこの判定が抜けていた。
  - `data/ability.py` 側の登録（`Event.ON_CALC_ATK_MODIFIER`, `subject_spec="defender:self"`）
    自体は仕様と整合しており変更なし。
  - `AbilityData.flags` に `mold_breaker_ignorable` が付与されていないこと（＝かたやぶりで
    無視できない仕様どおり）を確認し、変更なし。

  ログ: `.loop/test_logs/わざわいのうつわ.log`

## 追記（わざわいのおふだ レビュー時の連鎖修正）
- `わざわいのおふだ`のレビューで「複数体のわざわい系特性所持者が対面した場合、攻撃側自身も
  わざわい系特性を持っていれば無効化されるべき」という仕様漏れが発覚し、`わざわいのうつわ`にも
  同一パターンの欠落（`ctx.attacker.ability.name != "わざわいのうつわ"` 判定なし）があることを
  確認したため、統合ブランチ上でその場修正した。
  - `わざわいのうつわ_reduce_C` に `ctx.attacker.ability.name != "わざわいのうつわ"` を追加。
  - `test_わざわいのうつわ_持ち同士では互いの特攻が下がらない` を回帰テストとして追加。
