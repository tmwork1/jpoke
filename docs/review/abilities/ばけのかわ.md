## 実装（handlers/ / data/）
- 欠落を修正: こんらんによる自傷ダメージ（`Event.ON_MODIFY_NON_MOVE_DAMAGE`,
  `hp_change_reason == "self_attack"`）に対してばけのかわが発動していなかった
  （`Event.ON_MODIFY_MOVE_DAMAGE` のみ登録されており、仕様書「こんらん時の
  自分への攻撃には発動し、ダメージを防ぐ」を満たしていなかった）。
  - `src/jpoke/handlers/ability.py` に `ばけのかわ_block_confusion_damage` を
    追加（がんじょう特性の `がんじょう_survive_confusion_damage` と同じ
    パターン）。こんらんの自傷ダメージを0にし、最大HPの1/8を消費して
    `consumed` 理由でこのバトル中の再発動を止める。
    通常ヒット時のハンドラと異なり `hits_taken` は加算しない
    （こんらんで自傷した場合はそのターンの行動自体が失敗するため、
    ダメおし・きあいパンチの判定対象にならない）。
  - `src/jpoke/data/ability.py` の `ばけのかわ` に
    `Event.ON_MODIFY_NON_MOVE_DAMAGE`（`subject_spec="target:self"`）を追加登録。
- それ以外（`Event.ON_MODIFY_MOVE_DAMAGE` 側の通常ヒット処理、
  `flags`＝`uncopyable`/`protected`/`per_battle_once`/`mold_breaker_ignorable`/
  `gas_proof`）は仕様書・既存の汎用フラグ基盤と照合し、誤りなし。変更なし。

## リーサル計算
- 対象列は「-」（未実装）と記載されていたが、実際には
  `handlers/lethal.py` の `ばけのかわ_block_damage`
  （`LethalEvent.ON_BEFORE_HIT`, `subject="defender"`）がすでに実装済みだった
  （進捗表が更新されていなかった）。
  - 実装内容を仕様書・`.claude/loop/_common.md` §共通11 と照合: 攻撃ダメージ分布を
    0に上書きし、`ability_enabled` な状態のみ最大HPの1/8消費・
    `ability_enabled=False` に遷移させる実装で、正しく仕様（1発目のみ無効化・
    以後は通常ダメージ）を反映していた。誤り・欠落なし。変更なし。
  - こんらんの自傷ダメージはリーサル計算（攻撃側のダメージ交換系列の期待値計算）の
    対象外とし、追加実装は行わなかった（防御側の状態異常起因の自傷はリーサル計算の
    既存方針でも対象外）。
