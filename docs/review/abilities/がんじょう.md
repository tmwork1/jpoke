# がんじょう レビュー結果

## 仕様書
- 変更なし。`docs/wiki/abilities/がんじょう.html`（一次情報）と照合し、効果・特性の仕様（一撃必殺技無効化、
  HP1残し補正、かたやぶり無効、こんらん自傷ダメージへの適用、みちづれ/ほろびのうた/自滅技/反動での不発、
  ヌケニン特例、きあいのタスキ・こらえるとの優先順など）はすべて記述済みで一致していることを確認した。

## 実装計画書
- 全面的に更新した（旧版は存在しない `move_executor.py` 変更や誤った実装方針を記載しており、
  実際のイベント登録と乖離していたため）。
  - HP1残し補正を「技によるダメージ（`ON_MODIFY_MOVE_DAMAGE`）」と「こんらんの自傷ダメージ
    （`ON_MODIFY_NON_MOVE_DAMAGE`, `reason="self_attack"`）」の2経路に分けて明記した。
  - 一撃必殺技無効化は `Event.ON_TRY_MOVE_2`（priority=140、`docs/spec/turn.md` 準拠）を使う根拠を明記し、
    誤って `Event.ON_BEFORE_APPLY_MOVE`（命中判定より後）を使わないよう注記した。
  - きあいのタスキとの優先順がハンドラ登録順（特性→アイテム）で保証されることを明記した。

## 実装（handlers/ / data/）
- **バグ修正**: `がんじょう_block_ohko`（一撃必殺技無効化）が `Event.ON_BEFORE_APPLY_MOVE`
  （命中判定の後に発火）に登録されていた。`docs/spec/abilities/がんじょう.md` は「一撃必殺技を受けた
  ときは、命中判定が行われる前にがんじょうで無効化される」と明記しており、`docs/spec/turn.md` でも
  がんじょうの一撃必殺無効は `Event.ON_TRY_MOVE_2`（priority=140、命中判定Interruptより前）に
  分類されている。旧実装のままだと、一撃必殺技の命中率ロール（じわれ等は命中30%）でMISS判定された
  場合にがんじょうが発動せずMOVE_MISSEDになってしまい、余計な乱数消費も発生する不具合があった。
  `Event.ON_TRY_MOVE_2`（priority=140）に登録し直した。
  （`src/jpoke/data/ability.py`, `src/jpoke/handlers/ability.py`）
- **欠落追加**: こんらんの自傷ダメージ（`handlers/volatile.py` の `こんらん_try_action` が
  `battle.modify_hp(..., reason="self_attack")` で直接HPを減らす経路）は技のダメージ計算
  （`Event.ON_MODIFY_MOVE_DAMAGE`）を通らないため、旧実装ではHP満タン時でもがんじょうがこんらん
  自傷を耐えられなかった。仕様書「こんらんの自傷ダメージも耐えることができる」に反していたため、
  `Event.ON_MODIFY_NON_MOVE_DAMAGE`（`subject_spec="target:self"`, `reason=="self_attack"`限定）に
  新規ハンドラ `がんじょう_survive_confusion_damage` を追加した。
  だいばくはつ等の自滅（`self_cost`）・反動（`recoil`）・みちづれ/ほろびのうた（`perish`）は
  対象外のまま維持している（仕様どおり発動しない）。
- `がんじょう_survive_lethal` のdocstringが誤ったイベント名・subject_spec
  （`ON_BEFORE_DAMAGE_APPLY` / `target:self`）を記載していたため、実際の登録
  （`ON_MODIFY_MOVE_DAMAGE` / `defender:self`）に合わせて修正した。
- きあいのタスキとの優先順（`Event.ON_MODIFY_MOVE_DAMAGE` の同一priority・同一速度でのタイブレーク）は
  ハンドラ登録順（`switch_manager.py` で特性→アイテムの順に登録される）により、がんじょうが常に
  先に処理されることを確認した（既存実装のまま、変更不要）。

## リーサル計算
- 変更なし。`handlers/lethal.py` の `がんじょう_survive_lethal`（`LethalEvent.ON_APPLY_DAMAGE`,
  `subject="defender"`）は満タン枝限定でHP0をHP1に補正し、`consume="ability"` により何度でも発動する
  正しい実装であることを確認した。`_get_pokemon_handlers` が特性→アイテムの順でハンドラを取得するため、
  きあいのタスキとの優先順（がんじょうが先）もリーサル計算側で保証されている。
  リーサル計算エンジンはこんらんによる行動不能を確率分岐としてモデル化していないため、こんらん自傷用の
  リーサルハンドラは対象外（不要）と判断した。

## テスト
- `tests/abilities/test_ability_ka.py` を修正・追加した。
  - `test_がんじょう_一撃必殺技を無効化する`: イベント変更に伴い `move_applied` ではなく
    `move_success` を検査するよう修正（`move_applied` は `ON_TRY_MOVE_2` で失敗した場合セットされず
    `None` のまま残るため）。
  - `test_がんじょう_一撃必殺技は命中判定前に無効化される`（新規）: 命中率30%のじわれに対し
    `fix_random` で「本来なら外れる」乱数を固定し、それでもMISSにならずがんじょうが発動することを
    確認する（今回のイベント登録バグ修正の回帰防止）。
  - `test_がんじょう_こんらんの自傷ダメージでもHP1残る`（新規）: こんらん自傷を強制発動させ、
    `fix_damage` で致死量にしてもHP1で耐えることを確認する（新規ハンドラの動作確認）。
  - `test_がんじょう_連続技の1発目で発動すると2発目は不発`（新規）: ダブルアタック（2回攻撃技）で
    毎ヒット致死量を与え、1発目でHP1残し後、2発目で（HPが満タンでなくなるため）発動せずひんしに
    なることを確認する。
  - `test_がんじょう_きあいのタスキと併せ持つときはがんじょうが優先`（新規）: 実戦闘のイベント経路で
    がんじょうが先に発動し、きあいのタスキが消費されないことを確認する
    （`tests/test_lethal.py` はリーサル計算専用の確認のため、実戦闘側でも別途担保する）。
- `python -m pytest tests/ -v` を実行し、4756件全てパスすることを確認した
  （ログ: `.loop/test_logs/がんじょう.log`）。
