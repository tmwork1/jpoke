## 実装（handlers/ / data/）
- 変更なし（`パンクロック_modify_power` / `パンクロック_reduce_damage`
  （`src/jpoke/handlers/ability.py`）は `ctx.move.has_flag("sound")` 判定と
  `apply_fixed_modifier` による5325/4096倍・2048/4096倍の補正を正しく実装済み。
  `src/jpoke/data/ability.py` の `パンクロック` エントリも `ON_CALC_POWER_MODIFIER`
  (attacker:self) / `ON_CALC_DAMAGE_MODIFIER` (defender:self) を正しく登録済み。
  `flags={"mold_breaker_ignorable"}` はダメージ計算内部の
  `かたやぶり_disable_foe_ability` / `きんしのちから_disable_foe_ability` が
  防御側の特性を丸ごと無効化する既存パターンと同一で、こおりのりんぷん等の
  他の被ダメ軽減系特性と整合している。威力補正側（attacker:self）は防御側の特性無効化と
  無関係のスコープでしか発火しないため、フラグをアビリティ全体に付与しても
  誤って威力補正まで無効化される事故は起きないことを確認した。
  ON_CALC_POWER_MODIFIER / ON_CALC_DAMAGE_MODIFIER はダメージ計算内部でのみ発火し
  変化技では発火しないため、変化技の除外仕様も実装上自然に満たされている）。
