## 実装計画書
- `実装方針` / `疑似コード` を実態に合わせて修正した。
  - 急所判定は `ctx.critical` ではなく `battle.move_executor.critical` を参照する理由を明記した。
    `AttackContext.critical` はダメージ計算専用に `damage.py` 内で生成される別インスタンスにのみ設定され、
    `move_executor._execute_hit` が保持し `ON_DAMAGE_HIT` へ渡すメインの `ctx` には反映されない
    （常に `False` のまま）ため、`ctx.critical` を参照するとメインの `ctx` の急所判定を取得できない。
  - `変更候補` のハンドラ関数名・テストファイルパスを実際の実装（`いかりのつぼ_max_atk_on_crit`、
    `tests/abilities/test_ability_a.py`）に合わせて修正した。

## 実装（handlers/ / data/）
- 変更なし。以下を確認した。
  - `handlers/ability.py` の `いかりのつぼ_max_atk_on_crit` は `battle.move_executor.critical` で急所を判定し、
    `6 - mon.rank["atk"]` の差分を `battle.modify_stats` で適用している。差分が0以下（既に+6）の場合は
    何もせず、`modify_stats` が実際に変化した場合のみ `_announce_ability_triggered` で発動を通知する。
  - `data/ability.py` の登録は `Event.ON_DAMAGE_HIT` / `subject_spec="defender:self"` / `priority=20` で、
    `docs/spec/turn.md` の ON_DAMAGE priority 20「いかりのつぼ: 急所に当たった」と一致している。
  - みがわり・クリアスモッグ／ダイドラグーン・シャドースチール・コアパニッシャー・かがくへんかガス等の
    エッジケースはいずれもイベント発火の仕組み（みがわりが吸収した場合は `ON_DAMAGE_HIT` 自体が発火しない、
    priority 順序、特性無効化の共通機構）で自然に仕様どおりの挙動になっており、ハンドラ側に追加の分岐は不要。
  - いかりのこうら（固定量 +1/-1）との実装パターンの違い（本特性は `6 - rank` の差分計算による
    最大ランクへの一括上昇）を計画書・実装ともに確認済み。
