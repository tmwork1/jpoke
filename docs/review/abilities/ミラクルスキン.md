# ミラクルスキン レビュー結果

一次情報: https://wiki.pokemonwiki.com/wiki/ミラクルスキン
（`docs/wiki/abilities/miracle_skin.html` に保存して照合）

## 仕様書
- 変更なし（既存の `docs/spec/abilities/ミラクルスキン.md` は一次情報の内容と一致していることを確認した）

## 実装計画書
- 命中率スケールの記述を修正した。旧記述は「4096スケールで50%相当の`2048`を上限とする」と
  なっていたが、実際の `move_executor._check_hit` はパーセント表記（0〜100）で命中率を扱っており、
  実装（`min(value, 50)`）とも一致しないため、実態に合わせて修正した。
- `かたやぶり` 無効化の仕組みについて、ハンドラ側では判定せず
  `かたやぶり_disable_foe_ability`（`Event.ON_BEGIN_MOVE`）が `mold_breaker_ignorable`
  フラグを持つ特性を技実行中に無効化する既存の仕組みに乗っている旨を明記した。
- 検証コマンド・変更候補に記載されていた古いテストファイルパス（`tests/test_ability.py`）を
  実際のパス（`tests/abilities/test_ability_ma.py`）に修正した。
- 疑似コードを実装の実態（`ctx.move.is_attack` と `value is None` の判定、`min(value, 50)`）に
  合わせて修正した。

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/handlers/ability.py` の `ミラクルスキン_reduce_accuracy` と
  `src/jpoke/data/ability.py` の `ミラクルスキン` エントリ（`ON_MODIFY_ACCURACY` ハンドラ登録、
  `mold_breaker_ignorable` フラグ）を仕様書・計画書と照合し、以下を確認した。誤り・欠落はなかった。
  - 攻撃技（`ctx.move.is_attack`）・必中技（`value is None`）は対象外。
  - `min(value, 50)` により変化技の命中率が50を超える場合のみ50に固定される
    （50以下の変化技には影響しない）。
  - `かたやぶり` による無効化は既存の `mold_breaker_ignorable` フラグの仕組みで実現されており、
    ハンドラ側に重複ガードは不要。
  - `subject_spec="defender:self"` は `AttackContext` のロールと一致している。

## リーサル計算
- 対象外（`docs/progress/ability.md` のリーサル実装列が `n/a`。ミラクルスキンは相手の変化技の
  命中率にのみ影響し、ダメージ量・瀕死判定には影響しないため妥当）

## テスト
- 既存の `test_ミラクルスキン_変化技の命中率を50パーセントに固定する`
  （`tests/abilities/test_ability_ma.py`）を確認し、誤りはなかった。
- `かたやぶり` によるミラクルスキン無効化のテストが存在しなかったため、
  `test_ミラクルスキン_かたやぶりで無効化される` を追加した
  （他の `mold_breaker_ignorable` 特性の慣例に合わせ、かたやぶり持ちの攻撃側がどくどくを
  使用した際に命中率が50に固定されず本来の90のままであることを確認する）。
- `からぶりほけん` との相互作用は `tests/items/test_item_ka.py` の既存の汎用テスト
  （`accuracy=0` で技を外した場合に発動する一連のテスト）で命中判定失敗要因を問わず
  カバーされているため、ミラクルスキン専用のテストは追加しなかった。
- `python -m pytest tests/ -v` で全5562件（+スキップ1件）が通過することを確認した。
