# ふかしのこぶし レビュー結果

## 仕様書
- 変更なし（`docs/spec/abilities/ふかしのこぶし.md` は一次情報
  （https://wiki.pokemonwiki.com/wiki/ふかしのこぶし ）と照合済み。Champions での効果・特性の仕様
  （貫通できる状態の一覧、ダイウォール非対応、ゴーストダイブ/シャドーダイブ例外、ぼうごパット、
  パンチグローブ、ミイラ/さまようたましい/とれないにおいの連続攻撃技エッジケース等）はすべて
  一次情報と一致していた）。

## 実装計画書
- `docs/plan/abilities/ふかしのこぶし.md` を修正した:
  - 実装ゴールの貫通対象揮発性状態を「まもる・トーチカ・キングシールド・スレッドトラップ・
    かえんのまもり」の5種から「まもる・トーチカ・キングシールド・スレッドトラップ・かえんのまもり・
    ニードルガード・ファストガード」の7種に修正（仕様書に記載のこの2種が実装で漏れていたため）。
    仕様書にあるブロッキング/ワイドガード/たたみがえしは本プロジェクト未実装（ワイドガードは
    スプレッド技対象範囲を未モデル化のため実質ノーガード）である旨を明記。
  - `_PROTECT_VOLATILES` 定数の説明を「5種」→「7種」に更新。
  - テストファイルの参照を実体である `tests/abilities/test_ability__group.py` に修正
    （旧記載の `tests/test_ability.py` は存在しない）。
  - 更新日を 2026-07-12 に更新。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `_PROTECT_VOLATILES` に `"ニードルガード"`, `"ファストガード"`
  を追加した。仕様書（一次情報）はこの特性が貫通できる状態として
  まもる/ニードルガード/トーチカ/キングシールド/ブロッキング/スレッドトラップ/かえんのまもり/
  ファストガード/ワイドガード/たたみがえしを挙げているが、実装済みの `_PROTECT_VOLATILES` には
  ニードルガードとファストガードが漏れていた。これにより貫通（`ON_CHECK_PROTECT`）自体は
  `battle.query.is_contact` のみで判定されるため機能していたが、ダメージを1/4にする
  `ON_CALC_PROTECT_MODIFIER`（`ふかしのこぶし_reduce_damage`）が発動せず、本来1/4であるべき
  ダメージが等倍になるバグがあった。この2つを追加して修正。
  （ブロッキング/ワイドガード/たたみがえしは本プロジェクトで未実装のため対象外のまま）
- `src/jpoke/data/ability.py` は変更なし（ハンドラ登録は仕様通り）。
- かんつうドリル（同一ハンドラを共有）にも同じ修正が反映される。

## リーサル計算
- 対象外（`docs/progress/ability.md` の「リーサル実装」列が `n/a`）。

## テスト
- `tests/abilities/test_ability__group.py` の `test_ふかしのこぶし_接触技でまもるを貫通` を修正:
  - パラメータに `move_name` 軸を追加し、`("ふかしのこぶし", "ニードルガード", "たいあたり")` と
    `("ふかしのこぶし", "ファストガード", "マッハパンチ")` のケースを追加
    （ファストガードは優先度1以上の技のみブロックするため、優先度+1の接触技マッハパンチを使用）。
  - いずれも `battle.damage_calculator.protect_modifier == 1024`（1/4ダメージ）を検証する。

## 進捗表の修正
- `docs/progress/ability.md` の「ふかしのこぶし」行の「効果」列が
  「相手のまもる状態を無視して攻撃できる」となっており、Champions での実際の効果（1/4ダメージで
  貫通する）を反映していなかった（隣接する「かんつうドリル」行の記述と不整合だった）ため、
  「接触技を使うとき、相手のまもる状態を無視して本来の1/4のダメージを与える
  （かんつうドリルと同じ効果）」に修正した。
