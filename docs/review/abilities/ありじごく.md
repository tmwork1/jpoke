# ありじごく レビュー結果

## 仕様書
- 変更なし。`docs/spec/abilities/ありじごく.md` は既に存在し、一次情報（ポケモンWiki）と照合した結果、
  効果・特性の仕様の記述に誤り・欠落は見つからなかった。

## 実装計画書
- 新規作成。`docs/plan/abilities/ありじごく.md` が存在しなかったため、仕様書と実装
  （`ON_CHECK_TRAPPED` ハンドラ、`subject_spec="source:foe"`）をもとに新規作成した。
  `かげふみ`・`じりょく` と同系統の trapped 系実装であることを明記した。

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/handlers/ability.py` の `ありじごく_check_trapped` は
  「相手がゴーストタイプでなく、かつ `battle.query.is_floating(source)` が False（地面にいる）」の場合に
  交代を制限しており、仕様書と一致していることを確認した。
  `battle.query.is_floating` は `ON_CHECK_FLOATING` を経由してひこうタイプ・ふゆう等の浮遊判定を
  共通で行うため、ありじごく側で個別に判定ロジックを持つ必要がないことも確認した。
  `src/jpoke/data/ability.py` の登録（`Event.ON_CHECK_TRAPPED`, `subject_spec="source:foe"`）も
  `かげふみ`・`じりょく` と同一パターンで問題なし。

## リーサル計算
- 対象外。`docs/progress/ability.md` のリーサル実装列は `n/a`（交代制限効果はダメージ計算に影響しない
  ため対象外）。`handlers/lethal.py` にも該当ハンドラが存在しないことを確認した。

## テスト
- 追加。`tests/abilities/test_ability__group.py` の `test_交代抑制特性_param` に
  `("ありじごく", "ゲンガー", "", True)`（ゴーストタイプの相手は交代できる）のケースを追加し、
  第六世代以降の仕様（ゴーストタイプに対して無効）をカバーした。
  ふゆう特性等の浮遊判定は `battle.query.is_floating` の共通テスト（`tests/abilities/test_ability_ha.py`
  の `test_ふゆう_浮いている` 等）で既にカバーされているため重複追加はしなかった。
  `python -m pytest tests/ -v` を実行し、4670 passed, 1 skipped で全件パスすることを確認した
  （ログ: `.loop/test_logs/ありじごく.log`）。
