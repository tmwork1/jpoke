# りゅうのあぎと レビュー結果

## 仕様書
- 既存の仕様書（`docs/spec/abilities/りゅうのあぎと.md`）を一次情報
  （https://wiki.pokemonwiki.com/wiki/りゅうのあぎと）と照合した。効果（ドラゴンタイプの攻撃技で
  こうげき・とくこうを1.5倍）、特性の仕様（`はがねつかい#特性の仕様を参照。` という同型特性
  参照形式の内容 — STABとは補正箇所が異なり両立する／タイプが変わる技で変更後がドラゴンタイプ
  なら発動）、備考（レジドラゴの専用特性、対となるレジエレキの専用特性トランジスタとの倍率差）の
  いずれも一次情報の内容と一致しており、誤りはなかった。
- 「関連項目」が空欄だった（同型特性である いわはこび／トランジスタ のレビュー時と同様の欠落）ため、
  同型の特性 いわはこび・はがねつかい・ほのおのたてがみ へのリンクを追加した。

## 実装計画書
- `docs/plan/abilities/りゅうのあぎと.md` が存在しなかったため新規作成した。
- 実装は `Event.ON_CALC_ATK_MODIFIER`（`subject_spec="attacker:self"`、priorityは同型ハンドラに
  合わせた既定値100）で、ドラゴンタイプの攻撃技のときに `apply_fixed_modifier(value, 6144)` を
  適用するパターンであることを明記した。これは同型特性の `いわはこび`・`はがねつかい`・
  `ほのおのたてがみ` と同じ実装パターンであり、共通ヘルパー `_modify_by_move_condition` を
  再利用している。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `りゅうのあぎと_modify_atk`、`src/jpoke/data/ability.py` の
  `りゅうのあぎと` 登録内容（`Event.ON_CALC_ATK_MODIFIER`、`subject_spec="attacker:self"`）を
  仕様書・計画書と照合した結果、誤り・欠落はなかった（変更なし）。
- 同型特性（`いわはこび` / `はがねつかい` / `ほのおのたてがみ` / `トランジスタ`）と同一の共通
  ヘルパー `_modify_by_move_condition` を使っており、実装パターンとして一貫している。

## リーサル計算
- 対象外（`docs/progress/ability.md` のりゅうのあぎと行のリーサル実装・リーサルテスト列は
  いずれも `n/a`）。`ON_CALC_ATK_MODIFIER` は通常のダメージ計算パイプラインの一部であり
  リーサル計算でも同一ハンドラが使われるため、同型の いわはこび／はがねつかい／
  ほのおのたてがみ／トランジスタ と同様に専用のリーサルハンドラは不要。
  `handlers/lethal.py` にも該当ハンドラが存在しないことを確認した。

## テスト
- `tests/abilities/test_ability__group.py::test_タイプ強化系` にて
  `("りゅうのあぎと", "りゅうのはどう", 6144)` のケースで既にカバーされており、`atk_modifier` が
  期待通り 6144（1.5倍）になることを確認した。同型特性（いわはこび・はがねつかい・
  ほのおのたてがみ・トランジスタ）も同様にこのグループテストのみで検証する方針が確立されており、
  扱いに一貫性がある。追加の修正は不要だった（変更なし）。
- `python -m pytest tests/ -v` を実行し、全5711件（1件skip）がパスすることを確認した
  （ログ: `.loop/test_logs/りゅうのあぎと.log`）。今回の対象と無関係な flaky テストは見つからな
  かった。
