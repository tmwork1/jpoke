## 実装計画書
- 内容を更新した（更新日: 2026-06-05 → 2026-07-12）。
  - 対象を「シングルでは攻撃者のみ」という限定的な書き方から、「自分以外のポケモン＝シングルでは常に攻撃者」という正確な説明に修正した。
  - `battle.modify_stats(..., ctx=ctx)` という現行 API に存在しない引数を使った誤った疑似コードを、実装済みコードと一致する疑似コードに修正した。
  - みがわり状態の攻撃者には発動しないことを実装方針に明記した。
  - クリアボディ/しろいけむり/しろいきり/ミラーアーマー等によるブロック・反射は `Event.ON_BEFORE_MODIFY_STAT`（target:self）側の既存汎用ハンドラで処理され、わたげ側での個別対応が不要であることを明記した。
  - かたやぶり経由でわたげが発動した際に攻撃者自身のクリアボディ/あまのじゃく/たんじゅん等を無視する挙動、およびすりぬけ経由でしろいきりを無視する挙動は、現行アーキテクチャでは対象側（攻撃者）への無効化理由の伝播経路がなく、かつゲーム内でほぼ再現不可能な組み合わせ（かたやぶり所持者が同時にクリアボディ等を持つことは通常起こり得ない）であるため、本実装では対象外である旨を明記した。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `わたげ_lower_spd_on_hit` を修正した。
  - 到達不能な `if attacker is None:` の防御的ガードを削除した（`AttackContext.attacker` は `Pokemon`（Optional でない）ため常に非 None）。
  - 一次情報の特性の仕様「攻撃してきたポケモンがみがわり状態であるときも、わたげの効果を受けない」を実装していなかったバグを修正し、`attacker.has_volatile("みがわり")` のとき早期リターンするようにした（ぬめぬめ/カーリーヘアーはこの制限が無い点で異なるため、既存の類似特性からのコピーでは再現できていなかった）。
- `data/ability.py` の `わたげ` エントリ自体は変更なし（`Event.ON_DAMAGE_HIT` / `subject_spec="defender:self"` は仕様・アーキテクチャに合致していることを確認）。
- 以下の点は既存の汎用アーキテクチャで自動的に正しく処理されることを確認済み（わたげ側の個別実装は不要）:
  - タイプ相性・まもる等での無効化時に `ON_DAMAGE_HIT` が発火しない（`actual_damage <= 0` のとき発火しない）。
  - 自身がみがわり状態で技を防いだ場合に `ON_DAMAGE_HIT` が発火しない（みがわりが被ダメージを 0 にするため）。
  - ひんしになった場合も `ON_DAMAGE_HIT` は正常に発火する（`ON_MOVE_KO` より先）。
  - クリアボディ/しろいけむり/フラワーベール/メタルプロテクト/しろいきりによるブロック、ミラーアーマーによる反射。
  - かたやぶりの攻撃を受けてもわたげ自体は発動する（わたげに `mold_breaker_ignorable` フラグが付いていないため）。
  - 敵のわたげでまけんき/かちきが発動する（`ON_BEFORE_MODIFY_STAT` の `source`/`target` の敵味方判定による）。
