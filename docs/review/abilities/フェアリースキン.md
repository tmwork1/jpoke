## 実装（handlers/ / data/）
- `handlers/ability.py` の共通ヘルパー `_skin_modify_move_type` / `_skin_boost_power`
  （フェアリースキン・スカイスキン・ドラゴンスキン・フリーズスキンが共用）に、
  仕様書に記載されている「タイプが変わる技」の対象外判定が実装されていない不具合を発見し修正した。
  - 修正前は「変換後のタイプ（またはmove.data.type）がノーマルかどうか」のみで判定していたため、
    天候なし・地形なし・アイテムなしなどで結果的にノーマルタイプのままだったウェザーボール/
    さばきのつぶて/だいちのはどう/めざめるダンスが誤ってフェアリータイプに変換され、
    威力補正まで乗ってしまっていた。
  - 修正後は技名ベースで判定する `_skin_is_exempt()` を追加し、
    ウェザーボール/さばきのつぶて/しぜんのめぐみ/だいちのはどう/マルチアタック/めざめるダンスは
    常に対象外、テラバースト/テラクラスターはテラスタル中のみ対象外とした
    （テラスタルしていなければ通常のノーマル技として対象になる）。
  - しぜんのめぐみ・マルチアタックは技データ自体が未実装のため現状は判定対象になり得ないが、
    将来実装された際に正しく除外されるよう対象名リストに含めた。
  - この修正はフェアリースキン自身のON_MODIFY_MOVE_TYPE/ON_CALC_POWER_MODIFIERハンドラ
    （`フェアリースキン_modify_move_type` / `フェアリースキン_modify_power`）が
    上記共通ヘルパーを呼び出しているため、フェアリースキンの挙動にも直接影響する。
