## 実装（handlers/ / data/）
- `src/jpoke/data/ability.py`: どくしゅの `Event.ON_DAMAGE_HIT` ハンドラに `priority=10` を追加した。
  従来はデフォルト値（100）のままで、`docs/spec/turn.md` が明記する
  「10 | 攻撃側のどくしゅによるどく」に反していた。優先度100のままだと、ミイラ/さまようたましい/
  とれないにおい（同イベント priority=20）や、はたきおとす等のアイテム効果（同100、同順位のため
  順序不定）より後に発動する可能性があり、`docs/spec/abilities/どくしゅ.md` に明記された
  「どくしゅの判定を先に行ってからミイラ等になる」「どくしゅの効果できのみを消費したときは
  はたきおとす等の技の効果が発動しない」という順序仕様を満たせていなかった
- `src/jpoke/handlers/ability.py`: `どくしゅ_maybe_poison_on_contact` で `ailment_manager.apply()` の
  戻り値を見ず、常に無条件で処理を終えていたため、実際にどく状態を付与できた場合でも
  `_announce_ability_triggered()`（特性発動ログ・`ability.revealed` 更新）が呼ばれていなかった。
  戻り値が True のときのみ発動を明示するよう修正した
  （せいでんき等が使う既存ヘルパー `_apply_contact_counter_ailment` はロール成功時に
  無条件で発動を明示する実装だが、どくしゅでは実際に付与できた場合のみ明示する、より厳密な扱いとした）
- 上記2点以外の判定ロジック（`battle.query.is_contact(ctx)` によるパンチグローブ対応込みの接触判定、
  `resolve_secondary_chance` によるりんぷん/おんみつマント対応込みの確率解決、
  `ailment_manager.apply` へのタイプ相性判定委譲）は仕様と一致しており変更なし
- 既知の未対応事項として、`Event.ON_DAMAGE_HIT` は実HPダメージが0以下のとき発火しないため、
  こらえるでHP1のまま耐えた場合やばけのかわ/アイスフェイスで攻撃を肩代わりされた場合
  （実HPダメージ0）にどくしゅが発動しない。`Event.ON_HIT` に変更すればこの制約は解消できるが、
  ON_HIT は move_executor.py 内で ON_DAMAGE_HIT より必ず先に発火するため、技自身の追加効果
  （同じく ON_DAMAGE_HIT で処理される、例: どくづき）よりどくしゅの判定が先走ってしまい、
  「技の追加効果の判定の後にどくしゅの判定がある」という基本仕様（合成確率51%の根拠）が
  崩れる。両立できないため、影響範囲がより大きい後者を優先し、このエッジケースは
  handlers/ability.py のdocstringに明記した上で未対応のままとした
