## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `へんげんじざい_change_type` に3件のバグを発見し修正した。
  1. **わるあがきで誤発動するバグ**: わるあがき（`type=""`、タイプなし技）を使用した場合でも
     `[""] != types` が真になるため発動し、`ability_override_type=""` という不正なタイプ状態に
     なっていた（仕様書「わるあがきを使用したときは発動しない」に違反）。`move_type` が空文字の
     場合は発動しないガードを追加した。
  2. **もえつきる/でんこうそうげきとの相互作用バグ**: 仕様書は「もえつきる/でんこうそうげきが
     元タイプ不足で失敗したときは、へんげんじざいは発動しない」としているが、実装では両技の
     タイプ判定（`もえつきる_fail_if_no_fire_type` / `でんこうそうげき_fail_if_no_electric_type`）が
     `Event.ON_TRY_MOVE_1`（priority=10）に登録されており、これは `Event.ON_MOVE_CHARGE`
     （へんげんじざいの発動イベント、priority=100）より後に発火する別イベントだった。そのため
     本来失敗するはずの技でも、先にへんげんじざいがタイプを変えてしまい、判定をすり抜けて
     技が成功してしまう不具合を確認した（例: ノーマルタイプ＋へんげんじざいでもえつきるを使うと、
     成功してダメージを与えてしまう）。両技のタイプ判定を `Event.ON_TRY_ACTION`
     （priority=15、こおりの回復判定〈priority=10〉より後・へんげんじざい発動〈別イベント〉より前）に
     移動して修正した。あわせて `docs/spec/turn.md` の該当行（`ON_TRY_MOVE_1` セクションの
     「ほのおタイプでないもえつきるの失敗」「でんきタイプでないでんこうそうげきの失敗」）を
     `ON_CHECK_ACTION` セクションへ移設し、`ON_TRY_MOVE_1` セクションに残っていた
     「60 | へんげんじざい・リベロの発動」という重複・誤記の行（実際の登録先は `ON_MOVE_CHARGE`
     で既に別途正しく記載済み）を削除した。
  3. **もりののろい/ハロウィンの追加タイプが残留するバグ**: 仕様書「もりののろい/ハロウィンにより
     タイプが追加されている場合は複数タイプを持っているとみなされ、へんげんじざいが発動すると
     使用する技のタイプ単色になるため、これらの状態を解除することになる」を満たしておらず、
     `ability_override_type` を設定しても `Pokemon.types` プロパティが `added_types` を
     ベースタイプに再合成するため、発動後も追加タイプが残ったままになっていた。`ミラータイプ`/
     `テクスチャー` と同様のパターンで、発動時に `ctx.attacker.added_types = []` を追加して修正した。
