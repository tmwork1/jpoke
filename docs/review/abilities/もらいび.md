# もらいび レビュー結果

## 仕様書
- `docs/wiki/abilities/もらいび.html`（一次情報）と照合した。
- 「バトルでの効果」の要約が「自分がほのお技を使うときのこうげき・とくこうが1.5倍になる」と
  誤って記載されていた（一次情報の要約文をそのまま転記したもの）ため、
  「自分がほのお技を使うときの威力が1.5倍になる」に修正した。
  - もうか・げきりゅう等（第五世代以降にステータス補正へ変更された特性）とは異なり、
    もらいびは全世代を通じて技の威力補正として扱われる（一次情報の「第四世代まで」節に
    「もらいび状態は、ほのおわざを使用したときのダメージが1.5倍になる効果だった」と明記されており、
    以降の世代でステータス補正へ変更されたという記述は存在しない。第八世代説明文でも
    「威力を あげる」と明記されている）。この点を明確化する注記を追加した。
- それ以外の項目（特性の仕様・第五世代以降の変更点・備考）は一次情報と一致しており変更なし。

## 実装計画書
- 旧計画書は `model/ability.py` や `move_executor.py`、`tests/test_ability.py` など
  現行のディレクトリ構成と乖離した記述だったため、現行の
  `handlers/ability.py` / `data/ability.py` / `tests/abilities/test_ability_ma.py` に
  合わせてテンプレート形式（`docs/plan/abilities/_template.md`）で全面的に書き直した。
- `docs/spec/turn.md` を確認し、`Event.ON_BEFORE_APPLY_MOVE`（`turn.md` 上は見出しが
  歴史的経緯で `Event.ON_APPLY_MOVE` のままだが、表内に `Event.ON_BEFORE_APPLY_MOVE` の
  行があり同一イベントを指す）ではデフォルト優先度（100）のハンドラが大半であることを確認し、
  もらいびの `block_fire` がデフォルト優先度のままで既存の同種特性（ちくでん・ちょすい）と
  揃っていることを計画書に明記した。
- `ON_MOVE_CHARGE` / `ON_MOVE_END` / `ON_CALC_POWER_MODIFIER` は `turn.md` に専用節がない
  （ダメージ計算・技実行内部の補助イベント）ため、既存の同型特性（もうか等の
  `ON_CALC_ATK_MODIFIER`、ちくでん・ちょすいの `ON_BEFORE_APPLY_MOVE`）を参照して
  デフォルト優先度で問題ないと判断した旨を明記した。

## 実装（handlers/ / data/）
- `handlers/ability.py` の `もらいび_reserve_fire_boost` で
  `mon = getattr(ctx, "source", None) or getattr(ctx, "attacker", None)` という
  防御的なフォールバックが使われていたが、この関数は `subject_spec="attacker:self"` で
  `Event.ON_MOVE_CHARGE`（常に `AttackContext`）にのみ登録されており `ctx.attacker` の存在が
  イベント発火側で保証されているため、`mon = ctx.attacker` に簡略化した
  （CLAUDE.md「イベント発火側で前提が保証されている場合、ハンドラ側の重複ガードは不要」に準拠）。
- それ以外のロジック（`もらいび_init_state` / `もらいび_block_fire` /
  `もらいび_modify_power` / `もらいび_consume_fire_boost`、`data/ability.py` の
  イベント登録・`mold_breaker_ignorable` フラグ）は仕様書・計画書と整合しており変更なし。
  威力補正が `Event.ON_CALC_POWER_MODIFIER`（威力ベース）で実装されている点も、
  上記の仕様書修正の通り正しい実装であることを確認した。

## テスト
- `tests/abilities/test_ability_ma.py` に以下のテストを追加した。
  - `test_もらいび_おにびなど状態異常技も無効化する`: 状態異常技（おにび）も無効化しやけどにならないこと。
  - `test_もらいび_交代で状態がリセットされる`: 一度チャージ状態になった後、交代して戻ると
    `idle` にリセットされること。
  - `test_もらいび_非ほのお技を挟んでも次のほのお技で1回だけ強化される`: 非ほのお技を挟んでも
    チャージ状態が維持され、次のほのお技でのみ1.5倍になること（旧計画書で未テストとされていた項目）。
  - `test_もらいび_まもるで防がれた場合は発動しない`: まもるでほのお技をブロックした場合、
    もらいびが発動しない（`state` が `idle` のまま・特性が公開されない）こと。
  - 既存の3テスト（かたやぶり貫通・威力1.5倍・自分対象技では発動しない）は実装と一致しており変更なし。
