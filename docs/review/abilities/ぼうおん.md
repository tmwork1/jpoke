# ぼうおん レビュー結果

## 仕様書
- 変更なし。一次情報（https://wiki.pokemonwiki.com/wiki/ぼうおん ）と全文照合し、内容が完全に一致していることを確認した。

## 実装計画書
- 実装方針を、実際のコード構成（`ぼうおん_block_sound` と、ほろびのうた用の per-mon 免疫判定 `_blocked_by_bouon`）に合わせて具体化した。
- 「自分含む全員が対象の技（ほろびのうた）は技全体無効化の対象から除外し、対象ポケモンごとに免疫判定する」という設計方針を明記した（おうごんのからだの `_blocked_by_ougon_no_karada` と同一パターン）。
- 変更候補に `src/jpoke/handlers/move_status.py` を追加。
- テスト観点を実装内容に合わせて更新した。

## 実装（handlers/ / data/）
- **バグ修正**: `ほろびのうた` はプログラム上 `target="foe"`（デフォルト値）だが、実際には使用者自身を含む場に出ている全ポケモンに効果が及ぶ技である。従来の `ぼうおん_block_sound`（`Event.ON_BEFORE_APPLY_MOVE`、`stop_event=True`）は、相手（ctx.defender）がぼうおんを持つ場合に技全体を無効化していたため、**使用者自身へのほろびのうた状態付与まで防いでしまっていた**（一次情報「特性 ぼうおん のポケモンに対しては無効化され、ほろびのうた状態にならない」＝あくまでそのポケモン個別の免疫であり、技全体の無効化ではない）。
  - `src/jpoke/handlers/ability.py`: `_BOUON_EXCLUDED_MOVES = frozenset({"ほろびのうた"})` を追加し、`ぼうおん_block_sound` の全体無効化対象からほろびのうたを除外。
  - `src/jpoke/handlers/move_status.py`: `_blocked_by_ougon_no_karada` と同じパターンで `_blocked_by_bouon` を新設し、`ほろびのうた_apply` のループ内で使用者以外の各対象ごとにぼうおん免疫を判定するように修正（使用者自身は現行世代の仕様通り自身のぼうおんを無視するため対象から除外）。
  - この per-mon 判定はイベントハンドラを介さない直接呼び出しだが、`mon.ability.name`/`mon.ability.enabled` を参照するため、かたやぶり等による無効化は自動的に反映されることを確認済み。
- すてゼリフ（能力低下・交代効果を両方無効化）・通常の単一対象音技（ちょうおんぱ等）・いやしのすず/とおぼえ/ソウルビート（自分・味方のみが対象のため相手のぼうおんと無関係）については、既存実装が仕様と一致していることを確認し、変更なし。

  - `test_ぼうおん_ほろびのうたのかたやぶりで無効`: かたやぶりの使用者に対してはほろびのうたのぼうおん免疫（per-mon判定）が発動しないことを確認。
  - `test_ぼうおん_ほろびのうたは自分への付与のみ防ぐ`: 相手がぼうおんを持つ場合、相手への付与のみ防がれ、技自体は失敗せず使用者自身には通常通り付与されることを確認（今回のバグ修正の回帰テスト）。
