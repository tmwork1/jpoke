## 実装（handlers/ / data/）
- 既知の注意点どおり、`わざわいのつるぎ_reduce_B` に
  「場に複数体『わざわいの』系特性所持者がいる場合、攻撃側（このケースでは防御側）自身が
  同じ特性を持っていても誤って軽減されてしまう」バグが存在した。
  `ctx.attacker is not ctx.defender` の判定のみで、防御側自身が わざわいのつるぎ を
  持つ場合（コピー・交換等による取得を含む）の除外判定が欠けていた。
  兄弟特性 わざわいのおふだ_reduce_A・わざわいのうつわ_reduce_C と同じパターンで
  `ctx.defender.ability.name != "わざわいのつるぎ"` を追加し、防御側自身が同特性を
  持つ場合は防御が下がらないよう修正した。
  （つるぎ・たまは Def 系のため、攻撃側=特性所持者・防御側=効果を受ける側という関係になり、
  除外判定は `ctx.defender.ability.name` を見る点が Atk 系のおふだ・うつわ
  （`ctx.attacker.ability.name` を見る）と逆になる）
- `src/jpoke/data/ability.py` の登録内容（`Event.ON_CALC_DEF_MODIFIER`,
  `subject_spec="attacker:self"`）は仕様と整合しており変更なし。
