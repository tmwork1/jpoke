## 実装（handlers/ / data/）
- `src/jpoke/data/ability.py` の `"ミストメイカー"` に `Event.ON_ABILITY_ENABLED`
  のハンドラ登録が漏れていたため追加した。
  - 仕様書に「ポケモンの特性がミストメイカーに書き換わった場合、即座に特性が
    発動する」と明記されており、姉妹特性（エレキメイカー/グラスメイカー/
    サイコメイカー/ハドロンエンジン）はすべて `ON_SWITCH_IN` と
    `ON_ABILITY_ENABLED` の両方を同じハンドラ関数で登録しているのに対し、
    ミストメイカーだけ `ON_SWITCH_IN` のみが登録されており、スキルスワップ・
    トレース・かがくへんかガス解除等で特性がミストメイカーに変化しても
    フィールドが展開されないバグだった。
  - `src/jpoke/handlers/ability.py` の `ミストメイカー_activate_terrain` は
    共通ヘルパー `_activate_terrain` を呼ぶのみで `ctx.source` しか参照しない
    ため、関数自体の変更は不要だった。
