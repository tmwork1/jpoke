- 変更なし。`handlers/ability.py` の `いろめがね_boost_ineffective()` は `battle.query.is_not_very_effective(ctx)` で「効果は今ひとつ」（`0 < タイプ相性倍率 < 1`）のみを判定しており、無効化技（倍率0）や等倍・抜群では発動しない。
- `is_not_very_effective` は `damage_calculator.calc_def_type_modifier(ctx)`（= `Event.ON_CALC_DEF_TYPE_MODIFIER` の結果）を参照するため、もりののろい/ハロウィン/タールショット/フリーズドライ/フライングプレス/らんきりゅう/テラスシェルなど「最終的な相性」を変える効果もすべて反映された後の値で判定される。仕様書の記載と実装が一致していることを確認した。
- `data/ability.py` の登録（`Event.ON_CALC_DAMAGE_MODIFIER`, `subject_spec="attacker:self"`）は `AttackContext` の攻撃側特性として正しいロール指定になっている。
