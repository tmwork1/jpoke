# ハードロック レビュー結果

## 仕様書
- 変更なし。既存の `docs/spec/abilities/ハードロック.md` を一次情報（`docs/wiki/abilities/ハードロック.html`）と
  照合し、効果（効果バツグンダメージ3/4倍）・特性の仕様（かたやぶり無効、もりののろい/ハロウィン/
  タールショットで追加された弱点は発動に影響、らんきりゅうで等倍化されると発動しない、
  フリーズドライ/フライングプレスの相性変化は発動に影響、弱点半減きのみ等の相性を変えない効果は
  発動に影響しない）・備考（フィルター/プリズムアーマーとの異同、4倍弱点保有）すべて一致を確認した。

## 実装計画書
- 新規作成。`docs/plan/abilities/ハードロック.md` が存在しなかったため、仕様書と既存実装
  （`ON_CALC_DAMAGE_MODIFIER` + `is_super_effective` + `mold_breaker_ignorable` フラグ）を
  照合しながら計画書を作成した。実装済みのため「保留・論点」は「なし」とした。

## 実装（handlers/ / data/）
- 変更なし。`handlers/ability.py` の `ハードロック_reduce_effective`（`フィルター`/`プリズムアーマー`と共用）は
  `battle.query.is_super_effective(ctx)` で効果抜群判定を行い、真なら固定小数点3072/4096（0.75倍）を乗算する
  実装で、仕様と一致することを確認した。
  - `is_super_effective` は `calc_def_type_modifier` → `Event.ON_CALC_DEF_TYPE_MODIFIER` を経由するため、
    もりののろい/タールショット/フリーズドライ/らんきりゅう等の相性変化要因は一元的なタイプ相性計算に
    反映され、ハードロック側で個別判定する必要がないことを確認した。
  - `AbilityData` に `flags={"mold_breaker_ignorable"}` が付与されており、かたやぶり・シャドーレイ・
    フォトンゲイザー等で無効化される（プリズムアーマーはこのフラグを持たず、かたやぶりに対しても
    有効という仕様上の違いも実装に反映されている）ことを確認した。
