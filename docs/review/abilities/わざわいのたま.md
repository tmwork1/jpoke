## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `わざわいのたま_reduce_D` に防御側ステータス判定
  （`not battle.query.deals_physical_damage(ctx.attacker, ctx.move)`）が欠落していたバグを
  修正した。
  - 修正前は判定なしで、攻撃側がこの特性を持つ場合、物理技であっても防御側の防御補正を
    0.75倍にしてしまっていた（本来はとくぼうのみが対象で、ぼうぎょを参照する技には無関係）。
  - 兄弟特性 `わざわいのうつわ`（特殊版・とくこう対象）で既に同様の欠落バグが見つかり
    `ctx.move.category == "special"` 判定で修正済みだったが、`わざわいのたま`（防御側イベント
    `ON_CALC_DEF_MODIFIER` を使う）はそもそも分類ではなくボディプレス等の例外を含めて
    `_calc_final_defense` と同じ `battle.query.deals_physical_damage` で判定する必要があり、
    `ファーコート`・`フラワーギフト_modify_def` 等の既存ハンドラと同じ判定パターンを採用した。
  - `data/ability.py` 側の登録（`Event.ON_CALC_DEF_MODIFIER`, `subject_spec="attacker:self"`）
    自体は仕様と整合しており変更なし。
  - `AbilityData.flags` に `mold_breaker_ignorable` が付与されていないこと（＝かたやぶりで
    無視できない仕様どおり）を確認し、変更なし。

## テスト
- `tests/abilities/test_ability_yarawa.py` に以下を追加した:
  - `test_わざわいのたま_物理技には効果が及ばない`（今回のバグ修正の回帰防止）
  - `test_わざわいのたま_自分自身の特防には効果が及ばない`
  - かたやぶり関連のテストは追加していない: `わざわいのたま` は
    `subject_spec="attacker:self"`（自身が攻撃側のときのみ発動）で、かたやぶりの無効化ロジック
    （`かたやぶり_disable_foe_ability`）は `subject_spec="attacker:self"` で
    かたやぶり所持ポケモンが攻撃側のときに防御側の対象特性を無効化するものであるため、
    この特性が発動する場面（自身が攻撃側）とかたやぶりが介入できる場面（相手が攻撃側）は
    構造的に重ならず、意味のある回帰テストを構成できない。
- 既存の `test_わざわい_相手防御補正を0_75倍`（わざわいのたま分）はそのまま維持。
- `python -m pytest tests/ -v` で全件（5723件）成功、1件skip（本修正と無関係な既存skip）を確認。
