## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/data/ability.py` の `"スイートベール"` エントリは
  `ON_BEFORE_APPLY_AILMENT`（`prevent_sleep_ailment`）で「ねむり」の付与を、
  `ON_BEFORE_APPLY_VOLATILE`（`スイートベール_prevent_volatile`）で「ねむけ」の付与を
  それぞれ無効化しており、`ふみん`・`やるき` と全く同じパターンで実装されている。
  `flags={"mold_breaker_ignorable"}` によりかたやぶり等での無効化にも対応済み。
  - `subject_spec="target:self"` は両イベントとも
    `EventContext(source=source, target=target)` で発火されることと整合している。
  - CLAUDE.md のメモ（特性ハンドラにかがくへんかガス/かたやぶり無効化チェックを書かない）
    通り、ハンドラ内には個別のかたやぶりチェックは実装されておらず、
    `mold_breaker_ignorable` フラグ + かたやぶり側の一時無効化の仕組みに一元化されている。
  - 対象範囲がシングルバトルのみのため、「自分を含めた味方」は自分自身のみとなり、
    `target:self` のみの実装で仕様を満たす（アロマベール計画書と同様の考え方）。
  - すでにねむり状態のポケモンを治す効果は実装されておらず（`ON_ABILITY_ENABLED` の
    キュアハンドラが存在しない）、仕様（治療効果なし）と一致している。
