- 追加: うのミサイルの交換可否について、SV Ver.3.0.0 以降（本プロジェクトの対象範囲）は交換可能であることを明記する脚注を追加した

## 実装計画書
- 既存の計画書は初版（2026-06-04）作成時点の設計案であり、実際の実装（`さまようたましい_swap_ability_on_contact`）とAPI・priorityが乖離していたため、実装済みコードに合わせて全面的に更新した。
  - `battle.change_ability()` を2回呼ぶ設計 → 実際は `battle.ability_manager.swap_ability()` を1回呼ぶ実装（`ON_ABILITY_DISABLED`/`ON_ABILITY_ENABLED` をまとめて発火し、片方だけ入れ替わることを防ぐ）
  - `battle.query.is_contact(ctx)` → 実際は `battle.query.is_contact_reaction(ctx)`（ぼうごパットによる発動阻止を内包）
  - priority 100 → 実際は 20（同系統の `ミイラ`・`とれないにおい` に合わせた値）
  - うのミサイルの例外的な交換許可ロジックの記載を追加
  - 「既知の制限」セクションを追加（下記参照）

## 実装（handlers/ / data/）
- 実装は既に完了しており、仕様書と照合した結果、以下の点を確認した:
  - `protected` フラグを持つ特性（ARシステム等）は交換不可
  - かがくへんかガスは `base_name` で個別除外（とくせいなし状態でも対象外）
  - うのミサイルは `protected` フラグを持つが `base_name` で個別に例外許可（対象範囲がVer.3.0.0以降のため）
  - `is_contact_reaction` によりぼうごパットでの発動阻止・接触判定を包括的に処理
  - 攻撃者が fainted の場合は発動しない
- コード自体への修正は行わなかった（変更なし）。
- 既知の制限として、反動技（`Event.ON_HIT` で反動を計算）といしあたま等の相互作用について、仕様書記載の
  「特性を交換した後に反動ダメージが発生する」という順序が現状の実装では再現できていないことを確認した。
  この制限は `さまようたましい` 固有ではなく `ミイラ`・`とれないにおい` にも共通するアーキテクチャ上の制約
  （反動技ハンドラの発火イベントを `ON_HIT` から見直す必要がある）であり、本レビューの対象範囲外として
  計画書に「既知の制限」として明記し、対応を見送った。
