## 実装計画書
- 記述を実装・実際の挙動に合わせて修正した。
  - 実装ゴールを「こうげき・とくぼうが1.5倍」という曖昧な記述から、「物理技のこうげき補正のみ
    1.5倍」「特殊技のとくぼう補正のみ1.5倍」という正確な条件に修正した。
  - 実装方針に、`ON_CALC_ATK_MODIFIER`/`ON_CALC_DEF_MODIFIER` は物理/特殊どちらの技でも発火する
    ため、こうげき側は `ctx.move.category == "physical"`、とくぼう側は
    `not battle.query.deals_physical_damage(ctx.attacker, ctx.move)` で絞り込む必要があることを
    明記した（こんじょう・サンパワー・ちからもち・`ability_paradox.apply_def_modifier` と同じ
    判定パターン）。
  - ワンダールームとの実数値入れ替え相互作用は `docs/spec/fields/ワンダールーム.md` の
    「範囲外（現行実装非対象）」で明記済みの既知の対象外である旨を追記した。
  - `flags`・変更候補・テスト観点・疑似コードを実装に合わせて更新した。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/ability.py` の `フラワーギフト_modify_atk` / `フラワーギフト_modify_def` を
  修正した（バグ修正）。
  - 修正前は `ON_CALC_ATK_MODIFIER` が発火するたび（技の分類を問わず）1.5倍を適用しており、
    特殊技（例: はかいこうせん）でも「こうげき」補正が1.5倍になってしまっていた。
  - 修正前は `ON_CALC_DEF_MODIFIER` も同様に技の分類を問わず適用しており、物理技（例:
    たいあたり）を受けたときにも「とくぼう」補正が1.5倍になってしまっていた（本来は「ぼうぎょ」
    は変化しない）。
  - `フラワーギフト_modify_atk` に `ctx.move.category == "physical"` の判定を追加し、物理技の
    ときのみ補正するよう修正した。
  - `フラワーギフト_modify_def` に `not battle.query.deals_physical_damage(ctx.attacker, ctx.move)`
    の判定を追加し、サイコショック等の防御側実数値が「ぼうぎょ」参照になる技を除外したうえで、
    特殊技のときのみ補正するよう修正した。
  - `data/ability.py` の `"フラワーギフト"` エントリ（`flags`, ハンドラ登録）自体は仕様書と一致
    しており変更なし。
