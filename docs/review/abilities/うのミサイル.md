## 実装（handlers/ / data/）
- **flags の欠落を修正**: `data/ability.py` の `うのミサイル` に `protected` フラグを追加した。
  一次情報に「この特性は上書きできない。いえき/コアパニッシャーを受けてもとくせいなし状態にならず」と
  あるが、従来の実装は `gas_proof` のみでいえき・なかまづくり・シンプルビーム・なやみのタネ等の
  上書き系の技に対して無防備だった（バグ）。
- **スキルスワップ/さまようたましいの例外処理を追加**: 一次情報によると、うのミサイルは上書き系の技には
  抵抗するが、SV Ver.3.0.0以降はスキルスワップ・さまようたましいでの交換・トレース等によるコピーは
  可能になった。`protected` フラグは（他の上書き系判定と共用のため）このスキルスワップ/さまようたましい
  も一律でブロックしてしまうため、`handlers/move_status.py` の `スキルスワップ_can_apply` と
  `handlers/ability.py` の `さまようたましい_swap_ability_on_contact` に `うのミサイル` の base_name を
  個別に除外する例外を追加した（`なまけ` に対する `シンプルビーム_can_apply` の個別判定と同様のパターン）。
  `uncopyable` フラグは付与していないため、トレース・なりきり・うつしえによるコピーは元々可能。
- **ダイビングの溜めターン判定の欠落を修正**: 一次情報に「ダイビングは、1ターン目の溜める行動ができれば
  フォルムチェンジできる。2ターン目の攻撃がまもる状態やよびみずなどで無効化されたときや…でも関係ない」
  とあるが、従来の実装は `Event.ON_MOVE_END` のみで判定しており、`Event.ON_MOVE_CHARGE` が
  `False` を返す溜めターンには `ON_MOVE_END` が発火しないため、2ターン目がまもる等で無効化されると
  フォルムチェンジ自体が成立しない実装になっていた（バグ）。`うのミサイル_load_prey_on_charge`
  （`Event.ON_MOVE_CHARGE`, priority=90）を新設し、技自身の揮発状態「ダイビング」付与ハンドラ
  （priority=100）より先に判定することで、1ターン目の溜めターン成功時点でフォルムチェンジが確定する
  ようにした（パワフルハーブで1ターンで繰り出す場合も揮発状態付与前に判定されるため成立する）。
  既存の `うのミサイル_load_prey`（`Event.ON_MOVE_END`）は「なみのり」のみを対象とするよう変更した
  （ダイビング2ターン目時点では既にフォルムが変わっているため `mon.name != CRAMORANT_NORMAL` の
  ガードで自然に無効化される）。
- フォルム変化ロジックの重複を `_set_cramorant_form_by_hp` ヘルパーに統合した。
- 上記以外（吐き出し時の反撃ダメージ・副効果・みがわり貫通・マジックガード連携・交代時のフォルム復帰等）
  は仕様書・一次情報と照合し、既存実装が正しいことを確認した（`battle.modify_hp` / `battle.modify_stats` /
  `battle.ailment_manager.apply` を使った一般機構に委譲されており、追加のうのミサイル固有コードは不要）。
