  無効化されても発動する点、上書き・コピー不可、第七世代以降の行動判定順など）は
  すべて既存の `docs/spec/abilities/バトルスイッチ.md` に反映済みであることを確認した。

## 実装計画書
- 変更なし。`docs/plan/abilities/バトルスイッチ.md` は `Event.ON_CHECK_ACTION` を採用する記載に
  なっているが、実装は `docs/spec/turn.md` の現行仕様（`Event.ON_TRY_ACTION` priority=200）に
  合わせて `ON_TRY_ACTION` を使用しており、こちらが正（turn.md 基準）。計画書は初期設計時点の
  ものであり、実装・仕様書と実質的に矛盾しないため書き換えは行わなかった。

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/handlers/ability.py` の `バトルスイッチ_change_form` /
  `バトルスイッチ_revert_form` と `src/jpoke/data/ability.py` の登録
  （`Event.ON_TRY_ACTION` priority=200, `Event.ON_SWITCH_OUT`）を仕様書・turn.md と照合し、
  問題がないことを確認した。
  - `ON_TRY_ACTION` priority=200 は行動可否判定（まひ/ねむり等）より後段のため、
    「行動の判定を決めてから特性を発動する」（第七世代以降仕様）を満たしている。
  - わるあがき（category=physical）は `is_attack` 判定によりブレード遷移の対象になる。
  - ねごとは選択技の置き換え（priority=150）が本ハンドラ（priority=200）より先に走るため、
    出た技に応じて正しく発動する。
