## 実装計画書

## 実装（handlers/ / data/）
- `handlers/item.py` の `とつげきチョッキ_block_status_move`（`Event.ON_TRY_MOVE_1`）を削除した。
  このハンドラは `ctx.move.category == "status"` を一律ブロックしており、以下の仕様上の例外を満たせていなかった:
  - 変化技を選択してから行動するまでに他のポケモンからとつげきチョッキを受け取った場合、そのターンのみは変化技でも使用できる
  - とつげきチョッキを持っている状態で変化技をアンコールされた場合、そのターンのみは変化技も使用できる
  実際に `アンコール` で変化技を強制した状態で検証したところ、修正前は技が失敗し（`move_success=False`）、
  仕様（そのターンのみ使用できる）に反する挙動になっていることを確認した。
  `Event.ON_MODIFY_COMMAND_OPTIONS`（`とつげきチョッキ_modify_command_options`、コマンド選択時に変化技を除外）と
  `Event.ON_CALC_DEF_MODIFIER`（`とつげきチョッキ_boost_spdef`、特殊技被弾時にとくぼう1.5倍）の2ハンドラのみに整理した。
  「変化技しか出せない場合はわるあがきになる」仕様は `CommandManager.get_available_action_commands` 側の
  既存フォールバック（技コマンドが無い場合にわるあがきを追加）で自然に満たされることを確認済み。
- `data/item.py` から該当する `Event.ON_TRY_MOVE_1` ハンドラ登録を削除した
- `python scripts/sort_handlers.py src/jpoke/handlers/item.py` と `python scripts/sort_data/sort_items.py` を実行した
