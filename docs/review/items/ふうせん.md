## 実装計画書
- 修正した。
  - ダメージ発火イベントを `Event.ON_DAMAGE_HIT` から `Event.ON_HIT` に変更する根拠を明記した。
    `Event.ON_DAMAGE_HIT` は `core/move_executor.py` の `if actual_damage <= 0: return` により
    実ダメージが0以下のとき発火しないため、仕様書が明記する「みがわりで肩代わりした場合」
    「ばけのかわ・アイスフェイスで肩代わりした場合」「ダメージ補正やオーバーフローで0になった場合」
    でもふうせんが割れる、という要件を満たせない。`Event.ON_HIT` はダメージ量に関わらず必ず発火する
    ため、これらのケースを正しくカバーできる（`いのちのたま`の反動処理と同じ判断パターン。
    `src/jpoke/data/item.py` の `いのちのたま` エントリの既存コメントを参照して同様の方針を採用した）。
  - `docs/spec/turn.md` の `Event.ON_HIT`（優先度100で「ふうせんが割れる」と明記）と
    `Event.ON_DAMAGE`（実装上は `Event.ON_DAMAGE_HIT` に相当。優先度30でも同名の記載がある）の
    二重記載について、採用理由と根拠を計画書に追記した。
  - `どろぼう`・`ほしがる`（`Event.ON_HIT`, `subject_spec="attacker:self"`, デフォルト`priority=100`）
    との優先順位について、同一イベント内で同じ優先度のためすばやさ順に実行され、奪う側が速い場合は
    仕様どおりにならない可能性がある既知の制約として明記した（解消は `どろぼう`/`ほしがる` 側の
    レビュー範囲とする）。特性マジシャン（`Event.ON_DAMAGE_HIT`）は本実装が常に先に発火するため
    問題なく仕様どおりに動作することを確認した。
  - `さしおさえ`（Embargo）は本コードベースに未実装であり、`ItemDisabledReason` にも
    未対応であることを明記した（実装時は既存の無効化機構で自動対応できる見込み）。
