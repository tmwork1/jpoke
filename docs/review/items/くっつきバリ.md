  （`Event.ON_DAMAGE_HIT` がダメージ0のとき発火しないため、ばけのかわ等で肩代わりされた
  攻撃には転移が発動しない。`docs/spec/items/ゴツゴツメット.md` に既存の同種の制限と揃えた）。

## 実装（handlers/ / data/）
- `src/jpoke/data/item.py` の `くっつきバリ` の `Event.ON_DAMAGE_HIT` ハンドラに
  `priority=30`（`docs/spec/turn.md` ON_DAMAGE 「30 くっつきバリが攻撃側に渡る」）を明示した。
  修正前は priority 未指定（既定値100）だったため、はたきおとす等と同順位になり
  発動順が不定だった。
- **重大な不具合を発見・修正**: `src/jpoke/data/move.py` の `どろぼう` / `ほしがる`
  （`ha.どろぼう_steal_item`）と `はたきおとす`（`ha.はたきおとす_remove_item`）が
  `Event.ON_HIT`（`ON_DAMAGE_HIT` より前に発火するイベント）に登録されていたため、
  くっつきバリの転移判定（`ON_DAMAGE_HIT`）より先にアイテムが除去され、
  **転移が一切発生しない不具合**があった（実際に `はたきおとす` を使用したテストで、
  防御側の道具が消滅するだけで攻撃側に渡らないことを確認した上で修正）。
  仕様書の「どろぼう・ほしがる・はたきおとす・特性マジシャンの効果判定はくっつきバリの判定の後」
  という記述に反していたため、3つのハンドラを `Event.ON_DAMAGE_HIT` に変更した
  （`docs/spec/turn.md` ON_DAMAGE 「100 はたきおとす等のアイテム効果」に整合、
  優先度は既定値100のままで変更なし）。
- この修正により、アイテムなしの攻撃者がどろぼう・ほしがる・はたきおとすでくっつきバリ所持者を
  攻撃した場合に正しく攻撃者へ転移し、技自体の効果は発動しなくなることを確認した。
  攻撃者がすでにアイテムを持っている場合は転移せず、各技が従来通り発動することも確認した。

## テスト
- `tests/test_item.py` に以下のテストを追加した。
  - `test_くっつきバリ_はたきおとす等でアイテムなしの攻撃者に転送される`
    （はたきおとす・どろぼうをパラメータ化）
  - `test_くっつきバリ_はたきおとすで攻撃者がアイテム持ちのとき通常通りはたき落とされる`
  - `test_くっつきバリ_どろぼうで攻撃者がアイテム持ちのとき転送も奪取もしない`
    （どろぼうは攻撃者がアイテムを持っていると奪取自体に失敗するため、はたきおとすとは
    期待結果が異なる点に注意して個別のテストとした）
- `python scripts/sort_tests.py tests/test_item.py` でソート、
  `python scripts/generate_test_list.py` でテスト一覧を更新した。
- `python -m pytest tests/ -v`（3181 passed, 0 failed, 1 skipped）で全テストが通ることを確認した。
  結果は `.loop/test_logs/くっつきバリ.log` に保存した。
