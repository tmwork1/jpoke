# イアのみ レビュー結果

## 仕様書
- `docs/spec/items/イアのみ.md` という個別ファイルは存在しないが、`docs/spec/items/_HP回復ピンチきのみ.md` 内に
  「イアのみ」セクション（オレン/オボン/フィラ/ウイ/マゴ/バンジと共通）として既に記載されていた（本リポジトリの既存の慣習）。
- 効果・発動条件（HP1/4以下で最大HPの1/3回復・すっぱい味が嫌いだとこんらん）・備考（該当性格：さみしがり・おっとり・おとなしい・せっかち）は
  一次情報（`docs/wiki/items/イアのみ.html`）と一致しており誤りはなかった。
- 「詳細な仕様」節が「混乱木の実#詳細な仕様を参照」という壊れた参照（`docs/wiki/items/` に混乱木の実のページが存在せず、
  外部サイトも403で取得不可）のままだったため、実際に確認できた内容で補記した。
  - こんらんの継続ターン数は通常のこんらん付与と同じ（2〜5ターンのランダム）であること
  - どんかん（マイペース）・ミストフィールドが防ぐこと（既存の汎用チェックに任せればよいこと）
  - しんぴのまもりは自傷起因（source == target）のため防げないこと
  - 本リポジトリにミント機能が存在しないため、性格は `mon.nature` を単純参照すればよいこと
  - 参考として、外部サイト（p-battle-lab.com のフィラのみ効果解説）とBulbapediaで、現行世代でもこんらん効果が
    健在であることを確認した旨を追記

## 実装計画書
- `docs/plan/items/イアのみ.md` は既に存在していたが、**「SV では混乱しない」という誤った記述**があり、実装も
  回復のみでこんらんが未実装だったため、計画書を全面的に修正した。
  - こんらん付与の実装方針（`_heal_berry` に `confuse_natures` パラメータを追加し、イアのみ専用ラッパー
    `イアのみ_heal_on_quarter_hp` から性格タプルを渡す）を明記
  - priority の根拠（`docs/spec/turn.md` に `ON_HP_CHANGED` の個別記載はなく、既存の同種ハンドラ
    `heal_on_quarter_hp` 等のデフォルト priority=100 に合わせる）を明記
  - こんらんの無効化（どんかん・ミストフィールド）は `volatile_manager.apply` 内部の汎用チェックに任せ、
    本ハンドラでは判定しない旨を明記
  - lethal 計算はHP分布のみを扱うため、こんらん付与は対象外である旨を明記

## 実装（handlers/ / data/）
- **バグ修正**: イアのみは「すっぱい味が嫌いだとこんらんする」という一次情報上の効果があるにもかかわらず、
  実装（`heal_on_quarter_hp`）は回復のみでこんらんが一切実装されていなかった。外部情報（Bulbapedia、
  p-battle-lab.com のフィラのみ効果解説）で、この味覚起因のこんらん効果は第9世代（現行タイトル）でも
  健在であることを確認し、欠落として修正した。
- `src/jpoke/handlers/item.py`
  - 共通処理 `_heal_berry` に `confuse_natures: tuple[str, ...] | None = None` パラメータを追加。
    発動時（回復が起きたとき）に `mon.nature in confuse_natures` なら
    `battle.volatile_manager.apply_confusion(mon, source=mon)` でこんらんを付与するようにした。
  - 既存の共通関数 `heal_on_quarter_hp`（フィラ・ウイ・マゴ・バンジのみが使用）は変更せず、
    イアのみ専用の新関数 `イアのみ_heal_on_quarter_hp` を追加し、
    `confuse_natures=("さみしがり", "おっとり", "おとなしい", "せっかち")` を渡すようにした。
    他の4種（フィラ・ウイ・マゴ・バンジ）は今回のレビュー対象外のため意図的に変更していない
    （並行レビューでの競合を避けるため、また各アイテムの個別レビュー時に同様の専用ラッパーへ
    切り替える想定）。
  - `python scripts/sort_handlers.py src/jpoke/handlers/item.py` を実行済み。
- `src/jpoke/data/item.py`
  - 「イアのみ」の `Event.ON_HP_CHANGED` ハンドラ登録を `h.heal_on_quarter_hp` から
    `h.イアのみ_heal_on_quarter_hp` に変更。
  - `python scripts/sort_items.py` を実行済み。
- `src/jpoke/handlers/lethal.py` の `イアのみ_heal`（HP分布のみを扱う）は変更なし。こんらんはHPを
  直接変えないため lethal 側での対応は不要と判断。
