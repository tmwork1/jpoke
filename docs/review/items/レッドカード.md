## 実装（handlers/ / data/）
- 以下のバグを修正した
  1. **とらわれ状態の見落とし**: `battle.query.can_switch(opponent)` を使っていたため、
     にげられない・バインド・フェアリーロックや特性かげふみ・ありじごく・じりょくの
     相手に対して誤って発動しなくなっていた。控えの生存有無のみを見る自前の
     コマンド列挙に置き換え、これらのとらわれ状態を無視して発動するよう修正した。
  2. **ねをはる・特性きゅうばん/ばんけんの未考慮**: これらの相手を交代させられない
     （ただしアイテムは消費される）という仕様が未実装だったため追加した。
  3. **連続攻撃技での早期発動**: `ctx.hit_index != ctx.hit_count` の判定がなく、
     連続攻撃技の1ヒット目で発動してしまっていた（仕様は「全ての攻撃が終わってから発動」）。
     最後のヒットでのみ判定するよう修正した。
  4. **実ダメージ0のケースが未対応**: `Event.ON_DAMAGE_HIT` は `actual_damage<=0` のとき
     発火しないため、ばけのかわ・アイスフェイスの肩代わりやこらえるで耐えたときに
     発動しなかった。`Event.ON_HIT` 側で `value<=0` のケースのみを処理するハンドラ
     （`レッドカード_force_switch_on_zero_damage`）を追加した。
  5. **みがわり・ちからずくの未考慮**: `ctx.substitute_damage`（自分がみがわりで防いだ）と
     ちからずく（`ctx.attacker.ability.name == "ちからずく" and ctx.move.has_flag("secondary_effect")`）
     のチェックが抜けていたため追加した。
  6. **攻撃者側のひんし未考慮**: 持たせたポケモン自身のひんしチェックに加え、
     攻撃者が反動・自滅技で既にひんしになっている場合に発動しないためのチェック
     （`ctx.attacker.fainted`）が抜けていたため追加した。
- 上記1の関連で、`priority=150`（`docs/spec/turn.md` の値）を明示指定した
- `python scripts/sort_handlers.py src/jpoke/handlers/item.py` と
  `python scripts/sort_data/sort_items.py` を実行した
