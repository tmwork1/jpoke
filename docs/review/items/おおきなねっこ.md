## 実装（handlers/ / data/）
- **バグ修正**: `handlers/item.py` の `おおきなねっこ_boost_drain` が
  `apply_fixed_modifier(value, 5325)`（誤った倍率・切り捨て）になっていたのを
  `round_half_down(value * 5324 / 4096)`（正しい倍率・五捨五超入）に修正。
  例: 通常回復量32のとき、旧実装は41、正しくは42になる。
- **欠落追加**: `handlers/move_status.py` の `ちからをすいとる_apply` が
  `Event.ON_CALC_DRAIN` を発火しておらず、おおきなねっこの効果を受けていなかったため、
  `modify_hp` 呼び出し前に emit するよう追加。
- **欠落追加**: `handlers/volatile.py` の `アクアリング_self_heal` / `ねをはる_self_heal`
  （ターン終了時回復）が `Event.ON_CALC_DRAIN` を発火しておらず、おおきなねっこの効果を
  受けていなかったため追加。
- **欠落追加**: `handlers/volatile.py` の `やどりぎのタネ_drain_hp` が
  `Event.ON_CALC_DRAIN` を発火しておらず、回復側（吸い取る側）のおおきなねっこが
  反映されていなかったため、`ctx.derive(source=to_mon)` で回復側を主体とする
  コンテキストを作って emit するよう追加。
- `data/item.py` の `おおきなねっこ` の `Event.ON_CALC_DRAIN` ハンドラを、単一の
  `ItemHandler`（`subject_spec="attacker:self"`のみ）から、AttackContext用と
  EventContext用の2つの `ItemHandler`（`subject_spec="attacker:self"` /
  `"source:self"`）を持つ `list[Handler]` に変更。
- `enums/event.py` の `ON_CALC_DRAIN` のコメントを「未実装」から実際の発火元・
  ハンドラ元に更新。
