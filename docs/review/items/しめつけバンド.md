# しめつけバンド レビュー結果

## 仕様書
- 一次情報（ポケモンWiki）と照合し、内容自体は正確だったが「備考」セクションが空欄だったため補充した。
  - 第五世代→第六世代の効果改定経緯（1/16→1/8→1/6）
  - ねばりのかぎづめとの効果の違い（ダメージ増加 vs 持続ターン固定）

## 実装計画書
- 既存計画書は「バインドのターン終了時ダメージ処理（`バインド_damage`）内で毎ターン
  `ON_MODIFY_BIND_DAMAGE` を発火し、現在の所持アイテムを問い合わせる」設計になっていたが、
  これは仕様書の「詳細な仕様」（付与時にしめつけバンドを持っていなければ発動せず、以降の
  入手・喪失では増減しない）と矛盾していたため、以下の通り設計を修正した。
  - `ON_MODIFY_BIND_DAMAGE` の発火位置を `handlers/move_attack.py` の
    `apply_bind_to_defender`（バインド付与時、`AttackContext`）に変更し、
    結果を `Volatile.bind_damage_ratio` として保存する「付与時確定」方式に変更
  - `subject_spec` を `"source:foe"` → `"attacker:self"` に変更（AttackContext のロールに合わせる）
  - `バインド_damage`（ターン終了時処理）はイベントを再発火せず、保存済みの
    `bind_damage_ratio` をそのまま使うように変更
  - `ねばりのかぎづめ`（バインド継続ターンを付与時に確定する既存実装）と同じパターンに統一

## 実装（handlers/ / data/）
- `handlers/move_attack.py` の `apply_bind_to_defender`: バインド付与時に
  `Event.ON_MODIFY_BIND_DAMAGE` を発火し、結果を `volatile_manager.apply(...,
  bind_damage_ratio=...)` として渡すように修正（バグ修正: 従来は毎ターン再判定していたため、
  バインド中にアイテムを入手/喪失すると倍率が変動してしまっていた）
- `handlers/volatile.py` の `バインド_damage`: イベント再発火を削除し、
  `mon.volatiles["バインド"].bind_damage_ratio`（付与時に確定済みの値）をそのまま使用するよう修正
- `data/item.py` の `しめつけバンド` エントリ: `subject_spec` を `"source:foe"` から
  `"attacker:self"` に変更（AttackContext での発火に合わせる）
- `enums/event.py` の `ON_MODIFY_BIND_DAMAGE` コメントを「未実装」から実装済みの内容に更新
- `python scripts/sort_handlers.py src/jpoke/handlers/item.py` /
  `src/jpoke/handlers/move_attack.py` / `src/jpoke/handlers/volatile.py` を実行（並び順に変更なし）
- `python scripts/sort_items.py` を実行（並び順に変更なし）
