## 仕様書
- 変更なし。`docs/spec/items/どくどくだま.md` は一次情報（`docs/wiki/items/どくどくだま.html`）と照合済みで、記述に誤り・欠落はなかった。

## 実装計画書
- `priority` の記載を `90` → `150` に修正した。
  - `docs/spec/turn.md` の `Event.ON_TURN_END` 表では `どくどくだま` は priority=150（644行目）と明記されており、`かえんだま`（645行目、priority=150で実装済み）と同格である。旧計画書の priority=90 はどく・もうどくダメージの priority と誤って混同したものだった。
- テスト観点に、ねむけ優先・ミストフィールド解除ターンからの発動・発動ターンのもうどくダメージ非発生の3項目を追加した。

## 実装（handlers/ / data/）
- `src/jpoke/data/item.py` の `どくどくだま` エントリの `priority` を `90` から `150` に修正した（かえんだまと同じ優先度に統一）。
  - 修正前は priority=90 で「どく・もうどくダメージ」（turn.md 上でも priority=90）と同格のため、実行順序が不定・不正確になり、「発動するターンにどくダメージは受けない」という仕様を満たせないバグだった。
- `src/jpoke/handlers/item.py` の `どくどくだま_apply_poison`（共通ヘルパー `_apply_ailment_from_item` 経由でもうどくを付与）にロジック上の問題はなく、変更なし。
  - タイプによる無効化（どく・はがねタイプ、ふしょく特性の例外）は `AilmentManager._can_apply_by_type` で共通処理されており正しい。
  - 特性・ミストフィールド等による無効化は `Event.ON_BEFORE_APPLY_AILMENT` の共通イベントで処理されており正しい。
- `python scripts/sort_data/sort_items.py` を実行し、辞書の並びを確認（既存の並び順に変更なし）。
- 交代で出した場合はそのターンから発動し、瀕死による死に出しの場合は発動しない、という仕様はターン進行フェーズの順序（`ON_TURN_END` 発火後に `run_faint_switch` を実行）により自然に満たされているため、追加の実装は不要と判断した。
