- `docs/spec/items/_フィールドシード.md` の「グラスシード」節が「効果」「詳細な仕様」ともに空欄だったため、一次情報（`docs/wiki/items/グラスシード.html`）をもとに以下を追記した。

## 実装計画書
- `docs/plan/items/グラスシード.md` に以下の誤り・欠落があったため修正した。
  - `Event.ON_SWITCH_IN` の priority が 100 のまま記載されていたが、`docs/spec/turn.md` の ON_SWITCH_IN 表では「120 グラスシードの発動」と定義されているため 120 に修正し、根拠を明記した。
  - `Event.ON_FIELD_CHANGE` / `Event.ON_ITEM_ENABLED` は turn.md に記載がないためデフォルト優先度（100）を採用する旨と根拠（既存の同種ハンドラ実装を参照）を明記した。
  - 「ぼうぎょがすでに最大のときは消費しない」という一次情報の仕様が計画書に欠落していたため追記した。
  - once フラグの記載が実装（`consume_item` によるアイテム消滅）と整合しない内容だったため修正した。

## 実装（handlers/ / data/）
- `src/jpoke/data/item.py`: 「グラスシード」の `Event.ON_SWITCH_IN` ハンドラに `priority=120`（turn.md 準拠）を追加。他のイベント（`ON_FIELD_CHANGE` / `ON_ITEM_ENABLED`）・`subject_spec`・共通ヘルパー `_terrain_seed_boost` の呼び出しは仕様通りで変更なし。
  - なお共通ヘルパー `_terrain_seed_boost`（`battle.modify_stats` の戻り値を確認してランク最大時は消費しないロジック）は先行の「エレキシード」レビューで既に修正済みであり、グラスシードもこの修正の恩恵を受けている。
- `python scripts/sort_handlers.py src/jpoke/handlers/item.py` と `python scripts/sort_items.py` を実行（グラスシード自体の並び順・内容に変更なし。同時にコミット未了の他アイテム実装分の並び替えも反映されるが、それらはこのレビューの対象外かつ内容変更なし）。

## テスト
