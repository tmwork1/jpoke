# バンジのみ レビュー結果

## 仕様書
- 変更なし（`docs/spec/items/_HP回復ピンチきのみ.md` に既存の記述があり、一次情報（Wiki）と照合して
  誤り・欠落なし。「HPが1/4以下になったとき最大HPの1/3を回復するが、にがい味が嫌いだとこんらんする」
  という効果は現行世代（SVを含む第六〜九世代）でも一貫して有効であることを一次情報の説明文でも確認済み）。

## 実装計画書
- 新規作成に近い形で全面更新。旧版は「SVでは混乱しない」と誤って記載していたが、一次情報・仕様書と
  照合した結果、こんらん効果はSVでも健在であることが判明したため修正した。
  - `イアのみ`・`ウイのみ`と同じ実装パターン（`_heal_berry` 共通処理 + 専用ラッパー関数）に変更する
    方針を明記した。
  - マゴのみは本レビューの対象外のため、既存の共通関数のまま変更しない旨を明記した。

## 実装（handlers/ / data/）
- バグを修正した。従来 `バンジのみ` は他の未対応きのみと同じ汎用関数 `heal_on_quarter_hp`
  （こんらん処理なし）を使用しており、にがい味が嫌いな性格でもこんらんしない状態だった。
  - `handlers/item.py` に `バンジのみ_heal_on_quarter_hp` を新規実装（`イアのみ`・`ウイのみ` と同じ
    `_heal_berry` 呼び出しパターン、`confuse_natures=("やんちゃ", "のうてんき", "うっかりや", "むじゃき")`）。
  - `data/item.py` の `"バンジのみ"` エントリのハンドラを `h.heal_on_quarter_hp` から
    `h.バンジのみ_heal_on_quarter_hp` に変更。
  - lethal側 (`handlers/lethal.py` の `バンジのみ_heal`) はHP分布のみを扱うため変更なし（他の同種
    きのみと同じ扱い）。
  - `python scripts/sort_handlers.py src/jpoke/handlers/item.py` と
    `python scripts/sort_data/sort_items.py` を実行済み。

## テスト
- `tests/test_item.py` に `test_バンジのみ_それ以外の性格ではこんらんしない` と
  `test_バンジのみ_とくぼうが上がりにくい性格でこんらんする`（パラメータ化: やんちゃ・のうてんき・
  うっかりや・むじゃき）を追加した。既存の共通パラメータ化テスト（回復量・発動条件）はそのまま
  流用可能なため変更なし。
- `python scripts/sort_tests.py tests/test_item.py` と `python scripts/generate_test_list.py` を
  実行し、`python -m pytest tests/ -v` で全件パス（3028 passed, 1 skipped）を確認した。
