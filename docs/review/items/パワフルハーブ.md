## 実装（handlers/ / data/）
- 以下の欠落・不具合を発見し修正した。
  - **おおひでり下のダイビング失敗時に消費されない仕様が未実装だった。**
    `handlers/item.py` の `パワフルハーブ_skip_charge` は常にアイテムを消費・
    `stop_event=True` していたため、溜めをスキップした結果 `ON_TRY_MOVE_1` で
    おおひでり（みずタイプ技を無効化）により技が失敗する場合でも消費されていた。
    → 溜めをスキップする前に「おおひでり×みずタイプ」「おおあめ×ほのおタイプ」の
    組み合わせを判定し、該当する場合はアイテムを消費せずに溜めのみスキップするよう修正した。
  - **「天候の効果が優先されてパワフルハーブは消費しない」仕様（ソーラービーム/
    ソーラーブレードのにほんばれ、エレクトロビームのあめ）が実装されておらず、
    常にパワフルハーブが先に消費されてしまうバグを発見した。**
    アイテムハンドラは登場時（バトル開始・登場）に登録され、技側ハンドラは技使用の
    直前に登録されるため、`ON_MOVE_CHARGE` が同一 priority=100 の場合は登録順により
    常にアイテムハンドラが先に実行され、技側の天候チェックに到達する前にアイテムが
    消費・イベント停止されていた（天候がにほんばれ/あめでも無条件に消費される）。
    → `ソーラービーム系_charge` と `エレクトロビーム_charge` から天候チェック部分を
      `ソーラービーム系_weather_skip` / `エレクトロビーム_weather_skip`
      （priority=90）として分離し、パワフルハーブ（priority=100）より先に実行される
      ようにした。天候による自動スキップが成立する場合はそこで `stop_event=True` を
      返し、以降のパワフルハーブ判定に到達させない。天候による自動スキップが成立しない
      場合のみ、通常通りパワフルハーブ（同priority、登録順で先）または
      通常の溜め処理（`ソーラービーム系_charge` / `エレクトロビーム_charge`、
      priority=100）に処理が渡るようにした。
    - `ソーラービーム_weather_skip` / `ソーラーブレード_weather_skip` /
      `エレクトロビーム_weather_skip` を新規追加し、`data/move.py` の
      `ソーラービーム` / `ソーラーブレード` / `エレクトロビーム` の
      `Event.ON_MOVE_CHARGE` をリスト形式に変更して priority=90 で登録した。
    - `python scripts/sort_handlers.py src/jpoke/handlers/item.py` および
      `src/jpoke/handlers/move_attack.py` を実行し、
      `python scripts/sort_data/sort_moves.py` を実行した。
  - `パワフルハーブ_skip_charge` 自体の消費・アナウンス処理（`_announce_and_consume_item`
    の呼び出し）や、通常ケースでの溜めスキップ（`stop_event=True`）は仕様通りであり
    修正不要だった。
  - `メテオビーム`（天候による自動スキップなし）は今回の分割対象外であり、修正不要だった。
    `ロケットずつき` は本コードベースに未実装のため対象外（別タスクの範囲）。
