## 実装（handlers/ / data/）
以下のバグを修正した（`だっしゅつパック` の直前レビューで指摘された「ベンチ・トラップチェックが一切ない」という所見に基づく重点確認により発見）。

- **`src/jpoke/core/command_manager.py`**: `get_available_switch_commands()` が `Interrupt.PIVOT` と `EJECTPACK*` のみとらわれ状態チェックをスキップしており、`Interrupt.EJECTBUTTON` が対象外だった。にげられない・バインド・ねをはる等でとらわれた状態のポケモンがだっしゅつボタンを発動しても、交代コマンドの選択肢が0件になり交代できなくなるバグを修正（`state.interrupt != Interrupt.EJECTBUTTON` を条件に追加）。
- **`src/jpoke/handlers/item.py` の `だっしゅつボタン_reserve_switch`**: 従来は「`ctx.defender` の `Interrupt.EJECTBUTTON` を無条件に設定するだけ」で、以下の判定が一切なかった。
  - 交代先の控え（`has_available_bench()`）が残っているかの判定を追加。
  - 受けた攻撃でひんしになった場合に発動しないようにするガードを追加（`ctx.defender.fainted`）。既存実装のままだと瀕死交代の前に誤って `Interrupt.EJECTBUTTON` を立ててしまう可能性があった。
  - みがわりに阻まれた場合に発動しないようにするガードを追加（`ctx.substitute_damage`）。
  - 特性ちからずくの効果が発動した技を受けた場合に発動しないようにするガードを追加（`いのちのたま_recoil` 等と同一パターン）。
- **`src/jpoke/data/item.py`**: 登録イベントを `Event.ON_DAMAGE_HIT` から `Event.ON_HIT` に変更した。`ON_DAMAGE_HIT` は実HPダメージが0の場合に発火しないため、ばけのかわ/アイスフェイスの肩代わりやこらえるで耐えた場合（実ダメージ0でも発動する仕様）に対応できていなかった。`いのちのたま_recoil` と同様の理由付けをコメントで明記。

`python scripts/sort_handlers.py src/jpoke/handlers/item.py` と `python scripts/sort_data/sort_items.py` を実行済み。
