# サンのみ レビュー結果

## 仕様書
- `docs/spec/items/_ランク上昇ピンチきのみ.md` の「サンのみ」節（既存）を一次情報（`docs/wiki/items/サンのみ.html`）と照合し、内容自体（HP1/4以下で消費してきゅうしょアップ状態になる、すでにきゅうしょアップ状態のときは発動しない）は一致していることを確認した。
- 追加: なげつける・むしくい・ついばむ・ほおばる・おちゃかいで消費するときは残りHPに関わらず発動する旨を追記した（同ファイル内のチイラのみ等、同系統のピンチきのみ共通仕様との整合性を取るため）。

## 実装計画書
- `docs/plan/items/サンのみ.md` は旧設計（`mon.item.data` にフラグを立てて `ON_CALC_CRITICAL_RANK` で個別に急所ランクを加算する案）のまま放置されており、実際の実装（`きゅうしょアップ` 揮発性状態を `volatile_manager.apply` で付与するだけで、急所ランク補正は `data/volatile.py` 側の既存ハンドラが担う）と乖離していたため、現行実装に合わせて全面的に書き直した。

## 実装（handlers/ / data/）
- `handlers/item.py` の `サンのみ_apply_focus_energy` に、`ほおばる`・`むしくい`・`ついばむ`・`おちゃかい`等による強制発動（`item_manager.force_trigger_berry` が `ON_HP_CHANGED` を `value=mon.max_hp` で発火する経路）を考慮する分岐が欠落していたバグを修正した。
  - 修正前: `if mon.hp <= mon.max_hp // 4:` （HP実値のみで判定、強制発動を無視）
  - 修正後: `if mon.hp * 4 <= mon.max_hp or value >= mon.max_hp:` （`_boost_on_quarter_hp` 等、既存の同種きのみハンドラと同じ判定パターンに統一）
- `data/item.py` の登録内容（`Event.ON_HP_CHANGED`, `subject_spec="target:self"`, `fling_power=10`）は仕様・計画書と一致しており変更なし。
- `sort_handlers.py` を実行したが、関数の並び順に変更はなかった。

## テスト
- 既存テスト `test_サンのみ_HP25以下できゅうしょアップ状態` は実装と一致しており変更なし。
- 追加: `test_サンのみ_すでにきゅうしょアップ状態のときは発動しない`（既にきゅうしょアップ状態のときは発動しないことを確認）
- 追加: `test_サンのみ_ほおばるでHPに関わらず発動する`（`ほおばる` による強制発動でHP満タンでも発動することを確認。上記バグ修正の回帰テスト）
- `sort_tests.py tests/test_item.py` → `generate_test_list.py` を実行し、`docs/tests/test_item.md` を更新した。
- `python -m pytest tests/ -v` を実行し、サンのみ関連の3件はすべてPASSしたことを確認（`tests/test_item.py::test_サンのみ_*`）。
  - 全体としては `test_しめつけバンド_バインドダメージ増加` が1件FAILしているが、これは同一チェックアウト内で並行進行中の別タスクによる `handlers/volatile.py` / `data/item.py` の未コミット変更が原因であり、本タスク（サンのみ）の変更とは無関係であることを確認済み。
