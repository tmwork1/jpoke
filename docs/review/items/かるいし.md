# かるいし レビュー結果

## 仕様書
- 変更なし。一次情報（docs/wiki/items/かるいし.html）と照合し、誤り・欠落なしと確認した。

## 実装計画書
- 修正した。効果概要に「小数点第二位切り捨て・最低0.1kg・なげつける威力30」を明記し、
  下記の実装バグ修正の経緯をコメントとして追記した。
- テスト観点に「体重0.1kgのポケモンがかるいしを持っても0.1kgのままであること」を追加した。

## 実装（handlers/ / data/）
- `src/jpoke/model/pokemon.py` の `weight` プロパティにバグを発見・修正した。
  - 仕様書には「小数点は第二位を切り捨てる。切り捨てた結果おもさが0.1kgより軽くなった場合、
    0.1kgになる」とあるが、実装は下限クランプがなく、体重0.1kgのポケモン
    （例: ゴース）がかるいしを持つと `int(0.1*0.5*10)/10` が `0.0` になってしまっていた。
  - `return w` を `return max(w, 0.1)` に修正し、ライトメタル・ヘヴィメタル・かるいしの
    どの組み合わせでも最低0.1kgを下回らないようにした（ライトメタル単体でも同様の
    下限規定が仕様書に明記されているため、共通の `weight` プロパティで一括対応した）。
  - `data/item.py` の `かるいし` エントリ（`fling_power=30` のみ）は変更なし。
    体重半減効果は元々ハンドラではなく `weight` プロパティで直接実装されており、
    この方針も適切と判断し変更なし。

## テスト
- `tests/test_item.py` に `test_かるいし_体重の最低値は0_1kg` を追加した
  （ゴース（体重0.1kg）にかるいしを持たせても `mon.weight == 0.1` であることを確認）。
- 既存の `test_かるいし_体重半減` は変更なし。
- `scripts/sort_tests.py tests/test_item.py` でソート、`scripts/generate_test_list.py` で
  テスト一覧を更新後、`python -m pytest tests/ -v` を実行し、2758 passed, 1 skipped
  （既存の無関係なskip）で全テスト成功を確認した。
