## 実装計画書
- 既存の計画書は `value.get("atk", 0) < 0` を前提にした疑似コードのみで、以下の欠落があったため
  書き直した。
  - あまのじゃく所持者がいかくを受けてこうげきが上昇するケースでも本アイテムが発動する仕様の記載
  - `battle.modify_stats(mon, {"spe": +1})` の戻り値を確認しないと、すばやさが既に最大
    （あまのじゃく所持者はすばやさが既に最小）の場合でもアイテムが消費されてしまう問題の指摘
  - しろいきり・特性かいりきバサミ/クリアボディ/しろいけむり/メタルプロテクト/フラワーベール/
    きもったま/せいしんりょく（第八世代以降）でいかくの効果自体が無効化された場合も本来は
    ビビリだまが発動する仕様と、現アーキテクチャ（`Event.ON_MODIFY_STAT` はランクが実際に
    変化した後にのみ発火）では対応できないため本レビューでは見送る旨を明記した
  - ミラーアーマーによる保留発動、クリアチャームでの無効化時の発動、きょうせいを介した
    受け渡しも同様の理由で未対応と明記した（きょうせい自体が本コードベースで未実装、
    どんかん・マイペースもいかく無効化自体が未実装のため、これらの経路では問題が
    顕在化しないことも確認した）

## 実装（handlers/ / data/）
- `handlers/item.py` の `ビビリだま_boost_speed_on_intimidate` に以下の2点のバグを発見し修正した。
  - 発動条件が `value.get("atk", 0) < 0`（こうげきが下がった場合のみ）になっており、
    あまのじゃく所持者がいかくを受けてこうげきが上昇するケースで発動しなかった。
    → `"atk" in value`（こうげきランクが実際に変化したかどうか）に変更した。
  - `battle.modify_stats(mon, {"spe": +1})` の戻り値を確認せず無条件にアイテムを消費していたため、
    すばやさが既に最大ランク（あまのじゃく所持者はすばやさが既に最小ランク）で本来発動しない
    はずのケースでもアイテムが消費されていた。
    → 戻り値（実際に変化したランクの辞書）が真の場合のみ発動・消費するよう修正した。
  - `data/item.py` の登録（`Event.ON_MODIFY_STAT`, `subject_spec="target:self"`）は仕様と一致しており
    変更不要だった。
  - `python scripts/sort_handlers.py src/jpoke/handlers/item.py` を実行済み。
