## 実装計画書
- `docs/plan/items/スターのみ.md` を修正した。
  - 疑似コードが実装と乖離していた（`hp_before`/`hp_after` 差分方式で判定していたが、
    実装は他のピンチきのみ共通の `mon.hp * 4 <= mon.max_hp` 方式かつ `value >= mon.max_hp`
    による強制発動対応が抜けていた）ため、実装に合わせて更新した。
  - なげつける・むしくい・ついばむ・ほおばる・おちゃかいによる強制発動、こんらんの自傷
    ダメージでの不発、ランクが最大の能力を候補から除外する仕様、5箇所全て最大時の不発を
    明記した。
  - たんじゅん・じゅくせい・あまのじゃくの影響は既存の共通フック
    (`たんじゅん_double_stat` / `あまのじゃく_reverse_stat`) が自動適用するため個別対応
    不要であることを明記した。あまのじゃく時の候補除外反転（ランク最小を除外・最大を含む）
    は既存の類似実装（`ムラっけ_boost_stats`）でも未対応であるため、本実装でも既知の限界
    として対応しないことを明記した。
  - priority・イベント（`Event.ON_HP_CHANGED` / `subject_spec="target:self"`）は
    `docs/spec/turn.md` に未掲載のため、同種のピンチきのみ実装（チイラのみ等）に
    合わせている旨を根拠として追記した。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/item.py` の `スターのみ_random_boost` に以下のバグを発見し修正した。
  1. **ランクが最大の能力が選択候補から除外されていなかった**: 5能力から無条件で乱数選択
     していたため、最大ランクの能力が選ばれた場合に「他に上げられる能力があるのに不発・
     消費もされない」という誤った挙動になっていた。`mon.rank[s] < 6` で事前に候補を絞り込む
     よう修正（既存の `ムラっけ_boost_stats` と同様のパターン）。これにより5箇所全てが
     最大のときのみ不発（消費なし）になる。
  2. **こんらんの自傷ダメージ（`reason="self_attack"`）で発動してしまっていた**: 仕様書に
     明記された「こんらんの自傷ダメージを受けたときは、HPが発動圏内になっても発動しない」
     が未実装だったため、`_heal_berry` と同様の `ctx.hp_change_reason == "self_attack"`
     チェックを追加した。
  3. **なげつける・ほおばる等による強制発動（`value >= mon.max_hp`）に対応していなかった**:
     HPがすでに1/4以下でない限り発動しない実装だったため、ほおばる等で残りHPに関わらず
     強制発動させる仕様（チイラのみ系共通仕様、`docs/spec/items/_ランク上昇ピンチきのみ.md`
     参照）を満たしていなかった。他のピンチきのみ共通処理 `_boost_on_quarter_hp` と同様の
     `forced = value >= mon.max_hp` 分岐を追加した。
- 上記2点目（こんらん自傷除外）は、チイラのみ・ヤタピのみ・リュガのみ・ズアのみ・カムラのみ
  が共有する `_boost_on_quarter_hp` にも同じ欠落があったため、あわせて修正した
  （`if not forced: ctx.hp_change_reason == "self_attack" ならスキップ` を追加）。
  これにより上記5種のきのみにも同様の修正が反映されている。
- `data/item.py` 側の変更はなし（イベント登録・`subject_spec` は仕様通りで変更不要）。
- 修正後 `python scripts/sort_handlers.py src/jpoke/handlers/item.py` を実行済み。
