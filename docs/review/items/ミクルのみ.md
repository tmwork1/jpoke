# ミクルのみ レビュー結果

## 仕様書
- `docs/spec/items/_ランク上昇ピンチきのみ.md`（ピンチ系きのみをまとめた仕様書）に既存のミクルのみ節が存在した。内容自体は一次情報（ポケモンWiki）と整合していたが、以下を補足した。
  - こんらんの自傷ダメージでは発動しない旨を追記（個別のwikiページには明記されていないが、同じ「HP1/4以下トリガー」系のイバンのみ・チイラのみ等と同様の発動判定を用いるため、第五世代以降の仕様として共通と推定されることを明記した）。

## 実装計画書
- `docs/plan/items/ミクルのみ.md` は存在したが、実装と乖離していた（`mon.item.data` を使う疑似コードになっていたが、実装は `mon.item.count` を使用）。以下の点を修正・補完した。
  - フラグ管理を実装に合わせて `mon.item.count`（イバンのみと同じ手法）に修正。
  - こんらんの自傷ダメージ除外条件を追記。
  - 一撃必殺技には命中率倍率を適用しない旨を追記（消費はされる）。
  - 命中判定のない技（`move.accuracy is None`）を使った場合に `ON_MODIFY_ACCURACY` が発火しないため、`ON_END_MOVE` で行動完了時にフラグを解除するイベント3を追加。
  - 「効果発動中に別のミクルのみを再入手しても発動しない」という一次情報の挙動は、`mon.item.count` 方式ではアイテムスロットが実際には空にならないため構造上自然に満たされる旨を注記した。

## 実装（handlers/ / data/）
以下のバグを修正した。

1. **一撃必殺技への命中率倍率適用（仕様違反）**
   - `ミクルのみ_boost_accuracy` が一撃必殺技（`ohko`フラグ）にも無条件で1.2倍を適用していた。一撃必殺技には倍率を適用しないよう修正（アイテムの消費自体は行う）。
2. **こんらんの自傷ダメージでの誤発動**
   - `ミクルのみ_set_accuracy_flag` に `ctx.hp_change_reason == "self_attack"` の除外チェックが無かった（同種のイバンのみには実装済み）。追加した。
3. **命中判定のない技を使った場合に効果が消費されない**
   - `move.accuracy is None` の技（必中技・命中判定のない変化技）を使用すると `Event.ON_MODIFY_ACCURACY` が発火しないため（`core/move_executor.py` の `_check_hit` 参照）、フラグが消費されず何ターンも持ち越されてしまう不具合があった。
   - 新規ハンドラ `ミクルのみ_clear_flag_after_move`（`Event.ON_END_MOVE`）を追加し、行動完了時に未消費のフラグを解除するようにした。行動不能（状態異常・ひるみ等）や溜め技の溜めターンでは解除しない（メトロノームの `ON_END_MOVE` ハンドラと同じ判定ロジックを踏襲）。
4. `data/item.py` の登録から冗長な `priority=100`（デフォルト値と同じ）を削除し、新規ハンドラを追加登録した。

修正後、`python scripts/sort_handlers.py src/jpoke/handlers/item.py` と `python scripts/sort_data/sort_items.py` を実行済み。
