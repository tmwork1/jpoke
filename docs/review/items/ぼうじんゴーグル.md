    実装・登録済みイベント（`Event.ON_MODIFY_NON_MOVE_DAMAGE`）は `"target:self"` が正しいため修正した。
  - 疑似コードの `ctx.move.has_label("powder")` を実装と一致する `has_flag("powder")` に修正した。
  - 特性ぼうじんとの優先順位（switch_manager でのハンドラ登録順により特性側が先に評価される）を明記した。
  - 特性ほうし（Effect Spore）による状態異常付与の無効化について、実装方針（`handlers/ability.py` の
    `ほうし_maybe_inflict_ailment_on_contact` に直接ガード条件を追加する方針とその理由）を追記した。
  - 粉技無効化時にアイテムを公開する（`_announce_item_triggered`）仕様を追記した。
  - 現行仕様（あられ天候が存在しない）を踏まえた注意書き、いかりのこながシングルバトルでは

## 実装（handlers/ / data/）

    アイテムを公開するように修正（仕様書に記載のメッセージ表示に対応。以前は無効化するのみでアイテムが
    公開されなかった）。
  - `ぼうじんゴーグル_block_weather_damage`: 型ヒットを `ctx: Any` から `ctx: EventContext` に修正（コード品質）。
    ロジック自体（`hp_change_reason == "sandstorm"` のみをチェック）は、現行仕様に「あられ」天候が
    存在しないため妥当と判断し変更なし。
- `src/jpoke/handlers/ability.py`

    ぼうじんゴーグルを持っている場合は状態異常を付与しないガードを追加。
    仕様書（`docs/spec/items/ぼうじんゴーグル.md` および `docs/spec/abilities/ほうし.md`）の双方に
    「特性ほうしをぼうじんゴーグルが無効化する」旨が明記されていたが未実装だったため修正した
    （このアイテムの効果を実現するために必要な変更のため、本レビューの一環として修正した）。
  - `python scripts/sort_handlers.py` は関数の並び順に影響しないため実行不要と判断したが、
    `src/jpoke/handlers/item.py` については実行し差分がないことを確認した。
- `data/item.py` / `data/ability.py`: ハンドラ登録自体に変更はないため、`sort_abilities.py` /
  `sort_items.py` の実行は不要。
