## 実装（handlers/ / data/）
- `src/jpoke/data/item.py` の `ヨロギのみ` 登録内容自体に誤りはなし（`ON_CALC_DAMAGE_MODIFIER`
  ハンドラと `lethal_handlers` を登録済み、五十音順の並びも正しい）。
- レビュー中に、ヨロギのみを含む18種のタイプ半減きのみが共有する
  `src/jpoke/handlers/item.py` の `_modify_super_effective_damage` に2件のバグを発見し修正した。
  1. **消費処理の欠落**: 効果発動時に `_announce_and_consume_item` が呼ばれておらず、
     「1度だけ」の仕様に反して毎ターン再発動していた。
  2. **効果判定条件の誤り**: `calc_def_type_modifier(ctx)` はタイプ相性を4096基準の固定小数点
     （等倍=4096, 抜群=8192等）で返すにもかかわらず、条件が `> 1` になっていたため、
     免疫でない限り等倍・半減でも「効果バツグン」と誤判定され、対応タイプの技を受けるたびに
     常時ダメージを半減してしまっていた（例: ノーマルタイプのカビゴンがヨロギのみを持ち、
     等倍のいわ技を受けても誤って半減・消費されることを確認）。正しくは `> 4096` で判定すべき。
  - 併せて、ホズのみ（ノーマル専用・効果抜群不要で発動）が同じ共通関数を利用しているため、
    `super_effective_only: bool = True` 引数を追加し、ホズのみ側で `super_effective_only=False`
    を指定するよう変更した（`docs/spec/items/_タイプ半減きのみ.md` のホズのみ「備考」に合わせ、
    ノーマル技であれば相性に関係なく発動する挙動を維持）。
  - `src/jpoke/handlers/lethal.py` の `_type_resist_berry` docstring 内のコメントが旧条件
    （`> 1`）を前提とした説明のままだったため、修正後の条件（`> 4096`）に合わせて更新した。
    lethal側の判定ロジック自体（`def_type_modifier <= 4096` で早期リターン）は元々正しかった。
  - これらの修正は共通関数に対するものであり、ヨロギのみ以外の17種のタイプ半減きのみ
    （オッカのみ・ソクノのみ・ヨプのみ・ホズのみ等）にも同様に適用される。

Note: 本レビュー作業中に別の並行タスク（ナモのみのレビュー）が同じ共有チェックアウトで
`src/jpoke/handlers/item.py` を編集・コミットしており（コミット `review: ナモのみ`）、
上記の消費処理欠落・効果判定条件・ホズのみ対応・lethal.py docstring の修正は
そのコミットに取り込まれる形で既にリポジトリに反映済みであることを確認した
（重複コミットを避けるため、本レビューでは残りの差分のみをコミットする）。
