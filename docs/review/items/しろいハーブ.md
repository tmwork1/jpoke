# しろいハーブ レビュー結果

## 仕様書
- 変更なし（`docs/spec/items/しろいハーブ.md` は一次情報（Wiki）の内容と完全に一致していることを確認した）

## 実装計画書
- 既存の計画書は `Event.ON_MODIFY_STAT` によるランク低下キャンセルのみを記載しており、
  以下の仕様が実装計画から欠落していたため追記した。
  - なげつけるでしろいハーブを投げつけた場合、対象の下がったランクをリセットする効果
  - バトンタッチで下がった能力を引き継いだ場合・アイテムを入手した時点で既にランクが
    マイナスの場合に即座に発動する効果（`Event.ON_SWITCH_IN` / `Event.ON_ITEM_GAINED`）
  - `docs/spec/turn.md` の `ON_SWITCH_IN` に明記されている `160 しろいハーブの発動` の
    priority根拠

## 実装（handlers/ / data/）
- 以下の欠落を発見し実装を追加した。
  - `handlers/move_attack.py` の `なげつける_apply_item_effect` に
    `# TODO: しろいハーブ → 相手の下がったランクを0に戻す` という未実装のTODOが残っており、
    なげつけるでしろいハーブを投げても対象のランクがリセットされない状態だった。
    → `item_name == "しろいハーブ"` の分岐を追加し、`ctx.defender` の負のランクを
    全て0に戻す処理を実装した（`ON_HIT` で発火するため、`ON_DAMAGE_HIT` で発火する
    くだけるよろいより先に処理される）。
  - `docs/spec/turn.md` の `ON_SWITCH_IN` に「160 しろいハーブの発動」と明記されているにも
    関わらず、`data/item.py` にはこのイベントへの登録が存在せず、バトンタッチで下がった
    能力を引き継いだ場合や、トリック等で既にランクがマイナスの状態でアイテムを入手した
    場合に即座に発動する処理が未実装だった。
    → `handlers/item.py` に `しろいハーブ_reset_if_already_lowered` を追加し、
    `Event.ON_SWITCH_IN`（priority=160）と `Event.ON_ITEM_GAINED` に登録した
    （`くちたけん_form_change` 等、既存アイテムが同一関数を両イベントに登録する
    パターンを踏襲）。
  - 上記2箇所と既存の `しろいハーブ_cancel_stat_drop` で共通化できる「マイナスランクを
    全て0に戻す」処理を `_reset_negative_ranks` ヘルパーとして切り出した
    （`くろいきり_reset_all_ranks` と同様に `Event.ON_MODIFY_STAT` を経由せず直接
    `mon.rank` を書き換え、`LogCode.STAT_CHANGED` ログを追加する方式とし、
    この復元自体がかちき・びんじょう等を再誘発しないようにした）。
  - `しろいハーブ_cancel_stat_drop` 自体の発動条件（「今回の変化が負」かつ「結果のランクが
    マイナス」の場合のみ発動）、および発動時に「その時点で負になっている全ての能力を
    まとめてリセットする」実装は仕様と一致しており、修正不要だった。
