# じゅうでんち レビュー結果

## 仕様書
- `## 効果` の見出しが空欄だったため、一次情報（ポケモンWiki「概要」節）をもとに以下を追記した。
  - 「持たせたポケモンがでんきタイプの攻撃技を受けると、こうげきが1段階上がる。1度使うとなくなる（なげつける威力30）。」
- 「詳細な仕様」「備考」「関連項目」は一次情報と完全に一致しており修正なし。

## 実装計画書
- 消費条件の記述が実装（`_announce_and_consume_item` を成否判定後に呼ぶ）と不一致だったため、疑似コード・表を実装に合わせて修正した。
- たんじゅん・あまのじゃく・かたやぶりとの相互作用について、フレームワークの `ON_BEFORE_MODIFY_STAT`（たんじゅん/あまのじゃくの倍率反転）と
  かたやぶりのアビリティ無効化（`ON_BEGIN_MOVE`〜`ON_END_MOVE` 区間）により自動的に仕様通りの挙動になることを明記した。
- テスト観点に、ランク上限時不発・たんじゅん・あまのじゃく・あまのじゃく最小時不発・かたやぶり時の挙動を追加した。

## 実装（handlers/ / data/）
- `handlers/item.py` の共通ヘルパー `_boost_stat_on_type_hit`（じゅうでんち・きゅうこん・ひかりごけ・ゆきだま が共用）に、
  こうげき等のランクがすでに上限（あまのじゃくなら下限）でスタット変化が発生しなかった場合でも
  無条件にアイテムを消費・発動宣言してしまうバグがあったため修正した。
  - 修正前: `_announce_and_consume_item` を常に呼んでから `modify_stats` を実行（不発でも消費されてしまう）
  - 修正後: `battle.modify_stats` の戻り値（実際の変化）を確認し、変化があった場合のみ `_announce_and_consume_item` を呼ぶ
    （`じゃくてんほけん` 等の既存実装と同じパターンに統一）
  - このヘルパーを共用する `きゅうこん` `ひかりごけ` `ゆきだま` にも同じ修正が反映されるが、
    これらのテスト追加は各アイテムの担当レビューで行う想定（本タスクでは `じゅうでんち` のテストのみ追加）。
- たんじゅん・あまのじゃく・かたやぶりに関する固有ロジックは追加していない。これらは既存のフレームワーク機構
  （`たんじゅん_double_stat` / `あまのじゃく_reverse_stat` を `ON_BEFORE_MODIFY_STAT` で発火、
  `かたやぶり_disable_foe_ability` で防御側特性を技実行中無効化）によって自動的に仕様通りに動作することを確認した。
