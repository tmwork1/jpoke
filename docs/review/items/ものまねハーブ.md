## 実装（handlers/ / data/）
- `handlers/item.py` の `ものまねハーブ_copy_stat_boost` を修正:
  - **バグ修正**: 従来は「コピー元の上昇量が正であれば」無条件でアイテムを消費・発動メッセージを出していたが、
    実際のコピー結果（`battle.modify_stats` の戻り値）を確認せず消費していたため、
    自分のランクが既に最大でコピーしても実際には変化しない場合でも誤って消費される不具合があった。
    `changed = battle.modify_stats(...)` の結果を見てから消費・アナウンスするよう修正
    （`docs/spec/items/ものまねハーブ.md` の「所持者のすでに最大になっている能力を相手が上げたときは発動しない」に対応）。
  - **バグ修正**: 相手のびんじょう・ものまねハーブによるコピーで上昇した分を、自分のものまねハーブが
    再度コピーしてしまう不具合を修正。`Event.ON_MODIFY_STAT` の `ctx.stat_change_reason` が
    `"びんじょう"` / `"ものまねハーブ"` の場合は発動しないガードを追加した。
  - 誤解を招くdocstring（「+1でコピーする」）を「実際に上昇した分をコピーする」に修正。
  - `battle.modify_stats` 呼び出しに `source=ctx.target, reason="ものまねハーブ"` を追加
    （コピー自体が別のびんじょう・ものまねハーブを誤発動させないためのマーキング）。
- `handlers/ability.py` の `びんじょう_copy_stat_rise` も同様に修正（対称的な相互作用のため）:
  - `ctx.stat_change_reason` が `"びんじょう"` / `"ものまねハーブ"` の場合は発動しないガードを追加。
  - コピー呼び出しに `reason="びんじょう"` を追加。
- `types/literals.py` の `StatChangeReason` に `"びんじょう"` / `"ものまねハーブ"` を追加。
- `data/item.py` の登録内容（`Event.ON_MODIFY_STAT`, `subject_spec="target:foe"`, なげつける威力30）は
  仕様と一致しており変更なし。
- `python scripts/sort_handlers.py src/jpoke/handlers/item.py` および
  `src/jpoke/handlers/ability.py` を実行済み（並び順に変化なし）。
