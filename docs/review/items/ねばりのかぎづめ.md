# ねばりのかぎづめ レビュー結果

## 仕様書
- 変更なし。一次情報（ポケモンWiki）と照合し、効果・第四世代のみの例外・詳細な仕様（付与時確定、キョダイマックス技使用者退場後も継続ターンは減らない）がすべて正しく記載されていることを確認した。

## 実装計画書
- 修正した。旧計画書は「`ON_MODIFY_DURATION` または専用フックが必要（要確認）」という調査未了の内容のままだったが、
  実装は既に `Event.ON_MODIFY_DURATION` を `battle.query.get_volatile_duration` 経由で
  `handlers/move_attack.py` の `apply_bind_to_defender`（バインド付与時、`AttackContext`）で発火する形で完成していた。
  実態に合わせて計画書を全面的に更新し、`しめつけバンド`の計画書と対になる記述（付与時確定の設計、発火側コード、テスト観点）に揃えた。

## 実装（handlers/ / data/）
- 変更なし。`handlers/item.py` の `ねばりのかぎづめ_fix_bind_duration` は `value = [name, count]` を受け取り
  `"バインド"` の場合に `count` を 7 に固定して返す実装で、`data/item.py` で
  `Event.ON_MODIFY_DURATION` / `subject_spec="attacker:self"` として登録されている。
- `しめつけバンド` と同様、継続ターンはバインド付与時（`apply_bind_to_defender` 内の
  `battle.query.get_volatile_duration` 呼び出し）に一度だけ確定し、`Volatile.count` に保存されるため、
  付与後のアイテム入手・喪失で継続ターンが増減するバグは存在しないことを確認した
  （`バインド_damage` はターン終了時に `tick` で単純減算するのみで、`ON_MODIFY_DURATION` を再発火しない）。

## テスト
- 追加した。既存の `test_ねばりのかぎづめ_なしでは通常ターン` / `test_ねばりのかぎづめ_バインドターン固定` に加えて、
  `しめつけバンド`のテストと対になる以下2件を追加した。
  - `test_ねばりのかぎづめ_付与後の入手では延びない`: バインド付与後にアイテムを入手しても継続ターンが4〜5ターンのまま変わらないこと
  - `test_ねばりのかぎづめ_付与後の喪失では減らない`: バインド付与後にアイテムを失っても継続ターンが7ターンのまま変わらないこと
- `python scripts/sort_tests.py tests/test_item.py` でソート、`python scripts/generate_test_list.py` でテスト一覧を更新、
  `python -m pytest tests/ -v` で全件（3189 passed, 1 skipped）パスを確認した
  （実行ログ: `.loop/test_logs/ねばりのかぎづめ.log`）。
