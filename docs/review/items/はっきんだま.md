## 実装（handlers/ / data/）
- **バグ修正1（本タスクの主目的）**: `src/jpoke/data/item.py` の `はっきんだま` に、フォルムロック専用アイテム（`だいはっきんだま`）の値が誤って流用されており `removable=False, fling_power=0` になっていた。
  SV のはっきんだまは通常の持ち物としてトリック・はたきおとす等で奪われる仕様のため、`removable=False` を削除し `fling_power=60` に修正した。
- **バグ修正2**: `src/jpoke/handlers/item.py` の `はっきんだま_modify_power` が `{"ギラティナ(アナザー)"}` のみを対象としており、`ギラティナ(オリジン)` が持っている場合に威力補正が効かなかった。
  こんごうだま（`{"ディアルガ", "ディアルガ(オリジン)"}`）・しらたま（`{"パルキア", "パルキア(オリジン)"}`）と同様の両フォルム対応に修正し、
  `frozenset({"ギラティナ(アナザー)", "ギラティナ(オリジン)"})` を対象にした。
- **仕様欠落の実装追加**: 一次情報に「なげつける: ギラティナの場合は失敗する。それ以外は威力60」と明記されているにもかかわらず、
  この固有の失敗条件が未実装だった（こんごうだま・しらたまにはこの制限はなく、はっきんだま固有の仕様）。
  `src/jpoke/handlers/move_attack.py` の共通ハンドラ `なげつける_check_item` に、
  `attacker.item.base_name == "はっきんだま"` かつ `attacker.name.startswith("ギラティナ")` の場合のみ失敗する分岐を追加した
  （他のアイテム・他のポケモンの `なげつける` 挙動には影響しない）。
- `python scripts/sort_handlers.py src/jpoke/handlers/item.py` および `src/jpoke/handlers/move_attack.py` を実行済み（関数の並び順に変化なし）。
- `data/item.py` は既存エントリの値変更のみのため `scripts/sort_data/sort_items.py` の再実行は不要と判断（並び順・キー構成に変化なし）。念のため実行しても差分が発生しないことを確認。
