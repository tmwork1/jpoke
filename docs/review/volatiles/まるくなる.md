## 実装（handlers/ / data/）

### 修正内容

`src/jpoke/handlers/volatile.py` の `まるくなる_boost_power` 関数を修正:

1. **型アノテーション修正**: `ctx: EventContext` → `ctx: AttackContext`
   - `ON_CALC_POWER_MODIFIER` は攻撃フェーズイベントであり `AttackContext` が正しい。
   - `ctx.move` へのアクセスは `AttackContext` 前提なので修正が必要。

2. **docstring 拡充**: 「まるくなる状態のポケモンが使う技の威力補正」であることを明記。

3. **技名リストをタプルに変更**: `["ころがる", "アイスボール"]` → `("ころがる", "アイスボール")`（慣用パターンに合わせる）。

### subject_spec について

`subject_spec="attacker:self"` は正しい。
まるくなる状態の**自分**がころがるを使う → 攻撃側にまるくなる状態がある → `attacker:self`。
ちいさくなる（`defender:self`）とは逆の向きになる点を確認した。

### data/volatile.py

`まるくなる` の VolatileData 登録は正しい。変更なし。
