## 実装（handlers/ / data/）

**バグ発見・修正あり**。

### 問題

`src/jpoke/handlers/volatile.py` の `ロックオン_remove_volatile` 関数が、相手交代時にロックオン状態を正しく解除していなかった。

原因: 汎用 `remove_volatile` ヘルパーは `ctx.source`（交代するポケモン = 相手）に対して揮発状態の削除を試みていたが、ロックオン状態は自分（使用者）に付与されている。そのため正しい対象（自分）から削除できず、交代後もロックオン状態が残り続けていた。

### 修正内容

`ロックオン_remove_volatile` 関数を `バインド_remove` と同じパターンに変更し、`battle.foe(ctx.source)`（= 交代した相手から見た自分）からロックオン状態を削除するよう修正した。

```python
# 修正前
def ロックオン_remove_volatile(battle, ctx, value):
    return remove_volatile(battle, ctx, value, volatile="ロックオン")

# 修正後
def ロックオン_remove_volatile(battle, ctx, value):
    """相手が交代したときに自分のロックオン状態を解除する。"""
    mon = battle.foe(ctx.source)
    battle.volatile_manager.remove(mon, "ロックオン")
    return HandlerReturn(value=value)
```

`src/jpoke/data/volatile.py` の登録部分（subject_spec・イベント・priority）は計画書と一致しており変更なし。
