  追加効果技（クロスポイズン、ヘドロこうげき など）と同様に `docs/spec/moves/_move_secondary_1.md`
  （86行目: `どくづき	defender	0.3	...	1(どく)	...`）と `docs/spec/moves/_move_labels.md`
  （405行目、`## contact` セクション）の一覧表で管理する方針であり、一次情報（Wiki）と照合した結果、
  既に正しく記載されていた。個別ファイルを作成しないという既存の方針
  （あくのはどうレビュー・3ぼんのやレビュー等）を踏襲し、新規作成は行っていない。

  PP20・直接攻撃・追加効果「30%の確率で相手をどく状態にする」。すべて実装・進捗表と一致していた。

## 実装計画書
- `docs/plan/moves/どくづき.md` は存在しないが、グループ計画書 `docs/plan/moves/攻撃技_どく系.md` に
  どくづきの実装内容が記載されており、同種の単純な追加効果技には個別の計画書を作成しない慣例
  （あくのはどうレビュー等）に倣い、新規作成はしていない。
  なお同計画書内のコード例（`Event.ON_HIT` / `ha.どくづき_apply_ailment_to_defender` /
  `labels=[...]`）は執筆時点の草案であり、実際の実装は `Event.ON_DAMAGE_HIT` /
  `ha.どくづき_apply_poison_to_defender` / `flags={...}` という現行の命名・イベント規約に従って
  実装されている（他の同グループ技: どくばり・クロスポイズン・ヘドロこうげき 等もすべて同じ規約で
  統一されている）。この乖離はグループ全体に及ぶため、本タスクの対象（どくづき単体）を超える修正は
  見送った。

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/data/move.py` の `"どくづき"` エントリ

  `ha.どくづき_apply_poison_to_defender` を `MoveHandler`（デフォルト subject_spec="attacker:self"、
