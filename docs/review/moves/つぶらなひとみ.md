# つぶらなひとみ レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/つぶらなひとみ.md`）。`docs/wiki/moves/つぶらなひとみ.html` を参照し、
  タイプ・分類・PP・命中率・範囲・優先度・直接攻撃の有無・効果・判定フラグ（急所/追加効果/
  まもる/マジックコート/オウムがえし/よこどり）を記載した。PP は本家 Gen6-9 の 30 ではなく
  `docs/champions/move_list.txt` の値（20）を採用した旨を明記した。

## 実装計画書
- 既存の `docs/plan/moves/つぶらなひとみ.md` は「handlers 未実装」「PP: 30」「`data/move.py`
  エントリ」「`つきのひかり_heal_self` の後に挿入」など、現状の実装（handlers 実装済み、
  `data/moves/move_ta.py` に分割済み、実際の挿入位置は `ちょうはつ_apply` の後）と乖離した
  古い記述だったため、現行実装に合わせて全面的に書き直した。
- コード例の `ctx: EventContext` → `ctx: AttackContext`、`stats={"A": -1}` → `stats={"atk": -1}`、
  `category="変化"` → `category="status"`、`pp=30` → `pp=20` に修正した。
- 対象は相手（`target` はデフォルトの `"foe"` のまま。`target="self"` にしてはいけない）である旨を
  エッジケース節に明記した。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ta.py` の `つぶらなひとみ` エントリで `pp=30` になっていたバグを
  `pp=20` に修正した（`docs/champions/move_list.txt` の値が正。Gen9 本家の 30 のままだった
  移行漏れ）。
- `handlers/move_status.py` の `つぶらなひとみ_modify_defender_stats`（相手のこうげきを1段階
  下げる、`modify_defender_stats` 共通ヘルパー使用）自体は仕様通りで変更なし。`target` も
  デフォルトの `"foe"` のままで、まもる／マジックコートへの反応は
  `Move.is_blocked_by_protect` / `Move.is_reflectable` により自動的に正しく処理される
  （target="self" 誤設定のバグはなし）。
