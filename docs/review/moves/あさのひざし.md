## 実装（handlers/ / data/）
- `src/jpoke/handlers/move_status.py` の `あさのひざし_heal_self` は既に `battle.weather_for(mon)`
  を使っており、計画書の初版（`battle.weather` 直参照）より正しい実装になっていたため変更なし
- `src/jpoke/data/moves/move_a.py` の `"あさのひざし"` エントリを修正:
  - `pp=5` → `pp=8`（championsルール準拠。`src/jpoke/data/ps-champ-ja/moves.json` の値と一致させた）
  - `target="self"` を追加（**バグ修正**）。`MoveData.target` のデフォルト値は `"foe"` であり、
    未指定のままだと以下が誤って発生することを実測で確認した:
    - `move.is_blocked_by_protect` が `True` になり、相手の `まもる` で誤ってブロックされる
      （修正前: 相手が `まもる` を使っているとHP回復自体が失敗していた）
    - `マジックコート_reflect` の条件 `ctx.move.target in {"foe", "foe_side"}` を満たしてしまい、
      相手の `マジックコート` で誤って跳ね返される
    `target="self"` を追加することで両方が解消されることを確認済み
