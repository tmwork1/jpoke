## 実装（handlers/ / data/）

- `src/jpoke/data/move.py` の `"じごくづき"` エントリを修正した。
  - **`flags` に `secondary_effect` が欠落していたため追加**（`{"contact"}` → `{"contact", "secondary_effect"}`）。
    これにより特性ちからずくの「威力1.3倍・追加効果無効化」がじごくづきに対して機能していなかった。
  - **登録イベントを `Event.ON_HIT` から `Event.ON_DAMAGE_HIT` に修正**。
    `ON_HIT` はみがわりに防がれてダメージが0になった場合でも発火するため、みがわり状態の相手にも
    じごくづき状態が付与されてしまう不具合があった。同種の追加効果ハンドラ（`しおづけ`・`みずあめボム` 等）は
    すべて `ON_DAMAGE_HIT`（`actual_damage <= 0` のとき発火しない）を使用しており、これに合わせた。
  - `handlers/move_attack.py` の `じごくづき_apply_volatile_to_defender` 自体は変更なし（汎用ヘルパー
    `apply_volatile_to_defender` を呼ぶのみで問題なし）。
  - `python scripts/sort_data/sort_moves.py` を実行（差分なし、既存の並び順を維持）。
  - じごくづき状態側（`data/volatile.py` の `"じごくづき"` エントリ、`handlers/volatile.py` の
    `じごくづき_restrict_commands` / `じごくづき_try_action` / `じごくづき_tick_volatile`）は仕様書・実装ともに
    問題なし。変更なし。
