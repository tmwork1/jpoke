# ウェーブタックル レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/ウェーブタックル.md`）。`docs/wiki/moves/ウェーブタックル.html` を参照し、
  タイプ・分類・威力・命中率・対象・優先度・接触判定・反動比率（33%＝1/3）・
  判定フラグ（急所/命中判定/追加効果/まもる/おうじゃのしるし/マジックコート）をまとめた。
  PP は本家SVの基準値10ではなく `docs/champions/move_list.txt` の12を採用した。

## 実装計画書
- 新規作成（`docs/plan/moves/ウェーブタックル.md`）。`docs/plan/moves/とっしん.md` を
  テンプレートとして、ハンドラ構成（`ON_HIT` / priority=100 / `attacker:self`）、
  共通ヘルパー `_recoil` の利用、リーサル計算が対象外である根拠を記載した。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/move_attack.py` の `ウェーブタックル_recoil` は既存実装のまま
  （共通ヘルパー `_recoil(battle, ctx, value, 1/3)` を呼ぶのみで仕様と一致）。変更なし。
- `src/jpoke/data/moves/move_a.py` の `MoveData` で `pp=10`（本家SV基準値）のままに
  なっていたバグを修正し、チャンピオンズ準拠の `pp=12`（`docs/champions/move_list.txt`）に修正した。
  威力120・命中100・タイプみず・分類physical・`flags={"contact", "recoil"}` は仕様と一致していた。

## リーサル計算
- 対象外（`docs/progress/move.md` の「リーサル実装」列は n/a）。反動技は攻撃者自身のHPを
  減らすのみで、防御側の HP 分布 (`StateDist`) には影響しないためリーサル計算ハンドラの
  実装対象にはならない。

## テスト
- 変更なし。`tests/moves_attack/test_move_a.py` の
  `test_ウェーブタックル_使用後に攻撃者が反動ダメージを受ける` と、
  `tests/moves_attack/test_move__group.py` の `test_recoil技グループ_反動量が与ダメのrecoil_ratio倍になる`
  （`("ウェーブタックル", 1/3)` パラメータ）で反動量まで含めて既に検証されており、
  追加・修正の必要はなかった。
