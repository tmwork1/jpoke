# アストラルビット レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/アストラルビット.md`）。一次情報（`docs/wiki/moves/アストラルビット.html`）を基に、
  タイプ・分類・威力・命中率・判定フラグ（急所あり／まもる有効／おうじゃのしるし発動／マジックコート対象外等）、
  バドレックス（こくばじょうのすがた）専用技である旨を記載した。
- PPについては Gen9 本家値（5）と Champions 変換値（8）を明記し、`src/jpoke/data/ps-champ-ja/moves.json` に本技の
  記載がないため、base PP5の他技（でんじほう・はかいこうせん・じばく・だいばくはつ）が軒並み8に変換されている
  実例から類推した根拠を明記した。

## 実装計画書
- 新規作成（`docs/plan/moves/アストラルビット.md`）。追加効果を持たない通常攻撃技のため専用ハンドラ不要である旨、
  PPをChampions基準の8にする根拠、targetはデフォルトの`"foe"`のままでよい旨（本プロジェクトはシングルバトル
  のみを対象とするため「相手全員」用の別ターゲット種別が存在しない）を記載した。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_a.py` の `MOVES["アストラルビット"]` の `pp` を `5` → `8` に修正した
  （Champions のPP統一ルール：全技PPは8/12/16/20の4段階。base PP5の技は8に変換される）。
  それ以外のタイプ・分類・威力・命中率・handlers（追加効果なし）は仕様通りで変更なし。

## リーサル計算
- 対象外（進捗表の「リーサル実装」列が`n/a`。追加効果を持たない通常攻撃技のためリーサル専用ハンドラは不要）。

## テスト
- `tests/moves_attack/test_move_a.py` に `test_アストラルビット_タイプ分類威力命中が正しく反映されダメージのみ与える`
  を追加した。ゴーストタイプの攻撃技であるため、ノーマルタイプ（無効）を避け、でんきタイプのポケモンを
  防御側に使用してダメージが発生することと、追加効果（ランク変化・状態異常）が発生しないことを確認する。
