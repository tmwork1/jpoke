# かわらわり レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/かわらわり.md`）。
- 一次情報（`docs/wiki/moves/かわらわり.html`）の「技の仕様」節を精読し、以下を明記した。
  - 壁の解除はダメージを与える前に行われるため、かわらわり自身のダメージは
    リフレクター/オーロラベールの軽減を受けない。
  - 壁解除は追加効果ではない（りんぷん・おんみつマント・ちからずくの対象外）。
  - 技が外れた場合・まもるに防がれた場合は壁を解除できない。
  - みがわりに防がれた場合は壁を解除できる。
  - 相性・とくせいで無効化された場合、第五世代以降は壁を解除できない（本プロジェクトの採用挙動）。

## 実装計画書
- 新規作成（`docs/plan/moves/かわらわり.md`）。

## 実装（handlers/ / data/）
- **バグ修正**: `かわらわり_break_screens` の発動イベントが `Event.ON_HIT`（ダメージ計算後）に
  誤って登録されており、かわらわり自身のダメージがリフレクター/オーロラベールの軽減を
  受けてしまっていた（一次情報の「かわらわりによるダメージは、相手の場のリフレクター/
  オーロラベール状態の影響で減衰しない」に違反）。
  `Event.ON_BEFORE_APPLY_MOVE`（priority=30, ダメージ計算より前）に修正した。
  - `src/jpoke/data/move.py`: `Event.ON_HIT` → `Event.ON_BEFORE_APPLY_MOVE`（priority=30）
  - `src/jpoke/handlers/move_attack.py`: docstring を実際の発動イベントに合わせて更新
  - この修正により、命中判定に外れた場合・まもるに防がれた場合・タイプ相性で無効化された
    場合に壁が解除されない挙動は従来通り維持される（それぞれ本ハンドラの到達前に
    `_execute_move` が中断されるため）。
- `python scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` を実行。
- `python scripts/sort_data/sort_moves.py` を実行。
