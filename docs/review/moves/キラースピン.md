# キラースピン レビュー結果

## 仕様書
- 変更なし
- `docs/spec/moves/こうそくスピン.md`（こうそくスピン・キラースピン共通仕様書）の内容を一次情報
  （`docs/wiki/moves/キラースピン.html`）と照合し、誤り・欠落がないことを確認した。
  - 基本データ（どく/物理/威力30/命中100/追加効果どく100%）、バインド・やどりぎのタネ解除、
    自分の場のまきびし・どくびし・ステルスロック・ねばねばネット解除、りんぷん・おんみつマント
    無効化なし（解除自体は発動）、ちからずくブロックの記述はすべて一次情報と一致していた。
  - Champions固有の「使用者がひんしになる場合でも場の状態解除ができる」（SVとの差異）は、
    実装が `Event.ON_HIT`（`Event.ON_DAMAGE_HIT` より前に発火し、ゴツゴツメット等の反動より
    先に解除処理が走る）を採用しているため、仕様書に明記せずとも自然に満たされていることを確認した。
  - 複数対象時の仕様（ニードルガード関連）はダブルバトル専用の記述であり、本プロジェクトの対象は
    シングルバトルのみのため対象外。

## テスト
- 追加した内容（`tests/moves_attack/test_move_ka.py`）

    りんぷん・おんみつマントが「どく付与」だけを無効化し、「場の状態解除」は無効化しないという
    仕様上重要な非対称性を検証するテストを追加した（既存はこうそくスピン側にのみ同種のテストが
    存在し、キラースピン側には無かった）。

    使用者がちからずくの場合、場の状態解除・どく付与のいずれも発動しないことを検証するテストを
    追加した。
  - `test_キラースピン_まきびしを解除する`: 既存はステルスロックのみのテストだったため、
    まきびし解除も検証するテストを追加した（こうそくスピン側と同等のカバレッジに揃えた）。
  - `tests/test_lethal.py` の既存テスト `test_キラースピン_どく付与_secondary有り` は
    `れんごく_apply_やけど` と同等のパターンで実装されており、修正なし。
  - `python -m pytest tests/ -v` で全3710件（+1 skip）がパスすることを確認した
    （`.loop/test_logs/キラースピン.log`）。
