## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ka.py` の `MoveData("ゴーストダイブ")` に以下のバグを発見し、修正した。
  - **2ターン技としての `ON_MOVE_CHARGE` ハンドラが未登録で、`handlers={}` のまま放置されていた**。そのままでは通常の1ターン攻撃技として動作してしまい、「1ターン目に姿を消し2ターン目に攻撃する」という技の根幹仕様が実装されていなかった。`シャドーダイブ`（`move_sa.py`）と同じパターンで `Event.ON_MOVE_CHARGE: h.MoveHandler(lambda b,c,v: h.charge_into_volatile(b,c,v,"シャドーダイブ"))` を追加した。
  - `flags` に `unprotectable` が欠落しており、まもる系を無視して攻撃するという仕様（Wiki一次情報・champions効果文とも明記）が反映されていなかったため追加した。
  - `pp=10`（Gen9本家準拠、champions移行漏れのバグ）を `pp=12`（`docs/champions/move_list.txt` 290行目）に修正した。
- 揮発状態側（`src/jpoke/data/volatile.py` の `"シャドーダイブ"` エントリ、`src/jpoke/handlers/volatile.py` の関連ハンドラ）はシャドーダイブ用に実装済みのものをそのまま再利用でき、変更不要だった。

## テスト
- `tests/moves_attack/test_move_ka.py` に以下を追加した（`ゴッドバード` と `ゴールドラッシュ` のテストの間、五十音順の暫定位置）。
  - `test_ゴーストダイブ_2ターンで攻撃する`: 1ターン目はダメージなし・`"シャドーダイブ"` 揮発状態が付与、2ターン目でダメージが発生し揮発状態が解除されることを確認。
  - `test_ゴーストダイブ_まもる中の相手にダメージを与えられる`: `unprotectable` フラグにより、まもる状態の相手にもダメージを与えられることを確認（`test_シャドーダイブ_まもる中の相手にダメージを与えられる` と同型）。
  - 防御側はノーマルタイプ（ゴースト技を無効化してしまう）を避け、どくタイプの「ドガース」を使用した（既存のシャドーダイブ用テストと同じ理由）。
  - 当初計画していた「1ターン目はPPを消費しない」テストは、`tests/test_utils.py` の `run_move` ヘルパーが `core/command_manager.py` の強制行動時PP消費抑制ロジック（新規 `Move` インスタンス生成）を経由しないため、2連続 `run_move` では正しく検証できないと判明し、追加を見送った。この観点は既存の `tests/volatiles/test_volatile_ka.py::test_かくれる_強制行動ターンはPPを消費しない`（`hidden_move_name` パラメトライズに `"シャドーダイブ"` を含む）が正しいコマンド解決フローで既に検証済みのため、重複を避けた。
- `python -m pytest tests/ -v` を実行し、3758 passed, 1 skipped（既存の無関係なskip）で全テスト成功を確認した。ログは `C:\Users\tmtmp\Documents\pokemon\jpoke\.loop\test_logs\ゴーストダイブ.log` に保存済み。
