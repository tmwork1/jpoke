## 仕様書

- 新規作成（`docs/spec/moves/ふぶき.md`）。一次情報（`docs/wiki/moves/ふぶき.html`）と
  `src/jpoke/data/ps-champ-ja/moves.json` を基に、タイプ・分類・威力・命中率・PP・追加効果
  （10%こおり付与）・ゆき天候時の必中補正・風の技フラグなどを記載した。
- PPは一次情報（ポケモンWiki基準、PP5）と実際のプロジェクト値（PP8、
  `src/jpoke/data/ps-champ-ja/moves.json` 準拠）が異なる点を明記した。

## 実装計画書

- 新規作成（`docs/plan/moves/ふぶき.md`）。既存実装とのレビュー結果として検出した誤りを記載し、
  ハンドラ構成・priority根拠・エッジケースを整理した。

## 実装（handlers/ / data/）

- `src/jpoke/data/moves/move_ha.py`: `ふぶき` の `pp` が `5`（一次情報＝ポケモンWikiの値）に
  なっていた誤りを `pp=8`（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）に修正した。
- `src/jpoke/handlers/move_attack.py`: `ふぶき_accuracy` が `battle.weather_manager.current.name`
  を直接参照しており、エアロック・ノーてんきによる天候無効化を考慮せずに必中判定していた誤りを、
  `battle.weather_for(ctx.defender).name == "ゆき"` を使う実装（`かみなり_accuracy` 等の既存パターンと
  同様）に修正した。
  - なお、ばんのうがさ（`ばんのうがさ_weather_immune`）は「はれ・あめ・おおひでり・おおあめ」のみを
    対象とし『ゆき』は対象外（実際のゲーム仕様通り）のため、ばんのうがさ所持者に対しても
    ゆき天候による必中補正は通常通り適用される。これは修正対象ではないことを確認した。
- `scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` と
  `scripts/sort_data/sort_moves.py` を実行し、並び順を整えた。
