# きりばらい レビュー結果

## 仕様書
- 既存の `docs/spec/moves/きりばらい.md` は概ね一次情報（`docs/wiki/moves/きりばらい.html`）と
  一致していたが、処理順序に関する記述が欠けていたため「処理順序」節を追加した。
  - 回避率変化を場の効果解除より先に判定すること
  - しろいきりが回避率変化を防いだ後、しろいきり自体を含む場の効果は通常通り解除されること
  - クリアボディ等の特性・クリアチャームで回避率変化が防がれた場合も場の効果解除は通常通り
    行われること
- PPは仕様書内では一次情報（Gen9本家）どおり15のまま変更していない（`docs/champions/move_list.txt`
  の16は実装側でのみ反映する）。

## 実装計画書
- 存在しなかったため新規作成した（`docs/plan/moves/きりばらい.md`）。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ka.py`
  - PPが15（Gen9本家値）になっていたため、`docs/champions/move_list.txt` の値に合わせて
    16へ修正した。
  - `flags={"bypass_substitute"}` を追加した。追加しないと、みがわり状態の相手に対して
    `ON_BEFORE_APPLY_MOVE` の汎用みがわり判定（`みがわり_immune`）により変化技全体が
    ブロックされ、場の効果解除まで発動しなくなってしまうバグがあったため。
- `src/jpoke/handlers/move_status.py` の `きりばらい_defog`
  - 処理順序のバグを修正した。修正前は「壁・トラップ・フィールドの解除」→「回避率変化」の
    順で実装されていたため、対象側にしろいきりがある場合に、回避率変化を試みる前に
    しろいきりが除去されてしまい、本来防がれるはずの回避率変化が誤って成功してしまう
    問題があった。一次情報の「回避率の低下を防がれた後に、しろいきり状態を含めた場の状態が
    解除される」という記述に合わせ、回避率変化を先に行うよう順序を入れ替えた。
  - みがわり状態の相手に対しては `ctx.defender.has_volatile("みがわり")` を明示的に判定し、
    回避率変化のみをスキップするようにした（場の効果解除は `bypass_substitute` フラグにより
    通常通り実行される）。
  - 戻り値を `modify_defender_stats` の戻り値（能力変化の有無）のみに依存させず、
    回避率変化・壁解除・トラップ解除・フィールド解除のいずれかが実際に発生したかを
    集計して技の成否（`HandlerReturn(value=changed)`）とするよう修正した。
