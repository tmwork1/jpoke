## 実装（handlers/ / data/）
- `src/jpoke/handlers/move_status.py` の `パワートリック_swap_stats` はランク非依存で実数値インデックス1・2を入れ替える実装で仕様通り。変更なし
- `src/jpoke/data/moves/move_ha.py` の `"パワートリック"` エントリに以下のバグを発見・修正した:
  - **PP誤り**: `pp=10`（Gen9本家基準の移行漏れ）→ `pp=12`（`docs/champions/move_list.txt` 準拠）に修正
  - **`target` 未指定バグ**: `target` を省略すると `MoveTarget` のデフォルト値 `"foe"` が使われ、
    `Move.is_blocked_by_protect`（`target == "foe"` で True）・`Move.is_reflectable`
    （`target in ("foe", "foe_side")` で True）が誤って True になり、まもる・マジックコートに
    阻害されてしまう不具合があった。`target="self"` を明示指定して修正（`テクスチャー`・
    `つるぎのまい` 等の自分対象変化技と同一パターン）
  - 不要だった `accuracy=100`（自分対象の必中技のため）を削除（`テクスチャー`・`つるぎのまい` 等と同様に省略し `None`＝必中とする）
- `Pokemon.reset_on_switch_out()` が `update_stats()` を呼び、交代時に実数値を種族値ベースで
  再計算する既存修正（更新済み）を確認。パワートリックによる実数値入れ替えが交代で正しく
  解除されることを確認した（問題なし）
