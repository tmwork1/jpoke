# ポイズンアクセル レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/ポイズンアクセル.md`）。一次情報（docs/wiki/moves/ポイズンアクセル.html）と
  同系統「アクセル技」の既存レビュー済み仕様書（`docs/spec/moves/バーンアクセル.md`）を参照して作成した。
  - タイプ: どく / 分類: 物理 / 威力: 100 / 命中: 100 / PP: 10（本家）→ 12（Champions換算）
  - 対象: 相手1体 / 優先度: 0 / 接触なし（直接攻撃ではない）
  - 追加効果: 30%の確率で相手を「どく」状態にする
  - アンコール・ものまね・まねっこ・さいはいは失敗する（うちアンコール・まねっこのみ本プロジェクトで実装対象）
  - オウムがえしは反射不可（一次情報のオウムがえし可否フラグに明記。バーンアクセル仕様書の
    「反射可能」記載は誤りの可能性があるが、対象外の技のため本レビューでは修正しない）

## 実装計画書
- 新規作成（`docs/plan/moves/ポイズンアクセル.md`）。`docs/plan/moves/攻撃技_どく系.md` に記載されていた
  草案（威力70・PP20・`labels=["contact", "secondary_effect"]`・`Event.ON_HIT`）は一次情報未確認の
  誤りであったため、同ファイル冒頭に「ダブルニードル」「フェイタルクロー」と同様の訂正追記を行い、
  個別計画書に置き換えたことを明記した。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ha.py` の `"ポイズンアクセル"` エントリを修正:
  - `power`: `70` → `100`（一次情報の威力100と不一致だった）
  - `pp`: `20` → `12`（一次情報のPP10をChampions基準4段階（8/12/16/20）に変換した値。
    本家PP10の技が12に変換される前例（10まんばりき等）と同様の規則を適用）
  - `flags`: `{"contact", "non_copycat", "secondary_effect"}` →
    `{"non_copycat", "non_encore", "secondary_effect"}`
    （`"contact"` を削除: 本技は直接攻撃ではない非接触技のため誤り。
    `"non_encore"` を追加: アンコール失敗の仕様が反映されていなかった）
- `src/jpoke/handlers/move_attack.py` の `ポイズンアクセル_apply_poison_to_defender`
  （`apply_ailment_to_defender(..., ailment="どく", chance=0.3)`）は仕様と一致しており修正不要。

## リーサル計算
