# だいふんげき レビュー結果

## 仕様書

- `docs/spec/moves/だいふんげき.md` は存在しない。だいふんげき固有の効果（強制連続攻撃・
  使用後こんらん）は、あばれる・げきりん・はなびらのまい・だいふんげきで共用の
  揮発性状態として `docs/spec/volatiles/あばれる.md` に既に詳細な仕様書
  （付与条件・消滅条件・こんらん付与の特例・ターン処理優先度）が存在しており、
  これが技レベルの効果を包含している。`あばれる`（技）・`げきりん` のレビュー
  （`docs/review/moves/あばれる.md`, `docs/review/moves/げきりん.md`）で踏襲された
  方針（共用仕様書がある場合は技単体の個別ファイルを新規作成しない）に倣い、
  新規作成は行わなかった。
- 一次情報（`docs/wiki/moves/だいふんげき.html`）と実装を照合し、以下の誤りを発見した：
  - タイプ＝ほのお（一致）、**分類＝物理**（実装は誤って `special` になっていた）、
    **威力＝120**（実装は誤って `90` になっていた）、命中率＝100（一致）、
    PP＝10（一致）、範囲＝相手ランダム1体（シングル対象のため実質相手1体固定）、
    直接攻撃＝×（一致、`flags` に `contact` なし）、追加効果＝×、まもる＝○、
    おうじゃのしるし（第五世代以降）＝○。
  - 一次情報には「だいふんげきはさいはい（Instruct）に対して例外的に成功する」という
    固有仕様が記載されている。これは `docs/spec/moves/さいはい.md` に既に反映済み
    （`さいはいが失敗する技` の節で「例外的に、だいふんげきは指示できる」と明記）。
    さいはい自体は `docs/progress/move.md` で「ダブル専用」として扱われ本プロジェクト
    （シングルバトル専用）の対象外であるため、追加の実装対応は不要と判断した。

## 実装計画書

- `docs/plan/moves/だいふんげき.md` は存在しない。同様の理由で、あばれる系共有の
  揮発性状態実装（`あばれる_apply` / `あばれる_tick` / `force_command`）が
  技の実装内容を実質的にカバーしているため、個別の新規作成は行わなかった。

## 実装（handlers/ / data/）

  - `category="special"` → `category="physical"` に修正（一次情報では物理技）。
  - `power=90` → `power=120` に修正（一次情報の威力と一致させた）。
  - `pp=10`, `accuracy=100`, `Event.ON_HIT` に `ha.あばれる_apply` を登録している点は
    既に一次情報と一致しており変更なし。
  - 修正後 `python scripts/sort_data/sort_moves.py` を実行したが、キー順は変更なし
    （値のみの修正のため）。
- `src/jpoke/handlers/move_attack.py` の `あばれる_apply`
  （`ON_HIT`, `subject_spec="attacker:self"`）: 初回命中時にのみランダム
  2〜3ターンの揮発性状態を付与し、既に付与済み（強制行動2ターン目以降）の
  場合は何もしない実装で仕様と一致。あばれる・げきりん・はなびらのまいと共用。
  変更なし。
- `src/jpoke/data/volatile.py` の `"あばれる"`（`forced=True`,
  `ON_MODIFY_COMMAND_OPTIONS` で `force_command`, `ON_DAMAGE_HIT`

  `あばれる_tick`（ターン経過後カウント0でこんらん付与）は
  `docs/spec/volatiles/あばれる.md` の仕様通りで、既存レビュー（あばれる・げきりん）
  で確認済み。変更なし。
