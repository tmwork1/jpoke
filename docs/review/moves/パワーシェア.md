# パワーシェア レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/パワーシェア.md`）。既存の `ガードシェア.md` を対称技のテンプレートとして、
  `docs/wiki/moves/パワーシェア.html` の記述に沿って作成した。

## 実装計画書
- 既存の計画書 (`docs/plan/moves/パワーシェア.md`) を仕様書・実装と照合し、以下を修正した。
  - `data/move.py` のエントリ例が誤って `accuracy=100` になっていたのを `accuracy=None`（必中）に修正。
    `accuracy=100` のままだと `ON_MODIFY_ACCURACY` やランク補正（相手の回避率上昇）の影響を受けてしまい、
    Wiki記載の「命中判定: ×（命中判定自体が行われない）」と矛盾する。
  - `flags={"unreflectable"}`（マジックコートで跳ね返されない）の付与漏れを追記。
  - ファイルパスの記述を `src/jpoke/data/move.py` → `src/jpoke/data/moves/move_ha.py`（実際の分割ファイル）に修正。

## 実装（handlers/ / data/）
- `handlers/move_status.py` の `パワーシェア_equalize_stats` はガードシェアと対称の実装で問題なし（変更なし）。
- `data/moves/move_ha.py` の `"パワーシェア"` エントリに以下のバグを発見し修正した。
  - **PP誤り**: `pp=10`（Gen9本家値のまま）になっていたのを `pp=12`（`src/jpoke/data/ps-champ-ja/moves.json` 基準）に修正。
  - **命中判定誤り**: `accuracy=100` になっていたのを `accuracy=None`（必中・命中判定自体を行わない）に修正。
    修正前は相手の回避ランクが高いと理論上ミスしうる状態だった。
  - **マジックコート耐性の欠落**: `flags={"unreflectable"}` が付与されておらず、デフォルトの
    `Move.is_reflectable`（`status` かつ `target in ("foe", "foe_side")` の技は反射可能）判定により
    マジックコートで跳ね返される実装になっていたのを修正。
  - いずれも対称技である `ガードシェア` エントリとの比較で発見。
- `Pokemon.reset_on_switch_out()` の `update_stats()` 呼び出し（過去修正済みの共通バグ）は現状のコードに
  正しく存在することを確認済み（コメントにも「ガードシェア・パワーシェア・パワートリックなどによる
  実数値の書き換えを...リセットする」旨が明記されている）。
