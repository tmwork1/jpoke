## 実装（handlers/ / data/）
- 修正: `src/jpoke/data/moves/move_ra.py` の `"リサイクル"` エントリで `pp=10`
  （Gen9本家基準の移行漏れ）となっていたのを `pp=12`（`docs/champions/move_list.txt` 準拠）に修正した。
- `handlers/move_status.py` の `リサイクル_can_apply` / `リサイクル_restore_item` 自体は
  仕様・計画書と一致しており誤りなし。
- **`item_manager` の `track_loss` 判定漏れを修正**（リサイクル仕様書「持ち物を消費したかどうかの
  判定」に直結する不具合。既存の `swap_items`（トリック・すりかえ・どろぼう等）は
  `track_loss=False` を正しく使っていたが、以下は素の `remove_item`/`consume_item` を呼んでおり
  `last_lost_item_name` を誤って更新していたため修正した）:
  - `src/jpoke/core/item_manager.py`: `remove_item` / `consume_item` / `force_trigger_berry` に
    `track_loss: bool = True` 引数を追加し呼び出し元へ伝播できるようにした。
    `force_trigger_berry` は個々のきのみハンドラを変更せずに済ませるため、発動前後で
    `last_lost_item_name` を退避・復元する方式にした。
  - `src/jpoke/handlers/move_attack.py` の `はたきおとす_remove_item` / `やきつくす_burn_item`、
    `src/jpoke/handlers/move_status.py` の `ふしょくガス_remove_item`:
    `remove_item(..., track_loss=False)` を指定（場に存在したまま消滅する扱いのため）。
  - `src/jpoke/handlers/item.py` の `_announce_and_consume_item` に `track_loss` 引数を追加し、
    `ふうせん_pop_on_hit` から `track_loss=False` を渡すよう修正（攻撃を受けて割れたふうせんは
    復元対象にならないため。`docs/spec/items/ふうせん.md` の既存規定と一致）。
  - `src/jpoke/handlers/move_attack.py` の `むしくい_steal_and_use_berry`: 相手から奪って消費した
    きのみは復元対象にならないため `force_trigger_berry(attacker, track_loss=False)` に修正。
  - `おちゃかい_force_consume_berries` / `ほおばる_consume_berry_and_boost` /
    `なげつける_consume_item` / 通常のHP閾値きのみ消費は仕様上復元対象のため、
    デフォルトの `track_loss=True` のまま変更なし。
- **発見したが本タスクの対象外として修正しなかった既知の問題**: `はたきおとす` は
  `Event.ON_DAMAGE_HIT`（`modify_hp` によるダメージ適用・`ON_HP_CHANGED` 発火より後）で
  道具を除去しているため、相手がHPしきい値回復きのみ（オボンのみ等）を持っている場合、
  はたきおとすで奪う前にきのみが自動発動してしまう
  （`docs/spec/moves/はたきおとす.md`「持ち物との発動順序」の規定に反する）。
  `src/jpoke/data/moves/move_ha.py` の該当箇所に `# TODO` コメントを追加し、
  はたきおとす自体の damage パイプライン修正が必要な別課題として明記した
  （本レビューの対象はリサイクルのため、はたきおとす側のイベント設計変更は行わなかった）。
