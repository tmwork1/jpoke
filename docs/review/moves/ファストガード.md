## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ha.py`: `pp=15` → `pp=16` に修正
  （`docs/champions/move_list.txt` 744行目でPPは16。Gen9本家の15はchampions未移行のバグだった）
- `src/jpoke/handlers/volatile.py`: `ファストガード_protect` の優先度判定を修正
  - 修正前: `ctx.move.priority < 1`（技データの静的な `priority` フィールドを直接参照）
  - 修正後: `battle.speed_calculator.calc_move_priority(ctx.attacker, ctx.move) < 1`
    （`じょおうのいげん_block_priority`・`はやてがえし_can_apply` と同じパターンで実効優先度を算出）
  - 不具合の内容: `いたずらごころ`は `ON_MODIFY_MOVE_PRIORITY` イベントで動的に変化技の優先度を
    +1するのみで、`move.data.priority`（静的値）は変更しない。そのため修正前の実装では、
    技データ上は優先度0の変化技（例: でんじは）が `いたずらごころ` で実質優先度+1になっていても
    `ctx.move.priority` は0のままとなり、ファストガードで防げなかった。一次情報「技の仕様」節にも
    「いたずらごころの効果による技など」も防げる旨が明記されており、仕様違反だった
  - 修正の妥当性は `battle.speed_calculator.calc_move_priority` を用いる既存の同種実装
    （`じょおうのいげん_block_priority`・`はやてがえし_can_apply`）と同一パターンであることで確認した
