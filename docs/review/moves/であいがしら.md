## 実装（handlers/ / data/）
- 誤り・欠落を修正した。
  - `src/jpoke/data/move.py`: `power` を `90` → `100` に修正（Champions での威力変更が
    未反映だった）。
  - `src/jpoke/data/move.py`: `handlers={}`（追加効果なしのコメント付き）のままで、
    本技の根幹である成功条件（場に出てから最初の行動でなければ失敗する）が全く
    実装されていなかったため、`Event.ON_TRY_MOVE_1`（priority=30。
    `docs/spec/turn.md` の「30 | ねこだまし等: すでに行動をした」に対応）へ
    `であいがしら_check_first_turn` を新規登録した。
  - `src/jpoke/handlers/move_attack.py`: `であいがしら_check_first_turn` を新規実装
    （`ctx.attacker.acted_since_switch_in` が `True` なら `MOVE_FAILED` ログを出して
    不発にする）。`scripts/sort_handlers.py` を実行済み。
  - `src/jpoke/model/pokemon.py`: 「場に出てから一度でも行動したか」を表す新しい
    フラグ `acted_since_switch_in: bool` を追加し、`reset_on_switch_in()` で
    `False` にリセットするようにした（既存の `active_turn` は不使用のまま残る
    未使用フィールドだったため、新規フィールドとして切り出した）。
  - `src/jpoke/core/move_executor.py`: `MoveExecutor.run_move()` の `finally` 節
    （`failed_or_immobile_last_turn` を記録している箇所の直後）で
    `attacker.acted_since_switch_in = True` を無条件に設定するよう追加した。
    これにより、通常の技使用（成功・失敗・タイプ相性による無効を問わない）や、
    まひ・ねむり・こおり・こんらん自傷・ひるみ・なまけ等で行動できなかった場合も
    「行動済み」として記録され、次回以降の成功条件判定に反映される。であいがしら
    自身の成功判定（`ON_TRY_MOVE_1`）はこの `finally` 節より前（`_execute_move` の
    内部）で実行されるため、真に最初の行動である場合は正しく成功する。
  - `scripts/sort_data/sort_moves.py` を実行済み。
