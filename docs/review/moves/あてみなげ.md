  タイプ・分類・威力・命中率（必中）・PP・優先度・接触の有無・判定フラグ・備考をまとめた。

## 実装計画書
- 新規作成（`docs/plan/moves/あてみなげ.md`）。追加効果を持たない技であり、専用ハンドラは不要で
  `MoveData` のフィールドのみで挙動が完結することを明記した。

## 実装（handlers/ / data/）
- `src/jpoke/data/move.py` の `MOVES["あてみなげ"]` が一次情報と一致しない誤ったデータになっていたため修正した。
  - 修正前: `type="ノーマル"`, `power=50`, `accuracy=100`, `pp=20`（別の技のデータが誤って登録されていたと見られる）
  - 修正後: `type="かくとう"`, `power=70`, `accuracy=None`（必中）, `pp=10`, `priority=-1`, `flags={"contact"}`
  - `priority=-1` は既存の値のまま正しかったため変更なし。
  - `accuracy=None` は `core/move_executor.py` の `is_hit()` が `move.accuracy is None` の場合に無条件で命中と
    判定する既存の仕組み（`スピードスター` `でんげきは` `マジカルリーフ` `クリアスモッグ` 等と同様）で処理されるため、
    専用ハンドラの追加は不要と判断した。
  - `handlers={}`（追加効果なし）は変更なし。
  - `python scripts/sort_data/sort_moves.py` を実行し、辞書順を確認（並び替えによる差分なし）。

## テスト
- `tests/moves_attack/test_move_a.py` にテストが存在しなかったため、以下を追加した。
  - `test_あてみなげ_相手の回避率が高くても必ず命中する`: 相手の回避率を+6段階しても命中してダメージを与えることを確認。
  - `test_あてみなげ_優先度がマイナス1のため素早さが低い相手より後攻になる`: 素早さの低い相手（ヤドン）より
    後に行動することを確認。
- `python scripts/sort_tests.py tests/moves_attack/test_move_a.py` でソート、
  `python scripts/generate_test_list.py` でテスト一覧を更新した。
- `python -m pytest tests/ -v` を実行し、追加した2件はいずれも PASSED。
  なお、他の並列タスクによる未コミットの変更（`pokedex.json` / `model/stats.py` 等）に起因すると見られる
  無関係な既存失敗が7件あったが、本タスクの対象ファイルとは無関係であることを確認済み
  （`うなぎのぼり` / `ほのおのたてがみ` 特性が `data/ability.py` に未登録であることによるKeyError等）。
