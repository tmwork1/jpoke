# サイコファング レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/サイコファング.md`）。
  一次情報（`docs/wiki/moves/サイコファング.html`）に基づき、基本情報・効果・判定フラグを記載。
  PPは本家Gen9基準（10）ではなく `docs/champions/move_list.txt` の値（12）を明記した。
  同種の壁解除効果を持つ かわらわり の仕様書を参照し、壁解除の発動タイミング
  （ダメージ計算前）や各種判定（まもる／みがわり／タイプ相性無効時の挙動）を揃えた。

## 実装計画書
- 新規作成（`docs/plan/moves/サイコファング.md`）。
  `docs/plan/moves/かわらわり.md` と同一パターンの壁解除ロジックとして計画を記載。
  Priority根拠（`docs/spec/turn.md` の `Event.ON_APPLY_MOVE`／コード上は
  `Event.ON_BEFORE_APPLY_MOVE`、priority=30）を明記し、リーサル計算は対象外（n/a）と結論。

## 実装（handlers/ / data/）
- **バグ修正**: `data/moves/move_sa.py` の `サイコファング` エントリで、壁解除ハンドラ
  （`サイコファング_break_screens`）が `Event.ON_HIT`（ダメージ適用後に発火）に
  登録されていた。これはダメージ計算（`roll_damage`）より後に壁を解除することになり、
  相手側にリフレクター／ひかりのかべ／オーロラベールが張られている場合、
  サイコファング自身のダメージが誤って軽減されてしまう不具合だった。
  同種の壁解除技である かわらわり は既に `Event.ON_BEFORE_APPLY_MOVE`（priority=30）に
  修正済みであり、本技も同じ修正を適用した。
  （なお `レイジングブル` にも同種のタイミング誤り（`Event.ON_HIT`登録）が残っているが、
  本タスクの対象外のため未修正のまま。）
- `handlers/move_attack.py` の `サイコファング_break_screens` のdocstringを、
  実際の発動イベント（`Event.ON_BEFORE_APPLY_MOVE`）に合わせて更新した。
- **PP修正**: `pp=10`（Gen9本家基準）になっていたが、`docs/champions/move_list.txt` の
  値（PP12）に修正した（進捗表のPP列は元々12で一致していたため、進捗表ではなく
  実装側が移行漏れのバグだった）。

## リーサル計算
- 対象外（n/a）。壁解除効果はHP・ダメージ量の分布に影響を与えないため、
  同種の壁解除技である かわらわり（n/a）・レイジングブル（n/a）と同様に
  専用のリーサル計算ハンドラは不要と判断した。

  「n/a」に修正した。
