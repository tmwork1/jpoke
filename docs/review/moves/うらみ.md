# うらみ レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/うらみ.md`）
  - `docs/wiki/moves/うらみ.html` の一次情報を基に、基本情報・効果・失敗条件・判定（急所/命中/追加効果/まもる/マジックコート/みがわり）をまとめた
  - 失敗条件として「未行動」「直前の技がわるあがき」「直前の技のPPがすでに0」の3つを明記
  - `executed_move` の更新タイミング（`ON_TRY_ACTION` が偽の場合は更新されない）により、
    「最後にPPを消費した技を対象にする」仕様がエンジン側で自然に満たされることを記載

## 実装計画書
- 修正した内容:
  - PPを `10` → `12`（`docs/champions/move_list.txt` 準拠）に修正
  - `flags={"bypass_substitute"}` の追加を明記（みがわり貫通の一次情報に対応する実装が欠落していたため）
  - `うらみ_can_apply` の失敗条件に「直前の技がわるあがき」「直前の技のPPがすでに0」を追加
  - `うらみ_deplete_pp` から不要になった `move is None` ガードを削除（`can_apply` で確定済みのため）
  - 挿入位置の記述を実ファイルの関数名（`うたう_apply_sleep` の後）に修正

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_a.py`
  - `pp=10` → `pp=12`（`docs/champions/move_list.txt` の値に合わせて修正。Gen9本家値からの移行漏れ）
  - `flags={"bypass_substitute"}` を追加（一次情報「みがわり状態のポケモンにも効果がある」に対応、従来はみがわりでブロックされてしまう不具合だった）
- `src/jpoke/handlers/move_status.py`
  - `うらみ_can_apply`: `ctx.defender.executed_move is None` のみのチェックだったのを、
    `move.name == "わるあがき"`（PPが概念上無限のため対象外）と `move.pp <= 0`
    （すでにPPが尽きている技への使用は失敗）の条件を追加
  - `うらみ_deplete_pp`: `can_apply` で失敗条件が確定しているため、重複していた `move is None` ガードを削除
