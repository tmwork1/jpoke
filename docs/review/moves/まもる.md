# まもる レビュー結果

## 仕様書

新規作成: `docs/spec/moves/まもる.md`

一次情報（`docs/wiki/moves/まもる.html`、`docs/wiki/volatiles/まもる_(状態変化).html`）と
`src/jpoke/data/ps-champ-ja/moves.json` をもとに作成。主な記載内容:
- 基本情報（タイプ/分類/PP=8/優先度+4/対象=自分/接触なし）
- 連続使用による成功率低下（本来は1/3ずつ低下・下限1/729だが、本プロジェクトは
  「直前ターンに `protect` フラグ技が成功していれば必ず失敗する」簡略化モデルを採用。
  `docs/spec/moves/こらえる.md` で既に採用されている方針を踏襲）
- まねっこでコピーできない（`non_copycat` フラグ）
- 技本体の詳細防御仕様は既存の `docs/spec/volatiles/まもる.md` に委譲

また、既存の `docs/spec/volatiles/まもる.md` と `docs/plan/volatiles/まもる.md` に
「連続使用失敗判定は未実装・スコープ外」という記述が残っていた（実際は
`まもる系_連続使用失敗チェック` として実装済み）ため、実装済みである旨と簡略化モデルである旨に
修正した。あわせて `docs/spec/volatiles/まもる.md` の PP 表記（10）を champions 基準の8に修正した。

## 実装計画書

新規作成: `docs/plan/moves/まもる.md`

技側（`まもる` MoveData・`まもる_apply`・`まもる系_連続使用失敗チェック`）の設計根拠・priority根拠を
記録。turn.md では `Event.ON_TRY_MOVE_1` priority=30 に「まもる系: 連続使用により失敗」と
記載されているが、既存実装（まもる・みきり・キングシールド・こらえる等すべて）が
`Event.ON_TRY_MOVE_2` のデフォルト priority で統一されているため、その既存パターンを踏襲する
判断を明記した。また「スコープ外（既知の限界）」として、確率モデル未実装であることに加え、
まひ・ねむり等で行動できなかったターンを挟んでも `executed_move` がリセットされず、まれに
誤って連続使用扱いになりうる点を新たに記録した（まもる系共通の限界であり、本レビューでは
修正しない）。

`docs/plan/volatiles/まもる.md` の「スコープ外（未実装・保留）」節も、実際には実装済みである
ことを反映して「連続使用による失敗率上昇（実装済み）」に更新した。
