# シンプルビーム レビュー結果

## 仕様書
- 未作成だったため新規作成した（`docs/spec/moves/シンプルビーム.md`）。
  ポケモンWiki（`docs/wiki/moves/シンプルビーム.html`）を一次情報とし、
  `docs/champions/move_list.txt` の PP=16 を反映した。
- 失敗条件として、既存実装（PR前）が未対応だった以下2点をWikiの「技の仕様」節から追記した。
  - 対象の特性が「なまけ」の場合は失敗する。
  - 対象の特性がすでに「たんじゅん」の場合は失敗する。
- まもる／マジックコート／みがわりの判定はWiki通り（まもる○＝防がれる、
  マジックコート○＝反射される、みがわりに防がれる）であることを確認した。

## 実装計画書
- ability 名を誤って「シンプル」としていた記述を、正しい特性名「たんじゅん」に修正した
  （シンプルビームは相手の特性を「たんじゅん」に変える技で、「シンプル」という特性は存在しない）。
- 「シンプル特性が未実装のため先に実装が必要」という前提が誤りだったため削除した。
  `data/ability.py` の「たんじゅん」エントリと `handlers/ability.py` の
  `たんじゅん_double_stat` はすでに実装済みだった（別タスクで対応済み）。
- なまけ・すでにたんじゅんの2つの失敗条件をハンドラ構成に追記した。
- PP を 15 → 16（`champions/move_list.txt` 準拠）に修正した。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_sa.py` の「シンプルビーム」エントリの `pp` を
  `15` → `16` に修正した（`champions/move_list.txt` の値が正。
  ポケモンWikiの表記PP15は本家ゲームの基準でありプロジェクトの基準ではない）。
- `src/jpoke/handlers/move_status.py` の `シンプルビーム_can_apply` に、
  以下2つの失敗条件チェックを追加した（バグ修正）。
  - 対象の特性が「なまけ」の場合（`なまけ` には `protected` フラグが付与されていないため、
    `なかまづくり_can_apply` と同様に `base_name` での個別比較が必要）。
  - 対象の特性がすでに「たんじゅん」の場合。
    従来は `battle.change_ability` 内部の `if mon.ability.base_name == ability: return`
    という無言のno-opガードに処理を委ねており、`MOVE_FAILED` ログが出ないまま
    見かけ上「成功」扱いになっていた。ポケモンWikiの「技の仕様」節に明記された
    正式な失敗条件のため、`なりきり_can_apply` と同様に明示的な失敗判定に修正した。
- `シンプルビーム_change_ability` はロジック・優先度（デフォルト100）とも仕様書と一致しており変更なし。

## リーサル計算
- 対象外（`docs/progress/move.md` の「リーサル実装」「リーサルテスト」列は n/a。
  シンプルビームはダメージを与えない変化技のため）。
