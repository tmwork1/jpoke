# くろいまなざし レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/くろいまなざし.md`）。
- 一次情報（`docs/wiki/moves/くろいまなざし.html`）と `src/jpoke/data/ps-champ-ja/moves.json` を突き合わせ、
  以下を明記した:
  - PP は本家5だが Champions では8（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）。
  - 命中は「100%」ではなく「必中」（命中判定自体を行わない）。
  - 第六世代以降は「まもる」で防がれない（`move_list.txt` の「守×」に対応）。
  - みがわりには防がれる（`bypass_substitute` フラグなしの一般ルールに従う）。
  - マジックコートで反射可能・オウムがえしでコピー可能・ゴーストタイプには無効（にげられない状態側の仕様）。

## 実装計画書
- 既存の計画書の誤りを修正した:
  - PP5→8、命中率100→必中（`accuracy=None`）に修正。
  - `unprotectable` フラグの追加を明記（デフォルトでは target=foe の変化技はまもるで防がれるため、
    フラグなしでは「守×」の仕様を満たせないバグを発見・修正）。
  - ゴーストタイプ無効化・重複付与失敗の実処理経路を、実際のコードに基づいて訂正
    （`ON_TRY_MOVE_2`/`ON_APPLY_MOVE` の該当ハンドラは未実装であり、実際は `にげられない` 揮発性状態側の
    `Event.ON_CHECK_TRAPPED`（`check_trapped_not_ghost`）と `VolatileManager.apply()` の重複チェックが
    その役割を担っていることを明記）。

## 実装（handlers/ / data/）
