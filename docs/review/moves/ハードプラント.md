## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ha.py` の `ハードプラント` エントリの `pp` を `5` → `8` に修正した。
  `docs/champions/move_list.txt` では `ハードプラント 150 90 8` であり、Gen9本家PP(5)のまま
  移行漏れになっていたバグ。
- ハンドラ実装自体（`Event.ON_DAMAGE_HIT: ha.リチャージ_apply`）は仕様通りで変更なし。
  共用の「リチャージ」揮発状態（`ON_TRY_ACTION`, priority=10, `attacker:self`）による
  反動での行動不能化ロジックも他のリチャージ系技と同一実装で問題なし。
