  自分の『とくこう』1段階上昇）は `docs/spec/moves/_move_secondary_1.md` の一覧表（69行目、
  `しんぴのちから attacker 1 ... C+1 ...`）に既に記載されており、一次情報（
  `docs/wiki/moves/しんぴのちから.html`）と照合した結果、内容は正しい（エスパー / 特殊 / 威力70 /
  命中90 / PP10 / 対象1体選択 / 優先度0 / 直接攻撃なし / 追加効果100%で自分のとくこう+1 /

  マジックコート×）。変更なし。

## 実装計画書
- 個別の計画書（`docs/plan/moves/しんぴのちから.md`）は存在しないが、同種の「攻撃後に自分の能力
  ランクを上げる」技をまとめた共通計画書 `docs/plan/moves/攻撃技_攻撃側ランク変化系.md` に
  しんぴのちからの仕様・ハンドラ構成が既に記載されている（ON_HIT / priority=100省略 /
  subject_spec=attacker:self / C+1・100%）。同計画書ではturn.mdのpriority=10表記はイベント内の
  処理グループを示すものであり、既存実装（アームハンマー等）に倣ってMoveHandlerのデフォルト
  priority（100）で登録する方針が明記されており、実装と齟齬はない。変更なし。

## 実装（handlers/ / data/）
- `src/jpoke/data/move.py` の `しんぴのちから` エントリ（type=エスパー, category=special, pp=10,
  power=70, accuracy=90, flags={"secondary_effect"}, `Event.ON_HIT` に
  `しんぴのちから_boost_spa_C` を登録）は仕様と一致する。
- `src/jpoke/handlers/move_attack.py` の `しんぴのちから_boost_spa_C` は
  `modify_attacker_stats(battle, ctx, value, stats={"spa": 1})` を呼び出しており、正規API
  （`battle.modify_stats` 経由）で100%の確率で自分のとくこうを1段階上げる仕様と一致する。
  `subject_spec`（デフォルト `attacker:self`）も `AttackContext` のロールと整合している。
  五十音順の位置（フレアソング等と同グループ）にも問題なし。変更なし。
