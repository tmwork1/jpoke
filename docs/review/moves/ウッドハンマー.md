- 新規作成（`docs/spec/moves/ウッドハンマー.md`）。一次情報（`docs/wiki/moves/ウッドハンマー.html`）と
  `docs/spec/moves/_move_secondary_2.md`（反動比率一覧）を参照し、基本情報（くさ/物理/威力120/命中100/
  PP16/対象1体/優先度0/接触あり）、反動ダメージ計算（`max(1, round_half_down(与えたダメージ*1/3))`）、
  判定フラグ、特性との関係（いしあたま・マジックガード・すてみ・がんじょう）を記載した。
  PPは一次情報の15ではなく `src/jpoke/data/ps-champ-ja/moves.json` の16を正としている。

## 実装計画書

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_a.py` の `pp` を `15` → `16` に修正した。
  `src/jpoke/data/ps-champ-ja/moves.json` では `ウッドハンマー くさ 物理 120 100 16` と定義されており、
  `docs/champions/changes_from_sv.md` の「全技PPを8/12/16/20に統一」の対象技であるにもかかわらず、
  実装は本家Gen9基準の15のままだった（championsへの移行漏れ）。`docs/progress/move.md` の
  PP列は既に16で正しかったため、そちらに合わせて実装値を修正した。
- `handlers/move_attack.py` の `ウッドハンマー_recoil`（共通ヘルパー `_recoil` を反動比率1/3で呼び出す）は
  仕様と一致しており、修正不要と判断した。

## テスト
- 変更なし。既存の `tests/moves_attack/test_move_a.py::test_ウッドハンマー_使用後に攻撃者が反動ダメージを受ける`
  と `tests/moves_attack/test_move__group.py::test_recoil技グループ_反動量が与ダメのrecoil_ratio倍になる`
  （`ウッドハンマー` を含む反動比率1/3グループのパラメータ化テスト）で反動ダメージの挙動が確認されており、
  PP修正の影響を受ける既存テストはなかった。

## 補足
- `python -m pytest tests/ -v` 実行時に本レビューと無関係な2件のテスト
  （`tests/items/test_item_ta.py::test_だっしゅつパック_わるいてぐせに奪われた場合は発動しない`、
  `tests/items/test_item_ta.py::test_だっしゅつボタン_ちからずくの効果が発動した技を受けたときは発動しない`）
  が間欠的に失敗することを確認した。ウッドハンマーの変更を`git stash`で完全に取り除いた状態でも
  同じ間欠的な失敗（他のテストが失敗する場合もあり）が再現したため、乱数シードが固定されていない
  既存のテスト基盤側の問題であり、本レビューの変更とは無関係と判断した（再実行で全件パスすることを確認済み）。
