# ハードプレス レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/ハードプレス.md`）。
  - 一次情報（`docs/wiki/moves/ハードプレス.html`）に基づき、基本情報（はがね/物理/命中100/接触あり/優先度0）、
    威力計算式（`max(1, floor(100 × 対象の残りHP ÷ 対象の最大HP))`）、判定フラグ
    （急所○/命中判定○/追加効果×/まもる○/おうじゃのしるし○(第五世代以降)/マジックコート×）を記載。
  - PPは Gen9本家10 → Champions 12（`docs/champions/move_list.txt` の値）と明記。
  - `docs/spec/moves/_variable_power.md` 第5節「対象HP依存威力技」には既にハードプレスの
    威力計算式が記載済みであることを確認（変更不要）。

## 実装計画書
- 新規作成（`docs/plan/moves/ハードプレス.md`）。
  - 既存の `docs/plan/moves/攻撃技_威力変動系.md`（複数技バンドル計画書）に同等の内容が
    含まれていることを確認したが、`にぎりつぶす.md` 等の他の変動威力技と同様に単体の計画書
    としても新規作成し、priority根拠・実装コード・テスト方針・PP修正の経緯を明記した。

## 実装（handlers/ / data/）
- `handlers/move_attack.py` の `ハードプレス_calc_power` は仕様通り
  （`power = max(1, int(100 * ctx.defender.hp / ctx.defender.max_hp))`、`ON_CALC_POWER_MODIFIER`
  で `power * 4096` を返す方式）であり、変更なし。
- `data/moves/move_ha.py` の `pp` を **10 → 12** に修正した。
  `docs/champions/move_list.txt`（664行目: `ハードプレス はがね 物理 - 100 12`）が正であり、
  実装側がGen9本家PP値のまま移行漏れになっていたバグ。`docs/progress/move.md` のPP列は
  既に12で正しかったため、そちらは変更していない。

## リーサル計算
