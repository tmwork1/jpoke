## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ra.py` の `"ロックカット"` エントリに `target="self"` を追加した。
  - `target` を省略すると `MoveData` のデフォルト値 `"foe"` が使われ、`Move.is_blocked_by_protect`
    （`target == "foe"`）が True になってしまい、相手が `まもる` を使用しているとロックカットの
    すばやさ上昇が誤ってブロックされることを実測で確認した（修正前: `まもる` 使用中の相手に対して
    ランクが 0 のまま／修正後: 正しく +2 される）。
  - `is_reflectable`（`category == "status" and target in ("foe", "foe_side")`）も False になり、
    マジックコートで誤って跳ね返されることも防止される。
  - `handlers/move_status.py` の `ロックカット_modify_attacker_stats`（`modify_attacker_stats` を
    `stats={"spe": 2}` で呼び出す実装）自体は仕様と一致しており変更なし。
