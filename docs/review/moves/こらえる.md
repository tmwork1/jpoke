## 実装（handlers/ / data/）
- **バグ修正1（PP）**: `src/jpoke/data/moves/move_ka.py` の `こらえる` エントリで `pp=10` を
  `pp=12` に修正した（`docs/champions/move_list.txt` 286行目が正）。
- **バグ修正2（連続使用失敗チェックの欠落）**: `こらえる` エントリに `flags={"protect"}` と
  `Event.ON_TRY_MOVE_2: h.MoveHandler(hs.まもる系_連続使用失敗チェック)` を追加した。
  - 修正前は、まもる・みきり・キングシールド・ニードルガード・トーチカ・スレッドトラップ・
    かえんのまもり・ファストガードが共有する連続使用失敗ロジックが `こらえる` にだけ登録されておらず、
    何度連続で使用しても必ず成功してしまう欠落があった（`docs/spec/volatiles/まもる.md` は
    こらえるをこの連続使用失敗共有グループの一員として明記している）。
  - `まもる系_連続使用失敗チェック` は `mon.executed_move.has_flag("protect")` を見て判定するため、
    `こらえる` にも `flags={"protect"}` を付与する必要があった。
- **priorityの誤修正と復元**: レビュー当初、`docs/spec/turn.md` の記載通り
  `src/jpoke/data/volatile.py` の `こらえる` の `Event.ON_MODIFY_MOVE_DAMAGE` priority を
  95 → 60 に変更したところ、`tests/items/test_item_na.py::test_ナゾのみ_一撃必殺技を耐えたときは発動する`
  等が失敗することを確認した。原因は、一撃必殺技の確定ダメージ算出 `ohko_damage`
  （`ha.ohko_damage`, priority=90）が turn.md の表に掲載されておらず、priority=60では
  `こらえる_endure` が `ohko_damage` より先に実行されてしまい、後から確定ダメージで
  上書きされて一撃必殺を耐えられなくなるためと判明した。95に復元し、テストが全て通ることを確認した。
  この既知の齟齬は仕様書・計画書に明記した（turn.md 自体の修正は本レビューのスコープ外とした）。
- `src/jpoke/handlers/volatile.py` の `こらえる_endure`（HP1補正・hits_taken補正）と
  `こらえる_remove_volatile`、`src/jpoke/handlers/move_status.py` の `こらえる_apply` は
  仕様と一致しており変更なし。
