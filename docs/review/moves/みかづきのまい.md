## 実装（handlers/ / data/）
- **バグ修正**: 控えのポケモンがいない状態でみかづきのまいを使用すると、原作では技が失敗し
  使用者はひんしにならないはずが、既存実装では無条件にフィールド設置＋ひんし化していたため、
  控えがいない場合に実質的な負け確定行動になってしまう欠陥があった。
  - `handlers/move_status.py` に `みかづきのまい_can_apply` を新規実装
    （`battle.query.has_available_bench` でとらわれ状態を無視して控えの有無のみ判定し、
    控えがいなければ `MOVE_FAILED` ログを出して `HandlerReturn(value=False, stop_event=True)`）
  - `data/moves/move_ma.py` の `"みかづきのまい"` エントリに
    `Event.ON_BEFORE_APPLY_MOVE: h.MoveHandler(hs.みかづきのまい_can_apply)` を追加
    （`ねがいごと_can_apply` と同じパターン）
- `handlers/field.py` の `みかづきのまい_heal`、`data/field/side_field.py` の
  `"みかづきのまい"` エントリ（ON_SWITCH_IN priority=100）は仕様書と照合した結果、
  HP・PP・状態異常の回復条件、保留（発動しない場合はフィールドを残す）ロジック、
  かいふくふうじとの相互作用（HPのみブロックされ状態異常・PPは回復する）すべて正しく
  実装されていたため変更なし
