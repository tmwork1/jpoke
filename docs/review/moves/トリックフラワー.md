## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ta.py` の `PP` が `10`（Gen9本家基準）になっていたバグを修正し、
  `docs/champions/move_list.txt` に記載された `12` に修正した
  （`docs/champions/changes_from_sv.md` により全技PPは8/12/16/20の4段階に統一されているため、
  10という値自体が移行漏れであると判断）。
- `type="くさ"`, `category="physical"`, `power=70`, `critical_rank=3`, `handlers={}` は
  仕様通りで変更なし。`accuracy` 未指定（`None`＝必中）、`flags` 未指定（非接触）、
  `target` 省略時デフォルト`"foe"`（相手1体）もすべて仕様と一致。

## リーサル計算
- 対象外（`docs/progress/move.md` のリーサル実装・リーサルテスト列は `n/a`）。
  急所ランク3による確定急所は `calc_lethal` の `critical` 引数を用いる既存の共通枠組みで
  表現されるため、本技固有のリーサルハンドラは不要（`こおりのいぶき`・`やまあらし` と同様）。

## テスト

  そのまま維持。

  - `test_トリックフラワー_威力命中率PPが仕様通り`: タイプ・分類・威力・命中（None＝必中）・
    PP=12（champions基準）を検証。PP修正バグを検知できるよう明示的にアサート。
  - `test_トリックフラワー_相手の回避率が高くても必ず命中する`: 相手の回避ランクを+6にしても
    必ず命中することを確認（`どげざつき`の同種テストを踏襲）。
- `python -m pytest tests/ -v` で全3901件（1件skip）がパスすることを確認した
  （`.loop/test_logs/トリックフラワー.log`）。
  なお `tests/moves_attack/test_move_ka.py::test_くらいつく_...` と
