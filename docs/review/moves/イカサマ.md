## 実装（handlers/ / data/）
- **PP値の修正**: `src/jpoke/data/moves/move_a.py` の `"イカサマ"` エントリで `pp=15`
  （Gen9本家基準）になっていたのを `pp=16`（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）に修正した。
  `docs/progress/move.md` のPP列はすでに16で記載されており、実装側の移行漏れだった。
- **パラドックス特性（クォークチャージ/こだいかっせい）との相互作用バグを修正**:
  `src/jpoke/handlers/ability_paradox.py` の `apply_atk_modifier` が、イカサマ使用時は
  「対象（相手）自身のパラドックス補正状態」を見て1.3倍を適用していた
  （`boost_mon = defender; stat = "atk"` という技名分岐）。
  しかし `docs/spec/abilities/クォークチャージ.md` / `docs/spec/abilities/こだいかっせい.md` に
  「攻撃が上昇しても、イカサマを受けたときのダメージは変わらない」と明記されており、対象自身の
  補正はイカサマのダメージに影響しないのが正しい仕様。また同スペックの「攻撃が上昇したとき
  使用するボディプレスは防御の数値を1.3倍にしてダメージを与える。防御が上がってもダメージは
  変わらない」という記述から、パラドックス特性の補正は「ダメージ計算でどの実数値が使われて
  いるか」とは無関係に、常に「使用者自身の攻撃側スロット（物理技なら攻撃、特殊技なら特攻）が
  強化対象かどうか」だけで判定されるべきと判断した。技名による分岐（イカサマ/ボディプレス）を
  撤廃し、常に `ctx.attacker.paradox_boost_stat` を技の分類（物理/特殊）で判定するように修正した。
  この修正はボディプレス自体の挙動にも影響するが、ボディプレスの個別レビュー
  （`docs/progress/move.md` のボディプレス行）は別タスクの対象のため、本タスクではイカサマとの
  相互作用としてのみ確認・修正した。
- その他（`damage.py` の攻撃力入れ替え、てんねん/ちからもち/こだわりハチマキ/わざわいのおふだ
  との相互作用）はコンテキストロール（attacker/defender）ベースの既存アーキテクチャにより
  自動的に正しく動作していることを確認した。追加の修正は不要。
