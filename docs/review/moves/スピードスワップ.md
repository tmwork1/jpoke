# スピードスワップ レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/スピードスワップ.md`）。一次情報（`docs/wiki/moves/スピードスワップ.html`）を基に、基本情報表・効果・判定（まもる／マジックコート／みがわり／必中）・世代差異（メガシンカ・フォルムチェンジ・へんしん・ダイマックス/テラスタル）を記載した。

## 実装計画書
- 既存の `docs/plan/moves/スピードスワップ.md` を一次情報・修正後の実装と照合し、以下を修正した。
  - PP を 10 → 12 に修正（`src/jpoke/data/ps-champ-ja/moves.json` 準拠。Gen9本家基準の10は移行漏れのバグ。ガードシェア・ガードスワップと同種のバグ）。
  - 命中率を「命中判定は発生しない（`accuracy=100`のまま）」という誤った記述から、`accuracy=None` へ修正する旨に訂正した。従来の計画書は「`accuracy=100`は他の変化技に倣った設定値であり命中判定は発生しない」と誤認していたが、`move_executor._check_hit` の実装では `accuracy=100` の場合 `ON_MODIFY_ACCURACY` → 回避率ランク補正の計算経路を通るため、実際には回避率上昇の影響を受けてしまうバグだった。
  - マジックコート「×」（跳ね返されない）・みがわり「○」（貫通する）が一次情報に明記されているにもかかわらず `flags` が未設定だったため、`flags={"unreflectable", "bypass_substitute"}` の必要性を明記した。
  - ハンドラのコンテキスト型がプレースホルダ的に不明瞭だった箇所を、実装済みコード（`ctx: AttackContext`、`attacker`/`defender` ロール）に合わせて明記した。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/move_status.py` の `スピードスワップ_swap_speed` はガードシェア・パワートリック等と同一パターン（`_stats_manager.stats[5]` の直接入れ替え）で実装済みであり、ロジック自体に変更なし。
- `src/jpoke/data/moves/move_sa.py` の `"スピードスワップ"` エントリを以下の通り修正した（レビューで発見したバグ）。
  - `pp=10` → `pp=12`（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）。
  - `accuracy=100` → `accuracy=None`（必中技。回避率ランク補正の影響を受けないようにするため）。
  - `flags={"unreflectable", "bypass_substitute"}` を追加（マジックコートで跳ね返されず、みがわりを貫通するため）。
- 交代時リセットは `Pokemon.reset_on_switch_out()` が `update_stats()` を呼んで種族値ベースの実数値に再計算する既存修正（ガードシェア・パワーシェア・パワートリック等と共通）でカバーされることを確認した。専用の実装は不要で、重複実装も行っていない。

## テスト
- `tests/moves_status/test_move_sa.py` に以下のテストを追加した。
  - `test_スピードスワップ_PPと必中フラグが正しい`: `MOVES["スピードスワップ"]` の `pp`・`accuracy`・`flags` が仕様通りであることを確認。
  - `test_スピードスワップ_まもるで防がれる`: まもる状態の相手には防がれ、すばやさが入れ替わらないことを確認。
  - `test_スピードスワップ_マジックコートで跳ね返されない`: `unreflectable` フラグにより、マジックコート状態の相手に使用しても跳ね返されず、使用者側のすばやさが変化することを確認。
  - `test_スピードスワップ_みがわり状態の相手にも効果が発動する`: `bypass_substitute` フラグにより、みがわり状態の相手にもすばやさ入れ替えが成立することを確認。
  - `test_スピードスワップ_相手の回避率が高くても必ず命中する`: `accuracy=None` により、相手の回避率ランクが+6でも必ず命中し効果が発動することを確認（修正前の `accuracy=100` バグでは失敗し得たケース）。
