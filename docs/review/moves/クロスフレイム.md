# クロスフレイム レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/クロスフレイム.md`）。
- 一次情報（ポケモンWiki）に基づき、基本情報（ほのお/特殊/威力100/命中100/PP8/対象1体/
  優先度0/非接触）と効果（自分のこおり状態を無条件解凍してから攻撃、追加効果なし）を記載。
- クロスサンダーとの「デュアルクロス」（直前に成功した技がクロスサンダーの場合威力2倍）は、
  実装対象とした。効果に「味方」「自分以外」の制限はなく、シングルバトルでも相手が同ターン内に
  先にクロスサンダーを命中させていれば成立するため。
- PPは `src/jpoke/data/ps-champ-ja/moves.json` に記載がないが、同時にレビューされたクロスサンダー・
  ガリョウテンセイ等、第9世代PP5の技がchampionsで一様にPP8へ変換される前例に従い8とした。

（統合ブランチへのマージ時の補足: 本エントリは並列レビュー中のクロスサンダー側と同時期に
レビューされ、クロスサンダー側は「デュアルクロスは実装対象・PP8」、クロスフレイム側は
当初「デュアルクロスは対象外・PP5維持」という相反する結論だった。一次情報を再確認した結果
クロスサンダー側の結論が正しく、マージ時に仕様書・実装計画書・進捗表・実装を
クロスサンダー側の結論に合わせて修正した。）

## 実装計画書
- 新規作成（`docs/plan/moves/クロスフレイム.md`）。
- `かえんぐるま_thaw_attacker` / `もえつきる_thaw_attacker` と同一パターンでの
  `クロスフレイム_thaw_attacker` ハンドラ追加（`Event.ON_TRY_ACTION`, priority=5）を計画。
- priorityの根拠を `docs/spec/moves/かえんぐるま.md` の先行レビュー内容を引用して記載。
- デュアルクロスはクロスサンダー側と対になる `クロスフレイム_calc_power` /
  `クロスフレイム_record_hit` を追加する方針に修正。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/move_attack.py` に `クロスフレイム_thaw_attacker` を新規追加
  （使用者がこおり状態のとき無条件解凍する。`かえんぐるま_thaw_attacker` と同一パターン）。
- 同ファイルに `クロスフレイム_calc_power` / `クロスフレイム_record_hit` を新規追加
  （クロスサンダー側と対になるデュアルクロス実装。`battle.fusion_bolt_used_turn` /
  `fusion_flare_used_turn` を参照）。
- `src/jpoke/data/moves/move_ka.py` の `"クロスフレイム"` エントリに
  `Event.ON_TRY_ACTION`（thaw）、`Event.ON_CALC_POWER_MODIFIER`、`Event.ON_HIT`
  （デュアルクロス）を登録。`pp` を 5 から 8 に修正。
