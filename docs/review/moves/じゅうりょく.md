# じゅうりょく レビュー結果

## 仕様書
- `docs/spec/fields/じゅうりょく.md`（フィールド仕様）は一次情報と照合し、以下を修正した:
  - `ハンドラ優先度` テーブルの `ON_TRY_MOVE_1` を `priority=100（デフォルト）` から
    実装値である `priority=30`（`docs/spec/turn.md` の技個別失敗条件カテゴリ）に修正
  - `ON_CALC_POWER_MODIFIER`（Gのちから）・`ON_TRY_MOVE_1`（使用禁止技ブロック）は
    フィールド側の汎用ハンドラではなく各技側の個別ハンドラとして実装されている旨を明記
- `docs/spec/moves/じゅうりょく.md`（技仕様）を新規作成した。PP・対象・まもる/マジックコート
  判定など技データ側の情報を記載し、フィールド効果本体は `docs/spec/fields/じゅうりょく.md`
  を参照する形にした。

## 実装計画書
- `docs/plan/fields/じゅうりょく.md` を実装と一致するよう修正した:
  - `ON_FIELD_ACTIVATE` の実装関数名を `じゅうりょく_activate_release_volatiles` から
    実際の関数名 `じゅうりょく_remove_volatiles` に修正
  - Gのちから威力補正・使用禁止技ブロックのセクションを、実際の実装（技側の個別ハンドラ:
    `Gのちから_gravity_boost` / `gravity_restricted_fail`）に合わせて全面修正
  - `ON_MODIFY_ACCURACY` に一撃必殺技除外の記述を追加
  - `gravity_restricted` フラグ対象技の一覧のファイルパスを実ファイル
    （`data/moves/move_ha.py` 等）に修正
  - データ登録コード例を実装（4ハンドラのみ）に合わせて修正
- `docs/plan/moves/じゅうりょく.md` を新規作成した。技データ（PP=8、target="field"）と
  `ON_STATUS_HIT` ハンドラの実装方針を記載した。

## 実装（handlers/ / data/）
- **PP値のバグを修正**: `src/jpoke/data/moves/move_sa.py` の `"じゅうりょく"` エントリが
  `pp=5`（Gen9本家基準）になっていたのを `pp=8`（`docs/champions/move_list.txt` 準拠）に修正。
- **一撃必殺技の命中率補正バグを修正**: `src/jpoke/handlers/field.py` の
  `じゅうりょく_modify_accuracy` が一撃必殺技（`ohko` フラグ）にも命中率補正を適用してしまって
  いたため、`ctx.move.has_flag("ohko")` の場合は補正しないよう修正した
  （一次情報「一撃必殺技の命中は上がらない」に対応）。
- その他の効果（接地化・使用禁止技ブロック・Gのちから威力補正・そらをとぶ/でんじふゆう解除・
  カウントダウン）は仕様書と一致しており、修正なし。
