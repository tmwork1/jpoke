  もとに、タイプ・分類・威力・命中・PP・急所ランク+1・やけど追加効果（10%）・こおり解凍効果を記載した。
  PPは一次情報の10ではなく `src/jpoke/data/ps-champ-ja/moves.json`（Champions基準、12）を正とした。

## 実装計画書
- 新規作成（`docs/plan/moves/ブレイズキック.md`）。仕様書をもとに、下記「実装」節の修正内容の根拠を
  記載した。あわせて既存の共有計画書 `docs/plan/moves/攻撃技_やけど系.md` 内のブレイズキック例示コード
  （`pp=10`, `category="物理"`, `Event.ON_HIT` 等の旧記法）を現状の実装（`pp=12`, `category="physical"`,
  `Event.ON_DAMAGE_HIT`, `thaw`フラグ追加）に合わせて更新した。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ha.py` の `"ブレイズキック"` エントリを修正:
  - `pp=10` → `pp=12`（Gen9本家基準の値が移行漏れになっていたバグ。
    `src/jpoke/data/ps-champ-ja/moves.json` 788行目に合わせて修正）
  - `flags` に `"thaw"` を追加（ほのおタイプの攻撃技として、被弾した相手のこおりを解凍する効果が
    欠落していた。`data/ailment.py` の `こおり_cure_by_thaw_move` が `ctx.move.has_flag("thaw")` を
    参照するため、フラグ未設定では発動しない）
  - `docs/spec/moves/_move_labels.md` の `thaw` セクションに `ブレイズキック` を五十音順で追加
    （フレアドライブの次）
  - `critical_rank=1`（急所ランク+1）・やけど付与ハンドラ（`ブレイズキック_apply_burn_to_defender`,
    `handlers/move_attack.py`）は既存実装が仕様と一致しており修正不要。急所率+1段階は
    `docs/spec/critical.md` 89行目のブレイズキックの記載と一致することを確認した。
  - 使用者自身がこおり状態のとき無条件解凍する効果（`_thaw_attacker` 系、かえんぐるま等が持つ）は
    一次情報に記載がないため対象外とした。
