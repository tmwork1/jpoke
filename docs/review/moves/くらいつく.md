## 実装（handlers/ / data/）
- `src/jpoke/data/move.py`: `"くらいつく"` エントリに `Event.ON_DAMAGE_HIT` ハンドラ
  （`くらいつく_apply_no_escape`）を追加。従来は `handlers={}`（コメント「追加効果なし」）で、
  『にげられない』状態の付与が完全に未実装だった。
- `src/jpoke/handlers/move_attack.py`: `くらいつく_apply_no_escape` を新規実装。
  - 片方でもゴーストタイプ、または片方でもすでに『にげられない』状態の場合は双方とも付与しない。
  - `apply_volatile_to_defender`/`apply_volatile_to_attacker`（`resolve_secondary_chance` 経由）を
    使わず `battle.volatile_manager.apply` を直接呼び出すことで、追加効果とみなされない
    （ちからずく・りんぷんの影響を受けない）仕様を実現。
  - 相手をひんしにした場合は使用者に『にげられない』状態を付与しない。
  - `Event.ON_DAMAGE_HIT` を使用することで、まもる・みがわり・技無効化・命中失敗時に
    イベント自体が発火せず、追加のガードなしで「外れた/防がれた場合は付与されない」仕様を満たす。
- `src/jpoke/handlers/volatile.py` / `src/jpoke/data/volatile.py`: 『にげられない』揮発性状態に
  `Event.ON_SWITCH_OUT` ハンドラ（`にげられない_remove_on_foe_switch`、
  `subject_spec="source:foe"`）を追加。付与元（相手）が場を離れる（交代・ひんし後の交代のいずれも
  `_switch_out` 経由で発火）と、保持側の『にげられない』状態を解除する。
  - この修正は くらいつく に限らず、既存の『にげられない』付与技（くろいまなざし・とおせんぼう・
    かげぬい等）全般に共通して必要な一般的な修正であり、`docs/spec/volatiles/にげられない.md`
    の消滅条件（付与した側が場からいなくなると解除される）を満たすために追加した。
- `python scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` /
  `python scripts/sort_handlers.py src/jpoke/handlers/volatile.py` /
  `python scripts/sort_data/sort_moves.py` を実行済み。
