## 実装計画書

  - `stats={"B": 2}` という誤った Stat キー例を `stats={"def": 2}`（実装と一致）に修正
  - 存在しない `src/jpoke/data/move.py` への参照を実際の登録先 `src/jpoke/data/moves/move_ta.py` に修正し、
    `category="変化"` の誤記を実装値 `category="status"` に修正
  - データ例に欠落していた `target="self"` を追加（下記実装バグの修正に合わせる）
  - 注意点セクションから技と無関係な「ちいさくなる」に関する記述を削除し、まもる無効の根拠を追記

## 実装（handlers/ / data/）
- `handlers/move_status.py` の `とける_modify_attacker_stats` は `てっぺき_modify_attacker_stats` と同一パターンで
  問題なし（`modify_attacker_stats(battle, ctx, value, stats={"def": 2})`）。
- `data/moves/move_ta.py` の `"とける"` エントリに **`target="self"` が欠落していたバグを修正**。
  `MoveData.target` のデフォルトは `"foe"` のため、修正前は
  - `Move.is_blocked_by_protect` が `True` となり、相手の `まもる` で誤って防がれる
  - `Move.is_reflectable` が `True` となり、マジックコート/マジックミラーで誤って反射される
  という不具合があった（同種の `てっぺき` エントリには `target="self"` が設定済みで問題なし）。
  `target="self"` を追加してこれらを修正した。
