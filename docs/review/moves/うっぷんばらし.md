# うっぷんばらし レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/うっぷんばらし.md`）。一次情報（`docs/wiki/moves/うっぷんばらし.html`）を
  基に、基本情報（あく/物理/威力75/命中100/PP5/接触あり）と「そのターン中に自分の能力ランクが
  下がっていたら威力2倍」の効果、および一次情報に記載された細かい判定基準
  （対象以外からのランクダウンも対象、自分の特性によるランクダウンも対象、まけんき等で
  下がった直後に上がっても対象、いかく等の場に出た時のランクダウンも対象、死に出し直後の
  ランクダウンは次に行動できるターンには対象外、パワースワップ系・くろいきり・
  きみょうなくすり／きょうえんは対象外）をまとめた。

## 実装計画書
- 新規作成（`docs/plan/moves/うっぷんばらし.md`）。`Pokemon.stat_lowered_this_turn` フラグの
  役割、`core/status_manager.py` の `modify_stats()` でのフラグ設定箇所、ハンドラ構成
  （`ON_CALC_POWER_MODIFIER` / デフォルトpriority、根拠は同種の `たたりめ`・`からげんき`・
  `ダメおし` に合わせた）を記載した。
  また、死に出し直後のランクダウンが次ターンには対象外になる点について、
  `core/turn_controller.py` の `_begin_turn()`（ターン開始時の `reset_turn_state()`）と
  `_run_switch_phase()`（通常交代は `_begin_turn()` の後に実行される）の順序だけで
  仕様どおりの挙動が自然に成立することを確認し、専用の分岐実装が不要である根拠を明記した。

## 実装（handlers/ / data/）
- 変更なし。既存の `src/jpoke/model/pokemon.py`（`stat_lowered_this_turn` フィールドと
  `reset_turn_state()` でのリセット）、`src/jpoke/core/status_manager.py`
  （`modify_stats()` でのフラグ設定）、`src/jpoke/handlers/move_attack.py`
  （`うっぷんばらし_double_power_when_rank_dropped`）、`src/jpoke/data/move.py`
  （`ON_CALC_POWER_MODIFIER` 登録）を一次情報・新規作成した仕様書・計画書と照合したが、
  誤り・欠落は見つからなかった。
  - パワースワップ／ガードスワップ／ハートスワップ／ひっくりかえす／くろいきりは
    `mon.rank[stat]` を直接書き換える実装になっており `modify_stats()` を経由しないため、
    既に自動的に対象外になっていることを確認した。
  - きみょうなくすり／きょうえん特性は現状 `AbilityData()`（未実装のプレースホルダ）であり
    ランクを下げる処理自体が存在しないため、除外のための追加実装は不要と判断した。
  - 実機相当の確認として、バトル開始時のいかく（相手が `いかく` 持ちで初手交代）→
    ターン1で `うっぷんばらし` を使用するケースを実装レベルで検証し、威力2倍
    （`power_modifier == 8192`）になることを確認した。

- **本レビューとは別に、テスト収集を妨げていた無関係な2件の不整合を修正した**
  （どちらも既存の別レビュー結果ドキュメントで「修正済み」と記載されていたが、
  `src/jpoke/data/move.py` 側の変更が反映されていなかったため、テスト収集時に
  `AttributeError` が発生していた）。
  - `いてつくしせん`: `docs/review/moves/いてつくしせん.md` の記載どおり、
    `Event.ON_DAMAGE_HIT` のハンドラ登録をリスト形式から
    `いてつくしせん_apply_freeze_to_defender` 単体指定に修正した
    （`いてつくしせん_lower_defender_spd` は同レビューで既に廃止されていたが
    参照だけが残っていた）。
  - `いのちがけ`: `docs/review/moves/いのちがけ.md` / `docs/plan/moves/いのちがけ.md` の
    記載どおり、`Event.ON_PAY_HP`（`いのちがけ_pay_hp`、既に廃止済み関数への参照）を削除し、
    `flags={"fixed_damage"}` を追加した。
  - いずれも `python scripts/sort_data/sort_moves.py` を実行済み。

## リーサル計算

## テスト
- 既存の2件（`test_うっぷんばらし_ランクが下がったとき威力2倍`・
  `test_うっぷんばらし_ランクが下がっていないとき通常威力`）は
  `attacker.stat_lowered_this_turn = True` という内部フラグの直接書き換えで前提条件を
  作っていたため、`battle.modify_stats(attacker, {"atk": -1}, source=attacker)` を使う形に
  修正し、`status_manager.modify_stats()` によるフラグ設定の実挙動まで検証するようにした。
- 追加: `test_うっぷんばらし_ランクが下がった後に上がっても威力2倍`
  （まけんき等でランクが下がった直後に別の能力が上がって差し引き下がっていない状態に
  なっていても威力2倍になる一次情報の記載を検証）。
- `python scripts/sort_tests.py tests/moves_attack/test_move_a.py` を実行した。
- `python scripts/generate_test_list.py` を実行し、テスト一覧を更新した。
- `python -m pytest tests/ -v` を実行し、`.loop/test_logs/うっぷんばらし.log` に保存した。
  結果は 3242 passed, 15 failed, 1 skipped。`うっぷんばらし` 自体のテスト（3件）はすべて成功。
  失敗している15件は `いかりのまえば`・`いじげんホール`・`いじげんラッシュ`・`いびき`・
  `うたかたのアリア`・`ちからずく`・`がんじょう`・`きあいのタスキ`・`マルチスケイル` など、
  本レビューの対象外の技・特性に関する既存の不具合（他の並行レビュー作業に起因すると
  見られる）であり、`うっぷんばらし` の実装・テストとは無関係であることを確認した。
  これらは本タスクのスコープ外のため修正していない。
