## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ta.py` の `"テクスチャー"` エントリを修正した。
  - **バグ修正**: `target` が未指定でデフォルト値 `"foe"` のままだったため、
    `Move.is_blocked_by_protect`（`target == "foe"`）・`Move.is_reflectable`
    （`category == "status" and target in ("foe", "foe_side")`）が誤って `True` を返し、
    自分対象の技であるにもかかわらずまもる・マジックコートに阻害されうる状態だった。
    `target="self"` を明示的に追加して修正した。
  - `handlers={}`（「追加効果なし」扱い）だったが、これは誤り。テクスチャーの核心効果である
    「自分のタイプを変更する」処理が未実装だったため、`ON_BEFORE_APPLY_MOVE` /
    `ON_STATUS_HIT` ハンドラを新規登録した。
- `src/jpoke/handlers/move_status.py` に `テクスチャー_can_apply` / `テクスチャー_apply` を新規実装した。
  - `テクスチャー_can_apply`: テラスタル中、または自分が覚えている技のうち一番上の技
    （`moves[0]`）のタイプが自分の現在のタイプ（`added_types` 含む）のいずれかと同じ場合に失敗させる。
  - `テクスチャー_apply`: `moves[0].data.type`（そうでん状態や特性による実行時タイプ変化の影響を
    受けない「本来のタイプ」）を用いて `move_override_types` を設定し、`added_types` をリセットする。
