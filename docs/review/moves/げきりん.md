# げきりん レビュー結果

## 仕様書

- `docs/spec/moves/げきりん.md` は存在しない。げきりん固有の効果（強制連続攻撃・
  使用後こんらん）は、あばれる・げきりん・はなびらのまい・だいふんげきで共用の
  揮発性状態として `docs/spec/volatiles/あばれる.md` に既に詳細な仕様書
  （付与条件・消滅条件・こんらん付与の特例・ターン処理優先度）が存在しており、
  これが技レベルの効果を包含している。`あばれる` 自体のレビュー
  （`docs/review/moves/あばれる.md`）で踏襲された方針（共用仕様書がある場合は
  技単体の個別ファイルを新規作成しない）に倣い、新規作成は行わなかった。
- 一次情報（`docs/wiki/moves/げきりん.html`）と実装を照合し、以下を確認した：
  タイプ＝ドラゴン、分類＝物理、威力＝120（第四世代以降）、命中率＝100、
  PP＝10（第五世代以降）、範囲＝相手ランダム1体（シングル対象のため実質相手
  1体固定）、直接攻撃＝○、追加効果＝×、急所／命中判定＝各回、まもる＝○、
  マジックコート／よこどり＝×、オウムがえし＝○。いずれも実装
  （`src/jpoke/data/move.py`）と一致していた。

## 実装計画書

- `docs/plan/moves/げきりん.md` は存在しない。同様の理由で、あばれる系共有の
  揮発性状態実装（`あばれる_apply` / `あばれる_tick` / `force_command`）が
  技の実装内容を実質的にカバーしているため、個別の新規作成は行わなかった。

## 実装（handlers/ / data/）

- `src/jpoke/data/move.py` の `"げきりん"`: `type="ドラゴン"`,
  `category="physical"`, `pp=10`, `power=120`, `accuracy=100`,
  `flags={"contact"}`, `Event.ON_HIT` に `ha.あばれる_apply` を登録。
  `"あばれる"`（ノーマル版）とタイプ以外完全に同一の構造であり、一次情報とも
  一致しており妥当。変更なし。
- `src/jpoke/handlers/move_attack.py` の `あばれる_apply`
  （`ON_HIT`, `subject_spec="attacker:self"`）: 初回命中時にのみランダム
  2〜3ターンの揮発性状態を付与し、既に付与済み（強制行動2ターン目以降）の
  場合は何もしない実装で仕様と一致。げきりん・はなびらのまい・だいふんげきと
  共用。変更なし。
- `src/jpoke/data/volatile.py` の `"あばれる"`（`forced=True`,
  `ON_MODIFY_COMMAND_OPTIONS` で `force_command`, `ON_DAMAGE_HIT`
  priority=180 で `あばれる_tick`）、`src/jpoke/handlers/volatile.py` の
  `あばれる_tick`（ターン経過後カウント0でこんらん付与）は
  `docs/spec/volatiles/あばれる.md` の仕様通りで、`あばれる`（技）レビュー時に
  確認済み。変更なし。

## リーサル計算

- 対象外（`docs/progress/move.md` のげきりん行「リーサル実装」列が `n/a`）。
  `あばれる`（ノーマル）と同様、強制連続攻撃自体は通常のダメージ計算の繰り返し
  であり、リーサル計算上の特別な補正は不要なため。

## テスト

- `tests/moves_attack/test_move_ka.py` に既存の2件
  （`test_げきりん_初回命中時にあばれる揮発状態が付与される`,
  `test_げきりん_カウント終了後にこんらんが付与される`）で
  ON_HIT経由の初回付与・こんらん移行を確認済み。`あばれる`本体や
  `だいふんげき`・`はなびらのまい`と同水準のテストが揃っており、実装と照合し
  過不足なし。変更なし。
- `tests/volatiles/` 側の揮発性状態そのもの（カウント進行・強制行動）は
  `tests/volatiles/test_volatile_a.py` の `あばれる` 揮発状態テストで
  共用カバー済みのため、げきりん専用の追加テストは不要と判断した。
- `python -m pytest tests/ -v` を実行し、げきりん関連の2件はすべて成功
  （`tests/moves_attack/ -k げきりん` で 2 passed）。結果は
  `.loop/test_logs/げきりん.log` に保存した。全体では
  `tests/test_tree_search_framework.py` の2件が失敗しているが、これは
  同時実行中の別作業（`scripts/tree_search/` 関連の並行編集）による
  未コミットの作業中状態に起因するものであり、げきりんの変更（今回はテスト
  コード無変更）とは無関係であることを、げきりん・moves_attack 関連テストの
  みを対象にした単独実行（2 passed / 844 passed）で確認した。
