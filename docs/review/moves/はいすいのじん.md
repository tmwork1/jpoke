## 実装（handlers/ / data/）
- **バグ修正1: `MoveData.target` 未設定**（`data/moves/move_ha.py`）。デフォルト値 `"foe"` の
  ままだったため、`Move.is_blocked_by_protect` / `Move.is_reflectable` が誤って `True` になり、
  相手の `まもる`・`マジックコート`/`マジックミラー` で防がれ得るバグがあった（一次情報の
  「まもる: ×」「マジックコート: ×」に反する）。加えて `いたずらごころ`（あくタイプ相手への
  優先度技無効化）・`おうごんのからだ`（他のポケモンからの変化技無効化）等、
  `ctx.move.target == "foe"` を判定条件に含む特性ハンドラの誤爆リスクもあった。
  `target="self"` を明示して修正。
- **バグ修正2: `にげられない_remove_on_foe_switch` の誤爆**（`handlers/volatile.py`）。
  この共通ハンドラは元々「にげられない状態を与えた相手（発生源）が場を離れたら解除する」という
  相手対象トラップ技（とおせんぼう等）向けの処理だったが、はいすいのじんは自分自身が発生源かつ
  対象のため、対戦相手が単に交代しただけで誤ってにげられない状態が解除されてしまうバグがあった
  （一次情報「使用時に相手の場にいたポケモンが場から去っても、この技によるにげられない状態は
  解除されない」に反する）。`Volatile.move_name == "はいすいのじん"` の場合はスキップするガードを
  追加して修正（くらいつく等の相手対象付与は `move_name` を設定しないため影響なし）。
- **実装方式の統一**: `はいすいのじん_apply` を、`battle.modify_stats()` の戻り値が空辞書になる
  ことを利用した自動失敗方式に変更（`ちょうのまい`・`とぐろをまく`・`ソウルビート` 等、既存の
  複合ランク変化技と同じパターン）。従来は `はいすいのじん_can_apply`（`ON_TRY_MOVE_1`）内に
  「全能力ランクが+6なら失敗」という冗長な個別チェックが実装されており、`ON_STATUS_HIT` 側の
  `はいすいのじん_apply` は `battle.modify_stats()` の戻り値を無視して常に成功扱いにしていた
  （挙動としては結果的に等価だが、失敗判定の責務が二重化していた）。これを解消し、
  `ON_TRY_MOVE_1` 側は「はいすいのじん起因のにげられない状態がすでにある場合の失敗」のみを
  担当するよう整理した。
- PPは `src/jpoke/data/ps-champ-ja/moves.json` に該当エントリが存在しないため（タイレーツ系統の専用技で
  Champions図鑑に含まれない可能性がある）、Gen9本家基準のPP=5をそのまま維持した
  （既存実装値と一致・変更なし）。
