## 実装計画書
- 新規作成（`docs/plan/moves/やきつくす.md`）。
  - 既存実装（`やきつくす_burn_item`）のハンドラ構成を記載。
  - `docs/spec/turn.md` の `Event.ON_DAMAGE` 優先度表に「やきつくすできのみ消失」（優先度30）と
    記載されているが、`Event.ON_MODIFY_MOVE_DAMAGE`（`modify_hp` 呼び出し前）を採用する理由を
    明記した。`Event.ON_DAMAGE_HIT` は `modify_hp` によるHP反映（内部で `Event.ON_HP_CHANGED` が
    同期発火し、オボンのみ等のHP回復ピンチきのみが発動する）より後に発火するため、そちらを採用
    すると「回復せずに燃やされる」という仕様を満たせないことを、実際に `やきつくす` と
    `はたきおとす`（`Event.ON_DAMAGE_HIT` を採用）それぞれでオボンのみ保有相手への命中を検証して
    確認した（前者はオボンのみを消費するだけで回復せず、後者はオボンのみが発動し回復してしまう）。
  - はたきおとす側の不具合（`docs/spec/moves/はたきおとす.md` の記載と実装が食い違っている点）は
    本タスクのスコープ外として指摘のみに留め、修正はしていない。

## 実装（handlers/ / data/）
- 修正した内容:
  1. **ノーマルジュエルが焼却対象になっていなかった不具合を修正**
     - `やきつくす_remove_berry`（`handlers/move_attack.py`）が `ctx.defender.item.is_berry()` のみを
       判定しており、ノーマルジュエルを保有している場合に焼却されない不具合があった
       （`docs/wiki/moves/やきつくす.html` の「相手が きのみ・ノーマルジュエルを持っている場合に
       焼きつくして使えなくする」という仕様に反する）。
     - `item.name == "ノーマルジュエル"` の判定を追加し、きのみ・ノーマルジュエルのいずれかを
       保有していれば焼却するように修正した。
  2. **みがわりに防がれた場合でも実体の持ち物が燃やされる不具合を修正**
     - みがわりによるダメージ肩代わり判定（`ctx.substitute_damage`）を一切参照しておらず、
       みがわり状態の相手に対しても本体の持ち物が焼却されてしまっていた（`docs/wiki/moves/
       やきつくす.html` の「みがわり状態に防がれた場合、持ち物は燃やせない」という仕様に反する）。
     - `むしくい_steal_and_use_berry` と同じパターンで `ctx.substitute_damage > 0` のとき何もせず
       抜けるガードを追加した。
  - 上記2点の修正に伴い、関数名を `やきつくす_remove_berry` から `やきつくす_burn_item`
    （きのみに限らずジュエルも扱うことを明確化）へ変更した。`data/moves/move_ya.py` の参照も
    合わせて更新し、`scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` を実行した。
  - タイプ半減きのみ（オッカのみ等）との発動順序、特性ねんちゃく・マジシャンとの相互作用は
    いずれも既存の共通処理（`Event.ON_CALC_DAMAGE_MODIFIER` での消費順序・
    `battle.item_manager.remove_item` が内包する `can_change_item` 判定・マジシャンの
    `Event.ON_DAMAGE_HIT` への登録）だけで仕様通りに成立することを確認したため、個別対応は
    行っていない。
  - `data/moves/move_ya.py` を修正したため `python scripts/sort_data/sort_moves.py` を実行した。
