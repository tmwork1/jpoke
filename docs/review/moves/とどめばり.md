## 実装（handlers/ / data/）
- **修正1**: `src/jpoke/core/move_executor.py` の `_execute_hit` で、
  `actual_damage <= 0` により早期リターンする際、`ctx.defender.fainted` が
  真（ばけのかわのフォルムチェンジ消費ダメージ等、技ダメージ以外で
  このヒット中に新たにひんしになった場合のみ真になる）であれば
  `Event.ON_MOVE_KO` を追加で発火するよう修正した。
  - 修正前は、ばけのかわの相手をフォルムチェンジ消費ダメージで倒した場合、
    技自体のダメージが0であるため `Event.ON_MOVE_KO` が一切発火せず、
    とどめばりだけでなくじしんかじょう等のKOトリガー特性も発動しなかった
    （`docs/spec/abilities/ばけのかわ.md` の既存記載と不整合だった）。
- **修正2**: `src/jpoke/data/move.py` のとどめばりのハンドラを
  `Event.ON_DAMAGE_HIT` から `Event.ON_MOVE_KO`（`priority=50`）に変更した。
  - `Event.ON_MOVE_KO` は相手がひんしになったときのみ発火するため、
    修正1と合わせてばけのかわ経由のKOでも効果が発動するようになった。
  - じしんかじょう等の同種KOトリガー特性（`priority` デフォルト100）より
    先に発動するよう `priority=50` を明示し、ハンドラ本体
    （`src/jpoke/handlers/move_attack.py` の `とどめばり_boost_attacker_atk`）の
    冗長な生存チェックを削除した。
- **修正3**: `src/jpoke/data/ability.py` の `ミイラ`・`さまようたましい`・
  `とれないにおい` の `Event.ON_DAMAGE_HIT` ハンドラに `priority=20` を追加した。
  - 修正前はいずれも `priority` 未指定（デフォルト100）で、とどめばりと同じ
    優先度だったため、「相手を倒すと同時に自分の特性が変化する」場面で
    `EventManager._sort_handlers` が素早さでタイブレークしてしまい、
    攻撃側が速い場合に変化前の特性（たんじゅん/あまのじゃく）で
    とどめばりの効果が適用される誤りがあった
    （同種の受動特性・てつのトゲ等は既に `priority=20` で登録されている）。
  - `sort_handlers.py`・`sort_moves.py`・`sort_abilities.py` を実行済み。
