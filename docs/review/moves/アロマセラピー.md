# アロマセラピー レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/アロマセラピー.md`）。一次情報（ポケモンWiki）に基づき、基本情報・効果・
  判定フラグ・そうしょく/みがわり/かいふくふうじ/ぜったいねむりとの相互作用・Pokémon Champions での
  補足を記載した。
- 対象は「味方全体（自分と選出チーム全員、控え含む）」であり、ダブルバトル専用の「隣接する味方」限定
  効果とは異なるため、シングルバトルでも「控えの状態異常を回復できる」実効的な効果を持つことを明記した。
- PPは `src/jpoke/data/ps-champ-ja/moves.json` に本技のエントリが存在しないため、Gen9本家のPP=5を採用（他の
  同様のケースであるVジェネレート・あてみなげ等と同じ扱い）。

## 実装計画書
- 新規作成（`docs/plan/moves/アロマセラピー.md`）。既存の「いやしのすず」計画書と同一パターン
  （`ON_STATUS_HIT` priority=100、状態異常のポケモンがいない場合は失敗）を踏襲する内容で記載した。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/move_status.py`: `アロマセラピー_apply` を `アロマセラピー_cure_team_ailment` に
  改名し、`いやしのすず_cure_ailment` と同じ「対象チームに状態異常のポケモンが1体もいない場合は
  `HandlerReturn(value=False, stop_event=True)` を返して技を失敗させる」ロジックを追加した
  （従来は状態異常のポケモンがいなくても技が成功扱いになっており、ポケモンWiki「技の仕様」節の
  「治せるポケモンがいなければ しかし うまく決まらなかった!! と出る」という第五世代以降の仕様と
  食い違っていた）。
- `src/jpoke/data/moves/move_a.py`: `target` を `"self"` から `"own_side"` に修正した（味方全体を
  対象とする効果のため、`いやしのすず`・`おいかぜ` 等の同種の技と同じ `own_side` を用いるのが適切。
  `is_blocked_by_protect`・`is_reflectable` の判定には影響なし）。ハンドラの参照名も
  `アロマセラピー_cure_team_ailment` に更新した。

## リーサル計算
- 対象外（`docs/progress/move.md` のリーサル実装列が `n/a`。ダメージを与えない変化技のため）。

## テスト
- `tests/moves_status/test_move_a.py`:
  - `test_アロマセラピー_状態異常なしでも失敗しない`（従来の「技が失敗せず正常終了する」という誤った
    期待を検証していたテスト）を `test_アロマセラピー_状態異常なしなら失敗` に修正し、
    `いやしのすず` の対応テストと同じ内容（状態に変化がないことを確認）に更新した。
  - `test_アロマセラピー_まもるで防がれない` を新規追加し、相手の「まもる」状態でも本技が防がれ
    ないことを確認した（`いやしのすず` の対応テストと同一パターン）。
  - `test_アロマセラピー_使用者の状態異常が回復される`・`test_アロマセラピー_控えの状態異常も回復
    される` は変更なし。
