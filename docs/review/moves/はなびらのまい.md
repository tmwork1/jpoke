# はなびらのまい レビュー結果

## 仕様書

- `docs/spec/moves/はなびらのまい.md` は存在しない。はなびらのまい固有の効果（強制連続
  攻撃・使用後こんらん）は、あばれる・げきりん・はなびらのまい・だいふんげきで共用の
  揮発性状態として `docs/spec/volatiles/あばれる.md` に既に詳細な仕様書
  （付与条件・消滅条件・こんらん付与の特例・ターン処理優先度）が存在しており、
  これが技レベルの効果を包含している。`あばれる`・`げきりん`・`だいふんげき` の
  レビュー（`docs/review/moves/あばれる.md`, `docs/review/moves/げきりん.md`,
  `docs/review/moves/だいふんげき.md`）で踏襲された方針（共用仕様書がある場合は
  技単体の個別ファイルを新規作成しない）に倣い、新規作成は行わなかった。
- 一次情報（`docs/wiki/moves/はなびらのまい.html`）と実装を照合し、以下を確認した：
  タイプ＝くさ、分類＝特殊、威力＝120（第五世代以降）、命中率＝100、
  PP＝10（第五世代以降）、範囲＝相手ランダム1体（シングル対象のため実質相手
  1体固定）、直接攻撃＝○、追加効果＝×、急所／命中判定＝各回、まもる＝○、
  マジックコート／よこどり＝×、オウムがえし＝○、おどりのわざ（かがやくうろこ等の
  対象）。いずれも実装（`src/jpoke/data/moves/move_ha.py`）と一致していた。

## 実装計画書

- `docs/plan/moves/はなびらのまい.md` は存在しない。同様の理由で、あばれる系共有の
  揮発性状態実装（`あばれる_apply` / `あばれる_tick` / `force_command`）を扱う
  `docs/plan/moves/攻撃技_あばれ系.md` が技の実装内容を実質的にカバーしているため、
  個別の新規作成は行わなかった。
  なお同計画書内の「`はなびらのまい` はハンドラ未登録」という記述は、実装済みの
  現状（下記参照）と食い違う古い草稿段階の記述だが、`あばれる`・`げきりん`・
  `だいふんげき` レビュー時にも同様の古い記述（例: だいふんげきの分類・威力誤記）は
  当該計画書自体を書き換える対象にしていないため、本レビューでも踏襲し変更しなかった
  （進捗表・実装コードが現状の正としている）。

## 実装（handlers/ / data/）

- `src/jpoke/data/moves/move_ha.py` の `"はなびらのまい"`: `type="くさ"`,
  `category="special"`, `pp=10`, `power=120`, `accuracy=100`,
  `flags={"contact", "dance"}`, `Event.ON_HIT` に `ha.あばれる_apply` を登録。
  一次情報（タイプ・分類・威力・命中率・PP・直接攻撃・おどりのわざ）と一致しており
  妥当。変更なし。
- `src/jpoke/handlers/move_attack.py` の `あばれる_apply`
  （`ON_HIT`, `subject_spec` 省略＝`attacker:self` 相当）: 初回命中時にのみランダム
  2〜3ターンの揮発性状態を付与し、既に付与済み（強制行動2ターン目以降）の場合は
  何もしない実装で仕様と一致。あばれる・げきりん・だいふんげきと共用。変更なし。
- `src/jpoke/data/volatile.py` の `"あばれる"`（`forced=True`,
  `ON_MODIFY_COMMAND_OPTIONS` で `force_command`, `ON_DAMAGE_HIT`
  priority=180 で `あばれる_tick`）、`src/jpoke/handlers/volatile.py` の
  `あばれる_tick`（ターン経過後カウント0でこんらん付与）は
  `docs/spec/volatiles/あばれる.md` の仕様通りで、既存レビュー
  （あばれる・げきりん・だいふんげき）で確認済み。変更なし。

## テスト

- `tests/moves_attack/test_move_ha.py` に既存の2件
  （`test_はなびらのまい_初回命中時にあばれる揮発状態が付与される`,
  `test_はなびらのまい_カウント終了後にこんらんが付与される`）で
  ON_HIT経由の初回付与・こんらん移行を確認済み。`あばれる`本体や
  `げきりん`・`だいふんげき`と同水準のテストが揃っており、実装と照合し過不足なし。
  変更なし。
- `tests/volatiles/` 側の揮発性状態そのもの（カウント進行・強制行動）は
  `tests/volatiles/test_volatile_a.py` の `あばれる` 揮発状態テストで共用カバー済み
  のため、はなびらのまい専用の追加テストは不要と判断した。
- `python -m pytest tests/ -v` を実行し、3567 passed, 1 skipped で全件成功。
  結果は `.loop/test_logs/はなびらのまい.log` に保存した。
