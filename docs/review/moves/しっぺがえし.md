# しっぺがえし レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/しっぺがえし.md`）。
- 一次情報（`docs/wiki/moves/しっぺがえし.html`）に基づき、基本情報（あく/物理/威力50/命中100/PP10/接触あり）、
  「相手がすでに行動していると威力2倍」という効果、および「行動した」とみなされる／みなされない
  各ケース（状態異常・状態変化・なまけによる不発は行動扱い、反動技のリチャージによる不発は
  非行動扱い、きあいパンチ等の準備行動のみでは非行動扱い、交代直後で技未使用の相手は第五世代以降
  非行動扱い）を明記した。

## 実装計画書
- 新規作成（`docs/plan/moves/しっぺがえし.md`）。
- 既存実装の内容整理に加え、発見した不具合（交代直後の相手を誤って「行動済み」と判定してしまう
  問題）とその修正方針、および対応を見送った既知の制限（リチャージによる不発の判定不可）を記載した。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/move_attack.py` の `しっぺがえし_double_power_when_second` を修正した。
  - 修正前: `battle.query.is_second_actor(attacker_player)` のみで判定していたため、相手が
    その場に交代してきたばかりで技を未使用の場合でも威力2倍が誤って適用されていた
    （`_run_switch_phase` で交代したプレイヤーのインデックスも `action_order` に加わるため）。
  - 修正後: `battle.player_states[defender_player].has_switched` が真の場合は威力補正を適用せず
    早期リターンするようにした（`フォーカスレンズ_boost_accuracy_second` と同じパターン）。
- `src/jpoke/data/move.py` の `"しっぺがえし"` エントリ自体（type/category/pp/power/accuracy/flags）
  は一次情報と一致しており変更なし。
- `python scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` を実行済み（並び順に変更なし）。
