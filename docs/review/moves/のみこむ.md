# のみこむ レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/のみこむ.md`）。一次情報（`docs/wiki/moves/のみこむ.html`）と
  連動技である `たくわえる`・`はきだす` の仕様書を参照して作成した。
- PPは champions 基準（`docs/champions/move_list.txt`）の12を採用し、Gen9本家基準の10は
  参考として明記した。

## 実装計画書
- 既存の `docs/plan/moves/のみこむ.md` を実装コードと照合し、以下の食い違いを修正した。
  - 記載されていたファイルパス `data/move.py` / `handlers/move_status.py` のコード例が
    現在の実装（`data/moves/move_na.py` へ分割済み、`category="status"`、
    `flags={"heal"}`、ランクキーが `"B"/"D"` ではなく `"def"/"spd"`、全回復処理が
    `v=mon.max_hp - mon.hp` ではなく `r=1.0`（ヘルパー辞書 `_heal_ratio` 経由）で
    実装されている）と乖離していたため、実装済みコードの引用を最新化した。
  - PP を計画書上も 10 → 12 に修正した（後述「実装」参照）。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_na.py` の `"のみこむ"` エントリで `pp=10` となっていたのを
  `pp=12` に修正した（`docs/champions/move_list.txt` の `のみこむ 12` に合わせる。
  champions ではすべての技のPPが8/12/16/20の4段階に統一されているため、Gen9本家基準の
  10になっていたのは移行漏れのバグと判断）。
- `handlers/move_status.py` の `のみこむ_check_can_use` / `のみこむ_apply` 自体のロジックは
  仕様書と整合しており、修正なし。

## リーサル計算
- 対象外（`docs/progress/move.md` の のみこむ 行のリーサル実装列は n/a）。
  自分のHPを回復するだけの技であり、相手側HP分布を左右しないため専用ハンドラは不要。

## テスト
- `tests/moves_status/test_move_na.py` の既存テスト（たくわえカウント別回復量、
  たくわえなしでの失敗、クリアボディでのランク逆補正阻止不可、使用後のランク復元）は
  実装・仕様書と整合しており、修正なし。
- `python -m pytest tests/ -v` で全4318件（1件skip）がパスすることを確認した
  （実行中に無関係な乱数依存テスト `test_むしくい_ねんちゃく持ちをひんしにさせた場合は奪える`
  が1回だけflaky失敗したが、単体再実行では成功しており のみこむ の変更とは無関係）。
