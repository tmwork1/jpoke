## 実装（handlers/ / data/）

一次資料と突き合わせた結果、以下の実装誤り・欠落を発見し修正した。

1. **交代先が「次の控え」固定になっていた誤り**（最重要）
   - `handlers/move_attack.py` の `_force_switch_next`（`commands[0]` で先頭固定）を
     `_force_switch_random` にリネームし、`battle.random.choice(commands)` でランダムに
     選ぶよう修正。`ともえなげ_force_switch` / `ドラゴンテール_force_switch` の両方に適用。
   - `python scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` を実行済み。

2. **ねをはる状態が強制交代技を無効化しない欠落**
   - 一次資料に「対象が...ねをはる状態のポケモンである場合、ダメージのみを与えて
     交代効果は発動しない」と明記されているが、`ねをはる` の `VolatileData` には
     `Event.ON_TRY_BLOW` ハンドラが未登録で、きゅうばん・ばんけんのみが無効化される
     状態だった。
   - `handlers/volatile.py` に `ねをはる_block_blow` を追加し、
     `data/volatile.py` の `"ねをはる"` エントリに
     `Event.ON_TRY_BLOW: h.VolatileHandler(h.ねをはる_block_blow, subject_spec="defender:self")`
     を登録した。
   - `python scripts/sort_handlers.py src/jpoke/handlers/volatile.py` を実行済み。

3. **にげられない・バインド・フェアリーロックや特性かげふみ・ありじごく・じりょくを
   無視して発動すべきという仕様が満たされていなかった欠落**
   - 一次資料に「相手がにげられない/バインド状態であるときや、場がフェアリーロック
     状態であるとき、特性かげふみ/ありじごく/じりょくの効果を受けているときでも
     交代させることができる」と明記されている。
   - しかし `_force_switch_next`（旧実装）・`blow`（ほえる・ふきとばし共用）はいずれも
     `CommandManager.get_available_switch_commands` を経由しており、この関数は
     `Query.can_switch`（`ON_CHECK_TRAPPED` 経由でバインド等のとらわれ状態を判定）を
     通すため、対象がこれらの状態にあると強制交代自体が発動しない誤りがあった
     （ねをはるも同様に阻害されてしまっていたため、上記2の修正だけでは
     「とらわれ状態全般を無視する」という仕様を満たせない）。
   - `handlers/move.py` に共有ヘルパー `get_forced_switch_commands(battle, player)` を
     新設し、`CommandManager.get_available_switch_commands` を経由せず控えの生存
     ポケモンから直接コマンドを構築するようにした（とらわれ状態を一切参照しない）。
     `ねをはる`・きゅうばん・ばんけんの無効化は呼び出し側で行う `Event.ON_TRY_BLOW`
     の判定に委ねる。
   - `handlers/move_attack.py` の `_force_switch_random` と `handlers/move_status.py` の
     `blow`（ほえる・ふきとばし共用）の両方をこのヘルパー経由に変更した。
   - 併せて `core/command_manager.py` の `get_available_switch_commands` に
     `force: bool = False` 引数を追加したが、最終的な強制交代系ハンドラの実装は
     `get_forced_switch_commands` ヘルパー（handlers 層）を利用する方式に統一した
     （`force` 引数は将来的な用途のためそのまま残置）。

4. **PP誤り（進捗表のみ、実装は元々正しい）**
   - `data/move.py` の `ともえなげ`・`ドラゴンテール` は元々 `pp=10`（正しい値）だったが、
     `docs/progress/move.md` の当該行に `12` という誤ったPPが記載されていた。
     進捗表の値を `10` に修正した（実装コードの変更はなし）。

## リーサル計算

- 対象外（`docs/progress/move.md` の「リーサル実装」「リーサルテスト」列がいずれも `n/a`）。
  強制交代の追加効果はダメージ量そのものに影響せず、通常のダメージ計算で完結するため
  専用のリーサルハンドラは不要。
