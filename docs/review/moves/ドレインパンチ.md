## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ta.py` の `ドレインパンチ` エントリ
  （type=かくとう, category=physical, pp=12, power=75, accuracy=100,
  flags={"contact", "punch", "heal"}）は一次情報・仕様書と一致しており修正不要だった。
- 同ファイル: `Event.ON_HIT` ハンドラ登録に `docs/spec/turn.md` の priority 根拠コメントを追加
  （`ドレインキッス` 等の他のドレイン技と同じ記法に統一）。
- `handlers/move_attack.py` の `ドレインパンチ_drain` は共通関数 `_drain_hp`（heal_ratio=0.5）を
  使用しており、他のドレイン技（ギガドレイン・きゅうけつ・すいとる等）と同じ実装パターンで
  一次情報（回復量=与ダメの1/2、端数は四捨五入、最低1回復）と一致しており修正不要だった。
- パンチ技威力補正（てつのこぶし・パンチグローブ）、かいふくふうじでの選択不可／失敗は
  いずれも `flags` を参照する既存の汎用ハンドラで処理済み（`ドレインキッス` レビューで
  かいふくふうじ対応は既に実装済み）であることを確認した。
