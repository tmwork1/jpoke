## 実装（handlers/ / data/）
- 修正した内容:
  1. **ねんちゃく所持相手への威力補正が誤って無効化される不具合を修正**
     - `はたきおとす_power`（`handlers/move_attack.py`）が `ItemManager.can_change_item()` を
       そのまま呼んでいたため、除去可否の判定にねんちゃく特性の `ON_CHECK_ITEM_CHANGE` ハンドラまで
       含まれ、`docs/spec/abilities/ねんちゃく.md` に明記されている「威力補正はかかる」仕様に反して
       威力補正が無効化されていた。加えて、威力判定と実際の除去判定の両方で `can_change_item()` が
       呼ばれるため、ねんちゃくの特性発動ログが2回記録される不具合もあった。
     - `EventContext` に `dry_run: bool = False` フィールドを追加し、
       `ItemManager.can_change_item(..., dry_run: bool = False)` から伝播するようにした
       （`core/context.py`, `core/item_manager.py`）。
     - `ねんちゃく_prevent_item_change`（`handlers/ability.py`）で `ctx.dry_run` が True の場合は
       ブロック・ログ記録を行わず素通りするよう修正。
     - `はたきおとす_power` は `can_change_item(ctx.defender, source=ctx.attacker, dry_run=True)` を
       呼ぶように変更（`handlers/move_attack.py`）。
     - 種族+持ち物ロック系ハンドラ（だいこんごうだま・くちたけん・マルチタイプ等）は `dry_run` を
       参照しないため、これらのケースでは従来通り威力補正が無効化される（仕様通り）。
  2. **メガストーン・オーガポンのめんに種族ロックが未実装だった不具合を修正**
     - `removable=False` が設定済みにもかかわらず、メガストーン（`_add_mega_stones` 生成分）と
       オーガポンのめん（いしずえのめん・いどのめん・かまどのめん）には `ON_CHECK_ITEM_CHANGE`
       ハンドラが未登録で、対応する種族が持っていてもはたきおとす等で奪える状態になっていた
       （`docs/spec/moves/どろぼう.md` にも同様のロックが仕様として明記されていたため、
       どろぼう・ほしがる・トリック・すりかえ等、他の道具移動系にも影響する共通の不具合だった）。
     - `handlers/item.py` に `mega_prevent_item_change`（`MEGA_STONES` を参照し対応種族のみ除去を防ぐ）
       と `オーガポンのめん_prevent_item_change`（既存の `_dedicated_item_prevent_item_change`
       ヘルパーで種族プレフィックス「オーガポン」を判定）を追加。
     - `data/item.py` の `_add_mega_stones()` および いしずえのめん・いどのめん・かまどのめんに
       `ON_CHECK_ITEM_CHANGE` ハンドラを登録した。
