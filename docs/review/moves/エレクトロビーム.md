# エレクトロビーム レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/エレクトロビーム.md`）。
- `docs/wiki/moves/エレクトロビーム.html` を一次情報として、基本情報（タイプ・分類・威力・命中・PP=12）、
  溜め技としての挙動（1ターン目にとくこう+1で溜め、2ターン目に攻撃）、天気（あめ/おおあめ）による
  溜めスキップ、パワフルハーブとの優先順位（天気優先で消費されない）、ノーてんき/エアロック/
  ばんのうがさによるあめ無効化、ちからずくに関する本家 SV との Champions での差異（Champions では
  ちからずくの特別な効果は発動しない）を記載した。

## 実装計画書
- 既存の `docs/plan/moves/エレクトロビーム.md` は実装前段階（未実装扱い）の古い内容のままで、
  実際の実装（3ハンドラ構成: `boost_spa` priority=50 / `weather_skip` priority=90 /
  `charge` priority=100）と食い違っていたため、実装済み内容に合わせて全面更新した。
  - 旧計画は「天気による溜めスキップ」を `charge` ハンドラ内に埋め込む2ハンドラ構成だったが、
    これだとパワフルハーブ（デフォルト priority=100）より後に判定されてしまい、
    「天気があめのときはパワフルハーブが優先されず消費されない」という仕様に反する。
    実際の実装は `エレクトロビーム_weather_skip`（priority=90）を独立ハンドラとして切り出し、
    パワフルハーブより先に判定・`stop_event=True` で停止させる正しい設計になっていたため、
    この設計をそのまま計画書に反映した（ソーラービーム/ソーラーブレードと同一パターン）。
  - PP数の食い違いに関する古い注意書き（pp=10 vs 12）は、実装が既に pp=12
    （`docs/champions/move_list.txt` 準拠）で解決済みのため、解決済みである旨に修正した。
  - lethal ハンドラ構成（`LethalEvent.ON_BEFORE_MOVE` + `ctx.move_secondary` ゲート、
    メテオビームと同一パターン）を追記した。
  - ノーてんき/エアロック/メガソーラーとの相互作用は `weather_for` 等の汎用機構で自動対応済みで
    あり、エレクトロビーム固有の追加ハンドラは不要である旨を明記した。
