## 実装計画書
- 既存の `docs/plan/moves/ガードシェア.md` は「未実装」前提の記述だったが、実際には既に実装済みだったため、実態に合わせて全面的に更新した。
- 誤りを修正:
  - PP: 10 → 12（Championsの`move_list.txt`基準）
  - 命中判定: `accuracy=100` としていたが、必中技のため `accuracy=None` が正しい旨を明記
  - まもる・みがわりのブロックは既定動作（`target=="foe"`）で処理されるため追加フラグ不要である旨を明記
  - マジックコートの反射回避には `unreflectable` フラグが必須である旨を追記（実装に欠落していたバグ）
  - 交代時の実数値リセットの実装方法（`Pokemon.reset_on_switch_out()` に `update_stats()` を追加）を追記
- `_stats_manager._stats`（二重下線）ではなく実際の実装で使われている `_stats_manager.stats`（公開プロパティ、リストの実体を返す）を正として記述を修正した。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/move_status.py` の `ガードシェア_equalize_stats` 自体は仕様通りで変更なし（B/D実数値を平均化・切り捨て、ランク変化なし）。
- `src/jpoke/data/moves/move_ka.py` の `"ガードシェア"` エントリを修正:
  - `pp=10` → `pp=12`（Champions基準の move_list.txt に合わせて修正。Gen9本家値からの移行漏れバグ）
  - `accuracy=100` → `accuracy=None`（必中技。旧実装だと回避率・命中率ランクの影響を受けてしまうバグだった）
  - `flags={"unreflectable"}` を追加（マジックコートで跳ね返されない仕様だが、フラグが無く反射可能になっていたバグ）
- `src/jpoke/model/pokemon.py` の `Pokemon.reset_on_switch_out()` に `self.update_stats()` を追加。
  - ガードシェア（および同一パターンで実数値を直接書き換えるパワーシェア・パワートリック）は交代でリセットされる仕様だが、`_stats_manager.stats` への直接書き換えは `reset_on_switch_out()` で一切リセットされておらず、交代後も変更後の実数値が残り続けるバグだった。
  - `update_stats()` は現在の種族値・レベル・性格・個体値・努力値から実数値を再計算するのみで、`self.hp`（現在HP）自体には影響しない（`keep_damage` を指定しない限り）ため、他の交代処理への副作用はない。
