# トラップシェル レビュー結果

## 一次情報確認の結果（重要）

`docs/wiki/moves/トラップシェル.html` を確認したところ、インフォボックスに赤字で
「BDSPで使用不可」「SVで使用不可」と明記されており、ブリリアントダイヤモンド・シャイニングパール
以降・スカーレット/バイオレットでは使用できない技であることが分かった。
`docs/champions/move_list.txt`（ポケモンチャンピオンズの技一覧）にも本技の項目は存在せず、
ポケモンチャンピオンズ側でも採用されていないという二重の根拠が得られた。

README.md の「対象範囲」定義（「第9世代で実装されていない技・アイテム・特性・ポケモンなどは
実装しない」）に該当するため、**本技は実装対象外と判断した**（`docs/spec/moves/オクタンほう.md` /
`docs/review/moves/オクタンほう.md` に前例あり）。以降の項目はこの判断に基づく対応内容を記載する。

## 仕様書
- 新規作成（`docs/spec/moves/トラップシェル.md`）。
- 対象範囲外である旨と根拠（BDSP/SVで使用不可、ポケモンチャンピオンズにも項目なし）を明記した。
- 参考として第七・第八世代時点の技データ（タイプ:ほのお、特殊、威力150、命中100、PP5、
  優先度-3、直接攻撃なし、専用技バクガメス）と、設置→物理攻撃を受けて起爆→行動順繰り上げで
  即座に反撃する特殊な技の仕様を記録した。

## 実装計画書
- 新規作成（`docs/plan/moves/トラップシェル.md`）。
- 「実装しない」という結論と、過去に対象範囲確認なしに誤って登録されていた `MoveData`
  （タイプ「はがね」・優先度-5など、一次情報と一致しない誤った値を含む）の内容を記録した。

## 実装（handlers/ / data/）
- 誤って登録されていた実装を削除した。
  - `src/jpoke/data/moves/move_ta.py`: `"トラップシェル": MoveData(...)` エントリを削除
    （一次情報ではタイプ「ほのお」・優先度-3だが、タイプ「はがね」・優先度-5という誤った値で
    登録されており、かつ設置・起爆条件などの中核仕様も一切実装されていなかった）。
  - `src/jpoke/types/move.py`: `python scripts/generate_literals/generate_move_literal.py` を
    実行して `MoveName` Literal から `"トラップシェル"` を自動削除し、MOVES辞書と再整合させた。
  - `python scripts/sort_data/sort_moves.py` を実行（差分なし、既存ファイルは元々整列済みだった）。
  - `handlers/move_attack.py` 等に `トラップシェル` 専用ハンドラは存在しなかった
    （`non_negoto` フラグ自体は他技（きあいパンチ等）でも使う汎用フラグのため、
    フラグ機構自体には影響なし）。
- ついでに見つかった誤記を修正した。
  - `docs/spec/action_order.md`: 優先度-5ブロックに誤って掲載されていた「トラップシェル」を削除
    （実際の優先度は-3であり、対象範囲外のため-3ブロックにも追加しない）。
  - `docs/spec/turn.md`: `Event.ON_BEFORE_MOVE` の「トラップシェルの準備行動」、
    `Event.ON_TRY_MOVE_1` の「トラップシェル: 物理技を受けていない」の行を削除
    （対応する実装が存在せず、対象範囲外のため今後も実装されない）。
