# ミラータイプ レビュー結果

## 仕様書
- `docs/spec/moves/ミラータイプ.md` が存在しなかったため、`docs/wiki/moves/ミラータイプ.html`
  （一次情報）と `src/jpoke/data/ps-champ-ja/moves.json` をもとに新規作成した。
- 一次情報から以下の重要な仕様を新たに反映した。
  - PP は Wiki記載の15ではなく `move_list.txt` 準拠の16。
  - **使用者がテラスタル中の場合は失敗する**（旧計画書には記載がなかった）。
  - まもるで防がれる（対象がfoeで `unprotectable` フラグ無し）。
  - マジックコートで跳ね返されない（`unreflectable` フラグが必要）。
  - みがわり状態の対象にも成功する（`bypass_substitute` フラグが必要）。
  - 対象のタイプが「特性・技等の効果で変化している場合は現在のタイプをコピーする」
    （`defender.data.types` ではなく `defender.types` を参照すべき）。
  - 対象のタイプが完全に無い場合（純ひこうタイプの `はねやすめ` 等）はノーマルタイプとして扱う。
  - よこどり・オウムがえしは本プロジェクト未実装のため対象外と明記。
  - もりののろい/ハロウィンのボラタイル状態自体のコピー等、極めて限定的な複合エッジケースは
    未実装・未検証として「対象範囲外・簡略化した点」節に明記。

## 実装計画書
- `docs/plan/moves/ミラータイプ.md` は「未実装」を前提にした古い内容（`handlers なし`、PP15、
  優先度根拠のみ記載）だったため、実装済みの現状と新仕様書に合わせて全面更新した。
  - PP を15→16に修正。
  - 使用者テラスタル中の失敗チェックを追加。
  - 対象タイプの決定ロジックを `defender.data.types` 参照から `defender.types` 参照
    （`ミラータイプ_取得_対象タイプ`関数）に変更し、テラスタル・特性変化・追加/除去タイプを
    一括して考慮できるようにした。
  - `flags={"unreflectable", "bypass_substitute"}` の追加を明記。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ma.py` の `"ミラータイプ"` エントリに以下のバグを発見・修正した。
  1. **PP誤り**: `pp=15`（Gen9本家基準の値の移行漏れ）→ `pp=16`（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）。
  2. **priority誤り**: `ON_BEFORE_APPLY_MOVE` に `priority=130` を明示指定していたが、
     `docs/spec/turn.md` の `Event.ON_APPLY_MOVE`（「ミラータイプ: タイプ同じ」）は priority 100
     であり、同種の `テクスチャー_can_apply` もデフォルト値（100）で実装されている。
     明示指定を削除し、デフォルトの100に修正。
  3. **`unreflectable`/`bypass_substitute` フラグの欠落**: マジックコートで跳ね返されず、
     みがわりを貫通する一次情報の仕様が未反映だったため、`flags={"unreflectable", "bypass_substitute"}`
     を追加。
- `src/jpoke/handlers/move_status.py` の `ミラータイプ_can_apply`/`ミラータイプ_apply` に以下の
  修正を行った。
  1. **使用者テラスタル中の失敗チェックの欠落**: `attacker.terastallized` の場合に失敗させる
     チェックが存在しなかったため追加。
  2. **対象タイプ取得ロジックの誤り**: `defender.terastallized`（ステラ以外）→ `[tera_type]`、
     それ以外 → `defender.data.types` という分岐は、対象の特性変化（へんげんじざい等）・
     もりののろい/ハロウィンの追加タイプ・はねやすめの除去タイプを一切考慮しない誤りだった。
     `Pokemon.types` プロパティ（テラスタル優先度・追加/除去タイプを含めた現在の実効タイプ）を
     そのまま利用する `ミラータイプ_取得_対象タイプ(defender)` ヘルパー関数に置き換え、
     `can_apply`/`apply` の両方から共通利用するようにした。
  3. 対象のタイプが空リストになる場合（純ひこうタイプが `はねやすめ` でタイプを消している場合）は
     `["ノーマル"]` として扱うフォールバックを追加した。
- `src/jpoke/model/pokemon.py` の `move_override_types` フィールド・`types` プロパティの優先順位・
  `reset_on_switch_out` でのリセットは仕様と一致しており変更なし。
