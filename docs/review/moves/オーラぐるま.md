# オーラぐるま レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/オーラぐるま.md`）。docs/wiki/moves/オーラぐるま.html を一次情報として、
  基本情報（でんきタイプ、物理、威力110、命中100、PP12=champions版、対象1体、優先度0、直接攻撃なし）、
  追加効果（100%で自分のすばやさ+1）、フォルムによるタイプ変更（まんぷくもよう=でんき／
  はらぺこもよう=あく）、ちからずく・そうでん・きけんよちとの相互作用を記載した。

## 実装計画書
- 新規作成（`docs/plan/moves/オーラぐるま.md`）。ハンドラ構成・priority根拠（ON_MODIFY_MOVE_TYPE は
  turn.md 未掲載のため既存実装 `そうでん_move_type` の priority=110 を根拠に判断、ON_HIT はデフォルト
  priority を既存の類似技実装に合わせて採用）・ちからずくとの相互作用・リーサル対象外の理由を記載した。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_a.py`: `pp` を `10` → `12` に修正（champions版PPは
  `src/jpoke/data/ps-champ-ja/moves.json` 136行目より 12 であり、10 は本家基準の移行漏れバグだった）。
- `src/jpoke/data/moves/move_a.py`: `flags={"secondary_effect"}` を追加（ちからずく汎用処理の対象にするため）。
- `src/jpoke/data/moves/move_a.py`: `Event.ON_HIT: h.MoveHandler(ha.オーラぐるま_boost_attacker_spe)` を追加。
  従来は `Event.ON_MODIFY_MOVE_TYPE`（タイプ変更）のみが実装されており、100%で自分のすばやさが
  1段階上がる追加効果が完全に未実装だった（重大な欠落）。
- `src/jpoke/handlers/move_attack.py`: `オーラぐるま_boost_attacker_spe` を新規追加
  （既存の `オーラウイング_boost_attacker_spe` と同一パターン、`modify_attacker_stats(..., stats={"spe": 1})`）。
- `src/jpoke/handlers/ability.py`: 未使用のデッドコード `はらぺこスイッチ_modify_move_type` を削除。
  `data/ability.py` の `はらぺこスイッチ` エントリには登録されておらず、呼び出し元も存在しなかった。
  オーラぐるまのタイプ変更は既に move 側の `オーラぐるま_check_move_type`
  （`Event.ON_MODIFY_MOVE_TYPE`、ability の発動有無に依存せず現在のフォルムで判定する仕様と一致）
  で正しく処理されているため、重複していた特性側の未使用実装を整理した。
