## 実装（handlers/ / data/）
- 修正: `src/jpoke/data/moves/move_na.py` の `"ねがいごと"` エントリ
  - `pp=10` → `pp=12`（`src/jpoke/data/ps-champ-ja/moves.json` の値と不一致だったバグ。Gen9本家基準からの移行漏れ）
  - `target="field"`（天候・フィールド等バトル全体に影響する技専用の分類）→ `target="self"` に修正
    - `target="field"` は あまごい・エレキフィールド・トリックルーム等、盤面全体に影響する技専用の分類であり、
      自陣サイドのみに影響するねがいごとには不適切だった
    - `champions/move_list.txt` の対象欄が「自分」であることと、性質が近い `いやしのねがい`（target="self"）に
      合わせて `"self"` を採用
    - この engine はシングルバトル専用であり `MoveTarget` は `target == "foe"` / `target in ("foe","foe_side")`
      判定にのみ使われるため、修正前後で `is_blocked_by_protect` / `is_reflectable` の結果に変化はない
      （挙動を変えるバグ修正ではなく分類の正確性の修正）
- `handlers/field.py`・`handlers/move_status.py`・`data/field/side_field.py` のハンドラ実装自体は仕様と一致しており変更なし
