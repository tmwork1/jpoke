## 実装（handlers/ / data/）
- **バグ修正**: `handlers/move_attack.py` の `つけあがる_calc_power` が能力ランク合計の対象を
  `("atk", "def", "spa", "spd", "spe")` に限定し、命中率・回避率のランク上昇を除外していた。
  仕様書の根拠（アシストパワー備考の860という数値）と照合した結果、命中率・回避率も対象に含めるのが正しい
  仕様であることが判明したため、`k != "hp"` で hp 以外の全ランクを対象にするよう修正した。
- 同一の誤り・同一パターンのバグが `アシストパワー_boost_power_by_rank`（`アシストパワー` 用ハンドラ）にも
  存在したため、レビュー範囲外だが同時に修正した（レビュー観点「今回の対象エントリ以外の行でも、作業中に
  誤りに気づいた場合は修正する」に基づく）。
- `data/move.py` の `"つけあがる"` エントリ自体（type/category/pp/power/accuracy/flags/handler登録）は
  仕様と一致しており修正不要。
- `python scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` を実行済み。

## リーサル計算
