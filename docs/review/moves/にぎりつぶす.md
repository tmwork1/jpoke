- 共通仕様書 `docs/spec/moves/威力変動技.md`（第5節「対象HP依存威力技」）および、その旧名の

  端数処理ルール（**五捨五超入**、第五世代以降の仕様）を追記した。従来は

## 実装（handlers/ / data/）
- `src/jpoke/handlers/move_attack.py` の `にぎりつぶす_calc_power` を修正した。
  - **バグ修正**: 従来 `int(120 * hp / max_hp)`（floor）で威力を計算していたが、
    一次情報の「五捨五超入」仕様に反していた。
    `round_half_down`（`jpoke.utils.math`、既に同ファイルでインポート済み）を使うように修正。
  - 具体例: カビゴン（max_hp=235）のHPが半分（117）のとき、
    `120 × 117 / 235 = 59.744...` となり、floor では 59 だったが、
    正しくは round_half_down で 60 になる（`しおふき` 等の floor 仕様の技とは
    端数処理ルールが異なることを一次情報で確認済み）。
  - `max(1, ...)` による最低威力1の保証はそのまま維持。
  - `data/move.py` の `"にぎりつぶす"` エントリ自体は仕様と一致しており変更不要。
- `python scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` を実行済み。

## リーサル計算
- 対象外（`docs/progress/move.md` のリーサル実装/リーサルテスト列は `n/a`）。
- 備考: 威力が対象（相手）自身の残りHPに依存する技のため、リーサル計算で複数のHP分布枝が
  存在する場合、`core/lethal.py` は代表HP値（分布の最小値）で `battle.calc_damages` を
  1回だけ呼ぶ設計上、枝ごとの厳密な威力差までは反映されない可能性がある。ただし本タスクの
  対象行は `n/a` のためスコープ外とし、変更は行っていない。
