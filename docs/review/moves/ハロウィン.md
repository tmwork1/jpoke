# ハロウィン レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/ハロウィン.md`）
  - 基本情報（タイプ: ゴースト、分類: 変化、PP20、命中100、範囲: 相手1体、優先度0、直接攻撃×）
  - 効果: 相手のタイプにゴーストタイプを追加する（複合タイプへの追加であり上書きしない）
  - 失敗条件: 相手がすでにゴーストタイプを持つ場合、および相手がテラスタルしている場合
    （ハロウィン自体の一次情報にはテラスタル関連の記述がないが、同一構造の技「もりののろい」の
    一次情報には明記されているため、構造上の整合性からポケモンチャンピオンズでは同様に扱うと明記）
  - タイプ追加の実装方式（`added_types` を用いた volatile 方式）、イベントフロー、
    もりののろいとの相互作用に関する一次情報の記述と簡略化方針を記載

## 実装計画書
- 既存ファイル（`docs/plan/moves/ハロウィン.md`）が実装前の古い設計（`data.types` 直接変更 /
  `ON_CALC_DEF_TYPE_MODIFIER` 方式）のまま放置されていたため、現行実装（`added_types` を用いた
  volatile 方式）に合わせて全面的に書き直した
- テラスタル判定を `ON_BEFORE_APPLY_MOVE` に追加する方針・根拠を明記

## 実装（handlers/ / data/）

  バグがあった（`もりののろい_can_apply` と同一パターンにするための修正）
- その他（`data/moves/move_ha.py`, `data/volatile.py`, `handlers/volatile.py` の
  `ハロウィン_add_type` / `ハロウィン_remove_type`）は仕様通りで変更なし
