# むねんのつるぎ レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/むねんのつるぎ.md`）。
- 一次情報（`docs/wiki/moves/むねんのつるぎ.html`）に基づき、基本情報・判定フラグ・
  ドレイン技としての仕様を記載。
- 特筆点として、他の多くのドレイン技（すいとる・きゅうけつ等）は回復量の端数を
  切り捨てるが、むねんのつるぎは公式Wikiに「小数点以下は切り上げる」と明記されている
  ことを明記した。
- 進捗表（`docs/progress/move.md`）のPP欄が12と誤記されていること（正しくは10、
  実装済みの `pp=10` は正しい）も注記した。

## 実装計画書
- 新規作成（`docs/plan/moves/むねんのつるぎ.md`）。
- 既存のグループ計画書 `docs/plan/moves/HP吸収技.md` は、むねんのつるぎも他のドレイン技と
  同じ共通処理 `_drain_hp`（端数切り捨て）を再利用する想定になっていたが、これは
  一次情報と矛盾するため誤りと判断し、個別の計画書で上書きした
  （`HP吸収技.md` 側にも訂正の注記を追加）。
- priorityは `docs/spec/turn.md` の `Event.ON_HIT` 表「20 | HP吸収技による回復」に基づき
  既存実装（priority=20）のまま変更不要と確認した。

## 実装（handlers/ / data/）
- `handlers/move_attack.py` の `むねんのつるぎ_drain` を修正。
  - 修正前: 共通ヘルパー `_drain_hp(heal_ratio=0.5)` を呼び出しており、
    回復量が `int(damage * 0.5)`（端数切り捨て）で計算されるバグがあった。
  - 修正後: 一次情報どおり `(damage + 1) // 2`（端数切り上げ）で回復量を計算するよう変更。
    `ON_CALC_DRAIN` イベントの発火（おおきなねっこ対応）・`ctx.substitute_damage`
    フォールバック（みがわり対応）は従来どおり維持。
  - `ヘドロえき`（回復→反動ダメージ変換）は `battle.modify_hp(reason="drain")` を
    経由する限り `ON_MODIFY_HEAL` 側で自動適用されるため、追加対応は不要と確認。
- `data/moves/move_ma.py` の `MoveData`（type/category/pp/power/accuracy/flags/handlers）は
  一次情報と一致しており、変更なし。
- `python scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` を実行済み。
