# タネマシンガン レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/タネマシンガン.md`）。従来は `docs/spec/moves/_multi_hit.md`
  の「9.3 2-5回攻撃技」区分のみでカバーされていたが、Championsでの
  PP（`docs/champions/move_list.txt` 準拠で20。本家シリーズは30だが
  `docs/champions/changes_from_sv.md` の全技PP統一ルールにより20になる）を
  明記した個別仕様書を新規作成した（`アイアンテール.md` 等の個別仕様書がある技と
  同様の形式）。
- 弾のわざ（`bullet`フラグ、特性「ぼうだん」で無効化）である点、追加効果なしである点、
  連続攻撃技としての共通仕様は `_multi_hit.md` を参照する旨を明記した。

## 実装計画書
- 変更なし（作成しない）。本技は `handlers={}` で追加効果を持たない純粋な連続攻撃技であり、
  カスタムハンドラの設計が不要なため、個別の計画書は作成しない
  （`つららばり`・`ミサイルばり`・`ロックブラスト` 等、同種の無効果連続攻撃技にも
  個別計画書は存在しないという既存の慣例と一致）。

## 実装（handlers/ / data/）
- 修正した。`src/jpoke/data/moves/move_ta.py` の `タネマシンガン` エントリで
  `pp=30` となっていたが、これは本家シリーズ（SVまで）基準の値の移行漏れバグであり、
  `docs/champions/move_list.txt`（`タネマシンガン	くさ	物理	25	100	20	接×	守○`）に
  従い `pp=20` に修正した。
- その他のフィールド（`type=くさ`, `category=physical`, `power=25`, `accuracy=100`,
  `flags={"bullet"}`, `multi_hit={min:2, max:5, check_hit_each_time:False,
  power_sequence:()}`）は一次情報（ポケモンWiki）・`_multi_hit.md` と一致しており、
  修正不要と判断した。
- `handlers/` にはタネマシンガン専用のハンドラは存在せず、それで問題ない
  （連続攻撃・弾のわざの共通処理は `core/move_executor.py` および
  `handlers/ability.py` のぼうだん判定で汎用的に処理される）。

## リーサル計算
- 対象外。`docs/progress/move.md` のリーサル実装列は `n/a`。

## テスト
- 変更なし。`tests/moves_attack/test_move_ta.py` の
  `test_タネマシンガン_ヒット数が2から5の範囲で決まる`（乱数ロールに応じた2~5ヒット分布の検証）、
  `test_タネマシンガン_相手HP1で最初の1発で処理中断`（1発で相手がひんしになった場合に
  以降のヒットが発生しないことの検証）に加え、
  `tests/abilities/test_ability_sa.py` の `test_スキルリンク_2から5連続技が最大ヒット数になる`、
  `tests/abilities/test_ability_ta.py` の `test_テクニシャン_威力補正`（連続技の威力補正）
  でも本技が使われており、`_multi_hit.md` に記載の仕様（ヒット回数分布・
  スキルリンク・テクニシャン・ひんし判定）を過不足なくカバーしている。
  PP修正（30→20）はテストに影響しない（PP値を直接検証するテストは存在しない）。
