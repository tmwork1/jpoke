- 個別ファイル `docs/spec/moves/ドレインキッス.md` は存在せず、HP吸収技（ドレイン系）は
  `docs/spec/moves/_move_secondary_2.md`（共通仕様表）でまとめて管理されている方針を確認
  （きゅうけつ・デスウイング・むねんのつるぎ・ゆめくい等の既レビュー技も同様に個別ファイルなし）。
- `_move_secondary_2.md` に、ドレインキッスの回復量端数処理（`round_half_up`）について
  一次情報（`docs/wiki/moves/ドレインキッス.html`「技の仕様」節）で確認済みである旨の
  注記を追加した（「与ダメ101→round_half_up(75.75)=76」の具体例付き）。
- `docs/spec/volatiles/かいふくふうじ.md` に、HP吸収技・回復技（`heal`フラグ技）は
  かいふくふうじ状態では選択不可・行動前に付与された場合は失敗する、という一次情報
  （`ドレインキッス.html`／`ギガドレイン.html`／`ドレインパンチ.html`／`じこさいせい.html`／
  `はねやすめ.html` の「技の仕様」節）に基づく仕様を追記した（後述のバグ修正に対応）。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ta.py`: `docs/champions/move_list.txt`（PP=12）と実装（`pp=10`）が
  食い違っていたバグを修正（`pp=10` → `pp=12`）。Gen9本家基準の値が移行漏れとして残っていたもの。
- 同ファイル: `ON_HIT` ハンドラ登録に `docs/spec/turn.md` の priority 根拠コメントを付与
  （他のドレイン技と同様の記法に統一）。
- `handlers/move_attack.py` の `ドレインキッス_drain`（`_drain_hp(heal_ratio=0.75)`）自体は
  一次情報（回復量=与ダメの3/4、端数は四捨五入、最低1回復）と一致しており修正不要だった。
- **新規バグ発見・修正**: 一次情報（`ドレインキッス.html`／`ギガドレイン.html`／
  `ドレインパンチ.html`「技の仕様」節）で「かいふくふうじ状態のポケモンは技を選択できない。
  攻撃する前にかいふくふうじ状態にされた場合は失敗する。」という第六世代以降の仕様が
  明記されているにもかかわらず、実装ではHP吸収技・回復技（`heal`フラグ技）がかいふくふうじ状態でも
  普通に使用・命中してしまう欠落があった。かいふくふうじ揮発状態に `ON_TRY_ACTION` ハンドラ
  （`かいふくふうじ_try_action`、priority=100、既存の ちょうはつ/かなしばり/ふういん と同じ優先度）を
  新規実装し、`heal` フラグを持つ技の実行を失敗させるよう修正した
  （`src/jpoke/handlers/volatile.py` に関数追加、`src/jpoke/data/volatile.py` の
  `かいふくふうじ` エントリに登録）。この修正はドレインキッスだけでなく全HP吸収技・回復技に
  適用される、かいふくふうじ揮発状態レベルの汎用修正である。
