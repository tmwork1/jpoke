# ぼうぎょしれい レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/ぼうぎょしれい.md`）。一次情報（`docs/wiki/moves/ぼうぎょしれい.html`）に
  基づき、タイプ・分類・威力・命中率・PP・範囲・優先度・直接攻撃・効果・判定
  （急所/命中判定/追加効果/まもる/おうじゃのしるし/マジックコート/よこどり/オウムがえし）を記載した。
- 対象範囲について: 一次情報の「範囲」欄は「自分」であり、味方を対象とするダブルバトル専用技ではない
  ため、対象外の判断はせず通常どおり実装対象とした。
- PPについて: `docs/champions/move_list.txt` に「ぼうぎょしれい」自体の項目が存在しないことを確認した。
  一次情報に「ソード・シールドで使用不可」等の除外カテゴリの記載はなく、第9世代で通常に習得・使用
  可能な技であるため、README.md の「対象範囲」定義（第9世代で実装されていない技は実装しない）には
  該当しない。同じくビークイン専用の「こうげきしれい」も move_list.txt 未収録のまま PP15（Gen9本家値）
  で実装・レビュー済み（`docs/progress/move.md` で確認）という既存の前例があるため、これに倣い
  championsの PP統一ルール対象外（現行ロースターに習得者がいない等の理由で一覧に未掲載）と判断し、
  PPはGen9本家の値（10）をそのまま採用する旨を明記した。既存実装（`pp=10`）は変更不要と確認した。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ha.py` の `ぼうぎょしれい` エントリに `target="self"` を追加した
  （バグ修正）。
  - 修正前は `target` 未指定（デフォルト `"foe"`）だったため、`Move.is_blocked_by_protect`
    （`target == "foe"` で判定）が True になり、相手が「まもる」状態のときに自分対象であるにも
    かかわらず能力ランク上昇が阻止されてしまう不具合があった。回帰テストで確認・修正した。
  - `handlers/move_status.py` の `ぼうぎょしれい_modify_attacker_stats`
    （`modify_attacker_stats(battle, ctx, value, stats={"def": 1, "spd": 1})`）は
    `コスモパワー_modify_attacker_stats` と全く同じロジックで、仕様と一致しており変更なし。
