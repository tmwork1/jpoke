# いのちのしずく レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/いのちのしずく.md`）。一次情報（`docs/wiki/moves/いのちのしずく.html`）に
  基づき、以下を明記した:
  - 端数処理は **四捨五入**（`ジャングルヒール`・`みかづきのいのり`の切り捨てとは異なる、一次情報が
    明記する既知の差異）
  - `target="self"` を明示する必要がある根拠（`まもる`・`よびみず`/`ちょすい`/`かんそうはだ`・
    `おうごんのからだ`・マジックコートの各判定が `move.target == "foe"` を前提とするため）
  - PP はチャンピオンズ基準の 12（`docs/champions/move_list.txt` 準拠）

## 実装計画書
- 新規作成（`docs/plan/moves/いのちのしずく.md`）。旧計画書の内容（PP=10、`r=1/4` による切り捨て
  実装、`target` 未設定のまま）は誤りだったため、仕様書と整合する内容に全面的に書き直した。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_a.py`: `いのちのしずく` エントリの `pp` を `10` → `12`（champions基準）
  に修正し、`target="self"` を追加（未設定のままだとデフォルト値 `"foe"` になり、`まもる` や
  `よびみず`/`ちょすい`/`かんそうはだ`/`おうごんのからだ` の `move.target == "foe"` 判定に
  誤って合致し、敵の特性・状態に技をブロックされてしまうバグがあった）。
- `src/jpoke/handlers/move_status.py`: `いのちのしずく_heal` の回復量計算を
  `battle.modify_hp(mon, r=1/4)`（内部で切り捨て）から
  `battle.modify_hp(mon, v=round_half_up(mon.max_hp / 4))`（四捨五入）に修正。
  一次情報に明記された「`ジャングルヒール`/`みかづきのいのり`とは異なる端数処理」を反映した。
  `round_half_up` のインポートを追加。

## リーサル計算
- 対象外（`docs/progress/move.md` のリーサル実装列が `n/a`。使用者自身を回復する技のため
  対戦相手へのリーサル計算とは無関係）。

## テスト
- `tests/moves_status/test_move_a.py`:
  - `test_いのちのしずく_HPが4分の1回復する`: 期待値を `int(max_hp * 1/4)`（切り捨て）から
    四捨五入後の値（ピカチュウの最大HP=110のため `1 + 28`）に修正。
  - `test_いのちのしずく_回復量の端数は四捨五入で切り上げになる` を新規追加。
    110/4=27.5 が四捨五入で28になり、切り捨て(27)とは異なることを検証する
    （`ウッドホーン`等のドレイン技の四捨五入テストと同じパターン）。
  - `test_いのちのしずく_相手のまもるで防がれない` を新規追加。`target="self"` 修正の回帰検出用。
  - `test_いのちのしずく_相手のよびみずに引き寄せられず回復できる` を新規追加。
    `target="self"` 修正により、みずタイプ技だが敵の吸収特性が誤発動しないことを検証する。
