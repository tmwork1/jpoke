# さむいギャグ レビュー結果

## 仕様書
- 新規作成した（`docs/spec/moves/さむいギャグ.md`）。
  - 一次情報（`docs/wiki/moves/さむいギャグ.html`）を基に、タイプ・分類・PP・範囲・優先度・
    効果・判定を記載。
  - PP は一次情報の10ではなく `champions/move_list.txt` 準拠の12を採用（PP列は8/12/16/20の
    4段階統一ルールに従う）。
  - 「先にゆきを発生させ、その後交代する」という順序、既にゆき状態のときの失敗仕様（ゆき変更
    のみ失敗し交代は発動）、交代先もいない場合にのみ完全失敗する仕様、つめたいいわによる
    8ターン延長、交代先の天候上書きなどを明記。

## 実装計画書
- 既存の計画書を実装済みの実体と照合し、以下を修正した。
  - PP を 10 → 12（champions 準拠）に修正。
  - 「現状の実装状況」を「未実装」から「実装済み」に更新。
  - ハンドラのコード例を実装済みコードに合わせて修正（`ctx` の型を `EventContext` から
    `AttackContext` に、交代可否判定を `get_available_switch_commands` から
    `battle.query.can_switch()` に修正）。
  - `data/move.py` への追加という記述を `data/moves/move_sa.py` に修正し、`category="変化"` を
    `category="status"`、PP を 10→12 に修正。
  - `labels=["non_yubi"]` の記述を削除し、ゆびをふる自体が未実装（TODO）であるため現時点では
    対応する `flags` を追加しないことを明記。
  - Interrupt のインポートは既に追加済みである旨に修正。
  - 「交代メカニズム」の注釈を、`pivot()` を流用しない理由（独自の失敗条件を持つため）が
    正確に伝わるように修正。

## 実装（handlers/ / data/）
- `handlers/move_status.py` の `さむいギャグ_activate_weather_and_pivot`、
  `data/moves/move_sa.py` の `"さむいギャグ"` エントリともに仕様・計画と整合しており、
  変更なし。
