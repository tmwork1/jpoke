# けたぐり レビュー結果

## 仕様書
- 変更なし。個別の仕様書ファイルは存在しないが、`docs/spec/moves/威力変動技.md`（8章「体重依存威力技」）に
  けたぐり・くさむすびが共通仕様として記載されており、一次情報（`docs/wiki/moves/けたぐり.html`）の
  威力段階（10kg未満:20 / 10〜25kg未満:40 / 25〜50kg未満:60 / 50〜100kg未満:80 /
  100〜200kg未満:100 / 200kg以上:120、第9世代基準）と完全に一致していることを確認した。
  くさむすびの前例（個別ファイルを作らず共通仕様書で管理）に倣い、新規ファイルは作成しない。

## 実装計画書
- 新規作成。`docs/plan/moves/けたぐり.md` を作成し、既存実装（`けたぐり_calc_power` /
  共通関数 `_weight_to_power`、`くさむすび_calc_power` と共用）が仕様と一致していることを明記した。

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/handlers/move_attack.py` の `けたぐり_calc_power` と共通関数
  `_weight_to_power`、`src/jpoke/data/move.py` の `MoveData`（type=かくとう、category=physical、
  pp=20、power=1（`ON_CALC_POWER_MODIFIER` で実威力を算出）、accuracy=100、flags={"contact"}）を
  一次情報・共通仕様書と照合したが、誤り・欠落は見つからなかった。
  `Pokemon.weight` プロパティがライトメタル/ヘヴィメタル/かるいしによる体重補正を正しく反映しており、
  仕様（第3世代以降の補正順序）とも矛盾しない。

## リーサル計算
- 対象外（`docs/progress/move.md` の該当行は「リーサル実装: n/a」「リーサルテスト: n/a」）。
  理由: 威力は対象（defender）の体重という試合開始後は変動しない確定パラメータにのみ依存し、
  `StateDist`（HP分布）を左右する要素ではないため、`lethal.py` 側の専用ハンドラは不要。
