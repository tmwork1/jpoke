## 実装（handlers/ / data/）
- **重大な仕様漏れを修正**: 前回レビューの実装は `Event.ON_HIT`（ヒット成立後にのみ発火）で
  ランク低下・`たくわえる` リセットを行っていたため、技が外れた場合や、効果がない
  ゴーストタイプ（ノーマル技のため無効）に使用した場合に `ON_HIT` 自体が発火せず、
  一次情報が明記する「外れても効果がないタイプでもランクは低下する」という仕様に反していた。
  `はきだす_apply_after` の発火イベントを `Event.ON_HIT` から `Event.ON_END_MOVE`
  （`run_move()` の `try/finally` 内で必ず1回発火するイベント。`メトロノーム_update_count`
  と同種の「行動不能時を除き必ず発火する」性質を利用）に変更し、
  `battle.move_executor.action_success is False`（麻痺・ひるみ等でそもそも行動していない）
  の場合のみ何もしないよう修正した。これにより、ミス・タイプ相性0倍・まもるブロックの
  いずれの場合でもランク低下・`たくわえる` リセットが発生するようになった。
- **副次的な簡潔化**: `Event.ON_END_MOVE` は技全体につき1回のみ発火するため、
  前回実装で必要だった「おやこあいで2ヒット化されてもランク低下は1回のみ」を保証するための
  `ctx.hit_index != 1` ガードが不要になった（`ON_END_MOVE` がヒット回数に関わらず1回しか
  発火しないため、自然に「1回のみ」の仕様を満たす）。このガードを削除した。
- `src/jpoke/data/moves/move_ha.py` の `はきだす` エントリの `Event.ON_HIT` を
  `Event.ON_END_MOVE` に変更した。
- `scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` /
  `scripts/sort_data/sort_moves.py` を実行した。
