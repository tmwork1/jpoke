# ひっさつまえば レビュー結果

## 一次情報確認の結果（重要）

`docs/wiki/moves/ひっさつまえば.html` を確認したところ、本技はカテゴリ
「スカーレット・バイオレットで使用不可能なわざ」に分類されており、本文中の技データ表にも
赤字で「ソード・シールドで使用不可」「BDSPで使用不可」「SVで使用不可」と明記されている。

README.md の「対象範囲」定義（「第9世代で実装されていない技・アイテム・特性・ポケモンなどは実装しない」）
に該当するため、**本技は実装対象外と判断した**。以降の項目はこの判断に基づく対応内容を記載する。

## 仕様書
- 新規作成（`docs/spec/moves/ひっさつまえば.md`）。
- 対象範囲外である旨と根拠（ソード・シールド以降で使用不可）を明記した。
- 参考として第1〜7世代時点の技データ（タイプ:ノーマル、物理、威力80、命中90、PP15、追加効果10%ひるみ、
  かみつき技）を記録した。

## 実装計画書
- 新規作成（`docs/plan/moves/ひっさつまえば.md`）。
- 「実装しない」という結論と、過去の一括実装（`docs/plan/moves/攻撃技_ひるみ系.md`）で
  誤って登録されていた経緯を記録した。
- `docs/plan/moves/攻撃技_ひるみ系.md` にも追記セクションを追加し、当該箇所（威力120・PP5等）が
  誤った過去の登録内容であり削除済みであることを明記した。

## 実装（handlers/ / data/）
- 誤って登録されていた実装を削除した。
  - `src/jpoke/data/moves/move_ha.py`: `"ひっさつまえば": MoveData(...)` エントリを削除
    （威力120・PP5という誤った値で登録されていた。正しい基礎威力は80、PPは15であり、
    がんじょうあご適用後の値120がそのまま `MoveData.power` に登録されてしまっていた）
  - `src/jpoke/handlers/move_attack.py`: `ひっさつまえば_apply_flinch` ハンドラを削除
  - `python scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` を実行
  - `python scripts/sort_data/sort_moves.py` を実行
  - `python scripts/generate_literals/generate_move_literal.py` を実行し、`MoveName` Literal 型を
    MOVES辞書（733件）と整合させた
  - `src/jpoke/handlers/item.py` の `_INNATE_FLINCH_MOVES`（おうじゃのしるし・するどいキバの
    重複判定用リスト）から `"ひっさつまえば"` を削除（実体が存在しない技名の混入を解消）
