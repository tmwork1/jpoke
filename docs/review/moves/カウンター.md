## 実装（handlers/ / data/）
- `src/jpoke/core/move_executor.py`: `Pokemon.last_physical_damage_received` /
  `last_special_damage_received` の更新処理を `+=`（ターン中の合計）から `=`（直近1ヒット分で上書き）に修正した。
  一次情報に「連続技を受けた場合は最後の1回分のダメージのみを2倍にする」という明記があり、従来の実装では
  連続技（例: ダブルアタック）を受けた場合に合計ダメージが2倍返しされてしまうバグがあったため。
  この修正は共通コードのため `ミラーコート`（特殊版のカウンター）にも同様に適用される。
- `src/jpoke/model/pokemon.py`: 上記修正に合わせて `last_physical_damage_received` /
  `last_special_damage_received` のコメントを実態（直近1ヒット分）に合わせて修正した。
- `src/jpoke/data/moves/move_ka.py`: カウンターの `flags` に `non_copycat` を追加した。一次情報に
  「オウムがえし・さきどり・まねっこでコピーできない(第四世代以降)」との記載があり、本プロジェクトで実装済みの
  `まねっこ` がカウンターをコピー対象として選んでしまう欠落があったため（`きょじゅうだん`等の既存パターンに準拠）。
- `ha.カウンター_can_use` / `ha.カウンター_modify_damage` のロジック自体（priority=30固定、失敗判定、
  ダメージ倍返し計算）は仕様と一致しており修正なし。
