# テレポート レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/テレポート.md`）。
- `docs/wiki/moves/テレポート.html`（「技の仕様」節）を精査した結果、第7世代（USUM）までは
  トレーナー戦では効果が無かったが、**ピカブイ・第8世代以降はトレーナー戦でも優先度-6の
  交代技として機能する**ことを確認した（本プロジェクトが対象とする第9世代準拠）。
- とらわれ状態（バインド・にげられない・フェアリーロック・かげふみ・ありじごく・じりょく）を
  トレーナー戦では無視して確実に交代できる点を明記した。
- 関連する既存の汎用仕様書 `docs/spec/moves/_pivot.md`（`交代技.md` と内容同一の重複ファイル）に
  「テレポートは第9世代トレーナーバトルでは交代効果なし（仕様対象外）」という**誤った注記**が
  あったため、上記の正しい挙動に修正し、対象技一覧にテレポートの行を追加、
  技別固有仕様にテレポートの節を追加した（`_pivot.md` と `交代技.md` の両方に同一の修正を適用）。

## 実装計画書
- 新規作成（`docs/plan/moves/テレポート.md`）。
- `docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` 節に「30 | テレポート: トレーナー戦・交代不可」と
  既に明記されていたため、この priority をそのまま失敗判定チェックに採用した。
- 実際の交代処理（`ON_STATUS_HIT`）は、同じ「とらわれ状態を無視する交代技」である
  バトンタッチ（`バトンタッチ_check` / `バトンタッチ_apply`）の実装パターンを踏襲する方針とした。
  ただしテレポートは能力ランク・揮発性状態の引き継ぎを行わないため、その部分は実装しない。

## 実装（handlers/ / data/）
- 既存の `data/moves/move_ta.py` の `"テレポート"` エントリは `handlers={}`（交代効果なし）
  かつ `target` 未設定（デフォルトの `"foe"` のまま）という不完全な状態だったため、以下を修正した。
  - `target="self"` を追加（まもる・マジックコートへの誤反応を防止）。
  - `Event.ON_TRY_MOVE_1`（priority=30）に `テレポート_check` を登録（控えに生存個体がいない場合は失敗）。
  - `Event.ON_STATUS_HIT`（デフォルト priority=100）に `テレポート_apply` を登録
    （とらわれ状態を無視して `Interrupt.PIVOT` を設定し交代する）。
- `handlers/move_status.py` に `テレポート_check` / `テレポート_apply` を新規実装した
  （バトンタッチと同じロジックだが、ランク・揮発性状態の保存処理は行わない）。
- PP（20）は `docs/champions/move_list.txt` にテレポートのエントリが存在しなかったため、
  原作全世代共通の PP20（かつ本プロジェクトの4段階PP制 8/12/16/20 にも適合する値）を維持した。
