## 実装（handlers/ / data/）
- **`data/moves/move_sa.py` のサイコノイズに追加効果ハンドラが一切登録されておらず、命中してもかいふくふうじ状態が
  付与されない未実装バグがあった**（`handlers={}` のまま放置されていた）。
  - `Event.ON_DAMAGE_HIT` に新規ハンドラ `サイコノイズ_apply_volatile_to_defender`
    （`handlers/move_attack.py` に実装、`apply_volatile_to_defender(..., volatile="かいふくふうじ", count=2)`）を登録。
- **`flags` に `secondary_effect` が欠落していた**: ちからずく特性の威力1.3倍・追加効果無効化が機能していなかった。
  `flags={"sound"}` → `flags={"sound", "secondary_effect"}` に修正。
- **PP が Gen9本家基準の10のままだった**（`docs/champions/move_list.txt` の正しい値は12）。
  `pp=10` → `pp=12` に修正。
- かいふくふうじ揮発性状態自体（回復無効化・ターン管理）は既に `data/volatile.py` / `handlers/volatile.py` に
  実装済みで、変更不要だった。
- りんぷん・おんみつマント（追加効果無効化）・アロマベール（かいふくふうじ付与阻止）・音技のみがわり貫通は、
  いずれも既存の汎用処理（`resolve_secondary_chance` / `sound` フラグ / `アロマベール_prevent_volatile`）で
  自動的に対応済みのため、個別のハンドラ追加は不要だった。

## リーサル計算
- 新規実装。`docs/progress/move.md` のリーサル実装列が「-」（要実装）だった。
  - `handlers/lethal.py` に `サイコノイズ_apply_volatile`（`LethalEvent.ON_HIT`）を追加し、
    `data/moves/move_sa.py` の `lethal_handlers` に登録。バインド（`_apply_bind`）・しおづけ
    （`しおづけ_apply_volatile`）と同じパターンで `ctx.defender.volatiles` へ直接 `Volatile` を設定する。
  - かいふくふうじの本質的な効果（HP回復の無効化）を反映するため、既存の回復系リーサルハンドラが共通で使う
    `_heal` / `_heal_at_pinch`（`handlers/lethal.py`）の先頭に、`target.volatiles` にかいふくふうじが
    含まれる場合は回復せずそのまま返すガードを追加。これにより、たべのこし・オボンのみ・グラスフィールド・
    ポイズンヒール等、既存の全回復系リーサルハンドラに個別の書き換えなしで一括反映されるようにした。
