# かえんのまもり レビュー結果

## 仕様書
- `docs/spec/volatiles/かえんのまもり.md` が既に存在していたため、これを更新した
  （`docs/spec/moves/かえんのまもり.md` は新規作成しなかった。同系統の `まもる` /
  `キングシールド` / `トーチカ` / `スレッドトラップ` も専用の仕様書は `docs/spec/volatiles/`
  にのみ存在し `docs/spec/moves/` には存在しない運用に統一されているため、それに合わせた）。
- 一次情報（`docs/wiki/moves/かえんのまもり.html`）と照合し、以下を追記・修正した。
  - 「そのターンの最後に行動する場合は失敗する」という、かえんのまもり固有の失敗条件が
    仕様書に一切記載されていなかったため追記した（実装にも欠落していたため §実装 で対応）。
  - 「フェイント/ゴーストダイブ/シャドーダイブ/いじげんホール/いじげんラッシュによる解除」の
    節を新設し、現状の実装がフェイント/いじげんホール/いじげんラッシュのみ明示的に
    volatile を解除し、ゴーストダイブ/シャドーダイブは `unprotectable` フラグのみで
    守りを無視する（volatile 自体は解除しない）簡略実装である旨を明記した。
  - 「連続使用による失敗率上昇」の節に、本実装が確率的な逓減（1/3ずつ）ではなく
    「直前に protect 技が成功していれば必ず失敗する」という簡略化された決定的な実装
    （`まもる系_連続使用失敗チェック`）であることを明記した（`まもる.md` にある同様の
    注記と整合させた）。
  - 「連続使用により失敗」チェックの実際の実装イベントが `docs/spec/turn.md` に記載の
    `ON_TRY_MOVE_1` priority=30 ではなく、実装上は `ON_TRY_MOVE_2`（デフォルト
    priority=100）であることを明記した（`まもる` 系全体で共通の既存の齟齬であり、
    本レビューでは是正せず現状を正しく記載するに留めた）。
  - 「防御が無視されるケース」の見出しから誤解を招く「（今世代スコープ外）」を削除した
    （実際にはふかしのこぶし・ハイパードリル・パワフルエッジの貫通は実装済みのため）。
  - 出典を `docs/wiki/moves/かえんのまもり.html` に修正した（技の説明とかえんのまもり状態が
    同一ページに記載されているため両方のパスを記載）。

## 実装（handlers/ / data/）
- 一次情報で明記されている「そのターンの最後にかえんのまもりを使った場合は失敗する」
  という固有の失敗条件が実装に欠落していたため、以下を追加した。
  - `src/jpoke/handlers/move_status.py`: `かえんのまもり_check` を新規実装。
    `battle.query.is_second_actor()`（既存の `しっぺがえし_double_power_when_second` 等で
    使われている1vs1想定のユーティリティ）を用いて、自分がそのターンの後攻であれば
    技を失敗させ、`executed_move` を `None` にリセットする（連続使用扱いにしないため）。
  - `src/jpoke/data/moves/move_ka.py`: `"かえんのまもり"` の `Event.ON_TRY_MOVE_2` を
    リスト化し、既存の `まもる系_連続使用失敗チェック` に加えて新規の `かえんのまもり_check`
    を登録した。
- それ以外の既存実装（`かえんのまもり_apply` による volatile 付与、
  `data/volatile.py` の `かえんのまもり_protect` / `かえんのまもり_remove_volatile` による
  攻撃技無効化・やけど付与・ターン終了時解除）は一次情報・仕様書と整合しており修正不要と判断した。
  - 接触判定（`is_contact_reaction` によるぼうごパット対応）、変化技を防がない
    （`protect_non_attack=False`）、ふかしのこぶし・ハイパードリル・パワフルエッジによる
    貫通（`ON_CHECK_PROTECT` / `unprotectable` フラグ）はいずれも実装済みで一致を確認した。

## テスト
- `tests/moves_status/test_move_ka.py` に以下を追加した。
  - `test_かえんのまもり_そのターン最後の行動だと失敗する`:
    優先度が同じでも自分が後攻ならかえんのまもりが失敗し、`executed_move` が
    `None` にリセットされることを確認する。
  - `test_かえんのまもり_最後の行動で失敗しても次ターンは連続使用扱いにならない`:
    最後の行動で失敗した次のターンに自分が先に行動できれば、連続使用扱いにならず
    成功する（直接攻撃を防いでやけどを付与できる）ことを確認する。
- 攻撃技の無効化・変化技を防がない・やけど付与・ぼうごパット対応・ターン終了解除といった
  既存の挙動は `tests/volatiles/test_volatile_ka.py`・`tests/volatiles/test_volatile_hamayarawa.py`
  （`まもる系`パラメータ化テスト）で既に十分にカバーされているため重複するテストは追加しなかった。
- `python -m pytest tests/ -v` を実行し、4127 passed, 1 skipped で全件成功を確認した
  （ログ: `.loop/test_logs/かえんのまもり.log`）。
  - 実行中、本レビューと無関係な `tests/moves_attack/test_move_ma.py::test_むしくい_ねんちゃく持ちをひんしにさせた場合は奪える`
    や `tests/moves_attack/test_move_ta.py::test_DDラリアット_相手のぼうぎょランク低下も無視する`
    が乱数シード未固定（`Battle.__init__` の `seed` 未指定時は `time.time()` 由来）により
    低確率で失敗することを確認したが、いずれも本レビューの変更とは無関係な既存の flaky
    テストのため今回は修正対象としなかった（`git stash` して再現することを確認済み）。
