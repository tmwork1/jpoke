# トリック レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/トリック.md`）。
  - `docs/wiki/moves/トリック.html` の「技の仕様」節（第五世代以降）と、既存の `docs/spec/moves/すりかえ.md`
    （トリックはタイプを除きすりかえと全く同じ効果であるとWiki備考節に明記）を基に作成した。
  - 基本情報（タイプ: エスパー、分類: 変化、命中100、PP12、対象: 相手1体、優先度0、接触なし）、
    失敗条件（両者ともアイテム未所持／みがわり／ねんちゃく／専用道具の組み合わせ）、
    他効果との関係（ぶきよう・さしおさえ・マジックルームでも成功、こだわり系アイテムを入手しても
    ロックされない、ゆびをふる・ねこのてで選ばれない、まねっこでコピー不可）、
    判定フラグ（マジックコート反射不可等）を記載した。

## 実装計画書
- 新規作成（`docs/plan/moves/トリック.md`）。
  - `docs/plan/moves/すりかえ.md` を参考に、既存実装（`すりかえ_swap_items` /
    `すりかえ_release_choice_lock` の共有ハンドラ構成）をそのまま計画書として整理した。
  - `docs/spec/turn.md` に `ON_STATUS_HIT` / `ON_MOVE_END` の相対順位が掲載されていないため、
    すりかえの既存実装（`ON_STATUS_HIT` はデフォルト優先度100、`ON_MOVE_END` は優先度110）を
    根拠として採用したことを明記した。

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/data/moves/move_ta.py` の `"トリック"` エントリは、すりかえと同一の
  `すりかえ_swap_items`（`ON_STATUS_HIT`）・`すりかえ_release_choice_lock`（`ON_MOVE_END`,
  priority=110）ハンドラを使用しており、仕様書・計画書の内容と整合していることを確認した。
  `flags={"unreflectable", "non_copycat"}` によりマジックコート反射不可・まねっこコピー不可の
  両仕様にも対応済み。PP=12 は `src/jpoke/data/ps-champ-ja/moves.json` の値と一致している。
