# ダイビング レビュー結果

## 仕様書
- `docs/spec/moves/ダイビング.md` を一次情報（`docs/wiki/moves/ダイビング.html`）と照合し、生のWiki貼り付け形式から
  プロジェクト標準の簡潔な形式（基本情報／効果／判定フラグ／チャンピオンズ向け簡略化／参照）に全面改訂した。
  - PPは vanilla=10、本実装（チャンピオンズ）=12 である旨を明記（`docs/champions/move_list.txt` 準拠）。
  - なみのり・うずしお以外に、一次情報ではロックオン状態／ノーガード特性／どくどく（第六世代以降）でも
    命中するとされるが、本実装では未実装であることを明記（実装調査の結果を反映）。
  - 通常の姿のウッウが1ターン目の溜め成功だけで特性「うのミサイル」が発動する一次情報を追記
    （うのミサイル自体の実装可否は当該特性のレビュー対象として明記し、本レビューでは変更しない）。
  - `docs/spec/volatiles/ダイビング.md` の「回避」節・「本実装での対象外」節を、実装コード
    （`handlers/volatile.py` の `can_hit_hidden_target` / `HIDDEN_MOVE_ALLOWED_MOVES`）と突き合わせて修正。
    ロックオン状態・ノーガード特性による命中を「実装済み」と誤って記載していた箇所を、実際には
    `ON_TRY_MOVE_1`（`can_hit_hidden_target`）が技名一致のみで判定しており救済されないことが判明したため、
    未実装（`あなをほる`／`そらをとぶ`と共通の既知の未実装事項）として訂正した。どくどくによる命中も
    未実装として追記した。

## 実装計画書
- `docs/plan/moves/ダイビング.md` の「仕様要約」でPPが10と誤記されていたため12（チャンピオンズ仕様）に修正し、
  vanilla基準ではない旨の注記を追加。
- 「現状の実装状況」表の実装場所が古い `src/jpoke/data/move.py` を指していたため、実際の実装ファイル

  （正しくは `category="physical"` `flags={"contact"}`）になっていたため修正し、PPも12に修正。
- 「注意点・エッジケース」のロックオン・ノーガードに関する記述を、実装調査の結果（未実装）に基づいて訂正。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ta.py` の `"ダイビング"` エントリで `pp=10`（Gen9本家基準の移行漏れ）に
  なっていたバグを `pp=12`（`docs/champions/move_list.txt` 準拠）に修正した。
- それ以外（威力80・命中100・分類物理・タイプみず・`contact`フラグ、`ON_MOVE_CHARGE`ハンドラでの
  `charge_into_volatile` 呼び出し、揮発状態「ダイビング」側の `ON_CHECK_TRAPPED` / `ON_MODIFY_COMMAND_OPTIONS` /
  `ON_CALC_POWER_MODIFIER`（なみのり・うずしお威力2倍） / `ON_TRY_MOVE_1`（隠れる系共通の回避判定、priority=50） /
  `ON_HIT`（揮発状態解除））は仕様書・一次情報と一致しており、`あなをほる` / `そらをとぶ` と同一設計であることを
  確認した。修正不要。
- ロックオン状態・ノーガード特性・どくどくによる命中救済の未実装は `あなをほる` / `そらをとぶ` と共通の
  横断的な既知事項であり、`handlers/volatile.py` の共有関数 `can_hit_hidden_target` を変更すると
  他の2技のレビュー・テストにも影響するため、本レビュー（ダイビング単体）の範囲では実装を見送り、
  仕様書側に正確な未実装事項として記録するに留めた。
