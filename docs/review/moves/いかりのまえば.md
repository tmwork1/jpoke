## 実装（handlers/ / data/）
- `src/jpoke/data/move.py` の `いかりのまえば` エントリを修正:
  - `power=0` を追加（他の固定ダメージ技: カタストロフィ・ちきゅうなげ・ナイトヘッドと表記を統一。
    従来 `power` 未指定=`None` だったが、`damage.py` の `if not move.power` 判定により実質的な挙動差はない）。
  - `ON_MODIFY_MOVE_DAMAGE` ハンドラに `priority=15` を追加（**挙動修正**）。
    従来デフォルトの `priority=100` のままだったため、`みがわり`(priority=20)・
    `こらえる`(priority=95) より後に実行され、以下の誤動作が発生していた:
    - みがわり状態の相手に使用すると、みがわりのHPを消費せずに本体の実HPへ
      そのままダメージが適用されてしまう（みがわりが実質貫通する）。
    - こらえる状態・残りHP1で使用すると、こらえるの瀕死防止が機能せず瀕死になる。
    `priority=15` として `半減ダメージの確定` を `みがわり`・`こらえる` より前に
    実行させることで、本体HP基準の正しいダメージがまず確定し、
    みがわりはそのダメージをみがわりのHPを上限として肩代わりし、
    こらえるは瀕死を正しく防ぐようになった。
  - `handlers/move_attack.py` の `half_damage` 関数自体は修正不要（既存実装のまま正しい）。
