# ドラゴンテール レビュー結果

## 仕様書

- 個別の `docs/spec/moves/ドラゴンテール.md` は作成しなかった。ドラゴンテールは
  `docs/spec/moves/_blow.md`（および `docs/spec/moves/強制交代技.md`）でほえる・
  ふきとばし・ともえなげと共通仕様としてカバーされており、既存の進捗表の慣例
  （`あばれる` 等、共有仕様書でカバーされる技は個別ファイルを作らず仕様書列を `-`
  のままにする。`ともえなげ` レビュー時にも同じ判断がされている）に合わせた。
- `docs/wiki/moves/ドラゴンテール.html` と、参照先である
  `docs/wiki/moves/ともえなげ.html` の「技の仕様」節と照合し、
  `docs/spec/moves/強制交代技.md` の誤りを修正:
  - 「交代先は次に控えているポケモン（ドラゴンテール・ともえなげ）」という記載は誤り。
    一次資料には「交代で出てくるポケモンは対象の控えからランダムで選ばれる」と明記
    されており（ドラゴンテール側のページも「ともえなげ#技の仕様を参照」としている）、
    ほえる・ふきとばしと同様に**ランダム選択**が正しい仕様である。
    `docs/spec/moves/_blow.md` は既に正しい記載（ランダム選択）になっていたため、
    `強制交代技.md` 側を `_blow.md` に合わせて修正した。

## 実装計画書

- 個別の `docs/plan/moves/ドラゴンテール.md` は作成せず、既存の共有計画書
  `docs/plan/moves/攻撃技_強制交代系.md`（ともえなげ・ドラゴンテールを対象）を
  一次資料・現行実装と突き合わせて修正した。
- 修正内容:
  - サンプルコード中の `battle.get_available_switch_commands(player)` を、実際の
    最終実装である `get_forced_switch_commands(battle, player)`（`handlers/move.py`）
    に修正。あわせて「とらわれ状態・交代制限特性の無視」節を新設し、
    にげられない/バインド/フェアリーロックや特性かげふみ/ありじごく/じりょくを
    無視して発動する仕様と、それを実現するための専用ヘルパーの役割を明記した。
  - 「きゅうばん・ねをはる無効化」節を、ねをはるが実装済み（未実装ではない）である
    現状に合わせて更新。
  - 「控えが存在しない場合」節の関数名参照を `get_available_switch_commands` から
    `get_forced_switch_commands` に修正。
  - サンプルコード・PP値の誤り（後述）を `pp=10` → `pp=12` に修正。

## 実装（handlers/ / data/）

一次資料と突き合わせた結果、以下のPP誤りを発見し修正した。実装ロジック自体
（強制交代・ランダム選択・ON_TRY_BLOW によるきゅうばん/ばんけん/ねをはる無効化・
とらわれ状態を無視する交代先取得）は仕様どおりで問題なかった。

1. **PP誤り（実装バグ、`src/jpoke/data/ps-champ-ja/moves.json` 準拠に修正）**
   - `data/moves/move_ta.py` の `ドラゴンテール`・`ともえなげ` はいずれも `pp=10`
     になっていたが、`src/jpoke/data/ps-champ-ja/moves.json` では両技とも PP=12 と明記されて
     おり、`docs/champions/changes_from_sv.md` の「全技PPは8/12/16/20の4段階に統一」
     という仕様と矛盾していた。これは移行漏れのバグと判断し、両エントリを `pp=12`
     に修正した。
   - `docs/progress/move.md` の `ドラゴンテール ` `ともえなげ` 両行のPP列も `10` →
     `12` に修正した（`ともえなげ` の前回レビュー時に誤って進捗表側を実装値
     `10` に合わせてしまっていたため、あわせて是正した）。
   - `ドラゴンテール` 行の「レビュー」列を `-` → `✓` に更新した。

その他、強制交代のロジック（`handlers/move_attack.py` の `_force_switch_random`、
`handlers/move.py` の `get_forced_switch_commands`、`きゅうばん`/`ばんけん`/
`ねをはる` の `ON_TRY_BLOW` ハンドラ）は一次資料の記載（優先度-6、控えからランダム
選択、まもる等で防御可能、きゅうばん等で無効化、とらわれ状態を無視して発動）と
一致しており、修正は行っていない。

## リーサル計算

- 対象外（`docs/progress/move.md` の「リーサル実装」「リーサルテスト」列がいずれも
  `n/a`）。強制交代の追加効果はダメージ量そのものに影響せず、通常のダメージ計算で
  完結するため専用のリーサルハンドラは不要。
  なお `tests/test_lethal.py` では既にドラゴンテールが汎用の物理ドラゴン技サンプル
  として多数のリーサル計算テストに使われているが、これらは一般的なダメージ計算の
  検証であり、ドラゴンテール固有のリーサルロジックではない。
