  - 状態変化としての防御・貫通仕様は既存の `docs/spec/volatiles/みがわり.md` に委譲する形に整理した

## 実装（handlers/ / data/）
既存実装には重大な不具合が複数あったため、以下を修正した。

- **`target="self"` の欠落（重大）**: `src/jpoke/data/moves/move_ma.py` の `みがわり` エントリに
  `target` が未指定で、`MoveData.target` のデフォルト `"foe"` が適用されていた。これにより
  `Move.is_reflectable`（`category=="status" and target in ("foe","foe_side")`）が誤って `True` になり
  マジックコート／マジックミラーで跳ね返る扱いになっていたほか、`Move.is_blocked_by_protect`
  （`target=="foe"`）も誤って `True` になり、相手が「まもる」状態のときにみがわりが失敗する不具合が
  あった。`target="self"` を追加して修正。
- **PPの誤り**: `pp=10`（本家SV基準）になっていたが `docs/champions/move_list.txt` の実値 `pp=12` に
  修正した。
- **HP消費が発生しない不具合（重大）**: `handlers/move_status.py` の `みがわり_apply` が
  `battle.volatile_manager.apply` のみを呼び、`battle.modify_hp` を一切呼んでいなかったため、
  使用者のHPが全く減らずにみがわりが生成されていた。`battle.modify_hp(mon, -migawari_hp)` を追加した。
- **バインド解除の欠落**: 技成功時に自分のバインド状態を解除する仕様（第三世代以降、
  `docs/spec/volatiles/みがわり.md` に記載済みだったが未実装）が抜けていたため、
  `battle.volatile_manager.remove(mon, "バインド")` を追加した。
- `みがわり_check`（失敗条件判定）は仕様と一致しており変更なし。
- `data/volatile.py` の `みがわり` エントリ（防御・貫通ハンドラ）は本レビューの対象範囲だが確認の結果
  仕様どおりで変更なし。
