## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_na.py` の「なりきり」エントリを修正:
  - `pp=10` → `pp=12`（`src/jpoke/data/ps-champ-ja/moves.json` 準拠。Gen9本家基準の
    値が移行漏れとして残っていた）
  - `accuracy=100` → `accuracy=None`（必中技。move_list.txt でも命中欄は
    「-」で必中を表す）
  - `flags` が未設定だったため `{"unprotectable", "unreflectable",
    "bypass_substitute"}` を追加（まもる・みがわりを貫通し、マジックコート・
    マジックミラーで反射されない、という一次情報の記載を反映）
- `src/jpoke/handlers/move_status.py` の `なりきり_can_apply` を修正:
  - 対象の特性が `uncopyable` フラグを持つ場合の判定は既存のまま維持
  - 対象の特性が「ふしぎなまもり」の場合を個別に判定して失敗させる処理を追加
    （ふしぎなまもりは `uncopyable` フラグを持たず、トレース・なかまづくり
    では正常にコピーできるが、なりきりに対してのみ無効という一次情報上の
    例外のため、フラグとは別に名前判定が必要）
  - 使用者自身の特性が `protected` フラグを持つ場合（上書きできない特性）に
    失敗させる判定を追加（欠落していた）
  - 使用者と対象の特性が既に同じ場合に失敗させる判定を追加
    （第五世代以降の仕様。欠落していた）
- `src/jpoke/data/ability.py` の「うのミサイル」エントリを修正:
  - `uncopyable` フラグと `protected` フラグを除去し、`gas_proof` のみ残した
  - `docs/spec/abilities/うのミサイル.md`／`docs/spec/abilities/トレース.md`／
    `docs/spec/abilities/レシーバー.md` によれば、SV Ver.3.0.0以降（本
    プロジェクトが対象とする最新版）ではうのミサイルはコピー・スキルスワップ
    ともに可能になっているため、両フラグとも誤りだった
