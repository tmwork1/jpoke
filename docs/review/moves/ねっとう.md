# ねっとう レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/ねっとう.md`）。
- 一次情報（`docs/wiki/moves/ねっとう.html`）と `docs/spec/abilities/ちからずく.md` /
  `docs/spec/abilities/りんぷん.md` を突き合わせ、以下を明記した。
  - 基本情報（みず/特殊/威力80/命中100/PP15/相手1体単体選択/優先度0/非接触）。
  - 追加効果: 30%でやけど付与。
  - 使用者が『こおり』状態のとき、通常の25%解凍抽選とは独立して無条件で解凍してから攻撃する
    （かえんぐるま・ねっさのだいち等と同種の効果）。
  - 第六世代以降、命中した相手の『こおり』状態も治す。ただし使用者の特性が『ちからずく』の
    場合はこの効果が発動しない（`docs/spec/abilities/ちからずく.md` 13行目・39行目）。
    りんぷんはこの効果を防がない。

## 実装（handlers/ / data/）
- **バグ修正1**: `src/jpoke/handlers/move_attack.py` に `ねっとう_thaw_attacker` を新規追加し、
  `src/jpoke/data/move.py` の `"ねっとう"` エントリに `Event.ON_TRY_ACTION`
  （`ha.ねっとう_thaw_attacker`, priority=5）を登録した。
  - 修正前は、使用者がこおり状態のときにねっとうを選択しても通常の25%解凍抽選のみが適用され、
    抽選に外れると技が発動できなかった（仕様上は無条件解凍が必要）。
- **バグ修正2**: `src/jpoke/handlers/ailment.py` の `こおり_cure_by_thaw_move` を修正し、
  `"thaw"` フラグを持つ技のうちタイプが『ほのお』でないもの（シャカシャカほう・
  スチームバースト・ねっさのだいち・ねっとう・ハイドロスチーム）を使用者がちからずくで
  使用した場合、被弾側のこおり解除効果が発動しないようにした。ほのおタイプの技（タイプ由来の
  効果）は従来通りちからずくの影響を受けない。
  - 修正前は技のタイプに関わらず無条件で解除しており、ちからずく使用時にねっとうで相手の
    こおりが治ってしまう仕様外の挙動になっていた。
  - この修正は同じ `"thaw"` フラグ共通ハンドラを使う他の技（ねっさのだいち等）にも影響するが、
    いずれも仕様に沿う方向の修正であり退行はない（既存テストは全件パス）。
- `python scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` /
  `python scripts/sort_handlers.py src/jpoke/handlers/ailment.py` /
  `python scripts/sort_data/sort_moves.py` を実行済み。

## 補足（今回のレビュー対象外だが関連する既知の課題）
- シャカシャカほう・スチームバースト・ねっさのだいち・ハイドロスチームも `"thaw"` フラグにより
  被弾側のこおり解除は実装済みだが、ねっとうと同様に使用者自身の無条件解凍ハンドラ
  （`_thaw_attacker`）が未実装のものがある（ねっさのだいちは実装済み、シャカシャカほう・
  スチームバースト・ハイドロスチームは未実装）。別途のレビュー対象とする。
- ねっぷう（ほのおタイプ）は `"thaw"` フラグが付与されておらず、被弾した相手のこおりを
  治す効果が未実装の可能性がある。別途のレビュー対象とする。
