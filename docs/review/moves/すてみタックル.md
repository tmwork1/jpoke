# すてみタックル レビュー結果

## 仕様書
- 個別の仕様書ファイルは存在しないが、一次情報（`docs/wiki/moves/すてみタックル.html`）と照合した結果、
  効果内容（相手に与えたダメージの1/3を自分も受ける反動技）は既に
  `docs/spec/moves/_move_secondary_2.md`（18行目: `すてみタックル 0 0.33 0 0`）に記載済みで、内容は正しい。
  同種の反動技グループ（アフロブレイク・じごくぐるま・ウェーブタックル・ウッドハンマー・とっしん・
  はめつのひかり・フレアドライブ・ブレイブバード・ボルテッカー・もろはのずつき・ワイルドボルト）も
  同様に個別仕様書を持たず共通表で管理する既存方針（じごくぐるまレビュー時に確立）のため、
  個別ファイルの新規作成は不要と判断した。
- 一次情報を精査した結果、`_move_secondary_2.md` に記載されていなかった重要な仕様を発見したため追記した。
  ポケモンWikiの「技の仕様」節に、反動ダメージの端数処理は「第四世代までは切り捨て、第五世代以降では
  四捨五入する」と明記されている。これは本プロジェクトの `round_half_down`（五捨五超入）に相当する
  丸め規則であり、`_move_secondary_2.md` に「端数処理」節を新設して明記した
  （`_drain_hp` 側は比率0.75の技（デスウイング・ドレインキッス）で乖離がありうる旨も併記し、
  本レビューの対象外として明示した）。

## 実装計画書
- 個別の計画書ファイルは存在しないが、`docs/plan/moves/反動技_033.md` に
  すてみタックルを含む反動量1/3グループ（ウェーブタックル・ウッドハンマー・すてみタックル・
  フレアドライブ・ブレイブバード）の実装計画が記載されている。
- 計画書に記載されていた反動量の計算式 `max(1, int(damage / 3))` は、実際には
  「一次情報のレビュー」節で発見した端数処理仕様（`round_half_down`）と食い違っていたため誤りと判断し、
  `max(1, round_half_down(damage / 3))` に修正した。あわせてコード例（`ウェーブタックル_recoil` 等）も
  `round_half_down` を使う形に更新し、実装修正の経緯を記した節を追加した。
- 同じ `_recoil` ヘルパーを使う `反動技_025.md`（1/4比率グループ）・`反動技_05.md`（1/2比率グループ）にも
  同様の計算式記載があったため、実装（共通ヘルパー）の修正に合わせて式と補足説明を更新した
  （1/2比率グループは端数が必ず0か0.5のため実質的な挙動差はないが、記載の正確性のため修正した）。

## 実装（handlers/ / data/）
- `src/jpoke/data/move.py` の `すてみタックル` エントリ（タイプ: ノーマル、分類: 物理、威力: 120、
  命中率: 100、PP: 15、flags: `contact`, `recoil`）は一次情報と一致しており変更なし。
- `src/jpoke/handlers/move_attack.py` の共通ヘルパー `_recoil`（`すてみタックル_recoil` が呼び出す関数）で
  **反動ダメージの端数処理の誤りを発見**したため修正した。
  - 修正前: `recoil = max(1, int(value * ratio))` — 単純な切り捨て。
  - 修正後: `recoil = max(1, round_half_down(value * ratio))` — 五捨五超入（第五世代以降の正式な
    端数処理仕様）で丸める。
  - 反動比率1/3（すてみタックルほか）や1/4（アフロブレイク・とっしん等）では、与ダメが
    「比率で割った際の端数が0.5を超える」値のとき、旧実装は反動量が正しい値より1少なくなるバグが
    あった（例: 与ダメ101・比率1/3の場合、正しくは34だが旧実装では33になっていた）。
  - `round_half_down` は既に `src/jpoke/utils/math.py` に実装済みで、ダメージ計算本体
    （`core/damage.py`）でも同じ丸め規則が使われており、既存の丸め規則と整合させる形で修正した。
  - `_recoil` は複数の反動技（アフロブレイク・ウェーブタックル・ウッドハンマー・じごくぐるま・
    すてみタックル・とっしん・はめつのひかり・フレアドライブ・ブレイブバード・ボルテッカー・
    もろはのずつき・ワイルドボルト）で共有されているため、本修正は該当する全ての反動技に適用される
    （既存の完全一致テストはいずれも `fix_damage(battle, 100)` を使用しており、100に対しては
    どの比率でも端数が0.5以下のため、既存テストは全て通過することを確認した）。
  - `_drain_hp`（吸収技用ヘルパー）は本レビューの対象（すてみタックル）に無関係なため変更していない
    （比率0.75の吸収技で同様の乖離がありうる点は仕様書の「端数処理」節に注記した）。
- `python scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` を実行したが、
  `_recoil` は非公開ヘルパー関数（プレフィックス`_`）のため並び順に影響はなかった。

## テスト
- 既存の `test_すてみタックル_使用後に攻撃者が反動ダメージを受ける`（HPが減少することのみ確認）は
  変更なし。
- `tests/moves_attack/test_move_sa.py` に以下を追加した。
  - `test_すてみタックル_反動ダメージが与ダメの3分の1になる`:
    `fix_damage(battle, 100)` で反動量が `max(1, round_half_down(100/3)) = 33` になることを確認。
  - `test_すてみタックル_反動ダメージの端数が0_5を超える場合は切り上げられる`:
    `fix_damage(battle, 101)` で反動量が34になることを確認（`int()`切り捨てなら33になり、
    今回の修正が退行しないことを検証する回帰テスト）。

  （すてみタックルを使用してテスト）は引き続き通過することを確認した。変更なし。
- `python scripts/sort_tests.py tests/moves_attack/test_move_sa.py` を実行し、追加テストを適切な位置に整列した。
