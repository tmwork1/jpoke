## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ta.py`: `トーチカ` の PP を `10` → `8` に修正した。
  `src/jpoke/data/ps-champ-ja/moves.json` の532行目でPPが `8` と明記されており、実装値の `10` は
  第9世代本家基準のままの移行漏れだったため修正（`docs/progress/move.md` のPP列はもともと
  正しく `8` になっていた）。
- `handlers/volatile.py` の `トーチカ_protect` / `トーチカ_remove_volatile`、
  `handlers/move_status.py` の `トーチカ_apply`、`data/volatile.py` の `"トーチカ"` VolatileData
  エントリを確認したが、いずれも `まもる` / `ニードルガード` / `キングシールド` 等の既存protect系実装と
  同一パターン（`_run_protect` 共通関数を `ailment_on_contact="どく"` で呼び出す）で正しく実装されていた
  ため、コード変更なし。
  - `_FAINT_REMOVE_VOLATILES`（フェイント等による解除対象）、`ふかしのこぶし_bypass_protect`
    （`ON_CHECK_PROTECT`）にも `トーチカ` が正しく含まれていることを確認した。
  - `まねっこ_can_use` が `protect` フラグを持つ技を一律コピー不可としているため、`トーチカ` に
    個別の `non_copycat` フラグは不要であることを確認した。
