# グラスフィールド レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/グラスフィールド.md`）。`docs/wiki/moves/グラスフィールド.html` と
  `src/jpoke/data/ps-champ-ja/moves.json` を一次情報として、基本情報・効果・判定・チャンピオンズ向けの
  簡略化事項を記載。地形本体の詳細仕様は既存の `docs/spec/fields/グラスフィールド.md` を参照する
  形にした（`docs/spec/moves/エレキフィールド.md` と同じ構成）。

## 実装計画書
- 既存の計画書は `global_manager.activate` を使う旧設計のまま残っており、実際の実装
  （`terrain_manager.apply` を使う `グラスフィールド_activate_terrain`）と乖離していたため、
  `docs/plan/moves/エレキフィールド.md` と同一パターンに合わせて全面的に書き直した。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ka.py` の `"グラスフィールド"` エントリの `pp=10` は Gen9本家基準の
  値のままで `src/jpoke/data/ps-champ-ja/moves.json`（PP=12）への移行漏れだったため、`pp=12` に修正した。
- `handlers/move_status.py` の `グラスフィールド_activate_terrain`、`handlers/field.py` の
  `グラスフィールド_power_modifier` / `グラスフィールド_boost_move_priority` / `グラスフィールド_heal`、
  `data/field/terrain.py` のフィールド定義は仕様と一致しており、修正なし。

## リーサル計算
- `docs/progress/move.md` の当該行は「リーサル実装: n/a」（技本体はフィールド設定のみで
  HP/状態変化を持たないため対象外）。フィールド側の回復効果は `handlers/lethal.py` の
  `グラスフィールド_heal`（`data/field/terrain.py` の `lethal_handlers`）にすでに実装されており、
  接地判定・回復量（1/16）とも仕様と一致していたため変更なし。

## テスト
- `tests/moves_status/test_move__grouped.py` の `test_フィールド展開技_5ターン展開される` /
  `test_フィールド展開技_すでに同じフィールドなら失敗` でグラスフィールド技本体の発動・上書き失敗を
  カバー済み。`tests/test_field.py` で回復・くさ技威力補正・じしん系威力半減・浮遊時の補正なし・
  かいふくふうじでの回復無効・カウントダウンをカバー済み。`tests/moves_attack/test_move_ka.py` で
  グラススライダーの優先度上昇もカバー済みのため、テストの追加・修正は行わなかった。
