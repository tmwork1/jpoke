## 実装（handlers/ / data/）
- **バグ修正**: `src/jpoke/data/moves/move_ma.py` の `みねうち` エントリで、`Event.ON_MODIFY_MOVE_DAMAGE` ハンドラに `priority=60` を追加した。
  - 修正前は priority 未指定（デフォルト100）で、がんじょう・きあいのタスキ・きあいのハチマキ（いずれも priority=100）と同順位になり、`_sort_handlers` の素早さタイブレークにより **防御側が攻撃側より素早い場合、がんじょう/きあいのタスキが先に発動してしまう** バグを確認した。
  - 具体的には、HP満タンの相手（攻撃側より素早い）にみねうちで致死級ダメージを与えると、きあいのタスキが誤って消費され、がんじょうの特性発動ログが誤って出力されていた（修正前後で実際にバトルを実行し比較して確認）。
  - `priority=60`（`docs/spec/turn.md` 記載の値）を追加し、みねうちの1残し補正が常に先に適用されるよう修正した。修正後は防御側の素早さによらず、タスキは消費されず、がんじょうの特性バーも現れないことを確認した。
- `src/jpoke/handlers/move_attack.py` の `みねうち_modify_damage` 自体のロジック（HP1以下なら0、それ以外は `min(value, hp-1)`）は仕様と一致しており変更なし。
- 検討したが見送った項目: 「すでにHPが1の相手に使用したとき、ダメージを受けたときに発動する特性（せいでんき等）が発動する」効果。現在の実装では `actual_damage <= 0` のため `Event.ON_DAMAGE_HIT` が発火せず、せいでんき等の特性がトリガーされないことを実際に確認した。この修正には `core/move_executor.py` のダメージ処理フロー自体の変更が必要となり、みねうち単体の実装スコープを超えるため、仕様書に既知の未実装範囲として明記するに留めた。
