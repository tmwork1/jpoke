## 実装（handlers/ / data/）

- `src/jpoke/data/move.py`: とびげりの `power` を85→100、`accuracy` を90→95に修正
  （wikiに記載の第五世代以降の値）。`flags` に `"recoil"` を追加
  （すてみ特性が `ctx.move.has_flag("recoil")` で判定するため、欠落しているとすてみの威力上昇が発動しない）。
- `src/jpoke/handlers/move_attack.py`: `とびげり_crash` の `reason` を `"self_cost"` → `"fixed_recoil"` に修正。
  - `マジックガード_ignore_damage`（`handlers/ability.py`）は `"self_cost"` を除外リストに含めており、
    `self_cost` のままだとマジックガードで失敗反動を防げない仕様バグになっていた。
  - 実際のゲームではマジックガード所持時にとびげりが外れても反動ダメージを受けないため、
    `"fixed_recoil"`（マジックガードで防げるが、いしあたまでは防げない確定反動）に修正した。
  - `"fixed_recoil"` は `いしあたま`（`reason == "recoil"` のみ判定）では防げず、
    `recoil` フラグ（すてみ判定用）とも独立した別軸のため、上記の `recoil` フラグ追加とは矛盾しない。
- このレビューで判明した知見（`reason="self_cost"` → `"fixed_recoil"` の修正）は、同じ問題を抱えていた
  とびひざげり（並行レビュー）にも適用し、`とびひざげり_crash` を同様に修正した。
- `python scripts/sort_handlers.py src/jpoke/handlers/move_attack.py` と
  `python scripts/sort_data/sort_moves.py` を実行し、五十音順を維持した。
