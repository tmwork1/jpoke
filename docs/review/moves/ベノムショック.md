# ベノムショック レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/ベノムショック.md`）。一次情報（`docs/wiki/moves/ベノムショック.html`）
  に基づき、基本情報（どく/特殊/威力65/命中100/PP12/相手1体/優先度0/非接触）と
  効果（対象がどく・もうどく状態のとき威力2倍）、判定フラグを記載した。
  PPは一次情報の10ではなく `docs/champions/move_list.txt`（チャンピオンズのPP4段階統一）
  の値12を正として採用した。

## 実装計画書
- 新規作成（`docs/plan/moves/ベノムショック.md`）。既存の同種技（`たたりめ`）の計画書を
  テンプレートに、ハンドラ構成（`Event.ON_CALC_POWER_MODIFIER`、デフォルトpriority・
  subject_spec）、実装箇所、リーサル計算方針（n/a）、エッジケースを記載した。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/move_attack.py` の `ベノムショック_double_power_when_poisoned` は
  `ctx.defender.ailment.name in ("どく", "もうどく")` で正しく判定しており、仕様と一致。
  変更なし。
- `src/jpoke/data/moves/move_ha.py` の `"ベノムショック"` エントリで `pp=10` と
  なっていたバグを発見し、`pp=12`（`docs/champions/move_list.txt` 準拠）に修正した。
  `flags`（非接触）・`target`（デフォルト`foe`）・`priority`（デフォルト0）は仕様と一致
  しており変更不要。

## リーサル計算
- 対象外（`docs/progress/move.md` の該当行が `n/a`）。威力補正のみでダメージ計算モジュール
  側に反映されるため、`handlers/lethal.py` への専用ハンドラは不要。

## テスト
- 既存テスト（`tests/moves_attack/test_move_ha.py`）は、どく状態時の威力2倍・もうどく状態時の
  威力2倍・まひ状態（対象外）で補正なし・状態異常なし時の補正なしの4パターンを網羅しており、
  実装・仕様と整合していたため変更なし。
