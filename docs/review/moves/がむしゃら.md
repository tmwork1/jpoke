# がむしゃら レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/がむしゃら.md`）。
  - 基本情報（ノーマル/物理/威力-/命中100/PP5/相手1体/優先度0/接触あり）を一次情報から整理。
  - 「相手の残りHPが自分の残りHP以下の場合は無効化されたときと同様に失敗する」点を明記。
  - タイプ相性（ゴースト無効・固定ダメージのため半減/2倍は非適用）、`fixed_damage` フラグの必要性を明記。
  - 特性おやこあいで連続攻撃にならない点（`docs/spec/abilities/おやこあい.md` 参照）を明記。
  - リーサル計算は対象外（この技単独では相手を倒せないため）である旨を明記。

## 実装計画書
- 新規作成（`docs/plan/moves/がむしゃら.md`）。
  - `がむしゃら_can_use`（`Event.ON_TRY_MOVE_1`, priority=30, `docs/spec/turn.md` のカウンター・ミラーコートと同枠）を
    新設し、HP条件を満たさない場合に `LogCode.MOVE_IMMUNED` で技を失敗させる設計を明記。
  - `おやこあい_modify_hit_count` からがむしゃらを除外する修正の根拠を明記。

## 実装（handlers/ / data/）
- `src/jpoke/handlers/move_attack.py`
  - `がむしゃら_can_use` を新規追加。相手の残りHPが自分の残りHP以下の場合、
    `LogCode.MOVE_IMMUNED` を記録し `HandlerReturn(value=False, stop_event=True)` で技を失敗させる。
  - 修正前は `ON_MODIFY_MOVE_DAMAGE` 内で `max(0, defender.hp - attacker.hp)` により
    黙って0ダメージになるだけで、技としては「成功」扱いになってしまっていた
    （やけっぱち・じだんだ等の「前ターンの行動失敗」判定に反映されない誤りがあった）。
- `src/jpoke/data/move.py`
  - `がむしゃら` に `Event.ON_TRY_MOVE_1`（`がむしゃら_can_use`, priority=30）を追加。
  - `flags` に `fixed_damage` を追加（修正前は `{"contact"}` のみで欠落していた。
    じゃくてんほけん・ナゾのみ等の誤発動防止。ノーマル技は元々抜群を取れないため実害はほぼ無いが、
    他の固定ダメージ技との整合性のため追加）。
- `src/jpoke/handlers/ability.py`
  - `おやこあい_modify_hit_count` にがむしゃらの除外分岐を追加。
    修正前は単発攻撃技全般を2ヒット化する条件のみで、`docs/spec/abilities/おやこあい.md` に明記されている
    「がむしゃらはHPの変動有無に関わらずいかなる状況でも連続攻撃にならない」という規定に反し、
    誤って2ヒット化されてしまう不具合があった。
  - なお、いのちがけ・一撃必殺技等おやこあいの他の対象外規定は本レビューの対象外のため未修正
    （それぞれの技のレビュー時に個別対応する）。
- `docs/spec/turn.md`
  - `Event.ON_TRY_MOVE_1`（priority=30）の表に「がむしゃら: 相手の残りHPが自分以下」の行を追加。

## リーサル計算
- 対象外（`docs/progress/move.md` の「リーサル実装」列は n/a）。
  がむしゃらのダメージは常に「相手の残りHP−自分の残りHP」であり、結果として相手の残りHPは
  自分の残りHP以上にしかならないため、この技単独では相手をひんしにできない。

## テスト

    正常系1ケースのみに整理。
  - `test_がむしゃら_相手HPが自分以下の場合は失敗する`（相手HP<自分HP・相手HP=自分HPの2ケース）を新規追加し、

  - `test_がむしゃら_ゴーストタイプに無効` を新規追加。
- `tests/abilities/test_ability_a.py`
  - `test_おやこあい_がむしゃらには適用しない` を新規追加し、おやこあい所持時もがむしゃらが
    連続攻撃にならないことを検証。
- 上記2ファイルを `scripts/sort_tests.py` でソートし、`scripts/generate_test_list.py` で
  `docs/tests/` を更新。
- `scripts/generate_test_list.py` に既存バグを発見・修正: `@pytest.mark.parametrize` の第一引数が
  `("a", "b", "c")` のようなタプル形式（本プロジェクトの多くのテストで使われている記法）の場合、
  パラメータ名の抽出に失敗してケースが一覧から丸ごと消えてしまう不具合があった
  （新規追加した `test_がむしゃら_相手HPが自分以下の場合は失敗する` がこのバグで一覧に出てこなかったことで発覚）。
  タプル/リスト形式のパラメータ名にも対応し、さらに parametrize が検出されたのにケースを1件も
  抽出できなかった場合はベース名だけでも一覧に残すフォールバックを追加した（テストが一覧から
  サイレントに消えることを防止）。この修正により `docs/tests/` 配下の複数ファイル
  （`test_stats.md`, `test_terastal.md`, `test_damage.md`, `test_megaevol.md`,
  `test_types_generated.md`, `volatiles.md`, `abilities.md` 等）で、既存のタプル形式
  parametrize テストが正しく一覧に反映されるようになった（差分はすべて追記のみ）。
- `python -m pytest tests/ -q` で全3293件（+1 skipped）がパスすることを確認
  （`.loop/test_logs/がむしゃら.log`）。
