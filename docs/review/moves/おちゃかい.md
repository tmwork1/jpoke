# おちゃかい レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/おちゃかい.md`）
- 一次情報（`docs/wiki/moves/おちゃかい.html`）と `docs/champions/move_list.txt`（L152）を照合し、
  基本情報（ノーマル/変化/威力なし/命中なし〈必中〉/PP12/専用技〈ポットデス〉）、
  本来の効果（場の全ポケモンが持つきのみを発動条件を無視して強制消費させる、
  消費順は使用者→使用者の味方→相手左→相手右）、および各種エッジケース
  （姿を隠しているポケモンは消費しない、みがわり状態は消費する、まもる/トリックガードを無視、
  ねんちゃく/きんちょうかんの影響下でも消費、ぶきよう/マジックルームでも本来は消費・効果発動、
  ものひろい/しゅうかく/リサイクルでの復元対象になる）を明記した。
- ぶきよう/マジックルームのケースは本プロジェクトの `Item.enabled` ゲート機構（無効時は
  `item.name` が空文字列扱いになり `is_berry()` が False を返す）により実装上は
  対応できない既知の制限であることを明記した（`ほおばる` にも共通する制限）。
- **誤りを発見・修正**: `docs/progress/move.md` で本技が全列「ダブル専用」として
  扱われていたが、これは誤りと判断した。`てだすけ`・`いかりのこな`・`おさきにどうぞ` 等の
  「味方の存在が前提でシングルでは効果が成立しない技」とは異なり、おちゃかいは
  「使用者＋相手」の2体構成のシングルバトルでも実際に機能する（相手のきのみを
  強制消費させる実利がある）。さらに `src/jpoke/core/item_manager.py` の
  `force_trigger_berry()` と `src/jpoke/enums/event.py` の `ON_FORCE_BERRY_TRIGGER` の
  コメントには「ほおばる・おちゃかい等」と明記されており、フレームワーク自体が
  当初からおちゃかいの実装を前提に設計されていたことを確認し、対象外の分類を撤回した。

## テスト
- 追加した（`tests/moves_status/test_move_a.py`）
  - `test_おちゃかい_自分のきのみが消費される`: 使用者のきのみが強制消費されることを確認
  - `test_おちゃかい_相手のきのみも消費される`: 相手のきのみも同時に強制消費されることを確認
  - `test_おちゃかい_HP条件を満たす場合は効果を受けて回復する`: HPが半分以下のときに
    オボンのみを強制消費すると通常通りHPが回復することを確認
  - `test_おちゃかい_姿を隠しているポケモンはきのみを消費しない`: そらをとぶ状態の相手は
    きのみを消費しないことを確認
  - `test_おちゃかい_きのみを持つポケモンがいなくても技は成功する`: きのみを持つポケモンが
    いなくても技自体は失敗せずPPが通常通り消費されることを確認
- `python -m pytest tests/ -v` を実行し、4126 passed, 1 skipped で全件成功。
  結果は `.loop/test_logs/おちゃかい.log` に保存した
  （複数回実行したところ、`てんねん`・`だっしゅつパック`・`くらいつく`・`むしくい` 等
  おちゃかいと無関係な既存テストが乱数依存でごく稀にフレーキーに失敗する既存事象を確認したが、
  再実行すればいずれも全件成功する。本レビューの変更には起因しないため対応対象外とした）。
