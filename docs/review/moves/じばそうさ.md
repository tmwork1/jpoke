## 実装（handlers/ / data/）
- 修正した
  - `src/jpoke/data/moves/move_sa.py` の `"じばそうさ"` エントリに `target="self"` を追加
  - **発見したバグ**: `MoveData.target` が既定値の `"foe"` のままだったため、
    `MoveExecutor.run_move` が常に `defender = battle.foe(attacker)`（相手）としてコンテキストを構築する
    実装上、以下の2つの誤動作が発生していた（実機で再現・修正後に解消を確認済み）
    1. 相手がみがわり状態のとき、`みがわり_immune`（`ON_BEFORE_APPLY_MOVE` priority 30、
       `ctx.move.target == "foe"` を条件に判定）により、本来無関係なはずの相手のみがわりで
       じばそうさが無効化されてしまっていた
    2. 相手がちくでん/ひらいしん/でんきエンジンを持つとき、`handlers/ability.py` の
       `_apply_type_absorb`（`ctx.move.target == "foe"` を条件に判定）により、
       でんきタイプ技として誤って吸収・無効化されてしまっていた
    `target="self"` を指定することで `check_hit_substitute` が早期に `False` を返すようになり、
    上記2点とも解消した。これはポケモンWikiの「技の仕様」セクションの記述
    （「味方がちくでん/ひらいしん/でんきエンジンの場合、これらの特性に対してじばそうさは発動せず」）
    とも整合する
  - `handlers/move_status.py` の `じばそうさ_can_apply` / `じばそうさ_modify_attacker_stats` 自体は
    仕様と一致しており変更なし
