# パワースワップ レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/パワースワップ.md`）。`docs/wiki/moves/パワースワップ.html` の一次情報
  （PP=12基準前のGen9値=10、命中率「-」＝必中、まもる: 防がれる、マジックコート: 跳ね返されない、
  みがわり: 貫通する、おうじゃのしるし/よこどり: 対象外、オウムがえし: コピー可能）と、
  「技の仕様」節（ランク低下無効特性・あまのじゃく・まけんき/かちき等のトリガーをすり抜ける）を
  反映。`ガードスワップ`・`スピードスワップ` の既存仕様書と同一フォーマットで作成した。

## 実装計画書
- 既存の `docs/plan/moves/パワースワップ.md` を修正。
  - PP を 10→12（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）に修正。
  - `accuracy=100`→`accuracy=None`（必中）に修正。
  - `flags={"unreflectable", "bypass_substitute"}` の追加を明記（マジックコート非反射・みがわり貫通）。
  - ハンドラのサンプルコードを実装済みコードに合わせて修正（ランクキーを `"A","C"` から実際の
    `"atk","spa"` に、`data/move.py` を実ファイルパス `data/moves/move_ha.py` に修正）。
  - エッジケース節に「マジックコート」「必中」「PP」の項目を追加し、みがわりの説明を
    「デフォルトでは阻まれるため `bypass_substitute` が必須」という正しい内容に修正
    （旧記述は「ON_STATUS_HITは通常みがわりに阻まれない」という誤り）。

## 実装（handlers/ / data/）
- `handlers/move_status.py` の `パワースワップ_swap_ranks` はランクキー交換ロジックとして正しく
  実装済みで変更なし。
- `data/moves/move_ha.py` の `"パワースワップ"` エントリを修正:
  - `pp=10` → `pp=12`（`src/jpoke/data/ps-champ-ja/moves.json` の値。Gen9本家基準のまま移行されていなかった
    バグ）。
  - `accuracy=100` → `accuracy=None`（必中技であるにもかかわらず通常命中判定扱いになっていたバグ。
    このままだと回避率ランク補正の影響を受けてしまう）。
  - `flags={"unreflectable", "bypass_substitute"}` を追加（マジックコートで跳ね返される・みがわりを
    貫通しないという誤動作を修正）。
  - これは過去にレビュー済みの `スキルスワップ` と同種のバグ（Gen9本家値のまま移行漏れ）だった。
