# アクアブレイク レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/アクアブレイク.md`）。
  一次情報（`docs/wiki/moves/アクアブレイク.html`）は本セッションでは
  wiki.pokemonwiki.com へのネットワークアクセスがタイムアウトし取得できなかったため、
  `docs/champions/moves.md`・`docs/spec/moves/_move_secondary_1.md`（10行目: 対象=defender、
  確率=0.2、B=-1）・既存実装・既存テスト・`docs/spec/abilities/みずがため.md`（アクアブレイク/
  シェルブレードとの発動順序について既に一次情報照合済み）を突き合わせて内容を記載した。
  タイプ=みず／分類=物理／威力85／命中100／PP12（champions基準）／接触あり／
  追加効果20%でぼうぎょ-1、切る技には該当しない、という内容で確定した。

## 実装計画書
- 新規作成（`docs/plan/moves/アクアブレイク.md`）。シェルブレードの計画書と同一パターン
  （共通ヘルパー `modify_defender_stats` を使用、`Event.ON_DAMAGE_HIT priority=10` で
  みずがため`priority=20`より先に発動させる根拠を明記）で作成した。

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/data/moves/move_a.py` の `アクアブレイク` エントリ
  （type=みず, category=physical, pp=12, power=85, accuracy=100,
  flags={"contact", "secondary_effect"}、`Event.ON_DAMAGE_HIT priority=10` に
  `アクアブレイク_lower_defender_def` を登録）は仕様と一致していることを確認した。
  `src/jpoke/handlers/move_attack.py` の `アクアブレイク_lower_defender_def` は
  `modify_defender_stats(battle, ctx, value, stats={"def": -1}, chance=0.2)` を呼び出しており、
  20%の確率で相手のぼうぎょを1段階下げる仕様と一致する。priority=10（みずがための priority=20
  より先）の根拠もコード内コメントに明記されており正しい。既に五十音順の位置に実装済み。
