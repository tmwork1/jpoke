# デスウイング レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/デスウイング.md`）。
  - `docs/wiki/moves/デスウイング.html` を一次情報として、タイプ（ひこう）・分類（特殊）・
    威力（80）・命中（100）・対象・優先度・接触なし・ドレイン技共通仕様（四捨五入、
    おおきなねっこ1.3倍、ヘドロえきで反動化、みがわり吸収、かいふくふうじ対象）を記載。
  - PPは `docs/champions/move_list.txt` に単独エントリが存在しないため、第九世代の基準値
    （10）を採用する方針を明記（`すいとる` と同じ扱い）。

## 実装計画書
- 新規作成（`docs/plan/moves/デスウイング.md`）。
  - 既存ドレイン技共通処理 `_drain_hp`（`handlers/move_attack.py`）の再利用方針、
    `Event.ON_HIT` の priority=20（`docs/spec/turn.md` 準拠）を明記。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ta.py` の `MoveData` を仕様書に合わせて修正:
  - `category`: `physical` → `special`（誤り。実際は特殊技）
  - `power`: `90` → `80`（誤り。一次情報では威力80）
  - `flags`: `{"contact", "heal"}` → `{"heal"}`（`contact` を削除。デスウイングは直接攻撃ではない）
  - `Event.ON_HIT` の `MoveHandler` に `priority=20` を追加
    （デフォルトの `priority=100` のままだったため、他のドレイン技と処理順が異なっていた）
- `src/jpoke/handlers/move_attack.py` の `デスウイング_drain`（`_drain_hp` を heal_ratio=0.75 で
  呼び出す実装）は仕様と一致しており変更なし。

## リーサル計算
- 対象外（`docs/progress/move.md` のリーサル実装列は `n/a`。他のドレイン技と同様に
  ドレイン技グループとしてリーサル計算ハンドラは未実装の方針）。

## テスト
- `tests/moves_attack/test_move_ta.py::test_デスウイングとドレインキッスの回復量比較` を
  `test_デスウイングとドレインパンチの回復量比較` にリネーム（実際の比較対象はドレインパンチであり、
  テスト名が実装内容と一致していなかった）。
  - また、`power`/`category` 修正後にファイヤー（特殊高め）とカビゴン（特防高め・防御低め）の
    ステータス差により素の与ダメ比較が破綻したため、`t.fix_damage` で与ダメを固定して
    heal_ratio の違いのみを検証するように修正。
  - `test_デスウイング_使用後に攻撃者のHPが回復する` は変更なし。
  - `tests/moves_attack/test_move__group.py` の `test_drain技グループ_回復量が与ダメのheal_ratio倍になる`
    に既存の `("デスウイング", 0.75)` パラメータがあることを確認（変更なし）。
- `python scripts/sort_tests.py tests/moves_attack/test_move_ta.py` と
  `python scripts/generate_test_list.py` を実行済み。
- `python -m pytest tests/ -v` は 3899 passed, 1 skipped で全件パス
  （`.loop/test_logs/デスウイング.log` に保存）。
  ※ pytest-randomly によるテスト順序ランダム化に起因する既存の別テスト
  （てんねん関連・だっしゅつパック関連等）の順序依存フレークを確認したが、
  デスウイングの変更とは無関係であり、`-p no:randomly` では全件パスすることを確認済み。
