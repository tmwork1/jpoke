# サイコショック レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/サイコショック.md`）。一次情報（ポケモンWiki／ポケモン徹底攻略／ゲームウィズ）と
  `docs/champions/move_list.txt` を照合し、タイプ・分類・威力・命中・PP・対象・接触の有無・
  「相手の『とくぼう』ではなく『ぼうぎょ』でダメージ計算する」という固有効果を明記した。
  同種の「物理判定の特殊技」である `しんぴのつるぎ` の仕様書構成に合わせた。

## 実装計画書
- 新規作成（`docs/plan/moves/サイコショック.md`）。既存実装のバグ2件（PP誤り・`physical_damage`
  フラグ欠落）とその修正方針、`Query.deals_physical_damage()` の既存機構をそのまま利用できる旨を
  記載した。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_sa.py` の `"サイコショック"` エントリを修正した。
  - `pp=10`（Gen9本家基準）→ `pp=12`（`docs/champions/move_list.txt` の Champions基準値）に修正。
  - `flags={"physical_damage"}` を追加。従来は `flags` が未指定で、技の説明文
    （相手の『ぼうぎょ』でダメージ計算する）に反して実際には『とくぼう』で計算されてしまう
    バグがあった。`physical_damage` フラグにより `Query.deals_physical_damage()` →
    `DamageCalculator._calc_final_defense()` の既存経路で防御側の参照ステータスが
    `def`（ぼうぎょ）に切り替わることを確認した。
  - `handlers/move_attack.py` への追加・修正は不要（既存の汎用ダメージ計算フローで対応可能なため）。

## テスト
- `tests/moves_attack/test_move_sa.py` にテストを3件追加した（`サイコキネシス` と
  `サイコファング` のテストの間に配置）。
  - `test_サイコショック_相手にダメージを与える`: 通常のダメージが発生することを確認。
  - `test_サイコショック_相手のとくぼうランク変化の影響を受けない`: 相手の『とくぼう』ランクが
    上がっていてもダメージが変わらないことを確認（『ぼうぎょ』参照であることの裏付け）。
  - `test_サイコショック_相手のぼうぎょランク変化の影響を受ける`: 相手の『ぼうぎょ』ランクが
    上がっているとダメージが減ることを確認。
  - `しんぴのつるぎ` の既存テストと同型の構成にした。
