# テラクラスター レビュー結果

## 仕様書
- 既存（`docs/spec/moves/テラクラスター.md`）は一次情報（`docs/wiki/moves/テラクラスター.html`）と
  照合し、基本仕様・タイプ変化・分類変化・対象範囲変化・コピー制限は正しく記載されていた。
- 「防御・無効化相互作用」節に、他の技仕様書（例: `テラバースト.md`）と同様の
  命中判定／急所判定／まもる／マジックコート／追加効果の一覧表を追加した（欠落していたため補充）。

## 実装計画書
- 存在しなかったため新規作成（`docs/plan/moves/テラクラスター.md`）。
- `ON_MODIFY_MOVE_TYPE` / `ON_MODIFY_MOVE_CATEGORY` の2ハンドラ構成、priority根拠
  （`docs/spec/turn.md` 未掲載のため既存の同種ハンドラ〈テラバースト・フォトンゲイザー〉を根拠に
  デフォルトpriority=100を採用）を明記。
- 本プロジェクトがシングルバトル専用であるため、対象範囲変化（相手全体化）とワイドガード相互作用は
  戦闘結果に影響しないことを明記し、実装対象外とした（`ワイドガード` エントリのコメントと同じ理由）。
- PPについて、`docs/champions/move_list.txt` に本技の記載が存在しないため、
  `docs/champions/changes_from_sv.md` の「全技PPが8/12/16/20の4段階に統一」規則と、
  本家PP5の技（アーマーキャノン・アイアンローラー・はかいこうせん・しんそく・じばく・だいばくはつ）が
  すべてchampions PP8に統一されている実例から、championsPPを8と推定した根拠を記載した。

## 実装（handlers/ / data/）
- 従来の実装は `handlers={}`（コメント「追加効果なし」）となっており、ステラテラスタル時の
  タイプ変化・分類変化がまったく実装されていなかった（バグ）。
- `src/jpoke/handlers/move_attack.py` に `テラクラスター_modify_move_type` /
  `テラクラスター_modify_move_category` を新規実装した。
  - `ctx.attacker.active_tera_type == "ステラ"` の場合のみタイプを「ステラ」に変更する
    （テラバーストと異なり、ステラ以外のテラスタイプでは変化しない）。
  - 同条件下で `ranked_stats["atk"] > ranked_stats["spa"]` の場合のみ分類を「physical」に変更する。
  - テラバーストと異なり、威力補正・命中後の能力ランクダウンは実装しない（一次情報上そのような
    効果は存在しないため）。
- `src/jpoke/data/moves/move_ta.py` の `"テラクラスター"` エントリに上記2ハンドラを登録し、
  `flags={"non_copycat"}` を追加した（まねっこでコピー不可。ものまね・スケッチ・ゆびをふるは
  本プロジェクト未実装のため対応フラグなし）。
- PPを本家基準の `5` から championsの推定値 `8` に修正した（根拠は計画書参照）。

## リーサル計算

  通常のダメージ計算経路にタイプ・分類変化が自然に反映されるため、専用リーサルハンドラは不要。
