# サンダープリズン レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/サンダープリズン.md`）。
- 一次情報（ポケモンWiki）と `docs/spec/volatiles/バインド.md` の記載内容を突き合わせ、基本情報・効果・判定フラグ・備考を整理した。

## 実装計画書
- 新規作成（`docs/plan/moves/サンダープリズン.md`）。
- 既存のバインド系技（うずしお・すなじごく・まとわりつく等）と同じ `apply_bind_to_defender` / `_apply_bind` を流用する方針を明記した。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_sa.py` の `サンダープリズン` エントリに `lethal_handlers`（`LethalEvent.ON_HIT: LethalHandler(l._apply_bind)`）が欠落していたバグを修正した。
  - 他の同系統バインド技（うずしお・しめつける・すなじごく・まきつく・ほのおのうず・まとわりつく・トラバサミ・マグマストーム）はすべて `lethal_handlers` を登録済みだったが、サンダープリズンのみ抜けていた。
  - `handlers/move_attack.py` の `apply_bind_to_defender`、`handlers/lethal.py` の `_apply_bind` 自体は既存の共通実装をそのまま利用でき、修正不要だった。
  - `flags` に `contact` が付与されていない点は一次情報（直接攻撃: ×）と一致しており、変更なし。

## リーサル計算
- 修正（既存の `_apply_bind` の登録漏れを追加）。
  - `docs/progress/move.md` のリーサル実装列は `-`（未実装）だったが、実体はハンドラ自体は既存共通実装であり、`data/moves/move_sa.py` への登録のみが漏れていた。
  - 登録を追加し、`tests/test_lethal.py` に `test_サンダープリズン_バインド付与`（技経由のバインド付与とバインド事前付与で同一のリーサル結果になることを確認）を追加した。

## テスト
- `tests/moves_attack/test_move_sa.py` に以下4件を追加（`レジエレキ` の専用技のため、攻撃者は `Pokemon("レジエレキ", move_names=["サンダープリズン"])` を使用）:
  - `test_サンダープリズン_命中後にバインド状態になる`
  - `test_サンダープリズン_バインド中は相手が毎ターン最大HPの8分の1ダメージを受ける`
  - `test_サンダープリズン_バインド中は相手が交代できない`
  - `test_サンダープリズン_攻撃者が交代するとバインドが解除される`
- `tests/test_lethal.py` に `test_サンダープリズン_バインド付与` を追加。
- `python -m pytest tests/ -v` を実行し、3779 passed, 1 skipped（既存のスキップ1件のみで新規失敗なし）を確認した。
