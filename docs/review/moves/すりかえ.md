## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_sa.py`
  - `pp` を `10` → `12` に修正（`src/jpoke/data/ps-champ-ja/moves.json` の値に合わせる。チャンピオンズでは
    全技PPが8/12/16/20の4段階に統一されているため、本家Gen9基準の値は移行漏れのバグ）。
  - `flags={"unreflectable"}` を追加。`Move.is_reflectable` は変化技かつ対象が `foe`/`foe_side`
    の技をデフォルトでマジックコート反射対象とするが、すりかえは仕様上（一次情報の判定表で
    「マジックコート: ×」）反射されない例外技のため、フラグで明示的に除外した
    （まもる/マジックコート誤反応の典型パターンだったため要修正）。
  - `Event.ON_MOVE_END` に `すりかえ_release_choice_lock`（priority=110）を追加登録し、
    こだわり系アイテムの `ON_MOVE_END` ハンドラ（デフォルト優先度100）より後に発動させ、
    自身の効果で新たに入手したこだわり系アイテムによるロックを解除するようにした。
- `src/jpoke/data/moves/move_ta.py`（トリックは本レビュー対象外だが、すりかえと全く同一の
  共有ハンドラ・共有バグのため、整合性のため同時に修正）
  - `pp` を `10` → `12` に修正（`src/jpoke/data/ps-champ-ja/moves.json` に合わせる）。
  - `flags={"unreflectable"}` を追加。
  - `Event.ON_MOVE_END` に `すりかえ_release_choice_lock`（priority=110）を追加登録。
- `src/jpoke/handlers/move_status.py`
  - `すりかえ_release_choice_lock` を新規実装（すりかえ・トリックの `ON_MOVE_END` で、
    使用者の「こだわり」揮発状態を無条件に解除する）。

## テスト
- `tests/moves_status/test_move_sa.py` に以下を追加。
  - `test_すりかえ_マジックコートで跳ね返されない`: 対象がマジックコート状態でも技が跳ね返らず
    （`LogCode.MOVE_REFLECTED` ログが記録されない）、通常通り持ち物が交換されることを確認。
  - `test_すりかえ_こだわり系アイテムを入手してもロックされない`: すりかえの効果で新たに
    こだわりハチマキを入手しても、「こだわり」揮発状態が付与されないことを確認。
  - `test_すりかえ_既にロックされていた場合も解除される`: 使用前から「こだわり」でロックされて
    いた場合も、すりかえの使用でロックが解除されることを確認。
  - 既存の3件（両者アイテムなしで失敗／両者のアイテムが入れ替わる／片方のみアイテムを持つ場合も
    成功）はそのまま維持（変更なし）。
- `python -m pytest tests/ -v` で全4211件パス（1件skip）を確認。結果は
  `.loop/test_logs/すりかえ.log` に保存。
  - 実行中、`すりかえ` と無関係な複数テスト（`てんねん`・`むしくい`・`test_field.py` のクリティカル
    確率絡みのテスト等）がテスト順序のランダム化（pytest-randomly）に起因すると見られる既存の
    フレーキーな失敗を示すことがあったが、変更前のコードでも同様の非決定的な失敗が再現することを
    確認済みであり、本レビューの変更によるものではない。
