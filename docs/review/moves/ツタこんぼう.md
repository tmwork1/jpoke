# ツタこんぼう レビュー結果

## 仕様書
- 既存の `docs/spec/moves/ツタこんぼう.md` を一次情報（ポケモンWiki）と照合し、以下を修正した。
  - 「反射・無効化」節: 「ミラー系（ミラーコート等）で反射可能」は誤り
    （ミラーコートは特殊技専用で、本技は物理技のため対象外）。カウンターで反射可能に修正。
    マジックコート・よこどりについても「攻撃技のため対象外」である理由を明記。
  - 「フォーム依存処理」節: 「なげつける等で仮面を失うとタイプ変化が無効化される可能性がある」は誤り。
    オーガポンの仮面は `removable=False` かつ、なげつけるはオーガポン使用時に失敗するため
    （`src/jpoke/data/item.py`・`src/jpoke/handlers/move_attack.py` の `なげつける_check_item` で
    実装済み）、戦闘中に仮面を失うことは無い旨に修正。
  - 一次情報にあった以下の詳細を追記した。
    - みずびたし等で現在タイプが変わってもツタこんぼうのタイプは変わらない（フォルム基準判定）
    - マジックルーム・ぶきようで仮面の威力補正が無効化されてもタイプは変わらない
    - へんしんでオーガポンに変身した場合は変身対象のフォルムに従う（へんしん自体は未実装のため
      計画書側で「対象外」と明記）
    - イリュージョンでの見た目分岐、きけんよちが常にくさタイプとみなす点
  - PP(10)について、`docs/champions/move_list.txt` に本技のエントリが存在しない
    （コライドンのアクセルブレイク、テラパゴスのコラプシンドゥーム等の他の専用技も同様に未掲載）ため、
    第9世代本家基準の値を採用し、その旨を注記として追加した。

## 実装計画書
- `docs/plan/moves/ツタこんぼう.md` が存在しなかったため新規作成した。
  `ON_MODIFY_MOVE_TYPE`（priority=100、既存の類似ハンドラ `だいちのはどう_modify_move_type` /
  `さばきのつぶて_modify_move_type` / `テラバースト_modify_move_type` を根拠に決定）を用いた
  実装方針、へんしん依存部分の対象外化、テスト方針を記載。

## 実装（handlers/ / data/）
- 実装時点で `data/moves/move_ta.py` の `"ツタこんぼう"` エントリに `handlers={}` しか
  登録されておらず、本技の核心機能（フォルムに応じたタイプ変化）が未実装だった。
  - `src/jpoke/handlers/move_attack.py` に `ツタこんぼう_modify_move_type` を新規実装
    （`つけあがる_calc_power` と `つららおとし_apply_flinch` の間に配置）。
    使用者がオーガポン（`attacker.name.startswith("オーガポン")`）であれば
    `attacker.base_types[-1]`（フォルム基準の非くさ側タイプ、単色フォルムはくさ自身）を返し、
    それ以外はデフォルトのくさタイプのまま変更しない。
  - `src/jpoke/data/moves/move_ta.py` の `"ツタこんぼう"` エントリに
    `Event.ON_MODIFY_MOVE_TYPE: h.MoveHandler(ha.ツタこんぼう_modify_move_type)` を追加登録。
  - なげつける（`なげつける_check_item`）、オーガポンの仮面アイテム（`removable=False`・
    攻撃技威力1.2倍）は既に正しく実装済みであることを確認（変更なし）。

## リーサル計算
- `docs/progress/move.md` の該当行は着手前「リーサル実装: -」だったが、本技は
  固定威力・追加効果なしの通常攻撃技であり、タイプ変化も命中前に確定するためHP分布に
  影響しない。同種の「急所ランク+1・追加効果なしの通常攻撃技」である `つじぎり` や
  `あくうせつだん` も「リーサル実装: n/a」であることを確認し、同様に対象外と判断した
  （進捗表の誤記として `n/a` に修正）。新規のリーサルハンドラは実装していない。

    （カビゴンにいどのめんを持たせても、タイプはくさタイプ固定になることを確認）
- `python -m pytest tests/ -v` を実行し、`.loop/test_logs/ツタこんぼう.log` に保存
  （3877 passed, 1 skipped, 0 failed）。
  なお `tests/test_field.py::test_オーロラベール_ひかりのかべとは重複しない` 等、
  本技と無関係な一部テストが実行順序に依存して稀に失敗する既知のフレーキーな挙動を確認したが、
  変更前のコードでも再現することを確認済みのため、本レビューの対象外として扱った。
