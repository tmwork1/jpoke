# しっぽきり レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/しっぽきり.md`）。
  - `docs/wiki/moves/しっぽきり.html`、`docs/spec/moves/交代技.md`、`docs/spec/volatiles/みがわり.md`
    を基に、基本情報・効果・失敗条件・判定フラグ・交代処理との関係をまとめた。
  - PPは `src/jpoke/data/ps-champ-ja/moves.json` 基準の12を採用（本家SVは10）。
  - HP不足の失敗条件は「消費コスト `ceil(max_hp/2)` 以下で失敗」と明記し、
    通常の `みがわり`（`floor(max_hp/4)` 以下で失敗）と同じ「支払うとHPが0以下になる場合は
    使用不可」という一般則に基づくことを記載した。

## 実装計画書
- 修正した（`docs/plan/moves/しっぽきり.md`）。

  - HP不足の失敗条件の記述を、単純な `hp <= max_hp // 2`（切り捨て半分）から、
    実装済みの `hp <= (max_hp + 1) // 2`（消費コスト基準、みがわり_check と同じ一般則）に修正。
  - サンプルコードを実際の実装（`FailureLogPayload` の使用、`battle.query.can_switch`、
    `baton_pass_data` を利用したみがわり引き継ぎ）に合わせて更新。
  - 「現状の実装状況」表を実装済み状態に更新。
  - エッジケース節に PP・HP閾値の根拠を追記。
