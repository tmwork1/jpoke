# ダイヤストーム レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/ダイヤストーム.md`）。一次情報（`docs/wiki/moves/ダイヤストーム.html`）を元に、
  タイプ・分類・威力・命中・追加効果（50%で自分のぼうぎょ2段階上昇。第六世代は1段階だったが第七世代
  以降は2段階に変更）・判定フラグ（急所/命中判定/まもる/おうじゃのしるし/マジックコート/よこどり/
  オウムがえし/ちからずく）を整理して記載した。
- PPについて: `src/jpoke/data/ps-champ-ja/moves.json` に本技のエントリが存在しないことを確認した。本技は
  ディアンシーの専用技であり、ディアンシーは幻のポケモンのためポケモンチャンピオンズのレギュレーション
  では使用不可（`docs/spec/moves/シードフレア.md` に記載済みの前例と同じ扱い）。よってPPはGen9本家基準の
  `5` を正としてそのまま採用した（8/12/16/20への変換対象外）。
- ダブルバトル時の「複数体命中でも追加効果判定は1回のみ」という第七世代以降の仕様も記載したが、
  本プロジェクトはシングルバトルのみ対象のため実装不要である旨を明記した。

## 実装（handlers/ / data/）
- 変更なし。`src/jpoke/data/moves/move_ta.py` のダイヤストームのエントリ（type=いわ/category=physical/
  pp=5/power=100/accuracy=95/flags={"secondary_effect"}）と、`src/jpoke/handlers/move_attack.py` の
  `ダイヤストーム_sharply_boost_defender_B`（`modify_attacker_stats` で def +2 / chance 0.5）は
  仕様書と完全に一致しており、誤り・欠落はなかった。
  `flags={"secondary_effect"}` のみで `contact` は付与されておらず、非接触技である仕様とも一致。
  おうじゃのしるし等は本技固有のフラグ不要で汎用ハンドラが処理することも確認した。
