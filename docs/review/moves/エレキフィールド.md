- `docs/spec/moves/エレキフィールド.md` を新規作成した（技本体の基本情報・判定・簡略化事項）。
  エレキフィールドの継続効果（威力補正・ねむり／ねむけ防止等）は既存の `docs/spec/fields/エレキフィールド.md`
  を参照する形にした（内容は一次情報と照合済みで誤りなし）

## 実装計画書
- `docs/plan/moves/エレキフィールド.md` が `global_manager` / `data/move.py`（分割前）/ `category="変化"`
  など旧アーキテクチャを前提にした内容のまま放置されていたため、現行実装（`terrain_manager`,
  `data/moves/move_a.py`, `category="status"`, PP=12）に合わせて全面的に書き直した
- `docs/plan/fields/エレキフィールド.md` は現行実装と完全に一致していたため変更なし

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_a.py` の `"エレキフィールド"` エントリで PP が `10`
  （Gen9本家準拠、移行漏れ）になっていたのを `src/jpoke/data/ps-champ-ja/moves.json` 準拠の `12` に修正した
- `handlers/move_status.py` の `エレキフィールド_activate_terrain`、`handlers/field.py` の
  地形効果本体（威力補正・ねむり防止・ねむけ防止・カウントダウン）は仕様と一致しており変更なし

## リーサル計算

  発生させないステータス変化技のため）

## テスト
- 技本体（`ON_STATUS_HIT` での地形発動）を直接検証するテストが存在しなかったため、
  `tests/moves_status/test_move_a.py` に以下を追加した:
  - `test_エレキフィールド_地形がエレキフィールドになる`: 使用後に `battle.terrain.name` が
    `"エレキフィールド"` になり、`count == 5` になることを確認
  - `test_エレキフィールド_すでに同じ地形なら失敗`: すでにエレキフィールドが有効な場合、
    再使用してもカウントが変化しない（再設置されない）ことを確認
- 地形自体の効果（ねむり／ねむけ防止・威力補正・ねむる失敗など）を検証する `tests/test_field.py` の
  既存テストは実装と一致しており変更なし
