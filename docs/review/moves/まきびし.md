# まきびし レビュー結果

## 仕様書
- 新規作成（`docs/spec/side_fields/まきびし.md`）。
  - `docs/spec/moves/` ではなく、既存の `どくびし.md` と同じ配置ルールに合わせて
    `docs/spec/side_fields/まきびし.md` として作成した（まきびしは「技」自体の効果より
    「相手陣営に設置するフィールド効果」の詳細が主であり、どくびし・ステルスロック等の
    設置技も同様に `docs/spec/side_fields/` 側にまとめる方針のため）。
  - 内容は一次情報（`docs/wiki/moves/まきびし.html` および
    `docs/wiki/side_fields/まきびし_(場の状態).html`）に基づき、層数ごとのダメージ割合
    （1/8, 1/6, 1/4）、ダメージの切り捨て・最低1保証、マジックガード/あつぞこブーツでの
    無効化、ひこうタイプ・特性ふゆう・でんじふゆう/テレキネシス継承・ふうせん所持による
    無効化と、くろいてっきゅう・じゅうりょく・ねをはる継承による無効化の上書き、
    アイテム/特性無視条件（ぶきよう・さしおさえ継承・マジックルーム／いえき継承・
    かがくへんかガス）、消滅条件（こうそくスピン等・きりばらい・キョダイフウゲキ・
    おかたづけ）をまとめた。

## 実装計画書
- 既存の `docs/plan/moves/まきびし.md` を修正した。
  - 誤り: 「『どくタイプ』のポケモンには無効」という記述があったが、これは
    どくびし固有の仕様（毒タイプ着地でフィールド解除）との混同であり、まきびしには
    タイプによる無効は存在しない。削除し、「どくタイプによる無効はない」旨を明記。
  - 「現状の実装状況」表が「`handlers/move_status.py` 関数: 未実装」のままだったが、
    実際には既に実装済みだったため更新した。
  - ハンドラ実装例の `ctx` 型注釈が `EventContext` になっていたが、実装は
    `ON_STATUS_HIT`（攻撃フロー）のため正しくは `AttackContext`。修正した。
  - 実装ファイルパスが `src/jpoke/data/move.py` / `category="変化"` と誤っていたため、
    実際の `src/jpoke/data/moves/move_ma.py` / `category="status"` に修正した。
  - Priority根拠が `Event.ON_APPLY_MOVE` テーブルの「設置技: すでに最大」を誤って引用
    していた（別イベント）。`ON_STATUS_HIT` は `docs/spec/turn.md` に未掲載のイベントである
    旨と、既存の同種ハンドラ（どくびし・ステルスロック等）がすべてデフォルト優先度のまま
    登録されている根拠に修正した。
  - 「注意点・エッジケース」の記述を、実装済みの `StackableFieldManager.activate()` /
    `data/field/side_field.py` の実態に合わせて更新した（「確認が必要」等の未確認表現を除去）。
