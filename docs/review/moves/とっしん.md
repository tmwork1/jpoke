# とっしん レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/とっしん.md`）。一次情報（`docs/wiki/moves/とっしん.html`）に基づき、
  タイプ・分類・威力・命中率・PP・反動仕様（1/4反動、`round_half_down` による端数処理）・判定フラグをまとめた。

## 実装計画書
- 新規作成（`docs/plan/moves/とっしん.md`）。既存の共通計画書
  `docs/plan/moves/反動技_025.md`（とっしん含む反動技グループの計画）を踏まえつつ、
  とっしん単体のハンドラ構成・data/move.py 定義・エッジケースを整理した。

## 実装（handlers/ / data/）
- `src/jpoke/data/move.py` の `とっしん` の `MoveData`（type/category/pp/power/accuracy/flags）は
  仕様書と一致しており変更なし。
- 共通ヘルパー `_recoil`（`src/jpoke/handlers/move_attack.py`）にバグを発見し修正した：
  みがわりに阻まれてダメージが0になった場合でも `ctx.substitute_damage`（みがわりへの与ダメージ）に
  フォールバックする処理が抜けており、みがわり相手には常に最低反動量（1）しか発生しない誤りがあった。
  ドレイン系の共通ヘルパー `_drain_hp` と同じ `damage = value or ctx.substitute_damage` パターンを追加した。
  この修正は `とっしん` に加え、同じ `_recoil` を使う `アフロブレイク`・`じごくぐるま`・`ボルテッカー`・
  `ワイルドボルト` にも影響する（いずれも正しい方向への修正）。

## リーサル計算
- 対象外（`docs/progress/move.md` の「リーサル実装」列は n/a）。反動技は攻撃者自身のHPのみを変化させ、
  防御側の HP 分布に影響しないためリーサル計算の対象にならない。

  `test_とっしん_反動ダメージが与ダメの4分の1になる`）は実装と一致しており変更なし。
