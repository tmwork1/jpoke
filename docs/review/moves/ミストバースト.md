# ミストバースト レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/ミストバースト.md`）。一次情報（docs/wiki/moves/ミストバースト.html）を
  参照し、`だいばくはつ` / `じばく` 仕様書と同様の構成でHP全消費・ひんし判定・しめりけによる失敗の
  仕様をまとめたうえで、ミストバースト固有の「ミストフィールドの効果を受けている場合威力1.5倍」の
  仕様を追記した。PPは一次情報のGen9本家値（5）ではなく `docs/champions/move_list.txt` の値（8）を
  採用した。

## 実装計画書
- 新規作成（`docs/plan/moves/ミストバースト.md`）。従来は `docs/plan/moves/自爆技.md`
  （じばく・だいばくはつ・ミストバースト共通の暫定計画書）にまとめられていたが、個別レビューのため
  分離した。あわせて `自爆技.md` の冒頭コメントを更新し、ミストバーストが分離済みである旨を明記した。
- ミストフィールドによる威力1.5倍ハンドラの追加を計画書に記載（`Event.ON_CALC_POWER_MODIFIER`、
  `subject_spec="attacker:self"`、priority未指定）。既存の同種ハンドラ（`ワイドフォース_calc_power` /
  `サイコブレイド_calc_power` / `だいちのはどう_power_modifier`）を根拠として明記した。

## 実装（handlers/ / data/）
- 欠落を修正: `handlers/move_attack.py` に `ミストバースト_calc_power` を新規実装し、
  `battle.terrain.name == "ミストフィールド"` かつ `not battle.query.is_floating(ctx.attacker)` の
  場合に `apply_fixed_modifier(value, 6144)`（1.5倍）を適用するようにした。従来は `ON_PAY_HP` による
  HP全消費のみが実装されており、ミストフィールドによる威力上昇が未実装だった。
- 誤りを修正: `data/moves/move_ma.py` の `"ミストバースト"` エントリの `pp` を `5` → `8` に修正した
  （`docs/champions/move_list.txt` のPP統一ルールに未移行のバグだった）。同エントリに
  `Event.ON_CALC_POWER_MODIFIER` ハンドラを追加登録した。
- `ミストバースト_pay_hp`（HP全消費）自体は既存実装のまま変更なし。

## リーサル計算
- 対象外（`docs/progress/move.md` の該当行は「リーサル実装」「リーサルテスト」ともに `n/a`。
  使用者自身がひんしになる技であり、相手への通常特殊ダメージ算出以外に専用ロジックは不要のため）。
