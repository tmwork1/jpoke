# フルールカノン レビュー結果

## 仕様書
- 新規作成（`docs/spec/moves/フルールカノン.md`）。
- 一次情報（ポケモンWiki）に基づき、タイプ・分類・威力130・命中90・PP（本家5→Champions8）・
  直接攻撃なし・命中後に自分の『とくこう』ランクが2段階下がる（確定効果）ことを記載。
  同系統の反動技（オーバーヒート・サイコブースト等）の仕様書と同一パターンで作成した。
- `docs/spec/moves/_move_secondary_1.md` のフルールカノン行はC列（とくこう）が誤って0に
  なっている（同系統技はいずれも-2）ことに気づいたため、仕様書の備考・参照欄に注記した
  （このファイル自体は生成・参照専用のため今回は修正対象外とした）。

## 実装計画書
- 新規作成（`docs/plan/moves/フルールカノン.md`）。
- 既存実装（`フルールカノン_sharply_lower_spa_C`）の構成確認結果と、PP誤り・
  リーサル未実装という2点の問題点、修正方針を記載した。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ha.py` のフルールカノンエントリを `pp=5` → `pp=8` に修正
  （Champions基準のPP統一ルールに準拠。オーバーヒート・サイコブースト等の同系統技と同じ変換）。
- 同エントリに `lethal_handlers={LethalEvent.ON_HIT: LethalHandler(l.フルールカノン_lower_spa)}`
  を新規追加。
- `handlers/move_attack.py` の `フルールカノン_sharply_lower_spa_C`（`modify_attacker_stats`
  経由で `battle.modify_stats()` を呼ぶ実装）は仕様通りで変更なし。

## リーサル計算
- 新規実装。`docs/progress/move.md` のフルールカノン行「リーサル実装」列が `n/a` に
  なっていたが、同系統技（りゅうせいぐん・オーバーヒート・ゴールドラッシュ・サイコブースト）
  がいずれも `x`（実装済み）であり矛盾していたため、新規実装対象として扱った。
- `handlers/lethal.py` に `フルールカノン_lower_spa`（攻撃側のとくこうランクを2段階下げる）
  を新規実装。同系統技と同一パターン。
