## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_sa.py` の `"スケッチ"` エントリを修正した。
  - `pp`: `0` → `1` に修正。旧値`0`はPPが1しかないスケッチの実際のゲーム仕様（Gen9基準）と
    乖離したバグだった。`src/jpoke/data/ps-champ-ja/moves.json` にスケッチのエントリは存在しない
    （ものまね・ゆびをふる等と同様の「技コピー系の技」で未掲載）ため、前例に倣い確認可能な
    第9世代の値（1）を採用。
  - `accuracy`: `100` → 省略（`None`、必中）に修正。旧実装は回避率補正等の影響を受けてしまう
    誤りだった。
  - `flags`: `{"non_encore", "non_negoto"}` → `{"non_encore", "non_negoto", "non_copycat",
    "unprotectable", "unreflectable"}` に修正。
    - `non_copycat`: まねっこ/ものまね/オウムがえしでコピーできない仕様（`docs/spec/moves/
      スケッチ.md`「技の仕様」節）に対応。旧実装は未設定で漏れていた。
    - `unprotectable`: まもる状態を無視して成功する仕様（判定節「まもる: ×」）に対応。旧実装は
      未設定で漏れていた。
    - `unreflectable`: マジックコート状態を無視して成功する仕様（判定節「マジックコート: ×」）
      に対応。旧実装は未設定で漏れていた。
  - `handlers={}` は変更なし。相手の最後に使った技を技スロットへ恒久的に追加する機構は、
    ものまねの一時的な技スロット差し替えよりも復元条件（経験値が入るバトルか否か等）が複雑な
    新規モデル変更を要するため、前例（へんしん・ものまね・ゆびをふる）と同様に実装保留のまま
    とした。判断の詳細は `docs/plan/moves/スケッチ.md`「実装保留の判断」節を参照。
  - `_INSTRUCT_BLOCKED_MOVES`（`src/jpoke/handlers/move_status.py`）には既に「スケッチ」が
    含まれており、さいはい対応は変更不要だった。
