# フェアリーロック レビュー結果

## 仕様書
- `docs/spec/moves/フェアリーロック.md` を、Wiki生データの丸ごと転記（旧形式）から他の技（`じゅうりょく.md` 等）と同じ構造化フォーマットに書き直した。
- PP の記載を `10` → `12`（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）に修正。
- 「まもる無効」「マジックコート無効」の理由（`target="field"` により `Move.is_blocked_by_protect` / `Move.is_reflectable` が自動的に False になる）を明記。
- `docs/spec/fields/フェアリーロック.md`（場の状態側の仕様書）は内容が正確だったため変更なし。

## 実装計画書
- `docs/plan/moves/フェアリーロック.md` は「未実装」を前提にした古い計画書だったため、実装済みの現状に合わせて全面更新した。
  - PP を `10` → `12` に修正。
  - `MoveData.target="field"` を明示する必要がある旨を追記（後述のバグ修正に対応）。
  - ファイルパスを現行の分割構成（`data/moves/move_ha.py`, `data/field/global_field.py`, `handlers/field.py` など）に更新。
  - 「`きれいなぬけがら` との相互作用が要review」だった記述を、`docs/spec/items/きれいなぬけがら.md`（「場がフェアリーロック状態のときでも、所有者は交代できるようになる」と明記）と priority 順（`きれいなぬけがら_check_trapped` が priority=-100 で先に `stop_event=True` を返す）の突き合わせにより「仕様どおりで修正不要」と結論づけた。
  - 「オウムがえしでコピー可能」「`target="field"` の要確認」の未決事項を解消。

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_ha.py` の `"フェアリーロック"` エントリに2つのバグを発見・修正した。
  1. **PP誤り**: `pp=10`（Gen9本家基準の値の移行漏れ）→ `pp=12`（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）に修正。
  2. **`target` 未設定によるまもる/マジックコートの誤動作**: `MoveData.target` が未指定でデフォルト値 `"foe"` のままだったため、`Move.is_blocked_by_protect`（`target == "foe"` が条件）・`Move.is_reflectable`（`category=="status"` かつ `target in ("foe","foe_side")` が条件）が誤って `True` を返し、一次情報の「まもる: ×」「マジックコート: ×」（防がれない・跳ね返されない）に反して実際には防がれる・跳ね返されるバグがあった。同種の全体の場発動技「じゅうりょく」に倣い `target="field"` を明示して修正した。
- `handlers/field.py` の `フェアリーロック_check_trapped`（ゴーストタイプも含め無条件で交代禁止）・`フェアリーロック_tick`（`_tick_global_field` 共通処理を利用）、`handlers/move_status.py` の `フェアリーロック_activate_global_field`、`data/field/global_field.py` の `ON_CHECK_TRAPPED`（priority 100）・`ON_TURN_END`（priority 140）は仕様と一致しており変更なし。
