## 実装計画書

  - サンプルコードが `ctx: EventContext` / `stats={"A": -1, "C": -1}` / `battle.get_available_switch_commands` /
    `result.stop_event` 判定など、実際の実装（`AttackContext` / `stats={"atk": -1, "spa": -1}` /

  - 交代制限の無視に関する記述を、実装修正（後述）に合わせて `battle.query.can_switch` を
    使わず `state.bench` の生存判定を直接行う方式に更新した。
  - のどスプレーの消費タイミングに関する既知の未対応事項を注意点に追記した。

## 実装（handlers/ / data/）
- **バグ修正**: `すてゼリフ_modify_defender_stats_and_pivot`（`handlers/move_status.py`）が
  `battle.query.can_switch(player)` を使って交代可否を判定していたが、`can_switch` は
  とらわれ状態（`is_trapped`）を理由に `False` を返す実装になっている。すてゼリフは
  「使用者は交代を防ぐ効果（ありじごく・ねをはる・かげふみ・じりょく・バインド・
  にげられない・フェアリーロック等）を無視して交代できる」仕様のため、とらわれ状態下でも
  PIVOT が設定されるべきところ、誤って交代がブロックされるバグがあった。
  同じ「とらわれ無視の自発交代」であるバトンタッチ（`バトンタッチ_check` /
  `バトンタッチ_apply`）が `state.bench` の生存有無のみを直接判定している実装パターンに
  合わせ、`any(mon.alive for mon in state.bench)` による直接判定に修正した。
- ランク低下失敗判定（`if not result.value`）・`data/moves/move_sa.py` のエントリ内容
  （type/category/pp/accuracy/flags/handlers）は仕様書・一次情報と一致しており変更なし。
