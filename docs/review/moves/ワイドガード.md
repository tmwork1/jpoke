- PP・優先度・対象範囲（味方の場）・判定フラグ（まもる×／マジックコート×等）・技の仕様

## 実装（handlers/ / data/）
- `src/jpoke/data/moves/move_wa.py` の `ワイドガード` エントリを修正:
  - `pp=10` → `pp=12`（champions基準。docs/champions/move_list.txt 970行目。Gen9本家は10）
  - `target`未指定（デフォルト`"foe"`）→ `target="own_side"` に修正
    - `target`未指定のままだと `Move.is_reflectable` が `True` になり、相手のマジックコートで
      跳ね返される誤った挙動になっていた（一次情報「マジックコート: ×」と矛盾）
    - 同様に `Move.is_blocked_by_protect` も `True` になり、相手のまもるでワイドガードの使用
      自体がブロックされる誤った挙動になっていた（一次情報「まもる: ×」と矛盾）
    - `target="own_side"` は既存の同種技（`リフレクター`・`おいかぜ`・`ハッピータイム`等）と
      同じパターンで、champions記載「対象: 味方の場」に一致する
  - `handlers={}`は維持（本プロジェクトがスプレッド技の対象範囲区分をモデル化していないため。
    詳細は仕様書・計画書参照）
