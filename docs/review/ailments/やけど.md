## 実装（handlers/ / data/）

- **不具合を発見・修正**: `handlers/ailment.py` の `やけど_modifier` が、こんらんの自傷ダメージ
  （`battle.roll_damage(attacker, attacker, "_こんらん")`）に対しても物理技ダメージ半減（0.5倍）を
  誤って適用していた。一次情報によれば第五世代以降はこんらんの自傷ダメージにやけどの影響は無いため、
  Champions仕様として不正な挙動だった。
  - 実際に検証したところ、やけど状態＋こんらん自傷時に `burn_modifier == 2048`（誤って半減）が
    発生することを確認した
  - `ctx.move.name == "_こんらん"` の場合に半減を適用せずそのまま `value` を返すよう修正した
  - 当初、からげんきパターン（`_こんらん` 側の技データに `ON_CALC_BURN_MODIFIER` の
    `MoveHandler` を登録し、優先度で上書きする案）を試みたが、`battle.roll_damage` が
    `Move.register_handlers` を経由しないため技側のハンドラが一切発火せず機能しないことが判明した。
    そのため `やけど_modifier` 側で直接 `ctx.move.name` をチェックする方式に変更した
  - この設計は `こんじょう_modify_atk`（`ability.py`）が同様に `ctx.move.name != "_こんらん"` で
    こんらん自傷ダメージを対象外にしている既存パターンと整合する。修正前は「やけどが誤って
    こんらんダメージを半減する一方、こんじょうはこんらんダメージに攻撃補正をかけない」という
    非対称な状態だったが、修正後は両特性とも一貫してこんらん自傷ダメージを対象外にする
  - `handlers/ailment.py` の `やけど_damage`（`max(1, mon.max_hp // 16)`、`reason="burn"`）、
    `data/ailment.py` の `"やけど"` エントリ（`Event.ON_TURN_END` priority=100、
    `Event.ON_CALC_BURN_MODIFIER` priority=100（デフォルト）、`subject_spec` は
    `AttackContext`/`EventContext` のロールと一致）は仕様と一致しており、他の誤り・欠落は
    見つからなかった
  - タイプ免疫（ほのおタイプ）は `AilmentManager._can_apply_by_type` で実装済みで仕様と一致

## リーサル計算

- 変更なし
  - `docs/progress/ailment.md` の「やけど」行「リーサル実装」列は `x`（実装済み）
  - `handlers/lethal.py` の `やけど_damage`（`LethalEvent.ON_TURN_END`, `subject="defender"`）を確認:
    `max(1, ctx.defender.max_hp // 16)` でダメージ計算しており、通常ハンドラと計算式・最小値保証が
    一致している
  - ぶつりわざ半減（`ON_CALC_BURN_MODIFIER`）は致死率計算専用のハンドラを必要とせず、
    `core/lethal.py` の `_calc_damage_dist` が `battle.calc_damages` を再利用することで通常の
    `Event` システム（＝今回修正した `やけど_modifier` を含む）がそのまま適用される設計になっており、
    誤り・欠落は見つからなかった
  - マジックガード・たいねつ等によるやけどダメージの無効化/軽減は `docs/progress/ability.md` で
    各特性のリーサル列が `n/a` と明記されており、リーサル計算のスコープ外として意図的に未対応
    （どく・もうどくと同様、やけど固有の欠落ではない）
