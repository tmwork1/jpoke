更新日: 2026-07-05

## スコープ

- 仕様書: `docs/spec/items/_タイプ半減きのみ.md`（ヤチェのみ節）
- 実装状態: 実装済み

## 効果概要

- 効果バツグンの`こおり`タイプの技を受けたとき、ダメージを1/2にして消費する（1度きり）。
- フリーズドライ特有の注意点: フリーズドライは`みず`タイプに対して技タイプ相性を無視して
  常に効果抜群になる。技タイプ自体は`こおり`のままのため、ヤチェのみは通常通り
  `calc_def_type_modifier`の結果が抜群（4096超）であれば発動する。
- 特性`あついしぼう`はダメージ計算の攻撃側補正（`ON_CALC_ATK_MODIFIER`）でこおり技のダメージを
  弱める効果であり、タイプ相性（`calc_def_type_modifier`）自体を変えるものではないため、
  ヤチェのみの発動判定には影響しない。

## 実装方針

- 通常戦闘: `Event.ON_CALC_DAMAGE_MODIFIER`（`subject_spec="defender:self"`）で発火。
  タイプ半減きのみ共通処理 `handlers/item.py` の `_modify_super_effective_damage` を
  `type_="こおり"`, `modifier=2048/4096` で呼び出す。
  - `ctx.move.type == "こおり"` かつ `battle.damage_calculator.calc_def_type_modifier(ctx) > 4096`
    で効果バツグン判定する。
  - priority: 他のタイプ半減きのみ・ダメージ補正系アイテムと同じ既定値（`docs/spec/turn.md` に
    `ON_CALC_DAMAGE_MODIFIER` 個別の優先度記載なし。ダメージ計算内部イベントのため、
    既存の同種きのみハンドラの priority に合わせてデフォルト値を使用）。
  - 条件成立時に `_announce_and_consume_item` でログ記録とアイテム消費を行う。
- リーサル計算（簡易KO判定用の `LethalEvent`）: `LethalEvent.ON_BEFORE_HIT`（`subject="defender"`）で
  `handlers/lethal.py` の `ヤチェのみ_resist_ice` → 共通処理 `_type_resist_berry(..., "こおり")` を呼び出す。
  通常ハンドラ側で既にダメージ半減済みのため、リーサル側ではアイテム消費と `item_enabled` の更新のみ行う。

## テスト

- `tests/test_item.py` の `test_タイプ半減実`（パラメータ化テスト）で
  `("ヤチェのみ", "こおり", "ミニリュウ")` を検証。
  - `damage_modifier == 2048`（1/2倍）であること。
  - 使用後にアイテムが消費されていること（`not has_item()`）。
- `test_ヤチェのみ_フリーズドライのみずタイプに対する効果抜群でも発動する`
  みずタイプ（カメックス）がフリーズドライ（みずタイプに抜群）を受けたとき発動することを確認。
- `test_ヤチェのみ_あついしぼうの影響下でも発動する`
  くさ/どく複合タイプ（フシギダネ）に`あついしぼう`を持たせても、ふぶき（こおり抜群）を
  受けたとき発動することを確認。
