# きあいのタスキ 実装計画

## 効果概要
フルHP状態から OHKO（一撃ひんし）ダメージを受けたとき、HP1 で耐える（消費）。HP が満タンでない場合は発動しない。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_MODIFY_MOVE_DAMAGE` |
| subject_spec | `"defender:self"` |
| priority | 100（`docs/spec/turn.md` の `Event.ON_MODIFY_DAMAGE` 節に「100 \| きあいのタスキ」と明記されている） |
| once | False（アイテム消費は `_announce_and_consume_item` が担うため独自の once 管理は不要） |
| 消費条件 | 発動時に `_announce_and_consume_item(battle, mon)`（アイテム表示 + `battle.item_manager.consume_item(mon)`） |

### ハンドラロジック（疑似コード）
```python
def きあいのタスキ_survive_ohko(battle, ctx, value):
    mon = ctx.defender
    # HP が満タン かつ ダメージがHP以上の場合のみ発動
    if mon.hp == mon.max_hp and value >= mon.hp:
        value = mon.hp - 1
        _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)
```

### 一撃必殺技（OHKO）との優先度の整合（バグ修正）

`docs/spec/turn.md` の `Event.ON_MODIFY_DAMAGE` 節には以下の優先度が明記されている。

| Priority | 処理 |
|----------|------|
| 20 | みがわりに被弾 |
| 40 | ばけのかわ／アイスフェイスに被弾 |
| 60 | こらえる／みねうち・てかげん |
| 100 | がんじょう／きあいのタスキ／きあいのハチマキ |

一方、一撃必殺技（じわれ・ぜったいれいど・つのドリル・ハサミギロチン）の確定ダメージを
設定する `ha.ohko_damage`（`src/jpoke/data/move.py`）は turn.md に明記されておらず、
`MoveHandler` のデフォルト priority=100 のまま登録されていた。

これにより `ha.ohko_damage`（subject_spec 既定の `"attacker:self"`、素早さ=攻撃側）と
`きあいのタスキ_survive_ohko`（`subject_spec="defender:self"`、素早さ=防御側）が
同一 priority=100 で衝突し、`EventManager._sort_handlers` の
`(priority, -speed)` によるタイブレークで実行順が素早さ依存になっていた。
防御側（きあいのタスキ所持者）が攻撃側より素早い場合、
`きあいのタスキ_survive_ohko` が `ha.ohko_damage` より先に実行され、
確定ダメージがまだ設定されていない値で判定してしまい発動しない
→ その後 `ha.ohko_damage` が防御側の HP 全量で上書きしてしまい、
きあいのタスキが一撃必殺技を防げないバグが発生していた
（おうじゃのしるしレビューで指摘された不具合）。

**対応**: `ha.ohko_damage` の登録に `priority=90` を明示する
（既存仕様の「こらえる／みねうち」の 60 より後、
「がんじょう／きあいのタスキ／きあいのハチマキ」の 100 より前という、
未掲載イベントに対する既存同種ハンドラ準拠の判断）。
これにより一撃必殺技の確定ダメージが必ず先に確定してから
きあいのタスキ等の「致死ダメージをHP1に補正する」判定が行われるようになり、
素早さに関係なく安定して発動するようになる。
なお `みがわり`(20)・`ばけのかわ`/`アイスフェイス`(40)・`こらえる`/`みねうち`(60) は
priority=90 より前のままであり、既存の優先度関係・挙動は変更しない
（それらと一撃必殺技の相互作用は本レビューの対象外）。

## テスト観点
- フルHP状態から OHKO ダメージを受けたとき HP1 で耐えること
- HP が満タンでない場合（半減状態等）は発動しないこと
- 発動後にアイテムが消費されること
- 消費後に再度 OHKO を受けたとき耐えないこと
- すなあらし・まきびし等の継続ダメージには発動しないこと（ON_MODIFY_MOVE_DAMAGE は技ダメージのみ）
- 一撃必殺技（じわれ等）を受けたとき、防御側（きあいのタスキ所持者）が攻撃側より素早い場合でも
  HP1 で耐えること（素早さ依存で発動しない旧バグの回帰防止）
