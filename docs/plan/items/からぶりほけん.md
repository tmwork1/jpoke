# からぶりほけん 実装計画

## 効果概要
持たせたポケモンが使用した技が命中判定に失敗したとき、すばやさ（S）を +2 する（1度発動すると消費）。

## 実装仕様

技の命中判定失敗は `MoveExecutor._execute_move` 内で `_check_hit(ctx)` が False を返した場合に
`Event.ON_MISS` が発火される（`docs/spec/turn.md` に ON_MISS の明記はないため、命中判定失敗直後に
即時発火される核心ロジックとして扱う。タイプ相性・まもる等による無効化は `ON_TRY_MOVE_*` 系で
`move_success = False` として処理され、`ON_MISS` は発火されないため、命中判定失敗時のみ発動する
仕様と自然に一致する）。

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_MISS`（`MoveExecutor` が命中判定失敗時に発火） |
| subject_spec | `"attacker:self"` |
| priority | 100（デフォルト） |
| once | False（`battle.modify_stats` の戻り値を見て消費要否を判定するため、ハンドラ内で個別に消費処理する） |
| 消費条件 | すばやさランクが実際に上昇した場合のみ `_announce_and_consume_item` で消費する |

### 除外条件
- 一撃必殺技（`ctx.move.has_flag("ohko")`）を外したときは発動しない。
- 連続技（`ctx.hit_index != 1`）は最初の1発以外の命中判定失敗では発動しない。
  トリプルキック/トリプルアクセル/ネズミざん等の `check_hit_each_time` 技も同様に、
  1発目が外れたときのみ発動する。
- すでにすばやさランクが最大（+6）のときは `battle.modify_stats` の戻り値が空になるため、
  自然に消費されない。

### ハンドラロジック
```python
def からぶりほけん_boost_speed_on_miss(battle, ctx, value):
    mon = ctx.attacker
    if ctx.move.has_flag("ohko"):
        return HandlerReturn(value=value)
    if ctx.hit_index != 1:
        return HandlerReturn(value=value)
    changes = battle.modify_stats(mon, {"spe": +2})
    if changes:
        _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)
```

### 対象外の詳細仕様（シングルバトル対象外のため実装不要）
- 全体攻撃技・ドラゴンアロー・わたげ/ミラーアーマー/レッドカード/わるいてぐせ等との
  相互作用は、いずれも複数体を同時に相手取るダブルバトル前提の仕様であり、
  本プロジェクトはシングルバトルのみを対象とするため対応不要。
- マジックコートによる跳ね返りは `MoveExecutor._execute_move` 内で
  `ctx.attacker, ctx.defender` が入れ替わる既存の汎用処理により、
  跳ね返した側が `ctx.attacker` となるため、からぶりほけん固有の追加実装は不要。

## テスト観点
- 技が外れたとき、すばやさが +2 されること
- 技が命中したときは発動しないこと
- 発動後にアイテムが消費されること
- すでにすばやさランクが最大のときは発動しない（アイテムも消費されない）こと
- 一撃必殺技を外したときは発動しないこと
- 連続技で1発目が命中し途中の1発が外れた場合は発動しないこと（`check_hit_each_time` 技）
