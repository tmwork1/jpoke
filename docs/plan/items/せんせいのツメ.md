# せんせいのツメ 実装計画

## 効果概要
20% の確率で、同じ優先度の行動の中で最初に行動できるようになる（後攻ティア +1）。
クイックドロウ特性と同パターン。非消費アイテム。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `DomainEvent.ON_CALC_BACK_TIER` |
| subject_spec | `"attacker:self"` |
| priority | 未指定（デフォルト。クイックドロウ特性の登録と同様） |
| once | False |
| 消費条件 | なし（非消費アイテム） |

### ハンドラロジック（疑似コード）
```python
def せんせいのツメ_priority_boost(battle, ctx, value):
    # 20% の確率で先制（+1）
    if battle.random.random() < 0.2:
        return HandlerReturn(value=value + 1)
    return HandlerReturn(value=value)
```

## テスト観点
- せんせいのツメ発動時（確率）、行動順ティアが +1 されること
- 発動しない場合は行動順が変化しないこと
- クイックドロウ特性と同パターンで実装できること
- 乱数を固定したテストで発動・非発動の両ケースを確認すること
