# タラプのみ 実装計画

## 効果概要
HP が最大値の 25% 以下になった瞬間にフラグを立て（消費）、次に使う技で先制する（イバンのみと同効果）。

## 実装仕様

### イベント 1: HP 閾値トリガー（フラグ付与）

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_HP_CHANGED` |
| subject_spec | `"target:self"` |
| priority | 100 |
| once | True（消費アイテム） |
| 消費条件 | 発動時に `battle.consume_item(mon)` |

### イベント 2: 行動順の先制付与

| 項目 | 内容 |
|------|------|
| イベント | `DomainEvent.ON_CALC_BACK_TIER` |
| subject_spec | `"attacker:self"` |
| priority | 100 |
| once | False |

### ハンドラロジック（疑似コード）
```python
def タラプのみ_set_priority_flag(battle, ctx, value):
    mon = ctx.target
    hp_after = mon.hp
    hp_before = hp_after + value
    if hp_before * 4 > mon.max_hp and hp_after * 4 <= mon.max_hp:
        mon.item.data["custap_active"] = True
        battle.consume_item(mon)
        _announce_item_triggered(battle, mon)
    return HandlerReturn(value=value)


def タラプのみ_boost_priority(battle, ctx, value):
    mon = ctx.attacker
    if mon.item.data.get("custap_active"):
        mon.item.data["custap_active"] = False  # 1回限り
        return HandlerReturn(value=value + 1)
    return HandlerReturn(value=value)
```

## 注意
- イバンのみ（Custap Berry）と同効果の可能性あり。日本語名の対応を要確認。
- フラグ管理（`mon.item.data` 利用可否）を既存実装と合わせる。

## テスト観点
- HP が 25% 超 → 25% 以下になった瞬間にフラグが立ち、アイテムが消費されること
- フラグが立った後、次のターンに先制（行動ティア +1）で行動すること
- 先制後はフラグが解除され、2ターン目は通常行動順になること
