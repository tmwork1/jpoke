# するどいキバ 実装計画

## 効果概要
ダメージ技命中時、10% の確率でひるみを付与する（非消費）。おうじゃのしるしと同効果。
なげつけるで使用した場合、威力30・相手を100%の確率でひるませる。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_DAMAGE_HIT` |
| subject_spec | `"attacker:self"` |
| priority | 100（デフォルト値。turn.md 未掲載の内部イベントのため、既存の同種アイテムハンドラ（`おうじゃのしるし`等）に合わせてデフォルトを採用） |
| once | False |
| 消費条件 | なし（非消費アイテム） |

- `Event.ON_DAMAGE_HIT` は `actual_damage<=0` のとき発火しないため、
  「技のダメージが無効化されたとき」「みがわりに防がれたとき」は自然に効果が発動しない。

### ハンドラロジック（疑似コード）
```python
_INNATE_FLINCH_MOVES = frozenset({...})  # 元々ひるみの追加効果を持つ技名（アイアンヘッド等）

def flinch_on_hit_10pct(battle, ctx, value):
    defender = ctx.defender
    if (
        ctx.move.is_attack
        and defender is not None
        and not ctx.move.has_flag("ohko")          # 一撃必殺技には効果が無い
        and ctx.move.name not in _INNATE_FLINCH_MOVES  # 元々ひるみ効果がある技には重複しない（第五世代以降の仕様）
    ):
        chance = battle.resolve_secondary_chance(ctx, 0.1)  # りんぷんで無効化・てんのめぐみで2倍
        if battle.random.random() < chance:
            battle.volatile_manager.apply(defender, "ひるみ", source=ctx.attacker)
    return HandlerReturn(value=value)
```

- `おうじゃのしるし` と同一の関数 `flinch_on_hit_10pct` を共有し、`data/item.py` の
  `するどいキバ` エントリからも同じハンドラを登録する（関数の重複実装はしない）。
- `battle.resolve_secondary_chance` は `Event.ON_MODIFY_SECONDARY_CHANCE` を発火し、
  りんぷん（`りんぷん_block_secondary_chance`: chance=0 で無効化）・
  てんのめぐみ（`てんのめぐみ_boost_secondary_chance`: chance×2）の影響を反映する。

### なげつける連携
`handlers/move_attack.py` の `なげつける_apply_item_effect` に分岐がある：
```python
elif item_name in ("おうじゃのしるし", "するどいキバ"):
    battle.volatile_manager.apply(ctx.defender, "ひるみ", source=ctx.attacker)
```
（`resolve_secondary_chance` を通さず無条件で付与 = 100%）

## テスト観点
- ダメージ技命中時、10% の確率で相手がひるむこと
- 変化技では発動しないこと
- 一撃必殺技には効果が無いこと
- 元々ひるみの追加効果がある技（例: エアスラッシュ）には重複して効果が発動しないこと
- りんぷん所持者には効果が発動しないこと
- てんのめぐみ所持者は確率が2倍（20%）になること
- なげつけるでするどいキバを使用した場合、100%の確率でひるませること
- おうじゃのしるしと同パターンで実装できること（ハンドラ関数を共有）
