# ねらいのまと 実装計画

## 効果概要
タイプ相性による無効（0 倍）を無効化する。例: ゴーストタイプへのノーマル技が当たるようになる（非消費）。
無効化以外の相性補正（弱点・耐性）には一切影響しない。じめん技に対する浮遊由来の無効化
（フリーフォール）や特性由来の耐性には効果が及ばない。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_CALC_DEF_TYPE_MODIFIER` |
| subject_spec | `"defender:self"` |
| priority | -100（`Handler.priority` はソート昇順で先に実行される。他のタイプ相性ハンドラ（デフォルト優先度100）より先に実行して補正済みの基礎値を確定させ、後続ハンドラ〔フリーズドライ等の追加乗算〕はその値にさらに乗算する） |
| once | False |
| 消費条件 | なし（非消費アイテム） |

### ハンドラロジック（疑似コード）

`damage.py` の `calc_def_type_modifier` は、防御側の複数タイプの相性倍率をあらかじめ
1つの `int` に掛け合わせてから `ON_CALC_DEF_TYPE_MODIFIER` イベントを発火するため、
一度 0 倍に潰れた後では「どのタイプが無効化要因だったか」を区別できない
（例: ノーマル・エスパー複合であれば `4096 * 0.0(ノーマル) * 2.0(エスパー) = 0` となり、
エスパー側の弱点情報が失われる）。
そのため、`きもったま`/`しんがん` と同様に `TYPE_MODIFIER` を用いて防御側タイプごとに
再計算し、タイプ単位で 0 倍のみを 1 倍に置き換えてから掛け合わせる。

```python
def ねらいのまと_negate_immunity(battle, ctx, value):
    move_type = ctx.move.type
    if move_type == "ステラ" and ctx.defender.terastallized:
        return HandlerReturn(value=value)  # 抜群化は無効化ではないため対象外

    type_chart = TYPE_MODIFIER.get(move_type, {})
    if move_type == "じめん":
        if battle.query.is_floating(ctx.defender):
            # 浮遊由来の無効化（フリーフォール）はねらいのまとの対象外
            return HandlerReturn(value=value)
        type_chart = type_chart.copy()
        type_chart["ひこう"] = 1.0  # 接地しているひこうタイプは通常通り等倍

    base = 4096
    for def_type in ctx.defender.types:
        rate = type_chart.get(def_type, 1.0)
        if rate == 0.0:
            rate = 1.0  # タイプ相性による無効化のみを無効化する
        base = int(base * rate)
    return HandlerReturn(value=base)
```

## 注意
- ぶきよう・さしおさえ・マジックルームによるアイテム無効化は `handler.py` の
  ハンドラ有効性チェック（アイテム発動元の汎用無効化判定）で自動的に処理されるため、
  本ハンドラ側で個別に判定する必要はない。
- きけんよちの判定・技選択画面の「こうかなし」表示には影響しない（本アイテムはダメージ計算
  内部のみに作用するため、既存の判定ロジックには手を加えない）。
- でんじは（電気タイプの状態異常技）がじめんタイプに通常無効化される仕様は、
  `でんじは` の技実装側に免疫判定が未実装のため本アイテム側では対応しない
  （`docs/progress/move.md` のでんじは行を参照。技側の別課題）。

## テスト観点
- ねらいのまと持ちに対して通常無効になる技が命中すること（単タイプ）
- 複合タイプで無効化タイプ以外の相性（弱点等）はそのまま反映されること
  （ノーマル・エスパータイプがゴースト技を受けて効果抜群になる例）
- 浮遊（特性・道具・自然なひこうタイプ）によるじめん技無効化には効果が及ばないこと
- 特性ぶきようを持つ場合は効果が無効化され、通常通り無効になること
- ねらいのまとがない場合は通常通り無効になること
