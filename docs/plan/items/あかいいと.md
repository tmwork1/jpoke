# あかいいと 実装計画

更新日: 2026-06-12

## スコープ

- 仕様書: `docs/spec/items/あかいいと.md`
- 実装状態: 未実装
- 実装方針: `ON_VOLATILE_START` でメロメロが付与された持ち主を検知し、`battle.foe()` 経由で攻撃側にもメロメロを返す

## 実装ゴール（完了定義）

- 持ち主がメロメロになったとき、相手にもメロメロが付与される
- 相手がどんかん・アロマベールなら付与されない（ON_BEFORE_APPLY_VOLATILE で既存処理が担当）
- 相手がすでにメロメロ状態なら付与されない
- 持ち主がメロメロになっても消費されない

## 実装対象イベント

- `Event.ON_VOLATILE_START` — volatile が付与された直後に発火。`value` に揮発状態名が入る

## 疑似コード

```python
def あかいいと_infatuate_foe(battle, ctx, value):
    # value は揮発状態名。メロメロ以外は無視
    if value != "メロメロ":
        return HandlerReturn(value=value)
    foe = battle.foe(ctx.source)
    # シングルバトルでは foe が攻撃側。apply_volatile が どんかん等を検査する
    battle.apply_volatile(foe, "メロメロ", source=ctx.source)
    return HandlerReturn(value=value)
```

登録:
```python
"あかいいと": ItemData(
    fling_power=10,
    handlers={
        Event.ON_VOLATILE_START: h.ItemHandler(

        )
    }
),
```

（`ItemData` に `consumable` フィールドは存在しない。消費処理を呼び出さないことで「消費されない」を担保する。）
