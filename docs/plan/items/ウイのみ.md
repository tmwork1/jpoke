# ウイのみ 実装計画

## 効果概要
HP が最大値の 25% 以下になった瞬間に最大 HP の 33%（端数切り捨て）を回復する（消費）。
回復と同時に、とくこうが上がりにくい性格（しぶい味が嫌い）のポケモンはこんらんする。
フィラのみ・マゴのみ・バンジのみ・イアのみと回復量・発動条件は同じで、嫌う味と対応する性格が異なるのみ。

※旧計画書には「SV では混乱しない」と記載されていたが誤り。ウイのみのwikiページの
説明文（第六〜九世代、値段・効果節）では一貫して「きらいなあじだとこんらんする」と
明記されており、現行の対象タイトル（SV）でもこんらん効果は健在（docs/spec/items/_HP回復ピンチきのみ.md
のウイのみ節・詳細な仕様を参照）。本計画書はその修正版。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_HP_CHANGED` |
| subject_spec | `"target:self"` |
| priority | 100（`docs/spec/turn.md` に ON_HP_CHANGED の個別記載はなし。既存の同種ハンドラ `heal_on_quarter_hp` 等が priority=100（`ItemHandler` のデフォルト）で登録されているため、それに合わせる） |
| once | False（`ItemHandler` は消費アイテムでも once を使わず、`battle.item_manager.consume_item` で明示的に消費する既存の共通パターンに合わせる） |
| 消費条件 | 発動時に `_announce_and_consume_item(battle, mon)`（内部で `battle.item_manager.consume_item` を呼ぶ既存の共通処理） |
| こんらん付与 | 発動時、`mon.nature` が `("いじっぱり", "わんぱく", "ようき", "しんちょう")` のいずれかなら `battle.volatile_manager.apply_confusion(mon, source=mon)` |

### 性格の根拠
しぶい味はとくこう（特攻）に対応する（イアのみ計画書のすっぱい=ぼうぎょ、バンジのみ実装のにがい=とくぼう
と同様、`src/jpoke/data/nature.py` の `NATURE_MODIFIER` でとくこうが0.9倍（下降）になっている性格を
「しぶい味が嫌いな性格」とする）。該当性格は以下の4つ：
- いじっぱり: `[1.0, 1.1, 1.0, 0.9, 1.0, 1.0]`（こうげき↑・とくこう↓）
- わんぱく: `[1.0, 1.0, 1.1, 0.9, 1.0, 1.0]`（ぼうぎょ↑・とくこう↓）
- ようき: `[1.0, 1.0, 1.0, 0.9, 1.0, 1.1]`（すばやさ↑・とくこう↓）
- しんちょう: `[1.0, 1.0, 1.0, 0.9, 1.1, 1.0]`（とくぼう↑・とくこう↓）

### ハンドラロジック

共通処理 `_heal_berry`（`handlers/item.py`）は既にイアのみのレビューで `confuse_natures` パラメータが
追加済み。ウイのみ専用のラッパー関数 `ウイのみ_heal_on_quarter_hp` を追加し、しぶい味を嫌う性格タプルを渡す。
`data/item.py` の `"ウイのみ"` エントリの `Event.ON_HP_CHANGED` ハンドラを、汎用の `h.heal_on_quarter_hp`
からこの専用ラッパーに差し替える。
他の3種（フィラ・マゴ・バンジ）は本レビューの対象外のため、既存の共通関数 `heal_on_quarter_hp`
（こんらん未対応）のまま変更しない。将来的にそれぞれのレビュー時に同様の専用ラッパーへ切り替える想定。

```python
def ウイのみ_heal_on_quarter_hp(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """ウイのみ: HPが1/4以下になった瞬間に最大HPの1/3を回復するが、
    とくこうが上がりにくい性格（いじっぱり・わんぱく・ようき・しんちょう）は
    しぶい味を嫌うためこんらんする。
    """
    return _heal_berry(
        battle, ctx, value, denominator=4, heal_r=1/3,
        confuse_natures=("いじっぱり", "わんぱく", "ようき", "しんちょう"),
    )
```

`data/item.py` 側の変更（差分イメージ）:
```python
"ウイのみ": ItemData(
    fling_power=10,
    handlers={
        Event.ON_HP_CHANGED: h.ItemHandler(
            h.ウイのみ_heal_on_quarter_hp,
            subject_spec="target:self",
        )
    },
    lethal_handlers={
        LethalEvent.ON_EVERY_EVENT: LethalHandler(
            func=l.ウイのみ_heal,
            subject="defender",
        )
    }
),
```

- こんらんの発動条件チェック（どんかん／ミストフィールド等の無効化）は `volatile_manager.apply`
  内部の `ON_BEFORE_APPLY_VOLATILE` フックが担うため、本ハンドラ側で個別に判定しない
  （CLAUDE.md の方針・既存の `どくくぐつ_confuse_on_poison` や `イアのみ_heal_on_quarter_hp` と同じ扱い）。
- `source=mon`（自傷）として付与することで、しんぴのまもり等の「相手由来のみ防ぐ」ガードに
  誤って阻止されないようにする（`ctx.is_foe_target()` が false になるため防がれない＝実機仕様と一致）。
- lethal 計算（`handlers/lethal.py` の `ウイのみ_heal`）はHP分布のみを扱うため、こんらん付与は
  対象外（HPを直接変えない効果のため、既存の他アイテムと同様に lethal 側では未実装のままでよい）。

## テスト観点
- HP が 25% 超 → 25% 以下になった瞬間に 33% 回復すること
- とくこうが上がりにくくない性格（例: まじめ）ではこんらんしないこと
- とくこうが上がりにくい性格（いじっぱり・わんぱく・ようき・しんちょう）のとき、発動と同時にこんらんすること
- 発動後にアイテムが消費されること
