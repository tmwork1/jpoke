# じゅうでんち 実装計画

## 効果概要
でんき技でダメージを受けたとき、こうげき（A）を +1 する（消費）。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_DAMAGE_HIT` |
| subject_spec | `"defender:self"` |
| priority | 100（`ItemHandler` のデフォルト） |
| once | False（`_announce_and_consume_item` で都度アイテムを消費する消費型アイテム） |
| 消費条件 | `battle.modify_stats` が実際に変化を返したときのみ `_announce_and_consume_item(battle, mon)` を呼ぶ |

### ハンドラロジック（疑似コード）
```python
def じゅうでんち_boost_atk_on_electric_hit(battle, ctx, value):
    mon = ctx.defender
    if ctx.move.type == "でんき":
        changes = battle.modify_stats(mon, {"atk": +1})
        if changes:  # ランク上限などで不発の場合は消費しない
            _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)
```

`きゅうこん` / `ひかりごけ` / `ゆきだま` と共通の `_boost_stat_on_type_hit` ヘルパーを利用する。

### たんじゅん・あまのじゃく・かたやぶりとの関係（フレームワークで自動的に処理される）
- `battle.modify_stats` は `ON_BEFORE_MODIFY_STAT` を発火し、たんじゅん（`たんじゅん_double_stat`）・あまのじゃく
  （`あまのじゃく_reverse_stat`）がそこで能力変化量を2倍・反転するため、じゅうでんち側で個別対応は不要。
- こうげきランクがすでに最大（あまのじゃくの場合は最小）まで変化している場合、`modify_stats` の戻り値が空になるため
  発動せず、アイテムも消費されない。
- かたやぶり（`mold_breaker_ignorable` フラグを持つ特性を無効化）は `ON_BEGIN_MOVE`～`ON_END_MOVE` の間、防御側の
  たんじゅん・あまのじゃくを無効化する。じゅうでんちが発動する `ON_DAMAGE_HIT` はこの区間内で発火するため、
  かたやぶり技に対しては自動的にたんじゅん・あまのじゃくの効果が発動せず、通常通り +1 される。

## テスト観点
- でんき技でダメージを受けたとき A が +1 されること
- でんき以外の技では発動しないこと
- 発動後にアイテムが消費されること
- でんき免疫ポケモン（でんきタイプ等）では ON_DAMAGE_HIT が発火しないため発動しないこと
- こうげきランクが最大のときは発動せず、アイテムも消費されないこと
- 所持者がたんじゅんのとき、こうげきランクが2段階上がること
- 所持者があまのじゃくのとき、こうげきランクが1段階下がること
- 所持者があまのじゃくでこうげきランクが最小のとき、発動しないこと
- かたやぶりの効果を持つでんき技を受けた場合、たんじゅん・あまのじゃくでもこうげきランクが1段階上がること
