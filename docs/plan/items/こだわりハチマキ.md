# こだわりハチマキ 実装計画

## 効果概要
物理技のこうげきを1.5倍にする。最初に選んだ技しか使用できなくなる（こだわりロック）。

## 実装仕様

| 項目 | 内容 |
|------|------|
| 仕様書 | `docs/spec/items/_こだわり系.md` |
| 実装状態 | **実装済み** |
| イベント① | `Event.ON_CALC_ATK_MODIFIER` — 物理技のATK補正を1.5倍 |
| subject_spec① | `"attacker:self"` |
| イベント② | `Event.ON_MOVE_END` — 技使用後にこだわりロックを付与 |
| subject_spec② | `"attacker:self"` |
| priority | 100 |
| once | False |
| 消費条件 | なし（非消費アイテム） |

### ハンドラロジック
```python
def こだわりハチマキ_boost_physical(battle, ctx, value):
    # こんらんの自傷ダメージ（move.name == "_こんらん"）には適用しない
    # （第五世代以降の仕様。docs/spec/items/_こだわり系.md「詳細な仕様」参照）
    if ctx.move.category == "物理" and ctx.move.name != "_こんらん":
        value = apply_fixed_modifier(value, 6144)  # ×1.5
    return HandlerReturn(value=value)

def こだわり_lock_move(battle, ctx, value):  # こだわりスカーフと共通
    mon = ctx.attacker
    if not mon.has_volatile("こだわり"):
        battle.volatile_manager.apply(mon, "こだわり", source=mon, move_name=ctx.move.name)
    return HandlerReturn(value=value)
```

### 詳細な仕様に基づく追加確認事項
- イカサマ使用時: `_calc_final_attack` は防御側の攻撃実数値を使うが、
  `ON_CALC_ATK_MODIFIER` は攻撃側（こだわりハチマキ所持者）視点で発火するため、
  `subject_spec="attacker:self"` のまま補正がかかる（追加実装不要、既存挙動で仕様どおり）。
- イカサマを受ける側の場合: 所持者は防御側であり `ctx.attacker`（真の攻撃者）ではないため、
  `subject_spec="attacker:self"` により発動しない（追加実装不要）。
- こんらんの自傷ダメージ: 第四世代までは補正がかかるが、第五世代以降（Champions対応範囲）は
  補正がかからない。ハンドラ内で `ctx.move.name != "_こんらん"` を条件に加えて対応する。

## テスト観点
- こだわりハチマキ所持時、物理技のダメージが1.5倍になること
- 特殊技には補正がないこと
- 技を使用した後、同じ技しか選択できなくなること
- イカサマ使用時も攻撃補正が1.5倍になること
- イカサマを受ける側の場合は補正がかからないこと
- こんらんの自傷ダメージには補正がかからないこと
