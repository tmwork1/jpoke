# ブーストエナジー 実装計画

更新日: 2026-06-12

## スコープ

- 仕様書: `docs/spec/items/ブーストエナジー.md`
- 実装状態: 部分実装
- 実装方針: `ON_ITEM_GAINED` ハンドラを追加して `ON_REFRESH_PARADOX_BOOST` を発火する。登場時・天候変化時の既存能力ハンドラ（`ability_paradox.py`）とあわせてブーストを完結させる

## 現在の実装状態

- `ability_paradox.py`: `refresh_paradox_charge_state` がこだいかっせい/クォークチャージの `ON_SWITCH_IN`・`ON_FIELD_CHANGE`・`ON_REFRESH_PARADOX_BOOST` で呼ばれ、ブーストエナジーの消費を処理済み。
- `data/item.py`: `ブーストエナジー` に `handlers` が未登録。バトル中にブーストエナジーを新たに取得したとき（`ON_ITEM_GAINED`）の再判定が欠けている。

## 実装ゴール（完了定義）

- バトル中にブーストエナジーを取得したとき（消費後きょうせい等で再取得）、即座にブーストが発動する
- 場がにほんばれ（こだいかっせい）またはエレキフィールド（クォークチャージ）の場合はブーストエナジーは発動しない
- マジックルーム中は発動しない
- こだいかっせい/クォークチャージを持たないポケモンが取得しても何も起きない

## 実装対象イベント

- `Event.ON_ITEM_GAINED` — アイテムを新たに保持したとき。`ON_REFRESH_PARADOX_BOOST` を再発火する

## 疑似コード

```python
def ブーストエナジー_on_item_gained(battle, ctx, value):
    mon = ctx.source
    # 能力側のハンドラ（ON_REFRESH_PARADOX_BOOST）に処理を委譲する
    battle.events.emit(Event.ON_REFRESH_PARADOX_BOOST, EventContext(source=mon), None)
    return HandlerReturn(value=value)
```

登録:
```python
"ブーストエナジー": ItemData(
    consumable=True,
    removable=False,
    fling_power=30,
    handlers={
        Event.ON_ITEM_GAINED: h.ItemHandler(
            h.ブーストエナジー_on_item_gained,
            subject_spec="source:self",
        )
    }
),
```

## リスクと対策

- リスク: `removable=False` のため通常はバトル中にブーストエナジーが渡ることはないが、消費後に `きょうせい` で再取得するケースが存在する。
- 対策: ON_ITEM_GAINED ハンドラで再判定をかける実装で対応する。
