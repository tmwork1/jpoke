# かいがらのすず 実装計画

## 効果概要
攻撃技命中後、与えたダメージの 1/8 を HP として回復する（吸収技とは別枠）。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_HIT` |
| subject_spec | `"attacker:self"` |
| priority | 160（turn.md の Event.ON_DAMAGE「160 かいがらのすずの回復」に合わせる） |
| once | False |
| 消費条件 | なし（非消費アイテム） |

### イベント選定の根拠
turn.md では「かいがらのすずの回復」は priority=160 で `Event.ON_DAMAGE`
（実装上の `Event.ON_DAMAGE_HIT` に相当）に掲載されている。
しかし `ON_DAMAGE_HIT` は `actual_damage <= 0` のとき発火しない
（`src/jpoke/core/move_executor.py` の `_execute_hit` 参照）。
みがわりに阻まれた場合、本体への実ダメージは 0 になるため `ON_DAMAGE_HIT` は発火しない。
かいがらのすずは第五世代以降、みがわりに阻まれた場合も
「みがわりへの与ダメージ」から回復量を算出する仕様（詳細な仕様参照）のため、
`ON_DAMAGE_HIT` ではこの挙動を満たせない。
そのため、常に発火する `Event.ON_HIT`（`docs/spec/turn.md` の「ダメージ発生後の処理
（みがわりに被弾しても発動）」）を採用し、priority のみ turn.md の値（160）に合わせる。
これは `いのちのたま_recoil`（`src/jpoke/data/item.py` 内のコメント参照）と同じ判断。

### ハンドラロジック（疑似コード）
```python
def かいがらのすず_drain_on_hit(battle, ctx, value):
    # ちからずく所持者が追加効果ありの技を使った場合は回復効果が無くなる
    if (
        ctx.attacker.ability.name == "ちからずく"
        and ctx.move.has_flag("secondary_effect")
    ):
        return HandlerReturn(value=value)

    # みがわりに阻まれた場合（value=0）は substitute_damage を使う
    damage = value or ctx.substitute_damage
    total_damage = getattr(ctx, "_shell_bell_total_damage", 0) + damage

    # 連続攻撃技は最後のヒットでまとめて回復する。
    # ひんしで中断した場合、その時点のヒットを「最後のヒット」とみなす。
    is_last_hit = (
        ctx.hit_index == ctx.hit_count
        or ctx.defender.fainted
        or ctx.attacker.fainted
    )
    if not is_last_hit:
        ctx._shell_bell_total_damage = total_damage
        return HandlerReturn(value=value)

    heal_amount = total_damage // 8
    if heal_amount > 0 and battle.modify_hp(ctx.attacker, v=heal_amount):
        _announce_item_triggered(battle, ctx.attacker)
    return HandlerReturn(value=value)
```

### 仕様との対応関係
- 「与えたダメージが0だった場合は回復しない」
  → `value`（本体への実ダメージ）と `ctx.substitute_damage` がともに 0 の場合、
    `heal_amount = 0 // 8 = 0` となり `battle.modify_hp` を呼ばないため回復しない。
    ばけのかわ・アイスフェイス・HP1相手へのみねうちはいずれも
    `value=0` かつみがわり無関与のため、この分岐で正しく回復しない。
- 「みがわりに阻まれた場合も回復できる（第五世代以降）」
  → `ctx.substitute_damage`（`みがわり_block_damage` が設定する、みがわりへの与ダメージ）を使う。
    既存の `_drain_hp`（`src/jpoke/handlers/move_attack.py`）と同じ `value or ctx.substitute_damage` の慣用パターン。
- 「連続攻撃技は最後の攻撃の後に発動し、回復量は全ヒットの合計ダメージから算出する」
  → `ctx.hit_index`/`ctx.hit_count` を使い、最後のヒットまで `_shell_bell_total_damage` に累積し、
    最後のヒットでまとめて 1/8 を計算する（`いのちのたま_recoil` と同様に `ctx.hit_index == ctx.hit_count` を利用）。
- 「特性ちからずくの対象技を使うときは回復効果も無くなる」
  → `いのちのたま_recoil` と同じ条件（`ability.name == "ちからずく"` かつ `move.has_flag("secondary_effect")`）でスキップする。
- 「こんらんの自傷ダメージでは発動しない」「みらいよち/はめつのねがいでは発動しない」
  → これらは `ON_HIT`/`ON_DAMAGE_HIT` を経由せず直接 `battle.modify_hp` を呼ぶ実装
    （`こんらん_try_action`, `はめつのねがい_damage`, `みらいよち_damage`）のため、
    かいがらのすずのハンドラには到達せず、追加の実装は不要。
- 「わるいてぐせ発動前にかいがらのすずの回復が先に発動する」
  → `わるいてぐせ_steal_item` は `Event.ON_DAMAGE_HIT`（`ON_HIT` より後に発火）に登録されているため、
    `ON_HIT` を使うかいがらのすずは常に先に発動する（イベントの発火順序で保証される。優先度の数値比較ではない）。
- 「全体攻撃技で複数のポケモンに同時にダメージを与えた場合は合計する」
  → 本プロジェクトはシングルバトルのみ対象（CLAUDE.md）のため、この仕様は対象外・実装不要。

### 全体攻撃技（シングル対象外）の扱い
CLAUDE.md により「対象はポケモンチャンピオンズシングルバトルのみ」であるため、
複数対象への同時ダメージ合計（トリプルバトル等）は考慮しない。

## 注意
- `ちからずく`/`substitute_damage`/`hit_index`/`hit_count` の判定は
  `いのちのたま_recoil`（`src/jpoke/handlers/item.py`）・`_drain_hp`（`src/jpoke/handlers/move_attack.py`）を参考にした。
- `_shell_bell_total_damage` は `AttackContext` に宣言されていない動的属性。
  `AttackContext` は技の実行ごとに新規生成されるため（`move_executor.py` の `run_move`）、
  ターンをまたいだ値の混入は発生しない。

## テスト観点
- ダメージ技命中時、与ダメージの 1/8 が回復すること（既存）
- 変化技では回復しないこと（`ON_HIT` は攻撃技のダメージ発生時のみ実ダメージを持つ）
- みがわり状態の相手を攻撃した場合、みがわりへの与ダメージの 1/8 が回復すること
- 特性ちからずく所持者が追加効果ありの技を使った場合、回復効果が発生しないこと（威力上昇等の効果自体には影響しない特性ではないため、回復可否のみ確認）
- 連続攻撃技（例: 2回固定技）を使用した場合、各ヒットごとではなく最後のヒットの後にまとめて、合計ダメージの 1/8 が回復すること
