# だいはっきんだま 実装計画

更新日: 2026-07-04

## スコープ

- 仕様書: `docs/spec/items/だいはっきんだま.md`
- 実装状態: 実装済み
- 実装方針: `ON_SWITCH_IN` / `ON_ITEM_GAINED` でギラティナ(アナザー)をオリジンフォルムにフォルムチェンジし、`ON_CALC_POWER_MODIFIER` でドラゴン・ゴースト技を 1.2 倍にする。
  加えて `ON_CHECK_ITEM_CHANGE` で（1）ギラティナが持っている間の奪取/交換/除去防止と、（2）アナザーフォルムのギラティナへの譲渡防止を行う。

## 実装ゴール（完了定義）

- ギラティナ(アナザー)がだいはっきんだまを持って登場するとギラティナ(オリジン)になる
- ギラティナ(オリジン)がドラゴンまたはゴースト技を使うと威力が 4915/4096 倍になる
- `removable=False` はすでに ItemData に設定済み（なげつける威力0で失敗）
- ギラティナ以外が持っても効果なし
- ギラティナが持っている間はトリック・すりかえ・ほしがる・どろぼう・特性マジシャン/わるいてぐせ・
  ふしょくガス・はたきおとすによる奪取/交換/除去を防ぐ（`ON_CHECK_ITEM_CHANGE` を `False` で止める）
  - はたきおとすの威力補正（1.5倍）も、除去できないため発生しない
- アナザーフォルムのギラティナへはトリック・すりかえ等で渡すことができない
- マジックルーム中は威力補正が無効になるがフォルムは維持（ItemHandler が item_enabled チェックを通じて自動的に無効化される）

## 実装対象イベント

- `Event.ON_SWITCH_IN` — 登場時にフォルムチェンジを発動する
- `Event.ON_ITEM_GAINED` — バトル中にアイテムを入手したとき
- `Event.ON_CALC_POWER_MODIFIER` — ドラゴン/ゴースト技威力 1.2 倍
- `Event.ON_CHECK_ITEM_CHANGE` — 奪取/交換/除去の可否判定（`ItemManager.can_change_item` から発火。ターン進行に関する `docs/spec/turn.md` 掲載イベントではないため、既存の
  同種ハンドラ（`だいこんごうだま_prevent_item_change` 等）に倣い priority は既定値(100)のまま登録する）

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/item.py` — `だいはっきんだま` に `handlers` を登録する

## 実装ステップ

1. `src/jpoke/handlers/item.py` に `だいはっきんだま_form_change` と `だいはっきんだま_modify_power` を実装する。
2. `src/jpoke/handlers/item.py` の共通処理 `_dedicated_item_prevent_item_change` /
   `_dedicated_item_prevent_transfer_to_base_form` を呼び出す
   `だいはっきんだま_prevent_item_change` / `だいはっきんだま_prevent_transfer_to_base_form` を実装する。
3. `src/jpoke/data/item.py` の `だいはっきんだま` に各 handler を登録する（`ON_CHECK_ITEM_CHANGE` は
   `target:self` と `target:foe` の2つを登録する）。
4. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. ギラティナ(アナザー)がだいはっきんだまを持って登場するとギラティナ(オリジン)になる。
2. ギラティナ(オリジン)がドラゴン技を使うと威力が約 1.2 倍になる。
3. ギラティナ(オリジン)がゴースト技を使うと威力が約 1.2 倍になる。
4. ギラティナ以外が持っても効果がない。
5. ギラティナ(オリジン)がはたきおとす・どろぼうで奪われない。
6. ギラティナ(オリジン)がトリック・すりかえで交換されない。
7. アナザーフォルムのギラティナへトリック等で渡すことができない。

## 疑似コード

```python
BOOSTED_TYPES = {"ドラゴン", "ゴースト"}

def だいはっきんだま_form_change(battle, ctx, value):
    mon = ctx.source
    if mon.name != "ギラティナ(アナザー)":
        return HandlerReturn(value=value)
    mon.set_form("ギラティナ(オリジン)")
    return HandlerReturn(value=value)

def だいはっきんだま_modify_power(battle, ctx, value):
    if ctx.attacker.name != "ギラティナ(オリジン)":
        return HandlerReturn(value=value)
    if ctx.move.type not in BOOSTED_TYPES:
        return HandlerReturn(value=value)
    return HandlerReturn(value=apply_fixed_modifier(value, 4915))

def だいはっきんだま_prevent_item_change(battle, ctx, value):
    # ギラティナ（オリジン/アナザーどちらでも）が持っている間は除去/交換を防ぐ
    mon = ctx.target
    if mon is not None and mon.name.startswith("ギラティナ"):
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)

def だいはっきんだま_prevent_transfer_to_base_form(battle, ctx, value):
    # アナザーフォルムのギラティナへは渡せない
    mon = ctx.target
    if mon is not None and mon.name == "ギラティナ(アナザー)":
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

## リスクと対策

- リスク: `はたきおとす_power` は `ItemManager.can_change_item` を確認する共通実装のため、`ON_CHECK_ITEM_CHANGE`
  の登録漏れがあるとはたきおとすの威力補正判定に影響する。`test_はたきおとす_奪えない専用道具は威力補正なし`
  （だいこんごうだまで代表確認）と本アイテムの `test_だいはっきんだま_ギラティナから奪えない` で
  奪取不可であることを確認する。
