# ヒメリのみ 実装計画

## 効果概要
PP が 0 になったとき、使用した技の PP を 10 回復する（消費）。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_MODIFY_PP_CONSUMED` または PP 減少後のフック |
| subject_spec | `"source:self"` |
| priority | 100 |
| once | True（消費アイテム） |
| 消費条件 | PP が 0 になったとき `battle.consume_item(mon)` |

### ハンドラロジック（疑似コード）
```python
def ヒメリのみ_restore_pp_on_empty(battle, ctx, value):
    mon = ctx.source
    move = ctx.move
    # TODO: PP が 0 になった瞬間を検知する方法を確認
    if move.pp == 0:
        move.pp = min(move.pp + 10, move.max_pp)
        battle.consume_item(mon)
        _announce_item_triggered(battle, mon)
    return HandlerReturn(value=value)
```

## 注意
PP 管理の詳細確認が必要。`ON_MODIFY_PP_CONSUMED` は PP 消費量の計算用イベントであり、
PP が 0 になった後を検知するフックとしては使えない可能性がある（TODO）。

## テスト観点
- 技の PP が 0 になったとき、その技の PP が 10 回復すること
- PP が 0 でない場合は発動しないこと
- 発動後にアイテムが消費されること
