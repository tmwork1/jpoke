# ばんのうがさ 実装計画

## 効果概要
持たせたポケモンはにほんばれ・あめ状態の影響を受けなくなる。
すなあらし・ゆきには無効。天候を自分で発動させることは可能。

## 実装仕様

| 項目 | 内容 |
|------|------|
| 仕様書 | `docs/spec/items/ばんのうがさ.md` |
| 実装状態 | **未実装**（`# TODO : 実装` コメントあり） |
| 消費条件 | なし（非消費アイテム） |

### 影響する効果と対応イベント（候補）

仕様が広範なため、影響する効果を分類して段階的に実装する。

| 効果カテゴリ | 詳細 | 対応イベント候補 |
|---|---|---|
| 技威力補正（攻撃側） | 晴れ時ほのお技×1.5、雨時みず技×1.5（所持者が攻撃） | `Event.ON_CALC_POWER_MODIFIER` |
| 技威力補正（防御側） | 晴れ/雨による被ダメージ補正を無視（所持者が防御） | `Event.ON_CALC_POWER_MODIFIER` |
| 命中率（晴れ時かみなり等） | 晴れでの命中低下を無視（攻撃側） | `Event.ON_MODIFY_ACCURACY` |
| ソーラービーム | 晴れでも溜めが必要になる（攻撃側） | 溜め判定イベント |
| 特性抑制 | サンパワー・リーフガード・ようりょくそ等を抑制 | `Event.ON_CHECK_ABILITY_DISABLE` 相当 |

### 実装方針
まず最も利用頻度が高い「技威力への天候補正を無視」から着手し、
その他の細かい仕様（ソーラービーム・特性抑制等）は後続タスクとして順次追加する。

## 実装ステップ

1. 仕様書 `docs/spec/items/ばんのうがさ.md` の各項目をイベント単位に整理する。
2. 攻撃側・防御側ごとに天候威力補正を無視するハンドラを `src/jpoke/handlers/item.py` に実装。
3. `src/jpoke/data/item.py` の `ばんのうがさ` にハンドラを登録。
4. `tests/test_item.py` に各効果のテストを追加。

## テスト観点
- 晴れ状態でも攻撃側がばんのうがさを持つ場合、ほのお技威力が上昇しないこと
- 晴れ状態でも攻撃側がばんのうがさを持つ場合、みず技威力が半減しないこと
- 防御側がばんのうがさを持つ場合、晴れのほのお技・雨のみず技の被ダメージ補正が無視されること
- すなあらし・ゆきの影響（ダメージ等）はばんのうがさで防げないこと
