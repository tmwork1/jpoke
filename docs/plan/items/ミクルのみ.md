# ミクルのみ 実装計画

## 効果概要
HP が最大値の 25% 以下になったとき、フラグを立て（消費）、次に使う技の命中率を 1.2 倍にする。

## 実装仕様

### イベント 1: HP 閾値トリガー（フラグ付与）

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_HP_CHANGED` |
| subject_spec | `"target:self"` |
| priority | 100 |
| once | True（消費アイテム） |
| 消費条件 | 発動時に `battle.consume_item(mon)` |

### イベント 2: 命中率 1.2 倍適用

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_MODIFY_ACCURACY` |
| subject_spec | `"attacker:self"` |
| priority | 100 |
| once | False |

### ハンドラロジック（疑似コード）
```python
def ミクルのみ_set_accuracy_flag(battle, ctx, value):
    mon = ctx.target
    hp_after = mon.hp
    hp_before = hp_after + value
    if hp_before * 4 > mon.max_hp and hp_after * 4 <= mon.max_hp:
        mon.item.data["micle_active"] = True
        battle.consume_item(mon)
        _announce_item_triggered(battle, mon)
    return HandlerReturn(value=value)


def ミクルのみ_boost_accuracy(battle, ctx, value):
    mon = ctx.attacker
    if mon.item.data.get("micle_active"):
        mon.item.data["micle_active"] = False  # 1回限り
        value = apply_fixed_modifier(value, 4915)  # 1.2倍
    return HandlerReturn(value=value)
```

## 注意
フラグ管理（`mon.item.data` の利用可否）を確認する。
サンのみ・タラプのみと同パターンのフラグ管理。

## テスト観点
- HP が 25% 超 → 25% 以下になった瞬間にフラグが立ち、アイテムが消費されること
- フラグが立った後、次の技の命中率が 1.2 倍になること
- 命中率適用後はフラグが解除され、2回目以降は倍率が変化しないこと
