# くっつきバリ 実装計画

## 効果概要
仕様書: `docs/spec/items/くっつきバリ.md`

2つの効果を持つ：
1. **ターン終了時ダメージ**: 所持者は毎ターン最大HPの1/8のダメージを受ける。
2. **直接攻撃者へのアイテム転送**: アイテムを持たない相手から直接攻撃を受けると、相手のアイテムになる。

## 実装仕様

### 効果1: ターン終了時ダメージ

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_TURN_END` |
| subject_spec | `"source:self"` |
| priority | 90（かえんだま・どくどくだまと同水準） |
| once | False |
| 消費条件 | なし（非消費） |

### 効果2: 直接攻撃者へのアイテム転送

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_DAMAGE_HIT` |
| subject_spec | `"defender:self"` |
| priority | 100 |
| once | False |
| 消費条件 | 転送時に自分のアイテムを失う（`battle.remove_item(mon)` + `battle.gain_item(attacker, "くっつきバリ")`） |

### ハンドラロジック（疑似コード）
```python
def くっつきバリ_damage_on_turn_end(battle, ctx, value):
    """くっつきバリ: ターン終了時に最大HPの1/8ダメージ。"""
    mon = ctx.source
    assert mon is not None
    battle.modify_hp(mon, r=-1/8)
    return HandlerReturn(value=value)


def くっつきバリ_transfer_on_direct_hit(battle, ctx, value):
    """くっつきバリ: アイテムなし直接攻撃者に転送。"""
    attacker = ctx.attacker
    defender = ctx.defender
    assert defender is not None
    # 接触技かつ攻撃者がアイテムなしのとき転送
    if ctx.move.is_contact and not attacker.item.name:
        battle.gain_item(attacker, "くっつきバリ")
        battle.remove_item(defender)
        _announce_item_triggered(battle, defender)
    return HandlerReturn(value=value)
```

## 実装ステップ

1. `src/jpoke/handlers/item.py` に `くっつきバリ_damage_on_turn_end` と `くっつきバリ_transfer_on_direct_hit` を追加。
2. `src/jpoke/data/item.py` の `くっつきバリ` に両イベントのハンドラを登録。
3. `tests/test_item.py` にテストを追加し、全テストが通ることを確認。

## テスト観点

1. 毎ターン終了時に最大HPの1/8ダメージを受けること
2. 接触技を受け、攻撃者がアイテムなしのとき、くっつきバリが攻撃者に転送されること
3. 攻撃者がすでにアイテムを持っている場合は転送されないこと
4. 非接触技（飛び道具等）では転送されないこと
5. 転送後、自分のアイテム欄が空になること
6. 転送された攻撃者が次のターン終了時にダメージを受けること
