# クリアチャーム 実装計画

更新日: 2026-06-12

## スコープ

- 仕様書: `docs/spec/items/クリアチャーム.md`
- 実装状態: 未実装
- 実装方針: `ON_BEFORE_MODIFY_STAT` で相手由来のランク低下を全て無効化する（クリアボディのアイテム版）

## 実装ゴール（完了定義）

- 相手の技・特性によるランク低下を無効化できる
- 自分自身の技・特性によるランク低下（からをやぶる・じこあんじ等）は防げない
- かたやぶりでは無視されない（アイテムなので mold_breaker_ignorable フラグ不要）
- クリアチャームより先にみがわり・しろいきり等の状態が優先する（既存処理が担当）

## 実装対象イベント

- `Event.ON_BEFORE_MODIFY_STAT` — ランク変化を適用する直前。`value` は `dict[Stat, int]`

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/item.py` — `クリアチャーム` に `handlers` を登録する
- `src/jpoke/handlers/item.py` — `クリアチャーム_block_stat_drop` を実装する
- `tests/test_item.py` — テストを追加する

## 実装ステップ

1. `src/jpoke/handlers/item.py` に `クリアチャーム_block_stat_drop` を実装する。
   - `common.block_stat_drop_by_foe(value, ctx)` を使って相手由来のランク低下を除去する。
2. `src/jpoke/data/item.py` の `クリアチャーム` に `ON_BEFORE_MODIFY_STAT` handler を登録する。
3. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. 相手のいかく・すてゼリフ等によるランク低下が無効になる。
2. 自分の技（からをやぶる・じこあんじ等）によるランク低下は防げない。
3. ミラーアーマーで跳ね返ってきたランク低下は防げる（相手由来として扱われる）。
4. 持ち主が交代した後は効果がなくなる。
