# レッドカード 実装計画

## 効果概要
ダメージを受けたとき、攻撃した相手をランダムな控えポケモンと強制交代させる（消費）。
プレイヤーが交代先を選べないため Interrupt は不要。ふきとばし系の `blow()` と同パターン。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_DAMAGE_HIT` |
| subject_spec | `"defender:self"` |
| priority | 100 |
| once | True（消費アイテム） |
| 消費条件 | 発動時に `battle.consume_item(mon)` |

### ハンドラロジック（疑似コード）
```python
def レッドカード_force_switch(battle, ctx, value):
    mon = ctx.defender
    foe = ctx.attacker
    foe_player = battle.get_player(foe)
    commands = battle.get_available_switch_commands(foe_player)
    if commands:
        battle.consume_item(mon)
        _announce_item_triggered(battle, mon)
        command = battle.random.choice(commands)
        battle.run_switch(foe_player, battle.player_states[foe_player].team[command.index])
    return HandlerReturn(value=value)
```

## テスト観点
- ダメージを受けたとき相手がランダムに強制交代されること
- 相手の控えがいない場合（最後の1体）は交代が起きないこと
- 発動後にアイテムが消費されること
