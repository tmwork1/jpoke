# おおきなねっこ 実装計画

## 効果概要
吸収技（ドレイン技）によって回収する HP を 1.3 倍にする（非消費）。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_CALC_DRAIN` |
| subject_spec | `"source:self"` |
| priority | 100 |
| once | False |
| 消費条件 | なし（非消費アイテム） |

## 発火側の改修

`ON_CALC_DRAIN` はイベント自体は定義済みだが未発火。ドレイン処理の2箇所に発火を追加する。

### 1. `handlers/move.py` の `_drain_hp()` を改修
```python
def _drain_hp(battle, ctx, damage, heal_ratio):
    damage = damage or ctx.substitute_damage
    heal_amount = int(damage * heal_ratio)
    heal_amount = battle.events.emit(
        Event.ON_CALC_DRAIN,
        EventContext(source=ctx.attacker),
        heal_amount
    )
    battle.modify_hp(ctx.attacker, v=heal_amount, reason="drain")
```

### 2. `handlers/common.py` の `drain_hp()` を改修
```python
# battle.modify_hp(to_mon, -damage * heal_rate, ...) の前に発火
heal_amount = int(damage * heal_rate)
heal_amount = battle.events.emit(
    Event.ON_CALC_DRAIN,
    EventContext(source=to_mon),
    heal_amount
)
battle.modify_hp(to_mon, v=heal_amount, reason=reason)
```

### ハンドラロジック（アイテム側）
```python
def おおきなねっこ_boost_drain(battle, ctx, value):
    return HandlerReturn(value=int(value * 1.3))
```

## テスト観点
- おおきなねっこ持ちがドレイン技を使ったとき回収量が 1.3 倍になること
- おおきなねっこがない場合は通常の回収量（ダメージの 50% 等）であること
- やどりぎのタネ（common.drain_hp 経由）でも 1.3 倍になること
