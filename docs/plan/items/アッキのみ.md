# アッキのみ 実装計画

## 効果概要
物理技でダメージを受けたとき、ぼうぎょ（B）を +1 する（消費）。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_DAMAGE_HIT` |
| subject_spec | `"defender:self"` |
| priority | 100 |
| once | True（消費アイテム） |
| 消費条件 | 発動時に `battle.consume_item(mon)` |

### ハンドラロジック（疑似コード）
```python
def アッキのみ_boost_defense_on_physical_hit(battle, ctx, value):
    return _boost_on_attack_category(battle, ctx, value, "physical", "def", +1)

def _boost_on_attack_category(battle, ctx, value, category, stat, amount):
    mon = ctx.defender
    if ctx.move.category == category:
        # source=mon で自発的な変化として扱う（あまのじゃく反転時にしろいきり等で防がれないようにする）
        # modify_stats の戻り値が空（ランク上限/下限で変化なし）なら発動・消費しない
        if battle.modify_stats(mon, {stat: amount}, source=mon):
            _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)
```

## 注意
- じゅくせい持ちでも効果量は+1のまま（スター以外のピンチきのみと違い2倍にならない）
- ダメージが0のとき（相性無効・みがわり等）は ON_DAMAGE_HIT が発火しないため別途チェック不要
- すでに防御ランクが最大（+6）のときは `modify_stats` の戻り値が空になるため、発動・消費しない（`if battle.modify_stats(...):` で判定する）
- `battle.modify_stats(mon, {...}, source=mon)` は必ず `source=mon` を渡す。省略すると `EventContext.source=None` となり `is_foe_target()`（`source != target`）が真になってしまい、あまのじゃくで反転した下降がしろいきり・フラワーベールに防がれてしまう（仕様上は自発的な変化として防がれてはいけない）
- あまのじゃく/たんじゅんの反転・倍加は `Event.ON_BEFORE_MODIFY_STAT` に登録された特性ハンドラが `modify_stats` 経由で自動的に適用するため、本ハンドラ側で個別対応は不要
- かたやぶり由来のあまのじゃく/たんじゅん無効化も特性側の `mold_breaker_ignorable` フラグで汎用的に処理されるため、本ハンドラでは何もしない
- ちからずくの効果が発動した技を受けたときは発動しないという仕様があるが、この抑制機構自体がこのリポジトリに未実装（だっしゅつボタン・レッドカードなど他の「反応型アイテム」も同様に未実装）。今回のレビューでは対象外の既知の制限として残す

## テスト観点
- 物理技でダメージを受けたとき B が +1 されること
- 特殊技では発動しないこと
- 発動後にアイテムが消費されること
- すでに B ランクが+6のときは発動せず、アイテムも消費されないこと
- あまのじゃく持ちが物理技を受けたとき B が-1になり、しろいきりがあっても防がれないこと（自発的な変化として扱われる）
