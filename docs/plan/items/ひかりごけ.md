# ひかりごけ 実装計画

## 効果概要
みず技でダメージを受けたとき、とくぼう（D）を +1 する（消費）。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_DAMAGE_HIT` |
| subject_spec | `"defender:self"` |
| priority | 90（`ItemHandler` のデフォルト100より低い値を明示指定。理由は後述） |
| once | False（`_announce_and_consume_item` で都度アイテムを消費する消費型アイテム） |
| 消費条件 | `battle.modify_stats` が実際に変化を返したときのみ `_announce_and_consume_item(battle, mon)` を呼ぶ |

### ハンドラロジック（疑似コード）
```python
def ひかりごけ_boost_spdef_on_water_hit(battle, ctx, value):
    mon = ctx.defender
    if ctx.move.type == "みず":
        changes = battle.modify_stats(mon, {"spd": +1})
        if changes:  # ランク上限などで不発の場合は消費しない
            _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)
```

`きゅうこん` / `じゅうでんち` / `ゆきだま` と共通の `_boost_stat_on_type_hit` ヘルパーを利用する。

### たんじゅん・あまのじゃく・かたやぶりとの関係（フレームワークで自動的に処理される）
- `battle.modify_stats` は `ON_BEFORE_MODIFY_STAT` を発火し、たんじゅん（`たんじゅん_double_stat`）・あまのじゃく
  （`あまのじゃく_reverse_stat`）がそこで能力変化量を2倍・反転するため、ひかりごけ側で個別対応は不要。
- とくぼうランクがすでに最大（あまのじゃくの場合は最小）まで変化している場合、`modify_stats` の戻り値が空になるため
  発動せず、アイテムも消費されない。
- かたやぶり（`mold_breaker_ignorable` フラグを持つ特性を無効化）は `ON_BEGIN_MOVE`～`ON_END_MOVE` の間、防御側の
  たんじゅん・あまのじゃくを無効化する。ひかりごけが発動する `ON_DAMAGE_HIT` はこの区間内で発火するため、
  かたやぶり技に対しては自動的にたんじゅん・あまのじゃくの効果が発動せず、通常通り +1 される。

### マジシャンとの発動順（priority=90 の根拠）
- `docs/spec/abilities/マジシャン.md` に「きあいのハチマキ/きあいのタスキ/ゴツゴツメット/じゃくてんほけん/
  じゅうでんち/ゆきだま/きゅうこん/ひかりごけ/ふうせんの効果が発動する場合、マジシャンで奪う前に発動する」
  と明記されている。使い捨てアイテムは発動させてしまうと奪えない仕様のため、この順序は素早さに関係なく
  常にひかりごけが先でなければならない。
- 本フレームワークの `_sort_handlers` は同一イベント内で `(priority, -speed)` によりハンドラを並べ替える。
  マジシャン（`ON_DAMAGE_HIT`, subject=attacker）とひかりごけ（`ON_DAMAGE_HIT`, subject=defender）は
  異なるポケモンが subject であるため、両者とも `priority=100`（デフォルト）のままでは、攻撃側が防御側より
  速い場合にマジシャンが先に発動し道具を奪ってしまう不具合があった。
- これを防ぐため、ひかりごけの `ON_DAMAGE_HIT` ハンドラに `priority=90` を明示し、マジシャン特性ハンドラ
  （`priority=100`）より確実に先に発動するようにした（`いかさまダイス`の`priority=90`と同様の先例パターン）。

## 既知の課題（本アイテム単体の対応範囲外）
- 同じ `_boost_stat_on_type_hit` ヘルパーを使う `きゅうこん` / `じゅうでんち` / `ゆきだま` にも同様の
  マジシャンとの発動順不具合が残っている可能性がある。これらは各アイテムの担当レビューで個別に対応する。

## テスト観点
- みず技でダメージを受けたとき D が +1 されること
- みず以外の技では発動しないこと
- 発動後にアイテムが消費されること
- とくぼうランクが最大のときは発動せず、アイテムも消費されないこと
- 所持者がたんじゅんのとき、とくぼうランクが2段階上がること
- 所持者があまのじゃくのとき、とくぼうランクが1段階下がること
- 所持者があまのじゃくでとくぼうランクが最小のとき、発動しないこと
- かたやぶりの効果を持つみず技を受けた場合、たんじゅん・あまのじゃくでもとくぼうランクが1段階上がること
- 特性マジシャンのみず技を受けた場合、攻撃側の素早さが防御側より高くてもひかりごけが先に発動・消費され、
  道具を奪われないこと
