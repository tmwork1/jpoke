# くちたたて 実装計画

更新日: 2026-07-04

## スコープ

- 仕様書: `docs/spec/items/くちたたて.md`
- 実装状態: 実装済み
- 実装方針:
  - `ON_SWITCH_IN` / `ON_ITEM_GAINED` でザマゼンタ(れきせん)をザマゼンタ(たてのおう)にフォルムチェンジする
  - `ON_CHECK_ITEM_CHANGE` でザマゼンタが持っている間の奪取/交換/除去を防ぎ、
    れきせんのゆうしゃザマゼンタへの譲渡も防ぐ（`くちたけん` と同じ構成）

## 実装ゴール（完了定義）

- ザマゼンタ(れきせん)がくちたたてを持って登場すると、ザマゼンタ(たてのおう)にフォルムチェンジする
- ザマゼンタ以外が持っても追加効果はない（フォルムチェンジしない）
- ザマゼンタが持っている間はトリック・すりかえ・ほしがる・どろぼう・特性マジシャン/わるいてぐせ・
  ふしょくガス・はたきおとすによる奪取/交換/除去を防ぐ（ザマゼンタ以外が持つ場合は通常通り奪われる）
- れきせんのゆうしゃザマゼンタへはトリック・すりかえ等で渡せない
- `removable=False` および `fling_power=0` はすでに ItemData に設定済み

## 実装対象イベント

- `Event.ON_SWITCH_IN` — 登場時にフォルムチェンジを発動する
- `Event.ON_ITEM_GAINED` — バトル中にアイテムを入手したとき
- `Event.ON_CHECK_ITEM_CHANGE` — アイテムの奪取/交換/除去が許可されるかを判定する
  （`ItemManager.can_change_item()` が発火。`docs/spec/turn.md` に未掲載のため、
  同様の実装を持つ `くちたけん` の既存ハンドラと同じ登録方法・優先度を踏襲する）

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/item.py` — `くちたたて` に `handlers` を登録する
- `src/jpoke/handlers/item.py` — `くちたたて_form_change` / `くちたたて_prevent_item_change` /
  `くちたたて_prevent_transfer_to_hero` を実装する
- `tests/test_item.py` — テストを追加する

## 実装ステップ

1. `src/jpoke/handlers/item.py` に `くちたたて_form_change` を実装する。
2. `src/jpoke/handlers/item.py` に `くちたたて_prevent_item_change`（`target:self` がザマゼンタなら
   奪取/交換/除去を禁止）・`くちたたて_prevent_transfer_to_hero`（`target:foe` がれきせんの
   ゆうしゃザマゼンタなら禁止）を実装する。
3. `src/jpoke/data/item.py` の `くちたたて` に `ON_SWITCH_IN`・`ON_ITEM_GAINED`・
   `ON_CHECK_ITEM_CHANGE` handler を登録する。
4. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. ザマゼンタ(れきせん)がくちたたてを持って登場するとザマゼンタ(たてのおう)になる。
2. ザマゼンタ(たてのおう)がすでに場にいるときは再度フォルムチェンジしない。
3. ザマゼンタ以外のポケモンが持っても変化しない。
4. フォルムチェンジ後の種族値が正しく反映される。
5. ザマゼンタがくちたたてを持っている間ははたきおとす・どろぼうで奪われない。
6. ザマゼンタ以外が持っている場合は通常通り奪われる。
7. ザマゼンタがくちたたてを持っている間はトリック・すりかえで交換されない。
8. れきせんのゆうしゃザマゼンタへはトリック等で渡すことができない。

    mon = ctx.source
    if mon.name != "ザマゼンタ(れきせん)":

    mon.set_form("ザマゼンタ(たてのおう)")

## リスクと対策

- リスク: `set_form` がステータス再計算・ログ出力を自動で行うか確認が必要。`Pokemon.set_form()` は
  `update_stats()` を呼ぶ（実装確認済み）。
- リスク: ザマゼンタ(たてのおう)で登場した場合（フォームを直接指定した場合）は
  `name != "ザマゼンタ(れきせん)"` により再フォルムチェンジしない。
- `ON_CHECK_ITEM_CHANGE` は `docs/spec/turn.md` に priority の記載がないイベントのため、
  同種の `くちたけん` の登録（priority 未指定 = デフォルト）に合わせている。
