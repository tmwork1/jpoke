# のどスプレー 実装計画

## 効果概要
音技（sound ラベル）を使用したとき、とくこう（C）を +1 する（消費）。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_MOVE_END` |
| subject_spec | `"attacker:self"` |
| priority | 100 |
| once | True（消費アイテム） |
| 消費条件 | 発動時に `battle.item_manager.consume_item(mon)`（`_announce_and_consume_item` 経由） |

### ハンドラロジック（疑似コード）
```python
def のどスプレー_boost_spatk_on_sound(battle, ctx, value):
    mon = ctx.attacker
    executor = battle.move_executor
    if (
        ctx.move.has_flag("sound")
        and not executor.move_missed
        and executor.move_success is not False
    ):
        if battle.modify_stats(mon, {"spa": +1}):  # ランクが変化しなければ消費しない
            _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)
```

## 注意
- `ON_MOVE_END` の subject_spec: `attacker:self` (AttackContext) で正しい。
  event.py のコメントによると `ON_MOVE_END` は `AttackContext` で発火するため。
- `ON_MOVE_END` は技が失敗・無効化された場合（`ON_TRY_MOVE_1/2` 不成立、タイプ相性無効、
  `まもる` 等で `move_applied=False` になった場合）は `MoveExecutor._execute_move` が
  途中で `return` するため発火しない。よってこれらのケースは自然に「消費しない」になる。
- ただし「命中しなかったとき」（`move_missed=True`）と「命中はしたが技自体が不発だったとき」
  （例: いやしのすずで治すべき状態異常の対象がいない場合。`ON_STATUS_HIT` 内で
  `move_success=False` になるが、この失敗は `_execute_move` のループを継続し
  `ON_MOVE_END` まで到達してしまう）はガードなしでは誤発動する。
  そのため `battle.move_executor.move_missed` / `battle.move_executor.move_success` を
  明示的に参照してガードする（うのミサイルの `battle.move_executor.move_applied` 参照と同様の手法）。
- すでにとくこうが最大（+6）の場合は `battle.modify_stats` が空dictを返すため、
  その真偽値で消費判定を行う（他の「ランク上昇して消費」系アイテムと同じパターン）。

## テスト観点
- 音技を使用したとき、とくこうが +1 されること
- 音技以外では発動しないこと
- 発動後にアイテムが消費されること
- 消費後に再度音技を使っても発動しないこと
- 音技が外れたときは発動・消費しないこと
- とくこうがすでに最大（+6）のときは消費しないこと
- いやしのすずで治療対象がおらず技が不発になったときは発動・消費しないこと
