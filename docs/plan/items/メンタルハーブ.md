# メンタルハーブ 実装計画

## 効果概要
メロメロ・アンコール・いちゃもん・かなしばり・ちょうはつ・かいふくふうじのいずれかの
揮発性状態が付与されたとき、即座に解除する（消費）。
なげつけるで使用した場合は追加効果として相手の同じ揮発性状態を解除する。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_VOLATILE_START` |
| subject_spec | `"source:self"`（`VolatileManager.apply` が付与後に発火する `EventContext(source=target)` に合わせる） |
| priority | 未指定（デフォルト） |
| once | 消費アイテムのため `_announce_and_consume_item` で `item_manager.consume_item` を呼び出す |
| 消費条件 | 対象の揮発性状態を解除した場合のみ消費 |

### ハンドラロジック（実装済み）
```python
def メンタルハーブ_cure_mental_volatile(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """メンタルハーブ: 特定の揮発性状態が付与されたとき即解除する。"""
    mon = ctx.source
    assert mon is not None
    if value in {"メロメロ", "アンコール", "いちゃもん", "かなしばり", "ちょうはつ", "かいふくふうじ"}:
        battle.volatile_manager.remove(mon, value)
        _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)
```

`VolatileManager.apply()` は揮発性状態の付与に成功した直後に
`EventContext(source=target)` で `ON_VOLATILE_START` を発火するため、
`ctx.source` が状態変化を受けた本人（＝アイテムの持ち主）になる。
`value` には付与された揮発性状態名（`VolatileName`）がそのまま渡される。

### なげつけるの追加効果
`なげつける_apply_item_effect`（`handlers/move_attack.py`）内で
`item_name == "メンタルハーブ"` の分岐として、相手が対象の揮発性状態を
持っていれば解除する（メンタルハーブ自体は `ON_MOVE_END` で別途消費される）。

## 未対応（対象外）のスコープ
- どんかん（第4世代以前の発動順の違い）・バトンタッチでの引き継ぎ（第6・7世代限定の挙動）は
  現行世代（SV）を対象とする本プロジェクトの範囲外のため実装しない。

## テスト観点
- メロメロ・アンコール・いちゃもん・かなしばり・ちょうはつ・かいふくふうじが付与されたとき即解除されること
- 対象外の揮発性状態（こんらん等）では発動しないこと
- 発動後にアイテムが消費されること
- なげつけるで使用した場合、相手の対象揮発性状態が解除されること
