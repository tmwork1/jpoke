# サイコシード 実装計画

## 効果概要
サイコフィールドが発動中に登場したとき、または発動したとき、とくぼう（D）を +1 する（消費）。
とくぼうがすでに最大（+6）のときは発動せず、アイテムも消費されない。

## 実装仕様

### イベント 1: 登場時

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_SWITCH_IN` |
| subject_spec | `"source:self"` |
| priority | 120（`docs/spec/turn.md` ON_SWITCH_IN: 「120 サイコシードの発動」に一致） |
| once | False（`Pokemon.item.count` ではなく `consume_item` によるアイテム消滅で1度きりの発動を担保するため不要） |
| 消費条件 | ランクが実際に変化した場合のみ `battle.item_manager.consume_item(mon)` |

### イベント 2: フィールド変化時

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_FIELD_CHANGE` |
| subject_spec | `"source:self"` |
| priority | 100（デフォルト。`turn.md` に本イベントの記載なし。既存の同種ハンドラ（例: アイスフェイスの `ON_FIELD_CHANGE`）もデフォルト優先度のため踏襲） |

### イベント 3: アイテム有効化時（かがくへんかガス解除等）

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_ITEM_ENABLED` |
| subject_spec | `"source:self"` |
| priority | 100（デフォルト。`turn.md` に本イベントの記載なし） |

### ハンドラロジック（疑似コード）
```python
def _terrain_seed_boost(battle, ctx, value, terrain, stat):
    mon = ctx.source
    if battle.terrain.name == terrain:
        changes = battle.modify_stats(mon, {stat: +1})
        if changes:  # 実際にランクが変化した場合のみ消費（既に最大なら不発）
            _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)


def サイコシード_boost_spdef(battle, ctx, value):
    return _terrain_seed_boost(battle, ctx, value, "サイコフィールド", "spd")
```

## テスト観点
- サイコフィールド中に登場したとき D が +1 されること
- サイコフィールドが発動したとき D が +1 されること
- サイコフィールドがない場合は発動しないこと
- 発動後にアイテムが消費されること
- とくぼうが既に最大（+6）のときは発動せず、アイテムも消費されないこと
