更新日: 2026-07-05

## スコープ

- 仕様書: `docs/spec/items/_タイプ半減きのみ.md`（バコウのみ節）
- 実装状態: 実装済み

## 効果概要

- 効果バツグンの`ひこう`タイプの技を受けたとき、ダメージを1/2にして消費する（1度きり）。
- ついばむ特有の注意点: ついばむを効果抜群で受けた場合、技の効果（相手のきのみを奪って使用する）
  よりバコウのみの効果が優先して発動する。ダメージ計算時点（`ON_CALC_DAMAGE_MODIFIER`）で
  アイテムが消費されるため、ついばむの`ON_HIT`効果（きのみを奪う処理）が発火する時点では
  既にバコウのみが手元になく、奪われることはない。
- フライングプレス特有の注意点: フライングプレスは技タイプ自体は`かくとう`のままで、
  `ひこう`タイプとの複合相性（ダメージ計算内部でひこうタイプの相性を追加で掛け合わせる）で
  効果抜群かどうかが決まる。バコウのみは技タイプが`ひこう`と一致しないため発動しない
  （フライングプレスの複合相性で発動するのはヨプのみのみ。`ヨプのみ.md`参照）。

## 実装方針

- 通常戦闘: `Event.ON_CALC_DAMAGE_MODIFIER`（`subject_spec="defender:self"`）で発火。
  タイプ半減きのみ共通処理 `handlers/item.py` の `_modify_super_effective_damage` を
  `type_="ひこう"`, `modifier=2048/4096` で呼び出す。
  - `ctx.move.type == "ひこう"` かつ `battle.damage_calculator.calc_def_type_modifier(ctx) > 4096`
    で効果バツグン判定する。フライングプレスは技タイプが`かくとう`のため、この時点で
    条件不成立となり自動的に対象外になる。
  - priority: 他のタイプ半減きのみ・ダメージ補正系アイテムと同じ既定値（`docs/spec/turn.md` に
    `ON_CALC_DAMAGE_MODIFIER` 個別の優先度記載なし。ダメージ計算内部イベントのため、
    既存の同種きのみハンドラの priority に合わせてデフォルト値を使用）。
  - 条件成立時に `_announce_and_consume_item` でログ記録とアイテム消費を行う。
    この時点でアイテムが手元から無くなるため、後続の`ON_HIT`（ついばむの盗み効果）では
    `defender.has_item()` が `False` となり、`むしくい_steal_and_use_berry` は何もしない。
- リーサル計算（簡易KO判定用の `LethalEvent`）: `LethalEvent.ON_BEFORE_HIT`（`subject="defender"`）で
  `handlers/lethal.py` の `バコウのみ_resist_flying` → 共通処理 `_type_resist_berry(..., "ひこう")` を呼び出す。
  通常ハンドラ側で既にダメージ半減済みのため、リーサル側ではアイテム消費と `item_enabled` の更新のみ行う。

## テスト

- `tests/test_item.py` の `test_タイプ半減実`（パラメータ化テスト）で
  `("バコウのみ", "ひこう", "コジョフー")` を検証。
  - `damage_modifier == 2048`（1/2倍）であること。
  - 使用後にアイテムが消費されていること（`not has_item()`）。
- `test_バコウのみ_ついばむで奪われる前に消費されダメージ半減`
  くさ単タイプ（キモリ）がついばむを効果抜群で受けたとき、ダメージが半減し、
  かつバコウのみが消費済みのため攻撃側に奪われないことを確認。
- `test_バコウのみ_フライングプレスでは発動しない`
  くさ単タイプ（キモリ）がフライングプレス（かくとう等倍×ひこう2倍=抜群）を受けても
  バコウのみは発動しないことを確認。
