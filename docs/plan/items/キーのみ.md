# キーのみ 実装計画

## スコープ
- 優先度: -
- 仕様書: `docs/spec/items/_状態回復きのみ.md`（状態回復きのみ共有仕様書、キーのみの節）
- 実装状態: 実装済み（レビューにより ON_VOLATILE_START / ON_ITEM_ENABLED を追加）

## 仕様まとめ
- 持たせると戦闘中にこんらん状態になった瞬間に自動消費して回復する。
- こんらん状態のポケモンに「なげつける」で当てると相手のこんらんを回復する（威力10、`lethal_handlers` 側で別途処理される想定・本アイテムの handlers スコープ外）。
- こんらんは状態異常ではなく揮発性状態（`VolatileName`）。他の状態異常回復きのみ（カゴのみ・クラボのみ等）と同様の「即時発動＋ターン終了時の保険」という発動タイミングの構造を、揮発性状態向けのイベントに置き換えて踏襲する。

## 発動タイミングと priority

対応する既存アイテムの実装パターンを参照して決定（`docs/spec/turn.md` に該当イベントの記載なし）。

| Event | 参照した既存実装 | priority | 根拠 |
| --- | --- | --- | --- |
| `Event.ON_TURN_END` | カゴのみ／クラボのみ／ラムのみ 等（状態異常回復きのみ） | 50 | 既存の状態異常回復きのみが軒並み `priority=50` を使用しており、他のターン終了処理（天候・場・状態異常ダメージ等）との相対順序を揃えるため踏襲。実装済みのため変更なし。 |
| `Event.ON_VOLATILE_START` | あかいいと／メンタルハーブ（`ON_VOLATILE_START` を使う既存アイテム） | 指定なし（デフォルト100） | こんらんは `ON_APPLY_AILMENT` ではなく `VolatileManager.apply()` が発火する `ON_VOLATILE_START` で通知される。同イベントを使う既存アイテム（あかいいと・メンタルハーブ）はいずれも priority を明示していないため、それに合わせてデフォルト値を使用。 |
| `Event.ON_ITEM_ENABLED` | カゴのみ／クラボのみ／ラムのみ 等 | 50 | アイテムが有効化された時点（きんじて解除等）で既にこんらん中なら即座に回復させる。既存の状態異常回復きのみが `ON_ITEM_ENABLED` でも `priority=50` を使っているため踏襲。 |
| `Event.ON_FORCE_BERRY_TRIGGER` | カゴのみ／クラボのみ／ラムのみ 等 | 50 | ほおばる等による強制発動。既存実装済みのため変更なし。 |

いずれも `subject_spec="source:self"`（`EventContext.source` がきのみ所持者自身になる非攻撃フェーズのイベントのため）。

## 実装（`src/jpoke/handlers/item.py` / `src/jpoke/data/item.py`）

- `handlers/item.py`: 既存の `キーのみ_cure_confusion(battle, ctx, value)` を流用する。
  - `ctx.source` のこんらんを確認し、あれば `battle.volatile_manager.remove(mon, "こんらん")` → `_announce_and_consume_item(battle, mon)`。
  - `value` の中身（イベント種別ごとに異なる）には依存しないため、4イベント全てで同一関数を再利用できる。
- `data/item.py` の `"キーのみ"` エントリに以下の4ハンドラを登録する（`ON_TURN_END` と `ON_FORCE_BERRY_TRIGGER` は実装済み、`ON_VOLATILE_START` と `ON_ITEM_ENABLED` を追加）:

```python
"キーのみ": ItemData(
    fling_power=10,
    handlers={
        Event.ON_TURN_END: h.ItemHandler(
            h.キーのみ_cure_confusion,
            subject_spec="source:self",
            priority=50,
        ),
        Event.ON_VOLATILE_START: h.ItemHandler(
            h.キーのみ_cure_confusion,
            subject_spec="source:self",
        ),
        Event.ON_ITEM_ENABLED: h.ItemHandler(
            h.キーのみ_cure_confusion,
            subject_spec="source:self",
            priority=50,
        ),
        Event.ON_FORCE_BERRY_TRIGGER: h.ItemHandler(
            h.キーのみ_cure_confusion,
            subject_spec="source:self",
            priority=50,
        ),
    }
),
```

## テスト方針

- 既存: `test_キーのみ_ターン終了でこんらん回復`（ON_TURN_END）。
- 追加: `battle.volatile_manager.apply(mon, "こんらん", source=foe)` を直接呼び出し、`ON_VOLATILE_START` で即座に回復・消費されることを確認するテスト（メンタルハーブの `test_メンタルハーブ_いちゃもんを即解除` と同じパターン）。
