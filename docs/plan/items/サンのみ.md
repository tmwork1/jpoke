# サンのみ 実装計画

## 効果概要
HP が最大値の 25% 以下になった瞬間に急所ランクを +2 するフラグを立て（消費）、急所計算時に +2 を適用する。

## 実装仕様

### イベント 1: HP 閾値トリガー（フラグ付与）

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_HP_CHANGED` |
| subject_spec | `"target:self"` |
| priority | 100 |
| once | True（消費アイテム） |
| 消費条件 | 発動時に `battle.consume_item(mon)` |

### イベント 2: 急所ランク適用

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_CALC_CRITICAL_RANK` |
| subject_spec | `"attacker:self"` |
| priority | 100 |
| once | False |

### ハンドラロジック（疑似コード）
```python
def サンのみ_set_critical_flag(battle, ctx, value):
    mon = ctx.target
    hp_after = mon.hp
    hp_before = hp_after + value
    if hp_before * 4 > mon.max_hp and hp_after * 4 <= mon.max_hp:
        # フラグを立てる（アイテムの state 等を活用）
        mon.item.data["critical_boost"] = True
        battle.consume_item(mon)
        _announce_item_triggered(battle, mon)
    return HandlerReturn(value=value)


def サンのみ_boost_critical_rank(battle, ctx, value):
    mon = ctx.attacker
    if mon.item.data.get("critical_boost"):
        return HandlerReturn(value=value + 2)
    return HandlerReturn(value=value)
```

## 注意
フラグ管理の実装方法（`mon.item.data` の利用可否）を確認する。
既存の実装で同様のフラグ管理パターンがあるか調査すること。

## テスト観点
- HP が 25% 超 → 25% 以下になった瞬間にフラグが立ち、アイテムが消費されること
- フラグが立った後、急所ランクが +2 されること
- アイテム消費後もフラグは残ること（フラグ永続チェック）
