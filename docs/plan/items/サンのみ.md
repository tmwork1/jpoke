# サンのみ 実装計画

## 効果概要
HP が最大値の 1/4 以下になった瞬間に消費し、揮発性状態「きゅうしょアップ」（急所ランク+2）を付与する。
すでに「きゅうしょアップ」状態のときは `volatile_manager.apply` が自動的に失敗を返すため発動しない
（仕様書 `docs/spec/items/_ランク上昇ピンチきのみ.md` の「サンのみ」節を参照）。

## 実装仕様

### イベント: HP 閾値トリガー

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_HP_CHANGED` |
| subject_spec | `"target:self"` |
| priority | 100（デフォルト。他の HP 閾値系きのみ（イアのみ・チイラのみ等）と同様） |
| once | False（`volatile_manager.apply` が既存状態を検知して二重付与を防ぐため、アイテム自体は`once`不要。アイテムは消費後に外れるので再発火しない） |

### ハンドラロジック

`handlers/item.py` の `サンのみ_apply_focus_energy` として実装する。
`_boost_on_quarter_hp` と同様に、`value >= mon.max_hp` を「ほおばる・むしくい・ついばむ・おちゃかい等による
強制発動（`item_manager.force_trigger_berry` が `ON_HP_CHANGED` を `value=mon.max_hp` で発火する）」の
判定に使い、HP 閾値チェックを無視する。

```python
def サンのみ_apply_focus_energy(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """サンのみ: HP1/4以下できゅうしょアップ状態になる。

    value >= mon.max_hp はほおばる等による強制発動（HP閾値チェックを無視する）。
    """
    mon = ctx.target
    assert mon is not None
    if mon.hp * 4 <= mon.max_hp or value >= mon.max_hp:
        if battle.volatile_manager.apply(mon, "きゅうしょアップ", count=2):
            _announce_and_consume_item(battle, mon)
    return HandlerReturn(value=value)
```

- `battle.volatile_manager.apply(mon, "きゅうしょアップ", count=2)` は既に同名の揮発性状態があれば `False` を返す
  （`volatile_manager.apply` の共通仕様）。この戻り値を使ってアイテム消費の可否を判定する。
- 「きゅうしょアップ」状態自体の効果（急所ランク+2の適用）は `data/volatile.py` の `きゅうしょアップ` エントリ
  （`Event.ON_CALC_CRITICAL_RANK` ハンドラ `きゅうしょアップ_boost_critical_rank`）にすでに実装済みであり、
  なげつける等の他経路（相手にきゅうしょアップを付与する効果）とも共有される。サンのみ側で急所ランク補正を
  重複実装しない。

## 登録

`data/item.py` の `"サンのみ"` エントリに `Event.ON_HP_CHANGED` ハンドラとして登録する（`fling_power=10`）。

## テスト観点
- HP が 1/4 超 → 1/4 以下になった瞬間にきゅうしょアップ状態が付与され、アイテムが消費されること
- すでにきゅうしょアップ状態のときは発動しない（アイテムが消費されない）こと
