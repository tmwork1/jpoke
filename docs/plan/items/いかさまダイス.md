# いかさまダイス 実装計画

## 効果概要
1. 2-5回連続技のヒット数を必ず4回以上にする。4か5回を各50%の確率で選択。
2. トリプルキック・トリプルアクセル・ネズミざん（毎ヒット命中判定する連続技）を使用したとき、
   最初の1回しか命中判定を行わないようにする（通常の連続技と同じ扱いにする）。

## 実装仕様

### 1. ヒット数補正

| 項目 | 内容 |
|------|------|
| 仕様書 | `docs/spec/items/いかさまダイス.md` |
| 実装状態 | **実装済み** |
| イベント | `Event.ON_MODIFY_HIT_COUNT` |
| subject_spec | `"attacker:self"` |
| priority | **90**（turn.md未掲載イベント。根拠は下記「スキルリンクとの優先順位」を参照） |
| once | False |
| 消費条件 | なし（非消費アイテム） |

#### ハンドラロジック
```python
def いかさまダイス_modify_hit_count(battle, ctx, value):
    min_hits, max_hits = ctx.move.min_hits, ctx.move.max_hits
    if (min_hits, max_hits) == (2, 5):
        value = 4 if battle.random.random() < 0.5 else 5
    return HandlerReturn(value=value)
```

#### スキルリンクとの優先順位（レビューで発見した不具合の修正）
仕様書には「所持者の特性がスキルリンクである場合、特性の効果が優先されて必ず攻撃は最大回数になる」とある。
`EventManager.emit` は同一 priority のハンドラを登録順（安定ソート）で実行し、後に実行されたハンドラの戻り値が
最終値になる。デフォルト priority=100 のまま実装されていたため、スキルリンク（`ability.py`, priority=100）より
いかさまダイスの本ハンドラが後に実行されるケースがあり、スキルリンクで確定したはずの最大ヒット数を
いかさまダイスの50%抽選が上書きしてしまう不具合があった（レビュー時に実測で確認: 4ヒットに化けるケースを検出）。
これを修正するため、本ハンドラの priority を **90** に変更し、スキルリンク側（priority=100）より必ず先に
実行されるようにした。スキルリンク側のハンドラは「常に最大ヒット数を強制する」ため、後から実行されれば
必ず最終値を上書きでき、実行順に依存せず仕様通りの優先順位を実現できる。
（`おやこあい`もON_MODIFY_HIT_COUNTを利用するが、単発技=max_hits1が対象でありいかさまダイスの
対象=(2,5)技とは条件が排他のため、本priority変更による影響はない）

### 2. 毎ヒット命中判定の無効化（トリプルキック等）

トリプルキック・トリプルアクセル・ネズミざんは `check_hit_each_time` フラグを持ち、
`core/move_executor.py` のヒットループ内で毎ヒットごとに命中判定を行う
（`hit_index == 1 or ctx.move.has_flag("check_hit_each_time")`）。
いかさまダイスを持たせている場合はこの毎回判定を無効化し、初回ヒットのみの判定にする必要がある。

この分岐は `move_executor.py` の内部ループでハードコードされていたため、
新規イベント `Event.ON_MODIFY_HIT_CHECK_EACH_TIME` を追加し、ヒットループ開始前に1回だけ発火する形に変更した
（`docs/spec/turn.md` に本イベントの掲載なし。`ON_MODIFY_HIT_COUNT` と同じ「連続技のヒット回数決定」フェーズに属する
既存イベントの直後に配置し、デフォルトpriority=100を採用。根拠: 同フェーズの `ON_MODIFY_HIT_COUNT` が同様に
turn.md未掲載かつデフォルトpriorityで実装されているため）。

| 項目 | 内容 |
|------|------|
| 仕様書 | `docs/spec/items/いかさまダイス.md` |
| イベント | `Event.ON_MODIFY_HIT_CHECK_EACH_TIME`（新規追加） |
| emit元 | `core/move_executor.py`（ヒットループ開始前、`ON_MODIFY_HIT_COUNT` 発火直後に1回だけ発火） |
| subject_spec | `"attacker:self"` |
| priority | 100（デフォルト。根拠は上記） |
| once | False |
| 消費条件 | なし（非消費アイテム） |

#### ハンドラロジック
```python
def いかさまダイス_modify_hit_check_each_time(battle, ctx, value):
    """いかさまダイス: トリプルキック等、毎ヒット命中判定する技を初回ヒットのみの判定にする。"""
    return HandlerReturn(value=False)
```

#### 付随修正: トリプルキックのフラグ不備
`data/move.py` の「トリプルキック」の `flags` に `check_hit_each_time` が設定されておらず、
`トリプルアクセル`・`ネズミざん` と異なり毎ヒット命中判定が機能していなかった（データ不整合のバグ）。
`docs/spec/moves/連続技.md` 9.4節（最大3回攻撃・命中判定は1発ごと）に反しているため、
`flags={"contact", "check_hit_each_time"}` に修正した。いかさまダイスの効果を正しく検証するために必要な前提修正。

## テスト観点
- 2-5回連続技（みだれひっかき等）使用時、ヒット数が4か5になること
- 2回固定・3回固定などの連続技には効果がないこと（ヒット数を変更しない）
- スキルリンク所持時はスキルリンクが優先されること（元々ヒット数最大のため競合しない）
- トリプルキック・トリプルアクセル・ネズミざん使用時、2発目以降が外れる状況でも
  最初の1回だけ命中判定を行い、外れずに最大回数ヒットすること
