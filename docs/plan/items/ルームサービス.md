# ルームサービス 実装計画

## 効果概要
トリックルームが発動したとき、すばやさ（S）を -1 する（消費）。トリックルーム中は遅いポケモンが有利になるため、自ら遅くなることで先制する。

## 実装仕様

| 項目 | 内容 |
|------|------|
| イベント | `Event.ON_FIELD_ACTIVATE` または `Event.ON_FIELD_CHANGE` |
| subject_spec | `"source:self"` |
| priority | 100 |
| once | True（消費アイテム） |
| 消費条件 | 発動時に `battle.consume_item(mon)` |

### ハンドラロジック（疑似コード）
```python
def ルームサービス_lower_speed_on_trick_room(battle, ctx, value):
    mon = ctx.source
    if battle.field.has_trick_room():  # トリックルームが有効かどうか
        battle.modify_stats(mon, {"S": -1})
        battle.consume_item(mon)
        _announce_item_triggered(battle, mon)
    return HandlerReturn(value=value)
```

## 注意
トリックルームが「発動した直後」を検知するイベントが `ON_FIELD_ACTIVATE` か `ON_FIELD_CHANGE` かを確認する。
`battle.field.has_trick_room()` または `"トリックルーム" in battle.field.active_screens` 等の判定方法を確認する。

## テスト観点
- トリックルームが発動したとき S が -1 されること
- トリックルームが発動していない場合は発動しないこと
- 発動後にアイテムが消費されること
