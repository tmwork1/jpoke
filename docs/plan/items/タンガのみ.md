# タンガのみ 実装計画

更新日: 2026-07-05

## スコープ

- 仕様書: `docs/spec/items/_タイプ半減きのみ.md`（タンガのみ節）
- 実装状態: 実装済み

## 効果概要

- 効果バツグンの`むし`タイプの技を受けたとき、ダメージを1/2にして消費する（1度きり）。
- むしくいを効果抜群で受けた場合、技の効果（相手のきのみを奪って消費する）よりタンガのみの
  効果（ダメージ半減）が先に適用される。ダメージ半減は`ON_CALC_DAMAGE_MODIFIER`
  （ダメージ計算内部、命中判定より前）で行われ、むしくいのきのみ強奪処理は`Event.ON_HIT`
  （命中後）で行われるため、実行順序上タンガのみが先に消費され、むしくいが強奪しようとした
  時点では既にアイテムが無く不発になる。

## 実装方針

- 通常戦闘: `Event.ON_CALC_DAMAGE_MODIFIER`（`subject_spec="defender:self"`）で発火。
  タイプ半減きのみ共通処理 `handlers/item.py` の `_modify_super_effective_damage` を
  `type_="むし"`, `modifier=2048/4096` で呼び出す。
  - `ctx.move.type == "むし"` かつ `battle.damage_calculator.calc_def_type_modifier(ctx) > 4096`
    で効果バツグン判定する。
  - priority: 他のタイプ半減きのみ・ダメージ補正系アイテムと同じ既定値（`docs/spec/turn.md` に
    `ON_CALC_DAMAGE_MODIFIER` 個別の優先度記載なし。ダメージ計算内部イベントのため、
    既存の同種きのみハンドラの priority に合わせてデフォルト値を使用）。
  - 条件成立時に `_announce_and_consume_item` でログ記録とアイテム消費を行う。
- リーサル計算（簡易KO判定用の `LethalEvent`）: `LethalEvent.ON_BEFORE_HIT`（`subject="defender"`）で
  `handlers/lethal.py` の `タンガのみ_resist_bug` → 共通処理 `_type_resist_berry(..., "むし")` を
  呼び出す。通常ハンドラ側で既にダメージ半減済みのため、リーサル側ではアイテム消費と
  `item_enabled` の更新のみ行う。
