# こおり 実装計画書

## 概要

こおり状態異常の実装。Champions では解凍確率25%・3回目行動時に必ず解凍・ほのお技被弾で解凍。

## 実装対象

### 1. こおり_action（Event.ON_TRY_ACTION）

**ファイル**: `handlers/ailment.py`
**登録**: `data/ailment.py`

- ターンの行動時に **25%** の確率で解凍（Champions仕様）
- 解凍したらハンドラを解除して行動可能（`HandlerReturn(value=True)`）
- 行動不能になったとき `elapsed_turns` を +1 する（`ailment_manager.tick`）
- `elapsed_turns >= 2` の場合（3回目の行動時）は強制解凍する
- 行動不能時は `LogCode.ACTION_BLOCKED` を記録し `stop_event=True` で返す
- subject_spec: `"attacker:self"` （AttackContext）
- priority: **10**（`docs/spec/turn.md` ON_TRY_ACTION より「こおりの回復判定」が Priority 10）

### 2. こおり_cure_by_fire_damage（Event.ON_DAMAGE_HIT）

**ファイル**: `handlers/ailment.py`
**登録**: `data/ailment.py`

- ほのおタイプの攻撃技でダメージを受けたとき解凍する
- subject_spec: `"defender:self"` （AttackContext）
- priority: **100**（`docs/spec/turn.md` ON_DAMAGE_HIT より「ほのお技受けてこおり回復」が Priority 100）

## テスト方針

`battle.test_option.trigger_ailment` で確率を固定してテストする:
- `True`: 必ず解凍（行動可能）
- `False`: 解凍しない（行動不能）

3回目強制解凍のテストは `trigger_ailment=False` で2回行動不能にした後、3回目のrun_moveで行動成功・ailment解除を確認する。

## 注意点

- SV以前と異なり Champions では解凍確率が **25%**（0.25）
- 3回目強制解凍は Champions 固有仕様
- elapsed_turns のカウントアップは **行動不能になったとき** のみ（解凍したときはカウントしない）
- `test_option.trigger_ailment=True` の場合は `elapsed_turns` チェックより優先して解凍する
