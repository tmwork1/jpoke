# ねむり 実装計画書

## 概要

ねむり状態異常の実装。Champions ではカウント方式によるターン経過回復。

## 実装対象

### 1. ねむり_check_action（Event.ON_TRY_ACTION）

**ファイル**: `handlers/ailment.py`
**登録**: `data/ailment.py`

- ねむり状態のポケモンが行動する際、カウントを1消費する
- カウントが0になった場合: ねむりを解除して行動成功（True を返す）
- カウントが残っている場合: いびき・ねごと選択時または sleep_talk_active フラグがある場合は行動成功
- それ以外: `LogCode.ACTION_BLOCKED` を記録し `stop_event=True` で返す
- subject_spec: `"attacker:self"` （AttackContext相当のイベント）
- priority: 10（`docs/spec/turn.md` の `ON_TRY_ACTION` にて「ねむりのカウント消費」は Priority 10）

### 2. ねむる_apply の count 修正

**ファイル**: `handlers/move_status.py`

- `count=2` → `count=3` に修正する
- 根拠: Wikiの記述「ねむるによってねむり状態になったときは、カウントは3で固定」、
  Champions 説明文「技ねむるでは 3度目まで回復しない」

### 3. ねむけ_remove_and_apply_sleep の count 修正

**ファイル**: `handlers/volatile.py`

- `battle.random.randint(1, 3)` → Champions 仕様の確率分布に修正
- Champions 仕様: count=2（確率1/3）、count=3（確率2/3）
- 実装: `battle.random.random() < 1/3` で分岐

### 4. 通常ねむり付与時の count（apply_ailment_to_defender）

**ファイル**: `handlers/move.py` の `apply_ailment_to_defender`

- 技による通常のねむり付与では `count` は `AilmentManager.apply` のデフォルト（None）のまま
- `None` だとcountチェックで永続になる可能性がある → apply時に Champions 仕様の確率でcountを決定する必要がある
- 確認: `ねむりごな`・`さいみんじゅつ`等は `apply_ailment_to_defender` を呼んでおり、count=None
- 修正: `AilmentManager.apply` の呼び出し側または apply_ailment_to_defender でcountを決定する

## 実装場所

| 修正 | ファイル | 内容 |
|------|--------|------|
| ねむり_check_action | `handlers/ailment.py` | 既実装。内容は正しい。 |
| ねむる count修正 | `handlers/move_status.py` | count=2 → count=3 |
| ねむけ count修正 | `handlers/volatile.py` | randint(1,3) → 1/3:2/3 確率 |
| 通常付与 count | `handlers/move.py` | apply時にcount=2or3を決定 |

## Priority の根拠

- `ON_TRY_ACTION` の priority=10: `docs/spec/turn.md` の「ねむりのカウント消費」は Priority 10
- こおり (`priority=10`) と同一 Priority で実装済み、一致している

## テスト方針

- ねむりカウント: count=2でrun_move→2ターン目で回復
- ねむる技: count=3を付与することを確認
- いびき・ねごと: ねむり中も行動できることを確認
- 通常技ねむり付与: countが2or3の範囲内であることを確認

## 注意点

- `ねむり_check_action` では `tick()` → `has_ailment("ねむり")` でremove判定する仕様。
  `AilmentManager.tick()` の中で `count==0` の場合 `remove()` が呼ばれる。
- `has_ailment("ねむり")` は remove後は False を返すため、
  tick→removeを経たポケモンは行動成功と判定される（正しい動作）。
- count=None で `ねむり` を付与すると tick() 内の `count == 0` チェックが発動せず永続になる可能性あり。
  必ず count を指定すること。
