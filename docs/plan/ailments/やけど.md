# やけど 実装計画書

## 概要

やけど状態異常の実装。ターン終了時に最大HPの1/16（最小1）のダメージ。
物理技で与えるダメージが0.5倍になる。

## 実装対象

### 1. やけど_damage（Event.ON_TURN_END）

**ファイル**: `handlers/ailment.py`  
**登録**: `data/ailment.py`

- やけど状態のポケモンのターン終了時に最大HPの1/16ダメージを与える
- ダメージ計算: `max(1, mon.max_hp // 16)`（切り捨て、最小1）
- `modify_hp(mon, v=-damage, reason="burn")` で呼び出す
  - reason="burn" により将来のたいねつ特性のやけどダメージ軽減に対応できる
  - v < 0 で `ON_MODIFY_NON_MOVE_DAMAGE` が発火され、マジックガード特性が正しく動作する
- subject_spec: `"source:self"` （EventContext）
- priority: 100
  - 根拠: `docs/spec/turn.md` の `Event.ON_TURN_END` より「やけどダメージ」は Priority 100

### 2. やけど_modifier（Event.ON_CALC_BURN_MODIFIER）

**ファイル**: `handlers/ailment.py`  
**登録**: `data/ailment.py`

- やけど状態のポケモンが使う物理技のダメージを0.5倍にする
- 技カテゴリが「物理」のとき `apply_fixed_modifier(value, 2048)` を適用
- subject_spec: `"attacker:self"` （AttackContext）
- priority: 100（デフォルト）

## テスト方針

- やけど状態付与後に `end_turn` を呼び、HP が `max_hp - max_hp // 16` になることを確認する
- 物理技のダメージがやけどで半減することを確認する
- 特殊技のダメージはやけどの影響を受けないことを確認する

## 注意点

- `modify_hp` に `v=0` を渡すと処理がスキップされるため、`max(1, ...)` で最小値1を保証する
- `r=-1/16` で浮動小数点除算を使うと最小値1が保証されないため、整数除算 `max_hp // 16` を使う
- ほのおタイプへの付与不可は `AilmentManager._can_apply_by_type` で実装済み
- マジックガード（`ON_MODIFY_NON_MOVE_DAMAGE` で0に変換）は特性側で実装済み
- たいねつ特性のやけどダメージ軽減（1/32）は今後 reason="burn" を使って実装予定
