# 特性実装計画: ポイズンヒール

更新日: 2026-05-03

## 仕様参照

- `spec/ability/ポイズンヒール.md`

## 実装ゴール

- どく/もうどく状態のターン終了時に、ポイズンヒール有効時はダメージを最大HPの1/8回復へ置換する。
- もうどくでも回復量は固定1/8（もうどくターン経過の影響を受けない）。
- かいふくふうじ中は回復しない。かつ毒ダメージへフォールバックしない（結果としてHP変化なし）。
- かがくへんかガス中はポイズンヒール無効化により通常のどく/もうどくダメージを受ける。

## 実装方針

- 新規イベント `ON_MODIFY_POISON_DAMAGE` を追加し、`battle.modify_hp(..., reason="poison")` の内部で最終HP変化量を確定する。
- どく/もうどくハンドラは「基準ダメージ量の計算」のみを担当し、`reason="poison"` で `battle.modify_hp` を呼ぶ。
- ポイズンヒールの置換ロジックは特性ハンドラに集約し、`ON_MODIFY_POISON_DAMAGE` の戻り値を上書きする。

### `ON_MODIFY_POISON_DAMAGE` 設計

- 発火タイミング: `StatusManager.modify_hp` 内、`ON_BEFORE_HEAL` / `ON_BEFORE_DAMAGE_APPLY` より前（`reason == "poison"` の場合のみ）
- コンテキスト: `BattleContext(target=対象ポケモン, hp_change=v, hp_change_reason="poison")`
- 入出力 `value`: 最終HP変化量（負=ダメージ、正=回復、0=変化なし）
- 初期値:
	- どく: `-max_hp // 8`
	- もうどく: `-(max_hp * elapsed_turns // 16)`
- ポイズンヒール有効時: `+max_hp // 8` に上書き
- 非有効時（通常時/かがくへんかガス中）: 初期値をそのまま返す
- 後段処理:
	- `v > 0` になった場合は通常どおり `ON_BEFORE_HEAL` を通す
	- `v < 0` の場合は通常どおり `ON_BEFORE_DAMAGE_APPLY` を通す
	- これにより、かいふくふうじが有効な場面でも既存の回復ブロックフローを維持できる

## 実装ステップ

1. `src/jpoke/enums/event.py` に `ON_MODIFY_POISON_DAMAGE` を追加する。
2. `src/jpoke/utils/type_defs.py` の `HPChangeReason` に `"poison"` を追加する。
3. `src/jpoke/core/pokemon_state.py` の `modify_hp` で、`reason == "poison"` のとき `ON_MODIFY_POISON_DAMAGE` を `ON_BEFORE_HEAL` / `ON_BEFORE_DAMAGE_APPLY` より前に発火する。
4. `src/jpoke/handlers/ability.py` に `ポイズンヒール_modify_poison_damage` を追加する。
5. `src/jpoke/data/ability.py` の「ポイズンヒール」に `ON_MODIFY_POISON_DAMAGE` ハンドラを登録する。
6. `src/jpoke/handlers/ailment.py` の `どく_damage` / `もうどく_damage` は基準ダメージ値を計算して `battle.modify_hp(..., reason="poison")` を呼ぶだけにする。
7. `ABILITY_TRIGGERED` の payload キーを `{"ability": "ポイズンヒール", "success": True}` に統一する。

## テスト観点

1. どく状態で最大HPの1/8を回復する（ダメージを受けない）。
2. もうどく状態でも最大HPの1/8を回復する（ターン数に関係なく固定）。
3. かいふくふうじ中は回復もダメージも受けない。
4. かがくへんかガス中は通常のどくダメージを受ける。
5. HPが全快の場合は回復が発動しない。
6. ポイズンヒール非所持時は既存どおりどく/もうどくダメージが入る。
