# あめふらし 実装計画

## スコープ
- 仕様書: `spec/ability/あめふらし.md`
- 優先度: H（天候始動）
- 対象: 入場時に雨へ変更する効果と継続ターン管理

## 実装方針
- `Event.ON_SWITCH_IN` で `common.activate_weather(weather="あめ")` を実行する。
- 天候の実際の有効化・同一天候時の不更新・継続ターン計算は `WeatherManager.activate` に委譲する。
- 特性無効化（かがくへんかガス等）は既存の `ON_CHECK_ABILITY_ENABLED` 基盤に従う。

## 実装ステップ
1. `src/jpoke/data/ability.py` の `あめふらし` に `ON_SWITCH_IN` ハンドラを登録。
2. `tests/test_ability.py` に「入場で雨化」「同一天候時にターン据え置き」を追加。
3. 既存の天候終了処理（`field.py`）と整合を確認。

## テスト観点
- 入場時に `battle.weather.name == "あめ"` になる。
- 初回発動時の `count` が 5 になる。
- 既に雨のとき再入場しても `count` が更新されない。
