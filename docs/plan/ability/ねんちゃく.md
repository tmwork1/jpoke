# ねんちゃく 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/ねんちゃく.md`
- 実装方針: 既存の `Event.ON_CHECK_ITEM_CHANGE` 共通判定を単一の制御点として維持し、未対応の持ち物変更経路だけをそこへ集約する。

## 現状整理
- `src/jpoke/core/battle.py` の `can_change_item()` が `Event.ON_CHECK_ITEM_CHANGE` を発火し、持ち物の交換・奪取・除去可否を共通判定している。
- `src/jpoke/data/ability.py` の `ねんちゃく` は `Event.ON_CHECK_ITEM_CHANGE` に登録済みで、`src/jpoke/handlers/ability.py` の `ねんちゃく_prevent_item_change()` が相手由来の持ち物変更を拒否する。
- 現在の共通経路でカバー済みの系統は `トリック/すりかえ`、`どろぼう/ほしがる`、`むしくい/ついばむ`、`はたきおとす`、`やきつくす`。
- `tests/test_ability.py` には `トリック`、`どろぼう`、`はたきおとす`、`かがくへんかガス` の基本ケースがある。

## 仕様要点（実装に反映）
- もちもの を奪われたり、失ったりする効果を防ぐ。
- 相手から受ける トリック ・ すりかえ を無効化する。自身が道具を持っていない場合でも無効化する。
- 相手から受ける ふしょくガス を無効化する。自身が道具を持っていない場合でも特性の表示が出る。
- どろぼう ・ ほしがる のダメージを受けてももちものを奪われない。
- むしくい ・ ついばむ のダメージを受けても きのみ を奪われない。
- はたきおとす ・ やきつくす ・ マジシャン ・ わるいてぐせ による持ち物喪失も防ぐ。

## 未対応・要確認
- `ふしょくガス` はまだコードベースに見当たらず、`remove_item(..., reason=...)` を呼ぶ経路自体が未実装。
- `マジシャン` と `わるいてぐせ` も未実装で、ねんちゃくの共通判定にまだ接続されていない。
- `ほしがる`、`むしくい`、`ついばむ`、`やきつくす` は実装経路上は保護されるが、ねんちゃくとの組み合わせテストが不足している。
- 「場にいるときしか発動しない」仕様は、今後 `ドラゴンテール`、`ともえなげ`、`レッドカード`、`わるいてぐせ` を実装した時点で再確認が必要。

## 実装フロー
1. `ねんちゃく` の本体は現状維持とし、新規の持ち物変更効果は必ず `battle.swap_items()` / `battle.take_item()` / `battle.remove_item()` を経由させる。
2. `ふしょくガス` を実装する際は、対象が道具未所持でも `ねんちゃく` の表示が出る仕様を満たせるよう、失敗前に `ON_CHECK_ITEM_CHANGE` を確認できる呼び出し順を設計する。
3. `マジシャン` と `わるいてぐせ` を実装する際は、発動条件判定の後で `battle.take_item(..., reason="steal")` に集約し、ねんちゃく・かたやぶり・特性無効化の判定を共通化する。
4. 交代後に発生する奪取処理を追加する場合は、ハンドラ解除後に `can_change_item()` が走ることを前提に、「場にいるときしか発動しない」仕様と一致するかを確認する。

## テスト観点
- 既存ケース: `トリック/すりかえ`、`どろぼう/ほしがる`、`むしくい/ついばむ`、`はたきおとす`、`やきつくす` をそれぞれ相手発動で防げる。
- 非発動ケース: 自分の `トリック`、通常の道具消費、`おちゃかい` など自発・通常消費では防がない。
- 無効化ケース: `かがくへんかガス` 中と `かたやぶり` 系統の技で防げない。
- 境界ケース: 対象が道具なしでも `トリック` と `ふしょくガス` で特性表示が出る。
- 将来ケース: 交代を挟む `マジシャン` / `わるいてぐせ` / 強制交代との相互作用が現行世代仕様と一致する。
