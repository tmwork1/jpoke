# てんねん 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/てんねん.md`
- 実装方針: 仕様の発動条件と例外条件を先に固定し、イベント単位で段階実装する。

## 実装対象イベント（第一候補）
- `DomainEvent.ON_MODIFY_MOVE_PRIORITY`
- `Event.ON_FIELD_CHANGE`
- `Event.ON_BEFORE_APPLY_AILMENT`
- `Event.ON_BEFORE_MODIFY_STAT`

## 仕様要点（実装に反映）
- このポケモンが技を使用するときは、相手の 回避率 ・ ぼうぎょ ・ とくぼう の ランク補正 を無視する。
- 他のポケモンから技を受けるときは、相手の 命中率 ・ こうげき ・ とくこう のランク補正を無視する。
- 相手の低下した ランク補正 も無視する。
- 素早さ のランク補正に関しては影響がない。
- この特性はポケモンの行動順に影響しない。

## 実装フロー
1. `src/jpoke/data/ability.py` の `てんねん` に上記イベントの `handlers` を登録する。
2. `src/jpoke/handlers/ability.py` に `てんねん` の本体処理を追加する。
3. 既存の共通ロジックで再利用できる処理は `src/jpoke/handlers/common.py` に分離する。
4. 特性無効化（`かがくへんかガス` / `とくせいなし`）と交代時リセットの相互作用を確認する。

## テスト観点
- 発動条件を満たすケースでのみ効果が出る。
- 仕様の例外条件（無効化・対象外条件）で誤発動しない。
- 解除条件を満たしたタイミングで効果が停止する。
- 既存システム（場状態・交代・ダメージ計算）と競合しない。
