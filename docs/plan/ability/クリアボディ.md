# クリアボディ の実装計画

## 概要
特性所持者の能力ランク低下を無効化する特性。ON_MODIFY_STATイベントで低下を防ぐ。

## 実装対象のイベント
1. **`ON_MODIFY_STAT`** - 能力ランク変動イベント

## 実装内容

### イベント: ON_MODIFY_STAT（能力ランク変動）
- **担当**: `handlers/ability.py` に新規ハンドラを追加
- **処理**:
  - 対象（ランク低下を受ける側）がクリアボディ所持 → 0 を返す（変化なし）
  - ただし、以下は防げない：
    - 自分の技による低下（やなどが自身のランクを下げる技）
    - 反動によるランク低下
    - 持ち物による低下（いのちのたまなど）
    - 状態異常による能力低下（まひによる素早さ低下など）
  - 既に下がったランクは自動回復しない
  - `かたやぶり` 効果付き技では無視されて低下を受ける

### 実装順序
1. `handlers/ability.py` に `クリアボディ_modify_stat` ハンドラを追加
2. 修正値 < 0 の低下判定を組み込む
3. ランク低下の「直接低下」判定ロジック（source_spec による判定）を実装
4. `AbilityData` に handlers を登録
5. テスト追加

## 実装の詳細ポイント
- `return HandlerReturn(value=0)` でランク変化を無効化
- 修正値が負数（低下）かチェック: `value < 0`
- source_spec で外部からの低下か判定（target:self が特定条件の場合のみ防ぐ）
- イベント発火側で `source_spec="attacker:self"` など攻撃側からの低下が指定される場合にブロック
- `かたやぶり` チェックは事前に実施（本ハンドラは事後）

## テスト項目
- いかくの攻撃低下を防ぐ ✓
- 追加効果の能力低下を防ぐ ✓
- 変化技のランク低下を防ぐ ✓
- かたやぶり技では防げない
- 自己低下技は防げない ✓（実装完了）
- 複数ランク低下が同時適用される場合

## 実装状況
**完了**: クリアボディは相手由来の能力低下のみを防ぐ仕様で正確に実装されました。
- 相手ポケモンが原因（source ≠ target）の能力低下を防ぐ
- 自分の技や反動による自己低下は防がない（source == target または source is None の場合）
- テストにより仕様適合性を確認済み
