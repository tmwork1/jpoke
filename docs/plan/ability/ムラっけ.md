# 特性実装計画: ムラっけ

更新日: 2026-05-02

## 仕様参照

- `spec/ability/ムラっけ.md`

## 実装ゴール

- ターン終了時、攻撃/防御/特攻/特防/すばやさのいずれか1つをランダムに+2し、別の1つを-1する。
- 第八世代以降は命中・回避率はランダム対象に含まない。
- 各ランクの+6/−6境界を超えないように処理する。

## 実装方針

- `ON_TURN_END_3` で対象リスト（攻撃・防御・特攻・特防・すばやさ）からランダムに1つ選んで+2し、残りからランダムに1つ選んで-1する。
- ランダムは乱数シードによって再現可能な形で実装する（テスト用にシードを固定できるようにする）。
- ランク境界処理は既存の `apply_stat_change` の境界チェックに依存する。

## 変更候補

- `src/jpoke/data/ability.py`
- `src/jpoke/handlers/ability.py`
- `tests/test_ability.py`

## テスト観点

1. ターン終了時にいずれか1つの能力が+2になる。
2. 別の1つの能力が-1になる。
3. +2と-1の対象が同じ能力にならない。
4. ランク+6を超えない / ランク-6を下回らない。
5. かがくへんかガス発動中は発動しない。

## 保留・論点

- テストの再現性のため、乱数固定（`random.seed` またはモック）の方針を決める必要がある。
