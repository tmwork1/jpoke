## 実装概要
- ほのお技を無効化する。
- 吸収後、最初のほのお技は威力が1.5倍になる。
- 2回目以降は等倍に戻る。
- 交代で状態がリセットされる。
- かたやぶり相手には吸収できない（抜き型的に機能する）。

## 状態設計（Literal型 + 汎用 state）
`Ability.state`（単一の文字状態）を Literal 型で管理。
- `"idle"`: 未チャージ（初期状態、交代時）
- `"charged"`: 吸収済み（次のほのお技1回を強化）
- `"active"`: この行動中だけ強化適用中
- 型定義: `FlashFireState = Literal["idle", "charged", "active"]`

## イベント設計（採用版：ON_MOVE_END 導入）
| イベント | 役割 | 遷移 |
| --- | --- | --- |
| `ON_CHECK_IMMUNE` | ほのお技吸収 | `idle/active -> charged` |
| `ON_MOVE_CHARGE` | ほのお技使用時に強化適用予約 | `charged -> active` |
| `ON_CALC_POWER_MODIFIER` | ほのお技威力1.5倍（`state=="active"` のときだけ） | - |
| `ON_MOVE_END` | 行動終了時の強化消費 | `active -> idle` |

**利点**:
- `ON_MOVE_CHARGE`: 「この行動で何をするか」に集中（準備のみ）
- `ON_MOVE_END`: 「行動終了後の撤去」を明確に分離
- 責務が明確で、他の特性にも応用しやすい

## 実装方針（実装済み）

### ハンドラ実装
1. **`もらいび_check_immune()`**  
   - ほのお技を吸収し、`ability.state = "charged"` に遷移。
   - かたやぶり相手には発動しない。

2. **`もらいび_on_switch_in()`**  
   - ポケモン登場時に `ability.state = "idle"` に初期化。

3. **`もらいび_on_move_charge()`**  
   - ほのお技使用時に `state == "charged"` なら `active` に遷移。
   - 準備処理のみ（前回の消費はしない）。

4. **`もらいび_on_move_end()`** ← NEW
   - `state == "active"` なら `idle` に消費済みにする。
   - 前回行動の状態をクリア。

5. **`もらいび_modify_power()`**  
   - ほのお技で`state == "active"` のときのみ威力を1.5倍。

### モデル層
- `Ability.state: AbilityState` を追加。
- `AbilityState` は単一文字の Literal（`"" | "idle" | "charged" | "active"`）。
- `bench_reset()` で state を初期化。

### イベント層
- `Event` に `ON_MOVE_END = auto()` を追加（`ON_DAMAGE` の直後）。
- `MoveExecutor._execute_move()` の終了時に `ON_MOVE_END` を emit。
- もらいび `data/ability.py` で `ON_MOVE_END` ハンドラを登録。
