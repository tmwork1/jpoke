# ひでり 実装計画

## スコープ
- 仕様書: `spec/ability/ひでり.md`
- 優先度: H（天候始動）
- 対象: 入場時に晴れへ変更する効果と継続ターン管理

## 実装方針
- `Event.ON_SWITCH_IN` で `common.activate_weather(weather="はれ")` を実行する。
- 同一天候時の不更新は `WeatherManager.activate` の共通仕様を利用する。
- 天候ダメージ・威力補正は既存の `data/field.py` 側ハンドラに任せる。

## 実装ステップ
1. `src/jpoke/data/ability.py` の `ひでり` に `ON_SWITCH_IN` ハンドラを登録。
2. `tests/test_ability.py` に「入場で晴れ化」「同一天候時にターン据え置き」を追加。
3. 既存の天候処理と回帰テストを実施。

## テスト観点
- 入場時に `battle.weather.name == "はれ"` になる。
- 初回発動時の `count` が 5 になる。
- 既に晴れのとき再入場しても `count` が更新されない。
