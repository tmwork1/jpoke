# 特性実装計画: はじまりのうみ

更新日: 2026-05-01

## 仕様参照

- `spec/ability/はじまりのうみ.md`
- `spec/global_field/おおあめ.md`

## 実装ゴール

- `Event.ON_SWITCH_IN` で おおあめ を発動する。
- おおあめ 中は ほのおタイプ攻撃技を失敗扱いにする。
- 通常天候では上書き不可、特殊天候（おおひでり / らんきりゅう）では上書き可を満たす。

## 実装方針

- あめふらし実装をベースに、強天候タイプへ切り替える。
- 技失敗判定は おわりのだいち と対称な実装にそろえる。
- 天候依存技（かみなり / ぼうふう / ウェザーボール / ソーラー系）は既存の雨効果導線へ乗せる。

## イベント別実装計画（turn.md準拠）

1. Event.ON_SWITCH_IN（Priority 100）
- 特性 `はじまりのうみ` に `common.activate_weather(weather="おおあめ")` を接続する。
- 既存の通常天候始動特性と同優先度帯で処理し、同時発動時は既存のイベント順序に委ねる。

2. Event.ON_CHECK_MOVE（Priority 10）
- `おおあめ・おおひでりによる技失敗` 行に対応するハンドラを追加する。
- `おおあめ` 中は「ほのおタイプの攻撃技」を失敗扱いにし、`stop_event=True` で後続処理を止める。

3. Event.ON_CHECK_IMMUNE（Priority 100）
- `天気変更技: おおひでり等で失敗` 行に対応するハンドラを追加する。
- 強天候中に通常天候技（あまごい/にほんばれ/すなあらし/ゆきげしき）を失敗させる。

4. Event.ON_CALC_POWER_MODIFIER
- 既存の雨補正導線を `おおあめ` にも適用する（みず1.5倍、ほのお0.5倍）。
- ただし `ON_CHECK_MOVE` で失敗した技はダメージ計算に進まないため、実際の攻撃失敗と矛盾しない。

5. Event.ON_MODIFY_ACCURACY
- かみなり/ぼうふうの命中補正で `おおあめ` を「あめ相当」として扱う。

6. Event.ON_TURN_END_1
- `おおあめ` は強天候としてカウントダウンしない（`tick_weather` 非登録）。

7. Event.ON_CHECK_ABILITY_ENABLED / Event.ON_FIELD_CHANGE
- ノーてんき/エアロックの既存有効判定導線に乗せ、天候状態は保持しつつ効果のみ抑制する方針を維持する。

## テスト観点

1. 登場時に おおあめ が発動する。
2. ほのおタイプ攻撃技が失敗する。
3. 通常天候で上書きできない。
4. おわりのだいち との競合で後発天候が残る。
5. ノーてんき / エアロック同居時に状態は残り効果は止まる。

## 保留・論点

- 強天候名だけで分岐が増えないよう、天候効果判定の共通化を先行する。

## 実装ステップ

1. 型定義と天候データへ `おおあめ` を追加。
2. 特性データ（はじまりのうみ）に `ON_SWITCH_IN` ハンドラを接続。
3. `ON_CHECK_MOVE` / `ON_CHECK_IMMUNE` の強天候判定ハンドラを実装。
4. 既存の雨参照ロジック（命中/威力）を `おおあめ` 対応へ拡張。
5. ability/global_field/move テストを追加して回帰確認。
