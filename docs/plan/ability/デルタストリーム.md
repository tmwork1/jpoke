# 特性実装計画: デルタストリーム

更新日: 2026-05-01

## 仕様参照

- `spec/ability/デルタストリーム.md`
- `spec/global_field/らんきりゅう.md`

## 実装ゴール

- `Event.ON_SWITCH_IN` で らんきりゅう を発動する。
- らんきりゅう 中は、ひこうタイプの弱点（でんき / いわ / こおり）に対する被ダメージ補正を適用する。
- 通常天候では上書き不可、特殊天候（おおひでり / おおあめ）では上書き可を満たす。

## 実装方針

- おわりのだいち / はじまりのうみ と同じ強天候導線へ統合し、天候種別として らんきりゅう を追加する。
- 弱点緩和はタイプ相性計算の後段で適用し、固定割合ダメージや非ひこうタイプには影響させない。
- ノーてんき / エアロック下では「天候状態は残し、効果だけ無効化」の共通判定に従う。

## イベント別実装計画（turn.md準拠）

1. Event.ON_SWITCH_IN（Priority 100）
- 特性 `デルタストリーム` に `common.activate_weather(weather="らんきりゅう")` を接続する。

2. Event.ON_CHECK_IMMUNE（Priority 100）
- `天気変更技: おおひでり等で失敗` 行に対応するハンドラを追加する。
- 強天候中に通常天候技を失敗させる（らんきりゅう中も同様）。

3. Event.ON_CALC_DEF_TYPE_MODIFIER
- らんきりゅう中に、対象がひこうタイプを含み、かつ技タイプが「でんき/いわ/こおり」のとき相性倍率を半減する。
- 適用タイミングはタイプ相性計算の後段（イベント補正）に置く。
- これにより 2倍弱点は等倍、4倍弱点は2倍になる。

4. Event.ON_CALC_POWER_MODIFIER / Event.ON_MODIFY_MOVE_TYPE（確認項目）
- らんきりゅう中のウェザーボールが無天候相当（ノーマル50）になることを確認する。
- 必要なら専用分岐を追加するが、まず既存導線で成立するかを先に検証する。

5. Event.ON_TURN_END_1
- `らんきりゅう` は強天候としてカウントダウンしない（`tick_weather` 非登録）。

6. Event.ON_CHECK_ABILITY_ENABLED / Event.ON_FIELD_CHANGE
- ノーてんき/エアロック有効時は相性軽減効果を無効化する。
- ただし天候名そのものは保持し、再有効化時に復帰できる設計を維持する。

## テスト観点

1. 登場時に らんきりゅう が発動する。
2. ひこうタイプの弱点被弾が緩和される（2倍→等倍、4倍→2倍）。
3. 非ひこうタイプに影響しない。
4. 通常天候で上書きできない。
5. おおひでり / おおあめ との競合で後発天候が残る。
6. ノーてんき / エアロック同居時に状態は残り効果は止まる。

## 保留・論点

- じゃくてんほけん など「弱点判定」に依存する処理の評価タイミングを相性補正後で統一する。
- ウェザーボールや回復技の天候依存挙動が中立値になることを、既存天候分岐と干渉しない形で固定する。

## 実装ステップ

1. 型定義と天候データへ `らんきりゅう` を追加。
2. 特性データ（デルタストリーム）に `ON_SWITCH_IN` ハンドラを接続。
3. `ON_CALC_DEF_TYPE_MODIFIER` に弱点緩和ハンドラを実装。
4. `ON_CHECK_IMMUNE` の天気変更技失敗ハンドラを強天候共通で実装。
5. じゃくてんほけんを含む相性依存の回帰テストを追加。
