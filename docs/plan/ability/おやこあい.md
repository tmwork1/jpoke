# おやこあい 実装計画

## スコープ
- 優先度: H
- 仕様書: `spec/ability/おやこあい.md`
- 実装方針: ヒット数決定は `ON_MODIFY_HIT_COUNT` に委譲し、`おやこあい` はハンドラで「疑似2ヒット化 + 2発目ダメージ減衰」を付与する。

## 実装ゴール（完了定義）
- 単体対象の単発攻撃技が 2 ヒット化する。
- 2 ヒット目のみ固定倍率でダメージが減衰する。
- 既存連続技（`min_hits/max_hits > 1`）は非対象のまま。
- miss、1ヒット目で相手ひんし、特性無効化中では 2 ヒット目が発生しない。
- 既存テスト（特に `tests/test_multihit.py`）を壊さない。

## 事前に固定する仕様値
- 2ヒット目ダメージ倍率: `PARENTAL_BOND_SECOND_HIT_NUM` / `PARENTAL_BOND_SECOND_HIT_DEN`
- 初期値候補: 1/4（世代差分対応が必要なら定数を差し替え可能にする）
- 実装位置候補: `src/jpoke/utils/constants.py`

## 変更対象（ファイル/関数単位）
- `src/jpoke/enums/event.py`
	- 新規イベント `Event.ON_MODIFY_HIT_COUNT` を追加。
- `src/jpoke/core/move_executor.py`
	- `_resolve_hit_count` の責務を「ベース値算出 + `ON_MODIFY_HIT_COUNT` emit」に限定。
	- 特性・道具の固有分岐（スキルリンク、いかさまダイス、おやこあい）を直接書かない。
- `src/jpoke/handlers/ability.py`
	- 新規: `おやこあい_modify_hit_count`（適用可なら 2 を返す）
	- 新規: `おやこあい_modify_damage`（2ヒット目だけ倍率適用）
- `src/jpoke/data/ability.py`
	- `おやこあい` に `Event.ON_MODIFY_HIT_COUNT` と `Event.ON_MODIFY_DAMAGE` を登録。
- `src/jpoke/handlers/item.py` / `src/jpoke/data/item.py`
	- `いかさまダイス` のヒット数補正を `ON_MODIFY_HIT_COUNT` へ移設。
- `src/jpoke/handlers/ability.py` / `src/jpoke/data/ability.py`
	- `スキルリンク` のヒット数補正を `ON_MODIFY_HIT_COUNT` へ移設。

## 対象技判定ルール（実装条件）
- 非適用条件（最小）:
  - 変化技
  - 既存連続技（`move.data.max_hits > 1`）
- 上記以外は適用候補として扱い、特性有効/無効は既存の `check_ability_enabled` 経路で判定する。

## 実装ステップ
1. `Event.ON_MODIFY_HIT_COUNT` を追加する。
2. `move_executor._resolve_hit_count` を次の形に変更する。
   - ベースヒット数を算出（技データ由来の通常乱数）
   - `events.emit(Event.ON_MODIFY_HIT_COUNT, ctx, base_hit_count)` を実行
   - 返り値を最終ヒット数として採用
3. `スキルリンク` と `いかさまダイス` の既存ハードコードを削除し、各ハンドラへ移す。
4. `おやこあい_modify_hit_count` を追加し、非適用条件に該当しなければ 2 を返す。
5. `おやこあい_modify_damage` を追加し、`ctx.hit_index == 2` のときだけ倍率適用する。
6. 既存テスト + 追加テストを実行し、回帰なしを確認する。

## 疑似コード
```python
def _resolve_hit_count(attacker, move):
    base_hit_count = _resolve_base_hit_count(attacker, move)
    return events.emit(Event.ON_MODIFY_HIT_COUNT, ctx, base_hit_count)

def おやこあい_modify_hit_count(battle, ctx, value):
    if not ctx.move.is_attack:
        return value
    if ctx.move.data.max_hits > 1:
        return value
    return 2

def おやこあい_modify_damage(battle, ctx, value):
    if ctx.hit_count < 2 or ctx.hit_index != 2:
        return value
    return value * NUM // DEN
```

## リスクと対策
- リスク: ヒット数ロジックをイベント化すると優先度競合が起きる。
	- 対策: `ON_MODIFY_HIT_COUNT` の handler priority を明示し、優先順を固定する。
- リスク: `MoveData` を直接変更すると全体へ副作用が出る。
	- 対策: `MoveData` は変更せず、ヒット数はイベント戻り値で制御する。
- リスク: 既存連続技の命中判定仕様を壊す。
	- 対策: 既存テスト `tests/test_multihit.py` を回帰セットに含める。
- リスク: 世代差で2ヒット目倍率が変わる。
	- 対策: 倍率は定数化し、差分吸収しやすくする。
