# マジックガード 実装計画

## スコープ
- 優先度: H
- 仕様書: `spec/ability/マジックガード.md`
- 実装方針: HP変更の共通経路で `reason` を見て、間接ダメージだけを一括で無効化する。

## 実装対象（共通ゲート方式）

| 層 | 対応箇所 | 目的 |
|----|----------|------|
| HP変更共通経路 | `src/jpoke/core/pokemon_state.py` の `StatusManager.modify_hp` | `マジックガード` による一括無効化を実施する |
| 呼び出し側の理由付与 | 各 `battle.modify_hp(..., reason=...)` 呼び出し | `move_damage` / `pain_split` を正しく渡す |

## 仕様要点（実装に反映）
- 攻撃技の本体ダメージは通常どおり受ける。
- はらだいこ・みがわりのような自己HP消費は「間接ダメージ」ではないため通常どおり発生する。
- `かがくへんかガス` 等で特性が無効化されている間は間接ダメージを受ける。

## 実装アプローチ
`StatusManager.modify_hp` に以下の共通判定を追加する。

1. `v < 0`（ダメージ）かつ `reason` が `move_damage` / `pain_split` / `self_attack` / `self_cost` 以外なら判定対象にする。
2. 対象ポケモンの現在特性が `マジックガード` なら HP 減少を 0 にして終了する。
3. それ以外は既存どおり処理する。

概念コード:

```python
EXCLUDED_REASONS = {"move_damage", "pain_split", "self_attack", "self_cost"}

def modify_hp(self, target, v=0, r=0, reason="other"):
    # ... 既存の v / r 計算
    if v < 0 and reason not in EXCLUDED_REASONS and target.ability.name == "マジックガード":
        return 0
    # ... 既存処理
```

この方式により、個別ハンドラへ同じガードを重複実装せずに済み、漏れが減る。
また、自己HP消費は `self_cost` として明示して、マジックガード対象から確実に除外する。

## 実装フロー
1. `src/jpoke/core/pokemon_state.py` の `StatusManager.modify_hp` に、reason ベース共通ゲートを追加する。
2. `src/jpoke/utils/type_defs.py` の `HPChangeReason` に `self_cost` を追加する。
3. 既存呼び出しのうち `move_damage` / `pain_split` / `self_attack` / `self_cost` になる経路で reason が崩れないことを確認する。
4. 自己HP消費処理に `reason="self_cost"` を付与する。
    - `handlers/volatile.py`: `みがわり_apply`
    - `handlers/move.py` または関連処理: はらだいこ、その他自己HP支払い技
5. 必要に応じて呼び出し側の reason を付与・置換する。
   - `handlers/ailment.py`: やけど/どく/もうどく
   - `handlers/field.py`: すなあらし、まきびし、ステルスロック
   - `handlers/volatile.py`: のろい
   - `handlers/item.py`: いのちのたま反動
6. `src/jpoke/data/ability.py` に `マジックガード` を実装済み特性として追加する。
7. `tests/test_ability.py` に回帰テストを追加する。
8. 実装数が変わる場合のみ `script/dashboard.py` を実行して集計を更新する。

## テスト観点
- `HPChangeReason=move_damage` は無効化しない。
- `HPChangeReason=self_attack` は無効化しない。
- `HPChangeReason=pain_split` は無効化しない。
- `HPChangeReason=self_cost` は無効化しない。
- `HPChangeReason=other` は無効化する。
- `HPChangeReason=other` の代表入力（どく/やけど/すなあらし/ステルスロック/いのちのたま反動）で回帰する。
- 同一 `HPChangeReason`（例: `other`）で、`かがくへんかガス` 有効時のみ無効化されないことを確認する。
