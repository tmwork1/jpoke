# フェアリーロック 実装計画書

## 概要

全体の場の状態の一つ。発動した次のターンが終了するまで、場に出ている全ポケモン（ゴーストタイプ含む）の交代・にげるを禁止する。

## 実装対象イベント

### 1. ON_CHECK_TRAPPED（交代禁止）

**優先度**: priority=100（デフォルト）
**subject_spec**: `"source:self"`
**実装関数**: `フェアリーロック_check_trapped`
**場所**: `handlers/field.py`

- フィールドが有効な間、全ポケモンの交代・にげるを禁止する
- ゴーストタイプも含む（`True` を返すだけでよい）
- `HandlerReturn(value=True)` を返す

### 2. ON_TURN_END（カウントダウン・終了）

**優先度**: priority=140（docs/spec/turn.md 準拠）
**subject_spec**: `"source:self"`
**実装関数**: `フェアリーロック_tick_global_field`（共通 tick_global_field のラッパー）
**場所**: `handlers/field.py`

## 技のデータ登録（data/move.py）

```python
"フェアリーロック": MoveData(
    type="フェアリー",
    category="status",
    pp=10,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.フェアリーロック_activate_global_field,
        ),
    }
),
```

`ON_STATUS_HIT` ハンドラ内で `battle.global_manager.activate("フェアリーロック", 1)` を呼び出す。
フェアリーロックがすでに有効な場合は `False` を返すため、技が失敗する。

## データ登録（data/field/global_field.py）

```python
"フェアリーロック": FieldData(
    handlers={
        Event.ON_CHECK_TRAPPED: h.FieldHandler(
            h.フェアリーロック_check_trapped,
            subject_spec="source:self",
        ),
        Event.ON_TURN_END: h.FieldHandler(
            h.フェアリーロック_tick_global_field,
            subject_spec="source:self",
            priority=140,
        ),
    },
),
```
