## ハンドラ一覧

### 1. ゆき_boost_def（こおりタイプ物理防御1.5倍）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_CALC_DEF_MODIFIER |
| subject_spec | `defender:self`（AttackContext: 攻撃フェーズ） |
| priority | 100（デフォルト） |

**ロジック:**
1. `battle.weather.name == "ゆき"` かつ `ctx.defender.has_type("こおり")` かつ `ctx.move.category == "physical"` → `apply_fixed_modifier(value, 6144)`（×1.5）

**注意:**
- `ON_CALC_DEF_MODIFIER` は攻撃フェーズイベントなので subject_spec は `defender:self`。
- `battle.weather.name` チェックはエアロック・ノーてんき対応のため必須（`battle.weather` はこれらの特性中に `""` を返す）。
- ばんのうがさとの相互作用はなし（はれ/あめの威力補正とは異なる）。

### 2. tick_weather（カウントダウン）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_TURN_END |
| subject_spec | `source:self`（EventContext: 非攻撃フェーズ） |
| priority | 10 |
| 根拠 | docs/spec/turn.md: 「にほんばれ等の天気終了」Priority 10 |

**ロジック:** 1P側のみ `battle.weather_manager.tick_down()` を呼ぶ（共通関数 `tick_weather` を使用）。

## data/field/weather.py 登録

```python
"ゆき": FieldData(
    handlers={
        Event.ON_CALC_DEF_MODIFIER: h.FieldHandler(
            h.ゆき_boost_def,
            subject_spec="defender:self",
        ),
        Event.ON_TURN_END: h.FieldHandler(
            h.tick_weather,
            subject_spec="source:self",
            priority=10,
        ),
    },
),
```
