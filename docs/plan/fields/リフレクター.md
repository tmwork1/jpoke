## 実装の詳細

### リフレクター_reduce_damage

```python
def リフレクター_reduce_damage(battle, ctx, value):
    if (
        not ctx.critical
        and not ctx.can_bypass_screen(battle)
        and ctx.move.category == "physical"
    ):
        value = apply_fixed_modifier(value, 2048)
    return HandlerReturn(value=value)
```

- 急所（`ctx.critical`）の場合は軽減しない
- すりぬけ等の壁貫通（`ctx.can_bypass_screen`）の場合は軽減しない
- 技カテゴリが "physical" の場合のみ軽減する（特殊技は軽減しない）
- apply_fixed_modifier(value, 2048) = 1/2倍（2048/4096）

### リフレクター_tick_side_field

```python
def リフレクター_tick_side_field(battle, ctx, value):
    return tick_side_field(battle, ctx, value, name="リフレクター")
```

- ターン終了時に「リフレクター」のカウントを1減らす
- priority=130 は docs/spec/turn.md の壁カウントダウン順に従う

### リフレクター_set_side_field（move_status.py）

```python
def リフレクター_set_side_field(battle, ctx, value):
    side = battle.get_side(ctx.attacker)
    if not side.apply("リフレクター", 5, source=ctx.attacker):
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

- 5ターンを初期値として `side.apply()` を呼ぶ
- `side.apply()` 内で ON_MODIFY_DURATION イベントが発火し、ひかりのねんどがあれば 8ターンに延長される
- すでに発動中の場合は `side.apply()` が False を返し技が失敗する

## 特記事項

- オーロラベールとの重複を防ぐロジックはオーロラベール側ハンドラに実装済み
- ひかりのねんどによる8ターン延長は `data/item.py` のハンドラが担当
