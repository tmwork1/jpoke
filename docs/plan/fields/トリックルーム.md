# トリックルーム 実装計画書

## 概要

全体の場の状態の一つ。発動中すばやさが低いポケモンほど先に行動できるようになる。
技の優先度は通常通り適用される。

## 実装対象イベント

### 1. ON_CHECK_SPEED_REVERSE（素早さ反転）

**優先度**: priority=100（デフォルト）
**subject_spec**: `"source:self"`
**実装関数**: `トリックルーム_reverse_speed`
**場所**: `handlers/field.py`

- 実効素早さの値を負数化して返す（`value → -value`）
- これにより降順ソートで低速ポケモンが先攻になる
- `EventContext(source=mon)` から呼ばれるため `subject_spec="source:self"` を使用

### 2. ON_TURN_END（カウントダウン・終了）

**優先度**: priority=140（docs/spec/turn.md 準拠）
**subject_spec**: `"source:self"`
**実装関数**: `トリックルーム_tick_global_field`（共通 tick_global_field のラッパー）
**場所**: `handlers/field.py`

## 技ハンドラ（move_status.py）

`トリックルーム_activate_global_field` がトリックルームの発動・解除を担当する。

- トリックルームが未発動 → `global_manager.activate("トリックルーム", 5)` で発動
- トリックルームが発動中 → `global_manager.deactivate("トリックルーム")` で解除
- `Event.ON_STATUS_HIT` で呼ばれる（`data/move.py` に登録）

## データ登録（data/field/global_field.py）

```python
"トリックルーム": FieldData(
    handlers={
        DomainEvent.ON_CHECK_SPEED_REVERSE: h.FieldHandler(
            h.トリックルーム_reverse_speed,
            subject_spec="source:self",
        ),
        Event.ON_TURN_END: h.FieldHandler(
            h.トリックルーム_tick_global_field,
            subject_spec="source:self",
            priority=140,
        ),
    },
),
```
