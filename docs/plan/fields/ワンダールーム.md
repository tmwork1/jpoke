## 実装方針

ダメージ計算（`damage.py` の `_calc_final_defense`）で:
1. 最初に `deals_physical_damage` で物理/特殊を判定し、参照する基本ステータス (`def` / `spd`) を選択する
2. `ON_CALC_DEF_RANK_MODIFIER` イベントでランク補正を上書きする
3. `ON_CALC_DEF_MODIFIER` イベントで実数値の比率補正を掛ける

ワンダールームは 2. と 3. のイベントハンドラで防御/特防の参照先を入れ替える。

## ハンドラ実装詳細

### `ワンダールーム_def_rank_modifier`

```python
def ワンダールーム_def_rank_modifier(battle, ctx, value):
    category_to_stat = {"physical": "spd", "special": "def"}
    swapped_stat = category_to_stat.get(ctx.move.category)
    value = ctx.defender.rank_modifier(swapped_stat)
    return HandlerReturn(value=value)
```

- 物理技: 特防ランク補正を返す
- 特殊技: 防御ランク補正を返す

### `ワンダールーム_def_modifier`

```python
def ワンダールーム_def_modifier(battle, ctx, value):
    base_stat = "def" if battle.query.deals_physical_damage(ctx.attacker, ctx.move) else "spd"
    swapped_stat = "spd" if base_stat == "def" else "def"
    base_value = max(1, ctx.defender.stats[base_stat])
    swap_value = max(1, ctx.defender.stats[swapped_stat])
    return HandlerReturn(value=value * swap_value // base_value)
```

- `value` は 4096 ベースの修正値（初期値 4096）
- `swap_value / base_value` の比率を整数演算で適用することで、最終的に swap_stat の実数値が参照される

### `ワンダールーム_tick_global_field`

```python
def ワンダールーム_tick_global_field(battle, ctx, value):
    return tick_global_field(battle, ctx, value, name="ワンダールーム")
```

### `ワンダールーム_activate_global_field`（move_status.py）

```python
def ワンダールーム_activate_global_field(battle, ctx, value):
    manager = battle.global_manager
    if manager.fields["ワンダールーム"].is_active:
        return HandlerReturn(value=manager.deactivate("ワンダールーム"))
    return HandlerReturn(value=manager.activate("ワンダールーム", 5))
```
