# じゅうりょく 実装計画書

## 概要

全体の場の状態の一つ。命中率の上昇・全ポケモンの接地化・一部技の使用禁止・Gのちから強化の効果を持つ。

## 実装対象イベント

### 1. ON_FIELD_ACTIVATE（揮発性状態解除）

**優先度**: priority=100（デフォルト）
**subject_spec**: `"source:self"`
**実装関数**: `じゅうりょく_remove_volatiles`
**場所**: `handlers/field.py`

- 各プレイヤーのアクティブポケモンからそらをとぶ・でんじふゆう揮発性状態を解除する
- `battle.volatile_manager.remove(mon, volatile_name)` を使用
- 揮発性状態が存在しない場合は何もしない（idempotent）

### 2. Gのちから威力補正（実装方針変更: フィールド側ではなく技側で実装）

**優先度**: priority=100（デフォルト）
**subject_spec**: `"attacker:self"`
**実装関数**: `Gのちから_gravity_boost`
**場所**: `handlers/move_attack.py`（`data/moves/move_ha.py` の `"Gのちから"` エントリの
`ON_CALC_POWER_MODIFIER` に登録）

- `battle.get_global_field("じゅうりょく").is_active` が True の場合、威力を1.5倍
  （固定倍率 6144/4096）にする
- Gのちからは唯一の対象技のため、フィールド側の汎用ハンドラではなく技側の専用ハンドラとして
  実装した（他の威力補正技と同じ実装パターンに統一するため）

### 3. 使用禁止技ブロック（実装方針変更: フィールド側ではなく技側で実装）

**優先度**: priority=100（デフォルト）
**subject_spec**: `"attacker:self"`
**実装関数**: `gravity_restricted_fail`
**場所**: `handlers/move.py`（`gravity_restricted` フラグを持つ各技の `ON_TRY_MOVE_1` に
個別登録）

- `battle.get_global_field("じゅうりょく").is_active` が True の場合、技を失敗させる
- ログ: `LogCode.MOVE_FAILED` reason="じゅうりょく"
- 対象技ごとに `flags={"gravity_restricted", ...}` と
  `Event.ON_TRY_MOVE_1: h.MoveHandler(h.gravity_restricted_fail)` を個別登録する方式とし、
  フィールド側の汎用ハンドラは持たない（技ごとのフラグ判定パターンに統一するため）

### 4. ON_MODIFY_ACCURACY（命中率補正）

**優先度**: priority=100（デフォルト）
**subject_spec**: `"attacker:self"`
**実装関数**: `じゅうりょく_modify_accuracy`
**場所**: `handlers/field.py`

- 命中率を約1.67倍（固定倍率 6840/4096）にする
- `apply_fixed_modifier(value, 6840)` を使用
- 一撃必殺技（`ctx.move.has_flag("ohko")`）は対象外（威力補正なし。ゲーム内の
  「一撃必殺技の命中は上がらない」仕様に対応）

### 5. ON_CHECK_FLOATING（接地化）

**優先度**: priority=100（デフォルト）
**subject_spec**: `"source:self"`
**実装関数**: `じゅうりょく_grounded`
**場所**: `handlers/field.py`

- 全ポケモンを接地状態（浮いていない）にする
- `HandlerReturn(value=False, stop_event=True)` を返す

### 6. ON_TURN_END（カウントダウン・終了）

**優先度**: priority=140（docs/spec/turn.md 準拠）
**subject_spec**: `"source:self"`
**実装関数**: `じゅうりょく_tick_global_field`（共通 tick_global_field のラッパー）
**場所**: `handlers/field.py`

## データ登録（data/field/global_field.py）

Gのちから威力補正・使用禁止技ブロックは技側で個別実装するため、フィールド側の
`GLOBAL_FIELD["じゅうりょく"]` には4つのハンドラのみを登録する:

```python
"じゅうりょく": FieldData(
    handlers={
        Event.ON_FIELD_ACTIVATE: h.FieldHandler(
            h.じゅうりょく_remove_volatiles,
            subject_spec="source:self",
        ),
        Event.ON_MODIFY_ACCURACY: h.FieldHandler(
            h.じゅうりょく_modify_accuracy,
            subject_spec="attacker:self",
        ),
        Event.ON_CHECK_FLOATING: h.FieldHandler(
            h.じゅうりょく_grounded,
            subject_spec="source:self",
        ),
        Event.ON_TURN_END: h.FieldHandler(
            h.じゅうりょく_tick,
            subject_spec="source:self",
            priority=140,
        ),
    },
),
```

技側のデータ登録例（`data/moves/move_ha.py` の `"はねる"` エントリ、`data/moves/move_ha.py` の
`"Gのちから"` エントリ）:

```python
"はねる": MoveData(
    ...
    flags={"gravity_restricted"},
    handlers={
        Event.ON_TRY_MOVE_1: h.MoveHandler(
            h.gravity_restricted_fail,
        ),
    },
),
```

```python
"Gのちから": MoveData(
    ...
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
            ha.Gのちから_gravity_boost,
        ),
        ...
    },
),
```
