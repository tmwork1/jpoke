`docs/spec/fields/ねがいごと.md`（フィールド本体の仕様）、`docs/spec/moves/ねがいごと.md`（技としての基本情報）を参照。

- ハンドラ登録（技）: `src/jpoke/data/moves/move_na.py`

## ハンドラ一覧

### 1. ねがいごと_can_apply（失敗チェック）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_BEFORE_APPLY_MOVE |
| subject_spec | `attacker:self`（MoveHandler デフォルト） |
| priority | 100（デフォルト） |
| 根拠 | docs/spec/turn.md ON_APPLY_MOVE「ねがいごと: 前ターン効果残り」Priority 100 相当 |

**ロジック:**
1. 使用者 `ctx.attacker` の自陣 `side` を取得
2. `side.get("ねがいごと").is_active` が True なら失敗ログを出力して `HandlerReturn(value=False, stop_event=True)` を返す

### 2. ねがいごと_set_side_field（フィールド設置）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_STATUS_HIT |
| subject_spec | `attacker:self`（MoveHandler デフォルト） |
| priority | 100（デフォルト） |

**ロジック:**
1. 使用者 `ctx.attacker` の自陣 `side` を取得
2. `side.activate("ねがいごと", 2)` でフィールドを設置（次のターン終了時に発動するため count=2）
3. `side.get("ねがいごと").heal = mon.max_hp // 2` で回復量を設定

### 3. ねがいごと_tick（カウントダウン）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_TURN_END |
| subject_spec | `source:self` |
| priority | 50 |
| 根拠 | docs/spec/turn.md ON_TURN_END「ねがいごと」Priority 50 |

**ロジック:**
- 内部ヘルパー `_tick_side_field(battle, ctx, value, name="ねがいごと")` を呼び出してカウントを1減らす
- カウントが 0 になると `_deactivate_field` が発火し `ON_FIELD_DEACTIVATE` が発生する

### 4. ねがいごと_heal（HP回復）

| 項目 | 値 |
|------|----|
| イベント | Event.ON_FIELD_DEACTIVATE |
| subject_spec | `source:self` |
| priority | 100（デフォルト） |

**ロジック:**
1. `battle.modify_hp(ctx.source, v=value.heal)` で現在の場にいるポケモンを回復
   - `ctx.source` は `registered_subject=Player` から `battle.get_active(player)` で取得される（現在の場のポケモン）
   - `battle.modify_hp` → `ON_MODIFY_HEAL` が発火するため、かいふくふうじ状態は自動的にブロックされる
   - HP が満タンの場合は `Pokemon.modify_hp` のクランプ処理で変化量が 0 になる

## データ登録（side_field.py）

```python
"ねがいごと": FieldData(
    handlers={
        Event.ON_TURN_END: h.FieldHandler(
            h.ねがいごと_tick,
            subject_spec="source:self",
            priority=50,
        ),
        Event.ON_FIELD_DEACTIVATE: h.FieldHandler(
            h.ねがいごと_heal,
            subject_spec="source:self",
        ),
    },
),
```
