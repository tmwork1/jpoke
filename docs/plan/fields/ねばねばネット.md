# ねばねばネット 実装計画

## 概要

相手陣サイドフィールド。交代で場に出たポケモンのすばやさランクを1段階下げる。

## 技データ（data/move.py）

```python
"ねばねばネット": MoveData(
    type="むし",
    category="status",
    pp=20,
    target="foe_side",
    handlers={
        Event.ON_STATUS_HIT: MoveHandler(
            hs.ねばねばネット_set_side_field,
        ),
    }
)
```

## フィールドデータ（data/field/side_field.py）

```python
"ねばねばネット": FieldData(
    handlers={
        Event.ON_SWITCH_IN: FieldHandler(
            h.ねばねばネット_speed_drop,
            subject_spec="source:self",
            priority=100,  # デフォルト。まきびし・ステルスロックと同じ
        ),
    },
)
```

## ハンドラ実装

### ねばねばネット_set_side_field（handlers/move_status.py）

- `battle.get_side(ctx.defender).activate("ねばねばネット", 1)` で相手陣に設置
- すでに設置済みの場合は `False` が返るため `HandlerReturn(value=False, stop_event=True)` で失敗

### ねばねばネット_speed_drop（handlers/field.py）

```python
def ねばねばネット_speed_drop(battle, ctx, value):
    # ON_CHECK_HAZARD_IMMUNE でブーツ等の免疫チェック
    if battle.query.is_hazard_immune(ctx.source):
        return HandlerReturn(value=value)
    # 地面にいないポケモンは影響を受けない
    if battle.query.is_floating(ctx.source):
        return HandlerReturn(value=value)
    # すばやさを1段階下げる（相手由来: かちき・まけんき が発動する）
    battle.modify_stats(ctx.source, {"spe": -1}, source=battle.foe(ctx.source))
    return HandlerReturn(value=value)
```

- `source=battle.foe(ctx.source)` を渡すことで、かちき・まけんき が相手由来として発動する
- たんじゅん/あまのじゃく は `modify_stats` 内部の `ON_MODIFY_STAT_CHANGES` で処理される
- クリアボディ等のランク低下防止も `modify_stats` 内部で処理される

## 優先度

| イベント | priority | 根拠 |
|----------|----------|------|
| ON_SWITCH_IN（すばやさ低下） | 100（デフォルト） | 既存の まきびし・ステルスロック と同じ |

## テスト計画

| テスト名 | 確認内容 |
|---|---|
| test_ねばねばネット | 交代で出たポケモンのすばやさランクが-1になる |
| test_ねばねばネット_浮いているポケモンには効かない | ひこうタイプは効果を受けない |
| test_ねばねばネット_あつぞこブーツで無効 | あつぞこブーツ持ちは効果を受けない |
| test_ねばねばネット_あまのじゃくは1段階上がる | あまのじゃく持ちはすばやさ+1 |
| test_ねばねばネット_たんじゅんは2段階下がる | たんじゅん持ちはすばやさ-2 |
