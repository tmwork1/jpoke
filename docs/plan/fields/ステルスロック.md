# ステルスロック 実装計画

## 概要

相手陣サイドフィールド。場に出たポケモンにいわタイプ相性依存ダメージを与える。

## 技データ（data/move.py）

```
"ステルスロック": MoveData(
    type="いわ",
    category="status",
    pp=20,
    target="foe_side",
    handlers={
        Event.ON_STATUS_HIT: hs.ステルスロック_set_field,
    }
)
```

## フィールドデータ（data/field/side_field.py）

```
"ステルスロック": FieldData(
    handlers={
        Event.ON_SWITCH_IN: FieldHandler(
            h.ステルスロック_damage,
            subject_spec="source:self",
            priority=100,  # デフォルト。まきびし・どくびしと同じ
        ),
    },
)
```

## ハンドラ実装

### ステルスロック_set_field（handlers/move_status.py）

- `battle.get_side(ctx.defender).activate("ステルスロック", 1)` で設置
- すでに設置済みの場合は `False` が返るため `HandlerReturn(value=False, stop_event=True)` で失敗

### ステルスロック_damage（handlers/field.py）

```python
def ステルスロック_damage(battle, ctx, value):
    # ON_CHECK_HAZARD_IMMUNE でブーツ等の免疫チェック
    if battle.query.is_hazard_immune(ctx.source):
        return HandlerReturn(value=value)
    # らんきりゅう補正を受けないよう ON_CALC_DEF_TYPE_MODIFIER を経由しない
    type_chart = TYPE_MODIFIER.get("いわ", {})
    base = 4096
    for def_type in ctx.source.types:
        base = int(base * type_chart.get(def_type, 1.0))
    battle.modify_hp(ctx.source, r=-base / (4096 * 8))
    # マジックガードは ON_MODIFY_NON_MOVE_DAMAGE で v=0 に設定し無効化
    return HandlerReturn(value=value)
```

## 優先度

| イベント | priority | 根拠 |
|----------|----------|------|
| ON_SWITCH_IN（ダメージ） | 100（デフォルト） | 既存の まきびし・どくびし と同じ |

## 注意点

- `// 4096`（整数除算）を使うと ×1/4・×1/2 の場合にダメージが 0 になるバグが発生する
- `base / (4096 * 8)` の浮動小数点除算を使い、`modify_hp` の `min(-1, raw)` で最低1ダメージを保証する
- `ctx.source.types` を使うことでテラスタル・タイプ追加を正しく反映する
