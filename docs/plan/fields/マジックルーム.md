# マジックルーム 実装計画書

## 概要

- カテゴリ: 全体の場の状態（グローバルフィールド）
- 継続ターン: 5ターン（発動ターンを含む）

## 実装するイベントと担当ファイル

### handlers/field.py

| 関数名 | イベント | 内容 |
|--------|----------|------|
| `マジックルーム_apply` | ON_FIELD_ACTIVATE, ON_SWITCH_IN | `battle.item_manager.add_disabled_reason(ctx.source, "マジックルーム")` でアイテム無効化 |
| `マジックルーム_remove` | ON_FIELD_DEACTIVATE | `battle.item_manager.remove_disabled_reason(ctx.source, "マジックルーム")` でアイテム再有効化 |
| `マジックルーム_tick_global_field` | ON_TURN_END | `tick_global_field` でカウントダウン・終了処理 |

### data/field/global_field.py

## subject_spec の根拠

- `ON_FIELD_ACTIVATE`, `ON_SWITCH_IN`, `ON_FIELD_DEACTIVATE` は ctx.source が対象ポケモンなので `source:self`
- `ON_TURN_END` のカウントダウンは source:self（1P側のみ処理）
- いずれもアクティブポケモンが主体となる非攻撃フェーズなので EventContext

## アイテム無効化の仕組み

`item_manager.add_disabled_reason(mon, "マジックルーム")` を呼ぶと `mon.item` の disabled_reasons に
"マジックルーム" が追加される。`item.enabled` が False になり、ハンドラが無効化される。
解除時は `remove_disabled_reason` で disabled_reasons から削除し、アイテムを再有効化する。
