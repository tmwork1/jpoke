# ちょうはつ 実装計画書

## 概要

ちょうはつ揮発性状態の実装。
対象ポケモンが変化技を使用できなくなる。

## 実装対象

### 1. ちょうはつ_apply_volatile_to_defender（ON_STATUS_HIT）

**ファイル**: `handlers/move_status.py`
**登録**: `data/move.py`（ちょうはつ の ON_STATUS_HIT）

- 相手にちょうはつ状態（count=3）を付与する。
- subject_spec: `"attacker:self"`（MoveHandler のデフォルト）
- priority: 100（デフォルト）

根拠: `apply_volatile_to_defender` 共通関数を使用。すでにちょうはつ状態かどうかの失敗判定は `ON_STATUS_HIT` priority=100 の共通処理で実施済み（`docs/spec/turn.md` 参照）。

### 2. ちょうはつ_try_action（ON_TRY_ACTION）

**ファイル**: `handlers/volatile.py`
**登録**: `data/volatile.py`（ちょうはつ の ON_TRY_ACTION）

- 行動実行時に選択した技のカテゴリが「変化」であれば行動をブロックする。
- `HandlerReturn(value=False, stop_event=True)` を返す。
- subject_spec: `"attacker:self"`
- priority: **100**（`docs/spec/turn.md` の ON_TRY_ACTION「ちょうはつで変化技が出せない」より）

根拠: turn.md の ON_TRY_ACTION priority=100 に「ちょうはつで変化技が出せない」が記載されている。

### 3. ちょうはつ_tick_volatile（ON_TURN_END）

**ファイル**: `handlers/volatile.py`（`tick_volatile` 共通関数使用）
**登録**: `data/volatile.py`（ちょうはつ の ON_TURN_END）

- ターン終了時にカウントを減少させ、0になれば解除する。
- subject_spec: `"source:self"`
- priority: 110（`docs/spec/turn.md` の ON_TURN_END「ちょうはつの終了」より）

## テスト方針

- ちょうはつ状態のポケモンが変化技を使えないことを確認する
- ちょうはつ状態のポケモンが攻撃技を使えることを確認する
- ターン経過でちょうはつが解除されることを確認する
- 技「ちょうはつ」を使うと相手がちょうはつ状態になることを確認する
- すでにちょうはつ状態の相手への技「ちょうはつ」は失敗することを確認する
