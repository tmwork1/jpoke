# ロックオン 実装計画書

仕様書: docs/spec/volatiles/ロックオン.md

---

## 実装概要

ロックオン揮発性状態は「次のターンの自分の技を必中にする」効果を持つ。
ON_MODIFY_ACCURACY で命中率を None にして stop_event=True とすることで必中を実現する。

---

## ハンドラ設計

### handlers/volatile.py

#### `ロックオン_guarantee_hit`

```python
def ロックオン_guarantee_hit(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """ロックオン状態による命中補正"""
    return HandlerReturn(value=None, stop_event=True)
```

- イベント: `ON_MODIFY_ACCURACY`
- subject_spec: `"attacker:self"` （攻撃フェーズ, AttackContext）
- priority: デフォルト（100）
- 根拠: `ON_MODIFY_ACCURACY` は `move_executor.py` から発火される攻撃フェーズのイベント。他の必中実装（ノーガード、ちいさくなる、テレキネシス等）に合わせて `value=None, stop_event=True` を返す。

#### `ロックオン_tick_volatile`

```python
def ロックオン_tick_volatile(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    return tick_volatile(battle, ctx, value, volatile="ロックオン")
```

- イベント: `ON_TURN_END`
- subject_spec: `"source:self"`
- priority: デフォルト（100）
- 根拠: 他の tick 系ハンドラに倣う。

#### `ロックオン_remove_volatile`

```python
def ロックオン_remove_volatile(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    return remove_volatile(battle, ctx, value, volatile="ロックオン")
```

- イベント: `ON_SWITCH_OUT`
- subject_spec: `"source:foe"` （相手が交代したときに自分のロックオンを解除）
- 根拠: バインドの解除と同じパターン。相手が交代したらロックオン状態は意味を失うため解除。

## data/volatile.py の登録

```python
"ロックオン": VolatileData(
    handlers={
        Event.ON_MODIFY_ACCURACY: h.VolatileHandler(
            h.ロックオン_guarantee_hit,
            subject_spec="attacker:self",
        ),
        Event.ON_TURN_END: h.VolatileHandler(
            h.ロックオン_tick_volatile,
            subject_spec="source:self",
        ),
        Event.ON_SWITCH_OUT: h.VolatileHandler(
            h.ロックオン_remove_volatile,
            subject_spec="source:foe",
        ),
    }
),
```
