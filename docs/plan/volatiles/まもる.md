# まもる 実装計画書

対象: Pokémon Champions シングルバトル  
仕様書: `docs/spec/volatiles/まもる.md`

## 概要

まもる volatile は既に実装済み。本計画書は実装の意図・設計根拠を記録する。

## 設計方針

### 揮発性状態のデータ (`data/volatile.py`)

```python
"まもる": VolatileData(
    handlers={
        Event.ON_TRY_MOVE_1: VolatileHandler(
            まもる_protect,
            subject_spec="defender:self",
            priority=100,
        ),
        Event.ON_TURN_END: VolatileHandler(
            まもる_remove_volatile,
            subject_spec="source:self",
        ),
    }
),
```

- `ON_TRY_MOVE_1` priority=100: 相手の技をブロック（`docs/spec/turn.md` 参照）
- `ON_TURN_END`: ターン終了時に解除

### ハンドラ実装 (`handlers/volatile.py`)

#### まもる_protect

```python
def まもる_protect(battle, ctx, value):
    return _run_protect(battle, ctx, value)
```

- `_run_protect` の `protect_non_attack=True` (デフォルト) を使用
- 変化技も防ぐ（キングシールド等と異なる点）
- 接触時の追加効果なし

#### _run_protect の動作

1. `_check_protect_success(battle, ctx, protect_non_attack)` を呼び出す
   - `protect_non_attack=False` かつ `ctx.move.is_attack=False` の場合はスキップ
   - `ctx.move.is_blocked_by_protect=False` の場合はスキップ（フェイント等）
   - `ON_CHECK_PROTECT` イベントを発火（ふかしのこぶし等による貫通チェック）
2. 保護成功 → `HandlerReturn(value=False, stop_event=True)` で技を止める
3. 保護失敗（フェイント等）→ `HandlerReturn(value=value)` で通過

### 技データ (`data/move.py`)

```python
"まもる": MoveData(
    type="ノーマル",
    category="変化",
    pp=10,
    priority=4,
    target="self",
    handlers={
        Event.ON_STATUS_HIT: MoveHandler(
            まもる_apply_volatile_to_attacker,
        ),
    }
),
```

- `ON_STATUS_HIT` で `まもる` volatile を自分に付与
- priority=4: 先制技として動作

### 変化技ハンドラ (`handlers/move_status.py`)

```python
def まもる_apply_volatile_to_attacker(battle, ctx, value):
    return apply_volatile_to_attacker(battle, ctx, value, volatile="まもる")
```

みきりも同様に `まもる` volatile を付与する:
```python
def みきり_apply_volatile_to_attacker(battle, ctx, value):
    return apply_volatile_to_attacker(battle, ctx, value, volatile="まもる")
```

### priority の根拠

| イベント | Priority | 根拠 |
|---------|---------|------|
| ON_TRY_MOVE_1 | 100 | `docs/spec/turn.md`: まもる・キングシールド等による無効化 priority=100 |
| ON_TURN_END | (default=100) | ターン終了時解除、優先度未規定 |

## 連続使用による失敗率上昇（実装済み）

`docs/spec/turn.md` に「まもる系: 連続使用により失敗 priority=30」と記載されているが、
その後の実装（`まもる系_連続使用失敗チェック`、`handlers/move_status.py`、技側の
`Event.ON_TRY_MOVE_2` に登録）で対応済み。ただし本来の確率モデル（1/3ずつ低下、下限1/729）
ではなく、「直前ターンに `protect` フラグを持つ技を成功させていた場合、今ターンは必ず失敗する」
という簡略化モデルを採用している。詳細な設計・priority根拠は `docs/plan/moves/まもる.md` を参照。

厳密な確率モデルを実装する場合は:
- `Pokemon` に `protect_streak: int = 0` フィールドを追加
- `ON_TRY_MOVE_1` priority=30 のハンドラ（まもる技に登録）で `1/3^n` 確率チェック
- 成功時: `protect_streak += 1`、失敗時: `protect_streak = 0`
- 交代時（`reset_on_switch_out`）: `protect_streak = 0`

上記は本レビュー時点でも引き続きスコープ外（既知の限界）とする。

## テスト方針

- まもる volatile が attack を防ぐことを確認
- まもる volatile が変化技も防ぐことを確認
- まもる volatile がターン終了で解除されることを確認
- 相手自身が対象の技は無効化しないことを確認（target="self" の技はまもる対象外）
- ふかしのこぶし持ちが貫通することの確認（ON_CHECK_PROTECT で False が返る）
- みきりもまもる volatile を付与することの確認
