# トーチカ 実装計画書

## 概要

トーチカ揮発性状態の実装。
使用したターンに受ける攻撃技・変化技を無効化し、直接攻撃をしてきた相手をどく状態にする。
まもると同様に変化技も防ぐ点でキングシールド・かえんのまもり等と異なる。
ターン終了時に解除される。

## 実装対象

### 1. トーチカ_protect（Event.ON_TRY_MOVE_1）

**ファイル**: `handlers/volatile.py`  
**登録**: `data/volatile.py`

- `_run_protect(battle, ctx, value, ailment_on_contact="どく")` を呼び出す
  - `protect_non_attack` はデフォルト（`True`）のため変化技も防ぐ（まもると同様）
  - `ailment_on_contact="どく"` により直接攻撃者にどくを付与する
  - `battle.query.is_contact(ctx)` が True の場合のみどくを付与（ぼうごパット持ちは False になる）
  - `ON_CHECK_PROTECT` イベントで守りが無視される場合（ふかしのこぶし等）は無効化もどくも行わない
- subject_spec: `"defender:self"` （AttackContext）
- priority: 100
  - 根拠: `docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` より「まもる・キングシールド等による無効化」は Priority 100

### 2. トーチカ_remove_volatile（Event.ON_TURN_END）

**ファイル**: `handlers/volatile.py`  
**登録**: `data/volatile.py`

- ターン終了時にトーチカ状態を解除する
- subject_spec: `"source:self"` （EventContext）
- priority: 未指定（デフォルト）
  - 根拠: turn.md に ON_TURN_END でのまもる系解除の明示的な priority 記載なし。他のまもる系（まもる・キングシールド・かえんのまもり等）と同様に priority 指定なし

### 3. 技データ定義・apply_volatile（data/move.py）

**ファイル**: `handlers/move_status.py` + `data/move.py`

- 技使用時（自分への変化技として成功）、自分にトーチカ状態（count=1）を付与する
- subject_spec: 未指定（デフォルト `"attacker:self"`）

## テスト方針

- 接触技を受けると攻撃者がどく状態になることを確認する
- 非接触攻撃技を受けてもどく状態にならないことを確認する
- 攻撃技を防ぐことを確認する（まもる系共通テストで検証済み）
- 変化技も防ぐことを確認する（まもる系共通テストで検証済み）
- ターン終了でトーチカ状態が解除されることを確認する（まもる系共通テストで検証済み）

## 注意点

- `data/volatile.py` の登録順は五十音順に従う
- `protect_non_attack` のデフォルト（`True`）を使用するため引数指定不要
- 交代による解除は `switch_manager._switch_out` の `remove_all_volatiles` で自動的に処理される
- まもる系共通の攻撃技無効化・変化技への反応・ターン終了解除はパラメトリックテストで一括検証
