# いちゃもん 実装計画書

作成日: 2026-06-19
参照仕様書: docs/spec/volatiles/いちゃもん.md

## 概要

いちゃもん揮発性状態の実装。直前に使用した技をコマンド選択時に選べなくする。

## 実装対象

### 1. いちゃもん_modify_command_options（Event.ON_MODIFY_COMMAND_OPTIONS）

**ファイル**: `handlers/volatile.py`
**登録**: `data/volatile.py`

- いちゃもん状態のポケモンのコマンド選択肢から、揮発状態に記録された `move_name` と一致する技コマンドを除外する。
- 交代コマンド（`cmd.is_move_family == False`）はそのまま通す。
- 選択肢が空になった場合は `CommandManager` が自動的に `Command.STRUGGLE` を追加する。
- `subject_spec`: `"source:self"` （EventContext）
- `priority`: 100（デフォルト）

**根拠**: `ON_MODIFY_COMMAND_OPTIONS` は docs/spec/turn.md に優先度の記載がないため、
同種ハンドラ（かなしばり_modify_command_options priority=100）を参照して決定。

## 実装状況

- [x] `いちゃもん_modify_command_options` ハンドラ実装済み (`handlers/volatile.py`)
- [x] `data/volatile.py` への登録済み（priority=100, subject_spec="source:self"）

## 消滅条件の処理

- 交代によるいちゃもん解除は `switch_manager._switch_out` の `remove_all_volatiles` で自動処理される。
- ターン経過では解除しない（`ON_TURN_END` ハンドラ不要）。
- バトンタッチによる引き継ぎ禁止はバトンタッチ実装時に対応。

## テスト方針

- いちゃもん状態で禁止技が選択肢から除外されることを確認する。
- いちゃもん状態でも別の技は選択できることを確認する。
- いちゃもん状態で技が1つしかないときわるあがきになることを確認する。
- いちゃもん状態がターン経過で消えないことを確認する。
- 交代するといちゃもん状態が解除されることを確認する。
