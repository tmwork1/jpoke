# キングシールド 実装計画書

## 概要

キングシールド揮発性状態の実装。
使用したターンに受ける攻撃技を無効化し、直接攻撃をしてきた相手の攻撃ランクを1段階下げる。
変化技は防げない。ターン終了時に解除される。

## 実装対象

### 1. キングシールド_protect（Event.ON_TRY_MOVE_1）

**ファイル**: `handlers/volatile.py`  
**登録**: `data/volatile.py`

- `_run_protect(battle, ctx, value, stats_change_on_contact={"A": -1}, protect_non_attack=False)` を呼び出す
  - `protect_non_attack=False` により変化技を防がない（仕様: 変化技は防げない）
  - `stats_change_on_contact={"A": -1}` により直接攻撃者の攻撃ランクを1段階下げる
  - `battle.query.is_contact(ctx)` が True の場合のみランク変化を付与（ぼうごパット持ちは False）
  - `ON_CHECK_PROTECT` イベントで守りが無視される場合（ふかしのこぶし等）は無効化も行わない
- subject_spec: `"defender:self"` （AttackContext）
- priority: 100
  - 根拠: `docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` より「まもる・キングシールド等による無効化」は Priority 100

### 2. キングシールド_remove_volatile（Event.ON_TURN_END）

**ファイル**: `handlers/volatile.py`  
**登録**: `data/volatile.py`

- ターン終了時にキングシールド状態を解除する
- subject_spec: `"source:self"` （EventContext）
- priority: 未指定（デフォルト）
  - 根拠: turn.md に ON_TURN_END でのまもる系解除の明示的な priority 記載なし。他のまもる系（まもる・かえんのまもり等）と同様に priority 指定なし

### 3. キングシールド技の技使用ハンドラ（Event.ON_STATUS_HIT）

**ファイル**: `handlers/move_status.py`  
**登録**: `data/move.py`

- 技使用時（自分への変化技として成功）、自分にキングシールド状態（count=1）を付与する
- subject_spec: 未指定（デフォルト `"attacker:self"`）
  - 根拠: MoveHandler のデフォルト subject_spec = `"attacker:self"`

## テスト方針

- 技を使うとキングシールド状態が付与されることを確認する
- 接触技を受けると攻撃者の攻撃ランクが下がることを確認する（実装済み）
- 非接触攻撃技を受けても攻撃ランクが下がらないことを確認する（実装済み）
- **変化技は防がれないことを確認する**（まもる系共通テストのパラメータを修正済み）
- ターン終了でキングシールド状態が解除されることを確認する（まもる系共通テストで検証済み）
- 攻撃技を防ぐことを確認する（まもる系共通テストで検証済み）

## 注意点

- `data/volatile.py` の登録順は五十音順に従う
- 交代による解除は `switch_manager._switch_out` の `remove_all_volatiles` で自動的に処理される
- まもる系共通の攻撃技無効化・ターン終了解除はパラメトリックテストで一括検証
- **変化技への反応はかえんのまもり/スレッドトラップと同様に `False`（防がない）が正しい**
