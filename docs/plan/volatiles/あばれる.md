# あばれる 実装計画書

## 概要

あばれる揮発性状態の実装。
2〜3ターン間、技を強制使用し続け、終了後にこんらん状態になる。

## 実装対象

### 1. あばれる_apply（ON_HIT）

**ファイル**: `handlers/move_attack.py`
**登録**: `data/move.py`（あばれる、げきりん、はなびらのまい、だいふんげきの ON_HIT）

- 初回攻撃命中時に揮発性状態を付与する。
- ターン数はランダムで 2〜3 ターン（`battle.random.randint(2, 3)`）。
- あばれる状態がすでに付与されている場合は何もしない。
- subject_spec: `"attacker:self"`
- priority: 100（デフォルト）

根拠: `ON_HIT` は攻撃命中後に発火する。強制技適用中は ON_HIT が発火するので、
volatile.data.forced チェックで二重付与を防ぐ。
ただし `volatile_manager.apply` 内で既存チェックが行われるため、二重チェック不要。

### 2. あばれる_tick（ON_DAMAGE_HIT）

**ファイル**: `handlers/volatile.py`（実装済み）
**登録**: `data/volatile.py`（実装済み、priority=180）

- ダメージ適用後にターンカウントを減少させる。
- カウントが 0 になったらこんらん状態を付与する（`apply_confusion`）。
- subject_spec: `"attacker:self"`
- priority: 180（`docs/spec/turn.md` の `Event.ON_DAMAGE_HIT` より「あばれる状態終了でこんらん」）

### 3. あばれる強制コマンド（ON_MODIFY_COMMAND_OPTIONS）

**ファイル**: `handlers/volatile.py`（`force_command` 共通関数使用済み）
**登録**: `data/volatile.py`（実装済み、subject_spec="source:self"）

- あばれる状態中は常に `[Command.FORCED]` を返す。
