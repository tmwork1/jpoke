# メロメロ 実装計画書

## 概要

メロメロ（揮発性状態）の実装計画。
性別が異性のときに付与でき、ON_TRY_ACTION（priority=130）で 50% の確率で行動不能にする。

## 実装対象

### 1. src/jpoke/handlers/move_status.py

`メロメロ_check_gender` を追加する。
関数の配置は既存の `メロメロ_apply_volatile_to_defender` の直前（五十音順）。

```python
def メロメロ_check_gender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """メロメロの失敗条件: 相手が同性または性別不明の場合は失敗させる。"""
    attacker = ctx.attacker
    defender = ctx.defender
    # 性別不明（""）または同性の場合は失敗
    if (defender.gender == ""
            or attacker.gender == ""
            or attacker.gender == defender.gender):
        battle.add_event_log(
            attacker, LogCode.MOVE_FAILED,
            payload={"reason": "メロメロ_性別不一致"}
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### 2. src/jpoke/data/move.py

`"メロメロ"` の MoveData に `ON_TRY_MOVE_2` ハンドラを追加する（priority=120）。

```python
"メロメロ": MoveData(
```

## Priority の根拠

- `ON_TRY_MOVE_2` priority=120: `docs/spec/turn.md` の ON_TRY_MOVE_2 テーブルに「120 | メロメロ: 性別同じ・不明」と明記
- `ON_TRY_ACTION` priority=130: `docs/spec/turn.md` の ON_TRY_ACTION テーブルに「130 | メロメロの判定」と明記
