# ふういん 実装計画書

作成日: 2026-06-21
参照仕様書: docs/spec/volatiles/ふういん.md

## 概要

ふういん揮発性状態の実装。ふういん状態のポケモンが覚えている技と共通する技を相手が使えなくする。
揮発状態ハンドラ（data/volatile.py）で実装する。
技「ふういん」の技データは別途 data/move.py に登録する（揮発状態付与）が、揮発状態自体のロジックはここで扱う。

## 実装対象

### 1. ふういん_try_action（Event.ON_TRY_ACTION）

**ファイル**: `handlers/volatile.py`
**登録**: `data/volatile.py`

- ふういん状態のポケモン（`defender`）が覚えている技と同名の技を attacker が使おうとした場合、行動をブロックする。
- `ctx.defender.has_move(ctx.move.name)` で照合する。
- `subject_spec`: `"defender:self"`（AttackContext）
  - ふういん状態のポケモンが defender（技を受ける側）のときに発動する。
  - 攻撃者（attacker）が使おうとしている技と defender の技リストを照合することで、共通技の使用禁止を実現する。
- `priority`: 100（`docs/spec/turn.md` の `ON_CHECK_ACTION` / プロジェクト内 `ON_TRY_ACTION` 優先度100 に基づく）

## 実装状況

- [x] `ふういん_try_action` ハンドラ実装済み (`handlers/volatile.py`)
- [x] `data/volatile.py` への登録済み

## 消滅条件の処理

- ひんしによるふういん解除は `status_manager` でのひんし処理後、交代が発生することで `remove_all_volatiles` が自動処理される。
- 交代によるふういん解除は `switch_manager._switch_out` の `remove_all_volatiles` で自動処理される。
- バトンタッチによる引き継ぎ禁止は交代時の全揮発状態解除で自動的に実現される。

## テスト方針

- ふういん: 共通技は使えない
- ふういん: 非共通技は使える
- ふういん: 交代で解除される（ふういん使用者が交代した後は相手は共通技を使える）
