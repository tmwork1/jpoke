# アクアリング 実装計画書

## 概要

アクアリング揮発性状態の実装。ターン終了時に最大HPの1/16（最小1）を回復する。

## 実装対象

### 1. アクアリング_self_heal（Event.ON_TURN_END）

**ファイル**: `handlers/volatile.py`  
**登録**: `data/volatile.py`

- アクアリング状態のポケモンのターン終了時に最大HPの1/16を回復する
- 回復量計算: `max(1, mon.max_hp // 16)`（切り捨て、最小1）
- `modify_hp(source, r=1/16, source=source)` で呼び出す
  - v > 0 で `ON_MODIFY_HEAL` が発火され、かいふくふうじによる回復ブロックが正しく動作する
- subject_spec: `"source:self"` （EventContext）
- priority: 70
  - 根拠: `docs/spec/turn.md` の `Event.ON_TURN_END` より「アクアリング」は Priority 70

## 実装状況

- [x] アクアリング_self_heal ハンドラ実装済み (`handlers/volatile.py`)
- [x] `data/volatile.py` への登録済み（priority=70, subject_spec="source:self"）

## テスト方針

- アクアリング状態で `end_turn` を呼び、HPが `1 + mon.max_hp // 16` になることを確認する
- かいふくふうじ状態では回復しないことを確認する
- 交代でアクアリング状態が解除されることを確認する

## 未実装事項（将来対応）

- おおきなねっこによる回復量1.3倍: `ON_CALC_DRAIN` ではなく別のイベントが必要
- バトンタッチによる引き継ぎ: バトンタッチ実装時に対応

## 注意点

- `modify_hp` に `r=1/16` を渡すと `v = int(max_hp * r)` となる
  - max_hp < 16 の場合に v=0 になるが、現実のポケモンではmax_hpが十分大きいため問題にならない
  - かいふくふうじの回復ブロックは `ON_MODIFY_HEAL` イベント経由で自動的に機能する
- 交代による解除は `switch_manager._switch_out` の `remove_all_volatiles` で自動的に処理される
