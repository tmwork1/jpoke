# やどりぎのタネ 実装計画書

## 概要

やどりぎのタネ揮発性状態の実装。
- ターン終了時に最大HPの1/8を吸収（ダメージ）し、使用者側を回復する
- くさタイプには技が失敗する

## 実装対象

### 1. やどりぎのタネ_drain_hp（Event.ON_TURN_END）

**ファイル**: `handlers/volatile.py`
**登録**: `data/volatile.py`

- やどりぎのタネ状態のポケモンのターン終了時に最大HPの1/8を吸収する
- ダメージ計算: `r = -1/8`（`int(max_hp * r)` = 切り捨て）
- `battle.modify_hp(from_mon, r=-1/8, reason="drain")` でダメージを与える
  - `v < 0` のため `ON_MODIFY_NON_MOVE_DAMAGE` が発火 → マジックガードが機能する
- 戻り値（damage）は負の値なので、`battle.modify_hp(to_mon, -damage, reason="drain")` で回復
  - `v > 0` のため `ON_MODIFY_HEAL` が発火 → ヘドロえきが機能する
- ダメージが0（マジックガード等）の場合、回復は発生しない（`if damage:` ガード）
- `HPChangeReason = "drain"` を使用する（ヘドロえきのダメージ変換に必要）
- subject_spec: `"source:self"` （EventContext）
- priority: 80
  - 根拠: `docs/spec/turn.md` の `Event.ON_TURN_END` より「やどりぎのタネ」は Priority 80

### 2. やどりぎのタネ_can_apply（Event.ON_BEFORE_APPLY_MOVE）

**ファイル**: `handlers/move_status.py`
**登録**: `data/move.py`

- くさタイプのポケモンにはやどりぎのタネが失敗する
- `ctx.defender.has_type("くさ")` のとき `False` を返して失敗させる
- subject_spec: `"attacker:self"` （AttackContext）
- priority: 130
  - 根拠: `docs/spec/turn.md` の技失敗チェック表「くさタイプ: やどりぎ無効」= Priority 130

### 3. やどりぎのタネ_apply_volatile_to_defender（Event.ON_STATUS_HIT）

**ファイル**: `handlers/move_status.py`（既存）
**登録**: `data/move.py`（既存）

- 相手にやどりぎのタネ状態を付与する（既存実装）
- `apply_volatile_to_defender(battle, ctx, value, volatile="やどりぎのタネ")` を使用
- subject_spec: `"attacker:self"` （AttackContext）
- priority: 100（デフォルト）

## テスト方針

- やどりぎのタネ状態で `end_turn` を呼び、HPが `max_hp - max_hp // 8` になることを確認する
- 回復先が満タンの場合でもダメージは発生することを確認する
- くさタイプに対してやどりぎのタネが失敗することを確認する
- マジックガード持ちはダメージを受けず、相手も回復しないことを確認する
- ヘドロえき持ちがやどりぎ状態のとき、回復先がダメージを受けることを確認する

## 注意点

- `modify_hp` に `r=-1/8` を渡すと `v = int(max_hp * -1/8)` となる（切り捨て）
- `damage` は負の値（HP変化量）なので、回復側には `-damage`（正値）を渡す
- くさタイプ無効チェックは `ON_BEFORE_APPLY_MOVE` で行う（`ON_STATUS_HIT` ではない）
- 交代による解除は `switch_manager._switch_out` の `remove_all_volatiles` で自動的に処理される
