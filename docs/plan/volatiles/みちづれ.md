# みちづれ 実装計画書

## 概要

みちづれ揮発性状態の実装。
攻撃技でひんしにされたとき、その攻撃者もひんし状態にする。
自身が次に行動したとき（または行動しようとしたとき）に状態が解除される。

## 実装対象

### 1. みちづれ_faint（Event.ON_MOVE_KO）

**ファイル**: `handlers/volatile.py`
**登録**: `data/volatile.py`

- みちづれ状態のポケモン（defender）が攻撃技でひんしになったとき、攻撃者（attacker）もひんしにする
- `battle.modify_hp(mon, v=-mon.hp, reason="perish")` で HP を 0 にする
  - `reason="perish"` によってマジックガードを回避する
- ログ: `LogCode.VOLATILE_DISPLAY` でみちづれ発動を記録
- subject_spec: `"defender:self"` （AttackContext — みちづれ持ちが defender）
- priority: 30
  - 根拠: `docs/spec/turn.md` の `Event.ON_DAMAGE` 内「みちづれによる攻撃側ひんし」は Priority 30

### 2. みちづれ_remove（Event.ON_TRY_ACTION）

**ファイル**: `handlers/volatile.py`
**登録**: `data/volatile.py`

- みちづれ状態のポケモンが行動しようとするとき（次の自身の行動順）に状態を解除する
- おんねんの解除ハンドラーと同じパターン
- subject_spec: `"attacker:self"` （AttackContext — みちづれ持ちが attacker）
- priority: 10
  - 根拠: おんねんの解除（Priority 10）と同様。ひるみ（Priority 40）より前、こんらん（Priority 110）より前

## 実装コード

### handlers/volatile.py（みちづれ_removeを追加）

```python
def みちづれ_remove(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """みちづれ状態の解除（自身の行動時に解除）。"""
    return remove_volatile(battle, ctx, value, volatile="みちづれ")
```

### data/volatile.py（ON_TRY_ACTIONハンドラを追加）

```python
"みちづれ": VolatileData(
    handlers={
        Event.ON_MOVE_KO: h.VolatileHandler(
            h.みちづれ_faint,
            subject_spec="defender:self",
            priority=30
        ),
        Event.ON_TRY_ACTION: h.VolatileHandler(
            h.みちづれ_remove,
            subject_spec="attacker:self",
            priority=10
        ),
    }
),
```

## テスト方針

- 基本動作: 攻撃技でひんしになると相手もひんしになる
- 非発動: 倒しきれなければ発動しない
- 解除: 次のターンに行動するとみちづれ状態が解除される（解除後は攻撃でひんしになっても発動しない）

## 注意点

- `reason="perish"` によってマジックガード・きあいのタスキ・がんじょうを無視してひんしにする
- ON_TRY_ACTION の priority 10 はおんねんと同パターン（解除処理は最初に行う）
- 交代による解除は `switch_manager._switch_out` の `remove_all_volatiles` で自動処理される
