# タールショット 実装計画書

## 概要

タールショット揮発性状態の実装。
ほのおタイプに対するタイプ相性補正を2倍にし、ほのおが弱点に追加される。

## 実装対象

### 1. タールショット_boost_fire（Event.ON_CALC_DEF_TYPE_MODIFIER）

**ファイル**: `handlers/volatile.py`
**登録**: `data/volatile.py`

- タールショット状態のポケモンに対してほのお技のタイプ相性補正を2倍にする
- subject_spec: `"defender:self"` （AttackContext — `ctx.move.type` を使用）
- priority: デフォルト（100）
- 根拠: `docs/spec/turn.md` には ON_CALC_DEF_TYPE_MODIFIER は掲載されていない。
  既存の同種ハンドラ（きもったま: priority=100）を参照してデフォルト100とする。

### 2. タールショット技（ON_STATUS_HIT）

**ファイル**: `handlers/move_status.py`
**登録**: `data/move.py`

- 命中時に相手をタールショット状態にし、すばやさを1段階下げる
- subject_spec: 技側のデフォルト（attacker/defender）

## 実装状況

- [x] タールショット_boost_fire ハンドラ実装済み (`handlers/volatile.py`)
- [x] `data/volatile.py` への登録済み（subject_spec="defender:self"）
- [x] タールショット技実装済み (`handlers/move_status.py`, `data/move.py`)

## テスト方針

- ほのお技でタールショット状態のポケモンを攻撃すると def_type_modifier が2倍になる
- ほのお以外の技では変化しない
- 元々ほのおが半減のポケモン（例: ほのおタイプ）がタールショット状態で等倍になる

## 注意点

- ON_CALC_DEF_TYPE_MODIFIER の context は AttackContext であるため、
  ハンドラの型アノテーションを `AttackContext` とする（`EventContext` は誤り）。
- 交代による解除は `switch_manager._switch_out` の `remove_all_volatiles` で自動処理される。
- テラスタル時の予防はポケモンチャンピオンズ仕様の範囲外のため未実装。
