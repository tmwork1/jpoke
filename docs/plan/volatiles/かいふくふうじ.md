# かいふくふうじ 実装計画書

## 概要

かいふくふうじ揮発性状態の実装。
付与されたポケモンの HP 回復を無効化し、規定ターン数後に解除される。

## 実装対象

### 1. かいふくふうじ_block_heal（Event.ON_MODIFY_HEAL）

**ファイル**: `handlers/volatile.py`  
**登録**: `data/volatile.py`

- かいふくふうじ状態のポケモンへの HP 回復を 0 にする
- 以下の `hp_change_reason` は通過させる（無効化しない）
  - `"pain_split"`: いたみわけ
  - `"bench_heal"`: さいせいりょく（交代時回復）
- subject_spec: `"target:self"` （EventContext）
- priority: 未指定（デフォルト）
  - 根拠: turn.md に ON_MODIFY_HEAL の記載なし。既存の同種ハンドラ（ヘドロえき等）と同様に priority 指定なし

### 2. かいふくふうじ_tick_volatile（Event.ON_TURN_END）

**ファイル**: `handlers/volatile.py`  
**登録**: `data/volatile.py`

- ターン終了時にカウントを 1 減算し、0 になったらかいふくふうじ状態を解除する
- subject_spec: `"source:self"` （EventContext）
- priority: 110
  - 根拠: `docs/spec/turn.md` の `Event.ON_TURN_END` より「かいふくふうじの終了」は Priority 110

### 3. かいふくふうじ_modify_command_options（Event.ON_MODIFY_COMMAND_OPTIONS）

**ファイル**: `handlers/volatile.py`  
**登録**: `data/volatile.py`

- 第六世代以降、`flags` に `heal` を含む技（じこさいせい等の回復技、ドレインキッス・ギガドレイン
  等のHP吸収技、すなあつめ等）をコマンド選択肢から除外する
- `category == "status"` ではなく `has_flag("heal")` で判定する（ドレインキッス等の攻撃技も
  対象に含めるため）
- subject_spec: `"source:self"` （EventContext）
- priority: 未指定（デフォルト100）
  - 根拠: `docs/spec/turn.md` に `ON_MODIFY_COMMAND_OPTIONS` の記載はないため、同種の
    ハンドラ（かなしばり_modify_command_options）と同じ形にした

### 4. かいふくふうじ_try_action（Event.ON_TRY_ACTION）

**ファイル**: `handlers/volatile.py`  
**登録**: `data/volatile.py`

- 行動前にかいふくふうじ状態にされた場合（コマンド選択時点では制限がかからなかった場合）の
  フォールバックとして、`heal` フラグを持つ技の実行をブロックする
- subject_spec: `"attacker:self"` （EventContext）
- priority: 100
  - 根拠: `docs/spec/turn.md` に `ON_TRY_ACTION` の記載はないため、同種の行動制限ハンドラ
    （ちょうはつ・かなしばり・ふういん、いずれも priority=100）を参考に決定した

## テスト方針

- かいふくふうじ状態で HP 回復を試み、HP が増えないことを確認する
- いたみわけはかいふくふうじで防がれないことを確認する
- さいせいりょくはかいふくふうじで防がれないことを確認する
- カウントが正しく進行し、ターン経過で解除されることを確認する
- `heal` フラグを持つ技がコマンド選択の時点で除外されることを確認する
  （fuzz_log/seed744で発見: この除外が漏れていたため、コマンド候補に残ったまま
  `ON_TRY_ACTION` で毎ターン弾かれ、技名・PP消費ログを伴わない「動けない」ログになっていた）
- `heal` フラグを持たない技は引き続き選択可能なことを確認する
- 行動前にかいふくふうじ状態にされた場合（コマンド選択レベルの除外を経由しない場合）、
  `ON_TRY_ACTION` により実行が失敗しダメージも発生しないことを確認する
