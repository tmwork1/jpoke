# かいふくふうじ 実装計画書

## 概要

かいふくふうじ揮発性状態の実装。
付与されたポケモンの HP 回復を無効化し、規定ターン数後に解除される。

## 実装対象

### 1. かいふくふうじ_block_heal（Event.ON_MODIFY_HEAL）

**ファイル**: `handlers/volatile.py`  
**登録**: `data/volatile.py`

- かいふくふうじ状態のポケモンへの HP 回復を 0 にする
- 以下の `hp_change_reason` は通過させる（無効化しない）
  - `"pain_split"`: いたみわけ
  - `"bench_heal"`: さいせいりょく（交代時回復）
- subject_spec: `"target:self"` （EventContext）
- priority: 未指定（デフォルト）
  - 根拠: turn.md に ON_MODIFY_HEAL の記載なし。既存の同種ハンドラ（ヘドロえき等）と同様に priority 指定なし

### 2. かいふくふうじ_tick_volatile（Event.ON_TURN_END）

**ファイル**: `handlers/volatile.py`  
**登録**: `data/volatile.py`

- ターン終了時にカウントを 1 減算し、0 になったらかいふくふうじ状態を解除する
- subject_spec: `"source:self"` （EventContext）
- priority: 110
  - 根拠: `docs/spec/turn.md` の `Event.ON_TURN_END` より「かいふくふうじの終了」は Priority 110

## テスト方針

- かいふくふうじ状態で HP 回復を試み、HP が増えないことを確認する
- いたみわけはかいふくふうじで防がれないことを確認する
- さいせいりょくはかいふくふうじで防がれないことを確認する
- カウントが正しく進行し、ターン経過で解除されることを確認する
