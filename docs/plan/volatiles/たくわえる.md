# たくわえる 実装計画書

作成日: 2026-06-20
参照仕様書: docs/spec/volatiles/たくわえる.md

## 実装概要

たくわえる揮発性状態と、関連する技（たくわえる・はきだす・のみこむ）のハンドラを実装する。
揮発性状態はカウント管理のみで、実際の効果は各技のハンドラ（`handlers/move_status.py`）側で処理する。

## 実装済み内容

### data/volatile.py

```
"たくわえる": VolatileData(handlers={})
```

ハンドラなし。カウントの保持のみ。

### handlers/move_status.py + data/move.py

| ハンドラ | イベント | subject_spec | priority | 説明 |
|---|---|---|---|---|
| `たくわえる_check_can_use` | ON_TRY_MOVE_1 | attacker:self | 30 | count>=3なら失敗 |
| `たくわえる_apply` | ON_STATUS_HIT | attacker:self | 100 | B/D+1、カウントインクリメント |
| `はきだす_check_can_use` | ON_TRY_MOVE_1 | attacker:self | 30 | count==0なら失敗 |
| `はきだす_set_power` | ON_BEGIN_MOVE | attacker:self | 100 | count×100で威力設定 |
| `はきだす_apply_after` | ON_HIT | attacker:self | 100 | B/D-count、volatile削除 |
| `のみこむ_check_can_use` | ON_TRY_MOVE_1 | attacker:self | 30 | count==0なら失敗 |
| `のみこむ_apply` | ON_STATUS_HIT | attacker:self | 100 | count依存HP回復、B/D-count、volatile削除 |

### subject_spec の根拠

攻撃フェーズ（AttackContext）のハンドラなので、自分に作用するものはすべて `attacker:self`。

### priority の根拠

- ON_TRY_MOVE_1 priority=30: 既存の同種ハンドラ（のみこむ等）を参照
- ON_STATUS_HIT / ON_BEGIN_MOVE / ON_HIT priority=100: デフォルト値

## 動作確認事項

- たくわえる: 初回使用でcount=1付与、B/D各+1
- たくわえる: 2回目でcount=2、3回目でcount=3
- たくわえる: count=3状態で使用すると失敗
- はきだす: count=0なら失敗
- はきだす: count=1で威力100、count=3で威力300
- はきだす: 命中後にたくわえる解除・B/D-count
- のみこむ: count=0なら失敗
- のみこむ: count=1で1/4回復、count=2で1/2回復、count=3で全回復
- のみこむ: 使用後にたくわえる解除・B/D-count
- のみこむ: ランク低下はクリアボディに阻まれない（source=self）

## 実装ファイル

- `src/jpoke/handlers/move_status.py`: 各技のハンドラ実装（実装済み）
- `src/jpoke/data/move.py`: 各技へのハンドラ登録（実装済み）
- `src/jpoke/data/volatile.py`: VolatileData登録（実装済み、handlers={}）
