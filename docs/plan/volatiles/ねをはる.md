# ねをはる 実装計画書

## 概要

ねをはる揮発性状態の実装。
- ターン終了時に最大HPの1/16を回復する
- 交代禁止（ゴーストタイプは例外）
- 地面にいる扱い（浮遊無効）

## 実装対象

### 1. ねをはる_self_heal（Event.ON_TURN_END）

**ファイル**: `handlers/volatile.py`
**登録**: `data/volatile.py`

- ねをはる状態のポケモンのターン終了時に最大HPの1/16を回復する
- 回復量計算: `r=1/16`（切り捨て）
- `battle.modify_hp(ctx.source, r=1/16)` で呼び出す
  - v > 0 で `ON_MODIFY_HEAL` が発火され、かいふくふうじによる回復ブロックが正しく動作する
- `HPChangeReason` は `""` （通常の回復）。`"drain"` は使わない。
- subject_spec: `"source:self"` （EventContext）
- priority: 70
  - 根拠: `docs/spec/turn.md` の `Event.ON_TURN_END` より「ねをはる」は Priority 70

### 2. check_trapped_not_ghost（Event.ON_CHECK_TRAPPED）

**ファイル**: `handlers/volatile.py` の共通関数を使用
**登録**: `data/volatile.py`

- ゴーストタイプでなければ交代を禁止する
- subject_spec: `"source:self"` （EventContext）
- priority: 100（デフォルト）

### 3. ねをはる_check_floating（Event.ON_CHECK_FLOATING）

**ファイル**: `handlers/volatile.py`
**登録**: `data/volatile.py`

- ねをはる状態では浮遊しない（地面に根を張っているため）
- `HandlerReturn(value=False, stop_event=True)` を返す
- subject_spec: `"source:self"` （EventContext）
- priority: 100（デフォルト）

## テスト方針

- ねをはる状態で `end_turn` を呼び、HPが `1 + mon.max_hp // 16` になることを確認する
- ねをはる状態では交代できないことを確認する
- ゴーストタイプはねをはる状態でも交代できることを確認する（将来対応）
- ねをはる状態では is_floating が False になることを確認する
- かいふくふうじ状態では回復しないことを確認する

## 未実装事項（将来対応）

- おおきなねっこによる回復量1.3倍
- バトンタッチによる引き継ぎ: バトンタッチ実装時に対応

## 注意点

- `modify_hp` に `r=1/16` を渡すと `v = int(max_hp * r)` となる
  - かいふくふうじの回復ブロックは `ON_MODIFY_HEAL` イベント経由で自動的に機能する
- 交代による解除は `switch_manager._switch_out` の `remove_all_volatiles` で自動的に処理される
- HPChangeReason は `""` を使う（drain はやどりぎのタネ等の吸収技専用）
