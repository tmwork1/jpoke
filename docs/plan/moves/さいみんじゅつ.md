# 実装計画: さいみんじゅつ

## 仕様サマリ

- タイプ: エスパー / 分類: 変化 / PP: 20 / 命中率: 60
- 効果: 相手を 2〜4 ターン（実質 1〜3 ターン）のねむり状態にする
- まもる: ○ / マジックコート: ○ / 直接攻撃: ×
- 対象: 1体選択

## 現状の実装状況

`data/move.py` にエントリは存在するが `handlers` キーが未設定（ねむり付与が未実装）。

```python
"さいみんじゅつ": MoveData(
    type="エスパー",
    category="変化",
    pp=20,
    accuracy=60,
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | ねむり状態を相手に付与する |

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の同種ハンドラ（`あくまのキッス_apply_ailment_to_defender`、`ねむりごな_apply_ailment_to_defender`）がすべて priority=100 デフォルトで実装されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: 五十音順で `こわいかお_*` と `じこさいせい_*` の間（こ → さ の境目）

```python
def さいみんじゅつ_apply_ailment_to_defender(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """さいみんじゅつの効果: 相手をねむり状態にする。"""
    return apply_ailment_to_defender(battle, ctx, value, ailment="ねむり")
```

### data/move.py の更新

`さいみんじゅつ` の MoveData に `handlers` を追加する。

```python
"さいみんじゅつ": MoveData(
    type="エスパー",
    category="変化",
    pp=20,
    accuracy=60,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.さいみんじゅつ_apply_ailment_to_defender,
        ),
    }
),
```

## 注意点・エッジケース

- ねむり付与の基本ロジック（すでに状態異常がある場合の失敗など）は `ailment_manager` 内部で処理されるため、ハンドラ側のガードは不要
- `うたう`・`キノコのほうし`・`ねむりごな` と効果は同じだが音系・粉系ラベルなし（一切貫通効果なし）
- `さわぐ` 状態中はねむり付与が無効（`ON_TRY_MOVE_2` priority=130 で処理済み）
- `エレキフィールド` 中は地面にいるポケモンへのねむり付与が無効（フレームワーク側で処理済み）
- 命中率 60 は低いため、テストでは `accuracy=100` 固定で実行する
