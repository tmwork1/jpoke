# 実装計画: フォトンゲイザー

## 仕様サマリ（docs/spec/moves/フォトンゲイザー.md 参照）

- タイプ: エスパー / 分類: 特殊（可変） / 威力: 100 / 命中率: 100 / PP: 5
  （champions/move_list.txt に記載なしのため、Gen9 本家準拠の PP をそのまま採用。
  シャドーレイ・テラバースト等と同様の扱い）
- 対象: 相手1体 / 優先度: 0 / 直接攻撃なし
- 効果:
  1. 自分の実数値（ランク補正込み）がこうげき＞とくこうなら物理技になる
     （同値・とくこうが高い場合は特殊技のまま）。てんねん所持の相手に対してもランクを
     考慮して分類を判定し、分類確定後のダメージ計算でランク無視が適用される。
  2. 相手の特性を無視して攻撃する（かたやぶりと同様、`mold_breaker_ignorable` フラグを
     持つ特性が対象。攻撃前後で `ON_BEGIN_MOVE` / `ON_END_MOVE` により無効化・復元する）。

## 現状の実装状況（レビュー前）

`src/jpoke/data/moves/move_ha.py` に登録済みで、分類変化・特性無視ともに実装済みだった。
`docs/spec/moves/フォトンゲイザー.md` と `docs/plan/moves/フォトンゲイザー.md` が存在しなかった
ため、本レビューで新規作成した（実装自体に修正は不要と判断）。

## ハンドラ構成

| Event | Priority | subject_spec | 役割 |
|---|---|---|---|
| `Event.ON_MODIFY_MOVE_CATEGORY` | 100（デフォルト） | `attacker:self`（デフォルト、明示指定なし） | こうげき＞とくこう（ランク補正込み）なら物理技にする |
| `Event.ON_BEGIN_MOVE` | 100（デフォルト） | `attacker:self`（デフォルト、明示指定なし。関数内で `ctx.defender` を直接参照） | 攻撃直前に相手の特性を無効化する |
| `Event.ON_END_MOVE` | 100（デフォルト） | `attacker:self`（デフォルト、明示指定なし。関数内で `ctx.defender` を直接参照） | 攻撃終了後に相手の特性の無効化を解除する |

`MoveHandler` は `skip_subject_check=True` で登録されるため主体照合が行われず、
`subject_spec` を省略するとデフォルトの `"attacker:self"` になる（シャドーレイと同じ扱い。
`docs/plan/moves/シャドーレイ.md` 参照）。ハンドラ関数内で `ctx.attacker` / `ctx.defender` を
直接参照するため、実質的な対象特定には影響しない。

## Priority 根拠

- `ON_MODIFY_MOVE_CATEGORY` は `docs/spec/turn.md` に掲載が無いイベントのため、同種の
  既存実装（`テラバースト_modify_move_category`、`テラクラスター_modify_move_category`）を
  参照して決定した。いずれも priority を指定せずデフォルト（100）で登録されているため、
  同様にデフォルトを採用する。
- `ON_BEGIN_MOVE` / `ON_END_MOVE` は `docs/spec/turn.md` に掲載が無いイベントのため、
  同じ「攻撃前後で相手の特性を無効化・復元する」効果を持つ既存実装
  （`シャドーレイ_disable_defender_ability` / `シャドーレイ_restore_defender_ability`）が
  priority を指定せずデフォルト（100）で登録されているため、同様にデフォルトを採用する。

## 実装箇所（既存実装済み・修正なし）

### handlers/move_attack.py

```python
def フォトンゲイザー_disable_defender_ability(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """フォトンゲイザー: 攻撃直前に相手の特性を無効化する（かたやぶりと同様の対象特性）。"""
    mon = ctx.defender
    if mon.ability.has_flag("mold_breaker_ignorable"):
        battle.add_ability_disabled_reason(mon, "フォトンゲイザー")
    return HandlerReturn(value=value)


def フォトンゲイザー_modify_move_category(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """フォトンゲイザー: 補正込みAがCより高い場合は物理技として計算する。"""
    mon = ctx.attacker
    if mon.ranked_stats["atk"] > mon.ranked_stats["spa"]:
        return HandlerReturn(value="physical")
    return HandlerReturn(value=value)


def フォトンゲイザー_restore_defender_ability(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """フォトンゲイザー: 攻撃終了後に相手の特性の無効化を解除する。"""
    battle.remove_ability_disabled_reason(ctx.defender, "フォトンゲイザー")
    return HandlerReturn(value=value)
```

### data/moves/move_ha.py

```python
"フォトンゲイザー": MoveData(
    type="エスパー",
    category="special",
    pp=5,
    power=100,
    accuracy=100,
    flags={"ignore_ability"},
    handlers={
        Event.ON_MODIFY_MOVE_CATEGORY: h.MoveHandler(
            ha.フォトンゲイザー_modify_move_category,
        ),
        Event.ON_BEGIN_MOVE: h.MoveHandler(
            ha.フォトンゲイザー_disable_defender_ability,
        ),
        Event.ON_END_MOVE: h.MoveHandler(
            ha.フォトンゲイザー_restore_defender_ability,
        ),
    },
),
```

`flags={"ignore_ability"}` は分類用のフラグで、実際の無効化ロジックには使用しない
（シャドーレイと同じ扱い。`types/literals.py` のコメント参照）。

## リーサル計算

対象外（`docs/progress/move.md` のリーサル実装列が `n/a`）。分類変化・特性無視ともに
ダメージ計算の分岐に影響するのみで、HP状態分布を直接変更する効果ではないため、
リーサル計算の追加ハンドラは不要（`battle.calc_damages()` が通常の
`ON_MODIFY_MOVE_CATEGORY` / `ON_BEGIN_MOVE` / `ON_END_MOVE` ハンドラをそのまま経由する）。

## テスト方針（既存実装済み・修正なし）

- 補正込みAがCより高いポケモン（カイリキー）で物理技になることを確認
- 補正込みAがCと同値以下のポケモン（フーディン）で特殊技のままであることを確認
- ハードロック（`mold_breaker_ignorable` 特性）を持つ相手への効果抜群ダメージ軽減が
  無視されることを確認（`damage_modifier == 4096`）
- 攻撃終了後に相手の特性の無効化が解除される（`defender.ability.enabled is True`）ことを確認
- 上記はシャドーレイ・テラバーストと同じテストパターンを踏襲する

## 注意点・エッジケース

1. **PP**: `docs/champions/move_list.txt` にフォトンゲイザーの行が存在しない
   （タイプ・分類が可変の専用技はテーブル形式の一次情報から除外されていると考えられる。
   シャドーレイ・テラバースト・テラクラスター・メテオドライブも同様）。Gen9 本家準拠の
   PP=5 を維持する。
2. **分類判定とてんねん**: `フォトンゲイザー_modify_move_category` は `ctx.attacker.ranked_stats`
   を参照するため、防御側のてんねん（ランク無視ハンドラ）の影響を受けない。分類判定後の
   ダメージ計算では、`フォトンゲイザー_disable_defender_ability` により相手の特性
   （てんねんを含む `mold_breaker_ignorable` 対象特性）が無効化されるため、ダメージ計算でも
   相手のランクが考慮される（仕様と一致）。
3. **スロースタート**: 分類判定時は特性の影響を考慮しない（`ranked_stats` はスロースタートの
   実数値半減補正を含まない生のランク補正込み値のため）。物理技になった場合のダメージ計算
   ではスロースタートの補正が通常通り適用される（第八世代以降の仕様と一致。専用コード不要）。
4. **かがくへんかガス・かたやぶり無効化チェック**: 書かない
   （既存の `has_flag` 判定のみで十分）。
5. **他の技が出す技（ゆびをふる等）経由の発動**: 該当する技呼び出し機構自体が本プロジェクト
   で未実装のため対象外。
6. **わたげとの相互作用**: わたげ自体が本プロジェクト未実装のため対象外。
