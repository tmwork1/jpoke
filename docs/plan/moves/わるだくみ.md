# 実装計画書: わるだくみ

## 仕様

### 技データ

- タイプ: あく
- カテゴリ: 変化
- PP: 20
- 命中率: - （必中）
- 対象: 自分

### 効果

自分の **とくこう** ランクを2段階上げる。

### 参照仕様書

- `docs/spec/moves/わるだくみ.md`

---

## subject_spec と priority

### イベント: `ON_STATUS_HIT`

- **subject_spec**: `"attacker:self"`（`MoveHandler` のデフォルト値を使用）
- **priority**: 100

**根拠**: `docs/spec/turn.md` に `ON_STATUS_HIT` の独立エントリは無いが、同種の自分対象ランクアップ
変化技ハンドラ（`りゅうのまい_modify_attacker_stats`, `ロックカット_modify_attacker_stats` など）が
いずれも `h.MoveHandler(...)` の priority デフォルト値（100）で登録されているため、これと同一
パターンとして priority=100 を採用する。

## ハンドラ構成

### 1. `src/jpoke/handlers/move_status.py`

共通ユーティリティ `handlers/move.py` の `modify_attacker_stats` を利用する（既存実装済み）。

```python
def わるだくみ_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"spa": 2})
```

五十音順の挿入位置: `ロックカット_modify_attacker_stats` の後、`ワンダールーム_activate_global_field`
の前（`わたほうし_modify_defender_stats` の直後）。

### 2. `src/jpoke/data/moves/move_wa.py`

```python
"わるだくみ": MoveData(
    type="あく",
    category="status",
    pp=20,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.わるだくみ_modify_attacker_stats,
        )
    }
),
```

## 注意事項

- `modify_attacker_stats` は `battle.modify_stats(ctx.attacker, stats, source=ctx.attacker)` を
  呼び出す共通関数であり、特性による無効化などは既存の `battle.modify_stats` /
  `ON_STATUS_HIT` 発火経路側で共通処理されるため、本ハンドラ側で個別対応は不要
- 自分対象の変化技のため `accuracy` は設定しない（必中）
- 確率発動ではない（`chance` 引数はデフォルト1のまま呼び出す）ため、`resolve_secondary_chance`
  は常に1を返し無条件でランクアップする
- リーサル計算（`handlers/lethal.py`）は対象外（`docs/progress/move.md` のリーサル実装列は n/a）。
  わるだくみは相手にダメージを与えない変化技のため、リーサル計算ハンドラの実装は不要
