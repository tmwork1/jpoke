# 実装計画書: まきびし

## 仕様要約

- タイプ: じめん / カテゴリ: 変化 / PP: 20 / 対象: 相手の場（target="foe_side"）
- 使用後、相手はポケモンを交代するたびに出てきたポケモンが最大HPの 1/8 のダメージを受ける
- 2回使用すると最大HPの 1/6、3回使用すると最大HPの 1/4 のダメージになる（最大 3 層）
- ひこうタイプ・特性「ふゆう」のポケモンには無効
- 「どくタイプ」のポケモンには無効

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「設置技: すでに最大」が priority 100 として記載されている。`ステルスロック_set_field` と同じパターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「ま」行 — `ほたるび_modify_attacker_stats` より後、`まるくなる_apply` の前に挿入する。

```python
def まきびし_set_field(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """まきびし: 相手陣営に「まきびし」を 1 層設置する（最大 3 層）。"""
    side = battle.get_side(ctx.defender)
    if not side.activate("まきびし", 1):
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

既存の `"まきびし"` エントリに handlers を追加する。

```python
"まきびし": MoveData(
    type="じめん",
    category="変化",
    pp=20,
    target="foe_side",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.まきびし_set_field,
        ),
    }
),
```

## 注意点・エッジケース

- `side.activate("まきびし", 1)` は重複設置（層数加算）を side マネージャが管理する。最大 3 層を超える場合に False を返すか確認が必要
- `ステルスロック_set_field` は count=1 が「設置済み判定」として機能しているが、まきびしは多層なので `activate` の戻り値の意味をサイドフィールドの実装で確認すること
- ひこうタイプ・ふゆう特性・どくタイプの判定はサイドフィールドの `ON_SWITCH_IN` ハンドラ側で処理されるため、技ハンドラ側でのガードは不要
- ダメージ量（1/8 → 1/6 → 1/4）の段階制御もサイドフィールドハンドラ側の責務
