# 実装計画書: さいきのいのり

## 仕様要約

- タイプ: ノーマル / カテゴリ: 変化 / PP: 1（本家 SV 基準。`src/jpoke/data/ps-champ-ja/moves.json` に単独のエントリなし。詳細は仕様書参照） / 命中: なし（必中） / 対象: 自分
- ひんし状態の味方（控え）を1体、最大HPの1/2（切り捨て）を回復して復活させる
- ひんし状態の味方（控え）がいないときは失敗する
- 実機は復活対象をプレイヤーが選択するが、本プロジェクトは選択UIを持たないため選出順で最初のひんし状態の味方を自動選択する（タイムアウト時の実機挙動に準拠）
- `heal` フラグを持つ（かいふくふうじによる選択禁止に必要）
- `target="self"` を持つ（まもる・マジックコートに阻まれない）

## Priority 根拠

### ON_BEFORE_APPLY_MOVE（priority=100）

`docs/spec/turn.md` では実装コード上の `ON_BEFORE_APPLY_MOVE` は `Event.ON_APPLY_MOVE` として掲載されている。この節には

> | 100 | 自己回復技: HP満タン |
> | 100 | いやしのはどう等: HP満タン |

など、自己完結する回復技の失敗条件が priority=100 で列挙されている。`さいきのいのり` の「ひんし状態の味方不在で失敗」も同種の自己完結的な失敗条件のため、同じ priority=100（`MoveHandler` のデフォルト）を採用する。`バトンタッチ_check`（控え不在での交代不可判定）も同じ priority=100 を明示的に使用しており、パターンとして一致する。

### ON_STATUS_HIT（priority=100）

`docs/spec/turn.md` に `ON_STATUS_HIT` の直接記載はないが、同種の自己完結型ハンドラ（`じこさいせい_heal_self`、`つるぎのまい_modify_attacker_stats` 等）はすべて `ON_STATUS_HIT` priority=100（`MoveHandler` のデフォルト）を使用しており、これに合わせる。

## 実装箇所（現状のコードに準拠）

### 1. `src/jpoke/data/moves/move_sa.py`

```python
"さいきのいのり": MoveData(
    type="ノーマル",
    category="status",
    pp=1,
    target="self",
    flags={"heal"},
    handlers={
        Event.ON_BEFORE_APPLY_MOVE: h.MoveHandler(
            hs.さいきのいのり_check,
        ),
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.さいきのいのり_revive,
        ),
    }
),
```

`target="self"` を明示しないと `Move.target` が既定値 `"foe"` になり、
`Move.is_blocked_by_protect` / `Move.is_reflectable` が誤って `True` を返し、
`まもる`・`マジックコート` に阻まれる誤動作を起こす（自分専用技のため両方とも効かないのが正しい）。

### 2. `src/jpoke/handlers/move_status.py`

```python
def さいきのいのり_check(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """さいきのいのりの失敗条件: ひんし状態の味方（控え）が1体もいない場合は失敗する。"""
    player = battle.get_player(ctx.attacker)
    state = battle.player_states[player]
    if not any(mon.fainted for mon in state.bench):
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="さいきのいのり_ひんしなし")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def さいきのいのり_revive(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """さいきのいのりの効果: ひんし状態の味方を選出順で先頭から1体選び、
    最大HPの1/2（切り捨て）を回復して復活させる。

    実機では復活させる味方をプレイヤーが選ぶが、通信対戦で選択が時間切れに
    なった場合は選出順で最初のひんし状態のポケモンが自動的に選ばれる
    （一次情報: docs/wiki/moves/さいきのいのり.html 技の仕様節）。本プロジェクトは
    復活対象選択のUIを持たないため、常にこの自動選択と同じ挙動（選出順で
    最初にひんし状態の味方）を採用する。`さいきのいのり_check` が呼び出し前に
    候補の存在を保証しているため、ここでは存在チェックを行わない。
    """
    player = battle.get_player(ctx.attacker)
    state = battle.player_states[player]
    target = next(mon for mon in state.bench if mon.fainted)
    battle.modify_hp(target, v=max(1, target.max_hp // 2), source=ctx.attacker)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース

- **控えのひんし不在時の失敗判定**: `state.bench`（選出済み・非アクティブのポケモン）に `fainted` が1体もいない場合は `ON_BEFORE_APPLY_MOVE` で失敗させる。`バトンタッチ_check` と同じく `state.bench` を直接参照する（`battle.query` に専用ヘルパーがないため）。
- **復活対象の自動選択**: `state.bench` は選出順（`selected_indexes` の順序、通常はチーム登録順）を保つため、`next(mon for mon in state.bench if mon.fainted)` で選出順最初のひんし状態のポケモンを取得できる。これは実機の「タイムアウト時に一番上のひんし状態のポケモンを復活させる」挙動と一致する。
- **回復量の計算**: `max(1, target.max_hp // 2)` で最大HPの1/2を切り捨てで計算する（`fixed_recoil` 等の既存ハンドラと同じ `max_hp // 2` パターンを流用）。`battle.modify_hp` の `r=0.5` 引数も利用可能だが、既存の反動ダメージ系ハンドラとの実装パターン統一のため `v=` を用いる。
- **かいふくふうじとの相互作用**: `flags={"heal"}` を設定することで、使用者がかいふくふうじ状態のときに技を選択できない（`かいふくふうじ_try_action` の既存の汎用チェックがそのまま機能する）。復活対象（ひんし状態だった控え）はフェイント（交代処理）時に `remove_all_volatiles` で揮発性状態がクリアされているため、`かいふくふうじ_block_heal`（`ON_MODIFY_HEAL`）が誤って回復をブロックすることはない。
- **状態異常は回復しない**: 効果文言に「状態異常回復」の記載がないため、`battle.ailment_manager` は操作しない（`いやしのねがい` 等の全回復系とは異なる）。
- **HP変化の重複ガード不要**: `battle.modify_hp` は `v == 0` のとき早期リターンする。`target.max_hp // 2` は通常のポケモンの最大HP（10以上）では常に1以上になるため、`max(1, ...)` は実質的な安全策として付与するのみで、通常は発火しないケースである。
- **`battle.faint` で控えを疑似的にひんし化するテスト方法**: `battle.faint(mon)` は対象が非アクティブでも使用でき、`judge_winner` は他に生存個体がいる限り勝敗を確定させない。テストでは控えメンバーに対して `battle.faint(mon)` を呼び出すことで「ひんし状態の控え」を用意する。
- **ダブル/マルチ/テラレイド固有仕様は対象外**: 本プロジェクトはシングルバトル専用のため、即座に場に出る条件・マルチバトルの味方対象外などの仕様は実装しない。
- **`ゆびをふる`・`スケッチ` との相互作用は対象外**: いずれも本プロジェクトで未実装のため、現時点では対応不要。
