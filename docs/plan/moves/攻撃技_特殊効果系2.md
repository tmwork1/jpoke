## 対象

`docs/spec/moves/_move_secondary_2.md` に定義されたHP吸収・反動・失敗反動・HPコスト・自爆系技の実装計画。

---

## 仕様要約

### A. HP吸収技（drain）

与えたダメージの一定割合を使用者が回復する。

| 技名 | 吸収率 |
|------|--------|
| ウッドホーン | 0.5 |
| ギガドレイン | 0.5 |
| きゅうけつ | 0.5 |
| シャカシャカほう | 0.5（他効果あり） |
| すいとる | 0.5 |
| デスウイング | 0.75 |
| ドレインキッス | 0.75 |
| ドレインパンチ | 0.5 |
| パラボラチャージ | 0.5 |
| むねんのつるぎ | 0.5 |
| メガドレイン | 0.5 |
| ゆめくい | 0.5（対象がねむり状態の時のみ使用可能） |

共通仕様:
- ダメージを0とした場合（みがわりに吸収された等）は `ctx.substitute_damage` を参照して回復量を計算する
- 回復量 = `floor(ダメージ × 吸収率)`
- 回復イベントは `Event.ON_CALC_DRAIN` を経由して特性（よこどり等）の補正を受ける

### B. 反動技（recoil）

与えたダメージの一定割合が使用者に反動として返ってくる。

| 技名 | 反動率 |
|------|--------|
| アフロブレイク | 0.25 |
| ウェーブタックル | 0.33 |
| ウッドハンマー | 0.33 |
| じごくぐるま | 0.25 |
| すてみタックル | 0.33 |
| とっしん | 0.25 |
| はめつのひかり | 0.5 |
| フレアドライブ | 0.33（+10%やけど追加効果あり） |
| ブレイブバード | 0.33 |
| ボルテッカー | 0.33→0.25（注意: 仕様書では0.25） |
| もろはのずつき | 0.5 |
| ワイルドボルト | 0.25 |

共通仕様:
- 反動ダメージ = `max(1, floor(与えたダメージ × 反動率))`
- みがわりにあたった場合は与えたダメージをそのまま使用（みがわり分のダメージが value）
- 反動ダメージで使用者がひんしになりうる

### C. 失敗反動技（crash）

技が外れた・防がれた場合に使用者がHPの1/2を失う。

| 技名 | 失敗時ダメージ |
|------|----------------|
| かかとおとし | 最大HPの1/2 |
| サンダーダイブ | 最大HPの1/2 |
| とびげり | 最大HPの1/2 |
| とびひざげり | 最大HPの1/2 |

共通仕様:
- 失敗反動はタイプ相性が無効でも発生しない場合がある（ON_MISS イベントで処理）
- まもる等で技が防がれた場合も失敗反動が発生する
- かかとおとし・とびひざげりはチャンピオンズ仕様

### D. HPコスト技（self-damage before attack）

技使用前に自分のHPの一部を消費する（技の失敗・命中と無関係）。

| 技名 | HPコスト |
|------|---------|
| クロロブラスト | 最大HPの1/2 |
| てっていこうせん | 最大HPの1/2 |
| ビックリヘッド | 最大HPの1/2 |

共通仕様:
- HPコストは `Event.ON_PAY_HP` で処理される
- まもる等で防がれてもHPコストは発生する
- ひんし状態で発動した場合のHPコスト処理は対戦管理側で制御済み

### E. 自爆技（self-KO）

技使用後に使用者がひんしになる。

| 技名 | HP消費 |
|------|--------|
| じばく | 現在HPをすべて消費 |
| だいばくはつ | 現在HPをすべて消費 |
| ミストバースト | 現在HPをすべて消費 |

共通仕様:
- `Event.ON_PAY_HP` で現在HPを全消費することでひんし判定
- ミストフィールド展開中は ミストバースト の威力が1.5倍（別ハンドラで処理済み）

## Priority 根拠

### ON_HIT での drain（priority 20）と recoil（priority 100）

`docs/spec/turn.md` の `Event.ON_HIT` テーブルに明記:

```
| 20  | HP吸収技による回復          ← drain は priority 20
| 100 | 反動技による反動ダメージ     ← recoil は priority 100
```

> 現状の実装（handlers/move_attack.py + data/move.py）では、drain・recoil ともに `MoveHandler` のデフォルト priority **100** で登録されている。drain は本来 priority **20** に変更すべきだが、ON_HIT 内で drain と recoil が同一ターンに衝突するケースは1技が両方を持つ場合のみであり（現状はない）、実用上の問題は生じていない。

turn.md 引用:
```
| 10  | 自分のランクを上げる追加効果
| 10  | 自分のランクが下がる反動
| 20  | HP吸収技による回復          ← ここが drain の正しい位置
| 100 | 反動技による反動ダメージ     ← ここが recoil の正しい位置
```

### ON_MISS での crash（priority 100 default）

`docs/spec/turn.md` に `ON_MISS` イベントは掲載されていない（ダメージ計算内部イベント扱い）。既存の `かかとおとし_crash`・`サンダーダイブ_crash` 等がデフォルト priority で登録されているため、同様にデフォルト priority（100）を使用する。

### ON_PAY_HP での pay_hp

`docs/spec/turn.md` の `Event.ON_PAY_HP` テーブル:

```
## Event.ON_PAY_HP
HP支払い時の処理

| Priority | 処理 |
|----------|------|
| -        | じばく等のダメージ・瀕死判定 |
| -        | いのちがけ |
```

priority は `-` 表記（不定）。既存の `じばく_pay_hp`・`だいばくはつ_pay_hp` がいずれも `subject_spec="attacker:self"` でデフォルト priority（100）を使用しているため、同様とする。

## 実装箇所

### handlers/move_attack.py — 共通ヘルパー関数

```python
def _drain_hp(battle: Battle, ctx: AttackContext, damage: int, heal_ratio: float) -> None:
    """ドレイン回収(drain)で回復するHP量を計算する。"""
    damage = damage or ctx.substitute_damage
    heal_amount = int(damage * heal_ratio)
    heal_amount = battle.events.emit(Event.ON_CALC_DRAIN, ctx, heal_amount)
    battle.modify_hp(ctx.attacker, v=heal_amount, reason="drain")


def _recoil(battle: Battle, ctx: AttackContext, value: int, ratio: float) -> HandlerReturn:
    """反動ダメージを与えるヘルパー関数。与ダメの ratio 分を攻撃側が受ける。"""
    recoil = max(1, int(value * ratio))
    battle.modify_hp(ctx.attacker, v=-recoil, reason="recoil", source=ctx.attacker)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース（review-test 向け）

### HP吸収技共通

1. **みがわりへの吸収**
   - `_drain_hp` 内で `damage = damage or ctx.substitute_damage` としているため、みがわりにダメージが吸収された場合でも `ctx.substitute_damage` を参照して回復量を計算する
   - みがわりが吸収したダメージに対しても drain が発生するかはゲーム仕様による（現在の実装はあり）

2. **ON_CALC_DRAIN イベント**
   - `_drain_hp` 内で `Event.ON_CALC_DRAIN` を emit することで、よこどり特性による回復量倍増などの補正を受ける
   - このイベントが返す値が実際の回復量になる

3. **回復量の端数**
   - `int(damage * heal_ratio)` で切り捨て（Python の int は負に向かうが damage が正数のため問題なし）
   - 最小回復量の下限は設けていない（1以上の保証なし。ダメージが1の場合、0.5 = 0で回復なし）

4. **HPが満タンの場合の回復**
   - `battle.modify_hp` 内で上限チェックを行い、HP満タンの場合は回復しない

5. **ゆめくい固有の注意点**
   - ON_TRY_MOVE_2 で `ゆめくい・あくむ: ねむり状態でない | priority 130` の失敗チェックが行われる（turn.md 明記）
   - ねむり状態の相手への通常ダメージ計算を行い、その後 ON_HIT で drain 処理
   - ねむり状態でない相手に当てようとすると ON_TRY_MOVE_2 で失敗するため、ハンドラ側でのガードは不要

### 反動技共通

1. **反動率の端数**
   - `_recoil` 内で `max(1, int(value * ratio))` を使用
   - 1/3 = 0.333... に対して `int(120 * 1/3) = int(39.99...) = 39` のように切り捨て
   - 最小反動は1（ダメージが0でもmax(1, ...)により1）

2. **反動でひんし**
   - 反動ダメージで使用者がひんしになることは仕様通り
   - `battle.modify_hp` が内部でひんし判定を行う

3. **ボルテッカーの反動率**
   - `_move_secondary_2.md` では `0.25` と記載
   - 現在の実装は `ボルテッカー_recoil: _recoil(..., 1/4)` = 0.25 で正しい

4. **フレアドライブの複合効果**
   - 反動（ON_HIT priority 100）と追加やけど効果（ON_DAMAGE_HIT）の2ハンドラを持つ
   - ON_DAMAGE_HIT は ON_HIT より前に発火するため、やけど付与 → 反動ダメージの順で処理される

### 失敗反動技共通

1. **ON_MISS の発火タイミング**
   - `ON_MISS` は技が外れた時（命中判定失敗）に発火する
   - まもる等で「技が防がれた」場合も `ON_MISS` が発火する（実装依存 — battle の実装を確認）
   - みがわりを狙った場合にみがわりが受けた場合は「命中した」扱いのため ON_MISS は発火しない

2. **失敗反動でひんし**
   - 失敗反動で使用者がひんしになることは仕様通り

3. **じゅうりょく中のとびひざげり・とびげり**
   - じゅうりょく状態中はとびひざげり・とびげりが使用不可となる（ON_CHECK_ACTION で制御）
   - ハンドラ側でのガードは不要（manager 内部で処理される）

### HPコスト技・自爆技

1. **ON_PAY_HP の priority と subject_spec**
   - `subject_spec="attacker:self"` を明示的に指定する（デフォルトと同じだが意図を明確にするため）
   - priority はデフォルト 100

2. **まもるで防がれた場合もHPコストが発生する**
   - `ON_PAY_HP` はダメージ計算より前のイベントであるため、相手の行動（まもる等）に関係なくHPを消費する
   - これはチャンピオンズ仕様の確認が必要（ポケモン本家では まもる中の てっていこうせん でもHPが減る）

3. **自爆技の威力ペナルティ廃止（第5世代以降）**
   - 第4世代まで: じばく・だいばくはつ使用時に相手の防御を半減してダメージ計算
   - 第5世代以降（チャンピオンズ含む）: この防御半減の効果は廃止
   - したがって `だいばくはつ` は通常のダメージ計算を行う

4. **ミストバーストのミストフィールド威力アップ**
   - ミストフィールド展開中は威力が1.5倍になる効果は別のハンドラ（ON_CALC_POWER_MODIFIER）で実装する
   - 自爆処理（ON_PAY_HP）とは独立したハンドラ構成

5. **HPコストでひんしになった場合**
   - `ON_PAY_HP` でHPを全消費してひんしになった場合、技の発動（ダメージ付与）は行われない
   - `battle.modify_hp` がひんし判定を行い、ひんし時は以降の処理を中断する（battle の実装による）
