## 仕様要約

- タイプ: ノーマル / 分類: 変化 / PP: 10 / 命中: なし / 優先度: 0
- 効果: 使用者の **最大HPの1/2（小数点以下切り上げ）** を消費してみがわりを生成し、
  みがわり状態を引き継いだまま控えのポケモンに交代する。
  みがわりのHP自体は通常の `みがわり` と同じく **最大HPの1/4（切り捨て）**。
- 失敗条件:
  1. 使用者のHPが最大HPの1/2以下（すなわち `hp <= max_hp // 2`）
  2. すでにみがわり状態
  3. 交代できる控えのポケモンがいない
- 能力ランク変化やみがわり以外の揮発性状態は交代先に引き継がない（バトンタッチと異なる）。
- `さいせいりょく` はHP減少後・交代後に発動する。

## Priority 根拠

- `ON_BEFORE_APPLY_MOVE` priority=100 は `みがわり_check` と同じ値（`docs/spec/turn.md` の `ON_APPLY_MOVE` セクションに相当する事前チェック群はすべて priority=100）。
- `ON_STATUS_HIT` priority=100 は `みがわり_apply` (デフォルト) と同一。変化技命中後のメイン効果はすべて priority=100 が標準（`docs/spec/turn.md` では未掲載のため既存の同種ハンドラを根拠とする）。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順で `しっぽをふる` の前に挿入する（現在 `しっぽをふる` は614行目）。

```python
def しっぽきり_check(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """しっぽきりの失敗条件チェック。

    以下のいずれかに該当する場合は失敗する:
    - 使用者がすでにみがわり状態
    - 使用者のHPが最大HPの半分以下
    - 交代できる控えのポケモンがいない
    """
    mon = ctx.attacker
    player = battle.get_player(mon)
    if mon.has_volatile("みがわり"):
        battle.add_event_log(mon, LogCode.MOVE_FAILED,
                             payload={"reason": "しっぽきり_みがわり中"})
        return HandlerReturn(value=False, stop_event=True)
    if mon.hp <= mon.max_hp // 2:
        battle.add_event_log(mon, LogCode.MOVE_FAILED,
                             payload={"reason": "しっぽきり_HP不足"})
        return HandlerReturn(value=False, stop_event=True)
    if not battle.get_available_switch_commands(player):
        battle.add_event_log(mon, LogCode.MOVE_FAILED,
                             payload={"reason": "しっぽきり_交代不可"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def しっぽきり_apply(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """しっぽきりの効果: HP消費・みがわり生成・ピボット交代。

    消費HPは最大HPの1/2（小数点以下切り上げ）。
    みがわりのHPは最大HPの1/4（切り捨て）で通常みがわりと同じ。
    """
    mon = ctx.attacker
    player = battle.get_player(mon)
    # 最大HPの1/2を消費（切り上げ = -(max_hp + 1) // 2 の符号反転）
    cost = (mon.max_hp + 1) // 2
    battle.modify_hp(mon, -cost)
    # みがわり生成
    battle.volatile_manager.apply(mon, "みがわり", hp=mon.max_hp // 4)
    # ピボット交代（交代先選択をプレイヤーに委ねる）
    battle.player_states[player].interrupt = Interrupt.PIVOT
    return HandlerReturn(value=value)
```

インポートに `Interrupt` を追加する（`move_status.py` の import 行に `Interrupt` が不足している場合）。

## 注意点・エッジケース（review-test 向け）

1. **HP消費の端数処理**
   - 消費量は `(max_hp + 1) // 2`（切り上げ）。最大HPが奇数のとき `みがわり` の `max_hp // 4`（切り捨て）との差が生じる点に注意。
   - 例: 最大HP=201 → 消費=101、みがわりHP=50。

2. **失敗条件の評価順序**
   - みがわり状態チェック → HP不足チェック → 交代可否チェックの順。
   - 交代先がいない場合はみがわりも生成されない（Wikiより）。

3. **ピボット交代とさいせいりょく**
   - HP減少後に `さいせいりょく` 発動圏に入ったとき、きのみの発動が先に起きてから交代する（`Interrupt.PIVOT` はターン後処理で実行されるため自然に後続処理を挟む）。

4. **バトンタッチとの差異**
   - しっぽきりはみがわり以外の能力ランク・揮発性状態を引き継がない。バトンタッチとは別の実装が必要。

5. **みがわり_check との違い**
   - 通常みがわりは `hp <= max_hp // 4` が閾値、しっぽきりは `hp <= max_hp // 2`。
   - しっぽきりには「交代先がいない場合失敗」の条件が追加で存在する。
   - そのため `みがわり_check` の再利用は不可。

6. **ON_BEFORE_APPLY_MOVE のガード**
   - `skip_subject_check=True`（MoveHandler の継承により設定済み）のため、主体の照合はスキップされる。失敗条件のガードはハンドラ内で完結させる。

7. **まもる等による無効化**
   - まもる判定は `ON_TRY_MOVE_1` より前に処理される（ `docs/spec/turn.md` 参照）。
   - 変化技かつ自分を対象にするためまもるは無効。ハンドラ側での追加ガードは不要。
