# 実装計画書: みずびたし

## 仕様要約

- タイプ: みず / カテゴリ: 変化 / PP: 20 / 命中: 100 / 対象: 1 体選択
- 相手の「タイプ」を「みず」タイプに変える（タイプを追加するのではなく、完全に置き換える）
- 相手がすでにみずタイプのみの場合は失敗する（`docs/spec/turn.md` の `ON_APPLY_MOVE` priority 100 に「みずびたし等: すでに該当タイプ」として明記）

## Priority 根拠

- `ON_BEFORE_APPLY_MOVE` priority 100: `docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「みずびたし等: すでに該当タイプ」が priority 100 として明記されている
- `ON_STATUS_HIT` priority 100（デフォルト）: ハロウィン・もりののろいと同じパターン
- `ON_VOLATILE_START` / `ON_VOLATILE_END` priority 100（デフォルト）: ハロウィン volatile と同じパターン

## タイプ上書きの実装方針

みずびたしは「タイプを追加」ではなく「タイプをみずに変える」技であり、`added_types` の append だけでは実現できない。

**採用方針: `volatile_override_type` フィールドを追加する**

1. `src/jpoke/model/pokemon.py` に `self.volatile_override_type: Type | None = None` フィールドを追加する
2. `types` プロパティに `volatile_override_type` の参照を追加する（`ability_override_type` と同じ優先度またはそれより低い優先度）
3. `reset_on_switch_out()` で `volatile_override_type = None` をリセットする
4. volatile の `ON_VOLATILE_START` で `volatile_override_type = "みず"` に設定し、`added_types` をクリアする
5. volatile の `ON_VOLATILE_END` で `volatile_override_type = None` に戻す

`types` プロパティの優先順位（修正後）:
```
テラスタル > ability_override_type > volatile_override_type > data.types
```

失敗判定では「すでに `types == ["みず"]`」ではなく「すでに `volatile_override_type == "みず"` かつ `added_types` が空」または、シンプルに「`types` が `["みず"]` のみ」で判定する。

`types` プロパティを修正:
```python
@property
def types(self) -> list[Type]:
    if self.active_tera_type:
        if self.active_tera_type == 'ステラ':
            base = self.data.types
        else:
            base = [self.active_tera_type]
    elif self.ability_override_type is not None:
        base = [self.ability_override_type]
    elif self.volatile_override_type is not None:
        base = [self.volatile_override_type]
    else:
        base = self.data.types

    if not self.added_types:
        return base
    extra = [t for t in self.added_types if t not in base]
    return base + extra if extra else base
```

## 注意点・エッジケース

- **`volatile_override_type` は新規フィールドのため、`pokemon.py` の `__init__`・`reset_on_switch_out`・`types` プロパティの 3 箇所すべてに追加が必要**
- 失敗判定は「すでにみずタイプのみである」ことを確認するが、みずタイプに加えて他タイプ（ハロウィンや もりののろい などで追加されたタイプを含む）を持っている場合は成功させる必要がある。判定式 `ctx.defender.types == ["みず"]` はリストの完全一致であるため、複合タイプのケースで正しく成功する
- `ON_VOLATILE_START` で `added_types.clear()` を行うのは、ハロウィン・もりののろい状態が付与されている相手にみずびたしを使った場合、追加タイプが残らないようにするため
- テラスタル中の相手に使用した場合、`types` プロパティはテラスタルタイプを返すため `volatile_override_type` は反映されない（テラスタルが優先）。ただし volatile 自体は付与され、テラスタル解除後（交代含む）には volatile もリセットされるため実質問題なし
- 交代すると `volatile_override_type = None` にリセットされるため、みずタイプの効果は消える（`reset_on_switch_out` でリセット）
- みずびたし状態の相手にハロウィン・もりののろいを使うと、`added_types` に追加されるが `volatile_override_type = "みず"` が優先されるため「みず＋ゴースト」タイプになる。これは `types` プロパティが `[volatile_override_type] + extra` を返すことで自然に処理される
- `まほうのこな`（相手をエスパータイプにする）も同じパターンで実装可能。みずびたしと設計を統一することが望ましい
