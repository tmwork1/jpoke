# 計画書: メテオビーム

## 仕様要約

- いわタイプ・特殊・威力120・命中90・PP10
- 2ターン技。1ターン目は攻撃せず、**自分のとくこうを100%の確率で1段階上げる**。
- 2ターン目に攻撃する（命中判定・急所判定は攻撃ターンのみ）。
- まもる: ○、マジックコート: ×。追加効果ではないため**ちからずくの対象外**（威力も上がらない）。
- **パワフルハーブ使用時**（1ターンで攻撃する場合）も、わざが外れたり無効化されたりしても **とくこうは必ず1段階上がる**。
- ダイマックス技に変わった場合は1ターンで使用できる（本実装スコープ外）。

## ハンドラ構成

### move.py の handlers（技 "メテオビーム"）

MoveData.handlers の型は `dict[Event | DomainEvent, Handler | list[Handler]]` なので、同一イベントに複数ハンドラをリストで渡せる。

| イベント | priority | subject_spec | 役割 | ハンドラ関数 |
|---|---|---|---|---|
| `ON_MOVE_CHARGE` | 50 | `attacker:self` | とくこう+1（パワフルハーブのスキップより先に実行して必ず発動） | `メテオビーム_charge`（単体関数。下記参照） |

**推奨**: priority=50 の単一ハンドラ `メテオビーム_charge` 内でとくこう上昇と揮発付与の両方を行う。

### volatile.py の handlers（volatile "メテオビーム"）

| イベント | priority | subject_spec | 役割 | ハンドラ関数 |
|---|---|---|---|---|
| `ON_MODIFY_COMMAND_OPTIONS` | 100 | `source:self` | 行動を強制（FORCED）にする | `force_command` |
| `ON_HIT` | 100 | `attacker:self` | 攻撃成功後に揮発状態を解除 | `メテオビーム_remove_volatile` |

## Priority 根拠

### ON_MOVE_CHARGE (priority=50)

`docs/spec/turn.md` の `Event.ON_MOVE_CHARGE` セクションに以下の記載がある：

> Priority 100 | メテオビーム: とくこう上昇

しかし **パワフルハーブとの相互作用** のため priority=50 を採用する：

- パワフルハーブ（ItemHandler, priority=100）が `stop_event=True` を返すと、同 priority=100 のメテオビームとくこう上昇ハンドラが実行されない可能性がある（同一 priority での実行順は不定）。
- 仕様「パワフルハーブで溜めをスキップした場合もとくこうは上がる」を保証するため、**priority=50**（パワフルハーブより確実に先）を採用する。
- turn.md の priority=100 の記載はパワフルハーブを持たない通常ケースの順序感として記載されていると解釈する。

```python
VolatileName = Literal[

    ...
]
```

### 2. `handlers/move_attack.py` — 新規関数（五十音順の位置に挿入）

```python
def メテオビーム_charge(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """メテオビーム: 1ターン目にとくこうを上げて溜め、2ターン目に攻撃する。

    仕様:
    - 1ターン目（揮発状態なし）: とくこう+1 → 揮発状態付与 → イベント停止（攻撃しない）。
    - 2ターン目（揮発状態あり）: そのまま通過して攻撃に進む。
    - パワフルハーブ使用時: このハンドラ（priority=50）が先に実行され、とくこう上昇と
      揮発付与を行う。その後 priority=100 のパワフルハーブが stop_event=True を返し、
      揮発付与後の「溜め」フローをスキップして即攻撃に移行する。
    - ちからずくの追加効果ではないため、battle.modify_stats を直接呼ぶ（resolve_secondary_chance 不要）。
    """
    attacker = ctx.attacker
    if not attacker.has_volatile("メテオビーム"):
        # とくこう+1（追加効果ではなく必ず発動）
        battle.modify_stats(attacker, {"C": 1}, source=attacker)
        # 揮発状態を付与してターンを停止（攻撃しない）
        battle.volatile_manager.apply(attacker, "メテオビーム", count=1, source=attacker)
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### 3. `handlers/volatile.py` — 新規関数（五十音順の位置に挿入）

## 注意点・エッジケース（review-test 向け）

### パワフルハーブとの相互作用

- メテオビーム_charge (priority=50) が先に実行 → とくこう+1 + 揮発付与 + stop_event=True を返す。
- その後 priority=100 のパワフルハーブ (ItemHandler) は `stop_event=True` を返すが、すでに停止済みのため追加の中断は不要（ただし、メテオビーム_charge が stop_event=True を返すことでパワフルハーブが呼ばれないかもしれない点は要確認）。

**実際の問題**: メテオビーム_charge が priority=50 で `stop_event=True` を返すと、priority=100 のパワフルハーブが呼ばれない。その結果パワフルハーブが消費されず、揮発状態だけが付与されて即攻撃にならない可能性がある。

**解決案**: `メテオビーム_charge` 内では stop_event=True を返さず、代わりにパワフルハーブと同じ priority=100 のハンドラで揮発付与 + stop を行い、とくこう上昇のみを priority=50 の別ハンドラで行う。

`handlers: dict[Event | DomainEvent, Handler | list[Handler]]` の list 形式を使って：

```python
"メテオビーム": MoveData(
    type="いわ",
    category="特殊",
    pp=10,
    power=120,
    accuracy=90,
    handlers={
        Event.ON_MOVE_CHARGE: [
            h.MoveHandler(
                ha.メテオビーム_boost_spa,   # とくこう+1のみ（stop_event を返さない）
                priority=50,
            ),
            h.MoveHandler(
                lambda b, c, v: h.charge_into_volatile(b, c, v, "メテオビーム"),
                priority=100,  # デフォルト。パワフルハーブと同 priority（順不定）
            ),
        ],
    }
),
```

ただし ON_MOVE_CHARGE 内で `charge_into_volatile` と パワフルハーブの実行順が不定になる問題は残る。実装者はテストで動作を確認する必要がある。

**最もシンプルな実装**: 単一関数 `メテオビーム_charge`（priority=50、stop_event=True を返す）に統一し、パワフルハーブが呼ばれないことを前提にする。この場合パワフルハーブの消費処理が行われないため、別途パワフルハーブ側の実装確認が必要。

### ちからずくとの相互作用

- 仕様書に「追加効果ではないため、ちからずくでも威力は上がらず、とくこうを上げることができる」と明記。
- `battle.modify_stats(attacker, {"C": 1}, source=attacker)` を直接呼ぶことで、`resolve_secondary_chance` を経由せず必ず発動する。
- テスト: ちからずく持ちがメテオビームを使用 → とくこう+1 されること、威力補正がないことを確認。

### 揮発状態の解除タイミング

- ソーラービーム volatile と同じく ON_HIT で解除する。
- まもるで防がれた場合（ON_HIT が発火しない）は揮発状態が残る。次ターンも強制コマンドで再攻撃を試みる（ソーラービームと同じ挙動）。
- テスト: 2ターン目にまもる → 揮発状態が残り3ターン目に再攻撃を確認（ソーラービームの既存テストがあれば参照）。

### テスト一覧（推奨）

1. 通常2ターン攻撃: 1ターン目でとくこう+1 + 攻撃なし、2ターン目でダメージあり
2. パワフルハーブ: 1ターンで攻撃 + とくこう+1
3. パワフルハーブ + 外れ: とくこう+1 + ダメージなし
4. ちからずく: とくこう+1 発動、威力補正なし（ダメージ量が通常と同じ）
5. まもる: 2ターン目まもるで防御 → 揮発状態が解除される（またはソーラービームに準じた挙動）
