# 実装計画書: さいはい

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 15 / 命中: - / 対象: 1 体選択（相手）
- 対象（`ctx.defender`）が最後に PP を消費して使用した技（`executed_move`）を、対象の PP を消費して再度実行させる
- 使用者自身は行動を消費せずに対象に技を使わせる（さいはい自体の行動が技の実行となる）
- 命中判定なし、急所なし、追加効果なし、まもる: ○（まもる状態の対象には防がれる）
- マジックコート: × / よこどり: × / オウムがえし: ×

### 失敗条件（ON_BEFORE_APPLY_MOVE priority 100）

- 対象が場に出てから PP を消費する行動を一度もしていない（`executed_move is None`）
- 対象の `executed_move` が以下の技である場合：
  - さいはい
  - 他の技が出る技（オウムがえし / さきどり / しぜんのちから / ねごと / ねこのて / まねっこ / ゆびをふる）
  - 他者の技を覚える技（スケッチ / ものまね / へんしん）
  - 反動で次ターン動けなくなる技（はかいこうせん / ギガインパクト / ブラストバーン / ハイドロカノン / ハードプラント / ガリョウテンセイ等）
  - 溜め技（ソーラービーム / ソーラーブレード / そらをとぶ / あなをほる等）
  - 数ターン行動する技（あばれる / げきりん / はなびらのまい / さわぐ / アイスボール / ころがる / がまん）
  - きあいパンチ / くちばしキャノン / トラップシェル
  - キングシールド / ブロッキング（類似の まもる / かえんのまもり 等は指示可能）
  - ゲップ / わるあがき / Zワザ / ダイマックスわざ
  - おいわい / てをつなぐ

### 指示後の挙動

- 対象の PP を消費して `battle.run_move(ctx.defender, move)` で対象が技を発動する
- 対象の PP が 0 の技を指示した場合は失敗する
- 指示された技は本来の優先度を無視してさいはいの直後に使用される（シングルバトルでは `run_move` を同一ターン内で即実行することで再現できる）

#### さいはいのハンドラ関数

```python
def さいはい_can_apply(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """さいはいの失敗条件を判定する。

    - 対象が PP を消費した行動を一度もしていない場合は失敗する
    - 対象の最後の技が指示できない技の場合は失敗する
    - 対象の最後の技の PP が 0 の場合は失敗する
    """
    move = ctx.defender.executed_move
    if move is None:
        battle.add_event_log(ctx.attacker, LogCode.MOVE_FAILED,
                             payload={"reason": "さいはい"})
        return HandlerReturn(value=False, stop_event=True)
    if move.name in _INSTRUCT_BLOCKED_MOVES:
        battle.add_event_log(ctx.attacker, LogCode.MOVE_FAILED,
                             payload={"reason": "さいはい"})
        return HandlerReturn(value=False, stop_event=True)
    if move.pp == 0:
        battle.add_event_log(ctx.attacker, LogCode.MOVE_FAILED,
                             payload={"reason": "さいはい"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def さいはい_instruct(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """さいはいの効果: 対象に最後に使用した技をもう一度使わせる。

    対象の executed_move の PP を消費して、battle.run_move で技を発動する。
    """
    move = ctx.defender.executed_move
    if move is None:
        return HandlerReturn(value=False, stop_event=True)
    battle.run_move(ctx.defender, move)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース

### `executed_move` の保持と判定

- `Pokemon.executed_move` は `Move` オブジェクトへの参照。ターン内で技が成功した時点（`ON_TRY_MOVE_2` 通過後、`move_executor.py` line 327）に設定される
- 交代時（`reset_on_switch_out`）で `None` にリセットされるため、場に出たばかりのポケモンには必ず失敗する
- `executed_move` は「PP を消費した技」である。ON_CHECK_ACTION で行動失敗した場合（まひ、こんらん等）は PP を消費しないため `executed_move` は更新されない。仕様書の「PP 消費を伴う行動を一度もしていないとき」に合致する

### PP チェックと消費

- 仕様書: 「指示された技の PP が無くなった技を指示したときは失敗する」→ `_can_apply` で `move.pp == 0` をチェック
- `battle.run_move(ctx.defender, move)` 内で `_consume_pp` が呼ばれ、対象のPPが1消費される。さいはい自身の PP も別途消費されるため、1ターンに2PP消費されることになる

### `run_move` による再実行の挙動

- `battle.run_move(ctx.defender, move)` は `move_executor.run_move` に委譲され、通常の技実行フローが走る。これにより、ねむり・こんらん・まひ等の状態異常判定（`ON_TRY_ACTION`）も対象ポケモンに対して実行される
- 対象の状態異常（こんらん・まひ等）により行動が失敗した場合も、仕様通り「さいはいによる行動で状態変化の持続ターンは減少しない」の部分はフレームワーク側の処理に委ねる
- シングルバトルでは対象は 1 体のみのため、ターゲット選択の複雑な仕様（ダブルバトルのターゲット位置管理）は対象外

### 指示できない技リストの管理

- `_INSTRUCT_BLOCKED_MOVES` に収録されていない技（まもる類似技、きょじゅうざん等）は指示できる
- `"label"` での判定ができないため、名前直接比較で行う
- ソーラービーム/ソーラーブレードは「にほんばれ状態のときも失敗する」仕様があるが、シングルバトル向け実装のスコープでは溜め技として一律ブロックで問題ない
- 溜め技判定は技データの `labels` に `"charge"` があれば将来的に利用可能だが、現状は名前での判定を採用する

### まもる・よこどり・マジックコート

- まもる状態の対象には防がれる: `ON_TRY_MOVE_1` priority 100 の「まもる等による無効化」ハンドラがフレームワーク側で処理するため、さいはいハンドラ側での対処は不要
- よこどり: よこどりで盗まれない（`ON_TRY_MOVE_1` priority 80 の「よこどりで技が盗まれる」は発動しない）
- マジックコート: マジックコート状態でも防がれない（反射されない）。`ON_CHECK_REFLECT` で対象の技がマジックコート対象外として扱われる

### `ON_STATUS_HIT` の `value` 戻り値

- `battle.run_move` は戻り値を返さない（None）。`HandlerReturn(value=value)` で元の value（True）を維持して返す
