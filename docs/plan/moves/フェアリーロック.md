# 実装計画書: フェアリーロック

## 仕様要約

- タイプ: フェアリー / カテゴリ: 変化 / PP: 10 / 対象: 全体の場 / 優先度: 0
- 命中判定なし、まもる無効、マジックコート無効、よこどり無効
- 使用した次のターンが終了するまで、場に出ている**全ての**ポケモンが交代・にげるできない（フェアリーロック場の状態）
- 場がすでにフェアリーロック状態の場合は失敗する
- ゴーストタイプも例外なく拘束される（`check_trapped_not_ghost` は使わない）

## ハンドラ構成

| イベント | priority | subject_spec | 登録先 | 役割 |
|----------|----------|--------------|--------|------|
| `Event.ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | `data/move.py` | グローバルフィールドを「フェアリーロック」に設定する（失敗時は False を返す） |
| `Event.ON_CHECK_TRAPPED` | 100（デフォルト） | `source:self` | `data/field.py` | フェアリーロック中の全ポケモンの交代を禁止する（ゴーストタイプも含む） |
| `Event.ON_TURN_END` | 140 | `source:self` | `data/field.py` | グローバルフィールドのカウントを 1 減らし、0 になったら解除する |

## Priority 根拠

- `ON_STATUS_HIT` (priority 100): `docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「天候・フィールド: すでに同じ」が priority 100 として記載。`じゅうりょく_activate_global_field` など既存の同種ハンドラが priority デフォルト（100）を使用しているため同一パターンを採用する。
- `ON_TURN_END` (priority 140): `docs/spec/turn.md` の `Event.ON_TURN_END` テーブルに「トリックルームの継続・終了」「じゅうりょくの継続・終了」など全グローバルフィールドが priority 140 として明記されている。
- `ON_CHECK_TRAPPED` (priority 100): `docs/spec/turn.md` には掲載なし。既存の `にげられない` volatileハンドラ、`ありじごく` abilityハンドラ、`かげふみ` abilityハンドラいずれも priority 100（デフォルト）を使用しているため同一とする。

## 注意点・エッジケース

- **持続ターン数は `count=1`**: 仕様「使用した次のターンが終了するまで」に対応。`ON_TURN_END`（priority 140）でカウントが消費され解除される。使用したそのターンには既にロックがかかり、翌ターン終了時に解除される。
- **ゴーストタイプも拘束対象**: `check_trapped_not_ghost`（揮発状態や `かげふみ` 等で使われる関数）は使わず、`フェアリーロック_check_trapped` は `source` に無関係に `True` を返す。
- **すでにフェアリーロック状態のときの失敗**: `global_manager.activate` はすでに有効な場合に `False` を返す。`HandlerReturn(value=False)` が技の失敗として処理される（`ON_APPLY_MOVE` priority 100「すでに同じ」チェックはマネージャー側で対応される）。
- **`subject_spec="source:self"` と `ON_CHECK_TRAPPED` の発火**: `ON_CHECK_TRAPPED` は `EventContext(source=pokemon)` で発火される（`core/pokemon_query.py` 参照）。`FieldHandler` は両プレイヤーを `owner` とする `GlobalFieldManager` に登録されるため、両プレイヤーの `source` に対してハンドラが呼ばれる。
- **`きれいなぬけがら` との相互作用**: `きれいなぬけがら` は `ON_CHECK_TRAPPED` で `False` を返してトラップを無効化する。フェアリーロックは仕様上ゴーストタイプを含む全ポケモンを拘束するが、`きれいなぬけがら` の免除対象かどうかは仕様書に明記なし。実装上は priority 順で先に `True` を返した場合は `stop_event=True` でないため後続ハンドラが処理される。`フェアリーロック_check_trapped` が `stop_event=True` を返さなければ、`きれいなぬけがら_check_trapped` が `False` を返すことで上書きされる点に注意。必要に応じて `stop_event=True` の追加を検討する（要 review）。
- **クレッフィ専用技**: 仕様上クレッフィ専用だが、実装上は他のポケモンも使用できる MoveData として登録している（既存パターンに準じる）。
- **オウムがえしでコピー可能**: 仕様書に記載あり。技の `target="field"` 設定は別途確認が必要。既存エントリには `target` が未指定であり、現状維持でよいか確認すること。
