# 計画書: 攻撃技（先制技系）

## 仕様要約

- 先制技（優先度補正が付く攻撃技）は `MoveData.priority` フィールドで優先度を設定する
- 純粋な先制効果のみを持つ技はハンドラ登録不要
- 特殊な追加効果を持つ技のみハンドラが必要:
  - **マッハパンチ・バレットパンチ**: `punch` ラベルにより特性「てつのこぶし」の 1.2 倍補正が適用される（ラベル登録のみ）
  - **フェイント**: 命中時にまもる系揮発性状態（まもる・みきり・キングシールド等）を解除する
  - **くちばしキャノン**: 優先度 -3。技発動前に直接攻撃を受けると、その攻撃者をやけど状態にする

## Priority 根拠

### フェイント ON_HIT priority=100

`docs/spec/turn.md` の `Event.ON_HIT` セクションに「追加効果・固有効果」は明示されていないが、
既存の追加効果ハンドラ（やけど・まひ付与等）がすべて `MoveHandler` デフォルト（priority=100）で登録されている。
まもる解除も ON_HIT priority=100 で統一する。

### くちばしキャノン ON_TRY_MOVE_1 priority=100

`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` には
`| 100 | まもる系: 連続使用により失敗 |` など多くの項目が priority=100 で登録されている。
既存の `きあいパンチ_check_move` も `subject_spec="attacker:self"` で ON_TRY_MOVE_1 に priority=100 で登録されており、
同パターンを採用する。

### くちばしキャノン ON_DAMAGE_HIT priority=10

`docs/spec/turn.md` の `Event.ON_DAMAGE`（ダメージ付与後の最初の処理）セクションより：
```
| 10 | 防御側のくちばしキャノンでやけど |
```
ON_DAMAGE_HIT は turn.md の「Event.ON_DAMAGE」に相当する。priority=10 を採用する。

## 実装コード

### 1. `src/jpoke/handlers/move_attack.py` への追記（フェイント）

`_PROTECT_VOLATILES` を能力モジュールから借用するのではなく、
handler 内で直接 volatile_manager を操作する。
まもる系 volatile の一覧（`"まもる"`, `"みきり"`, `"トーチカ"`, `"キングシールド"`, `"スレッドトラップ"`, `"かえんのまもり"` 等）を解除する。

```python
_FAINT_REMOVE_VOLATILES: tuple[str, ...] = (
    "まもる", "みきり", "トーチカ", "キングシールド",
    "スレッドトラップ", "かえんのまもり",
)


def フェイント_remove_protect(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """フェイントの追加効果: 相手のまもる系揮発性状態を解除する。"""
    defender = ctx.defender
    for volatile in _FAINT_REMOVE_VOLATILES:
        if defender.has_volatile(volatile):
            battle.volatile_manager.remove(defender, volatile)
            battle.add_event_log(
                defender,
                LogCode.VOLATILE_REMOVED,
                payload={"volatile": volatile, "reason": "フェイント"}
            )
    return HandlerReturn(value=value)
```

### 2. `src/jpoke/handlers/move_attack.py` への追記（くちばしキャノン）

```python
def くちばしキャノン_apply_volatile(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """くちばしキャノン選択時に揮発性状態を付与する。

    技使用宣言時（ON_TRY_MOVE_1）に使用者に「くちばしキャノン」揮発性状態を付与する。
    この状態は直接攻撃を受けた際に攻撃者をやけどにし、ターン終了時に解除される。
    """
    battle.volatile_manager.apply(
        ctx.attacker, "くちばしキャノン", source=ctx.attacker
    )
    return HandlerReturn(value=value)
```

### 3. `src/jpoke/handlers/volatile.py` への追記（くちばしキャノン揮発性状態）

```python
def くちばしキャノン_burn_attacker(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """くちばしキャノン状態のポケモンが直接攻撃を受けたとき、攻撃者をやけど状態にする。"""
    if not battle.query.is_contact(ctx):
        return HandlerReturn(value=value)
    battle.ailment_manager.apply(
        ctx.attacker, "やけど", source=ctx.defender, ctx=ctx
    )
    return HandlerReturn(value=value)


def くちばしキャノン_remove_volatile(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """くちばしキャノン状態のターン終了時解除"""
    return remove_volatile(battle, ctx, value, volatile="くちばしキャノン")
```

## 注意点・エッジケース（review-test 向け）

### フェイントの「まもる系解除」タイミング

- フェイントは priority=+2 のため、通常は「まもる」（変化技・priority なし）より先に行動する
- しかし実際の発動順は「まもるが先に発動 → フェイントが命中してまもるを解除」であることに注意
- フェイントは `is_blocked_by_protect` フラグを持たない（まもるを貫通する）ため、
  まもる状態のポケモンにも命中し、ON_HIT でまもる状態を解除する
- `MoveData` に `labels=["unprotectable"]` を追加するか、または仕様どおりフェイントが
  まもるを貫通する専用処理が必要か確認が必要
  （現状 `is_blocked_by_protect` は `target=="foe" and not has_label("unprotectable")`）

### フェイントの「まもる解除ラベル」

フェイントが実際にまもる貫通かどうかは `labels=["unprotectable"]` の有無で決まる。
現状の MoveData に `unprotectable` ラベルがないため、まもる状態を貫通できない可能性がある。
仕様どおり「まもる状態を解除しつつダメージを与える」ためには
`labels=["unprotectable"]` を追加してまもる貫通を有効にする必要がある。

### くちばしキャノンの準備行動タイミング

- turn.md の `ON_BEFORE_MOVE` に「くちばしキャノンの準備行動」が記載されているが、
  現状の `ON_BEFORE_MOVE` は全体発火でコンテキストなし（`EventContext` なし）のため、
  個別ポケモンの技選択に基づく揮発性状態付与には使えない
- 代替として `ON_TRY_MOVE_1`（技使用前の個別ポケモン向けチェック）で揮発性状態を付与する
  （`きあいパンチ` が ON_TRY_MOVE_1 で失敗チェックを行っているパターンと同様）
- この方法では「技が選ばれたターンの最初の行動前」ではなく「技の実行開始時」に揮発性状態が付与される
  差異があるが、シングルバトル実装上は実質問題なし

### くちばしキャノンの揮発性状態解除タイミング

- `ON_TURN_END` でターン終了時に解除する
- 技発動後（攻撃完了後）に解除すべき場合は `ON_MOVE_END` の利用も検討する
- 仕様上「技が発動する前に」接触技を受けるとやけどになるため、技の発動が完了した後は
  状態を保持する必要はない → `ON_TURN_END` で解除が妥当

### くちばしキャノンの「技はそのまま受ける」

仕様：「技はそのまま受ける」——接触技を受けた際にやけど付与が行われるが、
攻撃そのものは通常通りダメージを受ける（技を無効化しない）。
`HandlerReturn(value=value)` でダメージ値は変更しない点が重要。

### くちばしキャノンのやけど条件

- `battle.query.is_contact(ctx)` で接触技判定が必要
- 接触技でない場合はやけどを付与しない
- 攻撃者がすでにやけど状態の場合は `ailment_manager.apply` 内部で重複付与を弾く

### マッハパンチ・バレットパンチの「てつのこぶし」補正

- 両技とも `punch` ラベルが登録済み
- 特性「てつのこぶし」は `punch` ラベルを持つ技の威力を 1.2 倍にする処理を
  能力側ハンドラで実装済み（または実装予定）
- 追加ハンドラは不要

### しんくうは の contact ラベル不要

- `_move_labels.md` の contact リストに `しんくうは` は含まれていない
- 特殊技のため直接攻撃ではない

### サイコフィールドによる先制技無効化

`docs/spec/turn.md` `Event.ON_TRY_MOVE_1` より：
```
| 100 | サイコフィールドによる無効化 |
```
優先度+1の先制技（アクアジェット・かげうち等）は、地面にいるポケモンへの使用時
サイコフィールドが発動中に無効化される。MoveData 側での処理は不要（フィールド側で処理）。
