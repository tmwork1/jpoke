# 計画書: パワフルエッジ

## 仕様要約

（`docs/spec/moves/パワフルエッジ.md` 参照）

* いわタイプ・物理、威力95、命中100、PP5、直接攻撃技。
* 相手の「まもる」状態を無視して攻撃する（まもる系状態は解除しない）。
* 「切る技」であり、`slash` フラグにより特性「きれあじ」で威力1.5倍。
* 急所ランク補正はなし（Wiki上「急所: ○」のみで「急所に当たりやすい」の記載がなく、
  `ストーンエッジ` の備考によりSVで急所補正を持ついわ技は `ツタこんぼう` のみと確認済み）。
* 追加効果なし。命中判定は通常通り。

## 既存実装の状況

`data/moves/move_ha.py` の `パワフルエッジ` エントリは既に実装済みで、
`flags={"contact", "unprotectable", "slash"}` を設定し、`critical_rank` は指定なし
（デフォルト0）、追加効果ハンドラは持たない（`handlers={}`）。
レビューの結果、実装・登録内容に誤りはなかった。特に急所ランク補正について
一次情報を再確認したが、パワフルエッジには急所ランク補正がなく、`critical_rank` を
指定していない現在の実装が正しいことを確認した。

`unprotectable` フラグは `model/move.py` の `Move.is_blocked_by_protect` プロパティで
参照されており、`target == "foe"` かつ `has_flag("unprotectable")` でない場合のみ
「まもる」でブロックされる判定になる。`handlers/volatile.py` の `_check_protect_success` /
`_run_protect`（まもる・かえんのまもり・トーチカ・ニードルガード・ファストガード等、
`_run_protect` を共通骨格とする全ての防御系変化技が対象）と
`handlers/ability.py`（みきり等の特性起因の防御判定）の双方がこのプロパティを参照して
判定するため、専用の追加実装は不要。

`_run_protect` は防御成立時のみ `stats_change_on_contact` / `ailment_on_contact` /
`chip_on_contact` を適用する構造になっており、`unprotectable` フラグにより
`_check_protect_success` が `False` を返す本技では防御自体が不成立となるため、
かえんのまもり等の接触時反撃効果も自動的に発生しない。仕様書の「防御系変化技の
反撃効果を受けない」という記述と実装が一致していることを確認した。

`_run_protect` はまもる状態そのものを解除する処理を含まないため
（防御不成立で早期returnするのみ）、フェイント専用の `ha.フェイント_remove_protect`
のような「まもる状態解除」ハンドラは本技には不要かつ実装されていない。仕様書の
「まもる系状態は解除しない」という記述とも整合する。

`slash` フラグは特性「きれあじ」のダメージ計算ハンドラ（`handlers/ability.py` の
きれあじ関連処理）で参照され、`slash` フラグを持つ技の威力を1.5倍にする。
本技は「切るわざ」カテゴリのWiki記載と一致するため `slash` フラグを持たせている。

## イベント選定の根拠

* 追加効果を持たないため、専用ハンドラの新規登録は不要。`unprotectable` / `slash`
  フラグのみで仕様を満たす。

## ハンドラ構成

| 対象 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| （なし） | - | - | - | `flags={"contact", "unprotectable", "slash"}` のみで仕様を満たすため、専用ハンドラは不要 |

```python
"パワフルエッジ": MoveData(
    type="いわ",
    category="physical",
    pp=5,
    power=95,
    accuracy=100,
    flags={"contact", "unprotectable", "slash"},
    handlers={},  # 追加効果なし
),
```

## リーサル計算

`docs/progress/move.md` の該当行は「リーサル実装: n/a」「リーサルテスト: n/a」のため対象外
（追加効果のない通常の物理攻撃技であり、専用のリーサル計算ハンドラは不要）。

## テスト

* きれあじ特性で威力が1.5倍になること
  （`test_パワフルエッジ_きれあじで威力1_5倍`）。
* まもる状態の相手にもダメージを与えること
  （`test_パワフルエッジ_まもる中の相手にダメージが入る`）。
* まもる状態を無視して命中してもまもる揮発状態自体は解除されないこと
  （`test_パワフルエッジ_まもる状態が解除されない`）。

## 注意点・エッジケース

* 「まもる」を無視するが状態自体は解除しないため、同ターン中に別の相手ポケモンが
  攻撃対象になるような効果（シングルバトル対象のため実質発生しない）や、次ターンの
  まもる連続成功率ペナルティには影響しない。専用実装・専用テストは不要。
* かえんのまもり・スレッドトラップ・トーチカ・ニードルガードの反撃効果を受けないことは
  `_run_protect` の共通構造により保証されるため、パワフルエッジ専用のテストは追加しない。
* 急所ランク補正は無いため、`critical_rank` を明示的に指定しない（デフォルト0のまま）。
  `ストーンエッジ` のように `critical_rank=1` を付与しないよう注意する。
