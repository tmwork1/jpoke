# 実装計画書: どくのいと

## 仕様要約

- タイプ: どく / カテゴリ: 変化 / PP: 20 / 命中: 100 / 対象: 1 体選択
- 相手を「どく」状態にし、さらに「すばやさ」ランクを 1 段階下げる
- どく・はがねタイプには無効（どく状態付与の前提として `ON_TRY_MOVE_2` で判定される）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手の「すばやさ」を 1 段階下げる |
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手を「どく」状態にする |

注: `MoveData.handlers` の型は `dict[Event, Handler | list[Handler]]` であるため、同一イベントに複数ハンドラを渡す場合はリストで指定する。

## Priority 根拠

`docs/spec/turn.md` には明示記載なし。`タールショット` が S-1 + タールショット状態付与という複合効果を `ON_STATUS_HIT` に実装しているパターンを参照。どくのいとも同じく複合ハンドラで実装する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「ど」行 — `どくのこな_apply_ailment_to_defender` の後、`どくびし_set_field` の前に挿入する（「どくのいと」は「どくのこな」より前の五十音順）。

```python
def どくのいと_apply_ailment_to_defender(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """どくのいとの効果（毒）: 相手をどく状態にする。"""
    return apply_ailment_to_defender(battle, ctx, value, ailment="どく")


def どくのいと_modify_defender_stats(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """どくのいとの効果（速度低下）: 相手のすばやさを 1 段階下げる。"""
    return modify_defender_stats(battle, ctx, value, stats={"S": -1})
```

### `src/jpoke/data/move.py`

既存の `"どくのいと"` エントリに handlers を追加する。`MoveData.handlers` は `dict[Event, Handler | list[Handler]]` 型なので、リストで複数ハンドラを渡す。

```python
"どくのいと": MoveData(
    type="どく",
    category="変化",
    pp=20,
    accuracy=100,
    handlers={
        Event.ON_STATUS_HIT: [
            h.MoveHandler(
                hs.どくのいと_modify_defender_stats,
            ),
            h.MoveHandler(
                hs.どくのいと_apply_ailment_to_defender,
            ),
        ],
    }
),
```

## 注意点・エッジケース

- `MoveData.handlers` が `dict[Event, Handler | list[Handler]]` 型のため、同一イベントに複数ハンドラを渡す場合はリスト形式を使う
- タールショットが同イベントに 2 つのハンドラを分けて持つパターン（`タールショット_modify_defender_stats` と `タールショット_apply_volatile_to_defender`）を参照すること。ただしタールショットは 2 エントリとも別 key なので要確認
- S 下げの失敗（すでに -6）とどく付与の失敗（すでにどく・はがね・タイプ）は個別に `modify_stats`・`ailment_manager.apply` の内部で処理されるため、ハンドラ側での追加ガードは不要
- どく・はがねタイプへの無効化は `ON_TRY_MOVE_2` の「どく・はがねタイプ: どく無効」（priority 130）で処理されるため、ハンドラ側では不要
