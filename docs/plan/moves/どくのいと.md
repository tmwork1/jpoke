# 実装計画書: どくのいと

## 仕様要約

- タイプ: どく / カテゴリ: 変化 / PP: 20 / 命中: 100 / 対象: 1 体選択
- 相手の「すばやさ」ランクを 2 段階下げ（Championsでの変更点。SVまでは1段階）、相手を「どく」状態にする
- 2つの効果は互いに独立している。すばやさを下げる効果の後に、どく状態にする効果が発動する
- どく・はがねタイプには「どく」付与のみ無効（すばやさ低下は通常通り発動する）

詳細は `docs/spec/moves/どくのいと.md` を参照。

## 現状の実装状況

| 項目 | 状態 |
|------|------|
| `data/moves/move_ta.py` エントリ | 登録済み（type=どく, category=status, pp=20, accuracy=100） |
| `handlers/move_status.py` 関数 | 実装済み（`どくのいと_apply`） |

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（`MoveHandler` デフォルト） | `attacker:self`（`MoveHandler` デフォルト） | すばやさを2段階下げ、その後どく状態にする単一の複合ハンドラ |

`ON_STATUS_HIT` は `move_executor.py` の `_execute_status_hit` から `AttackContext` を伴って発火されるため、
ハンドラは `ctx: AttackContext` を受け取り `ctx.defender` を直接参照する（`EventContext` ではない）。
同種のパターンは `タールショット_apply`（`handlers/move_status.py`）を参照。

## Priority 根拠

`docs/spec/turn.md` には `ON_STATUS_HIT` の記載なし。既存の状態変化技ハンドラはすべて
`MoveHandler` のデフォルト priority=100 を使用しており、どくのいとも同じくデフォルトを踏襲する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「ど」行 — `どくどく_apply_ailment_to_defender` の後、`どくのこな_apply_ailment_to_defender` の前。

```python
def どくのいと_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """どくのいとの効果: 相手のすばやさを2段階下げ、どく状態にする（Championsで1段階→2段階に変更）。

    2つの効果は互いに独立している。すばやさを下げる効果の後にどく状態にする効果が発動する。
    どちらか一方でも成功すれば技自体は成功扱いとなり、両方失敗した場合のみ技が失敗する。
    """
    assert ctx.defender is not None
    stat_changed = bool(battle.modify_stats(ctx.defender, {"spe": -2}, source=ctx.attacker))
    ailment_applied = battle.ailment_manager.apply(ctx.defender, "どく", source=ctx.attacker, ctx=ctx)
    return HandlerReturn(value=stat_changed or ailment_applied)
```

### `src/jpoke/data/moves/move_ta.py`

```python
"どくのいと": MoveData(

    category="status",
    pp=20,
    accuracy=100,
    handlers={

        ),

```

## 注意点・エッジケース

- すばやさ低下量はChampionsで2段階（SVまでは1段階）。`docs/champions/changes_from_sv.md` に明記あり
- すばやさを下げる効果とどく状態にする効果は独立しており、順序はすばやさ低下が先、どく付与が後
- 技全体の成否（`ON_STATUS_HIT` の戻り値、`MOVE_FAILED` ログの有無）は、すばやさ低下・どく付与の
  いずれかが成功していれば成功扱いとする（両方失敗した場合のみ失敗）
- どく・はがねタイプへの「どく」付与無効化は `ailment_manager.apply` 内のタイプ判定で処理されるため、
  ハンドラ側での追加ガードは不要
- みがわり状態の相手には技全体が無効（通常の状態変化技と同様の挙動。特性『すりぬけ』所持者は無視できる）
