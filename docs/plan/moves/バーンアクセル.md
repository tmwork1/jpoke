# バーンアクセル 実装計画書

## 仕様要約（docs/spec/moves/バーンアクセル.md 参照）

* タイプ: ほのお / 分類: 物理 / 威力: 80 / 命中: 100 / PP: 12（Champions基準、本家PP10からの変換）
* 対象: 相手1体 / 優先度: 0 / **接触なし**（物理技だが直接攻撃ではない）
* 効果: 追加効果として30%の確率で相手を「やけど」状態にする。
* この技に対する アンコール・まねっこ は失敗する（`non_encore` / `non_copycat` フラグで表現）。
  ものまね・さいはいは本プロジェクト未実装のため対応フラグなし。

## 現状の実装状況（レビュー前）

`src/jpoke/data/moves/move_ha.py` に既に登録済みだったが、以下の誤りがあった。

```python
"バーンアクセル": MoveData(
    type="ほのお",
    category="physical",
    pp=15,                                            # 誤り: Championsの4段階(8/12/16/20)に含まれない
    power=80,
    accuracy=100,
    flags={"contact", "non_copycat", "secondary_effect"},  # 誤り: "contact" は不要、"non_encore" が不足
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.バーンアクセル_apply_burn_to_defender,
        )
    }
),
```

やけど付与ハンドラ自体（`ha.バーンアクセル_apply_burn_to_defender` = `apply_ailment_to_defender(..., ailment="やけど", chance=0.3)`）は
確率30%で仕様と一致しており、修正不要。

## 修正後の実装

```python
"バーンアクセル": MoveData(
    type="ほのお",
    category="physical",
    pp=12,
    power=80,
    accuracy=100,
    flags={"non_copycat", "non_encore", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.バーンアクセル_apply_burn_to_defender,
        )
    }
),
```

## ハンドラ構成

| Event | 関数 | subject_spec | priority | 役割 |
|---|---|---|---|---|
| `Event.ON_DAMAGE_HIT` | `バーンアクセル_apply_burn_to_defender` | `attacker:self`（デフォルト） | 100（デフォルト） | 30%の確率で相手をやけど状態にする |

`ON_DAMAGE_HIT` は既存の同種やけど付与技（ひのこ・かえんほうしゃ・だいもんじ 等）と
同一イベントであり、`docs/spec/turn.md` に個別の priority 記載がないダメージ命中後の
追加効果ハンドラ群と同様、`MoveHandler` のデフォルト priority（100）をそのまま使用する。
これは既存の burn 付与ハンドラすべてが priority 未指定（デフォルト値）である前例を踏襲したもの。

## Priority 根拠

* `Event.ON_DAMAGE_HIT` は `docs/spec/turn.md` に個別の priority 記載がないイベントであり、
  CLAUDE.md の指針に従い既存の同種ハンドラ（`ひのこ_apply_burn_to_defender` 等、
  すべて priority 未指定＝デフォルト100）を参照して同じ扱いとする。

## 実装箇所

### handlers/move_attack.py

修正不要（`バーンアクセル_apply_burn_to_defender` は既存実装のまま、`apply_ailment_to_defender`
汎用ヘルパーを呼び出すのみ）。

### data/moves/move_ha.py

## 注意点・エッジケース（review-test 向け）

1. `flags` に `"contact"` を含めない → さめはだ・てつのトゲ等の接触特性・アイテム
   （ゆびをふる等ではなく、接触判定系）のトリガー対象外になることを確認する
   （`tests/abilities/test_ability_sa.py` の `test_さめはだ_接触ダメージ` に類する
   パラメータ追加でダメージ増加が発生しないことを確認）。
2. `flags` に `"non_encore"` を追加 → アンコールで固定できないことを確認する
   （`アンコール_can_apply` は `move.has_flag("non_encore")` を見て失敗させる）。
3. `flags` の `"non_copycat"` は既存実装済みで、まねっこ失敗テストも既に存在する
   （`tests/moves_status/test_move_ma.py` の
   `test_まねっこ_スターモービル専用技はコピーできない`）→ 変更不要。
4. PP を `12` に修正 → PP消費・わるあがき遷移等の一般テストは既存の共通ロジックで
   カバーされるため専用テスト不要（他のPP修正済み技と同様の扱い）。
5. `secondary_effect` フラグにより、ちからずく・てんのめぐみ等の特性判定対象になる
   （既存の共通ロジックで処理されるため専用ハンドラ不要）。
6. やけど付与は物理アタッカーへの反動として意味を持つ（やけど状態は物理攻撃力を半減させる）。
   本技自体は「相手をやけど状態にする」効果であり、自分の状態異常には影響しない。
