# 攻撃技_やけど系 実装計画書

## 仕様要約

`docs/spec/moves/_move_secondary_1.md` の「やけど」列（13列目）が非0の技が対象。
命中時に確率でやけど状態を相手に付与する。

- イベント: `ON_HIT`
- priority: 100（`docs/spec/turn.md` の `Event.ON_DAMAGE` セクション「追加効果（特殊なもの除く）」priority=10 に記載されているが、`ON_HIT` は別イベントで priority=100 が既存の類似ハンドラ（`でんじほう`、`ほうでん` 等）と一致）
- ハンドラヘルパー: `apply_ailment_to_defender(battle, ctx, value, ailment="やけど", chance=<確率>)`
- subject_spec: `"attacker:self"`（AttackContext のデフォルト）

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_HIT` セクション（行474-490）には「追加効果」の記載なし。
ただし `Event.ON_DAMAGE` セクションの「追加効果（特殊なもの除く）」は priority=10 であり、`ON_HIT` は別イベントである。

既存の類似ハンドラを根拠とする:
- `でんじほう_apply_ailment_to_defender` → `ON_HIT`, priority デフォルト（100）
- `ほうでん_apply_ailment_to_defender` → `ON_HIT`, priority デフォルト（100）
- `ほっぺすりすり_apply_ailment_to_defender` → `ON_HIT`, priority デフォルト（100）

`MoveHandler` の priority デフォルト値は 100。やけど系も同じく priority=100 を採用する。

## 注意事項・エッジケース（review-test 向け）

### やけど無効条件
- `docs/spec/turn.md` の `Event.ON_TRY_MOVE_2` セクション（行345）に「ほのおタイプ: やけど無効」がある。
- 同 priority=140 に「みずのベール等: やけど無効」がある。
- これらは ailment_manager 側または ON_TRY_MOVE_2 ハンドラ側で処理されるため、各技ハンドラ内でのガードは不要。

### すでにやけど状態の場合
- 既に状態異常を持つポケモンに重ねて付与することはできない。`ailment_manager.apply` 内部で処理されるためハンドラ側でのガードは不要。

### MoveData 未登録技の基本パラメータ（要確認）
以下は第九世代公式データに基づくが、実装前に実際のゲームデータと照合すること:
- **かえんだん**: ほのお/特殊, pp=15, power=75, accuracy=100
- **バーンアクセル**: ほのお/物理, pp=15, power=80, accuracy=100, contact

### せいなるほのお
- 確率50%は全やけど技中最高。

### れんごく
- 確率100%（chance=1）なので `apply_ailment_to_defender` の `chance` 引数は省略可だが、明示的に記述しない（デフォルト値1を利用）。

### フレアドライブ
- recoil ラベルが既についているが、反動ダメージの計算は別ハンドラ（共通）が担当するため競合なし。

### シャカシャカほう
- heal ラベルが既についているが、回復効果は別途実装が必要。今回はやけど効果の追加のみ。

### ほのおのキバ
- 仕様書でキバ系技（かみなりのキバ・こおりのキバ）はひるみも付与する（仕様書行169, 190）。
  ほのおのキバはひるみ列が0であることを確認済み（行145）。やけどのみ対応。
