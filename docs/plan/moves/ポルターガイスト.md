# ポルターガイスト 実装計画書

## 仕様要約（docs/spec/moves/ポルターガイスト.md 参照）

* タイプ: ゴースト / 分類: 物理 / 威力: 110 / 命中: 90 / PP: 8（champions基準）
* 対象: 相手1体 / 直接攻撃ではない
* 失敗条件: 相手が持ち物を持っていない場合
* マジックルーム状態・相手の特性「ぶきよう」で持ち物が無効化されていても、
  「持っている」こと自体は変わらないため成功する
  （`Pokemon.has_item()` の既定 `consider_enabled=False` は `base_name` を参照するため、
  アイテムの有効/無効に関わらず所持判定できる）

## 現状の実装状況

`src/jpoke/data/moves/move_ha.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。

```python
"ポルターガイスト": MoveData(
    type="ゴースト",
    category="physical",
    pp=8,
    power=110,
    accuracy=90,
    handlers={
        Event.ON_BEFORE_APPLY_MOVE: h.MoveHandler(
            ha.ポルターガイスト_check_item,
            priority=130,
        ),
    },
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_BEFORE_APPLY_MOVE` | 130 | attacker:self（`MoveHandler`既定値） | 相手が持ち物を持っていない場合、技を失敗させる |

## Priority 根拠

* `ON_BEFORE_APPLY_MOVE priority=130`: docs/spec/turn.md に本イベントの優先度テーブルは
  掲載されていないが、既存の「対象の状態を見て技の成否を判定する」失敗チェック
  （`あくび_can_apply` `ほろびのうた_can_apply` 等）が `ON_BEFORE_APPLY_MOVE priority=130` を
  採用しているため、同じパターンに倣う。

## 実装箇所

### handlers/move_attack.py（五十音順: ほ行、ポイズンテールの直後）

```python
def ポルターガイスト_check_item(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ポルターガイストの使用条件チェック: 相手がアイテムを持っていない場合は失敗する。"""
    if not ctx.defender.has_item():
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="ポルターガイスト_アイテムなし")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

`ctx.defender.has_item()` は既定で `consider_enabled=False`（`base_name` を参照）のため、
マジックルームや相手の特性「ぶきよう」でアイテムが無効化されていても、
持ち物自体を保持していれば成功する。逆に「なげつける」のようにアイテムの有効性を
要求する技は `has_item(consider_enabled=True)` を使う（本技とは異なる挙動）。

## テスト方針

`tests/moves_attack/` に以下を追加する。

* 相手がアイテムを持っているとき技が成功しダメージを与える（既存）
* 相手がアイテムを持っていないとき技が失敗しダメージを与えない（既存）
* マジックルーム状態でアイテムが無効化されていても、相手が持ち物を持っていれば成功する
* 相手の特性が「ぶきよう」でアイテムが無効化されていても、相手が持ち物を持っていれば成功する

`tests/items/` の `test_カシブのみ_ポルターガイストは成功しダメージ半減` で
カシブのみとの相互作用（技は成功しつつダメージ半減）を確認済み。
