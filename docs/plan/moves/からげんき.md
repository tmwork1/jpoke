# からげんき 実装計画書

参照: docs/spec/moves/からげんき.md

## 現状

* `src/jpoke/data/move.py` の `"からげんき"` エントリは以下の通り実装済み。
  - type="ノーマル", category="physical", pp=20, power=70, accuracy=100
  - flags={"contact"}
  - `Event.ON_CALC_POWER_MODIFIER`: `からげんき_double_power_when_ailment`（`MoveHandler` の
    デフォルト `subject_spec="attacker:self"` を使用）
  - `Event.ON_CALC_BURN_MODIFIER`: `からげんき_ignore_burn_modifier`（`subject_spec="attacker:self"`）
* `からげんき_ignore_burn_modifier` は `やけど`（`src/jpoke/data/ailment.py` の
  `Event.ON_CALC_BURN_MODIFIER: h.AilmentHandler(h.やけど_modifier, ...)`）が返す 2048（0.5倍）を
  4096（1.0倍）で上書きする実装であり、仕様（第六世代以降やけどのダメージ減少を無視）と一致する。
  実際に `test_からげんき_やけどの攻撃半減を無効化する` で優先度衝突なく期待通り動作することを
  確認済み（両ハンドラとも priority=100 だが、ムーブ側ハンドラの実行が最終的な burn_modifier
  として採用される）。

## 発見した不具合と修正

* `からげんき_double_power_when_ailment` は修正前、`ctx.attacker.ailment.is_active` を条件に
  していた。しかし `src/jpoke/model/ailment.py` の `is_active` は `name != ""` を返すだけであり、
  `AilmentName`（`src/jpoke/types/ailment.py`）に含まれる `ねむり` `こおり` `ゆめうつつ` でも
  true になってしまう。
* 一次情報（docs/spec/moves/からげんき.md 参照）では、からげんきの威力上昇対象は
  `どく` `もうどく` `まひ` `やけど` の4種類のみであり、`ねむり`（ねごと経由の使用を含む）や
  `ゆめうつつ`（ぜったいねむり特性による付与）では威力が上がらないと明記されている。
* このため `ctx.attacker.ailment.name in ("どく", "もうどく", "まひ", "やけど")` の判定に修正する。

```python
def からげんき_double_power_when_ailment(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """からげんき: 使用者が『どく』『もうどく』『まひ』『やけど』状態のとき威力が2倍になる。

    『ねむり』『こおり』『ゆめうつつ』は対象外（ねごと経由の使用や、ぜったいねむりによる
    ゆめうつつ状態でも威力は上がらない）。
    """
    if ctx.attacker.ailment.name in ("どく", "もうどく", "まひ", "やけど"):
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)
```

## リーサル計算

* `docs/progress/move.md` の「リーサル実装」「リーサルテスト」列は `n/a`。対象外
  （威力補正はダメージ計算内で完結し、専用のリーサルハンドラを要しない）。

## テスト

* `tests/moves_attack/test_move_ka.py` に、使用者が『ねむり』状態（ねごと経由でからげんきを
  発動させたケース）と『ゆめうつつ』状態（ぜったいねむり相当）では威力が2倍にならないことを
  確認するテストを追加する。
