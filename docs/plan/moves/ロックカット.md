# 実装計画: ロックカット

## 仕様サマリ

- タイプ: いわ / 分類: 変化 / 威力: - / 命中率: - / PP: 20 / 対象: 自分
- 効果: 自分のすばやさランクを2段階上げる
- まもる: 防がれない（自分対象） / マジックコート: 反射不可（自分対象） / 直接攻撃: いいえ
- よこどり: 対象になる（○）

出典: docs/spec/moves/ロックカット.md

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自分のすばやさを2段階上げる |

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`わるだくみ_modify_attacker_stats`, `つるぎのまい_modify_attacker_stats` 等）が
すべて priority=100（デフォルト）で登録されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py（実装済み）

```python
def ロックカット_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"spe": 2})
```

### data/moves/move_ra.py の更新

`target="self"` を指定する（`target` を省略するとデフォルトの `"foe"` になり、
`is_blocked_by_protect` / `is_reflectable` がまもる・マジックコートに誤反応してしまうため必須。
`コットンガード`・`つるぎのまい` 等の既存レビューと同様のパターン）。

```python
"ロックカット": MoveData(
    type="いわ",
    category="status",
    pp=20,
    target="self",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ロックカット_modify_attacker_stats,
        )
    }
),
```

## 注意点・エッジケース

- `target="self"` を指定しないと、相手の「まもる」で誤って防がれてしまう
  （実測で確認済み: `target` 未指定だと `まもる` 使用中の相手に対してすばやさが上がらない）
- ランクがすでに最大（+6）の場合の失敗は `battle.modify_stats()` 側で処理されるため、
  ハンドラ側にガード不要
- 対象は自分のため命中率なし。テストは命中率固定不要

## リーサル計算

対象外（`docs/progress/move.md` の該当行は `n/a`）。ダメージを与えない自己バフ技のため
リーサル計算ハンドラは不要。
