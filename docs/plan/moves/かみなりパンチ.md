# かみなりパンチ 実装計画書

## 仕様要約

`docs/spec/moves/かみなりパンチ.md` を参照。
でんきタイプ・物理・威力75・命中100・PP15の直接攻撃技。
追加効果として10%の確率で相手を「まひ」状態にする。パンチ技のため「てつのこぶし」で威力1.2倍。

本技は `docs/plan/moves/攻撃技_まひ系.md`（まひ付与技の一括実装計画）に含まれている。
以下はその内容のうち本技に関する部分の抜粋。

## ハンドラ構成

`Event.ON_HIT` に `MoveHandler` デフォルト設定（`subject_spec="attacker:self"`, `priority=100`）で登録する。
`apply_ailment_to_defender` は `handlers/move.py` の既存共通関数を使用する。

| 技 | event | priority | subject_spec | 役割 |
|---|---|---|---|---|
| かみなりパンチ | ON_HIT | 100 | attacker:self | まひ 10% |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_HIT` セクションに追加効果（状態異常付与）は明示されていないが、
既実装の `でんじほう`・`ほっぺすりすり` の登録コードと同じ priority=100（`h.MoveHandler` のデフォルト）を採用する。

## 実装箇所

### `src/jpoke/handlers/move_attack.py`

```python
def かみなりパンチ_apply_paralysis_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="まひ", chance=0.1)
```

### `src/jpoke/data/move.py`

```python
"かみなりパンチ": MoveData(
    type="でんき",
    category="physical",
    pp=15,
    power=75,
    accuracy=100,
    flags={"contact", "punch", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.かみなりパンチ_apply_paralysis_to_defender,
        )
    }
),
```

※ 実装コードでは登録イベントが `Event.ON_DAMAGE_HIT` になっている。
`Event.ON_HIT` と同様に命中後の追加効果付与タイミングとして機能する既存の仕組みであり、
他の直接攻撃・追加効果技（`かみなりのキバ` 等）と揃えた登録のため妥当と判断する。

## リーサル計算

- 対象外（`n/a`）。まひは行動阻害・素早さ低下効果であり、ダメージ量には影響しないため専用のリーサルハンドラは不要。

## エッジケース

- でんきタイプの相手はまひ無効（`ailment_manager` 側の共通処理で判定されるため、本技のハンドラに追加ガードは不要）。
- 特性「てつのこぶし」による威力1.2倍は `flags={"punch"}` を通じて特性側ハンドラが処理するため、本技側の実装は不要。
