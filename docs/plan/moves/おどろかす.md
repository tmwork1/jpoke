# 計画書: おどろかす

## 仕様要約（docs/spec/moves/おどろかす.md 参照）

- ゴーストタイプ / 物理 / 威力30 / 命中100 / PP15 / 対象: 1体選択 / 優先度0 / 接触あり
- 追加効果として、30%の確率で対象をひるませる。

## 既存実装との整合確認

既存実装は仕様と一致しており、修正不要。

### handlers/move_attack.py: `おどろかす_apply_flinch`（既存）

```python
def おどろかす_apply_flinch(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_defender(battle, ctx, value, volatile="ひるみ", chance=0.3)
```

- `apply_volatile_to_defender` ヘルパーは内部で `battle.resolve_secondary_chance` を経由するため、
  「追加効果」としてちからずく・りんぷん・おんみつマント等の影響を正しく受ける
  （仕様書の「追加効果: あり」と一致）。
- 他の同系統技（かみつく・いわなだれ・エアスラッシュ・あくのはどう等）と同一パターンで実装されている。

- イベント: `Event.ON_DAMAGE_HIT`。同種のひるみ追加効果技（かみつく・いわなだれ等）が
  すべて `Event.ON_DAMAGE_HIT` を使用しており、`docs/spec/turn.md` に本イベントの記載はないが
  （ダメージ計算内部処理のため未掲載）、既存の同種ハンドラとの整合を優先し、priority省略時の
  デフォルトのままとする。
- subject_spec: `MoveHandler` デフォルト（defender側への効果、既存パターンと同一）。

### data/move.py（既存）

```python
    "おどろかす": MoveData(
        type="ゴースト",
        category="physical",
        pp=15,
        power=30,
        accuracy=100,
        flags={"contact", "secondary_effect"},
        handlers={
            Event.ON_DAMAGE_HIT: h.MoveHandler(
                ha.おどろかす_apply_flinch,
            )
        }
    ),
```

- `flags={"contact", "secondary_effect"}` は「直接攻撃」「追加効果あり」の仕様と一致。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `critical_rank` 省略時のデフォルトは `0`（通常の急所判定）→ 仕様と一致。

## リーサル計算

対象外（`docs/progress/move.md` の「おどろかす」行はリーサル実装/リーサルテストともに n/a）。
ひるみは相手の行動を1ターン封じるだけの効果であり、確定数計算（HP分布）には影響しないため。

## 注意点・エッジケース

- 第三世代のみ存在した「ちいさくなる状態の相手へのダメージ2倍」仕様は過去世代限定のバグ的挙動であり、
  本プロジェクト対象のポケモンチャンピオンズ（現行世代ルール）には存在しないため実装しない。
