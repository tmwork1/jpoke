# 実装計画書: アフロブレイク

参照: docs/spec/moves/アフロブレイク.md

## 概要

物理技。相手に与えたダメージの1/4を自分も受ける反動技。追加効果なし。

## 現状の実装状況（レビュー前）

`src/jpoke/data/moves/move_a.py` に登録済み。タイプ・分類・威力・命中率・反動比率・ハンドラ登録は
仕様書と一致していたが、PPのみ誤り（`pp=10`）だった。本技は `src/jpoke/data/ps-champ-ja/moves.json` に
収録されていないため PP 統一（8/12/16/20）の対象外であり、本家の値である15を使うのが正しい。

```python
"アフロブレイク": MoveData(
    type="ノーマル",
    category="physical",
    pp=10,  # 誤り。正しくは15（src/jpoke/data/ps-champ-ja/moves.json 未収録のため本家PPをそのまま使用）
    power=120,
    accuracy=100,
    flags={"contact", "recoil"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.アフロブレイク_recoil,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---------|----------|--------------|------|
| ON_HIT | 100（`MoveHandler` デフォルト） | attacker:self | 与ダメの1/4を反動として攻撃者に与える |

Priority 根拠: `docs/spec/turn.md` の `Event.ON_DAMAGE`（実装上の `Event.ON_DAMAGE_HIT` に相当）に
「100 反動技による反動ダメージ」の記載があるが、`いのちのたま`（`src/jpoke/data/item.py`）と同様、
`ON_DAMAGE_HIT` は実ダメージが0以下のとき発火しないため、常に発火する `Event.ON_HIT` を採用し、
priority の値のみ turn.md の 100 に合わせる（`MoveHandler` のデフォルト priority がちょうど100のため
明示指定は不要）。同系統の反動技（とっしん・ワイルドボルト・じごくぐるま）と同じ実装パターン。

## handlers/move_attack.py

共通ヘルパー `_recoil(battle, ctx, value, ratio)` を利用する（既存実装、修正不要）。

```python
def アフロブレイク_recoil(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return _recoil(battle, ctx, value, 1/4)
```

## data/move.py

`"アフロブレイク"` エントリの `pp` を `10` から `15` に修正する（他のフィールドは変更不要）。

```python
"アフロブレイク": MoveData(
```

## リーサル計算

対象外（`docs/progress/move.md` の「リーサル実装」列は n/a）。反動技は攻撃者自身のHPを減らすのみで、
防御側の HP 分布 (`StateDist`) には影響しないため、リーサル計算ハンドラの実装対象にはならない。

## 注意点・エッジケース

* `reason="recoil"` を指定することで、いしあたま特性が実装された際に自動的に無効化される
* みがわりに命中した場合も `ON_HIT` は発動するため反動ダメージは発生する。`_recoil` は `value`
  （実際の被ダメージ、みがわり有効時は0）が0の場合 `ctx.substitute_damage`（みがわりへの与ダメージ）
  にフォールバックして反動量を算出する
* テストは `tests/moves_attack/test_move_a.py`（個別テスト）と
  `tests/moves_attack/test_move__group.py`（recoil技グループの共通パラメタライズテスト）の
  両方でカバー済み
