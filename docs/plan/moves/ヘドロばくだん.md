# ヘドロばくだん 実装計画書

## 仕様要約（docs/spec/moves/ヘドロばくだん.md 参照）

* タイプ: どく / 分類: 特殊 / 威力: 90 / 命中: 100 / PP: 12
* 対象: 相手1体 / 優先度: 0 / 接触なし / 弾の技（bullet）
* 効果: 追加効果として30%の確率で相手をどく状態にする。

## 現状の実装状況

`src/jpoke/data/moves/move_ha.py` に既に登録済み。

```python
"ヘドロばくだん": MoveData(
    type="どく",
    category="special",
    pp=12,
    power=90,
    accuracy=100,
    flags={"bullet", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ヘドロばくだん_apply_poison_to_defender,
        )
    }
),
```

`src/jpoke/handlers/move_attack.py` にハンドラ実装済み。

```python
def ヘドロばくだん_apply_poison_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="どく", chance=0.3)
```

レビュー時点で PP が Gen9本家基準の 10 のままだったため、`src/jpoke/data/ps-champ-ja/moves.json` の値
（12）に修正した。それ以外は仕様書と一致しており、修正不要。

## ハンドラ構成

| event | priority | subject_spec | 役割 |
|---|---|---|---|
| ON_DAMAGE_HIT | 100（デフォルト） | attacker:self（デフォルト） | 30%の確率で相手をどく状態にする |

`apply_ailment_to_defender` は `handlers/move.py` の既存共通関数を使用する
（`battle.resolve_secondary_chance` で確率解決後 `battle.ailment_manager.apply` を呼ぶ）。

## Priority 根拠

どく付与は `Event.ON_DAMAGE_HIT`。`docs/spec/turn.md` の `Event.ON_DAMAGE` には
「追加効果（特殊なもの除く）」が priority=10 と掲載されているが、10まんボルト・スパーク・ずつき・
スモッグ等、既存の同種「追加効果付与」ハンドラがすべて `ON_DAMAGE_HIT` + デフォルト priority=100 で
登録されているため、同一パターンを踏襲する。

## 注意点・エッジケース（review-test 向け）

### どくの無効条件
- どく・はがねタイプはどく無効。`apply_ailment_to_defender` → `battle.ailment_manager.apply` 内部で
  判定されるため、本技側ハンドラに追加ガードは不要。

### ぼうだん特性による無効化
- 弾の技（`bullet` フラグ）のため、ぼうだん特性を持つ相手には技自体が無効化される。
  `handlers/ability.py` の `ぼうだん_block_bullet` で共通処理されるため、本技側の実装への影響はない。

### PPの扱い
- `src/jpoke/data/ps-champ-ja/moves.json` にヘドロばくだんのエントリがあり、PP=12。Gen9本家基準のPP10は
  championsのPP再編で12に変更されているため、championsの値（12）を正とする。

### 確率発動のテスト方針
- `secondary_chance` 引数で `battle.random` を制御し、確率発動ケース（発動/非発動）を両方テストする。
