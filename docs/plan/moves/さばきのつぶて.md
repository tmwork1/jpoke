# さばきのつぶて 実装計画書

## 仕様要約（docs/spec/moves/さばきのつぶて.md 参照）

* タイプ: ノーマル（持たせたプレートに応じて変化） / 分類: 特殊 / 威力: 100 / 命中: 100 / PP: 10
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果: 攻撃者の持っているプレートに応じてこの技のタイプが変わる。
  プレートなし・アイテム無効化中（さしおさえ・マジックルーム等）はノーマル固定。
  テラスタルの影響は受けない。追加効果なし。

## 現状の実装状況

`src/jpoke/data/move.py` には登録済みだが、タイプ変化ハンドラが未実装（`handlers={}`）。
仕様に反するため修正が必要。

## ハンドラ構成

| Event | 関数 | subject_spec | priority | 役割 |
| --- | --- | --- | --- | --- |
| `Event.ON_MODIFY_MOVE_TYPE` | `さばきのつぶて_modify_move_type`（`handlers/move_attack.py`） | `attacker:self`（`MoveHandler` デフォルト） | 100（`MoveHandler` デフォルト） | 攻撃者の持つプレートに応じて技のタイプを変更する |

### 実装方針

- 特性マルチタイプの実装（`handlers/ability.py` の `_apply_multitype` / `PLATE_TO_TYPE`）と同じテーブルを再利用する。
- `PLATE_TO_TYPE` は `src/jpoke/data/signature_items.py` で定義済み。
  循環importを避けるため `jpoke.data` 経由ではなく `jpoke.data.signature_items` から直接importする
  （`handlers/ability.py` と同じパターン）。
- 攻撃者の持つアイテム名は `ctx.attacker.item.name`（無効化中は空文字を返す）を使う。
  `GameEffect.name` プロパティが埋め込み無効化状態（さしおさえ・マジックルーム等）を自動的に反映するため、
  本ハンドラ側で個別に埋め込み判定を書く必要はない。
- プレート未所持、または `PLATE_TO_TYPE` に該当しない場合は初期値（`move.data.type` = ノーマル）のまま。

### ハンドラ実装（`handlers/move_attack.py`）

```python
def さばきのつぶて_modify_move_type(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """さばきのつぶて: 持っているプレートに応じてタイプが変わる（プレートがない場合はノーマル固定）。"""
    plate_name = ctx.attacker.item.name if ctx.attacker.has_item() else ""
    new_type = PLATE_TO_TYPE.get(plate_name)
    if new_type is not None:
        value = new_type
    return HandlerReturn(value=value)
```

先頭 import に `from jpoke.data.signature_items import PLATE_TO_TYPE` を追加する。

## Priority 根拠

`docs/spec/turn.md` に `ON_MODIFY_MOVE_TYPE` の行は記載がない。
既存の同種ハンドラ（`ウェザーボール_modify_move_type`、`めざめるダンス_modify_type`）は
いずれも `MoveHandler` のデフォルト値（`priority=100`）のまま個別指定していないため、
本技も同様にデフォルト値を採用する。

## 注意点・エッジケース（review-test 向け）

1. **プレートなし**: `PLATE_TO_TYPE.get("")` は `None` となるため、初期値のノーマルタイプのまま。
2. **せいれいプレート所持**: 本プロジェクトで実装済みの唯一のプレート。フェアリータイプに変化することを確認する。
3. **他のプレート**: `src/jpoke/data/item.py` の `ITEMS` に未登録のため、現状は保持させても
   `Item(name=...)` の生成時に `KeyError` となる（マルチタイプ側のテストと同様、`ITEMS` に存在するプレートのみで検証する）。
4. **かがくへんかガス・かたやぶり**: 本技自体は特性ではないため影響を受けない（マルチタイプ側の話であり本技には無関係）。
5. **威力補正について**: `せいれいプレート_modify_power_by_type` はフェアリータイプの技の威力を1.2倍にする専用ハンドラであり、
   フェアリーに変化したさばきのつぶても対象になる（既存の `ON_CALC_POWER_MODIFIER` ハンドラがタイプ判定で自動的に処理するため、本技側の追加対応は不要）。
