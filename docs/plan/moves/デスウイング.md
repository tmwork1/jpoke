# デスウイング 実装計画書

参照: docs/spec/moves/デスウイング.md

## 概要

特殊・ひこうタイプのドレイン技。相手に与えたダメージの75%だけ自分のHPを回復する。
既存のドレイン技（きゅうけつ・すいとる・ギガドレイン・ドレインパンチ等）と同一の共通処理
`_drain_hp`（`handlers/move_attack.py`）を再利用する。

## MoveData（data/moves/move_ta.py）

```python
"デスウイング": MoveData(
    type="ひこう",
    category="special",
    pp=10,
    power=80,
    accuracy=100,
    flags={"heal"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.デスウイング_drain,
            priority=20,  # turn.md: ON_HIT priority 20 (HP吸収技による回復)
        )
    }
),
```

* PPは `docs/champions/move_list.txt` に単独エントリが存在しないため、第九世代の基準値（10）
  を使用する（`すいとる` と同じ方針）。
* `flags` に `"heal"` を含める（HPを吸収する技であることを示す）。`"contact"` は含めない
  （直接攻撃ではないため）。
* `subject_spec` は `MoveHandler` のデフォルト `"attacker:self"` を使用（明示指定不要）。

## ハンドラ（handlers/move_attack.py）

```python
def デスウイング_drain(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """デスウイングの回復量を計算する。"""
    _drain_hp(battle, ctx, value, heal_ratio=0.75)
    return HandlerReturn(value=value)
```

* 共通関数 `_drain_hp(battle, ctx, damage, heal_ratio)` を呼び出す。
  * `damage`（与えたダメージ）が0の場合（みがわりに命中した場合）は `ctx.substitute_damage` を使う。
  * `heal_amount = max(1, round_half_up(damage * heal_ratio))` で算出し、`Event.ON_CALC_DRAIN`
    を発火して `おおきなねっこ`（1.3倍）・`ヘドロえき`（回復→反動ダメージへの変換）を適用する。
  * `battle.modify_hp(ctx.attacker, v=heal_amount, reason="drain")` で攻撃者のHPを回復する。

## イベント・priority

* `Event.ON_HIT`（ダメージ発生後の処理）: priority=20
  * `docs/spec/turn.md` の `Event.ON_HIT` 表「20 | HP吸収技による回復」に対応。
  * 既存の全ドレイン技ハンドラ（きゅうけつ・すいとる・ギガドレイン・ドレインパンチ等）と同じ
    priority=20 で登録し、処理順の一貫性を保つ。

## テスト方針

* `tests/moves_attack/test_move__group.py` の `test_drain技グループ_回復量が与ダメのheal_ratio倍になる`
  に `("デスウイング", 0.75)` として含める（既存）。
* `tests/moves_attack/test_move_ta.py` に個別の基本動作テスト
  `test_デスウイング_使用後に攻撃者のHPが回復する` を用意する（既存）。
* heal_ratio の違い（0.75 vs 0.5）を確認する比較テストも用意する（既存。ただし比較対象は
  `ドレインパンチ` であり、テスト名を実装内容と一致させる）。
