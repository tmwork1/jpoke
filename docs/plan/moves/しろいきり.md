# 実装計画書: しろいきり

## 仕様要約

- タイプ: こおり / カテゴリ: 変化 / PP: 30 / 対象: 味方の場（target="own_side"）
- 5ターンの間、相手（または第三者）による能力ランクの低下を防ぐ
- すでに自陣営がしろいきり状態のときに使用すると失敗する
- 詳細: `docs/spec/moves/しろいきり.md`, `docs/spec/fields/しろいきり.md`

## 現状の実装状況（レビュー時点）

| 項目 | 状態 |
|------|------|
| サイドフィールド `しろいきり`（ランク低下防止・ターン経過） | 実装済み（`data/field/side_field.py` / `handlers/field.py`） |
| `data/moves/move_sa.py` エントリ | 登録済みだが `target="field"` かつ `handlers={}` で技を使っても何も起きないバグがあった |

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自陣営サイドフィールドに「しろいきり」を設定（5ターン） |

## Priority 根拠

`docs/spec/turn.md` に `Event.ON_STATUS_HIT` は未掲載。変化技の命中処理は `move_executor.py` が
内部的に発火するイベントであり、同種のサイドフィールド設置技（リフレクター・ひかりのかべ・
オーロラベール・おいかぜ・しんぴのまもり）がいずれもデフォルトpriority（100）を使用しているため、
同一パターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「し」ブロック内、`しびれごな_apply_ailment_to_defender` の後、
`しんぴのまもり_set_side_field` の前（gojuon順: しろ「ろ」がら行、しん「ん」が撥音のため）。

```python
def しろいきり_set_side_field(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """しろいきり: 自陣営に「しろいきり」を5ターン設定する。"""
    side = battle.get_side(ctx.attacker)
    if not side.apply("しろいきり", 5, source=ctx.attacker):
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/moves/move_sa.py`

既存の `"しろいきり"` エントリを更新する。`target` を `"field"` から `"own_side"` に修正し、
handlers を追加する。

```python
"しろいきり": MoveData(
    type="こおり",
    category="status",
    pp=30,
    target="own_side",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.しろいきり_set_side_field,
        ),
    },
),
```

## 注意点・エッジケース

- 修正前は `target="field"`（あまごい等の全体天候技と同じ扱い）になっており、技としての
  効果（サイドフィールド設定）が一切実装されていなかった（`handlers={}`）。バグとして修正する。
- `SideFieldManager.apply()` は `StackableFieldManager.activate()` とは別に定義されたメソッドで、
  `ON_MODIFY_DURATION` を発火してから設定する（ひかりのねんど等による延長に対応するための
  共通実装）。しろいきりを延長するアイテム・特性は存在しないため実質的な影響はないが、
  リフレクター・ひかりのかべ・おいかぜと同じ `apply()` パターンに統一する。
- `apply()` が False を返す（すでにしろいきりが有効）場合は失敗として
  `HandlerReturn(value=False, stop_event=True)` を返す。
- ランク低下防止・ターン経過の本体は `handlers/field.py` の `しろいきり_prevent_stat_drop` /
  `しろいきり_tick` 側で処理されており、技側はフィールド設定のみを行う。
- PPは `src/jpoke/data/ps-champ-ja/moves.json` に掲載がないため（本技を習得できるチャンピオンズ対象
  ポケモンが存在しない）、本家準拠のPP30を採用する。
