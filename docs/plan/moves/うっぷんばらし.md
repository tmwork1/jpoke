# 計画書: うっぷんばらし

## 仕様要約

（詳細: `docs/spec/moves/うっぷんばらし.md`）

* あくタイプ / 物理 / 威力75 / 命中100 / PP5 / 対象: 相手1体選択 / 優先度0 / 接触あり
* このわざを使用するターンに、自分のいずれかの能力ランクが「下がった」ことがあれば威力が2倍になる。
  * 誰がランクを下げたか・下げた能力の種類は問わない。
  * まけんき等でランクが下がった直後に上がって元に戻っていても、下がった事実自体が
    あれば条件を満たす。
  * パワースワップ等のランク変化の「交換・操作・リセット」系の効果でランクが下がった
    扱いになっても対象外。
  * 死に出し（ひんし後の交代）で受けたランクダウンは、次に行動できるターンには
    既に判定がリセットされているため対象外になる（後述）。

## 状態管理: `Pokemon.stat_lowered_this_turn`

「そのターン中に能力ランクが下がったか」を判定するための専用フラグを `Pokemon` に持たせる。

```python
# Pokemon.__init__ に追加
self.stat_lowered_this_turn: bool = False  # このターン中にランクが下がったか（うっぷんばらし用）

# Pokemon.reset_turn_state() に追加（ターン開始時にリセット）
self.stat_lowered_this_turn = False
```

* フラグの設定は `core/status_manager.py` の `StatusManager.modify_stats()` で行う。
  能力ランクを実際に下げるすべての経路（技・特性・アイテムいずれも `battle.modify_stats()` /
  `battle.modify_stat()` を通る）を一箇所で捕捉できる。

  ```python
  if actual_changes:
      # うっぷんばらし用: ランクが実際に下がった場合にフラグを立てる
      if any(v < 0 for v in actual_changes.values()):
          target.stat_lowered_this_turn = True
      ...
  ```

* パワースワップ・ガードスワップ・ハートスワップ・ひっくりかえす・くろいきりは、いずれも
  `mon.rank[stat]` を直接書き換える実装になっており、`modify_stats()` を経由しないため、
  自動的に `stat_lowered_this_turn` の対象外になる（実装追加不要）。
* きみょうなくすり・きょうえん特性は現状 `AbilityData()`（未実装のプレースホルダ）であり、
  ランクを下げる処理自体が存在しないため、除外のための特別な実装は不要。

### リセットタイミングと死に出しケースの整合性

* `core/turn_controller.py` の `_begin_turn()` は、新しいターンの開始時
  （`is_new_turn()` が真のとき）に `player_states` 経由で各プレイヤーの現在の場のポケモンに
  対して `reset_turn_state()` を呼ぶ。
* 通常の交代（`_run_switch_phase()`）は `_begin_turn()` の**後**に実行されるため、
  そのターン中に交代でいかくを受けた場合、`stat_lowered_this_turn` はそのターンの技実行まで
  `True` のまま保持される（威力2倍の対象になる = 仕様と一致）。
* 一方、ひんしによる死に出しは技実行フェーズ中に発生する。死に出しで送られたポケモンは
  その場では行動しない（そのターンの行動順に組み込まれていないため）。次に行動できるのは
  次のターン以降であり、次のターン開始時に `reset_turn_state()` が呼ばれてフラグが
  `False` にリセットされる。したがって「死に出し直後のランクダウンは、次に使用しても
  威力2倍にならない」という仕様は、既存のターン進行の順序だけで自然に満たされる。
  専用の分岐処理は不要。

## ハンドラ構成

| イベント | Priority | subject_spec | 役割 |
|---------|---------|-------------|------|
| ON_CALC_POWER_MODIFIER | デフォルト | attacker:self | 使用者が同ターン中にランクを下げられていたら威力2倍 |

## Priority 根拠

`ON_CALC_POWER_MODIFIER` は `docs/spec/turn.md` のターン進行イベント一覧に掲載されていない
（ダメージ計算内部のイベントのため）。同種の条件付き威力2倍ハンドラ
（`たたりめ_double_power_when_ailment` / `からげんき_double_power_when_ailment` /
`ダメおし_double_power_when_hit` など、いずれも `data/move.py` で priority 未指定
＝デフォルト priority）に合わせ、本ハンドラも priority を指定しない。

## 実装箇所

### handlers/move_attack.py

```python
def うっぷんばらし_double_power_when_rank_dropped(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """うっぷんばらし: そのターン中に使用者の能力ランクが下げられていたら威力が2倍になる。"""
    if ctx.attacker.stat_lowered_this_turn:
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)
```

### data/move.py

```python
    "うっぷんばらし": MoveData(
        type="あく",
        category="physical",
        pp=5,
        power=75,
        accuracy=100,
        flags={"contact"},
        handlers={
            Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
                ha.うっぷんばらし_double_power_when_rank_dropped,
            ),
        }
    ),
```

## 注意点・エッジケース

* まもる・おうじゃのしるし等の一般的な技判定はフレームワーク側の共通処理で自動的に
  処理されるため、本技固有のハンドラ追加は不要。
* リーサル計算（`handlers/lethal.py`）は対象外（`docs/progress/move.md` の
  「リーサル実装」列が `n/a`）。威力変動は攻撃側の行動に依存し相手の行動を伴わないため、
  既存のリーサル計算フレームワークの通常経路（威力補正込みのダメージ計算）でカバーされる。
