# 実装計画書: なみだめ

## 仕様

### 技データ

- タイプ: ノーマル
- カテゴリ: 変化
- PP: 20
- 命中率: None（必中）
- 対象: 相手1体
- フラグ: `unprotectable`（まもる等を無視。ダイウォールは除く）

### 効果

相手の **こうげき** ・ **とくこう** ランクをそれぞれ1段階下げる。まもる状態を無視して発動する
（みがわりは貫通しない）。

### 参照仕様書

- `docs/spec/moves/なみだめ.md`

---

## subject_spec と priority

### イベント: `ON_STATUS_HIT`

- **subject_spec**: `"attacker:self"`（`MoveHandler` のデフォルト値を使用）
- **priority**: 100

**根拠**: `docs/spec/turn.md` に `ON_STATUS_HIT` の独立エントリは無いが、同種の防御側ランクダウン
変化技ハンドラ（`なかよくする_modify_defender_stats`, `なきごえ_modify_defender_stats` など）が
いずれも `h.MoveHandler(...)` の priority デフォルト値（100）で登録されているため、これと同一
パターンとして priority=100 を採用する。

## ハンドラ構成

### 1. `src/jpoke/handlers/move_status.py`

共通ユーティリティ `handlers/move.py` の `modify_defender_stats` を利用する（既存実装済み）。
こうげき・とくこうを同時に1段階ずつ下げるため、`stats={"atk": -1, "spa": -1}` を渡す。

```python
def なみだめ_modify_defender_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """なみだめ: 相手のこうげき・とくこうをそれぞれ1段階下げる。"""
    return modify_defender_stats(battle, ctx, value, stats={"atk": -1, "spa": -1})
```

五十音順の挿入位置: `なまける_heal_self` の後、`なやみのタネ_can_apply` の前
（`data/moves/move_na.py` の定義順に合わせる）。

### 2. `src/jpoke/data/moves/move_na.py`

まもるを無視するため `flags={"unprotectable"}` を指定する（みがわりは貫通しないため
`bypass_substitute` は付与しない）。`unprotectable` フラグは `model/move.py` の
`is_blocked_by_protect` でエンジン側が判定するため、フラグ指定のみで良く、
個別の無効化処理をハンドラ側に実装する必要はない。

```python
"なみだめ": MoveData(
    type="ノーマル",
    category="status",
    pp=20,
    accuracy=None,  # 必中
    flags={"unprotectable"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.なみだめ_modify_defender_stats,
        )
    }
),
```

## 注意事項

- `modify_defender_stats` は `battle.modify_stats(ctx.defender, stats, source=ctx.attacker)` を
  呼び出す共通関数であり、まけんき／かちき（特性発動による差し引き上昇）、かいりきバサミ
  （こうげき低下の無効化）、クリアボディ等のランクダウン無効化・反射は、
  既存の `battle.modify_stats` / `ON_STATUS_HIT` 発火経路側で共通処理されるため、
  本ハンドラ側で個別対応は不要
- 確率発動ではない（`chance` 引数はデフォルト1のまま呼び出す）ため、`resolve_secondary_chance`
  は常に1を返し無条件でランクダウンする
- トリックガード（なみだめ等を無効化する特性）は本ライブラリ未実装のため対応不要
- 対象のランク変化が片方のみ阻まれる場合（例: かいりきバサミでこうげきのみ無効化）でも、
  `modify_stats` が dict でまとめて処理するため個別のガード実装は不要
