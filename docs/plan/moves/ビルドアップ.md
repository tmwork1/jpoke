# 計画書: ビルドアップ

## 仕様要約

- 自分の「こうげき」「ぼうぎょ」ランクをそれぞれ1段階ずつ上げる
- タイプ: かくとう、変化、PP: 20、対象: 自分

出典: `docs/spec/moves/status_move.md` 行 208

---

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100 (省略) | `attacker:self` (省略) | こうげき・ぼうぎょを各1段階上げる |

---

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`かたくなる_modify_attacker_stats` 等）がすべて priority=100 (デフォルト) で登録されているため、100 を使用する。

---

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: `まもる_apply_volatile_to_attacker` の後（ま〜み の間、「ビ」は「び」として五十音順でも「ひ」〜「ふ」の間に相当。
実際には `ほたるび_modify_attacker_stats` の後、`まるくなる_apply` の前に挿入する）

```python
def ビルドアップ_modify_attacker_stats(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"A": 1, "B": 1})
```

### `src/jpoke/data/move.py`

`"ビルドアップ"` エントリ（行 5801）に `handlers` を追加する:

```python
    "ビルドアップ": MoveData(
        type="かくとう",
        category="変化",
        pp=20,
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.ビルドアップ_modify_attacker_stats,
            ),
        }
    ),
```

---

## 注意点・エッジケース

- A と B の両方を1つの `stats` dict にまとめて `modify_attacker_stats` に渡せる
- どちらか一方でもランクが最大（+6）の場合は、そのランクの上昇のみ失敗し、もう一方は成功する（マネージャー内部処理）
- ランクがすでに最大の場合の失敗判定は `turn.md` Priority 130 でマネージャー内部処理されるため、ハンドラ側にガード不要
