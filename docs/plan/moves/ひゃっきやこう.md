# 計画書: ひゃっきやこう

## 仕様要約

- ゴーストタイプ / 特殊 / 威力65（Champions基準。SVでは60） / 命中100 / PP15 / 対象: 1体選択 /
  優先度0 / 非接触
- 対象が状態異常（どく・もうどく・まひ・やけど・ねむり・こおり・ゆめうつつ）のとき、威力が2倍になる
  （「たたりめ」と同じ判定条件。`docs/plan/moves/たたりめ.md` を参照）。
- ダメージ命中時、追加効果として30%の確率で相手をやけど状態にする。

## ハンドラ構成

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| ひゃっきやこう | `Event.ON_CALC_POWER_MODIFIER` | デフォルト（100） | デフォルト（`attacker:self`） | 対象（`ctx.defender`）が状態異常のとき威力を2倍にする |
| ひゃっきやこう | `Event.ON_DAMAGE_HIT` | デフォルト（100） | デフォルト（`attacker:self`） | 30%の確率で相手をやけど状態にする |

`Event.ON_CALC_POWER_MODIFIER` および `Event.ON_DAMAGE_HIT` は `docs/spec/turn.md` に priority の
掲載がないため、同種の技（`たたりめ`, `ひのこ` 等）がいずれも `MoveHandler` のデフォルト値
（priority=100, subject_spec="attacker:self"）のまま登録されていることを根拠に、デフォルト設定の
まま登録する。

## 実装箇所

### data/moves/move_ha.py への登録

```python
    "ひゃっきやこう": MoveData(
        type="ゴースト",
        category="special",
        pp=15,
        power=65,
        accuracy=100,
        flags={"secondary_effect"},
        handlers={
            Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
                ha.ひゃっきやこう_double_power_when_ailment,
            ),
            Event.ON_DAMAGE_HIT: h.MoveHandler(
                ha.ひゃっきやこう_apply_burn_to_defender,
            )
        }
    ),
```

- `flags` を指定しない → 直接攻撃ではない（仕様と一致）。既存実装は `secondary_effect` を
  設定済みで、追加効果あり技として扱われる（仕様と一致）。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `power` はChampions基準の65に修正する（既存実装はSV値の60のままだったため誤り）。

### handlers/move_attack.py への実装

```python
def ひゃっきやこう_apply_burn_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="やけど", chance=0.3)


def ひゃっきやこう_double_power_when_ailment(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ひゃっきやこう: 相手が状態異常のとき威力が2倍になる。"""
    if ctx.defender.ailment.is_active:
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)
```

- `ailment.is_active` は状態異常名が空でないことを判定するため、`ゆめうつつ` も対象に含まれる
  （「たたりめ」と同じロジック）。
- `apply_ailment_to_defender` は `ON_DAMAGE_HIT` で発火するため、ダメージ計算（威力2倍判定）の
  後に追加効果判定が行われる仕様と自然に一致する（やけど付与前の状態異常有無で威力を判定）。

## リーサル計算

対象が状態異常かどうかでダメージが変動するが、リーサル計算（`hp_dist` の変化）には影響しない
威力補正のみのため、`handlers/lethal.py` に専用ハンドラは不要（n/a）。「たたりめ」と同様の扱い。

## 注意点・エッジケース

- ネッコアラ（特性『ぜったいねむり』を持つポケモン）は通常ノーマルタイプのため、ゴーストタイプの
  本技はタイプ相性で無効化され命中しない。テラスタル等でタイプが変化している場合や、相手が
  『ねらいのまと』を持っている場合にのみ実際に命中し、威力2倍の効果を確認できる。
