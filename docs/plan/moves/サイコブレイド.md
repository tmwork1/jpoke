# 計画書: サイコブレイド

## 仕様要約（docs/spec/moves/サイコブレイド.md 参照）

- エスパータイプ / 物理 / 威力80 / 命中100 / PP16（チャンピオンズ換算、根拠は仕様書参照） /
  対象: 相手1体 / 優先度0 / 直接攻撃あり（contact）
- 場がエレキフィールド状態のとき、威力が1.5倍になる（接地判定なし。使用者・対象が地面にいなくても適用）。
- 切る技（slashフラグ）。特性きれあじで別枠威力1.5倍。
- 追加効果なし。

## 既存実装との差分

既存実装（`data/moves/move_sa.py` の `"サイコブレイド"` エントリ）には以下の不備がある。

1. **PPが本家基準の15のまま**
   チャンピオンズでは全技PPが8/12/16/20の4段階に統一されている
   （`docs/champions/changes_from_sv.md`）。`src/jpoke/data/ps-champ-ja/moves.json` に
   サイコブレイドの行自体が欠落しているため直接参照できないが、本家PP15の他技
   （シャドークロー・そらをとぶ・サンダーダイブ・シザークロス）が軒並みPP16に変換されている
   ことから、サイコブレイドもPP16が妥当。`pp=15` → `pp=16` に修正する。

2. **`flags` に `slash` が欠落**
   一次情報（Wiki）に「切る技の1つ。特性きれあじのポケモンが使うと威力が1.5倍になる」と明記されている。
   `flags={"contact"}` のみでは `きれあじ_modify_power`（`handlers/ability.py`）の
   `ctx.move.has_flag("slash")` 判定が通らず、きれあじの威力補正が発動しない。
   `flags={"contact", "slash"}` に修正する。

3. **エレキフィールド威力1.5倍のハンドラが未実装**
   このハンドラが技の核となる効果だが、既存実装は `handlers={}` で何も登録されていない。
   新規に `サイコブレイド_calc_power` を実装する。

   参考にする既存の類似実装:
   - `ライジングボルト_calc_power`（`handlers/move_attack.py`）: エレキフィールド中かつ
     相手接地時に威力2倍。ただし接地判定あり。
   - `ワイドフォース_calc_power`（同）: サイコフィールド中かつ自分接地時に威力1.5倍。
     こちらも接地判定あり。

   サイコブレイドはWikiの技の仕様に「使用者や対象が地面にいないときでも威力が1.5倍になる」と
   明記されており、上記2例と異なり **接地判定を行わない**。エレキフィールドかどうかのみで判定する。

## イベント・priority根拠

`docs/spec/turn.md` に `ON_CALC_POWER_MODIFIER` の記載はない（ダメージ計算内部イベントのため）。
既存の同種ハンドラ（`ライジングボルト_calc_power` / `ワイドフォース_calc_power` /
`きれあじ_modify_power`）がいずれも `priority` 未指定（デフォルト100）で登録されているため、
本ハンドラも `priority` 省略（デフォルト100）とする。`subject_spec="attacker:self"` は
`AttackContext` の役割に合わせ、上記2例と同様の指定とする。

## ハンドラ構成

### handlers/move_attack.py（新規追加）

`さ行` セクション（`サイコブレイク`の直後あたり、五十音順は最終整形時に
`sort_handlers.py` で調整されるため、暫定位置に追記する）に以下を追加する。

```python
def サイコブレイド_calc_power(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """サイコブレイド: エレキフィールド中は威力が1.5倍になる。

    ライジングボルト・ワイドフォースと異なり、使用者・対象の接地判定は行わない。
    """
    if battle.terrain.name == "エレキフィールド":
        return HandlerReturn(value=apply_fixed_modifier(value, 6144))
    return HandlerReturn(value=value)
```

### data/moves/move_sa.py（修正）

```python
"サイコブレイド": MoveData(
    type="エスパー",
    category="physical",
    pp=16,
    power=80,
    accuracy=100,
    flags={"contact", "slash"},
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
            ha.サイコブレイド_calc_power,
            subject_spec="attacker:self",
        ),
    },
),
```

対象外（`docs/progress/move.md` の「リーサル実装」列が `n/a`。追加効果を持たない通常の

## 注意点・エッジケース

- エレキフィールドの「でんき技威力1.3倍」ボーナスとは無関係（サイコブレイドはエスパータイプ技のため
  エレキフィールド自体のタイプ別ボーナスは乗らない）。本ハンドラの1.5倍のみが乗る。
- きれあじによる1.5倍とエレキフィールドによる1.5倍は別枠の `power_modifier` 補正として両方乗り得る
  （`apply_fixed_modifier` の固定小数点乗算が `ON_CALC_POWER_MODIFIER` 上で複数回発火する）。
- 接地判定を行わないため、`battle.query.is_floating` は使用しない。
