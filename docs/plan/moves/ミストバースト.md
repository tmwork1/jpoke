## 仕様要約

* タイプ: フェアリー / 分類: 特殊 / 威力: 100（ミストフィールド受け時1.5倍） / 命中: 100 /
  PP: 8（`docs/champions/move_list.txt` 準拠） / 対象: defender
* 使用前に自分の現在HPを全て消費する（`Event.ON_PAY_HP`）
* HP消費後にHPが0になっても技は発動する（`ON_PAY_HP` はヒット処理より前に発火されるため）
* 命中判定・タイプ相性・まもるにより攻撃が不発になっても、使用者は必ずひんしになる
* `flags={"explosion"}` を保有する（しめりけによる失敗判定用）
* 使用者がミストフィールドの効果を受けている（地面にいる状態でミストフィールドが有効）とき、
  威力が1.5倍になる

## 現状の実装状況

`data/moves/move_ma.py` に登録済み・`flags={"explosion"}`・`handlers/move_attack.py` に
`ミストバースト_pay_hp` 実装済み。ただし以下の欠落・誤りがあった（本レビューで修正）:

* PPが5（Gen9本家準拠）のまま実装されており、championsのPP統一ルール（8/12/16/20）に
  未移行だった → 8 に修正
* ミストフィールドによる威力1.5倍が未実装だった → `Event.ON_CALC_POWER_MODIFIER` ハンドラを追加

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `Event.ON_PAY_HP` | `-`（未掲載、デフォルト） | `attacker:self` | 現在HPを全消費する |
| `Event.ON_CALC_POWER_MODIFIER` | `-`（未掲載、デフォルト） | `attacker:self` | ミストフィールド受け時に威力を1.5倍にする |

### Priority 根拠

`docs/spec/turn.md` の `Event.ON_PAY_HP` テーブルには以下の記載がある。

```
| Priority | 処理 |
|----------|------|
| - | じばく等のダメージ・瀕死判定(外れても必ず自滅する技のみ。...) |
```

「じばく等」として明記されており、本実装が対象（`ミストバースト` も同種の自滅技）。priority は
未指定（デフォルト値）で実装する。既存の `いのちがけ_pay_hp` / `じばく_pay_hp` /
`だいばくはつ_pay_hp` も priority 未指定で実装されており、それと同じパターンを踏襲する。

`Event.ON_CALC_POWER_MODIFIER` は `docs/spec/turn.md` に掲載がないため、既存の同種ハンドラ
（フィールドによる威力補正: `ワイドフォース_calc_power` / `サイコブレイド_calc_power` /
`だいちのはどう_power_modifier`）を参照する。いずれも priority 未指定で実装されており、
本実装もそれに合わせる。

## 実装箇所

### handlers/move_attack.py（既存の `ミストバースト_pay_hp` に加え、以下を追加）

```python
def ミストバースト_calc_power(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ミストバースト: 使用者がミストフィールドの効果を受けている場合、威力が1.5倍になる。

    使用者が地面にいなければ、場がミストフィールドであっても威力は上がらない。
    防御側の設置状況は無関係。
    """
    if (
        battle.terrain.name == "ミストフィールド"
        and not battle.query.is_floating(ctx.attacker)
    ):
        return HandlerReturn(value=apply_fixed_modifier(value, 6144))
    return HandlerReturn(value=value)
```

`ミストバースト_pay_hp` は既存実装のまま変更しない。

## リーサル計算について

`docs/progress/move.md` の ミストバースト 行は「リーサル実装」「リーサルテスト」ともに `n/a`。
使用者自身がひんしになる技であり、相手へのダメージ算出は通常の特殊技と同一のため、専用の
リーサルハンドラを追加する必要はない（対象外のまま据え置く）。

## 注意点・エッジケース（テストで確認済み・追加予定）

* **技後にひんしになるか**: `ON_PAY_HP` でHP全消費するため、技使用後、即座にひんしになる。
  → `test_ミストバースト_使用後に攻撃者がひんしになる` で確認済み。
* **HP0でも攻撃が発動するか**: `ON_PAY_HP` はヒット処理より前に発火される（`いのちがけ` と
  同一パターン）。 → `test_ミストバースト_HP消費後も攻撃が相手に届く` で確認済み。
* **しめりけによる失敗**: `flags={"explosion"}` を保有しているため、相手が「しめりけ」特性を
  持つ場合は `ON_TRY_MOVE_1` で失敗し、HPも消費されない。
  → `test_ミストバースト_しめりけ持ちには技が失敗する` で確認済み。
* **ミストフィールドによる威力上昇**: 使用者が地面にいる状態でミストフィールドが有効なとき威力が
  1.5倍になる。使用者が地面にいなければ上がらない。→ 本レビューで新規テストを追加する。
* **ハンドラ側ガード不要**: `ON_PAY_HP` 発火時に使用者が生存していることはバトル側で保証されている
  （行動可能チェック済み）ため、`if not attacker.alive` 等の重複ガードは不要。
