# 実装計画書: いやなおと

## 仕様

### 技データ

- タイプ: ノーマル
- カテゴリ: 変化
- PP: 20（`docs/champions/move_list.txt` 準拠）
- 命中率: 85
- 対象: 相手1体
- フラグ: 音のわざ（みがわりを貫通する）

### 効果

相手の **ぼうぎょ** ランクを2段階下げる。

### 参照仕様書

- `docs/spec/moves/いやなおと.md`

---

## subject_spec と priority

### イベント: `ON_STATUS_HIT`

- **subject_spec**: `"attacker:self"`（`MoveHandler` のデフォルト値を使用）
- **priority**: 100

**根拠**: `docs/spec/turn.md` に `ON_STATUS_HIT` の独立エントリは無いが、同種の防御側ランクダウン
変化技ハンドラ（`あまえる_modify_defender_stats`, `うそなき_reduce_defender_spe` など）が
いずれも `h.MoveHandler(...)` の priority デフォルト値（100）で登録されているため、これと同一
パターンとして priority=100 を採用する。

## ハンドラ構成

### 1. `src/jpoke/handlers/move_status.py`

共通ユーティリティ `handlers/move.py` の `modify_defender_stats` を利用する（既存実装済み）。

```python
def いやなおと_lower_defender_def(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"def": -2})
```

五十音順の挿入位置: `いやしのはどう_heal_defender` の後、`うそなき_reduce_defender_spe` の前。

### 2. `src/jpoke/data/moves/move_a.py`

```python
"いやなおと": MoveData(
    type="ノーマル",
    category="status",
    pp=20,
    accuracy=85,
    flags={"sound"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.いやなおと_lower_defender_def,
        )
    }
),
```

- `flags={"sound"}` により、みがわり貫通（`core/move_executor.py` の sound フラグ汎用処理）・
  ぼうおん/ぼうよわん/せいしんりょく等の音技無効特性・パンクロック等の恩恵が自動的に適用される。

## 注意事項

- `modify_defender_stats` は `battle.modify_stats(ctx.defender, stats, source=ctx.attacker)` を
  呼び出す共通関数であり、まもる・マジックコート・特性による無効化などは既存の
  `battle.modify_stats` / まもる判定 / `ON_STATUS_HIT` 発火経路側で共通処理されるため、
  本ハンドラ側で個別対応は不要
- 確率発動ではない（`chance` 引数はデフォルト1のまま呼び出す）ため、`resolve_secondary_chance`
  は常に1を返し無条件でランクダウンする
- PPは一次情報（Wiki）ではGen9本家基準の40だが、`docs/champions/move_list.txt` により
  ポケモンチャンピオンズでは20に統一されているため、実装値は20を採用する
