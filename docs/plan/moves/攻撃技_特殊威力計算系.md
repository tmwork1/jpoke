# 計画書: 攻撃技_特殊威力計算系

対象技: ジャイロボール / つけあがる / おはかまいり

---

## 仕様要約

### ジャイロボール (はがね/物理/威力可変/命中100/PP5)
- 自分の素早さが相手より遅いほど威力が増加する
- 威力 = floor(25 × 相手の実効素早さ ÷ 自分の実効素早さ) + 1、最大150
- **実効素早さはランク補正・特性・もちもの・まひ・おいかぜ・しつげんの影響を受ける**（wiki「技の仕様」明記）
- 自分の実効素早さが 0 の場合は威力を1として扱う（第六世代以降）
- 弾のわざ（ぼうだんに無効化される）、直接攻撃

### つけあがる (あく/物理/威力可変/命中100/PP10)
- 自分の能力ランク上昇段階の合計1段階ごとに威力+20（初期威力20）
- 威力 = 20 + 20 × (A/B/C/D/Sの正ランク合計) ← 同計算の`アシストパワー`実装から確認
- 直接攻撃

### おはかまいり (ゴースト/物理/威力可変/命中100/PP10)
- その戦闘でひんしになった味方1体ごとに威力+50（初期威力50）
- 威力 = 50 + 50 × ひんし味方数 = 50 × (1 + ひんし味方数)
- ひんし味方数 = `battle.player_states[player].bench` の中でひんし（`fainted`）のもの（`そうだいしょう`特性の実装パターンを参照）
- 直接攻撃

## Priority 根拠

`docs/spec/turn.md` に `ON_CALC_POWER_MODIFIER` イベントの記載はない（ダメージ計算内部のイベントのため）。

既存の同種ハンドラを確認した結果、`data/move.py` の `ON_CALC_POWER_MODIFIER` 登録はすべて `h.MoveHandler(...)` で priority 引数なし（デフォルト priority）で統一されている。

- `けたぐり_calc_power`、`くさむすび_calc_power`、`きしかいせい_calc_power`、`じたばた_calc_power`、`アシストパワー_boost_power_by_rank`、`エレキボール_calc_power` のすべてが priority 指定なし

**根拠**: 上記類似ハンドラすべてがデフォルト priority を使用しているため、3技とも同様にデフォルト priority を採用する。

## 実装箇所

### `src/jpoke/handlers/move_attack.py` に追加する関数

```python
def ジャイロボール_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """ジャイロボール: 自分の実効素早さが相手より遅いほど威力が高くなる。

    威力 = floor(25 × 相手の実効素早さ ÷ 自分の実効素早さ) + 1、最大150。
    実効素早さはランク補正・特性・もちもの・まひ・おいかぜ・しつげんの影響を含む。
    自分の実効素早さが 0 の場合は威力を 1 として扱う（第六世代以降の仕様）。
    """
    atk_speed = battle.speed_calculator.calc_effective_speed(ctx.attacker)
    def_speed = battle.speed_calculator.calc_effective_speed(ctx.defender)
    if atk_speed <= 0:
        power = 1
    else:
        power = min(150, 25 * def_speed // atk_speed + 1)
    return HandlerReturn(value=power * 4096)


def おはかまいり_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """おはかまいり: ひんしになった味方の数に応じて威力が増加する。

    威力 = 50 × (1 + ひんし味方数)。
    ひんし味方数は控えポケモン（bench）のうちひんしのもの。
    場に出ている自分自身はひんしでないため bench に含まれない。
    """
    player = battle.get_player(ctx.attacker)
    state = battle.player_states[player]
    fainted_count = sum(1 for p in state.bench if p.fainted)
    power = 50 * (1 + fainted_count)
    return HandlerReturn(value=power * 4096)


def つけあがる_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """つけあがる: 使用者の能力ランク上昇段階の合計1段階ごとに威力が20増加する（基本威力20）。

    威力 = 20 + 20 × rank_sum = 20 × (1 + rank_sum)
    A/B/C/D/S の正ランク合計を使う（ACC/EVA は対象外）。
    アシストパワーと同計算。
    """
    attacker = ctx.attacker
    rank_sum = sum(max(0, v) for k, v in attacker.rank.items() if k in ("A", "B", "C", "D", "S"))
    power = 20 * (1 + rank_sum)
    return HandlerReturn(value=power * 4096)
```

**注意**: `sort_handlers.py` で五十音順に並び替えること（ア行・オ行・ツ行）。

## 注意点・エッジケース（review-test 向け）

### ジャイロボール

1. **速度計算に `battle.speed_calculator.calc_effective_speed()` を使う**
   - raw `stats["S"]` ではなく `calc_effective_speed()` を呼ぶことで、まひ・ランク補正・おいかぜ・こだわりスカーフ等の影響を正しく反映する
   - wiki「技の仕様」に「ランク補正・特性・もちもの・まひ・おいかぜ・しつげんの影響を受けて威力が決まる」と明記されている
   - `エレキボール_calc_power` が `stats["S"]` を直接使っている点と**異なる**（エレキボールは速度段階テーブルで区切るため実効値不要）

2. **自分の速度が 0 の場合は威力を 1 として扱う**（第六世代以降の仕様）

3. **最大威力は 150**（min でクランプ）

4. **トリックルームは威力に影響しない**（wiki 明記。トリックルームは行動順を変えるが速度実数値は変わらない）

5. **`power=1` は「可変」の意味でデータに設定されている**。ハンドラが必ず `power * 4096` を返すため、`power=1` という初期値は実際には使われない

### つけあがる

1. **ACC/EVA はランク合計に含めない**（`アシストパワー_boost_power_by_rank` と同様）

2. **ランク合計が 0 のとき威力は 20**（初期値）。ランク低下分は計上しない（`max(0, v)` で正値のみ）

3. **ランク最大 +6 が A/B/C/D/S の 5 ステータスなら最大合計 30 → 威力 = 20 × 31 = 620**（上限なし）

4. **アシストパワーとの違い**: アシストパワーは `rank_sum > 0` のときのみ modifier を乗算する実装だが、つけあがるは常に `power * 4096` を直接返す実装にする（基本威力 20 を自分で設定するため）

### おはかまいり

1. **ひんし味方のカウントに `bench` を使う**
   - `bench` = 選出ポケモンのうち現在場に出ていないもの
   - 使用者本人（場に出ている）はカウントから除外される
   - `そうだいしょう_modify_power` が同じ `state.bench` アクセスパターンを使用している

2. **選出外ポケモン（`team` にいるが `selection_indexes` に含まれないもの）はカウントしない**
   - `bench` は `selection` から `active` を除いたもの = 選出済みの控えのみ

3. **威力の上限なし**（最大3体ひんしで威力 200）

4. **「これまでひんしになった」味方のカウント方法**
   - 現在ひんし（`fainted` = True）の控えポケモンを数える
   - 一度ひんしになって戦闘不能になった後も `fainted` フラグは保持される
   - 控え（bench）にいる場合のみカウント（場に出た時点では active なので bench に含まれない）

5. **自分もひんしになりうる技ではない**（使用前に自分はひんしでないため bench に含まれない）
