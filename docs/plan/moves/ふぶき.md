# ふぶき 実装計画書

## 仕様要約（docs/spec/moves/ふぶき.md 参照）

* タイプ: こおり / 分類: 特殊 / 威力: 110 / 命中: 70 / PP: 8
* 対象: 相手（自分以外の全員。本プロジェクトはシングルバトルのみのため相手1体と同義）
* 優先度: 0 / 接触なし
* 効果: 追加効果として、10%の確率で相手を『こおり』状態にする。
* 天気が『ゆき』のときは必ず命中する。
* 「風の技」に分類され、特性 かぜのり／ふうりょくでんき の対象になる。

## 現状の実装状況・レビューで検出した誤り

`src/jpoke/data/moves/move_ha.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。
レビュー時点で以下の誤りを検出し修正した：

1. **PP誤り**: `pp=5`（一次情報＝ポケモンWikiの値）になっていたが、
   `docs/champions/move_list.txt` の `ふぶき` 行はPP8であり、本プロジェクトはポケモンチャンピオンズの
   値を優先するため `pp=8` に修正した。
2. **命中率補正が天候無効化を考慮していない誤り**: `ふぶき_accuracy` が
   `battle.weather_manager.current.name` を直接参照しており、エアロック・ノーてんきによる天候無効化を
   無視して必中判定していた。`かみなり_accuracy` 等の既存実装（`battle.weather_for(ctx.defender)` を
   使うパターン）に合わせ、`battle.weather_for(ctx.defender).name == "ゆき"` を使うよう修正した。
   なお、ばんのうがさ（`ばんのうがさ_weather_immune`）は「はれ・あめ・おおひでり・おおあめ」のみを
   対象にしており、`ゆき` は対象外（実際のゲーム仕様通り）のため、ばんのうがさ所持者に対しては
   本来通りゆき天候による必中補正が適用される。この点は今回の修正対象ではない。

```python
"ふぶき": MoveData(
    type="こおり",
    category="special",
    pp=8,
    power=110,
    accuracy=70,
    flags={"wind", "secondary_effect"},
    handlers={
        Event.ON_MODIFY_ACCURACY: h.MoveHandler(
            ha.ふぶき_accuracy,
            subject_spec="attacker:self"
        ),
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ふぶき_apply_freeze_to_defender,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_MODIFY_ACCURACY` | 100（デフォルト） | attacker:self | 天候による命中率補正（ゆき: 必中） |
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手を『こおり』状態にする |

`src/jpoke/handlers/move_attack.py` の実装（修正済み）:

```python
def ふぶき_accuracy(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ふぶき: ゆき状態の時は必中になる補正。
    防御側がばんのうがさを持つ場合や、エアロック・ノーてんきで天候が無効化されている場合は
    必中補正の対象外になる（`battle.weather_for` が天候無効化を考慮するため）。
    """
    if battle.weather_for(ctx.defender).name == "ゆき":
        return HandlerReturn(value=None)  # 必中
    return HandlerReturn(value=value)


def ふぶき_apply_freeze_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="こおり", chance=0.1)
```

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は汎用ハンドラ側で処理される。本技側に個別のガードは不要。

`battle.weather_for(mon)` はエアロック・ノーてんきが場に出ている場合、ゆき天候を「なし」として返すため、
天候無効化特性が場にいる場合でも正しく必中補正が無効化される（`ふぶき_accuracy` 側の実装で対応済み）。
なお、ばんのうがさは「はれ・あめ・おおひでり・おおあめ」のみを対象とし `ゆき` を対象としないため
（`handlers/item.py` の `ばんのうがさ_weather_immune` 参照）、ばんのうがさ所持者に対しても
ゆき天候によるふぶきの必中補正はそのまま適用される。

## Priority 根拠

* `ON_MODIFY_ACCURACY` はダメージ計算内部のイベントであり `docs/spec/turn.md` に priority 記載がないため、
  既存の同種ハンドラ（`かみなり_accuracy`）と同じデフォルト priority（100）のパターンに従う。
* `ON_DAMAGE_HIT` によるこおり付与も同様に、既存の同種ハンドラ（`れいとうビーム`・`こおりのキバ`等の
  こおり付与ハンドラ）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。

## リーサル計算

固定威力・追加効果（状態異常付与のみでダメージ量に影響しない）の通常攻撃技のため、

## 注意点・エッジケース（review-test 向け）

1. **天候による命中率補正**
   - ゆき（防御側から見た天候）: 必中。ランク補正（回避・命中ランク）も無視される
     （`ON_MODIFY_ACCURACY` が `None` を返すと `_check_hit` がランク補正前にTrueを返すため）。
   - エアロック・ノーてんきが場にいる場合はゆき天候自体が「なし」扱いになるため必中補正も無効化される
     （一般則として `tests/abilities/test_ability_a.py` の
     `test_エアロック_天候と強天候を無効化する` でゆきを含む全天候の無効化を検証済み。
     `かみなり` 同様、本技専用の追加テストは不要）。
   - ばんのうがさは「はれ・あめ」系のみを対象とするため、ばんのうがさ所持者に対してもゆき天候による
     必中補正はそのまま適用される（`かみなり`・`ぼうふう` 等の雨天候必中補正とは異なる挙動のため
     混同しないよう注意）。
2. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・こおり付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
3. **テスト方針**
   - `secondary_chance` を1.0に固定し、こおり付与を確認するテスト
     （`test_ふぶき_こおりが発動する`）は既存。
   - ゆき天候中の必中補正を確認するテスト（`test_ふぶき_ゆき中は必中になる`）は既存。
