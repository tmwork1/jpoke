# 計画書: じわれ

## 仕様要約

（`docs/spec/moves/じわれ.md`、`docs/spec/moves/_ohko.md` 参照）

* じめんタイプ・物理、威力なし（一撃必殺技）、命中30固定、PP5、非接触技。
* 命中した場合、対象のHPを0にする。反動・追加効果・急所判定なし。
* じめんタイプ技であるため、ひこうタイプ・特性「ふゆう」・「でんじふゆう」を受けたポケモンには
  タイプ無効/浮遊判定により無効化される（じわれ専用の実装は不要）。
* 一撃必殺技共通の無効化条件（がんじょう・きあいのタスキ等での生存、まもる等でのブロック、
  はりきりの命中率ペナルティ対象外、おうじゃのしるし等の追加効果ハンドラ対象外）は
  既存の `flags={"ohko"}` を参照する各特性・アイテムハンドラで共通対応済み。

## 既存実装の状況

`data/move.py` の `じわれ` エントリは既に実装済みで、`ha.ohko_damage`
（`handlers/move_attack.py`）を `Event.ON_MODIFY_MOVE_DAMAGE` に `priority=90` で登録する、
`つのドリル`・`ハサミギロチン`・`ぜったいれいど` と同一パターンを踏襲している。
レビューの結果、実装・登録内容に誤りはなかった（`power` フィールドが未指定だった点のみ、
他の一撃必殺技との表記統一のため `power=0` を明示するよう修正した。`MoveData.power` の
デフォルトは `None` であり、`damage.py` の `calc_damages` は `if not move.power: return [0]`
で `None`/`0` を同一に扱うため、この修正はスタイル統一のみで挙動に影響しない）。

## イベント選定の根拠

* **確定ダメージの適用**: `Event.ON_MODIFY_MOVE_DAMAGE`、`priority=90`。
  `docs/spec/turn.md` に個別記載のないダメージ計算内部イベントだが、既存の `つのドリル`・
  `ハサミギロチン`・`ぜったいれいど` と同一の `ha.ohko_damage` ハンドラ・同一 priority を
  用いることで、`きあいのタスキ`（`item.py` の `きあいのタスキ_survive_ohko`）等の生存ハンドラとの
  実行順を素早さに依存させず、常に一撃必殺技側が先に確定ダメージを設定してから生存判定を
  行う既存の合意済み設計を維持する。

## ハンドラ構成

| 対象 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| じわれ（技） | `Event.ON_MODIFY_MOVE_DAMAGE` | 90 | `attacker:self`（`ha.ohko_damage` は既存共通ハンドラ） | 対象のHPと同値を確定ダメージとして設定する |

```python
"じわれ": MoveData(
    type="じめん",
    category="physical",
    pp=5,
    power=0,
    accuracy=30,
    flags={"ohko"},
    handlers={
        Event.ON_MODIFY_MOVE_DAMAGE: h.MoveHandler(
            ha.ohko_damage,
            priority=90,
        ),
    }
),
```

## リーサル計算

`docs/progress/move.md` の該当行は「リーサル実装: n/a」「リーサルテスト: n/a」のため対象外。

## テスト

* 命中したとき、対象のHPが0になること（`test_一撃必殺技_命中時は相手を一撃で倒す`。
  代表技は `つのドリル` だが `flags={"ohko"}` 共通ハンドラのため `じわれ` も同様に成立する）。
* 外したとき、ダメージを与えないこと（`test_一撃必殺技_外した時はダメージを与えない`）。
* タイプ相性で無効化されること（`test_一撃必殺技_タイプ相性で無効化される`。
  `じわれ` 対 `ピジョン`（ひこうタイプ）のケースを追加）。
* がんじょう・きあいのタスキ等のアイテム/特性との組み合わせは `tests/test_item.py`・
  `tests/abilities/` 側の既存テストでカバーする。

## 注意点・エッジケース

* 特性「ふゆう」「でんじふゆう」による無効化は、じめんタイプ技全般に共通する既存の
  グラウンド判定に委譲しており、じわれ専用のテスト追加は行わない。
* 「テレキネシス」による無効化は一次情報に記載があるが、本プロジェクトでは
  テレキネシス自体が未実装のため対応不要（実装時にグラウンド判定へ自然に組み込まれる想定）。
