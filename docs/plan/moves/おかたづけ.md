# 計画書: おかたづけ

## 仕様要約

- タイプ: ノーマル / カテゴリ: 変化 / PP: 12（`docs/champions/move_list.txt` 準拠。Gen9本家は10） / 命中: なし / 対象: 自分
- 使用者を含む全ポケモンのみがわり状態を除去する（両陣営）
- 両陣営のサイドフィールドに設置されたトラップ（まきびし・どくびし・ステルスロック・ねばねばネット）をすべて除去する
- 自分のこうげきとすばやさを1段階ずつ上げる
- みがわり・トラップが1つも存在しない場合でも、能力上昇は独立して発生する
- まもる: × / マジックコート: × / 命中判定: ×

### wiki 仕様詳細（技の仕様節）

- 解除順序: みがわり（ホスト側→もう片方の順）→ホスト側トラップ（まきびし→どくびし→ステルスロック→ねばねばネット）→相手側トラップの同順
  （シングルバトルのみが対象のためログの見え方以外に実質的な影響はない）
- ランクがすでに最大のときも、みがわり・トラップの除去効果は発動する
- 姿を隠しているポケモンのみがわり状態も除去できる
- 使用者特性がすりぬけのときもみがわりを除去できる
- ゆびをふるで選ばれない（`ゆびをふる` 自体が本プロジェクトで未実装のため対象外）

## Priority 根拠

`docs/spec/turn.md` には `ON_STATUS_HIT` の内部priority表がない。既存の変化技ハンドラ（`おいかぜ_set_side_field`、`きりばらい_defog` など）がすべて priority 100（デフォルト）を使用している。おかたづけも同一パターンとして priority 100 を採用する。

## 実装内容

### `src/jpoke/handlers/move_status.py`

```python
def おかたづけ_cleanup(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """おかたづけ: 両陣営のみがわり・トラップを除去し、こうげき・すばやさを1段階上げる。"""
    for mon in battle.actives:
        if mon.has_volatile("みがわり"):
            battle.volatile_manager.remove(mon, "みがわり")
    trap_names = ["まきびし", "どくびし", "ステルスロック", "ねばねばネット"]
    for side in battle.side_managers:
        for trap in trap_names:
            side.deactivate(trap)
    return modify_attacker_stats(battle, ctx, value, stats={"atk": 1, "spe": 1})
```

### `src/jpoke/data/moves/move_a.py`

"おかたづけ": MoveData(

## 注意点・エッジケース

- **両陣営への適用**: `battle.actives` ですべてのみがわりを、`battle.side_managers` を直接走査することで
  両サイドのトラップを確実に処理する
- **失敗条件なし**: 各 `deactivate` の戻り値は使わない（`stop_event` は発生しない）。除去対象が
  何もなくても除去処理自体は正常に完了する
- **みがわり除去ログ**: `battle.volatile_manager.remove` は内部でログを出力するため、個別のメッセージ
  追加は不要
- **トラップ除去ログ**: `side.deactivate` は `ON_FIELD_DEACTIVATE` を経由してフレームワーク側でログが
  出る想定であり、専用の「かたづけ おわり!」メッセージ（`LogCode.MOVE_CUSTOM` 等）は本プロジェクトの
  ログ体系に存在しないため実装しない（フレーバーテキストの再現は対象外）
- **技全体の成否判定**: `modify_attacker_stats` の戻り値（`battle.modify_stats` の結果）がそのまま
  `ON_STATUS_HIT` の成否として扱われるため、こうげき・すばやさが共に最大でランク上昇が不発だった場合は
  みがわり・トラップを除去していても技全体は「しかし失敗した」扱いになる。これは `きりばらい_defog`
  （回避率下降の成否のみで技全体の成否を判定）と同一の設計判断であり、本プロジェクトの既存precedentに
  従う
- **こうそくスピン・きりばらい との違い**: こうそくスピン（攻撃技）は自陣トラップのみ除去。きりばらい
  （変化技）は壁系（対象側のみ）・トラップ（両陣営）・フィールドを除去し、回避率も-1する
