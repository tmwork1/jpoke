# マッドショット 実装計画書

## 仕様要約（docs/spec/moves/マッドショット.md 参照）

* タイプ: じめん / 分類: 特殊 / 威力: 55 / 命中: 95 / PP: 16
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果: 追加効果として100%の確率で相手の『すばやさ』ランクを1段階下げる。

## 現状の実装状況（修正前）

`src/jpoke/data/moves/move_ma.py` は以下の通り。

```python
"マッドショット": MoveData(
    type="じめん",
    category="special",
    pp=15,
    power=55,
    accuracy=95,
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.マッドショット_lower_defender_spd,
        )
    }
),
```

問題点は2点:

1. `pp=15` が本家シリーズ基準のままで、`docs/champions/move_list.txt` の PP=16 に移行できていない
   （Championsは全技PPを8/12/16/20の4段階に統一しているため、正しい値は16）。
2. `flags` に `"secondary_effect"` が設定されておらず、`docs/spec/moves/_move_secondary_1.md` の
   一覧（がんせきふうじ・あわ・からみつく 等）と同種の追加効果技であるにもかかわらず、
   `ちからずく_boost`（`handlers/ability.py`）による威力1.3倍化と `ちからずく_disable_secondary_effect`
   によるすばやさ低下無効化のいずれも発動しない状態になっている
   （`ctx.move.has_flag("secondary_effect")` が常に False になるため）。

### 修正後

```python
"マッドショット": MoveData(
    type="じめん",
    category="special",
    pp=16,
    power=55,
    accuracy=95,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.マッドショット_lower_defender_spd,
        )
    }
),
```

追加効果ハンドラ（`handlers/move_attack.py` の `マッドショット_lower_defender_spd`）は仕様と一致しており、
修正不要。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 100%の確率で相手の『すばやさ』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def マッドショット_lower_defender_spd(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spe": -1})
```

`modify_defender_stats`（`handlers/move.py`）は `chance` 省略時デフォルト1（100%）で
`battle.resolve_secondary_chance` を経由するため、ちからずくによる追加効果無効化は
汎用的に処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（能力ランク低下）は既存の同種ハンドラ（がんせきふうじ・あわ・からみつく 等）と
同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。`docs/spec/turn.md` にも
`ON_DAMAGE_HIT` は他の追加効果技と共通のタイミングとして扱われており、個別の priority 検討は不要。

### handlers/move_attack.py

追加・修正なし（既存の `マッドショット_lower_defender_spd` を流用）。

### data/moves/move_ma.py

`"マッドショット"` エントリの `pp` を15→16に修正し、`flags={"secondary_effect"}` を追加する。

## リーサル計算について

`docs/progress/move.md` の該当行「リーサル実装」列は `n/a`。本技の追加効果は相手の『すばやさ』ランクを
下げるのみでHPに影響しないため、`handlers/lethal.py` に専用ハンドラは不要で、対象外（n/a）のままで正しい。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` を追加したことで、ちからずく所持時は威力が1.3倍になり、
     すばやさ低下効果が発動しなくなる（`がんせきふうじ` 等と同じ挙動）。
2. **テスト方針**
   - `secondary_chance` を1に固定し、すばやさランク変化が発動することを確認するテストを
     `tests/moves_attack/test_move_ma.py` に追加する（`がんせきふうじ` の既存テストと同じパターン）。
   - ちからずく所持時に威力1.3倍・すばやさ低下無効化になることを確認するテストを追加する。
   - PPが16になっていることを確認するテスト（`move.pp` 直接確認 or 使用後残りPP確認）を追加する。
