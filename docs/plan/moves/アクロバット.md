# アクロバット 実装計画書

参照: docs/spec/moves/アクロバット.md

## 現状

* `src/jpoke/data/moves/move_a.py` の `"アクロバット"` エントリは以下の通り実装済みだった。
  - type="ひこう", category="physical", power=55, accuracy=100
  - flags={"contact"}
  - `pp=15`（本家基準の値のまま。移行漏れのバグ）
  - `handlers={}`（道具無し威力2倍の効果が未実装）

## 発見した不具合と修正

1. **PP誤り**: `src/jpoke/data/ps-champ-ja/moves.json` ではアクロバットのPPは16
   （チャンピオンズは全技PPを8/12/16/20の4段階に統一しているため）。
   実装は本家(15)のままだったので `pp=16` に修正する。
2. **効果未実装**: 「自分が道具を持っていない場合、威力が2倍になる」効果が
   ハンドラ未登録のため発動しない。`Event.ON_CALC_POWER_MODIFIER` に
   `MoveHandler` を追加して実装する。

### priority の根拠

`docs/spec/turn.md` に `ON_CALC_POWER_MODIFIER` の記載はない（ダメージ計算内部の
イベントのため）。同種の「技固有条件で威力が2倍になる」ハンドラ
（`からげんき_double_power_when_ailment` など）はいずれも `MoveHandler` の
デフォルト優先度（100）のまま登録されているため、本技も同様にデフォルト優先度を用いる。

### 実装

```python
def アクロバット_double_power_when_no_item(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """アクロバット: 自分が道具を持っていないとき威力が2倍になる。

    `has_item()` のデフォルト（`consider_enabled=False`）は道具の効果が
    ぶきよう・さしおさえ・マジックルームなどで無効化されていても、
    物理的に道具を持っていればTrueを返す。アクロバットの威力2倍判定は
    「道具を持っているかどうか」のみで行うため、このデフォルト挙動をそのまま利用する。
    """
    if not ctx.attacker.has_item():
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)
```

`data/moves/move_a.py` 側は次の通り登録する（`MoveHandler` のデフォルト
`subject_spec="attacker:self"` を使用）。

```python
"アクロバット": MoveData(
    type="ひこう",
    category="physical",
    pp=16,
    power=55,
    accuracy=100,
    flags={"contact"},
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
            ha.アクロバット_double_power_when_no_item,
        ),
    },
),
```

### ジュエル・テクニシャンとの相互作用について

* ジュエル消費（`ノーマルジュエル_modify_power_by_type`、`ItemHandler` デフォルト優先度100）は
  ポケモンの入れ替わり時に登録されるためハンドラリスト内で技ハンドラ（使用時に登録）より
  先に位置し、同一優先度・同速のスタブルソートで先に実行される。そのためジュエル消費により
  道具が空になった後にアクロバット自身のハンドラが判定するので、「ジュエル消費で道具無し扱いになり
  威力2倍、かつジュエル自身の威力上昇も乗る」という仕様通りの挙動になる。
* テクニシャン（`テクニシャン_boost_power`、`AbilityHandler`）は `ctx.move.data.power`
  （宣言時の固定威力55）を基準に判定しており、道具の有無によって変動した後の威力を
  見ていない。そのため現状の実装では、道具が無くアクロバットの威力が110になっている場合でも
  テクニシャンの1.5倍が適用されてしまう可能性があり、`docs/spec/abilities/テクニシャン.md`
  （8行目）が明記する「変動後の威力で判定する」仕様と厳密には一致しない。
  この不整合はアクロバット固有の問題ではなく、めざめるパワー・かぜおこし・おいうちなど
  技自身の条件で威力が変動する技全般に共通するテクニシャン側の実装課題であるため、
  本計画書では対象外とし、テクニシャンの実装レビュー（`docs/progress/ability.md` の
  テクニシャン行）側で扱うべき課題として記録するに留める。
