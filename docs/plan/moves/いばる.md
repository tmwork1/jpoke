# 計画書: いばる

## 仕様要約

- ノーマルタイプ / 変化 / 命中85 / PP15 / 対象: 1体選択
- 相手の『こうげき』ランクを2段階上げる
- 相手を1〜4ターンの間『こんらん』状態にする
- 効果の成否判定: こうげきが+6でこんらん済みなら失敗（`ON_APPLY_MOVE` priority 100: `いばる等: ランク+6でこんらん`）
- こうげきが+6でも、こんらんでなければ技は成功（こんらんのみ付与）
- こんらん済みでも、こうげきが+6未満であればこうげきアップのみ適用して成功

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | こうげき+2、こんらん付与 |

## Priority 根拠

`docs/spec/turn.md` の `ON_APPLY_MOVE` に `いばる等: ランク+6でこんらん` が priority 100 で記載されており、両方が+6かつこんらん済みの場合のみフレームワーク側で失敗にする。ハンドラ本体は既存の `ON_STATUS_HIT` priority 100 のパターンに従う。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `あくまのキッス` 関連の後、`あさのひざし` の前（五十音順）。

```python
def いばる_modify_defender_stats_and_apply_volatile(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """いばるの効果: 相手のこうげきを2段階上げ、相手をこんらん状態にする（2〜5ターン）。"""
    # こうげきランクアップ（失敗しても次のこんらん付与を試みる）
    battle.modify_stats(ctx.defender, {"A": 2}, source=ctx.attacker)
    count = battle.random.randint(2, 5)
    battle.volatile_manager.apply(ctx.defender, "こんらん", count=count, source=ctx.attacker, ctx=ctx)
    return HandlerReturn(value=value)
```

### data/move.py への更新

`"いばる"` エントリのハンドラを追加する。

## 注意点・エッジケース

- 失敗判定（こうげき+6かつこんらん済み）は `ON_APPLY_MOVE` priority 100 でフレームワーク側が処理するため、ハンドラ内にガードは不要
- こうげきが+6の場合でもこんらんが未付与であれば技は成功し、こんらんのみ付与される
- こんらん済みでもこうげきが+6未満であればこうげきアップのみ適用される
- `battle.modify_stats` は個別のランク上限チェックを行うが、こうげき+2が「すでに+6」であっても失敗扱いにはならず変化ゼロで返る
- あやしいひかり等と同じ `count = battle.random.randint(2, 5)` パターンを使用（2〜5ターン）
