# 計画書: いばる

## 仕様要約

- ノーマルタイプ / 変化 / 命中85 / PP16（`docs/champions/move_list.txt`） / 対象: 1体選択
- 相手の『こうげき』ランクを2段階上げる
- 相手を1〜4ターンの間『こんらん』状態にする（内部実装は `randint(2, 5)` のカウント管理）
- 失敗条件: こうげきが+6で、かつすでにこんらん状態なら技全体が失敗する
  （`docs/spec/turn.md` の `ON_APPLY_MOVE`＝実装上の `Event.ON_BEFORE_APPLY_MOVE` priority 100:
  `いばる等: ランク+6でこんらん`）
- こうげきが+6でも、こんらんでなければ技は成功（こんらんのみ付与）
- こんらん済みでも、こうげきが+6未満であればこうげきアップのみ適用して成功

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_BEFORE_APPLY_MOVE` | 100（デフォルト） | attacker:self（省略） | 失敗条件判定（こうげき+6かつこんらん済み） |
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | こうげき+2、こんらん付与 |

## Priority 根拠

`docs/spec/turn.md` の `ON_APPLY_MOVE`（実装上の `Event.ON_BEFORE_APPLY_MOVE`）に
`いばる等: ランク+6でこんらん` が priority 100 で記載されている。この失敗条件はフレームワーク側
では自動処理されないため、`はらだいこ_can_apply` や `ちからをすいとる_can_apply` と同様に
`ON_BEFORE_APPLY_MOVE` に専用の `_can_apply` ハンドラを登録して判定する。
`ON_STATUS_HIT` 本体は既存の priority 100 のパターンに従う。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `いのちのしずく` の後、`いやしのすず` の前（五十音順）。

```python
def いばる_can_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """いばるの失敗条件: 相手のこうげきランクがすでに+6、かつすでにこんらん状態なら失敗する。"""
    assert ctx.defender is not None
    if ctx.defender.rank["atk"] == 6 and ctx.defender.has_volatile("こんらん"):
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="いばる")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def いばる_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """いばるの効果: 相手のこうげきを2段階上げ、相手をこんらん状態にする。"""
    battle.modify_stats(ctx.defender, {"atk": 2}, source=ctx.attacker)
    battle.volatile_manager.apply_confusion(ctx.defender, source=ctx.attacker)
    return HandlerReturn(value=value)
```

`battle.volatile_manager.apply_confusion()` は内部で `randint(2, 5)` のカウントを設定するため、
呼び出し側でカウント値を指定する必要はない（あやしいひかり・おだてる等と共通のヘルパー）。

## 注意点・エッジケース

- 失敗判定（こうげき+6かつこんらん済み）はフレームワーク側で自動処理されないため、
  `ON_BEFORE_APPLY_MOVE` に明示的な `_can_apply` ハンドラが必要（はらだいこ・ちからをすいとる等と同型）
- こうげきが+6の場合でもこんらんが未付与であれば技は成功し、こんらんのみ付与される
- こんらん済みでもこうげきが+6未満であればこうげきアップのみ適用される
- `battle.modify_stats` は個別のランク上限チェックを行うが、こうげき+2が「すでに+6」であっても
  失敗扱いにはならず変化ゼロで返る（この単独の挙動だけでは技全体は失敗にならない点に注意）
- マイペース所持者へのこんらん付与は `volatile_manager.apply` 内の `ON_BEFORE_APPLY_VOLATILE`
  （priority=140）で個別に無効化される。こうげきアップ自体は別途適用されるため、
  マイペース所持者に対しても `いばる_can_apply` の判定（こんらん有無）には影響しない
