# 実装計画書: いとをはく

## 仕様

### 技データ

- タイプ: むし
- カテゴリ: 変化
- PP: 20（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）
- 命中率: 95
- 対象: 相手（シングルバトルでは相手1体）

### 効果

相手の **すばやさ** ランクを2段階下げる。

### 参照仕様書

- `docs/spec/moves/いとをはく.md`

---

## subject_spec と priority

### イベント: `ON_STATUS_HIT`

- **subject_spec**: `"attacker:self"`（`MoveHandler` のデフォルト値を使用）
- **priority**: 100

**根拠**: `docs/spec/turn.md` に `ON_STATUS_HIT` の独立エントリは無いが、同種の防御側ランクダウン
変化技ハンドラ（`あまえる_modify_defender_stats`, `あまいかおり_modify_defender_stats` など）が
いずれも `h.MoveHandler(...)` の priority デフォルト値（100）で登録されているため、これと同一
パターンとして priority=100 を採用する。

---

## ハンドラ構成

### 1. `src/jpoke/handlers/move_status.py`

共通ユーティリティ `handlers/move.py` の `modify_defender_stats` を利用する（既存実装済み）。

```python
def いとをはく_modify_defender_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """いとをはくの効果: 相手のすばやさを 2 段階下げる。"""
    return modify_defender_stats(battle, ctx, value, stats={"spe": -2})
```

五十音順の挿入位置: `いちゃもん_apply` の後、`いのちのしずく_heal` の前。

### 2. `src/jpoke/data/moves/move_a.py`

```python
"いとをはく": MoveData(
    type="むし",
    category="status",
    pp=20,
    accuracy=95,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.いとをはく_modify_defender_stats,
        ),
    }
),
```

## 注意事項

- `modify_defender_stats` は `battle.modify_stats(ctx.defender, stats, source=ctx.attacker)` を
  呼び出す共通関数であり、まもる・マジックコートによる無効化・反射は既存の
  `battle.modify_stats` / まもる判定 / `マジックコート_reflect`（`category == "status"` かつ
  `target in {"foe", "foe_side"}` で自動判定）側で共通処理されるため、本ハンドラ側で個別対応は不要
- 確率発動ではない（`chance` 引数はデフォルト1のまま呼び出す）ため、`resolve_secondary_chance`
  は常に1を返し無条件でランクダウンする
- PP は Gen9本家では40だが、ポケモンチャンピオンズでは全技PPが8/12/16/20の4段階に統一されている
  （`docs/champions/changes_from_sv.md`）ため20を採用する。`src/jpoke/data/ps-champ-ja/moves.json` の
  値を正とすること
