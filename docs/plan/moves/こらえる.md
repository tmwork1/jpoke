# 計画書: こらえる

## 仕様要約

- タイプ: ノーマル / カテゴリ: 変化 / PP: 10 / 命中: なし（必中） / 対象: 自分 / 優先度: +4
- 使用ターン中に受けたあらゆるダメージによるひんしを1HPで耐える
- 効果範囲: 通常ダメージ技のみ（砂嵐・やけど等のターン経過ダメージは防げない）
- まもる系共通の連続使用失敗ルールが適用される（turn.md に「まもる系: 連続使用により失敗」と記載）
- 「こらえる」volatile を自分に付与し、`ON_MODIFY_MOVE_DAMAGE` で致死ダメージを HP 1 に補正する
- ターン終了時に volatile を解除する（まもる・みきり等と同じパターン）

## Priority 根拠

### `ON_STATUS_HIT`: priority 100
`docs/spec/turn.md` に `ON_STATUS_HIT` の内部 priority 表がない。既存の変化技ハンドラ（`まもる_apply_volatile_to_attacker`、`みきり_apply_volatile_to_attacker` 等）がすべて priority 100（デフォルト）を使用しているため、同一パターンで 100 を採用する。

### `ON_MODIFY_MOVE_DAMAGE`: priority 60
`docs/spec/turn.md` の `Event.ON_MODIFY_DAMAGE` セクションに明記：

```
| 60 | こらえる                |
| 60 | みねうち・てかげん      |
| 100 | がんじょう             |
| 100 | きあいのタスキ         |
| 100 | きあいのハチマキ       |
```

### `ON_TURN_END`: priority 100（デフォルト）
`docs/spec/turn.md` に `ON_TURN_END` の `まもる系` の記載はない。既存の `まもる_remove_volatile`・`かえんのまもり_remove_volatile` 等がすべてデフォルト priority を使用しているため、同一パターンで 100 を採用する。

def こらえる_apply_volatile_to_attacker(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:

### 3. `src/jpoke/handlers/volatile.py`

挿入位置: 五十音順「こ」行 — `こんらん_try_action` の前に挿入する（きゅうしょアップ の後）。

```python
def こらえる_endure(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """こらえる状態: 致死ダメージを HP 1 残しに補正する。

    ダメージが防御側の現在 HP 以上の場合、ダメージを hp - 1 に抑えて HP 1 を残す。
    """
    mon = ctx.defender
    if value >= mon.hp:
        value = mon.hp - 1
    return HandlerReturn(value=value)


def こらえる_remove_volatile(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    return remove_volatile(battle, ctx, value, volatile="こらえる")
```

## 注意点・エッジケース

### まもる系の連続使用失敗

`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` priority 30 に「まもる系: 連続使用により失敗」と記載されている。この失敗判定は `ON_CHECK_PROTECT` イベントで実装されており、`_run_protect` 内の `battle.events.emit(Event.ON_CHECK_PROTECT, ctx, True)` を通じて管理されている。

こらえるの volatile ハンドラ `こらえる_endure` は `ON_MODIFY_MOVE_DAMAGE` にのみ登録するため、まもる系連続失敗チェックは `ON_STATUS_HIT` での volatile 付与が前提となる。`apply_volatile_to_attacker` がすでに volatile 重複付与（= 前ターンもこらえるを使った場合）を防ぐ仕組みを持つかを impl が確認し、必要なら「こらえる状態中は技が失敗する」チェックを `ON_BEFORE_APPLY_MOVE` に追加する検討を行うこと。

現状の連続まもる失敗は `ON_CHECK_PROTECT` を通じて管理されているが、こらえるは `_run_protect` 経由でないため、このイベントを経由しない可能性がある。`apply_volatile_to_attacker` ヘルパーが返す値（`False` = すでに付与済み = 技失敗）で連続失敗が自然に実現できる場合はそのまま使えるが、確認が必要。

### `ON_MODIFY_MOVE_DAMAGE` のコンテキスト型

このイベントは `AttackContext` で発火される（`move_executor.py` の `_execute_hit` 参照）。volatile ハンドラは `subject_spec="defender:self"` を使い、`ctx.defender` がこらえる状態のポケモンとして参照される。引数型は `ctx: AttackContext` とすること（他の volatile 関数が `EventContext` を使うのと異なる点に注意）。

ただし、既存の `VolatileHandler` の関数シグネチャは `(battle, ctx, value)` で統一されており、型は `EventContext` のアノテーションになっている場合がある。実装時は `AttackContext` として参照しても動作するが、型アノテーションは既存パターンに合わせるか `AttackContext` に明示するかを impl が判断すること。

### 砂嵐・やけど等のターン経過ダメージへの非適用

こらえるは「技のダメージによるひんしのみを防ぐ」仕様のため、`ON_TURN_END` 時の砂嵐ダメージや状態異常ダメージ（`ON_MODIFY_NON_MOVE_DAMAGE` 経由）では発動しない。`こらえる_endure` は `ON_MODIFY_MOVE_DAMAGE` のみに登録するため、これは自然に満たされる。

### HP が 0 のとき

`value >= mon.hp` の条件はダメージが現在 HP 以上のときに補正する。HP がすでに 0 や負値の場合は通常ありえないが、`mon.hp` が最低 1 であることが前提（`battle.modify_hp` が 0 未満にならないよう管理されている）。

### 一撃必殺技への対応

こらえるは一撃必殺技（`label="ohko"`）も防げる（まもる系と同様）。`ON_MODIFY_MOVE_DAMAGE` でダメージを `hp - 1` に補正するため、一撃必殺の確定ダメージ（`ctx.defender.hp` を返す `ohko_modify_damage`）も HP 1 に抑えられる。ただし、`がんじょう_block_ohko` が `ON_BEFORE_APPLY_MOVE` で一撃必殺を完全無効にするのと異なり、こらえるは「ダメージを受けた上で HP 1 を残す」動作になる。

### `target="self"` の追加

既存の `data/move.py` エントリには `target` が未設定（デフォルト `"foe"`）のため、`target="self"` を明示的に追加する必要がある。まもる・みきりも `target="self"` を設定している（data/move.py 参照）。
