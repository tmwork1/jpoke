# 計画書: こらえる

## 仕様要約

- タイプ: ノーマル / カテゴリ: 変化 / PP: 12（champions基準。move_list.txt参照） / 命中: なし（必中） / 対象: 自分 / 優先度: +4
- 使用ターン中に受けたあらゆる攻撃技ダメージによるひんしを1HPで耐える
- 効果範囲: 通常ダメージ技のみ（砂嵐・やけど等のターン経過ダメージは防げない）
- まもる系共通の連続使用失敗ルールが適用される（turn.md に「まもる系: 連続使用により失敗」と記載）。
  既存のまもる・みきり等と同様、`flags={"protect"}` を付与し `まもる系_連続使用失敗チェック`
  （`Event.ON_TRY_MOVE_2`）を登録する
- 「こらえる」volatile を自分に付与し、`ON_MODIFY_MOVE_DAMAGE` で致死ダメージを HP 1 に補正する
- ターン終了時に volatile を解除する（まもる・みきり等と同じパターン）

## Priority 根拠

### `ON_TRY_MOVE_2`: priority 100（デフォルト）
`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` priority 30 に「まもる系: 連続使用により失敗」と記載が
あるが、既存実装（まもる・みきり・キングシールド・ニードルガード・トーチカ・スレッドトラップ・
かえんのまもり・ファストガード）はすべて `Event.ON_TRY_MOVE_2` のデフォルト priority（100）で
`まもる系_連続使用失敗チェック` を共用している。こらえるも同一パターンに合わせて priority を
明示指定せず、既存の兄弟技グループと歩調を合わせる（turn.md との厳密な整合はプロジェクト全体の
まもる系実装方針の課題であり、こらえる単体のスコープでは既存パターンを踏襲する）。

### `ON_STATUS_HIT`: priority 100
`docs/spec/turn.md` に `ON_STATUS_HIT` の内部 priority 表がない。既存の変化技ハンドラ（`まもる_apply`、
`みきり_apply` 等）がすべて priority 100（デフォルト）を使用しているため、同一パターンで 100 を採用する。

### `ON_MODIFY_MOVE_DAMAGE`: priority 95（turn.mdの60から要修正）
`docs/spec/turn.md` の `Event.ON_MODIFY_DAMAGE` セクションには次のように記載されている：

```
| 60 | こらえる                |
| 60 | みねうち・てかげん      |
| 100 | がんじょう             |
| 100 | きあいのタスキ         |
| 100 | きあいのハチマキ       |
```

しかし turn.md のこの表には、ハサミギロチン等の一撃必殺技が確定ダメージ（`ctx.defender.hp`）を
算出する `ohko_damage`（`Event.ON_MODIFY_MOVE_DAMAGE`, priority=90）が掲載されていない。
こらえるは「一撃必殺技を含むダメージ固定技も耐えることができる」仕様（docs/spec/moves/こらえる.md
参照）のため、`こらえる_endure` は `ohko_damage`（90）より後に実行して確定ダメージをHP1へ補正する
必要がある。turn.md の記載通り 60 にすると、`ohko_damage` が後から確定ダメージで value を
上書きしてしまい、こらえるでの一撃必殺耐えが機能しなくなる（レビューで実際に
`tests/items/test_item_na.py::test_ナゾのみ_一撃必殺技を耐えたときは発動する` 等が失敗することを確認した）。
そのため、`ohko_damage`（90）より後・がんじょう等（100）より前となる **95** を採用する
（既存実装の値を踏襲）。turn.md 側は本レビューでは修正せず、既知の齟齬として本計画書に明記する。

### `ON_TURN_END`: priority 100（デフォルト）
`docs/spec/turn.md` に `ON_TURN_END` の `まもる系` の記載はない。既存の `まもる_remove_volatile`・
`かえんのまもり_remove_volatile` 等がすべてデフォルト priority を使用しているため、同一パターンで
100 を採用する。

### 3. `src/jpoke/handlers/volatile.py`

```python
def こらえる_endure(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """こらえる状態: 致死ダメージを HP 1 残しに補正する。

    ダメージが防御側の現在 HP 以上の場合、ダメージを hp - 1 に抑えて HP 1 を残す。
    すでに HP1 の場合は実際のダメージが0になるが、攻撃自体は命中しているため
    「攻撃を無効化した」扱いにはならない（きあいパンチ不発・ダメおし威力2倍の対象）。
    実ダメージ0だと move_executor.py 側で hits_taken の加算がスキップされるため、
    ここで明示的に加算する（みねうち_modify_damage と同じパターン）。
    """
    mon = ctx.defender
    if value >= mon.hp:
        if mon.hp <= 1:
            mon.hits_taken += 1
        value = mon.hp - 1
    return HandlerReturn(value=value)


def こらえる_remove_volatile(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    return remove_volatile(battle, ctx, value, volatile="こらえる")
```

（実装済み）

## 注意点・エッジケース

### まもる系の連続使用失敗（解決済み）

`まもる系_連続使用失敗チェック`（`handlers/move_status.py`）は `mon.executed_move.has_flag("protect")`
を見て、直前ターンに守る系技を成功させていた場合に今ターンの守る系技を失敗させる。こらえるにも
`flags={"protect"}` と `Event.ON_TRY_MOVE_2` ハンドラを追加することで、まもる・みきり等の兄弟技と
同様に連続使用失敗が機能するようにした。

なお、実際の対戦仕様（`src/jpoke/data/ps-champ-ja/moves.json`）では連続使用のたびに成功率が1/3ずつ低下する
確率モデルだが、本プロジェクトは `docs/spec/volatiles/まもる.md` の方針に合わせ「直前ターン成功で
次ターン必ず失敗」という簡略化モデルを採用する（プロジェクト全体のまもる系実装方針であり、
こらえる単体で確率モデルを実装することはしない）。

### `ON_MODIFY_MOVE_DAMAGE` のコンテキスト型

このイベントは `AttackContext` で発火される（`move_executor.py` の `_execute_hit` 参照）。volatile
ハンドラは `subject_spec="defender:self"` を使い、`ctx.defender` がこらえる状態のポケモンとして
参照される。`こらえる_endure` の引数型は `ctx: AttackContext` になっている（実装済み）。

### 砂嵐・やけど等のターン経過ダメージへの非適用

こらえるは「技のダメージによるひんしのみを防ぐ」仕様のため、`ON_TURN_END` 時の砂嵐ダメージや
状態異常ダメージ（`ON_MODIFY_NON_MOVE_DAMAGE` 経由）では発動しない。`こらえる_endure` は
`ON_MODIFY_MOVE_DAMAGE` のみに登録するため、これは自然に満たされる。

### 一撃必殺技への対応

こらえるは一撃必殺技（`flags={"ohko"}`）も防げる（まもる系と同様）。`ON_MODIFY_MOVE_DAMAGE` で
ダメージを `hp - 1` に補正するため、一撃必殺の確定ダメージ（`ctx.defender.hp` を返す
`ohko_modify_damage`）も HP 1 に抑えられる。ただし、`がんじょう_block_ohko` が
`ON_BEFORE_APPLY_MOVE` で一撃必殺を完全無効にするのと異なり、こらえるは「ダメージを受けた上で
HP 1 を残す」動作になる。

### `target="self"` の追加

`data/moves/move_ka.py` のエントリに `target="self"` を明示的に設定している（デフォルトは
`"foe"` のため必須）。まもる・みきりも `target="self"` を設定している。
