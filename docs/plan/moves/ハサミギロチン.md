# 計画書: ハサミギロチン

## 仕様要約

（`docs/spec/moves/ハサミギロチン.md`、`docs/spec/moves/_ohko.md` 参照）

* ノーマルタイプ・物理、威力なし（一撃必殺技）、命中30固定、PP8（`docs/champions/move_list.txt`
  準拠。本家Gen9基準は5）、接触技。
* 命中した場合、対象のHPを0にする。反動・追加効果・急所判定なし。
* ノーマルタイプ技であるため、ゴーストタイプの対象には既存のタイプ相性判定により無効化される
  （ハサミギロチン専用の実装は不要）。
* 一撃必殺技共通の無効化条件（がんじょう・きあいのタスキ等での生存、まもる等でのブロック、
  はりきりの命中率ペナルティ対象外、おうじゃのしるし等の追加効果ハンドラ対象外）は
  既存の `flags={"ohko"}` を参照する各特性・アイテムハンドラで共通対応済み。

## 既存実装の状況

`data/moves/move_ha.py` の `ハサミギロチン` エントリは既に実装済みで、`ha.ohko_damage`
（`handlers/move_attack.py`）を `Event.ON_MODIFY_MOVE_DAMAGE` に `priority=90` で登録する、
`つのドリル`・`じわれ`・`ぜったいれいど` と同一パターンを踏襲している。
レビューの結果、`pp=5`（Gen9本家基準）のまま実装されていたバグを発見し、
`docs/champions/move_list.txt` の値に合わせて `pp=8` に修正した。それ以外の実装・登録内容に
誤りはなかった。

## イベント選定の根拠

* **確定ダメージの適用**: `Event.ON_MODIFY_MOVE_DAMAGE`、`priority=90`。
  `docs/spec/turn.md` に個別記載のないダメージ計算内部イベントだが、既存の `つのドリル`・
  `じわれ`・`ぜったいれいど` と同一の `ha.ohko_damage` ハンドラ・同一 priority を
  用いることで、`きあいのタスキ`（`item.py` の `きあいのタスキ_survive_ohko`）等の生存ハンドラとの
  実行順を素早さに依存させず、常に一撃必殺技側が先に確定ダメージを設定してから生存判定を
  行う既存の合意済み設計を維持する。

## ハンドラ構成

| 対象 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| ハサミギロチン（技） | `Event.ON_MODIFY_MOVE_DAMAGE` | 90 | `attacker:self`（`ha.ohko_damage` は既存共通ハンドラ） | 対象のHPと同値を確定ダメージとして設定する |

```python
"ハサミギロチン": MoveData(
    type="ノーマル",
    category="physical",
    pp=8,
    power=0,
    accuracy=30,
    flags={"ohko", "contact"},
    handlers={
        Event.ON_MODIFY_MOVE_DAMAGE: h.MoveHandler(
            ha.ohko_damage,
            priority=90,
        ),
    }
),
```

## リーサル計算

`docs/progress/move.md` の該当行は「リーサル実装: n/a」「リーサルテスト: n/a」のため対象外。

## テスト

* タイプ・分類・威力・命中率・PP・フラグが仕様通りであること
  （`test_ハサミギロチン_タイプ分類命中率PPが仕様通り`）。
* ゴーストタイプの対象には無効であること（`test_ハサミギロチン_ゴーストタイプに無効`）。
* 命中したとき、対象のHPが0になること（`test_ハサミギロチン_命中時は相手を一撃で倒す`）。
* 外した時にダメージを与えないこと・タイプ相性で無効化されることは、`つのドリル` を代表技とした
  `tests/moves_attack/test_move__group.py` の `test_一撃必殺技_*` 系テストで
  `flags={"ohko"}` 共通ハンドラとして併せてカバーされる。
* がんじょう・きあいのタスキ等のアイテム/特性との組み合わせは `tests/items/`・
  `tests/abilities/` 側の既存テストでカバーする。

## 注意点・エッジケース

* ゴーストタイプへの無効化は、ノーマルタイプ技全般に共通する既存のタイプ相性判定に
  委譲しており、ハサミギロチン専用の特別なハンドラは実装しない。
* `docs/champions/move_list.txt` ではPPが8/12/16/20の4段階に統一されているため、
  一撃必殺技グループ（つのドリル・じわれ・ぜったいれいど）も同様に本家基準の
  PP5からPP8への移行漏れが疑われる。ただしそれらは本レビューの対象エントリ外のため、
  今回は修正せず記録のみ残す。
