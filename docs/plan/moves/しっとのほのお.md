# しっとのほのお 実装計画書

## 仕様要約（docs/spec/moves/しっとのほのお.md 参照）

* タイプ: ほのお / 分類: 特殊 / 威力: 70 / 命中: 100 / PP: 5
* 対象: 相手1体（シングル） / 優先度: 0 / 接触なし
* 効果: 追加効果として、そのターンに能力ランクが上がった相手ポケモンを100%の確率で
  『やけど』状態にする。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | そのターンにランクが上がった相手を『やけど』状態にする |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def しっとのほのお_apply_burn_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """しっとのほのおの追加効果: そのターンにランクが上がった相手をやけど状態にする。"""
    if not ctx.defender.stat_raised_this_turn:
        return HandlerReturn(value=value)
    return apply_ailment_to_defender(battle, ctx, value, ailment="やけど")
```

`ctx.defender.stat_raised_this_turn` は `Pokemon.reset_turn_state()` でターン開始時に
`False` にリセットされ、`Battle.modify_stats()` で実際にランク補正が発生した場合にのみ
`True` になる（みわくのボイスと共有するフラグ）。この汎用フラグにより、能力上昇の発生源
（技・特性・道具）を問わず一律で判定できる。

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を
経由するため、ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は汎用ハンドラ側で
処理される。また `ailment_manager.apply()` 内でほのおタイプの免疫・りんぷん・おんみつマント
によるやけど無効化も既存の共通処理で吸収されるため、本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（状態異常付与）は既存の同種ハンドラ（したでなめる・
シャカシャカほう・みわくのボイス 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）の
パターンに従う。個別の priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **`stat_raised_this_turn` フラグの共有**
   - みわくのボイスと同一フラグを使用する。`Battle.modify_stats()` で実際にランクが
     変化した場合にのみ立ち、`reset_turn_state()` でターン開始時にリセットされる。
   - バトンタッチによる能力上昇の引き継ぎ、へんしんによるコピーはランク補正自体が
     発生しないため、このフラグは立たない（`modify_stats` を経由しないため対象外）。
2. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・やけど付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
3. **テスト方針**
   - 事前に `battle.modify_stats(defender, {"atk": 1}, source=defender)` で
     `stat_raised_this_turn` を `True` にしてからしっとのほのおを実行し、やけどが
     付与されることを確認する。
   - ランクが上がっていない場合はやけどが付与されないことを確認する。
   - 既存テスト（`tests/moves_attack/test_move_sa.py`）で両ケースとも実装済み。
