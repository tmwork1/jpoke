# 攻撃技（連続技系）実装計画

## 仕様要約

1回の技選択で複数回の攻撃判定を行う技群。

- **通常型（2〜5回）**: 命中判定は1発目のみ。ヒット回数はランダム（2回:37.5%/3回:37.5%/4回:12.5%/5回:12.5%）
- **固定回数型（2回/3回）**: 命中判定は1発目のみ。常に指定回数命中
- **毎回命中判定型**: ヒットごとに独立した命中判定を行う。外れたらそこで終了
- **ネズミざん（最大10回）**: 毎回命中判定、外れるまで最大10回
- **トリプルアクセル（最大3回）**: 毎回命中判定、威力が20→40→60と増加
- **スケイルショット（2〜5回）**: 全ヒット後に攻撃者のぼうぎょ-1・すばやさ+1
- **ドラゴンアロー（2回固定）**: シングルバトルでのみ2回ヒット（この実装はシングル専用なので常に2回固定として扱う）

各ヒットは独立したダメージ計算・急所判定・追加効果判定を持つ。

## ハンドラ構成

### 通常型・固定型（追加効果なし）

追加効果がない技はハンドラ不要。`data/move.py` の `multi_hit` フィールドを正しく設定するだけで動作する。

対象: スイープビンタ、つららばり、ダブルアタック、ダブルウイング、ツインビーム、ドラゴンアロー、ネズミざん、ミサイルばり、ボーンラッシュ、トリプルアクセル、ロックブラスト、みずしゅりけん

### スケイルショット（ランク変化あり）

| イベント | Priority | subject_spec | 役割 |
|---|---|---|---|
| `Event.ON_DAMAGE` | 100 | `attacker:self` | 全ヒット後（技終了時）にぼうぎょ-1・すばやさ+1 |

**注意**: `turn.md` の `Event.ON_DAMAGE` Priority 100 に「スケイルショットによる能力変化」として明記されている。ただし `ON_DAMAGE` は1ヒットごとに発火するため、**最終ヒットのときだけ**発動させる必要がある。`ctx.hit_index == ctx.hit_count` で最終ヒットを判定する。

代替案として `Event.ON_MOVE_END` を使う方法もあるが、turn.md の記載に従い `ON_DAMAGE` を使う。`ON_DAMAGE` の priority 100 はすでに多くのハンドラが存在するため、この priority で登録する。

---

## Priority 根拠

- **`Event.ON_DAMAGE` priority 100**: `docs/spec/turn.md` の `Event.ON_DAMAGE` テーブルに「Priority 100 | スケイルショットによる能力変化」として明示されている。
- **ハンドラ不要の技**: `multi_hit` フィールドは `move_executor.py` の `_resolve_hit_count` / `_resolve_hit_power` で内部処理されるため、外部ハンドラは不要。

### 2. `handlers/move_attack.py` への関数追加

スケイルショット専用ハンドラを追加する。他技はハンドラ不要。

```python
def スケイルショット_apply_stat_change(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """スケイルショット: 最終ヒット後に攻撃側のぼうぎょ-1・すばやさ+1。"""
    if ctx.hit_index != ctx.hit_count:
        return HandlerReturn(value=value)
    battle.modify_stats(ctx.attacker, {"B": -1, "S": 1}, source=ctx.attacker)
    return HandlerReturn(value=value)
```

**subject_spec**: `MoveHandler` のデフォルト（`attacker:self`）を使用する。技ハンドラは `skip_subject_check=True` のため subject_spec は照合されないが、慣例に従いデフォルトのまま使う。

**登録イベント**: `Event.ON_DAMAGE`（turn.md 記載の Priority 100 に対応）

## 注意点・エッジケース（review-test 向け）

### check_hit_each_time フラグの二重管理

`MoveData.multi_hit["check_hit_each_time"]` と `MoveData.flags` の `"check_hit_each_time"` は別々に管理されている。`move_executor.py` は `ctx.move.has_flag("check_hit_each_time")` のみを参照するため、`multi_hit["check_hit_each_time"]: True` だけでは毎回命中判定が行われない。**両方に設定が必要**。

「トリプルキック」（`data/moves/move_ta.py`）は `flags={"contact", "check_hit_each_time"}` が設定済みで、この二重管理漏れは解消されている。

### スケイルショット: ランク変化の発動タイミング

`ctx.hit_index == ctx.hit_count` で最終ヒット判定を行う。途中で相手がひんしになった場合（`ctx.fainted` が True）、`move_executor.py` がその時点でループを抜けるため、最終ヒット（＝実際の最後のヒット）でランク変化が発動する。意図通りの動作になる。

### スケイルショット: 外れた場合

`check_hit_each_time: False` のため、1発目が外れたら技自体が失敗し `ON_DAMAGE` は発火しない。ランク変化は起きない。

### ドラゴンアロー: シングルバトル専用の2回ヒット

仕様書には「攻撃できる相手が1体のときだけ連続攻撃になる」とある。シングルバトル専用実装のため、常に `min=2, max=2` として扱う。ダブルバトルは実装対象外。

### ネズミざん: 最大10回の命中判定

`min=10, max=10` かつ `check_hit_each_time: True` のため、10回すべて命中判定が走る。実質「当たり続ける限り最大10回」の動作になる。

### トリプルアクセル: `power_sequence` の使われ方

`power_sequence=(20, 40, 60)` の場合、`move_executor.py` の `_resolve_hit_power` が `hit_index - 1` をインデックスとして参照する。1発目=20、2発目=40、3発目=60。途中で外れた場合はそこで止まる。

### ミサイルばり・ロックブラスト: labels 確認

ミサイルばり: labels なし（直接攻撃フラグなし）。
ロックブラスト: `labels=["bullet"]`（弾の技フラグ）。
どちらも変更不要。

### テクニシャン特性との相互作用

連続技の各ヒット威力が60以下ならテクニシャンで威力1.5倍。これは `damage_calculator.py` 側で処理されるため、連続技ハンドラ側での対応は不要。

### スキルリンク特性との相互作用

スキルリンクは `Event.ON_MODIFY_HIT_COUNT` で常に最大回数にする（別途 `docs/plan/abilities/スキルリンク.md` で計画済み）。連続技ハンドラ側での対応は不要。
