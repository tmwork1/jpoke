# 計画書: ミラーショット

## 仕様要約（docs/spec/moves/ミラーショット.md 参照）

- はがねタイプ / 特殊 / 威力65 / 命中85 / PP10 / 対象: 1体選択 / 優先度0 / 接触なし
- 追加効果として、30%の確率で相手の命中率を1段階下げる。

## 既存実装との整合確認

### handlers/move_attack.py: `ミラーショット_lower_acc`（既存）

```python
def ミラーショット_lower_acc(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"accuracy": -1}, chance=0.3)
```

- `modify_defender_stats` ヘルパーは内部で `battle.resolve_secondary_chance` を経由するため、
  「追加効果」としてちからずく・おうじゃのしるし等の影響を正しく受ける
  （仕様書の「追加効果: あり」と一致）。
- 同系統技のどろかけ（100%）・どろばくだん（30%）等と同一ヘルパーを使用しており、
  実装パターンとして問題なし。
- イベント: `Event.ON_DAMAGE_HIT`。同種の能力ランク低下追加効果技（どろかけ・どろばくだん・
  ナイトバースト等）がすべて `Event.ON_DAMAGE_HIT` を使用しており、`docs/spec/turn.md`
  に本イベントの記載はないが（ダメージ計算内部処理のため未掲載）、既存の同種ハンドラとの
  整合を優先し、priority省略時のデフォルトのままとする。
- subject_spec: `MoveHandler` デフォルト（`attacker:self`）。技ハンドラは
  `skip_subject_check=True` でコンテキストを直接参照する実装のため、defender側への
  効果であっても既存の同種ハンドラ（どろかけ等）と同じくデフォルトのまま省略してよい。

### data/moves/move_ma.py（既存・変更不要）

```python
    "ミラーショット": MoveData(

```

- `pp=10` は `docs/champions/move_list.txt` に記載がないため、Gen9本家PPをそのまま採用する
  （過去の ずつき レビュー前例に準拠）。仕様書・進捗表とも一致しており修正不要。
- `flags={"secondary_effect"}` は「追加効果あり」の仕様と一致（直接攻撃ではないため
  `contact` は不要で正しい）。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `critical_rank` 省略時のデフォルトは `0`（通常の急所判定）→ 仕様と一致。

修正不要。既存実装は仕様書と完全に一致している。

## リーサル計算

対象外（`docs/progress/move.md` の「ミラーショット」行はリーサル実装/リーサルテストともに n/a）。
命中率を下げる効果は相手の命中判定にのみ影響し、確定数計算（HP分布）には影響しないため。

## 注意点・エッジケース

- 命中率ランク低下は「相手の命中率」であり、自分の回避率ではない点に注意
  （`modify_defender_stats` で `stats={"accuracy": -1}` を defender に適用しており、正しい）。
- 発動確率は30%であり、100%のどろかけとは異なる点に注意（`chance=0.3` の明示指定で対応済み）。
