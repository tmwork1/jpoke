# 実装計画書: ハードプラント

## 仕様

- タイプ: くさ / カテゴリ: 特殊 / 威力: 150 / 命中率: 90 / PP: 8（champions基準。move_list.txtにより8に統一）
- 対象: defender
- 効果: 技を命中させて成功した次のターン、使用者は反動で行動できなくなる（「リチャージ」揮発状態）。
  がんせきほう・ギガインパクト・はかいこうせん・ブラストバーン・ハイドロカノンと共通の効果であり、
  既存の共用ハンドラ・共用揮発状態「リチャージ」をそのまま利用する（新規実装不要）。

詳細は `docs/spec/moves/ハードプラント.md` を参照。

## イベント・Priority

| イベント | Priority | subject_spec | 役割 |
|----------|----------|-------------|------|
| `ON_DAMAGE_HIT`（technique側） | 100（デフォルト） | `attacker:self` | 命中後、使用者に「リチャージ」揮発状態を付与 |
| `ON_TRY_ACTION`（volatile側、共通） | 10 | `attacker:self` | リチャージ状態中、行動をブロックし状態を解除 |

**根拠**: `ON_DAMAGE_HIT` / `ON_TRY_ACTION` は `turn.md` に独立エントリなし。
同じ「リチャージ」系統の技（がんせきほう・ギガインパクト・はかいこうせん・ブラストバーン・ハイドロカノン）が
共通で使用する `handlers/move_attack.py` の `リチャージ_apply`（`ON_DAMAGE_HIT`、priorityデフォルト）と
`data/volatile.py` の「リチャージ」揮発状態（`ON_TRY_ACTION`、priority=10）を流用するため、同パターンを採用。

## 実装

`handlers/move_attack.py` の `リチャージ_apply`（共用関数、新規実装不要）を
`ON_DAMAGE_HIT` に登録するのみ。

### data/moves/move_ha.py

```python
"ハードプラント": MoveData(
    type="くさ",
    category="special",
    pp=8,
    power=150,
    accuracy=90,
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.リチャージ_apply,
        )
    }
),
```

## 注意事項

- `リチャージ_apply` は `apply_volatile_to_attacker` を介して `battle.volatile_manager.apply` を呼び出す共用関数。
  がんせきほう・ギガインパクト・はかいこうせん・ブラストバーン・ハイドロカノンと同一実装のため、
  ハードプラント専用のハンドラ関数は追加しない。
- まもるで防がれた場合・命中しなかった場合・特性等で無効化された場合は `ON_DAMAGE_HIT` が発火しないため、
  リチャージ状態は付与されない（`apply_volatile_to_attacker` 側の追加ガードは不要）。
- リチャージ状態のポケモンが交代すると、揮発状態は自動的に解除される（`volatile_manager` の交代時クリア処理に準拠）。
- PPは第九世代本家基準では5だが、`src/jpoke/data/ps-champ-ja/moves.json` により本プロジェクトでは8に統一されている。
