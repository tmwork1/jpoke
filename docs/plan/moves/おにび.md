# おにび 実装計画書

## 仕様要約

- 相手を **やけど** 状態にする変化技
- 命中率 85%、ほのおタイプへは無効
- 失敗条件: 対象がすでに状態異常 OR ほのおタイプ

## 実装方針

やけど付与は `ailment_manager.apply(defender, "やけど")` で行う。
おにびは **やけどを付与するトリガー** の部分だけを実装する。

### ハンドラ構成

| イベント | Priority | 役割 |
|----------|----------|------|
| `ON_BEFORE_APPLY_MOVE` | 130 | 失敗チェック：状態異常 OR ほのおタイプなら失敗 |
| `ON_STATUS_HIT` | 100（デフォルト） | やけど状態を相手に付与 |

### Priority 根拠

- `ON_BEFORE_APPLY_MOVE priority=130`：`docs/spec/turn.md` に「状態異常にする技: すでに同じ/別の状態異常」（priority=130）および「ほのおタイプ: やけど無効」（priority=130）が明記されている

### subject_spec

- `ON_BEFORE_APPLY_MOVE`: `attacker:self`（MoveHandlerデフォルト）
- `ON_STATUS_HIT`: `attacker:self`（MoveHandlerデフォルト）

どちらも変化技フロー（`EventContext`）の中で `ctx.attacker`（使用者）・`ctx.defender`（対象）を参照する。

## 実装箇所

### handlers/move_status.py（五十音順: おにびは「お」行）

```python
def おにび_can_apply(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    mon = ctx.defender
    if mon.ailment or "ほのお" in mon.types:
        battle.add_event_log(ctx.attacker, LogCode.MOVE_FAILED,
                             payload={"reason": "おにび"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def おにび_apply_ailment(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="やけど")
```

### data/move.py（「おにび」MoveData にハンドラを追加）

```python
"おにび": MoveData(
    type="ほのお",
    category="変化",
    pp=15,
    accuracy=85,
    handlers={
        Event.ON_BEFORE_APPLY_MOVE: h.MoveHandler(
            hs.おにび_can_apply,
            priority=130,
        ),
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.おにび_apply_ailment,
        ),
    }
),
```

## 注意点・エッジケース（review-test 向け）

1. **みずのベール特性**: `ailment_manager.apply` 内で priority=140 のチェックが働くため、ハンドラ側での対応不要。
2. **しんぴのまもり・みがわり**: `apply_ailment_to_defender` の内部で既存ガードが働くため、追加チェック不要。
3. **ダブルタイプのほのお**: `mon.types` に `"ほのお"` が含まれるかを `in` でチェックするため、単タイプ・複合タイプどちらも正しく判定できる。
4. **命中率 85%**: `accuracy=85` を MoveData に設定する（既存エントリへの追加）。
