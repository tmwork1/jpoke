# クロスポイズン 実装計画書

## 仕様要約（docs/spec/moves/クロスポイズン.md 参照）

* タイプ: どく / 分類: 物理 / 威力: 70 / 命中: 100 / PP: 20
* 対象: 相手1体 / 優先度: 0 / 接触あり（直接攻撃）
* 急所ランク+1（急所に当たりやすい）。
* 追加効果として、10%の確率で相手を『どく』状態にする。
* 「切る技」（`flags` に `"slash"`）。特性きれあじの対象。

## 現状の実装状況

`src/jpoke/data/moves/move_ka.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。
仕様と一致していることを確認した（修正不要）。

```python
"クロスポイズン": MoveData(
    type="どく",
    category="physical",
    pp=20,
    power=70,
    accuracy=100,
    critical_rank=1,
    flags={"contact", "slash", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.クロスポイズン_apply_poison_to_defender,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手を『どく』状態にする |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def クロスポイズン_apply_poison_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="どく", chance=0.1)
```

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を
経由するため、ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は汎用ハンドラ側で処理される。
また `flags={"slash"}` により、特性きれあじ所持ポケモンが使用すると威力が1.5倍になる
（`handlers/ability.py` の `きれあじ_modify_power` が既存実装のまま適用される）。
`critical_rank=1` は `core/move_executor.py` の急所判定ロジックが参照するため、追加実装は不要。
本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（状態異常付与）は既存の同種ハンドラ（どくづき・どくどくのキバ 等）と
同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。`docs/spec/turn.md` には
未掲載のイベント（ダメージ計算内部）のため、個別の priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **急所ランク+1の網羅**
   - `critical_rank=1` の効果自体を確認するテストが既存テストファイルに存在しなかった
     （どくが発動するテストのみ）。きりさく・つじぎり・クロスチョップ等の既存パターンに合わせ、
     乱数0で急所発生・乱数0.5で急所なしとなる2テストの追加が必要。
2. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・どく付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
3. **きれあじとの相互作用**
   - `flags={"slash"}` により、きれあじ所持ポケモンが使用すると威力が1.5倍になる。
     既存の きれあじ テストファイル側で他の切る技と同様の網羅がされていれば個別追加は不要。
4. **どく状態異常の重複・免疫判定**
   - どくタイプ・はがねタイプへの無効化、めんえき等の免疫特性による無効化は
     `ailment_manager.apply` 側の共通実装が保証しており、本技固有のガードは不要。
5. **テスト方針**
   - `secondary_chance` を1に固定し、どく付与を確認するテストが既存
     （`test_クロスポイズン_どくが発動する`）。
   - 急所ランク+1のテスト（乱数0で急所発生・乱数0.5で急所なし）を追加する。
