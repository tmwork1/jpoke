# 計画書: じばそうさ

## 仕様要約

- タイプ: でんき、変化、PP: 20、対象: 味方全員（シングルバトルでは自分のみ）、必中
- 特性が「プラス」または「マイナス」のポケモンの「ぼうぎょ」「とくぼう」ランクをそれぞれ1段階ずつ上げる
- 使用者自身の特性が「プラス」/「マイナス」の場合、自分にも効果がある（シングルバトルでは使用者＝対象）
- 特性が「プラス」/「マイナス」でないポケモンには効果がない（失敗する）
- いえき・かがくへんかガスによりプラス/マイナスが消えている場合も効果なし
- みがわり状態・まもる状態でも効果がある
- よこどり可（Zワザ時は「とくぼう1段階上昇」の付加効果）

出典: `docs/wiki/moves/じばそうさ.html`

```python
def じばそうさ_can_apply(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """じばそうさの失敗条件: 使用者の特性がプラス/マイナスでない場合は失敗させる。"""
    if ctx.attacker.ability.name not in ("プラス", "マイナス"):
        battle.add_event_log(ctx.attacker, LogCode.MOVE_FAILED,
                             payload={"reason": "じばそうさ"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def じばそうさ_modify_attacker_stats(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """じばそうさの効果: ぼうぎょ・とくぼうをそれぞれ1段階上げる。"""
    return modify_attacker_stats(battle, ctx, value, stats={"B": 1, "D": 1})
```

## 注意点・エッジケース

- シングルバトルでは対象は常に使用者自身のため、「味方全員」効果は使用者に対してのみ発動する
- 失敗条件チェックは `ON_BEFORE_APPLY_MOVE` で使用者の特性を確認する。いえき状態・かがくへんかガスにより特性が消えている場合は `ability.name` が変わるため、別途ガードは不要
- B と D の両方を1つの `stats` dict にまとめて `modify_attacker_stats` に渡せる
- どちらか一方のランクが最大（+6）の場合は、そのランクの上昇のみ失敗し、もう一方は成功する（マネージャー内部処理）
- ランクがすでに最大の場合の失敗判定は `turn.md` Priority 130 でマネージャー内部処理されるため、ハンドラ側にガード不要
- 対象が「ちくでん」「ひらいしん」「でんきエンジン」の場合はでんきタイプ技として無効化されるが、これは `ON_TRY_MOVE_1`（priority 110）で処理されるため、ハンドラ側に追加ガード不要
