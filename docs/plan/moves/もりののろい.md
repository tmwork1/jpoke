# 実装計画書: もりののろい

## 仕様要約

- タイプ: くさ / カテゴリ: 変化 / PP: 20 / 命中: 100 / 対象: 1 体選択
- 相手の「タイプ」に「くさ」タイプを追加する
- 相手がすでにくさタイプを持っている場合は失敗する

## 現状の実装状況

| 項目 | 状態 |
|------|------|
| `data/move.py` エントリ | 登録済み（accuracy=100、handlers なし） |
| `handlers/move_status.py` 関数 | 未実装 |

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_BEFORE_APPLY_MOVE` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手がすでにくさタイプなら失敗 |
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手にもりののろい状態を付与してくさタイプを追加 |

## Priority 根拠

- `ON_BEFORE_APPLY_MOVE` priority 100: `docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「ハロウィン等: すでに該当タイプ保持」が priority 100 として明記されている（もりののろいは「ハロウィン等」に含まれる）
- `ON_STATUS_HIT` priority 100（デフォルト）: 同種ハンドラと同じパターン

## タイプ追加の実装方針

ハロウィンと同じ設計方針を採用する。volatile「もりののろい」を付与し、`ON_CALC_DEF_TYPE_MODIFIER` でくさタイプの被弾倍率を適用する方式を推奨する。

**簡易案（直接変更）**: `mon.data.types` に `"くさ"` を append する方式。ハロウィンとの実装統一を図るならこちらも同じ方式で実装する。

## 実装箇所（推奨: volatile 方式）

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「も」行 — `みちづれ_apply_volatile_to_attacker` の後、「め」行を経て「も」行に挿入する。具体的には `めいそう_modify_attacker_stats` の後、`みがわり_*` 系の後の「も」行に配置する。

```python
def もりののろい_can_apply(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """もりののろいの使用条件チェック: 相手がすでにくさタイプなら失敗する。"""
    if ctx.defender.has_type("くさ"):
        battle.add_event_log(ctx.attacker, LogCode.MOVE_FAILED,
                             payload={"reason": "もりののろい"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def もりののろい_apply_volatile_to_defender(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """もりののろいの効果: 相手にもりののろい状態を付与してくさタイプを追加する。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="もりののろい")
```

## 注意点・エッジケース

- **ハロウィンと同じ実装方針で統一すること**
- volatile 方式の場合、「もりののろい」volatile を `data/volatile.py` に登録する必要がある
- `VolatileName` の Literal 型定義に `"ハロウィン"` と `"もりののろい"` を追加する必要があるか `utils/type_defs.py` を確認すること
- テラスタル中の相手への挙動を確認すること
- 交代後は volatile がリセットされるため、くさタイプ追加も消える
