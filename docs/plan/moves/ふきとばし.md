# 計画書: ふきとばし

## 仕様要約

`docs/spec/moves/ふきとばし.md` を参照。

- 変化技 / ノーマルタイプ / 優先度 -6 / PP 20 / 命中判定なし（必中）
- ダメージなし。トレーナー戦で相手を強制的に控えのポケモンとランダムに交代させる
- 風技（`wind` フラグ）のため、かぜのりに無効化される
- まもる・みきり・トーチカを貫通する（`unprotectable` フラグ）
- みがわり状態の相手にも効果がある（`bypass_substitute` フラグ）
- 特性きゅうばん・ばんけん・ねをはる状態で無効化される（`ON_TRY_BLOW` で判定）
- 特性かげふみ・ありじごく・じりょく、にげられない・バインド状態、フェアリーロック状態を
  無視して強制交代を発動する
- 相手に交代できる控えポケモンがいない場合は技自体が失敗する

## レビューで発見した既存実装の不具合

旧実装（レビュー前）は以下の誤りがあった。

1. `category="special"` になっていた（正しくは `category="status"`）。この誤りにより
   ダメージ計算処理（`roll_damage`）を経由してしまい、`ON_DAMAGE_HIT` も発火するため、
   特性ふうりょくでんきが誤って反応しうる状態だった。
2. `Event.ON_HIT`（ダメージ技専用の命中後イベント）に `blow` を登録していた。変化技である
   ため、正しくは `Event.ON_STATUS_HIT` に登録する必要がある。
3. `unprotectable` フラグが未設定だったため、まもる状態の相手に対して防がれてしまっていた
   （`Move.is_blocked_by_protect` は `unprotectable` フラグの有無で判定するため）。
4. `bypass_substitute` フラグが未設定だったため、みがわり状態の相手に対して無効化されて
   しまっていた（`みがわり_immune` は `MoveExecutor.check_hit_substitute` の判定結果を
   使うため）。
5. 相手に交代できる控えポケモンがいない場合の失敗判定が存在しなかった
   （`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` priority=100
   「ほえる・ふきとばし: 交代不可・野生」に対応する処理が未実装）。

これらはレビューで修正済み。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `Event.ON_BEFORE_APPLY_MOVE` | 30 | （デフォルト） | `on_blow_apply` — `ON_TRY_BLOW` を emit し、きゅうばん・ばんけん・ねをはる等で無効化されたか判定。無効ならイベントを停止 |
| `Event.ON_BEFORE_APPLY_MOVE` | 100 | （デフォルト） | `on_blow_check_switch_target` — 相手に交代できる控えポケモンがいない場合に技を失敗させる |
| `Event.ON_STATUS_HIT` | （デフォルト） | （デフォルト） | `blow` — 控えからランダムに1体選んで交代を実行 |

`on_blow_apply` と `blow` は `ほえる` と共用。`on_blow_check_switch_target` は本レビューで
新規追加した共通ハンドラ（`ほえる` 側でも将来的に流用可能）。

## Priority 根拠

- `docs/spec/turn.md` の `Event.ON_APPLY_MOVE`（実装上は `Event.ON_BEFORE_APPLY_MOVE`）節:
  - `30 | ほえる・ふきとばしの無効化` → `on_blow_apply` の priority=30
  - `100 | ほえる・ふきとばし: 交代不可・野生` → `on_blow_check_switch_target` の priority=100
- `docs/spec/moves/強制交代技.md` に「全強制交代技の優先度: -6」と明記されている
  （`MoveData.priority=-6`、行動順を決める `DomainEvent` とは別枠）。

## 実装箇所

### `src/jpoke/data/moves/move_ha.py`

```python
"ふきとばし": MoveData(
    type="ノーマル",
    category="status",
    pp=20,
    priority=-6,
    flags={"wind", "unprotectable", "bypass_substitute"},
    handlers={
        Event.ON_BEFORE_APPLY_MOVE: [
            h.MoveHandler(
                hs.on_blow_apply,
                priority=30,
            ),
            h.MoveHandler(
                hs.on_blow_check_switch_target,
                priority=100,
            ),
        ],
        Event.ON_STATUS_HIT: h.MoveHandler(hs.blow),
    }
),
```

### `src/jpoke/handlers/move_status.py`

`on_blow_check_switch_target` を新規追加（`on_blow_apply` の直後に配置）。
`blow` 自体の実装は変更なし（相手に交代できるポケモンがいるか否かは事前に
`on_blow_check_switch_target` で判定済みという前提）。

## 注意点・エッジケース（review-test 向け）

- **カテゴリと発火イベント**: `category="status"` でなければ `MoveExecutor._execute_hit` が
  ダメージ処理経路（`roll_damage` → `ON_MODIFY_MOVE_DAMAGE` → `ON_HIT` →
  `ON_DAMAGE_HIT`）に入ってしまう。`ふきとばし` は必ず `category="status"` かつ
  `Event.ON_STATUS_HIT` で登録すること。
- **まもる貫通**: `unprotectable` フラグが無いと `Move.is_blocked_by_protect` が
  `True` になり、まもる状態の相手に防がれてしまう。
- **みがわり貫通**: `bypass_substitute` フラグが無いと、状態変化技は
  `みがわり_immune`（`ON_BEFORE_APPLY_MOVE` priority=30）によって無効化されてしまう。
  `ほえる` は `sound` フラグがあるため自動的にみがわりを貫通するが、`ふきとばし` は
  音技ではないため明示的に `bypass_substitute` フラグが必要。
- **交代不可の失敗判定**: `blow` 自身の中で失敗ログを出すと、`MoveExecutor._execute_status_hit`
  側の自動フォールバックログと二重ログになる可能性を避けるため、失敗判定は
  `Event.ON_BEFORE_APPLY_MOVE`（priority=100）側の専用ハンドラで行い、`blow` 自体は
  「交代先が必ず存在する」前提で実装する。
- **交代先選択**: `blow` 関数は `get_forced_switch_commands` でランダム選択する実装になっており、
  仕様の「控えポケモンからランダム選択」に対応済み。
- **リーサル計算**: `docs/progress/move.md` の「リーサル実装」列は `n/a`
  （ダメージを与えない変化技のため、致死率計算の対象外）。
