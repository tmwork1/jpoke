# 実装計画: カウンター

## 仕様サマリ

- タイプ: かくとう / 分類: 物理 / 威力: -（固定ダメージ） / 命中率: 100 / PP: 20 / 優先度: -5
- 効果: そのターンに相手から受けた物理技のダメージを2倍にして同じ相手に返す
- 使用したターンに物理技のダメージを受けていない場合は失敗する
- 複数回の物理ダメージを受けていた場合（連続技等）は最後の1回分のダメージのみを参照する

詳細は `docs/spec/moves/カウンター.md` を参照。

## 現状の実装状況

`data/moves/move_ka.py` にすでに実装済み。

```python
"カウンター": MoveData(
    type="かくとう",
    category="physical",
    pp=20,
    power=0,
    accuracy=100,
    priority=-5,
    flags={"contact", "non_copycat"},
    handlers={
        Event.ON_TRY_MOVE_1: h.MoveHandler(
            ha.カウンター_can_use,
            subject_spec="attacker:self",
            priority=30,
        ),
        Event.ON_MODIFY_MOVE_DAMAGE: h.MoveHandler(
            ha.カウンター_modify_damage,
            subject_spec="attacker:self",
        ),
    },
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_TRY_MOVE_1` | 30 | `attacker:self` | そのターン物理ダメージを受けていない場合に失敗させる |
| `ON_MODIFY_MOVE_DAMAGE` | 100（デフォルト） | `attacker:self` | ダメージを「直近に受けた物理ダメージ×2」の固定値にする |

## Priority 根拠

### ON_TRY_MOVE_1 Priority 30

`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` テーブルに以下の記述がある：

```
| 30 | カウンター・ミラーコート: 適格ダメージ無 |
```

したがって priority=30 を採用する。

### ON_MODIFY_MOVE_DAMAGE Priority 100（デフォルト）

`docs/spec/turn.md` に `Event.ON_MODIFY_MOVE_DAMAGE` の個別記載はない（ダメージ計算内部のイベントのため）。
同種の固定ダメージ技である `ミラーコート` の実装が既存のデフォルト優先度（100）を用いているため、それに合わせる。

## 実装済み・修正済み内容（review時点）

### `Pokemon.last_physical_damage_received` / `last_special_damage_received` の意味修正

review前は `+=`（ターン中の物理/特殊ダメージ合計）として実装されていたが、以下の仕様に反していた：

> 使用したターンに複数回物理技のダメージを受けていた場合、最後に受けた物理技によるダメージを2倍にする。
> （連続技を受けた場合、最後の1回分のダメージだけを倍返しする）

`core/move_executor.py` の `_execute_hit` 内で `+=` を `=` に変更し、「直近の1ヒット分のダメージ」を
保持するよう修正した（`last_damage_received` と同様の「最後の値で上書き」方式に統一）。
この修正は `ミラーコート`（特殊版のカウンター）にも同様に影響する（共通コードのため）。

### `non_copycat` フラグの追加

`まねっこ`（第四世代以降）はカウンターをコピーできない仕様のため、`flags` に `non_copycat` を追加した
（`きょじゅうだん` 等の既存パターンに合わせる）。

## 注意点・エッジケース（review-test 向け）

### みがわりが受けたダメージ

みがわりが物理技を身代わりに受けた場合、`last_physical_damage_received` は更新されない
（みがわりへのダメージは `ctx.substitute_damage` 経由で処理され、`_execute_hit` の
`last_physical_damage_received` 更新処理を通らない）。このため、みがわり越しの物理技を受けた
だけの場合カウンターは失敗する（仕様通り）。

### 特殊技のみ受けた場合

`last_special_damage_received` のみが更新され、`last_physical_damage_received` は0のままなので
カウンターは失敗する（`ミラーコート` と役割が分離されている）。

### 連続技（複数回ヒットする技）を受けた場合

`_execute_hit` はヒットごとに呼ばれるため、`last_physical_damage_received` はヒットのたびに
上書きされる。最終的に残るのは最後のヒットのダメージのみとなり、仕様と一致する。

### 未対応のエッジケース

`docs/spec/moves/カウンター.md` の「既知の未対応エッジケース」を参照。相手の物理攻撃がHPを
実際には減少させなかった（きあいのハチマキ・ばけのかわ等でダメージが0に補正された）場合の
「本来は1ダメージを返す」という第五世代以降の仕様は、既存のダメージ処理パイプラインでは
「無効化された攻撃」と区別する手段がないため、対象外とする。
