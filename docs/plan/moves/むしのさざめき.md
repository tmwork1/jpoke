# 計画書: むしのさざめき

## 仕様要約

`docs/spec/moves/むしのさざめき.md` を参照。

* むし/特殊/威力90/命中100/PP10/対象1体/優先度0/非接触
* 音（sound）技
* 追加効果: 命中時10%の確率で相手の『とくぼう』ランクを1段階下げる
* まもるで防がれる、オウムがえし可、マジックコート・よこどり対象外
* ちからずく・てんのめぐみの対象技（`secondary_effect` フラグが必要）

## 現状の実装状況

`src/jpoke/data/moves/move_ma.py` に既に `MoveData` が登録済み、
`src/jpoke/handlers/move_attack.py` に `むしのさざめき_lower_defender_spd` が実装済み。

### 発見した不備

* `MoveData.flags` が `{"sound"}` のみで `secondary_effect` が欠落している。
  この結果、`ちからずく`（`ちからずく_boost` / `ちからずく_disable_secondary_effect`、
  `src/jpoke/handlers/ability.py`）が `ctx.move.has_flag("secondary_effect")` で
  判定しているため、むしのさざめきを使った場合に威力1.3倍化・追加効果無効化のどちらも
  発動しない不具合がある。
* ハンドラ本体（`stats={"spd": -1}, chance=0.1`）・イベント登録（`Event.ON_DAMAGE_HIT`）は
  仕様通りで問題なし。

## ハンドラ構成

| 技 | イベント | priority | subject_spec | 効果 |
|---|---|---|---|---|
| むしのさざめき | ON_DAMAGE_HIT | 100（デフォルト） | attacker:self（デフォルト） | 防御側とくぼう-1、10% |

### Priority 根拠

`docs/spec/turn.md` には `Event.ON_HIT`（479-489行目）と `Event.ON_DAMAGE`（497-539行目）の
priority 表があるが、コード上の `Event.ON_DAMAGE_HIT`（`ダメージ適用後` `追加効果発動`、
`src/jpoke/enums/event.py` 216行目）に対応するのは `turn.md` の `Event.ON_DAMAGE` セクション
（「10 | 追加効果（特殊なもの除く）」）と考えられる。

ただし、同種の「命中時10%等の確率で相手のランクを下げる」ハンドラ（きあいだま・サイコキネシス・
エナジーボール・だいちのちから 等、`src/jpoke/data/moves/*.py` 内の同パターン全件）は
すべて `Event.ON_DAMAGE_HIT` に `MoveHandler` のデフォルト priority（100）で登録されている。
本タスクはこの既存の同種ハンドラの慣例（priority=100）に合わせる。
turn.md上の priority=10 という記載と実装が乖離している点はこの技固有の問題ではなく、
このカテゴリの技全体に共通する既存の実装慣例のため、本レビューの対象外として現状を維持する。

## 実装箇所

### 1. handlers/move_attack.py（変更なし）

既存の `むしのさざめき_lower_defender_spd` をそのまま使用する。

```python
def むしのさざめき_lower_defender_spd(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spd": -1}, chance=0.1)
```

### 2. data/moves/move_ma.py（修正）

`flags` に `secondary_effect` を追加する。

```python
# 変更前
"むしのさざめき": MoveData(
    type="むし",
    category="special",
    pp=10,
    power=90,
    accuracy=100,
    flags={"sound"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.むしのさざめき_lower_defender_spd,
        )
    }
),

# 変更後
"むしのさざめき": MoveData(
    type="むし",
    category="special",
    pp=10,
    power=90,
    accuracy=100,
    flags={"sound", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.むしのさざめき_lower_defender_spd,
        )
    }
),
```

修正後は `python scripts/sort_data/sort_moves.py` を実行する（キー順序は変わらないため
差分は `flags` 行のみになる想定）。

## テスト方針

`tests/moves_attack/test_move_ma.py` に以下を追加する（`test_utils.py` の
`start_battle` / `run_move` / `secondary_chance` パラメータを使用、きあいだま・
ハイパーボイスの既存テストを参考にする）：

1. `secondary_chance=0.0` のとき、とくぼうランクが変化しないこと
2. `secondary_chance=1.0` のとき、とくぼうランクが1段階下がること
3. 相手にダメージを与えること（特殊むし技として機能すること）
4. 音技のため、みがわり状態の相手の本体にダメージが通ること
   （`battle.volatile_manager.apply(defender, "みがわり", hp=999)` を使用）
