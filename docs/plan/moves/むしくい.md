# 計画書: むしくい

## 仕様要約（docs/spec/moves/むしくい.md 参照）

- むしタイプ / 物理 / 威力60 / 命中100 / PP20 / 対象: 相手1体 / 優先度0 / 接触あり
- 相手が「バトルに効果のある」きのみを持っていれば、命中後にそのきのみを奪い、
  自分（攻撃者）が即座に消費してその効果を受ける。
- ついばむ（ひこうタイプ）と全く同じ追加効果を持つ姉妹技で、ハンドラを共有する。
- 追加効果ではないため、りんぷん・おんみつマント・ちからずくの影響を受けない。
- `docs/champions/move_list.txt:894` に記載あり（むし / 物理 / 60 / 100 / 20 / 接触あり / まもる有効）。

## 既存実装との差分

既存実装（`data/moves/move_ma.py` の `"むしくい"` エントリ、
`handlers/move_attack.py` の `むしくい_steal_and_use_berry`）は、以下2点で仕様と食い違っていたため修正した。

### 修正1: みがわりに防がれた場合もきのみを奪ってしまう不具合

仕様書（一次情報）:「対象のみがわり状態に防がれた場合、きのみを食べることはできない。」

修正前の実装は `defender.has_item()` のみを見ており、みがわりに阻まれたかどうかを
判定していなかった。このため、みがわり越しに命中した場合でも実体のきのみを奪ってしまっていた
（実機で再現・確認済み）。

`Event.ON_MODIFY_MOVE_DAMAGE` でみがわりがダメージを肩代わりすると `ctx.substitute_damage`
（`AttackContext` のフィールド）に肩代わり量が記録される
（`handlers/volatile.py` の `みがわり_block_damage`）。これを利用し、
`むしくい_steal_and_use_berry` の先頭で `ctx.substitute_damage > 0` の場合は
何もせず処理を打ち切るようにした。

### 修正2: ねんちゃく持ちをこの技でひんしにさせた場合に奪えない不具合

仕様書（一次情報、第五世代以降）:「対象が特性ねんちゃくである場合、きのみを食べることは
できない。ただし、ねんちゃくのポケモンをこの技でひんしにさせた場合はきのみを食べることができる。」

修正前の実装は `take_item` → `can_change_item` → `Event.ON_CHECK_ITEM_CHANGE` を経由し、
`ねんちゃく_prevent_item_change`（`handlers/ability.py`）が対象の生死に関係なく常に
奪取を阻止していた。

この仕様はむしくい・ついばむに固有の挙動であり、他の道具奪取技（ぬすむ・すりかえ等）には
適用されない（ねんちゃくは通常、生死に関係なく道具の奪取・交換・除去を防ぐ）。
そのため、`ねんちゃく_prevent_item_change` を無条件に変更するのではなく、
呼び出し元が明示的に指定した場合のみねんちゃくを無視できる仕組みを追加した。

* `core/context.py` の `EventContext` に `ignore_sticky_hold: bool = False` を追加。
* `core/item_manager.py` の `can_change_item` / `swap_items` / `take_item` に
  `ignore_sticky_hold` キーワード引数を追加し、`EventContext` に伝播させる
  （デフォルト `False` のため既存の全呼び出し元は影響を受けない）。
* `handlers/ability.py` の `ねんちゃく_prevent_item_change` で
  `ctx.dry_run or ctx.ignore_sticky_hold` の場合は阻止しないよう変更。
* `むしくい_steal_and_use_berry` から
  `battle.item_manager.take_item(defender, ignore_sticky_hold=defender.fainted)` を呼び出す。
  `Event.ON_HIT` の時点で `defender.fainted` は既に確定しているため、
  「この技で対象をひんしにさせた場合」の判定として利用できる。

なお、ねんちゃく以外にアイテム奪取を阻止する特性（マルチタイプ・ARシステム）は
いずれも非きのみ専用アイテム（メガランチャー等の対象ではないプレート・ドライブ類）を
対象としており、`defender.item.is_berry()` が真の時点でこれらが競合することはない。

## ハンドラ構成

| Event | subject_spec | 関数 | 役割 |
|---|---|---|---|
| ON_HIT | attacker:self / defender:self | `むしくい_steal_and_use_berry` | defender がきのみを持っていれば奪取し、attacker が即座に消費する |

### handlers/move_attack.py: `むしくい_steal_and_use_berry`（修正）

```python
def むしくい_steal_and_use_berry(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """むしくい・ついばむ: 相手のバトルに効果のあるきのみを奪って自分が消費する。

    みがわりに防がれた場合はきのみを食べられない（実体への攻撃ではないため）。
    take_item はねんちゃくチェックを内包し、attacker がアイテムを
    持っている場合は失敗して何もしない。ただし、この技で対象をひんしにさせた場合は
    ねんちゃくによる阻止を無視して奪取できる（第五世代以降の仕様）。
    """
    if ctx.substitute_damage > 0:
        return HandlerReturn(value=value)
    defender = ctx.defender
    attacker = ctx.attacker
    if not defender.has_item() or not defender.item.is_berry():
        return HandlerReturn(value=value)
    # take_item でdefenderのきのみをattackerに移す（ねんちゃく等で失敗する場合あり）
    if not battle.item_manager.take_item(defender, ignore_sticky_hold=defender.fainted):
        return HandlerReturn(value=value)
    # attackerがきのみを得たので効果を発動して消費する
    battle.item_manager.force_trigger_berry(attacker)
    return HandlerReturn(value=value)
```

- イベント: `Event.ON_HIT`（ダメージ発生後の処理。みがわりに被弾しても発動する。
  docs/spec/turn.md の `## Event.ON_HIT` 参照）。
- 優先度: デフォルト100（`MoveHandler` のデフォルト値）。docs/spec/turn.md の
  `Event.ON_HIT` セクションに個別記載はないが、同一イベントの既存パターン
  「100 | なげつける使用者のアイテム消費」（アイテム消費系の効果は優先度100）に倣う。
  新規のpriority指定は不要（既存のデフォルト値のまま変更なし）。
- subject_spec: `MoveHandler` デフォルト（`attacker:self`）のまま。

### handlers/ability.py（修正）

```python
def ねんちゃく_prevent_item_change(battle: Battle, ctx: EventContext, value: bool) -> HandlerReturn:
    """ねんちゃく特性: 相手から受けるアイテム交換・奪取・除去を防ぐ。

    はたきおとすの威力補正判定（dry_run=True）では、除去自体は行われないため
    特性を発動・表示させず、威力補正の対象として扱う（除去は別途阻止される）。
    ignore_sticky_hold（むしくい・ついばむが対象をひんしにさせた場合等、
    第五世代以降の仕様）が指定されている場合も阻止しない。
    """
    if ctx.dry_run or ctx.ignore_sticky_hold:
        return HandlerReturn(value=value)
    if ctx.source != ctx.target:
        _announce_ability_triggered(battle, ctx.target)
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

## 既知の制限（今回未対応・要フォローアップ）

一次情報には「チイラのみなどのピンチきのみ、オボンのみなどの回復きのみ、アッキのみは、
消費される前に食べることができる」とある。これは、相手自身のHP閾値ベースのきのみ
（例: オボンのみ）が自動発動するより先に、むしくいの奪取が優先されるべきという仕様である。

しかし本フレームワークでは、HP閾値ベースのきのみの発動チェック（`Event.ON_HP_CHANGED`）は
`core/status_manager.py` の `modify_hp` 内でダメージ適用と同時（同期的）に行われており、
これは `Event.ON_HIT`（むしくいの奪取ハンドラが登録されているイベント）より前に発生する。
このため、むしくいの命中によって相手のHPがちょうど閾値を下回った場合、
相手自身のオボンのみ等が自動的に先に発動・消費されてしまい、むしくいは奪えない
（実機で再現・確認済み。例: カビゴンがオボンのみを持ち、むしくいの命中でHPが半分を
下回るケースで、オボンのみが自動発動して奪えなかった）。

この不具合の根本修正には、「ダメージの適用」と「HP閾値による反応的効果の判定」を
`modify_hp` 内で分離するか、むしくいの奪取処理を `Event.ON_MODIFY_MOVE_DAMAGE`
（ダメージ適用前）に移動する等、ダメージ処理パイプライン全体に影響する変更が必要となる。
影響範囲がむしくい単体の修正を大きく超え、他の多数の技・特性・アイテムの
挙動に対する回帰リスクが高いため、本レビューでは対応を見送る。
将来的に別タスクとして、ダメージ処理パイプラインの見直しを検討すべき事項として記録する。

なお、この制限はねんちゃく等による特別扱いを伴わない一般ケース
（相手がひんしにならず、かつHP閾値ベースのきのみを持つ場合）にのみ影響する。
状態異常治療きのみ（ラムのみ等）や、消費が即座に行われないランク変化系きのみ
（チイラのみ・アッキのみ等、`ON_HP_CHANGED` に登録されていないきのみ）には影響しない。

## 注意点・エッジケース（review-test 向け）

1. **きのみを持たない場合**: 追加効果は発生しない（既存テストで確認済み）。
2. **きのみ以外のアイテム**: 奪取は発生しない（既存テストで確認済み）。
3. **タンガのみ等ダメージ計算前消費型のきのみ**: ダメージ計算時に先に消費されるため、
   命中後の奪取処理では既にきのみを失っており不発になる（既存テストで確認済み）。
4. **ねんちゃく（対象が生存する場合）**: 奪取自体が失敗し attacker は効果を受けない
   （既存テストで確認済み、修正後も変わらず維持）。
5. **ねんちゃく（対象をこの技でひんしにさせた場合）**: 奪取が成功し attacker が
   効果を受ける（今回修正・新規テスト追加）。
6. **みがわり**: 命中してもみがわりに防がれた場合はきのみを食べられない
   （今回修正・新規テスト追加）。
