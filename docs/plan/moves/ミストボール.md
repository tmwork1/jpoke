# ミストボール 実装計画書

## 仕様要約（docs/spec/moves/ミストボール.md 参照）

* タイプ: エスパー / 分類: 特殊 / 威力: 95 / 命中: 100 / PP: 5
* 対象: 相手1体 / 優先度: 0 / 接触なし（弾のわざ）
* 効果: 追加効果として50%の確率で相手の『とくこう』ランクを1段階下げる。

## 現状の実装状況（修正前）

`src/jpoke/data/moves/move_ma.py` にはタイプ・分類・威力・命中・PPとも仕様書と一致する値で登録済みで、
`flags={"bullet"}` により特性『ぼうだん』での無効化も汎用処理（`handlers/ability.py` の
`ctx.move.has_flag("bullet")` チェック）で対応済みだった。しかし `flags` に `"secondary_effect"` が
設定されておらず、`docs/spec/moves/がんせきふうじ.md` と同様の確率発動追加効果技として
本来必要な `ちからずく` との相互作用（威力1.3倍化・追加効果無効化の判定材料）が抜けていた。

```python
"ミストボール": MoveData(
    type="エスパー",
    category="special",
    pp=5,
    power=95,
    accuracy=100,
    flags={"bullet"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ミストボール_lower_spa_C,
        )
    }
),
```

### 修正後

```python
"ミストボール": MoveData(
    type="エスパー",
    category="special",
    pp=5,
    power=95,
    accuracy=100,
    flags={"bullet", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ミストボール_lower_spa_C,
        )
    }
),
```

`flags={"secondary_effect"}` がないと、`ちからずく_boost`（`handlers/ability.py`）による威力1.3倍化と
`ちからずく_disable_secondary_effect` によるとくこう低下無効化のいずれも発動しなくなってしまう
（`ctx.move.has_flag("secondary_effect")` が常に False になるため）。同種の実装として
`アイアンテール_lower_defender_def`・`がんせきふうじ_lower_defender_spd` が全く同じパターンで
`flags={"secondary_effect"}` を持っている。

追加効果ハンドラ（`handlers/move_attack.py` の `ミストボール_lower_spa_C`）は仕様と一致しており、
修正不要。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 50%の確率で相手の『とくこう』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def ミストボール_lower_spa_C(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spa": -1}, chance=0.5)
```

`modify_defender_stats`（`handlers/move.py`）は `chance` を受け取り
`battle.resolve_secondary_chance` を経由するため、ちからずくによる追加効果無効化・確率補正は
汎用的に処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（能力ランク低下）は既存の同種ハンドラ（アイアンテール・がんせきふうじ 等）
と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。`docs/spec/turn.md` にも
`ON_DAMAGE_HIT` は他の追加効果技と共通のタイミングとして扱われており、個別の priority 検討は不要。

## リーサル計算について

`docs/progress/move.md` の該当行「リーサル実装」列は `n/a`。本技の追加効果は相手の『とくこう』ランクを
下げるのみでHPに影響しないため、`handlers/lethal.py` に専用ハンドラは不要で、`n/a` のままで妥当。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` を追加したことで、ちからずく所持時は威力が1.3倍になり、
     とくこう低下効果が発動しなくなる（`がんせきふうじ` 等と同じ挙動）。
2. **ぼうだんとの相互作用**
   - `flags={"bullet"}` により、特性『ぼうだん』所持の相手には技自体が無効化される
     （汎用ハンドラ `handlers/ability.py` で処理済み、個別実装不要）。
3. **テスト方針**
   - `secondary_chance` を1に固定し、とくこうランク変化が発動することを確認するテストを
     `tests/moves_attack/test_move_ma.py` に追加する（`がんせきふうじ` の既存テストと同じパターン）。
   - ちからずく所持時に威力1.3倍・とくこう低下無効化になることを確認するテストを追加する
     （`がんせきふうじ` の既存テストと同じパターン）。
