# 計画書: やどりぎのタネ

## 仕様要約

- くさタイプ / 変化 / 命中90 / PP10 / 対象: 1体選択
- 毎ターン相手のHPを最大HPの1/8ずつ減らし、その分自分のHPを回復する
- 自分が交代しても効果が引き継がれる
- くさタイプのポケモンには無効（`ON_TRY_MOVE_2` priority 130: `くさタイプ: やどりぎ無効` でフレームワーク側が処理）
- すでにやどりぎのタネ状態の場合は失敗（`ON_TRY_MOVE_2` priority 130 でフレームワーク側が処理）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | 相手にやどりぎのタネ付与 |

## Priority 根拠

`docs/spec/turn.md` の `ON_TRY_MOVE_2` に `くさタイプ: やどりぎ無効` が priority 130、`やどりぎのタネ: すでにやどりぎ状態` が priority 130 で記載されており、これらはフレームワーク側が処理する。ハンドラ本体は既存の `ON_STATUS_HIT` priority 100 のパターンに従う。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `まるくなる` 関連の後、`みがわり` の前（五十音順）。

```python
def やどりぎのタネ_apply_volatile_to_defender(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """やどりぎのタネの効果: 相手をやどりぎのタネ状態にする。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="やどりぎのタネ")
```

### data/move.py への更新

`"やどりぎのタネ"` エントリのハンドラを追加する。

## 注意点・エッジケース

- **くさタイプ無効化はフレームワーク側の処理**。`ON_TRY_MOVE_2` priority 130 に `くさタイプ: やどりぎ無効` として明記されているため、ハンドラ内にくさタイプチェックは不要
- すでにやどりぎのタネ状態の失敗チェックも同 priority 130 でフレームワーク側が処理
- 毎ターンのHP吸収処理は `ON_TURN_END` priority 80 で volatile の効果として別途動作する（ハンドラとは分離）
- 自分が交代しても効果が持続する点は volatile 側の実装に委ねる
- `apply_volatile_to_defender` の標準ラッパーパターンを使用
