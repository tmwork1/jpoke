# かみなりのキバ 実装計画書

## 仕様要約（docs/spec/moves/かみなりのキバ.md 参照）

* タイプ: でんき / 分類: 物理 / 威力: 65 / 命中: 95 / PP: 15
* 対象: 相手1体 / 優先度: 0 / 接触あり（直接攻撃）
* 効果: 追加効果として、10%の確率で相手を『まひ』状態にし、10%の確率で相手をひるませる。
  2つの効果はそれぞれ独立に判定される。
* かみつき技（`flags` に `"bite"` を含む）。特性『がんじょうあご』の対象。

## 現状の実装状況

`src/jpoke/data/move.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。
以下のとおり仕様と一致していることを確認した（修正不要）。

```python
"かみなりのキバ": MoveData(
    type="でんき",
    category="physical",
    pp=15,
    power=65,
    accuracy=95,
    flags={"bite", "contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: [
            h.MoveHandler(ha.かみなりのキバ_apply_paralysis_to_defender),
            h.MoveHandler(ha.かみなりのキバ_apply_flinch),
        ]
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手を『まひ』状態にする |
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手をひるませる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def かみなりのキバ_apply_flinch(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_defender(battle, ctx, value, volatile="ひるみ", chance=0.1)


def かみなりのキバ_apply_paralysis_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="まひ", chance=0.1)
```

`apply_ailment_to_defender` / `apply_volatile_to_defender`（いずれも `handlers/move.py`）が
`battle.resolve_secondary_chance` を経由するため、ちからずくによる追加効果無効化（確率0化）・
威力1.3倍化、`flags={"bite"}` によるがんじょうあごの威力1.5倍化は、いずれも汎用ハンドラ側で
処理される。本技側に個別のガードは不要。まひとひるみは別々の `MoveHandler` として登録されており、
それぞれ独立に `battle.random.random()` で抽選されるため、仕様どおり同時発生し得る。

## Priority 根拠

ダメージを与えた技の追加効果（状態異常・ひるみ付与）は既存の同種ハンドラ（かみなり・かみなりパンチ・
ほのおのキバ 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。個別の priority
検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **がんじょうあごとの相互作用**
   - `flags={"bite"}` により、がんじょうあご所持ポケモンが使用すると威力が1.5倍（65→97）になる。
     既存の がんじょうあご テストファイル側で他のかみつき技と同様の網羅がされていれば個別追加は不要。
2. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・まひ/ひるみ付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
3. **おうじゃのしるし・するどいキバとの相互作用**
   - `handlers/item.py` の `_INNATE_FLINCH_MOVES` に「かみなりのキバ」が既に含まれており、
     元々ひるみ追加効果を持つ技として、おうじゃのしるし等による追加のひるみ判定が重複しないよう
     処理済み（修正不要）。
4. **2つの追加効果の独立性**
   - まひとひるみはそれぞれ個別に `battle.random.random()` で抽選されるため、両方同時に発動する
     可能性がある（テストで固定シードや `secondary_chance=1.0` を使う場合はどちらか一方のみを
     確認する形で問題ない）。
5. **テスト方針**
   - `secondary_chance` を1に固定し、まひ・ひるみそれぞれの付与有無を確認するテストが既存
     （`test_かみなりのキバ_まひが発動する` / `test_かみなりのキバ_ひるみが発動する`）。
