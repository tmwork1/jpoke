# 実装計画書: アーマーキャノン

参照仕様書: docs/spec/moves/アーマーキャノン.md

## 仕様要約

* タイプ: ほのお / 分類: 特殊 / 威力: 120 / 命中率: 100 / PP: 8（champions版） / 対象: 1体選択
  / 優先度: 0 / 接触なし
* 技が成功しダメージを与えた後、自分の『ぼうぎょ』『とくぼう』ランクが1段階ずつ下がる
  （確率100%・確定効果、追加効果には分類されない）。

## 既存実装の確認結果

`src/jpoke/data/moves/move_a.py` に既に以下の通り実装済みであることを確認した。

```python
"アーマーキャノン": MoveData(
    type="ほのお",
    category="special",
    pp=5,
    power=120,
    accuracy=100,
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.アーマーキャノン_lower_attacker_stats,
        )
    }
),
```

* `handlers/move_attack.py` の `アーマーキャノン_lower_attacker_stats` は
  `modify_attacker_stats(battle, ctx, value, stats={"def": -1, "spd": -1})` を呼び出しており、
  `chance` を指定していないためデフォルトの100%（確定効果）で発動する。`battle.modify_stats`
  経由のため、あまのじゃく等の特性補正も自動的に適用される。
* `Event.ON_HIT` の `MoveHandler` は `subject_spec` を省略した場合デフォルトで `"attacker:self"`
  になる（`src/jpoke/handlers/move.py` の `MoveHandler.__init__`）。本ハンドラは攻撃者自身の
  ランクを変化させるため、この既定値と一致しており問題ない。
* `flags` は未設定（空集合）で、`secondary_effect` を含まない。一次情報の判定表に
  「追加効果: ×」と明記されており、自己ランクダウンは「追加効果」に分類されないため、
  ちからずくとの相互作用を避けるためにも `secondary_effect` を持たないこの実装は正しい
  （`ばちかまし`・`ばかぢから` 等の同系統技と同じ方針）。接触もパンチ技でもないため他の
  flag も不要。
* `Event.ON_HIT` は `docs/spec/turn.md` で「自分のランクが下がる反動」が priority=10 と
  記載されているが、既存の同種実装（`インファイト_lower_defender_stats`、
  `ぶちかまし_lower_defender_spd`、`ばかぢから_lower_attacker_def` 等）はいずれも
  `priority` を指定せずデフォルト（100）のまま登録されている。本技も同じ実装パターンの
  ため、既存実装のデフォルト優先度を踏襲し変更しない。

### 不整合の修正: PP

`src/jpoke/data/ps-champ-ja/moves.json`（正）では `アーマーキャノン` の PP は 8 と記載されている
（champions では全技PPが 8/12/16/20 の4段階に統一されているため）。既存実装は
`pp=5`（Gen9本家Wikiの値のまま移行漏れ）になっており、`pp=8` に修正する。
`docs/progress/move.md` は既に PP 列が `8` になっており、修正不要（実装側を進捗表に
合わせる）。

### リーサル計算の確認結果

`docs/progress/move.md` の「リーサル実装」列は `n/a`。アーマーキャノンが下げるのは
攻撃者自身の『ぼうぎょ』『とくぼう』であり、以降の与ダメージには影響しない
（`ばかぢから`・`ぶちかまし` の計画書と同じ方針）。よって `lethal_handlers` は登録不要で
あり、`n/a` は妥当。

## 修正内容

* `src/jpoke/data/moves/move_a.py`: `アーマーキャノン` の `pp` を `5` から `8` に修正する。
