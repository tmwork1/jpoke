# ブレイズキック 実装計画書

参照: docs/spec/moves/ブレイズキック.md

## 仕様要約

* タイプ: ほのお / 分類: 物理 / 威力: 85 / 命中: 90 / PP: 12（Champions基準。
  `src/jpoke/data/ps-champ-ja/moves.json` 788行目）
* 対象: 相手1体 / 優先度: 0 / 接触あり（直接攻撃）
* 追加効果: 10%の確率で相手を『やけど』状態にする
* 急所ランク+1（`docs/spec/critical.md` 参照）
* ほのおタイプの攻撃技のため、被弾した相手のこおりを解凍する（`thaw` フラグ）

## 現状の実装状況（レビュー時点）

`src/jpoke/data/moves/move_ha.py` に登録済み、`src/jpoke/handlers/move_attack.py` に
やけど付与ハンドラ・`critical_rank=1` は実装済みだったが、以下2点が仕様と不一致だった。

### 修正1: PP が Gen9本家基準の値のままだった

```python
"ブレイズキック": MoveData(
    type="ほのお",
    category="physical",
    pp=10,          # ← Champions基準では12（移行漏れ）
    power=85,
    accuracy=90,
    critical_rank=1,
    flags={"contact", "secondary_effect"},
    ...
),
```

`src/jpoke/data/ps-champ-ja/moves.json` 788行目は `ブレイズキック	ほのお	物理	85	90	12	接○	守○`
であり、PPは12（Champions は全技PPを8/12/16/20の4段階に統一しているため、Gen9本家の10は
そのままでは成立しない）。`pp=12` に修正する。

### 修正2: `thaw` フラグが欠落していた

ほのおタイプの攻撃技は、被弾した相手がこおり状態の場合に解凍する効果を持つ
（`docs/spec/ailments/こおり.md` 31-32行目）。この解凍処理は `data/ailment.py` の
`こおり_cure_by_thaw_move`（`Event.ON_DAMAGE_HIT`, `subject_spec="defender:self"`）が
`ctx.move.has_flag("thaw")` を判定して汎用的に処理するため、`flags` に `"thaw"` を
追加しないと発動しない。かえんほうしゃ・ひのこ等の他のほのお攻撃技と同様に本レビューで追加する。

なお、`かえんぐるま`・`かえんボール`等が持つ「使用者自身がこおり状態のとき無条件で解凍してから
攻撃する」効果（`_thaw_attacker` 系ハンドラ, `Event.ON_TRY_ACTION`）は、一次情報に
その旨の記載がないため対象外（実装しない）。

### 修正後

```python
"ブレイズキック": MoveData(
    type="ほのお",
    category="physical",
    pp=12,
    power=85,
    accuracy=90,
    critical_rank=1,
    flags={"contact", "secondary_effect", "thaw"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ブレイズキック_apply_burn_to_defender,
        )
    }
),
```

`handlers/move_attack.py` の `ブレイズキック_apply_burn_to_defender` は既存実装のまま変更不要。

```python
def ブレイズキック_apply_burn_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="やけど", chance=0.1)
```

### docs/spec/moves/_move_labels.md の変更

`thaw` セクションに `ブレイズキック` を五十音順で追加する（フレアドライブとふんえん…ではなく
`フレアドライブ` の次に挿入。ふんえんは `thaw` フラグ対象外のため対象外のまま）。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手を『やけど』状態にする |
| `ON_DAMAGE_HIT` | 100（デフォルト） | defender:self（`data/ailment.py` 側で登録済み） | `thaw` フラグにより被弾した相手のこおりを解凍する（既存の汎用ハンドラ、本技側での追加実装は不要） |

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は汎用ハンドラ側で処理される。
`critical_rank=1` は `MoveData` のフィールドとして宣言するだけで、既存の急所判定ロジック
（`core/move_executor.py`）が自動的に急所率+1段階を適用するため、専用ハンドラは不要。

## Priority 根拠

* ダメージを与えた技の追加効果（やけど付与）は既存の同種ハンドラ（かえんほうしゃ・だいもんじ等）と
  同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。個別の priority 検討は不要。
* こおり解凍も同じ `Event.ON_DAMAGE_HIT` で処理される（`docs/spec/ailments/こおり.md` の
  「Priority 100: ほのお技受けてこおり回復」を参照）。

## 注意点・エッジケース（review-test 向け）

1. **急所ランク+1の確認**
   - `critical_rank=1` により、乱数0（`Stage=1` の確率は1/8）では急所が発生し、
     急所閾値以上の乱数では急所が発生しないことをテストで確認する
     （`はっぱカッター` の `test_はっぱカッター_急所ランクが1` 系と同型のテストを追加する）。
2. **こおり解凍の確認**
   - 相手がこおり状態のときにブレイズキックを当てると解凍されることをテストで確認する
     （`かえんほうしゃ` の `test_かえんほうしゃ_こおり状態の相手に当てると解凍する` と同型）。
3. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・やけど付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
4. **PP修正の確認**
   - PPが12であることを確認する既存/新規テストがあれば整合を取る（`MoveData` 定義の直接確認で十分）。
