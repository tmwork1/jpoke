# いにしえのうた 実装計画書

## 仕様要約（docs/spec/moves/いにしえのうた.md 参照）

* タイプ: ノーマル / 分類: 特殊 / 威力: 75 / 命中: 100 / PP: 10
* 対象: 相手1体 / 優先度: 0 / 非接触 / 音のわざ
* 効果: 10%の確率で相手を『ねむり』状態にする（追加効果）。
* メロエッタのフォルムチェンジ効果は、本プロジェクト（ポケモンチャンピオンズシングルバトル）の対象外
  （`src/jpoke/data/ps-champ-ja/moves.json` に技自体が未収録のため）。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手を『ねむり』状態にする |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def いにしえのうた_apply_sleep_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """いにしえのうたの追加効果: 10%の確率で相手をねむり状態にする。"""
    return apply_ailment_to_defender(battle, ctx, value, ailment="ねむり", chance=0.1)
```

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）は汎用的に処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（状態異常付与）は既存の同種ハンドラ（いてつくしせん・いびき 等）と同じ
`ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。`docs/spec/turn.md` にも
`ON_DAMAGE_HIT` は他の追加効果技と共通のタイミングとして扱われており、個別の priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **音のわざフラグ**
   - `flags={"sound"}` により、みがわり貫通・ぼうおん/パンクロックの対象・ちょうおんぱ等と同様の判定を受ける。
     フレームワーク側（`move_executor.py` の `ctx.move.has_flag("sound")` 判定）で汎用的に処理されるため、
     本技側の実装への影響はない。
2. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` があるため、ちからずく所持時は `resolve_secondary_chance` が確率を0にし、
     ねむり状態が発動しなくなる（威力1.3倍化は `ちからずく_boost` 側で処理）。
3. **メロエッタのフォルムチェンジは対象外**
   - 一次情報にはメロエッタの専用フォルムチェンジ効果があるが、`src/jpoke/data/ps-champ-ja/moves.json` に
     いにしえのうた自体が収録されておらず、本プロジェクトの対象範囲外。実装しない。
4. **テスト方針**
   - `secondary_chance` を0/1に固定し、ねむり状態の有無を確認する既存テスト
     （`test_いにしえのうた_ねむりが発動しない` / `test_いにしえのうた_ねむりが発動する`）で十分。
