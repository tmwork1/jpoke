# 計画書: たくわえる（のみこむ・はきだすセット）

出典: `docs/spec/moves/たくわえる.md`

## 仕様要約

### たくわえる

- ノーマルタイプ / 変化 / 命中なし / PP20（champions 基準） / 対象: 自分
- 自分のぼうぎょ・とくぼうランクを1段階ずつ上げる
- 揮発性状態「たくわえる」を付与し、たくわえた回数を記録する（最大3回）
- 3回たくわえた状態でさらに使うと失敗（`ON_TRY_MOVE_1` priority 30: `たくわえる: カウントが3`）
- **自分を対象とする変化技のため `MoveData(target="self")` を明示する**
  （省略すると `MoveData.target` のデフォルト `"foe"` になり、`is_blocked_by_protect` /
  `is_reflectable` が誤って True になる。相手の まもる で防がれたり
  マジックコートで跳ね返される誤動作の原因になるため必須）

### のみこむ（参考・実装済み、本計画書の対象外）

- ノーマルタイプ / 変化 / 命中なし / 対象: 自分
- 自分のHPを回復する。回復量はたくわえた回数に依存:
  - 1回: 最大HPの1/4
  - 2回: 最大HPの1/2
  - 3回: 全回復（最大HP全量）
- 使用後、たくわえた回数はリセット（volatile「たくわえる」を削除）
- たくわえた回数で上がったぼうぎょ・とくぼうランクをリセット時に元に戻す（count 分だけ逆方向に変化）
- たくわえた回数が0（volatile なし）の場合は失敗（`ON_TRY_MOVE_1` priority 30: `はきだす・のみこむ: カウントが0`）
- 既知の未修正課題（本計画書のスコープ外。のみこむ自体のレビュー時に対応）:
  - `docs/champions/move_list.txt` では PP12 だが、現在の実装は PP10 のまま
  - `MoveData` に `target="self"` が未設定（デフォルト `"foe"` のままで、たくわえるやこうそくいどう等の
    自己対象変化技と同種のバグを抱えている可能性がある）

### はきだす（参考・実装済み、本計画書の対象外）

- ノーマルタイプ / 特殊 / 命中100 / 対象: 1体選択（相手を攻撃する技のため target は省略時デフォルトの `"foe"` のままでよい）
- 威力はたくわえた回数に依存:
  - 1回: 威力100
  - 2回: 威力200
  - 3回: 威力300
- 使用後、たくわえた回数はリセット（volatile「たくわえる」を削除）
- たくわえた回数で上がったぼうぎょ・とくぼうランクをリセット時に元に戻す（count 分だけ逆方向に変化）
- たくわえた回数が0（volatile なし）の場合は失敗（`ON_TRY_MOVE_1` priority 30: `はきだす・のみこむ: カウントが0`）
- 既知の未修正課題（本計画書のスコープ外。はきだす自体のレビュー時に対応）:
  - `docs/champions/move_list.txt` では PP12 だが、現在の実装は PP10 のまま

## Priority 根拠

- `ON_TRY_MOVE_1 priority 30`: `docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` 表に `| 30 | たくわえる: カウントが3 |` と明記

## 実装箇所

### handlers/move_status.py（実装済み）

挿入位置（五十音順）: `たてこもる_modify_attacker_stats` の前

```python
def たくわえる_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """たくわえるの効果: ぼうぎょ・とくぼうを1段階上げ、たくわえカウントをインクリメントする。"""
    mon = ctx.attacker
    # ぼうぎょ・とくぼうのランク上昇を試みる
    result = battle.modify_stats(mon, {"def": 1, "spd": 1}, source=mon)
    # volatile がなければ新規付与（count=1）、あればカウントを+1
    if not mon.has_volatile("たくわえる"):
        battle.volatile_manager.apply(mon, "たくわえる", count=1)
    else:
        mon.volatiles["たくわえる"].count = (mon.volatiles["たくわえる"].count or 0) + 1
    return HandlerReturn(value=result)


def たくわえる_check_can_use(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """たくわえるの使用条件チェック: たくわえた回数がすでに3なら失敗する。"""
    mon = ctx.attacker
    if mon.has_volatile("たくわえる") and (mon.volatiles["たくわえる"].count or 0) >= 3:
        battle.add_event_log(
            mon, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="たくわえる")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### data/moves/move_ta.py（実装済み）

```python
    "たくわえる": MoveData(
        type="ノーマル",
        category="status",
        pp=20,
        target="self",
        handlers={
            Event.ON_TRY_MOVE_1: h.MoveHandler(
                hs.たくわえる_check_can_use,
                priority=30,
            ),
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.たくわえる_apply,
            ),
        }
    ),
```

## 注意点・エッジケース

- **カウント3での失敗チェック**: `ON_TRY_MOVE_1 priority 30` にハンドラを登録する。volatile の `count` フィールドが `None` の可能性があるため `(count or 0) >= 3` とガードする
- **volatile 管理方針**: `volatile_manager.apply` は「既に同名 volatile があれば失敗（False）」を返すため、初回のみ `apply` でカウント1付与し、2回目以降は `volatiles["たくわえる"].count` を直接インクリメントする
- **ぼうぎょ・とくぼうのランク上昇**: ランクがすでに+6でも技自体は失敗せず（たくわえカウントのみ増える）。`battle.modify_stats` の戻り値をそのまま `ON_STATUS_HIT` の value として返す（他の自己ランク変化技と同じ規約）
- **target="self" が必須**: `MoveData.target` のデフォルトは `"foe"` であり、省略すると `Move.is_blocked_by_protect`（`target == "foe"` で True）や `Move.is_reflectable`（`target in ("foe", "foe_side")` で True、変化技の場合）が誤って成立し、相手の まもる・マジックコートに誤反応してしまう。自分専用の変化技は必ず `target="self"` を明示する
- **リーサル計算**: 対象外（ダメージを与えない変化技、n/a）
