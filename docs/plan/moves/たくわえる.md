# 計画書: たくわえる（のみこむ・はきだすセット）

## 仕様要約

### たくわえる

- ノーマルタイプ / 変化 / 命中なし / PP20 / 対象: 自分
- 自分のぼうぎょ・とくぼうランクを1段階ずつ上げる
- 揮発性状態「たくわえる」を付与し、たくわえた回数を記録する（最大3回）
- 3回たくわえた状態でさらに使うと失敗（`ON_TRY_MOVE_1` priority 30: `たくわえる: カウントが3`）

### のみこむ

- ノーマルタイプ / 変化 / 命中なし / PP10 / 対象: 自分
- 自分のHPを回復する。回復量はたくわえた回数に依存:
  - 1回: 最大HPの1/4
  - 2回: 最大HPの1/2
  - 3回: 全回復（最大HP全量）
- 使用後、たくわえた回数はリセット（volatile「たくわえる」を削除）
- たくわえた回数で上がったぼうぎょ・とくぼうランクをリセット時に元に戻す（count 分だけ逆方向に変化）
- たくわえた回数が0（volatile なし）の場合は失敗（`ON_TRY_MOVE_1` priority 30: `はきだす・のみこむ: カウントが0`）

### はきだす

- ノーマルタイプ / 特殊 / 命中100 / PP10 / 対象: 1体選択
- 威力はたくわえた回数に依存:
  - 1回: 威力100
  - 2回: 威力200
  - 3回: 威力300
- 使用後、たくわえた回数はリセット（volatile「たくわえる」を削除）
- たくわえた回数で上がったぼうぎょ・とくぼうランクをリセット時に元に戻す（count 分だけ逆方向に変化）
- たくわえた回数が0（volatile なし）の場合は失敗（`ON_TRY_MOVE_1` priority 30: `はきだす・のみこむ: カウントが0`）

## Priority 根拠

- `ON_TRY_MOVE_1 priority 30`: `docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` 表に `| 30 | たくわえる: カウントが3 |` と明記
- `ON_TRY_MOVE_1 priority 30` （のみこむ・はきだすの失敗）: 同表に `| 30 | はきだす・のみこむ: カウントが0 |` と明記
- `ON_STATUS_HIT priority 100`: `MoveHandler` のデフォルト値。`docs/spec/turn.md` には `ON_STATUS_HIT` セクションの priority 記載なし。`たてこもる`・`ちょうのまい` など他の変化技ハンドラと同じパターンを踏襲
- `ON_MODIFY_MOVE priority 100`: `MoveHandler` のデフォルト値。`docs/spec/turn.md` の `Event.ON_MODIFY_MOVE` に記載なし。威力を動的に変える既存ハンドラ（ウェザーボール等）と同パターン
- `ON_HIT priority 100`: `MoveHandler` のデフォルト値。後処理（volatile 削除・ランク戻し）を行う既存ハンドラと同パターン

## 実装箇所

### handlers/move_status.py への追加

挿入位置（五十音順）:
- `たてこもる_modify_attacker_stats` の後
- `たくわえる_check_can_use` → `たくわえる_apply` → `のみこむ_apply` → `はきだす_set_power` → `はきだす_apply_after` の順で追加

```python
def たくわえる_check_can_use(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """たくわえるの使用条件チェック: たくわえた回数がすでに3なら失敗する。"""
    mon = ctx.attacker
    if mon.has_volatile("たくわえる") and (mon.volatiles["たくわえる"].count or 0) >= 3:
        battle.add_event_log(mon, LogCode.MOVE_FAILED,
                             payload={"reason": "たくわえる"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def たくわえる_apply(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """たくわえるの効果: ぼうぎょ・とくぼうを1段階上げ、たくわえカウントをインクリメントする。"""
    mon = ctx.attacker
    # ぼうぎょ・とくぼうのランク上昇を試みる（最大+6でも続行できるが、上がった分だけ後で戻す）
    result = battle.modify_stats(mon, {"B": 1, "D": 1}, source=mon)
    # volatile がなければ新規付与（count=1）、あればカウントを+1
    if not mon.has_volatile("たくわえる"):
        battle.volatile_manager.apply(mon, "たくわえる", count=1)
    else:
        mon.volatiles["たくわえる"].count = (mon.volatiles["たくわえる"].count or 0) + 1
    return HandlerReturn(value=result)


def のみこむ_apply(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """のみこむの効果: たくわえ回数に応じてHPを回復し、たくわえ状態をリセットする。"""
    mon = ctx.attacker
    count = (mon.volatiles["たくわえる"].count or 0) if mon.has_volatile("たくわえる") else 0
    # たくわえていない場合は失敗（ON_TRY_MOVE_1 でチェック済みだが念のため）
    if count == 0:
        return HandlerReturn(value=False, stop_event=True)
    # 回復量: 1回=1/4, 2回=1/2, 3回=全回復
    ratios = {1: 1/4, 2: 1/2, 3: 1.0}
    r = ratios[count]
    if r == 1.0:
        battle.modify_hp(mon, v=mon.max_hp - mon.hp)
    else:
        battle.modify_hp(mon, r=r)
    # たくわえた回数分だけランクを戻す（modify_stats で count 分逆方向に）
    battle.modify_stats(mon, {"B": -count, "D": -count}, source=mon)
    # volatile 削除
    battle.volatile_manager.remove(mon, "たくわえる")
    return HandlerReturn(value=value)


def はきだす_set_power(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """はきだすの効果（ON_MODIFY_MOVE）: たくわえ回数に応じて威力を設定する。"""
    mon = ctx.attacker
    count = (mon.volatiles["たくわえる"].count or 0) if mon.has_volatile("たくわえる") else 0
    power = count * 100  # 1回=100, 2回=200, 3回=300
    ctx.move.power = power
    return HandlerReturn(value=value)


def はきだす_apply_after(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """はきだすのヒット後処理（ON_HIT）: たくわえ状態をリセットし、上がったランクを戻す。"""
    mon = ctx.attacker
    if not mon.has_volatile("たくわえる"):
        return HandlerReturn(value=value)
    count = mon.volatiles["たくわえる"].count or 0
    # ランクをたくわえた回数分だけ逆方向へ
    battle.modify_stats(mon, {"B": -count, "D": -count}, source=mon)
    battle.volatile_manager.remove(mon, "たくわえる")
    return HandlerReturn(value=value)
```

## 注意点・エッジケース

### たくわえる

- **カウント3での失敗チェック**: `ON_TRY_MOVE_1 priority 30` にハンドラを登録する。volatile の `count` フィールドが `None` の可能性があるため `(count or 0) >= 3` とガードする
- **volatile 管理方針**: `volatile_manager.apply` は「既に同名 volatile があれば失敗（False）」を返すため、初回のみ `apply` でカウント1付与し、2回目以降は `volatiles["たくわえる"].count` を直接インクリメントする（`きゅうしょアップ` の count 管理と同じパターン）
- **ぼうぎょ・とくぼうのランク上昇**: ランクがすでに+6でも失敗せず（たくわえカウントのみ増やす）。ただしランクが上がった分だけ `のみこむ`/`はきだす` 使用時に逆補正されるため、上昇量が実際に変化した量か否かが重要。この仕様の詳細は `docs/wiki/volatiles/たくわえる_(状態変化).html` の備考参照（ランクが上がらなかった場合は戻りもない）。現実装では `battle.modify_stats` の戻り値から実際の変化量を追跡することが理想だが、仕様が複雑なため初期実装は「たくわえた回数 = 実際に上がった量」と扱う簡略版で実装する

### のみこむ

- **失敗チェック（カウント0）**: `ON_TRY_MOVE_1 priority 30: はきだす・のみこむ: カウントが0` でフレームワーク側が処理するためハンドラ内でも念のためガードするが、到達しない前提
- **全回復**: count=3 の場合 `r=1.0` を渡すと `modify_hp(r=1.0)` で最大HP全量が回復するかは実装依存なため、`v=mon.max_hp - mon.hp` の形で明示する
- **ランク戻し**: `battle.modify_stats(mon, {"B": -count, "D": -count})` はしろいきり・クリアボディ等を無視しない（wiki には「クリアボディなどでも防げない」と記載）。防げないことをテストで確認する必要あり
- **バトンタッチ**: volatile 「たくわえる」はバトンタッチで引き継がれないが、上昇したランクは引き継がれる（フレームワーク側が処理）

### はきだす

- **威力0への対応**: `data/move.py` では初期値 `power=0` で登録されている。`ON_MODIFY_MOVE` で count×100 に書き換える。たくわえカウントが0の場合は `ON_TRY_MOVE_1` で失敗するため、ハンドラが実行されるときは常に count≥1 が保証される
- **ランク戻し**: のみこむと同様、`ON_HIT` でランクを逆補正し volatile を削除する。はきだしが外れた（命中失敗）場合もランクは下がる（wiki 記載の仕様）。**外れた場合の処理は `ON_HIT` では発動しないため `ON_APPLY_MOVE` 以降の別イベントで処理が必要になる可能性がある。初期実装では `ON_HIT` のみ対応し、外れ時の挙動はエッジケースとして保留する**
- **急所**: 第四世代以降は急所あり（wiki 記載）。フレームワーク側の急所判定で処理
