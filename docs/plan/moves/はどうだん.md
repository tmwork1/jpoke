# 実装計画書: はどうだん（再レビュー）

## 仕様要約

`docs/spec/moves/はどうだん.md` 参照。

* タイプ: かくとう / 分類: 特殊 / 威力: 80 / 命中率: - （必中） / PP: 20 / 対象: 1体選択
* 追加効果なし。必ず命中する（自分の命中率、相手の回避率に関係なく必中）。
* 弾（bullet）・波動（pulse）のわざ。ぼうだんで無効化、メガランチャーで威力1.5倍
  （いずれも既存の汎用ハンドラで対応済み・技側の個別実装は不要）。

## 既存実装の確認結果

`src/jpoke/data/moves/move_ha.py` にエントリが既に存在する。

```python
"はどうだん": MoveData(
    type="かくとう",
    category="special",
    pp=20,
    power=80,
    flags={"bullet", "pulse"},
    handlers={},  # 追加効果なし
),
```

* タイプ・分類・PP・威力・flags（bullet, pulse）は仕様と一致しており問題なし。
* 追加効果がないため `handlers={}` で正しい。
* **`accuracy` フィールドが未指定**だった。`MoveData.accuracy` のデフォルトは `None` であり、
  `move_executor._check_hit` は `accuracy is None` の場合に必中として扱うため、
  動作上は既に仕様どおり（必中）になっていた。
  ただし本プロジェクトの慣例では、必中技であることを明示するため
  `あてみなげ`（`src/jpoke/data/moves/move_a.py`）等で `accuracy=None` を明示的に記述している。
  可読性・保守性のため `はどうだん` にも `accuracy=None` を明示的に追加する
  （挙動に変化はない）。

## 修正内容

`src/jpoke/data/moves/move_ha.py` の `はどうだん` エントリに `accuracy=None` を追加する。

## 判定フラグの動作確認（コード変更不要）

* あなをほる・そらをとぶ・ダイビング・シャドーダイブへの非命中は、
  `handlers/volatile.py` の `can_hit_hidden_target`（`HIDDEN_MOVE_ALLOWED_MOVES` に
  `はどうだん` が含まれない）により、`accuracy=None` とは独立した別イベントで
  汎用的にブロックされるため、技側に個別対応は不要。
* ぼうだんによる無効化・メガランチャーによる威力1.5倍は、いずれも `handlers/ability.py` が
  技の `bullet` / `pulse` フラグを参照する汎用実装のため、技側に個別対応は不要。
