# 実装計画書: さむいギャグ

## 仕様要約

- タイプ: こおり / カテゴリ: 変化 / PP: 10 / 対象: 全体の場（target="field"）
- **先に**ゆきを5ターン発生させ、**その後**自発交代（控えポケモンを選択して入れ替え）
- すでにゆき状態の場合、ゆきの変更は失敗するが交代効果は発動する
- すでにゆき状態で交代先もいない場合にのみ技が完全失敗する
- 使用者がつめたいいわを持つ場合はゆきの持続が8ターンになる（weather_manager / ON_MODIFY_DURATION 経由で自動処理）
- ゆびをふる対象外（labels=["non_yubi"]）

|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | ゆき変更を試み、交代可能な場合は PIVOT 設定。ゆきも交代もどちらも発動できない場合にのみ失敗を返す |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「天候・フィールド: すでに同じ」が priority 100 として記載されている。同種ハンドラ `あまごい_activate_weather`・`ゆきげしき_activate_weather` が priority デフォルト（100）を使用しているため、同一の priority を採用する。

ON_STATUS_HIT は変化技の命中後処理であり、`docs/spec/turn.md` には直接掲載されていないが、同種の天気変更技（にほんばれ・あまごい・ゆきげしき）がいずれも `ON_STATUS_HIT` priority デフォルト（100）で実装されているため、これを根拠とする。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

import 行に `Interrupt` を追加する（現在は `Event, LogCode` のみ）。

```python
from jpoke.enums import Event, Interrupt, LogCode
```

関数を五十音順の適切な位置（「さいはい」「さきおくり」付近、「し」より前）に追加する。

```python
def さむいギャグ_activate_weather_and_pivot(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """さむいギャグ: ゆきを5ターン発生させた後、自発交代する。

    すでにゆき状態の場合はゆきの変更は失敗するが交代効果は発動する。
    すでにゆき状態で交代先もいない場合にのみ技が失敗する。
    """
    weather_changed = battle.weather_manager.apply("ゆき", 5, source=ctx.attacker)

    player = battle.get_player(ctx.attacker)
    can_switch = bool(battle.get_available_switch_commands(player))
    if can_switch:
        battle.player_states[player].interrupt = Interrupt.PIVOT

    # ゆき変更も交代もどちらも発動できない場合にのみ失敗
    if not weather_changed and not can_switch:
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=True)
```

## 注意点・エッジケース

- **ゆきと交代の分離**: ゆきの変更に失敗しても交代は発動する。`weather_manager.apply` の戻り値で失敗を即時返さず、交代可否を別途チェックする必要がある。

- **完全失敗条件**: 「すでにゆき状態 + 交代先なし」の場合のみ `value=False, stop_event=True` を返す。この条件を満たさない場合は `value=True` を返す。

- **交代メカニズム**: `Interrupt.PIVOT` を `player_states` に設定することで、技実行後に `turn_controller.py` の `run_interrupt_switch(Interrupt.PIVOT)` が自発交代を処理する。変化技から PIVOT を設定する際は `pivot()` 関数（move_attack.py）を直接呼べないため、同じ処理を手動で行う。

- **交代先が選ぶ天気特性**: 交代先がゆき以外の天気を発動させる特性を持つ場合、交代時にゆきが別天候に上書きされる。これは ON_SWITCH_IN で処理されるため、ハンドラ側での対応は不要。

- **つめたいいわの8ターン延長**: `weather_manager.apply` が `source=ctx.attacker` を受け取り、内部で `ON_MODIFY_DURATION` を発火する。つめたいいわのハンドラが `subject_spec="source:self"` で登録済みのため、ハンドラ側での追加処理は不要。

- **おおひでり等の永続天候下での失敗**: `weather_manager.apply` が失敗（False 返却）するが、交代先がいれば技は成功扱いで交代のみ発動する。これはゆきWiki仕様（「マップの効果で自然に天候が発生しているとき、天気をゆきに変えることはできないが、交代することはできる」）に一致する。

- **ハンドラ側のガードは不要**: ねむり・まひ等の行動不能チェック・ちょうはつチェック等はイベント発火側（`move_executor.py`）で処理されるため、ハンドラ内での重複ガードは不要。

- **move_status.py への Interrupt インポート追加**: 現在 `from jpoke.enums import Event, LogCode` のみであるため、`Interrupt` を追加する必要がある。
