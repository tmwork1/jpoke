# ひょうざんおろし 実装計画書

## 仕様要約（docs/spec/moves/ひょうざんおろし.md 参照）

* タイプ: こおり / 分類: 物理 / 威力: 120（Champions。SVまでは100） / 命中: 85 / PP: 10
* 対象: 相手1体 / 優先度: 0 / 接触なし（直接攻撃×）
* 効果: 追加効果として、30%の確率で相手をひるみ状態にする。

## 現状の実装状況（修正後）

`src/jpoke/data/moves/move_ha.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。

```python
"ひょうざんおろし": MoveData(
    type="こおり",
    category="physical",
    pp=10,
    power=120,
    accuracy=85,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ひょうざんおろし_apply_flinch,
        )
    }

```

```python
def ひょうざんおろし_apply_flinch(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_defender(battle, ctx, value, volatile="ひるみ", chance=0.3)
```

### 修正前の問題点

* `power=100` のままだった（Champions での威力変更 100→120 が未反映。SVまでの値のまま）。
  一次情報（`docs/wiki/moves/ひょうざんおろし.html`）のインフォボックスに明記されている
  世代差であり、`docs/champions/move_list.txt` の該当行（`ひょうざんおろし こおり 物理 120 85 12 …`）
  とも威力の値が一致する。README.md の対象範囲定義（Champions でルールが異なる場合は
  そちらを優先する）に基づき、`power=120` に修正する。
* PP については `docs/champions/move_list.txt` は `12` を記載しているが、これは PP アップ等の
  補正を含んだ値であり信頼できないことが既存レビュー（`docs/review/moves/であいがしら.md`・
  `docs/review/moves/つららおとし.md`・`docs/review/moves/いわなだれ.md`）で確認済みのため、
  一次情報どおり基礎 PP の `10` を維持する（修正不要）。
* 命中率 85 はSV・Championsとも共通のためこちらも修正不要。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | デフォルト（`MoveHandler` 既定値） | attacker:self | 30%の確率でひるみを付与する |

`_move_secondary_1.md` に記載されたひるみ付与技（いわなだれ・つららおとし等）と同じ
イベント（`ON_DAMAGE_HIT`）・共通ヘルパー（`apply_volatile_to_defender`）を使う定型実装のため、
priority は既存の同種ハンドラに合わせる。

## Priority 根拠

`docs/spec/turn.md` に `ON_DAMAGE_HIT` は掲載されていない（ダメージ計算内部のイベントのため）。
既存の同種ハンドラ（`いわなだれ_apply_flinch`・`つららおとし_apply_flinch` 等）がいずれも
`MoveHandler` のデフォルト priority で `Event.ON_DAMAGE_HIT` に登録されているため、
本技も同じ方針（デフォルト priority）を踏襲する。

## 注意点・エッジケース（review-test 向け）

1. **基本ケース**
   - 通常攻撃時、追加効果として 30% の確率で相手がひるみ状態になる。
   - 威力は 120（Champions 基準）でダメージ計算されること。
2. **アイテム相互作用**
   - `handlers/item.py` の `_INNATE_FLINCH_MOVES` に本技が含まれており、おうじゃのしるし・
     するどいキバによる追加ひるみ効果が重複しない（元々ひるみを追加効果に持つ技のため）。
     既存のパラメータ化テスト（`tests/test_item.py`）で一般的に検証済み。
3. **直接攻撃でないこと**
   - `flags` に `contact` を含まないため、ゴツゴツメット等の接触依存効果は発動しない。
4. **リーサル計算**
   - 対象外（`docs/progress/move.md` のリーサル実装列が `n/a`）。単発威力技で追加効果が
     ひるみ（HP に影響しない）のため、リーサル計算ハンドラは不要という既存方針
     （いわなだれ・つららおとし等と同様）に合致する。
