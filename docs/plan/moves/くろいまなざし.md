# 計画書: くろいまなざし

## 仕様要約

- ノーマルタイプ / 変化 / 命中100 / PP5 / 対象: 1体選択
- ゴーストタイプではない相手は逃げたり交代できなくなる
- 自分が場を離れると効果が消える
- ゴーストタイプには無効（`ON_TRY_MOVE_2` priority 120: `ゴーストタイプ: にげられない無効` でフレームワーク側が処理）
- すでに逃げられない状態の場合は失敗（`ON_APPLY_MOVE` priority 100: `にげられない技: すでに状態変化` でフレームワーク側が処理）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | 相手ににげられない状態付与 |

## Priority 根拠

`docs/spec/turn.md` の `ON_TRY_MOVE_2` priority 120 に `ゴーストタイプ: にげられない無効`、`ON_APPLY_MOVE` priority 100 に `にげられない技: すでに状態変化` が記載されており、失敗・無効チェックはフレームワーク側が処理する。ハンドラ本体は `ON_STATUS_HIT` priority 100 のパターンに従う。

## 注意点・エッジケース

- ゴーストタイプへの無効はフレームワーク側（`ON_TRY_MOVE_2` priority 120）が処理するためハンドラ内チェック不要
- すでに逃げられない状態の失敗チェックも `ON_APPLY_MOVE` priority 100 でフレームワーク側が処理
- とおせんぼうと同じ volatile（`にげられない`）を使用する
- くろいまなざしは命中率100を持つが、とおせんぼうは命中率なし（自分対象の違いに注意：どちらも相手に使う技で命中判定がある）
- 自分が場を離れると効果が消える処理は volatile 側で管理
