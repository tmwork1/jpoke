# 計画書: かなしばり

## 仕様要約

- タイプ: ノーマル / カテゴリ: 変化 / PP: 20 / 命中: 100 / 対象: 相手
- 相手が「直前に使用した技」を 4 ターン使用不能にする（第五世代以降）
- まだ PP を消費した行動をしていない相手に使うと失敗する（`executed_move is None` で判定）
- 相手がすでにかなしばり状態の場合は失敗する（`volatile_manager.apply` が False を返す）
- わるあがきに対して使うと失敗する
- 特性アロマベールに守られている相手には効果がない（フレームワーク側で阻止）
- みがわり状態の相手にも有効
- まもる: x / マジックコート: x

## Priority 根拠

`docs/spec/turn.md` の `ON_STATUS_HIT` には内部 priority 表が存在しない。  
既存の変化技ハンドラ（`アンコール_apply`、`ちょうはつ_apply_volatile_to_defender` 等）がすべて priority 100（`MoveHandler` デフォルト）を使用しており、かなしばりも同一パターンとして priority 100 を採用する。

turn.md の `Event.ON_CHECK_ACTION`（実装上の `ON_TRY_ACTION`）では「100: かなしばりで技が出せない」と明記されており、volatile 側のブロックが priority 100 であることも整合している。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: 五十音順「か」行 — `かたくなる_modify_attacker_stats`（line 423）の後、`からにこもる_modify_attacker_stats`（line 427）の前に挿入する。

```python
def かなしばり_can_apply(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """かなしばりの失敗条件を判定する。

    - 相手がまだ技を使っていない（executed_move が None）場合は失敗する
    - わるあがきに対して使うと失敗する
    """
    move = ctx.defender.executed_move
    if not move or move.name == "わるあがき":
        battle.add_event_log(ctx.attacker, LogCode.MOVE_FAILED,
                             payload={"reason": "かなしばり"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def かなしばり_apply_volatile_to_defender(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """かなしばりの効果: 相手に「かなしばり」状態を付与する（4 ターン）。"""
    move = ctx.defender.executed_move
    return apply_volatile_to_defender(battle, ctx, value, volatile="かなしばり",
                                      count=4, move_name=move.name)
```

### `src/jpoke/data/move.py`

## 注意点・エッジケース

- **`executed_move` の判定**: `executed_move` は `Pokemon.reset()` で None に戻される。交代直後に場に出たばかりのポケモン（`executed_move is None`）に使うと失敗する。アンコール（`アンコール_can_apply`）と同一パターン。
- **わるあがきへの失敗**: `executed_move.name == "わるあがき"` でガードする。第五世代以降の仕様に準拠。
- **すでにかなしばり状態の場合の失敗**: `volatile_manager.apply` が既に volatile がある場合 False を返すため、ハンドラ側での重複チェックは不要。`apply_volatile_to_defender` ヘルパーはその返り値をそのまま `HandlerReturn.value` に渡すため、自動的に失敗ログが出る。
- **アロマベール無効化**: turn.md の `Event.ON_APPLY_MOVE`（実装上 `ON_TRY_MOVE_2`）priority 100 に「アロマベール: かなしばり等無効」と明記されている。これはフレームワーク・アロマベール特性ハンドラ側が `ON_CHECK_BYPASS_STATUS_GUARD` で処理するため、かなしばりハンドラ側には記述しない。
- **move_name の保存**: `apply_volatile_to_defender` の `**kwargs` として `move_name=move.name` を渡す。`Volatile.__init__` が `move_name` 引数を受け取り `self.move_name` に格納する。volatile 側のハンドラ（`かなしばり_try_action` 等）は `mon.volatiles["かなしばり"].move_name` でアクセスする。
- **ハンドラの分離**: `can_apply` と `apply` を別 `MoveHandler` として `list` 登録することで、失敗時に stop_event=True が後続ハンドラを止める。アンコール（`アンコール_can_apply` + `アンコール_apply` の 2 ハンドラ構成）と同パターン。
- **PP が 0 の技に対しての挙動**: wiki によると第五世代以降は PP が 0 の技にも成功する。`executed_move` に PP=0 の技が入っていても失敗させないため、PP チェックは行わない。
