# 計画書: めいそう

## 仕様要約

- 自分の「とくこう」「とくぼう」ランクをそれぞれ1段階ずつ上げる
- タイプ: エスパー、変化、PP: 20、対象: 自分

出典: `docs/spec/moves/めいそう.md`

---

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100 (省略) | `attacker:self` (省略) | とくこう・とくぼうを各1段階上げる |

---

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`コスモパワー_modify_attacker_stats` 等）がすべて priority=100 (デフォルト) で登録されているため、100 を使用する。

---

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: `みをけずる_can_apply` の後、`メロメロ_apply` の前（五十音順）

```python
def めいそう_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"spa": 1, "spd": 1})
```

### `src/jpoke/data/moves/move_ma.py`

`"めいそう"` エントリに `handlers` を追加する:

```python
    "めいそう": MoveData(
        type="エスパー",
        category="status",
        pp=20,
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.めいそう_modify_attacker_stats,
            ),
        }
    ),
```

---

## 注意点・エッジケース

- spa と spd の両方を1つの `stats` dict にまとめて `modify_attacker_stats` に渡せる（コスモパワーと同パターン、spa/spd 版）
- どちらか一方でもランクが最大（+6）の場合は、そのランクの上昇のみ失敗し、もう一方は成功する（マネージャー内部処理）
- ランクがすでに最大の場合の失敗判定は `turn.md` Priority 130 でマネージャー内部処理されるため、ハンドラ側にガード不要
