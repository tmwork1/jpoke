詳細は `docs/spec/moves/フェイタルクロー.md` / `docs/review/moves/フェイタルクロー.md` を参照。

## 仕様要約

`docs/spec/moves/_move_secondary_1.md` の「どく」列（0=なし / 1=どく / 2=もうどく）が非0の攻撃技を実装する。

- **どく付与（値=1）**: キラースピン・クロスポイズン・シェルアームズ・スモッグ・ダストシュート・ダブルニードル・どくづき・どくばり・どくばりセンボン・ヘドロウェーブ・ヘドロこうげき・ヘドロばくだん・ポイズンアクセル・ポイズンテール
- **もうどく付与（値=2）**: どくどくのキバ・じゃどくのくさり
- **三択状態異常（どく/まひ/こおり 各1/3）**: フェイタルクロー

変化技（どくどく・どくのこな等）は本計画書の対象外。

## ハンドラ構成

すべての技で以下の共通パターンを使用する。

| イベント | Priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `Event.ON_HIT` | 100（MoveHandler デフォルト） | `attacker:self`（MoveHandler デフォルト） | 確率判定後にどく/もうどくを付与 |

フェイタルクロー のみ三択ランダムが必要なため、専用関数で実装する。

### Priority 根拠

`docs/spec/turn.md` の `Event.ON_HIT` テーブルには確率付与の追加効果専用エントリはない。既存実装（`でんじほう`・`チャージビーム`・`ほのおのまい` 等）がすべて `Event.ON_HIT` priority=100 を使用しているため、同じパターンを採用する。

def フェイタルクロー_apply_ailment_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """フェイタルクローの追加効果: どく/まひ/こおり のいずれかをランダムに付与する。

    確率0.5で発動し、発動した場合はどく・まひ・こおりを各1/3の確率で付与する。

    if chance < 1 and battle.random.random() >= chance:
        return HandlerReturn(value=value)
    ailment = battle.random.choice(["どく", "まひ", "こおり"])
    return HandlerReturn(value=battle.ailment_manager.apply(
        ctx.defender, ailment, source=ctx.attacker, ctx=ctx

## 注意点・エッジケース（review-test 向け）

1. **どく・はがねタイプへの無効化**
   `docs/spec/turn.md` の `Event.ON_TRY_MOVE_2` priority=130「どく・はがねタイプ: どく無効」が `ailment_manager.apply` の内部チェックで処理されるため、ハンドラ側に重複ガードは不要。

2. **めんえき特性**
   `Event.ON_TRY_MOVE_2` priority=140「めんえき等: どく無効」が `ailment_manager.apply` の内部チェックで処理されるため、ハンドラ側に重複ガードは不要。

3. **もうどく（どくどくのキバ・じゃどくのくさり）**
   `ailment="もうどく"` を指定する。通常のどく付与技と区別して実装すること。

4. **フェイタルクロー の三択実装**
   `battle.random.choice(["どく", "まひ", "こおり"])` でランダム選択後、各状態の付与失敗条件（まひ→でんきタイプ無効等）は `ailment_manager.apply` 内部で処理される。ハンドラでは選択した状態異常をそのまま渡せばよい。

5. **スモッグの命中率**
   `スモッグ` は命中率70（`accuracy=70`）が既存 MoveData で設定されている。

6. **ダブルニードルの連打**
   `ダブルニードル` は2回攻撃技だが、どく付与は「最後のヒット後に判定」ではなく「各ヒットごとに判定」される実装かどうか仕様書に明記なし。既存の他の2回技パターンを参照して判断が必要。MoveData の `multi_hit` 系フラグがあれば確認する。

7. **どくばりの「ひんしにする技」特性**
   `どくばり` は命中した場合に確率でどく付与（威力15）の通常攻撃。`どくどくのキバ` ともうどくの区別に注意。

8. **フェイタルクロー の確率0.5**
   三択発動のトリガー確率自体が0.5である（各状態への最終確率は約1/6ずつ）。`battle.resolve_secondary_chance` でてんのめぐみ等の補正が適用される。

9. **MoveData 未登録の3技**
   `ダブルニードル`・`ポイズンアクセル`・`じゃどくのくさり` は MoveData 自体が存在しないため、五十音順に従って新規追加する必要がある。
