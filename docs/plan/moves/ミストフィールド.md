# 実装計画書: ミストフィールド

## 仕様要約

- タイプ: フェアリー / カテゴリ: 変化 / PP: 10 / 対象: 全体の場（target="field"）
- 5 ターンの間、場の状態を「ミストフィールド」にする
- ひこうタイプや特性「ふゆう」などではない地面にいるすべてのポケモンは状態異常にならず、ドラゴン技のダメージが半減する
- また、道具「ミストシード」を持ったポケモンの「とくぼう」ランクが 1 段階上がる
- エレキフィールド等と同じくすでに同じフィールドの場合は失敗する

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | テレインを「ミストフィールド」に 5 ターン設定 |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「天気・フィールド: すでに同じ」が priority 100 として記載されている。同種ハンドラ `エレキフィールド_activate_terrain`・`グラスフィールド_activate_terrain`・`サイコフィールド_activate_terrain` と同じパターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「み」行 — `みがわり_apply` の後、`みきり_apply_volatile_to_attacker` の前（「みき」は「みが」の後、「みす」は「みき」の後のため「ミストフィールド」は「みき」と「みち」の間）に挿入する。

```python
def ミストフィールド_activate_terrain(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """ミストフィールド: 地形をミストフィールドにする。"""
    return HandlerReturn(value=battle.terrain_manager.apply("ミストフィールド", 5))
```

### `src/jpoke/data/move.py`

既存の `"ミストフィールド"` エントリに handlers を追加する。

```python
"ミストフィールド": MoveData(
    type="フェアリー",
    category="変化",
    pp=10,
    target="field",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ミストフィールド_activate_terrain,
        ),
    }
),
```

## 注意点・エッジケース

- `terrain_manager.apply` がすでに同じフィールドの場合は False を返すため、技ハンドラ側での重複チェックは不要
- フィールド効果（状態異常無効・ドラゴン半減・ミストシード発動）はテレインマネージャ側で処理されるため、技ハンドラ側では設定のみ行う
- `エレキフィールド_activate_terrain` と異なり `source=ctx.attacker` を渡さない（他の terrain ハンドラも渡していない）ことを実装時に統一する
