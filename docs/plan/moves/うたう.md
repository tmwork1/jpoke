# 実装計画: うたう

参照: docs/spec/moves/うたう.md

## 仕様サマリ

- タイプ: ノーマル / 分類: 変化 / PP: 16（champions/move_list.txtの値。Gen9本家は15） / 命中率: 55
- 効果: 相手を『ねむり』状態にする（ねむりターン数は `ailment_manager` 側の共通ロジックで決定）
- 音系の技（`sound` フラグ）: 相手の『みがわり』状態を貫通する
- まもる: ○ / マジックコート: ○ / 直接攻撃: ×
- 対象: 1体選択

## 実装状況（実装済み）

`data/moves/move_a.py` にエントリが存在し、`handlers` も設定済み。

```python
"うたう": MoveData(
    type="ノーマル",
    category="status",
    pp=16,
    accuracy=55,
    flags={"sound"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.うたう_apply_sleep,
        ),
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト、`MoveHandler` の既定値） | ねむり状態を相手に付与する |

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の同種ハンドラ（`あくまのキッス_apply_ailment_to_defender`、`でんじは_apply_ailment_to_defender` など）がすべて `MoveHandler(func)` のみ（priority=100 デフォルト）で実装されているため、同様に priority=100 を採用する。

## 実装箇所

### handlers/move_status.py

```python
def うたう_apply_sleep(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """うたうの効果: 相手をねむり状態にする。音系の技のためみがわりを貫通する。"""
    return apply_ailment_to_defender(battle, ctx, value, ailment="ねむり")
```

`apply_ailment_to_defender`（`handlers/move.py` 共通ヘルパー）を利用し、`ctx.defender` に対して
`battle.ailment_manager.apply(...)` を呼び出す。

### data/moves/move_a.py

`うたう` の `MoveData` に `handlers` を設定済み（上記コード参照）。

## 注意点・エッジケース

- ねむり付与の基本ロジック（すでに状態異常がある場合の失敗、同一状態異常の失敗）は `ailment_manager` 内部で処理されるため、ハンドラ側のガードは不要
- `sound` フラグにより `みがわり` 状態を貫通して命中する（みがわりへの当否は技フレームワーク側が処理）
- `さわぐ` 状態中はねむり付与が無効になる（`ON_TRY_MOVE_2` priority=130 で処理済み）
- `エレキフィールド` 中は地面にいるポケモンにねむりを付与できない（フレームワーク側で処理済み）
- 命中率 55 は低いため、テストでは `accuracy=100` 固定で実行する
