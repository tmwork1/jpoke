# 実装計画: うたう

## 仕様サマリ

- タイプ: ノーマル / 分類: 変化 / PP: 15 / 命中率: 55
- 効果: 相手を 2〜4 ターン（実質 1〜3 ターン）のねむり状態にする
- 音系の技（sound ラベル）: 相手のみがわり状態を貫通する
- まもる: ○ / マジックコート: ○ / 直接攻撃: ×
- 対象: 1体選択

## ハンドラ構成

|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | ねむり状態を相手に付与する |

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の同種ハンドラ（`あくまのキッス_apply_ailment_to_defender`、`でんじは_apply_ailment_to_defender` など）がすべて `MoveHandler(func)` のみ（priority=100 デフォルト）で実装されているため、同様に priority=100 を採用する。

## 実装箇所

### handlers/move_status.py への追加

```python
def うたう_apply_ailment_to_defender(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """うたうの効果: 相手をねむり状態にする。音系の技のためみがわりを貫通する。"""
    return apply_ailment_to_defender(battle, ctx, value, ailment="ねむり")
```

### data/move.py の更新

`うたう` の MoveData に `handlers` を追加する。

```python
"うたう": MoveData(
    type="ノーマル",
    category="変化",
    pp=15,
    accuracy=55,
    labels=["sound"],
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.うたう_apply_ailment_to_defender,
        ),
    }
),
```

## 注意点・エッジケース

- ねむり付与の基本ロジック（すでに状態異常がある場合の失敗、同一状態異常の失敗）は `ailment_manager` 内部で処理されるため、ハンドラ側のガードは不要
- sound ラベルにより `みがわり` 状態を貫通して命中する（みがわりへの当否は技フレームワーク側が処理）
- `さわぐ` 状態中はねむり付与が無効になる（`ON_TRY_MOVE_2` priority=130 で処理済み）
- `エレキフィールド` 中は地面にいるポケモンにねむりを付与できない（フレームワーク側で処理済み）
- 命中率 55 は低いため、テストでは `accuracy=100` 固定で実行する
