# 計画書: しおみず（再レビュー）

## 仕様要約

* 相手の現在HPが最大HPの半分以下のとき、威力が2倍になる（`docs/spec/moves/しおみず.md` 参照）。
* タイプ: みず / 分類: 特殊 / 威力: 65 / 命中: 100 / PP: 10 / 対象: 相手1体。

## 既存実装の確認結果

`src/jpoke/data/move.py` の `"しおみず"` は `handlers={}`（コメント「追加効果なし」）となっており、
一次情報にある「相手のHPが半分以下のとき威力2倍」の効果が一切実装されていなかった。
これはバグであり、修正する。

## ハンドラ構成（新規追加）

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| しおみず | `Event.ON_CALC_POWER_MODIFIER` | デフォルト（100） | `attacker:self` / `defender:self`（`MoveHandler` デフォルト） | 対象の現在HPが最大HPの半分以下なら威力を2倍（modifier=8192）にする |

* `Event.ON_CALC_POWER_MODIFIER` は `docs/spec/turn.md` に priority の記載がないため、
  既存の同種ハンドラ（`たたりめ_double_power_when_ailment` / `ベノムショック_double_power_when_poisoned`
  ほか、`ON_CALC_POWER_MODIFIER` を使う全ての `MoveHandler`）が明示的な priority 指定なし
  ＝デフォルト値（100）で登録されている慣例に合わせ、デフォルトのままとする。

```python
def しおみず_double_power_if_defender_hp_half_or_less(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """しおみず: 対象の現在HPが最大HPの半分以下のとき威力が2倍になる。"""
    if ctx.defender.hp * 2 <= ctx.defender.max_hp:
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)
```

```python
"しおみず": MoveData(
    type="みず",
    category="special",
    pp=10,
    power=65,
    accuracy=100,
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
            ha.しおみず_double_power_if_defender_hp_half_or_less,
        ),
    }
),
```

`たたりめ_double_power_when_ailment` / `ベノムショック_double_power_when_poisoned` と同じ
`apply_fixed_modifier(value, 8192)` パターンを使い、条件式のみ対象HPの半分以下判定に置き換える。

## リーサル計算

`docs/progress/move.md` の該当行は現状「リーサル実装: -」となっていたが、レビューの結果
`n/a` に修正する。理由は `docs/spec/moves/しおみず.md` の「リーサル計算」節に記載の通り、
対象の現在HPに連続的・閾値的に依存する威力可変技（`にぎりつぶす`・`ハードプレス`・`しぼりとる`）が
いずれも専用の `lethal_handlers` を持たず `n/a` 扱いになっている既存の割り切りに、
しおみずも合わせるため。`battle.calc_damages` が代表HP値（`_update_hp` 由来）を用いて
`Event.ON_CALC_POWER_MODIFIER` を自動的に発火させるため、追加のリーサルハンドラは不要。
