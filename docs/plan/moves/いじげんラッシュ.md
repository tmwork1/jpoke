# いじげんラッシュ 実装計画書

## 仕様要約（docs/spec/moves/いじげんラッシュ.md 参照）

* タイプ: あく / 分類: 物理 / 威力: 100 / 命中: - （必中） / PP: 5
* 対象: 相手1体 / 優先度: 0 / 接触なし
* フラグ: unprotectable（まもる無視）、bypass_substitute（みがわり無視）
* 効果: 必中技。まもる系統・みがわりを無視して攻撃する。命中成功時、自分の防御が1段階下がる。

## 現状の実装状況（修正前）

```python
"いじげんラッシュ": MoveData(
    type="あく",
    category="physical",
    pp=5,
    power=100,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.いじげんラッシュ_lower_attacker_def,
        )
    }
),
```

### 発見した誤り

1. **`accuracy=100` が誤り**: 一次情報（Wiki）で「命中率: -」「命中判定: 必中」と明記されている。
   `move_executor.py._check_hit()` は `move.accuracy is None` の場合のみ命中判定をスキップして必中とするため、
   `accuracy=100` のままでは通常の命中率100技として扱われ、回避ランク等の影響を受けてしまう
   （`あてみなげ` 等の必中技は `accuracy=None` で実装されている）。
   → `accuracy=None` に修正する。

2. **`flags` に `unprotectable` / `bypass_substitute` が欠落**: 一次情報に
   「まもる系統の状態を取り除いて攻撃する」「相手のみがわり状態を無視する」と明記されている。
   `model/move.py` の `is_blocked_by_protect` は `unprotectable` フラグ、
   `core/move_executor.py.check_hit_substitute()` は `bypass_substitute` フラグを参照するため、
   これらが無いと通常の技としてまもる・みがわりに防がれてしまう。
   同種の効果を持つ `いじげんホール` は `flags={"unprotectable", "bypass_substitute"}` を持つ。
   → `flags` に `"unprotectable"`, `"bypass_substitute"` を追加する。

3. **`flags` の `secondary_effect` が誤り**: 一次情報の判定表では「追加効果: ×」であり、
   自分の防御ランクを下げる効果は「みずからのステータス低下」であって対象への追加効果ではない。
   Bulbapedia（Sheer Force / ちからずく の項目）によれば、ちからずくは対象への追加効果のみを
   無効化・強化の対象とし、**使用者自身のランクダウンは対象外**と明記されている
   （インファイト・じしんのランク低下等と同様の扱い）。
   本プロジェクトの `secondary_effect` フラグは `ちからずく_boost` / `ちからずく_disable_secondary_effect`
   （`handlers/ability.py`）、`いのちのたま_recoil` 等（`handlers/item.py`）で「ちからずく対象技」判定に
   使われているため、このフラグを付けたままだとちからずく所持者が使用した際に
   本来発生するはずの防御低下が無効化され、威力が誤って1.3倍されてしまう。
   → `flags` から `"secondary_effect"` を削除する（`アーマーキャノン` と同様、自分ランクダウンのみの技として扱う）。

4. **まもる系状態の解除処理が欠落**: 一次情報に「まもる系統の状態を取り除いて攻撃する」と明記されている。
   `unprotectable` フラグは「まもるによってダメージが防がれない」効果のみを実現し、
   相手の `まもる` 揮発性状態そのものを解除する効果は含まれない。
   同じ効果文言を持つ `いじげんホール`（`data/move.py`）・`フェイント`（同ファイル）は、それぞれ専用の
   `いじげんホール_remove_protect` / `フェイント_remove_protect`（`handlers/move_attack.py`、共通の
   `_FAINT_REMOVE_VOLATILES` タプルを使用）を ON_HIT に登録して相手の `まもる`・`みきり`・`トーチカ`・
   `キングシールド`・`ニードルガード`・`スレッドトラップ`・`かえんのまもり` を解除している。
   いじげんラッシュにはこの処理が無かったため、`まもる` 状態の相手にダメージは通るものの
   `まもる` 揮発性状態自体が消えずに残ってしまう不具合があった。
   → `いじげんラッシュ_remove_protect`（`いじげんホール_remove_protect` と同一パターン）を新規実装し、
     ON_HIT に追加登録する。

修正後:

```python
"いじげんラッシュ": MoveData(
    type="あく",
    category="physical",
    pp=5,
    power=100,
    accuracy=None,
    flags={"unprotectable", "bypass_substitute"},
    handlers={
        Event.ON_HIT: [
            h.MoveHandler(ha.いじげんラッシュ_remove_protect),
            h.MoveHandler(ha.いじげんラッシュ_lower_attacker_def),
        ]
    }
),
```

```python
def いじげんラッシュ_remove_protect(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """いじげんラッシュの追加効果: 相手のまもる系揮発性状態を解除する。"""
    defender = ctx.defender
    for volatile in _FAINT_REMOVE_VOLATILES:
        if defender.has_volatile(volatile):
            battle.volatile_manager.remove(defender, volatile)
            battle.add_event_log(
                defender,
                LogCode.VOLATILE_REMOVED,
                payload={"volatile": volatile, "reason": "いじげんラッシュ"}
            )
    return HandlerReturn(value=value)
```

## ハンドラ構成

| Event | Priority | subject_spec | 役割 |
|---|---|---|---|
| ON_HIT | 100（デフォルト） | attacker:self（MoveHandlerのデフォルト） | 相手のまもる系揮発性状態を解除する |
| ON_HIT | 100（デフォルト） | attacker:self（MoveHandlerのデフォルト） | 命中成功時、自分の防御を1段階下げる |

`handlers/move_attack.py` の `いじげんラッシュ_lower_attacker_def` は既存実装のまま変更不要
（`modify_attacker_stats(battle, ctx, value, stats={"def": -1})` で確率100%・自分対象のランクダウンを実装済み）。
`いじげんラッシュ_remove_protect` は新規実装（`いじげんホール_remove_protect` と同一パターン）。
2つのハンドラは異なる対象（defenderの揮発性状態／attackerのランク）に作用するため実行順序に依存関係はない。

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_HIT` 表には「自分のランクが下がる反動」が priority 10 として
記載されている。しかし本プロジェクトの既存実装（`アーマーキャノン`・`アームハンマー`・`インファイト`等、
いずれも命中成功時に自分のランクを下げる技）は例外なく `MoveHandler` のデフォルト値（priority=100）で
登録されており、priority=10 を明示している例は無い。
一貫性を優先し、既存の同種技と同じ priority=100（デフォルト）のまま変更しない。

## まもる・みがわり無視の実装根拠

* `unprotectable`: `model/move.py` の `is_blocked_by_protect` が参照。フラグがあれば
  `target == "foe"` でも「まもるで防がれる技」と判定されなくなる。
* `bypass_substitute`: `core/move_executor.py` の `check_hit_substitute()` が参照。フラグがあれば
  みがわりに命中判定を行わず、本体にダメージを与える。
* いずれも `いじげんホール`（`data/move.py` 655行目付近）と同一の実装パターン。
  ただしダメージが「防がれない」ことと、まもる系揮発性状態を「解除する」ことは別処理であり、
  後者は `いじげんラッシュ_remove_protect` ハンドラで別途実装する（上記4番参照）。

## 命中判定（必中）の実装根拠

* `accuracy=None` の場合、`core/move_executor.py._check_hit()` が
  「命中率がNoneなら必中」として即座に `True` を返す（ON_MODIFY_ACCURACY 等のイベントも発火しない）。
* これにより、相手の回避ランクや自分の命中ランク補正の影響を受けずに必ず命中する
  （`あてみなげ` 等、既存の必中技と同一パターン）。

## 注意点・エッジケース（review-test 向け）

1. **必中の検証**: `test_option.accuracy` を上書きせず、相手の回避ランクを+6にしても命中することを確認する
   （`あてみなげ` の既存テストと同パターン）。
2. **まもる貫通・解除の検証**: `volatile1={"まもる": 1}` を設定した相手にダメージが通り、
   `いじげんラッシュ_remove_protect` によって `まもる` 揮発状態自体も解除されることを確認する
   （`いじげんホール` の既存テストと同パターン）。
3. **みがわり貫通の検証**: `battle.volatile_manager.apply(defender, "みがわり", hp=999)` を設定した相手の
   本体HPが減り、みがわりのHPは変化しないことを確認する（`いじげんホール` の既存テストと同パターン）。
4. **自分の防御低下**: 既存テスト `test_いじげんラッシュ_防御1段階低下が発動する` は変更不要
   （`accuracy=100` の test_option 上書きで命中を固定しているため、move側の `accuracy=None` 修正の影響は受けない）。
5. **ちからずく非対象**: `secondary_effect` フラグを削除したことで、ちからずく所持者が使用しても
   威力1.3倍・防御低下無効化が発生しなくなる（今回のテスト追加範囲では未検証、必要なら
   `ちからずく` 関連の既存テストパターンを参照して追加を検討する）。
