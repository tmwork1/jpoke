# 計画書: じこあんじ

## 仕様要約

- タイプ: ノーマル / カテゴリ: 変化 / PP: 10 / 命中: なし（必中） / 対象: 相手（foe）
- 相手の能力ランク変化（A/B/C/D/S/ACC/EVA）をすべて自分にコピーする
- 相手のランクは変化しない（コピーのみ、入れ替えではない）
- まもる: × / マジックコート: × / よこどり: × (第五世代以降)
- `docs/spec/moves/status_move.md` 記載: 「相手の能力ランクの変化を、すべて自分にコピーする。」
- `docs/spec/moves/_all_moves.md` 記載: accuracy なし、pp=12（実データは10）

## Priority 根拠

`docs/spec/turn.md` には `ON_STATUS_HIT` の内部 priority 表は掲載されていない。既存の変化技ハンドラ（`ガードスワップ_swap_ranks`・`くろいきり_reset_all_ranks`・`いたみわけ_equalize_hp` 等）はすべて priority 100（`MoveHandler` デフォルト）を使用している。じこあんじも同一パターンで priority 100 を採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: 五十音順「じ」行 — `さむいギャグ_activate_weather_and_pivot` の後、`じこさいせい_heal_self` の前（さ行終わりから「じこ」の順）。

ランクのコピーは `mon.rank` dict を直接代入する。`battle.modify_stats()` はランク変化量をデルタ加算する API のため「上書きコピー」には不向きであり、また ON_BEFORE_MODIFY_STAT / ON_MODIFY_STAT を経由するとクリアボディ等のランク低下防止特性に干渉される。じこあんじは「自分の技の効果で自分のランクが変わる」ケースであり、仕様上クリアボディ・はとむね等は発動しない（自分から使った場合は防げない）。これは直接代入によって自然に実現できる。

```python
def じこあんじ_copy_ranks(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """じこあんじ: 相手の能力ランク変化をすべて自分にコピーする。

    相手のランクは変化しない。コピー後、自分のランクが変化したことに
    よるしろいハーブ等のアイテム・特性トリガーを発火させるため、
    ランク変化ログを出力する。
    """
    attacker = ctx.attacker
    defender = ctx.defender
    rank_stats: list[Stat] = ["A", "B", "C", "D", "S", "ACC", "EVA"]
    for stat in rank_stats:
        attacker.rank[stat] = defender.rank[stat]
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

既存の `"じこあんじ"` エントリに `handlers` を追加する。accuracy は仕様上「必中（なし）」のため None のままとする。

```python
"じこあんじ": MoveData(
    type="ノーマル",
    category="変化",
    pp=10,
    accuracy=100,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.じこあんじ_copy_ranks,
        ),
    }
),
```

注: 現在の `data/move.py` では accuracy=100 と記載されているが、`docs/spec/moves/status_move.md` の仕様では命中判定なし（必中）と記載されている。accuracy フィールドの変更要否は実装時に確認すること。

## 注意点・エッジケース

- **ランク直接代入の是非**: `battle.modify_stats()` はデルタ加算 API のため「現在値の上書き」には不向き。また ON_MODIFY_STAT イベントを経由するとクリアボディ・はとむね・あまのじゃく等が干渉する可能性がある。じこあんじは「自分から自分のランクをコピーする」ためこれらの特性は本来発動しない（仕様: クリアボディ系は自分の技効果で自分のランクが下がるときは発動しない）。直接代入を採用し、干渉を避ける。

- **たんじゅん・あまのじゃくの非適用**: `docs/spec/abilities/たんじゅん.md` に「じこあんじを使用したとき」はたんじゅんの倍増効果が適用されないと明記されている。直接代入によりこれが自然に実現される（modify_stats 経由ではないため ON_MODIFY_STAT で倍増処理に渡らない）。

- **あまのじゃくの非適用**: `docs/spec/abilities/あまのじゃく.md` に「じこあんじによる能力変化は反転しない」と明記されている。直接代入により自然に実現される。

- **しろいハーブの発動**: `docs/spec/items/しろいハーブ.md` に「へんしん状態やじこあんじで-1以下のランク補正をコピーした場合も発動する」と記載されている。直接代入後にしろいハーブがトリガーされるかは、battle のランク変化監視フロー次第。ランク変化ログが必要な場合は実装時に `battle.add_event_log` を追加する。

- **ものまねハーブ・びんじょうの非発動**: `docs/spec/items/ものまねハーブ.md`・`docs/spec/abilities/びんじょう.md` に「じこあんじでランクが上がったときは発動しない」と明記されている。直接代入でランク変化イベントを経由しないため、自然に非発動となる。

- **だっしゅつパックの非発動**: `docs/spec/items/だっしゅつパック.md` に「じこあんじによりランクが下がっても発動しない」と明記されている。直接代入により自然に実現される。

- **かいりきバサミ・はとむね等の非発動**: `docs/spec/abilities/かいりきバサミ.md` に「じこあんじなどで自分の技の効果で攻撃が下がるときは発動しない」と明記されている。直接代入により自然に実現される。

- **コピー後のランク値の範囲**: コピー元（相手）のランクは ±6 の範囲内に収まっているため、コピー後も範囲外になることはない。clamp 処理は不要。

- **コピー元（相手）のランクが全て0の場合**: 自分のランクが全て0に上書きされる。既存のランクがプラスの場合は実質ランクが下がる処理になる。この場合 ON_TRY_MOVE_2 の「ランク補正下げ: すでに最低」チェックは通らないため（全0上書きはstatus_move.mdの「ランク変化なし」失敗条件には該当しない可能性がある）、要確認。

- **よこどり非対象（第五世代以降）**: `docs/spec/moves/よこどり.md` に「じこあんじ/つぼをつく - 第五世代以降はよこどりできない」と明記されている。ポケモンチャンピオンズは第九世代ベースのため、よこどり対象外として扱う。

- **オウムがえし非対象**: `docs/spec/moves/オウムがえし.md` に「対象の状態を参照する技: じこあんじ」として列挙されており、オウムがえしで使えない技の一つ。
