# 計画書: 攻撃技_反撃系（カウンター・ミラーコート・ほうふく・メタルバースト）

## 仕様要約

### カウンター
- かくとう / 物理 / 威力 - / 命中 100 / PP 20 / 優先度 -5
- そのターンに相手から受けた **物理** ダメージの **2倍** を与える固定ダメージ技
- ゴーストタイプには無効。それ以外のタイプ相性は適用されない
- 対象ダメージがない場合は失敗

### ミラーコート
- エスパー / 特殊 / 威力 - / 命中 100 / PP 20 / 優先度 -5
- そのターンに相手から受けた **特殊** ダメージの **2倍** を与える固定ダメージ技
- あくタイプには無効。それ以外のタイプ相性は適用されない
- 対象ダメージがない場合は失敗

### ほうふく
- あく / 物理 / 威力 - / 命中 100 / PP 10 / 優先度 なし
- そのターンに受けた最後のダメージ（物理・特殊問わず）の **1.5倍** を与える固定ダメージ技
- タイプ相性の影響を受けない
- ダメージを受けていない時は失敗

### メタルバースト
- はがね / 物理 / 威力 - / 命中 100 / PP 10 / 優先度 なし
- そのターンに受けた最後のダメージ（物理・特殊問わず）の **1.5倍** を与える固定ダメージ技
- タイプ相性の影響を受けない
- ダメージを受けていない時は失敗

## 前提: Pokemon モデルへのフィールド追加

「直前に受けたダメージ」を技側から参照するために、`Pokemon` に以下のフィールドを追加する必要がある。
現在 `Pokemon` には `hits_taken`（被弾回数）しか存在せず、ダメージ量と技カテゴリは記録されていない。

### 追加するフィールド（`src/jpoke/model/pokemon.py`）

```python
# バトル中に利用する属性はコンストラクタで明示的に定義する。
self.last_physical_damage_received: int = 0  # 今ターン受けた物理ダメージ合計
self.last_special_damage_received: int = 0   # 今ターン受けた特殊ダメージ合計
self.last_damage_received: int = 0           # 今ターン最後に受けたダメージ量
```

`reset_turn_state` に以下を追加する:
```python
def reset_turn_state(self):
    """ターン初期化処理。"""
    self.hits_taken = 0
    self.last_physical_damage_received = 0
    self.last_special_damage_received = 0
    self.last_damage_received = 0
```

### 追跡ハンドラの追加（グローバルハンドラとして登録）

被弾時に defender の `last_*_damage_received` を更新するハンドラが必要。
これはすべての攻撃技に共通するため、`event_manager` またはグローバルなシステムハンドラとして実装する方式を検討する。

**代替案（よりシンプル）**: `move_executor.py` の `_execute_attack_hit` 内で、`actual_damage` 確定後に直接記録する。

```python
# src/jpoke/core/move_executor.py の _execute_attack_hit 内に追記
# actual_damage が確定した後（約 L432 付近）
if actual_damage > 0:
    ctx.defender.hits_taken += 1
    ctx.defender.last_damage_received = actual_damage  # 追加
    if battle.query.deals_physical_damage(ctx.attacker, ctx.move):
        ctx.defender.last_physical_damage_received += actual_damage  # 追加
    else:
        ctx.defender.last_special_damage_received += actual_damage   # 追加
```

この方式が最もシンプルで、ハンドラ登録なしに全技で機能する。

## ハンドラ構成

### カウンター・ミラーコート（失敗判定 + 固定ダメージ）

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_TRY_MOVE_1` | 30 | `attacker:self` | 対象ダメージが 0 なら `stop_event=True` で失敗 |
| `ON_MODIFY_MOVE_DAMAGE` | 100（デフォルト） | `attacker:self` | ダメージ × 2 を返す |

### ほうふく・メタルバースト（失敗判定 + 固定ダメージ）

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_TRY_MOVE_1` | 30 | `attacker:self` | `last_damage_received == 0` なら失敗 |
| `ON_MODIFY_MOVE_DAMAGE` | 100（デフォルト） | `attacker:self` | ダメージ × 1.5（切り捨て）を返す |

## Priority 根拠

### 技の優先度（`priority` フィールド）
- カウンター: **-5**（`docs/progress/move.md` の仕様欄に明記）
- ミラーコート: **-5**（同上）
- ほうふく・メタルバースト: 優先度なし（0）

### イベント内 priority
- `ON_TRY_MOVE_1` の priority=30 は `docs/spec/turn.md` に明記:
  > `30 | カウンター・ミラーコート: 適格ダメージ無`
  > `30 | がまん: 跳ね返すダメージが無い`
  
  ほうふく・メタルバーストも同一カテゴリ（受けたダメージの存在確認）のため priority=30 を採用する。

- `ON_MODIFY_MOVE_DAMAGE`: 既存の `がむしゃら`、`ちきゅうなげ`、`いのちがけ` がすべてデフォルト priority=100 を使用しているため同様に採用。

### `src/jpoke/handlers/move_attack.py`

五十音順で適切な位置に以下の関数を追加する:

```python
def カウンター_check_can_use(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """カウンターの使用可否を判定する。

    そのターン相手から物理ダメージを受けていない場合は失敗する。
    """
    if ctx.attacker.last_physical_damage_received <= 0:
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload={"reason": "カウンター"}
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def カウンター_modify_damage(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """カウンターの固定ダメージを計算する（物理被弾ダメージ × 2）。"""
    return HandlerReturn(value=ctx.attacker.last_physical_damage_received * 2)


def ほうふく_check_can_use(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ほうふくの使用可否を判定する。

    そのターンダメージを受けていない場合は失敗する。
    """
    if ctx.attacker.last_damage_received <= 0:
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload={"reason": "ほうふく"}
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def ほうふく_modify_damage(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ほうふくの固定ダメージを計算する（最後に受けたダメージ × 1.5、切り捨て）。"""
```

## 注意点・エッジケース（review-test 向け）

### ダメージ追跡のターン境界
- `last_physical_damage_received`、`last_special_damage_received`、`last_damage_received` はターン開始時（`reset_turn_state`）に 0 にリセットされる
- 同一ターンに複数回ダメージを受けた場合:
  - `last_physical_damage_received` と `last_special_damage_received` は **累積加算**（カウンターは今ターン受けた物理ダメージ合計 × 2 が正しい挙動と思われる。ダブルバトル仕様では「最後に受けた技のみ」だが、シングルでは最後の相手からしかダメージを受けないため事実上同じ）
  - `last_damage_received` は **上書き**（「最後に受けた技のダメージ」）

### ほうふく・メタルバーストの違い
- 技名・タイプ・カテゴリが異なるが、動作は同一
- ほうふくは `contact` ラベルあり（現在の定義どおり）
- メタルバーストは `contact` ラベルなし（現在の定義どおり）

### カウンターとミラーコートのタイプ無効
- カウンター: ゴーストタイプの相手には無効（タイプ相性ロジックが適用される）
- ミラーコート: あくタイプの相手には無効
- これらは通常のタイプ相性チェックで処理されるため、ハンドラ側のガードは不要

### 固定ダメージとタイプ相性
- 仕様書 7.2 より: 無効化（0倍）は適用される。半減・倍化は適用されない
- ただしカウンター・ミラーコートは「タイプ相性の影響を受けない」とあるため、`ON_MODIFY_MOVE_DAMAGE` で返した値がそのままダメージになることを確認する必要がある
- ほうふく・メタルバーストも同様に「タイプ相性の影響を受けない」
- 既存の `がむしゃら`、`ちきゅうなげ`、`いのちがけ` で同様の固定ダメージが動作していることを確認し、タイプ相性が既に除外されているかを確認すること

### 先制・後攻と発動タイミング
- カウンター・ミラーコートは優先度 -5 のため必ず後攻になる
- そのため「相手が先に攻撃してダメージを与え、後から反撃する」のが通常の流れ
- 同一優先度 -5 内で相手も優先度 -5 の技を使う場合（カウンター同士など）は素早さで決定される

### きあいパンチとの類似点
- `きあいパンチ` は `ON_TRY_MOVE_1` priority=30 + `ctx.attacker.hits_taken > 0` で判定
- カウンター等は `ON_TRY_MOVE_1` priority=30 + `last_*_damage_received <= 0` で判定
- 同一イベント・同一 priority のため、`きあいパンチ` のパターンをそのまま流用可能

### みがわりの扱い
- みがわりへの攻撃でも `ON_MODIFY_DAMAGE` 後に `actual_damage` が確定する
- みがわりが被弾した場合にもダメージが `last_*` に記録されるかどうかを `move_executor.py` で確認する必要がある
- みがわりへのダメージはカウンター等で返せないはずなので、みがわりが被弾した場合は記録しないよう `move_executor.py` で条件分岐が必要になる可能性がある
  - 要確認: `ON_MODIFY_DAMAGE` でみがわりが `stop_event=True` を返す場合、`actual_damage` はどう扱われるか

### がまんとの差異
- `がまん` は「2ターン受けたダメージ合計 × 2」だが、カウンターは「今ターン受けた物理ダメージ合計 × 2」
- 実装は別々に管理する（`last_*_damage_received` はターン内リセットのみ）
