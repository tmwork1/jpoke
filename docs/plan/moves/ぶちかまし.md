# 実装計画書: ぶちかまし

参照仕様書: docs/spec/moves/ぶちかまし.md

## 仕様要約

* タイプ: じめん / 分類: 物理 / 威力: 120 / 命中率: 100 / PP: 5 / 対象: 1体選択 / 優先度: 0
  / 直接攻撃 / パンチわざ
* 技が成功しダメージを与えた後、自分の『ぼうぎょ』『とくぼう』ランクが1段階ずつ下がる
  （確率100%・確定効果、追加効果には分類されない）。

## 既存実装の確認結果

`src/jpoke/data/moves/move_ha.py` に既に以下の通り実装済みであることを確認した。

```python
"ぶちかまし": MoveData(
    type="じめん",
    category="physical",
    pp=5,
    power=120,
    accuracy=100,
    flags={"contact", "punch", "secondary_effect"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.ぶちかまし_lower_defender_spd,
        )
    }
),
```

* `handlers/move_attack.py` の `ぶちかまし_lower_defender_spd` は
  `modify_attacker_stats(battle, ctx, value, stats={"def": -1, "spd": -1})` を呼び出しており、
  `chance` を指定していないためデフォルトの100%（確定効果）で発動する。`battle.modify_stats`
  経由のため、あまのじゃく等の特性補正も自動的に適用される。
* `Event.ON_HIT` の `MoveHandler` は `subject_spec` を省略した場合デフォルトで `"attacker:self"`
  になる（`src/jpoke/handlers/move.py` の `MoveHandler.__init__`）。本ハンドラは攻撃者自身の
  ランクを変化させるため、この既定値と一致しており問題ない。
* 関数名 `ぶちかまし_lower_defender_spd` の「defender」はスタッツ名「ぼうぎょ(def)」の略であり、
  既存の `インファイト_lower_defender_stats` と同じ命名パターンを踏襲している（実際に変化する
  ポケモンは `ctx.attacker` 自身であり、相手側 `ctx.defender` ではない点に注意）。命名は既存の
  同系統実装（`インファイト`）と揃っており、本タスクでは範囲外のためリネームは行わない。
* `Event.ON_HIT` は `docs/spec/turn.md` で「自分のランクが下がる反動」が priority=10 と
  記載されているが、既存の同種実装（`インファイト_lower_defender_stats`、
  `アーマーキャノン_lower_attacker_stats`、`ばかぢから_lower_attacker_def` 等）はいずれも
  `priority` を指定せずデフォルト（100）のまま登録されている。本技も同じ関数群と同一パターンの
  ため、既存実装のデフォルト優先度を踏襲し変更しない。

### 不整合の修正: `secondary_effect` フラグ

一次情報（wiki）のぶちかまし判定表に「追加効果: ×」と明記されている（ばかぢから・
アーマーキャノンと同様）。`secondary_effect` フラグは `handlers/ability.py` の
`ちからずく_boost` / `ちからずく_disable_secondary_effect` が参照しており、「追加効果」を
持つ技にのみちからずくの威力上昇・追加効果無効化が適用される仕組みになっている。

ぶちかまし自身の能力ランクダウンは「追加効果」に分類されない（ばかぢからと同じ扱い）ため、
既存実装の `flags={"contact", "punch", "secondary_effect"}` から `secondary_effect` を削除し、
`flags={"contact", "punch"}` に修正する。

（参考: `インファイト`・`Vジェネレート` は同系統の技だが `secondary_effect` フラグが残ったままに
なっている。これらは本タスクの対象外のため今回は修正しない。）

### リーサル計算の確認結果

`docs/progress/move.md` の「リーサル実装」列は `n/a`。ぶちかましが下げるのは『ぼうぎょ』
『とくぼう』であり、攻撃者自身の以降の与ダメージには影響しない（`ばかぢから` の計画書と同じ
方針）。よって `lethal_handlers` は登録不要であり、`n/a` は妥当。

## 修正内容

* `src/jpoke/data/moves/move_ha.py`: `ぶちかまし` の `flags` から `secondary_effect` を削除
  （`{"contact", "punch"}` に修正）。
* `docs/progress/move.md`: `ぶちかまし` 行の PP 列が `8` になっていたが、一次情報・実装
  （`pp=5`）と一致する `5` に修正する。
