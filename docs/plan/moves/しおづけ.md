# 計画書: しおづけ

## 仕様要約

- いわタイプ / 物理 / 威力40 / 命中100 / PP15 / 対象: 相手1体 / 直接攻撃なし
- 追加効果（100%）として、命中した相手を『しおづけ』揮発性状態にする
- しおづけ状態のターン終了時ダメージ・解除条件などは揮発性状態側の実装（`docs/spec/volatiles/しおづけ.md`、`data/volatile.py`、`handlers/volatile.py`）に委ねる
- 詳細仕様は `docs/spec/moves/しおづけ.md` を参照

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | デフォルト | attacker:self（省略時デフォルト） | 相手にしおづけ状態を付与 |

## Priority 根拠

`ON_DAMAGE_HIT` は `docs/spec/turn.md` に未掲載のダメージ計算内部イベント（技命中直後・ダメージ確定後に発火）。
同種の追加効果ハンドラである `シェルブレード_lower_defender_def`（`ON_DAMAGE_HIT`、priority省略）に倣い、
デフォルト priority を使用する。

## 実装箇所（実装済み）

### handlers/move_attack.py

```python
def しおづけ_apply_volatile_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """しおづけの追加効果: 100%の確率で相手をしおづけ揮発状態にする。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="しおづけ")
```

### data/move.py

```python
    "しおづけ": MoveData(
        type="いわ",
        category="physical",
        pp=15,
        power=40,
        accuracy=100,
        flags={"secondary_effect"},
        handlers={
            Event.ON_DAMAGE_HIT: h.MoveHandler(
                ha.しおづけ_apply_volatile_to_defender,
            )
        },
        lethal_handlers={
            LethalEvent.ON_HIT: LethalHandler(l.しおづけ_apply_volatile)
        }
    ),
```

## 注意点・エッジケース

- `secondary_effect` フラグにより、ちからずく特性（威力1.3倍・追加効果無効化）、りんぷん特性・おんみつマント（追加効果無効化）は
  `apply_volatile_to_defender` 内の `resolve_secondary_chance` を通じて汎用的に処理される。ハンドラ側に個別のチェックは不要。
- みがわり状態の相手に技が防がれた場合、`ON_MODIFY_MOVE_DAMAGE` でダメージが0になり `actual_damage <= 0` となるため
  `ON_DAMAGE_HIT` 自体が発火しない（`move_executor._execute_hit` の共通処理）。ハンドラ側にみがわりチェックは不要。
- 直接攻撃技ではないため `contact` フラグは付与しない。
- ゆびをふる（Metronome）で選ばれない仕様があるが、ゆびをふる自体が本プロジェクトで未実装のため対応不要。
- リーサル計算（`handlers/lethal.py` の `しおづけ_apply_volatile`）は `ctx.move_secondary` が真かつ未付与の場合にのみ
  しおづけ状態を仮想的に付与する。ターン終了時ダメージ自体は揮発性状態側のリーサルハンドラ（`しおづけ_damage`、
  `LethalEvent.ON_TURN_END`）が担当するため、技側のリーサルハンドラでダメージ計算は行わない。
