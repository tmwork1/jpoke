# 実装計画書: にほんばれ

## 仕様要約

- タイプ: ほのお / カテゴリ: 変化 / PP: 5 / 対象: 全体の場（target="field"）
- 5ターンの間、天候を「はれ」にする
- ほのおタイプ技のダメージが 1.5 倍、みずタイプ技が半減
- こおり状態にならなくなる
- おおひでり・おおあめ等の永続天候が存在する場合は失敗する（turn.md: 「天気変更技: おおひでり等で失敗」priority 100）

## 現状の実装状況

| 項目 | 状態 |
|------|------|
| `handlers/move_status.py` 関数 `にほんばれ_activate_weather` | 実装済み |
| `data/move.py` エントリ | 登録済み（handlers 登録なし） |

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 天候を「はれ」に変更 |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「天候・フィールド: すでに同じ」が priority 100 として記載されている。同種ハンドラ `あまごい_activate_weather`・`すなあらし_activate_weather` が priority デフォルト（100）を使用しているため、同一パターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

関数はすでに実装済みのため、追加不要。

```python
# すでに存在する（line 355-356 付近）:
def にほんばれ_activate_weather(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    return HandlerReturn(value=battle.weather_manager.apply("はれ", 5, source=ctx.attacker))
```

### `src/jpoke/data/move.py`

既存の `"にほんばれ"` エントリに handlers を追加する。

```python
"にほんばれ": MoveData(
    type="ほのお",
    category="変化",
    pp=5,
    target="field",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.にほんばれ_activate_weather,
        ),
    }
),
```

## 注意点・エッジケース

- ハンドラ関数はすでに存在するため、`data/move.py` への登録のみが作業となる
- `weather_manager.apply` が既存天候と同じ場合・おおひでり等の永続天候が存在する場合に失敗を返すかは manager 側に委ねられる。ハンドラ側でのガードは不要
- `あまごい` と完全に同じ登録パターン（`target="field"`）
