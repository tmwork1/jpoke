# 実装計画書: グラスフィールド

## 仕様要約

- タイプ: くさ / カテゴリ: 変化 / PP: 10 / 対象: 全体の場（target="field"）
- 5ターンの間、場の状態を「グラスフィールド」にする
- 地面にいるポケモンが毎ターン最大HP の 1/16 ずつ回復する
- くさタイプの技の威力が 1.3 倍になる
- じしん・じならし・マグニチュードの受けるダメージが半減する
- グラスシードを持つポケモンの「ぼうぎょ」ランクが 1 段階上がる（地面外でも発動）

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「天候・フィールド: すでに同じ」が priority 100 として記載されている。同種ハンドラ `じゅうりょく_activate_global_field` が priority デフォルト（100）を使用しているため、同一パターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「グ」は「ぐ」として処理。現在のファイルで `こうそくいどう_modify_attacker_stats` の前、「き」系の後。「ぐら」は「くら」扱いで「こうそくいどう」より前に配置する。具体的には `きりばらい_modify_defender_stats` → `きんぞくおん_modify_defender_stats` の後、`こうそくいどう_modify_attacker_stats` の前。

```python
def グラスフィールド_activate_global_field(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """グラスフィールド: グローバルフィールドをグラスフィールドにする。"""
    return HandlerReturn(value=battle.global_manager.activate("グラスフィールド", 5))
```

### `src/jpoke/data/move.py`

既存の `"グラスフィールド"` エントリに handlers を追加する。

```python
"グラスフィールド": MoveData(
    type="くさ",
    category="変化",
    pp=10,
    target="field",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.グラスフィールド_activate_global_field,
        ),
    }
),
```

## 注意点・エッジケース

- `global_manager.activate` はすでに同じフィールドが存在する場合に False を返すことが想定される。ハンドラは戻り値をそのまま `HandlerReturn(value=...)` で返すだけでよい
- `じゅうりょく_activate_global_field` および `エレキフィールド_activate_global_field` と同一パターン
- エレキフィールド・サイコフィールドと同時に設定することはできない（global_manager が排他管理）
