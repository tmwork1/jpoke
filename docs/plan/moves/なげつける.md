# なげつける 実装計画書

## 仕様要約（docs/spec/moves/なげつける.md 参照）

* タイプ: あく / 分類: 物理 / 威力: 持たせているどうぐに依存 / 命中: 100 / PP: 10
* 対象: 相手1体 / 使用後どうぐを消費する
* 威力・追加効果はどうぐごとに異なる（詳細は仕様書の威力決定仕様・追加効果テーブルを参照）
* 失敗条件: どうぐを持っていない、fling_power=0の対象外どうぐ、ジュエル全種、
  ぶきよう/マジックルームでどうぐが無効化されている場合、
  はっきんだま+ギラティナ、ブーストエナジー+こだいかっせい/クォークチャージ、
  仮面（いしずえのめん・いどのめん・かまどのめん）+オーガポン

## 現状の実装状況

`src/jpoke/data/move.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。

```python
"なげつける": MoveData(
    type="あく",
    category="physical",
    pp=10,
    power=1,
    accuracy=100,
    handlers={
        Event.ON_TRY_MOVE_1: h.MoveHandler(
            ha.なげつける_check_item,
            subject_spec="attacker:self",
            priority=30,
        ),
        Event.ON_HIT: [
            h.MoveHandler(
                ha.なげつける_apply_item_effect,
                priority=90,
            ),
            h.MoveHandler(
                ha.なげつける_consume_item,
                priority=100,
            ),
        ],
        Event.ON_MOVE_END: h.MoveHandler(
            ha.なげつける_consume_item,
        ),
    },
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_TRY_MOVE_1` | 30 | attacker:self | どうぐの所持・種類を確認し、`ctx.move.power` にfling_powerを設定する。使用不可なら技を失敗させる |
| `ON_HIT` | 90 | attacker:self（`MoveHandler` 既定値） | どうぐに応じた追加効果を相手に適用する（どうぐはまだ消費されていない） |
| `ON_HIT` | 100 | attacker:self（既定値） | 使用者のどうぐを消費する |
| `ON_MOVE_END` | 100（既定値） | attacker:self（既定値） | 命中しなかった・まもる等で無効化された等、`ON_HIT` に到達しなかった場合の消費保険 |

## Priority 根拠

* `ON_TRY_MOVE_1` priority=30: docs/spec/turn.md の `ON_TRY_MOVE_1` テーブルには本技固有のエントリはないが、
  同テーブルの30番台（技固有の使用可否判定: がむしゃら・たくわえる 等）と同種の「アイテム所持による使用可否判定」
  であるため、既存の同種チェックに合わせてpriority=30とする。
* `ON_HIT` priority=100（アイテム消費）: docs/spec/turn.md の `ON_HIT` テーブルに
  「100 | なげつける使用者のアイテム消費」と明記されている値をそのまま採用する。
* `ON_HIT` priority=90（追加効果適用）: turn.md に追加効果自体の掲載はないが、
  アイテム消費（priority=100）より前に `ctx.attacker.item.base_name` を参照して
  対象アイテムを判定する必要があるため、消費より低い値（90）を明示して先に実行させる。
* `ON_HIT` に両方のハンドラを置く理由（いのちのたま反動バグの修正）:
  `いのちのたま` はEvent.ON_HIT priority=160で反動ダメージを発生させる（`handlers/item.py`）。
  なげつけるでいのちのたまを投げた場合、一次情報
  （`docs/spec/items/いのちのたま.md`: 「なげつけるでいのちのたまを投げた場合、
  与ダメージは増えているが反動は受けない。」）により反動が発生してはならない。
  アイテム消費をON_HIT priority=100（いのちのたまの反動160より前）で行うことで、
  消費後にいのちのたま側の反動ハンドラを無効化する。
  ただし `EventManager.emit` はハンドラ一覧をループ開始時にスナップショットするため、
  ループ中にアイテムを消費（登録解除）しても、同じ `emit` 呼び出し内で既にスナップショットされた
  `いのちのたま_recoil` のハンドラ本体は実行されてしまう（登録解除は次回以降のイベント発火にのみ反映される）。
  このため、`いのちのたま_recoil` 自身にも `ctx.attacker.has_item("いのちのたま")` の
  自己確認ガードを追加する必要がある（`handlers/item.py` 側の修正、下記参照）。
* `ON_MOVE_END`（消費の保険）: 命中しなかった場合、まもる等で無効化された場合、
  特性による無効化（テレパシー・ふしぎなまもり・ひらいしん等）の場合は `ON_HIT` に到達しないため、
  一次情報の「命中可否に関わらずどうぐを消費する」を満たすためON_MOVE_ENDでも消費を試みる。
  `なげつける_consume_item` は `battle.item_manager.remove_item` を呼ぶだけで、
  既にどうぐを持っていない場合は何もしない（冪等）ため、二重登録による副作用はない。

## りんぷん・おんみつマントとの相互作用

一次情報: 「対象の特性がりんぷんや、持ち物おんみつマントを持つとき、なげつけるによる追加効果は発動しない。
きのみによる回復効果も受けられない。」

`なげつける_apply_item_effect` の冒頭で `battle.resolve_secondary_chance(ctx, 1.0)` を呼び、
`ON_MODIFY_SECONDARY_CHANCE` を経由してりんぷん（`りんぷん_block_secondary_chance`）・
おんみつマント（`おんみつマント_negate_secondary`）による確率0化を反映する。
確率が1未満の場合は追加効果（状態異常付与・ランク変化・きのみ回復等すべて）を発動せず終了する。

なげつけるは `has_flag("secondary_effect")` を持たないため、ちからずくによる
威力1.3倍・追加効果無効化（`ちからずく_boost` / `ちからずく_disable_secondary_effect`）は
本技には影響しない（一次情報「使用者が特性ちからずくであるときでも威力は上がらず、
各道具の追加効果は発動する」と一致）。

## ぶきよう・マジックルームとの相互作用

一次情報: 「使用者が特性ぶきようやさしおさえ状態であるときや、場がマジックルーム状態であるとき、
第四世代のみなげつけるを使用できる。第五世代以降では失敗し、持ち物は失われない。」

`なげつける_check_item` で `attacker.has_item(consider_enabled=True)` を使うことで、
ぶきよう・マジックルームによりアイテムが無効化されている場合は「どうぐを持っていない」と
同じ扱いで失敗させる（display_reasonは共通の「なげつける_アイテムなし」を流用する。
一次情報上も失敗メッセージの区別はない）。さしおさえは本プロジェクトでは技として未実装のため対象外。

## オーガポン・仮面との相互作用

一次情報（`docs/wiki/items/いどのめん.html` 等）: 「なげつける: オーガポンの場合は失敗する。
それ以外は威力60」。

`なげつける_check_item` に `attacker.item.base_name in ("いしずえのめん", "いどのめん", "かまどのめん")
and attacker.name.startswith("オーガポン")` の分岐を追加し、はっきんだま+ギラティナ・
ブーストエナジー+パラドックス特性と同様のパターンで失敗させる。
アルセウス+プレート各種・ゲノセクト+カセット各種にも同種の失敗条件が存在するが、
本プロジェクトではこれらのアイテム自体が未実装のため対象外とする。

## 実装箇所

### handlers/move_attack.py

* `なげつける_check_item`: `has_item()` → `has_item(consider_enabled=True)` に変更し、
  オーガポン+仮面の分岐を追加。
* `なげつける_apply_item_effect`: 冒頭に `resolve_secondary_chance` によるりんぷん/おんみつマント
  ガードを追加。
* `なげつける_consume_item`: 複数イベントから呼ばれる旨をdocstringに追記（実装自体は変更なし）。

### handlers/item.py

* `いのちのたま_recoil`: `ctx.attacker.has_item("いのちのたま")` の自己確認ガードを追加
  （なげつけるで投げた直後の同一ON_HIT内での反動誤発動を防ぐ）。

### data/move.py

* `なげつける` の `Event.ON_HIT` をリスト化し、`なげつける_apply_item_effect`（priority=90）と
  `なげつける_consume_item`（priority=100）を登録。`Event.ON_MOVE_END` の消費呼び出しは保険として維持。

## リーサル計算

docs/progress/move.md の該当行は「保留」。威力がどうぐ依存で不確定要素が大きく、
リーサル計算での一般化が難しいため対象外（現状維持）。
