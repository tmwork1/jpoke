# 実装計画書: すなあつめ

## 仕様

### 技データ

- タイプ: じめん
- カテゴリ: 変化（`category="status"`）
- PP: 8（championsルールにより8/12/16/20の4段階へ統一。`docs/champions/move_list.txt` に
  個別掲載はないが、同じPP5系統の自己回復技（あさのひざし/こうごうせい/じこさいせい/
  つきのひかり/はねやすめ）がいずれもPP8に統一されているため、それらと同一の扱いとする）
- 対象: 自分（`target="self"`。デフォルト値 `"foe"` のままだと `まもる`/`マジックコート` の
  判定が `move.target == "foe"` を前提にしているため誤って防がれる/跳ね返されてしまう）
- flags: `heal`

### 効果

使用者の HP を以下の割合で回復する（天候により変化）:

| 天候 | 回復量 |
|------|--------|
| すなあらし | 最大HP × 2732/4096（≒2/3） |
| それ以外（天候なし・はれ・おおひでり・あめ・おおあめ・ゆき・らんきりゅう等） | 最大HP × 1/2 |

HP が満タンの場合は失敗する。

天候の判定は `battle.weather_for(mon)`（使用者視点の有効天候）を使う。これにより
場に **ノーてんき/エアロック** 持ちがいる場合、天候効果自体を無効化（常に1/2扱い）が
自動的に考慮される。

### 参照仕様書

- `docs/spec/moves/すなあつめ.md`
- `docs/spec/moves/あさのひざし.md`（自己回復技の一般的な実装パターン）

## subject_spec と priority

### イベント: `ON_STATUS_HIT`

- **ctx型**: `AttackContext`（変化技の `ON_STATUS_HIT` は攻撃フローとして発火するため
  `ctx.attacker` / `ctx.defender` を参照する。`EventContext` の `source`/`target` ではない）
- **subject_spec**: `"attacker:self"`（`MoveHandler` のデフォルト値を使用）
- **priority**: 100

**根拠**: `docs/spec/turn.md` の `ON_APPLY_MOVE` テーブルに「自己回復技: HP満タン」が
priority 100 として記載されている。`ON_STATUS_HIT` は `turn.md` に独立エントリが存在しないが、
同種の変化技自己回復ハンドラ `あさのひざし_heal_self`/`じこさいせい_heal_self` も
`h.MoveHandler(hs.<handler>)` で priority=100 のデフォルトを使用しているため、これと同一
パターンとして priority=100 を採用する。

## ハンドラ構成

### 1. `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: `すてゼリフ_modify_defender_stats_and_pivot` の後、
`すなあらし_activate_weather` の前。

```python
def すなあつめ_heal_self(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """すなあつめ: 自分のHPを最大HPの1/2回復する。すなあらし状態のときは2732/4096(≒2/3)回復する。

    天候の有効判定は battle.weather_for() に委譲する（エアロック・ノーてんき考慮済み）。
    端数は五捨五超入（round_half_down）で丸める。HPが満タンの場合は失敗する。
    """
    mon = ctx.attacker
    if mon.hp == mon.max_hp:
        return HandlerReturn(value=False, stop_event=True)
    weather = battle.weather_for(mon)
    if weather.name == "すなあらし":
        heal = round_half_down(mon.max_hp * 2732 / 4096)
    else:
        heal = round_half_down(mon.max_hp / 2)
    battle.modify_hp(mon, v=heal)
    return HandlerReturn(value=value)
```

### 2. `src/jpoke/data/moves/move_sa.py`

`"すなあつめ"` エントリ:

```python
"すなあつめ": MoveData(
    type="じめん",
    category="status",
    pp=8,
    target="self",
    flags={"heal"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.すなあつめ_heal_self,
        )
    }
),
```

## 注意事項

- すなあらし時の回復割合 `2732/4096` は `1/2`・`2/3` のような単純な分数ではないため、
  `battle.modify_hp(mon, r=r)`（内部で `int(max_hp * r)` による単純な切り捨てを行う）は
  使わず、`round_half_down(mon.max_hp * 2732 / 4096)` を明示的に計算して
  `battle.modify_hp(mon, v=heal)` で渡す（`いやしのはどう_heal_defender` 等の既存パターンに
  合わせる）。通常時の `1/2` 分岐についても同じ関数で統一し `round_half_down` を使う
  （`1/2` 分岐は端数が必ず `.0` か `.5` になるため、結果は `int(max_hp * 0.5)` と一致する）。
- HP満タン判定は `mon.hp == mon.max_hp` で行い、失敗時は `HandlerReturn(value=False, stop_event=True)` を返す
- `target="self"` を明示しないと `MoveData.target` のデフォルト値 `"foe"` になり、
  `is_blocked_by_protect`（`まもる`）・`マジックコート_reflect` の判定が `move.target == "foe"` を
  条件にしているため、誤って「まもるで防がれる」「マジックコートで跳ね返される」動作になる
- `heal` フラグにより `かいふくふうじ_try_action`（技選択の禁止）・ヒーリングシフト
  （優先度+3）は汎用ハンドラ側で自動的に処理されるため、本技専用の実装は不要
