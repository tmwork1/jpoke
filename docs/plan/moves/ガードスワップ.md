# 計画書: ガードスワップ

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 10 / 命中: 100 / 対象: 相手
- 使用者と相手の「ぼうぎょ（B）」「とくぼう（D）」の**ランク変化**を入れ替える
- 実数値は変化しない（ランク変化のみ入れ替え）
- 入れ替え後のランクは `mon.rank["B"]` および `mon.rank["D"]` に反映される
- まもる: × / マジックコート: × / 命中判定: あり（accuracy=100）

## Priority 根拠

`docs/spec/turn.md` には `ON_STATUS_HIT` の内部priority表は掲載されていない。既存の変化技ハンドラ（`りゅうのまい_modify_attacker_stats`・`おいかぜ_set_side_field`・`いたみわけ_equalize_hp` 等）はすべて priority 100（`MoveHandler` デフォルト）を使用している。ガードスワップも同一パターンで priority 100 を採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: 五十音順「が」行 — `ガードシェア_equalize_stats` の直後に挿入する（ガードシェア → ガードスワップの順）。

ランクの入れ替えは `mon.rank` dict を直接操作する。`battle.modify_stats()` はランク**変化量**を加算するAPIのため、「現在値を入れ替える」目的には使用せず、直接代入を行う。

```python
def ガードスワップ_swap_ranks(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """ガードスワップ: 使用者と相手のぼうぎょ・とくぼうのランク変化を入れ替える。

    実数値は変化せず、ランク変化のみを互いに入れ替える。
    """
    attacker = ctx.attacker
    defender = ctx.defender
    for stat in ("B", "D"):
        attacker.rank[stat], defender.rank[stat] = (
            defender.rank[stat], attacker.rank[stat]
        )
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

既存の `"ガードスワップ"` エントリに `handlers` を追加する。

```python
"ガードスワップ": MoveData(
    type="エスパー",
    category="変化",
    pp=10,
    accuracy=100,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ガードスワップ_swap_ranks,
        ),
    }
),
```

## 注意点・エッジケース

- **ランク直接代入の是非**: `battle.modify_stats()` は `StatusManager.modify_stats()` へ委譲し `ON_BEFORE_MODIFY_STAT` / `ON_MODIFY_STAT` イベントを発火する。「入れ替え」はデルタ加算でないため、API経由では実装困難（変化後にもON_MODIFY_STATが呼ばれ、クリアボディ等で弾かれる可能性がある）。直接 `mon.rank[stat]` 代入を採用し、ランクガード系特性の干渉を受けない設計とする
- **ON_MODIFY_STAT イベント非発火**: 直接代入のため、かちき・まけんき・ムラっけ等のON_MODIFY_STAT反応特性は発動しない。これはゲーム仕様上も正しい挙動（ガードスワップはランク変化技ではなく入れ替え技）
- **0との入れ替え**: 両者のランクが0の場合は変化なし（0と0を入れ替えるだけ）。ON_TRY_MOVE_2 の「ランク補正上げ: すでに最大」「ランク補正下げ: すでに最低」ガードはガードスワップには適用されない（turn.md には「ひっくりかえす: ランク変化なし」の失敗条件があるが、ガードスワップは対象外）
- **パワースワップとの類似**: パワースワップ（A・Cランク入れ替え）は同じパターンで実装できる。対象ステータスが `("A", "C")` になる点のみ異なる
- **ワンダールーム中**: B/D参照の入れ替え（ON_CALC_DEF_MODIFIER）はダメージ計算時に行われる。ガードスワップはランク変数自体を入れ替えるため、ワンダールーム中でも `rank["B"]` と `rank["D"]` を正しく入れ替えれば問題ない
- **入れ替え後のランク制限確認**: 入れ替え後のランク値がすでに ±6 の範囲内に収まっているはずだが（入れ替えなので既存値の範囲を超えない）、念のため実装後テストで確認する
- **みがわり貫通**: 相手のみがわり状態に対してガードスワップがランク入れ替えを成功させるかはwiki不在のため要確認。みがわりがランク変化を防ぐ通常の仕様（ON_TRY_MOVE_2 priority 130 「みがわりによるランク補正下げ無効」）は「下げ」のみに適用されるため、「入れ替え」は貫通する可能性が高い
