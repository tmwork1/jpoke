# 計画書: ガードスワップ

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 12（`src/jpoke/data/ps-champ-ja/moves.json` 準拠） / 命中: - （必中） / 対象: 相手
- 使用者と相手の「ぼうぎょ（B）」「とくぼう（D）」の**ランク変化**を入れ替える
- 実数値は変化しない（ランク変化のみ入れ替え）
- 入れ替え後のランクは `mon.rank["def"]` および `mon.rank["spd"]` に反映される
- まもる: 防がれる / マジックコート: 跳ね返されない（`flags={"unreflectable"}` が必要） /
  みがわり: 貫通する（`flags={"bypass_substitute"}` が必要） / 命中判定: なし（`accuracy=None`）

## Priority 根拠

`docs/spec/turn.md` には `ON_STATUS_HIT` の内部priority表は掲載されていない。既存の変化技ハンドラ（`りゅうのまい_modify_attacker_stats`・`おいかぜ_set_side_field`・`いたみわけ_equalize_hp` 等）はすべて priority 100（`MoveHandler` デフォルト）を使用している。ガードスワップも同一パターンで priority 100 を採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: 五十音順「が」行 — `ガードシェア_equalize_stats` の直後に挿入する（ガードシェア → ガードスワップの順）。

ランクの入れ替えは `mon.rank` dict を直接操作する。`battle.modify_stats()` はランク**変化量**を加算するAPIのため、「現在値を入れ替える」目的には使用せず、直接代入を行う。
`Stat` の実キーは `"def"`（ぼうぎょ）・`"spd"`（とくぼう）である（`src/jpoke/types/literals.py` 参照）。

```python
def ガードスワップ_swap_ranks(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ガードスワップ: 使用者と相手のぼうぎょ・とくぼうのランク変化を入れ替える。

    実数値は変化せず、ランク変化のみを互いに入れ替える。
    """
    atk_rank = ctx.attacker.rank
    def_rank = ctx.defender.rank
    for stat in ("def", "spd"):
        atk_rank[stat], def_rank[stat] = def_rank[stat], atk_rank[stat]
    return HandlerReturn(value=value)
```

### `src/jpoke/data/moves/move_ka.py`

既存の `"ガードスワップ"` エントリに `handlers` を追加する。PP は `src/jpoke/data/ps-champ-ja/moves.json` 準拠で 12。
必中技のため `accuracy=None`。マジックコートで跳ね返されず、みがわりを貫通するため
`flags={"unreflectable", "bypass_substitute"}` を設定する。

```python
"ガードスワップ": MoveData(
    type="エスパー",
    category="status",
    pp=12,
    accuracy=None,  # 必中
    # マジックコートで跳ね返されず、みがわりを貫通する
    flags={"unreflectable", "bypass_substitute"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ガードスワップ_swap_ranks,
        ),
    }
),
```

## 注意点・エッジケース

- **ランク直接代入の是非**: `battle.modify_stats()` は `StatusManager.modify_stats()` へ委譲し `ON_BEFORE_MODIFY_STAT` / `ON_MODIFY_STAT` イベントを発火する。「入れ替え」はデルタ加算でないため、API経由では実装困難（変化後にもON_MODIFY_STATが呼ばれ、クリアボディ等で弾かれる可能性がある）。直接 `mon.rank[stat]` 代入を採用し、ランクガード系特性の干渉を受けない設計とする
- **ON_MODIFY_STAT イベント非発火**: 直接代入のため、かちき・まけんき・ムラっけ等のON_MODIFY_STAT反応特性は発動しない。これはゲーム仕様上も正しい挙動（ガードスワップはランク変化技ではなく入れ替え技）
- **0との入れ替え**: 両者のランクが0の場合は変化なし（0と0を入れ替えるだけ）。ON_TRY_MOVE_2 の「ランク補正上げ: すでに最大」「ランク補正下げ: すでに最低」ガードはガードスワップには適用されない（turn.md には「ひっくりかえす: ランク変化なし」の失敗条件があるが、ガードスワップは対象外）
- **パワースワップ・ハートスワップとの類似**: パワースワップ（`atk`・`spa` ランク入れ替え）、ハートスワップ（全ランク入れ替え）は同一パターンで実装済み。ガードスワップはそのうち `def`・`spd` のみを対象とする
- **ワンダールーム中**: B/D参照の入れ替え（ON_CALC_DEF_MODIFIER）はダメージ計算時に行われる。ガードスワップはランク変数自体を入れ替えるため、ワンダールーム中でも `rank["def"]` と `rank["spd"]` を正しく入れ替えれば問題ない
- **入れ替え後のランク制限確認**: 入れ替え後のランク値がすでに ±6 の範囲内に収まっているはずだが（入れ替えなので既存値の範囲を超えない）、念のため実装後テストで確認する（テスト済み）
- **みがわり貫通**: 一次情報（`docs/wiki/moves/ガードスワップ.html` 技の仕様節）に「相手のみがわり状態を貫通する」と明記されている。`MoveData.flags` に `"bypass_substitute"` を設定することで実現する
- **マジックコート非対応**: 一次情報の判定節に「マジックコート: ×」と明記されている。`Move.is_reflectable` は `category == "status" and target in ("foe", "foe_side")` の技をデフォルトで跳ね返り対象とするため、`flags={"unreflectable"}` を明示的に設定しないと誤って跳ね返されてしまう
- **必中技**: 一次情報の `wgCategories` に「必中わざ」が含まれ、命中率欄も「-」表記。`accuracy=100` ではランク補正（回避率上昇等）の影響を受けてしまうため、`accuracy=None` を設定する必要がある
- **PP**: `src/jpoke/data/ps-champ-ja/moves.json` の値（12）を正とする。Gen9本家基準の10は移行漏れのバグだったため修正する
