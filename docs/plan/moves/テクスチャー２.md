# 実装計画書: テクスチャー２

## 仕様要約（`docs/spec/moves/テクスチャー２.md` 第五世代以降準拠。本プロジェクトはチャンピオンズ＝SV基準のため）

- タイプ: ノーマル / カテゴリ: 変化 / PP: 30 / 命中: なし（必中） / 対象: 1体選択（相手）
- 自分のタイプを、相手が直前に使った技を半減か無効にするタイプに変更する
  - 候補が複数ある場合はランダムに1つを選ぶ
  - タイプを2つ持つポケモンが使っても、変更後のタイプは1つだけになる
  - 現在自分が持っているタイプには変更されない（候補から除外する）
- 失敗条件
  - 自分がテラスタル中
  - 相手が場に出てからまだ技を使用していない
  - 相手の技タイプに抵抗できるタイプのうち、自分が現在持っていないものが存在しない
    （わるあがき・ステラタイプの技など、抵抗できるタイプが存在しない技を含む）
- まもる・みがわり状態を無視する（貫通する）。ただしマジックコート・よこどり・オウムがえしには
  対応しない（本プロジェクトはこれらの技を未実装のため対象外）
- もりののろい・ハロウィンで追加されているタイプも自分のタイプとして考慮する。変更成功時、
  これらの追加タイプ効果はリセットされる（ミラータイプと同仕様）
- フリーズドライ・フライングプレス・サウザンアローに対して使用した場合、技の複合効果
  （みず4倍弱点付与・ひこう複合弱点付与等）を考慮せず、それぞれ みず・むし・ひこう タイプとして
  扱う（原作のバグ仕様）
- 交代するか、ひんしになるか、戦闘が終了すると自分のタイプは元に戻る

## 「相手が直前に使った技」の追跡方法

既存の `Pokemon.executed_move` は以下の理由からそのまま使えない:

- `Move.type` は `move_executor.py` の `use_move()` 内で `resolve_move_type()`（ON_MODIFY_MOVE_TYPE、
  Normalize等の特性やプラズマシャワー等の効果を反映）により一時的に上書きされるが、
  同じ `use_move()` の `finally` 節で `ctx.move.reset()` が呼ばれ、次の技が終わるまでに
  実効タイプが失われる。ターンをまたいで「直前に使った技の実効タイプ」を参照する用途には使えない。
- `executed_move` はみちづれ等の連続使用判定ハンドラで意図的に `None` にリセットされることがあり
  （`handlers/move_status.py` の `みちづれ_連続使用失敗チェック` 等）、テクスチャー２の判定に
  流用すると誤動作する。

そのため `Pokemon` に汎用フィールドを新設する:

```python
# model/pokemon.py の __init__ に追加（executed_move の直後）
self.last_move_type: Type | None = None  # 直近で使用した技の実効タイプ（テクスチャー2用）
self.last_move_name: MoveName | None = None  # 直近で使用した技名（テクスチャー2用）
```

```python
# reset_on_switch_out に追加（executed_move のリセットの直後）
self.last_move_type = None
self.last_move_name = None
```

`core/move_executor.py` の `_execute_move()` 内、`ctx.attacker.executed_move = ctx.move` の直後
（「発動した技の確定」タイミング＝行動不能・PP0以外の理由で技が失敗しても実行されない先頭部分）で
記録する:

```python
ctx.attacker.executed_move = ctx.move
ctx.attacker.last_move_type = ctx.move.type
ctx.attacker.last_move_name = cast(MoveName, ctx.move.name)
```

このタイミングは `executed_move` と同一のため、タイプ相性で無効化された攻撃技は
`executed_move` 同様に「使用した技」として記録されない（本プロジェクトの既存仕様。
`アンコール` 等も同じ制約を共有しており、本タスクの対象外の既存挙動として踏襲する）。

## 変更候補タイプの決定ロジック

`jpoke.data.TYPE_MODIFIER` / `TYPES` を用いて、相手の実効タイプを半減か無効にする
（倍率 < 1.0 の）タイプのうち、自分が現在持っていない（`attacker.types` に含まれない）ものを
列挙する。"" （わるあがき等のタイプ無し）・"ステラ" は候補から除外する
（`TYPE_MODIFIER[""]` / `TYPE_MODIFIER["ステラ"]` は全タイプ 1.0 倍のため、除外しなくても
自然に候補0件になるが、意図を明確にするため明示的に除外する）。

```python
テクスチャー2_判定タイプ上書き: dict[MoveName, Type] = {
    "フリーズドライ": "みず",
    "フライングプレス": "むし",
    "サウザンアロー": "ひこう",
}

def テクスチャー2_取得_変更候補タイプ(attacker: Pokemon, defender: Pokemon) -> list[Type]:
    if defender.last_move_type is None:
        return []
    move_type = テクスチャー2_判定タイプ上書き.get(
        cast(MoveName, defender.last_move_name), defender.last_move_type
    )
    type_chart = TYPE_MODIFIER.get(move_type, {})
    current_types = set(attacker.types)
    return [
        t for t in TYPES
        if t not in ("", "ステラ")
        and type_chart.get(t, 1.0) < 1.0
        and t not in current_types
    ]
```

`attacker.types` は `もりののろい`・`ハロウィン` による `added_types` を含めた現在のタイプを
返すため、追加タイプも自動的に「現在持っているタイプ」として除外される。

## タイプ変更の適用方法

`ミラータイプ`（`docs/plan/moves/ミラータイプ.md`）で新設済みの `Pokemon.move_override_types`
（`list[Type] | None`）をそのまま利用する。ひんし・交代時のリセットは既存の
`reset_on_switch_out` の処理（`move_override_types = None`）で完結する。

```python
ctx.attacker.move_override_types = [chosen]
ctx.attacker.added_types = []  # もりののろい・ハロウィンの追加タイプ効果をリセット
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_BEFORE_APPLY_MOVE` | 100（デフォルト） | `attacker:self`（デフォルト） | 失敗条件（テラスタル中／相手未行動／変更先タイプなし）を判定する |
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 候補からランダムに1つ選び、自分のタイプを変更する |

## Priority 根拠

`docs/spec/turn.md` に `ON_BEFORE_APPLY_MOVE` / `ON_STATUS_HIT` の記載はない。
同種のタイプ変更技である `ミラータイプ`（`ON_STATUS_HIT` はデフォルト priority）や、
失敗条件のみを判定する `いえき_can_apply` 等の一般的な `can_apply` 系ハンドラ
（デフォルト priority）を参考に、両イベントともデフォルト priority（100）を採用する。
（`ミラータイプ_can_apply` の priority=130 は turn.md に明記された固有の理由によるものであり、
本技には適用しない。）

## flags

まもる・みがわりを無視し、マジックコートで跳ね返されない技のため:

```python
flags={"unprotectable", "unreflectable", "bypass_substitute"}
```

- `unprotectable`: まもる状態を無視する（`Move.is_blocked_by_protect` が参照）
- `bypass_substitute`: みがわり状態を無視する（`MoveExecutor` が参照）
- `unreflectable`: マジックコート・マジックミラーで跳ね返されない（`Move.is_reflectable` が参照）

よこどり・オウムがえしへの対応（表内「×」）は、本プロジェクトが両技を未実装のため対象外。
トリックガード・ダイウォールについても未実装（シングルバトル専用かつダイマックス非対応の
プロジェクト方針）のため対象外。

## テスト方針

`battle.random.choice` を固定化し、候補一覧から決定的に1つを選ばせて検証する
（`つぼをつく` 等の既存テストと同じ手法）。

- 相手の技タイプを半減できるタイプに変わること
- 現在持っているタイプは候補から除外されること
- 変更候補が存在しない場合は失敗すること
- 相手が場に出てからまだ技を使っていない場合は失敗すること
- 自分がテラスタル中の場合は失敗すること
- まもる状態の相手に対しても成立すること（まもるを無視する）
- 交代でタイプが元に戻ること
- フリーズドライに対して使用した場合、こおりタイプではなくみずタイプとして判定されること
