# 計画書: こうげきしれい

## 仕様要約

- むしタイプ / 物理 / 威力90 / 命中100 / PP15 / 対象: 1体選択 / 優先度0 / 接触なし
- 急所ランク+1（急所に当たりやすい）。追加効果なし。

## ハンドラ構成

追加効果を持たないため、専用ハンドラは不要。`handlers={}` のみで基本ダメージ計算に委ねる。
急所ランク上昇は `MoveData.critical_rank` で表現し、専用ハンドラは不要。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、priority の検討対象イベントは存在しない。

## 実装箇所

### data/move.py への更新

既存の `"こうげきしれい"` エントリは以下の通りで、仕様と一致している（変更不要）。

```python
    "こうげきしれい": MoveData(
        type="むし",
        category="physical",
        pp=15,
        power=90,
        accuracy=100,
        critical_rank=1,
        handlers={},  # 追加効果なし
    ),
```

- `flags` を指定していない（`contact` フラグなし）→ 直接攻撃ではないという仕様と一致。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `critical_rank=1` → 急所ランク+1という仕様と一致。

## 注意点・エッジケース

- 追加効果がないため、まもる・おうじゃのしるし・オウムがえし等の一般的な技判定は
  フレームワーク側の共通処理（`handlers/move_attack.py` の基本ダメージ計算経路）で
  自動的に処理され、本技固有のハンドラ実装は不要。
- 急所ランク+1の判定は `critical_rank` フィールドを参照する既存の共通ロジック
  （例: きりさく・つじぎり等の高確率急所技と同様）で処理される。
