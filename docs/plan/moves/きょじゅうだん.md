# 計画書: きょじゅうだん

## 仕様要約

- はがねタイプ / 物理 / 威力100 / 命中100 / PP5 / 対象: 1体選択 / 優先度0 / 接触
- 「相手がダイマックスしていればダメージ2倍」の効果はダイマックス非対応の本プロジェクトでは対象外
- 追加効果なし（通常攻撃のみ）
- ものまねでコピー不可・まねっこ失敗（`non_copycat` フラグで表現）
- ゆびをふるで選ばれない（ゆびをふる自体が本プロジェクト未実装のため今回は対応不要）

## ハンドラ構成

ダメージ計算に関わる追加効果を持たないため、専用ハンドラは不要。
`handlers={}` のみで基本ダメージ計算に委ねる。
コピー不可の挙動は `flags={"contact", "non_copycat"}` で表現する
（`non_copycat` は `handlers/move_status.py` の `まねっこ_can_use` が
`Move(...).has_flag("non_copycat")` を見て失敗判定するため、専用ハンドラの追加は不要）。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、priority の検討対象イベントは存在しない。

## 実装箇所

### data/move.py への更新

既存の `"きょじゅうだん"` エントリは以下の通り。

```python
    "きょじゅうだん": MoveData(
        type="はがね",
        category="physical",
        pp=5,
        power=100,
        accuracy=100,
        flags={"contact"},
        handlers={},  # 追加効果なし
    ),
```

- type/category/pp/power/accuracy は仕様と一致（変更不要）。
- `flags` に `non_copycat` を追加する（まねっこでコピーできないという一次情報の記述を反映するため）。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `critical_rank` 省略時のデフォルトは `0`（通常の急所判定）→ 仕様と一致。

## 注意点・エッジケース

- 追加効果がないため、まもる・おうじゃのしるし等の一般的な技判定はフレームワーク側の共通処理
  （`handlers/move_attack.py` の基本ダメージ計算経路）で自動的に処理され、本技固有のハンドラ実装は不要。
- ダイマックス関連の効果（相手ダイマックス時にダメージ2倍）は本プロジェクトが非対応のため実装しない。
- くちたたて所持時のザマゼンタでアイアンヘッドがきょじゅうだんに変化する仕様は、
  本プロジェクトが覚える技の妥当性検証を行わないため対象外。
