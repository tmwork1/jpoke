# 計画書: かげぬい

## 仕様要約（docs/spec/moves/かげぬい.md 参照）

- ゴーストタイプ / 物理 / 威力90（Champions基準。本家SVまでは80） / 命中100 / PP12（Champions基準。本家は10） /
  対象: 1体選択 / 優先度0 / 非接触
- 追加効果として、命中時に相手を『にげられない』状態にする（100%）。
- ゴーストタイプの相手は『にげられない』状態の効果を無視できる
  （`にげられない` 揮発性状態自体の ON_CHECK_TRAPPED ハンドラで判定済み。本技側の実装で個別対応する必要はない）。
- 追加効果に分類されるため、ちからずく（威力1.3倍・効果無効化）・りんぷん/おんみつマント
  （効果無効化）の対象技になる。

## 既存実装との差分

既存実装（`data/move.py` の `"かげぬい"` エントリ）は以下の点で仕様と乖離していた。

1. **威力・PPがChampions基準になっていない**
   既存: `pp=10, power=80`（本家SVまでの値）。
   → `docs/champions/move_list.txt` / `docs/progress/move.md` に準拠し `pp=12, power=90` に修正する。

2. **追加効果（にげられない状態の付与）が未実装**
   既存: `handlers={}`（コメントで「交代できない状態を付与する処理を実装」と記載されているのみで未実装）。
   → 追加効果ハンドラを実装し、`Event.ON_DAMAGE_HIT` に登録する。

3. **`secondary_effect` フラグが付与されていない**
   → ちからずくの対象技として扱うため `flags={"secondary_effect"}` を付与する。

## ハンドラ構成

### handlers/move_attack.py: `かげぬい_apply_no_escape`（新規）

```python
def かげぬい_apply_no_escape(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """かげぬいの追加効果: 相手をにげられない状態にする。

    ゴーストタイプの相手はにげられない状態の効果を無視できる
    （`にげられない`揮発性状態側のON_CHECK_TRAPPEDで判定するため、本ハンドラでの個別対応は不要）。
    追加効果のため、`apply_volatile_to_defender` 経由でちからずく・りんぷんの影響を受ける。
    """
    return apply_volatile_to_defender(battle, ctx, value, volatile="にげられない")
```

- `apply_volatile_to_defender`（`handlers/move.py`）は内部で `battle.resolve_secondary_chance` を
  呼び出すため、`Event.ON_MODIFY_SECONDARY_CHANCE` 経由でちからずく（確率0化）・
  りんぷん/おんみつマント（確率0化）の無効化ハンドラが自動的に適用される。
  くろいまなざし（`くろいまなざし_apply`）・とおせんぼう（`とおせんぼう_apply`）と同じパターン。
- イベント: `Event.ON_DAMAGE_HIT`（`はたきおとす` / `うたかたのアリア` 等の既存実装と同様に、
  ダメージが実際に適用された後に発動させる。docs/spec/turn.md の `## Event.ON_DAMAGE` 参照）。
  ダメージが発生しなかった場合（無効化・まもる等）はイベント自体が発火しないため、
  追加効果も発生しない仕様を自然に満たす。
- 優先度: デフォルト100（`MoveHandler` のデフォルト値。turn.md未掲載の技固有効果のため
  既存の同種ハンドラ（うたかたのアリア等）に倣う）。
- subject_spec: `MoveHandler` デフォルト（`attacker:self`）のまま。

### data/move.py

```python
"かげぬい": MoveData(
    type="ゴースト",
    category="physical",
    pp=12,
    power=90,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.かげぬい_apply_no_escape,
        )
    },
),
```

### data/volatile.py

既存の `"にげられない"` 揮発性状態エントリ（ゴーストタイプ判定込みでON_CHECK_TRAPPEDを処理）は
仕様と一致しているため変更不要。

## リーサル計算

対象外（`docs/progress/move.md` の「リーサル実装」列が `n/a`）。
にげられない状態の付与はHP分布に影響しないため、リーサル計算ハンドラは不要。

## 注意点・エッジケース

- ノーマルタイプなどに技自体が無効化された場合、`Event.ON_DAMAGE_HIT` 自体が発火しないため
  追加効果も発生しない。個別のガードは不要。
- まもるで防がれた場合も同様にダメージが発生せず `Event.ON_DAMAGE_HIT` が発火しないため、
  追加効果も発生しない。
- ゴーストタイプへの付与判定は `にげられない` 揮発性状態の `ON_CHECK_TRAPPED` ハンドラ
  （`check_trapped_not_ghost`）に委ねるため、本技のハンドラ内でゴーストタイプ判定を
  重複して行わない。
