# 計画書: ほおばる

## 仕様要約

- タイプ: ノーマル / カテゴリ: 変化 / PP: 10 / 命中: なし（必中） / 対象: 自分
- 自分が持っているきのみを即座に使って効果を発動させ、さらに自分のぼうぎょランクを2段階上げる
- きのみを持っていない場合は技が失敗する（priority 30 の失敗条件）
- ぼうぎょランクがすでに+6の場合も技が失敗する（priority 130 の失敗条件）
- きのみの効果は残りHP量に関係なく発動する（HP閾値チェックを無視）
- ゲップを「その戦闘できのみを食べたことがある」状態にする

### あまのじゃく持ちのポケモンとの相互作用（仕様書より）

- ぼうぎょランクが+6のときも技は成功してきのみを消費する（あまのじゃくのとき）
- ぼうぎょランクが-6のときに使うと失敗してきのみを消費しない（あまのじゃくのとき）

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: 五十音順「ほ」行 — `ほたるび_modify_attacker_stats` の後に挿入する。

```python
def ほおばる_check_has_berry(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ほおばるの失敗条件: きのみを持っていない場合に失敗させる。"""
    mon = ctx.attacker
    if not mon.item.is_berry():
        battle.add_event_log(mon, LogCode.MOVE_FAILED,
                             payload={"reason": "ほおばる_きのみなし"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def ほおばる_check_defense_max(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ほおばるの失敗条件: ぼうぎょランクがすでに+6の場合に失敗させる。

    あまのじゃく持ちのポケモンは B ランクが -6 のときに失敗する。
    このチェックは battle.modify_stats の内部（ON_TRY_MOVE_2 の後）で
    行われるわけではないため、ここで明示的にガードする。
    """
    mon = ctx.attacker
    if mon.rank["B"] >= 6:
        battle.add_event_log(mon, LogCode.MOVE_FAILED,
                             payload={"reason": "ほおばる_ぼうぎょ最大"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def ほおばる_consume_berry_and_boost(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ほおばるの効果: 自分のきのみを強制消費して効果を発動し、ぼうぎょを2段階上げる。

    きのみ効果はHP閾値を無視して即座に発動させる。
    現フレームワークではきのみ効果が ON_HP_CHANGED（HP 減少時）に
    紐付いているため、ON_HP_CHANGED を大きな value で直接発火して
    HP 閾値チェックを通過させる。
    消費後にぼうぎょを2段階上げる。
    """
    from jpoke.enums import Event
    from jpoke.core import EventContext

    mon = ctx.attacker
    # ON_HP_CHANGED を直接発火してきのみ効果を発動させる
    # value = mon.max_hp を渡すことで、任意の HP 閾値チェック（1/2, 1/4 以下）を通過させる
    hp_ctx = EventContext(target=mon, source=mon)
    battle.events.emit(Event.ON_HP_CHANGED, hp_ctx, mon.max_hp)
    # ぼうぎょを2段階上げる
    return modify_attacker_stats(battle, ctx, value, stats={"B": 2})
```

## 注意点・エッジケース

### きのみ効果の強制発動方法（最重要）

現在のフレームワークでは、きのみの効果（例: オボンのみのHP回復、アッキのみのぼうぎょ+1）は `ON_HP_CHANGED` イベントが発火した際にアイテムハンドラが自動トリガーされる仕組みになっている。`battle.consume_item(mon)` はアイテムを消費するだけで効果は発動しない。

ほおばるの「HP閾値を無視して発動」を実現するには、`ON_HP_CHANGED` を `value = mon.max_hp` で直接発火する。これにより `_heal_berry` 内の `mon.hp * denominator <= mon.max_hp` チェックが常に `True` になる。

ただし、この方法には以下の制限・注意点がある：
- `ON_HP_CHANGED` に登録されていないきのみ（例: 状態異常治療きのみは `ON_BEFORE_APPLY_AILMENT` 等で発動）には別途対応が必要な場合がある
- `battle.events.emit(Event.ON_HP_CHANGED, hp_ctx, mon.max_hp)` の後にきのみが自動消費されるかどうかは、各きのみハンドラの `_announce_and_consume_item` 呼び出しに依存する
- impl エージェントは発火後に `mon.item.is_berry()` が False（消費済み）になっているか確認し、なっていない場合は `battle.consume_item(mon)` を別途呼ぶ

`おちゃかい` の計画書（`docs/plan/moves/おちゃかい.md`）でも同じ課題が記録されており、impl エージェントは両計画書を参照して一貫した実装方針を採用すること。

### ゲップフラグ

仕様書によれば「ほおばるの効果で自分のきのみを食べたとき、ゲップが使用可能になる」。現状ゲップ機能は未実装（`data/move.py` のエントリに handlers なし）のため、ゲップフラグの設定は今回スコープ外とする。ゲップ実装時に `ほおばる_consume_berry_and_boost` を拡張して `mon.ate_berry = True`（または相当するフラグ設定）を追加すること。

### ON_TRY_MOVE_2 でのぼうぎょ最大チェックの必要性

turn.md に「ほおばる: ぼうぎょ最大」が priority 130 で `ON_TRY_MOVE_2` に明記されている。ただし `modify_attacker_stats` 内の `battle.modify_stats` が B ランクを上げようとする際に `ON_BEFORE_MODIFY_STAT` で `stop_event` になる可能性はあるが、それは技の失敗ではなく「ランク変化が起きないが技は成功」扱いになる。そのため、技全体を失敗させるには `ON_TRY_MOVE_2` での明示的なチェックが必要。

### あまのじゃく持ちとの相互作用

`docs/spec/abilities/あまのじゃく.md` によれば：
- あまのじゃく持ちは「ぼうぎょ最大（+6）のときも成功してきのみを消費する」 → `ON_TRY_MOVE_2` の失敗チェックはあまのじゃく側で上書きされるか、あるいはほおばる側のチェックをスキップする必要がある
- あまのじゃく持ちは「ぼうぎょ最低（-6）のときに失敗してきのみを消費しない」 → -6 のときに失敗させる

実装上の注意: `battle.modify_stats` にはあまのじゃく補正が内部で入るため、あまのじゃくに関するロジックはほおばる側では実装せず `modify_attacker_stats` 経由で自然に処理される。ただし `ON_TRY_MOVE_2` の失敗チェックはあまのじゃくの影響を受けないため、impl エージェントはあまのじゃくとほおばるの相互作用テストを追加して正しく動作することを確認すること。

### ON_STATUS_HIT での「きのみを既に消費している」ケース

`ON_TRY_MOVE_1` でのきのみ保持チェックは「技発動時点」のものだが、`ON_STATUS_HIT` までの間にきのみが奪われるケース（実践上は発生しにくいが）は想定外。ハンドラ側でガード不要（イベント発火側で前提が保証される）。

### ほおぶくろ・はんすう・じゅくせい特性との相互作用

仕様書によれば、ほおばるできのみを食べたときもほおぶくろ（HP回復）・はんすう（もう一度効果発動）・じゅくせい（効果2倍）が発動する。これらは各特性のハンドラが `ON_HP_CHANGED` 等の正規イベントを通じて発動するため、ほおばるが `ON_HP_CHANGED` を正しく発火していれば自動的に連動する見込み。impl エージェントは特性との組み合わせテストを追加すること。

### しゅうかく・ものひろいによるきのみ復活後のほおばる再使用

仕様書「ほおばる/おちゃかいで食べられたきのみも対象になる（しゅうかく・ものひろい）」は各特性ハンドラが管理するため、ほおばる側での追加実装は不要。

### ランク上昇ピンチきのみがぼうぎょ最大時に発動した場合（リュガのみ）

`docs/spec/items/_ランク上昇ピンチきのみ.md` によれば「ぼうぎょランクが4〜5段階のときにほおばるを使って消費した場合、リュガのみの効果は発動せずにほおぶくろも発動しないが、リュガのみは失われる」とある。この挙動は `modify_stats` の内部でランク上限チェックが行われるため、ほおばる側での追加実装は不要。
