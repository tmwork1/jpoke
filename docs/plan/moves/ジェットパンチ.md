# ジェットパンチ 実装計画書

## 仕様要約（docs/spec/moves/ジェットパンチ.md 参照）

* タイプ: みず / 分類: 物理 / 威力: 60 / 命中: 100 / PP: 16
* 対象: 相手1体 / 優先度: +1 / 接触あり / フラグ: パンチ技
* 効果: 先制攻撃技。追加効果なし。パンチ技のため特性「てつのこぶし」で威力1.2倍。

## 現状の実装状況

`src/jpoke/data/moves/move_sa.py` に登録済みだが、PP が誤っている（バグ）。

```python
"ジェットパンチ": MoveData(
    type="みず",
    category="physical",
    pp=15,  # 誤り: Gen9本家の値が移行漏れで残っている
    power=60,
    accuracy=100,
    priority=1,
    flags={"contact", "punch"},
    handlers={},  # 追加効果なし
),
```

`src/jpoke/data/ps-champ-ja/moves.json:360` により、本プロジェクトの全技PPは8/12/16/20の4段階に
統一されている。ジェットパンチは `pp=16` が正しい値であり、`pp=15` はGen9本家からの
移行漏れバグである。`pp=16` に修正する。

型・威力・命中・優先度・`flags={"contact", "punch"}` は仕様書と一致しており修正不要。

## ハンドラ構成

追加効果を持たないため専用ハンドラは不要。`priority=1` は `MoveData.priority` フィールドの
みで完結し、行動順決定（`turn_controller.py`）が参照する。`docs/spec/turn.md` には技の
`priority` フィールド自体の記載はなく、既存の同種先制技（でんこうせっか・マッハパンチ・
バレットパンチ 等、いずれも `priority=1`）と同様のパターンを採用している。
「てつのこぶし」によるパンチ技の威力補正は `flags={"punch"}` を通して既存の共通ハンドラ
（`src/jpoke/handlers/ability.py` の `_modify_by_move_condition(..., move_flag="punch")`）
が自動的に処理するため、本技固有の実装は不要。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、イベントハンドラの priority 検討対象は存在しない。
`MoveData.priority=1` は技自体の行動順優先度であり、`turn.md` のイベントハンドラ priority とは
別概念。

## 注意点・エッジケース（review-test 向け）

1. `flags={"contact", "punch"}` により直接攻撃技・パンチ技の両方として扱われる
   （せいでんき等の接触特性、パンチグローブ・てつのこぶし等のパンチ技補正の対象）。実装済み。
2. `target` を省略しているためデフォルトの `"foe"`（相手1体）が適用される → 仕様と一致。
3. `critical_rank` を省略しているためデフォルトの `0`（通常の急所判定）が適用される → 仕様と一致。
4. 追加効果がないため、まもる・おうじゃのしるし・オウムがえし等の一般的な技判定は
   フレームワーク側の共通処理（`handlers/move_attack.py` の基本ダメージ計算経路）で
   自動的に処理され、本技固有のハンドラ実装は不要。
5. リーサル計算は `docs/progress/move.md` 上 n/a のため対象外。
