# 計画書: せいちょう

## 仕様要約

- 自分の「こうげき」「とくこう」ランクがそれぞれ1段階ずつ上がる
- 天候が「にほんばれ」（はれ / おおひでり）の時は2段階ずつ上がる
- タイプ: ノーマル、変化、PP: 20、対象: 自分

出典: `docs/spec/moves/status_move.md` 行 121

---

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100 (省略) | `attacker:self` (省略) | 天候に応じてこうげき・とくこうを1段階または2段階上げる |

---

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`わるだくみ_modify_attacker_stats` 等）がすべて priority=100 (デフォルト) で登録されているため、100 を使用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: `スレッドトラップ_apply_volatile_to_attacker` の後（「せ」行は「す」の後、「タールショット」の前）

天候チェックは `あさのひざし_heal_self` の `weather.sunny` プロパティ参照パターンを踏襲する。
`weather.sunny` は「はれ」と「おおひでり」の両方をカバーしている。

```python
def せいちょう_modify_attacker_stats(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """せいちょう: 天候が晴れの時はこうげき・とくこうを2段階、通常時は1段階上げる。"""
    n = 2 if battle.weather.sunny else 1
    return modify_attacker_stats(battle, ctx, value, stats={"A": n, "C": n})
```

### `src/jpoke/data/move.py`

`"せいちょう"` エントリ（行 5256）に `handlers` を追加する:

```python
    "せいちょう": MoveData(
        type="ノーマル",
        category="変化",
        pp=20,
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.せいちょう_modify_attacker_stats,

```

## 注意点・エッジケース

- `battle.weather.sunny` プロパティが「はれ」と「おおひでり」の両方を `True` として返すことを確認済み（`あさのひざし_heal_self` で同じプロパティを使用）
- 天候チェックは `ON_STATUS_HIT` ハンドラ内で行う（技発動時点の天候を参照）
- ランクがすでに最大（+6）の場合の失敗は `turn.md` Priority 130 でマネージャー内部処理されるため、ハンドラ側にガード不要
- 晴れ時に A または C のどちらかがすでに +6 の場合、その上昇のみ失敗し、もう一方は成功する
- 「エアロック」「ノーてんき」特性による天候無効化はフレームワーク側で `weather.sunny` が `False` を返す形で対応される（ハンドラ内追加対応不要）
