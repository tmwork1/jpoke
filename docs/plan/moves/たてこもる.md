# 実装計画: たてこもる

## 仕様サマリ

- タイプ: はがね / 分類: 変化 / PP: 12（champions 基準。src/jpoke/data/ps-champ-ja/moves.json 参照）/ 命中率: なし（自分対象）
- 効果: 自分のぼうぎょランクを 2 段階上げる
- まもる: × / マジックコート: × / 直接攻撃: ×
- 対象: 自分（`MoveData.target="self"` を明示しないと `is_blocked_by_protect` / `is_reflectable` が
  誤って True になるため必須）

`data/move.py` にエントリは存在するが `handlers` キーが未設定（ランク上昇が未実装）。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自分のぼうぎょを 2 段階上げる |

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の同種ハンドラ（`てっぺき_modify_attacker_stats`（B+2）、`バリアー_modify_attacker_stats`（B+2 相当）、`ドわすれ_modify_attacker_stats`（D+2））が priority=100 デフォルトで実装されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: 五十音順で `タールショット_*` の後、`てっぺき_*` の前（た行内）

```python
def たてこもる_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """たてこもるの効果: 自分のぼうぎょを 2 段階上げる。"""
    return modify_attacker_stats(battle, ctx, value, stats={"def": 2})
```

### data/move.py の更新

`たてこもる` の MoveData に `handlers` を追加する。

```python
"たてこもる": MoveData(
    type="はがね",
    category="変化",
    pp=12,
    target="self",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.たてこもる_modify_attacker_stats,
        ),
    }
),
```

## 注意点・エッジケース

- `てっぺき`（はがね・B+2）と効果は同じ。タイプが異なるだけ
- ランク上限（+6）チェックはフレームワーク側で処理されるため、ハンドラ側のガードは不要
- 対象は自分（自分対象技）のため、命中率なし。テストは命中率固定不要
