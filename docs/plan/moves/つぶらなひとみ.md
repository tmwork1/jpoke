- タイプ: フェアリー / カテゴリ: 変化 / PP: 30 / 命中: 100 / 対象: 1 体選択
- 優先度: +1（必ず先制できる）
- 相手の「こうげき」ランクを 1 段階下げる

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_STATUS_HIT` には明示記載なし。同種ハンドラ `なきごえ_modify_defender_stats`（A-1）と同じパターンを採用し、priority はデフォルト（100）とする。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「つ」行 — `つきのひかり_heal_self` の後、`つるぎのまい_modify_attacker_stats` の前（「つぶ」は「つき」の後、「つる」の前）に挿入する。

```python
def つぶらなひとみ_modify_defender_stats(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """つぶらなひとみの効果: 相手のこうげきを 1 段階下げる。"""
    return modify_defender_stats(battle, ctx, value, stats={"A": -1})
```

### `src/jpoke/data/move.py`

既存の `"つぶらなひとみ"` エントリに handlers を追加する。priority=1 はすでに設定済みのため変更不要。

```python
"つぶらなひとみ": MoveData(
    type="フェアリー",
    category="変化",
    pp=30,
    accuracy=100,
    priority=1,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.つぶらなひとみ_modify_defender_stats,
        ),
    }
),
```

## 注意点・エッジケース

- 優先度 +1 はすでに `data/move.py` に設定済みのため、ハンドラ側では不要
- ランク下限チェック（-6 で失敗）は `modify_stats` → `modify_stat` 内部の `clamp_stats` が行うため、ハンドラ側では追加チェック不要
- しろいきり・クリアボディ等によるランク変化無効はイベント `ON_TRY_MOVE_2`・`ON_BEFORE_MODIFY_STAT` 側で処理されるため、ハンドラ側では不要
