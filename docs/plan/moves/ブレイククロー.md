# 実装計画書: ブレイククロー

参照仕様書: docs/spec/moves/ブレイククロー.md

## 実装方針

ブレイククローは「50%の確率で相手のぼうぎょランクを1段階下げる」という単純な追加効果を持つ
攻撃技であり、既存の共通ハンドラ `modify_defender_stats`（handlers/move.py）を利用して実装する。
専用の効果処理は不要。

* `type="ノーマル"`
* `category="physical"`
* `pp=12`（champions基準。docs/champions/move_list.txt 準拠）
* `power=75`
* `accuracy=95`
* `flags={"contact", "secondary_effect", "slash"}`
  * `"contact"`: 直接攻撃
  * `"secondary_effect"`: 追加効果を持つ技（ちからずく等の対象技であることを示す）
  * `"slash"`: Championsで「切る技」に追加された（docs/champions/changes_from_sv.md 参照）。
    特性きれあじの威力補正（`handlers/ability.py` の `きれあじ_modify_power`）が
    既存実装のまま適用される

## ハンドラ

`Event.ON_DAMAGE_HIT` で `handlers/move_attack.py` の `ブレイククロー_lower_defender_def` を呼び出す。

```python
def ブレイククロー_lower_defender_def(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"def": -1}, chance=0.5)
```

`modify_defender_stats` は内部で `battle.resolve_secondary_chance(ctx, chance)` を呼び出すため、
ちからずくによる追加効果無効化は自動的に処理される。

## 登録

`src/jpoke/data/moves/move_ha.py` の `MOVES["ブレイククロー"]` に上記の値を設定する。

## リーサル計算

対象外（n/a）。docs/progress/move.md のブレイククロー行「リーサル実装」「リーサルテスト」列が
n/a のため、専用ハンドラは実装しない。

## テスト方針

`tests/moves_attack/test_move_ha.py` に以下を追加する（シェルブレード・かみくだくと同様のパターン）。

* `secondary_chance=0.0` で追加効果不発時にぼうぎょランクが変化しないことを確認するテスト
* `secondary_chance=1.0` で追加効果発動時にぼうぎょランクが-1になることを確認するテスト
* 相手にダメージを与えることを確認するテスト
* `slash` フラグを持つため、特性きれあじ所持時に威力が1.5倍になることを確認するテスト
