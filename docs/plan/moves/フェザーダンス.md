# 実装計画書: フェザーダンス

## 仕様

### 技データ

- タイプ: ひこう
- カテゴリ: 変化
- PP: 16（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）
- 命中率: 100
- 対象: 相手1体

### 効果

相手の **こうげき** ランクを2段階下げる。

### 参照仕様書

- `docs/spec/moves/フェザーダンス.md`

## subject_spec と priority

### イベント: `ON_STATUS_HIT`

- **subject_spec**: `"attacker:self"`（`MoveHandler` のデフォルト値を使用）
- **priority**: 100

**根拠**: `docs/spec/turn.md` に `ON_STATUS_HIT` の独立エントリは無いが、同種の防御側ランクダウン
変化技ハンドラ（`あまえる_modify_defender_stats`, `あまいかおり_modify_defender_stats` など）が
いずれも `h.MoveHandler(...)` の priority デフォルト値（100）で登録されているため、これと同一
パターンとして priority=100 を採用する。

## ハンドラ構成

### 1. `src/jpoke/handlers/move_status.py`

共通ユーティリティ `handlers/move.py` の `modify_defender_stats` を利用する（既存実装済み）。

```python
def フェザーダンス_modify_defender_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"atk": -2})
```

五十音順の挿入位置: `フェアリーロック_activate_global_field` の後、`ふしょくガス_remove_item` の前
（`data/moves/move_ha.py` の定義順に合わせる）。

### 2. `src/jpoke/data/moves/move_ha.py`

```python
"フェザーダンス": MoveData(
    type="ひこう",
    category="status",
    pp=16,
    accuracy=100,
    flags={"dance"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.フェザーダンス_modify_defender_stats,
        )
    }
),
```

`flags={"dance"}`: Wiki掲載のカテゴリ「おどりのわざ」に該当するため、ミラータイプ等の
「おどりのわざ」判定に使われる既存フラグを付与する。

## 注意事項

- `modify_defender_stats` は `battle.modify_stats(ctx.defender, stats, source=ctx.attacker)` を
  呼び出す共通関数であり、まもる・マジックコート・特性による無効化などは既存の
  `battle.modify_stats` / まもる判定 / `ON_STATUS_HIT` 発火経路側で共通処理されるため、
  本ハンドラ側で個別対応は不要
- 確率発動ではない（`chance` 引数はデフォルト1のまま呼び出す）ため、`resolve_secondary_chance`
  は常に1を返し無条件でランクダウンする
- よこどり（Snatch）は未実装のためWiki記載の「×」判定に対応するコードは不要
