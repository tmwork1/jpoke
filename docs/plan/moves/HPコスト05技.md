# 計画書: HPコスト05技（クロロブラスト・てっていこうせん・ビックリヘッド）

## 仕様要約

- 使用前に自分の最大HPの1/2を消費する（小数切り捨て、最低1）
- HP消費後にHPが0になっても技は発動する（ON_PAY_HP はヒット処理より前に発火されるため）
- 消費は `Event.ON_PAY_HP` で処理される

| 技名 | タイプ | 分類 | PP | 威力 | 命中 | HPコスト |
|------|--------|------|-----|------|------|----------|
| クロロブラスト | くさ | 特殊 | 5 | 150 | 95 | max_hp // 2 |
| てっていこうせん | はがね | 特殊 | 5 | 140 | 95 | max_hp // 2 |
| ビックリヘッド | ノーマル | 物理 | 5 | 130 | 100 | max_hp // 2 |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_PAY_HP` テーブルは priority が `-`（未掲載）となっている。

```
## Event.ON_PAY_HP

| Priority | 処理 |
|----------|------|
| - | じばく等のダメージ・瀕死判定 |
| - | いのちがけ |
```

既存実装の `いのちがけ_pay_hp` は `Event.ON_PAY_HP` に priority を明示していない（デフォルト値を使用）。
HPコスト1/2技も同様に priority 未指定（デフォルト値）で実装する。

## 実装箇所

### handlers/move_attack.py に追加する関数

3技のHPコスト処理は同一ロジックなので1つのヘルパー関数で共用する。
五十音順の挿入位置: `クロロブラスト` → く行、`てっていこうせん` → て行、`ビックリヘッド` → び行。

```python
def クロロブラスト_pay_hp(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """クロロブラスト: 使用前に最大HPの1/2を消費する。"""
    cost = max(1, ctx.attacker.max_hp // 2)
    battle.modify_hp(ctx.attacker, v=-cost, reason="self_cost", source=ctx.attacker)
    return HandlerReturn(value=value)


def てっていこうせん_pay_hp(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """てっていこうせん: 使用前に最大HPの1/2を消費する。"""
    cost = max(1, ctx.attacker.max_hp // 2)
    battle.modify_hp(ctx.attacker, v=-cost, reason="self_cost", source=ctx.attacker)
    return HandlerReturn(value=value)


def ビックリヘッド_pay_hp(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ビックリヘッド: 使用前に最大HPの1/2を消費する。"""
    cost = max(1, ctx.attacker.max_hp // 2)
    battle.modify_hp(ctx.attacker, v=-cost, reason="self_cost", source=ctx.attacker)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース（review-test 向け）

- **HP0でも技が発動するか**: ON_PAY_HP はヒット処理より前に発火されるため、HP消費後に自分のHPが0になってもその後の攻撃処理（ダメージ付与）は継続する。テストで確認すること。
- **最低コスト1**: `max(1, max_hp // 2)` により、最大HPが1でもコストは1になる（0にならない）。
- **ビックリヘッドの contact ラベル**: 物理技なので `contact` ラベルを付与する（ゴツゴツメット・てつのトゲ等の接触効果を受ける）。
- **てっていこうせんの contact ラベル不要**: 特殊技のため contact なし（既存定義と一致）。
- **ハンドラ側ガード不要**: `ON_PAY_HP` 発火時に使用者が生存していることはバトル側で保証されている（行動可能チェック済み）。
- **3技は同一パターン**: ロジックが同一のため、将来的には共通ヘルパー関数への統合も検討できるが、現状は五十音順の個別関数として実装する。
- **ミストバースト（自爆技）との違い**: HPコスト05技は `labels=["explosion"]` を持たない（自爆フラグ不要）。
