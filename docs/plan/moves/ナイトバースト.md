# 計画書: ナイトバースト

## 仕様要約（docs/spec/moves/ナイトバースト.md 参照）

- あくタイプ / 特殊 / 威力90 / 命中95 / PP12 /
  対象: 相手1体 / 優先度0 / 接触なし
- 追加効果として、命中時に40%の確率で相手の命中率を1段階下げる。
- 追加効果に分類されるため、ちからずく（威力1.3倍・効果無効化）の対象技になる。

## 既存実装との差分

既存実装（`data/moves/move_na.py` の `"ナイトバースト"` エントリ）は本家シリーズ値
（`power=85, pp=10`）のまま Champions 移行が漏れていた。以下を修正する。

1. **基本情報を Champions 値に修正**
   `power=85` → `power=90`、`pp=10` → `pp=12` に修正する（`accuracy=95` は変更なし）。
   `docs/champions/move_list.txt` のナイトバースト行（威力90・命中95・PP12）を正とする。

2. **追加効果ハンドラ本体（`ナイトバースト_lower_acc`）は仕様通り実装済み**
   `modify_defender_stats(battle, ctx, value, stats={"accuracy": -1}, chance=0.4)`
   で、40%発動・命中率-1段階の仕様と一致している。変更不要。

3. **`flags` は既に `{"secondary_effect"}` が設定済み**
   弾のわざ（bullet）ではなく、追加効果を持つ技として正しく分類されている
   （`contact` フラグは付与されておらず、直接攻撃ではない仕様と一致）。変更不要。

## ハンドラ構成

### handlers/move_attack.py: `ナイトバースト_lower_acc`（既存・変更なし）

```python
def ナイトバースト_lower_acc(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"accuracy": -1}, chance=0.4)
```

- `modify_defender_stats`（`handlers/move.py`）は内部で `battle.resolve_secondary_chance` を
  呼び出すため、`Event.ON_MODIFY_SECONDARY_CHANCE` 経由でちからずく（確率0化・威力1.3倍）の
  ハンドラが自動的に適用される（`secondary_effect` フラグが前提）。
- イベント: `Event.ON_DAMAGE_HIT`（ダメージが実際に適用された後に発動。
  docs/spec/turn.md の `## Event.ON_DAMAGE` 参照）。
  ダメージが発生しなかった場合（無効化・まもる等）はイベント自体が発火しないため、
  追加効果も発生しない仕様を自然に満たす。
- 優先度: デフォルト100（`MoveHandler` のデフォルト値）。
- subject_spec: `MoveHandler` デフォルト（`attacker:self`）のまま。

### data/moves/move_na.py（`power` / `pp` を Champions 値に修正）

```python
"ナイトバースト": MoveData(
    type="あく",
    category="special",
    pp=12,
    power=90,
    accuracy=95,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ナイトバースト_lower_acc,
        )
    }
),
```

対象外（`docs/progress/move.md` の「リーサル実装」列が `n/a`。命中率ランク低下のみでHP分布に
影響しない追加効果のためリーサル計算ハンドラは不要）。

## 注意点・エッジケース

- あくタイプの技が無効化される場合（フェアリー複合等でタイプ相性0倍になる場合）、
  `Event.ON_DAMAGE_HIT` 自体が発火しないため追加効果も発生しない。個別のガードは不要。
- まもるで防がれた場合も同様にダメージが発生せず `Event.ON_DAMAGE_HIT` が発火しないため、
  追加効果も発生しない。
- おうじゃのしるし・するどいキバの10%ひるみ追加抽選（`handlers/item.py` の
  `flinch_on_hit_10pct`）は `_INNATE_FLINCH_MOVES` に「ナイトバースト」が含まれないため、
  本技の命中率低下効果とは独立して常に判定される（ナイトバースト側の実装への影響はない）。
