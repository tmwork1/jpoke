# 計画書: とおせんぼう

## 仕様要約

- ノーマルタイプ / 変化 / 必中（`accuracy=None`） / PP8（Champions値） / 対象: 1体選択
- 相手を『にげられない』状態にし、逃げたり交代できなくなる
- ゴーストタイプの相手は『にげられない』状態の効果を無視できる
  （`にげられない` 揮発性状態自体の仕様。ハンドラ内チェック不要）
- すでに『にげられない』状態の場合は失敗する
  （`battle.volatile_manager.apply()` の重複付与チェックにより自動的に処理される）
- まもるで防がれない（第六世代以降。`docs/champions/move_list.txt` の「守×」に対応）ため
  `unprotectable` フラグを付与する

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | 相手ににげられない状態付与 |

## Priority 根拠

`Event.ON_STATUS_HIT` は既存の同種ハンドラ（くろいまなざし等）と同じデフォルト優先度100を使用する。

ゴーストタイプへの無効化・重複付与の失敗は、いずれもハンドラ本体ではなくフレームワーク側の
既存機構が処理する:

- ゴーストタイプ無効化: `にげられない` 揮発性状態側の `Event.ON_CHECK_TRAPPED` ハンドラ
  `check_trapped_not_ghost`（`docs/spec/volatiles/にげられない.md` 参照。
  `turn.md` 未掲載イベントのため既存の同種ハンドラ（バインド・ねをはる等）と同じ
  priority=100（デフォルト）を使用）。
- 重複付与の失敗: `VolatileManager.apply()` 内の `if target.has_volatile(name): return False`
  （`src/jpoke/core/volatile_manager.py`）。

## 注意点・エッジケース

- くろいまなざしと全く同じ実装パターン（volatile 名も `にげられない` で共通）
- ゴーストタイプへの無効はフレームワーク側（`にげられない` 揮発性状態の `ON_CHECK_TRAPPED`）が
  処理するためハンドラ内チェック不要
- すでに逃げられない状態の失敗チェックも `VolatileManager.apply()` が処理するためハンドラ内
  チェック不要
- PP は本家シリーズでは5だが、Champions では8（`docs/champions/move_list.txt` 準拠）
- 命中判定は「100%」ではなく「必中」（`accuracy=None`）。ランク補正・回避率に関係なく必ず命中する
- まもるで防がれない（第六世代以降）ため `unprotectable` フラグが必要
  （デフォルトでは target=foe の変化技はまもるで防がれる仕様のため、明示的なフラグ付与が必須）
- みがわりには防がれる（`bypass_substitute` フラグを付けない。target=foe の変化技は
  デフォルトでみがわりにブロックされる一般ルールに従う）
- クモのすとも同じ volatile を使用する（自分が場を離れると消える処理は `にげられない`
  volatile 側の `ON_SWITCH_OUT` で管理）
