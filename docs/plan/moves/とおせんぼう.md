# 計画書: とおせんぼう

## 仕様要約

- ノーマルタイプ / 変化 / 命中なし（必中）/ PP5 / 対象: 1体選択
- ゴーストタイプではない相手は逃げたり交代できなくなる
- 自分が場を離れると効果が消える
- ゴーストタイプには無効（`ON_TRY_MOVE_2` priority 120: `ゴーストタイプ: にげられない無効` でフレームワーク側が処理）
- すでに逃げられない状態の場合は失敗（`ON_APPLY_MOVE` priority 100: `にげられない技: すでに状態変化` でフレームワーク側が処理）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | 相手ににげられない状態付与 |

## Priority 根拠

くろいまなざしと同じ。`ON_TRY_MOVE_2` priority 120 と `ON_APPLY_MOVE` priority 100 でフレームワーク側が失敗・無効チェックを処理する。ハンドラ本体は `ON_STATUS_HIT` priority 100 のパターンに従う。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `とける` の後、`トーチカ` の前（五十音順）。

```python
def とおせんぼう_apply_volatile_to_defender(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """とおせんぼうの効果: 相手をにげられない状態にする。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="にげられない")
```

### data/move.py への更新

`"とおせんぼう"` エントリのハンドラを追加する。

## 注意点・エッジケース

- くろいまなざしと全く同じ実装パターン（volatile 名も `にげられない` で共通）
- 仕様表では命中率が `-`（必中）と記載されているが、data/move.py の既存エントリに `accuracy=100` が設定されているため現状のデータに従う
- ゴーストタイプへの無効・すでに逃げられない状態の失敗はすべてフレームワーク側が処理
- クモのすとも同じ volatile を使用する（自分が場を離れると消える）
