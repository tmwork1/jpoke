# れんごく 実装計画書

参照: docs/spec/moves/れんごく.md

## 現状

* `src/jpoke/data/moves/move_ra.py` の `"れんごく"` エントリは以下の通り実装済み。
  - type="ほのお", category="special", pp=5, power=100, accuracy=50
  - flags={"secondary_effect"}
  - `Event.ON_DAMAGE_HIT`: `れんごく_apply_burn_to_defender`（100%でやけど付与、`apply_ailment_to_defender` の
    `chance` 省略によりデフォルト値1を使用）
  - `lethal_handlers`: `LethalEvent.ON_HIT`: `れんごく_apply_やけど`（追加効果有効時にやけど状態を確定付与）
* 上記のうち type/category/pp/power/accuracy/やけど確率/リーサル計算は仕様と一致しており修正不要。
* 一方、`flags` に `"thaw"` が欠落している。れんごくはほのおタイプの攻撃技であるため、
  被弾した相手がこおり状態の場合はこおりを解凍する必要がある
  （`docs/spec/ailments/こおり.md` 31-32行目）。この解凍処理は `こおり_cure_by_thaw_move`
  （`src/jpoke/handlers/ailment.py`, `Event.ON_DAMAGE_HIT`, `ctx.move.has_flag("thaw")` で判定）
  が担っており、`flags` に `"thaw"` を追加するだけで有効になる（既存の `かえんほうしゃ` と同型）。
  新規ハンドラの実装は不要。

## 追加実装

### data/moves/move_ra.py の変更

```python
    "れんごく": MoveData(
        type="ほのお",
        category="special",
        pp=5,
        power=100,
        accuracy=50,
        flags={"secondary_effect", "thaw"},
        handlers={
            Event.ON_DAMAGE_HIT: h.MoveHandler(
                ha.れんごく_apply_burn_to_defender,
            )
        },
        lethal_handlers={
            LethalEvent.ON_HIT: LethalHandler(l.れんごく_apply_やけど)
        }
    ),
```

### handlers/move_attack.py

変更不要。既存の `れんごく_apply_burn_to_defender` をそのまま使用する。

### handlers/lethal.py

変更不要。既存の `れんごく_apply_やけど` をそのまま使用する（`ctx.move_secondary` が
真かつ防御側が無状態異常のときにやけどを確定付与、`キラースピン_apply_どく` と同型）。

## Priority 根拠

`Event.ON_DAMAGE_HIT` は追加効果付与（やけど付与ハンドラ）とこおり解凍判定（`こおり_cure_by_thaw_move`）
の両方が使われる標準イベントであり、他のやけど付与技（かえんほうしゃ等）と同じ priority（デフォルト）
を使用する。本技独自の priority 検討は不要。

リーサル計算の `LethalEvent.ON_HIT` も、他の確定追加効果技（Gのちから等）と同じ priority（デフォルト）
を使用する。

## 注意点・エッジケース

* 追加効果（やけど付与）の確率変動（ちからずく・てんのめぐみ・りんぷん・おんみつマント等）は
  `apply_ailment_to_defender` 内の `battle.resolve_secondary_chance` が汎用的に処理するため、
  本技側での追加実装は不要。
* れんごく自体は「使用者がこおり状態のとき無条件で解凍してから攻撃する」対象技には含まれない
  （かえんぐるま・かえんボール・せいなるほのお・もえつきる等の専用リストとは別）。
  そのため `Event.ON_TRY_ACTION` の thaw_attacker 系ハンドラは追加しない。
