# にどげり 実装計画書

## 仕様要約（docs/spec/moves/にどげり.md 参照）

* タイプ: かくとう / 分類: 物理 / 威力: 30（1ヒットあたり） / 命中: 100 / PP: 30
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果:
  * 2回連続で攻撃する固定回数の連続攻撃技（命中判定は1発目のみ、各ヒット威力は固定30）。
  * 追加効果なし。

## 現状の実装状況

`src/jpoke/data/move.py` に既に登録済み。

```python
"にどげり": MoveData(
    type="かくとう",
    category="physical",
    pp=30,
    power=30,
    accuracy=100,
    flags={"contact"},
    multi_hit={
        "min": 2,
        "max": 2,
        "check_hit_each_time": False,
        "power_sequence": (),
    },
    handlers={},  # 追加効果なし
),
```

* `multi_hit.min == multi_hit.max == 2` のため、`_resolve_hit_count`（`core/move_executor.py`）は
  常に2を返す。
* `flags` に `check_hit_each_time` を含まないため、命中判定は1発目のみ行われる
  （`ダブルアタック` / `ダブルウイング` と同一パターン）。
* `power_sequence` が空タプルのため、`_resolve_hit_power` は各ヒットとも `move.power`（30）を返す。

## ハンドラ構成

固有ハンドラなし（`handlers={}`）。固定2ヒットの挙動は `core/move_executor.py` 側の
汎用機構（`multi_hit`）で処理されるため、`handlers/move_attack.py` への追加実装は不要。

## 注意点・エッジケース（review-test 向け）

1. **命中判定は1発目のみ**
   - `check_hit_each_time` フラグを持たないため、1発目が外れた場合は残りのヒットも発生しない。
2. **各ヒット威力は固定30**
   - `power_sequence` を使わず、威力上昇パターン（トリプルアクセル等）とは異なる。
3. **テスト方針**
   - `run_move` 実行後、`defender.hits_taken == 2` となることを確認する基本テスト
     （`ダブルアタック` と同一パターン）。
