# ほのおのパンチ 実装計画書

## 仕様要約

`docs/spec/moves/ほのおのパンチ.md` を参照。
ほのおタイプ・物理・威力75・命中100・PP15の直接攻撃技。
追加効果として10%の確率で相手を「やけど」状態にする。パンチ技のため「てつのこぶし」で威力1.2倍。

## ハンドラ構成

`Event.ON_DAMAGE_HIT` に `MoveHandler` デフォルト設定（`subject_spec="attacker:self"`, `priority=100`）で登録する。
`apply_ailment_to_defender` は `handlers/move.py` の既存共通関数を使用する。

| 技 | event | priority | subject_spec | 役割 |
|---|---|---|---|---|
| ほのおのパンチ | ON_DAMAGE_HIT | 100 | attacker:self | やけど 10% |

## Priority 根拠

`docs/spec/turn.md` に `Event.ON_DAMAGE_HIT` は掲載されていない（ダメージ計算内部の追加効果付与イベント）。
既実装の `かみなりパンチ`・`ほのおのキバ` と同じ priority=100（`h.MoveHandler` のデフォルト）を採用する。

## 実装箇所

### `src/jpoke/handlers/move_attack.py`

```python
def ほのおのパンチ_apply_burn_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="やけど", chance=0.1)
```

### `src/jpoke/data/moves/move_ha.py`

"ほのおのパンチ": MoveData(

## リーサル計算

- 対象外（`n/a`）。やけどは毎ターンのダメージ・物理攻撃半減効果であり、命中時に即座に決着を左右する類のものではないため、
  本技専用のリーサルハンドラは不要（`docs/progress/move.md` のリーサル実装列も `n/a`）。

## エッジケース

- ほのおタイプの相手はやけど無効（`ailment_manager` 側の共通処理で判定されるため、本技のハンドラに追加ガードは不要）。
- 特性「てつのこぶし」による威力1.2倍は `flags={"punch"}` を通じて特性側ハンドラが処理するため、本技側の実装は不要。
