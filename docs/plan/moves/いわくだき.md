# いわくだき 実装計画書

## 仕様要約（docs/spec/moves/いわくだき.md 参照）

* タイプ: かくとう / 分類: 物理 / 威力: 40 / 命中: 100 / PP: 15
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 追加効果として、50%の確率で相手の『ぼうぎょ』ランクを1段階下げる。
* パンチ技には分類されない（一次情報より。`flags` に `"punch"` を含めない）。

## 現状の実装状況

`src/jpoke/data/move.py` に既に登録済みだが、`flags` に `"secondary_effect"` が
欠落していた（**バグ**）。本技は追加効果（50%ぼうぎょ-1）を持つため、
ちからずく特性（`ちからずく_boost` / `ちからずく_disable_secondary_effect`、
`src/jpoke/handlers/ability.py`）が `ctx.move.has_flag("secondary_effect")` で判定する
対象に含める必要がある。欠落したままだと、ちからずく所持ポケモンが使用した場合に
威力1.3倍化とぼうぎょ低下無効化のどちらも発生せず、通常技として処理されてしまう。

### 修正後

```python
"いわくだき": MoveData(
    type="かくとう",
    category="physical",
    pp=15,
    power=40,
    accuracy=100,
    flags={"contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.いわくだき_lower_defender_def,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 50%の確率で相手の『ぼうぎょ』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def いわくだき_lower_defender_def(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"def": -1}, chance=0.5)
```

`modify_defender_stats`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は `flags={"secondary_effect"}` さえ
付与されていれば汎用的に処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（能力ランク低下）は既存の同種ハンドラ（あわ・いにしえのうた 等）と
同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。個別の priority 検討は不要。

### data/move.py

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用（修正の主眼）**
   - `flags={"secondary_effect"}` の追加により、ちからずく所持時は威力1.3倍（40→52）になり、
     ぼうぎょ低下の追加効果は発動しなくなる。
2. **パンチ技には分類されない**
   - 技の説明文は「パンチで攻撃する」だが、一次情報の「備考」欄に明記の通り
     てつのこぶし・パンチグローブの対象にはならない。`flags` に `"punch"` を追加しないこと。
3. **テスト方針**
   - `secondary_chance` を0/1に固定し、ぼうぎょランクの変化有無を確認するテストを追加する
     （`test_いわくだき_相手にダメージを与える` / `test_いわくだき_ぼうぎょ低下が発動する` /
     `test_いわくだき_ぼうぎょ低下が発動しない`）。
   - ちからずくとの相互作用テスト（`test_ちからずく_...`）は既存の ちからずく テストファイル側で
     他の secondary_effect 技と同様の網羅がされていれば個別追加は不要（既存パターンを確認する）。
