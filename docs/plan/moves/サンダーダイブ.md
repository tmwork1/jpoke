# サンダーダイブ 実装計画書

## 仕様要約（docs/spec/moves/サンダーダイブ.md 参照）

* タイプ: でんき / 分類: 物理 / 威力: 100 / 命中: 95 / PP: 16（champions基準、`docs/champions/move_list.txt` 準拠）
* 対象: 相手1体 / 優先度: 0 / 接触あり
* じゅうりょく状態でも使用可能（`gravity_restricted` フラグ対象外。とびひざげりとの相違点）
* ちいさくなる状態の相手に必中・威力2倍（`minimize` フラグ）
* 効果: 技が外れる・失敗すると自分の最大HPの1/2ダメージを受ける（切り捨て、最低1）
  * マジックガードで防げるが、いしあたまでは防げない（`reason="fixed_recoil"`）
  * すてみ所持時は威力1.2倍（`recoil` フラグの対象）

## 現状の実装状況

`src/jpoke/data/moves/move_sa.py` に登録済み。

```python
"サンダーダイブ": MoveData(
    type="でんき",
    category="physical",
    pp=15,
    power=100,
    accuracy=95,
    flags={"minimize", "contact"},
    handlers={
        Event.ON_MISS: h.MoveHandler(
            ha.サンダーダイブ_crash,
        ),
    }
),
```

```python
def サンダーダイブ_crash(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """サンダーダイブが外れた場合の失敗反動ダメージ。自分の最大HPの1/2を受ける。"""
    battle.modify_hp(ctx.attacker, v=-max(1, ctx.attacker.max_hp // 2), reason="self_cost", source=ctx.attacker)
    return HandlerReturn(value=value)
```

威力・命中率・タイプ・分類・`minimize` フラグ・ON_MISSの発火条件は、いずれも仕様書と一致しており問題なし。
`gravity_restricted` を持たない点も、とびひざげりと異なりじゅうりょく状態でも使用可能という仕様と一致している。

### レビューで判明した修正点

1. **PPが champions 値と不一致（バグ）**: `pp=15` は Gen9本家の値であり、champions 基準への移行漏れ。
   `docs/champions/move_list.txt` の該当行（`サンダーダイブ	でんき	物理	100	95	16	接○	守○`）に基づき
   `pp=16` に修正した。
2. **`recoil` フラグの欠落**: すてみ特性は `ctx.move.has_flag("recoil")` を判定条件にしており
   （`handlers/ability.py` の `すてみ_boost_power`）、このフラグが無いと威力1.2倍が発動しない。
   wiki の関連項目に「失敗すると自分がダメージを受けてしまうわざ」グループとして記載されており、
   同グループの かかとおとし・とびげり・とびひざげり は各レビューで既に `recoil` フラグを追加済みのため、
   同様に `flags` に `"recoil"` を追加した。
3. **失敗反動の `reason` の誤り（バグ）**: `サンダーダイブ_crash` は `reason="self_cost"` でHPを減少させていたが、
   `"self_cost"` は `マジックガード_ignore_damage`（`handlers/ability.py`）の除外リストに含まれており、
   マジックガードで**防げない**扱いになっていた。
   本来はマジックガードで防げる必要がある（とびげり・とびひざげりのレビューで判明した同種のバグパターン、
   `docs/plan/moves/失敗反動技.md` 注記2参照）。
   一方でいしあたまでは防げてはいけない。この「マジックガードで防げるが、いしあたまでは防げない」という
   組み合わせは、本プロジェクトの `HPChangeReason` に既に `"fixed_recoil"`（てっていこうせん 等が使用）として
   定義されているため、`reason="self_cost"` → `reason="fixed_recoil"` に修正した。

## 実装箇所

### handlers/move_attack.py

```python
def サンダーダイブ_crash(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """サンダーダイブが外れた場合の失敗反動ダメージ。自分の最大HPの1/2を受ける。

    マジックガードで防げるが、いしあたまでは防げない確定反動として扱う
    （reason="fixed_recoil"。"self_cost" だとマジックガードで防げなくなってしまうため区別する）。
    """
    battle.modify_hp(ctx.attacker, v=-max(1, ctx.attacker.max_hp // 2), reason="fixed_recoil", source=ctx.attacker)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース（review-test 向け）

1. **命中時に反動が発生しないこと**: `ON_MISS` は命中判定失敗時のみ発火するため、命中時はダメージを受けない。
2. **外れたとき失敗反動ダメージを受けること**: `max(1, max_hp // 2)` 減少する。
3. **いしあたまとの相互作用**: 失敗反動は `reason="fixed_recoil"` のため、いしあたまでは防げないことを確認する
   （本レビューでの新規修正点）。
4. **マジックガードとの相互作用**: `reason="fixed_recoil"` は `マジックガード_ignore_damage` の除外リストに
   含まれないため、マジックガード所持時は反動ダメージを受けないことを確認する（本レビューでの新規修正点）。
5. **すてみとの相互作用**: `recoil` フラグ追加により、すてみ特性所持時に威力補正 `apply_fixed_modifier(value, 4915)`
   (1.2倍) が発生することを確認する（本レビューでの新規修正点）。
6. **flags に "recoil" が含まれること**: すてみ対応の確認。
7. **PP が16であること**: champions 基準のPP値を確認する（本レビューでの新規修正点）。
8. **ちいさくなるとの相互作用**: 既存のパラメタライズドテスト（`tests/volatiles/test_volatile_satana.py` の
   `minimize_enhance_moves`）に既にサンダーダイブが含まれており、必中・威力2倍が確認済み。変更不要。
9. **じゅうりょくとの相互作用**: `gravity_restricted` フラグを持たないため、じゅうりょく状態でも通常通り
   使用できることを確認する（とびひざげりとの相違点であり、既存実装が正しいことの確認のみ。新規ハンドラ
   実装は不要）。
10. **既知の実装制約**: まもる・タイプ無効化・でんき技無効化特性による失敗時の反動は現状未実装
    （`docs/spec/moves/サンダーダイブ.md` 「実装上の制約」参照）。本レビューの対象範囲外として現状維持する。
