# ドレインパンチ 実装計画書

参照: docs/spec/moves/ドレインパンチ.md

## 概要

物理・かくとうタイプのドレイン技（パンチ技）。相手に与えたダメージの50%だけ自分のHPを
回復する。既存のドレイン技（ギガドレイン・すいとる・きゅうけつ等）と同一の共通処理
`_drain_hp`（`handlers/move_attack.py`）を再利用する。
パンチ技であるため特性『てつのこぶし』・道具『パンチグローブ』の威力補正対象になるが、
これらは `flags={"punch"}` を参照する既存の汎用ハンドラ（`handlers/ability.py` の
`てつのこぶし` 系ハンドラ、`handlers/item.py` の `パンチグローブ` 系ハンドラ）で処理されるため
本技側で個別実装は不要。

## MoveData（data/move.py）

```python
"ドレインパンチ": MoveData(
    type="かくとう",
    category="physical",
    pp=12,
    power=75,
    accuracy=100,
    flags={"contact", "punch", "heal"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.ドレインパンチ_drain,
            priority=20,  # turn.md: ON_HIT priority 20 (HP吸収技による回復)
        )
    }
),
```

* `pp=12` は `src/jpoke/data/ps-champ-ja/moves.json` 基準（本家基準の10ではなく、
  チャンピオンズ準拠の値を採用する）。
* `flags` に `"punch"`（パンチ技）・`"heal"`（HPを吸収する技）を含める。
* `subject_spec` は `MoveHandler` のデフォルト `"attacker:self"` を使用（明示指定不要）。

## ハンドラ（handlers/move_attack.py）

```python
def ドレインパンチ_drain(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """ドレインパンチの回復量を計算する。"""
    _drain_hp(battle, ctx, value, heal_ratio=0.5)
    return HandlerReturn(value=value)
```

* 共通関数 `_drain_hp(battle, ctx, damage, heal_ratio)` を呼び出す。
  * `damage`（与えたダメージ）が0の場合（みがわりに命中した場合）は `ctx.substitute_damage` を使う。
  * `heal_amount = max(1, round_half_up(damage * heal_ratio))` で算出し、
    `Event.ON_CALC_DRAIN` を発火して `おおきなねっこ`（1.3倍）・
    `ヘドロえき`（回復→反動ダメージへの変換）を適用する。
  * `battle.modify_hp(ctx.attacker, v=heal_amount, reason="drain")` で攻撃者のHPを回復する。
* かいふくふうじ状態での選択不可・失敗は `heal` フラグを参照する汎用ハンドラ
  （`handlers/volatile.py` の `かいふくふうじ_try_action`）で処理されるため、本技側の実装は不要。

## イベント・priority

* `Event.ON_HIT`（ダメージ発生後の処理）: priority=20
  * `docs/spec/turn.md` の `Event.ON_HIT` 表「20 | HP吸収技による回復」に対応。
  * 既存の全ドレイン技ハンドラ（ウッドホーン・ギガドレイン・きゅうけつ・すいとる等）と同じ
    priority=20 で登録し、処理順の一貫性を保つ。

## テスト方針

* `tests/moves_attack/test_move__group.py` の `test_drain技グループ_回復量が与ダメのheal_ratio倍になる`
  に `("ドレインパンチ", 0.5)` として含める（既存）。
* `tests/moves_attack/test_move_ta.py` に個別の基本動作テスト
  `test_ドレインパンチ_使用後に攻撃者のHPが回復する`（既存）、
  `test_ドレインパンチ_回復量の端数は四捨五入で切り上げになる`（与ダメが奇数のときの
  丸め挙動を検証する専用テスト）を用意する。
* パンチ技としての威力補正（てつのこぶし・パンチグローブ）は
  `tests/abilities/test_ability_ta.py`・`tests/items/test_item_ha.py` 側で
  グループテストとして検証済みのため本ファイルでは個別実装しない。
