# きあいだま 実装計画書

## 仕様要約（docs/spec/moves/きあいだま.md 参照）

* タイプ: かくとう / 分類: 特殊 / 威力: 120 / 命中: 70 / PP: 5
* 対象: 相手1体 / 優先度: 0 / 非接触
* 効果: 追加効果として、10%の確率で相手の『とくぼう』ランクを1段階下げる。
* 弾（bullet）技に分類される（一次情報の「弾のわざ」カテゴリより）。

## 現状の実装状況

`src/jpoke/data/move.py` に既に登録済みで、仕様と一致している（変更不要）。

```python
"きあいだま": MoveData(
    type="かくとう",
    category="special",
    pp=5,
    power=120,
    accuracy=70,
    flags={"bullet", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.きあいだま_lower_defender_spd,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手の『とくぼう』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def きあいだま_lower_defender_spd(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spd": -1}, chance=0.1)
```

`modify_defender_stats`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は `flags={"secondary_effect"}` さえ
付与されていれば汎用的に処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（能力ランク低下）は既存の同種ハンドラ（あわ・いにしえのうた・
いわくだき 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。
`ON_DAMAGE_HIT` は `docs/spec/turn.md` に未掲載のイベント（ダメージ計算内部）だが、
既存の同種ハンドラの実装パターンを踏襲するため個別の priority 検討は不要。

対象外（`docs/progress/move.md` のリーサル実装列は n/a）。確率発動の相手ランク低下効果であり、

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` が付与されているため、ちからずく所持時は威力1.3倍（120→156）になり、
     とくぼう低下の追加効果は発動しなくなる。
2. **弾（bullet）フラグ**
   - 特性『ぼうじん』の無効化対象となる。個別のガードは不要（`ぼうじん` 側で判定される）。
3. **テスト方針**
   - `secondary_chance` を0/1に固定し、とくぼうランクの変化有無を確認するテストを追加する
     （`test_きあいだま_相手にダメージを与える` / `test_きあいだま_とくぼう1段階低下が発動する` /
     `test_きあいだま_とくぼう1段階低下が発動しない`）。
