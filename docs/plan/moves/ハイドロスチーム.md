# ハイドロスチーム 実装計画書

参照: docs/spec/moves/ハイドロスチーム.md

## 現状（レビュー前）

* `src/jpoke/data/moves/move_ha.py` の `"ハイドロスチーム"` エントリは以下の通り実装済み。
  - type="みず", category="special", pp=**15**, power=80, accuracy=100
  - flags={"thaw"}
  - `Event.ON_CALC_POWER_MODIFIER`: `ハイドロスチーム_power_modifier`
    （にほんばれ下で威力1.5倍になる補正。攻撃側・防御側それぞれのばんのうがさ判定を含む）
* `"thaw"` フラグが設定されているため、`こおり_cure_by_thaw_move`
  （`Event.ON_DAMAGE_HIT`, defender:self）により、相手がこおり状態のときにこの技のダメージを
  受けるとこおりが解除される効果は既に実装済み。
* 一方、以下の3点が **未実装／バグだった**。

### 1. PP が Gen9本家基準の値のまま（移行漏れ）

`docs/champions/changes_from_sv.md` により「すべての技のPPが8・12・16・20の4段階に統一」
されているのがchampionsの仕様である。`docs/champions/move_list.txt` にはウネルミナモの
専用技であるハイドロスチームの行自体が存在しない（信頼できる直接の出典がない）。

しかし同ファイル内でGen9のPPが15だった他の特殊技を確認すると、あくのはどう（34行目）・
シャドーボール（348行目）はいずれもchampions PP=16に統一されていることが確認できる
（ねっとうもGen9本家PP15から644行目でPP=16に統一されている。ただし
`src/jpoke/data/moves/move_na.py` 側のねっとうの実装PPは本レビュー時点で15のままであり、
別途の移行漏れバグが残っているが、本レビューの対象外のため修正しない）。
この一貫したパターンから、ハイドロスチームのchampions PPも16であると推定するのが妥当。
**PP=15は移行漏れのバグであり16に修正する**。

### 2. 使用者自身の無条件解凍処理が未実装

`"thaw"` フラグは現状 `こおり_cure_by_thaw_move`（被弾側＝defender）にのみ使われており、
使用者自身の無条件解凍処理は個別実装が必要（かえんぐるま・かえんボール・せいなるほのお・
もえつきる・ねっさのだいち・ねっとう・スチームバーストは実装済みだが、ハイドロスチームは
**未実装だった**）。

### 3. ちからずくによる被弾側解凍効果の無効化条件が過剰

`src/jpoke/handlers/ailment.py` の `こおり_cure_by_thaw_move` は、`"thaw"` フラグを持つ技の
うち「タイプが『ほのお』でない」ことのみを条件にちからずくで被弾側の解凍効果を無効化していた。

```python
if (
    ctx.move.type != "ほのお"
    and ctx.attacker.ability.name == "ちからずく"
):
    return HandlerReturn(value=value)
```

しかし `docs/spec/abilities/ちからずく.md`（13行目・39行目）のちからずく無効化リストには
「シャカシャカほう/スチームバースト/ねっさのだいち/ねっとう」が明記されている一方、
**ハイドロスチームは含まれていない**。ハイドロスチーム自身のwiki記載にも「ハイドロスチームは
ちからずくの対象技ではない。この特性のポケモンが使用したときでも……相手や自分のこおり状態を
治す効果は発動する」と明記されており、ハイドロスチームはちからずくによる被弾側解凍効果の
無効化を受けない例外である。

上記の条件はタイプのみで判定しており、みずタイプであるハイドロスチームも「ほのおでない」に
該当するため、誤ってちからずくで解凍効果が無効化されるバグがあった。

## 追加実装

### 1. PP修正

`src/jpoke/data/moves/move_ha.py` の `"ハイドロスチーム"` エントリの `pp=15` を `pp=16` に
修正する。

### 2. 使用者自身の無条件解凍ハンドラ

`src/jpoke/handlers/move_attack.py` に `ハイドロスチーム_thaw_attacker` を追加する。
`ねっとう_thaw_attacker` / `スチームバースト_thaw_attacker`
（同ファイル内、Event.ON_TRY_ACTION, priority=5）と全く同じパターンで実装する。

```python
def ハイドロスチーム_thaw_attacker(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ハイドロスチーム: こおり状態でも使用可能にし、こおりを解凍する。

    こおり_action (priority=10) より先に発火させる (priority=5) ことで、
    ailment が除去された状態で こおり_action の validity check が走り、
    こおり_action がスキップされる。
    """
    mon = ctx.attacker
    if mon.ailment.name == "こおり":
        battle.ailment_manager.remove(mon)
    return HandlerReturn(value=value)
```

`src/jpoke/data/moves/move_ha.py` の `"ハイドロスチーム"` エントリに以下を追加する。

```python
Event.ON_TRY_ACTION: h.MoveHandler(
    ha.ハイドロスチーム_thaw_attacker,
    priority=5,
),
```

#### priority の根拠

`docs/plan/moves/ねっとう.md` の「priority の根拠」節（`こおり_action` が priority=10 で
即座に `stop_event=True` するため、`docs/spec/turn.md` 記載の priority=170 では無条件解凍が
機能しない、という分析）をそのまま踏襲する。既存の同種実装
（かえんぐるま・ねっさのだいち・ねっとう・スチームバースト）と同じ priority=5 を採用する。

### 3. ちからずくによる被弾側解凍効果の無効化条件を修正

`src/jpoke/handlers/ailment.py` の `こおり_cure_by_thaw_move` に、タイプ判定に加えて
`"secondary_effect"` フラグの有無を条件に追加する。

```python
def こおり_cure_by_thaw_move(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """thawラベルを持つ技でダメージを受けたら解凍する。

    ほのおタイプ技による解凍はタイプ由来の効果であり追加効果に該当しないため、
    ちからずくの影響を受けない。一方、シャカシャカほう/スチームバースト/
    ねっさのだいち/ねっとう（第六世代以降）等、ほのお以外のタイプでこの効果を
    持つ技のうち「secondary_effect」フラグを持つもの（＝ちからずく対象技）は
    追加効果として扱われるため、使用者がちからずくの場合はこの効果が発動しない
    （りんぷんの影響は受けない）。
    ハイドロスチームは「secondary_effect」フラグを持たずちからずくの対象技
    ではないため、使用者がちからずくでもこの効果は常に発動する
    （docs/spec/moves/ハイドロスチーム.md参照）。
    """
    if not ctx.move.has_flag("thaw"):
        return HandlerReturn(value=value)
    if (
        ctx.move.type != "ほのお"
        and ctx.move.has_flag("secondary_effect")
        and ctx.attacker.ability.name == "ちからずく"
    ):
        return HandlerReturn(value=value)
    battle.ailment_manager.remove(ctx.defender)
    return HandlerReturn(value=value)
```

この修正は `"thaw"` フラグを持つ全ての技（かえんぐるま・かえんほうしゃ・かえんボール・
クロスフレイム・シャカシャカほう・スチームバースト・せいなるほのお・だいもんじ・
ねっさのだいち・ねっとう・ハイドロスチーム・フレアドライブ・めらめらバーン・もえつきる・
れんごく）に影響するが、ハイドロスチーム以外は全て「ほのおタイプ」または
「`"secondary_effect"` フラグを持つ非ほのおタイプ」のいずれかに該当するため、
条件の変更前後で挙動は変わらない。ハイドロスチームのみが「非ほのおタイプかつ
`"secondary_effect"` フラグなし」に該当し、修正後は常に解凍効果が発動するようになる
（修正前は誤ってちからずくで無効化されていた）。

`_is_sheer_force_blocked`（`src/jpoke/handlers/move_attack.py` 840行目）と同等の判定条件
（`ctx.move.has_flag("secondary_effect")` かつ ちからずく）だが、ここではさらに
「ほのおタイプでない」条件も必要（ほのおタイプの `"secondary_effect"` 技である
かえんぐるま等は、タイプ由来の解凍効果自体はちからずくの影響を受けないため）なので、
既存の `_is_sheer_force_blocked` をそのまま呼び出すのではなく、型チェックと
`has_flag("secondary_effect")` チェックを併記する形にした。
