# 実装計画書: エレキフィールド

## 仕様要約

- タイプ: でんき / カテゴリ: 変化 / PP: 10 / 対象: 全体の場（target="field"）
- 5ターンの間、場の状態を「エレキフィールド」にする
- 地面にいるポケモンが「ねむり」「あくび（ねむけ）」状態にならない（すでにねむり状態の場合は回復しない）
- でんきタイプの技の威力が 1.3 倍になる
- エレキシードを持つポケモンの「ぼうぎょ」ランクが 1 段階上がる（地面外でも発動）

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | グローバルフィールドを「エレキフィールド」に設定 |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「天候・フィールド: すでに同じ」が priority 100 として記載されている。同種ハンドラ `じゅうりょく_activate_global_field` が priority デフォルト（100）を使用しているため、同一パターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: `えんまく_modify_defender_stats` の後、`おいかぜ` 関連の前（「え」→「お」の間、ただし「エ」は「え」として扱う）。現在のファイルには「え」ブロックに `えんまく_modify_defender_stats` があり、その直後に挿入する。

```python
def エレキフィールド_activate_global_field(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """エレキフィールド: グローバルフィールドをエレキフィールドにする。"""
    return HandlerReturn(value=battle.global_manager.activate("エレキフィールド", 5))
```

### `src/jpoke/data/move.py`

既存の `"エレキフィールド"` エントリに handlers を追加する。

```python
"エレキフィールド": MoveData(
    type="でんき",
    category="変化",
    pp=10,
    target="field",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.エレキフィールド_activate_global_field,
        ),
    }
),
```

## 注意点・エッジケース

- `global_manager.activate` はすでに同じフィールドが存在する場合に False を返すことが想定される。ハンドラは戻り値をそのまま `HandlerReturn(value=...)` で返すだけでよい
- `じゅうりょく_activate_global_field` と同一パターン
- グラスフィールド・サイコフィールドと同時に設定することはできない（global_manager が排他管理）
