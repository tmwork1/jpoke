# 実装計画書: りんごさん（再レビュー）

## 仕様要約

`docs/spec/moves/りんごさん.md` 参照。

* タイプ: くさ / 分類: 特殊 / 威力: 90（Champions。SVまでは80） / 命中率: 100 / PP: 12（Champions。SVまでは10） / 対象: 相手1体
* 直接攻撃ではない。追加効果として、100%の確率で相手の『とくぼう』ランクを1段階下げる。

## 既存実装の確認結果と問題点

`src/jpoke/handlers/move_attack.py` の `りんごさん_lower_defender_spd` が
`Event.ON_DAMAGE_HIT` で `modify_defender_stats(battle, ctx, value, stats={"spd": -1})`
（`chance` 省略でデフォルト100%）を実行しており、同種のとくぼうダウン追加効果を持つ
`Gのちから_lower_defender_def` と同じ実装パターンになっている。イベント・優先度
（デフォルト100・`docs/spec/turn.md` の「100 スケイルショットによる能力変化」相当）も
問題ない。

一方、`src/jpoke/data/moves/move_ra.py` の `MoveData` エントリに以下の誤りがあった。

* `power=80`（SVまでの値）になっていたが、`src/jpoke/data/ps-champ-ja/moves.json` /
  `docs/champions/changes_from_sv.md` / `docs/progress/move.md` によれば Champions では
  `90` が正しい値のため修正する。
* `pp=10`（SVまでの値）になっていたが、`src/jpoke/data/ps-champ-ja/moves.json` /
  `docs/progress/move.md` によれば Champions では `12` が正しい値のため修正する。

`flags={"secondary_effect"}` は一次情報（直接攻撃×、ちからずく対象技）と
一致しており問題ない（`contact` フラグを持たないことも正しい）。
`lethal_handlers={LethalEvent.ON_HIT: LethalHandler(l.りんごさん_lower_spd)}`
も `Gのちから_lower_def` と同一パターンで問題ない。

## ハンドラ構成（変更なし）

|------|----------|----------|--------------|------|
| りんごさん | `Event.ON_DAMAGE_HIT` | デフォルト（100） | `attacker:self`（デフォルト、`modify_defender_stats` 内部で `ctx.defender` を操作） | 命中後、100%の確率で相手のとくぼうランクを1段階下げる |

* `docs/spec/turn.md` の `## Event.ON_DAMAGE`（実装上の `Event.ON_DAMAGE_HIT` に相当）の
  「100 スケイルショットによる能力変化」に準じ、ダメージ適用後にランク変化を発動させる。

## リーサル計算

`docs/progress/move.md` の該当行は「リーサル実装: x」「リーサルテスト: x」。
`handlers/lethal.py` の `りんごさん_lower_spd` は `Gのちから_lower_def` と同一パターンで
`ctx.move_secondary` が真のときのみ `ctx.defender.rank["spd"]` を1段階下げており、
仕様・実装と整合している。修正不要。

## テスト

`tests/moves_attack/test_move_yarawa.py` の `test_りんごさん_とくぼう1段階低下が発動する` で
追加効果の発動を確認済み。`tests/test_lethal.py` の
`test_りんごさん_とくぼうダウン_secondary有り` / `_secondary無し` で
リーサル計算のランク反映を確認済み。今回の修正（威力・PP）はこれらのテストの
アサーションに直接影響しないため、既存テストのままで問題ない。
