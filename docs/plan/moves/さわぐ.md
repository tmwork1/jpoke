# さわぐ 実装計画書

## 概要

さわぐ技本体（`data/move.py` の `"さわぐ"` エントリ）の実装計画。
「さわぐ」揮発性状態そのものの詳細（強制行動・ねむり無効化・さわがしい補助状態など）は
`docs/plan/volatiles/さわぐ.md` に既存の計画があるため、本書では技側（`ON_HIT` での状態付与）
のみを扱う。

## 実装対象

### さわぐ_apply（ON_HIT）

**ファイル**: `handlers/move_attack.py`（実装済み）
**登録**: `data/move.py`（実装済み）

```python
def さわぐ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """さわぐ技の初回命中時にさわぐ揮発性状態を付与する。

    すでにさわぐ状態の場合（強制行動の2・3ターン目）は何もしない。
    カウントは3ターン固定。
    """
    attacker = ctx.attacker
    if attacker.has_volatile("さわぐ"):
        return HandlerReturn(value=value)
    battle.volatile_manager.apply(
        attacker, "さわぐ", count=3, source=attacker,
        move_name=ctx.move.name
    )
    return HandlerReturn(value=value)
```

```python
"さわぐ": MoveData(
    type="ノーマル",
    category="special",
    pp=10,
    power=90,
    accuracy=100,
    flags={"non_negoto", "sound"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.さわぐ_apply,
        ),
    }
),
```

- `subject_spec`: `MoveHandler` のデフォルト `"attacker:self"`
- `priority`: `MoveHandler` のデフォルト 100
- イベント: `Event.ON_HIT`（ダメージ発生後の処理。技が命中した場合のみ発火するため、
  「最初の攻撃が命中したときにさわぐ状態になる」という仕様と自然に一致する）

## Priority 根拠

- `Event.ON_HIT` は `docs/spec/turn.md` に個別の priority 一覧があるが「さわぐ状態の付与」に
  該当する項目は明記されていない。同種の「命中時に強制行動用の揮発性状態を付与する」技
  （`あばれる` 系 `あばれる_apply`）が同じく `ON_HIT` デフォルト priority (100) で
  登録されているため、これに倣う（`docs/plan/moves/攻撃技_あばれ系.md` 参照）。
- `Event.ON_HIT` 内で呼び出す `battle.volatile_manager.apply` は内部で
  `Event.ON_VOLATILE_START` を発火し、`さわぐ_start`（場のねむり解除・さわがしい付与）
  を実行する。これは `さわぐ` 揮発性状態側の実装（`docs/plan/volatiles/さわぐ.md`）に既定。
