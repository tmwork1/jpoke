# 計画書: ほえる

## 仕様要約

- 変化技 / ノーマルタイプ / 優先度 -6 / PP 20
- ダメージなし。相手を強制的に交代させる（トレーナーバトルではランダムな控えに交代）
- 音技（label: `sound`）のため、みがわり状態を貫通する
- まもる・みきり・トーチカ等の防御技を貫通する（ほえる・ふきとばし固有）
- 命中判定なし（必中）
- 失敗条件:
  - 相手の控えポケモンがいない
  - きゅうばん特性 / ねをはる状態で無効
  - ほえるの無効化が通過した場合のみ交代（`ON_TRY_BLOW` で判定）
- ゴーストタイプに対しても有効（ノーマル技の通常無効を受けない扱い）

## 現状の実装状況

| コンポーネント | 状態 | 備考 |
|---|---|---|
| `data/move.py` の `ほえる` エントリ | 登録済み（ハンドラなし） | `priority=-6`, `labels=["sound"]` のみ設定済み |
| `handlers/move_status.py` の `on_blow_apply` | 実装済み | `ふきとばし` と共用 |
| `handlers/move_status.py` の `blow` | 実装済み | `ふきとばし` と共用 |
| `Event.ON_TRY_BLOW` | 定義済み | `enums/event.py` に存在 |
| `Event.ON_BEFORE_APPLY_MOVE` | 定義済み | `ふきとばし` で priority=30 を確認済み |

`ふきとばし` は同一の `on_blow_apply` + `blow` を使用して実装済み。
`ほえる` には同じハンドラを登録するだけで実装が完成する。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `Event.ON_BEFORE_APPLY_MOVE` | 30 | （デフォルト） | `on_blow_apply` — `ON_TRY_BLOW` を emit し、きゅうばん・ねをはる等で無効化されたか判定。無効ならイベントを停止 |
| `Event.ON_STATUS_HIT` | （デフォルト） | （デフォルト） | `blow` — 控えからランダムに1体選んで交代を実行 |

> 注意: `ふきとばし` は `ON_HIT`（ダメージ技命中後）を使っているが、`ほえる` はダメージ技ではないため **`ON_STATUS_HIT`** を使用する。

## Priority 根拠

- `docs/spec/moves/強制交代技.md` および `docs/spec/moves/_blow.md` に「全強制交代技の優先度: -6」と明記されている。
- `Event.ON_BEFORE_APPLY_MOVE` の priority=30 は、`ふきとばし` の登録に揃える（turn.md 該当行: `30 | ほえる・ふきとばしの無効化`）。

```python
"ほえる": MoveData(
    type="ノーマル",
    category="変化",
    pp=20,
    priority=-6,
    labels=["sound"],
    handlers={
        Event.ON_BEFORE_APPLY_MOVE: h.MoveHandler(
```

```python
def test_ほえる_相手が強制交代する():
    """ほえる: 相手に控えポケモンがいる場合、強制的に交代させる"""
    battle = t.start_battle(
        team0=[Pokemon("ピカチュウ", move_names=["ほえる"])],

    t.run_move(battle, 0)

def test_ほえる_控えがいない場合失敗する():
    """ほえる: 相手に控えポケモンがいない場合は失敗する"""
    battle = t.start_battle(
        team0=[Pokemon("ピカチュウ", move_names=["ほえる"])],
        team1=[Pokemon("カビゴン")],
        accuracy=100,
    )
    defender = battle.actives[1]
    t.run_move(battle, 0)

    assert battle.actives[1] is defender


def test_ほえる_みがわりを貫通する():
    """ほえる: 音技のためみがわり状態の相手にも効く（みがわりを無視して交代させる）"""
    battle = t.start_battle(
        team0=[Pokemon("ピカチュウ", move_names=["ほえる"])],
        team1=[Pokemon("カビゴン"), Pokemon("フシギダネ")],
        volatile1={"みがわり": {"hp": 50}},
        accuracy=100,
    )
    defender_before = battle.actives[1]
    t.run_move(battle, 0)

    assert battle.actives[1] is not defender_before


def test_ほえる_きゅうばんで無効化される():
    """ほえる: 相手がきゅうばん特性を持つ場合は交代が発生しない"""
    battle = t.start_battle(
        team0=[Pokemon("ピカチュウ", move_names=["ほえる"])],
        team1=[Pokemon("ピカチュウ", ability_name="きゅうばん"), Pokemon("フシギダネ")],
        accuracy=100,
    )
    defender = battle.actives[1]
    t.run_move(battle, 0)

    assert battle.actives[1] is defender
```

## 注意点・エッジケース（review-test 向け）

- **`ON_STATUS_HIT` vs `ON_HIT`**: `ほえる` は変化技（ダメージなし）なので `ON_STATUS_HIT` を使う。`ふきとばし` は `ON_HIT` を使っており、同じハンドラ関数 `blow` を使いながらイベントが異なる点に注意。
- **まもる貫通**: `ほえる` はまもる系防御技を貫通する（変化技型の固有仕様）。`ON_BEFORE_APPLY_MOVE` の `on_blow_apply` では `ON_TRY_BLOW` を発火し、きゅうばん・ねをはるのみを無効化対象とする。まもる系のガードは別途 `ON_CHECK_PROTECT` で処理されるが、`ほえる` には `ON_CHECK_PROTECT` ハンドラを登録しないことで貫通を実現（`ふきとばし` も同様）。
- **音技とみがわり**: `sound` ラベルにより `ON_CHECK_HIT_SUBSTITUTE` で変化技がみがわりを貫通するか判定される。`ほえる` には `sound` ラベルが既に設定されているため、みがわり貫通は自動的に有効になる（実装不要）。
- **命中判定**: `accuracy` フィールドが未設定（`None`）の場合は命中判定がスキップされる。`ほえる` の現在の MoveData は `accuracy` が未設定であり必中扱いになる。
- **ゴーストタイプへの有効性**: 仕様書に「ゴーストタイプ: ほえる・ふきとばし有効」と記載されている。タイプ無効化チェックがほえるに適用されるか実装を確認すること。ノーマルタイプとゴーストタイプは通常無相性だが、仕様上ほえるは有効。`ふきとばし` も同様の扱いであるため、既存動作を確認して整合を取ること。
- **ねをはる無効化**: `ねをはる` 状態による無効化は `ON_TRY_BLOW` ハンドラ側（volatiles）で処理されるため、ほえるのハンドラ側での重複ガードは不要。
- **交代先選択**: `blow` 関数は `get_available_switch_commands` でランダム選択する実装になっており、仕様の「控えポケモンからランダム選択」に対応済み。
