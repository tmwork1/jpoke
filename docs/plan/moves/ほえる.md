# 計画書: ほえる

## 仕様要約

- 変化技 / ノーマルタイプ / 優先度 -6 / PP 20（`docs/champions/move_list.txt` 準拠）
- ダメージなし。相手を強制的に交代させる（トレーナーバトルではランダムな控えに交代）
- 音技（flag: `sound`）のため、みがわり状態を貫通する
- まもる・みきり・トーチカ等の防御技を貫通する（`unprotectable` フラグで実現。ほえる・ふきとばし固有）
- 命中判定なし（必中）
- 失敗条件:
  - 相手の控えポケモンがいない（`on_blow_check_switch_target` で判定）
  - きゅうばん特性 / ばんけん特性 / ねをはる状態で無効（`ON_TRY_BLOW` で判定）
- ゴーストタイプに対しても有効（変化技はダメージ計算・タイプ相性判定を経由しないため自動的に満たす）

## 現状の実装状況（レビュー時点）

| コンポーネント | 状態 | 備考 |
|---|---|---|
| `data/moves/move_ha.py` の `ほえる` エントリ | 実装済み | `priority=-6`, `flags={"sound", "unprotectable"}` |
| `handlers/move_status.py` の `on_blow_apply` | 実装済み | `ふきとばし` と共用 |
| `handlers/move_status.py` の `on_blow_check_switch_target` | 実装済み | `ふきとばし` と共用 |
| `handlers/move_status.py` の `blow` | 実装済み | `ふきとばし` と共用 |
| `Event.ON_TRY_BLOW` | 定義済み | `enums/event.py` に存在 |
| `Event.ON_BEFORE_APPLY_MOVE` | 定義済み | `ふきとばし` で priority=30・100 を確認済み |

`ふきとばし` は同一の `on_blow_apply` + `on_blow_check_switch_target` + `blow` を使用して実装済み。
レビュー時点で `ほえる` には `unprotectable` フラグと `on_blow_check_switch_target`
（priority=100、交代不可判定）が欠落していたため、`ふきとばし` に揃えて追加した
（`docs/review/moves/ほえる.md` 参照）。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `Event.ON_BEFORE_APPLY_MOVE` | 30 | （デフォルト） | `on_blow_apply` — `ON_TRY_BLOW` を emit し、きゅうばん・ばんけん・ねをはる等で無効化されたか判定。無効ならイベントを停止 |
| `Event.ON_BEFORE_APPLY_MOVE` | 100 | （デフォルト） | `on_blow_check_switch_target` — 相手に交代可能な控えがいるか判定。いなければ失敗 |
| `Event.ON_STATUS_HIT` | （デフォルト） | （デフォルト） | `blow` — 控えからランダムに1体選んで交代を実行 |

> 補足: `ふきとばし` は変化技（`category="status"`）であるため `ON_HIT` ではなく `ON_STATUS_HIT` を
> 使用している。`ほえる` も同様に `ON_STATUS_HIT` を使う（両技で挙動は一致）。

## Priority 根拠

- `docs/spec/moves/強制交代技.md` および `docs/spec/moves/_blow.md` に「全強制交代技の優先度: -6」と明記されている。
- `Event.ON_BEFORE_APPLY_MOVE` の priority=30・100 は、`ふきとばし` の登録に揃える
  （turn.md 該当行: `30 | ほえる・ふきとばしの無効化`, `100 | ほえる・ふきとばし: 交代不可・野生`）。

## 実装箇所

### `src/jpoke/data/moves/move_ha.py`

```python
"ほえる": MoveData(
    type="ノーマル",
    category="status",
    pp=20,
    priority=-6,
    flags={"sound", "unprotectable"},
    handlers={
        Event.ON_BEFORE_APPLY_MOVE: [
            h.MoveHandler(
                hs.on_blow_apply,
                priority=30,
            ),
            h.MoveHandler(
                hs.on_blow_check_switch_target,
                priority=100,
            ),
        ],
        Event.ON_STATUS_HIT: h.MoveHandler(hs.blow),
    }
),
```

`handlers/move_status.py` への追加は不要（`on_blow_apply` / `on_blow_check_switch_target` / `blow` は既存）。

## 注意点・エッジケース（review-test 向け）

- **`ON_STATUS_HIT`**: `ほえる` は変化技（`category="status"`）なので `ON_STATUS_HIT` を使う。
  `ふきとばし` も同じく `category="status"` のため同一の `ON_STATUS_HIT` を使用しており、
  当初計画書にあった「`ふきとばし` は `ON_HIT` を使う」という記述は誤りだったため修正した。
- **まもる貫通**: `ほえる` はまもる系防御技を貫通する（変化技型の固有仕様）。これは
  `MoveData.flags` に `unprotectable` を設定することで実現する（`Move.is_blocked_by_protect`
  プロパティがこのフラグを参照する）。レビュー時点で `ほえる` にはこのフラグが欠落しており
  まもるで防がれてしまうバグがあったため追加した。
  `ON_BEFORE_APPLY_MOVE` の `on_blow_apply` は `ON_TRY_BLOW` を発火し、きゅうばん・ばんけん・
  ねをはるの無効化のみを判定する（まもる判定とは別経路）。
- **交代不可判定の欠落**: レビュー時点で `ほえる` には `on_blow_check_switch_target`
  （priority=100、相手に交代可能な控えがいるか判定するハンドラ）が登録されておらず、
  `ふきとばし` と異なり失敗ログが出ないまま静かに何も起きない実装になっていたため追加した。
- **音技とみがわり**: `sound` フラグにより `ON_CHECK_HIT_SUBSTITUTE`（`move_executor.py`）で
  変化技がみがわりを貫通するか判定される。`ほえる` には `sound` フラグが設定されているため、
  みがわり貫通は自動的に有効になる（実装不要）。
- **命中判定**: `accuracy` フィールドが未設定（`None`）の場合は命中判定がスキップされる。`ほえる` の現在の MoveData は `accuracy` が未設定であり必中扱いになる。
- **ゴーストタイプへの有効性**: 仕様書に「ゴーストタイプ: ほえる・ふきとばし有効」と記載されている。本プロジェクトでは変化技はダメージ計算・タイプ相性判定を経由しないため、この仕様は実装不要で自動的に満たされる。
- **ねをはる無効化**: `ねをはる` 状態による無効化は `ON_TRY_BLOW` ハンドラ側（volatiles）で処理されるため、ほえるのハンドラ側での重複ガードは不要。
- **交代先選択**: `blow` 関数は `get_forced_switch_commands` でランダム選択する実装になっており、仕様の「控えポケモンからランダム選択」に対応済み。
