# 計画書: いのちがけ（再レビュー）

## 仕様要約

* 使用者の現在HP分の固定ダメージを相手に与え、使用者はひんしになる（`docs/spec/moves/いのちがけ.md` 参照）。
* 命中判定・まもる・タイプ無効化で不発になった場合はHPを消費しない。
* みがわり／ばけのかわに肩代わりされた場合でもHPは消費する。

## 既存実装の問題点（今回のレビューで検出）

### 1. `flags={"fixed_damage"}` が未設定

`docs/data/move.py` の `ちきゅうなげ`・`いかりのまえば` など他の固定ダメージ技は
`flags={"fixed_damage"}` を設定しているが、`いのちがけ` には設定されていなかった。
このフラグは `じゃくてんほけん_boost_on_super_effective` (`handlers/item.py:952`) と
`ナゾのみ_heal_on_super_effective` (`handlers/item.py:1246`) が参照しており、
`is_super_effective` はタイプ相性のみで判定するため（威力0・固定ダメージであることを考慮しない）、
フラグがないと「かくとうタイプが弱点の相手」に対して誤って発動してしまう。

実機確認（修正前）:

```
カイリキー「いのちがけ」 → カビゴン（じゃくてんほけん所持）
結果: atk+2 spa+2、じゃくてんほけん消費（誤発動）
```

### 2. HP消費イベント `Event.ON_PAY_HP` のタイミングが命中判定より前

`docs/spec/turn.md` の `Event.ON_PAY_HP` 節には `いのちがけ` が掲載されているが、
`core/move_executor.py` の `_execute_move` を確認すると

```
ON_PAY_HP (379行目)
  ↓
命中判定 _check_hit (411行目)
  ↓
ON_BEFORE_APPLY_MOVE (418行目)
  ↓
_execute_hit → ON_MODIFY_MOVE_DAMAGE (450行目) → 相手へのダメージ適用 (452行目)
```

という順序になっており、`ON_PAY_HP` は命中判定より **前** に発火される。
`じばく`・`だいばくはつ`・`クロロブラスト`・`てっていこうせん`・`ビックリヘッド`・`ミストバースト` は
「外れても必ず自滅する」仕様（`docs/plan/moves/HPコスト05技.md`・`自爆技.md` 参照）なので
このタイミングで問題ないが、`いのちがけ` は一次情報に

> 攻撃が外れた場合はひんしにならない。

と明記されており、命中判定より後でHPを消費する必要がある。

実機確認（修正前、命中率0%で強制ミス）:

```
カイリキー「いのちがけ」（命中率0%） → カビゴン
結果: 技が外れたにもかかわらずカイリキーのHPが0になり、ひんしとして敗北扱いになった
```

## 修正方針

`Event.ON_PAY_HP` によるHP消費ハンドラ（`いのちがけ_pay_hp`）を廃止し、
既存の `Event.ON_MODIFY_MOVE_DAMAGE`（`いのちがけ_modify_damage`）1つに統合する。

* `ON_MODIFY_MOVE_DAMAGE` は命中判定・`ON_BEFORE_APPLY_MOVE`（みがわり無効化含む）を
  通過した後、かつ相手への実際のダメージ適用（`modify_hp(ctx.defender, ...)`）より前に発火するため、
  - 命中判定に外れた場合・まもる等で不発になった場合はハンドラ自体が呼ばれずHPを消費しない（修正2の解消）。
  - みがわり状態のときは `みがわり_block_damage` が同じ `ON_MODIFY_MOVE_DAMAGE` 内で
    ダメージの向き先を変えるだけで `ON_MODIFY_MOVE_DAMAGE` 自体は発火するため、
    使用者のHP消費は変わらず行われる（仕様どおり）。
  - 使用者のHP消費（自滅）が相手へのダメージ適用より先に行われるため、
    双方全滅となるケースで本プロジェクトの `judge_winner`（生存数0になった時点で即決着）が
    使用者側の敗北として判定される（一次情報の特別裁定と一致する結果になる）。
* `docs/spec/turn.md` の `Event.ON_PAY_HP` 節から `いのちがけ` の行を削除する
  （`ON_MODIFY_MOVE_DAMAGE` は同ファイル未掲載のイベントであり、
  `docs/plan/moves/攻撃技_反撃系.md` に「がむしゃら・ちきゅうなげ・いのちがけは
  デフォルト priority=100 を使用」とすでに根拠が記載されている。今回もこの既存の
  priority（デフォルト）をそのまま踏襲する）。

## ハンドラ構成（修正後）

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| いのちがけ | `Event.ON_MODIFY_MOVE_DAMAGE` | デフォルト（既存踏襲） | `attacker:self` | 現在HPを固定ダメージとして返しつつ、使用者のHPを0にする |

```python
def いのちがけ_modify_damage(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """いのちがけ: 現在HPを固定ダメージとして与え、使用者のHPを0にする（命中時のみ）。"""
    hp_cost = ctx.attacker.hp
    battle.modify_hp(ctx.attacker, v=-hp_cost, reason="self_cost", source=ctx.attacker)
    return HandlerReturn(value=hp_cost)
```

```python
"いのちがけ": MoveData(
    type="かくとう",
    category="special",
    pp=5,
    power=0,
    accuracy=100,
    flags={"fixed_damage"},
    handlers={
        Event.ON_MODIFY_MOVE_DAMAGE: h.MoveHandler(
            ha.いのちがけ_modify_damage,
            subject_spec="attacker:self",
        ),
    }
),
```

## 注意点・エッジケース

* HP消費に伴う `battle.modify_hp` の `reason` は既存どおり `"self_cost"` とし、
  `source=ctx.attacker` を明示する（他の自己犠牲技: `かかとおとし_crash` 等の慣例に合わせる。
  従来実装では `source` が未指定だったが、`ON_HP_CHANGED`/`ON_MODIFY_NON_MOVE_DAMAGE` を
  参照する既存ハンドラはいずれも `subject_spec="target:self"` で使用者自身をターゲットとして
  判定しているため、`source` の有無自体は現状の挙動に影響しない。今回は他の自己犠牲技との
  実装統一のために明示する）。
* ゴーストタイプへの無効化・まもるによるブロックは、いずれも `ON_MODIFY_MOVE_DAMAGE` より前の
  段階で完結するため、今回の変更による影響はない（修正前後で実機確認済み）。
