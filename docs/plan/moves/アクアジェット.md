# アクアジェット 実装計画書

## 仕様要約（docs/spec/moves/アクアジェット.md 参照）

* タイプ: みず / 分類: 物理 / 威力: 40 / 命中: 100 / PP: 20
* 対象: 相手1体 / 優先度: +1 / 接触あり
* 効果: 先制攻撃技。追加効果なし。

## 現状の実装状況

`src/jpoke/data/moves/move_a.py` に既に登録済み。

```python
"アクアジェット": MoveData(
    type="みず",
    category="physical",
    pp=20,
    power=40,
    accuracy=100,
    priority=1,
    flags={"contact"},
    handlers={},  # 追加効果なし
),
```

内容は仕様書と一致しており、修正不要。

## ハンドラ構成

追加効果を持たないため専用ハンドラは不要。`priority=1` は `MoveData.priority`
フィールドのみで完結し、行動順決定（`turn_controller.py`）が参照する。
`docs/spec/turn.md` には技の `priority` フィールド自体の記載はなく、
既存の同種先制技（でんこうせっか・かげうち・アクセルロック・マッハパンチ・バレットパンチ 等、
いずれも `priority=1`）と同様のパターンを採用している。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、イベントハンドラの priority 検討対象は存在しない。
`MoveData.priority=1` は技自体の行動順優先度であり、`turn.md` のイベントハンドラ priority とは別概念。

## 注意点・エッジケース（review-test 向け）

1. `flags={"contact"}` により直接攻撃技として扱われる（せいでんき等の接触特性の対象）。実装済み。
2. `target` を省略しているためデフォルトの `"foe"`（相手1体）が適用される → 仕様と一致。
3. `critical_rank` を省略しているためデフォルトの `0`（通常の急所判定）が適用される → 仕様と一致。
4. 追加効果がないため、まもる・おうじゃのしるし・オウムがえし等の一般的な技判定は
   フレームワーク側の共通処理（`handlers/move_attack.py` の基本ダメージ計算経路）で
   自動的に処理され、本技固有のハンドラ実装は不要。
5. みずタイプであるため、でんこうせっか（ノーマル）と異なりタイプ相性補正が付く
   （ほのお・じめん・いわに効果抜群、みず・くさ・ドラゴンに耐性）。既存の共通ダメージ計算
   ロジックで処理される。
6. 優先度+1の先制技のため、地面にいるポケモンへの使用時サイコフィールドが発動中に無効化される
   （`ON_TRY_MOVE_1` priority=100、フィールド側の共通処理で対応済み。専用実装不要）。
