## 仕様要約

- 使用前に自分の現在HPを全て消費する（ON_PAY_HP）
- HP消費後にHPが0になっても技は発動する（ON_PAY_HP はヒット処理より前に発火されるため）
- 技使用後に使用者はひんしになる
- `labels=["explosion"]` を保有する（しめりけによる失敗、その他爆発判定用）

| 技名 | タイプ | 分類 | PP | 威力 | 命中 | HPコスト |
|------|--------|------|-----|------|------|----------|
| じばく | ノーマル | 物理 | 5 | 200 | 100 | hp（全消費） |
| だいばくはつ | ノーマル | 物理 | 5 | 250 | 100 | hp（全消費） |
| ミストバースト | フェアリー | 特殊 | 5 | 100 | 100 | hp（全消費） |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_PAY_HP` テーブルは priority が `-`（未掲載）となっている。

```
## Event.ON_PAY_HP

| Priority | 処理 |
|----------|------|
| - | じばく等のダメージ・瀕死判定 |
| - | いのちがけ |
```

`じばく等` として明記されており、これが本実装の対象。priority は未指定（デフォルト値）で実装する。
既存の `いのちがけ_pay_hp` も priority 未指定で実装されており、それと同じパターンを踏襲する。

## ミストバーストのフィールド消費について

仕様書 `docs/spec/moves/_move_secondary_2.md` の範囲では HP変化のみ扱っている（ミストバーストの列は HPコスト比=1.0 のみ記載）。
ミストフィールドの消費については仕様書の対象外のため、本計画書では **HP全消費のみ実装する**。
ミストフィールド消費が必要であれば別途仕様書を参照して実装すること。

## 実装箇所

### handlers/move_attack.py に追加する関数

3技は全て同一ロジック（現在HP全消費）。五十音順の挿入位置: `じばく` → じ行、`だいばくはつ` → だ行、`ミストバースト` → み行。

```python
def じばく_pay_hp(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """じばく: 使用前に現在HPを全消費する。"""
    battle.modify_hp(ctx.attacker, v=-ctx.attacker.hp, reason="self_cost", source=ctx.attacker)
    return HandlerReturn(value=value)


def だいばくはつ_pay_hp(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """だいばくはつ: 使用前に現在HPを全消費する。"""
    battle.modify_hp(ctx.attacker, v=-ctx.attacker.hp, reason="self_cost", source=ctx.attacker)
    return HandlerReturn(value=value)


def ミストバースト_pay_hp(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ミストバースト: 使用前に現在HPを全消費する。"""
    battle.modify_hp(ctx.attacker, v=-ctx.attacker.hp, reason="self_cost", source=ctx.attacker)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース（review-test 向け）

- **技後にひんしになるか**: ON_PAY_HP でHP全消費するため、技使用後（ダメージ付与後のターン終了処理ではなく）技発動フロー内で即座にひんしになる。テストで使用者がひんし状態になることを確認すること。
- **HP0でも攻撃が発動するか**: ON_PAY_HP はヒット処理より前に発火される（`いのちがけ` と同一パターン）。HP消費後にHPが0でも攻撃は継続する。テストで確認すること。
- **しめりけによる失敗**: `labels=["explosion"]` を保有しているため、相手が「しめりけ」特性を持つ場合は ON_TRY_MOVE_1 で失敗する（しめりけ側のハンドラが処理する）。ON_PAY_HP ハンドラ側では対応不要。
- **ハンドラ側ガード不要**: ON_PAY_HP 発火時に使用者が生存していることはバトル側で保証されている（行動可能チェック済み）。
- **3技は同一パターン**: ロジックが同一のため、将来的には共通ヘルパー関数への統合も検討できるが、現状は五十音順の個別関数として実装する。
- **コスト計算**: `ctx.attacker.hp`（現在HP）を全消費する。`max_hp` ではなく現在HPを使用する点に注意（HPコスト05技の `max_hp // 2` とは異なる）。
- **ミストバーストのフィールド消費**: 本計画書の実装対象外。別途仕様確認が必要。
