# 計画書: ソウルビート

## 仕様要約

- ドラゴンタイプ / 変化 / 命中なし / PP5 / 対象: 自分
- 自分のHPを最大HPの1/3だけ減らす
- 自分のすべての能力ランク（こうげき・ぼうぎょ・とくこう・とくぼう・すばやさ）を1段階ずつ上げる
- ダンス技かつ音系の技（labels: dance, sound）
- HPが足りない場合は失敗（`ON_TRY_MOVE_1` priority 30: `ソウルビート: HPが足りない` でフレームワーク側が処理）
- 全能力がすでに最大の場合は失敗（`ON_TRY_MOVE_2` priority 130: `ソウルビート等: 全能力最大` でフレームワーク側が処理）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | HP1/3消費＋全能力+1段階 |

## Priority 根拠

`docs/spec/turn.md` の `ON_TRY_MOVE_1` priority 30 に `ソウルビート: HPが足りない`、`ON_TRY_MOVE_2` priority 130 に `ソウルビート等: 全能力最大` と明記されており、失敗チェックはフレームワーク側が処理する。ハンドラ本体は既存の `ON_STATUS_HIT` priority 100 のパターンに従う（`からをやぶる_modify_attacker_stats` のような複合ランク変化と HP 消費を組み合わせた形式）。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `スレッドトラップ_apply_volatile_to_attacker` の後、`せいちょう_modify_attacker_stats` の前（五十音順）。

```python
def ソウルビート_pay_hp_and_modify_attacker_stats(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """ソウルビートの効果: 最大HPの1/3を消費し、すべての能力を1段階ずつ上げる。"""
    mon = ctx.attacker
    battle.modify_hp(mon, r=-1/3)
    return modify_attacker_stats(battle, ctx, value, stats={"A": 1, "B": 1, "C": 1, "D": 1, "S": 1})
```

### data/move.py への更新

`"ソウルビート"` エントリのハンドラを追加する。pp の誤記（現状 100）は修正し 5 に直す。

    "ソウルビート": MoveData(

## 注意点・エッジケース

- **HPが足りない失敗判定**（最大HPの1/3より残HPが少ないとき）は `ON_TRY_MOVE_1` priority 30 でフレームワーク側が処理するため、ハンドラ内にガード不要
- **全能力最大での失敗判定**は `ON_TRY_MOVE_2` priority 130 でフレームワーク側が処理するため、ハンドラ内にガード不要
- HP減算は `battle.modify_hp(mon, r=-1/3)` を使用する。`Pokemon.hp` への直接代入は禁止
- 1/3 切り捨て・切り上げの実装はフレームワーク側の `modify_hp` の丸め規則に従う（別途確認が必要な場合は `modify_hp` の実装を参照）
- ダンス技（labels: dance）かつ音系の技（labels: sound）のため、おどりこ特性のポケモンへのコピーとみがわり貫通はフレームワーク側が処理する
- data/move.py の既存エントリで pp=100 と誤記されている。正しくは pp=5 なので修正が必要
