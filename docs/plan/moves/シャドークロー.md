# 計画書: シャドークロー

## 仕様要約（docs/spec/moves/シャドークロー.md 参照）

- ゴーストタイプ / 物理 / 威力70 / 命中100 / PP16（チャンピオンズ、`move_list.txt` で確認済み） /
  対象: 相手1体 / 優先度0 / 直接攻撃あり（contact）
- 急所ランク+1（追加効果なし）
- 切る技（slashフラグ）。特性きれあじで威力1.5倍。
- リーサル計算: 対象外（`docs/progress/move.md` の「リーサル実装」列が `n/a`）

## 既存実装との差分

既存実装（`data/moves/move_sa.py` の `"シャドークロー"` エントリ）には以下の不備がある。

1. **PPが本家基準の15のまま**
   チャンピオンズでは全技PPが8/12/16/20の4段階に統一されている
   （`docs/champions/changes_from_sv.md`）。`docs/champions/move_list.txt` に
   `シャドークロー	ゴースト	物理	70	100	16	接○	守○` と明記されており、PPは16が正。
   本家基準の値がそのまま残っている移行漏れのバグ。`pp=15` → `pp=16` に修正する。

2. **`flags` に `slash` が欠落**
   一次情報（Wiki「技の仕様」節）に「Championsでは切る技の1つ。特性きれあじのポケモンが使うと
   威力が1.5倍になる」と明記されている（本家SVでは切る技非対象だが、Championsでは対象という
   世代差がWikiに明記されている）。`flags={"contact"}` のみでは
   `きれあじ_modify_power`（`handlers/ability.py`）の `ctx.move.has_flag("slash")` 判定が
   通らず、きれあじの威力補正が発動しない。`flags={"contact", "slash"}` に修正する。

3. **`critical_rank=1` は既存実装済みで正しい**
   急所ランク+1の効果は `MoveData(critical_rank=1)` として既に正しく設定されており、
   `model/move.py` の `Move.critical_rank` プロパティ経由でダメージ計算・急所判定に
   直接反映される（イベントハンドラを介さないデータ属性のため、追加のハンドラ実装は不要）。

## ハンドラ構成

追加効果を持たない技のため、新規ハンドラの実装は不要。`critical_rank` はデータ属性のみで
完結する。

### data/moves/move_sa.py（修正）

```python
"シャドークロー": MoveData(
    type="ゴースト",
    category="physical",
    pp=16,
    power=70,
    accuracy=100,
    critical_rank=1,
    flags={"contact", "slash"},
    handlers={},  # 追加効果なし
),
```

## リーサル計算

対象外（`docs/progress/move.md` の「リーサル実装」列が `n/a`。追加効果を持たない通常の
物理攻撃技のためリーサル計算ハンドラは不要）。

## 注意点・エッジケース

- `slash` フラグ追加により、きれあじ特性のポケモンが使用した場合のみ別枠で威力1.5倍が乗る
  （`apply_fixed_modifier` による固定小数点乗算、`power_modifier` に加算的に反映）。
- `critical_rank=1` によって急所発生率が通常より1段階高くなる（第6世代以降の段階表: 0→1/24,
  1→1/8）。ダメージ倍率自体（急所時1.5倍）には影響しない。
