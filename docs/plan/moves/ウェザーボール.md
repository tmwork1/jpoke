# 計画書: ウェザーボール

## 仕様要約

- ノーマルタイプ・特殊・威力50・命中100・PP12（champions基準。Gen9本家は10）
- `bullet`（弾）ラベル（特性ぼうだんで無効化される。`ぼうだん_block_bullet` は既存の汎用ハンドラで対応済み）
- 使用時の天候に応じてタイプが変化し、威力が2倍になる

| 天候 | 変化後タイプ |
|------|------------|
| はれ・おおひでり | ほのお |
| あめ・おおあめ | みず |
| すなあらし | いわ |
| ゆき（あられ） | こおり |
| 天候なし・らんきりゅう | ノーマル（変化なし・威力も変化なし） |

- 天候が `らんきりゅう`（デルタストリーム）のときはタイプ・威力いずれも対応した変化はない
  （`docs/wiki/moves/ウェザーボール.html` 技の仕様節）。本プロジェクトはメガシンカ非対応のため
  実戦では到達しない状態だが、`WeatherName` に `らんきりゅう` が定義されているため防御的に対応する。
- 特性ノーてんき・エアロックを持つポケモンが場にいるときは天候が変化していてもノーマルタイプの
  まま、威力も増えない。`battle.weather` はこれらの特性による無効化を `ON_CHECK_WEATHER_ENABLED`
  経由で既に反映済み。
- 攻撃側がばんのうがさを持つ場合、はれ・あめ状態の影響を受けずノーマルタイプのまま
  （すなあらし・ゆきはタイプ変化・威力上昇の対象のまま）。`battle.weather_for(attacker)` が
  `ON_CHECK_WEATHER_IMMUNE` 経由でこれを反映する。
- そうでん状態の効果を受けた場合はどの天候でもでんきタイプになる（そうでん優先）。
  実機確認済み（後述）。
- ダイマックス技・Zワザのタイプ変化は未実装機能のため対象外。

## Priority 根拠

`ON_MODIFY_MOVE_TYPE` と `ON_CALC_POWER_MODIFIER` は `docs/spec/turn.md` に個別行の記載がない
（ダメージ計算内部のイベント）。

既存の類似ハンドラを根拠とする（`docs/plan/moves/だいちのはどう.md` と同様のパターン）：

- `テラバースト_modify_move_type`（`MoveHandler` デフォルト priority=100）
- `だいちのはどう_modify_move_type` / `だいちのはどう_power_modifier`（同 priority=100）

いずれも `MoveHandler` のデフォルト priority=100 を使用しているため、ウェザーボールも同様とする。

## 実装箇所

### `src/jpoke/handlers/move_attack.py`

```python
WEATHERBALL_TYPE_MAP = {
    "はれ": "ほのお",
    "おおひでり": "ほのお",
    "あめ": "みず",
    "おおあめ": "みず",
    "すなあらし": "いわ",
    "ゆき": "こおり",
}
"""ウェザーボール: 天候名からタイプへの対応表。

らんきりゅう（デルタストリーム）は対応する変化がないため含めない。
"""


def ウェザーボール_modify_move_type(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ウェザーボール: 天候に応じてタイプを変化させる。

    天候が有効な場合（エアロック・ノーてんき・ばんのうがさで無効化されていない）、
    天候ごとに対応するタイプに変換する。らんきりゅうの場合は変化しない。
    """
    weather = battle.weather_for(ctx.attacker)
    new_type = WEATHERBALL_TYPE_MAP.get(weather.name)
    if new_type is not None:
        value = new_type
    return HandlerReturn(value=value)


def ウェザーボール_power_modifier(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ウェザーボール: タイプが変化する天候のとき威力を2倍にする。

    らんきりゅうは対応するタイプ変化がないため威力も変化しない。
    """
    weather = battle.weather_for(ctx.attacker)
    if weather.name in WEATHERBALL_TYPE_MAP:
        value = apply_fixed_modifier(value, 8192)  # ×2倍
    return HandlerReturn(value=value)
```

### `src/jpoke/data/moves/move_a.py` の登録

## 注意点・エッジケース（review-test 向け）

### らんきりゅうでの威力据え置き（本レビューで修正したバグ）

- 修正前は `ウェザーボール_power_modifier` が `weather.name != ""` のみを条件にしていたため、
  `らんきりゅう`（デルタストリーム）中もタイプは変化しないのに威力だけ2倍になる不整合があった。
  `WEATHERBALL_TYPE_MAP` に含まれる天候のときのみ2倍にするよう修正した。
- 本プロジェクトはメガシンカ非対応のため、`らんきりゅう` が実戦で発生することはないが、
  `WeatherName` リテラルに存在し `てんきや`（フォルム同期）等でも扱われているため防御的に修正した。

### そうでん状態との排他

- そうでん状態は `ON_MODIFY_MOVE_TYPE` で「でんき」に上書きする
  （`data/volatile.py` の `"そうでん"` エントリ、`h.そうでん_move_type`）。
- 実機確認: はれ中に攻撃側へ `そうでん` を付与してウェザーボールを撃った場合、
  `battle.move_executor.move_type == "でんき"` となり、そうでんが優先されることを確認済み
  （ハンドラ登録順で move データが先に登録され、volatile ハンドラが後から追加されて
  上書きするため）。

### ばんのうがさ・ノーてんき・エアロック

- いずれも `battle.weather_for(ctx.attacker)` を経由することで自動的に反映される。
  個別のガード実装は不要（既存の `weather_for` の仕組みに乗るだけでよい）。

### 弾技判定

- `flags={"bullet"}` により特性ぼうだんを持つ相手に対して技自体が無効化される。
  この無効化は `ぼうだん_block_bullet`（`handlers/ability.py`）による汎用実装であり、
  ウェザーボール専用のテストは不要（`tests/abilities/test_ability__group.py` の
  ぼうだんグループテストで技全般をカバー済み）。

### ダイマックス・Zワザ

- 天候対応のダイマックスわざ・Zワザへのタイプ変化は未実装機能であり対象外。

### リーサル計算

- `docs/progress/move.md` の「リーサル実装」列は `n/a`。通常のダメージ技であり、
  タイプ・威力の変化は `ON_MODIFY_MOVE_TYPE` / `ON_CALC_POWER_MODIFIER` を通じて
  通常のダメージ計算経路に自然に反映されるため、専用のリーサルハンドラは不要。
