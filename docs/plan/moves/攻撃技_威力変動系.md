## 仕様要約

### エレキボール
- でんき / 特殊 / PP 10 / 命中 100 / ラベル: bullet
- 自分の素早さ / 相手の素早さ の比率（整数切り捨て）で威力が決まる
- 比率 = floor(自分の素早さ / 相手の素早さ)
  - 1倍未満（比率0）: 威力 40
  - 1以上2未満: 威力 60
  - 2以上3未満: 威力 80
  - 3以上4未満: 威力 120
  - 4以上: 威力 150
- 相手の素早さが 0 の場合: ゼロ除算防止のため威力 40
- トリックルーム状態でも威力は変わらない（素早さ実数値で比較）

### くさむすび・けたぐり
- くさむすび: くさ / 特殊 / PP 20 / contact / 命中 100
- けたぐり: かくとう / 物理 / PP 20 / contact / 命中 100
- 対象の体重で威力が決まる（共通）
  - 10kg 未満: 20
  - 10kg 以上 25kg 未満: 40
  - 25kg 以上 50kg 未満: 60
  - 50kg 以上 100kg 未満: 80
  - 100kg 以上 200kg 未満: 100
  - 200kg 以上: 120
- ダイマックスしている相手には無効（`ON_TRY_MOVE_1` priority=30 に明記）

### ハードプレス
- はがね / 物理 / PP 10 / contact / 命中 100
- 対象の残りHP / 最大HP の比率で威力が決まる
- 威力 = max(1, floor(100 × 現在HP / 最大HP))
  - HP が満タンなら威力 100、瀕死寸前なら威力 1

### ヒートスタンプ・ヘビーボンバー
- ヒートスタンプ: ほのお / 物理 / PP 10 / contact / 命中 100 / ラベル: minimize
- ヘビーボンバー: はがね / 物理 / PP 10 / contact / 命中 100 / ラベル: minimize
- 自分の体重 / 相手の体重 の比率で威力が決まる（共通）
  - 相手の体重が自分の 1/5 以下（比率 5以上）: 威力 120
  - 1/4 以下（比率 4以上5未満）: 威力 100
  - 1/3 以下（比率 3以上4未満）: 威力 80
  - 1/2 以下（比率 2以上3未満）: 威力 60
  - 1/2 超（比率 2未満）: 威力 40
- ちいさくなるを使用した相手には必中かつ威力 2 倍（`minimize` ラベル技の共通仕様）
- ダイマックスしている相手には無効（`ON_TRY_MOVE_1` priority=30 に明記）

### きしかいせい・じたばた
- きしかいせい: かくとう / 物理 / PP 15 / contact / 命中 100
- じたばた: ノーマル / 物理 / PP 15 / contact / 命中 100
- 自分の残りHP が少ないほど威力が増える（共通）
- 計算式: X = floor(残りHP × 48 / 最大HP)（第5世代以降）
  - X ≥ 33（HP 68.75%以上）: 威力 20
  - 17 ≤ X ≤ 32（35.42-68.74%）: 威力 40
  - 10 ≤ X ≤ 16（20.83-35.41%）: 威力 80
  - 5 ≤ X ≤ 9（10.42-20.82%）: 威力 100
  - 2 ≤ X ≤ 4（4.17-10.41%）: 威力 150
  - 0 ≤ X ≤ 1（4.16%以下）: 威力 200
- `docs/spec/威力変動技.md` の表は最大 150 と誤記されているが、ポケモンWikiの実装に基づき最大 200 が正しい（「20〜200」）

### しおふき・ふんか
- しおふき: みず / 特殊 / PP 5 / 基礎威力 150 / 命中 100
- ふんか: ほのお / 特殊 / PP 5 / 基礎威力 150 / 命中 100
- 自分のHP が少ないほど威力が下がる
- 威力 = max(1, floor(150 × 現在HP / 最大HP))
  - HP 満タンなら威力 150、HP 1 でほぼ 0（最低 1）

## 威力変動の実装方式

### ON_CALC_POWER_MODIFIER を使う方式

`damage_calculator._calc_final_power()` の計算式:
```
final_power = round_half_down(ctx.move.power * power_modifier / 4096)
```

`power=1` のプレースホルダを使い、`ON_CALC_POWER_MODIFIER` ハンドラで `value`（初期値 4096）を
`target_power * 4096` に差し替えることで最終威力を `target_power` にできる。

```python
# 例: 威力を target_power に設定する
value = target_power * 4096  # → final_power = 1 * (target_power * 4096) / 4096 = target_power
```

`しおふき`・`ふんか` は `power=150` を維持しつつ、HP 比率 `r = HP / max_HP` を使って:
```
modifier = floor(r * 4096)
final_power = round_half_down(150 * modifier / 4096) = round_half_down(150 * r)
```

### ダイマックス無効チェック

`けたぐり`・`くさむすび`・`ヒートスタンプ`・`ヘビーボンバー` の「ダイマックス相手に無効」は
`docs/spec/turn.md` の `ON_TRY_MOVE_1 priority=30` に `けたぐり等: 対象がダイマックス` として明記されている。
この判定はイベント発火側（`move_executor`）または専用ハンドラで実装する。

### handlers/move_attack.py への追加関数

```python
def エレキボール_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """エレキボール: 自分の素早さ / 相手の素早さ の比率で威力を決定する。

    比率 = floor(自分のすばやさ / 相手のすばやさ)
    比率が 0 または相手の素早さが 0 の場合は威力 40。
    0:40 / 1:60 / 2:80 / 3:120 / 4以上:150
    """
    atk_speed = ctx.attacker.stats["S"]
    def_speed = ctx.defender.stats["S"]
    if def_speed <= 0:
        ratio = 0
    else:
        ratio = atk_speed // def_speed
    if ratio >= 4:
        power = 150
    elif ratio == 3:
        power = 120
    elif ratio == 2:
        power = 80
    elif ratio == 1:
        power = 60
    else:
        power = 40
    return HandlerReturn(value=power * 4096)


def くさむすび_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """くさむすび: 対象の体重で威力を決定する。"""
    return HandlerReturn(value=_weight_to_power(ctx.defender.weight) * 4096)


def けたぐり_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """けたぐり: 対象の体重で威力を決定する。"""
    return HandlerReturn(value=_weight_to_power(ctx.defender.weight) * 4096)


def _weight_to_power(weight: float) -> int:
    """体重（kg）から威力を算出する共通関数。

    10kg未満: 20 / 25kg未満: 40 / 50kg未満: 60 / 100kg未満: 80 / 200kg未満: 100 / 以上: 120
    """
    if weight < 10:
        return 20
    elif weight < 25:
        return 40
    elif weight < 50:
        return 60
    elif weight < 100:
        return 80
    elif weight < 200:
        return 100
    else:
        return 120


def ハードプレス_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """ハードプレス: 対象の残りHP / 最大HP の比率で威力を決定する（最大100）。

    威力 = max(1, floor(100 × 現在HP / 最大HP))
    """
    power = max(1, int(100 * ctx.defender.hp / ctx.defender.max_hp))
    return HandlerReturn(value=power * 4096)


def ヒートスタンプ_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """ヒートスタンプ: 自分の体重 / 相手の体重 の比率で威力を決定する。"""
    return HandlerReturn(value=_weight_ratio_to_power(ctx.attacker.weight, ctx.defender.weight) * 4096)


def ヘビーボンバー_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """ヘビーボンバー: 自分の体重 / 相手の体重 の比率で威力を決定する。"""
    return HandlerReturn(value=_weight_ratio_to_power(ctx.attacker.weight, ctx.defender.weight) * 4096)


def _weight_ratio_to_power(attacker_weight: float, defender_weight: float) -> int:
    """体重比率（自分 / 相手）から威力を算出する共通関数。

    相手の体重が自分の 1/5 以下（比率 5以上）: 120
    1/4 以下（比率 4以上5未満）: 100
    1/3 以下（比率 3以上4未満）: 80
    1/2 以下（比率 2以上3未満）: 60
    1/2 超（比率 2未満）: 40
    """
    if defender_weight <= 0:
        return 40
    ratio = attacker_weight / defender_weight
    if ratio >= 5:
        return 120
    elif ratio >= 4:
        return 100
    elif ratio >= 3:
        return 80
    elif ratio >= 2:
        return 60
    else:
        return 40


def きしかいせい_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """きしかいせい: 自分の残りHPが少ないほど威力が高くなる。

    計算式: X = floor(残りHP × 48 / 最大HP)
    X ≥ 33 → 20 / 17-32 → 40 / 10-16 → 80 / 5-9 → 100 / 2-4 → 150 / 0-1 → 200
    """
    return HandlerReturn(value=_hp_low_to_power(ctx.attacker.hp, ctx.attacker.max_hp) * 4096)


def じたばた_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """じたばた: 自分の残りHPが少ないほど威力が高くなる（きしかいせいと同計算）。"""
    return HandlerReturn(value=_hp_low_to_power(ctx.attacker.hp, ctx.attacker.max_hp) * 4096)


def _hp_low_to_power(current_hp: int, max_hp: int) -> int:
    """HP割合（自HP低下依存）から威力を算出する共通関数（第5世代以降）。

    X = floor(残りHP × 48 / 最大HP)
    X ≥ 33 → 20 / 17-32 → 40 / 10-16 → 80 / 5-9 → 100 / 2-4 → 150 / 0-1 → 200
    """
    x = current_hp * 48 // max_hp
    if x >= 33:
        return 20
    elif x >= 17:
        return 40
    elif x >= 10:
        return 80
    elif x >= 5:
        return 100
    elif x >= 2:
        return 150
    else:
        return 200


def しおふき_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """しおふき: 自分のHP比率に比例して威力が決まる（最大150）。

    威力 = max(1, floor(150 × 現在HP / 最大HP))
    ON_CALC_POWER_MODIFIER では: modifier = floor(HP比率 × 4096)
    final_power = round_half_down(150 × modifier / 4096) = round_half_down(150 × HP比率)
    """
    modifier = int(ctx.attacker.hp * 4096 / ctx.attacker.max_hp)
    modifier = max(1, modifier)  # 威力が最低1になるよう保証
    return HandlerReturn(value=modifier)


def ふんか_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """ふんか: 自分のHP比率に比例して威力が決まる（最大150）。しおふきと同計算。"""
    modifier = int(ctx.attacker.hp * 4096 / ctx.attacker.max_hp)
    modifier = max(1, modifier)
    return HandlerReturn(value=modifier)
```

**注**: `_weight_to_power`、`_weight_ratio_to_power`、`_hp_low_to_power` はモジュールプライベートのヘルパー関数。
ハンドラ関数の五十音順ソートは `sort_handlers.py` が公開関数（日本語始まり）のみを対象にするため、
アンダースコア始まりのヘルパーは適切な位置（使用する関数の直前）に配置する。

## 注意点・エッジケース（review-test 向け）

### power=1 プレースホルダの扱い

`damage_calculator._calc_final_power()` の内部:
```python
power = ctx.move.power  # 1
power_modifier = self._events.emit(Event.ON_CALC_POWER_MODIFIER, ctx, 4096)
power = round_half_down(power * power_modifier / 4096)  # 1 * (target * 4096) / 4096 = target
```
`power=1` のとき `modifier = target_power * 4096` を返せば `final_power = target_power` が得られる。
テラスタル威力60底上げ補正は `power = max(power, 60)` として適用されるため、
エレキボール等でテラスタル一致かつ威力40以下のケースでは最低60に底上げされる（仕様通り）。

### しおふき・ふんかの計算精度

`power=150` の技に `modifier = floor(HP比率 × 4096)` を返すと:
```
final_power = round_half_down(150 × modifier / 4096)
```
HP 1/max_HP が非常に小さい場合に modifier=0 になるとダメージ計算が `power=0` → `not move.power` で0ダメージになる可能性がある。
`modifier = max(1, ...)` で最低値 1 を保証することで、`power=150` × `1/4096` ≈ 0 → `max(1, ...)` により最終威力 1 を担保する。

### エレキボールのトリックルーム

トリックルーム状態では行動順が逆転するが、エレキボールの威力は「素早さ実数値」の直接比較のため影響なし。
`ctx.attacker.stats["S"]` および `ctx.defender.stats["S"]` はランク補正後の実数値を参照する。

### エレキボールのゼロ除算

相手の素早さが 0（まひの特殊ケース等）の場合、`def_speed <= 0` で早期リターンし威力 40。
まひによる素早さ補正は `stats["S"]` に反映済みのため、stats 参照で自動的に対応される。

### ヒートスタンプ・ヘビーボンバーの minimize 判定

`labels=["minimize", "contact"]` が設定されている。`minimize` ラベル技の「ちいさくなる使用済み相手に必中・威力2倍」の判定は
既存の `minimize` ラベル処理（`ON_CALC_POWER_MODIFIER` 等）で実装済みか確認すること。
既存の `のしかかり` 等を参照して、minimize 倍化が `ON_CALC_POWER_MODIFIER` 経由で追加適用されるか確認する。

### けたぐり・くさむすびのダイマックス無効

`docs/spec/turn.md` の `ON_TRY_MOVE_1 priority=30` に:
```
30 | けたぐり等: 対象がダイマックス
```
と明記されている。この処理が既存コード（`move_executor` や `ON_TRY_MOVE_1` の汎用ハンドラ）で実装済みか確認すること。
未実装の場合は別途 `ON_TRY_MOVE_1 priority=30` のハンドラを追加する必要がある。

### Pokemon.weight の特性・アイテム反映

`Pokemon.weight` プロパティは `ライトメタル`（÷2）・`ヘヴィメタル`（×2）・`かるいし`（÷2）を自動反映する。
ハンドラ側で特別な処理は不要。

### きしかいせい・じたばたの威力テーブル

`docs/spec/威力変動技.md` の表は最大 150 と誤記されているが、ポケモンWiki（`docs/wiki/moves/きしかいせい.html`）の
実装テーブルに基づき X=0〜1（HP 4.16%以下）の威力は **200** が正しい。
計算式 `X = floor(残りHP × 48 / 最大HP)` を使い、X の閾値で分岐する。

### ハードプレスの floor 計算

`floor(100 × HP / max_HP)` の実装は `int(100 * ctx.defender.hp / ctx.defender.max_hp)` で代替可能。
Python の `int()` は切り捨てのため OK。最低値保証として `max(1, ...)` を適用する。

### ヒートスタンプ・ヘビーボンバーの体重比率計算

「相手の体重が自分の 1/5 以下」を「`attacker_weight / defender_weight ≥ 5`」として実装する。
浮動小数点比較のため、小数点誤差に注意（整数比較に変換して比較することも検討）。
