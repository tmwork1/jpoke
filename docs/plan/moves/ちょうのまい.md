# 実装計画: ちょうのまい

## 仕様サマリ

- タイプ: むし / 分類: 変化 / 威力: - / 命中率: - / PP: 20 / 対象: 自分
- 効果: 自分のとくこう・とくぼう・すばやさランクを 1 段階ずつ上げる
- ダンス技（`dance` フラグ）
- まもる: 防がれない（自分対象） / マジックコート: 反射不可（自分対象） / 直接攻撃: いいえ
- よこどり: 対象になる（○）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自分のとくこう・とくぼう・すばやさを 1 段階ずつ上げる |

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の複数能力上昇ハンドラ（`しょうりのまい_modify_attacker_stats`（A+1,B+1,S+1）相当、`つるぎのまい_modify_attacker_stats` など）が priority=100 デフォルトで実装されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: 五十音順で `ちいさくなる_*` の後、`ちょうはつ_*` の前（ちい → ちょう の順）

```python
def ちょうのまい_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ちょうのまいの効果: 自分のとくこう・とくぼう・すばやさを 1 段階ずつ上げる。"""
    return modify_attacker_stats(battle, ctx, value, stats={"spa": 1, "spd": 1, "spe": 1})
```

### data/moves/move_ta.py の更新

`ちょうのまい` の MoveData に `target="self"` と `handlers` を設定する。
（`target` を指定しないとデフォルトの `"foe"` になり、`is_blocked_by_protect` /
`is_reflectable` がまもる・マジックコートに誤反応してしまうため必須。）

```python
"ちょうのまい": MoveData(
    type="むし",
    category="status",
    pp=20,
    target="self",
    flags={"dance"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ちょうのまい_modify_attacker_stats,
        ),
    }
),
```

## 注意点・エッジケース

- とくこう・とくぼう・すばやさの 3 種類のステータスを同時に上げる。`modify_attacker_stats` の
  `stats` 引数に辞書で渡せばよい
- dance フラグがあるため `おどりこ` 特性のポケモンが同じ技を使う（この処理はフレームワーク側）
- ランク上限（+6）チェックはフレームワーク側で処理されるため、ハンドラ側のガードは不要
- 対象は自分のため命中率なし。テストは命中率固定不要
- 全能力が最大のときのフレームワーク側の失敗処理（`ON_TRY_MOVE_2` priority=130）との組み合わせは、1 つでも上げられる状態であれば成功する
- `target="self"` により `まもる` では防がれず、`マジックコート` でも跳ね返らない（`よこどり` の対象にはなる）

## リーサル計算

対象外（`docs/progress/move.md` の該当行は `n/a`）。ダメージを与えない自己バフ技のため
リーサル計算ハンドラは不要。
