# 実装計画: ちょうのまい

## 仕様サマリ

- タイプ: むし / 分類: 変化 / PP: 20 / 命中率: なし（自分対象）
- 効果: 自分のとくこう・とくぼう・すばやさランクを 1 段階ずつ上げる
- ダンス技（dance ラベル）
- まもる: × / マジックコート: × / 直接攻撃: ×
- 対象: 自分

## 現状の実装状況

`data/move.py` にエントリは存在するが `handlers` キーが未設定（ランク上昇が未実装）。

```python
"ちょうのまい": MoveData(

),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自分のとくこう・とくぼう・すばやさを 1 段階ずつ上げる |

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の複数能力上昇ハンドラ（`しょうりのまい_modify_attacker_stats`（A+1,B+1,S+1）相当、`ギアチェンジ_modify_attacker_stats`（A+1,S+2）など）が priority=100 デフォルトで実装されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: 五十音順で `ちいさくなる_*` の後、`ちょうはつ_*` の前（ちい → ちょう の順）

```python
def ちょうのまい_modify_attacker_stats(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """ちょうのまいの効果: 自分のとくこう・とくぼう・すばやさを 1 段階ずつ上げる。"""
    return modify_attacker_stats(battle, ctx, value, stats={"C": 1, "D": 1, "S": 1})
```

### data/move.py の更新

`ちょうのまい` の MoveData に `handlers` を追加する。

```python
"ちょうのまい": MoveData(
    type="むし",
    category="変化",
    pp=20,
    labels=["dance"],
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ちょうのまい_modify_attacker_stats,
        ),
    }
),
```

## 注意点・エッジケース

- C・D・S の 3 種類のステータスを同時に上げる。`modify_attacker_stats` の `stats` 引数に辞書で渡せばよい
- dance ラベルがあるため `おどりこ` 特性のポケモンが同じ技を使う（この処理はフレームワーク側）
- ランク上限（+6）チェックはフレームワーク側で処理されるため、ハンドラ側のガードは不要
- 対象は自分のため命中率なし。テストは命中率固定不要
- 全能力が最大のときのフレームワーク側の失敗処理（`ON_TRY_MOVE_2` priority=130）との組み合わせは、1 つでも上げられる状態であれば成功する
