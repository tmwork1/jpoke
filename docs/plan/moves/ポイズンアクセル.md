# ポイズンアクセル 実装計画書

## 仕様要約（docs/spec/moves/ポイズンアクセル.md 参照）

* タイプ: どく / 分類: 物理 / 威力: 100 / 命中: 100 / PP: 12（Champions基準、本家PP10からの変換）
* 対象: 相手1体 / 優先度: 0 / **接触なし**（物理技だが直接攻撃ではない）
* 効果: 追加効果として30%の確率で相手を「どく」状態にする。
* この技に対する アンコール・まねっこ は失敗する（`non_encore` / `non_copycat` フラグで表現）。
  ものまね・さいはいは本プロジェクト未実装のため対応フラグなし。

## 現状の実装状況（レビュー前）

`src/jpoke/data/moves/move_ha.py` に既に登録済みだったが、以下の誤りがあった
（`docs/plan/moves/攻撃技_どく系.md` に記載された、一次情報を確認せず書かれた
草案の値をそのまま実装に転記してしまったことが原因と見られる）。

```python
"ポイズンアクセル": MoveData(
    type="どく",
    category="physical",
    pp=20,                                                  # 誤り: 一次情報のPPは10であり、
                                                             #        Championsの4段階(8/12/16/20)では12
    power=70,                                                # 誤り: 一次情報の威力は100
    accuracy=100,
    flags={"contact", "non_copycat", "secondary_effect"},   # 誤り: "contact" は不要（非接触技）、
                                                             #        "non_encore" が不足
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ポイズンアクセル_apply_poison_to_defender,
        )
    }
),
```

どく付与ハンドラ自体（`ha.ポイズンアクセル_apply_poison_to_defender` =
`apply_ailment_to_defender(..., ailment="どく", chance=0.3)`）は
確率30%で仕様と一致しており、修正不要。

## 修正後の実装

```python
"ポイズンアクセル": MoveData(
    type="どく",
    category="physical",
    pp=12,
    power=100,
    accuracy=100,
    flags={"non_copycat", "non_encore", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ポイズンアクセル_apply_poison_to_defender,
        )
    }
),
```

## ハンドラ構成

| Event | 関数 | subject_spec | priority | 役割 |
|---|---|---|---|---|
| `Event.ON_DAMAGE_HIT` | `ポイズンアクセル_apply_poison_to_defender` | `attacker:self`（デフォルト） | 100（デフォルト） | 30%の確率で相手をどく状態にする |

`ON_DAMAGE_HIT` は既存の同種どく付与技（どくづき・どくばり・ヘドロこうげき 等）と
同一イベントであり、`docs/spec/turn.md` に個別の priority 記載がないダメージ命中後の
追加効果ハンドラ群と同様、`MoveHandler` のデフォルト priority（100）をそのまま使用する。
これは既存のどく付与ハンドラすべてが priority 未指定（デフォルト値）である前例を踏襲したもの。

## Priority 根拠

* `Event.ON_DAMAGE_HIT` は `docs/spec/turn.md` に個別の priority 記載がないイベントであり、
  CLAUDE.md の指針に従い既存の同種ハンドラ（`どくづき_apply_poison_to_defender` 等、
  すべて priority 未指定＝デフォルト100）を参照して同じ扱いとする。

## 実装箇所

### handlers/move_attack.py

修正不要（`ポイズンアクセル_apply_poison_to_defender` は既存実装のまま、
`apply_ailment_to_defender` 汎用ヘルパーを呼び出すのみ）。

### data/moves/move_ha.py

`"ポイズンアクセル"` エントリの `power` を `70` → `100` に、`pp` を `20` → `12` に、
`flags` から `"contact"` を削除し `"non_encore"` を追加する。

## 注意点・エッジケース（review-test 向け）

1. `flags` に `"contact"` を含めない → さめはだ・てつのトゲ等の接触特性・アイテムの
   トリガー対象外になることを確認する。
2. `flags` に `"non_encore"` を追加 → アンコールで固定できないことを確認する
   （`アンコール_can_apply` は `move.has_flag("non_encore")` を見て失敗させる）。
3. `flags` の `"non_copycat"` は既存実装済みで、まねっこ失敗テストも既に存在する
   （`tests/moves_status/test_move_ma.py` の
   `test_まねっこ_スターモービル専用技はコピーできない`、バーンアクセルを使用）→ 変更不要。
4. `power` を `100` に修正 → ダメージ計算テストで威力差異が反映されることを確認する。
5. PP を `12` に修正 → PP消費・わるあがき遷移等の一般テストは既存の共通ロジックで
   カバーされるため専用テスト不要（他のPP修正済み技と同様の扱い）。
6. `secondary_effect` フラグにより、ちからずく・てんのめぐみ等の特性判定対象になる
   （既存の共通ロジックで処理されるため専用ハンドラ不要）。
7. リーサル計算: 追加効果（どく付与）はリーサル計算のHP分布に影響しない一般的な
   確率追加効果のため、`docs/progress/move.md` の「リーサル実装」「リーサルテスト」は
   `n/a` とする。
