# 計画書: こうそくいどう

## 仕様要約

- 自分の「すばやさ」ランクが2段階上がる
- タイプ: エスパー、変化、PP: 20（champions 基準）、対象: 自分
- 純粋な自己ランク上昇技（プラスのみ、混在なし）

出典: `docs/spec/moves/こうそくいどう.md`

---

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100 (省略) | `attacker:self` (省略) | S+2 を適用 |

---

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`つるぎのまい_modify_attacker_stats` 等）がすべて priority=100 (デフォルト) で登録されているため、100 を使用する。

---

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: `くろいまなざし_apply` 〜 `グラスフィールド_activate_terrain` の後、`コスモパワー_modify_attacker_stats` の前（五十音順）

```python
def こうそくいどう_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"spe": 2})
```

### `src/jpoke/data/moves/move_ka.py`

`"こうそくいどう"` エントリに `handlers` を追加する:

```python
    "こうそくいどう": MoveData(
        type="エスパー",
        category="status",
        pp=20,
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.こうそくいどう_modify_attacker_stats,
            )
        }
    ),
```

---

## 注意点・エッジケース

- ランクがすでに+6の場合、`battle.modify_stats` が空の結果を返し技が失敗する（フレームワーク側で処理、ハンドラ側にガード不要）
- 追加効果・急所判定は対象外（変化技のため）
- リーサル計算: 対象外（ダメージを与えない変化技、n/a）
