# 実装計画書: ちょうおんぱ

## 仕様

- タイプ: ノーマル / カテゴリ: 変化 / PP: 20（Gen9本家基準。`docs/champions/move_list.txt` に
  エントリなしのため上書きなし） / 命中率: 55
- 効果: 相手を**こんらん**（揮発性状態）にする（2〜5ターン）
- 無効条件:
  - 特性「マイペース」 → `マイペース_prevent_volatile` が `ON_BEFORE_APPLY_VOLATILE` で自動処理
  - 「ミストフィールド」（接地ポケモン） → `ミストフィールド_prevent_confusion` が自動処理
  - すでにこんらん → `volatile_manager.apply` が重複付与を自動拒否

詳細は `docs/spec/moves/ちょうおんぱ.md` を参照。

## イベント・Priority

| イベント | Priority | subject_spec | 役割 |
|----------|----------|-------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（MoveHandler デフォルト） | こんらん付与 |

**根拠**: `ON_STATUS_HIT` は `turn.md` に独立エントリなし。`あやしいひかり`・`てんしのキッス` と同パターン
（priority=100 デフォルト）。

## 実装

### handlers/move_status.py

配置位置: `ちからをすいとる_can_apply` の直後、`ちょうのまい_modify_attacker_stats` の直前（五十音順）

```python
def ちょうおんぱ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ちょうおんぱの効果: 相手をこんらん状態にする。"""
    return HandlerReturn(value=battle.volatile_manager.apply_confusion(
        ctx.defender, source=ctx.attacker
    ))
```

`volatile_manager.apply_confusion` はカウント（2〜5のランダム）の抽選を内部で行うため、
`あやしいひかり`・`てんしのキッス` と同じヘルパーをそのまま利用する。

### data/moves/move_ta.py

既存の `"ちょうおんぱ"` エントリに `ON_STATUS_HIT` ハンドラを追加:

```python
"ちょうおんぱ": MoveData(
    type="ノーマル",
    category="status",
    pp=20,
    accuracy=55,
    flags={"sound"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ちょうおんぱ_apply,
        ),
    }
),
```

## 注意事項

- `VolatileName` の `"こんらん"` は登録済み（`docs/spec/volatiles/こんらん.md`）
- こんらん揮発性状態の実装（`こんらん_try_action` 等）も既存済み
- かがくへんかガス/かたやぶり無効化チェックは書かない
- 音技のみがわり貫通は `flags={"sound"}` によりコア側（`move_executor.check_hit_substitute`）で
  汎用的に処理されるため、個別ハンドラは不要
