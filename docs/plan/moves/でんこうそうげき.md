# でんこうそうげき 実装計画書

## 仕様要約（docs/spec/moves/でんこうそうげき.md 参照）

* タイプ: でんき / 分類: 物理 / 威力: 120 / 命中: 100 / PP: 5
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果:
  1. 自分が「でんき」タイプでないと失敗する。
  2. 命中してダメージを与えると、自分の「でんき」タイプを無くす
     （テラスタル中は消失処理を無視するため、`Pokemon.types` の実装済み仕様に委ねられる）。

## 現状の実装状況（修正前）

`src/jpoke/data/move.py` には登録済みだが、`handlers={}` となっており、
「でんきタイプでない場合の失敗」と「命中後のでんきタイプ消失」のどちらも未実装だった。

```python
"でんこうそうげき": MoveData(
    type="でんき",
    category="physical",
    pp=5,
    power=120,
    accuracy=100,
    flags={"contact"},
    handlers={},  # 追加効果なし  ← 誤り。実際は専用効果を持つ
),
```

`もえつきる`（ほのお・特殊版、同種の「タイプ依存の失敗 + 命中後タイプ消失」効果を持つ専用技）が
既に同じパターンで実装済みのため、そのハンドラをそのまま踏襲する
（`もえつきる` は `flags={"thaw"}` によるこおり解凍処理を追加で持つが、
`でんこうそうげき` にはこおり解凍要素はないため、その部分は不要）。

## ハンドラ構成

| Event | Priority | subject_spec | 役割 |
|---|---|---|---|
| `Event.ON_TRY_MOVE_1` | 10 | `attacker:self` | 自分が「でんき」タイプでない場合に技を失敗させる |
| `Event.ON_DAMAGE_HIT` | 180 | `attacker:self` | 命中後、自分の「でんき」タイプを `removed_types` に追加して消失させる |

## Priority 根拠

* `docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` 表に
  `10 | ほのおタイプでないもえつきるの失敗` と明記されており、
  同種の「使用者の現在タイプに依存した技の失敗」処理として同じ `priority=10` を採用する。
  （`turn.md` にも「でんきタイプでないでんこうそうげきの失敗」を追記する。）
* `docs/spec/turn.md` の `Event.ON_DAMAGE` 表に
  `180 | もえつきるによるタイプ消失` と明記されており、同じ `priority=180` を採用する。
  （`turn.md` にも でんこうそうげき を追記し、`もえつきる・でんこうそうげきによるタイプ消失` とする。）

## 実装箇所

### handlers/move_attack.py

`もえつきる_fail_if_no_fire_type` / `もえつきる_remove_fire_type` と同じパターンで、
でんきタイプ版の2関数を追加する（五十音順で `でんこうそうげき_*` として登録）。

```python
def でんこうそうげき_fail_if_no_electric_type(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """でんこうそうげき: でんきタイプを持たない場合に失敗させる。"""
    if not ctx.attacker.has_type("でんき"):
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="でんこうそうげき_でんきタイプなし")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def でんこうそうげき_remove_electric_type(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """でんこうそうげき: 命中後に自分のでんきタイプを除去する。

    removed_types に追加することで、交代するまででんきタイプを持たない状態になる。
    """
    mon = ctx.attacker
    if "でんき" not in mon.removed_types:
        mon.removed_types.append("でんき")
    return HandlerReturn(value=value)
```

### data/move.py

```python
"でんこうそうげき": MoveData(
    type="でんき",
    category="physical",
    pp=5,
    power=120,
    accuracy=100,
    flags={"contact"},
    handlers={
        Event.ON_TRY_MOVE_1: h.MoveHandler(
            ha.でんこうそうげき_fail_if_no_electric_type,
            subject_spec="attacker:self",
            priority=10,
        ),
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.でんこうそうげき_remove_electric_type,
            subject_spec="attacker:self",
            priority=180,
        ),
    }
),
```

## 注意点・エッジケース（review-test 向け）

1. `Pokemon.types` はテラスタル中 `removed_types` を無視する実装（`model/pokemon.py`）に
   なっているため、でんきタイプにテラスタルした状態で成功しても実際のタイプは消えない
   （仕様書「テラスタル中はでんきタイプが消えない」を自動的に満たす）。
2. `has_type("でんき")` はテラスタル込みの現在タイプで判定されるため、
   元々でんきタイプでないポケモンがでんきタイプにテラスタルしている場合は成功する
   （ものまね・まねっこ経由の使用も含む）。
3. まもる・タイプ相性による無効化・命中判定失敗時は `ON_DAMAGE_HIT` に到達しないため、
   でんきタイプは消えない（仕様と一致）。
4. `もえつきる` と異なりこおり状態との相互作用はないため、`thaw` 関連ハンドラは実装しない。
5. `ゆびをふる`（メトロノーム）は本プロジェクトで `handlers={}` の未実装状態のため、
   「ゆびをふるで選ばれない」除外仕様は対象外（`ゆびをふる` 自体が実装されたときに検討する）。
