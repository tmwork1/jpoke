# 実装計画書: まねっこ

## 仕様要約

- タイプ: ノーマル / カテゴリ: 変化 / PP: 20 / 対象: 不定（コピーする技に依存）
- その戦闘で最後に使われた技をそのままコピーして繰り出す。自分が出した技も対象になる。
- 使用技が無い場合（バトル開始以降、誰もPPを消費する行動をしていない場合）は失敗する。
- 以下の技はコピーできず、直前の技が該当する場合はまねっこ自体が失敗する。
  - Zワザ・Z変化わざ（本シミュレータではZワザ未実装のため対象外）
  - スターモービル専用技（バーンアクセル / ダークアクセル / ポイズンアクセル / ファイトアクセル / マジカルアクセル）
  - Wikiの詳細表に「×」と記載されている技（表内容が仕様書markdownに反映されておらず、詳細な全リストは未確定。第五世代以降の「PPを消費せずに終わった技はスキップする」仕様は `battle.last_used_move_name` の更新タイミング（`ON_TRY_MOVE_1` 通過後）で自然に満たされる）
- まねっこ自体のPPのみ消費し、コピーした技のPPは消費しない。

## 現状の実装状況

| 項目 | 状態 |
|------|------|
| `data/move.py` エントリ | 実装済み |
| `handlers/move_status.py` 関数 | 実装済み（まねっこ_can_use, まねっこ_execute） |
| `core/battle.py` | `last_used_move_name: str` を追加し、バトル全体で最後に使用された技名を保持する |
| `core/move_executor.py` | `_execute_move` 内、`ON_TRY_MOVE_1` 通過後・PP支払い前に `last_used_move_name` を更新する |

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_BEFORE_APPLY_MOVE` | 100（デフォルト） | `attacker:self`（デフォルト） | 失敗条件チェック（使用技なし / コピー不可技） |
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | `battle.last_used_move_name` から `Move` を生成し `battle.run_move` で実行する |

## Priority 根拠

- `ON_BEFORE_APPLY_MOVE` / `ON_STATUS_HIT`: `docs/spec/turn.md` に該当イベント内部の priority 表は存在しない。既存の同種ハンドラ（`いのちのしずく_heal`、`さいはい_can_apply` 等）がすべて priority 100（デフォルト）を使用しており、まねっこも同一パターンを採用する。

## `last_used_move_name` の更新ロジック

- `move_executor.py` の `_execute_move` で `ON_TRY_MOVE_2` 通過直後・`ON_PAY_HP` 発火前に、`ctx.move.has_flag("non_negoto")` が偽の技のみ `last_used_move_name` を更新する。
- `non_negoto` フラグは本来 ねごと の選択除外用フラグだが、対象技（ねごと、オウムがえし、さきどり、しぜんのちから、ねこのて、まねっこ、ゆびをふる、スケッチ、ものまね、がまん、さわぐ、きあいパンチ、くちばしキャノン、トラップシェル、ゲップ、ダイマックスほう、おいわい、てをつなぐ）は実際のポケモン対戦でも Copycat（まねっこ）の対象から外れるものが多く重なるため、流用している。
  - 重要な副次効果: まねっこ自身も `non_negoto` フラグを持つため、まねっこ使用時に `last_used_move_name` が「まねっこ」で上書きされることがない。これによりまねっこがまねっこ自身を無限にコピーする自己参照が構造的に防止される。
  - 既知の限界: `non_negoto` の技リストは「ねごとで選べない技」の実測に基づくものであり、まねっこの正式なコピー禁止技リスト（Wikiの詳細表）と完全一致するとは限らない。将来的に差異が判明した場合は専用の `non_copycat` 相当のフラグ運用に切り出すことを検討する。
- `ON_TRY_MOVE_1` で失敗する技（まひ・こおり・ねむり等でPPを消費せず行動が終わった場合、ほのおタイプでないもえつきるの失敗、でんきタイプでないでんこうそうげきの失敗、はいすいのじんの重複失敗等）は本更新処理に到達しないため、`last_used_move_name` は変化しない。これにより「PPが消費されずに終わった技はスキップし、その前の技を選ぶ」という第五世代以降の仕様を自然に満たす。
- `ON_TRY_MOVE_2` 通過後に失敗した技（まもる等の防御効果、命中判定失敗等）は既に `last_used_move_name` が更新された後に失敗するため、「PPを消費した上で失敗した技は基本的にその技がまねっこの対象となる」という仕様を満たす。

## スターモービル専用技の除外

- `MoveFlag` に `non_copycat` を追加し、バーンアクセル・ポイズンアクセル・マジカルアクセルに付与した（ダークアクセル・ファイトアクセルは本シミュレータ未実装のため対象外）。
- `まねっこ_can_use` で `Move(battle.last_used_move_name).has_flag("non_copycat")` を確認し、該当する場合は失敗させる。

## コピー実行時のPP消費

- `まねっこ_execute` は `Move(battle.last_used_move_name)` で新規の `Move` インスタンスを生成し、`battle.run_move(ctx.attacker, copied_move)` で実行する。
- この `copied_move` は使用者の実際の技スロット（`attacker.moves`）とは別のオブジェクトであり、生成直後はフルPPを持つ。`run_move` 内の `_consume_pp` はこの一時オブジェクトのPPのみを減らすため、使用者の実際の技のPPには一切影響しない。
- まねっこ自体のPPは、まねっこ自身が通常の技実行フロー（`run_move` → `_consume_pp`）で使用された時点で1消費される。

## 対象・命中判定

- コピーした技の対象・命中判定はコピー先の技自身の仕様に従う（`battle.run_move` が通常の技実行フローをそのまま流用するため）。シングルバトルのため、複数体からのランダム対象選択は考慮不要。
