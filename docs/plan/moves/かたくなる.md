# 実装計画: かたくなる

## 仕様サマリ

- タイプ: ノーマル / 分類: 変化 / PP: 30（Gen9本家準拠。champions/move_list.txt に本技自体が未収録のため） / 命中率: なし（自分対象）
- 効果: 自分の防御を 1 段階上げる
- まもる: × / マジックコート: × / 直接攻撃: ×
- 対象: 自分

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自分の防御を 1 段階上げる |

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の自分のランク上昇ハンドラ（`こうそくいどう_modify_attacker_stats`（S+2）、`かげぶんしん_boost_attacker_evasion`（EVA+1）など）がすべて priority=100 デフォルトで実装されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py への追加（実装済み）

挿入位置: 五十音順で `かげぶんしん_*` の後、`かなしばり_*` の前（かげ → かた → かな の順）

```python
def かたくなる_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """かたくなるの効果: 自分の防御を1段階上げる。"""
    return modify_attacker_stats(battle, ctx, value, stats={"def": 1})
```

### data/moves/move_ka.py の更新（実装済み）

`かたくなる` の MoveData に `target="self"` と `handlers` を設定する。

```python
"かたくなる": MoveData(
    type="ノーマル",
    category="status",
    pp=30,
    target="self",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.かたくなる_modify_attacker_stats,
        )
    }
),
```

## 注意点・エッジケース

- 対象は自分であるため `target="self"` を必ず設定する。デフォルトの `target="foe"` のままだと
  `Move.is_blocked_by_protect` / `Move.is_reflectable` が誤って True になり、相手の「まもる」で
  阻止されたり「マジックコート」で跳ね返されたりする不具合になる（レビュー時に発見・修正）。
- `modify_attacker_stats` を使うことで、`subject_spec="attacker:self"`（`MoveHandler` のデフォルト）
  により対象が正しく自分になる
- ランク上限（+6）チェックはフレームワーク側（`ON_TRY_MOVE_2` priority=130）で処理されるため、ハンドラ側のガードは不要
- 命中率なし（自分対象）のため accuracy 設定不要。テストは命中率固定不要
- PPは champions/move_list.txt に本技自体の収録がないため、Gen9本家の値（30）をそのまま採用する
  （詳細は `docs/spec/moves/かたくなる.md` の「PPについて」を参照）
