# 実装計画: ちいさくなる

## 仕様サマリ

- タイプ: ノーマル / 分類: 変化 / PP: 12（champions/move_list.txt準拠） / 命中率: なし（自分対象）
- 効果: 自分の回避率ランクを2段階上げ、自分に「ちいさくなる」揮発状態を付与する
- まもる: × / マジックコート: × / 直接攻撃: ×
- 対象: 自分

## 現状の実装状況（レビュー時点）

`data/moves/move_ta.py` にエントリと `handlers/move_status.py` の `ちいさくなる_apply` が既に実装されているが、以下2点にバグがある。

```python
"ちいさくなる": MoveData(
    type="ノーマル",
    category="status",
    pp=10,  # ← champions/move_list.txt では 12
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ちいさくなる_apply,
        )
    }
    # target 未指定（デフォルト "foe"）
    # → is_blocked_by_protect / is_reflectable が target=="foe" を条件にしているため、
    #   まもる・マジックコートに誤反応する
),
```

揮発状態側（`data/volatile.py` の `"ちいさくなる"` エントリ、`handlers/volatile.py` の
`ちいさくなる_guaranteed_hit` / `ちいさくなる_boost_power`）は仕様通りに実装済みで問題なし。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自分の回避率を2段階上げ、ちいさくなる状態を付与する |

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の同種ハンドラ（`かげぶんしん_boost_attacker_evasion` など自分対象ランク上昇技）が
priority=100 デフォルトで実装されているため、同様に採用する。

## 修正内容

### data/moves/move_ta.py の修正

- `pp` を `10` → `12` に修正（champions/move_list.txt準拠）
- `target="self"` を追加（まもる・マジックコートの誤反応を防止）

```python
"ちいさくなる": MoveData(
    type="ノーマル",
    category="status",
    pp=12,
    target="self",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ちいさくなる_apply,
        )
    }
),
```

### handlers/move_status.py

変更なし（`ちいさくなる_apply` は既に `ctx.attacker` の回避率ランクを2段階上げ、
`ちいさくなる` 揮発状態を付与しており仕様通り）。

## 注意点・エッジケース

- ダメージ2倍・必中化効果自体は揮発状態側（`ちいさくなる` VolatileData）の責務であり、
  本技のハンドラでは揮発状態を付与するのみ。対象技リストは `minimize` ムーブフラグで管理される。
- 対象は自分（自分対象技）のため、命中率なし。テストは命中率固定不要。
- ランク上限（+6）チェックはフレームワーク側で処理されるため、ハンドラ側のガードは不要。
