# 計画書: じゃどくのくさり

## 仕様要約

- どくタイプ / 特殊 / 威力100 / 命中100 / PP5 / 対象: 相手1体 / 直接攻撃なし
- 追加効果（50%）として、命中した相手を『もうどく』状態にする
- 詳細仕様は `docs/spec/moves/じゃどくのくさり.md` を参照

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | デフォルト | attacker:self（省略時デフォルト） | 相手にもうどく状態を付与（50%） |

## Priority 根拠

`ON_DAMAGE_HIT` は `docs/spec/turn.md` に未掲載のダメージ計算内部イベント（技命中・ダメージ適用後に発火し、
みがわりで防がれた場合は発火しない）。同種の追加効果ハンドラ（毒付与）である `どくづき` 系実装や
`じごくづき_apply_volatile_to_defender` に倣い、デフォルト priority を使用する。

## 実装箇所（実装済み）

### handlers/move_attack.py

```python
def じゃどくのくさり_apply_toxic_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="もうどく", chance=0.5)
```

### data/move.py

```python
    "じゃどくのくさり": MoveData(
        type="どく",
        category="special",
        pp=5,
        power=100,
        accuracy=100,
        flags={"secondary_effect"},
        handlers={
            Event.ON_DAMAGE_HIT: h.MoveHandler(

            )
        }

```

## 注意点・エッジケース

- `secondary_effect` フラグにより、ちからずく特性（威力1.3倍・追加効果無効化）、りんぷん特性・おんみつマント
  （追加効果無効化）は `apply_ailment_to_defender` 内の `resolve_secondary_chance` を通じて汎用的に処理される。
  ハンドラ側に個別のチェックは不要。
- 直接攻撃ではないため `flags` に `contact` を含めない。
- みがわり状態の相手に技が防がれた場合、`ON_MODIFY_MOVE_DAMAGE` でダメージが0になり `actual_damage <= 0` となるため
  `ON_DAMAGE_HIT` 自体が発火しない（`move_executor._execute_hit` の共通処理）。ハンドラ側にみがわりチェックは不要。
- リーサル計算は対象外（`docs/progress/move.md` でリーサル実装=n/a）。もうどく状態はそのターンのダメージ
  分布そのものには影響せず（HP分布は毎ターン`ON_TURN_END`等で既存の毒処理が反映される想定）、
  この技固有のリーサルハンドラは不要。

## 修正履歴（レビュー時に発見した誤り）

- **分類が `physical` になっていた**: 一次情報（ポケモンWiki）では分類は `特殊`。`category="physical"` →
  `category="special"` に修正。
- **PPが `10` になっていた**: 一次情報では PP は `5`。`pp=10` → `pp=5` に修正。
- **`flags` に `contact` が誤って含まれていた**: 一次情報では「直接攻撃: ×」。`flags={"contact", "secondary_effect"}` →
  `flags={"secondary_effect"}` に修正。
