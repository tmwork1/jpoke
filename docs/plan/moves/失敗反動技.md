# 計画書: 失敗反動技

## 仕様要約

「失敗反動（自己HP比）」が 0.5 の技。技が**外れた・失敗した**ときに、自分の最大HPの1/2のダメージを受ける。

- 対象技: かかとおとし / サンダーダイブ / とびげり / とびひざげり
- 発火タイミング: `Event.ON_MISS`（命中判定失敗時）
- ダメージ量: `max(1, attacker.max_hp // 2)`
- reason: 技によって異なる（下記「reason の選択」参照）
- subject_spec: `"attacker:self"`（ON_MISS は `AttackContext` を渡す）

**注記1（かかとおとしレビュー時に判明・修正済み: `docs/spec/moves/かかとおとし.md` / `docs/plan/moves/かかとおとし.md` 参照）**

- `recoil` フラグは `いしあたま`（`ctx.hp_change_reason == "recoil"` を判定）ではなく、
  `すてみ`（`ctx.move.has_flag("recoil")` を判定）にのみ影響する。両者は判定軸が異なる別概念。
  失敗反動を `reason="self_cost"`（または `"fixed_recoil"`）で実装している限り、`recoil` フラグの有無は
  いしあたま の可否に影響しない。
- wiki には「特性がすてみのポケモンが使用した場合、威力が1.2倍になる」という記載があり、
  失敗反動技グループは本来すべて `recoil` フラグを保持すべき（すてみの対象とするため）。
  かかとおとし・とびげり・とびひざげり は各レビューで `recoil` フラグを追加済み。

**注記2（とびひざげりレビュー時に判明・修正済み: `docs/spec/moves/とびひざげり.md` / `docs/plan/moves/とびひざげり.md` 参照）**

- とびひざげり の wiki には「特性がマジックガードのポケモンが使用したときは失敗しても反動ダメージを
  受けない」という明記がある。しかし `reason="self_cost"` は `マジックガード_ignore_damage`
  （`handlers/ability.py`）の除外リストに含まれており、マジックガードで**防げない**扱いになってしまう
  （バグ）。正しくは `reason="fixed_recoil"`（`てっていこうせん` 等が使う「マジックガードで防げるが
  いしあたまでは防げない確定反動」）を使うべきであり、とびひざげり は本レビューで
  `reason="self_cost"` → `reason="fixed_recoil"` に修正済み。`fixed_recoil` は `いしあたま`
  （`reason == "recoil"` のみ判定）では防げず、`recoil` フラグ（すてみ判定用）とも独立した別軸のため、
  注記1の `recoil` フラグ追加とは矛盾しない。
- かかとおとし・サンダーダイブ は現状 `reason="self_cost"` のままであり、同じ理由で
  マジックガードによる反動無効化が未実装のバグを抱えている。
  とびげり は本レビューで `reason="fixed_recoil"` に修正済み。
  残る2技を個別レビューする際は本注記を踏まえて `reason="fixed_recoil"` へ修正すること。

## Priority 根拠

`docs/spec/turn.md` に `Event.ON_MISS` の記載はない（イベント一覧に未掲載）。

既存の `ON_MISS` ハンドラである `からぶりほけん_boost_speed_on_miss`（`data/item.py`）は `ItemHandler` のデフォルト priority=100 で登録されている。また `MoveHandler` のデフォルト priority も 100 である。

**結論: priority=100 を使用する。根拠は MoveHandler デフォルト値と既存の ON_MISS ハンドラ（からぶりほけん）との整合性。**

## 実装箇所（とびひざげり分。他3技は各自の計画書・現状実装を参照）

### handlers/move_attack.py

```python
def とびひざげり_crash(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """とびひざげりが外れた場合の失敗反動ダメージ。自分の最大HPの1/2を受ける。

    マジックガードで防げるが、いしあたまでは防げない確定反動として扱う
    （reason="fixed_recoil"。"self_cost" だとマジックガードで防げなくなってしまうため区別する）。
    """
    battle.modify_hp(ctx.attacker, v=-max(1, ctx.attacker.max_hp // 2), reason="fixed_recoil", source=ctx.attacker)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース（review-test 向け）

### reason の選択

- `"recoil"` は反動ダメージ（与ダメージの割合）であり、**いしあたま**・**マジックガード**の両方で無効化される
- `"self_cost"` はみがわり・いのちがけ・クロロブラスト等の「意図的な自己HP消費」と同じ扱いで、
  いしあたま・マジックガードいずれでも防げない
- `"fixed_recoil"`（てっていこうせん等）は**マジックガードで防げるが、いしあたまでは防げない確定反動**。
  とびひざげり の失敗反動はこの区分に該当する（マジックガード無効化の wiki 記載があるため）。

### とびひざげり の flags 修正

- `flags={"contact", "gravity_restricted"}` は誤り（`recoil` フラグ欠落によりすてみの1.2倍補正が働かない）。
- 修正後: `flags={"contact", "gravity_restricted", "recoil"}`

### ON_MISS の発火条件

- `core/move_executor.py` の `_check_hit` が False を返したときのみ発火する
- 技の使用自体が失敗した場合（ON_TRY_MOVE_1/2 で中断）は ON_MISS は発火しない
- ハンドラ側での追加ガードは不要（move_executor が保証する）

### ひんし後のダメージ

- `battle.modify_hp` はひんし状態のポケモンへの適用をバトル側で管理する
- ハンドラ内で `if not ctx.attacker.alive` のガードは不要

### テスト観点（とびひざげり）

1. 通常命中時 → ON_MISS ハンドラが発火しないこと（ダメージなし）
2. 外れた時 → 使用者の HP が `max(1, max_hp // 2)` 減ること
3. いしあたまを持つポケモンが使用して外れた場合 → `fixed_recoil` なのでダメージを受けること
4. マジックガードを持つポケモンが使用して外れた場合 → `fixed_recoil` なのでダメージを受けないこと
5. すてみを持つポケモンが使用した場合 → `recoil` フラグにより威力が1.2倍になること
6. flags に `"recoil"` が含まれること（すてみ対応）
