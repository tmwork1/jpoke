# 実装計画書: アイアンヘッド（再レビュー）

## 仕様要約

`docs/spec/moves/アイアンヘッド.md` 参照。

* タイプ: はがね / 分類: 物理 / 威力: 80 / 命中率: 100 / PP: 16（Champions。SVまでは15） / 対象: 相手1体
* 直接攻撃。追加効果として、20%の確率で相手をひるませる（Champions仕様。本家シリーズでは30%）。

## 既存実装の確認結果と問題点

`src/jpoke/handlers/move_attack.py` の `アイアンヘッド_apply_flinch` が
`Event.ON_DAMAGE_HIT` で `apply_volatile_to_defender(battle, ctx, value, volatile="ひるみ", chance=0.2)`
を実行しており、`しねんのずつき_apply_flinch` 等の同種のひるみ追加効果を持つ技と同じ実装パターンになっている。
イベント・優先度（デフォルト100）も問題ない。chance=0.2 はChampions仕様の20%と一致しており正しい。

`src/jpoke/data/moves/move_a.py` の `MoveData` エントリ（`type="はがね"`, `category="physical"`,
`pp=16`, `power=80`, `accuracy=100`, `flags={"contact", "secondary_effect"}`）はいずれも
`docs/champions/moves.md` / 一次情報と一致しており問題ない。

以上の確認の結果、実装に誤りはなく修正不要だった。
