# 計画書: のみこむ

## 仕様要約

- ノーマルタイプ / 変化 / 命中なし / PP10 / 対象: 自分
- 技「たくわえる」を使った回数に応じてHPを回復する:
  - たくわえ1回: 最大HPの1/4
  - たくわえ2回: 最大HPの1/2
  - たくわえ3回: 全回復（最大HP）
- 使用後、「たくわえる」揮発性状態を削除する
- 使用後、たくわえた回数分だけぼうぎょ・とくぼうランクを逆補正して元に戻す
- たくわえた回数が0（揮発状態なし）の場合は失敗する

## 実装箇所

実装は `たくわえる` の計画書（`docs/plan/moves/たくわえる.md`）のセットとして実装済み。

### handlers/move_status.py（実装済みコード）

```python
def のみこむ_check_can_use(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """のみこむの使用条件チェック: たくわえた回数が0なら失敗する。"""
    mon = ctx.attacker
    count = (mon.volatiles["たくわえる"].count or 0) if mon.has_volatile("たくわえる") else 0
    if count == 0:
        battle.add_event_log(mon, LogCode.MOVE_FAILED,
                             payload={"reason": "のみこむ"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def のみこむ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """のみこむの効果: たくわえ回数に応じてHPを回復し、たくわえ状態をリセットする。"""
    mon = ctx.attacker
    count = mon.volatiles["たくわえる"].count or 0
    # 回復量: 1回=1/4, 2回=1/2, 3回=全回復
    if count >= 3:
        battle.modify_hp(mon, v=mon.max_hp - mon.hp)
    elif count == 2:
        battle.modify_hp(mon, r=1 / 2)
    else:
        battle.modify_hp(mon, r=1 / 4)
    # たくわえた回数分だけランクを戻す
    battle.modify_stats(mon, {"B": -count, "D": -count}, source=mon)
    # volatile 削除
    battle.volatile_manager.remove(mon, "たくわえる")
    return HandlerReturn(value=value)
```

### data/move.py（実装済みコード）

## 注意点・エッジケース（review-test 向け）

- **全回復の実装**: `count >= 3` の場合は `r=1.0` ではなく `v=mon.max_hp - mon.hp` を使って明示的に全量回復している。`modify_hp(r=1.0)` が最大HPを超えるかが実装依存なため
- **ランク逆補正とクリアボディ**: `battle.modify_stats(mon, {"B": -count, "D": -count}, source=mon)` は `source=mon`（自分源）であるが、クリアボディ・しろいきり等の「相手からの能力低下を防ぐ」効果は「自分が自分のランクを下げる」場合には適用されない。テスト `test_のみこむ_ランク逆補正がクリアボディに阻まれない` で確認済み
- **ランクが上昇していない場合の逆補正**: たくわえるを使用したとき、すでにぼうぎょ・とくぼうが+6でランクが実際には上昇しなかった場合も、のみこむは `count` 分だけ逆補正を試みる。ランクが-6以下に下がる場合は内部クランプされる
- **HP満タン時の回復**: `modify_hp` で HP が既に最大の場合は変化なし。失敗扱いにはならず（`ON_APPLY_MOVE` の「自己回復技: HP満タン」チェックが `ON_TRY_MOVE_1` より後に来るため、フレームワーク側で処理される）
- **かいふくふうじ状態**: labels に `"heal"` を付与しているため、かいふくふうじ状態では使用不可（`ON_CHECK_ACTION` 側でフレームワークが制限する）
- **バトンタッチとの関係**: 揮発状態「たくわえる」はバトンタッチで引き継がれないが、たくわえるで上昇したぼうぎょ・とくぼうランクはバトンタッチで引き継がれる
