# 計画書: のみこむ

## 仕様要約

- ノーマルタイプ / 変化 / 命中なし / PP12（champions基準。src/jpoke/data/ps-champ-ja/moves.json 参照）/ 対象: 自分
- 技「たくわえる」を使った回数に応じてHPを回復する:
  - たくわえ1回: 最大HPの1/4
  - たくわえ2回: 最大HPの1/2
  - たくわえ3回: 全回復（最大HP）
- 使用後、「たくわえる」揮発性状態を削除する
- 使用後、たくわえた回数分だけぼうぎょ・とくぼうランクを逆補正して元に戻す
- たくわえた回数が0（揮発状態なし）の場合は失敗する

## 実装箇所

実装は `たくわえる`（`docs/plan/moves/たくわえる.md`）・`はきだす`（`docs/plan/moves/はきだす.md`）と
連動する技セットとして実装済み。

### handlers/move_status.py（実装済みコード）

```python
def のみこむ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """のみこむの効果: たくわえ回数に応じてHPを回復し、たくわえ状態をリセットする。"""
    mon = ctx.attacker
    count = mon.volatiles["たくわえる"].count or 0
    # 回復量: 1回=1/4, 2回=1/2, 3回=全回復
    _heal_ratio: dict[int, float] = {1: 1 / 4, 2: 1 / 2, 3: 1.0}
    battle.modify_hp(mon, r=_heal_ratio.get(min(count, 3), 1 / 4))
    # たくわえた回数分だけランクを戻す
    battle.modify_stats(mon, {"def": -count, "spd": -count}, source=mon)
    # volatile 削除
    battle.volatile_manager.remove(mon, "たくわえる")
    return HandlerReturn(value=value)


def のみこむ_check_can_use(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """のみこむの使用条件チェック: たくわえた回数が0なら失敗する。"""
    mon = ctx.attacker
    if (
        not mon.has_volatile("たくわえる")
        or mon.volatiles["たくわえる"].count == 0
    ):
        battle.add_event_log(
            mon, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="のみこむ")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース（review-test 向け）

- **全回復の実装**: `count >= 3` の場合は `r=1.0` を渡している。`battle.modify_hp` は内部で
  `raw = int(max_hp * r)` を計算後 `target.modify_hp(v)` が HP を最大値でクランプするため、
  `r=1.0` でも最大HPを超えることなく全回復として機能する
- **ランク逆補正とクリアボディ**: `battle.modify_stats(mon, {"def": -count, "spd": -count}, source=mon)` は `source=mon`（自分源）であるが、クリアボディ・しろいきり等の「相手からの能力低下を防ぐ」効果は「自分が自分のランクを下げる」場合には適用されない。テスト `test_のみこむ_ランク逆補正がクリアボディに阻まれない` で確認済み
- **ランクが上昇していない場合の逆補正**: たくわえるを使用したとき、すでにぼうぎょ・とくぼうが+6でランクが実際には上昇しなかった場合も、のみこむは `count` 分だけ逆補正を試みる。ランクが-6以下に下がる場合は内部クランプされる
- **HP満タン時の回復**: `modify_hp` で HP が既に最大の場合は変化なし。失敗扱いにはならず（`ON_TRY_MOVE_1` のチェックは「たくわえ回数が0か」のみで、HP満タンかどうかは判定していない）
- **かいふくふうじ状態**: `flags` に `"heal"` を付与しているため、かいふくふうじ状態では使用不可（フレームワーク側で制限する）
- **バトンタッチとの関係**: 揮発状態「たくわえる」はバトンタッチで引き継がれないが、たくわえるで上昇したぼうぎょ・とくぼうランクはバトンタッチで引き継がれる
- **PP**: champions では全技のPPが8/12/16/20の4段階に統一されているため、Gen9本家の10ではなく12を採用する（`src/jpoke/data/ps-champ-ja/moves.json` 参照）
