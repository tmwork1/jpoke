# メガドレイン 実装計画書

参照: docs/spec/moves/メガドレイン.md

## 概要

特殊・くさタイプのドレイン技。相手に与えたダメージの50%だけ自分のHPを回復する。
既存のドレイン技（ギガドレイン・きゅうけつ・すいとる・ドレインパンチ等）と同一の共通処理
`_drain_hp`（`handlers/move_attack.py`）を再利用する。

## MoveData（data/moves/move_ma.py）

```python
"メガドレイン": MoveData(
    type="くさ",
    category="special",
    pp=15,
    power=40,
    accuracy=100,
    flags={"heal"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.メガドレイン_drain,
            priority=20,  # turn.md: ON_HIT priority 20 (HP吸収技による回復)
        )
    }
),
```

* PPは `docs/champions/move_list.txt` に記載がないため、Gen9本家のPP（15）をそのまま使用する。
* `flags` に `"heal"` を含める（HPを吸収する技であることを示す）。`"contact"` は含めない
  （直接攻撃ではないため）。
* `subject_spec` は `MoveHandler` のデフォルト `"attacker:self"` を使用（明示指定不要）。

## ハンドラ（handlers/move_attack.py）

```python
def メガドレイン_drain(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """メガドレインの回復量を計算する。"""
    _drain_hp(battle, ctx, value, heal_ratio=0.5)
    return HandlerReturn(value=value)
```

* 共通関数 `_drain_hp(battle, ctx, damage, heal_ratio)` を呼び出す。
  * `damage`（与えたダメージ）が0の場合（みがわりに命中した場合）は `ctx.substitute_damage` を使う。
  * `heal_amount = max(1, round_half_up(damage * heal_ratio))` で算出し、`Event.ON_CALC_DRAIN` を
    発火して `おおきなねっこ`（1.3倍）・`ヘドロえき`（回復→反動ダメージへの変換）を適用する。
  * `battle.modify_hp(ctx.attacker, v=heal_amount, reason="drain")` で攻撃者のHPを回復する。

## イベント・priority

* `Event.ON_HIT`（ダメージ発生後の処理）: priority=20
  * `docs/spec/turn.md` の `Event.ON_HIT` 表「20 | HP吸収技による回復」に対応。
  * 既存の全ドレイン技ハンドラ（ギガドレイン・きゅうけつ・すいとる・ドレインパンチ等）と同じ
    priority=20 で登録し、処理順の一貫性を保つ。
