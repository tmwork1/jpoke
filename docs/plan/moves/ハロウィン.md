# 実装計画書: ハロウィン

## 仕様要約

- タイプ: ゴースト / カテゴリ: 変化 / PP: 20 / 命中: 100 / 対象: 1 体選択
- 相手の「タイプ」に「ゴースト」タイプを追加する
- 相手がすでにゴーストタイプを持っている場合は失敗する

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_BEFORE_APPLY_MOVE` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手がすでにゴーストタイプなら失敗 |
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手の `data.types` にゴーストタイプを追加 |

## Priority 根拠

- `ON_BEFORE_APPLY_MOVE` priority 100: `docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「ハロウィン等: すでに該当タイプ保持」が priority 100 として明記されている
- `ON_STATUS_HIT` priority 100（デフォルト）: 同種ハンドラ（タールショット等）と同じパターン

## タイプ追加の実装方針

`Pokemon.types` プロパティは `self.data.types` を参照して返す（テラスタル・`ability_override_type` が未設定時）。タイプを永続的に追加するには `mon.data.types` リストに直接 append する方法を採用する。

- ただし `data.types` は図鑑データを共有しているため、**同じ種族のポケモンが複数いる場合に影響を受ける可能性がある**
- 代替案: `volatile` を付与して `ON_CALC_DEF_TYPE_MODIFIER` でゴーストタイプの弱点倍率を上乗せする（タールショットと同パターン）。この場合は交代でリセットされる

**推奨**: 交代でリセットされる volatile 方式で実装し、`ON_CALC_DEF_TYPE_MODIFIER` でゴーストタイプ追加効果を適用する。ただし volatile「ハロウィン」の定義と `ON_CALC_DEF_TYPE_MODIFIER` ハンドラを `data/volatile.py` および `handlers/volatile.py` に追加する必要がある。

**簡易案（直接変更）**: `mon.data.types` に append する方式。要調査：`PokemonData.types` が `list` であることを `data/models.py` で確認すること。

```python
def ハロウィン_can_apply(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """ハロウィンの使用条件チェック: 相手がすでにゴーストタイプなら失敗する。"""
    if ctx.defender.has_type("ゴースト"):
        battle.add_event_log(ctx.attacker, LogCode.MOVE_FAILED,
                             payload={"reason": "ハロウィン"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def ハロウィン_apply_volatile_to_defender(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """ハロウィンの効果: 相手にハロウィン状態を付与してゴーストタイプを追加する。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="ハロウィン")
```

## 注意点・エッジケース

- **タイプ追加の実装方法（volatile vs data.types 直接変更）は実装前に `data/models.py` の `PokemonData.types` の型と共有状態を確認すること**
- volatile 方式の場合、「ハロウィン」volatile を `data/volatile.py` に登録する必要がある
- テラスタル中の相手には `types` プロパティがテラスタルタイプのみを返すため、ゴーストタイプ追加の効果がどう反映されるか要確認
- 交代した場合に volatile はリセットされるため、ゴーストタイプの追加効果も消える（仕様確認要）
- もりののろいと同じパターンで実装可能
