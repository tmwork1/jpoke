# 計画書: りんしょう

## 仕様要約

* ノーマルタイプ / 特殊 / 威力60（可変） / 命中100 / PP15 / 対象: 1体選択 / 優先度0 /
  直接攻撃なし / 音技（sound）（`docs/spec/moves/りんしょう.md` 参照）。
* 同じターン中に自分・相手の両方がりんしょうを選択した場合、後で命中判定に進んだ方の
  威力が120になる（2倍固定）。
* シングルバトル専用スコープのため、行動順の繰り上げ効果は実装対象外
  （常に通常の優先度・すばやさ順と一致するため。詳細は仕様書参照）。

## 既存実装の状況

`data/move.py`（`data/moves/move_ra.py`）にエントリはあるが `handlers={}`（追加効果なし
扱い）となっており、本技の核心である「同ターン2体目使用時に威力120になる」効果が
全く実装されていない（バグ）。`docs/progress/move.md` では実装列が `x` になっているが、
実際には核心効果が欠落しているため修正が必要。テストも存在しない。

## 必要な状態管理

「このターン中にりんしょうが既に使われたか」はポケモン単位ではなくバトル全体で
1つ判定すればよい情報である（本プロジェクトはシングルバトルのみのため、場に出ている
2体のうちどちらか一方が既に使っていれば、もう一方が2体目になる）。

`Battle` に新規フィールドを追加し、「りんしょうが最後に使われたターン番号」を記録する。

### `core/battle.py`

```python
self.round_used_turn: int | None = None  # りんしょう: 直近に使われたターン番号
```

* `last_used_move_name` と同様、ターンをまたいでリセットする必要はない
  （絶対ターン番号を保持し、`battle.turn` と一致するかどうかだけで判定できるため）。
* プリミティブ型（`int | None`）のため `fast_copy` の追加のディープコピー対象指定は不要。

## ハンドラ構成

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| りんしょう | `Event.ON_BEGIN_MOVE` | デフォルト（100） | `attacker:self`（デフォルト） | 同ターン中に既に使われていれば威力を120にする。以後の使用に備えてターン番号を記録する |

`Event.ON_BEGIN_MOVE` は `docs/spec/turn.md` に priority の掲載がないため、既存の
かたやぶり系ハンドラ（`data/ability.py` の `かたやぶり_disable_foe_ability` 等）が
デフォルト値のまま登録されていることを根拠に、デフォルト設定のまま登録する。

`ON_BEGIN_MOVE` を選んだ理由: この時点は「行動成功判定（まひ・ねむり等）」の後、かつ
「発動成功判定・タイプ相性判定・まもる判定」より前に発火する。りんしょうは「先に使われた
方がまもる・タイプ相性などで無効化されても、後発の威力上昇は成立する」仕様のため、
無効化判定より前のこのタイミングで「使われたこと」を記録する必要がある。

`ON_CALC_POWER_MODIFIER` ではなく `ctx.move.power` を直接書き換える方式を採用する理由:
`ON_CALC_POWER_MODIFIER` は無効化判定を通過した技のみで発火するため、
「先に使われた技が無効化されても後発の威力は上がる」という仕様を再現できない。
`ON_BEGIN_MOVE` の時点で威力を確定させれば、無効化判定の結果に関係なく
「使われたこと」の記録と威力の書き換えを行える（`プレゼント_roll_outcome` 等、
既存の可変威力技が `ctx.move.power` を直接書き換える前例に倣う）。

```python
def りんしょう_apply_chain_power(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """りんしょう: 同じターン中に既に使われていれば威力を120にする。"""
    if battle.round_used_turn == battle.turn:
        ctx.move.power = 120
    battle.round_used_turn = battle.turn
    return HandlerReturn(value=value)
```

```python
"りんしょう": MoveData(
    type="ノーマル",
    category="special",
    pp=15,
    power=60,
    accuracy=100,
    flags={"sound"},
    handlers={
        Event.ON_BEGIN_MOVE: h.MoveHandler(
            ha.りんしょう_apply_chain_power,
        ),
    },
),
```

## テスト

`tests/moves_attack/test_move_ra.py` に以下を追加する。

* 自分だけがりんしょうを使用した場合 → 威力60のままダメージ計算されることを確認。
* 自分・相手の両方がりんしょうを選択し、相手が先に行動した場合 → 自分（2体目）の
  威力が120になることを確認。
* 前のターンにりんしょうが使われていても、次のターンでは威力が60に戻ることを確認
  （`battle.round_used_turn` がターン番号で判定されるため、ターンをまたいで威力が
  上がったままにならないことの確認）。

## 注意点・エッジケース

* シングルバトル専用のため、3体目以降のりんしょうによる威力上限（120で頭打ち）や
  ダブル/トリプルバトル・テラレイドバトル固有の行動順仕様は本プロジェクトの対象外。
* せんせいのツメ・クイックドロウ等の行動順変更効果とりんしょうの行動順繰り上げが
  競合するケースは実装しない（仕様書の「本プロジェクトでの実装方針」を参照）。
