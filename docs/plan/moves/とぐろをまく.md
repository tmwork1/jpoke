# 計画書: とぐろをまく

## 仕様要約

- 自分の「こうげき」「ぼうぎょ」「命中率（ACC）」ランクをそれぞれ1段階ずつ上げる
- タイプ: どく、変化、PP: 20、対象: 自分

出典: `docs/spec/moves/status_move.md` 行 158

---

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100 (省略) | `attacker:self` (省略) | こうげき・ぼうぎょ・命中率を各1段階上げる |

---

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`かたくなる_modify_attacker_stats` 等）がすべて priority=100 (デフォルト) で登録されているため、100 を使用する。

---

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: `トリックルーム_activate_global_field` の後（「と」行は「トリックルーム」の後で「ドわすれ」の前、「とぐろ」は「とける」より前のため「とぐろ」→「とける」の順）

```python
def とぐろをまく_modify_attacker_stats(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"A": 1, "B": 1, "ACC": 1})
```

### `src/jpoke/data/move.py`

`"とぐろをまく"` エントリ（行 5517）に `handlers` を追加する:

```python
    "とぐろをまく": MoveData(
        type="どく",
        category="変化",
        pp=20,
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.とぐろをまく_modify_attacker_stats,
            ),
        }
    ),
```

---

## 注意点・エッジケース

- A・B・ACC の3つを1つの `stats` dict で同時に渡せる（`modify_attacker_stats` は辞書をそのまま `battle.modify_stats` に転送する）
- いずれかのランクがすでに最大の場合は、そのランクの上昇のみ失敗し、他のランクは成功する（マネージャー内部処理）
- `turn.md` Priority 130「ランク補正上げ: すでに最大」でマネージャー内部処理されるため、ハンドラ側にガード不要
