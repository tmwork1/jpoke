# むしのていこう 実装計画書

## 仕様要約（docs/spec/moves/むしのていこう.md 参照）

* タイプ: むし / 分類: 特殊 / 威力: 50 / 命中: 100 / PP: 20
* 対象: 相手全員（シングルバトルでは相手1体） / 優先度: 0 / 非接触
* 効果: 100%の確率で相手の『とくこう』ランクを1段階下げる（追加効果）。

## 現状の実装状況

`src/jpoke/data/moves/move_ma.py` に登録済み（レビューにより `secondary_effect` フラグの欠落を修正）。

```python
"むしのていこう": MoveData(
    type="むし",
    category="special",
    pp=20,
    power=50,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.むしのていこう_lower_spa_C,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 100%の確率で相手の『とくこう』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def むしのていこう_lower_spa_C(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spa": -1})
```

`modify_defender_stats`（`handlers/move.py`）は `chance` 省略時に100%として扱い、
`battle.resolve_secondary_chance` を経由するため、ちからずくによる追加効果無効化（確率0化）は
汎用的に処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（ランク変化）は既存の同種ハンドラ（ひやみず・ムーンフォース 等）と
同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。`docs/spec/turn.md` にも
`ON_DAMAGE_HIT` は他の追加効果技と共通のタイミングとして扱われており、個別の priority 検討は不要。

追加・修正なし（実装済み）。

### data/moves/move_ma.py

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` が必要。ちからずく所持時は `resolve_secondary_chance` が
     確率を0にし、とくこうランク低下が発動しなくなる（威力1.3倍化は `ちからずく_boost` 側で処理）。
2. **テスト方針**
   - `battle.actives[1].rank["spa"] == -1` を確認する（100%発動のため乱数固定は不要）。
   - `MOVES["むしのていこう"].flags` に `secondary_effect` が含まれることを確認するテストを追加する
     （ひやみずのレビュー時に追加された `test_とびかかる_secondary_effectフラグを持つ` と同型）。

## リーサル計算

対象外（`docs/progress/move.md` の「リーサル実装」「リーサルテスト」は n/a）。
相手の『とくこう』ランクを下げる効果は次ターン以降の相手の攻撃力に影響するのみで、
本技自体のダメージによる HP 分布計算には影響しないため（ひやみず・ムーンフォース と同様）。
