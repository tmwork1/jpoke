# 計画書: つぼをつく

## 仕様要約

- ランダムで「こうげき」「ぼうぎょ」「とくこう」「とくぼう」「すばやさ」「回避率」「命中率」のいずれか1つのランクを2段階上げる
- タイプ: ノーマル、変化、PP: 30、対象: 自分（シングルでは自分のみ）
- 命中率なし（自分対象のため常に命中）
- 「ダイウォール」の効果も受けない（ダイウォール未実装のため現時点では追加対応不要）
- 7つの能力から一様ランダムに選択

出典: `docs/spec/moves/status_move.md` 行 140、`docs/spec/moves/_all_moves.md` 行 250

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 内での自己ランク変化技の記載なし。
既存の同種ハンドラ（`つるぎのまい_modify_attacker_stats`・`からをやぶる_modify_attacker_stats` 等）がすべて priority=100 (MoveHandler デフォルト) で登録されているため、100 を使用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: `つぶらなひとみ_modify_defender_stats` の後（つ〜て の間、五十音順）

```python
def つぼをつく_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """つぼをつく: 7種類の能力からランダムに1つ選んでランクを2段階上げる。"""
    stat = battle.random.choice(["A", "B", "C", "D", "S", "ACC", "EVA"])
    return HandlerReturn(value=battle.modify_stats(ctx.attacker, {stat: 2}, source=ctx.attacker))
```

## 注意点・エッジケース

- 選択肢は `["A", "B", "C", "D", "S", "ACC", "EVA"]` の7種類。スターのみ（`["A", "B", "C", "D", "S"]` の5種類）とは異なり、命中率・回避率も含む点に注意する
- ランクがすでに最大（+6）の能力が選ばれた場合の失敗は `StatusManager.modify_stats` 内で `STAT_CHANGE_BLOCKED` ログを出して処理されるため、ハンドラ側にガード不要
- `target="self"` の技のため、まもる・ダイウォールの影響を受けない（仕様上の「ダイウォール無視」フラグは現時点では未実装のため追加対応不要）
- シングルバトルのみ対象のため「自分か味方」の対象選択ロジックは不要。`target="self"` で実装する
- `modify_attacker_stats` ヘルパーは `chance` パラメータにより確率発動をサポートするが、つぼをつくは確率なし（常に発動）のためデフォルト引数不要。ただし `battle.modify_stats` を直接呼ぶことで `resolve_secondary_chance` の干渉を避けた実装とする
- よこどりの対象外（`docs/spec/moves/よこどり.md` 記載で第五世代以降はよこどり不可）
- さいはいの対象（`docs/spec/moves/さいはい.md` 記載で対象技。さいはいからの呼び出しでもランダム再選択が行われる）
