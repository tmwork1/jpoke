# 実装計画書: すなかけ

## 仕様

### 技データ

- タイプ: じめん
- カテゴリ: 変化
- PP: 15
- 命中率: 100
- 対象: 相手1体

### 効果

相手の **命中率** ランクを1段階下げる。

### 参照仕様書

- `docs/spec/moves/すなかけ.md`

---

## subject_spec と priority

### イベント: `ON_STATUS_HIT`

- **subject_spec**: `"attacker:self"`（`MoveHandler` のデフォルト値を使用）
- **priority**: 100

**根拠**: `docs/spec/turn.md` に `ON_STATUS_HIT` の独立エントリは無いが、同種の防御側ランクダウン
変化技ハンドラ（`あまえる_modify_defender_stats`, `こわいかお_modify_defender_stats` など）が
いずれも `h.MoveHandler(...)` の priority デフォルト値（100）で登録されているため、これと同一
パターンとして priority=100 を採用する。

## ハンドラ構成

### 1. `src/jpoke/handlers/move_status.py`

共通ユーティリティ `handlers/move.py` の `modify_defender_stats` を利用する（既存実装済み）。

```python
def すなかけ_modify_defender_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"accuracy": -1})
```

五十音順の挿入位置: `すなあらし_activate_weather` の後、`スピードスワップ_swap_speed` の前。

### 2. `src/jpoke/data/moves/move_sa.py`

```python
"すなかけ": MoveData(
    type="じめん",
    category="status",
    pp=15,
    accuracy=100,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.すなかけ_modify_defender_stats,
        )
    }
),
```

## 注意事項

- `modify_defender_stats` は `battle.modify_stats(ctx.defender, stats, source=ctx.attacker)` を
  呼び出す共通関数であり、まもる・マジックコート・特性（するどいめ・クリアボディ等）による
  無効化などは既存の `battle.modify_stats` / まもる判定 / `ON_STATUS_HIT` 発火経路側で
  共通処理されるため、本ハンドラ側で個別対応は不要
- 確率発動ではない（`chance` 引数はデフォルト1のまま呼び出す）ため、`resolve_secondary_chance`
  は常に1を返し無条件で命中率ランクダウンする
- `docs/champions/move_list.txt` に `すなかけ` の記載は無い（同リストは全技を網羅していない）ため、
  PPは第9世代（スカーレット・バイオレット）基準の15をそのまま採用する
