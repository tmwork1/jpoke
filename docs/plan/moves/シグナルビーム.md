# シグナルビーム 実装計画書

## 仕様要約（docs/spec/moves/シグナルビーム.md 参照）

* タイプ: むし / 分類: 特殊 / 威力: 75 / 命中: 100 / PP: 15
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果: 追加効果として、10%の確率で相手を『こんらん』状態にする。

## 現状の実装状況

`src/jpoke/data/moves/move_sa.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。
以下のとおり仕様と一致していることを確認した（修正不要）。

```python
"シグナルビーム": MoveData(
    type="むし",
    category="special",
    pp=15,
    power=75,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.シグナルビーム_apply_confusion_to_defender,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手を『こんらん』状態にする |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def シグナルビーム_apply_confusion_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """シグナルビームの追加効果: 10%の確率で相手をこんらん状態にする。"""
    return apply_confusion_to_defender(battle, ctx, value, chance=0.1)
```

`apply_confusion_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を
経由するため、ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は汎用ハンドラ側で
処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（こんらん付与）は既存の同種ハンドラ（みずのはどう・ねんりき・
おしゃべり 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。個別の
priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・こんらん付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
2. **おうじゃのしるし・するどいキバとの相互作用**
   - 本技はもともとひるみの追加効果を持たない（こんらん付与のみ）ため、`handlers/item.py` の
     `_INNATE_FLINCH_MOVES` にも含まれておらず、おうじゃのしるし等の追加ひるみ効果と正しく
     重複しうる（仕様通り、修正不要）。
3. **テスト方針**
   - `secondary_chance` を1に固定し、こんらんの付与有無を確認するテストが既存
     （`test_シグナルビーム_こんらんが発動する`）。
