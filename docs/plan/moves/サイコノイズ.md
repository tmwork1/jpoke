# 計画書: サイコノイズ

## 仕様要約

- エスパータイプ / 特殊 / 威力75 / 命中100 / PP12（Champions基準） / 対象: 相手1体 / 直接攻撃なし / 音の技
- 追加効果（100%）として、命中した相手を『かいふくふうじ』揮発性状態（2ターン）にする
- かいふくふうじ状態の詳細（HP回復無効化・解除条件など）は揮発性状態側の実装
  （`docs/spec/volatiles/かいふくふうじ.md`、`data/volatile.py`、`handlers/volatile.py`。実装済み）に委ねる
- 詳細仕様は `docs/spec/moves/サイコノイズ.md` を参照

## ハンドラ構成（実戦系）

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | デフォルト | attacker:self（省略時デフォルト） | 相手にかいふくふうじ状態（count=2）を付与 |

### Priority 根拠

`ON_DAMAGE_HIT` は技命中・ダメージ適用後に発火するイベント（みがわりに阻まれた場合は発火しない）。
同種の追加効果ハンドラである `じごくづき_apply_volatile_to_defender`・`しおづけ_apply_volatile_to_defender`
（いずれも `ON_DAMAGE_HIT`、priority省略でデフォルト値を使用）に倣い、デフォルト priority を使用する。
サイコノイズは `sound` フラグを持つため `MoveExecutor.check_hit_substitute` がみがわりを貫通する扱いになり、
みがわりが存在してもダメージは本体に通り `ON_DAMAGE_HIT` は通常どおり発火する
（じごくづきのようなみがわりブロックは発生しない）。

## 実装箇所

### handlers/move_attack.py

```python
def サイコノイズ_apply_volatile_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """サイコノイズの追加効果: 100%の確率で、相手を2ターンの間『かいふくふうじ』状態にする。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="かいふくふうじ", count=2)
```

### data/moves/move_sa.py

```python
    "サイコノイズ": MoveData(
        type="エスパー",
        category="special",
        pp=12,
        power=75,
        accuracy=100,
        flags={"sound", "secondary_effect"},
        handlers={
            Event.ON_DAMAGE_HIT: h.MoveHandler(
                ha.サイコノイズ_apply_volatile_to_defender,
            )
        },
        lethal_handlers={
            LethalEvent.ON_HIT: LethalHandler(l.サイコノイズ_apply_volatile)
        }
    ),
```

## 注意点・エッジケース

- `secondary_effect` フラグにより、ちからずく特性（威力1.3倍・追加効果無効化）、りんぷん特性・おんみつマント
  （追加効果無効化）、アロマベール特性（かいふくふうじ状態の付与阻止）は `apply_volatile_to_defender` 内の
  `resolve_secondary_chance` および揮発状態付与の汎用処理を通じて処理される。ハンドラ側に個別のチェックは不要。
- `sound` フラグにより `check_hit_substitute` が常に False を返すため、対象がみがわり状態でも本体にダメージが通り、
  かいふくふうじ状態も本体に付与される（貫通する）。ハンドラ側にみがわりチェックは不要。
- まもるで防がれた場合は `ON_MODIFY_MOVE_DAMAGE` でダメージが0になり `ON_DAMAGE_HIT` 自体が発火しないため、
  かいふくふうじ状態は付与されない。ハンドラ側に個別チェックは不要。

## リーサル計算

`docs/progress/move.md` のサイコノイズ行はリーサル実装列が「-」（未実装・要実装）だった
（じごくづき・ちょうはつ・いちゃもんなど「行動制限のみ」の揮発状態は n/a だが、
かいふくふうじは対象のHP回復そのものを封じるため、リーサル計算のHP分布に実際に影響する）。

### handlers/lethal.py

`LethalEvent.ON_HIT` ハンドラでバインド（`_apply_bind`）・しおづけ（`しおづけ_apply_volatile`）と同じパターンに倣い、
`battle.volatile_manager.apply` を経由せず `ctx.defender.volatiles` へ直接 `Volatile` を設定する
（ログ出力等の実戦系副作用を避けるため）。

```python
def サイコノイズ_apply_volatile(battle: Battle, ctx: LethalContext, hp_dist: StateDist) -> StateDist:
    """サイコノイズ: 命中後、追加効果有効時に相手にかいふくふうじ状態（2ターン）を付与する。"""
    if ctx.move_secondary and "かいふくふうじ" not in ctx.defender.volatiles:
        from jpoke.model.volatile import Volatile
        ctx.defender.volatiles["かいふくふうじ"] = Volatile("かいふくふうじ", count=2)
    return hp_dist
```

かいふくふうじの実際の効果（回復無効化）は、既存の回復系リーサルハンドラが共通で使う `_heal` /
`_heal_at_pinch`（`handlers/lethal.py`）の先頭で `target.volatiles` にかいふくふうじが含まれる場合に
即座に `hp_dist` をそのまま返すよう変更することで、たべのこし・オボンのみ・あめうけざら・グラスフィールド・
ポイズンヒール等、既存の全回復ハンドラに一括で反映されるようにした（個別ハンドラの書き換えは不要）。

- リーサル計算エンジンはターン経過に応じた揮発状態のカウントダウン・解除を行わない
  （既存のバインドも同様に `_apply_bind` で付与後カウントを減算しない簡略化がされている）ため、
  一度付与した「かいふくふうじ」はシミュレーション区間内で解除されない前提とする
  （実戦仕様の2ターン制限は厳密には再現しないが、既存アーキテクチャの簡略化方針に合わせた）。
- `いたみわけ`（`pain_split`）・`さいせいりょく`（`bench_heal`）はどちらもリーサル計算未実装のため、
  かいふくふうじの対象外扱い（`_heal`/`_heal_at_pinch` を経由しない）は現状影響しない。

## テスト方針

- 通常命中でかいふくふうじ状態（count=2）が付与されることを確認する
- ちからずく使用時は威力が上がるがかいふくふうじ状態にできないことを確認する
- みがわり状態の相手にも貫通してかいふくふうじ状態を付与できることを確認する（`sound` フラグの効果）
- まもるで防がれた場合はかいふくふうじ状態にならないことを確認する
- かいふくふうじ状態中は回復技（じこさいせい等）が無効になることを確認する（かいふくふうじ側の既存挙動の再確認）
- `t.calc_lethal` を用いて、サイコノイズ命中後にたべのこし等の回復がリーサル計算上ブロックされることを確認する

## 修正履歴（レビュー時に発見した誤り）

- **`handlers` が空だった**: 命中後にかいふくふうじ状態を一切付与していなかった（未実装のバグ）。
  `ON_DAMAGE_HIT` に `サイコノイズ_apply_volatile_to_defender` を追加。
- **`flags` に `secondary_effect` が欠落していた**: ちからずく特性の威力1.3倍・追加効果無効化が機能していなかった。
  `flags={"sound"}` → `flags={"sound", "secondary_effect"}` に修正。
- **PP が Gen9本家基準の10のままだった**: `docs/champions/move_list.txt` の正しい値は12。`pp=10` → `pp=12` に修正。
- **リーサル計算が未実装だった**: `lethal_handlers` が空で、かいふくふうじによる回復無効化がリーサル計算に
  反映されていなかった。`ON_HIT` にかいふくふうじ付与ハンドラを追加し、`_heal`/`_heal_at_pinch` を
  かいふくふうじ状態をチェックするよう修正。
