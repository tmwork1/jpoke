# 実装計画書: みねうち

参照仕様書: docs/spec/moves/みねうち.md

## 実装方針

みねうちは「相手のHPを1残すよう補正する」効果を持つ攻撃技であり、`Event.ON_MODIFY_MOVE_DAMAGE`
のダメージ補正ハンドラのみで挙動が完結する。

* `type="ノーマル"`
* `category="physical"`
* `pp=40`
* `power=40`
* `accuracy=100`
* `flags={"contact"}`（直接攻撃）

### ハンドラ: `みねうち_modify_damage`

* イベント: `Event.ON_MODIFY_MOVE_DAMAGE`
* `subject_spec`: 省略時のデフォルト `"attacker:self"`（`MoveHandler`のデフォルト）のままでよい。
  みねうちは自分の技の効果として発動するため、defenderの状態を参照しつつattacker側に紐づく。
* `priority=60`
  * 根拠: `docs/spec/turn.md` の `Event.ON_MODIFY_DAMAGE` 表に「60 | みねうち・てかげん」と明記されている。
  * がんじょう・きあいのタスキ・きあいのハチマキ（いずれも`priority=100`）より先に発動する必要がある。
    先にダメージを「相手の現在HP − 1」へ補正しておくことで、これらのハンドラが参照する
    `value >= mon.hp` 判定が満たされなくなり、誤った発動・消費を防げる。
  * 検証: 修正前は`priority`未指定（デフォルト100）だったため、がんじょう/きあいのタスキと
    同priorityとなり、素早さ比較（`_sort_handlers`）で防御側の方が速いと防御側のハンドラが
    先に実行され、HPが満タンの相手にみねうちを撃った際にきあいのタスキが誤って消費される
    バグを実際に確認した（修正後は解消）。
* ロジック（既存実装のまま、変更なし）:
  * `ctx.defender.hp <= 1` の場合: 実際のダメージを0にする。ただし「攻撃を無効化した」扱いとは
    区別するため、`ctx.defender.hits_taken += 1` を手動でインクリメントする
    （きあいパンチの不発判定等、`hits_taken`を参照する既存ロジックのため）。
  * それ以外の場合: `value` を `min(value, ctx.defender.hp - 1)` に補正する。

### 既知の未対応スコープ

* 「すでにHPが1の相手に使用したとき、ダメージを受けたときに発動する特性（せいでんき等）が発動する」
  効果は本計画では対応しない。理由は仕様書の「既知の未実装範囲」節を参照。
  `Event.ON_DAMAGE_HIT`は`actual_damage <= 0`のとき発火しないため、この効果を実現するには
  `core/move_executor.py`のダメージ処理フロー自体の変更が必要であり、みねうち単体の実装
  スコープを超えるため見送る。

## 登録

`src/jpoke/data/moves/move_ma.py` の `MOVES_MA["みねうち"]` に上記の値を設定する。
`Event.ON_MODIFY_MOVE_DAMAGE`ハンドラに`priority=60`を追加する。

## テスト方針

`tests/moves_attack/test_move_ma.py` に以下を追加・維持する。

* 相手のHPがすでに1のとき、ダメージが0になり倒さないことを確認するテスト（既存）
* 大ダメージを受けても必ずHP1を残すことを確認するテスト（既存）
* 追加: HPが満タンの相手が持ち物きあいのタスキを持つ場合、みねうちで満タンから瀕死級の
  ダメージを与えてもタスキが消費されず、通常通りHP1で止まることを確認するテスト
  （防御側が攻撃側より素早い場合を想定し、priority指定がないと壊れることを検証する）
