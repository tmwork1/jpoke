# 実装計画書: はやてがえし（再レビュー）

## 仕様要約

`docs/spec/moves/はやてがえし.md` 参照。

* タイプ: かくとう / 分類: 物理 / 威力: 65 / 命中率: 100 / PP: 15 / 対象: 1体選択 / 優先度: +3
* 対象が先制攻撃技（優先度1以上の攻撃技）を選択している時のみ成功する。
* 対象がすでに行動済み、または対象の技の優先度が3を超える場合は失敗する。
* 命中時、100%の確率で対象をひるませる。

## 既存実装の確認結果と問題点

`src/jpoke/handlers/move_attack.py` に `_はやてがえし_can_apply` / `はやてがえし_try_move` /
`はやてがえし_apply_flinch` が実装され、`src/jpoke/data/moves/move_ha.py` の `はやてがえし`
エントリから登録済みだった。判定ロジック自体（`command_reserved()` による行動済みチェック、
`defender_move.is_attack` による変化技チェック、
`battle.speed_calculator.calc_move_priority(ctx.defender, defender_move)` による
動的な優先度再計算）は仕様と一致しており問題ない。

1. **`ON_TRY_MOVE_1` の priority 指定漏れ**: `はやてがえし_try_move` は同種の
   「対象行動済み・変化技選択」判定を行う `ねこだまし_check_first_turn`
   （`data/moves/move_na.py`、priority=30）・`ふいうち_try_move`
   （`data/moves/move_ha.py`、priority=30）・`じんらい_try_move`
   （`data/moves/move_sa.py`、priority=30）と同じ役割を持つにもかかわらず、
   `data/moves/move_ha.py` の登録で priority が指定されておらずデフォルト値（100）に
   なっていた。`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` 表には
   「30 | ねこだまし等: すでに行動をした」「30 | ふいうち: 対象行動済み・変化技選択」の
   行があり、はやてがえしの判定もこれらと同一カテゴリの判定であるため、
   priority=30 に統一する。
   * 実害: 現状は他に競合する `ON_TRY_MOVE_1` ハンドラが同一ターンで同時に発火する
     ケースが薄く目立った不具合は出ていないが、`docs/spec/turn.md` の記載順序
     （こだわり系アイテムでの技固定=10 → ミクルのみ=20 → 各種行動済みチェック=30 →
     じょおうのいげん=40 → …→ まもる・マジックコート反射=100）と矛盾したまま
     priority=100 で登録されていると、将来 priority 30〜99 の間に発火する新規ハンドラを
     追加した際に意図しない順序でこの判定が実行される回帰リスクがあるため修正する。

## ハンドラ構成（修正後）

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| はやてがえし | `Event.ON_TRY_MOVE_1` | 30 | `attacker:self`（デフォルト） | 対象が未行動かつ先制攻撃技（優先度1〜3）を選択している時のみ成功させる |
| はやてがえし | `Event.ON_DAMAGE_HIT` | デフォルト（100） | `attacker:self`（デフォルト） | 命中時に100%の確率で対象をひるませる |

* `Event.ON_TRY_MOVE_1` の priority=30 は `docs/spec/turn.md` に明記された
  「ねこだまし等: すでに行動をした」「ふいうち: 対象行動済み・変化技選択」の行に倣う。
* `Event.ON_DAMAGE_HIT` は turn.md に個別の priority 表がないため、同種の
  ひるみ付与ハンドラ（`ねこだまし_apply_flinch`、`ハートスタンプ_apply_flinch` 等、
  いずれもデフォルト優先度100で登録）に倣いデフォルトのままとする。

## 実装詳細・エッジケース

* 対象の技の優先度は `battle.speed_calculator.calc_move_priority(ctx.defender, defender_move)`
  を用いて、はやてがえしの攻撃が実際に行われる瞬間に再計算する。これにより
  はやてのつばさ（HP満タン時のみひこう技の優先度+1）やグラススライダーの
  グラスフィールドによる優先度上昇のように、技選択時からはやてがえし発動までの間に
  状態が変化して優先度が変わるケースにも正しく対応できる
  （`ON_MODIFY_MOVE_PRIORITY` イベントを都度発火するため）。
* 変化技チェック（`defender_move.is_attack`）は、いたずらごころ等で優先度が上がった
  変化技（つぶらなひとみ等）に対しても正しく失敗させる。
* 対象より使用者が先に行動しなければ失敗する条件は
  `priority > ctx.move.priority`（対象の優先度がはやてがえし自身の優先度3を超える場合）で
  判定する。ctx.move.priority は固定値3のため動的な再計算は不要
  （かくとうタイプの優先度を変更する既存の特性・道具は本プロジェクトに存在しない）。
