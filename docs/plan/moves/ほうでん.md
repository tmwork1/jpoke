# ほうでん 実装計画書

## 仕様要約（docs/spec/moves/ほうでん.md 参照）

* タイプ: でんき / 分類: 特殊 / 威力: 80 / 命中: 100 / PP: 15
* 対象: 相手1体（本プロジェクトはシングルバトルのみ対象） / 優先度: 0 / 接触なし
* 効果: 追加効果として30%の確率で相手をまひ状態にする。

## 現状の実装状況

`src/jpoke/data/moves/move_ha.py` に既に登録済み。

```python
"ほうでん": MoveData(
    type="でんき",
    category="special",
    pp=15,
    power=80,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ほうでん_apply_paralysis_to_defender,
        )
    }
),
```

`src/jpoke/handlers/move_attack.py` にハンドラ実装済み。

```python
def ほうでん_apply_paralysis_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="まひ", chance=0.3)
```

内容は仕様書と一致しており、修正不要。

## ハンドラ構成

| event | priority | subject_spec | 役割 |
|---|---|---|---|
| ON_DAMAGE_HIT | 100（デフォルト） | attacker:self（デフォルト） | 30%の確率で相手をまひ状態にする |

`apply_ailment_to_defender` は `handlers/move.py` の既存共通関数を使用する
（`battle.resolve_secondary_chance` で確率解決後 `battle.ailment_manager.apply` を呼ぶ）。

## Priority 根拠

まひ付与は `Event.ON_DAMAGE_HIT`（ダメージ適用後に発火、`docs/spec/turn.md` には専用イベント名では
掲載されていないが、10まんボルト・のしかかり・ねっぷう等、既存の同種「追加効果」ハンドラがすべて
`ON_DAMAGE_HIT` + デフォルト priority=100 で登録されているため、同一パターンを踏襲する。

## 注意点・エッジケース（review-test 向け）

### まひの無効条件
- でんきタイプはまひ無効。`apply_ailment_to_defender` → `battle.ailment_manager.apply` 内部で判定されるため、
  本技側ハンドラに追加ガードは不要。

### 全体攻撃技の範囲について
- 仕様上は「自分以外の全員」が対象の全体攻撃技だが、本プロジェクトはシングルバトルのみを対象とするため、
  実質的に相手1体を対象とする通常の攻撃技として扱ってよい。ダブル・トリプルバトルの追加効果発動率補正
  （相手2体命中時2/3、3体命中時1/2）は対象外。

### 確率発動のテスト方針
- `secondary_chance` 引数で `battle.random` を制御し、確率発動ケース（発動）およびタイプ耐性による
  無効化ケースを既存テストで検証済み。
