# 計画書: そうでん

## 仕様要約

- でんきタイプ / 変化 / 命中なし（必中） / PP20 / 対象: 相手1体
- 命中した相手を「そうでん状態」にし、そのターン中に相手が使う技のタイプをでんきタイプに変換する
- ターン終了時にそうでん状態は解除される（1ターン限定）
- 相手がすでに行動していた場合は失敗する（`ON_TRY_MOVE_1` priority 30: `ねこだまし等: すでに行動をした` と同じ判定）
- みがわり状態の相手には防がれる
- この技自体がでんきタイプのため、ひらいしん・ちくでん・でんきエンジンを持つポケモンには無効
- `わるあがき`はそうでん状態でもでんきタイプにならない
- 交代で出てきたポケモンに対しては成功する

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（MoveHandler デフォルト） | 相手にそうでん状態を付与する（技ハンドラ） |
| `ON_MODIFY_MOVE_TYPE` | 100（デフォルト） | `attacker:self` | そうでん状態の攻撃側が出す技のタイプをでんきに変換する（volatile ハンドラ） |
| `ON_TURN_END` | 110 | `source:self` | ターン終了時にそうでん状態を解除する（volatile ハンドラ） |

## Priority 根拠

- **`ON_STATUS_HIT` priority 100**: `MoveHandler` のデフォルト値。既存の `タールショット_apply_volatile_to_defender` や `ちょうはつ_apply_volatile_to_defender` と同パターン。
- **`ON_MODIFY_MOVE_TYPE` priority 100**: `AbilityHandler` のデフォルト値。`うるおいボイス_modify_move_type`（`data/ability.py` subject_spec=`"attacker:self"`, priority 100）と同形式。
- **`ON_TURN_END` priority 110**: `docs/spec/turn.md` の `ON_TURN_END` 節に「ちょうはつの終了」「いちゃもんの終了」「アンコールの終了」等が priority 110 に記載されており、1ターン限りの揮発状態の解除はすべて priority 110 に統一されている。

def そうでん_apply_volatile_to_defender(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:

### 4. `handlers/volatile.py` への追加

五十音順で `さわがしい` 〜 `しおづけ` の間に挿入する。

```python
def そうでん_turn_end(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """そうでん状態のターン終了処理: ターン終了時に無条件で解除する。"""
    battle.volatile_manager.remove(ctx.source, "そうでん")
    return HandlerReturn(True)


def そうでん_move_type(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """そうでん状態による技タイプ変換: 攻撃側の使う技をでんきタイプに変換する。

    わるあがきはでんきタイプに変換されない。
    """
    if ctx.move and ctx.move.name != "わるあがき":
        return HandlerReturn(value="でんき")
    return HandlerReturn(value=value)
```

## 注意点・エッジケース

- **「相手がすでに行動済みの場合は失敗」**: `docs/spec/turn.md` の `ON_TRY_MOVE_1` priority 30 に「ねこだまし等: すでに行動をした」が記載されている。このチェックはフレームワーク側が処理するためハンドラ内にガード不要。
- **みがわりによる防御**: `ON_TRY_MOVE_1` priority 100 に「まもる・キングシールド等による無効化」があり、みがわりについては `ON_APPLY_MOVE` priority 30 に「みがわりによる無効化」がある。これらはフレームワーク側が処理するためハンドラ内にガード不要。
- **ひらいしん等による無効**: `ON_TRY_MOVE_1` priority 110 「ひらいしん等: でんきタイプ無効」でフレームワーク側が処理するためハンドラ内にガード不要。
- **`わるあがき` 除外**: Wikiの仕様に明記されており、`そうでん_move_type` 内で `ctx.move.name != "わるあがき"` の条件でスキップする。
- **ターン終了解除タイミング**: `ON_TURN_END` priority 110 は `ちょうはつの終了` 等と同順。同一ターンに付与・解除が行われることを想定した1ターン効果。
- **`count` 指定なし**: `apply_volatile_to_defender` に `count=None` を渡す（ターン終了ハンドラが解除を担うため count による自動 tick は不要）。
- **`ON_MODIFY_MOVE_TYPE` のコンテキスト**: `resolve_move_type` が発火する `AttackContext` の `attacker` がそうでん状態を持つポケモンなので `subject_spec="attacker:self"` が正しい。
- **`.bak` ファイルの `そうでん_move_type`**: `.bak` は `ctx.move._type = "でんき"` と直接代入しており現行アーキテクチャと異なる。正しくは `HandlerReturn(value="でんき")` でイベント値を上書きする方式にする。
