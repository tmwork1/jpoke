# ブラストバーン 実装計画書

## 仕様要約（docs/spec/moves/ブラストバーン.md 参照）

* タイプ: ほのお / 分類: 特殊 / 威力: 150 / 命中: 90 / PP: 8（champions基準。move_list.txt により統一）
* 対象: 相手1体 / 優先度: 0 / 接触なし / 分類フラグなし
* 効果: 命中して成功した次のターン、使用者は反動で行動できなくなる（リチャージ状態）。
  がんせきほう・ギガインパクト・はかいこうせん・ハードプラント・ハイドロカノンと共通の効果。

## 現状の実装状況

`src/jpoke/data/moves/move_ha.py` に既に登録済み。

```python
"ブラストバーン": MoveData(

    category="special",
    pp=8,
    power=150,
    accuracy=90,
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(

```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 命中後、使用者に「リチャージ」揮発性状態を付与する |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要、6技共用）:

```python
def リチャージ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """リチャージ系技: 命中後に使用者へ「リチャージ」volatile を付与する。

    がんせきほう・ギガインパクト・はかいこうせん・ブラストバーン・
    ハードプラント・ハイドロカノンで共用。
    """
    return apply_volatile_to_attacker(battle, ctx, value, volatile="リチャージ")
```

「リチャージ」揮発性状態自体は `src/jpoke/data/volatile.py` / `src/jpoke/handlers/volatile.py` に
`リチャージ_block_action` として実装済み（次ターンの行動をブロックし、状態を自動解除する）。
交代すると `volatile_manager` の通常仕様によりリチャージ状態は解除される。

## Priority 根拠

ダメージを与えた技の追加効果（使用者自身への volatile 付与）は既存の同種ハンドラ
（がんせきほう・はかいこうせん 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）の
パターンに従う。個別の priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **まもるで防がれた場合**
   - 技が失敗した場合は `ON_DAMAGE_HIT` が発火しないため、リチャージ状態にならない。実装済み・テスト済み。
2. **命中しなかった場合**
   - 命中判定に失敗した場合も同様に `ON_DAMAGE_HIT` が発火せず、リチャージ状態にならない。実装済み・テスト済み。
3. **交代によるリチャージ解除**
   - リチャージ状態中に使用者が交代すると、揮発性状態は解除される（`リチャージ` volatile は他の交代解除系 volatile と同様の共通仕様に従う）。実装済み・テスト済み。
4. **次ターンの行動不能**
   - リチャージ状態のポケモンは次のターン行動できず、状態は自動的に解除される。実装済み・テスト済み。
5. **相手をひんしにした場合**
   - 相手のHPが0になってもリチャージ状態は通常通り付与される（`ON_DAMAGE_HIT` は命中判定後に発火するため）。既存の共用ハンドラで対応済み。実装済み・テスト済み。
6. **リーサル計算**
   - `docs/progress/move.md` で「リーサル実装: n/a」「リーサルテスト: n/a」。反動で動けなくなる効果は
     HP依存の状態ではないため、リーサル計算上は通常の固定威力技として扱われ、専用ハンドラは不要。
