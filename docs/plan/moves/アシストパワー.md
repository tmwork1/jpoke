# アシストパワー 実装計画書（再レビュー）

## 仕様要約

`docs/spec/moves/アシストパワー.md` 参照。

* タイプ: エスパー / 分類: 特殊 / 威力: 20（変動） / 命中率: 100 / PP: 12（Championsでは
  全技PPが8・12・16・20に統一。SVまでのPPは10） / 対象: 相手1体 / 非接触
* 効果: 自分の能力ランク（こうげき・ぼうぎょ・とくこう・とくぼう・すばやさ・命中率・回避率）
  の正のランク変化の合計1段階ごとに威力が20増加する（負のランクは合計に含めない）。
  追加効果なし。

## 既存実装の確認結果と問題点

`src/jpoke/handlers/move_attack.py` の `アシストパワー_boost_power_by_rank` は
`attacker.rank` のうち `hp` を除く全キー（`atk`/`def`/`spa`/`spd`/`spe`/`accuracy`/
`evasion`）について正のランクのみ合計し、`Event.ON_CALC_POWER_MODIFIER` で
`modifier = 4096 * (1 + rank_sum)` を `apply_fixed_modifier` により適用している。
基本威力20に対しこの倍率補正をかけることで `power = 20 * (1 + rank_sum) = 20 + 20 *
rank_sum` となり、仕様と一致する。イベント・`subject_spec="attacker:self"` も
`Event.ON_CALC_POWER_MODIFIER` を使う他の技・特性・アイテムハンドラと同じパターンで
問題ない（`ON_CALC_POWER_MODIFIER` は `docs/spec/turn.md` に未掲載のため、同種の威力
補正ハンドラ（`ちからずく`特性など）に倣いデフォルト優先度を採用する）。

一方、`src/jpoke/data/moves/move_a.py` の `MoveData` エントリに以下の誤りがあった。

* `pp=10`（SVまでの値）になっていたが、`docs/champions/move_list.txt` /
  `docs/champions/changes_from_sv.md`（技のPPが8・12・16・20の4段階に統一）によれば
  Championsでは `12` が正しい値のため修正する。

`flags` は指定なし（デフォルトの空集合）で、`直接攻撃: ×`（非接触）と一致しており問題ない。
`target` もデフォルトの `"foe"` で `1体選択` と一致している。
