## 仕様要約

- ゴーストタイプ・物理・威力 90・命中率 100・PP 12（`src/jpoke/data/ps-champ-ja/moves.json` 290行目、champions側で8/12/16/20の4段階に統一済み。Gen9本家PPは10）・優先度 0・直接攻撃
- 2ターン技
  - 1ターン目: 姿を消す（`シャドーダイブ` 揮発状態が付与される。ゴーストダイブ専用の揮発状態は存在せず、シャドーダイブと共有する）。攻撃は行わない。
  - 2ターン目: 相手を攻撃する。攻撃終了後に揮発状態が解除される。
- 姿を消している間の回避・交代禁止・コマンド固定などの仕様は `docs/spec/volatiles/シャドーダイブ.md` に準拠（シャドーダイブ側で実装済みの揮発状態をそのまま再利用する）。
- まもる・みきり・たたみがえし・トーチカ・キングシールド・ニードルガード・ブロッキング・スレッドトラップ・かえんのまもりの効果を無視して攻撃する（`unprotectable` フラグ）。ダイウォールは除く（ダイウォール自体は別途ダメージ無効化ロジックで処理されるためスコープ外）。
- 1ターン目はPPを消費しない（ためターン）。
- ゴーストダイブ状態中は交代できない（ゴーストタイプを除く）。シャドーダイブ側の揮発状態実装がそのまま適用される。

| 項目 | 修正前 | 修正後 |

## 追加すべきテスト（`tests/moves_attack/test_move_ka.py` への追加）

`test_ゴッドバード_ひるみが発動する()` と `test_ゴールドラッシュ_特攻1段階低下が発動する()` の間に追加する（五十音順の暫定位置。マージ後に `sort_tests.py` で再整形される）。

```python
def test_ゴーストダイブ_2ターンで攻撃する():
    """1ターン目はダメージを与えず、2ターン目にダメージを与えて揮発状態が解除される"""
    battle = t.start_battle(
        # ノーマルタイプはゴースト無効なのでどくタイプを使用
        team0=[Pokemon("ゲンガー", move_names=["ゴーストダイブ"])],
        team1=[Pokemon("ドガース")],
        accuracy=100,
    )
    defender = battle.actives[1]
    hp_before = defender.hp

    # 1ターン目: 揮発状態付与のみ、ダメージなし
    t.run_move(battle, 0)
    assert defender.hp == hp_before
    assert battle.actives[0].has_volatile("シャドーダイブ")

    # 2ターン目: ダメージあり、揮発状態解除
    t.run_move(battle, 0)
    assert defender.hp < hp_before
    assert not battle.actives[0].has_volatile("シャドーダイブ")


def test_ゴーストダイブ_まもる中の相手にダメージを与えられる():
    """ゴーストダイブはまもる状態を貫通する（unprotectable ラベル）"""
    battle = t.start_battle(
        team0=[Pokemon("ゲンガー", move_names=["ゴーストダイブ"])],
        team1=[Pokemon("ドガース")],
    )
    defender = battle.actives[1]
    initial_hp = defender.hp
    battle.volatile_manager.apply(defender, "まもる", count=1)
    # 溜めターン
    t.run_move(battle, 0)
    assert battle.actives[0].has_volatile("シャドーダイブ")
    # 攻撃ターン（まもる状態の相手に攻撃）
    t.run_move(battle, 0)
    assert not battle.actives[0].has_volatile("シャドーダイブ")
    assert battle.move_executor.move_success
    assert defender.hp < initial_hp
```

**レビューでの修正1（防御側のタイプ）**: 当初の計画ではカビゴン（ノーマルタイプ）を防御側にしていたが、
ノーマルタイプはゴースト技を無効化してしまうため、防御側はどくタイプの「ドガース」に変更した
（`test_シャドーダイブ_まもる中の相手にダメージを与えられる` と同じ理由）。

**レビューでの修正2（PPテストの削除）**: 当初「1ターン目はPPを消費しない」というテストを追加する計画だったが、
実装をトレースしたところ、`tests/test_utils.py` の `run_move` ヘルパーは `battle.run_move(attacker, move)` で
攻撃側の実際の `Move` インスタンスを直接渡しており、`core/command_manager.py` の
`resolve_move_from_command`（強制行動時にPP消費を防ぐため新しい `Move` インスタンスを生成するロジック、
`command == Command.FORCED` の分岐）を経由しない。そのため `run_move` を2連続で呼ぶと、本来は
強制行動でPPが消費されないはずの2ターン目でも実際の `Move` インスタンスのPPが減ってしまい、
実際のゲームコマンド解決フローとは異なる結果になる（テストヘルパーの制約によるものであり、
ハンドラ実装のバグではない）。この観点のPP消費抑制は既存の
`tests/volatiles/test_volatile_ka.py` の `test_かくれる_強制行動ターンはPPを消費しない`
（`hidden_move_name` パラメトライズに `"シャドーダイブ"` を含む）が
`battle.volatile_manager.apply` + `battle.step()` の正しいコマンド解決フローで既に検証しているため、
`run_move` ベースの重複テストは追加せず、2ターン攻撃テストとまもる貫通テストの2件のみを追加した。

## 注意点・エッジケース（review-test 向け）

### ゴーストダイブ専用の揮発状態を作らない理由

- `docs/spec/volatiles/シャドーダイブ.md` に明記の通り、シャドーダイブとゴーストダイブは同一の揮発状態を共有する（ゲーム内でも「シャドーダイブ状態」という共通名称）。
- `src/jpoke/types/volatile.py` / `src/jpoke/data/volatile.py` に `"ゴーストダイブ"` という揮発状態名は存在しない。既存の `tests/volatiles/test_volatile_ka.py` のパラメトライズ済みテスト（`hidden_move_name` が揮発状態名と技名の一致を前提にしている）に `"ゴーストダイブ"` を追加すると `has_volatile("ゴーストダイブ")` が常に `False` になり誤ったテストになるため、追加しない。

### PPについて

- `src/jpoke/data/ps-champ-ja/moves.json` 290行目が正: PP12。Gen9本家（Wiki一次情報）はPP10だが、`docs/champions/changes_from_sv.md` によりchampionsでは全技PPが8/12/16/20の4段階に統一されているため、実装は12を採用する。

### `unprotectable` フラグについて

- `src/jpoke/model/move.py` の `is_blocked_by_protect` が `unprotectable` フラグを参照しており、これだけで「まもる系を無視して攻撃する」動作が実現される。ニードルガード・キングシールド・トーチカ・ブロッキング・スレッドトラップの反撃・追加効果が発動しない点や、まもる状態を「取り除く」演出（「守りを打ち破った!」ログ）は本実装のスコープ外（シャドーダイブ側の既存実装のスコープ外指定に準ずる）。

## スコープ外（未実装、シャドーダイブ側の既存スコープ外指定を踏襲）

- フィールド効果回避（フィールドの影響を受けない）
- てだすけによる命中・毒タイプのどくどく命中
- フリーフォールによる必ず失敗
- まもる系状態を「取り除く」演出・メッセージ
- わたげによるすばやさ低下回避
- ボール捕獲不可・おどりこ不発動
- 第7世代以降は無い「ちいさくなる」相手への必中2倍効果（実装しない。Gen6のみの旧仕様のため対象外）
