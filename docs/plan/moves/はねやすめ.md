# 計画書: はねやすめ

## 仕様要約

- タイプ: ひこう / カテゴリ: 変化 / PP: 5 / 対象: 自分 / まもる: 無視
- 最大HPの半分（切り上げ）を回復する
- 使用したターン中、ひこうタイプを一時的に失う（ノーマルタイプ扱い）
- ターン終了時（priority 120）にひこうタイプを復帰する
- HPが満タンの場合は技が失敗し、ひこうタイプも失われない
- テラスタルでひこうタイプになっているポケモンが使用した場合もタイプは失われない
- ひこうタイプを持たないポケモンが使った場合はタイプ変化なし（HP回復のみ）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 | 登録場所 |
|---|---|---|---|---|
| `ON_BEFORE_APPLY_MOVE` | 100 | `attacker:self` | HP満タン時の失敗チェック | `data/move.py` の技ハンドラ |
| `ON_APPLY_MOVE` | 100 | `attacker:self` | 回復実行 + ひこうタイプ除去（volatile付与） | `data/move.py` の技ハンドラ |
| `ON_TURN_END` | 120 | `source:self` | ひこうタイプ復帰（volatile除去） | `data/volatile.py` の volatile ハンドラ |

## Priority 根拠

- `ON_TURN_END` priority 120: `docs/spec/turn.md` に「120 | はねやすめのひこうタイプ復帰」と明記されている
- `ON_BEFORE_APPLY_MOVE` priority 100: `docs/spec/turn.md` の `Event.ON_APPLY_MOVE` セクション「100 | 自己回復技: HP満タン」に対応する
- `ON_APPLY_MOVE` priority 100: 上記同セクション内、標準 priority

## モデル変更: `model/pokemon.py`

はねやすめのひこうタイプ除去を表現するために `removed_types` フィールドを追加する。

```python
# __init__ 内（added_types と同じ箇所に追加）
self.removed_types: list[Type] = []
```

```python
# reset_on_switch_out 内（added_types = [] と同じ箇所に追加）
self.removed_types = []
```

```python
# types プロパティ内（既存の extra = [...] の前に追加）
@property
def types(self) -> list[Type]:
    ...
    if not self.added_types:
        result = base
    else:
        extra = [t for t in self.added_types if t not in base]
        result = base + extra if extra else base
    # はねやすめ等によるタイプ除去を適用（テラスタル中は無視）
    if self.removed_types and not self.active_tera_type:
        result = [t for t in result if t not in self.removed_types]
    return result if result else result  # タイプなしは空リスト
```

注: `result` が空リストになるケース（ひこう単タイプ + はねやすめ）は空リストのまま返す。
`has_type` は `type_ in self.types` で動くため自動対応。

## 実装箇所

### `handlers/move_status.py` に追加する関数

五十音順に従い「は」行の適切な位置（`はきだす` 系の後、`はらだいこ` 系の前）に挿入する。

```python
def はねやすめ_check(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """はねやすめの失敗チェック: HPが満タンの場合は失敗する。"""
    mon = ctx.attacker
    if mon.hp == mon.max_hp:
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def はねやすめ_heal_and_remove_flying(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """はねやすめの効果: 最大HPの1/2を回復し、ひこうタイプを一時的に除去する。

    テラスタル中はタイプ除去を行わない。
    ひこうタイプを持たないポケモンはタイプ除去を行わない。
    """
    mon = ctx.attacker
    battle.modify_hp(mon, r=1/2)
    # テラスタル中はタイプ変化しない
    if mon.active_tera_type:
        return HandlerReturn(value=value)
    # ひこうタイプを持つ場合のみ除去 volatile を付与
    if mon.has_type("ひこう"):
        battle.volatile_manager.apply(mon, "はねやすめ")
    return HandlerReturn(value=value)
```

### `handlers/volatile.py` に追加する関数

五十音順に従い「は」行の適切な位置（`ハロウィン` 系の後）に挿入する。

```python
def はねやすめ_remove_flying(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """はねやすめ volatile 付与時: ひこうタイプを removed_types に追加する。"""
    mon = ctx.source
    if "ひこう" not in mon.removed_types:
        mon.removed_types.append("ひこう")
    return HandlerReturn(value=value)


def はねやすめ_restore_flying(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """はねやすめ volatile 解除時 (ターン終了): ひこうタイプを復帰する。"""
    mon = ctx.source
    if "ひこう" in mon.removed_types:
        mon.removed_types.remove("ひこう")
    battle.volatile_manager.remove(mon, "はねやすめ")
    return HandlerReturn(value=value)
```

## 注意点・エッジケース（review-test 向け）

### タイプ除去のスコープ

- `model/pokemon.py` の `types` プロパティに `removed_types` フィルタを追加する必要がある。これが最も影響範囲の広い変更。
- `removed_types` は `reset_on_switch_out` でもクリアすること（場を離れた際に自動解除）。
- `removed_types` は `__deepcopy__` のキーに含める必要はない（バトル中フィールドのため `fast_copy` でコピーされる）。

### テラスタル中の挙動

- テラスタルでひこうタイプになっている場合: `mon.active_tera_type == "ひこう"` であっても `is not None` なのでタイプ除去をスキップする（仕様通り）。
- テラスタルでひこう以外のタイプになっており、本来ひこうを持つポケモンが使用した場合: `mon.has_type("ひこう")` は False なので除去不要（タイプはテラスタル後のタイプのみ）。
- ステラテラスタルの場合: `active_tera_type == "ステラ"` のとき `types` は `data.types` ベース。ただし `has_type("ひこう")` は True になり得るが、仕様上ステラはテラスタルと同じく除去しない。`mon.active_tera_type` が `not None` なら除去スキップ。

### HP満タン失敗

- `ON_BEFORE_APPLY_MOVE` でチェックし、`stop_event=True` で中断する。HP回復もタイプ除去も発生しない。
- `じこさいせい_heal_self` と同パターン。

### ひこうタイプを持たないポケモン

- `has_type("ひこう")` が False の場合は volatile を付与しない。HP回復のみ発動。

### ON_TURN_END の volatile 解除タイミング

- `VolatileHandler` の `ON_TURN_END` で `priority=120` を指定することで、ほろびのうた（120）と同タイミングになる。仕様（turn.md）通り。
- `はねやすめ_restore_flying` の中で `volatile_manager.remove` を直接呼ぶ。`tick_volatile` は使わない（ターンカウントではなくターン終了で即解除する仕組みのため）。

### ON_VOLATILE_START による除去タイミング

- volatile 付与と同時に `ON_VOLATILE_START` が発火し `removed_types` に "ひこう" が追加される。これにより、技を使用した同ターン内の被弾相性計算（例: じめん技の命中可否、タイプ一致ボーナス等）でひこうタイプを参照しない状態になる。

### 同ターン内の地面判定への影響

- `removed_types` に "ひこう" が入っている間は `has_type("ひこう")` が False になるため、`Check_floating` イベント等でひこうの浮遊判定も受けない（地面にいる状態）。
- じめんタイプの技が当たるようになる。

### かいふくふうじ

- `ON_APPLY_MOVE` の前に `ON_BEFORE_APPLY_MOVE` でかいふくふうじが技失敗を返す（別ハンドラ、既存実装）。タイプ変化も発生しない。

### 交代時の自動解除

- `reset_on_switch_out` で `removed_types = []` がクリアされるため、はねやすめ使用ターン中に交代が発生した場合（さいはい等）でも次のポケモンは通常タイプになる。volatile 自体も交代で自動削除される（既存 volatile_manager の動作）。

### VolatileName 型定義

- `"はねやすめ"` を `utils/type_defs.py` の `VolatileName` Literal に追加する必要がある。
