# かみくだく 実装計画書

## 仕様要約（docs/spec/moves/かみくだく.md 参照）

* タイプ: あく / 分類: 物理 / 威力: 80 / 命中: 100 / PP: 15
* 対象: 相手1体 / 優先度: 0 / 接触あり（直接攻撃）
* 効果: 追加効果として、20%の確率で相手の『ぼうぎょ』ランクを1段階下げる。
* かみつき技（`flags` に `"bite"` を含む）。特性『がんじょうあご』の対象。

## 現状の実装状況

`src/jpoke/data/move.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。
以下のとおり仕様と一致していることを確認した（修正不要）。

```python
"かみくだく": MoveData(
    type="あく",
    category="physical",
    pp=15,
    power=80,
    accuracy=100,
    flags={"bite", "contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.かみくだく_lower_defender_def,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 20%の確率で相手の『ぼうぎょ』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def かみくだく_lower_defender_def(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"def": -1}, chance=0.2)
```

`modify_defender_stats`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）・威力1.3倍化、`flags={"bite"}` によるがんじょうあごの
威力1.5倍化は、いずれも汎用ハンドラ側で処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（能力ランク低下）は既存の同種ハンドラ（あわ・いにしえのうた・いわくだき 等）と
同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。個別の priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **がんじょうあごとの相互作用**
   - `flags={"bite"}` により、がんじょうあご所持ポケモンが使用すると威力が1.5倍（80→120）になる。
     既存の がんじょうあご テストファイル側で他のかみつき技と同様の網羅がされていれば個別追加は不要。
2. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・ぼうぎょ低下無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
3. **テスト方針**
   - `secondary_chance` を0/1に固定し、ぼうぎょランクの変化有無を確認するテストを追加する
     （`test_かみくだく_相手にダメージを与える` / `test_かみくだく_ぼうぎょ1段階低下が発動する` /
     `test_かみくだく_ぼうぎょ1段階低下が発動しない`）。
