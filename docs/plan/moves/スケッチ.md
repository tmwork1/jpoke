# 計画書: スケッチ

## 仕様要約

* ノーマルタイプ / 変化 / 威力- / 命中-（必中） / PP1 / 対象: 相手1体 / 優先度0 / 接触なし
* 相手が最後に使った技をコピーし、スケッチ自身の代わりに永続的に習得する。成功するとスケッチは
  消え、コピーした技が残る（交代・ひんし・バトル終了後も維持される）。
* まもる・マジックコートを無視して成功する。姿を隠している（あなをほる等の潜伏中）相手には失敗する。
* 詳細は `docs/spec/moves/スケッチ.md` を参照。

## 実装保留の判断

本技の完全な実装には、以下の大規模な変更が必要となる。

1. **技スロットの恒久的な差し替え**: `Pokemon.moves` の該当インデックスを、コピーした技の
   `Move` インスタンスへ恒久的に置き換える機構が必要。ものまね（`docs/plan/moves/ものまね.md`）
   は交代・ひんし・バトル終了で「ものまね」へ復元する一時的な差し替えだが、スケッチはバトルが
   終わってもスケッチには戻らず、コピーした技をそのまま覚え続ける（経験値が入るバトルの場合。
   経験値が入らない対戦施設等では戦闘終了後にスケッチへ戻る仕様もあり、分岐が更に複雑）。
   `Pokemon.reset_on_switch_out()`（`src/jpoke/model/pokemon.py`）や `set_moves()` はいずれも
   恒久的な技スロット書き換え・復元条件分岐に対応していない。
2. **コピー技のPP**: コピーした技のPPは1（スケッチ自身のPP）になる。`Move` のPP初期化ロジック
   （`src/jpoke/model/move.py` の `self.pp = self.data.pp`）を流用しつつ、スケッチ固有のPP=1
   ルールを適用する必要がある。
3. **コピー対象の取得と失敗条件**: 「相手が場に出てから技を1度も使っていない」「相手がへんしん
   状態」「自分がへんしん状態」「相手がみがわり状態（第二〜四世代のみ、Championsは第九世代基準
   のため対象外の可能性が高い）」「相手がトリックガードに守られている」等、`ものまね`より
   失敗条件が多い。
4. **習得禁止技リスト**: すでに覚えている技・スケッチ自身・ダークわざ・Zワザ・スターモービル
   専用技・オウムがえし/じばく/だいばくはつ/ねごと/へんしん/ものまね/ゆびをふる/わるあがき等の
   一部世代限定制限（`docs/spec/moves/スケッチ.md`「覚える技」表）を判定する専用ロジックが要る。

上記はへんしん（種族・実数値ステータス・特性・能力ランクの丸ごとコピー）ほど大規模ではないが、
ものまねの一時的差し替えよりも復元条件が複雑な恒久的差し替えを要する点で同種の新規モデル変更が
必要である。前例（へんしん・ものまね・ゆびをふる。いずれも `docs/progress/move.md` で実装保留
として扱われている技コピー系の技）と同様に、本技も実装保留として扱う。

## data/moves/move_sa.py の修正内容

```python
"スケッチ": MoveData(
    type="ノーマル",
    category="status",
    pp=1,  # champions/moves.mdに記載なし。Gen9本家基準の値をそのまま採用
           # （ものまね・ゆびをふる等、Championsに記載のない技コピー系の技と同様）。
    # accuracy省略=必中。潜伏中の相手への失敗はHIDDEN_MOVE_ALLOWED_MOVES側で処理される。
    flags={"non_encore", "non_negoto", "non_copycat", "unprotectable", "unreflectable"},
    # 実装保留: 相手が最後に使った技を技スロットへ恒久的に追加する技コピー機構
    # （ものまねの一時差し替えより復元条件が複雑）が必要なため対応を見送る。
    # 詳細は docs/plan/moves/スケッチ.md 参照（前例: へんしん・ものまね・ゆびをふる）。
    handlers={},
),
```

* `pp`: 1に修正する。`src/jpoke/data/ps-champ-ja/moves.json` にスケッチのエントリは存在しない
  （ものまね・ゆびをふる等と同様）ため、確認可能な第9世代の値（1）を採用する。旧実装値0は
  ユーザー指示によるTODO対応見送り時の暫定値であり、実際のゲーム仕様（PP1）と乖離したバグで
  あった。
* `accuracy`: 必中技のため明示的な値を持たせず省略する（デフォルト`None`）。旧実装値`accuracy=100`
  は誤り（回避率補正等の影響を受けてしまう）だったため修正する。
* 潜伏中の相手（あなをほる等）への失敗は `handlers/volatile.py` の `HIDDEN_MOVE_ALLOWED_MOVES`
  により自動的に処理される（スケッチは許可リストに含まれないため成功しない）。専用ハンドラは
  不要。
* `flags`:
  * `non_encore`: アンコール対象外。`src/jpoke/data/ps-champ-ja/moves.json` のアンコール効果説明（「スケッチ」を
    含む一覧）と一致（既存実装済み・変更なし）。
  * `non_negoto`: 第五世代以降ねごとで選ばれない仕様（既存実装済み・変更なし）。
  * `non_copycat`: まねっこ・ものまね・オウムがえしでコピーできない仕様
    （`docs/spec/moves/スケッチ.md`「技の仕様」節）のうち、まねっこ側の実装
    （`まねっこ_can_use`、`src/jpoke/handlers/move_status.py`）が
    `copied_move.has_flag("non_copycat")` を見て失敗判定するため、本フラグを追加する
    （旧実装は未設定で漏れていた）。ものまね・オウムがえし自体は本コードベースで未実装のため
    それらとの相互作用は対象外。
  * `unprotectable`: まもる状態を無視して成功する仕様（`docs/spec/moves/スケッチ.md`
    判定節「まもる: ×」）に対応するため追加する（旧実装は未設定で漏れていた）。
    `Move.is_reflectable`と対になる`Move.is_blocked_by_protect`
    （`src/jpoke/model/move.py`）が`target=="foe"`の変化技をデフォルトでまもる対象とみなすため、
    明示的なフラグ付与が必須。へんしん（`docs/plan/moves/へんしん.md`）と同じパターン。
  * `unreflectable`: マジックコート状態を無視して成功する仕様（同判定節「マジックコート: ×」）に
    対応するため追加する（旧実装は未設定で漏れていた）。`Move.is_reflectable` は
    `category="status" かつ target in ("foe", "foe_side")` の技をデフォルトで反射対象とみなすため、
    明示的なフラグ付与が必須。
* さいはいに対する失敗は `handlers/move_status.py` の `_INSTRUCT_BLOCKED_MOVES` に
  「スケッチ」が既に含まれており、追加対応は不要。

## 将来実装時の設計メモ

* `Pokemon` に「技スロットの恒久的な差し替え」を行う公開APIを追加し、通常のバトル
  （経験値が入るケース）では復元せず、対戦施設等の経験値が入らないケースでのみバトル終了時に
  スケッチへ戻す、という条件分岐を持たせる設計が有力。ものまね実装時の「ものまね」volatile方式
  （`docs/plan/moves/ものまね.md`「将来実装時の設計メモ」参照）とは異なり、スケッチは
  Volatileではなく`Pokemon`本体の永続状態として保持する必要がある。
* コピー対象は `Pokemon.executed_move` を基本とし、へんしん状態・トリックガード状態・場に出てから
  未行動、等の失敗条件を個別に実装する。
* 習得禁止技の判定は `_INSTRUCT_BLOCKED_MOVES`（`src/jpoke/handlers/move_status.py`）とは別に、
  スケッチ専用の禁止技集合（ダークわざ・Zワザ・スターモービル専用技等）を新設する必要がある。
