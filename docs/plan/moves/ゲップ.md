# 計画書: ゲップ

## 仕様要約（docs/spec/moves/ゲップ.md 参照）

- どくタイプ / 特殊 / 威力120 / 命中90 / PP12（Champions基準。本家は10） / 対象: 1体選択 /
  優先度0 / 非接触
- 追加効果なし。ダメージのみ。
- 使用条件: その戦闘で「きのみ」を食べたことがあるポケモンでなければ失敗する。
  Championsでは選択自体は可能で、自分の行動時点（`Event.ON_TRY_MOVE_1`）で条件を満たしているかどうかで
  成否が決まる（本家のような「選択そのものをキャンセルする」処理は不要）。
- ねごと・まねっこで選ばれない（`non_negoto` / `non_copycat` フラグ）。

## 既存実装との差分

既存実装（`data/moves/move_ka.py` の `"ゲップ"` エントリ、`handlers/move_attack.py` の
`ゲップ_check_ate_berry`）は概ね仕様通りだが、以下の点で仕様と乖離していた。

1. **PPがChampions基準になっていない**
   既存: `pp=10`（本家シリーズの値）。
   → `src/jpoke/data/ps-champ-ja/moves.json` / `docs/progress/move.md` に準拠し `pp=12` に修正する。

2. **`non_copycat` フラグが付与されていない**
   一次情報によると、ゲップはまねっこ（Copycat）で選ばれない技だが、既存実装は `non_negoto` のみで
   `non_copycat` が付いていなかった。まねっこ側の実装（`まねっこ_can_use`、
   `src/jpoke/handlers/move_status.py`）は直前使用技が `non_copycat` フラグを持つ場合に失敗させる
   仕組みになっているため、フラグを追加するだけで仕様を満たせる。

## ハンドラ構成（既存実装の確認）

### handlers/move_attack.py: `ゲップ_check_ate_berry`（既存・変更なし）

```python
def ゲップ_check_ate_berry(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ゲップの失敗条件: きのみを食べていない場合に失敗させる。"""
    mon = ctx.attacker
    if not mon.ate_berry:
        battle.add_event_log(mon, LogCode.MOVE_FAILED,
                             payload=FailureLogPayload(move=ctx.move.name, display_reason="ゲップ_きのみ未食"))
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

- `mon.ate_berry`（`model/pokemon.py`）は `memory["battle"]` スコープ（バトル中はリセットされない）で
  管理されており、交代・ひんし復活をまたいで保持され、バトンタッチでは引き継がれない
  （後続ポケモンは別オブジェクトのため自動的に満たされる）。
- `battle.item_manager.consume_item` が自分できのみを消費した場合に `ate_berry = True` をセットする。
  むしくい・ついばむで相手のきのみを奪って食べる場合は `force_trigger_berry` 経由で最終的に
  `consume_item(attacker)` が呼ばれるため、こちらも正しく反映される。
- イベント: `Event.ON_TRY_MOVE_1`。`docs/spec/turn.md` の該当イベント表に「30 | ほおばる: きのみを
  持っていない」と同種の失敗条件が並んでおり、これに準じて priority=30 とする（既存実装のまま）。
- subject_spec: `"attacker:self"`（既存実装のまま。CLAUDE.mdの規約通りAttackContextのロールと一致）。
