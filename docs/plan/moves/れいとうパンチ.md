# れいとうパンチ 実装計画書

## 仕様要約

`docs/spec/moves/れいとうパンチ.md` を参照。
こおりタイプ・物理・威力75・命中100・PP15の直接攻撃技。
追加効果として10%の確率で相手を「こおり」状態にする。パンチ技のため「てつのこぶし」で威力1.2倍。

## ハンドラ構成

`Event.ON_DAMAGE_HIT` に `MoveHandler` デフォルト設定（`subject_spec="attacker:self"`, `priority=100`）で登録する。
`apply_ailment_to_defender` は `handlers/move.py` の既存共通関数を使用する。

| 技 | event | priority | subject_spec | 役割 |
|---|---|---|---|---|
| れいとうパンチ | ON_DAMAGE_HIT | 100 | attacker:self | こおり 10% |

## Priority 根拠

`docs/spec/turn.md` に `Event.ON_DAMAGE_HIT` は掲載されていない（ダメージ計算内部の追加効果付与イベント）。
既実装の `かみなりパンチ`・`ほのおのパンチ`・`れいとうビーム` と同じ priority=100（`h.MoveHandler` のデフォルト）を採用する。

def れいとうパンチ_apply_freeze_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="こおり", chance=0.1)

## リーサル計算

- 対象外（`n/a`）。こおりは行動阻害（凍結中は解凍まで行動不能）の状態異常であり、ダメージ量には影響しないため、
  本技専用のリーサルハンドラは不要（`docs/progress/move.md` のリーサル実装列も `n/a`）。

## エッジケース

- こおりタイプの相手はこおり無効（`ailment_manager` 側の共通処理で判定されるため、本技のハンドラに追加ガードは不要）。
- 特性「てつのこぶし」による威力1.2倍は `flags={"punch"}` を通じて特性側ハンドラが処理するため、本技側の実装は不要。
