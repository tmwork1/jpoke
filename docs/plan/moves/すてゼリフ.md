# 計画書: すてゼリフ

## 仕様要約

- タイプ: あく / カテゴリ: 変化 / PP: 20 / 命中: 100 / 対象: 1体選択（相手）
- 音技（labels=["sound"]）— みがわりを貫通、ぼうおんで無効
- 相手のこうげきととくこうをそれぞれ1段階下げる
- その後、使用者が控えのポケモンと交代する（自発交代）
- 交代制限（ありじごく・ねをはる・かげふみ等）を無視して交代可能
- 控えポケモンが存在しない場合はランク低下のみ発動し交代は発生しない
- ランク低下が特性等（クリアボディ・しろいきり・クリアチャーム等）で阻まれた場合は交代も発動しない（第七世代以降）
- まもる: ○ / マジックコート: ○（反射可） / よこどり: ×

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: 五十音順「す」行 — 既存の「スキルスワップ」の後、「ステルスロック」より前に配置する。

```python
def すてゼリフ_modify_defender_stats_and_pivot(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """すてゼリフ: 相手のこうげき・とくこうを1段階下げ、自発交代する。

    ランク低下が成功した場合のみ交代が発動する。
    ランク低下が阻まれた（クリアボディ等）場合は交代も発動しない（第七世代以降）。
    控えポケモンがいない場合はランク低下のみ発動し交代は発生しない。
    とらわれ状態（ありじごく・ねをはる・かげふみ・じりょく・バインド・にげられない・
    フェアリーロック等）を無視して交代できるため、バトンタッチと同様に
    `battle.query.can_switch` は使わず控えの生存だけを直接判定する。
    """
    result = modify_defender_stats(battle, ctx, value, stats={"atk": -1, "spa": -1})

    # ランク低下が完全に阻まれた（実際の変化量が空）場合は交代しない
    if not result.value:
        return result

    player = battle.get_player(ctx.attacker)
    state = battle.player_states[player]
    if any(mon.alive for mon in state.bench):
        state.interrupt = Interrupt.PIVOT

    return HandlerReturn(value=value)
```

`modify_defender_stats`（`handlers/move.py`）は ON_STATUS_HIT（value が bool）の場合、
`battle.modify_stats()` の戻り値（実際に変化したランクの dict）を `HandlerReturn.value` に
そのまま返す。戻り値が空 dict（＝クリアボディ等で完全に阻止された）場合は falsy になるため、
`if not result.value` で交代をスキップできる。かいりきバサミのようにこうげきのみ無効化された
場合は spa の変化が dict に残るため non-empty となり、交代は通常どおり発動する。

## 注意点・エッジケース

- **ランク低下と交代の依存関係（第七世代以降）**: クリアボディ・しろいきり・クリアチャーム等でランク低下が阻まれた場合は交代も発動しない。`modify_defender_stats` の戻り値（実際に変化したランクの dict）が空の場合に交代処理をスキップすることで実現する。ランク低下が一部だけ阻まれた（どちらか一方のみ阻止）場合は非空の dict が返るため交代は発動する

- **かいりきバサミへの対応**: かいりきバサミ（こうげき下げ無効）を持つ相手には特攻のみ下がる。wiki 仕様「対象が特性かいりきバサミである場合は特攻のみ下がる」。これは `modify_defender_stats` → `battle.modify_stats()` 内部でかいりきバサミ判定が行われ、こうげき低下のみキャンセルされることで自動処理される。ハンドラ側での追加対応は不要

- **まけんき・かちき持ちへの逆効果**: まけんきはこうげき、かちきはとくこうが1段階下がるごとに特性が発動する。すてゼリフでは2箇所のランクを下げるため特性が計2回発動しうる。ランク変化ハンドラ経由で自動処理されるためハンドラ側での追加対応は不要

- **音技の性質**: `flags={"sound"}` により、みがわり状態の相手にもランク低下が通る（みがわり貫通）。ぼうおん特性を持つ相手には技自体が無効化され、`ON_STATUS_HIT` は発火しない。ハンドラ側のガードは不要

- **マジックコートによる反射**: 相手がマジックコート状態の場合、イベント発火側で反射処理が行われ、使用者の攻撃・特攻が1段階下がりマジックコート使用者が交代する。これはイベント発火側で処理されるためハンドラ側での対応は不要

- **交代制限の無視（重要・修正済み）**: ありじごく・ねをはる・かげふみ・じりょく・バインド・にげられない・フェアリーロック等の交代制限は無視して交代可能。`battle.query.can_switch()` はとらわれ状態を理由に `False` を返すため使用禁止。バトンタッチの `バトンタッチ_check`／`バトンタッチ_apply` と同様、`state.bench` の生存有無のみを直接判定して `Interrupt.PIVOT` を設定する

- **交代先がいない場合**: 控え（選出済みで場のポケモン以外）に生存個体が1匹もいない場合は PIVOT を設定しない。ランク低下のみ発動し、使用者は場に残る

- **Interrupt.PIVOT のインポート**: `move_status.py` はすでに `バトンタッチ_apply` 等で `Interrupt` をインポート済みのため、追加インポートは不要

- **ミラーアーマーで反射された場合も使用者は交代可能**: wiki 仕様「対象の特性ミラーアーマーに効果を跳ね返された場合も使用者は交代できる」。反射処理はイベント発火側で行われ、その後の交代判定は別処理となるため、ハンドラ内の PIVOT 設定は通常どおり実行される

- **のどスプレーとの厳密な消費タイミング（対象外）**: 一次情報では「交代効果が発動したときはのどスプレーを消費しない」とあるが、アイテム側は `ON_MOVE_END`（技成功時に一律発動）で汎用実装されており、PIVOT による実際の交代は `ON_MOVE_END` より後の `turn_controller` フェーズで行われる。この細かい消費タイミングの区別は本プロジェクトでは再現しない（交代によりランクはリセットされるため実害は無い）
