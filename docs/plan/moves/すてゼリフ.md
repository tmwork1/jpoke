# 計画書: すてゼリフ

## 仕様要約

- タイプ: あく / カテゴリ: 変化 / PP: 20 / 命中: 100 / 対象: 1体選択（相手）
- 音技（labels=["sound"]）— みがわりを貫通、ぼうおんで無効
- 相手のこうげきととくこうをそれぞれ1段階下げる
- その後、使用者が控えのポケモンと交代する（自発交代）
- 交代制限（ありじごく・ねをはる・かげふみ等）を無視して交代可能
- 控えポケモンが存在しない場合はランク低下のみ発動し交代は発生しない
- ランク低下が特性等（クリアボディ・しろいきり・クリアチャーム等）で阻まれた場合は交代も発動しない（第七世代以降）
- まもる: ○ / マジックコート: ○（反射可） / よこどり: ×

## Priority 根拠

`docs/spec/turn.md` の `ON_STATUS_HIT` イベントは表に掲載されていない（ダメージ計算外の変化技処理）。  
既存の同種変化技ハンドラ（`おたけび_modify_defender_stats`・`なみだめ_modify_defender_stats`・`さむいギャグ_activate_weather_and_pivot` 等）がすべて priority 100（デフォルト）を使用しているため、同一パターンとして priority 100 を採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: 五十音順「す」行 — 既存の「さむいギャグ」の後、「せいちょう」より前（「し」行と「せ」行の間）に追加する。

```python
def すてゼリフ_modify_defender_stats_and_pivot(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """すてゼリフ: 相手のこうげき・とくこうを1段階下げ、自発交代する。

    ランク低下が成功した場合のみ交代が発動する。
    ランク低下が阻まれた（クリアボディ等）場合は交代も発動しない（第七世代以降）。
    控えポケモンがいない場合はランク低下のみ発動し交代は発生しない。
    """
    result = modify_defender_stats(battle, ctx, value, stats={"A": -1, "C": -1})

    # ランク低下が完全に阻まれた（stop_event=True）場合は交代しない
    if result.stop_event:
        return result

    player = battle.get_player(ctx.attacker)
    if battle.get_available_switch_commands(player):
        battle.player_states[player].interrupt = Interrupt.PIVOT

    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

既存の `"すてゼリフ"` エントリに `handlers` を追加する。

```python
"すてゼリフ": MoveData(

    category="変化",
    pp=20,
    accuracy=100,

    handlers={

        ),
    }

```

## 注意点・エッジケース

- **ランク低下と交代の依存関係（第七世代以降）**: クリアボディ・しろいきり・クリアチャーム等でランク低下が阻まれた場合は交代も発動しない。`modify_defender_stats` が `stop_event=True` を返した場合に交代処理をスキップすることで実現する。ランク低下が一部だけ阻まれた（どちらか一方のみ阻止）場合の挙動は `modify_defender_stats` 内部処理に依存する

- **かいりきバサミへの対応**: かいりきバサミ（こうげき下げ無効）を持つ相手には特攻のみ下がる。wiki 仕様「対象が特性かいりきバサミである場合は特攻のみ下がる」。これは `modify_defender_stats` 経由の `ON_TRY_MOVE_2` priority 140 でかいりきバサミ判定が行われ、こうげき低下のみキャンセルされることで自動処理される。ハンドラ側での追加対応は不要

- **まけんき・かちき持ちへの逆効果**: まけんきはこうげき、かちきはとくこうが1段階下がるごとに特性が発動する。すてゼリフでは2箇所のランクを下げるため特性が計2回発動しうる。ランク変化ハンドラ経由で自動処理されるためハンドラ側での追加対応は不要

- **音技の性質**: `labels=["sound"]` により、みがわり状態の相手にもランク低下が通る（みがわり貫通）。ぼうおん特性を持つ相手には `ON_TRY_MOVE_1` priority 110 で技が無効化され、`ON_STATUS_HIT` は発火しない。ハンドラ側のガードは不要

- **マジックコートによる反射**: 相手がマジックコート状態の場合、`ON_TRY_MOVE_1` priority 100 で反射処理が行われ、使用者の攻撃・特攻が1段階下がりマジックコート使用者が交代する。これはイベント発火側で処理されるためハンドラ側での対応は不要

- **交代制限の無視**: ありじごく・ねをはる・かげふみ・じりょく等の交代制限は無視して交代可能。自発交代技（PIVOT）はこれらの制限を回避して交代するため、ハンドラ側での追加処理は不要（turn_controller 側で対応済み）

- **交代先がいない場合**: `battle.get_available_switch_commands(player)` が空の場合は PIVOT を設定しない。ランク低下のみ発動し、使用者は場に残る

- **Interrupt.PIVOT のインポート**: `move_status.py` はすでに `さむいギャグ_activate_weather_and_pivot` で `Interrupt` をインポート済みのため、追加インポートは不要

- **ミラーアーマーで反射された場合も使用者は交代可能**: wiki 仕様「対象の特性ミラーアーマーに効果を跳ね返された場合も使用者は交代できる」。反射処理は `ON_APPLY_MOVE` priority 30 で行われ、その後の交代判定は別処理となるため、ハンドラ内の PIVOT 設定は通常通り実行される
