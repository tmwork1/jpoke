# 実装計画書: アロマセラピー

## 仕様要約

- タイプ: くさ / カテゴリ: 変化 / PP: 5 / 命中: なし（必中） / 対象: 味方全体
- 自分と味方全体（選出チーム、控え含む）の状態異常を治す
- 使用者側で状態異常のポケモンが誰もいない場合は技が失敗する（`ON_APPLY_MOVE` priority=100:
  「アロマセラピー等: 状態異常味方なし」）
- 音技ではないため、ぼうおんの影響を受けない
- `heal` ラベルなし（`かいふくふうじ` の対象外）
- ダイウォールの効果を受けない（本プロジェクトはダイマックス対象外）

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自チームに状態異常のポケモンがいない場合に失敗させる → いる場合は全員の状態異常を回復する |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` セクション（コード上の `ON_STATUS_HIT` に対応）に
以下の記載がある：

> | 100 | アロマセラピー等: 状態異常味方なし |

同種の技「いやしのすず」も `ON_STATUS_HIT`（priority=100、デフォルト）内で失敗判定と回復処理を
一括して行うパターンを採用しており（`いやしのすず_cure_ailment` 参照）、アロマセラピーも同じ
パターンを踏襲する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「あ」行 — `あやしいひかり_apply` の後、`アンコール_apply` の前に挿入する。

```python
def アロマセラピー_cure_team_ailment(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """アロマセラピー: 自分（シングルバトルでは選出チーム）の状態異常を回復する。

    チームに状態異常のポケモンがいない場合は技が失敗する。
    """
    mon = ctx.attacker
    player = battle.get_player(mon)
    state = battle.player_states[player]
    targets = [m for m in state.selection if m.ailment.is_active]
    if not targets:
        return HandlerReturn(value=False, stop_event=True)
    for target in targets:
        battle.ailment_manager.remove(target)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/moves/move_a.py`

```python
"アロマセラピー": MoveData(
    type="くさ",
    category="status",
    pp=5,
    target="own_side",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.アロマセラピー_cure_team_ailment,
        ),
    },

```

## 注意点・エッジケース

- **`target="own_side"`**: 味方全体を対象とする効果のため、`いやしのすず`・`おいかぜ` 等と同様に
  `target="self"` ではなく `target="own_side"` を用いる。`is_blocked_by_protect`（`target == "foe"` のみ
  True）・`is_reflectable`（`target in ("foe", "foe_side")` のみ True）のいずれも `own_side` は該当しない
  ため、まもる・マジックコートの判定に影響はない。
- **シングルバトル限定実装**: `state.selection` はシングルバトルでは選出した控えを含むチーム全体。
  「チームに状態異常がいなければ失敗」の判定と「全員を治療するループ」を実装することで、
  ダブルバトル拡張時にも対応できる構造にしておく。
- **チームに状態異常がいない場合の失敗**: `ON_STATUS_HIT` 内でチェックし、
  `HandlerReturn(value=False, stop_event=True)` を返すと `move_executor.py` 内で `MOVE_FAILED` ログが
  記録される。
- **そうしょくとの相互作用**: `target != "foe"` のため `_apply_type_absorb`（そうしょくの実装）は
  何もせずスキップする。自分に使う本技に対してそうしょくが発動することはない
  （`docs/spec/abilities/そうしょく.md` 参照）。
- **ぜったいねむりとの相互作用**: `ailment_manager.remove()` は `uncurable` フラグを持つ「ゆめうつつ」を
  解除しない。ハンドラ側での追加チェックは不要（内部で保護されている）。
- **かいふくふうじとの相互作用**: `heal` ラベルが付いていないため、`かいふくふうじ` 状態でも使用可能。
  ハンドラ側での追加チェックは不要。
- **いやしのすずとの違い**: いやしのすず（ノーマルタイプ）は音技（`sound` ラベル）だが、アロマセラピー
  （くさタイプ）は音技ではない。機能はほぼ同等。
