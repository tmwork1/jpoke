# 実装計画書: トリックルーム

## 仕様

### 技データ

- タイプ: エスパー
- カテゴリ: 変化（`category="status"`）
- 威力/命中率: なし（`power`/`accuracy` とも未設定 = 常に命中）
- PP: 8（championsルールにより8/12/16/20の4段階へ統一。`src/jpoke/data/ps-champ-ja/moves.json` 準拠。
  本家 Gen4〜9 では PP5）
- 優先度: -7（全ての技の中で最低）
- 対象: 全体の場（`target="field"`。デフォルト値 `"foe"` のままだと `まもる`/`マジックコート` の
  判定が `move.target == "foe"` を前提にしているため誤って防がれる/跳ね返されてしまう）
- 直接攻撃: しない（`flags` に `"contact"` を含めない）

### 効果

場を『トリックルーム』状態（5ターン、発動ターンを含む）にする。すでに発動中の場合は
再使用によって即座に解除される（トグル動作）。フィールド自体の詳細効果（素早さ反転・
カウントダウン）は `docs/plan/fields/トリックルーム.md` / `docs/spec/fields/トリックルーム.md`
を参照。技自体はフィールドの発動・解除のみを行う。

### 参照仕様書

- `docs/spec/moves/トリックルーム.md`
- `docs/spec/fields/トリックルーム.md`

## subject_spec と priority

### イベント: `ON_STATUS_HIT`

- **ctx型**: `AttackContext`（変化技の `ON_STATUS_HIT` は攻撃フローとして発火するため
  `ctx.attacker` / `ctx.defender` を参照する）
- **subject_spec**: `"attacker:self"`（`MoveHandler` のデフォルト値を使用）
- **priority**: 100

**根拠**: `ON_STATUS_HIT` は `docs/spec/turn.md` に独立エントリが存在しない。同種の
全体の場を発動する変化技 `じゅうりょく_activate_global_field`
（`h.MoveHandler(hs.じゅうりょく_activate_global_field)`）も priority=100 のデフォルトを
使用しているため、これと同一パターンとして priority=100 を採用する。

## ハンドラ構成

### 1. `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: `とける_modify_attacker_stats` の後、`トーチカ_apply` の前。

```python
def トリックルーム_activate_global_field(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    manager = battle.global_manager
    if manager.fields["トリックルーム"].is_active:
        return HandlerReturn(value=manager.deactivate("トリックルーム"))
    return HandlerReturn(value=manager.activate("トリックルーム", 5))
```

- すでに『トリックルーム』状態が有効な場合、`deactivate` でフィールドを即座に解除する
  （じゅうりょくとは異なり、再使用による解除効果がある）。
- 未発動の場合は `activate` で5ターン（発動ターンを含む）発動する。

### 2. `src/jpoke/data/moves/move_ta.py`

`"トリックルーム"` エントリ:

```python
"トリックルーム": MoveData(
    type="エスパー",
    category="status",
    pp=8,
    priority=-7,
    target="field",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.トリックルーム_activate_global_field,
        ),
    }
),
```

### 3. `src/jpoke/data/field/global_field.py`

フィールド本体の効果（素早さ反転・カウントダウン）は既存の `GLOBAL_FIELD["トリックルーム"]` に
実装済み。詳細は `docs/plan/fields/トリックルーム.md` を参照。

## 注意事項

- `target="field"` を明示しないと `MoveData.target` のデフォルト値 `"foe"` になり、
  `is_blocked_by_protect`（`まもる`）・`マジックコート_reflect` の判定が `move.target == "foe"` を
  条件にしているため、誤って「まもるで防がれる」「マジックコートで跳ね返される」動作になる
  （wiki の判定表でも まもる: ×、マジックコート: × と明記されている）
- 技の優先度（-7）は `MoveData.priority` に設定するのみでよく、行動順序の計算（優先度→後だし
  ティア→素早さ）は `SpeedCalculator.resolve_action_order` が共通処理する
- トリックルームによる素早さ反転自体は技ハンドラの責務ではなく、フィールドハンドラ
  （`DomainEvent.ON_CHECK_SPEED_REVERSE`）の責務。技ハンドラはフィールドの発動・解除のみを行う
