# 計画書: トライアタック

## 仕様要約

- ノーマルタイプ / 特殊 / 威力80 / 命中100 / PP12（Champions基準） / 対象: 相手1体 / 直接攻撃なし
- ダメージ後、20%の確率で相手を『まひ』『やけど』『こおり』のいずれかの状態にする
- まひ・やけど・こおりは各1/3。単一乱数で判定する：
  `r < base/3` → まひ、`base/3 <= r < base*2/3` → やけど、`base*2/3 <= r < base` → こおり
- 詳細仕様は `docs/spec/moves/トライアタック.md` を参照

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | デフォルト | attacker:self（省略時デフォルト） | 20%でまひ/やけど/こおりをランダム付与 |

## Priority 根拠

`ON_DAMAGE_HIT` は `docs/spec/turn.md` に未掲載のダメージ計算内部イベント（技命中直後・ダメージ確定後に発火）。
同種の追加効果ハンドラ（かえんほうしゃ、かみなり等の状態異常付与ハンドラ）に倣い、デフォルト priority を使用する。

## 実装箇所（実装済み）

### handlers/move_attack.py

```python
def トライアタック_apply_ailment_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """トライアタックの追加効果: 20%の確率でまひ/やけど/こおりのいずれかを付与する。

    単一乱数で判定: r < base/3 → まひ、r < base*2/3 → やけど、r < base → こおり。
    """
    base = battle.resolve_secondary_chance(ctx, 0.2)
    r = battle.random.random()
    if r < base / 3:
        ailment = "まひ"
    elif r < base * 2 / 3:
        ailment = "やけど"
    elif r < base:
        ailment = "こおり"
    else:
        return HandlerReturn(value=value)
    return HandlerReturn(value=battle.ailment_manager.apply(
        ctx.defender, ailment, source=ctx.attacker, ctx=ctx
    ))
```

## 注意点・エッジケース

- PP は Champions 基準で 12（`docs/champions/move_list.txt` 準拠）。Gen9本家の10ではない。
- `secondary_effect` フラグにより、ちからずく特性（威力1.3倍・追加効果無効化）、りんぷん特性・おんみつマント
  （追加効果無効化）、てんのめぐみ（発動確率2倍）は `resolve_secondary_chance` を通じて汎用的に処理される。
  ハンドラ側に個別のチェックは不要。
- まひ/やけど/こおりの各1/3は「単一乱数で切り分け」であり、3回独立した判定ではない。
- 各状態異常の付与に `ailment_manager.apply` を使うため、すでに状態異常の場合・タイプ免疫
  （でんきタイプ→まひ免疫等は本プロジェクト対象外の仕様だが、ほのおタイプ→やけど免疫、こおりタイプ→こおり免疫等）・
  みずのベール等の特性チェックは ailment_manager 内部で自動処理される。
- みがわり状態の相手に技が防がれた場合、ダメージが0になり `ON_DAMAGE_HIT` 自体が発火しないため
  ハンドラ側にみがわりチェックは不要。
- リーサル計算対象外（`docs/progress/move.md` のリーサル実装列が n/a）。状態異常付与は確率分岐が複雑で
  期待値計算のリーサル計算対象としないため、リーサルハンドラは実装しない。
