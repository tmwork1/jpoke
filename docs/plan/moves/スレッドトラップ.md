# 計画書: スレッドトラップ

対象: Pokémon Champions シングルバトル
仕様書: `docs/spec/moves/スレッドトラップ.md`（技本体）、
`docs/spec/volatiles/スレッドトラップ.md`（付与される揮発性状態）

## 概要

スレッドトラップは既に実装済み。本計画書は実装の意図・設計根拠を記録する
（レビュー時点でバグは見つからなかった。`docs/spec/moves/`・`docs/plan/moves/` が未作成
だったため新規作成した）。

## 技データ (`data/moves/move_sa.py`)

```python
"スレッドトラップ": MoveData(
    type="むし",
    category="status",
    pp=10,
    priority=4,
    target="self",
    flags={"protect"},
    handlers={
        Event.ON_TRY_MOVE_2: h.MoveHandler(
            hs.まもる系_連続使用失敗チェック,
        ),
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.スレッドトラップ_apply,
        ),
    }
),
```

- `pp=10`: `docs/champions/move_list.txt` にスレッドトラップ自体のエントリが存在しないため
  （ワナイダー専用技で champions 図鑑に含まれない可能性がある。`はいすいのじん` と同様の扱い）、
  第九世代本家の値をそのまま維持する。
- `flags={"protect"}`: 「まもる系技」共通の連続使用失敗判定（後述）で使用するラベル。
  この `protect` フラグは `まねっこ_can_use`（`handlers/move_status.py`）でも
  参照され、守る系技を一律まねっこのコピー対象から除外する。まもるとは異なり
  `non_copycat` フラグは不要（`protect` フラグのみで両方の判定を兼ねる）。

## ハンドラ構成 (`handlers/move_status.py`)

```python
def スレッドトラップ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_attacker(battle, ctx, value, volatile="スレッドトラップ")


def まもる系_連続使用失敗チェック(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """守る系技の連続使用失敗チェック（まもる・みきり・こらえる・キングシールド・
    スレッドトラップ等で共有）。"""
    ...
```

- `スレッドトラップ_apply`: `ON_STATUS_HIT` で「スレッドトラップ」揮発状態を自分に付与する。
  まもる・キングシールド等と同一のヘルパー呼び出しパターン。
- `まもる系_連続使用失敗チェック`: まもる・みきり・こらえる・キングシールド・ニードルガード・
  トーチカ・スレッドトラップ・かえんのまもりで共有される既存関数（`docs/plan/moves/まもる.md`
  参照）。スレッドトラップ専用の実装は不要。

## スレッドトラップ volatile (`data/volatile.py` / `handlers/volatile.py`)

スレッドトラップ volatile 自体の設計（`ON_TRY_MOVE_1` priority=100 での攻撃技無効化
（`protect_non_attack=False` により変化技は防がない）・直接攻撃者の素早さランクダウン
（`stats_change_on_contact={"spe": -1}`）、`ON_TURN_END` での解除、`_run_protect` の共通骨格）は
`docs/spec/volatiles/スレッドトラップ.md`・`docs/plan/volatiles/スレッドトラップ.md`（既存レビュー済み）
を参照。本計画書では技側（`スレッドトラップ` MoveData・`スレッドトラップ_apply`）のみを扱う。

```python
def スレッドトラップ_protect(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """スレッドトラップの保護判定。攻撃技のみ防ぎ、接触した相手の素早さランクを1段階下げる"""
    return _run_protect(battle, ctx, value, stats_change_on_contact={"spe": -1}, protect_non_attack=False)


def スレッドトラップ_remove_volatile(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    return remove_volatile(battle, ctx, value, volatile="スレッドトラップ")
```

## Priority 根拠

### `Event.ON_STATUS_HIT`: priority 100（デフォルト）

`docs/spec/turn.md` に `ON_STATUS_HIT` の個別 priority 表がない。既存の変化技ハンドラ
（`まもる_apply`・`みきり_apply`・`こらえる_apply`・`キングシールド_apply` 等）がすべて
priority 100（デフォルト）を使用しているため、同一パターンで 100 を採用する。

### `Event.ON_TRY_MOVE_2`: priority 100（デフォルト）

`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` 30番に「まもる系: 連続使用により失敗」と
明記されているが、既存実装（まもる・みきり・キングシールド・こらえる等）はすべて
`Event.ON_TRY_MOVE_2` のデフォルト priority（100）で実装済みであり、このパターンは
`docs/spec/moves/まもる.md`・`docs/spec/moves/キングシールド.md`（レビュー済み）でも
踏襲することが確認されている。本レビューでも既存パターンを維持する。

## スコープ外（既知の限界）

まもる系共通の限界（連続使用による失敗率の確率モデル未実装、`executed_move` の
行動スキップ時の非リセット等）は `docs/spec/moves/まもる.md` を参照。スレッドトラップ固有の
追加スコープ外事項はない。

ダイマックスわざ・Zワザは第9世代（champions基準）に存在しないため対象外。

## テスト方針

- スレッドトラップを使うとスレッドトラップ状態が付与されること（新規追加:
  `test_スレッドトラップ_技使用でスレッドトラップ状態が付与される`、
  `tests/moves_status/test_move_sa.py`。従来はニードルガード等の兄弟技（`test_move_na.py`）が
  個別に持つ「揮発状態が付与される」テストが `test_move_sa.py` 側に欠落していたため追加）
- スレッドトラップを実際に使用した同ターンに相手の直接攻撃を無効化しつつ攻撃者の『すばやさ』
  ランクを下げること（新規追加: `test_スレッドトラップ_使用した同ターンに相手の直接攻撃を
  ブロックしてランクを下げる`、`tests/moves_status/test_move_sa.py`。従来は
  `volatile_manager.apply()` で直接付与するテスト（`tests/volatiles/test_volatile_satana.py`）
  のみで、技実行経由の統合テストが欠落していたため、`docs/review/moves/キングシールド.md` で
  追加した統合テストと同じ狙いで追加）
- スレッドトラップを実際に使用した同ターンに相手の変化技は防がないこと（新規追加:
  `test_スレッドトラップ_変化技は防がない`、`tests/moves_status/test_move_sa.py`。同上の理由で追加）
- スレッドトラップを2ターン連続使用すると2ターン目は失敗すること・異なる守る系技を挟んでも
  連続使用扱いになること（既存: まもる系共通テスト、
  `test_まもる系_キングシールドとニードルガードでも相互に失敗する` 等）
- 攻撃技を防ぐこと・変化技は防がないこと・ターン終了で解除されること（既存:
  `tests/volatiles/test_volatile_hamayarawa.py` のまもる系共通パラメトリックテスト）
- 直接攻撃を防ぐと相手の素早さランクが下がること・非接触では下がらないこと（既存:
  `tests/volatiles/test_volatile_satana.py` の
  `test_スレッドトラップ_接触技で素早さを下げる`・`test_スレッドトラップ_非接触では素早さが下がらない`）
- ふかしのこぶし・かんつうドリルで守りを貫通されること（既存:
  `tests/abilities/test_ability__group.py` の `test_ふかしのこぶし_接触技でまもるを貫通`）
- フェイントでスレッドトラップ状態を解除して攻撃が命中すること（既存:
  `tests/moves_attack/test_move_ha.py` の `test_フェイント_まもる系統の状態を解除して攻撃する`）
