# 計画書: パワートリック

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 10 / 命中: - / 対象: 自分
- 使用者自身の「こうげき（A）」と「ぼうぎょ（B）」の**実数値**を入れ替える
- ランク変化（`rank["A"]`・`rank["B"]`）は変化しない。実数値のみを入れ替える
- 命中判定: なし（必中）/ まもる: × / マジックコート: × / よこどり: × / オウムがえし: ×
- 出典: `docs/spec/moves/status_move.md` 205行目
  「パワートリック: 自分の『こうげき』と『ぼうぎょ』を入れ替える。『ランク』は変化しない。」
- 参考: `パワーシフト`（ノーマル・`status_move.md` 203行目）と完全に同一効果だが、別の技として独立実装する

## Priority 根拠

`docs/spec/turn.md` には `ON_STATUS_HIT` の内部 priority 表は掲載されていない。既存の実数値操作ハンドラ（`スピードスワップ_swap_speed`・`ガードシェア_equalize_stats`・`ガードスワップ_swap_ranks` 等）はすべて priority 100（`MoveHandler` デフォルト）を使用している。パワートリックも同一パターンで priority 100 を採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: 五十音順「は」行 — `はらだいこ_apply` の直後、`ハロウィン_can_apply` の直前に挿入する（はらだいこ → パワートリック → ハロウィン の順）。

実数値の入れ替えは `_stats_manager.stats[1]`（インデックス1 = こうげき）と `_stats_manager.stats[2]`（インデックス2 = ぼうぎょ）を直接操作する。`スピードスワップ` が `stats[5]` を直接操作するパターンを踏襲する。`battle.modify_stat()` はランク変化量加算APIであるため「実数値の入れ替え」には使用しない。

```python
def パワートリック_swap_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """パワートリック: 使用者のこうげきとぼうぎょの実数値を入れ替える。

    ランク変化は行わず、実数値のみを入れ替える。
    インデックス 1 = こうげき、インデックス 2 = ぼうぎょ
    （[HP, 攻撃, 防御, 特攻, 特防, 素早さ]）。
    """
    mon = ctx.attacker
    atk = mon._stats_manager.stats[1]
    dfn = mon._stats_manager.stats[2]
    mon._stats_manager.stats[1] = dfn
    mon._stats_manager.stats[2] = atk
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

既存の `"パワートリック"` エントリに `handlers` を追加する。

```python
"パワートリック": MoveData(
    type="エスパー",
    category="変化",
    pp=10,
    accuracy=100,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.パワートリック_swap_stats,
        ),
    }
),
```

## 注意点・エッジケース

- **対象は自分のみ**: スピードスワップ（相手と互いに入れ替え）と異なり、パワートリックは使用者自身のこうげきとぼうぎょのみを入れ替える。`ctx.defender` は参照しない
- **実数値直接操作の根拠**: `スピードスワップ_swap_speed` と同じパターンで `_stats_manager.stats` を直接書き換える。`battle.modify_stat()` はランク変化量加算APIのため「実数値の入れ替え」に使用不可
- **ランク変化は変わらない**: `mon.rank["A"]`・`mon.rank["B"]` は変化しない。ダメージ計算時には実数値にランク補正がかかる
- **ON_MODIFY_STAT 非発火**: 直接代入のため、ランク変化に反応する特性（かちき・まけんき等）は発動しない。これはゲーム仕様上も正しい（パワートリックは「ランク変化」ではなく「実数値交換」の技）
- **再度使用で元に戻る**: パワートリックを2回使用すると実数値が元の状態に戻る（入れ替え状態は volatile として管理せず、実数値のみを操作するため自動的に対称性が保たれる）。ただし入れ替え状態の追跡（表示用ログ等）が将来必要になった場合は volatile 管理への移行を検討する
- **交代による解除**: ゲーム仕様では、場を離れると入れ替えた実数値が元に戻る。現状の実装（実数値の直接書き換え）では交代時に元の値へ戻す処理が必要になるため、将来的な実装では volatile での管理が望ましい。シミュレーションの現状スコープでは交代時の復元は未実装で問題ない場合は計画通り進める
- **ワンダールーム中**: ワンダールームはぼうぎょとくぼうの参照を入れ替えるが、実数値レベルの操作であるパワートリックと干渉した場合の挙動は複雑になる（入れ替えの組み合わせ）。実装シンプル化のため両立する場合は別途検討する
- **パワーシフトとの関係**: `status_move.md` の記述上はほぼ同一効果（パワーシフトはノーマルタイプ・対象「自分」）だが、別の技として独立したハンドラ関数を実装する。`パワーシフト` 自体は `data/move.py` への登録も未実施のため、将来実装する際に共通関数化を検討してもよい
- **`data/move.py` の `accuracy=100` について**: 他の変化技に倣った既存の設定値であり、命中判定は発生しない（仕様上の命中率フィールドは `-`）
