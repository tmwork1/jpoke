# 計画書: レイジングブル

## 仕様要約

- 物理・ノーマル技（威力90・命中100・PP12〔`docs/champions/move_list.txt` 準拠。本家シリーズのPPは10〕・直接攻撃・優先度0）
- ケンタロス（パルデア）が使用したとき、フォルムに応じて技のタイプが変化する
  - ケンタロス（通常）: ノーマル
  - ケンタロス（パルデア闘 / コンバット種）: かくとう
  - ケンタロス（パルデア炎 / ブレイズ種）: ほのお
  - ケンタロス（パルデア水 / ウォーター種）: みず
- 攻撃が命中した際、相手側の「リフレクター」「ひかりのかべ」「オーロラベール」を解除する
- 壁解除は追加効果ではない（りんぷん・おんみつマント・ちからずくの対象外）
- タイプ相性/特性/まもる等で技が無効化された場合は壁を解除できない
- みずびたし・テラスタル等でケンタロス自身のタイプが変更されても技のタイプは変わらない
- ケンタロス以外のポケモンがものまね等で使用した場合はノーマルタイプになる
- ケンタロスが使用した場合は「技のタイプを変える特性」（ノーマルスキン等）の効果を受けない

## ハンドラ構成

| イベント | Priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_MODIFY_MOVE_TYPE` | 100（デフォルト） | `attacker:self` | ケンタロスのフォルムに応じて技タイプを返す |
| `ON_HIT` | 100（デフォルト） | `attacker:self` | 相手側の壁3種を解除する |

## Priority 根拠

### ON_MODIFY_MOVE_TYPE

`docs/spec/turn.md` には `ON_MODIFY_MOVE_TYPE` のテーブルは存在しない。
既存の同イベントハンドラ（`オーラぐるま_check_move_type`、`テラバースト_modify_move_type`）はすべて priority=100（デフォルト）で登録されている。これを根拠として priority=100 とする。

### ON_HIT（壁解除）

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE`（実装名: `ON_BEFORE_APPLY_MOVE`）に priority=30 で「かわらわり等の効果発動」が記載されている。

しかし仕様書に「相性/特性/まもる等で無効化されたときは壁を解除できない」とある。コード上、`ON_BEFORE_APPLY_MOVE` は命中判定後・ダメージ処理前に発火し、このイベントで `True` を返す（無効化しない）か否かを決定する。実際にダメージを与えた後に壁解除する「ヒット確認後の副作用」として `ON_HIT` の方が適切である。

類似実装として `こうそくスピン_clear_field`（設置物解除）が `ON_HIT priority=100` で実装されており、これを根拠として `ON_HIT priority=100` とする。

> 補足: turn.md の `ON_APPLY_MOVE priority=30` は「かわらわり等」の壁解除判定タイミングを指す可能性があるが、コードには `ON_APPLY_MOVE` イベントが存在せず、かわらわり自体もハンドラ未実装のため、現時点では `ON_HIT` ベースで実装する。

## 実装箇所

### handlers/move_attack.py

五十音順で「れ」の位置（`レイジングブル` = re → `る` のあたり、既存末尾付近）に追加する。

```python
# フォルム名 → 技タイプの対応表
_RAGING_BULL_TYPE_MAP: dict[str, str] = {
    "ケンタロス":           "ノーマル",
    "ケンタロス(パルデア闘)": "かくとう",
    "ケンタロス(パルデア炎)": "ほのお",
    "ケンタロス(パルデア水)": "みず",
}

def レイジングブル_modify_move_type(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """レイジングブルのタイプをケンタロスのフォルムで決定する。

    ケンタロス以外が使用した場合はノーマルタイプのまま変化しない。
    """
    move_type = _RAGING_BULL_TYPE_MAP.get(ctx.attacker.name)
    if move_type is not None:
        value = move_type
    return HandlerReturn(value=value)


def レイジングブル_break_screens(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """レイジングブル: 相手側のリフレクター・ひかりのかべ・オーロラベールを解除する。

    壁解除は追加効果ではないため、りんぷん・おんみつマント・ちからずくの対象外。
    技が無効化されなかった場合（ON_HITに到達した場合）のみ発動する。
    """
    defender_side = battle.get_side(ctx.defender)
    for wall in ("リフレクター", "ひかりのかべ", "オーロラベール"):
        defender_side.deactivate(wall)
    return HandlerReturn(value=value)
```

### data/move.py

`"レイジングブル"` のエントリを以下のように更新する（`ha` は `handlers.move_attack` のエイリアス）。

## 注意点・エッジケース（review-test 向け）

### タイプ判定

- ケンタロス（通常）は `pokemon.name == "ケンタロス"` でノーマルタイプになる（`_RAGING_BULL_TYPE_MAP` に含める）
- ケンタロス以外（ものまね・まねっこ等）は `_RAGING_BULL_TYPE_MAP.get()` が `None` を返し、技のデフォルトタイプ（ノーマル）のまま変化しない
- フォルム変化後（`Pokemon.set_form` により `data` が差し替わる）も `pokemon.name` で正しく判定できる
- `ON_MODIFY_MOVE_TYPE` は特性（ノーマルスキン等）と同じイベントで発火するが、ケンタロスが使用した場合に特性側の変換が適用されないようにする必要がある。ただし、仕様「ケンタロスが使用したときは技のタイプを変える特性の効果を受けない」の実装は特性側ハンドラの責務であり、本ハンドラでは対応不要

### 壁解除タイミング

- `ON_HIT` は実際にダメージが通った後に発火するため、タイプ相性無効・特性無効・まもる等で無効化された場合は自動的に壁解除されない（イベント自体が発火しない）
- みがわり状態でも `ON_HIT` は発火するため（コメント「みがわりに被弾しても発動」）、みがわりがいる場合も壁解除が実行される点に注意（仕様には特段の例外記述なし）
- `SideFieldManager.deactivate` は対象フィールドが存在しない場合は何もしないため、壁が張られていないケースのガード不要

### 壁解除は追加効果でない点

- `ON_MODIFY_SECONDARY_CHANCE` を経由しないため `ちからずく` の威力上昇対象にならない
- 同様に `りんぷん`・`おんみつマント` で防がれない
- 本ハンドラを `labels` に `"secondary_effect"` を付けないことで正しく区別できる

### テストケース例

1. ケンタロス（パルデア炎）がレイジングブルを使うと技タイプが「ほのお」になる
2. ケンタロス（通常）がレイジングブルを使うと技タイプが「ノーマル」になる
3. ケンタロス以外がものまねでレイジングブルを使うと技タイプが「ノーマル」のまま
4. レイジングブルが命中後、相手側のリフレクター・ひかりのかべ・オーロラベールが解除される
5. まもるで無効化されたときは壁が解除されない
6. 壁が張られていない場合でもエラーなく動作する
