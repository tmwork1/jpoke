# 計画書: つめとぎ

## 仕様要約

- 自分の「こうげき」「命中率」ランクをそれぞれ1段階上げる
- タイプ: あく、分類: 変化、PP: 16（第9世代PP15からの移行推定値。詳細は仕様書参照）、対象: 自分
- まもる・マジックコートの影響を受けない（自分が対象のため）

出典: `docs/spec/moves/つめとぎ.md`

---

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100 (省略) | `attacker:self` (省略) | こうげき・命中率を1段階ずつ上げる |

---

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`つるぎのまい_modify_attacker_stats`, `てっぺき_modify_attacker_stats` 等）が
すべて priority=100（デフォルト）で登録されているため、100 を使用する。

---

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: `つぼをつく_modify_attacker_stats` の後、`つるぎのまい_modify_attacker_stats` の前
（つ〜つぼ〜つめ〜つる の間、五十音順）

```python
def つめとぎ_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """つめとぎ: 自分のこうげき・命中率ランクをそれぞれ1段階上げる。"""
    return modify_attacker_stats(battle, ctx, value, stats={"atk": 1, "accuracy": 1})
```

### `src/jpoke/data/moves/move_ta.py`

`"つめとぎ"` エントリの `pp` と `target`、`handlers` を以下に更新する:

```python
    "つめとぎ": MoveData(
        type="あく",
        category="status",
        pp=16,
        target="self",
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.つめとぎ_modify_attacker_stats,
            ),
        }
    ),
```

---

## 注意点・エッジケース

- `target="self"` を明示しないと `MoveData.target` のデフォルト `"foe"` が使われてしまい、
  `move.is_blocked_by_protect`（`target == "foe"` で判定）や `マジックコート_reflect`
  （`ctx.move.target in {"foe", "foe_side"}` で判定）が誤って True になる。
  自分が対象の技なので `target="self"` を必ず設定する。
- ランクがすでに最大（+6）の場合の失敗は `turn.md` Priority 130「ランク補正上げ: すでに最大」で
  マネージャー内部処理されるため、ハンドラ側にガード不要
- PPは `docs/champions/move_list.txt` に本技のエントリがないため、同じ第9世代PP15を持つ他技
  （あくのはどう・シャドーボール・10まんボルト、いずれもmove_list.txt上でPP16）からの類推でPP16とする
