# びりびりちくちく 実装計画書

## 仕様要約（docs/spec/moves/びりびりちくちく.md 参照）

* タイプ: でんき / 分類: 物理 / 威力: 80 / 命中: 100 / PP: 10
* 対象: 相手1体 / 優先度: 0 / 直接攻撃（接触）: あり
* 効果: 追加効果として、30%の確率で相手をひるみ状態にする。

## 現状の実装状況

`src/jpoke/data/moves/move_ha.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。
仕様と一致しており修正不要。

```python
"びりびりちくちく": MoveData(
    type="でんき",
    category="physical",
    pp=10,
    power=80,
    accuracy=100,
    flags={"contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.びりびりちくちく_apply_flinch,
        )
    }
),
```

```python
def びりびりちくちく_apply_flinch(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_defender(battle, ctx, value, volatile="ひるみ", chance=0.3)
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | デフォルト（`MoveHandler` 既定値） | attacker:self | 30%の確率でひるみを付与する |

`_move_secondary_1.md` に記載されたひるみ付与技（いわなだれ・つららおとし・ひょうざんおろし等）と
同じイベント（`ON_DAMAGE_HIT`）・共通ヘルパー（`apply_volatile_to_defender`）を使う定型実装のため、
priority は既存の同種ハンドラに合わせる。

## Priority 根拠

`docs/spec/turn.md` に `ON_DAMAGE_HIT` は掲載されていない（ダメージ計算内部のイベントのため）。
既存の同種ハンドラ（`ひょうざんおろし_apply_flinch`・`アイアンヘッド_apply_flinch` 等）がいずれも
`MoveHandler` のデフォルト priority で `Event.ON_DAMAGE_HIT` に登録されているため、
本技も同じ方針（デフォルト priority）を踏襲する。

## 注意点・エッジケース（review-test 向け）

1. **基本ケース**
   - 通常攻撃時、追加効果として 30% の確率で相手がひるみ状態になる。
2. **アイテム相互作用**
   - `handlers/item.py` の `_INNATE_FLINCH_MOVES` に本技が含まれており、おうじゃのしるし・
     するどいキバによる追加ひるみ効果が重複しない（元々ひるみを追加効果に持つ技のため）。
     既存のパラメータ化テスト（`tests/test_item.py`）で一般的に検証済み。
3. **直接攻撃であること**
   - `flags` に `contact` を含むため、ゴツゴツメット等の接触依存効果が発動する。
4. **リーサル計算**
   - 対象外（`docs/progress/move.md` のリーサル実装列が `n/a`）。単発威力技で追加効果が
     ひるみ（HP に影響しない）のため、リーサル計算ハンドラは不要という既存方針
     （いわなだれ・つららおとし・ひょうざんおろし等と同様）に合致する。
