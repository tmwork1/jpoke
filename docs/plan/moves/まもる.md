# 計画書: まもる

対象: Pokémon Champions シングルバトル
仕様書: `docs/spec/moves/まもる.md`（技本体）、`docs/spec/volatiles/まもる.md`（付与される揮発性状態）

## 概要

まもるは既に実装済み。本計画書は実装の意図・設計根拠を記録する（レビュー時点で PP と
まねっこ耐性フラグの2点に誤り・欠落があったため修正した）。

## 技データ (`data/moves/move_ma.py`)

```python
"まもる": MoveData(
    type="ノーマル",
    category="status",
    pp=8,
    priority=4,
    target="self",
    flags={"protect", "non_copycat"},  # まねっこでコピー不可
    handlers={
        Event.ON_TRY_MOVE_2: h.MoveHandler(
            hs.まもる系_連続使用失敗チェック,
        ),
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.まもる_apply,
        ),
    }
),
```

- `pp=8`: champions基準（`docs/champions/move_list.txt` 862行目）。Gen9本家は10だが、
  champions は全技PPを8/12/16/20の4段階に統一しているため8を採用する
  （`docs/champions/changes_from_sv.md` 参照）。
- `flags={"protect", "non_copycat"}`: `protect` は「まもる系技」共通の連続使用失敗判定
  （後述）で使用するラベル。`non_copycat` はポケモンWiki（`docs/wiki/moves/まもる.html`）に
  明記されている「まねっこでコピーできない」仕様を反映する（レビューで追加）。

## ハンドラ構成 (`handlers/move_status.py`)

```python
def まもる_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_attacker(battle, ctx, value, volatile="まもる")


def まもる系_連続使用失敗チェック(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """守る系技の連続使用失敗チェック。

    直前ターンに守る系の技（protect フラグを持つ技）を正常に使用していた場合、
    今ターンの守る系技を失敗させる。
    失敗時は executed_move を None にリセットして次ターンは再度使えるようにする。
    """
    mon = ctx.attacker
    if (
        mon.executed_move is not None
        and mon.executed_move.has_flag("protect")
    ):
        battle.add_event_log(
            mon, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="まもる系_連続使用")
        )
        mon.executed_move = None
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

- `まもる_apply`: `ON_STATUS_HIT` で「まもる」揮発状態を自分に付与する。みきりも同一の
  ヘルパー呼び出しで同じ volatile を付与する（`みきり_apply`）。
- `まもる系_連続使用失敗チェック`: `protect` フラグを持つ技（まもる・みきり・こらえる・
  キングシールド・ニードルガード・トーチカ・スレッドトラップ・かえんのまもり）すべてで共有される。
  `mon.executed_move` は `MoveExecutor._execute_move` の `ON_TRY_MOVE_2` 通過後に更新される
  ため、直前ターンに `protect` フラグ技が成功した場合のみ `True` を返す。

## まもる volatile (`data/volatile.py` / `handlers/volatile.py`)

まもる volatile 自体の設計（`ON_TRY_MOVE_1` priority=100 での技無効化、`ON_TURN_END` での解除、
`_run_protect` / `_check_protect_success` の共通骨格）は `docs/plan/volatiles/まもる.md` を参照。
本計画書では技側（`まもる` MoveData・`まもる_apply`・連続使用失敗判定）のみを扱う。

## Priority 根拠

### `Event.ON_STATUS_HIT`: priority 100（デフォルト）

`docs/spec/turn.md` に `ON_STATUS_HIT` の個別 priority 表がない。既存の変化技ハンドラ
（`まもる_apply`・`みきり_apply`・`トーチカ_apply` 等）がすべて priority 100（デフォルト）を
使用しているため、同一パターンで 100 を採用する。

### `Event.ON_TRY_MOVE_2`: priority 100（デフォルト）

`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` 30番に「まもる系: 連続使用により失敗」と
明記されているが、既存実装（まもる・みきり・キングシールド・こらえる等）はすべて
`Event.ON_TRY_MOVE_2` のデフォルト priority（100）で実装済みであり、このパターンは
`docs/spec/moves/こらえる.md`（レビュー済み）でも踏襲することが確認されている。
本レビューでも既存パターンを維持し、turn.md との priority 番号の相違は「仕様書の分類上の
参考値」であり実装上の実害はない（`ON_TRY_MOVE_1`/`ON_TRY_MOVE_2` いずれも
「発動成功判定」フェーズであり、`protect` フラグ技同士の相対順序にのみ依存する処理のため）
と判断する。

## スコープ外（既知の限界）

### 連続使用による失敗率の確率モデル

実際の第六世代以降の仕様は「連続で使うごとに成功率が1/3になっていく（下限1/729）」という
確率モデルだが、本プロジェクトは「直前ターンに成功していれば必ず失敗する」という簡略化モデルを
採用する。厳密な確率モデルを実装する場合は:

- `Pokemon` に `protect_streak: int = 0` 等のフィールドを追加
- 成功時: `protect_streak += 1`（上限6）、失敗時・非連続時・交代時: `protect_streak = 0`
- `(1/3) ** min(protect_streak, 6)` の確率で `battle.random.random()` によって成功判定

を行う必要があるが、まもる系技全般（8技）に影響する大規模な変更であり、既存レビュー
（`docs/review/moves/こらえる.md` 等）でも一貫してスコープ外としているため、本レビューでも
踏襲する。

### `executed_move` の行動スキップ時の非リセット

まひ・ねむり・ひるみ等で `ON_TRY_ACTION` が失敗し行動自体ができなかった場合、
`mon.executed_move` は更新されない（前ターン以前の値が残る）。本来は「動けなかったターンを
挟むと連続使用とみなされない」仕様だが、この場合は直前の `protect` フラグ技の記録が
残ったままになり、まれに誤って連続使用扱いになる可能性がある。`executed_move` は
アンコール・いちゃもん等でも「直前に実行された技（ターンをまたいでも可）」として共有利用
されているため、まもる専用に意味を変えることは影響範囲が大きく、本レビューではスコープ外
（既知の限界として仕様書に記録）とする。

## テスト方針

- まもるを2ターン連続使用すると2ターン目は失敗すること（既存: `test_まもる系_2ターン目は失敗する`）
- 2ターン目失敗後、3ターン目は再び成功すること（既存: `test_まもる系_2ターン目失敗後3ターン目は再使用できる`）
- 異なる守る系技（みきり・キングシールド・ニードルガード等）を挟んでも連続使用扱いになること（既存）
- 別技を挟むと連続使用がリセットされること（既存）
- まもるを実際に使用したターンに、相手の攻撃技が同ターン内でブロックされること（レビューで追加）
- まねっこでまもるをコピーできないこと（レビューで追加）
