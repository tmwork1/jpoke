# 計画書: ガードシェア

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 12（Champions基準。docs/champions/move_list.txt参照） / 命中: 必中 / 対象: 相手
- 使用者と相手の「ぼうぎょ（B）」「とくぼう（D）」の**実数値**を平均化する
- ランク変化は一切行わない（実数値のみ書き換える）
- 平均化の計算式: `(使用者のB + 相手のB) // 2` を両者のBに、Dも同様
- まもる: ○（防がれる） / マジックコート: ×（反射不可） / 命中判定: 必中（accuracy=None、命中率calc自体を行わない）
- 交代で実数値がリセットされる（種族値ベースの値に戻る）

詳細は `docs/spec/moves/ガードシェア.md` を参照。

## Priority 根拠

`docs/spec/turn.md` には `ON_STATUS_HIT` の内部priority表は掲載されていない。既存の変化技ハンドラ（`いたみわけ_equalize_hp`・`おかたづけ_cleanup`・`りゅうのまい_modify_attacker_stats` 等）はすべて priority 100（`MoveHandler` デフォルト）を使用している。ガードシェアも同一パターンで priority 100 を採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: 五十音順「が」行 — 「か行」清音・濁音の順序により `かいでんぱ_modify_defender_stats`（かい）の前に挿入する。

実数値の書き換えは `mon._stats_manager.stats`（`PokemonStats.stats` プロパティ）が返す **実体のリスト**（コピーではない）に対してインデックス代入することで行う。`Pokemon.stats`（`mon.stats`）は `stats_dict` を経由した dict のコピーを返す読み取り専用プロパティのため区別すること。`battle.modify_stats()` は**ランク変化**用のAPIのため使用しない。

```python
def ガードシェア_equalize_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ガードシェア: 使用者と相手のぼうぎょ・とくぼうの実数値を平均化する。

    ランク変化は行わず、実数値のみを書き換える。
    平均は切り捨て（// 2）。
    """
    atk_stats = ctx.attacker._stats_manager.stats
    def_stats = ctx.defender._stats_manager.stats
    # B=インデックス2、D=インデックス4
    for idx in (2, 4):
        avg = (atk_stats[idx] + def_stats[idx]) // 2
        atk_stats[idx] = avg
        def_stats[idx] = avg
    return HandlerReturn(value=value)
```

### `src/jpoke/data/moves/move_ka.py`

```python
"ガードシェア": MoveData(
    type="エスパー",
    category="status",
    pp=12,
    accuracy=None,  # 必中
    # マジックコートで跳ね返されない
    flags={"unreflectable"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ガードシェア_equalize_stats,
        ),
    }
),
```

`まもる`・`みがわり` によるブロックは追加のフラグ無しで既定動作により処理される:
- `Move.is_blocked_by_protect` は `target == "foe"` かつ `unprotectable` フラグ無しの場合に True を返すため、既定でまもるに防がれる。
- `MoveExecutor.check_hit_substitute` も `target == "foe"` かつ `bypass_substitute` / `sound` フラグ無しの場合にみがわり判定へ進むため、既定でみがわりに防がれる。
- `Move.is_reflectable` は `category == "status"` かつ `target in ("foe", "foe_side")` の場合に `unreflectable` フラグが無ければ True になる。ガードシェアはマジックコートで反射されない仕様のため `unreflectable` フラグが**必須**。

### `src/jpoke/model/pokemon.py`

`Pokemon.reset_on_switch_out()` に `self.update_stats()` を追加し、交代時に実数値を種族値ベースへ再計算してリセットする（ガードシェア・パワーシェア・パワートリックなど実数値を直接書き換える技すべてに共通する仕様上の要請）。

```python
def reset_on_switch_out(self):
    self.active_turn = 0
    self.hits_taken = 0
    self.rank = {k: 0 for k in STATS}
    # ガードシェア・パワーシェア・パワートリックなどによる実数値の書き換えを
    # 種族値ベースの値に再計算してリセットする（交代でリセットされる仕様）
    self.update_stats()
    self.executed_move = None
    ...
```

`update_stats()` は現在の `base`・`level`・`nature`・`indiv`・`effort` から実数値を再計算するのみで、HP実数値（インデックス0）は再計算されても `self.hp`（被弾中の現在HP）自体は変更しない（`keep_damage` を指定しない限り）ため、HPポイントに副作用はない。

## 注意点・エッジケース

- **実数値変更 vs ランク変化の区別**: `battle.modify_stats()` はランク変化API（`status_manager.modify_stats` 委譲）であり、実数値には触れない。ガードシェアは実数値平均化のため `_stats_manager.stats` の実体リストを直接書き換える必要がある
- **`Pokemon.stats` の読み取り専用制約**: `mon.stats` は `_stats_manager.stats_dict`（dictコピー）を返すため、`mon.stats["B"] = avg` の代入は反映されない。必ず `mon._stats_manager.stats`（リストの実体）を経由する
- **HPへの非影響**: H=インデックス0 は対象外。誤って `stats[0]` を変更しないよう注意する（`max_hp` ズレが生じる）
- **ランク非変化の確認**: `ON_BEFORE_MODIFY_STAT` や `ON_MODIFY_STAT` 系イベントは一切発火しない設計のため、クリアボディ・かがくへんかガス等の能力変化ガードは影響しない
- **パワーシェアとの類似**: パワーシェア（A・C実数値平均化）は同じパターンで実装済み。インデックスが A=1、C=3 になる点のみ異なる
- **ワンダールーム中の扱い**: ワンダールーム効果（B/D参照を入れ替え）は `ON_CALC_DEF_MODIFIER` ハンドラ内で対処されるため、ガードシェアハンドラ側では考慮不要
- **交代によるリセット**: `Pokemon.reset_on_switch_out()` の `update_stats()` 呼び出しにより対応済み。フォルムチェンジ（`base` setter経由での `update_stats()` 呼び出し）でも同様にリセットされる
- **メガシンカ時の実数値維持（Champions固有仕様）**: 本プロジェクトでは未対応。メガシンカで `base` が変わると `update_stats()` が再計算され、ガードシェアの効果は消えてしまう（Gen7相当の挙動）。Championsで実数値を維持する仕様は非常に限定的なケースのため対応を見送る
- **オウムがえし・よこどり**: 本プロジェクトには未実装の機能のため対応不要
