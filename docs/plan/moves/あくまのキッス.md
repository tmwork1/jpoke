# 実装計画: あくまのキッス

## 仕様サマリ

- タイプ: ノーマル / 分類: 変化 / PP: 10 / 命中率: 75
- 効果: 相手をねむり状態にする（継続ターン数は `docs/spec/ailments/ねむり.md` の Champions 共通仕様に従う）
- まもる: ○ / マジックコート: ○ / 直接攻撃: ×

## 対象イベント

`ON_STATUS_HIT` — 変化技命中時に発火。`おにび`（やけど）・`でんじは`（まひ）と同パターン。

## Priority

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の `おにび`・`でんじは` が `MoveHandler(func)` のみ（priority=100デフォルト）で実装されているため、同様に priority=100 を採用する。

## 実装手順

### handlers/move_status.py

`あくまのキッス_apply_ailment_to_defender` 関数を追加する。

```python
def あくまのキッス_apply_ailment_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="ねむり")
```

配置位置: `あくび_*` 関数の直後（五十音順: あくび → あくまのキッス → あまいかおり）

### data/move.py

`あくまのキッス` の MoveData に `ON_STATUS_HIT` ハンドラを追加する。

```python
"あくまのキッス": MoveData(
    type="ノーマル",
    category="変化",
    pp=10,
    accuracy=75,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.あくまのキッス_apply_ailment_to_defender,
        ),
    }
),
```

## テスト方針

- ねむり付与の確認
- すでに状態異常がある場合は失敗する確認
- 命中率75のため `accuracy=100` 固定で実行

## 注意事項

- `うたう`（音系・みがわり貫通）と効果は同じだが `あくまのキッス` は音系ではなく通常の変化技
- あくびと異なり揮発効果を経由せず直接ねむり付与
- かがくへんかガス/かたやぶり無効化チェックは書かない
