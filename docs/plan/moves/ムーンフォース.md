# ムーンフォース 実装計画書

## 仕様要約（docs/spec/moves/ムーンフォース.md 参照）

* タイプ: フェアリー / 分類: 特殊 / 威力: 95 / 命中: 100 / PP: 16（Champions）
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果: 追加効果として、10%の確率で相手の『とくこう』ランクを1段階下げる
  （Championsで30%→10%に変更されている点に注意）。

## 現状の実装状況

`src/jpoke/data/moves/move_ma.py` に登録済み。

```python
"ムーンフォース": MoveData(
    type="フェアリー",
    category="special",
    pp=16,
    power=95,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ムーンフォース_lower_spa_C,
        )
    }
),
```

`handlers/move_attack.py` の `ムーンフォース_lower_spa_C` は `modify_defender_stats` を
`chance=0.3` で呼び出しているが、これはスカーレット・バイオレット基準の確率であり、
Champions では10%に変更されている。**`chance=0.1` に修正する必要がある。**

## ハンドラ構成

| event | priority | subject_spec | 役割 |
|---|---|---|---|
| `Event.ON_DAMAGE_HIT` | デフォルト（既存の同種ハンドラ `むしのていこう_lower_spa_C` 等に合わせる） | `attacker:self`（`MoveHandler` は技のハンドラのため attacker 視点） | 10%の確率で相手の『とくこう』ランクを1段階下げる |

`modify_defender_stats(battle, ctx, value, stats={"spa": -1}, chance=0.1)` を呼び出すのみで、
`chance` 引数の値以外は既存実装のまま変更不要。

## Priority 根拠

新規イベントハンドラを追加するわけではなく、既存の `ON_DAMAGE_HIT` ハンドラの `chance` 引数のみを
修正するため、priority の再検討は不要（既存の登録のまま据え置き）。

## 実装箇所

### handlers/move_attack.py

`ムーンフォース_lower_spa_C` の `chance=0.3` を `chance=0.1` に修正する。

```python
def ムーンフォース_lower_spa_C(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spa": -1}, chance=0.1)
```

### data/moves/move_ma.py

変更なし（`pp=16` は既に `src/jpoke/data/ps-champ-ja/moves.json` と一致）。

## 注意点・エッジケース（review-test 向け）

1. **確率がChampionsで30%→10%に変更されている**
   - 一次情報（`docs/wiki/moves/ムーンフォース.html`）の「Champions」欄に「10%の確率で相手の特攻を
     1段階下げる。」と明記されており、`docs/champions/changes_from_sv.md` にも同様の記載がある。
   - 既存実装が `chance=0.3`（SV基準）のままになっていたバグを本レビューで修正した。
2. **ちからずくとの連動**
   - `flags={"secondary_effect"}` が付与されているため、ちからずく所持ポケモンが使用すると
     追加効果（とくこうダウン）が発動しなくなる代わりに威力が1.3倍になる
     （`resolve_secondary_chance` 経由で処理されるため、本技側の実装に追加対応は不要）。
3. **テストの `secondary_chance` オプション**
   - `tests/test_utils.py` の `secondary_chance=1.0` は確率計算をバイパスして必ず発動させる
     テスト専用オプションであり、実際の10%という値そのものはテストで検証していない
     （既存の同種技のテストパターンに合わせる）。
