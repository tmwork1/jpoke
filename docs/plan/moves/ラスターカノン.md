# ラスターカノン 実装計画書

## 仕様要約（docs/spec/moves/ラスターカノン.md 参照）

* タイプ: はがね / 分類: 特殊 / 威力: 80 / 命中: 100 / PP: 12（docs/champions/move_list.txt準拠）
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果: 追加効果として、10%の確率で相手の『とくぼう』ランクを1段階下げる。

## 現状の実装状況（修正前）

`src/jpoke/data/moves/move_ra.py` に登録済みだが、以下2点の誤りがあった。

1. `pp=10` になっているが、`docs/champions/move_list.txt` の正しいPPは `12`。
   Gen9本家のPP(10)への移行漏れバグ。
2. `flags` に `"secondary_effect"` が欠落しているため、ちからずく特性による
   威力1.3倍化・追加効果無効化が適用されない誤りがあった。同じ「確率発動の相手ランク低下」
   パターンの `10まんボルト` `オーロラビーム` 等は `secondary_effect` を保持しており、
   `ラスターカノン` も対象技として扱うべきである。

## 修正後

```python
"ラスターカノン": MoveData(
    type="はがね",
    category="special",
    pp=12,
    power=80,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ラスターカノン_lower_defender_spd,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手の『とくぼう』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def ラスターカノン_lower_defender_spd(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spd": -1}, chance=0.1)
```

`modify_defender_stats`（`handlers/move.py`）は内部で
`battle.resolve_secondary_chance(ctx, chance)` を経由するため、ちからずくのポケモンが
使用した場合は確率が0化され効果が発動しなくなる（`secondary_effect` フラグが前提）。
本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（能力ランク低下）は既存の同種ハンドラ（10まんボルト・
オーロラビーム・らいめいげり 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）の
パターンに従う。`ON_DAMAGE_HIT` は `docs/spec/turn.md` に未掲載のイベント
（ダメージ計算内部）だが、既存の同種ハンドラの実装パターンを踏襲するため
個別の priority 検討は不要。

対象外（`docs/progress/move.md` のリーサル実装列は n/a）。確率発動の相手ランク低下効果であり、
ダメージ量そのものを変化させないため、リーサル計算（`core/lethal.py`）の対象としない。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` を追加したことで、ちからずく所持時は威力1.2倍
     （80→96、実際の倍率計算は特性側実装に従う）になり、とくぼう低下の追加効果は
     発動しなくなる。
2. **接触フラグ**
   - 一次情報（docs/wiki/moves/ラスターカノン.html）で「直接攻撃: ×」と明記されており、
     `flags` に `"contact"` は付与しない（現状も付与されておらず正しい）。
3. **テスト方針**
   - 10%の確率発動効果のため `secondary_chance`（`Battle.random` のシード固定）で
     発動・不発動の両パターンを確認するテストを追加する。
   - PPが12であることを確認するテストを追加する。
