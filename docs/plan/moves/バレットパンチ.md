# 計画書: バレットパンチ

## 仕様要約

- はがねタイプ / 物理 / 威力40 / 命中100 / PP20（champions基準） / 対象: 1体選択 / 優先度+1 / 接触
- 追加効果なし（通常攻撃のみ、優先度+1のみが特徴）
- パンチ技（てつのこぶし・パンチグローブの対象）

## ハンドラ構成

追加効果を持たないため、専用ハンドラは不要。`handlers={}` のみで基本ダメージ計算に委ねる。
優先度は `priority=1` を `MoveData` に指定するだけでよく、`turn_controller.py` の行動順決定処理が
既存の共通ロジックとして `priority` を参照するため、専用実装は不要。
てつのこぶし・パンチグローブの威力補正は `flags={"punch"}` を付与するだけで既存の
`handlers/ability.py` / `handlers/item.py` 側の共通ロジックが対応する。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、priority の検討対象イベントは存在しない。
技自体の行動順における優先度（`MoveData.priority=1`）は `docs/spec/turn.md` の「技の優先度」に
関する記述通り、通常の優先度制御の枠組みで処理される。

## 実装箇所

### data/moves/move_ha.py への更新

既存の `"バレットパンチ"` エントリは以下の通り。

```python
    "バレットパンチ": MoveData(
        type="はがね",
        category="physical",
        pp=20,
        power=40,
        accuracy=100,
        priority=1,
        flags={"contact", "punch"},
        handlers={},  # 追加効果なし
    ),
```

- `flags={"contact", "punch"}` → 接触判定あり・パンチ技（仕様と一致）。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致。
- `priority=1` → 優先度+1（仕様と一致）。
- `critical_rank` 省略時のデフォルトは `0`（通常の急所判定）→ 仕様と一致。
- `pp=20` → champions基準のPP値（本家PP30から移行）。

## リーサル計算

追加効果のない固定威力技のため、専用のリーサル計算ハンドラは不要（n/a）。
標準のダメージ分布計算のみでリーサル判定が成立する。

## 注意点・エッジケース

- 追加効果がないため、まもる・おうじゃのしるし・オウムがえし等の一般的な技判定は
  フレームワーク側の共通処理（`handlers/move_attack.py` の基本ダメージ計算経路）で
  自動的に処理され、本技固有のハンドラ実装は不要。
- 優先度+1のため、同一ターン内で通常優先度の技より先に発動する。すばやさ比較は
  同優先度同士の場合のみ行われる（`docs/spec/turn.md` 参照）。
- てつのこぶし所持時は威力が1.2倍になるが、これは特性側の共通ロジック
  （`handlers/ability.py` の `_modify_by_move_condition` を用いた実装）で処理されるため、
  本技側での実装は不要。
