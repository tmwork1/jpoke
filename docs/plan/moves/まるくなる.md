# 実装計画: まるくなる

## 仕様サマリ

- タイプ: ノーマル / 分類: 変化 / PP: 40（Gen9本家準拠。champions/move_list.txt に本技自体が未収録のため） / 命中率: なし（自分対象）
- 効果: 自分の防御を1段階上げると同時に、揮発性状態『まるくなる』を付与する
  （まるくなる状態のポケモンが`ころがる`・`アイスボール`を使うと威力が2倍になる。既存の
  `まるくなる_boost_power` ハンドラ・`docs/spec/volatiles/まるくなる.md` で対応済み）。
- まもる: × / マジックコート: × / 直接攻撃: ×
- 対象: 自分

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自分の防御を1段階上げ、揮発性状態『まるくなる』を付与する |

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の自分のランク上昇ハンドラ（`かたくなる_modify_attacker_stats`、`こうそくいどう_modify_attacker_stats` など）
がすべて priority=100 デフォルトで実装されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py（実装済み）

挿入位置: 五十音順で `まもる系_連続使用失敗チェック` の後、`みかづきのいのり_apply` の前
（まき → まね → まも → まる → みか の順）。

```python
def まるくなる_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """まるくなるの効果を発動する。"""
    mon = ctx.attacker
    battle.modify_stats(mon, {"def": 1}, source=mon)
    battle.volatile_manager.apply(mon, "まるくなる")
    return HandlerReturn(value=value)
```

### data/moves/move_ma.py（レビューで修正）

`まるくなる` の MoveData に `target="self"` が未設定でデフォルトの `"foe"` になっていたため、
明示的に `target="self"` を追加した。

```python
"まるくなる": MoveData(
    type="ノーマル",
    category="status",
    pp=40,
    target="self",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.まるくなる_apply,
        )
    }
),
```

### handlers/move_status.py の `_BATON_PASS_VOLATILES`（レビューで修正）

`まるくなる`（および同様の欠陥を持つ `ちいさくなる`）が誤って含まれており、バトンタッチで
揮発性状態自体まで交代先に引き継がれてしまう不具合があった。
`docs/wiki/moves/バトンタッチ.html`「状態変化の引き継ぎ」表では、`まるくなる`・`ちいさくなる`
状態は第2世代のみ引き継がれ第3〜8世代（Champions が準拠する第9世代相当を含む）では
引き継がれないと明記されており、`docs/spec/volatiles/まるくなる.md` 自身も
「バトンタッチによる引き継ぎ（第二世代のみ有効、Champions は第三世代以降の仕様を採用）」を
スコープ外と明記していた。よってセットから `まるくなる`・`ちいさくなる` を削除した
（防御ランクの上昇自体は能力変化として別途正しく引き継がれるため、揮発状態を対象外にしても
影響しない）。

## 注意点・エッジケース

- 対象は自分であるため `target="self"` を必ず設定する。デフォルトの `target="foe"` のままだと
  `Move.is_blocked_by_protect` / `Move.is_reflectable` が誤って True になり、相手の「まもる」で
  阻止されたり「マジックコート」で跳ね返されたりする不具合になる（レビュー時に発見・修正）。
- `battle.modify_stats` を直接呼び出しているため `source=mon`（自分）を明示している。
- ランク上限（+6）チェックはフレームワーク側（`ON_TRY_MOVE_2` priority=130）で処理されるため、
  ハンドラ側のガードは不要。
- 揮発性状態『まるくなる』はバトンタッチで引き継がれない（`_BATON_PASS_VOLATILES` に含めない。
  上記参照）。
- 命中率なし（自分対象）のため accuracy 設定不要。テストは命中率固定不要。
- PPは champions/move_list.txt に本技自体の収録がないため、Gen9本家の値（40）をそのまま採用する
  （詳細は `docs/spec/moves/まるくなる.md` の「PPについて」を参照）。
