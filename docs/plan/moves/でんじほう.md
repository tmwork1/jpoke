# でんじほう 実装計画書

## 仕様要約

`docs/spec/moves/でんじほう.md` を参照。
でんきタイプ・特殊・威力120・命中50・PP5の非接触攻撃技。
追加効果として100%の確率で相手を「まひ」状態にする。弾技のため「ぼうだん」持ちには無効。

## ハンドラ構成

`Event.ON_DAMAGE_HIT` に `MoveHandler` デフォルト設定（`subject_spec="attacker:self"`, `priority=100`）で登録する。
`apply_ailment_to_defender` は `handlers/move.py` の既存共通関数を使用する（`chance` はデフォルトの1のため省略）。

| 技 | event | priority | subject_spec | 役割 |
|---|---|---|---|---|
| でんじほう | ON_DAMAGE_HIT | 100 | attacker:self | まひ 100% |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_DAMAGE`（実装上の `Event.ON_DAMAGE_HIT`）セクションに
「10: 追加効果（特殊なもの除く）」とあるが、既実装の `かみなりパンチ`・`でんきショック` 等の
まひ付与技と登録コードを揃えるため、`h.MoveHandler` のデフォルト priority=100 を採用する
（`docs/plan/moves/攻撃技_まひ系.md` の他技と同じ扱い）。

## 実装箇所

### `src/jpoke/handlers/move_attack.py`

```python
def でんじほう_apply_paralysis_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="まひ")
```

### `src/jpoke/data/move.py`

```python
"でんじほう": MoveData(
    type="でんき",
    category="special",
    pp=5,
    power=120,
    accuracy=50,
    flags={"bullet", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.でんじほう_apply_paralysis_to_defender,
        ),
    }
),
```

## リーサル計算

- 対象外（`n/a`）。まひは行動阻害・素早さ低下効果であり、ダメージ量には影響しないため専用のリーサルハンドラは不要。

## エッジケース

- でんきタイプの相手はまひ無効（`ailment_manager` 側の共通処理で判定されるため、本技のハンドラに追加ガードは不要）。
- 「ぼうだん」特性持ちの相手には弾技として無効化される（`flags={"bullet"}` を通じて特性側ハンドラが処理するため、本技側の実装は不要）。
