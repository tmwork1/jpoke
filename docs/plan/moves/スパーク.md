# スパーク 実装計画書

## 仕様要約（docs/spec/moves/スパーク.md 参照）

* タイプ: でんき / 分類: 物理 / 威力: 65 / 命中: 100 / PP: 20
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 追加効果として30%の確率で相手をまひ状態にする。

## 現状の実装状況

`src/jpoke/data/moves/move_sa.py` に既に登録済み。

```python
"スパーク": MoveData(
    type="でんき",
    category="physical",
    pp=20,
    power=65,
    accuracy=100,
    flags={"contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.スパーク_apply_paralysis_to_defender,
        )
    }
),
```

`src/jpoke/handlers/move_attack.py` にハンドラ実装済み。

```python
def スパーク_apply_paralysis_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="まひ", chance=0.3)
```

内容は仕様書と一致しており、修正不要。

## ハンドラ構成

| event | priority | subject_spec | 役割 |
|---|---|---|---|
| ON_DAMAGE_HIT | 100（デフォルト） | attacker:self（デフォルト） | 30%の確率で相手をまひ状態にする |

`apply_ailment_to_defender` は `handlers/move.py` の既存共通関数を使用する
（`battle.resolve_secondary_chance` で確率解決後 `battle.ailment_manager.apply` を呼ぶ）。

## Priority 根拠

まひ付与は `Event.ON_DAMAGE_HIT`（`docs/spec/turn.md` には専用イベント名で掲載されていないが、
10まんボルト・のしかかり・ほうでん等、既存の同種「追加効果付与」ハンドラがすべて
`ON_DAMAGE_HIT` + デフォルト priority=100 で登録されているため、同一パターンを踏襲する）。

## 注意点・エッジケース（review-test 向け）

### まひの無効条件
- でんきタイプはまひ無効。`apply_ailment_to_defender` → `battle.ailment_manager.apply` 内部で判定されるため、
  本技側ハンドラに追加ガードは不要。

### PPの扱い
- `src/jpoke/data/ps-champ-ja/moves.json` にスパークのエントリが存在しない。第九世代基準のPP20は
  既に統一4段階（8/12/16/20）のひとつであり、`changes_from_sv.md` の技個別変更点一覧にも
  スパークの記載がないため、PP20のまま変更不要と判断する。

### 確率発動のテスト方針
- `secondary_chance` 引数で `battle.random` を制御し、確率発動ケース（発動/非発動）を両方テストする。
- 既存テストは「発動する」ケースのみだったため、10まんボルト・のしかかり等の同系統技に合わせて
  「発動しない」ケースのテストを追加する。
