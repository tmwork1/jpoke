# 計画書: ふるいたてる

## 仕様要約

- 自分の「こうげき」「とくこう」ランクをそれぞれ1段階上げる
- タイプ: ノーマル、変化、PP: 20（champions 基準）、対象: 自分
- 純粋な自己ランク上昇技（プラスのみ、混在なし）

出典: `docs/spec/moves/ふるいたてる.md`

---

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 攻撃・特攻を1段階ずつ上げる |

---

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`つるぎのまい_modify_attacker_stats`, `こうそくいどう_modify_attacker_stats` 等）が
すべて priority=100（デフォルト）で登録されているため、同様に採用する。

---

## 実装箇所

### `src/jpoke/handlers/move_status.py`

既存実装（五十音順で `フラワーヒール_heal_defender` の後、`ブレイブチャージ_apply` の前）:

```python
def ふるいたてる_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ふるいたてるの効果: 自分のこうげき・とくこうを1段階ずつ上げる。"""
    return modify_attacker_stats(battle, ctx, value, stats={"atk": 1, "spa": 1})
```

### `src/jpoke/data/moves/move_ha.py`

`target="self"` を指定する（`target` を指定しないとデフォルトの `"foe"` になり、
`is_blocked_by_protect` / `is_reflectable` やいたずらごころ・おうごんのからだ・もらいび等の
「相手対象の変化技」判定に誤反応してしまうため必須）:

```python
"ふるいたてる": MoveData(
    type="ノーマル",
    category="status",
    pp=20,
    target="self",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ふるいたてる_modify_attacker_stats,
        ),
    }
),
```

---

## 注意点・エッジケース

- ランクがすでに+6の場合、`battle.modify_stats` が空の結果を返し技が失敗する（フレームワーク側で
  処理、ハンドラ側にガード不要）。一方のランクのみ上限の場合は、もう一方のランクだけ上昇する
- 追加効果・急所判定は対象外（変化技のため）
- リーサル計算: 対象外（ダメージを与えない変化技、n/a）
