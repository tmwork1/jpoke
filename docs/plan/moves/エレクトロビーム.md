# 計画書: エレクトロビーム

## 仕様要約

出典: `docs/champions/move_list.txt`

- タイプ: でんき / 分類: 特殊 / 威力: 130 / 命中: 100 / PP: 12 (data/move.py は 10)
- 接触: × / まもる: ○
- **1ターン目**: 100% の確率で自分の「とくこう」を1段階上げ、溜め状態に入る（追加効果でない）
- **2ターン目**: 攻撃する
- **天気がいあめ/おおあめ**: 溜めをスキップして即攻撃（とくこう上昇は発動する）
- **パワフルハーブ**: 溜めをスキップして即攻撃（とくこう上昇は発動する）

### メテオビームとの比較

| 技 | SpAtk 変化 | 天気スキップ条件 | 天気による威力変化 |
|---|---|---|---|
| エレクトロビーム | +1（必ず） | あめ / おおあめ | なし |
| メテオビーム | +1（必ず） | なし | なし |
| ソーラービーム | なし | にほんばれ / おおひでり | あめ/すなあらし/ゆきで半減 |

## Priority 根拠

- **`ON_MOVE_CHARGE` priority 50 (boost_spa)**: `docs/spec/turn.md` の `ON_MOVE_CHARGE` テーブルにはエレクトロビームの記載がないが、同じく「溜め中に自分のとくこう+1」を持つメテオビームの実装が priority=50 (data/move.py L5766) を採用している。パワフルハーブが priority=100 の ON_MOVE_CHARGE 内で処理されるより前に boost を確実に実行するため、同じ priority=50 を採用する。
- **`ON_MOVE_CHARGE` priority 100 (charge)**: `docs/spec/turn.md` の `ON_MOVE_CHARGE` テーブルで `| 100 | メテオビーム: とくこう上昇 |`（ソーラービームや隠れる技も同 100）と記載。溜め系技は priority 100 が標準。
- **`ON_HIT` volatile 解除**: ソーラービーム/メテオビームと同じ priority デフォルト（100）を踏襲。

### 2. `src/jpoke/handlers/move_attack.py`

五十音順（エ行）に以下2関数を追加。既存の `エレキネット_*` 付近の後に挿入：

```python
def エレクトロビーム_boost_spa(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """エレクトロビーム: とくこうを1段階上げる（追加効果ではないため必ず発動）。

    仕様:
    - ちからずくの追加効果ではないため、battle.modify_stats を直接呼ぶ。
    - パワフルハーブ使用時・あめでスキップ時もこのハンドラ（priority=50）が
      先に実行されるため、とくこう上昇は必ず発動する。
    """
    battle.modify_stats(ctx.attacker, {"C": 1}, source=ctx.attacker)
    return HandlerReturn(value=value)


def エレクトロビーム_charge(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """エレクトロビーム: 1ターン目に溜め状態へ移行、あめ/おおあめ時または2ターン目は通過する。

    仕様:
    - 揮発状態なし + 非あめ天気（1ターン目）: 揮発状態を付与して攻撃を停止する。
    - 揮発状態なし + あめ/おおあめ（1ターン目）: 溜めをスキップして攻撃に進む。
    - 揮発状態あり（2ターン目）: そのまま通過して攻撃に進む。
    - とくこう上昇は エレクトロビーム_boost_spa（priority=50）で先に処理される。
    - ばんのうがさを持つ場合は weather_for により あめ の恩恵を受けない。
    """
    attacker = ctx.attacker
    if not attacker.has_volatile("エレクトロビーム"):
        # あめ/おおあめなら溜めをスキップ
        if battle.weather_for(attacker).rainy:
            return HandlerReturn(value=value)
        battle.volatile_manager.apply(attacker, "エレクトロビーム", count=1, source=attacker)
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース（review-test 向け）

1. **とくこう上昇は追加効果でない**: `battle.resolve_secondary_chance` を呼んではならない。`battle.modify_stats` を直接呼ぶ（メテオビームと同じパターン）。

2. **ばんのうがさ持ちはあめの恩恵を受けない**: `battle.weather_for(attacker).rainy` を使うことで自動的にばんのうがさ判定が入る（`battle.weather.rainy` は使わない）。

3. **パワフルハーブとの相互作用**: パワフルハーブによる溜めスキップは `ON_TRY_MOVE_1 priority=70` で処理される。`エレクトロビーム_boost_spa`（priority=50）は `ON_MOVE_CHARGE` 内で先に実行されるため、パワフルハーブ使用時もとくこうは+1される（メテオビームと同様の挙動）。

4. **あめ + 2ターン目の挙動**: 1ターン目にあめでスキップした場合、揮発状態は付与されない。そのため「揮発状態あり」が2ターン目の判定に使われるのは通常（あめなし）の2ターン目のみ。

5. **PP数の差異**: `data/move.py` は pp=10 だが `docs/champions/move_list.txt` では 12。ポケモンチャンピオンズ準拠にするなら pp=12 が正しい可能性がある。impl エージェントが確認して修正すること。

6. **ON_HIT での揮発解除**: ソーラービーム/メテオビームと同様に `ON_HIT` で揮発状態を解除する。これにより攻撃がみがわりに当たった場合でも解除される（`ON_HIT` は「みがわりに被弾しても発動」と `docs/spec/turn.md` に記載）。

7. **VolatileData の `forced=True`**: 溜め中は次ターンの行動が同技に強制される。`ON_MODIFY_COMMAND_OPTIONS` に `force_command` を登録することで実現（ソーラービーム/メテオビームと同一パターン）。
