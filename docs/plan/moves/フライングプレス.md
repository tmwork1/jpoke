# フライングプレス 実装計画書

## 仕様要約（docs/spec/moves/フライングプレス.md 参照）

* タイプ: かくとう（同時にひこうタイプでもある。相性計算のみ複合） / 分類: 物理 / 威力: 100 / 命中: 95 / PP: 12
* 対象: 相手1体 / 優先度: 0 / 接触あり
* じゅうりょく状態では使用不可（`gravity_restricted`）
* 「ちいさくなる」状態の相手には必ず命中し、威力2倍になる（`minimize` ラベル共通処理、`docs/spec/volatiles/ちいさくなる.md`）
* かくとうタイプ相性とひこうタイプ相性を掛け合わせてダメージ計算する（相性判定のみの複合。STAB・きけんよち・
  タイプ強化アイテム等は単純にかくとうタイプ技として扱う）

## 現状の実装状況

`src/jpoke/data/moves/move_ha.py` に登録済み。

```python
"フライングプレス": MoveData(
    type="かくとう",
    category="physical",
    pp=12,
    power=100,
    accuracy=95,
    flags={"contact", "gravity_restricted", "minimize"},
    handlers={
        Event.ON_TRY_MOVE_1: h.MoveHandler(
            h.gravity_restricted_fail,
            subject_spec="attacker:self",
            priority=30,
        ),
        Event.ON_CALC_DEF_TYPE_MODIFIER: h.MoveHandler(
            ha.フライングプレス_add_flying_type,
        ),
    }
),
```

```python
def フライングプレス_add_flying_type(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """フライングプレス: かくとうタイプに加えてひこうタイプとしても相性計算する。

    damage_calculator.py がすでにかくとうタイプ相性を value に計算済み。
    ここでひこうタイプの相性を追加で掛け算することで複合タイプ判定を実現する。
    """
    flying_chart = TYPE_MODIFIER.get("ひこう", {})
    for def_type in ctx.defender.types:
        rate = flying_chart.get(def_type, 1.0)
        value = int(value * rate)
    return HandlerReturn(value=value)
```

`minimize` フラグと `gravity_restricted` フラグは、それぞれ `ちいさくなる` ボラタイル側・`じゅうりょく` フィールド側の
共通ハンドラで処理されるため、フライングプレス固有の実装は不要（既存の共通処理をそのまま利用）。

### レビューで判明した修正点

1. **PPの誤り（バグ）**: `pp=10` はGen9本家準拠の値であり、チャンピオンズでは全技PPが8/12/16/20の4段階に
   統一されている（`docs/champions/changes_from_sv.md`）。`docs/champions/move_list.txt` の実測値は
   `12` であり、`docs/progress/move.md` のPP列も `12` になっていた（実装のみが移行漏れ）。
   `pp=10` → `pp=12` に修正した。

その他、タイプ複合計算・`minimize` 相互作用・アイテム相互作用（ヨプのみ/バコウのみ）・特性相互作用
（きけんよち/ハードロック/ブレインフォース/いろめがね/ノーマルスキン/へんげんじざい/へんしょく）は
いずれも仕様書・既存テストと一致しており問題なし。

## ハンドラ構成

| event | priority | subject_spec | 役割 |
|-------|----------|-------------|------|
| `ON_TRY_MOVE_1` | 30 | `attacker:self`（`AttackContext`） | じゅうりょく状態下では技を失敗させる（既存の `gravity_restricted_fail` を流用） |
| `ON_CALC_DEF_TYPE_MODIFIER` | デフォルト | （`AttackContext`。攻撃フェーズの内部イベントのため `subject_spec` 省略） | ひこうタイプの相性を追加で掛け算し、かくとう×ひこうの複合タイプ判定を実現する |

## Priority 根拠

`docs/spec/turn.md` には `ON_CALC_DEF_TYPE_MODIFIER` はダメージ計算内部イベントとして専用行が掲載されていない
（「Interrupt: ダメージ計算」としてまとめられている）。
既存の同種ハンドラ（`フリーズドライ_water_effectiveness`、`data/moves/move_ha.py` 登録）がデフォルト priority で
登録されていることを根拠に、本ハンドラもデフォルト priority とする。
`ON_TRY_MOVE_1` の priority=30 は既存の `gravity_restricted_fail`（とびげり・とびはねる等で使用）と同一のため
変更なし。

## 注意点・エッジケース（review-test 向け）

1. **相性計算**: いわ（かくとう2倍×ひこう0.5倍=等倍）、ノーマル（かくとう2倍×ひこう等倍=2倍）、
   ゴースト（かくとう0倍×ひこう等倍=0倍・無効）を確認する既存テストあり
   （`tests/moves_attack/test_move_ha.py`）。
2. **`minimize` 相互作用**: `tests/volatiles/test_volatile_satana.py` の `minimize_enhance_moves`
   パラメトライズドテストに含まれており、必中化・威力2倍を確認済み。
3. **`gravity_restricted` 相互作用**: `tests/test_field.py` の `test_じゅうりょく_使用禁止技ブロック` に
   含まれており、じゅうりょく中の使用失敗を確認済み。
4. **アイテム相互作用**: `tests/items/test_item_ha.py`（バコウのみ不発動）、
   `tests/items/test_item_yarawa.py`（ヨプのみ：複合相性が抜群のときのみ発動）で確認済み。
5. **PP**: `pp=12` であること（本レビューでの修正点。既存テストでの直接アサートはなし）。
