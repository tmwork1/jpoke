# マジカルフレイム 実装計画書

## 仕様要約（docs/spec/moves/マジカルフレイム.md 参照）

* タイプ: ほのお / 分類: 特殊 / 威力: 75 / 命中: 100 / PP: 12（Champions基準、本家PP10からの変換）
* 対象: 相手1体 / 優先度: 0 / 接触なし（直接攻撃ではない）
* 効果: 追加効果として100%の確率で相手の『とくこう』ランクを1段階下げる。

## 現状の実装状況（レビュー前）

`src/jpoke/data/moves/move_ma.py` に既に登録済みだったが、以下の誤りがあった。

```python
"マジカルフレイム": MoveData(
    type="ほのお",
    category="special",
    pp=10,                                                  # 誤り: Championsの4段階(8/12/16/20)に含まれない
    power=75,
    accuracy=100,
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.マジカルフレイム_lower_spa_C,
        )
    }
    # flags 未指定 → 誤り: secondary_effect フラグが不足
    # （ちからずく・てんのめぐみ等の判定対象にならない）
),
```

とくこう低下ハンドラ自体（`ha.マジカルフレイム_lower_spa_C` =
`modify_defender_stats(battle, ctx, value, stats={"spa": -1})`）はデフォルトのchance=1（100%）で
仕様と一致しており、修正不要。

## 修正後の実装

```python
"マジカルフレイム": MoveData(
    type="ほのお",
    category="special",
    pp=12,
    power=75,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.マジカルフレイム_lower_spa_C,
        )
    }
),
```

## ハンドラ構成

| Event | 関数 | subject_spec | priority | 役割 |
|---|---|---|---|---|
| `Event.ON_DAMAGE_HIT` | `マジカルフレイム_lower_spa_C` | `attacker:self`（デフォルト） | 100（デフォルト） | 100%の確率で相手のとくこうを1段階下げる |

`ON_DAMAGE_HIT` は既存の同種とくこう低下技（バークアウト・はいよるいちげき・むしのていこう 等）と
同一イベントであり、`docs/spec/turn.md` に個別の priority 記載がないダメージ命中後の追加効果
ハンドラ群と同様、`MoveHandler` のデフォルト priority（100）をそのまま使用する。

## Priority 根拠

* `Event.ON_DAMAGE_HIT` は `docs/spec/turn.md` に個別の priority 記載がないイベントであり、
  CLAUDE.md の指針に従い既存の同種ハンドラ（`バークアウト_lower_spa_C` 等、すべて priority
  未指定＝デフォルト100）を参照して同じ扱いとする。

### handlers/move_attack.py

### data/moves/move_ma.py

## 注意点・エッジケース（review-test 向け）

1. PP を `12` に修正 → PP消費・わるあがき遷移等の一般テストは既存の共通ロジックでカバーされる
   ため専用テスト不要。
2. `flags` に `"secondary_effect"` を追加 → ちからずく（威力1.3倍・追加効果無効化）・
   てんのめぐみ（追加効果確率2倍）等の特性判定対象になる（既存の共通ロジックで処理されるため
   専用ハンドラ不要）。修正前は判定対象外だったため、ちからずく所持ポケモンが使用した場合に
   威力上昇・追加効果無効化のどちらも発生しない不具合があった。
3. `flags` に `"contact"` を含めない → 一次情報の「直接攻撃：×」の通り、接触判定系の特性・
   アイテム（さめはだ等）のトリガー対象外であることを確認する。
4. リーサル計算: 対象1体へのダメージ＋相手のとくこうランク低下のみで、HP分布・回復・残留
   ダメージ等に影響しないため、`docs/progress/move.md` の「リーサル実装」「リーサルテスト」列は
   `n/a` とする（同種のバークアウト・はいよるいちげき・むしのていこう も同様に `n/a`）。
