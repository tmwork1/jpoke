# 実装計画書: ほのおのまい（再レビュー）

## 仕様要約

`docs/spec/moves/ほのおのまい.md` 参照。

* タイプ: ほのお / 分類: 特殊 / 威力: 80 / 命中率: 100 / PP: 10 / 対象: 1体選択
* 追加効果として、50%の確率で自分の『とくこう』ランクを1段階上げる。
* `dance` フラグを持つ（おどりこによるコピー対象。おどりこ側の実装で対応済みのため
  本技側の対応は不要）。

## 既存実装の確認結果

`src/jpoke/handlers/move_attack.py` に `ほのおのまい_boost_spa_C` が実装済みで、
共通ヘルパー `modify_attacker_stats`（`handlers/move.py`）を通じて
`battle.resolve_secondary_chance` 経由でちからずく補正を適用しつつ、
50%の確率で `ctx.attacker` の `spa` を+1する。`data/moves/move_ha.py` の
`ほのおのまい` エントリから `Event.ON_HIT` で登録されており、`しんぴのちから_boost_spa_C` /
`フレアソング_boost_spa_C`（同じ「自分のとくこうが50〜100%で上がる」パターン）と
実装方針が一致しているため、修正の必要はない。

## ハンドラ構成（既存のまま）

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| ほのおのまい | `Event.ON_HIT` | デフォルト（100） | `attacker:self`（デフォルト） | 50%の確率で自分のとくこうを1段階上げる |

* `docs/spec/turn.md` の `Event.ON_HIT` 表には「自分のランクを上げる追加効果」が
  priority=10 として記載されているが、しんぴのちから・フレアソング・チャージビーム等
  同種の既存実装がいずれもデフォルト優先度（100）で登録されており、単独の攻撃技として
  同一 ON_HIT 内で競合する他ハンドラが存在しないため、既存実装の一貫性を優先し
  priority は変更しない。

## リーサル計算（新規実装）

`docs/progress/move.md` の該当行は「リーサル実装: n/a」「リーサルテスト: n/a」と
記載されていたが、命中後に自分の『とくこう』ランクが1段階上がる効果は以降の
攻撃のダメージ分布に影響するため、同種のしんぴのちから・フレアソングと同様に
リーサルハンドラの対象とすべきである（n/a 表記は誤りと判断し、本レビューで修正する）。

* `src/jpoke/handlers/lethal.py` に `ほのおのまい_boost_spa` を新規実装する
  （`しんぴのちから_boost_spa` / `フレアソング_boost_spa` と同一パターン）。
  * イベント: `LethalEvent.ON_HIT`
  * subject: `"attacker"`
  * `ctx.move_secondary` が真のとき、`ctx.attacker.rank["spa"]` を `clamp_stats` で
    1段階上げる。
* `data/moves/move_ha.py` の `ほのおのまい` エントリに
  `lethal_handlers={LethalEvent.ON_HIT: LethalHandler(l.ほのおのまい_boost_spa)}` を追加する。
