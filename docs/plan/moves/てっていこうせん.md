# 計画書: てっていこうせん

参照: docs/spec/moves/てっていこうせん.md
参照: docs/plan/moves/HPコスト05技.md（旧・統合計画。本技については本ファイルの内容を正とする）

## 仕様要約

- 使用前に自分の最大HPの1/2（**切り上げ**、最低1）を消費してから攻撃する
- `Event.ON_PAY_HP` はヒット処理より前に発火するため、HP消費後にHPが0になっても攻撃は相手に届く
- マジックガードで反動を防げるが、いしあたまでは防げない
  （クロロブラストは逆に両方で防げるため区別が必要。専用の `HPChangeReason = "fixed_recoil"` を使う）

## 現状の実装状況（修正前）

| 項目 | 状態 |
|------|------|
| `data/move.py` | 登録済み（`Event.ON_PAY_HP` ハンドラあり） |
| `handlers/move_attack.py` | `てっていこうせん_pay_hp` 実装済みだが、以下2点に誤りあり |

### 発見された誤り

1. **端数処理が切り捨てになっていた**: `cost = max(1, ctx.attacker.max_hp // 2)` は切り捨て。
   一次情報は「小数点以下切り上げ」のため `(ctx.attacker.max_hp + 1) // 2` に修正が必要。
2. **`HPChangeReason` が `"self_cost"` になっていた**: `"self_cost"` はマジックガード・いしあたま
   どちらでも防げない理由のため、「マジックガードで防げるがいしあたまでは防げない」という
   本技の仕様と矛盾する。新設の `"fixed_recoil"` に変更する。

## 実装箇所

### `src/jpoke/types/literals.py`

`HPChangeReason` に `"fixed_recoil"` を追加する
（マジックガードで防げるがいしあたまでは防げない反動。てっていこうせん専用）。

```python
HPChangeReason = Literal[
    ...
    "self_cost",            # 自己HP消費（みがわり等、マジックガード・いしあたまで防げない）
    "fixed_recoil",         # 確定反動（てっていこうせん等、マジックガードで防げるがいしあたまでは防げない）
    "bench_heal",
    "drain",
    "perish",
]
```

### `src/jpoke/handlers/move_attack.py`

```python
def てっていこうせん_pay_hp(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """てっていこうせん: 使用前に最大HPの1/2（切り上げ）を消費する。

    マジックガードで防げるが、いしあたまでは防げない反動として扱う。
    """
    cost = max(1, (ctx.attacker.max_hp + 1) // 2)
    battle.modify_hp(ctx.attacker, v=-cost, reason="fixed_recoil", source=ctx.attacker)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

既存の `てっていこうせん` エントリの `handlers` はそのまま
（`ha.てっていこうせん_pay_hp` を `Event.ON_PAY_HP` / `subject_spec="attacker:self"` で登録済み）。
変更不要。

### `src/jpoke/handlers/ability.py`

`マジックガード_ignore_damage` / `いしあたま_ignore_recoil` は変更不要
（`"fixed_recoil"` は `マジックガード_ignore_damage` の除外リストに含まれないため自動的に防がれ、
`いしあたま_ignore_recoil` は `reason == "recoil"` のみを見るため反応しない）。

## 注意点・エッジケース（review-test 向け）

- **HP0でも技が発動するか**: `ON_PAY_HP` はヒット処理より前に発火されるため、HP消費後に自分の
  HPが0になってもその後の攻撃処理（ダメージ付与）は継続する。テストで確認すること。
- **最低コスト1**: `max(1, (max_hp + 1) // 2)` により、最大HPが1でもコストは1になる。
- **切り上げの確認**: 最大HPが奇数のポケモンで、消費HPが `(max_hp + 1) // 2` になることを
  テストで確認する（偶数の場合は切り捨てと同じ値になり区別できないため、奇数個体が必要）。
- **マジックガードで反動無効**: マジックガード特性のポケモンが使用した場合、HPが減らないこと。
- **いしあたまでも反動あり**: いしあたま特性のポケモンが使用した場合でも、通常通りHPが減ること
  （クロロブラストとの違いを明確にするため、両特性のテストを揃える）。
- **すてみで威力上昇しない**: `flags={"recoil"}` を付与していないため、すてみ特性を持っていても
  威力は上がらないことを確認する（フラグを追加しないことが正しい実装）。
- **ダメおしとの連携は対象外**: 本技の反動ダメージが `ダメおし` の「そのターンに攻撃を受けた」
  判定に加算される一次情報上の仕様があるが、これは反動技全般に共通する `ダメおし` 側の課題として
  別途対応する（本技のテストでは扱わない）。
