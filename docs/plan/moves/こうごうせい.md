# 実装計画書: こうごうせい

## 仕様要約

- タイプ: くさ / カテゴリ: 変化 / PP: 5 / 対象: 自分
- label: `heal`
- 天候に応じた割合で使用者の HP を回復する

| 天候 | 回復量 |
|------|--------|
| はれ・おおひでり | 最大HP × 2/3 |
| 通常（天候なし・らんきりゅう） | 最大HP × 1/2 |
| あめ・おおあめ・すなあらし・ゆき | 最大HP × 1/4 |

HP が満タンの場合は失敗する。あさのひざし・つきのひかりと効果が完全に同一。

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「自己回復技: HP満タン」が priority 100 として記載されている。`ON_STATUS_HIT` 自体は turn.md に独立エントリが存在しないが、同種ハンドラ `じこさいせい_heal_self` および既実装の `あさのひざし_heal_self` がともに priority デフォルト（100）を使用しているため、同一パターンで 100 を採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: `こうそくいどう_modify_attacker_stats` の前（`き` 〜 `こ` のブロック内、現在 `きんぞくおん` の後に `こうそくいどう` があるため、その間に挿入）。

```python
def こうごうせい_heal_self(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """こうごうせい: 天候に応じた割合で自分のHPを回復する。"""
    mon = ctx.attacker
    if mon.hp == mon.max_hp:
        return HandlerReturn(value=False, stop_event=True)
    weather = battle.weather
    if weather.sunny:
        r = 2 / 3
    elif weather.name in {"あめ", "おおあめ", "すなあらし", "ゆき"}:
        r = 1 / 4
    else:
        r = 1 / 2
    battle.modify_hp(mon, r=r)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

既存の `"こうごうせい"` エントリに handlers を追加する。

```python
"こうごうせい": MoveData(

    category="変化",
    pp=5,
    labels=["heal"],
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.こうごうせい_heal_self,
        )

```

## 注意点・エッジケース

- あさのひざし と全く同じ回復ロジックを持つ。コードは `あさのひざし_heal_self` を参考にしてそのまま複製する
- `weather.sunny` プロパティが `"はれ"` と `"おおひでり"` の両方をカバーしていることを前提とする
- `らんきりゅう` は不利天候に含まれないため、通常時（1/2 回復）として扱う
- HP 満タンチェックはハンドラ内で行う（turn.md の ON_APPLY_MOVE priority 100「自己回復技: HP満タン」は失敗判定の確認に相当するが、既存パターンに倣い ON_STATUS_HIT ハンドラ内で一括処理する）
