# 実装計画書: こうごうせい

## 仕様要約

- タイプ: くさ / カテゴリ: 変化 / PP: 8（championsルールにより8/12/16/20の4段階へ統一。
  docs/champions/move_list.txt準拠） / 対象: 自分
- flags: `heal`
- 天候に応じた割合で使用者の HP を回復する

| 天候 | 回復量 |
|------|--------|
| はれ・おおひでり | 最大HP × 2/3 |
| 通常（天候なし・らんきりゅう） | 最大HP × 1/2 |
| あめ・おおあめ・すなあらし・ゆき | 最大HP × 1/4 |

HP が満タンの場合は失敗する。あさのひざし・つきのひかりと効果が完全に同一。

詳細は `docs/spec/moves/こうごうせい.md` を参照。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 天候参照して HP 回復 |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「自己回復技: HP満タン」が priority 100 として記載されている。`ON_STATUS_HIT` 自体は turn.md に独立エントリが存在しないが、同種ハンドラ `じこさいせい_heal_self` および既実装の `あさのひざし_heal_self` がともに priority デフォルト（100）を使用しているため、同一パターンで 100 を採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

回復ロジックはあさのひざしと完全に同一のため、新規関数を追加せず既存の `あさのひざし_heal_self`
（`battle.weather_for(mon)` を用いてばんのうがさ／ノーてんき／エアロックを考慮済み）をそのまま
ハンドラとして登録する。専用関数の複製は不要。

### `src/jpoke/data/moves/move_ka.py`

```python
"こうごうせい": MoveData(
    type="くさ",
    category="status",
    pp=8,
    target="self",
    flags={"heal"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.あさのひざし_heal_self,
        )
    }
),
```

`target="self"` を明示しないとデフォルト値 `"foe"` になり、`まもる` に誤って防がれたり
`マジックコート` に誤って跳ね返されたりするため必須（`docs/spec/moves/あさのひざし.md` と同様の注意点）。

## 注意点・エッジケース

- あさのひざし と全く同じ回復ロジックを持つため、ハンドラ関数は複製せず共用する
  （`つきのひかり` も同様に `あさのひざし_heal_self` を共用している）
- `weather.sunny` プロパティが `"はれ"` と `"おおひでり"` の両方をカバーしていることを前提とする
- `らんきりゅう` は不利天候に含まれないため、通常時（1/2 回復）として扱う
- HP 満タンチェックはハンドラ内で行う（turn.md の ON_APPLY_MOVE priority 100「自己回復技: HP満タン」は失敗判定の確認に相当するが、既存パターンに倣い ON_STATUS_HIT ハンドラ内で一括処理する）
- `MoveData.target` を明示しないと `is_blocked_by_protect` / `is_reflectable` が `target == "foe"` を前提に判定するため、`まもる` に防がれたり `マジックコート` に跳ね返されたりする誤動作が起きる
