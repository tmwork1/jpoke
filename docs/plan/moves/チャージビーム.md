# 実装計画書: チャージビーム（再レビュー）

## 仕様要約

`docs/spec/moves/チャージビーム.md` 参照。

* タイプ: でんき / 分類: 特殊 / 威力: 50 / 命中率: 90 / PP: 12 / 対象: 1体選択
* 追加効果として、70%の確率で自分の『とくこう』ランクを1段階上げる。

## 既存実装の確認結果とレビュー指摘

`src/jpoke/handlers/move_attack.py` に `チャージビーム_boost_spa_C` が実装済みで、
共通ヘルパー `modify_attacker_stats`（`handlers/move.py`）を通じて
`battle.resolve_secondary_chance` 経由で追加効果確率補正を適用しつつ、
70%の確率で `ctx.attacker` の `spa` を+1する。ロジック自体はしんぴのちから・
フレアソング・ほのおのまい（同じ「自分のとくこうが50〜100%で上がる」パターン）と
一致しており、修正不要。

一方、`data/moves/move_ta.py` の `チャージビーム` エントリには以下の不備があり、
本レビューで修正した。

* `pp=10` になっていたが、`docs/champions/move_list.txt` の基準値 `12` に修正した
  （一次情報のGen9値10は移行漏れのバグ）。
* `secondary_effect` フラグが付与されていなかった。追加効果（70%でとくこう上昇）を
  持つ技であるため、ちからずく（威力1.3倍・追加効果無効化）との相互作用に必要な
  フラグを追加した。

## ハンドラ構成

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| チャージビーム | `Event.ON_HIT` | デフォルト（100） | `attacker:self`（デフォルト） | 70%の確率で自分のとくこうを1段階上げる |

* `docs/spec/turn.md` の `Event.ON_HIT` 表には「自分のランクを上げる追加効果」が
  priority=10 として記載されているが、しんぴのちから・フレアソング・ほのおのまい等
  同種の既存実装がいずれもデフォルト優先度（100）で登録されており、単独の攻撃技として
  同一 ON_HIT 内で競合する他ハンドラが存在しないため、既存実装の一貫性を優先し
  priority は変更しない。

## リーサル計算（新規実装）

`docs/progress/move.md` の該当行は「リーサル実装: n/a」「リーサルテスト: n/a」と
記載されていたが、命中後に自分の『とくこう』ランクが1段階上がる効果は以降の
攻撃のダメージ分布に影響するため、しんぴのちから・フレアソング・ほのおのまいと同様に
リーサルハンドラの対象とすべきである（n/a 表記は誤りと判断し、本レビューで修正する）。

* `src/jpoke/handlers/lethal.py` に `チャージビーム_boost_spa` を新規実装した
  （`しんぴのちから_boost_spa` / `フレアソング_boost_spa` / `ほのおのまい_boost_spa` と
  同一パターン）。
  * イベント: `LethalEvent.ON_HIT`
  * subject: `"attacker"`
  * `ctx.move_secondary` が真のとき、`ctx.attacker.rank["spa"]` を `clamp_stats` で
    1段階上げる。
* `data/moves/move_ta.py` の `チャージビーム` エントリに
  `lethal_handlers={LethalEvent.ON_HIT: LethalHandler(l.チャージビーム_boost_spa)}` を追加した。
