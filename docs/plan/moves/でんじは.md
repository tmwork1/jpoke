# でんじは 実装計画書

## 仕様要約

- 相手を **まひ** 状態にする変化技
- でんきタイプ / PP 20（champions基準。docs/champions/move_list.txt と一致） / 命中率 90
- 失敗条件:
  1. 対象がすでに状態異常 OR でんきタイプ
     （`ailment_manager.apply` 内部の汎用チェックで処理されるため、技固有の失敗判定ハンドラは不要）
  2. 対象が **じめんタイプ** である場合、タイプ相性により無効になる（でんじは固有の例外挙動。
     ポケモンWiki: 「変化技だが、例外的に『効果がない』場合のみタイプ相性の影響を受けて無効になる。
     じめんタイプのポケモンには無効となる。相手がねらいのまとを持っている場合は有効になる。」）
     - この判定はでんじは固有であり、フレームワーク側の汎用処理には存在しないため、
       技固有の `ON_BEFORE_APPLY_MOVE` ハンドラを追加する必要がある

## 実装方針

まひ付与そのものは `ailment_manager.apply(defender, "まひ")` で行う。
`apply` メソッドが既存の状態異常チェック・タイプ相性チェック（`_can_apply_by_type` の
`target.has_type("でんき")`）・`ON_BEFORE_APPLY_AILMENT` イベント（じゅうなん等の特性、
しんぴのまもり等の場の状態、みがわりによる無効化）を汎用的に処理する。

一方、でんじはは変化技でありながら「技そのもの」のタイプ相性（でんき技 対 じめんタイプ = 0倍）
の影響を受けるという例外仕様を持つ。これは `_can_apply_by_type` の対象外（あちらは「まひ状態」に
対する免疫であり、でんき技自体の無効化ではない）であり、他の状態異常技には見られない、
でんじは固有の失敗条件である。そのため `battle.damage_calculator.calc_def_type_modifier(ctx)` を
用いてでんき技のタイプ相性を計算し、0倍（無効）なら技を失敗させる専用ハンドラを追加する。
`calc_def_type_modifier` は `ON_CALC_DEF_TYPE_MODIFIER` イベントを経由するため、ねらいのまと
（タイプ無効を1倍に補正するアイテム）による無効化解除も自動的に反映される。

でんきエンジン・ちくでん・ひらいしんの各特性による無効化は、`handlers/ability.py` の
`_apply_type_absorb` が `Event.ON_BEFORE_APPLY_MOVE` で汎用的に処理する
（`ctx.move.type == "でんき"` かつ `ctx.move.target == "foe"` の技全般に効くため、
でんじは側で個別対応は不要）。

失敗時（`apply` が `False` を返した場合）は `MoveExecutor._execute_status_hit` が
`ON_STATUS_HIT` の戻り値の偽を検知し、汎用的に `LogCode.MOVE_FAILED` を記録するため、
まひ付与自体の失敗ログはハンドラ側で出す必要がない。ただしじめんタイプによる無効化は
「タイプ無効」であるため、`_check_hit_by_type`（攻撃技用）に倣い `LogCode.MOVE_IMMUNED` を
明示的に記録する。

### ハンドラ構成

| イベント | Priority | subject_spec | 役割 |
|----------|----------|---------------|------|
| `ON_BEFORE_APPLY_MOVE` | 130 | attacker:self（省略） | じめんタイプの相手に無効（タイプ相性判定） |
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | まひ状態を相手に付与 |

### Priority 根拠

- `ON_BEFORE_APPLY_MOVE` priority=130: `docs/spec/turn.md` に `ON_BEFORE_APPLY_MOVE` 自体の
  priority 一覧は未掲載のため、既存の同種ハンドラ（`やどりぎのタネ_can_apply` が
  `ON_BEFORE_APPLY_MOVE` priority=130 でくさタイプ無効を判定している）を参考に priority=130 を採用する。
  でんきエンジン等の吸収特性ハンドラ（`ON_BEFORE_APPLY_MOVE` priority=100、AbilityHandler
  デフォルト）より後に評価されるため、吸収特性が優先して発動する（正しい優先順位）。
- `ON_STATUS_HIT`：MoveHandler のデフォルト priority（`docs/spec/turn.md` 未掲載。既存の
  `おにび` `どくどく` 等の状態異常技ハンドラと同様にデフォルト値を使用）

### subject_spec

- 両イベントとも `attacker:self`（MoveHandlerデフォルト）。`AttackContext` の中で
  `ctx.attacker`（使用者）・`ctx.defender`（対象）を参照する。

## 実装箇所

### handlers/move_status.py（五十音順: でんじはは「て」行）

```python
def でんじは_can_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """でんじはの失敗条件: 変化技だが例外的にタイプ相性の影響を受け、
    でんき技が無効なじめんタイプの相手には無効になる（ねらいのまと所持時は無効化解除）。"""
    if battle.damage_calculator.calc_def_type_modifier(ctx) == 0:
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_IMMUNED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="タイプ無効")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def でんじは_apply_ailment_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="まひ")
```

### data/moves/move_ta.py（「でんじは」MoveData）

```python
"でんじは": MoveData(
    type="でんき",
    category="status",
    pp=20,
    accuracy=90,
    handlers={
        Event.ON_BEFORE_APPLY_MOVE: h.MoveHandler(
            hs.でんじは_can_apply,
            priority=130,
        ),
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.でんじは_apply_ailment_to_defender,
        ),
    }
),
```

## 注意点・エッジケース（review-test 向け）

1. **じめんタイプの相手には無効**: 単タイプ・複合タイプ（例: じめん/ひこう）どちらも
   `calc_def_type_modifier` の型相性計算で正しく判定される。
2. **ねらいのまと**: `ねらいのまと_negate_immunity`（`ON_CALC_DEF_TYPE_MODIFIER`）が
   0倍を1倍に補正するため、`calc_def_type_modifier` を使う実装であれば自動的に反映される。
3. **でんきタイプの相手には無効（まひ免疫）**: `ailment_manager._can_apply_by_type` が
   `target.has_type("でんき")` で判定するため、追加チェック不要。
4. **でんきエンジン・ちくでん・ひらいしん**: `handlers/ability.py` の `_apply_type_absorb` が
   `ON_BEFORE_APPLY_MOVE` priority=100（AbilityHandlerデフォルト）で汎用的に処理するため、
   でんじは固有のじめんタイプ判定（priority=130）より先に評価され、正しく優先される。
5. **命中率90%**: 第七世代以降の値。`accuracy=90` を MoveData に設定する（champions基準）。
6. **PP20**: `docs/champions/move_list.txt` の値と一致（Gen9本家も20で変更なし）。
7. **まもる・マジックコート**: 変化技かつ `target=foe` のデフォルト挙動でまもる・マジックコートの
   対象になる。個別フラグ設定は不要。
