# 計画書: きあいだめ

## 仕様要約

- 自分を「きゅうしょアップ」状態（急所ランク+2）にする
- すでに「きゅうしょアップ」状態の時は失敗する
- タイプ: ノーマル、変化、PP: 20（`docs/champions/move_list.txt` 準拠）、対象: 自分
- 「きゅうしょアップ」揮発性状態は実装済み

出典: `docs/spec/moves/きあいだめ.md`

---

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100 (省略・`MoveHandler`デフォルト) | `attacker:self` (省略・`MoveHandler`デフォルト) | きゅうしょアップ揮発性状態を自分に付与する（急所ランク+2） |

---

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 内の揮発性状態付与技の記載なし。
既存の同種ハンドラ（`アクアリング_apply`、`でんじふゆう_apply` 等）がすべて priority=100
（`MoveHandler` デフォルト）で登録されているため、100 を使用する。

「すでに状態変化」時の失敗は `docs/spec/turn.md` 100番台「きあいだめ: すでに状態変化」に
対応するが、専用の `ON_BEFORE_APPLY_MOVE` ハンドラは不要。`apply_volatile_to_attacker` が
呼び出す `battle.volatile_manager.apply()` が対象に既に同名の揮発性状態がある場合に
`False` を返す仕組みになっており、これがそのまま技の失敗として扱われる
（`アクアリング`・`でんじふゆう`・`ちょうはつ` など同種の技も同じ仕組みで処理しており、
専用の重複チェックハンドラを持たない）。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

```python
def きあいだめ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    # 第9世代: 急所ランク+2。countにランク加算値を格納する
    return apply_volatile_to_attacker(battle, ctx, value, volatile="きゅうしょアップ", count=2)
```

### `src/jpoke/data/moves/move_ka.py`

`"きあいだめ"` エントリに `target="self"` を必ず指定する
（`MoveData.target` のデフォルトは `"foe"` であり、指定を忘れると
`Move.is_blocked_by_protect` / `Move.is_reflectable` が誤って `True` になり、
相手の「まもる」で防がれたり「マジックコート」で跳ね返される不具合になる）:

```python
    "きあいだめ": MoveData(
        type="ノーマル",
        category="status",
        pp=20,
        target="self",
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.きあいだめ_apply,
            ),
        }
    ),
```

## 注意点・エッジケース

- すでに「きゅうしょアップ」状態の場合の失敗は `apply_volatile_to_attacker` →
  `battle.volatile_manager.apply()` が自動的に処理するため、ハンドラ側に追加のガードは不要
- `apply_volatile_to_attacker` は `battle.volatile_manager.apply(ctx.attacker, "きゅうしょアップ", count=2, source=ctx.attacker)` を呼び出す薄いラッパー
- 「サンのみ」との相互作用: サンのみもHP1/4以下で「きゅうしょアップ」を付与するが、
  すでに付与済みの場合は発動しない（仕様通り、`handlers/item.py` の `サンのみ` 実装を参照）
- `target="self"` を指定することで、命中率判定・まもる・マジックコート・おうじゃのしるし・
  オウムがえしの対象外という判定（一次情報の「判定」表）と自動的に整合する
