# したでなめる 実装計画書

## 仕様要約（docs/spec/moves/したでなめる.md 参照）

* タイプ: ゴースト / 分類: 物理 / 威力: 30 / 命中: 100 / PP: 30
* 対象: 相手1体 / 優先度: 0 / 接触あり（直接攻撃）
* 効果: 追加効果として、30%の確率で相手を『まひ』状態にする。

## 現状の実装状況

`src/jpoke/data/move.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。
以下のとおり仕様と一致していることを確認した（修正不要）。

```python
"したでなめる": MoveData(
    type="ゴースト",
    category="physical",
    pp=30,
    power=30,
    accuracy=100,
    flags={"contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.したでなめる_apply_paralysis_to_defender,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 30%の確率で相手を『まひ』状態にする |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def したでなめる_apply_paralysis_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="まひ", chance=0.3)
```

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を
経由するため、ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は汎用ハンドラ側で
処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（状態異常付与）は既存の同種ハンドラ（かみなりのキバ・かみなり・
あくのはどう 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。個別の
priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・まひ付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
2. **おうじゃのしるし・するどいキバとの相互作用**
   - 本技はもともとまひの追加効果を持つため、おうじゃのしるし等によるひるみ追加効果とは
     無関係（本技はひるみ効果を持たない）。
3. **テスト方針**
   - `secondary_chance` を1に固定し、まひの付与有無を確認するテストが既存
     （`test_したでなめる_まひが発動する`）。
