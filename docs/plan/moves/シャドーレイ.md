# 実装計画: シャドーレイ

## 仕様サマリ

- タイプ: ゴースト / 分類: 特殊 / 威力: 100 / 命中率: 100 / PP: 5
  （champions/move_list.txt に記載なしのため、Gen9 本家準拠の PP をそのまま採用）
- 効果: 相手の特性を無視して攻撃する（追加効果なし）
- まもる: ○ / おうじゃのしるし: ○ / よこどり: × / オウムがえし: ○ / 直接攻撃: ×

## 対象イベント

`ON_BEGIN_MOVE` / `ON_END_MOVE` — 特性「かたやぶり」（`src/jpoke/handlers/ability.py` の
`かたやぶり_disable_foe_ability` / `かたやぶり_restore_foe_ability`）と全く同じパターン。
攻撃開始直前に相手の特性を無効化し、攻撃終了後に復元する。

## Priority

`docs/spec/turn.md` に `ON_BEGIN_MOVE` / `ON_END_MOVE` の記載なし。
既存の「かたやぶり」特性ハンドラが `AbilityHandler(func, subject_spec=...)` のみ
（priority 省略＝デフォルト）で実装されているため、同様に priority を省略する。

## 実装手順（実装済み・現状確認）

### handlers/move_attack.py

```python
def シャドーレイ_disable_defender_ability(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """シャドーレイ: 攻撃直前に相手の特性を無効化する（かたやぶりと同様の対象特性）。"""
    mon = ctx.defender
    if mon.ability.has_flag("mold_breaker_ignorable"):
        battle.add_ability_disabled_reason(mon, "シャドーレイ")
    return HandlerReturn(value=value)


def シャドーレイ_restore_defender_ability(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """シャドーレイ: 攻撃終了後に相手の特性の無効化を解除する。"""
    battle.remove_ability_disabled_reason(ctx.defender, "シャドーレイ")
    return HandlerReturn(value=value)
```

`subject_spec` は攻撃フェーズのため `defender:self`（かたやぶりは特性側の実装のため
`attacker:self` だが、シャドーレイは技側の実装のため `defender` を直接参照する形で問題ない。
既存の `MoveHandler` は `subject_spec` を省略した場合デフォルトの対象特定ロジックに従うため、
かたやぶりの `AbilityHandler(subject_spec="attacker:self")` とは事情が異なる点に注意）。

### data/moves/move_sa.py

```python
"シャドーレイ": MoveData(
    type="ゴースト",
    category="special",
    pp=5,
    power=100,
    accuracy=100,
    flags={"ignore_ability"},
    handlers={
        Event.ON_BEGIN_MOVE: h.MoveHandler(
            ha.シャドーレイ_disable_defender_ability,
        ),
        Event.ON_END_MOVE: h.MoveHandler(
            ha.シャドーレイ_restore_defender_ability,
        ),
    },
),
```

`flags={"ignore_ability"}` は分類用のフラグで、実際の無効化ロジックには使用しない
（`types/literals.py` のコメント参照）。フォトンゲイザー・メテオドライブと同じ扱い。

## テスト方針

- ハードロック（`mold_breaker_ignorable` 特性）を持つ相手への効果抜群ダメージ軽減が
  無視されることを確認（`damage_modifier == 4096`）
- 攻撃終了後に相手の特性の無効化が解除される（`defender.ability.enabled is True`）ことを確認
- 上記2点はフォトンゲイザー・メテオドライブと全く同じテストパターンで踏襲する

## 注意事項

- 無視できる特性は「かたやぶり」と同じ対象（`mold_breaker_ignorable` フラグを持つ特性）。
  ファントムガード／プリズムアーマーはこのフラグを持たないため無視されない（対応不要）。
- 第七世代限定の「使用者自身の特性も無視する」仕様は第八世代以降で撤廃されており、
  本プロジェクトは SV（チャンピオンズ）基準のため実装しない（defender のみを対象とする）。
- 技呼び出し系の技（ゆびをふる等）経由で発動した場合は特性無視効果が発生しない仕様が
  あるが、該当する技呼び出し機構自体が本プロジェクトで未実装のため対象外。
- かがくへんかガス/かたやぶり無効化チェックは書かない（既存の `has_flag` 判定のみで十分）。
