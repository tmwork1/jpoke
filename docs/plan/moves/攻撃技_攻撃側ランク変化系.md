## Priority 根拠

`docs/spec/turn.md` の `Event.ON_HIT` セクション（L474〜L489）に以下の記載がある：

```
| 10 | 自分のランクを上げる追加効果 |
| 10 | 自分のランクが下がる反動 |
```

攻撃側のランク上昇は priority=10、反動によるランク低下も priority=10。
既存実装（`アームハンマー_modify_attacker_stats`、`アイスハンマー_modify_attacker_stats` 等）も `MoveHandler(func)` のデフォルト priority=100 を使用しているが、ON_HIT 内での順序は他のハンドラとの相対的な優先度が問題になる場合にのみ影響する。
本バッチでは既存実装と同様に `MoveHandler` デフォルト（priority=100）で登録する。turn.md の priority=10 は ON_HIT イベント内での処理グループの優先度を示しており、MoveHandler の `priority` 引数はイベント内での個別ハンドラ順序を指定するものであるため、既存パターン（priority 引数省略）に合わせる。

## 注意点・エッジケース（review-test 向け）

### All+1 技（あやしいかぜ・げんしのちから・ぎんいろのかぜ）
- 発動確率は 10%。`modify_attacker_stats` の `chance=0.1` 引数で制御される。
- 全能力（A, B, C, D, S）が同時に+1上昇する。`battle.modify_stats` が複数能力の同時変化を受け付けることを確認済み（既存の `ばかぢから` や `インファイト` が `{"A": -1, "B": -1}` 形式を使用している）。
- 10%発動のため、テストでは `battle.random` をモックして確率固定が必要。

### ダイヤストーム（B+2）
- 50% でぼうぎょが2段階上昇。`stats={"B": 2}` で問題なし（既存の `アシッドボム` が `{"D": -2}` を使用している）。

### Ｖジェネレート（BDS-1）
- ぼうぎょ・とくぼう・すばやさが同時に-1。一度のヒットで3能力が下がる。
- スケイルノイズ（B-1）やインファイト（BD-1）の延長上のパターンで対応可能。

### ホイールスピン（S-2）
- 100% でじぶんのすばやさが-2。リフレクターやしろいきりなどの場の状態は攻撃側のランク変化には影響しないため、`modify_attacker_stats` 内での `battle.modify_stats` が正しく処理する。

### こうそくスピン（S+1）と場の状態解除
- `turn.md` L534 に「こうそくスピン等で場の状態解除」が `Event.ON_DAMAGE` priority=100 に記載されている。場の状態解除（まきびしなど）は別のハンドラで実装すべき内容であり、本バッチの対象外（S+1効果のみ実装する）。

### ぶちかまし vs インファイト・アーマーキャノン
- 同じ BD-1 パターン（`インファイト_modify_attacker_stats`、`アーマーキャノン_modify_attacker_stats` と同実装パターン）。
- `_move_secondary_1.md` では `ぶちかまし attacker 1 0 -1 0 -1 0` と確認。

### いじげんラッシュの発動条件
- `turn.md` L252「いじげんラッシュ等: 姿が適格でない」で失敗条件が記述されているが、これは `ON_TRY_MOVE_1` priority=30 の別ハンドラで実装する内容。本バッチではヒット後の B-1 効果のみを実装する。

### MoveData の accuracy フィールドについて
- `ダイヤストーム` の accuracy は仕様通り 95 を使用（現行データでは accuracy キーが未設定の状態のため、95 に設定する）。
- `しんぴのちから` の accuracy は仕様通り 90。

### はがねのつばさ（B+1, 10%）
- スケイルノイズ（B-1, 100%）とは異なり確率発動。
- `はがねのつばさ` のラベルに関して、現行 data/move.py では `labels=["contact"]` のみ設定されているため、`secondary_effect` を追加する。

### ゴールドラッシュ（C-1）
- タイプが `はがね` で特殊技。`ゴールドラッシュ_modify_attacker_stats` という関数名で実装する。
- C-2系（オーバーヒート等）と異なり -1 のみ。

### フレアソング（C+1, 100%）
- sound ラベル付き。追加効果はヒット後のみ。
- チャージビームは 70% 確率だが、フレアソングは 100% 確率で C+1。

### バリアーラッシュ（B+1, 100%）
- 確率なし（chance=1 = デフォルト）。`modify_attacker_stats` の `chance` 引数は省略可能。

### ハンドラ側のガード不要な理由
- `battle.modify_stats` が内部でランク上限（+6）・下限（-6）のチェックを行うため、ハンドラ側での境界チェックは不要。
- `OnHit` イベントはダメージ発生後に発火されるため、攻撃が当たった場合のみ実行される。みがわりへのヒットでも発動する（turn.md: 「ダメージ発生後の処理（みがわりに被弾しても発動）」）。
