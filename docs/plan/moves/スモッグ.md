# スモッグ 実装計画書

## 仕様要約（docs/spec/moves/スモッグ.md 参照）

* タイプ: どく / 分類: 特殊 / 威力: 30 / 命中: 70 / PP: 20
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果: 追加効果として40%の確率で相手をどく状態にする。

## 現状の実装状況

`src/jpoke/data/moves/move_sa.py` に既に登録済み。

```python
"スモッグ": MoveData(
    type="どく",
    category="special",
    pp=20,
    power=30,
    accuracy=70,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.スモッグ_apply_poison_to_defender,
        )
    }
),
```

`src/jpoke/handlers/move_attack.py` にハンドラ実装済み。

```python
def スモッグ_apply_poison_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="どく", chance=0.4)
```

内容は仕様書と一致しており、修正不要。

## ハンドラ構成

| event | priority | subject_spec | 役割 |
|---|---|---|---|
| ON_DAMAGE_HIT | 100（デフォルト） | attacker:self（デフォルト） | 40%の確率で相手をどく状態にする |

`apply_ailment_to_defender` は `handlers/move.py` の既存共通関数を使用する
（`battle.resolve_secondary_chance` で確率解決後 `battle.ailment_manager.apply` を呼ぶ）。

## Priority 根拠

どく付与は `Event.ON_DAMAGE_HIT`。`docs/spec/turn.md` の `Event.ON_DAMAGE` には
「追加効果（特殊なもの除く）」が priority=10 と掲載されているが、10まんボルト・スパーク・ずつき等、
既存の同種「追加効果付与」ハンドラがすべて `ON_DAMAGE_HIT` + デフォルト priority=100 で登録されて
いるため、同一パターンを踏襲する。

`docs/progress/move.md` のスモッグ行は「リーサル実装」「リーサルテスト」ともに n/a のため対象外。

## 注意点・エッジケース（review-test 向け）

### どくの無効条件
- どく・はがねタイプはどく無効。`apply_ailment_to_defender` → `battle.ailment_manager.apply` 内部で
  判定されるため、本技側ハンドラに追加ガードは不要。

### PPの扱い
- `docs/champions/move_list.txt` にスモッグのエントリが存在しない。第九世代基準のPP20は
  既に統一4段階（8/12/16/20）のひとつであり、`changes_from_sv.md` の技個別変更点一覧にも
  スモッグの記載がないため、PP20のまま変更不要と判断する。

### 確率発動のテスト方針
- `secondary_chance` 引数で `battle.random` を制御し、確率発動ケース（発動/非発動）を両方テストする。
- 既存テストは「発動する」ケースのみだったため、10まんボルト・スパーク等の同系統技に合わせて
  「発動しない」ケースのテストを追加する。
