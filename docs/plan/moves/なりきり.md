# 計画書: なりきり

## 仕様要約

- タイプ: エスパー / 変化技 / PP 12 / 命中 - (必中) / 対象: 相手1体
- 効果: 自分の特性を対象の特性と同じものに変化させる
- 失敗条件:
  - 対象の特性が `uncopyable` フラグを持つ（マルチタイプ・イリュージョン等、
    「コピーできない特性」）
  - 対象の特性がふしぎなまもりである（`uncopyable` フラグとは別枠の例外。
    トレース・なかまづくりでは有効だが、なりきりのみ無効）
  - 使用者自身の特性が `protected` フラグを持つ（「上書きできない特性」。
    ただし使用者自身がふしぎなまもり所持者の場合は成功する。ふしぎなまもりは
    `protected` フラグを持たないため、この判定は自然に満たされる）
  - 使用者と対象の特性が既に同じ場合（第五世代以降の仕様）
  - 使用者自身の特性変更がとくせいガード等で防がれる場合
- コピー元は相手の現在の特性名（`ability.name`。とくせいなし状態なら空文字列）
- 使用者が交代すると元の特性に戻る（`change_ability` は `base_name` を更新しないため）
- 詳細は `docs/spec/moves/なりきり.md` を参照。
- `docs/spec/turn.md` 記載:
  - `Event.ON_APPLY_MOVE`（実装上 `ON_BEFORE_APPLY_MOVE`）priority 100:
    「なりきり: 失敗条件」

## 実装内容（修正後）

### `handlers/move_status.py`（五十音順: な の位置）

```python
def なりきり_can_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """なりきりの失敗条件チェック。

    以下の場合は失敗する:
    - 対象の特性が uncopyable フラグを持つ場合（かわりもの・マルチタイプなど固有の特性）
    - 対象の特性がふしぎなまもりの場合
      （ふしぎなまもりはトレースでは有効だが、なりきりに対してのみ無効という
        例外があるため uncopyable フラグとは別に個別判定する）
    - 使用者自身の特性が protected フラグを持つ場合（上書きできない特性）
    - 使用者と対象の特性が既に同じ場合
    - 使用者自身の特性変更がとくせいガード等で防がれる場合
    """
    attacker_ability = ctx.attacker.ability
    defender_ability = ctx.defender.ability
    if (
        defender_ability.has_flag("uncopyable")
        or defender_ability.base_name == "ふしぎなまもり"
        or attacker_ability.has_flag("protected")
        or attacker_ability.base_name == defender_ability.base_name
        or battle.ability_manager.is_change_blocked(ctx.attacker)
    ):
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="なりきり")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def なりきり_change_ability(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """なりきりの効果: 自分の特性を相手の特性と同じにする。"""
    battle.change_ability(ctx.attacker, ctx.defender.ability.name)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース（review-test 向け）

### uncopyable と protected の使い分け

- `uncopyable`: トレース・なかまづくり等で「コピー元」として使えない特性
  （`types/literals.py` コメント参照）。なりきりでは対象（defender）側の
  判定に使う。
- `protected`: スキルスワップ等で「上書き・交換」できない特性。なかまづくりの
  対象（defender）側判定でも使われている。なりきりでは使用者（attacker）
  自身が上書きされる側になるため、この判定を attacker 側に適用する
  （`なかまづくり_can_apply` の `ctx.attacker.ability.has_flag("uncopyable")`
  と対称的な構造になる）。
- ふしぎなまもりはどちらのフラグも持たない特殊な特性。トレース・なかまづくり・
  使用者自身のなりきり（自分から使う分には成功する）では正常に動作するが、
  「なりきりで対象からふしぎなまもりをコピーする」ケースだけ個別に阻止する
  必要があるため、`defender_ability.base_name == "ふしぎなまもり"` の
  名前判定を追加する
  （`docs/spec/abilities/トレース.md` 備考「ふしぎなまもりに対してなりきりは
  効かないが、トレースは有効となる。」、`docs/spec/abilities/ふしぎなまもり.md`
  「特性の変更」節を参照）。

### 同一特性チェック

- `battle.ability_manager.change_ability()` は内部で
  `mon.ability.base_name == ability` の場合に無言で何もせず戻る（no-op）。
  この内部チェックと不整合が起きないよう、`なりきり_can_apply` 側でも
  `attacker_ability.base_name == defender_ability.base_name` を判定し、
  同一特性の場合は明示的に失敗ログを出して停止する
  （なかまづくり `なかまづくり_can_apply` と同じ判定パターン）。

### いえき状態の相手をコピーする場合

- 相手がとくせいなし状態（`ability.name == ""`）でも失敗しない。
  `なりきり_change_ability` は `ctx.defender.ability.name`
  （enabled でなければ空文字列）をそのままコピー先に渡すため、
  結果として使用者の特性も空になる
  （既存テスト `test_なりきり_いえきで無効化された相手の特性はコピーすると空特性になる` 参照）。

### まもる・みがわり・マジックコート

- `flags={"unprotectable", "bypass_substitute"}` によりまもる・みがわりを
  貫通する。`unreflectable` によりマジックコート・マジックミラーで反射されない
  （`docs/wiki/moves/なりきり.html` 技の仕様節に「マジックコート: ×」と
  明記されており、旧計画書の「反射される対象」という記述は誤りだった）。

### 交代後の挙動

- `change_ability` は `base_ability_name` を更新しないため、交代すると元の
  特性に戻る（既存テスト `test_なりきり_交代後に元の特性に戻る` 参照）。

### かがくへんかガス中の挙動

- かがくへんかガス発動中は相手の特性が無効化されているが、特性オブジェクト
  自体は残る。`uncopyable` フラグの判定はオブジェクト（`data.flags`）に
  基づき有効/無効状態に左右されないため、かがくへんかガス下でも `uncopyable`
  特性のなりきりは失敗する。
