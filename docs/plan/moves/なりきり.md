# 計画書: なりきり

## 仕様要約

- タイプ: エスパー / 変化技 / PP 10 / 命中なし（自動命中）
- 効果: 自分の特性を相手の特性と同じにする
- 失敗条件（`ON_BEFORE_APPLY_MOVE` priority 100）:
  - 対象の特性が `uncopyable` フラグを持つ場合（ふしぎなまもり・かわりもの等の固有特性）
- コピー元は相手の現在の特性（変更後の特性を含む。例: シンプルビームで書き換えた後の特性）
- 使用者が交代すると元の特性に戻る（`change_ability` は `base_name` を更新しないため）

## 実装箇所

### `handlers/move_status.py` に追加する関数（五十音順: な の位置）

```python
def なりきり_can_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """なりきりの失敗条件チェック。

    対象の特性が uncopyable フラグを持つ場合は失敗する
    （ふしぎなまもり・かわりものなど固有の特性）。
    """
    if ctx.defender.ability.has_flag("uncopyable"):
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload={"reason": "なりきり"}
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def なりきり_change_ability(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """なりきりの効果: 自分の特性を相手の特性と同じにする。"""
    battle.change_ability(ctx.attacker, ctx.defender.ability.name)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース（review-test 向け）

### 失敗条件の判定フラグ

- なりきりの失敗条件は `uncopyable` フラグ（コピー禁止）であり、`protected` フラグ（上書き禁止）ではない
  - `protected`: スキルスワップ・シンプルビーム・なやみのタネなど「相手の特性を上書きする技」から守る
  - `uncopyable`: なりきり・トレース・なかまづくりなど「特性をコピーする技/特性」から守る
  - なりきりはコピー元の特性（相手の特性）が `uncopyable` なら失敗する
  - なりきりの対象である使用者自身の現在の特性には `protected` / `uncopyable` 制限は発生しない（上書きされるのは使用者自身）

### 特性変更の副作用

- `battle.change_ability` を呼ぶと `ON_ABILITY_DISABLED` → ハンドラ解除 → 特性オブジェクト更新 → `ON_ABILITY_ENABLED` の順でイベントが発火する
- つまりなりきり直後にコピーした特性のハンドラが即座に登録される（例: 「あめふらし」をコピーすれば雨が降り出す）
- 使用者が交代すると `change_ability` は `base_name` を書き換えないため、元の特性に戻る

### コピー元の特性名

- `ctx.defender.ability.name` を参照する（現在有効な特性名）
- `いえき` などで相手が「とくせいなし」状態の場合、`ability.name` は `""` になる可能性がある
  - `battle.change_ability(mon, "")` は `base_name == ""` のガード（`if mon.ability.base_name == ability: return`）に引っかかる可能性があるため、`""` チェックを can_apply に追加することを検討する
  - ただし「とくせいなし」状態への挙動は wiki に詳細な記載がなく、テスト優先度は低い

### 使用者自身の特性が保護された特性の場合

- 使用者（自分）の現在の特性は `change_ability` 内部で `base_name == ability` チェック以外は阻まれない
- コピーが自分と同じ特性になる場合（`ctx.defender.ability.name == ctx.attacker.ability.name`）は `change_ability` 内部のガードで処理され、技自体は成功扱いになる（turn.md 未記載のため fail 扱いかは要確認）

### マジックコート・マジックミラー

- なりきりは変化技かつ相手を対象とするため、マジックコート・マジックミラーで反射される対象となる
- 反射処理は `ON_TRY_MOVE_1` の priority 100 で manager 内部が扱うため、ハンドラ側で追加ガードは不要

### かがくへんかガス中の挙動

- かがくへんかガス発動中は相手の特性が無効化されているが、特性オブジェクト自体は残る
- `uncopyable` フラグの判定はオブジェクトに基づくため、かがくへんかガス下でも `uncopyable` 特性のなりきりは失敗する
