# 実装計画書: いやしのはどう

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 10 / 命中: なし（必中） / 対象: 自分
- 使用者の HP を最大 HP の **1/2** 回復する（端数切り捨て）
- `heal` ラベルあり（`かいふくふうじ` の対象になる）
- 使用者の HP が満タンの場合は技が失敗する

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` セクションに以下の記載がある：

> | 100 | いやしのはどう等: HP満タン |

このイベントはコード上の `ON_BEFORE_APPLY_MOVE` に対応する。ただし、同種の自己回復技（`じこさいせい`・`なまける`・`いのちのしずく`・`こうごうせい`・`つきのひかり`・`あさのひざし`）はすべて `ON_STATUS_HIT`（priority=100）内で HP 満タンチェックと回復処理を一括して行うパターンを採用している。`いやしのはどう` も同パターンで実装し、priority はデフォルト（100）とする。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: `いやしのすず_cure_team_ailment`（266行付近）の後、`いやしのねがい_faint_and_set_side_field`（129行付近）の前に挿入する。

```python
def いやしのはどう_heal_self(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """いやしのはどう: 最大HPの1/2を回復する。HPが満タンの場合は失敗する。"""
    mon = ctx.attacker
    if mon.hp == mon.max_hp:
        return HandlerReturn(value=False, stop_event=True)
    battle.modify_hp(mon, r=1/2)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

## 注意点・エッジケース

- **HP 満タン時の失敗**: `mon.hp == mon.max_hp` で判定し、`HandlerReturn(value=False, stop_event=True)` を返す。`move_executor.py` 側で `MOVE_FAILED` ログが記録されるため、ハンドラ内でのログ記録は不要
- **かいふくふうじとの相互作用**: `heal` ラベルを持つため、`かいふくふうじ` 状態のポケモンがこの技を使用した場合は無効化される。無効化処理は `かいふくふうじ` ハンドラ側で `heal` ラベルを参照して行うため、本ハンドラ側での追加チェックは不要
- **回復量の端数処理**: `battle.modify_hp(mon, r=1/2)` の端数切り捨ては `modify_hp` 内部で処理されるため、ハンドラ側では不要
- **HP が 1 でも減っていれば成功**: `mon.hp == mon.max_hp` で判定するため、回復量が 0 になるケース（最大 HP が 2 未満など）でも満タンでない限り成功扱いとなる
- **じこさいせい・なまけるとの関係**: 同効果（最大 HP の 1/2 回復）の別タイプ技。実装コードは `じこさいせい_heal_self` / `なまける_heal_self` とほぼ同一であり、関数名だけ異なる
- **`ON_STATUS_HIT` の `attacker` コンテキスト**: 変化技の `ON_STATUS_HIT` では `ctx.attacker` が技の使用者を指す。対象: 自分（使用者）なので `ctx.attacker` を参照するのが正しい
