# 実装計画書: いやしのはどう

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 10 / 命中: なし（必中） / 対象: 1体選択（相手）
- 相手（defender）の HP を最大 HP の **1/2** 回復する（端数切り捨て）
- `heal` ラベルあり（`かいふくふうじ` の対象になる）
- 相手の HP が満タンの場合は技が失敗する

## 現状の実装状況

|------|------|
| `data/move.py` エントリ | 登録済み（type="エスパー", category="変化", pp=10, labels=["heal"], handlers あり） |
| `handlers/move_status.py` 関数 | 実装済み |

## ハンドラ構成

|----------|----------|--------------|------|----------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | HP 満タン時失敗チェック → 相手（defender）の HP を最大 HP の 1/2 回復 | `handlers/move_status.py` |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` セクションに以下の記載がある：

> | 100 | いやしのはどう等: HP満タン |

このイベントはコード上の `ON_BEFORE_APPLY_MOVE` に対応する。ただし、同種の回復技（`じこさいせい`・`なまける`・`いのちのしずく`・`こうごうせい`・`つきのひかり`・`あさのひざし`）はすべて `ON_STATUS_HIT`（priority=100）内で HP 満タンチェックと回復処理を一括して行うパターンを採用している。`いやしのはどう` も同パターンで実装し、priority はデフォルト（100）とする。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

```python
def いやしのはどう_heal_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """いやしのはどう: 相手の最大HPの1/2を回復する。HPが満タンの場合は失敗する。"""
    mon = ctx.defender
    if mon.hp == mon.max_hp:
        return HandlerReturn(value=False, stop_event=True)
    battle.modify_hp(mon, r=1/2)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

```python
"いやしのはどう": MoveData(
    type="エスパー",
    category="変化",
    pp=10,

    labels=["heal"],
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.いやしのはどう_heal_defender,
        ),
    }
),
```

## 注意点・エッジケース

- **HP 満タン時の失敗**: `mon.hp == mon.max_hp` で判定し、`HandlerReturn(value=False, stop_event=True)` を返す。`move_executor.py` 側で `MOVE_FAILED` ログが記録されるため、ハンドラ内でのログ記録は不要
- **回復対象は defender（相手）**: `じこさいせい` などの自己回復技と異なり、`ctx.defender` を参照する点に注意。`heal` ラベルは attacker 側の `かいふくふうじ` ではなく defender 側に適用される
- **かいふくふうじとの相互作用**: `heal` ラベルを持つため、`かいふくふうじ` 状態のポケモンに対してこの技を使用した場合は無効化される。無効化処理は `かいふくふうじ` ハンドラ側で `heal` ラベルを参照して行うため、本ハンドラ側での追加チェックは不要
- **回復量の端数処理**: `battle.modify_hp(mon, r=1/2)` の端数切り捨ては `modify_hp` 内部で処理されるため、ハンドラ側では不要
- **HP が 1 でも減っていれば成功**: `mon.hp == mon.max_hp` で判定するため、回復量が 0 になるケース（最大 HP が 2 未満など）でも満タンでない限り成功扱いとなる
- **メガランチャーとの相互作用**: メガランチャー特性を持つポケモンが使うと、回復量が defender の最大 HP の 3/4 になる
