# ナイトヘッド 実装計画書（再レビュー）

## 仕様要約（docs/spec/moves/ナイトヘッド.md 参照）

* タイプ: ゴースト / 分類: 特殊 / 威力: -（固定ダメージ） / 命中: 100 / PP: 16
* 対象: 相手1体 / 優先度: 0 / 直接攻撃技ではない
* 効果: 使用者のレベルと同じ固定ダメージを与える。本プロジェクトはレベル50固定のため実質常に50ダメージ。
* タイプ相性は無効化（ノーマルタイプへは「効果がない」）のみ適用され、それ以外の相性補正は受けない（固定ダメージ技共通仕様）。

## 現状の実装状況

`src/jpoke/data/moves/move_na.py` に既に登録済み。

```python
"ナイトヘッド": MoveData(
    type="ゴースト",
    category="special",
    pp=16,
    power=0,
    accuracy=100,
    flags={"fixed_damage"},
    handlers={
        Event.ON_MODIFY_MOVE_DAMAGE: h.MoveHandler(
            ha.level_fixed_damage,
            subject_spec="attacker:self",
            priority=15,
        )
    }
),
```

`handlers/move_attack.py` の `level_fixed_damage`（既存の共通ハンドラ。ちきゅうなげと共用）:

```python
def level_fixed_damage(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """使用者レベルと同値の固定ダメージを計算する。"""
    return HandlerReturn(value=ctx.attacker.level)
```

### 今回のレビューで検出した問題点

`docs/plan/moves/ちきゅうなげ.md` および `docs/plan/moves/いかりのまえば.md` で修正された問題と
同種の不備が `ナイトヘッド` にも残っていた。

1. **PP の誤り**: `pp=15`（Gen9本家の値）のまま実装されていたが、
   `src/jpoke/data/ps-champ-ja/moves.json`（612行目: `ナイトヘッド ゴースト 特殊 - 100 16 ...`）では
   `16` が正しい値（`docs/champions/changes_from_sv.md` の全技PP 8/12/16/20 統一ルールに合致）。
   `pp=16` に修正した。

2. **`priority` 未指定によるバグ（`ちきゅうなげ` と同種の priority バグ）**: `level_fixed_damage` が
   `priority` 未指定（デフォルト100）のまま `Event.ON_MODIFY_MOVE_DAMAGE` に登録されていたため、
   以下のハンドラより **後** に実行されていた。

   - `みがわり_block_damage`（`data/volatile.py`, priority=20）
   - `こらえる_endure`（`data/volatile.py`, priority=95）

   固定ダメージ技は「通常のダメージ計算式を経由せず、`ON_MODIFY_MOVE_DAMAGE` ハンドラ内で
   初めて基礎ダメージ値（レベル値）が確定する」という実装方式のため、`level_fixed_damage` は
   「基礎ダメージの確定」の役割を持ち、みがわり・こらえる等の「確定済みダメージへの修正」
   ハンドラより **前** に実行されなければならない。`priority=15` を追加して修正した
   （`ちきゅうなげ`・`いかりのまえば` と同じ値・同じ根拠）。

   なお、`がんじょう`・`きあいのタスキ`（いずれも `ON_MODIFY_MOVE_DAMAGE` priority=100）との
   競合は理論上あり得るが、レベル50における実際のHP実数値の下限（`level+10=60` 程度）が
   ナイトヘッドの最大ダメージ（使用者レベル、本プロジェクトでは50固定）を常に上回るため、
   満タンHPからの一撃ひんしは実質発生せず、実害のある競合は生じない
   （`ちきゅうなげ`・`いかりのまえば` の計画書における同種の記述を参照）。

## 修正方針

1. `pp=16` に修正する（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）。
2. `level_fixed_damage` の登録に `priority=15` を追加する
   （`みがわり`=20 より前、`ばけのかわ`=10 より後）。

## ハンドラ構成

| Event | 対象 | subject_spec | priority | 関数 |
|---|---|---|---|---|
| ON_MODIFY_MOVE_DAMAGE | ナイトヘッド | attacker:self | 15 | `ha.level_fixed_damage` |

## Priority 根拠

`docs/spec/turn.md` に `ON_MODIFY_MOVE_DAMAGE`（固定ダメージ技のダメージ確定）は
個別掲載されていない未掲載イベントのため、既存の同種ハンドラ
（`みがわり`=20, `こらえる`=95, `ばけのかわ`=10, `ちきゅうなげ`の`level_fixed_damage`=15,
`いかりのまえば`の`half_damage`=15）を参照し、`みがわり`より前に基礎ダメージを確定させる
必要から `priority=15` とした（`ちきゅうなげ.md` と同じ値・同じ根拠）。

## 注意点・エッジケース（review-test 向け）

1. **ノーマルタイプへの無効化**
   - `_check_hit_by_type`（`move_executor.py`）が `calc_def_type_modifier` を
     参照して別途無効化判定を行うため、`fixed_damage` フラグや `power=0` とは
     独立して正しく無効化される。実装済み・修正不要（既存の同種技のテストで確認済みの
     共通経路のため、ナイトヘッド向けにも新規テストを追加する）。

2. **みがわり・こらえるとの相互作用**
   - 上記の priority 修正により正しく動作することをテストで確認する。

3. **がんじょう・きあいのタスキ**
   - レベル50におけるHP実数値の下限（60程度）がナイトヘッドの最大ダメージ（50）を
     常に上回るため、満タンHPからの一撃ひんしは発生せず、実害のある競合は生じない。

4. **使用者レベル依存の固定ダメージ**
   - 本プロジェクトは全ポケモンレベル50固定（Pokemon Champions ルール）のため、
     実質常に50固定ダメージとなる。`level=50` を明示したテストで確認する。
