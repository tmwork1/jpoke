# ほのおのキバ 実装計画書

## 仕様要約（docs/spec/moves/ほのおのキバ.md 参照）

* タイプ: ほのお / 分類: 物理 / 威力: 65 / 命中: 95 / PP: 15
* 対象: 相手1体 / 優先度: 0 / 接触あり（直接攻撃）
* 効果: 追加効果として、10%の確率で相手を『やけど』状態にし、10%の確率で相手をひるませる。
  2つの効果はそれぞれ独立に判定される（同時に発生することもある）。
* かみつき技（`flags` に `"bite"` を含む）。特性『がんじょうあご』の対象。

## 現状の実装状況（レビュー時点）

`src/jpoke/data/moves/move_ha.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済みだったが、
仕様と不一致があった（下記「修正内容」参照）。

### 修正前（誤り）

```python
def ほのおのキバ_apply_flinch_or_burn(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ほのおのキバの追加効果: 5%でやけど、5%でひるみを付与する。

    単一乱数で判定: r < base*0.5 → やけど、r < base → ひるみ。
    """
    base = battle.resolve_secondary_chance(ctx, 0.1)
    r = battle.random.random()
    if r < base * 0.5:
        return HandlerReturn(value=battle.ailment_manager.apply(
            ctx.defender, "やけど", source=ctx.attacker, ctx=ctx
        ))
    if r < base:
        return HandlerReturn(value=battle.volatile_manager.apply(
            ctx.defender, "ひるみ", source=ctx.attacker
        ))
    return HandlerReturn(value=value)
```

上記は単一の乱数でやけど・ひるみを排他的に選択する実装になっており、以下と矛盾していた。

* 一次情報（docs/wiki/moves/ほのおのキバ.html「技の仕様」節）:
  「2つある追加効果の発動判定はそれぞれ個別に行われる。ひるみとやけどの効果が同時に発生することもある。」
* docs/progress/move.md のほのおのキバ行: 「10%の確率で相手を『やけど』状態にする。10%の確率で、
  相手をひるませる。」（2つの独立した効果として記載）
* docs/spec/moves/_move_secondary_1.md: やけど0.1・ひるみ0.1の2行に分けて掲載されている
  （同時発動を許容する独立判定の技として扱われている）
* 兄弟技かみなりのキバ・こおりのキバは、いずれも独立2判定（`MoveHandler` を2つ登録し、
  それぞれ `apply_ailment_to_defender` / `apply_volatile_to_defender` を chance=0.1 で呼ぶ）で
  実装されており、ほのおのキバだけ異なる実装になっていた。

### 修正後

かみなりのキバ・こおりのキバと同じパターンに合わせ、やけど付与とひるみ付与を独立した
2つの `MoveHandler` に分離する。

```python
def ほのおのキバ_apply_burn_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="やけど", chance=0.1)


def ほのおのキバ_apply_flinch(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_defender(battle, ctx, value, volatile="ひるみ", chance=0.1)
```

```python
"ほのおのキバ": MoveData(
    type="ほのお",
    category="physical",
    pp=15,
    power=65,
    accuracy=95,
    flags={"bite", "contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: [
            h.MoveHandler(ha.ほのおのキバ_apply_burn_to_defender),
            h.MoveHandler(ha.ほのおのキバ_apply_flinch),
        ]
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手を『やけど』状態にする |
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手をひるませる |

`apply_ailment_to_defender` / `apply_volatile_to_defender`（いずれも `handlers/move.py`）が
`battle.resolve_secondary_chance` を経由するため、ちからずくによる追加効果無効化（確率0化）・
威力1.3倍化、`flags={"bite"}` によるがんじょうあごの威力1.5倍化は、いずれも汎用ハンドラ側で
処理される。本技側に個別のガードは不要。やけどとひるみは別々の `MoveHandler` として登録されており、
それぞれ独立に `battle.random.random()` で抽選されるため、仕様どおり同時発生し得る。

## Priority 根拠

ダメージを与えた技の追加効果（状態異常・ひるみ付与）は既存の同種ハンドラ（かみなりのキバ・
こおりのキバ 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。個別の
priority検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **がんじょうあごとの相互作用**
   - `flags={"bite"}` により、がんじょうあご所持ポケモンが使用すると威力が1.5倍（65→97）になる。
     既存の がんじょうあご テストファイル側で他のかみつき技と同様の網羅がされていれば個別追加は不要。
2. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・やけど/ひるみ付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
3. **おうじゃのしるし・するどいキバとの相互作用**
   - `handlers/item.py` の `_INNATE_FLINCH_MOVES` に「ほのおのキバ」が既に含まれており、
     元々ひるみ追加効果を持つ技として、おうじゃのしるし等による追加のひるみ判定が重複しないよう
     処理済み（修正不要）。
4. **2つの追加効果の独立性**
   - やけどとひるみはそれぞれ個別に `battle.random.random()` で抽選されるため、両方同時に発動する
     可能性がある。テストは `secondary_chance=1.0` に固定し、やけど・ひるみそれぞれの付与有無、
     および両方同時発動のケースを確認する（こおりのキバの
     `test_こおりのキバ_こおりとひるみが同時に発動する` と同型のテストを追加する）。
5. **既存テストの修正**
   - `tests/moves_attack/test_move_ha.py` の `test_ほのおのキバ_やけどが選択される` /
     `_ひるみが選択される` / `_やけどかひるみが発動する` は、旧・排他選択実装を前提にした
     テストのため、独立判定の仕様に合わせて書き直す。
   - `tests/moves_status/test_move_ha.py` に誤って混入していた
     `test_ほのおのキバ_ひるみが発動する`（攻撃技なのに moves_status 側に存在）は削除する。
