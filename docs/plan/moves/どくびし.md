# 実装計画書: どくびし

## 仕様要約

- タイプ: どく / カテゴリ: 変化 / PP: 20 / 対象: 相手の場（target="foe_side"）
- 使用後、相手はポケモンを交代するたびに出てきたポケモンが「どく」状態になる
- 2 回使用すると「もうどく」状態になる（最大 2 層）
- どく・はがね・ひこうタイプ、特性「ふゆう」のポケモンには無効
- どくタイプのポケモンが出ると効果が消える

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「設置技: すでに最大」が priority 100 として記載されている。`ステルスロック_set_field` および `まきびし_set_field` と同じパターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「ど」行 — `どくのこな_apply_ailment_to_defender` の後、`どくのいと_*` 系の前（五十音順では「どくびし」は「どくのいと」より前のため、`どくどく_apply_ailment_to_defender` の後、`どくのいと_*` の前）に挿入する。

```python
def どくびし_set_field(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """どくびし: 相手陣営に「どくびし」を 1 層設置する（最大 2 層）。"""
    side = battle.get_side(ctx.defender)
    if not side.activate("どくびし", 1):
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

既存の `"どくびし"` エントリに handlers を追加する。

```python
"どくびし": MoveData(
    type="どく",
    category="変化",
    pp=20,
    target="foe_side",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.どくびし_set_field,
        ),
    }
),
```

## 注意点・エッジケース

- `side.activate("どくびし", 1)` が最大 2 層を超えた場合に False を返すことを、サイドフィールドの実装で確認すること
- 1 層時はどく、2 層時はもうどくの判定はサイドフィールドの `ON_SWITCH_IN` ハンドラ側で処理されるため、技ハンドラ側でのガードは不要
- どくタイプが出た場合の層消去もサイドフィールドハンドラ側の責務
- どく・はがね・ひこうタイプ、ふゆう特性への無効化もサイドフィールドハンドラ側で処理
