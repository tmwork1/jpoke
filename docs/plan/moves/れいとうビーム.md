# れいとうビーム 実装計画書

## 仕様要約

`docs/spec/moves/れいとうビーム.md` を参照。
こおりタイプ・特殊・威力90・命中100・PP10の非直接攻撃技。
追加効果として10%の確率で相手を「こおり」状態にする。

本技は `docs/plan/moves/攻撃技_こおり系.md`（こおり付与技の一括実装計画）に含まれている。
以下はその内容のうち本技に関する部分の抜粋。

## ハンドラ構成

`Event.ON_DAMAGE_HIT` に `MoveHandler` デフォルト設定（`subject_spec="attacker:self"`, `priority=100`）で登録する。
`apply_ailment_to_defender` は `handlers/move.py` の既存共通関数を使用する。

| 技 | event | priority | subject_spec | 役割 |
|---|---|---|---|---|
| れいとうビーム | ON_DAMAGE_HIT | 100 | attacker:self | こおり 10% |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_HIT` セクションに追加効果（状態異常付与）は明示されていないが、
既実装の `かみなりパンチ`・`れいとうパンチ` 等の登録コードと同じ priority=100（`h.MoveHandler` のデフォルト）を採用する。

## 実装箇所

### `src/jpoke/handlers/move_attack.py`

```python
def れいとうビーム_apply_freeze_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="こおり", chance=0.1)
```

### `src/jpoke/data/moves/move_ra.py`

```python
"れいとうビーム": MoveData(
    type="こおり",
    category="special",
    pp=10,
    power=90,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.れいとうビーム_apply_freeze_to_defender,
        )
    }
),
```

※ 登録イベントは `Event.ON_DAMAGE_HIT`。命中後の追加効果付与タイミングとして機能する既存の仕組みであり、
他の直接攻撃・追加効果技（`れいとうパンチ`・`かみなりパンチ` 等）と揃えた登録のため妥当と判断する。

## リーサル計算

- 対象外（`n/a`）。こおりは行動阻害効果であり、ダメージ量には影響しないため専用のリーサルハンドラは不要。

## エッジケース

- こおりタイプの相手はこおり無効（`ailment_manager` 側の共通処理で判定されるため、本技のハンドラに追加ガードは不要）。
- ちからずく所持時は威力1.3倍・追加効果無効となるが、これは `apply_ailment_to_defender` 内の
  `battle.resolve_secondary_chance(ctx, chance)` で汎用的に処理されるため、本技側の実装は不要。
