# 計画書: ねむる

## 仕様要約

- HPと状態異常をすべて回復した後、2ターンの間ねむり状態になる
- タイプ: エスパー、変化、PP: 5、対象: 自分、命中: -（命中判定なし）
- 回復技（`heal` ラベル）

出典: `docs/spec/moves/status_move.md` 行 188

---

## 失敗条件

`docs/spec/turn.md` を踏まえ、以下の失敗条件が存在する。

| 優先度 | 失敗条件 | 処理場所 |
|---|---|---|
| ON_TRY_MOVE_1 Priority 30 | HP が満タン（最大HP = 現在HP） | `ねむる_check` ハンドラ内 |
| ON_TRY_MOVE_1 Priority 30 | すでにねむり状態（`mon.ailment.name == "ねむり"`） | `ねむる_check` ハンドラ内 |
| ON_BEFORE_APPLY_MOVE Priority 100 | ステータス無効化（turn.md「ねむる: ステータス無効化」） | `ねむる_check_apply` ハンドラ内 |

「ステータス無効化」の具体的内容は不明確なため、HP 満タン・ねむり状態の双方チェックを `ON_STATUS_HIT` の冒頭でも行う（他の自己回復技と同一パターン）。
`ON_BEFORE_APPLY_MOVE` Priority 100 では既存実装に倣い追加の失敗チェックハンドラを置く方針とする。

以下は ailment_manager 内の `ON_BEFORE_APPLY_AILMENT` で自動処理されるため、ハンドラ側での個別実装は不要:

- さわぐ状態によるねむり無効（`さわぐ_prevent_sleep` が `ON_BEFORE_APPLY_AILMENT` で処理済み）
- しんぴのまもりによる無効化（`しんぴのまもり_prevent_ailment` が `ON_BEFORE_APPLY_AILMENT` で処理済み）
- ミストフィールドによる無効化（`ミストフィールド_prevent_ailment` が `ON_BEFORE_APPLY_AILMENT` で処理済み）
- エレキフィールドによる無効化（同上）
- ふみん等の特性によるねむり無効（`ON_BEFORE_APPLY_AILMENT` でハンドラが処理済み）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_TRY_MOVE_1` | 30 | `attacker:self` (省略) | HP満タン・すでにねむり状態の失敗チェック |
| `ON_BEFORE_APPLY_MOVE` | 100 | `attacker:self` (省略) | turn.md「ステータス無効化」対応（HP満タン重複チェック） |
| `ON_STATUS_HIT` | 100 (省略) | `attacker:self` (省略) | HP全回復・状態異常全回復・ねむり付与（count=2） |

## Priority 根拠

- `ON_TRY_MOVE_1 Priority 30`: `docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` 表に「ねむる: 失敗条件多数」として Priority 30 に明記されている
- `ON_BEFORE_APPLY_MOVE Priority 100`: `docs/spec/turn.md` の `Event.ON_APPLY_MOVE` 表に「ねむる: ステータス無効化」として Priority 100 に明記されている（実装コード上は `ON_BEFORE_APPLY_MOVE`）
- `ON_STATUS_HIT Priority 100`: 既存の `じこさいせい_heal_self`・`なまける_heal_self` が priority=100（デフォルト）で登録されているため同一とする

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: `ねむりごな_apply_ailment_to_defender` の後、`ねばねばネット_set_side_field` の前（「ね」の中で「ねむりごな」の後）

```python
def ねむる_check(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ねむるの失敗条件チェック。

    以下のいずれかに該当する場合は失敗する:
    - HP が最大HP（すでに満タン）
    - すでにねむり状態
    """
    mon = ctx.attacker
    if mon.hp == mon.max_hp or mon.has_ailment("ねむり"):
        battle.add_event_log(
            mon,
            LogCode.MOVE_FAILED,
            payload={"reason": "ねむる"},
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def ねむる_check_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ねむるのステータス無効化チェック（ON_BEFORE_APPLY_MOVE）。

    HP 満タン・ねむり状態を再チェックする。
    """
    mon = ctx.attacker
    if mon.hp == mon.max_hp or mon.has_ailment("ねむり"):
        battle.add_event_log(
            mon,
            LogCode.MOVE_FAILED,
            payload={"reason": "ねむる_apply"},
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def ねむる_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ねむるの効果: HP全回復・状態異常全回復・2ターンのねむり付与。

    1. HP を最大HPまで回復する
    2. 現在の状態異常を解除する
    3. ねむり状態を付与する（count=2）
    """
    mon = ctx.attacker
    # HP全回復
    battle.modify_hp(mon, v=mon.max_hp - mon.hp)
    # 状態異常を解除（上書きフラグで強制解除）
    battle.ailment_manager.remove(mon)
    # ねむり付与（count=2: 残り2ターン眠る）
    battle.ailment_manager.apply(mon, "ねむり", count=2, source=mon)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース

- **HP回復と ailment.remove の順序**: HP 全回復を先に行い、次に状態異常解除、最後にねむり付与とする。ailment_manager.remove() は `uncurable` な状態異常（ぜったいねむりなど）に対しては `False` を返して解除しない。ねむるが `uncurable` 状態異常を解除しないかは検討要（通常ゲームではぜったいねむりに対してねむるが成功するため、uncurable でも解除できると考えられる）
- **count=2 の意味**: `ailment.py` の `ねむり_check_action` では `battle.ailment_manager.tick(mon)` を行動前に呼ぶため、ねむりの初ターンは tick 後も count=1 が残る（実質2ターン眠る）。count=2 が正しい値かはいびきの仕様書「実質1〜3ターン」との整合性で要確認
- **さわぐ・しんぴのまもり・ミストフィールドによるねむり無効**: ailment_manager.apply() が ON_BEFORE_APPLY_AILMENT を発火するため、既存の各ハンドラが自動的に無効化判定を行う。ねむる_apply 側での重複ガードは不要
- **かいふくふうじ状態での使用**: `labels=["heal"]` が付いているため、かいふくふうじ状態では技自体が使用不可（ON_CHECK_ACTION 内で管理）。ただし HP 回復部分は `ON_MODIFY_HEAL` 経由で 0 にされる可能性があるため、ailment_manager.apply の結果が正常に返るかも確認要
- **HP 満タン チェックの二重実装**: ON_TRY_MOVE_1（priority=30）と ON_BEFORE_APPLY_MOVE（priority=100）の両方でチェックしているが、これは turn.md の記述通りの実装。両イベントそれぞれに対応するハンドラを置く
- **状態異常解除の失敗時の挙動**: `ailment_manager.remove()` が `False` を返した場合（状態異常がない場合や uncurable の場合）でも、HP 回復とねむり付与は継続する（remove の戻り値でハンドラを中断しない）
- **ゆめうつつとの相互作用**: ゆめうつつ（ぜったいねむり特性）がある場合、ailment_manager.apply の uncurable フラグにより上書き不可になる可能性がある
