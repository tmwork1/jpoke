# 計画書: エコーボイス

## 仕様要約

* ノーマルタイプ / 特殊 / 威力40（可変、最大200） / 命中100 / PP16（チャンピオンズ基準、
  詳細は `docs/spec/moves/エコーボイス.md` の根拠を参照） / 対象: 1体選択 / 優先度0 /
  直接攻撃なし / 音技（sound）。
* 場の誰かが直前のターンから連続してこの技を使い続けた場合、使用するたびに威力が
  40ずつ上昇していく（最大200）。使用が途切れると威力は40に戻る。
* 同じターン内で複数回使われても、そのターンの威力上昇は1回分のみ。

## 既存実装の状況

`data/move.py`（`data/moves/move_a.py`）にエントリはあるが `handlers={}`（追加効果なし
扱い）となっており、本技の核心である「連続使用による威力上昇」が全く実装されていない
（バグ）。`docs/progress/move.md` では実装列が `x` になっているが、実際には核心効果が
欠落しているため修正が必要。テストも存在しない。PPも本編基準の15のままで、
チャンピオンズ基準に未移行だった。

## 必要な状態管理

「エコーボイスが直前のターンに（誰であれ）成立した使用があったか」と「現在の威力段階」は
ポケモン単位ではなくバトル全体で1つ判定すればよい情報である（りんしょうの
`battle.round_used_turn` と同じ設計方針）。

`Battle` に新規フィールドを2つ追加する。

### `core/battle.py`

```python
self.echoed_voice_power: int = 40  # エコーボイス: 現在の威力段階
self.echoed_voice_last_turn: int | None = None  # エコーボイス: 直近に成立したターン番号
```

* プリミティブ型（`int` / `int | None`）のため `fast_copy` の追加のディープコピー対象
  指定は不要（`round_used_turn` と同様）。

## ハンドラ構成

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| エコーボイス | `Event.ON_TRY_MOVE_1` | 50 | `attacker:self`（デフォルト） | 直前ターンからの連続使用なら威力を40上昇（最大200）、途切れていれば40にリセットして確定させる。以後の使用に備えてターン番号と威力を記録する |

`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` 節に「50 | エコーボイス: 次ターン威力上昇」と
明記されているため、この priority を採用する。

この時点（ON_TRY_MOVE_1 priority 50）は、まもる・キングシールド等による無効化判定
（同イベント内 priority 100）や、タイプ相性判定・命中判定（いずれもON_TRY_MOVE_1より後）
より前に発火する。そのため、技が外れた場合やまもる・ゴーストタイプ等に防がれた場合でも
「使われたこと」自体は確定しており、次のターンへの威力上昇の引き継ぎを正しく再現できる。
一方、まひ・ねむり等により行動自体ができなかった場合は `Event.ON_TRY_ACTION` の時点で
行動が失敗しているため、ON_BEGIN_MOVE や ON_TRY_MOVE_1 そのものが発火せず、
「カウントされない」という仕様も自然に満たされる。

`ON_MODIFY_POWER`（ダメージ計算内のイベント）ではなく `ctx.move.power` を直接書き換える
方式を採用する理由: りんしょう（`りんしょう_apply_chain_power`）の前例と同じく、
無効化判定より前の時点で威力を確定させる必要があるため。

```python
def エコーボイス_apply_chain_power(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """エコーボイス: 直前のターンから連続で使用され続けていれば威力を40上昇させる
    （最大200）。途切れていれば威力を40にリセットする。"""
    if battle.echoed_voice_last_turn == battle.turn - 1:
        battle.echoed_voice_power = min(battle.echoed_voice_power + 40, 200)
    elif battle.echoed_voice_last_turn != battle.turn:
        battle.echoed_voice_power = 40
    battle.echoed_voice_last_turn = battle.turn
    ctx.move.power = battle.echoed_voice_power
    return HandlerReturn(value=value)
```

* 1つ目の分岐（直前ターン == turn - 1）: 連続使用のため40上昇。
* 2つ目の分岐（直前ターン != turn、かつ直前ターン != turn - 1）: 使用が途切れているため
  40にリセット。
* どちらの分岐にも当てはまらない場合（`echoed_voice_last_turn == battle.turn`、
  つまり同じターン内で既に使用済み）: 何もせず、既に確定した今ターンの威力をそのまま使う。

```python
"エコーボイス": MoveData(
    type="ノーマル",
    category="special",
    pp=16,
    power=40,
    accuracy=100,
    flags={"sound"},
    handlers={
        Event.ON_TRY_MOVE_1: h.MoveHandler(
            ha.エコーボイス_apply_chain_power,
            priority=50,
        ),
    },
),
```

## テスト

`tests/moves_attack/test_move_a.py` に以下を追加する。

* 初回使用時は威力40のまま。
* 前のターンに使用し、次のターンも連続使用すると威力が80になる。
* 別のポケモンが前のターンに使用していても、連続使用と扱われて威力が引き継がれる。
* 間のターンに使用しなかった場合、次に使用したときは威力が40に戻る。
* 同じターン内で自分・相手の両方が使用しても、威力は両者とも同じ値（上昇は1回分のみ）。
* 連続使用を繰り返すと威力が200で頭打ちになる（5ターン連続で200、6ターン目も200のまま）。

## 注意点・エッジケース

* テラレイドバトル固有の「味方ごとに独立したターン」仕様は本プロジェクト（シングル
  バトルのみ）の対象外。
* 技が外れた・まもるで防がれた場合でも威力が引き継がれる挙動は、`ON_TRY_MOVE_1`
  priority 50 が無効化判定より前に発火することで自動的に再現される（専用の分岐は不要）。
