# 実装計画: ミラーコート

## 仕様サマリ

- タイプ: エスパー / 分類: 特殊 / 威力: -（固定ダメージ） / 命中率: 100 / PP: 20 / 優先度: -5
- 効果: そのターンに相手から受けた特殊技のダメージを2倍にして同じ相手に返す
- 使用したターンに特殊技のダメージを受けていない場合は失敗する
- 複数回の特殊ダメージを受けていた場合（連続技等）は最後の1回分のダメージのみを参照する
- まねっこ（第八世代以前は不可）は本プロジェクトの対象世代（SV）ではコピー可能なため、
  `non_copycat` フラグは付与しない（カウンターとの相違点）

詳細は `docs/spec/moves/ミラーコート.md` を参照。

## 現状の実装状況

`data/moves/move_ma.py` にすでに実装済み。

```python
"ミラーコート": MoveData(
    type="エスパー",
    category="special",
    pp=20,
    power=0,
    accuracy=100,
    priority=-5,
    handlers={
        Event.ON_TRY_MOVE_1: h.MoveHandler(
            ha.ミラーコート_check_can_use,
            subject_spec="attacker:self",
            priority=30,
        ),
        Event.ON_MODIFY_MOVE_DAMAGE: h.MoveHandler(
            ha.ミラーコート_modify_damage,
            subject_spec="attacker:self",
        ),
    },
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_TRY_MOVE_1` | 30 | `attacker:self` | そのターン特殊ダメージを受けていない場合に失敗させる |
| `ON_MODIFY_MOVE_DAMAGE` | 100（デフォルト） | `attacker:self` | ダメージを「直近に受けた特殊ダメージ×2」の固定値にする |

## Priority 根拠

### ON_TRY_MOVE_1 Priority 30

`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` テーブルに以下の記述がある：

```
| 30 | カウンター・ミラーコート: 適格ダメージ無 |
```

したがって priority=30 を採用する。

### ON_MODIFY_MOVE_DAMAGE Priority 100（デフォルト）

`docs/spec/turn.md` に `Event.ON_MODIFY_MOVE_DAMAGE` の個別記載はない（ダメージ計算内部のイベントのため）。
同種の固定ダメージ技である `カウンター` の実装が既存のデフォルト優先度（100）を用いているため、それに合わせる。

## 共通コードとの関係

`core/move_executor.py` の `_execute_hit` で、被弾ダメージが物理か特殊かに応じて
`last_physical_damage_received` / `last_special_damage_received` を「最後の1回分の値」で
上書きする実装になっている（連続技合算を防ぐため、カウンターのレビュー時に修正済み）。
この共通処理は既にミラーコートにも適用されているため、追加のコード変更は不要。

```python
if self.battle.query.deals_physical_damage(ctx.attacker, ctx.move):
    ctx.defender.last_physical_damage_received = actual_damage
else:
    ctx.defender.last_special_damage_received = actual_damage
```

## non_copycat フラグを付与しない理由

`カウンター` には `まねっこ`（第四世代以降）でコピーできない仕様のため `non_copycat` フラグが
付与されているが、ミラーコートは公式Wiki上で以下のとおり明記されている：

> オウムがえし・さきどりでコピーできない(第四世代以降)。
> 第八世代以前ではまねっこでコピーできない。
> スカーレット・バイオレットではコピーできる (カウンターとは異なる)。

本プロジェクトの対象はスカーレット・バイオレット（チャンピオンズ）基準のため、ミラーコートは
まねっこでコピー可能とする。したがって `flags` に `non_copycat` を付与しない現状の実装が正しい。

## 注意点・エッジケース（review-test 向け）

### みがわりが受けたダメージ

みがわりが特殊技を身代わりに受けた場合、`last_special_damage_received` は更新されない
（みがわりへのダメージは `ctx.substitute_damage` 経由で処理され、`_execute_hit` の
`last_special_damage_received` 更新処理を通らない）。このため、みがわり越しの特殊技を受けた
だけの場合ミラーコートは失敗する（仕様通り）。

### 物理技のみ受けた場合

`last_physical_damage_received` のみが更新され、`last_special_damage_received` は0のままなので
ミラーコートは失敗する（`カウンター` と役割が分離されている）。

### 連続技（複数回ヒットする技）を受けた場合

`_execute_hit` はヒットごとに呼ばれるため、`last_special_damage_received` はヒットのたびに
上書きされる。最終的に残るのは最後のヒットのダメージのみとなり、仕様と一致する。

### まねっこでのコピー（SV固有仕様）

`non_copycat` フラグを持たないため、まねっこで直前にミラーコートが使用された場合コピー可能。
コピーされたミラーコートは通常のミラーコートと同様に `attacker:self` の
`last_special_damage_received` を参照するため、まねっこ使用者自身がそのターン特殊ダメージを
受けていなければ失敗する。

### 未対応のエッジケース

`docs/spec/moves/ミラーコート.md` の「既知の未対応エッジケース」を参照。相手の特殊攻撃がHPを
実際には減少させなかった（きあいのハチマキ等でダメージが0に補正された）場合の
「本来は1ダメージを返す」という第五世代以降の仕様は、既存のダメージ処理パイプラインでは
「無効化された攻撃」と区別する手段がないため、対象外とする。
