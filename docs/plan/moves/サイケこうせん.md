# 計画書: サイケこうせん

## 仕様要約（docs/spec/moves/サイケこうせん.md 参照）

- エスパータイプ / 特殊 / 威力65 / 命中100 / PP20 /
  対象: 1体選択 / 優先度0 / 接触なし
- 追加効果として、命中時に10%の確率で相手をこんらん状態にする。
- 追加効果に分類されるため、ちからずく（威力1.3倍・効果無効化）の対象技になる。
- `src/jpoke/data/ps-champ-ja/moves.json` には未収録の技のため、本家シリーズ（ポケモンWiki）の値をそのまま採用する。
- `ねんりき`（威力50・PP25・同一の追加効果）のPP違い上位互換技であり、実装構成もねんりきと同一。

## 既存実装との差分

既存実装（`data/moves/move_sa.py` の `"サイケこうせん"` エントリ、`handlers/move_attack.py` の
`サイケこうせん_apply_confusion_to_defender`）は仕様と一致しており、差分なし。

## ハンドラ構成

### handlers/move_attack.py: `サイケこうせん_apply_confusion_to_defender`（既存・変更なし）

```python
def サイケこうせん_apply_confusion_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """サイケこうせんの追加効果: 10%の確率で相手をこんらん状態にする。"""
    return apply_confusion_to_defender(battle, ctx, value, chance=0.1)
```

- `apply_confusion_to_defender`（`handlers/move.py`）は内部で `battle.resolve_secondary_chance` を
  呼び出すため、`Event.ON_MODIFY_SECONDARY_CHANCE` 経由でちからずく（確率0化・威力1.3倍）の
  ハンドラが自動的に適用される（`secondary_effect` フラグが前提）。
- イベント: `Event.ON_DAMAGE_HIT`（ダメージが実際に適用された後に発動。
  docs/spec/turn.md の `## Event.ON_DAMAGE` 参照）。
  ダメージが発生しなかった場合（無効化・まもる等）はイベント自体が発火しないため、
  追加効果も発生しない仕様を自然に満たす。
- 優先度: デフォルト100（`MoveHandler` のデフォルト値）。
- subject_spec: `MoveHandler` デフォルト（`attacker:self`）のまま。

### data/moves/move_sa.py（既存・変更なし）

"サイケこうせん": MoveData(

## リーサル計算

対象外（`docs/progress/move.md` の「リーサル実装」列が `n/a`。こんらん付与のみでHP分布に影響しない
追加効果のためリーサル計算ハンドラは不要）。

## 注意点・エッジケース

- ノーマルタイプの相手など、技自体が無効化された場合、`Event.ON_DAMAGE_HIT` 自体が発火しないため
  追加効果も発生しない。個別のガードは不要。
- まもるで防がれた場合も同様にダメージが発生せず `Event.ON_DAMAGE_HIT` が発火しないため、
  追加効果も発生しない。
- 既にこんらん状態の相手に対しては `volatile_manager.apply_confusion` 側で重複付与しない
  （`VolatileManager` の既存ロジックに準拠、本技側での重複ガードは不要）。
