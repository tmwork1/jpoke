# 実装計画書: ひかりのかべ

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 30 / 対象: 味方の場（target="own_side"）
- 5ターンの間、相手の特殊攻撃のダメージを半分にする
- 味方が複数の場合は半分ではなく 2/3（シングルバトルでは半減固定）
- 急所に当たった場合は軽減されない
- 交代しても効果は続く

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「ひ」ブロック。現在 `フェザーダンス_modify_defender_stats` の後、`へびにらみ_apply_ailment_to_defender` の前あたり（「ひ」は「フ」「へ」の間）。正確には `ほたるび_modify_attacker_stats` の前、`フェザーダンス_modify_defender_stats` の後に配置する。

```python
def ひかりのかべ_set_side_field(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """ひかりのかべ: 自陣営に「ひかりのかべ」を5ターン設定する。"""
    side = battle.get_side(ctx.attacker)
    if not side.activate("ひかりのかべ", 5):
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

既存の `"ひかりのかべ"` エントリを更新する。`target` を `"own_side"` に変更し、handlers を追加する。

```python
"ひかりのかべ": MoveData(
    type="エスパー",
    category="変化",
    pp=30,
    target="own_side",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ひかりのかべ_set_side_field,
        ),
    }
),
```

## 注意点・エッジケース

- 現在 `data/move.py` の target は `"field"` になっているため `"own_side"` への変更が必要
- `side.activate` が False を返す（すでにひかりのかべが有効）場合は失敗として `HandlerReturn(value=False, stop_event=True)` を返す
- 特殊ダメージ半減の本体は side_field ハンドラ側で処理されており、技側はフィールド設定のみ
- リフレクター・しんぴのまもりと同一パターン（5ターン・自陣営）
- `turn.md` の ON_TURN_END に「ひかりのかべの継続・終了」が priority 130 として記載されている（フィールド管理は side_field 側の責任）
