# 実装計画書: サイコブースト（再レビュー）

## 仕様要約

`docs/spec/moves/サイコブースト.md` 参照。

* タイプ: エスパー / 分類: 特殊 / 威力: 140 / 命中率: 90 /
  PP: 8（Champions。SVまでの本家基準は5） / 対象: 相手1体
* 直接攻撃ではない。命中して技が成功した後、確定で自分の『とくこう』ランクを2段階下げる
  （反動によるランク低下であり、追加効果ではない）。

## 既存実装の確認結果と問題点

`src/jpoke/handlers/move_attack.py` の `サイコブースト_sharply_lower_spa_C` が
`modify_attacker_stats(battle, ctx, value, stats={"spa": -2})`（`chance` 省略でデフォルト100%）
を実行しており、同系統の反動技である `りゅうせいぐん_sharply_lower_spa_C` /
`オーバーヒート_lower_attacker_spa` と全く同じ実装パターンになっている。`Event.ON_HIT` での
登録・`h.MoveHandler` のデフォルト priority（100）もりゅうせいぐんと同一で問題ない。

一方、`src/jpoke/data/moves/move_sa.py` の `MoveData` エントリに以下の誤りがあった。

* `pp=5`（Gen9本家基準の値）になっていたが、同系統の反動技（りゅうせいぐん・
  オーバーヒート・リーフストーム）がいずれも本家PP5→ChampionsPP8に統一されている
  ことから、サイコブーストも `8` に修正する（`docs/spec/moves/サイコブースト.md`
  「基本情報」節に根拠を記載）。

`power=140`・`accuracy=90` はすでに一次情報（ポケモンWiki）・Champions値と一致しており
問題ない（サイコブーストはりゅうせいぐん・オーバーヒートと異なり第六世代以降も威力据え置き）。
`flags` を持たない点も一次情報（直接攻撃×、追加効果なし＝ちからずく非対象）と一致しており
問題ない。

## リーサル計算

`docs/progress/move.md` の該当行は「リーサル実装: -」（新規実装対象）。
`handlers/lethal.py` に `りゅうせいぐん_lower_spa` / `オーバーヒート_lower_attacker_spa` と
同一パターンで `サイコブースト_lower_spa` を新規実装する。

```python
def サイコブースト_lower_spa(battle: Battle, ctx: LethalContext, hp_dist: StateDist) -> StateDist:
    """サイコブースト: 命中後、攻撃側のとくこうを2段階下げる。"""
    ctx.attacker.rank["spa"] = clamp_stats(ctx.attacker.rank["spa"] - 2)
    return hp_dist
```

* イベント: `LethalEvent.ON_HIT`（りゅうせいぐん・オーバーヒートと同一）
* subject: `"attacker"`
* `src/jpoke/data/moves/move_sa.py` の `サイコブースト` エントリに
  `lethal_handlers={LethalEvent.ON_HIT: LethalHandler(l.サイコブースト_lower_spa)}` を追加する。
