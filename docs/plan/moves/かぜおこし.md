# 計画書: かぜおこし

## 仕様要約

- ひこうタイプ / 特殊 / 威力40 / 命中100 / PP35 / 対象: 1体選択（遠隔） / 優先度0 / 非接触
- 追加効果なし（通常攻撃のみ）
- 風技（`wind` フラグ）。`ふうりょくでんき` などの風技依存の効果の対象になる。
- `そらをとぶ` で滞空中の相手にも命中し、威力が2倍になる
  （`そらをとぶ` volatile 側の共通処理で実装済み。本技固有のハンドラは不要）。

## ハンドラ構成

追加効果を持たないため、専用ハンドラは不要。`handlers={}` のみで基本ダメージ計算に委ねる。
`そらをとぶ` 滞空中の命中判定・威力2倍処理は `handlers/volatile.py` の
`そらをとぶ_can_hit_hidden_target` / `そらをとぶ_boost_power` で技名 `"かぜおこし"` を
判定しており、既存実装済みのため本計画書では変更しない。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、priority の検討対象イベントは存在しない。

## 実装箇所

### data/move.py への更新

既存の `"かぜおこし"` エントリは以下の通りで、仕様と一致している（変更不要）。

```python
    "かぜおこし": MoveData(
        type="ひこう",
        category="special",
        pp=35,
        power=40,
        accuracy=100,
        flags={"wind"},
        handlers={},  # 追加効果なし
    ),
```

- `flags={"wind"}` により風技として扱われる → 仕様と一致。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `critical_rank` 省略時のデフォルトは `0`（通常の急所判定）→ 仕様と一致。
- 直接攻撃フラグ（`contact`）を持たない → 仕様（非接触）と一致。

## リーサル計算

追加効果を持たない通常攻撃技であり、HP依存の分岐やダメージ以外の状態変化がないため、
専用のリーサル計算ハンドラは不要（`docs/progress/move.md` の対象外区分: n/a）。

## 注意点・エッジケース

- 追加効果がないため、まもる・マジックコート・オウムがえし等の一般的な技判定は
  フレームワーク側の共通処理（`handlers/move_attack.py` の基本ダメージ計算経路）で
  自動的に処理され、本技固有のハンドラ実装は不要。
- おうじゃのしるし・するどいキバ（`flinch_on_hit_10pct`）は全ダメージ技に汎用的に
  適用される実装であり、`かぜおこし` は除外リスト（`_INNATE_FLINCH_MOVES`）に含まれない
  ため通常通り10%ひるみ判定の対象になる → 既存実装のままで仕様と一致。
- `そらをとぶ` 滞空中の相手への命中・威力2倍は `そらをとぶ` 側の共通処理に委ねる。
