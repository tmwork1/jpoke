# 実装計画書: おいかぜ

## 仕様要約

- タイプ: ひこう / カテゴリ: status / PP: 16（`docs/champions/move_list.txt` 準拠） / 対象: 味方の場（target="own_side"）
- flags: `wind`
- 4ターンの間、自分と味方の「すばやさ」が 2 倍になる
- 交代しても効果は続く
- 詳細は `docs/spec/moves/おいかぜ.md` / `docs/spec/fields/おいかぜ.md` を参照

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自陣営サイドフィールドに「おいかぜ」を設定（4ターン） |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「設置技: すでに最大」が priority 100 として記載されている。同種の `ステルスロック_set_field` が priority デフォルト（100）を使用しているため、同一パターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: `えんまく_modify_defender_stats` の後、`おにび_can_apply` の前（「お」ブロックの先頭付近）。

```python
def おいかぜ_set_side_field(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """おいかぜ: 自陣営に「おいかぜ」を4ターン設定する。"""
    side = battle.get_side(ctx.attacker)
    if not side.activate("おいかぜ", 4):
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/moves/move_a.py`

`"おいかぜ"` エントリは以下の内容で登録済み。

```python
"おいかぜ": MoveData(
    type="ひこう",
    category="status",
    pp=16,
    target="own_side",
    flags={"wind"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.おいかぜ_set_side_field,
        ),
    }
),
```

## 注意点・エッジケース

- PPは `docs/champions/move_list.txt` の値（16）を採用する。Gen9本家の15とは異なる点に注意
- `side.activate` の戻り値が False の場合（すでにおいかぜが有効）は `HandlerReturn(value=False, stop_event=True)` を返す
- `ステルスロック_set_field` は `ctx.defender` の陣営を使うが、おいかぜは自陣営（`ctx.attacker`）を使う点が異なる
- おいかぜの効果（すばやさ 2 倍）の本体は side_field ハンドラ側で処理されており、技側はフィールド設定のみ
