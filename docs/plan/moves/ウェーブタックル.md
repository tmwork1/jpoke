# 実装計画書: ウェーブタックル

参照: docs/spec/moves/ウェーブタックル.md

## 概要

物理技。相手に与えたダメージの33%（1/3）を自分も受ける反動技。追加効果なし。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---------|----------|--------------|------|
| ON_HIT | 100（`MoveHandler` デフォルト） | attacker:self | 与ダメの1/3を反動として攻撃者に与える |

Priority 根拠: `docs/spec/turn.md` の `Event.ON_HIT` に反動ダメージ専用の記載行はないが、
同系統の反動技（すてみタックル・とっしん・ワイルドボルト・じごくぐるま・アフロブレイク）と同じく
`MoveHandler` のデフォルト priority=100 を使用する（既存実装との整合性を優先）。

## handlers/move_attack.py

共通ヘルパー `_recoil(battle, ctx, value, ratio)` を利用する（既存実装）。

```python
def ウェーブタックル_recoil(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return _recoil(battle, ctx, value, 1/3)
```

## data/move.py

```python
    "ウェーブタックル": MoveData(
        type="みず",
        category="physical",
        pp=12,
        power=120,
        accuracy=100,
        flags={"contact", "recoil"},
        handlers={
            Event.ON_HIT: h.MoveHandler(
                ウェーブタックル_recoil,
            )
        }
    ),
```

## リーサル計算

対象外（`docs/progress/move.md` の「リーサル実装」列は n/a）。
反動技は攻撃者自身のHPを減らすのみで、防御側の HP 分布 (`StateDist`) には影響しないため
リーサル計算ハンドラの実装対象にはならない。

## 注意点・エッジケース

* `reason="recoil"` を指定することで、いしあたま特性により無効化される
* みがわりに命中した場合も `ON_HIT` は発動するため反動ダメージは発生する。
  `_recoil` は `value`（実際の被ダメージ、みがわり有効時は0）が0の場合
  `ctx.substitute_damage`（みがわりへの与ダメージ）にフォールバックして反動量を算出する
* PP は本家SVの基準値10ではなく、チャンピオンズ準拠の12を使用する
  （`src/jpoke/data/ps-champ-ja/moves.json` を正とする）
