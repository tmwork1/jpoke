# 計画書: しょうりのまい

## 仕様要約

- 自分の「こうげき」「ぼうぎょ」「すばやさ」ランクを1段階ずつ上げる
- タイプ: かくとう、変化、PP: 10（champions move_list.txt に記載なしのため第9世代基準を採用）、対象: 自分
- 踊り技（`dance` ラベル）

出典: docs/spec/moves/しょうりのまい.md

---

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100 (省略) | `attacker:self` (省略) | こうげき・ぼうぎょ・すばやさを1段階ずつ上げる |

---

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`つるぎのまい_modify_attacker_stats`, `からをやぶる_modify_attacker_stats` 等）が
すべて priority=100（デフォルト）で登録されているため、100 を使用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: 五十音順（「し」の区画）

```python
def しょうりのまい_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"atk": 1, "def": 1, "spe": 1})
```

### `src/jpoke/data/moves/move_sa.py`

`"しょうりのまい"` エントリの `handlers={}` を以下に更新する（`target="self"` を追加）:

```python
    "しょうりのまい": MoveData(
        type="かくとう",
        category="status",
        pp=10,
        target="self",
        flags={"dance"},
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.しょうりのまい_modify_attacker_stats,
            ),
        },
    ),
```

---

## 注意点・エッジケース

- `target="self"` が既存エントリに欠落していたため追加する（省略時のデフォルト対象と
  実際の技範囲が一致しないため必須）
- ランクがすでに最大（+6）の場合の失敗は `battle.modify_stats()` 内部処理でマネージャーが
  扱うため、ハンドラ側にガード不要
- 3ステータス同時上昇のため `modify_attacker_stats` の `stats` 引数に3キーをまとめて渡す
  （`からをやぶる` と同様のパターン）
- 踊り技として「おどりこ」特性への連鎖は既存フレームワーク内で処理される
