# 計画書: てんしのキッス

## 仕様要約

- フェアリータイプ / 変化 / 命中75 / PP12（Pokémon Champions基準） / 対象: 1体選択
- 相手を2〜5ターンの間『こんらん』状態にする
- マイペース特性には無効（`ON_TRY_MOVE_2` priority 140 でフレームワーク側が処理）
- こんらん済みの場合は失敗（`ON_TRY_MOVE_2` priority 130 でフレームワーク側が処理）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | 相手にこんらん付与 |

## Priority 根拠

`docs/spec/turn.md` の `ON_STATUS_HIT` に直接記載はないが、あやしいひかり・ちょうおんぱ等の同種こんらん付与技と同じ `ON_STATUS_HIT` priority 100 を採用する。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `てっぺき` の後、`でんじは` の前（五十音順）。

```python
def てんしのキッス_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """てんしのキッスの効果: 相手をこんらん状態にする。"""
    return HandlerReturn(value=battle.volatile_manager.apply_confusion(
        ctx.defender, source=ctx.attacker
    ))
```

こんらん付与時のターン数（2〜5）は `VolatileManager.apply_confusion` 内でランダム決定されるため、
ハンドラ側で乱数を扱わない。

### data/moves/move_ta.py への更新

`"てんしのキッス"` エントリのハンドラを追加する。

```python
    "てんしのキッス": MoveData(
        type="フェアリー",
        category="status",
        pp=12,
        accuracy=75,
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.てんしのキッス_apply,
            ),
        }
    ),
```

## 注意点・エッジケース

- あやしいひかり（`あやしいひかり_apply`）と全く同じ実装パターン
- こんらん済み判定はフレームワーク側（`ON_TRY_MOVE_2` priority 130）が処理するため、ハンドラ内にガード不要
- マイペース特性による無効もフレームワーク側（`ON_TRY_MOVE_2` priority 140）が処理
- count_range は 2〜5（`VolatileManager.apply_confusion` が内部で `battle.random.randint(2, 5)` を実行）
- PP は Pokémon Champions で 8/12/16/20 の4段階に統一されているため、原作の10ではなく12を採用する
  （`docs/champions/move_list.txt`、`docs/champions/changes_from_sv.md`）
