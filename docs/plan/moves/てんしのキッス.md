# 計画書: てんしのキッス

## 仕様要約

- フェアリータイプ / 変化 / 命中75 / PP10 / 対象: 1体選択
- 相手を1〜4ターンの間『こんらん』状態にする
- マイペース特性には無効（`ON_TRY_MOVE_2` priority 140 でフレームワーク側が処理）
- こんらん済みの場合は失敗（`ON_TRY_MOVE_2` priority 130 でフレームワーク側が処理）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | 相手にこんらん付与 |

## Priority 根拠

`docs/spec/turn.md` の `ON_STATUS_HIT` に直接記載はないが、あやしいひかり・ちょうおんぱ等の同種こんらん付与技と同じ `ON_STATUS_HIT` priority 100 を採用する。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `てっぺき` の後、`でんじは` の前（五十音順）。

```python
def てんしのキッス_apply_volatile_to_defender(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """てんしのキッスの効果: 相手をこんらん状態にする（2〜5ターン）。"""
    count = battle.random.randint(2, 5)
    return HandlerReturn(value=battle.volatile_manager.apply(
        ctx.defender, "こんらん", count=count, source=ctx.attacker, ctx=ctx
    ))
```

### data/move.py への更新

`"てんしのキッス"` エントリのハンドラを追加する。

```python
    "てんしのキッス": MoveData(

        category="変化",
        pp=10,
        accuracy=75,
        handlers={

```

## 注意点・エッジケース

- あやしいひかり（`あやしいひかり_apply_volatile_to_defender`）と全く同じ実装パターン
- こんらん済み判定はフレームワーク側（`ON_TRY_MOVE_2` priority 130）が処理するため、ハンドラ内にガード不要
- マイペース特性による無効もフレームワーク側（`ON_TRY_MOVE_2` priority 140）が処理
- count_range は 2〜5（あやしいひかりと同一。仕様表記の「1〜4ターン」はターン終了時消費分のため実際の乱数範囲は2〜5）
