# 実装計画: メテオドライブ

## 仕様サマリ

- タイプ: はがね / 分類: 物理 / 威力: 100 / 命中率: 100 / PP: 5
  （champions/move_list.txt に記載なしのため、Gen9 本家準拠の PP をそのまま採用）
- 効果: 相手の特性を無視して攻撃する（追加効果なし）
- まもる: ○ / おうじゃのしるし: ○ / よこどり: × / オウムがえし: ○ / 直接攻撃: ○

## 対象イベント

`ON_BEGIN_MOVE` / `ON_END_MOVE` — 特性「かたやぶり」（`src/jpoke/handlers/ability.py` の
`かたやぶり_disable_foe_ability` / `かたやぶり_restore_foe_ability`）と全く同じパターン。
攻撃開始直前に相手の特性を無効化し、攻撃終了後に復元する。シャドーレイ・フォトンゲイザーと
同一実装パターン。

## Priority

`docs/spec/turn.md` に `ON_BEGIN_MOVE` / `ON_END_MOVE` の記載なし。
既存の「かたやぶり」特性ハンドラ・シャドーレイ・フォトンゲイザーの `MoveHandler` が
いずれも priority 省略（デフォルト）で実装されているため、同様に priority を省略する。

## 実装手順（実装済み・現状確認）

### handlers/move_attack.py

```python
def メテオドライブ_disable_defender_ability(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """メテオドライブ: 攻撃直前に相手の特性を無効化する（かたやぶりと同様の対象特性）。"""
    mon = ctx.defender
    if mon.ability.has_flag("mold_breaker_ignorable"):
        battle.add_ability_disabled_reason(mon, "メテオドライブ")
    return HandlerReturn(value=value)


def メテオドライブ_restore_defender_ability(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """メテオドライブ: 攻撃終了後に相手の特性の無効化を解除する。"""
    battle.remove_ability_disabled_reason(ctx.defender, "メテオドライブ")
    return HandlerReturn(value=value)
```

`subject_spec` は省略（デフォルトの `attacker:self`）。`MoveHandler` はコンテキストの攻守を
直接参照する実装のため `skip_subject_check=True` であり、シャドーレイ・フォトンゲイザーと
同様に `subject_spec` を明示しなくても `ctx.defender` を直接参照して問題ない。

### data/moves/move_ma.py

```python
"メテオドライブ": MoveData(
    type="はがね",
    category="physical",
    pp=5,
    power=100,
    accuracy=100,
    flags={"contact", "ignore_ability"},
    handlers={
        Event.ON_BEGIN_MOVE: h.MoveHandler(
            ha.メテオドライブ_disable_defender_ability,
        ),
        Event.ON_END_MOVE: h.MoveHandler(
            ha.メテオドライブ_restore_defender_ability,
        ),
    },
),
```

`flags={"contact", "ignore_ability"}` の `ignore_ability` は分類用のフラグで、実際の
無効化ロジックには使用しない（`types/literals.py` のコメント参照）。シャドーレイ・
フォトンゲイザーと同じ扱い。`contact` はメテオドライブが直接攻撃技であるため付与する
（シャドーレイ・フォトンゲイザーは非接触技のため `contact` を持たない点が異なる）。

## テスト方針

- もふもふ（`mold_breaker_ignorable` 特性）を持つ相手への接触技ダメージ半減が
  無視されることを確認（`damage_modifier == 4096`）
- 攻撃終了後に相手の特性の無効化が解除される（`defender.ability.enabled is True`）ことを確認
- 上記2点はシャドーレイ・フォトンゲイザーと全く同じテストパターンで踏襲する

## 注意事項

- 無視できる特性は「かたやぶり」と同じ対象（`mold_breaker_ignorable` フラグを持つ特性）。
  ファントムガード／プリズムアーマーはこのフラグを持たないため無視されない（対応不要）。
- 第七世代限定の「使用者自身の特性も無視する」仕様は第八世代以降で撤廃されており、
  本プロジェクトは SV（チャンピオンズ）基準のため実装しない（defender のみを対象とする）。
- 技呼び出し系の技（ゆびをふる等）経由で発動した場合は特性無視効果が発生しない仕様が
  あるが、該当する技呼び出し機構自体が本プロジェクトで未実装のため対象外。
- かがくへんかガス/かたやぶり無効化チェックは書かない（既存の `has_flag` 判定のみで十分）。
