# みずあめボム 実装計画書

参照: docs/spec/moves/みずあめボム.md

## 現状

* `src/jpoke/data/moves/move_ma.py` の `"みずあめボム"` エントリは実装済みだが、以下の誤りがある。
  - `accuracy=85`（SV値）になっているが、本プロジェクトの対象範囲定義によりChampions値の
    `90` を採用すべき。
  - `flags={"secondary_effect"}` のみで `"bullet"` フラグが欠落している。仕様書の通り
    弾の技であり、特性ぼうだんの無効化対象にする必要がある。
* `Event.ON_DAMAGE_HIT`: `ha.みずあめボム_apply_volatile_to_defender`
  （`apply_volatile_to_defender` ヘルパーで「あめまみれ」volatile を count=3 で付与、既存実装のまま）
  は仕様と一致しており修正不要。

## 修正内容

`src/jpoke/data/moves/move_ma.py` の `"みずあめボム"` エントリを以下のように修正する。

```python
"みずあめボム": MoveData(
    type="くさ",
    category="special",
    pp=10,
    power=60,
    accuracy=90,
    flags={"bullet", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.みずあめボム_apply_volatile_to_defender,
        )
    }
),
```

`handlers/move_attack.py` の `みずあめボム_apply_volatile_to_defender` は既存実装のまま変更不要
（`apply_volatile_to_defender(battle, ctx, value, volatile="あめまみれ", count=3)` を呼ぶのみ）。

りんぷん・おんみつマント・ちからずくによる無効化は `apply_volatile_to_defender` 内部の
`battle.resolve_secondary_chance` が `flags` の `"secondary_effect"` を参照して汎用的に処理する
ため、みずあめボム固有の実装は不要。

## priority の根拠

* `Event.ON_DAMAGE_HIT` は `docs/spec/turn.md` に未掲載のイベント（ダメージ計算内部の追加効果
  フェーズ）。既存の同種ハンドラ（`ミストボール_lower_spa_C` 等、追加効果でランク変化・状態異常・
  volatile 付与を行うハンドラ）はすべて `MoveHandler` のデフォルト priority=100 を使用しており、
  本ハンドラも同様にデフォルト値のまま登録する（既存実装通り、変更不要）。
