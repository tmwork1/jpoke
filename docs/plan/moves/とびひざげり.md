# とびひざげり 実装計画書

## 仕様要約（docs/spec/moves/とびひざげり.md 参照）

* タイプ: かくとう / 分類: 物理 / 威力: 130 / 命中: 90 / PP: 10
* 対象: 相手1体 / 優先度: 0 / 接触あり
* じゅうりょく状態では使用不可（`gravity_restricted`）
* 効果: 技が外れる・失敗すると自分の最大HPの1/2ダメージを受ける（切り捨て、最低1）
  * マジックガードで防げるが、いしあたまでは防げない（`reason="fixed_recoil"`）
  * すてみ所持時は威力1.2倍（`recoil` フラグの対象）

## 現状の実装状況

`src/jpoke/data/move.py` に登録済み。

```python
"とびひざげり": MoveData(
    type="かくとう",
    category="physical",
    pp=10,
    power=130,
    accuracy=90,
    flags={"contact", "gravity_restricted"},
    handlers={
        Event.ON_TRY_MOVE_1: h.MoveHandler(
            h.gravity_restricted_fail,
            subject_spec="attacker:self",
            priority=30,
        ),
        Event.ON_MISS: h.MoveHandler(
            ha.とびひざげり_crash,
        ),
    }
),
```

```python
def とびひざげり_crash(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """とびひざげりが外れた場合の失敗反動ダメージ。自分の最大HPの1/2を受ける。"""
    battle.modify_hp(ctx.attacker, v=-max(1, ctx.attacker.max_hp // 2), reason="self_cost", source=ctx.attacker)
    return HandlerReturn(value=value)
```

威力・命中率・PP・タイプ・分類・`gravity_restricted` フラグ・ON_MISSの発火条件は、いずれも仕様書と一致しており
問題なし。

### レビューで判明した修正点

1. **`recoil` フラグの欠落**: すてみ特性は `ctx.move.has_flag("recoil")` を判定条件にしており
   （`handlers/ability.py` の `すてみ_boost_power`）、このフラグが無いと威力1.2倍が発動しない。
   wiki 記載（「特性がすてみのポケモンが使用した場合、威力が1.2倍になる」）と一致させるため、
   `flags` に `"recoil"` を追加した。
   なお `recoil` フラグ（すてみ判定用）は `HPChangeReason`（`ctx.hp_change_reason`、いしあたま・
   マジックガード判定用）とは別の型・別の判定軸であり、フラグを追加しても reason の判定には影響しない。
2. **失敗反動の `reason` の誤り（バグ）**: `とびひざげり_crash` は `reason="self_cost"` でHPを減少させていたが、
   `"self_cost"` は `マジックガード_ignore_damage`（`handlers/ability.py`）の除外リストに含まれており、
   マジックガードで**防げない**扱いになっていた。
   しかし wiki には「特性がマジックガードのポケモンが使用したときは失敗しても反動ダメージを受けない」と
   明記されており、本来はマジックガードで防げる必要がある。
   一方で「特性がいしあたまのポケモンが使用したとしても失敗時の反動ダメージを受ける」という記載もあり、
   いしあたまでは防げてはいけない。
   この「マジックガードで防げるが、いしあたまでは防げない」という組み合わせは、本プロジェクトの
   `HPChangeReason` に既に `"fixed_recoil"`（`てっていこうせん` 等が使用）として定義されているため、
   `reason="self_cost"` → `reason="fixed_recoil"` に修正した。
   （`docs/plan/moves/失敗反動技.md` 注記2、`docs/spec/moves/とびひざげり.md` にも記載）

```python
def とびひざげり_crash(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """とびひざげりが外れた場合の失敗反動ダメージ。自分の最大HPの1/2を受ける。

    マジックガードで防げるが、いしあたまでは防げない確定反動として扱う

    """

    return HandlerReturn(value=value)
```

## 注意点・エッジケース（review-test 向け）

1. **命中時に反動が発生しないこと**: `ON_MISS` は命中判定失敗時のみ発火するため、命中時はダメージを受けない。
2. **外れたとき失敗反動ダメージを受けること**: `max(1, max_hp // 2)` 減少する。
3. **いしあたまとの相互作用**: 失敗反動は `reason="fixed_recoil"` のため、いしあたまでは防げないことを確認する。
4. **マジックガードとの相互作用**: `reason="fixed_recoil"` は `マジックガード_ignore_damage` の除外リストに
   含まれないため、マジックガード所持時は反動ダメージを受けないことを確認する（本レビューでの新規修正点）。
5. **すてみとの相互作用**: `recoil` フラグ追加により、すてみ特性所持時に威力補正 `apply_fixed_modifier(value, 4915)`
   (1.2倍) が発生することを確認する（本レビューでの新規修正点）。
6. **flags に "recoil" が含まれること**: 既存テスト `test_とびひざげり_labelsにrecoilが含まれない` は
   誤り（かかとおとしレビュー時の知見を反映しておらず、すてみ対応前の古い前提のテスト）であるため、
   `"recoil" in move_data.flags` を確認するテストに修正する。
7. **じゅうりょくとの相互作用**: 既存実装（`gravity_restricted_fail`）を変更しないため、既存のとびげり等の
   テストパターンに準拠していれば問題ない（本レビューでは新規テスト追加は必須ではない）。
8. **既知の実装制約**: まもる・タイプ無効化による失敗時の反動は現状未実装（`docs/spec/moves/とびひざげり.md`
   「実装上の制約」参照）。本レビューの対象範囲外として現状維持する。
9. **ダメおしとの相互作用**: `fixed_recoil` は `hits_taken` カウンタを増加させないため、とびひざげりの
   反動を受けた相手に対するダメおしの威力2倍化は未実装（`docs/spec/moves/とびひざげり.md` 参照）。
   本レビューの対象範囲外として現状維持する。
