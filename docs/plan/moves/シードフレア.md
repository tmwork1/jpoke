# シードフレア 実装計画書

## 仕様要約（docs/spec/moves/シードフレア.md 参照）

* タイプ: くさ / 分類: 特殊 / 威力: 120 / 命中: 85 / PP: 5
  （move_list.txt に該当エントリなし＝シェイミ専用技かつ幻のポケモンは使用不可のため据え置き）
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果: 追加効果として、40%の確率で相手の『とくぼう』ランクを2段階下げる。

## 現状の実装状況

`src/jpoke/data/moves/move_sa.py` に既に登録済み。

```python
"シードフレア": MoveData(
    type="くさ",
    category="special",
    pp=5,
    power=120,
    accuracy=85,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.シードフレア_sharply_lower_defender_spd,
        )
    }
),
```

内容は仕様書と一致しており、修正不要（PP・威力・命中・追加効果確率・下降幅すべて一致）。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 40%の確率で相手の『とくぼう』ランクを2段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def シードフレア_sharply_lower_defender_spd(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spd": -2}, chance=0.4)
```

`modify_defender_stats`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は `flags={"secondary_effect"}` さえ
付与されていれば汎用的に処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（能力ランク低下）は既存の同種ハンドラ（エナジーボール・だいちのちから 等）と
同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。`ON_DAMAGE_HIT` は
`docs/spec/turn.md` に未掲載のイベント（ダメージ計算内部）だが、既存の同種ハンドラの実装パターンを
踏襲するため個別の priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` が付与されているため、ちからずく所持時は威力1.3倍（120→156）になり、
     とくぼう低下の追加効果は発動しなくなる。
2. **PPの扱い**
   - move_list.txt に本技のエントリが存在しないため、8/12/16/20への変換対象外。Gen9本家基準の5を
     正としてそのまま維持する（Vジェネレート等の他の幻専用技と同じ扱い）。
3. **テスト方針**
   - `secondary_chance` を0/1に固定し、とくぼうランクの変化有無を確認するテストを追加する
     （`test_シードフレア_相手にダメージを与える` /
     `test_シードフレアのとくぼう2段階低下が発動する` /
     `test_シードフレアのとくぼう2段階低下が発動しない`）。
