# 計画書: なみのり

## 仕様要約

- みずタイプ / 特殊 / 威力90 / 命中100 / PP16（Champions仕様） / 対象: 自分以外の全員
  （シングルでは相手1体） / 優先度0 / 非接触
- 追加効果なし（通常攻撃のみ）
- 『ダイビング』状態の相手には命中し、ダメージが2倍になる

## ハンドラ構成

- なみのり自体には追加効果がないため、`data/move.py` の `なみのり` エントリは `handlers={}` のみで
  基本ダメージ計算に委ねる。
- 『ダイビング』状態の相手への威力2倍は、`handlers/volatile.py` の `ダイビング_boost_power`
  （`Event.ON_CALC_POWER_MODIFIER`、`subject_spec="defender:self"`、対象は `ダイビング` volatile
  自体に登録済み）で処理される。`HIDDEN_MOVE_ALLOWED_MOVES["ダイビング"]` に `なみのり` が含まれる
  ことで命中判定も回避されずに命中する（`ダイビング_can_hit_hidden_target`）。
- 上記は既存実装済み（`ダイビング` の計画書・仕様書を参照）のため、なみのり側で新規に実装する
  ハンドラはない。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、priority の検討対象イベントは存在しない。
（`ダイビング_boost_power` は既存実装であり、`ダイビング` の計画書で priority 根拠が示されている）。

## 実装箇所

### data/move.py への更新

既存の `"なみのり"` エントリは以下の通りで、仕様と一致している（変更不要）。

```python
    "なみのり": MoveData(
        type="みず",
        category="special",
        pp=16,
        power=90,
        accuracy=100,
        handlers={},  # 追加効果なし
    ),
```

- `flags` を指定していないため `contact` フラグは付与されない → 非接触（仕様と一致）。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `critical_rank` 省略時のデフォルトは `0`（通常の急所判定）→ 仕様と一致。

## リーサル計算

- `docs/progress/move.md` のリーサル実装・リーサルテスト列は `n/a`。追加効果を持たない単純な
  攻撃技であり、HP依存の追加効果（毎ターン回復・被弾時発動など）が存在しないため、
  リーサル計算での固有ハンドラは不要（対象外）。

## 注意点・エッジケース

- 『ダイビング』状態の相手への威力2倍・命中確定は `ダイビング` 状態側のハンドラで処理されるため、
  なみのり固有のハンドラを重複実装しない。
- 追加効果がないため、まもる・おうじゃのしるし・オウムがえし等の一般的な技判定は
  フレームワーク側の共通処理（`handlers/move_attack.py` の基本ダメージ計算経路）で
  自動的に処理され、本技固有のハンドラ実装は不要。
