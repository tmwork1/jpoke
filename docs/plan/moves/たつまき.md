# 計画書: たつまき

## 仕様要約（docs/spec/moves/たつまき.md 参照）

- ドラゴンタイプ / 特殊 / 威力40 / 命中100 / PP20 /
  対象: 1体選択（遠隔） / 優先度0 / 非接触
- 追加効果として、命中時に20%の確率で相手をひるませる。
- 追加効果に分類されるため、ちからずく（威力1.3倍・効果無効化）の対象技になる。
- 風技（`wind` フラグ）。`ふうりょくでんき` などの風技依存の効果の対象になる。
- `そらをとぶ` で滞空中の相手にも命中し、威力が2倍になる
  （`そらをとぶ` volatile 側の共通処理で実装済み。本技固有のハンドラは不要）。

## 既存実装との差分

既存実装（`data/move.py` の `"たつまき"` エントリ）は仕様と一致しており、修正は不要。

1. **基本情報は仕様通り**
   `type="ドラゴン", category="special", pp=20, power=40, accuracy=100` で一致。

2. **追加効果ハンドラ本体（`たつまき_apply_flinch`）は仕様通り実装済み**
   `apply_volatile_to_defender(battle, ctx, value, volatile="ひるみ", chance=0.2)`
   で、20%発動の仕様と一致している。変更不要。

3. **`flags` は既に `{"wind", "secondary_effect"}` が設定済み**
   風技かつ追加効果を持つ技として正しく分類されている。変更不要。

4. **`そらをとぶ` 滞空中の命中・威力2倍は共通処理に委ねる**
   `handlers/volatile.py` の `HIDDEN_MOVE_ALLOWED_MOVES["そらをとぶ"]` に `"たつまき"` が
   含まれ、`そらをとぶ_boost_power` でも技名 `"たつまき"` を判定済み。変更不要。

## ハンドラ構成

### handlers/move_attack.py: `たつまき_apply_flinch`（既存・変更なし）

```python
def たつまき_apply_flinch(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_defender(battle, ctx, value, volatile="ひるみ", chance=0.2)
```

- `apply_volatile_to_defender`（`handlers/move.py`）は内部で `battle.resolve_secondary_chance` を
  呼び出すため、`Event.ON_MODIFY_SECONDARY_CHANCE` 経由でちからずく（確率0化・威力1.3倍）の
  ハンドラが自動的に適用される（`secondary_effect` フラグが前提）。
- イベント: `Event.ON_DAMAGE_HIT`（ダメージが実際に適用された後に発動。
  docs/spec/turn.md の `## Event.ON_DAMAGE` 参照）。
  ダメージが発生しなかった場合（無効化・まもる等）はイベント自体が発火しないため、
  追加効果も発生しない仕様を自然に満たす。
- 優先度: デフォルト100（`MoveHandler` のデフォルト値）。
- subject_spec: `MoveHandler` デフォルト（`attacker:self`）のまま。

### data/move.py（既存・変更なし）

```python
"たつまき": MoveData(
    type="ドラゴン",
    category="special",
    pp=20,
    power=40,
    accuracy=100,
    flags={"wind", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.たつまき_apply_flinch,
        )
    }
),
```

対象外（`docs/progress/move.md` の「リーサル実装」列が `n/a`。ひるみ付与のみでHP分布に影響しない

## 注意点・エッジケース

- ドラゴンタイプの技が無効化されるタイプの相手（フェアリータイプ）など、技自体が無効化された場合、
  `Event.ON_DAMAGE_HIT` 自体が発火しないため追加効果も発生しない。個別のガードは不要。
- まもるで防がれた場合も同様にダメージが発生せず `Event.ON_DAMAGE_HIT` が発火しないため、
  追加効果も発生しない。
- ひるみは `Event.ON_CHECK_ACTION` で消費される（`docs/spec/turn.md` 参照）。
  ON_DAMAGE_HIT での付与後、相手が次のターンに行動しようとする際に判定される。
  すでに行動済みのポケモンにひるみが付与されてもそのターンは効果がない。
- おうじゃのしるし・するどいキバ（`flinch_on_hit_10pct`）は元々ひるみの追加効果を持つ技には
  重複しない仕様（第五世代以降）。`handlers/item.py` の `_INNATE_FLINCH_MOVES` に `"たつまき"` が
  含まれており、既存実装済みで変更不要。
- `そらをとぶ` 滞空中の相手への命中・威力2倍は `そらをとぶ` 側の共通処理に委ねる。
