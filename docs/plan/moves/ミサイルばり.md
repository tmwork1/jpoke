# ミサイルばり 実装計画書

## 仕様要約（docs/spec/moves/ミサイルばり.md 参照）

* タイプ: むし / 分類: 物理 / 威力: 25（1ヒットあたり） / 命中: 95 / PP: 20
* 対象: 相手1体 / 優先度: 0 / 接触なし（直接攻撃ではない）
* 効果:
  * 2～5回連続で攻撃する可変回数の連続攻撃技（命中判定は1発目のみ、各ヒット威力は固定25）。
  * 急所判定は各ヒットごとに独立して行われる。
  * 追加効果なし。

## 現状の実装状況

`src/jpoke/data/moves/move_ma.py` に既に登録済み。

```python
"ミサイルばり": MoveData(
    type="むし",
    category="physical",
    pp=20,
    power=25,
    accuracy=95,
    multi_hit={
        "min": 2,
        "max": 5,
        "check_hit_each_time": False,
        "power_sequence": (),
    },
    handlers={},  # 追加効果なし
),
```

内容は仕様書と一致しており、修正不要。`flags` に `contact` を含めていない点も、
接触判定なし（直接攻撃ではない）という仕様と一致する。

* `multi_hit.min=2, max=5` かつ `check_hit_each_time` を指定していないため、
  `core/move_executor.py` の `_resolve_hit_count` が `MULTI_HIT_DISTRIBUTION_2_TO_5`
  （2回37.5%、3回37.5%、4回12.5%、5回12.5%）に従ってヒット回数を決定し、
  命中判定は1発目のみ行われる（`みだれひっかき` `スイープビンタ` と同一パターン）。
* `power_sequence` が空タプルのため、`_resolve_hit_power` は各ヒットとも `move.power`（25）を返す。
* 急所判定（`_check_critical`）はヒットごとに独立して呼び出されるため、既存の汎用機構のみで
  「各ヒットごとに独立して急所判定」という仕様を満たす。

## ハンドラ構成

固有ハンドラなし（`handlers={}`）。可変回数連続ヒットの挙動は `core/move_executor.py` 側の
汎用機構（`multi_hit`）で処理されるため、`handlers/move_attack.py` への追加実装は不要。

## 注意点・エッジケース（review-test 向け）

1. **ヒット回数は2～5回の乱数分布**
   - `docs/spec/moves/_multi_hit.md` の「2.2 可変回数技（2～5回）」の通り、
     `_resolve_hit_count` が乱数でヒット回数を決定する。テストでは戻り値の範囲（2〜5）を確認する。
2. **命中判定は1発目のみ**
   - `check_hit_each_time` フラグを持たないため、1発目が外れた場合は残りのヒットも発生しない。
3. **急所判定は各ヒットごと**
   - 急所ランク補正なし（`critical_rank=0`、デフォルト値）。既存の急所機構をそのまま利用する。
4. **接触判定なし**
   - `flags` に `contact` を含めないため、ゴツゴツメット等の接触判定系効果は発動しない。
5. **テスト方針**
   - `_resolve_hit_count` の戻り値が2〜5の範囲に収まることを確認するテスト
     （`みだれひっかき` `スイープビンタ` の複数ヒットテストと同一パターン）。
