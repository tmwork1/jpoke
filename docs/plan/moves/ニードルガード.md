# 計画書: ニードルガード

## 仕様要約

- タイプ: くさ / カテゴリ: 変化 / PP: 10 / 命中: なし（必中） / 対象: 自分 / 優先度: +4
- 使用ターン中、相手の攻撃を防ぐ
- 接触技（直接攻撃）で攻撃してきた相手に、相手の最大HPの1/8ダメージを与える
- まもる系共通の連続使用失敗ルールが適用される（`ON_TRY_MOVE_1` priority 30「まもる系: 連続使用により失敗」）
- `"ニードルガード"` volatile を自分に付与し、`ON_TRY_MOVE_1` で攻撃を無効化しつつ接触カウンターダメージを与える
- ターン終了時に volatile を解除する

| イベント | priority | subject_spec | 役割 | 実装場所 |
|----------|----------|--------------|------|----------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self` | 自分に `"ニードルガード"` volatile を付与する | `handlers/move_status.py` |
| `ON_TRY_MOVE_1` | 100 | `defender:self` | 攻撃を無効化し、接触技で攻撃した相手に最大HPの1/8ダメージを与える | `handlers/volatile.py` |

## Priority 根拠

### `ON_STATUS_HIT`: priority 100
`docs/spec/turn.md` に `ON_STATUS_HIT` の内部 priority 表がない。既存の変化技ハンドラ（`まもる_apply_volatile_to_attacker`・`トーチカ_apply_volatile_to_attacker`・`キングシールド_apply_volatile_to_attacker` 等）がすべて priority 100（デフォルト）を使用しているため、同一パターンで 100 を採用する。

### `ON_TRY_MOVE_1`: priority 100
`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` セクションに明記：

```
| 100 | まもる・キングシールド等による無効化 |
```

既存の `まもる` / `キングシールド` / `トーチカ` volatile ハンドラがすべて priority 100 を明示しているため、同一の 100 を採用する。

### `ON_TURN_END`: priority 100（デフォルト）
`docs/spec/turn.md` に `ON_TURN_END` の「まもる系 volatile 解除」の記載はない。既存の `まもる_remove_volatile`・`かえんのまもり_remove_volatile`・`キングシールド_remove_volatile`・`トーチカ_remove_volatile` 等がすべてデフォルト priority を使用しているため、同一パターンで 100 を採用する。

なお、`docs/spec/turn.md` の `ON_HIT` priority 100 に「ニードルガード等の効果」と記載されているが、これはダメージ発生後ではなく「まもる成功時の追加効果」を指す記述と解釈する。実際の接触カウンターダメージは `_run_protect` 内（`ON_TRY_MOVE_1`）で処理するのが既存パターン（キングシールドの攻撃ランク低下、トーチカのどく付与が同一場所）と一致するため、`ON_TRY_MOVE_1` での処理を採用する。

### 2. `src/jpoke/handlers/volatile.py` — `_run_protect` の拡張

`_run_protect` に `chip_on_contact: float | None = None` パラメータを追加し、接触時に攻撃者へ固定割合ダメージを与える処理を追加する。

```python
def _run_protect(battle: Battle,
                 ctx: EventContext,
                 value: Any,
                 stats_change_on_contact: dict[Stat, int] | None = None,
                 ailment_on_contact: AilmentName | None = None,
                 chip_on_contact: float | None = None,
                 protect_non_attack: bool = True) -> HandlerReturn:
    """protect系の共通骨格。

    Args:
        battle: バトルインスタンス
        ctx: コンテキスト
        value: イベント値
        stats_change_on_contact: 接触時に攻撃者に与えるランク変化（例: {"A": -1}）
        ailment_on_contact: 接触時に攻撃者に付与する状態異常名
        chip_on_contact: 接触時に攻撃者の最大HPから削る割合（例: 1/8）
        protect_non_attack: False の場合、変化技を保護しない
    """
    if not _check_protect_success(battle, ctx, protect_non_attack):
        battle.add_event_log(ctx.defender, LogCode.PROTECT_FAILED)
        return HandlerReturn(value=value)

    battle.add_event_log(ctx.defender, LogCode.PROTECT_SUCCEEDED)

    if battle.query.is_contact(ctx):
        if stats_change_on_contact:
            battle.modify_stats(ctx.attacker, stats_change_on_contact, source=ctx.defender)
        if ailment_on_contact:
            battle.ailment_manager.apply(ctx.attacker, ailment_on_contact, source=ctx.defender)
        if chip_on_contact is not None:
            battle.modify_hp(ctx.attacker, r=-chip_on_contact, reason="")

    return HandlerReturn(value=False, stop_event=True)
```

## 注意点・エッジケース

### まもる系の連続使用失敗

`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` priority 30 に「まもる系: 連続使用により失敗」と記載されている。この判定は `_run_protect` 内の `battle.events.emit(Event.ON_CHECK_PROTECT, ctx, True)` を通じて管理されており、ニードルガードも `_run_protect` を経由するため自動的に連続失敗ルールが適用される。

### `chip_on_contact` 追加による既存 `_run_protect` への影響

`_run_protect` を拡張するため、既存の `まもる`・`かえんのまもり`・`キングシールド`・`スレッドトラップ`・`トーチカ` への影響がないことを確認する必要がある。`chip_on_contact=None`（デフォルト）では既存動作と同一になるため、影響なし。

### 接触判定は `battle.query.is_contact(ctx)` で行う

さめはだ・てつのトゲ等の接触カウンター特性と同じ `battle.query.is_contact(ctx)` を使用する。ダイマックス技（`is_blocked_by_protect=False`）は `_check_protect_success` で防御判定をパスするため、`chip_on_contact` 分岐にも到達しない（`stop_event=True` が返される前に return するため問題なし）。実際には `_check_protect_success` が False を返した時点で関数が早期リターンし、接触チェックには入らない。

### ダイマックス技は1/4ダメージを受ける仕様

仕様書の記載「ダイマックス技や第7世代のZワザの攻撃技は貫通し、1/4のダメージを受ける」は、まもる共通の貫通ルール（`ON_CALC_PROTECT_MODIFIER` 等）で処理される。ニードルガードハンドラ側では対応不要。

### 変化技への保護

ニードルガードはデフォルト（`protect_non_attack=True`）で変化技も防ぐ。「なみだめ」・「ふきとばし」・「ほえる」がニードルガードを貫通する仕様は、それらの技側で `is_blocked_by_protect=False` ラベル等により管理されるため、ニードルガードハンドラ側では対応不要。

### turn.md の `ON_HIT` priority 100 との関係

`docs/spec/turn.md` の `Event.ON_HIT` セクションに「ニードルガード等の効果」が priority 100 で記載されている。これは「ダメージ発生後」のイベントだが、ニードルガードで技が防がれた場合はダメージが発生しない（`ON_TRY_MOVE_1` で `stop_event=True` により処理が中断）。この記述は「まもる成功時のカウンター処理全般」を指す仕様上の分類と解釈する。キングシールド・トーチカの接触カウンターが既存実装で `ON_TRY_MOVE_1` 内（`_run_protect`）で処理されており、同一パターンを踏襲する。
