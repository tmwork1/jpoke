# ぼうふう 実装計画書

## 仕様要約（docs/spec/moves/ぼうふう.md 参照）

* タイプ: ひこう / 分類: 特殊 / 威力: 110 / 命中: 70 / PP: 12
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果: 追加効果として、30%の確率で相手を『こんらん』状態にする。
* 天気が『あめ』のときは必ず命中する。天気が『にほんばれ』のときは命中率が50になる。
* `そらをとぶ` の溜め状態（回避状態）の相手にも命中する。
* 「風の技」に分類され、特性 かぜのり／ふうりょくでんき の対象になる。

## 現状の実装状況・レビューで検出した誤り

`src/jpoke/data/moves/move_ha.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。
レビュー時点で以下の誤りを検出し修正した：

1. **PP誤り**: `pp=10`（一次情報＝ポケモンWikiの値）になっていたが、
   `src/jpoke/data/ps-champ-ja/moves.json` の `ぼうふう` 行はPP12であり、本プロジェクトはポケモンチャンピオンズの
   値を優先するため `pp=12` に修正した（`ふぶき` のPP修正と同様のパターン）。

```python
"ぼうふう": MoveData(
    type="ひこう",
    category="special",
    pp=12,
    power=110,
    accuracy=70,
    flags={"wind", "secondary_effect"},
    handlers={
        Event.ON_MODIFY_ACCURACY: h.MoveHandler(
            ha.ぼうふう_accuracy,
            subject_spec="attacker:self"
        ),
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ぼうふう_apply_confusion_to_defender,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_MODIFY_ACCURACY` | 100（デフォルト） | attacker:self | 天候による命中率補正（あめ: 必中、にほんばれ: 50） |
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 30%の確率で相手を『こんらん』状態にする |

`src/jpoke/handlers/move_attack.py` の実装（既存・変更なし）:

```python
def ぼうふう_accuracy(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ぼうふうの天候による命中率補正。雨: 必中、晴れ: 50%
    攻撃側がばんのうがさを持つ場合、晴れでも命中率低下なし。
    防御側がばんのうがさを持つ場合、雨でも必中にならない。
    """
    if battle.weather_for(ctx.defender).rainy:
        return HandlerReturn(value=None)  # 必中
    elif battle.weather_for(ctx.attacker).sunny:
        return HandlerReturn(value=50)
    return HandlerReturn(value=value)


def ぼうふう_apply_confusion_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ぼうふうの追加効果: 30%の確率で相手をこんらん状態にする。"""
    return apply_confusion_to_defender(battle, ctx, value, chance=0.3)
```

`apply_confusion_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は汎用ハンドラ側で処理される。本技側に個別のガードは不要。

`battle.weather_for(mon)` はばんのうがさを持つポケモンに対して晴れ/雨系天候を「なし」として返すため、
攻撃側・防御側どちらがばんのうがさを持っていても正しく無効化される（`ぼうふう_accuracy` 側の実装で対応済み）。

`そらをとぶ` 状態の相手に命中する仕様は `handlers/volatile.py` の
`HIDDEN_MOVE_ALLOWED_MOVES["そらをとぶ"]` に `ぼうふう` が登録済みであることで実現しており、
本技側の追加実装は不要。

## Priority 根拠

* `ON_MODIFY_ACCURACY` はダメージ計算内部のイベントであり `docs/spec/turn.md` に priority 記載がないため、
  既存の同種ハンドラ（`かみなり_accuracy`・`ふぶき_accuracy`）と同じデフォルト priority（100）のパターンに従う。
* `ON_DAMAGE_HIT` によるこんらん付与も同様に、既存の同種ハンドラ（`マジカルアクセル_apply_confusion_to_defender`
  等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。

## 注意点・エッジケース（review-test 向け）

1. **天候による命中率補正**
   - あめ（防御側から見た天候）: 必中。ランク補正（回避・命中ランク）も無視される
     （`ON_MODIFY_ACCURACY` が `None` を返すと `_check_hit` がランク補正前にTrueを返すため）。
   - にほんばれ（攻撃側から見た天候）: 命中率50。ランク補正は適用される。
   - `battle.weather_for` によりばんのうがさ所持ポケモンには晴れ/雨の効果が及ばない。
2. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・こんらん付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
3. **そらをとぶとの相互作用**
   - `tests/volatiles/test_volatile_satana.py` の `test_そらをとぶ_ぼうふうが命中する` で検証済み。
4. **かぜのり／ふうりょくでんきとの相互作用**
   - `tests/abilities/test_ability_ka.py` の `test_かぜのり_風の技を吸収してこうげき上昇` /
     `test_かぜのり_かたやぶりで風技吸収無効` で検証済み。
5. **テスト方針**
   - `secondary_chance` を1.0に固定し、こんらん付与を確認するテスト
     （`test_ぼうふう_こんらんが発動する`）は既存。
   - 天候による命中率補正（あめ: 必中、にほんばれ: 命中率50）を確認するテストが未整備のため追加する
     （`test_ぼうふう_あめ中は必中になる` / `test_ぼうふう_にほんばれ中は命中率が50になる`）。
