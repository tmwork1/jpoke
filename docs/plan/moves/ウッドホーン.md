# ウッドホーン 実装計画書

参照: docs/spec/moves/ウッドホーン.md

## 概要

物理・くさタイプのドレイン技。相手に与えたダメージの50%だけ自分のHPを回復する。
既存のドレイン技（ギガドレイン・きゅうけつ・すいとる・ドレインパンチ等）と同一の共通処理
`_drain_hp`（`handlers/move_attack.py`）を再利用する。

## MoveData（data/moves/move_a.py）

既存定義（PPをチャンピオンズ準拠の12に修正済み）:

```python
"ウッドホーン": MoveData(
    type="くさ",
    category="physical",
    pp=12,
    power=75,
    accuracy=100,
    flags={"contact", "heal"},
    handlers={
        Event.ON_HIT: h.MoveHandler(ha.ウッドホーン_drain, priority=20)
    }
),
```

* `flags` に `"contact"`（直接攻撃）・`"heal"`（HPを吸収する技、かいふくふうじ／
  ヒーリングシフトの対象判定）を含める。
* `subject_spec` は `MoveHandler` のデフォルト `"attacker:self"` を使用（明示指定不要）。
* PP: `docs/champions/move_list.txt` の値（12）を正とする。第九世代本家値（10）は
  チャンピオンズのPP4段階統一（8/12/16/20）に移行する際の未修正バグだったため12に修正した。

## ハンドラ（handlers/move_attack.py）

```python
def ウッドホーン_drain(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """ウッドホーンの回復量を計算する。"""
    _drain_hp(battle, ctx, value, heal_ratio=0.5)
    return HandlerReturn(value=value)
```

* 共通関数 `_drain_hp(battle, ctx, damage, heal_ratio)` を呼び出す。
  * `damage`（与えたダメージ）が0の場合（みがわりに命中した場合）は `ctx.substitute_damage` を使う。
  * `heal_amount = max(1, round_half_up(damage * heal_ratio))` で算出する
    （**修正**: 従来は `int(damage * heal_ratio)` で切り捨てていたが、公式Wikiに
    「第五世代以降は四捨五入する」と明記されているため `round_half_up` に修正し、
    最低1回復を保証する `max(1, ...)` を追加した。この修正は `_drain_hp` を共用する
    全てのドレイン技に影響するが、heal_ratio=0.5/0.75 いずれも既存のグループテスト
    （与ダメ100・端数なし）の結果には影響しない）。
  * `Event.ON_CALC_DRAIN` を発火して `おおきなねっこ`（1.3倍）・`ヘドロえき`
    （回復→反動ダメージへの変換）を適用する。
  * `battle.modify_hp(ctx.attacker, v=heal_amount, reason="drain")` で攻撃者のHPを回復する。

## イベント・priority

* `Event.ON_HIT`（ダメージ発生後の処理）: priority=20
  * `docs/spec/turn.md` の `Event.ON_HIT` 表「20 | HP吸収技による回復」に対応。
  * 既存の全ドレイン技ハンドラ（ギガドレイン・きゅうけつ・すいとる・ドレインパンチ等）と同じ
    priority=20 で登録し、処理順の一貫性を保つ（既存実装のまま変更不要）。

## テスト方針

* `tests/moves_attack/test_move__group.py` の `test_drain技グループ_回復量が与ダメのheal_ratio倍になる`
  に `("ウッドホーン", 0.5)` として含める（既存）。
* `tests/moves_attack/test_move_a.py`:
  * `test_ウッドホーン_使用後に攻撃者のHPが回復する`（既存、基本動作）
  * `test_ウッドホーン_回復量が与ダメの半分になる`（既存、与ダメ100→回復50）
  * `test_ウッドホーン_回復量の端数は四捨五入で切り上げになる`（新規、
    与ダメ101→回復51になることを確認し、`_drain_hp` の四捨五入修正を検証する）
