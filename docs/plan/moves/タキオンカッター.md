# タキオンカッター 実装計画書

## 仕様要約（docs/spec/moves/タキオンカッター.md 参照）

* タイプ: はがね / 分類: 特殊 / 威力: 50（1ヒットあたり） / 命中: -（必中） / PP: 10
* 対象: 相手1体 / 優先度: 0 / 接触なし / フラグ: 切る技（`slash`）
* 効果:
  * 2回連続で攻撃する固定回数の連続攻撃技（命中判定自体が行われない必中技。各ヒット威力は固定50）。
  * 追加効果なし。
* テツノカシラの専用技。連続攻撃技として初の必中技（`ダブルアタック` 等と異なり `accuracy=None`）。

## 現状の実装状況（レビュー前）

`src/jpoke/data/moves/move_ta.py` に登録されていたが、以下の誤り・欠落があった。

```python
"タキオンカッター": MoveData(
    type="はがね",
    category="special",
    pp=10,
    power=50,
    accuracy=100,
    handlers={},  # 追加効果なし
),
```

* `accuracy=100` は誤り。一次情報では命中率「-」（必中）であり、`スピードスター` `でんげきは`
  `あてみなげ` 等と同様 `accuracy=None` にする必要がある（`accuracy=100` のままだと通常の
  命中率100%技として扱われ、命中ランク補正や回避率上昇の影響を受けてしまう）。
* `multi_hit` が設定されておらず、2回連続攻撃（連続攻撃技）の挙動が実装されていなかった
  （1回攻撃の通常技として動作してしまう）。
* `flags` に `slash`（切る技）が設定されておらず、特性『きれあじ』の対象外になってしまう。

## 修正後の実装

```python
"タキオンカッター": MoveData(
    type="はがね",
    category="special",
    pp=10,
    power=50,
    accuracy=None,
    flags={"slash"},
    multi_hit={
        "min": 2,
        "max": 2,
        "check_hit_each_time": False,
        "power_sequence": (),
    },
    handlers={},  # 追加効果なし
),
```

* `accuracy=None` により `core/move_executor.py` の `_check_hit` が呼ばれる前に必中として扱われる
  （`need_hit_check` が常に False になるため、`multi_hit.check_hit_each_time` の値は動作に影響しない。
  他の2回攻撃固定技との表記統一のため `False` を明記する）。
* `multi_hit.min == multi_hit.max == 2` のため、`_resolve_hit_count`（`core/move_executor.py`）は
  常に2を返す。
* `power_sequence` が空タプルのため、`_resolve_hit_power` は各ヒットとも `move.power`（50）を返す。
* `flags={"slash"}` により、特性『きれあじ』の威力補正判定（`move.has_flag("slash")` 等）の対象になる。

## ハンドラ構成

固有ハンドラなし（`handlers={}`）。固定2ヒット・必中の挙動は `core/move_executor.py` 側の
汎用機構（`accuracy=None` と `multi_hit`）で処理されるため、`handlers/move_attack.py` への
追加実装は不要。

## 注意点・エッジケース（review-test 向け）

1. **必中（`accuracy=None`）**
   - 命中率0%まで下げても、回避率を上げても必ず命中することを確認するテスト。
   - まもる・かげぶんしん等の「命中判定を介さない無効化」は別途 `ON_BEFORE_APPLY_MOVE` 等で処理されるため
     必中でも防がれる（`まもる: ○` の仕様通り）。
2. **2回連続攻撃**
   - `run_move` 実行後、`defender.hits_taken == 2` となることを確認する基本テスト
     （`ダブルアタック` `にどげり` と同一パターン）。
3. **切る技（`slash`）**
   - 特性『きれあじ』所持時に威力が上がることを確認するテスト（`handlers/ability.py` の
     きれあじハンドラが `move.has_flag("slash")` を参照している前提。既存の切る技テストと同一パターン）。
4. **PP**
   - `docs/champions/move_list.txt` にタキオンカッターの記載が存在しないため、Champions側の
     正式なPP値を確認できなかった。現状は第九世代本家のPP値（10）を維持している。
     同系統の2回攻撃固定技（ダブルアタック・ダブルウイング・ツインビーム）は本家PP10から
     Champions側でPP12に統一されているため、タキオンカッターも同様にPP12となる可能性が高いが、
     一次情報での確認が取れるまでは値を変更しない（推測での上書きを避ける）。
