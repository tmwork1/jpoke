# 計画書: みちづれ

出典: `docs/spec/moves/みちづれ.md`（技本体）、`docs/spec/volatiles/みちづれ.md`（付与される揮発性状態）

## 仕様要約

- ゴーストタイプ / 変化 / 威力・命中率なし / PP8（champions 基準） / 対象: 自分
- 使用すると自分に「みちづれ」揮発性状態を付与する
  - 次の自分の行動順までの間に相手の攻撃技でひんしになると、攻撃してきたポケモンもひんしにする
  - 揮発性状態そのものの詳細（発動条件・消滅条件・マジックガード等の無視）は `docs/spec/volatiles/みちづれ.md` を参照
- **第七世代以降**: みちづれを成功させた直後にもう一度みちづれを使うと必ず失敗する（連続使用不可）
  - 連続使用に失敗した直後にもう一度使った場合は成功する
  - 状態異常・なまけ等で行動できなかった場合も「連続使用」とみなされる
  - ねごと/さいはいによるみちづれは直接使用したときと同一視される
- `まねっこ` で使用不可（`flags={"non_copycat"}`）
- `ねごと` では選択可能（`non_negoto` フラグは付与しない）

| `ON_TRY_MOVE_2` | 未指定（デフォルト） | `attacker:self`（デフォルト） | 直前の自分の行動が成功裏の「みちづれ」だった場合、今回の使用を失敗させる |
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自分に「みちづれ」揮発性状態を付与する |

## Priority 根拠

- `ON_TRY_MOVE_2`: `docs/spec/turn.md` に「みちづれの連続使用失敗」は明記されていない。
  同じ「連続使用したら必ず失敗する」系統の `まもる` / `みきり` が
  `まもる系_連続使用失敗チェック` を `ON_TRY_MOVE_2` に priority 未指定（デフォルト）で
  登録しているため、同一パターンを採用する。
- `ON_STATUS_HIT`: 状態変化技の通常付与処理と同じデフォルト priority=100 を使用する
  （`apply_volatile_to_attacker` を使う他ハンドラと同様）。

## 実装箇所

### handlers/move_status.py（実装済み）

挿入位置（五十音順）: `みずびたし_can_apply` の後、`ミラータイプ_apply` の前

```python
def みちづれ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_attacker(battle, ctx, value, volatile="みちづれ")


def みちづれ_連続使用失敗チェック(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """みちづれの連続使用失敗チェック（第七世代以降）。

    直前の自分の行動で成功裏にみちづれを使用していた場合、今回のみちづれは失敗する
    （まもる系_連続使用失敗チェックと同じパターン）。
    失敗時は executed_move を None にリセットし、次回の使用は連続使用扱いにしない。
    """
    mon = ctx.attacker
    if (
        mon.executed_move is not None
        and mon.executed_move.name == "みちづれ"
    ):
        battle.add_event_log(
            mon, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="みちづれ_連続使用")
        )
        mon.executed_move = None
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース

- **連続使用失敗の判定ロジック**: `mon.executed_move`（直前に発動条件を満たして実行された技）が
  「みちづれ」自身であるかどうかで判定する。`executed_move` は `move_executor._execute_move` の
  `ON_TRY_MOVE_1` / `ON_TRY_MOVE_2` 通過後（実際のダメージ・効果適用より前）に更新されるため、
  以下の原作仕様を追加コードなしで自然に満たす:
  - 状態異常・なまけ等で行動できなかったターンは `executed_move` が更新されないため、
    「連続使用」とみなされる
  - きあいパンチが集中力切れで失敗した場合は `ON_TRY_MOVE_1`（priority 30）で
    `move_success=False` となり `executed_move` は更新されない
  - バッグから道具を使った／逃げようとして失敗した場合はそもそも `run_move` 自体が
    呼ばれないため `executed_move` は変化しない
  - ねごと/さいはいでみちづれを選んだ場合は `battle.run_move(attacker, みちづれ)` を経由して
    実行されるため、直接使用した場合と同じ経路で `executed_move` が更新される
- **既知の限界**: 特性「おどりこ」により相手の踊り技を強制的にコピー実行した場合、原作では
  それを挟んでもみちづれの連続使用扱いは継続するが、本実装では別の技が `run_move` を経由して
  実行されると `executed_move` が上書きされてしまうため、この極めて稀なケースには対応していない
  （`まもる系_連続使用失敗チェック` も同じ設計上の限界を持つ）
- 失敗時は `mon.executed_move = None` にリセットする。これにより直後にもう一度みちづれを
  使った場合は連続使用とみなされず成功する（原作仕様通り）
- **リーサル計算**: 対象外（ダメージを与えない変化技、n/a）
