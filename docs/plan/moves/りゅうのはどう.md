# 計画書: りゅうのはどう

## 仕様要約（docs/spec/moves/りゅうのはどう.md 参照）

- ドラゴンタイプ / 特殊 / 威力85 / 命中100 / PP12 /
  対象: 1体選択（遠隔） / 優先度0 / 接触なし
- 追加効果なし（通常攻撃のみ）。
- `pulse` フラグを持つ（波動技。メガランチャー等の対象）。
- `src/jpoke/data/ps-champ-ja/moves.json:944` に記載あり（ドラゴン / 特殊 / 85 / 100 / 接触× / まもる○）。
  PP欄は12であり、`docs/champions/changes_from_sv.md`（全技PPを8/12/16/20の4段階に統一）
  に整合する。既存実装は正しく `pp=12` になっている。

## 既存実装との差分

既存実装（`data/moves/move_ra.py` の `"りゅうのはどう"` エントリ）は `pp=12` で仕様と一致。
差分なし。

※ 他の類似技（りゅうせいぐん pp=5、リーフストーム pp=5、ラスターカノン pp=10 等）は
champions move_list.txt の PP（8/8/12）と乖離しており、Gen9基準PPのまま未修正である
可能性が高い。これらは本技のスコープ外だが、別途レビューが必要。

それ以外（type・category・power・accuracy・flags・handlers空）は仕様と一致しており差分なし。

## ハンドラ構成

追加効果を持たないため、専用ハンドラは不要。`handlers={}` のままでよい。

## data/moves/move_ra.py（修正後）

```python
"りゅうのはどう": MoveData(
    type="ドラゴン",
    category="special",
    pp=12,
    power=85,
    accuracy=100,
    flags={"pulse"},
    handlers={},  # 追加効果なし
),
```

## リーサル計算

対象外（`docs/progress/move.md` の「リーサル実装」列が `n/a`。HP依存の追加効果・威力変動を
持たない通常のダメージ技のためリーサル計算ハンドラは不要）。

## 注意点・エッジケース

- `pulse` フラグにより、メガランチャー等の波動技強化特性の対象となる（本技固有のハンドラ実装は
  不要、特性側のハンドラが `flags` を参照して処理する）。
- まもるで防がれた場合はダメージが発生しないが、追加効果自体を持たないため考慮不要。
