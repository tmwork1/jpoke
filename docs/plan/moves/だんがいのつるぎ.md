# 計画書: だんがいのつるぎ

## 仕様要約

- じめんタイプ / 物理 / 威力120 / 命中85 / PP10 / 対象: 相手全員（シングルでは相手1体） / 優先度0 / 非接触
- 追加効果なし（通常攻撃のみ）
- 『あなをほる』状態の相手への威力2倍は対象外（`じしん`・`マグニチュード`のみが対象）
- 『グラスフィールド』での威力半減も対象外（`じしん`・`じならし`・`マグニチュード`のみが対象）

## ハンドラ構成

- だんがいのつるぎ自体には追加効果がないため、`data/move.py` の `だんがいのつるぎ` エントリは
  `handlers={}` のみで基本ダメージ計算に委ねる。
- 『あなをほる』状態の相手への威力2倍（`handlers/volatile.py` の `あなをほる_boost_power`）は
  `ctx.move.name in ("じしん", "マグニチュード")` の条件で判定しており、`だんがいのつるぎ` は
  対象外のため、追加のハンドラは不要。
- 『グラスフィールド』での威力半減（`handlers/field.py` の `グラスフィールド_power_modifier`）も
  `ctx.move.name in ["じしん", "じならし", "マグニチュード"]` の条件で判定しており、
  `だんがいのつるぎ` は対象外のため、追加のハンドラは不要。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、priority の検討対象イベントは存在しない。

## 実装箇所

### data/move.py への更新

既存の `"だんがいのつるぎ"` エントリは以下の通りで、仕様と一致している（変更不要）。

```python
    "だんがいのつるぎ": MoveData(
        type="じめん",
        category="physical",
        pp=10,
        power=120,
        accuracy=85,
        handlers={},  # 追加効果なし
    ),
```

- `flags` を指定していないため `contact` フラグは付与されない → 非接触（仕様と一致）。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致（シングルバトルでは「相手全員」も
  実質「相手1体」で差異なし）。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `critical_rank` 省略時のデフォルトは `0`（通常の急所判定）→ 仕様と一致。

## 注意点・エッジケース

- じしんと混同して『あなをほる』『グラスフィールド』のハンドラにだんがいのつるぎを追加登録しない
  こと（公式仕様上、両方とも対象外）。
- 追加効果がないため、まもる・おうじゃのしるし・オウムがえし等の一般的な技判定は
  フレームワーク側の共通処理（`handlers/move_attack.py` の基本ダメージ計算経路）で
  自動的に処理され、本技固有のハンドラ実装は不要。
