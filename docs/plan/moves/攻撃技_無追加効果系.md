## 注意点・エッジケース（review-test 向け）

### なみのり

- ダイビング中の相手への命中・威力2倍は `data/volatile.py` の `ダイビング` VolatileData に実装済み。なみのり側のハンドラ追加は不要
- `HIDDEN_MOVE_ALLOWED_MOVES["ダイビング"]` に `"なみのり"` が含まれており、命中判定の特別処理も対応済み
- `ダイビング_boost_power` の条件が `ctx.move.name in ("なみのり", "うずしお")` であるため、なみのり側のラベル等は不要

### 音系技（ハイパーボイス・ばくおんぱ）

- `move_executor.py` の `_hits_substitute` で `has_label("sound")` が False を返すため、みがわり状態のポケモンにも直接ダメージが届く
- 両技はすでに `labels=["sound"]` を設定済みであるため、追加対応不要
- `ぼうおん` 特性（`ON_TRY_MOVE_1` priority 110）による無効化は既存特性ハンドラが処理する

### はなふぶき

- `labels=["wind"]` が設定済み。風技として `かぜのり` 特性等の対象になる
- `_move_labels.md` に `はなふぶき` が `wind` ラベルリストに含まれており、設定は正しい

### タネばくだん

- `labels=["bullet"]` が設定済み。弾技として `ぼうだん` 特性等の対象になる
- `_move_labels.md` に `タネばくだん` が `bullet` ラベルリストに含まれており、設定は正しい

### りゅうのはどう

- `labels=["pulse"]` が設定済み。はどう技として `メガランチャー` 特性等の対象になる
- `_move_labels.md` に `りゅうのはどう` が `pulse` ラベルリストに含まれており、設定は正しい

### PP不一致について

- なみのり（仕様: pp=16、現状: pp=15）とハイドロポンプ（仕様: pp=8、現状: pp=5）の2技でPPが仕様と異なる
- これら以外の13技のPPは仕様通り

### 実装済み扱いの基準

- 追加効果のない攻撃技において `MoveData` が正しいパラメータで登録されていれば実装完了とする
- ハンドラなしの技は `data/move.py` のエントリのみで動作する（攻撃・命中・ダメージ計算はフレームワークが処理）
