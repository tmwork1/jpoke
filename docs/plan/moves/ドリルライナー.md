# 計画書: ドリルライナー

## 仕様要約（docs/spec/moves/ドリルライナー.md 参照）

- じめんタイプ / 物理 / 威力80 / 命中95 / PP12（Champions基準） / 対象: 相手1体 / 優先度0 / 接触あり
- 追加効果なし。急所ランク+1（急所に当たりやすい）。

## 既存実装との差分

既存実装（`data/moves/move_ta.py` の `"ドリルライナー"` エントリ）は `pp=10` になっており、
`docs/champions/move_list.txt`（PP=12）と不一致。Gen9本家基準の値が移行漏れで残っているため、
`pp=12` に修正する。

1. **基本情報**
   `type="じめん", category="physical", power=80, accuracy=95` は仕様通り。
   `pp` のみ `10` → `12` に修正が必要。

2. **急所ランクは `critical_rank=1` で仕様通り**
   追加効果を持たない技のため専用ハンドラは不要。`critical_rank` フィールドのみで
   `core/move_executor.py` の急所判定ロジック（`CRIT_RATES[critical_rank]`）が
   自動的に適用される。

3. **`flags` は `{"contact"}` で仕様通り**
   Wikiのカテゴリ一覧に「斬る技」該当カテゴリの記載がないため `slash` フラグは不要。

## ハンドラ構成

専用ハンドラなし（`handlers={}`）。

## リーサル計算

対象外（`docs/progress/move.md` の「リーサル実装」列が `n/a`。追加効果を持たない通常の
物理攻撃技のためリーサル計算ハンドラは不要）。

## 注意点・エッジケース

- 急所ランクは通常のダメージ計算の急所判定に組み込まれるため、まもる等で防がれた場合は
  ダメージ自体が発生せず急所判定も行われない（既存の共通ロジックに準拠）。
- 追加効果を持たないため、ちからずくの影響を受けない（威力・効果とも変化しない）。
- PPはdocs/champions/move_list.txtを正とする（8/12/16/20の4段階統一のうち12）。Wiki記載のPP10は
  Gen9本家基準であり本プロジェクトでは採用しない。
