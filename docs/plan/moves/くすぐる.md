# 実装計画: くすぐる

参照: docs/spec/moves/くすぐる.md

## 仕様サマリ

- タイプ: ノーマル / 分類: 変化 / PP: 20 / 命中率: 100
- 効果: 相手のこうげき・ぼうぎょランクを 1 段階ずつ下げる
- まもる: ○ / マジックコート: ○ / 直接攻撃: ×
- 対象: 1体選択

## 現状の実装状況

`data/moves/move_ka.py` にエントリが存在し、`handlers` に `ON_STATUS_HIT` が登録済み（実装済み）。

```python
"くすぐる": MoveData(
    type="ノーマル",
    category="status",
    pp=20,
    accuracy=100,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.くすぐる_modify_defender_stats,

```

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の同種ハンドラ（`にらみつける_modify_defender_stats`（B-1）、`なきごえ_modify_defender_stats`（A-1）など）が priority=100 デフォルトで実装されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py への追加（実装済み）

挿入位置: 五十音順で `くさわけ_*` の後、`くちばしキャノン_*` の前（く の並び）

```python
def くすぐる_modify_defender_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """くすぐるの効果: 相手のこうげきとぼうぎょを 1 段階ずつ下げる。"""
    return modify_defender_stats(battle, ctx, value, stats={"atk": -1, "def": -1})
```

### data/moves/move_ka.py の登録（実装済み）

`くすぐる` の MoveData に `handlers` を設定済み。

```python
"くすぐる": MoveData(
    type="ノーマル",
    category="status",
    pp=20,
    accuracy=100,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.くすぐる_modify_defender_stats,
        ),
    }
),
```

## 注意点・エッジケース

- こうげき・ぼうぎょの 2 種類のステータスを同時に下げる。`modify_defender_stats` の `stats` 引数に辞書で両方渡せばよい
- ランク低下の無効化（しろいきり、クリアボディ等）はフレームワーク内で処理されるため、ハンドラ側のガードは不要
- `みがわり` 状態の相手には命中しない（sound ラベルなし。フレームワーク側で処理済み）
