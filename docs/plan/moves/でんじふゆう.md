# 計画書: でんじふゆう

## 仕様要約

- でんきタイプ / 変化 / 命中なし / PP12（`src/jpoke/data/ps-champ-ja/moves.json` 準拠。Gen9本家のPP10ではない） / 対象: 自分
- 5ターンの間、『ふゆう』状態となりじめんタイプの技やフィールド効果を受けなくなる
- すでにでんじふゆう状態の場合は失敗（`battle.volatile_manager.apply` の「既に同じ揮発性状態があれば失敗」という
  汎用ルールでフレームワーク側が処理するため個別ハンドラ不要）
- ねをはる状態・うちおとす状態のポケモンが使うと失敗する（`ON_TRY_MOVE_1` priority 30）
- じゅうりょく状態中は使用不可（`ON_TRY_MOVE_1` priority 30、`gravity_restricted` フラグ経由の共通ハンドラ）
- ターン終了時にでんじふゆうのカウントが消費される（`ON_TURN_END` priority 110、揮発性状態側で処理済み）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_TRY_MOVE_1` | 30 | attacker:self | `gravity_restricted_fail`: じゅうりょく状態中の失敗（共通ハンドラ、`gravity_restricted` フラグ） |
| `ON_TRY_MOVE_1` | 30 | attacker:self | `でんじふゆう_check_can_use`: ねをはる・うちおとす状態での失敗 |
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | 自分にでんじふゆう状態付与 |

## Priority 根拠

`docs/spec/turn.md` の `ON_TRY_MOVE_1` priority 30 に「たくわえる: カウントが3」など
使用条件チェック系の失敗が並んでいるため、同じ priority=30 を採用する
（`gravity_restricted_fail` を使う他の技（そらをとぶ・とびはねる・フライングプレス等）も
すべて priority=30 で登録されており、これに合わせる）。
`ON_TURN_END` priority 110 に `でんじふゆうの終了` が記載されており、揮発性状態側の
`でんじふゆう_tick_volatile`（`data/volatile.py`）で処理済み。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `でんじは` の後、`とおせんぼう` の前（五十音順）。

```python
def でんじふゆう_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """でんじふゆうの効果: 自分をでんじふゆう状態にする（5ターン）。"""
    return apply_volatile_to_attacker(battle, ctx, value, volatile="でんじふゆう", count=5)


def でんじふゆう_check_can_use(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """でんじふゆうの使用条件チェック: ねをはる・うちおとす状態のポケモンは失敗する。"""
    mon = ctx.attacker
    if mon.has_volatile("ねをはる") or mon.has_volatile("うちおとす"):
        battle.add_event_log(
            mon, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="でんじふゆう_接地状態")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### data/moves/move_ta.py への更新

```python
    "でんじふゆう": MoveData(
        type="でんき",
        category="status",
        pp=12,
        target="self",
        flags={"gravity_restricted"},
        handlers={
            Event.ON_TRY_MOVE_1: [
                h.MoveHandler(
                    h.gravity_restricted_fail,
                    subject_spec="attacker:self",
                    priority=30,
                ),
                h.MoveHandler(
                    hs.でんじふゆう_check_can_use,
                    subject_spec="attacker:self",
                    priority=30,
                ),
            ],
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.でんじふゆう_apply,
            ),
        }
    ),
```

## 注意点・エッジケース

- `target="self"` を必ず指定する。欠落すると `Move.is_blocked_by_protect` /
  `Move.is_reflectable` のデフォルト判定基準である `target == "foe"` に一致してしまい、
  相手のまもる・マジックコートに誤反応する（本レビューで発見・修正した既存バグ）。
- PP はデフォルト実装で10になっていたが、`src/jpoke/data/ps-champ-ja/moves.json` の12が正
  （本レビューで発見・修正した既存バグ）。Gen9本家のPP10へ差し戻さないこと。
- すでにでんじふゆう状態の失敗チェックは `battle.volatile_manager.apply` 内の汎用ロジックで
  処理されるためハンドラ内ガード不要（`data/move.py` 側の個別実装は行わない）。
- ねをはる・うちおとす状態での失敗は元々未実装だったため、`でんじふゆう_check_can_use` を
  新規追加した（本レビューで発見・修正した既存バグ）。
- じゅうりょく状態中の使用不可は `ON_TRY_MOVE_1` の `gravity_restricted_fail` で
  フレームワーク側が処理する（`flags={"gravity_restricted"}` を指定するのみ）。
- じゅうりょく状態になるとでんじふゆうが解除される処理は `field_manager` 側で担当（別スコープ）。
- count=5 を明示して渡す（アクアリングは count 指定なし=永続だが、でんじふゆうは5ターン制限）。
- `ON_TRY_MOVE_2` priority 110 に `でんじふゆう等によるじめん技無効` の判定があり、volatile 付与後は
  自動的にじめん技を無効化する（`data/volatile.py` の `でんじふゆう` VolatileData 側で実装済み）。
- くろいてっきゅうを持っていても、でんじふゆうの使用自体は成功する
  （`ON_CHECK_FLOATING` をアイテム側が上書きするだけで、技の成否には影響しない）。
