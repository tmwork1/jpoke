# 計画書: でんじふゆう

## 仕様要約

- でんきタイプ / 変化 / 命中なし / PP10 / 対象: 自分
- 5ターンの間、『ふゆう』状態となりじめんタイプの技やフィールド効果を受けなくなる
- すでにでんじふゆう状態の場合は失敗（`ON_APPLY_MOVE` priority 100: `でんじふゆう: すでに状態変化` でフレームワーク側が処理）
- じゅうりょく状態中は使用不可（`ON_CHECK_ACTION` priority 100: `じゅうりょくで技が出せない`）
- ターン終了時にでんじふゆうのカウントが消費される（`ON_TURN_END` priority 110）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | 自分にでんじふゆう状態付与 |

## Priority 根拠

`docs/spec/turn.md` の `ON_TURN_END` priority 110 に `でんじふゆうの終了` が記載されており、`ON_APPLY_MOVE` priority 100 に `でんじふゆう: すでに状態変化` が記載されている。ハンドラ本体は既存の `ON_STATUS_HIT` priority 100 のパターンに従う（`アクアリング_apply_volatile_to_attacker` と同形式）。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `でんじは` の後、`トーチカ` の前（五十音順）。

```python
def でんじふゆう_apply_volatile_to_attacker(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """でんじふゆうの効果: 自分をでんじふゆう状態にする（5ターン）。"""
    return apply_volatile_to_attacker(battle, ctx, value, volatile="でんじふゆう", count=5)
```

### data/move.py への更新

`"でんじふゆう"` エントリのハンドラを追加する。

```python
    "でんじふゆう": MoveData(
        type="でんき",
        category="変化",
        pp=10,
        handlers={

```

## 注意点・エッジケース

- すでにでんじふゆう状態の失敗チェックは `ON_APPLY_MOVE` priority 100 でフレームワーク側が処理するためハンドラ内ガード不要
- じゅうりょく状態中の使用不可は `ON_CHECK_ACTION` でフレームワーク側が処理
- じゅうりょく状態になるとでんじふゆうが解除される処理も別途フレームワーク側が担当
- count=5 を明示して渡す（アクアリングは count 指定なし=永続だが、でんじふゆうは5ターン制限）
- `ON_TRY_MOVE_2` priority 110 に `でんじふゆう等によるじめん技無効` の判定があり、volatile 付与後は自動的にじめん技を無効化する
