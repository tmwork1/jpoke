# 実装計画書: かふんだんご

## 仕様要約

- タイプ: むし / 分類: 特殊 / 威力: 90 / 命中率: 100 / PP: 15 / 対象: 1体選択（自分以外） / 優先度: 0
- 直接攻撃: × / 弾（bullet）のわざ
- 本来の効果: 対象が味方である場合のみ、ダメージを与えずに対象の最大HPの1/2を回復する回復技になる（味方選択時は変化技ではなく特殊技のまま）。
- 味方選択時もヒーリングシフトの効果（優先度+3）は適用されない。

出典: `docs/spec/moves/かふんだんご.md`

## シングルバトルにおける扱い

本プロジェクトはシングルバトルのみを対象とする。シングルバトルには「味方（ダブルバトルの相方）」が存在しないため、かふんだんごは常に相手（`defender`）を対象として選択される。したがって、味方への回復効果・かいふくふうじ関連の分岐・おうごんのからだ／テラスシェル／おやこあい／マジシャンとの相互作用は発生し得ず、実装上は「追加効果のない90威力のむし特殊技」として扱えばよい。

## 現状の実装状況

| 項目 | 状態 |
|------|------|
| `data/move.py` エントリ | 実装済み（`type="むし"`, `category="special"`, `power=90`, `accuracy=100`, `pp=15`） |
| `flags` | `{"bullet"}` のみを付与する（`"heal"` は付与しない） |
| `handlers/move_attack.py` 関数 | 不要（追加効果なし） |

## 修正点

- `flags` に `"heal"` が誤って含まれていた。`"heal"` フラグは `ヒーリングシフト_modify_priority`（`handlers/ability.py`）が優先度+3を判定する際に参照するものだが、`docs/spec/abilities/ヒーリングシフト.md` に「プレゼント/かふんだんごで回復効果が出る場合も、回復技とは見なされず適用されない」と明記されている。また `docs/spec/moves/カテゴリ.md` の「回復わざ / heal」カテゴリにも かふんだんご は含まれていない。よって `"heal"` フラグを外し、`{"bullet"}` のみとする。

## ハンドラ構成

追加のハンドラは不要。`flags={"bullet"}` のみで、既存の `ぼうだん` 特性ハンドラ（`ON_TRY_MOVE_2` 等、`bullet` ラベルを見て弾のわざを無効化する）がそのまま機能する。

## Priority 根拠

対象外（追加ハンドラなし）。

## 実装詳細・エッジケース

- シングルバトルでは味方が存在しないため、対象は常に相手（`defender`）。ダメージ計算は通常のむし特殊技と同様に行われる。
- `ぼうだん` 特性を持つ相手には無効化される（`bullet` フラグにより既存ハンドラが対応）。
- 味方への回復効果・かいふくふうじ関連分岐・ヒーリングシフト除外は、ダブルバトル専用の要素のため実装・テスト対象外。
