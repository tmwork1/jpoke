# 実装計画: かげぶんしん

## 仕様サマリ

- タイプ: ノーマル / 分類: 変化 / PP: 16（champions/move_list.txt準拠。Gen9本家は15） / 命中率: なし（自分対象）
- 効果: 自分の回避率を 1 段階上げる
- まもる: × / マジックコート: × / 直接攻撃: ×
- 対象: 自分

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自分の回避率を 1 段階上げる |

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の自分のランク上昇ハンドラ（`こうそくいどう_modify_attacker_stats`（S+2）、`かたくなる_modify_attacker_stats`（B+1）など）がすべて priority=100 デフォルトで実装されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: 五十音順で `かえんのまもり_*` の後、`かたくなる_*` の前（かげ → かた の順）

```python
def かげぶんしん_boost_attacker_evasion(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """かげぶんしんの効果: 自分の回避率を 1 段階上げる。"""
    return modify_attacker_stats(battle, ctx, value, stats={"evasion": 1})
```

### data/moves/move_ka.py の更新

`かげぶんしん` の MoveData に `handlers` を追加する。

```python
"かげぶんしん": MoveData(
    type="ノーマル",
    category="status",
    pp=16,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.かげぶんしん_boost_attacker_evasion,
        ),
    }
),
```

## 注意点・エッジケース

- 対象は自分（`target` 未指定で `self` がデフォルト）であり、`modify_attacker_stats` を使う
- ランク上限（+6）チェックはフレームワーク側（`ON_TRY_MOVE_2` priority=130）で処理されるため、ハンドラ側のガードは不要
- `ちいさくなる`（EVA+2 かつ揮発性状態付与）と異なり、かげぶんしんは EVA+1 のみで揮発性状態は付与しない
- 命中率なし（自分対象）のため accuracy 設定不要。テストは命中率固定不要
