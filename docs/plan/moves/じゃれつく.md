# じゃれつく 実装計画書

## 仕様要約（docs/spec/moves/じゃれつく.md 参照）

* タイプ: フェアリー / 分類: 物理 / 威力: 90 / 命中: 90 / PP: 10
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 追加効果として、10%の確率で相手の『こうげき』ランクを1段階下げる。

## 現状の実装状況

`src/jpoke/data/move.py` に既に登録済みで、仕様と一致している（変更不要）。

```python
"じゃれつく": MoveData(
    type="フェアリー",
    category="physical",
    pp=10,
    power=90,
    accuracy=90,
    flags={"contact", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.じゃれつく_lower_defender_atk,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 10%の確率で相手の『こうげき』ランクを1段階下げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def じゃれつく_lower_defender_atk(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"atk": -1}, chance=0.1)
```

`modify_defender_stats`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は `flags={"secondary_effect"}` さえ
付与されていれば汎用的に処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（能力ランク低下）は既存の同種ハンドラ（あわ・うらみつらみ・
きあいだま 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。
`ON_DAMAGE_HIT` は `docs/spec/turn.md` に未掲載のイベント（ダメージ計算内部）だが、
既存の同種ハンドラの実装パターンを踏襲するため個別の priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` が付与されているため、ちからずく所持時は威力1.3倍（90→117）になり、
     こうげき低下の追加効果は発動しなくなる。
2. **接触フラグ**
   - `flags={"contact"}` により、`ゆびがみのおの` 等の接触時発動効果・特性（『さめはだ』『てつのとげ』等）
     の対象となる。個別のガードは不要（各特性・アイテム側で判定される）。
3. **テスト方針**
   - `secondary_chance` を0/1に固定し、こうげきランクの変化有無を確認するテストを追加する
     （`test_じゃれつく_相手にダメージを与える` / `test_じゃれつく_こうげき1段階低下が発動する` /
     `test_じゃれつく_こうげき1段階低下が発動しない`）。
