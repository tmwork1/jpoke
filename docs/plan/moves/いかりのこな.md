# 実装計画書: いかりのこな

参照: docs/spec/moves/いかりのこな.md

## 仕様要約

- タイプ: むし / 分類: 変化 / PP: 20 / 命中: - / 優先度: +2
- 効果: **ちゅうもくのまと状態**になり、そのターンの相手の技をすべて自分が受ける。対象を選択する技にしか効果はない。
- **ダブル/トリプルバトル専有の効果**。本プロジェクト（シングルバトル専用）では
  `battle.actives` が常に2体以下であり、対象リダイレクト自体に意味がない。
- 粉・胞子技（第六世代以降）: くさタイプ・特性「ぼうじん」・道具「ぼうじんゴーグル」を持つ相手の
  行動はこの技によるちゅうもくのまと状態を無視するが、これも上記リダイレクト機構に付随する
  ダブル専用の判定であるため対象外。

出典: `docs/spec/moves/いかりのこな.md`、`docs/champions/move_list.txt` L66-67

## 実装方針の判断（対象外）

- `docs/progress/move.md` の いかりのこな 行は「仕様書」「計画書」「実装」「テスト」
  「リーサル実装」「リーサルテスト」列すべてが「ダブル専用」であり、本プロジェクトの
  対象外として扱う。
- 類似技「このゆびとまれ」も同じ理由で全列「ダブル専用」として扱われており
  （`docs/progress/move.md` 該当行）、一貫した方針とする。
- `docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` 表には「このゆびとまれ: シングルバトルである」
  （priority=30）という記載が存在するが、これは将来ダブルバトル対応を行う場合の参考情報として
  残されているものであり、このゆびとまれ自身が未実装のまま「ダブル専用」区分で据え置かれている
  以上、いかりのこなについても同じ判断（未実装のまま据え置き）とする。
  - なお、本プロジェクトの `Battle` は常に `len(battle.actives) <= 2`（各プレイヤー1体ずつ）
    という構造的制約を持つため、この失敗条件を実装すると「いかりのこなは常に必ず失敗する」という
    自明で分岐のないハンドラになる。この技だけを特別扱いして実装することは、同条件を満たす
    他の全ダブル専用技（このゆびとまれ・おさきにどうぞ・てだすけ等）との一貫性を欠くため見送る。

## 注意点・エッジケース（将来ダブルバトル対応時のために記録）

1. **いかりのこな自体に `powder` フラグを付与してはならない**:
   本コードベースの `powder` ラベルによる無効化判定（`ぼうじん`・`ぼうじんゴーグル`・くさタイプ）は
   `ctx.defender`（技を受ける側）に対して働く。いかりのこなは自分を対象とする技だが、この
   プロジェクトの `AttackContext` 生成規則上 `ctx.defender` は「使用者の相手」に固定されるため、
   `powder` フラグを付けると「相手がくさタイプならいかりのこな自体が無効化される」という
   誤った判定になってしまう（詳細は `docs/spec/moves/いかりのこな.md` 参照）。
   実際に無効化されるべきなのは「いかりのこなで発生したちゅうもくのまと状態を無視して行動する
   相手側」の判定であり、いかりのこな自身の成否とは無関係。
   （※以前の版の本計画書はこの点を誤り、`labels=["powder"]` の追加を提案していたため訂正した。）
2. **このゆびとまれとの関係**: 第五世代では両技は完全に同一効果だったが、第六世代以降は
   いかりのこなのみ粉技に分類され、無効化手段が追加された。
