# 実装計画書: いかりのこな

## 仕様要約

- タイプ: むし / 分類: 変化 / PP: 20 / 命中: - / 優先度: +2
- 効果: **ちゅうもくのまと状態**になり、そのターンの相手の技をすべて自分が受ける。対象を選択する技にしか効果はない。
- **ダブルバトル専用**。シングルバトルでは失敗する。
- **粉・胞子技（第六世代以降）**: 下記に無効:
  - くさタイプのポケモン
  - 特性「ぼうじん」のポケモン
  - 道具「ぼうじんゴーグル」を持つポケモン

出典: `docs/spec/moves/_all_moves.md` L33、`docs/spec/moves/status_move.md` L18

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_TRY_MOVE_1` | 30 | `attacker:self` | シングルバトル（場のポケモン数が2以下）では失敗させる |

**注意**: ちゅうもくのまと状態の付与（`ON_STATUS_HIT`）はダブルバトル用のターゲットリダイレクト機能であり、シングルバトル専用プロジェクトでは実装対象外。`powder` ラベルを付与するだけで「くさタイプ・ぼうじん・ぼうじんゴーグル」の無効化は既存ハンドラが担当する。

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` セクションに以下の記載あり：

```
| 30 | このゆびとまれ: シングルバトルである |
```

「いかりのこな」は「このゆびとまれ」と同一の効果区分（ダブルバトル専用のターゲットリダイレクト）であるため、同じ priority=30 を採用する。

## 実装箇所

### 1. `src/jpoke/handlers/move_status.py` への追加

以下のコードを `い` の位置（五十音順: いえき より前、いたみわけ より前）に追加する。

```python
def いかりのこな_check_single_battle(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """いかりのこな: シングルバトルでは失敗する（ダブルバトル専用）。"""
    if len(battle.actives) <= 2:
        battle.add_event_log(ctx.attacker, LogCode.MOVE_FAILED,
                             payload={"reason": "いかりのこな"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### 2. `src/jpoke/data/move.py` の更新

既存の `"いかりのこな"` エントリに `labels` と `handlers` を追加する。

## 注意点・エッジケース（review-test 向け）

1. **シングルバトルでの失敗**: `battle.actives` の長さが 2 以下（自分と相手の 1 対 1）のときは無条件で失敗する。`len(battle.actives) <= 2` で判定する。

2. **くさタイプ・ぼうじん・ぼうじんゴーグルの無効化はハンドラ不要**:
   - `labels=["powder"]` を付与することで既存の以下のハンドラが機能する:
     - `ON_TRY_MOVE_1` priority=110: ぼうじん特性による粉技無効 (`ability.py`)
     - `ON_TRY_MOVE_2` priority=110: ぼうじんゴーグルによる粉技無効 (`item.py`)
     - `ON_TRY_MOVE_2` priority=120: くさタイプによる粉技無効 (`volatile.py` or `move.py`)
   - ただし「いかりのこな」は自分に使う技（target="self"）であり、上記の無効化は「いかりのこなを**使用された**ポケモン」に関する話であることに注意。シングルバトルでは機能しないため、この無効化はダブルバトル実装時の考慮事項。

3. **いかりのこなを使用者が ぼうじん・くさタイプ・ぼうじんゴーグルを持っていても技は成功する**:
   - wikiによると「いかりのこなは自分を対象とする技だが、上記の条件を持つポケモンが使っても成功し、ちゅうもくのまと状態になれる」とある。
   - ただしシングルバトルでは ON_TRY_MOVE_1 priority=30 で事前に失敗するため、この点の実装は不要。

4. **このゆびとまれとの類似性**: 「このゆびとまれ」も同様にシングルバトルでは失敗させる必要がある（現在未実装）。両技は同一の失敗チェックロジックを持つ。

5. **テスト対象**: シングルバトルでの失敗確認のみテスト可能。ちゅうもくのまとによるターゲットリダイレクト効果はダブルバトル機能であり、本プロジェクトのテスト対象外。
