# 計画書: つばめがえし

## 仕様要約

- ひこうタイプ / 物理 / 威力60 / 命中- (必中) / PP20 / 対象: 1体選択 / 優先度0 / 接触
- ラベル: slash
- 追加効果なし（`accuracy=None` により必ず命中する）

## ハンドラ構成

追加効果を持たないため、専用ハンドラは不要。`handlers={}` のみで基本ダメージ計算に委ねる。
`accuracy` フィールドを省略（`None`）することで、`move_executor._check_hit` が命中判定自体を
スキップし必中となる（`docs/spec/moves/_move_labels.md` の他の必中技と同様の実装方針）。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、priority の検討対象イベントは存在しない。

## 実装箇所

### data/move.py への更新

既存の `"つばめがえし"` エントリは以下の通りで、仕様と一致している（変更不要）。

```python
    "つばめがえし": MoveData(
        type="ひこう",
        category="physical",
        pp=20,
        power=60,
        flags={"contact", "slash"},
        handlers={},  # 追加効果なし
    ),
```

- `accuracy` を指定していないため `None`（デフォルト）となり、必中技として扱われる。
- `flags={"contact", "slash"}` により接触判定・きれあじ／テクニシャン等の対象になる → 仕様と一致。
- `target` 省略時のデフォルトは `"foe"`（相手1体）→ 仕様と一致。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `critical_rank` 省略時のデフォルトは `0`（通常の急所判定）→ 仕様と一致。

## 注意点・エッジケース

- 必中技のため、`はりきり`の命中率ペナルティ・`ON_MODIFY_ACCURACY`イベント自体が発火しない
  （`move_executor.py` の `_check_hit` が `accuracy is None` の時点で即 `True` を返すため）。
  `はりきり_modify_accuracy` 側にも `ctx.move.accuracy is not None` のガードがあり、二重に安全。
- まもる・おうじゃのしるし・オウムがえし等の一般的な技判定はフレームワーク側の共通処理
  （`handlers/move_attack.py` の基本ダメージ計算経路）で自動的に処理され、本技固有のハンドラ実装は不要。
