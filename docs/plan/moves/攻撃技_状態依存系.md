## Priority 根拠

`docs/spec/turn.md` には ON_CALC_POWER_MODIFIER / ON_CALC_DEF_RANK_MODIFIER / ON_CALC_DEF_TYPE_MODIFIER / ON_CALC_BURN_MODIFIER はダメージ計算内部イベントとして掲載されていない（「Interrupt: ダメージ計算」としてまとめられている）。

既存の類似ハンドラの登録コードを根拠とする：
- `ON_CALC_POWER_MODIFIER`: `はたきおとす_power`（data/move.py）、`ちからずく_boost`（data/ability.py）がデフォルト priority で登録されている
- `ON_CALC_DEF_RANK_MODIFIER`: `てんねん_ignore_rank`（data/ability.py）がデフォルト priority で登録されている
- `ON_CALC_DEF_TYPE_MODIFIER`: `フリーズドライ_water_effectiveness`（data/move.py）がデフォルト priority で登録されている
- `ON_CALC_BURN_MODIFIER`: ailment.py 側の既存実装に合わせてデフォルト priority

以上より、今回追加するすべてのハンドラはデフォルト priority（MoveHandler に priority 未指定）とする。

### handlers/move_attack.py に追加する関数

```python
def からげんき_double_power_when_ailment(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """からげんき: 使用者が状態異常のとき威力が2倍になる。"""
    if ctx.attacker.ailment.is_active:
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)


def からげんき_ignore_burn_modifier(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """からげんき: やけど状態でも物理攻撃の威力が半減しない。

    ailment.py が ON_CALC_BURN_MODIFIER で 2048（0.5倍）を返すため、
    ここで 4096（1.0倍）で上書きして半減を打ち消す。
    """
    if ctx.attacker.ailment.name == "やけど":
        value = 4096
    return HandlerReturn(value=value)


def たたりめ_double_power_when_ailment(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """たたりめ: 対象が状態異常のとき威力が2倍になる。"""
    if ctx.defender.ailment.is_active:
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)


def ベノムショック_double_power_when_poisoned(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ベノムショック: 対象がどく/もうどく状態のとき威力が2倍になる。"""
    if ctx.defender.ailment.name in ("どく", "もうどく"):
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)


def ダメおし_double_power_when_hit(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ダメおし: 同ターン中に対象が既にダメージを受けていたら威力が2倍になる。"""
    if ctx.defender.hits_taken > 0:
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)


def うっぷんばらし_double_power_when_rank_lowered(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """うっぷんばらし: そのターン中に使用者の能力ランクが下げられていたら威力が2倍になる。"""
    if ctx.attacker.stat_lowered_this_turn:
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)


def しっぺがえし_double_power_when_second(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """しっぺがえし: 同ターン後攻で行動する場合、威力が2倍になる。"""
    attacker_player = battle.get_player(ctx.attacker)
    if battle.query.is_second_actor(attacker_player):
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)


def DDラリアット_ignore_def_rank(battle: Battle, ctx: AttackContext, value: float) -> HandlerReturn:
    """DDラリアット: 相手のランク変化を無視してダメージ計算する（防御ランク補正を1.0に固定）。"""
    return HandlerReturn(value=1)


def せいなるつるぎ_ignore_def_rank(battle: Battle, ctx: AttackContext, value: float) -> HandlerReturn:
    """せいなるつるぎ: 相手のランク変化を無視してダメージ計算する（防御ランク補正を1.0に固定）。"""
    return HandlerReturn(value=1)


def きまぐレーザー_maybe_double_power(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """きまぐレーザー: 30%の確率で威力が2倍になる。"""
    if battle.random.random() < 0.3:
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)


def アシストパワー_boost_power_by_rank(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """アシストパワー: 使用者のランク上昇合計1段階ごとに威力が20増加する（基本威力20）。

    威力 = 20 + 20 * rank_sum = 20 * (1 + rank_sum)
    ON_CALC_POWER_MODIFIER は 4096 = 1.0 倍基準のため
    modifier = 4096 * (1 + rank_sum) を apply_fixed_modifier に渡す。
    """
    attacker = ctx.attacker
    # 正のランク変化の合計を計算（A/B/C/D/S のみ対象、ACC/EVA を除く）
    rank_sum = sum(max(0, v) for k, v in attacker.rank.items() if k in ("A", "B", "C", "D", "S"))
    if rank_sum > 0:
        modifier = 4096 * (1 + rank_sum)
        value = apply_fixed_modifier(value, modifier)
    return HandlerReturn(value=value)


def フライングプレス_add_flying_type(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """フライングプレス: かくとうタイプに加えてひこうタイプとしても相性計算する。

    damage_calculator.py がすでにかくとうタイプ相性を value に計算済み。
    ここでひこうタイプの相性を追加で掛け算することで複合タイプ判定を実現する。
    """
    from jpoke.data import TYPE_MODIFIER
    flying_chart = TYPE_MODIFIER.get("ひこう", {})
    for def_type in ctx.defender.types:
        rate = flying_chart.get(def_type, 1.0)
        value = int(value * rate)
    return HandlerReturn(value=value)
```

| ハンドラ | subject_spec | 理由 |

| からげんき_double_power_when_ailment | （省略） | ON_CALC_POWER_MODIFIER はダメージ計算内部の attacker 固有イベント。はたきおとす等の既存実装に合わせて省略可 |

## 注意点・エッジケース (review-test 向け)

### からげんき
- **やけど補正の無効化**: ailment.py が `ON_CALC_BURN_MODIFIER` で `2048`（0.5倍）を返す。`からげんき_ignore_burn_modifier` が `4096`（1.0倍）で上書きすることで半減を打ち消す。ailment.py のやけど補正ハンドラより後に発火させるため priority の順序を確認すること（どちらもデフォルト priority のため発火順は不定。必要であれば priority を大きくする）。
- **どく/まひ状態でも威力2倍**: 状態異常なら種類を問わず 2 倍。`ailment.is_active` で判定。
- **みがわり越し**: ON_CALC_POWER_MODIFIER はダメージ計算内部で発火するため、みがわり越しでも適用される。

### たたりめ / ベノムショック
- **たたりめ**: どく/まひ/やけど/ねむり/こおりすべての状態異常で2倍。`ailment.is_active` で判定。
- **ベノムショック**: どく/もうどくのみ。`ailment.name in ("どく", "もうどく")` で限定。
- 対象が「ねむけ」（あくびの揮発性状態）の場合は `ailment.is_active=False` のため2倍にならない。

### ダメおし
- `ctx.defender.hits_taken` はターン開始時に `reset_turn_state()` でリセットされ、`move_executor.py` 内でダメージ後に `hits_taken += 1` される。
- ダメおし自身のダメージは hits_taken に加算されるが、判定は計算前（ON_CALC_POWER_MODIFIER は ON_HIT より前に発火）のため自己参照は発生しない。
- マルチヒット技の2回目以降は、1回目で `hits_taken >= 1` となるため2倍になる。これは仕様通り。

### うっぷんばらし
- チャンピオンズ版の仕様は「そのターン中に能力ランクを下げられると威力2倍」（ポケモン本家の「各-1毎に+20、最大140」とは異なる）。
- **前提**: `"ランクダウン"` volatile は存在しない（調査済み）。`Pokemon.stat_lowered_this_turn: bool` フィールドの新設が必要。
- **フィールド設定タイミング**: `core/battle.py` の `modify_stat` でランクが実際に下がった場合（返り値が負）に `target.stat_lowered_this_turn = True` を設定する。
- **リセットタイミング**: `pokemon.reset_turn_state()` で `stat_lowered_this_turn = False` にリセットする（ターン開始時）。
- **場に出た時のランクダウンも対象**: チャンピオンズ仕様では「場に出た時に能力ランクを下げられた場合も対象」とある。ON_SWITCH_IN 後のランクダウン（いかく等）も `stat_lowered_this_turn = True` になるため、`reset_turn_state()` がターン開始時（ON_SWITCH_IN より前）に呼ばれないよう注意。

### しっぺがえし
- `battle.query.is_second_actor(attacker_player)` は `player_states[player].action_order_index == 1` で判定（1vs1想定）。
- `action_order_index` はターン開始時に `_record_action_order` で記録され、ターン終了時の `reset_turn_state` で `None` にリセットされる。
- 同速同優先度でランダム先攻の場合、後攻になったプレイヤーは `action_order_index == 1` なので正しく判定される。
- `is_second_actor` が `None` を返す場合（ターン外等）は `False` として扱われるため問題なし。

### DDラリアット / せいなるつるぎ
- `ON_CALC_DEF_RANK_MODIFIER` に `attacker:self` を指定すると、「この技の攻撃者」が自分と一致するハンドラのみ発動する。
- `てんねん` は `defender:self`（防御側特性として相手の攻撃ランクを無視）なのとは逆の登録方向になる。
- **急所時の処理**: `damage_calculator.py` では急所時に `r_rank = min(r_rank, 1)` する（防御ランク+補正を無視）。DDラリアット/せいなるつるぎが `1` を返せば `min(1, 1) = 1` となり問題なし。
- **攻撃ランクは無視しない**: DDラリアット/せいなるつるぎは**防御ランクのみ**無視（攻撃ランクは通常通り計算）。`ON_CALC_DEF_RANK_MODIFIER` のみ登録することで実現。

### きまぐレーザー
- `battle.random.random() < 0.3` の固定確率を使用する（ON_CALC_POWER_MODIFIER は追加効果フェーズではないため `resolve_secondary_chance` は不適切）。
- チャンピオンズ仕様では「30%程度の確率で威力2倍」。

### アシストパワー
- `rank.items()` は ACC/EVA を含む辞書のため、A/B/C/D/S のみを集計する必要がある（`if k in ("A", "B", "C", "D", "S")`）。
- **固定小数点の扱い**: ON_CALC_POWER_MODIFIER の value は 4096 = 1.0 倍基準。基本威力 20 に対して `(1 + rank_sum)` 倍を適用すると威力 = `20 * (1 + rank_sum)` になる。
  - rank_sum=0 のとき: modifier なし（威力20のまま）
  - rank_sum=1 のとき: `4096 * 2 = 8192` → 威力40
  - rank_sum=6 のとき: `4096 * 7 = 28672` → 威力140
- **整数オーバーフロー**: Python の int は任意精度のため overflow なし。

### フライングプレス
- `damage_calculator.py` の `calc_def_type_modifier` はすでに `ctx.move.type`（= "かくとう"）で相性を計算して value に格納している。ON_CALC_DEF_TYPE_MODIFIER ハンドラで受け取った value にひこうタイプ相性を追加で掛け算する。
- **型注意**: `value` は int（4096 基準）。`int(value * rate)` で丸め込む。
- **フライングプレスとSTAB**: STAB（タイプ一致補正）は `ON_CALC_ATK_TYPE_MODIFIER` で処理され、技タイプ "かくとう" のみを参照する。ひこうタイプのSTABは発生しない（仕様通り）。
- **ちいさくなる補正**: チャンピオンズ仕様では「ちいさくなるを使用していると必ず命中し威力2倍」と記載あり。"minimize" ラベルがすでに設定されているため、minimize 判定の既存実装に依存する。
- **ノーマルタイプ対かくとう**: フライングプレスの複合タイプ（かくとう × ひこう）でノーマルタイプに対しては「かくとう: 2倍、ひこう: 等倍」→ 合計 2 倍になる。

### イカサマ / ボディプレス（追加実装不要）
- `damage_calculator.py` の `_calc_final_attack` 内で move.name による分岐が実装済み。
- `イカサマ`: 相手の A ステータス + ランク補正を参照。
- `ボディプレス`: 自分の B ステータス + ランク補正を参照。
- ハンドラの追加は不要。ただし `ability_paradox.py` にも対応コードがある点に注意（クォークチャージ/こだいかっせいがイカサマ/ボディプレスに対応している）。
