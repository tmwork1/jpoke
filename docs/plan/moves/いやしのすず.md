# 実装計画書: いやしのすず

## 仕様要約

- タイプ: ノーマル / カテゴリ: 変化 / PP: 5 / 命中: なし（必中） / 対象: 味方全体
- 自分と味方全体のポケモンの**状態異常を治す**
- **音系の技**（`sound` ラベル）: ぼうおんに無効化される。みがわり状態でも効果が発生する
- 使用者側で状態異常のポケモンが誰もいない場合は技が失敗する（`ON_APPLY_MOVE` priority=100: アロマセラピー等）
- `heal` ラベルなし（`かいふくふうじ` の対象外）
- ダイウォールの効果を受けない
- シングルバトルでは「味方全体」＝使用者自身のみが対象

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` セクションに以下の記載がある：

> | 100 | アロマセラピー等: 状態異常味方なし |

このイベントはコード上の `ON_BEFORE_APPLY_MOVE` に対応する。ただし、同種の類似技（`いのちのしずく`・`じこさいせい`・`なまける`）はすべて `ON_STATUS_HIT`（priority=100）内でチェックと効果処理を一括して行うパターンを採用している（`いのちのしずく_heal_self` 参照）。`いやしのすず` も同パターンで実装し、priority はデフォルト（100）とする。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「い」行 — `いばる_modify_defender_stats_and_apply_volatile` の後、`いやなおと_modify_defender_stats` の前に挿入する。

```python
def いやしのすず_cure_team_ailment(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """いやしのすず: 自分（シングルバトルでは選出チーム）の状態異常を回復する。

    音系の技のためみがわり状態でも効果が発生する。
    チームに状態異常のポケモンがいない場合は技が失敗する。
    """
    mon = ctx.attacker
    player = battle.get_player(mon)
    state = battle.player_states[player]
    # シングルバトルでは selection は場にいる 1 体のみ
    targets = [m for m in state.selection if m.ailment.is_active]
    if not targets:
        return HandlerReturn(value=False, stop_event=True)
    for target in targets:
        battle.ailment_manager.remove(target)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース

- **シングルバトル限定実装**: `state.selection` はシングルバトルでは場の 1 体のみ。「チームに状態異常がいなければ失敗」の判定と「全員を治療するループ」を実装することで、ダブルバトル拡張時にも対応できる構造にしておく
- **音技のみがわり貫通**: `sound` ラベルによりみがわり状態のポケモンにも効果が発生する。`ON_CHECK_SUBSTITUTE` フィルタが音技を通過するため、ハンドラ側での追加チェックは不要
- **ぼうおんによる無効化**: `sound` ラベルを持つため、相手が `ぼうおん` 特性の場合は技が無効化される。ただし「いやしのすず」は自分側を対象とする変化技（target="self"等）のため、シングルバトルでは相手のぼうおんは関係ない
- **チームに状態異常がいない場合の失敗**: ターン.md「アロマセラピー等: 状態異常味方なし」に対応。`ON_STATUS_HIT` 内でチェックし、`HandlerReturn(value=False, stop_event=True)` を返すと `move_executor.py` 内で `MOVE_FAILED` ログが記録される
- **アロマセラピーとの違い**: アロマセラピー（くさタイプ）は音技でない。いやしのすずは音技で `sound` ラベルを持つ。機能はほぼ同等だが `heal` ラベルは両方とも持たない
- **かいふくふうじとの相互作用**: `heal` ラベルが付いていないため、`かいふくふうじ` 状態でも使用可能。ハンドラ側での追加チェックは不要
- **ailment_manager.remove** は `uncurable` フラグの状態異常（例: 特性由来の絶対ねむり）を解除しない。ハンドラ側で追加チェックせずとも `remove` 内部で保護されている
