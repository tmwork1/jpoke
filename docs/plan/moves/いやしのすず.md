# 実装計画書: いやしのすず

参照仕様書: `docs/spec/moves/いやしのすず.md`

## 仕様要約

- タイプ: ノーマル / カテゴリ: 変化 / PP: 8（`docs/champions/move_list.txt` 準拠） / 命中: なし（必中） / 対象: 味方全体
- 自分と味方全体（シングルバトルでは選出した控えを含む）のポケモンの**状態異常を治す**
- **音系の技**（`sound` フラグ）: みがわり状態でも効果が発生する（音技は貫通するため）
- 使用者側のチームに状態異常のポケモンが誰もいない場合は技が失敗する
- `heal` フラグなし（`かいふくふうじ` の対象外）
- まもる・ダイウォールの効果を受けない（`target="own_side"` のため、相手を対象とする効果ではない）
- `target="own_side"` を明示する必要がある（後述のバグ参照）

## 現状の実装状況（レビュー前）

| 項目 | 状態 |
|------|------|
| `data/moves/move_a.py` エントリ | 登録済みだが誤りあり（`pp=5` はGen9本家値の移行漏れ、`target` 未設定でデフォルト`"foe"`のまま） |
| `handlers/move_status.py` 関数 | 実装済み（`いやしのすず_cure_ailment`） |

## レビューで発見したバグと修正

1. **PP誤り**: `pp=5`（Gen9本家値）になっていた。`docs/champions/move_list.txt` の値は `8`。→ `pp=8` に修正。
2. **`target` 未設定によるバグ**: `MoveData.target` のデフォルトは `"foe"` だが、いやしのすずは
   `docs/champions/move_list.txt` 上「味方全体」が対象であり、相手を対象とする技ではない。
   `target` を明示していなかったため、以下の副作用があった。
   - `is_blocked_by_protect`（`target=="foe" and not has_flag("unprotectable")`）が `True` になり、
     相手が `まもる` を使っていた場合に誤って技がブロックされていた（`守×` のはずが `守○` 相当の挙動になっていた）。
   - `マジックコート_reflect`（`ctx.move.category == "status" and ctx.move.target in {"foe", "foe_side"}`）が
     `True` になり、相手が `マジックコート` を使っていた場合に誤って技が跳ね返されていた。
   - `ぼうおん_block_sound`（`subject_spec="defender:self"`）が `ctx.move.target` を見ていなかったため、
     相手（`ctx.defender`）が `ぼうおん` の場合に誤って技が無効化されていた。本来 いやしのすず は
     相手を対象としないため、相手の `ぼうおん` は無関係（`docs/spec/abilities/ぼうおん.md` 参照）。
   - → `data/moves/move_a.py` に `target="own_side"` を追加し、`ぼうおん_block_sound`
     （`handlers/ability.py`）に `ctx.move.target != "foe"` の場合は無効化しないガードを追加した
     （`_apply_type_absorb` 等、既存の `defender:self` 系ハンドラと同じパターン）。
     この修正で `まもる`・`マジックコート`・相手の `ぼうおん` のいずれからも影響を受けなくなる。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自分（チーム）の状態異常がない場合に失敗させる → チーム全体の状態異常を回復 |

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 個別の記載はない（ダメージ計算内部等と同様、技実行フロー内部のイベント）。
同種の類似技（`いのちのしずく`・`じこさいせい`・`なまける`）はすべて `ON_STATUS_HIT`（priority=100=デフォルト）内で
失敗チェックと効果処理を一括して行うパターンを採用しており（`いのちのしずく_heal` 参照）、`いやしのすず` も
同パターンで実装済み。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

`いやしのすず_cure_ailment` として実装済み（五十音順: `いばる_apply` の後、`いやしのねがい_apply` の前）。

```python
def いやしのすず_cure_ailment(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """いやしのすず: 自分（シングルバトルでは選出チーム）の状態異常を回復する。

    音系の技のためみがわり状態でも効果が発生する。
    チームに状態異常のポケモンがいない場合は技が失敗する。
    """
    mon = ctx.attacker
    player = battle.get_player(mon)
    state = battle.player_states[player]
    targets = [m for m in state.selection if m.ailment.is_active]
    if not targets:
        return HandlerReturn(value=False, stop_event=True)
    for target in targets:
        battle.ailment_manager.remove(target)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/moves/move_a.py`

```python
"いやしのすず": MoveData(
    type="ノーマル",
    category="status",
    pp=8,
    target="own_side",
    flags={"sound"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.いやしのすず_cure_ailment,

```

### `src/jpoke/handlers/ability.py`

`ぼうおん_block_sound` に `ctx.move.target != "foe"` ガードを追加（相手を対象としない音技を誤って
無効化しないようにするため）。

## 注意点・エッジケース

- **チーム全体対応**: `state.selection` は選出した控えポケモンも含むため、シングルバトルでも
  控えの状態異常を含めて回復できる（第五世代以降の仕様に合致）
- **音技のみがわり貫通**: `sound` フラグによりみがわり状態のポケモンにも効果が発生する。
  `check_hit_substitute` は `target != "foe"` の場合に即 `False` を返すため、いずれにせよ
  みがわりの影響を受けない
- **ぼうおんによる無効化**: 使用者自身・相手のぼうおんは無関係（最新世代基準）。控えのぼうおんも無関係。
  詳細は `docs/spec/abilities/ぼうおん.md` を参照
- **チームに状態異常がいない場合の失敗**: `ON_STATUS_HIT` 内でチェックし、
  `HandlerReturn(value=False, stop_event=True)` を返すと `MOVE_FAILED` ログが記録される
- **かいふくふうじとの相互作用**: `heal` フラグが付いていないため、`かいふくふうじ` 状態でも使用可能
- **ailment_manager.remove** は `uncurable` フラグの状態異常（例: 特性ぜったいねむりによる「ゆめうつつ」）を
  解除しない。ハンドラ側で追加チェックせずとも `remove` 内部で保護されている
