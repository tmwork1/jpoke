# 実装計画: サンダープリズン

## 仕様サマリ

- タイプ: でんき / 分類: 特殊 / 威力: 80 / 命中率: 90 / PP: 15 / 接触: なし
- 効果: 命中すると相手を『バインド』状態にし、4〜5ターンの間、毎ターン終了後に最大HPの1/8のダメージを与え続ける
- バインド状態の詳細仕様は `docs/spec/volatiles/バインド.md` を参照
- まもる: ○ / マジックコート: ×（変化技ではない） / 追加効果: なし
- レジエレキの専用技
- PPは `docs/champions/move_list.txt` に記載がない（レジエレキ専用技のため同ファイルの記載漏れと
  判断）。原作（第八/第九世代）値の15を採用する。

## 対象イベント

`ON_DAMAGE_HIT` — ダメージ命中時に発火。既存のバインド系技（うずしお・しめつける・すなじごく・
まきつく・ほのおのうず・からではさむ・まとわりつく・トラバサミ・マグマストーム）と共通の
`apply_bind_to_defender` ハンドラを流用する。

## Priority

`docs/spec/turn.md` に `ON_DAMAGE_HIT` の記載なし。
既存のバインド系技がすべて `MoveHandler(apply_bind_to_defender)` のみ（priority=100デフォルト）
で実装されているため、同様にデフォルト priority を採用する。

## 実装状況（実装済み・修正1件）

### handlers/move_attack.py

`apply_bind_to_defender` を流用（新規実装不要）。

### data/moves/move_sa.py

既存実装は `handlers` のみ登録済みで、`lethal_handlers` が欠落していた（他のバインド系技は
すべて `lethal_handlers` を持つ）。以下のとおり追加する:

```python
"サンダープリズン": MoveData(
    type="でんき",
    category="special",
    pp=15,
    power=80,
    accuracy=90,
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(ha.apply_bind_to_defender)
    },
    lethal_handlers={
        LethalEvent.ON_HIT: LethalHandler(l._apply_bind)
    }
),
```

`flags` に `contact` は付与しない（直接攻撃ではないため、wiki記載の「直接攻撃: ×」と一致）。

## リーサル計算

`_apply_bind`（`handlers/lethal.py`）を流用（新規実装不要、`lethal_handlers` への登録漏れのみ修正）。
バインドの毎ターンダメージはバインド揮発性状態の `ON_TURN_END` リーサルハンドラ側で処理される。

## テスト方針

- 命中時に相手がバインド揮発状態になる確認
- バインド中は相手が毎ターン最大HPの1/8ダメージを受ける確認
- バインド中は相手が交代できない確認
- 攻撃者が交代するとバインドが解除される確認
- （`うずしお`・`すなじごく`・`まとわりつく` の既存テストと同パターン。攻撃者はレジエレキ専用技の
  ため `Pokemon("レジエレキ", move_names=["サンダープリズン"])` を使用する）
- `tests/test_lethal.py` に、バインド付与（技経由）とバインド事前付与の結果が一致することを確認する
  リーサルテストを追加する（`うずしお`・`すなじごく`・`まとわりつく` と同パターン）
