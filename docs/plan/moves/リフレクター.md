# 実装計画書: リフレクター

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 20 / 対象: 味方の場（target="own_side"）
- 5ターンの間、相手の物理攻撃のダメージを半分にする
- 味方が複数の場合は半分ではなく 2/3（シングルバトルでは半減固定）
- 急所に当たった場合は軽減されない
- 交代しても効果は続く

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「設置技: すでに最大」が priority 100 として記載されている。同種の `ステルスロック_set_field` が priority デフォルト（100）を使用しているため、同一パターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: `ロックオン_apply_volatile_to_defender` の後、`ロックカット_modify_attacker_stats` の前（「リ」→「ろ」の間、カタカナ「リ」は「り」として処理）。

```python
def リフレクター_set_side_field(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """リフレクター: 自陣営に「リフレクター」を5ターン設定する。"""
    side = battle.get_side(ctx.attacker)
    if not side.activate("リフレクター", 5):
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

既存の `"リフレクター"` エントリを更新する。`target` を `"own_side"` に変更し、handlers を追加する。

```python
"リフレクター": MoveData(
    type="エスパー",
    category="変化",
    pp=20,
    target="own_side",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.リフレクター_set_side_field,
        ),
    }
),
```

## 注意点・エッジケース

- 現在 `data/move.py` の target は `"field"` になっているため `"own_side"` への変更が必要
- `side.activate` が False を返す（すでにリフレクターが有効）場合は失敗として `HandlerReturn(value=False, stop_event=True)` を返す
- 物理ダメージ半減の本体は side_field ハンドラ側で処理されており、技側はフィールド設定のみ
- ひかりのかべ・しんぴのまもりと同一パターン（5ターン・自陣営）
