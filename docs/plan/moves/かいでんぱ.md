# かいでんぱ 実装計画書

## 仕様要約

- 相手の **特攻** を **2段階** 下げる変化技
- 命中率 100%、明示的な失敗チェックなし

## 実装方針

`いやなおと` / `うそなき` 等の既存能力低下技と同じパターン。
`ON_STATUS_HIT` で `modify_defender_stats({"C": -2})` を呼ぶだけ。

### ハンドラ構成

| イベント | Priority | 役割 |
|----------|----------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | 相手の特攻を2段階下げる |

### Priority 根拠

- `ON_STATUS_HIT priority=100`：既存の能力低下技（`いやなおと`, `うそなき`, `えんまく` 等）に準じる

### subject_spec

- `ON_STATUS_HIT`: `attacker:self`（MoveHandlerデフォルト）

## 実装箇所

### handlers/move_status.py（五十音順: かいでんぱは「か」行、アンコールの後）

```python
def かいでんぱ_modify_defender_stats(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"C": -2})
```

### data/move.py（「かいでんぱ」MoveData にハンドラを追加）

```python
"かいでんぱ": MoveData(
    type="でんき",
    category="変化",
    pp=15,
    accuracy=100,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.かいでんぱ_modify_defender_stats,
        ),
    }
),
```

## 注意点・エッジケース（review-test 向け）

1. **特攻最低ランク**: `modify_stats` 内で `actual_value == 0` になるためスキップ。ランク変化なしのログも出ないが、明示的な MOVE_FAILED ログも不要（既存技と同様）。
2. **クリアボディ・シールドダスト等**: `ON_BEFORE_MODIFY_STAT` または `ON_MODIFY_STAT` 内で既存ガードが働くため、ハンドラ側での追加チェック不要。
3. **みがわり**: `modify_defender_stats` → `battle.modify_stats` の中でみがわり越しの能力低下は防がれる（既存の仕組みで対応済み）。
