# 計画書: ツタこんぼう

## 仕様要約（docs/spec/moves/ツタこんぼう.md 参照）

- くさタイプ（条件により変化）・物理・威力100・命中100・PP10・優先度0・急所ランク+1
- 非接触技（`flags` に `contact` を含めない）
- 使用者が **オーガポン** の場合のみタイプが変化する。かぶっている仮面フォルムに応じて
  以下のように変化する（オーガポンの本来の（フォルム基準の）タイプ、`base_types` の末尾要素）。

| フォルム | 技タイプ |
|----------|---------|
| オーガポン(みどり) | くさ |
| オーガポン(いど) | みず |
| オーガポン(かまど) | ほのお |
| オーガポン(いしずえ) | いわ |

- オーガポン以外が使用した場合（まねっこ・ものまね・スケッチ等でのコピー含む）は常にくさタイプ
- タイプ判定はフォルムの `base_types`（`data.types`）で行うため、みずびたし等による現在タイプの
  変更や、マジックルーム・ぶきようによる仮面の威力補正無効化の影響を受けない
- オーガポンの仮面は `removable=False` かつ `なげつける` はオーガポン使用時に失敗するため、
  戦闘中に仮面を失うことはない（`src/jpoke/data/item.py` で既に実装済み）

## Priority 根拠

`ON_MODIFY_MOVE_TYPE` は `docs/spec/turn.md` に個別行の記載がない（ダメージ計算内部のイベント）。

既存の類似ハンドラ（いずれも `MoveHandler` のデフォルト priority=100 を使用）を根拠とする：

- `だいちのはどう_modify_move_type`（`data/moves/move_ta.py`）
- `さばきのつぶて_modify_move_type`（`data/moves/move_sa.py`）
- `テラバースト_modify_move_type`（`data/moves/move_ta.py`）

これらすべて priority を明示指定していない（デフォルト100）。ツタこんぼうも同様に priority=100 とする。

## 実装箇所

### `src/jpoke/handlers/move_attack.py` に追加する関数

`つけあがる_calc_power` と `つららおとし_apply_flinch` の間（五十音順: つ→つ→つ）に配置する。

```python
def ツタこんぼう_modify_move_type(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ツタこんぼう: 使用者がオーガポンの場合、かぶっている仮面（フォルム）に応じてタイプが変わる。

    フォルムの基本タイプ（base_types）で判定するため、みずびたし等による現在タイプの変更や、
    マジックルーム・ぶきよう等で仮面の威力補正が無効化されている場合でもタイプは変化しない。
    オーガポン以外が使用した場合は常にくさタイプになる。
    """
    attacker = ctx.attacker
    if not attacker.name.startswith("オーガポン"):
        return HandlerReturn(value=value)
    return HandlerReturn(value=attacker.base_types[-1])
```

`base_types[-1]` は、オーガポン(みどり) では `["くさ"]` の末尾（くさ）、
オーガポン(いど) では `["くさ", "みず"]` の末尾（みず）というように、
フォルムごとの「非くさ側」タイプ（単色フォルムはくさ自身）を返す。

### `src/jpoke/data/moves/move_ta.py` の登録変更

既存の `"ツタこんぼう"` エントリに `handlers` を追加する：

```python
    "ツタこんぼう": MoveData(
        type="くさ",
        category="physical",
        pp=10,
        power=100,
        accuracy=100,
        critical_rank=1,
        handlers={
            Event.ON_MODIFY_MOVE_TYPE: h.MoveHandler(
                ha.ツタこんぼう_modify_move_type,
            ),
        },
    ),
```

## 対象外（未実装依存のため保留）

- へんしん（Transform）でオーガポンに変身した場合のタイプ変化: `へんしん` 技自体が
  `src/jpoke/data/moves/move_ha.py` で `handlers={}`（実装保留）のため、本計画では対応しない。
  へんしん実装時に、変身後の見た目フォルムを参照する形で本ハンドラとの整合性を再確認すること。
- イリュージョンの見た目分岐（モーションのみ変化しタイプはくさ固定）: モーション表現は
  本シミュレーターの対象外（ダメージ計算・状態変化のみ扱うため）。

## リーサル計算

`docs/progress/move.md` の該当行「リーサル実装」列が `-`（対象）。ただし本技は追加効果のない
固定威力の通常攻撃技であり、タイプ変化は命中前に確定するため、既存の一般攻撃技用リーサル計算
ロジック（`handlers/lethal.py` の汎用処理）でカバーされる想定。本技専用のリーサルハンドラは不要。
