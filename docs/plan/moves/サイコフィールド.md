# 実装計画書: サイコフィールド

## 仕様要約

- タイプ: エスパー / カテゴリ: status / PP: 12（`src/jpoke/data/ps-champ-ja/moves.json` 準拠） / 対象: 全体の場（target="field"）
- 5ターンの間、地形を「サイコフィールド」にする
- すでにサイコフィールドが発動している場合は失敗する
- 詳細は `docs/spec/moves/サイコフィールド.md` / `docs/spec/fields/サイコフィールド.md` を参照。
  本計画書は技本体（ON_STATUS_HIT で地形を発動する処理）のみを扱う。地形自体の効果（先制技無効化・
  威力補正・カウントダウン等）は `docs/plan/fields/サイコフィールド.md`（存在しない場合は
  `docs/spec/fields/サイコフィールド.md`）を参照

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト、AttackContext） | `battle.terrain_manager.apply("サイコフィールド", 5)` を呼び地形を発動する |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「天気・フィールド: すでに同じ」が priority 100 として記載されている。同種ハンドラ `エレキフィールド_activate_terrain` / `あまごい_activate_weather` / `じゅうりょく_activate_global_field` が priority デフォルト（100）を使用しているため、同一パターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

```python
def サイコフィールド_activate_terrain(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """サイコフィールド: 地形をサイコフィールドにする。"""
    return HandlerReturn(value=battle.terrain_manager.apply("サイコフィールド", 5))
```

### `src/jpoke/data/moves/move_sa.py`

```python
"サイコフィールド": MoveData(
    type="エスパー",
    category="status",
    pp=12,
    target="field",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.サイコフィールド_activate_terrain,
        ),
    }
),
```

## 注意点・エッジケース

- `terrain_manager.apply` はすでに同じ地形が発動している場合に `False` を返す（技は失敗扱いになる）
- `エレキフィールド_activate_terrain` / `あまごい_activate_weather` / `じゅうりょく_activate_global_field` と同一パターン
- エレキフィールド・グラスフィールド・ミストフィールドと同時に発動することはできない（`terrain_manager` が `ExclusiveFieldManager` として排他管理する）
- グランドコートを持つポケモンが発動させた場合は継続ターンが8ターンに延長される（`ON_MODIFY_DURATION` 経由、`docs/spec/items/グランドコート.md` を参照）
