# 実装計画書: サイコフィールド

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 10 / 対象: 全体の場（target="field"）
- 5ターンの間、場の状態を「サイコフィールド」にする
- 地面にいるポケモンが相手の先制技を受けなくなる（味方には影響しない）
- エスパータイプの技の威力が 1.3 倍になる
- サイコシードを持つポケモンの「とくぼう」ランクが 1 段階上がる（地面外でも発動）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | グローバルフィールドを「サイコフィールド」に設定 |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「天候・フィールド: すでに同じ」が priority 100 として記載されている。同種ハンドラ `じゅうりょく_activate_global_field` が priority デフォルト（100）を使用しているため、同一パターンを採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「サ」は「さ」として処理。現在のファイルで `すりかえ_swap_items` の前、`さ` ブロックの最初に挿入する。具体的には `コットンガード_modify_attacker_stats` の後、`すりかえ_swap_items` の前。

```python
def サイコフィールド_activate_global_field(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """サイコフィールド: グローバルフィールドをサイコフィールドにする。"""
    return HandlerReturn(value=battle.global_manager.activate("サイコフィールド", 5))
```

### `src/jpoke/data/move.py`

既存の `"サイコフィールド"` エントリに handlers を追加する。

```python
"サイコフィールド": MoveData(
    type="エスパー",
    category="変化",
    pp=10,
    target="field",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.サイコフィールド_activate_global_field,
        ),
    }
),
```

## 注意点・エッジケース

- `global_manager.activate` はすでに同じフィールドが存在する場合に False を返すことが想定される。ハンドラは戻り値をそのまま `HandlerReturn(value=...)` で返すだけでよい
- `じゅうりょく_activate_global_field`・`エレキフィールド_activate_global_field`・`グラスフィールド_activate_global_field` と同一パターン
- サイコフィールドの「先制技無効」効果の本体は global_field ハンドラ側（`ON_TRY_MOVE_1` priority 100）で処理されており、技側のハンドラはフィールドを設定するのみでよい
