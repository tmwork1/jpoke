# クリアスモッグ 実装計画書

## 仕様要約

- `docs/spec/moves/クリアスモッグ.md` 参照
- どく特殊・威力50・命中必中・PP16（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）・対象1体（相手）
- ダメージを与えた相手の能力ランク変化を全て±0にリセットする
  （こうげき・ぼうぎょ・とくこう・とくぼう・すばやさ・命中率・回避率）
- リセットは攻撃を当てた対象のみに適用され、くろいきりのような場全体へのリセットではない
- リセットは「追加効果」ではないため、ちからずくで威力が上がらず、りんぷん・おんみつマントの
  影響も受けない
- クリアボディ等のランク低下防止特性・しろいきり状態を無視してリセットする
- あまのじゃく・たんじゅんの影響を受けない。まけんき・かちき・びんじょう・ものまねハーブ・
  しっとのほのお・みわくのボイス・うっぷんばらしといった「ランク変化検知」系の効果も誘発しない

## 実装方針

### ハンドラ構成（`handlers/move_attack.py`）

| イベント | Priority | 役割 |
|----------|----------|------|
| `Event.ON_DAMAGE_HIT` | 10 | 対象の能力ランクを±0にリセットする（`クリアスモッグ_reset_defender_rank`） |

### Priority 根拠

`docs/spec/turn.md` の「Event.ON_DAMAGE」セクション（実装上は `Event.ON_DAMAGE_HIT` に相当）に
「10 | クリアスモッグによるランクリセット」と明記されているため、これに従う。

`Event.ON_DAMAGE_HIT` は実ダメージが0（みがわりに阻まれた場合・ばけのかわ・こらえる等）のとき
発火しない設計だが、この技については turn.md 側で `Event.ON_DAMAGE`（`ON_DAMAGE_HIT`）が
明記済みのため、既存の設計判断（かいがらのすず・こうそくスピンのような「常に発火する
`Event.ON_HIT` を採用する」パターン）は適用せず、turn.md の記載通り `ON_DAMAGE_HIT` を採用する。
これにより、通常のダメージが発生するケースとみがわりに阻まれるケースは正しく再現できる
（みがわり吸収時は実ダメージ0のため `ON_DAMAGE_HIT` が発火せずリセットも起きない＝仕様通り）。
一方、ばけのかわ・こらえるで実ダメージが0になるケースでのリセットは本実装では発動しない
（`docs/spec/moves/クリアスモッグ.md` の「既知の未実装ニュアンス」を参照）。

### rank の書き換え方法

くろいきり（`くろいきり_reset_all_ranks`、`handlers/move_status.py`）と同じパターンで、
`battle.modify_stats` を経由せず `Pokemon.rank` を直接 `0` に書き換える。これにより
`ON_BEFORE_MODIFY_STAT` を経由するランク低下防止特性（クリアボディ等）・しろいきり状態・
あまのじゃく・たんじゅん・ランク変化検知系特性（まけんき等）の影響を受けずに済む。

対象は `ctx.defender` のみ（くろいきりは `battle.actives` 全体、クリアスモッグは単体）。

### subject_spec

`MoveHandler` のデフォルト `attacker:self`（`AttackContext` の攻撃フロー、`skip_subject_check=True`）。

## 実装箇所

### handlers/move_attack.py（五十音順: クリアスモッグは「く」行、くらいつくとクロスポイズンの間）

```python
def クリアスモッグ_reset_defender_rank(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """クリアスモッグの効果: ダメージを与えた相手の能力ランクを±0にリセットする。

    くろいきりと同様、しろいきり状態やクリアボディ等のランク低下防止特性を無視して
    リセットする（ON_BEFORE_MODIFY_STAT を経由しない直接リセット）。
    あまのじゃく・たんじゅんの影響も受けない。追加効果ではないため、ちからずくで
    威力は上がらず、りんぷん・おんみつマントの影響も受けない
    （ON_MODIFY_SECONDARY_CHANCE を経由しないため自然に満たされる）。
    """
    defender = ctx.defender
    changed = {s: v for s, v in defender.rank.items() if v != 0}
    if changed:
        for s in changed:
            defender.rank[s] = 0
        battle.add_event_log(
            defender, LogCode.STAT_CHANGED,
            payload=StatChangePayload(
                stats={s: -v for s, v in changed.items()}, display_reason="クリアスモッグ"
            ),
        )
    return HandlerReturn(value=value)
```

## リーサル計算

- `docs/progress/move.md` のクリアスモッグ行「リーサル実装」列は `-`（新規実装対象）。
- クリアスモッグ自体は `hp_dist`（HP分布）を変化させないが、防御側の能力ランクを
  リセットすることで、致死率計算が追跡している後続ターンのダメージ計算に影響する
  （`アシッドボム_reduce_spd` 等と同様、`ctx.defender.rank` を直接書き換えることで
  以降のダメージ計算に反映される）。
- `handlers/lethal.py` に `クリアスモッグ_reset_defender_rank(battle, ctx, hp_dist)` を
  新規実装し、`ctx.defender.rank` の全キーを `0` にリセットして `hp_dist` をそのまま返す。
- `data/moves/move_ka.py` のクリアスモッグ `MoveData` に
  `lethal_handlers={LethalEvent.ON_HIT: LethalHandler(l.クリアスモッグ_reset_defender_rank)}`
  を追加する（`アシッドボム` と同じ `LethalEvent.ON_HIT` を採用）。

## 注意点・エッジケース（review-test 向け）

1. **PP修正**: 15→16（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）。
2. **ランクリセット効果の追加**: 旧実装は `handlers={}` で追加効果なし扱いだったが、
   Wikiの説明文どおり「のうりょく へんかを もとに もどす」効果を持つため、これを実装する。
3. **みがわり**: `ON_DAMAGE_HIT` は実ダメージ0（みがわり肩代わり）のとき発火しないため、
   追加のガードなしでみがわりブロックが自然に成立する。すりぬけ特性で貫通した場合は
   実ダメージが発生するため通常通りリセットされる。
4. **まもる**: `_execute_hit` に到達しないため `ON_DAMAGE_HIT` 自体が発火せず、
   自然にリセットされない。
5. **クリアボディ等のランク低下防止特性・しろいきり**: `battle.modify_stats` を経由しない
   直接書き換えのため、これらの影響を受けずにリセットが成立する。
6. **ちからずく**: `secondary_effect` フラグを付与しないため、ちからずくの威力上昇・
   追加効果無効化のいずれの対象にもならない（技の効果は常に発動する）。
7. **場全体ではなく対象のみ**: くろいきり（`battle.actives` 全体）と異なり、
   `ctx.defender` のみを対象とする。
