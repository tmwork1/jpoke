# 計画書: ねをはる

## 仕様要約

- くさタイプ / 変化 / 命中なし / PP20 / 対象: 自分
- 使用後、毎ターン最大HPの1/16ずつ回復する
- 交代できなくなる（ねをはる状態）
- 自分がひこうタイプまたは特性ふゆうの場合、じめんタイプの技が当たるようになる
- すでにねをはる状態の場合は失敗（`ON_TRY_MOVE_2` priority 130 でフレームワーク側が処理）
- ターン終了時に回復処理が行われる（`ON_TURN_END` priority 70）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | 自分にねをはる状態付与 |

## Priority 根拠

`docs/spec/turn.md` の `ON_TURN_END` priority 70 に `ねをはる` が記載されており、回復はturend側が処理する。`ON_TRY_MOVE_2` priority 130 に `ねをはる: すでにねをはる状態` が記載されており、失敗チェックはフレームワーク側が処理する。ハンドラ本体は `ON_STATUS_HIT` priority 100 のパターンに従う（`アクアリング_apply_volatile_to_attacker` と同形式）。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `なみだめ` の後、`ねばねばネット` の前（五十音順）。

```python
def ねをはる_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ねをはるの効果: 自分をねをはる状態にする。"""
    return apply_volatile_to_attacker(battle, ctx, value, volatile="ねをはる")
```

### data/move.py への更新

`"ねをはる"` エントリのハンドラを追加する。`target="self"` を必ず指定する
（欠落すると `Move.is_blocked_by_protect` / `Move.is_reflectable` が
`target == "foe"` を基準に判定するため、まもる・マジックコートに誤反応する）。

```python
    "ねをはる": MoveData(
        type="くさ",
        category="status",
        pp=20,
        target="self",
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.ねをはる_apply,
            ),
        }
    ),
```

## 注意点・エッジケース

- すでにねをはる状態の失敗チェックは `ON_TRY_MOVE_2` priority 130 でフレームワーク側が処理するためハンドラ内ガード不要
- 毎ターンの回復処理（1/16 HP）は `ON_TURN_END` priority 70 で volatile 側が処理
- 交代不可処理は `ON_APPLY_MOVE` priority 30 の `ねをはる状態による無効化`（ふきとばし等に対する抵抗）もフレームワーク側が処理
- ひこうタイプ・ふゆう特性のじめん耐性剥奪も volatile 側が担当
- count 引数は省略（アクアリングと同様に永続型）
- `MoveData` に `target="self"` を指定しないと、デフォルトの `target="foe"` により
  まもる・マジックコートに誤って反応してしまう（`docs/champions/move_list.txt` の
  「守×」に反するバグとなるため必須）
