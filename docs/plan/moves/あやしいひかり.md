# 実装計画書: あやしいひかり

## 仕様

- タイプ: ゴースト / カテゴリ: 変化 / PP: 10 / 命中率: 100
- 効果: 相手を**こんらん**（揮発性状態）にする（2〜5ターン）
- 無効条件:
  - 特性「マイペース」 → `マイペース_prevent_volatile` が `ON_BEFORE_APPLY_VOLATILE` で自動処理
  - 「ミストフィールド」（接地ポケモン） → `ミストフィールド_prevent_confusion` が自動処理
  - すでにこんらん → `volatile_manager.apply` が重複付与を自動拒否

## イベント・Priority

| イベント | Priority | subject_spec | 役割 |
|----------|----------|-------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（MoveHandler デフォルト） | こんらん付与（count=2〜5 ランダム） |

**根拠**: `ON_STATUS_HIT` は `turn.md` に独立エントリなし。`あくまのキッス`・`でんじは` と同パターン (priority=100 デフォルト)。

## 実装

### handlers/move_status.py

配置位置: `あさのひざし_heal_self` の直後、`あまいかおり_modify_defender_stats` の直前（五十音順）

```python
def あやしいひかり_apply_volatile_to_defender(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """あやしいひかりの効果: 相手をこんらん状態にする（2〜5ターン）。"""
    count = battle.random.randint(2, 5)
    return HandlerReturn(value=battle.volatile_manager.apply(
        ctx.defender, "こんらん", count=count, source=ctx.attacker, ctx=ctx
    ))
```

`apply_volatile_to_defender` ヘルパーはカウントを静的引数として受け取るため、ランダム生成が必要なこんらんには専用関数を実装する。

### data/move.py

既存の `"あやしいひかり"` エントリに `ON_STATUS_HIT` ハンドラを追加:

```python
"あやしいひかり": MoveData(
    type="ゴースト",
    category="変化",
    pp=10,
    accuracy=100,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.あやしいひかり_apply_volatile_to_defender,
        ),
    }
),
```

## 注意事項

- `VolatileName` の `"こんらん"` は登録済み（`utils/type_defs.py`）
- こんらん揮発性状態の実装（`こんらん_try_action` 等）も既存済み
- かがくへんかガス/かたやぶり無効化チェックは書かない
