# 実装計画書: あやしいひかり

## 仕様

- タイプ: ゴースト / カテゴリ: 変化 / PP: 12（`src/jpoke/data/ps-champ-ja/moves.json` 準拠） / 命中率: 100
- 効果: 相手を**こんらん**（揮発性状態）にする（2〜5ターン）
- 無効条件:
  - 特性「マイペース」 → `マイペース_prevent_volatile` が `ON_BEFORE_APPLY_VOLATILE` で自動処理
  - 「ミストフィールド」（接地ポケモン） → `ミストフィールド_prevent_confusion` が自動処理
  - すでにこんらん → `volatile_manager.apply` が重複付与を自動拒否

詳細は `docs/spec/moves/あやしいひかり.md` を参照。

## イベント・Priority

| イベント | Priority | subject_spec | 役割 |
|----------|----------|-------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（MoveHandler デフォルト） | こんらん付与 |

**根拠**: `ON_STATUS_HIT` は `turn.md` に独立エントリなし。`でんじは`・`あくまのキッス` と同パターン (priority=100 デフォルト)。

## 実装

### handlers/move_status.py

配置位置: `あさのひざし_heal_self` の直後、`あまいかおり_modify_defender_stats` の直前（五十音順）

```python
def あやしいひかり_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """あやしいひかりの効果: 相手をこんらん状態にする。"""
    return HandlerReturn(value=battle.volatile_manager.apply_confusion(
        ctx.defender, source=ctx.attacker
    ))
```

`volatile_manager.apply_confusion` はカウント（2〜5のランダム）の抽選を内部で行うため、
`いばる`・`おだてる` のこんらん付与部分と同じヘルパーをそのまま利用する。

### data/moves/move_a.py

既存の `"あやしいひかり"` エントリに `ON_STATUS_HIT` ハンドラを追加:

```python
"あやしいひかり": MoveData(
    type="ゴースト",
    category="status",
    pp=12,
    accuracy=100,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.あやしいひかり_apply,
        ),
    }
),
```

## 注意事項

- `VolatileName` の `"こんらん"` は登録済み（`docs/spec/volatiles/こんらん.md`）
- こんらん揮発性状態の実装（`こんらん_try_action` 等）も既存済み
- かがくへんかガス/かたやぶり無効化チェックは書かない
