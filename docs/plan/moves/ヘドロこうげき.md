# ヘドロこうげき 実装計画書

## 仕様要約（docs/spec/moves/ヘドロこうげき.md 参照）

* タイプ: どく / 分類: 特殊 / 威力: 65 / 命中: 100 / PP: 20
* 対象: 相手1体 / 優先度: 0 / 接触なし（直接攻撃ではない）
* 効果: 追加効果として、30%の確率で相手を『どく』状態にする。

## 現状の実装状況

`src/jpoke/data/moves/move_ha.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。
以下のとおり仕様と一致していることを確認した（修正不要）。

```python
"ヘドロこうげき": MoveData(
    type="どく",
    category="special",
    pp=20,
    power=65,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ヘドロこうげき_apply_poison_to_defender,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 30%の確率で相手を『どく』状態にする |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def ヘドロこうげき_apply_poison_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="どく", chance=0.3)
```

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を
経由するため、ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は、汎用ハンドラ側で
処理される。本技側に個別のガードは不要。

## Priority 根拠

ダメージを与えた技の追加効果（状態異常付与）は既存の同種ハンドラ（どくばり・どくばりセンボン・
ヘドロウェーブ・ヘドロばくだん 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）の
パターンに従う。個別の priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` により、ちからずく所持時は威力1.3倍・どく付与無効化になる。
     既存の ちからずく テストファイル側で網羅されていれば個別追加は不要。
2. **どく状態異常の重複判定**
   - 対象が既に状態異常（`ailment`）を持つ場合、`ailment_manager.apply` 側で重複付与されない
     ことを既存の共通実装が保証している（本技固有のガードは不要）。
3. **世代差**
   - 第一世代のみ追加効果発動率40%だったが、それ以降30%に変更されている。
     本プロジェクトは現行仕様（30%）のみ対象。
4. **PPについて**
   - `src/jpoke/data/ps-champ-ja/moves.json` には本技の記載がない。Gen9本家PP(20)が
     Champions側のPP4段階(8/12/16/20)の最大値と一致するため変換の必要がなく、
     実装値(20)のまま維持する。
5. **テスト方針**
   - `secondary_chance` を1に固定し、どく付与を確認するテストが既存
     （`test_ヘドロこうげき_どくが発動する`）。
