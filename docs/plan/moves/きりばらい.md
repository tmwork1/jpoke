# 実装計画書: きりばらい

参照仕様書: docs/spec/moves/きりばらい.md

## 実装方針

きりばらいは「対象の回避率を1段階下げる」効果と「場の効果（壁・設置技・フィールド）を
解除する」効果を併せ持つ変化技。両者は独立した効果であり、みがわりやしろいきりによって
回避率変化が防がれても場の効果解除は独立して発動する（一次情報:
docs/wiki/moves/きりばらい.html 技の仕様節）。

### データ定義（`src/jpoke/data/moves/move_ka.py`）

* `type="ひこう"`
* `category="status"`
* `pp=16`（`src/jpoke/data/ps-champ-ja/moves.json` に基づく。Gen9本家は15だが
  チャンピオンズでは全技PPが8/12/16/20に統一されているため16とする）
* `target`: 指定なし（デフォルトの `"foe"` を使用）
* `flags={"bypass_substitute"}`
  * みがわりに対して技自体（ON_BEFORE_APPLY_MOVE の全体ブロック）は無効化されない
    ようにするためのフラグ。回避率変化そのものは後述のハンドラ内で明示的にみがわりを
    チェックして防ぐため、bypass_substitute を付与しても回避率変化が誤って成功する
    ことはない。
* `handlers={Event.ON_STATUS_HIT: h.MoveHandler(hs.きりばらい_defog)}`
  * priority は指定なし（デフォルト100）。`docs/spec/turn.md` に `ON_STATUS_HIT` の
    掲載はないため、既存の変化技ハンドラ（`Event.ON_STATUS_HIT` を使う他の技）と
    同様にデフォルト優先度を用いる。

### ハンドラ実装（`src/jpoke/handlers/move_status.py` の `きりばらい_defog`）

処理順序が重要。一次情報より「しろいきりは回避率変化を防いだ後に、しろいきり状態を
含めた場の状態が解除される」ため、**回避率変化を場の効果解除より先に行う**。

1. `ctx.defender.has_volatile("みがわり")` が False の場合のみ
   `battle.modify_stats(ctx.defender, {"evasion": -1}, source=ctx.attacker)` を実行する。
   みがわり以外の失敗要因（しろいきり・クリアボディ等の特性・クリアチャーム）は
   `battle.modify_stats` 内部で発火する `ON_BEFORE_MODIFY_STAT` の既存ハンドラ
   （`しろいきり_prevent_stat_drop` 等）が処理するため、ここでの追加実装は不要。
2. 対象側の壁系（ひかりのかべ・リフレクター・オーロラベール・しんぴのまもり・
   しろいきり）を `battle.get_side(ctx.defender).deactivate(...)` で解除する。
3. 両陣営のトラップ（まきびし・どくびし・ステルスロック・ねばねばネット）を
   `battle.side_managers` を走査して解除する。
4. `battle.terrain_manager.remove()` でフィールドを解除する。
5. いずれかの効果が実際に変化した場合（回避率変化 or 壁/トラップ/フィールドの
   いずれかが解除された場合）に `HandlerReturn(value=True)` を返す。
   何も変化しなかった場合は `HandlerReturn(value=False)`（技失敗）を返す。

## 相互作用の確認

* **まもる**: `flags` に `"protect"`/`"unprotectable"` を付与しないため、通常の
  `ON_BEFORE_APPLY_MOVE` の連続使用判定に従い、まもるで防がれる（一次情報通り）。
* **マジックコート**: `flags` に `"unreflectable"` を付与しないため、
  `model/move.py` の既定ロジックにより反射対象となる（一次情報通り）。
* **みがわり**: `bypass_substitute` フラグにより技自体はブロックされないが、
  ハンドラ内で明示的にチェックするため回避率変化のみ防がれる。
* **すりぬけ**: 既存の `すりぬけ_bypass_substitute`（`ON_CHECK_HIT_SUBSTITUTE`）が
  汎用的に処理するため追加実装は不要。
* **ミラーアーマー**: `battle.modify_stats` が発火する `ON_BEFORE_MODIFY_STAT` を
  既存の `ミラーアーマー_reflect_stat_drop` が処理し、回避率変化を跳ね返す。
  場の効果解除は対象側のまま解除されるため、一次情報の記載と整合する。

## リーサル計算

ダメージを与えない補助効果技のため、`handlers/lethal.py` への専用ハンドラ追加は
不要（n/a）。

## テスト方針

`tests/moves_status/test_move_ka.py` に以下を追加・拡張する。

* 対象の回避率が1段階下がることを確認するテスト（既存）
* 対象側の壁系（ひかりのかべ・リフレクター・オーロラベール・しんぴのまもり）が
  解除されることを確認するテスト
* 使用者側の壁は解除されないことを確認するテスト（既存）
* 両陣営の設置技が除去されることを確認するテスト（既存）
* フィールド（エレキ・グラス・サイコ・ミスト）が解除されることを確認するテスト
* しろいきりが回避率変化を防いだ上で、しろいきり自体は解除されることを確認する
  テスト
* みがわり状態の相手には回避率変化のみ防がれ、場の効果解除（壁・設置技・
  フィールド）は貫通することを確認するテスト
