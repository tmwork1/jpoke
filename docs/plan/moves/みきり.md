# 実装計画書: みきり

## 仕様要約

- タイプ: かくとう / カテゴリ: 変化 / PP: 8（`docs/champions/move_list.txt` 866行目） / 命中: なし
  （必中） / 対象: 自分 / 優先度: +4
- タイプとPPを除けば `まもる` と全く同じ効果を持つ（`docs/spec/moves/みきり.md` 参照）。
  そのターン自分を揮発性状態「まもる」（`docs/spec/volatiles/まもる.md`）にし、受ける技を無効化する
- `まもる` 系統技（`protect` フラグを持つ技）と連続使用成功率を共有する。直前ターンに `protect`
  フラグ技を成功させていた場合、今ターンの `みきり` は失敗する

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_TRY_MOVE_1` には「30 | まもる系: 連続使用により失敗」という
記載があるが、実装では `まもる` 本体を含む protect フラグ技すべて（まもる・キングシールド・
トーチカ・ニードルガード・スレッドトラップ・ふういん除く全8技）が例外なく `ON_TRY_MOVE_2` の
デフォルト priority（100）で `まもる系_連続使用失敗チェック` を登録している。自分対象の変化技
（`target="self"`）はタイプ相性判定の影響を受けないため、`ON_TRY_MOVE_1` / `ON_TRY_MOVE_2` の
どちらで実施しても挙動に差はない。この記述はコードベース全体で既存の統一パターンであり、
`みきり` 単独でこれを変更すると `まもる` 等の兄弟技と priority が食い違ってしまうため、本レビューの
スコープ外として既存パターンを踏襲する。

`Event.ON_STATUS_HIT` は自己付与型の変化技で使われる標準パターン（`まもる_apply` 等と同一）。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

既存の `みきり_apply` はそのまま（変更なし）:

```python
def みきり_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_attacker(battle, ctx, value, volatile="まもる")
```

`まもる` volatile をそのまま付与することで、`まもる` と全く同じ防御効果・解除条件
（ターン終了時解除、フェイント等での解除、変化技の無効化）を実現する。

### `src/jpoke/data/moves/move_ma.py`

```python
"みきり": MoveData(
    type="かくとう",
    category="status",
    pp=8,
    priority=4,
    target="self",
    flags={"protect"},
    handlers={
        Event.ON_TRY_MOVE_2: h.MoveHandler(
            hs.まもる系_連続使用失敗チェック,
        ),
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.みきり_apply,
        ),
    }
),
```

## 注意点・エッジケース

- `みきり_apply` は `まもる_apply` と同じく `volatile="まもる"` を付与するため、`まもる` と `みきり`
  を同一ターンには使えない（`volatile_manager.apply()` は既に付与済みの場合 False を返す）ものの、
  そもそも1ターンに1回しか行動しないため実質問題にならない
- `まもる系_連続使用失敗チェック` は `mon.executed_move.has_flag("protect")` を見るため、直前ターンに
  `まもる` を使って成功していた場合でも次ターンの `みきり` は失敗する（技名ではなく `protect` フラグ
  で判定するため、異なる protect 系技同士でも連続使用失敗が共有される）
- Zワザによる付加効果（回避率上昇）は対象外（README.md「対象範囲」によりZワザ非対応）
- 確率的な 1/3 逓減による連続使用失敗率上昇は未実装（`docs/spec/volatiles/まもる.md` に既知の
  スコープ外事項として明記済み）
