## 実装箇所

### `src/jpoke/handlers/move_status.py`

挿入位置: `なきごえ_modify_defender_stats` の後（「なまける」は「なきごえ」の後、「なみだめ」の前）

```python
def なまける_heal_self(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """なまける: 最大HPの1/2を回復する。"""
    mon = ctx.attacker
    if mon.hp == mon.max_hp:
        return HandlerReturn(value=False, stop_event=True)
    battle.modify_hp(mon, r=1/2)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

`"なまける"` エントリ（行 5602）に `handlers` を追加する:

## 注意点・エッジケース

- `じこさいせい` と全く同じ処理。`labels=["heal"]` はエントリに既記載のため変更不要
- HP満タン時の失敗チェック（`mon.hp == mon.max_hp` で早期リターン）は `turn.md` の `ON_APPLY_MOVE` Priority 100「自己回復技: HP満タン」で管理される場合もあるが、既存の `じこさいせい_heal_self` がハンドラ内でもチェックしているため同パターンに従う
- `かいふくふうじ` による使用不可チェックは `turn.md` Priority 100「かいふくふうじで技が出せない」でフレームワーク側が処理するため、ハンドラ内不要
