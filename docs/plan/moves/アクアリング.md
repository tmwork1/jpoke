# アクアリング 実装計画書

## 仕様要約

- 使用者自身に **アクアリング** 揮発性状態を付与する変化技
- 命中判定なし（必中、自分対象）
- 効果: ターン終了時（ON_TURN_END priority=70）に最大HPの1/16を回復
- 失敗条件: すでにアクアリング状態の場合（`volatile_manager.apply` が内部で重複チェック済み）

## ハンドラ構成

| イベント | Priority | subject_spec | 役割 |
|----------|----------|-------------|------|
| `ON_STATUS_HIT` | 100 (デフォルト) | `attacker:self` | 使用者自身にアクアリング揮発性状態を付与 |

### Priority 根拠

`ON_STATUS_HIT` priority=100: MoveHandlerデフォルト値。turn.md には ON_STATUS_HIT の個別 priority は掲載されていないが、既存の自己状態付与技（まもる等）はすべてデフォルト（100）を使用しており、これに倣う。

## 実装箇所

```python
def アクアリング_apply_volatile_to_attacker(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:

```

### 2. `data/move.py`（既存の `"アクアリング"` MoveData エントリを更新）

```python
"アクアリング": MoveData(
    type="みず",
```

## 注意点

1. **重複付与防止**: `volatile_manager.apply` が内部でチェックするため、ハンドラ側に追加ガード不要
2. **かいふくふうじ**: `かいふくふうじ_block_heal` ハンドラが担当するため、アクアリング側は不要
3. **target="self"**: MoveData に明示する
