# せいなるつるぎ 実装計画書

## 仕様要約（docs/spec/moves/せいなるつるぎ.md 参照）

* タイプ: かくとう / 分類: 物理 / 威力: 90 / 命中: 100 / PP: 15
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 相手の『ぼうぎょ』『回避率』のランク補正（正負問わず）を無視してダメージ・命中判定を行う。
  自分の『こうげき』『命中率』のランク補正は無視しない。
* 「切る技」の1つ（`きれあじ` 特性で威力1.5倍）。

## 現状の実装状況

`src/jpoke/data/move.py` に既に登録済みで、`Event.ON_CALC_DEF_RANK_MODIFIER` により
相手の『ぼうぎょ』ランク補正を無視するハンドラ（`せいなるつるぎ_ignore_def_rank`）は
実装済み（DDラリアットと同一パターン）。

ただし、相手の『回避率』ランク補正を無視する処理が欠落していた（**バグ**）。
一次情報（docs/wiki/moves/せいなるつるぎ.html「技の仕様」節）に
「相手の防御/回避率のランク補正を無視する」と明記されており、ダメージ計算だけでなく
命中判定でも相手の回避率ランクを無視する必要がある。

`src/jpoke/data/ability.py` の `するどいめ`（Keen Eye）特性がすでに
`Event.ON_GET_STAT_RANK` を用いて同様の「相手の回避率ランクを無視する」処理
（`するどいめ_ignore_evasion`）を実装済みであり、このパターンをそのまま流用する。

`Event.ON_GET_STAT_RANK` は `core/move_executor.py` の `_check_hit()` 内で

```python
ranks = {
    "accuracy": attacker.rank["accuracy"],
    "evasion": defender.rank["evasion"]
}
modified_rank = self._events.emit(Event.ON_GET_STAT_RANK, ctx, ranks)
rank_modifier = hit_rank_modifier(modified_rank["accuracy"], modified_rank["evasion"])
```

の形で発火し、命中率のランク補正計算に使われる。ハンドラで `value["evasion"] = 0` を
返すことで、相手の回避率ランクを常に0（無視）として扱える。

## ハンドラ構成

| イベント | Priority | subject_spec | 役割 |
|---------|---------|-------------|------|
| ON_CALC_DEF_RANK_MODIFIER | デフォルト | attacker:self | 相手の防御ランク補正を1.0(=ランク0相当)に固定（実装済み） |
| ON_GET_STAT_RANK | デフォルト | attacker:self | 相手の回避率ランクを0に固定して命中判定する（新規追加） |

## 実装コード

### handlers/move_attack.py

```python
def せいなるつるぎ_ignore_def_rank(battle: Battle, ctx: AttackContext, value: float) -> HandlerReturn:
    """せいなるつるぎ: 相手のランク変化を無視してダメージ計算する（防御ランク補正を1.0に固定）。"""
    return HandlerReturn(value=1)


def せいなるつるぎ_ignore_evasion(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """せいなるつるぎ: 相手の回避率ランク変化を無視して命中判定する。"""
    value["evasion"] = 0
    return HandlerReturn(value=value)
```

### data/move.py

```python
"せいなるつるぎ": MoveData(
    type="かくとう",
    category="physical",
    pp=15,
    power=90,
    accuracy=100,
    flags={"contact", "slash"},
    handlers={
        Event.ON_CALC_DEF_RANK_MODIFIER: h.MoveHandler(
            ha.せいなるつるぎ_ignore_def_rank,
            subject_spec="attacker:self",
        ),
        Event.ON_GET_STAT_RANK: h.MoveHandler(
            ha.せいなるつるぎ_ignore_evasion,
            subject_spec="attacker:self",
        ),
    }
),
```

## Priority 根拠

`docs/spec/turn.md` には `ON_CALC_DEF_RANK_MODIFIER` / `ON_GET_STAT_RANK` は
ダメージ計算・命中判定内部イベントとして掲載されていない。既存の同種ハンドラ
（`するどいめ_ignore_evasion` が `ON_GET_STAT_RANK` にデフォルト priority で登録、
`せいなるつるぎ_ignore_def_rank` が `ON_CALC_DEF_RANK_MODIFIER` にデフォルト priority で登録）
を根拠に、新規追加分もデフォルト priority（`MoveHandler` に priority 未指定）とする。

## subject_spec 根拠

`ON_GET_STAT_RANK` は `AttackContext` で発火するイベントであり、「この技の攻撃者」が
自分と一致するハンドラのみ発動させるため `attacker:self` を指定する。
`するどいめ`（特性ハンドラ）も同様に `attacker:self` を指定している
（特性所持者が攻撃側の場合のみ発動）。

## 注意点・エッジケース（review-test 向け）

1. **回避率ランクの正負どちらも無視**
   - 相手の回避率ランクが上昇していても（命中しにくくなる効果を無視）、下降していても
     （命中しやすくなる効果を無視）、通常時と命中率が変わらないことを確認する。
2. **自分の命中率ランクは無視しない**
   - 攻撃側の命中率ランクが変化している場合は、通常通り命中判定に反映されることを確認する
     （`ON_GET_STAT_RANK` では `ranks["accuracy"]` を変更しない）。
3. **ダメージ計算側の防御ランク無視（既存実装）との独立性**
   - `ON_CALC_DEF_RANK_MODIFIER`（ダメージ計算）と `ON_GET_STAT_RANK`（命中判定）は
     別イベントであるため、それぞれ個別にハンドラを登録する必要がある
     （どちらか一方だけでは仕様を満たさない）。
4. **するどいめ・ONE_GET_STAT_RANKとの多重発火**
   - 攻撃側がするどいめを持っている場合、せいなるつるぎ由来のハンドラとするどいめ由来の
     ハンドラが両方発火するが、どちらも `value["evasion"] = 0` を設定するだけなので競合しない。

## テスト方針

- 相手の回避率ランクが上昇していてもせいなるつるぎが必ず命中率通りに命中判定されること
  （`battle.move_executor.accuracy == move.accuracy` で確認、するどいめのテストと同様の手法）。
- 既存の防御ランク無視テスト（`test_せいなるつるぎ_相手のぼうぎょランクを無視する` /
  `test_せいなるつるぎ_相手のぼうぎょランク低下も無視する`）は変更なし。
