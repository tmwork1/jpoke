# やきつくす 実装計画

更新日: 2026-07-07

## スコープ

- 仕様書: `docs/spec/moves/やきつくす.md`
- 実装状態: 実装済み（本レビューで焼却条件・みがわり判定を修正）

## 効果概要

- 命中時、相手が「きのみ」または「ノーマルジュエル」を持っている場合、それを焼却して消滅させる。
- ちからずくの対象外（追加効果ではない）。`flags` に `secondary_effect` を含めないことで、
  威力上昇・追加効果無効化のいずれも自動的に適用されなくなる（既存の共通処理を流用）。
- タイプ半減きのみ（オッカのみ・ナゾのみ・ソクノのみ）は本ハンドラより先に発火する
  `Event.ON_CALC_DAMAGE_MODIFIER` で消費されるため、既に消費済みなら本ハンドラの焼却対象がない
  （個別対応不要、既存の共通処理のみで成立）。
- HP回復ピンチきのみ（オボンのみ等）・レンブのみ・タラプのみ・チイラのみは、ダメージがHPへ
  反映される時点の `Event.ON_HP_CHANGED` で発動判定される。本ハンドラを `modify_hp` 呼び出しより
  前の `Event.ON_MODIFY_MOVE_DAMAGE` に置くことで、これらのきのみが発動する前に焼却できる
  （詳細は「実装方針・イベント選定の根拠」参照）。
- みがわり状態に防がれた場合は実体の持ち物を燃やせない。`ctx.substitute_damage > 0` で判定する
  （`むしくい_steal_and_use_berry` と同じパターン）。
- 特性ねんちゃくの相手には燃やせない。`battle.item_manager.remove_item` が内包する
  `can_change_item`（`Event.ON_CHECK_ITEM_CHANGE`）判定により自動的に処理される（個別対応不要）。
- 特性マジシャンとの順序（本ハンドラが先に発火し、燃やす対象にならない持ち物だけをマジシャンが
  奪える）は、マジシャンの `Event.ON_DAMAGE_HIT`（`ON_MODIFY_MOVE_DAMAGE` より後）への登録により
  自動的に成立する（個別対応不要）。

## 実装方針・イベント選定の根拠

- `docs/spec/turn.md` の `Event.ON_DAMAGE`（実装上は `Event.ON_DAMAGE_HIT`）優先度表には
  「30 やきつくすできのみ消失」という記載があるが、本実装ではあえて
  `Event.ON_MODIFY_MOVE_DAMAGE`（`subject_spec="attacker:self"`, 既定priority）を採用する。
  - 理由: `Event.ON_DAMAGE_HIT` は `battle.modify_hp` によるダメージ反映（この内部で
    `Event.ON_HP_CHANGED` が同期的に発火し、オボンのみ等のHP回復ピンチきのみが発動・消費される）
    より後に発火するイベントである（`src/jpoke/core/move_executor.py` の `_execute_hit` 参照）。
    `Event.ON_DAMAGE_HIT` で焼却すると、オボンのみ等が先に発動・消費されてしまい、
    「回復・能力変化は起こらず燃やされてなくなる」という仕様（`docs/spec/items/_HP回復ピンチきのみ.md`
    等）を満たせない。
  - `Event.ON_MODIFY_MOVE_DAMAGE` は `roll_damage`（ダメージ計算・オッカのみ等の消費を含む）の
    直後、`modify_hp` によるHP反映より前に発火するため、タイプ半減きのみの消費順序を保ったまま、
    HP回復ピンチきのみの発動を防げる。
  - 実際に `python -c` で `やきつくす` → オボンのみ保持ポケモンへの命中を検証したところ、
    「オボンのみを失った」のログのみが出力され、「オボンのみが発動した」は出力されずHPも
    回復しないことを確認した（`Event.ON_MODIFY_MOVE_DAMAGE` 採用時の挙動）。
  - なお、はたきおとす（`Event.ON_DAMAGE_HIT` 採用）で同様の検証を行うと、オボンのみが
    先に発動してHPが回復してしまう（`docs/spec/moves/はたきおとす.md` の
    「はたきおとすの効果が優先される（もちもの効果を発動させずに奪う）」という記載と
    実装が食い違っている）。これは本タスクのスコープ外（はたきおとす自体の既存レビュー範囲）の
    ため、本計画では触れるのみで修正は行わない。
  - この既定は既存の `docs/plan/items/オッカのみ.md`・`docs/plan/items/ソクノのみ.md` 等の
    複数の計画書ですでに前提として明記されている。

## 実装

- `src/jpoke/data/moves/move_ya.py` の `"やきつくす"` に `Event.ON_MODIFY_MOVE_DAMAGE` で
  `handlers/move_attack.py` の `やきつくす_burn_item` を登録する（`subject_spec="attacker:self"`）。
- `やきつくす_burn_item(battle, ctx, value)`:
  1. `ctx.substitute_damage > 0` の場合は何もしない（みがわりに防がれた）。
  2. `ctx.defender.item` が「きのみ」（`is_berry()`）または「ノーマルジュエル」であれば
     `battle.item_manager.remove_item(target=ctx.defender, source=ctx.attacker)` を呼ぶ。
  3. `value`（ダメージ量）はそのまま通す。

## 登録

- `src/jpoke/data/moves/move_ya.py` の `"やきつくす"` に `handlers`（`ON_MODIFY_MOVE_DAMAGE`）を
  登録済み（イベント選定の根拠をコメントとして残す）。

## テスト

- `tests/moves_attack/test_move_yarawa.py` に以下を追加する。
  - `test_やきつくす_相手のきのみを燃やす`: きのみを保有する相手に命中させ、`has_item()` が
    `False` になることを確認する。
  - `test_やきつくす_ノーマルジュエルを燃やす`: ノーマルジュエルを保有する相手に命中させ、
    `has_item()` が `False` になることを確認する。
  - `test_やきつくす_きのみでもジュエルでもない道具は燃やされない`: たべのこし等を保有する
    相手に命中させても `has_item("たべのこし")` が `True` のままであることを確認する。
  - `test_やきつくす_HP回復ピンチきのみは発動せず燃やされる`: オボンのみ保有の相手に命中させ、
    HPが回復せず（ダメージ分だけ減少）、アイテムが失われることを確認する。
  - `test_やきつくす_みがわりに防がれた場合はきのみを燃やせない`: みがわり状態の相手に命中させ、
    アイテムが維持されることを確認する。
  - `test_やきつくす_ねんちゃく持ちのきのみは燃やせない`: ねんちゃく特性の相手に命中させ、
    アイテムが維持されることを確認する。
  - `test_やきつくす_相手がアイテムを持っていない場合は何も起きない`: アイテムなしの相手に
    命中させ、エラーなく通常通りダメージを与えることを確認する。

## 今後の対応

- しゅうかく・ものひろい・はんすう・リサイクルとの相互作用（本技で失った道具が復活・再取得の
  対象にならないこと）は、各特性・技側の実装がまだレビュー未了（`docs/progress/ability.md` で
  しゅうかく=テスト未了、ものひろい=実装未了、はんすうは実装保留）のため、それぞれの
  レビューループで別途確認・テストする。本タスクでは対応しない。
