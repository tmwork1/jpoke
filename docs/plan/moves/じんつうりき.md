# 計画書: じんつうりき

## 仕様要約（docs/spec/moves/じんつうりき.md 参照）

- エスパータイプ / 特殊 / 威力80 / 命中100 / PP20 /
  対象: 1体選択 / 優先度0 / 接触なし
- 追加効果として、命中時に10%の確率で相手をひるませる。
- 追加効果に分類されるため、ちからずく（威力1.3倍・効果無効化）の対象技になる。

## 既存実装との差分

既存実装（`data/move.py` の `"じんつうりき"` エントリ、`handlers/move_attack.py` の
`じんつうりき_apply_flinch`）は仕様と一致しており、差分なし。

## ハンドラ構成

### handlers/move_attack.py: `じんつうりき_apply_flinch`（既存・変更なし）

```python
def じんつうりき_apply_flinch(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_defender(battle, ctx, value, volatile="ひるみ", chance=0.1)
```

- `apply_volatile_to_defender`（`handlers/move.py`）は内部で `battle.resolve_secondary_chance` を
  呼び出すため、`Event.ON_MODIFY_SECONDARY_CHANCE` 経由でちからずく（確率0化・威力1.3倍）の
  ハンドラが自動的に適用される（`secondary_effect` フラグが前提）。
- イベント: `Event.ON_DAMAGE_HIT`（ダメージが実際に適用された後に発動。
  docs/spec/turn.md の `## Event.ON_DAMAGE` 参照）。
  ダメージが発生しなかった場合（無効化・まもる等）はイベント自体が発火しないため、
  追加効果も発生しない仕様を自然に満たす。
- 優先度: デフォルト100（`MoveHandler` のデフォルト値）。
- subject_spec: `MoveHandler` デフォルト（`attacker:self`）のまま。

### data/move.py（既存・変更なし）

```python
"じんつうりき": MoveData(
    type="エスパー",
    category="special",
    pp=20,
    power=80,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.じんつうりき_apply_flinch,
        )
    }
),
```

## リーサル計算

対象外（`docs/progress/move.md` の「リーサル実装」列が `n/a`。ひるみ付与のみでHP分布に影響しない
追加効果のためリーサル計算ハンドラは不要）。

## 注意点・エッジケース

- ノーマルタイプの相手など、技自体が無効化された場合、`Event.ON_DAMAGE_HIT` 自体が発火しないため
  追加効果も発生しない。個別のガードは不要。
- まもるで防がれた場合も同様にダメージが発生せず `Event.ON_DAMAGE_HIT` が発火しないため、
  追加効果も発生しない。
- ひるみは `Event.ON_CHECK_ACTION` で消費される（`docs/spec/turn.md` 参照）。
  ON_DAMAGE_HIT での付与後、相手が次のターンに行動しようとする際に判定される。
  すでに行動済みのポケモンにひるみが付与されてもそのターンは効果がない。
