## Priority 根拠

`docs/spec/turn.md` の `Event.ON_HIT` セクションより：

```
| 100 | ダイマックスわざの効果 |
| 100 | ニードルガード等の効果 |
```

追加効果（状態異常付与）は同表に明示されていないが、`docs/spec/turn.md` の `Event.ON_DAMAGE` セクションに
「Priority 10: 追加効果（特殊なもの除く）」と記載されているのは ON_DAMAGE 側の処理。

ON_HIT の priority=100 は、既実装の `でんじほう`・`ほっぺすりすり` の登録コードと一致するため、同値を採用する（`h.MoveHandler` のデフォルト priority=100）。

（レビュー追記）りゅうのいぶきの実装では、既に実装済みの同系統技（フリーズボルト・ほうでん・のしかかり・スパーク・10まんボルト等）に合わせて `Event.ON_DAMAGE_HIT`（`h.MoveHandler` のデフォルト priority=100）を採用している。`ON_DAMAGE_HIT` は `docs/spec/turn.md` に未掲載のイベントだが、既存の同種ハンドラ（追加効果付与系）の登録実績を根拠とする。

## 注意点・エッジケース（review-test 向け）

### まひの無効条件
- でんきタイプはまひ無効（`docs/spec/turn.md` の `Event.ON_TRY_MOVE_2` Priority 130「でんきタイプ: まひ無効」）。
  ただし、この判定は `ailment_manager.apply` の内部または別のハンドラで処理されるため、
  `apply_ailment_to_defender` を呼ぶハンドラ側に追加ガードは不要。

### かみなりのキバの二重追加効果
- まひ(10%) と ひるみ(10%) は**それぞれ独立した確率判定**を行う。
- ON_HIT ハンドラをリスト `[ailment ハンドラ, volatile ハンドラ]` で登録する。
- 両効果がどちらも発動する確率は 1%（0.1 × 0.1）。
- `apply_ailment_to_defender` と `apply_volatile_to_defender` 内の `battle.resolve_secondary_chance` がそれぞれ独立して呼ばれる。

### かみなりの命中率ハンドラとの共存
- `かみなり` は既存の `ON_MODIFY_ACCURACY` ハンドラを保持したまま `ON_HIT` ハンドラを追加する。
- 2 つのイベントは独立しているため干渉しない。

### かみなりあらしの wind ラベル
- `かみなりあらし` は既存の `labels=["wind"]` を維持する。`secondary_effect` を追加する。

### のしかかりの minimize ラベル
- `のしかかり` は既存の `labels=["minimize", "contact"]` を維持する。`secondary_effect` を追加する。

### フリーズボルトは物理技
- タイプはこおりだがまひを付与する（仕様の通り、こおりは付与しない）。
- contact ラベルは仕様書に contact 記載がないため付与しない。

### 確率発動のテスト方針
- `accuracy=100` 相当の確率固定で `battle.random` を制御し、確率発動ケース（発動/非発動）を両方テストする。
- `start_battle()` の `accuracy` 引数は技命中には影響するが追加効果確率には影響しない点に注意。
