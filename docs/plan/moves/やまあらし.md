- かくとうタイプ / 物理 / 威力60（第六世代以降） / 命中100 / PP10 / 対象: 1体選択 / 優先度0 / 接触あり
- 必ず急所に当たる（急所ランク3）。追加効果なし。

`src/jpoke/data/moves/move_ya.py` に既に登録済み（`critical_rank=3`、`flags={"contact"}`）。

```python
"やまあらし": MoveData(
    type="かくとう",
    category="physical",
    pp=10,
    power=60,
    accuracy=100,
    critical_rank=3,
    flags={"contact"},
    handlers={},  # 常に急所に当たる（critical_rank=3）
),
```

内容は仕様書と一致しており、修正不要。

## ハンドラ構成

急所ランクの効果は `MoveData.critical_rank` フィールドのみで完結する
（`move_executor.py` の `_check_critical` が `ctx.move.critical_rank` を初期値として使用し、
`clamp_critic` 適用後 `CRIT_RATES[3] = 1.0` となるため常に急所となる）。
**専用ハンドラ関数は不要**。`handlers/move_attack.py` への追加コードはない。

`docs/plan/moves/攻撃技_急所系.md` に確定急所グループの共通実装パターンが既に記載されており、
本技もそのパターン（`critical_rank=3` のみで完結）に従っている。

## Priority 根拠

急所ランクは `move_executor.py` の `_check_critical` 内部（ダメージ計算 Interrupt 内）で処理される。
`docs/spec/turn.md` には `ON_CALC_CRITICAL_RANK` イベントの行は独立して記載されていない。
既存の `critical_rank=3` 技（あんこくきょうだ・こおりのいぶき・トリックフラワー）と同様のパターンを
採用しており、priority の新規検討は不要（MoveData フィールドのみで完結するため）。

## 注意点・エッジケース（review-test 向け）

1. **カブトアーマー / シェルアーマー との相互作用**
   - 確定急所（`critical_rank=3`）であっても、相手の特性カブトアーマー・シェルアーマーが発動すれば急所無効になる。
   - `ON_CALC_CRITICAL_RANK` イベントでカブトアーマー・シェルアーマー側が急所ランクを0に上書きする実装になっており、本技側の対応は不要。

2. **SV での使用可否**
   - 一次情報（docs/wiki/moves/やまあらし.html）に「SV で使用不可」と記載があるが、
     `src/jpoke/data/ps-champ-ja/moves.json` には登録されており、本プロジェクト（ポケモンチャンピオンズ
     シングルバトル）の対象範囲では使用可能な技として扱う。

3. **contact フラグ**
   - 直接攻撃技のため `flags={"contact"}` が必要。実装済み。

4. **テスト方針**
   - `critical_rank=3` は乱数に関係なく急所が確定するため、`fix_random` で命中判定のみ通過させ
     `battle.move_executor.critical is True` を確認するシンプルなテストで十分
     （`こおりのいぶき`・`あんこくきょうだ` の既存テストと同様のパターン）。
