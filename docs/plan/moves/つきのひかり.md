# 実装計画書: つきのひかり

## 仕様要約

- タイプ: フェアリー / カテゴリ: 変化 / PP: 5 / 対象: 自分
- label: `heal`
- 天候に応じた割合で使用者の HP を回復する

| 天候 | 回復量 |
|------|--------|
| はれ・おおひでり | 最大HP × 2/3 |
| 通常（天候なし・らんきりゅう） | 最大HP × 1/2 |
| あめ・おおあめ・すなあらし・ゆき | 最大HP × 1/4 |

HP が満タンの場合は失敗する。あさのひざし・こうごうせいと効果が完全に同一。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 天候参照して HP 回復 |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「自己回復技: HP満タン」が priority 100 として記載されている。`ON_STATUS_HIT` 自体は turn.md に独立エントリが存在しないが、既実装の `あさのひざし_heal_self` がデフォルト priority（100）を使用しているため、同一パターンで 100 を採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

新規関数は追加しない。既存の `あさのひざし_heal_self`（`こうごうせい` でも共用済み）を
`つきのひかり` のハンドラとしてそのまま登録する。回復ロジックが完全に同一のため、
専用関数を複製せず共用することでコード重複を避ける。

### `src/jpoke/data/moves/move_ta.py`

既存の `"つきのひかり"` エントリに `target="self"` と `handlers` を追加する
（PP は `src/jpoke/data/ps-champ-ja/moves.json` 準拠で 8）。

```python
"つきのひかり": MoveData(
    type="フェアリー",
    category="status",
    pp=8,
    target="self",
    flags={"heal"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.あさのひざし_heal_self,
        )
    }
),
```

## 注意点・エッジケース

- あさのひざし・こうごうせいと回復ロジックが完全に同一。ハンドラ関数も `あさのひざし_heal_self` を共用する
- `weather.sunny` プロパティが `"はれ"` と `"おおひでり"` の両方をカバーしていることを前提とする
- `らんきりゅう` は不利天候に含まれないため、通常時（1/2 回復）として扱う
- **`target="self"` を明示すること**。未設定だとデフォルト値 `"foe"` になり、
  `is_blocked_by_protect` / `is_reflectable` が誤って `True` を返し、まもる・マジックコートに
  誤反応してしまう
- PP は `src/jpoke/data/ps-champ-ja/moves.json` の値（8）を正とする。Gen9本家基準の 5 は移行漏れのバグ
