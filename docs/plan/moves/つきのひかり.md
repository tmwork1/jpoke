# 実装計画書: つきのひかり

## 仕様要約

- タイプ: フェアリー / カテゴリ: 変化 / PP: 5 / 対象: 自分
- label: `heal`
- 天候に応じた割合で使用者の HP を回復する

| 天候 | 回復量 |
|------|--------|
| はれ・おおひでり | 最大HP × 2/3 |
| 通常（天候なし・らんきりゅう） | 最大HP × 1/2 |
| あめ・おおあめ・すなあらし・ゆき | 最大HP × 1/4 |

HP が満タンの場合は失敗する。あさのひざし・こうごうせいと効果が完全に同一。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: `タールショット_modify_defender_stats` の後、`てっぺき_modify_attacker_stats` の前（「つ」ブロック）。

```python
def つきのひかり_heal_self(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """つきのひかり: 天候に応じた割合で自分のHPを回復する。"""
    mon = ctx.attacker
    if mon.hp == mon.max_hp:
        return HandlerReturn(value=False, stop_event=True)
    weather = battle.weather
    if weather.sunny:
        r = 2 / 3
    elif weather.name in {"あめ", "おおあめ", "すなあらし", "ゆき"}:
        r = 1 / 4
    else:
        r = 1 / 2
    battle.modify_hp(mon, r=r)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース

- あさのひざし・こうごうせいと回復ロジックが完全に同一。ハンドラ関数のコードも同一構造になる
- `weather.sunny` プロパティが `"はれ"` と `"おおひでり"` の両方をカバーしていることを前提とする
- `らんきりゅう` は不利天候に含まれないため、通常時（1/2 回復）として扱う
- data/move.py にエントリが存在するか確認してから追記する（「つ」行の五十音順内に配置）
