# どくばりセンボン 実装計画書

## 仕様要約（docs/spec/moves/どくばりセンボン.md 参照）

* タイプ: どく / 分類: 物理 / 威力: 60 / 命中: 100 / PP: 10
* 対象: 相手1体 / 優先度: 0 / 接触なし（直接攻撃ではない）
* 効果: 相手が『どく』『もうどく』状態のとき威力2倍。追加効果として50%の確率で相手を
  『どく』状態にする。

## 現状の実装状況

`src/jpoke/data/move.py` に登録済み、`src/jpoke/handlers/move_attack.py` にどく付与
ハンドラは実装済みだが、**威力2倍化ハンドラ（`ON_CALC_POWER_MODIFIER`）が欠落していた**
ため、本レビューで追加する。

```python
"どくばりセンボン": MoveData(
    type="どく",
    category="physical",
    pp=10,
    power=60,
    accuracy=100,
    flags={"secondary_effect"},
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
            ha.どくばりセンボン_double_power_when_poisoned,
        ),
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.どくばりセンボン_apply_poison_to_defender,
        ),
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_CALC_POWER_MODIFIER` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 相手が『どく』『もうどく』状態のとき威力2倍 |
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 50%の確率で相手を『どく』状態にする |

`src/jpoke/handlers/move_attack.py` の実装:

```python
def どくばりセンボン_double_power_when_poisoned(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """どくばりセンボン: 対象がどく/もうどく状態のとき威力が2倍になる。"""
    if ctx.defender.ailment.name in ("どく", "もうどく"):
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)


def どくばりセンボン_apply_poison_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_ailment_to_defender(battle, ctx, value, ailment="どく", chance=0.5)
```

威力2倍化ハンドラは `ベノムショック_double_power_when_poisoned`（`handlers/move_attack.py`）
と全く同じロジックであり、`apply_fixed_modifier(value, 8192)` を用いる（8192 = 2倍の
固定小数点表現、既存実装の慣習に合わせる）。

`apply_ailment_to_defender`（`handlers/move.py`）が `battle.resolve_secondary_chance` を
経由するため、ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は、汎用ハンドラ側で
処理される。本技側に個別のガードは不要。

## Priority 根拠

* `ON_CALC_POWER_MODIFIER` はダメージ計算内部のイベントで `docs/spec/turn.md` に未掲載。
  既存の同種ハンドラ（`ベノムショック_double_power_when_poisoned` など）と同じデフォルト
  priority（100）に揃える。
* `ON_DAMAGE_HIT`（どく付与）はダメージを与えた技の追加効果（状態異常付与）の既存の
  同種ハンドラ（どくどくのキバ・どくづき・どくばり 等）と同じデフォルト priority に従う。

## リーサル計算

`docs/progress/move.md` の該当行「リーサル実装」列は `n/a`（対象外）。
威力2倍化は対象の状態異常に依存する確定効果だが、どく付与自体はターン経過依存の
追加効果でありリーサル計算対象に含めない方針。威力2倍化のみを切り出して
リーサル計算に組み込む対応も、本プロジェクトの既存方針（ベノムショックも `n/a`）に
合わせて対象外とする。

## 注意点・エッジケース（review-test 向け）

1. **威力2倍化とちからずくの相互作用**
   - `ON_CALC_POWER_MODIFIER` の2倍化とちからずくの1.3倍化は別々の乗算であり、両方が
     条件を満たせば両方とも適用される（ダメージ計算の乗算チェーンに従う）。
2. **どく付与が同ターンの威力に影響しないこと**
   - 追加効果判定はダメージ計算後のため、このターン内でどく状態にした場合でも
     そのターンの威力は2倍にならない（`ON_CALC_POWER_MODIFIER` が `ON_DAMAGE_HIT`
     より先に評価されるため、実装上も自然に満たされる）。
3. **もうどく状態でも威力2倍**
   - `ailment.name in ("どく", "もうどく")` で両方を判定する。
4. **テスト方針**
   - `secondary_chance` を1に固定し、どく付与を確認するテストが既存
     （`test_どくばりセンボン_どくが発動する`）。
   - 威力2倍化については新規にテストを追加する（`ベノムショック` のテストパターンを踏襲）。
