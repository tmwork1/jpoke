# 計画書: ぶんまわす

## 仕様要約

- あくタイプ / 物理 / 威力60 / 命中100 / PP20 / 対象: 自分以外の全員（シングルバトルのみのため実質相手1体） / 優先度0 / 接触あり
- 追加効果なし（通常攻撃のみ）

## ハンドラ構成

追加効果を持たないため、専用ハンドラは不要。`handlers={}` のみで基本ダメージ計算に委ねる。

## Priority 根拠

追加効果がなく専用ハンドラを実装しないため、priority の検討対象イベントは存在しない。

## 実装箇所

### data/move.py への更新

既存の `"ぶんまわす"` エントリは以下の通りで、仕様と一致している（変更不要）。

```python
    "ぶんまわす": MoveData(
        type="あく",
        category="physical",
        pp=20,
        power=60,
        accuracy=100,
        flags={"contact"},
        handlers={},  # 追加効果なし
    ),
```

- `flags={"contact"}` を指定しているため接触判定あり（仕様と一致）。
- `target` 省略時のデフォルトは `"foe"`（相手1体）。本プロジェクトはシングルバトルのみを
  対象とするため、「自分以外の全員」という原作の範囲指定は実質的に相手1体と同義であり、
  デフォルト値のままで仕様と一致する。
- `priority` 省略時のデフォルトは `0` → 仕様と一致。
- `critical_rank` 省略時のデフォルトは `0`（通常の急所判定）→ 仕様と一致。

## 注意点・エッジケース

- 追加効果がないため、まもる・おうじゃのしるし・オウムがえし等の一般的な技判定は
  フレームワーク側の共通処理（`handlers/move_attack.py` の基本ダメージ計算経路）で
  自動的に処理され、本技固有のハンドラ実装は不要。
