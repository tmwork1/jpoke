# ついばむ 実装計画書

## 仕様要約（docs/spec/moves/ついばむ.md 参照）

* タイプ: ひこう / 分類: 物理 / 威力: 60 / 命中: 100 / PP: 20
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 相手がバトルに効果のあるきのみを持っていれば、命中後にそれを奪って自分が消費する
  （むしくいと全く同じ追加効果）。

## 現状の実装状況

`src/jpoke/data/move.py` に既に登録済み。むしくいと共通のハンドラ関数
`ha.むしくい_steal_and_use_berry` を `Event.ON_HIT` に登録している。

```python
"ついばむ": MoveData(
    type="ひこう",
    category="physical",
    pp=20,
    power=60,
    accuracy=100,
    flags={"contact"},
    handlers={
        Event.ON_HIT: h.MoveHandler(ha.むしくい_steal_and_use_berry)
    }
),
```

内容は仕様書と一致しており、修正不要。

## ハンドラ構成

| Event | subject_spec | 関数 | 役割 |
|---|---|---|---|
| ON_HIT | attacker:self / defender:self | `むしくい_steal_and_use_berry` | defender がきのみを持っていれば奪取し、attacker が即座に消費する |

`むしくい_steal_and_use_berry`（`handlers/move_attack.py`）の内部処理:

1. `defender.has_item()` かつ `defender.item.is_berry()` を確認（きのみでなければ何もしない）
2. `battle.item_manager.take_item(defender)` で attacker に移す
   （`take_item` 内部で `can_change_item` を通してねんちゃく等のチェックを行う。
   奪取失敗時は何もしない）
3. `battle.item_manager.force_trigger_berry(attacker)` で attacker に強制的にきのみ効果を発動・消費させる

**専用ハンドラの新規実装は不要**（むしくいの既存実装を流用）。

## 注意点・エッジケース（review-test 向け）

1. **きのみを持たない場合**: 追加効果は発生しない（むしくい側で既にテスト済みのパターン）。
2. **きのみ以外のアイテム**: 奪取は発生しない。
3. **タンガのみ等ダメージ計算前消費型のきのみ**: ダメージ計算時に先に消費されるため、
   命中後の奪取処理では既にきのみを失っており不発になる。
4. **ねんちゃく**: defender がねんちゃくを持つ場合、奪取自体が失敗し attacker は効果を受けない。
5. **みがわり**: ON_HIT はみがわり被弾時にも発動するため、みがわりを貫通してきのみを奪う。

これらのエッジケースは `むしくい_steal_and_use_berry` を対象とした
`tests/moves_attack/test_move_ma.py` の むしくい 用テストで既に網羅されている
（同一関数を共有するため、ついばむ側は代表的な1ケースのみで十分）。
