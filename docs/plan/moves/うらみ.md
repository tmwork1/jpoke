# 実装計画書: うらみ

## 仕様要約

- タイプ: ゴースト / カテゴリ: 変化 / PP: 10 / 命中: 100 / 対象: 1 体選択
- 相手が直前に使った技の PP を 4 減らす
- 失敗条件: 相手が技を使っていない場合（`executed_move` が None）

## 現状の実装状況

| 項目 | 状態 |
|------|------|
| `data/move.py` エントリ | 登録済み（handlers なし、accuracy 未設定） |
| `handlers/move_status.py` 関数 | 未実装 |

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_BEFORE_APPLY_MOVE` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手が技を使っていない場合は失敗 |
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手の `executed_move` の PP を 4 減らす |

## Priority 根拠

- `ON_BEFORE_APPLY_MOVE` priority 100: `docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「うらみ: 失敗条件」が priority 100 として明記されている
- `ON_STATUS_HIT` priority 100（デフォルト）: 同種ハンドラと同じパターン

## PP 情報の保持場所

- `Pokemon.executed_move` は `Move | None` 型で、直前に使用した技が格納される（`model/pokemon.py` で確認済み）
- `Move.modify_pp(v: int)` メソッドが存在し、PP の増減に使用できる（`model/move.py` で確認済み）
- `Move` オブジェクトは `Pokemon.moves` リスト内の要素と同一オブジェクトであるため、`executed_move.modify_pp(-4)` で次ターン以降の PP に反映される

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「う」行 — `うたう_apply_ailment_to_defender` の後、`えんまく_modify_defender_stats` の前に挿入する（「うら」は「うた」の後、「え」行の前）。

```python
def うらみ_can_apply(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """うらみの失敗チェック: 相手が技を使っていない場合は失敗する。"""
    if ctx.defender.executed_move is None:
        battle.add_event_log(ctx.attacker, LogCode.MOVE_FAILED,
                             payload={"reason": "うらみ"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def うらみ_deplete_pp(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """うらみの効果: 相手が直前に使った技の PP を 4 減らす。"""
    move = ctx.defender.executed_move
    if move is None:
        return HandlerReturn(value=False, stop_event=True)
    move.modify_pp(-4)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

既存の `"うらみ"` エントリに accuracy と handlers を追加する。

```python
"うらみ": MoveData(
    type="ゴースト",
    category="変化",
    pp=10,
    accuracy=100,
    handlers={
        Event.ON_BEFORE_APPLY_MOVE: h.MoveHandler(

        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.うらみ_deplete_pp,
        ),

```

## 注意点・エッジケース

- `Move.modify_pp(-4)` は `max(0, min(max_pp, pp + v))` でアンダーフロー・オーバーフローを防ぐ実装になっているため、PP が 0 のときでも安全に呼び出せる
- `executed_move` が格納されるタイミングを確認すること: 相手が行動する際に設定され、交代時にリセットされる（`reset_on_switch_out` で `executed_move = None`）
- 相手が「わるあがき」を使った場合、`executed_move` が「わるあがき」の `Move` オブジェクトになるため PP 減少を試みる。「わるあがき」の PP は 0 なので `modify_pp(-4)` は何も起きない
- 相手がターン内でまだ行動していない場合（自分が先攻）、`executed_move` は前ターンの技を保持している。失敗条件として turn.md で「うらみ: 失敗条件」が priority 100 で記載されているが、具体的な条件は仕様書の記述（「相手の最後に使った技のPPを4減らす」）に従い、`executed_move is None` チェックで代用する
