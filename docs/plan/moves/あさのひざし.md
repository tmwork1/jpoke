# 実装計画書: あさのひざし

## 仕様

### 技データ

- タイプ: ノーマル
- カテゴリ: 変化
- PP: 5
- 対象: 自分
- label: `heal`

### 効果

使用者の HP を以下の割合で回復する（天候により変化）:

| 天候 | 回復量 |
|------|--------|
| はれ・おおひでり | 最大HP × 2/3 |
| 通常（天候なし・らんきりゅう） | 最大HP × 1/2 |
| あめ・おおあめ・すなあらし・ゆき | 最大HP × 1/4 |

HP が満タンの場合は失敗する。

### 参照仕様書

- `docs/spec/global_fields/はれ.md`（回復量の天候補正テーブル参照）

---

## subject_spec と priority

### イベント: `ON_STATUS_HIT`

- **subject_spec**: `"attacker:self"`（`MoveHandler` のデフォルト値を使用）
- **priority**: 100

**根拠**: `docs/spec/turn.md` の `ON_APPLY_MOVE` テーブルに「自己回復技: HP満タン」が priority 100 として記載されている。`ON_STATUS_HIT` は `turn.md` に独立エントリが存在しないが、同種の変化技自己回復ハンドラ `じこさいせい_heal_self` も `h.MoveHandler(hs.じこさいせい_heal_self)` で priority=100 のデフォルトを使用しているため、これと同一パターンとして priority=100 を採用する。

## ハンドラ構成

### 1. `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: `あくまのキッス_apply_ailment_to_defender` の後、`あまいかおり_modify_defender_stats` の前。

```python
def あさのひざし_heal_self(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    mon = ctx.attacker
    if mon.hp == mon.max_hp:
        return HandlerReturn(value=False, stop_event=True)
    weather = battle.weather
    if weather.sunny:
        r = 2 / 3
    elif weather.name in {"あめ", "おおあめ", "すなあらし", "ゆき"}:
        r = 1 / 4
    else:
        r = 1 / 2
    battle.modify_hp(mon, r=r)
    return HandlerReturn(value=value)
```

### 2. `src/jpoke/data/move.py`

現在の `"あさのひざし"` エントリに `handlers` フィールドを追加する:

```python
"あさのひざし": MoveData(
    type="ノーマル",
    category="変化",
    pp=5,

    labels=["heal"],
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.あさのひざし_heal_self,
        )
    }
),
```

## 注意事項

- `battle.modify_hp(mon, r=r)` の `r` は小数で指定。端数処理は `battle.modify_hp` 側に委ねる（`じこさいせい` と同パターン）
- HP満タン判定は `mon.hp == mon.max_hp` で行い、失敗時は `HandlerReturn(value=False, stop_event=True)` を返す
- `weather.sunny` プロパティは `"はれ"` と `"おおひでり"` の両方をカバー済み
- 不利天候の判定は `weather.name in {"あめ", "おおあめ", "すなあらし", "ゆき"}` とする
- `らんきりゅう` は不利天候に含まれず、回復量 1/2（通常扱い）
