# 実装計画書: あさのひざし

## 仕様

### 技データ

- タイプ: ノーマル
- カテゴリ: 変化（`category="status"`）
- PP: 8（championsルールにより8/12/16/20の4段階へ統一。`docs/champions/move_list.txt` 準拠。
  Gen9本家基準の5は使わない）
- 対象: 自分（`target="self"`。デフォルト値 `"foe"` のままだと `まもる`/`マジックコート` の
  判定が `move.target == "foe"` を前提にしているため誤って防がれる/跳ね返されてしまう）
- label: `heal`

### 効果

使用者の HP を以下の割合で回復する（天候により変化）:

| 天候 | 回復量 |
|------|--------|
| はれ・おおひでり | 最大HP × 2/3 |
| 通常（天候なし・らんきりゅう） | 最大HP × 1/2 |
| あめ・おおあめ・すなあらし・ゆき | 最大HP × 1/4 |

HP が満タンの場合は失敗する。

天候の判定は `battle.weather_for(mon)`（使用者視点の有効天候）を使う。これにより以下が
自動的に考慮される:

- 使用者が **ばんのうがさ** を持つ場合、はれ/あめの補正を無効化（1/2扱い）
- 場に **ノーてんき/エアロック** 持ちがいる場合、天候効果自体を無効化（1/2扱い）

### 参照仕様書

- `docs/spec/moves/あさのひざし.md`
- `docs/spec/fields/はれ.md`（回復量の天候補正テーブル参照）

## subject_spec と priority

### イベント: `ON_STATUS_HIT`

- **ctx型**: `AttackContext`（変化技の `ON_STATUS_HIT` は攻撃フローとして発火するため
  `ctx.attacker` / `ctx.defender` を参照する。`EventContext` の `source`/`target` ではない）
- **subject_spec**: `"attacker:self"`（`MoveHandler` のデフォルト値を使用）
- **priority**: 100

**根拠**: `docs/spec/turn.md` の `ON_APPLY_MOVE` テーブルに「自己回復技: HP満タン」が priority 100 として記載されている。`ON_STATUS_HIT` は `turn.md` に独立エントリが存在しないが、同種の変化技自己回復ハンドラ `じこさいせい_heal_self` も `h.MoveHandler(hs.じこさいせい_heal_self)` で priority=100 のデフォルトを使用しているため、これと同一パターンとして priority=100 を採用する。

## ハンドラ構成

### 1. `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: `あくまのキッス_apply_ailment_to_defender` の後、`あまいかおり_modify_defender_stats` の前。

```python
def あさのひざし_heal_self(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """あさのひざし: 天候に応じた割合で自分のHPを回復する。
    攻撃側がばんのうがさを持つ場合、晴れ/雨状態でも1/2回復。
    """
    mon = ctx.attacker
    if mon.hp == mon.max_hp:
        return HandlerReturn(value=False, stop_event=True)
    weather = battle.weather_for(mon)
    if weather.sunny:
        r = 2 / 3
    elif weather.name in {"あめ", "おおあめ", "すなあらし", "ゆき"}:
        r = 1 / 4
    else:
        r = 1 / 2
    battle.modify_hp(mon, r=r)
    return HandlerReturn(value=value)
```

### 2. `src/jpoke/data/moves/move_a.py`

`"あさのひざし"` エントリ:

```python
"あさのひざし": MoveData(
    type="ノーマル",
    category="status",
    pp=8,
    target="self",
    flags={"heal"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.あさのひざし_heal_self,
        )
    }
),
```

## 注意事項

- `battle.modify_hp(mon, r=r)` の `r` は小数で指定。端数処理は `battle.modify_hp` 側に委ねる（`int(max_hp * r)` による切り捨て。じこさいせい等の既存回復技と同じ挙動で、五捨五超入は実装しない）
- HP満タン判定は `mon.hp == mon.max_hp` で行い、失敗時は `HandlerReturn(value=False, stop_event=True)` を返す
- `weather.sunny` プロパティは `"はれ"` と `"おおひでり"` の両方をカバー済み
- 不利天候の判定は `weather.name in {"あめ", "おおあめ", "すなあらし", "ゆき"}` とする
- `らんきりゅう` は不利天候に含まれず、回復量 1/2（通常扱い）
- `target="self"` を明示しないと `MoveData.target` のデフォルト値 `"foe"` になり、
  `is_blocked_by_protect`（`まもる`）・`マジックコート_reflect` の判定が `move.target == "foe"` を
  条件にしているため、誤って「まもるで防がれる」「マジックコートで跳ね返される」動作になる
