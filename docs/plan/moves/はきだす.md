# 実装計画書: はきだす（再レビュー）

## 仕様要約

`docs/spec/moves/はきだす.md` 参照。

* タイプ: ノーマル / 分類: 特殊 / 威力: 可変（たくわえる回数×100） / 命中率: 100 / PP: 10 / 対象: 1体選択
* たくわえる回数が0のとき失敗する。
* 使用すると、たくわえるで上がっていた分だけぼうぎょ・とくぼうのランクが下がり、
  たくわえる volatile がリセットされる。
* 特性おやこあいで2ヒット化しても、ランク低下は1回のみ発生する。

## 既存実装の確認結果と問題点

`src/jpoke/handlers/move_status.py` に `はきだす_check_can_use` / `はきだす_set_power` /
`はきだす_apply_after` が実装され、`src/jpoke/data/move.py` の `はきだす` エントリから
`hs.` プレフィックスで登録されていた。以下の問題を確認し、修正する。

1. **モジュール配置の誤り**: `はきだす` は `category="special"`（攻撃技）だが、
   ハンドラが `handlers/move_status.py`（変化技専用モジュール、docstring: 「変化技関連の
   イベントハンドラ関数を提供するモジュール」）に実装されていた。攻撃技ハンドラは
   `handlers/move_attack.py`（docstring: 「物理・特殊攻撃技の実行に関連するハンドラ関数」）に
   実装するのが本プロジェクトの規約のため、3関数を `move_attack.py` へ移動し、
   `data/move.py` 側の登録も `ha.` プレフィックスに変更する。
   テストも `tests/moves_status/test_move_ha.py` → `tests/moves_attack/test_move_ha.py` へ移動する。
2. **おやこあいとの併用でクラッシュする**: `はきだす_apply_after`（`ON_HIT`）は
   ヒットごとに発火する（`move_executor.py` の `_execute_hit` ループ内）。おやこあい特性で
   2ヒット化された場合、1ヒット目で `battle.volatile_manager.remove(mon, "たくわえる")` が
   実行され `たくわえる` volatile が消えるため、2ヒット目で
   `mon.volatiles["たくわえる"].count` にアクセスすると `KeyError` が発生する。
   仕様上も「ランク低下は1回のみ」のため、`ctx.hit_index != 1` の場合は何もせず
   `HandlerReturn(value=value)` を返すガードを追加する（`からぶりほけん_boost_speed_on_miss`
   等、既存の「1ヒット目のみ処理する」パターンに準拠）。
3. **威力決定イベントの見直し**: `はきだす_set_power` は `Event.ON_BEGIN_MOVE` で
   `ctx.move.power` を設定していたが、`ON_BEGIN_MOVE` は
   `move_executor.py` のコメント上「かたやぶりを適用する」ためのイベントであり
   （`ON_TRY_MOVE_1` より前に発火する）、可変威力の決定に使う慣例はない。
   同種の「絶対値で `ctx.move.power` を設定する」パターン（`プレゼント_roll_outcome` 等）は
   `Event.ON_TRY_MOVE_1`（デフォルト優先度100）を使っているため、これに合わせて
   `ON_BEGIN_MOVE` → `ON_TRY_MOVE_1` に変更する。
   `はきだす_check_can_use`（priority=30）は同じ `ON_TRY_MOVE_1` 内でより低い優先度のため
   先に評価され、失敗時は `stop_event=True` で以降のハンドラ（威力設定）が実行されない。

## ハンドラ構成（修正後）

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| はきだす | `Event.ON_TRY_MOVE_1` | 30 | `attacker:self`（デフォルト） | たくわえる回数が0なら失敗させる |
| はきだす | `Event.ON_TRY_MOVE_1` | デフォルト（100） | `attacker:self`（デフォルト） | たくわえる回数×100を`ctx.move.power`に設定する |
| はきだす | `Event.ON_END_MOVE` | デフォルト（100） | `attacker:self`（デフォルト） | たくわえるで上がった分のぼうぎょ・とくぼうランクを戻し、たくわえる volatile を削除する |

* `Event.ON_TRY_MOVE_1` は `docs/spec/turn.md` に priority の記載があるが、本ハンドラ2つの
  役割（失敗判定・威力決定）に対応する行は明記されていない。失敗判定は既存の
  `はきだす_check_can_use`（priority=30、`のみこむ_check_can_use` と同一）を踏襲し、
  威力決定は同種の `プレゼント_roll_outcome`（デフォルト優先度100）に倣う。
* `Event.ON_END_MOVE` は turn.md に priority 表がないため、同種の「行動終了時に必ず1回だけ
  発火する」既存ハンドラ（`フォトンゲイザー_restore_defender_ability` 等、デフォルト優先度100で
  登録されているもの）に倣いデフォルトのままとする。

## 再レビューでの修正（ON_HIT → ON_END_MOVE）

前回レビューの実装は `Event.ON_HIT`（ヒットごとに発火、`_execute_hit` 内でダメージ適用後に
発火）でランク低下・`たくわえる` リセットを行い、おやこあいの2ヒット対応として
`ctx.hit_index != 1` のガードを入れていた。しかし `docs/wiki/moves/はきだす.html` の
「技の仕様」節には「わざを外したり、効果がないゴーストタイプに使用した場合でも、
ぼうぎょ・とくぼうのランクが低下する」と明記されている。`move_executor.py` の実装上、
命中失敗（`ON_MISS`）やタイプ相性0倍（`_check_hit_by_type` で `move_success=False` となり
`_execute_hit` 自体が呼ばれない）の場合は `ON_HIT` が一切発火しないため、
旧実装ではこれらのケースでランクが低下せず、一次情報と矛盾していた。

この矛盾を解消するため、ランク低下・`たくわえる` リセットの発火イベントを
`Event.ON_HIT` から `Event.ON_END_MOVE` に変更する。`ON_END_MOVE` は `run_move()` の
`try/finally` 内で必ず1回発火するイベントで、`メトロノーム_update_count`
（`handlers/item.py`）が同種の「行動不能時や溜めターンを含め必ず発火する」性質を利用した
先例として存在する。この前例に倣い、`battle.move_executor.action_success` が `False`
（麻痺・ひるみ等でそもそも技を出そうとしていない）の場合のみ何もせず、それ以外
（成功・ミス・タイプ相性0倍・まもるによるブロックいずれも含む）ではランク低下・
`たくわえる` リセットを行う。`ON_END_MOVE` は技全体につき1回しか発火しないため、
おやこあいで2ヒット化されても自然にランク低下は1回のみになり、旧実装で必要だった
`ctx.hit_index != 1` ガードは不要になる（`ctx.hit_index` を参照する必要自体がなくなる）。

なお、まもるで防がれた場合に関する明記は一次情報にはないが、`ON_TRY_MOVE_1` で
`まもる_protect`（`subject_spec="defender:self"`, priority=100）が `False` を返して
`move_success=False` になるケースも、上記の「タイプ相性0倍」と区別できず同じ経路
（`action_success` は `True` のまま）を通るため、実装上はまもるで防がれた場合も
ランクが低下する。これは「外れても効果が及ぶ」という一次情報の趣旨から見て妥当な
解釈であり、`docs/spec/moves/はきだす.md` にもその判断根拠を明記した。

## 実装詳細・エッジケース

* たくわえるで上がったランクを正確に戻すため、`mon.volatiles["たくわえる"].count` を
  参照する（たくわえる volatile 自体が実際に上昇した段階数を保持している前提。
  ランク上限などで一部上昇しなかった場合の扱いは `たくわえる` 側の実装に委譲する）。
* `battle.modify_stats` を使ってランクを下げるため、あまのじゃく・たんじゅん等の特性補正は
  自動的に適用される。しろいきり等のランクダウン防止効果は `modify_stats` の実装上、
  技由来のランク低下にのみ適用される想定のため、はきだす由来の低下は影響を受けない
  （既存の `modify_stats` 実装を変更する必要はない）。
* てだすけによるダメージ補正は、通常のダメージ計算パイプラインを経由するため
  特別な対応は不要。
* さきどり（Snatch）は本プロジェクト未実装のため対応不要。
