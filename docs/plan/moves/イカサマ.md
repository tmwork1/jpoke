# イカサマ 実装計画書

参照: docs/spec/moves/イカサマ.md

## 現状

* `src/jpoke/data/moves/move_a.py` の `"イカサマ"` エントリは以下の通り実装済み。
  - type="あく", category="physical", power=95, accuracy=100
  - flags={"contact"}
  - `handlers={}`（追加効果なし。ダメージ計算式自体が特殊なため専用ハンドラは不要）
* 攻撃力の入れ替え（自分の『こうげき』ではなく相手の『こうげき』実数値・ランク補正を使う）は
  `src/jpoke/core/damage.py` の `_calc_final_attack` 内で `move.name == 'イカサマ'` により
  ハードコードされている。この分岐の後は通常の物理技と同じ `ON_CALC_ATK_RANK_MODIFIER` /
  `ON_CALC_ATK_MODIFIER` フローに乗るため、アイテム・特性・状態異常の補正（こだわりハチマキ・
  ちからもち・わざわいのおふだ・やけど等）は使用者側の状態を正しく参照する。追加のハンドラ実装は
  不要。

## 発見した不具合と修正

### PP 値の不一致（実装バグ）

* `data/moves/move_a.py` の PP が `15`（Gen9本家基準）になっていたが、
  `src/jpoke/data/ps-champ-ja/moves.json` では `16` と記載されている
  （チャンピオンズでは全技PPが8/12/16/20の4段階に統一されているため）。
  `docs/progress/move.md` の PP 列はすでに `16` で記載されており、実装側が移行漏れだった。
  → `pp=16` に修正。

### パラドックス特性（クォークチャージ/こだいかっせい）との相互作用バグ

* `src/jpoke/handlers/ability_paradox.py` の `apply_atk_modifier` は、修正前は次のように
  技名で分岐していた。
  ```python
  if ctx.move.name == "イカサマ":
      boost_mon = defender
      stat = "atk"
  elif ctx.move.name == "ボディプレス":
      boost_mon = attacker
      stat = "def"
  else:
      boost_mon = attacker
      stat = "atk" if ctx.move.category == "physical" else "spa"
  ```
  つまりイカサマ使用時は「対象（相手）自身のパラドックス補正状態」を見て 1.3 倍を適用していた。
* しかし `docs/spec/abilities/クォークチャージ.md` / `docs/spec/abilities/こだいかっせい.md` の
  「特性の効果はランク補正上昇とは異なる」の項に明記されている通り、
  - 「攻撃が上昇しても、イカサマを受けたときのダメージは変わらない」
    → 対象（相手）自身のパラドックス補正はイカサマのダメージに一切影響しない。
  - 「攻撃が上昇したとき使用するボディプレスは防御の数値を1.3倍にしてダメージを与える。
    防御が上がってもダメージは変わらない」
    → ボディプレスは使用者自身の『ぼうぎょ』実数値を攻撃として使うが、パラドックス補正が
    乗るかどうかは技の分類（物理）に対応する「攻撃側スロット」（＝『こうげき』が強化対象か
    どうか）で判定される。『ぼうぎょ』が強化対象でも乗らない。
* まとめると、パラドックス特性の補正は「ダメージ計算でどの実数値が使われているか」とは無関係に、
  常に「使用者自身の攻撃側スロット（物理技なら攻撃、特殊技なら特攻）が強化対象かどうか」だけで
  判定される。イカサマ・ボディプレスのような特殊な参照先を持つ技であっても、通常の物理技/特殊技
  と全く同じ判定でよい。
* 修正後:
  ```python
  def apply_atk_modifier(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
      attacker = ctx.attacker
      stat = "atk" if ctx.move.category == "physical" else "spa"
      if attacker.paradox_boost_stat == stat:
          value = apply_fixed_modifier(value, 5325)
      return HandlerReturn(value=value)
  ```
  技名による分岐（`defender`/`boost_mon` の切り替え）を撤廃し、常に `ctx.attacker` の
  `paradox_boost_stat` を技分類（物理/特殊）で判定するのみにした。
* この修正はボディプレス側の挙動にも影響するが、ボディプレス自体の個別レビュー
  （`docs/progress/move.md` のボディプレス行）は別タスクの対象であり、本タスクでは
  イカサマとの相互作用としてのみ確認・修正する。

## リーサル計算

* `docs/progress/move.md` の「リーサル実装」「リーサルテスト」列は `n/a`。対象外
  （攻撃力の参照元入れ替えはダメージ計算内で完結し、専用のリーサルハンドラを要しない）。

## テスト

* `tests/moves_attack/test_move_a.py` に以下を追加する。
  - 使用者ではなく対象の『こうげき』実数値を攻撃として計算すること
  - 対象の攻撃ランク上昇でダメージが増加すること
  - 使用者自身の攻撃ランク上昇はダメージに影響しないこと
* `tests/abilities/test_ability__group.py` に、パラドックス特性修正の回帰テストを追加する。
  - 使用者自身のパラドックス補正（攻撃）はイカサマにも通常の物理技と同様に乗ること
  - 対象（相手）のパラドックス補正（攻撃）はイカサマのダメージに影響しないこと
* 既存テスト（`tests/items/test_item_ka.py` のこだわりハチマキ/くろいメガネ、
  `tests/abilities/test_ability_ta.py` のちからもち）はすでに使用者側の状態を正しく参照する
  ことを確認済みのため変更なし。
