# インファイト 実装計画書（再レビュー）

## 仕様要約

`docs/spec/moves/インファイト.md` 参照。

* タイプ: かくとう / 分類: 物理 / 威力: 120 / 命中率: 100 / PP: 8（Championsでは全技PPが
  8・12・16・20に統一。SVまでのPPは5） / 対象: 相手1体
* 直接攻撃（`contact`）。
* 技が成功すると、自分の『ぼうぎょ』『とくぼう』ランクが1段階ずつ下がる（確定効果）。

## 既存実装の確認結果と問題点

`src/jpoke/data/moves/move_a.py` の `インファイト` エントリ、
`src/jpoke/handlers/move_attack.py` の実装は以下の点で仕様と不一致だった（修正済み）。

* `pp=5`（SVまでの値）になっていたが、`src/jpoke/data/ps-champ-ja/moves.json` /
  `docs/champions/changes_from_sv.md`（技のPPが8・12・16・20の4段階に統一）によれば
  Championsでは `8` が正しい値のため修正した。
* `flags={"contact", "secondary_effect"}` のうち `secondary_effect` が誤り。
  `docs/wiki/abilities/ちからずく.html` の一次情報によれば、ちからずくの対象技は
  「相手のランク補正を下げる技」「自分のランク補正を上げる技」等に限られ、
  「自分のランク補正を下げる技」は対象外。既存実装の `アームハンマー`・`いじげんラッシュ`
  （`docs/spec/moves/アームハンマー.md`・`docs/spec/moves/いじげんラッシュ.md` 参照）でも
  同様の判断で `secondary_effect` を付与していない。インファイトの防御・特防ダウンは
  自分自身への確定効果であるため、`secondary_effect` フラグを削除する。
  ※ この `secondary_effect` フラグが付いたままだと、ちからずく所持時に
  `ちからずく_boost`（威力1.3倍）と `ちからずく_disable_secondary_effect`
  （確率0で自分のランクダウンを無効化）が誤発動してしまうバグになる。
* ハンドラ関数名が `インファイト_lower_defender_stats` となっていたが、実際には
  `modify_attacker_stats` を呼び出して自分自身（`ctx.attacker`）のランクを下げており、
  関数名と実装内容が矛盾していた（命名バグ）。`アーマーキャノン_lower_attacker_stats` 等の
  既存命名規則に合わせ `インファイト_lower_attacker_stats` に改名する。

```python
"インファイト": MoveData(
    type="かくとう",
    category="physical",
    pp=8,
    power=120,
    accuracy=100,
    flags={"contact"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.インファイト_lower_attacker_stats,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 命中後、自分の『ぼうぎょ』『とくぼう』ランクを1段階ずつ下げる（確定効果） |

`src/jpoke/handlers/move_attack.py` の実装（改名・修正済み）:

```python
def インファイト_lower_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """インファイト: 命中時に確定で自分の『ぼうぎょ』『とくぼう』ランクが1段階ずつ下がる。

    自分自身のランクを下げる確定効果のため、ちからずくの対象外（secondary_effect フラグ非付与）。
    """
    return modify_attacker_stats(battle, ctx, value, stats={"def": -1, "spd": -1})
```

`modify_attacker_stats`（`handlers/move.py`）の `chance` 既定値は1のため常に発動する。
自分自身への確定効果であり、`resolve_secondary_chance` 経由の「追加効果」判定
（ちからずく等）の対象にはならない（`flags` に `secondary_effect` を付与しない）。

## Priority 根拠

自分自身への確定ランク低下は既存の同種ハンドラ（アーマーキャノン・アームハンマー・
いじげんラッシュ等）と同じ `ON_HIT` / priority 100（デフォルト）のパターンに従う。
`ON_HIT` は `docs/spec/turn.md` に「10 自分のランクが下がる反動」として掲載されているが、
本プロジェクトの既存の同種ハンドラはいずれも優先度指定なし（デフォルト100）で
実装されているため、既存実装との一貫性を優先してそのパターンに従う。

## 注意点・エッジケース（review-test 向け）

1. **ちからずく等「追加効果」系特性・アイテムとの非該当**
   - 自分自身への確定ランク低下は「追加効果」ではないため、ちからずくの対象にならない。
     `flags` から `secondary_effect` を削除するのが正しい。
2. **まもる貫通なし**
   - 通常の攻撃技であり、`まもる` 等の対象になる。個別のガードは不要
     （`unprotectable` フラグ等は付与しない）。
3. **テスト方針**
   - 100%固定効果のため `secondary_chance` の調整は不要。命中後に自分の
     『ぼうぎょ』『とくぼう』ランクが1段階ずつ下がることを確認する通常テストを追加する。
   - ちからずく所持時にも威力上昇・効果無効化が発生しないことを確認するテストを追加する
     （`アームハンマー` の同種テストと同じ方針）。
