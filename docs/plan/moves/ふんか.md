# 計画書: ふんか（再レビュー）

## 仕様要約

* 使用者の残りHPが多いほど威力が大きくなる（`docs/spec/moves/ふんか.md` 参照）。
* 威力 = floor(150 × 残りHP ÷ 最大HP)。最小値は1。
* しおふき・ドラゴンエナジーと全く同じ威力計算式を使用する。

## 既存実装の確認結果

`src/jpoke/handlers/move_attack.py` の `ふんか_calc_power` を確認した結果、
仕様書の計算式（`modifier = floor(HP比率 × 4096)`、`final_power = round_half_down(150 × modifier / 4096)`）
と完全に一致しており、最小値1の保証（`max(1, modifier)`）も実装済み。
`src/jpoke/data/moves/move_ha.py` の `MoveData`（type=ほのお/category=special/pp=5/power=150/accuracy=100）も
一次情報と一致している。しおふき・ドラゴンエナジーも同じ計算式を使用しており、実装の修正は不要。

## ハンドラ構成（既存・変更なし）

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| ふんか | `Event.ON_CALC_POWER_MODIFIER` | デフォルト（100） | `attacker:self`（`MoveHandler` デフォルト） | 使用者の残りHP割合から技の実威力の modifier を算出して返す |

* `Event.ON_CALC_POWER_MODIFIER` は `docs/spec/turn.md` に priority の記載がないため、
  既存の同種ハンドラ（`しおふき_calc_power` ほか、`ON_CALC_POWER_MODIFIER` を使う全ての
  `MoveHandler`）が明示的な priority 指定なし＝デフォルト値（100）で登録されている慣例に
  合わせ、今回もデフォルトのままとする。

```python
def ふんか_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """ふんか: 自分のHP比率に比例して威力が決まる（最大150）。しおふきと同計算。"""
    modifier = int(ctx.attacker.hp * 4096 / ctx.attacker.max_hp)
    modifier = max(1, modifier)  # 威力が最低1になるよう保証
    return HandlerReturn(value=modifier)
```

```python
"ふんか": MoveData(
    type="ほのお",
    category="special",
    pp=5,
    power=150,
    accuracy=100,
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
            ha.ふんか_calc_power,
        ),
    }
),
```

## リーサル計算

`docs/progress/move.md` の該当行は「リーサル実装: n/a」「リーサルテスト: n/a」のため対象外。
理由: 威力は攻撃側自身の現在HPにのみ依存し、相手（`defender`）のHP分布 `StateDist` を
直接左右する要素ではないため、`lethal.py` 側の専用ハンドラは不要。

## テスト

既存の `tests/moves_attack/test_move_ha.py` に威力を確認するテストが2件ある
（HP満タン→威力150、HP半分→威力150未満・1以上）。しおふきの既存テストと同一パターンで
計算式・実装ともに正しいため、テスト内容自体の修正は不要。

## 注意点・エッジケース

  一次情報（wiki）・実装（`data/moves/move_ha.py` の `pp=5`）と一致する `5` に修正する。
