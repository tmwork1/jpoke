# 実装計画書: じゅうりょく

## 仕様

### 技データ

- タイプ: エスパー
- カテゴリ: 変化（`category="status"`）
- PP: 8（championsルールにより8/12/16/20の4段階へ統一。`docs/champions/move_list.txt` 準拠）
- 対象: 全体の場（`target="field"`。デフォルト値 `"foe"` のままだと `まもる`/`マジックコート` の
  判定が `move.target == "foe"` を前提にしているため誤って防がれる/跳ね返されてしまう）

### 効果

場を『じゅうりょく』状態（5ターン、発動ターンを含む）にする。フィールド自体の詳細効果
（命中率補正・接地化・使用禁止技・Gのちから威力上昇）は `docs/plan/fields/じゅうりょく.md` /
`docs/spec/fields/じゅうりょく.md` を参照。技自体はフィールドの発動のみを行う。

### 参照仕様書

- `docs/spec/moves/じゅうりょく.md`
- `docs/spec/fields/じゅうりょく.md`

## subject_spec と priority

### イベント: `ON_STATUS_HIT`

- **ctx型**: `AttackContext`（変化技の `ON_STATUS_HIT` は攻撃フローとして発火するため
  `ctx.attacker` / `ctx.defender` を参照する）
- **subject_spec**: `"attacker:self"`（`MoveHandler` のデフォルト値を使用）
- **priority**: 100

**根拠**: `ON_STATUS_HIT` は `docs/spec/turn.md` に独立エントリが存在しない。同種の
全体の場を発動する変化技 `トリックルーム_activate_global_field`
（`h.MoveHandler(hs.トリックルーム_activate_global_field)`）も priority=100 のデフォルトを
使用しているため、これと同一パターンとして priority=100 を採用する。

## ハンドラ構成

### 1. `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: `じゅうでん_apply` の後、`スキルスワップ_can_apply` の前。

```python
def じゅうりょく_activate_global_field(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return HandlerReturn(value=battle.global_manager.activate("じゅうりょく", 5))
```

すでに『じゅうりょく』状態が有効な場合、`activate` は `max_count<=1` のため `False` を
返し、技は失敗扱いになる（トリックルームのような再使用による解除効果はない）。

### 2. `src/jpoke/data/moves/move_sa.py`

`"じゅうりょく"` エントリ:

```python
"じゅうりょく": MoveData(
    type="エスパー",
    category="status",
    pp=8,
    target="field",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.じゅうりょく_activate_global_field,
        ),
    }
),
```

### 3. `src/jpoke/data/field/global_field.py`

フィールド本体の効果（命中率補正・接地化・使用禁止技・Gのちから威力上昇・
そらをとぶ/でんじふゆう解除・カウントダウン）は既存の `GLOBAL_FIELD["じゅうりょく"]` に
実装済み。詳細は `docs/plan/fields/じゅうりょく.md` を参照。

## 注意事項

- `target="field"` を明示しないと `MoveData.target` のデフォルト値 `"foe"` になり、
  `is_blocked_by_protect`（`まもる`）・`マジックコート_reflect` の判定が `move.target == "foe"` を
  条件にしているため、誤って「まもるで防がれる」「マジックコートで跳ね返される」動作になる
- 命中率補正（`じゅうりょく_modify_accuracy`）は一撃必殺技（`ctx.move.has_flag("ohko")`）を
  除外する。ゲーム内の「一撃必殺技の命中は上がらない」仕様に対応するため。
