# ソウルクラッシュ 実装計画書

## 仕様要約（docs/spec/moves/ソウルクラッシュ.md 参照）

* タイプ: フェアリー / 分類: 物理 / 威力: 75 / 命中: 100 / PP: 15
  （src/jpoke/data/ps-champ-ja/moves.json に該当行なし・changes_from_sv.md にも変更記載なしのため
  第9世代基準値を採用）
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 100%の確率で相手の『とくこう』ランクを1段階下げる（追加効果）。

## 現状の実装状況

`src/jpoke/data/moves/move_sa.py` に登録済み。

```python
"ソウルクラッシュ": MoveData(
    type="フェアリー",
    category="physical",
    pp=15,
    power=75,
    accuracy=100,
    flags={"contact"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.ソウルクラッシュ_lower_spa_C,
        )
    }
),
```

`src/jpoke/handlers/move_attack.py` の実装:

```python
def ソウルクラッシュ_lower_spa_C(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_defender_stats(battle, ctx, value, stats={"spa": -1})
```

### 問題点: `secondary_effect` フラグの欠落

同種の「100%の確率で相手のとくこうを1段階下げる」技（はいよるいちげき・むしのていこう）は
いずれも `flags={"contact", "secondary_effect"}` / `flags={"secondary_effect"}` を持ち、
`docs/spec/moves/_move_secondary_1.md` の追加効果一覧にもソウルクラッシュ自身が
「defender 1(=100%) C -1」として掲載済みである。
現状の `flags={"contact"}` には `secondary_effect` が欠落しており、ちからずく特性
（`ちからずく_boost` / `ちからずく_disable_secondary_effect`、共に `ctx.move.has_flag("secondary_effect")`
で判定）が正しく作用しない（威力1.3倍化されず、追加効果も無効化されない）バグとなっている。
`flags={"contact", "secondary_effect"}` に修正する。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_DAMAGE_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 100%の確率で相手の『とくこう』ランクを1段階下げる |

`modify_defender_stats`（`handlers/move.py`）は `chance` 省略時に100%として扱い、
`battle.resolve_secondary_chance` を経由するため、ちからずくによる追加効果無効化（確率0化）は
`secondary_effect` フラグさえ正しく付与されていれば汎用的に処理される。ハンドラ関数自体の
修正は不要。

## Priority 根拠

ダメージを与えた技の追加効果（ランク変化）は既存の同種ハンドラ（はいよるいちげき・むしのていこう・
とびかかる 等）と同じ `ON_DAMAGE_HIT` / priority 100（デフォルト）のパターンに従う。
`docs/spec/turn.md` にも `ON_DAMAGE_HIT` は他の追加効果技と共通のタイミングとして扱われており、
個別の priority 検討は不要。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"contact", "secondary_effect"}` が必要。ちからずく所持時は `resolve_secondary_chance` が
     確率を0にし、とくこうランク低下が発動しなくなる（威力1.3倍化は `ちからずく_boost` 側で処理）。
     修正前はフラグ欠落によりちからずくが無視され、威力据え置き・追加効果発動という誤動作になっていた。
2. **テスト方針**
   - `battle.actives[1].rank["spa"] == -1` を確認する（100%発動のため乱数固定は不要）。
   - ちからずく所持ポケモンに使わせた場合、威力1.3倍化・とくこうランク変化なしを確認するテストを追加する。
