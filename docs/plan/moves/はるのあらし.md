# 実装計画書: はるのあらし

## 仕様要約

`docs/spec/moves/はるのあらし.md` 参照。

* タイプ: フェアリー / 分類: 特殊 / 威力: 100 / 命中率: 80 / PP: 5 / 対象: 相手1体選択
* 追加効果として、30%の確率で相手の「こうげき」ランクを1段階下げる。
* 風の技（`wind` フラグ）に分類され、かぜのり・ふうりょくでんき等の対象になる。
* マジックコートでは反射できない（攻撃技のため、`マジックコート_reflect` の
  `ctx.move.category == "status"` 判定で自動的に対象外になる。個別対応不要）。

## ハンドラ構成

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| はるのあらし | `Event.ON_DAMAGE_HIT` | デフォルト（100） | `attacker:self`（デフォルト） | 30%の確率で相手のこうげきランクを1段階下げる |

* `Event.ON_DAMAGE_HIT` は `docs/spec/turn.md` に明記された行はないが、同種の
  「ダメージ命中後に確率で相手の能力ランクを下げる」既存ハンドラ
  （`はいよるいちげき` 等、`handlers/move_attack.py` の `modify_defender_stats`
  呼び出しパターン）がすべて `ON_DAMAGE_HIT` を使用しているため、これに倣う
  （`ハートスタンプ` はひるみ付与技であり本パターンの例ではない上、第9世代に存在しない技のため
  実装対象外。`docs/spec/moves/ハートスタンプ.md` 参照）。
* 既存の共通ユーティリティ `handlers/move.py` の `modify_defender_stats` を再利用し、
  `stats={"atk": -1}, chance=0.3` を渡す。`chance` は `battle.resolve_secondary_chance`
  経由でおうじゃのしるし・てんのめぐみ・ちからずく等の補正を自動的に受ける。

## 実装詳細・エッジケース

* `flags={"wind", "secondary_effect"}` を設定する。`wind` はかぜのり（無効化＋こうげき+1）・
  ふうりょくでんき（じゅうでん付与）等の対象判定に使われる既存フラグ。`secondary_effect` は
  ちからずく・てんのめぐみ等の判定に使われる。
* 対象は本プロジェクトがシングルバトル専用のため `target="foe"`（デフォルト）のままでよい
  （原作の「相手全員」はシングルバトルでは相手1体と等価）。
* まもる・みがわり・マジックコート等は既存の共通処理（`まもる_protect` 等）にすべて委譲され、
  本技固有の追加対応は不要。

## リーサル計算

`docs/progress/move.md` の該当行は「リーサル実装: n/a」「リーサルテスト: n/a」のため対象外
（追加効果はダメージ量に影響しない能力ランク変化のみのため）。

## テスト

`tests/moves_attack/test_move_ha.py` に以下を追加する。

* `secondary_effect` フラグを持つことの確認（`はいよるいちげき` の既存テストに倣う）。
* 追加効果なしでダメージを与えることの確認。
* 30%の確率で相手のこうげきランクが1段階下がることの確認（`secondary_chance=1.0` で固定）。

マジックコートで反射されないことは `10まんボルト` 等で既に汎用的にカバーされているため、
本技専用のテストは追加しない。
