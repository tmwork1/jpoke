# くちばしキャノン 実装計画書

## 仕様要約（docs/spec/moves/くちばしキャノン.md 参照）

* タイプ: ひこう / 分類: 物理 / 威力: 120（Champions） / 命中: 100 / PP: 15
* 対象: 相手1体 / 優先度: -3 / 接触なし（弾のわざ）
* 効果: 加熱してから攻撃を繰り出すまでの間に直接攻撃を受けると、攻撃者をやけど状態にする。
  やけど付与は追加効果扱いではなく、技はそのまま発動する。
* `non_negoto`（ねごとで選ばれない）・`bullet`（ぼうだんの対象）ラベルを持つ。

## 現状の実装状況（レビュー時点）

`src/jpoke/data/move.py` に登録済み、`src/jpoke/handlers/move_attack.py` に実装済み。

```python
"くちばしキャノン": MoveData(
    type="ひこう",
    category="physical",
    pp=15,
    power=120,
    accuracy=100,
    priority=-3,
    flags={"bullet", "non_negoto"},
    handlers={
        Event.ON_TRY_ACTION: h.MoveHandler(
            ha.くちばしキャノン_burn_contact_hitter,
            subject_spec="attacker:self",
            priority=5,
        ),
    }
),
```

### レビューで発見した問題と対応

1. **`ON_TRY_MOVE_1`（priority=100）で判定していたため、行動可否判定より後にしか実行されず、
   まひ／ねむり／こおり／こんらん等で使用者が行動できない場合にやけど効果が発動しない
   バグがあった。**
   → `Event.ON_TRY_ACTION`（priority=5、`かえんぐるま_thaw_attacker` 等の「行動可否判定より
   前に必ず実行する」パターンと同じ priority 帯）に変更し、行動可否とは独立してやけど判定が
   行われるよう修正した（詳細は下記「Priority 根拠」参照）。
2. **`src/jpoke/data/volatile.py` の `"くちばしキャノン"` VolatileData と、
   `src/jpoke/handlers/volatile.py` の `くちばしキャノン_burn_attacker` /
   `くちばしキャノン_remove_volatile` が、どこからも `volatile_manager.apply()` されておらず
   完全に未使用（デッドコード）だった。**
   → これは本来「技選択時（加熱時）にユーザーへ揮発性状態を付与し、ON_DAMAGE_HIT で反応する」
   という設計だったと推測されるが、実際に効果を発揮している実装は `contact_hitter`
   （`Pokemon.contact_hitter`、`core/move_executor.py` で接触技ヒット時に記録）を
   `ON_TRY_ACTION` 時点で参照する方式であり、volatile ベースの実装は一度も呼び出されず
   誤解を招くだけの残骸だった。レビューで削除し、`scripts/generate_literals/generate_volatile_literal.py`
   で `types/volatile.py` の `VolatileName` を再生成した。
3. **`docs/progress/move.md` の PP列が `8` になっていたが、実装・一次情報とも `15` が正しい。**
   → `15` に修正した。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_TRY_ACTION` | 5 | attacker:self | 使用者が今ターン既に接触技のダメージを受けていれば（`Pokemon.contact_hitter`）、その攻撃者をやけど状態にする |

```python
def くちばしキャノン_burn_contact_hitter(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """くちばしキャノン発動前の判定: 接触技を受けていた場合は攻撃者をやけどにする。

    くちばしキャノンは優先度-3で必ず後攻となるため、技発動前に相手の接触技を受けることがある。
    この時点で contact_hitter が記録されていれば、その攻撃者をやけど状態にする。技自体はそのまま発動する。
    ON_TRY_ACTION の中でも最も早い priority=5 で登録することで、まひ・ねむり・こおり・こんらん等により
    使用者自身が行動できない場合でも「加熱」自体は必ず行われ、やけど付与が判定されるという仕様
    （加熱状態については行動可否の判定と独立している）を再現する。
    """
    hitter = ctx.attacker.contact_hitter
    if hitter is not None:
        battle.ailment_manager.apply(hitter, "やけど", source=ctx.attacker, ctx=ctx)
    return HandlerReturn(value=value)
```

`Pokemon.contact_hitter` は `core/move_executor.py` の `_execute_hit` 内で、
`battle.query.is_contact_reaction(ctx)`（接触技かつぼうごパット等で反応効果が防がれていない）
を満たし、かつ実際にダメージを与えた（`actual_damage > 0`）場合にのみ defender 側に記録され、
`reset_turn_state()` によってターン開始時に None へリセットされる。これにより「そのターン中に
既に接触技のダメージを受けたか」を判定できる。

## Priority 根拠

### `ON_TRY_ACTION` priority=5

`docs/spec/turn.md` の `Event.ON_BEFORE_MOVE`（技使用前の処理）には
「くちばしキャノンの準備行動」が priority=100 で記載されているが、これは致死率計算専用の
`LethalEvent.ON_BEFORE_MOVE`（`core/lethal.py`）に相当する概念であり、通常戦闘の `Event` enum
には対応するイベントが存在しない（`Event.ON_BEFORE_ACTION` は「emit: 未実装（各アクション実行
直前に発火する予定）」とコメントされている未実装イベントであり、個別ポケモンの技選択に基づく
処理には現状使えない）。

くちばしキャノンの「加熱してから攻撃するまで」の効果は、実質的に「使用者自身の行動実行
（`run_move`）が始まった時点で、そのターン中に既に生じた別の行動の結果（受けたダメージの
有無）を参照する」という、`きあいパンチ_check_move`（`ON_TRY_MOVE_1`、priority=30、
「30 | カウンター・ミラーコート: 適格ダメージ無」と同区分）と同種の判定である。

ただし、きあいパンチは「自分の技が不発になるかどうか」の判定であるのに対し、くちばしキャノンの
やけど付与は「使用者自身が行動可能かどうかとは無関係に必ず判定される」効果である
（`docs/spec/turn.md` 出典の一次情報より「使用者がこんらんやまひ状態などにより行動できない
可能性があるときでも、加熱は必ずできる」）。`core/move_executor.py` の `run_move` では
`Event.ON_TRY_ACTION` の結果が False の場合、それ以降の `_execute_move`（`ON_MOVE_CHARGE`・
`ON_TRY_MOVE_1` 等を含む）は一切実行されない。そのため `ON_TRY_MOVE_1` にハンドラを置くと、
まひ・ねむり・こおり・こんらん等で使用者が行動できない場合にやけど判定自体が行われず、
一次情報と矛盾する。

これを避けるため、`ON_TRY_ACTION` の中でも最も早い段階（`docs/spec/turn.md` の
`ON_CHECK_ACTION`＝実装上の `Event.ON_TRY_ACTION` において最小の priority=10 グループより前）
にハンドラを置く。priority=5 は既存の「行動可否判定より前に必ず処理を行う」パターン
（`かえんぐるま_thaw_attacker` / `かえんボール_thaw_attacker` / `もえつきる_thaw_attacker`、
いずれも `ON_TRY_ACTION` priority=5、こおりの回復判定 priority=10 より前に解凍する）と
同じ区分を採用する。これにより、まひ（priority=120）・こおり（priority=10）・こんらん
（priority=110）等のいずれのハンドラが `stop_event=True` を返した場合でも、それより前の
priority=5 で本ハンドラは必ず実行される。

## 注意点・エッジケース（review-test 向け）

1. **ぼうごパット**: 接触技の使用者がぼうごパットを持つ場合、やけど効果は発動しない
   （`is_contact_reaction` が `ON_CHECK_CONTACT_REACTION` を経由して判定するため、既存の
   ぼうごパット実装により自動的に反映される）。
2. **やけど耐性（ほのおタイプ等）**: `ailment_manager.apply` 内の汎用免疫判定により、
   ほのおタイプの攻撃者はやけどにならない。
3. **みがわり**: くちばしキャノンの使用者自身がみがわり状態の場合、接触技をみがわりが
   肩代わりすると `actual_damage <= 0` となり `contact_hitter` が記録されないため、
   やけど効果は発動しない（一次情報と一致）。
4. **相手の優先度がより低い場合**: カウンター（-5）・ドラゴンテール（-6）等、くちばしキャノン
   （-3）より優先度の低い技を相手が使う場合、使用者の `ON_TRY_ACTION` 実行時点では
   まだ相手が行動しておらず `contact_hitter` が記録されていないため、やけど効果は発動しない
   （一次情報と一致）。
5. **まひ・ねむり・こおり・こんらん等で行動できない場合**: 上記の priority 変更により、
   使用者がこれらの状態で行動できない場合でも、やけど効果は正しく発動する。
6. **既知の未対応エッジケース（ひんし時のやけど付与）**: 使用者が接触技でひんしになった
   場合でも本来はやけど効果が発動する仕様だが、`turn_controller.py` の `_run_move_phase` は
   `attacker.fainted` の場合そのポケモンの `run_move` 自体を呼び出さないため、
   `ON_TRY_ACTION` に登録した本ハンドラも実行されず、やけど効果が発動しない。
   この挙動を正しく再現するには「技を選択した時点（ターン開始時）で加熱状態を示すフラグを
   立て、`ON_DAMAGE_HIT` で常時監視する」という、`ON_BEFORE_ACTION`（現状未実装）に相当する
   ターン開始前フックの新設が必要となり、影響範囲が本レビュー対象を超える大きな設計変更と
   なる。発生条件が「同一ターンに、くちばしキャノンの使用者が接触技でちょうど瀕死になる」
   という極めて限定的な状況に限られるため、`きあいパンチ`（一撃必殺技＋耐え系アイテムの
   例外）と同様、既知の未対応エッジケースとして本ファイル・`docs/spec/moves/くちばしキャノン.md`
   に明記し、対応を見送る。
7. **さいはい／まねっこ／オウムがえし**: 一次情報では「加熱動作は技の使用とみなされない」
   ため、これらの技で技Aの代わりにくちばしキャノンが選ばれることはない、という仕様がある。
   ただし `さいはい`／`まねっこ`／`オウムがえし` 自体が本実装では未実装のため、対応不要
   （それぞれの技を実装する際に、直前に使用した技を参照する既存ロジックで自然に満たされる
   想定）。
8. **テスト方針**:
   - 既存: `test_くちばしキャノン_接触技を受けると攻撃者がやけどになる` /
     `test_くちばしキャノン_非接触技を受けてもやけどにならない`
   - 追加: `test_くちばしキャノン_ぼうごパットを持つ相手はやけどにならない` /
     `test_くちばしキャノン_ほのおタイプの相手はやけどにならない` /
     `test_くちばしキャノン_まひで行動不能でもやけどは発動する` /
     `test_くちばしキャノン_みがわりで防いだ場合はやけどにならない` /
     `test_くちばしキャノン_相手の優先度がより低い場合はやけどにならない`
