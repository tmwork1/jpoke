# 計画書: おだてる

## 仕様要約

- あくタイプ / 変化 / 命中100 / PP15 / 対象: 1体選択
- 相手の『とくこう』ランクを1段階上げる
- 相手を1〜4ターンの間『こんらん』状態にする
- 効果の成否判定: とくこうが+6でこんらん済みなら失敗（`ON_APPLY_MOVE` priority 100: `いばる等: ランク+6でこんらん`）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | とくこう+1、こんらん付与 |

## Priority 根拠

`docs/spec/turn.md` の `ON_APPLY_MOVE` に `いばる等: ランク+6でこんらん` が priority 100 で記載されており、いばると同じカテゴリの技として扱われる。ハンドラ本体は既存の `ON_STATUS_HIT` priority 100 のパターンに従う。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `おたけび` 関連の後、`おにび` の前（五十音順）。

```python
def おだてる_modify_defender_stats_and_apply_volatile(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """おだてるの効果: 相手のとくこうを1段階上げ、相手をこんらん状態にする（2〜5ターン）。"""
    battle.modify_stats(ctx.defender, {"C": 1}, source=ctx.attacker)
    count = battle.random.randint(2, 5)
    battle.volatile_manager.apply(ctx.defender, "こんらん", count=count, source=ctx.attacker, ctx=ctx)
    return HandlerReturn(value=value)
```

### data/move.py への更新

`"おだてる"` エントリのハンドラを追加する。

## 注意点・エッジケース

- いばると同パターン。失敗判定（とくこう+6かつこんらん済み）はフレームワーク側（`ON_APPLY_MOVE` priority 100）が処理するため、ハンドラ内にガードは不要
- いばるとの違いはランクアップする能力（こうげき→とくこう）と段数（2段→1段）のみ
- あくタイプのため、いたずらごころによる先制は `ON_TRY_MOVE_2` priority 120 でフレームワーク側が無効化する
