# トリプルダイブ 実装計画書

## 仕様要約（docs/spec/moves/トリプルダイブ.md 参照）

* タイプ: みず / 分類: 物理 / 威力: 30（1ヒットあたり、固定） / 命中: 95 / PP: 10
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果:
  * 3回連続で攻撃する固定回数の連続攻撃技（命中判定は1発目のみ、各ヒット威力は固定30）。
  * 1発目が当たれば攻撃は必ず3回続く（どちらかがひんしになった時点で中断する連続攻撃技全般の仕様）。
  * 追加効果なし。

## 現状の実装状況

`src/jpoke/data/moves/move_ta.py` に既に登録済み。

```python
"トリプルダイブ": MoveData(
    type="みず",
    category="physical",
    pp=10,
    power=30,
    accuracy=95,
    flags={"contact"},
    multi_hit={
        "min": 3,
        "max": 3,
        "check_hit_each_time": False,
        "power_sequence": (),
    },
    handlers={},  # 追加効果なし
),
```

内容は仕様書と一致しており、修正不要。

* `multi_hit.min == multi_hit.max == 3` かつ `flags` に `check_hit_each_time` を含まないため、
  `core/move_executor.py` の `_resolve_hit_count` は常に3を返し、命中判定は1発目のみ行われる
  （`すいりゅうれんだ` と同一パターン）。
* `power_sequence` が空タプルのため、`_resolve_hit_power` は各ヒットとも `move.power`（30）を返す。
* `critical_rank` の指定なし（確定急所ではない、通常の急所判定。各ヒットごとに独立して判定される）。

## 注意点・エッジケース（review-test 向け）

1. **命中判定は1発目のみ**
   - `check_hit_each_time` フラグを持たないため、1発目が外れた場合は残りのヒットも発生しない。
     逆に1発目が当たれば、2発目・3発目は命中判定を行わずに必ず命中する
     （`トリプルキック`・`トリプルアクセル` とは異なる挙動）。
2. **急所判定は通常仕様**
   - `すいりゅうれんだ` と異なり確定急所ではない。各ヒットごとに独立した急所判定が行われる
     （`docs/spec/moves/_multi_hit.md` の「7.1 急所判定」）。
3. **テスト方針**
   - `_resolve_hit_count` が常に3を返すことを確認するテスト。
   - `_check_hit` を差し替えて2発目以降が外れる条件にしても、1発目さえ当たれば
     3ヒットとも発生することを確認するテスト（命中判定1発目のみの検証）。
   - ダメージが発生することを確認する基本テスト。
