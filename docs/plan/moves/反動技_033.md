# 計画書: 反動技（反動量 1/3） — 反動技_033

## 仕様要約

`docs/spec/moves/_move_secondary_2.md` より：与えたダメージの **1/3** を自分が反動として受ける技。

- 反動量: `max(1, round_half_down(damage / 3))`
  （端数は五捨五超入で丸める。第五世代以降の反動ダメージ計算仕様に合わせる。`int()` による切り捨てでは
  端数が0.5超の場合に反動量が1不足するため、`round_half_down` を使用する）
- 反動は `reason="recoil"` で処理する（いしあたま特性による無効化のため）
- フレアドライブは既存の追加効果ハンドラ（10%やけど、`ON_DAMAGE_HIT`）を維持したまま `ON_HIT` に反動ハンドラを追加する

def ウェーブタックル_recoil(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    recoil = max(1, round_half_down(value / 3))

## 実装上の補足（レビュー時の修正）

- 共通ヘルパー `_recoil`（`handlers/move_attack.py`）は `int()` による切り捨てを採用していたが、
  ポケモンWiki「すてみタックル」ページの「技の仕様」に記載の通り、反動ダメージの端数処理は
  第五世代以降 `round_half_down`（五捨五超入）が正しい。すてみタックルのレビュー時に
  `_recoil` を `round_half_down` を使うよう修正した（この修正は同ヘルパーを使う
  反動技_025・反動技_05 グループの技にも適用される）。

挿入位置の目安（五十音順）：
- `ウェーブタックル_recoil` — `ウッドハンマー_recoil` の直前（「ウェ」→「ウッ」順）
- `ウッドハンマー_recoil` — `ウッドホーン` 系ハンドラ付近の直後
- `すてみタックル_recoil` — 「す」行ハンドラ付近
- `フレアドライブ_recoil` — 既存 `フレアドライブ_apply_ailment_to_defender` の直後
- `ブレイブバード_recoil` — 「ブ」行ハンドラ付近

## 注意点・エッジケース

- `reason="recoil"` を指定することで、いしあたま特性が実装された際に自動的に無効化される（ハンドラ側で追加のガードは不要）
- `わるあがき_self_damage` は最大HP比（`r=-1/4`）で計算するが、反動技は与ダメ絶対値（`v=-recoil`）で計算する点が異なる
- フレアドライブの追加効果（やけど）は `ON_DAMAGE_HIT`（priority=10）、反動は `ON_HIT`（priority=100）で発火する。`ON_DAMAGE_HIT` が先に発火されるため、やけど付与 → 反動ダメージの順となる
- `ON_HIT` の `value` パラメータは与えたダメージ量を保持する。みがわり人形に当たった場合も `ON_HIT` は発火するが（仕様書「ダメージ発生後の処理（みがわりに被弾しても発動）」）、みがわりへのダメージ量が `value` として渡されるため計算は正常に機能する
- 攻撃側がすでにひんし（HP=0）になっていた場合でも反動処理が呼ばれる可能性があるが、`battle.modify_hp` 内部で HP ≤ 0 の処理は manager が担当するため、ハンドラ側での alive ガードは不要
- 反動ダメージ最小値は `max(1, ...)` により 1 で固定される（ダメージが 0 でも 1 になる）
