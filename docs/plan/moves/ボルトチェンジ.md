# ボルトチェンジ 実装計画書

## 仕様要約（docs/spec/moves/ボルトチェンジ.md 参照）

* タイプ: でんき / 分類: 特殊 / 威力: 70 / 命中: 100 / PP: 20
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果: 攻撃後、使用者が手持ちのポケモンと交代する（追加効果とはみなされない）。
* とんぼがえりのでんきタイプ版であり、判定・交代条件は共通（`docs/plan/moves/とんぼがえり.md` 参照）。

## 現状の実装状況

`src/jpoke/data/moves/move_ha.py` に既に登録済み。

```python
"ボルトチェンジ": MoveData(
    type="でんき",
    category="special",
    pp=20,
    power=70,
    accuracy=100,
    handlers={
        Event.ON_HIT: h.MoveHandler(ha.pivot)
    }
),
```

`flags` に `contact` を含まない点も仕様（非接触技）と一致しており、修正不要。

## ハンドラ構成

| Event | 関数 | subject_spec | priority | 役割 |
|---|---|---|---|---|
| ON_HIT | `pivot` | attacker:self | 100（デフォルト） | 交代可能な控えがいれば `player_states[player].interrupt = Interrupt.PIVOT` を設定 |

`pivot`（`handlers/move_attack.py`）は `とんぼがえり` と共通の実装であり、専用の新規関数は不要。
実際の交代処理は `SwitchManager.run_interrupt_switch(Interrupt.PIVOT)`
（`turn_controller.py` の `_run_move_phase` 内で実行）が担う共通インフラであり、本技側の
実装には含まれない。

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_HIT` 節（ダメージ発生後の処理、みがわりに被弾しても発動）を確認した。
一覧に交代技専用の行は明記されていないが、既存の `とんぼがえり` も同じ `pivot` 関数をデフォルト
priority（100）で `Event.ON_HIT` に登録しており、他の priority=100 の処理とは独立した状態変更
（`interrupt` フラグ設定のみ）であるため相互作用がなく、priority の新規検討は不要。
既存パターンを踏襲しデフォルト値のままとする。

## でんきタイプ固有の考慮事項

1. **じめんタイプへの無効化・ひらいしん/ちくでん/でんきエンジンによる吸収**
   - これらは `Event.ON_BEFORE_APPLY_MOVE`（型無効判定・特性による無効化）で技自体を
     不発にするため、`Event.ON_HIT` は発火せず `pivot` も実行されない。既存の汎用実装
     （型相性チェック・各特性ハンドラ）でカバー済みであり、本技固有の追加実装は不要。

## 注意点・エッジケース（review-test 向け、とんぼがえりと共通）

1. **控えがいない場合**
   - `pivot` は `battle.command_manager.get_available_switch_commands(player)` が空の場合、
     interrupt を設定しない。ダメージは通常通り与えられるが交代は発動しない。実装済み・動作確認済み。

2. **まもる・命中しなかった場合**
   - `Event.ON_BEFORE_APPLY_MOVE` で無効化される、または命中判定に失敗した場合は
     `Event.ON_HIT` 自体が発火しないため、交代も発動しない。既存の move_executor.py の
     フロー（本技固有の実装ではない）に依存しており、対応済み。

3. **対象を倒した場合の交代順序**
   - `Event.ON_HIT` は攻撃側の interrupt を設定し、`_run_move_phase` 内で
     攻撃側の PIVOT 交代→（ターン終了時の）防御側の瀕死交代、の順に処理されるため、
     「ボルトチェンジの交代先を見せた後に相手が次のポケモンを選択する」という仕様と一致する。

4. **じめんタイプ・ひらいしん等による無効化**
   - 上記「でんきタイプ固有の考慮事項」を参照。既存の汎用実装で対応済み。
