# 計画書: パワースワップ

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 12（`docs/champions/move_list.txt` 準拠。本家Gen9基準は10）/
  命中: -（必中。`accuracy=None`） / 対象: 1体選択
- マジックコートで跳ね返されない（`flags={"unreflectable"}`）、みがわりを貫通する
  （`flags={"bypass_substitute"}`）
- 自分と相手の「こうげき」「とくこう」のランク変化を入れ替える
- 実数値（種族値・個体値・努力値由来のステータス）は変化しない
- ランク変化のみを互いに交換する

| `data/moves/move_ha.py` の `"パワースワップ"` エントリ | 実装済み（`pp=12`, `accuracy=None`,

| `handlers/move_status.py` の実装関数 | 実装済み（`パワースワップ_swap_ranks`） |

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `Event.ON_STATUS_HIT` | 100（デフォルト） | `"attacker:self"` | 攻撃側・防御側の A/C ランクを入れ替える |

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載はない（ダメージ計算内部イベントの扱い）。

類似ハンドラである `ガードスワップ`（`ぼうぎょ`・`とくぼう` ランク交換）および `スピードスワップ`（`すばやさ` 実数値交換）がいずれも `priority=100`（`MoveHandler` のデフォルト）で登録されているため、同じ 100 を採用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

`ガードスワップ_swap_ranks` の後（五十音順: カ行の後、パ行に配置）に以下を追加する。

```python
def パワースワップ_swap_ranks(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """パワースワップ: 使用者と相手のこうげき・とくこうのランク変化を入れ替える。

    実数値は変化せず、ランク変化のみを互いに入れ替える。
    """
    atk_rank = ctx.attacker.rank
    def_rank = ctx.defender.rank
    for stat in ("atk", "spa"):
        atk_rank[stat], def_rank[stat] = def_rank[stat], atk_rank[stat]
    return HandlerReturn(value=value)
```

### `src/jpoke/data/moves/move_ha.py`

`"パワースワップ"` エントリに PP・必中フラグ・マジックコート/みがわり用フラグを設定する。

```python
    "パワースワップ": MoveData(
        type="エスパー",
        category="status",
        pp=12,
        accuracy=None,  # 必中
        # マジックコートで跳ね返されず、みがわりを貫通する
        flags={"unreflectable", "bypass_substitute"},
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.パワースワップ_swap_ranks,
            ),
        }
    ),
```

## 注意点・エッジケース（review-test 向け）

- **ランク変化のみ**: `ガードスワップ` と同様に実数値（`_stats_manager.stats`）は変化させない。`スピードスワップ` は実数値を交換するが、パワースワップは別仕様である点に注意。
- **両ランクが +0 の場合**: 入れ替え後も両者 +0 のまま。失敗にならない（仕様通り）。
- **両ランクが同値の場合**: 入れ替えても変化なし。失敗にならない（仕様通り）。
- **ランク上限/下限**: ランク変化の入れ替えなので、+6 や -6 がそのまま相手に移る。ランク変化の範囲を超える処理は発生しない。
- **みがわり**: `ON_STATUS_HIT` はデフォルトでは `みがわり` に阻まれるため、`flags={"bypass_substitute"}` を明示的に設定しないと貫通しない。
- **マジックコート**: 対象が `foe` の変化技はデフォルトで跳ね返り対象になるため、`flags={"unreflectable"}` を明示的に設定しないと跳ね返されてしまう。
- **必中**: `accuracy=100` のままだと ON_MODIFY_ACCURACY 経由で回避率ランク補正の影響を受けてしまうため、`accuracy=None` を設定する必要がある。
- **PP**: `docs/champions/move_list.txt` 準拠で 12（本家Gen9基準の 10 ではない）。
- **ガードスワップとの比較**: ガードスワップ（B/D）、パワースワップ（A/C）、ハートスワップ（全能力）という整合関係がある。テストでは 3 種の組み合わせで誤実装がないか確認するとよい。
- **subject_spec**: `MoveHandler` のデフォルト `"attacker:self"` を使用。`AttackContext` 内で `ctx.attacker` / `ctx.defender` を直接参照するため、subject_spec はマッチング用途でのみ機能し、対象選択には影響しない。
