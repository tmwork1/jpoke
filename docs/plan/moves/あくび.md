# あくび 実装計画書

## 仕様要約

- 相手を **ねむけ** 揮発性状態にする変化技
- 命中判定なし（必中）、みがわり・しんぴのまもりには防がれる
- 失敗条件: 対象がすでにねむけ状態 OR 状態異常を持っている

## 実装方針

`ねむけ` 揮発性状態はすでに `data/volatile.py` / `handlers/volatile.py` に実装済みのため、
あくびは **ねむけを付与するトリガー** の部分だけを実装する。

### ハンドラ構成

| イベント | Priority | 役割 |
|----------|----------|------|
| `ON_BEFORE_APPLY_MOVE` | 130 | 失敗チェック：ねむけ状態 OR 状態異常なら失敗 |
| `ON_STATUS_HIT` | 100（デフォルト） | ねむけ揮発性状態を相手に付与 |

### Priority 根拠

- `ON_BEFORE_APPLY_MOVE priority=130`：`docs/spec/turn.md` の「あくび: すでにねむけ・状態異常」が priority=130 に明記されている

### subject_spec

- `ON_BEFORE_APPLY_MOVE`: `attacker:self`（MoveHandlerデフォルト）
- `ON_STATUS_HIT`: `attacker:self`（MoveHandlerデフォルト）

どちらも技の攻撃フロー（`AttackContext`）の `ON_STATUS_HIT` / `ON_BEFORE_APPLY_MOVE` で発火され、`ctx.attacker`（使用者）・`ctx.defender`（対象）を参照する。
変化技であっても MoveHandler が登録するイベントは攻撃フロー（`core/move_executor.py`）上で発火するため `AttackContext` を使う点に注意（`EventContext` は揮発性状態・状態異常・特性等の非 MoveHandler 系ハンドラで使用する）。
MoveHandler はデフォルトで `attacker:self` / `skip_subject_check=True`。

## 実装箇所

### handlers/move_status.py（五十音順: あくびは「あ」行、いちゃもんの前後）

```python
def あくび_can_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """あくびの失敗チェック: 対象がねむけ状態または状態異常を持っている場合は失敗する。"""
    mon = ctx.defender
    if mon.has_volatile("ねむけ") or mon.ailment.is_active:
        battle.add_event_log(
            ctx.attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="すでに状態異常になっている"),
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def あくび_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """あくびの効果: 相手をねむけ状態にする。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="ねむけ", count=2)
```

`count=2` の根拠: `tick_volatile` は使用ターン自体のターン終了時にも1回消費されるため、
「次のターン終了時にねむり状態にする」という仕様（`docs/spec/volatiles/ねむけ.md`）を満たすには
count を2で開始する必要がある（count=1 だと使用ターンの終了時点でねむりに移行してしまう）。

### data/move.py（既存の「あくび」MoveData エントリーにハンドラを追加）

```python
"あくび": MoveData(
    type="ノーマル",
    category="status",
    pp=12,
    handlers={
        Event.ON_BEFORE_APPLY_MOVE: h.MoveHandler(
            hs.あくび_can_apply,
            priority=130,
        ),
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.あくび_apply,
        ),
    }
),
```

PP は `src/jpoke/data/ps-champ-ja/moves.json` の値（12）を正とする。

## 注意点・エッジケース（review-test 向け）

1. **ねむけ付与後もねむりを防ぐケース**：やるき / ふみん / ぜったいねむり / スイートベール等の特性チェックは `ailment_manager.apply` 内で行われるため、あくびハンドラ側では不要。
2. **さわぐとの相互作用（第五世代以降）**：あくびは成功してねむけを付与できる。ねむけ解除時にさわぐ状態があればねむりにならないが、これは `さわぐ_prevent_sleep` が担当済み。
3. **すでに別の状態異常がある場合**：`ON_BEFORE_APPLY_MOVE` で `mon.ailment.is_active` が真であれば失敗させる。
4. **みがわり・しんぴのまもり**：`apply_volatile_to_defender` の内部（`volatile_manager.apply` → `ON_BEFORE_APPLY_VOLATILE`）で既存のガードが働くため、ハンドラ側での追加チェック不要。
5. **命中率**：MoveData に `accuracy` を設定しない（必中）。
