# 実装計画: おたけび

## 仕様サマリ

- タイプ: ノーマル / 分類: 変化 / PP: 30 / 命中率: 100
- 効果: 相手のこうげき・とくこうランクを 1 段階ずつ下げる
- 音系の技（sound ラベル）: 相手のみがわり状態を貫通する
- まもる: ○ / マジックコート: ○ / 直接攻撃: ×
- 対象: 1体選択

## 現状の実装状況

`data/move.py` にエントリは存在するが `handlers` キーが未設定（ランク低下が未実装）。

```python
"おたけび": MoveData(
    type="ノーマル",
    category="変化",
    pp=30,
    accuracy=100,
    labels=["sound"],
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手のこうげき・とくこうを 1 段階ずつ下げる |

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の同種ハンドラ（`なきごえ_modify_defender_stats`（A-1）、`なみだめ_modify_defender_stats`（A-1, C-1）など）が priority=100 デフォルトで実装されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: 五十音順で `えんまく_*` の後、`おにび_*` の前（お行内）

```python
def おたけび_modify_defender_stats(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """おたけびの効果: 相手のこうげきととくこうを 1 段階ずつ下げる。音系の技のためみがわりを貫通する。"""
    return modify_defender_stats(battle, ctx, value, stats={"A": -1, "C": -1})
```

### data/move.py の更新

`おたけび` の MoveData に `handlers` を追加する。

```python
"おたけび": MoveData(
    type="ノーマル",
    category="変化",
    pp=30,
    accuracy=100,
    labels=["sound"],
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.おたけび_modify_defender_stats,
        ),
    }
),
```

## 注意点・エッジケース

- A と C の 2 種類のステータスを同時に下げる。`modify_defender_stats` の `stats` 引数に辞書で両方渡せばよい
- sound ラベルにより `みがわり` 状態を貫通して命中する（みがわりへの当否はフレームワーク側が処理）
- ランク低下の無効化（しろいきり、クリアボディなど）はフレームワーク内で処理されるため、ハンドラ側のガードは不要
- `なきごえ`（A-1 のみ）・`なみだめ`（A-1, C-1、貫通あり）と仕様が一部重なるが、おたけびは音系でみがわり貫通かつ両攻撃能力を下げる点が特徴
