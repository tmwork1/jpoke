# 実装計画: おたけび

参照: docs/spec/moves/おたけび.md

## 仕様サマリ

- タイプ: ノーマル / 分類: 変化 / PP: 20（`src/jpoke/data/ps-champ-ja/moves.json` 準拠。本家PPは30） / 命中率: 100
- 効果: 相手のこうげき・とくこうランクを 1 段階ずつ下げる
- 音系の技（sound フラグ）: 相手のみがわり状態を貫通する
- まもる: ○（防がれる） / マジックコート: ○（反射される） / 直接攻撃: ×
- 対象: 1体選択

`src/jpoke/data/ps-champ-ja/moves.json` の基準値（20）に修正する必要がある。

"おたけび": MoveData(

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の同種ハンドラ（`なきごえ_modify_defender_stats`（atk -1 のみ）、
`なかよくする_modify_defender_stats`（atk -2 のみ）など）が
priority=100 デフォルトで実装されているため、同様に採用する。

## 実装箇所（実装済み）

### handlers/move_status.py

`おきみやげ_apply` の後、`おだてる_apply` の前（お行内）に実装済み。

```python
def おたけび_modify_defender_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """おたけびの効果: 相手のこうげきととくこうを 1 段階ずつ下げる。音系の技のためみがわりを貫通する。"""
    return modify_defender_stats(battle, ctx, value, stats={"atk": -1, "spa": -1})
```

### data/moves/move_a.py の修正

`pp` を `30` から `20` に修正する（`src/jpoke/data/ps-champ-ja/moves.json` 準拠）。

## 注意点・エッジケース

- こうげき・とくこうの 2 種類のステータスを同時に下げる。`modify_defender_stats` の `stats` 引数に
  辞書で両方（`{"atk": -1, "spa": -1}`）渡せばよい
- sound フラグにより `みがわり` 状態を貫通して命中する（みがわりへの当否はフレームワーク側が処理）
- ランク低下の無効化（しろいきり、クリアボディ、かいりきバサミなど）はフレームワーク内の
  `battle.modify_stats` が処理するため、ハンドラ側のガードは不要
- `なきごえ`（atk -1 のみ）・`すてゼリフ`（atk -1, spa -1 に加えて交代を伴う）と仕様が一部重なるが、
  おたけびは音系でみがわり貫通かつ両攻撃能力を下げる点が特徴
