# 計画書: ゆびをふる

## 仕様要約

* ノーマルタイプ / 変化 / 威力- / 命中-（必中） / PP10 / 対象: 不定 / 優先度0 / 接触なし
* 一部を除いたほぼ全ての技の中からランダムに1つを選び、その技をゆびをふるの代わりに繰り出す。
  選ばれた技の優先度に関係なく、ゆびをふる自体の優先度で繰り出される。
* 詳細は `docs/spec/moves/ゆびをふる.md` を参照。

## PPについて

`docs/champions/move_list.txt` に「ゆびをふる」のエントリが存在しない
（同様に「ものまね」「スケッチ」「オウムがえし」「よこどり」「さきどり」「ねこのて」
「しぜんのちから」「わるあがき」など、他の技を呼び出す・コピーする系の複雑技の多くも
未掲載）。README記載の対象範囲規則「技・特性・アイテムなどの仕様ソースは第9世代を基本とし、
チャンピオンズ側でルールが異なる場合はそちらを優先する」に従い、チャンピオンズ側の記載が
ない本技は第9世代基準のPP10をそのまま採用する。

なお `docs/champions/changes_from_sv.md` は「すべての技のPPが8・12・16・20の4段階に統一」と
記載しているが、旧PPから新PPへの変換は単純な一律の変換式ではなく技ごとに個別に定められている
ことを確認した（例: 旧PP10の技でも、へんしん/じしん/ゆきなだれ等は12になる一方、はねやすめは
旧PP相当から8になっている）。move_list.txtに記載のない本技について新PP値を推測することは
できないため、確認可能な第9世代の値（10）を維持する。将来move_list.txtに本技の記載が追加され
次第、その値に修正すること。

## 実装保留の判断

本技の完全な実装には、以下の大規模な変更が必要となる。

1. 「一部を除いたほぼ全ての技の中からランダムに1つを選ぶ」ための除外リスト
   （他の技が出る技・まもる系統の技・行動順や反撃に関する技・専用技の一部等。
   詳細は `docs/spec/moves/ゆびをふる.md`「選ばれる技の範囲」参照）を保持し、選択された
   技を実行する仕組み。
2. 選ばれた技をその場で「ゆびをふるの代わりに」実行する、技の実行フロー全体を動的に
   差し替える汎用的な機構。現在の `Battle` / `MoveExecutor`（`src/jpoke/core/move_executor.py`）
   はコマンドで指定された技をそのまま実行する前提で作られており、実行時に別の技へ動的に
   差し替える仕組みは存在しない。同様の機構は `おうむがえし`・`ねごと`・`さきどり` 等の
   「他の技を呼び出す技」とも一部共有されうるが、ゆびをふるは対象範囲が「ほぼ全ての技」で
   あり除外リストの管理コストが最も高い。
3. 選ばれた技の優先度に関係なく、ゆびをふる自体の優先度で繰り出す必要がある（ターン順序への
   影響）。
4. 選ばれた技がゆびをふる自体の代わりに実行されるため、アンコール・ねごと・ねこのて・
   さいはい・ものまね等、既存の多くの技・状態変化との相互作用を個別に見直す必要がある。

上記の理由から、へんしん・スケッチ等の前例（`docs/progress/move.md` で実装保留として扱われて
いる複雑技）と同様に、本技は実装保留として扱う。

## data/moves/move_ya.py の修正内容

```python
"ゆびをふる": MoveData(
    type="ノーマル",
    category="status",
    pp=10,  # champions_move_list.txtに記載なし。第9世代の値を採用（docs/plan/moves/ゆびをふる.md参照）
    flags={"non_negoto", "non_copycat"},  # まねっこでコピー不可（第二世代以降一貫して×）
    # 実装保留: ほぼ全ての技の中からランダムに1つを選び、その場で実行する大規模な機構が
    # 必要なため対応を見送る。詳細は docs/plan/moves/ゆびをふる.md 参照（前例: へんしん・スケッチ）。
    handlers={},
),
```

* `pp`: 10のまま維持する（上記「PPについて」参照）。
* `flags={"non_negoto"}`: 第三世代以降ねごとで選ばれない仕様と一致しているため変更なし。
* `flags`に`non_copycat`を追加する。まねっこ（Copycat）は既に`handlers/move_status.py`の
  `まねっこ_can_use`で`non_copycat`フラグを持つ技のコピーを拒否する実装があり、
  ポケモンWiki「ゆびをふるで出る技」表でも「まねっこ」の行は第二世代以降一貫して×
  （ゆびをふるが選ばれない＝逆にまねっこがゆびをふるをコピーできない仕様）となっているため、
  フラグ未設定のままだと現行実装のまねっこがゆびをふるを誤ってコピーしてしまう
  （実装保留のため何も起きないだけだが、意図しない成功ログ等が出てしまう）不具合になる。
* さいはいに対する失敗は `handlers/move_status.py` の `_INSTRUCT_BLOCKED_MOVES` に
  「ゆびをふる」が既に含まれており、追加対応は不要。

## 将来実装時の設計メモ

* `Battle` / `MoveExecutor` に「実行中の技を動的に差し替える」ための公開APIを追加し、
  ゆびをふる・おうむがえし・ねごと・さきどり等の「他の技を呼び出す技」群から共通で
  呼び出せる設計が望ましい。
* 除外リスト（選ばれない技の一覧）は `handlers/move_status.py` の
  `_INSTRUCT_BLOCKED_MOVES` のような `frozenset` として管理し、将来のねこのて等の実装と
  共有できるようにする。
* 選ばれた技の優先度を無視してゆびをふる自体の優先度で行動する点は、ターン順決定
  （`core/turn_controller.py`）側の実装と整合させる必要がある。
