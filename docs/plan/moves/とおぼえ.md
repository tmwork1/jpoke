# 実装計画書: とおぼえ

## 仕様要約

- タイプ: ノーマル / カテゴリ: 変化 / PP: 40 / 対象: 味方全体（sound ラベルあり）
- 自分と味方全体の「こうげき」ランクを 1 段階ずつ上げる
- 第 8 世代以降は音系の技（sound）であり、「ダイウォール」は貫通しない
- **シングルバトルでは味方がいないため、自分の「こうげき」を 1 段階上げるだけ**

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「と」行 — `どくのいと_*` 系の後、`とぐろをまく_modify_attacker_stats` の前（「とお」は「どく」より前）。実際の配置は「と」行は「ど」よりも前の濁音なし行なので、`トーチカ_apply_volatile_to_attacker` の後、`どくどく_apply_ailment_to_defender` の前（平仮名五十音順）に挿入する。

```python
def とおぼえ_modify_attacker_stats(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """とおぼえの効果: 自分のこうげきを 1 段階上げる（シングルバトルでは自分のみ）。"""
    return modify_attacker_stats(battle, ctx, value, stats={"A": 1})
```

### `src/jpoke/data/move.py`

既存の `"とおぼえ"` エントリに handlers を追加する。

```python
"とおぼえ": MoveData(
    type="ノーマル",
    category="変化",
    pp=40,
    labels=["sound"],
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.とおぼえ_modify_attacker_stats,
        ),
    }
),
```

## 注意点・エッジケース

- 対象が「味方全体」だがシングルバトルのみ対象のため、自分（attacker）に対する A+1 のみ実装する
- 音系の技ラベル（sound）はすでに `data/move.py` に設定済みのため、ハンドラ側では不要
- `modify_attacker_stats` で A+1 を渡すと、A ランクが +6 の場合は `modify_stats` 内部の `clamp_stats` がガードするため、ハンドラ側での最大チェックは不要
