# 計画書: 反動技（反動量 1/2） — 反動技_05

## 仕様要約

`docs/spec/moves/_move_secondary_2.md` より：与えたダメージの **1/2** を自分が反動として受ける技。

- 反動量: `max(1, int(damage / 2))`
- 反動は `reason="recoil"` で処理する（いしあたま特性による無効化のため）
- はめつのひかりは未登録のため MoveData を新規追加する
- もろはのずつきは登録済みだが `labels=["contact"]` のみで recoil ラベルなし・ハンドラなし

## 実装箇所

### handlers/move_attack.py に追加する2関数

五十音順に従い「は行」の適切な位置に挿入する。

```python
def はめつのひかり_recoil(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    recoil = max(1, int(value / 2))
    battle.modify_hp(ctx.attacker, v=-recoil, reason="recoil", source=ctx.attacker)
    return HandlerReturn(value=value)


def もろはのずつき_recoil(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    recoil = max(1, int(value / 2))
    battle.modify_hp(ctx.attacker, v=-recoil, reason="recoil", source=ctx.attacker)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース

- `reason="recoil"` を指定することで、いしあたま特性（`ON_MODIFY_NON_MOVE_DAMAGE` で `hp_change_reason == "recoil"` を検査）が自動的に反動を無効化する
- `わるあがき_self_damage` は最大HP比（`r=-1/4`）、反動技は与ダメ絶対値（`v=-recoil`）で計算する点が異なる
- `max(1, ...)` によりダメージが2以下の場合でも最低1の反動が発生する
- はめつのひかりは接触技ではないため `labels` に `"contact"` は付けない（仕様書に contact 記載なし）
- もろはのずつきは接触技（`"contact"` を維持）かつ反動技（`"recoil"` を追加）
- `すてみ` 特性は `has_label("recoil")` を参照するため、両技に `labels=["recoil"]` を付けることで自動的に威力1.2倍の恩恵を受ける
- turn.md では「反動技による反動ダメージ」は `Event.ON_DAMAGE` のセクションに記載されているが、いのちのたまの反動等と同様に実装上は `Event.ON_HIT` に登録する（既存計画書・実装パターンに準拠）
- 攻撃側がひんしになっても反動処理はハンドラ内で `modify_hp` を呼ぶだけであり、ひんし判定は battle 側で行うため、ハンドラ側に alive ガードは不要
