# 実装計画: しびれごな

参照: `docs/spec/moves/しびれごな.md`

## 仕様サマリ

- タイプ: くさ / 分類: 変化 / PP: 20（Pokémon Champions基準。`docs/champions/move_list.txt` 準拠） / 命中率: 75
- 効果: 相手をまひ状態にする
- 粉技（powder ラベル）: でんきタイプ・くさタイプ・特性ぼうじん・ぼうじんゴーグル所持ポケモンには無効
- まもる: ○ / マジックコート: ○ / 直接攻撃: ×
- 対象: 1体選択

## 現状の実装状況

`data/moves/move_sa.py` に実装済み。ただし PP が Gen9本家基準の 30 のままになっており、
Pokémon Champions 基準（`docs/champions/move_list.txt`）の 20 への修正が必要。

```python
"しびれごな": MoveData(
    type="くさ",
    category="status",
    pp=30,  # ← 20 に修正が必要
    accuracy=75,
    flags={"powder"},

        Event.ON_STATUS_HIT: h.MoveHandler(

    }

```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | まひ状態を相手に付与する |

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` の記載なし。
既存の同種ハンドラ（`でんじは_apply_ailment_to_defender`、`へびにらみ_apply_ailment_to_defender`）が priority=100 デフォルトで実装されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: 五十音順で `じこさいせい_*` と `しっぽをふる_*` の間

```python
def しびれごな_apply_ailment_to_defender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """しびれごなの効果: 相手をまひ状態にする。"""
    return apply_ailment_to_defender(battle, ctx, value, ailment="まひ")
```

### data/moves/move_sa.py の更新

## 注意点・エッジケース

- でんきタイプ無効・くさタイプ無効・ぼうじん無効・ぼうじんゴーグル無効はすべてフレームワーク（`ON_TRY_MOVE_1` priority=110、`ON_TRY_MOVE_2` priority=110/120）が処理するため、ハンドラ側のガードは不要
- まひ付与の基本ロジック（でんきタイプへの免疫、すでに状態異常がある場合の失敗）は `ailment_manager` 内部でも処理される
- `でんじは`（でんきタイプ・じめんタイプ無効）と異なり、しびれごなはじめんタイプには有効（でんきタイプのみ無効）
- 命中率 75 のため、テストでは `accuracy=100` 固定で実行する
