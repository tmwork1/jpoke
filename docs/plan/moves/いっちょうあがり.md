# いっちょうあがり 実装計画書

## 仕様要約（docs/spec/moves/いっちょうあがり.md 参照）

* タイプ: ドラゴン / 分類: 物理 / 威力: 80 / 命中: 100 / PP: 10
* 対象: 相手1体 / 優先度: 0 / 接触なし
* 効果: ヘイラッシャが特性しれいとうでシャリタツを飲み込んでいる場合、自分の能力を1段階上げる。
  ただし特性しれいとうはダブル専用（docs/progress/ability.md で全列「ダブル専用」＝対象外）であり、
  本プロジェクトはシングルバトルのみ対象のため、このランクアップ効果自体は実装しない（常に発動しない）。
* ちからずくとの相互作用: 技自体に「追加効果は無いがちからずく対象」フラグが設定されており、
  ちからずくのポケモンが使用すると（シングルでも）威力が1.3倍になる
  （docs/spec/abilities/ちからずく.md 参照。エレクトロビームと違いChampions除外の記載がないため本技はChampionsでも対象）。

## 現状の実装状況

`src/jpoke/data/move.py` に登録済みだが、`flags` が未設定（空集合）だった。

```python
"いっちょうあがり": MoveData(
    type="ドラゴン",
    category="physical",
    pp=10,
    power=80,
    accuracy=100,
    handlers={},  # 追加効果なし
),
```

これは基本ダメージ計算としては仕様書と一致しているが、`flags={"secondary_effect"}` が欠落しており、
`ちからずく_boost`（`handlers/ability.py`）が `ctx.move.has_flag("secondary_effect")` を見て威力補正するため、
このままではちからずくのポケモンが使用しても威力が1.3倍にならない不具合がある。

### 修正後

```python
"いっちょうあがり": MoveData(
    type="ドラゴン",
    category="physical",
    pp=10,
    power=80,
    accuracy=100,
    flags={"secondary_effect"},  # 追加効果自体は無いが、ちからずく対象技として扱われる（docs/spec/abilities/ちからずく.md参照）
    handlers={},  # しれいとう連携のランクアップはダブル専用のため対象外（実装しない）
),
```

## ハンドラ構成

専用ハンドラは不要。`flags={"secondary_effect"}` の追加のみで完結する。
`ちからずく_boost` / `ちからずく_disable_secondary_effect`（`handlers/ability.py`）は既存実装のまま
`ctx.move.has_flag("secondary_effect")` を判定するため、本技側の追加コードは不要。

`handlers/move_attack.py` への追加コードはない。

## Priority 根拠

新規イベントハンドラを追加しないため、priority 検討は不要。
威力補正は既存の `Event.ON_CALC_POWER_MODIFIER`（`ちからずく_boost`）が担い、本技は `flags` のみで連動する。

追加なし。

### data/move.py

`"いっちょうあがり"` エントリに `flags={"secondary_effect"}` を追加する。

## 注意点・エッジケース（review-test 向け）

1. **しれいとう連携のランクアップは実装しない**
   - 特性しれいとうはダブル専用能力（docs/progress/ability.md で全列「ダブル専用」）であり、
     シャリタツを味方として繰り出すダブルバトルでのみ発動しうる。シングル専用の本プロジェクトでは
     この分岐が到達しないため、実装・テスト対象外。

2. **ちからずく相互作用のテストのみ追加する**
   - `flags={"secondary_effect"}` の追加により、ちからずく所持ポケモンがいっちょうあがりを使用したとき
     `battle.damage_calculator.power_modifier == 5325`（1.3倍固定小数表現）になることを確認するテストを
     `tests/abilities/test_ability_ta.py`（既存の `test_ちからずく_追加効果が発動せず威力が1_3倍` の直後）に追加する。
   - このテストは特性側（ちからずく）のフラグ判定ロジックの回帰確認であり、`tests/moves_attack/` 側には
     本技固有の追加効果（ランクアップ等）が実装されないため新規テストは追加しない。

3. **専用技**
   - ヘイラッシャの専用技（`data/pokedex.json` 側で対応していれば変更不要。技そのものの習得可否は本計画の対象外）。
