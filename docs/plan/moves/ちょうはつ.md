# 実装計画: ちょうはつ

## 仕様サマリ

- タイプ: あく / 分類: 変化 / PP: 20 / 命中率: 100
- 効果: 3 ターンの間、相手は攻撃技しか使えなくなる（変化技を封じる）
- まもる: ○ / マジックコート: ○ / 直接攻撃: ×
- 対象: 1体選択

## 現状の実装状況

`data/move.py` にエントリは存在するが `handlers` キーが未設定（ちょうはつ揮発性状態付与が未実装）。

```python
"ちょうはつ": MoveData(
    type="あく",
    category="変化",
    pp=20,
    accuracy=100,
),
```

ちょうはつ揮発性状態自体は `docs/progress/volatile.md` で実装済み（`x`）。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手にちょうはつ揮発性状態を付与する |

## Priority 根拠

`docs/spec/turn.md` の `ON_TURN_END` に「ちょうはつの終了 priority=110」と記載があるが、これは状態の終了処理である。付与側の `ON_STATUS_HIT` は記載なし。
既存の `apply_volatile_to_defender` を使うハンドラ（`アンコール_apply`、`いちゃもん_apply_volatile_to_defender`、`ロックオン_apply_volatile_to_defender`）がすべて priority=100 デフォルトで実装されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: 五十音順で `ちいさくなる_*` と `ちょうのまい_*` の間（ちい → ちょうは → ちょうのまい の順）

正確には: `ちいさくなる_*` の後、`ちょうのまい_*` の前

```python
def ちょうはつ_apply_volatile_to_defender(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """ちょうはつの効果: 相手にちょうはつ状態を付与する（3 ターン）。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="ちょうはつ")
```

### data/move.py の更新

`ちょうはつ` の MoveData に `handlers` を追加する。

```python
"ちょうはつ": MoveData(
    type="あく",
    category="変化",
    pp=20,
    accuracy=100,
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ちょうはつ_apply_volatile_to_defender,
        ),
    }
),
```

## 注意点・エッジケース

- ちょうはつ揮発性状態の持続ターン数（3 ターン）は `volatile_manager.apply` 内のちょうはつ揮発実装が管理している。`count` 引数は不要（デフォルト値を使用）
- すでにちょうはつ状態の相手には失敗する（`ON_APPLY_MOVE` priority=100「ちょうはつ: すでに状態変化」でフレームワークが処理）
- `どんかん` 特性の相手にはちょうはつが無効（`ON_TRY_MOVE_2` priority=140 でフレームワークが処理）
- `アロマベール` 特性の相手にもちょうはつが無効（`ON_TRY_MOVE_2` priority=140 でフレームワークが処理）
- ターン終了時のちょうはつ継続・終了は `ON_TURN_END` priority=110 で既に実装されている
