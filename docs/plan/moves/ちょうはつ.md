# 計画書: ちょうはつ

出典: `docs/spec/moves/ちょうはつ.md`

## 仕様要約

- あくタイプ / 変化 / 命中100 / PP20（champions 基準） / 対象: 相手1体
- 相手にちょうはつ揮発性状態を付与する（3ターン固定。Pokémon Champions 仕様）
- ちょうはつ状態の相手は変化技を選択・使用できなくなる（詳細は `docs/spec/volatiles/ちょうはつ.md`）
- **相手を対象とする変化技のため `MoveData(target=...)` は省略時デフォルトの `"foe"` のままでよい**
  （まもる・マジックコートに正しく反応するために必要な設定であり、既定値のままで問題ない）
- **みがわりを貫通する（原作Wiki: 「みがわり状態のポケモンにも効果がある」）ため `flags={"bypass_substitute"}` が必須**
  （`じこあんじ`・`スキルスワップ` と同じパターン。省略すると `みがわり_immune` により
  みがわり状態の相手には無効化されてしまう）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 相手にちょうはつ揮発性状態を付与する（count=3） |

## Priority 根拠

`docs/spec/turn.md` の `ON_TURN_END` に「ちょうはつの終了 priority=110」と記載があるが、これは状態の終了処理である。付与側の `ON_STATUS_HIT` は記載なし。
既存の `apply_volatile_to_defender` を使うハンドラ（`アンコール_apply`、`いちゃもん_apply_volatile_to_defender`、`ロックオン_apply_volatile_to_defender`）がすべて priority=100 デフォルトで実装されているため、同様に採用する。

## 実装箇所

### handlers/move_status.py（実装済み）

挿入位置（五十音順）: `ちょうのまい_modify_attacker_stats` の前、`ちいさくなる_*` の後

```python
def ちょうはつ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ちょうはつの効果: 相手にちょうはつ状態を付与する（3 ターン）。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="ちょうはつ", count=3)
```

### data/moves/move_ta.py（実装済み）

```python
"ちょうはつ": MoveData(
    type="あく",
    category="status",
    pp=20,
    accuracy=100,
    flags={"bypass_substitute"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ちょうはつ_apply,
        ),
    }
),
```

## 注意点・エッジケース

- ちょうはつ揮発性状態の持続ターン数（3 ターン）は `ちょうはつ_apply` が `count=3` を明示的に渡している（原作の「行動後は4ターン／行動前・交代直後は3ターン」の分岐を単純化）
- すでにちょうはつ状態の相手には失敗する（`volatile_manager.apply` が既存 volatile への重複付与を失敗として扱う）
- `どんかん` 特性の相手にはちょうはつが無効（`ON_BEFORE_APPLY_VOLATILE` で `どんかん_prevent_volatile` が処理。`data/ability.py`）
- `アロマベール` 特性の相手にもちょうはつが無効（`ON_BEFORE_APPLY_VOLATILE` で `アロマベール_prevent_volatile` が処理。味方全体にも及ぶ）
- `おうごんのからだ` 特性の相手には技「ちょうはつ」自体が無効（`ON_BEFORE_APPLY_MOVE` で `おうごんのからだ_block_status_move` が処理。他の対象foeの変化技全般も同様）
- `メンタルハーブ` を持つ相手には付与直後に即時解除される（`ON_BEFORE_APPLY_VOLATILE` で `メンタルハーブ_cure_mental_volatile` が処理。`data/item.py`）
- ターン終了時のちょうはつ継続・終了は `ON_TURN_END` priority=110 で既に実装されている（`ちょうはつ_tick_volatile`）
- バトンタッチで引き継がれない（原作Wiki記載の通り。`handlers/move_status.py` の `_BATON_PASS_VOLATILES` から意図的に除外。通常はちょうはつ状態の効果でバトンタッチ自体を使用できないため実戦では到達しないが、原作仕様に合わせて除外している）
- **リーサル計算**: 対象外（ダメージを与えない変化技、n/a）
