# 実装計画書: さきおくり

## 仕様要約

- タイプ: あく / カテゴリ: 変化 / PP: 15 / 命中: - / 対象: 相手1体
- 優先度: 0 / 直接攻撃: ×
- 相手の行動をそのターンの最後に強制する（ダブルバトル向けの主効果）
- 相手がそのターンにすでに行動済みの場合は失敗する
- **シングルバトルでは「行動順を最後にする」効果は実質無意味**（1対1のため相手が行動済みなら失敗するだけ）

### 失敗条件（ON_BEFORE_APPLY_MOVE priority 100）

- turn.md: `さきおくり等: すでに行動済み`
- 相手（`ctx.defender`）がそのターンにすでに行動済みの場合に失敗する
  - 判定: `battle.query.is_first_actor(defender_player) == True`（相手が先攻 = すでに行動した）
  - または: `defender_state.has_switched == True`（相手がそのターンに交代済み）

### 成功時の効果（シングルバトルのスコープ）

- シングルバトルでは行動順変更が不要のため、ON_STATUS_HIT ハンドラは空実装（`HandlerReturn(value=value)` を返すだけ）とする
- 実際の行動順変更は `action_order` リストの操作が必要だが、シングルバトル限定では対象外

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「さ」行 — `さいはい_instruct` の後、`さいみんじゅつ_apply_ailment_to_defender` の前に挿入する。

#### さきおくりのハンドラ関数

```python
def さきおくり_can_apply(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """さきおくりの失敗条件を判定する。

    相手がそのターンにすでに行動済みの場合は失敗する。
    「行動済み」とは以下のいずれかを満たす状態：
    - 相手がこのターン先攻で技を発動済み（action_order_index == 0）
    - 相手がこのターン交代済み（has_switched == True）

    Note:
        action_order_index が None の場合（行動順未確定）は失敗しない。
        シングルバトルでは相手が先攻なら既に技を発動しており、
        さきおくり使用者は後攻であるため、通常は常に失敗する。
        相手が後攻（未行動）のときのみ成功する。
    """
    defender_player = battle.get_player(ctx.defender)
    defender_state = battle.player_states[defender_player]
    defender_acted = (
        battle.query.is_first_actor(defender_player) is True
        or defender_state.has_switched
    )
    if defender_acted:
        battle.add_event_log(ctx.attacker, LogCode.MOVE_FAILED,
                             payload={"reason": "さきおくり"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

## 注意点・エッジケース

### シングルバトルにおける実用上の意味

- さきおくりはダブルバトルで有効な技であり、シングルバトルでは核心効果（相手の行動順を最後に変える）を発動させても意味がない
- 本実装ではシングルバトル限定のスコープとして、失敗条件チェックのみを実装する
- 行動順変更（`action_order` リスト操作）は実装しない

### 「行動済み」判定の精度

- `battle.query.is_first_actor(defender_player)` は `action_order_index == 0` で判定する
- `action_order_index` は `TurnController._record_action_order` でターン開始時に確定・記録される
- 相手が先攻（`action_order_index == 0`）の場合、さきおくり使用時点では相手はすでに技を発動している（`_run_move_phase` で先に処理されている）
- `has_switched` は交代フェーズで `True` に設定されるため、交代した相手にも正しく失敗する

### `executed_move` を使わない理由

- `Pokemon.executed_move` はターン間でリセットされないため（`reset_on_switch_out` のみリセット、`reset_turn_state` ではリセットされない）、前ターンの値が残り「今ターン行動済み」の判定に使えない
- `action_order_index` と `has_switched` の組み合わせが正確な「このターンの行動済み」判定になる

### おさきにどうぞとの関係

- turn.md に「おさきにどうぞ: 失敗条件」（priority 100、ON_APPLY_MOVE）も記載されており、類似パターンで実装される予定
- さきおくりとおさきにどうぞは「行動順を最後にする」対「行動順を最初にする」という逆の効果

### まもる・マジックコート

- まもる: まもる状態の相手に使うと防がれる（`ON_TRY_MOVE_1` priority 100 の「まもる等による無効化」がフレームワーク側で処理）
- マジックコート: ハンドラ側での対処不要（`ON_CHECK_REFLECT` でフレームワーク側が処理）

### かがくへんかガス・かたやぶり

- さきおくりはランク変化・状態異常を付与しないため、かがくへんかガス・かたやぶりの影響を受けない
- ハンドラ側でのチェック不要
