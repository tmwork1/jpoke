# かかとおとし 実装計画書

## 仕様要約（docs/spec/moves/かかとおとし.md 参照）

* タイプ: かくとう / 分類: 物理 / 威力: 120 / 命中: 90 / PP: 10
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果1: 30%の確率で相手を『こんらん』状態にする（3〜5ターン継続。通常のこんらん技は2〜5ターン）。
* 効果2: 技が外れると自分の最大HPの1/2のダメージを受ける（self_cost、いしあたまで防げない）。

## 現状の実装状況

`src/jpoke/data/move.py` に登録済み。

```python
"かかとおとし": MoveData(
    type="かくとう",
    category="physical",
    pp=10,
    power=120,
    accuracy=90,
    flags={"contact", "recoil", "secondary_effect"},
    handlers={
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.かかとおとし_apply_confusion,
        ),
        Event.ON_MISS: h.MoveHandler(
            ha.かかとおとし_crash,
        ),
    }
),
```

### レビューで判明した修正点

1. **`recoil` フラグの欠落**: すてみ特性は `ctx.move.has_flag("recoil")` を判定条件にしており
   （`handlers/ability.py` の `すてみ_boost_power`）、このフラグが無いと威力1.2倍が発動しない。
   wiki 記載（「特性がすてみのポケモンが使用した場合、威力が1.2倍になる」）と一致させるため追加した。
   なお `recoil` フラグは `MoveFlag`（すてみ判定用）であり、いしあたまが参照する `HPChangeReason`
   （`ctx.hp_change_reason`）とは別の型・別の判定軸のため、フラグを追加しても
   いしあたまによる反動無効化には影響しない（`いしあたま_ignore_recoil` は `reason == "recoil"` を見るが、
   本技の反動は `reason="self_cost"` で発生させているため元々対象外）。
2. **こんらんの継続ターン数**: 汎用ヘルパー `apply_confusion_to_defender`（`handlers/move.py`）は
   `battle.volatile_manager.apply_confusion` を呼び、2〜5ターンで固定される。
   本技は wiki 仕様上 3〜5ターンという固有の継続ターン数を持つため、汎用ヘルパーを使わず
   専用ハンドラで `battle.volatile_manager.apply(..., count=battle.random.randint(3, 5))` を直接呼ぶ形に修正した。

## ハンドラ構成

| `ON_DAMAGE_HIT` | 100（デフォルト） | `attacker:self`（`AttackContext`） | 30%の確率で相手にこんらん（3〜5ターン）を付与 |
| `ON_MISS` | 100（デフォルト） | `attacker:self`（`AttackContext`） | 外れたとき自分の最大HPの1/2ダメージ（self_cost） |

## 実装箇所

### handlers/move_attack.py

```python
def かかとおとし_apply_confusion(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """かかとおとしの追加効果: 30%の確率で相手をこんらん状態にする。

    通常のこんらん付与技（2〜5ターン）と異なり、この技のこんらんは3〜5ターン継続する。
    """
    chance = battle.resolve_secondary_chance(ctx, 0.3)
    if chance < 1 and battle.random.random() >= chance:
        return HandlerReturn(value=value)
    count = battle.random.randint(3, 5)
    return HandlerReturn(value=battle.volatile_manager.apply(
        ctx.defender, "こんらん", count=count, source=ctx.attacker
    ))


def かかとおとし_crash(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """かかとおとしが外れた場合の失敗反動ダメージ。自分の最大HPの1/2を受ける。"""
    battle.modify_hp(ctx.attacker, v=-max(1, ctx.attacker.max_hp // 2), reason="self_cost", source=ctx.attacker)
    return HandlerReturn(value=value)
```

### data/move.py

`flags` に `"recoil"` を追加（既存の `"contact"`, `"secondary_effect"` は維持）。

## 注意点・エッジケース（review-test 向け）

1. **こんらん継続ターン数**: `battle.random.randint` をモックして `count == 3` / `count == 5` を確認する。
   （通常のこんらん技が `randint(2, 5)` を呼ぶのに対し、本技は `randint(3, 5)` を呼ぶ点を区別する）
2. **ちからずくとの相互作用**: `resolve_secondary_chance` を経由するため、ちからずく特性を持つ攻撃側では
   こんらんの追加効果確率が0になる（威力1.3倍は `ちからずく_boost_power` 側で処理、変更不要）。
3. **すてみとの相互作用**: `recoil` フラグ追加により、すてみ特性所持時に威力補正 `apply_fixed_modifier(value, 4915)`
   (1.2倍) が発生することを確認する。
4. **いしあたまとの相互作用**: 失敗反動は `reason="self_cost"` のため、いしあたまでは防げないことを確認する
   （既存テスト `test_かかとおとし_いしあたまでも失敗反動ダメージを受ける` で担保済み）。
5. **命中時に反動が発生しないこと**: `ON_MISS` は命中判定失敗時のみ発火するため、命中時はダメージを受けない
   （既存テストで担保済み）。
6. **既知の実装制約**: まもる・タイプ無効化による失敗時の反動は現状未実装（`docs/spec/moves/かかとおとし.md`
   「実装上の制約」参照）。本レビューの対象範囲外として現状維持する。
