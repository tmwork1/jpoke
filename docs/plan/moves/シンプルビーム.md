# 計画書: シンプルビーム

## 仕様要約

- タイプ: ノーマル / 変化技 / PP 15 / 命中 100
- 効果: 相手の特性を「シンプル」に書き換える
- 「シンプル」特性の効果: 自分のランク変化の段階数が 2 倍になる（上昇・下降ともに）
- 失敗条件（`ON_BEFORE_APPLY_MOVE` priority 100）:
  - 対象の特性が `protected` フラグを持つ場合（かがくへんかガス・ふしぎなまもり等）

## 実装箇所

### 前提: 「シンプル」特性の実装

シンプルビームが付与する特性「シンプル」が未実装のため、先に実装が必要。

#### `handlers/ability.py` に追加する関数

```python
def シンプル_modify_stat_delta(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """シンプル特性: 自分のランク変化量を 2 倍にする。

    ON_BEFORE_MODIFY_STAT イベントで value（変化段階数）を 2 倍して返す。
    """
    return HandlerReturn(value=value * 2)
```

#### `data/ability.py` に追加するエントリ（五十音順: シ の位置）

```python
"シンプル": AbilityData(
    handlers={
        Event.ON_BEFORE_MODIFY_STAT: h.AbilityHandler(
            h.シンプル_modify_stat_delta,
            subject_spec="source:self",
        ),
    }
),
```

> 注意: `ON_BEFORE_MODIFY_STAT` が使用するコンテキストロールと priority は、既存の「シンプル」系ハンドラ（クリアボディ等）を参照して合わせること。

### シンプルビーム本体の実装

#### `handlers/move_status.py` に追加する関数（五十音順: シ の位置）

```python
def シンプルビーム_can_apply(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """シンプルビームの失敗条件チェック。

    対象の特性が protected フラグを持つ場合は失敗する。
    """
    if ctx.defender.ability.has_flag("protected"):
        battle.add_event_log(ctx.attacker, LogCode.MOVE_FAILED,
                             payload={"reason": "シンプルビーム"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def シンプルビーム_change_ability(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """シンプルビームの効果: 相手の特性を「シンプル」に書き換える。"""
    battle.change_ability(ctx.defender, "シンプル")
    return HandlerReturn(value=value)
```

#### `data/move.py` の「シンプルビーム」エントリを更新

```python
"シンプルビーム": MoveData(
    type="ノーマル",
    category="変化",
    pp=15,
    accuracy=100,
    handlers={
        Event.ON_BEFORE_APPLY_MOVE: h.MoveHandler(
            hs.シンプルビーム_can_apply,
            priority=100,
        ),
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.シンプルビーム_change_ability,
        ),
    }
),
```

## 注意点・エッジケース（review-test 向け）

### 失敗条件

- 対象の特性が `protected` フラグを持つ場合は失敗する（いえき・なりきり・スキルスワップと同じ判定）
- 対象が「とくせいなし」状態（`volatile`）の場合でも特性オブジェクト自体は残るため、`ability.has_flag("protected")` の判定は通常通り動作する
- 相手がすでに「シンプル」の場合は `battle.change_ability` 内部の `if mon.ability.base_name == ability: return` ガードで処理され、技は成功扱いになる（失敗しない）

### 特性変更の副作用

- `battle.change_ability` が呼ばれると `ON_ABILITY_DISABLED` → ハンドラ解除 → `ON_ABILITY_ENABLED` の順でイベントが発火し、「シンプル」ハンドラが登録される
- 交代すると `base_ability_name` に戻る（`change_ability` は `base_name` も更新しない）ため、交代後は元の特性に戻ることに注意

### 「シンプル」特性の実装依存

- シンプルビームのテストを書く前に「シンプル」特性の `ON_BEFORE_MODIFY_STAT` ハンドラ実装が完了している必要がある
- 「シンプル」特性がシンプルビームで付与された場合もランク変化 2 倍が適用されることをテストで確認すること

### マジックコート・マジックミラー

- シンプルビームは変化技かつ対象が相手のため、マジックコート・マジックミラーで反射される対象となる
- 反射処理は `ON_TRY_MOVE_1` の priority 100 で manager 内部が扱うため、ハンドラ側で追加ガードは不要

### かがくへんかガス中の挙動

- かがくへんかガス中の場合、対象の特性は機能しないが特性オブジェクト自体は変わらない
- `protected` フラグの判定はオブジェクトに基づくため、かがくへんかガス下でも `protected` 特性への使用は失敗する
