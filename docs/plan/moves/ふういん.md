# 計画書: ふういん

参照仕様書: `docs/spec/moves/ふういん.md`, `docs/spec/volatiles/ふういん.md`

## 仕様要約

- タイプ: エスパー / カテゴリ: 変化 / PP: 12（`src/jpoke/data/ps-champ-ja/moves.json` 基準） / 命中: なし（必中） / 対象: 自分
- 自分に「ふういん」揮発性状態を付与する
- すでにふういん状態の場合は失敗する（`volatile_manager.apply` が False を返す）
- まもる・マジックコートの対象外（対象が `self` のため）
- 揮発性状態側（相手の行動ブロック）は `data/volatile.py` / `handlers/volatile.py` に実装済み

| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self` | 自分にふういん volatile を付与 | `handlers/move_status.py` の `ふういん_apply` |

揮発性状態側（付与後の相手行動ブロック）はすでに `data/volatile.py` + `handlers/volatile.py` に実装済み

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 専用の記載はないが、既存の同種の自己付与揮発性状態技
（`ファストガード_apply`、`はいすいのじん` 等）と同様に `Event.ON_STATUS_HIT` の `MoveHandler` デフォルト
priority（100）を採用する。

揮発性状態側の `ON_TRY_ACTION` priority=100 の根拠は `docs/plan/volatiles/ふういん.md` を参照
（`docs/spec/turn.md` の `ON_CHECK_ACTION`〈実装上の `ON_TRY_ACTION`〉に「100: ふういんで技が出せない」と明記）。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

```python
def ふういん_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return apply_volatile_to_attacker(battle, ctx, value, volatile="ふういん")
```

### `src/jpoke/data/moves/move_ha.py`

```python
"ふういん": MoveData(
    type="エスパー",
    category="status",
    pp=12,
    target="self",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.ふういん_apply,
        ),
    }
),
```

## 注意点・エッジケース

- **`target="self"` の重要性**: `Move.is_blocked_by_protect` は `target == "foe"` を前提に判定するため、
  `target` を明示しないとデフォルトの `"foe"` が使われ、まもる・マジックコートの判定が誤動作する。
  自分を対象とする変化技は必ず `target="self"` を明示する（`はいすいのじん` 等の既存実装と同一パターン）。
- **重複付与の失敗**: `apply_volatile_to_attacker` → `volatile_manager.apply` が既存の
  ふういん状態を検知して `False` を返すため、専用の失敗条件チェックは不要。
- **揮発性状態側のロジックは別ファイル**: 相手の行動ブロックロジック（共通技判定）は
  `docs/plan/volatiles/ふういん.md` を参照。技側の実装はあくまで volatile 付与のみを担当する。
