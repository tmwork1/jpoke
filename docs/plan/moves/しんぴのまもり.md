# 実装計画書: しんぴのまもり

## 仕様要約

- タイプ: ノーマル / カテゴリ: 変化 / PP: 20（docs/champions/move_list.txt準拠） / 対象: 味方の場（target="own_side"）
- 5ターンの間、自分と味方のポケモンは他のポケモンからのわざの効果で状態異常・こんらん・ねむけ状態にならなくなる
- 交代しても効果は続く
- すでに味方の場が「しんぴのまもり」状態になっている場合は失敗する
- 状態異常無効化の詳細仕様は `docs/spec/fields/しんぴのまもり.md`（サイドフィールド仕様書）を参照

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | 自陣営サイドフィールドに「しんぴのまもり」を5ターン設定する |

状態異常・こんらん・ねむけの無効化ロジック自体はサイドフィールド側
（`ON_BEFORE_APPLY_AILMENT` / `ON_BEFORE_APPLY_VOLATILE` / `ON_TURN_END(priority=130)`）
に実装済みで、本計画書（技側）のスコープ外。詳細は `docs/spec/fields/しんぴのまもり.md` を参照。

## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 自体の記載はないが、同一パターンの壁設置技
（`リフレクター_set_side_field` / `ひかりのかべ_set_side_field` / `オーロラベール_set_side_field`）が
いずれも `ON_STATUS_HIT` のデフォルト priority（100）を使用しているため、同一パターンを踏襲する。
サイドフィールドの `ON_TURN_END` は `docs/spec/turn.md` の「130 | しんぴのまもりの継続・終了」に一致する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

```python
def しんぴのまもり_set_side_field(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """しんぴのまもり: 自陣営に「しんぴのまもり」を5ターン設定する。"""
    side = battle.get_side(ctx.attacker)
    if not side.activate("しんぴのまもり", 5):
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/moves/move_sa.py`

```python
"しんぴのまもり": MoveData(
    type="ノーマル",
    category="status",
    pp=20,
    target="own_side",
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.しんぴのまもり_set_side_field,
        ),
    }
),
```

## 注意点・エッジケース

- `side.activate(name, count)`（`StackableFieldManager.activate`）を使用しており、壁系（リフレクター等）が使う
  `side.apply(name, count, source=...)` とは異なる。`apply()` は `ON_MODIFY_DURATION` を発火し、
  ひかりのねんど所持時に持続ターンを延長するが、しんぴのまもりは仕様上ひかりのねんどで延長されない
  （一次情報: `docs/wiki/side_fields/しんぴのまもり_(場の状態).html` 備考節）ため、意図的に `activate()` を使い
  `ON_MODIFY_DURATION` を発火させないことで延長を防いでいる。壁とは異なる仕様である点に注意。
- `side.activate` が False を返す（すでにしんぴのまもりが有効）場合は失敗として `HandlerReturn(value=False, stop_event=True)` を返す
- リーサル計算は対象外（ダメージを与えない変化技のため `lethal_handlers` なし、`docs/progress/move.md` は n/a）
