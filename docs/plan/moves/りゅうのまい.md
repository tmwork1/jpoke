# 計画書: りゅうのまい

## 仕様要約

- ドラゴンタイプ / 変化 / 命中なし / PP20 / 対象: 自分
- 自分の『こうげき』ランクを1段階上げる
- 自分の『すばやさ』ランクを1段階上げる
- ダンス技（labels: dance）のため、おどりこ特性のポケモンに使用した場合に相手もコピーする

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | こうげき+1、すばやさ+1 |

## Priority 根拠

`docs/spec/turn.md` には `ON_STATUS_HIT` の詳細表はないが、既存の `ちょうのまい_modify_attacker_stats`（`ON_STATUS_HIT` priority 100）が同じ複数ランクアップパターンであるため、同じ priority を採用する。ランク上限チェック（すでに最大のとき失敗）は `ON_TRY_MOVE_2` priority 130 でフレームワーク側が処理する。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `ロックオン_apply_volatile_to_defender` の後、`ロックカット_modify_attacker_stats` の前（五十音順）。

```python
def りゅうのまい_modify_attacker_stats(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """りゅうのまいの効果: 自分のこうげき・すばやさを1段階ずつ上げる。"""
    return modify_attacker_stats(battle, ctx, value, stats={"A": 1, "S": 1})
```

### data/move.py への更新

`"りゅうのまい"` エントリのハンドラを追加する。

```python
    "りゅうのまい": MoveData(
        type="ドラゴン",
        category="変化",
        pp=20,
        labels=["dance"],
        handlers={
```

## 注意点・エッジケース

- こうげき・すばやさともに+6の場合の失敗は `ON_TRY_MOVE_2` priority 130 の `ランク補正上げ: すでに最大` でフレームワーク側が処理するため、ハンドラ内ガード不要
- ダンス技（labels: dance）のため、おどりこ特性のポケモンが相手にいる場合は `ON_DAMAGE` priority 190 でコピーされる。これもフレームワーク側が処理する
- `ちょうのまい_modify_attacker_stats` と全く同じ実装パターン（stats の内容のみ異なる）
- `modify_attacker_stats` ヘルパーを利用することで、ランク変化ログ出力まで一括処理される
