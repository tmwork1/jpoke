# 実装計画: りゅうのまい

## 仕様サマリ

- タイプ: ドラゴン / 分類: 変化 / 威力: - / 命中率: - / PP: 20 / 対象: 自分
- 効果: 自分のこうげき・すばやさランクを 1 段階ずつ上げる
- ダンス技（`dance` フラグ）
- まもる: 防がれない（自分対象） / マジックコート: 反射不可（自分対象） / 直接攻撃: いいえ
- よこどり: 対象になる（○）

出典: docs/spec/moves/りゅうのまい.md

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | こうげき+1、すばやさ+1 |

## Priority 根拠

`docs/spec/turn.md` には `ON_STATUS_HIT` の詳細表はないが、既存の `ちょうのまい_modify_attacker_stats` /
`つるぎのまい_modify_attacker_stats`（いずれも `ON_STATUS_HIT` priority 100 = デフォルト）が同じ
複数ランクアップパターンであるため、同じ priority を採用する。ランク上限チェック（すでに最大のとき失敗）は
`battle.modify_stats()` 側でマネージャー内部処理されるため、ハンドラ側にガード不要。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `ロックオン_apply_volatile_to_defender` の後、`ロックカット_modify_attacker_stats` の前（五十音順）。

```python
def りゅうのまい_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """りゅうのまいの効果: 自分のこうげき・すばやさを1段階ずつ上げる。"""
    return modify_attacker_stats(battle, ctx, value, stats={"atk": 1, "spe": 1})
```

### data/moves/move_ra.py の更新

`target="self"` を指定する（`target` を指定しないとデフォルトの `"foe"` になり、
`is_blocked_by_protect` / `is_reflectable` がまもる・マジックコートに誤反応してしまうため必須）。

```python
"りゅうのまい": MoveData(
    type="ドラゴン",
    category="status",
    pp=20,
    target="self",
    flags={"dance"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.りゅうのまい_modify_attacker_stats,
        ),
    }
),
```

## 注意点・エッジケース

- `target="self"` を指定し忘れると、まもる・マジックコートの判定にデフォルト値 `"foe"` の挙動が
  誤って適用されるため、必ず指定する
- こうげき・すばやさともに+6の場合の失敗は `battle.modify_stats()` 側でマネージャー内部処理されるため、
  ハンドラ内ガード不要
- ダンス技（`dance` フラグ）のため、おどりこ特性のポケモンが相手にいる場合は既存フレームワーク側で
  コピーされる
- `ちょうのまい_modify_attacker_stats` / `つるぎのまい_modify_attacker_stats` と全く同じ実装パターン
  （stats の内容のみ異なる）
- `modify_attacker_stats` ヘルパーを利用することで、ランク変化ログ出力まで一括処理される
