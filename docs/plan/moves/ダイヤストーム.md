# ダイヤストーム 実装計画書

## 仕様要約（docs/spec/moves/ダイヤストーム.md 参照）

* タイプ: いわ / 分類: 物理 / 威力: 100 / 命中: 95 / PP: 5
  （move_list.txt に該当エントリなし＝ディアンシー専用技かつ幻のポケモンは使用不可のため据え置き。
  シードフレアの前例（`docs/review/moves/シードフレア.md`）と同じ扱い）
* 対象: 相手1体（シングルバトルのみ対象） / 優先度: 0 / 接触なし
* 効果: 追加効果として、50%の確率で自分の『ぼうぎょ』ランクを2段階上げる

## 現状の実装状況

`src/jpoke/data/moves/move_ta.py` に既に登録済み。

```python
"ダイヤストーム": MoveData(
    type="いわ",
    category="physical",
    pp=5,
    power=100,
    accuracy=95,
    flags={"secondary_effect"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.ダイヤストーム_sharply_boost_defender_B,
        )
    }
),
```

内容は仕様書と一致しており、修正不要（PP・威力・命中・追加効果確率・上昇幅すべて一致）。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_HIT` | 100（デフォルト） | attacker:self（`MoveHandler` 既定値） | 50%の確率で自分の『ぼうぎょ』ランクを2段階上げる |

`src/jpoke/handlers/move_attack.py` の実装（登録済み・修正不要）:

```python
def ダイヤストーム_sharply_boost_defender_B(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"def": 2}, chance=0.5)
```

`modify_attacker_stats`（`handlers/move.py`）が `battle.resolve_secondary_chance` を経由するため、
ちからずくによる追加効果無効化（確率0化）・威力1.3倍化は `flags={"secondary_effect"}` さえ
付与されていれば汎用的に処理される。本技側に個別のガードは不要。

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_HIT` セクション（L475〜L489）に「10 | 自分のランクを上げる追加効果」
という記載があるが、既存の同種実装（`グロウパンチ_boost_attacker_A`、`はがねのつばさ_modify_attacker_stats`、
`しんぴのちから_modify_attacker_stats` 等、すべて `data/moves/move_*.py` で `MoveHandler(func)` の
デフォルト priority=100 のまま登録）と同じパターンに従う。ON_HIT 内で自分のランク上昇効果と
競合する別ハンドラが同時登録されるケースが実質存在しないため、turn.md の priority=10 は
イベント内の処理グループを示す文書上の整理であり、コード上は既存パターン（priority 引数省略）に
合わせて問題ない。

## 注意点・エッジケース（review-test 向け）

1. **ちからずくとの相互作用**
   - `flags={"secondary_effect"}` が付与されているため、ちからずく所持時は威力1.3倍（100→130）になり、
     ぼうぎょ上昇の追加効果は発動しなくなる。
2. **PPの扱い**
   - move_list.txt に本技のエントリが存在しないため、8/12/16/20への変換対象外。Gen9本家基準の5を
     正としてそのまま維持する（シードフレア等の他の幻専用技と同じ扱い）。
3. **ダブルバトルの複数回判定は対象外**
   - 第六世代・第七世代以降でダブルバトル時の追加効果判定回数の仕様が異なるが、本プロジェクトは
     シングルバトルのみ対象のため実装不要。1体の相手に対して1回の追加効果判定を行えば仕様を満たす。
4. **テスト方針**
   - `secondary_chance` を0/1に固定し、ぼうぎょランクの変化有無を確認するテストを追加・確認する
     （`test_ダイヤストーム_防御2段階上昇が発動する` は実装済み。
     `test_ダイヤストーム_防御2段階上昇が発動しない` を追加する）。
