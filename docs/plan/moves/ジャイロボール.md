# 計画書: ジャイロボール（再レビュー）

## 仕様要約

* 自分（使用者）の実効素早さが相手より遅いほど威力が大きくなる（`docs/spec/moves/ジャイロボール.md` 参照）。
* 威力 = floor(25 × 相手の実効素早さ ÷ 自分の実効素早さ) + 1、最大150。
* 自分の実効素早さが0の場合は威力1固定（第六世代以降の仕様、ゼロ除算防止）。
* 実効素早さは、両者のランク補正・特性・もちもの・まひ・おいかぜ・しつげんの影響を受ける
  （対義技のエレキボールと同様、`battle.speed_calculator.calc_effective_speed()` を使う）。

## 既存実装の確認結果・修正内容

* `src/jpoke/handlers/move_attack.py` の `ジャイロボール_calc_power` は、威力計算式
  （`floor(25 × 相手の実効素早さ ÷ 自分の実効素早さ) + 1`、最大150、自分の実効素早さが0のとき
  威力1固定）・実効素早さの取得方法（`calc_effective_speed()` 使用）ともに仕様書と一致しており、
  修正不要だった。
* `src/jpoke/data/moves/move_sa.py` の `MoveData` の `pp` が `5`（本家基準の値）になっていたが、
  `src/jpoke/data/ps-champ-ja/moves.json` の値（8）と食い違っていたため `8` に修正する。
  `docs/progress/move.md` のPP列は元々 `8` で一致していたため変更不要。

## ハンドラ構成（修正後）

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| ジャイロボール | `Event.ON_CALC_POWER_MODIFIER` | デフォルト（100） | `attacker:self`（`MoveHandler` デフォルト） | 自分と相手の実効素早さの比率から技の実威力を算出して返す |

* `Event.ON_CALC_POWER_MODIFIER` は `docs/spec/turn.md` に priority の記載がないため、
  既存の同種ハンドラ（`エレキボール_calc_power` ほか、`ON_CALC_POWER_MODIFIER` を使う
  全ての `MoveHandler`）が明示的な priority 指定なし＝デフォルト値（100）で登録されている
  慣例に合わせ、今回もデフォルトのままとする。

```python
def ジャイロボール_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """ジャイロボール: 自分の実効素早さが相手より遅いほど威力が高くなる。

    威力 = floor(25 × 相手の実効素早さ ÷ 自分の実効素早さ) + 1、最大150。
    実効素早さはランク補正・特性・もちもの・まひ・おいかぜ・しつげんの影響を含む。
    自分の実効素早さが 0 の場合は威力を 1 として扱う（第六世代以降の仕様）。
    """
    atk_speed = battle.speed_calculator.calc_effective_speed(ctx.attacker)
    def_speed = battle.speed_calculator.calc_effective_speed(ctx.defender)
    if atk_speed <= 0:
        power = 1
    else:
        power = min(150, 25 * def_speed // atk_speed + 1)
    return HandlerReturn(value=power * 4096)
```

```python
"ジャイロボール": MoveData(
    type="はがね",
    category="physical",
    pp=8,
    power=1,
    accuracy=100,
    flags={"bullet", "contact"},
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
            ha.ジャイロボール_calc_power,
        ),
    }
),
```

## リーサル計算

`docs/progress/move.md` の該当行は「リーサル実装: n/a」「リーサルテスト: n/a」のため対象外。
理由: 威力は使用者・対象双方の実効素早さにのみ依存し、相手（`defender`）のHP分布
`StateDist` を直接左右する要素ではないため、`lethal.py` 側の専用ハンドラは不要。

## テスト

既存の `tests/moves_attack/test_move_sa.py` に威力計算（速い攻撃側→低威力、遅い攻撃側→高威力、
上限150頭打ち、まひによる実効素早さ半減で威力増加）を確認するテストが一通り揃っており、
仕様・実装と照合した結果、修正・追加は不要と判断した。

## 注意点・エッジケース
