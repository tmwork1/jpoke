# 計画書: フラフラダンス

## 仕様要約

- ノーマルタイプ / 変化 / 命中100 / PP20 / 対象: 自分以外全体（シングルでは相手1体）
- 相手を1〜4ターンの間『こんらん』状態にする
- ダンス技（labels: dance）
- マイペース特性には無効（フレームワーク側が処理）
- こんらん済みの場合は失敗（フレームワーク側が処理）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | 相手にこんらん付与 |

## Priority 根拠

てんしのキッス・あやしいひかりと同種の `ON_STATUS_HIT` priority 100 を採用する。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `フェザーダンス` の後、`ひかりのかべ` の前（五十音順）。

```python
def フラフラダンス_apply_volatile_to_defender(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """フラフラダンスの効果: 相手をこんらん状態にする（2〜5ターン）。"""
    count = battle.random.randint(2, 5)
    return HandlerReturn(value=battle.volatile_manager.apply(
        ctx.defender, "こんらん", count=count, source=ctx.attacker, ctx=ctx
    ))
```

### data/move.py への更新

`"フラフラダンス"` エントリのハンドラを追加する。

```python
    "フラフラダンス": MoveData(
        type="ノーマル",
        category="変化",
        pp=20,
        accuracy=100,
        labels=["dance"],
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.フラフラダンス_apply_volatile_to_defender,
            ),
        }
    ),
```

## 注意点・エッジケース

- てんしのキッスと全く同じ実装パターン
- ダンス技であるため、おどりこ特性のポケモンにコピーされる（フレームワーク側が処理）
- シングルバトルでは事実上1体対象として動作する
- こんらん済み判定・マイペース無効はいずれもフレームワーク側が処理するためハンドラ内ガード不要
