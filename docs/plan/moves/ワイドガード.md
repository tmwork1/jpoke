# 計画書: ワイドガード

対象: Pokémon Champions シングルバトル
仕様書: `docs/spec/moves/ワイドガード.md`

## 仕様要約

- タイプ: いわ / カテゴリ: 変化 / PP: 12（champions基準。docs/champions/move_list.txt 970行目） /
  命中: なし（必中） / 対象: 味方の場 / 優先度: +3
- 使用ターン中、自分（ダブル・トリプルでは味方全体）を「複数のポケモンが対象になる技」から守る
- 一次情報によれば、シングルバトルでも技の対象範囲区分（スプレッド技かどうか）によって
  防御対象が決まり、実際に複数体へ命中したかどうかは関係ない

## 実装方針（本レビューでの結論）

本プロジェクトの `MoveData.target` はダブル・トリプルバトルにおける「相手全体」
「自分以外全体」等のスプレッド対象区分をモデル化していない（`MoveTarget` は
`"foe"` / `"foe_side"` / `"own_side"` / `"self"` / `"field"` のみ）。この区分を
再現するには技データ全体（いわなだれ・じしん・ハイパーボイス等、実機でダブル時に
相手全体を攻撃する技すべて）にスプレッド判定用フラグを追加する必要があり、
技システム全体に影響する大規模な変更になるため、本レビュー（ワイドガード単体）の
スコープ外とする。

したがって、本実装ではワイドガードを **防御効果を持たないデータ登録のみの技**
として扱う（`handlers={}`）。優先度+3による先制効果のみが機能する。
これは既存の `ハッピータイム`（`target="own_side"`、`handlers={}`）と同じパターン。

将来的にスプレッド技の対象範囲区分を実装する場合は、以下の対応が必要になる:

1. `MoveFlag` に `"spread"`（あるいは`MoveTarget`に`"all_foes"`等）を追加する
2. いわなだれ・じしん・マグニチュード・ハイパーボイス・ふぶき等、実機でダブル時に
   相手全体を攻撃する技すべてにフラグを付与する（docs/champions/move_list.txtの
   「対象」列が「相手全体」「自分以外全体」になっている技が該当）
3. ワイドガードに `Event.ON_TRY_MOVE_1` ハンドラ（`defender:self`想定、`まもる`と
   同様の`_run_protect`風の骨格）を追加し、上記フラグを持つ技をブロックする
4. ドラゴンアロー（`unprotectable`相当の除外）、ワイドフォース・テラクラスター
   （範囲拡張時のみ防御対象になる条件付き）などの例外を個別対応する

## `target` フィールド修正の根拠（レビューで発見・修正）

修正前は `target` フィールドが未指定のため `MoveData` のデフォルト値 `"foe"`
（相手1体を対象とする技）になっていた。これは以下の点で誤り・潜在バグだった:

* `Move.is_reflectable`（`model/move.py`）は `category == "status" and target in
  ("foe", "foe_side")` を満たすとマジックコート・マジックミラーで反射可能と判定する。
  `target="foe"`のままだと、ワイドガードが相手のマジックコートで跳ね返される
  誤った挙動になる（一次情報「マジックコート: ×」と矛盾）。
* `Move.is_blocked_by_protect`（`model/move.py`）は `target == "foe"` を満たすと
  まもる等で防がれる対象と判定する。`target="foe"`のままだと、相手のまもるに
  よってワイドガードの使用自体がブロックされる誤った挙動になる
  （一次情報「まもる: ×」と矛盾。ワイドガードは相手を対象としない技のため、
  相手のまもるに関係なく成功するべき）。
* `target="own_side"` に修正することで、上記いずれの判定も `target` が
  `"foe"`/`"foe_side"` に一致しなくなり、意図通り「反射不可・妨害不可」になる。
  `own_side` を採用したのは、champions記載「対象: 味方の場」および一次情報
  「範囲: 味方の場」に一致し、かつ既存の同種技（`リフレクター`・`おいかぜ`・
  `ハッピータイム`等）が`target="own_side"`を使っているパターンに合わせるため。

## テスト方針

- シングルバトルでは相手の通常攻撃を防げないこと（既存:
  `test_ワイドガード_シングルバトルでは相手の攻撃を防がない`）
- `target="own_side"`の修正を担保するテストを追加:
  - 相手のマジックコートで跳ね返されないこと
  - 相手のまもるに関係なく成功すること
