# 計画書: エレキボール（再レビュー）

## 仕様要約

* 自分（使用者）の実効素早さが相手より速いほど威力が大きくなる（`docs/spec/moves/エレキボール.md` 参照）。
* 比率 = floor(自分の実効素早さ ÷ 相手の実効素早さ)
* 比率0→40 / 1→60 / 2→80 / 3→120 / 4以上→150。相手の実効素早さが0のときは威力40固定。
* 実効素早さは、両者のランク補正・特性・もちもの・まひ・おいかぜ・しつげんの影響を受ける
  （対義技のジャイロボールと同様、`battle.speed_calculator.calc_effective_speed()` を使う）。

## 既存実装の確認結果・修正内容

* `src/jpoke/handlers/move_attack.py` の `エレキボール_calc_power` は、比率の閾値・威力段階
  （0→40/1→60/2→80/3→120/4以上→150）自体は仕様書と一致していたが、素早さの取得に
  `ctx.attacker.stats["spe"]` / `ctx.defender.stats["spe"]`（ランク・まひ・おいかぜ・特性・
  もちもの補正を含まない実数値）を使っており、仕様書の「両者のランク補正・特性・もちもの・
  まひ・おいかぜ・しつげんの影響を受ける」という記述と食い違っていた。
  対義技のジャイロボール（`ジャイロボール_calc_power`）が既に
  `battle.speed_calculator.calc_effective_speed()` を使って実効素早さを取得している実装と
  平仄を合わせるため、エレキボールも同様に `calc_effective_speed()` を使うよう修正する。
* `src/jpoke/data/moves/move_a.py` の `MoveData` の `pp` が `10`（本家基準の値）になっていたが、
  `src/jpoke/data/ps-champ-ja/moves.json` の値（12）と食い違っていたため `12` に修正する。
  `docs/progress/move.md` のPP列は元々 `12` で一致していたため変更不要。

## ハンドラ構成（修正後）

| 技名 | イベント | priority | subject_spec | 役割 |
|------|----------|----------|--------------|------|
| エレキボール | `Event.ON_CALC_POWER_MODIFIER` | デフォルト（100） | `attacker:self`（`MoveHandler` デフォルト） | 自分と相手の実効素早さの比率から技の実威力を算出して返す |

* `Event.ON_CALC_POWER_MODIFIER` は `docs/spec/turn.md` に priority の記載がないため、
  既存の同種ハンドラ（`ジャイロボール_calc_power` ほか、`ON_CALC_POWER_MODIFIER` を使う
  全ての `MoveHandler`）が明示的な priority 指定なし＝デフォルト値（100）で登録されている
  慣例に合わせ、今回もデフォルトのままとする。

```python
def エレキボール_calc_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """エレキボール: 自分の実効素早さ / 相手の実効素早さ の比率で威力を決定する。

    比率 = floor(自分の実効素早さ / 相手の実効素早さ)
    比率が 0 または相手の実効素早さが 0 の場合は威力 40。
    0: 40 / 1: 60 / 2: 80 / 3: 120 / 4以上: 150
    実効素早さはランク補正・特性・もちもの・まひ・おいかぜ・しつげんの影響を含む
    （トリックルームなどの行動順序補正は含まない）。
    """
    atk_speed = battle.speed_calculator.calc_effective_speed(ctx.attacker)
    def_speed = battle.speed_calculator.calc_effective_speed(ctx.defender)
    if def_speed <= 0:
        ratio = 0
    else:
        ratio = atk_speed // def_speed
    if ratio >= 4:
        power = 150
    elif ratio == 3:
        power = 120
    elif ratio == 2:
        power = 80
    elif ratio == 1:
        power = 60
    else:
        power = 40
    return HandlerReturn(value=power * 4096)
```

```python
"エレキボール": MoveData(

    pp=12,
    power=1,
    accuracy=100,

            ha.エレキボール_calc_power,

```

## リーサル計算

`docs/progress/move.md` の該当行は「リーサル実装: n/a」「リーサルテスト: n/a」のため対象外。
理由: 威力は使用者・対象双方の実効素早さにのみ依存し、相手（`defender`）のHP分布
`StateDist` を直接左右する要素ではないため、`lethal.py` 側の専用ハンドラは不要。

## テスト

既存の `tests/moves_attack/test_move_a.py` に威力段階（比率0/1/2/3/4以上）を確認するテストが
一通り揃っている。今回の修正（実効素早さの使用）を確認するため、まひによって攻撃側の
実効素早さが半減し威力段階が下がることを確認するテストを追加する
（対義技のジャイロボールの `test_ジャイロボール_まひで実効素早さが半減すると威力が増加する` と対になる観点）。
