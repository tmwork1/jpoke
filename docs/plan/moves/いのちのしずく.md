# 実装計画書: いのちのしずく

## 仕様要約

- タイプ: みず / カテゴリ: 変化 / PP: 10（チャンピオンズ: 12）/ 命中: なし / 対象: 味方全体
- 自分と味方全体の HP をそれぞれ最大 HP の 1/4 ずつ回復する（端数切り捨て）
- シングルバトルでは「味方全体」＝使用者自身のみが対象
- 特性「ちょすい」「よびみず」の味方は技の効果を無効化して特性が発動する（シングルバトルでは自分が使用者なので発動しない）
- 使用者のHP が満タンの場合は技が失敗する
- `heal` ラベルあり（`かいふくふうじ` の対象になる）
- ダイウォールの効果を受けない

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | HP 満タン時失敗チェック → 自分の HP を最大 HP の 1/4 回復 |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` セクションに以下の記載がある：

> | 100 | いのちのしずく: HP満タン |

このイベントはコード上の `ON_BEFORE_APPLY_MOVE` に対応する。ただし、同種の自己回復技（`じこさいせい`・`なまける`・`こうごうせい`・`つきのひかり`）はすべて `ON_STATUS_HIT`（priority=100）内でHP満タンチェックと回復処理を一括して行うパターンを採用している。`いのちのしずく` も同パターンで実装し、priority はデフォルト（100）とする。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「い」行 — `いたみわけ_equalize_hp` の後、`いちゃもん_apply_volatile_to_defender` の前に挿入する。

```python
def いのちのしずく_heal_self(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """いのちのしずく: 最大HPの1/4を回復する。HPが満タンの場合は失敗する。"""
    mon = ctx.attacker
    if mon.hp == mon.max_hp:
        return HandlerReturn(value=False, stop_event=True)
    battle.modify_hp(mon, r=1/4)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

既存の `"いのちのしずく"` エントリに handlers を追加する。

```python
"いのちのしずく": MoveData(
    type="みず",
    category="変化",
    pp=10,
    labels=["heal"],
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.いのちのしずく_heal_self,
        ),
    }
),
```

## 注意点・エッジケース

- シングルバトルでは「味方全体」の回復対象は使用者自身のみ。ダブルバトルは対象外のため、複数対象への回復処理は実装しない
- HP 満タン時の失敗チェックは ON_STATUS_HIT 内で行う（`じこさいせい` 等と同パターン）。`HandlerReturn(value=False, stop_event=True)` を返すと `move_executor.py` 内で `MOVE_FAILED` ログが記録される
- `ちょすい` / `よびみず` との相互作用はシングルバトルでは発生しない（自分が使用者の場合、これらの特性は発動しない）
- `かいふくふうじ` 状態のポケモンがこの技を使用した場合の無効化は、`かいふくふうじ` ハンドラ側で `heal` ラベルを参照して処理されるため、本ハンドラ側での追加チェックは不要
- `modify_hp(mon, r=1/4)` の端数処理（切り捨て）は `StatusManager.modify_hp` 内部で処理されるため、ハンドラ側では不要
- HP が 1 でも減っていれば成功し、回復量が 0（最大 HP が 4 未満など）でも失敗扱いにはならない点に注意（`mon.hp == mon.max_hp` で判定するため）
