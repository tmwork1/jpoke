# 実装計画書: いのちのしずく

## 仕様要約

- タイプ: みず / カテゴリ: 変化 / PP: 12（チャンピオンズ基準。src/jpoke/data/ps-champ-ja/moves.json準拠）/ 命中: なし（必中）/ 対象: 味方全体
- 自分と味方全体の HP をそれぞれ最大 HP の 1/4 ずつ回復する（**端数は四捨五入**。
  `ジャングルヒール` `みかづきのいのり` の切り捨てとは異なる仕様）
- シングルバトルでは「味方全体」＝使用者自身のみが対象
- `MoveData.target="self"` を明示する必要がある（デフォルト値 `"foe"` のままだと、
  `まもる` / `よびみず` / `ちょすい` / `かんそうはだ` / `おうごんのからだ` の
  `move.target == "foe"` 判定に誤って合致し、敵の特性・状態にブロックされてしまう）
- 特性「ちょすい」「よびみず」「かんそうはだ」の味方は技の効果を無効化して特性が発動する
  （シングルバトルでは自分が使用者なので、`target="self"` により発動しない）
- 使用者のHP が満タンの場合は技が失敗する
- `heal` ラベルあり（`かいふくふうじ` の対象・`ヒーリングシフト` の優先度+3対象になる）
- ダイウォールの効果を受けない（`target="self"` により対象外）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | HP 満タン時失敗チェック → 自分の HP を最大 HP の 1/4（四捨五入）回復 |

## Priority 根拠

`docs/spec/turn.md` の `Event.ON_APPLY_MOVE` セクションに以下の記載がある：

> | 100 | いのちのしずく: HP満タン |

このイベントはコード上の `ON_BEFORE_APPLY_MOVE` に対応する。ただし、同種の自己回復技（`じこさいせい`・`なまける`・`こうごうせい`・`つきのひかり`）はすべて `ON_STATUS_HIT`（priority=100）内でHP満タンチェックと回復処理を一括して行うパターンを採用している。`いのちのしずく` も同パターンで実装し、priority はデフォルト（100）とする。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「い」行 — `いたみわけ_equalize_hp` の後、`いちゃもん_apply_volatile_to_defender` の前に挿入する。

```python
def いのちのしずく_heal(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """いのちのしずく: 最大HPの1/4を回復する。HPが満タンの場合は失敗する。

    端数は四捨五入で丸める（一次情報: docs/wiki/moves/いのちのしずく.html 技の仕様節）。
    同じ1/4回復技でも `ジャングルヒール` `みかづきのいのり` は切り捨てのため、
    本技のみ `round_half_up` を使う点に注意。
    """
    mon = ctx.attacker
    if mon.hp == mon.max_hp:
        return HandlerReturn(value=False, stop_event=True)
    battle.modify_hp(mon, v=round_half_up(mon.max_hp / 4))
    return HandlerReturn(value=value)
```

`round_half_up` は `jpoke.utils.math` からインポートする（ドレイン技の回復量計算と同じ丸め方式）。

### `src/jpoke/data/moves/move_a.py`

既存の `"いのちのしずく"` エントリ:

```python
"いのちのしずく": MoveData(
    type="みず",
    category="status",
    pp=12,
    target="self",
    flags={"heal"},
    handlers={
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.いのちのしずく_heal,
        ),
    }
),
```

`target="self"` を明示することで、以下の判定が正しく機能する:

- `Move.is_blocked_by_protect`（`target == "foe"` が前提）→ `まもる` に防がれない
- `_apply_type_absorb`（`よびみず` / `ちょすい` / `かんそうはだ` の判定。`target == "foe"` が前提）
  → 敵の吸収特性が誤発動しない
- `おうごんのからだ_block_status_move` など（`target == "foe"` が前提）→ 誤ってブロックされない

## 注意点・エッジケース

- シングルバトルでは「味方全体」の回復対象は使用者自身のみ。ダブルバトルは対象外のため、複数対象への回復処理は実装しない
- HP 満タン時の失敗チェックは ON_STATUS_HIT 内で行う（`じこさいせい` 等と同パターン）。`HandlerReturn(value=False, stop_event=True)` を返すと `move_executor.py` 内で `MOVE_FAILED` ログが記録される
- `ちょすい` / `よびみず` / `かんそうはだ` との相互作用はシングルバトルでは発生しない（`target="self"` のため、自分が使用者の場合はこれらの特性は発動しない）
- `かいふくふうじ` 状態のポケモンがこの技を使用した場合の無効化は、`かいふくふうじ` ハンドラ側で `heal` ラベルを参照して処理されるため、本ハンドラ側での追加チェックは不要
- 端数処理（四捨五入）は `round_half_up(mon.max_hp / 4)` で行う。`modify_hp` の `r=` 引数は内部で切り捨て（`int()`）を行うため、本技では使用できない
- HP が 1 でも減っていれば成功し、回復量が 0（最大 HP が 1 など極端に小さい場合）でも失敗扱いにはならない点に注意（`mon.hp == mon.max_hp` で判定するため）
