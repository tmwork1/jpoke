# 計画書: メロメロ

## 仕様要約

- ノーマルタイプ / 変化 / 命中100 / PP16 / 対象: 1体選択
- 相手が自分とは異なる性別の場合、相手を『メロメロ』状態にする
- 性別が同じ、またはどちらかが性別不明の場合は失敗
- みがわり状態の相手にも成功する（`bypass_substitute`）
- 『メロメロ』状態になると50%の確率で攻撃できなくなる（詳細は `docs/spec/volatiles/メロメロ.md`）
- どんかん・アロマベール・重複付与による無効化は `volatile_manager.apply` 側が担保するため
  技側での個別チェックは不要

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_TRY_MOVE_2` | 120 | attacker:self | 性別が同じ・不明の場合に技を失敗させる |
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | 相手にメロメロ付与 |

## Priority 根拠

`docs/spec/turn.md` の `ON_TRY_MOVE_2` に「120 | メロメロ: 性別同じ・不明」と明記されている。
`ON_STATUS_HIT` は既存の同種ハンドラ（`しびれごな`・`でんじは` 等）と同じくデフォルト priority を使用する。

## 実装箇所

### handlers/move_status.py

`メロメロ_check_gender`（`ON_TRY_MOVE_2`）と `メロメロ_apply`（`ON_STATUS_HIT`）を実装済み。
配置は五十音順で `めいそう` の後、`もりののろい` の前。

```python
def メロメロ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """メロメロの効果: 相手をメロメロ状態にする。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="メロメロ")


def メロメロ_check_gender(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """メロメロの失敗条件: 相手が同性または性別不明の場合は失敗させる。"""
    attacker = ctx.attacker
    defender = ctx.defender
    # 性別不明（""）または同性の場合は失敗
    if (defender.gender == ""
            or attacker.gender == ""
            or attacker.gender == defender.gender):
        battle.add_event_log(
            attacker, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="メロメロ_性別不一致")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)
```

### data/moves/move_ma.py への更新

`"メロメロ"` エントリの PP を `docs/champions/move_list.txt` 基準の 16 に修正し、
`flags={"bypass_substitute"}` を追加する。

```python
"メロメロ": MoveData(
    type="ノーマル",
    category="status",
    pp=16,
    accuracy=100,
    flags={"bypass_substitute"},
    handlers={
        Event.ON_TRY_MOVE_2: h.MoveHandler(

            priority=120,

            hs.メロメロ_apply,
        ),

```

## 注意点・エッジケース

- 性別チェック（同性・不明の場合の失敗）は技側の `ON_TRY_MOVE_2` priority=120 ハンドラで実装する
  （フレームワーク側の共通処理ではない）。
- どんかん特性・アロマベール特性による無効、重複付与の防止は `volatile_manager.apply` が
  内部で処理するため、技側ハンドラでのチェックは不要。
- みがわり状態の相手にも技が成功するため、`flags={"bypass_substitute"}` を忘れないこと。
- あかいいとによる反射（使用者側もメロメロになる）はアイテム側
  （`handlers/item.py` の `あかいいと_infatuate_foe`）の実装であり、技側の対応は不要。
- `apply_volatile_to_defender` のラッパーパターンを使用（`もりののろい_apply` 等と同じ形式）。
