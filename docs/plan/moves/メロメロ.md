# 計画書: メロメロ

## 仕様要約

- ノーマルタイプ / 変化 / 命中100 / PP15 / 対象: 1体選択
- 相手が自分とは異なる性別の場合、相手を『メロメロ』状態にする
- 性別が同じ、またはどちらかが性別不明の場合は失敗
- 『メロメロ』状態になると50%の確率で攻撃できなくなる
- すでにメロメロ状態の場合は失敗（`ON_TRY_MOVE_2` priority 130 でフレームワーク側が処理）
- 性別チェック（同性/不明）は `ON_TRY_MOVE_2` priority 120 でフレームワーク側が処理

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略） | 相手にメロメロ付与 |

## Priority 根拠

`docs/spec/turn.md` の `ON_TRY_MOVE_2` に `メロメロ: 性別同じ・不明` が priority 120、`メロメロ: すでにメロメロ状態` が priority 130 で記載されており、これらはフレームワーク側が処理する。ハンドラ本体は既存の `ON_STATUS_HIT` priority 100 のパターンに従う。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `まもる` 関連の後、`みがわり` の前（五十音順）。

```python
def メロメロ_apply_volatile_to_defender(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """メロメロの効果: 相手をメロメロ状態にする。"""
    return apply_volatile_to_defender(battle, ctx, value, volatile="メロメロ")
```

### data/move.py への更新

`"メロメロ"` エントリのハンドラを追加する。

```python
    "メロメロ": MoveData(
        type="ノーマル",
        category="変化",
        pp=15,
        accuracy=100,
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(

```

## 注意点・エッジケース

- 性別チェック（同性・不明の場合の失敗）は `ON_TRY_MOVE_2` priority 120 でフレームワーク側が処理するためハンドラ内チェック不要
- すでにメロメロ状態の失敗チェックも `ON_TRY_MOVE_2` priority 130 でフレームワーク側が処理
- どんかん特性・アロマベール特性による無効も `ON_TRY_MOVE_2` priority 140 でフレームワーク側が処理
- `apply_volatile_to_defender` のラッパーパターンを使用（`ロックオン_apply_volatile_to_defender` 等と同じ形式）
