# 計画書: へんしん

## 仕様要約

* ノーマルタイプ / 変化 / 威力- / 命中-（必中） / PP12 / 対象: 相手1体 / 優先度0 / 接触なし
* 相手の種族・タイプ・特性・技（PPは一律5固定）・実数値ステータス・能力ランク補正をコピーして
  変身する。HP・レベル・個体値・性格・アイテム・状態異常は自身のものを維持する。
  交代/瀕死で元の姿に戻る。
* 詳細は `docs/spec/moves/へんしん.md` を参照。

## 実装保留の判断

本技の完全な実装には、`Pokemon` モデルに以下の大規模な変更が必要となる。

1. 種族（`self.data`）・タイプ・特性・技リスト・実数値ステータスを一時的に「対象のコピー」で
   上書きし、交代/瀕死のタイミングで完全に元へ復元する仕組み。
   現在の `set_form`（`src/jpoke/model/pokemon.py`）は種族切り替えとステータス再計算を伴う
   恒久的な変更専用であり、「対象の実数値をそのままコピーし、退場時に完全復元する」用途には
   使えない。
2. 実数値ステータスを自身の種族値・個体値・努力値から再計算するのではなく、対象の実数値を
   そのまま固定値として保持する仕組み。`PokemonStats`（`src/jpoke/model/stats.py`）は種族値
   ベースの計算専用で、外部からの実数値上書きに対応していない。
3. コピーした技のPPを一律5（元の技の最大PPが1の場合のみ1）に設定し、通常のPP消費・PPアップ等
   の仕組みとは独立してPPを管理する必要がある。
4. 特性かわりもの（メタモン専用の隠れ特性、`docs/spec/abilities/かわりもの.md`）とほぼ同一の
   コピー処理を共有する必要があり、へんしん単体を場当たり的に実装すると、かわりもの実装時に
   再度大規模な手戻りが発生する。

上記の理由から、スケッチ・ゆびをふる等の前例（`docs/progress/move.md` で実装保留として扱われて
いる複雑技）と同様に、本技は実装保留として扱う。

## data/moves/move_ha.py の修正内容

```python
"へんしん": MoveData(
    type="ノーマル",
    category="status",
    pp=12,  # champions基準（src/jpoke/data/ps-champ-ja/moves.json）。旧値10はSV本家基準の移行漏れ。
    # accuracy省略=必中。潜伏中の相手への失敗はHIDDEN_MOVE_ALLOWED_MOVES側で処理される。
    flags={"non_encore", "non_copycat", "unprotectable", "unreflectable"},
    # 実装保留: 相手の種族・実数値ステータス・特性・技をまるごとコピーし退場時に
    # 完全復元する大規模な変身機構（特性かわりものと共有基盤）が必要なため対応を見送る。
    # 詳細は docs/plan/moves/へんしん.md 参照（前例: スケッチ・ゆびをふる）。
    handlers={},
),
```

* `pp`: `src/jpoke/data/ps-champ-ja/moves.json` の値（12）に修正する。旧実装値10はSV本家基準の値の
  移行漏れである。
* `accuracy`: 必中技のため明示的な値を持たせず省略する（デフォルト`None`）。これは`あくび`・
  `ねむりごな`等の他の必中変化技と同じ書き方に合わせたもので、以前の`accuracy=100`は誤り
  （回避率補正等の影響を受けてしまう）だった。
* 潜伏中の相手（あなをほる等）への失敗は `handlers/volatile.py` の `HIDDEN_MOVE_ALLOWED_MOVES`
  により自動的に処理される（へんしんは許可リストに含まれないため成功しない）。専用ハンドラは
  不要。
* `flags`: `non_encore`（アンコール対象外。`docs/spec/volatiles/アンコール.md` に記載済み）に加え、
  `non_copycat`（まねっこで選ばれない）・`unprotectable`（まもるに防がれない。
  `src/jpoke/data/ps-champ-ja/moves.json` の「守×」に対応）・`unreflectable`（マジックコートで跳ね返されない。
  `Move.is_reflectable` はデフォルトで `category="status" かつ target="foe"` の技を反射対象と
  みなすため、明示的なフラグ付与が必須。実装漏れがあり本レビューで追加した）を持たせる。
  ものまね（`data/moves/move_ma.py`）も同様のパターンで
  `flags={"non_encore", "non_negoto", "non_copycat", "unreflectable"}` を持つ。

  `Pokemon.ranked_stats` の算出元を種族値ベースと固定値ベースで切り替えられるようにする
  必要がある。

  `Battle` に実装し、へんしん・かわりもの双方から呼び出す設計が望ましい。
