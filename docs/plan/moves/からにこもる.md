## Priority 根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 内の自己ランク変化技の記載なし。
既存の同種ハンドラ（`てっぺき_modify_attacker_stats` 等）が priority=100（デフォルト）で
登録されているため、100 を使用する。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

`からにこもる_modify_attacker_stats` は実装済み（`からげんき_ignore_burn_modifier` の後、
`からみつく_lower_defender_spd` の前）。

```python
def からにこもる_modify_attacker_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return modify_attacker_stats(battle, ctx, value, stats={"def": 1})
```

### `src/jpoke/data/moves/move_ka.py`

`"からにこもる"` エントリに `target="self"` を追加する（レビューで判明した欠落。
`target` 未指定時のデフォルトは `"foe"` であり、`Move.is_blocked_by_protect` /
`Move.is_reflectable` がいずれも `target == "foe"` を条件に判定するため、
未設定のままだと『まもる』で防がれたり『マジックコート』で反射されたりする誤動作になる）。

```python
    "からにこもる": MoveData(
        type="みず",
        category="status",
        pp=40,
        target="self",
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.からにこもる_modify_attacker_stats,
            )
        }
    ),
```

## 注意点・エッジケース

- `てっぺき`・`とける`と同型の自己ランク変化技だが、上昇幅は1段階のみ（`てっぺき`・`とける`は2段階）
- `target="self"`が未設定だとデフォルトの`"foe"`が使われ、`まもる`で防がれる・
  `マジックコート`で反射されるという誤った挙動になるため、必ず明示する
- ランクがすでに最大（+6）の場合の失敗は `modify_stats` 内部の `clamp_stats` がガードするため、
  ハンドラ側での最大チェックは不要
- `src/jpoke/data/ps-champ-ja/moves.json` に本技のエントリが存在しないため、PPはChampions向けの
  8/12/16/20への正規化対象外として第9世代の値（40）を維持する。将来的にChampions公式データで
  本技のPPが判明した場合は本ファイルと`docs/progress/move.md`を更新すること
