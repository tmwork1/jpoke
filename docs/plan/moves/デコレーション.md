# 計画書: デコレーション

## 仕様要約

- フェアリータイプ / 変化 / 命中-（必中）/ PP16（`src/jpoke/data/ps-champ-ja/moves.json` L522） /
  対象: 自分以外の1体（本プロジェクトはシングル限定のため実質的に相手固定）
- 相手の『こうげき』『とくこう』ランクを2段階ずつ上げる
- まもる・ダイウォールを貫通する（`unprotectable`）
- マジックコート・マジックミラーで跳ね返されない（`unreflectable`）
- みがわり状態には防がれる（`bypass_substitute` フラグなし。既存の汎用みがわり判定に従う）
- 失敗条件は専用の `_can_apply` ハンドラを持たず、`modify_defender_stats` ヘルパーが
  「両方のランクとも変化しなかった場合は空dictを返し、`move_executor` 側の汎用判定で
  技全体が失敗扱いになる」という既存の仕組みに従う（`つぶらなひとみ`・`ちょうのまい` と同型）。

## 既存実装の誤り（本レビューで修正が必要な点）

`src/jpoke/data/moves/move_ta.py` の既存 `"デコレーション"` エントリには以下の誤りがあった。

- PP が `15`（Gen9本家基準）のままで、Champions基準の `16` に移行されていなかった。
- `accuracy=100` が明示指定されており、必中（`accuracy` 省略でNone）になっていなかった。
- `handlers={}` のままで、能力ランク変化の効果が一切実装されていなかった。
- `flags` が未設定で、`unprotectable` / `unreflectable` が欠落しており、まもる・ダイウォール
  ・マジックコートで防がれてしまう状態だった。

さらに `docs/progress/move.md` のデコレーション行は全列「ダブル専用」（対象外）と
記載されていたが、`docs/spec/moves/デコレーション.md` の調査の通り、本技は「自分以外の
1体（相手を含む）」が対象であり、シングルバトルでも失敗せずに成立する技のため、
この区分は誤りだった（アロマミスト等の「味方1体が対象」専用技とは異なる）。

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_STATUS_HIT` | 100（デフォルト） | attacker:self（省略、AttackContextの規約） | こうげき+2、とくこう+2 |

## Priority根拠

`docs/spec/turn.md` に `ON_STATUS_HIT` 自体の記載はない（ダメージ計算・状態変化技の
内部処理イベントのため）。既存の同型パターン（`つぶらなひとみ_modify_defender_stats`・
`ちょうのまい_modify_attacker_stats` 等、能力ランク変化のみを行う変化技）がいずれも
デフォルト priority（100）の `ON_STATUS_HIT` のみで実装されているため、これに倣う。

## 実装箇所

### handlers/move_status.py への追加

挿入位置: `でんじふゆう_check_can_use` の後、`とおせんぼう_apply` の前（五十音順、「で」グループ）。

```python
def デコレーション_modify_defender_stats(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """デコレーションの効果: 相手のこうげき・とくこうを2段階ずつ上げる。"""
    return modify_defender_stats(battle, ctx, value, stats={"atk": 2, "spa": 2})
```

`modify_defender_stats` は `handlers/move.py` で定義済みの共通ヘルパー（`つぶらなひとみ` 等で
使用実績あり）であり、`handlers/move_status.py` の先頭で既にインポート済みのため追加の
import 変更は不要。

### data/moves/move_ta.py の更新

既存の `"デコレーション"` エントリを修正する。

```python
    "デコレーション": MoveData(
        type="フェアリー",
        category="status",
        pp=16,
        flags={"unprotectable", "unreflectable"},
        handlers={
            Event.ON_STATUS_HIT: h.MoveHandler(
                hs.デコレーション_modify_defender_stats,
            ),
        }
    ),
```

## 注意点・エッジケース

- 対象は `MoveTarget="foe"`（デフォルト値のため明示不要）。ゲーム内では「自分以外の1体
  （味方 or 相手）」が対象だが、本プロジェクトはシングルバトルのみを扱うため、味方が
  存在せず実質的に「相手」固定になる。
- 相手のランクを+2ずつ上げる、通常は自分に不利な（利敵的な）効果を持つ技だが、ゲーム
  ルール上シングルバトルでも正当に使用可能（Wikiの「こんなときに使おう」に、控えの
  メタモンにランク補正をコピーさせる運用例の記載あり）。よって未実装のまま「ダブル専用」
  として据え置くことはせず、通常のシングル対応技として実装する。
- トリックガードで防がれる仕様があるが、トリックガード自体が本プロジェクトで未実装のため
  対応するフラグ・ハンドラは追加しない（トリックガード実装時に追って対応）。
- 必中技のため `accuracy` は指定しない（デフォルト `None`）。姿を隠している（溜め技中）
  相手に当たらない挙動は、既存の汎用の命中判定の枠組み（各溜め技側の `ON_CHECK_HIT` 等）
  に従うため追加実装は不要。
- 専用技（マホイップ）だが、習得可否のバリデーションは本プロジェクトのバトルエンジンの
  対象外（他の専用技と同様、使用可否はテスト側で保証する）。
- `battle.modify_stats` は個別のランク上限チェックを行うため、こうげき・とくこうの
  どちらか一方のみが+6で頭打ちの場合でも、もう一方は正常にランクアップし、技全体は成功
  として扱われる（両方とも+6で頭打ちの場合のみ技全体が失敗する）。
