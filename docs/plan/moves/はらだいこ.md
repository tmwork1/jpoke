# 実装計画書: はらだいこ

## 仕様要約（docs/spec/moves/はらだいこ.md 参照）

- タイプ: ノーマル / カテゴリ: 変化 / PP: 12（champions基準） / 対象: 自分（`MoveData.target="self"` を明示）
- 自分の最大HPの半分（切り捨て）を消費して、こうげきランクを最大（+6）まで上げる
- 失敗条件:
  - こうげきランクがすでに+6の場合
  - 現在HPが最大HPの半分以下（`hp <= max_hp // 2`）の場合
- あまのじゃく特性の場合はランク変化の符号が反転する（`battle.modify_stats` が汎用的に処理するため
  本技側での特別対応は不要）。判定は反転前の生のランク値に対して行う。

## 現状の実装状況

| 項目 | 状態 |
|------|------|
| `data/move.py`（`data/moves/move_ha.py`） エントリ | 登録済み（`target="self"`, `pp=12`） |
| `handlers/move_status.py` 関数 | 実装済み（`はらだいこ_can_apply` / `はらだいこ_apply`） |

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_BEFORE_APPLY_MOVE` | 100（デフォルト） | `attacker:self`（デフォルト） | こうげきランクが+6、またはHPが最大HPの半分以下ならば失敗判定 |
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | こうげきランクを+6に設定し、HPを最大HPの半分（切り捨て）消費 |

## Priority 根拠

- `ON_BEFORE_APPLY_MOVE` priority 100: `docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに
  「はらだいこ: 失敗条件」が priority 100 として明記されている（実装上のイベント定数は
  `Event.ON_BEFORE_APPLY_MOVE`）。
- `ON_STATUS_HIT` priority 100（デフォルト）: turn.md に個別記載なし。同種の自己強化技
  （`つるぎのまい` 等）と同じデフォルト優先度パターン。

## 実装箇所

### `src/jpoke/handlers/move_status.py`

```python
def はらだいこ_can_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """はらだいこの使用条件チェック: こうげきランクがすでに+6、またはHPが最大HPの半分以下ならば失敗する。"""
    mon = ctx.attacker
    if mon.rank["atk"] >= 6 or mon.hp <= mon.max_hp // 2:
        battle.add_event_log(
            mon, LogCode.MOVE_FAILED,
            payload=FailureLogPayload(move=ctx.move.name, display_reason="はらだいこ")
        )
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def はらだいこ_apply(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """はらだいこの効果: こうげきランクを最大まで上げ、HPを最大HPの半分消費する。"""
    mon = ctx.attacker
    delta = 6 - mon.rank["atk"]
    battle.modify_stats(mon, {"atk": delta}, source=mon)
    battle.modify_hp(mon, r=-0.5)
    return HandlerReturn(value=value)
```

### `src/jpoke/data/moves/move_ha.py`

```python
"はらだいこ": MoveData(
    type="ノーマル",
    category="status",
    pp=12,
    target="self",
    handlers={
        Event.ON_BEFORE_APPLY_MOVE: h.MoveHandler(
            hs.はらだいこ_can_apply,
        ),
        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.はらだいこ_apply,
        ),

```

## 注意点・エッジケース

- `modify_stats` は相対変化量を受け取るため、`6 - mon.rank["atk"]` で現在ランクとの差分を計算して
  渡す。あまのじゃく特性の場合は `modify_stats` 内部で符号が反転し、こうげきランクは最低（-6）まで
  下がる（本技側の追加実装は不要）。
- `battle.modify_hp(mon, r=-0.5)` は `int(max_hp * -0.5)` により端数切り捨てのマイナス方向計算となり、
  `-(max_hp // 2)` と同じ結果になる。
- `MoveData.target` を `"self"` に明示しないとデフォルト値 `"foe"` になり、
  `Move.is_blocked_by_protect` / `Move.is_reflectable` が誤って真になる
  （相手のまもる・マジックコートで防がれてしまう）。**この誤りが過去の実装に存在したため、
  今回のレビューで `target="self"` を追加した。**
- PP は champions 基準で 12（`docs/champions/move_list.txt` 準拠）。Gen9本家基準の 10 ではない。
  **この誤りが過去の実装に存在したため、今回のレビューで 12 に修正した。**
- HP が最大HPの半分以下の場合の失敗チェックは、こうげきランクの失敗チェックと同じ
  `ON_BEFORE_APPLY_MOVE`（priority 100）で行う。同一の `FailureLogPayload` にまとめる方式は、
  `なかまづくり_can_apply` 等の既存の複数条件失敗ハンドラと同じパターン。
