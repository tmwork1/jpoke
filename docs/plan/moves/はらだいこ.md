# 実装計画書: はらだいこ

## 仕様要約

- タイプ: ノーマル / カテゴリ: 変化 / PP: 10 / 対象: 自分（target はデフォルト）
- 自分の HP を最大 HP の半分だけ減らす代わりに、「こうげき」ランクを最大（+6）まで上げる
- 現在の A ランクがすでに +6 なら失敗する
- HP が最大 HP の半分以下（減らす量以上のダメージを受けている状態）でも使用自体は可能だが、HP が 1 未満になる場合は使えないかどうかは仕様書に明記なし（失敗条件として HP 不足は記載なし）

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|----------|----------|--------------|------|
| `ON_BEFORE_APPLY_MOVE` | 100（デフォルト） | `attacker:self`（デフォルト） | A ランクが +6 ならば失敗判定 |
| `ON_STATUS_HIT` | 100（デフォルト） | `attacker:self`（デフォルト） | A ランクを +6 に設定し HP を半分消費 |

## Priority 根拠

- `ON_BEFORE_APPLY_MOVE` priority 100: `docs/spec/turn.md` の `Event.ON_APPLY_MOVE` テーブルに「はらだいこ: 失敗条件」が priority 100 として明記されている
- `ON_STATUS_HIT` priority 100（デフォルト）: 同種ハンドラ（自己強化技）と同じパターン

## 実装箇所

### `src/jpoke/handlers/move_status.py`

五十音順の挿入位置: 「は」行 — `ねをはる` 系の後、「は」行の適切な位置に挿入する。「はらだいこ」は「は」行内では「はねやすめ」「ハバネロエキス」の後に来る。

```python
def はらだいこ_can_apply(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """はらだいこの使用条件チェック: こうげきランクがすでに +6 ならば失敗する。"""
    if ctx.attacker.rank["A"] >= 6:
        battle.add_event_log(ctx.attacker, LogCode.MOVE_FAILED,
                             payload={"reason": "はらだいこ"})
        return HandlerReturn(value=False, stop_event=True)
    return HandlerReturn(value=value)


def はらだいこ_apply(battle: Battle, ctx: EventContext, value: Any) -> HandlerReturn:
    """はらだいこの効果: こうげきランクを最大まで上げ、HP を最大 HP の半分消費する。"""
    mon = ctx.attacker
    # こうげきランクを +6 まで上げる（現在値との差分を渡す）
    delta = 6 - mon.rank["A"]
    battle.modify_stats(mon, {"A": delta}, source=mon)
    # HP を最大 HP の半分消費する
    battle.modify_hp(mon, -(mon.max_hp // 2))
    return HandlerReturn(value=value)
```

### `src/jpoke/data/move.py`

既存の `"はらだいこ"` エントリに handlers を追加する。

```python
"はらだいこ": MoveData(

    category="変化",
    pp=10,
    handlers={
        Event.ON_BEFORE_APPLY_MOVE: h.MoveHandler(

        Event.ON_STATUS_HIT: h.MoveHandler(
            hs.はらだいこ_apply,
        ),

```

## 注意点・エッジケース

- `modify_stats` は相対変化量を受け取るため、`6 - mon.rank["A"]` で現在ランクとの差分を計算して渡す
- `battle.modify_hp(mon, -(mon.max_hp // 2))` で HP を消費する。HP が 1 以下になる場合のひんし処理は `modify_hp` 内部で行われるため、ハンドラ側では追加チェック不要
- A ランクがすでに +6 の場合の失敗チェックは `ON_BEFORE_APPLY_MOVE` で行う（turn.md に priority 100 で明記）
- `ON_TRY_MOVE_2` テーブルの「ランク補正上げ: すでに最大」（priority 130）と重複する可能性があるが、はらだいこは「攻撃ランクを最大にする」技なので `ON_BEFORE_APPLY_MOVE` で明示的に失敗させる方が確実
- HP 消費は `ON_STATUS_HIT`（技命中後）で行うため、命中率は関係なし（この技は命中率なし）
