# はたきおとす 実装計画書

## 仕様要約（docs/spec/moves/はたきおとす.md 参照）

* タイプ: あく / 分類: 物理 / 威力: 65 / 命中: 100 / PP: 20
* 対象: 相手1体 / 優先度: 0 / 接触あり
* 効果: 命中時に相手の道具を失わせる。相手が道具を持っている場合は威力が1.5倍になる。
  ただし特定の種族と持ち物の組み合わせ（専用道具・メガストーン等）は奪えず威力補正もかからない。

## 現状の実装状況

`src/jpoke/data/moves/move_ha.py` に登録済み。

```python
"はたきおとす": MoveData(
    type="あく",
    category="physical",
    pp=20,
    power=65,
    accuracy=100,
    flags={"contact"},
    handlers={
        Event.ON_CALC_POWER_MODIFIER: h.MoveHandler(
            ha.はたきおとす_power,
        ),
        # docs/spec/turn.md ON_DAMAGE: 「100 はたきおとす等のアイテム効果」
        # くっつきバリの転移判定（priority=30）より後に発動する必要があるため ON_DAMAGE_HIT を使用する。
        Event.ON_DAMAGE_HIT: h.MoveHandler(
            ha.はたきおとす_remove_item,
        )
    }
),
```

## ハンドラ構成

| イベント | priority | subject_spec | 役割 |
|---|---|---|---|
| `ON_CALC_POWER_MODIFIER` | 100（デフォルト。ダメージ計算内部の専用イベントで turn.md 未掲載） | attacker:self（`MoveHandler` 既定値） | 相手が道具を持ち、かつ奪える場合に威力を1.5倍にする |
| `ON_DAMAGE_HIT` | 100（`docs/spec/turn.md` ON_DAMAGE: 「100 はたきおとす等のアイテム効果」） | attacker:self（`MoveHandler` 既定値） | 命中後に相手の道具を除去する |

```python
def はたきおとす_power(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """はたきおとすのアイテム所持時1.5倍補正。

    奪う/落とすことができないアイテム（だいこんごうだま等の専用道具）の場合は補正なし。
    ねんちゃく等、除去のみを防ぐ特性の場合は威力補正の対象になるため、
    dry_run=True で実際の除去を伴わない判定として問い合わせる。
    """
    if (
        ctx.defender.has_item()
        and battle.item_manager.can_change_item(ctx.defender, source=ctx.attacker, dry_run=True)
    ):
        value = apply_fixed_modifier(value, 6144)
    return HandlerReturn(value=value)


def はたきおとす_remove_item(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """はたきおとすのアイテム除去効果。"""
    battle.item_manager.remove_item(target=ctx.defender, source=ctx.attacker)
    return HandlerReturn(value=value)
```

道具の奪取可否は `ItemManager.can_change_item()` が発火する `Event.ON_CHECK_ITEM_CHANGE` の共通イベントで
判定する。専用道具（だいこんごうだま・くちたけん・くちたたて・メガストーン・オーガポンのめん等）は
各アイテム側（一部はマルチタイプ・ARシステム特性側）に登録された `ON_CHECK_ITEM_CHANGE` ハンドラが
種族一致を判定して除去を防ぐ。`removable=False` は種族+持ち物ロックの対象アイテムを示すメタデータ。

みがわり状態は `ON_MODIFY_MOVE_DAMAGE` でダメージを0にするため、`actual_damage<=0` となり
`ON_DAMAGE_HIT` 自体が発火せず、道具除去も自然に行われない（威力補正は先に確定しているため影響しない）。
ばけのかわ・アイスフェイスは `ON_MODIFY_MOVE_DAMAGE` で少量の固定ダメージに変換するのみで
`ON_DAMAGE_HIT` は通常通り発火するため、追加の実装なしで「防がれても道具を失わせる」仕様を満たす。

## レビューで発見した不具合と修正

### 1. ねんちゃく所持相手への威力補正が誤って無効化されていた

`はたきおとす_power` が `can_change_item()` をそのまま呼び出していたため、除去可否判定に
ねんちゃく特性の `ON_CHECK_ITEM_CHANGE` ハンドラ（`ねんちゃく_prevent_item_change`）まで
含まれてしまい、以下の問題があった。

* 仕様上は「ねんちゃく持ちの相手には道具を落とせないが、威力補正はかかる」
  （`docs/spec/abilities/ねんちゃく.md` にも明記済み）にもかかわらず、威力補正がかからなかった。
* `はたきおとす_power`（威力判定）と `はたきおとす_remove_item`（実際の除去）の両方から
  `can_change_item()` が呼ばれるため、ねんちゃくの特性発動ログが二重に記録されていた
  （威力判定の時点で本来発動していないはずの特性表示が出てしまう）。

**修正**: `EventContext` に `dry_run: bool = False` フィールドを追加し、
`ItemManager.can_change_item(..., dry_run: bool = False)` から `ON_CHECK_ITEM_CHANGE` へ伝播するようにした。
`ねんちゃく_prevent_item_change` は `ctx.dry_run` が True の場合（実除去を伴わない問い合わせ）は
ブロック・特性発動ログの記録を行わずに素通りさせる。`はたきおとす_power` は
`can_change_item(ctx.defender, source=ctx.attacker, dry_run=True)` を呼ぶことで、
威力補正の判定時にはねんちゃくの発動を起こさないようにした。
種族+持ち物ロック側のハンドラ（だいこんごうだま・くちたけん・マルチタイプ等）は `dry_run` を
参照しないため、これらのケースでは引き続き威力補正が無効化される（仕様通り）。

### 2. メガストーン・オーガポンのめんに種族ロックが未実装だった

`removable=False` が設定されているにもかかわらず、メガストーン（`_add_mega_stones` で
生成される全アイテム）・オーガポンのめん（いしずえのめん・いどのめん・かまどのめん）には
`Event.ON_CHECK_ITEM_CHANGE` ハンドラが一切登録されておらず、対応する種族が持っていても
はたきおとす（および他の道具移動系の技・特性）で奪える状態になっていた。

**修正**:
* `handlers/item.py` に `mega_prevent_item_change`（`MEGA_STONES` を参照し、対象ポケモンの種族が
  対応するメガシンカ前後のフォームに含まれる場合のみ除去を防ぐ）を追加し、
  `_add_mega_stones()` で生成する全メガストームアイテムの `handlers` に登録した。
* `handlers/item.py` に `オーガポンのめん_prevent_item_change`（既存の `_dedicated_item_prevent_item_change`
  ヘルパーで種族プレフィックス「オーガポン」を判定）を追加し、いしずえのめん・いどのめん・
  かまどのめんの3アイテムにそれぞれ登録した。

いずれも既存の `だいこんごうだま_prevent_item_change` 等と同じ設計（`ON_CHECK_ITEM_CHANGE` で
種族一致時のみ `HandlerReturn(value=False, stop_event=True)`）に倣っている。
本プロジェクトの対象範囲（ポケモンチャンピオンズ）では種族と持ち物の組み合わせが一致するかのみを
判定すればよく、使用者側の種族判定（`_dedicated_item_prevent_transfer_to_base_form` 相当）は不要
（メガストーン・オーガポンのめんはアイテム保持による動的フォルムチェンジを行わないため対象外）。

## Priority 根拠

* `ON_CALC_POWER_MODIFIER`: ダメージ計算内部の専用イベントで `docs/spec/turn.md` に掲載がないため、
  同種の威力補正ハンドラ（`かたいいし_modify_power_by_type` 等）と同じデフォルト優先度を採用する。
* `ON_DAMAGE_HIT`: `docs/spec/turn.md` の `Event.ON_DAMAGE` 節「100 はたきおとす等のアイテム効果」に
  明記されている。同じ100番台にある「30 くっつきバリが攻撃側に渡る」より後に発動する必要があるため、
  `ON_DAMAGE` 相当の `ON_DAMAGE_HIT`（`actual_damage<=0` で発火しない点を除き同等）を使用する。

## 注意点・エッジケース（review-test 向け）

1. **ねんちゃく**: 威力補正はかかる（`power_modifier == 6144`）が、道具は失われない。
   特性発動ログは1回のみ記録される。
2. **メガストーン・オーガポンのめん・専用道具**: 威力補正はかからない（`power_modifier == 4096`）。
   道具も失われない。
3. **みがわり**: `actual_damage<=0` となるため `ON_DAMAGE_HIT` は発火せず道具は失われないが、
   威力補正はダメージ計算前に確定しているため影響を受けない（追加実装不要）。
4. **道具なしの相手**: 威力補正なし（`power_modifier == 4096`）。
