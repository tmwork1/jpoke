# もろはのずつき 実装計画書

参照: docs/spec/moves/もろはのずつき.md

## 概要

物理・いわタイプの反動技。相手に与えたダメージの1/2を自分も受ける。
共通ヘルパー `_recoil`（handlers/move_attack.py）を再利用して実装する。

既存のグループ計画書 `docs/plan/moves/反動技_05.md`（反動量1/2の技をまとめた計画書。
はめつのひかりと共通）で既にもろはのずつきの登録内容が計画・実装済みとなっている。
本ファイルは一次情報（Wiki）・仕様書との照合結果として、既存実装に誤り・欠落が
ないことを個別に確認した記録を兼ねる。

## MoveData（data/moves/move_ma.py）

既存定義（変更なし、確認済み）:

```python
"もろはのずつき": MoveData(
    type="いわ",
    category="physical",
    pp=5,
    power=150,
    accuracy=80,
    flags={"contact", "recoil"},
    handlers={
        Event.ON_HIT: h.MoveHandler(
            ha.もろはのずつき_recoil,
        )
    }
),
```

* `flags` に `"contact"`（直接攻撃）・`"recoil"`（反動技。すてみ特性の1.2倍威力補正、
  いしあたま特性の反動無効化判定に使用）を含める。
* `subject_spec` は `MoveHandler` のデフォルト `"attacker:self"` を使用。
* `priority` は `MoveHandler` のデフォルト値100を使用する。

## ハンドラ（handlers/move_attack.py）

既存実装（変更なし、確認済み）:

```python
def もろはのずつき_recoil(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    return _recoil(battle, ctx, value, 1/2)
```

* 共通ヘルパー `_recoil(battle, ctx, value, ratio)` を再利用する。
  `recoil = max(1, round_half_down(damage * ratio))` で反動量を計算し、
  `battle.modify_hp(ctx.attacker, v=-recoil, reason="recoil", source=ctx.attacker)`
  で自分のHPを減らす。
* `damage = value or ctx.substitute_damage` により、みがわりに命中した場合は
  みがわりへの与ダメージを基準に反動量を算出する。
* `reason="recoil"` を指定することで、いしあたま特性
  （`ability.py` の `いしあたま_ignore_recoil`、`ctx.hp_change_reason == "recoil"`
  を検査して無効化）・マジックガード特性
  （`マジックガード_ignore_damage`、`hp_change_reason` が除外リストに含まれないため無効化）
  が自動的に反動を無効化する。
* すてみ特性（`すてみ_boost_power`、`ctx.move.has_flag("recoil")` を検査）により、
  `flags={"recoil"}` を持つ本技は自動的に威力1.2倍の恩恵を受ける。

## イベント・priority

* `Event.ON_HIT`（ダメージ発生後の処理）: priority=100（`MoveHandler` のデフォルト）
  * `docs/spec/turn.md` の `Event.ON_HIT` 表「100 | 反動技による反動ダメージ」に対応。
  * 既存の同系統反動技（はめつのひかり・アフロブレイク・ウェーブタックル・
    ウッドハンマー・じごくぐるま・すてみタックル・とっしん）と同じ priority=100 で
    登録されており、処理順の一貫性が保たれている。

## 実装状況の確認結果

一次情報（`docs/wiki/moves/もろはのずつき.html`）・仕様書
（`docs/spec/moves/もろはのずつき.md`）と照合した結果、以下の点で既存実装は
一致しており、修正不要と判断した。

* タイプ（いわ）・分類（物理）・威力（150）・命中率（80）・PP（5）が一致。
* 反動比率1/2、共通ヘルパー `_recoil` を使用した実装が仕様と一致。
* `flags={"contact", "recoil"}` が接触技・反動技の両方の性質を正しく表現している。
* がんじょう特性は `ON_BEFORE_DAMAGE_APPLY`（メイン攻撃ダメージの適用時）のみに
  作用し、反動ダメージの `modify_hp` 呼び出しには影響しないため、本技の反動による
  瀕死をがんじょうが防がない仕様と実装が一致している。
