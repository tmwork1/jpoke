# 調査: ps-champ-ja に欠落しているフォルムの洗い出し

更新日: 2026-07-15（4回目の再確認）

[data_source.md](data_source.md) の再検討で見つかった、jpoke 1286件中21件の名前不一致
（初回56件→31件→26件→23件→本更新で21件）について、`num`（図鑑番号）・`baseStats`
（H/A/B/C/D/S）・`types` で突合し、真にps-champ-jaへ存在しないフォルムか、単なる命名・表記の
ゆれかを分類した。

## 経緯: ps-champ-ja側の対応

- 1回目の再検討まで: オドリドリ・ヨワシ・イキリンコの3種について「基本フォルムにも
  `FORM_RULES` による和名サフィックスを合成する」修正が取り込まれ、`pokedex.json` の全エントリに
  `showdown_id`/`num`/`forme`/`baseForme` が付与された。これにより文字列一致に頼らず、構造キー
  （`num`+`baseStats`+`types`）で突合できるようになった（詳細は [data_source.md](data_source.md)
  の「3. 重要な方針転換」）
- 2回目の再確認: `5e0d3db2`「基本フォルムのforme(form)表記漏れを修正し、data_jpを再生成」により、
  イルカマン(ナイーブ)・ギラティナ(アナザー)・シェイミ(ランド)・メロエッタ(ボイス)・
  ルガルガン(まひる) の5種が追加で `FORM_RULES` 対応され、jpokeの表記と文字列完全一致するように
  なった
- 3回目の再確認: `5d4529e3`「バケッチャ・パンプジン・フーパの基本フォルムのforme表記漏れを
  修正し、data_jpを再生成」により、バケッチャ(ふつう)・パンプジン(ふつう)・
  フーパ(いましめられし) の3種が追加で文字列完全一致するようになった
- 4回目の再確認: `b3f45df5`「バスラオ基本フォルムとオーガポン(テラスタル)のforme表記漏れを
  修正し、data_jpを再生成」により、バスラオ(あか)・オーガポン(みどり) の2種が追加で
  文字列完全一致するようになった（いずれも前回調査時点では「C. 構造突合だけでは一意に決まらない」
  に分類されていたもの。ps-champ-ja側で `abilities`/`forme` を踏まえた個別対応が入ったと見られる）

## 結論

jpoke 1286件中、直接の文字列一致で不一致になるのは **21件**（初回56件→31件→26件→23件→21件）。
内訳:

| 分類 | 件数 |
|---|---|
| D. 真の欠落（既知のカスタムメガ9体、対応候補なし） | 9 |
| A. 構造突合（num+baseStats+types）で一意に解決 | 6 |
| C. 構造突合だけでは一意に決まらない（abilities/forme併用 or 本質的に1対多） | 6 |
| 合計 | 21 |

**真に欠落しているのは既知のカスタムメガ9体のみ**（変更なし）。残り12件はすべて
ps-champ-ja側に対応データが存在し、うち6件は構造キーだけで自動解決できる。

## D. 真の欠落（対応候補なし） — 全9件

jpoke 独自実装のカスタムメガシンカ。PS champions mod に存在しないため ps-champ-ja にも無い。
取り込み時は追加専用マージでこれらを保持する必要がある（[data_source.md](data_source.md) 参照）。

- メガグソクムシャ
- メガシャリタツ
- メガジガルデ
- メガセグレイブ
- メガゼラオラ
- メガダークライ
- メガニャオニクス
- メガヒードラン
- メガマギアナ

## A. 構造突合で一意に解決 — 全6件

`num` + `baseStats` + `types` の突合で ps-champ-ja 側の対応先が一意に確定した。
`pokedex.json` 本体だけでなく `pokedex_excluded.json`（和名重複等でps-champ-ja自身が除外した
エントリ）も候補プールに含めている点に注意（コレクレー(とほ) が該当）。

| jpoke名 | ps-champ-ja側の対応先 | 備考 |
|---|---|---|
| コレクレー(はこ) | コレクレー | |
| コレクレー(とほ) | `gimmighoulroaming`（showdown_id） | `pokedex_excluded.json` 側にのみ存在 |
| タイプ：ヌル | タイプ:ヌル | 全角/半角表記ゆれ |
| テラパゴス(ノーマル) | テラパゴス | |
| ヒヒダルマ(ノーマル) | ヒヒダルマ | |
| ポリゴン２ | ポリゴン2 | 全角/半角表記ゆれ |

## C. 構造突合だけでは一意に決まらない — 全6件

種族値・タイプが同一の候補が複数存在し、`abilities` 配列や `forme` 文字列（`Gmax` 等）まで
見ないと一意に決まらない。ウッウ・モルペコのように、jpoke側では区別しない見た目/状態違いを
ps-champ-ja側が複数の showdown forme として持つため、本質的に1対多になるものも含む
（マージ上は許容できる。取り込み時は先頭候補を割り当てれば足りる）。

| jpoke名 | 候補（num+baseStats+types一致） | 備考 |
|---|---|---|
| ウッウ(うのみ) | ウッウ、`cramorantgulping`、`cramorantgorging` | Gulping/Gorgingはabilityでのみ区別。本質的に1対多 |
| ウッウ(まるのみ) | 同上 | 同上 |
| ウーラオス(キョダイ) | ウーラオス(いちげき,キョダイ)、ウーラオス(れんげき,キョダイ) | いちげき/れんげきはtypeで区別可（かくとう+あく vs みず+あく）。`abilities`（ふかしのこぶし/すいほう）併用で解決可 |
| ストリンダー(キョダイ) | ストリンダー(ハイ,キョダイ)、ストリンダー(ロー,キョダイ) | `abilities`（パンクロック/せいでんき等）併用で解決可 |
| ケルディオ(いつも) | ケルディオ、ケルディオ(かくご) | `abilities`（せいぎのこころ/ふくつのこころ）併用で解決可 |
| モルペコ | モルペコ(まんぷく)、モルペコ(はらぺこ) | 満腹度に応じた状態違いのみ。本質的に1対多 |
