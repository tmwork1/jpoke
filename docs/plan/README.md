### **Level 2: 詳細分析**

#### [01_未実装バトルロジック分析.md](01_未実装バトルロジック分析.md)
**最重要ドキュメント**

技・特性・アイテム実装に必要な欠けているバトルロジックを網羅的に分析。

**主要セクション**:
- エグゼクティブサマリー（実装状況統計）
- 【最優先】核心バトルロジック（6項目）
  - 急所判定、命中判定、優先度、ひるみ等
- 【高優先】技の追加効果実装に必要なロジック
- 【中優先】特性実装に必要なロジック
- イベント実装優先度マトリクス
- 技術的課題と対策
- ロードマップ（5フェーズ、14-19週間）

**用途**: 新機能実装前の依存関係チェック、核心ロジック確認

---

#### [02_技の実装状況分析.md](02_技の実装状況分析.md)
**技特化ドキュメント**

「01_未実装バトルロジック分析.md」の調査結果に基づいた技の分類分析。

**主要セクション**:
- 実装済み効果処理（フラグ別分類）
- 未実装効果処理（23カテゴリ）
  - 各カテゴリの仕様・実装要件・優先度
  - 対象技のリスト
  - 実装関連箇所

**用途**: 技の追加効果実装時の詳細参照、優先度確認

### **Level 3: 実装計画**

#### [03_実装計画書_フェーズ1-2.md](03_実装計画書_フェーズ1-2.md)
**フェーズ別実装計画**

フェーズ1-2（核心システム + 技追加効果拡充）の具体的な実装計画。

**主要セクション**:
- 現状分析（実装率、各フェーズの目標）
- 実装の核心（12タスク）
  - 6つの核心ロジック + 4つの技追加効果 + 2つのアイテム・特性
  - 各タスク: 工数・必要性・実装難易度・テスト対象・実装ファイル
- 4週間の詳細スケジュール
- テスト戦略・品質基準
- サクセスメトリクス

**用途**: 実装チーム用の詳細作業計画書、進捗管理

## 🎯 使用ガイダンス

### シナリオ1: 「新しい技を実装したい」
1. **[01_未実装バトルロジック分析.md](01_未実装バトルロジック分析.md)** で依存する核心ロジックを確認
2. 必要なイベントが実装済みか確認
3. **[02_技の実装状況分析.md](02_技の実装状況分析.md)** で該当カテゴリの実装仕様を確認
4. 実装開始

### シナリオ2: 「今週の実装タスクを確認したい」
1. **[03_実装計画書_フェーズ1-2.md](03_実装計画書_フェーズ1-2.md)** の「実装スケジュール」セクションを確認
2. 該当週のタスク詳細を読む
3. テスト対象と品質基準を確認

### シナリオ3: 「優先度を確認したい」
1. **[01_未実装バトルロジック分析.md](01_未実装バトルロジック分析.md)** の「エグゼクティブサマリー」セクション確認
2. 「【最優先】未実装の核心バトルロジック」で Top 6 確認
3. イベント実装優先度マトリクスで詳細確認

### シナリオ4: 「技別の実装詳細を知りたい」
1. **[02_技の実装状況分析.md](02_技の実装状況分析.md)** から該当カテゴリを検索
2. 仕様・実装要件・対象技リストを確認
