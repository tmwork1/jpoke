# 実装計画

最終更新日: 2026年3月21日

このディレクトリは、現在の実装優先度と着手順を管理するための場所です。過去フェーズの細かい予定表を積み上げるのではなく、今のコードベースに対して有効な計画だけを残します。

## 収録ファイル

- [00_実装計画_統合版.md](00_実装計画_統合版.md): 現在の実装状況、残課題、次に進める順序。

## 運用方針

- 進捗率は [dashboard.json](../../dashboard.json) を正とする。
- 仕様調査は `docs/spec/` に残し、このディレクトリには実行計画だけを書く。
- 特性、アイテム、技の実装方針が変わったときは、まず統合版を更新する。
- 完了した短期タスクは積み上げずに削るか書き換える。

## 今の見方

- バトル基盤の完成度確認: 統合版の「現状認識」
- 次の着手順確認: 統合版の「優先順位」
- 実装時の更新対象確認: 統合版の「実装時の更新ルール」
