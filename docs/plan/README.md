# 実装計画ドキュメント

このディレクトリには、jpokeプロジェクトの未実装要素と実装計画に関するドキュメントが含まれています。

## ドキュメント一覧

### [move_implementation_status.md](move_implementation_status.md)
**技の実装計画**  
`docs/research/move_gen9.md`の調査結果を実装段階に変換したドキュメント。

**内容**:
- 実装済み効果処理（11カテゴリ）
- 未実装効果処理の詳細仕様（23カテゴリ）
- 優先度評価マトリクス
- 実装フェーズの推奨順序
- 依存関係分析

### [未実装バトルロジック分析.md](未実装バトルロジック分析.md)
**最重要ドキュメント**  
現在のコードと調査書を詳細に分析し、技・特性・持ち物の実装に必要な欠けているバトルロジックとイベントを網羅的に抽出。

**内容**:
- 実装状況の統計サマリー
- 核心バトルロジック（急所、命中判定、行動順序等）
- 技の追加効果実装に必要なロジック
- 特性・持ち物実装に必要なロジック
- イベント実装優先度マトリクス
- 実装ロードマップ（5フェーズ、14-19週間）
- 技術的課題と解決策

### [未実装要素サマリー.md](未実装要素サマリー.md)
上記分析の要約版。クイックリファレンス用。

**内容**:

- 最優先実装項目Top 10

- 完全実装への道のり

## 使い方

### 新機能を実装する前に
1. [未実装バトルロジック分析.md](未実装バトルロジック分析.md) で該当項目を確認
2. 依存する核心システムが実装済みかチェック
3. 必要なイベントが定義されているか確認

### 実装計画を立てる時
1. [未実装要素サマリー.md](未実装要素サマリー.md) で優先度を確認
2. ロードマップに沿ってフェーズ単位で計画
3. イベント実装優先度マトリクスを参考に依存関係を整理

### 進捗を確認する時
1. 実装状況概要表を定期的に更新
2. チェックリスト（`docs/checklist/`）と照合
3. テストカバレッジを確認
