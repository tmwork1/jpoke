# 特性実装計画: かんろなミツ

更新日: 2026-06-05

## 仕様参照

- `docs/spec/abilities/かんろなミツ.md`

## 実装ゴール

- バトル中一度だけ、場に出たとき相手の回避率を1段階下げる。
- 2回目以降の入場では発動しない。
- クリアボディ等で EVA↓ 無効の相手には発動しない。

## 実装方針

- `Event.ON_SWITCH_IN`、`subject_spec="source:self"` で実装。
- 対象はシングルバトルのみのため相手は `battle.foe(mon)` の1体で足りる。
- **いかく** と同一パターン（下げる能力が atk→evasion に変わるだけ）。発動時は必ず
  `_announce_ability_triggered` でメッセージを出し、相手が「みがわり」状態なら
  `modify_stats` を呼ばずに終了する（効果なし）。
- `per_battle_once` フラグは data/ability.py に既定済み。`add_ability_disabled_reason(mon, "consumed")`
  は相手が場にいることを確認した後（＝発動が成立した後）に呼び、みがわり・クリアボディ等で
  効果が無かった場合でも次の再登場では発動しないようにする。

## 変更候補

- `src/jpoke/data/ability.py` — `かんろなミツ` に ON_SWITCH_IN handler を登録する
- `src/jpoke/handlers/ability.py` — `かんろなミツ_on_switch_in` を実装する
- `tests/test_ability.py` — テストを追加する

## テスト観点

1. 初回入場で相手の EVA が 1 段階下がる。
2. 2回目以降の入場では発動しない。
3. クリアボディ等の相手では EVA↓ が失敗する。
4. みがわり状態の相手には効果がない。

## 疑似コード

```python
def かんろなミツ_lower_foe_evasion(battle, ctx, value):
    mon = ctx.source
    if mon is None:
        return HandlerReturn(value=value)
    foe = battle.foe(mon)
    if foe is None:
        return HandlerReturn(value=value)
    battle.add_ability_disabled_reason(mon, "consumed")
    _announce_ability_triggered(battle, mon)
    if foe.has_volatile("みがわり"):
        return HandlerReturn(value=value)
    battle.modify_stats(foe, {"evasion": -1}, source=mon)
    return HandlerReturn(value=value)
```
