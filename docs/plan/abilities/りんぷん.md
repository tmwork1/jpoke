# 特性実装計画: りんぷん

更新日: 2026-05-03

## 仕様参照

- `spec/ability/りんぷん.md`

## 実装ゴール

- 相手の攻撃技による追加効果を受けない。
- なげつけるの追加効果も防ぐ。
- オボンのみ等の防御側メリット効果も防ぐ。
- かたやぶりの技による追加効果は防げない。
- ちからずくとの競合時はちからずくが優先されて追加効果消失（りんぷんも発動するが追加効果自体がない状態）。
- コメットパンチのように使用者自身の能力が変化する追加効果は、りんぷん所持者から
  受けたときも防げない（自分自身に対する効果であり、りんぷん所持者への効果ではないため）。

## 実装方針

- 追加効果判定フック `Event.ON_MODIFY_SECONDARY_CHANCE`（`battle.resolve_secondary_chance()`
  経由で発火）で追加効果適用を無効化する。
- 追加効果の無効化はおんみつマント（アイテム）と同様のパターンで処理する。
- かたやぶり判定は既存の共通機構（`かたやぶり_disable_foe_ability`/`かたやぶり_restore_foe_ability` が
  `Event.ON_BEGIN_MOVE`/`Event.ON_END_MOVE` で `battle.add_ability_disabled_reason` を呼び出す仕組み）で
  制御済みのため、りんぷん側で個別対応は不要。
- `AttackContext.secondary_effect_target`（既定 `"defender"`）で追加効果の対象ロールを表す。
  `battle.resolve_secondary_chance(ctx, chance, target="attacker")` で呼び出された場合
  （コメットパンチ等、使用者自身への効果。`handlers/move.py` の `modify_attacker_stats`/
  `apply_volatile_to_attacker` がこれに該当）は `ctx.secondary_effect_target == "attacker"`
  になり、りんぷん（`defender:self`）は反応しない。ちからずく・てんのめぐみ
  （`attacker:self`）はこの値に関わらず常に反応する。

  `target="attacker"` を指定）

## テスト観点

1. 追加効果付き技を受けても追加効果が発生しない。
2. なげつけるの追加効果も防ぐ。
3. オボンのみ等のメリット効果も防ぐ。
4. かたやぶりの技では追加効果を防げない。
5. ちからずくとの競合時: ちからずくポケモンの技威力は1.3倍になるが追加効果はない（りんぷんの発動とは無関係に追加効果消失）。
6. コメットパンチ・きょけんとつげきのように使用者自身への追加効果は、相手のりんぷんでも防げない。
