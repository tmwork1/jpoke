# 特性実装計画: シェルアーマー

更新日: 2026-07-11

## スコープ

- 仕様書: `docs/spec/abilities/シェルアーマー.md`
- 実装状態: 実装済み
- 実装方針: カブトアーマーと完全に同一の効果を持つため、`docs/plan/abilities/カブトアーマー.md` で定義した
  `カブトアーマー_block_crit` ハンドラをそのまま共用登録する。新規ハンドラは実装しない。

## 実装ゴール（完了定義）

- 他のポケモンから受ける攻撃技が急所に当たらなくなる。
- こおりのいぶき/やまあらし/あんこくきょうだ/すいりゅうれんだ/トリックフラワーなど、急所ランクが極めて高い（確定急所扱いの）技を受けても急所に当たらない。
- かたやぶり等かたやぶり系特性の効果が発動した技に対してはこの特性は無視され、急所に当たることがある。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_MODIFY_CRITICAL_RATE` — `move_executor.py` の `_check_critical` で急所ランクから算出した急所発生率（`CRIT_RATES[critical_rank]`）を補正するタイミング。ここで発生率を0に上書きすることで、急所ランクの大小によらず急所を無効化できる（カブトアーマーと同一のタイミング根拠）。

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `シェルアーマー` に `ON_MODIFY_CRITICAL_RATE` の handler を登録する（`subject_spec="defender:self"`）。関数はカブトアーマー用の `h.カブトアーマー_block_crit` を共用する（新規関数は作らない）。
- `src/jpoke/handlers/ability.py` — 変更なし（`カブトアーマー_block_crit` を流用）。
- `tests/abilities/test_ability_sa.py` — シェルアーマー用テストを追加する。

## 実装ステップ

1. 仕様書 `docs/spec/abilities/シェルアーマー.md`（および参照元の `docs/spec/abilities/カブトアーマー.md`）で発動条件・例外条件を確認する。
2. `src/jpoke/data/ability.py` の `シェルアーマー` に `ON_MODIFY_CRITICAL_RATE` の handler（`h.カブトアーマー_block_crit`）を登録する。
3. `tests/abilities/test_ability_sa.py` にテストを追加し、全テストが通ることを確認する。

## テスト観点

1. 通常の攻撃技を受けても急所に当たらない。
2. 確定急所級の技（トリックフラワー等）を受けても急所に当たらない。
3. かたやぶり所持ポケモンからの攻撃を受けると急所に当たり得る（特性が無視される）。
