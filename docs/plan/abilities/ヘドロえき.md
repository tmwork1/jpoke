# 特性実装計画: ヘドロえき

更新日: 2026-07-12

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/ヘドロえき.md`
- 実装状態: 実装済み（`handlers/ability.py` の `ヘドロえき_reverse_drain`、`data/ability.py` に登録済み）
- 実装方針: `Event.ON_MODIFY_HEAL` に subject_spec="target:foe" でフックし、
  `ctx.hp_change_reason == "drain"` のとき回復量を反転（ダメージ化）して返す。
  ctx.target は「回復しようとしているポケモン（吸収技の使用者）」であり、
  ヘドロえき所持者はその foe（`battle.foe(ctx.target)`）として特定する。
  吸収技・やどりぎのタネの回復処理側で `reason="drain"` を設定済みであることが前提。

## 実装ゴール（完了定義）

- 吸収技（ギガドレイン・ドレインパンチ等）を受けたとき、攻撃者は回復せずにダメージを受ける。
- やどりぎのタネを植えられている場合も毎ターンダメージに変換する。
- マジックガード持ちの攻撃者にはダメージを与えられない（回復効果も消える）。
- みがわりを貫通してダメージを与える。
- おおきなねっこ所持の攻撃者に対してはダメージが1.3倍になる。
- 相手がかいふくふうじ状態であってもダメージ効果は発動する（通常の回復のみブロックされる）。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_MODIFY_HEAL` — subject_spec="target:foe"（ヘドロえき所持者は ctx.target の foe）。
  `ctx.hp_change_reason == "drain"` でフィルタ。

`ON_MODIFY_HEAL` は `core/status_manager.py` の `modify_hp`（v > 0 の場合）で発火済み。
ctx.target=吸収技使用者（回復予定のポケモン）。

## Priority

| イベント | Priority | 根拠 |
|---|---|---|
| ON_HIT | 20 | docs/spec/turn.md「ヘドロえきによるダメージ」（参考。実装は ON_MODIFY_HEAL） |
| ON_MODIFY_HEAL | 90 | ON_MODIFY_HEAL は docs/spec/turn.md に未掲載。同イベントには「かいふくふうじ」の回復ブロック（`かいふくふうじ_block_heal`, priority=100（デフォルト））が登録されており、両者が同一優先度だと素早さ順で処理順が入れ替わってしまう。仕様書の「相手がかいふくふうじ状態であってもダメージ効果は発動する」を満たすため、ヘドロえきの回復→ダメージ変換を常にかいふくふうじの回復ブロックより先に処理させる必要があり、priority=90（かいふくふうじより小さい値）を明示する。あわせて `かいふくふうじ_block_heal` 側にも「変換済み（0以下）の値は再ブロックしない」ガードを追加している。 |

## 変更対象（ファイル/関数単位）

- `src/jpoke/handlers/volatile.py` — `かいふくふうじ_block_heal`（value<=0 のとき再ブロックしないガードを追加）
- `tests/abilities/test_ability_ha.py` — ヘドロえき単体のテスト
- `tests/volatiles/test_volatile_hamayarawa.py` — やどりぎのタネ経由のテスト（かいふくふうじとの相互作用を含む）

## 実装ステップ

1. 吸収技・やどりぎのタネの回復呼び出しで `reason="drain"` が設定されていることを確認する
   （`handlers/move_attack.py`, `handlers/move_status.py`, `handlers/volatile.py` の該当箇所）。
2. `src/jpoke/data/ability.py` の `ヘドロえき` に handler を登録する。
   - `Event.ON_MODIFY_HEAL`: `ヘドロえき_reverse_drain`, subject_spec="target:foe", priority=90
3. `かいふくふうじ_block_heal` に「変換済み（value<=0）の場合は再ブロックしない」ガードを追加し、
   かいふくふうじ状態の相手に対してもヘドロえきのダメージ効果が阻害されないようにする。
4. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. ギガドレイン等の吸収技を受けると攻撃者が回復せずにダメージを受ける。
2. やどりぎのタネを植えられたとき、毎ターン攻撃者（回復予定側）がダメージを受ける。
3. マジックガード持ちの攻撃者には吸収ダメージを与えられない（回復も発生しない）。
4. みがわり状態の相手に対してもみがわりを無視して攻撃者にダメージを与える。
5. おおきなねっこ持ちの攻撃者に対してはダメージが1.3倍になる。
6. 回復予定側（攻撃者）がかいふくふうじ状態でも、通常の回復はブロックされつつヘドロえきのダメージ効果は発動する。
7. drain以外の理由（じこさいせい等）の回復には影響しない。
