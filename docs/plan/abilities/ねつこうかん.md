# 特性実装計画: ねつこうかん

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/ねつこうかん.md`

## 実装ゴール

- ほのおタイプの攻撃技でダメージを受けたとき、こうげきが1段階上がる。
- こうげきがすでに最大まで上がっているときは、ほのおタイプの攻撃技を受けても発動しない
  （`battle.modify_stats` の戻り値が変化なしのときは特性発動アナウンスも出さない）。
- 変化技（おにび等）や、まもる・みがわりで防がれたときは発動しない（`ON_DAMAGE_HIT` は
  実HPダメージが発生したときのみ発火するため、これらのケースは自然に除外される。
  じょうききかんと同じ扱い）。
- 攻撃技でひんしになったときは発動しない。`ON_DAMAGE_HIT` はKO後にも発火するため、
  `ctx.defender.fainted` を明示的にチェックして除外する（へんしょく・いかりのこうら等と
  同じ idiom。`battle.modify_stats` 自体には alive/fainted チェックが無いため、ハンドラ側で
  ガードしないとひんし後もランクが変化してしまう）。
- やけど状態にならない（付与をブロックする）。おにびを受けても無効化するが、その際攻撃は
  上昇しない（`ON_BEFORE_APPLY_AILMENT` と `ON_DAMAGE_HIT` は独立したイベントであり、
  おにびは変化技のため `ON_DAMAGE_HIT` 自体が発火しない）。
- 特性が有効化された時点ですでにやけど状態なら即座に回復する
  （かがくへんかガス・かたやぶりの効果が終わって特性が再び有効になった場合や、
  なかまづくり等で特性が書き換わった場合が対象）。
- かたやぶりの効果がある技を受けたときは、特性の効果は2つとも発動しない。
  - かたやぶりの効果があるほのおタイプの攻撃技を受けても攻撃は上がらない。
  - かたやぶりの効果がある技によりやけど状態にされることはあるが、技終了直後
    （特性が再び有効になったタイミング）で回復する。
- 技や特性の効果でタイプが変わった攻撃技を受けた場合、変更後のタイプがほのおタイプであれば
  発動する（`ctx.move.type` は技実行中に変化しうる有効タイプを参照するため、自然に満たされる）。

## 実装方針

- こうげき+1: `Event.ON_DAMAGE_HIT`、`subject_spec="defender:self"` で実装する
  （`ねつこうかん_boost_atk_on_fire`）。
  - `ctx.move.type != "ほのお"` なら何もしない。
  - `ctx.defender.fainted` なら何もしない（ひんし後もガードなしでは `modify_stats` が
    通ってしまうため必須）。
  - `battle.modify_stats(ctx.defender, {"atk": +1}, source=ctx.attacker)` を呼び、戻り値が
    変化なし（すでに最大）のときは特性発動アナウンスを出さない。
  - `ON_DAMAGE_HIT` は実HPダメージが発生したときのみ発火するイベントのため、まもる・みがわり・
    ダメージ0のケースは自然に除外される（じょうききかんと同じ実装パターン）。
  - priority は既定値（100）を使う。技側の追加効果とのランク変化順序が問題になる
    じょうききかん（S+6）と異なり、ねつこうかんのこうげき+1に絡む技側の追加効果は
    一次情報に記載が無いため、既存の同種ハンドラに倣い priority 指定は不要と判断する。
- やけど付与ブロック: `Event.ON_BEFORE_APPLY_AILMENT`（`target:self`）で、既存の状態異常
  無効系特性と同じ共通ヘルパー `prevent_burn_ailment`（`blocked_ailments=["やけど"]` で
  `_prevent_ailment` を呼ぶ）をそのまま再利用する（すいほう・みずのベールと同じ）。
- 特性有効化時点で既にやけど状態なら回復する処理: `Event.ON_ABILITY_ENABLED`
  （`source:self`）で共通ヘルパー `_cure_ailment_on_enable` を `blocked_ailments=["やけど"]`
  で呼び出す `ねつこうかん_cure_burn_on_enable` を実装する（すいほうと同じパターン）。
- `flags={"mold_breaker_ignorable"}` をアビリティ全体に付与する。かたやぶりは
  `ctx.defender`（攻撃を受ける側）の特性を攻撃中のみ一時的に無効化する仕組みのため、この
  付与によりこうげき+1・やけど付与ブロックの両方が攻撃中は無効化され、攻撃終了後に特性が
  再有効化されたタイミングで `ON_ABILITY_ENABLED` が発火し、やけど状態のみ回復する
  （＝「効果は2つとも発動しないが、やけど状態は技終了直後に回復する」という仕様を自然に
  満たす）。
- `ON_DAMAGE_HIT` / `ON_BEFORE_APPLY_AILMENT` / `ON_ABILITY_ENABLED` はいずれも
  `docs/spec/turn.md` に priority が明記されていない内部イベントのため、既存の同種ハンドラ
  （じょうききかん・すいほう・めんえき等）に倣い priority は指定せず既定値（100）を使う。

## テスト観点

1. ほのお技を受けてダメージ後にこうげきが1段階上がる（実装済み）。
2. ノーマル等他タイプでは発動しない。
3. こうげきがすでに最大まで上がっているときは発動しない。
4. まもるで無効化されたときは発動しない。
5. みがわり状態で防がれたときは発動しない。
6. おにびを受けてもやけど状態にならず、攻撃は上昇しない（実装済み: `test_状態異常無効`
   パラメタライズケースに追加）。
7. かたやぶりの技（おにび）を受けたときやけど付与が発生しうるが技終了直後に回復する
   （`test_状態異常無効_かたやぶりで無効` パラメタライズケースに追加）。
8. タイプが変わる攻撃技（実際に出されたタイプがほのおタイプ）を受けた場合に発動する。
