# 特性実装計画: はやてのつばさ

更新日: 2026-06-03

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/はやてのつばさ.md`
- 実装状態: 未実装
- 実装方針: `DomainEvent.ON_MODIFY_MOVE_PRIORITY` で HP が満タン かつ 使用技がひこうタイプのとき優先度を +1 する。

## 実装ゴール（完了定義）

- HP が満タン（最大HP）のときにひこうタイプの技を使うと、優先度が +1 になる。
- HP が満タン以外のときは発動しない。
- めざめるパワー等の「タイプが変化する技」でひこうになっても発動しない。
- 既存テストを壊さない。

## 実装対象イベント

- `DomainEvent.ON_MODIFY_MOVE_PRIORITY` — 優先度計算時に HP 満タン かつ ひこう技なら +1 する

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `はやてのつばさ` に handler を登録する
- `src/jpoke/handlers/ability.py` — `はやてのつばさ_modify_priority` を実装する
- `tests/test_ability.py` — テストを追加する

## 実装ステップ

1. 仕様書 `docs/spec/abilities/はやてのつばさ.md` でひこう技の判定ルール（めざめるパワー等の除外）を確認する。
2. `src/jpoke/handlers/ability.py` に `はやてのつばさ_modify_priority` を実装する（`ctx.attacker.hp == ctx.attacker.max_hp` かつ `ctx.move.type == "ひこう"` のとき value + 1 を返す）。
3. `src/jpoke/data/ability.py` の `はやてのつばさ` に handler を登録する。
4. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. HP 満タン時にひこう技を使うと先制（優先度 +1）になる。
2. HP が 1 でも減っていると優先度は上がらない。
3. HP 満タン時でもひこう以外の技では優先度は上がらない。
