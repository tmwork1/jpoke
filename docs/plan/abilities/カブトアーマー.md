# 特性実装計画: カブトアーマー

更新日: 2026-07-11

## スコープ

- 仕様書: `docs/spec/abilities/カブトアーマー.md`
- 実装状態: 実装済み
- 実装方針: `ON_MODIFY_CRITICAL_RATE` で急所発生率を強制的に0にし、以降のハンドラを止める（`stop_event=True`）。

## 実装ゴール（完了定義）

- 他のポケモンから受ける攻撃技が急所に当たらなくなる。
- こおりのいぶき/やまあらし/あんこくきょうだ/すいりゅうれんだ/トリックフラワーなど、急所ランクが極めて高い（確定急所扱いの）技を受けても急所に当たらない。
- かたやぶり等かたやぶり系特性の効果が発動した技に対してはこの特性は無視され、急所に当たることがある。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_MODIFY_CRITICAL_RATE` — `move_executor.py` の `_check_critical` で急所ランクから算出した急所発生率（`CRIT_RATES[critical_rank]`）を補正するタイミング。ここで発生率を0に上書きすることで、急所ランクの大小によらず急所を無効化できる。

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `カブトアーマー` に `ON_MODIFY_CRITICAL_RATE` の handler を登録する（`subject_spec="defender:self"`）。同じ関数は `シェルアーマー` からも共用登録する。
- `src/jpoke/handlers/ability.py` — `カブトアーマー_block_crit(battle, ctx, value)` を実装する。発生率を0にして `stop_event=True` を返す。
- `tests/abilities/test_ability_ka.py` — テストを追加する。

## 実装ステップ

1. 仕様書 `docs/spec/abilities/カブトアーマー.md` で発動条件・例外条件を確認する。
2. `src/jpoke/handlers/ability.py` に `カブトアーマー_block_crit` を実装する。
3. `src/jpoke/data/ability.py` の `カブトアーマー`（および `シェルアーマー`）に `ON_MODIFY_CRITICAL_RATE` の handler を登録する。
4. `tests/abilities/test_ability_ka.py` にテストを追加し、全テストが通ることを確認する。

## テスト観点

1. 通常の攻撃技を受けても急所に当たらない。
2. 確定急所級の技（トリックフラワー等）を受けても急所に当たらない。
3. かたやぶり所持ポケモンからの攻撃を受けると急所に当たり得る（特性が無視される）。
