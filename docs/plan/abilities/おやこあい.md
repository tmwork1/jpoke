# 実装計画: おやこあい

作成日: 2026-06-03
更新日: 2026-07-11（レビューで実装との乖離を修正）

## 効果
仕様書: `docs/spec/abilities/おやこあい.md`

使用する単発の攻撃技を2ヒット化する。1ヒット目は通常ダメージ、2ヒット目は
1ヒット目のダメージの1/4（切り捨て、最低1）になる。既存の連続攻撃技（`max_hits > 1`）や
変化技には適用しない。

## イベント／ハンドラ

| イベント | priority | subject_spec | 関数名 |
|---------|---------|-------------|--------|
| `Event.ON_MODIFY_HIT_COUNT` | 100（デフォルト。`turn.md` 未掲載イベント。ヒット数決定は命中判定前のダメージ計算前段であり、同イベントに登録される他ハンドラ（スキルリンク・いかさまダイス）と適用条件が排他のため競合しない） | `attacker:self` | `おやこあい_modify_hit_count` |
| `Event.ON_MODIFY_MOVE_DAMAGE` | 100（デフォルト。`turn.md` 未掲載イベント。ダメージロール後・HP適用前に発火するイベントで、既存の他特性（マルチスケイル等は `ON_CALC_DAMAGE_MODIFIER` 側）と重複登録がなく競合しない） | `attacker:self` | `おやこあい_reduce_second_damage` |

`Event.ON_MODIFY_HIT_COUNT` はヒット数決定を一箇所に集約するための共通イベントとして
既に導入済み（`MoveExecutor._resolve_hit_count` が発火）で、スキルリンク
（`スキルリンク_modify_hit_count`）・いかさまダイス（`いかさまダイス_modify_hit_count`）も
同イベント経由で実装されている。おやこあいの適用条件（`max_hits == 1` の単発攻撃技）と
スキルリンク／いかさまダイスの適用条件（`min_hits, max_hits == (2, 5)`）は排他のため、
同一の技で両方が同時に発火することはない。

## 実装

```python
def おやこあい_modify_hit_count(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """おやこあい特性: 単発攻撃技を2ヒット化する。

    がむしゃらはHPの変動有無に関わらずいかなる状況でも連続攻撃にならないため対象外とする。
    ころがる/アイスボールも連続攻撃にならない（数ターン継続する強制行動技のため）。
    """
    if ctx.move.name in ("がむしゃら", "ころがる", "アイスボール"):
        return HandlerReturn(value=value)
    if ctx.move.is_attack and ctx.move.max_hits == 1:
        value = 2
    return HandlerReturn(value=value)


def おやこあい_reduce_second_damage(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """おやこあい特性: 2ヒット目のダメージを1/4に減衰させる（最低1ダメージ保証）。"""
    if ctx.hit_index == 2:
        value = max(1, value // 4)
    return HandlerReturn(value=value)
```

**非対象技（`ON_MODIFY_HIT_COUNT` に到達する前に別イベントで弾かれる／`is_attack` が False）**:
- 変化技（`is_attack` が False）
- 既存連続技（`max_hits > 1`）: れんぞくパンチ、すいりゅうれんだ等
- がむしゃら／ころがる／アイスボール（数ターン継続する強制行動技・自傷判定技のため個別除外）
- みらいよち／はめつのねがい・一撃必殺技・こうげき対象2体以上になる技などは、
  `ON_MODIFY_HIT_COUNT` より前段の判定（対象数決定・技個別の実行フロー）で
  そもそも通常の連続ヒットループに乗らないため、おやこあい側で個別分岐する必要はない。

## data/ability.py 登録

```python
"おやこあい": AbilityData(
    handlers={
        Event.ON_MODIFY_HIT_COUNT: h.AbilityHandler(
            h.おやこあい_modify_hit_count,
            subject_spec="attacker:self",
        ),
        Event.ON_MODIFY_MOVE_DAMAGE: h.AbilityHandler(
            h.おやこあい_reduce_second_damage,
            subject_spec="attacker:self",
        )
    },
    lethal_handlers={
        LethalEvent.ON_BEFORE_HIT: LethalHandler(l.おやこあい_boost_damage, subject="attacker")
    }
),
```

## リーサル計算

`LethalEvent.ON_BEFORE_HIT`（subject="attacker"）で `おやこあい_boost_damage` を登録し、
単発攻撃技（`move.is_attack and move.max_hits == 1`）のダメージ分布に
2ヒット目分（1ヒット目ダメージの1/4、最低1、命中0は0のまま）を加算する。
`ctx.damage_dist` と `ctx.damage_dist_full`（満タン枝）の両方を更新する。
