# 特性実装計画: ふかしのこぶし

更新日: 2026-06-03

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/ふかしのこぶし.md`
- 実装状態: 未実装
- 実装方針: `ON_CHECK_BYPASS_PROTECT` で接触技がまもる状態を貫通するよう設定し、貫通した場合のダメージを 1/4 に減衰させる（SVシリーズ仕様）。

## 実装ゴール（完了定義）

- 接触技でまもる状態（まもる・ニードルガード等）を貫通し、1/4 のダメージを与える。
- 非接触技はまもるを貫通しない。
- パンチグローブ持ちのパンチ技は非接触になるため貫通しない。
- ぼうごパットは接触判定を変えないため、ふかしのこぶしの効果で貫通する。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_CHECK_BYPASS_PROTECT`（または同等のイベント）— 接触技に対してまもる貫通フラグを立てる
- `Event.ON_MODIFY_MOVE_DAMAGE` — まもる貫通時のダメージを 1/4 に減衰させる

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `ふかしのこぶし` に handlers を登録する
- `src/jpoke/handlers/ability.py` — `ふかしのこぶし_bypass_protect`, `ふかしのこぶし_reduce_damage` を実装する
- `tests/test_ability.py` — テストを追加する

## 実装ステップ

1. まもる貫通チェックのイベントが `src/jpoke/enums/event.py` にあるか確認する（既存の `すりぬけ_bypass_screen` を参考に）。
2. `ふかしのこぶし_bypass_protect` を実装する（技が接触判定ありのとき貫通フラグを立てる）。
3. `ふかしのこぶし_reduce_damage` を実装する（まもる貫通時のダメージを 1/4 に）。
4. `src/jpoke/data/ability.py` の `ふかしのこぶし` に handlers を登録する。
5. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. まもる状態の相手に接触技を使うと、まもるを貫通して 1/4 のダメージを与える。
2. まもる状態の相手に非接触技を使うと通常通り防がれる。

## リスクと対策

- リスク: まもる貫通と 1/4 ダメージ減衰のタイミングがシステム依存で複雑。
  - 対策: 既存の `すりぬけ` のまもる貫通実装を確認し、ダメージ減衰を追加する方針で進める。
