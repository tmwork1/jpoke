# 特性実装計画: パステルベール

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/パステルベール.md`

## 実装ゴール

- 自分（シングルバトルでは味方は対象外）がどく・もうどく状態にならない（付与をブロックする）。
- 特性が有効化された時点ですでにどく・もうどく状態なら即座に回復する
  （かがくへんかガス・かたやぶりの効果が終わって特性が再び有効になった場合や、
  なやみのタネ等で特性が書き換わった場合が対象）。
- かたやぶりの効果がある技でどく・もうどく状態にされた場合は、技終了直後（特性が再び有効に
  なったタイミング）で回復する。
- どくびし・どくどくだま等、技以外の経路によるどく/もうどく付与も
  `Event.ON_BEFORE_APPLY_AILMENT` を経由するため、追加実装なしで同様にブロックされる。

## 実装方針

- 状態異常の付与ブロックは `Event.ON_BEFORE_APPLY_AILMENT`（`target:self`）で行う。
  めんえき・じゅうなん・みずのベール・やるき等、既存の状態異常無効系特性と同じ
  `_prevent_ailment` 共通ヘルパー（`handlers/ability.py`）を `blocked_ailments=["どく", "もうどく"]` で
  呼び出す `prevent_poison_ailment` を用いる（めんえきと共有・既存関数の再利用）。
- 既にどく/もうどく状態のときに特性が有効化されるケース（かがくへんかガス/かたやぶりの終了、
  なやみのタネによる特性書き換え等）は `Event.ON_ABILITY_ENABLED`（`source:self`）で
  `_cure_ailment_on_enable` 共通ヘルパーを `blocked_ailments=["どく", "もうどく"]` で呼び出す
  `パステルベール_cure_poison_on_enable` で回復させる。
- `mold_breaker_ignorable` フラグを付与し、かたやぶり・きんしのちからの効果中は
  `ON_BEFORE_APPLY_AILMENT` ハンドラ自体が無効化される（`add_ability_disabled_reason`）
  ことで、どく/もうどく状態の付与を許可する。技終了後に特性が再有効化されると
  `ON_ABILITY_ENABLED` が発火し、上記の回復ハンドラで即座に回復する
  （モモンのみ/ラムのみとの発動順は、きのみ側のトリガーが技終了前に先に発火する既存の
  優先度関係にそのまま乗るため、本特性固有の実装は不要）。
- `ON_BEFORE_APPLY_AILMENT` / `ON_ABILITY_ENABLED` はいずれも `docs/spec/turn.md` に priority が
  明記されていないイベント（状態異常付与・特性有効化の内部処理）のため、既存の同種ハンドラ
  （めんえき・じゅうなん・みずのベール・やるき等）に倣い priority は指定せず既定値を使う。
- どくびし（`handlers/field.py` の `どくびし_apply_poison`）・どくどくだま
  （`handlers/item.py` の `どくどくだま_apply_poison`）はいずれも最終的に
  `ailment_manager.apply`（`ON_BEFORE_APPLY_AILMENT` を発火）を経由してどく/もうどくを付与するため、
  `prevent_poison_ailment` のブロックが自然に効く。本特性固有の追加実装は不要。
  - どくびしを踏んでも状態異常にならない一方、どくタイプが地面にいればどくびしを消し去れる挙動は
    `どくびし_apply_poison` 側の既存ロジック（タイプ判定が状態異常判定より先）でそのまま成立する。
- 味方2匹を治す順序（自分から見て左→右）は、ダブルバトル以上の挙動であり本プロジェクトの対象外
  （シングルバトルのみ）のため実装しない。
- パステルベールの持ち主自身が場に出た時点で味方が既にどく/もうどく状態だった場合に治す挙動、
  および「どく状態の味方がいる場にパステルベールが出てきた場合／その逆」の相互発動も、
  いずれもダブルバトル以上限定の挙動のため本プロジェクトの対象外。

## 変更候補

- `src/jpoke/data/ability.py`（実装済み。めんえきと同一パターン）
- `src/jpoke/handlers/ability.py`（実装済み。共通ヘルパーを再利用）
- `tests/abilities/test_ability__group.py`（既存の状態異常無効系パラメタライズテストに含まれる）

## テスト観点

1. どく/もうどく状態の付与を防ぐ（`test_状態異常無効`）。
2. かたやぶりの技でどく/もうどく状態にされても技終了直後に回復する
   （`test_状態異常無効_かたやぶりで無効`）。

## 保留・論点

- ダブルバトル以上限定の「味方全体保護」「登場時の相互発動」「治療順序」は
  本プロジェクトの対象外（シングルバトルのみ）のため実装しない。
