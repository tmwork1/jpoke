# しゅうかく 実装計画

## スコープ
- 優先度: -
- 仕様書: `spec/ability/しゅうかく.md`
- 実装方針: バトル効果を先に実装し、例外は段階的に拡張する。
- 非スコープ: 第八世代のフィールド遭遇効果（本リポジトリはバトルロジック中心のため対象外）。
- 補足: きのみ消費後に別アイテムへ差し替わった場合の履歴は、`last_lost_item_name` が
  「直近で `track_loss=True` により失った持ち物」を単純に上書きする設計のため、追加実装なしで
  仕様通りに動作する（新しい持ち物を消費すればそちらが対象になり、消費せず手放しただけなら
  最初のきのみが対象のまま残る）。

## 実装対象イベント（第一候補）
- `Event.ON_TURN_END`（priority=150。`docs/spec/turn.md` の `Event.ON_TURN_END` 節「150 しゅうかく」を参照。
  同じ priority のナイトメア/ものひろい/はんすう等とはすばやさ順でタイブレークされる）

## イベント選定理由
- しゅうかく判定は「ターン終了時に1回」のため、ターン終了イベントに集約する。
- 天候カウントは先行フェーズ（priority=10）で減算されるため、判定時点の天候をそのまま参照できる。
- トリックルームの継続・終了（priority=140）より後に判定させる必要があるため、
  `docs/spec/turn.md` 記載の priority=150 を明示的に指定する（省略するとデフォルト値100になり
  順序が仕様と食い違うため注意）。

## 仕様要点（実装に反映）
- ターン 終了時、自分が最後に使用した きのみ を50%の確率で復活させる。 天気 が にほんばれ ・ おおひでり 状態のときは必ず復活する。
- もちもの を持っているときは発動しない。
- 消費した きのみ が復活するかどうかの判定は、毎 ターン 終了時に1回だけ行われる( ターン#5.ターン終了時の処理 も参照)。
- ターン経過でにほんばれ状態が終了する場合、しゅうかくの発動判定より早く天気が戻るためそのターンのしゅうかく発動率は50%となる。
- ノーてんき / エアロック のポケモンが場にいる場合、にほんばれ / おおひでり 状態でも発動率は50%となる。
- 復活後、使用条件を満たすきのみは即座に使用される。

## 実装フロー（実装済み。以下は実際の設計）
1. `Pokemon.last_lost_item_name: ItemName` に「最後に失った持ち物」を記録する
   （`lost`/`lost_cause` のような別プロパティは追加せず、単一フィールド + 更新可否フラグの
   組み合わせで表現する）。
2. `ItemManager` の `_change_item` / `remove_item` / `swap_items` / `consume_item` /
   `force_trigger_berry` に `track_loss: bool = True` 引数を持たせ、呼び出し元ごとに
   `last_lost_item_name` を更新するかどうかを制御する。
   - 自発消費（通常のきのみ発動・なげつける・しぜんのめぐみ・おちゃかい・ほおばる）:
     `track_loss=True`（デフォルト）→ しゅうかく対象。
   - どろぼう・ほしがる・トリック・すりかえ・ギフトパス、はたきおとす・やきつくす・
     ふしょくガス、むしくい・ついばむで奪って消費した場合: `track_loss=False` を明示 →
     場に存在したまま消滅/移動する扱いのためしゅうかく対象外。
   - `Pokemon.last_lost_item_name` は交代・ひんし復帰を挟んでも保持する
     （`reset_on_switch_out` ではクリアしない。バトル中を通じて保持する `docs/spec/moves/リサイクル.md`
     の「そのバトルで、自分が最後に消費した持ち物」と同じ考え方）。
3. `src/jpoke/handlers/ability.py` に `しゅうかく_restore_berry` を実装する。
	- 発動条件: `last_lost_item_name` がきのみ名（末尾が「のみ」）、かつ現在アイテムを持っていない
	- 確率条件: 通常50%、`battle.weather_for(mon).sunny` が真の場合は100%
	  （`weather_for` がノーてんき/エアロックの影響を内包して「なし」天候を返すため、
	  ハンドラ側で個別に判定する必要はない）
4. `src/jpoke/data/ability.py` の `しゅうかく` に `Event.ON_TURN_END`（priority=150）で登録する。
5. 復活後のきのみ即時使用は既知の未対応事項として残す（下記「既知の制約」を参照）。

## 既知の制約
- 復活直後に使用条件（HPしきい値等）を満たすきのみは仕様上その場で即時使用されるが、
  `gain_item` は `Event.ON_ITEM_GAINED` のみを発火し、オボンのみ等のHPしきい値判定は
  `Event.ON_HP_CHANGED` 側にのみ登録されているため即時発動しない。
  `Event.ON_HP_CHANGED` を直接emitすると、ききかいひ等の無関係なハンドラまで誤発火するため、
  安全な代替手段が見つかるまで対応を見送る（`リサイクル`（技）も同じ制約を持つ）。
- 特性無効化（かがくへんかガス・かたやぶり等）はイベント発火前の共通フィルタで処理されるため、
  本ハンドラ側で個別の無効化チェックは行わない。

## テスト観点
- 50%判定の成功/失敗を乱数固定で再現できる。
- 晴れ継続ターンは100%、晴れ終了ターンは50%になる。
- ノーてんき/エアロック下では晴れ中でも50%になる。
- アイテムあり、候補なし、控えにいる状態では発動しない。
- きのみ消費後に交代し、再度場に出た場合はしゅうかくが発動する（控え中は発動しない）。
- きのみ消費後にひんしになり、げんきのかけら等で復帰した場合もしゅうかくが発動する。
- どろぼう・トリック等で他のポケモンに渡った持ち物は復活しない。
- はたきおとす・やきつくす・ふしょくガスで失った持ち物は復活しない。
