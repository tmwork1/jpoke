# 特性実装計画: かたやぶり

更新日: 2026-07-11

## 仕様参照

- `docs/spec/abilities/かたやぶり.md`

## 実装ゴール

- 攻撃技・変化技を問わず、技を出すときに相手の「防御的な特性」を無視する。
- 無視できるのは `mold_breaker_ignorable` フラグを持つ特性のみ。技終了後は元の状態に復元する。
- 無視した結果、状態異常/状態変化が付与された場合でも、それらの特性が本来持つ
  「即座に回復する」副作用は、無視される側の特性自身の実装（`ON_ABILITY_ENABLED` での
  自己回復ハンドラ）によって再現される。かたやぶり側は特性の有効/無効を切り替えるだけで、
  個別の回復ロジックを持たない。
- かがくへんかガスによりかたやぶり自身が無効化されたときは発動しない
  （`ability_manager` の一般的な特性無効化の仕組みで自然に処理される）。

## 実装方針

- `Event.ON_BEGIN_MOVE`（`src/jpoke/enums/event.py` のコメントに明記された制御系イベント。
  `docs/spec/turn.md` には掲載されないダメージ計算外の制御イベントのため、priority は
  デフォルト（0）とする。技実行前に `core/move_executor.py` が action_success=True の
  ときのみ発火する）で、相手が `mold_breaker_ignorable` フラグを持つ場合、
  `battle.add_ability_disabled_reason(defender, "かたやぶり")` で相手の特性を無効化する。
- `Event.ON_END_MOVE`（技実行後、`finally` ブロックで常に発火）で、
  `battle.remove_ability_disabled_reason(defender, "かたやぶり")` により特性を復元する。
  復元時に `Event.ON_ABILITY_ENABLED` が発火し、無視された特性側の「即座に回復する」
  ハンドラ（実装されている場合）が起動する。
- `subject_spec="attacker:self"` を使い、かたやぶりのポケモンが攻撃者（技の使用者）の
  ときのみ発動する。
- 対象の特性（`mold_breaker_ignorable` フラグを持つ特性）ごとに、ロール一致の仕組み
  （`AttackContext` の `attacker:self` / `defender:self`）により、攻撃時専用の効果
  （例: するどいめの回避無視、しんがんのゴースト無効貫通）は「かたやぶりに攻撃されている
  defender」としては呼ばれないため、特性全体を一時的に無効化しても副作用が出ない。

## 変更候補

- `src/jpoke/data/ability.py` — `かたやぶり` / `ターボブレイズ` / `テラボルテージ`（同一効果）
- `src/jpoke/handlers/ability.py` — `かたやぶり_disable_foe_ability` / `かたやぶり_restore_foe_ability`
- 無視される側の各特性（`mold_breaker_ignorable` フラグを持つ特性）に、必要に応じて
  `Event.ON_ABILITY_ENABLED` の「即座に回復する」ハンドラを実装する
  （`ふみん_cure_sleep_on_enable` が既存パターン）。

## テスト観点

1. 攻撃技で相手の防御的な特性（がんじょう・クリアボディ等）を無視できる。
2. 変化技（なきごえ等）でも相手の特性を無視できる。
3. 技終了後は相手の特性が元に戻る（次のターンには通常通り発動する）。
4. `mold_breaker_ignorable` フラグを持たない特性（マジックガード等）は無視できない。
5. かがくへんかガスでかたやぶり自身が無効化されているときは発動しない。
6. 状態異常/状態変化を無効化する特性（めんえき・じゅうなん・みずのベール・
   マグマのよろい・やるき・パステルベール・すいほう・マイペース・どんかん等）を
   無視して状態異常/状態変化を付与できるが、技終了直後に特性の効果で回復する。
7. スイートベールのように「無視して状態異常にしても回復しない」特性は回復しない。
