# 特性実装計画: メロメロボディ

更新日: 2026-06-05

## 仕様参照

- `docs/spec/abilities/メロメロボディ.md`

## 実装ゴール

- 直接攻撃を受けたとき、30%の確率で攻撃者をメロメロ状態にする。
- 直接攻撃でない技では発動しない。
- 同性・性別不明のポケモンには無効（メロメロの性別制約は volatile_manager が処理）。

## 実装方針

- `Event.ON_DAMAGE_HIT`、`subject_spec="defender:self"` で実装。
- `battle.query.is_contact(ctx)` 後に 30% 判定、成立で `battle.volatile_manager.apply(ctx.attacker, "メロメロ", ...)` を呼ぶ。
- **せいでんき・ほのおのからだ** と同一パターン。

## 変更候補

- `src/jpoke/data/ability.py` — `メロメロボディ` に handlers を登録する
- `src/jpoke/handlers/ability.py` — `メロメロボディ_on_damage` を実装する
- `tests/test_ability.py` — テストを追加する

## テスト観点

1. 直接攻撃を受けて 30% でメロメロが発動する。
2. 非接触技では発動しない。
3. 同性・性別不明の攻撃者にはメロメロが付与されない（volatile_manager 側で制御）。

def メロメロボディ_on_damage(battle, ctx, value):
    if battle.query.is_contact(ctx) and battle.random.random() < 0.3:
        battle.volatile_manager.apply(
            ctx.attacker, "メロメロ", source=ctx.defender, ctx=ctx,
        )
