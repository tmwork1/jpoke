# 特性実装計画: メロメロボディ

更新日: 2026-07-13

## 仕様参照

- `docs/spec/abilities/メロメロボディ.md`

## 実装ゴール

- 直接攻撃を受けたとき、30%の確率で攻撃者をメロメロ状態にする。
- 直接攻撃でない技（非接触技・まもる/タイプ相性で無効化された技・みがわりで防がれた攻撃）では発動しない。
- 相手か自分のどちらかが性別不明、または相手と同性の場合は発動しない
  （技メロメロの `ON_TRY_MOVE_2` による性別チェックは技使用時専用であり、特性トリガーの
  `volatile_manager.apply` 経路には適用されないため、ハンドラ内で明示的に性別を判定する）。
- どんかん・アロマベールを持つ相手には発動しない（`ON_BEFORE_APPLY_VOLATILE` 側の
  `どんかん_prevent_volatile` / `アロマベール_prevent_volatile` が処理する）。
- ぼうごパットを持つ相手には発動しない（`is_contact_reaction` が処理する）。
- かたやぶりの効果がある技に対しても発動する（メロメロボディは `mold_breaker_ignorable`
  フラグを持たないため、`かたやぶり_disable_foe_ability` の対象外で自然に成立する）。

## 実装方針

- `Event.ON_DAMAGE_HIT`（実ダメージが発生したときのみ発火。相性無効・まもる・みがわりに
  よる無効化は自然に除外される）、`subject_spec="defender:self"` で実装。
- `battle.query.is_contact_reaction(ctx)`（接触技かつぼうごパットで防がれていない）に加え、
  性別が有効かつ異性であることを確認したうえで 30% 判定を行い、成立したら
  `battle.volatile_manager.apply(ctx.attacker, "メロメロ", source=ctx.defender)` を呼ぶ。
  適用試行後は成否に関わらず `_announce_ability_triggered(battle, ctx.defender)` で
  特性発動ログを記録する（**のろわれボディ・`_apply_contact_counter_ailment`** と同一パターン）。
- 攻撃側があかいいとを持つ場合にメロメロボディの持ち主もメロメロになる処理は、
  あかいいと側の `ON_VOLATILE_START` ハンドラ（`あかいいと_infatuate_foe`）が
  メロメロ付与全般に対して汎用的に処理するため、メロメロボディ側では実装しない。

## テスト観点

1. 直接攻撃を受けて 30% でメロメロが発動する。
2. かたやぶりの効果がある技に対しても発動する。
3. 相手が性別不明の場合は発動しない。
4. 相手が同性の場合は発動しない。
5. （汎用テストで確認済み）どんかん・アロマベールを持つ相手には発動しない。

def メロメロボディ_maybe_infatuate_attacker(battle, ctx, value):
    attacker = ctx.attacker
    defender = ctx.defender
    if (
        defender.gender != ""
        and attacker.gender != ""
        and attacker.gender != defender.gender
        and battle.query.is_contact_reaction(ctx)
        and battle.random.random() < 0.3
    ):
        battle.volatile_manager.apply(attacker, "メロメロ", source=defender)
        _announce_ability_triggered(battle, defender)
    return HandlerReturn(value=value)
