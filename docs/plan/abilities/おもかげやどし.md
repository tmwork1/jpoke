# 実装計画: おもかげやどし

作成日: 2026-06-04
更新日: 2026-07-11（レビューにより修正）

## 効果
テラスタル後のオーガポン専用。この特性に変化したとき、またはこの特性のまま場に出たとき、
フォルムに対応する能力が 1 段階上がる。場に出るたびに1回しか発動しない
（かがくへんかガスの発動・解除でも再発動しない）。

| フォルム | 対象能力 |
|---------|---------|
| オーガポン（みどりのめん） | S（素早さ） |
| オーガポン（いどのめん） | D（特防） |
| オーガポン（かまどのめん） | A（攻撃） |
| オーガポン（いしずえのめん） | B（防御） |

## flags

```python
{"uncopyable", "protected"}  # すでに設定済み
```

## イベント／ハンドラ

| イベント | priority | subject_spec | 関数名 |
|---------|---------|-------------|--------|
| `ON_SWITCH_IN` | 100 | `source:self` | `おもかげやどし_boost_stat` |
| `ON_ABILITY_ENABLED` | 100 | `source:self` | `おもかげやどし_boost_stat` |
| `ON_TERASTALLIZE` | 100 | `source:self` | `おもかげやどし_boost_stat` |

3イベントとも同一関数を使い、`activated_since_switch_in` フラグで場に出てから1回のみに制限する。
このフラグはどのイベント経由で発動しても共通で立てるため、
「かがくへんかガス発動中は発動せず、解除直後に1回だけ発動する」挙動と
「一度発動した後は再発動しない」挙動の両方を単一のフラグで表現できる。

## 実装

```python
def おもかげやどし_boost_stat(battle, ctx, value):
    """おもかげやどし特性: 場に出たとき（かこの特性に変化したとき）フォルムに対応する能力が1段階上がる。

    場に出るたびに1回しか発動しない（かがくへんかガスの発動・解除が起きても再発動しない）。
    """
    mon = ctx.source
    stat = _OGERPON_STAT.get(mon.name)
    if stat is None:
        return HandlerReturn(value=value)
    if mon.ability.activated_since_switch_in:
        return HandlerReturn(value=value)
    mon.ability.activated_since_switch_in = True
    _announce_ability_triggered(battle, mon)
    battle.modify_stats(mon, {stat: +1}, source=mon)
    return HandlerReturn(value=value)
```

**補足**:
- `activated_since_switch_in` は `pokemon.reset_on_switch_in()` で `False` にリセットされる
- フラグチェックを外すと、場に出た直後 → かがくへんかガス発動 → 解除、という流れで
  二重発動してしまう（レビューで確認済みのバグ）。3イベントとも同一フラグでガードすること。
- このシミュレータはオーガポンの元の特性（まけんき/ちょすい/かたやぶり/がんじょう）から
  テラスタル時に「おもかげやどし」へ動的に切り替える処理を持たない。
  テラスタル運用を想定するオーガポンはチーム編成時点で `ability_name="おもかげやどし"` を
  直接指定する（Pokémon Showdown のチームビルダーと同様の簡略化）。
  そのため `ON_SWITCH_IN` は登場時点で既に発動し、`ON_TERASTALLIZE` は
  （フラグが立っていない場合に限り）テラスタル操作を単体でテストする際の経路として機能する。

## data/ability.py 登録

```python
"おもかげやどし": AbilityData(
    flags={"uncopyable", "protected"},
    handlers={
        Event.ON_SWITCH_IN: h.AbilityHandler(
            h.おもかげやどし_boost_stat,
            subject_spec="source:self",
        ),
        Event.ON_ABILITY_ENABLED: h.AbilityHandler(
            h.おもかげやどし_boost_stat,
            subject_spec="source:self",
        ),
        Event.ON_TERASTALLIZE: h.AbilityHandler(
            h.おもかげやどし_boost_stat,
            subject_spec="source:self",
        ),
    }
),
```

## テストケース
- オーガポン以外には発動しない
- みどりのめん → S+1
- いどのめん → D+1
- かまどのめん → A+1
- いしずえのめん → B+1
- 同じポケモンが 2 回目に発動（交代して再登場後）→ 発動する（フラグリセット）
- テラスタルで能力が上昇する（登場時発動フラグをリセットしてテラスタル起因のみ検証）
- かがくへんかガスにより発動が阻害されている間は発動せず、解除直後に発動する
- 場に出てから既に発動している場合、かがくへんかガスの発動・解除が起きても再発動しない
