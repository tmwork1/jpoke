# 特性実装計画: どくげしょう

更新日: 2026-06-05

## 仕様参照

- `docs/spec/abilities/どくげしょう.md`

## 実装ゴール

- 物理技を受けると、攻撃者の場に「どくびし」を1層設置する（最大2層）。
- 特殊技では発動しない。
- すでに2層のどくびしがある場合は設置しない。

## 実装方針

- `Event.ON_DAMAGE_HIT`、`subject_spec="defender:self"`、priority=20 で実装。
- priority は `turn.md` ON_DAMAGE priority 20 に準拠。
- 物理技判定後、攻撃者のサイドにどくびしを1層追加する。
- `battle.get_side(ctx.attacker)` で攻撃者側の SideFieldManager を取得。どくびしの layer追加は既存の設置技APIを参照。

## 変更候補

- `src/jpoke/data/ability.py` — `どくげしょう` に handlers を登録する
- `src/jpoke/handlers/ability.py` — `どくげしょう_on_damage` を実装する
- `tests/test_ability.py` — テストを追加する

## リスクと対策

- リスク: どくびしの layer 追加に公開 API がない可能性がある。
  - 対策: 既存の「どくびし設置技（どくびし）」の move ハンドラ実装を参照し、同じ方法で追加する。

## テスト観点

1. 物理技受けると攻撃者側にどくびしが1層設置される。
2. 特殊技では発動しない。
3. すでに2層の場合は追加されない。

```python
def どくげしょう_on_damage(battle, ctx, value):
    if battle.resolve_move_category(ctx.attacker, ctx.move) != "物理":
        return HandlerReturn(value=value)
    attacker = ctx.attacker
    if attacker is None:
        return HandlerReturn(value=value)
    foe_side = battle.get_side(attacker)
    spikes = foe_side.get("どくびし")
    if spikes.count < 2:
        foe_side._activate_field("どくびし", 1)  # 1層追加 (APIは実装時に確認)
        announce_ability_triggered(battle, ctx, value, mon=ctx.defender)
    return HandlerReturn(value=value)
```
