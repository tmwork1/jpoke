# 特性実装計画: びびり

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/びびり.md`

## 実装ゴール

1. あく・ゴースト・むしタイプの攻撃技でダメージを受けると、すばやさが1段階上がる。
   - 上記3タイプ以外では発動しない。
   - ダメージが無かった場合（無効化・まもる・みがわり等）は発動しない。
   - S が最大のときは発動しない。
2. （第八世代以降）いかくの効果で自分のこうげきが下がると、すばやさが1段階上がる。
   - いかくで攻撃が下がること自体は防がない（発動条件の判定のみ行う）。
   - しろいきり・フラワーベール・クリアチャーム・クリアボディ等でいかくの効果が
     無効化された場合は発動しない。
   - こうげきが既に最低ランク（-6）でいかくの効果が無かった場合は発動しない。
   - S が最大のときは発動しない。

## 実装方針

### 1. ダメージ由来（既存実装）

- `Event.ON_DAMAGE_HIT`、`subject_spec="defender:self"`、priority=100（既定）で実装。
- `ctx.move.type in ("あく", "ゴースト", "むし")` チェック後、
  `battle.modify_stats(mon, {"spe": +1}, ...)` を呼ぶ。
- **せいぎのこころ** と同パターン（対象タイプが複数の点が異なる）。

### 2. いかく由来（未実装 → 新規追加）

- `Event.ON_BEFORE_MODIFY_STAT`、`subject_spec="target:self"` で実装。
- priority は `docs/spec/turn.md` に未掲載のイベントのため、既存の同種ハンドラ
  （いかくのランク低下に反応する ON_BEFORE_MODIFY_STAT ハンドラ群）を根拠に決定する：
  - `ビビリだま`（アイテム、`docs/spec/items/ビビリだま.md` 参照）: priority=50
    （クリアボディ等の無効化ハンドラより先に判定し、無効化されても発動する）
  - `クリアボディ` / `しろいけむり` / `メタルプロテクト` / `かいりきバサミ` /
    `フラワーベール`（無効化系特性）・`クリアチャーム`（アイテム）: 既定 priority=100
  - `ミラーアーマー`: priority=110（アイテムの既定優先度より後に判定）
  - `しろいきり`（場の効果）: priority=130
  - びびりは「いかくで実際に攻撃が下がったとき」にのみ発動する仕様
    （一次情報: `docs/wiki/abilities/びびり.html` 特性の仕様#第八世代以降。
    ビビリだまと異なり、しろいきり・フラワーベールでの無効化時は発動しない）ため、
    上記の無効化ハンドラがすべて処理を終えた後に判定する必要がある。
    → **priority=140**（しろいきりの130より後）とする。
  - なお、びびり自身は「こうげき」を変化させる特性・アイテムではないため、
    あまのじゃく/ばんけん（下降を上昇に反転する特性）との組み合わせは
    特性の重複所持ができない仕様上発生しない（同一ポケモンが
    びびりとあまのじゃく/ばんけんを同時に持つことはない）。よって
    「`"atk" in value` であれば下降とみなしてよい」という単純な判定で十分。
- `ctx.stat_change_reason == "いかく"` かつ `"atk" in value` かつ
  `ctx.target.rank["atk"] > -6`（既に最低ランクなら いかく は不発なので対象外）
  のときのみ、`battle.modify_stats(mon, {"spe": +1}, source=ctx.source)` を呼ぶ。
- 実際の攻撃ランク低下自体は無効化・変更しない（`value` はそのまま返す）。

## 変更候補

- `tests/abilities/test_ability_ha.py` — いかく由来の発動条件のテストを追加する

## テスト観点

### ダメージ由来（既存）
1. あく技受けると S が 1 段階上がる。
2. ゴースト技受けると S が 1 段階上がる。
3. むし技受けると S が 1 段階上がる。
4. ノーマル等他タイプでは発動しない。

### いかく由来（新規）
5. いかく所持ポケモンが場に出て自分のこうげきが下がると、S が 1 段階上がる。
6. しろいきり状態でいかくの効果が無効化されたときは発動しない。
7. こうげきが既に最低ランク（-6）でいかくが不発のときは発動しない。
8. S が既に最大ランク（+6）のときは発動しない（modify_stats が不発）。

## 疑似コード

```python
def びびり_boost_spd_on_fear_move(battle, ctx, value):
    """既存実装（ダメージ由来）。"""
    if ctx.move.type not in ("あく", "ゴースト", "むし"):
        return HandlerReturn(value=value)
    mon = ctx.defender
    if battle.modify_stats(mon, {"spe": +1}, source=ctx.attacker):
        _announce_ability_triggered(battle, mon)
    return HandlerReturn(value=value)


def びびり_boost_spd_on_intimidate(battle, ctx, value):
    """新規実装（いかく由来、第八世代以降）。"""
    mon = ctx.target
    if (
        ctx.stat_change_reason != "いかく"
        or "atk" not in value
        or mon.rank["atk"] <= -6
    ):
        return HandlerReturn(value=value)
    if battle.modify_stats(mon, {"spe": +1}, source=ctx.source):
        _announce_ability_triggered(battle, mon)
    return HandlerReturn(value=value)
```
