# 特性実装計画: さめはだ

更新日: 2026-07-11

## スコープ

- 仕様書: `docs/spec/abilities/さめはだ.md`
- 実装状態: 実装済み
- 実装方針: `ON_DAMAGE_HIT` で `is_contact_reaction()` を満たしたときに攻撃者へ固定割合ダメージを与える。
  同一パターンの「てつのトゲ」と共通のヘルパー関数 `_apply_contact_counter_chip()` を再利用する。

## 実装ゴール（完了定義）

- 直接攻撃（接触技）でダメージを受けたとき、攻撃してきた相手に最大HPの1/8（切り捨て、最低1）のダメージを与える。
- 非接触技・技が無効化された場合（まもる等）・みがわりに防がれた場合は発動しない。
- まもる/みがわりを無視する直接攻撃で本体がダメージを受けた場合は発動する。
- 攻撃者がマジックガードを持つ場合はダメージを受けない（`ON_MODIFY_NON_MOVE_DAMAGE` 側で汎用的に処理される）。
- ぼうごパット所持者に対しては発動しない（`ON_CHECK_CONTACT_REACTION` 側で汎用的に処理される）。
- 連続攻撃技は1発ごとに発動し、途中で攻撃者がひんしになると以降のヒットは発生しない。
- さめはだ側がひんしになったときも発動する。
- 反撃ダメージはダメおしの「同ターン中にダメージを受けた」判定に加算される。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_DAMAGE_HIT` — 技のダメージが実際にヒットし、防御側の実HPが減少した直後（`turn.md` に個別記載なし。同系統の他特性〔せいでんき等〕と同様デフォルト優先度100を使用）。

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `さめはだ` に `Event.ON_DAMAGE_HIT` ハンドラを登録する（実装済み）
- `src/jpoke/handlers/ability.py` — `さめはだ_chip_contact_attacker()`（内部で共通ヘルパー `_apply_contact_counter_chip()` を呼び出す）を実装する（実装済み）
- `tests/abilities/test_ability_sa.py` — テストを追加する

## 実装ステップ

1. 仕様書 `docs/spec/abilities/さめはだ.md` で発動条件・例外条件を確認する。
2. `src/jpoke/handlers/ability.py` に `さめはだ_chip_contact_attacker()` を実装する。
   接触判定は `battle.query.is_contact_reaction(ctx)` に委譲し、ぼうごパット等の反応効果ブロックを共通処理に任せる。
   ダメージ適用は `battle.modify_hp(ctx.attacker, r=-1/8, reason="")` を用い、マジックガードの無効化は
   `reason=""`（`ON_MODIFY_NON_MOVE_DAMAGE`）側の共通処理に任せる。
3. `src/jpoke/data/ability.py` の `さめはだ` に `Event.ON_DAMAGE_HIT` ハンドラを `subject_spec="defender:self"` で登録する。
4. `tests/abilities/test_ability_sa.py` にテストを追加し、全テストが通ることを確認する。

## テスト観点

1. 接触技を受けたとき、相手の最大HPの1/8（切り捨て、最低1）のダメージを与える。
2. 非接触技（例: みずでっぽう）では発動しない。
3. 技が無効化されたとき（例: タイプ相性0倍の技おはかまいり、まもる系のバーンアクセル）は発動しない。
4. えんかく等で直接攻撃でなくなった技には発動しない。
5. みがわりが技を防いだとき（実HPダメージ0）は発動しない。
6. 攻撃者がマジックガードを持つ場合はダメージを受けない。
7. ぼうごパット所持者に対しては発動しない（`tests/items/test_item_ha.py` で検証済み）。
8. ゴツゴツメット等との併用で合計ダメージになる（`tests/items/test_item_ka.py` で検証済み）。
9. さめはだ側がひんしになったときも反撃ダメージが発生する。
10. 反撃ダメージがダメおしの「同ターン中にダメージを受けた」判定に加算される。
