# 特性実装計画: うのミサイル

更新日: 2026-06-03

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/うのミサイル.md`
- 実装状態: 部分実装（flags=[uncopyable, protected, gas_proof] のみ設定済み）
- 実装方針: なみのり/ダイビング成功時にフォルムチェンジ、攻撃を受けてダメージが入ったとき獲物を吐き出して反撃ダメージ＋副効果を与える。ウッウ専用のため所持者チェックが必要。

## 実装ゴール（完了定義）

- ウッウがなみのり/ダイビングを成功させると HP に応じてフォルムチェンジする（HP 1/2 超 → うのみのすがた、1/2 以下 → まるのみのすがた）。
- 咥えたすがたのとき物理・特殊問わずダメージを受けると獲物を吐き出し、相手最大 HP の 1/4 ダメージ + 副効果（うのみ: ぼうぎょ -1、まるのみ: まひ）を与える。
- みがわりで防いだとき・ダメージ 0 のとき・姿を隠しているとき（一部例外除く）は発動しない。
- ウッウ以外のポケモンがこの特性を得ても発動しない。
- かがくへんかガスで無効化されない（gas_proof は実装済み）。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_MOVE_END` — なみのり/ダイビング成功後にフォルムチェンジを行う
- `Event.ON_DAMAGE_HIT` — 攻撃を受けたとき獲物を吐き出し反撃ダメージと副効果を与える
- `Event.ON_SWITCH_OUT` / `Event.ON_SWITCH_IN` — 交代時にフォルムを「通常のすがた」に戻す

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `うのミサイル` に handlers を登録する
- `src/jpoke/handlers/ability.py` — `うのミサイル_on_move_end`, `うのミサイル_on_damage_hit`, `うのミサイル_on_switch_out` を実装する
- `tests/test_ability.py` — テストを追加する

## 実装ステップ

1. 仕様書 `docs/spec/abilities/うのミサイル.md` でフォルムチェンジ条件・吐き出しの例外（みがわり・ダイビング中・じばく後など）を確認する。
2. `うのミサイル_on_move_end` を実装する（技がなみのり/ダイビングかつ命中成功のとき、HP に応じてフォルムチェンジ）。
3. `うのミサイル_on_damage_hit` を実装する（咥えたすがたでダメージ > 0 のとき、フォルムを「通常のすがた」に戻し反撃ダメージ + 副効果を与える）。
4. `うのミサイル_on_switch_out` を実装する（咥えたすがたで交代するとき「通常のすがた」に戻す）。
5. `src/jpoke/data/ability.py` の `うのミサイル` に handlers を登録する。
6. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. なみのりが命中するとHP半分超なら「うのみのすがた」になる。
2. なみのりが命中するとHP半分以下なら「まるのみのすがた」になる。
3. 「うのみのすがた」のとき攻撃を受けると通常に戻り相手のぼうぎょが -1 になる。
4. 「まるのみのすがた」のとき攻撃を受けると通常に戻り相手がまひ状態になる。
5. みがわりで防いだ場合は吐き出しが発動しない。
6. 交代するとフォルムが「通常のすがた」に戻る。
7. ウッウ以外のポケモンがこの特性を持っても発動しない。

## リスクと対策

- リスク: 「吐き出し」の反撃は攻撃技ではなくゴツゴツメット相当の処理のため、みがわり貫通や特性発動順に注意が必要。
  - 対策: `battle.modify_hp`（非技ダメージ）として実装し、既存のてつのトゲ・さめはだ等の処理パターンを参考にする。
