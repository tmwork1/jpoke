# 特性実装計画: うなぎのぼり

更新日: 2026-07-10

## スコープ

- 仕様書: `docs/spec/abilities/うなぎのぼり.md`
- 実装状態: 未実装（データ登録のみ、ハンドラ未登録）
- 実装方針: ふゆう（地面にいない扱い）とビーストブースト（撃破時に最も高い実数値の能力を1段階上げる）を
  組み合わせた特性であり、いずれも既存ハンドラ関数（`ふゆう_float` / `ビーストブースト_boost_best_stat_on_ko`）と
  同一の挙動なので、新規ハンドラ関数は実装せず既存関数を `data/ability.py` からそのまま登録する
  （既存に `ファントムガード` が `マルチスケイル_reduce_damage` を再利用している前例と同じパターン）。

## 実装ゴール（完了定義）

- じめんタイプの攻撃技を無効化する。
- まきびし・どくびし・ねばねばネット状態、特性ありじごくの影響を受けない（`battle.query.is_floating` が True を返す）。
- かたやぶりの技を受けるときは地面にいる扱いになり、じめん技が通る。
- 攻撃技で相手をひんしにさせたとき、自分の実数値が最も高い能力（同値ならこうげき→ぼうぎょ→とくこう→とくぼう→すばやさの順）が1段階上がる。
- 既存テスト（ふゆう・ビーストブーストのテスト含む）を壊さない。

## 実装対象イベント

- `Event.ON_CHECK_FLOATING` — 浮遊判定時（`battle.query.is_floating` から発火、`EventContext(source=pokemon)`）
- `Event.ON_MOVE_KO` — 攻撃技で相手を撃破した直後（`AttackContext`、`move_executor.py` から発火）

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `うなぎのぼり` に `ON_CHECK_FLOATING`（`ふゆう_float`）と
  `ON_MOVE_KO`（`ビーストブースト_boost_best_stat_on_ko`）の handler、および
  `mold_breaker_ignorable` フラグ（ふゆうと同様、かたやぶりで無効化されるため）を登録する。
- `handlers/ability.py` — 変更なし（既存の `ふゆう_float` / `ビーストブースト_boost_best_stat_on_ko` を再利用）。
- `tests/abilities/test_ability_a.py` — テストを追加する。

## 実装ステップ

1. 仕様書 `docs/spec/abilities/うなぎのぼり.md` で発動条件を確認する（ふゆう・ビーストブーストと同一）。
2. `src/jpoke/data/ability.py` の `"うなぎのぼり": AbilityData()` を、`flags={"mold_breaker_ignorable"}` と
   `ON_CHECK_FLOATING` / `ON_MOVE_KO` の2つの handler を持つ定義に置き換える。
   - `ON_CHECK_FLOATING`: `h.ふゆう_float`, `subject_spec="source:self"`
   - `ON_MOVE_KO`: `h.ビーストブースト_boost_best_stat_on_ko`, `subject_spec="attacker:self"`
3. `tests/abilities/test_ability_a.py` にテストを追加し、全テストが通ることを確認する。

## テスト観点

1. じめんタイプの攻撃技を無効化する（地面にいない扱い）。
2. かたやぶりを持つ相手のじめん技は無効化されない（地面にいる扱いになる）。
3. `battle.query.is_floating` が True を返す。
4. 攻撃技で相手を撃破すると、実数値が最も高い能力が1段階上がる（複数の実数値が候補になるケースで優先順位を確認）。
