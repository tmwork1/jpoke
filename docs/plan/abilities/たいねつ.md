# 特性実装計画: たいねつ

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/たいねつ.md`
- `docs/spec/ailments/やけど.md`

## 実装ゴール

- 相手からほのおタイプの攻撃技を受けたとき、攻撃側のこうげき・とくこう（技の分類に応じて
  ダメージ計算で参照される方）を半減してダメージを計算する（実装済み）。
- タイプ相性の判定には影響しない（そのままダメージが通るが数値が半減する。実装への追加対応不要）。
- かたやぶり等の効果がある技に対しては無効（`mold_breaker_ignorable` フラグで対応済み。
  ハンドラ側に無効化ロジックを重複実装しない）。
- **やけど状態のターン終了時ダメージを半分にする（最大HPの1/32相当、最低1）。**
  → 現状未実装（欠落）。今回のレビューで追加する。

## 現状分析（レビュー時点）

- `Event.ON_CALC_ATK_MODIFIER` によるほのお技ダメージ半減は実装済み・仕様通り。
- やけどのターン終了時ダメージ半減が未実装だった。`やけど_damage`
  (`handlers/ailment.py`) は `battle.modify_hp(mon, v=-damage, reason="burn")` を
  呼ぶだけで、たいねつ側のフックが存在しなかった。
- `modify_hp` は `v < 0` のとき `Event.ON_MODIFY_NON_MOVE_DAMAGE` を発火する
  (`core/status_manager.py`)。この仕組みを使い、`hp_change_reason == "burn"` の場合に
  ダメージを半減するハンドラを追加する。
  既存の同種実装として `いしあたま_ignore_recoil`（`hp_change_reason == "recoil"` を判定）、
  `マジックガード_ignore_damage`（`hp_change_reason` で間接ダメージ判定）がある。

## 実装方針

### ほのお技ダメージ半減（既存・変更なし）

- `Event.ON_CALC_ATK_MODIFIER`、`subject_spec="defender:self"`。
- priority は turn.md に記載のない内部（ダメージ計算内）イベントのため、同一イベント・
  同一 `subject_spec="defender:self"` の既存ハンドラ（**あついしぼう**の
  `あついしぼう_reduce_fire_ice` 等）に合わせ、`AbilityHandler` のデフォルト値
  `priority=100` を使用する。
- `ctx.move.type == "ほのお"` のとき `apply_fixed_modifier(value, 2048)`
  （4096 基準で 0.5 倍）を適用する。

### やけどダメージ半減（新規追加）

- `Event.ON_MODIFY_NON_MOVE_DAMAGE`、`subject_spec="target:self"`。
- priority は turn.md に記載のないイベントのため、同一イベントの既存ハンドラ
  （`いしあたま_ignore_recoil` 等）に合わせ `AbilityHandler` のデフォルト値
  `priority=100` を使用する。
- `ctx.hp_change_reason == "burn"` のとき、`value`（負のHP変化量）の絶対値に
  `apply_fixed_modifier(-value, 2048)` を適用して0.5倍にし、最低1ダメージを
  保証したうえで符号を戻す。
  - `やけど_damage` 側が既に `max(1, mon.max_hp // 16)` で1/16ダメージを算出して
    いるため、ここでさらに0.5倍・最低1を適用すると数学的に
    `max(1, max_hp // 32)` と一致する
    （`floor(floor(n/16)/2) == floor(n/32)` が任意のnで成立するため）。
  - 負の値へ直接 `apply_fixed_modifier` を適用すると floor 方向が逆転し
    ダメージが過大になる（例: 3ダメージ→期待値1だが直接適用だと2になる）ため、
    絶対値に適用してから符号を戻す実装にする。

## テスト観点

1. ほのおタイプの技を受けたとき `atk_modifier` が 2048（0.5倍）になる（既存カバー済み）。
2. かたやぶりを持つ相手の技に対しては無効（既存カバー済み）。
3. やけど状態でターン終了時、通常の半分（最大HPの1/32、端数切り捨て、最低1）の
   ダメージを受ける。
4. やけど状態でも最低1ダメージは保証される（最大HPが小さいケース）。

## 疑似コード

```python
def たいねつ_reduce_burn_damage(battle: Battle, ctx: EventContext, value: int) -> HandlerReturn:
    """たいねつ特性: やけどのターン終了時ダメージを半分にする（最低1）。"""
    if ctx.hp_change_reason == "burn":
        value = -max(1, apply_fixed_modifier(-value, 2048))
    return HandlerReturn(value=value)
```
