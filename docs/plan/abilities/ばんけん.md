# 特性実装計画: ばんけん

更新日: 2026-07-12
実装状況: **✅ 完了**

## 仕様参照

- `docs/spec/abilities/ばんけん.md`

## 実装ゴール

✅ いかくを受けると A が下がらず、逆に 1 段階上がる（特性バーが表れて発動する）。
✅ 相手を交代させる技（ほえる・ふきとばし・ともえなげ・ドラゴンテール等）による強制交代を防ぐ
　（きゅうばんと異なり、特性バー・メッセージは流れないまま無効化される）。
✅ 相手のレッドカードによる強制交代を防ぐ（アイテム自体は消費される）。
✅ かたやぶり等で特性が無視された場合は通常どおり効果を受ける（`mold_breaker_ignorable`）。
✅ 自分のだっしゅつボタン・だっしゅつパックの発動は制限しない。

## イベント／ハンドラ

| イベント | priority | subject_spec | 関数名 |
|---------|---------|-------------|--------|
| `Event.ON_BEFORE_MODIFY_STAT` | 100（`turn.md` 未掲載。イベント計算内部のため、同種の `あまのじゃく_reverse_stat` に合わせてデフォルト値を採用） | `target:self` | `ばんけん_boost_atk_on_intimidate` |
| `Event.ON_TRY_BLOW` | 100（`turn.md` 未掲載。`きゅうばん_block_blow` と同種の「無効化するかどうか」のみを返す制御系ハンドラのため、複数登録されていても実行順に依存せず、デフォルト値を採用） | `defender:self` | `ばんけん_block_blow` |

## 実装方針

- **`ON_BEFORE_MODIFY_STAT` でいかく由来のこうげき低下を反転**
  - `ctx.stat_change_reason == "いかく"` かつ `value` に `"atk"` キーがある場合のみ、
    `value["atk"]` を `+1` に書き換える（あまのじゃくと同じ「対象が自分自身のときに反応する」
    `target:self` 構成）。
  - あまのじゃく（`docs/spec/abilities/あまのじゃく.md`: 「発動してもメッセージは流れない」）と
    異なり、ばんけんはいかくを受けたときに**特性バーが表れて発動する**
    （`docs/spec/abilities/ばんけん.md` 特性の仕様節）。このため
    `_announce_ability_triggered` を呼ぶ。
  - こうげきが既に最大ランク（+6）でいかくの効果が無かったときは発動しない
    （あまのじゃくの同種の仕様・`ビビリだま_boost_speed_on_intimidate` の `at_limit`
    判定と同じロジックを流用し、`ctx.target.rank["atk"] < 6` のときのみ announce する）。
  - みがわり状態は `いかく_lower_foe_atk` 側で `battle.modify_stats` を呼ぶ前に判定して
    いるため、本ハンドラは呼ばれず自然に「発動しない」となる。しろいきり等の
    低下無効化ハンドラが本ハンドラより先に `value` から `"atk"` を除去した場合も、
    `"atk" not in value` により自然に「発動しない」となる。
  - クリアチャームによる能力低下ブロック（`クリアチャーム_block_stat_drop`）より先に
    本ハンドラが実行されるため、`+1` に反転済みの値がクリアチャームに渡り
    「ばんけんの発動が優先されて攻撃が上がる」仕様を満たす（優先度は両者ともデフォルト値で、
    同一ポケモン上の特性ハンドラがアイテムハンドラより先に登録されるため）。
  - くさタイプ + 味方フラワーベールとの素早さ比較による優先度仕様は、
    フラワーベールの現行実装（`subject_spec="target:self"`、味方非対応）を含めダブルバトル
    専用の相互作用のため、本プロジェクトの対象範囲（シングルバトルのみ）外として実装しない。

- **`ON_TRY_BLOW` で強制交代を防ぐ（きゅうばんとは別ハンドラ）**
  - `docs/spec/abilities/ばんけん.md` 「相手を交代させる技を受けたとき、特性バーは現れずに
    交代が無効化される」に対し、`きゅうばん_block_blow` は `_announce_ability_triggered` を
    呼ぶため流用できない。専用の `ばんけん_block_blow` を実装し、
    `HandlerReturn(value=False, stop_event=True)` のみを返す（発動演出なし）。
  - ほえる/ふきとばしの「しかし うまく 決まらなかった!!」表示は `on_blow_apply`
    （`handlers/move_status.py`）・`_force_switch_random`（`handlers/move_attack.py`）側の
    `MOVE_IMMUNED` ログで、ばんけんに限らず共通して処理されるため本ハンドラでは扱わない。
  - `subject_spec="defender:self"`（`AttackContext` の防御側ロールと一致）。

- レッドカードの強制交代は `handlers/item.py` の `_レッドカード_try_force_switch` 内で
  `foe.ability.name in ("きゅうばん", "ばんけん")` を直接判定して防ぐ（変更なし・既存実装）。
  アイテムの消費は先に行われるため、ばんけんが発動してもレッドカードは消費される。
- `flags={"mold_breaker_ignorable"}` は data/ability.py に既定済み。

## 実装完了内容

### src/jpoke/handlers/ability.py

```python
def ばんけん_boost_atk_on_intimidate(battle: Battle, ctx: EventContext, value: dict) -> HandlerReturn:
    """ばんけん特性: いかくによるこうげき低下を、こうげきの上昇に変える。

    あまのじゃく（メッセージなし）と異なり、いかくを受けたときは特性バーが
    表れて発動する（一次情報: docs/wiki/abilities/ばんけん.html 特性の仕様節）。
    こうげきが既に最大ランク（+6）でいかくの効果が無かったときは発動しない
    （あまのじゃくの同種の仕様に準拠。ビビリだまの at_limit 判定と同じロジック）。
    """
    if ctx.stat_change_reason != "いかく" or "atk" not in value:
        return HandlerReturn(value=value)
    if ctx.target.rank["atk"] < 6:
        _announce_ability_triggered(battle, ctx.target)
    return HandlerReturn(value={**value, "atk": 1})


def ばんけん_block_blow(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ばんけん特性: 強制交代技・レッドカードによる交代を防ぐ。

    きゅうばんと異なり、特性バーやメッセージは流れないまま交代のみが
    無効化される（一次情報: docs/wiki/abilities/ばんけん.html 特性の仕様節）。
    ほえる/ふきとばしの「しかし うまく 決まらなかった!!」表示は on_blow_apply /
    _force_switch_random 側の MOVE_IMMUNED ログで別途処理されるため、
    ここでは特性の発動演出（_announce_ability_triggered）を行わない。
    """
    return HandlerReturn(value=False, stop_event=True)
```

### src/jpoke/data/ability.py

```python
"ばんけん": AbilityData(
    flags={
        "mold_breaker_ignorable"
    },
    handlers={
        Event.ON_BEFORE_MODIFY_STAT: h.AbilityHandler(
            h.ばんけん_boost_atk_on_intimidate,
            subject_spec="target:self",
        ),
        Event.ON_TRY_BLOW: h.AbilityHandler(
            h.ばんけん_block_blow,
            subject_spec="defender:self",
        ),
    }
),
```

## テスト観点

✅ 1. いかく発動時 → A が下がらず +1 になり、特性バーが表れる（`ability.revealed == True`）。
✅ 2. こうげきが既に最大ランク（+6）のときはいかくが不発でばんけんも発動しない
　　（`ability.revealed == False` のまま）。
✅ 3. ほえる等の強制交代 → 無効になり、かつ特性バーは表れない（`ability.revealed == False`）。
✅ 4. ともえなげ・ドラゴンテール → ダメージのみで交代しない。
✅ 5. かたやぶり系では無効化されて交代させられる。
✅ 6. レッドカードによる強制交代を防ぐが、アイテムは消費される。

## 保留・論点

  （上記「実装方針」参照）。
