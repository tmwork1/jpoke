# 特性実装計画: ちからもち

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/ちからもち.md`

## 実装ゴール

- 物理技を使用するとき、ダメージ計算に用いる「攻撃」の実数値を2倍にする。
  - イカサマ使用時: イカサマは分類は物理技のまま「相手の攻撃実数値」を代わりに参照するが、
    参照元がどちらであっても攻撃側（技を使用する側）にちからもちがあれば2倍になる。
  - イカサマを受ける側: 自分がちからもちを持っていても、相手がイカサマを使う際の攻撃実数値
    （イカサマ側が参照する自分の実数値）には影響しない。
  - ボディプレス使用時: 分類は物理技のまま「自分の防御実数値」を代わりに参照するが、同様に
    2倍になる。
  - パワートリック状態: こうげき・ぼうぎょの実数値そのものが入れ替わっているため、物理技使用時
    は入れ替わった値（元の防御）が2倍になる。
  - テラバーストなど分類が変わる技: 分類判定はランク補正のみを反映した `ranked_stats` で行われ
    特性補正の影響を受けない。分類が物理に決定した後のダメージ計算でのみちからもちが適用される。
  - こんらんの自傷ダメージ（内部技 `"_こんらん"`）: 第五世代以降は特性の効果が無いため、
    Champions仕様でも適用しない。
- ヨガパワーも同一ハンドラを共有する（既存実装のまま）。

## 実装方針

- `Event.ON_CALC_ATK_MODIFIER`、`subject_spec="attacker:self"` で実装する。
- priority は turn.md に記載のない内部（ダメージ計算内）イベントのため、同一イベント・同一
  `subject_spec="attacker:self"` の既存ハンドラ（**いわはこび**の `いわはこび_modify_atk` など）
  に合わせ、`AbilityHandler` のデフォルト値 `priority=100` を使用する。
- `ctx.move.category == "physical"` のとき `apply_fixed_modifier(value, 8192)`（4096基準で2.0倍）
  を適用する。イカサマ・ボディプレスは分類が `"physical"` のまま実数値の参照元だけが入れ替わる
  実装（`core/damage.py` の `_calc_final_attack`）になっているため、`ctx.move.category` 判定のみで
  自動的に正しく動作し、専用の分岐は不要。パワートリックも同様に、実数値自体が
  `update_stats`／`_stats_manager` 側で入れ替わっているため専用分岐は不要。
- こんらんの自傷ダメージ（`"_こんらん"`）は `category="physical"` のためこのままでは誤って
  2倍になってしまう。コードベース全体で徹底されている除外パターン
  （`ctx.move.name != "_こんらん"`。例: `こだわりハチマキ_boost_physical`、
  `くさのけがわ_boost_B` など）に合わせ、条件に追加する。

## 変更候補

- `src/jpoke/data/ability.py` — `ちからもち` / `ヨガパワー` に `Event.ON_CALC_ATK_MODIFIER`
  ハンドラを登録する（実装済み）。
- `src/jpoke/handlers/ability.py` — `ちからもち_boost_physical` に `"_こんらん"` 除外を追加する
  （本レビューで修正）。

## テスト観点

1. 物理技使用時に `atk_modifier` が 8192（2倍）になる。
2. 特殊技使用時は `atk_modifier` が 4096 のまま変化しない。
3. イカサマ使用時も `atk_modifier` が 8192 になる。
4. イカサマを受ける側では `atk_modifier` が 4096 のまま変化しない。
5. ボディプレス使用時に `atk_modifier` が 8192 になる。
6. パワートリック状態で物理技を使うと、入れ替わった実数値（元の防御）が2倍になる。
7. こんらんの自傷ダメージには補正がかからない（`atk_modifier` が 4096 のまま）。
