## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/なまけ.md`
- 実装状態: 未実装
- 実装方針: ポケモンごとに「怠け変数 X（0: 行動可 / 1: なまける）」を管理し、`ON_TRY_ACTION` で X=1 のとき行動をスキップ、行動ごとに X を反転させる。

## 実装ゴール（完了定義）

- 場に出た最初のターンは行動でき、次のターンはなまけて行動不能になる（以降交互）。
- なまけるターンに「なまけている!」メッセージが表示される。
- ねむり・こおり状態で行動できないターンは X を消費しない（第五世代以降仕様）。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_TRY_ACTION` — 行動時に X を確認し、X=1 のとき行動をスキップして X を 0 に反転させる

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `なまけ` に handler を登録する
- `src/jpoke/handlers/ability.py` — `なまけ_try_action` を実装する
- `src/jpoke/data/models.py`（または `src/jpoke/core/pokemon_state.py`）— ポケモンごとの X 変数（`laziness_flag`）フィールドを追加する
- `tests/test_ability.py` — テストを追加する

## 実装ステップ

1. 仕様書 `docs/spec/abilities/なまけ.md` で X の消費ルール（ねむり/こおり時は消費しない等）を確認する。
2. `Pokemon` モデルまたは揮発性状態に `laziness_flag: bool = False` を追加する。
3. `なまけ_try_action` を実装する（`flag=True` のとき「なまけている!」ログを出して行動をキャンセル、flag 反転）。
4. `ON_SWITCH_IN` で X を初期化する handler も登録する。
5. `src/jpoke/data/ability.py` の `なまけ` に handlers を登録する。
6. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. 1 ターン目は行動でき、2 ターン目はなまけて行動不能になる。
2. なまけるターンに「なまけている!」メッセージが出る。
3. ねむり状態のターンは X を消費しない（起きたターンに行動できる）。

## リスクと対策

- リスク: 交代やかがくへんかガスによる X リセット漏れ。
  - 対策: `ON_SWITCH_IN` と `ON_ABILITY_ENABLED` で必ず X を初期化する。
