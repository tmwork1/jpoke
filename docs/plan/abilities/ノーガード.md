# ノーガード の実装計画

## 概要
命中判定を必中化する特性。攻撃側がノーガード所持 **または** 防御側がノーガード所持のいずれかで、命中率をNoneに設定する。

## 実装対象のイベント
1. **`ON_MODIFY_ACCURACY`** - 命中率補正イベント

## 実装内容

### イベント: ON_MODIFY_ACCURACY（命中率補正）
- **担当**: `handlers/ability.py` に新規ハンドラを追加
- **処理**:
  - 攻撃側がノーガード所持 → None を返す（必中）
  - または防御側がノーガード所持 → None を返す（必中）
  - 条件（無視される）：
    - タイプ相性無効、まもる系、対象不在などは無視できない（別イベントで処理）
    - 半無敵状態（そらをとぶ/あなをほる中）でも必中フラグは設定（当たらないのは別判定）
    - 一撃必殺技の命中判定部分にも適用
    - `かたやぶり` の有無に関わらず適用

### 実装順序
1. `handlers/ability.py` に `ノーガード_modify_accuracy` ハンドラを追加
2. 攻撃側と防御側の両方をチェック
3. `AbilityData` に handlers を **２つのハンドラとして登録**：
   - `subject_spec="attacker:self"` - 攻撃側がノーガード持ちの場合
   - `subject_spec="defender:self"` - 防御側がノーガード持ちの場合
4. テスト追加

## 実装の詳細ポイント
- `return HandlerReturn(value=None)` で必中化（move.accuracy が None のかわり）
- 攻撃側: `ctx.attacker.ability.name == 'ノーガード'`
- 防御側: `ctx.defender.ability.name == 'ノーガード'`
- 複数条件は OR で結合（いずれかなら必中）
- 同じハンドラを２つ登録して、攻撃側・防御側の両方に対応

## テスト項目
- 攻撃側ノーガード時に低命中技が必中になる
- 防御側ノーガード時に低命中技が必中になる
- 相互ノーガード時に両者とも必中化
- まもる系は通常通り防がれる
- 半無敵状態は当たらない（別判定なので正常）
- 一撃必殺技の命中判定でも機能
