# 特性実装計画: ヨガパワー

更新日: 2026-07-14

## 仕様参照

- `docs/spec/abilities/ヨガパワー.md`
- `docs/spec/abilities/ちからもち.md`（特性の仕様はちからもちを参照する仕様）

## 実装ゴール

- 物理技を使用するとき、ダメージ計算に用いる「攻撃」の実数値を2倍にする。
- 効果・仕様はちからもちと完全に同一（`docs/spec/abilities/ヨガパワー.md` の
  「特性の仕様」がちからもちを参照する仕様のため）。
  - イカサマ使用時: 分類は物理技のまま「相手の攻撃実数値」を代わりに参照するが、
    参照元がどちらであっても攻撃側（技を使用する側）にヨガパワーがあれば2倍になる。
  - イカサマを受ける側: 自分がヨガパワーを持っていても、相手がイカサマを使う際の攻撃実数値
    （イカサマ側が参照する自分の実数値）には影響しない。
  - ボディプレス使用時: 分類は物理技のまま「自分の防御実数値」を代わりに参照するが、同様に
    2倍になる。
  - パワートリック状態: こうげき・ぼうぎょの実数値そのものが入れ替わっているため、物理技使用時
    は入れ替わった値（元の防御）が2倍になる。
  - テラバーストなど分類が変わる技: 分類判定はランク補正のみを反映した `ranked_stats` で行われ
    特性補正の影響を受けない。分類が物理に決定した後のダメージ計算でのみヨガパワーが適用される。
  - こんらんの自傷ダメージ（内部技 `"_こんらん"`）: 第五世代以降は特性の効果が無いため、
    Champions仕様でも適用しない。

## 実装方針

- `Event.ON_CALC_ATK_MODIFIER`、`subject_spec="attacker:self"` で実装する。
- ちからもちと完全に同一効果のため、ハンドラ関数 `ちからもち_boost_physical` を共用する
  （新規関数は実装しない）。
- priority は turn.md に記載のない内部（ダメージ計算内）イベントのため、既存の同種ハンドラ
  （ちからもち・いわはこび等）と同様に `AbilityHandler` のデフォルト値 `priority=100` を使用する。
- `ctx.move.category == "physical"` かつ `ctx.move.name != "_こんらん"` のとき
  `apply_fixed_modifier(value, 8192)`（4096基準で2.0倍）を適用する。イカサマ・ボディプレスは
  分類が `"physical"` のまま実数値の参照元だけが入れ替わる実装（`core/damage.py` の
  `_calc_final_attack`）になっているため、`ctx.move.category` 判定のみで自動的に正しく動作し、
  専用の分岐は不要。パワートリックも同様に、実数値自体が `update_stats`／`_stats_manager` 側で
  入れ替わっているため専用分岐は不要。

## テスト観点

1. 物理技使用時に `atk_modifier` が 8192（2倍）になる。
2. 特殊技使用時は `atk_modifier` が 4096 のまま変化しない。
3. イカサマ使用時も `atk_modifier` が 8192 になる。
4. こんらんの自傷ダメージには補正がかからない（`atk_modifier` が 4096 のまま）。
