# マジックガード 実装計画

## スコープ
- 優先度: H
- 仕様書: `spec/abilities/マジックガード.md`
- 実装方針: `StatusManager.modify_hp` が発火する `Event.ON_MODIFY_NON_MOVE_DAMAGE` に
  特性ハンドラを登録し、`reason`（`HPChangeReason`）を見て間接ダメージだけを一括で
  無効化する。個別の間接ダメージ発生源（状態異常・場の状態・揮発性状態・道具・
  相手特性）側には重複ガードを実装しない。

## 現状整理
- `src/jpoke/core/status_manager.py` の `modify_hp` は、`v < 0`（ダメージ）のとき常に
  `Event.ON_MODIFY_NON_MOVE_DAMAGE` を発火してから実際のHP変化を確定する。
- 技によるダメージ本体は `move_executor.py` 側の別経路（`Event.ON_HIT` 等）で処理され、
  `modify_hp` 呼び出し時には `reason="move_damage"` が付与される。
- `types/literals.py` の `HPChangeReason` に、マジックガード等の判定に必要な理由が
  列挙済み: `move_damage` / `sandstorm` / `poison` / `burn` / `recoil` / `self_attack` /
  `pain_split` / `self_cost` / `fixed_recoil` / `bench_heal` / `drain` / `perish`、
  および理由なし（`""`、まきびし・ステルスロック・くろいヘドロ等の既定値）。
- 自己HP消費（はらだいこ・みがわり・しっぽきり・みをけずる・ソウルビート等）は
  `reason="self_cost"` で呼び出し済み。こんらん自傷は `reason="self_attack"`、
  いたみわけは `reason="pain_split"`、ほろびのうた/みちづれのひんしは
  `reason="perish"` で呼び出し済み。
- わるあがきの反動は仕様上マジックガードで防げない特殊な反動のため、他の反動技
  （`reason="recoil"` / `reason="fixed_recoil"`）と区別して `reason="self_cost"` を
  使っている（`handlers/move_attack.py` の `わるあがき_self_damage`）。

## 実装対象（共通ゲート方式）

| 層 | 対応箇所 | 目的 |
|----|----------|------|
| 特性ハンドラ | `src/jpoke/handlers/ability.py` の `マジックガード_ignore_damage` | `Event.ON_MODIFY_NON_MOVE_DAMAGE` を受け、除外理由以外の間接ダメージを0にする |
| 登録 | `src/jpoke/data/ability.py` の `マジックガード` | 上記ハンドラを `subject_spec="target:self"` で登録する |
| 呼び出し側の理由付与 | 各 `battle.modify_hp(..., reason=...)` 呼び出し | `move_damage` / `self_attack` / `pain_split` / `self_cost` / `perish` を正しく渡す（新規の間接ダメージ経路を追加する際は必ず遵守する） |

## 仕様要点（実装に反映）
- 攻撃技の本体ダメージ（`reason="move_damage"`）は通常どおり受ける。みらいよち/
  はめつのねがいの発動ダメージも `reason="move_damage"` を使うため通常どおり受ける。
- はらだいこ・みがわりのような自己HP消費（`reason="self_cost"`）は「間接ダメージ」
  ではないため通常どおり発生する。わるあがきの反動もこの区分に含める。
- こんらん自傷（`reason="self_attack"`）・いたみわけ（`reason="pain_split"`）・
  ほろびのうた/みちづれのひんし（`reason="perish"`）は防げない。
- それ以外の間接ダメージ（状態異常・天候・設置技・バインド・呪い・しおづけ・
  やどりぎのタネ・自分の道具・相手の道具・相手の特性による反撃ダメージ等）は
  すべて無効化する。
- `かがくへんかガス` 等で特性が無効化されている間はハンドラ自体が発火しないため、
  間接ダメージを通常どおり受ける（`gas_proof` フラグを持たないため）。
- かたやぶり系の効果を持つ技を受けても、この特性は防御側特性でありダメージ計算
  自体をブロックするものではないため、かたやぶりとは無関係に発動する（無視されない）。

## 実装アプローチ

`handlers/ability.py`:

```python
def マジックガード_ignore_damage(battle: Battle, ctx: EventContext, value: int) -> HandlerReturn:
    """マジックガード特性: 間接ダメージを無効化する。"""
    # 直接ダメージ・自己由来の特定HP変動・ほろびのうた/みちづれのひんしは無効化しない。
    if ctx.hp_change_reason not in {"move_damage", "pain_split", "self_attack", "self_cost", "perish"}:
        value = 0
    return HandlerReturn(value=value)
```

`data/ability.py`:

```python
"マジックガード": AbilityData(
    handlers={
        Event.ON_MODIFY_NON_MOVE_DAMAGE: h.AbilityHandler(
            h.マジックガード_ignore_damage,
            subject_spec="target:self",
        )
    }
),
```

- `Event.ON_MODIFY_NON_MOVE_DAMAGE` は `docs/spec/turn.md` に未掲載の内部イベント
  （ダメージ計算内部・HP変更共通経路）のため、既存の同種ハンドラ（`ポイズンヒール`
  等の `ON_MODIFY_NON_MOVE_DAMAGE` 登録済みハンドラ）を参照し、priority 指定なし
  （デフォルト）とする。同一イベントに登録される他の特性ハンドラとの優先順位が
  問題になる実例は確認されていない。

## テスト観点
- `HPChangeReason=move_damage` は無効化しない（技ダメージ）。
- `HPChangeReason=self_attack` は無効化しない（こんらん自傷）。
- `HPChangeReason=pain_split` は無効化しない（いたみわけ）。
- `HPChangeReason=self_cost` は無効化しない（はらだいこ・わるあがきの反動）。
- `HPChangeReason=perish` は無効化しない（ほろびのうた・みちづれのひんし）。
- 理由なし・`sandstorm` / `poison` / `burn` / `recoil` / `fixed_recoil` / `drain` の
  代表入力（どく/やけど/すなあらし/ステルスロック/まきびし/バインド/しおづけ/
  のろい/やどりぎのタネ/いのちのたま反動/ゴツゴツメット/ジャポのみ・レンブのみ/
  くろいヘドロ/くっつきバリ/さめはだ/ナイトメア/ヘドロえき/ニードルガード/
  てっていこうせん・クロロブラスト/とびげり・とびひざげり・サンダーダイブの
  失敗反動）で回帰する。
- 同一 `HPChangeReason`（理由なし）で、`かがくへんかガス` 有効時のみ無効化されない
  ことを確認する（`tests/abilities/test_gas.py`）。
