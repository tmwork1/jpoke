# 特性実装計画: プレッシャー

更新日: 2026-07-12

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/プレッシャー.md`
- 実装状態: ハンドラ実装済み（`handlers/ability.py`）、`data/ability.py` に登録済み
- 実装方針: `ON_SWITCH_IN` でアナウンスを出し、`ON_MODIFY_PP_CONSUMED` で PP 消費量を +1 する。両ハンドラは `handlers/ability.py` に実装済み。

## 実装ゴール（完了定義）

- 登場時に「プレッシャーを はなっている!」メッセージが表示される。
- 相手がこちら（プレッシャー所持者）を対象に技を使ったとき、その技の PP が通常より 1 多く減る。
- 命中しない・まもる・みがわりで防いだ場合でも PP が多く減る。
- 自分を対象にする技（つるぎのまい等）は基本的にプレッシャーの影響を受けない。
- ふういん・のろい（ゴーストタイプの"呪い"）は例外的にプレッシャーの影響を受ける。
- かたやぶりの影響を受けない（アーキテクチャ上、PP消費は `ON_BEGIN_MOVE`＝かたやぶり適用より前に発生するため自然に満たされる）。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_SWITCH_IN` — 登場時にアナウンスを出す（subject_spec="source:self"）
- `Event.ON_MODIFY_PP_CONSUMED` — PP消費時に +1 する（subject_spec="defender:self"、こちらが技の対象＝`AttackContext.defender` になったとき）

`ON_MODIFY_PP_CONSUMED` は `core/move_executor.py` の `_consume_pp` で
`emit(Event.ON_MODIFY_PP_CONSUMED, ctx, 1)` として発火済み。`ctx` は `AttackContext`
（`attacker`=技使用者, `defender`=対象）であり、`EventContext` の `source`/`target` ロールは
持たない。そのため `subject_spec` は `"defender:self"` を用いる（`"target:self"` は
`AttackContext.resolve_role` で `ValueError` になる不正な指定）。

### 自分を対象にする技の除外と例外（仕様書「特性の仕様」節）

`_consume_pp` 時点で `ctx.move.target` が確定しているため、これを使って
「基本的に自分が対象の技はプレッシャーの影響を受けない」を判定する。

- `ctx.move.target` が `"foe"` / `"foe_side"` / `"field"` のとき: 通常どおり +1 する
  （シングル対戦のみが対象のため、`"foe_side"`（相手の場全体）と `"field"`（全体の場）は
  常に唯一の相手ポケモンを含み、実質 `"foe"` と同じ扱いでよい）。
- `ctx.move.target` が `"self"` / `"own_side"` のとき: 原則 +1 しない。ただし例外:
  - `ふういん`（`target="self"`）: 例外的に相手のプレッシャーの影響を受けるため常に +1 する。
  - `のろい`（`target` は既定値 `"foe"` のため上記の分岐では常に+1 対象になってしまうが、
    ゴーストタイプ以外が使う"鈍い"はプレッシャーの影響を受けないため、
    `ctx.attacker.has_type("ゴースト")` が偽のときは +1 しない）。

### 現時点で個別対応が不要な例外（今後実装される場合は要再検討）

- `カウンター` / `ミラーコート` / `メタルバースト` / `ほうふく`: この実装では `target` は
  既定値 `"foe"`（自分対象として定義されていない）のため、追加の分岐なしで仕様どおり動作する。
  「対象技を使われず失敗したときはプレッシャーの効果が無い」という成否条件までは未対応
  （PP消費が `ON_TRY_MOVE_1` より前に発生するアーキテクチャ上の制約）。
- `テラバースト` / `ドラゴンアロー`: 複数体を相手にする場合の特例だが、シングル対戦のみが
  対象のため通常の `"foe"` 対象と同じ扱いで仕様を満たす。
- `きりふだ`: PP消費（プレッシャーの補正込み）は威力計算より前に確定するため追加対応不要。
- `よこどり` / `マジックコート` / `ものまね` / `さきどり` / `へんしん`: 本リポジトリ未実装のため対象外。
  実装時にプレッシャーとの相互作用を要再検討。
- 複数体のプレッシャー所持者・味方から受けた技（PP増加なし）: シングル対戦のみが対象のため
  発生しない。

## テスト観点

1. 登場時にアナウンスが出る。
2. 相手がこちらを対象に技を使うと、PP が通常より 1 多く減る。
3. 命中しなかった場合も PP が多く減る。
4. まもるで防いだ場合も PP が多く減る。
5. 自分を対象にする技（つるぎのまい等）は PP が多く減らない。
6. ふういんは自分対象の技だが PP が多く減る。
7. のろいはゴーストタイプ（呪い）のときのみ PP が多く減り、それ以外のタイプ（鈍い）では減らない。
8. かたやぶり所持者が技を使っても PP は多く減る（かたやぶりの影響を受けない）。
