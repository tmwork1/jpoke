# 特性実装計画: ゆうばく

更新日: 2026-07-14

## 仕様参照

- `docs/spec/abilities/ゆうばく.md`

## 実装ゴール

- 直接攻撃でひんしになったとき、攻撃者に攻撃者自身の最大HPの1/4ダメージを与える（最低1）。
- 直接攻撃でない技でのひんしでは発動しない。
- 攻撃者が反動等ですでにひんしのときは発動しない。
- マジックガード持ちの攻撃者にはダメージを与えられない（`マジックガード_ignore_damage`側で処理。
  `reason="ability"`が許可リストに含まれないため自動的にブロックされる）。
- ぼうごパットを持つ攻撃者にはダメージを与えられない（`is_contact_reaction`側で処理）。
- しめりけ持ちの攻撃者には発動しない（シングル対象のため、しめりけ判定は攻撃者自身の
  ability名で行う）。

## 実装方針

- `Event.ON_MOVE_KO`、`subject_spec="defender:self"` で実装。
- **とびだすなかみ**（被弾直前HPをダメージとして返す）と同じ `ON_MOVE_KO` イベントを使用。
- 反応対象の接触判定には`battle.query.is_contact_reaction(ctx)`を使う
  （ぼうごパット等での無効化を自動的に反映するため）。
- `max(1, ctx.attacker.max_hp // 4)` を計算し `battle.modify_hp(attacker, -damage, reason="ability")`
  を呼ぶ。`reason="ability"`によりマジックガードは自動的にこのダメージを無効化する。

## 実装上の既知の制限

- **ゴツゴツメット/ジャポのみ/レンブのみとの発動順序**: 仕様では
  「ゆうばくのダメージの後に持ち物のダメージ」だが、現在のエンジンでは
  これらの持ち物が`Event.ON_DAMAGE_HIT`（ゆうばくの`Event.ON_MOVE_KO`より前に発火）
  で実装されているため、実際は持ち物側が先に適用される。`Event.ON_MOVE_KO`が
  `Event.ON_DAMAGE_HIT`より後に発火する設計はミイラ等の特性書き換え順序を守るために
  意図的なものであり（`ソウルハート_boost_spa_on_ko`のdocstring参照）、本特性単体の
  レビューでエンジン全体の発火順序を変更するのは影響範囲が広く見送る。
  合計ダメージは攻撃者の残りHPで頭打ちになるため数値上の結果は変わらないが、
  片方の効果だけで丁度ひんしになるHPのとき、表示される特性バー/メッセージが
  本来と異なる場合がある（`docs/spec/abilities/ゆうばく.md`備考節に記載）。
- **かたやぶりによるしめりけ無視**: 仕様上は存在するが、本プロジェクトの対象は
  シングルバトルのみのため、かたやぶりを持つ攻撃者自身がしめりけを同時に持つことは
  ありえず（アビリティは1つのみ）、この組み合わせが成立するのはダブルバトルで
  第三のポケモンがしめりけを持つ場合に限られる。シングル環境では到達不可能な
  分岐のため、対応するテストは追加しない。

## テスト観点

1. 直接攻撃でKOされると攻撃者に最大HP1/4ダメージが入る。
2. 非接触技でのKOでは発動しない。
3. 攻撃者が反動（とっしん等）で既にひんしのときは発動しない。
4. マジックガード持ちの攻撃者にはダメージが入らない。
5. ぼうごパットを持つ攻撃者にはダメージが入らない。
6. しめりけ持ちの攻撃者には発動しない。
7. 攻撃者がみがわり状態でも本体のHPにダメージが入る。
8. ゆうばくの反撃で相手も同時にひんしになった場合、ゆうばく側が負ける
   （`battle.judge_winner()`で確認）。

def ゆうばく_damage_attacker_on_ko(battle, ctx, value):
    attacker = ctx.attacker
    if not battle.query.is_contact_reaction(ctx) or attacker.fainted:
        return HandlerReturn(value=value)

        return HandlerReturn(value=value)

    battle.modify_hp(attacker, -damage, reason="ability")
