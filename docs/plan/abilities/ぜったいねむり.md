# 特性実装計画: ぜったいねむり

更新日: 2026-06-03

## 【最優先】計画の修正
（以下はclaude codeとのやりとり）
ゆめうつつという状態異常を追加して入場時に付与し、ailment.is_sleep: bool でねむりとして判定する。

変更箇所	内容
type_defs.py	AilmentName に "ゆめうつつ" 追加
data/models.py	AilmentData に is_sleep: bool = False, uncurable: bool = False 追加
model/ailment.py	is_sleep / uncurable プロパティ追加（data へ委譲）
data/ailment.py	"ゆめうつつ" エントリ追加（ハンドラなし、is_sleep=True, uncurable=True）
core/ailment_manager.py	remove() / apply(overwrite=True) に uncurable ガード追加
data/ability.py + handlers/ability.py	ぜったいねむり：ON_SWITCH_IN でメッセージ出力 + ailment 付与

メリット（筋が良い理由）
状態異常ブロックが自動化される

apply() の if target.ailment.is_active and not overwrite チェックにより、ゆめうつつ持ちには他の状態異常が入らない。どくびし・どくどくだまも自動的に防げる。

ailment.is_sleep で「ねむり扱い」が一元化できる

めざましビンタ・いびき・ねごと・たたりめなどが ailment.is_sleep を見るだけでよくなる。特性名チェックが移動ハンドラに散らばらない。

交代後も残る

ailment は既に switch-out 後も Pokemon に保持されるので、永続性は自然に担保される。

必要な追加変更（提案に足りない部分）
① is_sleep は AilmentData に置く

既存パターンと揃えるために、フラグは data/models.py の AilmentData に持ち、Ailment.is_sleep は self.data へ委譲するプロパティにする。

② uncurable フラグが必要

アロマセラピー・ラムのみ・じょうか等はすべて ailment_manager.remove() を経由する。ゆめうつつを削除不能にするには、AilmentData に uncurable: bool = False を追加し、ailment_manager.remove() の先頭でガードする。overwrite=True の apply() も同様にガードが必要。

③ 入場メッセージの扱い

apply() は「同じ ailment の重ね掛け不可」ガードにより2回目以降は False を返すが、仕様では 毎回 「夢うつつの状態!」と表示する。よって ability ハンドラはメッセージ出力を ailment 付与の成否と分離して書く必要がある。

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/ぜったいねむり.md`
- 実装状態: 部分実装（flags=[uncopyable, protected, gas_proof] のみ設定済み）
- 実装方針: `ON_BEFORE_APPLY_AILMENT` で全状態異常を無効化し、「夢うつつ」フラグを登場時に立てる。めざましビンタ・たたりめ・ひゃっきやこうの威力 2 倍補正も追加する。いびき・ねごとが使えるよう「常にねむり状態と見なす」フラグを設定する。

## 実装ゴール（完了定義）

- あらゆる状態異常（ねむり含む）にならない。
- いびき・ねごとが使える（常にねむり状態扱い）。
- めざましビンタを受けると威力が 2 倍になる。
- たたりめ・ひゃっきやこうを受けると威力が 2 倍になる（タイプ相性無効の場合除く）。
- かがくへんかガスで無効化されない（gas_proof は実装済み）。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_BEFORE_APPLY_AILMENT` — 全状態異常を無効化する
- `Event.ON_CHECK_ASLEEP`（または同等のイベント）— 常にねむり状態と見なす（いびき・ねごと有効化）
- `Event.ON_CALC_POWER_MODIFIER` — めざましビンタ・たたりめ・ひゃっきやこうの威力を 2 倍にする

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `ぜったいねむり` に handlers を登録する
- `src/jpoke/handlers/ability.py` — `ぜったいねむり_prevent_ailment`, `ぜったいねむり_always_asleep`, `ぜったいねむり_boost_sleep_moves` を実装する
- `tests/test_ability.py` — テストを追加する

## 実装ステップ

1. 仕様書で「夢うつつ」の詳細仕様（技の制限なし、ねむり扱いの範囲）を確認する。
2. `ぜったいねむり_prevent_ailment` を実装する（全状態異常を無効）。
3. `ぜったいねむり_always_asleep` を実装する（ねむり状態確認イベントに対して常に `True` を返す）。
4. `ぜったいねむり_boost_sleep_moves` を実装する（めざましビンタ・たたりめ・ひゃっきやこうの威力を 2 倍）。
5. `src/jpoke/data/ability.py` の `ぜったいねむり` に handlers を登録する。
6. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. どく・まひ・やけど・ねむりなどいかなる状態異常にもならない。
2. いびき・ねごとが通常通り使える。
3. めざましビンタを受けると通常の 2 倍のダメージを受ける。
4. たたりめがタイプ有効なら通常の 2 倍のダメージを受ける。
