# 特性実装計画: わざわいのつるぎ

更新日: 2026-07-14

## スコープ

- 仕様書: `docs/spec/abilities/わざわいのつるぎ.md`
- 実装状態: 実装済み（本レビューで「防御側自身がわざわいのつるぎを持つ場合は対象外」判定の
  欠落を修正）
- 実装方針: `Event.ON_CALC_DEF_MODIFIER` にハンドラを登録し、`subject_spec="attacker:self"`
  でこの特性の所持者が攻撃側のときに発動する。防御側の防御補正を0.75倍にする。
  自分自身が防御側でもある場合（こんらんの自傷等）と、防御側自身もわざわいのつるぎを持つ場合
  （コピー・交換等による取得を含む）は対象外。

## 実装ゴール（完了定義）

- 場にいる間、この特性を持つポケモン以外の全員がダメージを受けるとき、防御側の防御補正が
  0.75倍になる。
- この特性の所持者自身が防御側であるとき（自分がダメージを受ける、こんらんの自傷を含む）は
  対象外（自分自身は効果を受けない）。
- 防御側自身もわざわいのつるぎを持つ場合（コピー・交換等による取得を含む）は対象外
  （防御は下がらない）。攻撃側が同じくわざわいのつるぎを持っていても、防御側の判定のみで
  免除が決まる。
- かたやぶりで無視できない（`AbilityData.flags` に `mold_breaker_ignorable` を付与しない）。
- ボディプレスは使用者自身の防御を参照する技だが、`ON_CALC_DEF_MODIFIER` は防御側
  （＝技の受け手）にしか作用しないため、使用者側の防御参照には影響しない
  （仕様どおり下降補正はかからない）。
- こんらんの自傷ダメージは通常のダメージ計算経路を通らない（固定計算）ため、本特性の影響を
  受けない（既存の自傷ダメージ実装に委ねる。特別分岐は不要）。
- サイコショック/サイコブレイク/しんぴのつるぎ等、分類が変わる技であっても、防御側の防御
  補正を参照する場合（分類が物理に決まった場合、あるいは常にぼうぎょを参照する技）は
  `ON_CALC_DEF_MODIFIER` を通るため自然に対象になる（特別分岐は不要）。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_CALC_DEF_MODIFIER` — 防御側の防御補正（final_defense への乗算補正）を計算する際に
  発火。`docs/spec/turn.md` 未掲載のダメージ計算内部イベント。既存の `わざわいのたま` 等
  同種ハンドラに合わせ、priority は指定せずデフォルト値 (100) を用いる。
  `subject_spec="attacker:self"` で、この特性の所持者が攻撃側のときのみ発動する。
