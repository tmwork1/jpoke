# 特性実装計画: びんじょう

更新日: 2026-06-03

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/びんじょう.md`
- 実装状態: 未実装
- 実装方針: `ON_MODIFY_STAT` イベントで相手のランクが上昇したとき、同じだけ自分のランクも上昇させる。自分の該当ランクが最大の場合は発動しない。

## 実装ゴール（完了定義）

- 相手のランクが上昇したとき、自分も同じだけランクが上昇する。
- 自分の該当ランクがすでに最大（+6）のときは発動しない。
- 実際に上昇した量（上限クリップ後）をコピーする。
- 味方のランクが上昇したときは発動しない。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_MODIFY_STAT` — ランク変化後に発動し、相手のランク上昇をコピーする

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `びんじょう` に handler を登録する
- `src/jpoke/handlers/ability.py` — `びんじょう_copy_stat_rise` を実装する
- `tests/test_ability.py` — テストを追加する

## 実装ステップ

1. 仕様書 `docs/spec/abilities/びんじょう.md` でコピー対象・例外条件（しろいハーブ等で上がった分は対象外など）を確認する。
2. `src/jpoke/handlers/ability.py` に `びんじょう_copy_stat_rise` を実装する（相手のランク上昇イベント時に自分の同ランクを同量上昇）。
3. `src/jpoke/data/ability.py` の `びんじょう` に handler を登録する。
4. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. 相手がつるぎのまいでこうげきを 2 上げたとき、自分のこうげきも 2 上がる。
2. 相手が複数ランクを同時に上げたとき（めいそうなど）、全ランクをコピーする。
3. 自分の対象ランクが最大のとき発動しない。
4. 実際に上昇した量（+5→+6 のとき実際は +1）のみコピーする。
5. 味方のランク上昇では発動しない。
