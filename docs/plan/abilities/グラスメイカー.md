# 特性実装計画: グラスメイカー

更新日: 2026-07-11

## スコープ

- 仕様書: `docs/spec/abilities/グラスメイカー.md`
- 実装状態: 実装済み（レビューでON_ABILITY_ENABLEDの登録漏れを修正）
- 実装方針: `Event.ON_SWITCH_IN` と `Event.ON_ABILITY_ENABLED` で
  `TerrainManager.apply` を呼び出し、グラスフィールドを5ターン展開する
  （既存の `_activate_terrain` 共通ヘルパーを利用）

## 実装ゴール（完了定義）

- 場に出たとき、フィールドをグラスフィールド状態にする（5ターン、
  グランドコート所持時は8ターン）
- 既にグラスフィールド状態のときは発動せず、継続ターンも書き換わらない
- 他のフィールド発生特性（エレキメイカー/ミストメイカー/サイコメイカー/
  ハドロンエンジン）と同時に場に出た場合、補正込みすばやさが高い方が
  優先されるが、実装はハンドラの `subject_spec="source:self"` により
  `ON_SWITCH_IN` の行動順（すばやさ順）に従うため個別実装は不要
- スキルスワップ・トレース・かがくへんかガス解除等で特性がグラスメイカーに
  書き換わった場合、即座に発動する（`Event.ON_ABILITY_ENABLED`）
- 既存テストを壊さない

## 実装対象イベント

- `Event.ON_SWITCH_IN` — 場に出たときにフィールドを展開する
  （`docs/spec/turn.md` ON_SWITCH_IN: priority 100「グラスメイカー（特性）」）
- `Event.ON_ABILITY_ENABLED` — 特性がグラスメイカーに変化した瞬間に発動する。
  `turn.md` に未掲載のイベントだが、同種の `エレキメイカー`/`ハドロンエンジン`
  が `ON_SWITCH_IN` と同じ関数を priority省略（デフォルト100）で登録して
  いるため、それに合わせる

## リスクと対策

- リスク: `ON_ABILITY_ENABLED` はメガシンカ・スキルスワップ・トレース・
  かがくへんかガス解除など複数の経路で発火するため、意図しない二重発動が
  起きないか確認が必要。
  - 対策: `TerrainManager.apply` は同名フィールドが既に有効な場合は何もせず
    `False` を返す実装になっており、二重発動しても副作用は起きない。
