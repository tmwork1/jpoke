# てんのめぐみ 実装計画

## スコープ
- 優先度: -
- 仕様書: `docs/spec/abilities/てんのめぐみ.md`
- 実装方針: `Event.ON_MODIFY_SECONDARY_CHANCE` で追加効果の発動確率を2倍にする
  （上限100%）。`battle.resolve_secondary_chance()` を経由するすべての追加効果
  （ランク変化・状態異常・ひるみ付与アイテム等）に共通して作用する。

## 実装対象イベント
- `Event.ON_MODIFY_SECONDARY_CHANCE`（`docs/spec/turn.md` 未掲載。ダメージ計算・
  技効果処理の内部で `battle.resolve_secondary_chance()`（`core/battle.py`）から
  発火する既存イベント。同イベントを利用する既存特性 ちからずく・りんぷん、
  既存アイテム おんみつマント と同じ priority 未指定＝デフォルト値を使う）
  - `subject_spec="attacker:self"`: てんのめぐみのポケモンが攻撃技を使用するとき、
    渡された発動確率 `value` を2倍にする（`min(1.0, value * 2)` で上限100%に丸める）

## 仕様要点（実装に反映）
- 追加効果の発動確率を一律2倍にする。50%以上の技は2倍計算後に100%へ丸められる
  （`min(1.0, value * 2)` で自動的に満たされる）。
- `resolve_secondary_chance()` を経由しない効果（本体効果・固定発動効果・
  なげつけるのきのみ効果等、ソース側で「resolve_secondary_chance は経由しない」
  と明記されている効果）には影響しない。ハンドラ側で個別に分岐する必要はなく、
  イベントを発火させない設計に委ねる。
- アイテム おうじゃのしるし/するどいキバ によるひるみ確率（10%）も
  `resolve_secondary_chance()` 経由のため2倍（20%）になる。ただし元々ひるみ効果を
  持つ技（`_INNATE_FLINCH_MOVES`）にはこれらのアイテム自体が効果を持たないため、
  てんのめぐみによる重ねがけも発生しない（アイテム側の仕様であり、てんのめぐみ側の
  実装で対応不要）。
- にじ (場の状態) との重複計算は、本プロジェクトの対象がシングルバトルであり
  にじ自体が未実装のため対象外（`docs/spec/abilities/てんのめぐみ.md` に一次情報として
  記録するのみ）。
- キョダイサイセイ/キョダイスイマの発動率は増えない → キョダイマックス自体が
  対象外のため実装不要。

## 実装フロー（実装済み）
1. `src/jpoke/handlers/ability.py` に `てんのめぐみ_boost_secondary_chance` を実装
   （`value` を2倍にし `min(1.0, ...)` で上限100%にする）。
2. `src/jpoke/data/ability.py` の `てんのめぐみ` に
   `Event.ON_MODIFY_SECONDARY_CHANCE`（`subject_spec="attacker:self"`）を登録する。

## テスト観点
- 追加効果確率が2倍になる（20%→40%の境界を乱数で確認、実装済み・テスト済み）。
- 発動確率50%以上の技は2倍計算後に100%に丸められる（チャージビーム70%を例に、
  本来外れる乱数値でも必ず発動することを確認、新規追加）。
- アイテム おうじゃのしるし/するどいキバ のひるみ確率が2倍(20%)になる
  （`tests/items/test_item_ha.py` で確認済み）。
- ちからずく・りんぷん・おんみつマント等、同じ `ON_MODIFY_SECONDARY_CHANCE` を
  使う既存ハンドラとの組み合わせで既存テストが壊れないこと。
