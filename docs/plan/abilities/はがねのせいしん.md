# 特性実装計画: はがねのせいしん

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/はがねのせいしん.md`
- `docs/spec/abilities/はがねつかい.md`（同型の特性仕様を参照）

## 実装ゴール

- はがねタイプの攻撃技の威力を1.5倍にしてダメージを計算する。
- はがねつかい（`Event.ON_CALC_ATK_MODIFIER`）とは補正ステージが異なる（本特性は技の
  威力そのものを補正する）ため、両立する。
- 自身の使う技にも効果が及ぶ。
- はめつのねがい（はがねタイプの技）の威力も上がる。はめつのねがいは使用ターン（溜め処理、
  `ON_MOVE_CHARGE`）の時点で `battle.roll_damage` によりダメージを確定させており、その際の
  攻撃者は本特性の所持者自身であるため、`Event.ON_CALC_POWER_MODIFIER` を通常どおり
  `attacker:self` で実装すれば追加対応なしに反映される。
- タイプが変わる技・技のタイプを変える技によって使用技のタイプが変わった場合、変更後の
  タイプがはがねタイプであれば効果がある（`ctx.move.type` は変更後のタイプを参照するため、
  追加対応は不要）。

## シングルバトル限定範囲の扱い

- 本特性の一次情報には「味方の場にこの特性を持つポケモンが複数匹いると、その分効果が
  累積する」という記載があるが、本プロジェクトの対象はポケモンチャンピオンズ
  **シングルバトルのみ**であり、自分の場には常に1匹しかポケモンが存在しない。そのため
  「味方全員（自分を含む）」は実質的に「自分自身」と等価になり、複数所持による累積effectは
  シングルバトルでは発生しえない（テスト対象外）。
- 同様の理由で、フレンドガードなど他の「味方」参照特性もこのプロジェクトでは自分自身への
  効果として扱われている（`docs/spec/abilities/フレンドガード.md` 等、既存実装なし）。

## 実装方針

- `Event.ON_CALC_POWER_MODIFIER`、`subject_spec="attacker:self"` で実装する（実装済み）。
- priority は turn.md に記載のない内部（ダメージ計算内）イベントのため、同一イベント・同一
  `subject_spec="attacker:self"` の既存ハンドラ（**ちからずく** の `ちからずく_boost`、
  **パンクロック** の音技威力補正など）に合わせ、`AbilityHandler` のデフォルト値
  `priority=100` を使用する（実装済み）。
- `ctx.move.type` が `"はがね"` のとき `apply_fixed_modifier(value, 6144)`（4096 基準で 1.5 倍）を
  適用する。共通ヘルパー `_modify_by_move_condition(move, value, modifier=6144, move_type="はがね")`
  を使う（はがねつかいとの重複実装を避ける）。

## テスト観点

1. はがねタイプの技を使ったとき `power_modifier` が 6144（1.5倍）になる。
2. はがねタイプ以外の技を使ったときは `power_modifier` が 4096 のまま変化しない。
3. はめつのねがい（はがねタイプ技）を使用したとき、溜め処理時点の `power_modifier` が
   6144（1.5倍）になる。
