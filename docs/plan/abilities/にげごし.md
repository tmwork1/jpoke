# にげごし 実装計画

## スコープ
- 優先度: H
- 仕様書: `docs/spec/abilities/にげごし.md`
- 実装の核: にげごしはグソクムシャ専用特性「ききかいひ」と完全に同一効果（コソクムシ専用、
  進化後はききかいひに変わるだけ）。HP が「1/2超 -> 1/2以下」に落ちたとき、既存の割り込み
  交代基盤 `Interrupt.EMERGENCY` を立てて処理する。ハンドラ関数 `ききかいひ_switch_on_half_hp`
  をききかいひと共用する。

## 事前確認済みの実装面
- `src/jpoke/core/turn_controller.py` に `Interrupt.EMERGENCY` の解決ポイントがあり、技実行後
  （`_run_action_phase`）とターン終了時（`_run_end_phase`）の各段で割り込み交代を処理する。
  順序は EJECTBUTTON → EMERGENCY → PIVOT → EJECTPACK(after_move) / EJECTPACK(turn_end) →
  FAINTED であり、仕様書の「だっしゅつボタンが先」「だっしゅつパックはにげごしの後」と一致する。
- `src/jpoke/core/switch_manager.py` の `run_interrupt_switch()` が汎用的に割り込み交代を処理する。
- 連続攻撃技については、各ヒットごとに `battle.modify_hp(..., reason="move_damage")` が呼ばれ
  `Event.ON_HP_CHANGED` が都度発火するが、`_trigger_emergency_switch()` はフラグを立てるだけで
  実際の交代実行はターンコントローラー側の割り込み解決ポイント（技の全ヒット終了後）まで遅延
  される。そのため「連続攻撃技はすべてのヒットが終わってから発動判定」という仕様は現状の
  アーキテクチャで自然に満たされる。
- **ききかいひとの共有関係**: `にげごし`・`ききかいひ` はどちらも `Event.ON_HP_CHANGED` /
  `subject_spec="target:self"` で同一の `ききかいひ_switch_on_half_hp` を登録する
  （`src/jpoke/data/ability.py`）。ききかいひ側の実装レビュー（`docs/review/abilities/ききかいひ.md`）
  で発見・修正された不整合はにげごしにもそのまま適用される。

## 実装方針
1. `src/jpoke/handlers/ability.py` の `ききかいひ_switch_on_half_hp` を、にげごしと `data/ability.py`
   側で共用登録する（`Event.ON_HP_CHANGED` / `subject_spec="target:self"`）。
2. 発動しない理由コードを `HPChangeReason` で判定する。
   - `self_attack`（こんらん自傷）
   - `pain_split`（いたみわけ。自身が使用した場合・相手から受けた場合の両方とも
     `いたみわけ_equalize_hp` が両者に同じ reason を付与するため区別不要）
   - `self_cost`（みがわり・はらだいこ・のろい(呪い)・ソウルビートなどの自己HP消費。
     マジックガードでも防げない性質の自己コストとして共通化されている）
3. 特性ちからずくの効果が発動した技のダメージについて: 第七世代からSVまではにげごし/
   ききかいひが発動しなかったが、Championsではこの制限が撤廃されている
   （ぎゃくじょう/いかりのこうらの仕様書に記載済みの修正内容と同様）。実装側はもともと
   ちからずくを判定していないため対応不要（レビューで確認済み。ききかいひ側で一時的に
   Gen7-SV仕様の判定ロジックが混入していたが、Champions仕様に合わせて削除済み）。
4. 交代先の選択・実行そのものは既存の `Interrupt.EMERGENCY` 解決導線に委ねる。

## 実装タスク（完了状況）
1. `src/jpoke/data/ability.py` の `にげごし`・`ききかいひ` 双方に `ききかいひ_switch_on_half_hp`
   を登録済み。
2. `src/jpoke/handlers/ability.py` に `_crossed_half_hp`（閾値判定の共通関数）と
   `_trigger_emergency_switch`（割り込み予約の共通関数）を実装済み。
3. `self_cost` を除外条件に含む（ききかいひ側レビューで修正済み。にげごしにも適用される）。
4. `はらだいこ_apply` / `みがわり_apply` / `ソウルビート_pay_hp_and_modify_attacker_stats`
   （`src/jpoke/handlers/move_status.py`）に `reason="self_cost"` を付与済み（ききかいひ側
   レビューで修正済み。にげごしにも適用される）。

## 判定ルール
### 発動
- 攻撃技でダメージを受け、最終的に HP が半分以下になった（連続技は全ヒット終了後）。
- どく・やけど・天候・しおづけ・バインド・設置技等の定期/間接ダメージで HP が半分以下になった。
- 特性・アイテムの効果によるダメージで HP が半分以下になった。
- ちからずくの効果が発動した技のダメージでも通常どおり発動する（Champions仕様）。
- かげふみ/ありじごく/じりょくの影響や、にげられない/バインド/ねをはる/フェアリーロック状態を
  無視して発動する（これらは通常交代コマンドの可否判定であり、`Interrupt.EMERGENCY` の強制交代は
  そもそも `get_available_switch_commands()` のトラップ判定を経由しないため、現状の実装で自然に
  満たされる）。

### 非発動
- ダメージ前からすでに HP が半分以下だった。
- 瀕死になった。
- 交代先（控え）がいない。
- こんらん自傷（`self_attack`）。
- いたみわけ（自身の使用・被弾のどちらも `pain_split`）。
- みがわり・はらだいこ・のろい(呪い)・ソウルビートの自己HP消費（`self_cost`）。
- 特性無効化中（かがくへんかガス・かたやぶり関連はフレームワーク側で処理されるため
  ハンドラ側で個別判定しない）。

## テスト計画
1. 単発攻撃で `1/2超 -> 1/2以下` になった直後に `Interrupt.EMERGENCY` が立ち、技後の割り込み
   交代で引っ込む。
2. HP がちょうど半分の状態からさらに減っても発動しない。
3. どく・やけど・すなあらし等のターン終了ダメージで条件を満たした場合に発動する。
4. こんらん自傷では発動しない。
5. 交代先不在では `Interrupt.EMERGENCY` が立たない。
