# 特性実装計画: てつのこぶし

更新日: 2026-07-12

## スコープ

- 仕様書: `docs/spec/abilities/てつのこぶし.md`
- 実装状態: 実装済み
- 実装方針: `ON_CALC_POWER_MODIFIER` で攻撃側として `ctx.move.has_flag("punch")` が True の技（パンチ技）に
  4915/4096倍の固定小数点補正を適用する。

## 実装ゴール（完了定義）

- パンチ技（`punch` フラグを持つ技）を使用したときの威力が1.2倍（正確には4915/4096倍）になる。
- パンチ技でない技（でんきショック等）には補正がかからない。
- アイテムパンチグローブの効果（1.1倍・4506/4096倍）とは累積し、パンチ技の威力は
  本来の1.32倍相当（4096 → てつのこぶし適用後4915 → パンチグローブ適用後5406）になる。
  補正の適用順序は `ON_CALC_POWER_MODIFIER` に登録されたハンドラの実行順に依存するが、
  1段階目の乗算が常に割り切れる（`4096 * X // 4096 == X`）ため、実行順に関わらず最終値は一致する。
- 直接攻撃の連続攻撃技であるパンチ技を特性ミイラ/さまようたましい/とれないにおいの
  ポケモンに使用した場合、1発目で特性が消されることで2発目以降の攻撃に効果が及ばなくなる。
  これはイベント駆動アーキテクチャにより、命中ごとに特性を再評価する既存の仕組みで
  自動的に実現されるため、本特性側で個別対応は不要。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_CALC_POWER_MODIFIER` — ダメージ計算時の威力補正イベント（`AttackContext`,
  `attacker:self`）。ターン全体のイベント一覧（`docs/spec/turn.md`）には掲載されていない
  ダメージ計算内部イベントのため、同種の威力補正特性（かたいツメ・がんじょうあご・
  パンクロック・メガランチャー・きれあじ）と同じ priority（`AbilityHandler` 既定値 100）を
  踏襲する。

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `てつのこぶし` に `Event.ON_CALC_POWER_MODIFIER` の handler を登録済み
- `src/jpoke/handlers/ability.py` — `てつのこぶし_modify_power` を実装済み
- `src/jpoke/data/moves/*.py` — 各パンチ技の `flags` に `"punch"` を設定済み
  （`アームハンマー`/`アイスハンマー`/`あんこくきょうだ`/`かみなりパンチ`/`きあいパンチ`/
  `グロウパンチ`/`コメットパンチ`/`ジェットパンチ`/`シャドーパンチ`/`すいりゅうれんだ`/
  `ドレインパンチ`/`ばくれつパンチ`/`バレットパンチ`/`ぶちかまし`/`ふんどのこぶし`/
  `ほのおのパンチ`/`マッハパンチ`/`メガトンパンチ`/`れいとうパンチ`）。
  なお `スカイアッパー`/`ダブルパンツァー`/`ピヨピヨパンチ`/`プラズマフィスト`/`れんぞくパンチ`
  は本レビュー時点で技自体が未実装のため対象外。技を実装する際に `"punch"` フラグの付与を
  忘れないこと。
- `src/jpoke/handlers/item.py` — `パンチグローブ_boost_punch_power` が同じ `"punch"` フラグを
  参照し、累積計算を成立させている（実装済み）
- `tests/abilities/test_ability_ta.py` — テスト実装済み

## 実装ステップ

1. 仕様書 `docs/spec/abilities/てつのこぶし.md` で発動条件・補正率・例外条件を確認する。
2. `src/jpoke/handlers/ability.py` の `てつのこぶし_modify_power(battle, ctx, value)` で
   `_modify_by_move_condition(ctx.move, value, modifier=4915, move_flag="punch")` を返す。
3. `src/jpoke/data/ability.py` の `てつのこぶし` に `Event.ON_CALC_POWER_MODIFIER` の
   `AbilityHandler(h.てつのこぶし_modify_power, subject_spec="attacker:self")` を登録する。
4. パンチ技には `flags` に `"punch"` を付与する（`MoveFlag` の定義は
   `src/jpoke/types/literals.py`）。
5. `tests/abilities/test_ability_ta.py` にテストを追加し、全テストが通ることを確認する。

## テスト観点

1. パンチ技（マッハパンチ等）を使用したとき威力補正倍率が4915/4096になる。
2. パンチ技でない技（でんきショック等）には補正がかからない（4096のまま）。
3. パンチグローブ所持時はパンチ技の威力がさらに1.1倍（4506/4096）乗算され、
   累積して4915/4096 × 4506/4096 相当（最終値5406）になる。
