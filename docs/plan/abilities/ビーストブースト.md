# 特性実装計画: ビーストブースト

更新日: 2026-06-05

## 仕様参照

- `docs/spec/abilities/ビーストブースト.md`

## 実装ゴール

- 攻撃技で他のポケモンをひんしにさせたとき、最も実数値が高い能力が1段階上がる。
- 同値の場合は こうげき → ぼうぎょ → とくこう → とくぼう → すばやさ の優先順。
- すでに最大ランクの能力が最高実数値でも、2番目の能力に移ることはない。

## 実装方針

- `Event.ON_MOVE_KO`、`subject_spec="attacker:self"` で実装。
- `mon.stats["A"], ["B"], ["C"], ["D"], ["S"]` の実数値を比較して最大値の能力を決定。
- Python の `max` は同値のとき最初の要素を返すため、順序付きタプルで優先順を表現。
- **じしんかじょう・くろのいななき** と同じ `ON_MOVE_KO` パターン。

## 変更候補

- `src/jpoke/data/ability.py` — `ビーストブースト` に handlers を登録する
- `src/jpoke/handlers/ability.py` — `ビーストブースト_on_ko` を実装する
- `tests/test_ability.py` — テストを追加する

## テスト観点

1. 倒したとき、最高実数値の能力が1段階上がる。
2. 同値の場合は優先順（A > B > C > D > S）で決まる。
3. 最高実数値の能力がすでに最大ランクなら発動しない。

## 疑似コード

```python
_BEAST_BOOST_ORDER: tuple[Stat, ...] = ("A", "B", "C", "D", "S")

def ビーストブースト_on_ko(battle, ctx, value):
    mon = ctx.attacker
    best_stat = max(_BEAST_BOOST_ORDER, key=lambda s: mon.stats[s])
    if battle.modify_stats(mon, {best_stat: +1}, source=mon):
        announce_ability_triggered(battle, ctx, value, mon=mon)
    return HandlerReturn(value=value)
```
