# 実装計画: たんじゅん

作成日: 2026-05-02
更新日: 2026-07-12（再レビュー、実装内容に合わせて全面改稿）

## 効果
自分のランク補正が変動するとき、変動量（プラスマイナスとも）が2倍になる（`docs/spec/abilities/たんじゅん.md` 参照）。

## イベント／ハンドラ

| イベント | priority | subject_spec | 関数名 |
|---------|---------|-------------|--------|
| `ON_BEFORE_MODIFY_STAT` | 未指定（デフォルト値）。`turn.md` 未掲載のイベント計算内部のため、同種の `あまのじゃく`（ランク変化量の符号を反転するハンドラ）に合わせてデフォルト値を採用 | `target:self` | `たんじゅん_double_stat` |

`ON_BEFORE_MODIFY_STAT` は `StatusManager.modify_stats()` が能力ランク変更のたびに発火するイベントで、
`value` は `{stat: 変化量}` の辞書（命中率・回避率を含む）。`subject_spec="target:self"` により、
**たんじゅんを持つポケモン自身が変化の対象（target）になるときのみ** 発動する
（たんじゅん所持者が相手の能力を下げる技を使った場合は `target` が相手になるため発動しない）。

急所ランク（`Pokemon.critical_rank`）は `modify_stats()` を経由しない別経路（専用セッター）で
変更されるため、`ON_BEFORE_MODIFY_STAT` は発火せず、たんじゅんの効果は自然に及ばない
（仕様書「特性の仕様」節: 急所ランクには効果が無い）。

## 実装

```python
def たんじゅん_double_stat(battle: Battle, ctx: EventContext, value: dict[str, int]) -> HandlerReturn:
    """たんじゅん特性: 能力変化量を2倍にする。"""
    return HandlerReturn(value={stat: delta * 2 for stat, delta in value.items()})
```

上限・下限（±6）へのクランプは `Pokemon.modify_stat()` 側（`modify_stats()` 呼び出し先）で
共通処理されるため、本ハンドラ内でクランプする必要はない。

## data/ability.py 登録

```python
"たんじゅん": AbilityData(
    flags={
        "mold_breaker_ignorable"
    },
    handlers={
        Event.ON_BEFORE_MODIFY_STAT: h.AbilityHandler(
            h.たんじゅん_double_stat,
            subject_spec="target:self",
        )
    }
),
```

`mold_breaker_ignorable` フラグにより、かたやぶり等でとくせいを無視する技を受けた場合は
たんじゅんが無効化され、通常通りランクが変化する（仕様書「特性の仕様」節）。

## 実装済みの補足対応（他ファイル）

以下は本特性に依存する別特性・アイテム・技側の実装で、たんじゅん単体のハンドラとは別に
それぞれの `subject_spec` / 実装内で「自分の能力変化かどうか」を判定して整合させている。
本計画では変更しないが、参照用に記載する。

- `くろいきり` / `クリアスモッグ` / `じこあんじ` / `パワースワップ` / `ガードスワップ` /
  `ハートスワップ` / `ひっくりかえす` / `バトンタッチ`: `modify_stats()` を経由しない、
  または対象外ロールで発動するため、たんじゅんの影響を受けない。
- `パワーシェア` / `パワートリック`: 実数値を直接書き換える処理でありランク変化ではないため、
  たんじゅんの影響を受けない（仕様書どおり）。
- シャドースチール（ランクリセット・吸収技）は第9世代（スカーレット・バイオレット）に
  存在しない技のため未実装。対象外。
- Z変化ワザ・ダイマックスわざ・マックスレイドバトルは第9世代（ポケモンチャンピオンズ対象）に
  存在しない要素のため未実装。対象外。
- 各種アイテム（シード系・プラスパワー系・ものまねハーブ等）・技（ちょうのまい・からをやぶる・
  ちからをすいとる等）は、たんじゅん所持者の `ON_BEFORE_MODIFY_STAT` 経由で自然に2倍になる。
  それぞれのテストは `tests/items/`, `tests/moves_status/`, `tests/moves_attack/` に存在する。

## 既知の未対応（対象外・低優先度）

- `わたげ`（`handlers/ability.py` の `わたげ_lower_spd_on_hit`）は `battle.modify_stats(attacker, ...)`
  を通常経路で呼ぶため、攻撃者がたんじゅんを持つ場合は2倍になってしまう。仕様書には
  「かたやぶり効果がある技で発動したわたげを受けた場合、たんじゅんの効果を無視してすばやさが
  1段階下がる」とあるが、これは「かたやぶり技で わたげ 自体の発動可否判定を無視した場合」の
  非常に限定的な edge case であり、既存実装は一般ケース（かたやぶりでない技でわたげが発動する
  場合）を正しく処理している。あまのじゃく（同型の特性）でも同じ理由で対応を見送っており、
  優先度が低いため本レビューでも対応を見送る。

## テストケース
- たんじゅん所持者が対象（`target`）となるランク変化は、上昇下降とも変化量が2倍になる
  （`tests/abilities/test_ability_ta.py::test_たんじゅん_能力変化量が2倍になる`）。
- かたやぶりの効果がある技を受けた場合はたんじゅんが無効化され、通常通り1段階のみ変化する
  （`tests/abilities/test_ability_ta.py::test_たんじゅん_かたやぶりで無効`）。
- アイテム・戦闘道具・追加効果技との相互作用は `tests/items/`, `tests/moves_status/`,
  `tests/moves_attack/` に多数のテストが存在する。
