# 特性実装計画: ハドロンエンジン

更新日: 2026-07-12

## スコープ

- 仕様書: `docs/spec/abilities/ハドロンエンジン.md`
- 実装状態: 実装済み（レビューにより実装内容の妥当性を確認）
- 実装方針:
  - `Event.ON_SWITCH_IN` と `Event.ON_ABILITY_ENABLED` で
    `TerrainManager.apply` を呼び出し、エレキフィールドを5ターン展開する
    （既存の `_activate_terrain` ヘルパーを利用。エレキメイカーと共通）
  - `Event.ON_CALC_ATK_MODIFIER` で、エレキフィールド下・特殊技のときのみ
    攻撃側の特攻補正を5461/4096倍にする

## 実装ゴール（完了定義）

- 場に出たとき、フィールドをエレキフィールド状態にする（5ターン、
  グランドコート所持時は8ターン。グランドコートの延長は
  `Event.ON_MODIFY_DURATION` 側で汎用的に処理されるためハドロンエンジン側の
  実装は不要）
- 既にエレキフィールド状態のときは発動せず、継続ターンも書き換わらない
  （`TerrainManager.apply` が同名フィールド発動時は `False` を返し
  何もしないため、個別実装は不要）
- 場がエレキフィールド状態の間、特殊技を使用したときのみ特攻が
  5461/4096倍になる（物理技・でんきタイプ以外の特殊技を問わず、
  カテゴリが `special` であれば発動する）
- 特性が無効化されている間（かがくへんかガス・とくせいなし等）は
  `ON_CALC_ATK_MODIFIER` ハンドラ自体が発火しないため、特攻補正は
  自動的に解除される。値を保存・復元するロジックは不要
  （ランク変化ではなくダメージ計算時点の乗算補正のため）
- ミライドン以外のポケモンがこの特性を入手した場合も同様に発動する
  （`uncopyable` フラグを付与しない。かがくへんかガスの無効化・
  かたやぶり系の無視判定は共通処理に委譲するため、ここでは扱わない）
- テラスタル中のテラバーストの分類決定（`テラバースト_modify_move_category`）
  は `ranked_stats`（ランク補正のみ）を参照しており、
  `ON_CALC_ATK_MODIFIER` の補正は含まれないため、仕様の
  「分類決定にはハドロンエンジンの効果が考慮されない」を自動的に満たす。
  ハドロンエンジン側の追加実装は不要
- 既存テストを壊さない

## 実装対象イベント

- `Event.ON_SWITCH_IN` — 場に出たときにフィールドを展開する
  （`docs/spec/turn.md` ON_SWITCH_IN: priority 100「ハドロンエンジン（特性）」）
- `Event.ON_ABILITY_ENABLED` — 特性がハドロンエンジンに変化した瞬間に発動する。
  `turn.md` に未掲載のイベントだが、同種の `あめふらし`/`ひでり`/`すなおこし`/
  `ゆきふらし`/`エレキメイカー` がいずれも `ON_SWITCH_IN` と同じ関数を
  priority省略（デフォルト100）で登録しているため、それに合わせる
- `Event.ON_CALC_ATK_MODIFIER` — ダメージ計算内部イベント。`turn.md` に
  未掲載だが、同種の特攻補正特性（`サンパワー`, `ひひいろのこどう` の
  こうげき版など）がいずれも priority省略（デフォルト）で登録しているため、
  それに合わせる

## テスト観点

1. 場に出たときにエレキフィールドが5ターン展開される
   （`ability.revealed` も確認）。
2. エレキフィールド下で特殊技を使うと特攻補正が5461/4096倍になる。

## リスクと対策

- リスク: `ON_ABILITY_ENABLED` はメガシンカ・スキルスワップ・トレース・
  かがくへんかガス解除など複数の経路で発火するため、意図しない二重発動が
  起きないか確認が必要。
  - 対策: `TerrainManager.apply` は同名フィールドが既に有効な場合は何もせず
    `False` を返す実装になっており、二重発動しても副作用は起きない。
- リスク: `ON_CALC_ATK_MODIFIER` がテラバーストの分類決定より後に評価される
  実装順序に依存している。
  - 対策: `テラバースト_modify_move_category` は `ranked_stats`
    （ランク補正込みだが `ON_CALC_ATK_MODIFIER` 適用前の値）を参照するため
    整合している。ダメージ計算パイプラインの順序が変わった場合は
    再確認が必要。
