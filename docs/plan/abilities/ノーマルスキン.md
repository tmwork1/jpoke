# 特性実装計画: ノーマルスキン

作成日: 2026-05-03
更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/ノーマルスキン.md`

## 効果

使用する技を（ステラタイプを除き）すべてノーマルタイプに変換する。
元がノーマルタイプ以外だった技は威力が4915/4096倍になる。
タイプ変換後はノーマルタイプ一致補正が適用される（自身がノーマルタイプの場合）。

他のスキン系特性（スカイスキン・ドラゴンスキン・フェアリースキン・フリーズスキン）は
「ノーマルタイプの技を特定タイプに変換する」のに対し、ノーマルスキンは唯一
「あらゆるタイプの技をノーマルタイプに変換する」向きが逆の特性である。そのため
共通ヘルパー `_skin_modify_move_type` / `_skin_boost_power`（from_type/trigger_type を
1つだけ指定する形）は使わず、専用ロジックを実装する。

## イベント／ハンドラ

`ON_MODIFY_MOVE_TYPE` / `ON_CALC_POWER_MODIFIER` はいずれも `docs/spec/turn.md` に
掲載されていないイベント（ダメージ計算内部）のため、既存のスキン系特性
（スカイスキン・ドラゴンスキン・フェアリースキン・フリーズスキン）のハンドラと
同じ既定値（100）に揃える。いずれも攻撃フェーズ（`AttackContext`）のイベントのため、
`subject_spec` は CLAUDE.md の約束事に従い `attacker:self` を使う。

| イベント | priority | subject_spec | 関数名 |
|---------|---------|-------------|--------|
| `ON_MODIFY_MOVE_TYPE` | 100 (既定) | `attacker:self` | `ノーマルスキン_modify_move_type` |
| `ON_CALC_POWER_MODIFIER` | 100 (既定) | `attacker:self` | `ノーマルスキン_boost_power` |

## 実装方針

- `ノーマルスキン_modify_move_type`: `ON_MODIFY_MOVE_TYPE` チェーンの現在値（`value`）が
  `""`（タイプなし）/ `ノーマル` / `ステラ` のいずれでもなければ `ノーマル` に変換する。
  ステラタイプ技（テラバースト/テラクラスターのステラタイプにテラスタルした場合の技タイプ）を
  除外する点は他のスキン系特性と共通の実装済みパターンを踏襲する。
  `""` の除外は、わるあがき（`type=""`）がこの特性で誤ってノーマルタイプに変換されない
  ようにするために必須（`types/literals.py` の `Type` は `""` を「タイプなし」として含む）。
- `ノーマルスキン_boost_power`: `ctx.move.data.type`（技本来の静的タイプ）が
  `""` / `ノーマル` / `ステラ` のいずれでもない場合のみ威力を4915/4096倍にする。静的タイプを見るのは
  他のスキン系特性の `_skin_boost_power` と同じ設計判断で、ウェザーボール等タイプが変わる技の
  「元の技タイプ」を基準に補正有無を判定するため。こちらも `""` を除外しないと、わるあがきの
  威力まで4915/4096倍されてしまう。
- わるあがきに対して特性が発動しない（仕様書「わるあがきを使用したときにこの特性は発動しない」）
  という要件は、上記2関数内で `type=""` を除外することで満たす。他のスキン系特性
  （ノーマルタイプの技のみを変換先タイプに変換する向き）は `value == "ノーマル"` 判定が
  `""` に一致しないため構造的に安全だが、ノーマルスキンは「ノーマル/ステラ以外を全て変換する」
  向きのため `""` を明示的に除外しないとわるあがきまで変換されてしまう
  （レビューで発見・修正: 修正前は `わるあがき` がノーマルタイプ・威力4915/4096倍になるバグがあった）。
- でんじはがノーマルタイプに変換された場合の相性判定（じめん有効・ゴースト無効）は、
  `でんじは_can_apply`（`handlers/move_status.py`）が `battle.damage_calculator.calc_def_type_modifier(ctx)`
  を実行時点の技タイプ（＝本特性で変換済みの値）で評価するため、特性側で個別分岐を持たなくても
  自然に成立する。

## テスト観点

1. ノーマル以外の技がノーマルタイプに変換される。
2. 変換された技の威力が4915/4096倍になる。
3. 元々ノーマルタイプの技には威力補正がかからない（タイプも威力も変化しない）。
4. わるあがきはタイプ・威力とも変換対象外になる（`type=""` 除外の確認）。
