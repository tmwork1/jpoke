# ぶきよう 実装計画

## スコープ
- 優先度: H
- 仕様書: `docs/spec/abilities/ぶきよう.md`
- 実装方針: `かがくへんかガス` と同様に `Effect._disabled_reasons` ベースの汎用無効化理由機構
  （`model/effect.py` の `add_disable_reason` / `remove_disable_reason` / `enabled_ignoring`）を
  利用する。`ぶきよう` 特性ハンドラは `battle.item_manager.add_disabled_reason` /
  `remove_disabled_reason` を呼び、所持者の `item._disabled_reasons` に `"ぶきよう"` を
  追加・削除するだけでよい。道具ハンドラ自体は `ItemHandler(source="item")` のため、
  `EventManager._check_handler_validity` が発火のたびに `subject.item.enabled_ignoring(...)` を
  判定しており、道具のイベントハンドラ登録状態を明示的に付け替える必要はない。

## 現状整理
- `src/jpoke/data/ability.py` の `ぶきよう` は次の3イベントにハンドラを登録済み:
  - `Event.ON_SWITCH_IN`: `h.ぶきよう_disable_item`（登場時に無効化理由を追加）
  - `Event.ON_ABILITY_ENABLED`: `h.ぶきよう_disable_item`（スキルスワップ等で特性を得た直後）
  - `Event.ON_ABILITY_DISABLED`: `h.ぶきよう_enable_item`（かがくへんかガス等で特性が無効化された直後）
- `src/jpoke/handlers/ability.py` の実装:
  - `ぶきよう_disable_item`: `battle.item_manager.add_disabled_reason(ctx.source, "ぶきよう")`
  - `ぶきよう_enable_item`: `battle.item_manager.remove_disabled_reason(ctx.source, "ぶきよう")`
- `src/jpoke/core/item_manager.py` の `add_disabled_reason` / `remove_disabled_reason` は
  `mon.item.enabled` の変化時のみ `Event.ON_ITEM_DISABLED` / `Event.ON_ITEM_ENABLED` を発火する。
- `src/jpoke/core/event_manager.py` の `_check_handler_validity` が `source="item"` の全ハンドラに対し
  `subject.item.enabled_ignoring(rh.handler.ignored_disable_reasons)` を判定するため、
  `ぶきよう` が有効な間は道具由来の全イベントハンドラが自動的に不発になる。
  （個別の道具ハンドラを `ぶきよう` 用に書き換える必要はない）
- 例外的に道具効果を通したい場合（例: `とくせいガード`）は、当該道具ハンドラに
  `ignored_disable_reasons=frozenset({"ぶきよう"})` を指定する。

## 仕様要点（実装に反映）
- `ぶきよう` が有効な間、所持道具のバトル効果は不発（`can_change_item` 等の道具変更可否判定
  自体には影響しない。トリック・すりかえ・リサイクル・ギフトパスは通常どおり使用可能）。
- `とくせいガード` の効果（`ぶきよう` 自身が書き換え・無効化されるのを防ぐ）は例外的に発動する
  （`ignored_disable_reasons=frozenset({"ぶきよう"})` で実現済み）。
- `かがくへんかガス` 等で `ぶきよう` が無効化されている間は、道具効果が通常どおり発動する。
- `ぶきよう` 復帰時は即座に道具効果が再び止まる（`ON_ABILITY_ENABLED` で毎回無効化理由を
  再付与するため、取りこぼしは起きない）。
- `くろいてっきゅう` のグラウンド化・すばやさ半減、`だいこんごうだま` 等のフォルムチェンジ・
  威力補正効果、`ふうせん` の浮遊効果など、`ItemHandler` として実装された効果は上記の汎用機構で
  自動的に無効化される。

## 未対応・要確認
- 第七世代以降の仕様「こだわり系アイテムで技を固定した“後”に `ぶきよう` を得た場合、固定技以外を
  選ぼうとするとPP消費なしで技が失敗する」は未実装（現状は `こだわり` 揮発状態の
  `restrict_commands` がコマンド選択自体を固定技のみに絞るままで、`ぶきよう` の影響を受けない）。
  スキルスワップ等で既にこだわり済みのポケモンへ `ぶきよう` を付与するという極めて限定的な状況でのみ
  顕在化する差異のため、優先度は低いものとして未対応のまま残す。

## 実装フロー（実施済み）
1. `ぶきよう` の `Event.ON_SWITCH_IN` / `Event.ON_ABILITY_ENABLED` / `Event.ON_ABILITY_DISABLED` に
   ハンドラを登録する。
2. `ぶきよう_disable_item` / `ぶきよう_enable_item` を `handlers/ability.py` に実装し、
   `item_manager.add_disabled_reason` / `remove_disabled_reason` を呼ぶ。
3. 道具効果側で例外的にぶきようの影響を受けたくないハンドラ（`とくせいガード` 等）には
   `ignored_disable_reasons=frozenset({"ぶきよう"})` を個別に指定する。

## テスト観点
- 発動ケース: `ぶきよう` + `たべのこし` でHP自動回復が発生しない。
- 発動ケース: `ぶきよう` + `くろいてっきゅう` で浮遊無効化・すばやさ半減がともに無視される。
- 発動ケース: `ぶきよう` + `だいこんごうだま` を持つディアルガがオリジンフォルムにならない。
- 発動ケース: `ぶきよう` + `ふうせん` でじめん技が命中する。
- 発動ケース: `ぶきよう` + `ねらいのまと` で無効化される（通常どおりタイプ相性で無効化）。
- 発動ケース: `ぶきよう` + `いのちのたま` を持つ攻撃側が `なげつける` を使用すると失敗する
  （第五世代以降の仕様）。
- 例外ケース: `ぶきよう` + `とくせいガード` は効果を発動し、特性の上書き・無効化を防ぐ。
- 例外ケース: `ぶきよう` でも `ポルターガイスト` は成功する（アイテムの所持自体は判定対象）。
- 例外ケース: `おみとおし` は `ぶきよう` で無効化中のアイテムも公開できる。
- 非発動ケース: `かがくへんかガス` 登場中は `ぶきよう` 持ちの道具が通常どおり発動する。
