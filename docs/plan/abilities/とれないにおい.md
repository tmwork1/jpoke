# 特性実装計画: とれないにおい

更新日: 2026-07-12

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/とれないにおい.md`
- 実装状態: 実装済み
- 実装方針: ミイラと全く同じロジックで実装する。`ON_DAMAGE_HIT` で直接攻撃を受けたとき攻撃者の特性を `とれないにおい` に書き換える。ミイラと共通の内部ヘルパー `_overwrite_ability_on_contact` を使う。

## 実装ゴール（完了定義）

- 直接攻撃でダメージを受けたとき、攻撃者の特性がとれないにおいに変わる。
- ぼうごパット持ちの攻撃者には発動しない。
- ダメージ 0（みがわり・まもるで防いだ）のときは発動しない。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_DAMAGE_HIT` — 直接攻撃でダメージを受けたとき、攻撃者の特性を書き換える

## Priority

| イベント | Priority | 根拠 |
|---|---|---|
| ON_DAMAGE | 20 | docs/spec/turn.md「てつのトゲ等: 直接攻撃を受けた」 |

turn.md にとれないにおいの直接掲載はないが、同カテゴリ（直接攻撃を受けたときに発動する防御側特性）である
「てつのトゲ等: 直接攻撃を受けた」（Priority 20）に該当するため Priority 20 とする。ミイラと同ロジックであり、
実装も `Event.ON_DAMAGE_HIT` に `priority=20` で登録済み。

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `とれないにおい` に handler を登録する
- `src/jpoke/handlers/ability.py` — `とれないにおい_overwrite_attacker_ability` を実装する（`_overwrite_ability_on_contact` 共通ヘルパーを使用し、ミイラの実装を参考にする）
- `tests/abilities/test_ability_ta.py` — テストを追加する

## 実装ステップ

1. 仕様書 `docs/spec/abilities/とれないにおい.md`（ミイラとの差異）を確認する。
2. `src/jpoke/handlers/ability.py` の `ミイラ_overwrite_attacker_ability` の実装を参考に
   `とれないにおい_overwrite_attacker_ability` を実装する（共通ヘルパー `_overwrite_ability_on_contact` を
   `new_ability="とれないにおい"` で呼び出す）。
3. `src/jpoke/data/ability.py` の `とれないにおい` に `Event.ON_DAMAGE_HIT` の handler を
   `priority=20` で登録する。
4. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. 直接攻撃でダメージを受けたとき、攻撃者の特性がとれないにおいに変わる。
2. みがわりで技を防いだとき（ダメージなし）は発動しない。
3. ぼうごパットで接触を防いだ攻撃者には発動しない。
4. 上書きできない特性（uncopyable）の攻撃者には発動しない。
