# 特性実装計画: マルチタイプ

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/マルチタイプ.md`

## 実装ゴール

- アルセウスが持っているプレートに応じてタイプ（＝フォルム相当）を変更する。
- プレートの奪取・交換（自分の道具・相手の道具どちらが対象でも）を防ぐ。
- かがくへんかガス・かたやぶりに影響されない。
- コピー・上書き不可（uncopyable / protected）。

## 実装方針

- `docs/spec/turn.md` にマルチタイプ個別の記載はないため、同種のタイプ変更系特性
  （ARシステム）と同じ Priority（デフォルト100）を踏襲する。
- `Pokemon.ability_override_type` に登場時（`Event.ON_SWITCH_IN`）でタイプを設定し、
  `Pokemon.types` プロパティで反映する（ARシステムと共通の `_apply_multitype()` ヘルパーを使用）。
  フォルム名・見た目の変化は本シミュレーターでは対象外（バトルに影響するのはタイプのみ）。
- 対応表は `src/jpoke/data/signature_items.py` の `PLATE_TO_TYPE` に集約する
  （ARシステムの `MEMORY_TO_TYPE` と対の構成）。
- 道具保護は `Event.ON_CHECK_ITEM_CHANGE` ハンドラ（`マルチタイプ_block_item_change`）で
  `ItemManager.can_change_item()` を通して判定する。
  - 自分の持つプレートの奪取・交換を防ぐ（`_block_item_change()` 共通ヘルパー）。
  - トリック/すりかえ等の**交換型**の道具変更では、相手がプレートを持っている場合も
    交換自体が失敗する（`EventContext.is_exchange` フラグ経由で判定。はたきおとす等の
    一方的な除去では `is_exchange=False` のため影響しない）。
- `uncopyable` / `protected` / `gas_proof` フラグで、コピー不可・上書き不可・
  かがくへんかガス非対象を表現する（`mold_breaker_ignorable` は付与しないため、
  かたやぶりの効果を受けない）。
- さばきのつぶてのタイプ・威力補正は特性ではなく技側（`さばきのつぶて_modify_move_type`）で
  使用者の所持アイテムを直接参照して決定するため、本特性の実装スコープには含めない
  （アルセウス以外の変身先でも同じロジックが動く設計）。

## 変更候補

- `src/jpoke/model/pokemon.py`（`ability_override_type` / `types`）
- `src/jpoke/data/ability.py`
- `src/jpoke/handlers/ability.py`
- `src/jpoke/data/signature_items.py`（`PLATE_TO_TYPE`）
- `src/jpoke/core/item_manager.py`（`can_change_item` の `is_exchange` フラグ）
- `src/jpoke/core/context.py`（`EventContext.is_exchange`）
- `src/jpoke/data/item.py`（17種のプレートアイテム登録。各プレートの対応タイプ技威力1.2倍補正
  ハンドラは `src/jpoke/handlers/item.py` に実装し、`せいれいプレート` と同じパターンを踏襲する）
- `tests/abilities/test_ability_ma.py`
- `tests/items/test_item_ta.py`（プレートのタイプ強化効果テスト）

## テスト観点

1. 17種のプレートそれぞれでタイプが対応するタイプに変わる。
2. プレートなしではタイプ変更なし。
3. 自分の持つプレートの奪取・交換は防がれる（はたきおとす等）。
4. プレートを持っていなければ通常の道具変更は防がれない。
5. 交換型の道具変更（トリック/すりかえ相当）は、相手がプレートを持つ場合も失敗する。
6. かがくへんかガスで無効化されない（`tests/abilities/test_gas.py`）。

## 保留・論点

- へんしん（Transform）は `docs/progress/move.md` で実装保留のため、
  「へんしん先がマルチタイプをコピーする」仕様、および「アルセウスがへんしんした
  ポケモンに対してはみずびたし/まほうのこなでタイプ変更できる」仕様は未実装
  （へんしん実装時に対応する）。
- さばきのつぶてを同時実装するか別タスク分離するかは着手時に確定する方針だったが、
  技側は既に `さばきのつぶて_modify_move_type`（`handlers/move_attack.py`）として
  実装済みであり、本特性のハンドラには依存していない。
