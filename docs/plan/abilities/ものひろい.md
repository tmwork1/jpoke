# 特性実装計画: ものひろい

更新日: 2026-06-03

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/ものひろい.md`
- 実装状態: 未実装
- 実装方針: `ON_TURN_END` でそのターン消費された道具を確認し、もちものなしのものひろい所持者が持ちものを取得する。きのみ実装後に完全対応可能。

## 実装ゴール（完了定義）

- ターン終了時、自分がアイテムを持っていないとき、そのターン相手が消費した道具を拾う。
- 自分が消費した道具は拾わない（SVシリーズは野生戦のみ可だが、シミュレーション環境では除外）。
- バッグから使用した道具は対象外。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_TURN_END` — ターン終了時に消費アイテムを確認して取得する

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `ものひろい` に handler を登録する
- `src/jpoke/handlers/ability.py` — `ものひろい_on_turn_end` を実装する
- `tests/test_ability.py` — テストを追加する

## 実装ステップ

1. バトルシステムが「そのターン消費されたアイテム」をどこで管理しているか確認する（`battle.turn_consumed_items` など）。
2. `src/jpoke/handlers/ability.py` に `ものひろい_on_turn_end` を実装する（アイテムなし かつ そのターン相手が消費した道具があれば取得）。
3. `src/jpoke/data/ability.py` の `ものひろい` に handler を登録する。
4. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. 相手がきのみ等のアイテムを消費したターン終了時に、ものひろい所持者（アイテムなし）がそのアイテムを取得する。
2. ものひろい所持者がすでにアイテムを持っている場合は拾わない。
3. そのターン誰もアイテムを消費しなかった場合は何も拾わない。

## 検証コマンド

```powershell
python -m pytest tests/test_ability.py -k "ものひろい" -v
```
