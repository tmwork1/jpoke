## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/ものひろい.md`
- 実装状態: 実装済み
- 実装方針: `ON_TURN_END` でそのターン消費された道具を確認し、もちものなしのものひろい所持者が持ちものを取得する。

## 実装ゴール（完了定義）

- ターン終了時、自分がアイテムを持っていないとき、そのターン中に相手が消費した道具を拾う。
  「そのターン」限定であることを `Pokemon.last_lost_item_turn`（`battle.turn` との一致判定）で保証する。
- 自分が消費した道具は拾わない（SVシリーズは野生戦のみ可だが、シミュレーション環境では除外）。
- バッグから使用した道具・どろぼう等で場に存在し続ける形で失った道具（`track_loss=False`）は対象外。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_TURN_END` — ターン終了時に消費アイテムを確認して取得する
  （priority=150。`docs/spec/turn.md` の ON_TURN_END priority表に基づき、
  どく・やけど等のダメージやねむけ等より後、しゅうかく・たまひろいと同順位で発動する）

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `ものひろい` に handler を登録する
- `src/jpoke/handlers/ability.py` — `ものひろい_pickup_foe_item` を実装する
- `src/jpoke/model/pokemon.py` — `last_lost_item_turn` プロパティを追加する（`last_lost_item_name` が

- `src/jpoke/core/item_manager.py` — `_change_item` / `force_trigger_berry` で
  `last_lost_item_turn` を `last_lost_item_name` と同期して更新・復元する
- `tests/abilities/test_ability_ma.py` — テストを追加する

## 実装ステップ

1. バトルシステムが「そのターン消費されたアイテム」をどこで管理しているか確認する
   （`Pokemon.last_lost_item_name` / `last_lost_item_turn`、`ItemManager._change_item`）。
2. `src/jpoke/handlers/ability.py` に `ものひろい_pickup_foe_item` を実装する（アイテムなし かつ
   相手の `last_lost_item_turn == battle.turn` かつ消費した道具があれば取得）。
3. `src/jpoke/data/ability.py` の `ものひろい` に handler を登録する。
4. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. 相手がきのみ等のアイテムを消費したターン終了時に、ものひろい所持者（アイテムなし）がそのアイテムを取得する。
2. ものひろい所持者がすでにアイテムを持っている場合は拾わない。
3. そのターン誰もアイテムを消費しなかった場合は何も拾わない。
4. 相手が過去のターンに消費した道具は、そのターン以降新たに何も消費していなければ拾わない
   （`last_lost_item_turn` によるターン限定判定）。
5. どろぼう相当（`track_loss=False`）で場に存在し続ける形で失われた道具は拾わない。
6. どく等の継続ダメージでHP閾値ベースのきのみ（オボンのみ等）が同ターンの ON_TURN_END 処理内で
   消費された場合も、priority=150（ダメージ処理の priority=90/100 より後）で発動するため
   同ターンのうちに拾える。
