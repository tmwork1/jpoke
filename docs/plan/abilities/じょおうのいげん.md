# 特性実装計画: じょおうのいげん

更新日: 2026-06-05

## 仕様参照

- `docs/spec/abilities/じょおうのいげん.md`

## 実装ゴール

- 優先度 +1 以上の技（いたずらごころで昇格した技も含む）を無効化する。
- かたやぶり系の技には無効化されない。
- 優先度 0 以下の技は通常通り通る。

## 実装方針

- `Event.ON_TRY_MOVE_1`、`subject_spec="defender:self"`、priority=40 で実装。
- priority は `turn.md` ON_TRY_MOVE_1 priority 40「じょおうのいげん: 優先度高い技失敗」に従う。
- `battle.speed_calculator.calc_move_priority(ctx.attacker, ctx.move)` で実効優先度を取得。
- **テイルアーマー・ビビッドボディ** と同一効果。3特性でハンドラ関数を共有する。

## 変更候補

- `src/jpoke/data/ability.py` — `じょおうのいげん` に ON_TRY_MOVE_1 handler を登録する
- `src/jpoke/handlers/ability.py` — `じょおうのいげん_block_priority` を実装する
- `tests/test_ability.py` — テストを追加する

## テスト観点

1. でんこうせっか等の優先度 +1 技 → 失敗する。
2. いたずらごころで優先度 +1 になった変化技 → 失敗する。
3. 優先度 0 の技 → 通常通り通る。
4. かたやぶり系の技 → 特性を無視して通る。

```python
def じょおうのいげん_block_priority(battle, ctx, value):
    effective_priority = battle.speed_calculator.calc_move_priority(
        ctx.attacker, ctx.move
    )
    if effective_priority <= 0:
        return HandlerReturn(value=value)
    announce_ability_triggered(battle, ctx, value, mon=ctx.defender)
    battle.add_event_log(ctx.attacker, LogCode.MOVE_FAILED,
                         payload={"reason": "じょおうのいげん"})
    return HandlerReturn(value=False, stop_event=True)
```
