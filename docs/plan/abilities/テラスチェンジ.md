# 特性実装計画: テラスチェンジ

更新日: 2026-07-12

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/テラスチェンジ.md`
- 実装状態: 実装済み（flags=[uncopyable, protected, gas_proof] + `ON_SWITCH_IN` priority=10 ハンドラ登録済み）
- 実装方針: `ON_SWITCH_IN` で最速（他の特性より前）にテラスタルフォルムへフォルムチェンジする。かがくへんかガスが発動中でも発動するが、フォルムチェンジ後の特性（テラスシェル等）はガスで無効化される。

## 実装ゴール（完了定義）

- 登場時に、他の特性より先にテラスタルフォルムへフォルムチェンジする。
- かがくへんかガスが発動中でも発動する。
- 設置技のダメージはフォルムチェンジ後の HP を基準に計算される。
- この特性は上書きできない。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_SWITCH_IN` — 登場時に最速（priority を高く設定）でテラスタルフォルムへフォルムチェンジする

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `テラスチェンジ` に handler を登録する
- `src/jpoke/handlers/ability.py` — `テラスチェンジ_form_change_on_entry` を実装する
- `tests/abilities/test_ability_ta.py` — テストを追加する

## 実装ステップ

1. 仕様書 `docs/spec/abilities/テラスチェンジ.md` で「他の特性より先に発動」の優先度（素早さ実数値順）を確認する。
   （単体対戦では自パーティにテラパゴスは1体のみのため、複数体同時発動時の素早さ順ロジックは対象外）
2. `テラスチェンジ_form_change_on_entry` を実装する（対象がテラパゴス(ノーマル)のときのみ `mon.set_form(TERAPAGOS_TERASTAL)` を呼ぶ）。
3. `src/jpoke/data/ability.py` の `テラスチェンジ` に `ON_SWITCH_IN` handler を登録する（priority=10: turn.md L29、かがくへんかガス priority=20 より前）。
4. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. 登場時に自動的にテラスタルフォルムになる。
2. 既にかがくへんかガスが発動している場に繰り出したときも、テラスチェンジ自体（フォルムチェンジ）は発動する。
3. フォルムチェンジ後、テラスシェル/ゼロフォーミングなどのフォルム後の特性はガスで無効化される（ゼロフォーミング側のテストで担保）。
4. テラスチェンジ自体はかがくへんかガスで無効化されない（`gas_proof` フラグ、`tests/abilities/test_gas.py` でパラメタライズ確認済み）。
