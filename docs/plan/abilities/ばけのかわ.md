# 特性実装計画: ばけのかわ

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/ばけのかわ.md`

## 実装ゴール

- 「ばけたすがた」のときに相手の攻撃技（こんらん時の自分への攻撃を含む）を受けると、
  そのヒットのダメージを0にし、代わりに最大HPの1/8（小数点以下切り捨て、最低1）を
  消費して「ばれたすがた」に変わる（＝以後このバトル中は再発動しない）。
- 連続攻撃技は1発目のみ防ぎ、2発目以降は通常ダメージを受ける。
- かたやぶり系の効果を持つ技に対しては発動しない（ばけたすがたのままダメージを受ける）。
- 一度発動すると、交代・ひんし後の復活（Champions/第九世代基準）に関わらず、
  バトルが終わるまで再発動しない。
- スキルスワップ/なりきり/なかまづくり/なやみのタネ/シンプルビーム/いえき/うつしえ・
  トレース/ミイラ/レシーバー/かがくのちから/さまようたましい/とれないにおい・
  かがくへんかガスによる無効化を防ぐ（＝既存の `uncopyable` / `protected` / `gas_proof`
  フラグ基盤の挙動をそのまま利用する）。

## 実装方針

- `Event.ON_MODIFY_MOVE_DAMAGE`（`AttackContext`, `subject_spec="defender:self"`）で、
  ダメージを0に書き換え、`battle.add_ability_disabled_reason(ctx.defender, "consumed")`
  でこのバトル中の再発動を止め、`battle.modify_hp(ctx.defender, r=-1/8)` で変身解除
  ダメージ（最大HPの1/8、切り捨て）を与える。
  - `ばれたすがた` はミミッキュの図鑑データ上ステータス・タイプが同一のため
    （`src/jpoke/data/ps-champ-ja/pokedex.json` にフォルム別エントリを持たない）、`set_form`
    によるフォルム切り替えは行わず、`per_battle_once` フラグ経由の
    ability-disabled 状態（`consumed`）で「発動済み＝ばれたすがた」を表現する。
    アイスフェイス（`コオリッポ(アイス)`/`コオリッポ(ナイス)` の2エントリを持ち
    実ステータスが異なる）とはこの点が異なる。
  - 「攻撃を無効化した」扱いにはせず「0ダメージを受けた」扱いにするため、
    `HandlerReturn(value=0, stop_event=False)` とし、`ctx.defender.hits_taken += 1`
    を明示的にインクリメントする（ダメおし2倍・きあいパンチ不発の判定に必要。
    ダメージが0のため `move_executor.py` 側の通常加算処理が働かないことへの対応）。
- `Event.ON_MODIFY_NON_MOVE_DAMAGE`（`EventContext`, `subject_spec="target:self"`）で、
  `ctx.hp_change_reason == "self_attack"`（こんらんの自傷ダメージ、
  `handlers/volatile.py` 参照）の場合に同様の処理（ダメージ0・consumed付与・
  最大HPの1/8消費）を行う。こんらんの自傷ダメージは技扱いではなく
  `ON_MODIFY_MOVE_DAMAGE` を経由しないため、`がんじょう_survive_confusion_damage`
  と同じパターンで別ハンドラとして実装する。
  - このパスでは `hits_taken` を加算しない。こんらんで自傷した場合はそのターンの
    行動自体が失敗するため、ダメおし・きあいパンチの判定対象にならない。
- `AbilityData` に以下のフラグを付与する（`src/jpoke/types/literals.py` の
  `AbilityFlag` に定義済みの汎用フラグをそのまま利用し、個別ハンドラでの
  重複判定は行わない）:
  - `uncopyable` — トレース/ミイラ/レシーバー/かがくのちから/さまようたましい/
    とれないにおいに対して無効（`handlers/move_status.py` 等の
    `has_flag("uncopyable")` 判定で一元的に処理される）。
  - `protected` — 自身に対するスキルスワップ/なりきり/なかまづくり/なやみのタネ/
    シンプルビーム/いえき/うつしえを失敗させる（同上、`has_flag("protected")`
    判定で一元的に処理される）。
  - `per_battle_once` — 発動後は交代してもバトル終了まで再有効化されない
    （`model/ability.py` の `reset_enable_state` が `consumed` 理由を維持する）。
  - `mold_breaker_ignorable` — かたやぶり系の技に対して発動しない。
  - `gas_proof` — かがくへんかガスで無効化されない。
- `Event.ON_MODIFY_MOVE_DAMAGE` は `docs/spec/turn.md` の `Event.ON_MODIFY_DAMAGE`
  表（40番「ばけのかわに被弾」）にそのまま対応するイベントのため、`priority=40`
  を指定する（アイスフェイスと同一の優先度。実装は同表と一致させる）。
  - `ちきゅうなげ`・`ナイトヘッド`など`level_fixed_damage`（`priority=15`）で
    攻撃側のダメージ値を確定する固定ダメージ技より後に実行する必要がある
    （先に実行すると、ばけのかわが0へ書き換えた値を固定ダメージ技が
    再度上書きしてしまい、肩代わりが機能しなくなる。fuzz_log seed=73で
    この不具合が確認された）。
  - `がんじょう_survive_lethal`・`きあいのタスキ_survive_ohko`・
    `きあいのハチマキ_survive_by_chance`（いずれもデフォルト優先度100）より
    先に完全ブロックする必要があるが、`priority=40`であれば
    固定ダメージ技（15）より後・これらのHP1残し系ハンドラ（100）より先という
    両条件を同時に満たす。
- `Event.ON_MODIFY_NON_MOVE_DAMAGE` は `がんじょう_survive_confusion_damage` と同様、
  デフォルト優先度（100）のままでよい（こんらんダメージの発生源は1つのみで、
  他ハンドラとの競合が実質発生しないため）。
- リーサル計算（`handlers/lethal.py` の `ばけのかわ_block_damage`,
  `LethalEvent.ON_BEFORE_HIT`, `subject="defender"`）は実装済み。
  `hp_dist` 内で `ability_enabled` な状態のみ「ダメージ0・
  最大HPの1/8消費・ability_enabled=False」に遷移させ、既に発動済みの状態は
  素通りさせる。こんらん自傷は攻撃側の技のダメージ計算とは無関係のため、
  リーサル計算の対象外とする（既存のリーサル計算はダメージ交換系列の
  期待値計算に特化しており、防御側の状態異常起因の自傷は対象外という
  既存方針と一致する）。

## テスト観点

1. 1回目の攻撃技を受けたときダメージが0になり、最大HPの1/8が減少する。
2. 2回目以降の攻撃技には通常通りダメージを受ける（`ability.enabled` が
   `False` になっている）。
3. かたやぶりを持つ相手の攻撃に対しては発動せず、通常ダメージを受ける
   （`ability.enabled` は `True` のまま）。
4. 交代して戻ってきても再有効化されない（`per_battle_once` の維持）。
5. 連続攻撃技を受けた場合、1発目のみ防ぎ2発目以降は通常ダメージを受ける。
6. こんらんの自傷ダメージ（`ON_MODIFY_NON_MOVE_DAMAGE` 経由）も防ぎ、
   最大HPの1/8のみ消費する。
7. （リーサル計算）1発目はダメージ分布が0、HP減少は変身解除ダメージのみ、
   全状態で `ability_enabled` が `False` になる。2発目以降は通常ダメージの
   分布と一致する。

## 保留・論点

- みずびたし等でタイプが変化している状態でばけのかわが発動した場合の
  「ゴースト/フェアリータイプに戻る」処理、パワーシェア等でステータスが
  変化している場合の「元のステータスに戻る」処理は未実装。
  本エンジンには「フォルムチェンジ」を表す汎用イベントが存在せず、ミミッキュの
  ばけたすがた/ばれたすがたも図鑑データ上は単一エントリ（同一ステータス・
  タイプ）であるため、これらのリセット処理を行う基盤がない。発生条件が
  非常に限定的（相手や自分の技でタイプ/ステータスを変更された直後に
  ばけのかわを発動させる必要がある）なニッチな相互作用のため、対象外とする。
