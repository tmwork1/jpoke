# 特性実装計画: フィルター

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/フィルター.md`
- `docs/spec/abilities/ハードロック.md`（特性の仕様はハードロックを参照する仕様）

## 実装ゴール

- 相手から受ける効果バツグンの技のダメージを3/4倍にする。
- 効果・仕様はハードロックと完全に同一（`docs/spec/abilities/フィルター.md` の
  「特性の仕様」がハードロックを参照する仕様のため）。タイプ相性自体が変わる要因
  （もりののろい/ハロウィンで追加されたタイプ、タールショットで追加された弱点、
  フリーズドライ/フライングプレスによる相性変化、らんきりゅうによる弱点無効化）は
  正しく反映しつつ、相性自体は変わらない効果（弱点半減のきのみ等）には影響されない
  ようにする。
- かたやぶり・かがくへんかガス等で無効化される（`mold_breaker_ignorable` フラグで
  一元的に処理されるため、ハンドラ側での個別判定は不要。
  `.claude/loop/_common.md` および memory の既存方針に従う）。プリズムアーマーとは
  異なり、かたやぶりに対しては無効化される点に注意。

## 実装方針

- `Event.ON_CALC_DAMAGE_MODIFIER`（`AttackContext`, `subject_spec="defender:self"`）で
  `battle.query.is_super_effective(ctx)` を判定し、真なら `value` に固定小数点で
  3072/4096（0.75倍）を乗算する。
  - ハードロックと完全に同一効果のため、ハンドラ関数 `ハードロック_reduce_effective`
    を共用する（新規関数は実装しない）。
  - `Event.ON_CALC_DAMAGE_MODIFIER` は `docs/spec/turn.md` に未掲載
    （ダメージ計算内部イベント）のため、既存の同種ハンドラ
    （`ハードロック`/`プリズムアーマー`/`ブレインフォース` 等）と同様に
    priority のデフォルト値をそのまま用いる。
- `AbilityData` に `flags={"mold_breaker_ignorable"}` を付与し、シャドーレイ・
  フォトンゲイザー・かたやぶり等の無視技で無効化される（プリズムアーマーとの違い＝
  プリズムアーマーは `mold_breaker_ignorable` を持たず、かたやぶりに対しても有効）。

## テスト観点

1. 効果バツグンの技を受けたとき、`damage_modifier` が3072（0.75倍）になる。
2. かたやぶりを持つ相手の攻撃に対しては無効化され、`damage_modifier` が4096のままになる。

## 保留・論点

- なし（実装済み。ハードロックと共有ハンドラ `ハードロック_reduce_effective` を
  使用しているため、個別実装の追加は不要）。
