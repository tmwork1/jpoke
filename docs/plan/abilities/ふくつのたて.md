# 特性実装計画: ふくつのたて

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/ふくつのたて.md`
- `docs/spec/abilities/ふとうのけん.md`（「上がる能力を除けば同様の効果」のため詳細仕様の参照元）

## 実装ゴール

- バトル中一度だけ、場に出たときにぼうぎょが1段階上がる。
- 2回目以降の入場では発動しない。
- 既にぼうぎょが最大まで上がっていて上昇が不発だった場合でも、以後その戦闘で発動できなくなる
  （＝発動判定ではなく「場に出た」こと自体で1回分を消費する）。
- かがくへんかガスの発動中は発動せず、解除された瞬間（特性を手にしたときと同じ扱い）に発動する。

## 実装方針

- `Event.ON_SWITCH_IN` と `Event.ON_ABILITY_ENABLED`（かがくへんかガス解除等での再有効化）の
  両方に同一ハンドラを登録し、`subject_spec="source:self"` とする（`ふとうのけん` と同じ構成）。
- `per_battle_once` フラグを `data/ability.py` の `flags` に設定する。
- 「バトル中1回」の管理は `mon.ability.state` ではなく
  `battle.add_ability_disabled_reason(mon, "consumed")` を使う。この呼び出しは
  `modify_stats` の成否に関わらず必ず行う（上昇が不発でも消費される仕様のため）。
  `Ability.reset_on_switch_out()` が `per_battle_once` フラグ保有時に `"consumed"` 理由を
  維持するため、交代・ひんしを経ても再発動しない。

## 変更候補

- `src/jpoke/data/ability.py` — `ふくつのたて` に `flags={"per_battle_once"}` と
  `ON_SWITCH_IN` / `ON_ABILITY_ENABLED` handler を登録する
- `src/jpoke/handlers/ability.py` — `ふくつのたて_boost_B` を実装する
- `tests/abilities/test_ability_ha.py` — 単体テストを追加する
- `tests/abilities/test_ability__group.py` — `ふとうのけん` と共通の「一度きりの能力上昇特性」
  パラメタライズテストに含める

## テスト観点

1. 初回入場で B が 1 段階上がる。
2. 2回目以降の入場では発動しない（交代でランクリセット後も再発動しない）。
3. かがくへんかガスの発動中は発動しない。
4. かがくへんかガスの解除と同時に発動する。
5. Bが既に最大でBが上がらなかった場合でも、以後その戦闘で発動できなくなる。
