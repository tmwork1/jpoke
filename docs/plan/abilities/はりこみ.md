# 実装計画: はりこみ

作成日: 2026-07-12

## スコープ

- 仕様書: `docs/spec/abilities/はりこみ.md`
- 実装状態: 実装済み（`src/jpoke/handlers/ability.py` の `はりこみ_modify_atk`）。
  本レビューで死に出し除外の不具合を修正した。

## 効果

ポケモンチェンジ（通常の交代コマンド、使うと交代するわざ、レッドカード・だっしゅつボタン・
だっしゅつパック、にげごし・ききかいひ）で出てきた相手に攻撃するときの こうげき・とくこう を
2倍にする。ただし以下は対象外:

- 初手で繰り出された相手
- 死に出し（瀕死交代）で繰り出された相手
- かわりものが発動したポケモンに対して

## 実装方針

### 「交代直後」の判定

`PlayerState.has_switched`（今ターン中に交代したか）を使う。この値は `reset_turn_state()` で
毎ターン開始時にリセットされるため、初手（0ターン目の初期交代）で場に出たポケモンに対しては
1ターン目の攻撃時点で既に `False` に戻っており、追加の除外実装なしに「初手は対象外」の仕様を
満たす。

### 「死に出しは対象外」の判定（本レビューで新規追加）

`has_switched` は初期交代・死に出し交代・通常交代のいずれでも `True` になる汎用フラグであり、
これだけでは死に出しを区別できない。実際に区別が必要になるケースがある: 自分より速い相手が
反動技等で自滅し瀕死交代が発生した後、自分（はりこみ所持者、後攻）の攻撃がその交代先に
当たる同ターン内のシナリオ。これは既存の類似実装（`しっぺがえし`・`フォーカスレンズ` の
`has_switched` 判定）が想定していない、はりこみ固有の追加条件のため、コア側に新しいフラグを
追加する。

- `PlayerState.switched_in_by_faint: bool`（`src/jpoke/core/player_state.py`）を追加。
  `has_switched` と同じく `reset_turn_state()` で毎ターンリセットする。
- `SwitchManager.run_switch()`（`src/jpoke/core/switch_manager.py`）で、`state.interrupt` を
  `Interrupt.NONE` に破棄する **前** に `state.interrupt == Interrupt.FAINTED` を判定し、
  `_switch_in()` 呼び出し後に `state.switched_in_by_faint` へ代入する。
  - `run_faint_switch()` は交代直前に `state.interrupt = Interrupt.FAINTED` を設定してから
    `run_interrupt_switch(Interrupt.FAINTED, False)` → `run_switch()` を呼ぶため、この時点で
    正しく判定できる。
  - 初期交代（`run_initial_switch()`）は `run_switch()` を経由せず `_switch_in()` を直接呼ぶため
    `switched_in_by_faint` は更新されない（デフォルト `False` のまま）。「初手は対象外」は
    上記の `has_switched` リセットで別途満たされているため問題ない。
  - レッドカード・だっしゅつボタン・だっしゅつパック・にげごし・ききかいひ・使うと交代するわざ
    (ピボット) は、いずれも `run_switch()` 呼び出し時点で `state.interrupt` が
    `Interrupt.FAINTED` ではない（`NONE` または `PIVOT`/`EMERGENCY`/`EJECTBUTTON`/`EJECTPACK_*`）
    ため `switched_in_by_faint = False` となり、はりこみは正しく発動する。

はりこみのハンドラは `has_switched and not switched_in_by_faint` を発動条件とする。

### 「かわりものが発動したポケモンに対しては発動しない」（既知の制限）

かわりもの（Imposter）は `docs/progress/ability.md` 上で実装保留（`保留`）であり、場に出た瞬間に
相手をコピーする効果自体がこのプロジェクトに存在しない。そのため「かわりものが発動した
ポケモンに対しては発動しない」という除外条件を判定する状態（今ターン中にかわりものが発動したか）
がそもそも存在せず、実装不可能。かわりものの実装が入るまでの既知の制限として対応を見送る。

| イベント | priority | subject_spec | 関数名 |
|---------|---------|-------------|--------|
| `ON_CALC_ATK_MODIFIER` | 規定なし（`はりきり_modify_atk` 等の既存同種ハンドラに合わせ priority 未指定） | `attacker:self` | `はりこみ_modify_atk` |

## 実装（`src/jpoke/handlers/ability.py`）

```python
def はりこみ_modify_atk(battle: Battle, ctx: AttackContext, value: int) -> HandlerReturn:
    """はりこみ特性: 交代直後の相手に対する攻撃補正を2倍にする。

    初手で繰り出された相手（1ターン目の初期交代は次ターン開始時に
    has_switched がリセットされるため自然に除外される）や、死に出しで
    繰り出された相手には発動しない（switched_in_by_faint で除外）。
    """
    if ctx.defender is None:
        return HandlerReturn(value=value)

    defender_state = battle.player_states[battle.get_player(ctx.defender)]
    if defender_state.has_switched and not defender_state.switched_in_by_faint:
        value = apply_fixed_modifier(value, 8192)
    return HandlerReturn(value=value)
```

```python
"はりこみ": AbilityData(

        )

```
