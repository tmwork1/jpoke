# 特性実装計画: はじまりのうみ

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/はじまりのうみ.md`
- `docs/spec/fields/おおあめ.md`（おおあめの詳細）

## 実装ゴール

- 場に出たとき（`ON_SWITCH_IN`）天候を「おおあめ」にする。
- とくせいが再度有効化されたとき（`ON_ABILITY_ENABLED`）も同様に発動する。
- おおあめ中は、ほのおタイプの技（攻撃技・変化技問わず）を失敗させる。
- おおあめ中は、あめ相当の効果（みず技威力1.5倍・ほのお技威力0.5倍）を適用する。
  （あめと異なり、こおり付与を防ぐ効果はない）
- 通常天候（はれ/あめ/すなあらし/ゆき）・おおひでりでは上書きされず、らんきりゅうは上書き可能。
- 場からこの特性のポケモンがいなくなったとき（`ON_SWITCH_OUT`）・とくせいが無効化されたとき
  （`ON_ABILITY_DISABLED`）おおあめを解除する。ただし相手も同じ特性を持つ場合は解除しない。

## 実装方針

- 天候の発動・解除は、あめふらし/おわりのだいち/デルタストリームと共通の
  `_activate_weather` / `_deactivate_strong_weather` ヘルパー（`handlers/ability.py`）を使う。
  強天候は count=1 で登録する（`ON_TURN_END` でのカウントダウン対象外にするため）。
- ほのおタイプ技の失敗・あめ相当効果（威力補正）・天候の上書き優先度制御は、
  特性側ではなく `data/field/weather.py` の `"おおあめ"` エントリと
  `WeatherManager.apply`（`core/field_manager.py`）の優先度比較に委譲する。
  これにより、あめの `あめ_power_modifier` ハンドラをそのまま再利用できる。
  （あめの `あめ_prevent_freeze` に相当するこおり無効効果は、おおあめには存在しないため登録しない）
- ノーてんき/エアロックによる効果抑制は `ON_CHECK_WEATHER_ENABLED` の共通導線（天候システム側）に従う。

## イベント別実装計画（turn.md準拠）

1. `Event.ON_SWITCH_IN`（Priority 100, turn.md「はじまりのうみ（特性）」行相当）
   - `はじまりのうみ_activate_weather` → `_activate_weather(weather="おおあめ", count=1)`
2. `Event.ON_ABILITY_ENABLED`（とくせいなし解除・スキルスワップ等での再有効化時、既存の他天候始動特性と同じ扱い）
   - `はじまりのうみ_activate_weather` を同様に接続。
3. `Event.ON_SWITCH_OUT` / `Event.ON_ABILITY_DISABLED`
   - `はじまりのうみ_deactivate_strong_weather` → `_deactivate_strong_weather(weather="おおあめ")`。
   - 現在の天候が「おおあめ」かつ、相手の特性名がこの特性と異なる場合のみ解除する
     （相手も同じ特性なら解除しない＝仕様書「場から特性はじまりのうみを持つポケモンがいなくなった場合、
     おおあめ状態は解除される」の裏として、相手が同特性を保持していれば場に残るため解除しない）。
4. `Event.ON_TRY_MOVE_1`（Priority 10, turn.md「おおあめ・おおひでりによる技失敗」行）
   - `data/field/weather.py` の `"おおあめ"` に `おおあめ_block_fire_move` を登録済み。
   - ほのおタイプの技を `stop_event=True` で失敗させる（新規ハンドラは不要、field側で実装済み）。
5. `Event.ON_CALC_POWER_MODIFIER`
   - `"おおあめ"` に `あめ_power_modifier` を流用登録済み（みず1.5倍・ほのお0.5倍）。
   - ほのお技は 4. で失敗するため、この補正が実際に効くのはみず技威力補正のみ。
   - こおり無効効果（`はれ_prevent_freeze` 相当）はおおあめには存在しないため未登録（仕様通り）。
6. 天候の上書き制御
   - `WEATHER_PRIORITY`（`data/field/weather.py`）で `おおあめ` は優先度1、通常天候は0、
     らんきりゅうは2。`WeatherManager.apply` が優先度比較で上書き可否を判定するため、
     特性側に個別の上書き禁止ロジックは不要。
7. `Event.ON_TURN_END`
   - `"おおあめ"` には `tick_weather` を登録しない（強天候はターン経過で自然解除されない）。

## テスト観点

1. 登場時に「おおあめ」が発動し、count=1で登録される。
2. ほのおタイプの攻撃技・変化技が失敗する。
3. 通常天候（はれ/あめ/すなあらし/ゆき/おおひでり）では上書きされる。らんきりゅうでは上書きされない。
4. 相手も同じ特性を持つ場合、退場しても天候は解除されない。
5. 退場（交代・ひんし・とくせいなし化）でおおあめが解除される。

## 実装ステップ（実装済み）

1. `data/field/weather.py` に `おおあめ` を追加し、あめ相当効果とほのお技失敗を登録。
2. `data/ability.py` の `はじまりのうみ` に `ON_SWITCH_IN` / `ON_ABILITY_ENABLED` /
   `ON_SWITCH_OUT` / `ON_ABILITY_DISABLED` ハンドラを接続。
3. `handlers/ability.py` に `はじまりのうみ_activate_weather` /
   `はじまりのうみ_deactivate_strong_weather` を実装（共通ヘルパー経由）。
4. `tests/abilities/test_ability__group.py` に登場時発動・上書き可否・退場時解除のテストを追加。
