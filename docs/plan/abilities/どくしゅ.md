# 特性実装計画: どくしゅ

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/どくしゅ.md`

## 実装ゴール

- 直接攻撃でダメージを与えたとき、30%の確率で相手をどく状態にする。
- 技自体の追加効果とは個別に判定する（基本的には技の追加効果の判定の後にどくしゅの判定がある）。
- みがわり状態・技の無効化で発動しない。
- りんぷん・おんみつマント所持の相手には発動しない。
- パンチグローブ所持時は直接攻撃でなくなるため発動しない。
- 連続攻撃技は1ヒットごとに独立して発動判定を行う。

## 実装方針

- `Event.ON_DAMAGE_HIT`、`subject_spec="attacker:self"`、`priority=10` で実装する。
  - `docs/spec/turn.md` の Event.ON_DAMAGE（実装上の `Event.ON_DAMAGE_HIT` に相当）に
    「10 | 攻撃側のどくしゅによるどく」と明記されているため priority=10 を指定する
    （`AbilityHandler` のデフォルト priority は 100 であり、明示しないと
    ミイラ/さまようたましい/とれないにおい（同イベント priority=20）や
    はたきおとす等のアイテム効果（同100）より後に発動してしまい、
    「どくしゅの判定を先に行ってからミイラ等になる」「どくしゅの効果できのみを消費した
    ときははたきおとす等の技の効果が発動しない」という仕様（`docs/spec/abilities/どくしゅ.md`）
    を満たせない）。
  - `Event.ON_DAMAGE_HIT` は実HPダメージが0以下のとき発火しないため、こらえるでHP1のまま
    耐えた場合やばけのかわ/アイスフェイスで攻撃を肩代わりされた場合（実HPダメージ0）は
    どくしゅが発動しないという制約がある。`Event.ON_HIT`（常に発火、みがわり被弾でも発火）
    へ変更すればこの制約は解消できるが、`Event.ON_HIT` は move_executor.py 内で
    `Event.ON_DAMAGE_HIT` より必ず先に発火するため、技自身の追加効果
    （例: どくづきの追加効果、こちらも `Event.ON_DAMAGE_HIT` で処理される）より
    どくしゅの判定が先に走ってしまい、「技の追加効果の判定の後にどくしゅの判定がある」
    （どくづき使用時の合成確率 51% の根拠）という基本仕様が壊れる。
    両立できないため、より影響範囲の大きい後者（追加効果との順序）を優先し、
    `Event.ON_DAMAGE_HIT` を採用する。こらえる/ばけのかわ/アイスフェイスの
    実ダメージ0エッジケースは既知の未対応事項として扱う。
- 発動条件は `battle.query.is_contact(ctx)`（「自分の技が接触技であること」を判定する
  ヘルパー。`is_contact_reaction()` ではない。パンチグローブによるパンチ技の接触判定
  無効化は `Event.ON_CHECK_CONTACT` 経由で自動的に反映される）。
- 発動確率は `battle.resolve_secondary_chance(ctx, 0.3)` で解決する
  （りんぷん・おんみつマントは `Event.ON_MODIFY_SECONDARY_CHANCE` で確率を0にする
  既存実装があるため、どくしゅ側で個別に判定を書く必要はない）。
- どく付与は `battle.ailment_manager.apply(ctx.defender, "どく", source=ctx.attacker, ctx=ctx)`
  を使う（タイプ相性によるどく無効化・りんぷん以外のどく耐性特性の無効化などは
  `apply()` 内部の `_can_apply_by_type` / `Event.ON_BEFORE_APPLY_AILMENT` に委ねられる）。
- `apply()` が True を返した（実際にどく状態を付与できた）場合のみ
  `_announce_ability_triggered(battle, ctx.attacker)` を呼び、特性発動を明示する
  （せいでんき等が使う `_apply_contact_counter_ailment` ヘルパーに倣うが、そちらは
  ロール成功時に無条件で発動ログを出す実装になっている。どくしゅでは実際に
  どく状態を付与できた場合のみ発動を明示する、より厳密な扱いとする）。
- かがくへんかガス・かたやぶりによる無効化は、特性ハンドラの発火自体が
  自動的に抑止される既存の共通機構に委ねる（ハンドラ内に個別チェックを書かない）。

## テスト観点

1. 直接攻撃ヒット後に30%の確率で相手がどく状態になる（既存）。
2. 非接触技では発動しない（既存）。
3. 発動時に特性発動（`ability.revealed`）が記録される。
4. みがわり状態に防がれた場合は発動しない。
5. りんぷん所持相手には発動しない。
6. パンチグローブ所持時はパンチ技に発動しない。
7. 連続攻撃技は1ヒットごとに独立して発動判定がある
   （`resolve_secondary_chance` の呼び出し回数で検証する。命中判定・急所判定でも
   `battle.random.random()` が呼ばれるため、乱数呼び出し回数の単純カウントでは
   検証できないことに注意）。
