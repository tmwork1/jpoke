# 特性実装計画: きんしのちから

更新日: 2026-06-03

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/きんしのちから.md`
- 実装状態: 未実装
- 実装方針: 変化技選択時のみ `DomainEvent.ON_MODIFY_MOVE_PRIORITY` で行動順を後手化し、技の使用直前に `ON_ACTIVATE_MOLD_BREAKER` / `ON_DEACTIVATE_MOLD_BREAKER` で特性無視を発動させる（かたやぶりと同じ機構）。

## 実装ゴール（完了定義）

- 変化技を選択したとき、同じ優先度の行動の中で最後に行動する（あとだし相当）。
- 変化技使用の直前に相手の特性を無効化できる（かたやぶり相当）。
- 攻撃技を選択したときは発動しない。
- 既存テストを壊さない。

## 実装対象イベント

- `DomainEvent.ON_MODIFY_MOVE_PRIORITY` — 変化技選択時に行動順を後手化する
- `Event.ON_ACTIVATE_MOLD_BREAKER` — 変化技使用直前に相手の特性を無効化する
- `Event.ON_DEACTIVATE_MOLD_BREAKER` — 変化技使用後に無効化を解除する

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `きんしのちから` に handlers を登録する
- `src/jpoke/handlers/ability.py` — `きんしのちから_delay_priority`, およびかたやぶりと共有の `ON_ACTIVATE_MOLD_BREAKER` / `ON_DEACTIVATE_MOLD_BREAKER` を登録する
- `tests/test_ability.py` — テストを追加する

## 実装ステップ

1. 仕様書で「変化技のみ」の判定タイミングと、攻撃技で出た技（ねごとなど）の扱いを確認する。
2. `きんしのちから_delay_priority` を実装する（変化技のとき、あとだしと同じ方法で行動を後手化）。
3. `ON_ACTIVATE_MOLD_BREAKER` / `ON_DEACTIVATE_MOLD_BREAKER` はかたやぶりと共有できるか確認し、変化技判定の有無だけを切り替える。
4. `src/jpoke/data/ability.py` の `きんしのちから` に handlers を登録する。
5. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. 変化技を選択したとき、同じ優先度内で最後に行動する。
2. 変化技を使うとき相手の特性（クリアボディなど）を無視できる。
3. 攻撃技を選択したときは後手化しない。
