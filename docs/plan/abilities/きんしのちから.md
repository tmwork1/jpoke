# 特性実装計画: きんしのちから

更新日: 2026-06-03

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/きんしのちから.md`
- 実装状態: 実装済み
- 実装方針: 変化技選択時のみ `DomainEvent.ON_CALC_BACK_TIER`（あとだしと共通のイベント）で後攻ティアを-1し、技の使用直前〜使用後に `Event.ON_BEGIN_MOVE` / `Event.ON_END_MOVE`（かたやぶりと共通のイベント）で特性無視を発動・解除させる。
  - `DomainEvent.ON_CALC_BACK_TIER` は `docs/spec/turn.md` に未掲載のイベントだが、既存の `あとだし_delay_move_order` ハンドラ（`src/jpoke/data/ability.py` の `あとだし`）が同イベントを優先度指定なしで登録しているため、それに合わせて優先度は指定しない。
  - `Event.ON_BEGIN_MOVE` / `Event.ON_END_MOVE` も既存の `かたやぶり_disable_foe_ability` / `かたやぶり_restore_foe_ability`（`src/jpoke/data/ability.py` の `かたやぶり`）が優先度指定なしで登録しているため、それに合わせる。

## 実装ゴール（完了定義）

- 変化技を選択したとき、同じ優先度の行動の中で最後に行動する（あとだし相当）。
- 変化技使用の直前に相手の特性を無効化できる（かたやぶり相当）。
- 攻撃技を選択したときは発動しない。
- 既存テストを壊さない。

## 実装対象イベント

- `DomainEvent.ON_CALC_BACK_TIER` — 変化技選択時に後攻ティアを-1して行動順を後手化する（`AttackContext`, `subject_spec="attacker:self"`）
- `Event.ON_BEGIN_MOVE` — 変化技使用直前に相手の特性を無効化する（`AttackContext`, `subject_spec="attacker:self"`）
- `Event.ON_END_MOVE` — 変化技使用後に相手の特性の無効化を解除する（`AttackContext`, `subject_spec="attacker:self"`）

`ctx.move` は選択時ではなく「実際に使用される技」を指すため、ねごと・さいはい等で変化技/攻撃技が
入れ替わった場合も、実行直前の `ON_BEGIN_MOVE`/`ON_END_MOVE` 時点で正しく変化技かどうか判定できる。
一方 `DomainEvent.ON_CALC_BACK_TIER` はターン開始時の行動順決定（`SpeedCalculator.resolve_action_order`）
でのみ1回発火するため、後攻ティアは「選択した技」の分類のみに依存する（仕様の「行動順が遅くなる効果は
技の選択時に判定される」と一致）。

## 実装ステップ

1. 仕様書で「変化技のみ」の判定タイミングと、攻撃技で出た技（ねごとなど）の扱いを確認する。
2. `きんしのちから_delay_status_move` を実装する（変化技のとき、あとだしと同じ後攻ティア-1で行動を後手化）。
3. `きんしのちから_disable_foe_ability` / `きんしのちから_restore_foe_ability` を、かたやぶりの
   `ON_BEGIN_MOVE` / `ON_END_MOVE` ハンドラと同じロジックで実装し、変化技判定を追加する。
4. `src/jpoke/data/ability.py` の `きんしのちから` に handlers を登録する。
5. せんせいのツメ/イバンのみのハンドラにきんしのちから判定を追加する。
6. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. 変化技を選択したとき、同じ優先度内で最後に行動する。
2. 変化技を使うとき相手の特性（クリアボディなど）を無視できる。
3. 攻撃技を選択したときは後手化・特性無視のいずれも発動しない。
4. 変化技を選択したターンは、所有者が持つせんせいのツメ/イバンのみが発動しない。攻撃技選択時は発動し得る。
