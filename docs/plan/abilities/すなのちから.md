# 特性実装計画: すなのちから

更新日: 2026-07-11

## スコープ

- 仕様書: `docs/spec/abilities/すなのちから.md`
- 実装状態: 実装済み
- 実装方針: 2つの独立したハンドラで実装する。
  1. `ON_CALC_POWER_MODIFIER`（攻撃側）: すなあらし中に、いわ・じめん・はがねタイプの技の
     威力を5325/4096倍（1.3倍）にする。
  2. `ON_MODIFY_NON_MOVE_DAMAGE`（対象側）: すなあらしのダメージ（`ctx.hp_change_reason ==
     "sandstorm"`）を無効化する（すなかき・すながくれと共通の `_ignore_sandstorm_damage`
     ヘルパーを流用）。

## 実装ゴール（完了定義）

- すなあらし中、いわ・じめん・はがねタイプの技の威力が1.3倍（5325/4096倍）になる。
- すなあらしでない天候・天候なしのときは威力補正がかからない。
- いわ・じめん・はがね以外のタイプの技には補正がかからない。
- タイプによらずすなあらしのダメージを受けない（他タイプでも無効化）。
- 天候がすなあらしであっても、ノーてんき/エアロックにより天候効果が無効化されている場合は
  発動しない（`battle.weather.name` が天候抑制時に反映される既存の天候システムに委譲）。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_CALC_POWER_MODIFIER` — ダメージ計算時の威力補正イベント（`AttackContext`,
  `attacker:self`）。`docs/spec/turn.md` に掲載のないダメージ計算内部イベントのため、
  同種の威力補正特性（かたいツメ・てつのこぶし・がんじょうあご・ちからずく等）と同じ
  priority（`AbilityHandler` 既定値）を踏襲する。
- `Event.ON_MODIFY_NON_MOVE_DAMAGE` — すなあらし等の技以外のダメージ量を補正するイベント
  （`EventContext`, `target:self`）。すなかき・すながくれの同種ハンドラと同じ priority
  （既定値）を踏襲する。

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `すなのちから` に上記2イベントの handler を登録する
- `src/jpoke/handlers/ability.py`
  - `すなのちから_modify_power` を実装する
  - `すなのちから_ignore_sandstorm_damage` を実装する（`_ignore_sandstorm_damage` を流用）
- `tests/abilities/test_ability_sa.py` — テストを追加する
- `tests/test_field.py` — すなあらしダメージ免疫のパラメタライズドテストに追加する

## 実装ステップ

1. 仕様書 `docs/spec/abilities/すなのちから.md` で発動条件を確認する。
2. `src/jpoke/handlers/ability.py` に `すなのちから_modify_power(battle, ctx, value)` を実装する。
   `battle.weather.name == "すなあらし"` かつ `ctx.move.type in ["いわ", "じめん", "はがね"]`
   のとき `apply_fixed_modifier(value, 5325)` を返す。
3. `すなのちから_ignore_sandstorm_damage(battle, ctx, value)` を実装し、
   `_ignore_sandstorm_damage(battle, ctx, value)` に委譲する。
4. `src/jpoke/data/ability.py` の `すなのちから` に
   `Event.ON_CALC_POWER_MODIFIER`（`attacker:self`）と
   `Event.ON_MODIFY_NON_MOVE_DAMAGE`（`target:self`）を登録する。
5. `tests/abilities/test_ability_sa.py` にテストを追加し、全テストが通ることを確認する。

## テスト観点

1. すなあらし中にいわ・じめん・はがねタイプの技を使うと威力補正倍率が5325/4096になる
   （いわなだれ・じならし・アイアンヘッド等）。
2. すなあらし中でも上記以外のタイプの技（でんきショック等）には補正がかからない（4096のまま）。
3. すなあらしでないときは威力補正がかからない（4096のまま）。
4. すなあらしのダメージを（タイプに関わらず）受けない（`tests/test_field.py` の
   `test_すなあらし_特性免疫` でパラメタライズ済み）。
