# 特性実装計画: すなかき

更新日: 2026-07-11

## 仕様参照

- `docs/spec/abilities/すなかき.md`

## 実装ゴール

- 天気がすなあらし状態のとき、すばやさが2倍になる。
- タイプによらず、すなあらしのダメージを受けなくなる。
- すなあらし状態でも特性ノーてんき/エアロックのポケモンが場にいる場合は発動しない。

## 実装方針

- すばやさ補正: `DomainEvent.ON_CALC_SPEED`、`subject_spec="source:self"` で実装。
  - `battle.weather.name == "すなあらし"` で判定する。
    - `battle.weather`（`WeatherManager.active`）は `ON_CHECK_WEATHER_ENABLED` を経由して
      ノーてんき/エアロックによる天候無効化を自動的に反映するため、追加の重複ガードは不要。
    - ばんのうがさは「にほんばれ・あめ」の効果のみを無効化する専用アイテムであり、すなあらしには
      影響しないため、`weather_for()`（`ON_CHECK_WEATHER_IMMUNE` 経由）は使わず `battle.weather`
      を直接参照する（すいすい・ようりょくそとの違い。`docs/plan/abilities/すいすい.md` 参照）。
  - **ゆきかき**（ゆき中 S×2）と同一パターン。すなあらしには「強天候」に相当する天候が存在しないため、
    すいすいの「おおあめ」のような追加ケースは不要。
- すなあらしダメージ免除: 既存の `_ignore_sandstorm_damage` ヘルパーを利用し、
  `Event.ON_MODIFY_NON_MOVE_DAMAGE`、`subject_spec="target:self"` で実装。
  - `ctx.hp_change_reason == "sandstorm"` のときにダメージを0にして `stop_event=True` を返す。
  - すながくれ・すなのちから・ぼうじんと共通のヘルパー。

## 変更候補

- `src/jpoke/data/ability.py` — `すなかき` に `ON_CALC_SPEED` / `ON_MODIFY_NON_MOVE_DAMAGE` を登録する
- `src/jpoke/handlers/ability.py` — `すなかき_modify_speed` / `すなかき_ignore_sandstorm_damage` を実装する
- `tests/abilities/test_ability__group.py` — すいすい等と共通の天候依存素早さ上昇テストに含める
- `tests/test_field.py` — すなあらしダメージ免除の共通テストに含める（すながくれ・すなのちから・ぼうじん）

## テスト観点

1. すなあらし中 → S 実数値が2倍になる。
2. すなあらし以外（天候なし等）では変化なし。
3. すなあらし中でもすなかき所持ポケモンはターン終了時ダメージを受けない。
