## 実装ゴール（完了定義）

- 登場時に相手の技一覧から最高威力の技を1つ読み取り、メッセージに表示する。
- 既存テストを壊さない。

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `よちむ` に handler を登録する
- `src/jpoke/handlers/ability.py` — `よちむ_reveal_strongest_move` を実装する。
  威力欄が「―」と表示される変動威力技（一撃必殺技=150、カウンター系=120、
  くさむすび等=80 など）は `data.power` がダメージ計算用のプレースホルダに
  なっているため、専用の見なし威力テーブルで判定する。
- `tests/abilities/test_ability_yarawa.py` — テストを追加する

## 実装ステップ

1. 仕様書 `docs/spec/abilities/よちむ.md` で技の威力判定ルール（変化技=0, 一撃必殺=150 など）を確認する。
2. `src/jpoke/handlers/ability.py` に `よちむ_reveal_strongest_move` を実装する（相手の技を走査して最大威力の技名をログに出す）。
   威力欄が「―」の変動威力技は見なし威力テーブルを参照する（`data.power` をそのまま
   使うと一撃必殺技やくさむすび等が過小評価されるため）。
3. `src/jpoke/data/ability.py` の `よちむ` に handler を登録する。
4. `tests/abilities/test_ability_yarawa.py` にテストを追加し、全テストが通ることを確認する。

## テスト観点

1. 相手が複数の技を持つとき、最高威力の技がメッセージに表示される。
2. 同威力の技が複数あるとき、いずれかの技名がメッセージに表示される。
3. 相手が変化技のみを持つとき、変化技が読み取られる（威力0として最大）。
4. 威力欄が「―」の変動威力技（くさむすび等）が見なし威力80として通常技より優先される。
5. 一撃必殺技が見なし威力150として最優先で読み取られる。
