# ねんちゃく 実装計画

## スコープ
- 優先度: -
- 仕様書: `docs/spec/abilities/ねんちゃく.md`
- 実装方針: 既存の `Event.ON_CHECK_ITEM_CHANGE` 共通判定を単一の制御点として維持し、
  アイテム変更経路はすべて `ItemManager.swap_items()` / `take_item()` / `remove_item()`
  を経由させる。

## 現状整理（2026-07-12 レビュー時点）
- `src/jpoke/core/item_manager.py` の `can_change_item()` が `Event.ON_CHECK_ITEM_CHANGE` を発火し、
  アイテムの交換・奪取・除去可否を共通判定している。
- `src/jpoke/data/ability.py` の `ねんちゃく` は `Event.ON_CHECK_ITEM_CHANGE` に登録済み（`flags={"mold_breaker_ignorable"}` でかたやぶり無効化にも対応済み）で、
  `src/jpoke/handlers/ability.py` の `ねんちゃく_prevent_item_change()` が相手由来のアイテム変更を拒否する。
- 現在の共通経路でカバー済みの系統は `トリック/すりかえ`、`どろぼう/ほしがる`、`むしくい/ついばむ`、
  `はたきおとす`、`やきつくす`、`ふしょくガス`、`マジシャン`、`わるいてぐせ`、`くっつきバリ` の転送。
  いずれも実装済み。
- `tests/abilities/test_ability_na.py` に `can_change_item()` を直接呼ぶ基本ケース（相手発動でブロック・
  自己起因では阻害しない・道具なしでもブロック）がある。加えて各技側テスト
  （`tests/moves_attack/test_move_ha.py` の どろぼう・はたきおとす、`test_move_ma.py` の むしくい、
  `test_move_yarawa.py` の やきつくす、`tests/moves_status/test_move_ha.py` の ふしょくガス、
  `tests/moves_status/test_move_ta.py` の トリック、`tests/items/test_item_ka.py` の くっつきバリ）が
  ねんちゃくとの組み合わせを検証する。

## 仕様要点（実装に反映）
- もちもの を奪われたり、失ったりする効果を防ぐ。
- 相手から受ける トリック ・ すりかえ を無効化する。自身が道具を持っていない場合でも無効化する。
- 相手から受ける ふしょくガス を無効化する。自身が道具を持っていない場合でも特性の表示が出る。
- どろぼう ・ ほしがる のダメージを受けてももちものを奪われない。
- むしくい ・ ついばむ のダメージを受けても きのみ を奪われない。
- はたきおとす ・ やきつくす ・ マジシャン ・ わるいてぐせ によるアイテム喪失も防ぐ。
- ねんちゃくのポケモンが自分から道具を交換・消費するとき（トリック・すりかえ・ギフトパスの使用、
  なげつける・しぜんのめぐみ・ほおばるでの消費）は発動しない。
- くっつきバリを持っていても、発動条件（接触・攻撃者が道具なし）を満たせば攻撃側に渡る
  （ねんちゃくは阻止しない）。
- かたやぶり効果がある技によりねんちゃくは無視される。

## レビューで判明した既存バグと修正（本レビューで対応済み）
`ItemManager.swap_items()` が交換の発生源（`source`）を受け取らず常に `can_change_item(source=None)`
で判定していたため、「ねんちゃくのポケモンが自分から道具を交換するときは発動しない」という仕様に
反し、以下がいずれも誤ってブロックされていた（`ctx.source(None) != ctx.target(self)` が常に成立して
しまうため）。
- ねんちゃく持ちの攻撃者が自分からトリックを使って渡す（本来は成功すべき）
- ねんちゃく持ちの攻撃者が どろぼう で相手からアイテムを奪う（`take_item()` 経由でも同様に発生）
- ねんちゃく持ちが持つ くっつきバリ の転送（`remove_item(mon)` が source なしで呼ばれていたため、
  かつ戻り値を確認せず `gain_item()` を呼んでいたため道具が複製される二次バグも併発）

修正方針:
1. `swap_items(*, source=None, ignore_sticky_hold=False)` に `source` を追加し、各 `can_change_item()`
   呼び出しへ伝播する。`source` は「この交換を起こした張本人（自分から行った側）」を表し、
   `ctx.source is ctx.target` のときのみ自己起因と判定される。
2. `take_item()` は内部で計算する `source = foe(target)` を `swap_items()` へ渡す。
3. `すりかえ_swap_items`（`handlers/move_status.py`）は `swap_items(source=ctx.attacker)` を渡す。
4. `くっつきバリ_transfer_on_contact`（`handlers/item.py`）は `remove_item(mon, source=mon)` とし、
   戻り値が `True` の場合のみ `gain_item()` を呼ぶ（複製防止）。
5. 副作用として `handlers/item.py` の `ブーストエナジー_prevent_item_change()` が
   `ctx.source is None`（=旧 `swap_items` の既定値）を「交換判定である」ことの代用シグナルとして
   使っていたため、本来の `ctx.is_exchange` フラグに置き換えて修正した。

## 実装フロー（今後の拡張時の指針）
1. `ねんちゃく` の本体は現状維持とし、新規のアイテム変更効果は必ず `ItemManager.swap_items()` /
   `take_item()` / `remove_item()` を経由させる。
2. 一方的な奪取・除去（はたきおとす・やきつくす・ふしょくガス等）は `remove_item(target, source=attacker)`
   のように必ず `source` を明示する。
3. 双方向の交換（トリック・すりかえ等）や `take_item()` 経由の奪取は `swap_items(source=...)` の
   `source` に「この行為を起こした張本人」を渡す。`source is target` の場合のみ自己起因として
   ねんちゃくを無効化する。
4. `ギフトパス`（未実装）を実装する際も同様に `swap_items(source=<渡す側>)` を使う。
5. 「場にいるときしか発動しない」仕様は、`ドラゴンテール`・`ともえなげ` 経由の強制交代と
   `マジシャン`/`わるいてぐせ` の組み合わせを実装・拡張する際に再確認する。

## テスト観点
- 既存ケース: `トリック/すりかえ`、`どろぼう/ほしがる`、`むしくい/ついばむ`、`はたきおとす`、`やきつくす`、
  `ふしょくガス` をそれぞれ相手発動で防げる。
- 非発動ケース: 自分の `トリック`、通常の道具消費、`おちゃかい`、`くっつきバリ` の転送など自発・仕様上の
  例外では防がない（本レビューで追加・修正）。
- 無効化ケース: `かがくへんかガス` 中と `かたやぶり` 系統の技で防げない。
- 境界ケース: 対象が道具なしでも `トリック` と `ふしょくガス` で特性表示が出る。
- 将来ケース: 交代を挟む `マジシャン` / `わるいてぐせ` / 強制交代との相互作用が現行世代仕様と一致する。
