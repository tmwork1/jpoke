# 特性実装計画: アイスフェイス

更新日: 2026-06-03

## スコープ

- 仕様書: `docs/spec/abilities/アイスフェイス.md`
- 実装状態: 部分実装（flags のみ設定済み）
- 実装方針: 物理ダメージを 0 にして「ナイスフェイス」にフォルムチェンジする処理と、ゆき発生時に「アイスフェイス」へ戻るフォルムチェンジを別々のハンドラで実装する。

## 実装ゴール（完了定義）

- アイスフェイスフォルム時に物理技のダメージを 0 にし、ナイスフェイスにフォルムチェンジする。
- ナイスフェイスフォルム時、天気がゆきに変化したときアイスフェイスにフォルムチェンジする。
- ゆき状態の場にナイスフェイスのコオリッポを繰り出したときアイスフェイスにフォルムチェンジする。
- 特殊技のダメージは防がない。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_MODIFY_MOVE_DAMAGE` — 物理技ダメージを 0 に書き換え、フォルムチェンジを実行する
- `Event.ON_SWITCH_IN` — 登場時にゆきなら即フォルムチェンジする
- `Event.ON_FIELD_CHANGE` — 天気変化時にナイスフェイスからアイスフェイスへ戻す

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `アイスフェイス` に handlers を登録する
- `src/jpoke/handlers/ability.py` — 以下の関数を実装する
  - `アイスフェイス_block_physical` — 物理ダメージを 0 にしてナイスフェイスへフォルムチェンジ
  - `アイスフェイス_restore_on_snow` — ゆき発生時にアイスフェイスへ復帰
  - `アイスフェイス_restore_on_switch_in` — 登場時のゆき確認と復帰
- `tests/test_ability.py` — テストを追加する

## 実装ステップ

1. 仕様書 `docs/spec/abilities/アイスフェイス.md` でフォルムチェンジ条件・例外条件を確認する。
2. `src/jpoke/handlers/ability.py` に `アイスフェイス_block_physical` を実装する。
   - `ctx.move.category == "物理"` かつ `ctx.target フォルム == "アイスフェイス"` のとき value を 0 に書き換え、ナイスフェイスへフォルムチェンジする。
   - かたやぶり状態のときは何もしない。
3. `src/jpoke/handlers/ability.py` に `アイスフェイス_restore_on_snow` を実装する。
   - 天気がゆきに変化し、自分がナイスフェイスフォルムのとき、アイスフェイスへフォルムチェンジする。
   - 場にノーてんき/エアロックがいるときは発動しない。
4. `src/jpoke/handlers/ability.py` に `アイスフェイス_restore_on_switch_in` を実装する。
   - 登場時、場がゆき状態かつ自分がナイスフェイスのときアイスフェイスへフォルムチェンジする。
5. `src/jpoke/data/ability.py` の `アイスフェイス` に各ハンドラを登録する。
6. `tests/test_ability.py` にテストを追加し、全テストが通ることを確認する。

## テスト観点

1. アイスフェイスフォルム時に物理技を受けてもダメージが 0 になる。
2. アイスフェイスフォルム時に物理技を受けるとナイスフェイスにフォルムチェンジする。
3. ナイスフェイスフォルム時は物理技のダメージを防がない。
4. 特殊技のダメージはアイスフェイスフォルム時も防がない。
5. ゆきが発生するとナイスフェイスからアイスフェイスに戻る。
6. ゆき状態の場にナイスフェイスで登場するとアイスフェイスに戻る。
7. かたやぶり系の物理技に対してはダメージを防がない。
8. かたやぶり系の技でのゆき発生時はフォルムチェンジしない。
9. ノーてんき/エアロック発動中はゆき変化でフォルムチェンジしない。

## リスクと対策

- リスク: フォルムチェンジでタイプ・ステータスが変動するため、ターン中の計算に影響する。
  - 対策: 仕様書の「フォルムチェンジした瞬間から行動順に影響」の記載に従い、変更は即時反映する。
- リスク: ノーてんき/エアロック との相互作用が複雑。
  - 対策: `ON_FIELD_CHANGE` ハンドラ内で `ON_CHECK_WEATHER_ENABLED` 相当のチェックを追加する。
