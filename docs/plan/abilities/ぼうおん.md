# 特性実装計画: ぼうおん

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/ぼうおん.md`
- `docs/spec/moves/ほろびのうた.md`

## 実装ゴール

- 音の技を受けても無効化する。
- すてゼリフの攻撃・特攻低下効果と交代効果の両方を無効化する。
- かたやぶりの効果がある音技は防げない。
- ほろびのうたのように使用者自身にも効果が及ぶ音技は、ぼうおん所持者への
  付与のみを防ぎ、他のポケモンへの効果は通常通り発動する。

## 実装方針

- `Event.ON_BEFORE_APPLY_MOVE`（`defender:self`）で `ctx.move.has_flag("sound")` かつ
  `ctx.move.target == "foe"` の技を技全体無効化する（`h.ぼうおん_block_sound`,
  `src/jpoke/handlers/ability.py`）。
- 音技の無効化は `MoveData.flags` の `"sound"` を基準に実装する。
- **自分含む全員が対象の技（ほろびのうたなど）は例外**: 技全体を無効化すると
  使用者自身への効果まで防いでしまうため、`_BOUON_EXCLUDED_MOVES`
  （`src/jpoke/handlers/ability.py`）で `ぼうおん_block_sound` の全体無効化対象から
  除外し、代わりに各技のハンドラ側（`handlers/move_status.py` の
  `_blocked_by_bouon`、おうごんのからだの `_blocked_by_ougon_no_karada` と同じ
  パターン）で対象ポケモンごとに免疫判定を行う。呼び出し側で使用者自身は対象から
  除外する（現行世代では自身のぼうおんは無視されるため）。
  この per-mon 免疫判定はイベントハンドラを介さない直接呼び出しだが、
  `mon.ability.name`/`mon.ability.enabled` を参照するためかたやぶり等による
  無効化も自動的に反映される（`AbilityHandler` の `enabled_ignoring` チェックと
  同じ仕組み）。
- すてゼリフは通常の単一対象技として扱い、技全体無効化（能力低下・交代効果の
  両方を防ぐ）で仕様を満たす。
- りんしょうの行動順繰り上げ・威力2倍効果は防げないことを注記する。

## テスト観点

1. おたけび等の音技を受けて無効化する（こんらん等の追加効果が付与されない）。
2. すてゼリフの攻撃・特攻低下と交代効果が無効化される。
3. ほろびのうたを受けたぼうおん所持者は状態にならないが、技自体は失敗しない
   （使用者自身には通常通り付与される）。
4. かたやぶりを持つ攻撃側に対してはぼうおんの免疫が発動しない（単純な音技・
   ほろびのうたの両方）。
