# 特性実装計画: ソウルハート

更新日: 2026-07-11

## 仕様参照

- `docs/spec/abilities/ソウルハート.md`

## 実装ゴール

- シングルバトルでは「自分」と「相手」の2体しか場に出ないため、実質的には
  「相手がどのような要因であれひんしになったとき、自分のとくこうが1段階上がる」
  特性として実装する（自分自身がひんしになったことは対象外。仕様書「特性の仕様」節参照）。
- C が最大のときは発動しない。

## 実装方針

- 攻撃技によるKO（reason="move_damage"）と、それ以外の要因（状態異常・天候・
  自滅技の反動など）によるKOとで、発火させるイベントを分ける。
  - 攻撃技によるKO: `Event.ON_MOVE_KO`、`subject_spec="attacker:self"`。
    **くろのいななき**（倒したらC↑1）と同じ `_boost_on_move_ko` パターンを踏襲する。
    ON_MOVE_KO は `ON_DAMAGE_HIT`（ミイラ/さまようたましい/とれないにおいの
    特性書き換え、priority=20）より後に発火するため、直接攻撃で倒した相手が
    これらの特性を持つ場合は特性が書き換えられ、ソウルハートは正しく発動しない
    （仕様書の該当箇所を参照）。
  - それ以外の要因によるKO: `Event.ON_HP_CHANGED`、`subject_spec="target:foe"`。
    `ctx.target.fainted` かつ `ctx.hp_change_reason != "move_damage"` のときのみ
    発動する。`reason == "move_damage"` を除外することで、ON_MOVE_KO 側との
    二重発動を防ぐ（`ON_HP_CHANGED` は `battle.modify_hp` 内部で `ON_DAMAGE_HIT`/
    `ON_MOVE_KO` より前に発火するため、攻撃技KOをここに含めると、ミイラ等による
    特性書き換えより先にソウルハートが発動してしまい仕様と食い違う）。
    `subject_spec="target:foe"` は「ctx.target の相手側が自分（subject）である」
    ことを意味するため、これにより「相手（foe）がひんしになったとき」にのみ
    反応し、自分自身がひんしになったときは対象外になる。

## 変更候補

- `src/jpoke/data/ability.py` — `ソウルハート` に `ON_MOVE_KO` と `ON_HP_CHANGED`
  の2つの handlers を登録する
- `src/jpoke/handlers/ability.py` — `ソウルハート_boost_spa_on_ko`（攻撃技KO用）と
  `ソウルハート_boost_spa_on_faint`（それ以外の要因によるKO用）を実装する
- `tests/abilities/test_ability_sa.py` — テストを追加する

## テスト観点

1. 自分が攻撃技で相手を倒すと C が 1 段階上がる（ON_MOVE_KO 経路）。
2. 相手が状態異常（どく等）のダメージでひんしになったときも C が 1 段階上がる
   （ON_HP_CHANGED 経路、reason != move_damage）。
3. 相手がだいばくはつ等の自滅技で自分自身をひんしにしたときも C が 1 段階上がる
   （reason="self_cost"、ON_HP_CHANGED 経路）。
4. 自分自身がひんしになっても C は上がらない（target:foe に一致しないため）。
5. C が最大のときは発動しない（両経路とも）。

## 疑似コード

```python
def ソウルハート_boost_spa_on_ko(battle, ctx, value):
    """攻撃技によるKO用（ON_MOVE_KO, attacker:self）。"""
    mon = ctx.attacker
    if not battle.is_active(mon):
        return HandlerReturn(value=value)
    if battle.modify_stats(mon, {"spa": +1}, source=mon):
        announce_ability_triggered(battle, mon)
    return HandlerReturn(value=value)


def ソウルハート_boost_spa_on_faint(battle, ctx, value):
    """攻撃技以外の要因によるKO用（ON_HP_CHANGED, target:foe）。"""
    if not ctx.target.fainted or ctx.hp_change_reason == "move_damage":
        return HandlerReturn(value=value)
    mon = battle.foe(ctx.target)
    if not battle.is_active(mon):
        return HandlerReturn(value=value)
    if battle.modify_stats(mon, {"spa": +1}, source=mon):
        announce_ability_triggered(battle, mon)
    return HandlerReturn(value=value)
```
