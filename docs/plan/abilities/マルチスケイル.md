# マルチスケイル 実装計画

更新日: 2026-07-12

## スコープ

- 仕様書: `docs/spec/abilities/マルチスケイル.md`
- 実装状態: 実装済み（本ドキュメントは既存実装の追認・整理。レビュー時にこんらん自傷
  ダメージの半減バグを検出し修正した）

## 仕様要点（実装に反映）

- 自分のHPが満タンのとき、攻撃技によって受けるダメージを0.5倍にする。
- ダメージ固定技のダメージは半減しない。
- こんらんの自傷ダメージ（内部技 `"_こんらん"`）は半減しない。
- かたやぶり効果を持つ技・特性を無視する技のダメージは半減しない
  （＝この特性自体は `mold_breaker_ignorable` フラグを持つ）。
- 一度HPが減っても、回復してHP満タンになれば特性は何度でも発動する。
- みがわり状態のときも、みがわりではなく本体のHPが満タンのときに特性が発動する。
- 連続攻撃技に対しては、1発ごとに特性が発動するかが判定される
  （各ヒット時点でのHPを見て都度判定）。

## 実装対象イベント

- `Event.ON_CALC_DAMAGE_MODIFIER` — ダメージ計算内部イベント。`docs/spec/turn.md` に
  掲載がないため、既存の同種ハンドラ（パンクロック・ハードロック等の被ダメージ補正系
  特性）の登録済み priority（未指定・デフォルト）に合わせる。

## 変更対象（ファイル/関数単位・実績）

- `src/jpoke/data/ability.py`
  - `マルチスケイル` エントリに `flags={"mold_breaker_ignorable", "full_hp_damage_modifier"}`
    を設定する。
  - `Event.ON_CALC_DAMAGE_MODIFIER` に `h.マルチスケイル_reduce_damage`
    （`subject_spec="defender:self"`）を登録する。
- `src/jpoke/handlers/ability.py`
  - `マルチスケイル_reduce_damage(battle, ctx: AttackContext, value: int)`:
    `ctx.defender.hp == ctx.defender.max_hp` かつ `ctx.move.name != "_こんらん"` のとき
    `apply_fixed_modifier(value, 2048)` で0.5倍化する。
    - `mold_breaker_ignorable` フラグにより、攻撃側がかたやぶり等を持つ場合は
      `ON_BEGIN_MOVE` で `かたやぶり_disable_foe_ability` がこの特性を一時的に
      disable する（`battle.add_ability_disabled_reason`）ため、`ON_CALC_DAMAGE_MODIFIER`
      でこのハンドラ自体が呼ばれず無効化される（`handlers/ability.py` の
      `かたやぶり_disable_foe_ability` / `かたやぶり_restore_foe_ability` を参照）。
    - こんらんの自傷ダメージ（`"_こんらん"`）は攻撃技扱いではないため、他の同種特性
      （ちからもち・とうそうしん等）と同様に技名で除外する。
- `src/jpoke/data/ability.py`（ファントムガード）
  - `h.マルチスケイル_reduce_damage` を共有利用する（`mold_breaker_ignorable` を
    持たない点のみ差分）。
- `src/jpoke/core/lethal.py`
  - `full_hp_damage_modifier` フラグを汎用的に見て、hp_dist の満タン枝/非満タン枝を
    分けてダメージ分布を計算する既存の仕組み（`_calc_damage_dist`）で対応済み。
    本特性専用のリーサルハンドラは不要（`data/ability.py` に `lethal_handlers` の
    個別エントリは存在しない）。
- `tests/abilities/test_ability_ma.py`
  - `test_マルチスケイル_HP満タンのとき半減`
  - `test_マルチスケイル_かたやぶりで無効`
  - `test_マルチスケイル_こんらんの自傷ダメージは半減しない`（レビューで追加。
    こんらん自傷ダメージ半減バグの回帰テスト）
- `tests/test_lethal.py`
  - `test_マルチスケイル_ダメージ半減`
  - `test_マルチスケイル_満タン非満タン混在時も枝ごとに正しく半減`
  - `test_多段技マルチスケイル_1ヒット目のみ半減`
  - `test_複数技_順に使用`（マルチスケイル所持ポケモンを使った複数技リーサル計算の確認）

## 実装フロー（実績）

1. 仕様書を一次情報（wiki）と照合し、内容の正確性を確認する。
2. `data/ability.py` の `マルチスケイル` に `mold_breaker_ignorable` /
   `full_hp_damage_modifier` の両フラグを付与する。
3. `Event.ON_CALC_DAMAGE_MODIFIER` へ `マルチスケイル_reduce_damage` を
   `subject_spec="defender:self"` で登録する。
4. レビューでこんらん自傷ダメージが誤って半減されるバグを検出し、
   `ctx.move.name != "_こんらん"` のガードを追加して修正する。
5. `tests/abilities/test_ability_ma.py` に回帰テストを追加する。

## テスト観点

1. 発動: HPが満タンのとき、攻撃技のダメージが0.5倍になる（`damage_modifier == 2048`）。
2. 非発動: HPが満タンでないとき、ダメージは半減しない（`damage_modifier == 4096`）。
3. 無効化: 攻撃側が `かたやぶり` を持つ場合は半減が適用されない
   （`damage_modifier == 4096`）。
4. 除外: こんらんの自傷ダメージ（`"_こんらん"`）はHPが満タンでも半減されない。
5. リーサル計算: 満タン枝と非満タン枝が混在する `hp_dist` でも、枝ごとに正しく
   半減が適用される（`tests/test_lethal.py` で担保）。
6. リーサル計算: 連続攻撃技は1ヒット目のみ半減し、2ヒット目以降は半減されない
   （`tests/test_lethal.py` の `test_多段技マルチスケイル_1ヒット目のみ半減`）。

## リスクと対策

- リスク: `マルチスケイル_reduce_damage` はファントムガードと共有されているため、
  この関数を変更すると両特性の挙動が連動して変わる。
  - 対策: 本関数を修正する際は、`tests/abilities/test_ability_ha.py` の
    ファントムガード系テストも合わせて確認する。
- リスク: こんらんの自傷ダメージ（`"_こんらん"`）を除外するガードを外す変更は、
  他の同種特性（ちからもち・とうそうしん・テクニシャン等）との一貫性を崩す。
  - 対策: `"_こんらん"` 除外は複数のハンドラで共通パターン化されているため、
    変更時は他ハンドラとの整合性を確認する。
