# 特性実装計画: じりょく

更新日: 2026-07-11

## 仕様参照

- `docs/spec/abilities/じりょく.md`

## 実装ゴール

- 相手のはがねタイプのポケモンの交代・にげるを防ぐ。
- ゴーストタイプを併せ持つ相手には効果がない（ゴーストタイプの性質が優先される）。
- 味方の交代には影響しない。

## 実装方針

- `Event.ON_CHECK_TRAPPED` にハンドラを登録する（`ありじごく`・`かげふみ` と同系統）。
- `subject_spec="source:foe"`: ハンドラの主体（じりょく所持ポケモン）から見て、交代しようとしている
  ポケモン（`ctx.source`）が相手側のときに発動する。
- 判定条件: `source` がはがねタイプであり、かつゴーストタイプでないとき交代を制限する
  （`HandlerReturn(value=True)`）。
  - タイプを変えるわざ/とくせい/テラスタルなどで現在のタイプが変化した場合も、判定時点の
    `source.has_type(...)` で現在のタイプを参照するため追加実装は不要。
- きれいなぬけがら・にげあし・けむりだま・テレポート例外・だっしゅつボタン等は `is_trapped` を利用する
  既存の共通処理（アイテム/技側のハンドラ）で解決されるため、じりょく側の実装は不要。

## テスト観点

1. はがねタイプの相手は交代できない。
2. はがねタイプでない相手は交代できる。
3. はがね・ゴースト複合タイプ（トリックオアトリート等でゴーストタイプが追加された場合）の相手は交代できる。
4. 味方の交代には影響しない。
