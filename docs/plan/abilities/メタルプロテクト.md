# 特性実装計画: メタルプロテクト

更新日: 2026-07-13

## 仕様参照

- `docs/spec/abilities/メタルプロテクト.md`

## 実装ゴール

- 相手から能力ランクを下げられなくなる（全能力対象）。
- 自分の技・効果によるランク低下（からをやぶる・じこあんじ・のろい等）は防げない。
- かたやぶり効果がある技を受けたときもランクが下がらない（クリアボディ/しろいけむりと異なり無視されない）。
- かたやぶりに無効化されない点を除き、クリアボディ/しろいけむりと完全に同一のバトル効果を持つ。

## 実装方針

- クリアボディ/しろいけむりと同じハンドラ関数 `クリアボディ_block_stat_drop`
  （`ON_BEFORE_MODIFY_STAT`, `subject_spec="target:self"`）を登録し、実装を共有する。
- `クリアボディ_block_stat_drop` は共通ヘルパー `_block_stat_drop_by_foe(value, ctx, stat=None)`
  を呼び出し、`stat=None` で全能力を対象に、相手由来（`source != target`）かつ低下方向（`v < 0`）の
  項目だけを取り除く。
- かたやぶり判定は `AbilityData.flags` に `mold_breaker_ignorable` を付与することで
  `かたやぶり_disable_foe_ability` 側の共通処理に委ねる仕組みだが、メタルプロテクトは
  かたやぶりに無視されない特性のため、この特性エントリには `flags` を付与しない
  （クリアボディ/しろいけむりのみ `flags={"mold_breaker_ignorable"}` を付与する）。

## 変更候補

- `src/jpoke/data/ability.py` — `メタルプロテクト` の `AbilityData` に
  `ON_BEFORE_MODIFY_STAT` ハンドラを登録（`flags` は付与しない）
- `src/jpoke/handlers/ability.py` — `クリアボディ_block_stat_drop` / `_block_stat_drop_by_foe`
  （クリアボディと共有、新規実装なし）
- `tests/abilities/test_ability_ma.py` — メタルプロテクト本体のテスト

## テスト観点

1. 相手の技・特性による全能力のランク低下（いかく等）を防ぐ。
2. 自己効果によるランク低下は防げない。
3. かたやぶりの効果がある技を受けてもランクが下がらない（クリアボディ/しろいけむりとの違い）。
4. かたやぶりに無効化されない点を除き、クリアボディと同一の挙動になること。
