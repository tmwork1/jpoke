# 特性実装計画: どしょく

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/どしょく.md`

## 実装ゴール

- じめんタイプの技（攻撃技・変化技を問わない）を受けたときに技を無効化し、最大HPの1/4を回復する（小数点以下切り捨て）。
- 対象は自分に向けられた（`target == "foe"`）じめん技のみ。まきびしなど場を対象とする技には発動しない。
- 連続攻撃技を受けた場合も無効化するが、回復は1回のみ。
- まもる等で防がれた場合は発動しない。
- みがわり状態でじめん技を受けた場合も発動して本体が回復する。
- クリアチャーム・フラワーベールなどランク低下無効化系の効果や、命中ランクがすでに最低の場合よりも、どしょくによる技自体の無効化が優先される。
- ふうせん・ひこうタイプによるじめんタイプ無効化よりもどしょくの発動が優先され、HPを回復できる。
- かたやぶり効果があるじめん技には無効化されない（`mold_breaker_ignorable`）。
- 使用者とのレベル差に関係無くじわれに対しても発動する。

## 実装方針

- 他のタイプ吸収特性（ちくでん・ちょすい・そうしょく・ひらいしん・よびみず・でんきエンジン・こんがりボディ・かんそうはだ）と同一の共通ヘルパー `_apply_type_absorb()`（`handlers/ability.py`）を利用する。
  - `ctx.move.target == "foe"` かつ `ctx.move.type == "じめん"` の場合のみ発動。これにより場を対象とする技（まきびし等）は自動的に除外される。
  - `heal_ratio=1/4` を渡し、無効化ログを出したうえで `battle.modify_hp(defender, r=1/4)` で最大HPの1/4回復する（HP満タン時は `modify_hp` が変化なしと判定し「効果がないようだ」旨のログになる）。
  - 連続技での複数回発動は呼び出し元（`Event.ON_BEFORE_APPLY_MOVE`、ヒットごとに評価）が、無効化された時点で `stop_event=True` を返し以降のヒット処理自体が中断されるため、実質的に回復は1回のみとなる（既存の型吸収特性と同様のパターン）。
  - ランク低下無効化系の判定（クリアチャーム・フラワーベール・ランク下限）や、めんえき等の状態異常無効化と同様、`Event.ON_BEFORE_APPLY_MOVE` で技自体を `stop_event=True` によりキャンセルするため、後続のランク変化・状態異常付与処理（いずれも `Event.ON_APPLY_MOVE` 以降）には到達せず、自然にどしょくの発動が優先される。
- `AbilityData` に `flags={"mold_breaker_ignorable"}` を設定し、かたやぶり・かがくへんかガスの影響を受ける（無効化を貫通される）ようにする。個別のかたやぶり判定はハンドラ内には書かない（コア側の共通処理に委譲）。
- 登録イベントは `Event.ON_BEFORE_APPLY_MOVE`（`subject_spec="defender:self"`）。技の無効化判定に使う既存イベントであり、他の型吸収特性と同一の優先度（デフォルト100）で問題ない。

## 既知の制約

- ゲーム内では第五世代以降、命中判定（回避率・命中率によるRNG判定）より前に特性判定が行われるため、どしょくは技の命中/回避に関わらず発動する（姿を隠している場合を除く）。使用者とのレベル差に関係無くじわれに対しても発動する、という記述も同じ性質。
  - 現状の実装では `MoveExecutor._execute_move()` が命中判定 (`_check_hit`) を `Event.ON_BEFORE_APPLY_MOVE` より先に行っているため、命中率100%未満のじめん技（じわれ含む）が乱数で外れた場合はどしょくが発動しない、というゲームと異なる挙動になる。
  - 実際に `test_option.accuracy=0` で検証し、この挙動を確認済み（外れた場合「技が外れた」ログになりどしょく非発動）。
  - これは他のタイプ吸収特性（ちくでん・ちょすい等）にも共通する既知の制約であり、`move_executor.py` の判定順序を変更する大きめの改修が必要（試験的に順序を入れ替えたところ、かわらわり・サイコファングの「命中時のみ壁を解除する」実装が `Event.ON_BEFORE_APPLY_MOVE` に依存しており壊れることを確認した）なため、本レビューのスコープ外として現状維持とする。
- ふうせん・ひこうタイプによるじめんタイプ無効化は `damage.py` の `calc_def_type_modifier`（`is_floating` 判定でタイプ相性補正を0にする）で処理され、これは `MoveExecutor._execute_move()` 内で `Event.ON_BEFORE_APPLY_MOVE` より前（`ON_TRY_MOVE_1` の直後）に評価されるため、ふうせん・ひこうタイプを持つポケモンがどしょくを持っている場合（改造構成でのみ起こりうる組み合わせ。ミミズズは専用特性でありひこうタイプでもふゆう持ちでもないため実戦では発生しない）、タイプ無効化が優先されてしまいどしょくは発動しない（HPも回復せず特性も明かされない）ことを実際に検証して確認した。
  - 「アイテムやタイプによる無効化よりもどしょくの発動が優先される」という一次情報の記載と異なる既知の制約。修正には `calc_def_type_modifier` または `_check_hit_by_type` へのどしょく（じめんタイプ吸収特性全般）を考慮したロジック追加が必要で、影響範囲の精査を要するため、本レビューのスコープ外として現状維持とする。
- クリアチャーム所持時・命中ランクがすでに最低のときにすなかけを受けた場合や、味方フラワーベールとの優先順位については、どしょくが `Event.ON_BEFORE_APPLY_MOVE` で技自体を無効化する実装のため、後続のランク変化処理に到達せず自然に仕様どおり動作することを実際に検証して確認した（追加のハンドラ実装は不要）。

## テスト観点

1. じめん技（攻撃技）を受けると無効化し、HPが減っていれば最大HPの1/4回復する。
2. じめんタイプの変化技（例: すなかけ）を受けても無効化し、ランク変化も発生しない。
3. クリアチャーム所持時でも、ランク低下無効化より先にどしょくが発動して回復する。
4. まきびしなど場を対象とするじめん技には発動しない。
5. じめん技の連続攻撃技を受けても、回復は1回のみ。
6. みがわり状態でじめん攻撃技を受けても発動して本体が回復する。
7. まもる状態でじめん技を防いだときは発動しない。
8. HPが満タンのときに発動した場合、効果がない旨のログになる（回復しない）。
9. かたやぶり・かがくへんかガスの影響下ではどしょくが無効化され、じめん技が通常通り命中する（既存の `test_タイプ無効回復` / `test_タイプ無効回復_かたやぶりで無効` グループテストで確認済み）。
