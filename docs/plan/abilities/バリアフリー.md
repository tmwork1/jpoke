# 特性実装計画: バリアフリー

更新日: 2026-06-05

## 仕様参照

- `docs/spec/abilities/バリアフリー.md`

## 実装ゴール

- 場に出たとき、敵味方全体のリフレクター・ひかりのかべ・オーロラベールを消す。
- すでに壁が一切ない場合はアナウンスなし。

## 実装方針

- `Event.ON_SWITCH_IN`、`subject_spec="source:self"` で実装。
- `battle.side_managers` を両サイドループし、各壁フィールドの `is_active` を確認。
- `SideFieldManager._deactivate_field(field)` で解除（プライベートメソッド。公開 API が追加された場合はそちらへ移行）。

## 変更候補

- `src/jpoke/data/ability.py` — `バリアフリー` に ON_SWITCH_IN handler を登録する
- `src/jpoke/handlers/ability.py` — `バリアフリー_on_switch_in` を実装する
- `tests/test_ability.py` — テストを追加する

## テスト観点

1. 相手側にリフレクターがあるとき → 入場で解除される。
2. 自分側にひかりのかべがあるとき → 入場で解除される。
3. 両サイドのオーロラベールが同時に解除される。
4. 壁がない場合 → アナウンスなし。

## 疑似コード

```python
_BARRIER_SCREENS = ("リフレクター", "ひかりのかべ", "オーロラベール")

def バリアフリー_on_switch_in(battle, ctx, value):
    mon = ctx.source
    triggered = False
    for side in battle.side_managers:
        for screen in _BARRIER_SCREENS:
            field = side.get(screen)
            if field.is_active:
                side._deactivate_field(field)
                triggered = True
    if triggered:
        announce_ability_triggered(battle, ctx, value, mon=mon)
    return HandlerReturn(value=value)
```
