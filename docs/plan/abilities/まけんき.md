# 特性実装計画: まけんき

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/まけんき.md`

## 実装ゴール

- 敵から能力ランクを下げられたとき、こうげきが2段階上昇する。
- 自分の技（自傷効果）や味方からの能力低下では発動しない。
- 複数の能力が同時に下がる技（くすぐる・おきみやげ等）を受けた場合、下がった能力の数だけ発動する。
- こうげきがすでに最大まで上がっている場合は特性バーが出ない（メッセージのみ、実装上は
  `battle.modify_stats` の戻り値が空になるため自然に非通知となる）。
- 下がった能力を元に戻す効果ではないため、こうげき以外の能力低下時は元の能力低下はそのまま残る。

## 実装方針

- `Event.ON_MODIFY_STAT`、`subject_spec="target:self"` で実装する。
- **かちき** と同一パターン。違いは上昇させる能力が C → A の点のみ。
- `ctx.source != ctx.target` で自傷チェック（本プロジェクトはシングルバトル専用のため、
  「味方からの低下では発動しない」は自傷チェックと別途の判定は不要。対戦相手は1体のみで、
  味方が同時に場に出ることはない）。
- 下がった能力の数だけ `battle.modify_stats(ctx.target, {"atk": +2}, source=ctx.source)` を
  繰り返し呼び出す（1回のみだと、くすぐる等の複数能力低下技で発動回数が不足する）。
- パワースワップ/ガードスワップ/ハートスワップ/ひっくりかえす/くろいきり/クリアスモッグ/
  バトンタッチは既存実装がいずれも `ON_BEFORE_MODIFY_STAT`/`ON_MODIFY_STAT` を経由しない
  直接ランク操作のため、まけんきは誤発動しない（実装済み・追加対応不要）。

## テスト観点

1. 相手のいかくを受けると A が 2 段階上がる。
2. 自分の技の追加効果（アームハンマー等）で能力が下がっても発動しない。
3. くすぐる等、複数能力が同時に下がる技を受けると下がった数だけ発動する（例: くすぐるで
   B・Dが同時に下がると A が4段階上がる）。
4. A がすでに最大のとき、能力を下げられても A はそれ以上上がらない（バー非表示）。
5. ミラーアーマーで能力低下が反射されたときも発動する。

def まけんき_boost_atk_on_stat_drop(battle, ctx, value):
    negative_count = sum(1 for v in value.values() if v < 0)
    if negative_count and ctx.source != ctx.target:
        for _ in range(negative_count):
            if battle.modify_stats(ctx.target, {"atk": +2}, source=ctx.source):
                announce_ability_triggered(battle, ctx.target)
