# 特性実装計画: ミイラ

更新日: 2026-06-03

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/ミイラ.md`
- 実装状態: 未実装
- 実装方針: `ON_DAMAGE_HIT` で直接攻撃によりダメージを受けたとき、攻撃者の特性を `ミイラ` に書き換える。上書き不可特性（uncopyable flag）には発動しない。

## 実装ゴール（完了定義）

- 直接攻撃でダメージを受けたとき、攻撃者の特性がミイラに書き換わる。
- みがわり・まもるでダメージが防がれた場合は発動しない。
- 上書きできない特性（ARシステム・かがくへんかガスなど）には発動しない。
- ミイラになった攻撃者が別のポケモンから直接攻撃を受けると、その相手もミイラになる（連鎖）。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_DAMAGE_HIT` — 直接攻撃でダメージを受けたとき、攻撃者の特性をミイラに書き換える

## 実装ステップ

2. `ミイラ_on_damage` を実装する（接触判定あり、ダメージあり、上書き可能特性のとき攻撃者の特性を `ミイラ` に書き換え）。

## テスト観点

1. 直接攻撃でダメージを受けると攻撃者の特性がミイラになる。
2. みがわりで防がれた場合は発動しない。
3. まもるで防がれた場合は発動しない。
4. uncopyable な特性（かがくへんかガスなど）の攻撃者には発動しない。
5. ミイラになった攻撃者が次に直接攻撃を受けると、その攻撃者もミイラになる（連鎖）。
