# 特性実装計画: くさのけがわ

更新日: 2026-07-11

## 仕様参照

- `docs/spec/abilities/くさのけがわ.md`

## 実装ゴール

- 場がグラスフィールド状態のとき、自分の防御（ダメージ計算で参照される実際のステータス）を1.5倍にする。
- 所有者が地面にいない（浮いている等）場合でもグラスフィールドの効果を受けずに発動する（＝設置状態や接地判定を条件にしない）。
- 通常特殊技には発動しないが、サイコショック/サイコブレイク/しんぴのつるぎのように防御を参照する特殊技には発動する。
- こんらんの自傷ダメージ（内部技 `_こんらん`）には発動しない。
- ワンダールーム中は防御と特防の参照が入れ替わるため、結果的に特防を1.5倍にする効果になる（追加実装は不要、既存のワンダールーム処理と組み合わせるだけで成立する）。
- ボディプレスで攻撃力として参照する自分の防御実数値には影響しない（`ON_CALC_ATK_MODIFIER` 側の処理であり、本特性は `ON_CALC_DEF_MODIFIER`・`defender:self` にのみ登録するため自動的に無関係になる）。
- かたやぶり・こだいのちからガス等で無効化される（`mold_breaker_ignorable` フラグ）。

## 実装対象イベント

- `Event.ON_CALC_DEF_MODIFIER`（`defender:self`）— ダメージ計算の防御補正を計算するタイミング。`docs/spec/turn.md` に個別記載なし（ダメージ計算内部イベント）。既存の同種特性（`ファーコート` 等、`ON_CALC_DEF_MODIFIER` に登録する特性群）と同じくデフォルト priority=100 とする。

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `くさのけがわ` に `Event.ON_CALC_DEF_MODIFIER` の handler を登録する（`flags={"mold_breaker_ignorable"}` 済み）
- `src/jpoke/handlers/ability.py` — `くさのけがわ_boost_B` を実装する
- `tests/abilities/test_ability_ka.py` — テストを追加する

## 実装ステップ

1. `battle.terrain.name == "グラスフィールド"` を確認する。
2. `ctx.move.name != "_こんらん"` でこんらんの自傷ダメージを除外する。
3. `battle.query.deals_physical_damage(ctx.attacker, ctx.move)` で「防御実数値を参照するダメージ計算か」を判定する（`move.category == "physical"` の技に加え、`physical_damage` フラグを持つ特殊技〈サイコショック等〉も対象になる）。
4. 条件を満たす場合 `apply_fixed_modifier(value, 6144)` で1.5倍にする。

## テスト観点

1. グラスフィールド中、物理技を受けると防御補正が1.5倍になる（`def_modifier == 6144`）。
2. グラスフィールドでないときは発動しない（`def_modifier == 4096`）。
3. 通常の特殊技（`physical_damage` フラグなし）には発動しない。
4. サイコショックのように防御を参照する特殊技には発動する。
5. こんらんの自傷ダメージには発動しない。
6. ワンダールーム中は物理技でも実質的に特防が1.5倍になる（防御/特防の入れ替えとの組み合わせ）。
7. かたやぶりで無効化される。

## リスクと対策

- リスク: こんらんの自傷ダメージ（`_こんらん`）も `category="physical"` のため、除外条件を書き忘れると誤って1.5倍が乗ってしまう。
  - 対策: `ctx.move.name != "_こんらん"` を明示的に条件へ追加する（他の防御・軽減系ハンドラ `リフレクター_reduce_damage` 等と同じパターン）。
