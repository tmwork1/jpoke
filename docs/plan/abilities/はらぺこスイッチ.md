# 特性実装計画: はらぺこスイッチ

更新日: 2026-07-12

## スコープ

- 優先度: M
- 仕様書: `docs/spec/abilities/はらぺこスイッチ.md`
- 実装状態: 実装済み
- 実装方針: モルペコの「まんぷくもよう / はらぺこもよう」は種族データを分けず、
  `Ability` インスタンスのブール値プロパティ `is_hangry` のみで管理する
  （タイプはどちらのフォルムでも でんき/あく で変化せず、フォルム差分は
  オーラぐるまの技タイプ判定にしか影響しないため、ダルマモード等のような
  別種族エントリは不要）。
  ターン終了時の交互切替は `ON_TURN_END`、交代時のリセットは `ON_SWITCH_OUT`
  で行い、オーラぐるまの技タイプ変換は特性側ではなく `オーラぐるま` の
  `ON_MODIFY_MOVE_TYPE` ハンドラで `ctx.attacker.ability.is_hangry` を
  直接参照して判定する（特性が消されていてもフォルムに応じたタイプに
  なる、という仕様を素直に満たせるため）。

## 実装ゴール（完了定義）

- ターン終了時に `is_hangry` が交互に反転する（まんぷく→はらぺこ→まんぷく…）。
- ターン終了時のフォルムチェンジは、スリップダメージ・状態異常解除・場の状態解除等
  （`ON_TURN_END` priority 10〜150）より後、かつダルマモード/リミットシールド/
  スワームチェンジ/ぎょぐんのフォルムチェンジ（priority 160）より後に発動する
  （すばやさに関係なく後）。
- はらぺこスイッチ持ち同士では、実効すばやさが高い方が先にフォルムチェンジする
  （トリックルーム下では逆転する。既存の `_sort_handlers` の
  `(priority, -speed)` ソートと `calc_effective_speed` に委譲）。
- テラスタル中はターン終了時のフォルムチェンジが発動しない（現在のフォルムを維持）。
- 交代時、テラスタル状態でなければ `is_hangry = False`（まんぷくもよう）に戻す。
  テラスタル状態で生きたまま交代した場合はフォルムを維持する。
  ただし、テラスタル状態でも **ひんしになった場合は例外的にまんぷくもように戻す**
  （一次情報: 「はらぺこもようでテラスタルした後に交代したときは、はらぺこもよう
  を維持する...テラスタルしてひんしになった場合は、まんぷくもように戻る」）。
- オーラぐるまは `is_hangry` が真のとき あくタイプ、それ以外は でんきタイプになる。
  この判定は特性の有効・無効（とくせいなし/かがくへんかガス）に関係なく、
  `is_hangry` の値のみで行う。
- この特性はコピー不可（うつしえ/なりきり等の一方向コピーで対象にできない＝
  `uncopyable` フラグ）。加えて、`スキルスワップ`/`さまようたましい` の
  双方向交換でも交換できない（`protected` フラグは持たないため、
  `かがくへんかガス`/`クォークチャージ`/`こだいかっせい`/`どくくぐつ` と同様に
  `スキルスワップ_can_apply` の `UNSWAPPABLE_ABILITIES` と
  `さまようたましい_swap_ability_on_contact` の base_name 個別判定に
  `はらぺこスイッチ` を追加する）。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_SWITCH_OUT` — 交代時のフォルムリセット（テラスタル中は維持）。
  `docs/spec/turn.md` に掲載なし（ダメージ計算外のフック）のため、既存の同種
  ハンドラ（`ダルマモード_revert_form` / `リミットシールド_revert_form`、
  いずれも priority 未指定＝デフォルト100）を参照し、同じくデフォルト優先度
  で登録する。
- `Event.ON_TURN_END` — ターン終了時の交互切替。`docs/spec/turn.md` L648-652 の
  「160 ダルマモード/リミットシールド/スワームチェンジ/ぎょぐんによる
  フォルムチェンジ」と同列に「160 はらぺこスイッチによるフォルムチェンジ」が
  記載されているが、一次情報に「すばやさに関係なくはらぺこスイッチが後に
  発動する」と明記されている。同じ priority=160 のままだと `_sort_handlers`
  の `(priority, -speed)` tie-break によりすばやさ次第で前後してしまうため、
  **priority=161** として必ず 160 グループの後に実行されるようにする
  （`なまけ`の`priority=15`と同様、turn.md記載の数値からコード側で意図的に
  ずらす前例に倣う）。
- `オーラぐるま`（技側, `Event.ON_MODIFY_MOVE_TYPE`） — `ctx.attacker.ability.is_hangry`
  が真なら「あく」、それ以外は元のタイプ（でんき）を返す。特性側イベントとしては
  登録しない（技のタイプ決定はダメージ計算の内部処理であり、モルペコの特性が
  かがくへんかガス等で無効化されていてもフォルムに応じたタイプになる仕様の
  ため、特性の有効・無効を問わず参照できる技側ハンドラで実装する）。

## 実装ステップ

1. `Ability` に `is_hangry` フラグを持たせる（実装済み）。
2. `はらぺこスイッチ_on_switch_out` を実装する（ひんし、またはテラスタル状態
   でなければ `is_hangry = False`。テラスタル状態かつ生存中のみ維持する）。
3. `はらぺこスイッチ_on_turn_end` を実装する（テラスタル状態なら何もしない。
   それ以外は `is_hangry` を反転する）。
4. `src/jpoke/data/ability.py` の `はらぺこスイッチ` に `ON_SWITCH_OUT`
   （デフォルト優先度）と `ON_TURN_END`（priority=161）を登録する。
5. `オーラぐるま_check_move_type` を実装し、`オーラぐるま` の
   `ON_MODIFY_MOVE_TYPE` に登録する（実装済み）。
6. `スキルスワップ_can_apply` の `UNSWAPPABLE_ABILITIES` タプルと
   `さまようたましい_swap_ability_on_contact` の base_name 除外条件に
   `はらぺこスイッチ` を追加し、交換不可にする。
7. テストを追加・修正し、全テストが通ることを確認する。

## 仕様反映チェックポイント

- 特性無効化（`かがくへんかガス` / `とくせいなし`）中はターン終了時の切替が
  停止する（`AbilityHandler` は `source="ability"` のため、
  `enabled_ignoring` チェックで自動的に無効化される。個別のガード実装は不要）。
- テラスタル中はターン終了時の切替、交代時のリセットのいずれも停止する。
- オーラぐるまのタイプ判定は特性の有効・無効を問わず `is_hangry` を直接参照する
  （特性が消されていてもフォルムに応じたタイプになる仕様のため）。
- 通常交代（コマンドによる交代）は `ON_TURN_END`（フォルムチェンジ判定）より
  前に行われるため、その場に出た同じターンの終わりに通常どおり判定される
  （＝一次情報の「そのターンから即座にはらぺこスイッチが発動する」を満たす。
  死に出し交代でその場に出た場合は、既にそのターンの `ON_TURN_END` を通過済み
  のため次のターン終了時まで判定されない。いずれもイベント発火順序から
  自然に満たされ、追加実装は不要）。
- へんしん個体はこの特性を持たない（コピー不可）ため対象外。

## 既知の制限（未対応）

- `is_hangry` は `mon.ability`（`Ability` インスタンス）上のフラグとして持つため、
  `change_ability`/`swap_ability` で `mon.ability` が新しい `Ability` インスタンスに
  差し替わると値がリセットされる。スキルスワップ/さまようたましいでの「交換」は
  本タスクで対象外化した（上記チェックポイント）ため影響しないが、モルペコが
  `なりきり` の使用者として自らの意思で はらぺこスイッチ を手放すケース
  （`protected` フラグを持たないため `なりきり_can_apply` は許可する）では、
  直前のフォルム状態が失われる可能性がある。一次情報は「特性やその効果を
  消された場合」の例として いえき/かがくへんかガス のみを挙げており、
  なりきりによる自発的な特性上書き時のフォルム維持まで明言していないため、
  本レビューでは仕様未確定として実装を見送った。今後 一次情報で明確化された
  場合に再検討する。

## テスト観点

1. ターン終了時に `is_hangry` が交互に反転する。
2. テラスタル中はターン終了時に切り替わらない。
3. テラスタル状態で（生きたまま）交代すると、`is_hangry` を維持したまま控えに戻る。
4. テラスタルしていない状態で交代すると `is_hangry = False` に戻る。
5. テラスタル状態でもひんしになった場合は `is_hangry = False` に戻る。
6. とくせいなし/かがくへんかガス中はターン終了時に切り替わらない。
7. `ON_TURN_END` の priority が、ダルマモード/リミットシールド/スワームチェンジ/
   ぎょぐんの `ON_TURN_END`（priority=160）より大きい（＝すばやさに関係なく後）。
8. オーラぐるまは `is_hangry = True` のとき あくタイプ、`False` のとき でんきタイプになる。
9. スキルスワップ・さまようたましいでこの特性を交換できない。
