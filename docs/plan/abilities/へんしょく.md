# 実装計画: へんしょく

作成日: 2026-06-04
更新日: 2026-07-12（レビューで実装済みの内容に合わせて修正）

## 効果
攻撃技のダメージを受けた後、その技と同じタイプになる。
以下の場合は発動しない:
- ひんしになったとき
- みがわり・まもる等でダメージが0だったとき（`ON_DAMAGE_HIT` は `actual_damage > 0` の
  ときにしか発火しないため、ハンドラ側での明示的なガードは不要）
- 連続攻撃技の最終ヒット以外（第五世代以降仕様: 全ヒット終了後にのみ変化）
- わるあがき（move.type == ""）
- すでにそのタイプを持つ（複合タイプの場合はいずれかのタイプが一致すれば発動しない）
- 特性ちからずくの効果が発動した技（攻撃側が有効な「ちからずく」を持ち、
  技が `secondary_effect` フラグを持つ場合）
- かがくへんかガスが発動している間（AbilityData に `gas_proof` フラグを付けない
  ことで、既存の特性抑制の仕組みにより自動的に無効化される）

## イベント／ハンドラ

`docs/spec/turn.md` の `Event.ON_DAMAGE` テーブルは実装上の `Event.ON_DAMAGE_HIT`
に相当する（`どくしゅ` 等の既存コメント参照）。

| イベント | priority | subject_spec | 関数名 |
|---------|---------|-------------|--------|
| `ON_DAMAGE_HIT` | 100（`turn.md`: 「へんしょく等（防御側特性）」、`AbilityHandler` のデフォルト値のため明示指定不要） | `defender:self` | `へんしょく_copy_move_type` |

## 実装

```python
def へんしょく_copy_move_type(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """へんしょく特性: 攻撃技を受けた後、その技のタイプになる。

    特性ちからずくの効果が発動した技（secondary_effect フラグを持つ技をちからずく
    所持者が使用した場合）を受けた場合はタイプが変化しない
    （docs/spec/abilities/へんしょく.md参照）。
    """
    mon = ctx.defender
    move_type = ctx.move.type
    if (
        mon.fainted
        or ctx.hit_index != ctx.hit_count
        or not move_type
        or move_type in mon.types
        or (
            ctx.attacker.ability.name == "ちからずく"
            and ctx.move.has_flag("secondary_effect")
        )
    ):
        return HandlerReturn(value=value)
    mon.ability_override_type = move_type
    _announce_ability_triggered(battle, mon)
    return HandlerReturn(value=value)
```

**補足**:
- `ON_DAMAGE_HIT` は `move_executor._execute_hit` で `actual_damage <= 0` の場合は
  発火しない（みがわり・まもる等で防がれた場合を含む）ため、ハンドラ側で
  ダメージ0を明示的に除外する必要はない。
- `ctx.move.type` はスキン系や `ON_MODIFY_MOVE_TYPE` 後の最終タイプを反映している前提
  （既存の `へんげんじざい` と同様）。
- `mon.types` は `ability_override_type` に加え `added_types`（もりののろい・ハロウィン等）
  を含めた現在のタイプを返すため、複合タイプ判定を追加で書く必要はない。
- `ctx.attacker.ability.name` は無効化されている場合は空文字を返す
  （`GameEffect.name` プロパティ）ため、かがくへんかガス等でちからずくが
  無効化されている場合は自動的に除外条件から外れる。

## data/ability.py 登録

```python
"へんしょく": AbilityData(
    handlers={
        Event.ON_DAMAGE_HIT: h.AbilityHandler(
            h.へんしょく_copy_move_type,
            subject_spec="defender:self",
        ),
    }
),
```

## テストケース
- 攻撃技を受けた後 → タイプが技のタイプに変化
- すでにそのタイプを持つ → 発動しない
- ひんし時 → 発動しない
- わるあがき → 発動しない（move.type == ""）
- 連続攻撃技（例: ダブルアタック）→ 最終ヒット後のみタイプ変化
- ちからずくの効果が発動した技 → 発動しない
- スキン変換後の技（例: フェアリースキン適用のハイパーボイス）→ 最終タイプに変化
