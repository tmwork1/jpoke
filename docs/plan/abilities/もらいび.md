# 特性実装計画: もらいび

更新日: 2026-07-13

## スコープ

- 仕様書: `docs/spec/abilities/もらいび.md`
- 実装状態: 実装済み
- 実装方針: ほのおタイプの技（対象が相手を狙うもの限定）を `Event.ON_BEFORE_APPLY_MOVE` で無効化し、
  自身の `ability.state`（`"idle" | "charged" | "active"`）を進めることで「もらいび状態」を管理する。
  次に自分がほのお技を使うときのみ `Event.ON_CALC_POWER_MODIFIER` で威力を1.5倍にし、
  行動終了時（`Event.ON_MOVE_END`）に強化状態を消費する。

## 実装ゴール（完了定義）

- ほのおタイプの技（相手を対象とするもの）を受けるとダメージ・追加効果（やけど等）をすべて無効化し、
  「もらいび状態」（`state == "charged"`）になる。
- にほんばれ等、場を対象とするほのお技には発動しない。
- まもる等、技自体を防ぐ効果でブロックされた場合はもらいびは発動しない（`ON_TRY_MOVE_1` で先に技が失敗するため）。
- もらいび状態になった後、最初に自分が出すほのお技のみ威力が1.5倍になる（攻撃・とくこうのステータス自体は変化しない）。
- 1.5倍の適用は1回限りで、行動終了後に等倍へ戻る。非ほのお技を挟んでも「もらいび状態」自体は維持される。
- 交代（退場）すると「もらいび状態」は消える（`ON_SWITCH_IN` で `idle` に初期化）。
- かたやぶり等 `mold_breaker_ignorable` を無視する特性を持つ相手には無効化されない（吸収できない）。

## 実装対象イベント

- `Event.ON_SWITCH_IN` — 登場時に `state` を `"idle"` にリセットする。
- `Event.ON_BEFORE_APPLY_MOVE` — 相手のほのお技を無効化し `state` を `"charged"` にする。
- `Event.ON_MOVE_CHARGE` — 自分がほのお技を使う際、`"charged"` なら `"active"` に遷移させる（強化予約）。
- `Event.ON_CALC_POWER_MODIFIER` — `state == "active"` のときのみ、ほのお技の威力を1.5倍にする。
- `Event.ON_MOVE_END` — 行動終了時に `"active"` を `"idle"` に戻す（強化の消費）。

## テスト観点

1. ほのお技を受けると無効化され、`state` が `"charged"` になる（かたやぶりでは無効化されない）。
2. `"charged"` 状態で最初に出したほのお技のみ威力1.5倍になり、以降は等倍に戻る。
3. 自分対象技（かえんのまもり等）を相手が使っても発動しない。
4. おにび等の状態異常技も無効化し、やけどにならない。
5. 非ほのお技を挟んでも `"charged"` 状態は維持される（次のほのお技で1回だけ強化）。
6. まもるで防がれた場合は発動しない。
7. 交代して戻ってくると `state` が `"idle"` にリセットされる。

## 補足: priority について

`ON_BEFORE_APPLY_MOVE`（`docs/spec/turn.md` 上は歴史的経緯で見出しが `Event.ON_APPLY_MOVE` のままの
節に該当。表内の「あくタイプ: いたずらごころで優先度を上げた変化技を無効化
（`Event.ON_BEFORE_APPLY_MOVE`）」の行が実体を示す）では大半のハンドラが `priority=100`
（デフォルト値）で登録されている。もらいびの `block_fire` も同様にデフォルト優先度のままとし、
既存の同種特性（ちくでん・ちょすい等）と揃えている。
`ON_MOVE_CHARGE` / `ON_MOVE_END` / `ON_CALC_POWER_MODIFIER` は `turn.md` に専用節がない
（ダメージ計算内部・技実行内部の補助イベントのため）。既存の同型特性
（もうか・げきりゅう等の `ON_CALC_ATK_MODIFIER` 登録、ちくでん・ちょすいの `ON_BEFORE_APPLY_MOVE`
登録）を参照し、同じくデフォルト優先度で問題ないと判断した。
