## 仕様まとめ（`docs/spec/abilities/メガソーラー.md` より）

1. 自分が行動するとき、天気がにほんばれ状態であるものとして技を使用する（実際の天候は変化しない）
2. ノーてんきのポケモンがいてもメガソーラーの効果は有効
3. ソーラービーム/ソーラーブレード: 溜めターンをスキップ（既存の溜めハンドラが `battle.weather.sunny` を参照するため自動対応）
4. ウェザーボール: 威力100のほのおタイプ（同上）
5. ダメージ補正: ほのおタイプ1.5倍、みずタイプ0.5倍
6. 命中補正: かみなり・ぼうふうが50%になる
7. こうごうせい/つきのひかり/あさのひざし: 最大HPの2/3回復
8. 岩タイプの特防補正・こおりタイプの防御補正を無視（各ハンドラが `battle.weather.name` を参照するため自動対応）
9. 相手のリーフガードを発動させない（リーフガード実装時に確認）

## 実装ゴール（完了定義）

- メガソーラー持ちが攻撃するとき、天候に関係なくほのお技が1.5倍・みず技が0.5倍になる
- ノーてんきが相手にいてもメガソーラーの効果が有効である
- 攻撃後（次のポケモンの行動前）に天候が元に戻る
- ソーラービームがはれ扱いで即撃てる（既存のソーラービームハンドラで対応）
- テストがすべて通る

## 実装方針

### なぜ直接書き換えか

`weather_manager.apply()` は以下の副作用を持つため使用しない：
- `ON_FIELD_ACTIVATE` / `ON_FIELD_DEACTIVATE` の発火（すなおこし等のカウントダウン等が動く）
- `ON_FIELD_CHANGE` の発火（こだいかっせい等の再判定が走る）
- ハンドラの登録・解除

直接 `weather_manager.current_name` と `fields["はれ"].count` を書き換えることで、
`battle.weather` / `battle.raw_weather` を参照する既存ハンドラが自動的に「はれ」として動作する。

### ノーてんき対策

`battle.weather` は `weather_manager.active` を呼び出し、`ON_CHECK_WEATHER_ENABLED` を発火する。
ノーてんきが有効なら `False` が返って inactive Field が返されるため、
`ON_CHECK_WEATHER_ENABLED` に `stop_event=True, value=True` を返すハンドラを登録する。
subject_spec は `"source:self"`。

### 発動・復元のタイミング

`move_executor.py` の `run_move` は `try/finally` で攻撃前後をブラケット。
現状このブラケットを利用できるイベントは `ON_BEGIN_MOVE` / `ON_END_MOVE`（priority: 既存の かたやぶりハンドラを参照）。

- **発動**: `ON_BEGIN_MOVE`（`"attacker:self"`）
  - 元の `weather_manager.current_name` と元 Field の `count` を `ability.state` に保存
  - `weather_manager.current_name = "はれ"` にセット
  - `weather_manager.fields["はれ"].count` を `1` にセット（`Field.name` が空文字を返すのを防ぐ）
- **復元**: `ON_END_MOVE`（`"attacker:self"`）
  - `ability.state` から元の `current_name` と `count` を復元

### 状態の保存形式

`ability.state` に以下のタプルを格納する：

```python
(元の current_name, 元の fields["はれ"].count)
```

## 実装ステップ

1. `handlers/ability.py` に3つのハンドラ関数を実装する
   - `メガソーラー_activate(battle, ctx, value)` — 天候を直接「はれ」に変更
   - `メガソーラー_deactivate(battle, ctx, value)` — 天候を元に戻す
   - `メガソーラー_override_weather_enabled(battle, ctx, value)` — `ON_CHECK_WEATHER_ENABLED` で True を返す
2. `data/ability.py` に登録する
3. テストを追加し、全テストが通ることを確認する

## テスト観点

- メガソーラー持ちが攻撃するとき: ほのお技が1.5倍になる（天候なしでも）
- メガソーラー持ちが攻撃するとき: みず技が0.5倍になる（天候なしでも）
- 攻撃後に天候が元の状態に戻る（あめ中なら攻撃後もあめのまま）
- ノーてんきが相手にいても: ほのお技が1.5倍になる
- 相手が攻撃するとき: 天候補正は元の天候のまま（メガソーラーの効果は相手に適用されない）
- ソーラービームを使用したとき: はれ扱いで即撃てる（溜めなし）

## 既知の考慮事項

- `ON_BEGIN_MOVE` / `ON_END_MOVE` を借用しているため、
  かたやぶり以外の用途で使われていることを明確にするコメントを付ける
- `ability.state` に元の天候情報を保存するため、メガソーラーはそれ以外の `ability.state` を使用しない
- リーフガードが実装済みの場合、リーフガードハンドラ側での対応が別途必要
