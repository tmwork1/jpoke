# 特性実装計画: マグマのよろい

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/マグマのよろい.md`

## 実装ゴール

- こおり状態にならない（付与をブロックする）。
- 特性が有効化された時点ですでにこおり状態なら即座に回復する
  （かがくへんかガス・かたやぶりの効果が終わって特性が再び有効になった場合や、
  なやみのタネ等で特性が書き換わった場合が対象）。
- かたやぶりの効果がある技でこおり状態にされた場合は、技終了直後（特性が再び有効になった
  タイミング）で回復する。

## 実装方針

- 状態異常の付与ブロックは `Event.ON_BEFORE_APPLY_AILMENT`（`target:self`）で行う。
  じゅうなん・めんえき・パステルベール・みずのベール・やるき等、既存の状態異常無効系特性と
  同じ `_prevent_ailment` 共通ヘルパー（`handlers/ability.py`）を `blocked_ailments=["こおり"]` で
  呼び出す `prevent_freeze_ailment` を用いる（新規実装ではなく既存関数の再利用）。
- 既にこおり状態のときに特性が有効化されるケース（かがくへんかガス/かたやぶりの終了、
  なやみのタネによる特性書き換え等）は `Event.ON_ABILITY_ENABLED`（`source:self`）で
  `_cure_ailment_on_enable` 共通ヘルパーを `blocked_ailments=["こおり"]` で呼び出す
  `マグマのよろい_cure_freeze_on_enable` で回復させる。
- `mold_breaker_ignorable` フラグを付与し、かたやぶり・きんしのちからの効果中は
  `ON_BEFORE_APPLY_AILMENT` ハンドラ自体が無効化される（`add_ability_disabled_reason`）
  ことで、こおり状態の付与を許可する。技終了後に特性が再有効化されると
  `ON_ABILITY_ENABLED` が発火し、上記の回復ハンドラで即座に回復する。
- `ON_BEFORE_APPLY_AILMENT` / `ON_ABILITY_ENABLED` はいずれも `docs/spec/turn.md` に priority が
  明記されていないイベント（状態異常付与・特性有効化の内部処理）のため、既存の同種ハンドラ
  （じゅうなん・めんえき・パステルベール・みずのベール・やるき等）に倣い priority は指定せず
  既定値を使う。
- どくびしとの相互作用（こおり状態で場に出るとどくびしの毒が不発してからマグマのよろいで
  回復する）は、どくびし側の「既に状態異常なら不発」判定とマグマのよろいの回復タイミングの
  順序で自然に成立するため、追加の実装は不要。
- ラムのみ/ナナシのみとの発動順序（第五世代以降は特性より先にきのみが発動して消費される）は
  きのみ側の状態異常回復トリガーとマグマのよろいの `ON_BEFORE_APPLY_AILMENT` ブロックが
  独立して動作するため、既存のきのみハンドラ・状態異常付与処理の発火順に委ね、本特性側での
  追加実装は不要。
- 進化・とくせいカプセル/とくせいパッチによる場外（非戦闘中）でのこおり→マグマのよろい取得後、
  次に場に出たときに回復する挙動は、jpoke が戦闘中の進化・場外での特性変更をサポートしない
  （単一バトルシミュレータでベンチ側ポケモンの特性変更手段が無い）ため対象外。
- フィールドでの効果（タマゴ孵化に必要な歩数半減）はバトルシミュレータの対象外のため実装しない。
