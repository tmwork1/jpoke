# 実装計画: ふしぎなまもり

作成日: 2026-06-04

## 効果
効果抜群でない攻撃技のダメージを無効化する。変化技・かたやぶり系は防げない。

## イベント／ハンドラ

| イベント | priority | subject_spec | 関数名 |
|---------|---------|-------------|--------|
| `ON_TRY_MOVE_1` | 110（`turn.md`） | `defender:self` | `ふしぎなまもり_block_non_effective` |

## flags

```python
"mold_breaker_ignorable"  # かたやぶり系で無視される
```

## 実装

```python
def ふしぎなまもり_block_non_effective(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """ふしぎなまもり特性: 効果抜群でない攻撃技を無効化する。

    タイプを持たない技（わるあがき）は相性判定の対象外のため無効化しない。
    """
    if (
        not ctx.move.is_attack
        or not ctx.move.type
        or battle.query.is_super_effective(ctx)
    ):
        return HandlerReturn(value=value)

    _announce_ability_triggered(battle, ctx.defender)
    battle.add_event_log(
        ctx.attacker,
        LogCode.MOVE_IMMUNED,
        payload=FailureLogPayload(move=ctx.move.name, display_reason="ふしぎなまもり")
    )
    return HandlerReturn(value=False, stop_event=True)
```

`battle.query.is_super_effective(ctx)` は `damage_calculator.calc_def_type_modifier(ctx)`
（`ON_CALC_DEF_TYPE_MODIFIER` を経由）を利用するため、フリーズドライのみずタイプ特効・
らんきりゅうによるひこう弱点無効化・でんじふゆう等による浮遊状態でのじめん無効化・
タイプなし状態（もえつきる等で `defender.types` が空になった場合は常に等倍＝ブロック）は
自動的に反映される。個別のハンドラ追加は不要。

こんらんの自傷ダメージ（`_こんらん`）は `battle.roll_damage()` で直接計算され、
`ON_TRY_MOVE_1` を経由する通常の攻撃フローを通らないため、この特性の対象外となる
（実装側で個別のガードは不要）。

## data/ability.py 登録

```python
"ふしぎなまもり": AbilityData(
    flags={"mold_breaker_ignorable"},
    handlers={
        Event.ON_TRY_MOVE_1: h.AbilityHandler(
            h.ふしぎなまもり_block_non_effective,
            subject_spec="defender:self",
            priority=110,
        ),
    }
),
```

## テストケース
- 等倍技 → 無効（ダメージ 0、特性発動ログが記録される）
- 今ひとつ技 → 無効
- 効果抜群技 → 通る（ダメージあり）
- 変化技（でんじは等）→ 通る
- かたやぶり持ちの攻撃 → 無視して通る
- タイプ無効（免疫）の技 → 相性判定より前なので ON_TRY_MOVE_1 で先に無効化
- わるあがき（タイプなし技）→ 相性判定の対象外のため無効化されない
- こんらんの自傷ダメージ → 通常の攻撃フローを経由しないため無効化されない
