# 特性実装計画: わざわいのおふだ

更新日: 2026-07-14

## スコープ

- 仕様書: `docs/spec/abilities/わざわいのおふだ.md`
- 実装状態: 実装済み
- 実装方針: `ON_CALC_ATK_MODIFIER` で、攻撃側が自分自身（わざわいのおふだの所有者）でなく、
  かつ攻撃側自身がわざわいのおふだを持っていない場合に、攻撃補正を0.75倍（3072/4096）にする。
  シングルバトル（場に2体のみ）が前提のため、「自分以外」の判定は
  `subject_spec="defender:self"` と `ctx.attacker is not ctx.defender` の組み合わせで表現できる。

## 実装ゴール（完了定義）

- わざわいのおふだ所有者以外の攻撃技の攻撃補正が0.75倍になる。
- わざわいのおふだ所有者自身の攻撃は補正されない。
- 場に複数体わざわいのおふだ所有者がいても、所有者同士は互いに効果を受けない
  （コピー・交換で入手した場合を含む）。
- かたやぶりで無効化されない。
- イカサマ・ボディプレス・こんらん自傷ダメージとの相互作用は、既存の一般的な
  ダメージ計算経路（`ON_CALC_ATK_MODIFIER` に渡る attacker/defender の実体）に
  委ねることで自然に仕様どおりになる（特別分岐は不要）。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_CALC_ATK_MODIFIER` — 攻撃側の攻撃補正（物理・特攻共通）を計算するタイミング
  （`docs/spec/turn.md` 未掲載のダメージ計算内部イベント。既存の `わざわいのつるぎ` 等
  同種ハンドラに合わせ、priority は指定せずデフォルト値 (100) を用いる）

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `わざわいのおふだ` に `ON_CALC_ATK_MODIFIER` handler
  （`subject_spec="defender:self"`）を登録する
- `src/jpoke/handlers/ability.py` — `わざわいのおふだ_reduce_A` を実装する
- `tests/abilities/test_ability_yarawa.py` — テストを追加する

## 実装ステップ

1. 仕様書 `docs/spec/abilities/わざわいのおふだ.md` で発動条件・例外条件を確認する。
2. `src/jpoke/handlers/ability.py` に `わざわいのおふだ_reduce_A` を実装する。
   - `ctx.attacker is not ctx.defender`（自分自身への効果を除外）かつ
     `ctx.attacker.ability.name != "わざわいのおふだ"`（攻撃側自身がわざわいのおふだを
     持つ場合を除外）のときのみ、`apply_fixed_modifier(value, 3072)` で0.75倍にする。
3. `src/jpoke/data/ability.py` の `わざわいのおふだ` に `ON_CALC_ATK_MODIFIER` handler を
   `subject_spec="defender:self"` で登録する。
4. `tests/abilities/test_ability_yarawa.py` にテストを追加し、全テストが通ることを確認する。

## テスト観点

1. わざわいのおふだ所有者以外が攻撃したとき、攻撃補正が0.75倍になる。
2. わざわいのおふだ所有者自身が攻撃したとき、自分の攻撃は補正されない。
3. わざわいのおふだ所有者同士が対峙するとき、互いに攻撃は補正されない。
4. かたやぶり所持の相手から攻撃を受けても効果が無効化されない。
