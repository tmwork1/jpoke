# 特性実装計画: どくくぐつ

更新日: 2026-06-05

## 仕様参照

- `docs/spec/abilities/どくくぐつ.md`

## 実装ゴール

- 相手を「どく」または「もうどく」状態にしたとき、同時にこんらん状態にする。
- 状態異常自体は付与後。こんらん付与はその直後。
- こんらん耐性（マイペース等）を持つ相手にはこんらん付与されない（volatile_manager が処理）。

## 実装方針

- `Event.ON_APPLY_AILMENT`、`subject_spec="source:self"` で実装。
- `source` = 状態異常を与えた側（自分）を subject にして、付与したのがどく/もうどくのとき `ctx.target` にこんらんを付与する。
- `value` の型（ailment 名か object か）は他の ON_APPLY_AILMENT ハンドラ（**シンクロ**）を参照して確認。

## 変更候補

- `src/jpoke/data/ability.py` — `どくくぐつ` に handlers を登録する
- `src/jpoke/handlers/ability.py` — `どくくぐつ_on_apply_ailment` を実装する
- `tests/test_ability.py` — テストを追加する

## テスト観点

1. どく付与の直後にこんらんも付与される。
2. もうどく付与でもこんらん付与される。
3. まひ等他の状態異常ではこんらん付与されない。
4. マイペース持ちにはこんらん付与されない。

def どくくぐつ_on_apply_ailment(battle, ctx, value):
    # value の型は実装時に ON_APPLY_AILMENT の他ハンドラ（シンクロ等）を参照
    ailment_name = value.name if hasattr(value, "name") else value
    if ailment_name not in ("どく", "もうどく"):
        return HandlerReturn(value=value)
    target = ctx.target
    if target is None:
        return HandlerReturn(value=value)
    battle.volatile_manager.apply(target, "こんらん", source=ctx.source)
    announce_ability_triggered(battle, ctx, value, mon=ctx.source)
    return HandlerReturn(value=value)
