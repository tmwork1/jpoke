# 特性実装計画: どくくぐつ

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/どくくぐつ.md`

## 実装ゴール

- 自身が使用した技で相手を「どく」または「もうどく」状態にしたとき、同時に2〜5ターンの
  こんらん状態にする。
- 状態異常自体は付与後。こんらん付与はその直後。
- どくびし・どくどくだま等の自傷（source と target が同一ポケモン）では発動しない。
- こんらん耐性（マイペース等）を持つ相手や、既にこんらん状態の相手にはこんらん付与されず、
  その場合は特性バーも表示されない（volatile_manager.apply_confusion の戻り値で判定）。

## 実装方針

- `Event.ON_APPLY_AILMENT`、`subject_spec="source:self"` で実装。
- `source` = 状態異常を与えた側（自分）を subject にして、付与したのがどく/もうどくのとき `ctx.target` にこんらんを付与する。
- `value` の型は string（ailment 名）。他の ON_APPLY_AILMENT ハンドラ（**シンクロ**）と同様。
- 自傷（どくびし・どくどくだま）を除外するため `ctx.is_foe_target()`
  （`source != target`）で判定する。シンクロ実装（`シンクロ_return_ailment`）が
  同様の除外に `ctx.is_foe_target()` を使っているのを踏襲する。
- こんらんの継続ターンは通常の混乱と同じ2〜5ターンのランダムのため、`volatile_manager.apply`
  を直接呼ばず `volatile_manager.apply_confusion` を使う（内部で `count = randint(2, 5)` を設定）。
- `apply_confusion` の戻り値が True のとき（＝実際に混乱を付与できたとき）のみ
  特性発動ログを出す（既に混乱状態・マイペース等で無効化された場合は特性バーを出さない）。
- 「この特性に対するスキルスワップは失敗する」は `handlers/move_status.py` の
  `スキルスワップ_can_apply` にある `UNSWAPPABLE_ABILITIES`（かがくへんかガス/
  クォークチャージ/こだいかっせい）に `どくくぐつ` を追加して対応する
  （どくくぐつは protected フラグを持たない＝上書きは可能だが、スキルスワップでの
  交換だけは対象外という既存パターンと同じ）。

## テスト観点

1. どく付与の直後にこんらんも付与される。
2. もうどく付与でもこんらん付与される。
3. まひ等他の状態異常ではこんらん付与されない。
4. マイペース持ちにはこんらん付与されない。
5. どくびし等の自傷（source == target）ではこんらん付与されない。
6. 付与されたこんらんの継続ターンが2〜5のランダムになる（apply_confusion 使用の確認）。
7. どくくぐつに対するスキルスワップが失敗する。

```python
def どくくぐつ_confuse_on_poison(battle, ctx, value):
    # value の型は ON_APPLY_AILMENT の他ハンドラ（シンクロ等）と同じく string
    if value not in ("どく", "もうどく"):
        return HandlerReturn(value=value)
    if not ctx.is_foe_target():  # source == target の自傷では発動しない
        return HandlerReturn(value=value)
    target = ctx.target
    if target is None:
        return HandlerReturn(value=value)
    if battle.volatile_manager.apply_confusion(target, source=ctx.source):
        _announce_ability_triggered(battle, ctx.source)
    return HandlerReturn(value=value)
```
