- 実装方針: シンクロ所持者がどく・まひ・やけど状態にされたとき、その状態にした相手にも同じ状態異常を付与する。`ON_AFTER_APPLY_AILMENT` 相当のイベントで発動させる。

## 実装ゴール（完了定義）

- 相手から受けた技でどく・まひ・やけど状態にされたとき、その相手にも同じ状態異常を付与する。
- かえんだま・どくどくだまによる自傷状態異常では発動しない。
- 相手がすでに状態異常の場合、タイプ耐性がある場合は発動しない。
- ねむり・こおり状態にされても発動しない。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_AFTER_APPLY_AILMENT`（または同等のイベント）— シンクロ所持者に状態異常が付与された後、付与した相手に同じ状態異常を試みる

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `シンクロ` に handler を登録する
- `src/jpoke/handlers/ability.py` — `シンクロ_return_ailment` を実装する
- `tests/test_ability.py` — テストを追加する

## 実装ステップ

1. 仕様書 `docs/spec/abilities/シンクロ.md` で発動条件（どく/まひ/やけどのみ、自傷除外）と例外条件を確認する。
2. 状態異常付与後のイベントが存在するか `src/jpoke/enums/event.py` で確認する。なければ追加し、`battle.apply_ailment` 相当の処理後に発火させる。
3. `src/jpoke/handlers/ability.py` に `シンクロ_return_ailment` を実装する。
   - `ctx.cause` が技（自分から付けた場合のみ）かを確認する。
   - `battle.try_apply_ailment(ctx.attacker, ailment)` で相手に返す。
4. `src/jpoke/data/ability.py` の `シンクロ` に handler を登録する。
5. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. 相手の技でどく状態にされたとき、相手もどく状態になる。
2. 相手の技でまひ状態にされたとき、相手もまひ状態になる。
3. 相手の技でやけど状態にされたとき、相手もやけど状態になる。
4. ねむり・こおり状態にされても発動しない。
5. かえんだまによる自傷やけどでは発動しない。
6. 相手がすでに状態異常の場合は発動しない（付与できない）。
