# 特性実装計画: ありじごく

更新日: 2026-07-10

## 仕様参照

- `docs/spec/abilities/ありじごく.md`

## 実装ゴール

- 相手の「地面にいる」ポケモン（ひこうタイプでなく、かつ浮いていない）の交代・にげるを防ぐ。
- ひこうタイプ、またはふゆうなど浮いている状態のポケモンは対象外。
- 第六世代以降はゴーストタイプの相手には効果がない。
- 味方の交代には影響しない。

## 実装方針

- `Event.ON_CHECK_TRAPPED` にハンドラを登録する（`かげふみ`・`じりょく` と同系統）。
- `subject_spec="source:foe"`: ハンドラの主体（ありじごく所持ポケモン）から見て、交代しようとしている
  ポケモン（`ctx.source`）が相手側のときに発動する。
- 判定条件: `source` がゴーストタイプでなく、かつ `battle.query.is_floating(source)` が `False`
  （= 地面にいる）のとき交代を制限する（`HandlerReturn(value=True)`）。
  - `is_floating` は `ON_CHECK_FLOATING` を発火し、ひこうタイプ・ふゆう・ふゆうユニット・でんきだま等の
    浮遊状態を総合判定する（既存の共通実装を利用するため、ありじごく側で個別に判定ロジックを持たない）。
- きれいなぬけがら・にげあし・けむりだま・テレポート例外・だっしゅつボタン等は `is_trapped` を利用する
  既存の共通処理（アイテム/技側のハンドラ）で解決されるため、ありじごく側の実装は不要。

## テスト観点

1. 地面にいる相手（通常ポケモン）は交代できない。
2. ひこうタイプの相手は交代できる。
3. （既存の `is_floating` 共通実装がカバーする）ふゆう特性・浮遊アイテム所持の相手は交代できる。
4. 味方の交代には影響しない。
