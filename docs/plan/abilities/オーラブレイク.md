# 特性実装計画: オーラブレイク

更新日: 2026-06-03

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/オーラブレイク.md`
- 実装状態: 部分実装（flags=[mold_breaker_ignorable] のみ設定済み）
- 実装方針: `ON_SWITCH_IN`・`ON_ABILITY_ENABLED` でアナウンスを出し、`ON_CALC_POWER_MODIFIER` でフェアリーオーラ/ダークオーラによる倍率を 1.33→0.75 に反転させる。フェアリーオーラ/ダークオーラの実装と一緒に進める。

## 実装ゴール（完了定義）

- 登場時に「すべての オーラを 制圧する!」メッセージが表示される。
- フェアリーオーラ/ダークオーラの威力 1.33 倍補正が、オーラブレイクがいる間は 0.75 倍に反転する。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_SWITCH_IN` — 登場時にアナウンスを出す
- `Event.ON_ABILITY_ENABLED` — 特性が有効化されたときにアナウンスを出す
- `Event.ON_CALC_POWER_MODIFIER` — フェアリーオーラ/ダークオーラの補正をオーラブレイクで反転させる

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `オーラブレイク` に handlers を登録する; `フェアリーオーラ` と `ダークオーラ` にも `ON_CALC_POWER_MODIFIER` handler を登録する
- `src/jpoke/handlers/ability.py` — `オーラブレイク_on_switch_in`, `オーラブレイク_reverse_aura_power` (および `フェアリーオーラ_boost_power`, `ダークオーラ_boost_power`) を実装する
- `tests/test_ability.py` — テストを追加する

## 実装ステップ

1. 仕様書でオーラブレイクの反転ロジック（1.33→0.75 の計算）を確認する。
2. フェアリーオーラ/ダークオーラの `ON_CALC_POWER_MODIFIER` handler を実装する（場にオーラブレイクがいなければ 1.33 倍、いれば 0.75 倍を返す）。
3. `src/jpoke/handlers/ability.py` に `オーラブレイク_on_switch_in`（アナウンス）を実装する。
4. フェアリーオーラ/ダークオーラ・オーラブレイクの handlers を `src/jpoke/data/ability.py` に登録する。
5. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. フェアリーオーラがいるときフェアリー技の威力が 1.33 倍になる。
2. フェアリーオーラ + オーラブレイクがいるときフェアリー技の威力が 0.75 倍になる。
3. ダークオーラ + オーラブレイクがいるときあく技の威力が 0.75 倍になる。
