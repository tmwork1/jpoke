# 特性実装計画: しめりけ

更新日: 2026-05-03

## 仕様参照

- `spec/ability/しめりけ.md`

## 実装ゴール

- しめりけ所持ポケモンが場にいる間、全ポケモンの爆発技（じばく・だいばくはつ・ミストバースト）を
  失敗させる（ビックリヘッドは第9世代未収録のため実装対象外。`docs/progress/move.md` 参照）。
- 相手の特性ゆうばくを無効化する。

## 実装方針

- 対象はシングルバトルのみのため、場にいるポケモンは攻撃側・防御側の2体に限られる。
  爆発技（`move.has_flag("explosion")`）の失敗判定は `Event.ON_TRY_MOVE_1`（priority=30、
  `docs/spec/turn.md` 参照）で攻撃側・防御側それぞれの特性ハンドラとして実装する。
  - `attacker:self` ロール: 自分自身が使う爆発技を失敗させる。
  - `defender:self` ロール: 相手が使う爆発技を失敗させる。
  - しめりけには `mold_breaker_ignorable` フラグを付与し、かたやぶり・特性を無視する技・
    かがくへんかガスによる無効化は既存の特性無効化の仕組み（`battle.add_ability_disabled_reason`
    経由の `mon.ability.enabled`）に委ねる。ハンドラ内で個別に判定しない。
  - 失敗時は `_announce_ability_triggered` でしめりけ所持ポケモンの特性を公開・発動ログを出し
    （じょおうのいげん等 `Event.ON_TRY_MOVE_1` の他の技失敗系ハンドラと同様のパターン）、
    `LogCode.MOVE_FAILED` で技使用側に失敗ログを出す。
- ゆうばく無効化は、ゆうばく側の特性ハンドラ（`ゆうばく_damage_attacker_on_ko`、
  `Event.ON_MOVE_KO`）内で `attacker.ability.enabled and attacker.ability.name == "しめりけ"`
  を判定し、成立する場合は発動をスキップする形で実装する
  （`_blocked_by_ougon_no_karada` と同様、他特性の存在を直接参照するパターン）。
  `.enabled` を経由することで、既存の特性無効化の仕組み（`mon.ability.enabled`）と整合させる。
  無効化時はしめりけ側・ゆうばく側いずれの特性発動ログも出さない。
  - 一次情報にはトリプルバトルでの対角ポケモン挙動やかたやぶり技で倒したときの
    ゆうばく発動（しめりけ無視）についての記載があるが、これらはダブル・トリプル
    バトル固有の挙動であり、本プロジェクトの対象範囲（シングルバトルのみ）では
    再現不可能なため実装・テストの対象外とする。

## テスト観点

1. じばく・だいばくはつ・ミストバーストが失敗する（ビックリヘッドは対象外実装のため対象外）。
2. 自分自身の爆発技も失敗する。
3. 爆発技のPPは消費される（失敗扱い）。
4. 攻撃側がしめりけを持つ場合、相手のゆうばくが無効化される。
5. かたやぶりの技には無視されて爆発技が成功する。
