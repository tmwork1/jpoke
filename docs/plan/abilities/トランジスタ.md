# 特性実装計画: トランジスタ

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/トランジスタ.md`
- `docs/spec/abilities/はがねつかい.md`（同型の特性仕様を参照）

## 実装ゴール

- でんきタイプの攻撃技を使うとき、自分のこうげき・とくこう（技の分類に応じてダメージ計算で
  参照される方）を1.3倍にしてダメージを計算する。
- タイプ一致ボーナス（STAB）とは補正箇所が異なるため両立する（ダメージ計算の別ステップで
  乗算される）。
- タイプが変わる技・技のタイプを変える技によって使用技のタイプが変わった場合、変更後の
  タイプがでんきタイプであれば効果がある（`ctx.move.type` は変更後のタイプを参照するため、
  追加対応は不要）。

## 実装方針

- `Event.ON_CALC_ATK_MODIFIER`、`subject_spec="attacker:self"` で実装する。
- priority は turn.md に記載のない内部（ダメージ計算内）イベントのため、同一イベント・同一
  `subject_spec="attacker:self"` の既存ハンドラ（**いわはこび**の `いわはこび_modify_atk`、
  **はがねつかい**の `はがねつかい_modify_atk`、**りゅうのあぎと**の `りゅうのあぎと_modify_atk`）
  に合わせ、`AbilityHandler` のデフォルト値 `priority=100` を使用する。
- `ctx.move.type` が `"でんき"` のとき `apply_fixed_modifier(value, 5325)`（4096 基準で 1.3 倍）を
  適用する。共通ヘルパー `_modify_by_move_condition(move, value, modifier=5325, move_type="でんき")`
  を使う（いわはこび・はがねつかい・りゅうのあぎととの重複実装を避ける）。

## 変更候補

- `src/jpoke/data/ability.py` — `トランジスタ` に `Event.ON_CALC_ATK_MODIFIER` ハンドラを登録する
  （実装済み）。
- `src/jpoke/handlers/ability.py` — `トランジスタ_modify_atk` を実装する（実装済みだが、
  `_modify_by_move_condition` を使わず if 文で直接判定している点を同系列ハンドラに合わせて
  修正する）。
- `tests/abilities/test_ability__group.py` — `test_タイプ強化系` にてカバー（実装済み）。

## テスト観点

1. でんきタイプの技を使ったとき `atk_modifier` が 5325（1.3倍）になる。
2. でんきタイプ以外の技を使ったときは `atk_modifier` が 4096 のまま変化しない。
3. タイプ一致ボーナス（STAB）とは別枠で補正されるため、両立する。
