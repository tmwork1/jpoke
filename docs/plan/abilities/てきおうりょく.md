# 特性実装計画: てきおうりょく

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/てきおうりょく.md`

## 実装ゴール

- タイプ一致ボーナス（STAB）を1.5倍ではなく2.0倍（8192/4096）にする。
- テラスタルとの兼ね合い:
  - テラスタイプが元タイプと一致する場合は2.25倍（9216/4096）になる。
  - テラスタイプが元タイプと不一致で、技のタイプがテラスタイプに一致する場合は2.0倍になる。
  - **テラスタルにより失った元タイプの技（元タイプには一致するがテラスタイプには
    一致しない技）には効果が発動しない**。STAB補正は元タイプ一致による1.5倍のまま
    据え置かれる（2.0倍にはならない）。
  - ステラタイプにテラスタルした場合は効果が発動しない（ステラ補正の有無・元タイプの
    一致有無に関係なく、特性を持たないポケモンと同じ値になる）。

## 実装方針

- `Event.ON_CALC_ATK_TYPE_MODIFIER`、`subject_spec="attacker:self"` で実装する
  （攻撃フェーズの `AttackContext` のため `attacker:self` が正しいロール指定）。
- priority は `docs/spec/turn.md` に記載のない内部（ダメージ計算内）イベントのため、
  `AbilityHandler` のデフォルト値 `priority=100` を使用する。この事象を対象とする
  ハンドラは本特性の1件のみであり（`ON_CALC_ATK_TYPE_MODIFIER` を利用する他の特性が
  存在しない）、複数ハンドラの実行順を調整する必要がないため根拠として妥当。
- `HandlerReturn` は `value` のみを更新して返す（`stop_event` は使用しない、後続の
  ハンドラをブロックする理由がないため）。
- `core/damage.py` の `_calc_atk_type_modifier` が計算するベース値（4096/6144/8192、
  テラスタル・ステラ補正込み）を `ON_CALC_ATK_TYPE_MODIFIER` イベントで受け取り、
  そこから補正する方針とする。ただし **ベース値の数値（6144 か 8192 か）だけでは
  「テラタイプ一致による6144」と「テラスタルで失った元タイプ一致による6144」を
  区別できない**（どちらも6144になる）ため、ハンドラ内で `ctx.attacker.active_tera_type`
  と `ctx.move.type` を直接比較し、テラスタル中かつ技タイプがテラスタイプと不一致の
  場合は無条件で無補正（元の値をそのまま返す）とする。
- ステラタイプへのテラスタル時は既存のステラ判定（`active_tera_type == "ステラ"`）で
  無補正にする。

## テスト観点

1. 通常時（非テラスタル）、タイプ一致技のSTABが2.0倍（8192）になる。
2. 通常時、不一致タイプの技には補正がかからない（4096のまま）。
3. テラスタルで元タイプと一致するタイプにテラスタルした場合、そのタイプの技の
   STABが2.25倍（9216）になる。
4. テラスタルで元タイプと異なるタイプにテラスタルした場合、テラスタイプに一致する
   技のSTABは2.0倍（8192）になる。
5. **テラスタルで元タイプと異なるタイプにテラスタルした場合、テラスタルにより
   失った元タイプ（テラスタイプには一致しないが元タイプには一致する）の技は
   1.5倍（6144）のまま2.0倍に補正されない。**
6. ステラタイプにテラスタルした場合、てきおうりょくの効果が発動しない
   （ステラ補正込みのベース値のまま変化しない）。

## 疑似コード

```python
def てきおうりょく_modify_stab(battle, ctx, value):
    attacker = ctx.attacker
    tera_type = attacker.active_tera_type

    if tera_type == "ステラ":
        return HandlerReturn(value=value)

    # テラスタル中、技タイプがテラスタイプと不一致なら発動しない
    # （テラスタルで失った元タイプの技のため）
    if tera_type and ctx.move.type != tera_type:
        return HandlerReturn(value=value)

    if value == 6144:
        return HandlerReturn(value=8192)  # 1.5倍 -> 2倍
    if value == 8192:
        return HandlerReturn(value=9216)  # 2倍 -> 2.25倍
    return HandlerReturn(value=value)
```

## リスクと対策

- リスク: `_calc_atk_type_modifier` のベース値計算（`core/damage.py`）が将来変更され、
  6144/8192 の意味（どの一致パターンに対応するか）が変わった場合、本ハンドラの
  `tera_type` と `ctx.move.type` を直接比較するロジックとズレる可能性がある。
  - 対策: ベース値の数値だけで判定せず、テラスタイプと技タイプの一致を直接
    再計算する現行方針を維持する（ベース値の内部実装に依存しすぎない）。
