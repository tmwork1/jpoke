# 特性実装計画: てんきや

更新日: 2026-06-03

## ユーザーコメント
ポワルンの姿違いをdata/pokedex.jsonに追加した


## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/てんきや.md`
- 実装状態: 部分実装（flags=[uncopyable] のみ設定済み）
- 実装方針: `ON_SWITCH_IN` と `ON_FIELD_CHANGE`（天気変化）で天気に対応したフォルムへ即時フォルムチェンジする。ノーてんき/エアロック発動中は通常フォルムに戻す。かがくへんかガス発動中は変化できない。

## 実装ゴール（完了定義）

- にほんばれ/おおひでり → ほのおタイプの「たいようのすがた」
- あめ/おおあめ → みずタイプの「あまみずのすがた」
- ゆき → こおりタイプの「ゆきぐものすがた」
- それ以外 → ノーマルタイプの「ポワルンのすがた」
- ノーてんき/エアロック発動中は通常フォルムに固定される。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_SWITCH_IN` — 登場時に現在の天気に合わせたフォルムに変化する
- `Event.ON_FIELD_CHANGE` — 天気が変わったとき即座にフォルムを切り替える

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `てんきや` に handlers を登録する
- `src/jpoke/handlers/ability.py` — `てんきや_sync_form` を実装する
- `tests/test_ability.py` — テストを追加する

## 実装ステップ

1. 仕様書で天気 → フォルム/タイプのマッピングを確認する。
2. `てんきや_sync_form` を実装する（現在の天気に対応したフォルムへ `battle.change_form`）。ノーてんき/エアロック発動中は常に「ポワルンのすがた」に戻す。
3. `src/jpoke/data/ability.py` の `てんきや` に `ON_SWITCH_IN`（priority=140: turn.md L111）と `ON_FIELD_CHANGE` handler を登録する。
4. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. にほんばれ状態で登場するとたいようのすがたになる。
2. バトル中に天気があめに変わると即座にあまみずのすがたになる。
3. ノーてんき/エアロックが発動中はフォルムが変化しない（通常フォルムに固定）。
