# 特性実装計画: くいしんぼう

更新日: 2026-07-11

## スコープ

- 仕様書: `docs/spec/abilities/くいしんぼう.md`
- 実装状態: 未実装
- 実装方針: くいしんぼう専用のイベントハンドラは追加せず、既存のHP閾値系きのみ（フィラのみ等14種）の
  発動判定処理内で `mon.ability.name == "くいしんぼう"` を判定し、発動閾値を1/4以下→1/2以下に
  緩和する（ちからずく等と同様のインライン判定パターン）。

## 実装ゴール（完了定義）

- くいしんぼう所持時、対象14種のきのみ（フィラ・ウイ・マゴ・バンジ・イア・チイラ・リュガ・カムラ・
  ヤタピ・ズア・サン・スター・ミクル・イバンの各のみ）がHP1/4超-1/2以下で発動する。
- くいしんぼう非所持時は従来通りHP1/4以下でのみ発動する（回帰なし）。
- もとから1/2以下で発動するきのみ（オボンのみ等）には影響しない。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_HP_CHANGED` — 対象14種のきのみが登録済みの既存イベント（変更なし、判定条件のみ変更）

## 変更対象（ファイル/関数単位）

- `src/jpoke/handlers/item.py`
  - `_gluttony_denominator(mon, denominator)` を新規追加。denominator=4のときのみ
    くいしんぼう所持で2に緩和するヘルパー。
  - `_heal_berry` — `denominator` をこのヘルパー経由に変更（イアのみ・ウイのみ・バンジのみ・
    フィラのみ・マゴのみが対象）。
  - `_boost_on_quarter_hp` — ハードコードされた `mon.hp * 4` をヘルパー経由に変更
    （カムラのみ・ズアのみ・チイラのみ・ヤタピのみ・リュガのみが対象）。
  - `イバンのみ_set_priority_flag` / `サンのみ_apply_focus_energy` /
    `スターのみ_random_boost` / `ミクルのみ_set_accuracy_flag` — 個別にハードコードされた
    `* 4` 判定をヘルパー経由に変更。
- `src/jpoke/data/ability.py` — `くいしんぼう` は専用ハンドラを持たないため変更なし
  （インライン判定の理由をコメントで明記）。
- `tests/abilities/test_ability_ka.py` — テストを追加する。

## テスト観点

1. くいしんぼう所持時、HP1/4超-1/2以下でイアのみ等の回復系きのみが発動する（1/4以下では
   従来通り発動していたことの回帰にはならない）。
2. くいしんぼう所持時、HP1/2超では発動しない。
3. くいしんぼう非所持時は、HP1/4超-1/2以下では発動せず、1/4以下で発動する（従来通り）。
4. くいしんぼう所持時でも、もとから1/2以下で発動するきのみ（オボンのみ）の発動閾値は変化しない。
5. くいしんぼう所持時、ランク上昇系きのみ（カムラのみ等）・きゅうしょ上昇（サンのみ）・
   ランダムランク上昇（スターのみ）・先制フラグ（イバンのみ）・命中率フラグ（ミクルのみ）が
   HP1/2以下で発動する。

## リスクと対策

- リスク: `_heal_berry` の `denominator` は元々オボンのみ（denominator=2）でも共用されている。
  くいしんぼうの緩和をdenominator=4の場合のみに限定しないと、オボンのみの閾値まで誤って
  変化してしまう。
  - 対策: `_gluttony_denominator` はdenominator=4のときのみ2に変換し、それ以外は素通しする。
- リスク: 特性が後から書き換わった瞬間の再判定・場に出た瞬間に既にHP1/4超-1/2以下である場合の
  即時発動・かがくへんかガス解除後のイバンのみ限定発動は、既存のHP閾値系きのみ全般が対応して
  いない（`Event.ON_HP_CHANGED` のみに依存する現行実装の構造的な制約）。
  - 対策: 本特性固有の欠落ではないため対象外とし、仕様書の備考に明記した。
