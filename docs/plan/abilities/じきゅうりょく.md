# 特性実装計画: じきゅうりょく

更新日: 2026-06-05

## 仕様参照

- `docs/spec/abilities/じきゅうりょく.md`

## 実装ゴール

- 攻撃技のダメージを受けたとき（物理・特殊問わず）、ぼうぎょが1段階上がる。
- 天候・状態異常等のスリップダメージでは発動しない。
- B が最大ランクのときは発動しない。
- 連続攻撃技は1発ヒットするたびに発動する。
- 相性/まもるなどでダメージを無効化したとき、みがわり状態で攻撃を防いだときは発動しない。
- こらえるでHP1のまま耐えたときなど、実HPダメージが0でも攻撃自体が無効化されていなければ発動する。

## 実装方針

- `Event.ON_HIT`（`docs/spec/turn.md` の「Event.ON_HIT」＝ダメージ発生後の処理。みがわりに
  被弾しても発動する）、`subject_spec="defender:self"`、priority は明示せずデフォルト。
  `Event.ON_DAMAGE_HIT`（`docs/spec/turn.md` の「Event.ON_DAMAGE」に相当）は
  `actual_damage<=0` のとき発火しないため採用しない。こらえるでHP1のまま耐えたときなど
  （実HPダメージ0）も発動する仕様（`docs/spec/abilities/じきゅうりょく.md`）を満たすため、
  常に発火する `Event.ON_HIT` を使用する。みがわりに阻まれた場合は `ctx.substitute_damage`
  を見て除外する（**こぼれダネ** と同パターン）。
- `battle.modify_stats(mon, {"def": +1}, source=ctx.attacker)` の成否でアナウンスを制御。
- ぼうぎょが最大ランクのときは `modify_stats` が変化なし（偽値）を返すため、追加の閾値
  チェックは不要（**ぎゃくじょう** と同パターン）。

## テスト観点

1. 物理技を受けると B が 1 段階上がる。
2. 特殊技を受けても B が 1 段階上がる。
3. B が最大（+6）のときは発動しない。
4. 連続攻撃技: 1発ごとに B が上がる。
5. こらえるでHP1のまま耐えたとき（実HPダメージ0）でも発動する。
6. みがわり状態で攻撃を防いだときは発動しない。

## 疑似コード

```python
def じきゅうりょく_boost_B_on_hit(battle, ctx, value):
    if ctx.substitute_damage:
        return HandlerReturn(value=value)
    mon = ctx.defender
    if battle.modify_stats(mon, {"def": +1}, source=ctx.attacker):
        announce_ability_triggered(battle, mon)
    return HandlerReturn(value=value)
```

## リーサル計算

`LethalEvent.ON_HIT`（subject="defender"）で `じきゅうりょく_boost_def` を登録する
（§共通11 のパターン。シグネチャ `(battle, ctx, hp_dist) -> StateDist`）。
`ctx.defender.rank["def"]` が6以上なら何もせず `hp_dist` を返し、そうでなければ
`clamp_stats(ctx.defender.rank["def"] + 1)` でランクを更新して `hp_dist` をそのまま返す。
