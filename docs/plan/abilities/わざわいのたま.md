# 特性実装計画: わざわいのたま

更新日: 2026-07-14

## スコープ

- 仕様書: `docs/spec/abilities/わざわいのたま.md`
- 実装状態: 実装済み（本レビューで防御側ステータス判定の欠落を修正）
- 実装方針: `Event.ON_CALC_DEF_MODIFIER` にハンドラを登録し、`subject_spec="attacker:self"`
  でこの特性の所持者が攻撃側のときに発動する。防御側で参照されるステータスが特防（とくぼう）に
  決まった技のみ、防御側の防御補正を0.75倍にする。自分自身が防御側でもある場合（こんらんの
  自傷等）は対象外。

## 実装ゴール（完了定義）

- 場にいる間、この特性を持つポケモン以外の全員が特防を参照して受けるダメージ計算時、
  防御側の防御補正が0.75倍になる。
- 防御（ぼうぎょ）を参照する技（分類が物理の技、およびサイコショック/サイコブレイク/
  しんぴのつるぎのように分類は特殊でも防御側でぼうぎょを参照する技）には効果が及ばない
  （とくぼうを参照しないため）。
- この特性の所持者自身が防御側であるとき（自分が受けるダメージ、こんらんの自傷を含む）は
  対象外（自分自身は効果を受けない）。
- かたやぶりで無視できない（`AbilityData.flags` に `mold_breaker_ignorable` を付与しない）。
- みらいよち/はめつのねがいのダメージは、場にわざわいのたまのポケモンがいるなら技の使用者が
  場にいなくても特性の効果でダメージが下がる（`Event.ON_CALC_DEF_MODIFIER` はダメージ着弾時点の
  場の状態で評価されるため、追加対応は不要）。
- この特性のポケモンがあなをほる状態になっても特性の効果は切れない（場にいる間は継続する仕様
  どおりで、揮発性状態の変化では特性は失われないため追加対応は不要）。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_CALC_DEF_MODIFIER` — 防御側の防御補正（final_defense への乗算補正）を計算する際に
  発火。`subject_spec="attacker:self"` で、この特性の所持者が攻撃側のときのみ発動する。

## リスクと対策

- リスク: みらいよち/はめつのねがいは、技使用時（溜め時点）の `battle.roll_damage` で
  ダメージ量を確定して `Field.damage` に保存し、2ターン後の着弾時にはその保存値をそのまま
  適用する実装になっている（`handlers/move_attack.py` の `みらいよち_charge` /
  `はめつのねがい_charge`）。そのため、わざわいのたま所持ポケモンが溜め時点では場におらず、
  着弾時点で場に出てきた場合（あるいはその逆）でも、防御補正は溜め時点の場の状態で固定される。
  仕様書は「技の使用者が場にいなくても」「着弾時点」を基準にした記述であり、本来は着弾時点の
  場の状態を再評価すべきだが、この時間差再評価は みらいよち/はめつのねがい 側の実装
  （技本体の計画・実装範囲）に関わる論点であり、わざわいのたま 単体の特性実装では対応しない。
  今後 みらいよち/はめつのねがい 側のレビューで着弾時点の再計算に対応する場合、本特性側の
  ハンドラ（`subject_spec="attacker:self"`）はそのまま動作する想定。
