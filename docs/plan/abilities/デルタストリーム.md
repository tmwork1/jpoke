# 特性実装計画: デルタストリーム

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/デルタストリーム.md`
- `docs/spec/fields/らんきりゅう.md`（らんきりゅう状態の詳細）

## 実装ゴール

- 場に出たとき（`ON_SWITCH_IN`）天候を「らんきりゅう」にする。
- とくせいが再度有効化されたとき（`ON_ABILITY_ENABLED`）も同様に発動する。
- らんきりゅう中は、ひこうタイプに対するでんき/いわ/こおりタイプの技のダメージを0.5倍に軽減する
  （エアロック・ノーてんきで無効化される）。
- 通常天候・おおひでり・おおあめでは上書きされず、らんきりゅうはすべての天候を上書きできる。
- 場からこの特性のポケモンがいなくなったとき（`ON_SWITCH_OUT`）・とくせいが無効化されたとき
  （`ON_ABILITY_DISABLED`）らんきりゅうを解除する。ただし相手も同じ特性を持つ場合は解除しない。

## 実装方針

- 天候の発動・解除は、あめふらし/ひでり/おわりのだいち/はじまりのうみと共通の
  `_activate_weather` / `_deactivate_strong_weather` ヘルパー（`handlers/ability.py`）を使う。
  強天候は count=1 で登録する（`ON_TURN_END` でのカウントダウン対象外にするため）。
- ひこうタイプの弱点軽減は特性側ではなく `data/field/weather.py` の `"らんきりゅう"` エントリに
  `ON_CALC_DEF_TYPE_MODIFIER` ハンドラとして実装し、天候の上書き優先度制御は `WeatherManager.apply`
  （`core/field_manager.py`）の優先度比較に委譲する。
- ノーてんき/エアロックによる無効化は、はれ/あめの威力補正ハンドラと同じパターンで
  `battle.weather_for(ctx.defender).name == ""` を先頭でチェックすることで対応する
  （天候システムが `ON_CHECK_WEATHER_ENABLED` を自動で適用してくれるのは `battle.weather.active`
  プロパティのみで、`ON_CALC_DEF_TYPE_MODIFIER` のようなフィールドハンドラは有効化中ずっと
  登録され続けるため、各ハンドラ側で `weather_for` を明示的に呼ぶ必要がある）。
- じゃくてんほけん・たつじんのおび・ブレインフォース・ナゾのみ・ふしぎなまもり等は
  `battle.query.is_super_effective` 経由で `ON_CALC_DEF_TYPE_MODIFIER` の最終値（らんきりゅう軽減後）
  を参照するため、個別対応は不要（既存実装のまま整合する）。
- きけんよち・ステルスロックのダメージ計算は型相性表を直接参照する実装のため、
  らんきりゅうの影響を受けない（既存実装のまま。個別対応不要）。
- ウェザーボールは `WEATHERBALL_TYPE_MAP` にらんきりゅうを含めないことで威力・タイプ据え置きに
  対応済み。こうごうせい等の回復量もらんきりゅうが分岐にマッチしないため通常時の1/2になる
  （個別対応不要）。

## イベント別実装計画（turn.md準拠）

1. `Event.ON_SWITCH_IN`（Priority 100, turn.md「おわりのだいち（特性）」等と同じ天候始動特性の並びに準拠。
   turn.mdの一覧自体にはデルタストリームの行が未掲載だが、他の天候始動特性と同一の扱いとする）
   - `デルタストリーム_activate_weather` → `_activate_weather(weather="らんきりゅう", count=1)`
2. `Event.ON_ABILITY_ENABLED`（とくせいなし解除・スキルスワップ等での再有効化時、既存の他天候始動特性と同じ扱い）
   - `デルタストリーム_activate_weather` を同様に接続。
3. `Event.ON_SWITCH_OUT` / `Event.ON_ABILITY_DISABLED`
   - `デルタストリーム_deactivate_strong_weather` → `_deactivate_strong_weather(weather="らんきりゅう")`。
   - 現在の天候が「らんきりゅう」かつ、相手の特性名がこの特性と異なる場合のみ解除する
     （相手も同じ特性なら解除しない＝仕様書「相手の特性が同じ天候を発生させるものなら解除しない」）。
4. `Event.ON_CALC_DEF_TYPE_MODIFIER`（turn.mdに未掲載のダメージ計算内部イベント。
   既存の同種ハンドラ〔きもったま・しんがん・テラスシェル等〕にならい優先度指定なし＝デフォルト100とする）
   - `らんきりゅう_type_modifier`（`data/field/weather.py` の `"らんきりゅう"` に登録）
   - 先頭で `battle.weather_for(ctx.defender).name == ""` をチェックし、エアロック/ノーてんき下では
     何もしない。
   - 防御側がひこうタイプを持ち、技タイプがでんき/いわ/こおりのいずれかのとき、ダメージを0.5倍する
     （`apply_fixed_modifier(value, 2048)`）。
5. 天候の上書き制御
   - `WEATHER_PRIORITY`（`data/field/weather.py`）で `らんきりゅう` は優先度2（最高）、
     おおひでり/おおあめは優先度1、通常天候は0。`WeatherManager.apply` が優先度比較で
     上書き可否を判定するため、特性側に個別の上書き禁止ロジックは不要。
6. `Event.ON_TURN_END`
   - `"らんきりゅう"` には `tick_weather` を登録しない（強天候はターン経過で自然解除されない）。

## テスト観点

1. 登場時に「らんきりゅう」が発動し、count=1で登録される。
2. ひこうタイプへのでんき/いわ/こおり技のダメージが軽減される（2倍弱点→等倍、4倍弱点→2倍）。
3. ひこうタイプ以外・でんき/いわ/こおり以外の技には影響しない。
4. 通常天候・おおひでり・おおあめでは上書きされない。らんきりゅうは全天候を上書きできる。
5. 相手も同じ特性を持つ場合、退場しても天候は解除されない。
6. 退場（交代・ひんし・とくせいなし化）でらんきりゅうが解除される。
7. エアロック・ノーてんき下では弱点軽減効果が発動しない（天候変更は引き続き失敗する）。

## 実装ステップ（実装済み）

1. `data/field/weather.py` に `らんきりゅう` を追加し、`ON_CALC_DEF_TYPE_MODIFIER` ハンドラを登録。
2. `data/ability.py` の `デルタストリーム` に `ON_SWITCH_IN` / `ON_ABILITY_ENABLED` /
   `ON_SWITCH_OUT` / `ON_ABILITY_DISABLED` ハンドラを接続。
3. `handlers/ability.py` に `デルタストリーム_activate_weather` /
   `デルタストリーム_deactivate_strong_weather` を実装（共通ヘルパー経由）。
4. `handlers/field.py` に `らんきりゅう_type_modifier` を実装
   （エアロック/ノーてんき無効化チェックを追加、本レビューで修正）。
5. `tests/abilities/test_ability__group.py` / `tests/test_field.py` に
   登場時発動・上書き可否・退場時解除・弱点軽減・エアロック無効化のテストを追加。
