# 特性実装計画: じゅうなん

更新日: 2026-07-11

## 仕様参照

- `docs/spec/abilities/じゅうなん.md`

## 実装ゴール

- まひ状態にならない（付与をブロックする）。
- 特性が有効化された時点ですでにまひ状態なら即座に回復する
  （かがくへんかガス・かたやぶりの効果が終わって特性が再び有効になった場合や、
  なやみのタネ等で特性が書き換わった場合が対象）。
- かたやぶりの効果がある技でまひ状態にされた場合は、技終了直後（特性が再び有効になった
  タイミング）で回復する。

## 実装方針

- 状態異常の付与ブロックは `Event.ON_BEFORE_APPLY_AILMENT`（`target:self`）で行う。
  めんえき・パステルベール・みずのベール・やるき等、既存の状態異常無効系特性と同じ
  `_prevent_ailment` 共通ヘルパー（`handlers/ability.py`）を `blocked_ailments=["まひ"]` で
  呼び出す `prevent_paralysis_ailment` を用いる（新規実装ではなく既存関数の再利用）。
- 既にまひ状態のときに特性が有効化されるケース（かがくへんかガス/かたやぶりの終了、
  なやみのタネによる特性書き換え等）は `Event.ON_ABILITY_ENABLED`（`source:self`）で
  `_cure_ailment_on_enable` 共通ヘルパーを `blocked_ailments=["まひ"]` で呼び出す
  `じゅうなん_cure_paralysis_on_enable` で回復させる。
- `mold_breaker_ignorable` フラグを付与し、かたやぶり・きんしのちからの効果中は
  `ON_BEFORE_APPLY_AILMENT` ハンドラ自体が無効化される（`add_ability_disabled_reason`）
  ことで、まひ状態の付与を許可する。技終了後に特性が再有効化されると
  `ON_ABILITY_ENABLED` が発火し、上記の回復ハンドラで即座に回復する。
- `ON_BEFORE_APPLY_AILMENT` / `ON_ABILITY_ENABLED` はいずれも `docs/spec/turn.md` に priority が
  明記されていないイベント（状態異常付与・特性有効化の内部処理）のため、既存の同種ハンドラ
  （めんえき・パステルベール・みずのベール・やるき等）に倣い priority は指定せず既定値を使う。
- 第六世代以降のでんきタイプによるまひ無効化は、じゅうなん固有の実装ではなくエンジン共通の
  タイプ免疫チェック（`handlers/move_attack.py` 等）で処理されるため、本特性側では対応不要。
- どくびしとの相互作用（まひ状態で場に出るとどくびしの毒が不発してからじゅうなんで回復する）は、
  どくびし側の「既に状態異常なら不発」判定とじゅうなんの回復タイミングの順序で自然に成立する
  ため、追加の実装は不要。
- 進化・とくせいカプセル/とくせいパッチによる場外（非戦闘中）でのまひ→じゅうなん取得後、
  次に場に出たときに回復する挙動は、jpoke が戦闘中の進化・場外での特性変更をサポートしない
  （単一バトルシミュレータでベンチ側ポケモンの特性変更手段が無い）ため対象外。

## テスト観点

1. まひ状態の付与を防ぐ（`test_状態異常無効`）。
2. かたやぶりの技でまひにされても技終了直後に回復する（`test_状態異常無効_かたやぶりで無効`）。
