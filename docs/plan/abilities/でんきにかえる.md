# 特性実装計画: でんきにかえる

更新日: 2026-06-05

## 仕様参照

- `docs/spec/abilities/でんきにかえる.md`

## 実装ゴール

- 攻撃技でダメージを受けたとき、じゅうでん状態になる。
- すでにじゅうでん状態なら重複しない。

## 実装方針

- `Event.ON_DAMAGE_HIT`、`subject_spec="defender:self"` で実装。
- `battle.volatile_manager.apply(mon, "じゅうでん", source=mon)` を呼ぶ。
- **ふうりょくでんき**（風技→じゅうでん）と同一の volatile 付与パターン。

## 変更候補

- `src/jpoke/data/ability.py` — `でんきにかえる` に handlers を登録する
- `src/jpoke/handlers/ability.py` — `でんきにかえる_on_damage` を実装する
- `tests/test_ability.py` — テストを追加する

## テスト観点

1. 攻撃技を受けるとじゅうでん状態になる。
2. すでにじゅうでん状態なら apply が失敗してアナウンスなし。
3. じゅうでん状態中はでんき技の威力が 2 倍になる（volatile 側の効果）。
