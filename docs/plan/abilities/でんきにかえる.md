# 特性実装計画: でんきにかえる

更新日: 2026-07-12

## 仕様参照

- `docs/spec/abilities/でんきにかえる.md`

## 実装ゴール

- 攻撃技でダメージを受けたとき、じゅうでん状態になる。
- すでにじゅうでん状態であるときも改めて発動する（アナウンスは常に出る）。
- その攻撃技でひんしになったときは発動しない（Ver.1.2.0 以降の仕様。ポケモンチャンピオンズは
  現行パッチ準拠のためこちらを採用）。
- 実HPダメージが0（こらえる・みねうち等）でも発動する。ただしみがわりに阻まれた場合は発動しない。

## 実装方針

- **`Event.ON_DAMAGE_HIT`（`subject_spec="defender:self"`）**: 実HPダメージ(>0)を受けたときの通常経路。
  - `mon.alive` が False（その攻撃でひんしになった）なら何もしない。
  - `battle.volatile_manager.apply(mon, "じゅうでん", source=mon)` を呼ぶ。
  - アナウンスは apply の戻り値（成否）に関わらず常に行う
    （すでにじゅうでん状態でも「発動する」という仕様のため。apply は既に付与済みのときFalseを返すが、
    それでも技を受けたことの発動自体は起きているとみなす）。
- **`Event.ON_HIT`（`subject_spec="defender:self"`）**: 実HPダメージが0だった場合の経路。
  - `Event.ON_DAMAGE_HIT` は実HPダメージが0のとき発火しないため、常に発火する `Event.ON_HIT`
    側で `value<=0` のケースのみを処理する（`レッドカード`アイテムと同じ idiom。
    `src/jpoke/handlers/item.py` の `レッドカード_force_switch_on_zero_damage` を参照）。
  - `ctx.substitute_damage` が真（みがわりに阻まれた）なら対象外とする
    （タイプ相性無効・まもるは `ON_BEFORE_APPLY_MOVE` 側で完全にブロックされ、
    そもそも `_execute_hit` に到達しないため `ON_HIT` は発火しない。除外判定不要）。
  - こんらんの自傷ダメージは `AttackContext`/`ON_HIT` の経路を通らないため、ガード不要。
- 優先度は `docs/spec/turn.md` に `ON_DAMAGE_HIT` / `ON_HIT`（ダメージ計算内部イベント）の掲載がないため、
  既存の同種ハンドラ（`いかりのこうら`のひんし除外パターン、`レッドカード`の0ダメージ二重登録パターン）を
  参照し、デフォルト priority（100）で登録する。
- **ふうりょくでんき**（風技→じゅうでん）とは volatile 付与の仕組みは同じだが、KO時の挙動が異なる
  （ふうりょくでんきは「風の技でHPが0になったときも、特性が発動してからひんしになる」= 発動する。
  でんきにかえるはVer.1.2.0以降「発動しない」）。したがって「同一の volatile 付与パターン」と
  一括りにはできず、でんきにかえる側にのみ `mon.alive` ガードを入れる。

## 変更候補

- `src/jpoke/data/ability.py` — `でんきにかえる` に `ON_DAMAGE_HIT` / `ON_HIT` の2つの handlers を登録する
- `src/jpoke/handlers/ability.py` — `でんきにかえる_charge_on_hit`（ON_DAMAGE_HIT側）と
  `でんきにかえる_charge_on_zero_damage`（ON_HIT側）を実装する
- `tests/abilities/test_ability_ta.py` — テストを追加する

## テスト観点

1. 攻撃技を受けるとじゅうでん状態になる。
2. その攻撃技でひんしになったときはじゅうでん状態にならない。
3. こらえる等で実ダメージが0だったときもじゅうでん状態になる。
4. みがわりに阻まれたときは発動しない（じゅうでん状態にならない）。
5. じゅうでん状態中はでんき技の威力が2倍になる（volatile側の効果。じゅうでんの仕様書を参照）。
