# 特性実装計画: はやおき

更新日: 2026-07-12

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/はやおき.md`
- 実装状態: 実装済み
- 実装方針: CLAUDE.md のハンドラ約束事（固有効果は `handlers/*` に名前付き関数で実装し、`data/*.py` から登録する）に従い、`handlers/ability.py` に `はやおき_extra_decrement` を実装し、`data/ability.py` の `ABILITIES["はやおき"]` から `Event.ON_TRY_ACTION` に登録する。`ねむり_check_action`（`handlers/ailment.py`, priority=10）の直前（priority=9）で追加の `ailment_manager.tick()` を1回実行することで、1ターンあたり合計2回消費を実現する。ハードコーディングはしない。

## 実装ゴール（完了定義）

- ねむり状態のはやおき所持者は、ターンごとにねむりカウントを 2 ずつ消費する。
- バッグ使用・フリーフォールなどカウント消費が行われないターンでは効果が発動しない
  （`ON_TRY_ACTION` は `MoveExecutor` の技実行フローでのみ発火するため、道具使用時は
  そもそもこのイベントが発火せず、自然に満たされる）。
- ねごとのサブ実行中（`mon.sleep_talk_active`）は追加消費を行わない
  （`ねむり_check_action` と同様のガードが必要。ねごと自身の `ON_TRY_ACTION` で
  既に2回消費済みのため、サブ技の `ON_TRY_ACTION` で3回目の消費をしてはならない）。
- 既存テストを壊さない。

## 実装対象イベント

`Event.ON_TRY_ACTION`（`AttackContext`, `subject_spec="attacker:self"`）。

## Priority

| イベント | Priority | 根拠 |

## 実装内容

```python
def はやおき_extra_decrement(battle: Battle, ctx: AttackContext, value: Any) -> HandlerReturn:
    """はやおき特性: ねむり状態のとき、ねむりカウンタを追加で1消費する。"""
    mon = ctx.attacker
    if mon.sleep_talk_active:
        # ねごとのサブ実行中は、選ばれた技の ON_TRY_ACTION でも本ハンドラが
        # 再度発火するが、ねむりのカウント消費は1ターンに1回のみで良いため
        # （ねごと自身の ON_TRY_ACTION 時点ですでに消費済み）、ここでは何もしない。
        return HandlerReturn(value=value)
    if not mon.has_ailment("ねむり"):
        return HandlerReturn(value=value)
    # ねむり_check_action より先 (priority=9) に追加tickを実行
    battle.ailment_manager.tick(mon)
    return HandlerReturn(value=value)
```

data/ability.py 側:

```python
"はやおき": AbilityData(
    handlers={
        Event.ON_TRY_ACTION: h.AbilityHandler(
            h.はやおき_extra_decrement,
            subject_spec="attacker:self",
            priority=9,  # ねむりカウント消費 (priority=10) の直前
        ),
    }
),
```

## テスト観点

1. ねむりカウントが N のとき、1ターンで2消費して N-2 になる（通常は N-1）。
2. ねごとを使用したターンでも合計2消費に収まる（ねごと自身の `ON_TRY_ACTION` +
   サブ技の `ON_TRY_ACTION` の2回発火があっても、3消費にならないことを確認する）。
