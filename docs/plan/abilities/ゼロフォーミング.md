# 特性実装計画: ゼロフォーミング

更新日: 2026-06-03

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/ゼロフォーミング.md`
- 実装状態: 部分実装（flags=[uncopyable] のみ設定済み）
- 実装方針: テラスタル発動時のイベントで場の天候とフィールドを消去する。発動は 1 回のみ（テラスタル時）。かがくへんかガスで無効化される。

## 実装ゴール（完了定義）

- テラパゴスがテラスタルしたとき、場の天候とフィールドを消去する。
- テラスタル後に発生した天候・フィールドは消去しない（発動は 1 回のみ）。
- ノーてんき/エアロックによる無効化中でも天候を消去できる。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_TERASTALLIZE`（または同等のイベント）— テラスタル発動時に天候とフィールドを消去する

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `ゼロフォーミング` に handler を登録する
- `src/jpoke/handlers/ability.py` — `ゼロフォーミング_on_terastallize` を実装する
- `tests/test_ability.py` — テストを追加する

## 実装ステップ

1. テラスタル処理がどこで行われているか `src/jpoke/core/` で確認し、テラスタルイベントの有無を確認する。
2. `ゼロフォーミング_on_terastallize` を実装する（天候を `None`、フィールドを `None` にクリア）。
3. `src/jpoke/data/ability.py` の `ゼロフォーミング` に handler を登録する。
4. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. テラスタル時に場の天候が消える。
2. テラスタル時に場のフィールドが消える。
3. テラスタル後に天候/フィールドを発生させることは可能。
4. テラスタル済みの状態で引っ込んで再登場しても発動しない。
