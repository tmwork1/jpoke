# 実装計画: スカイスキン

作成日: 2026-05-03
更新日: 2026-07-11

## 仕様参照

- `docs/spec/abilities/スカイスキン.md`

## 効果

ノーマルタイプの技をひこうタイプに変換し、威力を4915/4096倍にする。
タイプ変換後はひこうタイプ一致補正が適用される。
もともとひこうタイプである技の威力は上昇しない。

## イベント／ハンドラ

priority は turn.md 未掲載のイベント（ダメージ計算内部）のため、既存のスキン系特性
（フェアリースキン・ドラゴンスキン・フリーズスキン）のハンドラと同じ既定値（100）に揃える。

| イベント | priority | subject_spec | 関数名 |
|---------|---------|-------------|--------|
| `ON_MODIFY_MOVE_TYPE` | 100 (既定) | `attacker:self` | `スカイスキン_modify_move_type` |
| `ON_CALC_POWER_MODIFIER` | 100 (既定) | `attacker:self` | `スカイスキン_modify_power` |

## 実装方針

- フェアリースキン・ドラゴンスキン・フリーズスキンと完全に同一構造。共通ヘルパー
  `_skin_modify_move_type` / `_skin_boost_power`（`handlers/ability.py`）に
  変換先タイプ／トリガータイプを渡すだけで実装する。

```python
def スカイスキン_modify_move_type(battle, ctx, value):
    return _skin_modify_move_type(battle, ctx, value, from_type="ノーマル", to_type="ひこう")

def スカイスキン_modify_power(battle, ctx, value):
    return _skin_boost_power(battle, ctx, value, trigger_type="ノーマル")
```

- `_skin_boost_power` は `ctx.move.data.type`（技本来の静的タイプ）が `ノーマル` の場合のみ威力を
  4915/4096倍にする。ウェザーボール等タイプが変わる技は、その技自身の `ON_MODIFY_MOVE_TYPE`
  ハンドラが本特性のハンドラより後に登録される（技側ハンドラは技使用時に都度登録されるため）ため、
  最終的な技タイプは本特性による変換を技側の変換が上書きする形になる。この優先順位は
  既存のフェアリースキン実装・テスト（`test_テラバースト_テラスタイプがノーマルでもスキン特性の対象にならない`）
  で検証済みのパターンを踏襲する。
- わるあがきはタイプなし判定のため、わるあがき実装側で除外し、特性側で個別分岐は持たない。

- `src/jpoke/data/ability.py`

## テスト観点

1. ノーマル技をひこうタイプに変換する（`SKIN_CASES` に含めて汎用テストでカバー）。
2. 威力が4915/4096倍になる（同上）。
3. ひこうタイプ一致補正が適用される（ダメージ計算経由で確認）。
4. もともとひこうタイプの技は威力補正を受けない。
5. わるあがきはわるあがき実装側の判定で変換対象外になる。
