# 特性実装計画: きもったま

更新日: 2026-06-03

## スコープ

- 優先度: H
- 仕様書: `docs/spec/abilities/きもったま.md`
- 実装状態: 未実装
- 実装方針: `ON_CALC_DEF_TYPE_MODIFIER` でノーマル/かくとう技がゴーストタイプに当たるよう補正し、`ON_BEFORE_MODIFY_STAT` でいかくによるこうげきランク低下を防ぐ。

## 実装ゴール（完了定義）

- ノーマル・かくとうタイプの技がゴーストタイプに等倍（×1）として当たる。
- ゴースト+はがね複合への等倍・抜群の計算が正しく行われる。
- いかくによるこうげきのランク低下を防ぐ。
- 既存テストを壊さない。

## 実装対象イベント

- `Event.ON_CALC_DEF_TYPE_MODIFIER`（または同等のイベント）— タイプ相性計算時にノーマル/かくとう vs ゴーストの無効を等倍に変更する
- `Event.ON_BEFORE_MODIFY_STAT` — いかく由来のこうげきランク低下を阻止する

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `きもったま` に handlers を登録する
- `src/jpoke/handlers/ability.py` — `きもったま_ghost_immune_bypass`, `きもったま_block_intimidate` を実装する
- `tests/test_ability.py` — テストを追加する

## 実装ステップ

1. タイプ相性計算のイベント名と引数を `src/jpoke/enums/event.py` / `src/jpoke/core/` で確認する。
2. `きもったま_ghost_immune_bypass` を実装する（技がノーマルまたはかくとう かつ 相手がゴーストタイプのとき、無効補正を等倍に書き換えて返す）。
3. `きもったま_block_intimidate` を実装する（`するどいめ_block_ACC_drop` を参考にいかく由来のこうげきドロップを阻止する）。
4. `src/jpoke/data/ability.py` の `きもったま` に handlers を登録する。
5. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. きもったま所持者がゴーストタイプにノーマル技を使うと等倍ダメージが出る。
2. きもったま所持者がゴースト+はがねにかくとう技を使うと抜群ダメージが出る。
3. いかくを持つ相手が登場してもこうげきが下がらない。
