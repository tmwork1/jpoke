## スコープ

- 優先度: H

- 実装状態: 未実装

## 実装ゴール（完了定義）

- 仕様書に列挙された回復技・吸収技を使うと優先度が +3 になる。
- アクアリング・やどりぎのタネ等の「状態変化技」は対象外。
- 既存テストを壊さない。

## 実装対象イベント

- `DomainEvent.ON_MODIFY_MOVE_PRIORITY` — 優先度計算時に対象回復技なら +3 する

## 変更対象（ファイル/関数単位）

- `src/jpoke/data/ability.py` — `ヒーリングシフト` に handler を登録する
- `src/jpoke/handlers/ability.py` — `ヒーリングシフト_modify_priority` を実装する
- `tests/test_ability.py` — テストを追加する

## 実装ステップ

1. 仕様書 `docs/spec/abilities/ヒーリングシフト.md` の対象技リストを確認し、`HEALING_SHIFT_MOVES` 等の定数として定義する場所を決める（`src/jpoke/utils/constants.py` または handlers 内）。
2. `src/jpoke/handlers/ability.py` に `ヒーリングシフト_modify_priority` を実装する（技名が対象リストに含まれるとき value + 3 を返す）。
3. `src/jpoke/data/ability.py` の `ヒーリングシフト` に handler を登録する。
4. テストを追加し、全テストが通ることを確認する。

## テスト観点

1. じこさいせい・ドレインパンチなど対象回復技の優先度が +3 になる。
2. やどりぎのタネ・アクアリングなど除外技では優先度が上がらない。
3. ヒーリングシフト対象外の攻撃技は優先度が上がらない。
