# 特性実装計画: よわき

更新日: 2026-07-14

## 仕様参照

- `docs/spec/abilities/よわき.md`

## 実装ゴール

- 自分のHPが最大HPの半分以下のとき、ダメージ計算に用いる「こうげき」「とくこう」の実数値補正を
  0.5倍にする。
  - 判定は `_calc_final_attack` 内、ランク補正適用後の固定小数点補正（`ON_CALC_ATK_MODIFIER`）
    で行われるため、物理・特殊どちらの技でも同じハンドラで対応できる。
  - ボディプレス使用時: 分類は物理技のまま「自分の防御実数値」を代わりに参照するが、参照元が
    どちらであっても攻撃側（技を使用する側）によわきがあり、かつHP条件を満たせば0.5倍になる。
  - イカサマ使用時: 分類は物理技のまま「相手の攻撃実数値」を代わりに参照するが、よわきを持つ側が
    攻撃側（イカサマの使用者）であればHP条件で0.5倍になる。イカサマを受ける側によわきがあっても
    影響しない（`subject_spec="attacker:self"` のため自動的に満たされる）。
  - こんらんの自傷ダメージ（内部技 `"_こんらん"`）には効果が無い。
  - ダメージ固定技（威力0の技）は `calc_damages` が `_calc_final_attack` 自体を呼ばないため、
    `ON_CALC_ATK_MODIFIER` が発火せず自動的に影響を受けない。専用の分岐は不要。
  - みがわり状態でも本体のHPで判定する（`ctx.attacker.hp` は本体のHPであり、みがわりの残りHPとは
    独立しているため専用分岐は不要）。
  - HPが半分超に回復すれば毎回のダメージ計算時にHPを再判定するため、自動的に補正が外れる
    （状態を保持しないため専用の解除処理は不要）。

## 実装方針

- `Event.ON_CALC_ATK_MODIFIER`、`subject_spec="attacker:self"` で実装する（実装済み）。
- priority は turn.md に記載のない内部（ダメージ計算内）イベントのため、同一イベント・同一
  `subject_spec="attacker:self"` の既存ハンドラ（**ちからもち**の `ちからもち_boost_physical` など）
  に合わせ、`AbilityHandler` のデフォルト値 `priority=100` を使用する。
- `ctx.attacker.hp * 2 <= ctx.attacker.max_hp` のとき `apply_fixed_modifier(value, 2048)`
  （4096基準で0.5倍）を適用する。
- こんらんの自傷ダメージ（`"_こんらん"`）は物理技（`category="physical"`）として扱われるため、
  コードベース全体で徹底されている除外パターン（`ctx.move.name != "_こんらん"`。例:
  `こんじょう_modify_atk`、`ちからもち_boost_physical` など）に合わせ、条件に追加する
  （本レビューで修正）。

## 変更候補

- `src/jpoke/data/ability.py` — `よわき` に `Event.ON_CALC_ATK_MODIFIER` ハンドラを登録する
  （実装済み）。
- `src/jpoke/handlers/ability.py` — `よわき_modify_atk` に `"_こんらん"` 除外を追加する
  （本レビューで修正）。

## テスト観点

1. HPが最大HPの半分以下のとき、`atk_modifier` が2048（0.5倍）になる（物理・特殊技どちらも）。
2. HPが最大HPの半分超のときは `atk_modifier` が4096のまま変化しない。
3. こんらんの自傷ダメージには補正がかからない（`atk_modifier` が4096のまま）。
